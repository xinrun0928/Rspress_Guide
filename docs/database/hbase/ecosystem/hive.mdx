# Hive/Hadoop 集成

在大数据生态中，Hive 是离线分析的主力工具。当我们需要对 HBase 中的海量数据进行复杂分析时，Hive 与 HBase 的集成就派上了用场。

这篇文章会讲解如何让 Hive 查询 HBase 数据，以及在实际场景中的应用。

## 为什么需要 Hive 集成？

原生 HBase 的查询能力有限：
- 没有 SQL 支持
- 不支持 JOIN、GROUP BY 等复杂操作
- 聚合计算需要 MapReduce

Hive 的能力：
- 完整的 SQL 支持（HiveQL）
- 强大的聚合和 JOIN 能力
- 与 MapReduce、Spark、Tez 计算引擎集成

Hive 与 HBase 集成后，可以：
- 用 SQL 分析 HBase 中的数据
- 将 HBase 作为 Hive 的存储后端
- 实现 ETL 流程中的数据流转

## 集成方式

### 方式一：HBase 外部表

Hive 可以创建指向 HBase 表的外部表：

```sql
-- 在 Hive 中创建指向 HBase 表的外部表
CREATE EXTERNAL TABLE hbase_users (
    user_id STRING,
    name STRING,
    age INT,
    email STRING
)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES (
    "hbase.columns.mapping" = 
        ":key,info:name,info:age,info:email"
)
TBLPROPERTIES (
    "hbase.table.name" = "users"
);
```

**参数说明**：
- `:key` 表示 HBase 的 RowKey
- `info:name` 表示 Column Family:Qualifier
- `hbase.table.name` 指定 HBase 表名

### 方式二：映射已有 HBase 表

```sql
-- 直接映射已有 HBase 表（不需要 TBLPROPERTIES）
CREATE EXTERNAL TABLE hbase_orders (
    order_id STRING,
    user_id STRING,
    amount DECIMAL(10,2),
    status STRING
)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES (
    "hbase.columns.mapping" = 
        ":key,order:user_id,order:amount,order:status"
);
```

### 方式三：Hive 管理 HBase 表

```sql
-- Hive 创建并管理 HBase 表
CREATE TABLE hive_managed_table (
    id STRING,
    data STRING,
    created_at TIMESTAMP
)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES (
    "hbase.columns.mapping" = ":key,info:data,info:created_at"
)
TBLPROPERTIES (
    "hbase.table.name" = "hive_managed",
    "hbase.table.storage" = "true"  -- Hive 管理数据
);
```

## 数据类型映射

Hive 与 HBase 之间的数据类型映射：

| HBase 类型 | Hive 类型 |
| --- | --- |
| RowKey (:key) | STRING |
| Integer | INT / INTEGER |
| Long | BIGINT / LONG |
| Float | FLOAT |
| Double | DOUBLE |
| Boolean | BOOLEAN |
| Binary | BINARY / TINYINT ARRAY |
| Bytes | STRING (Base64 编码) |

## 复杂数据类型

### 多列族

```sql
CREATE EXTERNAL TABLE multi_cf_table (
    row_key STRING,
    -- profile 列族
    profile_name STRING,
    profile_age INT,
    -- behavior 列族
    behavior_view_count BIGINT,
    behavior_buy_count BIGINT
)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES (
    "hbase.columns.mapping" = 
        ":key,

         profile:age,
         behavior:view_count,
         behavior:buy_count"

TBLPROPERTIES ("hbase.table.name" = "multi_cf");
```

### Map 类型

HBase 的动态列可以映射为 Hive 的 MAP 类型：

```sql
CREATE EXTERNAL TABLE hbase_map_table (
    row_key STRING,

)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES (
    "hbase.columns.mapping" = ":key,attr:"
)
TBLPROPERTIES ("hbase.table.name" = "map_table");
```

### 数组类型

## 查询示例

### 基本查询

```sql
-- 查询所有数据
SELECT * FROM hbase_users;

-- 条件查询
SELECT * FROM hbase_users WHERE age > 25;

-- 聚合查询
SELECT 
    COUNT(*) as total_users,
    AVG(age) as avg_age
FROM hbase_users;
```

### JOIN 操作

```sql
-- JOIN HBase 表和普通 Hive 表
SELECT 
    u.user_id,
    u.name,
    o.order_count,
    o.total_amount
FROM hbase_users u
LEFT JOIN (
    SELECT 
        user_id,
        COUNT(*) as order_count,
        SUM(amount) as total_amount
    FROM hbase_orders
    GROUP BY user_id
) o ON u.user_id = o.user_id;
```

### 复杂分析

```sql
-- 分析用户行为
SELECT 
    CASE 
        WHEN age < 20 THEN '18-20'
        WHEN age < 30 THEN '20-30'
        WHEN age < 40 THEN '30-40'
        ELSE '40+'
    END as age_group,
    COUNT(*) as user_count,
    AVG(purchase_count) as avg_purchases
FROM (
    SELECT 
        u.user_id,
        u.age,
        COUNT(o.order_id) as purchase_count
    FROM hbase_users u
    LEFT JOIN hbase_orders o ON u.user_id = o.user_id
    GROUP BY u.user_id, u.age
) t
GROUP BY 
    CASE 
        WHEN age < 20 THEN '18-20'
        WHEN age < 30 THEN '20-30'
        WHEN age < 40 THEN '30-40'
        ELSE '40+'
    END;
```

### 使用 MapReduce 写入

```java
// 使用 MapReduce 批量写入 HBase
public class HBaseBulkLoad {
    
    public static class Mapper extends Mapper<LongWritable, Text, ImmutableBytesWritable, Put> {
        
        @Override
        protected void map(LongWritable key, Text value, Context context) 
            throws IOException, InterruptedException {
            
            String[] fields = value.toString().split(",");
            
            // 创建 Put
            Put put = new Put(Bytes.toBytes(fields[0]));
            put.addColumn(
                Bytes.toBytes("info"),
                Bytes.toBytes("name"),
                Bytes.toBytes(fields[1])
            );
            put.addColumn(
                Bytes.toBytes("info"),
                Bytes.toBytes("age"),
                Bytes.toBytes(fields[2])
            );
            
            context.write(new ImmutableBytesWritable(Bytes.toBytes(fields[0])), put);
        }
    }
    
    public static void main(String[] args) throws Exception {
        Job job = Job.getInstance();
        job.setJobName("HBase Bulk Load");
        
        HFileOutputFormat2.configureIncrementalLoad(
            job,
            new HTable(connection, "users"),
            connection.getRegionLocator(TableName.valueOf("users"))
        );
        
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}
```

## Sqoop 集成

Sqoop 可以将关系型数据库的数据导入 HBase：

```bash
# 导入 MySQL 数据到 HBase
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --hbase-table users \
  --column-family info \
  --hbase-row-key user_id \
  --class-name User \
  --jar-file user.jar \
  --增量导入（可选）
  --check-column id \
  --incremental append \
  --last-value 0
```

## 性能优化

### 1. 预分区

创建 HBase 表时指定分裂点：

```sql
CREATE TABLE partitioned_users (
    user_id STRING,
    name STRING,
    age INT
)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES (
    "hbase.columns.mapping" = ":key,info:name,info:age",
    "hbase.table.name" = "partitioned_users"
)
TBLPROPERTIES (
    "hbase.table.split.keys" = "g,m,t,z"
);
```

### 2. 使用 Tez 引擎

```sql
SET hive.execution.engine=tez;
SELECT * FROM hbase_users WHERE age > 25;
```

### 3. 并行执行

```sql
SET hive.vectorized.execution.enabled=true;
SET hive.vectorized.execution.reduce.enabled=true;
```

## 应用场景

### 场景1：用户画像分析

```sql
-- 合并用户基本信息（存储在 HBase）和行为数据（存储在 Hive）
SELECT 
    u.user_id,
    u.name,
    u.age,
    b.total_purchases,
    b.total_amount,
    b.avg_rating
FROM hbase_users u
LEFT JOIN (
    SELECT 
        user_id,
        SUM(purchase_count) as total_purchases,
        SUM(amount) as total_amount,
        AVG(rating) as avg_rating
    FROM hive_user_behaviors
    GROUP BY user_id
) b ON u.user_id = b.user_id;
```

### 场景3：冷热数据分离

```sql
-- 热数据存储在 MySQL（实时查询）
-- 温数据存储在 HBase（近历史）
-- 冷数据存储在 Hive/HDFS（归档分析）

-- 查询时自动路由

    SELECT * FROM mysql_hot_data WHERE date >= '2024-01-01'
    UNION ALL
    SELECT * FROM hbase_warm_data WHERE date BETWEEN '2023-06-01' AND '2023-12-31'
    UNION ALL
    SELECT * FROM hive_cold_data WHERE date < '2023-06-01'

WHERE user_id = 'user_001';
```

## 注意事项

### 1. 版本兼容性

| 组件 | 版本要求 |
| --- | --- |
| Hive | 1.x+ |
| HBase | 0.98+ |
| ZooKeeper | 3.4+ |

### 2. 列映射限制

- Column Family 必须与 HBase 表中的定义一致
- 列名不能包含特殊字符
- RowKey 映射为 `:key`

### 3. 性能考虑

- Hive 查询 HBase 有额外开销
- 不适合实时查询场景
- 大量小查询性能较差
