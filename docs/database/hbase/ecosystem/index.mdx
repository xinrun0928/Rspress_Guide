# HBase 生态总览

HBase 不是孤立存在的，它是 Hadoop 生态的核心组件之一。与其他组件的集成，让 HBase 能够处理更复杂的场景，发挥更大的价值。

这篇文章会带你了解 HBase 的生态系统，以及如何与其他组件配合使用。

## HBase 与其他组件的关系

### HBase + HDFS

- HBase 的数据最终存储在 HDFS 上
- HBase 利用 HDFS 的副本机制保证数据可靠性
- HBase 的 WAL 直接写入 HDFS

### HBase + Spark

- Spark 可以直接读写 HBase 数据
- 适合大规模数据分析
- 支持 DataFrame 和 RDD 两种 API

### HBase + Hive

- Hive 可以查询 HBase 表
- 可以创建 HBase 外部表
- 适合离线分析场景

### HBase + Phoenix

- Phoenix 提供 SQL 接口访问 HBase
- 支持二级索引
- 支持事务

### HBase + Sqoop/Flume

- Sqoop 用于关系型数据库与 HBase 之间的数据导入导出
- Flume 用于日志收集并写入 HBase
