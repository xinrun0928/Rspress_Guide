# Phoenix SQL 层

原生 HBase 只支持 Key-Value API，不支持 SQL。对于习惯了关系型数据库的开发者来说，这是一个不小的学习成本。

Apache Phoenix 的出现解决了这个问题——它为 HBase 提供了完整的 SQL 支持，包括 DDL、DML、函数、索引等。

## Phoenix 是什么？

Phoenix 是构建在 HBase 之上的 SQL 层，它将 SQL 查询编译成 HBase 的原生 API 调用。

```
┌─────────────────────────────────────────────────────────────┐
│                     Phoenix 架构                              │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   Phoenix SQL                         │   │
│  │  ├─ Parser (SQL 解析)                              │   │
│  │  ├─ Optimizer (查询优化)                           │   │
│  │  └─ Executor (执行计划)                             │   │
│  └─────────────────────────────────────────────────────┘   │
│                            │                                │
│                            ▼                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  Phoenix 协处理器                    │   │
│  │  ├─ QueryServer (客户端)                           │   │
│  │  └─ Coprocessor (服务端)                           │   │
│  └─────────────────────────────────────────────────────┘   │
│                            │                                │
│                            ▼                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                      HBase                           │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 安装与配置

### Phoenix 与 HBase 版本对应

| Phoenix 版本 | HBase 版本 |
| --- | --- |
| Phoenix 5.x | HBase 2.0+ |
| Phoenix 4.x | HBase 1.x |

### 部署 Phoenix

```bash
# 1. 下载 Phoenix
wget https://archive.apache.org/dist/phoenix/phoenix-5.1.0/bin/apache-phoenix-5.1.0-bin.tar.gz

# 2. 解压
tar -xzf apache-phoenix-5.1.0-bin.tar.gz

# 3. 复制到 HBase lib 目录
cp phoenix-5.1.0-server.jar $HBASE_HOME/lib/

# 4. 重启 HBase

# 5. 验证安装
$HBASE_HOME/bin/hbase pe
```

### Phoenix QueryServer（可选）

Phoenix 支持通过 JDBC 访问：

```bash
# 启动 QueryServer
phoenix-queryserver start

# QueryServer 默认端口：8765
```

## 基本 SQL 操作

### 创建表

```sql
-- 创建表（默认使用单个列族 '0'）
CREATE TABLE users (
    user_id VARCHAR PRIMARY KEY,
    name VARCHAR,
    age INTEGER,
    email VARCHAR,
    created_at TIMESTAMP
);

-- 创建表并指定列族
CREATE TABLE orders (
    order_id VARCHAR PRIMARY KEY,
    user_id VARCHAR,
    amount DECIMAL(10, 2),
    status VARCHAR,
    created_at TIMESTAMP
) COLUMN_ENCODED_BYTES = 0;

-- 创建带有预分区的表
CREATE TABLE large_table (
    id VARCHAR PRIMARY KEY,
    data VARCHAR
) SPLITS ON ('5000000', '10000000', '15000000');
```

### 插入数据

```sql
-- 单行插入
INSERT INTO users (user_id, name, age, email)
VALUES ('user_001', '张三', 28, 'zhang@example.com');

-- 批量插入
UPSERT INTO users (user_id, name, age)
VALUES ('user_002', '李四', 32),
       ('user_003', '王五', 25),
       ('user_004', '赵六', 40);

-- 更新（使用 UPSERT，如果存在则更新，不存在则插入）
UPSERT INTO users (user_id, name, age)
VALUES ('user_001', '张三丰', 88);
```

### 查询数据

```sql
-- 基本查询
SELECT * FROM users WHERE user_id = 'user_001';

-- 条件查询
SELECT name, age FROM users WHERE age > 25;

-- 分页查询
SELECT * FROM users ORDER BY created_at LIMIT 10 OFFSET 20;

-- 聚合查询
SELECT 
    COUNT(*) as total_users,
    AVG(age) as avg_age,
    MAX(age) as max_age
FROM users;

-- GROUP BY
SELECT 
    status,
    COUNT(*) as order_count,
    SUM(amount) as total_amount
FROM orders
GROUP BY status;
```

### 删除数据

```sql
-- 删除行
DELETE FROM users WHERE user_id = 'user_001';

-- 删除满足条件的行
DELETE FROM orders WHERE status = 'cancelled' AND created_at < '2024-01-01';
```

## 核心特性

### 1. 二级索引

Phoenix 支持创建二级索引，显著提升非主键查询的性能：

```sql
-- 创建全局索引（默认）
CREATE INDEX idx_user_age ON users(age);

-- 创建本地索引
CREATE LOCAL INDEX idx_user_email ON users(email);

-- 创建覆盖索引（包含额外列）
CREATE INDEX idx_user_status ON orders(status)
INCLUDE (amount, created_at);

-- 索引命名
CREATE INDEX idx_users_age_name ON users(age) INCLUDE (name);

-- 查看索引
SELECT * FROM SYSTEM.CATALOG WHERE TABLE_NAME = 'idx_user_age';
```

### 2. 视图

Phoenix 支持创建视图，将 HBase 表映射为视图：

```sql
-- 创建视图
CREATE VIEW my_view (
    pk VARCHAR PRIMARY KEY,
    col1 VARCHAR,
    col2 INTEGER
) AS SELECT * FROM "my_hbase_table" WHERE ...;

-- 删除视图
DROP VIEW my_view;
```

### 3. 函数支持

```sql
-- 字符串函数
SELECT UPPER(name), LOWER(email) FROM users;
SELECT SUBSTR(name, 1, 2) FROM users;
SELECT LENGTH(name) FROM users;

-- 数学函数
SELECT ROUND(amount, 2) FROM orders;
SELECT ABS(-10);
SELECT FLOOR(amount), CEIL(amount) FROM orders;

-- 日期函数
SELECT CURRENT_DATE(), CURRENT_TIMESTAMP();
SELECT DATEADD(created_at, 7) FROM orders;
SELECT DATEDIFF(created_at, CURRENT_DATE()) FROM orders;

-- 条件函数
SELECT 
    CASE 
        WHEN age < 18 THEN '未成年'
        WHEN age < 30 THEN '青年'
        WHEN age < 50 THEN '中年'
        ELSE '老年'
    END as age_group
FROM users;

-- DECODE 函数
SELECT DECODE(status, 'pending', '待处理', 'completed', '已完成', '其他')
FROM orders;
```

### 4. 事务支持

Phoenix 4.7+ 支持事务：

```sql
-- 开启事务（表级别）
CREATE TABLE transactional_table (
    id VARCHAR PRIMARY KEY,
    data VARCHAR
) TRANSACTIONAL = true;

-- 使用事务
UPSERT INTO transactional_table VALUES ('1', 'data1');
UPSERT INTO transactional_table VALUES ('2', 'data2');
COMMIT;
```

## JDBC 访问

### Maven 依赖

```xml
<dependency>
    <groupId>org.apache.phoenix</groupId>
    <artifactId>phoenix-queryserver-client</artifactId>
    <version>5.1.0</version>
</dependency>
```

### JDBC 连接

```java
// 方式1：通过 QueryServer
String url = "jdbc:phoenix:thin:url=http://localhost:8765";
Connection conn = DriverManager.getConnection(url);

// 方式2：直连 HBase（需要 Phoenix server jar 在 classpath 中）
String url = "jdbc:phoenix:localhost:2181:/hbase";
Connection conn = DriverManager.getConnection(url);

// 执行查询
try (Statement stmt = conn.createStatement()) {
    ResultSet rs = stmt.executeQuery("SELECT * FROM users");
    while (rs.next()) {
        System.out.println(rs.getString("user_id") + ": " + rs.getString("name"));
    }
}
```

### 使用 PreparedStatement

```java
String sql = "SELECT * FROM users WHERE age > ? AND status = ?";

try (PreparedStatement ps = conn.prepareStatement(sql)) {
    ps.setInt(1, 25);
    ps.setString(2, "active");
    
    try (ResultSet rs = ps.executeQuery()) {
        while (rs.next()) {
            System.out.println(rs.getString("name"));
        }
    }
}
```

### 批量操作

```java
String sql = "UPSERT INTO users (user_id, name, age) VALUES (?, ?, ?)";

try (PreparedStatement ps = conn.prepareStatement(sql)) {
    conn.setAutoCommit(false);
    
    for (User user : users) {
        ps.setString(1, user.getId());
        ps.setString(2, user.getName());
        ps.setInt(3, user.getAge());
        ps.addBatch();
    }
    
    ps.executeBatch();
    conn.commit();
}
```

## 性能调优

### 1. 盐值表

对于写入密集型表，使用盐值分散数据：

```sql
-- 创建盐值表（4 个 Region）
CREATE TABLE salted_table (
    id VARCHAR PRIMARY KEY,
    data VARCHAR
) SALT_BUCKETS = 4;

-- 注意：盐值表的 RowKey 会自动添加前缀
```

### 2. 跳序表

对于时序数据，使用跳序表优化范围查询：

```sql
-- 创建跳序表
CREATE TABLE time_series (
    metric_id VARCHAR,
    ts TIMESTAMP,
    value DOUBLE,
    CONSTRAINT pk PRIMARY KEY (metric_id, ts)
) IMMUTABLE_ROWS = true;

-- 查询特定时间范围
SELECT * FROM time_series 
WHERE metric_id = 'cpu_usage' 
  AND ts >= '2024-01-01' 
  AND ts < '2024-01-02';
```

### 3. 列编码

```sql
-- 禁用列编码（适用于有大量字符串列的表）
CREATE TABLE my_table (
    id VARCHAR PRIMARY KEY,
    data VARCHAR
) COLUMN_ENCODED_BYTES = 0;
```

### 4. 连接池配置

```java
// 使用 Apache DBCP2 连接池
BasicDataSource dataSource = new BasicDataSource();
dataSource.setUrl("jdbc:phoenix:localhost:2181:/hbase");
dataSource.setInitialSize(5);
dataSource.setMaxTotal(20);
dataSource.setMaxIdle(10);
dataSource.setMinIdle(5);
dataSource.setValidationQuery("SELECT 1");
dataSource.setTestOnBorrow(true);
```

## Phoenix 与原生 API 的对比

| 特性 | Phoenix | 原生 HBase API |
| --- | --- | --- |
| SQL 支持 | 完整 SQL | 无 |
| 二级索引 | 内置支持 | 需要协处理器 |
| 事务 | 支持 | 不支持 |
| 学习曲线 | 低 | 高 |
| 性能 | 略低（有额外开销） | 高 |
| 灵活性 | 有限制 | 高 |
