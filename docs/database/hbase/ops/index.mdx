# HBase 运维总览

HBase 集群的运维是一项复杂的工作，需要掌握监控、调优、故障排查等多种技能。与传统数据库相比，HBase 的运维有其独特性——它依赖于 ZooKeeper 和 HDFS，组件之间的关系更加复杂。

这篇文章会带你了解 HBase 运维的核心要点，为后续的专题深入打下基础。

## 运维架构概览

HBase 的运维涉及多个层面：

```
┌─────────────────────────────────────────────────────────────────┐
│                        HBase 运维架构                            │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    监控层                                 │   │
│  │  ├─ Metrics (HBase 内置)                               │   │
│  │  ├─ Ganglia / Prometheus                               │   │
│  │  ├─ Grafana (可视化)                                   │   │
│  │  └─ Ambari / Cloudera Manager                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│  │                                                             │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    HBase 集群层                          │   │
│  │  ├─ HMaster (管理节点)                                  │   │
│  │  ├─ RegionServer (数据节点)                             │   │
│  │  ├─ ZooKeeper (协调服务)                                │   │
│  │  └─ HDFS (底层存储)                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│  │                                                             │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    运维工具层                             │   │
│  │  ├─ HBase Shell                                         │   │
│  │  ├─ HBCK (修复工具)                                     │   │
│  │  ├─ WALPlayer / HFileUtil                              │   │
│  │  └─ 自动化脚本                                          │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## 核心监控指标

### RegionServer 指标

| 指标 | 说明 | 告警阈值 |
| --- | --- | --- |
| `regionCount` | Region 数量 | `>` 200 需关注 |
| `storeCount` | Store 数量 | `>` 500 需关注 |
| `storeFileCount` | HFile 数量 | `>` 30 需关注 |
| `memStoreSize` | MemStore 大小 | `>` 128MB 需关注 |
| `blockCacheHitCount` | BlockCache 命中率 | `<` 0.8 需关注 |
| `readRequestCount` | 读请求数 | 异常波动 |
| `writeRequestCount` | 写请求数 | 异常波动 |
| `exceptionCount` | 异常数量 | `>` 0 需关注 |

### HMaster 指标

| 指标 | 说明 | 告警阈值 |
| --- | --- | --- |
| `masterStartTime` | 启动时间 | - |
| `masterActiveTime` | 激活时间 | - |
| `serverStartTime` | 服务器启动时间 | - |
| `serverCrashCount` | 服务器崩溃次数 | `>` 0 需关注 |

### HDFS 指标

| 指标 | 说明 | 告警阈值 |
| --- | --- | --- |
| `hdfsLinks` | HDFS 链接数 | - |
| `hdfsFileHandleCount` | 文件句柄数 | `>` 10000 需关注 |

## 常用运维命令

### HBase Shell 命令

```bash
# 连接 HBase Shell
$HBASE_HOME/bin/hbase shell

# 创建表
create 'users', 'info', 'order'

# 列出所有表
list

# 查看表结构
describe 'users'

# 查看 Region 信息
list_regions 'users'

# 查看 Region 分布
status 'detailed'

# 手动触发 Compaction
major_compact 'users'

# 手动触发 Flush
flush 'users'

# 手动触发 Split
split 'region_name', 'split_key'

# 禁用表
disable 'users'

# 删除表
drop 'users'
```

### HBCK 命令

```bash
# HBCK 检查集群状态
hbase hbck

# HBCK 修复 Region 分配
hbase hbck -fix

# HBCK 修复元数据
hbase hbck -fixMeta

# HBCK 修复 HDFS
hbase hbck -fixHdfs
```

## 常见运维任务

### 1. 表管理

```java
// Java API 管理表
Admin admin = connection.getAdmin();

// 创建表
TableDescriptor table = TableDescriptorBuilder
    .newBuilder(TableName.valueOf("users"))
    .setColumnFamily(ColumnFamilyDescriptorBuilder.of("info"))
    .build();
admin.createTable(table);

// 修改表
admin.disableTable(TableName.valueOf("users"));
admin.modifyTable(tableDescriptor);
admin.enableTable(TableName.valueOf("users"));

// 删除表
admin.disableTable(TableName.valueOf("users"));
admin.deleteTable(TableName.valueOf("users"));
```

### 2. 集群扩容

```bash
# 新增 RegionServer 后，HMaster 会自动进行负载均衡
# 也可以手动触发
hbase shell
balance_switch true
balance
```

### 3. 数据备份

```bash
# 创建快照
snapshot 'users', 'users_snapshot_20240101'

# 查看快照
list_snapshots

# 恢复快照
restore_snapshot 'users_snapshot_20240101'

# 导出快照到其他集群
ExportSnapshot -snapshot users_snapshot_20240101 \
  -copy-to hdfs://target-cluster/hbase
```

### 4. 版本升级

```bash
# 滚动升级步骤：
# 1. 停止 HBase 写入
# 2. 创建快照备份
# 3. 升级 HMaster
# 4. 滚动升级 RegionServer
# 5. 验证功能
```

## 常见问题场景

| 场景 | 症状 | 解决方案 |
| --- | --- | --- |
| 热点问题 | 特定 Region 负载高 | 预分区 + RowKey 优化 |
| MemStore 刷写频繁 | 写入延迟高 | 调整刷写阈值 |
| Compaction 占用资源 | 读取延迟高 | 调整 Compaction 策略 |
| RegionServer 宕机 | 集群不稳定 | HMaster 自动恢复 |
| ZooKeeper 问题 | 集群不可用 | 检查 ZooKeeper 状态 |
