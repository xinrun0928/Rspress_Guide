# 快照与备份

数据是无价的。在生产环境中，数据丢失可能意味着灾难性的后果。HBase 提供了快照（Snapshot）机制，可以在不影响服务的情况下备份和恢复数据。

快照是 HBase 数据安全的基石。理解快照机制，对于运维 HBase 集群至关重要。

## 快照的基本概念

### 快照是什么？

HBase 的快照不是复制数据，而是一组元数据，记录了某个时间点的 HFile 列表。真正的数据文件不会被复制，仍然在原位置。

```
快照原理：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  无快照时：                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  HFile-1  HFile-2  HFile-3                            │   │
│  │     │        │        │                               │   │
│  │     ▼        ▼        ▼                               │   │
│  │  [数据文件] [数据文件] [数据文件]                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  有快照时：                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  HFile-1  HFile-2  HFile-3                            │   │
│  │     │        │        │                               │   │
│  │     ▼        ▼        ▼                               │   │
│  │  [数据文件] [数据文件] [数据文件]                       │   │
│  │                                                      │   │
│  │  ┌───────────────────────────────────────────────┐  │   │
│  │  │  Snapshot "snap_20240101":                    │  │   │
│  │  │  - HFile-1 (创建快照时的状态)                  │  │   │
│  │  │  - HFile-2 (创建快照时的状态)                  │  │   │
│  │  │  - HFile-3 (创建快照时的状态)                  │  │   │
│  │  └───────────────────────────────────────────────┘  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  快照后写入新数据：                                          │
│  - 新 HFile-4 被创建                                        │
│  - 快照不包含 HFile-4（新数据不影响快照）                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 快照 vs 全量复制

| 方式 | 原理 | 创建时间 | 空间占用 | 数据一致性 |
| --- | --- | --- | --- | --- |
| 快照 | 记录 HFile 列表 | 秒级 | 几乎为 0 | 一致 |
| 全量复制 | 复制所有 HFile | 分钟~小时 | 等于数据量 | 可能不一致 |

## 快照操作

### HBase Shell 操作

```bash
# 连接 HBase Shell
hbase shell

# 创建快照
snapshot 'users', 'users_snapshot_20240101'

# 列出所有快照
list_snapshots

# 查看快照详情
list_snapshots 'users_snapshot*'

# 重命名快照
rename_snapshot 'users_snapshot_20240101', 'users_snapshot_backup'

# 删除快照
delete_snapshot 'users_snapshot_backup'
```

### Java API 操作

```java
// 快照操作
Admin admin = connection.getAdmin();

// 创建快照
SnapshotDescription snapshot = admin.snapshot(
    "users_snapshot_20240101",
    TableName.valueOf("users")
);

// 列出所有快照
List<SnapshotDescription> snapshots = admin.listSnapshots();

// 列出特定表的快照
List<SnapshotDescription> tableSnapshots = 
    admin.listSnapshots("users_snapshot*");

// 检查快照是否存在
boolean exists = admin.hasSnapshot("users_snapshot_20240101");

// 删除快照
admin.deleteSnapshot("users_snapshot_backup");

// 获取快照信息
SnapshotDescription snapshotInfo = admin.getSnapshot("users_snapshot_20240101");
System.out.println("Snapshot: " + snapshotInfo.getName());
System.out.println("Table: " + snapshotInfo.getTable());
System.out.println("Creation time: " + snapshotInfo.getCreationTime());
```

## 快照恢复

### 恢复到原表

```bash
# HBase Shell：恢复快照到原表
restore_snapshot 'users_snapshot_20240101'
```

```java
// Java API：恢复快照
admin.restoreSnapshot("users_snapshot_20240101");
```

:::warning 恢复前准备
恢复快照会覆盖当前表数据。在恢复前，建议：
1. 禁用当前表
2. 备份当前表（或者先创建一个快照）
3. 确认恢复范围
:::

### 恢复到新表

```bash
# 克隆快照到新表（不修改原表）
clone_snapshot 'users_snapshot_20240101', 'users_restore'
```

```java
// Java API：克隆快照到新表
admin.cloneSnapshot("users_snapshot_20240101", TableName.valueOf("users_restore"));
```

```
克隆快照的流程：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  快照 "users_snapshot_20240101"                              │
│      │                                                      │
│      │ 克隆到新表 users_restore                              │
│      ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  新表 "users_restore"                                │   │
│  │  - 共享原快照引用的 HFile                            │   │
│  │  - 数据立即可用                                      │   │
│  │  - 对新表的修改不影响原表                            │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 快照导出与迁移

### ExportSnapshot

ExportSnapshot 可以将快照导出到其他集群：

```bash
# 导出快照到 HDFS
hbase org.apache.hadoop.hbase.snapshot.ExportSnapshot \
  -snapshot users_snapshot_20240101 \
  -copy-to hdfs://target-cluster/hbase \
  -mappers 16 \
  -bandwidth 100

# 参数说明：
# -snapshot: 快照名
# -copy-to: 目标 HDFS 路径
# -mappers: MapReduce 并行度
# -bandwidth: 带宽限制（MB/s）
```

### 跨集群恢复

```bash
# 在目标集群执行：
# 1. 先创建快照目录
hdfs dfs -mkdir -p /hbase/archive/snapshot

# 2. 从源集群复制快照数据
hadoop distcp \
  hdfs://source-cluster/hbase/archive/snapshot/users_snapshot_20240101 \
  hdfs://target-cluster/hbase/archive/snapshot/

# 3. 在目标集群创建快照
hbase shell
restore_snapshot 'users_snapshot_20240101'
```

## 快照原理详解

### 快照存储位置

快照信息存储在 HDFS 上：

```
快照存储位置：
/hbase/.hbase-snapshot/
├── users_snapshot_20240101/
│   ├── .snapshotinfo          # 快照元信息
│   ├── manifest.json          # 文件清单
│   └── ...
└── orders_snapshot_backup/
    └── ...
```

### 快照创建过程

```
快照创建流程：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  1. 获取表级锁                                              │
│     └─ 禁止表的 Schema 变更                                │
│                                                              │
│  2. 记录当前 HFile 列表                                     │
│     └─ 遍历所有 Region                                      │
│     └─ 记录每个 Column Family 的 HFile                     │
│                                                              │
│  3. 更新 .snapshotinfo                                      │
│     └─ 写入快照名称、表名、时间戳                          │
│                                                              │
│  4. 释放表级锁                                              │
│                                                              │
│  整个过程在秒级完成，不会复制数据                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 引用计数机制

当创建快照后，HBase 使用引用计数来管理 HFile：

```
引用计数机制：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  创建快照前：                                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  HFile-1  (引用计数 = 0)                             │   │
│  │  - 没有快照引用                                       │   │
│  │  - 可以被 Compaction 删除                           │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  创建快照后：                                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  HFile-1  (引用计数 = 1)                             │   │
│  │  - 被快照引用                                        │   │
│  │  - Compaction 时保留原文件                          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  Compaction 后：                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  HFile-1  (引用计数 = 0, 等待删除)                   │   │
│  │  HFile-2  (引用计数 = 1, 新文件)                    │   │
│  │  - Compaction 生成新文件                              │   │
│  │  - 原文件引用计数减 1                                │   │
│  │  - 当计数为 0 时，原文件可删除                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 备份策略

### 策略1：每日快照 + 定期导出

```bash
# 每日快照脚本
#!/bin/bash
DATE=$(date +%Y%m%d)

# 创建快照
hbase shell <<EOF
snapshot 'users', 'users_snapshot_$DATE'
snapshot 'orders', 'orders_snapshot_$DATE'
snapshot 'products', 'products_snapshot_$DATE'
EOF

# 保留最近 7 天的快照
hbase shell <<EOF
list_snapshots | grep 'snapshot_' | sort | head -n -7 | xargs -I {} delete_snapshot {}
EOF
```

### 策略2：增量备份 + 定期全量

```bash
# 每周日：全量导出
hbase org.apache.hadoop.hbase.snapshot.ExportSnapshot \
  -snapshot weekly_full_backup \
  -copy-to hdfs://backup-cluster/hbase/snapshots/weekly/

# 每周一~六：快照保留
snapshot 'users', "users_weekly_day$(date +%u)"
```

### 策略3：跨集群容灾

```
跨集群容灾架构：
┌─────────────────────────┐      异步复制      ┌─────────────────────────┐
│      生产集群            │  ─────────────▶  │       备份集群           │
│                         │                   │                         │
│  ┌─────────────────┐   │                   │  ┌─────────────────┐   │
│  │  用户表          │   │   ExportSnapshot   │  │  用户表(恢复)   │   │
│  │  快照           │   │   ━━━━━━━━━━━━━▶  │  │  快照           │   │
│  └─────────────────┘   │                   │  └─────────────────┘   │
│                         │                   │                         │
└─────────────────────────┘                   └─────────────────────────┘

故障时：
1. 停止生产集群写入
2. 导出最新快照到备份集群
3. 在备份集群恢复数据
4. 切换应用连接到备份集群
```

## 快照与 Replication

快照用于备份恢复，而 Replication 用于实时同步。两者可以结合使用：

```
快照 + Replication 架构：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  生产集群                          备份集群                   │
│  ┌──────────────┐                ┌──────────────┐          │
│  │   HBase      │── Replication ──▶│   HBase      │          │
│  │              │                │              │          │
│  │  ┌────────┐ │                │  ┌────────┐ │          │
│  │  │ 快照   │ │   ExportSnapshot │  │ 快照   │ │          │
│  │  └────────┘ │  ━━━━━━━━━━━━━━▶│  └────────┘ │          │
│  └──────────────┘                └──────────────┘          │
│                                                              │
│  Replication：实时同步（同步所有写入）                      │
│  快照：定时备份（保留历史版本）                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 常见问题与注意事项

### 快照过多影响性能

```bash
# 检查快照数量
hbase shell
list_snapshots | wc -l

# 建议：定期清理不需要的快照
delete_snapshot 'old_snapshot'
```

### 快照导出失败

```bash
# 检查 HDFS 空间
hdfs dfs -df -h

# 检查权限
hdfs dfs -ls /hbase/.hbase-snapshot

# 重试导出
hbase org.apache.hadoop.hbase.snapshot.ExportSnapshot \
  -snapshot snapshot_name \
  -copy-to hdfs://target/path \
  -overwrite
```

### 快照恢复失败

```bash
# 检查快照是否存在
hbase shell
list_snapshots

# 检查表状态
is_enabled 'users'
is_disabled 'users'

# 如果表被禁用，先启用
enable 'users'

# 然后恢复
restore_snapshot 'snapshot_name'
```
