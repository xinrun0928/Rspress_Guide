# 性能监控

监控是运维的眼睛。没有监控，你就像在黑暗中飞行——不知道方向，不知道速度，更不知道何时会撞山。

HBase 提供了丰富的 Metrics 接口，可以监控集群的各个方面。这篇文章会带你了解 HBase 的监控体系，以及如何通过监控发现问题。

## 关键监控指标

### 1. 请求相关

```java
// 关键指标
| 指标 | 说明 | 告警阈值 |
| --- | --- | --- |
| readRequestCount | 读请求总数 | 异常波动 |
| writeRequestCount | 写请求总数 | 异常波动 |
| requestCountPerSecond | 每秒请求数 | > 10000 需关注 |
| queueSize | RPC 队列大小 | > 100 需关注 |
| queueCallTime | 队列等待时间 | > 100ms 需关注 |
```

### 2. 存储相关

```java
| 指标 | 说明 | 告警阈值 |
| --- | --- | --- |
| regionCount | Region 数量 | > 200 需关注 |
| storeFileCount | HFile 数量 | > 30 需关注 |
| memStoreSize | MemStore 大小 | > 128MB 需关注 |
| hlogFileCount | WAL 文件数 | > 32 需关注 |
| maxStoreFileAge | 最旧 HFile 时间 | 超过 TTL |
```

### 3. 缓存相关

```java
| 指标 | 说明 | 告警阈值 |
| --- | --- | --- |
| blockCacheHitCount | BlockCache 命中数 | - |
| blockCacheMissCount | BlockCache 未命中数 | - |
| blockCacheHitRatio | BlockCache 命中率 | < 0.8 需关注 |
| blockCacheEvictedCount | BlockCache 淘汰数 | 过多需关注 |
```

### 4. Compaction 相关

```java
| 指标 | 说明 | 告警阈值 |
| --- | --- | --- |
| compactionSize | Compaction 数据量 | 异常增大 |
| compactionTime | Compaction 时间 | 过长需关注 |
| compactionQueueSize | Compaction 队列 | > 10 需关注 |
```

## 常见问题监控

### 1. 热点检测

```java
// 监控方法：检测 Region 间的负载不均衡
// 每个 RegionServer 的 Region 数量应该大致相等

// PromQL 查询：
// 不均衡度 = max(regionCount) / avg(regionCount)
max(hbase_region_count) / avg(hbase_region_count)

// 告警条件：不均衡度 > 1.5
```

### 2. 写入瓶颈检测

```java
// 监控方法：检查 MemStore 大小和 flush 频率

// MemStore 大小接近阈值
hbase_memStoreSize / hbase_memStore_flush_size > 0.8

// WAL 文件数量过多（说明 flush 太慢）
hbase_hlog_count > 30

// 告警：MemStore 持续高位，可能需要调参
```

### 3. 读取性能检测

```java
// 监控方法：BlockCache 命中率和 HFile 数量

// BlockCache 命中率下降
hbase_blockCache_hitRatio < 0.7

// HFile 数量过多（读取放大）
hbase_storeFile_count > 50

// 复合告警
hbase_blockCache_hitRatio < 0.7 AND hbase_storeFile_count > 50
```

### 4. Compaction 压力检测

```java
// 监控方法：Compaction 队列和执行时间

// Compaction 队列积压
hbase_compaction_queue_size > 10

// Compaction 执行时间长
hbase_compaction_time_max > 300000  // 5分钟

// Compaction 影响写入
hbase_compaction_size / hbase_write_request_count > 0.1
```

```java
| gc_pause_count | GC 次数 |
| memory_mb_used | 内存使用 |
| memory_mb_max | 最大内存 |
```

### GC 告警规则

```yaml
# Grafana 告警规则
- alert: HBaseGC_PAUSE
  expr: rate(gc_pause_time_sum[5m]) > 1000
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "HBase GC 暂停时间过长"
    description: "GC 暂停时间超过 1 秒，请检查内存配置"
```

## 自定义 Metrics

### 添加自定义 Metrics

```java
// 自定义 Metrics 示例
public class CustomObserver extends RegionObserver {
    
    private static final Counter WRITE_COUNTER = MetricsCounter.builder("custom_write_count")
        .description("Custom write counter")
        .tag("table", "unknown")
        .build();
    
    @Override
    public void postPut(ObserverContext<RegionCoprocessorEnvironment> e,
                        Put put, WALEdit edit, Durability durability) {
        
        String tableName = e.getEnvironment().getRegion()
            .getTableDescriptor().getTableName().getNameAsString();
        
        // 更新 Metrics
        WRITE_COUNTER.builder()
            .tag("table", tableName)
            .increment();
    }
}
```

## 监控最佳实践

### 1. 告警阈值设置

```yaml
# 建议的告警阈值
alerts:
  # 请求相关
  - name: high_write_latency
    condition: write_latency_p99 > 100
    severity: warning
    
  - name: queue_overflow
    condition: rpc_queue_size > 100
    severity: critical
    
  # 存储相关
  - name: too_many_hfiles
    condition: storeFile_count > 50
    severity: warning
    
  - name: memstore_full
    condition: memStoreSize > 128MB
    severity: critical
    
  # 缓存相关
  - name: low_cache_hit_ratio
    condition: blockCache_hitRatio < 0.7
    severity: warning
```

### 2. 监控指标保留

```yaml
# 数据保留策略
retention:
  - 1分钟精度: 保留 7 天
  - 5分钟精度: 保留 30 天
  - 1小时精度: 保留 1 年
```
