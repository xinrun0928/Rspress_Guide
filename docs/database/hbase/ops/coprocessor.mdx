# 协处理器：Observer 与 Endpoint

假设你有这样的需求：
1. 在写入数据时自动记录操作日志
2. 在查询时自动过滤敏感数据
3. 在更新数据时触发外部通知

传统的做法是在应用层实现这些逻辑，但这样会增加应用复杂度，也不够优雅。HBase 提供了协处理器（Coprocessor）机制，让这些逻辑可以在服务端执行。

协处理器是 HBase 的高级特性，用好它可以让 HBase 变得更加强大和灵活。

## 协处理器的类型

HBase 有两种协处理器：

### Observer

Observer 类似于数据库的触发器，在特定的钩子点执行自定义逻辑。

```
Observer 钩子点：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│                     Observer 钩子点                           │
│                                                              │
│  1. RegionObserver                                          │
│     ├─ preGet / postGet                                    │
│     ├─ prePut / postPut                                    │
│     ├─ preDelete / postDelete                              │
│     ├─ preIncrement / postIncrement                         │
│     ├─ preScan / postScan                                  │
│     ├─ preOpen / postOpen                                  │
│     └─ ...                                                  │
│                                                              │
│  2. MasterObserver                                          │
│     ├─ preCreateTable / postCreateTable                     │
│     ├─ preDeleteTable / postDeleteTable                     │
│     ├─ preModifyTable / postModifyTable                     │
│     └─ ...                                                  │
│                                                              │
│  3. WALObserver                                             │
│     ├─ preWALWrite / postWALWrite                           │
│     └─ ...                                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Endpoint

Endpoint 类似于数据库的存储过程，允许在服务端执行自定义计算。

```
Endpoint 执行流程：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  客户端调用 Endpoint                                         │
│      │                                                      │
│      ▼                                                      │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  RegionServer 1      RegionServer 2      RegionServer 3│   │
│  │      │                    │                    │      │
│  │      ▼                    ▼                    ▼      │
│  │  ┌────────────────────────────────────────────────┐ │   │
│  │  │           执行 Endpoint 逻辑                    │ │   │
│  │  │    (在服务端计算，减少网络传输)                 │ │   │
│  │  └────────────────────────────────────────────────┘ │   │
│  │      │                    │                    │      │
│      └──────┬─────────────────┼────────────────────┘      │
│             ▼                                       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              合并结果，返回客户端                       │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## Observer 协处理器

### 使用场景

1. **数据过滤**：查询时自动过滤敏感数据
2. **审计日志**：记录所有写操作
3. **数据验证**：写入前校验数据合法性
4. **二级索引**：写入时同步更新索引表
5. **访问控制**：基于用户权限过滤数据

### 示例：审计日志

```java
// 审计日志 Observer
public class AuditObserver extends RegionObserver {
    private static final Logger LOG = LoggerFactory.getLogger(AuditObserver.class);
    
    // 定义审计日志表
    private static final byte[] AUDIT_TABLE = Bytes.toBytes("audit_log");
    private static final byte[] CF_AUDIT = Bytes.toBytes("info");
    
    @Override
    public void postPut(ObserverContext<RegionCoprocessorEnvironment> e,
                        Put put, WALEdit edit, Durability durability) throws IOException {
        
        // 获取被操作的表和 Region 信息
        Region region = e.getEnvironment().getRegion();
        String tableName = region.getTableDescriptor().getTableName().getNameAsString();
        byte[] rowKey = put.getRow();
        
        // 记录审计日志
        Table auditTable = e.getEnvironment().getConnection()
            .getTable(TableName.valueOf(AUDIT_TABLE));
        
        try {
            // 审计日志 RowKey：时间戳 + 操作表名
            long timestamp = System.currentTimeMillis();
            String auditRowKey = timestamp + "_" + tableName;
            
            Put auditPut = new Put(Bytes.toBytes(auditRowKey));
            auditPut.addColumn(CF_AUDIT, Bytes.toBytes("table"), 
                Bytes.toBytes(tableName));
            auditPut.addColumn(CF_AUDIT, Bytes.toBytes("rowKey"), rowKey);
            auditPut.addColumn(CF_AUDIT, Bytes.toBytes("operation"), 
                Bytes.toBytes("PUT"));
            auditPut.addColumn(CF_AUDIT, Bytes.toBytes("timestamp"), 
                Bytes.toBytes(timestamp));
            
            auditTable.put(auditPut);
        } finally {
            auditTable.close();
        }
        
        LOG.info("Audit: {} operation on table {}, rowKey {}", 
            "PUT", tableName, Bytes.toString(rowKey));
    }
    
    @Override
    public void postDelete(ObserverContext<RegionCoprocessorEnvironment> e,
                           Delete delete, WALEdit edit, Durability durability) 
                           throws IOException {
        
        Region region = e.getEnvironment().getRegion();
        String tableName = region.getTableDescriptor().getTableName().getNameAsString();
        
        // 记录删除操作的审计日志
        // ... 类似 postPut 的实现
        
        LOG.info("Audit: DELETE operation on table {}, rowKey {}", 
            tableName, Bytes.toString(delete.getRow()));
    }
}
```

### 示例：二级索引

```java
// 二级索引 Observer
public class SecondaryIndexObserver extends RegionObserver {
    
    private static final byte[] MAIN_TABLE = Bytes.toBytes("orders");
    private static final byte[] INDEX_TABLE = Bytes.toBytes("orders_index");
    private static final byte[] CF_INDEX = Bytes.toBytes("info");
    
    @Override
    public void postPut(ObserverContext<RegionCoprocessorEnvironment> e,
                        Put put, WALEdit edit, Durability durability) throws IOException {
        
        // 获取主表的 Region 信息
        Region region = e.getEnvironment().getRegion();
        if (!region.getTableDescriptor().getTableName().getNameAsString()
            .equals("orders")) {
            return;  // 不是 orders 表，跳过
        }
        
        // 从主表数据中提取索引字段
        // 假设主表：RowKey=orderId, CF:info:userId, CF:info:productId
        byte[] orderId = put.getRow();
        byte[] userId = null;
        byte[] productId = null;
        
        Cell userIdCell = put.getCellLatest(Bytes.toBytes("info"), Bytes.toBytes("userId"));
        if (userIdCell != null) {
            userId = CellUtil.cloneValue(userIdCell);
        }
        
        Cell productIdCell = put.getCellLatest(Bytes.toBytes("info"), Bytes.toBytes("productId"));
        if (productIdCell != null) {
            productId = CellUtil.cloneValue(productIdCell);
        }
        
        // 写入索引表
        Table indexTable = e.getEnvironment().getConnection()
            .getTable(TableName.valueOf(INDEX_TABLE));
        
        try {
            // 索引表 RowKey = userId + orderId（按用户查订单）
            if (userId != null) {
                Put indexPut = new Put(userId);
                indexPut.addColumn(CF_INDEX, orderId, orderId);  // 存储订单 ID
                indexTable.put(indexPut);
            }
            
            // 索引表 RowKey = productId + orderId（按商品查订单）
            if (productId != null) {
                Put indexPut = new Put(productId);
                indexPut.addColumn(CF_INDEX, orderId, orderId);
                indexTable.put(indexPut);
            }
        } finally {
            indexTable.close();
        }
    }
}
```

### 示例：数据验证

```java
// 数据验证 Observer
public class ValidationObserver extends RegionObserver {
    
    @Override
    public void prePut(ObserverContext<RegionCoprocessorEnvironment> e,
                       Put put, WALEdit edit, Durability durability) throws IOException {
        
        // 获取要写入的列值
        List<Cell> cells = put.getCells();
        
        for (Cell cell : cells) {
            byte[] family = CellUtil.cloneFamily(cell);
            byte[] qualifier = CellUtil.cloneQualifier(cell);
            byte[] value = CellUtil.cloneValue(cell);
            
            // 验证 age 列必须在 0-150 之间
            if (Bytes.equals(family, Bytes.toBytes("info")) &&
                Bytes.equals(qualifier, Bytes.toBytes("age"))) {
                
                int age = Bytes.toInt(value);
                if (age < 0 || age > 150) {
                    throw new DoNotRetryIOException(
                        "Invalid age value: " + age + ". Must be between 0 and 150.");
                }
            }
            
            // 验证 email 列必须是有效格式
            if (Bytes.equals(family, Bytes.toBytes("info")) &&
                Bytes.equals(qualifier, Bytes.toBytes("email"))) {
                
                String email = Bytes.toString(value);
                if (!email.contains("@")) {
                    throw new DoNotRetryIOException(
                        "Invalid email format: " + email);
                }
            }
        }
    }
}
```

### 部署 Observer

```java
// 通过 Java API 部署 Observer
public void deployObserver() throws IOException {
    Admin admin = connection.getAdmin();
    
    // 创建表时指定 Observer
    TableDescriptor tableDescriptor = TableDescriptorBuilder
        .newBuilder(TableName.valueOf("orders"))
        .setColumnFamily(ColumnFamilyDescriptorBuilder.of("info"))
        .setCoprocessor("com.example.AuditObserver")
        .setCoprocessor("com.example.SecondaryIndexObserver")
        .build();
    
    admin.createTable(tableDescriptor);
}

// 或者修改已有表
public void modifyTableWithObserver() throws IOException {
    Admin admin = connection.getAdmin();
    
    TableDescriptor table = admin.getTableDescriptor(TableName.valueOf("orders"));
    TableDescriptor modified = TableDescriptorBuilder.newBuilder(table)
        .setCoprocessor("com.example.AuditObserver")
        .build();
    
    admin.modifyTable(modified);
}
```

## Endpoint 协处理器

### 使用场景

1. **聚合计算**：在服务端计算 COUNT、SUM、AVG 等
2. **复杂查询**：在服务端执行复杂过滤逻辑
3. **数据转换**：在服务端转换数据格式
4. **加密解密**：在服务端执行加解密

### 示例：聚合计算

```java
// 定义 Endpoint 接口
public interface AggregationProtocol extends CoprocessorProtocol {
    // 定义聚合方法
    long sum(byte[] family, byte[] qualifier) throws IOException;
    long count(byte[] family, byte[] qualifier) throws IOException;
    double avg(byte[] family, byte[] qualifier) throws IOException;
}

// 实现 Endpoint
public class AggregationEndpoint implements AggregationProtocol, 
                                            RegionCoprocessor {
    
    @Override
    public long sum(byte[] family, byte[] qualifier) throws IOException {
        long sum = 0;
        
        // 获取当前 Region
        Region region = getEnvironment().getRegion();
        InternalScanner scanner = region.getScanner(
            new Scan().addColumn(family, qualifier));
        
        try {
            List<Cell> cells = new ArrayList<>();
            while (scanner.next(cells)) {
                for (Cell cell : cells) {
                    sum += Bytes.toLong(CellUtil.cloneValue(cell));
                }
                cells.clear();
            }
        } finally {
            scanner.close();
        }
        
        return sum;
    }
    
    @Override
    public long count(byte[] family, byte[] qualifier) throws IOException {
        long count = 0;
        
        Region region = getEnvironment().getRegion();
        InternalScanner scanner = region.getScanner(
            new Scan().addColumn(family, qualifier));
        
        try {
            List<Cell> cells = new ArrayList<>();
            while (scanner.next(cells)) {
                count += cells.size();
                cells.clear();
            }
        } finally {
            scanner.close();
        }
        
        return count;
    }
    
    @Override
    public double avg(byte[] family, byte[] qualifier) throws IOException {
        long sum = sum(family, qualifier);
        long count = count(family, qualifier);
        return count > 0 ? (double) sum / count : 0;
    }
}
```

### 调用 Endpoint

```java
// 客户端调用 Endpoint
public void callEndpoint() throws IOException {
    Table table = connection.getTable(TableName.valueOf("orders"));
    
    // 创建 Invocation
    Map<byte[], Long> results = table.coprocessorService(
        AggregationProtocol.class,  // Endpoint 接口
        null,                      // 起始 RowKey（null 表示所有 Region）
        null,                      // 结束 RowKey（null 表示所有 Region）
        new Batch.Call<AggregationProtocol, Long>() {
            @Override
            public Long call(AggregationProtocol instance) throws IOException {
                // 在每个 Region 上执行
                return instance.sum(
                    Bytes.toBytes("info"), 
                    Bytes.toBytes("amount")
                );
            }
        }
    );
    
    // 合并所有 Region 的结果
    long totalSum = 0;
    for (Long sum : results.values()) {
        totalSum += sum;
    }
    
    System.out.println("Total sum: " + totalSum);
}
```

## 协处理器的限制与注意事项

### 协处理器的限制

```
协处理器限制：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  1. 异常处理                                                 │
│     - Observer 中抛出 IOException 会中断操作                │
│     - Endpoint 中抛出异常会导致整个请求失败                │
│                                                              │
│  2. 性能影响                                                 │
│     - 每个协处理器都会增加操作延迟                          │
│     - 协处理器中的耗时操作会影响吞吐量                      │
│                                                              │
│  3. 资源管理                                                 │
│     - 协处理器运行在 RegionServer 进程中                    │
│     - 内存泄漏会影响 RegionServer 稳定性                   │
│                                                              │
│  4. 事务限制                                                 │
│     - Observer 中的操作与主操作不在同一事务中              │
│     - 需要自行处理失败回滚                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 安全注意事项

```java
// 协处理器中的安全考虑

// 1. 避免直接访问文件系统
// 不要在协处理器中读写本地文件系统

// 2. 避免耗时操作
// 不要在协处理器中执行网络请求
// 不要在协处理器中执行大计算

// 3. 资源清理
public class ExampleObserver extends RegionObserver {
    private Connection connection;
    
    @Override
    public void start(CoprocessorEnvironment env) throws IOException {
        // 初始化资源
        connection = ConnectionFactory.createConnection(env.getConfiguration());
    }
    
    @Override
    public void stop(CoprocessorEnvironment env) throws IOException {
        // 清理资源
        if (connection != null) {
            connection.close();
        }
    }
}
```

## 协处理器与二级索引

协处理器最常见的应用场景是实现二级索引：

```
协处理器实现二级索引的架构：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│                      写入 orders 表                          │
│                           │                                  │
│                           ▼                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              SecondaryIndexObserver                     │   │
│  │                                                       │   │
│  │   postPut:                                            │   │
│  │   - 提取索引字段 (userId, productId)                  │   │
│  │   - 写入 orders_index 表                              │   │
│  └──────────────────────────────────────────────────────┘   │
│                           │                                  │
│                           ▼                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │                    查询流程                             │   │
│  │                                                       │   │
│  │   1. 先查索引表 orders_index（按 userId 查）         │   │
│  │   2. 获取 orderId                                     │   │
│  │   3. 再查 orders 表（按 orderId 查）                  │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```
