# 故障排查

凌晨 3 点，告警响起：HBase 集群不可用。你的第一反应是什么？从哪里开始排查？有哪些常见的故障模式？

故障排查是运维技能的集大成者。它需要你理解系统的每一个组件，知道正常状态是什么，才能在异常时快速定位问题。

这篇文章会带你了解 HBase 常见故障的排查方法。

## 常见故障场景

### 场景1：客户端无法连接

**症状**：
```
ERROR: org.apache.hadoop.hbase.client.NoServerForRegionException: 
    Unable to find region for xxx
```

**排查步骤**：

```bash
# 1. 检查 ZooKeeper 服务
zkCli.sh -server zookeeper:2181
# 查看 /hbase/master 是否存在
get /hbase/master

# 查看 /hbase/meta-region-server 是否存在
get /hbase/meta-region-server

# 2. 检查 HMaster 状态
curl http://master:16010/master-status

# 3. 检查 meta 表健康
hbase shell
scan 'hbase:meta', {LIMIT => 10}
```

**常见原因与解决方案**：

| 原因 | 解决方案 |
| --- | --- |
| ZooKeeper 未启动 | 启动 ZooKeeper |
| HMaster 未启动 | 启动 HMaster |
| Meta 表损坏 | 使用 HBCK 修复 |
| 网络分区 | 检查网络配置 |

### 场景2：写入延迟飙升

**症状**：
```
写入延迟从 5ms 飙升到 500ms
```

**排查步骤**：

```bash
# 1. 查看 RegionServer 日志
tail -f /var/log/hbase/hbase-hbase-regionserver-*.log

# 关键词：
# - "Running compaction"
# - "MemStore flush"
# - "WAL sync"

# 2. 检查 MemStore 大小
curl http://regionserver:16030/rs-status | grep -i memstore

# 3. 检查 HFile 数量
curl http://regionserver:16030/rs-status | grep -i storeFile

# 4. 检查 Compaction 队列
curl http://regionserver:16030/jmx | grep CompactionQueueSize
```

**可能原因与解决方案**：

```bash
# 原因1：Compaction 占用大量 I/O
# 解决：调整 Compaction 策略，或在低峰期执行 Major Compaction

# 原因2：MemStore 刷写频繁
# 解决：调大 flush 阈值
# hbase.hregion.memstore.flush.size = 256MB

# 原因3：热点 Region
# 解决：优化 RowKey，手动 split 热点 Region

# 原因4：WAL 写入慢
# 解决：检查 HDFS 写入延迟，考虑使用异步 WAL
```

### 场景3：RegionServer 频繁宕机

**症状**：
```
RegionServer 日志出现：
FATAL: org.apache.hadoop.hbase.util.JvmPauseMonitor: 
    Detected pause in JVM or machine
```

**排查步骤**：

```bash
# 1. 检查 JVM 状态
jstat -gcutil <pid> 1000

# 关键指标：
# - Old区使用率 > 90%
# - Full GC 频繁
# - GC 时间过长

# 2. 检查内存配置
grep -E "HBASE_HEAPSIZE|HEAP" /etc/hbase/conf/hbase-env.sh

# 3. 检查 Region 数量
curl http://regionserver:16030/rs-status | grep regionCount

# 4. 检查日志中的异常
grep -E "OutOfMemory|GC|ERROR|FATAL" /var/log/hbase/hbase-hbase-regionserver-*.log
```

**OOM 常见原因**：

```
┌─────────────────────────────────────────────────────────────┐
│  RegionServer OOM 常见原因：                                 │
│                                                              │
│  1. MemStore 过大                                           │
│     - Region 数过多                                          │
│     - 单 Region MemStore 上限未设置                         │
│     - 写入压力大，MemStore 无法及时刷写                     │
│                                                              │
│  2. BlockCache 过大                                         │
│     - 缓存配置过大                                          │
│     - 读取热点数据太多                                      │
│                                                              │
│  3. Handler 线程过多                                        │
│     - hbase.regionserver.handler.count 配置过大             │
│     - 请求堆积                                              │
│                                                              │
│  4. Compacted HFile 淘汰不及时                              │
│     - HFile 数量过多                                        │
│     - Compaction 线程不足                                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**解决方案**：

```xml
<!-- 建议的内存配置 -->
<property>
  <name>hbase.regionserver.global.memstore.size</name>
  <value>0.4</value>  <!-- 不超过堆内存 40% -->
</property>

<property>
  <name>hfile.block.cache.size</name>
  <value>0.3</value>  <!-- BlockCache 不超过 30% -->
</property>

<property>
  <name>hbase.regionserver.handler.count</name>
  <value>30</value>  <!-- 根据 CPU 核数调整 -->
</property>
```

### 场景4：Meta 表问题

**症状**：
```
ERROR: TableExistsException: hbase:meta
     at org.apache.hadoop.hbase.MetaTableAccessor
```

或：
```
ERROR: Region {METHOD => 'get', NAME => 'hbase:meta,,1'}
     is not on regionserver
```

**排查步骤**：

```bash
# 1. 检查 meta 表状态
hbase shell
describe 'hbase:meta'
scan 'hbase:meta', {LIMIT => 5}

# 2. 检查 meta 表 Region 位置
hbase shell
get 'hbase:meta', 'users,,1', 'info:regioninfo'

# 3. 使用 HBCK 检查
hbase hbck
hbase hbck -details
```

**修复方法**：

```bash
# 修复 Region 分配
hbase hbck -fixAssignments

# 修复 Meta 表
hbase hbck -fixMeta

# 修复 HDFS
hbase hbck -fixHdfs

# 全面修复
hbase hbck -fix -fixMeta -fixAssignments
```

### 场景5：数据不一致

**症状**：
```
读取结果与预期不符
Scan 结果丢失数据
```

**排查步骤**：

```bash
# 1. 检查 Region 是否在线
hbase shell
list_regions 'my_table'

# 2. 检查 HFile 和 WAL
hdfs dfs -ls /hbase/data/default/my_table/*/*

# 3. 检查是否有残留的 Reference 文件
hdfs dfs -ls /hbase/data/default/my_table/*/*.reference

# 4. 检查 Compaction 状态
curl http://regionserver:16030/jmx | grep compaction
```

**修复方法**：

```bash
# 方法1：Major Compaction 重建数据
hbase shell
major_compact 'my_table'

# 方法2：恢复快照
hbase shell
restore_snapshot 'my_snapshot'

# 方法3：重建表
# 1. 导出数据
hbase org.apache.hadoop.hbase.mapreduce.Export my_table /tmp/my_table_export

# 2. 禁用并删除表
disable 'my_table'
drop 'my_table'

# 3. 重建表
create 'my_table', 'info'

# 4. 导入数据
hbase org.apache.hadoop.hbase.mapreduce.Import my_table /tmp/my_table_export
```

## HBCK 工具使用

### HBCK 常用命令

```bash
# 检查集群状态
hbase hbck

# 详细检查
hbase hbck -details

# 修复 Region 分配
hbase hbck -fixAssignments

# 修复 Meta 表
hbase hbck -fixMeta

# 修复 HDFS 一致性
hbase hbck -fixHdfs

# 跳过 HDFS 检查（仅修复元数据）
hbase hbck -skipFS

# 修复所有问题
hbase hbck -fix -fixMeta -fixAssignments -fixHdfs

# 指定 Region 修复
hbase hbck -fix -r 'table,region'
```

### HBCK 2.0 新特性

HBase 2.0+ 的 HBCK 有重大改进：

```bash
# HBCK 2.0+ 命令
# 检查 Region 过渡状态
hbase hbck -j /path/to/hbase-hbck2.jar reports

# 修复 Region 状态
hbase hbck -j /path/to/hbase-hbck2.jar bypass <region>

# 强制上线 Region
hbase hbck -j /path/to/hbase-hbck2.jar unassign <region>

# 修复复制管道
hbase hbck -j /path/to/hbase-hbck2.jar repair
```

## 日志分析

### 关键日志位置

```bash
# HMaster 日志
/var/log/hbase/hbase-hbase-master-*.log

# RegionServer 日志
/var/log/hbase/hbase-hbase-regionserver-*.log

# ZooKeeper 日志
/var/log/zookeeper/zookeeper-*.log

# HDFS 日志
$HADOOP_HOME/logs/hadoop-*-datanode-*.log
```

### 日志关键词

```bash
# Region 分裂相关
grep "Split transaction completed" /var/log/hbase/hbase-hbase-regionserver-*.log

# Compaction 相关
grep "Compaction" /var/log/hbase/hbase-hbase-regionserver-*.log | tail -100

# Region 上线/下线
grep -E "Online|Offline" /var/log/hbase/hbase-hbase-master-*.log

# 异常
grep -E "ERROR|FATAL|Exception" /var/log/hbase/hbase-hbase-regionserver-*.log | head -50

# GC
grep "GC" /var/log/hbase/hbase-hbase-regionserver-*.log | tail -100
```
