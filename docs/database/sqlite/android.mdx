# SQLite 与 Android 嵌入式开发

当你在 Android 手机上打开微信、浏览相册、或者查看地图缓存时，你正在使用 SQLite 数据库。事实上，Android 系统内置了 SQLite，从第一天起就把它作为本地数据存储的核心。

Android 选择 SQLite 是必然的：手机需要一款零配置、零运维、单文件、崩溃安全的数据库。这与 SQLite 的设计哲学完美契合。

你能回答这个问题吗——**「Android 为什么选择 SQLite 作为内置数据库，而不是 MySQL 或 PostgreSQL？」**

## Android SQLite API

### 基本使用

```java
// 1. 继承 SQLiteOpenHelper
public class MyDatabaseHelper extends SQLiteOpenHelper {
    private static final String DATABASE_NAME = "myapp.db";
    private static final int DATABASE_VERSION = 1;

    public MyDatabaseHelper(Context context) {
        super(context, DATABASE_NAME, null, DATABASE_VERSION);
    }

    @Override
    public void onCreate(SQLiteDatabase db) {
        db.execSQL("CREATE TABLE users (" +
                   "_id INTEGER PRIMARY KEY AUTOINCREMENT, " +
                   "name TEXT NOT NULL, " +
                   "email TEXT" +
                   ")");
    }

    @Override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
        db.execSQL("DROP TABLE IF EXISTS users");
        onCreate(db);
    }
}

// 2. 使用 Helper
MyDatabaseHelper helper = new MyDatabaseHelper(context);
SQLiteDatabase db = helper.getReadableDatabase();  // 获取只读数据库
SQLiteDatabase dbWrite = helper.getWritableDatabase();  // 获取可写数据库

// 3. 查询数据
Cursor cursor = db.query("users", null, null, null, null, null, null);
while (cursor.moveToNext()) {
    long id = cursor.getLong(cursor.getColumnIndexOrThrow("_id"));
    String name = cursor.getString(cursor.getColumnIndexOrThrow("name"));
    String email = cursor.getString(cursor.getColumnIndexOrThrow("email"));
}
cursor.close();

// 4. 插入数据
ContentValues values = new ContentValues();
values.put("name", "Alice");
values.put("email", "alice@example.com");
long rowId = db.insert("users", null, values);

// 5. 更新数据
ContentValues updateValues = new ContentValues();
updateValues.put("email", "new_email@example.com");
db.update("users", updateValues, "_id = ?", new String[]{"1"});

// 6. 删除数据
db.delete("users", "_id = ?", new String[]{"1"});
```

    db.beginTransaction();

## Room：现代 Android 数据库

### Room 简介

Room 是 Android 官方推荐的数据库解决方案，建立在 SQLite 之上，提供了编译时检查和更好的抽象：

```groovy
// build.gradle 添加依赖
dependencies {
    implementation "androidx.room:room-runtime:2.6.1"
    annotationProcessor "androidx.room:room-compiler:2.6.1"
}
```

### Room 组件

```java
// 1. Entity：定义表结构
@Entity(tableName = "users")
public class User {
    @PrimaryKey(autoGenerate = true)
    public long id;

    @ColumnInfo(name = "name")
    public String name;

    @ColumnInfo(name = "email")
    public String email;

    @ColumnInfo(name = "created_at")
    public long createdAt;
}

// 2. DAO：定义数据访问方法
@Dao
public interface UserDao {
    @Query("SELECT * FROM users ORDER BY created_at DESC")
    List<User> getAllUsers();

    @Query("SELECT * FROM users WHERE id = :id")
    User getUserById(long id);

    @Insert
    long insert(User user);

    @Update
    void update(User user);

    @Delete
    void delete(User user);

    @Query("SELECT * FROM users WHERE name LIKE :pattern")
    List<User> searchByName(String pattern);
}

// 3. Database：定义数据库
@Database(entities = {User.class}, version = 1)
public abstract class AppDatabase extends RoomDatabase {
    public abstract UserDao userDao();
}

// 4. 使用数据库
AppDatabase db = Room.databaseBuilder(
    getApplicationContext(),
    AppDatabase.class,
    "myapp.db"
).build();

UserDao userDao = db.userDao();
List<User> users = userDao.getAllUsers();
```

### Room 的优势

```java
// 1. 编译时 SQL 检查
// 如果 SQL 有语法错误，编译时会报错
@Query("SELECT * FROM users WHER name = :id")  // WHER 拼写错误
// 编译错误：error: no such column: name

// 2. LiveData 支持：数据变化自动通知 UI
@Dao
public interface UserDao {
    @Query("SELECT * FROM users")
    LiveData<List<User>> getAllUsersLive();  // 返回 LiveData
}

// Activity 中观察数据
userDao.getAllUsersLive().observe(this, users -> {
    // 数据变化时自动回调
    adapter.setUsers(users);
});

// 3. Coroutines 支持：异步数据库操作
@Dao
public interface UserDao {
    @Query("SELECT * FROM users")
    suspend fun getAllUsers(): List<User>;  // 协程版本
}

// ViewModel 中调用
viewModelScope.launch {
    val users = userDao.getAllUsers()
}

// 4. 迁移管理
Migration migration = new Migration(1, 2) {
    @Override
    public void migrate(SupportSQLiteDatabase database) {
        database.execSQL("ALTER TABLE users ADD COLUMN phone TEXT");
    }
};

Room.databaseBuilder(context, AppDatabase.class, "myapp.db")
    .addMigrations(migration)
    .build();
```

## ContentProvider：数据共享

### ContentProvider 简介

ContentProvider 是 Android 的组件间数据共享机制，底层使用 SQLite：

```java
// 1. 定义 ContentProvider
public class UserProvider extends ContentProvider {
    public static final String AUTHORITY = "com.example.myapp.provider";
    public static final Uri CONTENT_URI = Uri.parse("content://" + AUTHORITY + "/users");

    private MyDatabaseHelper dbHelper;

    @Override
    public boolean onCreate() {
        dbHelper = new MyDatabaseHelper(getContext());
        return true;
    }

    @Override
    public Cursor query(Uri uri, String[] projection, String selection,
                       String[] selectionArgs, String sortOrder) {
        SQLiteDatabase db = dbHelper.getReadableDatabase();
        Cursor cursor = db.query("users", projection, selection,
                                 selectionArgs, null, null, sortOrder);
        cursor.setNotificationUri(getContext().getContentResolver(), uri);
        return cursor;
    }

    @Override
    public Uri insert(Uri uri, ContentValues values) {
        SQLiteDatabase db = dbHelper.getWritableDatabase();
        long id = db.insert("users", null, values);
        getContext().getContentResolver().notifyChange(uri, null);
        return Uri.withAppendedPath(CONTENT_URI, String.valueOf(id));
    }

    // 其他方法...
}

// 2. AndroidManifest.xml 注册
<provider
    android:name=".UserProvider"
    android:authorities="com.example.myapp.provider"
    android:exported="false" />
```

## 性能优化

### 数据库性能优化

```java
// 1. 创建索引
db.execSQL("CREATE INDEX IF NOT EXISTS idx_name ON users(name)");

// 2. 使用批量操作
public void batchInsert(SQLiteDatabase db, List<ContentValues> values) {
    db.beginTransaction();
    try {
        for (ContentValues cv : values) {
            db.insert("users", null, cv);
        }
        db.setTransactionSuccessful();
    } finally {
        db.endTransaction();
    }
}

// 3. 使用 WAL 模式
db.enableWriteAheadLogging();

// 4. 配置缓存大小
db.setMaximumSize(1024 * 1024 * 100);  // 100MB 最大大小

// 5. 分析查询
Cursor cursor = db.rawQuery("EXPLAIN QUERY PLAN SELECT * FROM users WHERE name = ?",
                            new String[]{"Alice"});
```

### 避免常见问题

```java
// 1. 避免主线程操作数据库
// ❌ 错误
new Thread(() -> {
    db.insert("users", null, values);  // 在后台线程
}).start();
// 读取也在后台线程
// 但 Cursor 可能在其他线程使用

// ✅ 正确：使用 Room 或 AsyncQueryHandler
Executor executor = Executors.newSingleThreadExecutor();
executor.execute(() -> {
    // 数据库操作
});

// 2. 及时关闭 Cursor
Cursor cursor = null;
try {
    cursor = db.query("users", null, null, null, null, null, null);
    while (cursor.moveToNext()) {
        // 处理数据
    }
} finally {
    if (cursor != null) {
        cursor.close();
    }
}

// 3. 使用事务保证原子性
db.beginTransaction();
try {
    // 多个操作
    db.setTransactionSuccessful();
} finally {
    db.endTransaction();
}

// 4. 预编译 SQL
SQLiteStatement stmt = db.compileStatement(
    "INSERT INTO users (name, email) VALUES (?, ?)"
);
for (User user : users) {
    stmt.bindString(1, user.getName());
    stmt.bindString(2, user.getEmail());
    stmt.executeInsert();
    stmt.clearBindings();
}
```

## 面试要点总结

### 面试官可能会问的问题

**Q1：Android 为什么选择 SQLite 作为内置数据库？**

回答要点：
- SQLite 零配置、零运维，适合嵌入式环境
- 单文件数据库，便于备份和迁移
- 崩溃安全，保证数据完整性
- 资源占用小，适合资源受限的移动设备
- 支持 SQL 查询，开发者熟悉

**Q2：Room 相比直接使用 SQLiteOpenHelper 有什么优势？**

回答要点：
- 编译时 SQL 检查，减少运行时错误
- 减少模板代码，提高开发效率
- 支持 LiveData 和 Coroutines，便于响应式编程
- 支持数据库迁移
- 支持关系查询和 FTS 全文搜索

**Q3：Android 中如何使用 ContentProvider 共享数据？**

回答要点：
- ContentProvider 提供统一的 URI 接口访问数据
- 底层使用 SQLite 存储数据
- 通过 ContentResolver 在不同应用间共享数据
- 支持增删改查操作
- 需要在 AndroidManifest 中注册

**Q4：SQLite 在 Android 中的性能优化有哪些？**

回答要点：
- 合理创建索引
- 使用批量操作减少事务次数
- 启用 WAL 模式提高并发性能
- 避免在主线程操作数据库
- 及时关闭 Cursor
- 使用预编译 SQL
