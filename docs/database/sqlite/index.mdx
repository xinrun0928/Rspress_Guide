# SQLite 总览

> 全球部署量最多的数据库——在你的手机、浏览器、甚至火星探测器上，都在运行着 SQLite。

## SQLite 是什么

SQLite 是一款轻量级的嵌入式数据库，它的核心设计哲学是：**简单、可靠、零运维**。

与 MySQL、PostgreSQL 这类需要独立服务进程的数据库不同，SQLite 没有服务器进程，数据库就是一个文件。应用程序通过动态库直接读写这个文件，不需要网络通信，不需要复杂的配置，也不需要专职的 DBA。

```
MySQL/PostgreSQL:
┌──────────┐     网络      ┌──────────┐
│  应用   │ ──────────▶ │  数据库   │
└──────────┘             │  服务器   │
                         └──────────┘

SQLite:
┌──────────┐     API      ┌──────────┐
│  应用   │ ◀──────────▶ │  数据库   │
└──────────┘             │   文件    │
                         └──────────┘
```

## 为什么选择 SQLite

SQLite 被广泛使用在以下场景：

| 场景 | 代表应用 | SQLite 的优势 |
|---|---|---|
| 移动应用 | Android、iOS | 零配置、崩溃安全、文件便于备份 |
| 桌面应用 | VS Code、Skype | 单文件、易分发、无需安装服务 |
| 嵌入式系统 | 无人机、汽车电子 | 资源占用小、可靠性高 |
| 浏览器 | Chrome、Firefox | 内嵌在浏览器进程中 |
| 文件格式 | PDF、DICOM | 作为数据容器使用 |

## 核心特性

### ACID 事务

SQLite 的事务遵循 ACID 原则，默认隔离级别是 SERIALIZABLE。它的原子性基于文件系统的能力——事务要么全部成功，要么全部失败，没有中间状态。

### 无服务器架构

数据库就是一个文件，复制、移动、备份都极其方便。这使得 SQLite 成为嵌入式场景的首选。

### WAL 模式

Write-Ahead Logging 模式允许读写并发执行，显著提升了并发性能。

### 跨平台

SQLite 支持所有主流操作系统（Windows、macOS、Linux）和编程语言。

## 快速开始

### 创建数据库

```bash
# 命令行创建数据库
sqlite3 myapp.db

# 或使用 Python
python3 -c "import sqlite3; conn = sqlite3.connect('myapp.db')"
```

### 基本操作

```sql
-- 创建表
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT
);

-- 插入数据
INSERT INTO users (name, email) VALUES ('Alice', 'alice@example.com');

-- 查询数据
SELECT * FROM users WHERE name = 'Alice';

-- 更新数据
UPDATE users SET email = 'new@example.com' WHERE id = 1;

-- 删除数据
DELETE FROM users WHERE id = 1;
```

### 启用 WAL 模式

```sql
-- 推荐的生产配置
PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
PRAGMA busy_timeout = 5000;
```
