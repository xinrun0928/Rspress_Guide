## 安全原则

MongoDB 安全需要从多个层面考虑：

| 层面 | 措施 |
|---|---|
| 网络安全 | 防火墙、VPC、网络隔离 |
| 认证 | SCRAM、x.509 证书、LDAP |
| 授权 | 基于角色的访问控制（RBAC） |
| 传输加密 | TLS/SSL |
| 静态加密 | WiredTiger 加密 |
| 审计 | 审计日志 |

## 认证机制

MongoDB 支持多种认证机制：

| 机制 | 说明 | 适用场景 |
|---|---|---|
| SCRAM-SHA-1 | 用户名密码，默认机制 | 通用场景 |
| SCRAM-SHA-256 | 更强的密码验证 | 高安全需求 |
| x.509 | 证书认证 | 企业级部署 |
| LDAP | LDAP 服务器认证 | 企业统一认证 |
| Kerberos | Kerberos 认证 | 大型企业 |
