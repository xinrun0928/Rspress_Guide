# MongoDB 备份与恢复

数据备份是保障数据安全的重要措施。MongoDB 提供了多种备份和恢复方案。

## 备份方法概览

| 方法 | 说明 | 适用场景 |
|---|---|---|
| `mongodump` | 逻辑备份 | 小规模数据、快速备份 |
| 文件系统快照 | 物理备份 | 大规模数据、热备份 |
| Ops Manager | 企业备份 | 生产环境、全面管理 |
| Cloud Backup | 云备份 | 云部署 |

## mongodump / mongorestore

### 基本备份

```bash
# 基本语法
mongodump --uri="mongodb://user:password@localhost:27017/database" --out=/backup/dump

# 备份指定数据库
mongodump --db=appdb --out=/backup/dump

# 备份指定集合
mongodump --db=appdb --collection=orders --out=/backup/dump

# 带压缩备份
mongodump --db=appdb --archive=/backup/appdb.gz --gzip

# 带认证备份
mongodump --uri="mongodb://user:password@localhost:27017/appdb?authSource=admin" --out=/backup/dump
```

### 基本恢复

```bash
# 基本语法
mongorestore --uri="mongodb://user:password@localhost:27017" /backup/dump

# 恢复指定数据库
mongorestore --db=appdb --drop /backup/dump/appdb

# 从压缩文件恢复
mongorestore --archive=/backup/appdb.gz --gzip

# 恢复时重命名数据库
mongorestore --db=newdb --nsFrom="olddb" --nsTo="newdb" /backup/dump/olddb
```

### 增量备份

```bash
# 使用 oplog 进行增量备份

# 1. 全量备份
mongodump --db=appdb --out=/backup/full

# 2. 记录备份时间点
BACKUP_TS=$(date +%s)

# 3. 增量备份
# 等待一段时间后备份 oplog
mongodump --db=local --collection=oplog.rs --query="{ ts: { \$gt: Timestamp($BACKUP_TS, 0) } }" --out=/backup/incremental

# 恢复
mongorestore --db=appdb /backup/full/appdb
mongorestore --db=local --collection=oplog.rs /backup/incremental/local/oplog.rs
```

## 文件系统快照

### LVM 快照

```bash
# 1. 刷新 Journal（确保数据一致）
mongo --eval "db.adminCommand({ fsync: 1 })"

# 2. 创建 LVM 快照
lvcreate --size 10G --snapshot --name mongodb-snap /dev/vg01/lv-mongodb

# 3. 挂载快照
mkdir -p /mnt/snapshot
mount /dev/vg01/mongodb-snap /mnt/snapshot

# 4. 复制数据
rsync -av /mnt/snapshot/var/lib/mongodb /backup/

# 5. 删除快照
umount /mnt/snapshot
lvremove /dev/vg01/mongodb-snap
```

### 云快照

```bash
# AWS EBS 快照
aws ec2 create-snapshot --volume-id vol-1234567890abcdef0 --description "MongoDB backup"

# 恢复
aws ec2 create-volume --snapshot-id snap-1234567890abcdef0 --availability-zone us-east-1a
```

## 副本集备份

### 从从节点备份

```bash
# 备份从节点（不影响主节点性能）
mongodump --host secondary-host:27017 --out=/backup/dump

# 排除 local 数据库
mongodump --excludeCollection=oplog.rs --host secondary-host:27017 --out=/backup/dump
```

### oplog 备份

```bash
# 副本集备份脚本
#!/bin/bash

BACKUP_DIR="/backup"
DATE=$(date +%Y%m%d_%H%M%S)
OPLOG_START=$(date +%s)

# 全量备份
mongodump --host rs0/primary:27017,secondary1:27017,secondary2:27017 \
  --out=${BACKUP_DIR}/${DATE}

# 记录 oplog 时间点
echo "Oplog start: ${OPLOG_START}" > ${BACKUP_DIR}/${DATE}/oplog_start.txt

# 模拟备份时间
sleep 10

OPLOG_END=$(date +%s)
echo "Oplog end: ${OPLOG_END}" > ${BACKUP_DIR}/${DATE}/oplog_end.txt

# 备份 oplog
mongodump --db=local --collection=oplog.rs \
  --query="{ ts: { \$gt: Timestamp(${OPLOG_START}, 0), \$lt: Timestamp(${OPLOG_END}, 0) } }" \
  --out=${BACKUP_DIR}/${DATE}

echo "Backup completed: ${DATE}"
```

## 分片集群备份

### 分片集群备份策略

```bash
# 分片集群备份需要：
# 1. 停止均衡器
sh.stopBalancer()

# 2. 备份 Config Server
mongodump --host config1:27017 --out=/backup/config

# 3. 备份每个 Shard
mongodump --host shard1-primary:27017 --out=/backup/shard1
mongodump --host shard2-primary:27017 --out=/backup/shard2

# 4. 重新开启均衡器
sh.startBalancer()
```

### 完整备份脚本

```bash
#!/bin/bash

BACKUP_DIR="/backup/sharded_cluster"
DATE=$(date +%Y%m%d_%H%M%S)

echo "Starting sharded cluster backup at $DATE"

# 1. 停止均衡器
echo "Stopping balancer..."
mongo --quiet --eval "sh.stopBalancer()"

# 2. 备份 Config Server
echo "Backing up Config Server..."
mongodump --host configReplSet/config1:27017,config2:27017,config3:27017 \
  --out=${BACKUP_DIR}/${DATE}/config

# 3. 备份每个 Shard
for shard in shard0001 shard0002 shard0003; do
  echo "Backing up $shard..."
  mongodump --host ${shard}/${shard}-1:27017,${shard}-2:27017,${shard}-3:27017 \
    --out=${BACKUP_DIR}/${DATE}/${shard}
done

# 4. 重新开启均衡器
echo "Starting balancer..."
mongo --quiet --eval "sh.startBalancer()"

echo "Backup completed: ${DATE}"
```

## 恢复操作

### 时间点恢复

```bash
# 1. 恢复全量备份
mongorestore --drop /backup/full/appdb

# 2. 应用 oplog
mongorestore --db=local --collection=oplog.rs /backup/oplog.bson

# 3. 回放到指定时间点
mongorestore --oplogReplay --oplogLimit="Timestamp($TS, 0)" /backup/dump
```

### 分片集群恢复

```bash
# 1. 恢复 Config Server
mongorestore --host configReplSet/config1:27017,config2:27017,config3:27017 \
  ${BACKUP_DIR}/config

# 2. 恢复每个 Shard
for shard in shard0001 shard0002 shard0003; do
  mongorestore --host ${shard}/${shard}-1:27017,${shard}-2:27017,${shard}-3:27017 \
    ${BACKUP_DIR}/${shard}
done
```

## 备份策略设计

### 备份策略示例

```javascript
// 备份策略设计

// 1. 全量备份：每周一次（周日）
// 2. 增量备份：每天一次
// 3. oplog 备份：每小时一次

// 备份保留策略
// - 每日备份：保留 7 天
// - 每周备份：保留 4 周
// - 每月备份：保留 12 个月
```

### 备份脚本

```bash
#!/bin/bash
# daily_backup.sh

BACKUP_ROOT="/backup/mongodb"
DATE=$(date +%Y%m%d)
TIMESTAMP=$(date +%s)

# 创建备份目录
mkdir -p ${BACKUP_ROOT}/daily/${DATE}

# 全量备份
mongodump --uri="mongodb://localhost:27017/appdb?authSource=admin" \
  --out=${BACKUP_ROOT}/daily/${DATE}

# 记录 oplog 时间点
mongosh --quiet --eval "printjson(db.getSiblingDB('local').oplog.rs.find().sort({ts:-1}).limit(1).toArray()[0].ts)" \
  > ${BACKUP_ROOT}/daily/${DATE}/oplog_end.txt

# 压缩备份
tar -czf ${BACKUP_ROOT}/daily/${DATE}.tar.gz -C ${BACKUP_ROOT}/daily ${DATE}
rm -rf ${BACKUP_ROOT}/daily/${DATE}

# 删除 7 天前的备份
find ${BACKUP_ROOT}/daily -name "*.tar.gz" -mtime +7 -delete

echo "Daily backup completed: ${DATE}"
```

## 验证备份

```bash
# 验证备份完整性
mongorestore --dryRun --db=test /backup/dump/appdb

# 测试恢复
mongorestore --db=test /backup/dump/appdb

# 验证数据
mongo --quiet --eval "
  db = db.getSiblingDB('test');
  print('Collections:', db.getCollectionNames().length);
  print('Documents:', db.users.count());
"
```

## 总结

备份恢复要点：

- **mongodump**：逻辑备份，适合小规模数据
- **文件系统快照**：物理备份，适合大规模数据
- **oplog**：增量备份，支持时间点恢复
- **分片集群备份**：需要备份 Config Server 和所有 Shard
- **备份策略**：全量 + 增量，定期验证备份有效性

制定完善的备份策略是保障数据安全的关键。
