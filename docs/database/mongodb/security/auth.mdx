# MongoDB 用户角色与权限管理

正确管理用户和权限是保障 MongoDB 安全的关键。本节详细介绍用户创建、权限配置和最佳实践。

## 用户管理基础

### 用户存储位置

```javascript
// 用户信息存储在 admin 数据库
// local 数据库也存储副本集成员信息

// 查看所有用户
db.getSiblingDB("admin").getUsers()

// 查看当前用户
db.runCommand({ connectionStatus: 1 }).user
```

### 创建用户语法

```javascript
// 基本语法
db.getSiblingDB("admin").createUser({
  user: "username",
  pwd: "password",
  roles: [
    { role: "roleName", db: "databaseName" },
    "builtInRole"
  ],
  mechanisms: ["SCRAM-SHA-1", "SCRAM-SHA-256"],  // 可选
  passwordDigestor: "server"  // 或 "client"
})
```

## 内置角色详解

### 数据库角色

| 角色 | 数据库级别权限 |
|---|---|
| `read` | find, collStats, dbStats, PlanCache.readPlans |
| `readWrite` | read + insert, update, remove, createCollection, dropCollection |
| `dbAdmin` | collStats, dbHash, dbStats, find, killCursors, PlanCache.readPlans |
| `dbOwner` | readWrite + dbAdmin + userAdmin |
| `userAdmin` | createUser, dropUser, grantRole, revokeRole, viewUser |

### 集群角色

| 角色 | 说明 |
|---|---|
| `clusterAdmin` | 所有集群管理操作 |
| `clusterManager` | 集群监控和管理 |
| `clusterMonitor` | 只读集群监控 |
| `hostManager` | 主机管理 |
| `backup` | 备份操作 |
| `restore` | 恢复操作 |

### 特殊角色

```javascript
// readAnyDatabase: 所有数据库的只读权限
// readWriteAnyDatabase: 所有数据库的读写权限
// userAdminAnyDatabase: 所有数据库的用户管理
// dbAdminAnyDatabase: 所有数据库的管理操作
// root: 所有权限
```

## 用户创建示例

### 管理员用户

```javascript
// 创建超级管理员
db.getSiblingDB("admin").createUser({
  user: "siteAdmin",
  pwd: "strong_password_here",
  roles: [
    { role: "root", db: "admin" }
  ]
})

// 创建数据库管理员
db.getSiblingDB("admin").createUser({
  user: "dbAdmin",
  pwd: "dbadmin_password",
  roles: [
    { role: "dbOwner", db: "admin" }
  ]
})
```

### 应用用户

```javascript
// 为订单服务创建用户
db.getSiblingDB("admin").createUser({
  user: "orders_service",
  pwd: "orders_password",
  roles: [
    { role: "readWrite", db: "orders" },
    { role: "read", db: "products" },
    { role: "read", db: "customers" }
  ]
})

// 为报表服务创建只读用户
db.getSiblingDB("admin").createUser({
  user: "reporting_service",
  pwd: "reporting_password",
  roles: [
    { role: "read", db: "orders" },
    { role: "read", db: "products" },
    { role: "read", db: "customers" }
  ]
})

// 为分析服务创建只读用户（所有数据库）
db.getSiblingDB("admin").createUser({
  user: "analytics_service",
  pwd: "analytics_password",
  roles: [
    { role: "readAnyDatabase", db: "admin" },
    { role: "read", db: "local" }
  ]
})
```

### 分片集群用户

```javascript
// 分片集群管理员
db.getSiblingDB("admin").createUser({
  user: "shardAdmin",
  pwd: "shardadmin_password",
  roles: [
    { role: "clusterAdmin", db: "admin" },
    { role: "userAdminAnyDatabase", db: "admin" }
  ]

// 分片应用用户

  user: "shardAppUser",
  pwd: "shardapp_password",

```

## 权限管理

### 查看用户权限

```javascript
// 查看用户信息（包括角色）
db.getSiblingDB("admin").getUser("appuser")

// 查看角色权限
db.getSiblingDB("admin").getRole("readWrite", { showPrivileges: true })
```

### 授予角色

```javascript
// 授予数据库角色
db.getSiblingDB("admin").grantRolesToUser("appuser", [
  { role: "readWrite", db: "newdb" }
])

// 授予集群角色
db.getSiblingDB("admin").grantRolesToUser("monitorUser", [
  { role: "clusterMonitor", db: "admin" }
])
```

### 撤销角色

```javascript
// 撤销用户角色
db.getSiblingDB("admin").revokeRolesFromUser("appuser", [
  { role: "read", db: "reporting" }
])
```

### 更新用户密码

```javascript
// 更新密码
db.getSiblingDB("admin").changeUserPassword("appuser", "new_strong_password")

// 或者
db.getSiblingDB("admin").updateUser("appuser", {
  pwd: "new_strong_password"
})
```

## 自定义角色

### 创建自定义角色

```javascript
// 创建数据分析师角色
db.getSiblingDB("admin").createRole({
  role: "dataAnalyst",
  privileges: [
    {
      resource: { db: "appdb", collection: "orders" },
      actions: ["find", "aggregate"]
    },
    {
      resource: { db: "appdb", collection: "customers" },
      actions: ["find", "aggregate"]
    },
    {
      resource: { db: "appdb", collection: "products" },
      actions: ["find", "aggregate"]
    }
  ],
  roles: []
})

// 创建备份操作员角色
db.getSiblingDB("admin").createRole({
  role: "backupOperator",
  privileges: [
    { resource: { db: "", collection: "" }, actions: ["backup"] },
    { resource: { db: "", collection: "" }, actions: ["restore"] }
  ],
  roles: []
})
```

### 角色继承

```javascript
// 创建高级开发角色（继承 + 额外权限）
db.getSiblingDB("admin").createRole({
  role: "seniorDeveloper",
  privileges: [
    {
      resource: { db: "appdb", collection: "logs" },
      actions: ["dropCollection", "dropIndex"]
    }
  ],
  roles: [
    { role: "readWrite", db: "appdb" }
  ]
})
```

## 用户认证

### SCRAM 认证

```javascript
// SCRAM-SHA-256（MongoDB 4.0+ 默认）
// 用户必须支持 SCRAM-SHA-256
db.getSiblingDB("admin").createUser({
  user: "secureUser",
  pwd: "very_strong_password",
  roles: [{ role: "read", db: "appdb" }],
  mechanisms: ["SCRAM-SHA-256"]
})

// 客户端连接
// mongo "mongodb://secureUser:very_strong_password@localhost:27017/appdb/?authSource=admin&authMechanism=SCRAM-SHA-256"
```

### 副本集内部认证

```javascript
// 副本集/分片集群内部认证

// 1. 创建密钥文件
openssl rand -base64 756 > /path/to/keyfile
chmod 400 /path/to/keyfile

// 2. 配置 mongod
// mongod.conf
security:
  authorization: enabled
  keyFile: /path/to/keyfile

// 3. 重启所有成员
```

## 用户维护脚本

### 批量创建用户

```javascript
// 批量创建应用用户
const users = [
  {
    user: "order_service",
    db: "admin",
    pwd: "order_password",
    roles: [
      { role: "readWrite", db: "orders" },
      { role: "read", db: "products" }
    ]
  },
  {
    user: "payment_service",
    db: "admin",
    pwd: "payment_password",
    roles: [
      { role: "readWrite", db: "payments" }
    ]
  },
  {
    user: "analytics_service",
    db: "admin",
    pwd: "analytics_password",
    roles: [
      { role: "readAnyDatabase", db: "admin" }
    ]
  }
];

users.forEach(userConfig => {
  try {
    db.getSiblingDB(userConfig.db).createUser({
      user: userConfig.user,
      pwd: userConfig.pwd,
      roles: userConfig.roles
    });
    print(`Created user: ${userConfig.user}`);
  } catch (e) {
    print(`Error creating ${userConfig.user}: ${e.message}`);
  }
});
```

### 用户审计

```javascript
// 导出所有用户
function exportUsers() {
  const users = db.getSiblingDB("admin").getUsers();

  const exportData = users.map(user => ({
    user: user.user,
    db: user.db,
    roles: user.roles,
    mechanisms: user.mechanisms,
    createdAt: user.createdAt
  }));

  printjson(exportData);
  return exportData;
}

// 检查未使用的角色
function findUnusedRoles() {
  const allRoles = db.getSiblingDB("admin").getRoles({ rolesInfo: 1, showPrivileges: false });
  const builtInRoles = ["read", "readWrite", "readAnyDatabase", "readWriteAnyDatabase",
                        "dbAdmin", "dbAdminAnyDatabase", "userAdmin", "userAdminAnyDatabase",
                        "clusterAdmin", "root"];

  allRoles.forEach(role => {
    if (!builtInRoles.includes(role.role)) {
      print(`Custom role: ${role.role} in ${role.db}`);
    }
  });
}
```

## 最佳实践

### 用户管理原则

```javascript
// 1. 最小权限原则
// ❌ 错误：应用使用 root 用户
db.getSiblingDB("admin").createUser({
  user: "app",
  pwd: "password",
  roles: [{ role: "root", db: "admin" }]
});

// ✅ 正确：只授予必要的权限
db.getSiblingDB("admin").createUser({
  user: "app",
  pwd: "password",
  roles: [
    { role: "readWrite", db: "appdb" }
  ]
});

// 2. 不同环境不同用户
// 开发环境、测试环境、生产环境使用不同用户

// 3. 定期更新密码
db.getSiblingDB("admin").changeUserPassword("appuser", "new_password")

// 4. 禁用或删除无用用户
db.getSiblingDB("admin").dropUser("unused_user")

// 5. 使用强密码
// 长度 >= 16 位，包含大小写、数字、特殊字符
```

### 密码策略

```javascript
// MongoDB Server 6.0+ 支持密码复杂度验证

// mongod.conf
security:
  passwordValidation:
    forbidWeakCiphers: true
    minLength: 16
    requireUppercase: true
    requireLowercase: true
    requireDigits: true
    requireSymbols: true
```

## 总结

用户管理要点：

- **用户存储**：admin 数据库
- **内置角色**：read、readWrite、dbAdmin、userAdmin、clusterAdmin、root
- **自定义角色**：根据业务需求创建
- **最小权限**：应用用户只授予必要权限
- **定期维护**：更新密码、清理无用用户

正确的用户和权限管理是 MongoDB 安全的基础。
