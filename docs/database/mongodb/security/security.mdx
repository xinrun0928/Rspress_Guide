# MongoDB 认证与授权

MongoDB 提供了完整的认证和授权机制，确保数据安全访问。

## 认证机制

### SCRAM 认证

SCRAM（Salt Challenge Response Authentication Mechanism）是 MongoDB 默认的认证机制。

```javascript
// SCRAM-SHA-1（MongoDB 3.0-4.0 默认）
// mongod.conf
security:
  authorization: enabled

// SCRAM-SHA-256（MongoDB 4.0+ 默认）
// mongod.conf
security:
  authorization: enabled
  authenticationMechanisms:
    - SCRAM-SHA-256
```

### x.509 证书认证

企业级环境推荐使用 x.509 证书认证。

```javascript
// 生成证书
// 1. 创建 CA
openssl genrsa -out ca.key 4096
openssl req -x509 -new -nodes -key ca.key -sha256 -days 3650 -out ca.pem

// 2. 生成服务器证书
openssl genrsa -out server.key 4096
openssl req -new -key server.key -out server.csr
openssl x509 -req -in server.csr -CA ca.pem -CAkey ca.key -CAcreateserial -out server.pem -days 365 -sha256

// 配置 mongod
// mongod.conf
net:
  port: 27017
  ssl:
    mode: requireSSL
    PEMKeyFile: /path/to/server.pem
    CAFile: /path/to/ca.pem

security:
  clusterAuthMode: x509
```

### LDAP 认证

企业统一认证可以使用 LDAP。

```javascript
// mongod.conf
security:
  authorization: "enabled"
  ldap:
    servers: "ldap.example.com:389"
    bind:
      method: "simple"
      queryUser: "cn=admin,dc=example,dc=com"
      queryPassword: "password"
    userToDNMapping:
      - match: "(.+)"
        ldapToDN:
          - base: "ou=users,dc=example,dc=com"
            filter: "(uid={0})"
    authz:
      queryTemplate: "ou=groups,dc=example,dc=com?member?uid={USER}"
```

## 启用认证

### 基本配置

```javascript
// mongod.conf
security:
  authorization: enabled  # 启用认证

net:
  port: 27017
  bindIp: 127.0.0.1  # 仅本地访问，生产环境应改为具体 IP
```

### 连接字符串

```javascript
// 无认证连接
mongodb://localhost:27017

// 用户名密码连接
mongodb://username:password@localhost:27017/?authSource=admin

// 分片集群连接
mongodb://mongos1:27017,mongos2:27017/?authSource=admin
```

## 基于角色的访问控制

### 内置角色

| 角色 | 说明 | 权限 |
|---|---|---|
| `read` | 读取所有非系统集合 | find,collStats,dbStats |
| `readWrite` | 读写所有非系统集合 | read + insert,update,remove |
| `dbAdmin` | 数据库管理 | dbStats,collStats,indexStats |
| `clusterAdmin` | 集群管理 | 所有集群操作 |
| `userAdmin` | 用户管理 | createUser,dropUser,grantRole |
| `root` | 超级管理员 | 所有权限 |

### 数据库级别角色

```javascript
// readAnyDatabase: 读取所有数据库
// readWriteAnyDatabase: 读写所有数据库
// dbAdminAnyDatabase: 管理所有数据库
// userAdminAnyDatabase: 管理所有数据库用户
```

## 用户管理

### 创建用户

```javascript
// 创建管理员用户
db.getSiblingDB("admin").createUser({
  user: "admin",
  pwd: "secure_password",
  roles: [
    { role: "root", db: "admin" }
  ]
})

// 创建应用用户
db.getSiblingDB("appdb").createUser({
  user: "appuser",
  pwd: "app_password",
  roles: [
    { role: "readWrite", db: "appdb" },
    { role: "read", db: "reporting" }
  ]
})

// 创建只读用户
db.getSiblingDB("appdb").createUser({
  user: "readonly",
  pwd: "readonly_password",
  roles: [
    { role: "read", db: "appdb" }
  ]
})
```

### 修改用户

```javascript
// 更新用户密码
db.getSiblingDB("admin").updateUser("admin", {
  pwd: "new_password"
})

// 更新用户角色
db.getSiblingDB("admin").grantRolesToUser("appuser", [
  { role: "readWrite", db: "anotherdb" }
])

// 删除用户角色
db.getSiblingDB("admin").revokeRolesFromUser("appuser", [
  { role: "read", db: "reporting" }
])
```

### 删除用户

```javascript
// 删除用户
db.getSiblingDB("admin").dropUser("appuser")
```

## 权限管理

### 资源权限

```javascript
// 数据库和集合权限
// { db: "mydb", collection: "mycoll" }
// { db: "mydb", collection: "" }  # 数据库所有集合
// { db: "", collection: "" }      # 所有数据库
```

### 自定义角色

```javascript
// 创建自定义角色
db.getSiblingDB("admin").createRole({
  role: "appAdmin",
  privileges: [
    {
      resource: { db: "appdb", collection: "" },
      actions: ["find", "insert", "update", "remove"]
    },
    {
      resource: { db: "appdb", collection: "logs" },
      actions: ["find", "insert", "update", "remove", "dropCollection"]
    }
  ],
  roles: []
})

// 为用户分配自定义角色
db.getSiblingDB("admin").createUser({
  user: "app_admin",
  pwd: "password",
  roles: ["appAdmin"]
})
```

### 角色继承

```javascript
// 创建继承其他角色的角色
db.getSiblingDB("admin").createRole({
  role: "seniorDeveloper",
  privileges: [],
  roles: [
    { role: "readWrite", db: "appdb" },
    { role: "dbAdmin", db: "appdb" }
  ]
})
```

## TLS/SSL 配置

### 服务端 TLS

```javascript
// mongod.conf
net:
  port: 27017
  ssl:
    mode: requireSSL
    PEMKeyFile: /path/to/server.pem
    CAFile: /path/to/ca.pem
    clusterFile: /path/to/cluster.pem  # 副本集集群证书
```

### 客户端 TLS

```javascript
// mongo shell 连接
mongo --ssl --sslCAFile /path/to/ca.pem --sslPEMKeyFile /path/to/client.pem

// MongoDB URI
mongodb://localhost:27017/?ssl=true&sslCAFile=/path/to/ca.pem
```

## 审计日志

### 配置审计

```javascript
// mongod.conf
security:
  auditLog:
    destination: file
    format: JSON
    path: /var/log/mongodb/audit.log
    # 或者 syslog
    # destination: syslog
```

### 审计事件

```javascript
// 审计的操作类型
// authenticate: 认证
// createUser: 创建用户
// dropUser: 删除用户
// dropRole: 删除角色
// grantRole: 授予角色
// revokeRole: 撤销角色
// createRole: 创建角色
// updateUser: 更新用户
// createCollection: 创建集合
// dropCollection: 删除集合
```

## 最佳实践

### 安全配置清单

```javascript
// 1. 启用认证
security:
  authorization: enabled

// 2. 使用强密码
// 长度 >= 12 位，包含大小写、数字、特殊字符

// 3. 最小权限原则
// 应用用户只授予必要的权限

// 4. 网络隔离
// 使用防火墙，限制访问 IP
net:
  bindIp: 192.168.1.0/24

// 5. 启用 TLS
net:
  ssl:
    mode: requireSSL

// 6. 定期轮换密码和证书

// 7. 启用审计
security:
  auditLog:
    destination: file

// 8. 监控安全事件
// 关注异常登录、权限变更等
```

## 总结

MongoDB 安全要点：

- **认证机制**：SCRAM（默认）、x.509、LDAP
- **角色**：内置角色和自定义角色
- **权限**：基于资源的权限控制
- **传输加密**：TLS/SSL
- **审计**：记录所有敏感操作

安全配置是生产环境 MongoDB 部署的必要步骤。
