## 核心概念

MongoDB 4.0 引入了多文档事务支持，4.2 进行了重大改进，使得事务的使用更加简单和高效。

| 版本 | 事务特性 |
|---|---|
| 4.0 | 支持副本集内的多文档事务 |
| 4.2 | 支持分片集群的多文档事务 |
| 5.0+ | 事务默认开启，无法禁用 |

## 为什么需要事务

MongoDB 的文档模型允许在一个文档内存储相关数据，大多数情况下**单文档操作是原子的**。但跨文档、跨集合的业务操作需要事务来保证一致性。

```javascript
// 转账场景：需要原子性
// 从账户A扣款，同时给账户B存款
// 单文档无法完成，需要多文档事务

// 订单场景：需要一致性
// 创建订单 + 扣减库存 + 更新用户积分
// 三个操作要么全部成功，要么全部回滚
```
