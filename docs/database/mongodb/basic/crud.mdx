# MongoDB CRUD 操作

「MongoDB 的增删改查和 SQL 有什么区别？」

这是初学者的第一个问题。MongoDB 的 CRUD 操作比 SQL 更直观——不需要写冗长的 INSERT 语句，也不需要 JOIN，一个方法搞定一切。

但别被简单迷惑了。MongoDB 的更新操作有多种模式，批量更新和原子操作是生产环境的必修课。

## 插入操作

### 插入单条文档

```javascript
// 基本插入
db.users.insertOne({
  name: "张三",
  age: 25,
  email: "zhangsan@example.com"
})

// 或者使用 insert（已废弃，但还能用）
db.users.insert({
  name: "李四",
  age: 30
})
```

### 插入多条文档

```javascript
// 批量插入
db.users.insertMany([
  { name: "王五", age: 28, city: "北京" },
  { name: "赵六", age: 35, city: "上海" },
  { name: "钱七", age: 22, city: "深圳" }
])

// 返回插入结果
// { "acknowledged" : true, "insertedIds" : [ ObjectId(...), ... ] }
```

:::tip insertMany 的原子性

`insertMany` 是原子操作——要么全部成功，要么全部失败。如果需要部分成功，使用 `insertMany` 的 `ordered: false` 选项。
:::

```javascript
// unordered 模式：继续插入即使部分失败
db.users.insertMany([
  { _id: 1, name: "重复ID" },    // 会失败
  { name: "钱七", age: 22 }        // 会继续尝试
], { ordered: false })
```

## 查询操作

### 基础查询

```javascript
// 查询所有文档
db.users.find()

// 条件查询：等价于 WHERE
db.users.find({ age: 25 })

// 多条件：AND
db.users.find({ age: 25, city: "北京" })

// OR 查询
db.users.find({
  $or: [
    { age: { $lt: 25 } },
    { city: "上海" }
  ]
})
```

### 比较运算符

| 运算符 | 说明 | SQL 等价 |
|---|---|---|
| `$eq` | 等于 | `=` |
| `$ne` | 不等于 | `!=` |
| `$gt` | 大于 | `>` |
| `$gte` | 大于等于 | `>=` |
| `$lt` | 小于 | `<` |
| `$lte` | 小于等于 | `<=` |
| `$in` | 在数组中 | `IN (...)` |
| `$nin` | 不在数组中 | `NOT IN (...)` |

```javascript
// 年龄大于等于 25 且小于 40
db.users.find({
  age: { $gte: 25, $lt: 40 }
})

// 城市在上海或北京
db.users.find({
  city: { $in: ["上海", "北京"] }
})
```

### 查询投影

只返回需要的字段，减少网络传输：

```javascript
// 只返回 name 和 email，不返回 _id
db.users.find(
  { age: { $gte: 25 } },
  { name: 1, email: 1, _id: 0 }
)

// 结果
// { "name" : "张三", "email" : "zhangsan@example.com" }
```

### 查询方法

```javascript
// 查询单条
db.users.findOne({ name: "张三" })

// 排序：1 升序，-1 降序
db.users.find().sort({ age: -1, name: 1 })

// 跳过和限制
db.users.find().skip(10).limit(5)

// 统计数量
db.users.countDocuments({ age: { $gte: 25 } })

// 检查是否存在
db.users.findOne({ name: "张三" }) !== null
```

:::tip 分页查询

分页的正确姿势是使用 `skip()` + `limit()`，或者更好的是使用**游标游标**（基于 `_id` 或索引字段）进行游标分页，避免 skip 大数据量时的性能问题。
:::

### 数组查询

```javascript
// 文档结构
{ name: "张三", tags: ["Java", "MongoDB", "Redis"] }

// 查询数组包含某元素
db.users.find({ tags: "MongoDB" })

// 查询数组同时包含多个元素（顺序无关）
db.users.find({
  tags: { $all: ["Java", "MongoDB"] }
})

// 按数组长度查询
db.users.find({ tags: { $size: 3 } })
```

## 更新操作

### 基本更新

```javascript
// updateOne：更新匹配的第一条
db.users.updateOne(
  { name: "张三" },
  { $set: { age: 26, city: "广州" } }
)

// updateMany：更新匹配的所有文档
db.users.updateMany(
  { age: { $lt: 25 } },
  { $set: { status: "未成年" } }
)

// replaceOne：替换整个文档（除了 _id）
db.users.replaceOne(
  { name: "张三" },
  { name: "张三", age: 26, city: "广州", email: "new@example.com" }
)
```

### 更新运算符

| 运算符 | 说明 | 示例 |
|---|---|---|
| `$set` | 设置字段值 | `{ $set: { age: 26 } }` |
| `$unset` | 删除字段 | `{ $unset: { email: "" } }` |
| `$inc` | 递增/递减 | `{ $inc: { age: 1 } }` |
| `$mul` | 乘 | `{ $mul: { price: 2 } }` |
| `$rename` | 重命名字段 | `{ $rename: { old_name: "new_name" } }` |
| `$setOnInsert` | 仅在插入时设置 | - |

```javascript
// 原子递增：点赞数 +1
db.articles.updateOne(
  { _id: ObjectId("...") },
  { $inc: { likes: 1 } }
)

// 如果不存在则设置，存在则不变
db.users.updateOne(
  { name: "新用户" },
  {
    $set: { name: "新用户", created_at: new Date() },
    $setOnInsert: { status: "active" }
  },
  { upsert: true }
)
```

### 数组更新

```javascript
// 数组添加元素
db.users.updateOne(
  { name: "张三" },
  { $push: { tags: "Golang" } }
)

// 数组批量添加（$each）
db.users.updateOne(
  { name: "张三" },
  { $push: { tags: { $each: ["Golang", "Python"] } } }
)

// 数组删除元素
db.users.updateOne(
  { name: "张三" },
  { $pull: { tags: "Java" } }
)

// 按位置更新数组元素
db.users.updateOne(
  { name: "张三", "scores.0": { $exists: true } },
  { $set: { "scores.0": 95 } }
)

// 数组元素作为查询条件时，可以直接用索引更新
db.users.updateOne(
  { name: "张三", scores: { $gte: 60 } },
  { $set: { "scores.$": 65 } }
)
```

:::warning 更新文档时注意字段类型

更新时 `$set` 会替换整个字段。如果原字段是数组，新值是数字，会导致类型变化。MongoDB 不进行类型强制转换。
:::

### 原子操作与竞态条件

多个客户端同时更新同一文档时，需要注意原子性：

```javascript
// ❌ 错误：读-改-写，存在竞态条件
const user = db.users.findOne({ name: "张三" })
user.score += 10
db.users.updateOne({ name: "张三" }, { $set: { score: user.score } })

// ✅ 正确：使用原子操作
db.users.updateOne(
  { name: "张三" },
  { $inc: { score: 10 } }
)

// ✅ 更安全：使用条件更新
db.users.updateOne(
  { name: "张三", score: { $gte: 0 } },  // 前置条件
  { $inc: { score: -5 } }                 // 扣款
)
```

## 删除操作

```javascript
// 删除单条
db.users.deleteOne({ name: "张三" })

// 删除多条
db.users.deleteMany({ status: "inactive", last_login: { $lt: ISODate("2024-01-01") } })

// 删除集合（慎用！）
db.users.deleteMany({})  // 删除所有文档
db.users.drop()           // 删除整个集合和索引

// 删除数据库
db.dropDatabase()
```

:::danger 删除操作不可逆

删除操作不会进入回收站。在生产环境中执行删除前，务必确认条件正确。建议先用 `find()` 确认文档范围，再执行删除。
:::

## 批量操作与性能

### 有序 vs 无序执行

```javascript
// 有序执行（默认）：一条失败后停止
db.users.insertMany([
  { _id: 1, name: "A" },
  { _id: 2, name: "B" }
])

// 无序执行：并行处理，忽略失败
db.users.insertMany([
  { _id: 1, name: "A" },
  { _id: 2, name: "B" }
], { ordered: false })
```

### 批量写入配置

```javascript
// 大批量插入时的配置
const bulkOps = [
  { insertOne: { document: { name: "用户1", data: "..." } } },
  { insertOne: { document: { name: "用户2", data: "..." } } },
  { updateOne: { filter: { name: "用户1" }, update: { $set: { status: "active" } } } },
  { deleteOne: { filter: { name: "旧用户" } } }
]

db.users.bulkWrite(bulkOps, { ordered: false })
```

## 面试实战

**面试官问**：「MongoDB 的更新是原子操作吗？」

**深入回答**：

> 单个文档的更新是原子操作。但多个文档的 `updateMany` 不是——它会逐个更新，中途失败不会回滚已完成的部分。
>
> 跨文档事务（Multi-Document ACID Transaction）是 4.0 引入的功能，但性能开销较大，应尽量用单文档原子操作代替。

**面试官问**：「如何实现点赞计数的并发安全？」

**深入回答**：

> 使用 `$inc` 运算符，它是原子操作：
>
> ```javascript
> db.articles.updateOne(
>   { _id: ObjectId("...") },
>   { $inc: { likes: 1 } }
> )
> ```
>
> 不要先读取 `likes` 值，再写回去。这样会产生竞态条件。

## 总结

MongoDB 的 CRUD 操作核心要点：

- **插入**：`insertOne`、`insertMany`，注意原子性和 `ordered` 选项
- **查询**：`find`、`findOne`，熟练使用比较运算符和投影
- **更新**：`updateOne`、`updateMany`，使用 `$set`、`$inc` 等运算符
- **删除**：`deleteOne`、`deleteMany`，删除操作不可逆
- **原子性**：单文档操作是原子的，跨文档需要事务
