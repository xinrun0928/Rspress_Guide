# MongoDB 基础入门

MongoDB 是最流行的 NoSQL 数据库之一，以灵活的文档模型、强大的查询能力和水平扩展能力著称。与传统关系型数据库不同，MongoDB 使用 JSON 类似的文档存储数据，无需预定义表结构，让开发更加敏捷。

本章节将带你从零认识 MongoDB，包括：

- MongoDB 的核心概念与数据模型
- CRUD 操作的基本语法
- BSON 数据类型与文档结构
- 与关系型数据库的概念对比

## 基本概念

| MongoDB | 关系型数据库 |
|---|---|
| Database | Database |
| Collection | Table |
| Document | Row |
| Field | Column |
| Index | Index |
| Embedded Document | Join |
| `$lookup` | JOIN |

MongoDB 的核心优势在于**文档嵌入**——可以将相关数据直接嵌套在一个文档中，减少跨文档查询。而 Sharded Cluster 架构则让 MongoDB 能够水平扩展，处理海量数据。

---

## 应用场景

MongoDB 适合以下场景：

- **内容管理系统**：文档结构天然适合存储文章、评论等半结构化数据
- **实时分析**：聚合管道支持复杂的统计分析
- **物联网数据**：时序数据的高效写入
- **移动应用后端**：灵活的数据模型适应快速迭代

不适合的场景：强事务需求（如银行转账）、高度规范化存储、复杂多表关联查询。

## 学习路径

建议按以下顺序学习 MongoDB：

1. 先掌握 CRUD 操作，理解文档的增删改查
2. 学习索引原理，这是性能优化的基础
3. 理解聚合管道，掌握数据分析能力
4. 学习副本集和分片集群，理解高可用架构
