# MongoDB 聚合管道

「MongoDB 的聚合管道是怎么工作的？为什么比 MapReduce 快？」

这是面试中经常被问到的问题。大多数人知道 `aggregate()` 可以做数据统计，但追问下去——「管道各阶段是怎么执行的？」「 `$lookup` 能不能做 join？」「 `$group` 的内存限制是多少？」「如何优化聚合查询的性能？」——能答上来的就很少了。

本文深入讲解 MongoDB 聚合管道的原理和实践，帮你彻底理解这个强大的数据处理框架。

## 聚合管道基础

### 什么是聚合管道

聚合管道（Aggregation Pipeline）是一个**数据处理流水线**——文档从管道的一端进入，经过多个阶段的处理，从另一端输出结果。

```javascript
// 聚合管道的基本结构
db.orders.aggregate([
  { $match: { status: "completed" } },    // 阶段1：过滤
  { $group: { _id: "$customer_id",       // 阶段2：分组
              total: { $sum: "$amount" } }},
  { $sort: { total: -1 } },               // 阶段3：排序
  { $limit: 10 }                          // 阶段4：限制
])
```

每个阶段称为一个**管道阶段**（Pipeline Stage），文档在管道中**顺序流动**，前一个阶段的输出作为后一个阶段的输入。

### 管道 vs MapReduce

| 特性 | 聚合管道 | MapReduce |
|---|---|---|
| 语法 | 声明式，简洁直观 | JavaScript 函数，较复杂 |
| 性能 | 优化器自动优化 | 需要手动优化 |
| 灵活性 | 受限于内置阶段 | 支持自定义 JS 函数 |
| 适用场景 | 简单到中等复杂度 | 复杂自定义逻辑 |
| 并行执行 | 自动并行 | 手动配置 |

MongoDB 推荐优先使用聚合管道，只有在管道无法满足需求时才考虑 MapReduce。

## 常用管道阶段

### `$match`：过滤阶段

`$match` 用于筛选符合条件的文档，应该**尽早放置在管道前部**，可以减少后续阶段处理的数据量。

```javascript
// 基本用法：过滤已完成订单
db.orders.aggregate([
  { $match: { status: "completed" } }
])

// 复杂条件：AND 和 OR
db.orders.aggregate([
  {
    $match: {
      $and: [
        { status: { $in: ["completed", "shipped"] } },
        { amount: { $gte: 100 } },
        { created_at: { $gte: ISODate("2024-01-01") } }
      ]
    }
  }
])
```

:::tip 优化原则

`$match` 尽量放在管道前部，原因：

1. 减少后续阶段需要处理的数据量
2. 如果 `$match` 使用了索引，MongoDB 可以**索引覆盖**
3. 越早过滤，数据越小，后续处理越快

:::

### `$group`：分组阶段

`$group` 是聚合管道的核心阶段，用于对文档进行分组统计。

```javascript
// 按客户分组，计算订单总额和数量
db.orders.aggregate([
  {
    $group: {
      _id: "$customer_id",           // 分组字段
      total_amount: { $sum: "$amount" },  // 累加金额
      order_count: { $sum: 1 },           // 计数
      avg_amount: { $avg: "$amount" },     // 平均金额
      max_amount: { $max: "$amount" },     // 最大金额
      min_amount: { $min: "$amount" }      // 最小金额
    }
  }
])
```

#### 分组操作符一览

| 操作符 | 说明 | 示例 |
|---|---|---|
| `$sum` | 求和，可累加字段值或计数 | `{ $sum: "$amount" }` 或 `{ $sum: 1 }` |
| `$avg` | 计算平均值 | `{ $avg: "$amount" }` |
| `$min` | 最小值 | `{ $min: "$amount" }` |
| `$max` | 最大值 | `{ $max: "$amount" }` |
| `$push` | 将值推入数组 | `{ $push: "$product_id" }` |
| `$addToSet` | 将值推入数组（去重） | `{ $addToSet: "$category" }` |
| `$first` | 获取分组第一条 | `{ $first: "$created_at" }` |
| `$last` | 获取分组最后一条 | `{ $last: "$status" }` |

### `$sort`：排序阶段

`$sort` 对文档进行排序，可以按一个或多个字段排序。

```javascript
// 按金额降序排序
db.orders.aggregate([
  { $sort: { amount: -1 } }
])

// 多字段排序：先按状态，再按金额
db.orders.aggregate([
  { $sort: { status: 1, amount: -1 } }
])
```

:::warning 内存限制

`$sort` 默认使用 **100MB 内存限制**。超过限制会报错或使用磁盘临时文件（性能下降）。

可以通过 `{ $sort: { field: 1 }, allowDiskUse: true }` 允许使用磁盘，但建议尽量在内存中完成。

:::

### `$limit` 和 `$skip`：限制与跳过

```javascript
// 分页查询：第2页，每页10条
db.orders.aggregate([
  { $sort: { created_at: -1 } },
  { $skip: 10 },
  { $limit: 10 }
])
```

### `$project`：投影阶段

`$project` 用于选择或排除字段，重命名字段，添加计算字段。

```javascript
// 只保留需要的字段，排除 _id
db.orders.aggregate([
  {
    $project: {
      _id: 0,
      order_id: "$_id",
      customer: "$customer_id",
      total: "$amount",
      created_year: { $year: "$created_at" }
    }
  }
])

// 结果示例
// { "order_id": ObjectId(...), "customer": "C001", "total": 150, "created_year": 2024 }
```

### `$unwind`：展开数组

`$unwind` 将数组字段展开为多个文档，每个元素一个文档。

```javascript
// 原始文档
{ _id: 1, name: "张三", orders: ["A001", "A002", "A003"] }

// unwind 后
{ _id: 1, name: "张三", orders: "A001" }
{ _id: 1, name: "张三", orders: "A002" }
{ _id: 1, name: "张三", orders: "A003" }
```

```javascript
// 基本用法
db.orders.aggregate([
  { $unwind: "$items" }
])

// 保留数组为 null 或 empty 的文档
db.orders.aggregate([
  {
    $unwind: {
      path: "$items",
      preserveNullAndEmptyArrays: true
    }
  }
])

// 添加索引字段
db.orders.aggregate([
  {
    $unwind: {
      path: "$items",
      includeArrayIndex: "item_index"
    }
  }
])
```

### `$lookup`：跨集合关联

`$lookup` 是 MongoDB 实现「JOIN」的核心阶段（MongoDB 4.2+）。

```javascript
// 基本语法：左外连接
db.orders.aggregate([
  {
    $lookup: {
      from: "customers",           // 关联的集合
      localField: "customer_id",  // 本集合的字段
      foreignField: "_id",        // 关联集合的字段
      as: "customer_info"         // 输出字段名
    }
  }
])

// 示例结果
// {
//   _id: ObjectId("..."),
//   customer_id: ObjectId("C001"),
//   amount: 150,
//   customer_info: [
//     { _id: ObjectId("C001"), name: "张三", email: "zhangsan@example.com" }
//   ]
// }
```

#### pipeline 模式（MongoDB 5.0+）

```javascript
// 使用 pipeline 进行更复杂的关联查询
db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      let: { order_cid: "$customer_id" },
      pipeline: [
        {
          $match: {
            $expr: { $eq: ["$_id", "$$order_cid"] },
            status: "vip"  // 只关联 VIP 客户
          }
        }
      ],
      as: "vip_customer"
    }
  }
])
```

## 聚合管道表达式

### 算术表达式

| 表达式 | 说明 | 示例 |
|---|---|---|
| `$add` | 加法 | `{ $add: ["$price", "$tax"] }` |
| `$subtract` | 减法 | `{ $subtract: ["$end_time", "$start_time"] }` |
| `$multiply` | 乘法 | `{ $multiply: ["$price", "$quantity"] }` |
| `$divide` | 除法 | `{ $divide: ["$total", "$count"] }` |
| `$mod` | 取模 | `{ $mod: ["$value", 10] }` |

### 字符串表达式

| 表达式 | 说明 | 示例 |
|---|---|---|
| `$substrBytes` | 截取子串 | `{ $substrBytes: ["$name", 0, 5] }` |
| `$concat` | 拼接字符串 | `{ $concat: ["$first_name", " ", "$last_name"] }` |
| `$toLower` | 转小写 | `{ $toLower: "$status" }` |
| `$toUpper` | 转大写 | `{ $toUpper: "$code" }` |
| `$trim` | 去除空白 | `{ $trim: { input: "$name" } }` |

### 日期表达式

| 表达式 | 说明 | 示例 |
|---|---|---|
| `$year` | 提取年份 | `{ $year: "$created_at" }` |
| `$month` | 提取月份 | `{ $month: "$created_at" }` |
| `$dayOfYear` | 提取年内天数 | `{ $dayOfYear: "$created_at" }` |
| `$dateToString` | 格式化日期 | `{ $dateToString: { format: "%Y-%m-%d", date: "$created_at" } }` |

### 条件表达式

```javascript
// $cond：三元表达式
{ $cond: [条件, 真值, 假值] }
{ $cond: [{ $gte: ["$amount", 1000] }, "高价值", "普通"] }

// $ifNull：处理 null
{ $ifNull: ["$discount", 0] }  // discount 为 null 时返回 0

// $switch：多分支
{
  $switch: {
    branches: [
      { case: { $gte: ["$score", 90] }, then: "A" },
      { case: { $gte: ["$score", 80] }, then: "B" },
      { case: { $gte: ["$score", 60] }, then: "C" }
    ],
    default: "D"
  }
}
```

## 高级聚合阶段

### `$bucket`：分桶

将连续数值分成若干区间（桶）。

```javascript
db.orders.aggregate([
  {
    $bucket: {
      groupBy: "$amount",           // 分桶字段
      boundaries: [0, 100, 500, 1000, Infinity],  // 边界
      default: "Other",            // 边界外的默认桶
      output: {
        count: { $sum: 1 },
        orders: { $push: "$_id" }
      }
    }
  }
])

// 结果示例
// { _id: 0, count: 10, orders: [...] }      // 0-100
// { _id: 100, count: 25, orders: [...] }   // 100-500
// { _id: 500, count: 8, orders: [...] }    // 500-1000
```

### `$bucketAuto`：自动分桶

自动将数据分成指定数量的桶。

```javascript
db.orders.aggregate([
  {
    $bucketAuto: {
      groupBy: "$amount",
      buckets: 5,                  // 分成 5 个桶
      output: {
        count: { $sum: 1 },
        min: { $min: "$amount" },
        max: { $max: "$amount" }
      }
    }
  }
])
```

### `$facet`：多维度聚合

在单个管道中对同一组数据执行多个聚合操作。

```javascript
db.orders.aggregate([
  {
    $facet: {
      // 子管道1：按月份统计
      monthly: [
        { $group: { _id: { $month: "$created_at" }, count: { $sum: 1 } } },
        { $sort: { "_id": 1 } }
      ],
      // 子管道2：按金额区间统计
      amount_ranges: [
        {
          $bucket: {
            groupBy: "$amount",
            boundaries: [0, 100, 500, 1000, Infinity],
            output: { count: { $sum: 1 } }
          }
        }
      ],
      // 子管道3：Top 5 客户
      top_customers: [
        { $group: { _id: "$customer_id", total: { $sum: "$amount" } } },
        { $sort: { total: -1 } },
        { $limit: 5 }
      ]
    }
  }
])
```

### `$graphLookup`：图遍历

递归查找关联的文档，适合层级数据（如组织架构、好友关系）。

```javascript
db.employees.aggregate([
  {
    $graphLookup: {
      from: "employees",
      startWith: "$_id",           // 起始字段
      connectFromField: "_id",     // 连接字段
      connectToField: "manager_id", // 目标字段
      as: "reporting_chain",      // 输出字段
      maxDepth: 10,               // 最大深度
      depthField: "level"          // 深度字段
    }
  }
])
```

## 聚合管道优化

### 管道顺序优化

MongoDB 聚合管道优化器会自动调整管道阶段顺序，但理解其原理有助于写出更好的查询。

```javascript
// 原始查询
db.orders.aggregate([
  { $sort: { amount: -1 } },
  { $match: { status: "completed" } },
  { $limit: 10 }
])

// 优化后：$match 提前
db.orders.aggregate([
  { $match: { status: "completed" } },  // 先过滤
  { $sort: { amount: -1 } },             // 再排序
  { $limit: 10 }                         // 最后限制
])
```

### 使用 `$match` + `$sort` + `$limit` 组合

当 `$match`、`$sort`、`$limit` 组合使用时，MongoDB 可以使用**索引**来优化。

```javascript
// 创建复合索引
db.orders.createIndex({ status: 1, amount: -1 })

// 这个查询可以使用索引
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $sort: { amount: -1 } },
  { $limit: 10 }
])
```

### `$limit` 的合并优化

连续的多个 `$limit` 会被合并为一个。

```javascript
// 原始
{ $limit: 5 }, { $limit: 10 }

// 优化后
{ $limit: 5 }  // 只保留最小的 limit
```

### `$project` 的列剪裁

`$project` 可以剪裁不必要的字段，减少数据在管道中的传输量。

```javascript
// 只保留需要的字段
db.orders.aggregate([
  {
    $project: {
      order_id: "$_id",
      total: "$amount",
      _id: 0
    }
  }
])
```

## 性能与资源管理

### 内存限制

每个 `$group` 和 `$sort` 阶段默认使用 **100MB 内存**。

```javascript
// 允许使用磁盘
db.orders.aggregate([
  { $group: { _id: "$customer_id", total: { $sum: "$amount" } } }
], { allowDiskUse: true })
```

:::warning 内存使用警告

当 `$group` 的内存使用超过 100MB 时：

1. 如果 `allowDiskUse: true`，会使用磁盘临时文件
2. 如果 `allowDiskUse: false`，会报错
3. 使用磁盘会导致性能显著下降

建议优化数据结构或分批处理。

:::

### 管道超时

```javascript
// 设置 30 秒超时
db.orders.aggregate([
  { $match: { status: "completed" } }
], { maxTimeMS: 30000 })
```

### 分片集合的聚合

在分片集合上执行聚合管道时：

```javascript
// MongoDB 会将管道拆分到各分片执行
// 优化：尽量让 $match 在分片上执行
db.orders.aggregate([
  { $match: { region: "华北" } },      // 可以在分片上执行
  { $group: { _id: "$product_id" } }  // 需要在 mongos 上执行
])
```

## 面试实战

**面试官问**：「聚合管道 `$lookup` 和关系型数据库的 JOIN 有什么区别？」

**深入回答**：

> 核心区别在于数据模型：
>
> 1. **JOIN 是表对表**，结果集是二维的；`$lookup` 是文档对集合，结果保留了被关联文档的完整结构
>
> 2. **JOIN 可以多表连接**；`$lookup` 只能一对一（from 一个集合），但可以用 pipeline 实现更复杂的逻辑
>
> 3. **JOIN 结果是扁平的**；`$lookup` 结果是嵌套的（as 指定的数组）
>
> ```javascript
> // $lookup 结果结构
> {
>   _id: 1,
>   order_id: "A001",
>   customer_info: [{ _id: "C001", name: "张三" }]  // 嵌套数组
> }
> ```

**面试官问**：「如何优化聚合管道的性能？」

**深入回答**：

> 优化聚合管道的主要策略：
>
> 1. **管道顺序**：把 `$match` 放在最前面，减少后续处理的数据量
>
> 2. **使用索引**：确保 `$match` 和 `$sort` 阶段可以使用索引
>
> 3. **限制中间结果**：使用 `$limit` 尽早减少数据量
>
> 4. **使用 `$match` + `$sort` + `$limit` 组合**：这是最优模式
>
> 5. **避免不必要的字段**：使用 `$project` 剪裁字段
>
> 6. **考虑分片策略**：在分片集合上，尽量让过滤在分片上完成

## 总结

聚合管道是 MongoDB 最强大的数据处理工具：

- **管道模型**：文档从一端进入，经过多个阶段处理后输出
- **常用阶段**：`$match`、`$group`、`$sort`、`$limit`、`$project`、`$unwind`、`$lookup`
- **高级阶段**：`$bucket`、`$facet`、`$graphLookup`
- **优化原则**：尽早过滤、使用索引、限制数据量
- **资源管理**：注意内存限制，必要时使用磁盘

熟练掌握聚合管道，是 MongoDB 开发者的必备技能。
