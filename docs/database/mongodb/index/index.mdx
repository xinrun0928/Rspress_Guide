# MongoDB 索引与聚合

索引和聚合是 MongoDB 性能优化的两大基石。

索引决定了查询能否高效执行，没有索引的查询往往是全表扫描。聚合管道则是 MongoDB 数据分析的利器——通过管道式的 stages 组合，可以完成复杂的数据统计分析。

本章节将深入讲解：

- 索引的类型与底层实现
- 索引的管理与优化
- 慢查询分析与优化策略
- 聚合管道的原理与各阶段用法
- `$lookup` 与关系型 JOIN 的对比

---

## 索引的重要性

想象一下在一本没有目录的书中查找内容——你只能逐页翻阅。索引就像是书的目录，让数据库能够快速定位数据，而不是扫描每一条记录。

```
无索引查询：
[文档1] [文档2] [文档3] ... [文档1000000]  ← 全表扫描

有索引查询：
[索引树] → 直接定位 → [目标文档]  ← 只需几次比较
```

---

## 索引与聚合的关系

索引可以加速查询，而聚合管道是执行查询和分析的框架。两者配合使用：

- `$match` 阶段可以利用索引过滤数据
- `$sort` 阶段可以利用索引避免排序
- `$lookup` 阶段则需要额外的处理

理解索引原理，才能写出高效的聚合管道。
