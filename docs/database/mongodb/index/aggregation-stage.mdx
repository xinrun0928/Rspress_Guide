# MongoDB 聚合阶段详解

聚合管道的威力来自各个阶段的组合使用。本节详细介绍常用聚合阶段的原理和最佳实践。

## $match：数据过滤

`$match` 是聚合管道中使用最频繁的阶段，负责筛选符合条件的文档。

### 基本用法

```javascript
// 过滤已完成订单
db.orders.aggregate([
  { $match: { status: "completed" } }
])

// 过滤条件组合
db.orders.aggregate([
  {
    $match: {
      status: "completed",
      amount: { $gte: 100 },
      created_at: { $gte: ISODate("2024-01-01") }
    }
  }
])
```

### 使用 $or 和 $and

```javascript
// OR 条件
db.orders.aggregate([
  {
    $match: {
      $or: [
        { status: "completed" },
        { status: "shipped" }
      ]
    }
  }
])

// AND 嵌套在 OR 中
db.orders.aggregate([
  {
    $match: {
      $or: [
        { status: "completed", amount: { $gte: 1000 } },
        { status: "cancelled", refund: { $exists: true } }
      ]
    }
  }
])
```

### 正则表达式匹配

```javascript
// 查询用户名以 "admin" 开头的用户
db.users.aggregate([
  {
    $match: {
      username: { $regex: "^admin" }
    }
  }
])
```

## $group：数据分组

`$group` 是聚合的核心阶段，用于对文档进行分组和聚合计算。

### 分组基础

```javascript
// 按客户分组，统计订单总额和数量
db.orders.aggregate([
  {
    $group: {
      _id: "$customer_id",
      total_amount: { $sum: "$amount" },
      order_count: { $sum: 1 },
      avg_amount: { $avg: "$amount" }
    }
  }
])
```

### 分组字段为 null 或表达式

```javascript
// 不分组（整个集合作为一组）
db.orders.aggregate([
  {
    $group: {
      _id: null,
      total_amount: { $sum: "$amount" },
      order_count: { $sum: 1 }
    }
  }
])

// 按表达式分组：按年龄段分组
db.users.aggregate([
  {
    $group: {
      _id: {
        $switch: {
          branches: [
            { case: { $lt: ["$age", 18] }, then: "未成年" },
            { case: { $lt: ["$age", 30] }, then: "青年" },
            { case: { $lt: ["$age", 50] }, then: "中年" }
          ],
          default: "老年"
        }
      },
      count: { $sum: 1 }
    }
  }
])
```

### 数组操作符

```javascript
// $push：将值推入数组
db.orders.aggregate([
  {
    $group: {
      _id: "$customer_id",
      order_ids: { $push: "$_id" },
      products: { $push: "$product_id" }
    }
  }
])

// $addToSet：去重推入
db.orders.aggregate([
  {
    $group: {
      _id: "$customer_id",
      categories: { $addToSet: "$category" }  // 不重复的分类
    }
  }
])

// $first 和 $last
db.orders.aggregate([
  {
    $sort: { created_at: 1 }
  },
  {
    $group: {
      _id: "$customer_id",
      first_order: { $first: "$_id" },
      last_order: { $last: "$_id" },
      first_date: { $first: "$created_at" },
      last_date: { $last: "$created_at" }
    }
  }
])
```

## $sort：结果排序

### 基本用法

```javascript
// 按金额降序
db.orders.aggregate([
  { $sort: { amount: -1 } }
])

// 多字段排序
db.orders.aggregate([
  { $sort: { status: 1, amount: -1 } }
])
```

### 与 $match 配合的索引优化

```javascript
// 创建索引
db.orders.createIndex({ status: 1, amount: -1 })

// 这个聚合可以使用索引
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $sort: { amount: -1 } },
  { $limit: 10 }
])
```

:::tip $sort + $limit 优化

当 `$sort` 和 `$limit` 组合使用时，如果 `$limit` 值较小，MongoDB 会使用** top-k sort **优化：只维护最小的 N 个结果，而不是对全部数据排序。

:::

## $lookup：跨集合关联

### 基础关联

```javascript
// 订单关联客户信息
db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customer_id",
      foreignField: "_id",
      as: "customer"
    }
  }
])
```

### 使用 pipeline（MongoDB 5.0+）

```javascript
// 带条件的关联
db.orders.aggregate([
  {
    $lookup: {
      from: "products",
      let: { order_pid: "$product_id" },
      pipeline: [
        {
          $match: {
            $expr: { $eq: ["$_id", "$$order_pid"] },
            price: { $gte: 100 }  // 只关联高价产品
          }
        }
      ],
      as: "premium_products"
    }
  }
])

// 关联多个字段
db.orders.aggregate([
  {
    $lookup: {
      from: "inventory",
      let: {
        pid: "$product_id",
        warehouse: "$warehouse_id"
      },
      pipeline: [
        {
          $match: {
            $expr: {
              $and: [
                { $eq: ["$product_id", "$$pid"] },
                { $eq: ["$warehouse_id", "$$warehouse"] }
              ]
            }
          }
        }
      ],
      as: "inventory_info"
    }
  }
])
```

### 自关联

```javascript
// 评论关联回复
db.comments.aggregate([
  {
    $lookup: {
      from: "comments",
      localField: "_id",
      foreignField: "parent_id",
      as: "replies"
    }
  }
])
```

## $unwind：数组展开

### 基本用法

```javascript
// 展开 tags 数组
db.articles.aggregate([
  { $unwind: "$tags" }
])
```

### 展开选项

```javascript
// 保留数组为 null 或 empty 的文档
db.orders.aggregate([
  {
    $unwind: {
      path: "$items",
      preserveNullAndEmptyArrays: true
    }
  }
])

// 包含数组索引
db.orders.aggregate([
  {
    $unwind: {
      path: "$items",
      includeArrayIndex: "index"
    }
  }
])

// 重命名字段
db.orders.aggregate([
  {
    $unwind: {
      path: "$items",
      preserveNullAndEmptyArrays: true
    }
  },
  {
    $project: {
      order_id: "$_id",
      item: "$items",
      position: "$index"
    }
  }
])
```

## $project：字段投影

### 包含和排除字段

```javascript
// 只保留必要字段
db.orders.aggregate([
  {
    $project: {
      _id: 0,
      order_id: "$_id",
      customer_id: 1,
      total: "$amount",
      created_date: {
        $dateToString: {
          format: "%Y-%m-%d",
          date: "$created_at"
        }
      }
    }
  }
])
```

### 计算字段

```javascript
// 添加计算字段
db.orders.aggregate([
  {
    $project: {
      order_id: "$_id",
      subtotal: "$amount",
      tax: { $multiply: ["$amount", 0.1] },
      total: { $add: ["$amount", { $multiply: ["$amount", 0.1] }] }
    }
  }
])
```

### 嵌套字段访问

```javascript
// 访问嵌套字段
db.orders.aggregate([
  {
    $project: {
      order_id: "$_id",
      customer_name: "$customer.name",
      customer_city: "$customer.address.city"
    }
  }
])
```

## $limit 和 $skip：分页处理

### 基本用法

```javascript
// 获取前 10 条
db.orders.aggregate([
  { $limit: 10 }
])

// 跳过前 20 条，获取 10 条
db.orders.aggregate([
  { $skip: 20 },
  { $limit: 10 }
])
```

### 分页查询

```javascript
// 第 N 页，每页 pageSize 条
function getPage(pageNum, pageSize) {
  return db.orders.aggregate([
    { $sort: { created_at: -1 } },
    { $skip: (pageNum - 1) * pageSize },
    { $limit: pageSize }
  ])
}
```

## $count：计数

### 基本用法

```javascript
// 统计符合条件的文档数
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $count: "completed_orders" }
])

// 结果
// { "completed_orders": 150 }
```

### 替代 $group 计数

```javascript
// $count 比 $group + $project 更简洁
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $count: "total" }
])

// 等价于
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $group: { _id: null, total: { $sum: 1 } } },
  { $project: { _id: 0, total: 1 } }
])
```

## $addFields：添加字段

### 基本用法

```javascript
// 添加计算字段
db.orders.aggregate([
  {
    $addFields: {
      tax: { $multiply: ["$amount", 0.1] },
      total_with_tax: { $add: ["$amount", { $multiply: ["$amount", 0.1] }] }
    }
  }
])
```

### 重写嵌套文档

```javascript
// 更新嵌套字段
db.users.aggregate([
  {
    $addFields: {
      "address.city": "新城市",  // 使用点号访问
      "address.updated": true
    }
  }
])
```

## 阶段组合最佳实践

### 订单统计：完整示例

```javascript
// 按月统计各状态订单
db.orders.aggregate([
  // 阶段1：过滤（使用索引）
  { $match: { created_at: { $gte: ISODate("2024-01-01") } } },

  // 阶段2：展开商品
  { $unwind: "$items" },

  // 阶段3：关联产品信息
  {
    $lookup: {
      from: "products",
      localField: "items.product_id",
      foreignField: "_id",
      as: "product_info"
    }
  },
  { $unwind: "$product_info" },

  // 阶段4：重算金额（使用关联后的价格）
  {
    $addFields: {
      real_amount: {
        $multiply: ["$product_info.price", "$items.quantity"]
      }
    }
  },

  // 阶段5：按月分组
  {
    $group: {
      _id: {
        year: { $year: "$created_at" },
        month: { $month: "$created_at" },
        status: "$status"
      },
      count: { $sum: 1 },
      total_amount: { $sum: "$real_amount" }
    }
  },

  // 阶段6：重塑输出
  {
    $project: {
      _id: 0,
      period: {
        $concat: [
          { $toString: "$_id.year" },
          "-",
          { $toString: "$_id.month" }
        ]
      },
      status: "$_id.status",
      count: 1,
      total_amount: { $round: ["$total_amount", 2] }
    }
  },

  // 阶段7：排序
  { $sort: { period: 1, status: 1 } }
])
```

### 用户画像：完整示例

```javascript
// 构建用户画像
db.orders.aggregate([
  // 过滤活跃用户
  { $match: { created_at: { $gte: ISODate("2023-01-01") } } },

  // 按用户分组
  {
    $group: {
      _id: "$customer_id",
      total_orders: { $sum: 1 },
      total_amount: { $sum: "$amount" },
      categories: { $addToSet: "$category" },
      products: { $push: "$product_id" },
      first_order: { $min: "$created_at" },
      last_order: { $max: "$created_at" }
    }
  },

  // 添加计算字段
  {
    $addFields: {
      avg_order_value: { $divide: ["$total_amount", "$total_orders"] },
      category_count: { $size: "$categories" },
      days_since_first: {
        $divide: [
          { $subtract: [new Date(), "$first_order"] },
          86400000  // 毫秒转天
        ]
      }
    }
  },

  // 添加用户等级
  {
    $addFields: {
      level: {
        $switch: {
          branches: [
            { case: { $gte: ["$total_amount", 10000] }, then: "VIP" },
            { case: { $gte: ["$total_amount", 5000] }, then: "高级" },
            { case: { $gte: ["$total_amount", 1000] }, then: "普通" }
          ],
          default: "新用户"
        }
      }
    }
  },

  // 排序
  { $sort: { total_amount: -1 } },

  // 分页
  { $skip: 0 },
  { $limit: 100 }
])
```

## 总结

聚合管道的各个阶段各有用途：

- `$match`：数据过滤，尽量靠前
- `$group`：分组聚合，核心阶段
- `$sort`：结果排序，配合索引
- `$lookup`：跨集合关联
- `$unwind`：数组展开
- `$project`：字段投影，剪裁数据
- `$limit/$skip`：分页处理
- `$count`：快速计数
- `$addFields`：添加字段

熟练掌握各阶段的用法和组合技巧，是 MongoDB 数据分析的必备技能。
