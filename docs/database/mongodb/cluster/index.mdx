## 核心概念

MongoDB 集群主要解决两个问题：**高可用**和**水平扩展**。

| 架构 | 解决问题 | 核心组件 |
|---|---|---|
| 副本集（Replica Set） | 高可用 | Primary + Secondary + Arbiter |
| 分片集群（Sharded Cluster） | 水平扩展 | mongos + config server + shard |

## 副本集 vs 分片集群

```
副本集架构（高可用）：

┌─────────────────────────────────────────┐
│         Replica Set                      │
│                                          │
│   ┌─────────┐                            │
│   │Primary  │  ← 接受写入                 │
│   │(主节点)  │                            │
│   └────┬────┘                            │
│        │                                 │
│        │ 同步复制                          │
│        ▼                                 │
│   ┌─────────┐  ┌─────────┐              │
│   │Secondary│  │Secondary│              │
│   │(从节点1) │  │(从节点2) │              │
│   └─────────┘  └─────────┘              │
│                                          │
└─────────────────────────────────────────┘

分片集群架构（水平扩展）：

┌─────────────────────────────────────────┐
│         Sharded Cluster                  │
│                                          │
│   ┌─────────┐                            │
│   │ mongos  │  ← 路由节点                 │
│   └────┬────┘                            │
│        │                                 │
│   ┌────┴────┐                           │
│   │          │                           │
│   ▼          ▼                           │
│ Shard1     Shard2     Shard3            │
│ ┌─────┐    ┌─────┐    ┌─────┐          │
│ │ RS1 │    │ RS2 │    │ RS3 │          │
│ └─────┘    └─────┘    └─────┘          │
│                                          │
│   ┌─────────────────────┐              │
│   │   Config Servers     │  ← 元数据      │
│   │   (配置服务器副本集)    │              │
│   └─────────────────────┘              │
└─────────────────────────────────────────┘
```
