# MongoDB 压缩机制

MongoDB WiredTiger 存储引擎内置了多种压缩算法，可以显著减少磁盘占用和 IO 开销。

## 压缩概述

### WiredTiger 支持的压缩算法

| 算法 | 压缩率 | CPU 开销 | 说明 |
|---|---|---|---|
| snappy | 中等 | 低 | 默认算法，速度快 |
| zstd | 高 | 中等 | MongoDB 4.2+，推荐 |
| zlib | 高 | 高 | 最高压缩率，最慢 |
| none | 无 | 无 | 不压缩 |

```
压缩流程示意：

┌─────────────────────────────────────────┐
│            数据写入                       │
│  { name: "张三", age: 25, ... }        │
└─────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│         WiredTiger 压缩                  │
│                                          │
│  snappy:  { "x00": "\x1f\x8b..." }    │
│  zstd:   { "x00": "\x28\xb5..." }      │
│  zlib:   { "x00": "\x78\x9c..." }      │
└─────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│            压缩后存储                     │
│         .wt 文件 (压缩后)                │
└─────────────────────────────────────────┘
```

## 压缩配置

### 集合压缩

```javascript
// 创建压缩集合
db.createCollection("logs", {
  storageEngine: {
    wiredTiger: {
      configString: "block_compressor=zstd"
    }
  }
});

// 或通过 collMod 修改
db.runCommand({
  collMod: "logs",
  storageEngine: {
    wiredTiger: {
      configString: "block_compressor=zstd"
    }
  }
});
```

### 索引压缩

```javascript
// 创建压缩索引
db.orders.createIndex(
  { customer_id: 1 },
  {
    storageEngine: {
      wiredTiger: {
        configString: "block_compressor=zstd"
      }
    }
  }
);
```

### 全局默认压缩

```javascript
// mongod.conf
storage:
  wiredTiger:
    collectionConfig:
      blockCompressor: zstd  # 默认集合压缩算法
    indexConfig:
      prefixCompression: true  # 索引前缀压缩
```

## 压缩算法对比

### snappy 压缩

```javascript
// snappy 特点：
// - 压缩速度极快
// - 压缩比约 2-3x
// - CPU 开销低
// - 适合写入密集型场景

// 配置 snappy
storage:
  wiredTiger:
    collectionConfig:
      blockCompressor: snappy
```

### zstd 压缩

```javascript
// zstd 特点（MongoDB 4.2+）：
// - 压缩比高（3-5x）
// - 压缩速度较快
// - 支持压缩级别调整
// - 推荐用于存储密集型场景

// 配置 zstd
storage:
  wiredTiger:
    collectionConfig:
      blockCompressor: zstd

// zstd 压缩级别
// level 1-3: 快速压缩
// level 10-15: 高压缩比
// level 19+: 超高压缩比

// MongoDB 不直接支持配置压缩级别
// 可通过 WiredTiger API 配置
```

### zlib 压缩

```javascript
// zlib 特点：
// - 最高压缩比
// - CPU 开销最高
// - 适合冷数据、归档数据

// 配置 zlib
storage:
  wiredTiger:
    collectionConfig:
      blockCompressor: zlib
```

## 索引压缩

### 前缀压缩

```javascript
// 索引前缀压缩：相邻索引键的共同前缀只存储一次

// 示例：
// 索引键: ["user_001", "user_002", "user_003"]
// 压缩后: ["user_001", "002", "003"]

// 开启前缀压缩
storage:
  wiredTiger:
    indexConfig:
      prefixCompression: true

// 创建索引时指定
db.orders.createIndex(
  { order_id: 1 },
  { storageEngine: { wiredTiger: { configString: "prefix_compression=true" } } }
);
```

### 不同数据类型压缩

```javascript
// 不同字段适合不同的压缩算法

// 适合 snappy（短字符串、数值）：
// - 状态码
// - 分类标签
// - 数值字段

// 适合 zstd（长文本、JSON）：
// - 描述文本
// - 日志内容
// - 文档内容

// 适合 zlib（冷数据）：
// - 历史归档
// - 备份数据
```

### 计算压缩率

```javascript
// 比较压缩前后大小
function showCompressionRatio(collection) {
  const stats = collection.stats();

  const storageSizeMB = (stats.storageSize / 1024 / 1024).toFixed(2);
  const totalSizeMB = (stats.totalSize / 1024 / 1024).toFixed(2);
  const indexSizeMB = (Object.values(stats.indexSizes).reduce((a, b) => a + b, 0) / 1024 / 1024).toFixed(2);

  print(`Storage Size: ${storageSizeMB} MB`);
  print(`Index Size: ${indexSizeMB} MB`);
  print(`Total Size: ${totalSizeMB} MB`);
  print(`Compression Ratio: ${(stats.totalSize / stats.storageSize).toFixed(2)}x`);
}

// 使用
showCompressionRatio(db.orders);
```

## 压缩与性能

### 压缩对性能的影响

| 方面 | 影响 | 说明 |
|---|---|---|
| 写入性能 | 降低 | 压缩需要额外 CPU |
| 读取性能 | 降低 | 解压缩需要额外 CPU |
| 磁盘 IO | 降低 | 压缩后数据更小 |
| 内存效率 | 提高 | 同样内存可缓存更多数据 |
| 网络传输 | 降低 | 压缩后传输数据更少 |

### 性能优化建议

```javascript
// 写入密集型场景
// 推荐 snappy 压缩
storage:
  wiredTiger:
    collectionConfig:
      blockCompressor: snappy

// 读取密集型场景
// 推荐 snappy 或关闭压缩
storage:
  wiredTiger:
    collectionConfig:
      blockCompressor: snappy
    indexConfig:
      prefixCompression: true

// 存储密集型场景
// 推荐 zstd 压缩
storage:
  wiredTiger:
    collectionConfig:
      blockCompressor: zstd
```

### CPU 与压缩

```javascript
// 高压缩场景下 CPU 使用增加
// MongoDB 单线程压缩，但可以并行

// 查看 CPU 使用
db.serverStatus().systemCpu

// 查看压缩相关统计
db.serverStatus().wiredTiger.cache
```

## 常见问题

### 压缩算法选择

```javascript
// 选择建议：

// 1. 通用场景
// → snappy（默认）

// 2. 存储空间有限
// → zstd（MongoDB 4.2+）

// 3. 冷数据/归档
// → zlib

// 4. 高性能要求
// → none 或 snappy

// 5. 混合场景
// → 根据集合特性分别配置
```

### 压缩配置错误

```javascript
// 常见错误：
// 1. 使用不支持的压缩算法
// 2. 混合压缩算法导致索引无法使用
// 3. 压缩级别设置不当

// 正确做法：
// 索引压缩算法应该与集合一致
db.orders.createIndex(
  { customer_id: 1 },
  { storageEngine: { wiredTiger: { configString: "block_compressor=zstd" } } }
)
```

## 总结

压缩配置要点：

- **snappy**：默认，速度快，适合写入密集型
- **zstd**：压缩率高，推荐用于存储密集型
- **zlib**：最高压缩率，适合冷数据
- **前缀压缩**：对索引效果好
- **性能权衡**：压缩减少 IO 但增加 CPU 开销

正确配置压缩可以显著减少存储成本，同时保持良好的性能。
