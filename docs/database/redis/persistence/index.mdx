# Redis 持久化概述

Redis 的数据存储在内存中，这是它高性能的根源。但内存是易失性的——一旦服务器宕机、断电或重启，内存中的数据就会丢失。

持久化机制解决了这个问题。它将内存中的数据定期或实时写入磁盘，确保即使 Redis 重启，数据也能从磁盘恢复。

## 两种持久化方式

Redis 提供了两种持久化机制：

| 方式 | 原理 | 文件格式 | 恢复速度 | 数据完整性 |
| --- | --- | --- | --- | --- |
| RDB | 定时快照 | 二进制 | 快 | 可能丢失数据（取决于快照间隔） |
| AOF | 记录命令 | 文本 | 慢 | 可配置（always/everysec/no） |

两种方式各有优缺点，实际生产中通常**同时使用**。

## 学习路径

建议按照以下顺序学习：

1. **先理解 RDB**：了解快照的工作原理和 fork 机制
2. **再学习 AOF**：掌握命令追加和三种同步策略
3. **理解 AOF 重写**：知道 AOF 文件会不断膨胀，需要重写优化
4. **对比两种方式**：根据业务场景选择合适的持久化配置
5. **掌握数据恢复**：了解故障后的恢复流程

:::tip 学习目标
学完本章后，你应该能够：
- 根据业务需求配置合适的持久化策略
- 理解 RDB 和 AOF 的优缺点
- 排查持久化相关的故障
- 设计可靠的数据恢复方案
:::
