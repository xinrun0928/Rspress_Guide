# 分布式 Session 管理

在分布式系统中，用户登录后需要在多台服务器间共享 Session。这是 Redis 最经典的应用场景之一。

## 为什么需要分布式 Session

传统单机部署时，Session 存储在应用服务器的内存中：

```
┌─────────────────────────────────────────────────────────────┐
│                      单机部署                                │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                   应用服务器                          │    │
│  │  ┌─────────────────────────────────────────────┐    │    │
│  │  │              Session 存储                    │    │    │
│  │  │  session:user:123 = {id:123, name:"张三"} │    │    │
│  │  └─────────────────────────────────────────────┘    │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

分布式部署时，用户请求可能被分发到不同服务器：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   负载均衡   │ ──▶ │  服务器 A    │     │  服务器 B    │
│             │ ──▶ │  Session A  │     │  Session B  │
│             │ ──▶ │  服务器 C    │     │  Session C  │
└─────────────┘     └─────────────┘     └─────────────┘
                          │                   │
                          │   用户 A 登录在 A
                          │   用户 B 登录在 B
                          │   问题：Session 不共享！
```

## Session 存储方案

### 方案一：Session 粘性（Sticky Session）

在负载均衡层保证同一用户请求到同一台服务器：

```nginx
# Nginx 配置
upstream backend {
    ip_hash;  # 按 IP 哈希，同一 IP 到同一服务器
    server 192.168.1.101:8080;
    server 192.168.1.102:8080;
}
```

**缺点**：
- 服务器宕机时，用户 Session 丢失
- 负载不均衡（热点用户集中）

### 方案二：Session 复制

服务器间同步 Session：

```
服务器 A ──复制──▶ 服务器 B
   │                     │
   └──复制──▶ 服务器 C ───┘
```

**缺点**：
- 占用大量网络带宽
- Session 越多，复制越慢

### 方案三：Session 存储在 Redis（推荐）

```
┌─────────────────────────────────────────────────────────────┐
│                      分布式 Session                          │
│                                                              │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐    │
│  │  服务器 A    │     │  服务器 B    │     │  服务器 C    │    │
│  │  Session    │ ──▶ │  Session    │ ──▶ │  Session    │    │
│  │  Redis 客户端│     │  Redis 客户端│     │  Redis 客户端│    │
│  └─────────────┘     └─────────────┘     └─────────────┘    │
│         │                   │                   │          │
│         └───────────────────┴───────────────────┘          │
│                             │                               │
│                             ▼                               │
│                   ┌─────────────────────┐                   │
│                   │       Redis          │                   │
│                   │  session:xxx:user1  │                   │
│                   │  session:xxx:user2  │                   │
│                   └─────────────────────┘                   │
└─────────────────────────────────────────────────────────────┘
```

## Spring Session + Redis

Spring Session 提供了完整的分布式 Session 解决方案：

### 依赖配置

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-data-redis</artifactId>
    <version>2.7.0</version>
</dependency>
```

### 配置类

```java
import org.springframework.session.data.redis.config.annotation.web.http.EnableRedisHttpSession;

@Configuration
@EnableRedisHttpSession(maxInactiveIntervalInSeconds = 1800)  // 30分钟过期
public class SessionConfig {
    
    @Bean
    public LettuceConnectionFactory connectionFactory() {
        return new LettuceConnectionFactory(
            new RedisStandaloneConfiguration("localhost", 6379)
        );
    }
}
```

### 使用 Session

```java
@RestController
public class UserController {
    
    @PostMapping("/login")
    public String login(HttpServletRequest request, 
                       @RequestParam String username,
                       @RequestParam String password) {
        // 验证用户
        User user = userService.login(username, password);
        
        // 设置 Session
        HttpSession session = request.getSession();
        session.setAttribute("userId", user.getId());
        session.setAttribute("username", user.getUsername());
        
        return "登录成功";
    }
    
    @GetMapping("/userinfo")
    public User getUserInfo(HttpServletRequest request) {
        HttpSession session = request.getSession();
        Long userId = (Long) session.getAttribute("userId");
        
        return userService.getById(userId);
    }
    
    @PostMapping("/logout")
    public String logout(HttpServletRequest request) {
        HttpSession session = request.getSession();
        session.invalidate();  // 销毁 Session
        return "退出成功";
    }
}
```

## 手动实现分布式 Session

如果不用 Spring Session，也可以手动实现：

### Session 管理类

```java
@Service
public class RedisSessionManager {
    
    private RedisTemplate<String, Object> redisTemplate;
    
    private static final String SESSION_PREFIX = "session:";
    private static final int SESSION_EXPIRE = 1800;  // 30分钟
    
    /**
     * 创建 Session
     */
    public String createSession(Long userId, String username) {
        String sessionId = UUID.randomUUID().toString();
        String key = SESSION_PREFIX + sessionId;
        
        Map<String, Object> session = new HashMap<>();
        session.put("userId", userId);
        session.put("username", username);
        session.put("loginTime", System.currentTimeMillis());
        
        redisTemplate.opsForHash().putAll(key, session);
        redisTemplate.expire(key, SESSION_EXPIRE, TimeUnit.SECONDS);
        
        return sessionId;
    }
    
    /**
     * 获取 Session
     */
    public Map<Object, Object> getSession(String sessionId) {
        String key = SESSION_PREFIX + sessionId;
        Map<Object, Object> session = redisTemplate.opsForHash().entries(key);
        
        if (!session.isEmpty()) {
            // 续期
            redisTemplate.expire(key, SESSION_EXPIRE, TimeUnit.SECONDS);
        }
        
        return session;
    }
    
    /**
     * 设置 Session 属性
     */
    public void setAttribute(String sessionId, String key, Object value) {
        String sessionKey = SESSION_PREFIX + sessionId;
        redisTemplate.opsForHash().put(sessionKey, key, value);
        redisTemplate.expire(sessionKey, SESSION_EXPIRE, TimeUnit.SECONDS);
    }
    
    /**
     * 获取 Session 属性
     */
    public Object getAttribute(String sessionId, String key) {
        String sessionKey = SESSION_PREFIX + sessionId;
        return redisTemplate.opsForHash().get(sessionKey, key);
    }
    
    /**
     * 删除 Session
     */
    public void deleteSession(String sessionId) {
        String key = SESSION_PREFIX + sessionId;
        redisTemplate.delete(key);
    }
}
```

### Session 拦截器

```java
@Component
public class SessionInterceptor implements HandlerInterceptor {
    
    @Autowired
    private RedisSessionManager sessionManager;
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) throws Exception {
        String sessionId = request.getHeader("X-Session-Id");
        
        if (sessionId == null) {
            sessionId = request.getParameter("sessionId");
        }
        
        if (sessionId == null) {
            // 从 Cookie 获取
            Cookie[] cookies = request.getCookies();
            if (cookies != null) {
                for (Cookie cookie : cookies) {
                    if ("SESSION_ID".equals(cookie.getName())) {
                        sessionId = cookie.getValue();
                        break;
                    }
                }
            }
        }
        
        if (sessionId != null) {
            Map<Object, Object> session = sessionManager.getSession(sessionId);
            if (!session.isEmpty()) {
                request.setAttribute("session", session);
                request.setAttribute("userId", session.get("userId"));
            }
        }
        
        return true;
    }
}
```

## Session 安全

### Session 固定攻击

攻击者先获取一个 Session ID，诱导用户登录后使用该 ID：

**防御措施**：

```java
// 登录成功后更换 Session ID
@PostMapping("/login")
public String login(HttpServletRequest request) {
    // 验证用户...
    
    // 旧 Session 失效
    HttpSession oldSession = request.getSession(false);
    if (oldSession != null) {
        oldSession.invalidate();
    }
    
    // 新 Session
    HttpSession newSession = request.getSession(true);
    newSession.setAttribute("userId", user.getId());
    
    return "登录成功";
}
```

### XSS 攻击

攻击者通过脚本窃取 Cookie 中的 Session ID：

**防御措施**：

```java
// 设置 HttpOnly Cookie
Cookie sessionCookie = new Cookie("SESSION_ID", sessionId);
sessionCookie.setHttpOnly(true);  // 禁止 JavaScript 访问
sessionCookie.setSecure(true);     // 仅 HTTPS 传输
sessionCookie.setPath("/");
response.addCookie(sessionCookie);
```

## 集群 Session 管理

### Redis Cluster 下的 Session

```java
@Configuration
@EnableRedisHttpSession
public class SessionConfig {
    
    @Bean
    public LettuceConnectionFactory connectionFactory() {
        RedisClusterConfiguration config = new RedisClusterConfiguration();
        config.addNode("192.168.1.101:6379");
        config.addNode("192.168.1.102:6379");
        config.addNode("192.168.1.103:6379");
        
        return new LettuceConnectionFactory(config);
    }
}
```

## 总结

分布式 Session 是分布式系统的必备功能：

1. **Session 粘性**：负载均衡层实现，简单但不灵活
2. **Session 复制**：服务器间同步，开销大
3. **Redis 存储**：推荐方案，高性能、易扩展
4. **Spring Session**：开箱即用，功能完善
5. **安全防护**：防止 Session 固定攻击、XSS

使用 Redis 存储 Session，是最主流、最可靠的方案。
