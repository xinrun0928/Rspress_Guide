# 延迟队列与定时任务

订单超时自动取消、用户注册成功后发送欢迎短信、会员到期前提醒续费……这些场景都需要「延迟执行」。Redis 的 ZSet 可以优雅地实现延迟队列。

## 什么是延迟队列

普通队列是 FIFO（先进先出），延迟队列则是按「延迟时间」决定执行顺序：

```
普通队列：                        延迟队列：
┌─────┐ ┌─────┐ ┌─────┐         10:00  ┌─────┐
│  A  │ │  B  │ │  C  │              │  B  │
└──┬──┘ └──┬──┘ └──┬──┘              └──┬──┘
   │        │        │                   │
   ▼        ▼        ▼              10:05 ┌─────┐
   A        B        C                   │  A  │
（立即执行）                           └──┬──┘
                                          │
                                     10:10 ┌─────┐
                                          │  C  │
（延迟 5 分钟）                        └──┬──┘
                                          │
（延迟 10 分钟）                       当前时间

A 延迟 5 分钟，B 延迟 10 分钟，C 延迟 15 分钟
按延迟时间顺序执行
```

## 延迟队列实现

### 使用 ZSet 实现

```java
@Service
public class DelayQueueService {
    
    private RedisTemplate<String, Object> redisTemplate;
    
    private static final String DELAY_QUEUE = "delay:queue:";
    
    /**
     * 添加延迟任务
     * @param jobId 任务 ID
     * @param data 任务数据
     * @param delaySeconds 延迟秒数
     */
    public void addJob(String jobId, Object data, long delaySeconds) {
        // 执行时间 = 当前时间 + 延迟时间
        long executeTime = System.currentTimeMillis() + delaySeconds * 1000;
        
        // 使用 ZSet，score 为执行时间
        String queueKey = DELAY_QUEUE + getQueueName(data);
        redisTemplate.opsForZSet().add(queueKey, jobId, executeTime);
    }
    
    /**
     * 获取可执行的任务
     */
    public Set<String> getExecutableJobs(String queueName) {
        String queueKey = DELAY_QUEUE + queueName;
        long now = System.currentTimeMillis();
        
        // 查询 score <= now 的任务
        return redisTemplate.opsForZSet().rangeByScore(queueKey, 0, now);
    }
    
    /**
     * 移除任务
     */
    public void removeJob(String queueName, String jobId) {
        String queueKey = DELAY_QUEUE + queueName;
        redisTemplate.opsForZSet().remove(queueKey, jobId);
    }
}
```

### 延迟队列消费者

```java
@Service
public class DelayQueueConsumer {
    
    @Autowired
    private DelayQueueService delayQueueService;
    
    @Autowired
    private OrderService orderService;
    
    /**
     * 轮询延迟队列，处理到期的任务
     */
    @Scheduled(fixedDelay = 1000)  // 每秒执行一次
    public void pollAndProcess() {
        // 获取所有可执行的任务
        Set<String> jobs = delayQueueService.getExecutableJobs("order");
        
        for (String jobId : jobs) {
            try {
                // 处理任务
                processOrderJob(jobId);
                
                // 处理成功后移除任务
                delayQueueService.removeJob("order", jobId);
            } catch (Exception e) {
                // 处理失败，可以重试或记录日志
                handleJobFailure(jobId, e);
            }
        }
    }
    
    private void processOrderJob(String jobId) {
        // 解析任务数据
        String[] parts = jobId.split(":");
        if (parts.length < 2) {
            return;
        }
        
        String orderId = parts[1];
        
        // 执行业务逻辑：取消订单
        orderService.cancelOrder(orderId);
    }
}
```

### 订单超时取消示例

```java
@Service
public class OrderService {
    
    @Autowired
    private DelayQueueService delayQueueService;
    
    /**
     * 创建订单时，添加延迟任务
     */
    public Order createOrder(Order order) {
        // 1. 创建订单
        order = orderRepository.save(order);
        
        // 2. 添加延迟取消任务（30 分钟超时）
        String jobId = "order:cancel:" + order.getId();
        delayQueueService.addJob(jobId, order, 30 * 60);  // 30 分钟
        
        return order;
    }
    
    /**
     * 支付成功，取消延迟任务
     */
    public void paySuccess(Long orderId) {
        // 移除延迟取消任务
        String jobId = "order:cancel:" + orderId;
        delayQueueService.removeJob("order", jobId);
    }
    
    /**
     * 取消订单
     */
    public void cancelOrder(Long orderId) {
        Order order = orderRepository.findById(orderId);
        
        // 只有未支付的订单才能取消
        if (order != null && "PENDING".equals(order.getStatus())) {
            order.setStatus("CANCELLED");
            orderRepository.save(order);
        }
    }
}
```

## 分布式延迟队列

如果系统部署了多个实例，需要保证只有一个实例处理任务：

```java
@Service
public class DistributedDelayQueueConsumer {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    private static final String PROCESSING_SET = "delay:processing:";
    
    /**
     * 分布式处理延迟任务
     */
    public void pollAndProcess(String queueName) {
        long now = System.currentTimeMillis();
        String queueKey = DELAY_QUEUE + queueName;
        
        // 获取到期的任务
        Set<ZSetOperations.TypedTuple<String>> jobs = 
            redisTemplate.opsForZSet().rangeByScoreWithScores(queueKey, 0, now);
        
        for (ZSetOperations.TypedTuple<String> job : jobs) {
            String jobId = job.getValue();
            Double score = job.getScore();
            
            // 尝试获取任务锁
            String lockKey = "lock:" + jobId;
            Boolean acquired = redisTemplate.opsForValue()
                .setIfAbsent(lockKey, "1", 30, TimeUnit.SECONDS);
            
            if (Boolean.TRUE.equals(acquired)) {
                try {
                    // 处理任务
                    processJob(jobId);
                    
                    // 从队列移除
                    redisTemplate.opsForZSet().remove(queueKey, jobId);
                } finally {
                    // 释放锁
                    redisTemplate.delete(lockKey);
                }
            }
        }
    }
}
```

## 定时任务 vs 延迟队列

| 特性 | 定时任务 | 延迟队列 |
| --- | --- | --- |
| 执行时间 | 固定时间点 | 相对当前时间的延迟 |
| 精确度 | 取决于扫描间隔 | 取决于扫描间隔 |
| 适用场景 | 每天定时、每小时定时 | 订单超时、消息重试 |
| 实现复杂度 | 低 | 中 |

## 定时任务实现

如果需要每天定时执行某个任务，可以使用 Redis 实现分布式定时锁：

```java
@Service
public class DistributedCronService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    private static final String CRON_LOCK = "cron:lock:";
    
    /**
     * 执行定时任务（分布式环境只有一个实例执行）
     */
    public void executeWithLock(String cronName, Runnable task) {
        String lockKey = CRON_LOCK + cronName;
        
        // 尝试获取锁
        Boolean acquired = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, "1", 5, TimeUnit.MINUTES);
        
        if (Boolean.TRUE.equals(acquired)) {
            try {
                task.run();
            } finally {
                // 执行完成后释放锁
                redisTemplate.delete(lockKey);
            }
        }
        // 获取锁失败，说明其他实例正在执行
    }
    
    /**
     * 每天凌晨 2 点清理过期数据
     */
    @Scheduled(cron = "0 0 2 * * ?")
    public void cleanupExpiredData() {
        executeWithLock("cleanup", () -> {
            // 清理逻辑
            log.info("开始清理过期数据");
            
            // 查询并删除过期数据
            List<ExpiredData> expiredList = dataRepository.findExpired();
            for (ExpiredData data : expiredList) {
                dataRepository.delete(data);
            }
            
            log.info("清理完成，共删除 {} 条数据", expiredList.size());
        });
    }
}
```

## 延迟队列的改进

### 问题：轮询消耗资源

定时轮询会消耗资源，而且有延迟（取决于轮询间隔）。

### 解决方案：使用 Redis 的 Keyspace Notifications

```bash
# 开启 Keyspace Notifications
redis-cli CONFIG SET notify-keyspace-events Ex

# 配置过期事件监听
# 当 key 过期时，发布事件
```

```java
@Service
public class KeyspaceNotificationConsumer {
    
    @Autowired
    private RedisMessageListenerContainer listenerContainer;
    
    @Autowired
    private OrderService orderService;
    
    @PostConstruct
    public void init() {
        // 监听过期事件
        listenerContainer.addMessageListener((message, pattern) -> {
            String expiredKey = new String(message.getBody());
            
            // 处理过期任务
            if (expiredKey.startsWith("order:cancel:")) {
                String orderId = expiredKey.substring("order:cancel:".length());
                orderService.cancelOrder(Long.parseLong(orderId));
            }
        }, new PatternTopic("__keyevent@0__:expired"));
    }
}
```

:::warning Keyspace Notifications 的限制
- 需要开启 `notify-keyspace-events Ex`
- 过期事件是「尽力而为」，可能有延迟
- 不保证每个过期事件都被触发
:::

## 完整示例：订单超时取消

```java
@RestController
@RequestMapping("/api/order")
public class OrderController {
    
    @Autowired
    private OrderService orderService;
    
    /**
     * 创建订单

     * 支付订单

        orderService.paySuccess(id);
        return Result.success("支付成功");
    }
    
    /**
     * 取消订单
     */
    @PostMapping("/cancel/{id}")
    public Result<String> cancelOrder(@PathVariable Long id) {
        orderService.cancelOrder(id);
        return Result.success("取消成功");
    }
}
```

## 总结

延迟队列是 Redis 的经典应用：

1. **ZSet 实现**：score 为执行时间戳
2. **轮询处理**：定时扫描到期的任务
3. **分布式锁**：保证任务不被重复执行
4. **可靠性**：任务执行后移除，失败时重试
5. **替代方案**：Redisson、RocketMQ 等专业队列

根据业务场景选择合适的实现方式，简单场景用 Redis 足够，复杂场景考虑专业队列中间件。
