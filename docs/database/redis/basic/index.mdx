## 学习路径

建议按照以下顺序学习：

1. **先理解线程模型**：搞清楚 Redis 为什么这么快，这是 Redis 最核心的设计理念
2. **再学习数据类型**：了解每种数据类型的能力和使用场景
3. **深入对象系统**：理解底层数据结构，知其然更知其所以然
4. **掌握高级类型**：Bitmap 和 HyperLogLog 是面试亮点

:::tip 学习建议
Redis 的很多设计都是围绕「高性能」和「节省内存」展开的。学习时可以带着这两个目标去理解每一种机制。
:::
