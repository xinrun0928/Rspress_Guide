# Redis 数据类型

Redis 之所以强大，很大程度上是因为它支持丰富的数据类型。与 Memcached 只能存储字符串不同，Redis 提供了五种基本数据类型和多种高级类型，每种类型都针对特定场景进行了优化。

理解数据类型不仅是面试的基础，更是正确使用 Redis 的前提。

## 数据类型总览

| 类型 | 命令示例 | 底层实现 | 典型场景 |
| --- | --- | --- | --- |
| String | SET/GET | SDS | 缓存、计数器、分布式锁 |
| List | LPUSH/RPOP | QuickList | 消息队列、最新列表 |
| Hash | HSET/HGET | ziplist / dict | 对象存储、表数据缓存 |
| Set | SADD/SMEMBERS | intset / dict | 标签、好友关系、去重 |
| ZSet | ZADD/ZRANGE | ziplist / skiplist+dict | 排行榜、延迟队列 |
| BitMap | SETBIT/GETBIT | String | 签到、在线统计 |
| HyperLogLog | PFADD/PFCOUNT | String | UV 统计 |
| Geo | GEOADD/GEORADIUS | Sorted Set | LBS、附近的人 |
| Stream | XADD/XREAD | Radix Tree | 消息队列 |

:::tip 编码与数据结构
Redis 的数据类型和底层实现是分离的。同一种数据类型可能对应多种编码，Redis 会根据数据量自动选择最优的编码。这种设计让 Redis 在小数据量时节省内存，大数据量时保证性能。
:::

## String 类型

### 概述

String 是 Redis 最基本的数据类型，可以存储字符串、整数或浮点数。

```bash
# 基本操作
SET name "redis"
GET name
> "redis"

# 数值操作
SET counter 100
INCR counter
> (integer) 101
INCRBY counter 50
> (integer) 151

# 浮点数操作
SET price 9.99
INCRBYFLOAT price 0.5
> "10.49"
```

### 常用命令

| 命令 | 说明 | 示例 |
| --- | --- | --- |
| SET | 设置值 | `SET key value [EX seconds] [PX milliseconds] [NX/XX]` |
| GET | 获取值 | `GET key` |
| SETNX | 不存在时设置 | `SETNX key value` |
| MSET/MGET | 批量设置/获取 | `MSET k1 v1 k2 v2` |
| APPEND | 追加字符串 | `APPEND key "suffix"` |
| STRLEN | 获取长度 | `STRLEN key` |
| SETRANGE | 范围修改 | `SETRANGE key offset value` |

```bash
# 批量操作提高效率
MSET user:1:name "Alice" user:1:age "25" user:1:city "Beijing"
MGET user:1:name user:1:age user:1:city
> 1) "Alice"
> 2) "25"
> 3) "Beijing"
```

### 分布式锁实现

String 类型是实现分布式锁的基础：

```bash
# 加锁：使用 SETNX + 过期时间
SET lock:order:123 UNIQUE_ID NX EX 30

# 释放锁：使用 Lua 脚本确保原子性
EVAL "if redis.call('get', KEYS[1]) == ARGV[1] then return redis.call('del', KEYS[1]) else return 0 end" 1 lock:order:123 UNIQUE_ID
```

### 计数器场景

```bash
# 文章阅读量
INCR article:views:1001
INCRBY article:views:1001 100

# 限流
INCR rate:limit:ip:192.168.1.1
EXPIRE rate:limit:ip:192.168.1.1 60
```

### String 的编码

String 类型有两种编码：

| 编码 | 说明 | 使用条件 |
| --- | --- | --- |
| int | 整数 | 存储的是整数，且能用 long 类型表示 |
| embstr | 短字符串 | 字符串长度 `` `<=` 44 字节 |
| raw | 长字符串 | 字符串长度 `` `>` 44 字节 |

```bash
# 查看编码
$ redis-cli DEBUG OBJECT ENCODING mykey
"int"

$ redis-cli SET mykey "hello"
$ redis-cli DEBUG OBJECT ENCODING mykey
"embstr"

$ redis-cli SET mykey "a very long string that exceeds 44 bytes..."
$ redis-cli DEBUG OBJECT ENCODING mykey
"raw"
```

:::tip 为什么是 44 字节
Redis 一个对象头（redisObject）占 16 字节，jemalloc 分配单元最小 64 字节，减去对象头后可用于存储 SDS 的空间为 48 字节。其中 SDS 结构本身占 3-4 字节，所以实际存储空间为 44 字节。
:::

## List 类型

### 概述

List 是一个有序的字符串列表，支持从两端插入（push）和弹出（pop）元素。

```bash
# 从左边插入
LPUSH fruits apple banana orange
> (integer) 3

# 从右边插入
RPUSH fruits grape
> (integer) 4

# 查看列表
LRANGE fruits 0 -1
> 1) "orange"
> 2) "banana"
> 3) "apple"
> 4) "grape"
```

### 常用命令

| 命令 | 说明 |
| --- | --- |
| LPUSH/RPUSH | 从左/右插入 |
| LPOP/RPOP | 从左/右弹出 |
| LRANGE | 范围查询 |
| LLEN | 获取长度 |
| LTRIM | 修剪列表 |
| BLPOP/BRPOP | 阻塞式弹出 |

### 消息队列实现

```bash
# 生产者：向列表添加消息
LPUSH queue:orders "order:1001"
LPUSH queue:orders "order:1002"
LPUSH queue:orders "order:1003"

# 消费者：阻塞式消费
BRPOP queue:orders 0
> 1) "queue:orders"
> 2) "order:1001"
```

:::warning 注意事项
List 作为消息队列没有消费确认机制，消息被消费后会从列表中删除。如果消费者在处理过程中崩溃，消息会丢失。对于可靠性要求高的场景，建议使用 Stream。
:::

### 最新列表场景

```bash
# 朋友圈最新动态
LPUSH user:feed:1001 "msg:2001"
LPUSH user:feed:1001 "msg:2002"
LPUSH user:feed:1001 "msg:2003"

# 只保留最新 100 条
LTRIM user:feed:1001 0 99

# 获取最新 10 条
LRANGE user:feed:1001 0 9
```

### List 的编码

List 类型有两种底层实现：

| 编码 | 说明 | 触发条件 |
| --- | --- | --- |
| ziplist | 压缩列表 | 列表元素数量 `` `<=` 512，且每个元素 `` `<=` 64 字节 |
| quicklist | 快速列表 | 默认使用 quicklist |

```
ziplist：紧凑连续内存，节省空间，但插入/删除效率低

quicklist：多个 ziplist 组成的双向链表，平衡了空间和性能
```

## Hash 类型

### 概述

Hash 是一个键值对集合，适合存储对象。

```bash
# 设置对象
HSET user:1001 name "Alice" age "25" city "Beijing"

# 获取字段
HGET user:1001 name
> "Alice"

# 获取所有字段
HGETALL user:1001
> 1) "name"
> 2) "Alice"
> 3) "age"
> 4) "25"
> 5) "city"
> 6) "Beijing"

# 批量操作
HMSET user:1002 name "Bob" age "30"
HMGET user:1002 name age
> 1) "Bob"
> 2) "30"
```

### 常用命令

| 命令 | 说明 |
| --- | --- |
| HSET/HGET | 设置/获取单个字段 |
| HMSET/HMGET | 批量设置/获取字段 |
| HGETALL | 获取所有字段值 |
| HLEN | 获取字段数量 |
| HEXISTS | 判断字段是否存在 |
| HINCRBY | 字段值增加 |
| HDEL | 删除字段 |
| HKEYS/HVALS | 获取所有键/值 |

### Hash vs String 存储对象

| 特性 | Hash | String |
| --- | --- | --- |
| 获取单个字段 | O(1) | O(n) 或需要额外解析 |
| 内存占用 | 较高（有字段开销） | 较低 |
| 批量操作 | 需要 HGETALL | 可用 MGET |
| 过期时间 | 整个 key | 可为每个字段单独设置过期 |

```bash
# Hash 存储：对象属性独立
HSET user:1001 name "Alice" age "25"
HINCRBY user:1001 age 1

# String 存储：需要 JSON 序列化
SET user:1001 '{"name":"Alice","age":25}'
GET user:1001
> "{\"name\":\"Alice\",\"age\":25}"
```

:::tip 选择建议
- 对象字段经常单独操作：用 Hash
- 对象整体操作频繁：用 String + JSON
- 需要过期时间控制：用 String（可对整个 key 设置过期）
:::

### Hash 的编码

| 编码 | 触发条件 |
| --- | --- |
| ziplist | 元素数量 `` `<=` 512，每个元素 `` `<=` 64 字节 |
| hashtable | 超过上述条件 |

```bash
# 查看编码
$ redis-cli DEBUG OBJECT ENCODING user:1001
"ziplist"
```

## Set 类型

### 概述

Set 是一个无序的、不重复的字符串集合。

```bash
# 添加元素
SADD tags:article:1001 "Redis" "Database" "Cache"

# 获取所有元素
SMEMBERS tags:article:1001
> 1) "Cache"
> 2) "Database"
> 3) "Redis"

# 判断元素是否存在
SISMEMBER tags:article:1001 "Redis"
> (integer) 1

SISMEMBER tags:article:1001 "MongoDB"
> (integer) 0
```

### 常用命令

| 命令 | 说明 |
| --- | --- |
| SADD | 添加元素 |
| SREM | 删除元素 |
| SMEMBERS | 获取所有元素 |
| SISMEMBER | 判断元素是否存在 |
| SCARD | 获取集合基数（元素个数） |
| SRANDMEMBER | 随机获取元素 |
| SPOP | 随机弹出元素 |
| SINTER/SUNION/SDIFF | 交集/并集/差集运算 |

### 标签系统

```bash
# 给文章添加标签
SADD article:tags:1001 "Java" "Redis" "Database"

# 给用户添加感兴趣的标签
SADD user:interest:100 "Java" "Python" "AI"

# 计算用户和文章的标签重合度
SINTER user:interest:100 article:tags:1001
> 1) "Java"

# 推荐：用户还感兴趣的其他标签
SDIFF user:interest:100 article:tags:1001
> 1) "Python"
> 2) "AI"
```

### 抽奖系统

```bash
# 参与者
SADD lottery:1001 user:1001 user:1002 user:1003 user:1004 user:1005

# 抽取 2 名获奖者
SRANDMEMBER lottery:1001 2
> 1) "user:1003"
> 2) "user:1001"

# 抽取后删除（不可重复中奖）
SPOP lottery:1001 2
> 1) "user:1002"
> 2) "user:1005"
```

### Set 的编码

| 编码 | 触发条件 |
| --- | --- |
| intset | 所有元素都是整数，且元素数量 `` `<=` 512 |
| hashtable | 超过上述条件 |

## ZSet 类型

### 概述

ZSet 是一个有序的、不重复的字符串集合，每个元素都有一个分数（score）用于排序。

```bash
# 添加排行榜数据
ZADD game:rank 1500 "player1"
ZADD game:rank 2000 "player2"
ZADD game:rank 1800 "player3"

# 获取排名前 3（从高到低）
ZREVRANGE game:rank 0 2 WITHSCORES
> 1) "player2"
> 2) "2000"
> 3) "player3"
> 4) "1800"
> 5) "player1"
> 6) "1500"

# 获取指定玩家的排名（从 0 开始）
ZREVRANK game:rank "player2"
> (integer) 0
```

### 常用命令

| 命令 | 说明 |
| --- | --- |
| ZADD | 添加元素 |
| ZRANGE/ZREVRANGE | 正序/倒序获取 |
| ZSCORE | 获取元素分数 |
| ZRANK/ZREVRANK | 获取排名 |
| ZINCRBY | 增加分数 |
| ZCARD | 获取元素数量 |
| ZCOUNT | 获取指定分数范围的元素数 |
| ZRANGEBYSCORE | 按分数范围查询 |
| ZUNIONSTORE/ZINTERSTORE | 并集/交集运算 |

### 排行榜场景

```bash
# 玩家加分
ZINCRBY game:rank 500 "player1"
ZINCRBY game:rank -200 "player2"

# 获取排名变化
ZREVRANK game:rank "player1"
> (integer) 0  # 第一名

# 获取指定分数段的用户
ZCOUNT game:rank 1000 2000
> (integer) 2

# 获取 Top 10 用户及其分数
ZREVRANGE game:rank 0 9 WITHSCORES
```

### ZSet 的编码

| 编码 | 触发条件 |
| --- | --- |
| ziplist | 元素数量 `` `<=` 128，每个元素 `` `<=` 64 字节 |
| skiplist + dict | 超过上述条件 |

:::tip 为什么是跳表 + 字典
ZSet 需要同时支持：
1. 按分数快速排序（跳表）
2. 按元素名快速查找（字典）

两种数据结构配合，保证所有操作都是 O(log N) 复杂度。
:::

## BitMap 类型

### 概述

BitMap 不是独立的数据类型，而是 String 类型的一种扩展。它通过位操作来实现高效的存储。

```bash
# 设置某一位的值
SETBIT user:checkin:1001:2024-03 0 1
SETBIT user:checkin:1001:2024-03 1 1
SETBIT user:checkin:1001:2024-03 2 1
SETBIT user:checkin:1001:2024-03 5 1

# 获取某一位的值
GETBIT user:checkin:1001:2024-03 0
> (integer) 1

# 统计连续 1 的个数（签到天数）
BITCOUNT user:checkin:1001:2024-03
> (integer) 4
```

### 签到系统

```bash
# 用户签到（假设今天是 2024-03-15，第 75 天）
SETBIT user:checkin:1001:2024-03 74 1

# 统计本月签到天数
BITCOUNT user:checkin:1001:2024-03

# 统计连续签到天数
BITCOUNT user:checkin:1001:2024-03 74 74  # 最新签到的前一天开始往前统计
```

### 在线状态统计

```bash
# 用户上线
SETBIT online:users 1001 1

# 用户下线
SETBIT online:users 1001 0

# 统计在线人数
BITCOUNT online:users

# 获取在线用户 ID
GETBIT online:users 1001
```

## HyperLogLog 类型

### 概述

HyperLogLog 是用于基数统计的算法，能用极小的内存统计海量数据。

```bash
# 添加数据
PFADD uv:2024-03-15 "user1" "user2" "user3"
PFADD uv:2024-03-15 "user4" "user5"

# 统计基数（UV）
PFCOUNT uv:2024-03-15
> (integer) 5

# 合并多个 HyperLogLog
PFADD uv:2024-03-16 "user3" "user6" "user7"
PFMERGE uv:2024-03 uv:2024-03-15 uv:2024-03-16
PFCOUNT uv:2024-03
> (integer) 7
```

:::tip 内存占用
HyperLogLog 的内存占用是固定的，约 12KB，但可以统计 2^64 个元素，标准误差约 0.81%。
:::

### UV 统计场景

```bash
# 每日 UV 统计
PFADD page:uv:2024-03-15 "192.168.1.1"
PFADD page:uv:2024-03-15 "192.168.1.2"
PFADD page:uv:2024-03-15 "192.168.1.1"  # 重复的不计数

# 活动页面总 UV
PFADD event:uv:promotion "user1" "user2" "user3"
PFCOUNT event:uv:promotion
```

## Geo 类型

### 概述

Geo 类型基于 ZSet 实现，存储地理位置信息。

```bash
# 添加地理位置
GEOADD cities 116.4074 39.9042 "Beijing"
GEOADD cities 121.4737 31.2304 "Shanghai"
GEOADD cities 113.2644 23.1291 "Guangzhou"

# 获取两个城市之间的距离
GEODIST cities Beijing Shanghai km
> "1088.1301"

# 获取附近的城市（150km 范围内）
GEORADIUS cities 116.4074 39.9042 150 km WITHDIST ASC COUNT 10
> 1) 1) "Beijing"
>    2) "0.0"
> 3) 1) "Tianjin"
>    2) "110.2894"
```

### 附近的人

```bash
# 用户位置更新
GEOADD user:location 116.4074 39.9042 "user:1001"
GEOADD user:location 116.4100 39.9060 "user:1002"

# 查找附近的人（3km 范围）
GEORADIUS user:location 116.4074 39.9042 3 km WITHDIST ASC
```
