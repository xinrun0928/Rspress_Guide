# Redis 监控

Redis 的运维离不开监控。及时发现性能问题、容量瓶颈，是保障服务稳定的关键。

## 监控维度

Redis 监控主要包括：

| 维度 | 监控指标 | 工具/命令 |
| --- | --- | --- |
| 性能 | QPS、延迟、命令耗时 | INFO stats、SLOWLOG |
| 内存 | 使用量、峰值、碎片 | INFO memory |
| 连接 | 连接数、拒绝连接数 | INFO clients |
| 持久化 | RDB/AOF 状态、fork 耗时 | INFO persistence |
| 复制 | 主从延迟、同步状态 | INFO replication |
| 键空间 | key 数量、过期率 | INFO keyspace |

## INFO 命令

INFO 是 Redis 最全面的监控命令：

```bash
# 基本信息
redis-cli INFO server        # 服务器信息
redis-cli INFO clients        # 客户端信息
redis-cli INFO memory        # 内存信息
redis-cli INFO persistence   # 持久化信息
redis-cli INFO stats         # 统计信息
redis-cli INFO replication   # 复制信息
redis-cli INFO cpu           # CPU 使用
redis-cli INFO commandstats  # 命令统计
redis-cli INFO keyspace      # 键空间信息

# 全部信息
redis-cli INFO all
```

## 常用监控命令

### 实时 QPS

```bash
# 实时查看每秒命令数
redis-cli --intrinsic-latency 100

# 使用 INFO stats 查看 QPS
redis-cli INFO stats | grep instantaneous_ops

# 实时监控
watch -n 1 'redis-cli INFO stats | grep instantaneous_ops'
```

### 延迟监控

```bash
# 测量延迟
redis-cli --latency

# 延迟历史
redis-cli --latency-history

# 延迟分布
redis-cli --latency-dist

# 测试特定命令延迟
redis-cli --latency-history -c GET -c SET
```

### 连接监控

```bash
# 查看所有客户端
redis-cli CLIENT LIST

# 返回格式
id=1 addr=127.0.0.1:54321 fd=6 name= age=100 idle=0 flags=N db=0 
sub=0 psub=0 multi=-1 qbuf=0 qbuf-free=0 obl=0 oll=0 omem=0 cmd=GET

# 字段说明
# id: 客户端 ID
# addr: IP 和端口
# fd: 文件描述符
# age: 连接时长（秒）
# idle: 空闲时长（秒）
# flags: 客户端类型（N=普通，M=主节点，S=从节点）
# cmd: 最后执行的命令

# 查看慢客户端
redis-cli CLIENT LIST | awk '$5>60 {print $2, $5}'

# 杀死空闲连接
redis-cli CLIENT KILL ID <id>
redis-cli CLIENT KILL ADDR <ip:port>
redis-cli CLIENT KILL TYPE normal
```

### 内存监控

```bash
# 查看内存使用
redis-cli INFO memory | grep -E "used|peak|fragment"

# 查看 key 大小分布
redis-cli --bigkeys

# 查看指定 key 内存
redis-cli MEMORY USAGE mykey

# 内存诊断
redis-cli MEMORY DOCTOR
```

## MONITOR 命令

实时监控所有命令：

```bash
# 实时监控所有命令（生产环境慎用）
redis-cli MONITOR

# 输出示例
1700000000.123456 [0 127.0.0.1:54321] "GET" "user:123"
1700000000.234567 [0 127.0.0.1:54322] "SET" "key" "value"
1700000000.345678 [0 127.0.0.1:54323] "HGETALL" "product:456"
```

```python
# Python 监控示例
import redis

def monitor_commands():
    r = redis.Redis()
    pubsub = r.pubsub()
    pubsub.subscribe('__keyspace@0__:user:*')
    
    for message in pubsub.listen():
        if message['type'] == 'message':
            print(f"Key {message['channel']} was {message['data']}")

# 使用 Redis 的键空间通知
redis-cli CONFIG SET notify-keyspace-events KEA
```

:::warning MONITOR 的性能影响
MONITOR 会捕获所有命令，对性能有较大影响。建议：
- 只在排查问题时临时使用
- 使用 redis-cli --intrinsic-latency 限制采样
- 考虑使用 CLIENT LIST + SLOWLOG 代替
:::

## Prometheus 监控

### 安装 redis_exporter

```bash
# Docker 部署
docker run -d \
  --name redis_exporter \
  -p 9121:9121 \
  oliver006/redis_exporter:latest \
  --redis.addr=redis://127.0.0.1:6379

# Kubernetes 部署
kubectl apply -f redis-exporter.yaml
```

### Prometheus 配置

```yaml
# prometheus.yml
- job_name: 'redis'
  static_configs:
    - targets: ['localhost:9121']
```

## 告警配置

### 内存告警

```yaml
# Prometheus 告警规则
groups:
- name: redis
  rules:
  - alert: RedisMemoryHigh
    expr: redis_memory_used_bytes / redis_memory_max_bytes > 0.8
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "Redis 内存使用率过高"
      description: "内存使用率 {{ $value | humanizePercentage }}"

  - alert: RedisMemoryFragmentation
    expr: redis_memory_fragmentation_ratio > 1.5
    for: 10m
    labels:
      severity: warning
    annotations:
      summary: "Redis 内存碎片率过高"
      description: "内存碎片率 {{ $value }}"
```

### 连接告警

```yaml
  - alert: RedisConnectionsHigh
    expr: redis_connected_clients > 10000
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "Redis 连接数过高"
      description: "当前连接数 {{ $value }}"
```

### 慢查询告警

```yaml
  - alert: RedisSlowQueries
    expr: redis_slowlog_length > 10
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "Redis 慢查询过多"
      description: "当前慢查询数 {{ $value }}"
```

## 日志分析

### Redis 日志

```bash
# Redis 日志位置
# Debian/Ubuntu: /var/log/redis/redis-server.log
# CentOS/RHEL: /var/log/redis/redis.log

# 日志级别
redis-cli CONFIG GET loglevel
# 1) "loglevel"
# 2) "notice"

# 查看最近日志
tail -f /var/log/redis/redis-server.log
```

### 日志格式

```bash
# Redis 日志示例
1:M 15 Mar 2024 10:00:00.123 * Server started
1:M 15 Mar 2024 10:00:00.234 * DB loaded from disk
1:M 15 Mar 2024 10:00:00.345 * Ready to accept connections
1:M 15 Mar 2024 10:30:00.456 * 10000 changes in 60 seconds
```

### 关键日志信息

| 日志内容 | 含义 |
| --- | --- |
| `10000 changes in 60 seconds` | 写入过于频繁 |
| `Client id connected` | 客户端连接 |
| `Client id disconnected` | 客户端断开 |
| `Background save started` | RDB 保存开始 |
| `Background save terminated` | RDB 保存完成 |
| `Replica connected` | 从节点连接 |
| `Replication backlog full` | 复制积压满 |

## 总结

Redis 监控是运维的基础：

1. **基础监控**：INFO 命令获取各项指标
2. **实时监控**：MONITOR、SLOWLOG 排查问题
3. **自动化监控**：redis_exporter + Prometheus + Grafana
4. **告警配置**：根据业务设置合理阈值
5. **日志分析**：从日志中发现问题

建立完善的监控体系，才能保障 Redis 服务的稳定运行。
