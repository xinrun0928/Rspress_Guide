# Pipeline 与事务

批量操作是提升 Redis 性能的重要手段。Pipeline、事务（MULTI/EXEC）、Lua 脚本都能实现批量操作，但它们有什么区别？什么时候该用哪个？

## 批量操作的三种方式

| 方式 | 原子性 | 事务支持 | 性能 | 适用场景 |
| --- | --- | --- | --- | --- |
| Pipeline | 否 | 否 | 高 | 普通批量操作 |
| MULTI/EXEC | 是 | 是 | 中 | 需要事务保证 |
| Lua 脚本 | 是 | 是 | 高 | 复杂业务逻辑 |

## Pipeline

### 普通批量操作的问题

```bash
# 普通方式：3 次网络往返
GET key1    # 第 1 次 RTT
GET key2    # 第 2 次 RTT
GET key3    # 第 3 次 RTT
```

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│  Client │───▶│  Redis  │───▶│ Response│  RTT 1
└─────────┘    └─────────┘    └─────────┘
┌─────────┐    ┌─────────┐    ┌─────────┐
│  Client │───▶│  Redis  │───▶│ Response│  RTT 2
└─────────┘    └─────────┘    └─────────┘
┌─────────┐    ┌─────────┐    ┌─────────┐
│  Client │───▶│  Redis  │───▶│ Response│  RTT 3
└─────────┘    └─────────┘    └─────────┘

总耗时 = 3 * RTT
```

### Pipeline 原理

Pipeline 将多个命令打包成一个请求发送，减少网络往返：

```bash
# Pipeline：1 次网络往返
GET key1
GET key2
GET key3
```

```
┌─────────────────────────────────────────┐
│  Client                                 │
│  GET key1                               │
│  GET key2                               │
│  GET key3                               │
└─────────────────────────────────────────┘
                    │
                    │ 1 个请求（打包多个命令）
                    ▼
┌─────────────────────────────────────────┐
│  Redis                                  │
│  Response1                              │
│  Response2                              │
│  Response3                              │
└─────────────────────────────────────────┘
                    │
                    │ 1 个响应
                    ▼
┌─────────────────────────────────────────┐
│  Client                                 │
│  Response1                              │
│  Response2                              │
│  Response3                              │
└─────────────────────────────────────────┘

总耗时 = 1 * RTT
```

### Pipeline 使用

```java
// Jedis
try (Pipeline pipeline = jedis.pipelined()) {
    pipeline.set("key1", "value1");
    pipeline.set("key2", "value2");
    pipeline.set("key3", "value3");
    pipeline.get("key1");
    pipeline.get("key2");
    
    // 执行所有命令
    List<Object> results = pipeline.syncAndReturnAll();
    
    // results[0] = OK
    // results[1] = OK
    // results[2] = OK
    // results[3] = "value1"
    // results[4] = "value2"
}
```

```python
# Python Redis
pipe = r.pipeline()
pipe.set('key1', 'value1')
pipe.set('key2', 'value2')
pipe.set('key3', 'value3')
pipe.get('key1')

# 执行所有命令
results = pipe.execute()
# ['OK', 'OK', 'OK', 'value1']
```

```bash
# redis-cli 支持 Pipeline
echo -e "SET k1 v1\nSET k2 v2\nGET k1" | redis-cli --pipe
```

### Pipeline 的限制

Pipeline **不保证原子性**：

```java
// Pipeline 中的命令是分开执行的
// 如果中间某条命令失败，后面的命令仍然会执行
try (Pipeline pipeline = jedis.pipelined()) {
    pipeline.incr("counter");     // 成功
    pipeline.incr("counter");     // 成功
    pipeline.incr("counter");     // 成功
    pipeline.rename("counter", "new_counter");  // 失败（counter 可能不存在）
    
    List<Object> results = pipeline.syncAndReturnAll();
    // 前 3 个成功，第 4 个失败
    // 但前 3 个命令已经执行了
}
```

## 事务（MULTI/EXEC）

### 事务原理

Redis 事务将多个命令打包，然后一次性顺序执行：

```bash
MULTI                # 开始事务
SET key1 value1      # 命令入队
SET key2 value2      # 命令入队
GET key1             # 命令入队
EXEC                 # 执行所有命令
```

```
┌──────────────────────────────────────────────────────────────┐
│                        事务流程                               │
│                                                              │
│  MULTI                                                        │
│     │                                                         │
│     ▼                                                         │
│  ┌─────────────────────────────────────────────────────┐     │
│  │ 命令入队                                               │     │
│  │ - SET key1 value1  → QUEUED                          │     │
│  │ - SET key2 value2  → QUEUED                          │     │
│  │ - GET key1         → QUEUED                          │     │
│  └─────────────────────────────────────────────────────┘     │
│     │                                                         │
│     ▼                                                         │
│  EXEC                                                         │
│     │                                                         │
│     ▼                                                         │
│  ┌─────────────────────────────────────────────────────┐     │
│  │ 执行结果                                              │     │
│  │ 1) OK                                                │     │
│  │ 2) OK                                                │     │
│  │ 3) "value1"                                         │     │
│  └─────────────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────────────┘
```

### 事务使用

```java
// Jedis
Transaction tx = jedis.multi();
tx.set("key1", "value1");
tx.set("key2", "value2");
tx.get("key1");
List<Object> results = tx.exec();

// results[0] = OK
// results[1] = OK
// results[2] = "value1"
```

### WATCH / UNWATCH

WATCH 用于乐观锁，监视 key 的变化：

```bash
WATCH mykey                 # 监视 mykey
GET mykey                   # 读取当前值
MULTI                       # 开始事务
SET mykey newvalue          # 设置新值
INCR mykey                  # 再次修改
EXEC                        # 执行
                             # 如果 mykey 在 WATCH 之后被修改，事务失败
```

```java
// 乐观锁实现
jedis.watch("balance");           // 监视 balance
int balance = Integer.parseInt(jedis.get("balance"));

if (balance >= amount) {          // 检查余额
    Transaction tx = jedis.multi();
    tx.decrBy("balance", amount);
    tx.incrBy("order", amount);
    
    // 如果 balance 被修改，exec 返回 null
    if (tx.exec() != null) {
        // 成功
    } else {
        // 失败，重试
    }
} else {
    jedis.unwatch();              // 取消监视
}
```

### DISCARD

取消事务，放弃所有入队的命令：

```bash
MULTI
SET key1 value1
SET key2 value2
DISCARD                # 取消事务，所有命令都不执行
```

### 事务的错误处理

Redis 事务的错误有两种：

1. **入队错误**：命令不存在或参数错误，EXEC 直接失败
2. **执行错误**：命令执行时失败，其他命令仍会执行

```bash
# 入队错误
MULTI
SET key1 value1
INCR key1         # 对 string 执行 INCR
LPUSH key1 value  # 对 string 执行 LPUSH
EXEC
# EXEC 直接失败，返回错误

# 执行错误
MULTI
SET key1 1
INCR key1         # OK
INCRBYFLOAT key1 0.5  # OK（Redis 6.2+ 支持）
LPUSH key1 value  # 错误：value 不是数字
EXEC
# 返回 [1, 2, error]
# 前两条命令执行了，第三条失败
```

:::warning 事务的错误处理
Redis 事务不像数据库事务那样支持回滚。EXEC 执行后，所有命令要么全部执行（部分失败也会执行），要么全部不执行（入队错误）。这是 Redis 的设计哲学：简单、高性能。
:::

## Lua 脚本

### 为什么需要 Lua

- Pipeline：命令打包，但**不保证原子性**
- 事务：保证原子性，但**不支持条件判断**
- Lua 脚本：**既有原子性，又有条件判断**

### Lua 脚本原理

Redis 内嵌 Lua 解释器，执行 Lua 脚本时：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Lua 脚本   │───▶│ Redis 命令   │───▶│   结果      │
│             │    │ 原子执行     │    │            │
└─────────────┘    └─────────────┘    └─────────────┘

整个脚本像一条命令一样执行，不会被其他命令打断
```

### Lua 脚本使用

```bash
# EVAL 基本语法
EVAL script numkeys [key ...] [arg ...]

# 示例
EVAL "return redis.call('GET', KEYS[1])" 1 mykey
# 等价于：GET mykey

# 多 key 和多参数
EVAL "
  local v1 = redis.call('GET', KEYS[1])
  local v2 = redis.call('GET', KEYS[2])
  return v1 .. ',' .. v2
" 2 key1 key2
```

```java
// Jedis 执行 Lua 脚本
String script = 
    "local current = redis.call('GET', KEYS[1]) " +
    "if current == false then " +
    "    redis.call('SET', KEYS[1], ARGV[1]) " +
    "    return 1 " +
    "else " +
    "    return 0 " +
    "end";

Jedis jedis = new Jedis("localhost", 6379);

// 执行脚本
String[] keys = {"mykey"};
String[] args = {"myvalue"};
Object result = jedis.eval(script, 1, keys, args);

// 返回 1 表示成功设置，返回 0 表示 key 已存在
```

### Lua 脚本示例

#### 分布式锁

```lua
-- 加锁脚本
local key = KEYS[1]
local value = ARGV[1]
local ttl = ARGV[2]

-- SETNX + EXPIRE 的原子操作
if redis.call('SET', key, value, 'NX', 'EX', ttl) then
    return 1
else
    return 0
end
```

```lua
-- 解锁脚本
local key = KEYS[1]
local value = ARGV[1]

-- 只有持有者才能释放锁
if redis.call('GET', key) == value then
    return redis.call('DEL', key)
else
    return 0
end
```

#### 点赞计数器

```lua
-- 点赞脚本
local user_key = KEYS[1]      -- user:likes:post:123
local post_key = KEYS[2]      -- post:likes:123
local user_id = ARGV[1]

-- 检查是否已点赞
if redis.call('SISMEMBER', user_key, user_id) == 1 then
    return 0  -- 已经点赞
end

-- 添加点赞
redis.call('SADD', user_key, user_id)
redis.call('INCR', post_key)

return 1  -- 点赞成功
```

#### 限流器

```lua
-- 滑动窗口限流
local key = KEYS[1]           -- rate:limit:ip:192.168.1.1
local window = tonumber(ARGV[1])  -- 时间窗口（秒）
local limit = tonumber(ARGV[2])  -- 限制次数

local current = redis.call('GET', key)
if current and tonumber(current) >= limit then
    return 0  -- 超过限制
end

current = redis.call('INCR', key)
if tonumber(current) == 1 then
    redis.call('EXPIRE', key, window)
end

return 1  -- 允许通过
```

### EVALSHA 缓存脚本

如果脚本很长，可以使用 EVALSHA 缓存脚本：

```bash
# 第一次：加载脚本，返回 sha1
SCRIPT LOAD "
  local v1 = redis.call('GET', KEYS[1])
  local v2 = redis.call('GET', KEYS[2])
  return v1 .. ',' .. v2
"
# 返回: "c4a5cce1d0e8f1a2b3c4d5e6f7a8b9c0d1e2f3a4"

# 后续：使用 sha1 执行（更快）
EVALSHA "c4a5cce1d0e8f1a2b3c4d5e6f7a8b9c0d1e2f3a4" 2 key1 key2
```

```java
// Jedis 使用 EVALSHA
String script = 
    "local v1 = redis.call('GET', KEYS[1]) " +
    "local v2 = redis.call('GET', KEYS[2]) " +
    "return v1 .. ',' .. v2";

// 加载脚本
String sha = jedis.scriptLoad(script);

// 执行
jedis.evalsha(sha, 2, "key1", "key2");
```

## 三种方式对比

| 特性 | Pipeline | 事务 | Lua 脚本 |
| --- | --- | --- | --- |
| 原子性 | 无 | 有 | 有 |
| 条件判断 | 无 | 无 | 有 |
| 错误处理 | 部分失败 | 部分失败 | 可处理 |
| 性能 | 最高 | 中 | 高 |
| 复杂度 | 低 | 低 | 中 |
| 回滚支持 | 无 | 无 | 手动 |
| 适用场景 | 普通批量操作 | 简单事务 | 复杂业务逻辑 |

## 最佳实践

### Pipeline 场景

```java
// 批量写入
try (Pipeline pipeline = jedis.pipelined()) {
    for (String key : keys) {
        pipeline.set(key, values.get(key));
    }
    pipeline.sync();
}

// 批量读取
try (Pipeline pipeline = jedis.pipelined()) {
    for (String key : keys) {
        pipeline.get(key);
    }
    List<Object> results = pipeline.syncAndReturnAll();
}
```

### 事务场景

```java
// 简单的批量更新
Transaction tx = jedis.multi();
tx.incr("page:views:2024-03-15");
tx.incr("page:views:2024-03");
tx.exec();
```

### Lua 脚本场景

```java
// 需要条件判断的复杂操作
String script = 
    "local balance = redis.call('GET', KEYS[1]) " +
    "if balance == false then " +
    "    return -1 " +  // key 不存在
    "end " +
    "local amount = tonumber(ARGV[1]) " +
    "if tonumber(balance) < amount then " +
    "    return 0 " +    // 余额不足
    "end " +
    "redis.call('DECRBY', KEYS[1], amount) " +
    "return 1";          // 成功

jedis.eval(script, 1, "balance", "100");
```

## 总结

三种批量操作方式各有特点：

1. **Pipeline**：适合普通批量操作，高性能，但不保证原子性
2. **事务**：适合简单的原子操作，有 WATCH 支持乐观锁
3. **Lua 脚本**：适合复杂的业务逻辑，既有原子性又有条件判断

根据业务需求选择合适的方式，才能发挥 Redis 的最大性能。
