# Redis 高级特性概述

学完了 Redis 的基础数据结构、主从复制、哨兵和 Cluster，你可能觉得 Redis 已经掌握得差不多了。

但 Redis 还有一些高级特性，它们不那么显眼，却非常重要——I/O 模型决定了 Redis 为什么这么快，淘汰策略决定了内存满了怎么办，Pipeline 和事务让批量操作更高效。

这一章，我们来深入理解这些高级特性。

## 为什么 Redis这么快

Redis 为什么这么快？很多人会说「因为是内存数据库」——这只是原因之一。

真正的原因是 **I/O 模型 + 数据结构 + 单线程** 三者结合。

```
Redis 性能影响因素：

1. I/O 模型
   - epoll 多路复用
   - 非阻塞 I/O
   - 事件循环

2. 数据结构
   - 简单动态字符串（SDS）
   - 压缩列表、跳表等高效结构
   - 时间复杂度 O(1) 的操作

3. 单线程
   - 避免锁竞争
   - 避免上下文切换
   - 原子操作保证
```

## 内存管理

Redis 运行在内存中，但内存是有限的。当内存满了怎么办？

```
内存管理机制：

1. 内存上限控制
   - maxmemory 配置
   - 超过上限触发淘汰

2. 淘汰策略
   - noeviction: 不淘汰，返回错误
   - volatile-*: 只淘汰带过期时间的 key
   - allkeys-*: 淘汰所有 key
   - LRU/LFU/RANDOM

3. 内存回收
   - 定时删除
   - 惰性删除
   - 定期删除
```

## 批量操作优化

Redis 命令是往返式的，每次命令都需要网络往返。Pipeline 让我们可以把多个命令打包发送：

```
普通模式：
命令1 -> 往返 -> 结果1
命令2 -> 往返 -> 结果2
命令3 -> 往返 -> 结果3

Pipeline 模式：
命令1 -> 命令2 -> 命令3 -> 往返 -> 结果1 + 结果2 + 结果3
```
