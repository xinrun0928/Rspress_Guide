# HTAP 实战

TiDB HTAP 架构将 OLTP 和 OLAP 能力整合在一起，本节介绍 HTAP 的实战应用。

## HTAP 架构设计

### 典型架构

```

┌─────────────────────────────────────────────────────┐

```

### 场景选择

| 场景 | 存储 | 说明 |
|---|---|---|
| 高并发写入 | TiKV | 行存储，高效写入 |
| 点查询 | TiKV | 行存储，快速响应 |
| 复杂分析 | TiFlash | 列存储，高效聚合 |
| 历史报表 | TiFlash | 列存储，大数据量 |
| 机器学习 | TiSpark | Spark 生态 |

## 实时分析场景

### 电商订单分析

```sql
-- 场景：实时订单统计

-- 1. 实时订单数
SELECT
    COUNT(*) as order_count,
    SUM(amount) as total_amount
FROM orders
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 1 HOUR);

-- 2. 分渠道统计
SELECT
    channel,
    COUNT(*) as order_count,
    SUM(amount) as total_amount
FROM orders
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 1 DAY)
GROUP BY channel;

-- 3. 分小时趋势
SELECT
    DATE_FORMAT(created_at, '%Y-%m-%d %H:00') as hour,
    COUNT(*) as order_count
FROM orders
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY DATE_FORMAT(created_at, '%Y-%m-%d %H:00')
ORDER BY hour;
```

### 用户行为分析

```sql
-- 场景：用户行为漏斗分析

-- 1. 漏斗转化
WITH events AS (
    SELECT
        user_id,
        MAX(CASE WHEN event = 'view' THEN 1 ELSE 0 END) as viewed,
        MAX(CASE WHEN event = 'cart' THEN 1 ELSE 0 END) as carted,
        MAX(CASE WHEN event = 'order' THEN 1 ELSE 0 END) as ordered
    FROM user_events
    WHERE created_at >= '2024-01-01'
    GROUP BY user_id
)
SELECT
    'View' as stage,
    SUM(viewed) as count
UNION ALL
SELECT
    'Cart' as stage,
    SUM(carted) as count
UNION ALL
SELECT
    'Order' as stage,
    SUM(ordered) as count;

-- 2. 用户留存
WITH cohort AS (
    SELECT
        user_id,
        DATE(created_at) as cohort_date,
        DATEDIFF(lead(DATE(created_at)) OVER (PARTITION BY user_id ORDER BY created_at), DATE(created_at)) as days
    FROM user_events
    WHERE event = 'order'
)
SELECT
    cohort_date,
    COUNT(DISTINCT user_id) as users,
    SUM(CASE WHEN days = 1 THEN 1 ELSE 0 END) as day1_retention,
    SUM(CASE WHEN days = 7 THEN 1 ELSE 0 END) as day7_retention
FROM cohort
GROUP BY cohort_date;
```

### 实时大屏

```sql
-- 场景：实时销售大屏

-- 1. 今日实时数据
SELECT
    SUM(CASE WHEN DATE(created_at) = CURDATE() THEN amount ELSE 0 END) as today_amount,
    SUM(CASE WHEN DATE(created_at) = DATE_SUB(CURDATE(), INTERVAL 1 DAY) THEN amount ELSE 0 END) as yesterday_amount,
    SUM(CASE WHEN DATE(created_at) = CURDATE() THEN 1 ELSE 0 END) as today_orders,
    SUM(CASE WHEN DATE(created_at) = DATE_SUB(CURDATE(), INTERVAL 1 DAY) THEN 1 ELSE 0 END) as yesterday_orders
FROM orders;

-- 2. TOP 商品
SELECT
    p.product_name,
    COUNT(*) as order_count,
    SUM(oi.quantity) as total_quantity,
    SUM(oi.quantity * oi.price) as total_amount
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
JOIN products p ON oi.product_id = p.id
WHERE o.created_at >= DATE_SUB(NOW(), INTERVAL 1 DAY)
GROUP BY p.id, p.product_name
ORDER BY total_amount DESC
LIMIT 10;

-- 3. 分地区统计
SELECT
    c.region,
    COUNT(*) as order_count,
    SUM(o.amount) as total_amount
FROM orders o
JOIN customers c ON o.customer_id = c.id
WHERE o.created_at >= DATE_SUB(NOW(), INTERVAL 1 DAY)
GROUP BY c.region
ORDER BY total_amount DESC;
```

## 历史数据分析场景

### 月度报表

```sql
-- 场景：生成月度报表

-- 1. 月度汇总
SELECT
    YEAR(created_at) as year,
    MONTH(created_at) as month,
    COUNT(*) as order_count,
    COUNT(DISTINCT customer_id) as customer_count,
    SUM(amount) as total_amount,
    AVG(amount) as avg_amount,
    MAX(amount) as max_amount,
    MIN(amount) as min_amount
FROM orders
WHERE created_at >= '2024-01-01'
GROUP BY YEAR(created_at), MONTH(created_at)
ORDER BY year, month;

-- 2. 月度趋势
SELECT
    DATE(created_at) as date,
    COUNT(*) as order_count,
    SUM(amount) as total_amount
FROM orders
WHERE created_at >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
GROUP BY DATE(created_at)
ORDER BY date;

-- 3. TOP 客户
SELECT
    c.customer_id,
    c.customer_name,
    COUNT(*) as order_count,
    SUM(o.amount) as total_amount
FROM orders o
JOIN customers c ON o.customer_id = c.id
WHERE o.created_at >= DATE_SUB(CURDATE(), INTERVAL 1 MONTH)
GROUP BY c.id, c.customer_name
ORDER BY total_amount DESC
LIMIT 20;
```

### 用户画像分析

```sql
-- 场景：用户画像分析

-- 1. 用户分层
WITH user_stats AS (
    SELECT
        customer_id,
        COUNT(*) as order_count,
        SUM(amount) as total_amount,
        AVG(amount) as avg_amount,
        MAX(created_at) as last_order_date,
        DATEDIFF(CURDATE(), MAX(DATE(created_at))) as days_since_last_order
    FROM orders
    WHERE created_at >= DATE_SUB(CURDATE(), INTERVAL 6 MONTH)
    GROUP BY customer_id
)
SELECT
    CASE
        WHEN total_amount >= 100000 THEN 'VIP'
        WHEN total_amount >= 50000 THEN '高级'
        WHEN total_amount >= 10000 THEN '普通'
        ELSE '新用户'
    END as customer_level,
    COUNT(*) as customer_count,
    SUM(total_amount) as total_amount
FROM user_stats
GROUP BY
    CASE
        WHEN total_amount >= 100000 THEN 'VIP'
        WHEN total_amount >= 50000 THEN '高级'
        WHEN total_amount >= 10000 THEN '普通'
        ELSE '新用户'
    END;

-- 2. 流失预警
SELECT
    customer_id,
    last_order_date,
    days_since_last_order,
    CASE
        WHEN days_since_last_order > 30 THEN '流失风险'
        WHEN days_since_last_order > 14 THEN '活跃下降'
        ELSE '活跃'
    END as status
FROM user_stats
WHERE days_since_last_order > 14
ORDER BY days_since_last_order DESC;
```

## 总结

HTAP 实战要点：

- **架构设计**：根据场景选择 TiKV 或 TiFlash
- **实时分析**：适合大屏、漏斗、实时统计
- **历史分析**：适合报表、用户画像
- **性能优化**：批量写入、分区裁剪、资源隔离
- **监控调优**：持续监控查询性能
