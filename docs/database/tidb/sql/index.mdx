# SQL 执行层

TiDB 的 SQL 执行层负责接收用户请求、解析 SQL、生成执行计划，并协调下层存储节点返回数据。

本节涵盖以下内容：

- SQL 执行引擎：火山模型与向量化执行
- MPP 架构：大规模并行分析处理
- DDL 实现：在线 schema 变更
- 查询优化器：代价模型与执行计划生成
- Percolator 事务：分布式事务的两阶段提交

---

## 核心概念速览

| 概念 | 说明 |
|---|---|
| Volcano Model | 算子通过 Next() 接口拉取数据的执行模型 |
| Vectorization | 向量化执行，一次处理一批数据 |
| MPP | Massively Parallel Processing，多节点并行分析 |
| CBO | Cost-Based Optimizer，基于代价的查询优化器 |
| Percolator | Google 的分布式事务模型 |

这些组件共同构成了 TiDB 的 SQL 处理能力。
