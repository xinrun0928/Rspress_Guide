# TiDB PD 调度组件

PD（Placement Driver）是 TiDB 集群的调度中心，负责元数据管理、调度决策和 TSO 分配。

## PD 核心功能

### 三大职责

| 功能 | 说明 |
|---|---|
| 集群元数据管理 | 存储 Region、Store、调度规则 |
| 调度决策 | 均衡 Region 分布、管理副本 |
| TSO 分配 | 提供全局唯一时间戳 |

## 调度策略

### Balance Scheduler

负责将 Region 均匀分布到各个 Store。

```sql
-- 查看调度器
SHOW SCHEDULERS;

-- 输出示例
-- +-------------------------------+---------------+
-- | NAME                          | STATUS        |
-- +-------------------------------+---------------+
-- | balance-region-scheduler       | STARTED       |
-- | balance-leader-scheduler      | STARTED       |
-- | hot-region-scheduler          | STARTED       |
-- | label-scheduler               | STARTED       |
-- +-------------------------------+---------------+

-- 禁用调度器
pd-ctl -u http://pd1:2379 scheduler delete balance-region-scheduler

-- 启用调度器
pd-ctl -u http://pd1:2379 scheduler add balance-region-scheduler
```

### Hot Region Scheduler

处理热点 Region 的调度。

```sql
-- 查看热点 Region
SHOW HOT REGION;

-- 输出示例
-- +----------+----------------------+--------+---------+
-- | STORE_ID | REGION_ID           | Leader | HotDegree|
-- +----------+----------------------+--------+---------+
-- | 1        | 12345               | 101    | 1000    |
-- | 2        | 67890               | 201    | 900     |
-- +----------+----------------------+--------+---------+

-- 查看热点 Region 详情
SHOW HOT REGION HISTORY;
```

### 调度参数配置

```bash
# pd.toml 配置

# 调度器参数
[schedule]
# 最大调度并发数
max-scheduler-count = 4096

# Region 分裂阈值
region-schedule-limit = 2048

# Leader 调度限制
leader-schedule-limit = 2048

# 热点 Region 调度限制
hot-region-schedule-limit = 4

# 调度器权重
scheduler-max-waiting-operator = 5

# 调度器类型
enable-joint-consensus = true

# Region 合并
merge-schedule-limit = 64
split-merge-interval = "1h"
```

## 调度规则

### 调度配置

```bash
# 配置调度规则
pd-ctl -u http://pd1:2379 config set balance-region-weight 5.0

# 配置副本数
pd-ctl -u http://pd1:2379 config set max-replicas 3

# 配置 Label
pd-ctl -u http://pd1:2379 label store store_id 1 zone "zone-a"

# 查看配置
pd-ctl -u http://pd1:2379 config show all
```

### 调度限制

```bash
# 配置调度速度
pd-ctl -u http://pd1:2379 config set max-pending-down-beacon 3
pd-ctl -u http://pd1:2379 config set max-store-down-time "30m"

# 配置调度器权重
pd-ctl -u http://pd1:2379 scheduler config balance-region-scheduler balance-weight 1
```

## 调度优化

### 常见问题

```sql
-- 问题：调度太慢
-- 解决：增加调度并发

-- 问题：调度影响性能
-- 解决：限制调度速度

-- 问题：热点 Region
-- 解决：启用热点调度

-- 问题：数据分布不均
-- 解决：检查 Store 状态和调度器
```

### 优化建议

```bash
# 1. 调整调度参数
pd-ctl -u http://pd1:2379 config set region-schedule-limit 2048
pd-ctl -u http://pd1:2379 config set leader-schedule-limit 2048

# 2. 开启跨机房调度
pd-ctl -u http://pd1:2379 placement-rules rule-save

# 3. 限制调度时间窗口
pd-ctl -u http://pd1:2379 config set allow-temporarily-store-down "1h"
```

## 调度监控

### 查看调度状态

```bash
# 使用 pd-ctl
pd-ctl -u http://pd1:2379 operator show

# 查看调度等待
pd-ctl -u http://pd1:2379 operator check

# 查看 Store 状态
pd-ctl -u http://pd1:2379 store
```


## 总结

PD 调度要点：

- **三大职责**：元数据管理、调度决策、TSO 分配
- **调度器类型**：Balance、Hot Region、Replica、Label
- **调度参数**：可配置调度速度、并发数
- **调度优化**：根据业务场景调整参数
