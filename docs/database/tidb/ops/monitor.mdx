# TiDB 性能监控

TiDB 提供了完善的监控体系，包括 Grafana 面板、Metrics 接口和日志分析。

## 监控架构

### 监控组件

| 组件 | 说明 |
|---|---|
| Prometheus | 指标收集和存储 |
| Grafana | 可视化展示 |
| Alertmanager | 告警通知 |
| TiDB Dashboard | TiDB 自带监控面板 |


## TiDB Dashboard

### 访问 Dashboard

```bash
# 通过 tiup 访问
tiup cluster display tidb-cluster
# 查看 Dashboard 地址

# 直接访问
# http://tidb_ip:10080/dashboard
```

## 关键监控指标

### TiDB 指标

```sql
-- 查看 TiDB 指标
-- 访问：http://tidb_ip:10080/metrics

-- 关键指标：
-- tidb_server_qps: QPS
-- tidb_query_duration_seconds: 查询延迟
-- tidb_connection_count: 连接数
-- tidb_transaction_duration_seconds: 事务延迟
-- tidb_executor_duration_seconds: 执行器延迟

-- 查看当前连接数
SHOW STATUS LIKE 'Threads_connected';

-- 查看 QPS
SHOW GLOBAL STATUS LIKE 'Com_select';

-- 查看事务统计
SHOW GLOBAL STATUS LIKE 'Com_commit';
SHOW GLOBAL STATUS LIKE 'Com_rollback';
```

### TiKV 指标

```bash
# TiKV 监控通过 Grafana

# 关键指标：
# tikv_engine_write_stall: 写入延迟
# tikv_engine_size_bytes: 存储大小
# tikv_region_count: Region 数量
# tikv_raftstore_message_count: Raft 消息数
# tikv_scheduler_concurrency: 调度器并发

# 通过 pd-ctl 查看 Region 分布
pd-ctl -u http://pd1:2379 region
pd-ctl -u http://pd1:2379 region --help
```

### PD 指标

```bash
# PD 监控

# 关键指标：
# pd_cluster_space_available: 可用空间
# pd_scheduler_status: 调度器状态
# pd_operator_status: Operator 状态
# pd_region_health: Region 健康状态

# 查看集群状态
pd-ctl -u http://pd1:2379 cluster

# 查看调度状态
pd-ctl -u http://pd1:2379 operator show
```

```bash
SELECT * FROM information_schema.metrics_schema;
```

## 告警配置

### 常用告警

```yaml
# alertmanager.yml

groups:
  - name: tidb
    rules:
      # TiDB 连接数过高
      - alert: TiDB_high_connection
        expr: tidb_connection_count > 10000
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "TiDB 连接数过高"
          description: "TiDB 连接数 {{ $value }} 超过 10000"

      # TiKV 写入延迟高
      - alert: TiKV_write_stall
        expr: tikv_engine_write_stall > 10
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "TiKV 写入延迟高"
          description: "Store {{ $labels.instance }} 写入延迟过高"

      # PD Region 不健康
      - alert: PD_region_unhealthy
        expr: count(pd_region_health_status{type="unhealthy"}) > 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Region 不健康"
          description: "有 {{ $value }} 个 Region 不健康"
```

## SQL 诊断

### 查看慢查询

```sql
-- 查看最近的慢查询
SELECT * FROM information_schema.slow_query
ORDER BY query_time DESC
LIMIT 10;

-- 查看特定时间的慢查询
SELECT * FROM information_schema.slow_query
WHERE time >= '2024-01-01 10:00:00'
ORDER BY query_time DESC;

-- 分析慢查询
EXPLAIN ANALYZE SELECT * FROM orders WHERE customer_id = 'C001';
```

### Top SQL

```sql
-- 启用 Top SQL
SET GLOBAL tidb_enable_top_sql = ON;

-- 查看 Top SQL
SELECT * FROM information_schema.top_sql_summary;

-- 查看特定 SQL 的执行统计
SELECT * FROM information_schema.top_sql_summary
WHERE query_sql LIKE '%orders%';
```

## 总结

监控要点：

- **Dashboard**：集群概览、SQL 分析、热点分析
- **Grafana**：多维度可视化监控
- **关键指标**：QPS、延迟、连接数、资源使用
- **告警**：配置 Prometheus 告警规则
- **SQL 诊断**：慢查询、Top SQL 分析
