# TiDB 故障排查

TiDB 故障排查需要从集群状态、日志、监控等多个维度进行分析。

## 常见问题

### 集群状态问题

```bash
# 1. 检查集群整体状态
tiup cluster display tidb-cluster

# 2. 检查节点状态
tiup cluster list

# 3. 查看日志
tiup cluster logs tidb-cluster -N 192.168.1.1:20160

# 4. 重启问题节点
tiup cluster restart tidb-cluster -N 192.168.1.1:20160
```

### TiDB 连接问题

```sql
-- 查看连接数
SHOW PROCESSLIST;

-- 查看最大连接数
SHOW VARIABLES LIKE 'max_connections';

-- 查看当前连接
SHOW STATUS LIKE 'Threads_connected';

-- 查看连接错误
SHOW ERRORS;
```

### 查询慢问题

```sql
-- 1. 查看当前运行的查询
SELECT * FROM information_schema.processlist
WHERE command != 'Sleep'
ORDER BY time DESC;

-- 2. 查看慢查询
SELECT * FROM information_schema.slow_query
ORDER BY query_time DESC
LIMIT 10;

-- 3. 分析执行计划
EXPLAIN ANALYZE SELECT * FROM orders WHERE customer_id = 'C001';

-- 4. 查看统计信息
SHOW STATS_META;
SHOW STATS_HISTOGRAMS;
```

## 日志分析

### 日志位置

```bash
# TiDB 日志
# 默认：/tmp/tidb.log
# 配置：log-file

# TiKV 日志
# 默认：/tmp/tikv.log

# PD 日志
# 默认：/tmp/pd.log

# 使用 tiup 查看日志
tiup cluster logs tidb-cluster
```

### 日志级别

```sql
-- 修改 TiDB 日志级别
SET GLOBAL tidb_log_level = 'debug';

-- 修改 TiKV 日志级别
# 配置文件
log-level = "debug"
```

### 常见日志分析

```bash
# 查找错误日志
grep -i "error" /tmp/tidb.log

# 查找慢查询相关日志
grep "slow query" /tmp/tidb.log

# 查找事务超时
grep "transaction timeout" /tmp/tidb.log

# 查找锁冲突
grep "lock" /tmp/tidb.log
```

## 性能问题排查

### CPU 使用率高

```bash
# 1. 查看 TiDB CPU
SHOW STATS_CPU;

# 2. 查看 Top SQL
SELECT * FROM information_schema.top_sql_summary
ORDER BY cpu_time DESC
LIMIT 10;

# 3. 检查慢查询
SELECT * FROM information_schema.slow_query
ORDER BY query_time DESC
LIMIT 10;
```

### 内存使用率高

```sql
-- 查看内存使用
SHOW STATUS LIKE 'Memory%';

-- 查看 OOM 相关配置
SHOW VARIABLES LIKE 'tidb_mem_oom_threshold';

-- 分析内存分配
SELECT * FROM information_schema.processlist
ORDER BY mem_query_digest DESC;
```

### TiKV 问题

```bash
# 查看 TiKV 状态
pd-ctl -u http://pd1:2379 store

# 查看 Region 分布
pd-ctl -u http://pd1:2379 region --limit 10

# 查看热点 Region
pd-ctl -u http://pd1:2379 region top WRITE

# 检查 RocksDB 状态
# curl http://tikv:20180/metrics | grep rocksdb
```

## PD 问题排查

### PD 不健康

```bash
# 查看 PD 成员
pd-ctl -u http://pd1:2379 member

# 检查 etcd 健康状态
pd-ctl -u http://pd1:2379 health

# 查看 etcd 日志
# journalctl -u etcd -n 100
```

### 调度问题

```bash
# 查看调度队列
pd-ctl -u http://pd1:2379 operator show

# 查看等待的 operator
pd-ctl -u http://pd1:2379 operator check

# 查看调度器状态
pd-ctl -u http://pd1:2379 scheduler show

# 查看 Region 健康
pd-ctl -u http://pd1:2379 region --jq='.regions[] | select(.region.peer_stores | length < 3)'
```

## 常见错误处理

### Region 不可用

```bash
# 1. 查看问题 Region
pd-ctl -u http://pd1:2379 region --jq='.regions[] | select(.region.peer_stores | length < 3)'

# 2. 查看 Leader
pd-ctl -u http://pd1:2379 region <region_id>

# 3. 移除不可用副本
pd-ctl -u http://pd1:2379 operator add remove-peer <region_id> <store_id>

# 4. 添加新副本
pd-ctl -u http://pd1:2379 operator add add-peer <region_id> <store_id>
```

### Leader 不存在

```bash
# 1. 转移 Leader
pd-ctl -u http://pd1:2379 operator add transfer-leader <region_id> <leader_store_id>

# 2. 移除 Follower
pd-ctl -u http://pd1:2379 operator add remove-peer <region_id> <store_id>

# 3. 添加新副本
pd-ctl -u http://pd1.2379 operator add add-peer <region_id> <new_store_id>
```

### TiDB OOM

```sql
-- 1. 查看内存使用
SHOW STATUS LIKE 'Memory%';

-- 2. 调整 OOM 阈值
SET GLOBAL tidb_mem_oom_threshold = 1073741824; -- 1GB

-- 3. 调整 tidb-server 内存限制
# 配置文件
[performance]
mem-quota-query = 1073741824

-- 4. 分析内存分配
SELECT * FROM information_schema.processlist ORDER BY mem_query_digest DESC;
```

## 故障恢复

### 数据恢复

```bash
# 1. 查看备份
tiup br validate checksum --storage "local:///backup/path"

# 2. 恢复到指定时间
tiup br restore full \
    --pd "192.168.1.1:2379" \
    --storage "s3://backup/full" \
    --restored-ts 442345678901234567

# 3. Flashback 表
FLASHBACK TABLE orders TO orders_before_drop;
```

### 集群恢复

```bash
# 1. 重启整个集群
tiup cluster restart tidb-cluster

# 2. 重启单个组件
tiup cluster restart tidb-cluster -R tidb
tiup cluster restart tidb-cluster -R tikv
tiup cluster restart tidb-cluster -R pd

# 3. 强制恢复 PD
pd-ctl -u http://pd1:2379 operator cancel all
```

## 总结

故障排查要点：

- **日志分析**：查看 TiDB/TiKV/PD 日志
- **状态检查**：使用 tiup/pd-ctl 检查状态
- **性能分析**：慢查询、Top SQL、监控
- **常见问题**：Region 不可用、OOM、调度问题
- **恢复方案**：备份恢复、Flashback、重启集群
