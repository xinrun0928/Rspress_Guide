# TiDB 备份恢复

TiDB 支持多种备份恢复方式，包括 BR 工具、物理备份、逻辑备份等。

## 备份方式对比

| 方式 | 工具 | 适用场景 | 特点 |
|---|---|---|---|
| 物理备份 | BR | 大规模备份 | 速度快，占用空间小 |
| 逻辑备份 | mydumper/tiup dumpling | 小规模备份 | 可跨版本恢复 |
| 快照备份 | Flashback | 误操作恢复 | 快速恢复 |

## BR 备份

### BR 概述

BR（Backup & Restore）是 TiDB 推荐的物理备份工具，直接备份 SST 文件。

```
BR 备份架构：

┌─────────────────────────────────────────────────────┐
│                       TiKV 集群                       │
│                                                       │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│   │  TiKV 1  │  │  TiKV 2  │  │  TiKV 3  │        │
│   │  Region │  │  Region │  │  Region │        │
│   │  (Leader)│  │(Follower)│  │(Follower)│        │
│   └─────┬────┘  └─────┬────┘  └─────┬────┘        │
│         │              │              │             │
│         └──────────────┴──────────────┘             │
│                          │                           │
└──────────────────────────┼───────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────┐
│                       BR 工具                          │
│                                                       │
│   ┌─────────────────────────────────────────────┐   │
│   │  Backup                                     │   │
│   │  - 遍历所有 Region                           │   │
│   │  - 调用 Store 备份接口                        │   │
│   │  - 写入备份存储                              │   │
│   └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
                           │
                           ▼
                   ┌───────────────┐
                   │  备份存储      │
                   │ (S3/GCS/NFS)  │
                   └───────────────┘
```

### 集群备份

```bash
# 备份整个集群
tiup br backup full \
    --pd "192.168.1.1:2379" \
    --storage "s3://backup/tidb-cluster/full" \
    --ratelimit 128 \
    --concurrency 4 \
    --log-file backup.log

# 查看备份进度
tiup br backup full --pd "192.168.1.1:2379" \
    --storage "s3://backup/tidb-cluster/full" \
    --log-file backup.log 2>&1 | grep "backup progress"

# 查看备份日志
cat backup.log
```

### 库表备份

```bash
# 备份单个数据库
tiup br backup db \
    --pd "192.168.1.1:2379" \
    --db "appdb" \
    --storage "s3://backup/tidb-cluster/appdb" \
    --log-file backup.log

# 备份单个表
tiup br backup table \
    --pd "192.168.1.1:2379" \
    --db "appdb" \
    --table "orders" \
    --storage "s3://backup/tidb-cluster/orders" \
    --log-file backup.log

# 备份多个表
tiup br backup table \
    --pd "192.168.1.1:2379" \
    --db "appdb" \
    --table "orders,customers,products" \
    --storage "s3://backup/tidb-cluster/multi-table" \
    --log-file backup.log
```

### 增量备份

```bash
# 增量备份需要先备份到某个时间点
# 然后记录 backupTSO

# 第一次全量备份
FULL_BACKUP_TS=$(tiup br backup full \
    --pd "192.168.1.1:2379" \
    --storage "s3://backup/tidb-cluster/full" \
    --log-file backup.log 2>&1 | grep "BackupTSO" | awk '{print $2}')

# 增量备份
tiup br backup point \
    --pd "192.168.1.1:2379" \
    --storage "s3://backup/tidb-cluster/inc-$FULL_BACKUP_TS" \
    --lastbackupts $FULL_BACKUP_TS \
    --log-file backup.log
```

## BR 恢复

### 全量恢复

```bash
# 恢复整个集群
tiup br restore full \
    --pd "192.168.1.1:2379" \
    --storage "s3://backup/tidb-cluster/full" \
    --log-file restore.log

# 恢复单个数据库
tiup br restore db \
    --pd "192.168.1.1:2379" \
    --db "appdb" \
    --storage "s3://backup/tidb-cluster/full" \
    --log-file restore.log

# 恢复单个表
tiup br restore table \
    --pd "192.168.1.1:2379" \
    --db "appdb" \
    --table "orders" \
    --storage "s3://backup/tidb-cluster/full" \
    --log-file restore.log
```

### 恢复到新集群

```bash
# 恢复前需要确保新集群存在
# 新集群的 PD 地址

# 恢复
tiup br restore full \
    --pd "192.168.1.2:2379" \
    --storage "s3://backup/tidb-cluster/full" \
    --log-file restore.log
```

## mydumper 逻辑备份

### mydumper 配置

```bash
# 安装 mydumper
tiup install mydumper

# 基本备份
mydumper -h 192.168.1.1 -P 4000 -u root -p password \
    -o /backup/mydumper \
    -t 4 \
    -F 64

# 备份选项
# -o: 输出目录
# -t: 并发线程数
# -F: 单文件大小（MB）
# -r: 行数分块
# -m: 只备份表结构
# -d: 只备份数据
```

### myloader 恢复

```bash
# 恢复
myloader -h 192.168.1.1 -P 4000 -u root -p password \
    -d /backup/mydumper \
    -t 4 \
    -o target_db

# 选项
# -d: 输入目录
# -o: 目标数据库（为空则自动创建）
# -t: 并发线程数
# -B: 目标数据库
```

## Flashback 恢复

### Flashback 概述

Flashback 是 TiDB 的快速恢复功能，可以恢复到指定时间点或 TSO。

### Flashback 到时间点

```sql
-- 查看当前 TSO
SHOW GLOBAL VARIABLES LIKE 'tidb_current_ts';

-- 执行 Flashback（需要 admin 权限）
FLASHBACK CLUSTER TO TIMESTAMP '2024-01-01 10:00:00';

-- Flashback 到指定 TSO
FLASHBACK CLUSTER TO TSO 442345678901234567;
```

### Flashback 表

```sql
-- 删除表后 Flashback
DROP TABLE orders;

-- 查看可恢复的表
FLASHBACK TABLE orders TO orders_before_drop;

-- Flashback 表到指定时间
FLASHBACK TABLE orders TO TSO 442345678901234567;
```

## 备份策略

### 推荐备份策略

```bash
# 备份策略设计

# 1. 全量备份：每天一次
# 2. 增量备份：每小时一次
# 3. 保留策略：每日保留 7 天，每周保留 4 周

# 备份脚本示例
#!/bin/bash

DATE=$(date +%Y%m%d)
PD_ADDR="192.168.1.1:2379"
BACKUP_DIR="/backup/tidb"

# 全量备份
tiup br backup full \
    --pd $PD_ADDR \
    --storage "local://${BACKUP_DIR}/full-${DATE}" \
    --log-file ${BACKUP_DIR}/logs/full-${DATE}.log

# 清理 7 天前的备份
find ${BACKUP_DIR}/full-* -mtime +7 -type d -exec rm -rf {} \;

# 保留 4 周的周日备份
if [ $(date +%w) -eq 0 ]; then
    cp -r ${BACKUP_DIR}/full-${DATE} ${BACKUP_DIR}/weekly-${DATE}
fi
```

## 总结

备份恢复要点：

- **BR**：物理备份，推荐用于大规模备份，速度快
- **mydumper**：逻辑备份，适合小规模，可跨版本恢复
- **Flashback**：快速恢复，支持时间点恢复
- **备份策略**：全量 + 增量，定期验证备份
