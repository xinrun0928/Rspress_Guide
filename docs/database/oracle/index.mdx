# Oracle 数据库

凌晨 2 点，某省政务云平台的数据库突然告警。

DBA 紧急排查：不是连接池问题，不是慢查询——是 Oracle RAC 的一个节点宕机了。

但奇怪的是，**应用层没有任何感知**，交易还在正常进行。

这就是 Oracle 在企业级市场屹立 40 年的底气：**极高的可用性、极强的事务处理能力、完备的安全机制**。

## 核心公式

```
Oracle 架构 = 实例（Instance） + 数据库（Database）
实例 = SGA（共享内存）+ PGA（私有内存）+ 后台进程
数据库 = 数据文件 + 日志文件 + 控制文件

SGA = Buffer Cache + Shared Pool + Redo Log Buffer + Java Pool + Streams Pool
PGA = Sort Area + Hash Area + Bitmap Merge Area

后台进程：
  DBWn = 数据库写入（脏页刷盘）
  LGWR = 日志写入（写前日志）
  CKPT = 检查点（触发 DBWn 写盘）
  SMON = 系统监控（实例恢复）
  PMON = 进程监控（清理失败进程）
  ARCn = 日志归档（归档日志）

PL/SQL 块结构 = DECLARE（声明） + BEGIN（执行） + EXCEPTION（异常） + END
```
