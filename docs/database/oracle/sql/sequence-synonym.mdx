# Oracle 序列与同义词

序列（Sequence）和同义词（Synonym）是 Oracle 数据库中两个非常实用但容易被忽视的对象。序列用于生成自增的唯一值，是主键生成的主流方案；同义词用于为对象创建别名，简化跨用户访问和对象引用的复杂度。

假设你需要为订单表生成唯一订单号，要求全局递增且不重复。如果每次插入都查询 `MAX(id) + 1`，在高并发下会产生冲突。而序列由 Oracle 内部管理，保证了在多用户并发环境下的唯一性和递增性。

## 序列

### 创建序列

```sql
CREATE SEQUENCE seq_order_id
START WITH 1          -- 起始值
INCREMENT BY 1        -- 增量
MINVALUE 1            -- 最小值
MAXVALUE 9999999999  -- 最大值
NOMAXVALUE           -- 无最大值（使用默认 10^27-1）
CYCLE                -- 循环
NOCACHE              -- 不预分配缓存
ORDER;               -- 保证顺序（高并发推荐）
```

### 序列参数详解

| 参数 | 说明 | 默认值 |
|---|---|---|
| START WITH | 起始值 | — |
| INCREMENT BY | 增量（正数递增，负数递减） | 1 |
| MINVALUE | 最小值 | NOMINVALUE（10^26 或 -10^26） |
| MAXVALUE | 最大值 | NOMAXVALUE |
| CYCLE | 达到最大值后循环 | NOCYCLE |
| CACHE | 预分配序列号数量 | 20 |
| ORDER | 保证序列号按请求顺序生成 | NOORDER |

### 序列的基本用法

```sql
-- NEXTVAL：获取下一个序列值
SELECT seq_order_id.NEXTVAL FROM DUAL;  -- 返回 1

-- CURRVAL：获取当前会话最近一次 NEXTVAL 返回的值
SELECT seq_order_id.CURRVAL FROM DUAL;  -- 返回 1

-- 在 INSERT 中使用
INSERT INTO orders (order_id, order_date, amount)
VALUES (seq_order_id.NEXTVAL, SYSDATE, 1000);

INSERT INTO orders (order_id, order_date, amount)
VALUES (seq_order_id.NEXTVAL, SYSDATE, 2000);

-- 查看序列当前值
SELECT seq_order_id.CURRVAL FROM DUAL;
```

### CACHE 与性能

序列的 CACHE 参数对性能影响很大：

- **CACHE 20**（默认）：预分配 20 个序列号，存放在 SGA 中，获取 NEXTVAL 极快
- **NOCACHE**：每次获取都访问数据字典，性能较差
- **CACHE 值过低**：高并发下序列号获取成为瓶颈
- **CACHE 值过高**：实例崩溃时丢失预分配但未使用的序列号

```sql
-- 高并发系统推荐：较大的 CACHE + ORDER
CREATE SEQUENCE seq_order_id
START WITH 1
INCREMENT BY 1
CACHE 1000
ORDER;
```

:::warning CACHE 的代价
如果实例异常崩溃（如 SHUTDOWN ABORT），预分配但未使用的序列号会丢失，导致序列号出现间隙。这是性能和连续性之间的权衡。
:::

### 修改和删除序列

```sql
-- 修改序列（部分参数不能修改，如 START WITH）
ALTER SEQUENCE seq_order_id
INCREMENT BY 2
MAXVALUE 99999999999
CACHE 500;

-- 删除序列
DROP SEQUENCE seq_order_id;
```

### 查看序列信息

```sql
-- 查看用户拥有的序列
SELECT sequence_name, min_value, max_value, increment_by,
       cache_size, last_number
FROM user_sequences;

-- 查看序列的当前值（即使未使用）
SELECT * FROM user_sequences WHERE sequence_name = 'SEQ_ORDER_ID';

-- 查看序列最后分配的值（NEXTVAL 后更新）
SELECT sequence_name, last_number
FROM user_sequences
WHERE sequence_name = 'SEQ_ORDER_ID';
```

### 序列的典型应用

**主键生成**：

```sql
CREATE TABLE users (
    user_id NUMBER PRIMARY KEY,
    username VARCHAR2(50) UNIQUE,
    email VARCHAR2(100)
);

CREATE SEQUENCE seq_user_id START WITH 1 CACHE 1000;

CREATE OR REPLACE TRIGGER trg_user_id
BEFORE INSERT ON users
FOR EACH ROW
BEGIN
    IF :NEW.user_id IS NULL THEN
        :NEW.user_id := seq_user_id.NEXTVAL;
    END IF;
END trg_user_id;
/

-- 使用时只需插入其他字段
INSERT INTO users (username, email) VALUES ('zhangsan', 'zhang@example.com');
```

**批量插入**：

```sql
-- PL/SQL 中批量获取序列
DECLARE
    TYPE num_list IS TABLE OF NUMBER INDEX BY PLS_INTEGER;
    v_ids num_list;
BEGIN
    FOR i IN 1..100 LOOP
        v_ids(i) := seq_user_id.NEXTVAL;
    END LOOP;
    
    -- 批量插入
    FORALL i IN 1..v_ids.COUNT
        INSERT INTO orders (order_id, order_date) VALUES (v_ids(i), SYSDATE);
    
    COMMIT;
END;
/
```

## 同义词

同义词是对象的别名，可以简化跨 Schema 访问、隐藏对象实际名称、提供公共接口。

### 公有同义词 vs 私有同义词

- **私有同义词**：创建在用户 Schema 下，仅当前用户可用
- **公有同义词**：创建在 PUBLIC Schema 下，所有用户可用

### 创建同义词

```sql
-- 私有同义词（默认）
CREATE SYNONYM orders FOR sh.orders;  -- sh 用户下的 orders 表

-- 公有同义词（需要 CREATE PUBLIC SYNONYM 权限）
CREATE PUBLIC SYNONYM emp FOR hr.employees;

-- 为视图创建同义词
CREATE SYNONYM my_view FOR sh.my_complex_view;

-- 为存储过程创建同义词
CREATE SYNONYM proc_test FOR user_package.procedure_name;

-- 为远程对象创建同义词（数据库链）
CREATE SYNONYM emp_remote FOR emp@remote_db;
```

### 同义词的应用场景

**场景一：跨用户访问**

```sql
-- 用户 A 访问用户 B 的表
-- 方式 1：直接使用全限定名
SELECT * FROM hr.employees;

-- 方式 2：创建同义词简化
CREATE SYNONYM employees FOR hr.employees;
SELECT * FROM employees;  -- 不需要前缀
```

**场景二：对象重命名而不影响引用**

```sql
-- 原表 orders_v1 需要替换为 orders_v2
-- 1. 创建 orders_v2 表
-- 2. 重命名同义词
CREATE OR REPLACE SYNONYM orders FOR orders_v2;
-- 3. 业务代码无需修改，仍然引用 orders
```

**场景三：隐藏数据库链信息**

```sql
-- 业务代码不直接暴露数据库链名称
CREATE PUBLIC SYNONYM remote_emp FOR emp@dblink_finance;
SELECT * FROM remote_emp;  -- 隐藏了 dblink_finance 的细节
```

### 管理同义词

```sql
-- 查看同义词
SELECT synonym_name, table_name, table_owner
FROM user_synonyms;

-- 查看公有同义词
SELECT synonym_name, table_name, table_owner
FROM all_synonyms
WHERE owner = 'PUBLIC' AND synonym_name = 'EMP';

-- 删除私有同义词
DROP SYNONYM orders;

-- 删除公有同义词
DROP PUBLIC SYNONYM emp;

-- 编译同义词（对象状态变更后）
ALTER SYNONYM orders COMPILE;
```

## 序列与同义词的结合

序列和同义词经常结合使用，简化主键生成和跨 Schema 访问：

```sql
-- DBA 在 HR Schema 下
CREATE TABLE employees (
    emp_id NUMBER PRIMARY KEY,
    name VARCHAR2(100)
);

CREATE SEQUENCE seq_emp_id CACHE 1000;

CREATE PUBLIC SYNONYM emp_seq FOR hr.seq_emp_id;
CREATE PUBLIC SYNONYM employees FOR hr.employees;

-- 普通用户在应用 Schema 下
INSERT INTO employees VALUES (seq_emp_id.NEXTVAL, '张三');
INSERT INTO employees VALUES (seq_emp_id.NEXTVAL, '李四');
```

## 常见面试问题

**问：序列的 CACHE 参数对性能和序列连续性有什么影响？**

CACHE 参数预分配序列号存放在 SGA 中，获取 NEXTVAL 时无需访问数据字典，性能很高。但实例异常崩溃时，预分配但未使用的序列号会丢失，导致序列号出现间隙。NOCACHE 每次都访问数据字典，性能较差但序列号不会丢失。

**问：ORDER 参数的作用是什么？**

ORDER 保证序列号按照请求的顺序生成（需要 Enqueue 锁）。在高并发多节点环境下，如果不使用 ORDER，不同节点的序列号可能不按请求顺序递增。ORDER 保证全局顺序，但会带来一定的性能开销。

**问：同义词有什么作用？**

同义词是对象的别名，主要作用包括：简化跨 Schema 访问（不需要写完整的 `schema.table`）；提供对象抽象层（底层表改名后只需修改同义词）；隐藏数据库链信息（通过同义词引用远程对象）；创建公共 API 接口。

**问：公有同义词和私有同义词的优先级是什么？**

当用户引用一个对象名时，Oracle 先查找用户的私有同义词，再查找公有同义词，最后查找用户 Schema 下的对象。如果私有同义词和公有同义词重名，私有同义词优先。

## 总结

序列是 Oracle 生成唯一自增键的标准方案，配合 CACHE 参数可以提供高性能的主键生成。同义词是对象引用的抽象层，可以简化跨 Schema 访问、隐藏实现细节、提供灵活的接口设计。两者都是数据库日常开发中经常使用的对象。
