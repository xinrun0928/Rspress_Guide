# Oracle 异常处理

异常处理是 PL/SQL 程序健壮性的保障。再完美的代码也可能遇到各种意外情况：网络中断、数据冲突、资源耗尽……异常处理机制让你能够优雅地应对这些情况，而不是让程序崩溃或产生不可预期的结果。

假设一个银行转账系统，转出账户扣款成功后，转入账户加款时发生网络故障。如果程序没有异常处理，转出账户的钱被扣了，但转入账户没收到——这就是"数据不一致"的典型案例。正确的异常处理应该是：检测到异常后，回滚整个事务，确保两边数据都回到转账前的状态。

## 异常的基本概念

异常（Exception）是 PL/SQL 程序运行过程中发生的错误或特殊状态。当异常被抛出时，程序的控制权转移到异常处理块。如果没有异常处理块，异常会向上传播直到被捕获或导致程序终止。

### 异常分类

**预定义异常**：Oracle 预定义的错误类型，如 NO_DATA_FOUND、TOO_MANY_ROWS 等。

**非预定义异常**：Oracle 预定义但没有预置名称的异常，需要通过 `PRAGMA EXCEPTION_INIT` 关联。

**用户定义异常**：由程序员显式抛出的异常。

## 预定义异常

Oracle 为常见的错误预定义了大量异常名称：

```sql
DECLARE
    v_name VARCHAR2(100);
BEGIN
    -- ORA-01403: 未找到数据
    SELECT first_name INTO v_name
    FROM employees
    WHERE employee_id = 99999;
    
    DBMS_OUTPUT.PUT_LINE(v_name);
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('未找到数据');
END;
/

DECLARE
    v_name VARCHAR2(100);
BEGIN
    -- ORA-01422: 返回多行
    SELECT first_name INTO v_name
    FROM employees
    WHERE department_id = 50;
EXCEPTION
    WHEN TOO_MANY_ROWS THEN
        DBMS_OUTPUT.PUT_LINE('返回多行数据');
END;
/
```

### 常用预定义异常

| 异常名称 | Oracle 错误码 | 说明 |
|---|---|---|
| NO_DATA_FOUND | ORA-01403 | SELECT INTO 未返回数据 |
| TOO_MANY_ROWS | ORA-01422 | SELECT INTO 返回多行 |
| DUP_VAL_ON_INDEX | ORA-00001 | 唯一索引冲突 |
| INVALID_CURSOR | ORA-01001 | 游标操作无效 |
| CURSOR_ALREADY_OPEN | ORA-06511 | 游标已打开 |
| NOT_LOGGED_ON | ORA-01012 | 未连接数据库 |
| STORAGE_ERROR | ORA-06500 | 内存不足 |
| VALUE_ERROR | ORA-06502 | 数值或转换错误 |
| ZERO_DIVIDE | ORA-01476 | 除数为零 |
| OTHERS | — | 捕获所有异常（应放在最后） |

## 非预定义异常

Oracle 定义了错误码但没有预定义名称的异常，可以通过 `PRAGMA EXCEPTION_INIT` 关联：

```sql
DECLARE
    e_child_exists EXCEPTION;  -- 声明异常
    PRAGMA EXCEPTION_INIT(e_child_exists, -02292);  -- 关联 ORA-02292（外键约束冲突）
BEGIN
    DELETE FROM departments WHERE department_id = 50;
EXCEPTION
    WHEN e_child_exists THEN
        DBMS_OUTPUT.PUT_LINE('无法删除：存在关联的子记录');
END;
/
```

### 常用约束错误码

| 错误码 | 说明 |
|---|---|
| ORA-00001 | 唯一约束冲突 |
| ORA-02291 | 外键约束不存在 |
| ORA-02292 | 外键约束冲突（子记录存在） |
| ORA-02443 | 约束不存在 |
| ORA-04068 | 包状态已丢弃 |
| ORA-04091 | Mutating Table 错误 |

## 用户定义异常

程序员可以显式抛出自己的异常：

```sql
DECLARE
    e_invalid_salary EXCEPTION;  -- 声明异常
    e_salary_too_high EXCEPTION; -- 声明异常
BEGIN
    UPDATE employees SET salary = 500000 WHERE employee_id = 100;
    
    IF SQL%ROWCOUNT > 0 THEN
        -- 检查是否超过合理范围
        IF 500000 > 100000 THEN
            RAISE e_salary_too_high;
        END IF;
    END IF;
EXCEPTION
    WHEN e_salary_too_high THEN
        DBMS_OUTPUT.PUT_LINE('工资过高，拒绝更新');
        ROLLBACK;
END;
/
```

### RAISE_APPLICATION_ERROR

`RAISE_APPLICATION_ERROR` 可以抛出带有自定义错误码和消息的异常：

```sql
BEGIN
    DELETE FROM employees WHERE employee_id = 100;
    
    IF SQL%NOTFOUND THEN
        RAISE_APPLICATION_ERROR(-20001, '员工不存在');
    END IF;
    
    COMMIT;
EXCEPTION
    WHEN OTHERS THEN
        IF SQLCODE = -20001 THEN
            DBMS_OUTPUT.PUT_LINE('业务错误: ' || SQLERRM);
        ELSE
            DBMS_OUTPUT.PUT_LINE('系统错误: ' || SQLERRM);
        END IF;
        ROLLBACK;
END;
/
```

**错误码范围**：自定义错误码必须在 -20999 到 -20000 之间。

## 异常传播

如果在内部块中未处理异常，异常会传播到外部块：

```sql
BEGIN
    BEGIN
        SELECT * FROM nonexistent_table;
    EXCEPTION
        WHEN OTHERS THEN
            DBMS_OUTPUT.PUT_LINE('内部捕获: ' || SQLERRM);
            -- 重新抛出异常
            RAISE;
    END;
    
    -- 这行不会执行，因为 RAISE 抛出异常
    DBMS_OUTPUT.PUT_LINE('这行不会执行');
EXCEPTION
    WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('外部捕获: ' || SQLERRM);
END;
/
```

### 内部块不捕获异常

```sql
BEGIN
    BEGIN
        -- 这里抛出的异常会传播到外部
        RAISE_APPLICATION_ERROR(-20001, '来自内部');
    END;
    
    -- 这里不会执行
    DBMS_OUTPUT.PUT_LINE('不会执行');
EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('被外部捕获');
END;
/
```

## SQLCODE 和 SQLERRM

在异常处理中使用这两个函数获取错误信息：

```sql
EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('错误码: ' || SQLCODE);    -- 如 -20001
        DBMS_OUTPUT.PUT_LINE('错误信息: ' || SQLERRM);  -- 如 ORA-20001: ...
        
        -- 获取错误堆栈
        DBMS_OUTPUT.PUT_LINE(DBMS_UTILITY.FORMAT_ERROR_STACK);
        DBMS_OUTPUT.PUT_LINE(DBMS_UTILITY.FORMAT_ERROR_BACKTRACE);
END;
/
```

### DBMS_UTILITY 错误追踪

```sql
EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('错误堆栈:');
        DBMS_OUTPUT.PUT_LINE(DBMS_UTILITY.FORMAT_ERROR_STACK);
        DBMS_OUTPUT.PUT_LINE('调用堆栈:');
        DBMS_OUTPUT.PUT_LINE(DBMS_UTILITY.FORMAT_CALL_STACK);
        DBMS_OUTPUT.PUT_LINE('背Trace:');
        DBMS_OUTPUT.PUT_LINE(DBMS_UTILITY.FORMAT_ERROR_BACKTRACE);
END;
/
```

## 异常处理最佳实践

### 记录错误日志

```sql
CREATE TABLE error_log (
    log_id NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    error_date TIMESTAMP DEFAULT SYSTIMESTAMP,
    user_name VARCHAR2(50),
    error_code NUMBER,
    error_message VARCHAR2(4000),
    call_stack VARCHAR2(4000)
);

CREATE OR REPLACE PROCEDURE log_error IS
    PRAGMA AUTONOMOUS_TRANSACTION;  -- 自治事务
BEGIN
    INSERT INTO error_log (user_name, error_code, error_message, call_stack)
    VALUES (
        USER,
        SQLCODE,
        SQLERRM,
        DBMS_UTILITY.FORMAT_ERROR_BACKTRACE
    );
    COMMIT;
END log_error;
/

BEGIN
    -- 业务代码
    NULL;
EXCEPTION
    WHEN OTHERS THEN
        log_error;  -- 记录错误
        RAISE;       -- 重新抛出
END;
/
```

### 自治事务

使用 `PRAGMA AUTONOMOUS_TRANSACTION` 可以在事务中开启独立事务，确保错误日志被持久化，即使主事务回滚：

```sql
CREATE OR REPLACE PACKAGE error_pkg AS
    PROCEDURE log_error(p_code NUMBER, p_message VARCHAR2);
END error_pkg;
/

CREATE OR REPLACE PACKAGE BODY error_pkg AS
    PROCEDURE log_error(p_code NUMBER, p_message VARCHAR2) IS
        PRAGMA AUTONOMOUS_TRANSACTION;
    BEGIN
        INSERT INTO error_log (user_name, error_code, error_message)
        VALUES (USER, p_code, p_message);
        COMMIT;
    END;
END error_pkg;
/
```

## 常见面试问题

**问：预定义异常和用户定义异常有什么区别？**

预定义异常是 Oracle 预先定义的名称，如 NO_DATA_FOUND、TOO_MANY_ROWS，无需声明即可使用。非预定义异常是 Oracle 预定义但没有名称的异常（如外键约束冲突 ORA-02292），需要用 `PRAGMA EXCEPTION_INIT` 关联。用户定义异常是程序员显式声明和抛出的异常。

**问：OTHERS 异常处理器的注意事项有哪些？**

OTHERS 捕获所有未显式处理的异常，应放在 WHEN 子句的最后；使用 OTHERS 后应记录错误日志或重新抛出异常；避免在 OTHERS 中忽略异常什么都不做；可以配合 SQLCODE 和 SQLERRM 获取错误详情。

**问：RAISE_APPLICATION_ERROR 和 RAISE 的区别是什么？**

RAISE 重新抛出当前捕获的异常，保持原有错误码和信息。RAISE_APPLICATION_ERROR 抛出用户自定义错误，必须指定错误码（-20999 到 -20000 之间）和错误信息，常用于业务规则验证失败时返回明确的错误给调用者。

## 总结

异常处理是 PL/SQL 程序健壮性的关键。预定义异常覆盖了常见错误类型；通过 `PRAGMA EXCEPTION_INIT` 可以关联 Oracle 错误码；用户定义异常和 `RAISE_APPLICATION_ERROR` 提供了灵活的错误报告机制。异常传播规则决定了异常如何层层上抛，自治事务确保错误日志不受主事务影响。
