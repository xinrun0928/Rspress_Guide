# Oracle 游标

游标是 Oracle 数据库处理结果集的机制。当你需要逐行处理查询结果时，游标提供了灵活的控制能力。虽然现代 SQL 鼓励使用集合操作，但某些场景下，游标仍然是不可或缺的工具。

假设你需要从订单表中读取所有逾期订单，对每条订单执行复杂计算，然后根据计算结果决定是发送催款通知还是自动取消订单。这种逐行处理、业务逻辑复杂的场景，游标是最佳选择。

## 游标概述

游标就像数据库中的"指针"，指向结果集中的某一行，允许你逐行遍历和处理数据。

Oracle 支持两类游标：

**隐式游标**：Oracle 自动为所有 SQL 语句创建和维护的游标。DML 操作和 SELECT INTO 语句使用隐式游标。

**显式游标**：由程序员显式声明和控制的游标。用于处理返回多行的查询。

```sql
-- 隐式游标：Oracle 自动创建
INSERT INTO employees VALUES (...);  -- 使用隐式游标

SELECT * INTO v_emp FROM employees WHERE employee_id = 100;  -- 使用隐式游标

-- 显式游标：程序员声明
CURSOR c_emp IS SELECT * FROM employees;
```

## 隐式游标

Oracle 为每条 SQL 语句自动创建隐式游标，开发者无法控制其打开和关闭，但可以访问其属性。

### 隐式游标属性

| 属性 | 说明 |
|---|---|
| SQL%FOUND | 最近一次操作影响了至少一行时返回 TRUE |
| SQL%NOTFOUND | 最近一次操作未影响任何行时返回 TRUE |
| SQL%ROWCOUNT | 最近一次操作影响的行数 |
| SQL%ISOPEN | 对于隐式游标始终返回 FALSE |

```sql
BEGIN
    UPDATE employees SET salary = salary * 1.1 WHERE department_id = 999;
    
    IF SQL%FOUND THEN
        DBMS_OUTPUT.PUT_LINE('更新了 ' || SQL%ROWCOUNT || ' 行');
    ELSE
        DBMS_OUTPUT.PUT_LINE('没有找到匹配的记录');
    END IF;
END;
/

BEGIN
    DELETE FROM employees WHERE employee_id = 9999;
    
    IF SQL%NOTFOUND THEN
        DBMS_OUTPUT.PUT_LINE('员工不存在');
    END IF;
END;
/
```

## 显式游标

显式游标需要程序员显式声明、打开、读取和关闭。

### 基本语法

```sql
-- 1. 声明游标
CURSOR cursor_name IS select_statement;

-- 2. 打开游标
OPEN cursor_name;

-- 3. 读取数据
FETCH cursor_name INTO variable_list;

-- 4. 关闭游标
CLOSE cursor_name;
```

### 显式游标属性

| 属性 | 说明 |
|---|---|
| cursor_name%FOUND | 最近一次 FETCH 成功获取了数据时返回 TRUE |
| cursor_name%NOTFOUND | 最近一次 FETCH 未获取数据时返回 TRUE |
| cursor_name%ROWCOUNT | 累计 FETCH 的行数 |
| cursor_name%ISOPEN | 游标是否打开 |

### 完整示例

```sql
DECLARE
    -- 声明记录类型
    TYPE emp_record_type IS RECORD (
        emp_id    employees.employee_id%TYPE,
        emp_name  VARCHAR2(100),
        emp_salary employees.salary%TYPE
    );
    
    v_emp emp_record_type;
    
    -- 声明游标
    CURSOR c_emp IS
        SELECT employee_id, first_name || ' ' || last_name, salary
        FROM employees
        WHERE department_id = 50
        ORDER BY salary DESC;
BEGIN
    OPEN c_emp;
    
    LOOP
        FETCH c_emp INTO v_emp;
        EXIT WHEN c_emp%NOTFOUND;
        
        DBMS_OUTPUT.PUT_LINE(
            'ID: ' || v_emp.emp_id || 
            ', Name: ' || v_emp.emp_name || 
            ', Salary: ' || v_emp.emp_salary
        );
    END LOOP;
    
    CLOSE c_emp;
    
    DBMS_OUTPUT.PUT_LINE('总共处理: ' || c_emp%ROWCOUNT || ' 行');
END;
/
```

### 使用 %ROWTYPE 简化

```sql
DECLARE
    -- 使用 %ROWTYPE，无需定义记录类型
    v_emp employees%ROWTYPE;
    
    CURSOR c_emp IS
        SELECT * FROM employees WHERE department_id = 50;
BEGIN
    OPEN c_emp;
    
    FETCH c_emp INTO v_emp;
    WHILE c_emp%FOUND LOOP
        DBMS_OUTPUT.PUT_LINE(v_emp.first_name || ' ' || v_emp.last_name);
        FETCH c_emp INTO v_emp;
    END LOOP;
    
    CLOSE c_emp;
END;
/
```

### FOR 循环简化游标处理

PL/SQL 提供了简化的游标 FOR 循环，自动处理打开、读取和关闭：

```sql
-- 方式 1：显式声明游标
BEGIN
    FOR rec IN c_emp LOOP
        DBMS_OUTPUT.PUT_LINE(rec.first_name || ' ' || rec.last_name);
    END LOOP;
END;
/

-- 方式 2：子查询方式（更简洁）
BEGIN
    FOR rec IN (SELECT * FROM employees WHERE department_id = 50) LOOP
        DBMS_OUTPUT.PUT_LINE(rec.first_name || ' ' || rec.last_name);
    END LOOP;
END;
/

-- 方式 3：显式声明 + FOR 循环
DECLARE
    CURSOR c_emp IS
        SELECT * FROM employees WHERE department_id = 50;
BEGIN
    FOR rec IN c_emp LOOP
        DBMS_OUTPUT.PUT_LINE(rec.first_name || ' ' || rec.last_name);
    END LOOP;
END;
/
```

## 带参数的游标

游标可以接受参数，使查询更加灵活：

```sql
DECLARE
    CURSOR c_emp(p_dept_id NUMBER, p_min_salary NUMBER) IS
        SELECT employee_id, first_name, salary
        FROM employees
        WHERE department_id = p_dept_id
          AND salary >= p_min_salary
        ORDER BY salary DESC;
BEGIN
    DBMS_OUTPUT.PUT_LINE('--- 部门 50，工资 >= 5000 ---');
    FOR rec IN c_emp(50, 5000) LOOP
        DBMS_OUTPUT.PUT_LINE(rec.first_name || ': ' || rec.salary);
    END LOOP;
    
    DBMS_OUTPUT.PUT_LINE('--- 部门 80，工资 >= 8000 ---');
    FOR rec IN c_emp(80, 8000) LOOP
        DBMS_OUTPUT.PUT_LINE(rec.first_name || ': ' || rec.salary);
    END LOOP;
END;
/
```

## 游标的锁定与更新

在打开游标时，可以使用 `FOR UPDATE` 对结果集加锁：

```sql
DECLARE
    CURSOR c_emp IS
        SELECT employee_id, salary
        FROM employees
        WHERE department_id = 50
        FOR UPDATE;  -- 加排他锁
BEGIN
    FOR rec IN c_emp LOOP
        -- 在锁的范围内，可以安全地 UPDATE
        UPDATE employees
        SET salary = salary * 1.05
        WHERE CURRENT OF c_emp;
    END LOOP;
    
    COMMIT;
END;
/
```

### CURRENT OF 语法

`CURRENT OF cursor_name` 引用游标当前行的 ROWID，用于定位更新：

```sql
CURSOR c IS SELECT * FROM t1 FOR UPDATE;
-- ...
UPDATE t1 SET col = val WHERE CURRENT OF c;
```

### FOR UPDATE NOWAIT 与 FOR UPDATE SKIP LOCKED

```sql
-- NOWAIT：立即返回，不等待锁
CURSOR c IS SELECT * FROM t1 FOR UPDATE NOWAIT;

-- SKIP LOCKED：跳过已锁定的行
CURSOR c IS SELECT * FROM t1 FOR UPDATE SKIP LOCKED;
```

## REF 游标：动态游标

REF 游标允许在运行时动态确定查询语句：

```sql
DECLARE
    -- 声明 REF CURSOR 类型
    TYPE ref_cursor_type IS REF CURSOR;
    v_cursor ref_cursor_type;
    
    v_empno NUMBER;
    v_ename VARCHAR2(100);
    v_query VARCHAR2(500);
BEGIN
    -- 动态构建 SQL
    v_query := 'SELECT employee_id, first_name FROM employees WHERE department_id = :1';
    
    OPEN v_cursor FOR v_query USING 50;
    
    LOOP
        FETCH v_cursor INTO v_empno, v_ename;
        EXIT WHEN v_cursor%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_empno || ': ' || v_ename);
    END LOOP;
    
    CLOSE v_cursor;
END;
/
```

### 强类型与弱类型 REF 游标

```sql
-- 强类型 REF CURSOR（指定返回类型）
TYPE emp_cursor_type IS REF CURSOR RETURN employees%ROWTYPE;

-- 弱类型 REF CURSOR（不指定返回类型）
TYPE generic_cursor_type IS REF CURSOR;
```

## SYS_REFCURSOR

Oracle 10g 引入了预定义的 `SYS_REFCURSOR` 类型，简化了 REF 游标的使用：

```sql
CREATE OR REPLACE FUNCTION get_employees(p_dept_id NUMBER)
RETURN SYS_REFCURSOR IS
    v_cursor SYS_REFCURSOR;
BEGIN
    OPEN v_cursor FOR
        SELECT employee_id, first_name, salary
        FROM employees
        WHERE department_id = p_dept_id;
    RETURN v_cursor;
END get_employees;
/

-- Java 中调用返回 SYS_REFCURSOR 的函数
CallableStatement cs = conn.prepareCall("{ ? = call get_employees(?) }");
cs.registerOutParameter(1, OracleTypes.CURSOR);
cs.setInt(2, 50);
cs.execute();
ResultSet rs = (ResultSet) cs.getObject(1);
```

## 常见面试问题

**问：隐式游标和显式游标的区别是什么？**

隐式游标由 Oracle 自动创建和管理，用于处理单行操作的 SELECT、DML 语句；开发者无法控制其生命周期，只能访问其属性（SQL%FOUND、SQL%ROWCOUNT 等）。显式游标由程序员显式声明和控制，可以处理多行查询，提供细粒度的遍历控制，包括 OPEN、FETCH、CLOSE 生命周期管理。

**问：游标的 %FOUND 和 %NOTFOUND 在什么时机判断？**

在 FETCH 之后判断：若最近一次 FETCH 成功获取了数据，`%FOUND` 返回 TRUE；否则返回 FALSE。首次 FETCH 之前，两者都返回 NULL，这是需要注意的陷阱——循环开始时不能直接用 `EXIT WHEN cursor%NOTFOUND`，否则首行会被跳过。正确写法是在 FETCH 之后判断，或使用 `FOR rec IN cursor LOOP` 让 PL/SQL 自动处理。

**问：FOR UPDATE 的作用是什么？**

FOR UPDATE 在打开游标时对结果集加排他锁，锁定被查询的行，防止其他会话修改。在事务提交或回滚后锁自动释放。常用于"读取-修改-更新"的场景，确保在修改时行未被其他事务修改。FOR UPDATE NOWAIT 立即返回，FOR UPDATE SKIP LOCKED 跳过已锁行。

## 总结

游标是 PL/SQL 处理结果集的基础工具。隐式游标适用于简单的单行操作，显式游标提供了逐行处理的灵活性，REF 游标支持动态 SQL。游标 FOR 循环是最简洁的遍历方式，而 FOR UPDATE 提供了事务性的锁定机制。理解游标的生命周期和属性，是掌握 PL/SQL 数据处理的关键。
