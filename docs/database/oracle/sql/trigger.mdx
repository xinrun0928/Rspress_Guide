# Oracle 触发器

触发器是 Oracle 数据库中一类特殊的存储过程，它们不能被直接调用，而是在特定事件发生时自动执行。触发器是实现复杂业务规则、审计跟踪、数据验证的利器——当你需要"每当某张表被修改时，自动执行某个操作"时，触发器就是答案。

假设你在管理一个金融系统，需要记录所有金额变更的审计日志。每次 UPDATE 或 DELETE 操作后，你希望自动记录变更前的值、变更后的值、变更时间、操作人。如果不用触发器，你需要在每一处业务代码中手动添加日志逻辑；而用触发器，这个行为可以被统一、自动、透明地实现。

## 触发器概述

触发器由三部分组成：**触发事件**（什么操作）、**触发条件**（什么时候触发）、**触发操作**（做什么）。

### 触发时机

- **BEFORE**：在触发事件执行之前触发
- **AFTER**：在触发事件执行之后触发
- **INSTEAD OF**：代替触发事件执行（仅用于视图）

### 触发事件

- **DML 触发器**：INSERT、UPDATE、DELETE（行级或语句级）
- **DDL 触发器**：CREATE、ALTER、DROP
- **数据库触发器**：SERVERERROR、LOGON、LOGOFF、STARTUP、SHUTDOWN

### 触发类型

- **行级触发器（FOR EACH ROW）**：每影响一行就触发一次
- **语句级触发器**：每执行一条语句触发一次

```sql
-- 语句级触发器示例
CREATE OR REPLACE TRIGGER trg_before_emp_insert
BEFORE INSERT ON employees
BEGIN
    IF TO_CHAR(SYSDATE, 'HH24') NOT BETWEEN 8 AND 18 THEN
        RAISE_APPLICATION_ERROR(-20001, '只能在工作时间创建员工');
    END IF;
END trg_before_emp_insert;
/

-- 行级触发器示例
CREATE OR REPLACE TRIGGER trg_after_salary_update
AFTER UPDATE OF salary ON employees
FOR EACH ROW
BEGIN
    DBMS_OUTPUT.PUT_LINE('工资变更: ' || :OLD.salary || ' -> ' || :NEW.salary);
END trg_after_salary_update;
/
```

## DML 触发器

DML 触发器是最常用的触发器类型，用于监听表的 INSERT、UPDATE、DELETE 操作。

### 语法结构

```sql
CREATE [OR REPLACE] TRIGGER trigger_name
[BEFORE | AFTER | INSTEAD OF]
[INSERT | UPDATE [OF column_list] | DELETE]
ON [schema.]table_name | [schema.]view_name
[FOR EACH ROW [WHEN (condition)]]
[ENABLE / DISABLE]
[DECLARE
    declaration_statements]
BEGIN
    trigger_body
[EXCEPTION
    exception_handlers]
END [trigger_name];
/
```

### :OLD 和 :NEW 引用

在行级触发器中，`:OLD` 和 `:NEW` 分别代表修改前后的行数据：

| 操作 | :OLD 值 | :NEW 值 |
|---|---|---|
| INSERT | NULL | 新插入的值 |
| UPDATE | 修改前的值 | 修改后的值 |
| DELETE | 删除前的值 | NULL |

```sql
-- 审计日志触发器
CREATE TABLE employee_audit (
    audit_id NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    employee_id NUMBER,
    action VARCHAR2(10),
    old_salary NUMBER,
    new_salary NUMBER,
    changed_by VARCHAR2(50),
    changed_at TIMESTAMP DEFAULT SYSTIMESTAMP
);

CREATE OR REPLACE TRIGGER trg_emp_salary_audit
AFTER UPDATE OF salary ON employees
FOR EACH ROW
BEGIN
    INSERT INTO employee_audit (employee_id, action, old_salary, new_salary, changed_by)
    VALUES (:OLD.employee_id, 'UPDATE', :OLD.salary, :NEW.salary, USER);
END trg_emp_salary_audit;
/

-- 当 UPDATE 发生时，自动记录审计日志
UPDATE employees SET salary = salary * 1.1 WHERE department_id = 50;
```

### WHEN 子句

使用 WHEN 子句可以添加额外的触发条件，只有满足条件时才执行触发器：

```sql
CREATE OR REPLACE TRIGGER trg_salary_check
BEFORE UPDATE OF salary ON employees
FOR EACH ROW
WHEN (NEW.salary < OLD.salary)  -- 仅当降薪时触发
BEGIN
    RAISE_APPLICATION_ERROR(-20001, '不允许降薪');
END trg_salary_check;
/
```

### 组合触发事件

一个触发器可以监听多个事件：

```sql
-- 监听 INSERT 和 UPDATE
CREATE OR REPLACE TRIGGER trg_emp_log
AFTER INSERT OR UPDATE OR DELETE ON employees
FOR EACH ROW
BEGIN
    IF INSERTING THEN
        DBMS_OUTPUT.PUT_LINE('INSERT: ' || :NEW.employee_id);
    ELSIF UPDATING THEN
        DBMS_OUTPUT.PUT_LINE('UPDATE: ' || :OLD.employee_id || ' -> ' || :NEW.employee_id);
    ELSIF DELETING THEN
        DBMS_OUTPUT.PUT_LINE('DELETE: ' || :OLD.employee_id);
    END IF;
END trg_emp_log;
/
```

### 谓词条件

在触发器体内，可以使用 `INSERTING`、`UPDATING`、`DELETING` 谓词判断触发事件类型：

```sql
CREATE OR REPLACE TRIGGER trg_audit
AFTER INSERT OR UPDATE OR DELETE ON employees
FOR EACH ROW
BEGIN
    IF INSERTING THEN
        -- INSERT 逻辑
        INSERT INTO audit_log VALUES ('INSERT', :NEW.employee_id, USER, SYSDATE);
    ELSIF UPDATING THEN
        -- UPDATE 逻辑
        INSERT INTO audit_log VALUES ('UPDATE', :OLD.employee_id, USER, SYSDATE);
    ELSIF DELETING THEN
        -- DELETE 逻辑
        INSERT INTO audit_log VALUES ('DELETE', :OLD.employee_id, USER, SYSDATE);
    END IF;
END trg_audit;
/
```

## INSTEAD OF 触发器

INSTEAD OF 触发器用于更新不可更新的视图。Oracle 不允许直接对复杂视图执行 DML，但可以通过 INSTEAD OF 触发器实现逻辑：

```sql
-- 创建复杂视图（包含 JOIN）
CREATE OR REPLACE VIEW emp_dept_view AS
SELECT e.employee_id, e.first_name, e.salary,
       d.department_id, d.department_name
FROM employees e
LEFT JOIN departments d ON e.department_id = d.department_id;

-- 创建 INSTEAD OF 触发器处理 INSERT
CREATE OR REPLACE TRIGGER trg_view_insert
INSTEAD OF INSERT ON emp_dept_view
FOR EACH ROW
DECLARE
    v_dept_id NUMBER;
BEGIN
    -- 先插入部门（如果不存在）
    BEGIN
        SELECT department_id INTO v_dept_id
        FROM departments WHERE department_name = :NEW.department_name;
    EXCEPTION
        WHEN NO_DATA_FOUND THEN
            INSERT INTO departments (department_name)
            VALUES (:NEW.department_name)
            RETURNING department_id INTO v_dept_id;
    END;
    
    -- 插入员工
    INSERT INTO employees (employee_id, first_name, salary, department_id)
    VALUES (:NEW.employee_id, :NEW.first_name, :NEW.salary, v_dept_id);
END trg_view_insert;
/

-- 现在可以对视图执行 INSERT
INSERT INTO emp_dept_view VALUES (1001, '张三', 10000, NULL, '新部门');
```

## 触发器执行顺序与 Mutating Table 问题

### Mutating Table 问题

行级触发器存在一个著名问题：**Mutating Table**。当触发器尝试查询或修改触发它的表本身时，会触发 ORA-04091 错误：

```sql
-- 错误示例：触发器中查询触发表
CREATE OR REPLACE TRIGGER trg_count_employees
AFTER INSERT ON employees
FOR EACH ROW
DECLARE
    v_count NUMBER;
BEGIN
    SELECT COUNT(*) INTO v_count FROM employees;  -- ORA-04091!
END trg_count_employees;
/
```

### 解决方案：使用复合触发器或包

Oracle 11g 引入了复合触发器，可以在同一个触发器中定义 BEFORE STATEMENT、BEFORE ROW、AFTER ROW、AFTER STATEMENT 四个部分，从而在 STATEMENT 阶段访问表：

```sql
CREATE OR REPLACE TRIGGER trg_employee_audit
FOR INSERT ON employees
COMPOUND TRIGGER

    TYPE t_audit_rec IS RECORD (
        emp_id NUMBER,
        salary NUMBER
    );
    TYPE t_audit_list IS TABLE OF t_audit_rec INDEX BY PLS_INTEGER;
    v_audit_list t_audit_list;
    v_index PLS_INTEGER := 0;

    AFTER EACH ROW IS
    BEGIN
        v_index := v_index + 1;
        v_audit_list(v_index).emp_id := :NEW.employee_id;
        v_audit_list(v_index).salary := :NEW.salary;
    END AFTER EACH ROW;

    AFTER STATEMENT IS
    BEGIN
        FOR i IN 1..v_index LOOP
            INSERT INTO audit_log VALUES (v_audit_list(i).emp_id, v_audit_list(i).salary);
        END LOOP;
    END AFTER STATEMENT;

END trg_employee_audit;
/
```

## 禁用与启用触发器

```sql
-- 禁用触发器
ALTER TRIGGER trg_emp_log DISABLE;

-- 启用触发器
ALTER TRIGGER trg_emp_log ENABLE;

-- 禁用表上所有触发器
ALTER TABLE employees DISABLE ALL TRIGGERS;

-- 启用表上所有触发器
ALTER TABLE employees ENABLE ALL TRIGGERS;
```

## 管理触发器

```sql
-- 查看触发器源代码
SELECT trigger_name, trigger_type, triggering_event, table_name, status
FROM user_triggers;

-- 查看触发器依赖的对象
SELECT name, type, referenced_name, referenced_type
FROM user_dependencies
WHERE type = 'TRIGGER' AND name = 'TRIGGER_NAME';

-- 重新编译触发器
ALTER TRIGGER trg_name COMPILE;

-- 删除触发器
DROP TRIGGER trg_name;
```

## 常见面试问题

**问：行级触发器和语句级触发器的区别是什么？**

行级触发器使用 `FOR EACH ROW`，每影响一行就触发一次，可以在触发器中使用 `:OLD` 和 `:NEW` 引用修改前后的值。语句级触发器每执行一条语句触发一次，不论影响多少行，在触发器中不能使用 `:OLD` 和 `:NEW`。

**问：什么是 Mutating Table 问题？如何解决？**

Mutating Table 发生在行级触发器尝试查询或修改触发它的表本身时，会触发 ORA-04091 错误。解决方案包括：使用复合触发器在 STATEMENT 阶段访问表、使用包变量暂存数据、使用 autonomous transaction（自治事务）分离事务上下文。

**问：BEFORE 和 AFTER 触发器的区别是什么？**

BEFORE 触发器在操作执行前触发，常用于数据验证、修改待插入的值；AFTER 触发器在操作执行后触发，常用于审计日志、统计更新等。选择哪种取决于业务需求——需要基于新值做判断用 BEFORE，需要基于最终结果做记录用 AFTER。

## 总结

触发器是 Oracle 实现自动化业务逻辑的重要工具。DML 触发器用于监听表的变更，INSTEAD OF 触发器用于处理不可更新的视图。行级触发器提供了 `:OLD` 和 `:NEW` 引用，但需要注意 Mutating Table 问题。合理使用触发器可以使业务逻辑更加内聚，减少应用层代码。
