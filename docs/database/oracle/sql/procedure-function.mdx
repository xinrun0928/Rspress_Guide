# Oracle 存储过程与函数

存储过程和函数是 PL/SQL 编程的核心组件，它们将 PL/SQL 代码持久化在数据库中，供应用程序重复调用。相比每次都发送完整的 SQL 语句，调用存储过程或函数可以显著减少网络流量，同时实现业务逻辑的集中管理。

假设你需要定期清理过期数据、计算用户积分、更新统计报表——这些操作如果放在应用层执行，需要多次数据库往返；如果封装为存储过程，一次调用即可完成所有操作。这就是存储过程和函数的核心价值。

## 存储过程

### 基本语法

```sql
CREATE [OR REPLACE] PROCEDURE procedure_name
[(parameter1 [IN | OUT | IN OUT] datatype,
  parameter2 [IN | OUT | IN OUT] datatype,
  ...)]
[AUTHID CURRENT_USER | DEFINER]
IS | AS
[declaration_section]
BEGIN
    execution_section
[EXCEPTION
    exception_handling_section]
END [procedure_name];
/
```

### 参数模式

- **IN**（默认）：参数从调用者传入过程
- **OUT**：参数从过程返回给调用者
- **IN OUT**：既传入又传出

```sql
-- IN 参数
CREATE OR REPLACE PROCEDURE increase_salary(
    p_emp_id IN employees.employee_id%TYPE,
    p_percent IN NUMBER
) AS
    v_new_salary employees.salary%TYPE;
BEGIN
    UPDATE employees
    SET salary = salary * (1 + p_percent / 100)
    WHERE employee_id = p_emp_id;
    
    SELECT salary INTO v_new_salary
    FROM employees WHERE employee_id = p_emp_id;
    
    DBMS_OUTPUT.PUT_LINE('新工资: ' || v_new_salary);
END increase_salary;
/

-- 调用
BEGIN
    increase_salary(100, 10);  -- 工号 100，工资增加 10%
END;
/
```

```sql
-- OUT 参数：返回多个值
CREATE OR REPLACE PROCEDURE get_employee_info(
    p_emp_id IN employees.employee_id%TYPE,
    p_name OUT VARCHAR2,
    p_salary OUT NUMBER,
    p_dept_name OUT VARCHAR2
) AS
BEGIN
    SELECT e.first_name || ' ' || e.last_name,
           e.salary, d.department_name
    INTO p_name, p_salary, p_dept_name
    FROM employees e
    LEFT JOIN departments d ON e.department_id = d.department_id
    WHERE e.employee_id = p_emp_id;
END get_employee_info;
/

-- 调用
DECLARE
    v_name VARCHAR2(100);
    v_salary NUMBER;
    v_dept VARCHAR2(100);
BEGIN
    get_employee_info(100, v_name, v_salary, v_dept);
    DBMS_OUTPUT.PUT_LINE(v_name || ', ' || v_salary || ', ' || v_dept);
END;
/
```

```sql
-- IN OUT 参数
CREATE OR REPLACE PROCEDURE swap_values(
    p_value1 IN OUT NUMBER,
    p_value2 IN OUT NUMBER
) AS
    v_temp NUMBER;
BEGIN
    v_temp := p_value1;
    p_value1 := p_value2;
    p_value2 := v_temp;
END swap_values;
/
```

### 无参数存储过程

```sql
CREATE OR REPLACE PROCEDURE print_hello AS
BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello, World!');
END print_hello;
/

BEGIN
    print_hello;
END;
/

-- SQL 中调用
CALL print_hello();
```

### 默认参数

```sql
CREATE OR REPLACE PROCEDURE create_user(
    p_user_name IN VARCHAR2,
    p_email IN VARCHAR2 DEFAULT 'noemail@example.com',
    p_status IN VARCHAR2 DEFAULT 'ACTIVE'
) AS
BEGIN
    DBMS_OUTPUT.PUT_LINE('用户: ' || p_user_name);
    DBMS_OUTPUT.PUT_LINE('邮箱: ' || p_email);
    DBMS_OUTPUT.PUT_LINE('状态: ' || p_status);
END create_user;
/

BEGIN
    create_user('张三');  -- 使用默认值
    create_user('李四', 'lisi@example.com');  -- 指定邮箱
    create_user('王五', 'wang@example.com', 'INACTIVE');  -- 全部指定
END;
/
```

## 函数

### 基本语法

```sql
CREATE [OR REPLACE] FUNCTION function_name
[(parameter1 [IN] datatype,
  parameter2 [IN] datatype,
  ...)]
RETURN return_datatype
[AUTHID CURRENT_USER | DEFINER]
IS | AS
[declaration_section]
BEGIN
    execution_section
[EXCEPTION
    exception_handling_section]
END [function_name];
/
```

### 无参数函数

```sql
CREATE OR REPLACE FUNCTION get_database_name
RETURN VARCHAR2 IS
    v_db_name VARCHAR2(50);
BEGIN
    SELECT name INTO v_db_name FROM v$database;
    RETURN v_db_name;
END get_database_name;
/

-- 调用
SELECT get_database_name FROM DUAL;
```

### 单参数函数

```sql
CREATE OR REPLACE FUNCTION get_employee_name(
    p_emp_id IN employees.employee_id%TYPE
) RETURN VARCHAR2 IS
    v_name VARCHAR2(100);
BEGIN
    SELECT first_name || ' ' || last_name INTO v_name
    FROM employees
    WHERE employee_id = p_emp_id;
    
    RETURN v_name;
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        RETURN '未知员工';
END get_employee_name;
/

-- 在 SQL 中调用
SELECT employee_id, get_employee_name(employee_id) AS name
FROM employees
WHERE department_id = 50;

-- 在 PL/SQL 中调用
DECLARE
    v_name VARCHAR2(100);
BEGIN
    v_name := get_employee_name(100);
    DBMS_OUTPUT.PUT_LINE(v_name);
END;
/
```

### 多参数函数

```sql
CREATE OR REPLACE FUNCTION calculate_bonus(
    p_salary IN NUMBER,
    p_years IN NUMBER,
    p_rating IN VARCHAR2
) RETURN NUMBER IS
    v_bonus NUMBER;
BEGIN
    v_bonus := p_salary * 0.1 * p_years;
    
    CASE p_rating
        WHEN 'A' THEN v_bonus := v_bonus * 1.5;
        WHEN 'B' THEN v_bonus := v_bonus * 1.2;
        WHEN 'C' THEN v_bonus := v_bonus * 1.0;
        ELSE v_bonus := v_bonus * 0.5;
    END CASE;
    
    RETURN NVL(v_bonus, 0);
END calculate_bonus;
/

SELECT first_name, salary, hire_date,
       TRUNC(MONTHS_BETWEEN(SYSDATE, hire_date) / 12) AS years,
       calculate_bonus(salary, TRUNC(MONTHS_BETWEEN(SYSDATE, hire_date) / 12), 'B') AS bonus
FROM employees;
```

### 函数限制

在 SQL 中调用的函数有以下限制：

- 只能使用 SQL 数据类型，不能使用 PL/SQL 特有类型（如 BOOLEAN）
- 不能包含 DML 语句（INSERT、UPDATE、DELETE）
- 不能调用其他有副作用的函数
- 必须遵守 purity level 规则

## 存储过程 vs 函数

| 特性 | 存储过程 | 函数 |
|---|---|---|
| 返回值 | 可有 0~N 个 OUT 参数 | 必须有 1 个 RETURN 值 |
| 调用方式 | 独立调用、EXECUTE、DBMS_JOB | SQL 表达式、PL/SQL 赋值 |
| DML 操作 | 支持 | SQL 中调用时不支持 |
| 返回结果集 | 通过 OUT 参数或 REF CURSOR | 可通过 RETURN TABLE() |
| 使用场景 | 复杂业务逻辑、数据维护 | 计算、转换、报表 |

## 管理存储过程和函数

```sql
-- 查看存储过程源代码
SELECT text FROM user_source WHERE name = 'PROCEDURE_NAME' ORDER BY line;

-- 查看函数源代码
SELECT text FROM user_source WHERE name = 'FUNCTION_NAME' ORDER BY line;

-- 重新编译
ALTER PROCEDURE procedure_name COMPILE;
ALTER FUNCTION function_name COMPILE;

-- 编译错误信息
SHOW ERRORS PROCEDURE procedure_name;
SHOW ERRORS FUNCTION function_name;

-- 删除
DROP PROCEDURE procedure_name;
DROP FUNCTION function_name;
```

## 常见面试问题

**问：存储过程和函数的区别是什么？**

函数必须有返回值（RETURN 语句），存储过程没有强制返回值要求；函数可以在 SQL 中直接调用（如 SELECT、WHERE 子句），存储过程需要独立调用（EXECUTE 或 CALL）；存储过程支持 DML 操作，SQL 中调用的函数不能包含 DML；存储过程通过 OUT 参数返回多个值，函数通过 RETURN 返回一个值。

**问：IN、OUT、IN OUT 参数的区别是什么？**

IN 参数将值传入存储过程/函数，是默认模式；OUT 参数将值传出给调用者，调用前初始值无效；IN OUT 参数既传入又传出，调用时传入值，过程可以修改并传出。

**问：如何返回结果集？**

可以使用 REF CURSOR 作为 OUT 参数；或者在包中定义游标并返回；Oracle 12c+ 支持函数返回 PL/SQL 表类型，结合 TABLE() 函数可在 SQL 中使用。

## 总结

存储过程和函数是 PL/SQL 编程的两大支柱。存储过程适合执行复杂业务逻辑和数据维护任务；函数适合计算、转换和可以在 SQL 中使用的场景。理解参数模式、掌握创建和管理方法，是 PL/SQL 开发的基础技能。
