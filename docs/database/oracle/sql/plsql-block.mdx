# PL/SQL 块结构

PL/SQL 是 Oracle 对 SQL 的过程化扩展，它不仅能执行 SQL 语句，还支持变量、流程控制、异常处理等编程特性。与纯 SQL 相比，PL/SQL 可以在数据库服务器上直接运行复杂业务逻辑，减少应用层与数据库之间的网络往返。

假设你需要将历史订单数据从 OLTP 表归档到 OLAP 表，同时更新统计信息、记录日志。如果用纯 SQL，你需要多次往返数据库；如果用 PL/SQL，所有操作可以在一次数据库调用中完成——这就是 PL/SQL 的价值。

## PL/SQL 块的基本结构

PL/SQL 程序由三部分组成：**声明部分（DECLARE）**、**执行部分（BEGIN...END）**、**异常处理部分（EXCEPTION）**。

```sql
[DECLARE
    -- 声明部分：变量、常量、游标、自定义类型
    v_var VARCHAR2(100);
    c_const CONSTANT NUMBER := 100;
]
BEGIN
    -- 执行部分：SQL 语句、流程控制、逻辑处理
    SELECT column_name INTO v_var FROM table_name WHERE id = 1;
    DBMS_OUTPUT.PUT_LINE('结果: ' || v_var);
[EXCEPTION
    -- 异常处理部分：捕获和处理异常
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('未找到数据');
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('其他错误');
]
END;
/
```

`/` 用于结束 PL/SQL 块并执行。

### 块的类型

**匿名块（Anonymous Block）**：没有名称的临时块，每次执行都要重新编译。

```sql
BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello PL/SQL');
END;
/
```

**命名块（Named Block）**：有名称的 PL/SQL 程序，包括存储过程、函数、触发器、包。

```sql
CREATE OR REPLACE PROCEDURE say_hello AS
BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello');
END say_hello;
/
```

## 变量与数据类型

### 基本数据类型

```sql
DECLARE
    v_name VARCHAR2(50);           -- 字符串
    v_age NUMBER(3);               -- 数值
    v_birthday DATE;               -- 日期
    v_salary employees.salary%TYPE;  -- 表列类型
    v_row employees%ROWTYPE;        -- 表行类型
    v_flag BOOLEAN := TRUE;        -- 布尔型
BEGIN
    v_name := '张三';
    v_age := 25;
    v_birthday := SYSDATE;
    DBMS_OUTPUT.PUT_LINE('姓名: ' || v_name);
END;
/
```

### %TYPE 和 %ROWTYPE

```sql
-- %TYPE：引用表列的数据类型
v_emp_name employees.first_name%TYPE;

-- %ROWTYPE：引用表中整行数据类型
DECLARE
    v_emp employees%ROWTYPE;
BEGIN
    SELECT * INTO v_emp FROM employees WHERE employee_id = 100;
    DBMS_OUTPUT.PUT_LINE(v_emp.first_name || ' ' || v_emp.last_name);
END;
/
```

使用 `%TYPE` 和 `%ROWTYPE` 的好处是：当表结构变更时，PL/SQL 代码自动适应，无需手动修改。

### 常量声明

```sql
DECLARE
    c_pi CONSTANT NUMBER := 3.14159;
    c_max_retries CONSTANT PLS_INTEGER := 3;
BEGIN
    DBMS_OUTPUT.PUT_LINE('圆周率: ' || c_pi);
END;
/
```

## 流程控制

### IF 条件判断

```sql
DECLARE
    v_score NUMBER := 85;
    v_grade VARCHAR2(10);
BEGIN
    IF v_score >= 90 THEN
        v_grade := 'A';
    ELSIF v_score >= 80 THEN
        v_grade := 'B';
    ELSIF v_score >= 70 THEN
        v_grade := 'C';
    ELSIF v_score >= 60 THEN
        v_grade := 'D';
    ELSE
        v_grade := 'F';
    END IF;
    
    DBMS_OUTPUT.PUT_LINE('等级: ' || v_grade);
END;
/
```

### CASE 语句

```sql
DECLARE
    v_dept_id NUMBER := 10;
    v_dept_name VARCHAR2(50);
BEGIN
    v_dept_name := CASE v_dept_id
        WHEN 10 THEN '行政部'
        WHEN 20 THEN '市场部'
        WHEN 30 THEN '销售部'
        ELSE '其他部门'
    END;
    
    DBMS_OUTPUT.PUT_LINE('部门: ' || v_dept_name);
END;
/
```

### 循环语句

**简单循环（LOOP...END LOOP）**：

```sql
DECLARE
    v_counter NUMBER := 1;
BEGIN
    LOOP
        DBMS_OUTPUT.PUT_LINE('计数器: ' || v_counter);
        v_counter := v_counter + 1;
        EXIT WHEN v_counter > 10;  -- 退出条件
    END LOOP;
END;
/
```

**WHILE 循环**：

```sql
DECLARE
    v_counter NUMBER := 1;
BEGIN
    WHILE v_counter <= 10 LOOP
        DBMS_OUTPUT.PUT_LINE('计数器: ' || v_counter);
        v_counter := v_counter + 1;
    END LOOP;
END;
/
```

**FOR 循环**：

```sql
BEGIN
    -- 数字范围循环
    FOR i IN 1..10 LOOP
        DBMS_OUTPUT.PUT_LINE('i = ' || i);
    END LOOP;
    
    -- 逆序循环
    FOR i IN REVERSE 1..10 LOOP
        DBMS_OUTPUT.PUT_LINE('i = ' || i);
    END LOOP;
END;
/
```

**CONTINUE 和 CONTINUE WHEN**（Oracle 11g+）：

```sql
BEGIN
    FOR i IN 1..10 LOOP
        CONTINUE WHEN MOD(i, 2) = 0;  -- 偶数跳过
        DBMS_OUTPUT.PUT_LINE('奇数: ' || i);
    END LOOP;
END;
/
```

## 异常处理

### 预定义异常

Oracle 预定义了大量异常，如 NO_DATA_FOUND、TOO_MANY_ROWS、DUP_VAL_ON_INDEX 等：

```sql
DECLARE
    v_emp_name VARCHAR2(50);
BEGIN
    SELECT first_name INTO v_emp_name 
    FROM employees 
    WHERE employee_id = 1000;  -- 不存在的员工
    
    DBMS_OUTPUT.PUT_LINE(v_emp_name);
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('员工不存在');
    WHEN TOO_MANY_ROWS THEN
        DBms_OUTPUT.PUT_LINE('返回多行，无法赋值');
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('其他错误: ' || SQLERRM);
END;
/
```

### SQLCODE 和 SQLERRM

```sql
EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('错误码: ' || SQLCODE);
        DBMS_OUTPUT.PUT_LINE('错误信息: ' || SQLERRM);
        RAISE;  -- 重新抛出异常
END;
/
```

### 用户定义异常

```sql
DECLARE
    e_invalid_salary EXCEPTION;  -- 声明异常
    PRAGMA EXCEPTION_INIT(e_invalid_salary, -20001);  -- 关联错误码
    v_salary NUMBER := -1000;
BEGIN
    IF v_salary < 0 THEN
        RAISE_APPLICATION_ERROR(-20001, '工资不能为负数');
    END IF;
EXCEPTION
    WHEN e_invalid_salary THEN
        DBMS_OUTPUT.PUT_LINE('自定义异常: ' || SQLERRM);
END;
/
```

### RAISE_APPLICATION_ERROR

```sql
-- 抛出用户自定义错误
BEGIN
    DELETE FROM employees WHERE employee_id = 100;
    IF SQL%NOTFOUND THEN
        RAISE_APPLICATION_ERROR(-20001, '员工不存在');
    END IF;
END;
/
```

### 异常传播

如果在内部块中未捕获异常，异常会传播到外部块：

```sql
BEGIN
    BEGIN
        SELECT * FROM nonexistent_table;
    EXCEPTION
        WHEN OTHERS THEN
            DBMS_OUTPUT.PUT_LINE('内部异常被捕获');
    END;
    
    -- 即使上面捕获了，这里仍会继续执行
    DBMS_OUTPUT.PUT_LINE('继续执行');
END;
/
```

## GOTO 语句

虽然 GOTO 不推荐使用，但 Oracle 确实支持：

```sql
BEGIN
    FOR i IN 1..10 LOOP
        IF i = 5 THEN
            GOTO end_loop;
        END IF;
    END LOOP;
    <<end_loop>>
    DBMS_OUTPUT.PUT_LINE('循环结束');
END;
/
```

:::warning 不推荐使用 GOTO
GOTO 会破坏程序的结构化特性，使代码难以理解和维护。绝大多数可以用 GOTO 实现的逻辑，都可以用 LOOP、EXIT 等结构化语句替代。
:::

## DBMS_OUTPUT 包

DBMS_OUTPUT 是 PL/SQL 的标准输出包，用于调试和显示信息：

```sql
BEGIN
    DBMS_OUTPUT.PUT_LINE('第一行');
    DBMS_OUTPUT.PUT_LINE('第二行');
    DBMS_OUTPUT.PUT('不换行输出');
    DBMS_OUTPUT.NEW_LINE;  -- 换行
END;
/

-- 需要先启用输出
SET SERVEROUTPUT ON
```

## 常见面试问题

**问：PL/SQL 块的三部分是什么？**

DECLARE（声明部分，可选）：定义变量、常量、游标、类型等。BEGIN...END（执行部分，必需）：包含实际执行的 SQL 和 PL/SQL 语句。EXCEPTION（异常处理部分，可选）：捕获和处理运行时异常。

**问：%TYPE 和 %ROWTYPE 的区别是什么？**

%TYPE 引用单个列的数据类型，适合定义存储单值的变量；%ROWTYPE 引用整行的数据类型，适合定义存储整条记录的变量。两者都能随表结构变化自动适应，提高代码可维护性。

**问：预定义异常和用户定义异常有什么区别？**

预定义异常是 Oracle 预先定义的，如 NO_DATA_FOUND、TOO_MANY_ROWS；用户定义异常需要手动声明，可以关联错误码并用 RAISE_APPLICATION_ERROR 抛出。

## 总结

PL/SQL 块结构是 PL/SQL 编程的基础。声明部分定义数据和对象，执行部分实现业务逻辑，异常处理部分捕获和处理错误。掌握这些基本概念后，才能进一步学习存储过程、函数、触发器等高级特性。
