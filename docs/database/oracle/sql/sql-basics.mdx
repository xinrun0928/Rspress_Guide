# Oracle SQL 基础

Oracle SQL 是 Oracle 数据库的核心操作语言。虽然 SQL 是标准化语言，但 Oracle 在 SQL 语法上有很多独有特性——分析函数、层级查询、MODEL 子句等，都是面试中展示技术深度的好素材。

假设你正在为一家电商公司开发报表系统，需要从订单表中计算每个用户的累计消费金额、排名、同比增长率——这些需求用标准 SQL 实现会非常复杂，而 Oracle 的分析函数可以优雅地解决。

## SELECT 语句：数据查询的核心

Oracle SELECT 语句的基本结构：

```sql
SELECT [DISTINCT] column_list | *
FROM table_list
[JOIN ...]
[WHERE condition]
[GROUP BY group_by_list]
[HAVING group_condition]
[ORDER BY order_list]
[LIMIT n [OFFSET m]];  -- Oracle 12c+
```

Oracle 12c 之前，分页需要借助 ROWNUM 或 ROW_NUMBER；Oracle 12c+ 支持了 `FETCH FIRST n ROWS ONLY` 语法。

### 别名与表达式

```sql
-- 列别名（AS 关键字可选）
SELECT employee_id AS "工号", 
       first_name || ' ' || last_name AS 姓名,
       salary * 1.1 AS 新工资
FROM employees e
WHERE department_id = 50;

-- AS 关键字的可选性
SELECT employee_id "工号" FROM employees;  -- 不带 AS
SELECT employee_id AS "工号" FROM employees;  -- 带 AS
```

### 连接操作

**等值连接（INNER JOIN）**：

```sql
-- 显式连接语法
SELECT e.employee_id, e.first_name, d.department_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.department_id;

-- 隐式连接语法（不推荐）
SELECT e.employee_id, e.first_name, d.department_name
FROM employees e, departments d
WHERE e.department_id = d.department_id;
```

**外连接**：

```sql
-- 左外连接
SELECT e.employee_id, e.first_name, d.department_name
FROM employees e
LEFT OUTER JOIN departments d ON e.department_id = d.department_id;

-- 右外连接
SELECT e.employee_id, e.first_name, d.department_name
FROM employees e
RIGHT OUTER JOIN departments d ON e.department_id = d.department_id;

-- 全外连接
SELECT e.employee_id, e.first_name, d.department_name
FROM employees e
FULL OUTER JOIN departments d ON e.department_id = d.department_id;
```

Oracle 9i 之前的专用语法使用 `(+)` 表示外连接，但在 Oracle 10g 之后逐渐被标准语法取代：

```sql
-- Oracle 旧式外连接语法（了解即可，不推荐使用）
SELECT e.employee_id, d.department_name
FROM employees e, departments d
WHERE e.department_id(+) = d.department_id;  -- 左表用(+)
```

### 集合操作

```sql
-- UNION：合并去重
SELECT department_id FROM employees
UNION
SELECT department_id FROM departments;

-- UNION ALL：合并不去重（效率更高）
SELECT department_id FROM employees
UNION ALL
SELECT department_id FROM departments
WHERE department_id IS NOT NULL;

-- INTERSECT：交集
SELECT department_id FROM employees
INTERSECT
SELECT department_id FROM departments;

-- MINUS：差集（第一个查询有而第二个没有的）
SELECT department_id FROM employees
MINUS
SELECT department_id FROM departments
WHERE department_name LIKE 'S%';
```

## DML 语句：数据操作

### INSERT

```sql
-- 单行插入
INSERT INTO employees (employee_id, first_name, last_name, email, hire_date, job_id, salary)
VALUES (1001, '张', '三', 'zhangsan@example.com', SYSDATE, 'IT_PROG', 10000);

-- 子查询插入
INSERT INTO employees_backup
SELECT * FROM employees WHERE hire_date > '2020-01-01';

-- 多行插入（Oracle 23c+ 支持 VALUES 直接多行）
INSERT ALL
    INTO employees VALUES (1002, '李四')
    INTO employees VALUES (1003, '王五')
SELECT 1 FROM DUAL;
```

### UPDATE

```sql
-- 更新单个列
UPDATE employees
SET salary = salary * 1.1
WHERE department_id = 50;

-- 更新多个列
UPDATE employees
SET (salary, commission_pct) = (
    SELECT salary * 1.2, 0.15 
    FROM employees 
    WHERE employee_id = 100
)
WHERE employee_id = 200;

-- 使用 CASE 表达式
UPDATE employees
SET salary = CASE 
    WHEN salary < 5000 THEN salary * 1.2
    WHEN salary < 10000 THEN salary * 1.1
    ELSE salary
END;
```

### DELETE 与 TRUNCATE

```sql
-- DELETE：带条件删除，可回滚
DELETE FROM employees
WHERE employee_id = 1001;

-- TRUNCATE：清空表，高效但不可回滚
TRUNCATE TABLE employees;

-- TRUNCATE 的特性
TRUNCATE TABLE employees REUSE STORAGE;  -- 保留存储空间
```

:::warning 重要区别
DELETE 是 DML 语句，可以回滚，触发触发器，生成 redo 日志（但使用 undo 回滚）；TRUNCATE 是 DDL 语句，无法回滚，不触发触发器，生成少量 redo 日志，执行速度远快于 DELETE。对于大表清空，TRUNCATE 是首选。
:::

## Oracle 特有 SQL 特性

### DECODE 函数

DECODE 是 Oracle 特有的条件表达式，等价于 CASE WHEN：

```sql
-- DECODE 基本语法
SELECT employee_id, first_name, 
       DECODE(department_id, 
              10, '行政部',
              20, '市场部',
              30, '销售部',
              '其他部门') AS dept_name
FROM employees;

-- 嵌套 DECODE
SELECT DECODE(SIGN(salary - 10000),
              -1, '低薪',
              0, '中薪',
              1, '高薪',
              '未知') AS salary_level
FROM employees;
```

### NVL、NVL2、COALESCE

```sql
-- NVL：如果第一个参数为 NULL，返回第二个参数
SELECT first_name, NVL(commission_pct, 0) AS comm
FROM employees;

-- NVL2：如果第一个参数不为 NULL，返回第二个参数；为 NULL 返回第三个参数
SELECT first_name, 
       NVL2(commission_pct, '有提成', '无提成') AS comm_status
FROM employees;

-- COALESCE：返回第一个非 NULL 值
SELECT first_name,
       COALESCE(commission_pct, manager_id, -1) AS fallback_value
FROM employees;
```

### 层级查询

Oracle 的 `CONNECT BY` 语法支持层级查询，非常适合处理组织架构、菜单树等层级数据：

```sql
-- 层级查询：员工-经理关系
SELECT employee_id, first_name, last_name, manager_id, LEVEL
FROM employees
START WITH manager_id IS NULL  -- 从 CEO 开始
CONNECT BY PRIOR employee_id = manager_id;

-- 层级伪列
-- LEVEL：当前层级（1 为根）
-- CONNECT_BY_ISLEAF：是否为叶子节点
-- SYS_CONNECT_BY_PATH：路径
SELECT SYS_CONNECT_BY_PATH(last_name, '/') AS path
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id;

-- 使用 CONNECT_BY_ROOT 获取根节点值
SELECT CONNECT_BY_ROOT last_name AS root_name, last_name, LEVEL
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id;
```

### 行列转换

```sql
-- PIVOT：行转列
SELECT * FROM (
    SELECT job_id, department_id, salary
    FROM employees
)
PIVOT (
    AVG(salary)  -- 聚合函数
    FOR department_id IN (10, 20, 30, 50, 80)  -- 列值
);

-- UNPIVOT：列转行
SELECT * FROM (
    SELECT * FROM pivot_table
)
UNPIVOT (
    salary FOR department_id IN (dept10, dept20, dept30)
);
```

## WHERE 子句的高级用法

```sql
-- BETWEEN ... AND ...
SELECT * FROM employees WHERE salary BETWEEN 5000 AND 10000;

-- IN (列表)
SELECT * FROM employees WHERE department_id IN (10, 20, 30);

-- LIKE 模糊匹配
SELECT * FROM employees WHERE last_name LIKE 'S%';  -- 以 S 开头
SELECT * FROM employees WHERE email LIKE '%@%.%';  -- 包含 @

-- IS NULL / IS NOT NULL
SELECT * FROM employees WHERE manager_id IS NULL;

-- EXISTS 子查询
SELECT * FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e 
    WHERE e.department_id = d.department_id 
    AND e.salary > 10000
);

-- 关联子查询
SELECT e.* FROM employees e
WHERE salary > (
    SELECT AVG(salary) FROM employees 
    WHERE department_id = e.department_id
);
```

## Oracle SQL 中的日期处理

```sql
-- 日期常量（必须使用 TO_DATE）
SELECT * FROM employees 
WHERE hire_date > TO_DATE('2020-01-01', 'YYYY-MM-DD');

-- 日期格式
SELECT TO_CHAR(SYSDATE, 'YYYY-MM-DD HH24:MI:SS') FROM DUAL;
SELECT TO_CHAR(SYSDATE, 'YYYY"年"MM"月"DD"日"') FROM DUAL;

-- 日期计算（加减数字表示天数）
SELECT SYSDATE + 1 AS tomorrow,
       SYSDATE - 1 AS yesterday,
       SYSDATE + 7/24 AS one_week_later_hours
FROM DUAL;

-- MONTHS_BETWEEN：月份差
SELECT MONTHS_BETWEEN(SYSDATE, hire_date) / 12 AS years_of_service
FROM employees;

-- ADD_MONTHS：加减月份
SELECT ADD_MONTHS(SYSDATE, 6) AS six_months_later FROM DUAL;

-- LAST_DAY：月份最后一天
SELECT LAST_DAY(SYSDATE) AS month_end FROM DUAL;
```

## 分页查询

Oracle 12c 之前，分页需要嵌套查询：

```sql
-- Oracle 11g 分页：使用 ROWNUM
SELECT * FROM (
    SELECT a.*, ROWNUM rn FROM (
        SELECT * FROM employees ORDER BY salary DESC
    ) a WHERE ROWNUM <= 20
) WHERE rn > 10;

-- Oracle 12c+：使用 OFFSET/FETCH
SELECT * FROM employees
ORDER BY salary DESC
OFFSET 10 ROWS FETCH NEXT 10 ROWS ONLY;

-- FETCH FIRST n ROWS ONLY（Oracle 18c+）
SELECT * FROM employees
ORDER BY salary DESC
FETCH FIRST 10 ROWS ONLY;
```

## 常见面试问题

**问：DELETE 和 TRUNCATE 的区别是什么？**

DELETE 是 DML 语句，可以回滚，会触发触发器，生成 undo 数据用于回滚；DELETE 可以有条件删除，释放部分空间。TRUNCATE 是 DDL 语句，无法回滚，不触发触发器，生成少量 redo 日志，执行速度远快于 DELETE；TRUNCATE 保留表结构，释放全部数据空间（REUSE STORAGE 参数可改变行为）。

**问：NVL 和 NVL2 的区别是什么？**

NVL(expr1, expr2) 在 expr1 为 NULL 时返回 expr2；NVL2(expr1, expr2, expr3) 在 expr1 不为 NULL 时返回 expr2，为 NULL 时返回 expr3。NVL2 提供了更灵活的两分支处理。

**问：CONNECT BY 的作用是什么？**

CONNECT BY 用于处理层级数据，通过 START WITH 指定根节点，CONNECT BY PRIOR 指定父子关系。配合 LEVEL、CONNECT_BY_ISLEAF、SYS_CONNECT_BY_PATH 等伪列，可以实现组织架构查询、菜单树遍历等场景。

## 总结

Oracle SQL 提供了丰富的功能，从基础的 DML 操作到特有的 DECODE、层级查询、行列转换。理解这些特性不仅能提高 SQL 编写效率，也是在面试中展示 Oracle 技术深度的重要素材。
