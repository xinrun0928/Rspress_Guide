# Oracle ASM 存储管理：ASM 实例与 Disk Group

存储管理，是 DBA 最头疼的工作之一。

磁盘满了、条带化不均匀、LUN 规划混乱……传统存储管理需要大量人工操作，而且容易出错。Oracle **Automatic Storage Management（ASM）** 把存储管理变成了自动化的工作，让 DBA 从繁琐的磁盘管理中解放出来。

## ASM 概述

### 为什么需要 ASM

传统存储管理的问题：

```
传统方案：
┌─────────────────────────────────────────────────────────────────┐
│  DBA 需要管理的内容：                                             │
│  - 每个数据文件的存放位置                                         │
│  - 条带化参数（Stripe Width, Stripe Depth）                     │
│  - 磁盘 I/O 负载均衡                                             │
│  - 空间分配与扩容                                                │
│  - 存储路径规划                                                  │
│                                                                 │
│  挑战：                                                          │
│  - 100+ 数据文件，分布在 10+ 磁盘                                 │
│  - 新增磁盘需要手动重新分布数据                                   │
│  - 磁盘故障恢复复杂                                              │
└─────────────────────────────────────────────────────────────────┘
```

ASM 的解决方案：

```
ASM 方案：
┌─────────────────────────────────────────────────────────────────┐
│  DBA 只需要管理 Disk Group：                                      │
│  - CREATE DISKGROUP ...                                          │
│  - ALTER DISKGROUP ...                                            │
│                                                                 │
│  ASM 自动处理：                                                   │
│  - 数据文件的分布（AuTO 条带化）                                  │
│  - I/O 负载均衡（动态再平衡）                                     │
│  - 空间分配                                                      │
│  - 故障恢复（镜像保护）                                          │
└─────────────────────────────────────────────────────────────────┘
```

### ASM 核心概念

| 概念 | 说明 |
|---|---|
| ASM Instance | 管理 ASM 磁盘的专门实例 |
| Disk Group | 磁盘组，多个磁盘组成的逻辑单元 |
| Disk | ASM 管理的物理磁盘或 LUN |
| Failure Group | 故障组，用于镜像的磁盘分组 |
| AU (Allocation Unit) | 分配单元，默认 1MB |
| Extent | 盘区，ASM 的空间分配单位 |

## ASM 实例架构

### ASM 实例 vs 数据库实例

```
┌─────────────────────────────────────────────────────────────────┐
│                    ASM 实例 vs 数据库实例                         │
│                                                                 │
│  ┌─────────────────────────┐    ┌─────────────────────────┐     │
│  │      ASM Instance      │    │    Database Instance    │     │
│  ├─────────────────────────┤    ├─────────────────────────┤     │
│  │ SGA: 少量（~256MB）    │    │ SGA: 根据负载配置       │     │
│  │ 后台进程: RBAL, ARBn   │    │ 后台进程: DBWn, LGWR... │     │
│  │ 不运行 SQL             │    │ 运行 SQL                │     │
│  │ 只管理磁盘元数据        │    │ 管理数据和事务           │     │
│  └─────────────────────────┘    └─────────────────────────┘     │
│                                                                 │
│              两个实例共享同一个存储                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### ASM 后台进程

| 进程 | 说明 |
|---|---|
| RBAL | Rebalancer，协调磁盘再平衡 |
| ARBn | Actual Rebalancer，执行数据迁移（多个）|
| GMON | Disk Group Monitor，监控磁盘组状态 |
| MARK | AU Marking，标记分配状态 |
| X000 | ASMB 连接进程 |

### ASM 实例配置

```bash
# ASM 实例初始化参数
# init+ASM1.ora

*.asm_diskgroups='DATA','FRA'           # 启动时自动挂载的磁盘组
*.asm_diskstring='/dev/oracleasm/disks/*'  # 发现的磁盘
*.asm_power_limit=1                      # 再平衡速度（1-11）
*.diagnostic_dest='/u01/app/oracle'
*.memory_target=1536M
```

## Disk Group

### Disk Group 类型

```
┌─────────────────────────────────────────────────────────────────┐
│                    Disk Group 类型                               │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │               Normal Redundancy (2-way)                 │   │
│  │                                                          │   │
│  │  AU1 ──▶ AU1' (镜像)                                     │   │
│  │  AU2 ──▶ AU2' (镜像)                                     │   │
│  │  AU3 ──▶ AU3' (镜像)                                     │   │
│  │                                                          │   │
│  │  需要至少 2 个 Failure Group                             │   │
│  │  可承受 1 个磁盘故障                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │               High Redundancy (3-way)                   │   │
│  │                                                          │   │
│  │  AU1 ──▶ AU1' ──▶ AU1''                                │   │
│  │  AU2 ──▶ AU2' ──▶ AU2''                               │   │
│  │                                                          │   │
│  │  需要至少 3 个 Failure Group                             │   │
│  │  可承受 2 个磁盘故障                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │               External Redundancy                       │   │
│  │                                                          │   │
│  │  AU1 ──▶ AU1                                            │   │
│  │  AU2 ──▶ AU2                                            │   │
│  │  无镜像，依赖外部存储 RAID                               │   │
│  │  最大可用空间                                            │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### 创建 Disk Group

```sql
-- 使用 SQL*Plus 或 ASMCMD

-- 1. 连接到 ASM 实例
$ asmcmd
ASMCMD> connect sys/oracle as sysasm

-- 2. 创建 Normal Redundancy Disk Group
CREATE DISKGROUP DATA NORMAL REDUNDANCY
    FAILGROUP CONTROLLER1 DISK
        '/dev/oracleasm/disks/ASM_DATA1' NAME DATA_DISK1,
        '/dev/oracleasm/disks/ASM_DATA2' NAME DATA_DISK2
    FAILGROUP CONTROLLER2 DISK
        '/dev/oracleasm/disks/ASM_DATA3' NAME DATA_DISK3,
        '/dev/oracleasm/disKS/ASM_DATA4' NAME DATA_DISK4;

-- 3. 创建 External Redundancy Disk Group
CREATE DISKGROUP FRA EXTERNAL REDUNDANCY
    DISK '/dev/oracleasm/disks/ASM_FRA1' NAME FRA_DISK1;

-- 4. 查看 Disk Group
SELECT name, group_number, type, sector_size, block_size,
       state, total_mb, free_mb, usable_file_mb
FROM v$asm_diskgroup;

-- 5. 查看 Disk
SELECT disk_number, group_number, header_status, mode_status,
       state, redundancy, failgroup, path, name
FROM v$asm_disk;
```

### Disk Group 操作

```sql
-- 添加磁盘到 Disk Group
ALTER DISKGROUP DATA ADD DISK
    '/dev/oracleasm/disks/ASM_DATA5' NAME DATA_DISK5;

-- 从 Disk Group 删除磁盘
ALTER DISKGROUP DATA DROP DISK DATA_DISK4;

-- 重新平衡（Rebalance）
-- 添加/删除磁盘后，ASM 会自动重新分布数据
ALTER DISKGROUP DATA REBALANCE POWER 8;  -- 手动调整再平衡速度

-- 挂载/卸载 Disk Group
ALTER DISKGROUP DATA MOUNT;
ALTER DISKGROUP DATA DISMOUNT;

-- 删除 Disk Group
DROP DISKGROUP DATA INCLUDING CONTENTS;
```

## AU 与 Extent

### AU（Allocation Unit）

AU 是 ASM 空间分配的基本单位：

```sql
-- 默认 AU 大小
-- 11g: 1MB
-- 12c+: 1MB/4MB（根据磁盘大小自动选择）

-- 查看 AU 大小
SELECT group_number, name, allocation_unit_size
FROM v$asm_diskgroup;

-- AU 大小影响：
-- 1. 大 AU 减少元数据开销
-- 2. 小 AU 提供更细粒度的分配
```

### Extent（盘区）

Extent 是逻辑上的分配单位，由多个 AU 组成：

```sql
-- Extent 分布策略（AUTO vs FLEX）
-- 11g: 固定大小 Extent
-- 12c: 可变大小 Extent (Variable Extents)

-- 查看文件 Extent 信息
SELECT group_number, file_number, extent_hwm, total_extents
FROM v$asm_file
WHERE file_number = 256;

-- Extent 分布示例
/*
 * File Extent Map:
 *   Extent 0: AU[0] Disk1
 *   Extent 1: AU[0] Disk2  (镜像)
 *   Extent 2: AU[1] Disk1
 *   Extent 3: AU[1] Disk2  (镜像)
 */
```

## ASM 文件管理

### ASM 文件命名

```sql
-- 完整 ASM 文件名格式
-- +DATA/RACDB/DATAFILE/SYSTEM.256.123456789

-- +DATA: Disk Group
-- RACDB: 数据库名
-- DATAFILE: 文件类型
-- SYSTEM: 文件名
-- 256: 文件编号
-- 123456789: 序号

-- 使用别名简化管理
ALTER DISKGROUP DATA ADD ALIAS 
    '+DATA/RACDB/PARAMFILE.ORA' 
    FOR '+DATA/RACDB/PARAMETERFILE/SPFILE.267.987654321';
```

### 创建 ASM 文件

```sql
-- 创建数据文件（使用 ASM 文件名）
CREATE TABLESPACE app_data
    DATAFILE '+DATA' SIZE 10G AUTOEXTEND ON NEXT 1G;

-- 创建临时文件
CREATE TEMPORARY TABLESPACE app_temp
    TEMPFILE '+FRA' SIZE 10G;

-- 创建控制文件
ALTER DATABASE BACKUP CONTROLFILE TO '+FRA/backup/control.bak';

-- 创建日志文件
ALTER DATABASE ADD LOGFILE
    ('+DATA', '+FRA') SIZE 1G;

-- 使用别名
CREATE TABLESPACE app_index
    DATAFILE '+DATA/app_index01.dbf' SIZE 5G;
```

### 管理 ASM 文件

```sql
-- 查看 ASM 文件
SELECT file_number, file_name, group_number, type, bytes, space
FROM v$asm_file
WHERE group_number = (SELECT group_number FROM v$asm_diskgroup WHERE name = 'DATA');

-- 删除 ASM 文件
ALTER DISKGROUP DATA DROP FILE '+DATA/RACDB/TEMPFILE/TEMP.268.987654321';

-- 重命名 ASM 文件
ALTER DISKGROUP DATA RENAME ALIAS
    '+DATA/users01.dbf' TO '+DATA/users01_old.dbf';

-- 移动 ASM 文件
ALTER DATABASE MOVE DATAFILE '+DATA/users01.dbf' TO '+FRA/users01.dbf';
```

## 镜像与冗余

### Failure Group

Failure Group 定义了共享同一故障点的磁盘集合：

```sql
-- 创建带 Failure Group 的 Disk Group
CREATE DISKGROUP DATA NORMAL REDUNDANCY
    FAILGROUP FG_CONTROLLER1 DISK
        '/dev/oracleasm/disks/ASM_DISK1' SIZE 500G,
        '/dev/oracleasm/disks/ASM_DISK2' SIZE 500G
    FAILGROUP FG_CONTROLLER2 DISK
        '/dev/oracleasm/disks/ASM_DISK3' SIZE 500G,
        '/dev/oracleasm/disKS/ASM_DISK4' SIZE 500G;

-- 说明：
-- FG_CONTROLLER1: 控制器 1 上的磁盘
-- FG_CONTROLLER2: 控制器 2 上的磁盘
-- 如果一个控制器故障，另一个控制器的磁盘仍有完整数据
```

### 镜像策略

```sql
-- 查看文件镜像状态
SELECT file_number, mirror_segs
FROM v$asm_file
WHERE file_number = 256;

-- mirror_segs 含义：
-- 0: 无镜像（External Redundancy）
-- 1: 2 路镜像（Normal Redundancy）
-- 2: 3 路镜像（High Redundancy）

-- 修改文件镜像
-- 需要使用 RMAN
RMAN> CONFIGURE DATAFILE '+DATA/RACDB/DATAFILE/USERS.257.dbinc' 
      MIRROR COPY;
```

## 再平衡（Rebalance）

### 何时触发再平衡

| 触发条件 | 说明 |
|---|---|
| 添加磁盘 | 新磁盘需要参与数据分布 |
| 删除磁盘 | 数据需要迁移到其他磁盘 |
| 磁盘故障 | 数据需要重新分布 |
| 手动触发 | 调整再平衡参数 |

### 再平衡过程

```
再平衡前：
┌─────────────────────────────────────────┐
│  Disk 1: ████████████████ 90%          │
│  Disk 2: ████████████████ 90%          │
│  Disk 3: ████░░░░░░░░░░░░░ 20%        │
│  Disk 4: ████░░░░░░░░░░░░░ 20%        │
└─────────────────────────────────────────┘

再平衡后：
┌─────────────────────────────────────────┐
│  Disk 1: ████████████░░░░░ 60%         │
│  Disk 2: ████████████░░░░░ 60%         │
│  Disk 3: ████████████░░░░░ 60%         │
│  Disk 4: ████████████░░░░░ 60%         │
└─────────────────────────────────────────┘
```

### 再平衡参数

```sql
-- ASM_POWER_LIMIT 参数（1-11）
-- 值越大，再平衡越快，但 I/O 开销越大

-- 临时调整再平衡速度
ALTER DISKGROUP DATA REBALANCE POWER 11;

-- 查看再平衡进度
SELECT group_number, operation, state, actual, sofar, est_work, est_rate
FROM v$asm_operation;

-- 取消再平衡
ALTER DISKGROUP DATA CANCEL REBALANCE;
```

## ASM 最佳实践

### 磁盘规划

```sql
-- 推荐配置
/*
 * 1. 每个控制器至少 2 个磁盘
 * 2. 使用 NORMAL REDUNDANCY
 * 3. 磁盘大小相同
 * 4. 使用 Oracle ASM Filter Driver (Oracle 18c+)
 */

-- 不推荐的配置
/*
 * - 单一控制器多磁盘（单点故障）
 * - 磁盘大小差异大（导致空间浪费）
 * - 混合不同性能的磁盘
 */
```

### Disk Group 划分

| Disk Group | 用途 | 冗余级别 | 典型大小 |
|---|---|---|---|
| DATA | 数据文件、表空间 | Normal | 较大 |
| FRA | 快速恢复区 | Normal/External | 中等 |
| REDO | 在线日志 | High/Normal | 较小 |
| OCR/VOTE | 集群配置 | High | 较小 |

```sql
-- 典型 Disk Group 配置
CREATE DISKGROUP DATA NORMAL REDUNDANCY
    FAILGROUP FG1 DISK '/dev/oracleasm/disks/ASM_DATA1', '/dev/oracleasm/disks/ASM_DATA2'
    FAILGROUP FG2 DISK '/dev/oracleasm/disks/ASM_DATA3', '/dev/oracleasm/disks/ASM_DATA4'
    ATTRIBUTE
        'compatible.asm' = '18.0',
        'compatible.rdbms' = '18.0',
        'sector_size' = '512',
        'au_size' = '4M';

CREATE DISKGROUP FRA EXTERNAL REDUNDANCY
    DISK '/dev/oracleasm/disks/ASM_FRA1'
    ATTRIBUTE
        'compatible.asm' = '18.0',
        'compatible.rdbms' = '18.0';
```

### 监控与维护

```sql
-- 查看 Disk Group 使用情况
SELECT name, total_mb, free_mb, 
       ROUND(free_mb / total_mb * 100, 2) AS free_pct,
       usable_file_mb
FROM v$asm_diskgroup;

-- 查看磁盘状态
SELECT disk_number, group_number, header_status, mode_status,
       state, failgroup, path, name, reads, writes
FROM v$asm_disk
ORDER BY group_number, disk_number;

-- 查看 I/O 统计
SELECT d.name, g.name AS disk_group,
       p Reads, p Write, p Read_Err, p Write_Err
FROM v$asm_disk p
JOIN v$asm_diskgroup g ON p.group_number = g.group_number
JOIN v$asm_disk d ON d.group_number = p.group_number AND d.disk_number = p.disk_number;

-- 查看再平衡历史
SELECT * FROM v$asm_disk_iostat
ORDER BY sample_time DESC;
```

## 总结

Oracle ASM 简化了存储管理的复杂度：

- **Disk Group**：逻辑存储单元，简化磁盘管理
- **Normal/High Redundancy**：自动镜像，无需担心磁盘故障
- **AU 和 Extent**：灵活的分配单元
- **再平衡**：数据自动均匀分布
- **兼容性**：支持 11g、12c、18c、19c、21c

使用 ASM，DBA 只需要关注 Disk Group 的容量和健康状态，而不需要关心具体的磁盘分配细节——**存储管理从此变得简单**。
