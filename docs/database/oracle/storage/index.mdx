# Oracle 存储结构

存储结构模块深入探讨 Oracle 数据库中数据的物理存储方式。从表类型的选择到索引的设计，从分区表的管理到表压缩技术，你将掌握如何根据业务特点选择最优的存储策略。

合理的存储结构设计直接影响数据库的读写性能、存储成本和可维护性。选择不当可能导致查询缓慢、存储浪费甚至数据倾斜等一系列问题。

## 模块内容

**表类型与索引基础**

- [表类型](/database/oracle/storage/table-type)：堆表、IOT、聚簇表、外部表的特点与选择
- [索引类型](/database/oracle/storage/index-type)：B-Tree、位图、函数、反向键索引的原理与应用
- [索引组织表与聚簇因子](/database/oracle/storage/iot)：IOT 表结构、聚簇因子的影响与优化

**分区与压缩**

- [分区表](/database/oracle/storage/partition)：Range、Hash、List、复合分区的策略与裁剪
- [表压缩](/database/oracle/storage/compression)：OLTP 压缩、归档压缩的原理与应用

:::tip 学习建议
存储结构的设计需要在开发早期考虑。一旦表上积累了大量数据，再进行结构变更的成本会非常高。建议结合业务场景（OLTP/OLAP）、数据特点（冷热分离、分布均匀度）和维护需求（分区裁剪、备份策略）综合决策。
:::
