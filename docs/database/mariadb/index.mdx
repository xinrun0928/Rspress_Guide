# MariaDB

2010 年，Oracle 收购 Sun，MySQL 正式落入 Oracle 之手。

同年，MySQL 的创始人 Michael "Monty" Widenius 创立了 MariaDB——MySQL 的「官方开源分支」。

十几年后，MariaDB 已经成为世界上最重要的开源数据库之一，Google、Red Hat、Wikipedia、腾讯云等巨头都在用。

## MariaDB vs MySQL：为什么选 MariaDB？

| 维度 | MySQL | MariaDB |
|------|-------|---------|
| 线程池 | 企业版收费 | ✅ 开源 |
| Galera Cluster | ❌ | ✅ 开源 |
| 窗口函数 | 8.0+ | ✅ 10.2+ |
| CTEs | 8.0+ | ✅ 10.2+ |
| 存储引擎 | InnoDB/MyISAM | +XtraDB/Aria/TokuDB |
| 审计插件 | 企业版 | ✅ 开源 |
| 开源协议 | GPL + 商业 | GPL |

**选 MariaDB 的理由**：
- 需要开源的线程池（MySQL 企业版收费）
- 需要 Galera 多主集群
- 担心 MySQL 许可证风险
- 需要 MariaDB 独有一些特性（系统版本表、虚拟列、Sequence）

**选 MySQL 的理由**：
- 需要 MySQL 8.0 的新特性（CTE、窗口函数 MySQL 也有了）
- 需要 MySQL 官方商业支持
- 团队已经熟悉 MySQL 生态

## 核心公式

```
MariaDB 存储引擎 = InnoDB（默认，事务）+ Aria（增强 MyISAM）+ XtraDB（增强 InnoDB）
Galera Cluster = 同步多主复制（所有节点同时写入，保证强一致）
线程池 = 多线程处理连接（解决高并发连接瓶颈）
GTID = 全局事务 ID（简化主从复制 failover）

复制模式：
  异步复制 = 主节点写入后立即返回，从节点异步同步
  半同步复制 = 主节点等待至少一个从节点确认
  Galera 同步 = 所有节点同时写入，强一致
```
