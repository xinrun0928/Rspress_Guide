# MariaDB

MariaDB 是 MySQL 的一个重要分支，由 MySQL 创始人 Michael "Monty" Widenius 在 MySQL 被 Oracle 收购后创建。作为一个社区驱动的开源数据库，MariaDB 致力于保持与 MySQL 的完全兼容性，同时提供更多的存储引擎、更新的功能和更强的性能。

---

## 为什么 MariaDB 值得关注

2010 年，Oracle 收购 Sun Microsystems 并获得 MySQL 的控制权后，MySQL 创始人 Michael Widenius 决定创建 MariaDB 作为 MySQL 的"社区分支"。这个决定背后有三重考量：

1. **兼容性优先**：MariaDB 从设计之初就将兼容性作为核心原则，确保现有 MySQL 应用能够零成本迁移
2. **开源承诺**：MariaDB 由 MariaDB Foundation 维护，保证代码永远开源，不会出现闭源"突变"
3. **持续创新**：在 MySQL 原地踏步的领域（如线程池、Galera 集群、JSON 函数），MariaDB 持续发力

---

## MariaDB 的核心优势

### 存储引擎的多样性

MariaDB 提供了比 MySQL 更为丰富的存储引擎选择。从默认的 InnoDB（实际是 XtraDB）到 Aria、TokuDB、MyRocks，每种引擎都有其独特的适用场景：

- **XtraDB/InnoDB**：事务型工作负载的首选
- **Aria**：MyISAM 的增强版，提供崩溃安全支持
- **TokuDB**：大规模数据插入场景下的利器
- **MyRocks**：极端写入密集型场景的最优选择

### Galera Cluster：真正的多主集群

MySQL 的主从复制是异步的，读写分离虽然能分担负载，但无法保证强一致性。MariaDB 的 Galera Cluster 提供了同步多主复制能力，多个节点可以同时接受读写请求，数据一致性由 Galera 协议保证。

### 线程池：连接管理的利器

MariaDB 企业版和社区版都提供了线程池插件，能够在高并发场景下显著减少线程创建和销毁的开销。对于需要处理大量短查询的应用，线程池是性能优化的关键。

### 查询优化器的演进

MariaDB 10.x 系列带来了全新的优化器架构。成本模型重新校准，连接优化、范围优化、排序优化等多个子模块都进行了改进。在复杂查询场景下，MariaDB 10.6+ 的优化器表现往往优于 MySQL 5.7 和 8.0。

---

## 适用场景

MariaDB 的设计目标明确，它最适合以下场景：

| 场景 | 推荐理由 |
|---|---|
| MySQL 迁移 | 兼容性极佳，零成本迁移 |
| 多主写入需求 | Galera Cluster 支持同步多主 |
| 高并发短查询 | 线程池插件降低连接开销 |
| 大数据量归档 | TokuDB/MyRocks 提供高压缩比 |
| PostgreSQL 替代 | MariaDB 10.6+ 兼容部分 PostgreSQL 语法 |
