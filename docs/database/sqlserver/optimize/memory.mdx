# 内存管理

「为什么 SQL Server 占用了这么多内存？」

SQL Server 的内存使用策略与 Oracle 类似——**尽可能缓存数据，减少磁盘 I/O**。理解内存管理，是调优 SQL Server 的关键。

## Buffer Pool

### Buffer Pool 的作用

Buffer Pool 是 SQL Server 最重要的内存组件：

- 缓存数据页（8KB）
- 支持读写操作
- 减少磁盘 I/O

```sql
-- 查看 Buffer Pool 使用
SELECT 
    COUNT(*) AS TotalPages,
    COUNT(*) * 8 / 1024.0 AS TotalBufferMB,
    SUM(CASE WHEN database_id = DB_ID() THEN 1 ELSE 0 END) AS CurrentDBPages,
    SUM(CASE WHEN is_modified = 1 THEN 1 ELSE 0 END) AS DirtyPages,
    SUM(CASE WHEN is_modified = 0 THEN 1 ELSE 0 END) AS CleanPages
FROM sys.dm_os_buffer_descriptors;
```

### 缓冲池页面状态

| 状态 | 说明 |
|---|---|
| FREE | 空闲页面，可分配 |
| CLEAN | 已提交但未修改 |
| DIRTY | 已修改，需刷盘 |
| STALE | 数据过时 |

### 缓冲池内存配置

```sql
-- 查看最大内存设置
sp_configure 'max server memory';
sp_configure 'min server memory';

-- 设置最大内存（单位：MB）
sp_configure 'max server memory', 32768;  -- 32GB
RECONFIGURE;

-- 设置最小内存（确保预留）
sp_configure 'min server memory', 8192;  -- 8GB
RECONFIGURE;
```

### 内存压力检测

```sql
-- 查看内存压力
SELECT 
    * 
FROM sys.dm_os_ring_buffers
WHERE ring_buffer_type = 'RING_BUFFER_RESOURCE_MONITOR';

-- 查看缓冲池页面分布
SELECT 
    DB_NAME(database_id) AS DatabaseName,
    COUNT(*) AS PageCount,
    COUNT(*) * 8 / 1024.0 AS BufferMB,
    SUM(CASE WHEN is_modified = 1 THEN 1 ELSE 0 END) AS DirtyPages
FROM sys.dm_os_buffer_descriptors
GROUP BY database_id
ORDER BY PageCount DESC;
```

## 数据页缓存

### 页面读取

```sql
-- 查看哪些对象占用最多缓存
SELECT 
    OBJECT_NAME(p.object_id) AS TableName,
    i.name AS IndexName,
    COUNT(*) AS PageCount,
    COUNT(*) * 8 / 1024.0 AS BufferMB,
    p.index_id
FROM sys.dm_os_buffer_descriptors bd
JOIN sys.allocation_units au ON bd.allocation_unit_id = au.allocation_unit_id
JOIN sys.partitions p ON au.container_id = p.partition_id
JOIN sys.indexes i ON p.object_id = i.object_id AND p.index_id = i.index_id
WHERE bd.database_id = DB_ID()
    AND OBJECTPROPERTY(p.object_id, 'IsUserTable') = 1
GROUP BY p.object_id, i.name, p.index_id
ORDER BY PageCount DESC;
```

## 执行计划缓存

### 计划缓存结构

```sql
-- 查看计划缓存
SELECT 
    cp.bucketid,
    cp.usecounts AS UseCount,
    cp.size_in_bytes / 1024.0 AS SizeKB,
    cp.objtype AS ObjectType,
    qs.total_worker_time / 1000.0 AS TotalCPUTimeMS,
    qs.total_elapsed_time / 1000.0 AS TotalDurationMS,
    qs.total_logical_reads,
    SUBSTRING(qt.text, 1, 200) AS QueryText
FROM sys.dm_exec_cached_plans cp
LEFT JOIN sys.dm_exec_query_stats qs ON cp.plan_handle = qs.plan_handle
CROSS APPLY sys.dm_exec_sql_text(cp.plan_handle) qt
WHERE cp.objtype IN ('Adhoc', 'Prepared')
ORDER BY cp.usecounts DESC;
```

### 计划缓存清理

```sql
-- 清除特定计划
DBCC FREEPROCCACHE (0x060001...);

-- 清除所有计划
DBCC FREEPROCCACHE;

-- 清除特定数据库的计划
DECLARE @dbid INT = DB_ID();
DELETE FROM sys.dm_exec_cached_plans
WHERE CAST(pa.value AS INT) = @dbid;
```

### 计划缓存问题

| 问题 | 说明 | 解决 |
|---|---|---|
| 计划太多 | 占用大量内存 | 限制计划数量 |
| 参数嗅探 | 错误计划被缓存 | Query Store |
| 零散计划 | Adhoc 查询产生 | 启用优化 |

```sql
-- 配置优化零散计划（SQL Server 2008+）
sp_configure 'optimize for ad hoc workloads', 1;
RECONFIGURE;
-- 第一次编译只缓存计划摘要，不缓存完整计划
```

## 内存组件

### 锁内存（Lock Memory）

```sql
-- 查看锁内存使用
SELECT 
    SUM(virtual_memory_reserved_kb) / 1024.0 AS ReservedMB,
    SUM(virtual_memory_committed_kb) / 1024.0 AS CommittedMB,
    SUM(lock_owner_blocked) AS BlockedCount
FROM sys.dm_os_memory_clerks
WHERE type = 'LOCK';

-- 查看当前锁数量
SELECT COUNT(*) AS LockCount FROM sys.dm_tran_locks;
```

### 连接内存

```sql
-- 查看连接内存
SELECT 
    cntr_value / 1024.0 AS ConnectionMemoryKB
FROM sys.dm_os_performance_counters
WHERE counter_name = 'Connection Memory (KB)';
```

### 优化器内存

```sql
-- 查看优化器内存
SELECT 
    SUM(virtual_memory_reserved_kb) / 1024.0 AS OptimizerReservedMB,
    SUM(virtual_memory_committed_kb) / 1024.0 AS OptimizerCommittedMB
FROM sys.dm_os_memory_clerks
WHERE type = 'MEMORYCLERK_SQLOPTIMIZER';
```

## 内存优化

### 监控内存使用

```sql
-- 综合内存监控
SELECT 
    type,
    name,
    pages_kb / 1024.0 AS PagesMB,
    virtual_memory_reserved_kb / 1024.0 AS VirtualReservedMB,
    virtual_memory_committed_kb / 1024.0 AS VirtualCommittedMB,
    shared_memory_reserved_kb / 1024.0 AS SharedReservedMB,
    shared_memory_committed_kb / 1024.0 AS SharedCommittedMB
FROM sys.dm_os_memory_clerks
ORDER BY pages_kb DESC;

-- 查看内存 clerk
SELECT 
    type,
    name,
    SUM(pages_kb) / 1024.0 AS TotalPagesMB
FROM sys.dm_os_memory_clerks
GROUP BY type, name
ORDER BY SUM(pages_kb) DESC;
```

### 内存配置建议

| 配置项 | 推荐值 | 说明 |
|---|---|---|
| `max server memory` | 物理内存 70-80% | 留内存给 OS |
| `min server memory` | 与 max 接近 | 预留最小内存 |
| `min memory per query` | 默认或增加 | 查询最小内存 |
| `index create memory` | 默认 | 索引创建内存 |

```sql
-- 推荐的 max server memory 计算
-- max server memory = 物理内存 - OS预留 - 其他服务预留
-- 例如：64GB 物理内存，SQL Server 用 50GB

sp_configure 'max server memory', 51200;  -- 50GB
RECONFIGURE;
```

### 内存压力诊断

```sql
-- 检测内存压力
SELECT 
    record.value('(./Record/ResourceMonitor/IndicatorsProcess)[1]', 'INT') AS ProcessIndicators,
    record.value('(./Record/ResourceMonitor/IndicatorsSystem)[1]', 'INT') AS SystemIndicators,
    record.value('(./Record/ResourceMonitor/TargetMemory)[1]', 'BIGINT') AS TargetMemoryKB,
    record.value('(./Record/ResourceMonitor/GrantedMemory)[1]', 'BIGINT') AS GrantedMemoryKB,
    record.value('(./Record/ResourceMonitor/ForegroundWorkers)[1]', 'INT') AS ForegroundWorkers,
    record.value('(./Record/ResourceMonitor/AvailableMemory)[1]', 'BIGINT') AS AvailableMemoryKB,
    record.timestamp
FROM sys.dm_os_ring_buffers
WHERE ring_buffer_type = 'RING_BUFFER_RESOURCE_MONITOR'
ORDER BY timestamp DESC;
```

## 内存优化表（In-Memory OLTP）

### 内存优化表简介

```sql
-- 创建内存优化文件组
ALTER DATABASE [YourDB] 
ADD FILEGROUP YourDB_Hekaton CONTAINS MEMORY_OPTIMIZED_DATA;

ALTER DATABASE [YourDB] 
ADD FILE (
    NAME = 'YourDB_Hekaton',
    FILENAME = 'D:\Hekaton\YourDB_Hekaton'
) TO FILEGROUP YourDB_Hekaton;
```

### 内存优化表

```sql
CREATE TABLE Orders_InMemory (
    OrderID INT IDENTITY(1,1) PRIMARY KEY NONCLUSTERED,
    OrderDate DATETIME2 NOT NULL,
    CustomerID INT NOT NULL,
    Amount DECIMAL(10,2),
    INDEX IX_Orders_Date NONCLUSTERED (OrderDate)
) WITH (MEMORY_OPTIMIZED = ON, DURABILITY = SCHEMA_AND_DATA);
```

### 限制和注意事项

| 限制 | 说明 |
|---|---|
| 数据类型 | 不支持 TEXT、IMAGE、XML 等大对象 |
| 大小 | 受内存限制 |
| 约束 | 有限的约束类型 |
| 功能 | 不支持外键、触发器（2016+ 除外） |

理解内存管理是 SQL Server 调优的核心。合理配置内存，能显著提升数据库性能。
