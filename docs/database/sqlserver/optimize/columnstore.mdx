# 列存储索引

「分析查询跑了几分钟，怎么提速？」

传统行存储索引在大数据分析场景下效率低下。SQL Server 2012+ 引入了**列存储索引（Columnstore Index）**，专为分析查询设计。

## 列存储索引概述

### 行存储 vs 列存储

```
┌─────────────────────────────────────────────────────────────────┐
│                    行存储 vs 列存储                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  行存储（传统 B+ 树）：                                          │
│  ┌────────┬────────┬────────┬────────┬────────┐                │
│  │ Col1   │ Col2   │ Col3   │ Col4   │ Col5   │ ← 一行       │
│  ├────────┼────────┼────────┼────────┼────────┤                │
│  │ 行数据 │ 行数据 │ 行数据 │ 行数据 │ 行数据 │                │
│  └────────┴────────┴────────┴────────┴────────┘                │
│  读取整行，即使只查 1 列                                        │
│                                                                 │
│  列存储：                                                        │
│  ┌────────┬────────┬────────┬────────┬────────┐                │
│  │ Col1   │ Col1   │ Col1   │ Col1   │ Col1   │ ← 同一列     │
│  │ Col1   │ Col1   │ Col1   │ Col1   │        │                │
│  ├────────┼────────┼────────┼────────┼────────┤                │
│  │ Col2   │ Col2   │ Col2   │ Col2   │        │                │
│  ├────────┼────────┼────────┼────────┼────────┤                │
│  │ Col3   │ Col3   │ Col3   │        │        │                │
│  └────────┴────────┴────────┴────────┴────────┘                │
│  只读取需要的列                                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 列存储索引优势

| 特性 | 行存储 | 列存储 |
|---|---|---|
| 数据压缩 | 1-3x | 10-100x |
| I/O 效率 | 读取整行 | 只读所需列 |
| 批量处理 | 行级 | 列批处理 |
| 内存效率 | 整行缓存 | 高密度压缩 |
| 适合场景 | OLTP | OLAP/分析 |

## 列存储索引类型

### 非聚集列存储索引（SQL Server 2012+）

在行存储表上创建列存储索引，适合混合场景：

```sql
-- 在行存储表上创建非聚集列存储索引
CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    OrderDate DATETIME2,
    CustomerID INT,
    ProductID INT,
    Amount DECIMAL(10,2),
    Status TINYINT
);

CREATE NONCLUSTERED COLUMNSTORE INDEX IX_Orders_ColumnStore
ON Orders(OrderDate, CustomerID, ProductID, Amount, Status);
```

### 聚集列存储索引（SQL Server 2014+）

整表使用列存储：

```sql
-- 创建聚集列存储索引
CREATE TABLE Orders_Analytics (
    OrderID BIGINT,
    OrderDate DATETIME2,
    CustomerID INT,
    ProductID INT,
    Amount DECIMAL(10,2),
    Status TINYINT
)
WITH (
    CLUSTERED COLUMNSTORE INDEX IX_Orders_CCS
);

-- 聚集列存储索引只能有一个（替代聚集索引）
```

### 混合模式（SQL Server 2016+）

同时使用行存储和列存储：

```sql
-- 聚集列存储 + 非聚集行存储索引
CREATE TABLE Orders_Hybrid (
    OrderID BIGINT NOT NULL,
    CustomerID INT NOT NULL,
    Amount DECIMAL(10,2),
    Status TINYINT
)
WITH (
    CLUSTERED COLUMNSTORE INDEX IX_Orders_CCS,
    INDEX IX_Customer NONCLUSTERED (CustomerID)
);
```

## 列存储压缩

### 压缩算法

列存储使用多种压缩算法：

| 算法 | 说明 |
|---|---|
| 字典压缩 | 对重复值创建字典 |
| 位图压缩 | 对低基数列 |
| 行程长度编码 | 对有序重复数据 |
| 增量压缩 | 最新数据的追加优化 |

### 查看压缩率

```sql
-- 查看列存储索引压缩统计
SELECT 
    OBJECT_NAME(object_id) AS TableName,
    index_id,
    index_type_desc,
    partition_number,
    column_store_compression_stats,
    column_store_dictionary_stats,
    row_group_trim_stats
FROM sys.dm_db_column_store_row_group_physical_stats
ORDER BY OBJECT_NAME(object_id), index_id;
```

## 行组（Row Group）

### 行组结构

列存储将数据分成行组，每组约 100 万行：

```
┌─────────────────────────────────────────────────────────────────┐
│                    列存储索引结构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Columnstore Index                                              │
│  │                                                             │
│  ├─ Row Group 1 (1M rows) ──▶ Column 1 ──▶ 压缩列数据          │
│  │                           ├─ Column 2 ──▶ 压缩列数据          │
│  │                           ├─ Column 3 ──▶ 压缩列数据          │
│  │                           └─ ...                              │
│  ├─ Row Group 2 (1M rows) ──▶ Column 1 ──▶ 压缩列数据          │
│  ├─ Row Group 3 (1M rows) ──▶ ...                              │
│  └─ Delta Store ◀─── 新插入数据（行格式）                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 行组状态

| 状态 | 说明 | 可查询 |
|---|---|---|
| COMPRESSED | 已压缩 | 是 |
| OPEN | 正在填充 | 是 |
| CLOSED | 等待压缩 | 是 |
| TOMBSTONE | 已删除（等待清理） | 否 |

```sql
-- 查看行组状态
SELECT 
    OBJECT_NAME(object_id) AS TableName,
    index_id,
    partition_number,
    row_group_id,
    state_desc,
    total_rows,
    deleted_rows,
    size_in_bytes / 1024.0 / 1024.0 AS SizeMB
FROM sys.dm_db_column_store_row_group_physical_stats
ORDER BY row_group_id DESC;
```

## 查询优化

### 批量模式执行

列存储使用批量模式（Batch Mode）处理数据：

```sql
-- 查看是否使用批量模式
SET STATISTICS XML ON;
GO
SELECT CustomerID, SUM(Amount) AS TotalAmount
FROM Orders
GROUP BY CustomerID;
GO
SET STATISTICS XML OFF;

-- 在 XML 计划中查找：<BatchMode>true</BatchMode>
```

### 列消除（Column Elimination）

只读取查询涉及的列：

```sql
-- 查询只涉及 3 列
SELECT CustomerID, OrderDate, Amount
FROM Orders
WHERE CustomerID = 100;

-- 列存储只读取这 3 列，忽略其他列
```

### 行组消除（Row Group Elimination）

跳过不包含目标数据的行组：

```sql
-- 查询特定日期范围
SELECT *
FROM Orders
WHERE OrderDate BETWEEN '2024-01-01' AND '2024-03-31';

-- 如果行组按 OrderDate 排序，只读取相关行组
```

## 创建列存储索引

### 基本语法

```sql
-- 非聚集列存储索引
CREATE NONCLUSTERED COLUMNSTORE INDEX IX_Orders_CS
ON Orders(OrderDate, CustomerID, ProductID, Amount, Status);

-- 聚集列存储索引
CREATE CLUSTERED COLUMNSTORE INDEX IX_Orders_CCS
ON Orders;

-- 包含所有列
CREATE CLUSTERED COLUMNSTORE INDEX IX_Orders_CCS
ON Orders
WITH (MAXDOP = 4);
```

### 排序列存储索引（SQL Server 2019+）

```sql
-- 排序列存储索引（按指定列排序存储）
CREATE CLUSTERED COLUMNSTORE INDEX IX_Orders_Sorted
ON Orders(OrderDate, CustomerID)
ORDER (OrderDate ASC, CustomerID ASC);
-- 减少行组消除，提升范围查询性能
```

### 列存储索引选项

```sql
CREATE NONCLUSTERED COLUMNSTORE INDEX IX_Orders_CS
ON Orders(OrderDate, CustomerID, Amount)
WITH (
    COMPRESSION_DELAY = 60 MINUTES,  -- 延迟压缩（SQL Server 2016+）
    MAXDOP = 4                         -- 并行度
);
```

## 维护

### 压缩行组

```sql
-- 查看需要压缩的行组
SELECT 
    OBJECT_NAME(object_id) AS TableName,
    row_group_id,
    state_desc,
    total_rows
FROM sys.dm_db_column_store_row_group_physical_stats
WHERE state_desc = 'CLOSED';

-- 手动压缩
ALTER INDEX IX_Orders_CCS ON Orders
REORGANIZE PARTITION = ALL
WITH (COMPRESS_ALL_ROW_GROUPS = ON);
```

### 重建索引

```sql
-- 重建列存储索引
ALTER INDEX IX_Orders_CCS ON Orders
REBUILD;

-- 重建所有分区
ALTER INDEX IX_Orders_CCS ON Orders
REBUILD PARTITION = ALL;
```

### 删除碎片

```sql
-- 查看碎片
SELECT 
    OBJECT_NAME(object_id) AS TableName,
    index_id,
    AVG_fragmentation_in_percent,
    fragment_count,
    avg_fragment_size_in_bytes / 1024.0 / 1024.0 AS AvgFragmentMB
FROM sys.dm_db_column_store_row_group_physical_stats
WHERE state_desc = 'COMPRESSED';

-- 重建解决碎片
ALTER INDEX IX_Orders_CCS ON Orders REBUILD;
```

## 限制

### 不支持的功能

| 功能 | 状态 |
|---|---|
| 主键/唯一约束 | 需额外非聚集索引 |
| 外键 | 不支持 |
| 触发器 | 不支持 |
| 镜象 | 不支持 |
| 全文索引 | 不支持 |

### 数据类型限制

| 支持 | 不支持 |
|---|---|
| 数值类型 | BINARY, VARBINARY |
| 字符串类型（有限长度） | VARCHAR/CHAR > 8000 |
| 日期/时间类型 | - |
| 唯一标识符 | - |

## 使用场景

### 适合场景

| 场景 | 原因 |
|---|---|
| 数据仓库 | 大量分析查询 |
| 大表汇总 | 高压缩减少 I/O |
| 列聚合 | 批量处理 |
| 历史数据 | 压缩率高 |

### 不适合场景

| 场景 | 原因 |
|---|---|
| 高频单行插入 | 批量加载更优 |
| 小表 | 开销不划算 |
| 窄范围查询 | 行存储可能更快 |

## 监控

```sql
-- 查看列存储使用统计
SELECT 
    OBJECT_NAME(object_id) AS TableName,
    index_id,
    partition_number,
    delta_rows_scheduled_for_merge AS DeltaRows,
    total_rows,
    size_in_bytes / 1024.0 / 1024.0 AS SizeMB
FROM sys.dm_db_column_store_row_group_physical_stats;

-- 查看批量模式使用
SELECT 
    SUBSTRING(qt.text, 1, 100) AS QueryText,
    rs.last_execution_time,
    rs.total_rows,
    rs.last_row_count
FROM sys.dm_exec_query_stats rs
CROSS APPLY sys.dm_exec_sql_text(rs.sql_handle) qt
WHERE qt.text LIKE '%columnstore%'
ORDER BY rs.last_execution_time DESC;
```
