# 备份策略

「数据丢了怎么办？」

备份是数据安全的最后一道防线。合理的备份策略能在灾难发生时快速恢复业务。

## 备份类型

### 完整备份

备份整个数据库：

```sql
-- 基础完整备份
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full.bak'
WITH COMPRESSION, CHECKSUM, NAME = 'Full Backup';

-- 完整备份到多个文件
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full_1.bak',
     DISK = 'E:\Backup\YourDB_Full_2.bak'
WITH COMPRESSION, CHECKSUM;

-- 镜像备份（副本）
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full.bak'
MIRROR TO DISK = 'E:\Backup\YourDB_Full_Mirror.bak'
WITH FORMAT, COMPRESSION;
```

### 差异备份

备份自上次完整备份以来的变化：

```sql
-- 差异备份
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Diff.bak'
WITH DIFFERENTIAL, COMPRESSION, CHECKSUM;
```

### 日志备份

仅备份事务日志（需要 FULL 或 BULK_LOGGED 恢复模式）：

```sql
-- 日志备份
BACKUP LOG [YourDB]
TO DISK = 'D:\Backup\YourDB_Log.trn'
WITH COMPRESSION, CHECKSUM;
```

### 备份类型对比

| 类型 | 备份内容 | 大小 | 备份时间 | 恢复时间 |
|---|---|---|---|---|
| 完整备份 | 全部数据 | 大 | 长 | 需 + 日志 |
| 差异备份 | 变化数据 | 中 | 中 | 需 + 日志 |
| 日志备份 | 日志记录 | 小 | 短 | 组合恢复 |

## 备份策略设计

### 策略一：完整 + 日志

适用于小型数据库：

```
每日完整备份：
┌─────┐─────┐─────┐─────┐─────┐─────┐─────┐
│ 完整 │ 完整 │ 完整 │ 完整 │ 完整 │ 完整 │ 完整 │
└─────┘─────┘─────┘─────┘─────┘─────┘─────┘
 周一     周二    周三    周四    周五    周六    周日

RPO：最长 24 小时
```

### 策略二：完整 + 差异 + 日志（推荐）

适用于大多数生产数据库：

```
┌─────┐─────┐─────┐─────┐─────┐─────┐─────┐
│ 完整 │     │     │     │ 差异 │     │     │
└─────┘─────┘─────┘─────┘─────┘─────┘─────┘
 周日备份  周一备份  周二备份  周三备份  周四备份  周五备份  周六备份

日志备份（每 15-30 分钟）：
F F F F F F F F F F F F F F F F F F F F F F F
│ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │
每 15 分钟
```

### 策略三：多地点备份

```
本地磁盘备份 ──────────▶ 磁带/云存储
      │
      │ 复制
      ▼
远程站点备份
```

## 备份选项

### 压缩备份

```sql
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full.bak'
WITH COMPRESSION;
```

### 校验和

```sql
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full.bak'
WITH CHECKSUM;
```

### 加密备份

```sql
-- 创建主密钥
CREATE MASTER KEY ENCRYPTION BY PASSWORD = 'YourPassword';

-- 创建证书
CREATE CERTIFICATE BackupCert
WITH SUBJECT = 'Backup Certificate';

-- 创建对称密钥
CREATE SYMMETRIC KEY BackupKey
BY CERTIFICATE = BackupCert;

-- 加密备份
OPEN SYMMETRIC KEY BackupKey
DECRYPTION BY CERTIFICATE BackupCert;

BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full_Encrypted.bak'
WITH COMPRESSION, ENCRYPTION (
    ALGORITHM = AES_256,
    SERVER CERTIFICATE = BackupCert
);

CLOSE SYMMETRIC KEY BackupKey;
```

### 剥离和继续

```sql
-- 剥离备份（多个文件）
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full_1.bak',
     DISK = 'D:\Backup\YourDB_Full_2.bak',
     DISK = 'D:\Backup\YourDB_Full_3.bak'
WITH COMPRESSION, CHECKSUM, STATS = 25;

-- 继续备份（从中断处继续）
BACKUP DATABASE [YourDB]
TO DISK = 'D:\Backup\YourDB_Full.bak'
WITH COMPRESSION, CONTINUE_IF_ERROR;
```

## 恢复模式

### 完整恢复模式

```sql
-- 查看当前恢复模式
SELECT 
    name,
    recovery_model_desc
FROM sys.databases
WHERE name = 'YourDB';

-- 设置为完整恢复模式
ALTER DATABASE [YourDB] SET RECOVERY FULL;

-- 完整恢复模式下的备份
BACKUP DATABASE [YourDB] TO DISK = 'D:\Backup\Full.bak';
BACKUP LOG [YourDB] TO DISK = 'D:\Backup\Log1.trn';
```

### 简单恢复模式

```sql
-- 设置为简单恢复模式
ALTER DATABASE [YourDB] SET RECOVERY SIMPLE;

-- 简单恢复模式下只能完整和差异备份
BACKUP DATABASE [YourDB] TO DISK = 'D:\Backup\Full.bak';
BACKUP DATABASE [YourDB] TO DISK = 'D:\Backup\Diff.bak'
WITH DIFFERENTIAL;
```

## 恢复操作

### 完整恢复

```sql
-- 恢复到最近状态
RESTORE DATABASE [YourDB]
FROM DISK = 'D:\Backup\Full.bak'
WITH NORECOVERY;

RESTORE DATABASE [YourDB]
FROM DISK = 'D:\Backup\Log1.trn',
     DISK = 'D:\Backup\Log2.trn'
WITH RECOVERY;
```

### 时间点恢复

```sql
-- 恢复到特定时间点
RESTORE DATABASE [YourDB]
FROM DISK = 'D:\Backup\Full.bak'
WITH NORECOVERY;

RESTORE LOG [YourDB]
FROM DISK = 'D:\Backup\Log1.trn',
     DISK = 'D:\Backup\Log2.trn'
WITH RECOVERY,
STOPAT = '2024-03-01 14:30:00';
```

### 恢复到新数据库

```sql
-- 恢复到新数据库
RESTORE DATABASE [YourDB_Restored]
FROM DISK = 'D:\Backup\Full.bak'
WITH NORECOVERY,
MOVE 'YourDB_Data' TO 'D:\Data\YourDB_Restored.mdf',
MOVE 'YourDB_Log' TO 'E:\Logs\YourDB_Restored.ldf';

RESTORE DATABASE [YourDB_Restored]
FROM DISK = 'D:\Backup\Log1.trn'
WITH RECOVERY;
```

### 页面恢复

```sql
-- 恢复损坏的页面
RESTORE DATABASE [YourDB]
PAGE = '1:823'
FROM DISK = 'D:\Backup\Full.bak'
WITH NORECOVERY;

RESTORE LOG [YourDB]
FROM DISK = 'D:\Backup\Log1.trn'
WITH NORECOVERY;

RESTORE LOG [YourDB]
FROM DISK = 'D:\Backup\Log2.trn'
WITH RECOVERY;
```

## 备份验证

### 验证备份

```sql
-- 验证备份完整性
RESTORE VERIFYONLY
FROM DISK = 'D:\Backup\Full.bak'
WITH CHECKSUM;

-- 验证并列出备份内容
RESTORE HEADERONLY
FROM DISK = 'D:\Backup\Full.bak';

-- 恢复数据库到测试位置（不覆盖原库）
RESTORE DATABASE [YourDB_Test]
FROM DISK = 'D:\Backup\Full.bak'
WITH MOVE 'YourDB_Data' TO 'D:\Data\YourDB_Test.mdf',
     MOVE 'YourDB_Log' TO 'E:\Logs\YourDB_Test.ldf';
```

### 自动化验证

```sql
-- 创建定期验证任务
EXEC sp_create_plan_guide 
    @name = 'BackupVerify',
    @stmt = 'RESTORE VERIFYONLY FROM DISK = ''D:\Backup\Full.bak''',
    @type = 'SQL_STMT';
```

## 备份维护

### 清理旧备份

```sql
-- 使用 SQL Server 维护计划
-- 或使用 PowerShell 脚本

# PowerShell 清理脚本
$BackupPath = "D:\Backup"
$DaysToKeep = 7

Get-ChildItem $BackupPath -Filter "*.bak" |
Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-$DaysToKeep) } |
Remove-Item -Force
```

### 监控备份

```sql
-- 查看最近的备份
SELECT 
    bs.database_name,
    bs.backup_start_date,
    bs.backup_finish_date,
    bs.backup_size / 1024.0 / 1024.0 AS BackupSizeMB,
    bs.type,
    bm.physical_device_name
FROM msdb.dbo.backupset bs
JOIN msdb.dbo.backupmediafamily bm ON bs.media_set_id = bm.media_set_id
WHERE bs.database_name = 'YourDB'
ORDER BY bs.backup_start_date DESC;

-- 查看备份历史
SELECT 
    database_name,
    backup_start_date,
    backup_finish_date,
    DATEDIFF(MINUTE, backup_start_date, backup_finish_date) AS DurationMinutes,
    type,
    backup_size / 1024.0 / 1024.0 AS BackupSizeMB
FROM msdb.dbo.backupset
WHERE database_name = 'YourDB'
ORDER BY backup_start_date DESC;
```

## 最佳实践

### 备份建议

| 建议 | 说明 |
|---|---|
| 使用压缩 | 减少备份时间和存储 |
| 使用校验和 | 检测备份损坏 |
| 测试恢复 | 定期验证备份可恢复 |
| 多个副本 | 本地 + 远程备份 |
| 加密敏感数据 | 防止备份泄露 |
| 监控备份失败 | 设置警报 |

### 备份脚本模板

```sql
-- 完整备份存储过程
CREATE PROCEDURE usp_BackupFull
    @BackupPath NVARCHAR(500)
AS
BEGIN
    DECLARE @DBName NVARCHAR(128) = DB_NAME();
    DECLARE @BackupFile NVARCHAR(500);
    DECLARE @DateStr NVARCHAR(20) = FORMAT(GETDATE(), 'yyyyMMdd_HHmmss');
    
    SET @BackupFile = @BackupPath + '\' + @DBName + '_Full_' + @DateStr + '.bak';
    
    BACKUP DATABASE @DBName
    TO DISK = @BackupFile
    WITH COMPRESSION, CHECKSUM, NAME = @DBName + ' Full Backup';
    
    PRINT 'Backup completed: ' + @BackupFile;
END;
GO

-- 日志备份存储过程
CREATE PROCEDURE usp_BackupLog
    @BackupPath NVARCHAR(500)
AS
BEGIN
    DECLARE @DBName NVARCHAR(128) = DB_NAME();
    DECLARE @BackupFile NVARCHAR(500);
    DECLARE @DateStr NVARCHAR(20) = FORMAT(GETDATE(), 'yyyyMMdd_HHmmss');
    
    SET @BackupFile = @BackupPath + '\' + @DBName + '_Log_' + @DateStr + '.trn';
    
    BACKUP LOG @DBName
    TO DISK = @BackupFile
    WITH COMPRESSION, CHECKSUM;
    
    PRINT 'Log backup completed: ' + @BackupFile;
END;
GO
```

合理的备份策略是数据安全的基石。根据业务 RPO/RTO 要求，选择合适的备份组合。
