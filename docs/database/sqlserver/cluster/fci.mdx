# 故障转移群集

SQL Server 故障转移群集（FCI）是基于 Windows Server Failover Cluster (WSFC) 的高可用解决方案，保护整个 SQL Server 实例。

## 故障转移群集概述

### 什么是 FCI

FCI 将多个 SQL Server 实例组合成一个虚拟服务器，当主节点故障时，整个实例自动转移到备用节点。

```
┌─────────────────────────────────────────────────────────────────┐
│                    SQL Server 故障转移群集                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │               Windows Server Failover Cluster            │   │
│  │                                                         │   │
│  │  ┌─────────────┐          ┌─────────────┐               │   │
│  │  │   节点 A     │          │   节点 B     │               │   │
│  │  │  (PRIMARY)  │◀────────▶│  (STANDBY)  │               │   │
│  │  └──────┬──────┘          └──────┬──────┘               │   │
│  │         │                         │                      │   │
│  │         │     共享存储 (SAN)      │                      │   │
│  │         │◀────────────────────────│                      │   │
│  │         ▼                         ▼                      │   │
│  │  ┌─────────────────────────────────────────────┐          │   │
│  │  │     共享磁盘阵列                            │          │   │
│  │  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ │          │   │

│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### FCI vs Always On AG

| 特性 | FCI | Always On AG |
|---|---|---|
| 保护单位 | 整个实例 | 单个数据库组 |
| 存储 | 共享存储（SAN） | 独立存储 |
| 故障转移粒度 | 实例级 | 数据库级 |
| 多副本 | 不支持 | 支持（最多 8） |
| 读写分离 | 不支持 | 支持 |
| 数据同步 | 自动 | 同步/异步可选 |

### 适用场景

| 场景 | 推荐方案 |
|---|---|
| 需要保护整个实例 | FCI |
| 数据库独立故障转移 | Always On AG |
| 读写分离需求 | Always On AG |
| 跨地域灾备 | Always On AG |
| 简单高可用 | FCI |

## 架构组件

### Windows Server Failover Cluster (WSFC)

WSFC 是 FCI 的基础：

- 管理节点成员资格
- 检测节点故障
- 触发故障转移
- 提供虚拟网络名称（VNN）

### 共享存储

FCI 需要共享存储：

| 存储类型 | 说明 |
|---|---|
| SAN | 企业级存储，推荐 |
| SMB 3.0 | 文件共享，适用于 Azure |
| Storage Spaces Direct | 软件定义存储 |

### Quorum

Quorum 决定集群存活：

| Quorum 模式 | 说明 | 适用 |
|---|---|---|
| 节点多数 | 多数节点在线存活 | 3+ 节点 |
| 节点和磁盘多数 | 多数节点+共享磁盘 | 2 节点 + 见证磁盘 |
| 节点和文件共享多数 | 多数节点+见证文件 | 2 节点 + 见证文件共享 |
| 仅节点 | 不推荐 | 测试 |

## 故障转移过程

### 检测故障

```
┌─────────────────────────────────────────────────────────────────┐
│                    故障检测与转移                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. WSFC 检测节点心跳失败                                       │
│  2. 确定需要故障转移                                            │
│  3. 验证 Quorum 多数                                            │
│  4. 在备用节点启动 SQL Server                                    │
│  5. 恢复数据库（恢复阶段）                                       │
│  6. 客户端重连到虚拟服务器                                        │
│                                                                 │
│  总时间：通常 30 秒到 2 分钟                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 故障恢复阶段

```sql
-- 查看数据库恢复状态
SELECT 
    DB_NAME(database_id) AS DatabaseName,
    state_desc,
    recovery_health_desc,
    synchronization_state_desc,
    lastredo_lsn,
    last_received_lsn
FROM sys.dm_hadr_database_replica_states
WHERE is_primary_replica = 1;

-- 查看恢复进度
SELECT 
    DB_NAME(database_id),
    percent_complete,
    wait_time,
    wait_type
FROM sys.dm_exec_requests
WHERE command = 'RESTORE LOG';
```

## 安装与配置

### 前置要求

| 要求 | 说明 |
|---|---|
| Windows Server | Enterprise 或 Datacenter |
| 域账户 | Active Directory 域成员 |
| 存储 | 共享磁盘，LUN |
| 网络 | 心跳网络 + 公网网络 |
| Quorum | 配置见证 |

### 安装步骤

```powershell
# 1. 安装故障转移群集功能
Install-WindowsFeature -Name Failover-Clustering -IncludeManagementTools

# 2. 验证群集准备
Test-Cluster -Node ServerA, ServerB

# 3. 创建群集
New-Cluster -Name "SQLCluster" -Node ServerA, ServerB `
    -StaticAddress 192.168.1.100 -NoStorage

# 4. 配置 Quorum
Set-ClusterQuorum -Cluster SQLCluster `
    -NodeAndFileShareMajority `
    -SharePath "\\FileShare\Witness"

# 5. 添加共享存储
Get-ClusterAvailableDisk
Add-ClusterDisk -Name "SQLData"
Add-ClusterDisk -Name "SQLLog"
```

### SQL Server 安装

```powershell
# 6. 安装 SQL Server（选择故障转移群集）
# 在安装向导中：
# - 选择 "New SQL Server failover cluster installation"
# - 配置网络名称和 IP
# - 选择共享磁盘作为数据目录
# - 配置实例 ID
```

## 监控与维护

### 群集状态监控

```powershell
# 查看群集状态
Get-Cluster -Name SQLCluster | Get-ClusterNode

# 查看资源状态
Get-ClusterResource -Cluster SQLCluster | Where-Object {$_.State -ne "Online"}

# 查看故障转移历史
Get-ClusterLog -Cluster SQLCluster -TimeSpan 24 | Select-Object -Last 100
```

### SQL Server 监控

```sql
-- 查看 FCI 状态
SELECT 
    @@SERVERNAME AS FailoverClusterInstance,
    SERVERPROPERTY('ComputerNamePhysicalNetBIOS') AS CurrentNode;

-- 查看故障转移事件
SELECT 
    creation_time,
    message,
    severity
FROM sys.event_log
WHERE type = 'FAILOVER'
ORDER BY creation_time DESC;
```

    wait_type,
