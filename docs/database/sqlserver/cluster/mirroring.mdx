# 数据库镜像

数据库镜像是 SQL Server 的数据保护技术，为 Always On 可用性组的前身。

## 数据库镜像概述

### 什么是数据库镜像

数据库镜像将主数据库的事务日志实时发送到镜像服务器，在镜像上重做，实现数据保护。

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据库镜像架构                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐                ┌─────────────┐                │
│  │   主服务器   │                │   镜像服务器  │                │
│  │ (Principal) │◀──────────────▶│  (Mirror)   │                │
│  └──────┬──────┘                └──────┬──────┘                │
│         │                                │                       │
│         │  事务日志                       │                       │
│         │◀──────────────────────────────│                       │
│         │                                │                       │
│         ▼                                ▼                       │
│  ┌─────────────┐                ┌─────────────┐                │
│  │  数据库 A   │  镜像 ───────▶ │  数据库 A   │                │
│  │  (在线)     │                │  (恢复中)    │                │
│  └─────────────┘                └─────────────┘                │
│                                                                 │
│  ┌─────────────┐                                                │
│  │  见证服务器  │  可选，用于自动故障转移                         │
│  │  (Witness)  │                                                │
│  └─────────────┘                                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 镜像模式

| 模式 | 说明 | 数据丢失 | 故障转移 |
|---|---|---|---|
| 高安全模式（同步） | 事务等待镜像确认 | 零数据丢失 | 手动或自动（有见证） |
| 高性能模式（异步） | 事务不等待镜像 | 可能少量丢失 | 仅手动 |

### 与 Always On AG 的关系

| 特性 | 数据库镜像 | Always On AG |
|---|---|---|
| SQL Server 版本 | 2005+ | 2012+ |
| 副本数量 | 1 | 最多 8 |
| 多个数据库 | 不支持 | 支持 |
| 读写分离 | 不支持 | 支持 |
| 自动故障转移 | 需要见证服务器 | 内置 |
| 推荐场景 | 简单数据保护 | 企业级高可用 |

## 配置数据库镜像

### 前置条件

| 条件 | 说明 |
|---|---|
| 恢复模式 | 必须 FULL |
| 兼容版本 | Enterprise 对 Enterprise，Standard 对 Standard |
| 域环境 | 推荐 AD 域，Workgroup 可用但有限制 |
| 端点 | TCP 端口，服务器间可通信 |

### 配置步骤

```sql
-- 1. 确保恢复模式为 FULL
ALTER DATABASE [YourDB] SET RECOVERY FULL;

-- 2. 备份主数据库
BACKUP DATABASE [YourDB]
TO DISK = 'E:\Backup\YourDB_Full.bak'
WITH COMPRESSION;

-- 3. 备份事务日志
BACKUP LOG [YourDB]
TO DISK = 'E:\Backup\YourDB_Log.trn'
WITH COMPRESSION;

-- 4. 复制备份到镜像服务器

-- 5. 在镜像服务器恢复数据库（NORECOVERY）
RESTORE DATABASE [YourDB]
FROM DISK = 'E:\Backup\YourDB_Full.bak'
WITH NORECOVERY;

RESTORE LOG [YourDB]
FROM DISK = 'E:\Backup\YourDB_Log.trn'
WITH NORECOVERY;
```

### 建立镜像会话

```sql
-- 6. 在主服务器创建端点（如果不存在）
CREATE ENDPOINT DatabaseMirroring
    STATE = STARTED
    AS TCP (LISTENER_PORT = 5022)
    FOR DATABASE_MIRRORING (
        ROLE = ALL,
        AUTHENTICATION = WINDOWS NEGOTIATE,
        ENCRYPTION = REQUIRED
    );

-- 7. 在主服务器配置镜像伙伴
ALTER DATABASE [YourDB]
SET PARTNER = 'TCP://MirrorServer:5022';

-- 8. 在镜像服务器配置镜像伙伴
ALTER DATABASE [YourDB]
SET PARTNER = 'TCP://PrincipalServer:5022';

-- 9. 可选：添加见证服务器实现自动故障转移
ALTER DATABASE [YourDB]
SET WITNESS = 'TCP://WitnessServer:5022';
```

## 镜像操作

### 手动故障转移

```sql
-- 在主服务器执行故障转移（仅高安全模式）
USE master;
ALTER DATABASE [YourDB] SET PARTNER FAILOVER;
-- 主服务器变为镜像，镜像变为主
```

### 自动故障转移

需要高安全模式 + 见证服务器：

```
┌─────────────────────────────────────────────────────────────────┐
│                    自动故障转移流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 主服务器不可达                                               │
│  2. 见证服务器检测连接丢失                                       │
│  3. 见证服务器通知镜像服务器                                      │
│  4. 镜像服务器自动提升为主服务器                                  │
│  5. 应用程序重连到新的主服务器                                    │
│                                                                 │
│  通常 10-30 秒内完成                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 暂停和恢复镜像

```sql
-- 暂停镜像（在主服务器）
ALTER DATABASE [YourDB] SET PARTNER SUSPEND;

-- 恢复镜像（在主服务器）
ALTER DATABASE [YourDB] SET PARTNER RESUME;
```

### 删除镜像

```sql
-- 在主服务器移除镜像
ALTER DATABASE [YourDB] SET PARTNER OFF;

-- 在镜像服务器也执行（确保清理）
ALTER DATABASE [YourDB] SET PARTNER OFF;
```

## 监控镜像

### DMV 监控

```sql
-- 查看镜像状态
SELECT 
    DB_NAME(database_id) AS DatabaseName,
    mirroring_role_desc,
    mirroring_state_desc,
    mirroring_safety_level_desc,
    mirroring_partner_name,
    mirroring_witness_name,
    mirroring_witness_state_desc
FROM sys.database_mirroring
WHERE mirroring_guid IS NOT NULL;

-- 查看镜像性能
SELECT 
    DB_NAME(database_id) AS DatabaseName,
    mirroring_connection_state_desc,
    mirroring_connection_timeout,
    mirroring_end_of_log_lsn,
    mirroring_replay_lag,
    mirroring_redo_queue_lsn,
    mirroring_redo_queue,
    mirroring_redone_buffer_size
FROM sys.dm_database_mirroring_connections;
```

## 客户端重连

### 连接字符串

```sql
-- Failover Partner 参数用于自动重连
"Server=PrincipalServer; Failover Partner=MirrorServer;
  Database=YourDB; Integrated Security=True;"

-- Always On 可用性组使用不同连接方式
"Server=ListenerName; Database=YourDB; ApplicationIntent=ReadOnly;
  MultiSubnetFailover=Yes"
```

### 故障转移后的行为

```
┌─────────────────────────────────────────────────────────────────┐
│                    客户端重连行为                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  应用程序 ──▶ 主服务器（正常）                                    │
│                                                                 │
│  故障转移后：                                                     │
│  应用程序 ──▶ 原主服务器（连接失败）                             │
│  自动重试 ──▶ 见证服务器查询新主                                 │
│  成功重连 ──▶ 新主服务器                                        │
│                                                                 │
│  重试次数和超时由客户端设置                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 迁移到 Always On AG

### 为什么迁移

| 原因 | 说明 |
|---|---|
| 限制太多 | 单个数据库，不支持读写分离 |
| 功能有限 | 不支持备份到辅助副本 |
| 微软不再增强 | 已转向 Always On AG |
| Enterprise 专属 | Standard 不支持 |

### 迁移步骤

```sql
-- 1. 准备 Always On AG（创建 AG、添加副本）

-- 2. 将镜像数据库加入 AG
-- 在主副本：
ALTER AVAILABILITY GROUP [AG1] ADD DATABASE [YourDB];

-- 3. 等待数据同步完成
-- 监控 sys.dm_hadr_database_replica_states

-- 4. 验证 AG 状态
SELECT 
    ag.name AS AGName,
    ar.replica_server_name,
    drs.database_name,
    drs.synchronization_state_desc
FROM sys.dm_hadr_database_replica_states drs
JOIN sys.availability_replicas ar ON drs.replica_id = ar.replica_id
JOIN sys.availability_groups ag ON ar.group_id = ag.group_id;

-- 5. 断开镜像
ALTER DATABASE [YourDB] SET PARTNER OFF;

-- 6. 更新客户端连接字符串
```

## 常见问题

### 性能问题

```sql
-- 检查日志发送队列

    mirroring_redo_queue_lsn,
    mirroring_redo_queue
FROM sys.database_mirroring
WHERE mirroring_guid IS NOT NULL;
```
