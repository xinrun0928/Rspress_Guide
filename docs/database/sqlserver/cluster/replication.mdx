# 复制

SQL Server 复制是一组技术，用于在数据库之间分发和同步数据和数据库对象。

## 复制概述

### 复制的用途

| 用途 | 说明 |
|---|---|
| 负载均衡 | 将读操作分布到多个服务器 |
| 数据移动 | 跨地域数据分发 |
| 报表 | 为报表创建独立数据副本 |
| 灾难恢复 | 数据实时复制到灾备中心 |
| 集成 | 与其他系统数据同步 |

### 复制类型

```
┌─────────────────────────────────────────────────────────────────┐
│                    SQL Server 复制类型                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  复制                                                            │
│  │                                                             │
│  ├── 快照复制 (Snapshot Replication)                            │
│  │     └── 定期全量同步                                        │
│  │                                                             │
│  ├── 事务复制 (Transactional Replication)                      │
│  │     └── 实时增量同步                                        │
│  │                                                             │
│  └── 合并复制 (Merge Replication)                              │
│        └── 双向同步，支持离线                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 快照复制

### 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    快照复制流程                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  发布服务器                                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 1. 生成快照                                             │   │
│  │    - 锁定表（可选）                                     │   │
│  │    - 导出数据到快照文件夹                               │   │
│  │    - 释放锁                                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│         │                                                        │
│         │ 快照文件                                              │
│         ▼                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 2. 分发服务器                                             │   │
│  │    存储快照文件                                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│         │                                                        │
│         │ 复制快照                                              │
│         ▼                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 3. 订阅服务器                                             │   │
│  │    应用快照到订阅数据库                                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  定时重复：每日/每周/按需                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 配置快照复制

```sql
-- 1. 在发布服务器配置分发服务器
EXEC sp_adddistributor 
    @distributor = 'DistributorServer',
    @password = 'YourPassword';

-- 2. 创建分发数据库
EXEC sp_adddistributiondb 
    @database = 'distribution',
    @security_mode = 1;

-- 3. 创建发布
EXEC sp_addpublication 
    @publication = 'OrdersSnapshot',
    @status = 'active',
    @sync_method = 'native';

-- 4. 添加文章
EXEC sp_addarticle 
    @publication = 'OrdersSnapshot',
    @article = 'Orders',
    @source_object = 'Orders',
    @destination_table = 'Orders';

-- 5. 创建订阅
EXEC sp_addsubscription 
    @publication = 'OrdersSnapshot',
    @subscriber = 'SubscriberServer',
    @destination_db = 'OrdersReplica',
    @subscription_type = 'Push';

-- 6. 生成并分发快照
EXEC sp_startpublication_snapshot @publication = 'OrdersSnapshot';
```

## 事务复制

### 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    事务复制流程                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  发布服务器                                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Log Reader 代理                                          │   │
│  │  - 读取事务日志                                          │   │
│  │  - 筛选已提交事务                                       │   │
│  │  - 写入分发数据库                                        │   │
│  └─────────────────────────────────────────────────────────┘   │
│         │                                                        │
│         │ 事务命令                                              │
│         ▼                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 分发服务器 (Distribution Database)                      │   │
│  │  - 存储待复制命令                                        │   │
│  │  - 保留历史                                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│         │                                                        │
│         │ 分发命令                                              │
│         ▼                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 订阅服务器                                               │   │
│  │  Distribution Agent                                      │   │
│  │  - 读取分发数据库                                       │   │
│  │  - 应用命令到订阅数据库                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 配置事务复制

```sql
-- 1. 配置分发（与快照复制相同）

-- 2. 创建发布（使用默认同步方法）
EXEC sp_addpublication 
    @publication = 'OrdersTrans',
    @allow_push = N'true',
    @allow_pull = N'true',
    @type = N'logical';

-- 3. 添加文章
EXEC sp_addarticle 
    @publication = 'OrdersTrans',
    @article = 'Orders',
    @source_object = 'Orders',
    @type = N'logbased',
    @ins_cmd = N'CALL sp_MSins_Orders',
    @del_cmd = N'CALL sp_MSdel_Orders',
    @upd_cmd = N'SCALL sp_MSupd_Orders';

-- 4. 添加订阅
EXEC sp_addsubscription 
    @publication = 'OrdersTrans',
    @subscriber = 'SubscriberServer',
    @destination_db = 'OrdersReplica',
    @subscription_type = N'Push',
    @sync_type = N'automatic';
```

### 事务复制选项

```sql
-- 查看复制拓扑
SELECT 
    publisher,
    publisher_db,
    publication,
    subscriber,
    subscriber_db
FROM MSsubscription_properties;

-- 修改复制属性
EXEC sp_change_subscription_properties
    @publication = 'OrdersTrans',
    @subscriber = 'SubscriberServer',
    @destination_db = 'OrdersReplica',
    @property = 'subscription_type',
    @value = 'Pull';
```

## 合并复制

### 工作原理

合并复制支持双向同步和离线场景：

```
┌─────────────────────────────────────────────────────────────────┐
│                    合并复制流程                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  发布服务器 ──────────────── 订阅服务器                         │
│       │                            │                            │
│       │  初始化同步                │  初始化同步                  │
│       │◀──────────────────────────│▶                          │
│       │                            │                            │
│       │  各自独立修改              │  各自独立修改                │
│       │  (离线工作)                │  (离线工作)                  │
│       │                            │                            │
│       │  同步时冲突检测            │  同步时冲突检测              │
│       │◀──────────────────────────│▶                          │
│       │                            │                            │
│       │  解决冲突                  │  解决冲突                  │
│       │◀──────────────────────────│▶                          │
│       │                            │                            │
└─────────────────────────────────────────────────────────────────┘
```

### 配置合并复制

```sql
-- 1. 创建发布
EXEC sp_addmergepublication 
    @publication = 'OrdersMerge',
    @status = 'active',
    @sync_mode = 'native';

-- 2. 添加文章（需要唯一标识符）
EXEC sp_addarticle 
    @publication = 'OrdersMerge',
    @article = 'Orders',
    @source_object = 'Orders',
    @type = 'table',
    @pub_identity_range = 10000,
    @identity_range = 1000,
    @threshold = 80;

-- 3. 添加订阅
EXEC sp_addmergepullsubscription 
    @publication = 'OrdersMerge',
    @subscriber = 'SubscriberServer',
    @destination_db = 'OrdersReplica';

EXEC sp_addmergepushsubscription 
    @publication = 'OrdersMerge',
    @subscriber = 'SubscriberServer',
    @destination_db = 'OrdersReplica';

-- 4. 初始化订阅
EXEC sp_mergesubscription_subscriptiondb_level_init 
    @publication = 'OrdersMerge',
    @subscriber = 'SubscriberServer',
    @subscription_type = 'Pull';
```

## 复制监控

### 常用监控查询

```sql
-- 查看复制状态
SELECT 
    publisher,
    publisher_db,
    publication,
    publication_type,
    status,
    publication_frequency
FROM MSpublications;

-- 查看分发代理状态
SELECT 
    name,
    status,
    publisher,
    subscriber,
    last_dts,
    comments
FROM MSdistribution_agents;

-- 查看复制延迟
SELECT 
    publisher,
    publisher_db,
    publication,
    subscriber,
    subscriber_db,
    current_delivery_rate,
    current_delivery_latency
FROM MSdistribution_status;

-- 查看错误
SELECT 
    time,
    error_code,
    error_text,
    session_id
FROM MSrepl_errors
ORDER BY time DESC;
```

## 常见问题

### 延迟问题

```sql
-- 检查分发队列
SELECT 
    publisher_database_id,
    publisher_id,
    article_id,
    subscriber_id,
    batch_id,
    commands_in_batch,
    spid
FROM MSrepl_commands WITH (NOLOCK)
WHERE xact_seqno IS NULL;

-- 检查代理历史
EXEC sp_expired_subscription_cleanup;
```

### 冲突解决

```sql
-- 查看合并复制冲突
EXEC sp_helpmergeconflictrows 
    @publication = 'OrdersMerge';

-- 解决冲突
EXEC sp_deleteconflictrow
    @publication = 'OrdersMerge',
    @conflict_table = 'MSmerge_conflict_OrdersMerge_Orders';
```

复制是数据分发和多服务器场景的强大工具，选择合适的复制类型是关键。
