# Always On 可用性组架构

「数据库的高可用怎么做？」

对于 SQL Server，答案是 **Always On 可用性组（Availability Groups）**。这是微软为企业级高可用场景打造的核心技术。

## Always On 概述

### 什么是 Always On

Always On 是 SQL Server 的高可用和灾难恢复解决方案，核心概念：

- **可用性组（Availability Group）**：一组用户数据库作为一个整体进行故障转移
- **可用性副本（Availability Replica）**：可用性组中的 SQL Server 实例
- **主副本（Primary Replica）**：承载主数据库的副本
- **辅助副本（Secondary Replica）**：承载辅助数据库的副本

```
┌─────────────────────────────────────────────────────────────────┐
│                    Always On 可用性组                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    可用性组 AG1                          │   │
│  │                                                         │   │
│  │  ┌───────────────┐         ┌───────────────┐           │   │
│  │  │   主副本     │ ──────▶ │   辅助副本    │           │   │
│  │  │  (PRIMARY)   │◀─────── │  (SECONDARY) │           │   │
│  │  └───────────────┘         └───────────────┘           │   │
│  │         │                           │                   │   │
│  │         │  同步/异步复制            │                   │   │
│  │         ▼                           ▼                   │   │
│  │  ┌─────────────┐           ┌─────────────┐            │   │
│  │  │ DB1 (PRIMARY)│         │ DB1 (SECONDARY)│          │   │

```

### 版本与功能

| 版本 | Always On 功能 |
|---|---|
| Enterprise Edition | 完整功能，支持最多 8 个副本 |
| Standard Edition | 基本可用性组，最多 2 个副本 |
| Express Edition | 无 Always On |

### Always On vs 数据库镜像

| 特性 | Always On | 数据库镜像 |
|---|---|---|
| 副本数量 | 最多 8 个 | 最多 1 个 |
| 多个数据库 | 支持 | 不支持 |
| 读写分离 | 支持 | 只读辅助副本 |
| 自动故障转移 | 支持 | 需见证服务器 |
| 备份到辅助副本 | 支持 | 支持 |

## 数据同步模式

### 同步提交模式

同步提交确保零数据丢失：

```
┌─────────────────────────────────────────────────────────────────┐
│                    同步提交模式                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 应用程序写入主副本                                          │
│  2. 主副本 写入本地日志                                          │
│  3. 主副本 发送日志到辅助副本                                    │
│  4. 辅助副本 写入本地日志                                        │
│  5. 辅助副本 发送确认到主副本                                    │
│  6. 主副本 返回成功给应用程序                                    │
│                                                                 │
│  延迟 = 网络延迟 × 2                                            │
│  优点：零数据丢失                                              │
│  缺点：有延迟，受网络影响                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 异步提交模式

异步提交最大化性能，可能少量数据丢失：

```
┌─────────────────────────────────────────────────────────────────┐
│                    异步提交模式                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 应用程序写入主副本                                          │
│  2. 主副本 写入本地日志，返回成功                               │
│  3. 主副本 后台发送日志到辅助副本                               │
│                                                                 │
│  延迟 = 主副本响应延迟 + 辅助副本处理                            │
│  优点：性能好，不阻塞应用                                        │
│  缺点：可能丢失少量数据（最后几个事务）                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 选择模式

| 场景 | 模式 | 原因 |
|---|---|---|
| 同一数据中心 | 同步提交 | 网络延迟低，零数据丢失 |
| 跨数据中心 | 异步提交 | 网络延迟高，保证性能 |
| 灾难恢复 | 异步提交 | 接受少量数据丢失 |
| 读写分离 | 同步 + 辅助只读 | 本地副本同步，远程副本异步 |

## 故障转移

### 故障转移类型

| 类型 | 条件 | 恢复时间 |
|---|---|---|
| 自动故障转移 | 同步 + 自动故障转移配置 | < 30 秒 |
| 手动故障转移 | 同步 + 管理员触发 | < 30 秒 |
| 强制故障转移 | 异步（可能数据丢失） | 手动 |

### 自动故障转移

```
┌─────────────────────────────────────────────────────────────────┐
│                    自动故障转移                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌───────────┐                              ┌───────────┐       │
│  │  主副本   │                              │  辅助副本  │       │
│  │  (PRIMARY)│                              │ (SECONDARY)│      │
│  └─────┬─────┘                              └─────┬─────┘       │
│        │                                          │             │
│        │ 检测到故障                               │             │
│        ▼                                          │             │
│  ┌───────────┐                                      │             │
│  │  自动    │◀──────────────────────────────────────│             │
│  │  故障转移 │                                      │             │
│  └─────┬─────┘                                      │             │
│        │                                            │             │
│        │  角色切换                                  ▼             │
│        │                              ┌───────────┐             │
│        │                              │  新的     │             │
│        └────────────────────────────▶│  主副本   │             │
│                                       └───────────┘             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 故障转移过程

1. WSFC 检测主副本不可用
2. 可用性组自动故障转移
3. 辅助副本角色变为 PRIMARY
4. 客户端重定向到新主副本
5. 应用程序透明重连

## 读写分离

### 辅助副本只读访问

Always On 支持将只读查询负载到辅助副本：

```sql
-- 配置辅助副本只读访问
ALTER AVAILABILITY GROUP [AG1]
MODIFY REPLICA ON 'Instance2'
WITH (SECONDARY_ROLE(ALLOW_CONNECTIONS = READ_ONLY));

-- 客户端连接字符串（读取意图路由）
"Server=tcp:ListenerName,1433;
  ApplicationIntent=ReadOnly;
  Database=YourDB;
  User ID=YourUser;
  Password=YourPassword;"
```

### 读取意图路由

```
┌─────────────────────────────────────────────────────────────────┐
│                    读取意图路由                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  应用程序 (ApplicationIntent=ReadOnly)                           │
│        │                                                        │
│        ▼                                                        │
│  ┌───────────────┐                                              │
│  │   监听器     │                                              │
│  │  (AG Listener)│                                             │
│  └───────┬───────┘                                              │
│          │                                                      │
│          │ 读取意图路由                                          │
│          ▼                                                      │
│  ┌───────────────┐                                              │
│  │   辅助副本    │  ← 只读连接自动路由到这里                      │
│  │ (SECONDARY)   │                                              │
│  └───────────────┘                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 只读工作负载隔离

```sql
-- 在辅助副本创建只读路由
ALTER AVAILABILITY GROUP [AG1]
MODIFY REPLICA ON 'Instance1'
WITH (PRIMARY_ROLE(READ_ONLY_ROUTING_LIST=('Instance2','Instance1')));

ALTER AVAILABILITY GROUP [AG1]
MODIFY REPLICA ON 'Instance2'
WITH (PRIMARY_ROLE(READ_ONLY_ROUTING_LIST=('Instance1','Instance2')));
```

## 备份策略

### 备份到辅助副本

Always On 允许备份到辅助副本，减少主副本负载：

```sql
-- 配置备份优先级
ALTER AVAILABILITY GROUP [AG1]
MODIFY REPLICA ON 'Instance2'
WITH (BACKUP_PRIORITY = 50);

-- 备份到辅助副本（优先）
BACKUP DATABASE [YourDB]
TO DISK = 'E:\Backup\YourDB.bak'
WITH COPY_ONLY, COMPRESSION;
```

### 备份类型

| 备份类型 | 位置 | 说明 |
|---|---|---|
| 完整备份 | 主副本或辅助副本 | 可在任一副本执行 |
| 日志备份 | 主副本或辅助副本 | 建议在主副本 |
| 文件备份 | 主副本 | 只能在主副本 |

### 自动备份首选项

```sql
-- 查看备份首选项
SELECT 
    a.name AS AGName,
    a.primary_recovery_health_desc,
    a.automated_backup_preference_desc
FROM sys.availability_groups;

-- 配置自动备份偏好
ALTER AVAILABILITY GROUP [AG1]
SET (AUTOMATED_BACKUP_PREFERENCE = SECONDARY);
```

## 监控与诊断

### DMV 监控

```sql
-- 查看副本同步状态
SELECT 
    ar.replica_server_name,
    adc.database_name,
    adr.synchronization_state_desc,
    adr.last_redone_lsn,
    adr.last_redone_time,
    adr.last_commit_lsn,
    adr.last_commit_time,
    adr.filestream_send_rate,
    adr.log_send_rate,
    adr.log_send_queue_size
FROM sys.dm_hadr_database_replica_states adr
JOIN sys.availability_replicas ar ON adr.replica_id = ar.replica_id
JOIN sys.availability_databases_cluster adc ON adr.group_database_id = adc.group_database_id;
```

## 最佳实践

### 网络要求

| 要求 | 建议 |
|---|---|
| 副本间延迟 | 同步 < 3ms，异步 < 100ms |
| 带宽 | 最低 1 Gbps |
| 防火墙 | 开放集群端口 |

### 性能优化

| 优化项 | 建议 |
|---|---|
| 文件初始化 | 启用即时文件初始化 |
| 压缩 | 启用日志压缩 |
| 批大小 | 优化日志批处理 |
| 端点加密 | 使用默认端点或配置加密 |

### 高可用设计

```sql
-- 推荐配置
-- 副本 1：主副本（数据中心 A）
-- 副本 2：同步辅助副本（数据中心 A）
-- 副本 3：异步辅助副本（数据中心 B，用于灾难恢复）
-- 副本 4+：异步辅助副本（云端，用于备份）
```

Always On 可用性组是企业级 SQL Server 高可用的标准解决方案。
