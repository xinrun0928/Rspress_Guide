# 索引类型

SQL Server 提供了多种索引类型，满足不同场景的需求。理解每种索引的适用场景，才能做出正确的设计选择。

## 索引分类概览

```
索引类型
├── 聚集索引
│   └── 每个表最多 1 个，数据按索引键排序存储
├── 非聚集索引
│   └── 每个表最多 999 个，与数据分离的 B+ 树
├── 唯一索引
│   └── 确保索引键唯一
├── 筛选索引（Filtered Index）
│   └── 只索引满足条件的行
├── 包含列索引（Covering Index）
│   └── 非聚集索引 + INCLUDE
├── 列存储索引（Columnstore Index）
│   └── 列式存储，适合分析查询
├── XML 索引
│   └── XML 数据类型专用
├── 空间索引
│   └── 地理位置数据专用
└── 全文索引
    └── 文本搜索
```

## 聚集索引

### 特点

- 每个表只能有 1 个聚集索引
- 数据按索引键顺序物理存储
- 叶节点是完整的数据行
- 适合范围查询和主键查询

```sql
-- 创建聚集索引
CREATE CLUSTERED INDEX IX_Orders_OrderID ON Orders(OrderID);

-- 主键自动创建唯一聚集索引（除非指定 NONCLUSTERED）
CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,  -- 自动创建唯一聚集索引
    ...
);
```

## 非聚集索引

### 特点

- 每个表最多 999 个
- 叶节点存储索引键和行定位器
- 行定位器指向聚集索引键或堆的 RID
- 支持覆盖查询（INCLUDE）

```sql
-- 创建非聚集索引
CREATE NONCLUSTERED INDEX IX_Orders_CustomerID ON Orders(CustomerID);

-- 复合索引
CREATE NONCLUSTERED INDEX IX_Orders_CustDate 
ON Orders(CustomerID, OrderDate DESC);

-- 包含列索引
CREATE NONCLUSTERED INDEX IX_Orders_CustomerID 
ON Orders(CustomerID)
INCLUDE (OrderDate, TotalAmount);
```

## 唯一索引

### 特点

- 确保索引键值唯一
- 可应用于聚集或非聚集索引
- 支持多列组合唯一

```sql
-- 唯一非聚集索引
CREATE UNIQUE NONCLUSTERED INDEX UX_Users_Email ON Users(Email);

-- 组合唯一索引
CREATE UNIQUE NONCLUSTERED INDEX UX_OrderItems 
ON OrderItems(OrderID, ProductID);

-- 如果已有重复数据，创建唯一索引会失败
-- 需要先清理重复数据
```

## 筛选索引（Filtered Index）

### 特点

- 只索引满足条件的行
- 索引更小，查询更快
- 适合稀疏数据

```sql
-- 只索引活跃订单
CREATE NONCLUSTERED INDEX IX_Orders_Active 
ON Orders(CustomerID, OrderDate)
WHERE Status = 'Active';  -- 筛选条件

-- 查询优化
SELECT OrderID, CustomerID, OrderDate, Status
FROM Orders
WHERE Status = 'Active' AND CustomerID = 100;
-- 使用 IX_Orders_Active

-- 不能使用 IX_Orders_Active 的查询
SELECT OrderID, CustomerID, OrderDate
FROM Orders
WHERE Status = 'Shipped';  -- 状态不同
```

### 筛选索引的 WHERE 子句规则

```sql
-- 支持的表达式
WHERE Column IS NOT NULL
WHERE Column > 0
WHERE Column IN (1, 2, 3)
WHERE Column LIKE 'A%'
WHERE Column = 'Active' AND Amount > 100

-- 不支持的表达式
WHERE UPPER(Column) = 'VALUE'  -- 函数
WHERE Column + 1 = 10           -- 表达式
WHERE GETDATE() BETWEEN ...     -- 函数
```

## 包含列索引（Covering Index）

### 特点

- 非聚集索引 + INCLUDE 子句
- 将查询需要的列包含在索引中
- 避免回表（Key Lookup）

```sql
-- 不使用覆盖索引
CREATE NONCLUSTERED INDEX IX_Orders_CustomerID ON Orders(CustomerID);

SELECT OrderID, OrderDate, TotalAmount
FROM Orders
WHERE CustomerID = 100;
-- 执行计划：Index Seek + Key Lookup（回表）

-- 使用覆盖索引
CREATE NONCLUSTERED INDEX IX_Orders_CustomerID 
ON Orders(CustomerID)
INCLUDE (OrderDate, TotalAmount);

SELECT OrderID, OrderDate, TotalAmount
FROM Orders
WHERE CustomerID = 100;
-- 执行计划：仅 Index Seek，无需回表
```

### 包含列 vs 复合索引

| 对比 | 复合索引 | 包含列索引 |
|---|---|---|
| 列存储位置 | B+ 树键 | 索引叶级页（非键） |
| 键大小影响 | 影响索引深度 | 不影响深度 |
| 支持列数 | 受键大小限制 | 最多 1023 列 |
| 排序支持 | 支持 | 不支持 |

```sql
-- 复合索引（列在键中）
CREATE INDEX IX_Test1 ON TestTable(Col1, Col2, Col3);

-- 包含列索引（排序列在键中，其他列在 INCLUDE）
CREATE INDEX IX_Test2 ON TestTable(Col1, Col2) INCLUDE (Col3);

-- 如果 Col3 需要排序，用复合索引
SELECT * FROM TestTable 
WHERE Col1 = 1 ORDER BY Col2, Col3;
-- IX_Test2 满足（按 Col2 排序）
-- IX_Test3 无 INCLUDE，需要回表
```

## 列存储索引（Columnstore Index）

### 特点

- 列式存储，不是传统 B+ 树
- 高度压缩（10-100x）
- 批量模式处理
- 适合分析查询（OLAP）

```sql
-- 创建列存储索引（SQL Server 2012+）
CREATE NONCLUSTERED COLUMNSTORE INDEX IX_Orders_ColumnStore
ON Orders(OrderDate, CustomerID, TotalAmount, Status);

-- 聚集列存储索引（SQL Server 2014+）
CREATE CLUSTERED COLUMNSTORE INDEX IX_Orders_CCS
ON Orders;

-- 混合：聚集索引 + 列存储索引
CREATE CLUSTERED INDEX IX_Orders_PK ON Orders(OrderID);
CREATE NONCLUSTERED COLUMNSTORE INDEX IX_Orders_CS 
ON Orders(OrderDate, CustomerID, TotalAmount);
```

### 行存储 vs 列存储

| 特性 | 行存储（B+ 树） | 列存储（Columnstore） |
|---|---|---|
| 数据组织 | 按行 | 按列 |
| 压缩率 | 1-3x | 10-100x |
| 读取方式 | 整行读取 | 按列读取 |
| 适合场景 | OLTP（点查询） | OLAP（分析） |
| 批量插入 | 高效 | 需要批量模式 |

```sql
-- 分析查询：使用列存储
SELECT 
    YEAR(OrderDate) AS Year,
    MONTH(OrderDate) AS Month,
    COUNT(*) AS OrderCount,
    SUM(TotalAmount) AS TotalSales
FROM Orders
GROUP BY YEAR(OrderDate), MONTH(OrderDate);
-- 列存储可以只读取需要的列，避免读取其他列
```

## XML 索引

### 特点

- 针对 XML 数据类型的索引
- 支持 XML 列的查询优化

```sql
-- XML 列
CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    OrderData XML
);

-- 主 XML 索引
CREATE PRIMARY XML INDEX PXML_Orders_Data ON Orders(OrderData);

-- 次级 XML 索引
CREATE XML INDEX SXML_Orders_Value 
ON Orders(OrderData)
USING XML INDEX PXML_Orders_Data
FOR VALUE;

CREATE XML INDEX SXML_Orders_Path
ON Orders(OrderData)
USING XML INDEX PXML_Orders_Data
FOR PATH;

CREATE XML INDEX SXML_Orders_Property
ON Orders(OrderData)
USING XML INDEX PXML_Orders_Data
FOR PROPERTY;
```

## 空间索引

### 特点

- 针对地理位置数据（geometry/geography）
- 支持空间查询优化

```sql
-- 空间列
CREATE TABLE Stores (
    StoreID INT PRIMARY KEY,
    StoreName NVARCHAR(100),
    Location GEOGRAPHY
);

-- 空间索引
CREATE SPATIAL INDEX SIX_Stores_Location 
ON Stores(Location)
USING GEOGRAPHY_AUTO_GRID;

-- 查询：查找范围内的门店
SELECT * FROM Stores
WHERE Location.STWithin(
    GEOGRAPHY::Point(116.4, 39.9, 4326).STBuffer(5000)
);
```

## 全文索引

### 特点

- 文本搜索引擎
- 支持词根、分词、模糊匹配

```sql
-- 创建全文目录
CREATE FULLTEXT CATALOG ft_Catalog AS DEFAULT;

-- 全文索引
CREATE FULLTEXT INDEX ON Products(Description, Name)
KEY INDEX PK_Products
ON ft_Catalog;

-- 全文搜索
SELECT ProductName, Description
FROM Products
WHERE CONTAINS((Name, Description), 'laptop OR notebook');
```

## 索引选项

### 常用选项

```sql
CREATE NONCLUSTERED INDEX IX_Orders_Date 
ON Orders(OrderDate)
WITH (
    FILLFACTOR = 80,          -- 填充因子
    PAD_INDEX = ON,           -- 非叶节点也应用填充因子
    SORT_IN_TEMPDB = ON,      -- 排序在 tempdb
    ONLINE = ON,              -- 联机创建（Enterprise）
    ALLOW_ROW_LOCKS = ON,     -- 允许行锁
    ALLOW_PAGE_LOCKS = ON,    -- 允许页锁
    MAXDOP = 4                -- 最大并行度
);
```

| 选项 | 说明 |
|---|---|
| `FILLFACTOR` | 叶级页填充率 |
| `PAD_INDEX` | 非叶节点填充 |
| `SORT_IN_TEMPDB` | 使用 tempdb 排序，减少日志 |
| `ONLINE` | 联机创建，不阻塞表访问 |
| `ALLOW_ROW_LOCKS` | 允许行锁 |
| `ALLOW_PAGE_LOCKS` | 允许页锁 |

## 查看索引信息

```sql
-- 查看表的所有索引
EXEC sp_helpindex 'Orders';

-- 查看索引详情
SELECT 
    i.name AS IndexName,
    i.type_desc AS IndexType,
    i.is_primary_key,
    i.is_unique,
    i.fill_factor,
    STRING_AGG(c.name, ', ') WITHIN GROUP (ORDER BY ic.key_ordinal) AS KeyColumns,
    COALESCE(STRING_AGG(cc.name, ', '), '') AS IncludeColumns
FROM sys.indexes i
JOIN sys.index_columns ic ON i.object_id = ic.object_id AND i.index_id = ic.index_id AND ic.is_included_column = 0
LEFT JOIN sys.index_columns icc ON i.object_id = icc.object_id AND i.index_id = icc.index_id AND icc.is_included_column = 1
JOIN sys.columns c ON ic.object_id = c.object_id AND ic.column_id = c.column_id
LEFT JOIN sys.columns cc ON icc.object_id = cc.object_id AND icc.column_id = cc.column_id
WHERE i.object_id = OBJECT_ID('Orders')
GROUP BY i.name, i.type_desc, i.is_primary_key, i.is_unique, i.fill_factor;
```

理解不同索引类型的特点，才能选择最适合当前场景的索引方案。
