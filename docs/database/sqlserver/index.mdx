# SQL Server 数据库

凌晨 3 点，你被生产环境的报警电话叫醒——数据库连接超时，应用完全无法响应。

这不是某个小众场景的边缘 bug，而是 SQL Server 在高并发写入下的典型症状：锁升级（Lock Escalation）导致整张表被独占锁定，所有并发请求都在排队等待。

相比 MySQL 的 InnoDB，SQL Server 的锁机制和存储引擎有着截然不同的设计哲学。了解这些差异，是成为 SQL Server 专家的必经之路。

## 为什么选择 SQL Server

SQL Server 是微软的企业级关系型数据库，在 Windows 生态中占据统治地位。它的核心优势体现在三个方面：

**深度集成 Windows 生态系统**。AD 域身份验证、SSIS 数据集成、SSRS 报表服务、SSAS 分析服务，这些组件构成了完整的数据平台。AD 组的精细权限控制，比 MySQL 的用户权限体系强大得多。

**企业级特性开箱即用**。Always On 可用性组、在线索引重建、快照隔离级别、列存储索引，这些在 MySQL 中需要额外配置或根本不支持的特性，在 SQL Server 中只需勾选即可启用。

**性能调优工具链完善**。执行计划的可视化分析、DMV 动态管理视图、扩展事件（Extended Events）、Query Store 查询存储，这些工具让 DBA 能够精准定位性能瓶颈，而不是靠猜。

## SQL Server 架构概览

SQL Server 采用两层架构：**协议层**负责客户端通信，**关系引擎**负责 SQL 解析和执行，**存储引擎**负责数据读写和事务管理。

```
客户端请求
    ↓
协议层 (Tabular Data Stream/TDS)
    ↓
关系引擎 (解析器 → 查询优化器 → 执行器)
    ↓
存储引擎 (访问方法 → 缓冲池 → 磁盘文件)
    ↓
数据文件 (.mdf) / 日志文件 (.ldf)
```

**缓冲池（Buffer Pool）** 是 SQL Server 内存的核心。数据页从磁盘读取后缓存在内存中，写操作先写入日志，再修改缓冲池中的页，最后由检查点（Checkpoint）异步刷回磁盘。这种 Write-Ahead Logging 机制保证了事务的持久性。

## 与 MySQL 的核心差异

对于从 MySQL 转型的开发者，以下几点需要特别注意：

| 特性 | MySQL (InnoDB) | SQL Server |
|---|---|---|
| 索引结构 | B+ 树 | B+ 树 |
| 默认隔离级别 | REPEATABLE READ | READ COMMITTED |
| 锁粒度 | 行锁（辅助以-gap锁） | 行锁、页锁、表锁 |
| 锁升级 | 无（只在高水位标记） | 有（阈值触发） |
| MVCC | 支持（Read View） | 支持（行版本） |
| 高可用 | 主从复制、Group Replication | Always On、镜像、复制 |

面试中，面试官经常问到「SQL Server 和 MySQL 在锁机制上的区别」，这正是理解两者架构差异的关键切入点。
