# PostgreSQL 统计信息

统计信息是 PostgreSQL 查询优化器做出正确决策的基础。如果统计信息不准确，优化器可能会选择错误的执行计划，导致查询变慢。

---

## 什么是统计信息

### 统计信息的内容

PostgreSQL 为每个列维护以下统计信息：

| 统计项 | 说明 |
|---|---|
| `n_distinct` | 不同值的数量 |
| `avg_width` | 平均字节宽度 |
| `null_frac` | NULL 值比例 |
| `most_common_vals` | 最常见的值 |
| `most_common_freqs` | 最常见值的频率 |
| `histogram_bounds` | 直方图边界值 |

```sql
-- 查看统计信息
SELECT * FROM pg_stats WHERE tablename = 'orders' AND attname = 'status';
```

### 统计信息的用途

优化器使用统计信息估算：

1. **行数**：查询会返回多少行
2. **选择性**：WHERE 条件过滤掉多少行
3. **成本**：不同执行计划的相对成本

## 查看统计信息

### 基本查询

```sql
-- 查看表的所有列统计信息
SELECT
    attname,
    n_distinct,
    null_frac,
    avg_width,
    correlation
FROM pg_stats
WHERE tablename = 'orders'
ORDER BY attname;
```

### 查看直方图

```sql
-- 查看数值列的直方图
SELECT
    attname,
    histogram_bounds
FROM pg_stats
WHERE tablename = 'orders'
  AND attname = 'total';
```

### 查看高频值

```sql
-- 查看高频值
SELECT
    attname,
    most_common_vals,
    most_common_freqs
FROM pg_stats
WHERE tablename = 'orders'
  AND attname = 'status';
```

## 统计信息的收集

### ANALYZE 命令

```sql
-- 收集表的统计信息
ANALYZE orders;

-- 收集特定列的统计信息
ANALYZE orders(status, created_at);

-- 收集整个数据库的统计信息
ANALYZE VERBOSE;
```

### autovacuum 自动收集

autovacuum 进程会自动执行 ANALYZE：

```
触发条件 = autovacuum_analyze_threshold + 表大小 * autovacuum_analyze_scale_factor
```

```sql
-- 查看 autovacuum 配置
SHOW autovacuum_analyze_threshold;
SHOW autovacuum_analyze_scale_factor;
```

### 手动设置阈值

```sql
-- 为特定表设置更低的阈值
ALTER TABLE orders SET (
    autovacuum_analyze_threshold = 100,
    autovacuum_analyze_scale_factor = 0.01
);
```

## 统计信息不准确的后果

### 问题场景

```sql
-- 表突然增加大量数据，但没有 ANALYZE
INSERT INTO orders SELECT * FROM orders;  -- 数据翻倍
INSERT INTO orders SELECT * FROM orders;  -- 再翻倍

-- 查询使用旧的统计信息
EXPLAIN SELECT * FROM orders WHERE status = 'pending';
-- 预估行数：10000（实际可能是 100000）
```

### 解决方案

```sql
-- 手动 ANALYZE
ANALYZE orders;

-- 或者使用 pg_stats_statements 发现异常查询
SELECT
    query,
    rows / calls AS avg_rows,
    100.0 * rows / (SELECT SUM(rows) FROM pg_stat_statements) AS pct
FROM pg_stat_statements
ORDER BY pct DESC
LIMIT 10;
```

## 扩展统计信息

### PostgreSQL 10+ 功能

```sql
-- 创建表达式统计信息
CREATE STATISTICS s_orders_status_created (dependencies)
ON status, created_at
FROM orders;

-- 收集统计信息
ANALYZE orders;

-- 查看相关性
SELECT
    statistics_name,
    dependency_matrix
FROM pg_statistic_ext
WHERE statistics_name = 's_orders_status_created';
```

### 使用扩展统计信息

```sql
-- 创建统计信息
CREATE STATISTICS s_region_status (dependencies, ndistinct)
ON region, status
FROM customers;

ANALYZE customers;

-- 查询会使用相关性统计
SELECT * FROM customers
WHERE region = 'North' AND status = 'active';
```

## 统计信息参数调优

### 全局参数

```sql
-- 默认统计目标
SHOW default_statistics_target;

-- 设置全局统计目标（影响所有列）
SET default_statistics_target = 200;

-- 立即生效
ANALYZE;
```

### 列级别参数

```sql
-- 设置特定列的统计目标（范围 1-10000）
ALTER TABLE orders ALTER COLUMN status SET STATISTICS 500;
ALTER TABLE orders ALTER COLUMN created_at SET STATISTICS 1000;

-- 收集统计信息
ANALYZE orders;
```

### 推荐配置

```sql
-- 高频查询列：提高统计目标
ALTER TABLE orders ALTER COLUMN customer_id SET STATISTICS 1000;
ALTER TABLE orders ALTER COLUMN status SET STATISTICS 500;

-- 低频查询列：降低统计目标
ALTER TABLE orders ALTER COLUMN notes SET STATISTICS 100;
```

## 统计信息与查询优化

### 常见问题

```sql
-- 问题 1：预估行数远大于实际行数
-- 原因：统计信息过时
EXPLAIN SELECT * FROM orders WHERE status = 'pending';
-- Solution: ANALYZE orders

-- 问题 2：预估行数远小于实际行数
-- 原因：统计信息过时或数据分布不均匀
EXPLAIN SELECT * FROM orders WHERE created_at > '2024-01-01';
-- Solution: ANALYZE orders

-- 问题 3：JOIN 顺序错误
-- 原因：统计信息不准确导致成本估算错误
EXPLAIN SELECT * FROM orders o
JOIN customers c ON o.customer_id = c.id;
-- Solution: ANALYZE orders; ANALYZE customers;
```

## 监控统计信息

### pg_stat_user_tables

```sql
-- 查看表的统计信息收集状态
SELECT
    relname,
    n_live_tup,
    n_dead_tup,
    last_analyze,
    last_autoanalyze,
    n_mod_since_analyze
FROM pg_stat_user_tables
WHERE relname = 'orders';
```

### pg_statistic vs pg_stats

```sql
-- pg_stats 是 pg_statistic 的视图，更易读
SELECT * FROM pg_stats WHERE tablename = 'orders';
```

## 总结

统计信息是查询优化的基础：

1. **作用**：帮助优化器估算行数、选择性和成本
2. **收集**：ANALYZE 命令和 autovacuum 自动收集
3. **问题**：统计信息不准确导致错误的执行计划
4. **优化**：调整统计目标，创建扩展统计信息

定期监控统计信息的收集情况，对性能问题保持警惕，是数据库维护的重要工作。
