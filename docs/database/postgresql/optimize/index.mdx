# PostgreSQL 性能优化

这一章讲解 PostgreSQL 的性能优化，从执行计划分析到慢查询优化，从分区表到并行查询，是 PostgreSQL 调优的核心内容。

## 内容导航

- **[执行计划分析](explain)**：理解 EXPLAIN 输出，优化查询计划
- **[慢查询分析](slow-query)**：发现、分析、优化慢查询
- **[统计信息](statistics)**：统计信息的作用和调优
- **[连接池](connection-pool)**：PostgreSQL 连接池配置
- **[分区表](partition)**：Range、List、Hash 分区的设计与优化
- **[并行查询](parallel-query)**：利用多核加速查询
- **[性能监控](monitor)**：监控数据库性能指标

## 学习路径建议

建议按以下顺序学习：

1. 先看 [执行计划分析](explain)，理解查询计划的含义
2. 再看 [慢查询分析](slow-query)，掌握分析方法
3. 学习 [统计信息](statistics)，理解查询优化器的工作原理
4. 然后学习 [分区表](partition) 和 [并行查询](parallel-query)
5. 最后看 [性能监控](monitor)，建立监控体系
