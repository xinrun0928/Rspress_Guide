# PostgreSQL Citus 分布式集群

Citus 是 PostgreSQL 的分布式扩展，可以将 PostgreSQL 变成分布式数据库，支持水平扩展。

---

## Citus 概述

Citus 通过分片（Shard）将数据分布在多个节点上，实现水平扩展。

```

│    Worker 1   │                               │    Worker 2   │
```

## 安装配置

### 安装 Citus

```bash
# Ubuntu/Debian
sudo apt-get install postgresql-14-citus

# 或者使用 Docker
docker run -d --name citus \
  -e POSTGRES_PASSWORD=mypasswd \
  citusdata/citus
```

### 初始化集群

```sql
-- 在协调节点执行
CREATE EXTENSION citus;

-- 添加 worker 节点
SELECT citus_add_node('worker1', 5432);
SELECT citus_add_node('worker2', 5432);

-- 查看集群状态
SELECT * FROM citus_get_active_worker_nodes();
SELECT * FROM citus_shards();
```

## 表类型

### 分布式表

```sql
-- 创建分布式表（按 tenant_id 分片）
CREATE TABLE orders (
    id BIGSERIAL,
    tenant_id BIGINT,
    customer_id BIGINT,
    total NUMERIC,
    created_at TIMESTAMPTZ
);

-- 选择分片键
SELECT create_distributed_table('orders', 'tenant_id');

-- 现在 orders 表会自动分片到 worker 节点
```

### 引用表

```sql
-- 创建引用表（复制到所有节点）
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    email VARCHAR UNIQUE
);

SELECT create_reference_table('users');
```

### 本地表

```sql
-- 普通 PostgreSQL 表，不分片
CREATE TABLE app_config (
    key VARCHAR PRIMARY KEY,
    value TEXT
);
-- 不调用 create_distributed_table，保持为本地表
```

## 分片策略

### Hash 分片（默认）

```sql
-- 按 tenant_id 哈希分片
SELECT create_distributed_table('orders', 'tenant_id', 'hash');
```

### Range 分片

```sql
-- 按 ID 范围分片
SELECT create_distributed_table('events', 'event_id', 'range');

-- 手动定义分片范围
SELECT create_range_partition(
    'events',
    0,          -- 起始值
    1000000     -- 结束值
);
```

### 滚动分片

```sql
-- 按时间分片
SELECT create_distributed_table('logs', 'created_at', 'append');
```

## 查询路由

### Coordinator 自动路由

```sql
-- 查询自动路由到正确的分片
SELECT * FROM orders WHERE tenant_id = 1;

-- 聚合查询自动合并结果
SELECT tenant_id, COUNT(*), SUM(total)
FROM orders
GROUP BY tenant_id;
```

### 跨分片查询

```sql
-- 两表 JOIN（需要分片键关联）
SELECT * FROM orders o
JOIN users u ON o.tenant_id = u.tenant_id
WHERE o.tenant_id = 1;

-- 无分片键关联会失败
-- SELECT * FROM orders o JOIN users u ON o.user_id = u.id;
```

## 分布式事务

### 事务支持

```sql
-- 简单事务（单分片）
BEGIN;
INSERT INTO orders (tenant_id, total) VALUES (1, 100);
COMMIT;

-- 分布式事务（多节点）
BEGIN;
INSERT INTO orders (tenant_id, total) VALUES (1, 100);
UPDATE users SET order_count = order_count + 1 WHERE tenant_id = 1;
COMMIT;
```

### 2PC 支持

```sql
-- Citus 使用 2PC 保证分布式事务一致性
-- 配置：
SET citus.multi_shard_commit_protocol = '2pc';
```

## 扩容

### 添加节点

```sql
-- 在线添加节点
SELECT citus_add_node('worker3', 5432);

-- 重新分配分片
SELECT citus_rebalance_start();
```

### 分片再平衡

```sql
-- 查看分片分布
SELECT
    nodename,
    COUNT(*) AS shard_count
FROM citus_shards
GROUP BY nodename;

-- 手动移动分片
SELECT citus_move_shard_placement(
    102041,           -- 分片 ID
    'worker1',        -- 源节点
    'worker3'         -- 目标节点
);
```

## 性能调优

### 分片数量

```sql
-- 创建更多分片
CREATE TABLE orders (
    id BIGSERIAL,
    tenant_id BIGINT,
    ...
);
SELECT create_distributed_table(
    'orders',
    'tenant_id',
    'hash',
    shard_count := 64  -- 默认 32
);
```

### 写入优化

```sql
-- 批量插入（减少网络往返）
COPY orders (tenant_id, total) FROM STDIN WITH BINARY;

-- 使用 INSERT INTO ... ON CONFLICT
INSERT INTO orders (tenant_id, total) VALUES (1, 100)
ON CONFLICT (tenant_id, id) DO NOTHING;
```

## 监控

```sql
-- 查看集群状态
SELECT * FROM citus_check_cluster_size();

-- 查看分片大小
SELECT
    shardid,
    shardlength,
    nodename
FROM citus_shard_placements
ORDER BY shardlength DESC;

-- 查看复制状态
SELECT * FROM citus_get_active_worker_nodes();
```

## 限制

### 不支持的功能

- 序列的 CURRVAL/LASTVAL（跨分片）
- 某些触发器
- 外键跨分片（引用表除外）
- 某些窗口函数

### 使用限制

- JOIN 需要分片键或引用表
- 分片键不能修改
- 分布式事务有一定延迟

---

## 总结

Citus 是 PostgreSQL 水平扩展的解决方案：

1. **分片**：数据分布到多个节点
2. **查询路由**：Coordinator 自动路由查询
3. **引用表**：跨节点 JOIN
4. **扩容**：在线添加节点和分片再平衡
5. **限制**：需要考虑分片键设计

Citus 让你用 PostgreSQL 的方式构建分布式数据库，适合多租户 SaaS、实时分析等场景。
