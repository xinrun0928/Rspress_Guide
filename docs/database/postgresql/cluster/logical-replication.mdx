# PostgreSQL 逻辑复制

逻辑复制是 PostgreSQL 10 引入的特性，允许在表级别进行数据复制，支持异构复制、选择性复制等场景。

## 逻辑复制 vs 物理复制

| 特性 | 逻辑复制 | 物理复制 |
|---|---|---|
| 复制粒度 | 表级别 | 数据库级别 |
| 数据格式 | 逻辑数据 | 物理 WAL |
| 异构支持 | 支持（不同版本） | 不支持 |
| 复制延迟 | 可能稍高 | 极低 |
| 冲突处理 | 需要处理 | 自动处理 |

## 配置步骤

### 1. 主节点配置

```ini
# postgresql.conf

# 启用逻辑复制
wal_level = logical

# 复制槽数量
max_replication_slots = 10

# 发布数量
max_wal_senders = 10
```

### 2. 创建发布

```sql
-- 创建数据库级别的发布
CREATE PUBLICATION my_publication FOR ALL TABLES;

-- 创建表级别的发布
CREATE PUBLICATION my_publication FOR TABLE users, orders;

-- 带有筛选条件的发布
CREATE PUBLICATION active_users FOR TABLE users
    WHERE (status = 'active');

-- 查看发布
SELECT * FROM pg_publication;
SELECT * FROM pg_publication_tables;
```

### 3. 创建订阅

```sql
-- 在从节点创建订阅
CREATE SUBSCRIPTION my_subscription
    CONNECTION 'host=master port=5432 dbname=mydb user=postgres password=xxx'
    PUBLICATION my_publication;

-- 查看订阅
SELECT * FROM pg_subscription;
SELECT * FROM pg_subscription_rel;
```

## 复制管理

### 添加表到发布

```sql
-- 添加表
ALTER PUBLICATION my_publication ADD TABLE products;

-- 移除表
ALTER PUBLICATION my_publication DROP TABLE old_products;
```

### 刷新订阅

```sql
-- 刷新订阅数据（重新同步）
ALTER SUBSCRIPTION my_subscription REFRESH PUBLICATION;

-- 禁用/启用订阅
ALTER SUBSCRIPTION my_subscription DISABLE;
ALTER SUBSCRIPTION my_subscription ENABLE;
```

### 删除订阅

```sql
-- 先删除订阅
DROP SUBSCRIPTION my_subscription;

-- 如果有复制槽，需要先删除槽
SELECT pg_drop_replication_slot('subscription_slot');
```

## 复制方式

### 表结构复制

逻辑复制会复制表结构，但**不会复制**：

- 索引
- 触发器
- 约束（NOT NULL 等）
- 序列

需要手动同步这些对象。

### 初始数据同步

创建订阅时，会自动进行初始数据同步。如果初始同步失败：

```sql
-- 查看同步状态
SELECT * FROM pg_stat_subscription;

-- 手动同步表
-- 1. 在订阅端清空表
TRUNCATE TABLE my_table;

-- 2. 刷新订阅
ALTER SUBSCRIPTION my_subscription REFRESH PUBLICATION;
```

## 冲突处理

### 常见冲突

```sql
-- 唯一键冲突
-- DELETE/UPDATE 在订阅端不存在

-- 序列冲突
-- 主键生成的值不一致
```

### 解决方案

```sql
-- 方案 1：跳过冲突
SELECT pg_replication_origin_advance('my_subscription', '0/0');

-- 方案 2：使用 COPY 选项
CREATE SUBSCRIPTION my_subscription
    CONNECTION 'host=master port=5432 dbname=mydb'
    PUBLICATION my_publication
    WITH (copy_data = false);

-- 方案 3：手动处理
-- 在订阅端解决冲突后，重新启用订阅
ALTER SUBSCRIPTION my_subscription ENABLE;
```

### 复制冲突监控

```sql
-- 查看订阅错误
SELECT
    subname,
    pid,
    received_lsn,
    last_msg_send_time,
    last_msg_receipt_time,
    latest_end_lsn,
    latest_end_time
FROM pg_stat_subscription;
```

## 实战案例

### 案例一：只复制部分表

```sql
-- 只复制活跃数据
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR,
    status VARCHAR
);

-- 创建发布
CREATE PUBLICATION active_data FOR TABLE users
    WHERE (status = 'active');

-- 创建订阅
CREATE SUBSCRIPTION active_sub
    CONNECTION 'host=master port=5432 dbname=mydb'
    PUBLICATION active_data;
```

### 案例二：多订阅者

```sql
-- 总部订阅所有数据
CREATE PUBLICATION full_data FOR ALL TABLES;

-- 分支只订阅需要的表
CREATE PUBLICATION branch_data FOR TABLE orders, customers;

-- 创建多个订阅
CREATE SUBSCRIPTION headquarter_sub ...;
CREATE SUBSCRIPTION branch1_sub ...;
CREATE SUBSCRIPTION branch2_sub ...;
```

### 案例三：级联复制

```
Master -> Replica1 -> Replica2
```

```sql
-- 在 Replica1 上创建发布
CREATE PUBLICATION replica_data FOR ALL TABLES;

-- 在 Replica2 上创建订阅
CREATE SUBSCRIPTION replica_sub ...;
```

## 与外部系统集成

### 逻辑复制到 Kafka

```python
import psycopg2
from kafka import KafkaProducer

conn = psycopg2.connect(...)
conn.set_isolation_level(pyscopg2.extensions.ISOLATION_LEVEL_AUTOCOMMIT)

cur = conn.cursor()
cur.execute("SELECT * FROM pg_logical_slot_get_changes('slot_name', NULL, NULL)")

producer = KafkaProducer(...)

while True:
    msg = cur.fetchone()
    if msg:
        producer.send('postgres-changes', value=msg)
```

### 逻辑复制到其他数据库

通过 Debezium 等工具，可以将逻辑复制的数据同步到 MySQL、MongoDB、Elasticsearch 等。

## 性能调优

### 参数配置

```ini
# 订阅端
max_logical_replication_workers = 4
max_sync_workers_per_subscription = 1

# 主节点
max_wal_senders = 10
wal_sender_delay = 200ms
```

### 监控

```sql
-- 查看复制状态
SELECT * FROM pg_stat_subscription;

-- 查看复制延迟
SELECT
    subname,
    pg_wal_lsn_diff(pg_current_wal_lsn(), confirmed_flush_lsn) AS lag
FROM pg_stat_subscription;
```

## 总结

逻辑复制是 PostgreSQL 的强大特性：

1. **表级别复制**：灵活选择复制对象
2. **异构支持**：支持不同版本间的复制
3. **选择性复制**：通过 WHERE 条件筛选
4. **冲突处理**：需要处理唯一键、序列等冲突
5. **集成能力**：可以与 Kafka、Debezium 等集成

逻辑复制适用于数据同步、CDC（Change Data Capture）、多活架构等场景。
