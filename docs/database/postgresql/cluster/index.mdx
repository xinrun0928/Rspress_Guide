# PostgreSQL 集群与高可用

这一章介绍 PostgreSQL 的集群和HA（High Availability，高可用）方案，包括流复制、逻辑复制、主从切换等。

## 内容导航

- **[流复制原理](streaming-replication)**：物理复制的主从同步
- **[主从切换](failover)**：故障转移和故障恢复
- **[逻辑复制](logical-replication)**：逻辑级别的复制，支持异构复制
- **[Citus 分布式集群](citus)**：水平扩展的分布式数据库
- **[备份恢复](backup)**：数据备份和恢复策略
- **[高可用方案](ha-solution)**：常见 HA 架构和方案

## 学习路径建议

建议按以下顺序学习：

1. 先看 [流复制原理](streaming-replication)，理解 PostgreSQL 的主从同步机制
2. 再看 [主从切换](failover)，理解故障转移
3. 学习 [逻辑复制](logical-replication)，理解异构复制
4. 了解 [备份恢复](backup) 和 [高可用方案](ha-solution)
