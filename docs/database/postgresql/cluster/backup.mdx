# PostgreSQL 备份恢复

数据是企业的核心资产。完善的备份策略是数据库运维的重中之重。

## 备份类型

### 物理备份 vs 逻辑备份

| 类型 | 说明 | 工具 |
|---|---|---|
| 物理备份 | 复制数据文件 | pg_basebackup |
| 逻辑备份 | 导出 SQL | pg_dump, pg_dumpall |

### 全量备份 vs 增量备份

| 类型 | 说明 | 工具 |
|---|---|---|
| 全量备份 | 备份所有数据 | pg_basebackup, pg_dump |
| 增量备份 | 只备份变更的 WAL | WAL 归档 |

---

## 物理备份

### pg_basebackup

```bash
# 基本用法
pg_basebackup -h master -U replicator -D /backup/base -Ft -z -P

# 参数说明：
# -Ft：使用 tar 格式
# -z：gzip 压缩
# -P：显示进度
# -R：创建恢复配置文件
```

### 备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR=/backup/postgresql
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_PATH=$BACKUP_DIR/base_$DATE

mkdir -p $BACKUP_PATH

# 执行备份
pg_basebackup -h localhost -U postgres -D $BACKUP_PATH -Ft -z -P -Xs

# 删除 7 天前的备份
find $BACKUP_DIR -type d -mtime +7 -exec rm -rf {} \;
```

## 逻辑备份

### pg_dump

```bash
# 备份单个数据库
pg_dump -h localhost -U postgres -d mydb -F c -b -v -f mydb.dump

# 备份多个表
pg_dump -h localhost -U postgres -d mydb \
       -t orders -t customers \
       -F c -f tables.dump

# 备份结构（不含数据）
pg_dump -h localhost -U postgres -d mydb \
       --schema-only \
       -F c -f schema.dump

# 备份数据（不含结构）
pg_dump -h localhost -U postgres -d mydb \
       --data-only \
       -F c -f data.dump
```

参数说明：
- `-F c`：自定义格式（推荐）
- `-F p`：纯 SQL 格式
- `-F t`：tar 格式
- `-b`：包含大对象
- `-v`：详细输出

### pg_dumpall

```bash
# 备份所有数据库（包括角色、表空间）
pg_dumpall -h localhost -U postgres -f alldbs.sql
```

### 恢复

```bash
# 恢复自定义格式
pg_restore -h localhost -U postgres -d mydb -v mydb.dump

# 恢复前清空数据库
pg_restore -h localhost -U postgres -d mydb --clean -v mydb.dump

# 恢复纯 SQL 格式
psql -h localhost -U postgres -d mydb -f mydb.sql
```

## WAL 归档备份

### 配置 WAL 归档

```ini
# postgresql.conf

# 启用归档
archive_mode = on

# 归档命令
archive_command = 'cp %p /archive/wal/%f'

# 或使用 rsync
# archive_command = 'rsync -a %p postgres@backup:/archive/wal/%f'
```

### WAL 归档备份

```bash
# 查看当前 WAL
SELECT pg_switch_wal();

# 强制归档
SELECT pg_switch_wal();
```

## Point-in-Time Recovery (PITR)

### 恢复流程

```bash
# 1. 准备恢复目录
rm -rf /var/lib/postgresql/data
mkdir -p /var/lib/postgresql/data
chown postgres:postgres /var/lib/postgresql/data

# 2. 恢复基础备份
cd /var/lib/postgresql/data
tar -xzf /backup/base/base_20240115.tar.gz

# 3. 配置恢复参数
cat >> /var/lib/postgresql/data/postgresql.conf << EOF
restore_command = 'cp /archive/wal/%f %p'
recovery_target_time = '2024-01-15 14:30:00'
recovery_target_action = 'promote'
EOF

# 4. 创建恢复信号文件
touch /var/lib/postgresql/data/recovery.signal

# 5. 启动数据库
pg_ctl start -D /var/lib/postgresql/data
```

### 恢复配置参数

```ini
# postgresql.conf

# 恢复到某个时间点
recovery_target_time = '2024-01-15 14:30:00'

# 恢复到某个事务 ID
recovery_target_xid = '12345'

# 恢复到某个 LSN
recovery_target_lsn = '0/7000000'

# 恢复到最近时间
recovery_target_action = 'pause'  # 或 'promote', 'shutdown'

# 是否包含事务 ID
recovery_target_inclusive = on

# 恢复到命名恢复点
# recovery_target_name = 'before_upgrade'
```

## Barman 备份工具

Barman 是 PostgreSQL 的备份和恢复管理工具。

### 安装

```bash
sudo apt-get install barman
```

### 配置

```ini
# /etc/barman.conf
[barman]
barman_home = /var/lib/barman
configuration_files_directory = /etc/barman.d
barman_user = barman

[mydb]
ssh_command = ssh postgres@localhost
conninfo = host=localhost user=postgres dbname=mydb
backup_method = rsync
parallel_backups = 1
retention_policy = REDUNDANCY 3
wal_level = replica
```

### 基本命令

```bash
# 列出备份
barman list-backups mydb

# 执行备份
barman backup mydb

# 查看备份状态
barman check mydb

# 恢复
barman recover mydb latest /var/lib/postgresql/data
```

## 备份策略

### 推荐策略

```bash
# 每日全量 + WAL 归档
# 备份脚本 - daily_backup.sh

#!/bin/bash
DATE=$(date +%Y%m%d)

# 1. 执行 pg_basebackup
pg_basebackup -h localhost -U postgres -D /backup/base_$DATE -Ft -z -P -Xs

# 2. 清理旧备份（保留 7 天）
find /backup/base_* -type d -mtime +7 -exec rm -rf {} \;

# 3. WAL 归档自动执行
```

### 3-2-1 原则

1. **3 份数据副本**
2. **2 种不同存储介质**
3. **1 份异地备份**

## 总结

PostgreSQL 备份恢复的核心：

1. **pg_basebackup**：物理全量备份
2. **pg_dump**：逻辑备份
3. **WAL 归档**：增量备份
4. **PITR**：任意时间点恢复
5. **Barman**：备份管理工具

完善的备份策略 + 定期演练 = 数据安全。
