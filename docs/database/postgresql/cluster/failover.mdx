# PostgreSQL 主从切换

主从切换（Failover）是指当主节点故障时，自动或手动将某个从节点提升为新的主节点的过程。

## 切换类型

### 手动切换（Switchover）

计划内的主从切换，用于维护升级等场景。

```bash
# 1. 停止主节点写入
pg_ctl stop -D /var/lib/postgresql/data

# 2. 在从节点执行提升
pg_ctl promote -D /var/lib/postgresql/data

# 3. 更新连接字符串
```

### 自动切换（Failover）

主节点故障时自动触发，需要配合高可用工具。

---

## pgpool-II 自动切换

```ini
# pgpool.conf
failover_command = '/etc/pgpool/failover.sh %d %H %h %m %M %p'

# 自动检测和切换
use_watchdog = on
```

### 故障检测脚本

```bash
#!/bin/bash
# /etc/pgpool/failover.sh

FALLING_NODE=$1
NEW_MASTER=$3
OLD_MASTER=$5
NEW_PRIMARY_PORT=$6

echo "Failover triggered: $FALLING_NODE -> $NEW_MASTER" | tee -a /var/log/pgpool/failover.log

# 更新 DNS 或 VIP
```

## Patroni 自动切换

Patroni 是常用的 PostgreSQL 高可用解决方案。

### 配置示例

```yaml
# patroni.yml
scope: postgres-cluster
name: node1

restapi:
  listen: 0.0.0.0:8008
  connect_address: 192.168.1.1:8008

etcd:
  host: 192.168.1.10:2379

postgresql:
  listen: 0.0.0.0:5432
  connect_address: 192.168.1.1:5432
  data_dir: /var/lib/postgresql/data
  parameters:
    wal_level: replica
    max_wal_senders: 10

  replication:
    username: replicator
    password: your_password
    slots: yes

  callbacks:
    on_start: /etc/patroni/callbacks/on_start.sh
    on_stop: /etc/patroni/callbacks/on_stop.sh
    on_failover: /etc/patroni/callbacks/on_failover.sh
```

### 启动 Patroni

```bash
patroni patroni.yml
```

## 故障恢复

### 使用 pg_rewind

主节点故障后恢复时，可以使用 pg_rewind 快速同步。

```bash
# 1. 停止旧主节点
pg_ctl stop -D /var/lib/postgresql/data

# 2. 使用 pg_rewind 同步
pg_rewind --target-pgdata=/var/lib/postgresql/data \
          --source-server='host=new_master port=5432 user=postgres'

# 3. 启动节点
pg_ctl start -D /var/lib/postgresql/data

# 4. 设置为从节点
pg_ctl promote -D /var/lib/postgresql/data
```

## 注意事项

### VIP 漂移

故障切换时需要漂移 VIP：

```bash
#!/bin/bash
# /usr/local/bin/vip-manager.sh

VIP=192.168.1.100
INTERFACE=eth0

if [ "$1" = "start" ]; then
    ip addr add $VIP/24 dev $INTERFACE
    ip link set $INTERFACE up
elif [ "$1" = "stop" ]; then
    ip addr del $VIP/24 dev $INTERFACE
fi
```

### 连接重试

应用层需要配置重试机制：

```python
import psycopg2
import time

def execute_with_retry(sql, max_retries=3):
    for i in range(max_retries):
        try:
            conn = psycopg2.connect(connection_string)
            cur = conn.cursor()
            cur.execute(sql)
            return cur.fetchall()
        except psycopg2.OperationalError as e:
            if i == max_retries - 1:
                raise
            time.sleep(2 ** i)
```

## 总结

主从切换是 PostgreSQL 高可用的核心：

1. **手动切换**：用于计划维护
2. **自动切换**：需要高可用工具（Patroni、pgpool-II）
3. **故障恢复**：使用 pg_rewind 快速同步
4. **VIP 漂移**：确保连接可达
5. **应用重试**：容忍短暂的切换时间

配置可靠的主从切换是高可用部署的关键。
