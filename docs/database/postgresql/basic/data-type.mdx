# PostgreSQL 常用数据类型与高级类型

很多从 MySQL 迁移到 PostgreSQL 的开发者，第一反应是"数据类型怎么这么丰富？"。

没错，PostgreSQL 的类型系统是其核心优势之一。从简单的整数到复杂的几何类型，从传统的字符串到现代的 JSONB，从标量值到数组，PostgreSQL 几乎覆盖了所有你可能用到的数据类型。

## 数值类型

### 整数类型

PostgreSQL 的整数类型比 MySQL 简洁，只有三种：

| 类型 | 存储大小 | 范围 |
|---|---|---|
| `SMALLINT` (或 `INT2`) | 2 字节 | -32768 ~ 32767 |
| `INTEGER` (或 `INT`, `INT4`) | 4 字节 | -2147483648 ~ 2147483647 |
| `BIGINT` (或 `INT8`) | 8 字节 | -9223372036854775808 ~ 9223372036854775807 |

MySQL 有 TINYINT、SMALLINT、MEDIUMINT、INT、BIGINT 五种，PostgreSQL 统一为三种，语义更清晰。

### 浮点类型

| 类型 | 存储大小 | 说明 |
|---|---|---|
| `REAL` | 4 字节 | 单精度浮点，约 6 位十进制精度 |
| `DOUBLE PRECISION` | 8 字节 | 双精度浮点，约 15 位十进制精度 |

```sql
CREATE TABLE physics (
    x REAL,
    y DOUBLE PRECISION,
    z DOUBLE PRECISION
);
```

### 精确数值类型

对于需要精确计算的场景（如金融、货币），使用 `NUMERIC` 类型：

```sql
CREATE TABLE accounts (
    id SERIAL PRIMARY KEY,
    balance NUMERIC(15, 2)  -- 总共 15 位，其中 2 位小数
);

-- NUMERIC 的运算是精确的，不会丢失精度
SELECT 0.1::NUMERIC + 0.2::NUMERIC;  -- 结果是 0.3
```

:::warning
NUMERIC 的性能比 FLOAT/DOUBLE 差很多。只在必须精确计算的场景使用。
:::

### 序列（Serial）

PostgreSQL 没有 MySQL 的 AUTO_INCREMENT，而是使用序列：

```sql
-- 这两种写法等价
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100)
);

CREATE TABLE users (
    id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    name VARCHAR(100)
);

-- 插入数据时，id 会自动递增
INSERT INTO users (name) VALUES ('张三');
INSERT INTO users (name) VALUES ('李四');
```

## 字符串类型

### 基本类型

| 类型 | 说明 |
|---|---|
| `CHAR(n)` | 定长字符串，不足部分用空格填充 |
| `VARCHAR(n)` | 变长字符串，最大 n 字符 |
| `TEXT` | 变长字符串，无长度限制 |

```sql
CREATE TABLE articles (
    title CHAR(50),      -- 定长，查询时可能更快
    summary VARCHAR(200),
    content TEXT         -- 无长度限制，适合长文本
);
```

:::tip
PostgreSQL 的 VARCHAR(n) 和 TEXT 本质上是同一种存储方式（varlena），区别只是 VARCHAR 有长度检查。如果你不需要限制长度，直接用 TEXT。
:::

### 常用函数

```sql
-- 字符串拼接
SELECT 'Hello' || ' ' || 'World';  -- Hello World

-- 大小写转换
SELECT UPPER('hello');  -- HELLO
SELECT LOWER('HELLO');  -- hello

-- 子字符串
SELECT SUBSTRING('Hello World', 1, 5);  -- Hello
SELECT SUBSTRING('Hello World' FROM 7);  -- World

-- 字符串替换
SELECT REPLACE('Hello World', 'World', 'PostgreSQL');  -- Hello PostgreSQL

-- 去除空格
SELECT TRIM('  hello  ');   -- hello（两端）
SELECT LTRIM('  hello');    -- hello（左侧）
SELECT RTRIM('hello  ');    -- hello（右侧）

-- 字符串长度
SELECT LENGTH('Hello');     -- 5（字符数）
SELECT CHAR_LENGTH('你好'); -- 2（字符数）
SELECT OCTET_LENGTH('你好');-- 6（字节数）
```

## 日期时间类型

PostgreSQL 的日期时间类型非常丰富，特别是对时区的原生支持。

### 基本类型

| 类型 | 说明 |
|---|---|
| `DATE` | 日期，不含时间 |
| `TIME` | 时间，不含日期 |
| `TIMESTAMP` | 日期时间，不带时区 |
| `TIMESTAMPTZ` | 日期时间，带时区 |

```sql
CREATE TABLE events (
    event_date DATE,
    event_time TIME,
    created_at TIMESTAMP,
    updated_at TIMESTAMPTZ
);
```

### TIMESTAMPTZ 的优势

TIMESTAMPTZ 是 PostgreSQL 的一大优势——它存储 UTC 时间戳，但在显示时自动转换为客户端时区。

```sql
-- 设置会话时区
SET TIME ZONE 'Asia/Shanghai';

-- 插入 UTC 时间戳
INSERT INTO events (created_at) VALUES ('2024-01-15 10:00:00+00');

-- 查询时会自动转换为北京时间
SELECT created_at FROM events;
-- 结果：2024-01-15 18:00:00+08
```

```sql
-- 设置为美国东部时间
SET TIME ZONE 'America/New_York';

-- 同一个时间，显示为美东时间
SELECT created_at FROM events;
-- 结果：2024-01-15 07:00:00-05
```

### 常用函数

```sql
-- 当前时间
SELECT NOW();                  -- 2024-01-15 18:00:00+08
SELECT CURRENT_TIMESTAMP;       -- 同上
SELECT CURRENT_DATE;            -- 2024-01-15
SELECT CURRENT_TIME;            -- 18:00:00+08

-- 日期计算
SELECT NOW() + INTERVAL '1 day';      -- 明天
SELECT NOW() - INTERVAL '1 week';      -- 一周前
SELECT NOW() - INTERVAL '2 hours 30 minutes';  -- 2.5 小时前

-- 提取日期部分
SELECT EXTRACT(YEAR FROM NOW());       -- 2024
SELECT EXTRACT(MONTH FROM NOW());      -- 1
SELECT EXTRACT(DAY FROM NOW());         -- 15
SELECT EXTRACT(DOW FROM NOW());         -- 1（周日=0，周一=1）

-- 日期格式化
SELECT TO_CHAR(NOW(), 'YYYY-MM-DD HH24:MI:SS');
-- 结果：2024-01-15 18:00:00

SELECT TO_CHAR(NOW(), 'YYYY年MM月DD日');
-- 结果：2024年01月15日

-- 字符串转日期
SELECT '2024-01-15'::DATE;
SELECT TO_DATE('20240115', 'YYYYMMDD');
```

### 时区转换

```sql
-- 将时间戳转换为特定时区
SELECT created_at AT TIME ZONE 'UTC' FROM events;
SELECT created_at AT TIME ZONE 'America/Los_Angeles' FROM events;

-- 时间戳与时区转换
SELECT NOW()::TIMESTAMP;              -- 去除时区信息
SELECT NOW()::TIMESTAMPTZ;           -- 添加时区信息
```

## 布尔类型

PostgreSQL 的布尔类型非常直观：

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    is_active BOOLEAN DEFAULT TRUE,
    is_admin BOOLEAN DEFAULT FALSE
);

-- 插入
INSERT INTO users (name, is_active) VALUES ('张三', TRUE);

-- 查询
SELECT * FROM users WHERE is_active = TRUE;
SELECT * FROM users WHERE is_active;  -- 简写

-- 布尔表达式
SELECT
    name,
    CASE WHEN is_admin THEN '管理员' ELSE '普通用户' END as role
FROM users;
```

## UUID 类型

UUID 是分布式系统的理想选择，可以避免多节点环境下的 ID 冲突。

```sql
CREATE TABLE distributed_tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_name VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 插入
INSERT INTO distributed_tasks (task_name) VALUES ('Task A');

-- 查询
SELECT * FROM distributed_tasks WHERE id = 'a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11';

-- 也可以使用 uuid-ossp 扩展生成更规范的 UUID
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
INSERT INTO distributed_tasks (task_name) VALUES ('Task B');
SELECT uuid_generate_v4();
```

## 数组类型

PostgreSQL 原生支持数组类型，这在与 MySQL 对比时是一个巨大优势。

### 创建数组类型

```sql
CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    skills TEXT[],          -- 技能数组
    phone_numbers VARCHAR(20)[]
);
```

### 数组操作

```sql
-- 插入数组
INSERT INTO employees (name, skills)
VALUES ('张三', ARRAY['Java', 'Python', 'PostgreSQL']);

INSERT INTO employees (name, skills)
VALUES ('李四', ARRAY['Go', 'Kubernetes']);

-- 访问数组元素（下标从 1 开始）
SELECT skills[1] FROM employees WHERE name = '张三';  -- Java

-- 数组长度
SELECT array_length(skills, 1) FROM employees;  -- 3

-- 数组包含
SELECT * FROM employees WHERE 'PostgreSQL' = ANY(skills);

-- 数组包含（另一种写法）
SELECT * FROM employees WHERE skills @> ARRAY['Java', 'PostgreSQL'];

-- 数组重叠
SELECT * FROM employees WHERE skills && ARRAY['Java'];

-- 展开数组
SELECT id, unnest(skills) as skill FROM employees;

-- 添加元素
UPDATE employees SET skills = skills || ARRAY['Docker'] WHERE name = '张三';

-- 删除元素
UPDATE employees SET skills = array_remove(skills, 'Python');
```

### 数组与索引

数组列可以创建 GIN 索引加速查询：

```sql
CREATE INDEX idx_employees_skills ON employees USING GIN (skills);

-- 这个查询会使用 GIN 索引
SELECT * FROM employees WHERE skills @> ARRAY['Java'];
```

## JSON 与 JSONB

PostgreSQL 对 JSON 的支持非常完善，提供了两种类型：JSON 和 JSONB。

### JSON vs JSONB

| 特性 | JSON | JSONB |
|---|---|---|
| 存储格式 | 文本 | 二进制 |
| 查询性能 | 每次查询需解析 | 预解析，直接查询 |
| 保留格式 | 完全保留 | 忽略空白、key 顺序 |
| 索引支持 | 有限 | 完全支持 GIN 索引 |
| 大小 | 略小 | 略大 |

:::tip
大多数场景下，推荐使用 JSONB。它的查询性能优势远超存储空间的微小损失。
:::

### JSONB 操作

```sql
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    order_data JSONB
);

-- 插入 JSONB 数据
INSERT INTO orders (order_data) VALUES
    ('{"customer": "张三", "items": [{"product": "iPhone", "qty": 1}], "total": 9999}'),
    ('{"customer": "李四", "items": [{"product": "MacBook", "qty": 2}], "total": 19998}');

-- 查询 JSONB 字段
-- -> 返回 JSON 类型
-- ->> 返回 text 类型
SELECT order_data->>'customer' as customer FROM orders;
SELECT order_data->'items'->0->>'product' as first_item FROM orders;

-- 使用路径查询
SELECT order_data#>>'{customer}' as customer FROM orders;

-- 查询特定值
SELECT * FROM orders WHERE order_data->>'customer' = '张三';

-- 查询嵌套值
SELECT * FROM orders WHERE order_data#>>'{items,0,product}' = 'iPhone';

-- JSONB 包含查询
SELECT * FROM orders WHERE order_data @> '{"customer": "张三"}';
SELECT * FROM orders WHERE order_data @> '{"items": [{"product": "iPhone"}]}';

-- JSONB 存在查询
SELECT * FROM orders WHERE order_data ? 'customer';
SELECT * FROM orders WHERE order_data ?| array['customer', 'total'];
SELECT * FROM orders WHERE order_data ?& array['customer', 'total'];

-- JSONB 数组包含
SELECT * FROM orders WHERE order_data #> '{items}' @> '[{"product": "iPhone"}]';
```

### JSONB 函数

```sql
-- 获取所有 key
SELECT jsonb_object_keys(order_data) FROM orders;

-- 获取 JSONB 大小（顶层元素数）
SELECT jsonb_size(order_data) FROM orders;

-- 删除字段
SELECT order_data - 'total' FROM orders;

-- 添加字段
SELECT order_data || '{"status": "completed"}'::jsonb FROM orders;

-- 更新字段
SELECT jsonb_set(order_data, '{total}', '10000') FROM orders;

-- 展开为多行
SELECT id, jsonb_array_elements(order_data->'items') as item FROM orders;
```

### JSONB 索引

这是 JSONB 最强大的特性——可以创建 GIN 索引加速查询：

```sql
CREATE INDEX idx_orders_data ON orders USING GIN (order_data);

-- 这个查询会使用 GIN 索引
SELECT * FROM orders WHERE order_data @> '{"customer": "张三"}';
```

也可以在特定路径上创建表达式索引：

```sql
CREATE INDEX idx_orders_customer ON orders ((order_data->>'customer'));

SELECT * FROM orders WHERE order_data->>'customer' = '张三';
```

## 范围类型

范围类型是 PostgreSQL 独有的特性，对时序数据和范围查询特别有用。

### 内置范围类型

| 类型 | 说明 |
|---|---|
| `int4range` | INTEGER 范围 |
| `int8range` | BIGINT 范围 |
| `numrange` | NUMERIC 范围 |
| `tsrange` | TIMESTAMP 范围（无时区） |
| `tstzrange` | TIMESTAMPTZ 范围（有时区） |
| `daterange` | DATE 范围 |

### 范围类型操作

```sql
CREATE TABLE reservations (
    id SERIAL PRIMARY KEY,
    room_id INT,
    time_range TSRANGE
);

-- 插入范围
INSERT INTO reservations (room_id, time_range) VALUES
    (101, '[2024-01-15 09:00, 2024-01-15 10:00)'),
    (101, '[2024-01-15 14:00, 2024-01-15 16:00)');

-- 范围包含
SELECT * FROM reservations
WHERE time_range @> '2024-01-15 09:30'::timestamp;

-- 范围重叠
SELECT * FROM reservations
WHERE time_range && '[2024-01-15 09:30, 2024-01-15 10:30)'::tsrange;

-- 排除重叠（会议室不能重复预订）
ALTER TABLE reservations ADD EXCLUDE USING GIST (
    room_id WITH =,
    time_range WITH &&
);
```

### 多范围类型

PostgreSQL 14 引入了多范围类型（multirange），可以表示多个不连续的范围：

```sql
SELECT '[2024-01-15 09:00, 2024-01-15 10:00), [2024-01-15 14:00, 2024-01-15 16:00)'::tsmultirange;
```

## 枚举类型

PostgreSQL 支持原生枚举类型：

```sql
-- 创建枚举类型
CREATE TYPE order_status AS ENUM ('pending', 'processing', 'shipped', 'delivered', 'cancelled');

-- 使用枚举类型
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    status order_status DEFAULT 'pending',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 枚举操作
INSERT INTO orders (status) VALUES ('pending');
INSERT INTO orders (status) VALUES ('processing');

SELECT * FROM orders WHERE status = 'pending';
SELECT * FROM orders WHERE status IN ('pending', 'processing');

-- 获取枚举值
SELECT enum_label, enum_range FROM pg_enum WHERE enumtypid = 'order_status'::regtype;
```

## 总结

PostgreSQL 的类型系统是其最强大的特性之一：

- **数值类型**：简洁清晰，NUMERIC 支持精确计算
- **字符串类型**：TEXT 无长度限制，VARCHAR 按需限制
- **日期时间**：TIMESTAMPTZ 原生支持时区，是 PostgreSQL 的一大优势
- **数组类型**：PostgreSQL 独有，MySQL 需要用 JSON 模拟
- **JSONB**：二进制存储，可索引，是半结构化数据的理想选择
- **范围类型**：PostgreSQL 独有，适合时序数据和范围查询
- **枚举类型**：原生支持，比字符串更规范

理解这些类型，能帮助你在实际项目中做出更好的数据类型选择，提升查询性能和数据一致性。
