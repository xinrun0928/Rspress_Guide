# PostgreSQL 全文检索

"我想在数据库里实现类似 Google 的搜索功能，支持关键词匹配、相关性排序。"

很多人会想到 Elasticsearch，但在某些场景下，PostgreSQL 内置的全文检索就能满足需求，而且不需要引入额外的服务。

---

## 全文检索基础

### 全文检索 vs LIKE 匹配

```sql
-- LIKE 匹配：简单但低效
SELECT * FROM articles WHERE title LIKE '%PostgreSQL%';

-- 全文检索：高效且支持相关性排序
SELECT * FROM articles
WHERE to_tsvector('english', title) @@ to_tsquery('english', 'PostgreSQL')
ORDER BY ts_rank(to_tsvector('english', title), to_tsquery('english', 'PostgreSQL')) DESC;
```

### 核心概念

PostgreSQL 的全文检索有三个核心概念：

| 概念 | 说明 |
|---|---|
| `tsvector` | 文本的词向量表示，预先解析和索引的文本 |
| `tsquery` | 搜索词向量，用于匹配 |
| `ts_rank` | 相关性排名函数 |

---

## 全文检索函数

### to_tsvector：文本转词向量

```sql
-- 解析文本为词向量
SELECT to_tsvector('english', 'PostgreSQL is a powerful database system');

-- 结果：
-- 'database':1 'power':3 'postgresql':2 'system':5
-- （数字表示词在原文中的位置）
```

### to_tsquery：构建搜索词

```sql
-- 简单词
SELECT to_tsquery('english', 'PostgreSQL');

-- 多个词（AND 关系）
SELECT to_tsquery('english', 'PostgreSQL & database');

-- OR 关系
SELECT to_tsquery('english', 'PostgreSQL | MySQL');

-- NOT 关系
SELECT to_tsquery('english', 'PostgreSQL & !MySQL');
```

### @@：匹配操作符

```sql
-- 匹配
SELECT to_tsvector('english', 'PostgreSQL is great') @@ to_tsquery('english', 'PostgreSQL');
-- 结果：true
```

## 基本使用

### 创建全文检索列

```sql
CREATE TABLE articles (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255),
    content TEXT,
    search_vector TSVECTOR  -- 预计算的搜索向量
);

-- 使用触发器自动更新搜索向量
CREATE OR REPLACE FUNCTION articles_search_trigger() RETURNS trigger AS $$
BEGIN
    NEW.search_vector :=
        setweight(to_tsvector('english', COALESCE(NEW.title, '')), 'A') ||
        setweight(to_tsvector('english', COALESCE(NEW.content, '')), 'B');
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER tsvector_articles_update
BEFORE INSERT OR UPDATE ON articles
FOR EACH ROW EXECUTE FUNCTION articles_search_trigger();

-- 插入数据
INSERT INTO articles (title, content) VALUES
    ('PostgreSQL Tutorial', 'Learn PostgreSQL basics...'),
    ('MySQL vs PostgreSQL', 'Compare MySQL and PostgreSQL...'),
    ('Advanced PostgreSQL', 'Deep dive into PostgreSQL internals...');

-- 查询
SELECT title, ts_rank(search_vector, query) AS rank
FROM articles, to_tsquery('english', 'PostgreSQL & tutorial') query
WHERE search_vector @@ query
ORDER BY rank DESC;
```

## 搜索向量配置

### 支持的语言

```sql
-- 支持的语言
SELECT to_tsvector('simple', 'PostgreSQL is great');       -- 简单分词
SELECT to_tsvector('english', 'PostgreSQL is great');     -- 英文
SELECT to_tsvector('chinese', 'PostgreSQL 很强大');         -- 中文
SELECT to_tsvector('japanese', 'PostgreSQLは強い');        -- 日文
SELECT to_tsvector('korean', 'PostgreSQL很强');            -- 韩文
```

### 中文分词

PostgreSQL 默认不支持中文分词，需要安装扩展：

```sql
-- 安装 pg_jieba 或 zhparser
CREATE EXTENSION pg_jieba;  -- 或
CREATE EXTENSION zhparser;

-- 配置中文分词
ALTER TEXT SEARCH CONFIGURATION chinese
    ADD MAPPING FOR zh WITH simple;
```

### 词权重

```sql
-- 给不同字段设置权重（A 最高，D 最低）
SELECT
    setweight(to_tsvector('english', title), 'A') ||
    setweight(to_tsvector('english', content), 'B') AS search_vector
FROM articles;
```

## 相关性排名

### ts_rank：基本排名

```sql
SELECT
    title,
    ts_rank(search_vector, query) AS rank
FROM articles, to_tsquery('english', 'PostgreSQL | MySQL') query
WHERE search_vector @@ query
ORDER BY rank DESC;
```

### ts_rank_cd：覆盖密度排名

```sql
SELECT
    title,
    ts_rank_cd(search_vector, query) AS rank
FROM articles, to_tsquery('english', 'PostgreSQL & database') query
WHERE search_vector @@ query
ORDER BY rank DESC;
```

### 归一化排名

```sql
-- 1.0 / (1.0 + ts_rank(...))
SELECT
    title,
    ts_rank(search_vector, query) AS rank,
    ts_rank(search_vector, query) /
        (SELECT MAX(ts_rank(search_vector, query))
         FROM articles, query
         WHERE search_vector @@ query) AS normalized_rank
FROM articles, to_tsquery('english', 'PostgreSQL') query
WHERE search_vector @@ query
ORDER BY rank DESC;
```

## 搜索提示

### 前缀搜索

```sql
-- 搜索以 "data" 开头的词
SELECT to_tsvector('english', 'database data entry') @@ to_tsquery('english', 'data:*');
```

### 短语搜索

```sql
-- 搜索 "PostgreSQL database" 作为短语
SELECT to_tsvector('english', 'PostgreSQL database system') @@ to_tsquery('english', '''PostgreSQL database''');
```

### 模糊搜索

需要 pg_jieba 的扩展支持：

```sql
-- 使用 pg_jieba 模糊搜索
CREATE EXTENSION pg_jieba;

-- 安装模糊搜索字典
SELECT jieba_dict_loading();
```

## 全文索引

### 创建 GIN 索引

```sql
-- 在搜索向量上创建 GIN 索引
CREATE INDEX idx_articles_search ON articles USING GIN(search_vector);

-- 在表达式上创建索引
CREATE INDEX idx_articles_title_search ON articles USING GIN(to_tsvector('english', title));
```

### 索引后的查询

```sql
-- 查看是否使用索引
EXPLAIN SELECT * FROM articles
WHERE search_vector @@ to_tsquery('english', 'PostgreSQL');

-- 结果应该包含：Bitmap Heap Scan on articles
--               ->  GIN Scan on articles_search
```

## 高级搜索

### 搜索多个字段

```sql
-- 搜索多个字段
SELECT *
FROM articles
WHERE to_tsvector('english', title || ' ' || content)
    @@ to_tsquery('english', 'PostgreSQL & database')
ORDER BY
    ts_rank(to_tsvector('english', title || ' ' || content),
            to_tsquery('english', 'PostgreSQL & database')) DESC;
```

### 高亮显示

```sql
-- 使用 ts_headline 高亮匹配词
SELECT
    title,
    ts_headline('english', content, query,
                 'StartSel=<b>, StopSel=</b>, MaxWords=50') AS snippet
FROM articles, to_tsquery('english', 'PostgreSQL') query
WHERE search_vector @@ query;
```

### 相似搜索

```sql
-- 使用 pg_trgm 进行相似搜索
CREATE EXTENSION pg_trgm;

CREATE INDEX idx_articles_title_trgm ON articles USING GIN(title gin_trgm_ops);

-- 搜索相似的标题
SELECT *
FROM articles
WHERE similarity(title, 'PostgestSQL') > 0.3
ORDER BY similarity(title, 'PostgestSQL') DESC;
```

## 性能优化

### 1. 预计算搜索向量

使用触发器或物化视图预计算搜索向量：

```sql
-- 物化视图
CREATE MATERIALIZED VIEW articles_search AS
SELECT
    id,
    title,
    setweight(to_tsvector('english', COALESCE(title, '')), 'A') ||
    setweight(to_tsvector('english', COALESCE(content, '')), 'B') AS search_vector
FROM articles;

CREATE UNIQUE INDEX ON articles_search(id);
CREATE INDEX ON articles_search USING GIN(search_vector);

-- 刷新
REFRESH MATERIALIZED VIEW articles_search;
```

### 2. 分区表全文检索

对于超大型表，可以按时间分区：

```sql
CREATE TABLE articles (
    id SERIAL,
    title TEXT,
    search_vector TSVECTOR
) PARTITION BY RANGE (id);

CREATE TABLE articles_2024 PARTITION OF articles
    FOR VALUES FROM (0) TO (1000000);
```

### 3. 限制搜索范围

```sql
-- 只搜索最近的数据
SELECT *
FROM articles
WHERE search_vector @@ to_tsquery('english', 'PostgreSQL')
  AND created_at > NOW() - INTERVAL '30 days'
ORDER BY created_at DESC;
```

## 全文检索 vs Elasticsearch

| 特性 | PostgreSQL 全文检索 | Elasticsearch |
|---|---|---|
| 部署复杂度 | 低（单数据库） | 高（需要独立集群） |
| 搜索能力 | 基本够用 | 强大（分布式、实时） |
| 数据量 | 亿级以下 | PB 级 |
| 延迟 | 低 | 毫秒级 |
| 中文支持 | 需要扩展 | 插件支持 |
| 适用场景 | 中小型应用 | 大型搜索系统 |

---

## 总结

PostgreSQL 的全文检索功能可以满足大多数中小型应用的搜索需求：

1. **核心概念**：tsvector、tsquery、ts_rank
2. **索引**：使用 GIN 索引加速搜索
3. **相关性**：ts_rank 函数实现相关性排序
4. **中文支持**：需要 pg_jieba 或 zhparser 扩展
5. **高级特性**：高亮、相似搜索、前缀搜索

对于大型搜索系统，Elasticsearch 是更好的选择。但如果你的数据量在亿级以下，PostgreSQL 全文检索是一个简单高效的解决方案。
