# PostgreSQL 索引

索引是数据库查询性能的核心。PostgreSQL 提供了六种索引类型，每种都有其独特的适用场景。理解这些索引类型的原理，是进行性能优化的基础。

## 内容导航

- **[索引类型详解](index-type)**：B-Tree、Hash、GIN、GiST、BRIN、Partial Index，各类型的底层实现和适用场景
- **[复合索引](composite-index)**：多列索引的设计原则和使用技巧
- **[表达式索引](expression-index)**：在索引中使用函数和表达式
- **[索引失效场景](index-invalid)**：常见导致索引无法使用的场景
- **[索引膨胀与 REINDEX](index-bloat)**：索引膨胀的原因和处理方法
- **[全文检索](fulltext)**：PostgreSQL 的全文搜索功能

## 学习路径建议

建议按以下顺序学习：

1. 先看 [索引类型详解](index-type)，理解每种索引的原理
2. 再看 [复合索引](composite-index)，掌握多列索引的设计
3. 学习 [表达式索引](expression-index)，扩展索引的使用场景
4. 最后看 [索引失效场景](index-invalid) 和 [全文检索](fulltext)
