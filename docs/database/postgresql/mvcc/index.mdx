# PostgreSQL MVCC 与事务

这一章深入探讨 PostgreSQL 最核心的机制——MVCC（Multi-Version Concurrency Control，多版本并发控制）。理解 MVCC，你才能真正理解 PostgreSQL 的并发模型、事务隔离、垃圾回收机制。

MVCC 是 PostgreSQL 区别于 MySQL 的核心特性，也是面试的高频考点。

## 内容导航

- **[MVCC 原理](mvcc)**：深入理解 PostgreSQL 的 MVCC 实现，xmin/xmax、快照、可见性判断
- **[事务隔离级别](isolation)**：四种隔离级别的实现原理，MVCC 下的异常处理
- **[VACUUM 机制](vacuum)**：MVCC 的代价、垃圾回收、膨胀控制
- **[锁机制](lock)**：表级锁、行级锁、轻量锁的工作原理
- **[事务回滚与 Savepoint](transaction)**：事务控制、savepoint、部分回滚

## 学习路径建议

建议按以下顺序学习：

1. 先看 [MVCC 原理](mvcc)，理解 PostgreSQL 如何实现多版本并发控制
2. 再看 [事务隔离级别](isolation)，理解 MVCC 如何影响隔离级别
3. 然后学习 [VACUUM](vacuum)，理解 MVCC 的代价和垃圾回收
4. 最后看 [锁机制](lock) 和 [事务回滚](transaction)
