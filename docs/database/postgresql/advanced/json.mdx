# PostgreSQL JSON/JSONB

JSON 是现代 Web 应用中最常用的数据格式之一。PostgreSQL 从 9.2 开始支持 JSON，从 9.4 开始支持 JSONB。JSONB 是 PostgreSQL 处理半结构化数据的利器。

---

## JSON vs JSONB

| 特性 | JSON | JSONB |
|---|---|---|
| 存储格式 | 文本 | 二进制 |
| 查询性能 | 每次查询需解析 | 预解析，直接查询 |
| 索引支持 | 有限 | 完全支持 GIN 索引 |
| 保留格式 | 完全保留 | 忽略空白、key 顺序 |
| 大小 | 略小 | 略大 |

:::tip
大多数场景下，推荐使用 JSONB。
:::

## JSON/JSONB 操作符

### 创建 JSON

```sql
-- 创建 JSON
SELECT '{"name": "张三", "age": 30}'::JSON;
SELECT '{"items": [1, 2, 3]}'::JSON;

-- 从表数据创建
SELECT row_to_json(employees) FROM employees WHERE id = 1;
```

### 访问 JSON 字段

```sql
-- 创建 JSONB 数据
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    data JSONB
);

INSERT INTO products (data) VALUES
    ('{"name": "iPhone", "price": 9999, "tags": ["手机", "苹果"]}'),
    ('{"name": "MacBook", "price": 19999, "tags": ["电脑", "苹果"]}');

-- -> 返回 JSON 类型
SELECT data->>'name' AS name FROM products;
SELECT data->'tags'->0 AS first_tag FROM products;

-- ->> 返回 TEXT 类型
SELECT data->>'name' AS name FROM products;

-- #> 获取嵌套路径
SELECT data#>>'{tags,0}' AS first_tag FROM products;
```

### JSONB 操作符

```sql
-- 包含
SELECT * FROM products WHERE data @> '{"name": "iPhone"}';

-- 存在
SELECT * FROM products WHERE data ? 'name';
SELECT * FROM products WHERE data ?| array['name', 'price'];

-- 包含任意
SELECT * FROM products WHERE data ?& array['name', 'price'];
```

## JSON 函数

### 聚合函数

```sql
-- 聚合为 JSON
SELECT jsonb_agg(name) FROM users;

-- 聚合为 JSON 对象
SELECT jsonb_object_agg(name, email) FROM users;
```

### 转换函数

```sql
-- JSONB 数组展开为多行
SELECT * FROM products, jsonb_array_elements_text(data->'tags');

-- 统计标签出现次数
SELECT
    tag,
    COUNT(*) AS count
FROM products,
     jsonb_array_elements_text(data->'tags') AS tag
GROUP BY tag
ORDER BY count DESC;
```

### 构造函数

```sql
-- 从记录构造 JSONB
SELECT jsonb_build_object(
    'name', name,
    'email', email,
    'department', department
) FROM users;

-- 从数组构造 JSONB
SELECT jsonb_build_array(1, 2, 'three', true);
```

## JSONB 索引

### GIN 索引

```sql
-- 创建 GIN 索引
CREATE INDEX idx_products_data ON products USING GIN (data);

-- 包含查询使用索引
SELECT * FROM products WHERE data @> '{"name": "iPhone"}';

-- 存在查询使用索引
SELECT * FROM products WHERE data ? 'price';
```

### 表达式索引

```sql
-- 在特定路径上创建索引
CREATE INDEX idx_products_price ON products ((data->>'price'));

-- 查询
SELECT * FROM products WHERE (data->>'price')::NUMERIC > 10000;
```

## 实战案例

### 动态属性

```sql
-- 产品表支持动态属性
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    attributes JSONB
);

INSERT INTO products (name, attributes) VALUES
    ('T恤', '{"color": "红色", "size": "XL", "material": "纯棉"}'),
    ('手机', '{"color": "黑色", "storage": "256GB", "ram": "8GB"}');

-- 查询所有有 color 属性的产品
SELECT * FROM products WHERE attributes ? 'color';

-- 查询 color 为红色的产品
SELECT * FROM products WHERE attributes @> '{"color": "红色"}';

-- 查询所有属性的所有值
SELECT
    key AS attribute,
    value
FROM products,
     jsonb_each_text(attributes);
```

### 事件日志

```sql
-- 事件日志表
CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    event_type VARCHAR(50),
    event_data JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 插入事件
INSERT INTO events (event_type, event_data) VALUES
    ('user_signup', '{"user_id": 1, "source": "web"}'),
    ('user_login', '{"user_id": 1, "ip": "192.168.1.1"}'),
    ('user_purchase', '{"user_id": 1, "amount": 999, "items": 3}');

-- 统计事件类型
SELECT event_type, COUNT(*) FROM events GROUP BY event_type;

-- 查询特定事件
SELECT * FROM events
WHERE event_type = 'user_purchase'
  AND (event_data->>'amount')::NUMERIC > 500;
```

## 总结

JSONB 是 PostgreSQL 处理半结构化数据的强大工具：

1. **性能**：预解析的二进制存储，支持 GIN 索引
2. **操作符**：丰富的查询和操作符
3. **索引**：支持 GIN 索引和表达式索引
4. **应用**：动态属性、事件日志、配置存储

合理使用 JSONB，可以让 PostgreSQL 同时处理结构化和半结构化数据。
