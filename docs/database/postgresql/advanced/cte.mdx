# PostgreSQL CTE 与递归查询

CTE（Common Table Expression，公用表表达式）是一种临时的命名结果集，可以在查询中多次引用。递归 CTE 更是处理树形结构和图结构数据的利器。

---

## CTE 基础

### 基本语法

```sql
-- CTE 定义
WITH cte_name AS (
    SELECT ...
)
-- 主查询
SELECT * FROM cte_name;

-- 等价于子查询，但更易读
SELECT *
FROM (
    SELECT ...
) AS cte_name;
```

### 简单示例

```sql
-- 使用 CTE 查询部门平均工资
WITH dept_avg AS (
    SELECT department, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department
)
SELECT
    e.name,
    e.department,
    e.salary,
    da.avg_salary,
    ROUND(100.0 * (e.salary - da.avg_salary) / da.avg_salary, 2) AS diff_pct
FROM employees e
JOIN dept_avg da ON e.department = da.department;
```

## 多个 CTE

```sql
-- 多个 CTE
WITH
    active_orders AS (
        SELECT customer_id, SUM(total) AS total_amount
        FROM orders
        WHERE status = 'completed'
        GROUP BY customer_id
    ),
    top_customers AS (
        SELECT customer_id
        FROM active_orders
        WHERE total_amount > 10000
    ),
    top_customer_orders AS (
        SELECT o.*
        FROM orders o
        JOIN top_customers tc ON o.customer_id = tc.customer_id
    )
SELECT
    tc.customer_id,
    ao.total_amount,
    COUNT(*) AS order_count,
    AVG(tc.total) AS avg_order_value
FROM top_customer_orders tc
JOIN active_orders ao ON tc.customer_id = ao.customer_id
GROUP BY tc.customer_id, ao.total_amount;
```

## 递归 CTE

### 语法

```sql
WITH RECURSIVE cte_name AS (
    -- 基础查询（非递归）
    SELECT ...
    UNION ALL
    -- 递归查询（引用自身）
    SELECT ...
    FROM cte_name
    WHERE ...
)
SELECT * FROM cte_name;
```

### 实战：生成数字序列

```sql
-- 生成 1-10 的数字
WITH RECURSIVE numbers AS (
    -- 基础查询
    SELECT 1 AS n
    UNION ALL
    -- 递归查询
    SELECT n + 1
    FROM numbers
    WHERE n < 10
)
SELECT * FROM numbers;
```

### 实战：斐波那契数列

```sql
WITH RECURSIVE fibonacci AS (
    -- 基础查询
    SELECT 0 AS n, 0 AS fib, 1 AS next_fib
    UNION ALL
    -- 递归查询
    SELECT n + 1, next_fib, fib + next_fib
    FROM fibonacci
    WHERE n < 10
)
SELECT n, fib FROM fibonacci;
```

## 递归 CTE 处理树形结构

### 组织架构查询

```sql
-- 组织架构表
CREATE TABLE org_structure (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    manager_id INT REFERENCES org_structure(id)
);

INSERT INTO org_structure (id, name, manager_id) VALUES
    (1, 'CEO', NULL),
    (2, 'CTO', 1),
    (3, 'CFO', 1),
    (4, 'Engineering Manager', 2),
    (5, 'Frontend Lead', 4),
    (6, 'Backend Lead', 4),
    (7, 'Senior Developer', 5),
    (8, 'Junior Developer', 5);
```

### 查询所有下属

```sql
-- 查询某个员工的所有下属（包括多层级）
WITH RECURSIVE subordinates AS (
    -- 基础查询：直接下属
    SELECT id, name, manager_id, 1 AS level
    FROM org_structure
    WHERE manager_id = 4  -- Engineering Manager

    UNION ALL

    -- 递归查询：下属的下属
    SELECT o.id, o.name, o.manager_id, s.level + 1
    FROM org_structure o
    JOIN subordinates s ON o.manager_id = s.id
)
SELECT
    id,
    name,
    level,
    REPEAT('  ', level) || name AS indented_name
FROM subordinates
ORDER BY level, name;
```

### 查询完整路径

```sql
-- 查询从根到叶子的完整路径
WITH RECURSIVE org_path AS (
    -- 基础查询：CEO（根节点）
    SELECT id, name, manager_id, name AS path
    FROM org_structure
    WHERE manager_id IS NULL

    UNION ALL

    -- 递归查询：添加父节点路径
    SELECT o.id, o.name, o.manager_id, op.path || ' > ' || o.name
    FROM org_structure o
    JOIN org_path op ON o.manager_id = op.id
)
SELECT path FROM org_path WHERE id = 8;
-- 结果：CEO > CTO > Engineering Manager > Frontend Lead > Junior Developer
```

## 递归 CTE 处理图结构

### 查询图中两点间的路径

```sql
-- 图关系表
CREATE TABLE graph (
    from_node INT,
    to_node INT,
    cost NUMERIC
);

INSERT INTO graph VALUES
    (1, 2, 10), (1, 3, 20),
    (2, 4, 30), (2, 5, 15),
    (3, 5, 10), (3, 6, 25),
    (4, 7, 5),
    (5, 7, 10), (5, 8, 20),
    (6, 8, 15);
```

```sql
-- 查询从节点 1 到节点 7 的最短路径
WITH RECURSIVE shortest_path AS (
    -- 基础查询：起点
    SELECT
        1 AS node,
        ARRAY[1] AS path,
        0 AS cost

    UNION ALL

    -- 递归查询：扩展路径
    SELECT
        g.to_node,
        sp.path || g.to_node,
        sp.cost + g.cost
    FROM shortest_path sp
    JOIN graph g ON sp.node = g.from_node
    WHERE NOT g.to_node = ANY(sp.path)  -- 防止循环
      AND sp.cost + g.cost < 100         -- 成本上限
)
SELECT
    path,
    cost
FROM shortest_path
WHERE node = 7
ORDER BY cost
LIMIT 1;
```

## CTE vs 子查询

### CTE 的优势

1. **可读性**：给子查询起有意义的名字
2. **可复用**：在同一查询中多次引用
3. **可递归**：处理树形结构和图结构
4. **可维护**：修改 CTE 名称，全局生效

```sql
-- 子查询写法
SELECT *
FROM (
    SELECT ... FROM orders WHERE ...
) AS sub1
JOIN (
    SELECT ... FROM customers WHERE ...
) AS sub2 ON sub1.customer_id = sub2.id;

-- CTE 写法
WITH
    active_orders AS (
        SELECT ... FROM orders WHERE ...
    ),
    vip_customers AS (
        SELECT ... FROM customers WHERE ...
    )
SELECT *
FROM active_orders ao
JOIN vip_customers vc ON ao.customer_id = vc.id;
```

### 性能对比

CTE 和子查询通常有相同的执行计划，但 CTE 有以下特殊场景：

1. **物化 CTE**（PostgreSQL 12+）：`MATERIALIZED` 和 `NOT MATERIALIZED` 提示
2. **CTE 作为写入目标**：可以用于 INSERT/UPDATE/DELETE

```sql
-- 物化 CTE
WITH MATERIALIZED cte AS (
    SELECT * FROM large_table
)
SELECT * FROM cte WHERE ...

-- 非物化 CTE
WITH NOT MATERIALIZED cte AS (
    SELECT * FROM another_table WHERE ...
)
SELECT * FROM cte WHERE ...
```

## 写入操作中使用 CTE

```sql
-- INSERT 使用 CTE
WITH pending_orders AS (
    SELECT id, total FROM orders WHERE status = 'pending'
)
INSERT INTO order_audit (order_id, amount, action)
SELECT id, total, 'processing'
FROM pending_orders;

-- UPDATE 使用 CTE
WITH updated AS (
    UPDATE orders SET status = 'completed'
    WHERE id = 123
    RETURNING id, total
)
UPDATE order_stats SET total = total + (SELECT total FROM updated)
WHERE date = CURRENT_DATE;

-- DELETE 使用 CTE
WITH old_orders AS (
    DELETE FROM orders
    WHERE created_at < NOW() - INTERVAL '1 year'
    RETURNING id
)
INSERT INTO order_archive (order_id, archived_at)
SELECT id, NOW() FROM old_orders;
```

## 总结

CTE 是 SQL 的重要特性：

1. **基础 CTE**：临时结果集，提高可读性
2. **多个 CTE**：逻辑清晰地组织复杂查询
3. **递归 CTE**：处理树形结构和图结构
4. **写入操作**：CTE 可以用于 INSERT/UPDATE/DELETE

递归 CTE 是处理层级数据的利器，从组织架构到地理区域，从文件目录到社交网络图，都能优雅地处理。
