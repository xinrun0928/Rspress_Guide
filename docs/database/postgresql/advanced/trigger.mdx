# PostgreSQL 触发器

触发器是在特定事件发生时自动执行的数据库逻辑。比如，每次插入订单时，自动更新订单统计表；每次删除用户时，自动记录删除日志。

---

## 触发器基础

### 创建触发器

```sql
-- 创建触发器函数
CREATE OR REPLACE FUNCTION log_orders_change()
RETURNS TRIGGER AS $$
BEGIN
    IF TG_OP = 'INSERT' THEN
        INSERT INTO orders_log (action, order_id, new_data)
        VALUES ('INSERT', NEW.id, row_to_json(NEW));
        RETURN NEW;
    ELSIF TG_OP = 'UPDATE' THEN
        INSERT INTO orders_log (action, order_id, old_data, new_data)
        VALUES ('UPDATE', NEW.id, row_to_json(OLD), row_to_json(NEW));
        RETURN NEW;
    ELSIF TG_OP = 'DELETE' THEN
        INSERT INTO orders_log (action, order_id, old_data)
        VALUES ('DELETE', OLD.id, row_to_json(OLD));
        RETURN OLD;
    END IF;
    RETURN NULL;
END;
$$ LANGUAGE plpgsql;

-- 创建触发器
CREATE TRIGGER orders_change_trigger
AFTER INSERT OR UPDATE OR DELETE ON orders
FOR EACH ROW EXECUTE FUNCTION log_orders_change();
```

### 触发器变量

| 变量 | 说明 |
|---|---|
| `NEW` | 新行（INSERT/UPDATE） |
| `OLD` | 旧行（UPDATE/DELETE） |
| `TG_OP` | 触发器操作（INSERT/UPDATE/DELETE） |
| `TG_TABLE_NAME` | 触发器所在的表名 |
| `TG_LEVEL` | 触发级别（ROW/STATEMENT） |

## 触发器类型

### 行级触发器

```sql
-- FOR EACH ROW：每行变化都触发
CREATE TRIGGER update_inventory
AFTER UPDATE ON order_items
FOR EACH ROW
EXECUTE FUNCTION update_inventory_stock();
```

### 语句级触发器

```sql
-- FOR EACH STATEMENT：每个语句触发一次
CREATE TRIGGER log_orders_delete
AFTER DELETE ON orders
FOR EACH STATEMENT
EXECUTE FUNCTION log_orders_delete_event();
```

### BEFORE 和 AFTER

```sql
-- BEFORE：在操作之前执行，可以修改数据
CREATE TRIGGER validate_order
BEFORE INSERT ON orders
FOR EACH ROW
EXECUTE FUNCTION validate_order_data();

-- AFTER：在操作之后执行，用于日志、统计
CREATE TRIGGER update_stats
AFTER INSERT ON orders
FOR EACH ROW
EXECUTE FUNCTION update_order_stats();
```

## 实战示例

### 自动更新统计

```sql
-- 订单统计表
CREATE TABLE order_stats (
    date DATE PRIMARY KEY,
    total_orders INT DEFAULT 0,
    total_amount NUMERIC DEFAULT 0
);

-- 触发器函数
CREATE OR REPLACE FUNCTION update_daily_stats()
RETURNS TRIGGER AS $$
BEGIN
    IF TG_OP = 'INSERT' THEN
        INSERT INTO order_stats (date, total_orders, total_amount)
        VALUES (NEW.created_at::DATE, 1, NEW.total)
        ON CONFLICT (date) DO UPDATE
        SET total_orders = order_stats.total_orders + 1,
            total_amount = order_stats.total_amount + NEW.total;
        RETURN NEW;
    END IF;
    RETURN NULL;
END;
$$ LANGUAGE plpgsql;

-- 创建触发器
CREATE TRIGGER order_stats_trigger
AFTER INSERT ON orders
FOR EACH ROW
EXECUTE FUNCTION update_daily_stats();
```

### 审计日志

```sql
-- 审计表
CREATE TABLE audit_log (
    id BIGSERIAL PRIMARY KEY,
    table_name VARCHAR(100),
    action VARCHAR(10),
    old_data JSONB,
    new_data JSONB,
    changed_by VARCHAR(100),
    changed_at TIMESTAMPTZ DEFAULT NOW()
);

-- 审计函数
CREATE OR REPLACE FUNCTION audit_trigger()
RETURNS TRIGGER AS $$
BEGIN
    IF TG_OP = 'INSERT' THEN
        INSERT INTO audit_log (table_name, action, new_data, changed_by)
        VALUES (TG_TABLE_NAME, 'INSERT', row_to_json(NEW), current_user);
        RETURN NEW;
    ELSIF TG_OP = 'UPDATE' THEN
        INSERT INTO audit_log (table_name, action, old_data, new_data, changed_by)
        VALUES (TG_TABLE_NAME, 'UPDATE', row_to_json(OLD), row_to_json(NEW), current_user);
        RETURN NEW;
    ELSIF TG_OP = 'DELETE' THEN
        INSERT INTO audit_log (table_name, action, old_data, changed_by)
        VALUES (TG_TABLE_NAME, 'DELETE', row_to_json(OLD), current_user);
        RETURN OLD;
    END IF;
    RETURN NULL;
END;
$$ LANGUAGE plpgsql;

-- 为表创建审计触发器
CREATE TRIGGER customers_audit
AFTER INSERT OR UPDATE OR DELETE ON customers
FOR EACH ROW EXECUTE FUNCTION audit_trigger();
```

## 触发器管理

### 查看触发器

```sql
-- 查看表的触发器
SELECT
    trigger_name,
    event_manipulation,
    action_statement,
    action_timing,
    action_reference_old_table,
    action_reference_new_table
FROM information_schema.triggers
WHERE event_table_name = 'orders';
```

### 删除触发器

```sql
DROP TRIGGER IF EXISTS orders_change_trigger ON orders;
```

### 禁用/启用触发器

```sql
-- 禁用触发器
ALTER TABLE orders DISABLE TRIGGER orders_change_trigger;

-- 启用触发器
ALTER TABLE orders ENABLE TRIGGER orders_change_trigger;

-- 禁用所有触发器
ALTER TABLE orders DISABLE TRIGGER ALL;

-- 启用所有触发器
ALTER TABLE orders ENABLE TRIGGER ALL;
```

## 触发器注意事项

### 性能影响

- 触发器会增加每次 DML 操作的开销
- 复杂触发器可能导致性能问题
- 考虑使用规则（Rule）代替触发器

### 递归触发器

```sql
-- 禁用递归触发器
ALTER TABLE orders DISABLE TRIGGER REPLICA TRIGGER_NAME;

-- 配置最大触发器嵌套深度
SHOW max_trigger_depth;
```

---

## 总结

触发器是 PostgreSQL 的重要特性：

1. **类型**：BEFORE/AFTER、ROW/STATEMENT
2. **变量**：NEW、OLD、TG_OP
3. **用途**：审计日志、自动统计、数据验证
4. **注意**：性能影响、递归触发

合理使用触发器可以减少应用层逻辑，但也要注意不要过度使用。
