# PostgreSQL 函数重载

函数重载是指同一个函数名可以有多个不同的实现，根据参数类型或参数个数选择执行哪个实现。

---

## 函数重载基础

### 创建重载函数

```sql
-- 根据不同参数类型重载
CREATE OR REPLACE FUNCTION calculate_total(
    p_price NUMERIC,
    p_quantity INT
)
RETURNS NUMERIC AS $$
BEGIN
    RETURN p_price * p_quantity;
END;
$$ LANGUAGE plpgsql;

CREATE OR REPLACE FUNCTION calculate_total(
    p_items JSONB
)
RETURNS NUMERIC AS $$
DECLARE
    total NUMERIC := 0;
    item JSONB;
BEGIN
    FOR item IN SELECT * FROM jsonb_array_elements(p_items)
    LOOP
        total := total + (item->>'price')::NUMERIC * (item->>'quantity')::INT;
    END LOOP;
    RETURN total;
END;
$$ LANGUAGE plpgsql;

-- 调用时会根据参数类型选择
SELECT calculate_total(99.99, 2);        -- 调用第一个
SELECT calculate_total('[{"price": 99.99, "quantity": 2}]');  -- 调用第二个
```

### 参数个数重载

```sql
-- 三个参数版本
CREATE OR REPLACE FUNCTION make_date(
    p_year INT,
    p_month INT,
    p_day INT
)
RETURNS DATE AS $$
BEGIN
    RETURN make_date(p_year, p_month, p_day);
END;
$$ LANGUAGE plpgsql;

-- 两个参数版本（月和日默认为 1）
CREATE OR REPLACE FUNCTION make_date(
    p_year INT,
    p_day_of_year INT
)
RETURNS DATE AS $$
BEGIN
    RETURN make_date(p_year, 1, 1) + (p_day_of_year - 1);
END;
$$ LANGUAGE plpgsql;
```

## 默认参数

### 使用默认值

```sql
-- 使用 DEFAULT 关键字
CREATE OR REPLACE FUNCTION create_user(
    p_name VARCHAR,
    p_email VARCHAR,
    p_status VARCHAR DEFAULT 'active',
    p_role VARCHAR DEFAULT 'user'
)
RETURNS BIGINT AS $$
DECLARE
    v_user_id BIGINT;
BEGIN
    INSERT INTO users (name, email, status, role)
    VALUES (p_name, p_email, p_status, p_role)
    RETURNING id INTO v_user_id;

    RETURN v_user_id;
END;
$$ LANGUAGE plpgsql;

-- 调用
SELECT create_user('张三', 'zhang@example.com');
SELECT create_user('李四', 'li@example.com', 'inactive');
SELECT create_user('王五', 'wang@example.com', 'inactive', 'admin');
```

## 函数重载的限制

### 不能重载的情况

```sql
-- 1. 参数名称不同但类型相同
-- 错误：无法区分
CREATE FUNCTION foo(a INT) ...
CREATE FUNCTION foo(b INT) ...

-- 2. 仅返回类型不同
-- 错误：调用时无法确定
CREATE FUNCTION foo(INT) RETURNS INT ...
CREATE FUNCTION foo(INT) RETURNS TEXT ...

-- 3. 参数的默认参数顺序
-- 错误：可能产生歧义
CREATE FUNCTION foo(a INT, b INT DEFAULT 10) ...
CREATE FUNCTION foo(a INT, b TEXT) ...
```

## 变长参数

### VARIADIC 关键字

```sql
-- 变长参数函数
CREATE OR REPLACE FUNCTION sum_values(
    VARIADIC arr INT[]
)
RETURNS INT AS $$
DECLARE
    total INT := 0;
BEGIN
    FOREACH val IN ARRAY arr
    LOOP
        total := total + val;
    END LOOP;
    RETURN total;
END;
$$ LANGUAGE plpgsql;

-- 调用
SELECT sum_values(1, 2, 3);              -- 6
SELECT sum_values(1, 2, 3, 4, 5);        -- 15
SELECT sum_values(VARIADIC ARRAY[1,2,3]); -- 6
```

## 实战示例

### 格式化函数

```sql
-- 格式化整数
CREATE OR REPLACE FUNCTION format_number(
    p_value BIGINT,
    p_separator VARCHAR DEFAULT ','
)
RETURNS TEXT AS $$
BEGIN
    RETURN REPLACE(FORMAT('%s', p_value), ',', p_separator);
END;
$$ LANGUAGE plpgsql;

-- 格式化浮点数
CREATE OR REPLACE FUNCTION format_number(
    p_value NUMERIC,
    p_decimals INT DEFAULT 2,
    p_separator VARCHAR DEFAULT ','
)
RETURNS TEXT AS $$
BEGIN
    RETURN REPLACE(FORMAT('%.' || p_decimals || 'f', p_value), ',', p_separator);
END;
$$ LANGUAGE plpgsql;

-- 调用
SELECT format_number(1234567);              -- 1234567
SELECT format_number(1234567.89);           -- 1234567.89
SELECT format_number(1234567.891, 1);       -- 1234567.9
```

## 查看重载函数

```sql
-- 查看函数定义
SELECT
    p.proname,
    pg_get_functiondef(p.oid) AS definition
FROM pg_proc p
JOIN pg_namespace n ON p.pronamespace = n.oid
WHERE n.nspname = 'public'
  AND p.proname = 'format_number';
```

## 总结

函数重载是 PostgreSQL 的重要特性：

1. **重载类型**：根据参数类型或参数个数
2. **默认参数**：使用 DEFAULT 关键字
3. **变长参数**：使用 VARIADIC 关键字
4. **限制**：参数名不同但类型相同不可重载

合理使用函数重载，可以让 API 更简洁、更易用。
