# PostgreSQL PostGIS：空间数据库扩展深度解析

地理位置数据无处不在。

外卖配送需要计算骑手到商家的距离，共享单车需要判断哪些车在附近，物流系统需要规划最优配送路线……这些场景的核心都是**空间计算**。

传统方案是把经纬度存成两个浮点数，查询时用 Haversine 公式计算距离。但当数据量达到百万级别，每次查询都要遍历全表计算距离——性能无法接受。

**PostGIS** 让 PostgreSQL 成为专业的地理信息系统，提供了高效的空间数据存储、索引和计算能力。

## PostGIS 概述

### 什么是 PostGIS

PostGIS 是 PostgreSQL 的空间数据库扩展，提供了：

| 功能 | 说明 |
|---|---|
| 空间数据类型 | Point、LineString、Polygon、MultiPoint 等 |
| 空间函数 | 距离计算、面积计算、缓冲区分析等 |
| 空间索引 | R-Tree（GiST）索引，加速空间查询 |
| 坐标系统 | 支持 WGS84、北京 54、西安 80 等坐标系 |
| 空间关系判断 | ST_Contains、ST_Intersects、ST_DWithin 等 |

## 空间数据类型

### 基础几何类型

```sql
-- Point：点
SELECT ST_GeomFromText('POINT(116.4074 39.9042)');
-- 格式：POINT(经度 纬度) 或 POINT(x y)

-- LineString：线
SELECT ST_GeomFromText('LINESTRING(116.4074 39.9042, 116.4274 39.9142, 116.4474 39.9242)');

-- Polygon：多边形
SELECT ST_GeomFromText('POLYGON((116.40 39.90, 116.41 39.90, 116.41 39.91, 116.40 39.91, 116.40 39.90))');

-- MultiPoint：多点
SELECT ST_GeomFromText('MULTIPOINT(116.40 39.90, 116.41 39.91, 116.42 39.92)');

-- MultiPolygon：多边形集合
SELECT ST_GeomFromText('MULTIPOLYGON(((0 0, 1 0, 1 1, 0 1, 0 0)), ((2 2, 3 2, 3 3, 2 3, 2 2)))');
```

### 创建空间表

```sql
-- 创建门店表
CREATE TABLE stores (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    location GEOGRAPHY(POINT, 4326),  -- POINT 类型，WGS84 坐标系
    address VARCHAR(200),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 创建配送区域表
CREATE TABLE delivery_zones (
    id SERIAL PRIMARY KEY,
    zone_name VARCHAR(100),
    boundary GEOGRAPHY(POLYGON, 4326),  -- POLYGON 类型
    zone_type VARCHAR(50)
);

-- 创建空间索引
CREATE INDEX idx_stores_location ON stores USING GIST (location);
CREATE INDEX idx_delivery_zones_boundary ON delivery_zones USING GIST (boundary);

-- 插入数据
INSERT INTO stores (name, location, address) VALUES
('三里屯店', ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326), '北京市朝阳区三里屯'),
('望京店', ST_SetSRID(ST_MakePoint(116.4695, 39.9986), 4326), '北京市朝阳区望京');

INSERT INTO delivery_zones (zone_name, boundary, zone_type) VALUES
('朝阳区北部', ST_SetSRID(ST_GeomFromText('POLYGON((116.40 39.90, 116.50 39.90, 116.50 40.00, 116.40 40.00, 116.40 39.90))'), 4326), 'standard');
```

## 空间关系函数

### 常用空间关系

```sql
-- ST_Contains：包含（A 完全在 B 内部）
SELECT ST_Contains(
    ST_SetSRID(ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))'), 4326),
    ST_SetSRID(ST_GeomFromText('POINT(5 5)'), 4326)
);  -- 返回 true

-- ST_Intersects：相交（两个几何对象有交集）
SELECT ST_Intersects(
    ST_SetSRID(ST_GeomFromText('LINESTRING(0 0, 10 10)'), 4326),
    ST_SetSRID(ST_GeomFromText('LINESTRING(0 10, 10 0)'), 4326)
);  -- 返回 true

-- ST_DWithin：在指定距离内
SELECT ST_DWithin(
    location,
    ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326),
    1000  -- 1000 米
)
FROM stores
WHERE name = '三里屯店';

-- ST_Equals：相等（几何形状相同）
SELECT ST_Equals(
    ST_SetSRID(ST_GeomFromText('POINT(1 1)'), 4326),
    ST_SetSRID(ST_GeomFromText('POINT(1 1)'), 4326)
);  -- 返回 true
```

### 距离计算

#### ST_Distance：距离计算

```sql
-- 计算两点之间的距离（米）
SELECT ST_Distance(
    ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326)::GEOGRAPHY,
    ST_SetSRID(ST_MakePoint(116.4695, 39.9986), 4326)::GEOGRAPHY
);
-- 返回约 6900 米

-- 使用 geography 类型直接计算
SELECT ST_Distance(
    'POINT(116.4536 39.9385)'::GEOGRAPHY,
    'POINT(116.4695 39.9986)'::GEOGRAPHY
);
```

#### ST_Distance_Sphere：球面距离（简化版）

```sql
-- 近似计算，使用球面模型
SELECT ST_Distance_Sphere(
    ST_MakePoint(116.4536, 39.9385),
    ST_MakePoint(116.4695, 39.9986)
);
-- 返回约 6898 米

-- Haversine 公式手写实现
SELECT 6371 * 2 * ASIN(SQRT(
    POWER(SIN((RADIANS(39.9986) - RADIANS(39.9385)) / 2), 2) +
    COS(RADIANS(39.9385)) * COS(RADIANS(39.9986)) *
    POWER(SIN((RADIANS(116.4695) - RADIANS(116.4536)) / 2), 2)
)) AS distance_km;
```

### 实际应用：查找附近的门店

```sql
-- 问题：查找距离用户位置 3 公里内的所有门店

-- 用户当前位置
DECLARE user_location GEOGRAPHY := ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326);

-- 查询附近门店（使用 ST_DWithin，利用空间索引）
SELECT 
    s.id,
    s.name,
    s.address,
    ROUND(ST_Distance(s.location, user_location)) as distance_m
FROM stores s
WHERE ST_DWithin(s.location, user_location, 3000)  -- 先用索引过滤
ORDER BY ST_Distance(s.location, user_location);

-- 使用 ST_DWithin + ST_Distance 的组合
SELECT 
    s.id,
    s.name,
    s.address,
    ST_Distance(s.location, user_location) as distance_m
FROM stores s
WHERE ST_DWithin(s.location, user_location, 3000)
ORDER BY distance_m
LIMIT 10;
```

## 空间计算函数

### ST_Length：长度计算

```sql
-- 计算线段长度（米）
SELECT ST_Length(
    'LINESTRING(116.4074 39.9042, 116.4274 39.9142)'::GEOGRAPHY
);
-- 返回约 2400 米
```

### ST_Area：面积计算

```sql
-- 计算多边形面积（平方米）
SELECT ST_Area(
    'POLYGON((116.40 39.90, 116.50 39.90, 116.50 39.95, 116.40 39.95, 116.40 39.90))'::GEOGRAPHY
);
-- 返回约 55550000 平方米

-- 计算面积（平方公里）
SELECT ST_Area(
    'POLYGON((116.40 39.90, 116.50 39.90, 116.50 39.95, 116.40 39.95, 116.40 39.90))'::GEOGRAPHY
) / 1000000 as area_km2;
```

### ST_Buffer：缓冲区分析

```sql
-- 创建距离某点 1 公里范围内的缓冲区
SELECT ST_AsText(
    ST_Buffer(
        ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326)::GEOGRAPHY,
        1000  -- 1000 米
    )
);

-- 使用缓冲区查找范围内的门店
SELECT s.name
FROM stores s, (
    SELECT ST_Buffer(
        ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326)::GEOGRAPHY,
        1000
    ) as buffer
) center
WHERE ST_Contains(center.buffer, s.location);
```

### ST_Centroid：质心计算

```sql
-- 计算多边形的质心
SELECT ST_AsText(
    ST_Centroid(
        'POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))'::GEOGRAPHY
    )
);
-- 返回 POINT(5 5)
```

## 空间索引与性能优化

### GiST 索引原理

PostGIS 使用 **GiST（Generalized Search Tree）** 索引来加速空间查询：

```sql
-- 查看空间索引
\d stores

-- 索引信息
/*
Indexes:
    "idx_stores_location" gist (location)
*/

-- 强制使用空间索引
SET enable_seqscan = off;
EXPLAIN SELECT * FROM stores WHERE ST_DWithin(location, ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326), 5000);
```

### 索引生效的关键

```sql
-- 1. 使用 geography 类型（自动计算投影，精确米制）
ALTER TABLE stores ALTER COLUMN location TYPE GEOGRAPHY(POINT, 4326);

-- 2. 查询条件中引用索引列
-- 正确写法：ST_DWithin(location, point, distance)
SELECT * FROM stores WHERE ST_DWithin(location, ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326), 5000);

-- 错误写法：先转换列，无法使用索引
SELECT * FROM stores WHERE ST_DWithin(location::GEOMETRY, ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326), 5000);
```

### 性能对比

```sql
-- 无空间索引（全表扫描）
EXPLAIN SELECT * FROM stores 
WHERE ST_Distance(location, ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326)) < 5000;

-- 有空间索引
EXPLAIN SELECT * FROM stores 
WHERE ST_DWithin(location, ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326), 5000);

-- 对比结果
-- 无索引：Seq Scan on stores  (cost=0.00..100.00 rows=10 width=200)
-- 有索引：Index Scan using idx_stores_location on stores
--          Index Cond: (location && ST_Expand(...))
```

## 实战案例

### 案例一：配送范围查询

```sql
-- 创建门店和配送范围表
CREATE TABLE restaurants (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    location GEOGRAPHY(POINT, 4326)
);

CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    address VARCHAR(200),
    location GEOGRAPHY(POINT, 4326)
);

CREATE INDEX idx_restaurants_location ON restaurants USING GIST (location);
CREATE INDEX idx_customers_location ON customers USING GIST (location);

-- 查询某门店 3 公里内的所有顾客
SELECT 
    r.name as restaurant,
    c.name as customer,
    c.address,
    ROUND(ST_Distance(r.location, c.location)) as distance_m
FROM restaurants r, customers c
WHERE r.id = 1
  AND ST_DWithin(r.location, c.location, 3000)
ORDER BY distance_m;

-- 批量计算每个门店的配送覆盖顾客数
SELECT 
    r.id,
    r.name,
    COUNT(c.id) as customer_count
FROM restaurants r
LEFT JOIN customers c ON ST_DWithin(r.location, c.location, 3000)
GROUP BY r.id, r.name
ORDER BY customer_count DESC;
```

### 案例二：地理围栏判断

```sql
-- 判断用户是否在某个配送区域内
CREATE OR REPLACE FUNCTION is_in_delivery_zone(
    user_point GEOGRAPHY,
    zone_id INT
) RETURNS BOOLEAN AS $$
DECLARE
    zone_boundary GEOGRAPHY;
BEGIN
    SELECT boundary INTO zone_boundary
    FROM delivery_zones
    WHERE id = zone_id;
    
    RETURN ST_Contains(zone_boundary, user_point);
END;
$$ LANGUAGE plpgsql;

-- 使用函数判断
SELECT 
    c.name,
    c.address,
    is_in_delivery_zone(c.location, 1) as in_zone
FROM customers c
WHERE c.id = 1;
```

### 案例三：路径规划

```sql
-- 模拟路径：经过多个门店的最短路径
WITH门店 AS (
    SELECT location FROM stores WHERE id IN (1, 2, 3)
)
SELECT 
    ST_MakeLine(array_agg(location ORDER BY location)) as route
FROM 门店;

-- 计算路径总长度
WITH门店 AS (
    SELECT location FROM stores WHERE id IN (1, 2, 3)
)
SELECT 
    ST_Length(ST_MakeLine(array_agg(location ORDER BY location))) as total_length_m
FROM 门店;
```

### 案例四：热力图数据生成

```sql
-- 生成网格热力图数据
CREATE TABLE heatmap_data AS
WITH RECURSIVE grid AS (
    -- 生成网格点
    SELECT 
        ST_SetSRID(ST_MakePoint(116.40 + x * 0.01, 39.90 + y * 0.01), 4326) as point,
        x,
        y
    FROM generate_series(0, 10) as x,
         generate_series(0, 10) as y
)
SELECT 
    point,
    x,
    y,
    (SELECT COUNT(*) FROM stores s 
     WHERE ST_DWithin(s.location, grid.point, 5000)) as store_count
FROM grid;

-- 输出热力图数据（可被前端地图库使用）
SELECT json_build_object(
    'type', 'FeatureCollection',
    'features', json_agg(
        json_build_object(
            'type', 'Feature',
            'geometry', ST_AsGeoJSON(point)::JSON,
            'properties', json_build_object('count', store_count)
        )
    )
) as geojson
FROM heatmap_data;
```

## 坐标系转换

### 坐标系说明

| 坐标系 | SRID | 说明 |
|---|---|---|
| WGS84 | 4326 | GPS 标准，最常用 |
| Web Mercator | 3857 | 地图渲染常用 |
| 北京 54 | 4214 | 中国老坐标系 |
| 西安 80 | 4610 | 中国新坐标系 |

### 坐标系转换

```sql
-- WGS84 转 Web Mercator
SELECT ST_AsText(
    ST_Transform(
        ST_SetSRID(ST_MakePoint(116.4536, 39.9385), 4326),
        3857
    )
);
-- 输出：POINT(12956803.3 4825454.8)

-- Web Mercator 转 WGS84
SELECT ST_AsText(
    ST_Transform(
        ST_SetSRID(ST_MakePoint(12956803.3, 4825454.8), 3857),
        4326
    )
);
-- 输出：POINT(116.4536 39.9385)
```

## 总结

PostGIS 将 PostgreSQL 变成了专业的空间数据库：

- **空间数据类型**：Point、LineString、Polygon 支持经纬度存储
- **GiST 索引**：让空间查询能够高效执行，不用遍历全表
- **ST_DWithin**：查找附近目标，支持空间索引加速
- **ST_Distance**：精确计算两点间距离（米级精度）
- **ST_Contains/ST_Intersects**：空间关系判断

当你的应用需要处理地理位置数据时，PostGIS 是 PostgreSQL 带来的强大武器——**高效的空间存储、索引和计算**，一个数据库全搞定。
