# PostgreSQL 窗口函数

面试官问："如何计算每个部门的工资排名？如何在同一查询中计算累计销售额？"

如果你只知道 GROUP BY，你需要多表 JOIN 或者子查询。但如果知道窗口函数，这些问题可以优雅地解决。

窗口函数是 SQL 的高级特性，PostgreSQL 从 8.4 开始支持。理解窗口函数，是进行数据分析的基础。

## 窗口函数基础

### 什么是窗口函数

窗口函数在一组行上执行计算，并为每行返回一个结果。与聚合函数不同，窗口函数不会将多行合并为一行——每行保持独立，只是增加了额外的"窗口"列。

```sql
-- 普通聚合：多行合并为一行
SELECT department, SUM(salary) FROM employees GROUP BY department;

-- 窗口函数：每行都有聚合结果
SELECT
    name,
    department,
    salary,
    SUM(salary) OVER (PARTITION BY department) AS dept_total
FROM employees;
```

### 窗口函数语法

```sql
window_function(arguments) OVER (
    PARTITION BY column_list    -- 分区：分组
    ORDER BY column_list        -- 排序：窗口内的排序
    ROWS/RANGE BETWEEN ...      -- 帧：窗口的范围
)
```

## 常用窗口函数

### 排名函数

```sql
-- ROW_NUMBER：行号（无并列）
SELECT
    name,
    department,
    salary,
    ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS row_num
FROM employees;

-- RANK：排名（有并列，跳过后续名次）
SELECT
    name,
    department,
    salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank
FROM employees;

-- DENSE_RANK：密集排名（有并列，不跳过后续名次）
SELECT
    name,
    department,
    salary,
    DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rank
FROM employees;

-- NTILE：分桶
SELECT
    name,
    salary,
    NTILE(4) OVER (ORDER BY salary) AS quartile
FROM employees;
```

**示例结果对比**：

| name | salary | row_num | rank | dense_rank |
|---|---|---|---|---|
| 张三 | 10000 | 1 | 1 | 1 |
| 李四 | 10000 | 2 | 1 | 1 |
| 王五 | 8000 | 3 | 3 | 2 |
| 赵六 | 6000 | 4 | 4 | 3 |

### 偏移函数

```sql
-- LAG：获取前一行
SELECT
    order_date,
    total,
    LAG(total, 1) OVER (ORDER BY order_date) AS prev_total,
    total - LAG(total, 1) OVER (ORDER BY order_date) AS diff
FROM orders;

-- LEAD：获取后一行
SELECT
    order_date,
    total,
    LEAD(total, 1) OVER (ORDER BY order_date) AS next_total
FROM orders;

-- FIRST_VALUE：窗口内第一个值
SELECT
    order_date,
    total,
    FIRST_VALUE(total) OVER (ORDER BY order_date) AS first_total
FROM orders;

-- LAST_VALUE：窗口内最后一个值
SELECT
    order_date,
    total,
    LAST_VALUE(total) OVER (
        ORDER BY order_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS last_total
FROM orders;

-- NTH_VALUE：窗口内第 N 个值
SELECT
    order_date,
    total,
    NTH_VALUE(total, 3) OVER (ORDER BY order_date) AS third_total
FROM orders;
```

## 帧（Frame）

### 默认帧

默认情况下，帧是 `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`，即从分区开始到当前行。

```sql
-- 默认帧的行为
SELECT
    order_date,
    daily_total,
    SUM(daily_total) OVER (ORDER BY order_date) AS running_total
FROM daily_sales;
```

### 自定义帧

```sql
-- 滑动窗口：最近 7 天
SELECT
    order_date,
    daily_total,
    SUM(daily_total) OVER (
        ORDER BY order_date
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) AS last_7_days
FROM daily_sales;

-- 滑动窗口：前后各 3 天
SELECT
    order_date,
    daily_total,
    AVG(daily_total) OVER (
        ORDER BY order_date
        ROWS BETWEEN 3 PRECEDING AND 3 FOLLOWING
    ) AS moving_avg
FROM daily_sales;

-- 整个分区
SELECT
    department,
    salary,
    SUM(salary) OVER (
        PARTITION BY department
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS dept_total
FROM employees;
```

### ROWS vs RANGE

| 模式 | 说明 |
|---|---|
| `ROWS` | 按物理行号 |
| `RANGE` | 按逻辑值（相同的值视为同一帧） |

```sql
-- ROWS：按行计算
SELECT
    salary,
    SUM(salary) OVER (ORDER BY salary ROWS UNBOUNDED PRECEDING) AS running_sum
FROM employees;

-- RANGE：按值计算（相同值一起计算）
SELECT
    salary,
    SUM(salary) OVER (ORDER BY salary RANGE UNBOUNDED PRECEDING) AS running_sum
FROM employees;
```

## 实战案例

### 案例一：部门工资排名

```sql
-- 查询每个部门的工资排名
SELECT
    department,
    name,
    salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank,
    DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_dense_rank
FROM employees;

-- 查询每个部门的最高工资
SELECT *
FROM (
    SELECT
        department,
        name,
        salary,
        RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank
    FROM employees
) ranked
WHERE rank = 1;
```

### 案例二：累计销售额

```sql
-- 按月累计销售额
SELECT
    month,
    monthly_sales,
    SUM(monthly_sales) OVER (ORDER BY month) AS cumulative_sales,
    ROUND(
        100.0 * SUM(monthly_sales) OVER (ORDER BY month) /
        SUM(monthly_sales) OVER (),
        2
    ) AS pct_of_total
FROM monthly_sales
ORDER BY month;
```

### 案例三：同比环比

```sql
-- 计算同比和环比
SELECT
    month,
    sales,
    LAG(sales, 1) OVER (ORDER BY month) AS prev_month,           -- 环比
    LAG(sales, 12) OVER (ORDER BY month) AS prev_year,            -- 同比
    ROUND(
        100.0 * (sales - LAG(sales, 1) OVER (ORDER BY month)) /
        LAG(sales, 1) OVER (ORDER BY month),
        2
    ) AS mom_growth,                                              -- 环比增长率
    ROUND(
        100.0 * (sales - LAG(sales, 12) OVER (ORDER BY month)) /
        LAG(sales, 12) OVER (ORDER BY month),
        2
    ) AS yoy_growth                                               -- 同比增长率
FROM monthly_sales;
```

### 案例四：分组排名 TOP N

```sql
-- 每个部门工资最高的前三名
WITH ranked AS (
    SELECT
        department,
        name,
        salary,
        ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS rank
    FROM employees
)
SELECT *
FROM ranked
WHERE rank <= 3;
```

### 案例五：移动平均

```sql
-- 计算 7 日移动平均
SELECT
    order_date,
    daily_revenue,
    ROUND(
        AVG(daily_revenue) OVER (
            ORDER BY order_date
            ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
        ),
        2
    ) AS ma_7
FROM daily_revenue
ORDER BY order_date;
```

## 窗口函数与聚合函数的对比

### 等价写法

```sql
-- 聚合函数
SELECT department, SUM(salary) FROM employees GROUP BY department;

-- 窗口函数
SELECT DISTINCT
    department,
    SUM(salary) OVER (PARTITION BY department) AS total
FROM employees;

-- 使用 CTE
WITH dept_salary AS (
    SELECT department, salary FROM employees
)
SELECT DISTINCT
    department,
    SUM(salary) OVER (PARTITION BY department) AS total
FROM dept_salary;
```

### 窗口函数的优势

1. **简洁**：不需要 JOIN 或子查询
2. **灵活**：可以在同一查询中计算多个窗口
3. **保持行数**：不会减少原始行数
4. **易于维护**：逻辑清晰

## 性能考虑

### 窗口函数的开销

窗口函数在内存中维护窗口状态，对于大窗口可能有性能影响。

```sql
-- 优化：减少窗口大小
-- 不好：全窗口
SELECT
    order_date,
    SUM(amount) OVER (ORDER BY order_date) AS running_total
FROM orders;

-- 好：限制窗口大小
SELECT
    order_date,
    SUM(amount) OVER (
        ORDER BY order_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total
FROM orders;
```

### 物化视图优化

对于频繁使用的窗口计算，可以考虑物化视图：

```sql
CREATE MATERIALIZED VIEW monthly_sales_stats AS
SELECT
    month,
    monthly_sales,
    SUM(monthly_sales) OVER (ORDER BY month) AS cumulative,
    LAG(monthly_sales, 1) OVER (ORDER BY month) AS prev_month
FROM monthly_sales
ORDER BY month;

CREATE UNIQUE INDEX ON monthly_sales_stats(month);
```

## 高级用法

### 多窗口

```sql
SELECT
    order_date,
    amount,
    SUM(amount) OVER w1 AS running_total,
    AVG(amount) OVER w2 AS moving_avg,
    RANK() OVER w3 AS daily_rank
FROM orders
WINDOW
    w1 AS (ORDER BY order_date),
    w2 AS (ORDER BY order_date ROWS 6 PRECEDING),
    w3 AS (ORDER BY amount DESC);
```

### FILTER 子句

```sql
-- 只统计满足条件的值
SELECT
    order_date,
    amount,
    SUM(amount) FILTER (WHERE amount > 100) OVER (ORDER BY order_date) AS large_orders_total
FROM orders;
```

## 总结

窗口函数是 SQL 的强大特性：

1. **核心概念**：OVER、PARTITION BY、ORDER BY、帧
2. **排名函数**：ROW_NUMBER、RANK、DENSE_RANK、NTILE
3. **偏移函数**：LAG、LEAD、FIRST_VALUE、LAST_VALUE、NTH_VALUE
4. **聚合函数**：SUM、AVG、COUNT、MAX、MIN
5. **帧**：ROWS BETWEEN、RANGE BETWEEN

窗口函数可以优雅地解决：
- 排名问题
- 累计计算
- 同比环比
- 移动平均
- TOP N 查询

掌握窗口函数，是进行 SQL 数据分析的必备技能。
