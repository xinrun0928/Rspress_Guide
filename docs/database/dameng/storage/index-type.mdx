# 索引类型详解

<Badge text="核心" type="tip" />

## 索引概述

索引是数据库性能优化的核心手段。正确的索引设计可以让查询速度提升数百甚至数千倍；错误的索引设计不仅浪费空间，还可能拖累写入性能。

你可能听说过：索引是双刃剑。

这句话在达梦数据库中同样适用。理解各种索引类型的特点和适用场景，是成为达梦 DBA 的必备技能。

## 索引类型总览

达梦数据库支持多种索引类型：

| 类型 | 英文名 | 适用场景 | 特点 |
|---|---|---|---|
| B 树索引 | B-Tree Index | 大多数场景 | 默认索引 |
| 位图索引 | Bitmap Index | 低基数列 | 位图存储 |
| 函数索引 | Function Index | 表达式查询 | 计算后索引 |
| 全文索引 | Full-Text Index | 文本搜索 | 分词检索 |
| 向量索引 | Vector Index | AI 场景 | 向量相似度 |

## B 树索引

### B 树索引的原理

B 树索引是达梦数据库的默认索引类型，适用于大多数查询场景。它的结构类似于二叉查找树，但每个节点可以有多个子节点。

```
B 树索引结构示意：

                    [50]
                 /         \
            [25, 40]      [75, 90]
            /    \        /     \
         [..] [..]    [..]   [..]
            ↓         ↓        ↓
          叶子页    叶子页    叶子页
            ↓         ↓        ↓
        [数据行指针] [数据行指针] [数据行指针]
```

B 树索引的特点：

- **平衡树**：所有叶子节点在同一层级
- **有序存储**：叶子节点按索引列排序
- **范围查询友好**：叶子节点链表支持范围扫描

### B 树索引的创建

```sql
-- 单列索引（默认 B 树索引）
CREATE INDEX idx_emp_name ON employees(name);

-- 指定索引类型
CREATE INDEX idx_emp_name ON employees(name) USING BTREE;

-- 唯一索引
CREATE UNIQUE INDEX idx_emp_code ON employees(emp_code);

-- 组合索引
CREATE INDEX idx_emp_dept_salary ON employees(dept_id, salary DESC);
```

### B 树索引的适用场景

| 场景 | 适用性 | 说明 |
|---|---|---|
| 等值查询 | <Badge text="推荐" type="tip" /> | `WHERE col = value` |
| 范围查询 | <Badge text="推荐" type="tip" /> | `WHERE col BETWEEN a AND b` |
| 排序 | <Badge text="推荐" type="tip" /> | `ORDER BY col` |
| 前缀匹配 | <Badge text="推荐" type="tip" /> | `WHERE col LIKE 'abc%'` |

```sql
-- 等值查询
SELECT * FROM employees WHERE emp_code = 'E001';

-- 范围查询
SELECT * FROM employees WHERE salary BETWEEN 5000 AND 10000;

-- 排序查询
SELECT * FROM employees ORDER BY hire_date DESC;

-- 前缀匹配
SELECT * FROM employees WHERE name LIKE '张%';

-- 组合索引使用
-- 假设有索引 idx(dept_id, salary)
SELECT * FROM employees 
WHERE dept_id = 10 
  AND salary `>=` 5000;
-- 组合索引可以高效支持
```

## 位图索引

### 位图索引的原理

位图索引针对低基数（取值种类少）列设计，使用位图存储每个取值对应的行位置。

<Badge text="适用低基数" type="tip" />

```
位图索引原理：

假设有表 employees，性别列有 2 种取值：男、女

表数据：
RowID   姓名    性别
1       张三    男
2       李四    女
3       王五    男
4       赵六    女
...

位图索引：
性别='男': [1, 0, 1, 0, ...]  →  1010...
性别='女': [0, 1, 0, 1, ...]  →  0101...

查询 性别='男' AND 性别='女':
  1010... AND 0101... = 0000...  →  无结果
```

### 位图索引的创建

```sql
-- 创建位图索引
CREATE BITMAP INDEX idx_gender ON employees(gender);

-- 创建位图组合索引
CREATE BITMAP INDEX idx_status_type ON orders(order_status, order_type);
```

### 位图索引的适用场景

| 场景 | 适用性 | 说明 |
|---|---|---|
| 性别、国籍等枚举值 | <Badge text="强烈推荐" type="tip" /> | 取值种类少 |
| 状态、类型等字段 | <Badge text="推荐" type="tip" /> | 取值种类少 |
| 需要 AND/OR 组合查询 | <Badge text="推荐" type="tip" /> | 位运算优化 |
| 高基数列 | <Badge text="不适用" type="danger" /> | 空间爆炸 |

```sql
-- 低基数列示例
CREATE BITMAP INDEX idx_nation ON customers(nationality);    -- 假设只有几十个国家
CREATE BITMAP INDEX idx_grade ON students(grade);           -- 年级：1-12

-- 组合位图查询（适合 OLAP 场景）
SELECT * FROM students
WHERE grade = 5 AND class = 'A';

-- 对比 B 树索引
-- B 树索引需要扫描多个索引项
-- 位图索引直接位运算
```

### 位图索引的优缺点

**优点**：

- 空间效率高（低基数列）
- 组合查询快（位运算）
- 适合 OLAP 场景

**缺点**：

- 更新代价高（行锁）
- 不适合高并发 OLTP
- 空间随数据量线性增长

## 函数索引

### 函数索引的原理

函数索引是对列值经过计算后的结果建立索引，适用于需要对列进行函数处理的查询场景。

```sql
-- 函数索引示例
-- 查询时经常使用 UPPER() 函数处理姓名
CREATE INDEX idx_upper_name ON employees(UPPER(name));

-- 现在的查询可以直接使用索引
SELECT * FROM employees WHERE UPPER(name) = 'ZHANG SAN';
```

### 函数索引的创建

```sql
-- 常用函数索引
CREATE INDEX idx_upper_name ON employees(UPPER(name));
CREATE INDEX idx_lower_email ON employees(LOWER(email));
CREATE INDEX idx_salary_tax ON employees(salary * 0.85);
CREATE INDEX idx_full_address ON employees(city || ' ' || street);

-- 带表达式的索引
CREATE INDEX idx_year ON orders(EXTRACT(YEAR FROM order_date));
CREATE INDEX idx_month ON orders(TO_CHAR(order_date, 'YYYY-MM'));
```

### 函数索引的适用场景

| 场景 | 示例 |
|---|---|
| 大小写不敏感查询 | `WHERE UPPER(name) = 'XXX'` |
| 日期部分查询 | `WHERE TO_CHAR(date, 'YYYY-MM') = '2024-01'` |
| 计算列查询 | `WHERE salary * 0.85 `>=` 5000` |
| 字符串拼接查询 | `WHERE city || street = 'xxx'` |

```sql
-- 应用案例：用户名登录（大小写不敏感）
CREATE INDEX idx_login ON users(LOWER(username));

-- 登录查询（可以使用索引）
SELECT * FROM users WHERE LOWER(username) = LOWER('ZhangSan');

-- 应用案例：月份统计查询
CREATE INDEX idx_order_month ON orders(TRUNC(order_date, 'MM'));

-- 月份查询（可以使用索引）
SELECT COUNT(*), SUM(amount)
FROM orders
WHERE TRUNC(order_date, 'MM') = DATE '2024-01-01';
```

## 全文索引

### 全文索引的原理

全文索引用于文本内容的快速搜索，支持中文分词和模糊匹配。

```sql
-- 创建全文索引
CREATE FULLTEXT INDEX idx_content ON articles(title, content);

-- 全文搜索
SELECT * FROM articles 
WHERE MATCH(title, content) AGAINST('数据库 优化');
```

### 全文索引的使用

```sql
-- 基本全文搜索
SELECT id, title, 
    SNIPPET(content, '达梦', '...', '<b>', '</b>', -1, 32) AS snippet
FROM articles
WHERE MATCH(title, content) AGAINST('达梦' IN BOOLEAN MODE);

-- 布尔全文搜索
SELECT * FROM articles
WHERE MATCH(title, content) AGAINST('+达梦 -Oracle +数据库' IN BOOLEAN MODE);
-- + 必须包含
-- - 必须不包含
-- 多词默认为 OR
```

## 向量索引

### 向量索引的原理

向量索引是 DM8 的新特性，用于 AI 场景下的向量相似度搜索。

<Badge text="DM8 新特性" type="tip" />

```sql
-- 创建向量列
CREATE TABLE document_embeddings (
    id INT PRIMARY KEY,
    doc_id INT,
    content VARCHAR(1000),
    embedding VECTOR(768)          -- 向量维度
);

-- 创建向量索引（HNSW 算法）
CREATE VECTOR INDEX idx_embedding ON document_embeddings(embedding)
TYPE HNSW DIMENSION 768;

-- 向量相似度查询
SELECT id, doc_id, content
FROM document_embeddings
ORDER BY VECTOR_L2_DISTANCE(embedding, :query_embedding)
LIMIT 10;
```

## 索引创建语法

### 基本语法

```sql
CREATE [UNIQUE|BITMAP] INDEX index_name 
ON table_name (column [, column2...])
[REVERSE]
[TABLESPACE tablespace_name]
[STORAGE (INITIAL n NEXT n ...)];

-- UNIQUE: 唯一索引
-- BITMAP: 位图索引
-- REVERSE: 反向键索引（适合序列增长列）
-- TABLESPACE: 指定存储表空间
```

### 反向键索引

```sql
-- 反向键索引：将索引键值反转存储
-- 适用场景：序列值作为主键，连续插入导致右侧竞争
CREATE INDEX idx_reverse_id ON transactions(transaction_id) REVERSE;

-- 普通索引：1, 2, 3, 4, 5, ... 集中在右侧
-- 反向索引：1, 2, 3, 4, 5, ... 分散到整个索引树
```

### 索引选项

```sql
-- 指定表空间
CREATE INDEX idx_tbs ON employees(name) TABLESPACE idx_tbs;

-- 指定存储参数
CREATE INDEX idx_storage ON employees(name)
STORAGE(INITIAL 1024 NEXT 256);

-- 在线创建索引
CREATE INDEX idx_online ON employees(name) ONLINE;

-- 并行创建索引
CREATE INDEX idx_parallel ON large_table(product_id) PARALLEL 4;
```

## 索引管理

### 查看索引

```sql
-- 查看表的所有索引
SELECT 
    index_name,
    column_name,
    column_position,
    uniqueness,
    index_type
FROM user_ind_columns
WHERE table_name = 'EMPLOYEES'
ORDER BY index_name, column_position;

-- 查看索引详情
SELECT 
    index_name,
    table_name,
    index_type,
    uniqueness,
    status,
    num_rows,
    distinct_keys
FROM user_indexes
WHERE table_name = 'EMPLOYEES';

-- 查看索引使用情况
SELECT * FROM v$object_usage;
```

### 重建索引

```sql
-- 重建单个索引
ALTER INDEX idx_name REBUILD;

-- 重建所有失效索引
ALTER INDEX idx_name REBUILD ONLINE;

-- 查看索引状态
SELECT index_name, status FROM user_indexes WHERE status != 'VALID';
```

### 删除索引

```sql
-- 删除索引
DROP INDEX idx_name;

-- 删除约束关联的索引（先删除约束）
ALTER TABLE employees DROP PRIMARY KEY;
-- 或
ALTER TABLE employees DROP CONSTRAINT emp_pk;
```

## 面试要点

<Badge text="面试重点" type="warning" />

### 索引类型对比

**Q：B 树索引和位图索引的区别是什么？**

| 维度 | B 树索引 | 位图索引 |
|---|---|---|
| 适用列基数 | 高基数 | 低基数 |
| 存储方式 | 树结构 | 位图 |
| 查询效率 | 等值/范围查询 | 组合 AND/OR 查询 |
| 更新代价 | 低 | 高 |
| 并发支持 | 好 | 差 |

### 索引创建原则

**Q：什么时候应该创建索引？**

应该创建索引的场景：

- 主键和唯一约束（自动创建）
- WHERE 条件频繁使用的列
- JOIN 连接列
- ORDER BY、GROUP BY 列
- 区分度高的列

不应该创建索引的场景：

- 低区分度列（如性别）
- 频繁更新的列
- 数据量很小的表
- 不在 WHERE 条件中使用的列

### 函数索引

**Q：函数索引的作用是什么？**

函数索引允许在索引中存储计算后的值，解决以下问题：

1. 大小写不敏感查询：`UPPER(name)` 索引
2. 日期部分查询：`TRUNC(date)` 索引
3. 计算表达式查询：`salary * 0.85` 索引

## 总结

达梦数据库的索引类型丰富：

- **B 树索引**：默认索引，适合大多数场景
- **位图索引**：适合低基数列的组合查询
- **函数索引**：适合表达式查询
- **全文索引**：适合文本搜索
- **向量索引**：适合 AI 场景

理解各种索引的特点，是设计高效数据库的基础。
