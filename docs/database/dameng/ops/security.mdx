# DM 安全体系：用户、角色与权限管理

数据库安全，是数据保护的最后一道防线。

你的财务数据、用户隐私、商业机密——都存在数据库里。如果权限管理混乱，就像把金库钥匙挂在墙上，谁都能进来拿。**用户能做什么、不能做什么、谁可以授权、谁可以审计**，都需要精心设计。

DM 数据库提供了完整的 **RBAC（Role-Based Access Control，基于角色的访问控制）** 安全体系。

## DM 安全模型概述

### 安全架构

DM 的安全体系分为三个层次：

```
┌─────────────────────────────────────────────────────────────┐
│                    DM 安全架构                              │
│                                                             │
│  ┌───────────────┐                                         │
│  │    用户       │  ← 数据库账号，用于认证                  │
│  └───────┬───────┘                                         │
│          │                                                 │
│  ┌───────▼───────┐                                         │
│  │    角色       │  ← 权限集合，简化授权管理                  │
│  └───────┬───────┘                                         │
│          │                                                 │
│  ┌───────▼───────┐                                         │
│  │    权限       │  ← 对对象的操作能力                        │
│  └───────────────┘                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**核心概念**：

| 概念 | 说明 |
|---|---|
| 用户（USER）| 数据库账号，用于登录认证 |
| 角色（ROLE）| 权限的集合，简化授权管理 |
| 权限（PRIVILEGE）| 对数据库对象（表、视图、序列等）的操作能力 |
| 对象（OBJECT）| 表、视图、存储过程、序列等 |
| 模式（SCHEMA）| 对象的命名空间，与用户关联 |

### 内置角色

DM 预定义了一些常用角色：

```sql
-- 查看所有内置角色
SELECT * FROM SYS.SYSROLES WHERE SYSRBA IS NOT NULL;

-- 常用内置角色
-- DBA: 超级管理员，几乎所有权限
-- RESOURCE: 可以创建数据库对象
-- CONNECT: 可以连接数据库
-- PUBLIC: 所有用户的默认角色
```

## 用户管理

### 创建用户

```sql
-- 基本创建用户
CREATE USER SYSDBA IDENTIFIED BY "YourPassword123";

-- 创建带默认表空间的用户
CREATE USER app_user IDENTIFIED BY "AppUser123"
    DEFAULT TABLESPACE MAIN
    TEMPORARY TABLESPACE TEMP;

-- 创建带资源限制的用户（防止资源耗尽）
CREATE USER limited_user IDENTIFIED BY "Limited123"
    PROFILE DEFAULT
    PASSWORD EXPIRE
    ACCOUNT UNLOCK;

-- 创建只读用户
CREATE USER readonly_user IDENTIFIED BY "ReadOnly123"
    DEFAULT TABLESPACE MAIN;
```

### 用户属性

```sql
-- 查看用户信息
SELECT USERNAME, ACCOUNT_STATUS, CREATED, DEFAULT_TABLESPACE
FROM DBA_USERS
WHERE USERNAME = 'APP_USER';

-- 用户状态说明
-- OPEN: 正常
-- LOCKED: 被锁定（多次登录失败）
-- EXPIRED: 密码过期
-- EXPIRED & LOCKED: 密码过期且被锁定
```

### 用户修改与删除

```sql
-- 修改用户密码
ALTER USER app_user IDENTIFIED BY "NewPassword456";

-- 锁定用户
ALTER USER app_user ACCOUNT LOCK;

-- 解锁用户
ALTER USER app_user ACCOUNT UNLOCK;

-- 修改默认表空间
ALTER USER app_user DEFAULT TABLESPACE USERS;

-- 删除用户（需要先删除关联对象）
DROP USER app_user CASCADE;
```

## 角色管理

### 创建角色

```sql
-- 创建角色
CREATE ROLE app_developer;
CREATE ROLE app_dba;
CREATE ROLE data_analyst;

-- 带密码的角色
CREATE ROLE app_admin IDENTIFIED BY "AdminPassword123";
```

### 角色授权

```sql
-- 将权限授予角色
GRANT CREATE SESSION TO app_developer;
GRANT CREATE TABLE TO app_developer;
GRANT CREATE VIEW TO app_developer;
GRANT CREATE PROCEDURE TO app_developer;

-- 将角色授予用户
GRANT app_developer TO user_zhang;
GRANT app_developer TO user_li;

-- 查看用户拥有的角色
SELECT GRANTEE, GRANTED_ROLE
FROM DBA_ROLE_PRIVS
WHERE GRANTEE = 'USER_ZHANG';
```

### 角色继承

```sql
-- 角色可以继承其他角色
-- 创建一个高级角色，继承基础角色
CREATE ROLE app_senior_developer;

GRANT app_developer TO app_senior_developer;
GRANT CREATE ANY TABLE TO app_senior_developer;
GRANT CREATE ANY VIEW TO app_senior_developer;

-- 将高级角色授予资深开发
GRANT app_senior_developer TO user_wang;
```

### 角色使用

```sql
-- 角色授权给用户后，用户默认没有激活角色
-- 需要显式激活角色
SET ROLE app_developer;
SET ROLE app_developer, app_dba;

-- 设置默认角色（登录时自动激活）
ALTER USER user_zhang DEFAULT ROLE app_developer;

-- 设置多个默认角色
ALTER USER user_zhang DEFAULT ROLE app_developer, data_analyst;

-- 查看当前会话激活的角色
SELECT * FROM SESSION_ROLES;
```

## 权限管理

### 权限类型

DM 的权限分为两大类：

| 类型 | 说明 | 示例 |
|---|---|---|
| 系统权限 | 对整个数据库的操作能力 | CREATE TABLE, CREATE USER, CREATE ANY VIEW |
| 对象权限 | 对特定对象的操作能力 | SELECT ON table, UPDATE ON column |

### 系统权限授权

```sql
-- 授予系统权限
GRANT CREATE SESSION TO app_user;           -- 连接数据库
GRANT CREATE TABLE TO app_user;              -- 创建表
GRANT CREATE VIEW TO app_user;               -- 创建视图
GRANT CREATE PROCEDURE TO app_user;           -- 创建存储过程
GRANT CREATE SEQUENCE TO app_user;            -- 创建序列
GRANT CREATE DATABASE LINK TO app_user;       -- 创建数据库链接

-- 授予所有系统权限（谨慎使用）
GRANT DBA TO app_admin;

-- WITH ADMIN OPTION：允许被授权者再授权给别人
GRANT CREATE TABLE TO app_developer WITH ADMIN OPTION;

-- 撤销系统权限
REVOKE CREATE TABLE FROM app_user;
REVOKE DBA FROM app_admin;
```

### 对象权限授权

```sql
-- 授予表的对象权限
GRANT SELECT ON app_db.orders TO app_user;
GRANT INSERT ON app_db.orders TO app_user;
GRANT UPDATE ON app_db.orders TO app_user;
GRANT DELETE ON app_db.orders TO app_user;

-- 授予所有对象权限
GRANT ALL ON app_db.orders TO app_user;

-- 授予列级权限（只允许更新特定列）
GRANT UPDATE (balance, status) ON app_db.accounts TO app_user;

-- 授予视图权限
GRANT SELECT ON app_db.v_order_summary TO app_user;

-- 授予存储过程权限
GRANT EXECUTE ON app_db.sp_create_order TO app_user;

-- 授予序列权限
GRANT SELECT ON app_db.order_seq TO app_user;

-- WITH GRANT OPTION：允许被授权者再授权给别人
GRANT SELECT ON app_db.orders TO app_user WITH GRANT OPTION;

-- 撤销对象权限
REVOKE SELECT ON app_db.orders FROM app_user;
REVOKE ALL ON app_db.orders FROM app_user;
```

### 权限传递

```sql
-- 场景：A 授权给 B，B 再授权给 C
-- A 必须有 WITH GRANT OPTION

-- A 授权给 B
GRANT SELECT ON app_db.orders TO B WITH GRANT OPTION;

-- B 授权给 C
GRANT SELECT ON app_db.orders TO C;

-- 查看权限传递链
SELECT GRANTOR, GRANTEE, TABLE_SCHEMA, TABLE_NAME, PRIVILEGE
FROM DBA_TAB_PRIVS
WHERE TABLE_NAME = 'ORDERS';
```

## 模式与对象权限

### 模式概念

```sql
-- 每个用户有一个同名模式
-- 模式包含该用户拥有的所有对象

-- 查看用户模式
SELECT SCHEMA_NAME
FROM USER_SCHEMAS
WHERE SCHEMA_OWNER = 'APP_USER';

-- 在模式中创建对象（默认使用当前用户模式）
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    user_id INT,
    amount DECIMAL(10,2)
);
-- 默认创建在当前用户的模式中
```

### 跨模式访问

```sql
-- 访问其他模式的对象
SELECT * FROM app_admin.orders;  -- 需要有 SELECT 权限

-- 授权跨模式访问
GRANT SELECT ON app_admin.orders TO app_user;

-- 创建同义词简化访问
CREATE SYNONYM orders FOR app_admin.orders;
GRANT SELECT ON orders TO app_user;
```

### 模式对象权限总结

| 对象类型 | 可授予的权限 |
|---|---|
| 表 | SELECT, INSERT, UPDATE, DELETE, ALTER, INDEX, REFERENCES |
| 视图 | SELECT, INSERT, UPDATE, DELETE |
| 序列 | SELECT, ALTER |
| 存储过程/函数 | EXECUTE |
| 数据库 | CREATE TABLE, CREATE VIEW, CREATE PROCEDURE 等 |

## 最佳安全实践

### 最小权限原则

```sql
-- 错误做法：直接给 DBA 权限
GRANT DBA TO app_user;

-- 正确做法：按需分配权限
GRANT CREATE SESSION TO app_user;
GRANT SELECT, INSERT, UPDATE, DELETE ON app_db.orders TO app_user;
GRANT SELECT ON app_db.v_order_summary TO app_user;

-- 更好的做法：通过角色授权
GRANT app_developer TO app_user;
```

### 分离职责

```sql
-- 创建不同职责的角色
-- 应用开发角色
CREATE ROLE role_app_dev;
GRANT CREATE SESSION TO role_app_dev;
GRANT CREATE TABLE TO role_app_dev;
GRANT CREATE VIEW TO role_app_dev;
GRANT CREATE PROCEDURE TO role_app_dev;

-- 数据分析角色
CREATE ROLE role_analyst;
GRANT CREATE SESSION TO role_analyst;
GRANT SELECT ON app_db.orders TO role_analyst;
GRANT SELECT ON app_db.products TO role_analyst;
GRANT SELECT ON app_db.users TO role_analyst;

-- 运维管理角色
CREATE ROLE role_ops;
GRANT CREATE SESSION TO role_ops;
GRANT SELECT ANY TABLE TO role_ops;
GRANT BACKUP ANY TABLE TO role_ops;
```

### 密码策略

```sql
-- 创建密码策略
CREATE PROFILE app_profile LIMIT
    FAILED_LOGIN_ATTEMPTS 5        -- 密码错误 5 次后锁定
    PASSWORD_LOCK_TIME 1           -- 锁定 1 天
    PASSWORD_LIFE_TIME 90         -- 密码 90 天后过期
    PASSWORD_GRACE_TIME 7;         -- 过期后有 7 天宽限期

-- 将策略分配给用户
CREATE USER app_user IDENTIFIED BY "StrongPass123"
    PROFILE app_profile;

-- 修改用户的策略
ALTER USER app_user PROFILE app_profile;

-- 修改默认策略
ALTER PROFILE DEFAULT LIMIT
    PASSWORD_LIFE_TIME 60;
```

### 审计策略

```sql
-- 开启审计
AUDIT ALL ON app_db.orders BY ACCESS;

-- 审计登录
AUDIT SESSION;

-- 审计权限使用
AUDIT DBA;

-- 查看审计记录
SELECT * FROM DBA_AUDIT_TRAIL;

-- 审计敏感操作
AUDIT INSERT, UPDATE, DELETE ON app_db.accounts BY ACCESS;
```

## 权限管理脚本示例

```sql
-- 完整的企业级权限管理脚本

-- 1. 创建角色
CREATE ROLE role_erp_developer;
CREATE ROLE role_erp_analyst;
CREATE ROLE role_erp_readonly;

-- 2. 授予角色权限
-- 开发角色
GRANT CREATE SESSION TO role_erp_developer;
GRANT CREATE TABLE TO role_erp_developer;
GRANT CREATE VIEW TO role_erp_developer;
GRANT CREATE PROCEDURE TO role_erp_developer;
GRANT CREATE SEQUENCE TO role_erp_developer;

-- 分析角色
GRANT CREATE SESSION TO role_erp_analyst;
GRANT SELECT ON app_erp.orders TO role_erp_analyst;
GRANT SELECT ON app_erp.products TO role_erp_analyst;
GRANT SELECT ON app_erp.customers TO role_erp_analyst;

-- 只读角色
GRANT CREATE SESSION TO role_erp_readonly;
GRANT SELECT ON app_erp.orders TO role_erp_readonly;
GRANT SELECT ON app_erp.products TO role_erp_readonly;
GRANT SELECT ON app_erp.customers TO role_erp_readonly;

-- 3. 创建用户并分配角色
CREATE USER erp_dev_zhang IDENTIFIED BY "DevZhang123";
GRANT role_erp_developer TO erp_dev_zhang;
ALTER USER erp_dev_zhang DEFAULT ROLE role_erp_developer;

CREATE USER erp_dev_li IDENTIFIED BY "DevLi123";
GRANT role_erp_developer TO erp_dev_li;
ALTER USER erp_dev_li DEFAULT ROLE role_erp_developer;

CREATE USER erp_analyst_wang IDENTIFIED BY "AnalystWang123";
GRANT role_erp_analyst TO erp_analyst_wang;
ALTER USER erp_analyst_wang DEFAULT ROLE role_erp_analyst;

CREATE USER erp_readonly IDENTIFIED BY "ReadOnly123";
GRANT role_erp_readonly TO erp_readonly;
ALTER USER erp_readonly DEFAULT ROLE role_erp_readonly;

-- 4. 查看用户权限
SELECT u.USERNAME, r.GRANTED_ROLE
FROM DBA_USERS u
LEFT JOIN DBA_ROLE_PRIVS r ON u.USERNAME = r.GRANTEE
WHERE u.USERNAME LIKE 'ERP_%';
```

## 总结

DM 的安全体系提供了精细化的权限控制能力：

- **用户**：认证的实体，按职责分配
- **角色**：权限的集合，简化管理
- **权限**：系统权限和对象权限，粒度可控
- **模式**：对象的命名空间，组织清晰

遵循 **最小权限原则**、**分离职责**、**密码策略** 和 **审计监控**，才能构建起数据库的安全防线。

权限管理不是一次性的工作，而是需要持续优化和监控的长期工程。
