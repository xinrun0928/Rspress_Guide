# 达梦安装与配置

<Badge text="入门" type="tip" />

## 安装前准备

你拿到了一个信创项目订单，数据库选型是达梦。项目环境要求：

- 麒麟操作系统（国产 Linux）
- 飞腾 CPU（ARM 架构）
- 8 核 CPU，32GB 内存

达梦完全支持这些环境。但在此之前，你需要先了解安装前的准备工作。

### 系统要求

达梦数据库对硬件和软件环境的基本要求：

| 要求 | 最低配置 | 推荐配置 |
|---|---|---|
| CPU | 2 核 | 8 核及以上 |
| 内存 | 4GB | 16GB 及以上 |
| 磁盘 | 10GB | 100GB 及以上 |
| 操作系统 | Linux/Windows | 麒麟、统信、CentOS |

达梦支持的操作系统：

| 操作系统 | CPU 架构 | 说明 |
|---|---|---|
| 银河麒麟 | x86_64、ARM64 | 国产主流操作系统 |
| 统信 UOS | x86_64、ARM64 | 国产主流操作系统 |
| 中标麒麟 | x86_64、ARM64 | 国产服务器操作系统 |
| CentOS | x86_64 | 社区版 Linux |
| Windows | x86_64 | 仅开发测试使用 |

## 安装步骤

### 解压安装包

```bash
# 创建安装目录
mkdir -p /dm8/software

# 解压安装包
unzip dm8_xxx_x86_rh6_64_ent.zip -d /dm8/software/

# 查看解压内容
ls -la /dm8/software/
```

解压后会得到以下文件：

```
dm8_xxx_x86_rh6_64_ent.zip
    ├── DMInstall.bin          # 安装程序
    ├── dm书生客户端工具.zip    # 客户端工具
    └── 使用手册/
        ├── DM8 快速上手.pdf
        ├── DM8 安装手册.pdf
        └── DM8 SQL 使用手册.pdf
```

### 创建用户和目录

达梦建议使用非 root 用户安装：

```bash
# 创建用户组
groupadd dinstall

# 创建用户
useradd -g dinstall -m -d /home/dmdba -s /bin/bash dmdba

# 设置密码
passwd dmdba

# 创建安装目录
mkdir -p /dm8
chown -R dmdba:dinstall /dm8
```

### 开始安装

```bash
# 切换到 dmdba 用户
su - dmdba

# 设置图形界面转发（如果使用 SSH）
export DISPLAY=:0.0

# 运行安装程序
cd /dm8/software
./DMInstall.bin
```

安装程序会启动图形界面，按照向导操作即可：

1. **语言选择**：选择中文
2. **许可协议**：阅读并接受
3. **安装类型**：典型安装
4. **安装路径**：/dm8
5. **时区选择**：选择 Asia/Shanghai
6. **安装确认**：确认安装信息
7. **安装完成**：显示安装摘要

### 命令行静默安装

对于服务器环境，可以使用命令行静默安装：

```bash
# 创建配置文件
cat > /dm8/software/dm_silent.ini << 'EOF'
# 安装路径
INSTALL_PATH=/dm8

# 时区
TIME_ZONE=Asia/Shanghai

# 安装类型
INSTALL_TYPE=典型安装

# 确认安装
KEY=
EOF

# 执行静默安装
su - dmdba
cd /dm8/software
./DMInstall.bin -q /dm8/software/dm_silent.ini
```

### 配置环境变量

```bash
# 编辑 dmdba 用户的环境变量
su - dmdba
vi ~/.bash_profile

# 添加以下内容
export DM_HOME=/dm8/dmdbms
export PATH=$DM_HOME/bin:$DM_HOME/tool:$PATH
export LD_LIBRARY_PATH=$DM_HOME/bin:$LD_LIBRARY_PATH

# 使配置生效
source ~/.bash_profile
```

## 初始化数据库实例

### 使用图形工具初始化

达梦提供了图形化初始化工具：

```bash
# 以 dmdba 用户启动数据库配置助手
cd /dm8/dmdbms/tool
./dbca.sh
```

初始化步骤：

1. **创建数据库**：选择创建新实例
2. **选择模板**：通用模板OLTP/OLAP
3. **指定实例名**：如 DAMENG
4. **配置参数**：端口号（5236）、簇大小等
5. **选择表空间**：使用默认或创建新表空间
6. **配置日志**：日志文件大小和数量
7. **完成初始化**：创建数据库实例

### 命令行初始化

```bash
# 使用 dminit 工具初始化
cd /dm8/dmdbms/bin

# 基本初始化
./dminit path=/dm8/dmdbms/data

# 定制初始化
./dminit \
    path=/dm8/dmdbms/data \
    db_name=DAMENG \
    instance_name=DMSERVER \
    port_num=5236 \
    extent_size=16 \
    page_size=8 \
    log_size=256 \
    log_file_num=3 \
    charset=1 \
    TIME_ZONE=480
```

参数说明：

| 参数 | 说明 | 默认值 |
|---|---|---|
| `path` | 数据文件目录 | 当前目录 |
| `db_name` | 数据库名 | DAMENG |
| `instance_name` | 实例名 | DMSERVER |
| `port_num` | 端口号 | 5236 |
| `extent_size` | 区大小（页数） | 16 |
| `page_size` | 页大小（KB） | 8 |
| `log_size` | 日志文件大小（MB） | 256 |
| `log_file_num` | 日志文件数量 | 3 |
| `charset` | 字符集（0:GBK, 1:UTF-8） | 0 |

## 启动与停止

### 启动数据库

```bash
# 启动服务（前台）
cd /dm8/dmdbms/bin
./dmserver /dm8/dmdbms/data/DAMENG/dm.ini

# 启动服务（后台，使用 systemd）
systemctl start DmServiceDAMENG

# 检查服务状态
systemctl status DmServiceDAMENG
```

### 创建 systemd 服务

```bash
# root 用户执行
cat > /usr/lib/systemd/system/DmServiceDAMENG.service << 'EOF'
[Unit]
Description=DM Database Service
After=network.target

[Service]
Type=forking
ExecStart=/dm8/dmdbms/bin/DmServiceDAMENG start
ExecStop=/dm8/dmdbms/bin/DmServiceDAMENG stop
User=dmdba
Group=dinstall

[Install]
WantedBy=multi-user.target
EOF

# 重载 systemd
systemctl daemon-reload

# 启用开机启动
systemctl enable DmServiceDAMENG
```

### 停止数据库

```bash
# 使用服务脚本停止
systemctl stop DmServiceDAMENG

# 或使用 dmserver 命令
# 在前台运行时，按 Ctrl+C 或在另一个终端执行
# cd /dm8/dmdbms/bin
# ./DmServiceDAMENG stop
```

## 客户端连接

### 使用 disql 连接

达梦的命令行客户端工具是 `disql`：

```bash
# 基本连接
cd /dm8/dmdbms/bin
./disql sysdba/SYSDBA@localhost:5236

# 连接本地数据库（不指定地址）
./disql sysdba/SYSDBA

# 使用 IP 连接
./disql sysdba/SYSDBA@192.168.1.100:5236
```

连接成功后会显示：

```
disql V8
Connected to DM SQL Editor V8

SQL>
```

### 基本 SQL 操作

```sql
-- 查看当前用户
SELECT USER;

-- 查看数据库版本
SELECT * FROM v$version;

-- 查看数据库信息
SELECT * FROM v$database;

-- 查看会话
SELECT * FROM v$sessions;

-- 创建测试表
CREATE TABLE test_users (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    created_at DATETIME DEFAULT SYSDATE
);

-- 插入测试数据
INSERT INTO test_users VALUES (1, '张三', SYSDATE);
INSERT INTO test_users VALUES (2, '李四', SYSDATE);

-- 查询数据
SELECT * FROM test_users;

-- 查看表结构
DESC test_users;
```

## 配置参数优化

### 关键配置参数

```sql
-- 查看当前配置
SELECT para_name, para_value, description
FROM v$dm_ini
WHERE para_name IN (
    'BUFFER',           -- 缓冲区大小
    'MAX_SESSIONS',     -- 最大会话数
    'WORKER_THREADS',   -- 工作线程数
    'TASK_THREADS',     -- 任务线程数
    'HJ_BUF_GLOBAL_SIZE',  -- 哈希连接缓冲区
    'SORT_BUF_SIZE'     -- 排序缓冲区
);

-- 修改参数（动态生效）
ALTER SYSTEM SET 'MAX_SESSIONS' = 500;

-- 修改参数（重启生效）
-- 需要在 dm.ini 中修改
```

### 内存参数配置建议

根据服务器内存配置：

| 服务器内存 | BUFFER 配置 | MAX_SESSIONS |
|---|---|---|
| 4GB | 256MB | 100 |
| 16GB | 4GB | 500 |
| 32GB | 10GB | 1000 |
| 64GB | 20GB | 2000 |

```bash
# 修改 dm.ini
vi /dm8/dmdbms/data/DAMENG/dm.ini

# 主要参数调整
BUFFER = 4096000          # 4GB，页数为单位（4GB / 8KB）
MAX_SESSIONS = 1000        # 最大会话数
WORKER_THREADS = 4         # CPU 核数
SORT_BUF_SIZE = 200        # 排序缓冲区 MB
HJ_BUF_GLOBAL_SIZE = 200  # 哈希连接缓冲区 MB
```

## 常见问题处理

### 安装问题

**Q：安装时报错 "cannot execute binary file"**

可能是 CPU 架构不匹配，检查下载的安装包是否对应正确的 CPU 架构（x86_64 或 aarch64）。

**Q：安装时报错 "cannot open display"**

使用命令行静默安装模式：

```bash
./DMInstall.bin -q /path/to/response.ini
```

### 连接问题

**Q：disql 连接报错 "Connection failed"**

检查服务是否启动：

```bash
systemctl status DmServiceDAMENG
```

检查端口是否正确：

```bash
netstat -tlnp | grep 5236
```

**Q：连接超时**

检查防火墙设置：

```bash
firewall-cmd --list-ports
firewall-cmd --add-port=5236/tcp --permanent
firewall-cmd --reload
```

- 后台启动：`systemctl start DmServiceDAMENG`

停止方式：

- `systemctl stop DmServiceDAMENG`
- 或在 disql 中执行 `SHUTDOWN IMMEDIATE;`

**Q：达梦的默认端口是什么？如何修改？**

默认端口是 **5236**。

修改方式：

1. 初始化时指定：`./dminit port_num=5436 ...`
2. 修改 dm.ini：`PORT_NUM = 5436`
