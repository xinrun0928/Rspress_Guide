# 达梦数据库概述

<Badge text="入门" type="tip" />

## 达梦是什么？

达梦数据库管理系统（DM Database Management System，简称 DM）是武汉达梦数据库股份有限公司自主研发的国产关系型数据库。40 多年的技术积累，让达梦从最初的实验室项目成长为国内市场份额最高的国产数据库之一。

达梦的核心定位是 **Oracle 国产替代**，它的架构设计借鉴了 Oracle 的许多优秀特性，包括：

- 相同的物理存储结构
- 相似的逻辑组织方式
- 兼容的 SQL 语法
- 类似的索引机制

这种设计让 Oracle 开发者可以快速上手达梦，降低了技术迁移的成本。

## 发展历程：从实验室到信创先锋

### 起步阶段（1988-2000）

1988 年，达梦创始人开始研究关系数据库理论。1997 年，达梦推出第一个数据库产品 DM3。在此期间，达梦完成了从理论到产品的关键跨越，填补了国内数据库技术的空白。

### 成熟阶段（2000-2010）

进入 21 世纪，达梦推出了 DM5、DM6 等版本，在事务处理能力、可靠性和性能方面持续提升。这一时期，达梦在金融、电力、电信等行业开始得到应用，积累了宝贵的实践经验。

### 突破阶段（2010-2020）

2010 年，达梦推出具有里程碑意义的 DM7 版本，在兼容 Oracle 方面取得重大突破。2019 年，DM8 发布，引入更多企业级特性，包括 MPP 集群、混合负载等。

## 核心架构

达梦数据库采用分层架构设计，各层职责清晰、相互协作：

```
┌─────────────────────────────────────────────────────────────┐
│                      应用层                                 │
├─────────────────────────────────────────────────────────────┤
│                     SQL 引擎层                               │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐       │
│  │ 解析器  │  │ 优化器  │  │ 执行器  │  │ 计划器  │       │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘       │
├─────────────────────────────────────────────────────────────┤
│                     存储引擎层                               │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐       │
│  │ 事务管理 │  │  锁管理  │  │ 索引管理 │  │ 空间管理 │       │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘       │
├─────────────────────────────────────────────────────────────┤
│                     文件系统层                              │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐       │
│  │ 数据文件 │  │ 日志文件 │  │ 控制文件 │  │ 备份文件 │       │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 存储层

存储层负责数据的物理存储，包括：

- **数据文件**：存储表、索引等数据库对象
- **日志文件**：记录数据库的所有修改操作
- **控制文件**：记录数据库的物理结构信息
- **临时文件**：存储排序等临时操作的数据

### SQL 引擎层

SQL 引擎层负责处理 SQL 语句：

1. **词法分析**：将 SQL 语句分解为 token
2. **语法分析**：生成语法树
3. **语义分析**：检查表、列是否存在
4. **查询优化**：生成最优执行计划
5. **执行计划生成**：将优化后的计划交给执行器

```sql
-- SQL 执行流程示例
SELECT e.name, d.dept_name 
FROM employees e 
JOIN departments d ON e.dept_id = d.id
WHERE e.salary `>=` 10000;

-- 解析过程
-- 1. 识别关键词: SELECT, FROM, WHERE, JOIN
-- 2. 识别表: employees, departments
-- 3. 识别列: name, salary, dept_id, dept_name
-- 4. 优化: 可能选择索引扫描而非全表扫描
-- 5. 执行: 嵌套循环连接
```

### 事务管理层

事务管理层负责 ACID 特性的实现：

- **原子性**：事务要么全部成功，要么全部失败
- **一致性**：事务执行前后，数据库状态保持一致
- **隔离性**：并发事务相互隔离
- **持久性**：事务提交后，数据永久保存

### 高可用层

高可用层提供各种集群方案：

- **数据守护**：主备同步，保证数据安全
- **读写分离**：分散读压力，提高性能
- **MPP**：大规模并行处理，支持数据分析
