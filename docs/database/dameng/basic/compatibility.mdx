# 达梦兼容性对比

<Badge text="核心" type="tip" />

## 兼容性概述

凌晨三点，你正在处理一个紧急迁移任务：从 Oracle 迁移到一个新项目，必须使用国产数据库。

项目组选择了达梦。但问题是：

- 现有系统的 Oracle 存储过程能直接迁移吗？
- MySQL 的语法需要改动多少？
- 哪些特性是达梦独有的？

这些问题的答案，都在兼容性设计里。

## 达梦的兼容性策略

达梦数据库采用了「**高兼容、低风险**」的兼容性策略：

```
┌─────────────────────────────────────────────────────────┐
│                    达梦兼容性层次                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────────────────────────────────────────────┐  │
│   │              SQL 标准兼容性                       │  │
│   │        (ANSI SQL 标准，达梦完全支持)              │  │
│   └─────────────────────────────────────────────────┘  │
│                         ↓                               │
│   ┌─────────────────────────────────────────────────┐  │
│   │              Oracle 兼容性                       │  │
│   │    (PL/SQL、数据类型、系统包，达梦高度兼容)        │  │
│   └─────────────────────────────────────────────────┘  │
│                         ↓                               │
│   ┌─────────────────────────────────────────────────┐  │
│   │              MySQL 兼容性                        │  │
│   │         (常用语法，达梦兼容支持)                   │  │
│   └─────────────────────────────────────────────────┘  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

<Badge text="DM8" type="tip" />

达梦 DM8 版本在 Oracle 兼容性方面做了大量优化，目标是**零修改或最小修改**完成迁移。

## 与 Oracle 的兼容性

### 数据类型对比

达梦与 Oracle 的数据类型高度一致：

| Oracle | 达梦 | 说明 |
|---|---|---|
| `NUMBER(p,s)` | `NUMBER(p,s)` | 数值类型，完全兼容 |
| `VARCHAR2(n)` | `VARCHAR(n)` | 变长字符串，推荐使用 |
| `CHAR(n)` | `CHAR(n)` | 定长字符串 |
| `DATE` | `DATE` | 日期时间类型 |
| `TIMESTAMP` | `TIMESTAMP` | 时间戳 |
| `CLOB` | `CLOB` | 大文本对象 |
| `BLOB` | `BLOB` | 大二进制对象 |
| `ROWID` | `ROWID` | 行标识 |

```sql
-- Oracle 兼容的数据类型定义
CREATE TABLE oracle_compat (
    id NUMBER(10),                    -- 主键
    name VARCHAR2(100),               -- 字符串（达梦用 VARCHAR）
    birth_date DATE,                  -- 日期
    salary NUMBER(10,2),             -- 工资
    resume CLOB,                      -- 简历
    photo BLOB,                       -- 照片
    row_id ROWID                      -- 行标识
);

-- Oracle 的 VARCHAR2 在达梦中的使用建议
-- 达梦推荐使用 VARCHAR，VARCHAR2 被视为 VARCHAR 的别名
```

### PL/SQL 兼容性

PL/SQL 是 Oracle 的核心特性，达梦提供了**完整的 PL/SQL 支持**：

```sql
-- 存储过程
CREATE OR REPLACE PROCEDURE calc_bonus(
    p_emp_id IN INT,
    p_bonus OUT DECIMAL(10,2)
) AS
    v_salary DECIMAL(10,2);
BEGIN
    SELECT salary INTO v_salary 
    FROM employees 
    WHERE id = p_emp_id;
    
    p_bonus := v_salary * 0.2;
END;
/

-- 函数
CREATE OR REPLACE FUNCTION get_emp_count(
    p_dept_id INT
) RETURN INT AS
    v_count INT;
BEGIN
    SELECT COUNT(*) INTO v_count 
    FROM employees 
    WHERE dept_id = p_dept_id;
    
    RETURN v_count;
END;
/

-- 包
CREATE OR REPLACE PACKAGE emp_package AS
    PROCEDURE hire_employee(
        p_name VARCHAR,
        p_dept_id INT
    );
    
    FUNCTION get_dept_avg_salary(
        p_dept_id INT
    ) RETURN DECIMAL(10,2);
END emp_package;
/

CREATE OR REPLACE PACKAGE BODY emp_package AS
    PROCEDURE hire_employee(
        p_name VARCHAR,
        p_dept_id INT
    ) AS
    BEGIN
        INSERT INTO employees(name, dept_id, hire_date)
        VALUES (p_name, p_dept_id, SYSDATE);
    END;
    
    FUNCTION get_dept_avg_salary(
        p_dept_id INT
    ) RETURN DECIMAL(10,2) AS
        v_avg DECIMAL(10,2);
    BEGIN
        SELECT AVG(salary) INTO v_avg
        FROM employees
        WHERE dept_id = p_dept_id;
        
        RETURN NVL(v_avg, 0);
    END;
END emp_package;
/
```

### 系统包对比

| Oracle 包 | 达梦包 | 功能 |
|---|---|---|
| `DBMS_OUTPUT` | `DBMS_OUTPUT` | 输出信息 |
| `DBMS_LOB` | `DBMS_LOB` | 大对象操作 |
| `DBMS_UTILITY` | `DBMS_UTILITY` | 工具函数 |
| `DBMS_RANDOM` | `DBMS_RANDOM` | 随机数生成 |
| `DBMS_STATS` | `DBMS_STATS` | 统计信息 |
| `DBMS_SCHEDULER` | `DBMS_JOB` | 作业调度 |

```sql
-- DBMS_OUTPUT 使用（完全兼容）
BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello from DM!');
    DBMS_OUTPUT.PUT_LINE('Employee count: ' || 100);
END;
/

-- DBMS_RANDOM 使用（完全兼容）
BEGIN
    -- 生成随机字符串
    DBMS_OUTPUT.PUT_LINE(DBMS_RANDOM.STRING('A', 10));
    
    -- 生成随机数
    DBMS_OUTPUT.PUT_LINE(DBMS_RANDOM.VALUE(1, 100));
END;
/

-- DBMS_LOB 使用
DECLARE
    v_clob CLOB;
    v_text VARCHAR2(100);
BEGIN
    SELECT content INTO v_clob FROM articles WHERE id = 1;
    
    -- 获取 CLOB 长度
    DBMS_OUTPUT.PUT_LINE('Length: ' || DBMS_LOB.GETLENGTH(v_clob));
    
    -- 读取部分内容
    v_text := DBMS_LOB.SUBSTR(v_clob, 100, 1);
END;
/
```

### 语法兼容性

达梦支持 Oracle 的关键语法：

```sql
-- Oracle 特有的 (+) 外连接语法
-- 达梦支持，但推荐使用标准 JOIN
SELECT e.name, d.dept_name
FROM employees e, departments d
WHERE e.dept_id(+) = d.dept_id;

-- 标准写法（推荐）
SELECT e.name, d.dept_name
FROM departments d
LEFT JOIN employees e ON e.dept_id = d.dept_id;

-- 层次查询（Oracle CONNECT BY）
-- 达梦完全支持
SELECT level, emp_name, manager_id
FROM employees
CONNECT BY PRIOR emp_id = manager_id
START WITH manager_id IS NULL
ORDER SIBLINGS BY emp_name;

-- 高级分析函数
SELECT 
    emp_name,
    dept_id,
    salary,
    SUM(salary) OVER(PARTITION BY dept_id) AS dept_total,
    RANK() OVER(PARTITION BY dept_id ORDER BY salary DESC) AS dept_rank,
    ROW_NUMBER() OVER(ORDER BY hire_date) AS hire_seq
FROM employees;

--  flashback 查询（时间点回溯）
-- 达梦支持类似功能
-- SELECT * FROM employees AS OF TIMESTAMP TO_TIMESTAMP('2024-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS');
```

### 不兼容的 Oracle 特性

| 特性 | 状态 | 替代方案 |
|---|---|---|
| `REF CURSOR` | 部分兼容 | 使用达梦游标类型 |
| 分区表交换 | 兼容 | 直接使用 |
| 物化视图日志 | 兼容 | 使用达梦刷新机制 |
| 外部表 | 兼容 | 使用达梦外部表 |
| 序列的 `CACHE` | 兼容 | 默认缓存 20 |

## 与 MySQL 的兼容性

### 数据类型对比

| MySQL | 达梦 | 说明 |
|---|---|---|
| `INT` | `INT` | 整数类型 |
| `BIGINT` | `BIGINT` | 长整数 |
| `VARCHAR(n)` | `VARCHAR(n)` | 变长字符串 |
| `TEXT` | `TEXT` | 大文本 |
| `DATETIME` | `DATETIME` | 日期时间 |
| `TIMESTAMP` | `TIMESTAMP` | 时间戳 |
| `DECIMAL(p,s)` | `DECIMAL(p,s)` | 精确数值 |

```sql
-- MySQL 风格建表（达梦兼容）
CREATE TABLE mysql_compat (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 达梦中实现自增主键
CREATE TABLE dm_auto_incr (
    id INT IDENTITY(1,1) PRIMARY KEY,  -- 达梦自增语法
    name VARCHAR(100)
);

-- 达梦的序列也可以实现自增
CREATE SEQUENCE seq_users START WITH 1 INCREMENT BY 1;

CREATE TABLE users (
    id INT DEFAULT seq_users.NEXTVAL,
    name VARCHAR(100)
);
```

### 语法差异

| MySQL | 达梦 | 说明 |
|---|---|---|
| `` ` `` 反引号 | `"` 双引号或无引号 | 标识符引用 |
| `IFNULL()` | `NVL()` | 空值处理 |
| `LIMIT n, m` | `TOP m OFFSET n` | 分页查询 |
| `INSERT INTO ... ON DUPLICATE KEY` | `MERGE INTO` | 合并操作 |
| `REPLACE INTO` | `MERGE INTO` | 替换操作 |

```sql
-- MySQL 分页语法
SELECT * FROM employees LIMIT 10 OFFSET 20;

-- 达梦分页语法（兼容）
SELECT * FROM employees LIMIT 20, 10;  -- 也支持

-- 达梦标准分页
SELECT * FROM employees OFFSET 20 ROWS FETCH NEXT 10 ROWS ONLY;

-- 达梦 TOP 分页
SELECT TOP 10 * FROM employees WHERE id NOT IN (
    SELECT TOP 20 id FROM employees ORDER BY id
) ORDER BY id;

-- MySQL 的 IFNULL
SELECT IFNULL(salary, 0) FROM employees;

-- 达梦使用 NVL
SELECT NVL(salary, 0) FROM employees;

-- MySQL 的 CONCAT
SELECT CONCAT(name, '-', dept_name) FROM employees;

-- 达梦使用 || 或 CONCAT
SELECT name || '-' || dept_name FROM employees;
SELECT CONCAT(name, '-', dept_name) FROM employees;
```

### 函数差异

| MySQL | 达梦 | 说明 |
|---|---|---|
| `NOW()` | `SYSDATE` | 当前时间 |
| `DATE_FORMAT()` | `TO_CHAR()` | 日期格式化 |
| `IF(condition, a, b)` | `DECODE()` 或 `CASE` | 条件表达式 |
| `GROUP_CONCAT()` | `LISTAGG()` | 字符串聚合 |
| `FIND_IN_SET()` | `INSTR()` | 字符串查找 |

```sql
-- 日期函数对比
-- MySQL
SELECT NOW(), DATE_FORMAT(created_at, '%Y-%m-%d') FROM employees;

-- 达梦
SELECT SYSDATE, TO_CHAR(created_at, 'YYYY-MM-DD') FROM employees;

-- 条件表达式对比
-- MySQL
SELECT 
    name,
    IF(salary `>=` 10000, '高薪', IF(salary `>=` 5000, '中等', '普通')) AS level
FROM employees;

-- 达梦（DECODE）
SELECT 
    name,
    DECODE(SIGN(salary - 10000), 
        1, '高薪', 
        0, '高薪',
        DECODE(SIGN(salary - 5000),
            1, '中等',
            0, '中等',
            '普通')) AS level
FROM employees;

-- 达梦（CASE，更直观）
SELECT 
    name,
    CASE 
        WHEN salary `>=` 10000 THEN '高薪'
        WHEN salary `>=` 5000 THEN '中等'
        ELSE '普通'
    END AS level
FROM employees;
```

## 迁移注意事项

### 从 Oracle 迁移

| 注意事项 | 说明 | 处理方法 |
|---|---|---|
| VARCHAR2 | 达梦推荐用 VARCHAR | 自动转换 |
| 序列语法 | 完全兼容 | 直接使用 |
| PL/SQL | 高度兼容 | 需检查边界情况 |
| 系统包 | 大部分兼容 | 使用达梦对应包 |
| 特殊字符集 | 注意编码 | 使用 UTF-8 |

```sql
-- 迁移检查清单
-- 1. 检查不兼容的数据类型
SELECT * FROM user_tab_columns 
WHERE data_type IN ('NVARCHAR2', 'NCHAR');

-- 2. 检查存储过程依赖的系统包
SELECT name, type 
FROM user_dependencies 
WHERE referenced_name IN (
    SELECT object_name FROM user_objects 
    WHERE object_type = 'PACKAGE'
);

-- 3. 检查大对象字段
SELECT table_name, column_name, data_type
FROM user_tab_columns
WHERE data_type IN ('CLOB', 'BLOB', 'NCLOB');
```

### 从 MySQL 迁移

| 注意事项 | 说明 | 处理方法 |
|---|---|---|
| 自增主键 | 使用 IDENTITY 或序列 | 转换语法 |
| 分页语法 | 转换分页方式 | 使用 OFFSET |
| 字符串连接 | 改用 `||` | 转换函数 |
| 关键字 | 注意保留字 | 加引号 |
| 编码 | 确保一致 | UTF-8 |

```sql
-- MySQL 迁移检查清单
-- 1. 检查自增主键
SELECT table_name, column_name 
FROM user_tab_columns
WHERE column_name = 'id';

-- 2. 检查大文本字段
SELECT table_name, column_name, data_type
FROM user_tab_columns
WHERE data_type IN ('TEXT', 'LONGTEXT');

-- 3. 检查编码设置
SHOW PARAMETER LIKE '%CHAR%';
```

## 面试要点

<Badge text="面试重点" type="warning" />

### Oracle 兼容性

**Q：达梦对 Oracle 的兼容性如何？哪些特性不兼容？**

达梦对 Oracle 的兼容性很高，体现在：

1. **数据类型**：NUMBER、VARCHAR、DATE 等完全兼容
2. **PL/SQL**：存储过程、函数、包完全支持
3. **系统包**：DBMS_OUTPUT、DBMS_LOB 等常用包兼容
4. **SQL 语法**：CONNECT BY、层次查询、分析函数等支持

不兼容或部分兼容的包括：

- `REF CURSOR` 的某些用法
- Oracle 的特定 hint 语法
- 部分高级特性如 `VPD`、` Label Security`

### 迁移策略

**Q：从 Oracle 迁移到达梦，迁移策略是什么？**

迁移策略分为四个阶段：

1. **评估阶段**：分析源库对象，识别不兼容项
2. **转换阶段**：使用 DM Migration 工具转换对象
3. **验证阶段**：验证数据和业务逻辑正确性
4. **上线阶段**：制定回滚方案，灰度上线

### 性能差异

**Q：迁移后性能可能有哪些变化？**

迁移后可能出现的性能问题：

1. **统计信息**：需要重新收集统计信息
2. **执行计划**：需要检查和优化 SQL 执行计划
3. **参数调整**：根据达梦特性调整参数
4. **索引优化**：检查索引是否有效

## 总结

达梦数据库的兼容性设计让迁移变得更加简单：

- **Oracle 兼容**：数据类型、PL/SQL、系统包高度兼容
- **MySQL 兼容**：常用语法和函数支持
- **迁移工具**：提供 DM Migration 工具辅助迁移
- **兼容性参数**：可配置兼容模式

理解兼容性差异，是成功迁移的第一步。
