# 达梦数据库（DM）

你的项目要从 Oracle 迁移到国产数据库，要求零代码改造、数据不丢、业务不停。

这是所有正在推进「国产化替代」的企业面临的真实挑战。

达梦数据库（DM）正是这个场景下的主角——它是国产数据库中，Oracle 兼容性最高的存在。

## 达梦 vs Oracle：为什么迁移选达梦？

| 维度 | Oracle | 达梦 DM |
|------|--------|---------|
| 语法兼容性 | — | 高度兼容 PL/SQL |
| 存储过程 | ✅ | ✅ 兼容迁移 |
| 触发器 | ✅ | ✅ 兼容迁移 |
| 分区表 | ✅ | ✅ 兼容迁移 |
| RAC 集群 | ✅ | DM 数据守护替代 |
| 授权费用 | 昂贵 | 开源/商业可选 |
| 国密支持 | ❌ | ✅ 内置支持 |

达梦的核心价值在于：**降低从 Oracle 迁移的成本**。如果你已经精通 Oracle，学达梦的边际成本很低。

## 核心公式

```
DM 架构 = 实例（Instance） + 数据库（Database）
实例 = 内存结构（SGA/PGA）+ 后台进程
数据库 = 数据文件 + 日志文件 + 控制文件

事务保证 = MVCC（多版本） + 锁机制 + WAL（写前日志）
高可用 = 数据守护（主备同步） / MPP（分布式集群）
```
