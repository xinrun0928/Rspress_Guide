# 达梦数据库面试汇总

<Badge text="面试重点" type="warning" />

本篇文章汇总达梦数据库的高频面试题，帮助你快速把握面试重点，系统化复习核心知识点。

## 一、基础概念类

### Q1：达梦数据库的发展历程是怎样的？

达梦数据库的发展可以追溯到 1988 年，经过 40 多年的研发迭代：

| 阶段 | 时间 | 版本 | 主要特性 |
|---|---|---|---|
| 起步阶段 | 1988-2000 | DM1-DM3 | 原型研发，填补国内空白 |
| 成长阶段 | 2000-2010 | DM4-DM5 | 事务处理能力提升 |
| 成熟阶段 | 2010-2020 | DM6-DM7 | 兼容 Oracle，企业级特性 |
| 领先阶段 | 2020-至今 | DM8 | 混合事务分析，性能大幅提升 |

### Q2：达梦与 Oracle 的核心区别是什么？

| 维度 | 达梦 | Oracle |
|---|---|---|
| 研发背景 | 武汉达梦自主研发 | 甲骨文公司产品 |
| 知识产权 | 完全自主可控 | 引进消化 |
| SQL 兼容性 | 高度兼容 Oracle | 标准制定者 |
| 部署平台 | 全面支持国产芯片和操作系统 | 跨平台支持 |
| 成本 | 社区版免费 | 商业授权昂贵 |

### Q3：达梦数据库的主要版本有哪些？

- **DM8**：最新主流版本，支持混合负载
- **DM7**：成熟稳定版本，企业广泛应用

### Q5：表空间、段、区、页的关系是什么？

达梦的逻辑存储结构是层次化的：

```
表空间（Tablespace）
    ↓
段（Segment）
    ↓
区（Extent）
    ↓
页（Page）
```

- **页（Page）**：最小存储单元，大小为 8KB
- **区（Extent）**：一组连续的页，默认 16 页
- **段（Segment）**：由多个区组成，对应表、索引等对象
- **表空间（Tablespace）**：逻辑容器，管理多个段

### Q6：达梦有哪些表空间类型？

| 类型 | 说明 | 用途 |
|---|---|---|

| TEMP 表空间 | 临时数据 | 排序、哈希操作 |
| USER 表空间 | 用户数据 | 存放用户表和索引 |

## 四、事务与锁类

### Q10：达梦支持哪些事务隔离级别？

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---|---|---|---|
| 读未提交 | 可能 | 可能 | 可能 |
| 读已提交 | 不可能 | 可能 | 可能 |
| 可重复读 | 不可能 | 不可能 | 可能 |
| 串行化 | 不可能 | 不可能 | 不可能 |

<Badge text="默认" type="tip" /> 达梦默认隔离级别为**读已提交**。

### Q11：达梦的锁类型有哪些？

| 锁类型 | 说明 | 作用对象 |
|---|---|---|
| 排他锁（X） | 独占访问 | 行、页、表 |
| 共享锁（S） | 共享访问 | 行、页、表 |
| 意向锁 | 表级锁表示 | 表 |
| 行共享锁（RS） | 表级锁定 | 防止表级排他锁 |
| 行排他锁（RX） | 修改行 | 防止其他行修改 |

### Q12：什么是死锁？如何处理？

死锁是指两个或多个事务互相等待对方持有的锁，导致都无法继续执行。

**处理方式**：
- **预防**：按照固定顺序访问资源
- **检测**：达梦自动检测死锁
- **解决**：回滚其中一个事务，释放锁

## 五、集群高可用类

### Q13：什么是数据守护？

数据守护（Data Guard）是达梦提供的高可用解决方案，通过主库和备库的日志同步，保证数据一致性和系统可用性。

| 模式 | 说明 | 适用场景 |
|---|---|---|
| 实时归档 | 实时同步日志 | 最高可用性 |
| 即时归档 | 事务提交时同步 | 高可用 |
| 定期归档 | 定时同步日志 | 灾备场景 |

### Q14：主备集群的切换方式有哪些？

| 类型 | 说明 | 触发方式 |
|---|---|---|
| 自动切换 | 系统故障时自动切换 | 心跳检测 |
| 手动切换 | 管理员主动切换 | 维护操作 |
| 强制切换 | 紧急情况强制切换 | 故障恢复 |

### Q15：读写分离集群的原理是什么？

读写分离集群通过配置多个只读实例，将读请求分发到备库执行，减轻主库压力：

```sql
-- 配置读写分离
ALTER SYSTEM WRITE 'PRIMARY' NODE '主库地址';
ALTER SYSTEM WRITE 'STANDBY' NODE '备库地址';
```

| 优势 | 说明 |
|---|---|
| 负载分担 | 读请求分发到备库 |
| 性能提升 | 主库专注写操作 |
| 可用性增强 | 备库故障不影响主库 |

### Q16：MPP 集群和读写分离集群的区别？

| 维度 | MPP 集群 | 读写分离集群 |
|---|---|---|
| 架构 | 多节点并行处理 | 主从复制 |
| 扩展方式 | 水平扩展 | 增加备库 |
| 适用场景 | 大数据分析 | 事务处理 |
| 数据分布 | 分片存储 | 完整副本 |

## 面试技巧

<Badge text="加分项" type="tip" />

### 如何回答架构类问题

不要只背概念，要结合实际场景：

> **问题**：请描述达梦数据守护的原理。
>
> **普通回答**：数据守护是通过日志同步实现主备数据一致的。
>
> **加分回答**：数据守护通过主库将重做日志实时传输到备库，备库接收后应用日志实现数据同步。这种架构可以在主库故障时快速切换到备库，保证业务连续性。我们在实际项目中配置了实时归档模式，RPO 可以达到秒级。

### 如何回答对比类问题

采用「是什么、优缺点、适用场景」的结构：

> **问题**：达梦和 MySQL 有什么区别？
>
> 达梦是大型关系型数据库，定位与 Oracle 相同，支持完整的事务处理和企业级特性；MySQL 是开源数据库，定位轻量级互联网场景。达梦的索引结构、锁机制、存储结构都与 Oracle 类似，而 MySQL 使用 InnoDB 引擎，架构差异较大。在信创项目中，达梦是 Oracle 的主要替代品。

### 如何回答实操类问题

给出具体的步骤和注意事项：

> **问题**：如何配置达梦主备集群？
>
> 配置主备集群主要分为以下步骤：首先在主库配置归档，然后初始化备库并创建实例，接着配置守护进程，最后验证同步状态。关键点包括：网络配置要确保主备之间可达、日志传输模式要根据业务要求选择、切换时机要合理设置。
