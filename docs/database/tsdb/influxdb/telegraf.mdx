# Telegraf 数据采集

Telegraf 是 InfluxDB 生态中的数据采集代理，支持 200+ 输入插件。本篇文章将详细介绍 Telegraf 的使用方法和最佳实践。

## 一、Telegraf 概述

### 1.1 什么是 Telegraf

```
Telegraf:

定位: 数据采集代理

特点:
1. 插件化架构 (输入/输出/处理)
2. 200+ 输入插件
3. 50+ 输出插件
4. 轻量级 Go 语言编写
5. 配置简单
```

### 1.2 架构

```
Telegraf 架构:

┌─────────────────────────────────────────────────────────────┐
│                        Telegraf                             │
│                                                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐     │
│  │   Inputs    │ → │   Processors│ → │   Outputs    │     │
│  │  (采集插件)  │    │  (处理插件) │    │  (输出插件)  │     │
│  │             │    │             │    │             │     │
│  │  - CPU      │    │  - regex    │    │  - InfluxDB │     │
│  │  - Memory   │    │  - tag     │    │  - Kafka    │     │
│  │  - Disk     │    │  - convert │    │  - Prometheus│     │
│  │  - Nginx    │    │  - limit   │    │  - File     │     │
│  │  - MySQL    │    │             │    │  - MQTT     │     │
│  └─────────────┘    └─────────────┘    └─────────────┘     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 二、安装配置

### 2.1 安装

```bash
# macOS (Homebrew)
brew install telegraf

# Ubuntu/Debian
wget https://dl.influxdata.com/telegraf/releases/telegraf_2.0.1_amd64.deb
sudo dpkg -i telegraf_2.0.1_amd64.deb

# Docker
docker pull telegraf

# 二进制
wget https://dl.influxdata.com/telegraf/releases/telegraf-2.0.1_linux_amd64.tar.gz
tar xf telegraf-2.0.1_linux_amd64.tar.gz
```

### 2.2 基础配置

```toml
# /etc/telegraf/telegraf.conf

# 全局标签
[global_tags]
  env = "production"
  datacenter = "us-west-1"

# Agent 配置
[agent]
  interval = "10s"        # 采集间隔
  round_interval = true
  metric_batch_size = 1000
  metric_buffer_limit = 10000
  collection_jitter = "0s"
  flush_interval = "10s"
  flush_jitter = "0s"

# 输入插件 (CPU, Memory, Disk)
[[inputs.cpu]]
  percpu = true
  totalcpu = true
  collect_cpu_time = false

[[inputs.mem]]
  # 无需配置

[[inputs.disk]]
  ignore_fs = ["tmpfs", "devtmpfs", "devfs", "iso9660", "overlay", "aufs", "squashfs"]

# 输出插件 (InfluxDB)
[[outputs.influxdb]]
  urls = ["http://localhost:8086"]
  database = "telegraf"
  precision = "s"
  timeout = "5s"
```

```toml

[[inputs.tail]]

  watch_method = "inotify"

  [inputs.tail.parser]
    data_format = "grok"

# Grok 模式示例
[[inputs.tail]]

```

## 四、处理插件

### 4.1 数据转换

```toml
# 添加标签
[[inputs.processor]]
  namepass = ["cpu"]
  [inputs.processor.tags]
    environment = "production"

# 字段重命名
[[inputs.processor.rename]]
  namepass = ["disk"]
  [[inputs.processor.rename.replace]]
    from = "used"
    to = "disk_used"

# 类型转换
[[inputs.processor.converter]]
  namepass = ["nginx"]
  [inputs.processor.converter.fields]
    integer = ["status_code"]
    float = ["response_time"]
```

### 4.2 数据过滤

```toml
# 过滤标签
[[inputs.processor.defaulttagvalue]]
  namepass = ["cpu"]
  tag = "host"
  value = "unknown"

# 删除标签
[[inputs.processor.taginclude]]
  namepass = ["cpu"]
  tags = ["host", "region"]

# 正则过滤
[[inputs.processor.regex]]
  namepass = ["nginx"]
  [[inputs.processor.regex.tags]]
    key = "status"
    pattern = "^(\\d+)"
    replacement = "${1}"
    result_key = "status_code"
```

### 4.3 聚合

```toml
# 限流
[[inputs.processor.limit]]
  namepass = ["nginx"]
  drop_original = false
  limit = 100
  limit_period = "1s"
```

## 七、最佳实践

### 7.1 性能优化

```toml
# 性能调优

[agent]
  interval = "10s"           # 采集间隔
  flush_interval = "10s"      # 发送间隔
  metric_batch_size = 1000   # 批量大小
  metric_buffer_limit = 10000 # 缓冲大小
  
# 建议:
# - 采集间隔 10-30s 足够大多数监控
# - 避免过小的采集间隔
# - 合理设置 flush_interval 减少网络开销
```

### 7.2 监控自身

```toml
# 监控 Telegraf 自身
[[inputs.internal]]
  collect_memstats = true
  interval = "60s"

# 收集 Go 运行时指标
[[inputs.exec]]
  commands = ["go run ./scripts/get_goroutine.go"]
  data_format = "influx"
  interval = "60s"
```

### 7.3 故障处理

```
常见问题排查:

1. Telegraf 无法启动
   - 检查配置文件语法
   - telegraf --config /path/to/config --test

2. 数据丢失
   - 增加 metric_buffer_limit
   - 检查输出插件状态

3. CPU 使用率高
   - 减少采集间隔
   - 减少插件数量
   - 使用高效的数据格式
```

## 总结

Telegraf 是 InfluxDB 生态的重要组成部分：

1. **插件丰富**：200+ 输入，50+ 输出
2. **配置简单**：TOML 格式，易于理解和维护
3. **性能优秀**：Go 语言编写，高效采集
4. **灵活处理**：丰富的处理插件支持数据转换
5. **部署多样**：单机、集中、Kafka 中转

配合 InfluxDB 和 Grafana，Telegraf 可以构建完整的监控解决方案。
