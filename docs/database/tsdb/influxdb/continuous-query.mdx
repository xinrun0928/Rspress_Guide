# 连续查询与降采样

你的监控系统每天写入 10 亿条监控数据，但你只需要查看小时级别的趋势，分钟级和秒级的原始数据 7 天后就没用了。

手动清理太麻烦，每次查询都让数据库实时聚合又太慢。

连续查询，正是为解决这个问题而生。

## 什么是连续查询

连续查询（Continuous Query，CQ）是 InfluxDB 定期自动执行的查询，结果存储到指定的目标 Measurement。

```
┌─────────────────────────────────────────────────────────────────────┐
│                        连续查询工作原理                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  原始数据 (1秒精度)                                                  │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │  cpu,host=server01 usage=45.5  1709308800                   │  │
│  │  cpu,host=server01 usage=46.2  1709308801                   │  │
│  │  cpu,host=server01 usage=44.8  1709308802                   │  │
│  │  ... (每分钟 60 条)                                         │  │
│  │  cpu,host=server01 usage=50.1  1709308859                   │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                            │                                        │
│                            │ 连续查询 (每分钟执行一次)               │
│                            ▼                                        │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │  cpu_1m,host=server01 avg_usage=46.1  1709308860            │  │
│  │  -- 这 1 分钟的 60 条数据聚合为 1 条                         │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 核心特点

| 特点 | 说明 |
|---|---|
| 自动执行 | 根据指定的时间间隔自动运行 |
| 结果存储 | 查询结果写入目标 Measurement |
| 数据聚合 | 原始高频数据聚合为低频数据 |
| 降低存储 | 减少存储空间占用 |

## 基本语法

### 创建连续查询

```influxdb
CREATE CONTINUOUS QUERY <cq_name> ON <database>
BEGIN
  SELECT <function>(<field>) INTO <target_measurement>
  FROM <source_measurement>
  WHERE <condition>
  GROUP BY time(<interval>), <tag>
END
```

### 简单示例

```influxdb
-- 将每分钟的数据聚合为每小时的平均值
CREATE CONTINUOUS QUERY "cpu_1h_avg" ON "monitoring"
BEGIN
  SELECT mean(usage) INTO "cpu_hourly"
  FROM "cpu"
  GROUP BY time(1h), *
END
```

### 关键参数

| 参数 | 说明 | 示例 |
|---|---|---|
| `cq_name` | 连续查询名称 | `cpu_1h_avg` |
| `database` | 执行的数据库 | `monitoring` |
| `SELECT` | 聚合函数和字段 | `mean(usage)` |
| `INTO` | 目标 Measurement | `"cpu_hourly"` |
| `FROM` | 源 Measurement | `"cpu"` |
| `GROUP BY time()` | 时间分组间隔 | `time(1h)` |
| `GROUP BY *` | 保留所有 Tag | `*` |

## 聚合函数

连续查询支持多种聚合函数：

### 基础聚合

```influxdb
-- 平均值
SELECT mean(usage) INTO "cpu_hourly" FROM "cpu" GROUP BY time(1h), *

-- 总和
SELECT sum(count) INTO "requests_hourly" FROM "http_requests" GROUP BY time(1h), *

-- 最大值
SELECT max(usage) INTO "cpu_max_hourly" FROM "cpu" GROUP BY time(1h), *

-- 最小值
SELECT min(usage) INTO "cpu_min_hourly" FROM "cpu" GROUP BY time(1h), *

-- 计数
SELECT count(status) INTO "error_count_hourly" FROM "logs" WHERE status = 'error' GROUP BY time(1h), *
```

### 分位数聚合

```influxdb
-- P50 (中位数)
CREATE CONTINUOUS QUERY "latency_p50" ON "monitoring"
BEGIN
  SELECT percentile(usage, 50) INTO "latency_p50_hourly"
  FROM "request_latency"
  GROUP BY time(1h), service
END

-- P99
CREATE CONTINUOUS QUERY "latency_p99" ON "monitoring"
BEGIN
  SELECT percentile(usage, 99) INTO "latency_p99_hourly"
  FROM "request_latency"
  GROUP BY time(1h), service
END
```

### 多字段聚合

```influxdb
-- 同时聚合多个字段
CREATE CONTINUOUS QUERY "cpu_stats" ON "monitoring"
BEGIN
  SELECT mean(usage_user) AS mean_user,
         mean(usage_system) AS mean_system,
         max(usage_user) AS max_user,
         max(usage_system) AS max_system
  INTO "cpu_hourly_stats"
  FROM "cpu"
  GROUP BY time(1h), host
END
```

## 降采样实践

### 场景一：监控数据降采样

```influxdb
-- 原始数据：秒级精度
-- 保留：7 天

-- 1 分钟聚合：保留 30 天
CREATE CONTINUOUS QUERY "downsample_1m" ON "monitoring"
BEGIN
  SELECT mean(usage) AS usage,
         mean(memory) AS memory,
         mean(disk_io) AS disk_io
  INTO "metrics_1m"
  FROM "metrics_raw"
  GROUP BY time(1m), host, region
END

-- 1 小时聚合：永久保留
CREATE CONTINUOUS QUERY "downsample_1h" ON "monitoring"
BEGIN
  SELECT mean(usage) AS usage,
         mean(memory) AS memory
  INTO "metrics_1h"
  FROM "metrics_raw"
  GROUP BY time(1h), host, region
END
```

### 场景二：多级降采样

```
┌─────────────────────────────────────────────────────────────────────┐
│                        多级降采样架构                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Level 1: 原始数据 (秒级)                                           │
│  ├── 保留: 7 天                                                     │
│  └── 容量: 100GB                                                    │
│                            │                                        │
│                            ▼ 连续查询 (每分钟)                       │
│  Level 2: 1 分钟聚合                                                │
│  ├── 保留: 30 天                                                    │
│  └── 容量: 10GB                                                     │
│                            │                                        │
│                            ▼ 连续查询 (每小时)                       │
│  Level 3: 1 小时聚合                                                │
│  ├── 保留: 365 天                                                   │
│  └── 容量: 2GB                                                      │
│                            │                                        │
│                            ▼ 连续查询 (每天)                         │
│  Level 4: 1 天聚合                                                  │
│  ├── 保留: 永久                                                     │
│  └── 容量: 100MB                                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```influxdb
-- Level 1: 秒级原始数据 (自动过期 7 天)
-- 数据直接写入 metrics_raw

-- Level 2: 1 分钟聚合
CREATE CONTINUOUS QUERY "cq_downsample_1m" ON "monitoring"
BEGIN
  SELECT mean(usage) AS usage, mean(memory) AS memory
  INTO "metrics_1m" FROM "metrics_raw"
  GROUP BY time(1m), *
END

-- Level 3: 1 小时聚合
CREATE CONTINUOUS QUERY "cq_downsample_1h" ON "monitoring"
BEGIN
  SELECT mean(usage) AS usage, mean(memory) AS memory
  INTO "metrics_1h" FROM "metrics_1m"
  GROUP BY time(1h), *
END

-- Level 4: 1 天聚合
CREATE CONTINUOUS QUERY "cq_downsample_1d" ON "monitoring"
BEGIN
  SELECT mean(usage) AS usage, mean(memory) AS memory
  INTO "metrics_1d" FROM "metrics_1h"
  GROUP BY time(1d), *
END
```

### 场景三：条件聚合

```influxdb
-- 只聚合特定条件的数据
CREATE CONTINUOUS QUERY "error_rate" ON "monitoring"
BEGIN
  SELECT mean(error_count) * 100.0 / mean(total_count) AS error_rate
  INTO "error_rate_hourly"
  FROM (
    SELECT count(*) AS error_count
    FROM "http_requests"
    WHERE status >= 500
    GROUP BY time(1m), service, *
  ),
  (
    SELECT count(*) AS total_count
    FROM "http_requests"
    GROUP BY time(1m), service, *
  )
END
```

## 时间窗口对齐

### 何时执行

连续查询的执行时间由 `RESAMPLE` 子句控制：

```influxdb
-- 每 10 分钟执行一次，查询最近 10 分钟的数据
CREATE CONTINUOUS QUERY "cq_10m" ON "monitoring"
RESAMPLE EVERY 10m FOR 10m
BEGIN
  SELECT mean(usage) INTO "metrics_10m"
  FROM "metrics"
  GROUP BY time(10m), *
END
```

| 参数 | 说明 |
|---|---|
| `EVERY` | 执行间隔 |
| `FOR` | 每次查询的时间窗口 |

### 常用配置

```influxdb
-- 每 5 分钟执行，查询 5 分钟窗口
CREATE CONTINUOUS QUERY "cq_5m" ON "monitoring"
RESAMPLE EVERY 5m FOR 5m
BEGIN
  SELECT mean(usage) INTO "metrics_5m"
  FROM "metrics"
  GROUP BY time(5m), *
END

-- 每小时执行，查询 1 小时窗口（默认行为）
CREATE CONTINUOUS QUERY "cq_1h" ON "monitoring"
BEGIN
  SELECT mean(usage) INTO "metrics_1h"
  FROM "metrics"
  GROUP BY time(1h), *
END
```

## 常用函数

### 基础统计

```influxdb
SELECT
  count(*),
  distinct(*),
  mean(*),
  median(*),
  mode(*),
  spread(*),      -- max - min
  stddev(*),      -- 标准差
  sum(*)
INTO "target"
FROM "source"
GROUP BY time(*), *
```

### 选择

```influxdb
SELECT
  bottom(*, 10),  -- 最小的 10 个值
  first(*),       -- 窗口第一个值
  last(*),        -- 窗口最后一个值
  max(*),
  min(*),
  percentile(*, 99),
  sample(*, 10),  -- 随机采样 10 个
  top(*, 10)      -- 最大的 10 个值
INTO "target"
FROM "source"
GROUP BY time(*), *
```

### 转换

```influxdb
SELECT
  derivative(*),          -- 变化率
  non_negative_derivative(*),  -- 非负变化率
  difference(*),          -- 差值
  moving_average(*, 10),  -- 移动平均
  cumulative_sum(*)       -- 累计和
INTO "target"
FROM "source"
GROUP BY time(*), *
```

## 管理操作

### 查看连续查询

```influxdb
-- 列出所有连续查询
SHOW CONTINUOUS QUERIES

-- 查看特定数据库的连续查询
SHOW CONTINUOUS QUERIES ON "monitoring"
```

### 删除连续查询

```influxdb
DROP CONTINUOUS QUERY "cq_name" ON "database"
```

### 修改连续查询

InfluxDB 不支持直接修改连续查询，需要删除后重新创建：

```influxdb
-- 删除旧的
DROP CONTINUOUS QUERY "cpu_1h_avg" ON "monitoring"

-- 创建新的
CREATE CONTINUOUS QUERY "cpu_1h_avg" ON "monitoring"
BEGIN
  SELECT mean(usage) INTO "cpu_hourly_v2"
  FROM "cpu"
  GROUP BY time(1h), *
END
```

## 性能考虑

### 执行频率

| 间隔 | 适用场景 | 精度损失 |
|---|---|---|
| 10s | 实时性要求高 | 几乎无 |
| 1m | 标准监控 | 轻微 |
| 5m | 大规模数据 | 明显 |
| 1h | 历史趋势 | 明显 |

### 数据量估算

```influxdb
-- 原始数据量估算
-- 100 台服务器，每台每秒 10 个指标，保留 7 天
-- = 100 × 10 × 86400 × 7 = 604,800,000 条

-- 降采样后 (1 小时聚合)
-- = 100 × 10 × 24 × 7 = 168,000 条
-- 压缩比: 3600:1
```

### 存储策略配合

```influxdb
-- 创建降采样用的保留策略
CREATE RETENTION POLICY "one_month" ON "monitoring"
DURATION 30d
REPLICATION 1
SHARD DURATION 1h

-- 连续查询写入到新 RP
CREATE CONTINUOUS QUERY "cq_downsample" ON "monitoring"
BEGIN
  SELECT mean(usage) INTO "monitoring"."one_month"."metrics_1h"
  FROM "metrics"
  GROUP BY time(1h), *
END
```

## 常见问题

### 1. 数据重复

```influxdb
-- 原因：连续查询和原始数据的时间窗口重叠
-- 解决：使用 INTO 写入到不同的 Measurement
```

### 2. 数据丢失

```influxdb
-- 原因：查询窗口太小
-- 解决：增大 FOR 窗口
RESAMPLE EVERY 5m FOR 10m
```

### 3. 查询超时

```influxdb
-- 原因：数据量太大
-- 解决：
-- 1. 提高 RP 的 SHARD DURATION
-- 2. 增加查询执行频率
-- 3. 减少每次查询的数据范围
```

## 总结

连续查询是 InfluxDB 实现数据降采样的核心机制：

1. **自动执行**：根据设定的时间间隔自动运行
2. **聚合存储**：将聚合结果写入目标 Measurement
3. **多级降采样**：建立数据金字塔，减少存储占用
4. **配合 RP**：与保留策略配合，实现数据自动过期
5. **灵活配置**：支持多种聚合函数和查询条件

合理使用连续查询，可以在保留数据价值的同时，大幅降低存储成本。
