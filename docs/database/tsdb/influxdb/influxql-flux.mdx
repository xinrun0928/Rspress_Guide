# InfluxQL 与 Flux 查询语言

你的监控面板上有个奇怪的需求：需要同时展示过去 1 小时 CPU 使用率的平均值，以及与上周同期的环比变化。

用 InfluxQL？一个查询搞定平均值，另一个查询拿上周数据，然后在代码里算变化率。

用 Flux？一个查询，全部搞定。

InfluxDB 提供了两种查询语言：InfluxQL 和 Flux。它们各有特点，适用于不同场景。

## InfluxQL：类 SQL 的查询语言

InfluxQL 是 InfluxDB 最早提供的查询语言，语法类似于 SQL，易于学习和使用。

### 基础查询

```txt
-- 选择所有字段
SELECT * FROM "cpu"

-- 选择特定字段
SELECT "usage", "user" FROM "cpu"

-- 带条件过滤
SELECT * FROM "cpu"
WHERE "host" = 'server01'
  AND "time" > now() - 1h

-- 限制返回条数
SELECT * FROM "cpu" LIMIT 10
```

### 时间范围

```txt
-- 最近 1 小时
SELECT * FROM "cpu" WHERE time > now() - 1h

-- 指定时间范围
SELECT * FROM "cpu"
WHERE time > '2024-01-01T00:00:00Z'
  AND time < '2024-01-02T00:00:00Z'

-- 相对时间
SELECT * FROM "cpu" WHERE time > now() - 7d
```

:::tip 时间单位
| 单位 | 说明 | 示例 |
|---|---|---|
| `u` 或 `us` | 微秒 | `1u` |
| `ms` | 毫秒 | `1ms` |
| `s` | 秒 | `1s` |
| `m` | 分钟 | `1m` |
| `h` | 小时 | `1h` |
| `d` | 天 | `1d` |
| `w` | 周 | `1w` |
:::

### 聚合查询

```txt
-- 按时间分组聚合
SELECT MEAN("usage") FROM "cpu"
WHERE time > now() - 1h
GROUP BY time(5m), "host"

-- 按标签分组
SELECT MEAN("usage") FROM "cpu"
WHERE time > now() - 1h
GROUP BY "region", "host"

-- 多个聚合函数
SELECT MEAN("usage"), MAX("usage"), MIN("usage")
FROM "cpu"
WHERE time > now() - 1h
GROUP BY time(5m)
```

### 常用聚合函数

| 函数 | 说明 | 示例 |
|---|---|---|
| `COUNT()` | 计数 | `COUNT(*)` |
| `MEAN()` | 平均值 | `MEAN(usage)` |
| `MEDIAN()` | 中位数 | `MEDIAN(usage)` |
| `SUM()` | 求和 | `SUM(count)` |
| `MIN()` | 最小值 | `MIN(usage)` |
| `MAX()` | 最大值 | `MAX(usage)` |
| `STDDEV()` | 标准差 | `STDDEV(usage)` |
| `PERCENTILE()` | 百分位数 | `PERCENTILE(usage, 99)` |
| `DIFFERENCE()` | 差值 | `DIFFERENCE(usage)` |
| `DERIVATIVE()` | 变化率 | `DERIVATIVE(usage)` |

### 数学运算

```txt
-- 基础运算
SELECT "usage" / 100 FROM "cpu"           -- 除法
SELECT "usage" * 100 FROM "cpu"           -- 乘法
SELECT "usage" + "user" FROM "cpu"        -- 加法

-- 百分比计算
SELECT ("usage" - "idle") / "usage" * 100 FROM "cpu"

-- 组合使用
SELECT MEAN("usage") * 100 FROM "cpu"
WHERE time > now() - 1h
GROUP BY time(5m)
```

### 填充空值

```txt
-- 默认不填充
SELECT MEAN("usage") FROM "cpu"
WHERE time > now() - 1h
GROUP BY time(5m)

-- 用前一个值填充
SELECT MEAN("usage") FROM "cpu"
WHERE time > now() - 1h
GROUP BY time(5m)
FILL(previous)

-- 用指定值填充
SELECT MEAN("usage") FROM "cpu"
WHERE time > now() - 1h
GROUP BY time(5m)
FILL(0)

-- 线性填充
SELECT MEAN("usage") FROM "cpu"
WHERE time > now() - 1h
GROUP BY time(5m)
FILL(linear)
```

### 嵌套查询

```txt
-- 先聚合再计算
SELECT MEAN("mean_usage") FROM (
  SELECT MEAN("usage") AS "mean_usage"
  FROM "cpu"
  WHERE time > now() - 1h
  GROUP BY time(1m), *
)
WHERE time > now() - 30m
```

### 正则表达式

```txt
-- 匹配 Measurement
SELECT * FROM /cpu.*/

-- 匹配 Tag 值
SELECT * FROM "cpu"
WHERE "host" =~ /server\d+/

-- 排除 Tag 值
SELECT * FROM "cpu"
WHERE "host" !~ /test.*/
```

## Flux：函数式数据脚本语言

Flux 是 InfluxDB 2.0 主推的查询语言，功能更强大，但也更复杂。

### 基础语法

Flux 的核心是**管道式编程**，每个函数处理数据后传递给下一个函数：

```javascript
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> filter(fn: (r) => r.host == "server01")
  |> mean()
```

### 数据源

```javascript
// 从指定数据库查询
from(bucket: "monitoring")
  |> range(start: -1h)

// 从指定时间范围查询
from(bucket: "monitoring")
  |> range(start: 2024-01-01T00:00:00Z, stop: 2024-01-02T00:00:00Z)

// 查询最近的数据
from(bucket: "monitoring")
  |> range(start: -7d)
```

### 过滤器

```javascript
// 过滤 Measurement
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")

// 过滤 Tag
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r.host == "server01")

// 多个过滤条件
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) =>
    r._measurement == "cpu" and
    (r.host == "server01" or r.host == "server02") and
    r.region == "us-west"
  )
```

### 时间窗口

```javascript
// 按时间窗口聚合
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> aggregateWindow(every: 5m, fn: mean)

// 使用 group by
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> group(columns: ["host", "region"])
  |> mean()
```

### 聚合与转换

```javascript
// 平均值
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> mean()

// 多个聚合
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> reduce(
    identity: {count: 0.0, sum: 0.0, min: 0.0, max: 0.0},
    fn: (r, accumulator) => ({
      count: accumulator.count + 1.0,
      sum: accumulator.sum + r._value,
      min: if accumulator.count == 0.0 then r._value else min(min: accumulator.min, max: r._value),
      max: if accumulator.count == 0.0 then r._value else max(min: accumulator.max, max: r._value)
    })
  )
```

### 高级功能

#### join 连接

```javascript
// 多个数据源的 Join
cpu = from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> mean()

memory = from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "memory")
  |> mean()

join(tables: {cpu: cpu, memory: memory}, on: ["host", "_time"])
```

#### union 合并

```javascript
us_west = from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu" and r.region == "us-west")

us_east = from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu" and r.region == "us-east")

union(tables: [us_west, us_east])
  |> mean()
```

#### 移动平均

```javascript
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> movingAverage(n: 10)  // 10 个点的移动平均
```

#### 变化率

```javascript
// 差值
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> difference()

// 导数 (变化率)
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> derivative()
```

## InfluxQL vs Flux 对比

| 维度 | InfluxQL | Flux |
|---|---|---|
| **语法风格** | SQL-like | 函数式脚本 |
| **学习曲线** | 低 | 较高 |
| **灵活性** | 中等 | 高 |
| **适用场景** | 简单查询 | 复杂处理 |
| **跨Measurement查询** | 不支持 | 支持 |
| **Join操作** | 不支持 | 支持 |
| **自定义函数** | 不支持 | 支持 |
| **性能** | 较好 | 略低 |

### InfluxQL 适用场景

```txt
-- 简单的时间范围查询
SELECT * FROM "cpu" WHERE time > now() - 1h

-- 基础聚合
SELECT MEAN("usage") FROM "cpu" GROUP BY time(5m), "host"

-- 带条件的过滤
SELECT * FROM "http_requests"
WHERE status >= 500 AND time > now() - 1d
GROUP BY "service"
```

### Flux 适用场景

```javascript
// 跨多个 Measurement 查询
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) =>
    r._measurement == "cpu" or
    r._measurement == "memory"
  )
  |> mean()

// 复杂的时序计算
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> map(fn: (r) => ({
    r with
    anomaly: if r._value > 90.0 then true else false
  }))
```

## 实践建议

### 1. 简单查询用 InfluxQL

```txt
-- ✅ InfluxQL 足够
SELECT MEAN(usage) FROM cpu
WHERE time > now() - 1h
GROUP BY time(5m), host
```

### 2. 复杂处理用 Flux

```javascript
// 需要自定义处理时用 Flux
from(bucket: "monitoring")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> map(fn: (r) => ({
    r with
    percent_used: r.usage / r.total * 100.0
  }))
  |> filter(fn: (r) => r.percent_used > 80.0)
```

### 3. 时间范围尽量精确

```txt
-- ❌ 全表扫描
SELECT * FROM "cpu"

-- ✅ 限定范围
SELECT * FROM "cpu"
WHERE time > now() - 1h
  AND time < now()
```

### 4. 避免 SELECT *

```txt
-- ❌ 全字段
SELECT * FROM "cpu"

-- ✅ 指定字段
SELECT "usage", "user" FROM "cpu"
```

## 总结

InfluxQL 和 Flux 是 InfluxDB 提供的两种查询语言：

1. **InfluxQL**：类 SQL 语法，学习成本低，适合简单查询
2. **Flux**：函数式编程，灵活强大，适合复杂场景

选择建议：
- 简单的时间聚合查询用 InfluxQL
- 需要 Join、Union、自定义逻辑时用 Flux
- InfluxDB 2.0 推荐学习 Flux，它将是未来

两者的目标都是让时序数据的查询变得简单，只是走了不同的路径。
