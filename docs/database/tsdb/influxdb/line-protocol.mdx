# Line Protocol 数据写入格式

凌晨 2 点，你的监控数据采集脚本在向 InfluxDB 写入数据时神秘失败了。日志里只有一行错误：`invalid tag format`。

你排查了半天，发现问题出在标签值里有个意外的空格。

这就是 Line Protocol——InfluxDB 的数据写入格式。看起来简单，但细节满满。

## Line Protocol 基本格式

Line Protocol 是 InfluxDB 最核心的写入协议，所有的数据写入最终都转换成这个格式：

```
<measurement>[,<tags> ]<fields> <timestamp>
```

| 组成部分 | 是否必须 | 说明 |
|---|---|---|
| `measurement` | 必须 | 测量名，必填 |
| `tags` | 可选 | 标签集，逗号分隔，键值对用 `=` |
| `fields` | 必须 | 字段集，空格分隔，键值对用 `=` |
| `timestamp` | 可选 | 纳秒时间戳，不填则用服务器当前时间 |

:::tip 空格的重要性
Line Protocol 中有严格的空格分隔规则：
- Tags 和 Fields 之间用**单个空格**分隔
- Fields 和 Timestamp 之间用**单个空格**分隔
- Tags 内部的逗号后**不要有空格**

`cpu,host=server01 usage=45.5 1709308800000000000` ✅
`cpu,host=server01, usage=45.5 1709308800000000000` ❌ (逗号后有空格)
:::

## 格式详解

### Measurement 测量名

测量名定义数据的类型或来源：

```
cpu
temperature
http_requests
stock_prices
```

命名规则：
- 大小写敏感
- 可以包含字母、数字、下划线
- 不能单独使用下划线 `_`（保留给系统）
- 建议使用小写和下划线分隔

### Tags 标签集

Tags 是可选的维度信息，用于过滤和分组：

```
cpu,host=server01,region=us-west
       │           │
       └── Tag Set │
                   └── Tag: host=server01,region=us-west
```

单个 Tag：`host=server01`
多个 Tags：逗号分隔 `host=server01,region=us-west`

:::warning Tag 值的限制
- Tag Key 和 Tag Value 都必须是字符串
- Tag Value 不能包含逗号 `,`、空格 ` `、等号 `=`
- 如果包含这些字符，需要转义或用双引号包裹

```influxdb
-- ❌ 错误：Tag Value 包含逗号
cpu,hosts=server01,server02 usage=45.5

-- ✅ 正确：使用转义
cpu,hosts="server01\,server02" usage=45.5

-- ✅ 正确：使用双引号
cpu,hosts="server01" usage=45.5
```
:::

### Fields 字段集

Fields 存储实际的指标值：

```
cpu usage=45.5,user=30.2
        │       │
        └── Field Set │
                       └── Field: usage=45.5,user=30.2
```

单个 Field：`usage=45.5`
多个 Fields：逗号分隔 `usage=45.5,user=30.2`

Field Value 支持多种类型：

| 类型 | 示例 | 说明 |
|---|---|---|
| Float | `usage=45.5` | 64 位浮点数 |
| Integer | `count=100i` | 64 位整数，需加 `i` 后缀 |
| String | `status="error"` | 字符串需用双引号 |
| Boolean | `active=true` | true/false，需要双引号 |

:::tip Integer 类型必须加 i 后缀
```influxdb
-- ✅ 正确：整数加 i 后缀
request_count=1000i

-- ❌ 错误：整数没有后缀，会被当作 Float
request_count=1000
```
:::

### Timestamp 时间戳

Timestamp 是可选的纳秒级时间戳：

```
cpu usage=45.5 1709308800000000000
```

如果不指定，使用服务器当前时间。常见的时间戳精度：

| 精度 | 长度 | 示例 |
|---|---|---|
| 纳秒 | 19 位 | `1709308800000000000` |
| 微秒 | 16 位 | `1709308800000000` |
| 毫秒 | 13 位 | `1709308800000` |
| 秒 | 10 位 | `1709308800` |

## 完整示例

### 单条数据点

```influxdb
cpu,host=server01,region=us-west usage=45.5,user=30.2 1709308800000000000
```

分解：

```
cpu                    ← Measurement
,host=server01         ← Tag 1
,region=us-west        ← Tag 2
 usage=45.5            ← Field 1 (空格分隔)
,user=30.2             ← Field 2 (逗号分隔)
 1709308800000000000   ← Timestamp (空格分隔)
```

### 多条数据点

多条数据点用换行分隔：

```influxdb
cpu,host=server01,region=us-west usage=45.5,user=30.2 1709308800000000000
cpu,host=server02,region=us-west usage=30.1,user=15.8 1709308800000000000
cpu,host=server01,region=us-east usage=50.2,user=25.0 1709308800000000000
cpu,host=server02,region=us-east usage=35.7,user=18.4 1709308800000000000
```

## 写入方式

### HTTP API

```bash
curl -i -XPOST "http://localhost:8086/write?db=mydb" \
  --data-binary 'cpu,host=server01 usage=45.5,user=30.2 1709308800000000000'
```

带精度参数：

```bash
curl -i -XPOST "http://localhost:8086/write?db=mydb&rp=one_week&precision=s" \
  --data-binary 'cpu,host=server01 usage=45.5 1709308800'
```

参数说明：

| 参数 | 说明 |
|---|---|
| `db` | 数据库名（必填） |
| `rp` | 保留策略（可选） |
| `precision` | 时间戳精度：`ns`、`u`、`ms`、`s`、`m`、`h` |

### 批量写入

```bash
curl -i -XPOST "http://localhost:8086/write?db=mydb" \
  --data-binary '
cpu,host=server01,region=us-west usage=45.5,user=30.2 1709308800000000000
cpu,host=server02,region=us-west usage=30.1,user=15.8 1709308800000000000
cpu,host=server03,region=us-east usage=50.2,user=25.0 1709308800000000000
cpu,host=server01,region=us-east usage=35.7,user=18.4 1709308800000000000
'
```

:::tip 批量写入优化
批量写入可以显著提升性能：
- 减少网络往返次数
- 减少客户端和服务端的交互开销
- 建议每批次 5000-10000 条数据
:::

### CLI 写入

```bash
# 单条写入
influx -execute "INSERT cpu,host=server01,region=us-west usage=45.5,user=30.2"

# 指定时间戳
influx -execute "INSERT cpu,host=server01,region=us-west usage=45.5 1709308800" -precision s
```

### 客户端库写入

```python
from influxdb import InfluxDBClient

client = InfluxDBClient('localhost', 8086, 'root', 'root', 'mydb')

# 单条数据
json_body = [
    {
        "measurement": "cpu",
        "tags": {
            "host": "server01",
            "region": "us-west"
        },
        "fields": {
            "usage": 45.5,
            "user": 30.2
        },
        "time": "1709308800000000000"
    }
]

client.write_points(json_body)
```

```java
import influxdb.InfluxDB;
import influxdb.InfluxDBFactory;
import influxdb.dto.Point;
import java.util.concurrent.TimeUnit;

InfluxDB influxDB = InfluxDBFactory.connect("http://localhost:8086", "root", "root");

Point point = Point.measurement("cpu")
    .tag("host", "server01")
    .tag("region", "us-west")
    .addField("usage", 45.5)
    .addField("user", 30.2)
    .time(1709308800L, TimeUnit.SECONDS)
    .build();

influxDB.write("mydb", "autogen", point);
```

## 特殊字符处理

### 逗号

Tag Key/Value 中包含逗号需要转义：

```influxdb
-- Tag Value 包含逗号
events,action=click,button="Submit,Form" count=1

-- Field Key 包含逗号
events,type=click "user,count"=100
```

### 空格

Tag Value 中包含空格需要双引号：

```influxdb
-- Tag Value 包含空格
cpu,location="us west 1" usage=45.5

-- Field Value 包含空格（字符串类型）
log,level=error message="Failed to connect" timestamp=1709308800
```

### 等号

Field Value 中包含等号需要双引号：

```influxdb
-- Field Value 是字符串且包含 =
log message="status=200 OK" 1709308800
```

### 双引号

如果字符串中包含双引号，需要转义：

```influxdb
-- Field Value 包含双引号
log message="Say \"Hello\" to the world" 1709308800
```

## 常见错误与排查

### 1. invalid tag format

```bash
# 错误：逗号后有空格
cpu,host =server01 usage=45.5

# 错误：空格分隔符位置不对
cpu usage=45.5 host=server01
```

### 2. partial write success

```bash
# 部分数据写入成功，部分失败
# 检查返回的错误信息，确认哪些行失败
```

### 3. timestamp out of range

```bash
# 时间戳超出范围
# InfluxDB 支持的时间范围：1677-09-21 ~ 2262-04-11
```

### 4. field type conflict

```bash
# 同一个 Field Key 在不同时间点写了不同类型的数据
cpu,host=server01 usage=45.5      -- Float
cpu,host=server01 usage="error"   -- String ❌ 冲突
```

## 性能优化建议

### 1. 批量写入

```python
# 一次性写入大量数据
points = []
for i in range(10000):
    point = {
        "measurement": "cpu",
        "tags": {"host": f"server{i % 100}"},
        "fields": {"usage": random.random() * 100},
        "time": current_timestamp + i
    }
    points.append(point)

client.write_points(points)  # 批量写入
```

### 2. 合理使用 Tags

```influxdb
-- ✅ 推荐：低基数的 Tag
cpu,host=server01,region=us-west

-- ❌ 避免：高基数的 Tag
events,user_id=12345
```

### 3. 选择合适的时间戳精度

```bash
# 毫秒精度就够用时，不要用纳秒
# 纳秒：1709308800000000000 (19 字符)
# 毫秒：1709308800000 (13 字符)
# 节省 30% 存储空间
```

### 4. 控制单次写入大小

```bash
# 单次 HTTP 请求建议不超过 5MB
```

## 总结

Line Protocol 是 InfluxDB 数据写入的核心：

1. **基本格式**：`measurement,tag_key=tag_value field_key=field_value timestamp`
2. **Tags vs Fields**：Tags 索引化用于过滤，Fields 存储实际数值
3. **Timestamp**：可选，默认纳秒精度，可指定其他精度
4. **批量写入**：显著提升写入性能
5. **特殊字符**：逗号、等号、空格需要正确处理

掌握 Line Protocol，是熟练使用 InfluxDB 的基础。
