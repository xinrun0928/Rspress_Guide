# InfluxDB

InfluxDB 是目前最流行的开源时序数据库之一。

它的优势在于**功能全面**——数据保留策略、连续查询、降采样、Telegraf 生态，几乎你想得到的时序场景它都能 cover。

但它的弱点也很明显：**集群版收费**，开源版只能单机。

## Tag vs Field：最核心的设计决策

这是 InfluxDB 最重要的设计决策，很多初学者都会搞错：

| 特性 | Tag（标签） | Field（字段） |
|------|------------|--------------|
| 索引 | ✅ 自动索引 | ❌ 不索引 |
| 存储类型 | 只能是字符串 | 任意类型（数值、字符串、布尔） |
| 查询性能 | 快（可过滤、可分组） | 慢（全表扫描） |
| 适用场景 | 过滤条件、分组维度 | 存储实际数据值 |

```influxql
# 正确设计：
# host 是 Tag（用于过滤和分组）
# cpu、memory 是 Field（存储数据值）
cpu_monitor,host=server01,region=us-east cpu=45.2,memory=8192 1709808000000000000

# 错误设计：
# user_id 唯一值太多，作为 Tag 会导致 Series 爆炸
user_behavior,user_id=12345 action=click value=1 1709808000000000000
```

**记住**：Tag 用于**过滤和分组**（低基数），Field 用于**存储数据值**（高基数）。

## 核心公式

```
InfluxDB 数据模型：
Point（数据点）= Measurement + Tags + Fields + Timestamp
Series（时间线）= Measurement + Tag 组合

Line Protocol：
measurement,tag1=value1,tag2=value2 field1=value1,field2=value2 timestamp

写入格式：
cpu_monitor,host=server01,region=us-east usage=45.2 1709808000000000000
        ①              ②                            ③              ④

① Measurement：测量名称
② Tag Set：标签集（逗号分隔，= 连接）
③ Field Set：字段集（逗号分隔，= 连接）
④ Timestamp：纳秒时间戳（可选）

InfluxQL vs Flux：
  InfluxQL = 类 SQL，简单易学，适合简单查询
  Flux = 脚本语言，灵活性高，适合复杂数据处理

存储架构：
  WAL → MemTable → TSM File → Compaction → 多版本 TSM
```
