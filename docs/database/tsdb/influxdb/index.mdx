# InfluxDB 总览

InfluxDB 是目前最流行的开源时序数据库之一，由 InfluxData 公司开发。它以其丰富的功能、强大的查询语言和活跃的社区赢得了广泛应用。

本模块将带你深入了解 InfluxDB 的核心特性和使用方法。

## 一、InfluxDB 概述

### 1.2 InfluxDB 版本

```
InfluxDB 版本对比:

InfluxDB 1.x:
- 开源版: 单机，功能完整
- InfluxDB Enterprise: 集群，商业版
- 查询语言: InfluxQL (类似 SQL)

InfluxDB 2.x:
- 开源版: 单机 + Cloud API
- InfluxDB Cloud: 全托管云服务
- 查询语言: Flux (脚本语言)
- 特点: 统一了 TICK 组件

InfluxDB OSS (开源) vs Cloud:
- OSS: 完全免费，自托管
- Cloud: 按使用量计费，无需运维
```

### 1.3 TICK 技术栈

```
TICK 技术栈:

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Telegraf (采集) ──→ InfluxDB (存储) ──→ Chronograf (可视化) │
│         │                           │                       │
│         │ Kapacitor                 │                       │
│         │ (告警) ───────────────────┘                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘

组件:
- Telegraf: 数据采集代理，支持 200+ 输入插件
- InfluxDB: 时序数据库
- Chronograf: 可视化和告警界面
- Kapacitor: 流式处理和告警引擎
```

## 二、核心特性

### 2.1 高写入吞吐

```
InfluxDB 写入性能:

测试环境: 4核 / 16GB 内存 / SSD

写入吞吐量:
- 单次 Batch: 100K - 500K 点/秒
- 并发写入: 可达 2M+ 点/秒

性能优化:
1. Batch 写入 (减少网络往返)
2. 异步写入 (不阻塞)
3. Line Protocol 简洁高效
```

### 2.2 数据压缩

```
压缩技术:

1. 时间戳压缩
   - Delta Encoding
   - Delta of Delta

2. 数值压缩
   - Gorilla XOR Encoding
   - 无损压缩

3. 标签压缩
   - 字典编码
   - 相同标签值只存储一次

压缩效果:
- 原始数据: 100 GB
- 压缩后: 10-20 GB
- 压缩率: 5-10x
```

### 2.3 保留策略

```
保留策略 (Retention Policy, RP):

CREATE RETENTION POLICY "one_day"
ON "mydb"
DURATION 1d
REPLICATION 1
SHARD DURATION 1h

含义:
- 名称: one_day
- 保留时间: 1 天
- 副本数: 1 (单机)
- 分片时长: 1 小时
```

## 三、数据写入

### 3.1 Line Protocol

```
Line Protocol 格式:

<measurement>,<tags> <fields> <timestamp>

示例:

cpu,host=server1,region=us-west usage=45.5,user=20 1705305600000000000

解释:
- Measurement: cpu
- Tags: host=server1,region=us-west (逗号分隔)
- Fields: usage=45.5,user=20 (空格分隔)
- Timestamp: 1705305600000000000 (纳秒)
```

### 3.2 写入方式

```bash
# 方式 1: HTTP API
curl -i -XPOST "http://localhost:8086/write?db=mydb" \
  --data-binary 'cpu,host=server1,region=us-west usage=45.5 1705305600000000000'

# 方式 2: CLI
influx -execute "INSERT cpu,host=server1,region=us-west usage=45.5"

# 方式 3: 客户端库
# Python, Java, Go, Node.js 等
```

### 3.3 批量写入

```python
# Python 批量写入示例
from influxdb import InfluxDBClient

client = InfluxDBClient('localhost', 8086, 'root', 'root', 'mydb')

# 构造批量数据
points = [
    {
        "measurement": "cpu",
        "tags": {"host": "server1", "region": "us-west"},
        "fields": {"usage": 45.5, "user": 20},
        "time": 1705305600000000000
    },
    {
        "measurement": "cpu",
        "tags": {"host": "server2", "region": "us-east"},
        "fields": {"usage": 30.1, "user": 15},
        "time": 1705305600000000000
    }
]

# 批量写入
client.write_points(points)
```

## 四、数据查询

### 4.1 InfluxQL 基础

```sql
-- 基础查询
SELECT * FROM cpu WHERE time > now() - 1h;

-- 带条件过滤
SELECT * FROM cpu
WHERE host = 'server1'
AND time > now() - 24h;

-- 聚合查询
SELECT MEAN(usage) FROM cpu
WHERE time > now() - 1h
GROUP BY time(5m), host;
```

### 4.2 Flux 查询

```javascript
// Flux 查询示例
from(bucket: "mydb")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "cpu")
  |> filter(fn: (r) => r.host == "server1")
  |> aggregateWindow(every: 5m, fn: mean)
```

## 总结

InfluxDB 是功能完整的时序数据库：

1. **TICK 栈**：完整的数据采集、存储、分析、可视化解决方案
2. **高吞吐**：支持百万级点/秒写入
3. **高压缩**：5-10x 压缩率
4. **双查询语言**：InfluxQL (SQL-like) 和 Flux (脚本语言)
5. **生态丰富**：Telegraf 200+ 插件，Grafana 原生支持

适合需要完整时序数据处理能力的企业级应用。
