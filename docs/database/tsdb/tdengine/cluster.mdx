# TDengine 集群架构

你的物联网平台从 1 万台设备增长到 100 万台设备，单节点 TDengine 开始力不从心。

写入延迟从 1ms 飙升到 50ms，查询超时频发。

TDengine 的水平扩展架构，让你在不改变代码的情况下，线性扩展集群容量和性能。

## 核心组件

### vnode（虚拟节点）

vnode 是 TDengine 的数据存储和计算单元：

| 组件 | 说明 |
|---|---|
| **数据存储** | 负责存储数据的 LSM-Tree 结构和文件 |
| **数据计算** | 本地执行聚合、过滤等计算 |
| **Replication** | 支持多副本，保证高可用 |
| **分区** | 按时间分区管理数据 |

每个 vnode 只属于一个数据库，但一个数据库可以有多个 vnode 分布在不同节点上。

### mnode（管理节点）

mnode 负责集群管理：

| 功能 | 说明 |
|---|---|
| **元数据管理** | 管理数据库、表、用户等元数据 |
| **集群协调** | 协调数据分布和副本同步 |
| **路由管理** | 管理数据分布映射 |

mnode 至少需要 3 个节点以保证高可用，通常是奇数个。

### dnode（数据节点）

dnode 是物理节点，一个物理服务器运行一个 dnode：

```bash
# 启动 dnode
taosd -c /etc/taos
```

### qnode（查询节点）

qnode 专门处理查询请求（可选）：

```sql
-- 创建查询节点
CREATE QNODE ON DNODE "dnode_1";
```

### 创建集群数据库

```sql
-- 创建高可用数据库
CREATE DATABASE power_db
  REPLICA 3              -- 3 副本
  QUORUM 2               -- 写入确认数
  DAYS 10                -- 每个分片 10 天
  KEEP 365               -- 保留 365 天
  BUFFER 256             -- 写入 Buffer 256MB
  COMPRESSION 'ZSTD'     -- ZSTD 压缩
  WAL_LEVEL 2           -- WAL 持久化
  VGROUPS 3;             -- 3 个 vgroup
```

| 参数 | 说明 | 建议 |
|---|---|---|
| `REPLICA` | 副本数 | 3（数据安全） |
| `QUORUM` | 写入确认数 | 2（多数确认） |
| `VGROUPS` | vgroup 数量 | 建议 `>=` 节点数 |
| `KEEP` | 数据保留天数 | 根据业务需求 |
| `COMPRESSION` | 压缩算法 | ZSTD（高压缩） |

### 手动均衡

```sql
-- 创建新的 dnode
CREATE DNODE "192.168.1.105:6030";

-- 迁移 vnode 到新节点
BALANCE VGROUP ON DATABASE power_db;

-- 迁移所有 vgroup
BALANCE VGROUP;
```

## 节点管理

### 添加节点

```sql
-- 1. 在新服务器启动 taosd
taosd -c /etc/taos

-- 2. 在集群中添加节点
CREATE DNODE "192.168.1.105:6030";

-- 3. 查看节点状态
SHOW DNODES;
```

### 移除节点

```sql
-- 1. 将数据迁移到其他节点
BALANCE VGROUP ON DNODE "192.168.1.104:6030";

-- 2. 确认迁移完成
SHOW VGROUPS;

-- 3. 删除节点
DROP DNODE "192.168.1.104:6030";
```

### 节点状态监控

```sql
-- 查看所有节点状态
SHOW DNODES;

-- 查看所有 vgroup 分布
SHOW VGROUPS;

-- 查看数据库详情
SHOW power_db.VGROUPS;
```

## 总结

TDengine 集群架构的关键点：

1. **vnode 是核心**：每个 vnode 是独立的数据存储和计算单元
2. **mnode 管理集群**：负责元数据和协调工作
3. **多副本保证高可用**：副本数 `>=` 3，QUORUM `>=` 2
4. **自动负载均衡**：新增节点后可自动迁移数据
5. **线性扩展**：增加节点即可提升容量和性能
6. **查询节点可选**：复杂查询场景可添加 qnode 分担负载

集群部署时，合理规划副本数和 vgroup 数量，是保证系统稳定性和性能的关键。
