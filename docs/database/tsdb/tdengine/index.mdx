# TDengine 总览

你的 IoT 项目里有 100 万个传感器，每个传感器每秒采集 10 个指标。一天的数据量是多少？

100 万 `×` 10 `×` 86400 `=` 8640 亿条。

用 MySQL？一张表根本装不下，分库分表后查询复杂得像噩梦。

InfluxDB？单节点能抗住吗？集群版要花钱。

TDengine，专为物联网海量时序数据而生。

## TDengine 是什么

TDengine 是涛思数据开发的国产时序数据库（Time-Series Database），专为物联网、车联网、工业互联网等场景优化。

## 核心概念

### 超级表（Super Table）

超级表是 TDengine 的核心概念，定义了某一类设备的数据模型模板：

```sql
-- 创建超级表：定义所有发电机的数据模型
CREATE STABLE power_meter (
    ts TIMESTAMP,           -- 时间戳
    voltage FLOAT,          -- 电压
    current FLOAT,          -- 电流
    power FLOAT,            -- 功率
    energy FLOAT            -- 电能
) TAGS (
    meter_id INT,           -- 电表 ID
    location VARCHAR(100),  -- 位置
    manufacturer VARCHAR(50)-- 厂商
);
```

### 子表（Sub Table）

子表是超级表的具体实例，代表一个具体的设备：

```sql
-- 为每台发电机创建子表
CREATE TABLE meter_001 USING power_meter TAGS (1, 'Beijing', 'ABB');
CREATE TABLE meter_002 USING power_meter TAGS (2, 'Shanghai', 'Siemens');
CREATE TABLE meter_003 USING power_meter TAGS (3, 'Guangzhou', 'GE');
```

### 优势对比

| 传统方式 | TDengine 超级表方式 |
|---|---|
| 每台设备一张表 | 所有同类设备共用一个超级表 |
| 百万设备 = 百万张表 | 百万设备 = 百万子表 + 1 个超级表 |
| Schema 变更要改百万张表 | Schema 变更只需改超级表 |
| 跨设备聚合查询复杂 | 跨设备聚合只需查超级表 |

## 写入性能

TDengine 的写入性能在单节点场景下达到了惊人的 1000 万点/秒：

| 指标 | TDengine | InfluxDB | TimescaleDB |
|---|---|---|---|
| 写入吞吐 (单节点) | 1000 万点/秒 | 50 万点/秒 | 10 万点/秒 |
| 写入延迟 | < 1ms | < 5ms | < 10ms |
| Bulk Insert 吞吐 | 5000 万点/秒 | 200 万点/秒 | 50 万点/秒 |

:::tip 性能优化的秘密
TDengine 高性能的秘密：
1. **无锁写入**：采用 WAL（Write-Ahead Log）顺序写
2. **列式存储**：同类型数据连续存储，压缩率高
3. **SIMD 加速**：利用 CPU 指令集加速计算
4. **零拷贝**：避免不必要的数据复制
:::

### 压缩技术

| 技术 | 说明 | 效果 |
|---|---|---|
| Delta 编码 | 时间戳存储差值 | 时间戳压缩 5-10x |
| ZigZag 编码 | 有符号整数压缩 | 整数压缩 2x |
| XOR 编码 | 浮点数相邻压缩 | 浮点压缩 3-5x |
| 字典编码 | 重复标签压缩 | 标签压缩 10-50x |
| 列式压缩 | 同列数据连续存储 | 整体压缩 5-10x |

## SQL 兼容

TDengine 支持标准 SQL，包括 JOIN、子查询等高级特性：

```sql
-- 基础查询
SELECT * FROM meter_001 WHERE ts > NOW() - 1h;

-- 超级表聚合 (跨所有子表)
SELECT AVG(voltage), MAX(current)
FROM power_meter
WHERE ts > NOW() - 1d
GROUP BY location;

-- JOIN 查询
SELECT m.meter_id, m.location, e.energy
FROM power_meter m
JOIN equipment e ON m.meter_id = e.id
WHERE m.ts > NOW() - 1h;

-- 子查询
SELECT * FROM (
  SELECT tbname, AVG(voltage) as avg_v
  FROM power_meter
  WHERE ts > NOW() - 1h
  GROUP BY tbname
) t
WHERE t.avg_v > 220;
```

## 内置功能

### 消息队列

TDengine 原生支持消息队列功能，无需额外部署 Kafka：

```sql
-- 创建主题 (类似 Kafka Topic)
CREATE TOPIC topic_power_meter AS SELECT * FROM power_meter;

-- 订阅数据
-- 通过 TAOS Adapter 连接 Kafka/RocketMQ 等客户端消费
```

### 缓存

TDengine 原生支持缓存功能，无需额外部署 Redis：

```sql
-- 写入缓存
INSERT INTO cache_latest_power
VALUES (NOW(), 'meter_001', 220.5, 10.5, 2.3);

-- 查询缓存
SELECT * FROM cache_latest_power WHERE meter_id = 'meter_001';
```

### 流计算

TDengine 支持流式计算：

```sql
-- 创建流
CREATE STREAM power_stream INTO power_stream_out
AS SELECT _wstart, tbname, AVG(voltage), MAX(current)
FROM power_meter
PARTITION BY tbname
INTERVAL (1m);
```

## 总结

TDengine 是专为物联网设计的时序数据库：

1. **超高写入性能**：单节点 1000 万点/秒，轻松应对海量设备
2. **超级表架构**：超级表定义模板，子表存储设备数据，适合百万级设备管理
3. **极低存储成本**：行列混合压缩，存储空间仅为其他 TSDB 的 10%
4. **SQL 完全兼容**：标准 SQL 查询，JOIN、子查询都支持
5. **内置消息队列**：减少系统复杂度，一站式解决方案
6. **国产开源**：社区活跃，文档完善，国产支持响应快

如果你的场景是物联网、车联网、工业互联网等设备数据采集场景，TDengine 是非常值得考虑的选择。
