# 查询与降采样

你的监控面板需要展示两个视图：
- 实时视图：过去 1 小时的分钟级数据
- 历史视图：过去 30 天的小时级汇总

这两种视图背后，是 TDengine 的两种查询能力：原始数据查询和聚合查询。

## 基础查询

### SELECT 查询

```sql
-- 查询所有字段
SELECT * FROM meter_001 WHERE ts > NOW() - 1h;

-- 查询指定字段
SELECT ts, voltage, current, power FROM meter_001
WHERE ts > NOW() - 1h;

-- 带条件过滤
SELECT * FROM power_meter
WHERE ts > NOW() - 1d
  AND voltage > 220
  AND location = 'Beijing';
```

### 时间范围

```sql
-- 最近 1 小时
SELECT * FROM meter_001 WHERE ts > NOW() - 1h;

-- 指定时间范围
SELECT * FROM meter_001
WHERE ts > '2024-01-01 00:00:00'
  AND ts < '2024-01-02 00:00:00';

-- 时间窗口切片
SELECT * FROM meter_001
WHERE ts > '2024-01-01 00:00:00'
  AND ts < '2024-01-01 01:00:00';
```

## 聚合查询

### 按时间窗口聚合

```sql
-- 按 5 分钟聚合
SELECT AVG(voltage) AS avg_voltage,
       MAX(current) AS max_current,
       SUM(power) AS total_power
FROM meter_001
WHERE ts > NOW() - 1d
GROUP BY tbname, INTERVAL(5m);

-- 按 1 小时聚合
SELECT AVG(voltage) AS avg_voltage
FROM power_meter
WHERE ts > NOW() - 30d
GROUP BY location, INTERVAL(1h);
```

### INTERVAL 参数

| 参数 | 说明 | 示例 |
|---|---|---|
| `1s` | 1 秒 | `INTERVAL(1s)` |
| `1m` | 1 分钟 | `INTERVAL(1m)` |
| `1h` | 1 小时 | `INTERVAL(1h)` |
| `1d` | 1 天 | `INTERVAL(1d)` |
| `1w` | 1 周 | `INTERVAL(1w)` |

### 多级时间窗口

```sql
-- 按 1 小时聚合，每 10 分钟滑动一次
SELECT AVG(voltage)
FROM meter_001
WHERE ts > NOW() - 1d
GROUP BY tbname, INTERVAL(1h) SLIDE(10m);
```

## 降采样实践

### 场景：构建数据金字塔

```
┌─────────────────────────────────────────────────────────────────────┐
│                        多级降采样架构                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Level 1: 原始数据 (秒级)                                           │
│  ├── 保留: 7 天                                                    │
│  ├── 查询: SELECT * FROM metrics WHERE ts > NOW() - 1h              │
│  └── 用途: 实时监控、故障诊断                                       │
│                                                                     │
│  Level 2: 1 分钟聚合                                               │
│  ├── 保留: 30 天                                                   │
│  ├── 查询: SELECT AVG(*) FROM metrics GROUP BY INTERVAL(1m)          │
│  └── 用途: 小时级趋势分析                                           │
│                                                                     │
│  Level 3: 1 小时聚合                                               │
│  ├── 保留: 365 天                                                  │
│  ├── 查询: SELECT AVG(*) FROM metrics GROUP BY INTERVAL(1h)         │
│  └── 用途: 日报、周报、月报                                          │
│                                                                     │
│  Level 4: 1 天聚合                                                 │
│  ├── 保留: 永久                                                    │
│  ├── 查询: SELECT AVG(*) FROM metrics GROUP BY INTERVAL(1d)          │
│  └── 用途: 年度统计、趋势分析                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 实现方式一：连续查询

```sql
-- 1 分钟降采样
CREATE STABLE metrics_1m (
    ts TIMESTAMP,
    avg_voltage FLOAT,
    max_voltage FLOAT,
    min_voltage FLOAT,
    avg_current FLOAT
) TAGS (
    location VARCHAR(50)
);

CREATE CQ cq_downsample_1m ON power_db
BEGIN
  INSERT INTO power_db.metrics_1m
  SELECT _wstart, AVG(voltage), MAX(voltage), MIN(voltage), AVG(current)
  FROM power_db.power_meter
  PARTITION BY location
  INTERVAL(1m);
END;

-- 1 小时降采样
CREATE STABLE metrics_1h (
    ts TIMESTAMP,
    avg_voltage FLOAT,
    max_voltage FLOAT,
    min_voltage FLOAT
) TAGS (
    location VARCHAR(50)
);

CREATE CQ cq_downsample_1h ON power_db
BEGIN
  INSERT INTO power_db.metrics_1h
  SELECT _wstart, AVG(voltage), MAX(voltage), MIN(voltage)
  FROM power_db.metrics_1m
  PARTITION BY location
  INTERVAL(1h);
END;
```

### 实现方式二：流计算

```sql
-- 创建流进行实时降采样
CREATE STREAM power_stream_1m INTO power_stream_out_1m
AS SELECT _wstart AS ts,
       AVG(voltage) AS avg_voltage,
       MAX(voltage) AS max_voltage,
       MIN(voltage) AS min_voltage,
       location
FROM power_meter
PARTITION BY location, tbname
INTERVAL(1m)
WATERMARK 1m;
```

## 跨表查询

### UNION ALL

```sql
-- 合并多表数据
(SELECT ts, voltage, 'meter_001' AS meter_name FROM meter_001 WHERE ts > NOW() - 1h)
UNION ALL
(SELECT ts, voltage, 'meter_002' AS meter_name FROM meter_002 WHERE ts > NOW() - 1h)
UNION ALL
(SELECT ts, voltage, 'meter_003' AS meter_name FROM meter_003 WHERE ts > NOW() - 1h)
ORDER BY ts;
```

### 子查询

```sql
-- 计算与平均值的偏差
SELECT ts, voltage, (voltage - avg_v) AS deviation
FROM (
  SELECT ts, voltage, AVG(voltage) OVER () AS avg_v
  FROM meter_001
  WHERE ts > NOW() - 1h
);
```

### JOIN 查询

```sql
-- 关联设备信息表
SELECT m.meter_id, e.equipment_name, m.voltage, m.current
FROM power_meter m
JOIN equipment e ON m.meter_id = e.id
WHERE m.ts > NOW() - 1h;
```

## 窗口函数

### 滑动窗口

```sql
-- 5 分钟滑动窗口，每分钟输出一次
SELECT tbname,
       _wstart AS window_start,
       AVG(voltage) AS avg_voltage
FROM power_meter
WHERE ts > NOW() - 1h
GROUP BY tbname, INTERVAL(5m) SLIDE(1m);
```

### 状态窗口

```sql
-- 按状态变化分窗口
SELECT _wstart, tbname,
       AVG(voltage) AS avg_voltage
FROM power_meter
WHERE ts > NOW() - 1h
GROUP BY tbname, STATE_WINDOW(status);
```

### 会话窗口

```sql
-- 30 秒内无数据则断开
SELECT _wstart, tbname,
       COUNT(*) AS data_points
FROM power_meter
WHERE ts > NOW() - 1h
GROUP BY tbname, SESSION(ts, 30s);
```

## 统计函数

### 基础统计

```sql
SELECT
    COUNT(*) AS total_count,
    AVG(voltage) AS avg_voltage,
    SUM(power) AS total_power,
    MIN(voltage) AS min_voltage,
    MAX(voltage) AS max_voltage,
    STDDEV(voltage) AS std_voltage
FROM meter_001
WHERE ts > NOW() - 1d;
```

### 百分位数

```sql
-- 计算 P50, P90, P99
SELECT
    PERCENTILE(voltage, 50) AS p50,
    PERCENTILE(voltage, 90) AS p90,
    PERCENTILE(voltage, 99) AS p99
FROM meter_001
WHERE ts > NOW() - 1d;
```

### 变化率

```sql
-- 计算差值
SELECT ts,
       voltage,
       voltage - LAG(voltage, 1) AS voltage_diff
FROM meter_001
WHERE ts > NOW() - 1h;

-- 计算变化率
SELECT ts,
       voltage,
       DIFF(voltage) / INTERVAL(1s) AS voltage_rate
FROM meter_001
WHERE ts > NOW() - 1h;
```

## 分组聚合

### 按 Tags 分组

```sql
-- 按位置分组
SELECT location,
       COUNT(*) AS meter_count,
       AVG(voltage) AS avg_voltage,
       MAX(current) AS max_current
FROM power_meter
WHERE ts > NOW() - 1d
GROUP BY location;

-- 按多个 Tags 分组
SELECT region,
       manufacturer,
       COUNT(*) AS meter_count,
       AVG(voltage) AS avg_voltage
FROM power_meter
WHERE ts > NOW() - 1d
GROUP BY region, manufacturer;
```

### 按子表分组

```sql
-- 每个子表的统计
SELECT tbname,
       FIRST(voltage) AS first_voltage,
       LAST(voltage) AS last_voltage,
       MAX(voltage) - MIN(voltage) AS voltage_range
FROM power_meter
WHERE ts > NOW() - 1d
GROUP BY tbname;
```

## 查询优化技巧

### 1. 使用时间过滤

```sql
-- ✅ 推荐：限定时间范围
SELECT * FROM meter_001
WHERE ts > NOW() - 1h;

-- ❌ 避免：全表扫描
SELECT * FROM meter_001;
```

### 2. 避免 SELECT *

```sql
-- ✅ 推荐：指定字段
SELECT ts, voltage, current FROM meter_001
WHERE ts > NOW() - 1h;

-- ❌ 避免：返回所有字段
SELECT * FROM meter_001
WHERE ts > NOW() - 1h;
```

### 3. 利用降采样

```sql
-- ✅ 推荐：查询已降采样的数据
SELECT * FROM metrics_1h
WHERE ts > NOW() - 30d;

-- ❌ 避免：实时聚合大量历史数据
SELECT AVG(voltage)
FROM power_meter
WHERE ts > NOW() - 30d
GROUP BY INTERVAL(1h);
```

### 4. 利用缓存

```sql
-- 最近的数据使用缓存查询
SELECT LAST(*) FROM power_meter
GROUP BY tbname;
```

## 总结

TDengine 的查询能力强大且灵活：

1. **基础查询**：支持完整的时间范围过滤和条件筛选
2. **聚合查询**：INTERVAL 时间窗口聚合是核心能力
3. **降采样**：通过连续查询或流计算实现多级降采样
4. **跨表查询**：支持 UNION、JOIN、子查询
5. **窗口函数**：滑动窗口、状态窗口、会话窗口满足各种需求
6. **统计函数**：COUNT、AVG、SUM、STDDEV、PERCENTILE 等

合理利用降采样和聚合查询，可以在保证数据价值的同时，大幅提升查询性能。
