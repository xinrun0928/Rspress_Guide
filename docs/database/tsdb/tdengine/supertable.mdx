# 超级表与子表设计

你有 10 万台发电机需要监控，每台发电机每秒采集电压、电流、功率等 10 个指标。

如果每台发电机一张表，你需要创建 10 万张表，查询跨设备数据时 join 操作让人头疼。

TDengine 的超级表设计，让你用一个模板管理所有同类设备。

## 为什么需要超级表

### 传统方式的困境

```
┌─────────────────────────────────────────────────────────────────────┐
│                    传统方式：每设备一张表                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  meter_001 ─┐                                                       │
│  meter_002 ─┼─ 10 万张表 ── 10 万个文件 ── 复杂的元数据管理         │
│  meter_003 ─┤                                                       │
│  ...        │                                                       │
│  meter_100000┘                                                       │
│                                                                     │
│  问题:                                                               │
│  1. 创建 10 万张表耗时极长                                           │
│  2. 每张表独立索引，查询效率低                                       │
│  3. Schema 变更需要改 10 万张表                                      │
│  4. 跨设备聚合需要全表扫描或手动 union                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 超级表的优势

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TDengine：超级表 + 子表                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                    Super Table (超级表)                             │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │  power_meter                                               │  │
│  │  Columns: ts, voltage, current, power, energy              │  │
│  │  Tags: meter_id, location, manufacturer                    │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                            │                                        │
│         ┌──────────────────┼──────────────────┐                     │
│         ▼                  ▼                  ▼                     │
│  ┌────────────┐     ┌────────────┐     ┌────────────┐                │
│  │ meter_001 │     │ meter_002 │     │ meter_003 │                │
│  │ (子表)    │     │ (子表)    │     │ (子表)    │                │
│  └────────────┘     └────────────┘     └────────────┘                │
│  │                 │                 │                             │
│  └─────────────────┼─────────────────┘                             │
│                    │                                               │
│           共享 Tags 索引 (高效)                                     │
│                                                                     │
│  优势:                                                               │
│  1. 一次创建 N 个子表                                                │
│  2. Tags 统一索引                                                    │
│  3. Schema 变更只需改超级表                                          │
│  4. 跨设备聚合只需查超级表                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 创建超级表

### 基本语法

```sql
CREATE STABLE [IF NOT EXISTS] <stable_name> (
    <col_name> <data_type>, ...
) TAGS (
    <tag_name> <data_type>, ...
);
```

### 示例：电力监控超级表

```sql
-- 创建超级表
CREATE STABLE power_meter (
    ts TIMESTAMP,           -- 时间戳，TDengine 强制要求第一列
    voltage FLOAT,           -- 电压 (V)
    current FLOAT,           -- 电流 (A)
    power FLOAT,            -- 有功功率 (kW)
    reactive_power FLOAT,    -- 无功功率 (kVar)
    power_factor FLOAT,      -- 功率因数
    energy_consumed FLOAT,   -- 累计消耗电能 (kWh)
    energy_generated FLOAT   -- 累计发电电能 (kWh)
) TAGS (
    meter_id BIGINT,         -- 电表 ID
    location VARCHAR(100),   -- 地理位置
    region VARCHAR(50),      -- 区域
    manufacturer VARCHAR(50),-- 设备厂商
    model VARCHAR(50)        -- 设备型号
);
```

### 常用数据类型

| 类型 | 说明 | 示例 |
|---|---|---|
| `TIMESTAMP` | 时间戳，必需的第一列 | `ts TIMESTAMP` |
| `INT` / `INT UNSIGNED` | 32 位整数 | `count INT` |
| `BIGINT` / `BIGINT UNSIGNED` | 64 位整数 | `meter_id BIGINT` |
| `FLOAT` | 单精度浮点 | `temperature FLOAT` |
| `DOUBLE` | 双精度浮点 | `precise_value DOUBLE` |
| `VARCHAR(n)` | 可变长度字符串 | `name VARCHAR(100)` |
| `NCHAR(n)` | Unicode 字符串 | `description NCHAR(200)` |
| `BOOL` | 布尔值 | `is_online BOOL` |
| `BINARY(n)` | 二进制数据 | `data BINARY(1024)` |

## 创建子表

### 基本语法

```sql
CREATE TABLE [IF NOT EXISTS] <table_name> USING <stable_name> TAGS (<tag_value>, ...);
```

### 示例

```sql
-- 为单个设备创建子表
CREATE TABLE meter_001 USING power_meter TAGS (1, 'Beijing Chaoyang', 'Beijing', 'ABB', 'ABB-5000');
CREATE TABLE meter_002 USING power_meter TAGS (2, 'Shanghai Pudong', 'Shanghai', 'Siemens', 'S-3000');
CREATE TABLE meter_003 USING power_meter TAGS (3, 'Guangzhou Tianhe', 'Guangzhou', 'GE', 'GE-Pro');

-- 使用自动生成的表名
CREATE TABLE meter_004 USING power_meter TAGS (4, 'Shenzhen Nanshan', 'Shenzhen', ' Schneider', 'SC-2000');
```

### 批量创建子表

```sql
-- 从 CSV 文件批量创建
CREATE TABLE meters USING power_meter TAGS ('AutoGen_001', 'Location_A', 'Region_A', 'Mfr_A', 'Model_A');
CREATE TABLE meters USING power_meter TAGS ('AutoGen_002', 'Location_B', 'Region_B', 'Mfr_B', 'Model_B');
-- ...
```

:::tip 批量创建技巧
如果需要创建大量子表，建议：
1. 使用脚本批量生成 CREATE TABLE 语句
2. 在事务中批量提交
3. 或使用超级表的自动建表功能
:::

## 自动建表

TDengine 支持在写入时自动创建子表：

```sql
-- 自动创建子表
INSERT INTO meter_auto_001
USING power_meter TAGS (1001, 'Auto_Loc_001', 'Auto_Region', 'Auto_Mfr', 'Auto_Model')
VALUES (NOW(), 220.5, 10.2, 2.25, 0.5, 0.95, 1000.5, 500.2);
```

```python
# Python 客户端自动建表
from taosrest import TaosRestDemo

conn = TaosRestDemo(url="http://localhost:6041", token="your-token")

# 写入时自动建表
conn.execute("INSERT INTO meter_auto_002 "
             "USING power_meter TAGS (1002, 'Auto_Loc_002', 'Auto_Region', 'Auto_Mfr', 'Auto_Model') "
             "VALUES (NOW(), 221.0, 10.5, 2.30, 0.6, 0.94, 1001.0, 501.0)")
```

## 数据写入

### 单条写入

```sql
-- 写入单个子表
INSERT INTO meter_001 VALUES ('2024-01-15 10:00:00.000', 220.5, 10.2, 2.25, 0.5, 0.95, 1000.5, 500.2);

-- 指定时间戳
INSERT INTO meter_001 VALUES (1705305600000, 220.5, 10.2, 2.25, 0.5, 0.95, 1000.5, 500.2);
```

### 批量写入


## 查询操作

### 查询子表

```sql
-- 查询单个子表
SELECT * FROM meter_001 WHERE ts > NOW() - 1h;

-- 查询多个子表
SELECT * FROM meter_001, meter_002 WHERE ts > NOW() - 1h;
```

### 查询超级表（跨所有子表聚合）

```sql
-- 统计所有电表的平均电压
SELECT AVG(voltage), MAX(current), SUM(power)
FROM power_meter
WHERE ts > NOW() - 1h;

-- 按 Tags 分组聚合
SELECT location, AVG(voltage), MAX(current)
FROM power_meter
WHERE ts > NOW() - 1h
GROUP BY location;

-- 按多个 Tags 分组
SELECT region, manufacturer, AVG(voltage)
FROM power_meter
WHERE ts > NOW() - 1h
GROUP BY region, manufacturer;
```

### 使用 tbname 获取子表名

```sql
-- 获取每个子表的最新数据
SELECT LAST(*), tbname
FROM power_meter
WHERE ts > NOW() - 1h
GROUP BY tbname;

-- 按子表过滤
SELECT * FROM power_meter
WHERE tbname IN ('meter_001', 'meter_002', 'meter_003')
  AND ts > NOW() - 1h;
```

### 修改 Schema

```sql
-- 添加列
ALTER STABLE power_meter ADD COLUMN frequency FLOAT;

-- 添加 Tag
ALTER STABLE power_meter ADD TAG is_virtual BOOL;

-- 修改列宽
ALTER STABLE power_meter MODIFY COLUMN voltage DOUBLE;
```

### 删除操作

```sql
-- 删除子表
DROP TABLE meter_001;

-- 删除超级表（同时删除所有子表）
DROP STABLE power_meter;
```

## 设计最佳实践

### 1. 合理的 Tags 设计

```sql
-- ✅ 推荐：低基数的 Tags
CREATE STABLE sensor_data (
    ts TIMESTAMP,
    temperature FLOAT,
    humidity FLOAT
) TAGS (
    device_type INT,          -- 低基数：几十种类型
    location VARCHAR(50),    -- 低基数：几十个位置
    firmware VARCHAR(20)      -- 中基数：固件版本有限
);

-- ❌ 避免：高基数的 Tags
CREATE STABLE sensor_data (
    ts TIMESTAMP,
    value FLOAT
) TAGS (
    device_id BIGINT,         -- 高基数：百万设备
    user_id BIGINT           -- 高基数：百万用户
);
```

### 2. 合理的列设计

```sql
-- ✅ 推荐：分析型列
CREATE STABLE app_metrics (
    ts TIMESTAMP,
    request_count BIGINT,     -- 请求计数 (INT/BIGINT)
    response_time DOUBLE,     -- 响应时间 (DOUBLE)
    error_count INT           -- 错误计数 (INT)
);

-- ❌ 避免：冗余数据
CREATE STABLE app_metrics (
    ts TIMESTAMP,
    total_count BIGINT,        -- 冗余：可以计算
    today_count BIGINT        -- 冗余：可以计算
);
```

### 3. 分区策略

```sql
-- 创建数据库时指定分区
CREATE DATABASE power_db
  KEEP 365            -- 保留 365 天
  DAYS 10             -- 每个 Partition 10 天
  BUFFER 256          -- 写入 Buffer 256MB
  COMPRESSION '列压缩方式'  -- 支持 LZ4, ZSTD, ZLIB 等
  MAXROWS 4096        -- 每个文件最大行数
  MINROWS 100;        -- 每个文件最小行数
```

## 总结

TDengine 的超级表 + 子表架构是处理海量设备数据的利器：

1. **超级表定义模板**：同类设备共享 Schema 和 Tags 定义
2. **子表代表具体设备**：每台设备一张子表，数据隔离
3. **Tags 支持索引**：高效的跨设备聚合查询
4. **支持自动建表**：写入时可自动创建子表
5. **Schema 灵活变更**：修改超级表即可影响所有子表
6. **SQL 完全兼容**：标准 SQL 操作，学习成本为零

设计好超级表和 Tags，是用好 TDengine 的关键。
