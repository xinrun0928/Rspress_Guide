# Kubernetes 服务发现

你的 Kubernetes 集群里有 100 个 Pod，每个 Pod 的 IP 都在动态变化。

手动维护监控配置？运维团队会疯掉。

Prometheus 的 Kubernetes 服务发现，让你配置一次，自动感知所有变化。

## 服务发现原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Prometheus Kubernetes SD                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │                    Kubernetes API Server                    │  │
│  │                                                             │  │
│  │   ┌───────────┐  ┌───────────┐  ┌───────────┐             │  │
│  │   │   Pods    │  │ Services  │  │  Nodes    │             │  │
│  │   │           │  │           │  │           │             │  │
│  │   │ - nginx   │  │ - api     │  │ - node-1  │             │  │
│  │   │ - api     │  │ - web     │  │ - node-2  │             │  │
│  │   │ - worker  │  │ - worker  │  │           │             │  │
│  │   └───────────┘  └───────────┘  └───────────┘             │  │
│  │                                                             │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                              │                                      │
│                              │ Kubernetes API                       │
│                              ▼                                      │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │                   Prometheus                                 │  │
│  │                                                             │  │
│  │   kubernetes_sd_configs:                                    │  │
│  │     - role: pod           ──────► 自动发现所有 Pod          │  │
│  │     - role: service       ──────► 自动发现所有 Service      │  │
│  │     - role: node          ──────► 自动发现所有 Node         │  │
│  │     - role: endpoints     ──────► 自动发现所有 Endpoints     │  │
│  │     - role: ingress       ──────► 自动发现所有 Ingress      │  │
│  │                                                             │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 角色类型

### Pod 角色

```yaml title="prometheus.yml"
scrape_configs:
  - job_name: 'kubernetes-pods'
    kubernetes_sd_configs:
      - role: pod
```

自动生成的标签：

| 标签 | 说明 | 示例 |
|---|---|---|
| `__meta_kubernetes_pod_name` | Pod 名称 | `nginx-abc123` |
| `__meta_kubernetes_pod_ip` | Pod IP | `10.0.0.5` |
| `__meta_kubernetes_namespace` | 命名空间 | `default` |
| `__meta_kubernetes_pod_label_*` | Pod 标签 | `app=nginx` |
| `__meta_kubernetes_pod_container_name` | 容器名称 | `nginx` |
| `__meta_kubernetes_pod_container_port_number` | 容器端口 | `80` |

### Node 角色

```yaml title="prometheus.yml"
scrape_configs:
  - job_name: 'kubernetes-nodes'
    kubernetes_sd_configs:
      - role: node
```

自动生成的标签：

| 标签 | 说明 | 示例 |
|---|---|---|
| `__meta_kubernetes_node_name` | Node 名称 | `node-1` |
| `__meta_kubernetes_node_ip` | Node IP | `192.168.1.10` |
| `__meta_kubernetes_node_label_*` | Node 标签 | `node-role=worker` |
| `__meta_kubernetes_node_annotation_*` | Node 注解 | |

### Endpoints 角色

```yaml title="prometheus.yml"
scrape_configs:
  - job_name: 'kubernetes-endpoints'
    kubernetes_sd_configs:
      - role: endpoints
```

自动生成的标签：

| 标签 | 说明 | 示例 |
|---|---|---|
| `__meta_kubernetes_namespace` | 命名空间 | `default` |
| `__meta_kubernetes_endpoints_name` | Endpoints 名称 | `api` |
| `__meta_kubernetes_endpoint_port_name` | 端口名称 | `http` |
| `__meta_kubernetes_endpoint_port` | 端口号 | `8080` |
| `__meta_kubernetes_endpoint_ready` | 就绪状态 | `true` |

## relabel_configs

`relabel_configs` 是 Prometheus 服务发现的核心，用于过滤、重写标签：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        relabel_configs 流程                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Target (原始)                                                      │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │ __address__: "10.0.0.5:80"                                 │  │
│  │ __meta_kubernetes_pod_name: "nginx-abc123"                  │  │
│  │ __meta_kubernetes_namespace: "production"                   │  │
│  │ __meta_kubernetes_pod_label_app: "nginx"                   │  │
│  │                                                             │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                              │                                      │
│                              │ relabel_configs                      │
│                              ▼                                      │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │  1. keep: 只保留包含特定标签的 targets                       │  │
│  │  2. labelmap: 将 __meta_kubernetes_pod_label_* 映射为普通标签│  │
│  │  3. replace: 替换标签值                                      │  │
│  │                                                             │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                              │                                      │
│                              ▼                                      │
│  Final Target (用于采集)                                           │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │ __address__: "10.0.0.5:80"                                  │  │
│  │ pod: "nginx-abc123"          # 映射后的标签                 │  │
│  │ namespace: "production"      # 映射后的标签                 │  │
│  │ app: "nginx"                 # 映射后的标签                 │  │
│  │                                                             │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 常用操作

#### 1. 过滤 Targets

```yaml
relabel_configs:
  # 只保留包含特定标签的 targets
  - source_labels: [__meta_kubernetes_namespace]
    action: keep
    regex: 'production|monitoring'

  # 排除特定 targets
  - source_labels: [__meta_kubernetes_pod_label_tier]
    action: drop
    regex: 'frontend'
```

#### 2. 标签映射

```yaml
relabel_configs:
  # 将 __meta_kubernetes_pod_label_* 映射为 pod_label_*
  - action: labelmap
    regex: __meta_kubernetes_pod_label_(.+)
    replacement: pod_label_$1

  # 将 __meta_kubernetes_pod_label_app 映射为 app
  - action: labelmap
    regex: __meta_kubernetes_pod_label_app
    replacement: app
```

#### 3. 标签替换

```yaml
relabel_configs:
  # 替换标签值
  - source_labels: [__meta_kubernetes_namespace]
    target_label: namespace
    replacement: $1

  # 使用正则提取
  - source_labels: [__meta_kubernetes_pod_name]
    regex: '(.+)-[a-z0-9]+'
    target_label: app
    replacement: $1
```

#### 4. 标签删除

```yaml
relabel_configs:
  # 删除不需要的标签
  - action: labeldrop
    regex: '__meta_kubernetes_pod_label_version'

  # 删除以 __ 开头的内部标签
  - action: labeldrop
    regex: '__meta_kubernetes_.*'
```

## 完整配置示例

### Pod 监控

```yaml title="pod_scrape.yml"
scrape_configs:
  # 监控所有 Pod (过滤掉未就绪的)
  - job_name: 'kubernetes-pods'
    kubernetes_sd_configs:
      - role: pod
        namespaces:
          names:
            - production
            - monitoring
    relabel_configs:
      # 只保留包含 /metrics 端点的 Pod
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_path]
        action: keep
        regex: (.+)

      # 设置默认路径
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_path]
        action: replace
        target_label: __metrics_path__
        regex: (.+)

      # 从注解中获取端口，默认 80
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_port]
        action: replace
        target_label: __address__
        regex: (.+)
        replacement: $1:${1}

      # 映射标签
      - action: labelmap
        regex: __meta_kubernetes_pod_label_(.+)

      # 用 namespace 覆盖默认的 job 标签
      - source_labels: [__meta_kubernetes_namespace]
        action: replace
        target_label: kubernetes_namespace
```

### Service 监控

```yaml title="service_scrape.yml"
scrape_configs:
  - job_name: 'kubernetes-services'
    kubernetes_sd_configs:
      - role: service
    relabel_configs:
      # 只保留包含 prometheus.io/scrape 注解的 Service
      - source_labels: [__meta_kubernetes_service_annotation_prometheus_io_scrape]
        action: keep
        regex: true

      # 设置采集间隔 (默认 60s)
      - source_labels: [__meta_kubernetes_service_annotation_prometheus_io_scheme]
        action: replace
        target_label: __scheme__
        regex: (https?)

      # 设置路径
      - source_labels: [__meta_kubernetes_service_annotation_prometheus_io_path]
        action: replace
        target_label: __metrics_path__
        regex: (.+)

      # 设置端口
      - source_labels: [__meta_kubernetes_service_annotation_prometheus_io_port]
        action: replace
        target_label: __param_target_port
        regex: (.+)

      # 标签映射
      - action: labelmap
        regex: __meta_kubernetes_service_label_(.+)

      # 设置 target 实例
      - source_labels: [__meta_kubernetes_namespace]
        action: replace
        target_label: kubernetes_namespace
      - source_labels: [__meta_kubernetes_service_name]
        action: replace
        target_label: kubernetes_service_name
```

### Node 监控

```yaml title="node_scrape.yml"
scrape_configs:
  - job_name: 'kubernetes-nodes'
    kubernetes_sd_configs:
      - role: node
    relabel_configs:
      # 用 Node IP 替换默认地址
      - source_labels: [__meta_kubernetes_node_name]
        regex: (.+)
        target_label: __address__
        replacement: $1:9100

      # 映射标签
      - action: labelmap
        regex: __meta_kubernetes_node_label_(.+)

      # 标签改名
      - source_labels: [__meta_kubernetes_node_label_node_role_kubernetes_io]
        action: replace
        target_label: node_role
        regex: worker
```

### Endpoints 监控

```yaml title="endpoints_scrape.yml"
scrape_configs:
  - job_name: 'kubernetes-endpoints'
    kubernetes_sd_configs:
      - role: endpoints
    relabel_configs:
      # 只保留包含 prometheus.io/scrape 注解的 Endpoints
      - source_labels: [__meta_kubernetes_service_annotation_prometheus_io_scrape]
        action: keep
        regex: true

      # 只保留 Ready 的 endpoints
      - source_labels: [__meta_kubernetes_endpoint_ready]
        action: keep
        regex: true

      # 标签映射
      - action: labelmap
        regex: __meta_kubernetes_(.+)
```

## Pod 注解配置

在应用的 Pod 中添加注解，让 Prometheus 自动发现：

```yaml title="pod-annotation.yml"
apiVersion: v1
kind: Pod
metadata:
  name: my-app
  annotations:
    prometheus.io/scrape: "true"           # 是否采集
    prometheus.io/scheme: "http"          # 协议 (http/https)
    prometheus.io/path: "/metrics"        # 路径
    prometheus.io/port: "8080"            # 端口
spec:
  containers:
    - name: my-app
      image: my-app:latest
      ports:
        - containerPort: 8080
```

### 注解说明

| 注解 | 说明 | 默认值 |
|---|---|---|
| `prometheus.io/scrape` | 是否采集 | `true` |
| `prometheus.io/scheme` | 采集协议 | `http` |
| `prometheus.io/path` | 采集路径 | `/metrics` |
| `prometheus.io/port` | 采集端口 | 容器端口 |

## ServiceMonitor (推荐方式)

使用 Prometheus Operator 时，用 ServiceMonitor 定义监控目标：

```yaml title="service-monitor.yml"
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: my-app-monitor
  labels:
    release: prometheus
spec:
  selector:
    matchLabels:
      app: my-app
  namespaceSelector:
    matchNames:
      - production
  endpoints:
    - port: http
      path: /metrics
      interval: 30s
      scrapeTimeout: 10s
```

```yaml title="service.yml"
apiVersion: v1
kind: Service
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  selector:
    app: my-app
  ports:
    - name: http
      port: 80
      targetPort: 8080
```

## RBAC 配置

Prometheus 需要相应权限才能访问 Kubernetes API：

```yaml title="rbac.yml"
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: prometheus
rules:
  - apiGroups: [""]
    resources:
      - nodes
      - nodes/proxy
      - services
      - endpoints
      - pods
      - ingresses
    verbs: ["get", "list", "watch"]
  - apiGroups: [""]
    resources:
      - configmaps
      - secrets
    verbs: ["get"]
  - apiGroups: ["networking.k8s.io"]
    resources:
      - ingresses
    verbs: ["get", "list", "watch"]
  - nonResourceURLs:
      - /metrics
    verbs: ["get"]

---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: prometheus
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: prometheus
subjects:
  - kind: ServiceAccount
    name: prometheus
    namespace: monitoring
```

## 总结

Prometheus 的 Kubernetes 服务发现让监控配置变得简单：

1. **角色类型**：Pod、Node、Service、Endpoints、Ingress
2. **relabel_configs**：过滤、重写、映射标签
3. **Pod 注解**：通过注解控制采集行为
4. **ServiceMonitor**：Prometheus Operator 的声明式配置
5. **RBAC**：正确的权限配置是服务发现的基础

合理的服务发现配置，可以实现「部署即监控」，无需手动维护监控目标列表。
