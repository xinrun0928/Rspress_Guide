# Prometheus 总览

凌晨 3 点，你的服务开始报警：API 响应时间从 50ms 飙升到 5 秒。

你打开监控面板，发现过去 5 分钟内有大量请求超时。但这些请求调用了什么服务？数据库是否正常？Redis 有没有问题？

Prometheus，可能是你需要的答案。

## Prometheus 是什么

Prometheus 是 CNCF 的毕业项目，专为云原生环境的服务监控设计。

### 核心特点

| 特性 | 说明 | 优势 |
|---|---|---|
| **Pull 模式** | Prometheus 主动拉取指标 | 易于部署、无需修改应用代码 |
| **多维数据模型** | 标签维度灵活 | 支持多维度分析和过滤 |
| **PromQL** | 强大的查询语言 | 支持聚合、计算、函数 |
| **服务发现** | 自动发现监控目标 | 无缝对接 Kubernetes |
| **告警管理** | 告警规则 + AlertManager | 完整的告警生命周期 |
| **社区生态** | Exporter 丰富 | 200+ 官方/社区 Exporter |

```
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐  │
│  │    Counter      │  │     Gauge       │  │   Histogram     │  │

```

| 类型 | 说明 | 适用场景 | 示例 |
|---|---|---|---|
| `Counter` | 只增不减的计数器 | 请求总数、错误总数 | `http_requests_total` |
| `Gauge` | 可增可减的仪表 | 当前值、瞬时状态 | `cpu_usage_percent` |
| `Histogram` | 直方图，区间统计 | 请求延迟、响应大小 | `http_request_duration_seconds` |
| `Summary` | 分位数统计 | P99 延迟等 | `http_request_latency_seconds` |

### 标签（Labels）

标签是 Prometheus 多维数据模型的核心：

```promql
# 标签用于区分不同的维度
http_requests_total{method="GET", status="200"}    # GET 请求
http_requests_total{method="POST", status="200"}   # POST 请求
http_requests_total{method="GET", status="500"}    # GET 错误请求
```

| 良好标签设计 | 不良标签设计 |
|---|---|
| 低基数：method、status、handler | 高基数：user_id、request_id |
| 有意义的维度 | 唯一标识 |
| 可聚合 | 不可聚合 |

## 数据采集

### Exporter

Exporter 是 Prometheus 生态的核心，将指标暴露为 `/metrics` 端点：

| Exporter | 监控目标 | 端口 |
|---|---|---|
| `node_exporter` | 服务器硬件/系统 | 9100 |
| `mysqld_exporter` | MySQL 数据库 | 9104 |
| `postgres_exporter` | PostgreSQL | 9187 |
| `redis_exporter` | Redis 缓存 | 9121 |
| `nginxExporter` | Nginx | 9113 |
| `cadvisor` | Docker/Kubernetes | 8080 |
| `blackbox_exporter` | HTTP/TCP/ICMP | 9115 |

### 配置示例

```yaml title="prometheus.yml"
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'node'
    static_configs:
      - targets: ['node-exporter:9100']

  - job_name: 'mysql'
    static_configs:
      - targets: ['mysql-exporter:9104']
```

## 查询语言 PromQL

```promql
# 基础查询
http_requests_total

# 带标签过滤
http_requests_total{status="200", method="GET"}

# 范围查询
http_requests_total{status="200"}[5m]

# 聚合
sum(rate(http_requests_total[5m]))

# 分组聚合
sum(rate(http_requests_total[5m])) by (method, status)
```

## 告警规则

```yaml title="alert_rules.yml"
groups:
  - name: example
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_total{status="500"}[5m]) > 0.05
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "High error rate detected"
          description: "Error rate is {{ $value | printf \"%.2f\" }}%"
```

## 总结

Prometheus 是云原生监控的事实标准：

1. **Pull 模式**：服务端主动拉取，架构简单，易于部署
2. **多维数据模型**：标签提供灵活的维度分析能力
3. **PromQL**：强大的查询语言，支持聚合、计算、函数
4. **服务发现**：与 Kubernetes 深度集成，自动发现监控目标
5. **告警管理**：AlertManager 提供完整的告警生命周期
6. **丰富生态**：200+ Exporter，覆盖各类监控需求

如果你正在构建云原生应用，Prometheus 几乎是必选项。
