# 数据模型与 Metrics 类型

你在埋点监控数据时犯难了：用户请求延迟应该用 Counter、Gauge 还是 Histogram？

选错了指标类型，后续的数据分析会变得很痛苦。

Prometheus 定义了四种指标类型，每种都有明确的适用场景。

## Prometheus 数据模型

Prometheus 的数据模型是**基于标签的时间序列**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Prometheus 时间序列                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  格式: <metric_name>{<label_name>=<label_value>, ...}             │
│                                                                     │
│  示例:                                                              │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │                                                             │  │
│  │   http_requests_total{                                      │  │
│  │     method="GET",                                           │  │
│  │     status="200",                                           │  │
│  │     handler="/api/users"                                    │  │
│  │   }                                                         │  │
│  │                                                             │  │
│  │   │                                                         │  │
│  │   └── 指标名 + 标签 = 时间序列的唯一标识                     │  │
│  │                                                             │  │
│  │   值: 1234567.89                                            │  │
│  │   时间戳: 1709308800                                         │  │
│  │                                                             │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 时间序列的结构

```
┌─────────────────────────────────────────────────────────────────────┐
│                    时间序列的组成部分                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. 指标名称 (Metric Name)                                         │
│     - 标识指标的类型                                                │
│     - 命名规范: [a-zA-Z_:][a-zA-Z0-9_:*]*                          │
│     - 示例: http_requests_total, process_cpu_seconds_total          │
│                                                                     │
│  2. 标签 (Labels)                                                  │
│     - 维度的键值对                                                  │
│     - 用于区分同一指标的不同实例                                    │
│     - 示例: method="GET", status="200"                             │
│                                                                     │
│  3. 值 (Value)                                                     │
│     - 64 位浮点数                                                  │
│     - 示例: 12345.67                                               │
│                                                                     │
│  4. 时间戳 (Timestamp)                                             │
│     - Unix 时间戳 (毫秒)                                           │
│     - 示例: 1709308800000                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 指标命名规范

```promql
# ✅ 正确命名
http_requests_total           -- 总量
http_request_duration_seconds -- 延迟（带单位）
node_memory_usage_bytes       -- 内存使用量（带单位）

# ❌ 错误命名
HttpRequestsTotal            -- 不应该大写
request-count                -- 不应该用横杠
my_metric                    -- 缺少后缀
```

命名建议：

| 模式 | 说明 | 示例 |
|---|---|---|
| `<name>_total` | 计数器，表示总数 | `http_requests_total` |
| `<name>_seconds` | 直方图/摘要，单位秒 | `http_request_duration_seconds` |
| `<name>_bytes` | 大小指标，单位字节 | `file_size_bytes` |

## 四种指标类型

### 1. Counter（计数器）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Counter 计数器                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  特点: 只增不减，从 0 开始累积                                      │
│                                                                     │
│  用途: 统计累计发生的事件数量                                        │
│                                                                     │
│  0 │                           ┌───┐                              │
│    │                       ┌───┘   │                              │
│    │                   ┌───┘       │                              │
│    │               ┌───┘           │                              │
│    │           ┌───┘               │                              │
│    │       ┌───┘                   │                              │
│    │   ┌───┘                       │                              │
│    └───┘                           │                              │
│  ────────────────────────────────────────────────────────────────  │
│  0s    10s   20s   30s   40s   50s   60s   70s   80s   90s        │
│                                                                     │
│  使用场景:                                                          │
│  ├── 请求总数                                                        │
│  ├── 错误总数                                                        │
│  ├── 处理字节数                                                      │
│  └── 运行时间                                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```go
// Go 语言示例
var httpRequestsTotal = prometheus.NewCounterVec(
    prometheus.CounterOpts{
        Name: "http_requests_total",
        Help: "Total number of HTTP requests",
    },
    []string{"method", "status", "handler"},
)

// 每次请求时
httpRequestsTotal.WithLabelValues("GET", "200", "/api/users").Inc()
```

```python
# Python 语言示例 (prometheus_client)
from prometheus_client import Counter

http_requests_total = Counter(
    'http_requests_total',
    'Total HTTP requests',
    ['method', 'status', 'handler']
)

# 每次请求时
http_requests_total.labels(method='GET', status='200', handler='/api/users').inc()
```

### 2. Gauge（仪表）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Gauge 仪表                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  特点: 可增可减，反映当前瞬时值                                      │
│                                                                     │
│  用途: 测量可以任意变化的指标                                        │
│                                                                     │
│  100│     ┌───┐                                                    │
│    │     │   │     ┌───┐                                          │
│  80│ ────┘   └───┐ │   │                                          │
│    │             │ └───┘                                          │
│  60│             │                                                │
│    │             │                                                │
│  40│             │                                                │
│    │             │                                                │
│  20│             │                                                │
│    └───────────────────────────────────────────────────────────────  │
│  ────────────────────────────────────────────────────────────────  │
│  0s    10s   20s   30s   40s   50s   60s   70s   80s   90s        │
│                                                                     │
│  使用场景:                                                          │
│  ├── 当前连接数                                                      │
│  ├── CPU 使用率                                                     │
│  ├── 内存使用量                                                      │
│  ├── 温度/电压                                                      │
│  └── 队列长度                                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```go
// Go 语言示例
var cpuUsage = prometheus.NewGauge(prometheus.GaugeOpts{
    Name: "cpu_usage_percent",
    Help: "Current CPU usage percentage",
})

// 设置当前值
cpuUsage.Set(45.5)

// 增加
cpuUsage.Inc()

// 减少
cpuUsage.Dec()

// 增加指定值
cpuUsage.Add(10.0)
```

### 3. Histogram（直方图）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Histogram 直方图                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  特点: 将值分布到多个桶中，统计每个桶的累计计数                      │
│                                                                     │
│  用途: 分析值的分布情况                                             │
│                                                                     │
│  Bucket: ≤0.005s   ≤0.01s   ≤0.025s   ≤0.05s   ≤0.1s   ≤0.25s    │
│  ────────────────────────────────────────────────────────────────  │
│  请求:   ████      ████████  ██████████████  ████████████████      │
│                                                                     │
│  自动生成的时间序列:                                                 │
│  <metric>_bucket{le="0.01"}      -- ≤10ms 的请求数                  │
│  <metric>_bucket{le="0.05"}      -- ≤50ms 的请求数                  │
│  <metric>_bucket{le="+Inf"}      -- 所有请求数 (= _count)           │
│  <metric>_sum                    -- 所有值的总和                    │
│  <metric>_count                  -- 所有值的计数                    │
│                                                                     │
│  使用场景:                                                          │
│  ├── 请求延迟分布                                                   │
│  ├── 响应大小分布                                                   │
│  └── 计算分位数                                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```go
// Go 语言示例
var httpRequestDuration = prometheus.NewHistogramVec(
    prometheus.HistogramOpts{
        Name:    "http_request_duration_seconds",
        Help:    "HTTP request latency distribution",
        Buckets: []float64{0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1, 2.5, 5, 10},
    },
    []string{"method", "handler"},
)

// 记录请求
httpRequestDuration.WithLabelValues("GET", "/api/users").Observe(0.023)
```

Histogram 暴露的指标：

```promql
# 查看具体的桶
http_request_duration_seconds_bucket{handler="/api/users", le="0.1"}
# 结果: 950 (95% 的请求在 100ms 内完成)

# 计算 P95 延迟
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))
```

### 4. Summary（摘要）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Summary 摘要                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  特点: 直接计算并暴露分位数                                         │
│                                                                     │
│  用途: 获取精确的分位数                                             │
│                                                                     │
│  自动生成的时间序列:                                                 │
│  <metric>_seconds{quantile="0.5"}    -- P50                        │
│  <metric>_seconds{quantile="0.9"}    -- P90                        │
│  <metric>_seconds{quantile="0.99"}   -- P99                        │
│  <metric>_sum                          -- 所有值的总和              │
│  <metric>_count                        -- 所有值的计数              │
│                                                                     │
│  ⚠️ 注意: Summary 无法重新计算其他分位数                            │
│                                                                     │
│  对比 Histogram vs Summary:                                         │
│  ┌────────────────┬─────────────────────────┬────────────────────┐ │
│  │  特性          │ Histogram               │ Summary            │ │
│  ├────────────────┼─────────────────────────┼────────────────────┤ │
│  │ 分位数计算     │ 服务端计算              │ 客户端计算         │ │
│  │ 重新计算       │ ✅ 支持                 │ ❌ 不支持          │ │
│  │ 性能影响       │ 记录值 + 查找桶         │ 计算分位数         │ │
│  │ 精度           │ 取决于桶定义            │ 精确分位数         │ │
│  │ 推荐场景       │ 通用场景               │ 预知分位数需求    │ │
│  └────────────────┴─────────────────────────┴────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```go
// Go 语言示例
var httpRequestLatency = prometheus.NewSummaryVec(
    prometheus.SummaryOpts{
        Name:       "http_request_latency_seconds",
        Help:       "HTTP request latency",
        Objectives: map[float64]float64{0.5: 0.05, 0.9: 0.01, 0.99: 0.001},
    },
    []string{"method", "handler"},
)

// 记录请求
httpRequestLatency.WithLabelValues("GET", "/api/users").Observe(0.023)
```

## 指标类型对比

```
┌─────────────────────────────────────────────────────────────────────┐
│                    四种指标类型对比                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌──────────────┬──────────────┬──────────────┬──────────────┐      │
│  │   Counter    │    Gauge     │   Histogram  │   Summary    │      │
│  ├──────────────┼──────────────┼──────────────┼──────────────┤      │
│  │   只增不减   │   可增可减   │   区间统计   │   分位统计   │      │
│  ├──────────────┼──────────────┼──────────────┼──────────────┤      │
│  │    Inc()    │   Set()      │  Observe()  │  Observe()  │      │
│  │    Add()    │   Inc()      │              │              │      │
│  │             │   Dec()      │              │              │      │
│  ├──────────────┼──────────────┼──────────────┼──────────────┤      │
│  │  rate()     │  无          │ histogram_   │ quantile()  │      │
│  │  increase() │             │  quantile()  │              │      │
│  └──────────────┴──────────────┴──────────────┴──────────────┘      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

| 指标类型 | 特点 | 操作 | 适用场景 | 不适用场景 |
|---|---|---|---|---|
| Counter | 只增不减 | `Inc()`, `Add()` | 请求数、错误数 | 需要显示当前值 |
| Gauge | 可增可减 | `Set()`, `Inc()`, `Dec()` | CPU、内存、连接数 | 累计值 |
| Histogram | 区间分布 | `Observe()` | 延迟分布、计算分位数 | 精确分位数 |
| Summary | 分位统计 | `Observe()` | 精确 P99 | 需要事后计算其他分位数 |

## 常见问题

### 1. Counter vs Gauge 混淆

```go
// ❌ 错误：用 Gauge 统计请求数
var requestCount = prometheus.NewGauge(prometheus.GaugeOpts{
    Name: "request_count",
    Help: "Request count",
})
requestCount.Inc() // 每次请求 +1

// ✅ 正确：用 Counter 统计请求数
var requestCount = prometheus.NewCounter(prometheus.CounterOpts{
    Name: "request_count",
    Help: "Request count",
})
requestCount.Inc() // 每次请求 +1
```

### 2. Histogram vs Summary 选择

```go
// ✅ 通用场景用 Histogram（推荐）
// 原因：可以事后计算任意分位数
var latency = prometheus.NewHistogram(prometheus.HistogramOpts{
    Name:    "request_latency_seconds",
    Help:    "Request latency",
    Buckets: []float64{0.01, 0.05, 0.1, 0.5, 1, 5},
})

// 在 PromQL 中计算 P95
histogram_quantile(0.95, rate(latency_bucket[5m]))

// ❌ 只有预知分位数需求时才用 Summary
// 原因：无法事后计算其他分位数
var latency = prometheus.NewSummary(prometheus.SummaryOpts{
    Name:       "request_latency_seconds",
    Help:       "Request latency",
    Objectives: map[float64]float64{0.5: 0.05, 0.95: 0.01},
})
```

### 3. 标签基数问题

```promql
# ❌ 高基数标签（会导致内存爆炸）
http_requests_total{user_id="12345", request_id="abc123"}

// ✅ 低基数标签
http_requests_total{method="GET", status="200", handler="/api/users"}
```

## 总结

Prometheus 的四种指标类型各有用途：

1. **Counter**：累计计数器，只增不减，适合统计请求数、错误数等
2. **Gauge**：瞬时仪表，可增可减，适合测量 CPU、内存等当前值
3. **Histogram**：直方图，统计分布，适合分析延迟分布，支持事后计算任意分位数
4. **Summary**：摘要，直接计算分位数，适合需要精确分位数的场景

选择正确的指标类型，可以让后续的数据分析事半功倍。
