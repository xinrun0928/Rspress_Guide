# TSM 与远程存储

Prometheus 的本地存储设计，限制了你只能存储 15-90 天的数据。但业务需要看一年前的历史趋势，Prometheus 本身做不到。

远程存储，正是解决这个问题的方案。

### 本地存储的限制

| 限制 | 说明 | 影响 |
|---|---|---|
| 单机存储 | 无法水平扩展 | 无法存储 PB 级数据 |
| 保留期 | 默认 15 天 | 无法查看长期趋势 |
| 副本 | 无内置副本 | 节点故障数据丢失 |
| 容量 | 受单机磁盘限制 | 大规模部署受限 |

### 远程写入协议

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Prometheus 远程写入架构                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────┐      Remote Write       ┌─────────────┐          │
│  │ Prometheus  │ ─────────────────────► │  Remote     │          │
│  │             │                         │  Storage    │          │
│  │  TSDB       │ ◄───────────────────── │  (Thanos/   │          │
│  │             │      Remote Read       │   Cortex/   │          │
│  └─────────────┘                         │   M3DB)     │          │
│                                           └──────┬──────┘          │
│                                                  │                  │
│                                                  ▼                  │
│                                           ┌─────────────┐          │
│                                           │   对象存储   │          │
│                                           │ (S3/GCS)    │          │
│                                           └─────────────┘          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 配置示例

```yaml title="prometheus.yml"
remote_write:
  - url: http://thanos-receive:19291/api/v1/receive
    name: "thanos"
    # 写入队列配置
    queue_config:
      capacity: 10000           # 队列容量
      max_shards: 30            # 最大并发写入数
      min_shards: 10            # 最小并发写入数
      max_samples_per_send: 2000 # 每批最大样本数

remote_read:
  - url: http://thanos-query:10912/api/v1/read
    name: "thanos"
    read_recent: true          # 从远程读取历史数据
```

## Thanos 架构

### 核心组件

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Thanos 架构                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                     Thanos Query                              │  │
│  │                     (统一查询入口)                            │  │
│  └──────────────────────────────────────────────────────────────┘  │
│        ▲                    ▲                    ▲                  │
│        │                    │                    │                  │
│        │                    │                    │                  │
│  ┌─────┴─────┐         ┌────┴────┐         ┌────┴────┐           │
│  │  Sidecar  │         │ Sidecar │         │  Ruler  │           │
│  │ (StoreAPI)│         │(StoreAPI)│        │(告警/记录)│          │
│  └─────┬─────┘         └────┬────┘         └─────────┘           │
│        │                    │                                      │
│        └────────┬───────────┘                                      │
│                 │                                                  │
│                 ▼                                                  │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                     对象存储 (S3/GCS/MinIO)                    │  │
│  │                     长期存储所有历史数据                        │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                     Thanos Receive                             │  │
│  │                     (接收远程写入)                              │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 核心特性

| 组件 | 功能 | 说明 |
|---|---|---|
| `Sidecar` | 数据共享 | 读取 Prometheus 本地数据，通过 StoreAPI 暴露 |
| `Store` | 数据查询 | 从对象存储读取历史数据 |
| `Query` | 统一入口 | 聚合多个 Store 的数据进行查询 |
| `Receive` | 接收写入 | 接收 Prometheus 的 Remote Write |
| `Ruler` | 告警/记录 | 基于历史数据的告警和录制规则 |
| `Compact` | 数据压缩 | 合并小 Block，清理过期数据 |

### 部署示例

```yaml title="thanos-query-deployment.yaml"
apiVersion: apps/v1
kind: Deployment
metadata:
  name: thanos-query
spec:
  replicas: 2
  template:
    spec:
      containers:
        - name: thanos
          image: quay.io/thanos/thanos:v0.32.0
          args:
            - query
            - --http-address=0.0.0.0:9090
            - --store=prometheus-1:10901
            - --store=prometheus-2:10901
            - --store=thanos-store:10901
          ports:
            - containerPort: 9090
```

| 场景 | 推荐方案 | 原因 |
|---|---|---|
| 小规模/个人项目 | VictoriaMetrics | 轻量、简单、低资源 |

| 已有 InfluxDB | InfluxDB Remote | 复用现有基础设施 |

## 总结

Prometheus 的本地存储限制了数据的保留期和规模：

1. **本地 TSDB**：适合短期数据 (< 30天)，单机部署
2. **远程写入**：通过 Remote Write 协议将数据发送到外部存储
3. **Thanos**：CNCF 项目，适合 Kubernetes 环境，提供统一查询
4. **VictoriaMetrics**：轻量级高性能，适合中小规模
5. **Cortex**：多租户支持，适合 SaaS 平台
6. **M3DB**：超大规模，Uber 生产验证

选择远程存储方案时，需要考虑数据规模、团队能力、运维成本等因素。
