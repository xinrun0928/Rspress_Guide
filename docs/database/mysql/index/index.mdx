# 索引专题总览

索引是数据库查询性能的核心。面试中，索引几乎是逢问必考的知识点——索引底层数据结构、B+ 树原理、聚簇索引与覆盖索引、最左前缀原则、索引失效场景——每一个点都可能成为面试官深挖的方向。

本模块带你深入理解 MySQL 索引的底层原理，掌握索引设计的最佳实践。

## 为什么要用索引？

假设有一张 100 万行的用户表，没有索引时按 name 查询：

```sql
SELECT * FROM users WHERE name = '张三';
```

数据库需要全表扫描，逐行比较 name 字段，最多需要 100 万次磁盘 IO。创建索引后：

```sql
CREATE INDEX idx_name ON users(name);
```

查询变成索引查找，只需几次磁盘 IO 就能定位到目标记录。索引就像书的目录——没有目录，找内容要翻完整本书；有目录，直接翻到对应章节。

## MySQL 索引体系

MySQL 支持多种类型的索引：

**B+ 树索引**：MySQL 最主要的索引类型，InnoDB 和 MyISAM 都支持。B+ 树是多路平衡查找树，支持范围查询，适合排序。

**Hash 索引**：基于哈希表实现，只支持等值查询，不支持范围查询和排序。Memory 引擎支持 Hash 索引，InnoDB 有自适应哈希索引（AHI）。

**全文索引**：用于文本搜索，支持 `MATCH AGAINST` 语法。MyISAM 支持全文索引，InnoDB 从 5.6 版本开始支持。

**空间索引**：用于地理空间数据类型，如 `GEOMETRY`、`POINT`。只有 MyISAM 支持（R-Tree）。

## 索引的核心概念

**聚簇索引 vs 非聚簇索引**：聚簇索引的叶子节点存储完整数据行，一个表只有一个聚簇索引；非聚簇索引的叶子节点存储索引值和主键值，需要回表查询完整数据。

**覆盖索引**：查询的所有列都在索引中，无需回表，是重要的优化手段。

**联合索引**：多个列组成的索引，遵循最左前缀原则。

**索引下推**：Index Condition Pushdown，ICP，在索引层面过滤数据，减少回表次数。
