# COUNT 性能优化：COUNT(*) vs COUNT(1) vs COUNT(column)

「COUNT(*) 和 COUNT(1) 哪个更快？如何优化 COUNT 查询？」

COUNT 查询看似简单，但在大表上可能很慢。本文深入剖析 COUNT 的性能差异和优化方法。

## COUNT 家族

### COUNT 的几种写法

```sql
-- COUNT(*)
SELECT COUNT(*) FROM orders;

-- COUNT(1)
SELECT COUNT(1) FROM orders;

-- COUNT(column)
SELECT COUNT(id) FROM orders;
SELECT COUNT(user_id) FROM orders;  -- 可能有 NULL
SELECT COUNT(DISTINCT user_id) FROM orders;

-- COUNT(expression)
SELECT COUNT(IF(status = 'completed', 1, NULL)) FROM orders;
```

## COUNT(column) 的区别

### COUNT(*) vs COUNT(column)

```sql
-- COUNT(*)：统计所有行，包括 NULL
SELECT COUNT(*) FROM orders;
-- 返回：总行数

-- COUNT(column)：只统计非 NULL 的行
SELECT COUNT(user_id) FROM orders;
-- 返回：user_id 不为 NULL 的行数
```

### 示例

```sql
-- 创建测试数据
INSERT INTO orders (id, user_id, amount) VALUES
(1, 1, 100),   -- user_id = 1
(2, NULL, 200), -- user_id = NULL
(3, 1, 150);   -- user_id = 1

SELECT COUNT(*) FROM orders;          -- 结果：3
SELECT COUNT(1) FROM orders;          -- 结果：3
SELECT COUNT(user_id) FROM orders;    -- 结果：2（忽略 NULL）
```

## COUNT 优化方案

### 方案一：使用 EXPLAIN 估算

```sql
-- 使用 EXPLAIN 估算行数
EXPLAIN SELECT COUNT(*) FROM orders;

-- rows 字段就是估算值
```

```
+----+-------------+-------+------+------+--------+------+----------+-------+
| id | select_type| table | type | key  |key_len | ref  | rows     | Extra |
+----+-------------+-------+------+------+--------+------+----------+-------+
|  1 | SIMPLE     | orders| index| PRIMARY| 8    | NULL | 10000000 |       |
+----+-------------+-------+------+------+--------+------+----------+-------+
-- rows: 10000000（估算值）
```

### 方案二：维护统计表

```sql
-- 创建统计表
CREATE TABLE order_stats (
    stat_date DATE PRIMARY KEY,
    order_count BIGINT
);

-- 定时任务更新统计表
INSERT INTO order_stats (stat_date, order_count)
VALUES (CURDATE(), (SELECT COUNT(*) FROM orders))
ON DUPLICATE KEY UPDATE order_count = (SELECT COUNT(*) FROM orders);

-- 查询总数
SELECT order_count FROM order_stats ORDER BY stat_date DESC LIMIT 1;
```

### 方案三：使用触发器

```sql
-- 创建统计表
CREATE TABLE order_count (
    total BIGINT DEFAULT 0
);
INSERT INTO order_count VALUES (0);

-- 创建触发器
DELIMITER //

CREATE TRIGGER after_order_insert
AFTER INSERT ON orders
FOR EACH ROW
BEGIN
    UPDATE order_count SET total = total + 1;
END//

CREATE TRIGGER after_order_delete
AFTER DELETE ON orders
FOR EACH ROW
BEGIN
    UPDATE order_count SET total = total - 1;
END//

DELIMITER ;

-- 查询总数
SELECT total FROM order_count;
```

### 方案四：使用缓存

```sql
-- 使用 Redis 缓存总数
-- 伪代码
function getOrderCount() {
    $count = Redis::get('order_count');
    if ($count === null) {
        $count = DB::select("SELECT COUNT(*) FROM orders");
        Redis::set('order_count', $count, 3600);  // 缓存 1 小时
    }
    return $count;
}
```

## COUNT 与索引

### 索引对 COUNT 的影响

```sql
-- 主键索引
SELECT COUNT(*) FROM orders;
-- 使用主键索引，性能好

-- 普通索引
SELECT COUNT(status) FROM orders;
-- 使用 status 索引

-- 无索引
SELECT COUNT(amount) FROM orders;
-- amount 没有索引，全表扫描
```

### 创建优化索引

```sql
-- 为 COUNT 查询创建索引
CREATE INDEX idx_status ON orders(status);

-- 对于 COUNT(*) 优化不大
-- 对于 COUNT(status) 有帮助
```

## 面试实战

### 问题一：COUNT(*) 和 COUNT(1) 哪个更快？

**面试官问**：「COUNT(*) 和 COUNT(1) 有什么区别？哪个更快？」

**深入回答**：

> **结论：COUNT(*) 更快（或一样快）**
>
> MySQL 8.0 之前：
> - 两者性能几乎相同
> - MySQL 优化器会将 COUNT(1) 转换为 COUNT(*)
>
> MySQL 8.0 之后：
> - COUNT(*) 有专门优化，不需要检查数据
> - COUNT(1) 仍需处理每一行
>
> **建议**：使用 COUNT(*)，这是标准且最优的写法

### 问题二：COUNT(column) 和 COUNT(*) 有什么区别？

**面试官问**：「COUNT(column) 和 COUNT(*) 有什么区别？」

**深入回答**：

> **区别**：COUNT(column) 忽略 NULL 值
>
> | 查询 | 含义 |
> |---|---|
> | COUNT(*) | 统计所有行，包括 NULL |
> | COUNT(column) | 统计 column 不为 NULL 的行数 |
>
> ```sql
> SELECT COUNT(*) FROM orders;          -- 总行数
> SELECT COUNT(user_id) FROM orders;     -- user_id 不为 NULL 的行数
> ```

### 问题三：如何优化大表的 COUNT 查询？

**面试官问**：「大表 COUNT 查询很慢怎么优化？」

**深入回答**：

> **优化方法**：
>
> 1. **使用 EXPLAIN 估算**：
> ```sql
> EXPLAIN SELECT COUNT(*) FROM orders;
> ```
>
> 2. **维护统计表**：定期计算 COUNT 并存储
> ```sql
> CREATE TABLE order_stats (
>     stat_date DATE PRIMARY KEY,
>     order_count BIGINT
> );
> ```
>
> 3. **使用缓存**：Redis 等缓存 COUNT 值
>
> 4. **接受估算值**：对于实时性要求不高的场景，使用统计值

## 总结

COUNT 查询优化要点：

- **COUNT(*) 最优**：MySQL 专门优化
- **COUNT(column)**：会忽略 NULL
- **COUNT(DISTINCT)**：最慢，需要去重
- **大表优化**：使用统计表、缓存

掌握 COUNT 的性能差异，是 SQL 优化的基础。
