# 子查询分类与优化

「子查询一定比 JOIN 慢吗？如何优化子查询？」

子查询和 JOIN 是两种不同的查询方式，各有优劣。本文深入剖析子查询的分类和优化技巧。

## 子查询概述

### 什么是子查询？

子查询是指在一个 SELECT 语句中嵌套另一个 SELECT 语句。

```
┌─────────────────────────────────────────────────────────────────┐
│                    子查询结构                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  SELECT column                                          │   │
│  │  FROM table1                                           │   │
│  │  WHERE column OPERATOR (                               │   │
│  │      SELECT column FROM table2                         │   │

│  └─────────────────────────────────────────────────────────┘   │
```

```
└─────────────────────────────────────────────────────────────────┘
```

## 子查询与 JOIN 的对比

### 对比示例

```sql
-- 子查询方式
SELECT * FROM users
WHERE id IN (SELECT user_id FROM orders WHERE amount > 100);

-- JOIN 方式
SELECT DISTINCT u.*
FROM users u
INNER JOIN orders o ON u.id = o.user_id
WHERE o.amount > 100;
```

```

│  │  子查询优点：                                            │   │
│  │  - 语义清晰，易于理解                                    │   │
│  │  - 可以使用标量子查询返回单一值                          │   │
│  │  - 某些场景下 MySQL 优化更好                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  JOIN 优点：                                            │   │
│  │  - 可读性根据查询而异                                    │   │
```

## 子查询优化

### 问题：子查询性能差

```sql
-- 低效的子查询
SELECT * FROM users
WHERE id NOT IN (SELECT user_id FROM orders WHERE status = 'completed');

EXPLAIN SELECT * FROM users
WHERE id NOT IN (SELECT user_id FROM orders WHERE status = 'completed');
```

```
+----+-------------+--------+------+------+------+--------+-------+
| id | select_type| table  | type | key  |key_len | ref  | rows |
+----+-------------+--------+------+------+------+--------+-------+
|  1 | PRIMARY    | users  | ALL  | NULL |  NULL | NULL  | 1000 |
|  1 | PRIMARY    | users  | ALL  | NULL |  NULL | NULL  | 1000 |
|  2 | SUBQUERY   | orders | ALL  | NULL |  NULL | NULL  |10000 |
+----+-------------+--------+------+------+------+--------+-------+
-- 全表扫描！
```

### 优化一：使用 JOIN 替代

```sql
-- 用 LEFT JOIN + IS NULL 替代 NOT IN
SELECT u.*
FROM users u
LEFT JOIN orders o ON u.id = o.user_id AND o.status = 'completed'
WHERE o.id IS NULL;

EXPLAIN SELECT u.*
FROM users u
LEFT JOIN orders o ON u.id = o.user_id AND o.status = 'completed'
WHERE o.id IS NULL;
```

```
+----+-------------+-------+--------+------+---------------+-------------+---------+------------------+------+-------+
| id | select_type| table | type   | key  | key_len       | ref        | rows    | filtered         | Extra|       |
+----+-------------+-------+--------+------+---------------+-------------+---------+------------------+------+-------+
|  1 | SIMPLE     | u     | ALL    | NULL |  NULL         | NULL       |    1000 |  100.00|                |       |
|  1 | SIMPLE     | o     | ref    | idx_user_id | 5   | u.id       |       3 |   10.00|Using where|
+----+-------------+-------+--------+------+---------------+-------------+---------+------------------+------+-------+
-- 使用索引了！
```

### 优化二：使用 NOT EXISTS

```sql
-- 用 NOT EXISTS 替代 NOT IN
SELECT * FROM users u
WHERE NOT EXISTS (
    SELECT 1 FROM orders o
    WHERE o.user_id = u.id AND o.status = 'completed'
);

EXPLAIN SELECT * FROM users u
WHERE NOT EXISTS (
    SELECT 1 FROM orders o
    WHERE o.user_id = u.id AND o.status = 'completed'
);
```

```
+----+-------------+-------+--------+------+---------------+-------------+---------+------------------+-------------+
| id | select_type| table | type   | key  | key_len       | ref        | rows    | filtered         | Extra       |
+----+-------------+-------+--------+------+---------------+-------------+---------+------------------+-------------+
|  1 | SIMPLE     | u     | ALL    | NULL |  NULL         | NULL       |    1000 |  100.00|                |             |
|  1 | SIMPLE     | o     | ref    | idx_user_id | 5   | u.id       |       3 |   10.00|Using where|

+----+-------------+-------+--------+------+---------------+-------------+---------+------------------+-------------+

```

### 优化三：使用 IN vs EXISTS

```sql
-- IN 子查询
SELECT * FROM users
WHERE id IN (SELECT user_id FROM orders WHERE amount > 100);

-- EXISTS 子查询
SELECT * FROM users u
WHERE EXISTS (
    SELECT 1 FROM orders o
    WHERE o.user_id = u.id AND o.amount > 100
);
```

```

│                    IN vs EXISTS                                 │
├─────────────────────────────────────────────────────────────────┤

│  ┌─────────────────────────────────────────────────────────┐   │
│  │  MySQL 对 IN 和 EXISTS 的优化：                         │   │
│  │                                                         │   │
│  │  - IN 子查询：如果子查询列有索引，效果好                │   │
│  │  - EXISTS：外层表使用索引时效果好                        │   │
│  │                                                         │   │
│  │  一般情况下，性能相近                                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │

│  - 语义清晰优先                                               │

│  - 外层表大、内层表小：用 IN                                   │
│                                                                 │

```

## 常见子查询模式

### 模式一：IN 子查询

```sql
-- 原始
SELECT * FROM orders
WHERE user_id IN (SELECT id FROM users WHERE status = 'active');

-- 优化：添加索引
CREATE INDEX idx_user_id ON users(status, id);
CREATE INDEX idx_user_id ON orders(user_id);

-- 验证
EXPLAIN SELECT * FROM orders
WHERE user_id IN (SELECT id FROM users WHERE status = 'active');
```

### 模式二：标量子查询

```sql
-- 获取每个用户的最新订单
SELECT o.*,
    (SELECT name FROM users WHERE id = o.user_id) as user_name
FROM orders o;

-- 优化：使用 JOIN
SELECT o.*, u.name
FROM orders o
INNER JOIN users u ON o.user_id = u.id;
```

### 模式三：FROM 子句子查询（派生表）

```sql
-- 派生表
SELECT AVG(order_count)
FROM (
    SELECT user_id, COUNT(*) as order_count
    FROM orders
    GROUP BY user_id
) as user_stats;

-- 注意：派生表必须有别名
```

## 子查询与 JOIN 转换

### NOT IN → LEFT JOIN + IS NULL

```sql
-- NOT IN
SELECT * FROM users
WHERE id NOT IN (SELECT user_id FROM orders);

-- LEFT JOIN + IS NULL
SELECT u.*
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.id IS NULL;
```

### NOT EXISTS → LEFT JOIN + IS NULL

```sql
-- NOT EXISTS
SELECT * FROM users u
WHERE NOT EXISTS (
    SELECT 1 FROM orders o WHERE o.user_id = u.id
);

-- LEFT JOIN + IS NULL
SELECT u.*
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.id IS NULL;
```

### IN → JOIN

```sql
-- IN
SELECT * FROM users
WHERE id IN (SELECT user_id FROM orders WHERE amount > 100);

-- JOIN + DISTINCT
SELECT DISTINCT u.*
FROM users u
INNER JOIN orders o ON u.id = o.user_id
WHERE o.amount > 100;
```

## 面试实战

### 问题一：子查询和 JOIN 哪个更快？

**面试官问**：「子查询一定比 JOIN 慢吗？」

**深入回答**：

> **不一定。性能取决于具体情况**：
>
> - MySQL 对子查询和 JOIN 都有优化
> - 大多数情况下，性能相近
> - 但某些场景下差异很大：
>   - `NOT IN` 子查询通常比 `LEFT JOIN + IS NULL` 慢
>   - `IN` 子查询有索引时，性能与 JOIN 相近
> - 建议：根据语义选择易读的方式，性能有问题时再优化

### 问题二：如何优化子查询？

**面试官问**：「子查询性能差怎么优化？」

**深入回答**：

> **优化方法**：
>
> 1. **添加索引**：子查询列和 JOIN 列都要有索引
> 2. **改写为 JOIN**：`NOT IN` 改为 `LEFT JOIN + IS NULL`
> 3. **使用 EXISTS**：`IN` 子查询可以尝试 `EXISTS`
> 4. **避免相关子查询**：相关子查询每次外层行都要执行
>
> ```sql
> -- 优化 NOT IN
> SELECT * FROM users WHERE id NOT IN (SELECT user_id FROM orders);
> -- 改为
> SELECT u.* FROM users u LEFT JOIN orders o ON u.id = o.user_id WHERE o.id IS NULL;
> ```

### 问题三：什么是相关子查询？

**面试官问**：「什么是相关子查询？为什么性能差？」

**深入回答**：

> **相关子查询**：子查询依赖外层查询的值
>
> ```sql
> SELECT * FROM orders o
> WHERE amount > (SELECT AVG(amount) FROM orders WHERE user_id = o.user_id);
> -- 子查询依赖外层的 o.user_id
> ```
>
> **性能问题**：外层每行都要执行一次子查询
> - 1000 行订单，就要执行 1000 次子查询
>
> **优化方法**：使用 JOIN 或窗口函数
> ```sql
> SELECT o.*
> FROM orders o
> INNER JOIN (
>     SELECT user_id, AVG(amount) as avg_amount
>     FROM orders
>     GROUP BY user_id
> ) t ON o.user_id = t.user_id
> WHERE o.amount > t.avg_amount;
> ```

## 总结

子查询优化要点：

- **分类**：标量子查询、列子查询、表子查询
- **相关 vs 非相关**：相关子查询性能差
- **优化方法**：添加索引、改写为 JOIN
- **NOT IN 优化**：改为 `LEFT JOIN + IS NULL`

掌握子查询优化，是 SQL 优化的重要技能。
