# DML 优化：INSERT、UPDATE、DELETE

「如何让批量插入更快？如何安全地执行 UPDATE 和 DELETE？」

DML（Data Manipulation Language）优化是后端开发的必备技能。本文介绍 INSERT、UPDATE、DELETE 的优化技巧。

## INSERT 优化

### 批量插入

```sql
-- 逐条插入（慢）
INSERT INTO users VALUES (1, '张三');
INSERT INTO users VALUES (2, '李四');
INSERT INTO users VALUES (3, '王五');

-- 批量插入（快）
INSERT INTO users VALUES
(1, '张三'),
(2, '李四'),
(3, '王五');
```

```sql
-- LOAD DATA 是最快的导入方式
LOAD DATA INFILE '/tmp/users.csv'
INTO TABLE users
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
(id, name);
```

### 插入优化配置

```sql
-- 查看配置
SHOW VARIABLES LIKE 'bulk_insert_buffer_size';
-- 默认：8MB

SHOW VARIABLES LIKE 'max_allowed_packet';
-- 默认：4MB

-- 调整配置
SET GLOBAL bulk_insert_buffer_size = 256 * 1024 * 1024;  -- 256MB
SET GLOBAL max_allowed_packet = 64 * 1024 * 1024;        -- 64MB
```

### 插入时关闭索引

```sql
-- 禁用索引
ALTER TABLE orders DISABLE KEYS;

-- 批量插入
INSERT INTO orders VALUES ...;

-- 启用索引
ALTER TABLE orders ENABLE KEYS;
```

## UPDATE 优化

### 基本 UPDATE

```sql
-- 更新单行
UPDATE users SET name = '新名字' WHERE id = 1;

-- 更新多行
UPDATE users SET status = 'inactive' WHERE last_login < '2023-01-01';
```

### UPDATE 优化技巧

```sql
-- 不好的写法：每次更新都重新查询
UPDATE orders o,
    (SELECT SUM(amount) as total FROM orders WHERE user_id = 1) t
SET o.status = 'completed'
WHERE o.user_id = 1;

-- 好的写法：使用 LIMIT 限制每次更新数量
UPDATE orders SET status = 'completed'
WHERE status = 'pending' AND id <= 1000;

-- 分批更新
UPDATE orders SET status = 'completed'
WHERE status = 'pending' AND id BETWEEN 1 AND 1000;

UPDATE orders SET status = 'completed'
WHERE status = 'pending' AND id BETWEEN 1001 AND 2000;
```

### UPDATE 与索引

```sql
-- 确保 WHERE 条件有索引
UPDATE users SET name = '新名字'
WHERE status = 'active' AND last_login < '2023-01-01';

-- 添加索引
CREATE INDEX idx_status_login ON users(status, last_login);
```

### 安全 UPDATE 技巧

```sql
-- 先查询确认
SELECT * FROM users WHERE id = 1;
-- 确认数据后再更新

-- 使用事务
BEGIN;
SELECT * FROM users WHERE id = 1 FOR UPDATE;
UPDATE users SET name = '新名字' WHERE id = 1;
-- 检查结果
SELECT * FROM users WHERE id = 1;
COMMIT;
```

## DELETE 优化

### 基本 DELETE

```sql
-- 删除单行
DELETE FROM users WHERE id = 1;

-- 删除多行
DELETE FROM users WHERE status = 'deleted' AND delete_time < '2023-01-01';
```

### DELETE 优化技巧

```sql
-- 分批删除：避免长时间锁表
DELETE FROM orders WHERE status = 'completed'
ORDER BY id LIMIT 1000;
-- 执行 1000 次，删除 100 万行

-- 删除旧数据示例
DELIMITER //

CREATE PROCEDURE delete_old_orders()
BEGIN
    DECLARE done INT DEFAULT FALSE;
    DECLARE last_id INT DEFAULT 0;

    REPEAT
        SELECT MIN(id) INTO last_id FROM orders
        WHERE id > last_id AND create_time < '2022-01-01';

        IF last_id IS NOT NULL THEN
            DELETE FROM orders WHERE id = last_id;
        END IF;

        UNTIL last_id IS NULL END REPEAT;
END//

DELIMITER ;

-- 调用存储过程
CALL delete_old_orders();
```

### TRUNCATE vs DELETE

```sql
-- TRUNCATE 更快，但不可回滚
TRUNCATE TABLE orders;

-- DELETE 可以回滚
DELETE FROM orders;
```

## 事务与 DML

### 批量 DML 使用事务

```sql
-- 开启事务
BEGIN;

-- 批量 UPDATE
UPDATE users SET points = points + 100 WHERE id BETWEEN 1 AND 1000;

-- 批量 DELETE
DELETE FROM logs WHERE create_time < '2023-01-01';

-- 提交
COMMIT;

-- 或者回滚
-- ROLLBACK;
```

### 事务大小控制

```sql
-- 不好：大事务
BEGIN;
-- 处理 100 万行数据
COMMIT;
-- 执行时间长，占用大量 undo log

-- 好：小事务分批处理
BEGIN;
-- 处理 1000 行
COMMIT;

BEGIN;
-- 处理下一批 1000 行
COMMIT;
```

## DML 优化配置

### 关键配置参数

```sql
SHOW VARIABLES LIKE 'innodb_flush_log_at_trx_commit';
-- 1：每次提交都刷盘（安全）
-- 2：每秒刷盘（较快）
-- 0：最慢（不建议）

SHOW VARIABLES LIKE 'sync_binlog';
-- 1：每次提交都同步 binlog（安全）
-- N：每 N 次提交同步一次（较快）
```

### 配置建议

```ini
[mysqld]
# 批量插入优化
bulk_insert_buffer_size = 256M

# 批量导入允许
max_allowed_packet = 64M

# 事务日志刷盘策略（安全模式）
innodb_flush_log_at_trx_commit = 1
sync_binlog = 1

# 如果可以接受数据丢失，可设为：
# innodb_flush_log_at_trx_commit = 2
# sync_binlog = 1000
```

## 面试实战

### 问题一：如何快速插入 100 万行数据？

**面试官问**：「需要快速插入 100 万行数据，怎么做？」

**深入回答**：

> **最快方法：LOAD DATA INFILE**
>
> ```sql
> -- 1. 准备 CSV 文件
> -- 2. 执行 LOAD DATA
> LOAD DATA INFILE '/tmp/data.csv'
> INTO TABLE orders
> FIELDS TERMINATED BY ','
> LINES TERMINATED BY '\n';
> ```
>
> **性能对比**：
> - INSERT 批量：30-60 秒
> - LOAD DATA：3-5 秒
>
> **优化技巧**：
> - 禁用唯一检查和外键检查
> - 禁用索引，完成后重建

### 问题二：如何安全地执行批量 UPDATE？

**面试官问**：「需要批量更新 100 万行数据，怎么做才安全？」

**深入回答**：

> **安全批量 UPDATE 方法**：
>
> 1. **分批执行**：
> ```sql
> UPDATE orders SET status = 'completed'
> WHERE id BETWEEN 1 AND 1000;
> -- 循环执行 1000 次
> ```
>
> 2. **使用事务**：
> ```sql
> BEGIN;
> -- 每批 1000 行
> UPDATE orders SET status = 'completed'
> WHERE id BETWEEN 1 AND 1000;
> COMMIT;
> ```
>
> 3. **先备份**：执行前先备份表

### 问题三：DELETE 和 TRUNCATE 有什么区别？

**面试官问**：「DELETE 和 TRUNCATE 有什么区别？」

**深入回答**：

> | 维度 | DELETE | TRUNCATE |
> |---|---|---|
> | 速度 | 慢（逐行删除） | 快（物理删除） |
> | 可回滚 | 可以 | 不可以 |
> | 重置 AUTO_INCREMENT | 不 | 是 |
> | WHERE 条件 | 支持 | 不支持 |
> | 触发 DELETE 触发器 | 是 | 否 |
>
> **选择**：
> - 清空表：TRUNCATE
> - 按条件删除：DELETE

## 总结

DML 优化要点：

- **INSERT**：批量插入、LOAD DATA、禁用索引
- **UPDATE**：分批执行、使用索引、注意事务
- **DELETE**：分批删除、软删除优先

掌握这些技巧，让 DML 操作飞起来！
