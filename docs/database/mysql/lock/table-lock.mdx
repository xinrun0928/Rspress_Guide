# 全局锁与表级锁：MDL 锁

「什么情况下表会被锁住，导致整个库的所有操作都变慢？」

这个问题可能和 MDL 锁有关。MySQL 的锁机制分为多个层次，其中表级锁是日常最容易遇到的一类。

## 全局锁

### 什么是全局锁？

全局锁是锁住整个数据库实例，所有表都无法读写。

### FLUSH TABLES WITH READ LOCK

```sql
-- 加全局读锁
FLUSH TABLES WITH READ LOCK;

-- 解锁
UNLOCK TABLES;
```

### 全局锁的特点

```
┌─────────────────────────────────────────────────────────────────┐
│                    全局锁的特点                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  执行后：                                               │   │
│  │                                                         │   │
│  │  - 所有表被锁定，只允许读                               │   │
│  │  - 所有写操作被阻塞                                     │   │
│  │  - 其他线程无法执行 DDL                                 │   │
│  │  - 其他线程无法执行 DML                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  主要用途：                                              │   │
│  │                                                         │   │
│  │  - 全库备份（mysqldump 使用）                           │   │
│  │  - 的一致性快照                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 全局锁的影响

```sql
-- 线程 1：加全局读锁
FLUSH TABLES WITH READ LOCK;

-- 线程 2：尝试写入
INSERT INTO users VALUES (1, '张三');
-- 被阻塞！

-- 线程 3：尝试读取
SELECT * FROM users;
-- 可以读取（因为是读锁）

-- 线程 4：尝试 DDL
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
-- 被阻塞！

-- 解锁
UNLOCK TABLES;
```

### MySQLDump 的备份原理

```bash
# mysqldump 使用全局锁进行备份
mysqldump -uroot -p --single-transaction --master-data=2 dbname > backup.sql

# --single-transaction 的原理：
# 1. 先 FLUSH TABLES WITH READ LOCK
# 2. 开启事务（REPEATABLE READ）
# 3. 读取数据（快照读）
# 4. UNLOCK TABLES
# 5. 导出数据
```

:::warning 全局锁的影响

全局锁会锁住整个数据库，对业务影响很大。在生产环境中执行前必须谨慎评估。
:::

## MDL 锁（Meta Data Lock）

### 什么是 MDL 锁？

MDL 锁是 MySQL 5.5 引入的表级锁，用于保护表结构。当对表执行 DDL（ALTER TABLE、CREATE INDEX 等）或 DML 操作时，会自动加 MDL 锁。

### MDL 锁的类型

```
┌─────────────────────────────────────────────────────────────────┐
│                    MDL 锁类型                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  MDL_SHARED_READ                                       │   │
│  │  - 共享锁                                               │   │
│  │  - 读操作获取                                           │   │
│  │  - 兼容其他 MDL_SHARED_READ                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  MDL_SHARED_WRITE                                      │   │
│  │  - 共享锁                                               │   │
│  │  - 写操作获取                                           │   │
│  │  - 兼容 MDL_SHARED_READ                                │   │
│  │  - 阻塞其他 MDL_SHARED_WRITE                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  MDL_SHARED_READ_WRITE                                 │   │
│  │  - 共享锁                                               │   │
│  │  - 读写操作获取                                         │   │
│  │  - 与 MDL_SHARED_READ 兼容                             │   │
│  │  - 与 MDL_SHARED_WRITE 互斥                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  MDL_EXCLUSIVE                                         │   │
│  │  - 排他锁                                               │   │
│  │  - DDL 操作获取                                         │   │
│  │  - 与所有其他 MDL 锁互斥                                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### MDL 锁的兼容矩阵

| 请求锁 | 已持有 SHARED_READ | SHARED_WRITE | SHARED_READ_WRITE | EXCLUSIVE |
|---|---|---|---|---|
| SHARED_READ | ✓ | ✓ | ✓ | ✗ |
| SHARED_WRITE | ✓ | ✗ | ✗ | ✗ |
| SHARED_READ_WRITE | ✓ | ✗ | ✗ | ✗ |
| EXCLUSIVE | ✗ | ✗ | ✗ | ✗ |

### MDL 锁的触发场景

```sql
-- 场景 1：查询（获取 MDL_SHARED_READ）
SELECT * FROM users;

-- 场景 2：插入/更新（获取 MDL_SHARED_WRITE）
INSERT INTO users VALUES (1, '张三');
UPDATE users SET name = '李四' WHERE id = 1;

-- 场景 3：DDL（获取 MDL_EXCLUSIVE）
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
DROP TABLE users;
CREATE INDEX idx_name ON users(name);
```

### MDL 锁的典型问题

#### 问题场景：长查询阻塞 DDL

```sql
-- 线程 1：开启事务，执行查询（获取 SHARED_READ）
BEGIN;
SELECT * FROM users;
-- 不提交事务！

-- 线程 2：尝试执行 DDL（需要 EXCLUSIVE 锁）
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
-- 被阻塞！因为线程 1 持有 SHARED_READ

-- 线程 3：尝试执行 DML（获取 SHARED_WRITE）
INSERT INTO users VALUES (2, '王五');
-- 被阻塞！因为线程 2 等待 EXCLUSIVE，线程 1 持有 SHARED_READ
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    MDL 锁等待链                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  线程 1：SELECT（SHARED_READ）←───────────────┐               │
│                                               │                 │
│  线程 3：INSERT（SHARED_WRITE）──→ 阻塞！←───┤                 │
│                                               │                 │
│  线程 2：ALTER（EXCLUSIVE）──→ 阻塞！←───────┘                 │
│                                               │                 │
│  等待链：线程 1 → 线程 2 → 线程 3                                │
│  形成死锁！                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### MDL 锁监控

```sql
-- 查看 MDL 锁状态
SHOW PROCESSLIST;
-- 或
SELECT * FROM information_schema.PROCESSLIST;

-- 查看 MDL 锁详情（需要开启 performance_schema）
SELECT
    OBJECT_TYPE,
    OBJECT_SCHEMA,
    OBJECT_NAME,
    LOCK_TYPE,
    LOCK_STATUS,
    THREAD_ID
FROM performance_schema.metadata_locks
WHERE OBJECT_TYPE = 'TABLE';
```

### MDL 锁问题排查

```sql
-- 查看长事务（可能导致 MDL 锁等待）
SELECT
    trx_mysql_thread_id,
    trx_started,
    trx_state,
    trx_query
FROM information_schema.INNODB_TRX
WHERE trx_started < NOW() - INTERVAL 10 SECOND;

-- 杀掉阻塞的线程
KILL <thread_id>;
```

## 表锁（LOCK TABLE）

### LOCK TABLE 语句

```sql
-- 加表级读锁
LOCK TABLES users READ;

-- 加表级写锁
LOCK TABLES users WRITE;

-- 解锁
UNLOCK TABLES;
```

### 表锁的特点

```
┌─────────────────────────────────────────────────────────────────┐
│                    表锁的特点                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  LOCK TABLES ... READ（表级读锁）：                       │   │
│  │                                                         │   │
│  │  - 当前线程可以读                                       │   │
│  │  - 其他线程可以读（但不能写）                           │   │
│  │  - 其他线程写操作被阻塞                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  LOCK TABLES ... WRITE（表级写锁）：                      │   │
│  │                                                         │   │
│  │  - 只有当前线程可以读写                                 │   │
│  │  - 其他线程读写都被阻塞                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 表锁 vs MDL 锁

| 维度 | 表锁（LOCK TABLE） | MDL 锁 |
|---|---|---|
| 获取方式 | 手动执行 LOCK TABLE | 自动获取 |
| 作用域 | 当前连接 | 所有连接 |
| 事务关系 | 释放时机可控制 | 与事务相关 |
| 典型场景 | 数据迁移 | DDL/DML 操作 |

## 面试实战

### 问题一：什么是 MDL 锁？它有什么用？

**面试官问**：「什么是 MDL 锁？它有什么用？」

**深入回答**：

> MDL（Meta Data Lock）是 MySQL 5.5 引入的表级锁，用于保护表结构。
>
> **作用**：
> - DML 操作需要获取 MDL_SHARED_WRITE
> - DDL 操作需要获取 MDL_EXCLUSIVE
> - 保证在查询过程中，表结构不会被修改
>
> **问题**：长事务会导致 MDL 锁等待，甚至形成等待链。

### 问题二：MDL 锁如何导致数据库变慢？

**面试官问**：「什么情况下 MDL 锁会导致整个库变慢？」

**深入回答**：

> **典型场景**：
> 1. 线程 A 开启事务执行查询，但不提交（持有 SHARED_READ）
> 2. 线程 B 执行 DDL（需要 EXCLUSIVE），被阻塞
> 3. 线程 C 执行 DML（需要 SHARED_WRITE），被线程 B 阻塞
> 4. 其他所有操作都被阻塞
>
> **解决方案**：
> - 避免长事务
> - 使用 `pt-online-schema-change` 工具
> - 监控 MDL 锁等待
> - 设置 `lock_wait_timeout`

### 问题三：全局锁和 MDL 锁有什么区别？

**面试官问**：「FLUSH TABLES WITH READ LOCK 和 MDL 锁有什么区别？」

**深入回答**：

> | 维度 | FTWRL | MDL 锁 |
> |---|---|---|
> | 作用域 | 整个数据库实例 | 单个表 |
> | 用途 | 全库备份 | 保护表结构 |
> | 获取方式 | 手动执行 | 自动获取 |
> | 锁类型 | 全局读锁 | 多种类型（共享/排他） |
> | 影响范围 | 所有表 | 目标表 |

## 总结

MySQL 的全局锁和表级锁：

- **全局锁**：锁住整个库，用于全库备份，影响很大
- **MDL 锁**：保护表结构，自动获取，可能导致锁等待
- **表锁**：手动锁定，用于特定场景

理解这些锁的特点和影响，是排查数据库性能问题的基础。
