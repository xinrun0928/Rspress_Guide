# 意向锁：IX 与 IS

「为什么表级锁和行级锁可以共存？」

这个问题的答案，在意向锁（Intention Lock）里。意向锁是 InnoDB 为了在同一表上同时支持表级锁和行级锁而设计的。

## 为什么要用意向锁？

### 问题背景

在没有意向锁的情况下，如果要对整个表加表级锁，需要检查表中每一行是否有行级锁。这需要遍历所有记录，性能极差。

```
┌─────────────────────────────────────────────────────────────────┐
│                    意向锁解决的问题                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  没有意向锁：                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  事务 A：对某行加行级锁                                  │   │
│  │  事务 B：想对整个表加表级锁                              │   │
│  │  → 事务 B 需要检查表中每一行是否有锁                    │   │
│  │  → 需要遍历整个表，性能极差！                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  有意向锁：                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  事务 A：对某行加行级锁，同时在表上加 IX                 │   │
│  │  事务 B：看到表上有 IX，直接知道有行级锁，无需逐行检查    │   │
│  │  → 只需检查表级锁，性能好！                             │   │
│  └─────────────────────────────────────────────────────────┘   │
```


## 意向锁的类型

### 两种意向锁

```

├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  意向共享锁（Intention Shared Lock, IS）                │   │
│  │                                                         │   │
│  │  - 表级锁                                               │   │
│  │  - 表示事务要获取共享锁（S Lock）                       │   │

│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  意向排他锁（Intention Exclusive Lock, IX）             │   │
│  │                                                         │   │
│  │  - 表级锁                                               │   │
│  │  - 表示事务要获取排他锁（X Lock）                       │   │
│  │  - 获取方式：自动加在表上，无需手动                     │   │
│  │  - 典型场景：INSERT/UPDATE/DELETE                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 意向锁的兼容性

```
┌─────────────────────────────────────────────────────────────────┐
│                    锁兼容性矩阵                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                  已持有的锁                              │   │
│  │           IS       IX       S       X                  │   │
│  │        ┌──────┬──────┬──────┬──────┐                  │   │
│  │   IS   │  ✓   │  ✓   │  ✓   │  ✗   │                  │   │
│  │        ├──────┼──────┼──────┼──────┤                  │   │
│  │   IX   │  ✓   │  ✓   │  ✗   │  ✗   │                  │   │
│  │  请   ├──────┼──────┼──────┼──────┤                  │   │
│  │  求   │   S  │  ✗   │  ✗   │  ✗   │                  │   │
│  │  的   ├──────┼──────┼──────┼──────┤                  │   │
│  │   锁   │   X  │  ✗   │  ✗   │  ✗   │                  │   │
│  │        └──────┴──────┴──────┴──────┘                  │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ✓ = 兼容（可以同时持有）                                       │
│  ✗ = 不兼容（互斥）                                             │
│                                                                 │
│  重要结论：                                                     │
│  - 意向锁之间互相兼容（IS + IX = ✓）                           │
│  - 意向锁与行级锁的关系：                                       │
│    - IS 与 S 兼容，与 X 不兼容                                  │
│    - IX 与 S、X 都不兼容                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 兼容性详解

| 锁类型 | IS | IX | S | X |
|---|---|---|---|---|
| IS | ✓ | ✓ | ✓ | ✗ |
| IX | ✓ | ✓ | ✗ | ✗ |
| S | ✓ | ✗ | ✓ | ✗ |
| X | ✗ | ✗ | ✗ | ✗ |

## 意向锁的实际场景

### 场景一：行级读锁 + 表级锁

```sql
-- 事务 A：行级共享锁
BEGIN;
SELECT * FROM users WHERE id = 1 LOCK IN SHARE MODE;
-- 自动对 users 表加 IS，对 id=1 加行级 S 锁

-- 事务 B：尝试对整个表加表级锁
LOCK TABLES users WRITE;
-- 被阻塞！因为 IX 与 X 不兼容
-- 事务 B 知道表中已有行级锁，无需逐行检查

COMMIT;
UNLOCK TABLES;
```

### 场景二：行级写锁 + 表级锁

```sql
-- 事务 A：行级排他锁
BEGIN;
UPDATE users SET name = '新张三' WHERE id = 1;
-- 自动对 users 表加 IX，对 id=1 加行级 X 锁

-- 事务 B：尝试加表级共享锁
LOCK TABLES users READ;
-- 被阻塞！因为 IX 与 S 不兼容

-- 事务 C：尝试加表级排他锁
LOCK TABLES users WRITE;
-- 被阻塞！因为 IX 与 X 不兼容

COMMIT;
UNLOCK TABLES;
```

### 场景三：两个行级锁

```sql
-- 事务 A：锁定 id=1
BEGIN;
SELECT * FROM users WHERE id = 1 FOR UPDATE;
-- 对 users 表加 IX，对 id=1 加行级 X 锁

-- 事务 B：锁定 id=2
BEGIN;
SELECT * FROM users WHERE id = 2 FOR UPDATE;
-- 对 users 表加 IX（可以！因为 IX 与 IX 兼容）
-- 对 id=2 加行级 X 锁

COMMIT;
COMMIT;
```

### 场景四：读写并发

```sql
-- 事务 A：行级读锁
BEGIN;
SELECT * FROM users WHERE id = 1 LOCK IN SHARE MODE;
-- 对 users 表加 IS

-- 事务 B：行级写锁
BEGIN;
SELECT * FROM users WHERE id = 2 FOR UPDATE;
-- 对 users 表加 IX（可以！因为 IS 与 IX 兼容）
-- 对 id=2 加行级 X 锁

COMMIT;
COMMIT;
```

### 意向锁与 MDL 锁的关系

```sql
-- 事务 A：行级锁
BEGIN;
SELECT * FROM users WHERE id = 1 FOR UPDATE;
-- 自动加：IX（InnoDB 引擎） + MDL_SHARED_WRITE（Server 层）

-- 事务 B：DDL 操作
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
-- 需要获取 MDL_EXCLUSIVE
-- 被 IX 阻塞！
```

## 面试实战

### 问题一：什么是意向锁？为什么要用意向锁？

**面试官问**：「什么是意向锁？为什么要用意向锁？」

**深入回答**：

> **意向锁是表级锁，分为 IS（意向共享锁）和 IX（意向排他锁）**。
>
> **为什么需要意向锁**：
> 1. 当事务要对表中某行加行级锁时，会自动在表上加对应的意向锁
> 2. 其他事务要加表级锁时，只需检查意向锁即可
> 3. 不需要逐行检查是否有行级锁，性能好
>
> **举例**：
> ```sql
> SELECT * FROM users WHERE id = 1 FOR UPDATE;
> -- 自动加 IX（表级），表示表中某行有排他锁
> ```

### 问题二：IS 和 IX 锁兼容吗？为什么？

**面试官问**：「IS 和 IX 锁兼容吗？」

**深入回答**：

> **IS 和 IX 锁是互相兼容的**。
>
> 原因：
> - 意向锁只是「表示」事务可能要加行级锁
> - 意向锁之间不冲突，因为它们不真正锁定任何资源
> - 真正的冲突发生在行级锁
>
> | 锁 | IS | IX | S | X |
> |---|---|---|---|---|
> | IS | ✓ | ✓ | ✓ | ✗ |
> | IX | ✓ | ✓ | ✗ | ✗ |
>
> 注意：IS/IX 与行级 X 不兼容，IX 与行级 S 不兼容。

### 问题三：意向锁和行级锁的关系是什么？

**面试官问**：「意向锁和行级锁是什么关系？」

**深入回答**：

> **意向锁是行级锁的「预告」**：
>
> 1. 当事务对某行加行级锁时，自动在表上加对应意向锁
> 2. 行级共享锁 → 表级 IS
> 3. 行级排他锁 → 表级 IX
>
> **作用**：
> - 表级锁检查时，看意向锁就知道有没有行级锁
> - 不需要逐行检查
>
> ```sql
> -- 行级锁会自动加意向锁
> SELECT * FROM users WHERE id = 1 LOCK IN SHARE MODE;
> -- 自动加：IS（表级）+ 行级 S 锁
>
> SELECT * FROM users WHERE id = 1 FOR UPDATE;
> -- 自动加：IX（表级）+ 行级 X 锁
> ```

## 总结

意向锁是 InnoDB 实现表级锁和行级锁共存的关键：

- **IS（意向共享锁）**：表示事务要获取行级共享锁
- **IX（意向排他锁）**：表示事务要获取行级排他锁
- **兼容性**：IS 与 IS/IX 兼容，IX 与 IS 兼容，但都与 S/X 不兼容
- **作用**：避免检查行级锁时逐行遍历，提高性能

理解意向锁的原理，是掌握 InnoDB 锁机制的重要一环。
