# 分片键选择与分片算法

「如何选择分片键？用什么算法？」

分片键和分片算法是分库分表的核心，决定了数据如何分散到各个分片。

## 分片算法

### 哈希分片

```
┌─────────────────────────────────────────────────────────────────┐
│                    哈希分片算法                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  算法：shard_id = hash(sharding_key) % 分片数             │   │
│  │                                                         │   │
│  │  示例：user_id % 10                                     │   │
│  │                                                         │   │
│  │  user_id = 1   → 1 % 10 = 1   → orders_1               │   │
│  │  user_id = 11  → 11 % 10 = 1   → orders_1               │   │
│  │  user_id = 21  → 21 % 10 = 1   → orders_1               │   │
│  │  user_id = 2   → 2 % 10 = 2   → orders_2               │   │
│  │  ...                                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  特点：                                                         │
│  - 数据分布均匀                                                │
│  - 无法按范围查询                                              │
│  - 适合随机查询场景                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 哈希分片实现

```java
public class HashShardingAlgorithm implements ShardingAlgorithm {

    @Override
    public String doSharding(
            Collection<String> availableTargetNames,
            ShardingValue shardingValue) {

        // 计算哈希值
        long value = hash(shardingValue.getValue());

        // 取模分片
        long shardIndex = value % availableTargetNames.size();

        // 返回目标分片
        for (String targetName : availableTargetNames) {
            if (targetName.endsWith(String.valueOf(shardIndex))) {
                return targetName;
            }
        }

        throw new ShardingException("No target found");
    }

    private long hash(Object value) {
        return value.hashCode();
    }
}
```

### 范围分片实现

```java
public class RangeShardingAlgorithm implements ShardingAlgorithm {

    private final long shardSize = 10_000_000;  // 每个分片 1000 万

    @Override
    public String doSharding(
            Collection<String> availableTargetNames,
            ShardingValue shardingValue) {

        long id = (Long) shardingValue.getValue();

        // 计算分片索引
        long shardIndex = id / shardSize;

        // 返回目标分片
        for (String targetName : availableTargetNames) {
            if (targetName.endsWith(String.valueOf(shardIndex))) {
                return targetName;
            }
        }

        throw new ShardingException("No target found");
    }
}
```

### 时间分片实现

```java
public class TimeShardingAlgorithm implements ShardingAlgorithm {

    @Override
    public String doSharding(
            Collection<String> availableTargetNames,
            ShardingValue shardingValue) {

        LocalDateTime dateTime = (LocalDateTime) shardingValue.getValue();

        // 生成时间分片名
        String shardName = String.format("logs_%d%02d",
                dateTime.getYear(),
                dateTime.getMonthValue());

        // 返回目标分片
        for (String targetName : availableTargetNames) {
            if (targetName.equals(shardName)) {
                return targetName;
            }
        }

        // 默认返回最新的分片
        return Collections.max(availableTargetNames);
    }
}
```

## 复合分片

### 什么是复合分片？

```
┌─────────────────────────────────────────────────────────────────┐
│                    复合分片示意                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  复合分片键：user_id + create_time                       │   │
│  │                                                         │   │
│  │  路由规则：                                             │   │
│  │  hash(user_id) % 4 + year(create_time) * 4              │   │
│  │                                                         │   │
│  │  示例：                                                 │   │
│  │  user_id = 1, 2024-01 → hash(1) % 4 = 1, year = 2024   │   │
│  │  分片 = 1 + 2024 * 4 = 8097                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  适用场景：既需要按用户查，又需要按时间范围查                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 复合分片实现

```java
public class ComplexShardingAlgorithm implements ShardingAlgorithm {

    @Override
    public Collection<String> doSharding(
            Collection<String> availableTargetNames,
            List<ShardingValue> shardingValues) {

        ShardingValue userIdValue = shardingValues.get(0);  // user_id
        ShardingValue timeValue = shardingValues.get(1);    // create_time

        long userId = (Long) userIdValue.getValue();
        LocalDateTime time = (LocalDateTime) timeValue.getValue();

        // 计算分片索引
        long shardIndex = (userId % 4) + (time.getYear() - 2020) * 4;

        // 返回目标分片
        return availableTargetNames.stream()
                .filter(name -> name.endsWith(String.valueOf(shardIndex)))
                .collect(Collectors.toList());
    }
}
```

## 面试实战

### 问题一：分片键怎么选择？

**面试官问**：「分片键如何选择？有什么原则？」

**深入回答**：

> **选择原则**：
>
> 1. **查询必需**：分片键必须是查询条件
> 2. **分布均匀**：避免数据倾斜
> 3. **业务相关**：符合业务查询习惯
>
> **常见选择**：
> - 用户表：user_id
> - 订单表：user_id（按用户查）或 order_id（按订单查）
> - 日志表：create_time

### 问题二：分片算法有哪些？各有什么优缺点？

**面试官问**：「哈希分片和范围分片各有什么优缺点？」

**深入回答**：

> **哈希分片**：
> - 优点：数据分布均匀
> - 缺点：无法按范围查询，跨分片查询困难
>
> **范围分片**：
> - 优点：范围查询高效
> - 缺点：可能数据倾斜
>
> **时间分片**：
> - 优点：时间查询高效，历史归档方便
> - 缺点：近期数据可能集中

### 问题三：分片数如何规划？

**面试官问**：「分片数怎么定？考虑哪些因素？」

**深入回答**：

> **规划步骤**：
>
> 1. 预估未来 3 年的数据量
> 2. 确定单分片容量（建议 1000 万行）
> 3. 计算分片数 = 数据量 / 单分片容量
> 4. 预留扩容空间（× 2）
> 5. 最好选择 2 的幂次
>
> **示例**：
> - 3 年预估 9 亿条
> - 单分片 1000 万
> - 分片数 = 90
> - 预留扩容 = 180
> - 选择 256 个分片

## 总结

分片键和分片算法是分库分表的核心：

- **分片键选择**：查询必需、分布均匀、业务相关
- **分片算法**：哈希（均匀）、范围（高效）、时间（归档）
- **复合分片**：满足多维度查询
- **分片数规划**：考虑数据量和扩容

合理选择分片键和分片算法，是分库分表成功的关键。
