# MySQL 基础总览

MySQL 基础是整个 MySQL 知识体系的根基。理解 MySQL 的整体架构、存储引擎特性、数据类型选择，才能更好地理解索引、事务、锁等高级特性。

本模块带你从宏观视角认识 MySQL，理解它的分层架构、核心存储引擎 InnoDB 的特性，以及一条 SQL 语句在 MySQL 中是如何执行的。

## MySQL 整体架构

MySQL 采用分层架构，这种设计让每一层专注于自己的职责，便于维护和扩展：

**连接层**：负责处理客户端连接、管理连接池、进行身份认证。连接池复用线程，减少频繁创建销毁线程的开销。

**服务层**：包含 SQL 接口、解析器、优化器、缓存等核心组件。解析器将 SQL 解析为语法树，优化器生成最优执行计划，缓存（Query Cache）在 8.0 之前用于加速查询。

**存储引擎层**：负责数据的物理存储和提取。MySQL 采用插件式存储引擎架构，InnoDB、MyISAM、Memory 等引擎各有特点。

## 存储引擎对比

MySQL 支持多种存储引擎，每种引擎都有其适用场景：

**InnoDB**：MySQL 5.5.5 之后的默认引擎。支持事务、行级锁、外键、崩溃自动恢复。采用聚簇索引组织数据，索引和数据在一起存储。适合大多数 OLTP 场景。

**MyISAM**：早期默认引擎，不支持事务、表级锁、无崩溃恢复。采用非聚簇索引，索引和数据分开存储。适合只读场景、快速插入场景（如日志表）。

**Memory**：数据存储在内存中，访问速度极快，但数据易丢失。适合临时表、缓存表。

## InnoDB 核心特性

InnoDB 是 MySQL 最核心的存储引擎，需要重点掌握：

**行级锁**：InnoDB 支持行级锁，相比表级锁，并发性能更好。行锁通过索引实现，因此查询条件必须命中索引才能加行锁。

**事务支持**：InnoDB 完全支持 ACID 事务特性，通过 Redo Log 实现持久性，通过 Undo Log 实现原子性和隔离性。

**MVCC**：Multi-Version Concurrency Control，多版本并发控制。InnoDB 通过 MVCC 实现读写并发，在 REPEATABLE READ 隔离级别下解决幻读问题。

**聚簇索引**：InnoDB 表数据按主键顺序存储，主键索引就是聚簇索引，叶子节点存储完整数据行。

**外键约束**：InnoDB 支持外键，提供参照完整性保护，但外键会增加开销，高并发场景需谨慎使用。

## SQL 执行过程

一条 SQL 在 MySQL 中的执行过程如下：

1. **连接管理**：连接器接收客户端连接，验证用户身份，从连接池获取线程
2. **查询缓存**（8.0 之前）：检查 Query Cache 是否有缓存结果（8.0 已移除）
3. **解析器**：词法分析、语法分析，生成抽象语法树（AST）
4. **预处理器**：检查表名、列名是否合法，进行语义检查
5. **优化器**：生成执行计划，选择最优索引、决定 JOIN 顺序等
6. **执行器**：根据执行计划，调用存储引擎接口获取数据
7. **存储引擎**：InnoDB 执行数据读取、写入操作
8. **结果返回**：将结果返回给客户端
