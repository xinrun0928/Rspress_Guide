# 计算机基础

计算机基础是 P5/P6 面试的必考模块，涵盖计算机网络、操作系统、网络安全和数据结构与算法四大方向。这四个方向各有侧重：网络重点考察 TCP/HTTP 协议细节，操作系统关注进程线程与内存管理，安全领域聚焦加密与常见攻击，算法则是面试的硬核考察。

## 模块导航

### 计算机网络

计算机网络是面试中的高频考点，尤其是 TCP 协议的三次握手四次挥手、HTTP 各版本的差异、HTTPS 的加密流程。面试官往往不满足于背诵答案，而是追问底层原理，比如为什么是三次握手而不是两次、四次挥手后为什么要等待 2MSL。

| 专题 | 核心知识点 | 面试权重 |
|------|-----------|----------|
| TCP 协议 | 三次握手/四次挥手、可靠传输、流量控制/拥塞控制 | 🔴 高频 |
| HTTP 协议 | 1.1 特性、2.0 多路复用、3.0 QUIC | 🔴 高频 |
| 网络安全 | HTTPS 加密过程、TLS 握手流程 | 🟡 中频 |
| 应用层协议 | DNS 解析、Session/Cookie、CDN | 🟡 中频 |

### 操作系统

操作系统考察的核心是**并发与内存**。进程线程的区别、线程同步机制、用户态与内核态切换是理解并发的基础；虚拟内存、页面置换算法、死锁则是内存管理的关键。IO 模型和 epoll 是后台开发面试的标配问题。

| 专题 | 核心知识点 | 面试权重 |
|------|-----------|----------|
| 进程与线程 | 进程线程区别、协程、线程同步 | 🔴 高频 |
| 内存管理 | 虚拟内存、分页分段、页面置换 | 🟡 中频 |
| 进程调度 | 调度算法、银行家算法、死锁 | 🟡 中频 |
| IO 模型 | select/poll/epoll、零拷贝 | 🔴 高频 |

### 网络安全

安全领域的面试题分为两类：**加密基础**和**攻防实战**。前者考察对称非对称加密、哈希、数字签名等原理；后者考察 SQL 注入、XSS、CSRF、DDOS 等常见攻击的原理与防御。

| 专题 | 核心知识点 | 面试权重 |
|------|-----------|----------|
| 加密基础 | 对称/非对称加密、常见加密算法 | 🔴 高频 |
| 协议安全 | TLS/SSL 握手、数字证书 | 🟡 中频 |
| 认证机制 | OAuth2、JWT 与 Session | 🟡 中频 |
| 攻防安全 | SQL 注入、XSS、CSRF、DDOS | 🔴 高频 |

### 数据结构与算法

算法是面试的核心模块，无论哪个方向都会考察。数组链表是基础，树结构要掌握遍历和平衡二叉树，排序算法要能分析复杂度，常见设计题如 LRU、布隆过滤器要能手写实现。

| 专题 | 核心知识点 | 面试权重 |
|------|-----------|----------|
| 基础结构 | 数组、链表、栈、队列 | 🔴 高频 |
| 树结构 | 二叉树遍历、BST、AVL、红黑树、B/B+树 | 🔴 高频 |
| 排序算法 | 快排、归并、堆排、各算法对比 | 🔴 高频 |
| 高频设计题 | LRU 缓存、布隆过滤器 | 🟡 中频 |

## 面试准备建议

计算机基础模块内容庞杂，建议分阶段突破：

**第一阶段（1-2 周）**：搞定高频考点：TCP 三次握手四次挥手、HTTP 各版本差异、进程线程区别、常见排序算法和二叉树遍历。这些是面试中出现频率最高的问题。

**第二阶段（1 周）**：深入原理：TCP 可靠传输机制、epoll 原理、虚拟内存与分页、IO 模型。这些问题面试官喜欢追问底层。

**第三阶段（持续）**：扩展知识面：CDN 架构、分布式系统安全、动态规划、滑动窗口等。这些是 P6+ 面试的加分项。

:::tip 面试技巧

在计算机基础面试中，**原理比结论更重要**。面试官往往不满足于"是什么"的答案，而是追问"为什么"。比如问到三次握手，不仅要能说出 SYN、SYN-ACK、ACK 三个步骤，更要理解为什么需要这三步、为什么是三次而不是两次、为什么第三次握手可以携带数据。

:::
