# 系统层安全总览

操作系统是所有软件的底层支撑，系统层安全出了问题，应用层的安全措施可能全部失效。

系统层安全关注的是操作系统层面的安全机制——内存保护、进程隔离、权限控制。从经典的缓冲区溢出，到现代的容器沙箱，每一个环节都关乎系统的整体安全性。

## 系统层安全威胁

### 攻击链

```
初始访问 → 代码执行 → 权限提升 → 持久化 → 数据窃取
   ↓           ↓           ↓           ↓
  漏洞利用   缓冲区溢出   本地提权    Rootkit
```

### 主要威胁类型

| 威胁类型 | 说明 | 示例 |
|---|---|---|
| 内存安全漏洞 | 缓冲区溢出、Use-After-Free | Stack Overflow |
| 权限提升 | 利用漏洞获得更高权限 | sudo 漏洞 |
| 后门 | 隐藏的入侵通道 | 键盘记录器 |
| 恶意软件 | 病毒、木马、勒索软件 | 加密货币挖矿 |
| 容器逃逸 | 突破容器边界访问宿主机 | Docker runc 漏洞 |

## 系统层安全机制

### 纵深防御

```
┌─────────────────────────────────────────┐
│           应用层安全                      │
│  输入验证、输出编码、参数化查询           │
├─────────────────────────────────────────┤
│           系统层安全                      │
│  ASLR、DEP、Stack Canary、CFI            │
├─────────────────────────────────────────┤
│           内核安全                        │
│  SELinux、AppArmor、seccomp              │
├─────────────────────────────────────────┤
│           容器隔离                        │
│  Namespace、Cgroup、Capability           │
└─────────────────────────────────────────┘
```

### 安全边界

| 边界 | 说明 |
|---|---|
| 用户态 vs 内核态 | CPU 特权级别 Ring 0/3 |
| 进程间边界 | 虚拟内存隔离、文件描述符隔离 |
| 容器边界 | Namespace 隔离、Cgroup 资源限制 |
| 网络边界 | iptables、firewalld |

## 本专题内容

| 文档 | 核心内容 |
|---|---|
| 缓冲区溢出 | 栈溢出原理、ASLR、DEP、Stack Canary |
| ROP 攻击 | 返回导向编程、堆风水、ROP 链构造 |
| 权限提升 | 本地提权、垂直越权、Sudo 漏洞 |
| 后门与 Rootkit | 隐藏技术、内核 Rootkit 检测 |
| 安全沙箱 | 容器隔离、Seccomp、AppArmor |
| SELinux | 强制访问控制、策略配置 |

## 安全原则

1. **最小权限原则**：只授予必要的权限
2. **纵深防御**：多层防护，单一措施不足
3. **默认安全**：系统默认配置应该是安全的
4. **持续监控**：实时检测异常行为

> 系统层安全是整个安全体系的基石。理解底层原理，才能更好地设计上层应用的安全措施。
