# 身份认证

你输入密码登录网站，系统怎么知道「这是你」？

身份认证（Authentication）的核心问题是：**如何证明你是你声称的那个人？**

## 认证的基本概念

### 什么是认证因素

认证因素分为三类：

| 因素类型 | 英文 | 示例 |
|---|---|---|
| 知识因素 | Something you know | 密码、PIN、安全问题 |
| 持有因素 | Something you have | 手机、U盾、硬件令牌 |
| 生物因素 | Something you are | 指纹、虹膜、人脸、声纹 |

单因素认证只使用一种因素，安全性较低。多因素认证（MFA）组合多种因素，安全性更高。

### 认证的三个步骤

```
1. 识别（Identification）
   - 用户声明身份：输入用户名/邮箱/手机号
   - 系统定位用户记录

2. 验证（Verification）
   - 用户提供认证因素
   - 系统验证因素是否正确

3. 授权（Authentication）
   - 验证通过，创建会话
   - 记录认证状态
```

## 密码认证

### 密码存储

:::danger
绝对不能明文存储密码！
:::

密码必须哈希后存储。正确的存储方式：

```python
import bcrypt
import hashlib
import secrets

# 正确的密码存储（使用 bcrypt）
def store_password(password):
    # bcrypt 会自动生成盐
    salt = bcrypt.gensalt(rounds=12)
    hashed = bcrypt.hashpw(password.encode(), salt)
    db.save(hashed)
    return salt  # bcrypt 的 hash 中已包含 salt

def verify_password(password, hashed):
    return bcrypt.checkpw(password.encode(), hashed)

# 错误的密码存储方式（绝对不要这样做！）
def bad_store_password(password):
    # 错误1：直接 MD5 存储
    hashed = hashlib.md5(password.encode()).hexdigest()
    db.save(hashed)

    # 错误2：简单加盐但用不安全算法
    salt = "fixed_salt"  # 固定盐
    hashed = hashlib.sha256(salt + password).hexdigest()
    db.save(hashed)
```

### 密码策略

| 策略 | 说明 | 评估 |
|---|---|---|
| 最小长度 | 至少 8 位，推荐 12 位以上 | 长度是最重要的 |
| 复杂性 | 大小写字母、数字、特殊字符 | 容易被预测模式绕过 |
| 禁止常见密码 | 字典攻击词库 | 必须的 |
| 定期更换 | 每 90 天更换 | 可能降低安全性（用户倾向简单密码） |
| 禁止重复使用 | 历史密码检查 | 推荐的 |

### 密码哈希算法对比

| 算法 | 速度 | 推荐用途 | 说明 |
|---|---|---|---|
| bcrypt | 慢 | 密码存储 | 自适应，可配置难度 |
| scrypt | 慢 | 密码存储 | 内存密集型 |
| Argon2 | 慢 | 密码存储 | 2015 年 PHC 冠军 |
| PBKDF2 | 中等 | 密码存储 | FIPS 标准 |
| SHA-256 | 快 | **不适合密码** | 太快，容易被暴力破解 |
| MD5 | 很快 | **禁止使用** | 已被攻破 |

```python
# Argon2 密码哈希示例
import argon2

ph = argon2.PasswordHasher(
    time_cost=3,      # 迭代次数
    memory_cost=65536,  # 内存成本（KB）
    parallelism=4       # 并行度
)

def store_password_argon2(password):
    hashed = ph.hash(password)
    db.save(hashed)

def verify_password_argon2(password, hashed):
    try:
        ph.verify(hashed, password)
        return True
    except argon2.exceptions.VerifyMismatchError:
        return False
```

## 多因素认证（MFA）

### 常见 MFA 方式

| 类型 | 方式 | 安全性 | 便利性 |
|---|---|---|---|
| SMS OTP | 手机短信验证码 | 中 | 高 |
| TOTP | Google Authenticator 等 APP | 高 | 高 |
| HOTP | 硬件令牌 | 高 | 中 |
| Push 通知 | 手机 APP 推送确认 | 高 | 高 |
| U2F/FIDO2 | 硬件安全钥匙（YubiKey） | 最高 | 中 |

### TOTP 实现原理

TOTP（Time-based One-Time Password）基于 HMAC：

```python
import pyotp
import qrcode

# 生成密钥（首次设置时）
secret = pyotp.random_base32()
print(f"密钥: {secret}")

# 生成 TOTP
totp = pyotp.TOTP(secret)
current_code = totp.now()
print(f"当前验证码: {current_code}")

# 验证（用户输入的验证码）
def verify_totp(user_secret, user_code):
    totp = pyotp.TOTP(user_secret)
    return totp.verify(user_code)

# 生成二维码（用于绑定 Authenticator APP）
uri = totp.provisioning_uri(name="user@example.com", issuer_name="MyApp")
qrcode.make(uri).save("totp_qr.png")
```

### U2F/FIDO2 安全钥匙

U2F 是目前最安全的 MFA 方式：

| 特性 | 说明 |
|---|---|
| 物理存在 | 硬件钥匙必须物理插入或靠近 |
| 公钥密码学 | 服务器只存储公钥，私钥永不离开硬件 |
| 钓鱼防护 | 绑定特定域名，无法被钓鱼网站使用 |
| 无共享密钥 | 每个网站使用不同的密钥对 |

```javascript
// WebAuthn/FIDO2 示例
async function registerSecurityKey() {
    const credential = await navigator.credentials.create({
        publicKey: {
            challenge: new Uint8Array([/* 服务器提供的随机数 */]),
            rp: {
                name: "My App",
                id: window.location.hostname
            },
            user: {
                id: new Uint8Array([/* 用户 ID */]),
                name: "user@example.com",
                displayName: "User Name"
            },
            pubKeyCredParams: [
                { alg: -7, type: "public-key" },   // ES256 (secp256r1)
                { alg: -257, type: "public-key" }   // RS256 (RSA)
            ],
            authenticatorSelection: {
                authenticatorAttachment: "cross-platform",
                userVerification: "preferred",
                residentKey: "preferred"
            },
            timeout: 60000,
            attestation: "indirect"
        }
    });

    // 将 credential.id 发送到服务器存储
    await saveCredentialToServer(credential);
}

async function authenticateWithSecurityKey() {
    const assertion = await navigator.credentials.get({
        publicKey: {
            challenge: new Uint8Array([/* 服务器提供的随机数 */]),
            allowCredentials: [{
                type: "public-key",
                id: new Uint8Array([/* 存储的 credential.id */])
            }],
            userVerification: "preferred",
            timeout: 60000
        }
    });

    // 验证签名
    await verifyAssertionOnServer(assertion);
}
```

## 生物认证

### 指纹识别

```python
# 指纹识别原理（简化）
class FingerprintMatcher:
    """指纹匹配器"""
    def extract_features(self, fingerprint_image):
        """从指纹图像提取特征点"""
        # 1. 图像增强
        enhanced = self.enhance_image(fingerprint_image)
        # 2. 方向场提取
        orientation = self.extract_orientation(enhanced)
        # 3. 特征点提取（细节点）
        minutiae = self.extract_minutiae(enhanced, orientation)
        return minutiae

    def match(self, probe_features, gallery_features):
        """匹配两个指纹特征"""
        # 计算特征点之间的欧氏距离
        matches = 0
        for p in probe_features:
            for g in gallery_features:
                if euclidean_distance(p, g) < THRESHOLD:
                    matches += 1
        # 计算匹配分数
        score = matches / max(len(probe_features), len(gallery_features))
        return score > MATCH_THRESHOLD
```

### 人脸识别

```python
# 人脸识别流程
class FaceRecognizer:
    def __init__(self):
        # 加载预训练模型
        self.model = load_face_recognition_model()

    def encode_face(self, image):
        """将人脸编码为特征向量"""
        face = self.detect_face(image)
        encoding = self.model.compute_face_descriptor(face)
        return encoding

    def verify(self, image, stored_encoding, threshold=0.6):
        """验证是否为同一人"""
        encoding = self.encode_face(image)
        distance = euclidean_distance(encoding, stored_encoding)
        return distance < threshold

    def identify(self, image, known_encodings):
        """识别是哪个人"""
        encoding = self.encode_face(image)
        distances = [euclidean_distance(encoding, e) for e in known_encodings]
        min_idx = argmin(distances)
        if distances[min_idx] < IDENTIFY_THRESHOLD:
            return known_users[min_idx]
        return None  # 未识别
```

## 认证流程设计

### 完整的登录流程

```python
class AuthenticationService:
    def login(self, username, password, factors=None):
        """完整的登录流程"""
        # 1. 速率限制检查
        if self.is_rate_limited(username):
            raise RateLimitExceeded()

        # 2. 密码验证
        user = db.find_user(username)
        if not user or not self.verify_password(password, user.password_hash):
            self.record_failed_attempt(username)
            raise InvalidCredentials()

        # 3. 检查是否需要 MFA
        if user.mfa_enabled:
            if not factors or not self.verify_mfa(user, factors):
                raise MfaRequired()

        # 4. 检查账户状态
        if user.is_locked:
            raise AccountLocked()
        if user.is_disabled:
            raise AccountDisabled()

        # 5. 登录成功
        self.clear_failed_attempts(username)
        session = self.create_session(user)
        self.record_login(user)

        return session

    def verify_mfa(self, user, factors):
        """验证多因素"""
        for factor in factors:
            if factor.type == 'totp':
                return self.verify_totp(user.mfa_secret, factor.code)
            elif factor.type == 'sms':
                return self.verify_sms_otp(user.phone, factor.code)
            elif factor.type == 'push':
                return self.verify_push_notification(user, factor.transaction_id)
        return False
```

### Session vs Token

| 特性 | Session | Token (JWT) |
|---|---|---|
| 存储位置 | 服务器内存/Redis | 客户端存储 |
| 状态管理 | 有状态 | 无状态 |
| 可扩展性 | 需要共享 Session | 无需服务端存储 |
| 失效机制 | 服务端直接失效 | 需配合黑名单或短有效期 |
| 体积 | 小 | 较大 |
| 适用场景 | 传统 Web 应用 | API 服务、移动端 |

```python
# Session 认证
class SessionAuth:
    def create_session(self, user):
        session_id = secrets.token_urlsafe(32)
        session_data = {
            'user_id': user.id,
            'created_at': time.time(),
            'expires_at': time.time() + SESSION_TIMEOUT
        }
        redis.setex(f"session:{session_id}", SESSION_TIMEOUT, json.dumps(session_data))
        return session_id

    def verify_session(self, session_id):
        data = redis.get(f"session:{session_id}")
        if not data:
            return None
        session = json.loads(data)
        if time.time() > session['expires_at']:
            redis.delete(f"session:{session_id}")
            return None
        return session

# Token 认证
class TokenAuth:
    def create_token(self, user):
        payload = {
            'sub': user.id,
            'iat': time.time(),
            'exp': time.time() + TOKEN_EXPIRY
        }
        token = jwt.encode(payload, SECRET_KEY, algorithm='HS256')
        return token

    def verify_token(self, token):
        try:
            payload = jwt.decode(token, SECRET_KEY, algorithms=['HS256'])
            return payload
        except jwt.ExpiredSignatureError:
            raise TokenExpired()
        except jwt.InvalidTokenError:
            raise InvalidToken()
```
