# 身份认证与授权

你登录银行 App 时，输入密码后系统怎么知道「这是你」而不是别人？手机收到的验证码又是如何证明「你拥有这台手机」？

身份认证（Authentication）和授权（Authorization）是信息安全的核心问题。简单说：

- **认证**：你是谁？（Who are you?）
- **授权**：你能做什么？（What can you do?）

## 知识图谱

本章节涵盖以下核心内容：

| 分类 | 内容 | 应用场景 |
|---|---|---|
| 身份认证 | 密码认证、多因素认证、生物认证 | 登录验证 |
| OAuth 2.0 | 授权框架、Token 机制 | 第三方登录 |
| OIDC | 身份层协议、ID Token | 单点登录 |
| SSO | 单点登录、CAS、SAML | 企业身份集成 |
| RBAC | 角色权限模型 | 权限管理 |
| LDAP | 目录服务集成 | 企业用户管理 |
| 零信任 | 永不信任、始终验证 | 现代安全架构 |

## 认证与授权的区别

很多初学者容易混淆认证和授权。来看一个生活中的例子：

```
认证：你有钥匙（证明你是钥匙的主人）
授权：钥匙能开哪把锁（主人允许你进入哪些房间）
```

在系统中：

```python
# 认证：验证用户身份
def authenticate(username, password):
    """验证用户名密码是否匹配"""
    user = db.find_user(username)
    if user and verify_password(password, user.password_hash):
        return user  # 认证成功
    return None

# 授权：检查用户权限
def authorize(user, resource, action):
    """检查用户是否有权限执行动作"""
    permissions = db.get_permissions(user.role)
    return (resource, action) in permissions
```

## 认证因素

认证因素分为三类：

| 因素类型 | 示例 | 特点 |
|---|---|---|
| 你知道的 | 密码、PIN、安全问题 | 记忆成本高，可能泄露 |
| 你拥有的 | 手机、U盾、硬件令牌 | 需要物理持有 |
| 你是什么 | 指纹、虹膜、人脸、声纹 | 生理特征，难以伪造 |

**多因素认证（MFA）**：组合多种因素，提供更强的安全性。

## 学习路径

建议按照以下顺序学习：

1. **先理解认证基础**：单因素认证、多因素认证
2. **再学习 OAuth 2.0**：授权框架的核心概念
3. **理解 OIDC**：在 OAuth 基础上添加身份层
4. **掌握 SSO**：企业级身份管理
5. **学习 RBAC**：权限管理模型
6. **理解零信任**：现代安全架构理念
