# 单点登录（SSO）

你在公司登录了邮箱系统，然后打开内部 Wiki、知识库、代码平台，都不需要再输入密码。这是怎么办到的？

**SSO（Single Sign-On）** 让你登录一次，访问所有系统。

## SSO 是什么？

SSO 的核心价值：
- **用户体验**：一次登录，到处访问
- **安全性**：减少密码暴露面
- **效率**：统一身份管理

```
没有 SSO：
用户 -> 系统A -> 输入密码 -> 系统A
用户 -> 系统B -> 输入密码 -> 系统B  # 又要输入！
用户 -> 系统C -> 输入密码 -> 系统C  # 烦死了！

有 SSO：
用户 -> 系统A -> 重定向到 SSO -> 输入密码 -> SSO -> 返回系统A
用户 -> 系统B -> 检查 SSO 会话 -> 直接访问（无需登录）
用户 -> 系统C -> 检查 SSO 会话 -> 直接访问（无需登录）
```

## CAS 协议

CAS（Central Authentication Service）是实现 SSO 的经典协议。

### CAS 流程

```
用户                                          浏览器                   CAS 服务器                    应用
  │                                               │                          │                       │
  │  1. 访问应用（未登录）                          │                          │                       │
  │ ─────────────────────────────────────────────>│                          │                       │
  │                                               │                          │                       │
  │  2. 重定向到 CAS 登录页面                       │                          │                       │
  │ <─────────────────────────────────────────────│                          │                       │
  │                                               │                          │                       │
  │  3. 输入用户名密码登录                          │                          │                       │
  │ ─────────────────────────────────────────────────────────────────────────>│                       │
  │                                               │                          │                       │
  │  4. 验证成功，颁发 TGC（Ticket Granting Cookie）│                          │                       │
  │ <────────────────────────────────────────────────────────────────────────│                       │
  │                                               │                          │                       │
  │  5. 重定向回应用，带 ST（Service Ticket）       │                          │                       │
  │ <─────────────────────────────────────────────│                          │                       │
  │                                               │                          │                       │
  │  6. 用 ST 验证用户身份                          │                          │                       │
  │ ──────────────────────────────────────────────────────────────────────────────────────────────────────>│
  │                                               │                          │                       │
  │  7. 返回用户信息，建立本地会话                   │                          │                       │
  │ <─────────────────────────────────────────────────────────────────────────────────────────────────────│
```

### CAS 实现

```python
# CAS 服务器端实现
from flask import Flask, request, redirect, session
import secrets
import hashlib

app = Flask(__name__)
app.secret_key = 'cas_secret_key'

# 已认证用户存储（实际用数据库）
authenticated_users = {}

@app.route('/login', methods=['GET', 'POST'])
def login():
    """CAS 登录端点"""
    service = request.args.get('service')  # 目标服务

    if request.method == 'POST':
        username = request.form['username']
        password = request.form['password']

        # 验证用户（实际用数据库）
        if verify_user(username, password):
            # 创建 TGC（Ticket Granting Cookie）
            tgc = secrets.token_urlsafe(32)
            session['tgc'] = tgc
            authenticated_users[tgc] = username

            # 如果有目标服务，重定向回去
            if service:
                # 生成 Service Ticket
                st = generate_service_ticket(service, username)
                return redirect(f"{service}?ticket={st}")
            return "登录成功"
        else:
            return "用户名或密码错误"

    # 显示登录表单
    return f'''
    <form method="POST">
        <input name="username" placeholder="用户名">
        <input name="password" type="password" placeholder="密码">
        <button type="submit">登录</button>
    </form>
    '''

@app.route('/validate')
def validate():
    """验证 Service Ticket"""
    service = request.args.get('service')
    ticket = request.args.get('ticket')

    if not ticket:
        return "no\n"

    # 验证 ST
    username = verify_service_ticket(service, ticket)
    if username:
        return f"yes\n{username}\n"
    return "no\n"

@app.route('/logout')
def logout():
    """登出"""
    tgc = session.get('tgc')
    if tgc and tgc in authenticated_users:
        del authenticated_users[tgc]
    session.clear()
    return "已登出"
```

```python
# CAS 客户端实现
from flask import Flask, request, session, redirect
import requests

app = Flask(__name__)
app.secret_key = 'app_secret_key'

CAS_SERVER = "https://cas.example.com"
SERVICE = "http://myapp.example.com/callback"

@app.before_request
def check_auth():
    """检查是否已登录"""
    # 已登录，放行
    if session.get('user'):
        return

    # 静态资源放行
    if request.path.startswith('/static'):
        return

    # 跳转到 CAS 登录
    if request.path != '/callback':
        return redirect(f"{CAS_SERVER}/login?service={SERVICE}")

@app.route('/callback')
def cas_callback():
    """处理 CAS 回调"""
    ticket = request.args.get('ticket')
    if not ticket:
        return redirect('/')

    # 向 CAS 服务器验证 Ticket
    response = requests.get(
        f"{CAS_SERVER}/validate",
        params={'service': SERVICE, 'ticket': ticket}
    )

    lines = response.text.split('\n')
    if lines[0] == 'yes':
        username = lines[1].strip()
        session['user'] = username
        return redirect('/')

    return "CAS 验证失败", 401
```

## SAML 2.0

SAML（Security Assertion Markup Language）是企业级 SSO 的另一重要标准，使用 XML 格式。

### SAML 工作流程

```
用户          应用(SP)                    身份提供商(IdP)
 │               │                              │
 │  1. 访问 SP    │                              │
 │ ─────────────>│                              │
 │               │                              │
 │  2. 生成 AuthnRequest (XML)                   │
 │               │───── SAML Request ──────────>│
 │               │                              │
 │  3. 用户在 IdP 登录                            │
 │               │                              │
 │  4. 生成 SAMLResponse (XML，包含断言)          │
 │               │<──── SAML Response ──────────│
 │               │                              │
 │  5. 验证断言，登录用户                         │
 │ <─────────────│                              │
```

### SAML Response 示例

```xml
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol">
    <saml:Issuer xmlns:saml="urn:oasis:names:tc:SAML:2.0:assertion">
        https://idp.example.com
    </saml:Issuer>
    <samlp:Status>
        <samlp:StatusCode Value="urn:oasis:names:tc:SAML:2.0:status:Success"/>
    </samlp:Status>
    <saml:Assertion xmlns:saml="urn:oasis:names:tc:SAML:2.0:assertion">
        <saml:Subject>
            <saml:NameID>user@example.com</saml:NameID>
        </saml:Subject>
        <saml:AttributeStatement>
            <saml:Attribute Name="email">
                <saml:AttributeValue>user@example.com</saml:AttributeValue>
            </saml:Attribute>
            <saml:Attribute Name="role">
                <saml:AttributeValue>admin</saml:AttributeValue>
            </saml:Attribute>
        </saml:AttributeStatement>
    </saml:Assertion>
</samlp:Response>
```

## OIDC 单点登录

现代应用更倾向于使用 OIDC 做 SSO：

```python
# OIDC SSO 实现
import requests
from flask import Flask, redirect, session

app = Flask(__name__)

# OIDC 配置
OIDC_CONFIG = {
    'issuer': 'https://auth.example.com',
    'client_id': 'your_client_id',
    'client_secret': 'your_client_secret',
    'redirect_uri': 'http://yourapp.com/callback'
}

@app.route('/login')
def login():
    config = requests.get(
        f"{OIDC_CONFIG['issuer']}/.well-known/openid-configuration"
    ).json()

    # 构建授权请求
    import secrets
    state = secrets.token_urlsafe(32)
    session['oauth_state'] = state

    auth_url = f"{config['authorization_endpoint']}?"
    auth_url += f"client_id={OIDC_CONFIG['client_id']}"
    auth_url += f"&response_type=code"
    auth_url += f"&scope=openid%20profile%20email"
    auth_url += f"&redirect_uri={OIDC_CONFIG['redirect_uri']}"
    auth_url += f"&state={state}"

    return redirect(auth_url)

@app.route('/callback')
def callback():
    # 验证 state
    # 交换 token
    # 获取用户信息
    # 建立本地会话
    pass
```

## Session 共享

多应用 SSO 需要共享会话：

### Redis Session 共享

```python
# 使用 Redis 共享 Session
from flask import Flask, session
from flask_session import Session
import redis

app = Flask(__name__)
app.config['SESSION_TYPE'] = 'redis'
app.config['SESSION_PERMANENT'] = False
app.config['SESSION_USE_SIGNER'] = True
app.config['SESSION_KEY_PREFIX'] = 'sso:'
app.config['SESSION_REDIS'] = redis.Redis(host='redis', port=6379)

Session(app)

# 应用 A 和应用 B 共享同一个 Redis
# 登录一次，所有应用都能读到 Session
```

```python
        }, SECRET_KEY, algorithm='HS256')

        # 设置 Cookie，domain 为根域名
        response = make_response(redirect('/dashboard'))
        response.set_cookie(
            'sso_token',
            token,
            domain='.example.com',  # 根域名，所有子域共享
            httponly=True,
            secure=True,
            samesite='Lax'
        )
        return response

    return "登录失败", 401

# 各子应用验证 JWT
def verify_token(token):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=['HS256'])
        return payload
    except jwt.ExpiredSignatureError:
        return None
```

## SSO 安全考虑

| 风险 | 防御措施 |
|---|---|
| 会话劫持 | HTTPS + HttpOnly + Secure Cookie |
| CSRF | CSRF Token |
| 会话持久化 | 合理设置 Session 过期时间 |
| 单点故障 | SSO 服务器高可用部署 |
| 账户关联攻击 | 多因素认证 |
