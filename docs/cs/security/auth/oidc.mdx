# OIDC 与身份认证

OAuth 2.0 是一个**授权**协议，不是**认证**协议。授权解决的是「你能访问什么」，但没有解决「你是谁」。

**OIDC（OpenID Connect）** 在 OAuth 2.0 基础上添加了身份认证层，让第三方应用不仅能获取用户授权，还能知道用户是谁。

## OAuth 2.0 vs OIDC

| 方面 | OAuth 2.0 | OIDC |
|---|---|---|
| 目的 | 授权（你能做什么） | 认证 + 授权（你是谁，你能做什么） |
| 解决的问题 | 访问权限委托 | 用户身份验证 |
| 返回内容 | Access Token | Access Token + **ID Token** |
| 用途 | API 访问 | 用户登录 |

### OAuth 2.0 授权流程

```
OAuth 2.0 返回：
{
  "access_token": "...",
  "token_type": "Bearer",
  "expires_in": 3600
}
用户是谁？不知道！
```

### OIDC 认证流程

```
OIDC 返回：
{
  "access_token": "...",    // 访问资源
  "id_token": "...",        // 身份信息
  "token_type": "Bearer",
  "expires_in": 3600
}
用户是谁？一清二楚！
```

## ID Token

ID Token 是 OIDC 的核心，它是一个 **JWT（JSON Web Token）**，包含用户身份信息。

### JWT Payload 声明

```json
{
  "iss": "https://auth.example.com",        // 签发者
  "sub": "1234567890",                       // 主题（用户唯一标识）
  "aud": "client_id",                        // 受众（你的应用）
  "exp": 1516239022,                         // 过期时间
  "iat": 1516239022,                         // 签发时间
  "nonce": "n-0S6_WzA2Ms",                   // 防重放随机数
  "name": "John Doe",                         // 姓名
  "email": "john@example.com",               // 邮箱
  "email_verified": true,                    // 邮箱是否验证
  "picture": "https://example.com/john.jpg", // 头像
  "locale": "en-US"                          // 语言偏好
}
```

### JWT 验证

```python
import jwt
import requests

# OIDC 提供商配置
ISSUER = "https://auth.example.com"
CLIENT_ID = "your_client_id"
JWKS_URI = f"{ISSUER}/.well-known/jwks.json"

# 获取公钥
def get_public_keys():
    jwks = requests.get(JWKS_URI).json()
    keys = {}
    for key in jwks['keys']:
        # 构建 RSA 公钥
        public_key = jwt.algorithms.RSAAlgorithm.from_jwk(json.dumps(key))
        keys[key['kid']] = public_key
    return keys

def verify_id_token(id_token):
    # 1. 解码 Header，获取 kid
    header = jwt.get_unverified_header(id_token)
    kid = header['kid']
    alg = header['alg']

    # 2. 获取对应公钥
    public_key = get_public_keys().get(kid)
    if not public_key:
        raise InvalidKeyError("Unknown key")

    # 3. 验证签名
    payload = jwt.decode(
        id_token,
        public_key,
        algorithms=[alg],
        audience=CLIENT_ID,
        issuer=ISSUER
    )

    # 4. 验证通过，返回 payload
    return payload

# 使用示例
id_token = request.args.get('id_token')
user_info = verify_id_token(id_token)
print(f"用户邮箱: {user_info['email']}")
print(f"用户名称: {user_info['name']}")
```

## OIDC 流程

### 完整认证流程

```
用户                                          应用                          授权服务器
  │                                               │                                  │
  │  1. 点击「登录」                               │                                  │
  │ ─────────────────────────────────────────────>│                                  │
  │                                               │                                  │
  │  2. 重定向到授权页面                           │                                  │
  │ <─────────────────────────────────────────────│                                  │
  │                                               │                                  │
  │  3. 用户输入凭证并授权                         │                                  │
  │ ────────────────────────────────────────────────────────────────────────────────>│
  │                                               │                                  │
  │  4. 重定向回应用，带授权码                     │                                  │
  │ <───────────────────────────────────────────────────────────────────────────────│
  │                                               │                                  │
  │  5. 用授权码换 Token（含 ID Token）            │                                  │
  │ ─────────────────────────────────────────────>│                                  │
  │                                               │  6. 请求 Token                    │
  │                                               │ ──────────────────────────────────>│
  │                                               │                                  │
  │                                               │  7. 返回 access_token + id_token  │
  │                                               │ <─────────────────────────────────│
  │                                               │                                  │
  │  8. 返回登录成功                               │                                  │
  │ <─────────────────────────────────────────────│                                  │
```

### 代码实现

```python
# OIDC 登录实现
from flask import Flask, request, redirect, session
import requests
import jwt

app = Flask(__name__)
app.secret_key = 'your_secret_key'

OIDC_CONFIG = {
    'issuer': 'https://auth.example.com',
    'client_id': 'your_client_id',
    'client_secret': 'your_client_secret',
    'redirect_uri': 'http://localhost:5000/callback',
    'scope': 'openid profile email'
}

@app.route('/login')
def login():
    # 1. 发现 OIDC 配置
    config = requests.get(
        f"{OIDC_CONFIG['issuer']}/.well-known/openid-configuration"
    ).json()

    # 2. 构建授权 URL
    import secrets
    state = secrets.token_urlsafe(32)
    nonce = secrets.token_urlsafe(32)
    session['oidc_state'] = state
    session['oidc_nonce'] = nonce

    auth_url = f"{config['authorization_endpoint']}?"
    auth_url += f"client_id={OIDC_CONFIG['client_id']}"
    auth_url += f"&response_type=code"
    auth_url += f"&scope=openid%20profile%20email"
    auth_url += f"&redirect_uri={OIDC_CONFIG['redirect_uri']}"
    auth_url += f"&state={state}"
    auth_url += f"&nonce={nonce}"

    return redirect(auth_url)

@app.route('/callback')
def callback():
    # 验证 state
    if request.args.get('state') != session.get('oidc_state'):
        abort(400, "Invalid state")

    # 获取授权码
    code = request.args.get('code')

    # 发现 Token 端点
    config = requests.get(
        f"{OIDC_CONFIG['issuer']}/.well-known/openid-configuration"
    ).json()

    # 交换 Token
    token_response = requests.post(
        config['token_endpoint'],
        data={
            'grant_type': 'authorization_code',
            'code': code,
            'redirect_uri': OIDC_CONFIG['redirect_uri'],
            'client_id': OIDC_CONFIG['client_id'],
            'client_secret': OIDC_CONFIG['client_secret']
        },
        headers={'Content-Type': 'application/x-www-form-urlencoded'}
    )

    tokens = token_response.json()
    id_token = tokens['id_token']
    access_token = tokens['access_token']

    # 验证并解析 ID Token
    user_info = verify_id_token(id_token)

    # 验证 nonce
    if user_info['nonce'] != session.get('oidc_nonce'):
        abort(400, "Invalid nonce")

    # 登录成功
    session['user'] = user_info
    return redirect('/dashboard')
```

## UserInfo 端点

除了 ID Token，还可以通过 UserInfo 端点获取更详细的用户信息：

```python
def get_user_info(access_token):
    """通过 UserInfo 端点获取用户信息"""
    config = requests.get(
        f"{OIDC_CONFIG['issuer']}/.well-known/openid-configuration"
    ).json()

    response = requests.get(
        config['userinfo_endpoint'],
        headers={'Authorization': f'Bearer {access_token}'}
    )

    return response.json()
    # 返回：
    # {
    #   "sub": "1234567890",
    #   "name": "John Doe",
    #   "given_name": "John",
    #   "family_name": "Doe",
    #   "email": "john@example.com",
    #   "email_verified": true,
    #   "picture": "https://..."
    # }
```

## 与 SAML 的对比

| 方面 | OIDC | SAML |
|---|---|---|
| 协议基础 | OAuth 2.0 | XML |
| Token 格式 | JWT | XML |
| 使用场景 | 移动端、Modern Web | 企业 SSO |
| 复杂度 | 简单 | 复杂 |
| 生态系统 | 广泛 | 企业级 |
