# OAuth 2.0

你想让第三方应用读取你的 GitHub 仓库列表，但不想告诉它你的密码，怎么办？

**OAuth 2.0** 就是解决这个问题的：让你在不泄露密码的情况下，授权第三方应用访问你的资源。

## 问题背景

在 OAuth 出现之前，如果你想让第三方应用访问你的数据，唯一的办法是把用户名密码给它。这带来了严重的问题：

```
传统方式的问题：
1. 用户必须信任第三方应用不会滥用密码
2. 第三方应用获得了完整的账户访问权限
3. 无法限制第三方应用的访问权限
4. 无法撤回授权（只能改密码）
5. 密码在网络中多次传输，增加泄露风险
```

## 核心概念

OAuth 2.0 引入了一套角色和概念：

| 角色 | 说明 |
|---|---|
| Resource Owner（资源所有者） | 拥有账户的用户，你 |
| Client（客户端） | 想访问资源的第三方应用 |
| Authorization Server（授权服务器） | 验证用户并颁发 Token 的服务器 |
| Resource Server（资源服务器） | 存储用户资源的服务器 |
| User Agent | 用户的浏览器或 APP |

### OAuth 2.0 的四个角色

```
┌─────────┐     授权     ┌──────────────────┐     Token      ┌─────────┐
│  用户   │ <──────────> │  授权服务器      │ <─────────────> │  资源   │
│(你)     │              │ (如 GitHub)      │                │  服务器 │
└─────────┘              └──────────────────┘                └─────────┘
        │                         │
        │    授权许可             │
        └────────────────────────>┘
                                      │
        ┌─────────────────────────────┘
        │
        │  授权码 / Token
        ▼
┌─────────┐
│ 第三方   │ (你的 APP)
│ 客户端   │
└─────────┘
```

## 四种授权模式

### 1. 授权码模式（Authorization Code）

**最安全、最完整的模式**，适合有后端服务器的应用。

```
用户                                          第三方应用                          授权服务器
  │                                               │                                  │
  │  1. 点击「用 GitHub 登录」                     │                                  │
  │ ─────────────────────────────────────────────>│                                  │
  │                                               │                                  │
  │  2. 重定向到授权页面                           │                                  │
  │ <─────────────────────────────────────────────│                                  │
  │                                               │                                  │
  │  3. 用户点击「授权」                           │                                  │
  │ ────────────────────────────────────────────────────────────────────────────────>│
  │                                               │                                  │
  │  4. 重定向回第三方应用，带上授权码               │                                  │
  │ <───────────────────────────────────────────────────────────────────────────────│
  │                                               │                                  │
  │  5. 后端用授权码换 Token                       │                                  │
  │ ──────────────────────────────────────────────│                                  │
  │                                               │  6. 换 Token                      │
  │                                               │ ──────────────────────────────────>│
  │                                               │                                  │
  │                                               │  7. 返回 Access Token             │
  │                                               │ <─────────────────────────────────│
```

代码示例：

```python
# Flask 应用示例
from flask import Flask, request, redirect
import requests
import secrets

app = Flask(__name__)

# OAuth 配置
GITHUB_CLIENT_ID = "your_client_id"
GITHUB_CLIENT_SECRET = "your_client_secret"
GITHUB_AUTHORIZE_URL = "https://github.com/login/oauth/authorize"
GITHUB_TOKEN_URL = "https://github.com/login/oauth/access_token"
REDIRECT_URI = "http://localhost:5000/callback"

@app.route('/login/github')
def login_github():
    """发起 OAuth 授权"""
    state = secrets.token_urlsafe(32)
    # 将 state 存入 session 防止 CSRF
    session['oauth_state'] = state

    params = {
        'client_id': GITHUB_CLIENT_ID,
        'redirect_uri': REDIRECT_URI,
        'scope': 'read:user user:email',
        'state': state
    }

    url = f"{GITHUB_AUTHORIZE_URL}?client_id={params['client_id']}&redirect_uri={params['redirect_uri']}&scope={params['scope']}&state={params['state']}"
    return redirect(url)

@app.route('/callback')
def callback():
    """处理授权回调"""
    # 1. 验证 state 防止 CSRF
    state = request.args.get('state')
    if state != session.get('oauth_state'):
        abort(400, "Invalid state")

    # 2. 用授权码换 Token
    code = request.args.get('code')

    response = requests.post(
        GITHUB_TOKEN_URL,
        data={
            'client_id': GITHUB_CLIENT_ID,
            'client_secret': GITHUB_CLIENT_SECRET,
            'code': code,
            'redirect_uri': REDIRECT_URI
        },
        headers={'Accept': 'application/json'}
    )

    token_data = response.json()
    access_token = token_data['access_token']

    # 3. 用 Token 获取用户信息
    user_response = requests.get(
        'https://api.github.com/user',
        headers={'Authorization': f'token {access_token}'}
    )
    user = user_response.json()

    # 4. 创建本地会话
    session['user'] = user
    return redirect('/dashboard')
```

### 2. 隐式授权模式（Implicit）

**已废弃，不推荐使用**。适合纯前端 SPA 应用（已被 PKCE 取代）。

### 3. 密码凭证模式（Password Credentials）

**仅适用于可信的第一方应用**。

```python
# 密码凭证模式示例
@app.route('/login', methods=['POST'])
def login():
    username = request.form['username']
    password = request.form['password']

    # 直接用用户名密码换 Token
    # 仅适用于自己开发的应用，绝不能用于第三方应用！
    response = requests.post(
        TOKEN_URL,
        data={
            'grant_type': 'password',
            'username': username,
            'password': password,
            'client_id': CLIENT_ID
        }
    )

    return response.json()
```

### 4. 客户端凭证模式（Client Credentials）

**用于机器对机器认证**。

```python
# 客户端凭证模式示例
@app.route('/api/data')
def get_data():
    # 服务之间直接用 Client ID 和 Secret 换 Token
    response = requests.post(
        TOKEN_URL,
        data={
            'grant_type': 'client_credentials',
            'client_id': SERVICE_CLIENT_ID,
            'client_secret': SERVICE_CLIENT_SECRET,
            'scope': 'read:data'
        }
    )

    token_data = response.json()
    access_token = token_data['access_token']

    # 用 Token 访问资源
    data = fetch_protected_data(access_token)
    return data
```

## Token 类型

OAuth 2.0 使用两种 Token：

| Token | 说明 | 用途 |
|---|---|---|
| Access Token | 访问令牌 | 调用 API |
| Refresh Token | 刷新令牌 | 获取新的 Access Token |

### Access Token

```json
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "read:user user:email"
}
```

### Token 刷新机制

```python
# 刷新 Access Token
def refresh_access_token(refresh_token):
    response = requests.post(
        TOKEN_URL,
        data={
            'grant_type': 'refresh_token',
            'refresh_token': refresh_token,
            'client_id': CLIENT_ID,
            'client_secret': CLIENT_SECRET
        }
    )

    return response.json()
    # 返回新的 access_token 和 refresh_token

# 使用 Token 调用 API
def call_api(access_token):
    response = requests.get(
        'https://api.github.com/user',
        headers={
            'Authorization': f'Bearer {access_token}',
            'Accept': 'application/json'
        }
    )
    return response.json()
```

## 安全考虑

### CSRF 攻击

攻击者诱导用户访问恶意页面，该页面携带用户已有的授权发起请求。

**防御措施**：使用 state 参数

```python
@app.route('/login/github')
def login_github():
    state = secrets.token_urlsafe(32)
    session['oauth_state'] = state  # 存储用于验证

    # 在授权 URL 中包含 state
    return redirect(f"{AUTH_URL}?state={state}...")

@app.route('/callback')
def callback():
    # 验证 state
    if request.args.get('state') != session.get('oauth_state'):
        abort(400, "CSRF detected!")
```

### Token 泄露

**防御措施**：

| 措施 | 说明 |
|---|---|
| HTTPS | 必须使用 HTTPS 传输 |
| 短有效期 | Access Token 有效期不要太长 |
| Token 刷新 | 使用 Refresh Token 定期刷新 |
| Token 撤销 | 提供 Token 撤销接口 |

### 授权范围（Scope）

明确指定应用需要的权限：

```python
# 请求最小必要权限
params = {
    'scope': 'read:user user:email'  # 只请求读取用户信息和邮箱
}
# 而非
# 'scope': 'repo admin:org'  # 不要请求过多权限
```

## PKCE 扩展

PKCE（Proof Key for Code Exchange）增强了授权码模式的安全性，防止 Token 被拦截。

```python
# PKCE 实现
import hashlib
import base64
import secrets

def generate_pkce_params():
    # 1. 生成 code_verifier（随机字符串，43-128 字符）
    code_verifier = secrets.token_urlsafe(64)

    # 2. 生成 code_challenge
    #    S256 方法：SHA256(code_verifier) 再 base64url 编码
    digest = hashlib.sha256(code_verifier.encode()).digest()
    code_challenge = base64.urlsafe_b64encode(digest).decode().rstrip('=')

    return {
        'code_verifier': code_verifier,
        'code_challenge': code_challenge,
        'code_challenge_method': 'S256'
    }

def authorize_with_pkce():
    pkce = generate_pkce_params()
    session['code_verifier'] = pkce['code_verifier']

    params = {
        'client_id': CLIENT_ID,
        'redirect_uri': REDIRECT_URI,
        'scope': 'read:user',
        'code_challenge': pkce['code_challenge'],
        'code_challenge_method': 'S256'
    }
    return params

def exchange_code_with_pkce(code, code_verifier):
    response = requests.post(
        TOKEN_URL,
        data={
            'grant_type': 'authorization_code',
            'code': code,
            'redirect_uri': REDIRECT_URI,
            'client_id': CLIENT_ID,
            'code_verifier': code_verifier  # 必须提供
        }
    )
    return response.json()
```
