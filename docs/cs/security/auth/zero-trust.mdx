# 零信任安全架构

传统安全模式：你在公司内网，所有系统都信任你。
零信任模式：**永不信任，始终验证。**

2019 年 Google 宣布完成BeyondCorp 转型，标志着大企业零信任实践的开始。2021 年，美国政府发布 M-22-09 备忘录，要求联邦机构在 2024 财年前实现零信任架构。

## 传统安全 vs 零信任

### 传统边界安全

```
传统安全模型：

    Internet
        │
    ┌───┴───┐
    │防火墙 │
    └───┬───┘
        │
    ┌───┴───┐
    │ 内网   │  <- 所有内网系统默认信任
    │       │
    │ ┌───┐ │
    │ │DB │ │  <- 直接访问
    │ └───┘ │
    │ ┌───┐ │
    │ │App│ │  <- 直接访问
    │ └───┘ │
    └───────┘

问题：
1. 一旦突破边界，攻击者可以在内网自由漫游
2. 内网系统缺乏强身份验证
3. 横向移动风险巨大
```

### 零信任架构

```
零信任安全模型：

    Internet
        │
    ┌───┴───┐
    │ 接入层  │
    │(身份验证)│
    └───┬───┘
        │
    ┌───┴───┐
    │ 策略   │  <- 每次访问都验证
    │ 引擎   │
    └───┬───┘
        │
    ┌───┴───┐
    │ 资源   │
    │       │
    │ ┌───┐ │  <- 只有通过验证才能访问
    │ │DB │ │
    │ └───┘ │
    │ ┌───┐ │
    │ │App│ │
    │ └───┘ │
    └───────┘

原则：
1. 永不信任，无论来源
2. 始终验证，无论位置
3. 最小权限，无论角色
```

## 零信任核心原则

| 原则 | 说明 |
|---|---|
| 始终验证 | 每次请求都需要身份验证和授权 |
| 最小权限 | 只授予完成任务所需的最小权限 |
| 微分段 | 将网络划分为细粒度的安全段 |
| 假设已被入侵 | 设计和运营假设系统已被攻陷 |
| 持续监控 | 持续评估风险和信任 |
| 不依赖网络 | 访问决策与网络位置无关 |

## 关键技术

### 身份感知代理（Identity-Aware Proxy）

```python
# 简化版身份感知代理
class IdentityAwareProxy:
    def __init__(self, policy_engine):
        self.policy_engine = policy_engine

    async def handle_request(self, request):
        # 1. 提取身份信息
        user = request.user
        device = await self.get_device_info(request)

        # 2. 评估风险
        risk_score = await self.assess_risk(user, device, request)

        # 3. 访问决策
        decision = await self.policy_engine.evaluate(
            user=user,
            device=device,
            resource=request.resource,
            action=request.action,
            risk_score=risk_score
        )

        if decision.allow:
            # 验证通过，转发请求
            return await self.forward_request(request)
        elif decision.conditional:
            # 需要额外验证（MFA）
            return await self.require_mfa(request)
        else:
            # 拒绝访问
            return Response(status=403, body="Access Denied")

    async def assess_risk(self, user, device, request):
        """评估访问风险"""
        risk_score = 0

        # 设备风险
        if not device.managed:
            risk_score += 30
        if not device.encrypted:
            risk_score += 20
        if not device.os_updated:
            risk_score += 10

        # 用户风险
        if user.new_device:
            risk_score += 20
        if user.unusual_location:
            risk_score += 30
        if user.time_since_mfa > 24 * 3600:
            risk_score += 15

        # 请求风险
        if request.sensitive_data:
            risk_score += 25
        if request.bulk_operation:
            risk_score += 20

        return risk_score
```

### 软件定义边界（SDP）

SDP（Software Defined Perimeter）是实现零信任的一种架构：

```python
# SDP 客户端和服务端
class SDPServer:
    """SDP 服务器"""
    def __init__(self):
        self.registered_services = {}

    async def handle_hip(self, client_hip):
        """处理主机身份证明（Host Identity Proof）"""
        # 1. 验证设备状态
        device_compliant = await self.check_device_compliance(client_hip)

        # 2. 检查用户权限
        user_authorized = await self.check_user_authorization(client_hip)

        # 3. 颁发临时凭证
        if device_compliant and user_authorized:
            return self.issue_spf(client_hip)

        return None

class SDPController:
    """SDP 控制器"""
    def __init__(self):
        self.sp_info = {}  # 服务提供者信息

    async def register_service(self, service_id, allowed_users):
        """注册受保护服务"""
        self.sp_info[service_id] = {
            'allowed_users': allowed_users,
            'last_updated': time.time()
        }

    async def authorize_connection(self, client_id, target_service):
        """授权连接请求"""
        allowed = self.sp_info.get(target_service, {}).get('allowed_users', [])
        return client_id in allowed
```

## 实现步骤

### 第一阶段：身份安全

```python
# 实施强身份认证
class StrongAuthentication:
    def require_mfa(self, user, resource):
        """敏感资源必须 MFA"""
        if resource.sensitivity >= HIGH:
            return self.enforce_mfa(user)

    def continuous_auth(self, user, behavior):
        """持续身份验证"""
        risk = self.calculate_risk(user, behavior)
        if risk > THRESHOLD:
            self.require_reauthentication(user)
```

### 第二阶段：设备信任

```python
# 设备信任评估
class DeviceTrust:
    async def evaluate_device(self, device_id):
        """评估设备信任分数"""
        checks = [
            await self.check_managed(device_id),      # 是否 MDM 管理
            await self.check_encrypted(device_id),    # 是否加密
            await self.check_os_updated(device_id),   # 系统是否更新
            await self.check_compliant(device_id),    # 是否合规
        ]

        trust_score = sum(c.weight * c.result for c in checks)
        return DeviceTrustLevel.from_score(trust_score)
```

## 与传统模型对比

| 方面 | 传统边界模型 | 零信任模型 |
|---|---|---|
| 信任基础 | 网络位置（内网/外网） | 身份和设备状态 |
| 访问控制 | 粗粒度（网络层） | 细粒度（应用/数据层） |
| 横向移动 | 相对容易 | 受限 |
| 边界定义 | 清晰的网络边界 | 无边界 |
| 适用场景 | 固定办公环境 | 远程办公、云环境 |
| 复杂度 | 相对简单 | 复杂 |

## 实施挑战

| 挑战 | 解决方案 |
|---|---|
| 遗留系统改造 | 使用代理/网关隐藏旧系统 |
| 性能影响 | 优化策略引擎，缓存决策 |
| 用户体验 | 无摩擦的 Step-up 认证 |
| 多云环境 | 统一的身份和策略层 |
| 监控复杂度 | 集中日志和 SIEM 集成 |
