# 应用层安全

你的网站被注入了恶意脚本，用户信息被窃取。你检查了服务器，没有被入侵。问题出在哪？

**应用层安全**关注的是应用程序本身的安全问题，而非网络或系统层面的问题。根据 OWASP 的统计，应用层漏洞是大多数安全事件的原因。

## 知识图谱

本章节涵盖以下核心内容：

| 分类 | 漏洞类型 | 风险等级 |
|---|---|---|
| 注入 | XSS、SQL 注入、命令注入、LDAP 注入 | 高 |
| 认证 | 暴力破解、会话劫持、密码重置漏洞 | 高 |
| 敏感数据 | 数据泄露、弱加密、不安全存储 | 高 |
| XML 外部实体 | XXE 注入 | 高 |
| 访问控制 | 水平越权、垂直越权 | 高 |
| 安全配置 | 默认配置、不安全配置 | 中 |
| XSS | 反射型、存储型、DOM 型 | 高 |
| 访问控制 | IDOR、权限绕过 | 高 |

## 学习路径

建议按照以下顺序学习：

1. **理解注入原理**：XSS、SQL 注入、命令注入都是「注入」的不同形式
2. **掌握防护机制**：输入验证、输出编码、参数化查询
3. **理解认证安全**：会话管理、密码存储、MFA
4. **学习访问控制**：RBAC、水平越权、垂直越权
5. **理解安全配置**：最小权限、安全默认配置

## 面试要点

应用层安全是面试中的高频考点，以下问题值得重点准备：

- XSS 的三种类型及区别？
- CSRF 和 XSS 的区别？如何防御？
- SQL 注入的原理和防御？
- 什么是同源策略？
- JWT 的安全考虑？
- 如何设计安全的密码重置功能？

> 应用层安全是 Web 安全的核心。理解漏洞原理才能有效防御。
