# Web 应用防火墙（WAF）

你有想过在攻击者到达应用之前，就把他们挡在门外吗？

WAF（Web Application Firewall，Web 应用防火墙）是专门保护 Web 应用的安全设备。与传统防火墙不同，WAF 能够理解 HTTP 协议和应用层协议，能够检测和阻断 SQL 注入、XSS、CSRF 等应用层攻击。

## WAF 工作原理

### 工作层次

```
请求流程：
客户端 → WAF → 负载均衡 → 应用服务器

WAF 检查请求：
1. 请求头（User-Agent、Cookie、Referer）
2. 请求参数（GET、POST、URL 参数）
3. 请求体（JSON、XML、表单数据）
4. 请求频率和模式

如果检测到攻击 → 返回 403 或重定向到错误页
正常请求 → 转发到后端服务器
```

### 部署方式

| 部署方式 | 说明 | 优点 | 缺点 |
|---|---|---|---|
| 硬件 WAF | 独立网络设备 | 性能高、不影响应用 | 成本高、扩展性差 |
| 软件 WAF | 安装在服务器上 | 灵活、成本低 | 占用服务器资源 |
| 云 WAF | 云服务商提供（Cloudflare、AWS WAF） | 零部署、弹性扩展 | 依赖网络、有延迟 |
| 混合部署 | CDN + WAF | 静态内容加速 + 安全 | 配置复杂 |

## WAF 规则类型

### 基于签名的规则

```yaml
# ModSecurity 规则示例
# SQL 注入检测
SecRule REQUEST_URI|REQUEST_BODY|ARGS "@rx (\')|(\")|(\-\-)|(\;)|(\()|(\))|(\bunion\b)|(\bselect\b)" \
    "id:1001,\
    phase:2,\
    deny,\
    status:403,\
    msg:'SQL Injection Attempt',\
    logdata:'Matched Data: %{MATCHED_VAR} found within %{MATCHED_VAR_NAME}',\
    severity:CRITICAL"
```

### 基于行为的规则

```yaml
# 速率限制规则
SecRule REQUEST_HEADERS:User-Agent "@streq empty" \
    "id:1002,\
    phase:1,\
    deny,\
    status:403,\
    msg:'Missing User-Agent',\
    severity:WARNING"

# 限制单个 IP 的请求频率
SecRule REQUEST_BODY "@streq /login" \
    "id:1003,\
    phase:2,\
    rateLimitBy:IP,\
    nolog,\
    pass,\
    auditlogParts:+E"
```

### OWASP Core Rule Set (CRS)

OWASP CRS 是最广泛使用的开源 WAF 规则集：

```bash
# 安装 OWASP CRS for ModSecurity
# 1. 下载规则
wget https://github.com/coreruleset/coreruleset/archive/refs/tags/v4.0.0.tar.gz

# 2. 配置
# crs-setup.conf - 基础配置
IncludeOptional /etc/modsecurity/crs/crs-setup.conf

# 3. 启用规则
IncludeOptional /etc/modsecurity/crs/rules/*.conf
```

## WAF 绕过技术

### 绕过并非总是必要

理解 WAF 绕过不是为了攻击，而是为了**理解 WAF 的局限性**，设计更健壮的防御方案。

### 常见绕过方式

#### 1. 大小写混合

```sql
-- WAF 可能只检测小写
UNION SELECT password FROM users--

-- 绕过：大写混合
UniOn SeLeCt password FrOm users--
```

#### 2. 编码绕过

```bash
# URL 编码
%27 OR %271%27=%271

# 双重 URL 编码
%2527 OR %25271%2527=%25271

# Unicode 编码
\u0027 OR \u00271\u0027=\u00271

# HTML 实体编码
&#39; OR &#39;1&#39;=&#39;1
```

#### 3. 注释混淆

```sql
-- 注释绕过
UNION/**/SELECT/**/password/**/FROM/**/users

-- 内联注释（MySQL）
UNION/*!SELECT*//*!password*//*!FROM*//*!users*/

-- 注释可以插入关键字中间，绕过正则匹配
```

#### 4. 字符串拼接

```sql
-- MySQL 字符串拼接
UNION SELECT pass/**/FROM users
'pass' = 'pas' 's'
CHAR(112, 97, 115, 115)

-- 使用变量
SET @sql = 'UNION SELECT ' + 'password FROM users';
EXEC(@sql);
```

#### 5. 等价替换

```sql
-- 空格过滤 → 使用注释或括号
UNION/**/SELECT/**/password/**/FROM users
(username)BETWEEN(1)AND(1)

-- OR/AND 过滤 → 使用 ||、&&
'1'||'1'='1'
'1'&&'1'='1'

-- 等号过滤 → 使用 LIKE、BETWEEN、IN
WHERE id IN (1)
WHERE id BETWEEN 1 AND 1
WHERE username LIKE 'admin'
```

#### 6. HTTP 方法绕过

```bash
# POST 请求伪装成 GET
X-HTTP-Method-Override: GET
X-Original-URL: /admin/delete?id=1

# 或者反过来
GET /admin/delete?id=1 HTTP/1.1
Host: target.com
X-Original-URL: /normal/page
```

### WAF 绕过检测工具

```bash
# 使用 WAFW00F 检测 WAF 类型
```

## WAF 最佳实践

### 1. 规则调优

```nginx
# Nginx + ModSecurity 调优
# 排除误报
SecRule REQUEST_URI "@beginsWith /static/" \
    "id:9001,\
    phase:1,\
    pass,\
    nolog,\
    ctl:ruleRemoveById=920350"

# 排除特定 IP
SecRule REMOTE_ADDR "@ipMatch 192.168.1.0/24" \
    "id:9002,\
    phase:1,\
    pass,\
    nolog,\
    ctl:ruleEngine=Off"

# 降低误报级别的规则
SecRule ARGS "@rx <script>" \
    "id:9003,\
    phase:2,\
    pass,\
    log,\
    msg:'Potential XSS',\
    severity:WARNING,\
    chain"
SecRule MATCHED_VAR "@contains javascript:"
    "setvar:MATCHED_VAR='[BLOCKED]'"
```

### 2. 日志分析

```python
# 分析 WAF 日志，识别误报和漏报
import re
from collections import Counter

def analyze_waf_logs(log_file):
    """分析 WAF 日志"""
    patterns = []
    blocked_ips = Counter()
    false_positive_rules = []

    with open(log_file) as f:
        for line in f:
            # 提取规则 ID 和请求信息
            rule_match = re.search(r'id:(\d+)', line)
            ip_match = re.search(r'(\d+\.\d+\.\d+\.\d+)', line)

            if rule_match:
                patterns.append(rule_match.group(1))

            if ip_match:
                blocked_ips[ip_match.group(1)] += 1

    # 输出分析结果
    print("Top 10 触发的规则：")
    for rule_id, count in Counter(patterns).most_common(10):
        print(f"  Rule {rule_id}: {count} 次")

    print("\n被阻止最多的 IP：")
    for ip, count in blocked_ips.most_common(10):
        print(f"  {ip}: {count} 次")
```

### 3. 与应用安全配合

WAF 不是银弹，必须配合其他安全措施：

```
多层防护体系：

第一层：WAF（边界防护）
   ↓ 检测常见攻击
第二层：应用层安全（输入验证、输出编码）
   ↓ 防御未被 WAF 阻断的攻击
第三层：数据库层（参数化查询）
   ↓ 防御 SQL 注入
第四层：身份认证与会话管理
   ↓ 防御会话劫持
```

## WAF 的局限性

| 局限性 | 说明 |
|---|---|
| 无法防御业务逻辑漏洞 | 越权、暴力破解等业务漏洞 |
| 存在绕过可能 | 高级攻击者可能绕过 WAF |
| 误报问题 | 可能阻断正常请求 |
| 性能开销 | 增加请求延迟 |
| 配置复杂 | 需要持续调优 |
