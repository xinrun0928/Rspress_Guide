# CSRF 跨站请求伪造

你在银行网站登录后，没有退出，直接访问了一个恶意网站。该网站自动发送了一个转账请求，利用你的登录状态把钱转走了。

**CSRF（Cross-Site Request Forgery）** 跨站请求伪造，利用用户已登录的身份，诱导用户浏览器发送恶意请求。

## CSRF 原理

CSRF 利用的是浏览器发送请求时会**自动带上同源 Cookie**的特性：

```
攻击条件：
1. 用户已登录目标网站
2. 用户访问恶意网站
3. 目标网站没有 CSRF 防护
```

```
攻击流程：

用户浏览器                              恶意网站                          银行网站
    │                                        │                                │
    │  1. 用户已登录银行                       │                                │
    │ ──────────────────────────────────────────────────────────────────────>│
    │                                        │                                │
    │  2. 用户访问恶意网站                     │                                │
    │ ──────────────────────────────────────>│                                │
    │                                        │                                │
    │  3. 恶意页面自动发送请求                  │                                │
    │     <form action="bank.com/transfer">   │                                │
    │       <input name="to" value="hacker"> │                                │
    │       <input name="amount" value="1000">│                                │
    │     </form>                            │                                │
    │ ──────────────────────────────────────────────────────────────────────>│
    │                                        │   4. 请求带上 Cookie！           │
    │                                        │                                │
    │                                        │                                │ 转账成功！
```

## 攻击类型

### 1. GET 型 CSRF

```html
<!-- 恶意网站 -->
<img src="https://bank.com/transfer?to=hacker&amount=1000" style="display:none">
<!-- 用户访问时，浏览器自动发起 GET 请求 -->
```

```python
# Flask 错误示例
@app.route('/transfer')
def transfer():
    # 直接使用 GET 参数，没有验证
    to_account = request.args.get('to')
    amount = request.args.get('amount')

    # 执行转账
    bank.transfer(to_account, amount)
    return '转账成功'
```

### 2. POST 型 CSRF

```html
<!-- 恶意网站 -->
<form action="https://bank.com/transfer" method="POST" id="csrf-form">
    <input type="hidden" name="to" value="hacker">
    <input type="hidden" name="amount" value="1000">
</form>
<script>
    document.getElementById('csrf-form').submit();
</script>
```

```python
# Flask 错误示例
@app.route('/transfer', methods=['POST'])
def transfer():
    # 没有 CSRF Token 验证
    to_account = request.form['to']
    amount = request.form['amount']

    bank.transfer(to_account, amount)
    return '转账成功'
```

### 3. JSON CSRF

```html
<!-- 恶意网站，绕过 Content-Type 检查 -->
<script>
fetch('https://api.example.com/profile', {
    method: 'POST',
    credentials: 'include',  // 带上 Cookie
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        email: 'attacker@evil.com',
        password: 'hacked'
    })
});
</script>
```

## CSRF vs XSS

| 方面 | CSRF | XSS |
|---|---|---|
| 原理 | 利用用户身份发送请求 | 在目标网站执行脚本 |
| 前提 | 用户已登录 | 用户访问恶意页面 |
| 攻击目标 | 目标网站的业务逻辑 | 目标网站的用户 |
| Cookie | 自动发送 | 无法直接获取（HttpOnly） |
| 防护 | Token/验证请求来源 | 输入验证/输出编码 |

```
CSRF 是「借刀杀人」：攻击者借助受害者的身份做事
XSS 是「偷天换日」：攻击者直接冒充受害者
```

## 防御策略

### 1. CSRF Token

最常用的防御方式：

```python
# Flask CSRF 保护实现
import secrets
from functools import wraps

csrf_tokens = {}

def generate_csrf_token():
    """生成 CSRF Token"""
    token = secrets.token_urlsafe(32)
    csrf_tokens[token] = True  # 存储到服务端
    return token

def verify_csrf_token(token):
    """验证 CSRF Token"""
    if token and token in csrf_tokens:
        del csrf_tokens[token]  # 使用后删除，防止重放
        return True
    return False

def csrf_protect(f):
    """CSRF 保护装饰器"""
    @wraps(f)
    def decorated(*args, **kwargs):
        if request.method in ['POST', 'PUT', 'DELETE']:
            token = request.form.get('csrf_token') or request.headers.get('X-CSRF-Token')
            if not verify_csrf_token(token):
                abort(403, 'CSRF token invalid')
        return f(*args, **kwargs)
    return decorated

# 正确使用
@app.route('/transfer', methods=['POST'])
@csrf_protect
def transfer():
    to_account = request.form['to']
    amount = request.form['amount']
    bank.transfer(to_account, amount)
    return '转账成功'

@app.route('/transfer-form')
def transfer_form():
    # 生成带 Token 的表单
    return f'''
    <form action="/transfer" method="POST">
        目标账户: <input name="to"><br>
        金额: <input name="amount"><br>
        <input type="hidden" name="csrf_token" value="{generate_csrf_token()}">
        <button type="submit">转账</button>
    </form>
    '''
```

### 2. Double Submit Cookie

不需要服务端存储 Token：

```python
# Double Submit 实现
import secrets
from flask import request, make_response

@app.before_request
def csrf_protect():
    if request.method in ['POST', 'PUT', 'DELETE']:
        cookie_token = request.cookies.get('csrf_token')
        header_token = request.headers.get('X-CSRF-Token')
        form_token = request.form.get('csrf_token')

        # 检查两个 Token 是否匹配
        if not cookie_token or not header_token:
            abort(403)

        if cookie_token != header_token or cookie_token != form_token:
            abort(403)

@app.route('/transfer-form')
def transfer_form():
    token = secrets.token_urlsafe(32)
    response = make_response(render_template('form.html', csrf_token=token))
    # 设置 Cookie
    response.set_cookie('csrf_token', token, samesite='Strict')
    return response
```

```html
<!-- form.html -->
<form action="/transfer" method="POST">
    <input type="hidden" name="csrf_token" value="{{ csrf_token }}">
    <!-- 页面加载时从 Cookie 读取设置到 Header -->
</form>
<script>
    document.addEventListener('DOMContentLoaded', function() {
        var token = document.cookie.match(/csrf_token=([^;]+)/)?.[1];
        if (token) {
            fetch('/transfer', {
                method: 'POST',
                headers: { 'X-CSRF-Token': token },
                body: 'to=hacker&amount=1000&csrf_token=' + token
            });
        }
    });
</script>
```

### 3. SameSite Cookie

```python
# 设置 SameSite Cookie
response.set_cookie(
    'session_id',
    session_id,
    samesite='Strict'  # 完全禁止跨站发送
    # 或者
    # samesite='Lax'   # GET 请求允许，其他不行
)
```

| 值 | 说明 | 限制程度 |
|---|---|---|
| `Strict` | 完全禁止跨站发送 | 最严格 |
| `Lax` | GET 请求允许，POST 限制 | 中等 |
| `None` | 不限制（需配合 Secure） | 无 |

```python
# 跨域请求携带 Cookie
response.set_cookie(
    'session_id',
    session_id,
    samesite='None',  # 允许跨站
    secure=True       # 必须 HTTPS
)
```

### 4. 验证请求来源

```python
from urllib.parse import urlparse

def verify_origin(request):
    """验证请求来源"""
    origin = request.headers.get('Origin')
    referer = request.headers.get('Referer')

    allowed_origins = ['https://example.com', 'https://app.example.com']

    if origin and origin in allowed_origins:
        return True

    if referer:
        parsed = urlparse(referer)
        if parsed.origin in allowed_origins:
            return True

    return False

@app.route('/transfer', methods=['POST'])
def transfer():
    if not verify_origin(request):
        abort(403, 'Invalid origin')

    # 处理请求
    pass
```

### 5. 密码确认

对于敏感操作，要求用户重新输入密码：

```python
@app.route('/transfer', methods=['POST'])
def transfer():
    to_account = request.form['to']
    amount = request.form['amount']
    password = request.form['password']

    # 验证密码
    if not verify_password(current_user, password):
        return '密码错误', 400

    bank.transfer(to_account, amount)
    return '转账成功'
```

## 完整防护示例

```python
from flask import Flask, request, abort, session
from flask_wtf.csrf import CSRFProtect

app = Flask(__name__)
app.secret_key = secrets.token_hex(32)

# 使用 Flask-WTF 的 CSRF 保护
csrf = CSRFProtect(app)

# 自动为所有 POST 请求验证 CSRF Token
@app.route('/transfer', methods=['POST'])
@login_required
def transfer():
    to_account = request.form['to']
    amount = int(request.form['amount'])

    # 业务逻辑
    bank.transfer(session['user_id'], to_account, amount)
    return '转账成功'

# 模板中使用
@app.route('/transfer-form')
@login_required
def transfer_form():
    return render_template('transfer.html')
```

```html
<!-- transfer.html，使用 Flask-WTF -->
<form method="POST">
    {{ form.hidden_tag() }}  <!-- 自动生成 CSRF Token -->
    {{ form.to() }}
    {{ form.amount() }}
    <button type="submit">转账</button>
</form>
```
