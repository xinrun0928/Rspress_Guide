# 消息摘要

你的文件下载完成了吗？你怎么确认文件和原始文件是一样的？

MD5、SHA-1、SHA-256 这些名字你一定见过。它们不是加密算法，而是**摘要算法**——一种不可逆的「数字指纹」。

## 什么是消息摘要？

消息摘要（Message Digest）是一种单向函数，将任意长度的输入映射为固定长度的输出：

```
输入：任意长度的数据
输出：固定长度的「指纹」
```

核心特性：

| 特性 | 说明 |
|---|---|
| 确定性 | 相同输入必产生相同输出 |
| 单向性 | 无法从输出反推输入 |
| 抗碰撞性 | 难以找到两个不同输入产生相同输出 |
| 雪崩效应 | 输入微小变化导致输出完全不同 |

```python
# 雪崩效应示例
import hashlib

msg1 = "Hello, World!"
msg2 = "Hello, World"  # 少了感叹号

digest1 = hashlib.sha256(msg1.encode()).hexdigest()
digest2 = hashlib.sha256(msg2.encode()).hexdigest()

print(f"原文1: {msg1}")
print(f"原文2: {msg2}")
print(f"摘要1: {digest1}")
print(f"摘要2: {digest2}")
print(f"相同? {digest1 == digest2}")
```

```
原文1: Hello, World!
原文2: Hello, World
摘要1: dffd6021bb2bd5b0af676290809ec3a53191dd81c7f70a4b28688a362182986f
摘要2: 315f5bdb76d078c43b8ac0064e081a5fab9f2f53e94c4f7e6e4d5c5e1e0b6d8
相同? False
```

仅仅少了一个感叹号，256 位的摘要就有超过一半的位发生了变化。

## MD5

MD5（Message-Digest Algorithm 5）由 Ronald Rivest 于 1991 年设计，输出 128 位（16 字节）摘要。

### 算法结构

```
输入消息 -> 填充 -> 分组(512位) -> 迭代压缩 -> 128位摘要
```

MD5 的核心是四轮非线性函数 F、G、H、I，每轮 16 步操作：

```python
# MD5 核心函数（Python 示意）
import struct

def F(x, y, z):
    return (x & y) | (~x & z)

def G(x, y, z):
    return (x & z) | (y & ~z)

def H(x, y, z):
    return x ^ y ^ z

def I(x, y, z):
    return y ^ (x | ~z)

def md5_round(F, X, A, B, C, D, k, s, i):
    """MD5 单轮操作"""
    A = (A + F(B, C, D) + X[k] + i) & 0xffffffff
    A = (A << s | A >> (32 - s)) & 0xffffffff
    A = (A + B) & 0xffffffff
    return A, B, C, D, A
```

### MD5 的碰撞攻击

:::danger
MD5 已废弃！2004 年被王小云教授团队攻破，可以在数秒内找到碰撞。
:::

MD5 的碰撞示例：

```
原始文件 A 和恶意文件 B 具有相同的 MD5 值
可以构造相同的 HTML 前缀，但不同可执行代码
```

实际攻击场景：

| 攻击类型 | 说明 | 危害 |
|---|---|---|
| 长度扩展攻击 | 可在已知摘要的情况下伪造摘要 | 伪造消息 |
| 碰撞攻击 | 找到两个不同消息具有相同摘要 | 伪造签名 |
| 原像攻击 | 从摘要反推原始消息 | **理论上不可行，但有改进攻击** |

## SHA 系列

SHA（Secure Hash Algorithm）家族是美国国家安全局（NSA）设计的标准算法。

### SHA-1

输出 160 位，已被废弃。

```
SHA-1 碰撞攻击：
2017 年，Google 和 CWI Amsterdam 合作
展示了第一次 SHA-1 碰撞（SHAttered 攻击）
```

### SHA-2 家族

SHA-2 是 SHA 家族的继任者，包括多种变体：

| 算法 | 输出长度 | 状态 |
|---|---|---|
| SHA-224 | 224 位 | 安全 |
| SHA-256 | 256 位 | **最常用** |
| SHA-384 | 384 位 | 安全 |
| SHA-512 | 512 位 | 安全 |
| SHA-512/224 | 224 位 | 安全 |
| SHA-512/256 | 256 位 | 安全 |

SHA-256 的核心结构：

```python
# SHA-256 核心操作
import math

# 初始哈希值（前 8 个素数的平方根的小数部分前 32 位）
H0 = [
    0x6a09e667, 0xbb67ae85, 0x3c6ef372, 0xa54ff53a,
    0x510e527f, 0x9b05688c, 0x1f83d9ab, 0x5be0cd19
]

# 常量（前 32 个素数的立方根的小数部分前 32 位）
K = [
    0x428a2f98, 0x71374491, 0xb5c0fbcf, 0xe9b5dba5, ...
]

def sigma0(x):
    return (x >> 2 | x << 30) ^ (x >> 13 | x << 19) ^ (x >> 22 | x << 10)

def sigma1(x):
    return (x >> 6 | x << 26) ^ (x >> 11 | x << 21) ^ (x >> 25 | x << 7)

def sha256_compress(W):
    """SHA-256 单块压缩"""
    A, B, C, D, E, F, G, H = H0

    for i in range(64):
        S1 = sigma1(E) + F(E, F, G) + H + K[i] + W[i]
        S2 = sigma0(A) + F(A, B, C)
        H = G
        G = F
        F = E
        E = D + S1
        D = C
        C = B
        B = A
        A = S1 + S2

    return A, B, C, D, E, F, G, H
```

### SHA-3 家族

SHA-3 于 2015 年成为标准，采用全新的 **Keccak** 算法设计：

| 算法 | 输出长度 | 内部宽度 |
|---|---|---|
| SHA3-224 | 224 位 | 1600 位 |
| SHA3-256 | 256 位 | 1600 位 |
| SHA3-384 | 384 位 | 1600 位 |
| SHA3-512 | 512 位 | 1600 位 |

SHA-3 的核心是 **海绵结构**（Sponge）：

```
输入 -> 吸收(Absorbing) -> 挤压(Squeezing) -> 输出
        内部状态 1600 位
```

## HMAC

单纯的摘要算法无法抵抗长度扩展攻击。HMAC（Hash-based Message Authentication Code）通过引入密钥来提供认证：

```python
import hmac
import hashlib

def hmac_sha256(key, message):
    """HMAC-SHA256 实现"""
    block_size = 64  # SHA-256 的分组长度

    # 密钥填充到 block_size
    if len(key) > block_size:
        key = hashlib.sha256(key).digest()
    if len(key) < block_size:
        key = key + b'\x00' * (block_size - len(key))

    # 内部和外部填充
    o_key_pad = bytes(k ^ 0x5c for k in key)
    i_key_pad = bytes(k ^ 0x36 for k in key)

    # 嵌套哈希
    inner = hashlib.sha256(i_key_pad + message).digest()
    outer = hashlib.sha256(o_key_pad + inner).digest()

    return outer

# 使用示例
key = b"secret_key"
message = b"Hello, HMAC!"

mac = hmac_sha256(key, message)
print(f"HMAC: {mac.hex()}")
```

## SM3 国密算法

SM3 是中国国家密码管理局于 2010 年发布的商用密码杂凑算法标准，输出 256 位，与 SHA-256 相当。

```python
# SM3 核心结构（简化）
def SM3_compress(iv, block):
    """SM3 压缩函数"""
    # 消息扩展：将 512 位扩展为 132 个 32 位字
    W = expand_message(block)

    # 压缩函数：64 轮迭代
    A, B, C, D, E, F, G, H = iv

    for i in range(64):
        if i < 16:
            SS1 = rotl((rotl(A, 12) + E + T(i)) & 0xffffffff, 7)
        else:
            SS1 = rotl((rotl(A, 12) + E + T(i)) & 0xffffffff, 7)
        SS2 = SS1 ^ rotl(A, 12)
        TT1 = (F(A, B, C) + D + SS2 + W[i] ^ W[i+4]) & 0xffffffff
        TT2 = (F(E, F, G) + H + SS1 + W[i]) & 0xffffffff
        D = C
        C = rotl(B, 9)
        B = A
        A = TT1
        H = G
        G = rotl(F, 19)
        F = E
        E = P0(TT2)

    return A, B, C, D, E, F, G, H
```

## 实际应用

### 文件完整性校验

```bash
# Linux/Mac 计算 SHA-256
shasum -a 256 filename.zip

# Windows PowerShell
Get-FileHash filename.zip -Algorithm SHA256

# 验证下载
# 下载页面通常会提供 SHA-256 值
echo "下载文件的 SHA-256" | shasum -a 256 -c
```

### 密码存储

:::warning
绝对不能直接存储明文密码！
:::

```python
# 错误的密码存储方式
def bad_store_password(password):
    hashed = hashlib.sha256(password.encode()).hexdigest()
    db.save(hashed)  # 危险！相同密码产生相同哈希

# 正确的密码存储方式
import bcrypt

def store_password(password):
    # bcrypt 会自动生成随机盐
    hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt())
    db.save(hashed)

def verify_password(password, hashed):
    return bcrypt.checkpw(password.encode(), hashed)
```

### HMAC 验证请求

```java
// Java HMAC 验证示例
import javax.crypto.Mac;
import javax.crypto.spec.SecretKeySpec;
import java.util.Base64;

public class HmacExample {

    public static String calculateHmac(String data, String key) throws Exception {
        Mac mac = Mac.getInstance("HmacSHA256");
        SecretKeySpec secretKey = new SecretKeySpec(key.getBytes(), "HmacSHA256");
        mac.init(secretKey);
        byte[] hmacBytes = mac.doFinal(data.getBytes());
        return Base64.getEncoder().encodeToString(hmacBytes);
    }

    // 验证请求
    public static boolean verifyRequest(String data, String key, String providedHmac) {
        try {
            String expectedHmac = calculateHmac(data, key);
            return constantTimeEquals(expectedHmac, providedHmac);
        } catch (Exception e) {
            return false;
        }
    }

    // 使用恒定时间比较防止时序攻击
    private static boolean constantTimeEquals(String a, String b) {
        if (a.length() != b.length()) return false;
        int result = 0;
        for (int i = 0; i < a.length(); i++) {
            result |= a.charAt(i) ^ b.charAt(i);
        }
        return result == 0;
    }
}
```
