# 国密算法

2014 年，中国银监会要求银行系统使用国密算法。2020 年，《密码法》正式实施。**国密算法**不再是可选项，而是很多场景的必选项。

什么是国密算法？简单说，就是**中国国家密码管理局发布的中国商用密码标准**。和国际算法对应：

| 国际算法 | 国密算法 | 类型 |
|---|---|---|
| DES | SM1 | 对称加密（芯片内置，不公开） |
| AES | SM4 | 对称加密 |
| RSA | SM2 | 非对称加密 |
| SHA-256 | SM3 | 消息摘要 |
| - | SM7 | 对称加密（轻量级） |
| - | SM9 | 标识密码算法 |

## SM1 对称加密

SM1 是对称分组密码算法，密钥长度 128 位，分组长度 128 位。

**关键特点**：算法本身不对外公开，以芯片形式提供。

```
SM1 特点：
- 密钥和算法都在芯片内部
- 对外只提供芯片接口
- 安全性依赖于硬件保护
```

这意味着：
- 无法在纯软件中实现 SM1
- 芯片成本较高
- 但也意味着算法本身不会被逆向分析

## SM4 对称加密

SM4 是公开的对称分组密码算法，于 2006 年发布，2012 年成为国家标准 GB/T 32907-2016。

### 算法参数

| 参数 | SM4 | AES-128 | DES |
|---|---|---|---|
| 分组长度 | 128 位 | 128 位 | 64 位 |
| 密钥长度 | 128 位 | 128 位 | 56 位 |
| 轮数 | 32 | 10 | 16 |
| 结构 | 类 DES (Feistel) | SPN | Feistel |

### 算法结构

SM4 使用 **Feistel 结构**，每轮结构：

```python
# SM4 核心操作
def SM4_round(X, rk):
    """
    X: 当前状态（4 个 32 位字）
    rk: 轮密钥
    """
    # 1. T 函数：非线性变换
    T_input = X[0] ^ X[1] ^ X[2] ^ X[3] ^ rk

    # 2. T 函数内部
    #    - S 盒替换
    #    - 循环左移
    #    - 线性变换

    return (X[3], X[2], X[1], X[0] ^ T(T_input))

def SM4_encrypt(plaintext_blocks, round_keys):
    """
    plaintext_blocks: 4 个 32 位字为一组
    round_keys: 32 个轮密钥
    """
    state = plaintext_blocks

    for i in range(32):
        state = SM4_round(state, round_keys[i])

    # 最后一轮的反序变换
    return state[3], state[2], state[1], state[0]
```

### SM4 的 S 盒

SM4 使用 8 个并行的 S 盒，与 AES 的 S 盒不同：

```python
# SM4 S 盒（简化示意）
SBOX = [
    0xD6, 0x90, 0xE9, 0xFE, 0xCC, 0xE1, 0x3D, 0xB7, ...
]

def sm4_sbox(byte):
    return SBOX[byte]

def T(x):
    """非线性变换 T"""
    # 1. S 盒替换
    result = 0
    for i in range(4):
        result |= sm4_sbox((x >> (8*i)) & 0xFF) << (8*i)

    # 2. 循环左移 + 线性变换
    result = result ^ rotl(result, 2) ^ rotl(result, 10) ^ rotl(result, 18) ^ rotl(result, 24)

    return result
```

## SM2 非对称加密

SM2 是基于椭圆曲线的非对称密码算法，标准于 2010 年发布。

### SM2 曲线参数

SM2 使用专用的椭圆曲线，参数公开：

```python
# SM2 曲线参数
SM2_CURVE = {
    "p": 0xFFFFFFFEFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF00000000FFFFFFFFFFFFFFFF,
    "a": 0xFFFFFFFEFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF00000000FFFFFFFFFFFFFFFC,
    "b": 0x28E9FA9E9D9F5E344D5A9E4BCF6509A7F39789F515AB8F92DDBCBD414D940E93,
    "n": 0xFFFFFFFEFFFFFFFFFFFFFFFFFFFFFFFF7203DF6B21C6052B53BBF40939D54123,
    "G": (  # 基点
        0x32C4AE2C1F1981195F9904466A39C9948FE30BBFF2660BE1715A4589334C74C7,
        0xBC3736A2F4F6779C59BDCEE36B692153D0A9877CC62A474002DF32E52139F0A0
    )
}
```

### SM2 加密流程

```python
# SM2 加密（简化）
def sm2_encrypt(plaintext, public_key):
    """
    SM2 加密过程
    """
    # 1. 生成随机数 k
    k = generate_random(32)

    # 2. 计算点 C1 = k * G = (x1, y1)
    C1 = k * SM2_CURVE["G"]

    # 3. 计算点 S = h * PB（验证公钥）
    # 4. 计算共享点 (x2, y2) = k * PB
    S_point = k * public_key
    x2, y2 = S_point.x, S_point.y

    # 5. 计算 t = KDF(x2 || y2, klen)
    t = KDF(x2 + y2, len(plaintext))

    # 6. C2 = plaintext ⊕ t
    C2 = bytes([m ^ t[i] for i, m in enumerate(plaintext)])

    # 7. C3 = SM3(x2 || plaintext || y2)
    C3 = SM3(x2 + plaintext + y2)

    return C1 + C2 + C3  # 格式：x1||y1||C3||C2
```

### SM2 签名

SM2 签名与 ECDSA 类似，但使用了 Z 值预处理：

```python
# SM2 签名
def sm2_sign(message, private_key, user_id, public_key):
    """
    SM2 签名过程
    user_id: 用户标识
    public_key: 用户公钥
    """
    # 1. 计算 Z = SM3(ENTL || ID || a || b || G || P)
    Z = sm3_hash(user_id + curve_params)

    # 2. 计算 e = SM3(Z || message)
    e = sm3_hash(Z + message)

    # 3. 生成随机数 k
    k = generate_random(32)

    # 4. 计算点 R = k * G = (x1, y1)
    R = k * SM2_CURVE["G"]
    x1 = R.x

    # 5. 计算 r = (e + x1) mod n
    r = (e + x1) % SM2_CURVE["n"]

    # 6. 计算 s = (d + 1)^(-1) * (k - r * d) mod n
    d = private_key
    s = pow(d + 1, -1, SM2_CURVE["n"]) * (k - r * d) % SM2_CURVE["n"]

    return (r, s)
```

## SM3 消息摘要

SM3 是密码散列算法，输出 256 位，与 SHA-256 相当。

### 算法结构

```python
# SM3 核心结构
import struct

IV = [
    0x7380166f, 0x4914b2b9, 0x172442d7, 0xda8a0600,
    0xa96f30bc, 0x163138aa, 0xe38dee4d, 0xb0fb0e4e
]

def sm3_permutation(X, Y, Z):
    """SM3 置换函数"""
    return X ^ Y ^ Z

def sm3_ff_j(X, Y, Z, j):
    """SM3 非线性函数"""
    if j < 16:
        return X ^ Y ^ Z
    else:
        return (X & Y) | (X & Z) | (Y & Z)

def sm3_gg_j(X, Y, Z, j):
    """SM3 非线性函数"""
    if j < 16:
        return X ^ Y ^ Z
    else:
        return (X & Y) | (~X & Z)

def sm3_round(A, B, C, D, E, F, G, H, W, W_prime, j):
    """SM3 一轮操作"""
    SS1 = rotl((rotl(A, 12) + E + T(j)) & 0xFFFFFFFF, 7)
    SS2 = SS1 ^ rotl(A, 12)
    TT1 = (ff_j(A, B, C, j) + D + SS2 + W_prime[j]) & 0xFFFFFFFF
    TT2 = (gg_j(E, F, G, j) + H + SS1 + W[j]) & 0xFFFFFFFF
    D = C
    C = rotl(B, 9)
    B = A
    A = TT1
    H = G
    G = rotl(F, 19)
    F = E
    E = P0(TT2)
    return A, B, C, D, E, F, G, H
```

## SM9 标识密码

SM9 是标识密码算法（IBC），最大的特点是**不需要数字证书**。

传统 PKI vs SM9：

| 传统 PKI | SM9 |
|---|---|
| 需要 CA 颁发证书 | 不需要证书 |
| 公钥是一串随机数 | 公钥可以是邮箱、手机号等标识 |
| 证书管理复杂 | 密钥管理简单 |
| 验证需要证书链 | 直接验证 |

```python
# SM9 标识加密示例
def sm9_encrypt(identifier, message, master_public):
    """
    SM9 加密
    identifier: 接收方标识（如邮箱 "alice@example.com"）
    """
    # 1. 生成随机数 r
    r = generate_random(32)

    # 2. 计算点 C1 = r * Ppub（主公钥点）
    C1 = r * master_public

    # 3. 计算共享密钥
    #    Q = H1(identifier || hid, N) * Ppub
    #    point = r * Q
    Q_id = hash_to_point(identifier, 3)  # hid = 3 表示加密
    shared_point = r * Q_id

    # 4. 用 KDF 派生密钥，加密消息
    klen = len(message)
    k = KDF(shared_point.x || shared_point.y, klen)
    C2 = bytes([m ^ k[i] for i, m in enumerate(message)])

    # 5. 计算 Tag = SM3(x2 || message || y2)
    C3 = sm3_hash(shared_point.x + message + shared_point.y)

    return C1, C3, C2
```

## 实际应用

### 国密 HTTPS

```nginx
# Nginx 国密配置
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers ECDHE-SM2-SM4-CBC-SM3:ECDHE-SM2-SM4-GCM-SM3;

# 需要国密证书
ssl_certificate /path/to/sm2_cert.pem;
ssl_certificate_key /path/to/sm2_key.pem;
ssl_ca_certificate /path/to/sm2_ca.pem;
```

### Java 国密实现

```java
import org.bouncycastle.jce.provider.GM;

public class GmExample {

    public static void main(String[] args) throws Exception {
        // 注册国密提供者
        Security.addProvider(new GM());

        // SM4 加密
        KeyGenerator keyGen = KeyGenerator.getInstance("SM4", "BC");
        keyGen.init(128);
        SecretKey sm4Key = keyGen.generateKey();

        Cipher cipher = Cipher.getInstance("SM4/CBC/PKCS5Padding", "BC");
        byte[] iv = new SecureRandom().generateSeed(16);
        IvParameterSpec ivSpec = new IvParameterSpec(iv);
        cipher.init(Cipher.ENCRYPT_MODE, sm4Key, ivSpec);

        byte[] plaintext = "Hello, GM!".getBytes();
        byte[] ciphertext = cipher.doFinal(plaintext);

        // SM2 签名
        KeyPairGenerator keyGen2 = KeyPairGenerator.getInstance("SM2", "BC");
        keyGen2.initialize(256);
        KeyPair keyPair = keyGen2.generateKeyPair();

        Signature signature = Signature.getInstance("SM3withSM2", "BC");
        signature.initSign(keyPair.getPrivate());
        signature.update(plaintext);
        byte[] sign = signature.sign();
    }
}
```
