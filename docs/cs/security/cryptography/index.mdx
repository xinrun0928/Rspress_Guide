# 密码学基础

密码学是信息安全的基石。无论是 HTTPS 传输、银行交易还是区块链应用，背后都有密码学在默默守护。理解密码学原理，是成为安全工程师的必修课。

## 知识图谱

本章节涵盖以下核心内容：

| 分类 | 内容 | 应用场景 |
|---|---|---|
| 对称加密 | AES、DES、3DES、SM4 | 数据加密存储 |
| 非对称加密 | RSA、ECC、SM2 | 密钥协商、数字签名 |
| 混合加密 | 密钥封装 + 数据加密 | TLS/SSL 协议 |
| 消息摘要 | MD5、SHA 系列、SM3 | 数据完整性校验 |
| 数字签名 | RSA 签名、ECDSA、SM2 签名 | 身份认证、抗抵赖 |
| 密钥交换 | DH、ECDH、X25519 | 密钥协商协议 |
| 国密算法 | SM2、SM3、SM4、SM9 | 国产化替代 |

## 学习路径

建议按照以下顺序学习：

1. **先理解对称加密**：最直观，加解密使用同一把钥匙
2. **再学习非对称加密**：理解公钥和私钥的数学关系
3. **混合加密**：解决密钥分发难题
4. **消息摘要**：理解单向函数
5. **数字签名**：结合非对称加密和摘要
6. **密钥交换**：理解如何安全传输密钥
7. **国密算法**：了解国产化要求

## 面试要点

密码学是面试中的高频考点，以下问题值得重点准备：

- AES 和 DES 的区别？为什么 DES 被废弃？
- RSA 的数学原理？大数分解为什么困难？
- 什么是对称加密的密钥分发问题？
- HTTPS 握手过程？用了哪些密码学原语？
- 数字签名和消息认证码的区别？
- 什么是前向安全？

> 密码学不仅要知道「是什么」，更要理解「为什么这样设计」和「代价是什么」。
