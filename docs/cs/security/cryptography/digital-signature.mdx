# 数字签名

老板发来一封邮件："请转账 100 万到 xxx 账户"。你敢转吗？

数字签名就是为了解决**身份认证**和**抗抵赖**的问题。谁发的消息，谁签过字，事后无法否认。

## 什么是数字签名？

数字签名是**非对称加密的逆向应用**：

```
加密：公钥加密 -> 私钥解密
签名：私钥签名 -> 公钥验证
```

核心特性：
- **真实性**：签名只能用私钥产生，公钥持有者可以验证
- **完整性**：签名覆盖整个消息，修改消息签名失效
- **抗抵赖**：只有私钥持有者能签名，无法否认

```
Alice                                Bob
  │                                   │
  │  1. 计算消息摘要                   │
  │     digest = SHA256(message)      │
  │                                   │
  │  2. 用私钥签名摘要                 │
  │     signature = RSA(digest, alice_private) │
  │                                   │
  │  3. 发送消息 + 签名                │
  ├───────────────────────────────────┤
  │  message + signature              │
  ├───────────────────────────────────┤
  │                                   │
  │  4. Bob 获得 Alice 的公钥         │
  │  5. 验证签名                       │
  │     expected = SHA256(message)   │
  │     actual = verify(signature, alice_public) │
  │     if expected == actual: 验证通过 │
```

## RSA 签名

RSA 签名是最经典的数字签名方案：

```
签名：signature = message^d mod N
验证：message == signature^e mod N
```

### 签名的是摘要，不是消息

:::warning
绝对不能直接对消息进行签名！
:::

为什么？因为 RSA 的数学性质，存在选择消息攻击：

```
已知：签名 σ = m^d mod N

攻击者可以伪造签名：
σ' = σ * s^e mod N  （s 是任意值）
m' = m * s mod N

验证：σ'^e = (σ * s^e)^e = σ^e * s = m * s = m'
```

正确的做法是签名消息摘要：

```
签名：signature = (H(message))^d mod N
验证：H(message) == signature^e mod N
```

### RSA-PSS 填充

与 RSA-OAEP 类似，RSA 签名也需要填充，使用 **RSA-PSS**：

```python
import hashlib
import os
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.padding import PKCS7
from cryptography.hazmat.backends import default_backend

def rsa_sign(private_key, message):
    """RSA-PSS 签名"""
    # 1. 计算消息摘要
    digest = hashes.SHA256()._hashed_with_data(message)

    # 2. PSS 填充
    #    EM = DB || salt || T_0
    #    其中 T_0 = 00...0 (8位0)
    #    salt 是随机生成的

    # 3. 用私钥解密 EM
    signature = private_key.sign(
        digest,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        )
    )
    return signature

def rsa_verify(public_key, message, signature):
    """RSA-PSS 验证"""
    digest = hashes.SHA256()._hashed_with_data(message)

    try:
        public_key.verify(
            signature,
            digest,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()),
                salt_length=padding.PSS.MAX_LENGTH
            )
        )
        return True
    except:
        return False
```

## ECDSA 椭圆曲线签名

ECDSA 是基于椭圆曲线的签名算法，比 RSA 签名更高效：

| 指标 | RSA-2048 | ECDSA-P256 |
|---|---|---|
| 签名长度 | 256 字节 | 64 字节 |
| 签名速度 | 慢（15ms） | 快（1ms） |
| 验证速度 | 慢 | 快 |
| 密钥长度 | 2048 位 | 256 位 |

### ECDSA 数学原理

ECDSA 使用椭圆曲线上的数学运算：

```python
# ECDSA 签名算法
import random
import hashlib

class ECDSA:
    def __init__(self, curve):
        self.curve = curve  # 例如 secp256k1

    def sign(self, private_key, message_hash):
        G = self.curve.G  # 基点
        n = self.curve.n  # 阶

        # 1. 生成随机数 k（关键：必须随机且不可预测）
        k = random.randrange(1, n)

        # 2. 计算点 R = k * G
        R = k * G
        r = R.x % n  # 取 x 坐标模 n

        if r == 0:
            return self.sign(private_key, message_hash)  # 重新生成

        # 3. 计算 s = k^(-1) * (H(m) + r * d) mod n
        #    其中 d 是私钥，H(m) 是消息哈希
        s = (pow(k, -1, n) * (int(message_hash, 16) + r * private_key)) % n

        if s == 0:
            return self.sign(private_key, message_hash)  # 重新生成

        return (r, s)

    def verify(self, public_key, message_hash, signature):
        G = self.curve.G
        n = self.curve.n
        r, s = signature

        if not (1 <= r < n and 1 <= s < n):
            return False

        # 1. 计算 w = s^(-1) mod n
        w = pow(s, -1, n)

        # 2. 计算 u1 = H(m) * w mod n
        #    计算 u2 = r * w mod n
        u1 = (int(message_hash, 16) * w) % n
        u2 = (r * w) % n

        # 3. 计算点 P = u1 * G + u2 * public_key
        P = u1 * G + u2 * public_key

        # 4. 验证 v = P.x mod n == r
        v = P.x % n

        return v == r
```

### ECDSA 的危险：随机数重用

:::danger
ECDSA 的随机数 k 如果泄露，私钥将被完全暴露！
:::

如果相同的 k 被重用，可以从两个签名中恢复私钥：

```
签名1: (r, s1) 来自消息 m1
签名2: (r, s2) 来自消息 m2
使用相同的 k

s1 = k^(-1) * (H(m1) + r * d) mod n
s2 = k^(-1) * (H(m2) + r * d) mod n

s1 - s2 = k^(-1) * (H(m1) - H(m2)) mod n
k = (H(m1) - H(m2)) * (s1 - s2)^(-1) mod n
d = (s1 * k - H(m1)) * r^(-1) mod n
```

PlayStation 3 的 ECDSA 私钥泄露事件就是因为代码中硬编码了随机数。

## Ed25519

Ed25519 是现代化的签名算法，基于 Curve25519 曲线：

| 特性 | Ed25519 | ECDSA-P256 | RSA-2048 |
|---|---|---|---|
| 签名长度 | 64 字节 | 64 字节 | 256 字节 |
| 密钥长度 | 256 位 | 256 位 | 2048 位 |
| 签名速度 | 最快 | 快 | 慢 |
| 安全性 | 高 | 高 | 中 |
| 确定性签名 | 是 | 否 | 是 |

Ed25519 的优势：
- 签名是确定性的（不依赖随机数）
- 对侧信道攻击有更好的抵抗力
- 简化实现，减少 bug 可能性

```python
# Ed25519 签名示例
from nacl.signing import SigningKey

# 生成密钥
signing_key = SigningKey.generate()
verify_key = signing_key.verify_key

# 签名
message = b"Hello, Ed25519!"
signed = signing_key.sign(message)
signature = signed.signature  # 前 64 字节

# 验证
verify_key.verify(signed)
```

## SM2 国密签名

SM2 签名是中国国家密码标准，基于 SM2 椭圆曲线：

```python
# SM2 签名核心概念
"""
SM2 签名与 ECDSA 类似，但：
1. 使用 SM2 专用曲线（256 位）
2. 使用 SM3 哈希算法
3. 签名过程有预处理（z 值）

签名：
1. 计算 e = SM3(ENTL || IDA || PA || message)
2. 生成随机数 k
3. 计算点 R = k * G = (x1, y1)
4. 计算 r = (e + x1) mod n
5. 计算 s = (d + k)^(-1) * (k - r * d) mod n

验证：
1. 计算 e' = SM3(ENTL || IDA || PA || message)
2. 计算 t = (r + s) mod n
3. 计算点 (x1', y1') = s * G + t * PA
4. 验证 r == (e' + x1') mod n
"""
```

## 实际应用

### 代码签名

```bash
# 使用 OpenSSL 生成自签名代码签名证书
openssl req -x509 -new -nodes -key private.key \
  -sha256 -days 1024 -out code_signing.crt \
  -subj "/CN=Code Signing/O=My Company"

# 对二进制文件签名
codesign -d -vvv MyApp.app  # 查看签名信息
codesign -s "Developer ID: My Company" MyApp.app  # macOS 代码签名

# Windows Authenticode
signtool sign /f certificate.pfx /p password /tr http://timestamp.digicert.com MyApp.exe
```

### 文档签名

```java
// PDF 签名示例
import org.bouncycastle.cms.CMSSignedData;
import org.bouncycastle.cms.SignerInformationStore;
import org.bouncycastle.cms.jcajce.JcaSignerInfoGeneratorBuilder;

public class PDFSignature {
    public void signPDF(File pdfFile, PrivateKey privateKey, Certificate cert) throws Exception {
        // 1. 计算 PDF 的摘要
        // 2. 用私钥签名摘要
        // 3. 将签名嵌入 PDF

        CMSSignedData signedData = new CMSSignedData(
            new SignerInformationStore(
                new JcaSignerInfoGeneratorBuilder(
                    new JcaDigestCalculatorProviderBuilder().build()
                ).build(privateKey, cert)
            )
        );
        // ...
    }
}
```

### Git 签名提交

```bash
# 生成 GPG 签名密钥
gpg --full-generate-key
gpg --list-secret-keys

# 配置 Git 使用签名
git config --global user.signingkey <key-id>
git config --global commit.gpgsign true

# 签名提交
git commit -S -m "feat: add new feature"

# 查看签名状态
git log --show-signature
```
