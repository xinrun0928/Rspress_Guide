# 密钥交换

Alice 想给 Bob 发加密消息，但她没有安全的信道传输密钥。怎么办？

这个问题困扰了密码学家几十年。直到 Diffie 和 Hellman 在 1976 年提出了一个优雅的解决方案——**Diffie-Hellman 密钥交换**。

## Diffie-Hellman 密钥交换

DH 的核心思想是：**利用数学的不可逆性，即使通信被完全监听，双方也能共享一个秘密。**

### 数学原理

DH 基于**离散对数难题**：

```
已知：g, p, y
求：x 使得 g^x ≡ y (mod p)

对于大素数 p，这个问题是计算上不可行的。
```

### 协议流程

```
Alice                                          Bob
  │                                             │
  │  1. 双方协商公开参数                         │
  │     p = 大素数 (例如 23)                     │
  │     g = 原根 (例如 5)                       │
  │ ─────────────────────────────────────────────>
  │                                             │
  │  2. Alice 选择私钥 a = 6                    │
  │     计算 A = g^a mod p = 5^6 mod 23 = 8      │
  │ ─────────────────────────────────────────────>
  │                              Alice 的公钥 A=8
  │                                             │
  │                              Bob 选择私钥 b = 15
  │ 3. Bob 计算 B = g^b mod p = 5^15 mod 23 = 19
  │ <─────────────────────────────────────────────
  │                              Bob 的公钥 B=19
  │                                             │
  │  4. Alice 计算共享密钥                       │
  │     K = B^a mod p = 19^6 mod 23 = 2         │
  │                                             │
  │     Bob 计算共享密钥                         │
  │     K = A^b mod p = 8^15 mod 23 = 2         │
  │                                             │
  │  共享密钥 K = 2                              │
```

数学验证：

```
Alice: K = B^a = (g^b)^a = g^(ab) = (g^a)^b = A^b = Bob 计算的 K
```

### 代码实现

```python
def diffie_hellman(p, g, a, b):
    """
    Diffie-Hellman 密钥交换
    p: 大素数
    g: 原根（生成元）
    a: Alice 的私钥
    b: Bob 的私钥
    """
    # Alice 计算公钥
    A = pow(g, a, p)

    # Bob 计算公钥
    B = pow(g, b, p)

    # 双方计算共享密钥
    # Alice: K = B^a mod p
    K_alice = pow(B, a, p)

    # Bob: K = A^b mod p
    K_bob = pow(A, b, p)

    assert K_alice == K_bob, "密钥不一致！"

    return K_alice

# 使用示例
p = 23
g = 5
a = 6  # Alice 的私钥
b = 15  # Bob 的私钥

shared_key = diffie_hellman(p, g, a, b)
print(f"共享密钥: {shared_key}")  # 输出: 2
```

### 安全性分析

DH 的安全性依赖于：

| 假设 | 说明 |
|---|---|
| 离散对数难题 | 已知 g、p、A 无法在合理时间内计算出 a |
| 随机数 | 私钥必须是随机生成的 |
| 足够大的参数 | p 至少 2048 位，推荐 3072 位或 4096 位 |

#### 中间人攻击（MITM）

:::danger
标准 DH 不提供身份认证！存在中间人攻击风险！
:::

```
Attacker (Mallory)                           Alice
    │                                            │
    │        拦截 Alice 的 g^a mod p              │
    │   A_mal = g^a mod p                        │
    │ <───────────────────────────────────────────
    │                                            │
    │        发送自己的公钥给 Bob                  │
    │   B_mal = g^b mod p                        │
    │ ─────────────────────────────────────────────>
    │                                    Bob
    │
    │  与 Alice 共享 K1 = g^(a*b1)                │
    │  与 Bob 共享 K2 = g^(a1*b)                  │
    │
    │  Mallory 可以解密、修改、重加密所有消息！
```

**防御措施**：使用认证的密钥交换（Authenticated DH）或数字签名。

## ECDH 椭圆曲线 DH

ECDH 是 DH 的椭圆曲线版本，更高效：

| 指标 | 经典 DH (2048 位) | ECDH (P-256) |
|---|---|---|
| 公钥大小 | 256 字节 | 64 字节 |
| 私钥大小 | 2048 位 | 256 位 |
| 共享密钥大小 | 256 字节 | 32 字节 |
| 计算速度 | 慢 | 快 10 倍 |

### ECDH 原理

ECDH 使用椭圆曲线点乘法代替模幂运算：

```
Alice:
  - 选择私钥 d_A（随机数）
  - 计算公钥 P_A = d_A * G

Bob:
  - 选择私钥 d_B（随机数）
  - 计算公钥 P_B = d_B * G

共享密钥：
  - Alice: K = d_A * P_B = d_A * (d_B * G)
  - Bob: K = d_B * P_A = d_B * (d_A * G)
  - 两人得到相同的点 K
```

```python
# ECDH 示例（使用 ECDH 库）
from cryptography.hazmat.primitives.asymmetric import ec
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

def ecdh_key_exchange():
    """ECDH 密钥交换示例"""
    # 1. Alice 生成密钥对
    alice_private = ec.generate_private_key(ec.SECP256R1())
    alice_public = alice_private.public_key()

    # 2. Bob 生成密钥对
    bob_private = ec.generate_private_key(ec.SECP256R1())
    bob_public = bob_private.public_key()

    # 3. 双方交换公钥（可明文传输）
    # 4. Alice 计算共享密钥
    alice_shared = alice_private.exchange(ec.ECDH(), bob_public)

    # 5. Bob 计算共享密钥
    bob_shared = bob_private.exchange(ec.ECDH(), alice_public)

    assert alice_shared == bob_shared

    # 6. 派生会话密钥（HKDF）
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.hkdf import HKDF
    session_key = HKDF(
        algorithm=hashes.SHA256(),
        length=32,
        salt=None,
        info=b'handshake data'
    ).derive(alice_shared)

    return session_key

session_key = ecdh_key_exchange()
print(f"会话密钥: {session_key.hex()}")
```

## X25519

X25519 是 Curve25519 曲线的 ECDH 实现，是现代密码系统推荐的算法：

| 特性 | X25519 | ECDH-P256 | DH-2048 |
|---|---|---|---|
| 密钥长度 | 256 位 | 256 位 | 2048 位 |
| 安全性 | 128 位 | 128 位 | 112 位 |
| 性能 | 最快 | 快 | 慢 |
| 抗侧信道 | 好 | 中 | 差 |

X25519 的优势：

1. **简单**：曲线参数固定，减少错误
2. **快速**：使用优化的蒙哥马利阶梯算法
3. **安全**：设计时考虑了各种攻击
4. **无歧义**：公钥表示唯一

## TLS 中的密钥交换

TLS 握手使用多种密钥交换算法：

### TLS 1.3 支持的密钥交换

| 算法 | 名称 | 前向安全 |
|---|---|---|
| X25519 | ECDH over Curve25519 | 是 |
| P-256 | ECDH over NIST P-256 | 是 |
| P-384 | ECDH over NIST P-384 | 是 |
| P-521 | ECDH over NIST P-521 | 是 |

## 前向安全（Forward Secrecy）

### 什么是前向安全？

如果长期私钥泄露，**前向安全**确保之前的会话密钥不被泄露。

```
无前向安全（RSA 密钥交换）：
  - 服务器私钥泄露 -> 所有历史会话可解密

有前向安全（ECDHE）：
  - 服务器私钥泄露 -> 无法恢复会话密钥（因为每次使用新密钥）
```

### TLS 中的前向安全

```bash
# 查看网站支持的 TLS 密码套件
openssl s_client -connect example.com:443 -tls1_3

# 推荐的服务器配置（只启用前向安全的密码套件）

SSLProtocol -all +TLSv1.3
```
