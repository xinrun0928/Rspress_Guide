# 混合加密

对称加密快但密钥分发困难，非对称加密安全但速度太慢。怎么办？答案是：**两者结合，取长补短。**

混合加密不是一种新的加密算法，而是一种**系统设计思想**：用非对称加密安全地传输对称密钥，然后用对称加密高效地加解密数据。

## 为什么需要混合加密？

让我们比较一下对称和非对称加密的特点：

| 特性 | 对称加密 | 非对称加密 |
|---|---|---|
| 速度 | 快 100-1000 倍 | 慢 |
| 密钥分发 | 困难，需要安全通道 | 简单，公钥可公开 |
| 密钥数量 | O(n²) 增长 | O(n)，每人只需一对 |
| 适用场景 | 大量数据加密 | 密钥协商、身份认证 |

非对称加密的加解密速度对比：

```
RSA-2048 加密：约 0.5 ms/次
RSA-2048 解密：约 15 ms/次
AES-256-GCM 加密：约 0.01 ms/次
```

处理 1MB 数据：
- 纯 RSA：约 15 秒
- AES-256-GCM：约 0.01 秒

**结论**：非对称加密不适合加密大量数据，但可以安全地传输对称密钥。

## 混合加密的流程

```
Alice                                Bob
  │                                   │
  │  1. 生成会话密钥 (对称)            │
  │     session_key = random(256)     │
  │                                   │
  │  2. 用 Bob 的公钥加密会话密钥      │
  │     encrypted_key = RSA(session_key, bob_public)  │
  │                                   │
  │  3. 用会话密钥加密数据             │
  │     encrypted_data = AES(data, session_key)       │
  │                                   │
  ├───────────────────────────────────┤
  │  发送: encrypted_key + encrypted_data             │
  ├───────────────────────────────────┤
  │                                   │
  │  4. 用私钥解密会话密钥             │
  │     session_key = RSA(encrypted_key, bob_private)  │
  │                                   │
  │  5. 用会话密钥解密数据             │
  │     data = AES(encrypted_data, session_key)       │
  │                                   │
```

## 简单实现

```python
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.backends import default_backend
import os

class HybridEncryption:
    """混合加密实现"""

    @staticmethod
    def generate_keypair():
        """生成 RSA 密钥对"""
        private_key = rsa.generate_private_key(
            public_exponent=65537,
            key_size=2048,
            backend=default_backend()
        )
        public_key = private_key.public_key()
        return private_key, public_key

    @staticmethod
    def generate_session_key():
        """生成 AES 会话密钥（256 位）"""
        return os.urandom(32)  # 256 位

    @staticmethod
    def encrypt(public_key, plaintext):
        """
        混合加密：
        1. 生成随机会话密钥
        2. 用 RSA-OAEP 加密会话密钥
        3. 用 AES-GCM 加密数据
        """
        # 1. 生成会话密钥
        session_key = HybridEncryption.generate_session_key()

        # 2. 用 RSA 加密会话密钥
        encrypted_key = public_key.encrypt(
            session_key,
            padding.OAEP(
                mgf=padding.MGF1(algorithm=hashes.SHA256()),
                algorithm=hashes.SHA256(),
                label=None
            )
        )

        # 3. 用 AES-GCM 加密数据
        nonce = os.urandom(12)  # 96 位 nonce
        aesgcm = AESGCM(session_key)
        ciphertext = aesgcm.encrypt(nonce, plaintext, None)

        # 返回：密钥长度(4字节) + 加密密钥 + nonce + 密文
        key_length = len(encrypted_key).to_bytes(4, 'big')
        return key_length + encrypted_key + nonce + ciphertext

    @staticmethod
    def decrypt(private_key, encrypted_data):
        """
        混合解密：
        1. 提取加密的会话密钥
        2. 用 RSA 解密会话密钥
        3. 用 AES-GCM 解密数据
        """
        # 1. 提取加密的会话密钥
        offset = 0
        key_length = int.from_bytes(encrypted_data[offset:offset+4], 'big')
        offset += 4
        encrypted_key = encrypted_data[offset:offset+key_length]
        offset += key_length

        # 2. 用 RSA 解密会话密钥
        session_key = private_key.decrypt(
            encrypted_key,
            padding.OAEP(
                mgf=padding.MGF1(algorithm=hashes.SHA256()),
                algorithm=hashes.SHA256(),
                label=None
            )
        )

        # 3. 提取 nonce 和密文
        nonce = encrypted_data[offset:offset+12]
        offset += 12
        ciphertext = encrypted_data[offset:]

        # 4. 用 AES-GCM 解密
        aesgcm = AESGCM(session_key)
        plaintext = aesgcm.decrypt(nonce, ciphertext, None)

        return plaintext

# 使用示例
if __name__ == "__main__":
    # 生成密钥对
    private_key, public_key = HybridEncryption.generate_keypair()

    # 加密
    message = b"这是一条秘密消息，需要安全传输"
    encrypted = HybridEncryption.encrypt(public_key, message)
    print(f"加密后长度: {len(encrypted)} 字节")

    # 解密
    decrypted = HybridEncryption.decrypt(private_key, encrypted)
    print(f"解密结果: {decrypted.decode('utf-8')}")
```

## TLS 中的混合加密

TLS（Transport Layer Security）是混合加密最典型的应用场景。

### TLS 1.2 握手（RSA 密钥交换）

```
Client                              Server
  │                                   │
  │  1. ClientHello                   │
  │     (支持的密码套件列表)           │
  │ ─────────────────────────────────>│
  │                                   │
  │  2. ServerHello                   │
  │     (选定的密码套件)               │
  │  3. Certificate                   │
  │     (服务器证书，含公钥)           │
  │ ─────────────────────────────────>│
  │                                   │
  │  4. ClientKeyExchange             │
  │     (用服务器公钥加密的             │
  │      客户端随机数 +                  │
  │      预主密钥 PremasterSecret)    │
  │ ─────────────────────────────────>│
  │                                   │
  │  5. 生成主密钥                     │
  │     master_secret =                │
  │       PRF(premaster_secret,        │
  │            client_random +          │
  │            server_random)          │
  │                                   │
  │  6. Finished                       │
  │ ─────────────────────────────────>│
  │                                   │
  │                                   │
  │  7. 生成主密钥 (同上)              │
  │  8. Finished                       │
  │ <─────────────────────────────────│
```

### TLS 1.3 握手（改进）

TLS 1.3 改进了握手过程：

1. **更短**：从 2-RTT 减少到 1-RTT
2. **更安全**：废弃 RSA 密钥交换，只支持 ECDHE
3. **更快**：0-RTT 恢复会话

```
Client                              Server
  │                                   │
  │  1. ClientHello                   │
  │     (支持的椭圆曲线)               │
  │     + 客户端 Diffie-Hellman 参数  │
  │ ─────────────────────────────────>│
  │                                   │
  │  2. ServerHello                   │
  │     + 服务器 Diffie-Hellman 参数  │
  │  3. EncryptedExtensions           │
  │  4. Certificate                   │
  │  5. CertificateVerify             │
  │  6. Finished                      │
  │ <─────────────────────────────────│
  │                                   │
  │  7. 生成主密钥                     │
  │  8. Finished                      │
  │ ─────────────────────────────────>│
```

## 安全性分析

### 需要避免的危险模式

| 错误模式 | 风险 |
|---|---|
| 使用 RSA 直接加密数据 | 小明文攻击 |
| 使用 ECB 模式加密大量数据 | 模式泄露 |
| 重复使用相同的会话密钥 | 密钥相关攻击 |
| 使用非确定性填充 | 信息泄露 |

### 正确的设计原则

| 原则 | 说明 |
|---|---|
| 每次加密使用新的随机 nonce | 防止重放攻击 |
| 每次加密使用新的随机种子 | 确保密钥流不可预测 |
| 验证完整性 | 使用 GCM 等认证加密模式 |
| 清理敏感数据 | 使用后立即清零密钥 |
