# 网络层安全

你在一咖啡厅连接了公共 WiFi，登录了银行 App。殊不知，隔壁可能有人正在用 Wireshark 抓包，分析你的流量。这就是网络层安全问题的缩影——**数据在网络中传输时，面临着窃听、篡改、伪造等多种威胁**。

网络层安全是信息安全的护城河。无论是浏览网页、发送邮件还是远程办公，数据都要经过复杂的网络路径。在这条路径上，任何一个节点被攻破，都可能导致数据泄露。本章将带你了解网络层的安全机制，从协议加密到攻击防御，构建完整的安全知识体系。

## 知识图谱

本章节涵盖以下核心内容：

| 分类 | 内容 | 防御目标 |
|---|---|---|
| 传输加密 | TLS/SSL、HTTPS、HSTS | 数据保密性、完整性 |
| DNS 安全 | DoH、DoT、DNSSEC | DNS 劫持防护 |
| 局域网安全 | ARP 欺骗、MITM 攻击 | 局域网安全 |
| DoS/DDoS | SYN Flood、UDP Flood、CC 攻击 | 服务可用性 |
| VPN 技术 | IPsec、SSL VPN、MPLS | 传输安全、远程访问 |
| 边界防护 | 防火墙、IDS、IPS | 访问控制、入侵检测 |

## 学习路径

建议按照以下顺序学习：

1. **先理解 TLS/HTTPS**：最常见的网络加密方案
2. **再学习 DNS 安全**：DNS 是互联网的基础服务
3. **了解局域网攻击**：ARP 欺骗是内网渗透的经典手法
4. **掌握 DoS/DDoS 防御**：保障服务可用性
5. **学习 VPN 技术**：安全远程访问
6. **理解边界防护**：防火墙、IDS、IPS 构建最后防线

## 面试要点

网络层安全是面试中的高频考点，以下问题值得重点准备：

- HTTPS 的握手过程？用了哪些密码学原语？
- TLS 1.2 和 TLS 1.3 的区别？
- 数字证书的验证流程？
- DNS 污染和 DNS 劫持的区别？
- ARP 欺骗的原理？如何防御？
- DoS 和 DDoS 的区别？
- SYN Flood 的原理和防御方法？
- IPsec 的两种模式：传输模式和隧道模式
- 防火墙和 IDS/IPS 的区别？

> 网络层安全不仅是防御，更是攻守平衡的艺术。理解攻击才能更好地防御。
