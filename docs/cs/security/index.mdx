# 计算机安全

你有没有想过，为什么一个看似普通的 Web 应用，会被黑客轻松拿下？

很多安全漏洞的根源，不是代码写得烂，而是**开发者对安全的认知存在盲区**。你以为加了防火墙就安全了？不对。你以为用了 HTTPS 就高枕无忧了？也不对。

安全是一个**纵深防御**的系统工程——从密码学的底层原理，到应用层的攻防对抗，再到系统内核的安全机制，每一层都有它独特的攻击面和防御手段。

本专题将带你系统掌握计算机安全的核心知识，无论你是准备安全面试，还是想构建更安全的系统，这里都有你需要的答案。

## 安全全景图

### 五大安全领域

| 领域 | 核心问题 | 典型威胁 |
|---|---|---|
| 密码学基础 | 如何安全地加密、签名、交换密钥？ | 弱加密、密钥泄露 |
| 身份认证与授权 | 如何证明「你是你」？如何控制你能做什么？ | 凭证盗取、越权访问 |
| 应用层安全 | Web 应用如何被攻击？ | XSS、SQL 注入、CSRF |
| 系统层安全 | 操作系统如何被攻破？ | 缓冲区溢出、权限提升 |
| 网络层安全 | 网络通信如何被窃听或篡改？ | 中间人攻击、DNS 劫持 |

### 攻击与防御的博弈

```
攻击者视角：发现入口 →  reconnaissance 信息收集 → 漏洞利用 → 权限提升 → 持久化
防御者视角：边界防护 → 纵深防御 → 实时监控 → 应急响应 → 事后审计
```

## 为什么学习安全知识

### 对开发者

- **避免写出不安全的代码**：理解常见漏洞的原理，从源头杜绝
- **设计更安全的架构**：知道在哪里加锁、加什么锁
- **快速定位安全问题**：能够识别日志中的异常行为

### 对面试

- **安全类岗位**：密码学、Web 安全、系统安全是必考内容
- **后端/全栈岗位**：应用层安全（SQL 注入、XSS）是高频考点
- **DevOps 岗位**：安全运维、容器安全、CI/CD 安全越来越受重视

### 对运维

- **理解安全机制**：知道 SELinux、iptables 背后的原理
- **配置安全策略**：正确配置防火墙、最小权限、密钥管理
- **应急响应能力**：能够识别和处理安全事件

## 学习路径

```
第一阶段：密码学基础
├── 对称加密 vs 非对称加密
├── 哈希与消息认证
├── 数字签名与证书
└── 密钥交换协议

第二阶段：身份认证与授权
├── 密码认证与多因素认证
├── OAuth2 与 OIDC
├── SSO 单点登录
└── RBAC 权限模型

第三阶段：应用层安全
├── Web 漏洞原理（XSS、SQL 注入、CSRF）
├── 服务端请求伪造（SSRF）
├── 文件上传与代码注入
└── 反序列化漏洞

第四阶段：系统层安全
├── 缓冲区溢出与 ROP
├── 权限提升与后门
├── 容器安全与沙箱
└── 强制访问控制（SELinux）

第五阶段：安全运维
├── 安全监控与日志分析
├── 漏洞扫描与渗透测试
├── 应急响应流程
└── DevSecOps 实践
```

## 核心安全原则

无论学习哪个领域，这几条原则都适用：

| 原则 | 说明 |
|---|---|
| 最小权限 | 只授予完成工作所需的最小权限 |
| 纵深防御 | 不要依赖单一安全措施，多层防护 |
| 默认安全 | 系统默认配置应该是安全的 |
| 失败安全 | 出错时的默认行为应该是拒绝访问 |
| 职责分离 | 关键操作需要多人或多步骤确认 |
