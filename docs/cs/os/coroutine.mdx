# 协程与线程对比

> 目标级别：P6

面试官问：「协程和线程有什么区别？」你回答「协程更轻量」——然后面试官追问：「为什么协程比线程轻量？」「协程调度和线程调度有什么区别？」「Go 和 Java 的协程实现有什么不同？」

协程是现代高并发编程的重要技术，理解协程对于设计高性能系统至关重要。

## 一、协程基础

### 1.1 什么是协程

```
协程（Coroutine）：
- 用户态的轻量级线程
- 由应用程序调度，不是操作系统
- 切换开销极小（只有寄存器）
- 主动让出（yield）而非抢占
```

### 1.2 线程 vs 协程

| 维度 | 线程 | 协程 |
|------|------|------|
| 调度者 | 操作系统 | 应用程序 |
| 切换开销 | 大（内核切换） | 极小（用户态切换） |
| 栈大小 | 1MB | 几 KB |
| 创建速度 | 慢 | 快 |
| 阻塞影响 | 阻塞线程 | 让出执行权 |
| 数量限制 | 几千 | 几十万 |

---

## 二、协程调度

### 2.1 调度方式

```
协程调度：

1. 协作式调度
   - 协程主动让出 CPU
   - 需要显式 yield
   - 简单，但可能被恶意协程阻塞

2. 抢占式调度
   - 定时器中断强制切换
   - 更公平，不会被单个协程阻塞
   - 如 Go 的 GMP 调度器
```

---

## 三、Go 协程

### 3.1 GMP 模型

```
Go GMP 调度器：

G（Goroutine）：协程
M（Machine）：线程
P（Processor）：调度器上下文

调度流程：
1. P 从本地队列获取 G
2. M 绑定 P 执行 G
3. G 阻塞时，M 去找新的 P
4. G 完成后，回到 P 的队列
```

### 3.2 Java 协程

```
Java 协程（Loom 项目）：

1. 虚拟线程（Virtual Thread）
   - JVM 层面的轻量级线程
   - 由 JVM 调度

2. continuation
   - 暂停和恢复执行
   - 底层机制
```

## 四、面试题精讲

### 🔴 【高频】协程和线程的区别

**问题**：协程和线程有什么区别？

**标准答案**：

```
1. 调度方式
   - 线程：操作系统调度（抢占式）
   - 协程：应用程序调度（通常协作式）

2. 开销
   - 线程：需要内核态切换，栈 1MB
   - 协程：用户态切换，栈几 KB

3. 数量
   - 线程：几千个
   - 协程：可以几十万

4. 阻塞影响
   - 线程阻塞：线程阻塞
   - 协程阻塞：让出，调度器切到其他协程

5. 场景
   - 线程：CPU 密集型
   - 协程：I/O 密集型（高并发）
```

### 🟡 【中频】Go GMP 模型

**问题**：Go 的 GMP 调度模型是怎么工作的？

**标准答案**：

```
GMP 调度模型：

G（Goroutine）：Go 协程
M（Machine）：系统线程
P（Processor）：调度器上下文

调度流程：
1. P 有本地协程队列
2. M 从 P 获取 G 执行
3. G 阻塞时，M 去找新 P
4. 负载均衡：从其他 P 偷协程

特点：
- 抢占式调度
- 工作窃取（Work Stealing）
- 绑定 P 的 M 数量可控
```

## 五、对比总结

### 线程 vs 协程

| 维度 | 线程 | 协程 |
|------|------|------|
| 栈大小 | 1MB | 几 KB |
| 创建 | 慢 | 快 |
| 调度 | OS（抢占式） | 应用（协作/抢占） |
| 切换开销 | 1000+ CPU 周期 | 几十 CPU 周期 |
| 并发数 | 几千 | 几十万 |
| 阻塞影响 | 线程阻塞 | 让出执行权 |

## 六、扩展思考

### 💡 协程适用场景

```
适合协程的场景：
1. 高并发 I/O（Web 服务器、爬虫）
2. 异步 I/O 封装（网络请求）
3. 游戏逻辑
4. 事件驱动编程

不适合协程的场景：
1. CPU 密集型计算
2. 需要真正并行（多核）
3. 低延迟要求（切换有开销）
```

> 协程是处理高并发的一种有效方式。Go 语言从设计之初就支持协程（goroutine），Java 也在 Loom 项目中引入了虚拟线程。理解协程的原理有助于选择合适的并发方案。
