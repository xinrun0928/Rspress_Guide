## 一、算法背景

### 1.1 命名来源

银行家算法的命名来源于银行放贷业务：

```
银行放贷 analogy：

银行：
- 有固定资金
- 需要确保放贷后能收回

操作系统：
- 有有限资源
- 需要确保分配后进程能完成

银行家算法的核心思想：
- 银行放贷前，检查放贷后是否有足够资金让客户还款
- 操作系统分配资源前，检查分配后是否仍能保证所有进程完成
```

### 1.2 基本思想

```
银行家算法的核心思想：

1. 在资源分配前，检查分配后系统是否仍处于安全状态
2. 如果是安全状态，分配资源
3. 如果不是安全状态，拒绝分配

安全状态：存在一个资源分配序列，所有进程都能完成
```

## 二、核心数据结构

### 2.1 数据结构定义

```java
public class BankerAlgorithm {
    // 可用资源向量：每种资源的可用数量
    int[] available;  // 例如 [3, 3, 2]

    // 最大需求矩阵：每个进程对每种资源的最大需求
    int[][] maximum;  // 例如 max[0][1] = 7 表示 P0 对 R1 的最大需求

    // 分配矩阵：每个进程已分配的资源
    int[][] allocation;  // 例如 alloc[0][1] = 3 表示 P0 已分配 R1 的数量

    // 需求矩阵：每个进程还需要多少资源
    int[][] need;  // need[i][j] = maximum[i][j] - allocation[i][j]

    // 进程数
    int processCount;

    // 资源类型数
    int resourceCount;
}
```

### 2.2 矩阵关系

```
核心关系：

need[i][j] = maximum[i][j] - allocation[i][j]

当前资源状态 = 已分配资源 + 可用资源
available + Σ(allocation[i]) = 总资源
```

## 三、算法步骤

### 3.1 安全性检测算法

```java
/**
 * 安全性检测算法
 * @return 是否存在安全序列
 */
public boolean safetyCheck() {
    // work = available：复制可用资源
    int[] work = available.clone();

    // finish[i] = true 表示进程 Pi 可以完成
    boolean[] finish = new boolean[processCount];

    // 初始化所有进程为未完成
    for (int i = 0; i < processCount; i++) {
        finish[i] = false;
    }

    // 尝试找到可以完成的进程
    int count = 0;
    while (count < processCount) {
        boolean found = false;

        // 查找满足条件的进程
        for (int i = 0; i < processCount; i++) {
            // 条件1：进程尚未完成
            // 条件2：进程的需求 <= 当前可用资源
            if (!finish[i] && canSatisfyNeed(i, work)) {
                // 假设进程 Pi 完成，释放其资源
                work = addVectors(work, allocation[i]);
                finish[i] = true;
                found = true;
                count++;
            }
        }

        // 如果一轮查找没有找到可以完成的进程
        if (!found) {
            break;
        }
    }

    // 所有进程都能完成 = 安全状态
    for (int i = 0; i < processCount; i++) {
        if (!finish[i]) {
            return false;  // 不安全状态
        }
    }
    return true;  // 安全状态
}

private boolean canSatisfyNeed(int i, int[] work) {
    for (int j = 0; j < resourceCount; j++) {
        if (need[i][j] > work[j]) {
            return false;
        }
    }
    return true;
}

private int[] addVectors(int[] a, int[] b) {
    int[] result = new int[a.length];
    for (int i = 0; i < a.length; i++) {
        result[i] = a[i] + b[i];
    }
    return result;
}
```

### 3.2 资源请求算法

```java
/**
 * 进程 Pi 请求资源
 * @param processId 进程 ID
 * @param request 请求向量
 * @return 是否分配成功
 */
public boolean request(int processId, int[] request) {
    // 步骤1：检查请求是否超过需求
    for (int j = 0; j < resourceCount; j++) {
        if (request[j] > need[processId][j]) {
            // 请求超过最大需求，错误
            return false;
        }
    }

    // 步骤2：检查请求是否超过可用资源
    for (int j = 0; j < resourceCount; j++) {
        if (request[j] > available[j]) {
            // 可用资源不足，进程等待
            return false;
        }
    }

    // 步骤3：尝试分配资源
    for (int j = 0; j < resourceCount; j++) {
        available[j] -= request[j];
        allocation[processId][j] += request[j];
        need[processId][j] -= request[j];
    }

    // 步骤4：安全性检测
    if (safetyCheck()) {
        // 安全状态，分配成功
        return true;
    } else {
        // 不安全状态，恢复状态，拒绝分配
        for (int j = 0; j < resourceCount; j++) {
            available[j] += request[j];
            allocation[processId][j] -= request[j];
            need[processId][j] += request[j];
        }
        return false;
    }
}
```

### 4.1 初始状态

```
系统状态：

资源总量：
R1 = 10, R2 = 5, R3 = 7

进程最大需求（Maximum）：
        R1  R2  R3
P0      7   5   3
P1      3   2   2
P2      9   0   2

已分配（Allocation）：
        R1  R2  R3
P0      0   1   0
P1      2   0   0
P2      3   0   2

当前可用（Available）：
V = [10-5, 5-1, 7-2] = [5, 4, 5]

需求（Need = Maximum - Allocation）：
        R1  R2  R3
P0      7   4   3
P1      1   2   2
P2      6   0   0
```

### 4.2 安全性检测过程

```
安全性检测过程：

Step 1：
- 检查 P0：Need = [7,4,3] > Available = [5,4,5]？是，无法满足
- 检查 P1：Need = [1,2,2] <= Available = [5,4,5]？是，可以满足
- P1 可以完成，释放资源
  Available = [5,4,5] + [2,0,0] = [7,4,5]
  P1 finish = true

Step 2：
- 检查 P0：Need = [7,4,3] <= Available = [7,4,5]？是，可以满足
- P0 可以完成，释放资源
  Available = [7,4,5] + [0,1,0] = [7,5,5]
  P0 finish = true

Step 3：
- 检查 P2：Need = [6,0,0] <= Available = [7,5,5]？是，可以满足
- P2 可以完成，释放资源
  Available = [7,5,5] + [3,0,2] = [10,5,7]
  P2 finish = true

安全序列：P1 → P0 → P2（存在安全序列）
结论：当前状态是安全状态
```

## 五、完整代码实现

### 5.1 完整类实现

```java
import java.util.*;

public class BankerAlgorithm {
    private int[] available;      // 可用资源向量
    private int[][] maximum;      // 最大需求矩阵
    private int[][] allocation;   // 分配矩阵
    private int[][] need;         // 需求矩阵
    private int processCount;      // 进程数
    private int resourceCount;      // 资源类型数

    public BankerAlgorithm(int[] totalResources, int[][] maxDemand, int[][] allocated) {
        this.resourceCount = totalResources.length;
        this.processCount = maxDemand.length;
        this.maximum = maxDemand;
        this.allocation = allocated;

        // 初始化可用资源 = 总资源 - 已分配资源
        this.available = new int[resourceCount];
        for (int i = 0; i < resourceCount; i++) {
            int used = 0;
            for (int j = 0; j < processCount; j++) {
                used += allocated[j][i];
            }
            this.available[i] = totalResources[i] - used;
        }

        // 初始化需求矩阵
        this.need = new int[processCount][resourceCount];
        for (int i = 0; i < processCount; i++) {
            for (int j = 0; j < resourceCount; j++) {
                this.need[i][j] = maximum[i][j] - allocation[i][j];
            }
        }
    }

    /**
     * 资源请求
     */
    public synchronized boolean requestResource(int processId, int[] request) {
        System.out.println("P" + processId + " 请求: " + Arrays.toString(request));

        // 步骤1：检查请求是否超过需求
        for (int j = 0; j < resourceCount; j++) {
            if (request[j] > need[processId][j]) {
                System.out.println("P" + processId + " 请求超过最大需求，拒绝");
                return false;
            }
        }

        // 步骤2：检查请求是否超过可用资源
        for (int j = 0; j < resourceCount; j++) {
            if (request[j] > available[j]) {
                System.out.println("P" + processId + " 资源不足，等待");
                return false;
            }
        }

        // 步骤3：尝试分配
        for (int j = 0; j < resourceCount; j++) {
            available[j] -= request[j];
            allocation[processId][j] += request[j];
            need[processId][j] -= request[j];
        }

        // 步骤4：安全性检测
        if (safetyCheck()) {
            System.out.println("P" + processId + " 分配成功");
            return true;
        } else {
            // 恢复状态
            for (int j = 0; j < resourceCount; j++) {
                available[j] += request[j];
                allocation[processId][j] -= request[j];
                need[processId][j] += request[j];
            }
            System.out.println("P" + processId + " 分配后不安全，拒绝");
            return false;
        }
    }

    /**
     * 进程释放资源
     */
    public synchronized void releaseResource(int processId, int[] release) {
        System.out.println("P" + processId + " 释放: " + Arrays.toString(release));
        for (int j = 0; j < resourceCount; j++) {
            allocation[processId][j] -= release[j];
            need[processId][j] += release[j];
            available[j] += release[j];
        }
    }

    /**
     * 安全性检测
     */
    private boolean safetyCheck() {
        int[] work = available.clone();
        boolean[] finish = new boolean[processCount];
        Arrays.fill(finish, false);

        // 记录安全序列
        List<Integer> safeSequence = new ArrayList<>();

        int count = 0;
        while (count < processCount) {
            boolean found = false;

            for (int i = 0; i < processCount; i++) {
                if (!finish[i] && canSatisfyNeed(i, work)) {
                    // P[i] 可以完成
                    for (int j = 0; j < resourceCount; j++) {
                        work[j] += allocation[i][j];
                    }
                    finish[i] = true;
                    safeSequence.add(i);
                    found = true;
                    count++;
                }
            }

            if (!found) {
                break;
            }
        }

        boolean isSafe = true;
        for (boolean f : finish) {
            if (!f) {
                isSafe = false;
                break;
            }
        }

        if (isSafe) {
            System.out.println("安全序列: " + safeSequence);
        } else {
            System.out.println("不安全状态！");
        }

        return isSafe;
    }

    private boolean canSatisfyNeed(int i, int[] work) {
        for (int j = 0; j < resourceCount; j++) {
            if (need[i][j] > work[j]) {
                return false;
            }
        }
        return true;
    }

    public void printState() {
        System.out.println("=== 系统状态 ===");
        System.out.println("Available: " + Arrays.toString(available));
        System.out.println("\nMaximum:\n" + matrixToString(maximum));
        System.out.println("\nAllocation:\n" + matrixToString(allocation));
        System.out.println("\nNeed:\n" + matrixToString(need));
    }

    private String matrixToString(int[][] matrix) {
        StringBuilder sb = new StringBuilder();
        for (int[] row : matrix) {
            sb.append(Arrays.toString(row)).append("\n");
        }
        return sb.toString();
    }

    public static void main(String[] args) {
        // 资源总量
        int[] total = {10, 5, 7};

        // 最大需求
        int[][] max = {
            {7, 5, 3},
            {3, 2, 2},
            {9, 0, 2}
        };

        // 已分配
        int[][] alloc = {
            {0, 1, 0},
            {2, 0, 0},
            {3, 0, 2}
        };

        BankerAlgorithm banker = new BankerAlgorithm(total, max, alloc);
        banker.printState();
        banker.safetyCheck();

        // P1 请求 [1, 0, 2]
        banker.requestResource(1, new int[]{1, 0, 2});

        // P0 请求 [1, 0, 1]
        banker.requestResource(0, new int[]{1, 0, 1});
    }
}

## 六、算法局限性

### 6.1 局限性分析

```
银行家算法的局限性：

1. 需要预先知道最大需求
   - 实际系统中，进程可能不知道需要多少资源
   - 动态变化的资源需求无法处理

2. 进程数量必须固定
   - 无法处理动态创建/销毁进程

3. 资源类型必须固定
   - 无法处理动态添加资源类型

4. 算法开销大
   - 每次请求都要进行安全性检测
   - O(m*n) 时间复杂度（m 资源类型，n 进程数）

5. 不适用于实时系统
   - 银行家算法可能导致无限等待
   - 实时系统需要确定性保证
```

### 6.2 实际应用

```
银行家算法实际应用较少的原因：

1. 资源类型动态变化
2. 进程行为不可预测
3. 最大需求难以预先确定
4. 开销与实际收益不成正比

实际系统更多使用：
- 死锁预防（如按序分配）
- 死锁检测 + 恢复
- 资源分配图检测（单资源类型）
```

## 七、面试题精讲

### 🔴 【高频】银行家算法原理

**问题**：请描述银行家算法的工作原理。

**标准答案**：

```
银行家算法的核心思想：在资源分配前，检查分配后系统是否处于安全状态。

算法步骤：

1. 初始化
   - Available = 总资源 - 已分配资源
   - Need = Maximum - Allocation

2. 安全性检测
   - 找 need <= work 的进程 Pi
   - 假设 Pi 完成，work += allocation[i]
   - 重复直到所有进程都能完成

3. 资源请求（进程 Pi 请求 request）
   - 检查 request <= need
   - 检查 request <= available
   - 尝试分配
   - 安全性检测
   - 安全则分配，不安全则拒绝

安全状态：存在一个序列使所有进程都能完成
不安全状态：不保证所有进程能完成（不一定死锁）
```

### 🟡 【中频】安全状态与不安全状态

**问题**：什么是安全状态？银行家算法如何判断安全状态？

**标准答案**：

```
安全状态：
- 存在一个资源分配序列
- 按照这个序列，所有进程都能顺利完成
- 系统处于安全状态

不安全状态：
- 不存在这样的序列
- 不一定真的死锁，只是不能保证所有进程都能完成
- 如果后续分配不当，可能导致死锁

银行家算法判断安全状态：
1. work = available
2. 找 need <= work 的进程 Pi
3. Pi 完成，work += allocation[i]
4. 重复直到所有进程都能完成

如果存在安全序列，系统处于安全状态
```

### 🟢 【低频】银行家算法局限性

**问题**：银行家算法有什么局限性？为什么实际系统很少使用？

**标准答案**：

```
局限性：

1. 需要预先知道最大需求
   - 实际系统中难以确定

2. 进程数量必须固定
   - 无法处理动态进程

3. 资源类型必须固定
   - 无法处理动态资源

4. 算法开销大
   - 每次请求都要安全性检测
   - O(m*n) 时间复杂度

5. 实时性差
   - 可能导致进程无限等待

实际应用：学校教学为主，生产系统较少使用
```

## 九、扩展思考

### 💡 加分话题：资源分配图检测

```
单资源类型的死锁检测：

资源分配图：
- 进程节点：P1, P2
- 资源节点：R1（圆圈），数字表示实例数
- 边：申请边、分配边

检测方法：
- 简化资源分配图
- 如果能简化所有进程节点，不死锁
- 如果有剩余进程节点，死锁

简化规则：
- 找一个已分配的进程，移除其所有边
- 重复直到无法简化
```

### 💡 加分话题：哲学家就餐问题的银行家算法分析

```
哲学家就餐问题用银行家算法分析：

资源：筷子（每种 1 个）

最大需求（拿筷子）：
- 每个哲学家最多需要 2 根

分配（已拿筷子）：
- 每个哲学家最多拿 1 根
