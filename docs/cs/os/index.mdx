# 操作系统

凌晨 3 点，你的服务器突然 CPU 100%，请求全部超时。你登录服务器，`top` 命令一敲，发现一个进程占用了所有资源。你想杀掉它，但 `kill` 命令没有反应。这时候你会怎么办？

这几乎是每个后端工程师都会遇到的场景。表面上是「进程占满 CPU」，背后却是进程调度、内存管理、I/O 阻塞等一系列操作系统机制的综合体现。

操作系统是连接硬件与软件的桥梁，是每一个程序员必须掌握的基础知识。无论你是写 Java 后端、C++ 游戏引擎，还是 Python 机器学习，都逃不开和操作系统打交道。

## 知识体系总览

本章节将围绕以下几个核心模块展开：

**进程与线程** —— 程序运行起来就变成了进程，而线程是进程内的执行单元。理解进程与线程的区别、进程状态转换、调度算法，是理解并发的基础。还会深入讲解进程间通信（IPC）和线程同步机制，以及令人头疼的死锁问题。

**内存管理** —— 为什么 32 位系统最多只能使用 4GB 内存？为什么程序访问的地址是虚拟地址而不是物理地址？分页、分段、页面置换算法、局部性原理、虚拟内存……这些概念将帮助你理解操作系统如何高效管理宝贵的内存资源。

**文件系统** —— 从点击一个文件到文件内容呈现在屏幕上，操作系统经历了什么？文件如何存储、如何分配空间、目录如何实现、磁盘调度算法有哪些？文件系统章节将为你揭开这些问题的答案。

**操作系统原理** —— 内核态和用户态有什么区别？系统调用是如何实现的？中断和异常处理机制是什么？设备管理如何工作？这些底层原理将帮助你建立对操作系统的完整认知。

**Linux** —— 作为最流行的服务器操作系统，Linux 的进程管理、内存管理、文件系统、网络协议栈都有其独特设计。掌握 Linux 的常用命令和性能分析工具，是每个工程师的必备技能。

## 学习路线建议

如果你对操作系统完全陌生，建议按以下顺序学习：

```
进程与线程（最直观，容易理解）
    ↓
内存管理（需要一些抽象思维）
    ↓
文件系统（与实际应用联系紧密）
    ↓
操作系统原理（理论升华）
    ↓
Linux（实践验证）
```

如果你已经有一定基础，可以直接跳到自己薄弱的模块深入学习。

## 面试关联

操作系统是后端面试的重灾区，尤其是进程线程、内存管理、文件系统三个模块。常见的面试问题包括：

- 进程和线程的区别是什么？
- 进程间有哪些通信方式？
- 什么是死锁？死锁的必要条件是什么？
- 页面置换算法有哪些？LRU 是怎么实现的？
- 虚拟内存是什么？有什么作用？
- 什么是缓冲区溢出？如何防护？

这些问题看似基础，但能回答出「为什么」的人却不多。本章节不仅讲解「是什么」，更注重讲解「为什么」，帮助你真正理解操作系统的设计思想。
