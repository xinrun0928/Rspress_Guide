# 进程与线程

想象一下：你打开了一个浏览器，播放着音乐，同时下载着文件。这三个任务看起来是「同时」运行的——但你的 CPU 可能只有 4 核。

这就是操作系统最核心的能力：**让有限的 CPU 资源看起来像是无限的多任务并行**。而实现这个能力的关键，正是进程与线程。

本章节将带你深入理解进程与线程的区别、进程状态转换、调度算法、进程间通信、线程同步，以及令人头疼的死锁问题。

## 核心内容

**进程、线程、协程的区别** —— 三者看似相似，实则有着本质的区别。进程是资源分配的基本单位，线程是 CPU 调度的基本单位，而协程是用户态的轻量级线程。你知道为什么 Java 线程比进程轻量吗？

**进程状态转换与 PCB** —— 进程从创建到退出，经历了哪些状态？PCB（进程控制块）是进程的「档案」，操作系统如何管理这些档案？

**进程调度算法** —— 当多个进程同时就绪，谁先执行？FCFS、SJF、RR、优先级调度……每种算法都有其适用场景，没有最优，只有最适合。

**多级反馈队列调度** —— 这是现代操作系统广泛采用的调度算法，既能保证响应时间，又能提高系统吞吐量。它的核心思想是什么？

**进程间通信（IPC）** —— 管道、消息队列、共享内存、信号量、Socket……五种主流 IPC 方式各有特点，如何选择？

**线程同步机制** —— 互斥锁、读写锁、自旋锁、条件变量……同步是为了解决什么问题？用错了会怎样？

**进程同步经典问题** —— 生产者-消费者、哲学家就餐、读者-写者……这些问题看似简单，却蕴含着并发控制的核心思想。

**死锁** —— 死锁是如何产生的？四个必要条件是什么？如何预防、避免、检测和解除死锁？
