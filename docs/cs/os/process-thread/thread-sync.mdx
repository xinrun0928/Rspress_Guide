# 线程同步机制

多线程程序最怕什么？**数据竞争**。

两个线程同时修改同一个变量，结果不可预测；一个线程在读数据，另一个线程在写数据，读到的可能是半成品。这些问题，用「锁」来解决。

但锁的种类那么多：互斥锁、自旋锁、读写锁、条件变量……什么时候用什么锁？用错了会怎样？

## 为什么需要同步？

看一个经典的计数器问题：

```java
public class Counter {
    private long count = 0;

    public void increment() {
        count++;  // 这个操作不是原子的！
    }
}
```

`count++` 在 Java 中对应三条字节码：

```java
// count++
0: aload_0
1: dup
2: getfield      #2  // 获取 count 值
5: iconst_1
6: iadd          // 加 1
7: putfield      #2  // 写回 count
```

如果两个线程同时执行：

```
线程 A: getfield count(0) → add 1 → putfield count(1)
线程 B: getfield count(0) → add 1 → putfield count(1)
// 结果：count = 1，期望是 2！
```

这就是数据竞争（Data Race）。**同步机制的目的，就是让对共享数据的访问串行化。**

## 互斥锁（Mutex）

互斥锁是最常用的同步机制。一次只有一个线程能持有互斥锁，其他线程必须等待。

```c
#include <pthread.h>

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;

void *worker(void *arg) {
    pthread_mutex_lock(&mutex);   // 获取锁
    // 访问共享资源
    shared_counter++;
    pthread_mutex_unlock(&mutex); // 释放锁
    return NULL;
}
```

### Java 中的互斥锁

```java
public class Counter {
    private final Lock lock = new ReentrantLock();
    private long count = 0;

    public void increment() {
        lock.lock();
        try {
            count++;
        } finally {
            lock.unlock();
        }
    }
}
```

### 互斥锁的特点

| 特点 | 说明 |
|---|---|
| 排他性 | 同一时刻只能有一个线程持有 |
| 阻塞 | 等待锁的线程会进入阻塞态，释放 CPU |
| 可重入 | 同一线程可多次获取同一把锁 |

## 自旋锁（Spin Lock）

自旋锁不会让线程阻塞，而是不停地「自旋」检查锁是否可用：

```c
#include <pthread.h>

// 简单的自旋锁实现
void spin_lock(int *lock) {
    while (__sync_lock_test_and_set(lock, 1)) {
        // 忙等待
    }
}

void spin_unlock(int *lock) {
    __sync_lock_release(lock, 0);
}
```

### 自旋锁 vs 互斥锁

| 对比 | 自旋锁 | 互斥锁 |
|---|---|---|
| 等待方式 | 忙等待（占用 CPU） | 阻塞（不占用 CPU） |
| 适用场景 | 临界区极短 | 临界区较长或不确定 |
| 切换开销 | 无上下文切换 | 有上下文切换 |
| 可能死锁 | 可能（持锁线程被挂起） | 不会 |

### 选择原则

- 临界区执行时间 **< 上下文切换时间** → 自旋锁
- 临界区执行时间 **> 上下文切换时间** → 互斥锁
- 临界区包含 **阻塞操作** → 互斥锁

## 读写锁（Read-Write Lock）

读操作可以并发，写操作必须独占。

```c
#include <pthread.h>

pthread_rwlock_t rwlock = PTHREAD_RWLOCK_INITIALIZER;

// 读操作
void read_data() {
    pthread_rwlock_rdlock(&rwlock);
    // 多个线程可以同时读
    printf("reading: %d\n", shared_value);
    pthread_rwlock_unlock(&rwlock);
}

// 写操作
void write_data(int value) {
    pthread_rwlock_wrlock(&rwlock);
    // 同时只能有一个线程写
    shared_value = value;
    pthread_rwlock_unlock(&rwlock);
}
```

### 读写锁的效率分析

在读多写少的场景下，读写锁比互斥锁效率高很多：

```bash
# 假设：读操作 1000 次，写操作 1 次
# 互斥锁：1001 次串行执行
# 读写锁：1 次写串行 + 1000 次读并发 = 大幅提升
```

### Java 中的读写锁

```java
import java.util.concurrent.locks.ReadWriteLock;
import java.util.concurrent.locks.ReentrantReadWriteLock;

public class Cache {
    private final ReadWriteLock rwlock = new ReentrantReadWriteLock();
    private String data = "";

    public String read() {
        rwlock.readLock().lock();
        try {
            return data;
        } finally {
            rwlock.readLock().unlock();
        }
    }

    public void write(String value) {
        rwlock.writeLock().lock();
        try {
            data = value;
        } finally {
            rwlock.writeLock().unlock();
        }
    }
}
```

## 条件变量（Condition Variable）

条件变量用于线程间的**等待通知**机制。当某个条件不满足时，线程主动等待；条件满足时，被唤醒继续执行。

```c
#include <pthread.h>

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t cond = PTHREAD_COND_INITIALIZER;
int ready = 0;

// 等待方
void *consumer(void *arg) {
    pthread_mutex_lock(&mutex);
    while (ready == 0) {
        pthread_cond_wait(&cond, &mutex);  // 解锁并等待
    }
    // 醒来时重新持有锁
    printf("consumed\n");
    ready = 0;
    pthread_mutex_unlock(&mutex);
    return NULL;
}

// 通知方
void *producer(void *arg) {
    pthread_mutex_lock(&mutex);
    ready = 1;
    pthread_cond_signal(&cond);  // 唤醒一个等待线程
    pthread_mutex_unlock(&mutex);
    return NULL;
}
```

### 为什么需要 while 而非 if？

```c
// 错误写法
if (ready == 0) {
    pthread_cond_wait(&cond, &mutex);  // 虚假唤醒！
}
// 醒来后直接继续，但 ready 可能是 0

// 正确写法
while (ready == 0) {
    pthread_cond_wait(&cond, &mutex);  // 醒来后再次检查
}
```

**虚假唤醒（Spurious Wakeup）**：pthread 的条件变量可能在没有调用 `signal`/`broadcast` 的情况下被唤醒，所以必须用 while 循环重新检查条件。

### Java 中的条件变量

```java
import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.ReentrantLock;

public class Buffer {
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();
    private final String[] buffer = new String[100];
    private int count = 0;

    public void put(String item) throws InterruptedException {
        lock.lock();
        try {
            while (count == buffer.length) {
                notFull.await();  // 等待有空间
            }
            buffer[count++] = item;
            notEmpty.signal();  // 通知有数据
        } finally {
            lock.unlock();
        }
    }

    public String take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();  // 等待有数据
            }
            String item = buffer[--count];
            notFull.signal();  // 通知有空间
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

## 生产者-消费者问题

理解了条件变量，我们来实现一个生产者-消费者问题：

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

#define BUFFER_SIZE 5
int buffer[BUFFER_SIZE];
int count = 0;

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t not_empty = PTHREAD_COND_INITIALIZER;
pthread_cond_t not_full = PTHREAD_COND_INITIALIZER;

void *producer(void *arg) {
    for (int i = 0; i < 20; i++) {
        pthread_mutex_lock(&mutex);
        while (count == BUFFER_SIZE) {
            pthread_cond_wait(&not_full, &mutex);
        }
        buffer[count++] = i;
        printf("produced: %d\n", i);
        pthread_cond_signal(&not_empty);
        pthread_mutex_unlock(&mutex);
        usleep(100000);  // 模拟生产时间
    }
    return NULL;
}

void *consumer(void *arg) {
    for (int i = 0; i < 20; i++) {
        pthread_mutex_lock(&mutex);
        while (count == 0) {
            pthread_cond_wait(&not_empty, &mutex);
        }
        int item = buffer[--count];
        printf("consumed: %d\n", item);
        pthread_cond_signal(&not_full);
        pthread_mutex_unlock(&mutex);
        usleep(150000);  // 模拟消费时间（比生产慢）
    }
    return NULL;
}

int main() {
    pthread_t p, c;
    pthread_create(&p, NULL, producer, NULL);
    pthread_create(&c, NULL, consumer, NULL);
    pthread_join(p, NULL);
    pthread_join(c, NULL);
    return 0;
}
```

## 面试高频问题

**Q1：为什么 wait() 要在 while 循环中调用？**

防止虚假唤醒。pthread 的条件变量规范允许在没有调用 signal 的情况下被唤醒，所以必须重新检查条件。

**Q2：synchronized 和 ReentrantLock 的区别？**

| 对比 | synchronized | ReentrantLock |
|---|---|---|
| 语法 | 关键字 | API |
| 获取锁 | 自动释放 | 必须手动释放 |
| 公平锁 | 否 | 可选（fair 参数） |
| 条件变量 | 无 | 可创建多个 |
| 可中断等待 | 否 | 是（tryLock） |

**Q3：什么是死锁？和线程同步有什么关系？**

死锁是指两个或多个线程相互等待对方持有的锁，导致都无法继续执行。线程同步可以解决数据竞争问题，但用错同步方式（如嵌套锁）可能导致死锁。

**Q4：读写锁的写锁饥饿问题如何解决？**

可以采用「读写优先」或「写优先」策略。写优先意味着新来的写请求会阻止后续的读请求，防止写锁长时间饥饿。
