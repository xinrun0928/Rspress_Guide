# 进程同步与经典问题

理论学多了容易枯燥，但这些经典问题，是理解同步机制最好的练兵场。

哲学家就餐问题曾经让无数天才头疼；读者-写者问题在数据库系统中无处不在；生产者-消费者问题是所有队列系统的雏形。理解了这些问题，你再看任何并发系统，都会有一种「一览众山小」的感觉。

## 生产者-消费者问题

### 问题描述

```
┌─────────┐     ┌─────────────┐     ┌─────────┐
│ 生产者  │────►│   缓冲区    │────►│ 消费者  │
│         │     │ (有界队列)  │     │         │
└─────────┘     └─────────────┘     └─────────┘
```

两个进程（或线程）共享一个固定大小的缓冲区：
- **生产者**生成数据，放入缓冲区
- **消费者**从缓冲区取出数据，消费掉

问题在于：缓冲区满时，生产者必须等待；缓冲区空时，消费者必须等待。

### 问题分析

我们需要解决三个问题：
1. **互斥**：同一时刻只能有一个进程访问缓冲区
2. **同步**：缓冲区满/空时的等待-通知
3. **避免死锁**：确保不会所有进程都在等待

### C 语言实现

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

#define BUFFER_SIZE 5
#define ITEM_COUNT 20

int buffer[BUFFER_SIZE];
int in = 0;   // 下一个写入位置
int out = 0;  // 下一个读取位置
int count = 0;  // 缓冲区中的数据数量

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t not_full = PTHREAD_COND_INITIALIZER;
pthread_cond_t not_empty = PTHREAD_COND_INITIALIZER;

void *producer(void *arg) {
    for (int i = 0; i < ITEM_COUNT; i++) {
        pthread_mutex_lock(&mutex);

        // 缓冲区满，等待消费者
        while (count == BUFFER_SIZE) {
            pthread_cond_wait(&not_full, &mutex);
        }

        // 生产数据
        buffer[in] = i * 10;
        printf("P: produced %d at %d\n", buffer[in], in);
        in = (in + 1) % BUFFER_SIZE;
        count++;

        // 通知消费者
        pthread_cond_signal(&not_empty);

        pthread_mutex_unlock(&mutex);
        usleep(50000);  // 生产较慢
    }
    return NULL;
}

void *consumer(void *arg) {
    for (int i = 0; i < ITEM_COUNT; i++) {
        pthread_mutex_lock(&mutex);

        // 缓冲区空，等待生产者
        while (count == 0) {
            pthread_cond_wait(&not_empty, &mutex);
        }

        // 消费数据
        int item = buffer[out];
        printf("C: consumed %d from %d\n", item, out);
        out = (out + 1) % BUFFER_SIZE;
        count--;

        // 通知生产者
        pthread_cond_signal(&not_full);

        pthread_mutex_unlock(&mutex);
        usleep(80000);  // 消费较慢
    }
    return NULL;
}

int main() {
    pthread_t p, c;
    pthread_create(&p, NULL, producer, NULL);
    pthread_create(&c, NULL, consumer, NULL);
    pthread_join(p, NULL);
    pthread_join(c, NULL);
    return 0;
}
```

### Java 实现

```java
import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.ReentrantLock;

public class ProducerConsumer {
    private static final int BUFFER_SIZE = 5;
    private static final int[] buffer = new int[BUFFER_SIZE];
    private static int count = 0;
    private static int in = 0;
    private static int out = 0;

    private static final ReentrantLock lock = new ReentrantLock();
    private static final Condition notFull = lock.newCondition();
    private static final Condition notEmpty = lock.newCondition();

    static class Producer extends Thread {
        public void run() {
            for (int i = 0; i < 20; i++) {
                lock.lock();
                try {
                    while (count == BUFFER_SIZE) {
                        notFull.await();
                    }
                    buffer[in] = i * 10;
                    System.out.println("P: produced " + buffer[in] + " at " + in);
                    in = (in + 1) % BUFFER_SIZE;
                    count++;
                    notEmpty.signal();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
                try { sleep(50); } catch (InterruptedException e) {}
            }
        }
    }

    static class Consumer extends Thread {
        public void run() {
            for (int i = 0; i < 20; i++) {
                lock.lock();
                try {
                    while (count == 0) {
                        notEmpty.await();
                    }
                    int item = buffer[out];
                    System.out.println("C: consumed " + item + " from " + out);
                    out = (out + 1) % BUFFER_SIZE;
                    count--;
                    notFull.signal();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
                try { sleep(80); } catch (InterruptedException e) {}
            }
        }
    }

    public static void main(String[] args) {
        new Producer().start();
        new Consumer().start();
    }
}
```

## 哲学家就餐问题

### 问题描述

五个哲学家围坐在一张圆桌旁，桌上有五盘意大利面和五把叉子：

```
        P0
       /   \
      /     \
     P4     P1
      \     /
       \   /
        P3
         P2
```

哲学家交替进行「思考」和「吃饭」。吃饭需要同时拿起左右两边的叉子。

### 问题分析

如果每个哲学家都先拿起左边的叉子，再拿右边的叉子：

```
哲学家 0: 拿起叉子 0 → 等待叉子 1
哲学家 1: 拿起叉子 1 → 等待叉子 2
哲学家 2: 拿起叉子 2 → 等待叉子 3
哲学家 3: 拿起叉子 3 → 等待叉子 4
哲学家 4: 拿起叉子 4 → 等待叉子 0（被哲学家 0 拿着）
→ 死锁！
```

### 解决方案一：资源顺序化

为所有资源（叉子）分配固定顺序，按顺序获取：

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

#define N 5

pthread_mutex_t forks[N];
int philosopher_id[N] = {0, 1, 2, 3, 4};

// 获取叉子：总是先拿编号小的
void get_forks(int id) {
    int left = id;
    int right = (id + 1) % N;

    // 按顺序加锁，避免循环等待
    if (left < right) {
        pthread_mutex_lock(&forks[left]);
        pthread_mutex_lock(&forks[right]);
    } else {
        pthread_mutex_lock(&forks[right]);
        pthread_mutex_lock(&forks[left]);
    }
}

void put_forks(int id) {
    pthread_mutex_unlock(&forks[id]);
    pthread_mutex_unlock(&forks[(id + 1) % N]);
}

void *philosopher(void *arg) {
    int id = *(int *)arg;
    while (1) {
        printf("Philosopher %d is thinking\n", id);
        sleep(1);

        printf("Philosopher %d is hungry\n", id);
        get_forks(id);

        printf("Philosopher %d is eating\n", id);
        sleep(1);

        put_forks(id);
    }
}

int main() {
    pthread_t threads[N];
    for (int i = 0; i < N; i++) {
        pthread_mutex_init(&forks[i], NULL);
    }
    for (int i = 0; i < N; i++) {
        pthread_create(&threads[i], NULL, philosopher, &philosopher_id[i]);
    }
    for (int i = 0; i < N; i++) {
        pthread_join(threads[i], NULL);
    }
    return 0;
}
```

### 解决方案二：限制同时吃饭的人数

最多允许 N-1 个哲学家同时吃饭：

```c
pthread_mutex_t room_mutex = PTHREAD_MUTEX_INITIALIZER;

void *philosopher(void *arg) {
    int id = *(int *)arg;
    while (1) {
        printf("Philosopher %d is thinking\n", id);
        sleep(1);

        printf("Philosopher %d is hungry\n", id);

        // 进入房间（限制人数）
        pthread_mutex_lock(&room_mutex);

        get_forks(id);
        printf("Philosopher %d is eating\n", id);
        sleep(1);
        put_forks(id);

        pthread_mutex_unlock(&room_mutex);  // 离开房间
    }
}
```

### 解决方案三：奇偶策略

奇数号哲学家先拿左边，偶数号哲学家先拿右边：

```c
void get_forks(int id) {
    if (id % 2 == 0) {
        // 偶数：先左后右
        pthread_mutex_lock(&forks[id]);
        pthread_mutex_lock(&forks[(id + 1) % N]);
    } else {
        // 奇数：先右后左
        pthread_mutex_lock(&forks[(id + 1) % N]);
        pthread_mutex_lock(&forks[id]);
    }
}
```

## 读者-写者问题

### 问题描述

一个数据库被多个进程共享：
- **读者**：只读取数据，可以并发
- **写者**：读取并修改数据，必须独占

问题在于如何平衡读者和写者的优先级，防止写者饥饿或读者饥饿。

### 问题分析

三种典型的公平策略：

| 策略 | 特点 |
|---|---|
| 读者优先 | 读者到来时直接进入，读者可能让写者永远等待 |
| 写者优先 | 写者到来时阻止新读者进入，写者可能让读者饥饿 |
| 公平策略 | 先到先服务，谁先申请谁先执行 |

### 读者优先实现

```c
#include <pthread.h>
#include <stdio.h>

int read_count = 0;  // 当前读者数量
pthread_mutex_t read_mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_mutex_t write_mutex = PTHREAD_MUTEX_INITIALIZER;

void *reader(void *arg) {
    int id = *(int *)arg;

    pthread_mutex_lock(&read_mutex);
    read_count++;
    if (read_count == 1) {
        // 第一个读者获取写锁，阻止写者
        pthread_mutex_lock(&write_mutex);
    }
    pthread_mutex_unlock(&read_mutex);

    // 读取数据（多个读者可以同时执行）
    printf("Reader %d is reading\n", id);
    usleep(100000);

    pthread_mutex_lock(&read_mutex);
    read_count--;
    if (read_count == 0) {
        // 最后一个读者释放写锁，允许写者
        pthread_mutex_unlock(&write_mutex);
    }
    pthread_mutex_unlock(&read_mutex);

    return NULL;
}

void *writer(void *arg) {
    int id = *(int *)arg;

    pthread_mutex_lock(&write_mutex);

    // 写入数据（独占）
    printf("Writer %d is writing\n", id);
    usleep(200000);

    pthread_mutex_unlock(&write_mutex);

    return NULL;
}
```

## 经典问题总结

| 问题 | 核心矛盾 | 典型解决方案 |
|---|---|---|
| 生产者-消费者 | 缓冲区满/空 | 条件变量同步 |
| 哲学家就餐 | 资源竞争与死锁 | 资源顺序化、限制人数 |
| 读者-写者 | 并发读与独占写 | 读写锁、优先级控制 |

这些问题的核心思想，在实际的分布式系统、数据库、消息队列中无处不在。

## 面试高频问题

**Q1：哲学家就餐问题中，如何避免死锁？**

三种方案：
1. 资源顺序化：所有线程按固定顺序获取锁
2. 限制同时执行人数：最多 N-1 个人同时拿叉子
3. 改变拿叉子的顺序：奇数号先左后右，偶数号先右后左

**Q2：读者-写者问题的写者优先如何实现？**

让写者在有新读者到来时阻止其进入，写者完成后再放行新读者。这样可以防止写者长时间饥饿。

**Q3：生产者-消费者问题中，为什么用 while 而不是 if？**

防止虚假唤醒。条件变量的 wait 可能被意外唤醒，必须重新检查条件。

理解这些经典问题，不仅是为了面试，更是理解并发控制本质的最佳途径。
