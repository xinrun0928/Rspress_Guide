# 虚拟内存与物理内存映射

> 目标级别：P6

面试官问：「什么是虚拟内存？」你回答「程序使用的假内存，比物理内存大」——然后面试官追问：「虚拟地址怎么映射到物理地址？」「MMU 是什么？」「分页机制是怎么工作的？」

虚拟内存是操作系统最核心的机制之一，理解虚拟内存对于理解程序运行原理至关重要。

## 一、基本概念

### 1.1 虚拟地址 vs 物理地址

```
虚拟地址：
- 程序使用的地址
- 每个进程有独立的虚拟地址空间

物理地址：
- 实际内存的地址
- 所有进程共享

虚拟地址到物理地址的转换由 MMU 完成。
```

### 1.2 虚拟内存的作用

```
虚拟内存的作用：

1. 进程隔离
   - 每个进程有独立的虚拟地址空间
   - 进程之间不会相互影响

2. 内存保护
   - 防止非法内存访问
   - 只读区域不能写入

3. 内存抽象
   - 程序可以使用连续的虚拟地址
   - 不需要关心物理内存的实际布局

4. 内存扩展
   - 可以使用比物理内存更大的虚拟空间
   - 通过换页机制实现
```

## 二、分页机制

### 2.1 页和页框

```
页（Page）：
- 虚拟内存的基本单位
- 通常 4KB

页框（Page Frame）：
- 物理内存的基本单位
- 与页大小相同

虚拟页 → 物理页框的映射
```

### 2.2 页表

```c
// 简化的页表结构
struct page_table_entry {
    unsigned int valid : 1;      // 页是否有效
    unsigned int page_frame : 20; // 物理页框号
    unsigned int protection : 3; // 保护位
    unsigned int accessed : 1;   // 是否被访问
    unsigned int dirty : 1;      // 是否被修改
};
```

### 2.3 地址转换

```
虚拟地址结构（32 位系统，4KB 页）：
[ 页号（20位） | 页内偏移（12位） ]

物理地址结构：
[ 页框号（20位） | 页内偏移（12位） ]

转换过程：
1. 从虚拟地址提取页号
2. 查页表得到页框号
3. 用页框号和页内偏移组合物理地址
```

## 三、TLB（快表）

### 3.1 为什么需要 TLB

```
页表访问的瓶颈：

- 每次内存访问都需要查页表
- 页表在内存中，多一次内存访问
- 性能下降约一半

解决方案：TLB（Translation Lookaside Buffer）
- CPU 内部的高速缓存
- 缓存最近使用的页表项
- 命中率 95% 以上
```

### 3.2 TLB 工作原理

```
TLB 工作流程：

1. CPU 生成虚拟地址
2. MMU 先查 TLB
   - 命中：直接使用物理地址
   - 未命中：查页表，更新 TLB
3. 访问物理内存
```

## 四、面试题精讲

### 🔴 【高频】虚拟内存的作用

**问题**：什么是虚拟内存？有什么作用？

**标准答案**：

```
虚拟内存的作用：

1. 进程隔离
   - 每个进程有独立的虚拟地址空间
   - 32 位系统每个进程有 4GB 虚拟空间

2. 内存保护
   - 防止非法内存访问
   - 页表有访问权限控制

3. 内存抽象
   - 程序使用连续的虚拟地址
   - 不需要关心物理内存布局

4. 内存扩展
   - 虚拟空间可以大于物理内存
   - 通过换页机制实现

5. 共享内存
   - 不同进程可以共享物理内存
   - 只读区域可以共享
```

### 🟡 【中频】页表和 TLB

**问题**：页表是什么？TLB 是什么？

**标准答案**：

```
页表：
- 虚拟地址到物理地址的映射表
- 每个进程有自己的页表
- 存储在内存中

TLB（Translation Lookaside Buffer）：
- CPU 内部的高速缓存
- 缓存最近使用的页表项
- 避免每次访问内存查页表

工作流程：
1. MMU 先查 TLB
2. 命中则直接使用物理地址
3. 未命中则查页表，并更新 TLB
```

## 五、对比总结

### 虚拟内存 vs 物理内存

| 维度 | 虚拟内存 | 物理内存 |
|------|----------|----------|
| 性质 | 逻辑地址空间 | 实际硬件 |
| 大小 | 可以大于物理内存 | 受硬件限制 |
| 进程 | 每个进程独立 | 所有进程共享 |
| 访问 | 需要 MMU 转换 | 直接访问 |

## 六、扩展思考

### 💡 多级页表

```
32 位系统 4GB 虚拟空间，4KB 页：
- 4GB / 4KB = 1M 个页表项
- 每个页表项 4 字节 = 4MB 页表

多级页表：
- 一级页表 4KB（1024 项）
- 每项指向一个二级页表
- 只有使用的二级页表才分配

优势：节省内存
```

> 虚拟内存是操作系统最伟大的发明之一。它让每个程序都以为拥有独立的超大内存空间，同时又保护了系统的安全性和稳定性。
