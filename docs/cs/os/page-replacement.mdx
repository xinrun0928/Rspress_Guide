# 页面置换算法

> 目标级别：P6

面试官问：「页面置换算法有哪些？」你回答「FIFO、LRU」——然后面试官追问：「为什么 LRU 最优但实际很少用？」「什么是 Belady 现象？」「Clock 算法是怎么工作的？」

页面置换算法是虚拟内存管理的核心，理解它们对于理解操作系统内存管理至关重要。

## 一、页面置换基础

### 1.1 什么是页面置换

```
页面置换：
- 当物理内存已满，需要换出某个页
- 选择哪个页被换出的算法

缺页中断：
- 访问的页不在内存时发生
- 需要从磁盘加载缺失的页
```

### 1.2 缺页率

```
缺页率 = 缺页次数 / 总访问次数

影响因素：
1. 物理内存大小
2. 页面置换算法
3. 访问模式
```

## 二、常见置换算法

### 2.1 FIFO（先进先出）

```java
// FIFO 置换算法
public class FIFO置换 {
    private Queue<Integer> queue = new LinkedList<>();

    public int selectPageToReplace() {
        // 返回最早进入的页
        return queue.poll();
    }

    public void addPage(int page) {
        queue.offer(page);
    }
}
```

**特点**：简单，但可能有 Belady 现象。

### 2.2 LRU（最近最少使用）

```java
// LRU 置换算法
public class LRU置换 {
    private LinkedHashMap<Integer, Integer> cache;

    public LRU置换(int capacity) {
        cache = new LinkedHashMap<>(capacity, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry eldest) {
                return size() > capacity;
            }
        };
    }

    public int selectPageToReplace() {
        // 返回最久未使用的
        return cache.keySet().iterator().next();
    }
}
```

**特点**：性能好，但实现成本高。

### 2.3 OPT（最佳置换）

```
理论最优算法：
- 选择将来最长时间不再访问的页换出
- 实际无法实现（需要预知未来）

用途：作为其他算法的理论上限
```

## 三、Belady 现象

### 3.1 什么是 Belady 现象

```
Belady 现象：
- 物理内存增加，缺页率反而增加
- FIFO 算法会出现这个现象
- LRU、OPT 不会出现
```

### 3.2 FIFO Belady 示例

```
访问序列：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

物理页框 = 3 时：
1 2 3 4 1 2 5 1 2 3 4 5
缺 页 页 页 页 页 页
缺页次数：9

物理页框 = 4 时：
1 2 3 4 1 2 5 1 2 3 4 5
缺 页 页 页 缺 页 页 缺 页 页 页
缺页次数：10（更多！Belady 现象）
```

## 四、Clock 算法

### 4.1 原理

```
Clock（第二次机会）算法：
- FIFO 的改进
- 每页有一个访问位（A）
- 被访问时设为 1
- 置换时跳过 A=1 的页，将其设为 0
- 找到 A=0 的页进行置换
```

### 4.2 实现

```java
public class Clock算法 {
    private int[] pages;       // 页帧
    private boolean[] accessed; // 访问位
    private int pointer;        // 指针

    public int selectPageToReplace() {
        while (true) {
            if (!accessed[pointer]) {
                // 找到访问位为 0 的页
                int victim = pointer;
                pointer = (pointer + 1) % pages.length;
                return victim;
            } else {
                // 访问位清零，给第二次机会
                accessed[pointer] = false;
                pointer = (pointer + 1) % pages.length;
            }
        }
    }
}
```

## 五、面试题精讲

### 🔴 【高频】页面置换算法

**问题**：有哪些页面置换算法？各有什么特点？

**标准答案**：

```
1. FIFO（先进先出）
   - 最早进入的页最先换出
   - 简单，可能有 Belady 现象

2. LRU（最近最少使用）
   - 最久未使用的页换出
   - 性能好，实现成本高

3. OPT（最佳置换）
   - 选择将来最长时间不用的页
   - 理论最优，实际无法实现

4. Clock（第二次机会）
   - FIFO + 访问位改进
   - 性能和实现成本平衡

5. LFU（最不经常使用）
   - 访问次数最少的页换出
   - 实现成本高
```

### 🟡 【中频】Belady 现象

**问题**：什么是 Belady 现象？

**标准答案**：

```
Belady 现象：
- 物理内存增加，缺页率反而增加
- FIFO 算法会出现

原因：
- FIFO 按进入顺序置换，与访问频率无关
- 增加内存后，最早进入的页可能是高频访问的
- 这些页频繁换入换出

LRU 和 OPT 不会出现 Belady 现象。
```

## 六、对比总结

### 页面置换算法对比

| 算法 | 性能 | 实现复杂度 | Belady 现象 |
|------|------|------------|-------------|
| FIFO | 差 | 低 | 有 |
| OPT | 最优 | 不可能实现 | 无 |
| LRU | 好 | 高 | 无 |
| Clock | 较好 | 中 | 无 |
| LFU | 较好 | 高 | 有 |

## 七、扩展思考

### 💡 工作集模型

```
工作集：
- 最近 n 次访问的页面集合
- 表示程序当前活跃使用的页面

工作集置换：
- 淘汰不在工作集中的页
- 维持程序的局部性
```

> 页面置换算法是虚拟内存的核心。理解各种算法的优缺点，有助于理解操作系统内存管理的工作原理。
