# 操作系统原理

你写的每一行代码，最终都运行在操作系统之上。

但操作系统并不只是「运行环境」——它是硬件与软件之间的桥梁，是资源的管理者，是安全的守护者。

本章节将带你深入理解操作系统的底层原理：从内核态与用户态的划分，到系统调用的实现；从中断与异常的处理，到设备管理的方式。这些知识帮助你理解程序是如何运行的，也能让你在遇到系统级问题时找到方向。

## 知识体系总览

**内核态 vs 用户态** —— 为什么程序不能直接访问硬件？内核态和用户态是如何划分的？系统调用是如何实现的？

**中断与异常** —— 当你按下键盘时，操作系统是如何知道的？中断是现代操作系统高效运行的基础。

**设备管理** —— 键盘、鼠标、磁盘、网卡……这些设备是如何被操作系统管理的？轮询、中断、DMA，各自的优缺点是什么？

**死锁处理策略** —— 如果你已经理解了死锁是什么，那如何预防、避免、检测和解除死锁呢？

## 为什么需要内核态和用户态？

想象一个没有内核态/用户态区分的世界：

```c
// 如果程序可以直接访问硬件...
void malicious_code() {
    // 直接修改其他进程的内存？
    other_process->password_hash = 0;
    
    // 直接关闭磁盘控制器？
    disk_controller->power = 0;
    
    // 直接读写任意磁盘扇区？
    read_sector(0, "secret_file");
}
```

这样的系统是极其不安全的。一个有 bug 或恶意的程序可以轻易摧毁整个系统。

**内核态/用户态的划分**，正是为了解决这个问题。

## 核心问题

操作系统需要解决以下核心问题：

1. **如何保护**：防止恶意或错误的程序破坏系统和其他程序
2. **如何调度**：在多个进程之间公平高效地分配 CPU
3. **如何通信**：让进程能够安全地共享数据和协作
4. **如何管理**：高效管理硬件资源

这些问题将在后续章节中逐一解答。
