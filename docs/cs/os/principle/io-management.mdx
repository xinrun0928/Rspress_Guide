# 设备管理：I/O 控制方式、DMA、缓冲区管理

键盘、鼠标、显示器、磁盘、网卡……这些五花八门的设备，是如何被操作系统统一管理的？

一个磁盘读写可能需要几十毫秒，但 CPU 执行一条指令只需要纳秒级别。如果 CPU 一直等待磁盘读写完成再执行下一条指令，那 99.999% 的时间都在等待。

I/O 控制方式，正是为了解决这个问题。

## I/O 控制方式

### 方式一：轮询（Polling）

CPU 不断检查设备状态寄存器，直到设备准备好。

```c
// 轮询方式读取键盘
char read_keyboard_polling() {
    while (1) {
        // 检查键盘控制器状态
        unsigned char status = inb(KEYBOARD_STATUS_PORT);
        
        if (status & KEYBOARD_OUTPUT_BUFFER_FULL) {
            // 输出缓冲区有数据
            unsigned char scan_code = inb(KEYBOARD_DATA_PORT);
            return convert_to_ascii(scan_code);
        }
    }
}

// 问题：CPU 一直在循环检查，浪费大量时间
```

**优缺点：**

| 优点 | 缺点 |
|---|---|
| 实现简单 | CPU 浪费大量时间等待 |
| 延迟确定（最多一次轮询延迟） | 不适合多设备 |
| 适合高优先级设备 | 效率极低 |

### 方式二：中断驱动（Interrupt Driven）

设备完成操作后主动通知 CPU。

```c
// 中断方式读取键盘
char keyboard_buffer[256];
int buffer_head = 0;
int buffer_tail = 0;

// 键盘中断处理程序
void keyboard_interrupt_handler() {
    unsigned char scan_code = inb(KEYBOARD_DATA_PORT);
    
    // 放入缓冲区
    keyboard_buffer[buffer_head] = scan_code;
    buffer_head = (buffer_head + 1) % 256;
    
    // 发送 EOI
    outb(PIC_CMD_PORT, PIC_EOI);
}

// 应用程序读取
char read_keyboard() {
    // 阻塞等待数据
    while (buffer_head == buffer_tail) {
        sleep();  // 让出 CPU
    }
    
    char c = keyboard_buffer[buffer_tail];
    buffer_tail = (buffer_tail + 1) % 256;
    return convert_to_ascii(c);
}
```

**优缺点：**

| 优点 | 缺点 |
|---|---|
| CPU 可以做其他事 | 中断处理有开销 |
| 适合多设备 | 中断频繁时开销大 |
| 响应及时 | 复杂设备的控制复杂 |

### 方式三：DMA（Direct Memory Access）

DMA 控制器在设备和内存之间直接传输数据，CPU 不参与。


### DMA 的工作原理

```c
// DMA 控制器编程
struct dma_channel {
    unsigned int memory_address;  // 目标内存地址
    unsigned int count;          // 传输字节数
    unsigned char direction;     // 读/写
    unsigned char mode;          // 传输模式
};

// 设置 DMA
void dma_setup(int channel, void* buffer, int size, int direction) {
    // 1. 关闭 DMA 通道
    outb(DMA_MASK_REG, channel | DMA_MASK);
    
    // 2. 清除flip-flop（用于处理 16/32 位传输）
    outb(DMA_CLEAR_REG, 0);
    
    // 3. 设置传输地址（低 16 位）
    outb(DMA_CHANNEL_REG(channel), (unsigned long)buffer & 0xFF);
    outb(DMA_CHANNEL_REG(channel), ((unsigned long)buffer >> 8) & 0xFF);
    
    // 4. 设置传输计数
    outb(DMA_COUNT_REG(channel), (size - 1) & 0xFF);
    outb(DMA_COUNT_REG(channel), ((size - 1) >> 8) & 0xFF);
    
    // 5. 设置方向（读/写）
    outb(DMA_MODE_REG, direction | DMA_MODE_SINGLE);
    
    // 6. 开启 DMA 通道
    outb(DMA_MASK_REG, channel);
}

// 启动 DMA 传输
void dma_start(int channel) {
    // DMA 控制器开始工作
    // 数据直接从设备传输到内存
    // CPU 可以做其他事
}

// DMA 完成中断处理
void dma_interrupt_handler() {
    int channel = read_dma_status();
    // 清除中断标志
    // 唤醒等待进程
    wake_up_process(waiting_process[channel]);
}
```

### DMA 的类型

| 类型 | 说明 | 特点 |
|---|---|---|
| **总线仲裁型（Bus Master）** | DMA 控制器获得总线控制权 | 效率最高，独立总线 |
| **透明型（Transparent）** | DMA 只在 CPU 不使用时传输 | 不需要总线仲裁 |
| **块传输型（Block）** | 独占总线，传输完释放 | 简单，阻塞 CPU |
| **周期窃取型（Cycle Stealing）** | DMA 周期性地窃取总线周期 | 折中方案 |

### 三种 I/O 方式的对比

| 特性 | 轮询 | 中断驱动 | DMA |
|---|---|---|---|
| **CPU 参与度** | 高（每个字节） | 中（开始/结束） | 低（仅设置） |
| **适用场景** | 简单设备、低速设备 | 大多数设备 | 块设备、高速设备 |
| **延迟** | 确定（最短） | 不确定 | 不确定 |
| **实现复杂度** | 简单 | 中等 | 复杂 |
| **数据传输单位** | 字节 | 字节/块 | 块 |

## 缓冲区管理

### 为什么需要缓冲区？

```c
// 没有缓冲区的问题
// 场景：生产者每秒生产 1000 个数据，消费者每秒只能处理 100 个
// 没有缓冲区时：
// - 生产者必须等待消费者
// - 或数据丢失

// 有缓冲区时：
// - 生产者和消费者可以异步工作
// - 缓冲区吸收速度差异
// - 提供一定的容错能力
```

### 循环缓冲区

当需要更大的吞吐量时，使用循环缓冲区：

```c
// 循环缓冲区实现
struct circular_buffer {
    char* buffer;        // 缓冲区数组
    int capacity;         // 缓冲区容量
    int head;            // 读位置
    int tail;            // 写位置
    int count;           // 当前数据量
};

// 生产者写入
int write_to_buffer(struct circular_buffer* cb, char data) {
    if (cb->count == cb->capacity) {
        return -1;  // 缓冲区满
    }
    cb->buffer[cb->tail] = data;
    cb->tail = (cb->tail + 1) % cb->capacity;
    cb->count++;
    return 0;
}

// 消费者读取
int read_from_buffer(struct circular_buffer* cb, char* data) {
    if (cb->count == 0) {
        return -1;  // 缓冲区空
    }
    *data = cb->buffer[cb->head];
    cb->head = (cb->head + 1) % cb->capacity;
    cb->count--;
    return 0;
}
```

### 缓冲区与缓存

```
缓冲区 vs 缓存

缓冲区（Buffer）:
- 用于弥补速度差异
- 数据最终会传输到目的地
- 是数据传输的中间站
- 例子：磁盘写入缓冲区

缓存（Cache）:
- 用于存储热点数据的副本
- 提高访问速度
- 数据可能在某处有原始副本
- 例子：CPU Cache、Page Cache
```

### Linux 驱动程序结构

```c
// Linux 字符设备驱动示例
#include <linux/fs.h>
#include <linux/module.h>
#include <linux/cdev.h>

#define DEVICE_NAME "my_device"
#define BUF_SIZE 1024

static char device_buffer[BUF_SIZE];
static int major_number;
static struct cdev my_cdev;

// 文件操作集合
static struct file_operations fops = {
    .owner   = THIS_MODULE,
    .read    = device_read,
    .write   = device_write,
    .open    = device_open,
    .release = device_release,
};

// 打开设备
static int device_open(struct inode* inode, struct file* file) {
    // 初始化、计数等
    return 0;
}

// 读取设备
static ssize_t device_read(struct file* file, char __user* buf,
                           size_t len, loff_t* offset) {
    int bytes_read = 0;
    
    if (*offset >= BUF_SIZE)
        return 0;
    
    if (*offset + len > BUF_SIZE)
        len = BUF_SIZE - *offset;
    
    if (copy_to_user(buf, device_buffer + *offset, len))
        return -EFAULT;
    
    *offset += len;
    return len;
}

// 写入设备
static ssize_t device_write(struct file* file, const char __user* buf,
                            size_t len, loff_t* offset) {
    int bytes_written = 0;
    
    if (*offset >= BUF_SIZE)
        return -ENOSPC;
    
    if (*offset + len > BUF_SIZE)
        len = BUF_SIZE - *offset;
    
    if (copy_from_user(device_buffer + *offset, buf, len))
        return -EFAULT;
    
    *offset += len;
    return len;
}

// 模块初始化
static int __init my_driver_init(void) {
    // 注册字符设备
    register_chrdev(0, DEVICE_NAME, &fops);
    return 0;
}

// 模块退出
static void __exit my_driver_exit(void) {
    unregister_chrdev(major_number, DEVICE_NAME);
}

module_init(my_driver_init);
module_exit(my_driver_exit);
```

## 面试高频问题

**Q1：轮询、中断、DMA 三种 I/O 方式的区别？**

轮询是 CPU 不断检查设备状态，效率最低。中断是设备主动通知 CPU，CPU 可以做其他事。DMA 是让 DMA 控制器直接在设备和内存之间传输数据，CPU 几乎不参与。

**Q2：DMA 解决了什么问题？**

DMA 解决了 CPU 在 I/O 操作中的瓶颈问题。CPU 不需要参与每个字节的传输，可以去做其他计算工作，提高系统整体效率。

**Q3：什么是缓冲区？缓冲区的作用是什么？**

缓冲区是临时存储数据的内存区域，用于弥补数据生产者和消费者之间的速度差异，提供异步操作能力。缓冲区可以让输入和输出设备并行工作。

**Q4：为什么网络设备常用中断驱动？**

因为网络数据到达的时机不确定，无法预知何时有数据。如果用轮询，CPU 会浪费大量时间检查；用中断，数据到达时主动通知 CPU，效率更高。

**Q5：设备驱动程序的作用是什么？**

驱动程序是操作系统和硬件之间的桥梁。它向上提供统一的设备接口（文件操作），向下访问具体的硬件寄存器，控制设备工作。驱动程序屏蔽了硬件的复杂性。

理解 I/O 控制方式和设备管理，是理解操作系统如何与硬件交互的关键。
