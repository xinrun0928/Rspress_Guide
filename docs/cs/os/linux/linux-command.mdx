# Linux 常用命令与 Shell 脚本

`top`、`ps`、`grep`、`awk`、`sed`……这些命令是每个 Linux 工程师的瑞士军刀。

但很多人只是机械地使用，不理解背后的原理。本篇将带你理解这些命令的工作机制，以及如何组合使用它们。

## 进程管理命令

### ps：查看进程

```bash
# 基本用法
ps                # 显示当前终端的进程
ps -e             # 显示所有进程
ps -f             # Full format
ps aux            # BSD 风格（显示所有进程 + 详细信息）
ps -ef            # System V 风格

# 常用组合
ps aux | grep nginx          # 查找 nginx 进程
ps -eo pid,ppid,cmd,%mem,%cpu | sort -k4 -nr  # 按内存排序

# 显示特定用户的进程
ps -u username

# 显示进程树
ps -ef --forest
# 输出:
#   PID   CMD
#     1   /sbin/init
#   456   ├─ /usr/sbin/sshd -D
#   789   │   └─ sshd: user@pts/0
#   790   │       └─ -bash
#   801   │           └─ ps -ef --forest
```

### top：实时监控系统

```bash
# 基本用法
top

# 交互命令（在 top 界面中按）
# h: 帮助
# q: 退出
# 1: 显示所有 CPU 核心
# M: 按内存排序
# P: 按 CPU 排序
# N: 按 PID 排序
# t: 切换 CPU 显示
# m: 切换内存显示
# l: 切换负载显示
# f: 选择显示字段
# k: 杀死进程
# r: 调整进程优先级
# s: 改变更新间隔

# 常用选项
top -d 1              # 每秒更新（默认 3 秒）
top -p 1234           # 只监控特定 PID
top -u username       # 只显示某用户的进程
top -b -n 10 > out.txt  # 批处理模式，输出 10 次
```

### kill：发送信号

```bash
# 发送信号到进程
kill -l                    # 列出所有信号
kill -9 <pid>             # 强制杀死进程 (SIGKILL)
kill -15 <pid>            # 正常终止进程 (SIGTERM，默认)
kill -2 <pid>             # 中断 (Ctrl+C)
kill -STOP <pid>          # 暂停进程
kill -CONT <pid>          # 继续进程

# 杀死进程组
kill -9 -<pgid>

# 杀死特定名称的所有进程
pkill nginx               # 杀死所有 nginx
killall java              # 杀死所有 java 进程
```

## 内存监控命令

### free：查看内存

```bash

free

# 人类可读格式
free -h

# 以字节为单位
free -b

# 显示总和
free -t

# 连续显示
watch -n 1 free -h

# 输出:
#               total        used        free      shared  buff/cache   available
# Mem:          31Gi       8.5Gi       2.1Gi       456Mi       20Gi       22Gi
# Swap:         8Gi           0B         8Gi
```

### vmstat：虚拟内存统计

```bash

#  r  b   swpd   free   buff  cache   si   so    bi    bo   in   cs us sy id wa
#  1  0      0 102400  12345 204800    0    0     0     0   50  100  5 10 30 55
```

## 网络命令

### ss：Socket 统计

```bash
# 基本用法
ss                          # 显示所有 socket
ss -l                       # 只显示监听 socket
ss -t                       # TCP
ss -u                       # UDP
ss -w                       # Raw
ss -x                       # Unix

# 常用组合
ss -ltnp                    # 监听 TCP + PID
ss -ltn sport = :80         # 过滤端口
ss -tnp state established   # 已建立连接

# 统计
ss -s                       # 统计摘要

# 输出:
# Netid  State   Recv-Q  Send-Q   Local Address:Port   Peer Address:Port
# tcp    LISTEN  0       128      0.0.0.0:22          0.0.0.0:*
# tcp    ESTAB   0       0        10.0.0.1:22          10.0.0.2:54321
# ...
```

### netstat：经典网络工具

```bash
# 基本用法
netstat -a                   # 所有连接
netstat -t                  # TCP
netstat -u                  # UDP
netstat -l                  # 监听
netstat -p                  # 显示进程
netstat -n                  # 数字地址

# 常用组合
netstat -tulnp              # 监听端口 + 进程
netstat -anp | grep :80     # 查找 80 端口
netstat -i                   # 网络接口统计
netstat -r                   # 路由表

# ss 是 netstat 的现代替代，更快
```

### tcpdump：抓包

```bash
# 基本用法
tcpdump -i eth0              # 监听特定网卡
tcpdump -i any               # 监听所有网卡

# 表达式
tcpdump host 192.168.1.1     # 特定主机
tcpdump port 80              # 特定端口
tcpdump tcp                  # TCP 协议
tcpdump icmp                 # ICMP 协议
tcpdump 'port 80 and host 192.168.1.1'  # 组合条件

# 保存到文件
tcpdump -w capture.pcap -i eth0

# 读取文件
tcpdump -r capture.pcap

# 常用选项
tcpdump -vv                  # 详细输出
tcpdump -c 100               # 只抓 100 个包
tcpdump -nn                  # 不要解析域名
```

## Shell 脚本基础

### 变量和字符串

```bash
#!/bin/bash

# 变量定义
name="World"
echo "Hello, $name!"
echo "Hello, ${name}!"

# 命令替换
current_date=$(date)
echo "Today is $current_date"

# 字符串操作
str="Hello, World!"
echo ${#str}              # 长度: 13
echo ${str:0:5}          # 子串: Hello
echo ${str/World/Linux}  # 替换: Hello, Linux!

# 数组
arr=(one two three four)
echo ${arr[0]}           # one
echo ${arr[@]}            # 全部
echo ${#arr[@]}           # 长度: 4
```

### 条件判断

```bash
#!/bin/bash

# 字符串比较
if [ "$name" == "World" ]; then
    echo "Hello, World!"
fi

# 数值比较
if [ $count -gt 10 ]; then
    echo "Count is greater than 10"
fi

# 文件测试
if [ -f "file.txt" ]; then
    echo "File exists"
fi

if [ -d "/path/to/dir" ]; then
    echo "Directory exists"
fi

# 常用测试操作符
# -f  文件存在
# -d  目录存在
# -r  可读
# -w  可写
# -x  可执行
# -z  字符串为空
# -n  字符串非空
# -eq 等于
# -ne 不等于
# -gt 大于
# -lt 小于
```

### 循环

```bash
#!/bin/bash

# for 循环
for i in {1..5}; do
    echo "Number: $i"
done

# C 风格 for 循环
for ((i=0; i<5; i++)); do
    echo "i = $i"
done

# while 循环
count=0
while [ $count -lt 5 ]; do
    echo "Count: $count"
    ((count++))
done

# 遍历文件
for file in /path/to/*; do
    if [ -f "$file" ]; then
        echo "File: $file"
    fi
done

# 读取文件行
while read line; do
    echo "Line: $line"
done < file.txt
```

### 函数

```bash
#!/bin/bash

# 定义函数
function hello() {
    echo "Hello, $1!"  # $1 是第一个参数
}

# 调用函数
hello "World"  # 输出: Hello, World!

# 返回值
function add() {
    local sum=$(($1 + $2))
    echo $sum
}

result=$(add 3 5)
echo "3 + 5 = $result"

# 局部变量
function counter() {
    local count=0
    ((count++))
    echo $count
}
```

### 实用脚本示例

```bash
#!/bin/bash
# 监控进程内存使用，超过阈值则告警

THRESHOLD=80  # 内存使用率阈值
PROCESS_NAME="java"

while true; do
    # 获取进程内存使用率
    mem_usage=$(ps -eo pid,comm,%mem | grep $PROCESS_NAME | awk '{print $3}' | head -1)
    
    if (( $(echo "$mem_usage > $THRESHOLD" | bc -l) )); then
        echo "[$(date)] WARNING: $PROCESS_NAME memory usage: ${mem_usage}%"
        # 发送告警邮件
        # mail -s "Memory Alert" admin@example.com < /dev/null
    fi
    
    sleep 60  # 每分钟检查一次
done
```

```bash
#!/bin/bash
# 批量日志清理
```

## 管道与重定向

```bash
# 管道：将一个命令的输出传给另一个命令
ps aux | grep nginx | head -5

# 重定向
command > output.txt      # 标准输出重定向（覆盖）
command >> output.txt     # 标准输出重定向（追加）
command 2> error.txt      # 标准错误重定向
command &> all.txt       # 标准输出和错误重定向

# 丢弃输出
command > /dev/null

# Here 文档
cat <<EOF > file.txt
This is line 1
This is line 2
Variable: $HOME
EOF

# Here 字符串
grep "pattern" <<< "This is a test string"
```

## 面试高频问题

**Q1：ps 和 top 的区别？**

ps 是静态快照，显示某个时刻的进程信息。top 是动态更新，持续显示系统状态和进程信息，适合实时监控。

**Q2：free 和 top 的内存显示有什么区别？**

free 显示的是操作系统层面的内存统计。top 的 RES（实际使用物理内存）显示的是进程实际占用的物理内存。available 是可分配给新进程的内存（包括可回收的缓存）。

**Q3：tcpdump 输出的 "flags [S]" 是什么意思？**

这是 TCP 标志位：[S] 是 SYN（连接建立请求），[S.] 是 SYN+ACK，[.] 是 ACK，[F] 是 FIN（连接关闭），[P] 是 PSH（推送数据）。

**Q4：如何写一个脚本来分析日志？**

可以用 `grep` 过滤，`awk` 处理字段，`sort` 排序，`uniq` 去重，`wc` 计数。组合这些工具可以完成复杂的日志分析任务。

掌握这些命令和脚本技巧，是每个 Linux 工程师的必备技能。
