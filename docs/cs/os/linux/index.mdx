# Linux

全球超过 90% 的服务器运行着 Linux，从 Google 到 Facebook，从 Amazon 到 Netflix。

理解 Linux 的内部机制，是成为优秀后端工程师的必经之路。

本章节将带你深入理解 Linux 的核心子系统：从进程调度到内存管理，从文件系统到网络协议栈。这些知识不仅帮助你更好地使用 Linux，更能在遇到问题时快速定位根源。

## 知识体系总览

**Linux 进程与线程管理** —— Linux 如何管理数十万个进程？调度器是如何工作的？线程和进程在 Linux 中有什么区别？

**Linux 内存管理** —— 伙伴系统和 slab 分配器如何协作？什么是内存映射？如何监控内存使用？

**Linux 文件系统** —— Ext4 的 inode 结构是什么？VFS 是如何统一不同文件系统的？

**Linux 网络协议栈** —— 数据包从网卡到应用程序，经历了什么？

**Linux 常用命令** —— top、vmstat、iostat、netstat……这些命令如何帮助分析系统问题？

**Linux 性能分析** —— 如何找到系统瓶颈？是 CPU、内存、I/O 还是网络？

## 为什么选择 Linux？

**服务器领域：** Linux 是绝对霸主。稳定、高效、安全、免费。

**云原生时代：** Kubernetes、Docker、Prometheus……所有容器化和云原生技术都基于 Linux。

**学习价值：** Linux 是理解操作系统原理的最佳实践平台，代码开源，设计优雅。

## 面试关联

Linux 是后端面试的重点：

- Linux 的进程调度算法是什么？
- 什么是伙伴系统和 slab 分配器？
- Linux 的文件系统结构是怎样的？
- 如何排查 CPU、内存、I/O 瓶颈？
- 什么是 OOM Killer？

理解 Linux 内部机制，能让你在面试中脱颖而出。
