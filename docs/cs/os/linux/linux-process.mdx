# Linux 进程与线程管理

`ps aux` 命令显示了数百个进程。init、systemd、sshd、nginx……它们是如何被 Linux 管理的？

Linux 的进程管理是其最核心的部分之一。理解 Linux 如何创建、调度、切换进程，是理解整个操作系统的关键。

## 进程与线程的统一视图

### Linux 的独特设计

Linux 把一切都看作「任务」（Task），进程和线程没有本质区别。

```c
// Linux 内核中的任务描述符（简化版）
struct task_struct {
    // 标识
    pid_t pid;              // 进程 ID
    pid_t tgid;             // 线程组 ID（主线程 PID）
    char name[TASK_COMM_LEN]; // 进程名
    
    // 状态
    volatile long state;    // 进程状态
    unsigned int flags;     // 进程标志
    
    // 调度
    int prio;               // 动态优先级
    int static_prio;        // 静态优先级
    int normal_prio;        // 正常优先级
    unsigned int rt_priority; // 实时优先级
    
    // 调度实体（CFS 使用）
    struct sched_entity se; // 完全公平调度实体
    struct sched_rt_entity rt; // 实时调度实体
    struct sched_dl_entity dl;  // 延迟调度实体
    
    // 内存
    struct mm_struct* mm;   // 内存描述符
    struct mm_struct* active_mm;
    
    // 文件
    struct files_struct* files; // 文件描述符表
    
    // 信号处理
    struct signal_struct* signal; // 信号描述符
    struct sighand_struct* sighand;
    
    // 关系
    struct task_struct *real_parent;  // 真实父进程
    struct task_struct *parent;       // 当前父进程
    struct list_head children;        // 子进程链表
    struct list_head sibling;         // 兄弟链表
    
    // 时间
    struct timespec utime;  // 用户态时间
    struct timespec stime;  // 内核态时间
    struct timespec utimescaled; // 缩放后的用户态时间
    struct timespec stimescaled; // 缩放后的内核态时间
    
    // 命名空间
    struct nsproxy* nsproxy;
    
    // ... 还有更多字段
};
```

**关键洞察：**
- 每个进程有一个 `task_struct`
- 线程本质上是共享地址空间的 `task_struct`
- `mm`（内存描述符）共享就是线程，不共享就是进程

## 进程创建

### fork + exec 模型

Linux 使用经典的 Unix 模型：fork 创建进程，exec 加载新程序。

```c
// fork + exec 示例
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();  // 创建子进程
    
    if (pid < 0) {
        perror("fork failed");
        return 1;
    }
    
    if (pid == 0) {
        // 子进程
        printf("Child: pid=%d, ppid=%d\n", getpid(), getppid());
        execlp("ls", "ls", "-la", NULL);  // 替换为新程序
        perror("execlp failed");  // 只有 execlp 失败才会执行
        return 1;
    } else {
        // 父进程
        printf("Parent: pid=%d, child=%d\n", getpid(), pid);
        int status;
        waitpid(pid, &status, 0);  // 等待子进程结束
        printf("Child exited with %d\n", WEXITSTATUS(status));
    }
    
    return 0;
}
```

### fork 的实现

```c
// 简化版的 fork 实现
long do_fork(unsigned long clone_flags) {
    struct task_struct* p;
    
    // 1. 分配新的 task_struct
    p = dup_task_struct(current);
    
    // 2. 设置进程关系
    p->real_parent = current;
    p->parent = current;
    
    // 3. 设置调度参数
    p->prio = current->prio;
    
    // 4. 复制文件描述符（如果 CLONE_FILES 未设置）
    if (!(clone_flags & CLONE_FILES))
        p->files = dup_fd(p->files);
    
    // 5. 复制内存空间（如果 CLONE_VM 未设置）
    if (!(clone_flags & CLONE_VM))
        dup_mm(p);
    
    // 6. 分配 PID
    p->pid = alloc_pid();
    
    // 7. 设置进程状态为就绪
    p->state = TASK_RUNNING;
    
    // 8. 加入调度队列
    wake_up_new(p);
    
    return p->pid;
}
```

### vfork 与 clone

```c
// vfork：共享地址空间的 fork
// 现代 Linux 中，vfork 和 fork 几乎等价
// 因为写时复制（COW）让 fork 也很高效

// clone：更灵活的任务创建
// 用于实现线程
clone(CLONE_VM | CLONE_FS | CLONE_FILES | CLONE_SIGHAND,
      child_stack,      // 子进程的栈
      0,                // pid
      NULL);            // 传递给函数的参数

// 等价于 pthread_create
```

## 调度器

### Linux 调度器的演进

| 版本 | 调度器 | 特点 |
|---|---|---|
| 2.4 | O(n) 调度器 | 简单，扫描所有任务，开销大 |
| 2.6 | O(1) 调度器 | 优先级数组，固定时间 |
| 2.6.23 | CFS | 完全公平调度器 |
| 3.14 | BFS | 脑残调度器（桌面优化） |
| 4.20+ | EEVDF | 延迟公平调度器 |

### CFS（Completely Fair Scheduler）

CFS 的核心思想：**绝对公平**——每个进程按权重分配 CPU 时间，虚拟运行时间最短的先跑。

```c
// CFS 调度实体
struct sched_entity {
    // 虚拟运行时间（vruntime）
    // 越小表示获得 CPU 时间越少，越应该先跑
    u64 vruntime;
    
    // 在红黑树中的位置
    struct rb_node run_node;
    
    // 时间片统计
    u64 exec_start;
    u64 sum_exec_runtime;
    
    // 权重（nice 值影响）
    unsigned long weight;
};

// 红黑树组织
// 所有可运行进程按 vruntime 排序
// 最左边的节点（最小 vruntime）获得 CPU
```

```

│                    红黑树（按 vruntime 排序）                │
```

### CFS 的时间片

CFS 没有固定时间片，而是用「调度延迟」和「最小粒度」控制：

```c
// CFS 参数
unsigned int sysctl_sched_latency = 60000000ULL;  // 60ms
unsigned int sysctl_sched_min_granularity = 4000000ULL;  // 4ms

// 每个进程在一个调度周期内的理想运行时间
// = 调度周期 / 可运行进程数
// 如果进程太多，每个进程的最小时间 = 最小粒度
```

### 实时调度策略

Linux 提供实时调度策略，优先级高于普通进程：

| 策略 | 说明 | 优先级 |
|---|---|---|
| `SCHED_FIFO` | 先进先出，无时间片 | 1-99 |
| `SCHED_RR` | 轮转，有时间片 | 1-99 |
| `SCHED_DEADLINE` | 期限调度 | 最高 |

```bash
# 查看进程的调度策略
chrt -p <pid>

# 设置实时调度
chrt -f 50 ./my_program  # FIFO，优先级 50
chrt -r 50 ./my_program  # RR，优先级 50
```

## 上下文切换

### 上下文切换的过程

```c
// 上下文切换的核心（简化）
__switch_to(prev, next) {
    // 1. 保存 prev 的上下文
    save callee-saved registers  // 保存调用者保存的寄存器
    save FPU/SSE/AVX state      // 保存浮点/向量状态
    prev->sp = current_stack_pointer
    
    // 2. 恢复 next 的上下文
    current_stack_pointer = next->sp
    restore FPU/SSE/AVX state    // 恢复浮点/向量状态
    restore callee-saved registers
    
    // 3. 返回到新进程
    return;
}
```

## 面试高频问题

**Q1：Linux 进程和线程的区别？**

Linux 不区分进程和线程，都是 `task_struct`。区别在于是否共享地址空间（mm_struct）。通过 `clone(CLONE_VM)` 创建的共享地址空间的任务就是线程。

**Q2：什么是写时复制（COW）？**

fork 时父子进程共享相同的物理内存，只有当其中一方尝试写入时才复制物理页。这样 fork 的开销大大减少。

**Q3：CFS 调度器的核心思想是什么？**

完全公平调度器让每个进程按权重分配 CPU 时间。进程按「虚拟运行时间」排序，最小的先跑。这保证了系统总体上的公平性。

**Q4：上下文切换的开销有哪些？**

直接开销包括寄存器保存/恢复、浮点状态保存、TLB 刷新。间接开销包括 Cache miss 率上升、分支预测失败等。

**Q5：僵尸进程和孤儿进程有什么区别？**

僵尸进程是子进程退出后，父进程没有调用 wait() 回收其退出状态。孤儿进程是父进程先于子进程退出，子进程被 init/systemd 收养。

理解 Linux 进程与线程管理，是理解 Linux 内核的关键。
