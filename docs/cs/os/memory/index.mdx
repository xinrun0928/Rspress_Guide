# 内存管理

你的电脑有 16GB 内存，但同时运行着浏览器、IDE、音乐播放器、微信……加起来可能需要 32GB。为什么系统还能正常运行？

这就是内存管理的魔力——它让有限的物理内存，看起来像是无限的虚拟地址空间。

本章节将带你深入理解操作系统如何管理宝贵的内存资源：从连续分配到分页分段，从页面置换算法到虚拟内存，这些知识将帮助你写出更高效的程序，也能让你在面试中应对各种内存相关的问题。

## 知识体系总览

**连续分配 vs 非连续分配** —— 早期的内存管理是连续的，就像一整块披萨要分给不同的人。分页和分段打破了这种连续性，让内存管理更加灵活。

**分页机制** —— 把物理内存和虚拟内存都划分成固定大小的页框和页面。就像把披萨切成固定大小的块，按需分配。

**分段与段页式** —— 分段按程序的逻辑结构划分，分页按固定大小划分。段页式结合两者优点，是现代操作系统的常见方案。

**页面置换算法** —— 当物理内存不够时，需要把不常用的页换出到磁盘。FIFO、LRU、LFU、Clock……每种算法都有其适用场景。

**局部性原理与抖动** —— 程序为什么会有「热点」？抖动是如何产生的？理解局部性原理，是理解缓存技术的关键。

**虚拟内存** —— 操作系统为每个进程提供的独立地址空间抽象，让程序可以访问比物理内存更大的空间。

## 为什么需要内存管理？

**问题一：地址空间隔离**

如果所有程序都直接访问物理内存，程序 A 可能不小心覆盖了程序 B 的数据，整个系统会混乱不堪。

**问题二：内存高效利用**

一个程序可能只需要 10MB 内存，但你买了 16GB 的电脑。如果每个程序都要占用连续的大块内存，内存很快就会碎片化，利用率极低。

**问题三：简化编程**

程序员不需要关心物理内存的具体位置。操作系统提供虚拟地址空间，程序只需要使用虚拟地址，剩下的交给操作系统。

## 核心问题

内存管理要解决三个核心问题：

1. **重定位**：程序编译时使用相对地址，运行时需要映射到物理地址
2. **保护**：防止一个进程访问另一个进程的内存
3. **共享**：让多个进程共享同一段物理内存（如共享库）

这三个问题将在后续章节中逐一解答。

## 面试关联

内存管理是操作系统面试的重灾区：

- 分页和分段的区别是什么？
- 页表有哪些优化？为什么需要多级页表？
- LRU 是怎么实现的？
- 什么是抖动？如何解决？
- 虚拟内存的作用是什么？

这些问题看似基础，但能回答出「为什么」的人却不多。本章节不仅讲解「是什么」，更注重讲解「为什么」，帮助你真正理解内存管理的设计思想。
