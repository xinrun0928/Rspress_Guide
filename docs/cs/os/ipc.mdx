# 进程间通信方式

> 目标级别：P5

面试官问：「进程间通信有哪些方式？」你回答「管道、消息队列、共享内存」——然后面试官追问：「共享内存为什么最快？」「Socket 可以用于本地通信吗？」「信号和信号量的区别是什么？」

进程间通信（IPC）是操作系统的重要概念，理解各种 IPC 方式的优缺点对于系统设计至关重要。

## 一、IPC 方式总览

| 方式 | 速度 | 复杂度 | 适用场景 |
|------|------|--------|----------|
| 管道 | 快 | 简单 | 父子进程单向通信 |
| 命名管道 | 快 | 中等 | 无亲缘关系进程 |
| 消息队列 | 中 | 中等 | 多进程消息传递 |
| 共享内存 | 最快 | 复杂 | 大量数据交换 |
| 信号 | 快 | 简单 | 异步事件通知 |
| Socket | 较慢 | 复杂 | 跨机器通信 |
| 信号量 | 快 | 简单 | 同步控制 |

## 二、管道

### 2.1 匿名管道

```c
// 父子进程通过管道通信
int pipefd[2];
pipe(pipefd);  // 创建管道

if (fork() == 0) {
    // 子进程写
    close(pipefd[0]);
    write(pipefd[1], "hello", 5);
    close(pipefd[1]);
} else {
    // 父进程读
    close(pipefd[1]);
    char buf[100];
    read(pipefd[0], buf, 100);
    close(pipefd[0]);
}
```

### 2.2 命名管道（FIFO）

```bash
# 创建命名管道
mkfifo /tmp/myfifo

# 进程 A 写
echo "hello" > /tmp/myfifo

# 进程 B 读
cat /tmp/myfifo
```

## 三、共享内存

### 3.1 为什么最快

```
共享内存是最快的 IPC 方式：

原因：
- 数据直接在内存中共享
- 没有内核参与数据传输
- 只是将同一块物理内存映射到不同进程的地址空间

其他方式需要：
- 管道：数据需要从用户态复制到内核态，再复制到用户态
- 消息队列：同样需要内核中转
```

### 3.2 实现示例

```c
#include <sys/shm.h>

// 创建共享内存
int shmid = shmget(IPC_PRIVATE, 1024, IPC_CREAT | 0666);

// 附加到进程地址空间
void *shm = shmat(shmid, NULL, 0);

// 写入数据
strcpy(shm, "Hello from process 1");

// 分离共享内存
shmdt(shm);

// 删除共享内存
shmctl(shmid, IPC_RMID, NULL);
```

## 四、面试题精讲

### 🔴 【高频】进程间通信方式及特点

**问题**：进程间通信有哪些方式？各有什么特点？

**标准答案**：

```
1. 管道
   - 字节流通信，半双工
   - 只能用于有亲缘关系的进程（匿名管道）
   - 无亲缘关系用命名管道

2. 消息队列
   - 消息结构，有格式
   - 独立于进程
   - 可按优先级排序

3. 共享内存
   - 速度最快
   - 需要配合信号量使用
   - 同步问题需要额外处理

4. 信号
   - 异步通知
   - 用于进程控制
   - 处理信号需要注册信号处理函数

5. Socket
   - 支持跨机器通信
   - 本地通信可用 Unix Domain Socket

速度排序：共享内存 > 管道 > 消息队列 > Socket
```

### 🟡 【中频】共享内存为什么最快

**问题**：共享内存为什么是速度最快的 IPC 方式？

**标准答案**：

```
共享内存不需要在进程和内核之间复制数据：

其他 IPC 方式：
进程 A → 内核缓冲区 → 进程 B（两次拷贝）

共享内存：
进程 A 和进程 B 映射同一块物理内存
→ 直接在内存中共享数据（零拷贝）

共享内存需要注意：
- 同步问题（需要信号量或互斥锁）
- 竞态条件
- 内存管理
```

## 五、对比总结

### IPC 方式对比

| 方式 | 速度 | 是否需要拷贝 | 同步方式 | 持久性 |
|------|------|--------------|----------|--------|
| 管道 | 快 | 2 次 | 读写阻塞 | 随进程 |
| 消息队列 | 中 | 2 次 | 消息结构 | 随内核 |
| 共享内存 | 最快 | 0 次 | 信号量 | 随内核 |
| 信号 | 快 | 0 次 | 异步 | 随进程 |
| Socket | 慢 | 2 次 | 读写阻塞 | 可跨机器 |

## 六、扩展思考

### 💡 信号 vs 信号量

```
信号（Signal）：
- 异步通知机制
- 用于终止进程、暂停进程等
- 如 SIGINT（Ctrl+C）、SIGKILL

信号量（Semaphore）：
- 计数器，用于同步
- P（减）操作和 V（加）操作
- 用于保护共享资源
```

> 进程间通信是并发编程的基础。选择合适的 IPC 方式需要考虑：通信速度、数据量、同步需求、进程关系等因素。
