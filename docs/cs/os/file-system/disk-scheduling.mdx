# 磁盘调度算法：FCFS、SCAN、C-SCAN、LOOK

磁盘是计算机中最慢的组件之一。

CPU 访问寄存器：纳秒级
CPU 访问 L1 Cache：1 纳秒
访问主存：100 纳秒
访问 SSD：微秒级
访问磁盘：**毫秒级**

访问一次磁盘的时间，CPU 可以执行上百万条指令。而磁盘操作的主要延迟来自**寻道时间**——磁头臂移动到目标磁道的时间。

磁盘调度算法，正是为了减少这个寻道时间。

### 磁盘访问时间

磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间

```c
// 访问时间计算
struct disk_stats {
    double seek_time;      // 寻道时间（磁头移动到目标磁道）
    double rotation_delay; // 旋转延迟（等待目标扇区转到磁头下）
    double transfer_time; // 传输时间（读写数据）
};

// 寻道时间
double calculate_seek_time(int from_track, int to_track, double seek_cost) {
    return abs(to_track - from_track) * seek_cost;
}

// 旋转延迟（平均是半圈）
double rotation_delay(int rpm) {
    double rotation_time = 60.0 / rpm;  // 一圈的时间（秒）
    return rotation_time / 2;  // 平均半圈
}

// 总访问时间
double access_time(struct disk_stats* disk, int current_track, int target_track) {
    double seek = calculate_seek_time(current_track, target_track, 0.1);
    double rotation = rotation_delay(7200);
    double transfer = disk->transfer_time;
    return seek + rotation + transfer;
}
```

### 为什么需要调度？

```bash
# 假设请求序列（磁道号）：98, 183, 37, 122, 14, 124, 65, 67
# 磁头初始位置：53

# 无调度（按请求顺序）：
53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67
总寻道距离 = |53-98| + |98-183| + |183-37| + |37-122| + |122-14| + |14-124| + |124-65| + |65-67|
           = 45 + 85 + 146 + 85 + 108 + 110 + 59 + 2 = 640

# 优化调度（见下文）：
53 → 65 → 67 → 37 → 14 → 98 → 122 → 124 → 183
总寻道距离 = 12 + 2 + 30 + 23 + 84 + 24 + 2 + 59 = 236

差距：2.7 倍！
```

## FCFS（First Come First Serve）

### 原理

最简单的调度：按请求到达的顺序服务。

```c
// FCFS 调度
void fcfs_schedule(int requests[], int n, int initial_position) {
    int total_seek = 0;
    int current = initial_position;
    
    printf("FCFS 调度顺序:\n");
    for (int i = 0; i < n; i++) {
        total_seek += abs(current - requests[i]);
        printf("  %d -> %d (寻道: %d)\n", 
               current, requests[i], abs(current - requests[i]));
        current = requests[i];
    }
    printf("总寻道距离: %d\n", total_seek);
}
```

### 示例

```
磁道请求: 98, 183, 37, 122, 14, 124, 65, 67
磁头初始位置: 53

FCFS:
  53 → 98 (45) → 183 (85) → 37 (146) → 122 (85) → 14 (108) → 124 (110) → 65 (59) → 67 (2)
  总寻道距离: 640
```

### 优缺点

| 优点 | 缺点 |
|---|---|
| 公平，每个请求都会被服务 | 磁头移动可能很远 |
| 实现简单 | 效率低 |
| 不会饿死 | 不适合高负载场景 |

## SSTF（Shortest Seek Time First）

### 原理

总是选择离当前磁头位置最近的请求。

```c
// SSTF 调度
void sstf_schedule(int requests[], int n, int initial_position) {
    int total_seek = 0;
    int current = initial_position;
    bool served[n] = {false};
    
    printf("SSTF 调度顺序:\n");
    for (int count = 0; count < n; count++) {
        int nearest = -1;
        int min_distance = INT_MAX;
        
        // 找最近的未服务请求
        for (int i = 0; i < n; i++) {
            if (!served[i]) {
                int distance = abs(current - requests[i]);
                if (distance < min_distance) {
                    min_distance = distance;
                    nearest = i;
                }
            }
        }
        
        served[nearest] = true;
        total_seek += abs(current - requests[nearest]);
        printf("  %d -> %d (寻道: %d)\n", 
               current, requests[nearest], abs(current - requests[nearest]));
        current = requests[nearest];
    }
    printf("总寻道距离: %d\n", total_seek);
}
```

### 示例

```
磁道请求: 98, 183, 37, 122, 14, 124, 65, 67
磁头初始位置: 53

SSTF:
  53 → 65 (12) → 67 (2) → 37 (30) → 14 (23) → 98 (84) → 122 (24) → 124 (2) → 183 (59)
  总寻道距离: 236
```

### 优缺点

| 优点 | 缺点 |
|---|---|
| 比 FCFS 效率高 | 可能导致饥饿（远处的请求一直等不到） |
| 实现简单 | 不保证公平性 |
| 减少平均寻道时间 | |

## SCAN（电梯算法）

### 原理

磁头像电梯一样单向移动，处理沿途的所有请求，到达一端后反转方向。

```c
// SCAN 调度
void scan_schedule(int requests[], int n, int initial_position, int max_track) {
    int total_seek = 0;
    int current = initial_position;
    int direction = 1;  // 1=向内(增加), -1=向外(减少)
    
    printf("SCAN 调度顺序:\n");
    
    // 先向内扫描
    while (true) {
        int nearest = -1;
        int min_distance = INT_MAX;
        
        for (int i = 0; i < n; i++) {
            int dist = requests[i] - current;
            // 只考虑同方向的请求
            if (dist > 0 && dist < min_distance) {
                min_distance = dist;
                nearest = i;
            }
        }
        
        if (nearest == -1) break;  // 没有更多同方向的请求
        
        total_seek += abs(current - requests[nearest]);
        printf("  %d -> %d (寻道: %d)\n", 
               current, requests[nearest], abs(current - requests[nearest]));
        current = requests[nearest];
        requests[nearest] = -1;  // 标记已服务
    }
    
    // 反向到达另一端
    printf("  %d -> %d (到端点)\n", current, direction > 0 ? max_track : 0);
    current = direction > 0 ? max_track : 0;
    
    // 向外扫描
    direction = -direction;
    while (true) {
        int nearest = -1;
        int min_distance = INT_MAX;
        
        for (int i = 0; i < n; i++) {
            if (requests[i] == -1) continue;
            int dist = requests[i] - current;
            if (dist > 0 && dist < min_distance) {
                min_distance = dist;
                nearest = i;
            }
        }
        
        if (nearest == -1) break;
        
        total_seek += abs(current - requests[nearest]);
        printf("  %d -> %d (寻道: %d)\n", 
               current, requests[nearest], abs(current - requests[nearest]));
        current = requests[nearest];
        requests[nearest] = -1;
    }
    
    printf("总寻道距离: %d\n", total_seek);
}
```

### 示例

```
磁道请求: 98, 183, 37, 122, 14, 124, 65, 67
磁头初始位置: 53, 最大磁道: 199

SCAN (向内方向):
  53 → 65 (12) → 67 (2) → 98 (31) → 122 (24) → 124 (2) → 183 (59) → 199 (16)
  → 反向 → (略)
  总寻道距离: 146 + 199 = 345
```

### 优缺点

| 优点 | 缺点 |
|---|---|
| 请求响应时间均匀 | 磁头可能过度移动（到达最远端） |
| 比 SSTF 公平 | 端点的请求等待时间较长 |
| 吞吐量高 | |

## C-SCAN（Circular SCAN）

### 原理

磁头单向扫描（一端到另一端），到端后快速返回起点，重新开始扫描。不像 SCAN 那样在返回时也服务请求。

```
SCAN vs C-SCAN:

SCAN:
  53 → ... → 199 → 反向处理请求 → 0 → ...

C-SCAN:
  53 → ... → 199 → 快速返回 0 → 服务请求 → ...

```

### 示例

```
磁道请求: 98, 183, 37, 122, 14, 124, 65, 67
磁头初始位置: 53, 最大磁道: 199

C-SCAN:
  53 → 65 (12) → 67 (2) → 98 (31) → 122 (24) → 124 (2) → 183 (59) → 199 (16)
  → 快速返回 0 (199)
  → (从 0 开始向内服务剩余请求，无)
  总寻道距离: 146 + 199 = 345
```

### 优缺点

| 优点 | 缺点 |
|---|---|
| 请求响应时间更均匀 | 吞吐量略低于 SCAN |
| 没有 SCAN 的"偏向端点"问题 | 磁头返回时空跑 |

## LOOK

### 原理

SCAN 的改进版：不到达最远端，而是在最后一个请求处反转。

```c
// LOOK 调度
void look_schedule(int requests[], int n, int initial_position) {
    // 类似 SCAN，但不到达端点
    // 磁头只移动到最远的请求处，然后反向
}
```

### 示例

```
磁道请求: 98, 183, 37, 122, 14, 124, 65, 67
磁头初始位置: 53

LOOK:
  53 → 65 (12) → 67 (2) → 98 (31) → 122 (24) → 124 (2) → 183 (59)
  → 反向 → 67 (116) → 65 (2) → 37 (28) → 14 (23)
  总寻道距离: 236 + 169 = 405
```

## 算法对比

| 算法 | 思想 | 平均寻道 | 公平性 | 吞吐量 |
|---|---|---|---|---|
| **FCFS** | 先来先服务 | 较差 | 最公平 | 低 |
| **SSTF** | 最短寻道优先 | 最好 | 差（可能饥饿） | 高 |
| **SCAN** | 电梯算法 | 好 | 较公平 | 高 |
| **C-SCAN** | 单向电梯 | 较好 | 非常公平 | 高 |
| **LOOK** | SCAN 改进 | 好 | 较公平 | 高 |

## 实际系统中的应用

### Linux 的 I/O 调度器

Linux 提供了多种 I/O 调度器：

```bash
# 查看当前调度器
$ cat /sys/block/sda/queue/scheduler
noop anticipatory deadline [cfq]

# noop: 最简单，只做合并，不排序（适合 SSD）
# anticipatory: 预测性调度
# deadline: 基于期限的调度，保证请求的响应时间
# cfq (Completely Fair Queuing): 完全公平队列（Linux 2.6-4.x 默认）
# mq-deadline: 多队列期限调度器（Linux 4.x+）
# bfq (Budget Fair Queuing): 预算公平队列（适合桌面和交互）

# 更改调度器
$ echo "mq-deadline" > /sys/block/sda/queue/scheduler
```

### SSD 的特殊情况

SSD 没有机械部件（磁头、磁盘），所以传统磁盘调度算法意义不大。SSD 使用 **TRIM** 命令告诉控制器哪些块已删除，可以回收。

```bash
# 查看 SSD 是否支持 TRIM
$ hdparm -I /dev/sda | grep TRIM
        *    Data Set Management TRIM supported
```

## 面试高频问题

**Q1：为什么需要磁盘调度？**

磁盘操作的主要延迟是寻道时间。好的调度算法可以减少磁头移动距离，提高吞吐量。

**Q2：SSTF 和 SCAN 哪个更好？**

SSTF 平均寻道时间最短，但可能饿死远处的请求。SCAN 响应时间更均匀，更公平。实际系统中通常使用 SCAN 变体。

**Q3：SCAN 和 C-SCAN 有什么区别？**

SCAN 在往返途中都服务请求，C-SCAN 只在一个方向服务请求，到达端点后快速返回起点。C-SCAN 的响应时间更均匀。

**Q4：LOOK 算法解决了什么问题？**

LOOK 不移动到最远端，只移动到最远的请求处。这减少了不必要的磁头移动。

**Q5：现代 SSD 还需要磁盘调度吗？**

SSD 没有寻道时间，传统调度算法意义不大。但 SSD 仍需要调度来合并请求、平衡磨损。Linux 对 SSD 使用 noop 或 mq-deadline 调度器。

理解磁盘调度算法，是理解 I/O 系统性能优化的关键。
