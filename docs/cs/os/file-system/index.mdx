# 文件系统

你双击了一个文件，视频开始播放。这个过程看似简单，背后却经历了复杂的旅程：

文件在哪里？磁盘上。如何找到它？通过目录。如何读取？文件系统在起作用。

本章节将带你深入理解文件系统的工作原理：从文件如何存储，到目录如何组织，从磁盘调度算法，到文件系统一致性保障。这些知识不仅能帮助你理解操作系统，更能在实际开发中写出更高效的文件操作代码。

## 知识体系总览

**文件系统的结构** —— 文件系统如何组织磁盘？FCB 和索引节点（inode）是如何工作的？

**文件分配方式** —— 文件数据是如何存放在磁盘块中的？连续分配、链接分配、索引分配各有什么优缺点？

**目录实现** —— 目录本身也是一个文件，它是如何实现的？线性列表、哈希表、B+ 树，哪种方案更好？

**磁盘调度算法** —— 磁盘读写需要移动磁头臂，磁头移动的距离直接影响性能。FCFS、SCAN、C-SCAN……各种算法如何选择？

## 从点击到播放

当你双击一个视频文件时，操作系统经历了什么？

```
1. 用户双击文件
         │
         ▼
2. 根据文件名查找目录
   /home/user/Videos/movie.mp4
         │
         ▼
3. 读取文件的 inode
   inode 包含文件元数据和数据块位置
         │
         ▼
4. 根据 inode 中的块地址读取数据
   可能需要多次磁盘 I/O
         │
         ▼
5. 把数据交给播放器
         │
         ▼
6. 播放流畅进行
```

每一步都涉及复杂的磁盘操作，理解这些原理，才能写出高效的文件访问代码。

## 为什么需要文件系统？

**裸磁盘的问题：**

如果直接把数据写在磁盘上，会有以下问题：

1. **如何找到数据？** 没有索引，数据放哪都不知道
2. **如何组织多个文件？** 文件之间是什么关系？
3. **如何保护数据？** 谁能访问，谁不能访问？
4. **磁盘空间如何管理？** 哪些块是空闲的？

文件系统解决了这些问题：

| 问题 | 文件系统的解决方案 |
|---|---|
| 找到数据 | 目录 + inode 索引 |
| 组织文件 | 目录树结构 |
| 保护数据 | 权限控制（rwx） |
| 管理空间 | 空闲空间管理（位图/空闲链表） |

## 面试关联

文件系统是操作系统面试的常客：

- 文件系统有哪些结构？
- inode 是什么？包含哪些信息？
- 文件分配方式有哪些？各自的优缺点？
- 磁盘调度算法有哪些？如何选择？
- 什么是文件系统一致性？

这些问题看似简单，但能回答出「为什么」的人却不多。本章节不仅讲解「是什么」，更注重讲解「为什么」，帮助你真正理解文件系统的设计思想。
