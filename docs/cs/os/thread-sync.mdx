# 线程同步机制

> 目标级别：P5/P6

面试官问：「线程同步有哪些方式？」你回答「互斥锁、信号量」——然后面试官追问：「互斥锁和读写锁的区别？」「什么是死锁？怎么避免？」「 synchronized 和 ReentrantLock 有什么区别？」

线程同步是并发编程的核心问题，理解各种同步机制是写出正确多线程代码的基础。

## 一、为什么需要同步

### 1.1 竞态条件

```java
// 竞态条件示例
public class Counter {
    private int count = 0;

    public void increment() {
        count++;  // 不是原子操作！
    }
}

// 两个线程同时执行 count++：
// 1. 线程 A 读取 count=0
// 2. 线程 B 读取 count=0
// 3. 线程 A 写入 count=1
// 4. 线程 B 写入 count=1
// 结果：应该是 2，实际是 1
```

## 二、同步机制分类

### 2.1 互斥锁

| 类型 | 说明 |
|------|------|
| synchronized | Java 关键字，JVM 实现 |
| ReentrantLock | Java API，显式锁 |
| std::mutex | C++ 标准库 |

```java
// synchronized 示例
public class SafeCounter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int get() {
        return count;
    }
}

// ReentrantLock 示例
public class SafeCounterWithLock {
    private int count = 0;
    private final ReentrantLock lock = new ReentrantLock();

    public void increment() {
        lock.lock();
        try {
            count++;
        } finally {
            lock.unlock();  // 必须释放
        }
    }
}
```

### 2.2 读写锁

适用于读多写少场景。

```java
// ReadWriteLock 示例
public class Cache {
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
    private Map<String, Object> cache = new HashMap<>();

    public Object get(String key) {
        rwLock.readLock().lock();
        try {
            return cache.get(key);
        } finally {
            rwLock.readLock().unlock();
        }
    }

    public void put(String key, Object value) {
        rwLock.writeLock().lock();
        try {
            cache.put(key, value);
        } finally {
            rwLock.writeLock().unlock();
        }
    }
}
```

### 2.3 信号量

控制并发数量。

```java
// Semaphore 示例：限制 10 个并发
public class ConnectionPool {
    private final Semaphore sem = new Semaphore(10);

    public void connect() throws InterruptedException {
        sem.acquire();
        try {
            // 使用连接
        } finally {
            sem.release();
        }
    }
}
```

### 2.4 条件变量

等待某个条件成立。

```java
// Condition 示例：生产者消费者
public class BoundedBuffer<T> {
    private final Lock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();

    public void put(T item) throws InterruptedException {
        lock.lock();
        try {
            while (isFull()) {
                notFull.await();
            }
            doPut(item);
            notEmpty.signal();
        } finally {
            lock.unlock();
        }
    }

    public T take() throws InterruptedException {
        lock.lock();
        try {
            while (isEmpty()) {
                notEmpty.await();
            }
            T item = doTake();
            notFull.signal();
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

## 三、对比总结

### 同步机制对比

| 机制 | 特点 | 适用场景 |
|------|------|----------|
| synchronized | JVM 实现，自动释放 | 一般同步 |
| ReentrantLock | API，可定时，可中断 | 复杂同步 |
| ReadWriteLock | 读多写少 | 缓存 |
| Semaphore | 控制并发数 | 资源池 |
| Condition | 条件等待 | 生产者消费者 |

### synchronized vs ReentrantLock

| 维度 | synchronized | ReentrantLock |
|------|--------------|---------------|
| 获取锁 | 隐式 | 显式 |
| 释放 | 自动释放 | 必须 finally 释放 |
| 公平锁 | 否 | 支持 |
| 定时锁 | 否 | tryLock(timeout) |
| 可中断锁 | 否 | lockInterruptibly() |
| 多条件 | 否 | newCondition() |

## 四、面试题精讲

### 🔴 【高频】线程同步方式

**问题**：线程同步有哪些方式？

**标准答案**：

```
1. 互斥锁
   - synchronized（Java 关键字）
   - ReentrantLock（显式锁）
   - 保证同一时刻只有一个线程执行

2. 读写锁
   - 读共享，写独占
   - 适用于读多写少场景

3. 信号量
   - 控制并发数量
   - P/V 操作

4. 条件变量
   - 线程等待某个条件
   - await/signal

5. volatile
   - 保证可见性
   - 不保证原子性

6. 原子变量
   - AtomicInteger 等
   - CAS 保证原子性
```

### 🟡 【中频】死锁及避免

**问题**：什么是死锁？如何避免？

**标准答案**：

```
死锁的四个必要条件：
1. 互斥条件：资源只能被一个线程使用
2. 持有并等待：持有资源同时请求其他资源
3. 不可抢占：资源不能被强制释放
4. 循环等待：形成循环等待链

避免死锁的方法：
1. 固定加锁顺序
   - 多把锁按固定顺序获取
2. 使用定时锁
   - lock.tryLock(timeout)
3. 减小锁粒度
   - 减少锁的持有时间
4. 使用高层并发工具
   - ConcurrentHashMap、BlockingQueue
```

## 五、扩展思考

### 💡 无锁编程

```
无锁编程使用 CAS（Compare-And-Swap）：

AtomicInteger 示例：
public class AtomicCounter {
    private final AtomicInteger count = new AtomicInteger(0);

    public void increment() {
        count.incrementAndGet();  // CAS 实现
    }
}

CAS 原理：
1. 读取当前值
2. 计算新值
3. CAS 更新，如果期间值被修改则重试
```

> 线程同步是并发编程的基础。选择合适的同步机制需要考虑：性能要求、代码复杂度、场景特点等因素。
