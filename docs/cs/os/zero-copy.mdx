# 零拷贝技术

> 目标级别：P6

面试官问：「什么是零拷贝？」你回答「不需要 CPU 拷贝」——然后面试官追问：「mmap 和 sendfile 的区别？」「Kafka/RocketMQ 怎么实现零拷贝？」「DMA 是什么？」

零拷贝是高性能 IO 的核心技术，理解零拷贝对于设计高性能系统至关重要。

## 一、传统 IO 的问题

### 1.1 数据拷贝次数

```
传统 read/write IO 的数据拷贝：

1. 磁盘 → 内核缓冲区（DMA 拷贝）
2. 内核缓冲区 → 用户缓冲区（CPU 拷贝）
3. 用户缓冲区 → Socket 缓冲区（CPU 拷贝）
4. Socket 缓冲区 → 网卡（DMA 拷贝）

总计：4 次拷贝，2 次 CPU 拷贝，2 次 DMA 拷贝
```

### 1.2 上下文切换

```
传统 IO 的上下文切换：

1. 用户态 → 内核态（read）
2. 内核态 → 用户态（read 返回）
3. 用户态 → 内核态（write）
4. 内核态 → 用户态（write 返回）

总计：4 次上下文切换
```

## 二、零拷贝实现

### 2.1 mmap

```
mmap（内存映射）：

将文件映射到内存，减少一次 CPU 拷贝：

1. 磁盘 → 内核缓冲区（DMA）
2. 内核缓冲区 → Socket 缓冲区（CPU）
3. Socket 缓冲区 → 网卡（DMA）

总计：3 次拷贝，1 次 CPU 拷贝

优点：减少一次 CPU 拷贝
缺点：仍然需要 CPU 参与
```

### 2.2 sendfile

```c
// sendfile 系统调用
#include <sys/sendfile.h>

ssize_t sendfile(int out_fd, int in_fd, off_t *offset, size_t count);

// 数据拷贝过程：
1. 磁盘 → 内核缓冲区（DMA）
2. 内核缓冲区 → 网卡（DMA，通过描述符）
3. Socket 缓冲区 → 网卡（DMA）

总计：2 次 DMA 拷贝，0 次 CPU 拷贝
```

### 2.3 Linux 零拷贝演进

```
Linux 零拷贝技术演进：

1. 传统 read/write：4 次拷贝
2. mmap：3 次拷贝
3. sendfile：2 次拷贝（Linux 2.4+）
4. splice：2 次拷贝
5. io_uring：真正的异步 IO
```

## 三、DMA

### 3.1 什么是 DMA

```
DMA（Direct Memory Access）：
- 允许硬件设备直接访问内存
- 不需要 CPU 参与数据传输
- CPU 只在传输开始和结束时介入

DMA 控制器：
- 独立于 CPU 的硬件设备
- 控制内存和外设之间的数据传输
```

## 四、面试题精讲

### 🔴 【高频】零拷贝原理

**问题**：什么是零拷贝？它是怎么实现的？

**标准答案**：

```
零拷贝：避免数据在用户态和内核态之间的重复拷贝

传统 IO 问题：
- 4 次数据拷贝
- 4 次上下文切换

零拷贝实现：
1. mmap：将文件映射到内存，减少一次 CPU 拷贝
2. sendfile：内核直接转发，2 次 DMA 拷贝，0 次 CPU 拷贝

关键：
- 数据直接从内核缓冲区到网卡
- 不经过用户缓冲区
- 减少 CPU 参与和上下文切换
```

### 🟡 【中频】Kafka/RocketMQ 零拷贝

**问题**：消息队列是怎么实现零拷贝的？

**标准答案**：

```
Kafka 零拷贝：

1. 使用 sendfile
   - 文件数据直接发送到网卡
   - 不经过应用层缓冲区

2. 消费时
   - 使用 transferTo
   - 从 PageCache 直接发送到网卡

RocketMQ 类似：
- 使用 MappedByteBuffer（mmap）
- 消费时使用 sendfile

效果：
- 高吞吐量
- 低 CPU 占用
```

## 六、扩展思考

### 💡 io_uring

```
io_uring（Linux 5.1）：
- 新一代异步 IO 接口
- 两个环形缓冲区：Submission Queue 和 Completion Queue
- 支持真正的异步 IO
- 性能优于传统接口
```

> 零拷贝是高性能 IO 的核心技术。Kafka、RocketMQ 等高性能消息队列都大量使用零拷贝技术来提升吞吐量。
