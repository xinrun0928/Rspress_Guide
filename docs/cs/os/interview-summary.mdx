# 操作系统面试高频问题汇总

操作系统是后端面试的重灾区，尤其是进程线程、内存管理、文件系统三个模块。

本章节汇总高频面试题，帮你快速梳理知识体系，查漏补缺。

## 进程与线程

### Q1：进程和线程的区别？

| 对比维度 | 进程 | 线程 |
|---|---|---|
| 资源分配 | 独立的地址空间 | 共享进程资源 |
| 开销 | 创建/切换开销大 | 创建/切换开销小 |
| 通信 | 需借助 IPC | 直接共享内存 |
| 隔离性 | 高隔离 | 低隔离 |
| 并行性 | 多进程可实现多核并行 | 多线程可实现多核并行 |

### Q2：进程间有哪些通信方式？

1. **管道**：面向字节流，血缘进程间通信
2. **命名管道（FIFO）**：无血缘进程间通信
3. **消息队列**：消息链表，持久化
4. **共享内存**：最快的 IPC，需要同步
5. **信号量**：计数器，用于同步
6. **Socket**：本机和网络通信

### Q3：线程同步方式有哪些？

1. **互斥锁**：排他访问
2. **读写锁**：读多写少场景
3. **自旋锁**：忙等待，适用临界区极短
4. **条件变量**：等待通知
5. **信号量**：计数器同步

### Q4：什么是死锁？死锁的必要条件？

**死锁**：两个或多个进程相互等待对方持有的资源，形成循环等待。

**四个必要条件**：
1. **互斥条件**：资源只能被一个进程使用
2. **占有并等待**：进程持有资源的同时请求其他资源
3. **不可抢占条件**：已分配资源不能被强制夺走
4. **循环等待条件**：形成进程-资源的循环链

### Q5：如何预防死锁？

- 破坏互斥条件：共享资源
- 破坏占有并等待：一次性申请所有资源
- 破坏不可抢占：强制抢占资源
- 破坏循环等待：资源排序，按顺序获取

## 内存管理

### Q6：分页和分段有什么区别？

| 对比 | 分页 | 分段 |
|---|---|---|
| 单位大小 | 固定 | 变量 |
| 碎片类型 | 内部碎片 | 外部碎片 |
| 逻辑意义 | 无 | 有（代码段、数据段等） |
| 视角 | 程序员看到连续的虚拟地址 | 程序员看到逻辑段 |

### Q7：页面置换算法有哪些？

| 算法 | 说明 | 特点 |
|---|---|---|
| FIFO | 先进先出 | 简单，可能有 Belady 异常 |
| LRU | 最近最少使用 | 效果好，需要硬件支持 |
| LFU | 最不经常使用 | 考虑访问频率 |
| Clock | 第二次 chance | LRU 近似实现 |

### Q8：什么是虚拟内存？有什么作用？

虚拟内存让程序使用比物理内存更大的地址空间，通过将不活跃的页面换出到磁盘实现。作用：
1. 扩大可用地址空间
2. 实现内存保护
3. 简化内存分配
4. 实现共享内存

### Q9：什么是局部性原理？

- **时间局部性**：最近访问的地址可能再次访问
- **空间局部性**：访问某地址后倾向于访问附近地址

CPU 缓存、虚拟内存的页面置换都基于局部性原理。

### Q10：什么是抖动？如何解决？

抖动是物理内存不足时，频繁的页面换入换出，导致 CPU 大部分时间在等待 I/O。

解决方法：
- 增加物理内存
- 优化程序局部性
- 减少并发进程数
- 调整页面置换算法

## 文件系统

### Q11：文件系统有哪些组成部分？

1. **超级块**：文件系统元信息
2. **inode 位图**：inode 使用情况
3. **块位图**：数据块使用情况
4. **inode 表**：文件元信息
5. **数据块**：文件实际内容

### Q12：硬链接和软链接的区别？

| 对比 | 硬链接 | 软链接 |
|---|---|---|
| 本质 | 同一 inode 的多个目录项 | 独立 inode，内容是路径 |
| 跨文件系统 | 否 | 是 |
| 链接目录 | 否 | 是 |
| 源文件删除 | 链接仍有效 | 链接失效 |

### Q13：文件分配方式有哪些？

1. **连续分配**：顺序访问快，有外部碎片
2. **链接分配**：无碎片，随机访问慢
3. **索引分配**：支持随机访问，索引块占空间

### Q14：磁盘调度算法有哪些？

| 算法 | 说明 | 特点 |
|---|---|---|
| FCFS | 先来先服务 | 公平，效果差 |
| SSTF | 最短寻道时间优先 |效果好，可能饥饿 |
| SCAN | 电梯算法 | 向一个方向扫描 |
| C-SCAN | 单向扫描后返回 | 响应时间均匀 |
| LOOK | 到达最远请求后返回 | 比 SCAN 优化 |

## 操作系统原理

### Q15：用户态和内核态的区别？

- **用户态**：低特权级，只能访问受限资源
- **内核态**：高特权级，可访问所有硬件和内存

系统调用、中断、异常会触发用户态到内核态的切换。

### Q16：什么是中断？中断的处理流程？

中断是 CPU 暂停当前执行流，转去处理突发事件。

**处理流程**：
1. 中断发生，CPU 保存状态
2. 根据中断号查中断向量表
3. 执行中断处理程序
4. 恢复状态，继续执行

### Q17：DMA 是什么？有什么优点？

DMA（直接内存访问）是 DMA 控制器在设备和内存之间直接传输数据，不需要 CPU 介入。

优点：减少 CPU 负担，减少中断次数，提高系统性能。

### Q18：什么是银行家算法？

死锁避免算法，在资源分配前检查分配后系统是否仍处于安全状态。

核心：找到一个安全序列，使得所有进程都能完成。

## Linux

### Q19：Linux 进程和线程的区别？

Linux 中进程和线程都是 task_struct，区别在于共享资源的程度。线程共享地址空间、文件描述符等；进程拥有独立的资源。

### Q20：什么是 OOM Killer？

OOM Killer（Out-of-Memory Killer）在物理内存不足时，选择并杀死进程释放内存。被选中的是 oom_score 最高的进程。

### Q21：如何排查服务器负载高的问题？

```bash
# 1. 查看系统负载
uptime / top / w

# 2. 查看 CPU 使用
vmstat 1

# 3. 查看内存使用
free -h

# 4. 查看 I/O
iostat -x 1

# 5. 找到高负载进程
top -c
```

### Q22：Linux 如何实现内存管理？

- **伙伴系统**：以 2 的幂次方分配内存页，解决外部碎片
- **slab 分配器**：为小对象分配内存，减少内部碎片
- **vmalloc**：用于大块内存映射
- **OOM Killer**：内存不足时杀死进程

## 面试技巧

### 如何回答「进程和线程的区别」

不要只背答案，要展示理解深度：

1. **基础层面**：资源分配 vs CPU 调度
2. **实现层面**：Linux 中线程和进程都是 task_struct
3. **实际场景**：什么时候用进程，什么时候用线程
4. **潜在问题**：线程安全、死锁等

### 如何回答「什么是死锁」

从生活场景切入，然后给出技术定义：

> 就像两个和尚挑水喝，三个和尚没水喝。进程 A 持有资源 1 等资源 2，进程 B 持有资源 2 等资源 1，形成循环等待。

然后回答四个必要条件。

### 如何回答「什么是虚拟内存」

先解释是什么，再解释为什么：

> 虚拟内存为每个进程提供独立的连续地址空间。程序以为自己有 4GB 空间，实际上只是使用了一部分物理内存和磁盘。解决了地址空间隔离、内存扩展、共享等问题。
