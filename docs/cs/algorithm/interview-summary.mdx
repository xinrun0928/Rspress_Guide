# 算法高频面试题汇总

面试中的算法题，表面上考的是「解题能力」，实际上考的是「思维方式」。面试官想看到的，不是你背了多少答案，而是你面对问题时如何思考、如何分析、如何优化。

本章节汇总了算法方向的高频面试题，帮助你系统性地复习和准备。

## 面试核心技巧

### 1. 理解题意再动手

:::tip 关键提问

面试官出一道题，首先确认以下信息：

1. 输入的数据范围是多少？
2. 是否有特殊约束（如空间复杂度要求）？
3. 返回值是什么？索引还是值？
4. 如何处理边界情况？

不要急着写代码，先和面试官确认清楚。
:::

### 2. 先说思路，再写代码

正确的面试流程：

```
1. 分析问题，确定算法思路（1-2分钟）
2. 描述算法思路和大致复杂度（30秒）
3. 询问面试官确认（是否正确）
4. 写代码实现（5-10分钟）
5. 测试几个用例验证（1-2分钟）
```

### 3. 代码要有「面试感」

| 要求 | 说明 |
|------|------|
| 变量命名清晰 | 用 `left`、`right`、`res` 等有意义的名字 |
| 代码结构整洁 | 适当空行分隔逻辑块 |
| 注释关键步骤 | 解释「为什么」而不是「是什么」 |
| 处理边界情况 | null、空数组等情况 |

## 常见面试追问

### 关于复杂度

- 这个算法的时间/空间复杂度是多少？
- 能不能优化？还能更好吗？
- 最坏情况和平均情况分别是什么？

### 关于实现

- 如果数据量扩大 100 倍，你的算法还能工作吗？
- 如果要求 O(1) 空间呢？
- 递归会不会导致栈溢出？怎么改写成迭代？

### 关于延伸

- 这个算法在实际项目中用过吗？
- 如果让你设计一个类似的系统，你会怎么做？

## 复习建议

:::tip 系统性复习路线

**第一阶段（1-2周）：基础打牢**

- 数组和链表：双指针、滑动窗口
- 二叉树：遍历、常见题型
- 哈希表：基本应用

**第二阶段（2-3周）：重点突破**

- 回溯和 DFS/BFS
- 动态规划（从简单到困难）
- 最短路径和拓扑排序

**第三阶段（考前冲刺）：**

- 薄弱环节专项练习
- 高频题多遍复习
- 模拟面试找感觉

:::

## 心态调整

面试算法题时，心态比实力更重要。

| 常见心态 | 正确认识 |
|----------|----------|
| 遇到难题就慌 | 难题是加分题，做不出来也能过 |
| 一定要写出最优解 | 暴力解法先写出来，再优化 |
| 担心面试官在盯着看 | 把面试官当成一起讨论的同事 |
| 这题没见过 | 所有题都是基础题的组合 |

**面试是双向选择**：你在展示能力，面试官在判断你是否合适。保持自信，展示你的思考过程，即使没做出来，也要让面试官看到你的潜力。
