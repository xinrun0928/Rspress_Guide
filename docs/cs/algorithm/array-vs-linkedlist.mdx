# 数组 vs 链表

> 目标级别：P5

面试官问：「数组和链表有什么区别？」你回答「数组连续，链表不连续」——然后面试官追问：「为什么数组查询快？」「为什么链表插入快？」「LRU 缓存用什么实现？」

数组和链表是最基础的数据结构，理解它们的区别是学习其他数据结构的基础。

## 一、数组（Array）

### 1.1 定义与特点

```
数组特点：
- 连续内存空间
- 通过下标随机访问
- 固定大小（静态数组）
- 动态数组（ArrayList）可自动扩容
```

### 1.2 数组操作复杂度

| 操作 | 复杂度 | 说明 |
|------|---------|------|
| 访问（下标） | O(1) | 随机访问 |
| 查找（无序） | O(n) | 遍历 |
| 查找（二分） | O(log n) | 有序数组 |
| 插入 | O(n) | 需要移动元素 |
| 删除 | O(n) | 需要移动元素 |
| 尾部插入 | O(1) | amortized |

---

## 二、链表（LinkedList）

### 2.1 定义与特点

```
链表特点：
- 不连续内存空间
- 通过指针连接节点
- 大小动态变化
- 节点包含数据和指针
```

### 2.2 单链表 vs 双链表

```java
// 单链表节点
class ListNode {
    int val;
    ListNode next;
}

// 双链表节点
class DListNode {
    int val;
    DListNode prev;
    DListNode next;
}
```

### 2.3 链表操作复杂度

| 操作 | 单链表 | 双链表 |
|------|--------|--------|
| 访问 | O(n) | O(n) |
| 头部插入 | O(1) | O(1) |
| 尾部插入 | O(n) | O(1) |
| 头部删除 | O(1) | O(1) |
| 尾部删除 | O(n) | O(1) |
| 任意位置插入 | O(n) | O(n) |

---

## 三、核心区别

### 3.1 内存布局

```
数组：[0][1][2][3][4]  （连续内存）

链表：  [0] -> [1] -> [2] -> [3] -> [4]  （不连续）
        0x100   0x200   0x150   0x300   0x250
```

### 3.2 性能对比

| 维度 | 数组 | 链表 |
|------|------|------|
| 访问 | O(1) | O(n) |
| 插入/删除（头部） | O(n) | O(1) |
| 插入/删除（尾部） | O(1) amortized | O(1) |
| 插入/删除（中间） | O(n) | O(n) |
| 内存利用率 | 高（无额外开销） | 低（需要存储指针） |
| CPU 缓存 | 友好（连续内存） | 不友好（跳转多） |

## 四、面试题精讲

### 🔴 【高频】数组和链表的区别

**问题**：数组和链表有什么区别？各适合什么场景？

**标准答案**：

```
核心区别：
1. 内存布局
   - 数组：连续内存，支持 CPU 缓存预取
   - 链表：不连续，通过指针连接

2. 访问性能
   - 数组：O(1) 随机访问
   - 链表：O(n) 需要遍历

3. 插入/删除
   - 数组：O(n) 需要移动元素
   - 链表：O(1) 只改指针（已知位置）

4. 内存
   - 数组：固定大小，无额外开销
   - 链表：需要额外存储指针

适用场景：
- 数组：频繁随机访问，CPU 缓存友好
- 链表：频繁插入/删除，大小动态变化
```

### 🟡 【中频】LRU 缓存实现

**问题**：LRU 缓存用什么数据结构实现？为什么？

**标准答案**：

```
LRU 缓存实现：HashMap + 双向链表

- HashMap：O(1) 查找
- 双向链表：O(1) 插入/删除

操作：
1. 查找：HashMap 找到节点，移动到头部
2. 插入：HashMap 添加，插入链表头部
3. 删除：HashMap 删除，链表尾部删除

Java LinkedHashMap 直接支持这个功能。
```

## 五、对比总结

### 数组 vs 链表

| 维度 | 数组 | 链表 |
|------|------|------|
| 内存 | 连续 | 不连续 |
| 访问 | O(1) | O(n) |
| 头部插入 | O(n) | O(1) |
| 尾部插入 | O(1)* | O(1) |
| 中间插入 | O(n) | O(n) |
| 内存开销 | 无 | 指针开销 |
| CPU 缓存 | 友好 | 不友好 |

## 六、扩展思考

### 💡 CPU 缓存与数组

```
CPU 缓存预取（Cache Prefetching）：

连续内存的数组可以触发预取：
- CPU 预测将要访问的数据
- 提前加载到缓存
- 访问时不需要等待内存

这就是为什么遍历数组比遍历链表快：
- 数组：顺序访问，触发预取
- 链表：随机访问，每次都需等待内存
```

### 💡 跳表（Skip List）

```
跳表是链表 + 索引的结合：

        1 ───────────────────────────→ 9
        ↑                               ↑
        1 ───→ 3 ────────────────→ 7 → 9
        ↑       ↑                   ↑   ↑
        1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9

查找：O(log n)（类似二分查找）
空间：O(n log n)

Redis ZSet 底层使用跳表。
```

> 数组和链表是数据结构的基础。理解它们的优缺点，才能在不同的场景选择合适的数据结构。实际开发中，数组（Vector/ArrayList）是最常用的，只有在频繁插入/删除时才考虑链表。
