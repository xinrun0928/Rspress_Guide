# 并查集

**如何在 O(1) 时间内判断两个人是否在同一个朋友圈？**

朋友圈会不断合并：今天 A 和 B 成了朋友，明天 B 和 C 也成了朋友。如何快速维护这种动态连通关系？

这就是**并查集**（Union-Find）要解决的问题。

## 核心概念

### 什么是并查集

并查集是一种「分组」的数据结构，支持：

- **Union(x, y)**：将 x 和 y 所在的集合合并
- **Find(x)**：找到 x 所在集合的代表元素

### 核心思想

每个元素指向一个「父节点」，父节点指向自己时就是代表。最终，所有同一集合的元素都指向同一个代表。

### 应用场景

| 场景 | 说明 |
|------|------|
| 朋友圈 | 朋友之间自动连通 |
| 岛屿问题 | 相邻的 1 连成一片 |
| 最小生成树 | Kruskal 算法 |
| 动态连通性 | 网络连接、变量等价 |

## 基本实现

### 朴素版本

```java
class UnionFind {
    int[] parent;

    UnionFind(int n) {
        parent = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;  // 每个节点的父节点是自己
        }
    }

    // 查找：找到 x 的根节点
    int find(int x) {
        while (parent[x] != x) {
            x = parent[x];
        }
        return x;
    }

    // 合并：将 x 和 y 合并到同一集合
    void union(int x, int y) {
        int px = find(x);
        int py = find(y);
        if (px != py) {
            parent[px] = py;  // px 的父节点指向 py
        }
    }

    // 判断是否连通
    boolean connected(int x, int y) {
        return find(x) == find(y);
    }
}
```

## 优化：路径压缩

### 问题

树可能会退化：每次 Union 都把一个集合的根接到另一个上，导致树变深，Find 变慢。

### 解决方案

在 Find 过程中，让每个节点直接指向根节点。

```java
int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);  // 递归压缩路径
    }
    return parent[x];
}
```

:::tip 路径压缩的效果

路径压缩后，树的高度变成 2：所有节点都直接指向根节点。

Find 操作接近 O(1)。

:::

## 优化：按秩合并

### 核心思想

合并时，把较小的树接到较大的树下，避免树变深。

```java
class UnionFind {
    int[] parent;
    int[] rank;  // 秩：树的深度上界

    UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            rank[i] = 1;
        }
    }

    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }

    void union(int x, int y) {
        int px = find(x);
        int py = find(y);

        if (px == py) return;  // 已经在同一集合

        // 按秩合并
        if (rank[px] < rank[py]) {
            parent[px] = py;
        } else if (rank[px] > rank[py]) {
            parent[py] = px;
        } else {
            parent[py] = px;
            rank[px]++;
        }
    }
}
```

## 复杂度分析

| 操作 | 朴素 | +路径压缩 | +按秩合并 |
|------|------|-----------|-----------|
| Find | O(n) | O(α(n)) | O(α(n)) |
| Union | O(n) | O(α(n)) | O(α(n)) |
| Connected | O(n) | O(α(n)) | O(α(n)) |

其中 α(n) 是**Ackermann 函数的反函数**，对于实际应用，α(n) `<=` 4，几乎等于 O(1)。

## 经典应用

### 1. 朋友圈（LeetCode 547）

```java
public int findCircleNum(int[][] isConnected) {
    int n = isConnected.length;
    UnionFind uf = new UnionFind(n);

    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if (isConnected[i][j] == 1) {
                uf.union(i, j);
            }
        }
    }

    int count = 0;
    for (int i = 0; i < n; i++) {
        if (uf.find(i) == i) count++;
    }
    return count;
}
```

### 2. 岛屿数量（LeetCode 200）

```java
public int numIslands(char[][] grid) {
    if (grid == null || grid.length == 0) return 0;

    int m = grid.length, n = grid[0].length;
    UnionFind uf = new UnionFind(m * n);

    int[][] dirs = {{1, 0}, {0, 1}};  // 只往右和往下，避免重复
    int waterCount = 0;

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (grid[i][j] == '0') {
                waterCount++;
                continue;
            }

            int id = i * n + j;
            // 合并相邻的 1
            if (i + 1 < m && grid[i + 1][j] == '1') {
                uf.union(id, (i + 1) * n + j);
            }
            if (j + 1 < n && grid[i][j + 1] == '1') {
                uf.union(id, i * n + j + 1);
            }
        }
    }

    // 独立岛屿数 = 总数 - 水格子数（但同一岛屿被合并）
    // 或者直接数有多少个不同的根
    Set<Integer> roots = new HashSet<>();
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (grid[i][j] == '1') {
                roots.add(uf.find(i * n + j));
            }
        }
    }
    return roots.size();
}
```

### 3. 图的连通分量数

```java
public int countComponents(int n, int[][] edges) {
    UnionFind uf = new UnionFind(n);
    for (int[] e : edges) {
        uf.union(e[0], e[1]);
    }
    // 找有多少个不同的根
    Set<Integer> roots = new HashSet<>();
    for (int i = 0; i < n; i++) {
        roots.add(uf.find(i));
    }
    return roots.size();
}
```

## 面试模板

```java
class UnionFind {
    int[] parent;
    int[] rank;

    public UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    public int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);  // 路径压缩
        }
        return parent[x];
    }

    public boolean union(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false;

        // 按秩合并
        if (rank[px] < rank[py]) {
            parent[px] = py;
        } else if (rank[px] > rank[py]) {
            parent[py] = px;
        } else {
            parent[py] = px;
            rank[px]++;
        }
        return true;
    }
}
```

## 面试追问

1. **为什么需要路径压缩？**
   - 避免树退化成链表，导致 Find 变慢
   - 压缩后几乎 O(1)

2. **按秩合并的「秩」是什么？**
   - 可以是树的高度，也可以是节点数
   - 两者效果类似，都是防止大树被小树「吃」

3. **路径压缩和按秩合并能同时用吗？**
   - 能！而且通常同时使用
   - 这就是「几乎 O(1)」的来源

**记住：并查集的核心是「分而治之」。路径压缩 + 按秩合并，让它成为处理动态连通性问题的利器。**
