# 最短路径

**导航软件是怎么知道从 A 到 B 的最短路的？**

它不会枚举所有路径——那太多了。它用的是最短路径算法。在地图、路由、网络延迟分析等场景中，最短路径问题无处不在。

## 三种经典算法

| 算法 | 时间复杂度 | 适用场景 |
|------|------------|----------|
| **Dijkstra** | O(V²) 或 O(E log V) | 非负权重、单元源最短路 |
| **Bellman-Ford** | O(VE) | 可以有负权、检测负环 |
| **Floyd-Warshall** | O(V³) | 多源最短路、所有节点对 |

## Dijkstra 算法

### 核心思想

贪心策略：每次选择当前已知最短距离的未访问节点。

:::warning 关键前提

**不能有负权边！** 如果存在负权边，贪心策略会失效。

:::

### 实现

```java
public class Dijkstra {
    public int[] dijkstra(int[][] graph, int src) {
        int n = graph.length;
        int[] dist = new int[n];
        boolean[] visited = new boolean[n];

        // 初始化距离为无穷大
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        for (int i = 0; i < n; i++) {
            // 1. 找未访问节点中距离最小的
            int u = -1;
            for (int j = 0; j < n; j++) {
                if (!visited[j] && (u == -1 || dist[j] < dist[u])) {
                    u = j;
                }
            }

            if (dist[u] == Integer.MAX_VALUE) break;  // 无法到达
            visited[u] = true;

            // 2. 用这个节点更新其他节点的距离
            for (int v = 0; v < n; v++) {
                if (!visited[v] && graph[u][v] != Integer.MAX_VALUE) {
                    int newDist = dist[u] + graph[u][v];
                    if (newDist < dist[v]) {
                        dist[v] = newDist;
                    }
                }
            }
        }
        return dist;
    }
}
```

### 堆优化版本

```java
public int[] dijkstraWithHeap(int[][] graph, int src) {
    int n = graph.length;
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;

    // 小顶堆：(距离, 节点)
    PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
    pq.offer(new int[]{0, src});

    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int d = cur[0], u = cur[1];

        if (d > dist[u]) continue;  // 跳过无效的旧条目
        visited[u] = true;

        for (int v = 0; v < n; v++) {
            if (graph[u][v] != Integer.MAX_VALUE) {
                int newDist = dist[u] + graph[u][v];
                if (newDist < dist[v]) {
                    dist[v] = newDist;
                    pq.offer(new int[]{newDist, v});
                }
            }
        }
    }
    return dist;
}
```

### LeetCode 实战：网络延迟时间（743）

```java
public int networkDelayTime(int[][] times, int n, int k) {
    // 构建邻接表
    List<int[]>[] graph = new ArrayList[n + 1];
    for (int i = 1; i <= n; i++) graph[i] = new ArrayList<>();

    for (int[] t : times) {
        graph[t[0]].add(new int[]{t[1], t[2]});
    }

    int[] dist = new int[n + 1];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[k] = 0;

    PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
    pq.offer(new int[]{0, k});

    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int d = cur[0], u = cur[1];

        if (d > dist[u]) continue;

        for (int[] edge : graph[u]) {
            int v = edge[0], w = edge[1];
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.offer(new int[]{dist[v], v});
            }
        }
    }

    int ans = 0;
    for (int i = 1; i <= n; i++) {
        ans = Math.max(ans, dist[i]);
    }
    return ans == Integer.MAX_VALUE ? -1 : ans;
}
```

## Bellman-Ford 算法

### 核心思想

动态规划思想：对所有边进行 V-1 轮松弛操作。

### 为什么能处理负权边？

因为每次松弛操作只考虑「一步可达」的改进，多轮松弛后能得到最优解。

### 实现

```java
public class BellmanFord {
    public int[] bellmanFord(int[][] edges, int n, int src) {
        int[] dist = new int[n];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // 松弛 V-1 次
        for (int i = 0; i < n - 1; i++) {
            boolean updated = false;
            for (int[] edge : edges) {
                int u = edge[0], v = edge[1], w = edge[2];
                if (dist[u] != Integer.MAX_VALUE && dist[u] + w < dist[v]) {
                    dist[v] = dist[u] + w;
                    updated = true;
                }
            }
            if (!updated) break;  // 没有更新，提前结束
        }

        // 第 V 次检查负环
        for (int[] edge : edges) {
            int u = edge[0], v = edge[1], w = edge[2];
            if (dist[u] != Integer.MAX_VALUE && dist[u] + w < dist[v]) {
                System.out.println("存在负环");
            }
        }
        return dist;
    }
}
```

## Floyd-Warshall 算法

### 核心思想

动态规划：`dp[k][i][j]` 表示经过前 k 个节点后，i 到 j 的最短距离。

### 实现

```java
public class FloydWarshall {
    public int[][] floydWarshall(int[][] graph) {
        int n = graph.length;
        int[][] dist = new int[n][n];

        // 初始化
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (i == j) dist[i][j] = 0;
                else if (graph[i][j] != 0) dist[i][j] = graph[i][j];
                else dist[i][j] = Integer.MAX_VALUE / 2;
            }
        }

        // 三重循环
        for (int k = 0; k < n; k++) {
            for (int i = 0; i < n; i++) {
                for (int j = 0; j < n; j++) {
                    dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
                }
            }
        }
        return dist;
    }
}
```

## 算法选择

:::tip 选择指南

**单源最短路，非负权重** → Dijkstra（堆优化）

**单源最短路，可能有负权重** → Bellman-Ford

**多源最短路** → Floyd-Warshall 或 n 次 Dijkstra

:::

## 面试追问

1. **为什么 Dijkstra 不能处理负权边？**
   - 贪心策略：选择当前最短路径后，不会再更新它
   - 负权边可能导致「绕路更近」的情况

2. **Bellman-Ford 为什么只松弛 V-1 次？**
   - 最长最短路径最多有 V-1 条边（无环）
   - 第 V 次还有更新，说明存在负环

3. **Floyd-Warshall 的空间优化？**
   - 可以压缩到二维数组，因为 `dp[k]` 只依赖 `dp[k-1]`

**记住：没有最好的算法，只有最合适的算法。理解原理，才能灵活应用。**
