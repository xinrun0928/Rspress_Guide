# 图的表示

**面试官问：「如何存储一个社交网络？」**

你会怎么回答？直接说「用图」，那只是理解了问题。真正的高手，会进一步分析：「用户数量级是多少？需要频繁查询两人的关系吗？需要遍历某个用户的所有好友吗？」

不同的场景，要用不同的存储方式。这就是**邻接矩阵**和**邻接表**的核心区别。

## 两种表示方式

### 邻接矩阵

用一个二维数组 `matrix`，`matrix[i][j] = 1` 表示节点 i 到节点 j 有边相连。

```java
public class AdjMatrix {
    int[][] matrix;
    int n;  // 节点数

    public AdjMatrix(int n) {
        this.n = n;
        this.matrix = new int[n][n];
    }

    // 添加无向边
    public void addEdge(int i, int j) {
        matrix[i][j] = 1;
        matrix[j][i] = 1;
    }

    // 添加有向边
    public void addDirectedEdge(int i, int j) {
        matrix[i][j] = 1;
    }

    // 查询两点是否相连
    public boolean hasEdge(int i, int j) {
        return matrix[i][j] == 1;
    }

    // 获取某节点的所有邻居
    public List<Integer> neighbors(int i) {
        List<Integer> result = new ArrayList<>();
        for (int j = 0; j < n; j++) {
            if (matrix[i][j] == 1) {
                result.add(j);
            }
        }
        return result;
    }
}
```

### 邻接表

用链表（或其他结构）存储每个节点的邻居。

```java
public class AdjList {
    List<Integer>[] adj;
    int n;

    public AdjList(int n) {
        this.n = n;
        this.adj = new ArrayList[n];
        for (int i = 0; i < n; i++) {
            adj[i] = new ArrayList<>();
        }
    }

    // 添加无向边
    public void addEdge(int i, int j) {
        adj[i].add(j);
        adj[j].add(i);
    }

    // 添加有向边
    public void addDirectedEdge(int i, int j) {
        adj[i].add(j);
    }

    // 查询两点是否相连
    public boolean hasEdge(int i, int j) {
        return adj[i].contains(j);
    }

    // 获取某节点的所有邻居 - 直接返回
    public List<Integer> neighbors(int i) {
        return adj[i];
    }
}
```

## 对比分析

| 维度 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| **空间复杂度** | O(V²) | O(V + E) |
| **查询两点是否相连** | O(1) | O(deg(v)) |
| **遍历某节点的所有边** | O(V) | O(deg(v)) |
| **适用场景** | 稠密图、需要快速判断连通性 | 稀疏图、需要遍历邻居 |

:::tip 空间换时间

邻接矩阵用 O(V²) 的空间，换来了 O(1) 的边查询。

邻接表用 O(V + E) 的空间，换来了遍历邻居的效率。

:::

## 实际选择

### 选择邻接矩阵的场景

1. **稠密图**：E 接近 V²时，邻接表的优势不明显
2. **需要频繁判断两点是否相连**：如某些动态规划问题
3. **需要快速获取两个节点的权重**：加权图中 `matrix[i][j]` 直接存权重

### 选择邻接表的场景

1. **稀疏图**：社交网络、互联网网页链接
2. **需要遍历某个节点的所有邻居**：BFS/DFS 遍历
3. **空间敏感**：V = 10000，E = 50000 时，邻接矩阵需要 1 亿单元，邻接表只需要 6 万

## 面试追问

**面试官可能会问：**

1. **如果图是动态的，边会频繁添加删除，怎么存储？**
   - 可以用邻接表 + 哈希表（如 `HashMap<ListNode>`）来加速边查询
   - 或者用 `LinkedList` 替代 `ArrayList`，加速删除

2. **如何存储带权图？**
   - 邻接矩阵：`matrix[i][j]` 直接存权重，不连通可用 `Integer.MAX_VALUE` 表示
   - 邻接表：`List<int[]>` 存 `{neighbor, weight}`

3. **如何判断图中是否有环？**
   - DFS 中遇到已访问节点（非父节点）→ 有环
   - 或用拓扑排序，能排完则无环

## 总结

```
选择存储方式 = 判断图的特点

稠密？用矩阵
稀疏？用邻接表

查询多？用矩阵
遍历多？用邻接表

没有标准答案，只有最适合的选择
```

**记住这个原则：没有最好的数据结构，只有最适合当前场景的数据结构。**
