# 最小生成树

**要在 n 个城市之间修建公路，如何让总长度最短，同时保证任意两城市可以互相到达？**

这就是**最小生成树**（Minimum Spanning Tree，MST）问题。生成树是包含所有顶点的无环子图，最小生成树是权值和最小的那个。

## 核心概念

### 什么是生成树

一个连通图的**生成树**是：
- 包含图中的所有顶点
- 是无环的（V 个顶点，V-1 条边）
- 连通（任意两顶点之间有路径）

### 什么是最小生成树

在加权图中，权值和最小的生成树就是**最小生成树**。

:::tip 重要性质

**切分定理（Cut Property）**：任意切分，横跨切分的所有边中，权重最小的边一定在某个最小生成树中。

这个性质是 Prim 和 Kruskal 算法的理论基础。

:::

## Prim 算法

### 核心思想

从某个顶点开始，像「染色」一样，一步步向外扩展。每次选择**离已扩展部分最近**的顶点加入。

### 类比理解

想象你是一个农民，要围出一块地。你有一根绳子，每次把绳子绑在最近的树上，直到围住所有树。

### 实现

```java
public class Prim {
    public int prim(int[][] graph, int n) {
        boolean[] visited = new boolean[n];
        int[] minDist = new int[n];  // 到已访问集合的最小距离
        Arrays.fill(minDist, Integer.MAX_VALUE);
        minDist[0] = 0;

        int totalWeight = 0;

        for (int i = 0; i < n; i++) {
            // 1. 找未访问节点中距离最小的
            int u = -1;
            for (int j = 0; j < n; j++) {
                if (!visited[j] && (u == -1 || minDist[j] < minDist[u])) {
                    u = j;
                }
            }

            if (minDist[u] == Integer.MAX_VALUE) {
                throw new IllegalStateException("图不连通");
            }

            visited[u] = true;
            totalWeight += minDist[u];

            // 2. 用这个节点更新其他节点的距离
            for (int v = 0; v < n; v++) {
                if (!visited[v] && graph[u][v] < minDist[v]) {
                    minDist[v] = graph[u][v];
                }
            }
        }
        return totalWeight;
    }
}
```

### 堆优化

```java
public int primWithHeap(int[][] graph, int n) {
    boolean[] visited = new boolean[n];
    PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[1]));
    pq.offer(new int[]{0, 0});  // (节点, 距离)

    int totalWeight = 0;

    while (!pq.isEmpty() && !allVisited(visited)) {
        int[] cur = pq.poll();
        int u = cur[0], dist = cur[1];

        if (visited[u]) continue;
        visited[u] = true;
        totalWeight += dist;

        for (int v = 0; v < n; v++) {
            if (!visited[v] && graph[u][v] < Integer.MAX_VALUE) {
                pq.offer(new int[]{v, graph[u][v]});
            }
        }
    }
    return totalWeight;
}
```

## Kruskal 算法

### 核心思想

每次选择**全局最小的边**，但要保证加入后不会形成环。可以用**并查集**判断是否成环。

### 类比理解

想象你有所有的边，每次选最短的那条。只要它连接的两个点还没被连起来，就把它加上。

### 实现

```java
public class Kruskal {
    // 边：起点、终点、权重
    static class Edge {
        int u, v, w;
        Edge(int u, int v, int w) {
            this.u = u;
            this.v = v;
            this.w = w;
        }
    }

    public int kruskal(int n, List<Edge> edges) {
        // 1. 按权重排序
        edges.sort(Comparator.comparingInt(e -> e.w));

        // 2. 初始化并查集
        UnionFind uf = new UnionFind(n);
        int totalWeight = 0, count = 0;

        // 3. 依次选择边
        for (Edge edge : edges) {
            if (uf.union(edge.u, edge.v)) {
                totalWeight += edge.w;
                count++;
                if (count == n - 1) break;  // 已有 n-1 条边
            }
        }
        return totalWeight;
    }
}

// 并查集实现
class UnionFind {
    int[] parent, rank;

    UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);  // 路径压缩
        }
        return parent[x];
    }

    boolean union(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false;  // 已连通，会成环

        // 按秩合并
        if (rank[px] < rank[py]) {
            parent[px] = py;
        } else if (rank[px] > rank[py]) {
            parent[py] = px;
        } else {
            parent[py] = px;
            rank[px]++;
        }
        return true;
    }
}
```

## 算法对比

| 维度 | Prim | Kruskal |
|------|------|---------|
| **策略** | 选顶点（加点） | 选边（加边） |
| **数据结构** | 优先队列 | 并查集 + 排序 |
| **时间复杂度** | O(V²) 或 O(E log V) | O(E log E) |
| **适用场景** | 稠密图 | 稀疏图 |
| **贪心策略** | 局部最优 | 全局最优 |

:::tip 实际选择

**稠密图**（E ≈ V²）→ Prim（避免排序）

**稀疏图**（E ≈ V）→ Kruskal（排序开销小）

:::

## 实战：连接城市的最小成本（LeetCode 1135）

```java
public int minimumCost(int n, int[][] connections) {
    List<Edge> edges = new ArrayList<>();
    for (int[] conn : connections) {
        edges.add(new Edge(conn[0] - 1, conn[1] - 1, conn[2]));
    }
    edges.sort(Comparator.comparingInt(e -> e.w));

    UnionFind uf = new UnionFind(n);
    int total = 0, count = 0;

    for (Edge edge : edges) {
        if (uf.union(edge.u, edge.v)) {
            total += edge.w;
            count++;
            if (count == n - 1) return total;
        }
    }
    return -1;  // 无法连通
}
```

## 面试追问

1. **为什么 Kruskal 可以用贪心？**
   - 切分定理保证了全局最优
   - 每次选最小边，如果它连接的两个顶点在同一集合，就跳过

2. **最小生成树是唯一的吗？**
   - 当所有边权重都不同时，唯一
   - 有相同权重边时，可能不唯一

3. **Prim 和 Dijkstra 有什么区别？**
   - 看起来很像，但目标不同
   - Dijkstra：最短路径（到源点）
   - Prim：最小生成树（连接所有点）

**记住：最小生成树的核心是「连接所有点，成本最小」。两种算法都是贪心，区别在于贪心的对象不同。**
