# 哈夫曼树与哈夫曼编码

面试官问：**「什么是哈夫曼树？它有什么用？」**

你可能听说过哈夫曼编码，知道它是用来压缩数据的。但哈夫曼树具体是怎么构造的？**为什么用哈夫曼编码可以实现无损压缩？**

这一节，我们来揭开哈夫曼编码的神秘面纱。

## 引例：不等长编码的问题

假设有一篇文章，只包含 A、B、C、D 四个字母。如果用定长编码：

| 字母 | 定长编码（2位） |
|------|-----------------|
| A | 00 |
| B | 01 |
| C | 10 |
| D | 11 |

每写一个字母需要 2 位。但如果这四个字母的出现频率差异很大呢？

```
假设频率：
A: 70%
B: 10%
C: 10%
D: 10%
```

如果我们用**变长编码**：

| 字母 | 变长编码 |
|------|----------|
| A | 0 |
| B | 10 |
| C | 110 |
| D | 111 |

**编码长度 = 频率 × 编码位数**

```
定长编码：70%×2 + 10%×2 + 10%×2 + 10%×2 = 2
变长编码：70%×1 + 10%×2 + 10%×3 + 10%×3 = 1.5
```

变长编码的平均编码长度更短！

## 前缀码：避免歧义

但是变长编码有个问题：**如何区分 0 和 10？**

```
编码序列：10110
可能的解释：
1. B A C B (10 1 11 10)
2. D A C B (111 1 11 10)
```

**前缀码**（Prefix Code）解决了这个问题：**任何一个字符的编码都不是另一个字符编码的前缀。**

```
前缀码示例：
A: 0        ✓ 不是其他编码的前缀
B: 10       ✓ 不是其他编码的前缀
C: 110      ✓ 不是其他编码的前缀
D: 111      ✓ 不是其他编码的前缀

非前缀码示例：
A: 0
B: 01       ✗ 01 以 0 开头，是 A 编码的前缀
```

**前缀码的特点**：编码序列可以唯一地解码，因为每个字符的结束位置是确定的。

## 哈夫曼树的构造

**哈夫曼树**是一棵带权二叉树，从根到叶子的路径就是该字符的编码。权值是字符的频率。

### 构造算法

```
步骤：
1. 将所有字符及其频率作为叶子节点，放入优先队列（最小堆）
2. 每次取出频率最小的两个节点，合并成一个新节点
3. 新节点的权值 = 两个子节点权值之和
4. 重复步骤 2-3，直到只剩一个节点（根节点）
```

### 具体示例

```
频率：A:45, B:13, C:12, D:16, E:9, F:5

第一步：放入优先队列
[A:45] [B:13] [C:12] [D:16] [E:9] [F:5]

第二步：取出最小的两个 F:5 和 E:9，合并
新节点 G:14 (F+E)

队列：[C:12] [B:13] [D:16] [G:14]
      ↓ 重新排序
队列：[C:12] [G:14] [B:13] [D:16]  → [C:12] [B:13] [G:14] [D:16]

第三步：取出 C:12 和 B:13，合并
新节点 H:25

队列：[G:14] [D:16] [H:25]

第四步：取出 G:14 和 D:16，合并
新节点 I:30

队列：[H:25] [I:30]

第五步：取出 H:25 和 I:30，合并
新节点 Root:55

哈夫曼树构造完成！
```

### 哈夫曼树可视化

```
              Root:55
            /            \
        H:25              I:30
        /    \            /    \
    C:12    B:13       G:14     D:16
                       /   \
                   F:5    E:9

编码：
C: 00 (根→左→左)
B: 01 (根→左→右)
F: 100 (根→右→左→左)
E: 101 (根→右→左→右)
D: 11 (根→右→右)
```

**编码规则**：左子树边为 0，右子树边为 1。

## 哈夫曼编码实现

```java
import java.util.*;

class HuffmanNode implements Comparable<HuffmanNode> {
    int freq;
    char ch;
    HuffmanNode left, right;

    public int compareTo(HuffmanNode node) {
        return this.freq - node.freq;
    }
}

public class HuffmanCoding {
    private Map<Character, String> codes = new HashMap<>();

    // 构建哈夫曼树
    public HuffmanNode buildTree(Map<Character, Integer> freqMap) {
        PriorityQueue<HuffmanNode> pq = new PriorityQueue<>();

        for (Map.Entry<Character, Integer> entry : freqMap.entrySet()) {
            HuffmanNode node = new HuffmanNode();
            node.ch = entry.getKey();
            node.freq = entry.getValue();
            pq.offer(node);
        }

        while (pq.size() > 1) {
            HuffmanNode left = pq.poll();
            HuffmanNode right = pq.poll();

            HuffmanNode parent = new HuffmanNode();
            parent.freq = left.freq + right.freq;
            parent.left = left;
            parent.right = right;

            pq.offer(parent);
        }

        return pq.poll();
    }

    // 生成编码表
    public void generateCodes(HuffmanNode root, String code) {
        if (root == null) return;

        if (root.left == null && root.right == null) {
            codes.put(root.ch, code);
            return;
        }

        generateCodes(root.left, code + "0");
        generateCodes(root.right, code + "1");
    }

    // 编码
    public String encode(String text) {
        StringBuilder sb = new StringBuilder();
        for (char c : text.toCharArray()) {
            sb.append(codes.get(c));
        }
        return sb.toString();
    }

    // 解码
    public String decode(String encoded, HuffmanNode root) {
        StringBuilder sb = new StringBuilder();
        HuffmanNode node = root;

        for (char bit : encoded.toCharArray()) {
            node = bit == '0' ? node.left : node.right;

            if (node.left == null && node.right == null) {
                sb.append(node.ch);
                node = root;
            }
        }
        return sb.toString();
    }
}
```

## 哈夫曼编码的特性

| 特性 | 说明 |
|------|------|
| 前缀码 | 任何字符的编码都不是其他编码的前缀 |
| 最优性 | 生成的编码是加权路径长度最短的前缀码 |
| 压缩率 | 与字符频率分布相关，频率差异越大压缩率越高 |
| 无损压缩 | 编码和解码完全可逆 |

### 压缩率计算

假设原始数据用 8 位 ASCII 编码，总共 n 个字符。

```
原始大小 = n × 8 位

哈夫曼编码后的平均位数 = Σ(频率 × 编码长度)

压缩率 = (原始大小 - 压缩后大小) / 原始大小
```

## 实际应用

| 应用 | 说明 |
|------|------|
| 文件压缩 | ZIP、GZIP、RAR 等压缩工具 |
| 通信传输 | 早期传真、调制解调器 |
| 图片压缩 | JPEG 的量化步骤（部分使用） |
| 数据存储 | 节省磁盘空间 |

## 面试追问预判

1. **哈夫曼编码为什么是最优前缀码？**
   - 构造过程保证了树的加权路径长度最短
   - 任何更短的编码都会导致某个字符的编码变成另一个的前缀

2. **哈夫曼编码的局限性是什么？**
   - 需要先统计字符频率（需要两遍扫描或预知频率）
   - 对于小文件，频率表的开销可能抵消压缩收益
   - 无法处理流式数据（需要完整的频率统计）

3. **如果字符频率相同，哈夫曼树是什么样的？**
   - 哈夫曼树不唯一
   - 任意一棵平衡的二叉树都可以作为编码树
   - 实际编码取决于合并节点的顺序

4. **LZ77、LZ78 和哈夫曼编码的区别？**
   - 哈夫曼：静态编码，基于字符频率
   - LZ77/LZ78：字典编码，基于重复模式
   - 实际压缩工具（如 DEFLATE）通常结合两者

**理解哈夫曼编码的关键是理解「最优前缀码」的含义。** 频率高的字符用短编码，频率低的用长编码，整体平均编码长度最短。
