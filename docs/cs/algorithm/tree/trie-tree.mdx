# Trie树（前缀树）：字符串检索

**如何在海量字符串中快速查找「以某个前缀开头的所有字符串」？**

如果用哈希表，每次都需要遍历所有字符串检查前缀，效率极低。

**Trie 树**（前缀树，又叫字典树）正是解决这类问题的利器——它把字符串的公共前缀「共享」存储，以空间换时间。

## Trie 树的核心思想

Trie 树把每个字符作为树的节点，同一个字符串的字符在同一条路径上。

```java
class TrieNode {
    Map<Character, TrieNode> children = new HashMap<>();
    boolean isEnd = false; // 标记是否是某个字符串的结尾
}
```

```
插入 "apple", "app", "application"

        root
         |
         a
         |
         p
         |
         p
       / | \
      l  l  l   <-- 分叉
      |  |  |
      e  e  i
      |     |
     [end] c
           |
          t
           |
          i
           |
          o
           |
          n
           |
         [end]
```

**关键观察**：
- `app` 是 `apple` 的前缀，所以 `app` 的路径是 `apple` 路径的前缀
- 路径 `a -> p -> l -> e` 代表字符串 `apple`
- 标记 `isEnd = true` 表示这是一个完整的字符串结尾

## 基本操作

### 插入

```java
public void insert(String word) {
    TrieNode node = root;
    for (char c : word.toCharArray()) {
        node.children.putIfAbsent(c, new TrieNode());
        node = node.children.get(c);
    }
    node.isEnd = true;
}
```

### 查找

```java
public boolean search(String word) {
    TrieNode node = findNode(word);
    return node != null && node.isEnd;
}

public boolean startsWith(String prefix) {
    return findNode(prefix) != null;
}

private TrieNode findNode(String prefix) {
    TrieNode node = root;
    for (char c : prefix.toCharArray()) {
        if (!node.children.containsKey(c)) {
            return null;
        }
        node = node.children.get(c);
    }
    return node;
}
```

### 删除

```java
public void delete(String word) {
    delete(root, word, 0);
}

private boolean delete(TrieNode node, String word, int index) {
    if (index == word.length()) {
        if (!node.isEnd) return false; // 单词不存在
        node.isEnd = false;
        return node.children.isEmpty(); // 返回是否需要删除该节点
    }

    char c = word.charAt(index);
    TrieNode child = node.children.get(c);
    if (child == null) return false; // 单词不存在

    boolean shouldDeleteChild = delete(child, word, index + 1);

    if (shouldDeleteChild) {
        node.children.remove(c);
        return node.children.isEmpty() && !node.isEnd;
    }
    return false;
}
```

**删除的关键**：只有当一个节点既不是某个单词的结尾，也没有子节点时，才应该被删除。

## 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 插入 | O(m) | O(m) |
| 查找 | O(m) | O(1) |
| 前缀匹配 | O(m) | O(1) |
| 删除 | O(m) | O(m) |

其中 `m` 是字符串的长度。

**Trie 的时间复杂度与数据规模无关**，只与要操作的字符串长度有关。

## 时间空间权衡

### 空间优化：压缩前缀树（Radix Tree）

如果树的分支较少，可以用**压缩 Trie**（也叫 Radix Tree）把只有单一子节点的路径合并：

```
普通 Trie：
root -> a -> p -> p -> l -> e -> [end]

压缩 Trie（合并单一路径）：
root -> "apple" -> [end]
```

### 空间优化：字典序优化

实际应用中，可以用**数组**代替 `HashMap` 存储子节点：

```java
class TrieNode {
    TrieNode[] children = new TrieNode[26]; // 假设只有小写字母
    boolean isEnd;
}
```

但这会浪费空间（未使用的字母位置）。

## 典型应用

### 应用一：前缀联想

输入法、搜索引擎的自动补全：

```java
public List<String> autocomplete(String prefix) {
    List<String> result = new ArrayList<>();
    TrieNode node = findNode(prefix);
    if (node == null) return result;

    dfs(node, prefix, result);
    return result;
}

private void dfs(TrieNode node, String path, List<String> result) {
    if (node.isEnd) {
        result.add(path);
    }
    for (Map.Entry<Character, TrieNode> entry : node.children.entrySet()) {
        dfs(entry.getValue(), path + entry.getKey(), result);
    }
}
```

### 应用二：字符串统计

```java
public int countPrefix(String prefix) {
    TrieNode node = findNode(prefix);
    if (node == null) return 0;
    return countAll(node);
}

private int countAll(TrieNode node) {
    int count = node.isEnd ? 1 : 0;
    for (TrieNode child : node.children.values()) {
        count += countAll(child);
    }
    return count;
}
```

### 应用三：敏感词过滤

```java
public boolean containsSensitiveWord(String text) {
    for (int i = 0; i < text.length(); i++) {
        if (checkFrom(text, i) != null) {
            return true;
        }
    }
    return false;
}

private String checkFrom(String text, int start) {
    TrieNode node = root;
    for (int i = start; i < text.length(); i++) {
        char c = text.charAt(i);
        if (!node.children.containsKey(c)) {
            return null;
        }
        node = node.children.get(c);
        if (node.isEnd) {
            return text.substring(start, i + 1); // 找到敏感词
        }
    }
    return null;
}
```

### 应用四：最长公共前缀

```java
public String longestCommonPrefix(String[] strs) {
    if (strs == null || strs.length == 0) return "";
    if (strs.length == 1) return strs[0];

    TrieNode node = root;
    StringBuilder prefix = new StringBuilder();

    while (node.children.size() == 1 && !node.isEnd) {
        char c = node.children.keySet().iterator().next();
        prefix.append(c);
        node = node.children.get(c);
    }

    return prefix.toString();
}
```

## 与其他数据结构的对比

| 数据结构 | 查找时间 | 前缀匹配 | 空间效率 |
|----------|----------|----------|----------|
| 哈希表 | O(m) | 不支持 | 较好 |
| BST（按字典序） | O(m log n) | O(m + k) | 较好 |
| Trie | O(m) | O(m + k) | 较差（但可优化） |
| 后缀数组 | O(m log n) | O(log n) | 较好 |

## 面试追问预判

1. **Trie 树的空间复杂度怎么算？**
   - 最坏情况：所有字符串完全不同且长度都是 m
   - 空间 = O(m * n)，n 是字符串数量

2. **Trie 树和前缀哈希有什么区别？**
   - 前缀哈希：把字符串哈希后存储，O(m) 查找
   - Trie 树：能精确匹配和前缀匹配，能按字典序遍历

3. **如果字符集很大（Unicode），怎么处理？**
   - 用 `HashMap` 代替数组
   - 或用**双数组 Trie**（Double Array Trie）优化空间

4. **Trie 树和 AC 自动机的区别？**
   - Trie：单模式匹配
   - AC 自动机：多模式匹配（多个敏感词同时匹配）

**Trie 树的核心是「前缀共享」。** 当你需要频繁进行前缀匹配时，Trie 树是最佳选择。
