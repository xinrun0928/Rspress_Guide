# AVL树：平衡条件与旋转操作

**AVL 树是计算机科学中第一种自平衡二叉搜索树。** 它比红黑树更「严格」，查找更快，但插入删除时的调整更频繁。

面试中，AVL 树可能不会让你手写完整实现，但**旋转操作**和**平衡条件**是高频考点。

## AVL 树的定义

AVL 树是一种**高度平衡**的二叉搜索树，由 G.M. Adelson-Velsky 和 E.M. Landis 于 1962 年提出。

**平衡条件**：对于每个节点，其左右子树的高度差（平衡因子）绝对值不超过 1。

```
平衡因子 = 左子树高度 - 右子树高度
平衡因子的取值：-1, 0, 1
```

```java
class AVLNode {
    int val;
    AVLNode left, right;
    int height; // 节点高度

    AVLNode(int val) {
        this.val = val;
        this.height = 1;
    }
}
```

## 高度计算

```java
public int height(AVLNode node) {
    return node == null ? 0 : node.height;
}

public int balanceFactor(AVLNode node) {
    return height(node.left) - height(node.right);
}
```

## 四种旋转操作

当插入或删除节点后，可能导致某个节点的平衡因子超出 [-1, 1] 范围，需要通过**旋转**来恢复平衡。

### 右旋（LL 型）

**场景**：左子树的左孩子导致失衡。

```
原始状态：                    旋转后：
      z                            y
     / \                          / \
    y   T4      --右旋-->         x   z
   / \                            / \ / \
  x   T3                         T1 T2 T3 T4
 /
T1
```

```java
public AVLNode rightRotate(AVLNode z) {
    AVLNode y = z.left;
    AVLNode T3 = y.right;

    // 旋转
    y.right = z;
    z.left = T3;

    // 更新高度（先更新 z，再更新 y）
    z.height = Math.max(height(z.left), height(z.right)) + 1;
    y.height = Math.max(height(y.left), height(y.right)) + 1;

    return y;
}
```

### 左旋（RR 型）

**场景**：右子树的右孩子导致失衡。

```
原始状态：                    旋转后：
    z                              y
   / \                            / \
  T1  y        --左旋-->         z   x
     / \                        / \ / \
    T2  x                      T1 T2 T3 T4
       / \
      T3 T4
```

```java
public AVLNode leftRotate(AVLNode z) {
    AVLNode y = z.right;
    AVLNode T2 = y.left;

    // 旋转
    y.left = z;
    z.right = T2;

    // 更新高度
    z.height = Math.max(height(z.left), height(z.right)) + 1;
    y.height = Math.max(height(y.left), height(y.right)) + 1;

    return y;
}
```

### 左右旋（LR 型）

**场景**：左子树的右孩子导致失衡。需要先对左子树左旋，变成 LL 型，再整体右旋。

```
原始状态：                        对 T2 左旋后：           整体右旋后：
      z                               z                        x
     / \                             / \                      / \
    y   T4        --左旋-->          x   T4    --右旋-->      y   z
   / \                             / \                      / \ / \
  T1  x                           y  T3                    T1 T2 T3 T4
     / \                         / \
    T2 T3                       T1 T2
```

```java
public AVLNode leftRightRotate(AVLNode z) {
    z.left = leftRotate(z.left);  // 先对左子树左旋
    return rightRotate(z);         // 再整体右旋
}
```

### 右左旋（RL 型）

**场景**：右子树的左孩子导致失衡。

```java
public AVLNode rightLeftRotate(AVLNode z) {
    z.right = rightRotate(z.right); // 先对右子树右旋
    return leftRotate(z);            // 再整体左旋
}
```

## 插入操作

```java
public AVLNode insert(AVLNode node, int val) {
    // 1. 普通 BST 插入
    if (node == null) return new AVLNode(val);

    if (val < node.val) {
        node.left = insert(node.left, val);
    } else if (val > node.val) {
        node.right = insert(node.right, val);
    } else {
        return node; // 不允许重复
    }

    // 2. 更新高度
    node.height = Math.max(height(node.left), height(node.right)) + 1;

    // 3. 获取平衡因子，判断失衡类型
    int balance = balanceFactor(node);

    // 4. 旋转恢复平衡
    // LL 型：右旋
    if (balance > 1 && val < node.left.val) {
        return rightRotate(node);
    }

    // RR 型：左旋
    if (balance < -1 && val > node.right.val) {
        return leftRotate(node);
    }

    // LR 型：左右旋
    if (balance > 1 && val > node.left.val) {
        return leftRightRotate(node);
    }

    // RL 型：右左旋
    if (balance < -1 && val < node.right.val) {
        return rightLeftRotate(node);
    }

    return node;
}
```

## AVL vs 红黑树

| 特性 | AVL 树 | 红黑树 |
|------|--------|--------|
| 平衡标准 | 严格（高度差 `<= 1`） | 近似（高度差 `<= 2*log(n)`） |
| 查找效率 | 更优 | 略低（可能稍微不平衡） |
| 插入/删除开销 | 较大（旋转次数多） | 较小（旋转次数少） |
| 应用场景 | 读密集型 | 插入/删除频繁型 |
| 典型应用 | 数据库索引（部分） | `TreeMap`、Linux 进程调度 |

## 实际应用

虽然 AVL 树在实际工程中使用较少（红黑树更常见），但它的思想很重要：

- **数据库索引**：部分数据库使用 AVL 或类似机制
- **内存数据库**：如 Redis 的有序集合（Sorted Set）使用跳跃表，但 AVL 树也是可选实现
- **编译器**：符号表管理

## 面试追问预判

1. **AVL 树的查找时间复杂度一定是 O(log n) 吗？**
   - 是的。因为树高度 `h <= log₂(n+1)`，查找最多 h 步

2. **AVL 树最多需要几次旋转？**
   - 最多需要 2 次旋转（LR 或 RL 型）
   - LL 或 RR 型只需 1 次

3. **删除操作会导致哪些失衡情况？**
   - 删除和插入的失衡情况类似
   - 删除可能触发多次调整（沿路径向上回溯）

4. **如果 AVL 树的高度是 h，最多有多少个节点？**
   - `n = 2^h - 1`（满二叉树时）
   - 最少节点数满足：`n(h) = n(h-1) + n(h-2) + 1`，类似斐波那契数列

**理解 AVL 树的关键是理解「平衡」和「旋转」的对应关系。** 四种失衡类型对应四种旋转方式，记住这个对应关系就能举一反三。
