# 二叉查找树（BST）与平衡树

面试官问：**「什么是二叉搜索树？它的时间复杂度是多少？」**

你答：**「左子树所有节点小于根节点，右子树所有节点大于根节点。查询、插入、删除平均 O(log n)。」**

面试官追问：**「最坏情况是什么？什么会导致最坏情况？」**

你犹豫了一下......

**BST 的最坏情况是 O(n)——当树退化成链表时。** 这就是为什么需要平衡树。

## BST 的定义

**二叉搜索树**（Binary Search Tree，BST）是一种特殊的二叉树，满足以下性质：

1. 左子树所有节点的值 `<` 根节点的值
2. 右子树所有节点的值 `>` 根节点的值
3. 左右子树也分别是 BST

```java
class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;

    TreeNode(int val) {
        this.val = val;
    }
}
```

这个简单的定义，使得 BST 天然支持**二分查找**。

## BST 的基本操作

### 查找

```java
public TreeNode search(TreeNode root, int target) {
    TreeNode cur = root;
    while (cur != null) {
        if (target == cur.val) {
            return cur;
        } else if (target < cur.val) {
            cur = cur.left;
        } else {
            cur = cur.right;
        }
    }
    return null;
}
```

**查找过程**：从根开始，如果目标小于当前节点，向左走；如果大于，向右走。每次比较都能排除一半的节点。

### 插入

```java
public TreeNode insert(TreeNode root, int val) {
    if (root == null) {
        return new TreeNode(val);
    }

    if (val < root.val) {
        root.left = insert(root.left, val);
    } else if (val > root.val) {
        root.right = insert(root.right, val);
    }
    // 如果 val == root.val，通常选择忽略或更新
    return root;
}

// 迭代版本
public TreeNode insertIterative(TreeNode root, int val) {
    TreeNode newNode = new TreeNode(val);
    if (root == null) return newNode;

    TreeNode cur = root, parent = null;
    while (cur != null) {
        parent = cur;
        if (val < cur.val) {
            cur = cur.left;
        } else {
            cur = cur.right;
        }
    }

    if (val < parent.val) {
        parent.left = newNode;
    } else {
        parent.right = newNode;
    }
    return root;
}
```

**插入位置**：一定是叶子节点的下方。

### 删除（最复杂）

删除有三种情况：

```java
public TreeNode delete(TreeNode root, int key) {
    if (root == null) return null;

    if (key < root.val) {
        root.left = delete(root.left, key);
    } else if (key > root.val) {
        root.right = delete(root.right, key);
    } else {
        // 找到了要删除的节点

        // 情况1：叶子节点，直接删除
        if (root.left == null && root.right == null) {
            return null;
        }

        // 情况2：只有一个子节点，用子节点替代
        if (root.left == null) return root.right;
        if (root.right == null) return root.left;

        // 情况3：有两个子节点，用后继节点替代
        TreeNode successor = findMin(root.right); // 右子树中的最小值
        root.val = successor.val;
        root.right = delete(root.right, successor.val); // 删除后继
    }
    return root;
}

private TreeNode findMin(TreeNode node) {
    while (node.left != null) {
        node = node.left;
    }
    return node;
}
```

**删除的核心思想**：用后继节点（右子树的最小值）或前驱节点（左子树的最大值）替代被删除的节点。

## BST 的时间复杂度

| 操作 | 平均 | 最坏 |
|------|------|------|
| 查找 | O(log n) | O(n) |
| 插入 | O(log n) | O(n) |
| 删除 | O(log n) | O(n) |
| 找最大/最小 | O(log n) | O(n) |

**最坏情况**：当插入的数据是有序的（递增或递减），BST 会退化成链表。

```
插入: 1, 2, 3, 4, 5, 6, 7

退化成链表：
    1
     \
      2
       \
        3
         \
          4
           \
            5
```

## 平衡树：解决 BST 退化问题

### 为什么需要平衡

如果 BST 退化成链表，查找就变成了线性搜索，失去了 BST 的优势。

**平衡树**通过各种机制保证树的深度是 O(log n)，从而保证所有操作的时间复杂度稳定在 O(log n)。

### 平衡的衡量：高度平衡 vs 近似平衡

| 类型 | 平衡条件 | 代表 |
|------|----------|------|
| **严格平衡** | 左右子树高度差 `<= 1` | AVL 树 |
| **近似平衡** | 左右子树高度差 `<= 常数` | 红黑树 |
| **弱平衡** | 左右子树大小（节点数）接近 | 替罪羊树 |

### 平衡的代价

平衡不是免费的。**每次插入/删除后，可能需要「旋转」操作来维护平衡**。

旋转操作：
- **单旋**：LL 型（右旋）、RR 型（左旋）
- **双旋**：LR 型（左旋+右旋）、RL 型（右旋+左旋）

旋转的次数越多，插入/删除的成本越高。这就是为什么红黑树的插入/删除比 AVL 树更快。

## BST 的实际应用

### 场景一：数据库索引

数据库的索引结构通常是 B+ 树（多叉平衡搜索树），本质上是对 BST 的扩展。

### 场景二：符号表

编译器用 BST（或哈希表）存储变量名和作用域信息。

### 场景三：优先队列

用 BST 可以实现优先队列，但更常用的是堆。

## 验证 BST

```java
public boolean isValidBST(TreeNode root) {
    return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
}

private boolean validate(TreeNode node, long min, long max) {
    if (node == null) return true;

    if (node.val <= min || node.val >= max) {
        return false;
    }

    return validate(node.left, min, node.val)
        && validate(node.right, node.val, max);
}
```

**关键点**：BST 的验证不能只比较「直接」父子关系，必须传递上下界。

## 面试追问预判

1. **AVL 树和红黑树的区别？**
   - AVL：严格平衡，查找更快，但插入删除需要更多旋转
   - 红黑：近似平衡，插入删除更快，查找略慢
   - Java `TreeMap` 用红黑树，C++ `std::map` 用红黑树

2. **为什么数据库用 B+ 树而不是 BST？**
   - B+ 树是多叉树，深度更浅，磁盘 I/O 更少
   - B+ 树叶子节点链表连接，适合范围查询

3. **如果让你实现一个动态查找的数据结构，你会选什么？**
   - 数据量小：`HashMap`（O(1) 但无序）
   - 需要有序：`TreeMap`（O(log n)）
   - 范围查询多：B+ 树

**理解 BST 的关键是理解「有序性」和「平衡」的权衡。** 没有完美的数据结构，只有最适合场景的选择。
