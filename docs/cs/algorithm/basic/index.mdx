# 基础数据结构

**数组**是怎么在内存中连续排列的？**链表**为什么要用指针连接？**栈**和**队列**有什么区别？**哈希表**为什么查询这么快？

这些问题，是每个程序员都必须回答的面试题，也是理解更复杂数据结构的基石。

## 为什么从基础开始

很多人在学习算法时，喜欢直接跳到「高级」的数据结构——红黑树、B+ 树、各种复杂的图算法。但往往发现，学着学着就迷失了。

**因为他们忽略了最重要的东西：基础。**

数组和链表是几乎所有复杂数据结构的构建单元。栈和队列是解决递归、遍历问题的利器。哈希表是面试中查询优化的常客。理解这些基础，你会发现后面的内容其实都是「组合」。

## 学习路径

:::tip 建议顺序

1. **数组与链表**：打牢基础，理解「连续 vs 链式」存储的本质
2. **栈与队列**：掌握两种受限访问模式的区别和应用
3. **哈希表**：理解散列思想，这是很多优化方案的核心
4. **字符串匹配**：理解模式匹配问题，掌握 KMP 等高效算法

:::
