# 字符串匹配：KMP、Boyer-Moore、Rabin-Karp

**文本编辑器中的「查找」功能是怎么实现的？** 输入关键词，瞬间定位到匹配位置——这背后是一系列的字符串匹配算法。

从简单的暴力匹配，到高效的 KMP、Boyer-Moore、Rabin-Karp，每种算法都有其独特的思想。**理解这些算法的本质，才能在面试中灵活应对。**

## 问题定义

> 给定文本串 T（长度为 n）和模式串 P（长度为 m），找出 P 在 T 中所有出现的位置。

## 暴力匹配：最朴素的思路

```java
public List<Integer> bruteForce(String text, String pattern) {
    List<Integer> result = new ArrayList<>();
    for (int i = 0; i <= text.length() - pattern.length(); i++) {
        int j = 0;
        for (j = 0; j < pattern.length(); j++) {
            if (text.charAt(i + j) != pattern.charAt(j)) {
                break; // 不匹配，移动到下一个起点
            }
        }
        if (j == pattern.length()) {
            result.add(i); // 找到匹配
        }
    }
    return result;
}
```

**时间复杂度**：O(n * m)。最坏情况下，比如 `text = "AAAA...A"`, `pattern = "AAA...B"`，每次匹配都要比较到模式串末尾才发现不匹配。

**为什么暴力算法慢？** 因为每次匹配失败后，只向右移动一位，**完全没有利用已匹配的信息**。

## KMP 算法：利用已匹配的信息

### 核心思想

KMP 的核心是：**当匹配失败时，不是简单地右移一位，而是利用已经匹配的信息，跳过不可能匹配的位置。**

```java
public List<Integer> kmp(String text, String pattern) {
    List<Integer> result = new ArrayList<>();
    int[] lps = buildLPS(pattern); // 最长公共前后缀数组

    int i = 0, j = 0; // i 遍历 text，j 遍历 pattern
    while (i < text.length()) {
        if (text.charAt(i) == pattern.charAt(j)) {
            i++;
            j++;
        }

        if (j == pattern.length()) {
            result.add(i - j); // 找到匹配
            j = lps[j - 1];    // 继续查找下一个匹配
        } else if (i < text.length() && text.charAt(i) != pattern.charAt(j)) {
            // 失配：根据 lps 数组跳过已匹配的部分
            if (j != 0) {
                j = lps[j - 1];
            } else {
                i++;
            }
        }
    }
    return result;
}
```

### 最长公共前后缀（LPS）

**LPS 数组**（Longest Proper Prefix which is also Suffix）是 KMP 的关键。每个位置 i 表示：`pattern[0..i]` 中，最长的相等前后缀长度。

```java
private int[] buildLPS(String pattern) {
    int[] lps = new int[pattern.length()];
    lps[0] = 0; // 长度为 1 的串，没有真前后缀

    for (int i = 1; i < pattern.length(); i++) {
        int j = lps[i - 1]; // 前一个位置的最长公共前后缀长度
        while (j > 0 && pattern.charAt(i) != pattern.charAt(j)) {
            j = lps[j - 1]; // 回退到更小的前缀
        }
        if (pattern.charAt(i) == pattern.charAt(j)) {
            j++;
        }
        lps[i] = j;
    }
    return lps;
}
```

### LPS 的计算示例

以 `pattern = "ABABAC"` 为例：

| i | pattern[0..i] | 最长公共前后缀 | lps[i] |
|---|---------------|----------------|--------|
| 0 | A | 无 | 0 |
| 1 | AB | 无 | 0 |
| 2 | ABA | "A" | 1 |
| 3 | ABAB | "AB" | 2 |
| 4 | ABABA | "ABA" | 3 |
| 5 | ABABAC | 无 | 0 |

### KMP 为什么快？

**关键洞察**：当匹配到 `pattern[j]` 失败时，已匹配的 `pattern[0..j-1]` 中，**前缀和后缀可能有重叠**。

如果 `pattern[0..j-1]` 有长度为 k 的公共前后缀，那么模式串的前 k 个字符一定和文本串的最后 k 个字符匹配——这部分不需要重新比较。

**时间复杂度**：O(n + m)。预处理 LPS 数组 O(m)，匹配过程 O(n)。

## Boyer-Moore 算法：从右向左匹配

### 核心思想

Boyer-Moore 的巧妙之处在于：**从模式串的末尾开始匹配**。这样一旦发现不匹配的字符，可以跳过更多的位置。

### 坏字符规则

当文本字符在模式串中不存在时，直接跳过整个模式串长度：

```
文本:    ABCDEFGH
模式:      XYZAB
          ↑
          匹配失败，但 'F' 不在 XYZAB 中
          直接跳过整个模式串（5个位置）
```

### 好后缀规则

当部分匹配成功后，如果后面部分与模式串前缀相同，可以跳过相同部分：

```java
public int boyerMoore(String text, String pattern) {
    Map<Character, Integer> lastOccurrence = new HashMap<>();

    // 记录每个字符最后出现的位置
    for (int i = 0; i < pattern.length(); i++) {
        lastOccurrence.put(pattern.charAt(i), i);
    }

    int n = text.length(), m = pattern.length();
    int s = 0; // 模式串在文本中的起始位置

    while (s <= n - m) {
        int j = m - 1; // 从右向左比较

        while (j >= 0 && text.charAt(s + j) == pattern.charAt(j)) {
            j--;
        }

        if (j < 0) {
            return s; // 找到匹配
        }

        // 坏字符规则：计算跳过距离
        char badChar = text.charAt(s + j);
        int lastPos = lastOccurrence.getOrDefault(badChar, -1);

        if (lastPos < j) {
            s += j - lastPos; // 模式串右移
        } else {
            s += 1; // 至少右移一位
        }
    }
    return -1; // 未找到
}
```

**时间复杂度**：最好情况 O(n/m)，最坏情况 O(n * m)。

## Rabin-Karp 算法：哈希的妙用

### 核心思想

把字符串看作数字，用**滚动哈希**实现快速匹配。

```java
public int rabinKarp(String text, String pattern) {
    int n = text.length(), m = pattern.length();

    // 计算模式串的哈希值
    int patternHash = hash(pattern);
    int textHash = hash(text.substring(0, m));

    for (int i = 0; i <= n - m; i++) {
        if (patternHash == textHash) {
            // 哈希相同，再逐字符确认（避免哈希碰撞）
            if (text.substring(i, i + m).equals(pattern)) {
                return i;
            }
        }
        // 滚动哈希：移除最高位，加上新字符
        if (i < n - m) {
            textHash = nextHash(textHash, text.charAt(i), text.charAt(i + m));
        }
    }
    return -1;
}

private int hash(String s) {
    int h = 0;
    for (char c : s.toCharArray()) {
        h = h * 31 + c;
    }
    return h;
}

private int nextHash(int oldHash, char left, char right) {
    // 典型的滚动哈希：h = 31 * (h - left * 31^(m-1)) + right
    return (oldHash * 31 - left * (int) Math.pow(31, 4) + right);
}
```

**关键点**：滚动哈希使得哈希值可以在 O(1) 时间内更新，不需要重新计算整个子串的哈希。

## 三种算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|------------|------------|------|
| 暴力匹配 | O(n * m) | O(1) | 简单，不适用于大数据 |
| KMP | O(n + m) | O(m) | 最常用，预处理有意义 |
| Boyer-Moore | O(n/m) ~ O(n*m) | O(σ) | 模式串长时效率高 |
| Rabin-Karp | O(n + m) | O(1) | 支持多模式匹配 |

## 实际应用

| 场景 | 常用算法 |
|------|----------|
| 文本编辑器查找 | KMP、Boyer-Moore |
| DNA 序列匹配 | Rabin-Karp（支持多模式） |
| 拼写检查 | Tire 树（更适合前缀匹配） |
| 敏感词过滤 | AC 自动机（多模式匹配） |

## 面试追问预判

1. **KMP 中 LPS 数组的作用是什么？**
   - 记录模式串每个位置的最长公共前后缀长度
   - 匹配失败时，快速定位到下一个可能匹配的位置

2. **KMP 和朴素匹配的本质区别是什么？**
   - 朴素匹配：每次匹配失败只右移一位
   - KMP：利用已匹配信息，跳过不可能匹配的位置

3. **如果模式串中所有字符都不同，KMP 怎么处理？**
   - LPS 数组大部分为 0
   - 匹配失败时，直接右移到当前位置

4. **Boyer-Moore 为什么从右向左匹配？**
   - 如果末尾字符不匹配，可以跳过的距离更大
   - 模式串中不存在的字符可以直接跳过整个模式串

**理解 KMP 的关键是理解「公共前后缀」的概念。** 这个概念不仅用于字符串匹配，在很多算法中都会出现。
