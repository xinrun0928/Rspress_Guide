# 数组与链表：原理与对比

面试官问：**「数组和链表的区别是什么？」**

大多数人会回答：数组查询快、链表增删快。然后，就没有然后了。

但如果你能继续说出：**「为什么数组查询快？是因为连续内存和 CPU 缓存预取。为什么链表增删快？是因为只需要改变指针指向。」** 那么面试官可能会多看你一眼。

**真正理解底层原理，才能在解题时游刃有余。**

## 内存模型：连续 vs 链式

### 数组：内存中的「连续小区」

```java
int[] arr = new int[5];
```

数组在内存中是一块**连续的空间**。假设首地址是 `0x1000`，每个 `int` 占 4 字节，那么：

| 索引 | 0 | 1 | 2 | 3 | 4 |
|------|---|---|---|---|---|
| 地址 | 0x1000 | 0x1004 | 0x1008 | 0x100C | 0x1010 |

**查询原理**：给定下标 i，直接计算地址：`addr = base + i * sizeof(type)`。这是 O(1) 操作。

**为什么数组查询快？** 不仅仅是 O(1) 的计算，关键是 **CPU 缓存机制**。当访问 `arr[0]` 时，CPU 会把 `arr[0]` 附近的数据（通常是一整个缓存行，64 字节）一起加载到缓存。接下来的访问如果命中缓存，就不需要去内存读，速度相差可达 **100 倍**。

### 链表：内存中的「散落家庭」

```java
class ListNode {
    int val;
    ListNode next;
}
```

链表在内存中是**零散分布**的，每个节点包含数据和指向下一个节点的指针：

```
节点1: [值|指针] -> 节点2: [值|指针] -> 节点3: [值|null]
```

**查询原理**：给定下标 i，必须从头节点开始，沿着 `next` 指针逐个遍历。这是 O(n) 操作。

**为什么链表增删快？** 因为只需要改变前后节点的指针指向，不需要移动其他数据。但前提是——你得先找到那个位置。查找本身是 O(n) 的。

## 核心对比

| 特性 | 数组 | 链表 |
|------|------|------|
| 存储方式 | 连续内存 | 零散内存（靠指针连接） |
| 随机访问 | O(1) | O(n) |
| 头部插入/删除 | O(n)（需移动元素） | O(1) |
| 尾部插入/删除 | O(1)（已知容量时） | O(n)（未知尾节点时） |
| 中间插入/删除 | O(n)（需移动元素） | O(n)（需遍历定位） |
| 空间占用 | 小（无需额外指针） | 大（需要存储 next 指针） |
| CPU 缓存友好性 | 高 | 低 |
| 内存分配 | 预分配，大小固定 | 按需分配，动态扩展 |

## 双指针：从两端向中间逼近

双指针是处理数组问题的利器。核心思想是：**用两个指针从不同位置出发，向中间逼近**，从而减少不必要的遍历。

### 经典问题：两数之和 II（有序数组）

> 题目：给你一个升序数组和一个目标值，找出两个数的下标，使它们的和等于目标值。假设有且仅有唯一解。

```java
public int[] twoSum(int[] numbers, int target) {
    int left = 0, right = numbers.length - 1;
    while (left < right) {
        int sum = numbers[left] + numbers[right];
        if (sum == target) {
            return new int[]{left + 1, right + 1}; // 返回 1-indexed
        } else if (sum < target) {
            left++;  // 和太小，左指针右移
        } else {
            right--; // 和太大，右指针左移
        }
    }
    return new int[]{-1, -1};
}
```

**为什么这样可行？** 因为数组有序：
- 如果 `numbers[left] + numbers[right] < target`，说明需要更大的数，只能 `left++`
- 如果 `numbers[left] + numbers[right] > target`，说明需要更小的数，只能 `right--`

**复杂度**：时间 O(n)，空间 O(1)。比暴力遍历的 O(n²) 好太多。

### 快慢指针：控制遍历节奏

快慢指针的妙处在于：**两个指针速度不同，可以用来找中点、判断环、定位特定元素**。

### 经典问题：链表中点

```java
public ListNode middleNode(ListNode head) {
    ListNode slow = head, fast = head;
    // fast 走两步，slow 走一步
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow; // slow 正好是中点
}
```

**关键点**：
- 奇数个节点：slow 正好是中点
- 偶数个节点：slow 是下中位点（靠右的那个）

### 经典问题：环形链表检测

```java
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true; // 相遇，有环
    }
    return false;
}
```

**为什么有环就一定相遇？** 想象两个人在环形跑道上跑步，一个快一个慢。快的人每圈都会「追上」慢的人——因为跑道是环形的，相对距离在不断缩小。

**进阶：找环形链表入口**

```java
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    // 第一步：判断是否有环
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) break;
    }
    if (fast == null || fast.next == null) return null; // 无环

    // 第二步：找入口
    // 数学推导：设 F = 头到入口距离，C = 环长
    // 相遇时：slow 走了 F + a，fast 走了 F + a + nC
    // 2(F + a) = F + a + nC  =>  F = nC - a
    // 即：从相遇点走 F 步 = 从头走 F 步 = 入口
    ListNode ptr = head;
    while (ptr != slow) {
        ptr = ptr.next;
        slow = slow.next;
    }
    return ptr;
}
```

## 滑动窗口：变长窗口的艺术

滑动窗口是双指针的进阶版。窗口大小是**可变的**——左边界和右边界分别移动，像一扇推拉的窗户。

### 经典问题：长度最小的子数组

> 题目：给定数组和目标值，找出最短的连续子数组，使和 `>=` 目标值。

```java
public int minSubArrayLen(int target, int[] nums) {
    int left = 0, curSum = 0, result = Integer.MAX_VALUE;
    for (int right = 0; right < nums.length; right++) {
        curSum += nums[right]; // 扩大窗口
        while (curSum >= target) { // 收缩窗口
            result = Math.min(result, right - left + 1);
            curSum -= nums[left];
            left++;
        }
    }
    return result == Integer.MAX_VALUE ? 0 : result;
}
```

**核心思想**：右指针扩展找到满足条件的窗口，左指针收缩找更短的窗口。每次收缩都更新最优解。

## 总结

| 技巧 | 适用场景 |
|------|----------|
| 对撞指针 | 有序数组两端逼近，如两数之和 |
| 快慢指针 | 链表找中点、判断环 |
| 滑动窗口 | 子数组/子串问题，连续区间性质 |
| 前后指针 | 删除数组元素（快慢指针变体） |

**记住**：双指针的核心是**减少不必要的遍历**。每次移动指针时，问自己：这一步能跳过什么？

## 面试追问预判

1. **数组和链表在实际使用中如何选择？**
   - 需要频繁随机访问：选数组
   - 需要频繁插入删除：选链表（或双向链表）
   - 数据量小且已知：数组
   - 数据量动态变化：链表或动态数组（如 `ArrayList`）

2. **Java 中的 `ArrayList` 是如何实现的？**
   - 底层是数组，但支持动态扩容
   - 每次扩容通常是原来的 1.5 倍
   - 扩容时需要复制数组，有一定开销

3. **如何实现一个 LRU 缓存？**
   - 使用双向链表 + 哈希表
   - 哈希表提供 O(1) 的 key 查找
   - 双向链表维护访问顺序，支持 O(1) 的节点移动
