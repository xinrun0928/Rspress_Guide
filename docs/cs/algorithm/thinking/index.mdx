# 算法思想

**为什么有些人看到一道算法题，脑子里就能快速想到解法？**

不是因为他们背了更多题，而是因为他们掌握了**算法思想**。每一种思想都是一把钥匙，能打开一类问题的锁。

## 四种核心思想

| 思想 | 核心 | 特点 |
|------|------|------|
| **递归** | 函数调用自己 | 化繁为简，堆栈配合 |
| **分治** | 分而治之 | 分解、解决、合并 |
| **回溯** | 枚举 + 剪枝 | 搜索解空间 |
| **贪心** | 局部最优 | 不回退，简单高效 |
| **动态规划** | 最优子结构 | 状态转移，空间换时间 |

:::tip 思想的关系

递归是基础——几乎所有算法思想都建立在递归之上。

分治是递归的延伸——分解问题，递归求解，合并结果。

回溯是递归的扩展——多了「走不通就退」的能力。

贪心是回溯的简化——只走一条路，不回头。

动态规划是回溯的优化——用空间记录中间状态，避免重复计算。

:::

## 思想选择指南

### 问题有「选择」吗？

- 有 → 回溯/贪心/DP
- 无 → 分治

### 贪心能否得到最优解？

- 能 → 贪心
- 不确定 → DP 或验证贪心

### 子问题是否重叠？

- 重叠 → DP
- 不重叠 → 分治

:::tip 快速判断

1. **排列组合枚举** → 回溯
2. **最优问题，无后效性** → DP
3. **局部最优能推导全局最优** → 贪心
4. **大问题能分解为小问题** → 分治

:::

## 学习路径

:::tip 建议顺序

1. **递归**：理解函数调用自己的思维方式
2. **分治**：在递归基础上，学会分解与合并
3. **回溯**：理解「穷举 + 剪枝」的本质
4. **贪心**：为什么局部最优能导出全局最优
5. **动态规划**：最难，但也是面试最爱考的

:::

**算法思想不是孤立的，它们相互联系、相互借鉴。掌握这些思想，你就掌握了解决问题的钥匙。**
