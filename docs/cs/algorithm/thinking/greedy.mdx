# 贪心算法

**如何用最少的纸币找零钱？**

每次都选面值最大但不超过剩余金额的纸币——这就是贪心。虽然这不是总能得到最优解（如硬币找零问题），但在很多场景下，贪心就是正确答案。

## 什么是贪心

**贪心算法**（Greedy Algorithm）在每一步都做出**局部最优**的选择，希望通过一系列局部最优的选择得到全局最优解。

### 关键问题

- 局部最优一定能导出全局最优吗？
- 答案是：**不一定**，但很多问题确实可以。

:::tip 贪心 vs 回溯 vs DP

| 算法 | 策略 | 回退 |
|------|------|------|
| **贪心** | 选最优，不回头 | 不回退 |
| **回溯** | 尝试所有，不行就退 | 回退 |
| **DP** | 选最优，可能回退 | 不需要回退 |

:::

## 经典问题：活动选择

### 问题描述

有 n 个活动，每个活动有开始时间 si 和结束时间 ei。选择最多能参加多少个活动（要求活动不重叠）。

### 贪心策略

**选择结束时间最早的活动**。这样能为后续活动留出更多时间。

### 代码实现

```java
public class ActivitySelection {
    public int maxActivities(int[] start, int[] end) {
        int n = start.length;
        int[][] activities = new int[n][2];
        for (int i = 0; i < n; i++) {
            activities[i][0] = start[i];
            activities[i][1] = end[i];
        }

        // 按结束时间排序
        Arrays.sort(activities, Comparator.comparingInt(a -> a[1]));

        int count = 1;
        int lastEnd = activities[0][1];

        for (int i = 1; i < n; i++) {
            if (activities[i][0] >= lastEnd) {
                count++;
                lastEnd = activities[i][1];
            }
        }
        return count;
    }
}
```

### 正确性证明

为什么选结束时间最早的最优？

假设最优解是 A，选择最早结束的活动是 E：

- 如果 E 在最优解中，结论成立
- 如果 E 不在最优解中，可以把最优解中第一个活动替换为 E
- 替换后其他活动仍然不冲突（因为 E 结束最早）
- 所以选 E 不会更差

## 经典问题：哈夫曼编码

### 问题背景

如何用最少的二进制位编码字符？

- A: 4次，B: 2次，C: 1次，D: 1次

普通定长编码需要 2 位 × 4 = 8 位。

### 贪心策略

**优先队列**：每次选频率最小的两个节点合并。

```java
class HuffmanCoding {
    class Node implements Comparable<Node> {
        char ch;
        int freq;
        Node left, right;

        Node(char ch, int freq) {
            this.ch = ch;
            this.freq = freq;
        }

        Node(int freq, Node left, Node right) {
            this.freq = freq;
            this.left = left;
            this.right = right;
        }

        public int compareTo(Node other) {
            return this.freq - other.freq;
        }
    }

    public Node buildHuffmanTree(Map<Character, Integer> freq) {
        PriorityQueue<Node> pq = new PriorityQueue<>();

        for (Map.Entry<Character, Integer> entry : freq.entrySet()) {
            pq.offer(new Node(entry.getKey(), entry.getValue()));
        }

        while (pq.size() > 1) {
            Node left = pq.poll();
            Node right = pq.poll();
            pq.offer(new Node(left.freq + right.freq, left, right));
        }

        return pq.poll();
    }
}
```

### 编码结果

```
A: 0      (1位)
B: 10     (2位)
C: 110    (3位)
D: 111    (3位)

总位数 = 4×1 + 2×2 + 1×3 + 1×3 = 4 + 4 + 3 + 3 = 14 位
比定长编码节省了 8-4 = 4 位
```

## 经典问题：合并区间

```java
public int[][] merge(int[][] intervals) {
    if (intervals.length `<=` 1) return intervals;

    // 按起点排序
    Arrays.sort(intervals, Comparator.comparingInt(a -> a[0]));

    List<int[]> result = new ArrayList<>();
    int[] current = intervals[0];

    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] `<=` current[1]) {
            // 重叠，合并
            current[1] = Math.max(current[1], intervals[i][1]);
        } else {
            // 不重叠，加入结果
            result.add(current);
            current = intervals[i];
        }
    }
    result.add(current);

    return result.toArray(new int[0][]);
}
```

## 经典问题：跳跃游戏

```java
public boolean canJump(int[] nums) {
    int furthest = 0;

    for (int i = 0; i < nums.length; i++) {
        if (i > furthest) return false;  // 当前位置不可达
        furthest = Math.max(furthest, i + nums[i]);
        if (furthest >= nums.length - 1) return true;
    }
    return true;
}
```

## 何时用贪心

:::tip 判断标准

1. **最优子结构**：问题的最优解包含子问题的最优解
2. **贪心选择性质**：每一步的局部最优选择能导出全局最优

两个条件都满足时，贪心算法正确。

:::

### 能用贪心的问题

| 问题 | 贪心策略 |
|------|----------|
| 活动选择 | 选结束时间最早的 |
| 哈夫曼编码 | 选频率最小的合并 |
| 最小生成树 | Prim / Kruskal |
| 最短路径 | Dijkstra |
| 找零钱（特定面值） | 选面值最大的 |

### 不能用贪心的问题

| 问题 | 原因 |
|------|------|
| 背包问题（分数背包可以） | 0-1 背包需要 DP |
| 最长路径 | DAG 才能用 DP |
| 编辑距离 | 需要 DP |

## 面试追问

1. **如何证明贪心是对的？**
   - 反证法：假设最优解不包含当前贪心选择
   - 替换论证：用贪心选择替换最优解中的选择，不变差
   - 数学归纳法

2. **贪心和 DP 的区别？**
   - 贪心：只看眼前，不考虑子问题
   - DP：考虑所有子问题，选最优
   - 能用贪心就不用 DP（贪心更简单更快）

3. **跳跃游戏 II（最少跳跃次数）**
   - 贪心：每一步选能跳到最远的
   - 用 `curEnd` 和 `nextFurthest` 跟踪

**记住：贪心的关键是「局部最优能导出全局最优」。找不到这个性质，就别用贪心。**
