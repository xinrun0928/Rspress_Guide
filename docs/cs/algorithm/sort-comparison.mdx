## 二、O(n²) 排序算法

### 2.1 冒泡排序

```java
// 冒泡排序
public void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr, j, j + 1);
                swapped = true;
            }
        }
        if (!swapped) break;  // 优化：已排序则退出
    }
}
```

### 2.2 选择排序

```java
// 选择排序
public void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        swap(arr, i, minIdx);
    }
}
```

### 2.3 插入排序

```java
// 插入排序
public void insertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

## 三、O(n log n) 排序算法

### 3.1 快速排序

```java
// 快速排序
public void quickSort(int[] arr, int low, int high) {
    if (low < high) {
        int pivot = partition(arr, low, high);
        quickSort(arr, low, pivot - 1);
        quickSort(arr, pivot + 1, high);
    }
}

private int partition(int[] arr, int low, int high) {
    int pivot = arr[high];  // 选择最后一个元素作为基准
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr, i, j);
        }
    }
    swap(arr, i + 1, high);
    return i + 1;
}
```

### 3.2 归并排序

```java
// 归并排序
public void mergeSort(int[] arr, int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);
        merge(arr, left, mid, right);
    }
}

private void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;
    while (i <= mid && j <= right) {
        temp[k++] = arr[i] <= arr[j] ? arr[i++] : arr[j++];
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];
    System.arraycopy(temp, 0, arr, left, temp.length);
}
```

### 3.3 堆排序

```java
// 堆排序
public void heapSort(int[] arr) {
    int n = arr.length;

    // 构建最大堆
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }

    // 逐个提取元素
    for (int i = n - 1; i > 0; i--) {
        swap(arr, 0, i);  // 移动当前最大到末尾
        heapify(arr, i, 0);  // 调整堆
    }
}

private void heapify(int[] arr, int n, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;

    if (left < n && arr[left] > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        swap(arr, i, largest);
        heapify(arr, n, largest);
    }
}
```

## 四、稳定性

### 4.1 什么是稳定排序

```
稳定排序：相等的元素相对顺序不变

不稳定排序示例：[2, 1a, 1b, 3] → [1b, 1a, 2, 3]
               1a 和 1b 的相对顺序改变了

稳定排序示例：[2, 1a, 1b, 3] → [1a, 1b, 2, 3]
             1a 仍在 1b 前面
```

### 4.2 稳定性对比

| 稳定排序 | 不稳定排序 |
|----------|------------|
| 冒泡、插入、归并、计数、基数 | 选择、快排、堆排、希尔 |

## 五、面试题精讲

### 🔴 【高频】快排和归排的区别

**问题**：快速排序和归并排序有什么区别？

**标准答案**：

```
1. 时间复杂度
   - 两者平均都是 O(n log n)
   - 快排最坏 O(n²)，归排始终 O(n log n)

2. 空间复杂度
   - 快排 O(log n)（递归栈空间）
   - 归排 O(n)（需要额外数组）

3. 稳定性
   - 快排不稳定
   - 归排稳定

4. 缓存友好性
   - 快排：原地排序，访问局部性好
   - 归排：需要额外数组，访问局部性差

5. 适用场景
   - 快排：通用排序，内部排序
   - 归排：外部排序，需要稳定排序
```

### 🔴 【高频】快排最坏情况

**问题**：快速排序的最坏情况是什么？怎么优化？

**标准答案**：

```
最坏情况：每次划分都极度不均匀

场景：
- 已排序数组 + 选择第一个/最后一个作为基准
- 时间复杂度退化为 O(n²)

优化方法：
1. 随机选择基准
2. 三数取中（首、中、尾的中位数）
3. 双基准快排（三分法）

优化代码：
private int medianOfThree(int[] arr, int low, int high) {
    int mid = low + (high - low) / 2;
    if (arr[low] > arr[high]) swap(arr, low, high);
    if (arr[mid] > arr[high]) swap(arr, mid, high);
    if (arr[low] > arr[mid]) swap(arr, low, mid);
    return low;  // arr[low] 现在是中位数
}
```

### 🟡 【中频】为什么快排比归排快

**问题**：为什么快速排序通常比归并排序快？

**标准答案**：

```
快排通常比归排快的原因：

1. 原地排序
   - 快排是原地排序，不需要额外数组
   - 归排需要 O(n) 额外空间

2. 缓存友好
   - 快排数据访问局部性好
   - 归排需要跳跃访问，缓存命中率低

3. 常数因子小
   - 快排只有交换和比较
   - 归排需要大量数组拷贝

实际测试：快排通常比归排快 2-3 倍。
```

## 六、对比总结

### 排序算法对比

| 算法 | 平均 | 最坏 | 空间 | 稳定 | 适用场景 |
|------|------|------|------|------|----------|
| 快排 | O(n log n) | O(n²) | O(log n) | 否 | 通用，内部排序 |
| 归排 | O(n log n) | O(n log n) | O(n) | 是 | 外部排序，稳定需求 |
| 堆排 | O(n log n) | O(n log n) | O(1) | 否 | 固定空间，TopK |
| 插入 | O(n²) | O(n²) | O(1) | 是 | 小数据，近有序 |
| 计数 | O(n + k) | O(n + k) | O(k) | 是 | 范围小整数 |
| 基数 | O(nk) | O(nk) | O(n + k) | 是 | 范围大整数 |

## 七、扩展思考

### 💡 Top K 问题

```java
// Top K 问题：O(n) 解法
public int[] topK(int[] arr, int k) {
    // 小顶堆实现
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();

    for (int num : arr) {
        if (minHeap.size() < k) {
            minHeap.offer(num);
        } else if (num > minHeap.peek()) {
            minHeap.poll();
            minHeap.offer(num);
        }
    }

    int[] result = new int[k];
    for (int i = 0; i < k; i++) {
        result[i] = minHeap.poll();
    }
    return result;
}
```

### 💡 外部排序

```
外部排序（大数据排序）：

方法：分块排序 + 归并

1. 将大文件分成小块（如 100 万条）
2. 对每个小块在内存中排序
3. 逐段归并输出

优化：
- 多路归并（k 路归并）
- 减少归并趟数
```
