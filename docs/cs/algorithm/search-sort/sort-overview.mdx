# 排序算法分类与复杂度

**你知道排序算法有多少种吗？**

说出来可能吓你一跳：已知的排序算法有几十种。但真正需要掌握的，只有那么几种。

理解排序算法的分类和复杂度，是选择排序算法的基础。

## 算法分类图

```
排序算法
├── 比较排序（需要比较元素大小）
│   ├── O(n²)：冒泡、选择、插入
│   ├── O(n log n)：归并、快速、堆排序
│   └── 其他：希尔排序、树排序
│
└── 非比较排序（不比较元素大小）
    ├── 计数排序
    ├── 桶排序
    └── 基数排序
```

## 复杂度全景图

### 比较排序

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 |
|------|------|------|------|------|--------|
| **冒泡排序** | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| **选择排序** | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| **插入排序** | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| **希尔排序** | O(n log n) | O(n^1.3) | O(n²) | O(1) | 不稳定 |
| **归并排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 |
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) | 不稳定 |
| **堆排序** | O(n log n) | O(n log n) | O(n log n) | O(1) | 不稳定 |

### 非比较排序

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 |
|------|------------|------------|--------|
| **计数排序** | O(n + k) | O(n + k) | 稳定 |
| **桶排序** | O(n + k) | O(n + k) | 稳定 |
| **基数排序** | O(d(n + k)) | O(n + k) | 稳定 |

:::tip 关键说明

- n：数组长度
- k：数据范围
- d：位数
- 稳定性：相等的元素排序后相对位置是否保持不变

:::

## 什么是稳定性

### 定义

稳定排序：排序前后，相等元素的相对位置不变。

不稳定排序：排序前后，相等元素的相对位置可能改变。

### 举例

```
原始数组：[5(甲), 3, 5(乙), 2]

稳定排序后：[2, 3, 5(甲), 5(乙)]
不稳定排序后：[2, 3, 5(乙), 5(甲)]
```

### 为什么重要

```java
// 场景：先按年龄排序，再按姓名排序
Student[] students = {
    new Student("Alice", 20),
    new Student("Bob", 20),   // 第一次排序后，Bob 在 Alice 前面
    new Student("Charlie", 19)
};

// 稳定排序：年龄相同的学生，姓名顺序保持
// 不稳定排序：年龄相同的学生，姓名顺序可能打乱
```

## 时间复杂度详解

### O(n²) 为什么会慢

假设 n = 10000：
- O(n²) = 100,000,000 次比较
- O(n log n) = 10000 × 14 = 140,000 次比较

差距约 700 倍。

### 什么让 O(n log n) 更快

比较排序有一个理论下限：**任何基于比较的排序算法，最坏情况下的比较次数至少是 n log n**。

这叫**决策树下界**。

```
原因：n 个元素有 n! 种可能的排列
决策树高度至少是 log₂(n!) ≈ n log₂n
```

## 空间复杂度详解

### 原地排序 vs 非原地排序

| 类型 | 说明 | 算法 |
|------|------|------|
| **原地排序** | O(1) 额外空间 | 冒泡、选择、插入、堆排序、快速排序 |
| **非原地排序** | 需要 O(n) 或更多空间 | 归并排序 |

### 递归栈空间

快速排序和归并排序使用递归，需要考虑栈空间：

- 快速排序：平均 O(log n) 栈空间，最坏 O(n)
- 归并排序：O(log n) 栈空间（递归版本）

## 选择排序的决策

:::tip 实际选择

**小数据量（n `<=` 100）**
- 插入排序：常数因子小，实际速度快

**数据基本有序**
- 插入排序：最好情况 O(n)

**追求稳定**
- 归并排序：始终 O(n log n)，稳定

**通用场景**
- 快速排序：平均最快，实际工程首选

**内存受限**
- 堆排序：O(1) 空间

**数据范围小且是整数**
- 计数排序：O(n + k)

:::

## 面试高频问题

1. **什么是稳定排序？什么场景下需要稳定排序？**
   - 稳定排序保持相等元素的相对顺序
   - 多字段排序时，稳定排序保证第一次排序的结果不被破坏

2. **为什么快速排序实际最快？**
   - 虽然归并和堆排序都是 O(n log n)
   - 但快速排序的常数因子小
   - 缓存命中率高（顺序访问数据）

3. **排序算法的下限是什么？**
   - 比较排序：Ω(n log n)
   - 非比较排序：可以突破，但有条件限制

**记住：没有最好的排序算法，只有最适合当前场景的排序算法。理解原理，才能做出正确的选择。**
