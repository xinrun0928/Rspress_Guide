# O(n²) 排序算法

**面试官问：「手写一个排序算法。」**

大多数人会写冒泡排序。问题是：你能写对吗？你知道它的优化吗？你理解它和插入排序的区别吗？

O(n²) 排序算法虽然在大数据量下不实用，但它们是理解排序思想的基础，而且在小数据量下往往比复杂算法更快。

## 冒泡排序

### 核心思想

像水中的气泡一样，大的元素慢慢「浮」到右边。每次比较相邻的两个元素，如果顺序错误就交换。

### 代码实现

```java
public class BubbleSort {
    public void bubbleSort(int[] arr) {
        int n = arr.length;

        for (int i = 0; i < n - 1; i++) {
            boolean swapped = false;

            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    swap(arr, j, j + 1);
                    swapped = true;
                }
            }

            // 如果没有交换，说明已经有序
            if (!swapped) break;
        }
    }

    private void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### 优化：提前终止

```java
// 优化：记录最后一次交换的位置
public void bubbleSortOptimized(int[] arr) {
    int n = arr.length;
    int lastSwap = n - 1;

    while (lastSwap > 0) {
        int currentLastSwap = 0;

        for (int j = 0; j < lastSwap; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr, j, j + 1);
                currentLastSwap = j;
            }
        }

        lastSwap = currentLastSwap;
    }
}
```

### 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|------------|------|
| 最好 | O(n) | 已排序，只需 n-1 次比较 |
| 平均 | O(n²) | 每次冒泡最多交换 n-i-1 次 |
| 最坏 | O(n²) | 逆序，每次都要交换 |

## 选择排序

### 核心思想

从未排序部分选择最小的，放到已排序部分的末尾。

### 代码实现

```java
public class SelectionSort {
    public void selectionSort(int[] arr) {
        int n = arr.length;

        for (int i = 0; i < n - 1; i++) {
            int minIdx = i;

            // 找未排序部分的最小值
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[minIdx]) {
                    minIdx = j;
                }
            }

            // 交换
            if (minIdx != i) {
                swap(arr, i, minIdx);
            }
        }
    }
}
```

### 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|------------|------|
| 最好 | O(n²) | 无论如何都要找最小值 |
| 平均 | O(n²) | n-1 + n-2 + ... + 1 = n(n-1)/2 |
| 最坏 | O(n²) | 同上 |

:::warning 为什么选择排序不稳定

选择排序在交换时，可能改变相等元素的相对位置：

```
[5(甲), 3, 5(乙), 2]
交换位置 0 和 3 后：[2, 3, 5(乙), 5(甲)]
两个 5 的相对位置变了，所以不稳定。
```

:::

## 插入排序

### 核心思想

像打扑克牌一样，把每张牌插入到已排序的手牌中。

### 代码实现

```java
public class InsertionSort {
    public void insertionSort(int[] arr) {
        int n = arr.length;

        for (int i = 1; i < n; i++) {
            int key = arr[i];
            int j = i - 1;

            // 把比 key 大的元素往后移
            while (j >= 0 && arr[j] > key) {
                arr[j + 1] = arr[j];
                j--;
            }

            arr[j + 1] = key;
        }
    }
}
```

### 优化：二分插入

```java
public void binaryInsertionSort(int[] arr) {
    int n = arr.length;

    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int pos = binarySearch(arr, key, 0, i - 1);

        for (int j = i; j > pos; j--) {
            arr[j] = arr[j - 1];
        }
        arr[pos] = key;
    }
}

private int binarySearch(int[] arr, int key, int left, int right) {
    while (left <= right) {
        int mid = (left + right) / 2;
        if (arr[mid] <= key) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return left;
}
```

### 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|------------|------|
| 最好 | O(n) | 已排序，每个元素只需比较一次 |
| 平均 | O(n²) | 移动次数约为 n²/4 |
| 最坏 | O(n²) | 逆序，每个元素需要移动 i 次 |

## 三者对比

| 维度 | 冒泡排序 | 选择排序 | 插入排序 |
|------|----------|----------|----------|
| **时间复杂度** | O(n²) | O(n²) | O(n²) |
| **最好情况** | O(n) | O(n²) | O(n) |
| **交换次数** | 最多 O(n²) | 固定 n-1 | 最多 O(n²) |
| **比较次数** | O(n²) | O(n²) | O(n²) |
| **稳定性** | 稳定 | 不稳定 | 稳定 |
| **特点** | 简单，但慢 | 交换次数少 | 对基本有序的数据快 |

## 实际应用

### 小数据量

```java
// 小数据量用插入排序
public void smallSort(int[] arr) {
    if (arr.length <= 16) {
        new InsertionSort().insertionSort(arr);
    } else {
        new QuickSort().quickSort(arr);
    }
}
```

### 近似有序数据

```java
// 检测是否近似有序
public boolean isNearlySorted(int[] arr) {
    int inversions = 0;
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] < arr[i - 1]) inversions++;
    }
    return inversions <= arr.length / 10;
}
```

## 面试追问

1. **冒泡排序能优化到 O(n) 吗？**
   - 在最好情况下（已排序），可以提前终止
   - 但这是特殊情况，一般仍是 O(n²)

2. **为什么插入排序在小数据量下比快速排序快？**
   - 插入排序的常数因子小
   - 快速排序的递归开销在小数据量下不可忽略
   - 实际应用中，常用「混合排序」：小数据量用插入排序

3. **选择排序为什么不稳定？**
   - 直接交换会导致相等元素的相对位置改变
   - 解决：改用插入式的选择（把后面的元素插入到前面）

**记住：O(n²) 算法不是没用的——它们是基础，是理解更复杂算法的窗口。插入排序在小数据量和近似有序时，往往是最佳选择。**
