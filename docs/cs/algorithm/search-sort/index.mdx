# 查找与排序

**面试官问：「排序算法你了解多少？」**

这个问题看似简单，但面试官想要的绝不是「冒泡排序我会」。他想知道的是：你是否理解不同排序算法的本质区别？是否能在不同场景下做出正确的选择？

排序和查找是算法中最基础的问题，也是最能体现程序员功力的地方。

## 为什么排序重要

1. **基础中的基础**：排序是很多高级算法的前置步骤
2. **面试高频点**：手撕、快排、归并，几乎每次面试都会问到
3. **工程应用**：日志排序、排行榜、数据预处理
4. **理解算法的窗口**：通过排序，可以理解时间复杂度、空间复杂度、递归、分治等核心概念

## 算法分类

| 类别 | 算法 | 时间复杂度 | 空间复杂度 | 稳定性 |
|------|------|------------|------------|--------|
| **O(n²)** | 冒泡、选择、插入 | O(n²) | O(1) | 有/无 |
| **O(n log n)** | 归并、快速、堆排序 | O(n log n) | O(log n) ~ O(n) | 有/无 |
| **线性** | 计数、桶、基数 | O(n) | O(n + k) | 有 |

:::tip 选择指南

- **数据量小** → 插入排序（常数因子小）
- **数据基本有序** → 插入排序
- **追求稳定性** → 归并排序
- **通用场景** → 快速排序（实际常数因子小）
- **内存受限** → 堆排序（O(1) 空间）
- **数据范围有限** → 计数排序

:::

## 面试核心问题

1. **手撕快排/归并**：能否在 10 分钟内写出来？
2. **时间复杂度分析**：最好、最坏、平均情况
3. **稳定性**：什么是稳定排序？为什么重要？
4. **优化**：快排的优化手段（三数取中、随机化）
5. **应用**：Top K 问题怎么用排序解决？

## 学习路径

:::tip 建议顺序

1. **先理解思想**：每种排序算法的核心思想是什么
2. **手动实现**：能写出代码是检验是否真正理解的标准
3. **复杂度分析**：为什么是 O(n²)、O(n log n)
4. **对比学习**：为什么有这么多排序算法？各自的优缺点是什么
5. **应用场景**：什么时候用什么排序

:::

**排序不仅是面试的常考内容，更是理解算法思维的绝佳窗口。掌握排序，你就迈出了算法学习的关键一步。**
