# 海量数据处理

**面试官问：「有 10 亿个整数，怎么找出最大的 1000 个？」**

如果你的答案是「全部排序」，面试官会直接问下一个问题。如果你能说出「维护一个大小为 1000 的小顶堆」或「分区 + 堆」，说明你真正理解了大数据处理。

## Top K 问题

### 问题分析

| 方案 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|------------|------------|----------|
| 全部排序 | O(n log n) | O(n) | 需要多次查询 |
| 快排 Partition | O(n) | O(log n) | 只需找一次 |
| 堆排序 | O(n log k) | O(k) | 数据流、内存有限 |

### 方案一：全排序（不可取）

### 方案二：快排 Partition

```java
public int[] topKByPartition(int[] arr, int k) {
    int left = 0, right = arr.length - 1;
    int target = arr.length - k;  // 第 k 大的索引

    while (true) {
        int pivotIndex = partition(arr, left, right);

        if (pivotIndex == target) {
            break;
        } else if (pivotIndex < target) {
            left = pivotIndex + 1;
        } else {
            right = pivotIndex - 1;
        }
    }

    return Arrays.copyOfRange(arr, target, arr.length);
}

private int partition(int[] arr, int left, int right) {
    int pivot = arr[right];
    int i = left - 1;

    for (int j = left; j < right; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr, i, j);
        }
    }
    swap(arr, i + 1, right);
    return i + 1;
}
```

### 方案三：最小堆（推荐）

```java
public int[] topKByMinHeap(int[] arr, int k) {
    if (k <= 0) return new int[0];

    // 1. 用前 k 个元素建立最小堆
    PriorityQueue<Integer> minHeap = new PriorityQueue<>(k);
    for (int i = 0; i < k; i++) {
        minHeap.offer(arr[i]);
    }

    // 2. 遍历剩余元素
    for (int i = k; i < arr.length; i++) {
        if (arr[i] > minHeap.peek()) {
            minHeap.poll();
            minHeap.offer(arr[i]);
        }
    }

    // 3. 转成数组
    int[] result = new int[k];
    for (int i = 0; i < k; i++) {
        result[i] = minHeap.poll();
    }
    return result;
}
```

### 方案四：计数排序思想（数据范围已知）

```java
public int[] topKByCounting(int[] arr, int k, int max) {
    int[] count = new int[max + 1];

    for (int num : arr) {
        count[num]++;
    }

    int[] result = new int[k];
    int index = 0;

    for (int i = max; i >= 0 && index < k; i--) {
        while (count[i] > 0 && index < k) {
            result[index++] = i;
            count[i]--;
        }
    }
    return result;
}
```

## 位图（Bitmap）

### 什么是位图

用每一位表示一个数是否存在，1 表示存在，0 表示不存在。

### 适用场景

- 数据范围有限（如 0 到 10 亿）
- 需要去重、判重

### Java 实现

```java
public class Bitmap {
    private byte[] bits;

    public Bitmap(int maxValue) {
        this.bits = new byte[(maxValue + 8) / 8];
    }

    // 设置第 i 位为 1
    public void set(int i) {
        bits[i / 8] |= (1 << (i % 8));
    }

    // 检查第 i 位是否为 1
    public boolean get(int i) {
        return (bits[i / 8] & (1 << (i % 8))) != 0;
    }

    // 清除第 i 位
    public void clear(int i) {
        bits[i / 8] &= ~(1 << (i % 8));
    }
}
```

### 实战：100 亿整数判重

```java
public boolean[] checkDuplicates(long[] numbers, long maxValue) {
    Bitmap bitmap = new Bitmap((int) maxValue);
    boolean[] duplicates = new boolean[numbers.length];

    for (int i = 0; i < numbers.length; i++) {
        if (bitmap.get((int) numbers[i])) {
            duplicates[i] = true;
        } else {
            bitmap.set((int) numbers[i]);
            duplicates[i] = false;
        }
    }
    return duplicates;
}
```

## 布隆过滤器（Bloom Filter）

### 什么是布隆过滤器

一种空间效率高的概率数据结构，用于判断元素「可能存在」或「一定不存在」。

### 工作原理

1. 使用 k 个哈希函数，将元素映射到 k 个位置
2. 所有位置都设为 1，表示元素可能存在
3. 如果任何一个位置是 0，表示元素一定不存在

```
元素映射：hash1(x) = 3, hash2(x) = 7, hash3(x) = 10
布隆过滤器：[0,0,0,1,0,0,0,1,0,0,1,...]
                              ↑     ↑
```

### Java 实现

```java
public class BloomFilter {
    private byte[] bits;
    private int size;
    private int hashCount;

    public BloomFilter(int size, int hashCount) {
        this.size = size;
        this.hashCount = hashCount;
        this.bits = new byte[(size + 7) / 8];
    }

    public void add(int item) {
        for (int i = 0; i < hashCount; i++) {
            setBit(hash(item, i));
        }
    }

    public boolean mightContain(int item) {
        for (int i = 0; i < hashCount; i++) {
            if (!getBit(hash(item, i))) {
                return false;  // 一定不存在
            }
        }
        return true;  // 可能存在
    }

    private int hash(int item, int seed) {
        int result = item * seed;
        result ^= (result >>> 16);
        result *= 0x85ebca6b;
        result ^= (result >>> 13);
        result *= 0xc2b2ae35;
        result ^= (result >>> 16);
        return Math.abs(result) % size;
    }

    private void setBit(int pos) {
        bits[pos / 8] |= (1 << (pos % 8));
    }

    private boolean getBit(int pos) {
        return (bits[pos / 8] & (1 << (pos % 8))) != 0;
    }
}
```

### 误判率分析

| 参数 | 说明 |
|------|------|
| n | 插入的元素数量 |
| m | 位数组大小 |
| k | 哈希函数数量 |

最优哈希函数数量：`k = (m/n) * ln 2`

### 适用场景

- 缓存穿透防护
- 网页去重
- 推荐系统过滤

### 题一：10 亿整数找中位数

```java
public double findMedian(int[] arr) {
    int min = Integer.MAX_VALUE, max = Integer.MIN_VALUE;

    // 统计范围
    for (int num : arr) {
        min = Math.min(min, num);
        max = Math.max(max, num);
    }

    int range = max - min + 1;
    int[] count = new int[range];

    for (int num : arr) {
        count[num - min]++;
    }

    int sum = 0;
    int medianIndex = arr.length / 2;

    for (int i = 0; i < range; i++) {
        sum += count[i];
        if (sum > medianIndex) {
            return (arr.length % 2 == 1) ? i + min : (i + min + findNextMedian(count, i, min)) / 2.0;
        }
    }
    return -1;
}
```

### 题二：两个大文件找共同 URL

```java
public Set<String> findCommonURLs(String file1, String file2) {
    Set<String> set1 = new HashSet<>();
    Set<String> result = new HashSet<>();

    // 读取小文件到内存
    try (BufferedReader reader = new BufferedReader(new FileReader(file2))) {
        String line;
        while ((line = reader.readLine()) != null) {
            set1.add(line);
        }
    }

    // 分块处理大文件
    try (BufferedReader reader = new BufferedReader(new FileReader(file1))) {
        String line;
        while ((line = reader.readLine()) != null) {
            if (set1.contains(line)) {
                result.add(line);
            }
        }
    }

    return result;
}
```

## 面试追问

1. **布隆过滤器的误判率能降到 0 吗？**
   - 不能，只能通过增加位数和哈希函数降低误判率
   - 需要零误判的场景，用「备份布隆过滤器」或「确定过滤器」

2. **如果数据量比位图能表示的范围还大怎么办？**
   - 分桶：把数据分成多个小文件，每个小文件用位图处理
   - 或者用哈希分桶

3. **Top K 的堆优化原理是什么？**
   - 最小堆的大小是 k，堆顶是最小值
   - 新来的数比堆顶大就替换，最后堆里就是 top k
   - 复杂度 O(n log k)，比全排序 O(n log n) 更优

**记住：海量数据处理的核心是「分而治之」和「空间换时间」。位图和布隆过滤器是两种典型的空间优化技巧。**
