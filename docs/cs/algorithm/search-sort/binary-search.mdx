# 二分查找

**如何在 100 万人里找到一个人？**

遍历？O(n) 太慢。二分查找！每次把搜索范围缩小一半，log₂(1,000,000) ≈ 20 次就够了。

二分查找是面试中最基础也最重要的算法之一。它的思想简单，但要写对、写完整并不容易。

## 基本二分查找

### 核心思想

在**有序数组**中，每次取中间元素，根据大小关系缩小搜索范围。

### 代码实现

```java
public class BinarySearch {
    public int binarySearch(int[] arr, int target) {
        int left = 0, right = arr.length - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (arr[mid] == target) {
                return mid;
            } else if (arr[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return -1;  // 没找到
    }
}
```

### 递归版本

```java
public int binarySearchRecursive(int[] arr, int target, int left, int right) {
    if (left > right) return -1;

    int mid = left + (right - left) / 2;

    if (arr[mid] == target) {
        return mid;
    } else if (arr[mid] < target) {
        return binarySearchRecursive(arr, target, mid + 1, right);
    } else {
        return binarySearchRecursive(arr, target, left, mid - 1);
    }
}
```

:::warning 常见错误

1. **mid 计算溢出**：`(left + right)` 可能超过 `Integer.MAX_VALUE`
   - 正确：`left + (right - left) / 2`

2. **循环终止条件**：`left <= right` 还是 `left < right`？
   - 取决于是否要检查 right 位置的元素

3. **边界更新**：`mid + 1` / `mid - 1` 还是 `mid`？
   - 不确定时，用 `left = mid + 1` 和 `right = mid - 1`

:::

## 二分查找的变体

### 变体一：查找左边界

找到**第一个**大于等于 target 的位置。

```java
public int leftBound(int[] arr, int target) {
    int left = 0, right = arr.length;

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;  // 可能是答案，也可能是 n
}
```

### 变体二：查找右边界

找到**最后一个**小于等于 target 的位置。

```java
public int rightBound(int[] arr, int target) {
    int left = 0, right = arr.length;

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] <= target) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    return left - 1;  // 可能是答案，也可能是 -1
}
```

### 变体三：寻找插入位置

```java
public int insertPosition(int[] arr, int target) {
    int left = 0, right = arr.length;

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}
```

## 旋转数组中的二分查找

### 问题描述

把一个有序数组旋转后（如 `[1,2,3,4,5]` → `[3,4,5,1,2]`），如何在 O(log n) 时间内查找？

### 关键思路

旋转数组中，**至少有一半是有序的**。根据有序的那一半判断 target 在哪边。

### 代码实现

```java
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }

        // 判断哪一半是有序的
        if (nums[left] <= nums[mid]) {
            // 左半有序
            if (nums[left] <= target && target < nums[mid]) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        } else {
            // 右半有序
            if (nums[mid] < target && target <= nums[right]) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
    }

    return -1;
}
```

### 寻找旋转点

```java
public int findMin(int[] nums) {
    int left = 0, right = nums.length - 1;

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] > nums[right]) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    return left;  // 旋转点索引，也是最小值索引
}
```

## 实战应用

### 1. 搜索区间（LeetCode 34）

```java
public int[] searchRange(int[] nums, int target) {
    return new int[]{leftBound(nums, target), rightBound(nums, target)};
}
```

### 2. 寻找峰值（LeetCode 162）

```java
public int findPeakElement(int[] nums) {
    int left = 0, right = nums.length - 1;

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] < nums[mid + 1]) {
            left = mid + 1;  // 峰值在右边
        } else {
            right = mid;     // 峰值在左边或当前
        }
    }
    return left;
}
```

### 3. 寻找重复数（LeetCode 287）

```java
public int findDuplicate(int[] nums) {
    int left = 1, right = nums.length - 1;

    while (left < right) {
        int mid = left + (right - left) / 2;

        int count = 0;
        for (int num : nums) {
            if (num <= mid) count++;
        }

        if (count > mid) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}
```

## 二分模板

```java
public int binarySearchTemplate(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;  // 或 arr.length

    while (left <= right) {  // 或 left < right
        int mid = left + (right - left) / 2;

        if (arr[mid] == target) {
            return mid;  // 或做其他处理
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return -1;  // 或 left（插入位置）
}
```

## 面试追问

1. **为什么 `mid = left + (right - left) / 2` 而不是 `(left + right) / 2`？**
   - 防止 left + right 溢出
   - 虽然 int 溢出概率低，但 best practice

2. **二分查找一定能找到答案吗？**
   - 如果存在，一定能找到
   - 如果不存在，返回 -1 或插入位置

3. **数据量小时适合用二分吗？**
   - 不适合，小数据量遍历更快
   - 二分查找有额外的比较和计算开销

**记住：二分查找的核心是「有序」。只要能二分，就能 O(log n) 搞定。关键是想清楚边界条件和搜索空间的定义。**
