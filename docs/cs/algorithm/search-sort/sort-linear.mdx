# 线性排序算法

**面试官问：「有没有 O(n) 的排序算法？」**

大多数人会说「没有」。但实际上是有的——只要满足特定条件，排序可以突破 O(n log n) 的下界。

这就是**线性排序**：计数排序、桶排序、基数排序。它们不基于比较，但有各自的适用条件。

## 为什么比较排序有下限

n 个元素有 n! 种可能的排列。决策树高度至少是 log₂(n!) ≈ n log₂n，这就是比较排序的理论下限。

**线性排序不比较元素大小**，所以可以突破这个限制。

## 计数排序

### 核心思想

统计每个值出现的次数，然后根据计数依次输出。

### 代码实现

```java
public class CountingSort {
    public int[] countingSort(int[] arr) {
        if (arr == null || arr.length == 0) return arr;

        // 1. 找最大值和最小值，确定计数数组大小
        int max = arr[0], min = arr[0];
        for (int num : arr) {
            max = Math.max(max, num);
            min = Math.min(min, num);
        }

        // 2. 计数
        int range = max - min + 1;
        int[] count = new int[range];

        for (int num : arr) {
            count[num - min]++;
        }

        // 3. 累加计数（稳定性的关键）
        for (int i = 1; i < range; i++) {
            count[i] += count[i - 1];
        }

        // 4. 输出（从后往前保证稳定）
        int[] result = new int[arr.length];
        for (int i = arr.length - 1; i >= 0; i--) {
            result[count[arr[i] - min] - 1] = arr[i];
            count[arr[i] - min]--;
        }

        return result;
    }
}
```

### 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n + k) | k 是数据范围 |
| 空间 | O(n + k) | 计数数组 + 结果数组 |

:::tip 适用条件

- **数据范围不大**：k = O(n)，否则空间开销太大
- **非负整数**：如果是负数或浮点数，需要偏移处理
- **可以排序任何类型**：把「比较」换成「哈希」即可

:::

### 变形：整数排序

```java
// 适用于 0 到 k 范围的整数
public void countingSort(int[] arr, int k) {
    int[] count = new int[k + 1];

    // 计数
    for (int num : arr) {
        count[num]++;
    }

    // 累加
    for (int i = 1; i <= k; i++) {
        count[i] += count[i - 1];
    }

    // 输出
    int[] result = new int[arr.length];
    for (int i = arr.length - 1; i >= 0; i--) {
        result[count[arr[i]] - 1] = arr[i];
        count[arr[i]]--;
    }

    System.arraycopy(result, 0, arr, 0, arr.length);
}
```

## 桶排序

### 核心思想

把数据分配到有限数量的桶里，每个桶内分别排序，最后合并。

```
原始数据：[0.42, 0.68, 0.15, 0.73, 0.25, 0.54]

分桶（5个桶）：
桶0: [0.15]
桶1: []
桶2: [0.25]
桶3: [0.42]
桶4: [0.54]
桶5: []

排序后：[0.15, 0.25, 0.42, 0.54, 0.68, 0.73]
```

### 代码实现

```java
public class BucketSort {
    public void bucketSort(float[] arr) {
        int n = arr.length;
        if (n == 0) return;

        // 1. 创建 n 个桶
        List<Float>[] buckets = new ArrayList[n];
        for (int i = 0; i < n; i++) {
            buckets[i] = new ArrayList<>();
        }

        // 2. 分桶
        for (float num : arr) {
            int bucketIndex = (int) (num * n);
            buckets[bucketIndex].add(num);
        }

        // 3. 桶内排序（可以用其他排序，这里用插入排序）
        for (List<Float> bucket : buckets) {
            Collections.sort(bucket);
        }

        // 4. 合并
        int index = 0;
        for (List<Float> bucket : buckets) {
            for (float num : bucket) {
                arr[index++] = num;
            }
        }
    }
}
```

### 复杂度分析

| 维度 | 复杂度 |
|------|--------|
| 时间 | O(n + k·(n/k)·log(n/k)) = O(n)（均匀分布时） |
| 空间 | O(n + k) |

### 适用场景

- **数据均匀分布**：每个桶的元素数量差不多
- **数据范围已知**：可以设计合适的桶数量

## 基数排序

### 核心思想

按位数（个位、十位、百位...）依次排序，利用计数排序作为稳定排序。

```
原始数据：[170, 45, 75, 90, 802, 24, 2, 66]

按个位排序：[170, 90, 802, 2, 24, 45, 75, 66]
按十位排序：[802, 2, 24, 45, 66, 170, 75, 90]
按百位排序：[2, 24, 45, 66, 75, 90, 170, 802]

结果：[2, 24, 45, 66, 75, 90, 170, 802]
```

### 代码实现

```java
public class RadixSort {
    public void radixSort(int[] arr) {
        if (arr == null || arr.length == 0) return;

        // 1. 找最大值
        int max = arr[0];
        for (int num : arr) {
            max = Math.max(max, num);
        }

        // 2. 按每位排序
        for (int exp = 1; max / exp > 0; exp *= 10) {
            countingSortByDigit(arr, exp);
        }
    }

    private void countingSortByDigit(int[] arr, int exp) {
        int n = arr.length;
        int[] output = new int[n];
        int[] count = new int[10];  // 0-9

        // 计数
        for (int num : arr) {
            int digit = (num / exp) % 10;
            count[digit]++;
        }

        // 累加
        for (int i = 1; i < 10; i++) {
            count[i] += count[i - 1];
        }

        // 输出（从后往前保证稳定）
        for (int i = n - 1; i >= 0; i--) {
            int digit = (arr[i] / exp) % 10;
            output[count[digit] - 1] = arr[i];
            count[digit]--;
        }

        System.arraycopy(output, 0, arr, 0, n);
    }
}
```

### 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(d(n + k)) | d 是位数，k 是进制 |
| 空间 | O(n + k) | |

### 适用场景

- **非负整数**：位数为 d
- **数据范围大**：计数排序会消耗太多空间
- **可以处理负数**：偏移处理或分正负两部分

## 三者对比

| 维度 | 计数排序 | 桶排序 | 基数排序 |
|------|----------|--------|----------|
| **核心思想** | 计数 | 分桶 + 桶内排序 | 按位排序 |
| **时间复杂度** | O(n + k) | O(n + k·m) | O(d(n + k)) |
| **空间复杂度** | O(n + k) | O(n + k) | O(n + k) |
| **稳定性** | 稳定 | 稳定 | 稳定 |
| **适用条件** | 整数，范围小 | 均匀分布浮点数 | 非负整数 |

:::tip 选择指南

- **整数，范围小** → 计数排序
- **浮点数，均匀分布** → 桶排序
- **整数，范围大，位数不多** → 基数排序

:::

## 面试追问

1. **基数排序为什么从低位开始？**
   - 如果从高位开始，高位排序后，低位的相对位置可能被破坏
   - 从低位开始，利用计数排序的稳定性，保证高位相同时按低位排序

2. **桶排序的桶数量怎么选？**
   - 一般取 n（和元素数量相同）
   - 或者根据数据分布调整

3. **计数排序的空间能优化吗？**
   - 可以用原地计数，但会失去稳定性
   - 或者压缩计数数组（用 HashMap 代替数组）

**记住：线性排序不是万能的，但满足条件时，它们是最优选择。理解原理，才能判断何时该用。**
