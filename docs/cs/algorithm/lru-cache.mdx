# LRU缓存实现

> 目标级别：P5/P6

面试官问：「手写一个 LRU 缓存」——然后面试官追问：「为什么用双向链表？」「LinkedHashMap 怎么实现 LRU？」「怎么做到 O(1) 时间复杂度？」

LRU（最近最少使用）缓存是面试中的高频手写算法题。

## 一、LRU 原理

### 1.1 什么是 LRU

```
LRU（Least Recently Used）：

核心思想：
- 最近使用的元素排在前面
- 当缓存满时，淘汰最久未使用的元素

使用场景：
- 浏览器缓存
- CPU 缓存
- 数据库缓存
```

### 1.2 数据结构选择

```
为什么用双向链表 + HashMap？

1. HashMap：O(1) 查找
2. 双向链表：O(1) 插入/删除

组合实现：
- HashMap 存储 key → 链表节点
- 双向链表维护使用顺序

操作：
- 访问：查找 + 移动到头部
- 插入：添加到头部 + 检查容量
- 淘汰：删除尾部节点
```

## 二、手写 LRU 缓存

### 2.1 代码实现

```java
import java.util.*;

public class LRUCache<K, V> {
    // 容量
    private final int capacity;
    // HashMap：快速查找
    private final Map<K, Node<K, V>> cache;
    // 双向链表：维护顺序
    private final Node<K, V> head;
    private final Node<K, V> tail;

    // 双向链表节点
    private static class Node<K, V> {
        K key;
        V value;
        Node<K, V> prev;
        Node<K, V> next;

        Node(K key, V value) {
            this.key = key;
            this.value = value;
        }
    }

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.cache = new HashMap<>(capacity);
        // 虚拟头尾节点
        this.head = new Node<>(null, null);
        this.tail = new Node<>(null, null);
        head.next = tail;
        tail.prev = head;
    }

    // 获取元素
    public V get(K key) {
        Node<K, V> node = cache.get(key);
        if (node == null) {
            return null;
        }
        // 移动到链表头部（最近使用）
        moveToHead(node);
        return node.value;
    }

    // 添加元素
    public void put(K key, V value) {
        Node<K, V> node = cache.get(key);

        if (node == null) {
            // 新建节点
            Node<K, V> newNode = new Node<>(key, value);
            cache.put(key, newNode);
            addToHead(newNode);

            // 检查容量
            if (cache.size() > capacity) {
                Node<K, V> removed = removeTail();
                cache.remove(removed.key);
            }
        } else {
            // 更新节点
            node.value = value;
            moveToHead(node);
        }
    }

    // 添加到链表头部
    private void addToHead(Node<K, V> node) {
        node.prev = head;
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
    }

    // 移动到链表头部
    private void moveToHead(Node<K, V> node) {
        removeNode(node);
        addToHead(node);
    }

    // 删除节点
    private void removeNode(Node<K, V> node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    // 删除尾部节点（最久未使用）
    private Node<K, V> removeTail() {
        Node<K, V> node = tail.prev;
        removeNode(node);
        return node;
    }

    // 打印链表（调试用）
    public void print() {
        StringBuilder sb = new StringBuilder();
        Node<K, V> curr = head.next;
        while (curr != tail) {
            sb.append(curr.key).append("=").append(curr.value);
            if (curr.next != tail) {
                sb.append(" -> ");
            }
            curr = curr.next;
        }
        System.out.println("Cache: " + sb);
    }
}
```

### 2.2 使用示例

```java
public class LRUCacheTest {
    public static void main(String[] args) {
        LRUCache<String, Integer> cache = new LRUCache<>(3);

        cache.put("A", 1);
        cache.put("B", 2);
        cache.put("C", 3);
        cache.print();  // Cache: C=3 -> B=2 -> A=1

        // 访问 A
        cache.get("A");
        cache.print();  // Cache: A=1 -> C=3 -> B=2

        // 添加 D，会淘汰 B
        cache.put("D", 4);
        cache.print();  // Cache: D=4 -> A=1 -> C=3

        // 访问不存在
        System.out.println(cache.get("X"));  // null
    }
}
```

## 三、LinkedHashMap 实现

### 3.1 使用 LinkedHashMap

```java
import java.util.*;

public class LRUCacheLinkedHashMap<K, V> extends LinkedHashMap<K, V> {
    private final int capacity;

    public LRUCacheLinkedHashMap(int capacity) {
        // accessOrder = true 开启访问顺序模式
        super(capacity, 0.75f, true);
        this.capacity = capacity;
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        // 当 size 超过容量时，删除最老的元素
        return size() > capacity;
    }

    public static void main(String[] args) {
        LRUCacheLinkedHashMap<String, Integer> cache =
            new LRUCacheLinkedHashMap<>(3);

        cache.put("A", 1);
        cache.put("B", 2);
        cache.put("C", 3);
        System.out.println(cache);  // {A=1, B=2, C=3}

        cache.get("A");
        cache.put("D", 4);
        System.out.println(cache);  // {B=2, C=3, D=4}（A 被淘汰）
    }
}
```

## 四、面试题精讲

### 🔴 【高频】手写 LRU 缓存

**问题**：请手写一个 LRU 缓存。

**标准答案**：

```
数据结构：
- HashMap：O(1) 查找
- 双向链表：O(1) 插入/删除

核心操作：
1. get(key)：
   - HashMap 找到节点
   - 移动到链表头部
   - 返回值

2. put(key, value)：
   - 如果存在，更新值并移动到头部
   - 如果不存在，新建节点添加到头部
   - 如果超过容量，删除链表尾部节点

关键点：
- 使用虚拟头尾节点简化边界处理
- 删除节点时需要更新 HashMap
- put 时需要先检查是否已存在
```

## 五、对比总结

### 实现方式对比

| 实现方式 | 优点 | 缺点 |
|----------|------|------|
| HashMap + 双向链表 | 灵活可控 | 代码较多 |
| LinkedHashMap | 简单 | 需要继承 |

### 时间复杂度

| 操作 | 时间复杂度 |
|------|------------|
| get | O(1) |
| put | O(1) |
| 删除最老元素 | O(1) |

## 六、扩展思考

### 💡 LFU 缓存

```
LFU（Least Frequently Used）：
- 淘汰访问频率最低的元素
- 需要维护访问次数

LRU vs LFU：
- LRU：最近使用
- LFU：使用频率最高

实际场景：
- 一次性数据：LRU 更合适
- 热点数据：LFU 更合适
```

### 💡 Redis LRU

```
Redis LRU 实现：

1. 近似 LRU
   - 不精确的 LRU
   - 采样淘汰

2. LFU
   - 记录访问频率
   - 使用衰减

配置：
maxmemory-policy allkeys-lru
```
