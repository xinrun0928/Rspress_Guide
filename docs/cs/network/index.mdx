# 计算机网络

你打开浏览器，输入一个网址，不到一秒，页面就呈现在眼前。这个过程看似简单，背后却藏着计算机网络几十年的技术演进。

从你敲下回车键的那一刻起，数据就开始了一场跨越全球的旅程：DNS 解析找到服务器 IP、TCP 三次握手建立连接、TLS 握手加密通信、HTTP 请求往返传输……每一层都有精密的协议在协作。

本章节将带你从零开始，系统掌握计算机网络的核心知识。

## 知识全景

计算机网络采用分层思想，最经典的是 OSI 七层模型和 TCP/IP 四层模型：

| OSI 七层 | TCP/IP 四层 | 代表协议 | 核心职责 |
|---|---|---|---|
| 应用层 | 应用层 | HTTP、DNS、FTP | 为用户提供网络服务 |
| 表示层 | 应用层 | TLS、SSL | 数据格式转换、加密 |
| 会话层 | 应用层 | NetBIOS、SOCKS | 会话管理 |
| 传输层 | 传输层 | TCP、UDP | 端到端可靠传输 |
| 网络层 | 网际层 | IP、ICMP、路由协议 | 路径选择、逻辑寻址 |
| 数据链路层 | 网络接口层 | Ethernet、ARP、VLAN | 帧封装、MAC 寻址 |
| 物理层 | 网络接口层 | 电缆、光纤、无线电 | 比特流传输 |

## 学习路径

本章节按照由浅入深的顺序组织：

**网络基础**：从宏观角度理解分层模型，搞清楚 URI、URL、URN 的区别，建立整体认知。

**物理层与数据链路层**：了解数据如何在局域网中传输，MAC 地址的作用，ARP 协议如何将 IP 映射到 MAC，交换机和 VLAN 如何构建高效局域网。

**网络层**：掌握 IP 地址的奥秘，子网划分的方法，IPv4 与 IPv6 的区别，以及路由和 NAT 如何实现跨网络通信。

**传输层**：深入 TCP 的连接管理、可靠性保证、流量控制、拥塞控制，理解 UDP 的轻量级设计，以及 QUIC 如何融合两者优点。

**应用层**：这是你日常接触最多的层。HTTP 各版本的演进、HTTPS 的加密机制、DNS 的域名解析、CDN 的内容分发、WebSocket 的双向通信——理解这些，才能真正掌握网络编程。

## 为什么必须学透网络

很多人觉得网络是"背一背就行"的知识。但面试中，网络问题往往是区分候选人水平的关键。

面试官问 TCP 三次握手，不只是考你流程，而是想看你是否理解为什么需要三次、为什么是 SYN 和 ACK 而不是其他标志位、第四次握手行不行。

这些问题，只有真正理解了协议的设计哲学，才能回答得让面试官眼前一亮。

- 传输层：深入 TCP/UDP 的设计权衡
