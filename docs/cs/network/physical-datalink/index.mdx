# 物理层与数据链路层

你在一台电脑上访问网站，数据包从网卡发出，经历网线（或光纤、Wi-Fi）传输，最终到达交换机或路由器。这整个过程，发生在物理层和数据链路层。

很多人觉得这两层"太低级"，不屑于深入理解。但实际上，理解 MAC 地址、ARP 协议、交换机的工作原理，是排查网络问题的基本功，也是面试中经常被追问的细节。

## 为什么从这两层开始

在 OSI 七层模型中，物理层和数据链路层是最接近硬件的两层。它们解决的问题很具体：

- 物理层：如何在各种介质上传输比特流？用铜线、光纤还是无线电？
- 数据链路层：如何让相邻节点可靠地传输数据帧？如何标识发送者和接收者？

搞清楚这两层，你才能理解为什么 ARP 协议如此重要，交换机和路由器的区别是什么，VLAN 为什么能隔离广播域。

## 学习路线

本章节包含两个核心主题：

**以太网与 ARP**：理解 MAC 地址的结构和作用，ARP 协议如何将 IP 地址解析为 MAC 地址，以及 ARP 欺骗的原理和防护。

**交换机与 VLAN**：理解交换机如何基于 MAC 地址转发帧，VLAN 如何划分广播域，以及 Trunk 链路的作用。

掌握了这些，你就能理解数据在局域网中是如何流转的。
