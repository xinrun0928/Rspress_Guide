# 交换机与 VLAN

集线器（Hub）把数据广播给所有人，网卡自行判断是不是发给自己的。这种"一对多"的模式，效率低下，安全性差。

交换机的出现解决了这个问题。它维护一张 MAC 地址表，知道每个 MAC 地址在哪个端口，只把帧发送到目标端口。这不仅提高了效率，还为 VLAN 提供了技术基础。

## 交换机基础

### 集线器 vs 交换机

```
集线器（Hub）- 物理层设备
┌────────────────────────────────────────┐
│                                        │
│  Port 1 ──────→ 广播到所有端口          │
│  Port 2 ←─────────────────────────────│
│  Port 3 ←─────────────────────────────│
│  Port 4 ←─────────────────────────────│
│                                        │
│  问题：所有设备都能收到所有数据，安全差   │
└────────────────────────────────────────┘

交换机（Switch）- 数据链路层设备
┌────────────────────────────────────────┐
│                                        │
│  Port 1 ──────→ 查找 MAC 表 → Port 3   │
│  Port 2 ←─────────── Port 4            │
│  Port 3 ←────── Port 1 ───────────────│
│  Port 4 ←────── Port 2                │
│                                        │
│  MAC 地址表：{MAC: Port}               │
│  00:11:22:33:44:55 → Port 1           │
│  00:11:22:33:44:66 → Port 2           │
└────────────────────────────────────────┘
```

### MAC 地址表

交换机内部维护一张 MAC 地址表（Forwarding Information Base，FIB），记录每个 MAC 地址对应的端口：

```bash
# 查看交换机 MAC 地址表（管理型交换机）
show mac address-table

# Cisco 交换机
show mac address-table

# 华为交换机
display mac-address

# 清除 MAC 地址表
clear mac address-table dynamic
```

MAC 地址表项的来源：
- **动态学习**：收到帧时，从源 MAC 学习
- **静态配置**：手动指定 MAC 与端口的映射

```
┌─────────────────────────────────────────────────────────────┐
│                    MAC 地址表结构                           │
├───────────────┬──────────┬─────────────┬────────────────────┤
│  VLAN ID      │ MAC 地址 │   端口      │     类型           │
├───────────────┼──────────┼─────────────┼────────────────────┤
│  1            │ 00:11:22 │   Gi0/1     │  dynamic（动态）   │
│  1            │ 00:33:44 │   Gi0/2     │  dynamic           │
│  10           │ 00:55:66 │   Gi0/3     │  static（静态）    │
└───────────────┴──────────┴─────────────┴────────────────────┘
```

### 交换机工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                    交换机处理流程                            │
└─────────────────────────────────────────────────────────────┘

收到帧：
  │
  ├─→ 1. 读取帧头（目的 MAC）
  │
  ├─→ 2. 查找 MAC 地址表
  │        │
  │        ├─→ 找到 → 从对应端口转发
  │        │
  │        └─→ 未找到 → 广播（除接收端口外）
  │
  └─→ 3. 学习源 MAC（如果表中没有）
```

### 帧转发方式

| 方式 | 说明 | 特点 |
|---|---|---|
| 直通转发（Cut-through） | 收到目的 MAC 后立即转发 | 延迟低，不检查错误 |
| 存储转发（Store-and-Forward） | 收完整帧，检查 CRC 后转发 | 延迟高，丢弃错误帧 |
| 碎片隔离（Fragment-free） | 检查前 64 字节后转发 | 折中方案 |

现代交换机大多采用**存储转发**模式，因为硬件性能足够，且能避免错误帧传播。

## VLAN 原理

### 为什么需要 VLAN？

没有 VLAN 的交换机，所有端口在同一个广播域：

```
┌──────────────────────────────────────────────────────────────┐
│                    单一广播域                                │
│                                                              │
│  PC1 ──┐                                                    │
│  PC2 ──┼──→ 交换机 ──→ 所有端口广播                         │
│  PC3 ──┘                                                    │
│  服务器 ──┐   ↑                                             │
│  打印机 ──┴───→ 广播域太大，安全性差，管理复杂                 │
└──────────────────────────────────────────────────────────────┘
```

问题：
- 广播风暴：一个设备的广播包影响整个网络
- 安全性：任何设备都能收到其他设备的流量
- 管理复杂：无法按部门/功能隔离

VLAN（Virtual Local Area Network）把一个物理交换机划分成多个逻辑交换机：

```
┌──────────────────────────────────────────────────────────────┐
│                   VLAN 划分后的逻辑拓扑                       │
│                                                              │
│  VLAN 10（研发部）         VLAN 20（市场部）                   │
│  ┌──────────────┐         ┌──────────────┐                   │
│  │ PC1         │         │ PC3          │                   │
│  │ PC2         │         │ PC4          │                   │
│  │ 服务器 A    │         │ 服务器 B      │                   │
│  └──────────────┘         └──────────────┘                   │
│                                                              │
│  VLAN 10 的广播包不会到达 VLAN 20                            │
└──────────────────────────────────────────────────────────────┘
```

### VLAN 的作用

- **隔离广播域**：每个 VLAN 是独立的广播域
- **提高安全性**：不同 VLAN 默认不能直接通信
- **简化管理**：按部门/功能划分，便于管理
- **成本优化**：减少物理交换机的购买

### VLAN 帧格式

标准以太网帧中，VLAN 信息通过 802.1Q 标签携带：

```
标准以太网帧：
┌────────┬────────┬────────┬─────────────┬────────┬────────┐
│ 前导码 │ DST MAC│ SRC MAC│  Type/Length│  Data │  CRC  │
└────────┴────────┴────────┴─────────────┴────────┴────────┘

802.1Q VLAN 标签帧：
┌────────┬────────┬────────┬──────┬────────┬────────┬────────┬────────┐
│ 前导码 │ DST MAC│ SRC MAC│ TPID │ TCI   │Type/   │  Data │  CRC  │
│        │        │        │ 0x8100│       │Length  │       │       │
└────────┴────────┴────────┴──────┴────────┴────────┴────────┴────────┘
                                ↑
                            新增 4 字节 VLAN 标签
```

802.1Q 标签结构：

```
┌──────────────────────────────────────────────────────────────┐
│  TPID (2B)      │  TCI (2B)                                  │
│  0x8100         │  PCP │ DEI │ VID                           │
│  固定值          │  3bit │ 1bit │ 12bit (0-4095)              │
│                 │      │     │  ↑                            │
│                 │      │     │  VLAN ID                       │
│                 │      │     │  0 = no VLAN (legacy)         │
│                 │      │     │  1-4094 = valid VLAN ID       │
└─────────────────┴──────┴─────┴───────────────────────────────┘

TCI 字段：
- PCP (Priority Code Point): 802.1p 优先级 (0-7)
- DEI (Drop Eligibility Indicator): 是否可丢弃
```

### VLAN 配置

#### Cisco 交换机配置

```bash
# 创建 VLAN
vlan 10
 name Engineering
vlan 20
 name Marketing

# 将端口加入 VLAN（接入端口 - 连接终端设备）
interface GigabitEthernet0/1
 switchport mode access
 switchport access vlan 10

# 配置 trunk 端口（连接其他交换机/路由器）
interface GigabitEthernet0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20

# 查看 VLAN 信息
show vlan brief
show interfaces trunk
```

#### Linux 配置 VLAN（使用 ip 命令）

```bash
# 创建 VLAN 设备（假设物理网卡是 eth0，VLAN ID 10）
ip link add link eth0 name eth0.10 type vlan id 10

# 或使用 vconfig
vconfig add eth0 10

# 配置 IP
ip addr add 192.168.10.1/24 dev eth0.10
ip link set eth0.10 up

# 查看 VLAN 配置
ip link show
cat /proc/net/vlan/config
```

## Trunk 链路

### 为什么需要 Trunk？

当多个 VLAN 跨越多台交换机时，需要一种机制让交换机传递 VLAN 信息：

```
┌─────────────────┐               ┌─────────────────┐
│   交换机 A       │   Trunk   │   交换机 B       │
│                 │   链路     │                 │
│ VLAN 10: PC1    │ ═══════════ │ VLAN 10: PC3    │
│ VLAN 20: PC2    │           │ VLAN 20: PC4    │
└─────────────────┘           └─────────────────┘

普通链路：
- PC1 的帧到交换机 A，只能在 VLAN 10 内部广播
- 无法告诉交换机 B 这个帧属于哪个 VLAN

Trunk 链路：
- 在帧上打 VLAN 标签（802.1Q）
- 交换机 B 根据标签判断属于哪个 VLAN
```

### 封装协议

| 协议 | 说明 |
|---|---|
| 802.1Q | IEEE 标准，通用，4 字节标签 |
| ISL（Inter-Switch Link） | Cisco 私有，基本废弃 |

### Native VLAN

Trunk 链路上，可以指定一个 VLAN 为 Native VLAN，这个 VLAN 的帧不打标签：

```bash
# Cisco 配置 native VLAN
interface GigabitEthernet0/1
 switchport mode trunk
 switchport trunk native vlan 99

# 注意：两端交换机的 native VLAN 应一致，否则可能丢包
```

:::warning Native VLAN 安全
Native VLAN 默认为 VLAN 1，且大多数设备默认不打标签。如果攻击者伪造 Native VLAN 标签，可能实现 VLAN hopping 攻击。建议将 native VLAN 改为不使用的 VLAN。
:::

### VLAN 间路由

不同 VLAN 之间默认不能通信，需要路由器或三层交换机进行路由：

```
┌─────────────────────────────────────────────────────────────┐
│               VLAN 间路由的三种方式                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  方式一：Router-on-a-Stick                                 │
│  ┌──────────┐                                              │
│  │  路由器   │                                              │
│  │ Gi0/0.10 ├──── Trunk ──────→ 交换机                    │
│  │ Gi0/0.20 │                                              │
│  └──────────┘                                              │
│   子接口 .10 → VLAN 10                                      │
│   子接口 .20 → VLAN 20                                      │
│                                                             │
│  方式二：三层交换机                                          │
│  ┌──────────┐                                              │
│  │ 三层交换机│                                              │
│  │ SVI VLAN10│ ←──────── VLAN 10                           │
│  │ SVI VLAN20│ ←──────── VLAN 20                           │
│  └──────────┘                                              │
│                                                             │
│  方式三：单臂路由（简化版 Router-on-a-Stick）               │
└─────────────────────────────────────────────────────────────┘
```

### Cisco 三层交换机配置

```bash
# 1. 开启三层交换功能
ip routing

# 2. 创建 VLAN
vlan 10
vlan 20

# 3. 配置 SVI（Switch Virtual Interface）
interface Vlan10
 ip address 192.168.10.1 255.255.255.0
 no shutdown

interface Vlan20
 ip address 192.168.20.1 255.255.255.0
 no shutdown

# 4. 将端口加入 VLAN
interface GigabitEthernet0/1
 switchport mode access
 switchport access vlan 10

# 5. 开启端口的三层模式（可选）
interface GigabitEthernet0/24
 no switchport
 ip address 10.0.0.1 255.255.255.0
```

## VTP 与 GVRP

### VTP（VLAN Trunking Protocol）

Cisco 专有协议，用于在交换网络中自动分发 VLAN 信息：

```bash
# VTP 模式
vtp mode {server | client | transparent}

# VTP 配置
vtp domain mycompany
vtp password mypassword

# 查看 VTP 状态
show vtp status
```

:::warning VTP 风险
VTP server 删除 VLAN 时，client 会同步删除，可能导致网络故障。生产环境慎用，或使用 VTP transparent 模式。
:::

### GVRP（GARP VLAN Registration Protocol）

IEEE 标准协议，与 VTP 类似，但更通用：

```bash
# 开启 GVRP
set gvrp enable
set gvrp registrationmode normal
```

## 常见问题排查

### VLAN 不通

```bash
# 1. 检查端口模式
show interfaces GigabitEthernet0/1 switchport

# 2. 检查 VLAN 是否存在
show vlan brief

# 3. 检查 trunk 是否允许该 VLAN
show interfaces trunk

# 4. 检查 native VLAN 是否一致
show interfaces trunk

# 5. 检查交换机间链路是否为 trunk
# 两端都需要配置 trunk
```

### 常见错误

| 错误 | 症状 | 解决 |
|---|---|---|
| trunk 未配置 | 多台交换机的同一 VLAN 不通 | 两端都配置 `switchport mode trunk` |
| native VLAN 不一致 | untagged 帧丢包 | 两端设置相同的 native VLAN |
| VLAN ID 超范围 | 帧被丢弃 | 确保 VLAN ID 在 1-4094 |
| trunk allowed VLAN 未包含 | 某些 VLAN 不通 | 配置 `switchport trunk allowed vlan` |

## 总结

交换机和 VLAN 是现代局域网的核心技术：

- 交换机基于 MAC 地址表转发帧，提高效率
- VLAN 划分广播域，隔离不同业务
- 802.1Q 标签在帧中携带 VLAN 信息
- Trunk 链路承载多个 VLAN 的流量
- VLAN 间路由需要三层设备

- VLAN 标签是什么时候加的、什么时候去的？
- Trunk 链路上的 Native VLAN 是什么？为什么要注意？
- 如何实现不同 VLAN 之间的通信？
