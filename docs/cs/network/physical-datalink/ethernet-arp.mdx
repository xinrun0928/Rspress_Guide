# 以太网与 ARP 协议

你的电脑网卡上贴着一个地址——MAC 地址。当你访问局域网内的另一台设备时，数据包的目的 MAC 填的就是这个地址。但 IP 地址是逻辑地址，MAC 地址是物理地址，两者是如何关联起来的？

答案是 ARP（Address Resolution Protocol）协议。这个看似简单的协议，却是局域网通信的基石。

## 以太网基础

以太网（Ethernet）是局域网最常用的技术标准，由 IEEE 802.3 定义。它工作在数据链路层，负责相邻节点之间的帧传输。

### MAC 地址

MAC（Media Access Control）地址是网卡的唯一标识，固化在网卡的 ROM 中，48 位（6 字节），通常用十六进制表示：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        MAC 地址结构                                 │
│                                                                     │
│  00:1A:2B:3C:4D:5E                                                │
│  └─┬─┘ └─┬─┘ └─┬─┘                                                │
│  厂商 ID  厂商 ID  厂商 ID  厂商分配的设备 ID                       │
│                                                                     │
│  前 24 位（OUI）：IEEE 分配给厂商的唯一标识                          │
│  后 24 位：厂商自行分配给每块网卡                                   │
└─────────────────────────────────────────────────────────────────────┘
```

```bash
# 查看本机 MAC 地址
# Linux/macOS
ifconfig | grep ether
ip link show

# Windows
ipconfig /all | findstr "Physical Address"
```

MAC 地址的一些特点：
- 全球唯一，由 IEEE 统一管理
- 理论上不可更改（可软件伪造）
- 用于局域网内帧的寻址
- 类型：`unicast`（单播）、`multicast`（多播）、`broadcast`（广播）

### 以太网帧结构

以太网帧是数据链路层的基本传输单元：

```
┌──────────┬──────────┬──────────┬────────────┬──────────┬────────┬─────┐
│  前导码  │  目的 MAC │  源 MAC  │   类型/长度  │  数据    │  CRC  │ Gap │
│  8 字节  │  6 字节  │  6 字节  │   2 字节    │ 46-1500 │ 4 字节 │ 12B │
└──────────┴──────────┴──────────┴────────────┴──────────┴────────┴─────┘
  前导码     帧开始     目的地址    源地址        类型字段   数据载荷   FCS
  (SFD)     标志       (DA)       (SA)         (EtherType)           校验
```

| 字段 | 长度 | 说明 |
|---|---|---|
| 前导码 + SFD | 8 字节 | 同步时钟，固定模式 `10101010...10101011` |
| 目的 MAC | 6 字节 | 帧的接收者 |
| 源 MAC | 6 字节 | 帧的发送者 |
| 类型/长度 | 2 字节 | `>=` `1536` 表示类型（IPv4: `0x0800`，ARP: `0x0806`），`<=` `1500` 表示长度 |
| 数据 | 46-1500 字节 | 上层协议载荷，最少 46 字节，不足填充 PAD |
| CRC | 4 字节 | 帧校验序列，检测传输错误 |

```bash
# 用 Wireshark 抓包看以太网帧
# 过滤 ARP 包
arp

# 过滤特定 MAC 地址
eth.addr == 00:1A:2B:3C:4D:5E

# 过滤特定网卡的帧
eth.src == 00:1A:2B:3C:4D:5E
```

## ARP 协议详解

### 为什么需要 ARP？

IP 地址是网络层的逻辑地址，MAC 地址是数据链路层的物理地址。当数据包要发往同一局域网内的目标时：

```
发送过程：
1. 已知：目标 IP 地址（如 192.168.1.100）
2. 需要：目标 MAC 地址（帧必须填写目的 MAC）
3. 怎么办？ → ARP 请求！
```

ARP 的作用就是：**在同一个局域网内，通过 IP 地址找到对应的 MAC 地址。**

### ARP 工作流程

**场景**：主机 A（IP: `192.168.1.10`）要发消息给主机 B（IP: `192.168.1.20`），但 A 不知道 B 的 MAC 地址。

```
┌──────────────────────────────────────────────────────────────────────┐
│                        ARP 请求-响应过程                              │
└──────────────────────────────────────────────────────────────────────┘

主机 A                                   广播（ARP 请求）
┌─────────────┐                   ┌─────────────┬─────────────┐
│ IP: 192.168 │ ── 帧 ────────────→ │  交换机      │              │
│ .1.10       │   DST: ff:ff:ff:ff │  （广播）    │  主机 C     │
│ MAC: aaaa   │   SRC: aaaa        │              │  192.168.1.30
└─────────────┘                   └─────────────┼─────────────┘
                                      │
                                      ↓ 广播
                              ┌─────────────────┐
                              │ 谁有 192.168.1.20│
                              │ 的 MAC 地址？    │
                              └─────────────────┘

主机 B（收到广播，认出是自己）
┌─────────────┐                   ┌─────────────┐
│ IP: 192.168 │ ←── ARP 响应 ─── │ 交换机      │
│ .1.20       │   DST: aaaa      │ （单播）    │
│ MAC: bbbb   │   SRC: bbbb      │             │
└─────────────┘                   └─────────────┘

ARP 表更新：
┌─────────────────────────────────────┐
│ IP 地址           MAC 地址          │
├─────────────────────────────────────┤
│ 192.168.1.20      bbbb              │
└─────────────────────────────────────┘
```

### ARP 报文格式

```
┌─────────┬──────────┬──────────┬──────────┬───────────────┬───────────────┐
│  硬件类型│ 协议类型  │  硬件地址│  协议地址│   发送端 MAC  │   发送端 IP   │
│  2 字节  │ 2 字节   │  长度    │  长度    │   6 字节      │   4 字节      │
├─────────┼──────────┼──────────┼──────────┼───────────────┼───────────────┤
│ 操作码  │  目的 MAC │  目的 IP │           │               │               │
│ 2 字节  │  6 字节   │  4 字节  │           │               │               │
└─────────┴──────────┴──────────┴──────────┴───────────────┴───────────────┘
  操作码：1=ARP请求  2=ARP响应  3=RARP请求  4=RARP响应
```

| 字段 | 值 | 说明 |
|---|---|---|
| 硬件类型 | 1 | Ethernet |
| 协议类型 | `0x0800` | IPv4 |
| 硬件地址长度 | 6 | MAC 地址长度 |
| 协议地址长度 | 4 | IPv4 地址长度 |
| 操作码 | 1 或 2 | 1=请求，2=响应 |

### ARP 表

每台主机的操作系统维护一张 ARP 表，缓存 IP-MAC 映射：

```bash
# 查看 ARP 表
# Linux
arp -a

# Windows
arp -a

# macOS
arp -a

# 清除 ARP 缓存（强制重新 ARP）
# Linux
ip neigh flush all
arp -d *

# Windows
netsh interface ip delete arpcache

# macOS
sudo arp -d -a
```

ARP 表项有老化时间，通常 20 分钟（可配置），避免网络拓扑变化后缓存不更新。

```bash
# Linux 查看 ARP 老化时间
cat /proc/sys/net/ipv4/neigh/default/gc_stale_time
# 默认 60 秒检查一次，300 秒（5分钟）未使用则删除
```

### ARP 欺骗与防护

#### 原理

ARP 协议没有认证机制，任何主机都可以声称"这个 IP 对应我的 MAC"。攻击者利用这一点：

```
正常情况：
┌──────────┐     ARP 请求/响应     ┌──────────┐
│  主机 A   │ ←─────────────────→ │  主机 B   │
└──────────┘                       └──────────┘

ARP 欺骗：
┌──────────┐     ARP 欺骗         ┌──────────┐     ARP 欺骗     ┌──────────┐
│  主机 A   │ ←───────────────────→ │  攻击者   │ ←──────────────→ │  主机 B   │
└──────────┘     "我是网关 MAC"      └──────────┘     "我是主机 A MAC" └──────────┘
```

#### 攻击后果

1. **中间人攻击（MITM）**：流量经过攻击者，可窃听、篡改
2. **拒绝服务（DoS）**：向网关发送虚假 ARP，把流量引向不存在 MAC
3. **会话劫持**：截获认证信息

#### 防护措施

```bash
# 1. 静态 ARP 绑定（临时，手动添加）
# Linux
arp -s 192.168.1.1 00:11:22:33:44:55

# Windows
netsh interface ip add neighbors "本地连接" 192.168.1.1 00-11-22-33-44-55

# 2. 动态 ARP 检测（交换机功能，DAI）
# 需要交换机支持 802.1X
# 交换机验证 ARP 请求/响应中的 MAC 与 DHCP Snooping 表一致

# 3. 开启 Linux 内核 ARP 防护
# /etc/sysctl.conf
net.ipv4.conf.all.arp_announce = 2
net.ipv4.conf.all.arp_ignore = 1
net.ipv4.conf.all.arp_accept = 0

# 4. 使用 ARP 防火墙工具
# arptables（Linux）
arptables -A INPUT -s 192.168.1.0/24 --jump ACCEPT

# 5. 企业级方案：802.1X 端口认证 + EAP
# 只有通过认证的设备才能接入网络
```

#### Windows 手动绑定 ARP

```powershell
# 查看当前 ARP 表
arp -a

# 绑定静态 ARP（需要管理员权限）
arp -s 192.168.1.1 00-11-22-33-44-55

# 删除静态绑定
arp -d 192.168.1.1
```

## 代理 ARP

代理 ARP 是一种特殊行为：路由器代替其他网段的设备回应 ARP 请求。

```
场景：
PC1 (192.168.1.10)  ──── 路由器 (192.168.1.1, 10.0.0.1)  ──── PC2 (10.0.0.10)

PC1 想知道 10.0.0.10 的 MAC → 发送 ARP 请求
路由器收到后，返回自己的 MAC（代理行为）
PC1 把发给 10.0.0.10 的帧发给路由器，由路由器转发
```

```bash
# Linux 开启/关闭代理 ARP
# 开启
echo 1 > /proc/sys/net/ipv4/conf/all/proxy_arp
echo 1 > /proc/sys/net/ipv4/conf/eth0/proxy_arp

# 关闭
echo 0 > /proc/sys/net/ipv4/conf/all/proxy_arp
```

## 总结

以太网和 ARP 是局域网通信的基础：

- MAC 地址是 48 位的物理地址，全球唯一
- 以太网帧封装了源 MAC 和目的 MAC
- ARP 协议通过 IP 地址解析同网段内主机的 MAC 地址
- ARP 表缓存 IP-MAC 映射，有老化机制
- ARP 协议缺乏认证，可被利用进行欺骗攻击

理解 ARP 的工作原理，是理解局域网通信的关键，也是排查"能 ping 通 IP 但 ping 不通"这类问题的前提。
