# IP 地址与子网划分

当你配置电脑网络时，会填入 IP 地址、子网掩码、默认网关。这三个数字决定了你的电脑如何与网络中的其他设备通信。

很多人会配置，却不理解背后的原理。比如：为什么子网掩码是 255.255.255.0？IP `192.168.1.100` 和 `192.168.2.100` 在同一个网段吗？

这一节，帮你彻底搞清楚这些问题。

## IPv4 地址基础

IPv4 地址是 32 位的二进制数，通常用点分十进制表示：

```
二进制：11000000 10101000 00000001 01100100
十进制：192       .168      .1        .100
```

### 地址分类（Classful Addressing）

传统的 IPv4 地址分为五类：

| 类别 | 地址范围 | 网络位数 | 主机位数 | 网络数 | 主机数/网络 | 用途 |
|---|---|---|---|---|---|---|
| A 类 | `0.0.0.0` `~` `127.255.255.255` | 8 | 24 | 128 | 16,777,214 | 大型网络 |
| B 类 | `128.0.0.0` `~` `191.255.255.255` | 16 | 16 | 16,384 | 65,534 | 中型网络 |
| C 类 | `192.0.0.0` `~` `223.255.255.255` | 24 | 8 | 2,097,152 | 254 | 小型网络 |
| D 类 | `224.0.0.0` `~` `239.255.255.255` | — | — | — | — | 多播（组播） |
| E 类 | `240.0.0.0` `~` `255.255.255.255` | — | — | — | — | 保留 |

```bash
# 查看 IP 地址分类
# A 类：首位 0，范围 1-126（127 保留给环回）
# B 类：首位 10，范围 128-191
# C 类：首位 110，范围 192-223
# D 类：首位 1110，范围 224-239
# E 类：首位 1111，范围 240-255
```

### 特殊地址

| 地址 | 说明 | 用途 |
|---|---|---|
| `0.0.0.0` | 本网络，当前主机 | 路由表中表示"默认路由" |
| `127.0.0.0` `~` `127.255.255.255` | 环回地址 | 本机测试，发送给自己的数据 |
| `255.255.255.255` | 受限广播地址 | 广播到本网络（路由器不转发） |
| `<net>.0` | 网络地址 | 表示整个网络，不可分配 |
| `<net>.255` | 广播地址 | 发送给网络内所有主机 |

### 私有地址

私有地址（Private Address）不能在公网路由，用于局域网内部：

| 范围 | 说明 | CIDR 表示 |
|---|---|---|
| `10.0.0.0` `~` `10.255.255.255` | A 类私有 | `10.0.0.0/8` |
| `172.16.0.0` `~` `172.31.255.255` | B 类私有 | `172.16.0.0/12` |
| `192.168.0.0` `~` `192.168.255.255` | C 类私有 | `192.168.0.0/16` |

```bash
# 常用私有地址段
# 10.0.0.0/8      - 大型企业内部
# 172.16.0.0/12   - 中型企业
# 192.168.0.0/16  - 家庭/小型办公室
```

## 子网掩码

### 为什么需要子网掩码？

IP 地址由网络部分和主机部分组成。但仅从 IP 地址本身，无法区分哪部分是网络、哪部分是主机。子网掩码就是用来解决这个问题的。

```
IP 地址：    192.168.1.100
子网掩码：   255.255.255.0
            ↓ 按位与运算
网络地址：   192.168.1.0     ← 同一网络的所有主机共享
主机地址：              100  ← 标识网络内的具体设备
```

### 子网掩码表示

子网掩码有两种表示方式：

| 方式 | 示例 | 说明 |
|---|---|---|
| 点分十进制 | `255.255.255.0` | 4 段十进制 |
| CIDR 表示 | `/24` | 斜线后跟网络位数 |

```bash
# 常用子网掩码与 CIDR 对照
/8   = 255.0.0.0
/16  = 255.255.0.0
/24  = 255.255.255.0
/25  = 255.255.255.128
/26  = 255.255.255.192
/27  = 255.255.255.224
/28  = 255.255.255.240
/29  = 255.255.255.248
/30  = 255.255.255.252
/31  = 255.255.255.254   ← .point-to-point link
/32  = 255.255.255.255   ← 单主机
```

### 子网掩码的二进制规则

子网掩码必须满足：**连续 1 后跟连续 0**。

```bash
# 正确
255.255.255.0   = 11111111.11111111.11111111.00000000
255.255.252.0   = 11111111.11111111.11111100.00000000

# 错误（不连续）
255.255.255.240 = 11111111.11111111.11111111.11110000 ✓
255.255.254.0   = 11111111.11111111.11111110.00000000 ✓
255.255.254.240 = 11111111.11111111.11111110.11110000 ✗ (不连续)
```

## 子网划分计算

### 核心公式

给定网络前缀 `/n`（n = 网络位数）：

```bash
总位数 = 32
主机位数 = 32 - n

主机数 = 2^(32-n) - 2
# 减 2 是排除网络地址和广播地址

可用主机数 = 2^(32-n) - 2
```

### 常用计算表

| CIDR | 主机位数 | 总主机数 | 可用主机数 | 子网掩码 |
|---|---|---|---|---|
| /24 | 8 | 256 | 254 | 255.255.255.0 |
| /25 | 7 | 128 | 126 | 255.255.255.128 |
| /26 | 6 | 64 | 62 | 255.255.255.192 |
| /27 | 5 | 32 | 30 | 255.255.255.224 |
| /28 | 4 | 16 | 14 | 255.255.255.240 |
| /29 | 3 | 8 | 6 | 255.255.255.248 |
| /30 | 2 | 4 | 2 | 255.255.255.252 |
| /31 | 1 | 2 | 0* | 255.255.255.254 |
| /32 | 0 | 1 | 0 | 255.255.255.255 |

:::tip /31 和 /32 的特殊情况
`/31` 用于点对点链路，RFC 3021 定义了两个地址都可用于主机通信，无需网络/广播地址。

`/32` 用于环回地址或单主机路由。
:::

### 手动计算示例

**问题**：将 `192.168.1.0/24` 划分成 4 个等大小的子网，每个子网多少主机？

```bash
分析：
- 原始网络：/24
- 需要划分：4 = 2^2
- 需要借位：2 位
- 新子网掩码：/26 (24 + 2)

每个子网的范围：
┌─────────────────────────────────────────────────────────────┐
│ 子网 1：192.168.1.0/26                                      │
│   网络地址：192.168.1.0                                     │
│   广播地址：192.168.1.63                                    │
│   可用范围：192.168.1.1 ~ 192.168.1.62  (62 台主机)        │
├─────────────────────────────────────────────────────────────┤
│ 子网 2：192.168.1.64/26                                     │
│   网络地址：192.168.1.64                                    │
│   广播地址：192.168.1.127                                   │
│   可用范围：192.168.1.65 ~ 192.168.1.126  (62 台主机)      │
├─────────────────────────────────────────────────────────────┤
│ 子网 3：192.168.1.128/26                                    │
│   网络地址：192.168.1.128                                   │
│   广播地址：192.168.1.191                                   │
│   可用范围：192.168.1.129 ~ 192.168.1.190  (62 台主机)     │
├─────────────────────────────────────────────────────────────┤
│ 子网 4：192.168.1.192/26                                    │
│   网络地址：192.168.1.192                                   │
│   广播地址：192.168.1.255                                   │
│   可用范围：192.168.1.193 ~ 192.168.1.254  (62 台主机)      │
└─────────────────────────────────────────────────────────────┘
```

### 计算方法

```
步骤 1：确定子网大小
子网大小 = 256 / 子网数 = 256 / 4 = 64

步骤 2：确定每个子网的网络地址
子网 1：0
子网 2：64
子网 3：128
子网 4：192

步骤 3：计算广播地址（下一个子网 - 1）
子网 1 广播：64 - 1 = 63
子网 2 广播：128 - 1 = 127
子网 3 广播：192 - 1 = 191
子网 4 广播：256 - 1 = 255
```

## 判断两台主机是否在同一网段

**规则**：两个 IP 地址与各自的子网掩码按位与后，得到的网络地址相同，则在同一网段。

```bash
# 示例
主机 A: 192.168.1.100/24
主机 B: 192.168.1.200/24
子网掩码: 255.255.255.0

计算：
主机 A: 192.168.1.100 AND 255.255.255.0 = 192.168.1.0
主机 B: 192.168.1.200 AND 255.255.255.0 = 192.168.1.0

结论：192.168.1.0 = 192.168.1.0，在同一网段，可以直接通信
```

```bash
# 跨网段示例
主机 A: 192.168.1.100/24
主机 B: 192.168.2.100/24

计算：
主机 A: 192.168.1.100 AND 255.255.255.0 = 192.168.1.0
主机 B: 192.168.2.100 AND 255.255.255.0 = 192.168.2.0

结论：不在同一网段，需要通过路由器通信
```

## 超网（Supernetting）

超网是将多个连续的小网络合并成一个大网络，与子网划分相反。

**问题**：将 `192.168.0.0/24`、`192.168.1.0/24`、`192.168.2.0/24`、`192.168.3.0/24` 合并成一个超网。

```bash
分析：
- 4 个网络，需要向右移动 2 位（2^2 = 4）
- 新的网络前缀：/24 - 2 = /22
- 合并后的网络：192.168.0.0/22

验证：
192.168.0.0/22 包含：
  192.168.0.0/24
  192.168.1.0/24
  192.168.2.0/24
  192.168.3.0/24
```

## VLSM（可变长子网掩码）

VLSM 允许在同一个网络内使用不同的子网掩码，更好地利用地址空间。

**场景**：公司有三个部门，需要不同数量的 IP。

| 部门 | 主机需求 | 最小子网 |
|---|---|---|
| 工程部 | 100 人 | /25（126 地址） |
| 市场部 | 50 人 | /26（62 地址） |
| 财务部 | 20 人 | /27（30 地址） |

```bash
假设公司网络：192.168.1.0/24

分配方案：

┌──────────────────────────────────────────────────────────────┐
│ 192.168.1.0/25  → 工程部（126 可用）                         │
│  范围：192.168.1.1 ~ 192.168.1.126                          │
├──────────────────────────────────────────────────────────────┤
│ 192.168.1.128/26 → 市场部（62 可用）                         │
│  范围：192.168.1.129 ~ 192.168.1.190                         │
├──────────────────────────────────────────────────────────────┤
│ 192.168.1.192/27 → 财务部（30 可用）                         │
│  范围：192.168.1.193 ~ 192.168.1.222                         │
├──────────────────────────────────────────────────────────────┤
│ 192.168.1.224/27 → 预留（未来扩展）                          │
└──────────────────────────────────────────────────────────────┘
```

## 常用网络计算工具

### Python 计算

```python
import ipaddress

# 创建网络
net = ipaddress.ip_network('192.168.1.0/24')

# 基本信息
print(f"网络地址: {net.network_address}")
print(f"广播地址: {net.broadcast_address}")
print(f"子网掩码: {net.netmask}")
print(f"主机数: {net.num_addresses}")
print(f"可用主机: {net.num_addresses - 2}")

# 遍历子网
for sub in net.subnets(prefixlen_diff=2):
    print(f"子网: {sub}")

# 判断 IP 是否在网络中
ip = ipaddress.ip_address('192.168.1.100')
print(f"{ip} 在网络中: {ip in net}")

# 子网划分
small_nets = list(net.subnets(new_prefix=26))
for i, small in enumerate(small_nets):
    print(f"子网 {i+1}: {small} ({small.num_addresses - 2} 可用)")
```

## 面试追问

**面试官问**：一个 /24 的网络，能分配多少台主机？

/24 网络有 8 位主机位，总地址 256 个，减 2（网络地址和广播地址），可用主机数是 **254**。

**追问**：为什么需要网络地址和广播地址？

- 网络地址用于标识整个网络，路由表中使用
- 广播地址用于向网络内所有主机发送数据（ARP、DHCP Discovery 等）

**追问**：子网掩码 255.255.255.252 有什么用？

/30 子网只有 4 个地址，2 个可用。常用于点对点链路（如路由器间的直连接口），节省 IP 地址。

## 总结

- IPv4 地址是 32 位，点分十进制表示
- 子网掩码用连续 1（网络位）和连续 0（主机位）确定网络边界
- 可用主机数 `=` `2^(32-n)` ` -` ` 2`
- 子网划分通过借位增加网络位，减少每个子网的主机位
- 判断是否在同一网段：将两个 IP 分别与子网掩码按位与，结果相同则同网段

**记住公式**：`可用主机数 = 2^(32-前缀长度) - 2`
