# 网络层

网络层是整个互联网的核心。

传输层以下，TCP 和 UDP 只负责端到端的通信，但数据如何从你的电脑到达万里之外的服务器？这就是网络层要解决的问题。

IP（Internet Protocol）是网络层的核心协议，它定义了数据包的格式，规定了路由的选择方式。理解 IP 地址的编址方案、子网划分的原理、路由的工作机制，是掌握网络通信的关键。

## IP 的核心挑战

IP 协议面对的是一个看似简单却极其复杂的问题：**如何让数据包从任意一台主机到达另一台主机？**

答案是路由。全球数万台路由器协同工作，每台路由器只知道到某些网络的路径，数据包一跳一跳地前进，最终到达目的地。

这个过程中，有几个关键问题需要解决：

- 如何给主机分配地址，使得路由聚合成为可能？
- 如何划分子网，平衡网络规模和地址利用率？
- IPv4 地址即将耗尽，IPv6 如何解决地址空间问题？
- 私有 IP 如何访问互联网？NAT 技术如何工作？

## 学习路线

本章节按照从基础到深入的顺序展开：

**IP 地址与子网划分**：理解 IPv4 地址的结构，学会计算子网掩码和可用主机数，这是网络工程师的基本功。

**IPv4 与 IPv6**：对比两代协议，理解 IPv6 的地址空间优势和改进点。

**ICMP**：ping 和 traceroute 的基础，理解 ICMP 报文类型和作用。

**路由**：理解路由表、路由选择算法、常见路由协议。

**NAT**：理解网络地址转换的原理，以及它在 IPv4 延续中的历史角色。

掌握了这些，你就理解了互联网的"交通规则"。
