# 路由与路由协议

你的数据包是如何从北京发送到纽约的？

答案是：经过无数个路由器的"接力"，一跳一跳地前进。每个路由器只负责自己的"下一跳"，最终到达目的地。

这个过程看似简单，背后却藏着复杂的技术：路由表如何构建？路由器如何选择最优路径？自治系统之间如何协调？这些问题，决定了互联网的效率和稳定。

## 路由基础

### 路由表

路由表是路由器的"导航地图"，每条路由条目告诉路由器如何到达目标网络：

```bash
# Linux 查看路由表
ip route show

# 或
route -n

# 输出示例
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
0.0.0.0         192.168.1.1     0.0.0.0         UG    600    0        0 wlan0
192.168.1.0     0.0.0.0         255.255.255.0   U     600    0        0 wlan0
```

路由表条目字段：

| 字段 | 说明 |
|---|---|
| Destination | 目标网络或主机 |
| Genmask | 子网掩码（ip route 用 CIDR 表示） |
| Gateway | 下一跳网关，0.0.0.0 表示直连 |
| Flags | U=启用，G=使用网关，H=主机路由 |
| Metric | 路由代价，越小越优先 |
| Iface | 出接口 |

### 路由查找过程

当数据包到达时，路由器按以下顺序查找路由表：

```
┌─────────────────────────────────────────────────────────────┐
│                    路由查找顺序                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 主机路由（Host Route）  → 最精确匹配                     │
│     192.168.1.100/32                                        │
│                                                             │
│  2. 子网路由                                                 │
│     192.168.1.0/24                                          │
│                                                             │
│  3. 超网路由                                                 │
│     192.168.0.0/16                                          │
│                                                             │
│  4. 默认路由（Default Route）                                │
│     0.0.0.0/0 或 ::/0                                       │
│                                                             │
│  5. 丢弃（ICMP 不可达）                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

最长前缀匹配（Longest Prefix Match）是路由查找的核心原则。

### 默认路由

默认路由（Default Route）是"兜底"路由，匹配所有找不到明确路由的流量：

```bash
# Linux 添加默认路由
ip route add default via 192.168.1.1
# 或
ip route add 0.0.0.0/0 via 192.168.1.1

# 查看默认路由
ip route | grep default
```

:::warning 默认路由的风险
默认路由将所有未知流量发送到指定网关。如果配置错误，可能导致"路由黑洞"，流量被发送到不存在的地方。生产环境中谨慎使用。
:::

## 路由类型

### 直连路由

路由器接口配置的 IP 所在网段，自动生成直连路由：

```
路由器接口：192.168.1.1/24
直连路由：192.168.1.0/24 via 0.0.0.0 dev eth0
```

### 静态路由

手动配置的路由，适合小型网络或特殊路径：

```bash
# Linux 添加静态路由
# 到达 10.0.0.0/8 的流量经过 192.168.1.1
ip route add 10.0.0.0/8 via 192.168.1.1

# 到达特定主机
ip route add 10.0.0.1 via 192.168.1.1

# 删除静态路由
ip route del 10.0.0.0/8 via 192.168.1.1
```

静态路由的特点：
- 配置简单，适合小型网络
- 管理员完全控制路由行为
- 网络拓扑变化时需要手动更新
- 无法自动适应网络故障

### 动态路由

路由器之间自动交换路由信息，自动适应网络变化：

| 类型 | 说明 | 代表协议 |
|---|---|---|
| 距离矢量 | 分享到达目标网络的距离 | RIP、IGRP |
| 链路状态 | 分享链路状态信息 | OSPF、IS-IS |
| 混合型 | 综合两者特点 | EIGRP、BGP |

## 路由协议详解

### RIP（Routing Information Protocol）

RIP 是最简单的动态路由协议，适用于小型网络：

```bash
# RIP 特点
# - 使用跳数（Hop Count）作为度量值
# - 最大跳数 15，16 表示不可达
# - 每 30 秒广播更新
# - 简单但不适合大型网络
```

RIP 的问题：
- 跳数限制（`<=` 15），不适合大规模网络
- 收敛慢（网络变化后更新慢）
- 可能产生路由环路
- 不考虑链路带宽

:::tip RIP vs 路由环路
RIP 使用"计数到无穷大"机制避免环路：跳数超过 16 则认为不可达。但这会延缓收敛时间。
:::

### OSPF（Open Shortest Path First）

OSPF 是大型企业网络的主流协议：

```bash
# OSPF 特点
# - 基于链路状态（Link State）
# - 使用 Dijkstra 算法计算最短路径
# - 支持区域（Area）划分，减少更新量
# - 收敛快，触发式更新
# - 支持 VLSM、CIDR
# - 无跳数限制
```

OSPF 区域结构：

```
┌─────────────────────────────────────────────────────────────┐
│                    OSPF 区域结构                            │
│                                                             │
│     Area 0 (Backbone)                                       │
│    ┌───────────────────────┐                               │
│    │                       │                               │
│  ┌─┴───┐   ┌───┐   ┌───┴─┐                                │
│  │Area 1│   │Area│   │Area │                                │
│  │     │   │ 0  │   │  2  │                                │
│  └──┬──┘   └───┘   └───┬──┘                                │
│     │                   │                                    │
│     └───────────────────┘                                    │
│                                                             │
│  所有区域必须连接到 Area 0                                  │
│  Area 0 是骨干区域，其他区域是末梢区域                       │
└─────────────────────────────────────────────────────────────┘
```

### BGP（Border Gateway Protocol）

BGP 是互联网的核心协议，用于自治系统（AS）之间的路由：

```bash
# BGP 特点
# - 路径向量协议（Path Vector）
# - 传输 AS-Path、Next-Hop、MED 等属性
# - 支持路由策略和过滤
# - 路由表巨大（当前约 90 万条）
# - 主要用于 ISP 之间（eBGP）和大型企业（iBGP）
```

BGP 与 IGP 的区别：

| 方面 | IGP（如 OSPF、RIP） | BGP |
|---|---|---|
| 作用域 | 单个 AS | 整个互联网 |
| 路由数量 | 几千条 | 几十万条 |
| 收敛速度 | 快 | 慢 |
| 度量值 | 带宽、延迟等 | AS-Path、MED、Local Preference |
| 用途 | AS 内部路由 | AS 之间的路由 |

### BGP 路径属性

BGP 使用多种属性决定最优路径：

| 属性 | 说明 | 分类 |
|---|---|---|
| AS-Path | 经过的 AS 序列 | 公认必遵 |
| Next-Hop | 下一跳地址 | 公认必遵 |
| MED | 多出口鉴别器 | 可选传递 |
| Local Preference | 本地优先级 | 公认自由 |
| Origin | 路由来源（IGP/EGP/不完整） | 公认必遵 |

```
# BGP 选路顺序（简化）
1. 最高 Local Preference
2. 最短 AS-Path
3. 最低 Origin 类型（IGP > EGP > Incomplete）
4. 最低 MED
5. eBGP > iBGP
6. 最低 IGP Metric 到 Next-Hop
7. 最早学习到的路由
```

## 路由选择过程

### 完整路由选择流程

```
┌─────────────────────────────────────────────────────────────┐
│                   路由器转发数据包流程                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 解封装数据链路层帧，提取 IP 数据报                       │
│                                                             │
│  2. 检查 IP 头 checksum                                      │
│     ↓ 失败则丢弃                                            │
│                                                             │
│  3. TTL 减 1，检查是否为 0                                  │
│     ↓ 为 0 则发送 ICMP 超时，丢弃                            │
│                                                             │
│  4. 路由查找（最长前缀匹配）                                 │
│     ↓ 未找到则发送 ICMP 不可达，丢弃                        │
│                                                             │
│  5. TTL 再减 1，更新 checksum                                │
│                                                             │
│  6. 查找下一跳 MAC（ARP 或缓存）                            │
│                                                             │
│  7. 重新封装数据链路层帧并转发                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 负载均衡

当有多条等代价路由时，路由器可以负载均衡：

```bash
# Linux 查看负载均衡配置
ip route show

# 等代价负载均衡（默认行为）
# 路由缓存
ip route show cache

# ECMP（Equal-Cost Multi-Path）
# 多个下一跳分担流量
```

## 路由协议配置

### Cisco OSPF 配置

```bash
# 进入 OSPF 配置模式
router ospf 1

# 宣告直连网络（反掩码）
network 192.168.1.0 0.0.0.255 area 0
network 10.0.0.0 0.0.0.255 area 1

# 查看 OSPF 邻居
show ip ospf neighbor

# 查看 OSPF 数据库
show ip ospf database

# 查看路由表中的 OSPF 路由
show ip route ospf
```

### Linux Quagga/Zebra 配置

```bash
# 配置文件
/etc/quagga/zebra.conf
/etc/quagga/ospfd.conf

# zebra.conf 示例
hostname Router1
password zebra
enable password zebra

# ospfd.conf 示例
hostname OSPF-Router1
password zebra
router ospf
  network 192.168.1.0/24 area 0.0.0.0
  network 10.0.0.0/8 area 0.0.0.0
```

### BGP 配置（Cisco）

```bash
# 进入 BGP 配置
router bgp 65001

# 指定邻居
neighbor 192.168.1.1 remote-as 65002

# 宣告网络
network 192.168.0.0 mask 255.255.0.0

# 查看 BGP 邻居
show ip bgp neighbors

# 查看 BGP 路由表
show ip bgp
```

## 常见问题排查

### 路由环路

```bash
# 症状：数据包在两个路由器之间来回跳
# TTL 耗尽后数据包被丢弃

# 排查方法
traceroute <destination>

# 查看路由表是否有错误
show ip route
```

### 非对称路由

```bash
# 去程和回程走不同路径
# 正常情况下无害，但可能导致：
# - ACL 难以配置
# - NAT 问题
# - 流量监控困难

# 查看双向路由
traceroute <destination>
traceroute <source> from <destination>
```

### 默认路由问题

```bash
# 默认路由指向错误
# 所有流量被发送到不存在的地方（路由黑洞）

# 检查默认路由
ip route show default
traceroute 8.8.8.8

# 查看是否有默认路由
ip route | grep default
```

## 总结

路由是互联网的核心技术：

- 路由表使用最长前缀匹配选择最佳路径
- 静态路由适合小型简单网络，动态路由适合大型复杂网络
- RIP 简单但有限制（跳数 `,` `15`），OSPF 是企业网主流
- BGP 是互联网骨干协议，处理 AS 间路由
- 路由选择考虑多个因素：带宽、延迟、跳数、策略
