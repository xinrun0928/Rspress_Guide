# 常见端口与协议

你的浏览器能同时打开十几个网页，微信能一边下载文件一边接收消息。这是因为操作系统用**端口**来区分同一台主机上的不同网络连接。

端口是传输层最重要的概念之一。理解端口的编号规则、掌握常用端口的用途，是网络基础中的基础。

## 端口的概念

当你的电脑访问一个网站时，数据包的目标 IP 确定了服务器，但服务器上可能运行着几十个服务——HTTP 服务、Nginx、MySQL、Redis……服务器怎么知道把数据包交给谁处理？

答案就是**端口号**。

```
┌─────────────────────────────────────────────────────────────┐
│                        服务器                               │
│                                                             │
│   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐     │
│   │   :80       │   │   :443      │   │   :3306     │     │
│   │   HTTP      │   │   HTTPS     │   │   MySQL     │     │
│   └─────────────┘   └─────────────┘   └─────────────┘     │
│                                                             │
│              端口号  ←  操作系统路由到对应进程              │
└─────────────────────────────────────────────────────────────┘
```

IP 地址解决了"主机到主机"的通信问题，端口号解决了"主机到进程"的通信问题。两者结合，才能实现完整的端到端通信。

## 端口号范围

端口号是一个 16 位的无符号整数，范围 `0` `~` `65535`：

| 范围 | 名称 | 说明 |
|---|---|---|
| `0` `~` `1023` | 知名端口（Well-Known Ports） | 保留给系统服务，如 HTTP、HTTPS、SSH |
| `1024` `~` `49151` | 注册端口（Registered Ports） | 需向 IANA 注册，如 MySQL、Redis |
| `49152` `~` `65535` | 动态端口（Dynamic Ports） | 客户端随机分配，不需注册 |

:::tip 为什么是 1024
Unix 系统规定，只有 root 用户才能绑定 `0` `~` `1023` 端口。这是为了防止普通用户占用常用服务端口。
:::

## 常用端口一览

| 端口 | 协议 | 说明 |
|---|---|---|
| 80 | TCP | HTTP 明文传输 |

| 8080 | TCP | HTTP 代理端口，常见于开发环境 |
| 8443 | TCP | HTTPS 代理端口 |

### 远程访问

| 端口 | 协议 | 说明 |
|---|---|---|
| 22 | TCP | SSH（Secure Shell），加密远程登录 |
| 23 | TCP | Telnet，明文传输，已废弃 |
| 3389 | TCP | Windows RDP 远程桌面 |

:::warning Telnet 的危险
Telnet 传输数据是明文的，用户名和密码在网络上"裸奔"。生产环境严禁使用 Telnet，SSH 是唯一的远程登录选择。
:::

## 查看端口状态

### Linux 查看端口

```bash
# 查看所有监听的端口
ss -tlnp

# 等价于 netstat
netstat -tlnp

# 查看特定端口是否被占用
ss -tlnp | grep :80
netstat -tlnp | grep :80

# 检查端口连通性
nc -zv 192.168.1.100 80
telnet 192.168.1.100 80

# 查看端口对应的进程
lsof -i :80
```

### macOS 查看端口

```bash
# 查看监听端口
lsof -i -P | grep LISTEN

# 查看特定端口
lsof -i :8080

# 使用 netstat
netstat -anv | grep LISTEN
```

### Windows 查看端口

```powershell
# 查看监听端口
netstat -ano | findstr LISTENING

# 查看特定端口
netstat -ano | findstr :80

# 查看端口对应的进程 PID
netstat -ano | findstr :3306

# 根据 PID 查看进程名
tasklist | findstr <PID>
```

## 端口与协议的关系

端口和协议是绑定的吗？

答案是：**看情况。**

- 80 端口通常跑 HTTP，但理论上任何服务都可以监听 80 端口
- HTTP 协议通常跑在 80 端口，但也可以跑在其他端口（如 8080）

```
┌────────────────────────────────────────────────────────────┐
│  端口是传输层的地址，协议是应用层的语义                       │
│                                                            │
│  80 端口 + HTTP 协议 = Web 服务                            │
│  80 端口 + SSH 协议 = 错误配置（但系统不阻止）               │
│  8080 端口 + HTTP 协议 = 开发环境 Web 服务                  │
└────────────────────────────────────────────────────────────┘
```

不过，有一些约定俗成的惯例：
- Web 服务通常在 80/443 端口
- SSH 在 22 端口
- 数据库在各自的标准端口

遵守这些惯例能让运维更简单，也减少沟通成本。

## 端口安全

### 最小化暴露原则

生产环境中，只开放必要的端口：

```bash
# 检查开放端口
ss -tlnp

# 关闭不必要的服务
systemctl stop <service-name>
systemctl disable <service-name>

# 用防火墙限制访问
# iptables 示例：只允许 192.168.1.0/24 访问 MySQL
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 3306 -j ACCEPT
iptables -A INPUT -p tcp --dport 3306 -j DROP
```

### 常见攻击面

| 端口 | 风险 | 安全建议 |
|---|---|---|
| 22 | SSH 暴力破解 | 使用密钥登录，禁用密码；修改默认端口；使用 fail2ban |
| 23 | Telnet 明文传输 | 禁止使用，迁移到 SSH |
| 3306 | 数据库未授权访问 | 仅允许内网访问；使用强密码；绑定 localhost |
| 6379 | Redis 未授权访问 | 绑定 127.0.0.1；设置密码；禁用危险命令 |
| 11211 | Memcached DDoS 放大 | 绑定 127.0.0.1；禁用 UDP |

:::danger 数据库端口暴露的教训
每年都有因 MongoDB、Redis 未授权访问导致数据泄露的新闻。这些数据库默认绑定 `0.0.0.0` 且无密码，上线前务必检查配置。
:::

## 端口转发与代理

### 本地端口转发

将远程服务器的端口映射到本地：

```bash
# 将远程 3306 端口映射到本地 33306
ssh -L 33306:localhost:3306 user@remote-server

# 现在连接本地的 33306 就等于连接远程的 3306
mysql -h localhost -P 33306 -u root -p
```

### 远程端口转发

将本地端口暴露给远程服务器：

```bash
# 将本地 8080 暴露给远程服务器的 80
ssh -R 80:localhost:8080 user@remote-server
```

### Nginx 反向代理

```nginx
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 总结

端口是传输层的核心概念：

- 端口号范围 `0` `~` `65535`，分为知名端口、注册端口、动态端口
- 常用服务有约定俗成的端口：HTTP(80)、HTTPS(443)、SSH(22)、MySQL(3306)
- 生产环境遵循最小化暴露原则，非必要端口不开放
- 用 `ss`、`netstat`、`lsof` 等工具查看端口状态

记住几个高频端口：80、443、22、3306、6379、8080。面试中问到端口相关问题，这些是最可能被追问的。
