# OSI 七层模型与 TCP/IP 四层模型

凌晨 2 点，你的程序突然连不上数据库。日志显示"Connection refused"。你开始排查：网络通吗？端口通吗？防火墙开了吗？服务启动了吗？

这个排查顺序，从物理层一路到应用层，恰好体现了网络分层思想的核心：**问题定位时，从底层往高层逐层排查是最有效的方式。**

## 为什么需要分层

计算机网络是一个极其复杂的系统。如果把所有功能混在一起实现，网络协议的设计会变成一团乱麻： HTTP 要关心电信号怎么传？ TCP 要处理 wifi 信号干扰？

分层思想解决这个问题的方式很简单：**每层只关心自己的职责，通过接口与上下层交互。**

```
┌────────────────────────────────────────────────────────────┐
│  应用层：HTTP 请求要发什么数据？                              │
├────────────────────────────────────────────────────────────┤
│  传输层：这些数据如何可靠地送到对方端口？                      │
├────────────────────────────────────────────────────────────┤
│  网络层：数据包如何跨越路由器到达目标网络？                    │
├────────────────────────────────────────────────────────────┤
│  数据链路层：帧如何通过交换机到达下一个节点？                  │
├────────────────────────────────────────────────────────────┤
│  物理层：比特流如何在网线/光纤上传输？                        │
└────────────────────────────────────────────────────────────┘
```

当你无法访问某个网站时，最先检查的是物理连接（网线插好没？wifi 连上没？），然后是 IP 层（能 ping 通网关吗？），接着是端口（telnet 80 端口通吗？），最后才是应用层（服务正常吗？）。

## OSI 七层模型

OSI（Open Systems Interconnection）模型是国际标准化组织在 1984 年提出的参考模型。它不是具体实现，而是理论框架，为理解网络通信提供了统一语言。

### 七层详解

**1. 物理层（Physical Layer）**

物理层关注的是最底层的问题：如何用电压、电流、光脉冲、无线电波来表示比特流。

```bash
# 物理层的设备和技术
- 集线器（Hub）：广播信号，不做任何处理
- 中继器（Repeater）：放大信号，延长传输距离
- 传输介质：双绞线、光纤、同轴电缆、无线电波
- 传输速率：100Mbps、1Gbps、10Gbps、100Gbps
```

物理层只传输原始比特流，不关心这些比特代表什么含义。

**2. 数据链路层（Data Link Layer）**

数据链路层在物理层之上，把比特流组织成"帧"（Frame），并实现**相邻节点**之间的可靠传输。

核心职责：
- 帧的封装和差错检测（CRC 校验）
- MAC 地址寻址
- 媒体访问控制（CSMA/CD、CSMA/CA）
- 流量控制（部分协议）

```bash
# 数据链路层的协议和设备
- 协议：Ethernet（IEEE 802.3）、PPP、HDLC、帧中继
- 设备：交换机（Switch）、网桥（Bridge）
- 地址：MAC 地址（48 位）
```

交换机工作在数据链路层，基于 MAC 地址转发帧。

**3. 网络层（Network Layer）**

网络层实现**端到端**的数据传输，关注的是如何跨越多个网络到达目的地。

核心职责：
- 逻辑寻址（IP 地址）
- 路由选择
- 分段与重组
- 拥塞控制

```bash
# 网络层的协议和设备
- 协议：IP、ICMP、ARP、RARP、OSPF、BGP
- 设备：路由器（Router）、三层交换机
- 地址：IP 地址（32 位 IPv4 / 128 位 IPv6）
```

路由器的核心功能是根据路由表决定数据包的下一跳。

**4. 传输层（Transport Layer）**

传输层在网络层之上，实现了端系统（不是路由器）之间的逻辑通信。核心问题是：**哪个应用程序收到了这个数据包？**

```bash
# 传输层的协议
- TCP：面向连接、可靠传输、流量控制、拥塞控制
- UDP：无连接、不可靠、面向报文
- SCTP：面向连接、可同时支持多个流
```

端口号是传输层的核心概念，范围 `0` `~` `65535`，其中 `0` `~` `1023` 是知名端口。

**5. 会话层（Session Layer）**

会话层管理应用程序之间的会话，负责建立、维护和终止通信连接。

```bash
# 会话层的职责
- 会话管理：建立、同步、检查点
- 认证和授权
- 协议示例：NetBIOS、RPC、SQL
```

在实际实现中，会话层功能通常被合并到应用层或传输层中。

**6. 表示层（Presentation Layer）**

表示层关注数据的表示方式，负责语法和语义之间的转换。

```bash
# 表示层的职责
- 数据格式转换：ASCII、EBCDIC
- 数据压缩：gzip、deflate、brotli
- 数据加密：SSL/TLS
- 序列化：JSON、XML、Protocol Buffers
```

HTTPS 中的 TLS 加密，就是表示层的典型应用。

**7. 应用层（Application Layer）**

应用层是用户与网络的接口，直接为用户提供网络服务。

```bash
# 应用层的协议
- Web：HTTP、HTTPS
- 邮件：SMTP、POP3、IMAP
- 文件：FTP、SFTP
- 远程登录：SSH、Telnet
- 域名：DNS
- 消息队列：AMQP、MQTT
```

## TCP/IP 四层模型

TCP/IP 模型是实际互联网使用的标准，它将 OSI 七层简化为四层：

| TCP/IP 四层 | OSI 七层对应 | 核心协议 |
|---|---|---|
| 应用层 | 应用层 + 表示层 + 会话层 | HTTP、DNS、FTP、SMTP、SSH |
| 传输层 | 传输层 | TCP、UDP |
| 网际层 | 网络层 | IP、ICMP、ARP |
| 网络接口层 | 数据链路层 + 物理层 | Ethernet、PPP、Wi-Fi |

:::tip 模型对比
OSI 是"教学模型"，分层细致，概念清晰。TCP/IP 是"工程模型"，实用性强，被实际部署。
:::

## 数据封装与解封装

数据从发送端到接收端，每层都会加上自己的头部（有时还有尾部），这个过程叫做**封装**。

以发送一个 HTTP 请求为例：

```
发送端：
┌────────────┐
│   HTTP     │  ← 应用层：我要访问 /api/users
├────────────┤
│   TCP      │  ← 传输层：源端口 54321，目标端口 80，SYN=1
├────────────┤
│   IP       │  ← 网络层：源 IP 192.168.1.100，目标 IP 93.184.216.34
├────────────┤
│  Ethernet  │  ← 数据链路层：源 MAC aa:bb:cc:dd:ee:ff，目标 MAC 路由器的 MAC
├────────────┤
│  比特流    │  ← 物理层：转换为电/光/射频信号
└────────────┘

传输过程中，每经过一个路由器：
┌────────────┐
│  旧 IP 头  │  ← 路由器剥去旧 IP 头
├────────────┤
│  新 IP 头  │  ← 路由器添加新 IP 头（目标地址不变）
├────────────┤
│  旧帧头    │  ← 路由器剥去旧帧头
├────────────┤
│  新帧头    │  ← 路由器添加新帧头（下一跳 MAC）
└────────────┘

接收端（反向解封装）：
┌────────────┤
│  解帧头    │  ← 数据链路层：剥去 Ethernet 头
├────────────┤
│  解IP头    │  ← 网络层：剥去 IP 头
├────────────┤
│  解TCP头   │  ← 传输层：剥去 TCP 头
├────────────┤
│   HTTP     │  ← 应用层：获得原始请求
└────────────┘
```

这个封装-解封装的过程，完美体现了分层设计的精髓：**每层只关心自己的头部，对上层和下层透明。**

## 常见的协议与设备对应关系

理解每层的代表性协议和设备，有助于在排查问题时快速定位：

```
┌──────────────────────────────────────────────────────────────┐
│  应用层    ← 路由器、负载均衡器（应用层负载均衡）              │
├──────────────────────────────────────────────────────────────┤
│  传输层    ← 防火墙（四层安全策略）                            │
├──────────────────────────────────────────────────────────────┤
│  网络层    ← 路由器、三层交换机                                │
├──────────────────────────────────────────────────────────────┤
│  数据链路层 ← 交换机、二层交换机、网桥                          │
├──────────────────────────────────────────────────────────────┤
│  物理层    ← 集线器、中继器、网卡、网线、光纤                  │
└──────────────────────────────────────────────────────────────┘
```

## 面试追问

**面试官问**：路由器和交换机有什么区别？

这是考察对网络分层理解的经典问题。

**标准答案**：
- 交换机工作在数据链路层，基于 MAC 地址转发帧，连接的是同一网络内的设备
- 路由器工作在网络层，基于 IP 地址转发数据包，连接的是不同网络
- 交换机内部有一张 MAC 地址表，路由器内部有一张路由表

**追问**：那三层交换机和普通交换机有什么区别？

三层交换机在普通二层交换机的基础上增加了路由功能，可以基于 IP 地址进行转发，性能比路由器更高，适合作为局域网的核心设备。

## 总结

分层是计算机网络最核心的设计思想：

- OSI 七层模型提供了理解网络的标准框架
- TCP/IP 四层模型是实际部署的工程标准
- 数据封装体现了"关注点分离"的设计哲学
- 理解每层的核心协议和设备，是排查网络问题的基本功

记住一个排查问题的顺序：物理层 → 数据链路层 → 网络层 → 传输层 → 应用层。这是从底层到高层、从简单到复杂的排查路径。
