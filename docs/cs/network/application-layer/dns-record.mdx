# DNS 记录类型

DNS 不只是把域名解析成 IP 地址。它还能指定邮件服务器、验证域名所有权、负载均衡……

理解不同的 DNS 记录类型，是配置域名服务的基础。

## 记录类型一览

| 类型 | 名称 | 用途 |
|---|---|---|
| A | 地址记录 | 域名 -> IPv4 地址 |
| AAAA | IPv6 地址记录 | 域名 -> IPv6 地址 |
| CNAME | 规范名称记录 | 域名别名 |
| MX | 邮件交换记录 | 邮件服务器 |
| NS | 域名服务器记录 | 指定 DNS 服务器 |
| TXT | 文本记录 | 验证、SPF、DKIM |
| SOA | 授权开始记录 | 区域配置 |
| PTR | 指针记录 | IP -> 域名（反向查询） |
| SRV | 服务记录 | 指定服务位置 |
| CAA | 证书颁发机构授权 | 指定可信 CA |

## A 记录：IPv4 地址

最常用的 DNS 记录，将域名指向 IPv4 地址。

### 格式

```
主机名    TTL    记录类型    值
www      3600    IN        A      93.184.216.34
```

### 示例

```bash
dig A www.example.com

# 输出：
; <<>> DiG 9.18.1 <<>> A www.example.com
;; ANSWER SECTION:
www.example.com.    86400    IN    A    93.184.216.34

# TTL = 86400 秒（1 天）
# IN = Internet 类别
# A = IPv4 地址记录
# 93.184.216.34 = 目标 IP
```

### 多个 A 记录（负载均衡）

DNS 可以返回多个 IP 地址，实现简单的负载均衡：

```bash
dig A lb.example.com

# 输出：
lb.example.com.    300    IN    A    1.2.3.4
lb.example.com.    300    IN    A    5.6.7.8
lb.example.com.    300    IN    A    9.10.11.12
```

## AAAA 记录：IPv6 地址

与 A 记录类似，但指向 IPv6 地址。

### 格式

```
主机名    TTL    记录类型    值
www      3600    IN        AAAA    2606:2800:220:1::
```

### 示例

```bash
dig AAAA www.example.com

# 输出：
www.example.com.    86400    IN    AAAA    2606:2800:220:1::
```

## CNAME 记录：别名

CNAME 将域名指向另一个域名，而不是直接指向 IP。

### 格式

```
主机名    TTL    记录类型    值
blog      3600    IN        CNAME    www.example.com.
```

### CNAME vs A 记录

```
CNAME 的作用：
blog.example.com -> www.example.com -> 93.184.216.34

访问 blog.example.com 时：
1. DNS 查询 blog.example.com -> www.example.com
2. DNS 查询 www.example.com -> 93.184.216.34
3. 最终解析为 IP
```

### 使用场景

- **博客平台**：用户博客 -> 平台域名
- **CDN**：静态资源 -> CDN 域名
- **负载均衡**：多个服务 -> 统一入口

```bash
# 常见 CDN 配置
static.example.com.    300    IN    CNAME    cdn.example.net.
cdn.example.net.       300    IN    A        123.45.67.89
```

### CNAME 的限制

CNAME 不能与其他记录共存于同一主机名：

```
错误配置：
@    3600    IN    A      93.184.216.34
@    3600    IN    MX     10 mail.example.com.

正确配置：
@    3600    IN    A      93.184.216.34
mail    3600    IN    MX     10 mail.example.com.
```

## MX 记录：邮件服务器

MX 记录指定处理邮件的服务器。

### 格式

```
主机名    TTL    记录类型    优先级    值
@      3600    IN        MX      10      mail.example.com.
```

### 优先级

数字越小优先级越高。如果高优先级服务器不可用，会尝试低优先级服务器：

```bash
dig MX example.com

# 输出：
example.com.    3600    IN    MX    10    mail.example.com.
example.com.    3600    IN    MX    20    mail2.example.com.
```

### MX 配置要求

MX 记录必须指向一个域名（通常是 A 记录），不能直接指向 IP：

```
错误：
@    IN    MX    10    93.184.216.34

正确：
mail    IN    A      93.184.216.34
@      IN    MX    10    mail.example.com.
```

## NS 记录：域名服务器

NS 记录指定域名的权威 DNS 服务器。

### 格式

```
主机名    TTL    记录类型    值
@      172800    IN        NS      ns1.example.com.
@      172800    IN        NS      ns2.example.com.
```

### 查看 NS 记录

```bash
dig NS example.com

# 输出：
example.com.    172800    IN    NS    a.iana-servers.net.
example.com.    172800    IN    NS    b.iana-servers.net.
```

### 权威服务器 vs 递归服务器

- **权威服务器**：存储域名的实际 DNS 记录（由域名注册商配置）
- **递归服务器**：替客户端查询 DNS（如 8.8.8.8）

## TXT 记录：文本信息

TXT 记录存储任意文本信息，常用于验证和反垃圾邮件。

### SPF：发送方策略框架

SPF 记录指定哪些服务器可以发送该域名的邮件：

```bash
# 只允许 mx1.example.com 发送邮件
example.com.    3600    IN    TXT    "v=spf1 mx mx1.example.com -all"

# 解释：
# v=spf1：SPF 版本 1
# mx：允许 MX 服务器发送
# mx1.example.com：允许这个 IP 发送
# -all：其他来源全部拒绝
```

### DKIM：域名密钥识别邮件

DKIM 记录存储公钥，用于验证邮件签名：

```bash
# DKIM 记录格式
selector._domainkey.example.com.    3600    IN    TXT    "v=DKIM1; k=rsa; p=MIGfMA0GCSqGSIb3DQE..."
```

### DMARC：基于域名的消息认证

DMARC 记录结合 SPF 和 DKIM，指定处理策略：

```bash
example.com.    3600    IN    TXT    "v=DMARC1; p=quarantine; rua=mailto:dmarc@example.com"
```

### Google 域名验证

添加 TXT 记录验证域名所有权：

```bash
# Google Search Console 验证
google-site-verification    3600    IN    TXT    "google-site-verification=xxxxx"

# Google Workspace
google._domainkey    3600    IN    TXT    "v=DKIM1; k=rsa; p=xxxxx"
```

## CAA 记录：证书授权

CAA 记录指定哪些 CA 可以为该域名颁发证书。

### 格式

```
主机名    TTL    记录类型    值
@      3600    IN        CAA    0    issue    "letsencrypt.org"
@      3600    IN        CAA    0    issuewild    "letsencrypt.org"
```

### 参数

| 参数 | 说明 |
|---|---|
| `issue` | 允许颁发证书的 CA |
| `issuewild` | 允许颁发通配符证书的 CA |
| `iodef` | 违规通知 URL |

## SRV 记录：服务定位

SRV 记录指定特定服务的位置。

### 格式

```
_service._proto.name.    TTL    IN    SRV    优先级    权重    端口    目标
_sip._tcp.example.com.    3600    IN    SRV    10        5       5060    sip.example.com.
```

### 示例

```bash
# SIP 服务
_sip._tcp.example.com.    3600    IN    SRV    10    5    5060    sip.example.com.

# XMPP 服务
_xmpp._tcp.example.com.    3600    IN    SRV    10    0    5222    xmpp.example.com.
```

## PTR 记录：反向 DNS

PTR 记录将 IP 地址反向解析为域名（用于反垃圾邮件验证）。

### 格式

```
# IP 地址反序 + .in-addr.arpa（IPv4）或 .ip6.arpa（IPv6）
# 93.184.216.34 -> 34.216.184.93.in-addr.arpa
34.216.184.93.in-addr.arpa.    3600    IN    PTR    www.example.com.
```

### 查看 PTR 记录

```bash
dig -x 93.184.216.34

# 或使用 host 命令
host 93.184.216.34
```

## SOA 记录：授权开始

SOA 记录包含区域管理的基本信息。

```bash
dig SOA example.com

# 输出：
example.com.    3600    IN    SOA    ns1.example.com. admin.example.com. 2026032901 7200 3600 1209600 86400
```

### 参数说明

| 参数 | 说明 |
|---|---|
| `Serial` | 序列号，区域传输时比较 |
| `Refresh` | 辅助 DNS 刷新间隔 |
| `Retry` | 辅助 DNS 重试间隔 |
| `Expire` | 辅助 DNS 过期时间 |
| `Minimum TTL` | 默认 TTL |

## 实际应用

### 查看所有记录

```bash
# 使用 dig 查看所有记录
dig example.com ANY

# 使用 dig 查看完整信息
dig @8.8.8.8 example.com +trace
```

### DNS 服务商配置示例

```bash
# Cloudflare DNS 配置
# A 记录
www    300    A    93.184.216.34
api    300    A    1.2.3.4

# CNAME 记录
blog    300    CNAME    @

# MX 记录
@      300    MX    10    mail.example.com.

# TXT 记录
@      300    TXT    "v=spf1 mx ~all"
```

## 面试追问

### DNS 记录 TTL 设为多少合适？

没有固定答案，取决于需求：
- 需要快速更新：TTL = 300 秒或更低
- 稳定服务：TTL = 3600 秒或更高
- 迁移时：提前降低 TTL，迁移后再提高

### 为什么 MX 记录的优先级是反的？

MX 优先级数字越小，优先级越高。数字 10 比 20 优先级高。

### TXT 记录最长可以多长？

单个 TXT 记录的字符串最长 255 字符。如果需要更长的文本，可以分成多个字符串。

### CNAME 和 A 记录哪个更快？

理论上 A 记录更快（少一次 DNS 查询）。但现代 DNS 解析器会缓存 CNAME 链，性能差异可以忽略。

## 总结

DNS 记录类型丰富：

| 类型 | 用途 |
|---|---|
| A/AAAA | 域名 -> IP 地址 |
| CNAME | 域名别名 |
| MX | 邮件服务器 |
| NS | DNS 服务器 |
| TXT | 验证、SPF、DKIM |
| CAA | 证书授权 |
| SRV | 服务定位 |
| PTR | 反向查询 |

理解这些记录类型，才能正确配置域名服务。
