# 邮件协议

当你在 Gmail 或 Outlook 中发送一封邮件时，背后是 SMTP、POP3、IMAP 三个协议在协同工作。

理解这些协议，你才能理解邮件系统的工作原理。

## 邮件系统架构

### 邮件收发流程

```
发件人                               收件人
   |                                    |
   |  1. MUA（邮件客户端）              |
   |     编写邮件                       |
   v                                    |
   |  2. SMTP（发送）                   |
   +-----------> MTA（邮件服务器）--------+
                        |
                        | 3. SMTP（中转）
                        v
                MTA（邮件服务器）
                        |
                        | 4. SMTP（投递）
                        v
   +-----------> MDA（邮件投递代理）------+
   |                                    |
   |  5. POP3/IMAP（拉取）              |
   v                                    |
收件人 MUA <----------------------------+
```

### 术语解释

| 缩写 | 全称 | 说明 |
|---|---|---|
| MUA | Mail User Agent | 邮件用户代理（邮件客户端） |
| MTA | Mail Transfer Agent | 邮件传输代理（邮件服务器） |
| MDA | Mail Delivery Agent | 邮件投递代理 |
| MSA | Mail Submission Agent | 邮件提交代理 |

## SMTP 协议

SMTP（Simple Mail Transfer Protocol）是**发送**邮件的协议。

### 协议端口

```
SMTP：25（标准）、587（邮件提交）
SMTPS：465（加密，SMTP over SSL）
```

### 基本命令

```
客户端                          服务器
  |                               |
  |<------ 220 mail.example.com ----|  服务就绪
  |                               |
  |------ EHLO localhost --------->|
  |<------ 250 HELP ----------------|
  |<------ 250-AUTH LOGIN PLAIN ----|
  |<------ 250 SIZE 10240000 -------|
  |                               |
  |------ AUTH LOGIN ------------->|
  |<------ 334 VXNlcm5hbWU6 -------|
  |------ dXNlckBleGFtcGxlLmNvbQ--|
  |<------ 334 UGFzc3dvcmQ6 --------|
  |------ c2VjcmV0 ----------------|
  |<------ 235 Authentication OK ---|
  |                               |
  |------ MAIL FROM:<a@example.com> ->|
  |<------ 250 OK ------------------|
  |                               |
  |------ RCPT TO:<b@example.org> -->|
  |<------ 250 OK ------------------|
  |                               |
  |------ DATA ------------------->|
  |<------ 354 Start mail input ---|
  |                               |
  |------ From: a@example.com ---->|
  |------ To: b@example.org ------->|
  |------ Subject: Hello ----------->|
  |------                          -->|
  |------ This is the body. ------->|
  |------ . ------------------------>|
  |<------ 250 OK Message accepted ->|
  |                               |
  |------ QUIT ------------------->|
  |<------ 221 Bye ----------------|
```

### SMTP 命令一览

| 命令 | 说明 |
|---|---|
| `HELO` | 开始会话（简单认证） |
| `EHLO` | 开始会话（扩展认证） |
| `AUTH` | 身份认证 |
| `MAIL FROM` | 发件人地址 |
| `RCPT TO` | 收件人地址 |
| `DATA` | 开始传输邮件内容 |
| `QUIT` | 关闭连接 |
| `RSET` | 重置会话 |
| `NOOP` | 空操作（保活） |

### 发送邮件示例

```python
import smtplib

# 使用 SMTP 发送邮件
server = smtplib.SMTP('smtp.example.com', 587)
server.starttls()  # 启用 TLS
server.login('user@example.com', 'password')

msg = '''From: sender@example.com
To: recipient@example.org
Subject: Test Email

This is the email body.'''

server.sendmail('sender@example.com', 'recipient@example.org', msg)
server.quit()
```

## POP3 协议

POP3（Post Office Protocol version 3）是**接收**邮件的协议，用于将邮件从服务器下载到本地。

### 协议端口

```
POP3：110
POP3S：995（加密，POP3 over SSL）
```

### 会话流程

```
客户端                          服务器
  |                               |
  |<------ +OK POP3 server ready --|
  |                               |
  |------ USER user --------------->|
  |<------ +OK --------------------|
  |                               |
  |------ PASS password ---------->|
  |<------ +OK --------------------|
  |                               |
  |------ STAT -------------------->|
  |<------ +OK 2 1024 --------------|  2 封信，共 1024 字节
  |                               |
  |------ LIST -------------------->|
  |<------ +OK --------------------|
  |<------ 1 512 ------------------|
  |<------ 2 512 ------------------|
  |<------ . ----------------------|
  |                               |
  |------ RETR 1 ----------------->|
  |<------ +OK 512 octets --------|
  |<------ [邮件内容] ---------------|
  |<------ . ----------------------|
  |                               |
  |------ DELE 1 ----------------->|
  |<------ +OK --------------------|
  |                               |
  |------ QUIT ------------------->|
  |<------ +OK -------------------|
```

### POP3 命令一览

| 命令 | 说明 |
|---|---|
| `USER` | 指定用户名 |
| `PASS` | 指定密码 |
| `STAT` | 获取邮件数量和总大小 |
| `LIST` | 列出所有邮件 |
| `RETR n` | 获取第 n 封信 |
| `DELE n` | 删除第 n 封信 |
| `TOP n m` | 获取第 n 封信的头 m 行 |
| `UIDL` | 获取邮件唯一标识 |
| `QUIT` | 退出 |

### POP3 的问题

```
POP3 的缺点：
- 邮件下载后通常从服务器删除
- 多设备使用不便（一个设备下载后，其他设备看不到）
- 不能同步邮件状态（已读/未读）
```

## IMAP 协议

IMAP（Internet Message Access Protocol）解决了 POP3 的问题，支持**多设备同步**。

### 协议端口

```
IMAP4：143
IMAPS：993（加密，IMAP over SSL）
```

### 核心概念

```
IMAP vs POP3：
POP3：下载邮件到本地，服务器只是"仓库"
IMAP：服务器是"主副本"，本地只是"缓存"
```

### 会话流程

```
客户端                          服务器
  |                               |
  |<------ * OK IMAP4rev1 ready --|
  |                               |
  |------ A001 LOGIN user pass -->|
  |<------ A001 OK LOGIN completed|
  |                               |
  |------ A002 LIST "" "*" ------>|
  |<------ * LIST (\HasNoChildren) "/" INBOX|
  |<------ A002 OK LIST completed |
  |                               |
  |------ A003 SELECT INBOX ----->|
  |<------ * 3 EXISTS -------------|  3 封信
  |<------ * 0 RECENT ------------|
  |<------ * FLAGS (\Seen \Answer...)|
  |<------ A003 OK SELECT completed|
  |                               |
  |------ A004 FETCH 1 BODY[TEXT] ->|
  |<------ * 1 FETCH (BODY[TEXT] {...})|
  |<------ A004 OK FETCH completed|
  |                               |
  |------ A005 STORE 1 +FLAGS (\Seen) ->|
  |<------ * 1 FETCH (FLAGS (\Seen))|
  |<------ A005 OK STORE completed|
```

### IMAP 命令一览

| 命令 | 说明 |
|---|---|
| `LOGIN` | 登录 |
| `SELECT` | 选择邮箱 |
| `LIST` | 列出邮箱 |
| `FETCH` | 获取邮件内容 |
| `STORE` | 修改邮件标志 |
| `SEARCH` | 搜索邮件 |
| `COPY` | 复制邮件 |
| `MOVE` | 移动邮件 |
| `CREATE/DELETE/RENAME` | 邮箱管理 |
| `LOGOUT` | 退出 |

### IMAP 的优势

```
IMAP 的优势：
- 多设备同步：所有设备看到相同的邮件状态
- 按需获取：只下载邮件头部或部分内容
- 远程操作：在服务器上管理邮件
- 搜索：服务器端搜索邮件
```

## 三种协议对比

| 特性 | SMTP | POP3 | IMAP |
|---|---|---|---|
| 用途 | 发送邮件 | 下载邮件 | 同步邮件 |
| 方向 | 客户端 -> 服务器 | 服务器 -> 客户端 | 双向同步 |
| 多设备支持 | N/A | 差 | 好 |
| 按需获取 | N/A | 全部下载 | 支持 |
| 服务器操作 | N/A | 有限 | 完整支持 |
| 端口 | 25/587 | 110/995 | 143/993 |

## 实际应用

### Python 发送邮件

```python
import smtplib
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart

msg = MIMEMultipart()
msg['From'] = 'sender@example.com'
msg['To'] = 'recipient@example.org'
msg['Subject'] = 'Test Email with Attachment'

# 文本内容
body = MIMEText('This is the email body.', 'plain')
msg.attach(body)

# 附件
with open('file.pdf', 'rb') as f:
    attachment = MIMEText(f.read(), 'base64', 'utf-8')
    attachment['Content-Type'] = 'application/pdf'
    attachment['Content-Disposition'] = 'attachment; filename=file.pdf'
    msg.attach(attachment)

# 发送
with smtplib.SMTP('smtp.example.com', 587) as server:
    server.starttls()
    server.login('user@example.com', 'password')
    server.send_message(msg)
```

### Python 读取邮件（IMAP）

```python
import imaplib
import email
from email.header import decode_header

mail = imaplib.IMAP4_SSL('imap.example.com', 993)
mail.login('user@example.com', 'password')
mail.select('INBOX')

# 搜索邮件
status, messages = mail.search(None, 'UNSEEN')  # 未读邮件
status, messages = mail.search(None, 'FROM "boss@example.com"')

# 获取邮件
for num in messages[0].split():
    status, msg_data = mail.fetch(num, '(RFC822)')
    msg = email.message_from_bytes(msg_data[0][1])
    
    # 解码主题
    subject, encoding = decode_header(msg['Subject'])[0]
    if isinstance(subject, bytes):
        subject = subject.decode(encoding or 'utf-8')
    
    print(f'From: {msg["From"]}')
    print(f'Subject: {subject}')
```

## 安全考虑

### TLS/SSL 加密

所有现代邮件服务都要求加密：

```python
# SMTP with TLS
server = smtplib.SMTP('smtp.example.com', 587)
server.starttls()

# IMAP with SSL
mail = imaplib.IMAP4_SSL('imap.example.com', 993)
```

### 认证方式

| 方式 | 说明 |
|---|---|
| LOGIN | 明文用户名密码（不推荐） |
| PLAIN | Base64 编码的用户名密码 |
| CRAM-MD5 | 挑战-响应认证 |
| OAuth 2.0 | 第三方认证（推荐） |

## 面试追问

### 为什么 SMTP 使用 25 和 587 两个端口？

- **25 端口**：MTA 之间转发邮件（服务器到服务器）
- **587 端口**：MUA 提交邮件（客户端到服务器）

465 端口曾用于 SMTPS（SMTP over SSL），后来被废弃，现在有些服务重新启用。

### IMAP 的离线支持怎么实现？

IMAP 客户端通常会缓存邮件到本地，并记录服务器状态。重新上线时，客户端会与服务器同步状态（通过 IDLE 命令或轮询）。

### 什么是 SPF、DKIM、DMARC？

这些都是反垃圾邮件的技术：

- **SPF**：验证发件服务器 IP 是否被允许
- **DKIM**：验证邮件内容是否被篡改
- **DMARC**：结合 SPF 和 DKIM，指定处理策略

## 总结

邮件系统依赖三个协议：

```
SMTP（发送）：
- 客户端 -> 服务器
- 587 端口（提交）
- 25 端口（服务器间转发）

POP3（下载）：
- 服务器 -> 客户端
- 邮件下载后通常删除
- 适合单设备使用

IMAP（同步）：
- 服务器与客户端双向同步
- 适合多设备使用
- 支持服务器端邮件管理
```

理解这三个协议，你才能理解电子邮件系统的全貌。
