# HTTP 状态码

当你在浏览器中看到 404 Not Found 或 500 Internal Server Error 时，你已经在和 HTTP 状态码打交道了。

状态码是服务器对请求的"回应"，它告诉客户端请求的结果。理解状态码，是排查 Web 问题的基础。

## 状态码分类

HTTP 状态码是三位数字，分为五类：

| 类别 | 范围 | 含义 |
|---|---|---|
| 1xx | 100-199 | 信息性状态码，请求已接收，继续处理 |
| 2xx | 200-299 | 成功，请求已被服务器理解和接受 |
| 3xx | 300-399 | 重定向，需要进一步操作完成请求 |
| 4xx | 400-499 | 客户端错误，请求有语法错误或无法完成 |
| 5xx | 500-599 | 服务器错误，服务器处理请求时发生错误 |

## 2xx：成功

请求已成功被服务器接收、理解、并处理。

### 200 OK

最常见的状态码，表示请求成功：

```http
HTTP/1.1 200 OK
Content-Type: application/json

{"success":true,"data":[...]}
```

### 201 Created

表示请求已成功，并且服务器创建了新的资源。通常用于 POST 请求创建资源：

```http
HTTP/1.1 201 Created
Location: /users/123
Content-Type: application/json

{"id":123,"name":"张三"}
```

响应头中的 `Location` 指明新资源的 URI。

### 202 Accepted

表示请求已接收，但尚未完成处理。常用于异步操作：

```http
HTTP/1.1 202 Accepted
Content-Type: application/json

{"message":"任务已提交","taskId":"abc123"}
```

### 204 No Content

表示请求成功，但响应没有内容。用于 DELETE 操作或不返回实体的成功响应：

```http
HTTP/1.1 204 No Content
```

### 206 Partial Content

表示服务器成功返回了部分数据，用于断点续传或范围请求：

```http
HTTP/1.1 206 Partial Content
Content-Range: bytes 0-499/1000
Content-Length: 500

[前 500 字节数据]
```

## 3xx：重定向

客户端需要采取进一步操作才能完成请求。

### 301 Moved Permanently

永久重定向。请求的资源已永久移动到新位置：

```http
HTTP/1.1 301 Moved Permanently
Location: https://www.example.com/new-page
```

**用途**：
- 域名更换
- URL 规范化
- 资源永久迁移

浏览器会缓存 301 重定向，下次访问直接请求新地址。

### 302 Found（临时重定向）

临时重定向。请求的资源临时在其他位置：

```http
HTTP/1.1 302 Found
Location: /temporary-page
```

**注意**：302 可能会导致 POST 变成 GET（部分浏览器行为）。新的临时重定向状态码是 **303 See Other**（POST -> GET）和 **307 Temporary Redirect**（保持方法不变）。

### 303 See Other

强制 POST 转为 GET：

```http
HTTP/1.1 303 See Other
Location: /result-page
```

常用于 POST 表单提交后的跳转。

### 307 Temporary Redirect

保持 HTTP 方法不变：

```http
HTTP/1.1 307 Temporary Redirect
Location: /temp-page
```

POST 请求重定向后仍然是 POST。

### 304 Not Modified

表示资源未修改，可以直接使用缓存。这是 HTTP 缓存机制的关键状态码：

```http
HTTP/1.1 304 Not Modified
```

304 响应没有消息体，配合 `ETag` 或 `Last-Modified` 使用。

## 4xx：客户端错误

请求有语法错误或无法被服务器处理。

### 400 Bad Request

通用客户端错误，表示请求格式错误或参数无效：

```http
HTTP/1.1 400 Bad Request
Content-Type: application/json

{"error":"参数缺失：username"}
```

### 401 Unauthorized

请求需要认证。响应必须包含 `WWW-Authenticate` 头：

```http
HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm="api"
Content-Type: application/json

{"error":"未授权访问"}
```

**区分 401 和 403**：
- 401：未认证，需要登录
- 403：已认证但无权限

### 403 Forbidden

服务器拒绝请求，通常是因为权限不足：

```http
HTTP/1.1 403 Forbidden
Content-Type: application/json

{"error":"没有权限访问此资源"}
```

### 404 Not Found

资源不存在：

```http
HTTP/1.1 404 Not Found
Content-Type: text/html

<h1>404 - Page Not Found</h1>
```

### 405 Method Not Allowed

请求方法不支持：

```http
HTTP/1.1 405 Method Not Allowed
Allow: GET, POST, PUT
```

### 409 Conflict

请求与服务器状态冲突：

```http
HTTP/1.1 409 Conflict
Content-Type: application/json

{"error":"用户名已存在"}
```

### 422 Unprocessable Entity

请求格式正确，但语义错误：

```http
HTTP/1.1 422 Unprocessable Entity
Content-Type: application/json

{"error":"邮箱格式不正确"}
```

### 429 Too Many Requests

请求频率超限：

```http
HTTP/1.1 429 Too Many Requests
Retry-After: 3600
Content-Type: application/json

{"error":"请求过于频繁，请1小时后再试"}
```

## 5xx：服务器错误

服务器在处理请求时发生错误。

### 500 Internal Server Error

通用服务器错误：

```http
HTTP/1.1 500 Internal Server Error
Content-Type: application/json

{"error":"服务器内部错误"}
```

### 502 Bad Gateway

网关或代理服务器收到无效响应：

```http
HTTP/1.1 502 Bad Gateway

Bad Gateway
```

常见原因：Nginx 代理的后端服务异常。

### 503 Service Unavailable

服务暂时不可用：

```http
HTTP/1.1 503 Service Unavailable
Retry-After: 300
Content-Type: text/plain

Service temporarily unavailable
```

### 504 Gateway Timeout

网关或代理服务器超时：

```http
HTTP/1.1 504 Gateway Timeout
```

## 常见问题排查

### 301 vs 302 vs 307

| 状态码 | 方法变化 | 缓存 |
|---|---|---|
| 301 | 可能变成 GET | 会被缓存 |
| 302 | 可能变成 GET | 可能会被缓存 |
| 303 | 强制 GET | 不缓存 |
| 307 | 保持不变 | 不会被缓存 |

### 401 vs 403

- 401：未认证，需要提供凭证
- 403：已认证但权限不足

### 502 vs 504

- 502：上游服务器返回了错误响应
- 504：上游服务器超时未响应

## 面试追问

### 为什么 301/302 重定向有时会导致 POST 变成 GET？

这是 HTTP/1.0 的历史遗留问题。HTTP/1.1 通过 303 和 307 修正了这个问题：
- 303：明确要求转为 GET
- 307：明确要求保持方法不变

### 304 响应有没有消息体？

没有。304 响应只包含头部，不包含消息体。客户端使用缓存的版本。

### HTTP 状态码是否可以自定义？

可以，WebSocket 的 101 Switching Protocols 就是非标准状态码。但自定义状态码应该遵循类别语义（1xx 信息、2xx 成功等）。

### 504 和 503 有什么区别？

503 是服务暂时不可用（可能过一会儿就好了），通常会返回 `Retry-After` 提示重试时间。504 是网关超时，说明上游服务响应太慢。

## 总结

HTTP 状态码是服务器与客户端沟通的"语言"：

| 类别 | 含义 | 示例 |
|---|---|---|
| 1xx | 信息 | 101 Switching Protocols |
| 2xx | 成功 | 200 OK, 201 Created |
| 3xx | 重定向 | 301, 302, 304 |
| 4xx | 客户端错误 | 400, 401, 403, 404 |
| 5xx | 服务器错误 | 500, 502, 503, 504 |

记住几个关键状态码，排查问题时会事半功倍。
