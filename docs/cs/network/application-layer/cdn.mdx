# CDN 原理与缓存策略

当你在北京打开一个来自美国服务器的网页时，如果页面加载只需要几百毫秒，你很可能已经在使用 CDN（Content Delivery Network，内容分发网络）。

CDN 是互联网的"物流网络"——把内容放到离用户更近的地方。

## 为什么需要 CDN

### 延迟问题

物理距离影响网络延迟：

```
从北京访问：
- 北京服务器：10ms
- 上海服务器：30ms
- 美国服务器：200ms
```

即使服务器处理速度一样，用户体验差异巨大。

### 服务器压力

大量用户同时访问时，单一服务器可能崩溃：

```
单一服务器：
用户 A -> 服务器（可能崩溃）
用户 B -> 服务器
用户 C -> 服务器
...

CDN：
用户 A -> CDN 边缘节点 A
用户 B -> CDN 边缘节点 B
用户 C -> CDN 边缘节点 C
服务器 <- CDN 源站（稳定的回源请求）
```

## CDN 的工作原理

### 基本架构

```
用户浏览器
    |
    | 1. DNS 解析
    v
CDN 全局负载均衡（GSLB）
    |
    | 2. 返回最近边缘节点 IP
    v
CDN 边缘节点（Edge Server）
    |
    | 3. 缓存命中？
    |    是 -> 返回缓存内容
    |    否 -> 4. 回源获取
    v
CDN 源站（Origin Server）
```

### 智能 DNS 解析

CDN 的 DNS 服务器会根据用户位置返回最优的边缘节点 IP：

```bash
# 用户在北京
dig cdn.example.com

# 返回：
cdn.example.com.    300    IN    A    220.181.12.1    # 北京节点

# 用户在美国
dig cdn.example.com

# 返回：
cdn.example.com.    300    IN    A    104.21.56.78    # 美国节点
```

### 边缘节点缓存

CDN 边缘节点缓存静态资源：

```
缓存命中流程：
1. 用户请求 www.example.com/static/app.js
2. CDN 边缘节点检查缓存
3. 缓存命中 -> 直接返回
4. 缓存未命中 -> 回源获取 -> 缓存 -> 返回
```

## CDN 核心概念

### 缓存命中率（Hit Rate）

```
缓存命中率 = 缓存命中次数 / 总请求次数

高命中率的好处：
- 更快响应（边缘节点直接返回）
- 更低源站压力
- 更低带宽成本
```

### 回源（Origin Pull / Pull Zone）

边缘节点在缓存未命中时，向源站请求内容：

```
回源流程：
1. 边缘节点：缓存中没有 app.js
2. 边缘节点 -> 源站：GET /static/app.js
3. 源站 -> 边缘节点：返回 app.js（带 Cache-Control）
4. 边缘节点：缓存 app.js
5. 边缘节点 -> 用户：返回 app.js
```

### 推送（Origin Push / Push Zone）

源站主动推送内容到 CDN：

```
推送流程：
1. 源站主动将内容推送到 CDN
2. CDN 存储并分发
3. 用户请求时直接命中
```

## 缓存策略

### Cache-Control 头部

源站通过 `Cache-Control` 控制 CDN 缓存行为：

```http
Cache-Control: max-age=3600
```

| 指令 | 说明 |
|---|---|
| `max-age=秒数` | 缓存有效期 |
| `s-maxage=秒数` | CDN 缓存有效期（忽略 max-age） |
| `no-cache` | 每次使用前必须验证 |
| `no-store` | 不缓存 |
| `public` | 可被任何缓存存储 |
| `private` | 只有浏览器可缓存 |

### 缓存示例

```bash
# 静态资源：缓存 1 年
Cache-Control: public, max-age=31536000, immutable

# API 响应：不缓存
Cache-Control: no-cache, no-store, must-revalidate

# 用户相关内容：私有缓存
Cache-Control: private, max-age=600
```

### CDN 缓存进阶策略

```http
# 启用 Brotli 压缩
Accept-Encoding: gzip, deflate, br

# 版本化 URL（用于缓存更新）
/static/app.js?v=1.2.3
/static/app.1.2.3.js

# 不带参数的请求默认缓存
# 带参数的请求通常不缓存（除非配置）
```

## CDN 缓存清除

### 缓存失效（Purge）

当源站更新内容后，需要清除 CDN 缓存：

```bash
# Cloudflare API 清除缓存
curl -X POST "https://api.cloudflare.com/client/v4/zones/{zone_id}/purge_cache" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  --data '{"files":["https://example.com/app.js"]}'

# 清除所有缓存
curl -X POST "https://api.cloudflare.com/client/v4/zones/{zone_id}/purge_cache" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  --data '{"purge_everything":true}'
```

### 缓存预热（Preload）

提前将热门资源加载到 CDN：

```bash
# CDN 服务通常提供预热 API
# 或使用爬虫工具
wget -r -np -l 1 https://example.com/
```

## CDN 与 HTTPS

### SSL 证书配置

CDN 通常支持三种证书模式：

| 模式 | 说明 |
|---|---|
| 源站 HTTP | CDN 提供证书，CDN 与源站用 HTTP |
| 源站 HTTPS | CDN 提供证书，CDN 与源站用 HTTPS |
| 源站 HTTPS + 客户端证书 | 双向认证（高安全场景） |

### 免费证书

主流 CDN 都提供免费 SSL 证书：

```
Cloudflare：自动提供
Let's Encrypt：通过 API 自动申请
阿里云 CDN：免费 DV 证书
AWS CloudFront：免费证书
```

## CDN 性能优化

### 边缘计算

现代 CDN 支持边缘计算（Edge Computing）：

```javascript
// Cloudflare Workers 示例
addEventListener('fetch', event => {
  event.respondWith(handleRequest(event.request))
})

async function handleRequest(request) {
  const cache = caches.default
  let response = await cache.match(request)
  
  if (!response) {
    response = await fetch(request)
    response = new Response(response.body, response)
    response.headers.set('Cache-Control', 'max-age=3600')
    event.waitUntil(cache.put(request, response.clone()))
  }
  
  return response
}
```

### Brotli 压缩

比 Gzip 压缩率更高的压缩算法：

```
压缩效果对比：
原始大小：100KB
Gzip 压缩：35KB
Brotli 压缩：30KB（节省 14%）
```

## 实际应用

### Nginx 配置源站缓存头

```nginx
server {
    location /static/ {
        # 静态资源缓存 1 年
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    location /api/ {
        # API 不缓存
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }
}
```

### Vary 头处理

```http
# 根据 Accept-Encoding 缓存不同版本
Vary: Accept-Encoding

# 根据 User-Agent 缓存不同版本（移动端/桌面端）
Vary: User-Agent
```

## 常见问题

### 缓存不一致

源站更新后，用户可能仍访问到旧内容。

解决方案：
- 使用版本化 URL
- 及时清除缓存
- 设置合理的 TTL

### 源站 IP 泄露

如果攻击者获取了源站真实 IP，可以通过绕过 CDN 直接攻击。

解决方案：
- 源站只允许 CDN IP 访问（防火墙配置）
- 使用云防火墙服务

### CDN 回源率过高

回源率高说明缓存命中率低。

原因：
- TTL 设置太短
- 动态内容过多
- 缓存键配置不当

## 面试追问

### CDN 和 DNS 负载均衡有什么区别？

DNS 负载均衡只是"把请求分发到不同 IP"，没有缓存能力。CDN 在 DNS 基础上增加了缓存、压缩、安全防护等功能。

### 如何判断 CDN 是否生效？

```bash
# 查看响应头中的 CDN 标识
curl -I https://example.com

# Cloudflare
cf-ray: xxxxxxx
server: cloudflare

# 阿里云 CDN
server: AlibabaCloud
x-cache: MISS TCP_MISS from cdc-xxxx
```

### CDN 能缓存 POST 请求吗？

通常 CDN 不缓存 POST 请求（需要发送 body，缓存 key 复杂）。但可以在 CDN 配置中开启 POST 缓存（Cloudflare 支持）。

### 什么是 Tiered Architecture？

多级 CDN 架构：
- 边缘节点（Edge）：最接近用户
- 中间节点（Mid-tier/POP）：区域缓存
- 源站（Origin）

## 总结

CDN 是互联网内容分发的基础设施：

```
CDN 核心功能：
- 就近访问：降低延迟
- 缓存静态资源：减少源站压力
- 负载均衡：提升可用性
- 安全防护：抵御 DDoS

缓存策略要点：
- 静态资源：长缓存 + 版本化 URL
- 动态内容：不缓存或短缓存
- 合理设置 TTL 和 Cache-Control
```

理解 CDN，你才能设计出高性能、高可用的 Web 应用。
