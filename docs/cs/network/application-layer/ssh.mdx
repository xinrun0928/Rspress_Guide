# SSH 远程登录

当你需要登录远程服务器时，你会怎么做？

早期的 `telnet` 是明文传输密码，任何人都能截获。**SSH（Secure Shell）**就是为了解决这个问题而诞生的——它提供了加密的远程登录方式。

## SSH 是什么

SSH（Secure Shell）是一种加密的网络协议，用于：
- 远程登录服务器
- 远程执行命令
- 文件传输（SFTP、SCP）
- 端口转发
- 隧道代理

### SSH vs Telnet

```
Telnet：明文传输
用户 ---> 密码明文 ---> 服务器
          ↓
      任何人都能截获

SSH：加密传输
用户 ---> 加密数据 ---> 服务器
              ↓
          只有服务器能解密
```

## SSH 连接原理

### 密钥交换

SSH 使用多种加密算法保证安全：

```
SSH 连接建立过程：

1. TCP 连接建立（22 端口）
2. 版本协商
3. 密钥交换（Diffie-Hellman 或 ECDHE）
4. 服务器认证（验证服务器身份）
5. 客户端认证（密码或密钥）
6. 加密会话建立
```

### 密钥交换算法

现代 SSH 使用 Diffie-Hellman（DH）或 Elliptic Curve DH（ECDH）：

```
DH 密钥交换：

客户端                          服务器
  |                               |
  |------ Client Hello ----------->|  支持的算法列表
  |<------ Server Hello ----------|  选择的算法
  |                               |
  |------ DH 公钥（g^a mod p）---->|
  |<------ DH 公钥（g^b mod p）----|  服务器公钥
  |                               |
  |  计算共享密钥                  |
  |  K = g^(ab) mod p             |
  |                               |
  |  双方使用 K 派生出会话密钥      |
```

### 认证方式

#### 密码认证

```
客户端                          服务器
  |                               |
  |------ 用户名 ----------------->|
  |<------ 密码提示 --------------|
  |------ 加密的密码 ------------->|
  |<------ 认证结果 --------------|
```

#### 公钥认证

更安全、更方便的方式：

```
公钥认证流程：

1. 客户端生成密钥对（公钥 + 私钥）
2. 公钥添加到服务器 ~/.ssh/authorized_keys
3. 登录时，客户端用私钥签名
4. 服务器用公钥验证签名
5. 验证通过，无需密码
```

```bash
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your_email@example.com"

# 或 RSA（兼容性更好）
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 公钥内容
cat ~/.ssh/id_ed25519.pub
# ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... your_email@example.com

# 添加到服务器
ssh-copy-id user@server.com
# 或手动添加
cat ~/.ssh/id_ed25519.pub | ssh user@server.com "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

## SSH 常用命令

### 基本连接

```bash
# 基本连接
ssh user@hostname

# 指定端口
ssh -p 2222 user@hostname

# 指定密钥文件
ssh -i ~/.ssh/my_key user@hostname

# 跃屏连接（通过跳板机）
ssh -J jump_host user@target_host
```

### 执行远程命令

```bash
# 执行单条命令
ssh user@hostname "ls -la /home"

# 执行本地脚本
ssh user@hostname < local_script.sh

# 多命令
ssh user@hostname "cd /var/log && tail -f syslog"
```

### 文件传输

```bash
# SCP 下载
scp user@hostname:/remote/file.txt /local/

# SCP 上传
scp /local/file.txt user@hostname:/remote/

# 递归复制目录
scp -r user@hostname:/remote/dir /local/

# SFTP 交互式传输
sftp user@hostname
sftp> get remote.txt
sftp> put local.txt
sftp> bye
```

### 端口转发

#### 本地端口转发

将远程端口映射到本地：

```bash
# 将远程 3306 映射到本地 3333
ssh -L 3333:localhost:3306 user@hostname
```

```
本地端口转发：
本地 3333 -> SSH 隧道 -> 服务器 -> localhost 3306
```

#### 远程端口转发

将本地端口暴露到远程：

```bash
# 将本地 8080 暴露到远程 80
ssh -R 80:localhost:8080 user@hostname
```

```
远程端口转发：
远程 80 -> SSH 隧道 -> 本地 8080
```

#### 动态端口转发（SOCKS 代理）

```bash
# 创建 SOCKS 代理
ssh -D 1080 user@hostname

# 配置浏览器使用代理
# SOCKS Host: localhost:1080
```

### SSH Config

配置文件简化连接：

```bash
# ~/.ssh/config
Host myserver
    HostName server.example.com
    User admin
    Port 2222
    IdentityFile ~/.ssh/my_key
    ForwardAgent yes

Host github
    HostName github.com
    User git
    IdentityFile ~/.ssh/github_key

Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

```bash
# 使用别名连接
ssh myserver
```

## SSH 安全实践

### 基本安全

```bash
# 禁用密码登录
# 编辑 /etc/ssh/sshd_config
PasswordAuthentication no
PubkeyAuthentication yes

# 禁用 root 登录
PermitRootLogin no

# 修改默认端口
Port 2222

# 限制用户登录
AllowUsers admin deploy

# 限制 IP 访问
AllowUsers admin@192.168.1.0/24
```

### SSH Agent

管理私钥，避免重复输入密码：

```bash
# 启动 SSH Agent
eval "$(ssh-agent -s)"

# 添加密钥
ssh-add ~/.ssh/id_ed25519

# 查看已添加的密钥
ssh-add -l

# 关机前关闭 Agent
ssh-agent -k
```

### 双因素认证

```bash
# 安装 Google Authenticator
sudo apt install libpam-google-authenticator

# 配置 PAM
# 编辑 /etc/pam.d/sshd
auth required pam_google_authenticator.so

# 编辑 /etc/ssh/sshd_config
ChallengeResponseAuthentication yes
AuthenticationMethods keyboard-interactive
```

### SSH 证书认证

大规模部署时，使用证书更方便：

```bash
# 1. 创建 CA 密钥
ssh-keygen -t ed25519 -f ca_key -C "CA"

# 2. 签署用户密钥
ssh-keygen -s ca_key -I "user@example.com" -n user1,user2 -V +1d user_key.pub

# 3. 配置服务器信任 CA
# 编辑 /etc/ssh/sshd_config
TrustedUserCAKeys /etc/ssh/trusted-user-ca-keys.pem

# 4. 分发 CA 公钥到服务器
sudo cp ca_key.pub /etc/ssh/trusted-user-ca-keys.pem
```

## SSH 高级用法

### SSH 隧道

#### 本地 Web 应用暴露到公网

```bash
# 将本地 3000 端口通过跳板机暴露
ssh -R 80:localhost:3000 user@bastion.com

# 需要服务器配置 GatewayPorts yes
```

#### 远程访问内网数据库

```bash
# 通过跳板机访问内网数据库
ssh -L 5432:192.168.1.100:5432 user@bastion.com
# 本地 5432 -> 跳板机 -> 内网 192.168.1.100:5432
```

### SSH + Git

```bash
# 配置 Git SSH
git config --global user.email "you@example.com"
git config --global user.name "Your Name"

# 使用特定密钥
# 编辑 ~/.ssh/config
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/github_key

# 测试连接
ssh -T git@github.com
```

### SSH ProxyJump

多级跳板机：

```bash
# 单级跳板
ssh -J jump1 target

# 多级跳板
ssh -J jump1,jump2 target

# 或在 config 中配置
Host target
    HostName target.internal
    ProxyJump jump1,jump2
```

### SSHFS

挂载远程文件系统：

```bash
# 安装
sudo apt install sshfs

# 挂载远程目录
mkdir ~/remote
sshfs user@hostname:/home/user ~/remote

# 卸载
fusermount -u ~/remote
```

## 面试追问

### SSH 密钥类型怎么选？

| 类型 | 安全性 | 兼容性 | 性能 | 推荐场景 |
|---|---|---|---|---|
| RSA 4096 | 高 | 最好 | 较慢 | 兼容旧系统 |
| Ed25519 | 很高 | 好 | 快 | 现代系统 |
| ECDSA | 高 | 一般 | 快 | 中等兼容性 |

### SSH 连接慢的原因有哪些？

- DNS 反向解析：禁用 `UseDNS yes`
- GSSAPI 认证：禁用 `GSSAPIAuthentication no`
- 密钥权限问题：确保 `~/.ssh` 权限 700，密钥 600
- 网络延迟

### SSH 如何实现会话共享？

```bash
# 启用 SSH Agent 转发
ssh -A user@hostname

# 配置 ControlMaster
Host *
    ControlMaster auto
    ControlPath ~/.ssh/sockets/%r@%h-%p
    ControlPersist 600
```

### SSH 密钥丢了怎么办？

- 使用密码保护私钥
- 定期备份私钥和 authorized_keys
- 使用硬件密钥（如 YubiKey）

## 总结

SSH 是远程管理的基石：

```
SSH 核心概念：
- 加密传输：保护数据安全
- 公钥认证：无密码登录
- 端口转发：安全访问内网服务
- SSH Agent：简化密钥管理

安全实践：
- 使用密钥认证
- 禁用密码登录
- 修改默认端口
- 限制登录用户和 IP
- 定期更换密钥
```

掌握 SSH，你才能安全地管理服务器。
