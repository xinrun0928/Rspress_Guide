# FTP 协议

FTP（File Transfer Protocol）是早期互联网最重要的协议之一——它专门用于在两台计算机之间传输文件。

虽然今天有 HTTP、云存储等方式共享文件，但 FTP 仍然是服务器管理、批量传输等场景的重要工具。

## FTP 的设计

### 协议特点

```
FTP vs HTTP：

FTP：
- 专门为文件传输设计
- 有状态协议（维护连接状态）
- 两个通道（命令通道 + 数据通道）
- 支持文件操作（列表、删除、重命名等）

HTTP：
- 万用的 Web 协议
- 无状态
- 单通道
- 主要用于下载
```

### 双通道架构

FTP 使用两个 TCP 连接：

```
命令通道（Control Connection）：
- 端口 21（控制端口）
- 传输命令和响应
- 保持打开状态

数据通道（Data Connection）：
- 传输文件列表和文件内容
- 按需建立和关闭
```

## FTP 连接模式

### 主动模式（PORT）

客户端开放一个随机端口，告知服务器：

```
主动模式流程：

客户端（随机端口）                    服务器（21）
    |                                   |
    |<---------- 欢迎信息 -------------|
    |                                   |
    |-------- USER username ----------->|
    |<-------- 331 OK -----------------|
    |-------- PASS password ----------->|
    |<-------- 230 OK -----------------|
    |                                   |
    |-------- PORT 192,168,1,100,4,15 ->|  客户端开放端口
    |<-------- 200 PORT command ok -----|
    |                                   |
    |-------- LIST -------------------->|
    |                                   |
    |                   服务器（20）    客户端（端口 4*256+15）
    |                        |<-------- 数据连接（服务器主动连接）--|
    |                        |                                    |
    |---------- 文件列表 ---------------->|                       |
    |                                   |                            |
```

PORT 命令格式：`PORT h1,h2,h3,h4,p1,p2`
- h1-h4：客户端 IP
- p1*256 + p2：端口号

### 被动模式（PASV）

服务器开放一个随机端口，客户端连接：

```
被动模式流程：

客户端（随机端口）                    服务器（21）
    |                                   |
    |<---------- 欢迎信息 -------------|
    |                                   |
    |-------- USER username ----------->|
    |<-------- 331 OK -----------------|
    |-------- PASS password ----------->|
    |<-------- 230 OK -----------------|
    |                                   |
    |-------- PASV -------------------->|
    |<-------- 227 Entering PASV mode.  |  服务器开放端口
    |           (192,168,1,200,4,16)    |  192.168.1.200:1040
    |                                   |
    |                                   |  服务器（1040）
    |客户端随机端口 ---- 数据连接 ------>|-----------------------|
    |                                   |                        |
    |-------- LIST -------------------->|
    |---------- 文件列表 ---------------->|
    |                                   |                        |
```

### 防火墙问题

```
主动模式的问题：
- 服务器主动连接客户端
- 客户端防火墙可能阻止

被动模式的问题：
- 客户端连接服务器随机端口
- 服务器防火墙需要开放大量端口

解决方案：
- 使用被动模式
- 服务器配置 PASV 地址范围
- 防火墙放行指定端口
```

## FTP 命令

### 连接相关命令

| 命令 | 说明 |
|---|---|
| `USER username` | 用户名 |
| `PASS password` | 密码 |
| `QUIT` | 退出 |
| `ACCT` | 账户信息 |

### 目录操作命令

| 命令 | 说明 |
|---|---|
| `PWD` | 显示当前目录 |
| `CWD directory` | 切换目录 |
| `CDUP` | 切换到上级目录 |
| `LIST` | 列出文件 |
| `NLST` | 列出文件名 |
| `MKD directory` | 创建目录 |
| `RMD directory` | 删除目录 |

### 文件操作命令

| 命令 | 说明 |
|---|---|
| `RETR filename` | 下载文件 |
| `STOR filename` | 上传文件 |
| `DELE filename` | 删除文件 |
| `RNFR filename` | 重命名（源） |
| `RNTO filename` | 重命名（目标） |
| `SIZE filename` | 获取文件大小 |

### 其他命令

| 命令 | 说明 |
|---|---|
| `TYPE [A/I]` | 设置传输模式（ASCII/Binary） |

## FTP 响应码

FTP 使用三位数字响应码：

| 首位 | 含义 |
|---|---|
| 1xx | 信息性响应（请求已接受，继续） |
| 2xx | 成功响应 |
| 3xx | 命令正确，需要更多信息 |
| 4xx | 暂时失败 |
| 5xx | 永久失败 |

### 常用响应码

| 响应码 | 说明 |
|---|---|
| `125` | 数据连接已打开，传输开始 |
| `150` | 文件状态正确，准备打开数据连接 |
| `200` | 命令成功 |
| `220` | 服务就绪 |
| `226` | 传输完成 |
| `227` | 进入被动模式 |
| `230` | 用户登录成功 |
| `250` | 文件操作完成 |
| `331` | 需要密码 |
| `425` | 无法打开数据连接 |
| `500` | 语法错误 |
| `530` | 未登录 |
| `550` | 文件不可用 |

## 传输模式

### ASCII 模式

用于传输文本文件，自动转换行结束符：

```
Windows: CRLF (\r\n)
Unix: LF (\n)
Mac: CR (\r)

ASCII 模式自动转换：
发送端 Unix -> 接收端 Windows：自动加 \r
发送端 Windows -> 接收端 Unix：自动去 \r
```

### 二进制模式

用于传输二进制文件（图片、视频、压缩包等）：

```
二进制模式：
- 不做任何转换
- 逐字节传输
- 所有字节保持原样
```

### 模式选择

```bash
# FTP 命令行中选择模式
ftp> binary    # 切换到二进制模式
ftp> ascii      # 切换到 ASCII 模式

# 自动检测（推荐）
# 大多数 FTP 客户端会自动根据文件类型选择模式
```

## FTPS 与 SFTP

### FTPS（FTP over TLS/SSL）

在 FTP 基础上添加 TLS/SSL 加密：

```
FTPS 两种模式：
- 显式 FTPS（FTPES）：21 端口，命令通道加密
- 隐式 FTPS：990 端口，全连接加密
```

### SFTP（SSH File Transfer Protocol）

与 FTP 完全不同，基于 SSH 协议：

```
SFTP vs FTP：

FTP：
- 明文传输（FTPS 除外）
- 端口 21（控制）
- 两个连接

SFTP：
- 加密传输
- 端口 22（SSH）
- 单个连接
- 实际上是一个 SSH 子系统
```

| 特性 | FTP | FTPS | SFTP |
|---|---|---|---|
| 安全性 | 明文 | 加密 | 加密 |
| 端口 | 21 | 21/990 | 22 |
| 防火墙 | 复杂 | 复杂 | 简单 |
| 证书 | N/A | 需要 | SSH 密钥 |
| 速度 | 快 | 略慢（加密开销）| 略慢 |

## 实际应用

### 命令行 FTP

```bash
# 连接 FTP 服务器
ftp ftp.example.com

# 登录
ftp> user username

# 切换模式
ftp> binary

# 下载文件
ftp> get remote.txt local.txt

# 上传文件
ftp> put local.txt remote.txt

# 下载多个文件
ftp> mget *.txt

# 上传多个文件
ftp> mput *.jpg

# 目录操作
ftp> ls
ftp> cd /public
ftp> mkdir backup
ftp> rmdir backup

# 删除文件
ftp> delete remote.txt

# 重命名
ftp> rename oldname.txt newname.txt

# 查看当前目录
ftp> pwd

# 退出
ftp> bye
```

### Python ftplib

```python
from ftplib import FTP

# 连接 FTP
ftp = FTP('ftp.example.com')
ftp.login('username', 'password')

# 切换到二进制模式
ftp.voidcmd('TYPE I')

# 下载文件
with open('local.txt', 'wb') as f:
    ftp.retrbinary('RETR remote.txt', f.write)

# 上传文件
with open('local.txt', 'rb') as f:
    ftp.storbinary('STOR remote.txt', f)

# 列出文件
ftp.retrlines('LIST')

# 创建目录
ftp.mkd('new_directory')

# 删除文件
ftp.delete('remote.txt')

# 退出
ftp.quit()
```

### lftp（增强型 FTP 客户端）

```bash
# lftp 支持多种协议和高级功能
lftp ftp://user:pass@ftp.example.com

# 镜像下载
lftp -e "mirror /remote/dir /local/dir; bye"

# 镜像上传
lftp -e "mirror -R /local/dir /remote/dir; bye"

# 断点续传
lftp -c "pget -c large-file.iso"
```

## 面试追问

### FTP 为什么需要两个连接？

FTP 设计于 1970 年代，当时的计算机和网络设备处理能力有限。双连接设计便于：
- 命令和数据分开处理
- 数据传输中断不会影响命令通道
- 实现复杂的目录操作

### 主动模式和被动模式哪个更好？

现代网络环境中，被动模式更常用：
- 客户端通常在防火墙后面
- 服务器更容易配置防火墙
- 云计算环境更适合被动模式

### SFTP 和 FTPS 哪个更安全？

两者都可以提供安全的文件传输：
- FTPS 使用 TLS/SSL，证书管理复杂
- SFTP 使用 SSH，密钥管理更简单
- SFTP 通常被认为更易部署

## 总结

FTP 是文件传输的经典协议：

```
FTP 核心概念：
- 双通道架构：命令通道 + 数据通道
- 主动模式：服务器主动连接客户端
- 被动模式：客户端连接服务器
- 传输模式：ASCII（二进制文件）vs Binary（文本）

FTP vs FTPS vs SFTP：
- FTP：明文传输
- FTPS：FTP + TLS/SSL
- SFTP：基于 SSH
```

虽然 HTTP 和云存储越来越流行，但 FTP 在服务器管理、批量传输等场景仍有广泛应用。
