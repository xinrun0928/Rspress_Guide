## 并发的三大特性

### 可见性：一个线程的修改，另一个线程看不到

这是并发编程中最容易出错的地方。

```java
public class VisibilityProblem {
    private static boolean flag = true;  // 共享变量

    public static void main(String[] args) throws InterruptedException {
        // 线程 A：不断循环等待 flag 变为 false
        new Thread(() -> {
            while (flag) {
                // 问题：这里可能永远看不到 flag 的变化
                // 因为 CPU 缓存没有及时刷新到主内存
            }
            System.out.println("线程 A 退出循环");
        }).start();

        // 线程 B：1 秒后修改 flag
        Thread.sleep(1000);
        flag = false;
        System.out.println("线程 B 修改了 flag");
    }
}
```

**为什么会有可见性问题？**

CPU 为了提升性能，引入了多级缓存：

```
CPU Core 1          CPU Core 2
    │                    │
 L1 Cache          L1 Cache
    │                    │
 L2 Cache          L2 Cache
    │                    │
    └────────┬───────────┘
             │
         L3 Cache
             │
         主内存
```

每个 CPU 核心都有自己的缓存，数据可能不一致。

### 原子性：操作不能被分割

```java
// 不是原子操作！
i++;  // 等价于：
// 1. 读取 i 的值到寄存器
// 2. 将 i+1
// 3. 将结果写回内存
// 中间可能被其他线程打断
```

**解决方案：**

```java
// 方案一：synchronized
synchronized (lock) {
    i++;
}

// 方案二：AtomicInteger
private AtomicInteger i = new AtomicInteger(0);
i.incrementAndGet();
```

### 有序性：代码的执行顺序可能被重排

```java
// Java 编译器/CPU 可能重排指令
int a = 1;      // 语句 1
int b = 2;      // 语句 2
a = a + 1;      // 语句 3
b = b * 2;      // 语句 4

// 重排后可能变成：
int b = 2;      // 语句 2 先执行
int a = 1;      // 语句 1 后执行
b = b * 2;      // 语句 4
a = a + 1;      // 语句 3
// 结果不变，但执行顺序变了
```

**为什么允许重排？**

为了提升性能。只要保证**单线程结果一致**，就可以重排。

---

## 核心公式：并发问题的解决之道

```
并发问题 = 可见性 + 原子性 + 有序性

解决思路 = 禁止重排 + 保证原子 + 确保可见

具体手段：
├─ 禁止重排：Happens-Before 规则、内存屏障
├─ 保证原子：synchronized、Lock、原子类
└─ 确保可见：volatile、synchronized、final
```

## 延伸思考

并发编程有三个层次：

**第一层：会用**
知道 `synchronized` 可以加锁，`volatile` 可以保证可见性，但不知道为什么。

**第二层：懂原理**
理解 JMM 的三大特性，知道内存屏障的作用，理解锁升级的过程。

**第三层：能设计**
能够设计出线程安全的并发组件，能够排查死锁问题，能够根据场景选择合适的并发工具。
