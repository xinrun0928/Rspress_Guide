# Sentinel

当系统流量突然激增，或者下游服务响应变慢，如果没有限流和熔断机制，系统会像多米诺骨牌一样逐渐崩溃——一个服务的故障拖垮整个系统。

Sentinel 就是解决这个问题的流量控制组件。

## 问题的本质

```
双十一零点，用户请求暴增 100 倍：

没有限流：
用户请求 → 应用服务器 → 数据库 → 崩溃
              → Redis → 崩溃
              → 下游服务 → 崩溃

有 Sentinel：
用户请求 → Sentinel 检查 → [限流] → 拒绝部分请求
              → [熔断] → 返回降级响应
              → [通过] → 正常处理
```

## Sentinel 是什么？

Sentinel 是阿里巴巴开源的流量控制、熔断降级组件，具有以下特点：

- **流量控制**：防止瞬时流量冲垮系统
- **熔断降级**：下游故障时快速失败
- **系统自适应**：根据系统负载自动限流
- **实时监控**：实时查看流量、响应时间等指标

## 模块学习路径

```
1. Sentinel 核心概念：资源、规则、插槽链
2. 流量控制：QPS 限流、并发线程数限流
3. 熔断降级：RT、异常比例、异常数熔断
4. 系统自适应限流：根据系统负载自动调整
5. 注解支持：@SentinelResource 与 Dubbo 集成
6. Sentinel vs Hystrix：选型对比
```

> [!TIP]
> Sentinel 的核心思想是「在流量达到阈值时保护系统，而不是让系统崩溃后再救火」。
