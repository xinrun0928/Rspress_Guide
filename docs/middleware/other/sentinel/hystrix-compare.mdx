# Sentinel vs Hystrix 对比

Sentinel 和 Hystrix 都是流量控制和熔断降级的组件。Hystrix 已经停止维护，Sentinel 是其优秀的替代方案。

## 背景

### Hystrix 简介

Hystrix 是 Netflix 开源的熔断器组件，2018 年进入维护模式，2020 年正式停止维护。

### Sentinel 简介

Sentinel 是阿里巴巴开源的流量控制组件，提供限流、熔断、降级、系统保护等能力。

## 核心功能对比

| 功能 | Sentinel | Hystrix |
|---|---|---|
| 限流 | 支持 | 不支持 |
| 熔断降级 | 支持 | 支持 |
| 系统保护 | 支持 | 不支持 |
| 热点参数限流 | 支持 | 不支持 |
| 流量整形 | 支持 | 简单 |
| 实时监控 | 支持 | 支持 |

## 熔断机制对比

### Hystrix 熔断

```java
@HystrixCommand(
    commandProperties = {
        @HystrixProperty(name = "circuitBreaker.requestVolumeThreshold", value = "20"),
        @HystrixProperty(name = "circuitBreaker.sleepWindowInMilliseconds", value = "10000"),
        @HystrixProperty(name = "circuitBreaker.errorThresholdPercentage", value = "50")
    },
    fallbackMethod = "fallback"
)
public String callService() {
    return restTemplate.getForObject("http://service/", String.class);
}

public String fallback() {
    return "降级响应";
}
```

### Sentinel 熔断

```java
@SentinelResource(value = "callService",
    blockHandler = "blockHandler",
    fallback = "fallback")
public String callService() {
    return restTemplate.getForObject("http://service/", String.class);
}

public String blockHandler(BlockException e) {
    return "限流/熔断";
}

public String fallback(Throwable t) {
    return "降级响应";
}
```

## 资源隔离对比

### Hystrix 隔离

Hystrix 支持线程池隔离和信号量隔离：

```java
// 线程池隔离
@HystrixCommand(
    groupKey = "orderGroup",
    threadPoolKey = "orderThreadPool",
    threadPoolProperties = {
        @HystrixProperty(name = "coreSize", value = "10"),
        @HystrixProperty(name = "maxQueueSize", value = "100")
    }
)
public String callOrder() { ... }

// 信号量隔离
@HystrixCommand(
    commandProperties = {
        @HystrixProperty(name = "execution.isolation.strategy", value = "SEMAPHORE")
    }
)
public String callService() { ... }
```

### Sentinel 隔离

Sentinel 主要通过并发线程数限流实现隔离：

```java
// 并发线程数限流
FlowRule rule = new FlowRule("callService")
    .setGrade(RuleConstant.FLOW_GRADE_THREAD)
    .setCount(10);

FlowRuleManager.loadRules(Collections.singletonList(rule));
```

## 限流能力对比

### Hystrix

Hystrix 不直接支持限流，需要配合 Netflix Zuul 或其他组件。

### Sentinel

Sentinel 支持多种限流方式：

| 限流方式 | 说明 |
|---|---|
| QPS 限流 | 按每秒请求数限流 |
| 并发线程数限流 | 按并发线程数限流 |
| 冷启动 | 预热后逐步达阈值 |
| 排队等待 | 匀速通过 |
| 热点参数限流 | 按参数值限流 |

## 选型建议

| 场景 | 推荐 |
|---|---|
| 新项目 | Sentinel |
| 已有 Hystrix 项目 | 迁移到 Sentinel |
| 需要限流 | Sentinel |
| 只需熔断 | 两者皆可 |
| Spring Cloud 项目 | Sentinel |

## 迁移方案

### Hystrix → Sentinel

```java
// Hystrix
@HystrixCommand(
    fallbackMethod = "fallback",
    commandProperties = {
        @HystrixProperty(name = "circuitBreaker.requestVolumeThreshold", value = "20")
    }
)
public String callService() {
    return restTemplate.getForObject("http://service/", String.class);
}

// 迁移到 Sentinel
@SentinelResource(value = "callService",
    fallback = "fallback")
public String callService() {
    return restTemplate.getForObject("http://service/", String.class);
}

public String fallback(Throwable t) {
    return "降级响应";
}
```

## 总结

| 维度 | Sentinel | Hystrix |
|---|---|---|
| 限流 | 支持 | 不支持 |
| 熔断 | 支持 | 支持 |
| 系统保护 | 支持 | 不支持 |
| 热点参数 | 支持 | 不支持 |
| 活跃度 | 活跃 | 停止维护 |
| 社区 | 阿里主导 | Netflix 已停止 |

**结论**：新项目推荐使用 Sentinel，已有 Hystrix 项目建议逐步迁移。
