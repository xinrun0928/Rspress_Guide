# Sentinel 流量控制

流量控制是 Sentinel 的核心能力，通过限制请求速率，防止系统被瞬时流量冲垮。

## 流量控制原理

```
没有限流：
请求 → 系统 → 正常（阈值内）
请求 → 系统 → 正常（阈值内）
请求 → 系统 → 正常（阈值内）
请求 → 系统 → 崩溃（超过阈值）

有限流：
请求 → Sentinel 检查 → 通过
请求 → Sentinel 检查 → 通过
请求 → Sentinel 检查 → [限流] → 拒绝
```

## 限流类型

| 类型 | 说明 | 应用场景 |
|---|---|---|
| QPS 限流 | 按每秒请求数限流 | 保护接口不被冲垮 |
| 并发线程数限流 | 按并发执行线程数限流 | 保护线程池不被耗尽 |

## QPS 限流

### 基本配置

```java
FlowRule rule = new FlowRule("createOrder");
rule.setGrade(RuleConstant.FLOW_GRADE_QPS);  // 按 QPS
rule.setCount(100);                           // 每秒 100 次
FlowRuleManager.loadRules(Collections.singletonList(rule));
```

### 限流效果

| 效果 | 说明 | 参数 |
|---|---|---|
| 直接拒绝 | 直接拒绝超出的请求 | `CONTROL_BEHAVIOR_DEFAULT` |
| 冷启动 | 逐步预热到阈值 | `CONTROL_BEHAVIOR_WARM_UP` |
| 排队等待 | 超出的请求排队等待 | `CONTROL_BEHAVIOR_WARM_UP_QUEUE` |

### 直接拒绝

```java
FlowRule rule = new FlowRule("createOrder");
rule.setGrade(RuleConstant.FLOW_GRADE_QPS);
rule.setCount(100);
rule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);

FlowRuleManager.loadRules(Collections.singletonList(rule));
```

```
QPS 100 的直接拒绝效果：

请求:  1    2    3    4    5 ... 99  100  101  102  103
结果: 通过 通过 通过 通过 通过 ... 通过 通过 拒绝  拒绝  拒绝
```

### 冷启动（Warm Up）

冷启动适合有预热过程的场景，如缓存预热：

```java
FlowRule rule = new FlowRule("createOrder");
rule.setGrade(RuleConstant.FLOW_GRADE_QPS);
rule.setCount(100);
rule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_WARM_UP);
rule.setWarmUpPeriodSec(10);  // 预热时间 10 秒

FlowRuleManager.loadRules(Collections.singletonList(rule));
```

```
冷启动效果：

初始阈值: 100 * (1/10) = 10 QPS
每秒递增: 10 QPS
10 秒后: 100 QPS

请求 1-10QPS: 通过
请求 11-20QPS: 通过
请求 21-30QPS: 通过
...
请求 91-100QPS: 通过（预热完成）
```

### 排队等待

让请求均匀通过，不直接拒绝：

```java
FlowRule rule = new FlowRule("createOrder");
rule.setGrade(RuleConstant.FLOW_GRADE_QPS);
rule.setCount(10);                              // 10 QPS
rule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_WARM_UP_QUEUE);
rule.setMaxQueueingTimeMs(500);                // 最多排队 500ms

FlowRuleManager.loadRules(Collections.singletonList(rule));
```

```
排队等待效果：

请求:  1    2    3    4    5    6 ... 50
时间:  0s   0.1s 0.2s 0.3s 0.4s 0.5s
结果: 通过 通过 通过 通过 通过 通过 ... 拒绝(超时)
```

## 并发线程数限流

### 基本配置

```java
FlowRule rule = new FlowRule("createOrder");
rule.setGrade(RuleConstant.FLOW_GRADE_THREAD);  // 按线程数
rule.setCount(10);                               // 最多 10 个线程
FlowRuleManager.loadRules(Collections.singletonList(rule));
```

### 场景

并发线程数限流适合以下场景：

- 保护线程池不被耗尽
- 保护下游服务不被并发请求打垮
- 限制资源占用

```java
// 场景：限制调用下游服务的并发数
try (Entry entry = SphU.entry("callDownstream")) {
    // 调用下游服务
    downstreamService.call();
} catch (BlockException e) {
    // 并发数已满，被限流
    return Result.fail("系统繁忙");
}
```

## 热点参数限流

### 基本概念

热点参数限流是更细粒度的限流，可以针对特定参数值限流：

```java
// 规则：对 itemId=123 的请求限流
ParamFlowRule rule = new ParamFlowRule("getItem")
    .setParamIdx(0)  // 第一个参数
    .setCount(10)     // 每个参数值 10 QPS
    .setDurationInSec(1);

ParamFlowRuleManager.loadRules(Collections.singletonList(rule));
```

### 配置示例

```java
// 热点参数限流
ParamFlowRule rule = ParamFlowRuleManager
    .getRulesOfResource("getItem")
    .get(0);

if (rule == null) {
    rule = new ParamFlowRule("getItem", ParamFlowRule.DEFAULT_BLOCK_MSG)
        .setParamIdx(0)
        .setCount(10);

    ParamFlowRuleManager.loadRules(Collections.singletonList(rule));
}
```

## 流量控制配置

### Dashboard 配置

Sentinel 提供 Dashboard 控制台，可以图形化配置限流规则：

```yaml title="application.yml"
spring:
  cloud:
    sentinel:
      # 连接 Dashboard
      dashboard: sentinel-dashboard:8080
```

启动后访问 `http://sentinel-dashboard:8080`，可以：

- 查看实时监控
- 配置限流规则
- 配置熔断规则
- 查看机器列表

### API 配置

```java
// 通过 API 添加规则
FlowRule rule = new FlowRule("createOrder")
    .setGrade(RuleConstant.FLOW_GRADE_QPS)
    .setCount(100)
    .setStrategy(RuleConstant.STRATEGY_DIRECT)
    .setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);

FlowRuleManager.loadRules(Collections.singletonList(rule));

// 获取规则
List<FlowRule> rules = FlowRuleManager.getRules();

// 删除规则
FlowRuleManager.clear("createOrder");
```

## 限流异常处理

### 方式一：try-catch

```java
try {
    Entry entry = SphU.entry("createOrder");
    // 业务逻辑
    orderService.create();
} catch (BlockException e) {
    // 被限流
    log.warn("请求被限流: {}", e.getClass().getSimpleName());
    return Result.fail("系统繁忙，请稍后重试");
} catch (Exception e) {
    // 其他异常
    Tracer.trace(e);
    throw e;
} finally {
    if (entry != null) {
        entry.exit();
    }
}
```

### 方式二：注解

```java
@SentinelResource(value = "createOrder",
    blockHandler = "createOrderBlockHandler")
public Result createOrder() {
    return orderService.create();
}

// 限流处理方法，签名必须与原方法一致
public Result createOrderBlockHandler(BlockException e) {
    return Result.fail("系统繁忙，请稍后重试");
}
```

### 方式三：全局异常处理

```java
@Component
public class SentinelBlockExceptionHandler
        implements BlockExceptionHandler {

    @Override
    public void handle(HttpServletRequest request,
                       HttpServletResponse response,
                       BlockException e) throws Exception {

        String msg;
        if (e instanceof FlowException) {
            msg = "请求被限流";
        } else if (e instanceof DegradeException) {
            msg = "服务降级";
        } else {
            msg = "系统繁忙";
        }

        response.setStatus(429);
        response.setContentType("application/json;charset=utf-8");
        response.getWriter().write(JSON.toJSONString(Result.fail(msg)));
    }
}
```

## 总结

限流核心配置：

| 配置项 | 说明 |
|---|---|
| `grade` | 限流类型：QPS / 线程数 |
| `count` | 阈值 |
| `controlBehavior` | 限流效果：直接拒绝 / 冷启动 / 排队等待 |
