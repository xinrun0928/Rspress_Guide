# Sentinel 核心概念

理解 Sentinel 的三个核心概念——资源、规则、插槽链，是掌握 Sentinel 的基础。

## 三大核心概念

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Sentinel 核心概念                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   资源（Resource）                                                   │
│   └── 需要保护的目标：接口、方法、URL                                 │
│                                                                     │
│   规则（Rule）                                                      │
│   └── 限流规则、熔断规则、系统规则                                    │
│                                                                     │
│   插槽链（Slot Chain）                                              │
│   └── 一个个插槽串联起来，实现流量控制                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 资源（Resource）

资源是 Sentinel 保护的对象，可以是：

| 资源类型 | 示例 | 说明 |
|---|---|---|
| URL | `/api/order/create` | HTTP 接口 |
| 方法 | `com.example.OrderService.create()` | Java 方法 |
| 注解方法 | `@SentinelResource("createOrder")` | 注解标注的方法 |

### 定义资源

```java
// 方式一：SphU.entry() 定义资源
try {
    Entry entry = SphU.entry("createOrder");
    // 业务逻辑
    orderService.create();
} catch (BlockException e) {
    // 被限流或熔断
    return Result.fail("系统繁忙，请稍后重试");
} finally {
    if (entry != null) {
        entry.exit();
    }
}

// 方式二：@SentinelResource 注解
@SentinelResource(value = "createOrder",
    blockHandler = "createOrderBlockHandler",
    fallback = "createOrderFallback")
public Result createOrder() {
    return orderService.create();
}

// 限流处理
public Result createOrderBlockHandler(BlockException e) {
    return Result.fail("系统繁忙，请稍后重试");
}

// 降级处理
public Result createOrderFallback(Throwable t) {
    return Result.fail("服务暂时不可用");
}
```

## 规则（Rule）

Sentinel 支持多种规则：

### 限流规则（FlowRule）

```java
FlowRule rule = new FlowRule("createOrder");  // 资源名
rule.setGrade(RuleConstant.FLOW_GRADE_QPS);   // 按 QPS 限流
rule.setCount(100);                            // 每秒 100 次
rule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);
rule.setMaxQueueingTimeMs(500);               // 排队等待时间
rule.setClusterMode(false);                   // 非集群模式

FlowRuleManager.loadRules(Collections.singletonList(rule));
```

### 熔断规则（DegradeRule）

```java
DegradeRule rule = new DegradeRule("createOrder");
rule.setGrade(RuleConstant.DEGRADE_GRADE_RT);  // 按平均响应时间
rule.setCount(2000);                            // 阈值 2000ms
rule.setTimeWindow(10);                         // 熔断窗口 10 秒

FlowRuleManager.loadRules(Collections.singletonList(rule));
```

### 系统规则（SystemRule）

```java
SystemRule rule = new SystemRule();
rule.setHighestSystemLoad(3.0);    // 最高系统负载
rule.setQps(1000);                 // 最高 QPS
rule.setAvgRt(5000);               // 最高平均响应时间
rule.setMaxThread(100);            // 最高并发线程数

SystemRuleManager.loadRules(Collections.singletonList(rule));
```

## 插槽链（Slot Chain）

插槽链是 Sentinel 的核心执行链路：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Sentinel 插槽链                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────┐                                                  │
│   │   请求入口   │                                                  │
│   └──────┬──────┘                                                  │
│          ▼                                                          │
│   ┌─────────────┐                                                  │
│   │  NodeSelector │ → 构建调用链节点                                 │
│   └──────┬──────┘                                                  │
│          ▼                                                          │
│   ┌─────────────┐                                                  │
│   │ ClusterBuilder│ → 构建集群节点                                   │
│   └──────┬──────┘                                                  │
│          ▼                                                          │
│   ┌─────────────┐                                                  │
│   │  Statistic  │ → 统计数据                                       │
│   └──────┬──────┘                                                  │
│          ▼                                                          │
│   ┌─────────────┐                                                  │
│   │   FlowSlot  │ → 限流检查                                        │
│   └──────┬──────┘                                                  │
│          ▼                                                          │
│   ┌─────────────┐                                                  │
│   │ DegradeSlot │ → 熔断检查                                        │
│   └──────┬──────┘                                                  │
│          ▼                                                          │
│   ┌─────────────┐                                                  │
│   │ SystemSlot  │ → 系统自适应检查                                   │
│   └──────┬──────┘                                                  │
│          ▼                                                          │
│   ┌─────────────┐                                                  │
│   │   请求通过   │                                                  │
│   └─────────────┘                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 核心插槽

| 插槽 | 职责 |
|---|---|
| NodeSelectorSlot | 构建资源调用链节点 |
| ClusterBuilderSlot | 构建集群节点，统计集群数据 |
| StatisticSlot | 统计数据（QPS、线程数、响应时间） |
| FlowSlot | 限流检查 |
| DegradeSlot | 熔断降级检查 |
| SystemSlot | 系统自适应检查 |

## 统计节点（StatisticNode）

StatisticNode 使用滑动窗口算法统计数据：

```java
public class StatisticNode {

    // 滑动窗口计数器
    private final Metric rollingCounter;
    private final ClusterNode clusterNode;

    // 当前线程数
    private final AtomicInteger curThread;

    // 增加 QPS
    public void addPassRequest() {
        rollingCounter.addPass();
    }

    // 增加拒绝
    public void addBlockRequest() {
        rollingCounter.addBlock();
    }

    // 增加线程
    public void increaseThread() {
        curThread.incrementAndGet();
    }

    // 获取 QPS
    public double getQPS() {
        return rollingCounter.getQPS();
    }
}
```

### 滑动窗口

```
滑动窗口：将时间线划分为多个小窗口，定期滑动

┌─────────────────────────────────────────────────────────────────┐
│ 时间线                                                              │
│ 0s        1s        2s        3s        4s        5s            │
│ ├─────────┼─────────┼─────────┼─────────┼─────────┤            │
│ │ Window1 │ Window2 │ Window3 │ Window4 │ Window5 │            │
│ │ 0-1s    │ 1-2s    │ 2-3s    │ 3-4s    │ 4-5s    │            │
│ └─────────┴─────────┴─────────┴─────────┴─────────┘            │
│                                           │                       │
│                                    滑动窗口（统计最近 1 秒）          │
│                                    Window4 + Window5              │
│                                                                     │
└─────────────────────────────────────────────────────────────────┘
```

## 规则加载

### 静态加载

```java
// 直接加载规则
FlowRuleManager.loadRules(rules);
DegradeRuleManager.loadRules(rules);
```

### 动态加载

Sentinel 支持从配置中心动态加载规则：

```java
// 使用 DataSource 动态加载
ReadableDataSource<String, List<FlowRule>> flowRuleDataSource =
    new ApolloDataSource("sentinel-flow-rules",
        "flow-rules",
        JSON.parseObject($$`{"data": $0}`, s -> {
            List<FlowRule> rules = new ArrayList<>();
            JSON.parseArray(s).forEach(e -> {
                FlowRule rule = JSON.parseObject(e.toString(), FlowRule.class);
                rules.add(rule);
            });
            return rules;
        }));

FlowRuleManager.register2Property(flowRuleDataSource.getProperty());
```

### 配置中心集成

```yaml title="application.yml"
spring:
  cloud:
    sentinel:
      enabled: true
      eager:
        # 懒加载改为启动时加载
        enabled: true
      datasource:
        ds:
          nacos:
            server-addr: nacos-server:8848
            data-id: sentinel-rules
            group-id: SENTINEL_GROUP
            data-type: json
            rule-type: flow
```

## 总结

Sentinel 核心概念：

| 概念 | 说明 |
|---|---|
| 资源 | Sentinel 保护的对象 |
| 规则 | 限流、熔断、系统规则 |
| 插槽链 | 串联各个检查环节 |
| StatisticNode | 滑动窗口统计数据 |
