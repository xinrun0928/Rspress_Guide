# 分布式任务调度面试题

面试官问分布式任务调度，往往不是想让你背 API，而是想考察你对分布式系统的理解——任务不重复执行怎么保证、任务分片怎么实现、高可用怎么设计。

## 常见面试题

### 1. 分布式任务调度的核心问题是什么？

这道题考察对分布式任务调度本质的理解。

**核心问题：**

1. **任务不重复执行**：多台机器同时触发，如何保证只执行一次
2. **高可用**：调度器或执行器宕机，任务能否正常执行
3. **任务分片**：大任务如何拆分到多台机器
4. **一致性**：任务执行状态如何保证

### 2. 如何保证任务不重复执行？

**常见方案：**

| 方案 | 原理 | 适用场景 |
|---|---|---|
| 分布式锁 | 只有获取锁的节点执行 | 单机到多机 |
| 调度中心 | 调度中心只分配一个执行器 | XXL-Job |
| 数据库标记 | 抢任务标记，执行后清除 | 所有方案 |
| 分片路由 | 每个分片只在一个节点 | 分片任务 |

**代码示例（分布式锁）：**

```java
@Scheduled(cron = "0 0 2 * * ?")
public void dailyTask() {
    String lockKey = "task:dailySettlement";
    Boolean acquired = redis.setIfAbsent(lockKey, "locked", Duration.ofMinutes(30));

    if (Boolean.TRUE.equals(acquired)) {
        try {
            doSettlement();
        } finally {
            redis.delete(lockKey);
        }
    }
}
```

### 5. ElasticJob 如何实现任务分片？

```java
@ElasticJobConf(
    name = "orderShardingJob",
    shardingTotalCount = 4,
    shardingItemParameters = "0=0-9999,1=10000-19999,2=20000-29999,3=30000-39999"
)
public class OrderShardingJob implements SimpleJob {

    @Override
    public void execute(ShardingContext context) {
        int shardingItem = context.getShardingItem();
        String range = context.getShardingParameter();

        String[] parts = range.split("-");
        long startId = Long.parseLong(parts[0]);
        long endId = Long.parseLong(parts[1]);

        // 根据分片处理对应范围的数据
        processOrders(startId, endId);
    }
}
```

### 6. 如何实现任务失败重试？

```java
// XXL-Job 重试配置
@XxlJob("retryJobHandler")
public ReturnT<String> execute(String param) {
    int maxRetries = 3;
    for (int i = 0; i < maxRetries; i++) {
        try {
            // 业务逻辑
            doSomething();
            return ReturnT.SUCCESS;
        } catch (Exception e) {
            if (i == maxRetries - 1) {
                // 最后一次也失败
                return new ReturnT<>(500, "任务失败: " + e.getMessage());
            }
            // 等待后重试
            Thread.sleep((i + 1) * 1000);
        }
    }
    return ReturnT.SUCCESS;
}
```

### 8. 如何保证任务幂等性？

```java
// 使用唯一键保证幂等
public void processOrder(Long orderId) {
    String key = "processed:order:" + orderId;

    // 检查是否已处理
    if (redis.exists(key)) {
        log.info("订单已处理，跳过: {}", orderId);
        return;
    }

    // 标记正在处理（防止重复处理）
    redis.setex(key, 3600, "processing");

    try {
        // 业务逻辑
        doProcess(orderId);

        // 标记已处理
        redis.set(key, "done");
    } catch (Exception e) {
        // 失败时删除标记，允许重试
        redis.del(key);
        throw e;
    }
}
```

## 面试追问方向

1. **Quartz 集群**：锁的粒度是什么？如何避免死锁？
2. **XXL-Job**：调度中心挂了怎么办？执行器如何发现调度中心？
3. **ElasticJob**：ZooKeeper 挂了怎么办？分片任务如何处理不均衡？
4. **任务分片**：分片数如何决定？数据倾斜怎么办？
5. **高可用**：HA 如何实现？脑裂如何避免？

这些追问方向，也是你深入学习的方向。
