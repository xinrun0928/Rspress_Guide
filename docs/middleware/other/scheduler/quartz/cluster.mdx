# Quartz 集群原理

Quartz 集群通过数据库实现分布式协调，保证任务不重复执行。

## 为什么需要集群？

```
单机问题：

服务器 A: 定时器触发 → 执行任务
服务器 B: 定时器也触发 → 执行任务
→ 任务执行了两次！

集群方案：

调度器 A ──→ 获取锁 ──→ 执行任务
调度器 B ──→ 获取锁失败 ──→ 等待

→ 只有一个调度器执行任务
```

## 分布式锁

### 锁表结构

```sql
CREATE TABLE QRTZ_LOCKS (
    LOCK_NAME VARCHAR(40) NOT NULL PRIMARY KEY
);
```

### 锁类型

| 锁名 | 用途 |
|---|---|
| TRIGGER_ACCESS | 触发器访问锁 |
| JOB_ACCESS | 任务访问锁 |
| STATE_ACCESS | 状态访问锁 |
| CALENDAR_ACCESS | 日历访问锁 |

### 获取锁

```java
// 伪代码
public synchronized String acquireTriggerLock(SchedulingContext ctx, String triggerName) {
    Connection conn = getConnection();

    try {
        conn.setAutoCommit(false);

        // 尝试获取锁（FOR UPDATE 悲观锁）
        PreparedStatement ps = conn.prepareStatement(
            "SELECT * FROM QRTZ_LOCKS WHERE LOCK_NAME = ? FOR UPDATE"
        );
        ps.setString(1, triggerName);
        ResultSet rs = ps.executeQuery();

        if (rs.next()) {
            return triggerName;  // 获取成功
        } else {
            return null;  // 获取失败
        }
    } finally {
        conn.close();
    }
}
```

## 集群配置

### quartz.properties

```yaml
org.quartz.scheduler.instanceName = MyClusteredScheduler
org.quartz.scheduler.instanceId = AUTO
org.quartz.scheduler.instanceIdGenerator.class = org.quartz.simpl.HostnameInstanceIdGenerator

# JobStore 配置
org.quartz.jobStore.class = org.quartz.impl.jdbcjobstore.JobStoreTX
org.quartz.jobStore.driverDelegateClass = org.quartz.impl.jdbcjobstore.StdJDBCDelegate
org.quartz.jobStore.dataSource = quartzDS
org.quartz.jobStore.tablePrefix = QRTZ_
org.quartz.jobStore.isClustered = true
org.quartz.jobStore.clusterCheckinInterval = 10000
org.quartz.jobStore.useProperties = true

# ThreadPool 配置
org.quartz.threadPool.class = org.quartz.simpl.SimpleThreadPool
org.quartz.threadPool.threadCount = 10
org.quartz.threadPool.threadPriority = 5

# 数据源配置
org.quartz.dataSource.quartzDS.driver = com.mysql.cj.jdbc.Driver
org.quartz.dataSource.quartzDS.URL = jdbc:mysql://localhost:3306/quartz
org.quartz.dataSource.quartzDS.user = root
org.quartz.dataSource.quartzDS.password = password
org.quartz.dataSource.quartzDS.maxConnections = 10
```

### Spring Boot 配置

```yaml
spring:
  quartz:
    job-store-type: jdbc

      org:
        quartz:
          scheduler:
            instanceName: MyClusteredScheduler
```

## 节点发现问题

### 心跳机制

```
节点启动：
1. 向 QRTZ_SCHEDULER_STATE 表插入记录
   INSERT INTO QRTZ_SCHEDULER_STATE
   (SCHED_NAME, INSTANCE_NAME, LAST_CHECKIN_TIME, CHECKIN_INTERVAL)
   VALUES ('scheduler', 'instance1', NOW(), 10000)

2. 定期更新心跳
   UPDATE QRTZ_SCHEDULER_STATE
   SET LAST_CHECKIN_TIME = NOW()
   WHERE INSTANCE_NAME = 'instance1'

3. 检查其他节点心跳
   SELECT * FROM QRTZ_SCHEDULER_STATE
   WHERE LAST_CHECKIN_TIME < NOW() - CHECKIN_INTERVAL * 2
```

### 节点失效检测

```sql
-- 检测失效节点
SELECT INSTANCE_NAME FROM QRTZ_SCHEDULER_STATE
WHERE LAST_CHECKIN_TIME < NOW() - (CHECKIN_INTERVAL * 2)
```

## 任务分配

### 分配算法

```
触发器分配给节点：

1. 查询所有可用的触发器
   SELECT * FROM QRTZ_TRIGGERS
   WHERE STATUS = 'WAITING'

2. 尝试获取锁
   SELECT * FROM QRTZ_LOCKS WHERE LOCK_NAME = 'TRIGGER_ACCESS' FOR UPDATE

3. 分配给当前节点
   UPDATE QRTZ_TRIGGERS
   SET INSTANCE_NAME = 'instance1', STATUS = 'ACQUIRED'
   WHERE TRIGGER_NAME = 'myTrigger'
```

## 注意事项

### 时钟同步

集群节点之间时钟必须同步，建议使用 NTP：

```bash
# 所有节点同步 NTP
ntpdate ntp-server
```

### 实例 ID 唯一

每个节点必须有唯一的 instanceId：

```yaml
# 方式一：自动生成（基于主机名）
org.quartz.scheduler.instanceId = AUTO

# 方式二：手动指定
org.quartz.scheduler.instanceId = instance1
```

## 总结

Quartz 集群核心机制：

| 机制 | 说明 |
|---|---|
| 数据库锁 | 保证同一时刻只有一个节点获取任务 |
| 心跳检测 | 检测节点存活 |
| 状态同步 | 任务状态在集群间同步 |
