# Quartz 任务状态与持久化

Quartz 支持将任务和触发器持久化到数据库，实现任务重启不丢失。

## 任务状态

### JobExecutionContext

Job 执行时会传入 JobExecutionContext，包含执行信息：

```java
public class ContextJob implements Job {

    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        // 获取任务信息
        JobDetail jobDetail = context.getJobDetail();
        JobKey jobKey = jobDetail.getKey();

        // 获取触发器信息
        Trigger trigger = context.getTrigger();
        TriggerKey triggerKey = trigger.getKey();

        // 获取执行时间
        Date fireTime = context.getFireTime();
        Date scheduledTime = context.getScheduledFireTime();

        // 判断是否恢复执行
        boolean recovered = context.isRecovering();

        // 获取重试次数
        int refireCount = context.getRefireCount();
    }
}
```

### JobExecutionException

任务执行时抛出异常：

```java
public class ErrorJob implements Job {

    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        try {
            // 业务逻辑
            doSomething();
        } catch (Exception e) {
            // 方案一：重新抛出，Quartz 会标记为失败
            throw new JobExecutionException(e);

            // 方案二：立即重新执行
            throw new JobExecutionException(e, true);  // 第二个参数为 true

            // 方案三：停止触发器
            JobExecutionException ex = new JobExecutionException(e);
            ex.setUnscheduleFiringTrigger(true);
            throw ex;
        }
    }
}
```

## 任务持久化注解

### @DisallowConcurrentExecution

禁止并发执行，同一个 JobDetail 不会同时执行：

```java
@DisallowConcurrentExecution
public class NoConcurrentJob implements Job {

    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        // 如果上一个执行还没结束，这个不会开始
    }
}
```

### @PersistJobDataAfterExecution

任务完成后持久化 JobDataMap：

```java
@PersistJobDataAfterExecution
public class StatefulJob implements Job {

    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        JobDataMap dataMap = context.getJobDetail().getJobDataMap();

        int count = dataMap.getInt("count", 0);
        count++;
        dataMap.put("count", count);  // 下次执行时能获取到

        System.out.println("执行次数: " + count);
    }
}
```

:::warning 配合使用
@PersistJobDataAfterExecution 应该和 @DisallowConcurrentExecution 配合使用，避免并发时数据冲突。
:::

## JobStore

JobStore 决定任务的存储方式。

### 内存存储（RAMJobStore）

```yaml
org.quartz.jobStore.class = org.quartz.simpl.RAMJobStore
```

- 任务存储在内存
- 重启后任务丢失
- 性能最好

### 数据库存储（JDBCJobStore）

```yaml
org.quartz.jobStore.class = org.quartz.impl.jdbcjobstore.JobStoreTX
org.quartz.jobStore.driverDelegateClass = org.quartz.impl.jdbcjobstore.StdJDBCDelegate
org.quartz.jobStore.dataSource = quartzDS
org.quartz.jobStore.tablePrefix = QRTZ_
org.quartz.jobStore.isClustered = true
org.quartz.jobStore.clusterCheckinInterval = 10000
```

### 数据库表结构

| 表名 | 说明 |
|---|---|
| QRTZ_JOB_DETAILS | 任务详情 |
| QRTZ_TRIGGERS | 触发器 |
| QRTZ_SIMPLE_TRIGGERS | SimpleTrigger 信息 |
| QRTZ_CRON_TRIGGERS | CronTrigger 信息 |
| QRTZ_BLOB_TRIGGERS | Blob 类型触发器 |
| QRTZ_CALENDARS | 日历 |
| QRTZ_PAUSED_TRIGGER_GRPS | 暂停的触发器组 |
| QRTZ_FIRED_TRIGGERS | 正在执行的触发器 |
| QRTZ_SCHEDULER_STATE | 调度器状态 |
| QRTZ_LOCKS | 分布式锁 |
