# Quartz Misfire 机制

Misfire 是指触发器应该触发但由于某种原因没有触发的现象。

## 什么是 Misfire？

```
场景：
- 任务配置为每 10 秒执行一次
- 9:00:00 执行一次
- 9:00:05 服务器关闭
- 9:00:10 服务器恢复

问题：
- 9:00:10 的触发应该如何处理？
→ 这就是 Misfire 需要处理的问题
```

## Misfire 原因

| 原因 | 说明 |
|---|---|
| 服务器关闭 | 服务器关闭期间错过的触发 |
| 调度器暂停 | 使用 standby() 暂停期间 |
| 负载过高 | 线程池无可用线程 |
| 数据库锁定 | 获取数据库锁失败 |

## Misfire 策略

### SimpleTrigger 策略

| 策略 | 说明 |
|---|---|
| SMART_POLICY | 默认，根据配置智能选择 |
| MISFIRE_INSTRUCTION_FIRE_NOW | 立即触发一次 |
| MISFIRE_INSTRUCTION_RESCHEDULE_NEXT_WITH_EXISTING_COUNT | 下一次触发，使用剩余重复次数 |
| MISFIRE_INSTRUCTION_RESCHEDULE_NEXT_WITH_REMAINING_COUNT | 下一次触发，使用剩余重复次数 |
| MISFIRE_INSTRUCTION_RESCHEDULE_NOW_WITH_EXISTING_REPEAT_COUNT | 立即触发，使用剩余重复次数 |
| MISFIRE_INSTRUCTION_RESCHEDULE_NOW_WITH_REMAINING_REPEAT_COUNT | 立即触发，使用剩余重复次数 |
| MISFIRE_INSTRUCTION_NOOP | 不执行任何操作 |

### CronTrigger 策略

| 策略 | 说明 |
|---|---|
| SMART_POLICY | 默认 |
| MISFIRE_INSTRUCTION_FIRE_ONCE_NOW | 立即触发一次 |
| MISFIRE_INSTRUCTION_DO_NOTHING | 不执行任何操作 |

## 配置 Misfire 策略

### Java 配置

```java
Trigger trigger = TriggerBuilder.newTrigger()
    .withIdentity("myTrigger", "group1")
    .withSchedule(SimpleScheduleBuilder.simpleSchedule()
        .withIntervalInSeconds(10)
        .withRepeatCount(5)
        .withMisfireHandlingInstructionFireNow())  // 立即触发错过的
    .build();

Trigger trigger = TriggerBuilder.newTrigger()
    .withIdentity("myTrigger", "group1")
    .withSchedule(CronScheduleBuilder.cronSchedule("0 0 2 * * ?")
        .withMisfireHandlingInstructionFireAndProceed())  // 触发后继续
    .build();
```

### Spring Boot 配置

```yaml
spring:
  quartz:
    properties:
      org:
        quartz:
          trigger:
            myTrigger:
              misfire-instruction: FIRE_NOW
```

## Misfire 阈值

### 配置检查间隔

```yaml
org.quartz.scheduler.misfireThreshold = 60000
```

超过 60 秒未触发的触发器会被标记为 misfire。

### 示例

```
触发器：每 10 秒执行一次

正常情况：
9:00:00 → 执行
9:00:10 → 执行
9:00:20 → 执行

Misfire 情况：
9:00:00 → 执行
9:00:10 → 服务器关闭（Misfire！）
9:00:20 → 服务器恢复
           触发器应该恢复到什么状态？
```

## 处理策略选择

### 选择原则

| 场景 | 推荐策略 |
|---|---|
| 重要任务，不允许丢失 | FIRE_NOW |
| 统计报表任务 | RESCHEDULE_NEXT_WITH_REMAINING_COUNT |
| 日志收集任务 | DO_NOTHING（避免重复） |

### 示例

```java
// 重要任务：立即触发
SimpleScheduleBuilder.simpleSchedule()
    .withIntervalInSeconds(10)
    .withMisfireHandlingInstructionFireNow()

// 统计任务：下次执行
SimpleScheduleBuilder.simpleSchedule()
    .withIntervalInSeconds(10)
    .withMisfireHandlingInstructionRescheduleNextWithExistingCount()

// 日志任务：忽略
SimpleScheduleBuilder.simpleSchedule()
    .withIntervalInSeconds(10)
    .withMisfireHandlingInstructionNoOp()
```

## 总结

Misfire 处理策略：

| 策略 | 适用场景 |
|---|---|
| FIRE_NOW | 重要任务，不允许丢失 |
| RESCHEDULE_NEXT | 普通任务 |
| DO_NOTHING | 允许丢失的任务 |
