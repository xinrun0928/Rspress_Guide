# XXL-Job 任务类型

XXL-Job 支持三种任务类型：Bean 模式、GLUE 模式、脚本模式。

## Bean 模式

Bean 模式是最常用的方式，将任务类注册到 Spring 容器中。

### 定义任务

```java
@Component
public class OrderJobHandler {

    @XxlJob("orderSettlementJob")
    public ReturnT<String> execute(String param) {
        // 执行业务逻辑
        try {
            List<Order> orders = orderService.getUnsettledOrders();
            for (Order order : orders) {
                orderService.settle(order);
            }
            return ReturnT.SUCCESS;
        } catch (Exception e) {
            return new ReturnT<>(500, "结算失败: " + e.getMessage());
        }
    }
}
```

### 启动执行器

```java
@SpringBootApplication
public class XxlJobExecutorApplication {

    public static void main(String[] args) {
        SpringApplication.run(XxlJobExecutorApplication.class, args);
    }

    @Bean
    public XxlJobSpringExecutor xxlJobExecutor() {
        XxlJobSpringExecutor executor = new XxlJobSpringExecutor();
        executor.setAdminAddresses("http://xxl-job-admin:8080/xxl-job-admin");
        executor.setAppname("order-service");
        executor.setPort(9999);
        executor.setAccessToken("xxl-job-token");
        executor.setLogPath("/data/logs/xxl-job");
        executor.setLogRetentionDays(30);
        return executor;
    }
}
```

### 注册 Handler

```java
@Component
public class MyJobHandler {

    @XxlJob(value = "myJob", init = "init", destroy = "destroy")
    public ReturnT<String> execute(String param) {
        // 业务逻辑
        return ReturnT.SUCCESS;
    }

    // 初始化方法
    public void init() {
        System.out.println("任务初始化");
    }

    // 销毁方法
    public void destroy() {
        System.out.println("任务销毁");
    }
}
```

## GLUE 模式

GLUE 模式支持在线编辑代码，适合快速验证和简单任务。

### GLUE(Java)

```java
// 在线编辑 Java 代码
@XxlJob("glueJavaJob")
public ReturnT<String> execute(String param) {
    System.out.println("GLUE Java Job 执行");
    return ReturnT.SUCCESS;
}
```

### GLUE(Shell)

```bash
#!/bin/bash
echo "GLUE Shell Job 执行"
echo "参数: $1"
exit 0
```

### GLUE(Python)

```python
#!/usr/bin/env python
import sys

print("GLUE Python Job 执行")
print("参数:", sys.argv[1] if len(sys.argv) > 1 else "")
sys.exit(0)
```

### GLUE 配置

```yaml
# 执行器配置
xxl:
  job:
    admin:
      addresses: http://xxl-job-admin:8080/xxl-job-admin
    executor:
      appname: glue-service
      port: 9999
```

## 脚本模式

脚本模式适合运维场景，如清理日志、备份数据等。

### 支持的脚本

| 类型 | 后缀 | 示例 |
|---|---|---|
| Shell | .sh | bash、zsh |
| Python | .py | python、python3 |
| PowerShell | .bat | Windows |
| Perl | .pl | perl |

### 脚本任务配置

```yaml
# 在调度中心配置
任务类型: SCRIPT
脚本类型: SHELL
脚本内容: |
  #!/bin/bash
  echo "开始清理日志"
  find /var/log -name "*.log" -mtime +7 -exec rm {} \;
  echo "清理完成"
```

## 任务参数

### 参数传递

```java
@XxlJob("paramJob")
public ReturnT<String> execute(String param) {
    // param = "userId=123,type=settlement"

    // 解析参数
    Map<String, String> params = parseParams(param);
    String userId = params.get("userId");

    // 根据参数执行
    processByUserId(userId);
    return ReturnT.SUCCESS;
}
```

### 分片参数

```java
@XxlJob("shardingJob")
public ReturnT<String> execute(String param) {
    // 获取分片信息
    int shardingIndex = XxlJobHelper.getShardIndex();
    int shardingTotal = XxlJobHelper.getShardTotal();

    // 根据分片执行
    System.out.println("分片 " + shardingIndex + " / " + shardingTotal);

    // 分片处理
    long start = shardingIndex * 10000L;
    long end = (shardingIndex + 1) * 10000L;
    processRange(start, end);

    return ReturnT.SUCCESS;
}
```

## 总结

任务类型对比：

| 类型 | 特点 | 适用场景 |
|---|---|---|
| Bean | 类注入、编译检查 | 生产环境 |
| GLUE | 在线编辑、动态执行 | 快速验证 |
| 脚本 | 运维友好 | 运维任务 |
