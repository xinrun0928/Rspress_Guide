# XXL-Job 路由策略

当任务配置了多个执行器时，路由策略决定任务分配给哪个执行器。

## 路由策略类型

```
┌─────────────────────────────────────────────────────────────────────┐
│                         XXL-Job 路由策略                              │
├─────────────────────────────────────────────────────────────────────┤

│   │     FIRST        │ │     LAST        │ │     ROUND       │  │
│   │                  │ │                  │ │                  │  │
│   │   选择第一个     │ │   选择最后一个   │ │   轮询选择       │  │
│   │                  │ │                  │ │                  │  │
│   └─────────────────┘ └─────────────────┘ └─────────────────┘  │
│                                                                     │

│   │                  │ │                  │ │                  │  │

```

## FIRST（第一个）

选择注册表中第一个执行器。

```java
public class FirstRouteStrategy {

    public String route(List<String> addresses) {
        // 返回第一个
        return addresses.get(0);
    }
}
```

适用场景：固定执行器，如主备切换。

## LAST（最后一个）

选择注册表中最后一个执行器。

```java
public class LastRouteStrategy {

    public String route(List<String> addresses) {
        return addresses.get(addresses.size() - 1);
    }
}
```

适用场景：需要固定使用某个执行器。

## ROUND（轮询）

按顺序轮流选择执行器。

```java
public class RoundRouteStrategy {

    private AtomicLong count = new AtomicLong(0);

    public String route(List<String> addresses) {
        int index = (int) (count.getAndIncrement() % addresses.size());
        return addresses.get(index);
    }
}
```

适用场景：负载均衡场景。

## CONSISTENT_HASH（一致性哈希）

相同参数的任务路由到同一执行器。

```java
public class ConsistentHashRouteStrategy {

    private final TreeMap<Long, String> hashRing = new TreeMap<>();

    public String route(List<String> addresses, String jobKey) {
        // 将执行器添加到哈希环
        for (String address : addresses) {
            for (int i = 0; i < 160; i++) {
                long hash = hash("address-" + i);
                hashRing.put(hash, address);
            }
        }

        // 计算任务哈希值
        long jobHash = hash(jobKey);

        // 顺时针查找最近的执行器
        Map.Entry<Long, String> entry = hashRing.ceilingEntry(jobHash);
        if (entry == null) {
            entry = hashRing.firstEntry();
        }
        return entry.getValue();
    }
}
```

适用场景：分片任务，保证同一分片固定执行器。

## LEAST_RECENTLY（最少最近使用）

选择最久未使用（LRU）的执行器。

```java
public class LeastRecentlyUsedStrategy {

    private ConcurrentHashMap<String, Long> lastUseTime = new ConcurrentHashMap<>();

    public String route(List<String> addresses) {
        long minTime = Long.MAX_VALUE;
        String selected = null;

        for (String address : addresses) {
            long lastTime = lastUseTime.getOrDefault(address, 0L);
            if (lastTime < minTime) {
                minTime = lastTime;
                selected = address;
            }
        }

        // 更新最近使用时间
        lastUseTime.put(selected, System.currentTimeMillis());
        return selected;
    }
}
```

适用场景：希望任务均匀分布到各执行器。

## LEAST_LOADED（最低负载）

选择当前负载最低的执行器。

```java
public class LeastLoadedStrategy {

    // 获取执行器的负载信息
    private Map<String, ExecutorRegistryInfo> registry = new ConcurrentHashMap<>();

    public String route(List<String> addresses) {
        String selected = null;
        int minLoad = Integer.MAX_VALUE;

        for (String address : addresses) {
            ExecutorRegistryInfo info = registry.get(address);
            int load = info.getLoad();  // 活跃线程数

            if (load < minLoad) {
                minLoad = load;
                selected = address;
            }
        }

        return selected;
    }
}
```

适用场景：动态负载均衡。

## BROADCAST（广播）

任务同时分发到所有执行器。

```java
public class BroadcastRouteStrategy {

    public List<String> route(List<String> addresses) {
        // 返回所有执行器
        return addresses;
    }
}
```

适用场景：需要在所有机器上同时执行，如通知发布。

## 选择建议

| 策略 | 特点 | 适用场景 |
|---|---|---|
| FIRST | 简单固定 | 主备切换 |
| LAST | 简单固定 | 固定执行器 |
| ROUND | 均匀分布 | 普通负载均衡 |
| CONSISTENT_HASH | 参数一致 | 分片任务 |
| LRU | 动态均衡 | 需要负载均衡 |
| LEAST_LOADED | 最低负载 | 动态负载均衡 |
| BROADCAST | 所有执行 | 广播通知 |

## 总结

路由策略决定了任务如何分配到执行器，选择合适的策略可以优化系统性能和可靠性。
