# XXL-Job 架构

XXL-Job 采用经典的两层架构：调度中心（Admin）和执行器（Executor）。

## 调度中心（Admin）

### 功能

| 功能 | 说明 |
|---|---|
| 任务管理 | CRUD 任务 |
| 执行器管理 | 注册/注销执行器 |
| 任务触发 | 触发任务执行 |
| 日志查询 | 查看执行日志 |
| 监控 | 执行统计、失败告警 |

## 执行器（Executor）

### 功能

| 功能 | 说明 |
|---|---|
| 注册 | 向调度中心注册自己 |
| 心跳 | 定期向调度中心发送心跳 |
| 执行 | 执行分配的任务 |
| 汇报 | 向调度中心汇报执行结果 |

## 通信机制

### HTTP 注册

```java
// 执行器启动时注册
@PostMapping("/registry")
public ReturnT<String> registry(@RequestBody RegistryParam registryParam) {
    // 向调度中心注册
    return adminClient.registry(
        "executor-server",
        "http://executor:9999",
        TriggerType.BEAN
    );
}

// 定期发送心跳
@PostMapping("/registry")
public ReturnT<String> heartbeat() {
    // 更新心跳时间
    return adminClient.heartbeat(executorKey);
}
```

### 执行器接收任务

```java
// 执行器接收任务调度
@Component
public class XxlJobSpringExecutor extends XxlJobExecutor {

    @Override
    public void start() {
        // 启动 Netty Server，接收任务调度
        super.start();
    }
}

// 处理任务
@Component
public class DemoJobHandler {

    @XxlJob("demoJobHandler")
    public ReturnT<String> execute(String param) {
        // 执行业务逻辑
        System.out.println("执行任务");
        return ReturnT.SUCCESS;
    }
}
```

## 执行流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                         XXL-Job 执行流程                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   1. 启动                                                         │
│      执行器 ──→ 注册 ──→ 调度中心                                 │
│                                                                     │
│   2. 配置任务                                                     │
│      管理员 ──→ 配置 cron、执行器 ──→ 调度中心                   │
│                                                                     │
│   3. 触发执行                                                     │
│      调度中心 ──→ 查询执行器 ──→ HTTP 调用执行器               │
│                                                                     │
│   4. 执行任务                                                     │
│      执行器线程池 ──→ 执行 JobHandler ──→ 返回结果             │
│                                                                     │
│   5. 记录日志                                                     │
│      执行器 ──→ 保存日志 ──→ 调度中心查询                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 执行器集群

```
┌─────────────────────────────────────────────────────────────────────┐
│                         执行器集群                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   调度中心                                                         │
│       │                                                           │
│       ├──→ 执行器 1 (机器 A)                                       │

```

## 总结

XXL-Job 架构要点：

| 组件 | 说明 |
|---|---|
| 调度中心 | 管理任务、触发执行、记录日志 |
| 执行器 | 注册自己、接收任务、执行任务 |
| 通信 | HTTP 调用 |
