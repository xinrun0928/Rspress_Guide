# XXL-Job GLUE 代码开发模式

GLUE 模式支持在线编辑代码，无需重新部署即可修改任务逻辑。

## GLUE 模式概述

```
┌─────────────────────────────────────────────────────────────────────┐
│                         GLUE 代码开发模式                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   调度中心                                                         │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │  GLUE 编辑器                                               │  │
│   │                                                              │  │
│   │  ┌─────────────────────────────────────────────────────┐ │  │
│   │  │ public class GlueJob {                             │ │  │
│   │  │     public ReturnT<String> execute(String param) {│ │  │
│   │  │         System.out.println("Hello GLUE");          │ │  │
│   │  │         return ReturnT.SUCCESS;                    │ │  │
│   │  │     }                                               │ │  │
│   │  │ }                                                   │ │  │
│   │  └─────────────────────────────────────────────────────┘ │  │
│   │                                                              │  │
│   │  [保存] [更新] [执行]                                     │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                              │                                        │
│                              ▼                                        │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                     执行器                                   │  │
│   │                                                              │  │
│   │   Groovy / JavaCompiler 动态编译                           │  │
│   │   执行编译后的代码                                          │  │
│   │                                                              │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## GLUE(Java)

### 代码结构

```java
package com.xxl.job.service.handler;

import com.xxl.job.core.context.XxlJobHelper;
import com.xxl.job.core.handler.IJobHandler;
import com.xxl.job.core.log.XxlJobLogger;

public class GlueJob extends IJobHandler {

    @Override
    public void execute() throws Exception {
        XxlJobLogger.log("GLUE Java Job 开始执行");

        // 执行业务逻辑
        String param = XxlJobHelper.getJobParam();
        System.out.println("参数: " + param);

        // 业务代码
        doSomething();

        XxlJobLogger.log("GLUE Java Job 执行完成");
    }
}
```

### 在线编辑

```
1. 在调度中心选择任务类型：GLUE(Java)
2. 点击「GLUE 代码」进入编辑器
3. 输入 Java 代码
4. 保存并触发执行
```

## GLUE(Shell)

### 基本语法

```bash
#!/bin/bash

echo "开始执行 Shell 脚本"
echo "参数: $1"

# 业务逻辑
date
whoami
df -h

echo "执行完成"
```

### 常用变量

| 变量 | 说明 |
|---|---|
| `$1` `$2` ... | 任务参数 |
| `$XXL_JOB_RUN_ID` | 本次运行 ID |
| `$XXL_JOB_LOG_ID` | 日志 ID |

## GLUE(Python)

### 基本语法

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-

import sys
from xxljob import XxlJobHelper

def execute():
    XxlJobHelper.log("开始执行 Python 脚本")

    # 获取参数
    param = XxlJobHelper.getJobParam()
    XxlJobHelper.log("参数: " + param)

    # 业务逻辑
    print("Hello Python")

    XxlJobHelper.log("执行完成")

if __name__ == "__main__":
    execute()
```

## GLUE(Go)

### 基本语法

```go
package main

import (
    "fmt"
    "github.com/xxl-job/xxljob-executor-go/xxljob"
)

func Execute() {
    xxljob.Logger.Info("开始执行 Go 脚本")

    // 获取参数
    param := xxljob.GetJobParam()
    xxljob.Logger.Info("参数: " + param)

    // 业务逻辑
    fmt.Println("Hello Go")

    xxljob.Logger.Info("执行完成")
}

func main() {
    Execute()
}
```

## 版本管理

### 版本历史

GLUE 代码支持版本历史：

```
┌─────────────────────────────────────────────────────┐
│  版本历史                                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  v3  2024-01-15 14:30:00  当前版本                 │
│  v2  2024-01-10 10:20:00                          │
│  v1  2024-01-05 09:15:00                          │
│                                                     │
│  [回滚到 v2] [查看 v2 代码]                       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 回滚代码

```java
// 如果发现新代码有问题，可以回滚到历史版本
// 调度中心点击「回滚到 v2」，任务将使用 v2 的代码执行
```

## 热更新原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                         GLUE 热更新原理                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   1. 保存代码                                                       │
│      调度中心 ──→ 保存到数据库 ──→ 生成版本号                       │
│                                                                     │
│   2. 同步到执行器                                                   │
│      执行器 ──→ 定时拉取 ──→ 缓存到本地                             │
│                                                                     │
│   3. 执行代码                                                       │
│      触发任务 ──→ 加载最新代码 ──→ 动态编译 ──→ 执行               │
│                                                                     │
│   4. 编译后缓存                                                     │
│      编译成功 ──→ 缓存 Class ──→ 下次直接使用                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 总结

GLUE 模式特点：

| 特点 | 说明 |
|---|---|

| 版本管理 | 支持历史版本回滚 |
| 热更新 | 立即生效 |
| 多语言 | Java/Shell/Python/Go |
