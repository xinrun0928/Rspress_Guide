# XXL-Job 高可用部署

XXL-Job 的高可用通过调度中心集群和执行器集群实现。

```
│   │  Nginx    │       │  Nginx    │       │            │           │
│   │  (主)     │       │  (备)     │       │            │           │
│   └─────┬─────┘       └─────┬─────┘       │            │           │
│         │                     │              │            │           │
│         └─────────┬───────────┘              │            │           │
│                   │                          │            │           │
```

## 调度中心集群

### 部署架构

```
调度中心 1:
├── 端口: 8080
├── MySQL: 读写
└── 注册到 Nginx

调度中心 2:
├── 端口: 8080
├── MySQL: 只读
└── 注册到 Nginx
```

### Nginx 配置

```nginx
upstream xxl-job-admin {
    server 192.168.1.1:8080 weight=10;
    server 192.168.1.2:8080 weight=10;
}

server {
    listen 80;
    server_name xxl-job.example.com;

    location / {
        proxy_pass http://xxl-job-admin;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 执行器配置

```yaml
xxl:
  job:
    admin:
      # 配置调度中心地址（多个用逗号分隔）
      addresses: http://192.168.1.1:8080,http://192.168.1.2:8080
    executor:
      appname: order-service
      port: 9999
```

## 执行器高可用

### 执行器集群

```
执行器 1 ──→ 注册 ──→ 调度中心
执行器 2 ──→ 注册 ──→ 调度中心
执行器 3 ──→ 注册 ──→ 调度中心
```

### 路由策略

任务分发时，调度中心会根据路由策略选择执行器：

```yaml
# 路由策略配置
路由策略: CONSISTENT_HASH
```

同一分片的任务会路由到同一执行器，保证一致性。

## 故障转移

### 执行器故障

```
执行器 1 宕机：
├── 执行器 1 心跳超时（30 秒）
├── 调度中心标记执行器 1 为失活
├── 任务自动路由到其他执行器
└── 执行器 1 恢复后重新注册
```

### 调度中心故障

```
主调度中心宕机：
├── Nginx 检测到不可用
├── 自动切换到备调度中心
├── 备调度中心接管任务调度
└── 主调度中心恢复后自动同步
```

## 监控

```bash
# 返回
{
  "code": 200,
  "msg": null,
  "data": {
    "total": 100,
    "running": 10
  }
}
```

### 执行器心跳

```
执行器心跳：每 30 秒发送一次
超时时间：90 秒（3 次心跳失败）
```

## 总结

高可用部署要点：

| 组件 | 高可用方式 |
|---|---|
| 调度中心 | 集群 + Nginx 负载均衡 |
| 执行器 | 集群 + 心跳检测 |
| 数据库 | MySQL 主从复制 |
| 注册中心 | 内置（MySQL） |
