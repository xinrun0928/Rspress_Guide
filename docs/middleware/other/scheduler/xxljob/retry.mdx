# XXL-Job 重试机制

任务执行失败时，XXL-Job 支持多种重试机制和失败处理策略。

## 重试配置

### 执行器配置

```yaml
xxl:
  job:
    executor:
      # 任务执行超时时间（秒）
      timeout: 60
      # 失败重试次数
      failRetryCount: 3
```

### 任务级别配置

在调度中心配置：

```
任务配置：
├── 超时时间：60 秒
├── 失败重试次数：3
└── 阻塞处理策略：单机串行
```

## 重试策略

### 自动重试

任务执行失败后，自动重试指定次数：

```
执行流程：

第 1 次执行 → 失败
    ↓
等待 1 分钟
    ↓
第 2 次执行 → 失败
    ↓
等待 1 分钟
    ↓
第 3 次执行 → 失败
    ↓
第 4 次执行（最后一次重试） → 失败
    ↓
记录失败日志 → 发送告警
```

### 代码示例

```java
@XxlJob("retryJobHandler")
public ReturnT<String> execute(String param) {
    try {
        // 业务逻辑
        doSomething();
        return ReturnT.SUCCESS;
    } catch (RetryableException e) {
        // 可重试异常，由框架重试
        return ReturnT.FAIL;
    } catch (BusinessException e) {
        // 业务异常，不重试
        return new ReturnT<>(500, "业务异常: " + e.getMessage());
    }
}
```

## 阻塞处理策略

当任务执行时间超过调度间隔时，阻塞策略决定如何处理。

### 单机串行（单机策略）

```
策略：上一个任务执行完，才执行下一个

时间线：
├── 10:00 → 任务开始
├── 10:05 → 任务执行中
├── 10:10 → 任务执行中（超过了 10:00 的调度间隔）
├── 10:15 → 任务完成
└── 10:15 → 执行被阻塞的任务（跳过 10:00 的触发）
```

```yaml
xxl:
  job:
    executor:
      block-strategy: DISCARD_LATER
```

### 丢弃后续调度（DISCARD_LATER）

丢弃调度时间点之后的所有触发，只执行一次。

```
时间线：
├── 10:00 → 任务开始
├── 10:05 → 第二次调度触发
│            ↓
│         丢弃（上一个还在执行）
├── 10:10 → 第三次调度触发
│            ↓
│         丢弃
├── 10:15 → 任务完成
└── 不再执行（已丢弃）
```

### 覆盖执行（SERIAL_EXECUTION）

如果上一次任务还未完成，用新触发覆盖执行。

```
时间线：
├── 10:00 → 任务开始
├── 10:05 → 第二次触发
│            ↓
│         取消上一次，切换到新的执行
├── 10:15 → 新任务完成
└── 不丢失任何触发
```

## 超时控制

### 执行器超时

```yaml
xxl:
  job:
    executor:
      timeout: 60  # 60 秒超时
```

### 代码中检查超时

```java
@XxlJob("timeoutJobHandler")
public ReturnT<String> execute(String param) {
    long startTime = System.currentTimeMillis();

    try {
        // 业务逻辑
        process();

        // 检查是否超时
        long cost = System.currentTimeMillis() - startTime;
        if (cost > XxlJobHelper.getTimeout()) {
            return new ReturnT<>(500, "执行超时");
        }

        return ReturnT.SUCCESS;
    } catch (Exception e) {
        return new ReturnT<>(500, "执行失败: " + e.getMessage());
    }
}
```

## 失败告警

### 配置告警

在调度中心配置告警联系人：

```
告警配置：
├── 负责人：张三
├── 邮箱：zhangsan@example.com
├── 电话：13800138000
└── 钉钉：zhangsan
```

### 代码触发告警

```java
@XxlJob("alertJobHandler")
public ReturnT<String> execute(String param) {
    try {
        doSomething();
        return ReturnT.SUCCESS;
    } catch (Exception e) {
        // 触发告警
        XxlJobHelper.log("任务执行失败", e);

        // 返回失败，让框架重试或告警
        return new ReturnT<>(500, "任务失败");
    }
}
```

## 总结

重试机制核心配置：

| 配置项 | 说明 |
|---|---|
| `failRetryCount` | 失败重试次数 |
| `timeout` | 执行超时时间 |
| `blockStrategy` | 阻塞处理策略 |

阻塞策略对比：

| 策略 | 特点 | 适用场景 |
|---|---|---|
| DISCARD_LATER | 丢弃后续触发 | 允许任务丢失 |
| COVER_EARLY | 覆盖执行 | 不允许任务丢失 |
| SERIAL_EXECUTION | 串行执行 | 任务互斥 |
