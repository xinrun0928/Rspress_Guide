# 任务调度基础

理解任务调度的核心概念，是学习各种调度框架的基础。

## 核心概念

### 触发器（Trigger）

触发器定义「什么时候执行」：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Trigger 类型                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────┐    ┌─────────────────────┐             │
│   │   Simple Trigger    │    │    Cron Trigger     │             │
│   │                      │    │                      │             │
│   │  固定时间间隔执行     │    │   Cron 表达式执行    │             │
│   │                      │    │                      │             │
│   │  每 5 分钟执行       │    │  每天凌晨 2 点执行  │             │
│   │  间隔 10 秒执行       │    │  每周一 9 点执行    │             │
│   │                      │    │                      │             │
│   └─────────────────────┘    └─────────────────────┘             │
│                                                                     │
│   ┌─────────────────────┐    ┌─────────────────────┐             │
│   │  Date Trigger      │    │  Calendar Trigger    │             │
│   │                      │    │                      │             │
│   │  指定日期时间执行     │    │  排除某些日期执行    │             │
│   │                      │    │                      │             │
│   │  2024-01-01 10:00  │    │  工作日执行          │             │
│   │                      │    │                      │             │
│   └─────────────────────┘    └─────────────────────┘             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 任务（Job）

Job 定义「执行什么」：

```java
// Job 接口
public interface Job {
    void execute(JobExecutionContext context) throws JobExecutionException;
}

// 示例
public class MyJob implements Job {
    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        System.out.println("定时任务执行！");
    }
}
```

### 调度器（Scheduler）

Scheduler 管理触发器和任务的注册、触发：

```java
// 创建调度器
Scheduler scheduler = StdSchedulerFactory.getDefaultScheduler();

// 创建任务
JobDetail job = JobBuilder.newJob(MyJob.class)
    .withIdentity("myJob", "group1")
    .build();

// 创建触发器（每天凌晨 2 点执行）
Trigger trigger = TriggerBuilder.newTrigger()
    .withIdentity("myTrigger", "group1")
    .withSchedule(CronScheduleBuilder.cronSchedule("0 0 2 * * ?"))
    .build();

// 注册并启动
scheduler.scheduleJob(job, trigger);
scheduler.start();
```

## Cron 表达式

### 格式

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Cron 表达式格式                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│    ┌─────────────┬─────────────┬─────────────┬─────────────┬─────────────┐│
│    │   Seconds    │   Minutes   │    Hours    │    Day      │   Month    ││
│    │    (秒)      │    (分)      │    (时)      │    (日)     │   (月)      ││
│    └─────────────┴─────────────┴─────────────┴─────────────┴─────────────┘│
│    ┌─────────────┐
│    │    Week     │
│    │    (周)      │
│    └─────────────┘
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 示例

| 表达式 | 说明 |
|---|---|
| `0 0 * * * ?` | 每小时整点执行 |
| `0 0 2 * * ?` | 每天凌晨 2 点 |
| `0 30 9 ? * MON-FRI` | 工作日 9:30 |
| `0 0/5 * * * ?` | 每 5 分钟 |
| `0 0 0 1 * ?` | 每月 1 号凌晨 |
| `0 0 0 ? * 1` | 每周一凌晨 |
| `0 0 10,14,16 * * ?` | 每天 10 点、14 点、16 点 |

### Quartz 特殊字符

| 字符 | 说明 |
|---|---|
| `*` | 任意值 |
| `?` | 无特定值（用于日/周冲突） |
| `-` | 范围 |
| `,` | 列表 |
| `/` | 步长 |
| `L` | 最后 |
| `W` | 工作日 |
| `#` | 第几个 |

## 任务类型

### 有状态 vs 无状态

```java
// 无状态任务
@DisallowConcurrentExecution
public class StatelessJob implements Job {
    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        // 每次都是新实例
    }
}

// 有状态任务
@PersistJobDataAfterExecution
public class StatefulJob implements Job {
    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        // 可以访问和修改 JobDataMap
        int count = context.getJobDetail().getJobDataMap().getInt("count", 0);
        count++;
        context.getJobDetail().getJobDataMap().put("count", count);
    }
}
```

### 持久化 vs 内存

```java
// 内存存储（重启丢失）
JobDetail job = JobBuilder.newJob(MyJob.class)
    .storeDurably()
    .build();

// 数据库持久化
scheduler.addJob(job, true);
```

## 监听器

### Job 监听器

```java
// Job 执行前
public class MyJobListener implements JobListener {
    @Override
    public String getName() {
        return "MyJobListener";
    }

    @Override
    public void jobToBeExecuted(JobExecutionContext context) {
        System.out.println("任务即将执行: " + context.getJobDetail().getKey());
    }

    @Override
    public void jobWasExecuted(JobExecutionContext context, JobExecutionException e) {
        System.out.println("任务执行完成");
    }
}

// 注册监听器
scheduler.getListenerManager().addJobListener(new MyJobListener(), KeyMatcher.keyEquals(JobKey.jobKey("myJob")));
```

### Trigger 监听器

```java
public class MyTriggerListener implements TriggerListener {
    @Override
    public String getName() {
        return "MyTriggerListener";
    }

    @Override
    public void triggerFired(Trigger trigger, JobExecutionContext context) {
        System.out.println("触发器触发了");
    }

    @Override
    public boolean vetoJobExecution(Trigger trigger, JobExecutionContext context) {
        // 返回 true 表示否决任务执行
        return false;
    }
}
```

## 总结

任务调度核心概念：

| 概念 | 说明 |
|---|---|
| Trigger | 触发器，定义执行时间 |
| Job | 任务，定义执行内容 |
| Scheduler | 调度器，管理触发和执行 |
| JobDetail | 任务详情，包含数据和配置 |
| JobDataMap | 任务数据存储 |
