# 单机 vs 分布式任务调度

当系统规模从小到大演进时，任务调度方案也需要升级。这一章我们对比单机调度和分布式调度的差异。

## 单机调度的问题

### 定时器 + 线程池

最简单的方式：

```java
// 单机定时器
@Scheduled(cron = "0 0 2 * * ?")
public void dailyTask() {
    // 每天凌晨 2 点执行
    doSomething();
}
```

### 问题一：重复执行

```
服务器 A: 定时器触发 → 执行任务
服务器 B: 定时器也触发 → 执行任务
→ 任务执行了两次！
```

### 问题二：单点故障

```
服务器 A: 宕机了
→ 定时任务无法执行
→ 业务中断
```

### 问题三：无统一管理

```
问题：
- 无法查看任务执行状态
- 无法手动触发任务
- 无法修改执行时间
- 无法查看执行日志
```

## 分布式调度

### 架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         分布式调度架构                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                      调度中心（Admin）                         │  │
│   │                                                              │  │
│   │  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │  │
│   │  │ 任务管理 │  │ 任务触发 │  │ 执行日志 │                     │  │
│   │  └─────────┘  └─────────┘  └─────────┘                     │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                              │                                      │
│                              │ 调度指令                              │
│         ┌───────────────────┼───────────────────┐                  │
│         │                   │                   │                   │
│         ▼                   ▼                   ▼                   │
│   ┌───────────┐       ┌───────────┐       ┌───────────┐           │
│   │ 执行器 1  │       │ 执行器 2  │       │ 执行器 3  │           │
│   │  (Executor)│       │  (Executor)│       │  (Executor)│           │
│   │  App A    │       │  App B    │       │  App C    │           │
│   └───────────┘       └───────────┘       └───────────┘           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 优势

| 能力 | 单机调度 | 分布式调度 |
|---|---|---|
| 任务分发 | 只能本地执行 | 跨机器分发 |
| 高可用 | 单点故障 | 自动切换 |
| 负载均衡 | 无 | 支持 |
| 任务管理 | 无 | 统一管理界面 |
| 执行日志 | 无 | 集中存储 |
| 失败重试 | 无 | 自动重试 |

## 调度模式

### 中心化调度

```
┌─────────────────────────────────────────────────────────────────────┐
│                         中心化调度                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   调度中心                                                         │
│      │                                                             │
│      ├──→ 执行器 1                                               │
│      ├──→ 执行器 2                                               │
│      └──→ 执行器 3                                               │
│                                                                     │
│   特点：                                                          │
│   - 调度器是单点（可用 HA 解决）                                   │
│   - 所有调度请求经过调度中心                                       │
│   - 适合任务量适中的场景                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 去中心化调度

```
┌─────────────────────────────────────────────────────────────────────┐
│                         去中心化调度                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   执行器 1 ←→ 执行器 2 ←→ 执行器 3                                 │
│      │              │              │                               │
│      └──→ 竞争获取任务 ←→ 竞争获取任务                             │
│                                                                     │
│   特点：                                                          │
│   - 无中心调度器                                                   │
│   - 节点之间竞争                                                   │
│   - 适合大规模任务                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 任务分片

### 什么是任务分片？

将大任务拆分为多个子任务，分布到不同机器执行：

```
总数据: 100 万条
分片数: 4

执行器 1: 处理 0-25 万
执行器 2: 处理 25-50 万
执行器 3: 处理 50-75 万
执行器 4: 处理 75-100 万
```

### 分片策略

```java
// XXL-Job 分片任务
@XxlJob("shardingJobHandler")
public ReturnT<String> execute(String param) {
    // 获取分片参数
    int shardIndex = XxlJobHelper.getShardIndex();
    int shardTotal = XxlJobHelper.getShardTotal();

    // 根据分片执行
    int start = shardIndex * (total / shardTotal);
    int end = (shardIndex + 1) * (total / shardTotal);

    processData(start, end);
    return ReturnT.SUCCESS;
}
```

## 选择建议

| 场景 | 推荐方案 |
|---|---|
| 单机应用，定时任务简单 | Spring @Scheduled |
| 多机器部署，需要任务管理 | XXL-Job |
| 超大规模，需要任务分片 | ElasticJob |
| 需要工作流编排 | PowerJob |

## 总结

| 维度 | 单机调度 | 分布式调度 |
|---|---|---|
| 适用规模 | 小 | 中/大 |
| 高可用 | 无 | 有 |
| 任务管理 | 无 | 有 |
| 任务分片 | 无 | 有 |
| 运维复杂度 | 低 | 中/高 |
