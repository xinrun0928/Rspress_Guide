# 任务调度核心概念

理解任务调度的核心概念，是学习各种调度框架的基础。

## 核心概念

### 触发器（Trigger）

触发器定义「什么时候执行」：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Trigger 类型                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────┐    ┌─────────────────────┐             │
│   │   Simple Trigger    │    │    Cron Trigger     │             │
│   │                      │    │                      │             │
│   │  固定时间间隔执行     │    │   Cron 表达式执行    │             │
│   │                      │    │                      │             │
│   │  每 5 分钟执行       │    │  每天凌晨 2 点执行  │             │
│   │  间隔 10 秒执行       │    │  每周一 9 点执行    │             │
│   │                      │    │                      │             │
│   └─────────────────────┘    └─────────────────────┘             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 任务（Job）

Job 定义「执行什么」：

```java
// Job 接口
public interface Job {
    void execute(JobExecutionContext context) throws JobExecutionException;
}

// 示例
public class MyJob implements Job {
    @Override
    public void execute(JobExecutionContext context) throws JobExecutionException {
        System.out.println("定时任务执行！");
    }
}
```

### 调度器（Scheduler）

Scheduler 管理触发器和任务的注册、触发：

```java
// 创建调度器
Scheduler scheduler = StdSchedulerFactory.getDefaultScheduler();

// 创建任务
JobDetail job = JobBuilder.newJob(MyJob.class)
    .withIdentity("myJob", "group1")
    .build();

// 创建触发器（每天凌晨 2 点执行）
Trigger trigger = TriggerBuilder.newTrigger()
    .withIdentity("myTrigger", "group1")
    .withSchedule(CronScheduleBuilder.cronSchedule("0 0 2 * * ?"))
    .build();

// 注册并启动
scheduler.scheduleJob(job, trigger);
scheduler.start();
```

## Cron 表达式

### 格式

```
秒 分 时 日 月 周 [年]
```

### 常用示例

| 表达式 | 说明 |
|---|---|
| `0 0 * * * ?` | 每小时整点 |
| `0 0 2 * * ?` | 每天凌晨 2 点 |
| `0 30 9 ? * MON-FRI` | 工作日 9:30 |
| `0 0/5 * * * ?` | 每 5 分钟 |
| `0 0 0 1 * ?` | 每月 1 号凌晨 |
| `0 0 0 ? * 1` | 每周一凌晨 |
| `0 0 10,14,16 * * ?` | 每天 10、14、16 点 |

### 特殊字符

| 字符 | 说明 | 示例 |
|---|---|---|
| `*` | 任意值 | `* * * * * ?` 每秒 |
| `?` | 无特定值（日周互斥） | `0 0 0 ? * 1` |
| `-` | 范围 | `0 0 9-17 * * ?` |
| `,` | 列表 | `0 0 10,14,16 * * ?` |
| `/` | 步长 | `0 0 0/2 * * ?` |
| `L` | 最后 | `L * * * ?` |
| `W` | 工作日 | `0 0 0 15W * ?` |

## 任务类型

### 有状态 vs 无状态

```java
// 无状态任务（不允许并发）
@DisallowConcurrentExecution
public class StatelessJob implements Job {
    // 同一个 JobDetail 的多个执行会串行
}

// 有状态任务（允许并发）
@PersistJobDataAfterExecution
public class StatefulJob implements Job {
    // JobDataMap 会持久化
}
```

### 任务分组

```java
// 创建任务组
JobKey jobKey = JobKey.jobKey("myJob", "group1");
JobDetail job = JobBuilder.newJob(MyJob.class)
    .withIdentity(jobKey)
    .build();

// 创建触发器组
TriggerKey triggerKey = TriggerKey.triggerKey("myTrigger", "group1");
```

## 监听器

### Job 监听器

```java
public class MyJobListener implements JobListener {
    @Override
    public String getName() {
        return "MyJobListener";
    }

    @Override
    public void jobToBeExecuted(JobExecutionContext context) {
        // 任务即将执行
    }

    @Override
    public void jobWasExecuted(JobExecutionContext context, JobExecutionException e) {
        // 任务执行完成
    }
}

// 注册
scheduler.getListenerManager().addJobListener(new MyJobListener());
```

## 总结

任务调度核心概念：

| 概念 | 说明 |
|---|---|
| Trigger | 触发器，定义执行时间 |
| Job | 任务，定义执行内容 |
| Scheduler | 调度器，管理触发和执行 |
| JobDetail | 任务详情 |
| JobDataMap | 任务数据 |
