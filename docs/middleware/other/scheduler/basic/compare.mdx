# Quartz vs XXL-Job vs ElasticJob 选型对比

当前主流的分布式任务调度框架有三个：Quartz、XXL-Job、ElasticJob。这一章我们对比它们的特点，帮助你选择合适的框架。

## 基本对比

| 维度 | Quartz | XXL-Job | ElasticJob |
|---|---|---|---|
| 开发语言 | Java | Java | Java |
| 定时类型 | Cron | Cron | Cron + 多种 |
| 架构 | 中心化 | 中心化 | 去中心化 |
| 任务管理 | 无 UI | Web UI | Web UI |
| 分片 | 不支持 | 支持 | 强大支持 |
| 失败重试 | 简单 | 完善 | 完善 |
| 监控 | 简单 | 完善 | 完善 |
| 运维 | 简单 | 中等 | 中等 |

## Quartz

### 特点

Quartz 是 Java 领域最成熟的定时任务框架：

- 功能完善
- 与 Spring 集成良好
- 支持集群
- 需要自己管理任务

### 适用场景

- 已有 Quartz 应用迁移
- 需要与 Spring 深度集成
- 任务量不大，不需要复杂分片

## XXL-Job

### 特点

XXL-Job 是一个轻量级分布式任务调度平台：

- 管理界面完善
- 任务分片支持
- GLUE 模式（在线编辑代码）
- 高可用部署

### 架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         XXL-Job 架构                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │

│   │                    调度中心 (Admin)                            │  │
│   │                                                              │  │
│   │  Web UI ──→ 任务管理 ──→ 执行器注册 ──→ 日志查询           │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                              │                                      │
│                              │ RPC 调用                              │
│         ┌───────────────────┼───────────────────┐                  │
│         │                   │                   │                   │
│         ▼                   ▼                   ▼                   │
│   ┌───────────┐       ┌───────────┐       ┌───────────┐           │
│   │ 执行器 1   │       │ 执行器 2   │       │ 执行器 3   │           │
```

### GLUE 模式

```java
// GLUE(Java) 模式 - 在线编辑 Java 代码
@XxlJob("glueJobHandler")
public ReturnT<String> execute(String param) {
    XxlJobLogger.log("GLUE JOB START");
    // 在线编写的代码
    System.out.println("执行任务");
    return ReturnT.SUCCESS;
}
```

### 适用场景

- 需要任务管理界面
- 任务量中等
- 需要 GLUE 在线编辑

## ElasticJob

### 特点

ElasticJob 是面向互联网的海量任务调度框架：

- 去中心化架构
- 强大的分片能力
- 多种作业类型
- 云原生支持

### 分片原理

```java
// ElasticJob 分片任务
@ElasticJobConf(
    name = "myShardingJob",
    shardingTotalCount = 4,
    shardingItemParameters = "0=北京,1=上海,2=广州,3=深圳"
)
public class MyShardingJob implements SimpleJob {
    @Override
    public void execute(ShardingContext context) {
        switch (context.getShardingItem()) {
            case 0:
                processCity("北京");
                break;
            case 1:
                processCity("上海");
                break;
            // ...
        }
    }
}
```

### 适用场景

- 海量任务分片
- 需要高可用
- 云原生环境

## 功能对比

| 功能 | Quartz | XXL-Job | ElasticJob |
|---|---|---|---|
| Cron 定时 | 支持 | 支持 | 支持 |
| 固定间隔 | 支持 | 支持 | 支持 |
| 任务分片 | 不支持 | 支持 | 强大支持 |
| 失败重试 | 简单 | 完善 | 完善 |
| 任务依赖 | 不支持 | 支持 | 不支持 |
| 任务编排 | 不支持 | 部分支持 | 不支持 |
| 管理界面 | 无 | 完善 | 完善 |
| 告警通知 | 无 | 支持 | 支持 |
| 日志查询 | 无 | 完善 | 完善 |

## 选择建议

| 场景 | 推荐 |
|---|---|
| 简单定时任务，Spring 项目 | Quartz |
| 需要管理界面，中等规模 | XXL-Job |
| 海量任务，需要分片 | ElasticJob |
| 需要 GLUE 在线编辑 | XXL-Job |
| 需要任务依赖 | XXL-Job |
| 云原生环境 | ElasticJob |

## 总结

三大框架对比：

| 框架 | 优势 | 劣势 |
|---|---|---|
| Quartz | 成熟、简单 | 无管理界面、无分片 |
| XXL-Job | 管理完善、GLUE | 中心化架构 |
| ElasticJob | 分片强大、云原生 | ZooKeeper 依赖 |
