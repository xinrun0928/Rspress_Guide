# ElasticJob 作业类型

ElasticJob 支持三种作业类型：SimpleJob、DataflowJob、ScriptJob。

## SimpleJob

SimpleJob 是最简单的作业类型，一次性执行完成。

### 定义 SimpleJob

```java
@ElasticJobConf(
    name = "dailySettlementJob",
    shardingTotalCount = 4,
    shardingItemParameters = "0=Beijing,1=Shanghai,2=Guangzhou,3=Shenzhen",
    cron = "0 0 2 * * ?"
)
public class DailySettlementJob implements SimpleJob {

    @Override
    public void execute(ShardingContext context) {
        // 获取当前分片
        int shardingItem = context.getShardingItem();
        String shardingParameter = context.getShardingParameter();

        System.out.println("开始结算分片: " + shardingItem + ", 区域: " + shardingParameter);

        // 分片处理
        List<Order> orders = orderRepository.findByRegion(shardingParameter);
        for (Order order : orders) {
            settle(order);
        }

        System.out.println("分片 " + shardingItem + " 结算完成");
    }
}
```

### 完整配置

```java
@ElasticJobConf(
    name = "mySimpleJob",
    shardingTotalCount = 3,
    shardingItemParameters = "0=北京,1=上海,2=广州",
    cron = "0/5 * * * * ?",

    // 作业参数
    jobParameter = "default",

    // 监控配置
    monitorExecution = true,
    monitorPort = 9888,

    // 熔断配置
    maxTimeDiffSeconds = 60,
    misfire = true,

    // 错过执行策略
   错过触发策略 = SIMPLE,

    // 分片策略
    shardingStrategyType = "AVG_ALLOCATION",

    // 业务调度器类型
    jobShardingStrategyType = "AVG_ALLOCATION",

    // 弹性扩缩容
    allowSendJobResultNotification = true,

    // 禁用
    disabled = false,

    // 覆盖
    overwrite = true
)
public class MySimpleJob implements SimpleJob {
    @Override
    public void execute(ShardingContext context) {
        // 业务逻辑
    }
}
```

## DataflowJob

DataflowJob 是流式作业类型，需要多次拉取数据并处理。

### 定义 DataflowJob

```java
@ElasticJobConf(
    name = "orderDataflowJob",
    shardingTotalCount = 4,
    cron = "0/10 * * * * ?",
    streamingProcessing = true
)
public class OrderDataflowJob implements DataflowJob<Order> {

    @Override
    public List<Order> fetchData(ShardingContext context) {
        // 拉取待处理数据
        int shardingItem = context.getShardingItem();
        return orderRepository.findUnprocessed(shardingItem, 100);
    }

    @Override
    public void processData(ShardingContext context, List<Order> data) {
        // 处理数据
        for (Order order : data) {
            process(order);
            order.setProcessed(true);
            orderRepository.save(order);
        }
    }
}
```

### 处理流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                       DataflowJob 处理流程                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────┐                                                  │
│   │ fetchData() │ → 拉取待处理数据                                    │
│   └──────┬──────┘                                                  │
│          │ 有数据                                                  │
│          ▼                                                         │
│   ┌─────────────┐                                                  │
│   │ processData()│ → 处理数据                                        │
│   └──────┬──────┘                                                  │
│          │                                                         │
│          │ 还有数据                                                │
│          └───────────┘                                              │
│                                                                     │
│   ┌─────────────┐                                                  │
│   │ fetchData() │ → 继续拉取                                        │
│   └──────┬──────┘                                                  │
│          │ 无数据                                                  │
│          ▼                                                         │
│   ┌─────────────┐                                                  │
│   │   作业完成  │                                                  │
│   └─────────────┘                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## ScriptJob

ScriptJob 是脚本作业类型，执行 Shell、Python 等脚本。

### 定义 ScriptJob

```java
@ElasticJobConf(
    name = "cleanupJob",
    shardingTotalCount = 1,
    shardingItemParameters = "0=/bin/bash /opt/scripts/cleanup.sh"
)
public class CleanupScriptJob implements ScriptJob {

    @Override
    public void execute(ShardingContext context) {
        // 脚本任务由框架自动执行
        // 这里可以添加额外的逻辑
    }
}
```

### Spring 配置

```xml
<elasticjob:job id="scriptJob"
    class="org.apache.shardingsphere.elasticjob.api.ScriptJob"
    sharding-total-count="1"
    sharding-item-parameters="0=/bin/bash /opt/cleanup.sh"
    cron="0 0 2 * * ?"/>
```

### 脚本示例

```bash
#!/bin/bash
echo "开始清理日志"

# 查找并删除 7 天前的日志
find /var/log -name "*.log" -mtime +7 -exec rm {} \;

# 清理临时文件
rm -rf /tmp/*

echo "清理完成"
```

## 作业监听器

### ElasticJobListener

```java
public class MyJobListener implements ElasticJobListener {

    @Override
    public void beforeJobExecuted(ShardingContexts shardingContexts) {
        System.out.println("作业执行前: " + shardingContexts.getJobName());
    }

    @Override
    public void afterJobExecuted(ShardingContexts shardingContexts) {
        System.out.println("作业执行后: " + shardingContexts.getJobName());
    }
}
```

### 注册监听器

```java
@ElasticJobConf(
    name = "myJob",
    shardingTotalCount = 3,
    cron = "0/5 * * * * ?",
    listenerClasses = MyJobListener.class
)
public class MyJob implements SimpleJob {
    @Override
    public void execute(ShardingContext context) {
        // 业务逻辑
    }
}
```

## 总结

| 作业类型 | 说明 | 适用场景 |
|---|---|---|
| SimpleJob | 普通任务，一次执行 | 对账、统计 |
| DataflowJob | 流式任务，多次拉取 | 数据同步、处理 |
| ScriptJob | 脚本任务 | 运维任务 |
