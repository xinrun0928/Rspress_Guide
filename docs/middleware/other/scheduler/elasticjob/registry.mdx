# ElasticJob 注册中心原理

ElasticJob 使用 ZooKeeper 作为注册中心，实现服务注册、分片协调、选举等功能。

## ZooKeeper 节点结构

```
/elasticjob-lite
├── jobs                                          # 作业目录
│   └── {job-name}
│       ├── config                              # 作业配置
│       ├── instances                           # 运行实例
│       │   ├── {instance-1-id}@instance
│       │   └── {instance-2-id}@instance
│       ├── sharding                           # 分片状态
│       │   ├── 0
│       │   │   └── instance                   # 分片 0 分配的实例
│       │   ├── 1
│       │   │   └── instance
│       │   └── ...
│       └── leader                             # 主节点选举
│           └── election
└── ...
```

## 分片协调

### 分片分配算法

```java
// 平均分配算法
public class AvgAllocationShardingStrategy {

    public Map<Instance, List<Integer>> sharding(
            List<Integer> shardingItems,
            List<Instance> instances) {

        Map<Instance, List<Integer>> result = new HashMap<>();

        // 按实例数量轮询分配
        for (int i = 0; i < shardingItems.size(); i++) {
            Instance instance = instances.get(i % instances.size());
            result.computeIfAbsent(instance, k -> new ArrayList<>())
                    .add(shardingItems.get(i));
        }

        return result;
    }
}
```

### 分片变化

```bash
# 分片前：2 个实例，4 个分片
/jobs/myJob/sharding/0/instance -> instance-1
/jobs/myJob/sharding/1/instance -> instance-1
/jobs/myJob/sharding/2/instance -> instance-2
/jobs/myJob/sharding/3/instance -> instance-2

# 分片后：3 个实例，4 个分片
/jobs/myJob/sharding/0/instance -> instance-1
/jobs/myJob/sharding/1/instance -> instance-2
/jobs/myJob/sharding/2/instance -> instance-3
/jobs/myJob/sharding/3/instance -> instance-1
```

## 主节点选举

### 选举原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                         ZooKeeper 选举                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   节点启动：                                                         │
│   /jobs/myJob/leader/election/instance-1 ──→ 创建临时节点               │
│                                                                     │
│   检查主节点：                                                        │
│   getData(/jobs/myJob/leader/election) ──→ 返回主节点 ID              │
│                                                                     │
│   主节点故障：                                                        │
│   ZooKeeper ──→ 临时节点消失 ──→ 触发重新选举                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 选举实现

```java
public class LeaderService {

    public void elect() {
        // 1. 创建临时节点
        String instanceId = generateInstanceId();

        try {
            // 尝试创建为主节点
            zkClient.createEphemeral(path + "/leader/election/" + instanceId);

            // 2. 设置为主节点
            setLeader(instanceId);

        } catch (KeeperException.NodeExistsException e) {
            // 3. 已被占用，等待其他节点释放
            waitForLeader();
        }
    }

    private void setLeader(String instanceId) {
        // 更新配置节点
        String data = JSON.toJSONString(new LeaderNode(instanceId));
        zkClient.setData(path + "/leader/election", data);
    }
}
```

## 节点监听

### Watch 机制

```java
// 监听实例变化
zkClient.getContentWanted(
    "/jobs/myJob/instances",
    (event) -> {
        if (event.getType() == EventType.NodeChildrenChanged) {
            // 实例列表变化，重新分配分片
            rebalanceSharding();
        }
    }
);

// 监听分片变化
zkClient.getDataWanted(
    "/jobs/myJob/sharding/0/instance",
    (event) -> {
        if (event.getType() == EventType.NodeChanged) {
            // 分片分配变化
            updateAssignment();
        }
    }
);
```

### 事件类型

| 事件 | 说明 |
|---|---|
| NodeCreated | 节点创建 |
| NodeDeleted | 节点删除 |
| NodeDataChanged | 节点数据变化 |
| NodeChildrenChanged | 子节点变化 |

## 总结

ZooKeeper 在 ElasticJob 中的作用：

| 功能 | ZooKeeper 操作 |
|---|---|
| 服务注册 | 临时节点 + 心跳 |
| 分片协调 | 持久节点 + Watch |
| 主节点选举 | 临时顺序节点 |
| 配置管理 | 持久节点 |
