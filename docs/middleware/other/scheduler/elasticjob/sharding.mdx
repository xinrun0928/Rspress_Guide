# ElasticJob 分片原理

分片是 ElasticJob 的核心能力，通过将大任务拆分为多个小任务，分布到不同节点执行。

## 分片概念

```

│                         ElasticJob 分片概念                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   总数据: 100 万条                                                  │
│   分片数: 4                                                         │
│                                                                     │
│   ┌───────────┐ ┌───────────┐ ┌───────────┐ ┌───────────┐        │
│   │  分片 0   │ │  分片 1   │ │  分片 2   │ │  分片 3   │        │
│   │ 0-25万    │ │ 25-50万   │ │ 50-75万   │ │75-100万   │        │
```

## 分片配置

### 配置分片数和参数

```java
// 方式一：注解配置
@ElasticJobConf(
    name = "orderJob",
    shardingTotalCount = 4,
    shardingItemParameters = "0=Beijing,1=Shanghai,2=Guangzhou,3=Shenzhen"
)
public class OrderJob implements SimpleJob {
    @Override
    public void execute(ShardingContext context) {
        int shardingItem = context.getShardingItem();
        String shardingParameter = context.getShardingParameter();
        // 根据分片执行
    }
}

// 方式二：Spring 配置
<elasticjob:job id="orderJob"
    class="com.example.OrderJob"
    sharding-total-count="4"
    sharding-item-parameters="0=Beijing,1=Shanghai,2=Guangzhou,3=Shenzhen"/>
```

### 分片参数

```java
// 获取分片信息
ShardingContext context = jobExecutionContext;

int shardingItem = context.getShardingItem();        // 分片序号
String shardingParameter = context.getShardingParameter();  // 分片参数
String jobName = context.getJobName();              // 作业名称
int shardingTotalCount = context.getShardingTotalCount();  // 总分片数
```

```java
// ElasticJob 内置平均分片策略
```

### 自定义分片策略

```java
// 实现 ShardingStrategy 接口
public class MyShardingStrategy implements ShardingStrategy {

    @Override
    public List<Integer> sharding(
            List<Integer> shardingItems,
            ShardingConfiguration shardingConfiguration) {

        // 自定义分片逻辑
        // 例如：按实例 IP 分配
        List<Integer> result = new ArrayList<>();
        String serverIp = getServerIp();

        for (int item : shardingItems) {
            if (shouldAssign(item, serverIp)) {
                result.add(item);
            }
        }

        return result;
    }
}
```

## ZooKeeper 分片协调

### 节点结构

```
/elasticjob-lite/jobs/myJob
├── config                                      # 作业配置
├── instances                                   # 运行实例
│   ├── 192.168.1.1@12345@instance
│   └── 192.168.1.2@12345@instance
├── sharding                                   # 分片信息
│   ├── 0
│   │   └── instance                           # 分片 0 分配的实例
│   ├── 1
│   │   └── instance
│   └── ...
└── leader                                     # 主节点选举
    └── election
```

### 分片分配

```java
// 主节点执行分片分配
public class JobShardingService {

    public void sharding() {
        // 1. 获取所有在线实例
        List<Instance> instances = getOnlineInstances();

        // 2. 获取总分片数
        int totalCount = getShardingTotalCount();

        // 3. 平均分配
        Map<Instance, List<Integer>> assignment = new HashMap<>();

        for (int i = 0; i < totalCount; i++) {
            Instance instance = instances.get(i % instances.size());
            assignment.computeIfAbsent(instance, k -> new ArrayList<>())
                    .add(i);
        }

        // 4. 更新 ZooKeeper
        updateShardingAssignment(assignment);
    }
}
```

## 动态分片

### 运行时修改分片

```java
// 方式一：通过 Spring
JobFacade jobFacade = jobExecutionContext.getJobFacade();
jobFacade.getShardingService().rescheduleJob("myJob", "0/10 * * * * ?");

// 方式二：通过 ZooKeeper
// 直接修改 /elasticjob-lite/jobs/myJob/config 节点
```

### 分片重新分配

当节点加入或离开时，会触发重新分片：

```
节点变化：
实例 A 运行 ──→ 实例 B 加入 ──→ 触发重新分片

重新分片：
旧分配：实例 A (0, 1, 2, 3)
新分配：实例 A (0, 1), 实例 B (2, 3)
```

## 分片监听

```java
// 分片变化监听
public class MyJob implements SimpleJob {

    @Override
    public void execute(ShardingContext context) {
        // 获取当前节点的分片
        List<Integer> shardingItems = context.getShardingItems();

        for (int item : shardingItems) {
            processItem(item);
        }
    }
}

// 监听分片变化
public class MyElasticJobListener implements ElasticJobListener {

    @Override
    public void beforeJobExecuted(ShardingContexts shardingContexts) {
        // 作业执行前
        System.out.println("即将执行分片: " + shardingContexts.getShardingItems());
    }

    @Override
    public void afterJobExecuted(ShardingContexts shardingContexts) {
        // 作业执行后
        System.out.println("分片执行完成");
    }
}
```
