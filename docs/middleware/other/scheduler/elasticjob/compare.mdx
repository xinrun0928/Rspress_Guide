# ElasticJob vs XXL-Job vs Quartz

三大分布式任务调度框架的对比总结。

## 基本对比

| 维度 | ElasticJob | XXL-Job | Quartz |
|---|---|---|---|
| 开发语言 | Java | Java | Java |
| 架构 | 去中心化 | 中心化 | 无中心 |
| 注册中心 | ZooKeeper | MySQL | JDBC/MySQL |
| 分片支持 | 强大 | 支持 | 不支持 |
| 管理界面 | 有 | 有 | 无 |
| 任务类型 | 3 种 | 3 种 | 多种 |
| Spring Boot | 支持 | 支持 | 支持 |

## 功能对比

| 功能 | ElasticJob | XXL-Job | Quartz |
|---|---|---|---|
| Cron 定时 | 支持 | 支持 | 支持 |
| 分片任务 | 强大 | 支持 | 不支持 |
| 失败重试 | 支持 | 支持 | 支持 |
| 任务依赖 | 不支持 | 支持子任务 | 不支持 |
| 广播任务 | 支持 | 支持 | 不支持 |
| 任务编排 | 弱 | 强 | 弱 |
| 监控告警 | 支持 | 支持 | 需要配置 |

## 分片能力对比

### ElasticJob

```java
@ElasticJobConf(
    name = "myJob",
    shardingTotalCount = 4,
    shardingItemParameters = "0=Beijing,1=Shanghai,2=Guangzhou,3=Shenzhen"
)
public class MyJob implements SimpleJob {
    @Override
    public void execute(ShardingContext context) {
        int item = context.getShardingItem();
        // 根据分片执行
    }
}
```

### XXL-Job

```java
@XxlJob("shardingJob")
public ReturnT<String> execute(String param) {
    int shardingIndex = XxlJobHelper.getShardIndex();
    int shardingTotal = XxlJobHelper.getShardTotal();
    // 根据分片执行
}
```

### Quartz

Quartz 不支持原生分片，需要手动实现：

```java
// 需要手动实现分片逻辑
for (int i = 0; i < shardTotal; i++) {
    if (i == shardIndex) {
        // 执行分片任务
    }
}
```

## 架构对比

### ElasticJob

```
应用 ──→ ZooKeeper ──→ 分片协调
                │
                └──→ 无调度中心
```

### XXL-Job

```
调度中心（Admin）──→ 执行器（Executor）
      │
      ├──→ 任务管理
      ├──→ 触发调度
      └──→ 日志收集
```

### Quartz

```
应用 ──→ 数据库 ──→ 集群锁
```

## 选择建议

| 场景 | 推荐 | 理由 |
|---|---|---|
| 海量任务分片 | ElasticJob | 分片能力强 |
| 任务依赖 | XXL-Job | 支持子任务链 |
| 轻量级 | Quartz | 简单依赖 |
| 有管理界面 | XXL-Job / ElasticJob | XXL-Job 更完善 |
| ZooKeeper 环境 | ElasticJob | 原生集成 |
| MySQL 环境 | XXL-Job | 原生支持 |

## 性能对比

| 指标 | ElasticJob | XXL-Job | Quartz |
|---|---|---|---|
| 调度延迟 | 低 | 低 | 低 |
| 扩展性 | 好 | 好 | 好 |
| 资源占用 | 中 | 中 | 低 |
| 数据库压力 | 无 | 中 | 高 |

## 总结

三大框架特点：

| 框架 | 优势 | 劣势 |
|---|---|---|
| ElasticJob | 分片强大、去中心化 | ZooKeeper 依赖 |
| XXL-Job | 管理完善、任务依赖 | 中心化架构 |
| Quartz | 简单、成熟 | 无管理界面、无分片 |

> 分布式任务调度模块的内容就到这里，「其他中间件」模块全部完成。
