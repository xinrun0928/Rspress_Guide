## CloudJob 配置

### Maven 依赖

```xml title="pom.xml"
<dependency>
    <groupId>org.apache.shardingsphere.elasticjob</groupId>
    <artifactId>elasticjob-cloud-spring-boot-starter</artifactId>
    <version>3.0.4</version>
</dependency>
```

### Spring 配置

```yaml title="application.yml"
spring:
  elasticjob:
    cloud:
      # ZooKeeper 配置
      zookeeper:
        server-lists: localhost:2181
        namespace: elasticjob-cloud

      # Mesos 配置
      mesos:
        url: localhost:5050
        failover-timeout-milliseconds: 300000

    jobs:
      myCloudJob:
        class: com.example.MyCloudJob
        sharding-total-count: 4
        cpu-count: 1
        memory-mb: 1024
        cron: 0/5 * * * * ?
```

## 资源分配

### CPU 和内存

```yaml
spring:
  elasticjob:
    jobs:
      myJob:
        sharding-total-count: 4
        cpu-count: 2
        memory-mb: 2048
        cron: 0/5 * * * * ?
```

### 软硬限制

```yaml
spring:
  elasticjob:
    jobs:
      myJob:
        sharding-total-count: 4
        cpu-count: 2
        soft-cpu-count-limit: 1.5
        memory-mb: 2048
        soft-memory-mb-limit: 1536
```

## 任务分配

### 任务分配流程

```
1. 任务调度
   Scheduler ──→ 读取作业配置 ──→ 计算资源需求

2. 资源请求
   Scheduler ──→ Mesos Master ──→ 请求 Offer

3. 任务分配
   Mesos Master ──→ Offer ──→ Scheduler

4. 任务启动
   Scheduler ──→ 启动任务 ──→ 执行 Job
```

### 任务状态

```
PENDING ──→ READY ──→ RUNNING ──→ COMPLETED
                │                      ↑
                │                      │
                └─── FAILED ──→ FAILOVER
```

## 总结

CloudJob vs LiteJob：

| 维度 | LiteJob | CloudJob |
|---|---|---|
| 架构 | 去中心化 | Mesos 调度 |
| 资源 | 固定 | 动态分配 |
| 扩缩容 | 手动 | 自动 |
| 适用 | 中小规模 | 大规模集群 |
