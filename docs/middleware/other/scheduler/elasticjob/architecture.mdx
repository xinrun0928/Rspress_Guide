# ElasticJob 架构

ElasticJob 有两种部署模式：LiteJob（轻量级）和 CloudJob（云原生）。

## 两种部署模式

```
┌─────────────────────────────────────────────────────────────────────┐
│                         ElasticJob 部署模式                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────┐  ┌─────────────────────┐             │
│   │    ElasticJob-Lite    │  │  ElasticJob-Cloud   │             │
│   │                      │  │                      │             │
│   │  轻量级              │  │  云原生              │             │
│   │  嵌入应用            │  │  Mesos 调度         │             │
│   │  ZooKeeper 协调      │  │  资源分配           │             │
│   │                      │  │                      │             │
│   │  适合中小规模        │  │  适合大规模          │             │
│   └─────────────────────┘  └─────────────────────┘             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## LiteJob 架构

LiteJob 采用去中心化架构，无需独立的调度中心：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         ElasticJob-Lite 架构                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                         ZooKeeper 集群                                │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  /elasticjob-lite                                      │   │
│   │    ├── jobs                                           │   │
│   │    │   └── myJob                                      │   │
│   │    │       ├── instances                               │   │
│   │    │       │   ├── 192.168.1.1@instance                │   │
│   │    │       │   └── 192.168.1.2@instance                │   │
│   │    │       ├── sharding                                │   │
│   │    │       │   └── 0=Beijing,1=Shanghai,2=Guangzhou    │   │
│   │    │       └── config                                  │   │
│   │    └── leader                                          │   │
│   │        └── election                                    │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                     │
│   ┌───────────┐       ┌───────────┐                           │
│   │ 作业服务 1 │       │ 作业服务 2 │                           │
│   │ (实例 A)  │       │ (实例 B)  │                           │
│   └───────────┘       └───────────┘                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 组件职责

| 组件 | 说明 |
|---|---|
| Registry | ZooKeeper 注册中心 |
| JobScheduler | 作业调度器 |
| JobExecutor | 作业执行器 |
| ShardingService | 分片服务 |

## LiteJob 配置

### Maven 依赖

```xml title="pom.xml"
<dependency>
    <groupId>org.apache.shardingsphere.elasticjob</groupId>
    <artifactId>elasticjob-lite-spring</artifactId>
    <version>3.0.4</version>
</dependency>
```

### Spring 配置

```xml title="applicationContext.xml"
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:elasticjob="http://shardingsphere.apache.org/schema/elasticjob">

    <!-- ZooKeeper 配置 -->
    <elasticjob:zookeeper id="registryCenter"
        server-lists="localhost:2181"
        namespace="elasticjob"
        base-sleep-time-milliseconds="1000"
        max-sleep-time-milliseconds="3000"
        max-retries="3"/>

    <!-- 作业配置 -->
    <elasticjob:jobs>
        <elasticjob:job id="myJob"
            class="com.example.MyJob"
            registry-center-ref="registryCenter"
            sharding-total-count="3"
            sharding-item-parameters="0=Beijing,1=Shanghai,2=Guangzhou"
            cron="0/5 * * * * ?"/>
    </elasticjob:jobs>

</beans>
```

### Spring Boot 配置

```yaml title="application.yml"
spring:
  elasticjob:
    reg-center:
      server-lists: localhost:2181
      namespace: elasticjob
    jobs:
      myJob:
        class: com.example.MyJob
        sharding-total-count: 3
        sharding-item-parameters: 0=Beijing,1=Shanghai,2=Guangzhou
        cron: 0/5 * * * * ?
```

## 作业执行流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                         ElasticJob 作业执行流程                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   1. 作业启动                                                       │
│      应用启动 ──→ 注册到 ZooKeeper ──→ 选举主节点                       │
│                                                                     │
│   2. 作业调度                                                       │
│      主节点 ──→ 读取分片 ──→ 分配分片任务                           │
│                                                                     │
│   3. 作业执行                                                       │
│      各节点 ──→ 获取分片 ──→ 执行 Job                             │
│                                                                     │
│   4. 作业完成                                                       │
│      各节点 ──→ 汇报状态 ──→ 主节点记录                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 总结

两种部署模式对比：

| 模式 | 特点 | 适用场景 |
|---|---|---|
| LiteJob | 嵌入应用、ZooKeeper 协调 | 中小规模 |
| CloudJob | Mesos 调度、资源分配 | 大规模集群 |
