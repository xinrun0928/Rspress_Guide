# Canal 应用场景

Canal 的核心价值在于「让其他系统实时感知数据库变更」。这一章我们从实际业务场景出发，讲解 Canal 的典型应用。

## 场景概览

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Canal 应用场景                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   MySQL ──→ Canal ──→ 变化数据                                      │
│                    │                                                │
│                    ├──→ Elasticsearch ──→ 全文搜索                   │
│                    │                                                │
│                    ├──→ Redis ──→ 缓存更新                          │
│                    │                                                │
│                    ├──→ 其他 MySQL ──→ 数据同步                     │
│                    │                                                │
│                    └──→ Kafka ──→ 消息队列                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 场景一：数据库实时同步

### MySQL 到 MySQL

```
MySQL Master ──→ Canal ──→ MySQL Slave
                    │
                    └──→ 实时同步数据
```

### 实现

```java
public class MySQLSyncHandler implements EntryHandler {

    @Autowired
    private JdbcTemplate jdbcTemplate;

    @Override
    public void onInsert(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            String insertSql = buildInsertSql(entry, rowData);
            jdbcTemplate.execute(insertSql);
        }
    }

    @Override
    public void onUpdate(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            String updateSql = buildUpdateSql(entry, rowData);
            jdbcTemplate.execute(updateSql);
        }
    }

    @Override
    public void onDelete(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            String deleteSql = buildDeleteSql(entry, rowData);
            jdbcTemplate.execute(deleteSql);
        }
    }

    private String buildInsertSql(Entry entry, RowData rowData) {
        String table = entry.getHeader().getTableName();
        List<Column> columns = rowData.getAfterColumnsList();

        String columnsStr = columns.stream()
            .map(Column::getName)
            .collect(Collectors.joining(", "));

        String valuesStr = columns.stream()
            .map(c -> "'" + c.getValue() + "'")
            .collect(Collectors.joining(", "));

        return "INSERT INTO " + table + " (" + columnsStr + ") VALUES (" + valuesStr + ")";
    }
}
```

## 场景二：ES 索引更新

### 架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                      MySQL → Elasticsearch 同步                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────┐       ┌─────────┐       ┌─────────┐               │
│   │  MySQL  │──────→│  Canal   │──────→│  ES     │               │
│   │         │ binlog │ Server  │       │         │               │
│   │ orders  │       │         │       │ orders  │               │
│   └─────────┘       └─────────┘       └─────────┘               │
│                                                                     │
│   订单表结构:                       ES 索引结构:                     │
│   - id (PK)                       - id                             │
│   - order_no                      - order_no                       │
│   - user_id                      - user_id                        │
│   - status                       - status                         │
│   - amount                       - amount                         │
│   - created_at                   - created_at                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 实现

```java
@Component
public class OrderEsSyncHandler {

    @Autowired
    private ElasticsearchRestTemplate esTemplate;

    @Autowired
    private OrderRepository orderRepository;

    public void onInsert(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            OrderDocument doc = toDocument(rowData.getAfterColumnsList());
            esTemplate.save(doc);
        }
    }

    public void onUpdate(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            OrderDocument doc = toDocument(rowData.getAfterColumnsList());
            esTemplate.save(doc);
        }
    }

    public void onDelete(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            Long id = getId(rowData.getBeforeColumnsList());
            esTemplate.delete(id.toString(), OrderDocument.class);
        }
    }

    private OrderDocument toDocument(List<Column> columns) {
        OrderDocument doc = new OrderDocument();
        for (Column column : columns) {
            switch (column.getName()) {
                case "id": doc.setId(column.getValue()); break;
                case "order_no": doc.setOrderNo(column.getValue()); break;
                case "user_id": doc.setUserId(Long.parseLong(column.getValue())); break;
                case "status": doc.setStatus(column.getValue()); break;
                case "amount": doc.setAmount(new BigDecimal(column.getValue())); break;
                case "created_at": doc.setCreatedAt(LocalDateTime.parse(column.getValue())); break;
            }
        }
        return doc;
    }
}
```

### 批量更新

public void onInsert(Entry entry) {
    RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());

    BulkOperations bulkOps = esTemplate.bulkOps(
        BulkOptions.DEFAULT_OPTIONS,
        OrderDocument.class

        bulkOps.addIndex(doc);
    }

}

## 场景三：缓存更新

### 架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        MySQL → Redis 缓存更新                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────┐       ┌─────────┐       ┌─────────┐               │
│   │  MySQL  │──────→│  Canal   │──────→│  Redis  │               │
│   │         │ binlog │ Server  │       │         │               │
│   │ users   │       │         │       │ users   │               │
│   └─────────┘       └─────────┘       └─────────┘               │
│                                                                     │
│   变更检测:                                                         │
│   - 用户信息变更 → 更新 Redis Hash                                  │
│   - 用户状态变更 → 更新 Redis String                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 实现

```java
@Component
public class UserCacheSyncHandler {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String CACHE_KEY_PREFIX = "user:";

    public void onInsert(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            Long userId = getColumnValue(rowData.getAfterColumnsList(), "id");
            String key = CACHE_KEY_PREFIX + userId;

            Map<String, String> userMap = toMap(rowData.getAfterColumnsList());
            redisTemplate.opsForHash().putAll(key, userMap);

            // 设置过期时间
            redisTemplate.expire(key, Duration.ofHours(24));
        }
    }

    public void onUpdate(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            Long userId = getColumnValue(rowData.getAfterColumnsList(), "id");
            String key = CACHE_KEY_PREFIX + userId;

            // 只更新变化的字段
            Map<String, String> updates = toMap(rowData.getAfterColumnsList());
            redisTemplate.opsForHash().putAll(key, updates);
        }
    }

    public void onDelete(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            Long userId = getColumnValue(rowData.getBeforeColumnsList(), "id");
            String key = CACHE_KEY_PREFIX + userId;
            redisTemplate.delete(key);
        }
    }

    private Map<String, String> toMap(List<Column> columns) {
        return columns.stream()
            .collect(Collectors.toMap(
                Column::getName,
                Column::getValue,
                (v1, v2) -> v2
            ));
    }
}
```

## 场景四：业务告警

### 异常数据监控

```java
@Component
public class AlertHandler {

    @Autowired
    private AlertService alertService;

    public void onUpdate(Entry entry) {
        RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());
        for (RowData rowData : rowChange.getRowDatasList()) {
            // 监控金额异常
            Long amount = getColumnValue(rowData.getAfterColumnsList(), "amount");
            if (amount != null && amount > 100000) {
                alertService.send(
                    "高额订单告警",
                    "订单金额超过 10 万，请确认: " + amount
                );
            }

            // 监控状态异常
            String newStatus = getColumnValue(rowData.getAfterColumnsList(), "status");
            String oldStatus = getColumnValue(rowData.getBeforeColumnsList(), "status");
            if ("cancelled".equals(newStatus) && !"pending".equals(oldStatus)) {
                alertService.send(
                    "订单状态异常",
                    "订单状态从 " + oldStatus + " 直接变为 cancelled"
                );
            }
        }
    }
}
```

## 注意事项

### 数据一致性

```java
// 补偿机制
public class DataSync {

    private Date lastSyncTime;
    private ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);

    public void init() {
        // 每小时检查一次同步状态
        scheduler.scheduleAtFixedRate(() -> {
            Date now = new Date();
            long diff = now.getTime() - lastSyncTime.getTime();

            // 如果超过 5 分钟没有同步，告警
            if (diff > 5 * 60 * 1000) {
                alertService.send("数据同步延迟", "已延迟 " + diff + " 毫秒");
            }
        }, 0, 1, TimeUnit.HOURS);
    }
}
```

## 总结

Canal 典型应用场景：

| 场景 | 说明 |
|---|---|
| 数据库同步 | MySQL 到 MySQL 的实时同步 |
| ES 索引更新 | 数据库变更实时同步到 Elasticsearch |
| 缓存更新 | 数据库变更实时同步到 Redis |
| 业务告警 | 异常数据实时监控 |
| 消息队列 | 变化数据发送到 Kafka |
