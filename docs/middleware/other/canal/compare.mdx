# Canal vs Maxwell vs Debezium

Canal、Maxwell 和 Debezium 是三个主流的 MySQL binlog 订阅组件。这一章我们对比它们的特点，帮助你选择合适的方案。

## 基本对比

| 维度 | Canal | Maxwell | Debezium |
|---|---|---|---|
| 开发语言 | Java | Java | Java |
| 维护方 | 阿里巴巴 | Zendesk | Red Hat |
| GitHub Stars | 18k+ | 5k+ | 8k+ |
| 活跃度 | 活跃 | 一般 | 活跃 |
| MySQL 支持 | 完整 | 完整 | 完整 |
| 其他数据库 | PostgreSQL | MySQL | MySQL/PostgreSQL/MongoDB/SQL Server |
| 消息格式 | JSON | JSON | JSON/Avro |
| 输出方式 | TCP | Kafka/MySQL | Kafka/其他 |

## 架构对比

### Canal 架构

```
MySQL ──→ Canal Server ──→ 客户端（TCP 推送）
              │
              └──→ Canal Admin（管理界面）
```

### Maxwell 架构

```
MySQL ──→ Maxwell Daemon ──→ Kafka（消息队列）
              │
              └──→ MySQL（存储 position）
```

### Debezium 架构

```
MySQL ──→ Debezium Connector ──→ Kafka Connect ──→ Kafka
                                        │
                                        └──→ 其他 Sink
```

**Canal:**

```json
{
  "data": [
    {"id": "1", "name": "张三"}
  ],
  "database": "order_db",
  "table": "orders",
  "type": "INSERT",

```

**Maxwell:**

```json
{
  "database": "order_db",
  "table": "orders",
  "type": "insert",
  "data": {"id": "1", "name": "张三"},
  "ts": 1234567890,
  "position": "mysql-bin.000001:12345"
```

```json
{
  "before": null,
  "after": {"id": 1, "name": "张三"},

    "db": "order_db",
    "table": "orders",
    "ts_ms": 1234567890
  },
  "op": "c"  // c=INSERT, u=UPDATE, d=DELETE, r=READ
}
```

## 使用对比

### Canal

```java
// Canal 客户端
CanalConnector connector = CanalConnectors.newSingleConnector(
    new InetSocketAddress("localhost", 11111),
    "destination", "", ""
);

connector.connect();
connector.subscribe();

while (true) {
    Message message = connector.get(100);
    // 处理消息
}
```

### Maxwell

```bash
# Maxwell 启动
./bin/maxwell --user='maxwell' --password='maxwell' \
  --host='mysql-host' \
  --producer=kafka \
  --kafka.bootstrap.servers='kafka:9092' \
  --kafka_topic=maxwell
```

### Debezium

```json title="register-mysql.json"
{
  "name": "mysql-connector",
  "config": {
    "connector.class": "io.debezium.connector.mysql.MySqlConnector",
    "database.hostname": "mysql-host",
    "database.port": "3306",
    "database.user": "debezium",
    "database.password": "password",
    "database.server.id": "184054",
    "topic.prefix": "mysql",
    "database.include.list": "order_db",
    "table.include.list": "order_db.orders"
  }
}
```

## 选择建议

### 选 Canal

- 需要本地部署，不依赖 Kafka
- 需要 TCP 推送模式
- 需要完整的 Admin 管理界面
- 已有 Canal 生态

### 选 Maxwell

- 已经使用 Kafka
- 只需要 MySQL 到 Kafka 的同步
- 需要简单的部署和运维

### 选 Debezium

- 需要多数据库支持
- 需要 Kafka Connect 生态
- 需要 Schema Registry 集成
- 需要 Debezium Cloud

## 性能对比

| 指标 | Canal | Maxwell | Debezium |
|---|---|---|---|
| 吞吐量 | 高 | 中 | 高 |
| 延迟 | 低 | 低 | 低 |
| 内存占用 | 中 | 低 | 中 |
| CPU | 中 | 低 | 中 |

## 生态集成

| 生态 | Canal | Maxwell | Debezium |
|---|---|---|---|
| Kafka | 支持（需要适配） | 原生 | 原生 |
| RocketMQ | 支持 | 不支持 | 不支持 |
| Elasticsearch | 需要适配 | 需要适配 | 需要适配 |
| 管理界面 | Admin | 无 | Control Center（商业） |
| 监控 | Prometheus | 无 | Prometheus |

## 总结

| 场景 | 推荐 |
|---|---|
| 本地 TCP 推送 | Canal |
| Kafka 集成 | Debezium / Maxwell |
| 多数据库支持 | Debezium |
| 完整管理界面 | Canal |
| 简单轻量 | Maxwell |
