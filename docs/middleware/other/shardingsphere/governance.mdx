# ShardingSphere 编排治理

ShardingSphere 的编排治理能力包括配置中心、注册中心、熔断追踪等功能，帮助实现配置的统一管理和集群的动态治理。

## 配置中心

### 支持的配置中心

| 配置中心 | 支持版本 | 说明 |
|---|---|---|
| Nacos | 1.5+ | 阿里巴巴配置中心 |
| Apollo | 1.5+ | 携程配置中心 |
| ZooKeeper | 3.4.14+ | Apache 注册中心 |
| Etcd | 3.4+ | CoreOS 配置中心 |

### Nacos 配置

```yaml title="application.yml"
spring:
  cloud:
    nacos:
      discovery:
        server-addr: nacos-server:8848
      config:
        server-addr: nacos-server:8848
        file-extension: yaml

  shardingsphere:
    # 从 Nacos 获取配置
    config-center:
      type: nacos
      server-lists: nacos-server:8848
      namespace: sharding
      group: SHARDING_SPHERE_DEFAULT_GROUP
```

### Apollo 配置

```yaml title="application.yml"
spring:
  shardingsphere:
    config-center:
      type: apollo
      server-lists: apollo-server:8080
      app-id: sharding-app
      cluster: default
      namespace: application
```

### ZooKeeper 配置

```yaml
spring:
  shardingsphere:
    config-center:
      type: zookeeper
      server-lists: zk-server:2181
      namespace: sharding
      base-sleep-time-milliseconds: 1000
      max-sleep-time-milliseconds: 3000
      max-retries: 3
```

## 注册中心

### 功能

- 服务发现：分片节点注册与注销
- 健康检查：心跳检测节点状态
- 分布式锁：分片规则变更时的协调
- 配置同步：节点间配置一致性

### ZooKeeper 注册中心

```yaml title="application.yml"
spring:
  shardingsphere:
    registry-center:
      type: zookeeper
      server-lists: zk-server:2181
      namespace: sharding
      base-sleep-time-milliseconds: 1000
      max-sleep-time-milliseconds: 3000
      max-retries: 3
      time-to-live-seconds: 30
      operation-timeout-milliseconds: 3000
```

### ZooKeeper 节点结构

```
/sharding
├── data_sources
│   ├── ds_0
│   └── ds_1
├── rules
│   └── sharding
├── schema
│   └── sharding_db
└── instance
    ├── 192.168.1.1:8888
    └── 192.168.1.2:8888
```

## 熔断追踪

### 使用 OpenTracing

```xml title="pom.xml"
<dependency>
    <groupId>io.shardingsphere</groupId>
    <artifactId>sharding-raytracing</artifactId>
    <version>${shardingsphere.version}</version>
</dependency>
```

### 配置链路追踪

```yaml title="application.yml"
spring:
  shardingsphere:
    # 启用追踪
    tracing:
      type: DBTractingWatcher
      props:
        open-watch: true
```

### 集成 SkyWalking

```xml title="pom.xml"
<dependency>
    <groupId>org.apache.skywalking</groupId>
    <artifactId>apm-toolkit-trace</artifactId>
    <version>8.12.0</version>
</dependency>
```

```yaml
spring:
  shardingsphere:
    tracing:
      type: SkyWalking
      props:
        agent-host: skywalking-oap-server
        agent-port: 11800
```

## 最佳实践

### 配置中心隔离

```yaml
spring:
  cloud:
    nacos:
      config:
        namespace: ${ENV:dev}
        group: sharding
```

不同环境使用不同的 namespace，实现配置隔离。

### 注册中心集群

```yaml
spring:
  shardingsphere:
    registry-center:
      type: zookeeper
      server-lists: |
        zk1:2181,
        zk2:2181,
        zk3:2181
      namespace: sharding
```

ZooKeeper 集群保证高可用。

### 配置热更新

```java
@Configuration
public class ShardingConfig {

    @Bean
    @ConditionalOnProperty(name = "spring.shardingsphere.config-center.type")
    public ConfigurationRepository configurationRepository(
            @Value("${spring.shardingsphere.config-center.type}") String type) {
        return ConfigurationRepositoryFactory.getRepository(type);
    }
}
```

## 总结

编排治理核心功能：

| 功能 | 说明 |
|---|---|
| 配置中心 | 统一配置管理，动态下发 |
| 注册中心 | 节点注册与发现 |
| 链路追踪 | SQL 执行链路追踪 |
| 数据脱敏 | 敏感数据加密 |
| 高可用 | 故障转移，集群选举 |
