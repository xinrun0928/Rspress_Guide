# ShardingSphere

当数据库单表数据量突破千万级、读写请求远超单机处理能力时，你需要考虑数据的水平扩展。ShardingSphere 就是解决这个问题的利器。

## 为什么需要数据分片？

想象一下，你的电商系统订单表有 1 亿条数据，每次查询都要全表扫描。索引失效、连接变慢、备份耗时……

没有分片时：

```
                    用户请求
                        │
                        ▼
              ┌─────────────────┐
              │   应用服务器      │
              └────────┬────────┘
                       │
                       ▼
              ┌─────────────────┐
              │   单机数据库     │
              │                 │
              │  订单表: 1亿条   │
              │  性能严重下降    │
              └─────────────────┘
```

分片后：

```
                    用户请求
                        │
                        ▼
              ┌─────────────────┐
              │   应用服务器      │
              │  + ShardingSphere│
              └────────┬────────┘
                       │
           ┌───────────┴───────────┐
           ▼                       ▼
   ┌───────────────┐       ┌───────────────┐
   │  数据库分片 1  │       │  数据库分片 2  │
   │  订单表: 5000万│       │  订单表: 5000万│
   └───────────────┘       └───────────────┘
```

## ShardingSphere 是什么？

ShardingSphere 是一套生态化的数据分片中间件，包含三个核心产品：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         ShardingSphere 产品                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
│  │   ShardingJDBC  │  │  ShardingProxy  │  │  ShardingSphere │    │
│  │                 │  │                 │  │      Mesh       │    │
│  │  Java 客户端    │  │  数据库代理层    │  │   服务网格      │    │
│  │  轻量级         │  │  多语言支持      │  │                │    │
│  │  无代理层        │  │  独立部署        │  │   整合 Istio   │    │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 模块学习路径

```
1. ShardingSphere 架构：Sharding-JDBC vs Sharding-Proxy
2. 数据分片原理：分片键、分片算法、分片策略
3. 读写分离：主从复制、路由策略
4. 分布式事务：XA 两阶段提交、Saga、TCC
5. 分布式 ID：雪花算法、UUID
6. 编排治理：配置中心、注册中心
```

> [!TIP]
> ShardingSphere 定位是「数据分片的生态系统」，不只是分片，还包括读写分离、分布式事务、数据加密等能力。
