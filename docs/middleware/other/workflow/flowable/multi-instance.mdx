# Flowable 会签（Multi-Instance）

会签是工作流中常见的场景——多个节点同时执行，只有全部完成后流程才继续。会签在 BPMN 中通过「多实例活动」（Multi-Instance Activity）实现。

## 什么是会签？

会签（Counter Sign）是多个审批人同时审批一个事项，只有所有人都通过才通过的审批模式。

```
普通审批：张三 → 李四 → 王五（串行，一个人处理完另一个人接手）

会签：张三 + 李四 + 王五 同时审批（三个人同时处理，全部通过才通过）
      ↓
    只有所有人都同意，才进入下一节点
```

## 会签类型

### 串行会签（Sequential）

多人按顺序依次处理：

```
开始 → ○ → ○ → ○ → 结束
      ↑   ↑   ↑
      │   │   │
      │   │   └── 第三个人
      │   └────── 第二个人
      └────────── 第一个人
```

### 并行会签（Parallel）

多人同时处理：

```
开始
  ↓
◇+
  ├─→ ○ → │
  ├─→ ○ → ├─→ ◇+ → 结束
  └─→ ○ → │
```

## BPMN 配置

### 配置多实例

```xml
<userTask id="multiApproval" name="多实例审批">
  <multiInstanceLoopCharacteristics
    isSequential="false"
    activiti:completionThreshold="1">
    <!-- 循环次数 -->
    <loopCardinality>3</loopCardinality>
  </multiInstanceLoopCharacteristics>
</userTask>
```

### 关键属性

| 属性 | 说明 |
|---|---|
| isSequential | `true`=串行，`false`=并行 |
| loopCardinality | 固定循环次数（数字） |
| loopDataInputRef | 循环数据引用（集合） |
| inputDataItem | 集合中的单个元素 |
| completionCondition | 完成条件（支持表达式） |
| activiti:completionThreshold | 完成阈值（只用于包容网关组合） |

## 串行会签示例

### 固定次数串行

```xml
<!-- 三级审批：串行执行 -->
<userTask id="threeLevelApproval" name="三级审批">
  <multiInstanceLoopCharacteristics isSequential="true">
    <loopCardinality>3</loopCardinality>
  </multiInstanceLoopCharacteristics>
</userTask>
```

### 动态串行

```xml
<!-- 审批人列表：按顺序审批 -->
<userTask id="approvalTask" name="审批">
  <multiInstanceLoopCharacteristics isSequential="true"
    loopDataInputRef="approvers"
    inputDataItem="currentApprover">
  </multiInstanceLoopCharacteristics>
</userTask>
```

```java
// 启动流程时传入审批人列表
List<String> approvers = Arrays.asList("manager", "director", "ceo");
Map<String, Object> variables = new HashMap<>();
variables.put("approvers", approvers);

ProcessInstance instance = runtimeService.startProcessInstanceByKey(
    "approval-process", variables
);
```

## 并行会签示例

### 固定次数并行

```xml
<!-- 三人会签：同时审批 -->
<userTask id="counterSign" name="会签审批">
  <multiInstanceLoopCharacteristics isSequential="false">
    <loopCardinality>3</loopCardinality>
  </multiInstanceLoopCharacteristics>
</userTask>
```

### 动态并行

```xml
<!-- 动态会签：审批人由变量决定 -->
<userTask id="counterSign" name="会签审批">
  <multiInstanceLoopCharacteristics isSequential="false"
    loopDataInputRef="approvers"
    inputDataItem="currentApprover">
    <!-- 当前审批人 -->
    <extensionElements>
      <flowable:formProperty id="approver"
        expression="${currentApprover}"
        writable="false"/>
    </extensionElements>
  </multiInstanceLoopCharacteristics>
</userTask>
```

```java
// 传入审批人列表
List<String> approvers = Arrays.asList(
    "legal-zhang",
    "finance-li",
    "business-wang"
);
variables.put("approvers", approvers);
```

### 分配处理人

```xml
<userTask id="counterSign" name="会签审批">
  <multiInstanceLoopCharacteristics isSequential="false"
    loopDataInputRef="approvers"
    inputDataItem="currentApprover">
    <extensionElements>
      <!-- 动态设置 assignee -->
      <flowable:taskListener event="create"
        class="com.example.SetApproverListener"/>
    </extensionElements>
  </multiInstanceLoopCharacteristics>
</userTask>
```

```java
public class SetApproverListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        // 获取当前审批人
        String approver = (String) delegateTask.getVariable("currentApprover");
        delegateTask.setAssignee(approver);

        // 发送通知
        notificationService.notifyApprover(approver, delegateTask.getProcessInstanceId());
    }
}
```

## 完成条件

### 默认完成条件

串行会签：所有实例完成
并行会签：所有实例完成

### 自定义完成条件

```xml
<!-- 只要有一个审批通过就继续 -->
<userTask id="counterSign" name="会签审批">
  <multiInstanceLoopCharacteristics isSequential="false"
    activiti:completionThreshold="1">
    <loopCardinality>3</loopCardinality>
  </multiInstanceLoopCharacteristics>
</userTask>
```

### 表达式完成条件

```xml
<!-- 表达式判断完成条件 -->
<userTask id="counterSign" name="会签审批">
  <multiInstanceLoopCharacteristics isSequential="false"
    completionCondition="${nrOfCompletedInstances / nrOfInstances >= 0.66}">
    <loopCardinality>3</loopCardinality>
  </multiInstanceLoopCharacteristics>
</userTask>
```

### 内置变量

| 变量 | 说明 |
|---|---|
| `nrOfInstances` | 总实例数 |
| `nrOfCompletedInstances` | 已完成实例数 |
| `nrOfActiveInstances` | 正在执行的实例数 |

## 会签结果汇总

### 查询会签状态

```java
// 查询当前会签的实例数
List<Execution> executions = runtimeService.createExecutionQuery()
    .processInstanceId(processInstanceId)
    .activityId("counterSign")
    .list();

System.out.println("当前会签实例数: " + executions.size());

// 查询已完成的数量
long completed = historyService
    .createHistoricActivityInstanceQuery()
    .processInstanceId(processInstanceId)
    .activityId("counterSign")
    .finished()
    .count();
```

### 获取会签结果

```java
// 查询所有会签任务的审批结果
List<HistoricTaskInstance> tasks = historyService
    .createHistoricTaskInstanceQuery()
    .processInstanceId(processInstanceId)
    .taskDefinitionKey("counterSign")
    .finished()
    .list();

List<String> approvers = new ArrayList<>();
int approveCount = 0;
for (HistoricTaskInstance task : tasks) {
    Boolean approved = (Boolean) task.getVariables().get("approved");
    if (Boolean.TRUE.equals(approved)) {
        approveCount++;
    }
}

// 判断是否通过（例如：超过半数通过）
boolean passed = approveCount > tasks.size() / 2;
runtimeService.setVariable(processInstanceId, "passed", passed);
```

## 子流程会签

### 嵌入式子流程会签

```xml
<subProcess id="multiSubProcess" name="会签子流程">
  <multiInstanceLoopCharacteristics isSequential="false"
    loopDataInputRef="approvers"
    inputDataItem="approver">
    <!-- 子流程内部的任务 -->
    <startEvent id="subStart"/>
    <userTask id="subApproval" name="审批 - ${approver}"/>
    <endEvent id="subEnd"/>
  </multiInstanceLoopCharacteristics>
</subProcess>
```

### 调用活动会签

```xml
<!-- 对整个子流程进行会签 -->
<callActivity id="callSubProcess" name="调用会签子流程"
  calledElement="subApprovalProcess">
  <multiInstanceLoopCharacteristics isSequential="false"
    loopDataInputRef="subProcessIds"
    inputDataItem="subProcessId"/>
</callActivity>
```

## 总结

会签（Multi-Instance）核心配置：

| 配置项 | 说明 |
|---|---|
| isSequential | `true`=串行，`false`=并行 |
| loopCardinality | 固定次数 |
| loopDataInputRef | 动态集合 |
| completionCondition | 完成条件 |
| nrOfInstances | 总实例数（内置变量） |
| nrOfCompletedInstances | 已完成数（内置变量） |
