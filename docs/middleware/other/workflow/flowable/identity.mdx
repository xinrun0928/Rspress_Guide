# Flowable 身份管理与权限控制

Flowable 提供完整的身份管理能力，支持用户、用户组、候选人等多种任务分配模式，以及基于角色的权限控制。

## 身份管理核心概念

### 用户（User）

用户是系统的操作者，每个用户有唯一的标识和基本信息。

```sql
-- 用户表结构
ACT_ID_USER
├── ID_           -- 用户ID
├── FIRST_        -- 名
├── LAST_         -- 姓
├── EMAIL_        -- 邮箱
├── PWD_          -- 密码（加密存储）
└── PICTURE_ID_   -- 头像
```

### 用户组（Group）

用户组是一组用户的集合，方便批量分配任务。

```sql
-- 用户组表结构
ACT_ID_GROUP
├── ID_           -- 组ID
├── NAME_         -- 组名
└── TYPE_         -- 类型（assignment、security-role等）
```

### 成员关系（Membership）

用户与用户组的关联关系。

```sql
-- 成员关系表
ACT_ID_MEMBERSHIP
├── USER_ID_      -- 用户ID
└── GROUP_ID_     -- 组ID
```

## IdentityService 使用

### 查询用户

```java
IdentityService identityService = engine.getIdentityService();

// 查询所有用户
List<User> users = identityService.createUserQuery().list();

// 按组查询用户
List<User> groupUsers = identityService
    .createUserQuery()
    .memberOfGroup("manager")
    .list();

// 按邮箱查询
User user = identityService.createUserQuery()
    .userEmail("zhangsan@example.com")
    .singleResult();
```

### 管理用户

```java
// 创建用户
User user = identityService.newUser("user-001");
user.setFirstName("三");
user.setLastName("张");
user.setEmail("zhangsan@example.com");
user.setPassword("encrypted-password");
identityService.saveUser(user);

// 更新用户
User existingUser = identityService.createUserQuery()
    .userId("user-001")
    .singleResult();
existingUser.setEmail("new-email@example.com");
identityService.saveUser(existingUser);

// 删除用户
identityService.deleteUser("user-001");
```

### 管理用户组

```java
// 创建用户组
Group group = identityService.newGroup("manager");
group.setName("主管组");
group.setType("assignment");
identityService.saveGroup(group);

// 添加用户到组
identityService.createMembership("user-001", "manager");

// 从组中移除用户
identityService.deleteMembership("user-001", "manager");

// 删除用户组
identityService.deleteGroup("manager");
```

### 检查用户组成员

```java
// 检查用户是否属于组
boolean isMember = identityService
    .createGroupQuery()
    .groupMember("user-001")
    .groupId("manager")
    .count() > 0;

// 获取用户所属的所有组
List<Group> userGroups = identityService
    .createGroupQuery()
    .groupMember("user-001")
    .list();
```

## 任务分配模式

### 1. 直接分配（Assignee）

直接将任务分配给指定用户：

```xml
<userTask id="managerApproval" name="主管审批"
  flowable:assignee="zhangsan"/>
```

运行时动态分配：

```java
Task task = taskService.newTask("task-001");
task.setName("主管审批");
task.setAssignee("zhangsan");  // 分配给张三
taskService.saveTask(task);
```

### 2. 候选用户（Candidate Users）

多个用户都可以处理该任务：

```xml
<userTask id="managerApproval" name="主管审批">
  <humanPerforms>
    <resourceAssignmentExpression>
      <formalExpression>user(zhangsan), user(lisi), user(wangwu)</formalExpression>
    </resourceAssignmentExpression>
  </humanPerforms>
</userTask>
```

运行时添加候选用户：

```java
Task task = taskService.newTask("task-001");
task.addCandidateUser("zhangsan");
task.addCandidateUser("lisi");
task.addCandidateUser("wangwu");
taskService.saveTask(task);
```

### 3. 候选组（Candidate Groups）

分配给用户组：

```xml
<userTask id="managerApproval" name="主管审批">
  <humanPerforms>
    <resourceAssignmentExpression>
      <formalExpression>group(manager)</formalExpression>
    </resourceAssignmentExpression>
  </humanPerforms>
</userTask>
```

运行时添加候选组：

```java
Task task = taskService.newTask("task-001");
task.addCandidateGroup("manager");
taskService.saveTask(task);
```

### 4. 组合分配

可以同时设置候选用户和候选组：

```xml
<userTask id="managerApproval" name="主管审批">
  <humanPerforms>
    <!-- 候选用户 -->
    <resourceAssignmentExpression>
      <formalExpression>user(zhangsan), user(lisi)</formalExpression>
    </resourceAssignmentExpression>
  </humanPerforms>
</userTask>
```

## 查询任务

### 查询个人任务

```java
// 查询分配给我的任务
List<Task> myTasks = taskService.createTaskQuery()
    .taskAssignee("zhangsan")
    .list();

// 查询我作为候选人的任务
List<Task> candidateTasks = taskService.createTaskQuery()
    .taskCandidateUser("zhangsan")
    .list();

// 查询我所在组的任务
List<Task> groupTasks = taskService.createTaskQuery()
    .taskCandidateGroup("manager")
    .list();
```

### 高级查询

```java
// 查询候选人或候选组的任务（并集）
List<Task> tasks = taskService.createTaskQuery()
    .taskCandidateOrGroup("zhangsan")
    .list();

// 按流程定义查询
List<Task> processTasks = taskService.createTaskQuery()
    .processDefinitionKey("leave-process")
    .taskAssignee("zhangsan")
    .list();

// 按变量查询
List<Task> tasks = taskService.createTaskQuery()
    .processDefinitionKey("leave-process")
    .taskVariableValueEquals("urgent", true)
    .list();

// 分页查询
List<Task> page = taskService.createTaskQuery()
    .taskAssignee("zhangsan")
    .orderByTaskPriority()
    .desc()
    .listPage(0, 10);
```

## 任务操作

### 签收任务

用户从候选池中认领任务：

```java
// 签收任务（张三抢了这个任务）
taskService.claim(taskId, "zhangsan");

// 签收后，其他候选人就看不到了
```

### 反签收

释放任务，回到候选池：

```java
// 反签收
taskService.unclaim(taskId);

// 或者设置 assignee 为 null
taskService.setAssignee(taskId, null);
```

### 委托任务

将任务委托给他人处理：

```java
// 委托给李四处理，处理完后回到张三
taskService.delegateTask(taskId, "lisi");

// 查询委托给我的任务
List<Task> delegatedTasks = taskService.createTaskQuery()
    .taskDelegationState(DelegationState.PENDING)
    .taskAssignee("zhangsan")
    .list();
```

### 转让任务

完全转让给其他人：

```java
// 转让给王五，不再回来
taskService.setAssignee(taskId, "wangwu");
```

## 权限控制

### 基于候选人的权限

Flowable 通过候选机制实现基本的权限控制：

```java
// 只有候选人才能签收
taskService.claim(taskId, "zhangsan");  // 如果不是候选人，会抛异常

// 只有 assignee 才能完成任务
taskService.complete(taskId);  // 如果不是 assignee，会抛异常
```

### 自定义权限检查

在实际应用中，通常需要结合业务系统做更细粒度的权限控制：

```java
@Service
public class WorkflowPermissionService {

    @Autowired
    private TaskService taskService;

    @Autowired
    private CustomPermissionService permissionService;

    public void completeTask(String taskId, String userId) {
        // 获取任务
        Task task = taskService.createTaskQuery()
            .taskId(taskId)
            .singleResult();

        // 自定义权限检查
        if (!permissionService.canCompleteTask(userId, task)) {
            throw new PermissionDeniedException("无权完成此任务");
        }

        taskService.complete(taskId);
    }
}
```

### Spring Security 集成

```java
@Service
public class WorkflowSecurityService {

    @Autowired
    private TaskService taskService;

    @Autowired
    private AuthenticationManager authManager;

    public void claimTask(String taskId) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        String currentUser = auth.getName();

        // 检查用户是否有资格签收此任务
        Task task = taskService.createTaskQuery()
            .taskId(taskId)
            .singleResult();

        boolean canClaim = taskService.createTaskQuery()
            .taskId(taskId)
            .taskCandidateUser(currentUser)
            .count() > 0;

        if (!canClaim) {
            throw new AccessDeniedException("无权签收此任务");
        }

        taskService.claim(taskId, currentUser);
    }
}
```

## 与 LDAP 集成

### 配置 LDAP

```yaml title="application.yml"
spring:
  ldap:
    urls: ldap://localhost:389
    base: dc=flowable,dc=org
    username: cn=admin
    password: admin

flowable:
  idm:
    enabled: true
    ldap:
      enabled: true
      server: ldap://localhost:389
      base-dn: dc=flowable,dc=org
      user:
        base-dn: ou=users
        filter: (objectClass=person)
        id-attribute: uid
        first-name-attribute: givenName
        last-name-attribute: sn
        email-attribute: mail
      group:
        base-dn: ou=groups
        filter: (objectClass=groupOfNames)
        name-attribute: cn
        type-attribute: businessCategory
```

### LDAP 用户同步

```java
@Configuration
public class LdapSyncConfig {

    @Bean
    public LdapUserManager ldapUserManager() {
        LdapUserManagerFactory factory = new LdapUserManagerFactory();
        // 配置同步策略
        return factory.createLdapUserManager(ldapConfig);
    }
}
```
