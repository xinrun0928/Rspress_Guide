# Flowable 数据库表结构

Flowable 使用关系数据库存储流程引擎的所有数据。了解表结构有助于排查问题、性能调优和二次开发。

## 表前缀

Flowable 使用五类表前缀：

| 前缀 | 说明 | 示例 |
|---|---|---|
| `ACT_RE_*` | Repository | 流程定义 |
| `ACT_RU_*` | Runtime | 运行时数据 |
| `ACT_HI_*` | History | 历史数据 |
| `ACT_ID_*` | Identity | 身份管理 |
| `ACT_GE_*` | General | 通用数据 |

## 核心表结构

### ACT_RE_* - Repository（仓储服务）

存储流程定义的元数据。

#### ACT_RE_DEPLOYMENT（部署表）

```sql
CREATE TABLE ACT_RE_DEPLOYMENT (
    ID_              VARCHAR(64)      -- 部署ID
    NAME_            VARCHAR(255)     -- 部署名称
    CATEGORY_        VARCHAR(255)     -- 分类
    KEY_             VARCHAR(255)     -- 业务键
    TENANT_ID_       VARCHAR(255)     -- 租户ID
    DEPLOY_TIME_     TIMESTAMP(3)     -- 部署时间
    DERIVED_FROM_    VARCHAR(64)      -- 父部署ID
    DERIVED_FROM_ROOT_ VARCHAR(64)    -- 根部署ID
    PARENT_DEPLOYMENT_ID_ VARCHAR(64) -- 父部署ID
);
```

#### ACT_RE_PROCDEF（流程定义表）

```sql
CREATE TABLE ACT_RE_PROCDEF (
    ID_              VARCHAR(64)      -- 流程定义ID（key:version）
    REV_             INT              -- 乐观锁版本
    CATEGORY_        VARCHAR(255)     -- 分类
    NAME_            VARCHAR(255)     -- 名称
    KEY_             VARCHAR(255)     -- 业务键
    VERSION_         INT              -- 版本号
    RESOURCE_NAME_   VARCHAR(4000)    -- BPMN文件名称
    DGRM_RESOURCE_NAME_ VARCHAR(4000) -- 流程图名称
    DEPLOYMENT_ID_   VARCHAR(64)      -- 部署ID
    SUSPENSION_STATE_ INT              -- 挂起状态（1=激活，2=挂起）
    TENANT_ID_       VARCHAR(255)     -- 租户ID
    ENGINE_VERSION_  VARCHAR(255)     -- 引擎版本
);
```

### ACT_RU_* - Runtime（运行时数据）

存储当前正在执行的流程实例和任务。

#### ACT_RU_EXECUTION（执行表）

```sql
CREATE TABLE ACT_RU_EXECUTION (
    ID_              VARCHAR(64)      -- 执行ID

    BUSINESS_KEY_    VARCHAR(255)     -- 业务键

    PROC_DEF_ID_     VARCHAR(64)      -- 流程定义ID
    ACT_ID_          VARCHAR(255)     -- 当前节点ID
    IS_ACTIVE_       TINYINT          -- 是否激活
    IS_CONCURRENT_   TINYINT          -- 是否并发
    IS_SCOPE_        TINYINT          -- 是否作用域
    IS_EVENT_SCOPE_  TINYINT          -- 是否事件作用域
    SUSPENSION_STATE_ INT             -- 挂起状态

    NAME_            VARCHAR(255)     -- 名称
    START_ACT_ID_   VARCHAR(255)     -- 开始节点ID
    START_TIME_      TIMESTAMP(3)     -- 开始时间
    START_USER_ID_   VARCHAR(255)     -- 启动人ID
    LOCK_TIME_       TIMESTAMP(3)     -- 锁时间
);
```

#### ACT_RU_TASK（任务表）

## 数据生命周期

### 运行时数据清理

```yaml title="application.yml"
flowable:
  history:
    # 历史数据保留天数（0 表示永久保留）
    retention-days: 180
    # 异步清理
    enable-history-cleaning: true
    history-cleaning-cycle: 86400000  # 24小时
```

### 定时清理

```java
@Component
public class HistoryCleanupScheduler {

    @Autowired
    private HistoryService historyService;

    @Scheduled(cron = "0 0 2 ? * *")  // 每天凌晨 2 点
    public void cleanupHistory() {
        HistoricProcessInstanceQuery query = historyService
            .createHistoricProcessInstanceQuery()
            .finished()
            .startedBefore(DateUtils.addDays(new Date(), -30));  // 30 天前的

        List<HistoricProcessInstance> oldInstances = query.list();

        for (HistoricProcessInstance instance : oldInstances) {
            historyService.deleteHistoricProcessInstance(instance.getId());
        }
    }
}
```

## 总结

表前缀速查：

| 前缀 | 说明 | 生命周期 |
|---|---|---|
| `ACT_RE_*` | Repository | 永久 |
| `ACT_RU_*` | Runtime | 流程结束时删除 |
| `ACT_HI_*` | History | 可配置清理策略 |
| `ACT_ID_*` | Identity | 永久 |
| `ACT_GE_*` | General | 永久 |
