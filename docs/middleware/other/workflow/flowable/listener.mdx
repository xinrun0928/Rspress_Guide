# Flowable 事件监听

Flowable 的事件监听机制让你能够在流程执行的关键节点插入自定义逻辑，实现日志记录、外部通知、数据同步等功能。

## 事件类型

Flowable 事件分为两大类：执行监听器和任务监听器。

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Flowable 事件分类                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   执行监听器（ExecutionListener）                                    │
│   ├── 流程启动/结束                                                  │
│   ├── 节点进入/离开                                                  │
│   ├── 网关分支/合并                                                  │
│   └── 变量变化                                                      │
│                                                                     │
│   任务监听器（TaskListener）                                        │
│   ├── 任务创建                                                      │
│   ├── 任务分配                                                      │
│   ├── 任务完成                                                      │
│   └── 任务删除                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 执行监听器（ExecutionListener）

### 监听事件类型

| 事件 | 触发时机 |
|---|---|
| start | 流程实例启动 |
| end | 流程实例结束 |
| take | 顺序流被执行 |
| transition | 节点间流转（Flowable 6.x 新增） |

### BPMN 配置

```xml
<!-- 监听整个流程 -->
<process id="leaveProcess"
  flowable:eventListener="com.example.MyProcessListener">

  <!-- 监听特定节点 -->
  <userTask id="managerApproval" name="主管审批">
    <extensionElements>
      <flowable:executionListener
        event="start"
        class="com.example.ApprovalStartListener"/>
      <flowable:executionListener
        event="end"
        class="com.example.ApprovalEndListener"/>
    </extensionElements>
  </userTask>
</process>
```

### 监听器实现

```java
public class ApprovalStartListener implements ExecutionListener {

    @Override
    public void notify(DelegateExecution execution) {
        String eventName = execution.getEventName();
        String activityId = execution.getCurrentActivityId();

        System.out.println("节点开始: " + activityId);
        System.out.println("事件类型: " + eventName);
        System.out.println("流程实例: " + execution.getProcessInstanceId());

        // 发送通知
        String assignee = (String) execution.getVariable("assignee");
        notificationService.notify(assignee, "您有新任务待处理");
    }
}
```

### 监听器获取事件类型

```java
public class MyListener implements ExecutionListener {

    @Override
    public void notify(DelegateExecution execution) {
        // 获取事件名称
        String eventName = execution.getEventName();

        if ("start".equals(eventName)) {
            // 节点开始时的逻辑
        } else if ("end".equals(eventName)) {
            // 节点结束时的逻辑
        } else if ("take".equals(eventName)) {
            // 顺序流执行时的逻辑
        }
    }
}
```

## 任务监听器（TaskListener）

### 监听事件类型

| 事件 | 触发时机 |
|---|---|
| create | 任务创建时 |
| assignment | 任务分配时 |
| complete | 任务完成时 |
| delete | 任务删除时 |
| all | 所有事件 |

### BPMN 配置

```xml
<userTask id="managerApproval" name="主管审批">
  <extensionElements>
    <flowable:taskListener event="create"
      class="com.example.TaskCreateListener"/>
    <flowable:taskListener event="assignment"
      class="com.example.TaskAssignmentListener"/>
    <flowable:taskListener event="complete"
      class="com.example.TaskCompleteListener"/>
  </extensionElements>
</userTask>
```

### 监听器实现

```java
// 任务创建监听器
public class TaskCreateListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        String taskName = delegateTask.getName();
        String processInstanceId = delegateTask.getProcessInstanceId();

        // 记录日志
        auditService.log("TASK_CREATED",
            processInstanceId,
            delegateTask.getId(),
            delegateTask.getAssignee()
        );

        // 设置优先级
        Integer days = (Integer) delegateTask.getVariable("days");
        if (days > 7) {
            delegateTask.setPriority(50);  // 高优先级
        } else {
            delegateTask.setPriority(10);  // 普通优先级
        }

        // 设置到期时间
        Date dueDate = DateUtils.addDays(new Date(), 3);
        delegateTask.setDueDate(dueDate);
    }
}

// 任务分配监听器
public class TaskAssignmentListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        String assignee = delegateTask.getAssignee();
        String taskName = delegateTask.getName();

        // 发送通知
        emailService.send(
            to: assignee,
            subject: "任务分配: " + taskName,
            body: "您有一个新任务: " + taskName
        );
    }
}

// 任务完成监听器
public class TaskCompleteListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        // 获取任务结果
        Boolean approved = (Boolean) delegateTask.getVariable("approved");
        String comment = (String) delegateTask.getVariable("comment");

        // 记录审计日志
        auditService.recordApproval(
            delegateTask.getProcessInstanceId(),
            delegateTask.getAssignee(),
            approved,
            comment
        );
    }
}
```

## 委托式监听器（Delegate Expression）

除了直接指定类名，还可以使用委托表达式，运行时动态解析：

```xml
<userTask id="managerApproval" name="主管审批">
  <extensionElements>
    <!-- 使用委托表达式 -->
    <flowable:taskListener event="create"
      delegateExpression="${taskNotificationListener}"/>
  </extensionElements>
</userTask>
```

```java
@Component("taskNotificationListener")
public class TaskNotificationListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        // 通知逻辑
    }
}
```

## 监听器与流程变量

### 在监听器中访问变量

```java
public class ApprovalListener implements ExecutionListener {

    @Override
    public void notify(DelegateExecution execution) {
        // 获取变量
        String employee = (String) execution.getVariable("employee");
        Integer days = (Integer) execution.getVariable("days");

        // 设置变量
        execution.setVariable("listenerProcessed", true);

        // 设置局部变量
        execution.setVariableLocal("listenerLocal", "局部变量");
    }
}
```

### 变量变化监听

```java
public class VariableListener implements ExecutionListener {

    @Override
    public void notify(DelegateExecution execution) {
        // 获取变化前的值（如果实现了 HistoricVariableListener）
        // 可以通过 HistoryService 查询
    }
}
```

## 全局监听器

### 配置方式

在流程引擎配置中添加全局监听器：

```java
ProcessEngineConfiguration config = processEngine.getProcessEngineConfiguration();

List<ExecutionListener> listeners = new ArrayList<>();
listeners.add(new GlobalStartListener());
listeners.add(new GlobalEndListener());

config.setExecutionListeners(listeners);
```

### Spring Boot 配置

```yaml title="application.yml"
flowable:
  process:
    execution-listeners:
      - class: com.example.GlobalStartListener
        event: start
      - class: com.example.GlobalEndListener
        event: end
```

## 事件顺序

一个典型的任务执行，事件触发顺序：

```
1. ExecutionListener: transition (进入节点)
2. TaskListener: create (任务创建)
3. TaskListener: assignment (任务分配)
4. ... 用户处理任务 ...
5. TaskListener: complete (任务完成)
6. ExecutionListener: transition (离开节点)
7. ExecutionListener: start (下一个节点开始)
```

## 异步监听器

监听器默认同步执行，如果逻辑耗时较长，可以配置为异步：

```xml
<userTask id="managerApproval" name="主管审批">
  <extensionElements>
    <flowable:taskListener event="complete"
      delegateExpression="${asyncListener}"
      flowable:async="true"/>
  </extensionElements>
</userTask>
```

```java
@Component("asyncListener")
public class AsyncTaskListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        // 这个方法会在异步线程池中执行
        // 适合发送邮件、调用外部 API 等耗时操作
    }
}
```

## 总结

事件监听器类型对比：

| 类型 | 监听对象 | 常用事件 |
|---|---|---|
| ExecutionListener | 流程实例、节点、网关 | start、end、take |
| TaskListener | 用户任务 | create、assignment、complete、delete |
