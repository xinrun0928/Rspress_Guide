# Flowable 子流程

子流程是 BPMN 中将复杂流程分解为可复用单元的重要手段。Flowable 支持两种子流程：嵌入式子流程和调用活动。

## 子流程的价值

想象一个请假流程：

```
请假申请 → 主管审批 → [天数>3?] → HR审批 → 财务审批 → 通知
                           ↓
                         直接通过
```

其中「多级审批」这个逻辑可能在多个流程中复用。如果没有子流程，我们需要重复定义这个逻辑。

```
子流程的价值：
1. 代码复用：避免重复定义相同的流程逻辑
2. 关注点分离：每个子流程关注一个具体业务
3. 可维护性：修改子流程，所有引用的地方都生效
```

## 嵌入式子流程（Embedded SubProcess）

嵌入式子流程是完全嵌入在父流程中的子流程，共享同一个执行上下文。

### 基本语法

```xml
<subProcess id="embeddedSubProcess">
  <!-- 子流程内部定义 -->
  <startEvent id="subStart"/>
  <userTask id="subTask1" name="子任务1"/>
  <userTask id="subTask2" name="子任务2"/>
  <endEvent id="subEnd"/>
</subProcess>
```

### 完整示例

```xml title="leave-process.bpmn"
<process id="leaveProcess">
  <startEvent id="start"/>

  <!-- 主流程任务 -->
  <userTask id="submitLeave" name="提交请假"/>

  <!-- 嵌入式子流程：审批流程 -->
  <subProcess id="approvalSubProcess">
    <startEvent id="approvalStart"/>
    <userTask id="managerApproval" name="主管审批"/>
    <userTask id="hrApproval" name="HR审批"/>
    <endEvent id="approvalEnd"/>
  </subProcess>

  <!-- 主流程任务 -->
  <serviceTask id="notifyTask" name="发送通知"/>
  <endEvent id="end"/>

  <!-- 顺序流 -->
  <sequenceFlow sourceRef="start" targetRef="submitLeave"/>
  <sequenceFlow sourceRef="submitLeave" targetRef="approvalSubProcess"/>
  <sequenceFlow sourceRef="approvalSubProcess" targetRef="notifyTask"/>
  <sequenceFlow sourceRef="notifyTask" targetRef="end"/>
</process>
```

### 执行流程

```
开始 → 提交请假 → ╔═══════════════════╗ → 发送通知 → 结束
                        ║  子流程开始          ║
                        ║  主管审批 → HR审批  ║
                        ║  子流程结束          ║
                        ╚═══════════════════╝
```

## 事件子流程（Event SubProcess）

事件子流程由事件触发（错误、信号、消息），不会在主流程中直接调用。

### 错误事件子流程

常用于捕获子流程中的错误：

```xml
<subProcess id="errorHandling" triggeredByEvent="true">
  <startEvent id="errorStart">
    <errorEventDefinition errorRef="ns:validationError"/>
  </startEvent>
  <serviceTask id="handleError" name="错误处理"/>
  <endEvent id="errorEnd"/>
</subProcess>
```

### 信号事件子流程

```xml
<subProcess id="signalHandling" triggeredByEvent="true">
  <startEvent id="signalStart">
    <signalEventDefinition signalRef="ns:pauseSignal"/>
  </startEvent>
  <serviceTask id="pauseProcess" name="暂停处理"/>
  <endEvent id="signalEnd"/>
</subProcess>
```

### 使用场景

| 类型 | 触发方式 | 场景 |
|---|---|---|
| 错误事件子流程 | 错误事件 | 异常处理、补偿 |
| 信号事件子流程 | 信号事件 | 全局暂停、通知 |
| 消息事件子流程 | 消息事件 | 外部响应处理 |

## 调用活动（Call Activity）

调用活动是调用**独立流程**的方式，被调用的流程有自己独立的执行上下文。

### 基本语法

```xml
<!-- 调用请假审批子流程 -->
<callActivity id="callLeaveApproval" name="调用请假审批流程">
  <calledElement>leave-approval-process</calledElement>
</callActivity>
```

### 传递数据

```xml
<callActivity id="callLeaveApproval" name="调用请假审批流程">
  <calledElement>leave-approval-process</calledElement>

  <!-- 输入映射：将父流程变量传递给子流程 -->
  <in>
    <extensionElements>
      <flowable:in source="employeeId" target="employeeId"/>
      <flowable:in source="days" target="days"/>
    </extensionElements>
  </in>

  <!-- 输出映射：将子流程变量传递给父流程 -->
  <out>
    <extensionElements>
      <flowable:out source="approved" target="leaveApproved"/>
      <flowable:out source="approvedBy" target="lastApprover"/>
    </extensionElements>
  </out>
</callActivity>
```

### 表达式映射

```xml
<!-- 使用表达式传递 -->
<callActivity id="callLeaveApproval">
  <calledElement>leave-approval-process</calledElement>

  <in>
    <flowable:in sourceExpression="${employeeService.getId()}"
      target="employeeId"/>
  </in>

  <out>
    <flowable:out sourceExpression="${approved ? 'PASS' : 'REJECT'}"
      target="result"/>
  </out>
</callActivity>
```

## 嵌入式 vs 调用活动

| 维度 | 嵌入式子流程 | 调用活动 |
|---|---|---|
| 执行上下文 | 共享父流程上下文 | 独立执行上下文 |
| 流程定义 | 内嵌在父流程中 | 独立部署 |
| 版本管理 | 随父流程版本 | 可以独立升级 |
| 数据传递 | 直接访问父流程变量 | 需要输入/输出映射 |
| 复用性 | 只能在一个流程中复用 | 可被多个流程调用 |
| BPMN 结构 | 必须是 SubProcess 元素 | 可以是任何流程 |

### 选择建议

```
需要复用？           是 → 调用活动
                    │
                    └── 独立部署、独立升级？ 是 → 调用活动
                                                │
                                                └── 否 → 嵌入式子流程
           │
           └── 否 → 嵌入式子流程
```

## 事务子流程（Transaction SubProcess）

事务子流程保证一组活动的原子性——要么全部成功，要么全部回滚。

### 基本语法

```xml
<subProcess id="transactionSubProcess">
  <transaction id="transaction1">
    <startEvent id="transStart"/>
    <userTask id="task1" name="任务1"/>
    <userTask id="task2" name="任务2"/>
    <endEvent id="transEnd"/>
  </transaction>
</subProcess>
```

### 事务边界行为

| 场景 | 行为 |
|---|---|
| 所有任务完成 | 事务提交，流程继续 |
| 外部补偿事件触发 | 执行补偿（Compensation） |
| 引擎异常/取消事件 | 执行补偿，事务取消 |

### 补偿处理

```xml
<subProcess id="orderTransaction">
  <transaction id="orderTransaction">
    <serviceTask id="createOrder" name="创建订单"
      flowable:class="com.example.CreateOrderTask"/>
    <serviceTask id="deductInventory" name="扣减库存"
      flowable:class="com.example.DeductInventoryTask"/>

    <!-- 补偿事件 -->
    <boundaryEvent id="compensate">
      <compensateEventDefinition/>
    </boundaryEvent>

    <!-- 补偿处理（反向操作） -->
    <serviceTask id="cancelOrder" name="取消订单"
      flowable:isForCompensation="true"
      flowable:class="com.example.CancelOrderTask"/>
    <serviceTask id="restoreInventory" name="恢复库存"
      flowable:isForCompensation="true"
      flowable:class="com.example.RestoreInventoryTask"/>

    <endEvent id="transEnd"/>
  </transaction>
</subProcess>
```

## 子流程中的变量

### 嵌入式子流程变量

```java
// 嵌入式子流程可以直接访问父流程变量
public class SubProcessDelegate implements JavaDelegate {
    @Override
    public void execute(DelegateExecution execution) {
        // 读取父流程变量
        String employeeId = (String) execution.getVariable("employeeId");

        // 设置变量（父流程可见）
        execution.setVariable("subResult", "子流程完成");
    }
}
```

### 调用活动变量

```java
// 方式一：输入/输出映射（BPMN 中配置）

// 方式二：手动设置（运行时）
ProcessInstance subProcess = runtimeService.createProcessInstanceByKey(
    "leave-approval-process",
    processInstanceId  // 父流程实例 ID，用于数据关联
);

// 通过监听器设置
public class CallActivityListener implements ExecutionListener {

    @Override
    public void notify(DelegateExecution execution) {
        // 设置输入变量
        runtimeService.setVariable(execution.getId(), "employeeId",
            execution.getVariable("employeeId"));

        // 子流程完成后，获取输出变量
        Object result = execution.getVariable("leaveApproved");
    }
}
```

## 总结

子流程类型对比：

| 类型 | 关键字 | 特点 | 适用场景 |
|---|---|---|---|
| 嵌入式子流程 | `<subProcess>` | 共享上下文 | 局部复用 |
| 事件子流程 | `<subProcess triggeredByEvent>` | 事件触发 | 异常处理 |
| 调用活动 | `<callActivity>` | 独立上下文 | 全局复用、独立部署 |
| 事务子流程 | `<transaction>` | 原子性保证 | 需补偿的业务 |
