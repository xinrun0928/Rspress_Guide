# Flowable 架构

Flowable 的核心是「引擎」——每个引擎负责处理一种类型的流程模型。这种设计让 Flowable 不仅仅是一个 BPMN 引擎，而是一个完整的流程平台。

## 六大核心引擎

Flowable 6.x 版本包含六个核心引擎：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Flowable 六大引擎                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌───────────────┐    ┌───────────────┐    ┌───────────────┐     │
│   │   Process     │    │    Case       │    │    Form       │     │
│   │   Engine      │    │   Engine      │    │   Engine      │     │
│   │   BPMN 2.0    │    │   CMMN 1.1    │    │               │     │
│   │               │    │               │    │               │     │
│   │ 业务流程建模   │    │ 案例任务管理   │    │  表单引擎     │     │
│   └───────────────┘    └───────────────┘    └───────────────┘     │
│                                                                     │
│   ┌───────────────┐    ┌───────────────┐    ┌───────────────┐     │
│   │     DMN       │    │   Content     │    │     App       │     │
│   │   Engine      │    │   Engine      │    │   Engine      │     │
│   │               │    │               │    │               │     │
│   │  决策表管理    │    │  文件附件管理  │    │   Web 应用    │     │
│   └───────────────┘    └───────────────┘    └───────────────┘     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Process Engine（BPMN 引擎）

Process Engine 是 Flowable 的核心，负责处理 BPMN 2.0 规范的流程。

**支持的元素：**
- 事件：开始、结束、中间事件
- 活动：用户任务、服务任务、脚本任务、子流程
- 网关：排他、并行、包容、事件
- 流向：顺序流、消息流

### Case Engine（CMMN 引擎）

CMMN（Case Management Model and Notation）是案例管理的建模标准，适合非结构化、探索性的业务流程。

**典型场景：**
- 客服工单处理
- 事故调查流程
- 客户服务支持

### Form Engine（表单引擎）

Form Engine 管理动态表单，用于用户任务的数据收集。

**表单类型：**
- 动态表单：定义表单属性，运行时渲染
- 外置表单：表单模板文件，运行时加载
- 表单键：引用外部表单定义

### DMN Engine（决策引擎）

DMN（Decision Model and Notation）是决策管理的建模标准。

**典型场景：**
- 贷款审批规则
- 保险理赔规则
- 价格计算规则

### Content Engine（内容引擎）

Content Engine 管理流程中的文件和附件。

**功能：**
- 文件上传和下载
- 文件版本管理
- 附件与流程实例关联

### App Engine（应用引擎）

App Engine 提供 Flowable 的 Web 应用套件。

**内置应用：**
- Flowable Modeler：流程/表单/决策表设计器
- Flowable IDM：身份管理
- Flowable Task：任务管理
- Flowable Admin：系统管理

## 核心服务

Flowable 引擎提供多个服务接口，每个服务负责特定的功能领域：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Flowable 核心服务                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   RepositoryService      ──→  流程定义管理                            │
│   RuntimeService         ──→  流程实例运行时                          │
│   TaskService            ──→  用户任务管理                            │
│   HistoryService         ──→  历史数据查询                            │
│   IdentityService        ──→  用户和组管理                            │
│   ManagementService     ──→  定时器和作业管理                         │
│   DynamicBpmnService    ──→  动态流程修改                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### RepositoryService

管理流程定义的生命周期：

```java
// 部署流程
Deployment deployment = repositoryService.createDeployment()
    .addClasspathResource("leave-process.bpmn")
    .name("请假流程")
    .deploy();

// 查询流程定义
ProcessDefinition processDefinition = repositoryService
    .createProcessDefinitionQuery()
    .processDefinitionKey("leave-process")
    .latestVersion()
    .singleResult();

// 获取流程定义 XML
String bpmnXml = repositoryService.getResourceAsString(
    processDefinition.getDeploymentId(),
    processDefinition.getResourceName()
);
```

### RuntimeService

管理流程实例的运行：

```java
// 启动流程实例
ProcessInstance processInstance = runtimeService
    .startProcessInstanceByKey("leave-process",
        Variables.createVariables()
            .putValue("employee", "张三")
            .putValue("days", 5)
    );

// 查询运行中的流程实例
List<ProcessInstance> instances = runtimeService
    .createProcessInstanceQuery()
    .processDefinitionKey("leave-process")
    .active()
    .list();

// 触发信号事件
runtimeService.signalEventReceived("continueSignal");

// 触发消息事件
runtimeService.messageEventReceived("orderMessage", processInstanceId);
```

### TaskService

管理用户任务：

```java
// 查询待办任务
List<Task> tasks = taskService.createTaskQuery()
    .taskAssignee("zhangsan")
    .list();

// 完成任务
taskService.complete(taskId,
    Variables.createVariables()
        .putValue("approved", true)
        .putValue("comment", "同意请假")
);

// 添加评论
taskService.addComment(taskId, processInstanceId, "approved", "同意，祝旅途愉快！");
```

### HistoryService

查询历史数据：

```java
// 查询已完成的流程实例
List<HistoricProcessInstance> completedProcesses = historyService
    .createHistoricProcessInstanceQuery()
    .processDefinitionKey("leave-process")
    .finished()
    .list();

// 查询流程实例的执行路径
List<HistoricActivityInstance> activities = historyService
    .createHistoricActivityInstanceQuery()
    .processInstanceId(processInstanceId)
    .orderByHistoricActivityInstanceStartTime()
    .asc()
    .list();
```

## 引擎配置

### 流程引擎配置

```java
ProcessEngineConfiguration config = StandaloneProcessEngineConfiguration
    .createProcessEngineConfigurationFromResource("flowable.cfg.xml");
    // jdbcUrl
    // jdbcDriver
    // jdbcUsername
    // jdbcPassword
    // databaseSchemaUpdate
    // asyncExecutorEnabled
    // asyncExecutorActivate
    // jobExecutorActivate
    ;

ProcessEngine engine = config.buildProcessEngine();
```

### Spring Boot 自动配置

```xml title="pom.xml"
<dependency>
    <groupId>org.flowable</groupId>
    <artifactId>flowable-spring-boot-starter</artifactId>
</dependency>
```

```yaml title="application.yml"
spring:
  datasource:
    url: jdbc:h2:mem:flowable
  flowable:
    async-executor-enabled: true
    async-executor-activate: true
    .check-process-definitions: true
    database-schema-update: true
    history-level: full
```

## 总结

Flowable 六大引擎与核心服务的对应关系：

| 引擎 | 核心服务 | 处理内容 |
|---|---|---|
| Process Engine | RuntimeService, TaskService | BPMN 2.0 流程 |
| Case Engine | CaseService | CMMN 1.1 案例 |
| Form Engine | FormService | 动态表单 |
| DMN Engine | DecisionService | DMN 决策表 |
| Content Engine | ContentService | 文件附件 |
| App Engine | AppService | Web 应用 |
