# Flowable 与 Activiti 的历史渊源

Flowable 和 Activiti 的关系，可以用一个词概括：**同源分流**。它们有着共同的血脉，却在发展中走出了不同的道路。

## 历史背景

### Activiti 的起源

2007 年，Tom Baeyens 离开 JBoss，创建了 Activiti 项目。Activiti 5.x 版本成为 Java 领域最流行的开源工作流引擎。

**关键人物：**
- Tom Baeyens：Activiti 创始人
- Joram Barrez：Activiti 联合创始人

### Flowable 的诞生

2016 年，由于对 Activiti 发展方向的分歧，Activiti 核心团队决定另起炉灶，成立了 Flowable 项目。

**分裂原因（据社区传闻）：**
- 对开源版本功能的限制
- 对云原生方向的不同理解
- 对商业化路线的分歧

```
Activiti 5.x（2007 年）
        │
        ├──→ Activiti 6.x（2016 年，Alfresco 主导）
        │         │
        │         └──→ 项目停滞，核心团队出走
        │
        └──→ Flowable 6.x（2016 年，核心团队创立）
                  │
                  └──→ Flowable 7.x（云原生方向）
```

## 版本对照

| 时间 | Activiti | Flowable |
|---|---|---|
| 2007 年 | Activiti 5.x 发布 | - |
| 2016 年 | Activiti 6.x 发布 | Flowable 6.x 发布 |
| 2019 年 | Activiti 7.x（云化） | Flowable 6.4（稳定版） |
| 2020 年 | - | Flowable 6.5 - 6.7 |
| 2021 年 | - | Flowable 7.x（云原生） |

## 核心区别

### 架构设计

| 维度 | Flowable | Activiti |
|---|---|---|
| 引擎数量 | 6 个（BPMN、CMMN、DMN、Form、Content、App） | 1 个（核心 BPMN） |
| 架构模式 | 微服务友好 | 单体应用 |
| Spring Boot | 原生支持 | 需要手动配置 |
| REST API | 内置 | 需要额外集成 |

### 功能特性

| 特性 | Flowable | Activiti |
|---|---|---|
| CMMN 支持 | 支持 | 不支持 |
| DMN 支持 | 支持 | 需要额外模块 |
| 表单引擎 | 内置 | 需要额外模块 |
| 内容引擎 | 内置 | 需要额外模块 |
| Web 应用 | 完整套件（IDM、Task、Admin、Modeler） | 基础 |

### 社区活跃度

| 维度 | Flowable | Activiti |
|---|---|---|
| GitHub Stars | 10k+ | 5k+ |
| 社区活跃度 | 活跃 | 较低 |
| 文档完善度 | 完善 | 一般 |
| 问题响应 | 较快 | 较慢 |

## 为什么 Flowable 更受欢迎？

### 1. 功能完整性

Flowable 一次性提供了六大引擎，而 Activiti 需要额外集成多个模块：

```xml title="Flowable 一站式依赖"
<dependency>
    <groupId>org.flowable</groupId>
    <artifactId>flowable-spring-boot-starter</artifactId>
    <!-- 内置所有引擎 -->
</dependency>

<!-- Activiti 需要多个依赖 -->
<dependency>
    <groupId>org.activiti</groupId>
    <artifactId>activiti-spring-boot-starter</artifactId>
</dependency>
<dependency>
    <groupId>org.activiti</groupId>
    <artifactId>activiti-form-spring-boot-starter</artifactId>
</dependency>
<dependency>
    <groupId>org.activiti</groupId>
    <artifactId>activiti-dmn-spring-boot-starter</artifactId>
</dependency>
```

### 2. Spring Boot 集成

Flowable 对 Spring Boot 的集成更加友好：

```yaml title="Flowable 只需几行配置"
spring:
  datasource:
    url: jdbc:h2:mem:flowable
  flowable:
    async-executor-enabled: true
    database-schema-update: true
```

Activiti 7.x 之前的版本配置相对复杂。

### 3. 发展路线

Flowable 坚持开源优先，核心功能都在开源版本中。而 Activiti 被 Alfresco 收购后，部分高级功能被移到商业版本。

## 迁移建议

### 从 Activiti 迁移到 Flowable

如果你有遗留的 Activiti 项目，可以考虑迁移到 Flowable：

**迁移步骤：**

1. **评估依赖**
   - 检查 Activiti 版本
   - 确定使用的模块

2. **修改依赖**
   ```xml
   <!-- 替换依赖 -->
   <dependency>
       <groupId>org.flowable</groupId>
       <artifactId>flowable-spring-boot-starter</artifactId>
   </dependency>
   ```

3. **修改 API 调用**
   - `RuntimeService.startProcessInstanceByKey()` → 保持不变
   - `TaskService.complete()` → 保持不变
   - 包名从 `org.activiti` 改为 `org.flowable`

4. **测试验证**
   - 验证流程定义兼容
   - 验证 API 调用结果
   - 验证历史数据迁移

### BPMN 兼容性

Flowable 对 Activiti 的 BPMN 文件有很好的兼容性：

```xml title="Activiti 格式（兼容）"
<activiti:formProperty id="employee" name="员工"
  expression="${employee}"/>

<!-- Flowable 格式（等价） -->
<flowable:formProperty id="employee" name="员工"
  expression="${employee}"/>
```

## 总结

Flowable 与 Activiti 的核心差异：

| 维度 | Flowable | Activiti |
|---|---|---|
| 引擎完整性 | 六大引擎，开箱即用 | 需要额外集成 |
| Spring Boot | 原生友好 | 一般 |
| 社区活跃度 | 活跃 | 较低 |
| 开源策略 | 核心功能开源 | 部分功能商业化 |
| 发展方向 | 稳定迭代 + 云原生 | 云原生优先 |
