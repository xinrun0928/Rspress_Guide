# Flowable 异步执行

Flowable 的异步执行机制让你能够将耗时操作从主流程中剥离出来，避免阻塞流程执行。这在发送邮件、调用外部系统等场景下特别有用。

## 为什么要异步执行？

同步执行的问题：

```
用户提交申请 → 发送邮件（等待 2 秒）→ 记录日志（等待 1 秒）→ 完成
               ↑
           用户等待 3 秒

异步执行的优势：

用户提交申请 → ╔═══════════════════╗ → 完成（用户无感知）
                ║ 后台异步执行：      ║
                ║ - 发送邮件（2秒）  ║
                ║ - 记录日志（1秒）  ║
                ╚═══════════════════╝
```

## 异步执行器（AsyncExecutor）

Flowable 使用 AsyncExecutor 管理异步作业的执行：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         AsyncExecutor 架构                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────┐                                                  │
│   │  流程引擎    │                                                  │
│   └──────┬──────┘                                                  │
│          │ 创建异步作业                                              │
│          ↓                                                          │
│   ┌─────────────┐      ┌─────────────┐                             │
│   │  作业表      │ ←──→ │  作业执行器   │                             │
│   │ ACT_RU_JOB   │      │ AsyncJobHandler│                            │
│   │ ACT_RU_TIMER │      └──────┬──────┘                             │
│   └─────────────┘             │ 执行作业                            │
│                               ↓                                     │
│                    ┌─────────────────────┐                          │
│                    │   线程池执行器       │                          │
│                    │  (默认 8 个线程)     │                          │
│                    └─────────────────────┘                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 说明 |
|---|---|
| JobExecutor | 同步作业执行器（已废弃，不推荐） |
| AsyncExecutor | 异步作业执行器（推荐） |
| Job | 异步作业（立即执行） |
| TimerJob | 定时作业（延迟执行） |
| SuspendedJob | 暂停的作业 |
| DeadLetterJob | 死信作业（执行失败后移入） |

## 配置异步执行

### Spring Boot 配置

```yaml title="application.yml"
flowable:
  async:
    # 启用异步执行器
    executor-enabled: true
    # 激活异步执行器
    executor-activate: true
    # 异步核心线程数
    core-pool-size: 8
    # 异步最大线程数
    max-pool-size: 16
    # 队列容量
    queue-size: 100
    # 获取作业等待时间（秒）
    wait-time: 10
    # 重试次数
    retry-cycle-count: 3
```

### Java 配置

```java
@Bean
public ProcessEngine processEngine(ProcessEngineConfigurationImpl config) {
    config.setAsyncExecutorEnabled(true);
    config.setAsyncExecutorActivate(true);

    // 配置线程池
    AsyncExecutor asyncExecutor = new DefaultAsyncJobExecutor();
    ((DefaultAsyncJobExecutor) asyncExecutor).setCorePoolSize(8);
    ((DefaultAsyncJobExecutor) asyncExecutor).setMaxPoolSize(16);
    ((DefaultAsyncJobExecutor) asyncExecutor).setQueueSize(100);

    config.setAsyncExecutor(asyncExecutor);

    return config.buildProcessEngine();
}
```

## 异步作业标记

### 服务任务异步

```xml
<!-- 服务任务异步执行 -->
<serviceTask id="sendNotification"
  name="发送通知"
  flowable:async="true"
  flowable:class="com.example.SendNotificationTask"/>
```

### 脚本任务异步

```xml
<serviceTask id="calculateTask"
  name="计算"
  flowable:async="true"
  flowable:expression="${calculator.calculate(execution)}"/>
```

### 发送信号事件异步

```xml
<sendTask id="sendTask"
  flowable:async="true"
  flowable:expression="${mailService.send()}"/>
```

### BPMN 触发异步

```xml
<!-- 整个流程异步执行 -->
<process id="asyncProcess"
  flowable:async="true">

  <!-- 或者节点级别 -->
  <userTask id="task1" flowable:async="true"/>
</process>
```

## 定时器事件

### 定时开始事件

流程定时启动：

```xml
<!-- 每天凌晨 2 点执行 -->
<startEvent id="scheduledStart">
  <timerEventDefinition>
    <timeCycle>0 0 2 ? * *</timeCycle>
  </timerEventDefinition>
</startEvent>
```

### 边界定时器

任务超时处理：

```xml
<userTask id="approvalTask" name="主管审批">
  <!-- 24 小时超时 -->
  <boundaryEvent id="timeout">
    <timerEventDefinition>
      <timeDuration>PT24H</timeDuration>
    </timerEventDefinition>
  </boundaryEvent>

  <!-- 超时后的处理 -->
  <sequenceFlow sourceRef="timeout" targetRef="autoReject"/>
</userTask>
```

### 中间定时事件

等待一段时间后继续：

```xml
<!-- 等待 30 分钟后重试 -->
<intermediateTimerEventDefinition>
  <timeDuration>PT30M</timeDuration>
</intermediateTimerEventDefinition>
```

### Cron 表达式

Flowable 支持标准 Cron 表达式：

| 表达式 | 说明 |
|---|---|
| `0 0 * * * ?` | 每小时整点 |
| `0 0 2 ? * *` | 每天凌晨 2 点 |
| `0 0/15 * * ?` | 每 15 分钟 |
| `0 0 2 ? * MON-FRI` | 工作日凌晨 2 点 |

## 异步执行器管理

### ManagementService

```java
ManagementService managementService = engine.getManagementService();

// 查询待执行的作业
List<Job> jobs = managementService.createJobQuery()
    .list();

// 查询定时作业
List<TimerJob> timerJobs = managementService.createTimerJobQuery()
    .list();

// 执行作业（手动触发）
managementService.executeJob(jobId);

// 删除作业
managementService.deleteJob(jobId);

// 获取作业执行器状态
String status = managementService.getProperties()
    .get("asyncExecutorStatus");
```

### 暂停和恢复

```java
// 暂停异步执行器
managementService.suspendAsyncOperation();

// 恢复异步执行器
managementService.activateAsyncOperation();
```

## 失败处理

### 失败重试

作业失败后，Flowable 会自动重试：

```xml
<serviceTask id="unreliableService"
  flowable:async="true"
  flowable:class="com.example.UnreliableServiceTask"
  flowable:failedJobRetryTimeCycle="3"  <!-- 重试 3 次 -->
  flowable:defaultRetryTimeout="PT1H"/>  <!-- 默认超时 1 小时 -->
```

### 失败监听

```xml
<serviceTask id="unreliableService"
  flowable:async="true"
  flowable:class="com.example.UnreliableServiceTask">
  <extensionElements>
    <flowable:executionListener
      event="onFailure"
      class="com.example.JobFailureListener"/>
  </extensionElements>
</serviceTask>
```

```java
public class JobFailureListener implements ExecutionListener {

    @Override
    public void notify(DelegateExecution execution) {
        // 获取失败信息
        Throwable exception = execution.getVariable("flowableJobException");
        int retryCount = (int) execution.getVariable("flowableJobRetryCount");

        // 发送告警
        alertService.send("Job执行失败",
            "作业ID: " + execution.getId(),
            "重试次数: " + retryCount,
            "异常: " + exception.getMessage()
        );
    }
}
```

### 死信队列

多次重试仍失败的作业会进入死信队列：

```java
// 查询死信作业
List<Job> deadLetterJobs = managementService.createDeadLetterJobQuery()
    .list();

// 重新执行死信作业
managementService.moveDeadLetterJobToExecutableJob(jobId, 3);

// 删除死信作业
managementService.deleteDeadLetterJob(jobId);
```

## 总结

异步执行核心配置：

| 配置 | 说明 |
|---|---|
| `flowable:async="true"` | 标记为异步执行 |
| `flowable:failedJobRetryTimeCycle` | 失败重试次数 |
| `flowable:exclusive="true"` | 排他执行（同一流程实例的作业顺序执行） |
