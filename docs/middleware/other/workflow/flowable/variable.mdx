# Flowable 流程变量

流程变量是流程实例的「上下文数据」——它在整个流程执行过程中传递和共享，支撑着条件判断、表单数据、业务计算等核心功能。

## 为什么需要流程变量？

没有流程变量，流程引擎只能执行「死流程」——每个步骤都固定，无法根据业务数据做决策。

```
没有变量：开始 → 任务A → 任务B → 结束（固定流程）

有变量：开始 → 任务A（设置 days=10）→ ◇（判断 days>3?）→ 主管审批
                                                          ↓
                                                        ◇（判断 days>7?）→ 总监审批
                                                          ↓
                                                        HR审批
                                                      → 结束（动态流程）
```

## 变量作用域

Flowable 支持两种变量作用域：

```
Global（全局变量）
├── 整个流程实例生命周期内可见
├── 所有任务和执行都可以访问
└── 流程启动时创建，流程结束时销毁

Local（局部变量）
├── 仅在当前任务或执行上下文中可见
├── 不影响其他任务或执行
└── 任务完成后自动清理
```

### 使用场景

| 场景 | 推荐作用域 | 原因 |
|---|---|---|
| 订单 ID、金额 | 全局变量 | 所有节点都需要 |
| 审批意见 | 局部变量 | 只在当前审批节点使用 |
| 循环计数 | 局部变量 | 每次循环独立 |
| 表单数据 | 局部变量 | 每个任务的表单独立 |

## 设置变量

### RuntimeService 设置变量

```java
RuntimeService runtimeService = engine.getRuntimeService();

// 启动流程时设置变量
Map<String, Object> variables = new HashMap<>();
variables.put("employee", "张三");
variables.put("days", 5);
variables.put("reason", "年假旅行");

ProcessInstance instance = runtimeService.startProcessInstanceByKey(
    "leave-process", variables
);

// 流程运行时设置全局变量
runtimeService.setVariable(processInstanceId, "approved", true);
runtimeService.setVariable(processInstanceId, "approvalNote", "同意，祝旅途愉快");

// 批量设置
Map<String, Object> updates = new HashMap<>();
updates.put("status", "APPROVED");
updates.put("approvedAt", new Date());
runtimeService.setVariables(processInstanceId, updates);
```

### TaskService 设置局部变量

```java
TaskService taskService = engine.getTaskService();

// 设置局部变量（仅当前任务可见）
taskService.setVariableLocal(taskId, "taskNote", "张三已处理");
taskService.setVariableLocal(taskId, "processedAt", new Date());

// 局部变量不会影响全局
// 其他任务无法获取 taskNote 的值
```

### 在 BPMN 中使用变量

```xml title="leave-process.bpmn"
<!-- 排他网关的条件表达式 -->
<sequenceFlow id="flow1" sourceRef="gateway" targetRef="managerTask">
  <conditionExpression>${days <= 3}</conditionExpression>
</sequenceFlow>

<!-- 用户任务的 assignee 从变量获取 -->
<userTask id="managerApproval" name="主管审批"
  flowable:assignee="${managerId}"/>

<!-- 服务任务的输入参数 -->
<serviceTask id="sendNotification" name="发送通知"
  flowable:class="com.example.NotificationTask">
  <extensionElements>
    <flowable:field name="recipient">
      <flowable:expression>${employeeEmail}</flowable:expression>
    </flowable:field>
    <flowable:field name="message">
      <flowable:expression>您的请假申请已${approved ? '批准' : '驳回'}</flowable:expression>
    </flowable:field>
  </extensionElements>
</serviceTask>
```

## 读取变量

### 读取全局变量

```java
RuntimeService runtimeService = engine.getRuntimeService();

// 读取单个变量
String employee = (String) runtimeService.getVariable(processInstanceId, "employee");
Integer days = (Integer) runtimeService.getVariable(processInstanceId, "days");

// 读取多个变量
Map<String, Object> allVariables = runtimeService.getVariables(processInstanceId);

// 按名称过滤读取
Map<String, Object> filtered = runtimeService.getVariables(processInstanceId,
    Arrays.asList("employee", "days"));
```

### 读取局部变量

```java
TaskService taskService = engine.getTaskService();

// 读取当前任务的局部变量
Map<String, Object> localVars = taskService.getVariablesLocal(taskId);
String taskNote = (String) taskService.getVariableLocal(taskId, "taskNote");
```

### 在 JavaDelegate 中读取

```java
public class ApprovalDelegate implements JavaDelegate {

    @Override
    public void execute(DelegateExecution execution) {
        // 读取变量
        String employee = (String) execution.getVariable("employee");
        Integer days = (Integer) execution.getVariable("days");

        // 设置变量
        execution.setVariable("approved", true);
        execution.setVariable("approver", "manager-zhang");

        // 设置局部变量
        execution.setVariableLocal("processingTime", System.currentTimeMillis());

        // 获取流程实例 ID
        String processInstanceId = execution.getProcessInstanceId();
        String executionId = execution.getId();
    }
}
```

## 变量类型

### 支持的类型

| 类型 | 示例 | 说明 |
|---|---|---|
| String | `"hello"` | 字符串 |
| Integer | `123` | 整数 |
| Long | `123L` | 长整数 |
| Double | `3.14` | 浮点数 |
| Boolean | `true`/`false` | 布尔值 |
| Date | `new Date()` | 日期 |
| List | `Arrays.asList("a","b")` | 列表 |
| Map | `new HashMap<>()` | 映射 |
| Serializable | 自定义对象 | 必须实现 Serializable |

### 自定义对象

```java
// 定义可序列化对象
public class LeaveRequest implements Serializable {
    private static final long serialVersionUID = 1L;

    private String employeeName;
    private Integer days;
    private LocalDate startDate;
    private String reason;

    // getters and setters
}

// 使用自定义对象
LeaveRequest request = new LeaveRequest();
request.setEmployeeName("张三");
request.setDays(5);

runtimeService.setVariable(processInstanceId, "leaveRequest", request);
```

:::warning 序列化注意
自定义对象必须实现 `Serializable` 接口。如果对象包含其他自定义类型的字段，这些字段也必须实现序列化。
:::

## 执行上下文

### DelegateExecution

`DelegateExecution` 是当前执行的上下文对象：

```java
public class PrintContextDelegate implements JavaDelegate {

    @Override
    public void execute(DelegateExecution execution) {
        System.out.println("======== 执行上下文 ========");
        System.out.println("流程定义 ID: " + execution.getProcessDefinitionId());
        System.out.println("流程实例 ID: " + execution.getProcessInstanceId());
        System.out.println("执行 ID: " + execution.getId());
        System.out.println("当前节点 ID: " + execution.getCurrentActivityId());
        System.out.println("当前节点名称: " + execution.getCurrentActivityName());
        System.out.println("父执行 ID: " + execution.getParentId());
        System.out.println("流程变量: " + execution.getVariables());
    }
}
```

### 历史变量

```java
HistoryService historyService = engine.getHistoryService();

// 查询历史变量
List<HistoricVariableInstance> historyVars = historyService
    .createHistoricVariableInstanceQuery()
    .processInstanceId(processInstanceId)
    .list();

// 查询特定变量的历史值
HistoricDetailQuery query = historyService.createHistoricDetailQuery()
    .variableUpdates()
    .processInstanceId(processInstanceId)
    .variableName("days");

List<HistoricVariableUpdate> updates = (List<HistoricVariableUpdate>) query.list();

// 变量值变化历史
for (HistoricVariableUpdate update : updates) {
    System.out.println(update.getTime() + ": " + update.getValue());
}
```

## 变量与并发

### 并行分支中的变量

并行执行时，每个分支有独立的执行上下文：

```
                        流程实例
                            │
                        ◇+ 分支
                      ┌───┴───┐
                      ↓       ↓
                  执行 A    执行 B
                  days=3    days=7
                      │       │
                      └───┬───┘
                          ↓
                        ◇+ 合并
```

```java
// 在并行分支中设置变量
public class BranchDelegate implements JavaDelegate {
    @Override
    public void execute(DelegateExecution execution) {
        String branchId = execution.getCurrentActivityId();
        // 每个分支只能看到自己的变量
        execution.setVariable("branchId", branchId);
        // 也可以设置到流程实例级别
        // execution.setVariable() // 全局
        // execution.setVariableLocal() // 局部
    }
}
```

### 子流程中的变量

子流程可以访问父流程的变量：

```java
// 父流程
runtimeService.setVariable(parentInstanceId, "parentVar", "来自父流程");

// 子流程（嵌入式子流程）
public class SubProcessDelegate implements JavaDelegate {
    @Override
    public void execute(DelegateExecution execution) {
        // 可以读取父流程的变量
        String parentVar = (String) execution.getVariable("parentVar");

        // 可以设置变量到父流程
        execution.setVariable("subResult", "子流程完成");

        // 也可以设置局部变量
        execution.setVariableLocal("subLocal", "仅子流程可见");
    }
}
```

## 总结

流程变量核心操作：

| 操作 | 方法 | 说明 |
|---|---|---|
| 设置全局变量 | `runtimeService.setVariable()` | 整个流程实例可见 |
| 设置局部变量 | `taskService.setVariableLocal()` | 仅当前任务可见 |
| 读取变量 | `execution.getVariable()` | JavaDelegate 中使用 |
| 批量操作 | `getVariables()` / `setVariables()` | 一次操作多个 |
