# Flowable 流程部署

流程部署是 Flowable 的起点——将 BPMN 流程定义文件加载到引擎中，让引擎能够启动流程实例。

## 部署方式

### 方式一：Classpath 资源部署

最简单的方式，直接从 classpath 加载 BPMN 文件：

```java
RepositoryService repositoryService = engine.getRepositoryService();

Deployment deployment = repositoryService.createDeployment()
    .name("请假流程")
    .key("leave-process")
    .addClasspathResource("bpmn/leave-process.bpmn20.xml")
    .deploy();
```

### 方式二：ZIP 包部署

支持一次部署多个 BPMN 文件和相关的表单、决策表：

```java
Deployment deployment = repositoryService.createDeployment()
    .name("完整请假流程")
    .addZipInputStream(new ZipInputStream(
        new FileInputStream("leave-process.zip")
    ))
    .deploy();
```

ZIP 包结构：

```
leave-process.zip
├── leave-process.bpmn20.xml
├── approve-form.form
├── leave-decision.dmn
└── icons/
    └── custom-icon.png
```

### 方式三：字符串部署

直接传入 BPMN XML 字符串：

```java
String bpmnXml = "<?xml version=\"1.0\" encoding=\"UTF-8\"?>...";

Deployment deployment = repositoryService.createDeployment()
    .name("动态流程")
    .addString("dynamic-process.bpmn20.xml", bpmnXml)
    .deploy();
```

### 方式四：InputStream 部署

从文件输入流或网络流部署：

```java
// 从文件部署
InputStream inputStream = new FileInputStream("process.bpmn");
Deployment deployment = repositoryService.createDeployment()
    .addInputStream("process.bpmn20.xml", inputStream)
    .deploy();

// 从 URL 部署（动态加载）
URL url = new URL("http://config-server/bpmn/leave-process.bpmn");
repositoryService.createDeployment()
    .addInputStream("leave-process.bpmn20.xml", url.openStream())
    .deploy();
```

## 部署监听器

部署过程中可以添加监听器，用于扩展或验证：

```java
repositoryService.createDeployment()
    .name("请假流程")
    .addClasspathResource("bpmn/leave-process.bpmn20.xml")
    .deploy()
    .getDeploymentId();

// 部署监听器
public class MyDeploymentListener implements DeploymentListener {

    @Override
    public void notify(DeploymentWithDefinitions deploymentWithDefinitions) {
        System.out.println("部署完成: " + deploymentWithDefinitions.getDeployment().getName());

        // 部署后处理
        // - 发送通知
        // - 更新缓存
        // - 初始化数据
    }
}

repositoryService.createDeployment()
    .name("请假流程")
    .addClasspathResource("bpmn/leave-process.bpmn20.xml")
    .deploymentBuilderFactory()
    .addDeploymentListener(new MyDeploymentListener())
    .deploy();
```

## 流程定义查询

部署后，可以通过多种方式查询流程定义：

### 查询单个流程定义

```java
// 根据 Key 查询最新版本
ProcessDefinition pd = repositoryService
    .createProcessDefinitionQuery()
    .processDefinitionKey("leave-process")
    .latestVersion()
    .singleResult();

// 根据 ID 查询
ProcessDefinition pd = repositoryService
    .getProcessDefinition("leave-process:2:10003");
```

### 批量查询

```java
// 查询所有流程定义
List<ProcessDefinition> all = repositoryService
    .createProcessDefinitionQuery()
    .orderByProcessDefinitionName()
    .desc()
    .list();

// 查询指定版本的流程定义
List<ProcessDefinition> v1 = repositoryService
    .createProcessDefinitionQuery()
    .processDefinitionKey("leave-process")
    .processDefinitionVersion(1)
    .list();

// 分页查询
List<ProcessDefinition> page = repositoryService
    .createProcessDefinitionQuery()
    .orderByProcessDefinitionName()
    .desc()
    .listPage(0, 10);
```

### 获取流程定义资源

```java
// 获取 BPMN XML
String bpmnXml = repositoryService.getResourceAsString(
    deploymentId,
    resourceName
);

// 获取流程图（用于可视化）
InputStream diagram = repositoryService.getProcessDiagram(processDefinitionId);
```

## 流程定义状态

流程定义有两种状态：激活（Active）和挂起（Suspended）。

### 挂起流程定义

挂起后，新的流程实例无法启动，但正在运行的实例不受影响：

```java
// 挂起流程定义
repositoryService.suspendProcessDefinitionByKey("leave-process");

// 指定时间后自动挂起
repositoryService.suspendProcessDefinitionByKey("leave-process",
    false,  // suspendProcessInstances
    new Date(System.currentTimeMillis() + 86400000)  // 24 小时后
);
```

### 激活流程定义

```java
// 激活流程定义
repositoryService.activateProcessDefinitionByKey("leave-process");
```

### 批量操作

```java
// 挂起多个流程定义
repositoryService.createProcessDefinitionQuery()
    .processDefinitionKeyLike("leave-%")
    .list()
    .forEach(pd -> {
        repositoryService.suspendProcessDefinitionById(pd.getId());
    });
```

## 级联操作

如果流程定义下有正在运行的实例，需要使用级联操作：

```java
// 普通挂起（如果有运行中的实例会报错）
repositoryService.suspendProcessDefinitionByKey("leave-process");

// 级联挂起（同时挂起所有运行中的实例）
repositoryService.suspendProcessDefinitionByKey("leave-process",
    true,   // cascade
    null    // suspendDate
);

// 级联激活
repositoryService.activateProcessDefinitionByKey("leave-process",
    true,   // cascade
    null    // activateDate
);
```

:::warning 级联操作的代价
级联操作会同时修改所有相关实例的状态，可能导致正在执行的任务被意外中断。使用前请确认影响范围。
:::

## 删除部署

### 普通删除

删除部署后，该部署下的流程定义不可用，但运行中的实例不受影响：

```java
// 删除单个部署
repositoryService.deleteDeployment(deploymentId);

// 级联删除（同时删除运行中的实例）
repositoryService.deleteDeployment(deploymentId, true);
```

### 批量删除

```java
// 删除所有版本的请假流程
repositoryService.createProcessDefinitionQuery()
    .processDefinitionKey("leave-process")
    .list()
    .forEach(pd -> {
        repositoryService.deleteDeployment(pd.getDeploymentId());
    });
```

## 自动部署

Spring Boot 环境下，Flowable 可以自动部署 `processes/` 目录下的 BPMN 文件：

```
src/main/resources/
└── processes/
    ├── leave-process.bpmn20.xml
    ├── order-process.bpmn20.xml
    └── deployment.json
```

```yaml title="application.yml"
flowable:
  process:
    # 自动部署 processes 目录下的 BPMN 文件
    definition-location-base-path: processes/
    .check-process-definitions: true
```

### deployment.json

可以配置部署的额外参数：

```json title="deployment.json"
{
  "processDefinitions": [
    {
      "key": "leave-process",
      "isSuspended": false,
      "category": "HR"
    }
  ]
}
```

## 部署与版本管理

Flowable 根据 Key 和 Version 管理流程定义：

```java
// 每次部署，同一个 Key 会递增版本号
// 第一次部署
repositoryService.createDeployment()
    .addClasspathResource("bpmn/leave-process.bpmn20.xml")
    .key("leave-process")  // 设置 Key
    .deploy();
// Version: 1

// 第二次部署
repositoryService.createDeployment()
    .addClasspathResource("bpmn/leave-process-v2.bpmn20.xml")
    .key("leave-process")
    .deploy();
// Version: 2

// 启动时使用 Key，会自动选择最新版本
runtimeService.startProcessInstanceByKey("leave-process");
// 实际启动的是 Version 2
```

## 总结

流程部署核心操作：

| 操作 | 方法 |
|---|---|
| 部署 | `createDeployment().addXxx().deploy()` |
| 查询 | `createProcessDefinitionQuery()` |
| 挂起 | `suspendProcessDefinitionByKey()` |
| 激活 | `activateProcessDefinitionByKey()` |
| 删除 | `deleteDeployment()` |
