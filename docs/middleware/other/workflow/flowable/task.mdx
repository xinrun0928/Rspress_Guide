# Flowable 用户任务

用户任务是工作流引擎的核心——它代表需要人工参与的工作。在 Flowable 中，用户任务的管理涉及任务的分配、签收、完成、委派等多个方面。

## 任务分配方式

### 方式一：直接分配（Assignee）

直接将任务分配给指定用户：

```xml title="bpmn/leave-approval.bpmn20.xml"
<userTask id="managerApproval" name="主管审批"
  flowable:assignee="${managerId}"/>
```

```java
// 运行时动态设置
Task task = taskService.newTask("task-001");
task.setName("主管审批");
task.setAssignee("manager-zhang");  // 直接指定办理人
taskService.saveTask(task);
```

### 方式二：候选用户（Candidate Users）

多个用户都可以认领该任务，但只能由一人完成：

```xml
<userTask id="managerApproval" name="主管审批">
  <potentialOwner>
    <resourceAssignmentExpression>
      <formalExpression>user(zhangsan), user(lisi), user(wangwu)</formalExpression>
    </resourceAssignmentExpression>
  </potentialOwner>
</userTask>
```

```java
// 动态添加候选用户
Task task = taskService.newTask("task-001");
task.addCandidateUser("zhangsan");
task.addCandidateUser("lisi");
task.addCandidateUser("wangwu");
taskService.saveTask(task);
```

### 方式三：候选组（Candidate Groups）

将任务分配给一个用户组，组内任何人都可以认领：

```xml
<userTask id="managerApproval" name="主管审批">
  <potentialOwner>
    <resourceAssignmentExpression>
      <formalExpression>group(manager)</formalExpression>
    </resourceAssignmentExpression>
  </potentialOwner>
</userTask>
```

```java
// 动态添加候选组
Task task = taskService.newTask("task-001");
task.addCandidateGroup("manager");
taskService.saveTask(task);

// 也可以用 fluent API
taskService.newTask()
    .setName("主管审批")
    .addCandidateGroup("manager")
    .save();
```

## 任务签收（Claim）

候选任务需要先签收，才能办理：

```java
TaskService taskService = engine.getTaskService();

// 查询候选任务（张三可以看到的任务）
List<Task> candidateTasks = taskService.createTaskQuery()
    .taskCandidateUser("zhangsan")
    .list();

// 签收任务（张三抢了这个任务）
taskService.claim(taskId, "zhangsan");

// 签收后，任务就变成了张三的「个人任务」
// 其他人的候选列表中就看不到这个任务了
```

### 签收与反签收

```java
// 反签收（释放任务，重新进入候选池）
taskService.unclaim(taskId);

// 转让给其他人
taskService.setAssignee(taskId, "lisi");
```

## 任务完成（Complete）

完成任务后，流程会流转到下一个节点：

```java
// 完成任务（最简单的场景）
taskService.complete(taskId);

// 完成任务时设置流程变量
taskService.complete(taskId,
    Variables.createVariables()
        .putValue("approved", true)
        .putValue("comment", "同意请假")
);

// 获取下一个任务的 ID
List<Task> nextTasks = taskService.createTaskQuery()
    .processInstanceId(processInstanceId)
    .list();
```

## 任务委派（Delegate）

委派是将任务转给他人办理，办理完后任务会回到原办理人：

### 委派场景

```
场景：张三提交了请假申请，任务分配给主管李四
      李四出差了，把任务委派给王五代为处理
      王五处理完后，任务回到李四
      李四最终审批
```

```java
// 委派任务（张三 -> 李四 -> 王五）
taskService.delegateTask(taskId, "wangwu");

// 委派后，当前办理人变成王五
// 任务有一个 delegationState 标记，记录了委派链

// 王五完成任务后，任务回到李四
taskService.complete(taskId);
```

### 委派 vs 转让

| 方式 | 原办理人 | 任务去向 | 适用场景 |
|---|---|---|---|
| 委派（Delegate） | 记录 | 办完后回到原办理人 | 代为处理，但最终由原人审批 |
| 转让（Transfer） | 变更 | 不再回来 | 完全转交给他人 |

```java
// 转让（完全转交）
taskService.setAssignee(taskId, "newAssignee");
```

## 任务评论与附件

### 添加评论

```java
// 添加评论
taskService.addComment(taskId, processInstanceId, "comment", "同意，祝旅途愉快！");

// 添加评论（不带类型）
taskService.addComment(taskId, processInstanceId, null, "同意，祝旅途愉快！");

// 查询评论
List<Comment> comments = taskService.getTaskComments(taskId);
```

### 添加附件

```java
// 添加文件附件
taskService.createAttachment(
    "url",           // 类型：url、file、processInstance
    taskId,
    processInstanceId,
    "流程图",
    "流程示意图",
    "http://example.com/diagram.png"
);

// 上传文件附件
InputStream inputStream = new FileInputStream("document.pdf");
taskService.createAttachment(
    "file",
    taskId,
    processInstanceId,
    "申请材料",
    "application/pdf",
    inputStream
);
```

## 任务表单

### 内置表单属性

```xml
<userTask id="managerApproval" name="主管审批">
  <extensionElements>
    <flowable:formProperty id="reason"
      name="审批意见"
      type="string"
      required="true"/>
    <flowable:formProperty id="approved"
      name="是否同意"
      type="boolean"
      default="true"/>
  </extensionElements>
</userTask>
```

### 读取表单数据

```java
// 获取表单属性定义
List<FormProperty> formProperties = formService.getTaskFormData(taskId)
    .getFormProperties();

// 获取表单数据
Map<String, Object> formData = formService.getVariables(taskId);

// 提交表单数据
Map<String, Object> properties = new HashMap<>();
properties.put("reason", "同意，祝旅途愉快");
properties.put("approved", true);
formService.submitTaskFormData(taskId, properties);
```

## 任务超时

### 设置截止日期

```java
// 设置 3 天后到期
Date dueDate = new Date(System.currentTimeMillis() + 3 * 24 * 60 * 60 * 1000);
taskService.setDueDate(taskId, dueDate);

// 查询过期任务
List<Task> overdueTasks = taskService.createTaskQuery()
    .taskAssignee("manager-zhang")
    .overdue()
    .list();
```

### BPMN 中定义超时

```xml
<userTask id="managerApproval" name="主管审批">
  <multiInstanceLoopCharacteristics
    isSequential="false"
    activiti:completionThreshold="1">
    <loopCardinality>3</loopCardinality>
  </multiInstanceLoopCharacteristics>
</userTask>

<!-- 任务超时边界事件 -->
<boundaryEvent id="timeout" attachedToRef="managerApproval">
  <timerEventDefinition>
    <timeDuration>PT72H</timeDuration>  <!-- 72 小时超时 -->
  </timerEventDefinition>
</boundaryEvent>
```

## 任务查询

### 常用查询方法

```java
TaskService taskService = engine.getTaskService();

// 查询个人任务
taskService.createTaskQuery()
    .taskAssignee("zhangsan")
    .list();

// 查询候选任务
taskService.createTaskQuery()
    .taskCandidateUser("zhangsan")
    .list();

// 查询候选组任务
taskService.createTaskQuery()
    .taskCandidateGroup("manager")
    .list();

// 查询候选用户或候选组的任务（并集）
taskService.createTaskQuery()
    .taskCandidateOrGroup("zhangsan")
    .list();

// 查询委托任务
taskService.createTaskQuery()
    .taskDelegationState(DelegationState.PENDING)
    .taskAssignee("zhangsan")
    .list();

// 按流程定义 Key 查询
taskService.createTaskQuery()
    .processDefinitionKey("leave-process")
    .taskAssignee("zhangsan")
    .list();

// 按流程实例 ID 查询
taskService.createTaskQuery()
    .processInstanceId(processInstanceId)
    .list();

// 分页查询
List<Task> page = taskService.createTaskQuery()
    .taskAssignee("zhangsan")
    .orderByTaskPriority()
    .desc()
    .listPage(0, 10);
```

## 任务监听器

### 任务创建时

```java
// BPMN 中定义
<userTask id="managerApproval" name="主管审批">
  <extensionElements>
    <flowable:taskListener event="create"
      class="com.example.MyTaskListener"/>
  </extensionElements>
</userTask>
```

```java
public class MyTaskListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        // 任务创建时触发
        delegateTask.setVariable("createdTime", new Date());

        // 动态设置办理人
        String managerId = determineManager(delegateTask);
        delegateTask.setAssignee(managerId);

        // 发送通知
        notificationService.notify(
            delegateTask.getAssignee(),
            "您有一个新任务: " + delegateTask.getName()
        );
    }
}
```

### 任务完成时

```java
<flowable:taskListener event="complete"
  class="com.example.CompleteListener"/>
```

```java
public class CompleteListener implements TaskListener {

    @Override
    public void notify(DelegateTask delegateTask) {
        // 任务完成时触发
        Boolean approved = (Boolean) delegateTask.getVariable("approved");

        // 记录审批结果
        auditService.record(
            delegateTask.getProcessInstanceId(),
            delegateTask.getAssignee(),
            approved
        );
    }
}
```

## 总结

用户任务核心操作：

| 操作 | 方法 | 说明 |
|---|---|---|
| 签收 | `claim(taskId, userId)` | 从候选池抢任务 |
| 反签收 | `unclaim(taskId)` | 释放回候选池 |
| 完成 | `complete(taskId)` | 完成任务，流程流转 |
| 委派 | `delegateTask(taskId, userId)` | 转给他人处理，之后回到原人 |
| 转让 | `setAssignee(taskId, userId)` | 完全转交 |
| 评论 | `addComment()` | 添加评论 |
| 附件 | `createAttachment()` | 添加附件 |
