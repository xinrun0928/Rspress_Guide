# 工作流引擎选型

当企业决定引入工作流引擎时，面对的第一个问题往往是：**选择哪个引擎？**

Flowable、Camunda、Activiti——这三个名字在 Java 工作流领域几乎是绕不开的选择。它们有着相似的血统（都源于 Activiti），却又走出了不同的道路。

## 背景渊源

### Activiti 的诞生

2007 年，Tom Baeyens 离开 JBoss，创建了 Activiti 项目。Activiti 5.x 版本一度是 Java 领域最流行的开源工作流引擎。

### Flowable 的诞生

2016 年，由于对 Activiti 发展方向的分歧，Activiti 核心团队出走，成立了 Flowable 项目。Flowable 在 Activiti 6.x 基础上继续发展，并逐步形成了自己的生态。

### Camunda 的独立发展

Camunda 同样起源于 Activiti 社区，但走了一条不同的路——它更专注于企业级场景，并引入了 BPMN 2.0 + DMN（决策模型和表示法）的组合。

```
Activiti 5.x（2007 年）
      │
      ├──→ Activiti 6.x（2016 年，停止维护）
      │         │
      │         └──→ Flowable 6.x（2016 年，继续发展）
      │                   │
      │                   └──→ Flowable 7.x（云原生方向）
      │
      └──→ Camunda 7.x（2014 年，企业级方向）
                │
                └──→ Camunda 8.x（云原生方向）
```

## 功能对比

| 特性 | Flowable | Camunda | Activiti |
|---|---|---|---|
| BPMN 2.0 支持 | 完整 | 完整 | 完整 |
| DMN 支持 | 支持 | 支持 | 不支持 |
| CMMN 支持 | 支持 | 不支持 | 不支持 |
| Spring Boot 集成 | flowable-spring-boot-starter | camunda-bpm-spring-boot-starter | activiti-spring-boot-starter |
| REST API | 内置 | 内置 | 需要额外配置 |
| 流程设计器 | Flowable Modeler（Web） | Camunda Modeler（桌面/Web） | Activiti Modeler（Web） |
| 流程引擎数 | 6 个 | 1 个（核心）+ DMN | 1 个 |
| 数据库支持 | 主流关系型数据库 | 主流关系型数据库 | 主流关系型数据库 |
| 社区活跃度 | 活跃 | 活跃 | 较低 |

## Flowable vs Camunda vs Activiti

### Flowable

**定位**：轻量级、功能丰富、企业级

**优点**：
- 六大引擎（BPMN、CMMN、DMN、Form、Content、App），覆盖多种流程场景
- 对 Spring Boot 集成非常友好，开箱即用
- 社区活跃，文档完善
- 6.x 版本架构清晰，扩展性强

**缺点**：
- 版本迭代较快，从 6.x 到 7.x 有较大变化
- 某些高级特性需要商业授权

```xml title="Flowable Spring Boot 依赖"
<dependency>
    <groupId>org.flowable</groupId>
    <artifactId>flowable-spring-boot-starter</artifactId>
    <version>7.0.0</version>
</dependency>
```

### Camunda

**定位**：企业级、RESTful 友好、BPMN + DMN

**优点**：
- DMN 决策表支持，适合需要复杂业务规则的场景
- REST API 设计优雅，易于集成
- Tasklist、Cockpit、Admin 三个 Web 应用功能完善
- 商业支持稳定，适合对稳定性要求高的企业

**缺点**：
- 相比 Flowable，CMMN 支持较弱
- 社区功能相对闭源（部分高级特性在企业版）

```xml title="Camunda Spring Boot 依赖"
<dependency>
    <groupId>org.camunda.bpm</groupId>
    <artifactId>camunda-bpm-spring-boot-starter</artifactId>
    <version>7.20.0</version>
</dependency>
```

### Activiti

**定位**：历史遗留系统、轻量级

**优点**：
- 最早的开源工作流引擎，有大量历史项目
- 社区积累丰富（虽然活跃度下降）

**缺点**：
- 6.x 版本后停止维护
- 功能相比 Flowable 和 Camunda 较落后
- Spring Boot 集成不如前两者友好

:::warning 项目选型建议
新项目**强烈不建议选择 Activiti**。如果需要迁移遗留项目，可以考虑迁移到 Flowable。
:::

## Spring Boot 集成对比

### Flowable

```java
@SpringBootApplication
public class FlowableApplication {
    public static void main(String[] args) {
        SpringApplication.run(FlowableApplication.class, args);
    }
}
```

```yaml title="application.yml"
spring:
  datasource:
    url: jdbc:h2:mem:flowable
    driver-class-name: org.h2.Driver
  flowable:
    # 自动部署 BPMN 文件
   .check-process-definitions: true
    database-schema-update: true
```

### Camunda

```java
@SpringBootApplication
@EnableProcessApplication
public class CamundaApplication {
    public static void main(String[] args) {
        SpringApplication.run(CamundaApplication.class, args);
    }
}
```

```yaml title="application.yml"
spring:
  datasource:
    url: jdbc:h2:mem:camunda
    driver-class-name: org.h2.Driver
camunda:
  bpm:
    auto-deployment-enabled: true
    database-schema-update: true
```

## 选择建议

| 场景 | 推荐选择 | 理由 |
|---|---|---|
| 新项目，内部系统 | Flowable | 轻量、功能丰富、Spring Boot 友好 |
| 企业级，需要 DMN | Camunda | DMN 支持完善、企业级稳定 |
| 遗留系统迁移 | Flowable | 从 Activiti 迁移成本低 |
| 需要 CMMN（案例管理） | Flowable | Camunda 不支持 CMMN |
| 需要商业支持 | Camunda | 商业版支持稳定 |

## 总结

三个引擎的核心差异：

```
Flowable：功能全面、引擎众多、适合复杂流程场景
Camunda：企业稳定、DMN 强大、RESTful 友好
Activiti：历史遗产、不推荐新项目使用
```
