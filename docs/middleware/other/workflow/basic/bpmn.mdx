# BPMN 2.0 规范

BPMN（Business Process Model and Notation）2.0 是 OMG（对象管理组织）发布的业务流程建模标准。它用统一的图形符号描述业务流程，让技术人员和业务人员可以用同一套语言沟通。

如果你用过 Visio、ProcessOn 画流程图，BPMN 2.0 的图形看起来会很眼熟。但 BPMN 2.0 不只是画图——它是可执行的。

## BPMN 2.0 的核心元素

BPMN 2.0 的元素可以分为四大类：

```
┌─────────────────────────────────────────────────────────────┐
│                      BPMN 2.0 元素分类                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 事件（Event）    ──  ○ ──  流程开始、进行中、结束       │
│                                                             │
│  2. 活动（Activity） ──  □ ──  任务、子流程                  │
│                                                             │
│  3. 网关（Gateway）  ──  ◇ ──  条件分支、并行合并            │
│                                                             │
│  4. 流向（Flow）     ──  → ──  顺序流、消息流                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 事件（Event）

事件是流程中的「瞬间」——它代表发生的事情。事件用圆圈表示，圆圈的样式区分不同类型。

### 开始事件（Start Event）

开始事件是流程的入口点，每个流程有且只有一个开始事件。

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   ○ 空开始事件            ○ 消息开始事件（信封图标）          │
│   (None Start Event)     (Message Start Event)             │
│                                                            │
│   ○ 定时器开始事件（钟表）  ○ 错误开始事件（闪电）            │
│   (Timer Start Event)     (Error Start Event)              │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

```xml title="开始事件示例"
<!-- 空开始事件（最常用） -->
<startEvent id="start"/>

<!-- 定时器开始事件（用于定时触发的流程） -->
<startEvent id="scheduledStart">
  <timerEventDefinition>
    <timeCycle>0 0 2 ? * *</timeCycle>  <!-- 每天凌晨 2 点触发 -->
  </timerEventDefinition>
</startEvent>

<!-- 消息开始事件（用于被外部消息触发的流程） -->
<startEvent id="messageStart">
  <messageEventDefinition messageRef="ns:newOrderMessage"/>
</startEvent>
```

### 结束事件（End Event）

结束事件表示流程的终点。结束事件用粗圆圈表示。

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   ● 空结束事件            ● 消息结束事件                     │
│   (None End Event)       (Message End Event)              │
│                                                            │
│   ● 错误结束事件（闪电）    ● 终止结束事件（黑色实心）         │
│   (Error End Event)       (Terminate End Event)            │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

```xml title="结束事件示例"
<!-- 空结束事件（最常用） -->
<endEvent id="end"/>

<!-- 错误结束事件 -->
<endEvent id="errorEnd">
  <errorEventDefinition errorRef="ns:paymentFailedError"/>
</endEvent>

<!-- 终止结束事件（立即终止整个流程实例） -->
<endEvent id="terminateEnd">
  <terminateEventDefinition/>
</endEvent>
```

### 中间事件（Intermediate Event）

中间事件发生在流程执行过程中，用双层圆圈表示。

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   ◎ 定时器中间事件（等待）   ◎ 消息中间事件（捕获）           │
│   (Timer Intermediate Event)  (Message Intermediate Event) │
│                                                            │
│   ◎ 错误中间事件（捕获）     ◎ 信号中间事件（捕获）           │
│   (Error Intermediate Event)  (Signal Intermediate Event)   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 活动（Activity）

活动是流程中的「工作单元」，用圆角矩形表示。

### 任务（Task）

任务是原子活动，不可再分。

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   ┌──────────────┐    用户任务（人像图标）                  │
│   │   用户任务    │    (User Task)                         │
│   └──────────────┘                                        │
│                                                            │
│   ┌──────────────┐    服务任务（齿轮图标）                  │
│   │   服务任务    │    (Service Task)                     │
│   └──────────────┘                                        │
│                                                            │
│   ┌──────────────┐    脚本任务（脚本图标）                  │
│   │   脚本任务    │    (Script Task)                      │
│   └──────────────┘                                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

```xml title="任务示例"
<!-- 用户任务 -->
<userTask id="managerApproval" name="主管审批">
  <documentation>审批员工提交的请假申请</documentation>
  <potentialOwner>
    <resourceAssignmentExpression>
      <formalExpression>manager-group</formalExpression>
    </resourceAssignmentExpression>
  </potentialOwner>
</userTask>

<!-- 服务任务（调用 Java 类） -->
<serviceTask id="sendNotification"
  name="发送通知"
  implementation="WebService"
  operationRef="ns:sendNotificationOperation"/>

<!-- 脚本任务 -->
<scriptTask id="calculateAmount" name="计算金额">
  <script>
    <![CDATA[
      var price = execution.getVariable("unitPrice");
      var quantity = execution.getVariable("quantity");
      var amount = price * quantity;
      execution.setVariable("totalAmount", amount);
    ]]>
  </script>
</scriptTask>
```

### 子流程（SubProcess）

子流程是一组活动的容器，可以折叠显示。

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   ┌──────────────────────────────┐                        │
│   │      嵌入式子流程             │  ← 有边界框             │
│   │  ┌────┐  ┌────┐            │                        │
│   │  │ A  │→ │ B  │            │                        │
│   │  └────┘  └────┘            │                        │
│   └──────────────────────────────┘                        │
│                                                            │
│   ┌──────────────────────────────┐                        │
│   │  ⎈ 调用活动（Call Activity） │  ← 粗边框               │
│   │  引用子流程名称              │                        │
│   └──────────────────────────────┘                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

```xml title="子流程示例"
<!-- 嵌入式子流程 -->
<subProcess id="approvalProcess">
  <startEvent id="subStart"/>
  <userTask id="firstApproval" name="一级审批"/>
  <userTask id="secondApproval" name="二级审批"/>
  <endEvent id="subEnd"/>
</subProcess>

<!-- 调用活动（引用其他流程） -->
<callActivity id="callLeaveProcess" name="调用请假流程">
  <calledElement>leaveProcess</calledElement>
</callActivity>
```

## 网关（Gateway）

网关用于控制流程的分支和合并，用菱形表示。

### 排他网关（XOR Gateway）

排他网关只能选择一条路径——条件互斥，只满足其一。

```
              ○ 开始
               │
               ▼
          ╱─ ◇ ─╲
         ╱  XOR   ╲
        ╱           ╲
    [天数<=3]      [天数>3]
        ╲           ╱
         ╲  XOR   ╱
          ╲─ ◇ ─╱
               │
          ┌────┴────┐
          │         │
          ▼         ▼
       主管审批   主管+HR审批
          │         │
          └────┬────┘
               ▼
            ○ 结束
```

```xml title="排他网关示例"
<!-- 排他分支 -->
<exclusiveGateway id="approvalGateway" name="审批分流"/>

<!-- 顺序流上的条件 -->
<sequenceFlow id="flow1" sourceRef="approvalGateway" targetRef="managerOnlyTask">
  <conditionExpression xsi:type="tFormalExpression">
    ${days <= 3}
  </conditionExpression>
</sequenceFlow>
<sequenceFlow id="flow2" sourceRef="approvalGateway" targetRef="managerAndHrTask">
  <conditionExpression xsi:type="tFormalExpression">
    ${days > 3}
  </conditionExpression>
</sequenceFlow>

<!-- 排他合并 -->
<exclusiveGateway id="mergeGateway" name="审批合并"/>
```

### 并行网关（AND Gateway）

并行网关所有路径同时执行——需要所有分支都完成才能继续。

```
              ○ 开始
               │
               ▼
          ╱─ ◇ ─╲
         ╱  AND   ╲
        ╱           ╲
        │           │
        ▼           ▼
     任务 A       任务 B
        │           │
        ╲           ╱
         ╲  AND   ╱
          ╲─ ◇ ─╱
               │
               ▼
            ○ 结束
```

```xml title="并行网关示例"
<!-- 并行分支 -->
<parallelGateway id="fork" name="并行执行"/>
<sequenceFlow sourceRef="fork" targetRef="taskA"/>
<sequenceFlow sourceRef="fork" targetRef="taskB"/>

<!-- 并行合并（等待所有分支完成） -->
<parallelGateway id="join" name="并行合并"/>
<sequenceFlow sourceRef="taskA" targetRef="join"/>
<sequenceFlow sourceRef="taskB" targetRef="join"/>
```

### 包容网关（OR Gateway）

包容网关可以同时选择多条路径——只要条件满足就执行。

```
              ○ 开始
               │
               ▼
          ╱─ ◇ ─╲
         ╱  OR    ╲
        ╱           ╲
    [金额大]      [紧急]
        ╲           ╱
         ╲  OR    ╱
          ╲─ ◇ ─╱
```

```xml title="包容网关示例"
<!-- 包容分支（默认条件也可以不写表达式） -->
<inclusiveGateway id="inclusiveFork"/>

<sequenceFlow sourceRef="inclusiveFork" targetRef="managerApproval">
  <conditionExpression>${amount > 10000}</conditionExpression>
</sequenceFlow>
<sequenceFlow sourceRef="inclusiveFork" targetRef="urgentApproval">
  <conditionExpression>${urgent == true}</conditionExpression>
</sequenceFlow>
<!-- 默认流（无条件） -->
<sequenceFlow sourceRef="inclusiveFork" targetRef="normalApproval"/>
```

### 事件网关（Event Gateway）

事件网关根据事件类型选择路径，而不是条件表达式。

```
              ○ 开始
               │
               ▼
          ╱─ ◇ ─╲
         ╱ Event  ╲
        ╱           ╲
        ◎消息       ◎定时器
        │           │
        ▼           ▼
      流程 A      流程 B
```

```xml title="事件网关示例"
<eventBasedGateway id="eventGateway"/>

<sequenceFlow sourceRef="eventGateway" targetRef="paymentReceivedEvent">
  <conditionExpression>paymentType == 'online'</conditionExpression>
</sequenceFlow>

<receiveTask id="paymentReceivedEvent" name="等待在线支付"/>
<receiveTask id="cashReceivedEvent" name="等待现金支付"/>
```

## 流向（Sequence Flow）

流向连接各个元素，控制执行顺序。

### 顺序流（Sequence Flow）

实线箭头表示顺序流。

```xml title="顺序流示例"
<sequenceFlow id="flow1"
  sourceRef="startEvent"
  targetRef="submitTask"/>
```

### 条件顺序流

条件顺序流在箭头上标注条件。

```xml title="条件顺序流示例"
<sequenceFlow id="approveFlow"
  sourceRef="reviewTask"
  targetRef="approveTask">
  <conditionExpression>
    ${approved == true}
  </conditionExpression>
</sequenceFlow>

<sequenceFlow id="rejectFlow"
  sourceRef="reviewTask"
  targetRef="rejectTask">
  <conditionExpression>
    ${approved == false}
  </conditionExpression>
</sequenceFlow>
```

### 默认流

默认流在没有其他条件满足时执行。

```xml title="默认流示例"
<exclusiveGateway id="gateway" default="defaultFlow"/>

<!-- 无条件的是默认流 -->
<sequenceFlow id="defaultFlow"
  sourceRef="gateway"
  targetRef="defaultTask"/>
```

## 完整示例

```xml title="leave-process.bpmn" wrapCode
<?xml version="1.0" encoding="UTF-8"?>
<definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL"
             xmlns:bpmndi="http://www.omg.org/spec/BPMN/20100524/DI"
             targetNamespace="http://bpmn.example.com/leave">

  <process id="leaveProcess" name="请假流程" isExecutable="true">

    <!-- 开始事件 -->
    <startEvent id="start" name="开始"/>

    <!-- 用户任务：提交请假 -->
    <userTask id="submitLeave" name="提交请假">
      <documentation>员工填写请假申请</documentation>
      <potentialOwner>
        <resourceAssignmentExpression>
          <formalExpression>user</formalExpression>
        </resourceAssignmentExpression>
      </potentialOwner>
    </userTask>

    <!-- 排他网关：分流 -->
    <exclusiveGateway id="gateway" name="审批分流"/>

    <!-- 用户任务：主管审批 -->
    <userTask id="managerApproval" name="主管审批">
      <potentialOwner>
        <resourceAssignmentExpression>
          <formalExpression>manager</formalExpression>
        </resourceAssignmentExpression>
      </potentialOwner>
    </userTask>

    <!-- 用户任务：HR 审批（天数 > 3 时执行） -->
    <userTask id="hrApproval" name="HR审批"/>

    <!-- 排他网关：合并 -->
    <exclusiveGateway id="mergeGateway" name="审批合并"/>

    <!-- 服务任务：发送通知 -->
    <serviceTask id="notifyTask" name="发送通知"
      implementation="WebService"
      operationRef="ns:notifyOperation"/>

    <!-- 结束事件 -->
    <endEvent id="end" name="结束"/>

    <!-- 顺序流 -->
    <sequenceFlow id="flow1" sourceRef="start" targetRef="submitLeave"/>
    <sequenceFlow id="flow2" sourceRef="submitLeave" targetRef="gateway"/>

    <!-- 分流条件 -->
    <sequenceFlow id="flow3" sourceRef="gateway" targetRef="managerApproval">
      <conditionExpression>${days <= 3}</conditionExpression>
    </sequenceFlow>
    <sequenceFlow id="flow4" sourceRef="gateway" targetRef="hrApproval">
      <conditionExpression>${days > 3}</conditionExpression>
    </sequenceFlow>

    <!-- 合流 -->
    <sequenceFlow id="flow5" sourceRef="managerApproval" targetRef="mergeGateway"/>
    <sequenceFlow id="flow6" sourceRef="hrApproval" targetRef="mergeGateway"/>

    <sequenceFlow id="flow7" sourceRef="mergeGateway" targetRef="notifyTask"/>
    <sequenceFlow id="flow8" sourceRef="notifyTask" targetRef="end"/>

  </process>

</definitions>
```

## 总结

BPMN 2.0 元素速查：

| 类别 | 元素 | 图形 | 用途 |
|---|---|---|---|
| 事件 | 开始事件 | ○ | 流程入口 |
| 事件 | 结束事件 | ● | 流程终点 |
| 事件 | 中间事件 | ◎ | 流程中发生的事 |
| 活动 | 用户任务 | □+人 | 需要人参与的工作 |
| 活动 | 服务任务 | □+⚙ | 自动执行的工作 |
| 活动 | 子流程 | □+□ | 嵌套的流程 |
| 网关 | 排他网关 | ◇ | 条件分支（一选一） |
| 网关 | 并行网关 | ◇+加号 | 并行执行（都执行） |
| 网关 | 包容网关 | ◇+圆圈 | 条件分支（多选） |
| 网关 | 事件网关 | ◇+闪电 | 基于事件选择 |
| 流向 | 顺序流 | → | 执行顺序 |
