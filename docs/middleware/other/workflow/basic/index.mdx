# 工作流基础

在深入学习具体的工作流引擎之前，我们需要先理解工作流的核心概念。这些概念是所有工作流引擎的「通用语言」，掌握了它们，学习任何工作流引擎都会事半功倍。

## 流程定义（Process Definition）

流程定义是工作流的「蓝图」——它描述了一个业务流程的结构和行为。

### 流程定义的关键属性

| 属性 | 说明 | 示例 |
|---|---|---|
| ID | 流程定义唯一标识 | `leave-process` |
| Name | 流程定义名称 | `请假流程` |
| Version | 版本号，每次部署自动递增 | `v1`、`v2` |
| Key | 业务标识，同一 Key 的不同版本属于同一流程 | `leave-process` |
| XML / BPMN | 流程定义文件 | `leave-process.bpmn` |

### 流程定义的生命周期

```
创建 → 部署（Deploy） → 激活（Active） → 挂起（Suspended） → 删除（Deleted）
```

```java
// 部署流程定义
RepositoryService repositoryService = engine.getRepositoryService();
Deployment deployment = repositoryService.createDeployment()
    .addClasspathResource("leave-process.bpmn")
    .name("请假流程")
    .deploy();

// 查询已部署的流程定义
ProcessDefinition processDefinition = repositoryService
    .createProcessDefinitionQuery()
    .processDefinitionKey("leave-process")
    .latestVersion()
    .singleResult();

// 挂起流程定义（新流程实例无法启动）
repositoryService.suspendProcessDefinitionByKey("leave-process");

// 激活流程定义
repositoryService.activateProcessDefinitionByKey("leave-process");
```

:::tip 版本管理
同一个 Key 的流程可以有多个版本。当流程需要升级时，部署新版本即可。旧版本的流程实例继续执行到结束，新流程实例使用新版本。
:::

## 流程实例（Process Instance）

流程定义是「模板」，流程实例是「实例」——一个流程定义可以被启动多次，每次启动创建一个新的流程实例。

### 流程实例 vs 流程定义

```
流程定义（Process Definition）
├── 流程定义 ID: leave-process:2:10003
├── Key: leave-process
└── Version: 2
    │
    ├── 流程实例 1（张三的请假）
    │   ├── 流程实例 ID: 2501
    │   └── 当前节点：主管审批
    │
    └── 流程实例 2（李四的请假）
        ├── 流程实例 ID: 2502
        └── 当前节点：结束
```

### 启动流程实例

```java
RuntimeService runtimeService = engine.getRuntimeService();

// 方式一：根据流程定义 Key 启动（使用最新版本）
ProcessInstance instance = runtimeService.startProcessInstanceByKey("leave-process");

// 方式二：传入业务键（用于与业务系统关联）
ProcessInstance instance = runtimeService
    .startProcessInstanceByKey("leave-process",
        Variables.createVariables()
            .putValue("employee", "张三")
            .putValue("days", 5));

// 业务键（businessKey）用于关联业务系统
// 例如：ProcessInstance instance = runtimeService.startProcessInstanceByKey("order-process", orderId);
```

### 流程实例状态

| 状态 | 说明 |
|---|---|
| Running | 流程实例正在执行 |
| Suspended | 流程实例被挂起，任务暂停执行 |
| Completed | 流程实例正常结束 |
| Terminated | 流程实例被终止 |

## 任务（Task）

任务是工作流中的「工作单元」——它代表了一个需要人或者系统完成的工作。

### 任务类型

| 类型 | 说明 | 示例 |
|---|---|---|
| User Task | 需要人参与的任务 | 审批、填写表单 |
| Service Task | 自动执行的任务 | 发送邮件、调用 API |
| Manual Task | 不由引擎执行的任务 | 线下签字 |
| Script Task | 引擎执行脚本的任务 | 计算、数据转换 |
| Receive Task | 等待外部信号的任务 | 等待回调 |

### 用户任务的核心属性

```java
// 创建用户任务
TaskService taskService = engine.getTaskService();
Task task = taskService.newTask("task-001");
task.setName("主管审批");
task.setDescription("审批张三的请假申请");
task.setAssignee("manager-001");  // 指定办理人
task.setPriority(10);  // 优先级 1-50
taskService.saveTask(task);
```

| 属性 | 说明 |
|---|---|
| ID | 任务唯一标识 |
| Name | 任务名称（展示用） |
| Description | 任务描述 |
| Assignee | 办理人（具体的人） |
| CandidateUsers | 候选用户列表（多选一） |
| CandidateGroups | 候选组列表（组内任何人都可以办理） |
| Priority | 优先级（1-50，数字越大优先级越高） |
| Due Date | 到期时间 |
| Category | 任务分类 |

### 任务查询

```java
// 查询个人任务
List<Task> myTasks = taskService.createTaskQuery()
    .taskAssignee("manager-001")
    .list();

// 查询候选任务（我可以签收的任务）
List<Task> candidateTasks = taskService.createTaskQuery()
    .taskCandidateUser("manager-001")
    .list();

// 查询组任务
List<Task> groupTasks = taskService.createTaskQuery()
    .taskCandidateGroup("manager-group")
    .list();

// 按流程实例 ID 查询任务
List<Task> tasks = taskService.createTaskQuery()
    .processInstanceId("2501")
    .list();
```

## 流程变量（Process Variable）

流程变量是流程实例的「上下文数据」——它在整个流程执行过程中传递和共享。

### 变量类型

| 类型 | 示例 |
|---|---|
| 基本类型 | `int`、`String`、`Boolean`、`Date` |
| 复杂类型 | `Map` `List` `CustomObject` |
| 文件类型 | `Binary`（附件、二进制数据） |

### 变量作用域

```
Global（全局变量）
└── 整个流程实例生命周期内可见

Local（局部变量）
└── 某个任务或执行上下文内可见
```

```java
RuntimeService runtimeService = engine.getRuntimeService();
TaskService taskService = engine.getTaskService();

// 设置全局变量（整个流程实例可见）
runtimeService.setVariable(processInstanceId, "employee", "张三");
runtimeService.setVariable(processInstanceId, "days", 5);

// 设置局部变量（仅当前任务可见）
taskService.setVariableLocal(taskId, "approvalNote", "同意，祝旅途愉快！");

// 获取变量
String employee = (String) runtimeService.getVariable(processInstanceId, "employee");
Integer days = (Integer) runtimeService.getVariable(processInstanceId, "days");

// 获取所有变量
Map<String, Object> variables = runtimeService.getVariables(processInstanceId);
```

### 变量与 BPMN 表达式

流程变量可以在 BPMN 中通过 `${}` 或 `#{` `}` 表达式引用：

```xml title="leave-process.bpmn"
<userTask id="managerApproval" name="主管审批">
  <extensionElements>
    <!--  assignee 从流程变量中获取 -->
    <activiti:formProperty id="employeeName"
      name="员工姓名"
      expression="${employee}"/>
    <activiti:formProperty id="days"
      name="请假天数"
      expression="${days}"/>
  </extensionElements>
</userTask>
```

## 执行上下文（Execution Context）

执行上下文是工作流引擎在运行时维护的「执行状态」——它包含了当前执行的位置、历史数据、以及流程追踪信息。

### 执行（Execution）

在 BPMN 中，一个执行（Execution）代表了一条执行路径。

```
┌─────────────────────────────────────────────┐
│                                             │
│  Start ──→ Task A ──→ Gateway ──→ End      │
│                                             │
└─────────────────────────────────────────────┘
                  ↑
              当前执行点
```

并发执行时，一个流程实例可能有多个执行：

```
┌─────────────────────────────────────────────┐
│                                             │
│  Start ──→ Fork ──→ Task A ─┬─→ Join ──→ End │
│                  │          └─→ Task B ─┘    │
│                  ↑                           │
│           并行执行（2 个 Execution）           │
└─────────────────────────────────────────────┘
```

```java
RuntimeService runtimeService = engine.getRuntimeService();

// 获取当前流程实例的执行列表
List<Execution> executions = runtimeService.createExecutionQuery()
    .processInstanceId(processInstanceId)
    .list();

// 获取当前活动节点 ID
String activityId = runtimeService.createExecutionQuery()
    .processInstanceId(processInstanceId)
    .singleResult()
    .getActivityId();
```

### 历史数据

工作流引擎会记录流程执行的历史：

```java
HistoryService historyService = engine.getHistoryService();

// 查询已完成的流程实例
List<HistoricProcessInstance> completedProcesses = historyService
    .createHistoricProcessInstanceQuery()
    .processDefinitionKey("leave-process")
    .finished()
    .list();

// 查询流程实例的所有历史任务
List<HistoricTaskInstance> tasks = historyService
    .createHistoricTaskInstanceQuery()
    .processInstanceId(processInstanceId)
    .list();

// 查询历史变量
List<HistoricVariableInstance> variables = historyService
    .createHistoricVariableInstanceQuery()
    .processInstanceId(processInstanceId)
    .list();
```

## 总结

工作流核心概念的关系：

```
流程定义（Blueprint）
├── 定义业务流程结构
└── 部署后创建流程定义记录

流程实例（Running Process）
├── 流程定义的运行时实例
├── 包含执行上下文和变量
└── 由开始事件触发

任务（Work Item）
├── 流程实例中的工作单元
├── 分配给用户或系统
└── 完成后推动流程流转

流程变量（Context Data）
├── 在流程实例中传递
├── 支持多种数据类型
└── 可用于条件判断
```
