# 工作流引擎

当你的系统需要处理「审批」「流转」「状态变更」这类业务流程时，你会怎么实现？

大多数人会想到：写一个状态机，加一堆 if-else，每次新增流程就改代码。短期内没问题，但当流程变复杂、业务规则变多时，代码就成了「意大利面条」——改一处可能影响另一处，测试不敢跑，上线靠祈祷。

工作流引擎，就是为了解决这个问题。

## 什么是工作流？

工作流（Workflow），简单说就是「按照预定规则流转的业务流程」。它有三个核心要素：

```
谁（Who）    →  做什么（What）  →  怎么流转（How）
参与者        任务/活动          流转规则/条件
```

### 生活中的工作流

- **请假流程**：员工提交 → 主管审批 → HR 审批 → 抄送相关人
- **订单处理**：下单 → 支付 → 发货 → 确认收货 → 评价
- **报销流程**：员工提单 → 部门领导审批 → 财务复核 → 出纳打款

这些流程的共同特点是：**步骤明确、条件分支、涉及多方参与**。

### 代码实现 vs 工作流引擎

用代码实现审批流：

```java
// 反模式：硬编码流程
public void processLeave(LeaveRequest request) {
    if (request.getDays() <= 3) {
        // 主管审批
        manager.approve(request);
    } else {
        // 主管 + HR 审批
        manager.approve(request);
        hr.approve(request);
    }
    // 继续加...
}
```

用工作流引擎：

```xml title="leave-process.bpmn"
<process id="leaveProcess">
  <startEvent id="start"/>
  <userTask id="managerApproval" assignee="${manager}"/>
  <userTask id="hrApproval" assignee="${hr}"
    activiti:skipExpression="${leaveRequest.days <= 3}"/>
  <endEvent id="end"/>
</process>
```

工作流引擎负责「流程执行」，业务代码负责「业务逻辑」，各司其职。

## BPMN 2.0 标准

BPMN（Business Process Model and Notation）是业务流程建模的标准 notation，现在已经到了 2.0 版本，被所有主流工作流引擎支持。

BPMN 2.0 的核心价值在于：**让业务流程「可视化定义、标准化执行」**。

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   ○ 开始事件                                             │
│   │                                                     │
│   ▼                                                     │
│   □ 用户任务（主管审批）                                   │
│   │                                                     │
│   ◇ 排他网关（天数 > 3？）                                │
│   ├─ 是 ─→ □ 用户任务（HR 审批）                          │
│   └─ 否 ─→ ○ 结束事件                                    │
│                │                                        │
│                ▼                                        │
│            ○ 结束事件                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 工作流引擎选型

| 引擎 | 特点 | 适用场景 |
|---|---|---|
| Flowable | 轻量、功能丰富、社区活跃 | 中型企业，内部系统 |
| Camunda | 企业级、RESTful 友好 | 需要 BPMN + DMN 的场景 |
| Activiti | 历史悠久，但已停止维护 | 遗留系统迁移 |

本模块将重点介绍 Flowable 和 Camunda。
