# Camunda vs Flowable 对比

Flowable 和 Camunda 是 Java 领域两个最流行的开源工作流引擎，它们各有特点。这一章我们深入对比两者，帮助你做出选择。

## 核心对比

| 维度 | Flowable | Camunda |
|---|---|---|
| 起源 | Activiti 6.x 分支 | Activiti 5.x 分支 |
| 引擎数量 | 6 个（BPMN、CMMN、DMN、Form、Content、App） | 1 个（核心） |
| DMN 支持 | 支持 | 完整支持 |
| CMMN 支持 | 支持 | 不支持 |
| Spring Boot | 原生友好 | 友好 |
| REST API | 需要额外配置 | 内置 |
| Web 应用 | Flowable Apps | Camunda Web Apps |
| 社区活跃度 | 活跃 | 活跃 |
| 商业支持 | Flowable 公司 | Camunda 公司 |

## 功能对比

### BPMN 支持

两者对 BPMN 2.0 的支持都很完整，都能处理：

- 用户任务、服务任务、脚本任务
- 排他网关、并行网关、包容网关、事件网关
- 子流程、调用活动
- 边界事件、计时器事件
- 多实例（会签）

### DMN 支持

| 特性 | Flowable | Camunda |
|---|---|---|
| 决策表 | 支持 | 完整支持 |
| 命中策略 | FIRST, ANY, COLLECT | 全部 8 种 |
| 表达式 | FEEL, S-FEEL | FEEL, S-FEEL |
| 与 BPMN 集成 | 支持 | 更强大 |

Camunda 的 DMN 支持更完善，特别是 `COLLECT` 策略和 `RULE_ORDER` 命中策略。

### CMMN 支持

| 特性 | Flowable | Camunda |
|---|---|---|
| CMMN 1.1 | 支持 | 不支持 |

如果你需要 Case Management（案例管理），Flowable 是唯一选择。

## 集成对比

### Spring Boot 集成

**Flowable:**

```yaml title="application.yml"
spring:
  datasource:
    url: jdbc:h2:mem:flowable
  flowable:
    async-executor-enabled: true
    database-schema-update: true
```

**Camunda:**

```yaml title="application.yml"
spring:
  datasource:
    url: jdbc:h2:mem:camunda
camunda:
  auto-deployment-enabled: true
  database-schema-update: true
```

两者对 Spring Boot 的支持都很友好，配置方式类似。

### REST API

| 维度 | Flowable | Camunda |
|---|---|---|
| 内置 | 否 | 是 |
| 独立部署 | 需要额外模块 | 支持 |
| 认证 | Spring Security | Basic Auth / OAuth2 |

Camunda 的 REST API 是内置的，开箱即用。

## 使用场景对比

### 何时选择 Flowable

```
✓ 需要 CMMN（案例管理）
✓ 需要 Form Engine（表单引擎）
✓ 需要 Content Engine（内容引擎）
✓ 需要一个套件解决所有问题
✓ 从 Activiti 迁移
```

### 何时选择 Camunda

```
✓ 主要关注 BPMN + DMN
✓ 需要完整的 REST API
✓ 企业级，稳定优先
✓ 需要商业支持
✓ 多语言客户端（Go、JavaScript）
```

## 性能对比

| 维度 | Flowable | Camunda |
|---|---|---|
| 启动速度 | 快 | 快 |
| 任务查询 | 优化 | 优化 |
| 历史数据 | 可配置 | 可配置 |
| 集群支持 | 支持 | 支持 |

两者性能相近，都适合生产环境使用。

## 学习曲线

### Flowable

- 概念多：6 个引擎，每个都有 API
- 文档完善：官方文档详细
- 示例丰富：GitHub 有大量示例

### Camunda

- 概念清晰：核心只有 BPMN + DMN
- 文档完善：官方文档详细
- 示例丰富：官方提供了完整的示例

## 社区对比

| 维度 | Flowable | Camunda |
|---|---|---|
| GitHub Stars | 10k+ | 15k+ |
| 贡献者 | 100+ | 200+ |
| 版本迭代 | 快 | 稳定 |
| 问题响应 | 快 | 快 |

两者社区都很活跃，都有商业公司支持。

## 迁移兼容性

如果你已有 Activiti 项目：

| 迁移方向 | 难度 | 说明 |
|---|---|---|
| Activiti → Flowable | 低 | 兼容性好 |
| Activiti → Camunda | 中 | 需要调整部分 API |

## 总结建议

| 场景 | 推荐 | 理由 |
|---|---|---|
| 新项目，功能全面 | Flowable | 6 个引擎，功能完整 |
| 企业级，BPMN + DMN | Camunda | DMN 支持更完善 |
| 需要 CMMN | Flowable | Camunda 不支持 |
| 需要轻量集成 | Camunda | REST API 内置 |
| 从 Activiti 迁移 | Flowable | 兼容性好 |

**没有绝对的好坏，只有适合与否。**
