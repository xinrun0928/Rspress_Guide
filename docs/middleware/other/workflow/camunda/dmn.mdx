# Camunda DMN 决策表

DMN（Decision Model and Notation）是业务决策建模的标准 notation。与 BPMN 描述「流程」不同，DMN 描述的是「决策逻辑」。

## 为什么需要 DMN？

业务系统中有很多复杂的决策逻辑，比如：

```
贷款审批：
- 如果年收入 > 50万 且 负债率 < 30% → 直接通过
- 如果年收入 20-50万 且 负债率 < 50% → 人工审核
- 如果年收入 < 20万 → 拒绝

价格计算：
- 如果是 VIP 客户 且 购买数量 > 10 → 8 折
- 如果是 VIP 客户 且 购买数量 <= 10 → 9 折
- 如果是普通客户 → 不打折
```

这些规则如果用 if-else 实现，代码会变成「意大利面条」。DMN 让你用表格的方式可视化这些规则。

## DMN 决策表结构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         DMN 决策表                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  贷款审批决策                                                       │
│  ══════════════════════════════════════════════════════════════    │
│                                                                     │
│  输入表达式                    │  输入表达式      │ 输出            │
│  ──────────────────────────────┼──────────────────┼───────────      │
│  年收入                       │  负债率          │ 审批结果         │
│  ──────────────────────────────┼──────────────────┼───────────      │
│  > 500000                     │  < 0.3           │ ✓ 直接通过      │
│  > 200000 and <= 500000       │  < 0.5           │ ⚠ 人工审核      │
│  otherwise                     │  -               │ ✗ 拒绝          │
│                                                                     │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  命中策略: FIRST | UNIQUE | ANY | COLLECT                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## DMN 文件结构

### 文件格式

```xml title="loan-decision.dmn"
<?xml version="1.0" encoding="UTF-8"?>
<definitions xmlns="https://www.omg.org/spec/DMN/20191111/MODEL/"
             xmlns:dmndi="https://www.omg.org/spec/DMN/20191111/DMNDI/"
             xmlns:dc="https://www.omg.org/spec/DMN/20180521/DC/"
             xmlns:di="https://www.omg.org/spec/DMN/20180521/DI/"
             id="definitions_loan"
             name="贷款审批决策"
             namespace="http://camunda.org/schema/1.0/dmn">

  <decision id="loan_decision" name="贷款审批决策">
    <decisionTable id="loan_decision_table" hitPolicy="FIRST">
      <!-- 输入 -->
      <input id="input_income">
        <inputExpression typeRef="number" id="inputExpression_income">
          <text>income</text>
        </inputExpression>
      </input>
      <input id="input_debt_ratio">
        <inputExpression typeRef="number" id="inputExpression_debt_ratio">
          <text>debtRatio</text>
        </inputExpression>
      </input>

      <!-- 输出 -->
      <output id="output_result" name="result" typeRef="string"/>
      <output id="output_approval_level" name="approvalLevel" typeRef="string"/>

      <!-- 规则 -->
      <rule id="rule_pass">
        <inputEntry id="inputEntry_income_1">
          <text>> 500000</text>
        </inputEntry>
        <inputEntry id="inputEntry_debt_ratio_1">
          <text>< 0.3</text>
        </inputEntry>
        <outputEntry id="outputEntry_pass">
          <text>"PASS"</text>
        </outputEntry>
        <outputEntry id="outputEntry_level_pass">
          <text>"AUTO"</text>
        </outputEntry>
      </rule>

      <rule id="rule_review">
        <inputEntry id="inputEntry_income_2">
          <text>[200000..500000]</text>
        </inputEntry>
        <inputEntry id="inputEntry_debt_ratio_2">
          <text>< 0.5</text>
        </inputEntry>
        <outputEntry id="outputEntry_review">
          <text>"REVIEW"</text>
        </outputEntry>
        <outputEntry id="outputEntry_level_review">
          <text>"MANUAL"</text>
        </outputEntry>
      </rule>

      <rule id="rule_reject">
        <inputEntry id="inputEntry_reject">
          <text>-</text>
        </inputEntry>
        <outputEntry id="outputEntry_reject">
          <text>"REJECT"</text>
        </outputEntry>
        <outputEntry id="outputEntry_level_reject">
          <text>"AUTO"</text>
        </outputEntry>
      </rule>
    </decisionTable>
  </decision>

</definitions>
```

## 命中策略（Hit Policy）

命中策略决定多条规则匹配时的行为：

| 策略 | 说明 | 适用场景 |
|---|---|---|
| FIRST | 返回第一条匹配的规则 | 优先级明确的场景 |
| UNIQUE | 所有规则互斥，不会有多条匹配 | 互斥条件 |
| ANY | 所有匹配规则必须返回相同输出 | 冗余校验 |
| COLLECT | 返回所有匹配的规则 | 需要聚合结果的场景 |

### FIRST 策略

```xml
<decisionTable id="loan_decision_table" hitPolicy="FIRST">
```

```
输入: income = 600000, debtRatio = 0.2
结果: 返回第一条匹配的规则 → PASS
```

### COLLECT 策略

```xml
<decisionTable id="discount_decision_table" hitPolicy="COLLECT">
```

```
输入: customer = VIP, isHoliday = true
结果:
  - 规则1: VIP 折扣 10%
  - 规则2: 节假日折扣 5%
  → 返回两个折扣结果
```

## 表达式语法

### 比较运算符

| 表达式 | 说明 |
|---|---|
| `> 500000` | 大于 |
| `< 0.3` | 小于 |
| `>= 100000` | 大于等于 |
| `<= 1000000` | 小于等于 |
| `[200000..500000]` | 范围（闭区间） |
| `!= 0` | 不等于 |
| `-` | 匹配任意值 |

### 逻辑组合

| 表达式 | 说明 |
|---|---|
| `> 500000, < 0.3` | AND 关系（同时满足） |
| `< 200000` or `> 1000000` | OR 关系（用 or 关键字） |
| `not(500000)` | NOT 关系 |

### 复合条件

```xml
<!-- 如果年收入 > 50万 且 负债率 < 30% -->
<inputEntry>
  <text>> 500000</text>
</inputEntry>
<inputEntry>
  <text>< 0.3</text>
</inputEntry>
```

## BPMN 集成 DMN

### 在 BPMN 中调用 DMN

```xml title="loan-process.bpmn"
<businessRuleTask id="loanApprovalDecision"
  name="贷款审批决策"
  camunda:decisionRef="loan_decision"
  camunda:resultVariable="decisionResult"/>
```

### 获取决策结果

```java
@Named
public class ProcessService {

    @Autowired
    private RuntimeService runtimeService;

    public void startLoanProcess(String customerId, BigDecimal income, BigDecimal debtRatio) {
        Map<String, Object> variables = new HashMap<>();
        variables.put("income", income);
        variables.put("debtRatio", debtRatio);
        variables.put("customerId", customerId);

        ProcessInstance instance = runtimeService.startProcessInstanceByKey(
            "loan-process", variables
        );
    }

    public void handleDecisionResult(String executionId) {
        // 从流程变量获取决策结果
        Map<String, Object> decisionResult =
            runtimeService.getVariable(executionId, "decisionResult");

        String result = (String) decisionResult.get("result");
        String approvalLevel = (String) decisionResult.get("approvalLevel");

        if ("REJECT".equals(result)) {
            // 拒绝处理
        } else if ("PASS".equals(result)) {
            // 自动通过
        } else {
            // 人工审核
        }
    }
}
```

### JavaDelegate 中调用 DMN

```java
public class LoanDecisionDelegate implements JavaDelegate {

    @Autowired
    private DmnEngine dmnEngine;

    @Override
    public void execute(DelegateExecution execution) {
        // 准备输入
        VariableMap inputs = Variables
            .createVariables()
            .putValue("income", execution.getVariable("income"))
            .putValue("debtRatio", execution.getVariable("debtRatio"));

        // 执行决策
        DmnDecisionTableResult result = dmnEngine
            .getDecisionService()
            .evaluateDecisionTableByKey("loan_decision", inputs);

        // 获取结果
        if (result.isEmpty()) {
            throw new BpmnError("NO_DECISION_RESULT", "未匹配到决策结果");
        }

        DmnDecisionResultEvaluatedEntry entry = result.getFirstResult();
        execution.setVariable("decisionResult", entry.get("result"));
        execution.setVariable("approvalLevel", entry.get("approvalLevel"));
    }
}
```

## DMN Spring Boot 集成

### 添加依赖

```xml title="pom.xml"
<dependency>
    <groupId>org.camunda.bpm</groupId>
    <artifactId>camunda-engine-dmn-bridge</artifactId>
</dependency>
```

### 配置文件

```yaml title="application.yml"
camunda:
  dmn:
    enabled: true
    # DMN 文件位置
    .definition-resources: classpath:/dmn/*.dmn
```

### 目录结构

```
src/main/resources/
├── processes/
│   └── loan-process.bpmn
└── dmn/
    ├── loan-decision.dmn
    └── discount-decision.dmn
```

## 总结

DMN 决策表核心概念：

| 概念 | 说明 |
|---|---|
| 决策表 | 核心元素，包含输入、输出、规则 |
| 输入表达式 | 条件判断表达式 |
| 输出 | 决策结果 |
| 命中策略 | 决定多条匹配时的行为 |
| 规则 | 条件 + 输出的组合 |

DMN 的价值：

- **可视化**：用表格描述业务规则，易于理解
- **可变更**：修改规则不需要改代码
- **可测试**：决策表可以独立测试
- **可复用**：同一决策表可被多个流程调用
