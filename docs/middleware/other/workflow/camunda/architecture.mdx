## 核心组件

### Process Engine

Camunda Process Engine 是 BPMN 2.0 的实现核心：

```java
ProcessEngine engine = ProcessEngines.getDefaultProcessEngine();

// 核心服务
RuntimeService runtimeService = engine.getRuntimeService();
TaskService taskService = engine.getTaskService();
RepositoryService repositoryService = engine.getRepositoryService();
HistoryService historyService = engine.getHistoryService();
IdentityService identityService = engine.getIdentityService();
ManagementService managementService = engine.getManagementService();
```

### Job Executor

Job Executor 负责异步作业的执行：

- **异步执行**：服务任务的异步执行
- **定时器**：定时事件、超时处理
- **重试机制**：失败作业自动重试

```yaml title="camunda.cfg.xml"
<process-engine-configuration>
  <jobExecutor>
    <threadPoolSettings>
      <corePoolSize>8</corePoolSize>
      <maxPoolSize>16</maxPoolSize>
      <queueSize>100</queueSize>
    </threadPoolSettings>
  </jobExecutor>
</process-engine-configuration>
```

### Persistence Layer

Camunda 使用 MyBatis 进行数据库持久化：

| 表前缀 | 说明 |
|---|---|
| `ACT_RE_*` | Repository（流程定义） |
| `ACT_RU_*` | Runtime（运行时数据） |
| `ACT_HI_*` | History（历史数据） |
| `ACT_GE_*` | General（通用数据） |
| `ACT_ID_*` | Identity（身份管理） |

## REST API

Camunda 提供完整的 REST API，可以独立部署：

```bash
# 流程定义
GET    /process-definition
GET    /process-definition/{key}
POST   /deployment/create

# 流程实例
POST   /process-instance/start
GET    /process-instance/{id}
GET    /process-instance/count

# 任务
GET    /task
POST   /task/{id}/complete
POST   /task/{id}/claim
POST   /task/{id}/delegate

# 历史
GET    /history/process-instance
GET    /history/variable-instance
```

### REST API 示例

```bash
# 启动流程
curl -X POST http://localhost:8080/engine-rest/process-definition/key/leave-process/start \
  -H "Content-Type: application/json" \
  -d '{"variables": {"employee": {"value":"张三"}, "days": {"value": 5}}}'

# 查询任务
curl http://localhost:8080/engine-rest/task?assignee=zhangsan

# 完成任务
curl -X POST http://localhost:8080/engine-rest/task/{taskId}/complete \
  -H "Content-Type: application/json" \
  -d '{"variables": {"approved": {"value": true}}}'
```

## 部署模式

### 嵌入式部署

将引擎嵌入到应用程序中：

```xml title="pom.xml"
<dependency>
    <groupId>org.camunda.bpm</groupId>
    <artifactId>camunda-engine</artifactId>
</dependency>
```

```java
ProcessEngineConfiguration config = StandaloneProcessEngineConfiguration
    .createStandaloneProcessEngineConfiguration()
    .setDataSource(myDataSource)
    .setDatabaseSchemaUpdate("true")
    .buildProcessEngine();
```

## 总结

Camunda 架构核心组件：

| 组件 | 职责 |
|---|---|
| Process Engine | 核心引擎，执行 BPMN/DMN |
| Job Executor | 异步作业和定时器执行 |
| REST API | 与外部系统集成 |
| Tasklist | 用户任务办理界面 |
| Cockpit | 流程监控运维界面 |
| Admin | 系统管理界面 |
