# Kong 插件体系与常用插件

Kong 的插件体系是其最大的竞争优势。通过插件，你可以为 Kong 添加认证、限流、日志、安全等各种能力，而且这些插件大多是开箱即用的。

## 插件架构

### 插件的工作原理

Kong 插件基于 OpenResty 的请求处理生命周期钩子实现：

```lua
-- 插件的完整生命周期钩子
plugins = {
    -- access 阶段：请求到达后、转发到上游前
    access = function(self, conf)
        -- 认证、限流检查、路由决策
    end,

    -- header_filter 阶段：上游响应 Header 到达后
    header_filter = function(self, conf)
        -- 添加/修改响应头
    end,

    -- body_filter 阶段：上游响应 Body 处理中
    body_filter = function(self, conf)
        -- 修改响应体
    end,

    -- log 阶段：请求完成后
    log = function(self, conf)
        -- 记录日志、发送指标
    end
}
```

### 插件执行顺序

同一阶段内的插件按 `priority` 排序，**数字越大越先执行**：

| Phase | 插件执行顺序 | 说明 |
|---|---|---|
| `access` | priority 从高到低 | 认证、限流通常 priority 较高 |
| `header_filter` | priority 从高到低 | Header 处理 |
| `body_filter` | priority 从高到低 | Body 转换 |
| `log` | priority 从高到低 | 日志记录 |

### 插件配置作用域

插件可以绑定到不同的作用域：

| 作用域 | 说明 | 优先级 |
|---|---|---|
| **全局插件** | 所有 Route/Service 都生效 | 最低 |
| **Service 插件** | 该 Service 下所有 Route 生效 | 中 |
| **Route 插件** | 仅该 Route 生效 | 最高 |
| **Consumer 插件** | 指定 Consumer 生效 | 最高 |

```
优先级：Route > Service > Consumer > Global
```

## 常用插件

### 认证插件

**JWT 插件**

```bash
# 启用 JWT 插件
curl -X POST http://localhost:8001/plugins/ \
    --data "name=jwt" \
    --data "config.secret_is_base64=false" \
    --data "config.claims_to_verify=exp"
```

创建 Consumer 和 JWT 凭证：

```bash
# 创建 Consumer
curl -X POST http://localhost:8001/consumers/ \
    --data "username=user-1"

# 创建 JWT 凭证
curl -X POST http://localhost:8001/consumers/user-1/jwt \
    --data "algorithm=HS256" \
    --data "key=user-1-key" \
    --data "secret=my-secret-key"
```

请求示例：

```bash
# 带 JWT Token 的请求
curl http://localhost:8000/api/users \
    -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

**Key Authentication 插件**

```bash
# 启用 Key Auth 插件
curl -X POST http://localhost:8001/plugins/ \
    --data "name=key-auth" \
    --data "config.key_names=apikey,X-API-Key"

# 为 Consumer 创建 API Key
curl -X POST http://localhost:8001/consumers/user-1/key-auth \
    --data "key=my-api-key-12345"
```

### 限流插件

**Rate Limiting 插件**

```bash
# 启用限流插件
curl -X POST http://localhost:8001/plugins/ \
    --data "name=rate-limiting" \
    --data "config.minute=100" \
    --data "config.hour=1000" \
    --data "config.policy=local"  # 或 redis/cluster
```

使用 Redis 实现分布式限流：

```bash
curl -X POST http://localhost:8001/plugins/ \
    --data "name=rate-limiting" \
    --data "config.minute=100" \
    --data "config.policy=redis" \
    --data "config.redis_host=127.0.0.1" \
    --data "config.redis_port=6379"
```

### 安全插件

**CORS 插件**

```bash
curl -X POST http://localhost:8001/plugins/ \
    --data "name=cors" \
    --data "config.origins=*" \
    --data "config.methods=GET,POST,PUT,DELETE" \
    --data "config.headers=Content-Type,Authorization" \
    --data "config.exposed_headers=X-Total-Count" \
    --data "config.credentials=true" \
    --data "config.max_age=3600"
```

**IP Restriction 插件**

```bash
# 只允许特定 IP 访问
curl -X POST http://localhost:8001/plugins/ \
    --data "name=ip-restriction" \
    --data "config.allow=192.168.1.0/24" \
    --data "config.allow=10.0.0.0/8" \
    --data "config.deny=192.168.1.100"  # 排除特定 IP
```

### 日志插件

**Logger 插件**

```bash
# 启用日志插件（输出到文件）
curl -X POST http://localhost:8001/plugins/ \
    --data "name=file-log" \
    --data "config.path=/var/log/kong/access.log" \
    --data "config.reopen=true"

# 启用 HTTP 日志（发送到外部服务）
curl -X POST http://localhost:8001/plugins/ \
    --data "name=http-log" \
    --data "config.http_endpoint=http://log-collector:8080/logs" \
    --data "config.method=POST" \
    --data "config.content_type=application/json"
```

**Prometheus 插件**

```bash
# 启用 Prometheus 指标插件
curl -X POST http://localhost:8001/plugins/ \
    --data "name=prometheus"
```

Prometheus 端点：

```bash
curl http://localhost:8001/metrics
```

暴露的指标包括请求数、延迟、带宽等。

## 插件配置示例

### 完整配置示例

```bash
# 1. 创建 Service
curl -X POST http://localhost:8001/services/ \
    --data "name=my-api" \
    --data "url=http://backend-service:8080"

# 2. 创建 Route
curl -X POST http://localhost:8001/services/my-api/routes/ \
    --data "name=my-route" \
    --data "paths[]=/api" \
    --data "strip_path=true"

# 3. 启用插件链
curl -X POST http://localhost:8001/routes/my-route/plugins/ \
    --data "name=jwt"

curl -X POST http://localhost:8001/routes/my-route/plugins/ \
    --data "name=rate-limiting" \
    --data "config.minute=100"

curl -X POST http://localhost:8001/routes/my-route/plugins/ \
    --data "name=cors" \
    --data "config.origins=*"

curl -X POST http://localhost:8001/routes/my-route/plugins/ \
    --data "name=ip-restriction" \
    --data "config.allow=0.0.0.0/0"
```

### 使用 declarative configuration

```yaml
# kong.yml
_format_version: "3.0"

services:
  - name: my-api
    url: http://backend-service:8080
    routes:
      - name: my-route
        paths:
          - /api
        strip_path: true
    plugins:
      - name: rate-limiting
        config:
          minute: 100
          hour: 1000
          policy: redis
          redis_host: 127.0.0.1
          redis_port: 6379

routes:
  - name: my-route
    service: my-api
    plugins:
      - name: jwt
        config:
          secret_is_base64: false
          claims_to_verify:
            - exp
      - name: cors
        config:
          origins:
            - "*"
          methods:
            - GET
            - POST
            - PUT
            - DELETE
          headers:
            - Content-Type
            - Authorization
          credentials: true
          max_age: 3600

consumers:
  - username: user-1
    plugins:
      - name: key-auth
        config:
          key: user-1-api-key
```

## 自定义插件开发

### 插件结构

Kong 插件是一个包含特定文件结构的 Lua 包：

```
kong/plugins/my-plugin/
├── handler.lua      # 插件主逻辑
├── schema.lua       # 配置校验
└── migrations/      # 数据库迁移（可选）
    └── 000_base_my_plugin.lua
```

### handler.lua

```lua
-- kong/plugins/my-plugin/handler.lua
local MyPlugin = {
    PRIORITY = 1000,  -- 优先级，数字越大越先执行
    VERSION = "1.0.0",
}

-- access 阶段
function MyPlugin:access(conf)
    kong.log.debug("MyPlugin: access phase")

    -- 获取请求信息
    local method = kong.request.get_method()
    local path = kong.request.get_path()

    -- 添加请求头
    kong.service.request.set_header("X-Processed-By", "my-plugin")

    -- 修改请求路径
    kong.service.request.set_path("/new" .. path)

    -- 调用下一个插件或继续处理
    -- （不调用 kong.service.go()，则继续 Kong 内部处理）
end

-- header_filter 阶段
function MyPlugin:header_filter(conf)
    kong.log.debug("MyPlugin: header_filter phase")

    -- 添加响应头
    kong.response.set_header("X-My-Plugin", "processed")
end

-- body_filter 阶段
function MyPlugin:body_filter(conf)
    kong.log.debug("MyPlugin: body_filter phase")

    -- 检查是否有响应体
    local chunk = kong.response.get_chunk()
    if chunk then
        -- 修改响应体
        local modified = string.gsub(chunk, "old", "new", 1)
        kong.response.set_chunk(modified)
    end
end

-- log 阶段
function MyPlugin:log(conf)
    kong.log.inspect(conf)  -- 记录配置

    -- 异步发送日志
    local ok, err = kong.log.notice("Request completed")
end

return MyPlugin
```

### schema.lua

```lua
-- kong/plugins/my-plugin/schema.lua
local typedefs = require "kong.db.schema.typedefs"

return {
    name = "my-plugin",
    fields = {
        { consumer = typedefs.no_consumer },  -- 是否绑定 Consumer
        { config = {
            type = "record",
            fields = {
                { enabled = { type = "boolean", default = true } },
                { timeout = { type = "number", default = 1000 } },
                { header_name = { type = "string", default = "X-Custom-Header" } },
            },
        }, },
    },
}
```

### 安装和使用

```bash
# 1. 将插件目录放到 Kong 插件路径
cp -r kong/plugins/my-plugin /usr/local/share/lua/5.1/kong/plugins/

# 2. 在 kong.yml 中启用插件
plugins:
  - name: my-plugin
    config:
      enabled: true
      timeout: 2000

# 3. 重启 Kong
kong restart --conf kong.yml
```

## 总结

| 插件类型 | 功能 | 常用插件 |
|---|---|---|
| 认证 | 验证请求合法性 | jwt, key-auth, basic-auth, oauth2 |
| 限流 | 控制请求速率 | rate-limiting, response-ratelimiting |
| 安全 | 保护 API 安全 | cors, ip-restriction, acl |
| 日志 | 记录请求信息 | file-log, http-log, prometheus, syslog |
| 转换 | 修改请求/响应 | request-transformer, response-transformer |
| 协议 | 协议转换 | grpc-gateway, http2rpc |

Kong 的插件体系非常丰富，社区插件和官方插件加起来超过 100 个。在选择插件时，建议：

1. **优先使用官方插件**：经过充分测试，稳定可靠
2. **社区插件要评估质量**：查看 GitHub stars、更新频率、issue 反馈
3. **复杂需求考虑自研**：基于 Kong 插件开发规范，开发满足特定需求的插件
