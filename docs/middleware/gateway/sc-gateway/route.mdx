# Spring Cloud Gateway 路由配置

路由是 Gateway 的核心配置单元。掌握静态路由和动态路由的配置方式，是使用 Gateway 的基本功。

## 基础路由配置

### 路由结构

一个完整的路由配置包含以下部分：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service       # 路由 ID，唯一标识
          uri: lb://user-service # 目标 URI，lb:// 表示负载均衡
          predicates:            # 谓词数组，所有谓词都满足才匹配
            - Path=/api/users/**
          filters:               # 过滤器数组
            - StripPrefix=1
          metadata:              # 元数据，可用于自定义逻辑
            weight: high
```

### URI 的两种形式

| URI 格式 | 说明 | 示例 |
|---|---|---|
| `http://host:port` | 固定地址，不做负载均衡 | `http://192.168.1.100:8080` |
| `lb://service-name` | 服务名，通过负载均衡器解析 | `lb://user-service` |

`lb://` 格式需要配合 Spring Cloud LoadBalancer 使用，会从服务发现（Nacos/Consul/Eureka）中获取服务实例列表。

## 静态路由配置

静态路由是最常见的配置方式，所有路由规则在启动时加载。

### Path 路由

最基础的路径路由：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-route
          uri: http://user-service:8080
          predicates:
            - Path=/users/**
```

### 路径重写

使用 `RewritePath` 过滤器重写路径：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: rewrite-route
          uri: http://backend-service
          predicates:
            - Path=/api/v1/**
          filters:
            - RewritePath=/api/v1(?<segment>/?.*), $\{segment}
```

| 请求路径 | 实际转发路径 |
|---|---|
| `/api/v1/users` | `/users` |
| `/api/v1/orders/123` | `/orders/123` |

### 多个 Predicate 组合

多个 Predicate 是 AND 关系，必须同时满足：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: combined-route
          uri: http://backend
          predicates:
            - Path=/api/admin/**
            - Method=GET,POST          # GET 或 POST 方法
            - Header=X-Token, .+       # 必须有 X-Token 头
            - Query=version, v[0-9]+   # version 参数必须是 v 开头加数字
```

### 动态路由路径变量

使用 `{变量名}` 捕获路径中的部分内容：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-detail-route
          uri: http://user-service:8080
          predicates:
            - Path=/api/users/{userId}
```

转发的请求 URL 中，`{userId}` 会被实际值替换：

| 请求路径 | 转发 URL |
|---|---|
| `/api/users/123` | `http://user-service:8080/api/users/123` |
| `/api/users/abc` | `http://user-service:8080/api/users/abc` |

## Header 路由

根据请求头进行路由，适合灰度发布、多租户等场景。

### 基本用法

```yaml
spring:
  cloud:
    gateway:
      routes:
        # 移动端用户路由到 mobile-backend
        - id: mobile-route
          uri: http://mobile-backend
          predicates:
            - Header=X-Client-Type, mobile

        # 管理员路由到 admin-backend
        - id: admin-route
          uri: http://admin-backend
          predicates:
            - Header=X-User-Role, admin
```

### 正则匹配

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: versioned-route
          uri: http://v2-backend
          predicates:
            # X-API-Version 头匹配 v2 或 v3
            - Header=X-API-Version, v[23]
```

## Query 参数路由

根据 URL 查询参数进行路由。

### 基本用法

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: v2-api-route
          uri: http://v2-backend
          predicates:
            - Query=api_version, v2
```

### 带默认值的查询

如果需要可选的查询参数，可以结合多个 Predicate：

```yaml
spring:
  cloud:
    gateway:
      routes:
        # 有 version 参数时路由到对应版本
        - id: v2-with-param
          uri: http://v2-backend
          predicates:
            - Path=/api/**
            - Query=version, v2

        # 无 version 参数或非 v2 时路由到默认版本
        - id: default-route
          uri: http://default-backend
          predicates:
            - Path=/api/**
```

## 动态路由

静态路由的缺点是修改配置后需要重启 Gateway。动态路由通过外部配置源实现运行时更新。

### 基于 Nacos 的动态路由

```yaml
spring:
  cloud:
    nacos:
      server-addr: 127.0.0.1:8848
    gateway:
      discovery:
        locator:
          enabled: true  # 开启服务发现路由
      # 从 Nacos 读取路由配置
      routes-source: nacos
```

在 Nacos 配置中心添加路由配置（Data ID: `gateway-router.json`）：

```json
[
  {
    "id": "user-service-dynamic",
    "uri": "lb://user-service",
    "predicates": [
      {
        "name": "Path",
        "args": {
          "pattern": "/dynamic/users/**"
        }
      }
    ],
    "filters": [
      {
        "name": "StripPrefix",
        "args": {
          "_genkey_0": "1"
        }
      }
    ]
  }
]
```

### 基于 Redis 的动态路由

```yaml
spring:
  cloud:
    gateway:
      routes:  # 静态路由
        - id: static-route
          uri: http://static-backend
          predicates:
            - Path=/static/**

      redis:
        route:
          enabled: true  # 开启 Redis 路由源
```

通过 Redis 发布路由变更：

```bash
# 发布路由配置到 Redis
redis-cli > PUBLISH spring.cloud.gateway.routes '{"id":"dynamic-route","uri":"lb://dynamic-service","predicates":[{"name":"Path","args":{"pattern":"/dynamic/**"}}]}'
```

### 自定义路由定义源

如果现有的动态路由方式不满足需求，可以自定义 `RouteDefinitionRepository`：

```java
@Component
public class CustomRouteDefinitionRepository implements RouteDefinitionRepository {

    private final Map<String, RouteDefinition> routes = new ConcurrentHashMap<>();

    @Override
    public Flux<RouteDefinition> getRouteDefinitions() {
        return Flux.fromIterable(routes.values());
    }

    @Override
    public Mono<RouteDefinition> save(Mono<RouteDefinition> route) {
        return route.flatMap(def -> {
            routes.put(def.getId(), def);
            return Mono.just(def);
        });
    }

    @Override
    public Mono<Void> delete(Mono<String> routeId) {
        return routeId.flatMap(id -> {
            routes.remove(id);
            return Mono.empty();
        });
    }
}
```

## 服务发现路由

Gateway 可以自动将从服务发现中心获取的服务实例注册为路由。

### 开启服务发现路由

```yaml
spring:
  cloud:
    gateway:
      discovery:
        locator:
          enabled: true  # 开启服务发现路由
          lower-case-service-id: true  # 服务 ID 转小写
          url-expression: "'lb://' + serviceId"  # URI 表达式
```

开启后，Gateway 会自动为每个注册的服务创建路由：

| 服务名 | 自动路由 | 实际路由 |
|---|---|---|
| `User-Service` | `/user-service/**` | `lb://User-Service/**` |
| `Order-Service` | `/order-service/**` | `lb://Order-Service/**` |

### 路由路径重写

自动生成的路径会包含服务名，通常需要重写去掉：

```yaml
spring:
  cloud:
    gateway:
      discovery:
        locator:
          enabled: true
          predicates:
            - name: Path
              args:
                pattern: "'/'+serviceId+'/**'"
          filters:
            - name: RewritePath
              args:
                regexp: "'/' + serviceId + '/(?<remaining>.*)'"
                replacement: "'/${remaining}'"
```

| 原始请求 | 路由到 |
|---|---|
| `/user-service/users/123` | `lb://user-service/users/123` |
| `/order-service/orders` | `lb://order-service/orders` |

## 路由优先级

当多个路由匹配同一个请求时，使用 **最具体优先** 的策略：

| 优先级规则 | 示例 |
|---|---|
| 精确匹配优先于前缀匹配 | `/api/users` 优先于 `/api/users/**` |
| 前缀路径越短优先级越高 | `/api` 优先于 `/api/usesrs` |
| 配置顺序（可配置） | 相同条件下，配置在前的优先 |

### 显式指定优先级

通过 `order` 属性显式指定路由顺序：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: high-priority
          uri: http://high-backend
          predicates:
            - Path=/admin/**
          order: -1  # 数值越小，优先级越高

        - id: normal-priority
          uri: http://normal-backend
          predicates:
            - Path=/admin/**
          order: 0
```

## 常见配置示例

### 前后端分离场景

```yaml
spring:
  cloud:
    gateway:
      routes:
        # API 服务
        - id: api-route
          uri: lb://api-service
          predicates:
            - Path=/api/**
          filters:
            - StripPrefix=1
            - AddRequestHeader=X-Gateway, gateway
            - AddResponseHeader=X-Response-Time, #{/executionTime}

        # 静态资源
        - id: static-route
          uri: http://static-server:8080
          predicates:
            - Path=/static/**
            - -Query=debug, true  # 排除 debug 参数的请求
```

### 微服务聚合场景

```yaml
spring:
  cloud:
    gateway:
      routes:
        # 用户服务
        - id: user-route
          uri: lb://user-service
          predicates:
            - Path=/gateway/users/**
          filters:
            - StripPrefix=1

        # 订单服务
        - id: order-route
          uri: lb://order-service
          predicates:
            - Path=/gateway/orders/**
          filters:
            - StripPrefix=1

        # 商品服务
        - id: product-route
          uri: lb://product-service
          predicates:
            - Path=/gateway/products/**
          filters:
            - StripPrefix=1
```

## 总结

| 配置类型 | 说明 | 适用场景 |
|---|---|---|
| 静态路由 | 配置文件定义，启动加载 | 大部分场景 |
| 动态路由 | 运行时可更新，无需重启 | 需要频繁变更的场景 |
| 服务发现路由 | 自动为注册服务创建路由 | 微服务架构 |
| 组合路由 | 多个 Predicate 组合 | 复杂匹配规则 |

路由配置的关键点：

1. **URI 格式**：固定地址用 `http://`，服务名用 `lb://`
2. **Predicate 组合**：多个 Predicate 是 AND 关系
3. **过滤器顺序**：`StripPrefix` 通常放在重写类过滤器之后
4. **动态路由**：根据团队技术栈选择 Nacos/Redis/Custom 实现
