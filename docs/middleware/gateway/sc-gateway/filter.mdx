# Spring Cloud Gateway 过滤器

过滤器是 Gateway 最强大的扩展机制。通过过滤器，你可以在请求到达后端服务之前、响应返回给客户端之前，做任何自定义处理。

Spring Cloud Gateway 提供两种过滤器：**全局过滤器（GlobalFilter）** 和 **路由过滤器（GatewayFilter）**。

## 全局过滤器 vs 路由过滤器

| 维度 | GlobalFilter | GatewayFilter |
|---|---|---|
| 作用范围 | 所有路由生效 | 仅对配置的路由生效 |
| 配置方式 | `@Component` + 实现 `GlobalFilter` 接口 | YAML 配置或 `GatewayFilterFactory` |
| 执行时机 | 在所有路由过滤器之前 | 在全局过滤器之后 |
| 典型用途 | 认证、限流、日志 | 请求转换、Header 处理 |

## 全局过滤器

### 执行顺序

全局过滤器通过实现 `Ordered` 接口或 `getOrder()` 方法来控制执行顺序。**数字越小，执行越早**。

```java
@Component
public class MyGlobalFilter implements GlobalFilter, Ordered {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, FilterChain chain) {
        // 过滤逻辑
        return chain.filter(exchange);
    }

    @Override
    public int getOrder() {
        return -100;  // 优先级，数值越小越先执行
    }
}
```

常见内置全局过滤器的顺序：

| 过滤器 | Order | 作用 |
|---|---|---|
| NettyRoutingFilter | -1 | HTTP 客户端路由 |
| NettyWriteResponseFilter | -1 | 响应写入 Netty |
| WebClientHttpRoutingFilter | -3 | WebClient 路由 |
| LoadBalancerClientFilter | 101 | 负载均衡客户端 |

### 常用全局过滤器实现

**认证过滤器**

```java
@Component
public class JwtAuthenticationFilter implements GlobalFilter, Ordered {

    private static final Logger log = LoggerFactory.getLogger(JwtAuthenticationFilter.class);

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, FilterChain chain) {
        String path = exchange.getRequest().getPath().value();

        // 放行公开路径
        if (isPublicPath(path)) {
            return chain.filter(exchange);
        }

        String authHeader = exchange.getRequest().getHeaders().getFirst("Authorization");

        if (StringUtils.isBlank(authHeader) || !authHeader.startsWith("Bearer ")) {
            log.warn("Unauthorized request to: {}", path);
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete();
        }

        String token = authHeader.substring(7);

        return verifyToken(token)
            .flatMap(claims -> {
                // 将用户信息存入上下文
                exchange.getAttributes().put("userId", claims.getUserId());
                exchange.getAttributes().put("username", claims.getUsername());
                return chain.filter(exchange);
            })
            .onErrorResume(e -> {
                log.error("Token verification failed: {}", e.getMessage());
                exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
                return exchange.getResponse().setComplete();
            });
    }

    @Override
    public int getOrder() {
        return -10;  // 在路由过滤器之前执行
    }

    private boolean isPublicPath(String path) {
        return path.startsWith("/api/auth/") || path.startsWith("/api/public/");
    }

    private Mono<TokenClaims> verifyToken(String token) {
        // JWT 验证逻辑
        return Mono.justOrEmpty(jwtService.verify(token));
    }
}
```

**请求日志过滤器**

```java
@Component
public class RequestLoggingFilter implements GlobalFilter, Ordered {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, FilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();

        // 记录请求开始时间
        long startTime = System.currentTimeMillis();
        exchange.getAttributes().put("startTime", startTime);

        // 打印请求信息
        String method = request.getMethod().name();
        String path = request.getPath().value();
        String clientIp = getClientIp(request);

        log.info("==> {} {} from {}", method, path, clientIp);

        return chain.filter(exchange)
            .then(Mono.fromRunnable(() -> {
                // Post 逻辑：记录响应信息
                long duration = System.currentTimeMillis() - startTime;
                int status = exchange.getResponse().getStatusCode() != null
                    ? exchange.getResponse().getStatusCode().value()
                    : 0;
                log.info("<== {} {} {} ({}ms)", method, path, status, duration);
            }));
    }

    @Override
    public int getOrder() {
        return -1000;  // 最早执行，记录所有请求
    }

    private String getClientIp(ServerHttpRequest request) {
        String xForwardedFor = request.getHeaders().getFirst("X-Forwarded-For");
        if (StringUtils.hasText(xForwardedFor)) {
            return xForwardedFor.split(",")[0].trim();
        }
        return request.getRemoteAddress() != null
            ? request.getRemoteAddress().getAddress().getHostAddress()
            : "unknown";
    }
}
```

## 路由过滤器

路由过滤器只对特定路由生效，通过 YAML 配置或自定义 `GatewayFilterFactory` 实现。

### 内置过滤器

Gateway 内置了大量实用的路由过滤器：

| 过滤器 | 作用 | 示例 |
|---|---|---|
| `AddRequestHeader` | 添加请求头 | `AddRequestHeader=X-Request-Time, now` |
| `RemoveRequestHeader` | 移除请求头 | `RemoveRequestHeader=X-Debug` |
| `AddResponseHeader` | 添加响应头 | `AddResponseHeader=X-Gateway, SCG` |
| `RemoveResponseHeader` | 移除响应头 | `RemoveResponseHeader=Server` |
| `RewritePath` | 重写请求路径 | `RewritePath=/api/(?<segment>.*), /$\{segment}` |
| `StripPrefix` | 去掉路径前缀 | `StripPrefix=1` |
| `SetPath` | 设置路径 | `SetPath=/{segment}` |
| `SetRequestHeader` | 设置请求头（存在则覆盖） | `SetRequestHeader=X-Source, gateway` |
| `PrefixPath` | 添加路径前缀 | `PrefixPath=/api` |
| `RedirectTo` | 重定向 | `RedirectTo=302, https://example.com` |
| `SetStatus` | 设置响应状态码 | `SetStatus=401` |
| `RequestSize` | 限制请求大小 | `RequestSize=5000000` |
| `Retry` | 重试机制 | `Retry=3` |

### 过滤器使用示例

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service-route
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          filters:
            # 去掉第一层路径前缀
            - StripPrefix=1
            # 添加请求头
            - AddRequestHeader=X-Gateway, gateway
            - AddRequestHeader=X-Request-Id, #{T(java.util.UUID).randomUUID()}
            # 移除敏感响应头
            - RemoveResponseHeader=Server
            - RemoveResponseHeader=Date
            # 重试机制
            - name: Retry
              args:
                retries: 3
                statuses: SERVICE_UNAVAILABLE,GATEWAY_TIMEOUT
                methods: GET,POST
                exceptions: java.io.IOException,java.util.concurrent.TimeoutException
```

### 自定义 GatewayFilterFactory

如果内置过滤器不满足需求，可以自定义 `GatewayFilterFactory`：

**请求体添加字段**

```java
@Component
public class AddRequestBodyGatewayFilterFactory
        extends AbstractGatewayFilterFactory<AddRequestBodyGatewayFilterFactory.Config> {

    public AddRequestBodyGatewayFilterFactory() {
        super(Config.class);
    }

    @Override
    public GatewayFilter apply(Config config) {
        return (exchange, chain) -> {
            if (!config.isEnabled()) {
                return chain.filter(exchange);
            }

            return DataBufferUtils.join(exchange.getRequest().getBody())
                .flatMap(dataBuffer -> {
                    ByteBuffer byteBuffer = dataBuffer.toByteBuffer();
                    byte[] bytes = new byte[byteBuffer.remaining()];
                    byteBuffer.get(bytes);
                    DataBufferUtils.release(dataBuffer);

                    String body = new String(bytes, StandardCharsets.UTF_8);

                    // 添加字段
                    try {
                        JsonNode jsonNode = new ObjectMapper().readTree(body);
                        ((ObjectNode) jsonNode).put(config.getFieldName(), config.getFieldValue());
                        String modifiedBody = new ObjectMapper().writeValueAsString(jsonNode);

                        DataBufferFactory bufferFactory = exchange.getResponse().bufferFactory();
                        DataBuffer newBuffer = bufferFactory.wrap(modifiedBody.getBytes(StandardCharsets.UTF_8));

                        ServerHttpRequest mutatedRequest = new ServerHttpRequestDecorator(exchange.getRequest()) {
                            @Override
                            public Flux<DataBuffer> getBody() {
                                return Flux.just(newBuffer);
                            }
                        };

                        return chain.filter(exchange.mutate().request(mutatedRequest).build());
                    } catch (JsonProcessingException e) {
                        return Mono.error(new RuntimeException("Failed to parse JSON", e));
                    }
                });
        };
    }

    public static class Config {
        private boolean enabled = true;
        private String fieldName;
        private String fieldValue;

        public boolean isEnabled() { return enabled; }
        public void setEnabled(boolean enabled) { this.enabled = enabled; }
        public String getFieldName() { return fieldName; }
        public void setFieldName(String fieldName) { this.fieldName = fieldName; }
        public String getFieldValue() { return fieldValue; }
        public void setFieldValue(String fieldValue) { this.fieldValue = fieldValue; }
    }
}
```

使用方式：

```yaml
filters:
  - name: AddRequestBody
    args:
      enabled: true
      fieldName: gateway_processed
      fieldValue: "true"
```

## 过滤器组合使用

过滤器按配置顺序执行，后续过滤器可以依赖前面过滤器的处理结果。

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: full-process-route
          uri: lb://backend-service
          predicates:
            - Path=/api/**
          filters:
            # 1. 认证检查（全局过滤器，这里列出会再执行一次）
            # 2. 去掉前缀
            - StripPrefix=1
            # 3. 添加 Header
            - AddRequestHeader=X-Processed-By, gateway
            - AddRequestHeader=X-Request-Time, #{T(System).currentTimeMillis()}
            # 4. 重写路径
            - RewritePath=/api/(?<segment>.*), /$\{segment}
            # 5. 限流（通过 bean 名称引用）
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 100
                redis-rate-limiter.burstCapacity: 200
            # 6. 熔断
            - name: CircuitBreaker
              args:
                name: backendCircuitBreaker
                fallbackUri: forward:/fallback
```

## 过滤器开发最佳实践

| 原则 | 说明 |
|---|---|
| **快速失败** | 能在前置过滤器判断的（如认证失败），不要留到后置 |
| **避免阻塞** | 过滤器中不要使用阻塞 I/O，会阻塞 EventLoop |
| **异常处理** | 使用 `onErrorResume` 兜底，避免异常穿透 |
| **上下文共享** | 使用 `exchange.getAttributes()` 在过滤器间共享数据 |
| **幂等性** | 同一个过滤器的多次执行不应该产生副作用 |
| **可配置** | 将阈值、超时等参数通过配置外部化 |

## 总结

| 类型 | 特点 | 使用场景 |
|---|---|---|
| GlobalFilter | 所有路由生效，实现接口 | 认证、限流、全局日志 |
| GatewayFilter | 仅特定路由生效，YAML 配置 | 请求转换、Header 处理 |
| GatewayFilterFactory | 自定义路由过滤器 | 复杂业务逻辑 |

过滤器链的执行顺序：
```
请求 → GlobalFilter (按 order 升序) → GatewayFilter (按配置顺序) → 后端 → 响应
```

掌握过滤器的设计，你就能将 Gateway 打造成符合业务需求的强大网关。
