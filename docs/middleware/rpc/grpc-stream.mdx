# gRPC 流式调用

传统的 RPC 调用是一元调用（Unary Call）：客户端发一个请求，服务端返回一个响应，一问一答，简单明了。

但现实中的业务场景往往更复杂：

- **文件上传**：客户端需要发送大量数据，不可能一次性全部加载到内存
- **实时日志**：服务端需要持续推送日志，客户端需要实时接收
- **数据推送**：服务端需要向客户端推送消息，而不是等待客户端请求

这些场景下，**流式调用**就派上用场了。

## 四种调用模式

gRPC 支持四种调用模式：

| 模式 | 描述 | 典型场景 |
|---|---|---|
| **Unary** | 一元调用，1 请求 - 1 响应 | 普通 API 调用 |
| **Server Streaming** | 服务端流，1 请求 - N 响应 | 实时推送、文件下载 |
| **Client Streaming** | 客户端流，N 请求 - 1 响应 | 文件上传、批量处理 |
| **Bidirectional Streaming** | 双向流，N 请求 - N 响应 | 聊天、实时交互 |

```
┌─────────────────────────────────────────────────────────────────┐
│                        gRPC 四种调用模式                         │
│                                                                  │
│   Unary                        Server Streaming                   │
│   ────►                    ────► ────► ────►                    │
│   ◄────                      ◄──── ◄──── ◄────                    │
│                               (服务端持续推送)                     │
│                                                                  │
│   Client Streaming            Bidirectional Streaming             │
│   ────► ────► ────►          ◄───► ◄───► ◄───►                  │
│   ◄────                        ◄───► ◄───► ◄───►                  │
│   (客户端分批发送)                (双方持续交互)                     │
└─────────────────────────────────────────────────────────────────┘
```

## Protobuf 定义

首先定义 `.proto` 文件，声明各种流式服务：

```protobuf
syntax = "proto3";

package chat;

option java_multiple_files = true;
option java_package = "com.example.chat";

// 聊天消息
message ChatMessage {
    string sender = 1;
    string content = 2;
    int64 timestamp = 3;
}

// 发送请求
message SendMessageRequest {
    string content = 1;
}

// 发送响应
message SendMessageResponse {
    bool success = 1;
    string message_id = 2;
}

// 订阅请求（空消息）
message SubscribeRequest {
    string room_id = 1;
}

// 聊天服务定义
service ChatService {
    // 一元调用：发送消息
    rpc SendMessage(SendMessageRequest) returns (SendMessageResponse);

    // 服务端流：订阅聊天室（接收消息）
    rpc Subscribe(SubscribeRequest) returns (stream ChatMessage);

    // 客户端流：批量发送消息
    rpc BatchSendMessage(stream SendMessageRequest) returns (SendMessageResponse);

    // 双向流：聊天室
    rpc Chat(stream ChatMessage) returns (stream ChatMessage);
}
```

## 一元调用：最基础的模式

### 服务端实现

```java
public class ChatServiceImpl extends ChatServiceGrpc.ChatServiceImplBase {

    @Override
    public void sendMessage(
            SendMessageRequest request,
            StreamObserver<SendMessageResponse> responseObserver) {

        // 1. 业务处理
        String content = request.getContent();
        String messageId = generateMessageId();

        // 2. 构建响应
        SendMessageResponse response = SendMessageResponse.newBuilder()
                .setSuccess(true)
                .setMessageId(messageId)
                .build();

        // 3. 发送响应
        responseObserver.onNext(response);
        responseObserver.onCompleted();
    }
}
```

### 客户端调用

```java
public class ChatClient {

    private final ManagedChannel channel;
    private final ChatServiceGrpc.ChatServiceBlockingStub blockingStub;

    public ChatClient(String host, int port) {
        this.channel = ManagedChannelBuilder
                .forAddress(host, port)
                .usePlaintext()
                .build();
        this.blockingStub = ChatServiceGrpc.newBlockingStub(channel);
    }

    public String sendMessage(String content) {
        SendMessageRequest request = SendMessageRequest.newBuilder()
                .setContent(content)
                .build();

        SendMessageResponse response = blockingStub.sendMessage(request);

        if (response.getSuccess()) {
            return response.getMessageId();
        } else {
            throw new RuntimeException("发送消息失败");
        }
    }

    public void shutdown() throws InterruptedException {
        channel.shutdown().awaitTermination(5, TimeUnit.SECONDS);
    }
}
```

一元调用的特点是**阻塞式**，客户端会等待服务端返回后再继续执行。

## 服务端流：Server Streaming

服务端流是**一个请求，服务端分多次返回**。常用于实时推送、文件下载等场景。

### 服务端实现

```java
public class ChatServiceImpl extends ChatServiceGrpc.ChatServiceImplBase {

    private final ConcurrentMap<String, StreamObserver<ChatMessage>> subscribers =
            new ConcurrentHashMap<>();

    @Override
    public void subscribe(
            SubscribeRequest request,
            StreamObserver<ChatMessage> responseObserver) {

        String roomId = request.getRoomId();
        subscribers.put(roomId, responseObserver);

        try {
            // 模拟：持续接收消息直到客户端断开
            while (true) {
                // 这里可以用阻塞队列或 CompletableFuture 接收消息
                // 简化示例，实际应该用消息队列
                Thread.sleep(1000);
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            subscribers.remove(roomId);
            responseObserver.onCompleted();
        }
    }

    // 广播消息到订阅者
    public void broadcastToRoom(String roomId, ChatMessage message) {
        StreamObserver<ChatMessage> subscriber = subscribers.get(roomId);
        if (subscriber != null) {
            subscriber.onNext(message);
        }
    }
}
```

### 客户端调用

```java
public class ChatClient {

    private final ChatServiceGrpc.ChatServiceBlockingStub blockingStub;

    // 服务端流调用示例
    public void subscribeRoom(String roomId) {
        SubscribeRequest request = SubscribeRequest.newBuilder()
                .setRoomId(roomId)
                .build();

        // 服务端流返回的是 Iterator
        Iterator<ChatMessage> stream = chatStub.subscribe(request);

        // 持续读取服务端推送的消息
        while (stream.hasNext()) {
            ChatMessage message = stream.next();
            System.out.printf("[%s] %s: %s%n",
                    LocalDateTime.now(),
                    message.getSender(),
                    message.getContent());
        }

        // 流结束
        System.out.println("订阅结束");
    }
}
```

### 典型应用场景

1. **实时日志推送**：服务器主动推送日志到客户端
2. **股票行情推送**：实时推送股价、汇率等数据
3. **文件下载**：服务端分块发送文件，客户端分块接收
4. **智能客服**：AI 分段返回回复内容

## 客户端流：Client Streaming

客户端流是**客户端分多次发送，服务端返回一次**。常用于文件上传、批量处理等场景。

### 服务端实现

```java
public class ChatServiceImpl extends ChatServiceGrpc.ChatServiceImplBase {

    @Override
    public StreamObserver<SendMessageRequest> batchSendMessage(
            StreamObserver<SendMessageResponse> responseObserver) {

        // 使用 AtomicReference 保存聚合结果
        AtomicInteger successCount = new AtomicInteger(0);
        AtomicInteger failCount = new AtomicInteger(0);

        // 返回 StreamObserver，处理客户端发送的每条消息
        return new StreamObserver<SendMessageRequest>() {

            @Override
            public void onNext(SendMessageRequest request) {
                // 每收到一条消息，处理一次
                try {
                    processMessage(request.getContent());
                    successCount.incrementAndGet();
                } catch (Exception e) {
                    failCount.incrementAndGet();
                }
            }

            @Override
            public void onError(Throwable t) {
                // 客户端异常断开
                System.err.println("客户端流异常: " + t.getMessage());
            }

            @Override
            public void onCompleted() {
                // 客户端发送完毕，返回聚合结果
                SendMessageResponse response = SendMessageResponse.newBuilder()
                        .setSuccess(true)
                        .setMessageId(String.format("batch:%d/%d",
                                successCount.get(), failCount.get()))
                        .build();
                responseObserver.onNext(response);
                responseObserver.onCompleted();
            }
        };
    }
}
```

### 客户端调用

```java
public class ChatClient {

    private final ManagedChannel channel;

    // 客户端流调用示例
    public void batchSendMessages(List<String> messages) {
        // 使用 AsyncStub 进行异步调用
        ChatServiceGrpc.ChatServiceStub asyncStub =
                ChatServiceGrpc.newStub(channel);

        CountDownLatch latch = new CountDownLatch(1);
        final String[] resultMessageId = new String[1];

        // 设置回调
        StreamObserver<SendMessageResponse> responseObserver =
                new StreamObserver<SendMessageResponse>() {
            @Override
            public void onNext(SendMessageResponse response) {
                resultMessageId[0] = response.getMessageId();
            }

            @Override
            public void onError(Throwable t) {
                System.err.println("批量发送失败: " + t.getMessage());
                latch.countDown();
            }

            @Override
            public void onCompleted() {
                System.out.println("批量发送完成: " + resultMessageId[0]);
                latch.countDown();
            }
        };

        // 获取请求发送器
        StreamObserver<SendMessageRequest> requestObserver =
                asyncStub.batchSendMessage(responseObserver);

        // 发送消息
        for (String message : messages) {
            SendMessageRequest request = SendMessageRequest.newBuilder()
                    .setContent(message)
                    .build();
            requestObserver.onNext(request);
        }

        // 发送完毕，标记结束
        requestObserver.onCompleted();

        // 等待完成
        try {
            latch.await(30, TimeUnit.SECONDS);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

2. **批量导入**：批量导入用户数据
3. **日志收集**：客户端持续发送日志到服务端
4. **实时监控**：客户端持续发送监控指标

## 双向流：Bidirectional Streaming

双向流是**客户端和服务端都可以分多次发送**。常用于聊天、实时交互等场景。

### 服务端实现

```java
public class ChatServiceImpl extends ChatServiceGrpc.ChatServiceImplBase {

    private final Map<String, String> onlineUsers = new ConcurrentHashMap<>();

    @Override
    public StreamObserver<ChatMessage> chat(
            StreamObserver<ChatMessage> responseObserver) {

        // 每个连接维护一个用户上下文
        AtomicReference<String> currentUser = new AtomicReference<>();

        // 返回 StreamObserver 处理客户端发送的消息
        return new StreamObserver<ChatMessage>() {

            @Override
            public void onNext(ChatMessage message) {
                // 首次消息：注册用户
                if (currentUser.get() == null) {
                    currentUser.set(message.getSender());
                    onlineUsers.put(currentUser.get(), "online");
                    System.out.println("用户上线: " + currentUser.get());
                    return;
                }

                // 处理普通消息
                System.out.printf("[%s] %s: %s%n",
                        message.getSender(),
                        message.getSender(),
                        message.getContent());

                // 构建回复
                ChatMessage reply = ChatMessage.newBuilder()
                        .setSender("Server")
                        .setContent("收到消息: " + message.getContent())
                        .setTimestamp(System.currentTimeMillis())
                        .build();
                responseObserver.onNext(reply);
            }

            @Override
            public void onError(Throwable t) {
                if (currentUser.get() != null) {
                    onlineUsers.remove(currentUser.get());
                    System.out.println("用户离线: " + currentUser.get());
                }
            }

            @Override
            public void onCompleted() {
                if (currentUser.get() != null) {
                    onlineUsers.remove(currentUser.get());
                    System.out.println("用户断开: " + currentUser.get());
                }
                responseObserver.onCompleted();
            }
        };
    }
}
```

### 客户端调用

```java
public class ChatClient {

    private final ManagedChannel channel;
    private final ChatServiceGrpc.ChatServiceStub asyncStub;

    public ChatClient(String host, int port) {
        this.channel = ManagedChannelBuilder
                .forAddress(host, port)
                .usePlaintext()
                .build();
        this.asyncStub = ChatServiceGrpc.newStub(channel);
    }

    public void chat(String username, Scanner scanner) {
        CountDownLatch latch = new CountDownLatch(1);

        // 设置响应处理器
        StreamObserver<ChatMessage> responseObserver = new StreamObserver<ChatMessage>() {
            @Override
            public void onNext(ChatMessage message) {
                System.out.printf("[%s] %s: %s%n",
                        LocalDateTime.now(),
                        message.getSender(),
                        message.getContent());
            }

            @Override
            public void onError(Throwable t) {
                System.err.println("聊天错误: " + t.getMessage());
                latch.countDown();
            }

            @Override
            public void onCompleted() {
                latch.countDown();
            }
        };

        // 启动双向流
        StreamObserver<ChatMessage> requestObserver = asyncStub.chat(responseObserver);

        // 新线程处理用户输入
        new Thread(() -> {
            try {
                // 首先发送一条注册消息
                ChatMessage registerMsg = ChatMessage.newBuilder()
                        .setSender(username)
                        .setContent("joined")
                        .setTimestamp(System.currentTimeMillis())
                        .build();
                requestObserver.onNext(registerMsg);

                // 循环读取用户输入
                while (scanner.hasNextLine()) {
                    String input = scanner.nextLine();
                    if ("exit".equalsIgnoreCase(input)) {
                        break;
                    }

                    ChatMessage message = ChatMessage.newBuilder()
                            .setSender(username)
                            .setContent(input)
                            .setTimestamp(System.currentTimeMillis())
                            .build();
                    requestObserver.onNext(message);
                }
            } finally {
                requestObserver.onCompleted();
            }
        }).start();

        try {
            latch.await();
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

### 典型应用场景

1. **聊天室**：多个用户实时聊天
2. **实时协作**：Google Docs 式的多人协作
3. **游戏对战**：实时游戏指令传输
4. **视频通话信令**：传输通话控制信令

## 流式调用的注意事项

### 背压（Backpressure）问题

在双向流中，如果服务端处理速度慢于客户端发送速度，会导致内存堆积。这就是**背压问题**。

gRPC 的解决方案：

- **同步 Stub**：使用阻塞的 Iterator，自动处理背压
- **异步 Stub**：需要手动控制请求速率

```java
// 同步客户端：自动背压
Iterator<ChatMessage> stream = stub.subscribe(request);
while (stream.hasNext()) {  // 阻塞等待下一个
    process(stream.next());
}
```

### 连接管理与重试

流式调用的连接比一元调用更敏感，需要考虑：

1. **心跳保活**：长时间空闲的连接可能被中间设备断开
2. **重连机制**：网络波动时的自动重连
3. **优雅关闭**：确保正在处理的请求不被丢弃

```java
// 配置连接参数
ManagedChannel channel = ManagedChannelBuilder
        .forAddress(host, port)
        .keepAliveTime(30, TimeUnit.SECONDS)     // 心跳间隔
        .keepAliveTimeout(10, TimeUnit.SECONDS)  // 心跳超时
        .idleTimeout(5, TimeUnit.MINUTES)       // 空闲超时
        .build();
```

### 错误处理

```java
// StreamObserver 错误处理
StreamObserver<Request> observer = new StreamObserver<>() {
    @Override
    public void onNext(Request request) {
        // 处理消息
    }

    @Override
    public void onError(Throwable t) {
        // 处理错误
        Status status = Status.fromThrowable(t);
        if (status.getCode() == Status.Code.UNAVAILABLE) {
            // 连接断开，尝试重连
        } else {
            // 其他错误
        }
    }

    @Override
    public void onCompleted() {
        // 流正常结束
    }
};
```

## 总结

gRPC 的四种调用模式覆盖了绝大多数远程调用场景：

- **Unary**：简单请求-响应
- **Server Streaming**：服务端主动推送
- **Client Streaming**：客户端分批发送
- **Bidirectional Streaming**：实时双向交互

理解每种模式的特点和使用场景，才能在架构设计时做出正确的选择。
