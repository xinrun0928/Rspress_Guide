## 为什么选择 Kafka

Kafka 在大数据和实时处理领域占据统治地位，有以下优势：

| 优势 | 说明 |
|---|---|
| **超高吞吐量** | 单机可达百万级 QPS |
| **低延迟** | 端到端延迟可达毫秒级 |
| **持久化** | 消息持久化到磁盘，支持回溯 |
| **水平扩展** | 通过分区实现水平扩展 |
| **生态完善** | 与 Flink、Spark、Kafka Streams 无缝集成 |
| **高可用** | 多副本机制保证高可用 |

## 学习建议

1. **先理解架构**：熟悉 Kafka 的整体架构和数据流
2. **掌握高性能原理**：理解顺序写、零拷贝、批量处理的原理
3. **深入副本机制**：理解 ISR、HW/LEO、故障转移
4. **掌握生产者和消费者**：理解发送模式、消费模式、Offset 管理
