# Kafka 消息发送模式：同步、异步、异步回调

## 场景切入：send 方法的三种姿势

你用过 `producer.send()`，但你知道它有三种发送模式吗？

- 同步发送：等消息发送成功才返回
- 异步发送：立即返回，通过 Future 获取结果
- 异步回调：立即返回，发送完成后自动回调

不同的场景选择不同的发送模式，直接影响系统的吞吐量和可靠性。

## 一、三种发送模式对比

| 模式 | 方法 | 返回值 | 阻塞 | 适用场景 |
|---|---|---|---|---|
| **同步发送** | `send(record).get()` | Future | 是 | 高可靠场景 |
| **异步发送** | `send(record)` | Future | 否 | 普通场景 |
| **异步回调** | `send(record, callback)` | 无 | 否 | 高吞吐场景 |

## 二、同步发送

### 2.1 基本用法

```java
// 同步发送
KafkaProducer<String, String> producer = new KafkaProducer<>(props);

try {
    // 发送消息，阻塞等待结果
    Future<RecordMetadata> future = producer.send(new ProducerRecord<>(
        "order-topic",
        "order-123",
        "order data"
    ));

    // get() 会阻塞，直到收到响应
    RecordMetadata metadata = future.get();

    System.out.println("发送成功：topic=" + metadata.topic()
        + ", partition=" + metadata.partition()
        + ", offset=" + metadata.offset());

} catch (ExecutionException e) {
    // 发送失败
    System.err.println("发送失败：" + e.getCause().getMessage());
}
```

### 2.2 同步发送流程

```
同步发送流程：

Producer                          Broker
    │                                │
    │──── send() ────────────────────>│
    │                                │
    │     写入日志                    │
    │                                │
    │<─── 返回 ACK ──────────────────│
    │                                │
    │  get() 返回                    │
    ▼                                ▼

主线程阻塞，等待结果
```

### 2.3 同步发送的优缺点

| 优点 | 缺点 |
|---|---|
| 简单直观 | 阻塞主线程 |
| 发送结果确定 | 吞吐量低 |
| 便于错误处理 | 不适合高并发场景 |

## 三、异步发送

### 3.1 基本用法

```java
// 异步发送
KafkaProducer<String, String> producer = new KafkaProducer<>(props);

// 发送消息，立即返回 Future
Future<RecordMetadata> future = producer.send(new ProducerRecord<>(
    "order-topic",
    "order-123",
    "order data"
));

// 可以做其他事情
doSomethingElse();

// 稍后获取结果
try {
    RecordMetadata metadata = future.get();
    System.out.println("发送成功：offset=" + metadata.offset());
} catch (ExecutionException e) {
    System.err.println("发送失败：" + e.getCause().getMessage());
}
```

### 3.2 带超时的异步发送

```java
// 带超时的异步发送
try {
    // 等待 5 秒获取结果
    RecordMetadata metadata = future.get(5, TimeUnit.SECONDS);
    System.out.println("发送成功：offset=" + metadata.offset());
} catch (TimeoutException e) {
    // 超时处理
    System.err.println("发送超时");
} catch (ExecutionException e) {
    System.err.println("发送失败：" + e.getCause().getMessage());
}
```

## 四、异步回调

### 4.1 基本用法

```java
// 异步回调发送
KafkaProducer<String, String> producer = new KafkaProducer<>(props);

// 发送消息，指定回调
producer.send(new ProducerRecord<>(
    "order-topic",
    "order-123",
    "order data"
), new Callback() {
    @Override
    public void onCompletion(RecordMetadata metadata, Exception exception) {
        if (exception == null) {
            // 发送成功
            System.out.println("发送成功：topic=" + metadata.topic()
                + ", partition=" + metadata.partition()
                + ", offset=" + metadata.offset());
        } else {
            // 发送失败
            System.err.println("发送失败：" + exception.getMessage());
        }
    }
});

// 主线程可以继续做其他事情
doSomethingElse();
```

### 4.2 Lambda 简化写法

```java
// Lambda 简化写法
producer.send(new ProducerRecord<>(
    "order-topic",
    "order-123",
    "order data"
), (metadata, exception) -> {
    if (exception == null) {
        System.out.println("发送成功：offset=" + metadata.offset());
    } else {
        System.err.println("发送失败：" + exception.getMessage());
    }
});
```

### 4.3 回调的执行时机

```
回调执行时机：

Producer                           Broker
    │                                │
    │──── send() ────────────────────>│
    │                                │
    │     写入日志                    │
    │                                │
    │<─── 返回 ACK ──────────────────│
    │                                │
    │  触发回调 onCompletion()        │
    ▼                                ▼

回调在线程池中执行，不阻塞主线程
```

## 五、发送模式选择

### 5.1 场景对比

| 场景 | 推荐模式 | 原因 |
|---|---|---|
| **高可靠订单系统** | 同步发送 | 需要确认发送成功 |
| **普通业务消息** | 异步回调 | 吞吐量高，不阻塞 |
| **日志收集** | 异步发送 | 允许少量丢失，追求吞吐 |
| **批量发送** | 异步回调 + Future | 异步并行发送 |
| **消息队列** | 异步回调 | 高吞吐量 |

### 5.2 代码示例

#### 场景一：订单系统（同步）

```java
// 订单系统：需要确认发送成功
public class OrderService {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void placeOrder(Order order) {
        ProducerRecord<String, String> record = new ProducerRecord<>(
            "order-topic",
            order.getId(),
            order.toString()
        );

        try {
            // 同步发送，确认成功
            RecordMetadata metadata = kafkaTemplate.send(record).get();

            log.info("订单发送成功：orderId={}, offset={}",
                order.getId(), metadata.offset());

        } catch (Exception e) {
            // 发送失败，记录日志，稍后重试
            log.error("订单发送失败：orderId={}", order.getId(), e);
            retryLater(order);
        }
    }
}
```

#### 场景二：日志收集（异步回调）

```java
// 日志收集：追求高吞吐
public class LogService {

    private KafkaProducer<String, String> producer;

    public void log(String event) {
        ProducerRecord<String, String> record = new ProducerRecord<>(
            "log-topic",
            event.getId(),
            event.toString()
        );

        // 异步回调，不阻塞
        producer.send(record, (metadata, exception) -> {
            if (exception != null) {
                // 记录错误，但不阻塞主流程
                logger.error("日志发送失败", exception);
            }
        });
    }
}
```

#### 场景三：批量发送

```java
// 批量发送：使用 CompletableFuture
public class BatchSender {

    private KafkaProducer<String, String> producer;

    public void sendBatch(List<Order> orders) {
        // 发送所有消息
        List<CompletableFuture<RecordMetadata>> futures = orders.stream()
            .map(order -> {
                ProducerRecord<String, String> record = new ProducerRecord<>(
                    "order-topic",
                    order.getId(),
                    order.toString()
                );
                return CompletableFuture.supplyAsync(
                    () -> producer.send(record).get(),
                    executor
                );
            })
            .collect(Collectors.toList());

        // 等待所有发送完成
        CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .thenRun(() -> System.out.println("批量发送完成"))
            .exceptionally(e -> {
                System.err.println("批量发送失败：" + e.getMessage());
                return null;
            });
    }
}
```

## 六、错误处理

### 6.1 可重试错误

```java
// 可重试错误处理
producer.send(record, (metadata, exception) -> {
    if (exception != null) {
        if (exception instanceof RetriableException) {
            // 可重试错误，稍后重试
            retryProducer.send(record, this.callback);
        } else {
            // 不可重试错误，记录日志
            logger.error("发送失败，不重试", exception);
        }
    }
});
```

### 6.2 不可重试错误

```java
// 不可重试错误
// - SerializationException：序列化失败
// - TimeoutException：超时
// - RecordTooLargeException：消息太大
```

### 6.3 全局错误处理

```java
// 配置全局错误处理器
props.put("error.handler", new CustomErrorHandler());

// 自定义错误处理器
public class CustomErrorHandler implements ErrorHandler {

    @Override
    public void handle(Exception exception, ProducerRecord<String, String> record) {
        // 记录错误
        logger.error("发送失败：record={}", record.value(), exception);

        // 发送死信消息
        sendToDeadLetterTopic(record);

        // 或者写入本地文件
        saveToLocalFile(record);
    }
}
```

## 八、面试追问预判

### 追问一：send 方法是同步还是异步？

`send()` 方法本身是异步的，它返回一个 `Future`。如果调用 `Future.get()`，则变为同步阻塞。

### 追问二：回调函数在哪个线程执行？

回调函数在 Producer 的后台线程（Sender 线程）执行，不阻塞主线程。

### 追问三：如何保证消息发送成功？

1. 使用 `acks=all` 确保所有副本确认
2. 使用同步发送或回调中检查结果
3. 实现重试机制
4. 开启幂等性

### 追问四：批量发送和逐条发送有什么区别？

| 模式 | 网络请求 | 吞吐量 | 延迟 |
|---|---|---|---|
| 逐条发送 | 每条消息一次请求 | 低 | 低 |
| 批量发送 | 多条消息一次请求 | 高 | 高 |
