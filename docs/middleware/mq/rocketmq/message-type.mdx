# RocketMQ 消息类型：普通消息、顺序消息、事务消息、延迟消息

## 场景切入：RocketMQ 的四种消息

你发送了一条消息到 RocketMQ，但你知道它有四种类型吗？

普通消息、顺序消息、事务消息、延迟消息……每种类型的发送方式和适用场景都不同。选对消息类型，才能发挥 RocketMQ 的最大价值。

## 一、四种消息类型

| 类型 | 说明 | 适用场景 |
|---|---|---|
| **普通消息** | 无特殊语义的异步消息 | 普通异步处理 |
| **顺序消息** | 消息按顺序消费 | 需要保证顺序的场景 |
| **事务消息** | 支持本地事务的半消息 | 分布式事务 |
| **延迟消息** | 定时投递的消息 | 延迟任务、延迟处理 |

---

## 二、普通消息

### 2.1 发送方式

```java
// 普通消息发送
DefaultMQProducer producer = new DefaultMQProducer("order-producer-group");
producer.setNamesrvAddr("127.0.0.1:9876");
producer.start();

// 创建消息
Message msg = new Message(
    "order-topic",           // Topic
    "order-created",        // Tag
    "order-123",           // Key
    "订单数据".getBytes()   // Body
);

// 发送消息
SendResult result = producer.send(msg);

System.out.println("发送成功：" + result.getSendStatus());
```

### 2.2 消费方式

```java
// 普通消息消费
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer("order-consumer-group");
consumer.setNamesrvAddr("127.0.0.1:9876");
consumer.subscribe("order-topic", "*");

consumer.registerMessageListener(new MessageListenerConcurrently() {
    @Override
    public ConsumeConcurrentlyStatus consumeMessage(List<MessageExt> msgs,
                                                    ConsumeConcurrentlyContext context) {
        for (MessageExt msg : msgs) {
            System.out.println("收到消息：" + new String(msg.getBody()));
        }
        return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;
    }
});

consumer.start();
```

### 2.3 特点

| 特点 | 说明 |
|---|---|
| **无顺序要求** | 消息可以并行消费 |
| **无事务要求** | 不保证本地事务 |
| **无延迟要求** | 立即投递 |

## 三、顺序消息

### 3.1 顺序消息类型

| 类型 | 说明 |
|---|---|
| **全局顺序** | Topic 中所有消息按顺序消费 |
| **分区顺序** | 同一 MessageQueue 中的消息按顺序消费 |

### 3.2 发送方式

```java
// 顺序消息发送
producer.send(msg, new MessageQueueSelector() {
    @Override
    public MessageQueue select(List<MessageQueue> mqs, Message msg, Object arg) {
        // 按订单 ID 选择 Queue，保证同一订单的消息到同一 Queue
        Long orderId = (Long) arg;
        int index = (int) (orderId % mqs.size());
        return mqs.get(index);
    }
}, orderId);  // 订单 ID 作为参数
```

### 3.3 消费方式

```java
// 顺序消息消费
consumer.registerMessageListener(new MessageListenerOrderly() {
    @Override
    public ConsumeOrderlyStatus consumeMessage(List<MessageExt> msgs,
                                             ConsumeOrderlyContext context) {
        // 单线程顺序处理
        for (MessageExt msg : msgs) {
            System.out.println("收到消息：" + new String(msg.getBody()));
        }
        return ConsumeOrderlyStatus.SUCCESS;
    }
});
```

## 四、事务消息

### 4.1 事务消息原理

事务消息用于保证分布式事务的一致性：

```
事务消息流程：

Producer                    Broker                    Consumer
   │                           │                          │
   │──发送半消息──────────────>│                          │
   │<─返回 OK ────────────────│                          │
   │                           │                          │
   │执行本地事务                 │                          │
   │                           │                          │
   │──提交事务结果──────────>│                          │
   │   (COMMIT/ROLLBACK)       │                          │
   │                           │                          │
   │                           │──消息可见───────────────>│
   │                           │                          │
   │     (定时回查本地事务)      │                          │
```

### 4.2 发送方式

```java
// 事务消息发送
TransactionMQProducer producer = new TransactionMQProducer("order-producer-group");
producer.setNamesrvAddr("127.0.0.1:9876");
producer.setTransactionListener(new TransactionListener() {

    @Override
    public LocalTransactionState executeLocalTransaction(Message msg, Object arg) {
        // 执行本地事务
        try {
            orderService.createOrder(order);
            return LocalTransactionState.COMMIT_MESSAGE;
        } catch (Exception e) {
            return LocalTransactionState.ROLLBACK_MESSAGE;
        }
    }

    @Override
    public LocalTransactionState checkLocalTransaction(MessageExt msg) {
        // 回查本地事务状态
        String orderId = msg.getKeys();
        if (orderService.exists(orderId)) {
            return LocalTransactionState.COMMIT_MESSAGE;
        }
        return LocalTransactionState.ROLLBACK_MESSAGE;
    }
});

producer.start();

// 发送事务消息
Message msg = new Message("order-topic", order.toString().getBytes());
TransactionSendResult result = producer.sendMessageInTransaction(msg, order.getId());
```

### 4.3 特点

| 特点 | 说明 |
|---|---|
| **半消息** | 消息对消费者不可见 |
| **本地事务** | 执行本地事务并提交结果 |
| **回查机制** | 定时回查未提交的消息 |

## 五、延迟消息

### 5.1 延迟等级

RocketMQ 支持 18 个延迟等级：

| 等级 | 延迟时间 |
|---|---|
| 1 | 1 秒 |
| 2 | 5 秒 |
| 3 | 10 秒 |
| 4 | 30 秒 |
| 5 | 1 分钟 |
| 6 | 2 分钟 |
| 7 | 3 分钟 |
| 8 | 4 分钟 |
| 9 | 5 分钟 |
| 10 | 6 分钟 |
| 11 | 7 分钟 |
| 12 | 8 分钟 |
| 13 | 9 分钟 |
| 14 | 10 分钟 |
| 15 | 20 分钟 |
| 16 | 30 分钟 |
| 17 | 1 小时 |
| 18 | 2 小时 |

### 5.2 发送方式

```java
// 延迟消息发送
Message msg = new Message("order-delay-topic", "延迟消息".getBytes());
msg.setDelayTimeLevel(3);  // 延迟 10 秒

SendResult result = producer.send(msg);
```

### 5.3 应用场景

| 场景 | 说明 |
|---|---|
| **订单超时取消** | 下单后 30 分钟未支付，自动取消 |
| **延迟重试** | 消息处理失败后延迟重试 |
| **定时任务** | 定时发送通知 |

## 六、消息类型对比

### 6.1 对比表

| 类型 | 顺序 | 事务 | 延迟 | 适用场景 |
|---|---|---|---|---|
| **普通消息** | ❌ | ❌ | ❌ | 普通异步处理 |
| **顺序消息** | ✅ | ❌ | ❌ | 需要顺序的场景 |
| **事务消息** | ❌ | ✅ | ❌ | 分布式事务 |
| **延迟消息** | ❌ | ❌ | ✅ | 延迟任务 |

### 6.2 选择建议

| 场景 | 选择 |
|---|---|
| 普通异步处理 | 普通消息 |
| 订单流程（需要顺序） | 顺序消息 |
| 下单+扣库存（需要事务） | 事务消息 |
| 订单超时取消 | 延迟消息 |

## 七、面试追问预判

### 追问一：事务消息的原理是什么？

事务消息通过**半消息**和**回查机制**实现：

1. 发送半消息（对消费者不可见）
2. 执行本地事务
3. 提交或回滚消息
4. 如果未收到结果，定时回查本地事务

### 追问二：普通消息和顺序消息有什么区别？

| 维度 | 普通消息 | 顺序消息 |
|---|---|---|
| **消费方式** | 并行消费 | 串行消费 |
| **分区策略** | 无 | 按 Key 哈希 |
| **性能** | 高 | 低 |

### 追问三：延迟消息和定时任务有什么区别？

| 维度 | 延迟消息 | 定时任务 |
|---|---|---|
| **实现方式** | RocketMQ 原生 | 定时器 |
| **精度** | 秒级 | 依赖定时器精度 |
| **可靠性** | 高 | 中 |
