# RocketMQ

> [!TIP]
> RocketMQ 是阿里巴巴开发的消息队列，最初用于支撑双十一的海量消息处理。它原生支持事务消息和延迟消息，是电商、金融场景的首选。本模块深入讲解 RocketMQ 的架构、消息类型、事务消息等。

## 模块内容导航

| 文档 | 内容简介 |
|---|---|
| **架构设计** | NameServer、Broker、Producer、Consumer 的角色 |
| **消息类型** | 普通消息、顺序消息、事务消息、延迟消息 |
| **事务消息** | 半消息与事务状态回查原理 |
| **顺序消息** | 全局顺序与分区顺序实现 |
| **延迟消息** | 延迟等级实现原理 |
| **高可用** | 主从复制与 DLeger |
| **选型对比** | RocketMQ vs Kafka vs RabbitMQ |

---

## 为什么选择 RocketMQ

RocketMQ 的核心优势：

| 优势 | 说明 |
|---|---|
| **事务消息** | 原生支持分布式事务 |
| **延迟消息** | 支持多级延迟消息 |
| **顺序消息** | 支持全局顺序和分区顺序 |
| **Java 原生** | 与 Spring 生态深度集成 |
| **高可靠** | 金融级可靠性 |

4. **对比学习**：与 Kafka、RabbitMQ 对比，理解各自的适用场景
