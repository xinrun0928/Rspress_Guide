# RocketMQ 延迟消息实现原理：延迟等级

## 场景切入：订单超时取消，你是怎么实现的？

用户下单后 30 分钟未支付，自动取消订单。这个功能如何实现？

定时轮询？太浪费资源。
RocketMQ 原生支持延迟消息，轻松实现订单超时取消。

## 一、延迟消息概述

### 1.1 什么是延迟消息

延迟消息是指定时投递的消息：

| 类型 | 说明 |
|---|---|
| **普通消息** | 立即投递 |
| **延迟消息** | 定时投递 |

```
延迟消息流程：

Producer ──发送延迟消息(30分钟后)──> Broker
                                        │
                                        ▼
                              消息存入延迟队列
                                        │
                                        │ 30 分钟后
                                        ▼
                              消息投递到目标 Queue ──> Consumer
```

### 1.2 延迟等级

RocketMQ 支持 18 个延迟等级：

| 等级 | 延迟时间 | 等级 | 延迟时间 |
|---|---|---|---|
| 1 | 1 秒 | 10 | 6 分钟 |
| 2 | 5 秒 | 11 | 7 分钟 |
| 3 | 10 秒 | 12 | 8 分钟 |
| 4 | 30 秒 | 13 | 9 分钟 |
| 5 | 1 分钟 | 14 | 10 分钟 |
| 6 | 2 分钟 | 15 | 20 分钟 |
| 7 | 3 分钟 | 16 | 30 分钟 |
| 8 | 4 分钟 | 17 | 1 小时 |
| 9 | 5 分钟 | 18 | 2 小时 |

## 二、延迟消息实现原理

### 2.1 延迟消息存储

```
延迟消息存储架构：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Producer ──发送延迟消息──> Broker                         │
│                                         │                  │
│                                         ▼                  │
│                              ┌─────────────────┐          │
│                              │  延迟队列        │          │
│                              │  (SCHEDULE_TOPIC)│          │
│                              └────────┬────────┘          │
│                                         │                  │
│                    ┌────────────────────┼────────────────┐│
│                    │                    │                    │
│                    ▼                    ▼                    ▼
│              delayLevel=1        delayLevel=2      delayLevel=3
│              (1秒后投递)         (5秒后投递)        (10秒后投递)
│                    │                    │                    │
│                    └────────────────────┼────────────────┘│
│                                         │                  │
│                                         ▼                  │
│                              定时扫描器（Timer）           │
│                                         │                  │
│                                         ▼                  │
│                              消息投放到目标 Queue           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 定时扫描器

Broker 内部有一个定时扫描器：

| 组件 | 说明 |
|---|---|
| **定时器** | 定时扫描延迟队列 |
| **时间索引** | 记录消息的投递时间 |
| **调度器** | 将到期的消息投递到目标 Queue |

## 三、发送延迟消息

### 3.1 基本发送

```java
// 发送延迟消息
DefaultMQProducer producer = new DefaultMQProducer("order-producer-group");
producer.setNamesrvAddr("127.0.0.1:9876");
producer.start();

// 创建延迟消息
Message msg = new Message(
    "order-topic",
    "order-data".getBytes()
);

// 设置延迟等级（3 = 10 秒）
msg.setDelayTimeLevel(3);

SendResult result = producer.send(msg);
System.out.println("延迟消息发送成功：" + result.getMsgId());
```

### 3.2 完整示例

```java
// 订单超时取消场景
public class OrderCancelScheduler {

    private DefaultMQProducer producer;

    public void scheduleOrderCancel(Order order) {
        try {
            // 创建延迟消息
            Message msg = new Message(
                "order-cancel-topic",    // Topic
                "cancel",                // Tag
                order.getId(),           // Key
                ("取消订单:" + order.getId()).getBytes()  // Body
            );

            // 设置延迟等级（15 = 30 分钟）
            msg.setDelayTimeLevel(15);

            // 发送延迟消息
            producer.send(msg);
            System.out.println("订单取消任务已调度：" + order.getId());

        } catch (MQClientException e) {
            throw new RuntimeException("调度失败", e);
        }
    }
}
```

### 3.3 消费延迟消息

```java
// 消费延迟消息
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer("order-consumer-group");
consumer.setNamesrvAddr("127.0.0.1:9876");
consumer.subscribe("order-cancel-topic", "cancel");

consumer.registerMessageListener(new MessageListenerConcurrently() {
    @Override
    public ConsumeConcurrentlyStatus consumeMessage(List<MessageExt> msgs,
                                                    ConsumeConcurrentlyContext context) {
        for (MessageExt msg : msgs) {
            String orderId = msg.getKeys();
            System.out.println("收到订单取消消息：" + orderId);

            // 检查订单状态
            Order order = orderService.getOrder(orderId);
            if ("待支付".equals(order.getStatus())) {
                // 取消订单
                orderService.cancelOrder(orderId);
                System.out.println("订单已取消：" + orderId);
            } else {
                System.out.println("订单状态已变化，无需取消：" + orderId);
            }
        }
        return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;
    }
});

consumer.start();
```

## 六、面试追问预判

### 追问一：延迟消息的原理是什么？

RocketMQ 通过**定时扫描器**实现延迟消息：

1. 延迟消息存入 `SCHEDULE_TOPIC` 延迟队列
2. Broker 定时扫描延迟队列
3. 消息到期后，投递到目标 Queue

### 追问二：RocketMQ 延迟消息有什么限制？

| 限制 | 说明 |
|---|---|
| **延迟等级固定** | 只能使用预设的 18 个等级 |
| **不支持任意时间** | 不能指定精确的投递时间 |
| **不支持毫秒级** | 最小精度为秒级 |

### 追问三：如何实现任意延迟？

如果要实现任意延迟时间，可以使用：

1. **RocketMQ 延迟插件**：第三方实现
2. **外部调度器**：定时轮询数据库
3. **其他 MQ**：如 RabbitMQ 的延迟插件
