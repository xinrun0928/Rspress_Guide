# RocketMQ 高可用：主从复制与 DLeger

## 场景切入：Broker 宕机了，消息还在吗？

如果 RocketMQ 的 Broker 宕机了，消息会丢失吗？

这取决于你的高可用配置。RocketMQ 支持**主从复制**和**DLeger** 两种高可用模式。

## 一、高可用模式对比

| 模式 | 数据一致性 | 故障转移 | 复杂度 |
|---|---|---|---|
| **主从复制** | 异步/同步 | 手动 | 低 |
| **DLeger** | Raft 强一致 | 自动 | 中 |

---

## 二、主从复制

### 2.1 复制模式

| 模式 | 说明 | 可靠性 | 性能 |
|---|---|---|---|
| **同步复制** | Master 和 Slave 都写入成功才返回 | 高 | 低 |
| **异步复制** | Master 写入成功即返回 | 中 | 高 |

```
同步复制：

Producer ──写入──> Master ──同步──> Slave
                    ▲                        │
                    └────────────────────────┘
                          等待确认

同步复制：Master 和 Slave 都写入成功才返回
```

```
异步复制：

Producer ──写入──> Master ──异步同步──> Slave
                    ▲
                    │
               立即返回

异步复制：Master 写入成功即返回
```

### 2.2 配置

```properties
# 主从复制配置
# Master 配置
brokerRole=SYNC_MASTER  # 同步 Master
# 或
brokerRole=ASYNC_MASTER  # 异步 Master

# Slave 配置
brokerId=1  # Slave 的 brokerId（必须大于 0）
```

### 2.3 故障转移

主从复制的故障转移需要手动操作：

```bash
# 查看 Broker 状态
mqadmin clusterList -n 127.0.0.1:9876

# 手动切换 Master
mqadmin brokerStatus -b broker-a -n 127.0.0.1:9876
```

## 三、DLeger 模式

### 3.1 DLeger 原理

DLeger 使用 Raft 协议实现高可用：

| 特性 | 说明 |
|---|---|
| **Raft 协议** | Leader 选举、日志复制 |
| **自动故障转移** | Leader 宕机自动选举 |
| **强一致性** | 写入需要多数节点确认 |

```
DLeger 架构：

┌─────────────────────────────────────────────────────────────┐
│                    DLeger 组                                │
│                                                             │
│   Leader ──> Follower ──> Follower                        │
│                                                             │
│   Raft 共识：                                               │
│   · 写入需要多数节点确认                                     │
│   · Leader 宕机自动选举新 Leader                            │
│   · 数据强一致性                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 DLeger 配置

```properties
# DLeger 配置
enableDLegerCommitLog=true
dLegerGroup=broker-a
dLegerPeers=n0-127.0.0.1:40911;n1-127.0.0.2:40911;n2-127.0.0.3:40911
dLegerSelfId=n0
```

### 3.3 故障转移

DLeger 支持自动故障转移：

```
故障转移流程：

1. Leader 宕机
   │
2. Follower 检测到 Leader 超时
   │
3. 发起选举
   │
4. 多数节点投票
   │
5. 选出新 Leader
   │
6. 恢复服务

故障恢复时间：通常 < 30 秒
```

## 四、刷盘策略

### 4.1 刷盘模式

| 模式 | 说明 | 可靠性 | 性能 |
|---|---|---|---|
| **同步刷盘** | 写入 Page Cache 后立即刷盘 | 高 | 低 |
| **异步刷盘** | 批量写入，定期刷盘 | 中 | 高 |

```properties
# 刷盘配置
flushDiskType=SYNC_FLUSH  # 同步刷盘
# 或
flushDiskType=ASYNC_FLUSH  # 异步刷盘
```

### 4.2 配置建议

| 场景 | 刷盘模式 | 复制模式 |
|---|---|---|
| **金融级** | 同步刷盘 | 同步复制 + DLeger |
| **普通生产** | 异步刷盘 | 同步复制 |
| **开发测试** | 异步刷盘 | 异步复制 |

---

## 五、高可用配置

### 5.1 推荐配置

```properties
# 高可用配置
# Broker 配置
brokerClusterName=DefaultCluster
brokerName=broker-a
brokerId=0
namesrvAddr=127.0.0.1:9876;127.0.0.2:9876;127.0.0.3:9876

# 刷盘配置
flushDiskType=SYNC_FLUSH

# 复制配置
brokerRole=SYNC_MASTER

# DLeger 配置（可选）
enableDLegerCommitLog=true
dLegerGroup=broker-a
dLegerPeers=n0-127.0.0.1:40911;n1-127.0.0.2:40911;n2-127.0.0.3:40911
```

## 六、面试追问预判

### 追问一：主从复制和 DLeger 有什么区别？

| 维度 | 主从复制 | DLeger |
|---|---|---|
| **一致性** | 最终一致 | 强一致 |
| **故障转移** | 手动 | 自动 |
| **协议** | 无 | Raft |

### 追问二：同步复制和异步复制有什么区别？

| 模式 | 说明 | 可靠性 | 性能 |
|---|---|---|---|
| **同步复制** | Master + Slave 都写入成功才返回 | 高 | 低 |
| **异步复制** | Master 写入成功即返回 | 中 | 高 |

### 追问三：如何保证消息不丢失？

需要三个层面都配置：

1. **刷盘**：同步刷盘
2. **复制**：同步复制或 DLeger
3. **消费**：手动确认
