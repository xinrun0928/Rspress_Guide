# RocketMQ vs Kafka vs RabbitMQ 选型对比

## 场景切入：选型不是选最好的，而是选最合适的

消息队列三巨头：Kafka、RocketMQ、RabbitMQ。它们各有特点，如何选型？

Kafka 高吞吐，RocketMQ 功能强，RabbitMQ 路由灵活。选对 MQ，才能发挥系统的最大价值。

## 一、三种 MQ 核心对比

### 1.1 设计理念

| MQ | 设计理念 | 核心优势 |
|---|---|---|
| **Kafka** | 日志优先 | 高吞吐、海量数据 |
| **RocketMQ** | 金融级 | 事务消息、延迟消息 |
| **RabbitMQ** | 路由优先 | 灵活路由、功能丰富 |

### 1.2 核心指标对比

| 指标 | Kafka | RocketMQ | RabbitMQ |
|---|---|---|---|
| **吞吐量** | 100 万+ QPS | 10-15 万 QPS | 1-5 万 QPS |
| **延迟** | 毫秒级 | 毫秒级 | 微秒级 |
| **消息回溯** | ✅ | ✅ | ❌ |
| **事务消息** | 有限支持 | ✅ 原生 | ❌ |
| **延迟消息** | ❌ 需插件 | ✅ 原生 | ✅ 插件 |
| **顺序消息** | ✅ 分区有序 | ✅ 分区有序 | ✅ 单队列有序 |

## 二、功能对比

### 2.1 核心功能对比

| 功能 | Kafka | RocketMQ | RabbitMQ |
|---|---|---|---|
| **消息回溯** | ✅ | ✅ | ❌ |
| **优先级队列** | ❌ | ❌ | ✅ |
| **延迟消息** | ❌ | ✅ 原生 | ✅ 插件 |
| **事务消息** | ✅ 有限 | ✅ 原生 | ❌ |
| **死信队列** | ❌ 需手动 | ✅ | ✅ 原生 |
| **消息过滤** | ✅ 客户端 | ✅ 服务端 | ✅ 服务端 |

### 2.2 架构对比

| 维度 | Kafka | RocketMQ | RabbitMQ |
|---|---|---|---|
| **协调服务** | ZooKeeper / KRaft | NameServer（无状态） | Erlang 分布式 |
| **存储** | 本地文件系统 | 本地文件系统 | Mnesia |
| **主从同步** | ISR | 同步/异步复制 | 镜像队列 |

---

## 三、适用场景对比

### 3.1 场景匹配表

| 场景 | 推荐 | 原因 |
|---|---|---|
| **日志收集** | Kafka | 高吞吐、海量数据 |
| **流处理** | Kafka | 与 Flink、Spark 集成 |
| **电商交易** | RocketMQ | 事务消息、延迟消息 |
| **金融消息** | RocketMQ | 金融级可靠性 |
| **复杂路由** | RabbitMQ | Exchange 灵活路由 |
| **任务队列** | RabbitMQ | 功能丰富、管理界面 |
| **大数据管道** | Kafka | 高吞吐、生态完善 |

### 3.2 选型决策树

```
开始选型
    │
    ▼
是否是日志收集 / 流处理？
    │
    ├── 是 → 选择 Kafka
    │
    └── 否 → 继续
            │
            ▼
        是否需要事务消息或延迟消息？
            │
            ├── 是 → 选择 RocketMQ
            │
            └── 否 → 继续
                    │
                    ▼
                是否需要复杂路由？
                    │
                    ├── 是 → 选择 RabbitMQ
                    │
                    └── 否 → 选 Kafka 或 RocketMQ
                            （根据吞吐和可靠性要求）
```

## 四、技术细节对比

### 4.1 消息顺序

| MQ | 顺序保证 |
|---|---|
| **Kafka** | 分区内有序 |
| **RocketMQ** | 分区内有序 |
| **RabbitMQ** | 单队列有序 |

### 4.2 消息可靠性

| MQ | 可靠性机制 |
|---|---|
| **Kafka** | ISR + HW |
| **RocketMQ** | 主从复制 / DLeger |
| **RabbitMQ** | 镜像队列 |

### 4.3 消息积压

| MQ | 堆积能力 |
|---|---|
| **Kafka** | 无限（依赖磁盘） |
| **RocketMQ** | 百万级 |
| **RabbitMQ** | 数十万级 |

## 五、选型建议

### 5.1 企业选型建议

| 企业类型 | 推荐 | 说明 |
|---|---|---|
| **互联网公司** | Kafka + RocketMQ | 日志用 Kafka，交易用 RocketMQ |
| **电商公司** | RocketMQ | 事务消息、延迟消息 |
| **金融公司** | RocketMQ + DLeger | 金融级可靠性 |
| **中小企业** | RabbitMQ | 功能丰富、易上手 |

### 5.2 团队能力建议

| 团队能力 | 推荐 | 说明 |
|---|---|---|
| **Java 团队** | RocketMQ | Spring 生态集成 |
| **大数据团队** | Kafka | 生态完善 |
| ** Erlang 团队** | RabbitMQ | 原生支持 |

## 六、面试追问预判

### 追问一：Kafka 和 RocketMQ 有什么区别？

| 维度 | Kafka | RocketMQ |
|---|---|---|
| **设计理念** | 日志优先 | 金融级 |
| **事务消息** | 有限支持 | 原生支持 |
| **延迟消息** | 不支持 | 原生支持 |
| **协调服务** | ZooKeeper / KRaft | NameServer（无状态） |

### 追问二：RocketMQ 和 RabbitMQ 有什么区别？

| 维度 | RocketMQ | RabbitMQ |
|---|---|---|
| **事务消息** | 原生支持 | 不支持 |
| **延迟消息** | 原生支持 | 插件支持 |
| **吞吐量** | 高 | 中 |
| **路由** | 基于 Queue | 基于 Exchange |

### 追问三：如何根据业务选择 MQ？

| 优先级 | 场景 | 选择 |
|---|---|---|
| **高吞吐** | 日志收集、流处理 | Kafka |
| **事务** | 订单、支付 | RocketMQ |
| **路由** | 复杂路由 | RabbitMQ |
| **金融级** | 金融交易 | RocketMQ + DLeger |

### 追问四：可以用多个 MQ 吗？

可以。根据不同场景使用不同的 MQ：

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   日志收集 ────────────────────────> Kafka                  │
│                                                             │
│   订单处理 ────────────────────────> RocketMQ               │
│                                                             │
│   任务队列 ────────────────────────> RabbitMQ               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
