# RocketMQ 架构：NameServer、Broker、Producer、Consumer

## 场景切入：RocketMQ 的无状态设计

RocketMQ 的架构和 Kafka 最大的区别是什么？

Kafka 使用 ZooKeeper 管理集群元数据，而 RocketMQ 使用**无状态的 NameServer**。这让 RocketMQ 的架构更加简单，也更容易部署。

### 1.2 核心组件

## 二、NameServer

### 2.1 NameServer 的角色

NameServer 是 RocketMQ 的协调服务：

| 职责 | 说明 |
|---|---|
| **Broker 管理** | 管理 Broker 的注册和心跳 |
| **路由信息** | 提供 Topic 到 Broker 的路由信息 |
| **无状态** | 各 NameServer 之间不通信 |

### 2.2 无状态设计

```
NameServer 无状态架构：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Broker-1 ───────────────┐                               │
│                            │                               │
│   Broker-2 ───────────────┼────> NameServer-1             │
│                            │                               │
│   Broker-3 ───────────────┘                               │
│                                                             │
│   Producer ◄───────────────────────────────── NameServer-1│
│                                                             │
│   NameServer-1 和 NameServer-2 不通信，各自独立             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 路由注册

```java
// Broker 注册到 NameServer
// Broker 每 30 秒发送心跳
// NameServer 超过 2 分钟未收到心跳，删除 Broker 信息

// Topic 路由信息
TopicRouteData route = namesrvProxy.getRouteInfoForTopic("order-topic");

// 返回结果
// - queueDataList: 队列信息
// - brokerDataList: Broker 信息
```

## 三、Broker

### 3.1 Broker 的角色

Broker 是消息存储的核心节点：

| 职责 | 说明 |
|---|---|
| **消息存储** | 存储消息到磁盘 |
| **消息收发** | 处理 Producer 和 Consumer 的请求 |
| **主从同步** | Master 和 Slave 之间同步数据 |

### 3.2 Broker 配置

```properties
# Broker 配置
brokerClusterName=DefaultCluster
brokerName=broker-a
brokerId=0  # 0=Master, 1+ = Slave
namesrvAddr=127.0.0.1:9876
deleteWhen=04
fileReservedTime=48
flushDiskType=ASYNC_FLUSH  # 异步刷盘
brokerRole=SYNC_MASTER  # 同步 Master
```

### 3.3 消息存储结构

```
RocketMQ 存储结构：

/store/
├── commitlog/           # 存储所有消息
│   ├── 00000000000000000000
│   └── 00000000000000001000
├── consumequeue/       # 消费队列（索引）
│   └── topic/
│       └── queue/
├── index/              # 消息索引
│   └── topic/
└── config/            # 配置文件
```

## 四、Producer

### 4.1 Producer 的角色

Producer 负责发送消息：

| 类型 | 说明 |
|---|---|
| **DefaultMQProducer** | 默认生产者 |
| **TransactionMQProducer** | 事务消息生产者 |

### 4.2 Producer 配置

```java
// Producer 配置
DefaultMQProducer producer = new DefaultMQProducer("order-producer-group");
producer.setNamesrvAddr("127.0.0.1:9876");
producer.start();

// 发送消息
Message msg = new Message("order-topic", "order-data".getBytes());
SendResult result = producer.send(msg);
```

## 五、Consumer

### 5.1 Consumer 的角色

Consumer 负责消费消息：

| 类型 | 说明 |
|---|---|
| **DefaultMQPushConsumer** | 推模式消费者 |
| **DefaultMQPullConsumer** | 拉模式消费者 |

### 5.2 Consumer 配置

```java
// Consumer 配置
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer("order-consumer-group");
consumer.setNamesrvAddr("127.0.0.1:9876");
consumer.subscribe("order-topic", "*");

consumer.registerMessageListener(new MessageListenerConcurrently() {
    @Override
    public ConsumeConcurrentlyStatus consumeMessage(List<MessageExt> msgs,
                                                    ConsumeConcurrentlyContext context) {
        for (MessageExt msg : msgs) {
            System.out.println("收到消息：" + new String(msg.getBody()));
        }
        return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;
    }
});

consumer.start();
```

## 六、架构对比

### 6.1 RocketMQ vs Kafka

| 维度 | RocketMQ | Kafka |
|---|---|---|
| **协调服务** | NameServer（无状态） | ZooKeeper（强一致） |
| **存储** | 本地文件系统 | 本地文件系统 |
| **延迟消息** | 原生支持 | 需插件 |
| **事务消息** | 原生支持 | 有限支持 |

### 6.2 NameServer vs ZooKeeper

| 维度 | NameServer | ZooKeeper |
|---|---|---|
| **状态** | 无状态 | 有状态 |
| **一致性** | 最终一致 | 强一致 |
| **复杂度** | 低 | 高 |
| **依赖** | 无 | 需要部署 |

## 七、面试追问预判

### 追问一：NameServer 是如何工作的？

NameServer 是无状态服务：

1. Broker 启动时注册到所有 NameServer
2. Broker 每 30 秒发送心跳
3. NameServer 超过 2 分钟未收到心跳，删除 Broker 信息
4. Producer 和 Consumer 从 NameServer 获取路由信息

### 追问二：NameServer 和 ZooKeeper 有什么区别？

| 维度 | NameServer | ZooKeeper |
|---|---|---|
| **设计** | 无状态 | 有状态 |
| **一致性** | 最终一致 | 强一致 |
| **复杂度** | 简单 | 复杂 |

### 追问三：Broker 的 Master 和 Slave 有什么区别？

| 角色 | 职责 |
|---|---|
| **Master** | 处理读写请求 |
| **Slave** | 同步 Master 数据，提供读服务 |
