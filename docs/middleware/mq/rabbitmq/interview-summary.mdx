## 一、基础概念篇

### 问题一：RabbitMQ 的核心组件有哪些？

**参考答案**：

| 组件 | 说明 |
|---|---|
| **Producer** | 消息生产者，发送消息到 Exchange |
| **Exchange** | 交换机，根据规则路由消息 |
| **Queue** | 队列，存储消息 |
| **Consumer** | 消费者，从队列获取消息 |
| **Binding** | 绑定关系，定义 Exchange 和 Queue 的路由规则 |

```
数据流：
Producer → Exchange → Binding → Queue → Consumer
```

---

### 问题二：Exchange 有哪些类型？

**参考答案**：

| 类型 | 路由规则 | 适用场景 |
|---|---|---|
| **Direct** | 精确匹配路由键 | 点对点消息 |
| **Fanout** | 广播到所有队列 | 发布订阅 |
| **Topic** | 通配符匹配 | 复杂路由 |
| **Headers** | 消息头匹配 | 按属性路由 |

---

### 问题三：消息是如何路由的？

**参考答案**：

```
路由流程：
1. Producer 发送消息到 Exchange
2. Exchange 根据类型和 Binding 规则路由
3. 匹配成功的 Queue 收到消息
4. Consumer 从 Queue 拉取消息
```

---

## 二、可靠性篇

### 问题四：如何保证消息不丢失？

**参考答案**：

需要三个层面都配置持久化：

| 层面 | 配置 |
|---|---|
| **交换机持久化** | `durable=true` |
| **队列持久化** | `durable=true` |
| **消息持久化** | `deliveryMode=2` |

```java
// 三层持久化
channel.exchangeDeclare("exchange", "direct", true);  // 交换机持久化
channel.queueDeclare("queue", true, false, false, null);  // 队列持久化
channel.basicPublish("exchange", "routingKey",
    MessageProperties.PERSISTENT_TEXT_PLAIN,  // 消息持久化
    body);
```

---

### 问题五：消费者如何确认消息？

**参考答案**：

| 方法 | 说明 |
|---|---|
| **ACK** | 确认消息处理成功，删除消息 |
| **NACK** | 拒绝消息，可选是否重入队列 |
| **Reject** | 拒绝消息，不重入队列 |

```java
// 手动 ACK
channel.basicConsume("queue", false, consumer);
channel.basicAck(deliveryTag, false);

// 处理失败拒绝
channel.basicNack(deliveryTag, false, true);  // 重试
channel.basicNack(deliveryTag, false, false);  // 不重试，进入死信
```

## 三、进阶特性篇

### 问题六：什么是死信队列？

**参考答案**：

消息满足以下条件时，成为死信：

| 条件 | 说明 |
|---|---|
| **消息被拒绝** | Consumer 拒绝且不重入队列 |
| **消息过期** | TTL 到期 |
| **队列满** | 超过最大长度 |

```java
// 配置死信队列
Map<String, Object> args = new HashMap<>();
args.put("x-dead-letter-exchange", "dlx-exchange");
channel.queueDeclare("queue", true, false, false, args);
```

---

### 问题七：如何实现延迟消息？

**参考答案**：

两种方式：

| 方式 | 原理 |
|---|---|
| **TTL + DLX** | 消息过期后进入死信队列 |
| **延迟插件** | `rabbitmq_delayed_message_exchange` 插件 |

```java
// TTL + DLX 延迟消息
Map<String, Object> args = new HashMap<>();
args.put("x-message-ttl", 10000);  // 10 秒
args.put("x-dead-letter-exchange", "dlx");
channel.queueDeclare("delay-queue", true, false, false, args);
```

---

### 问题八：RabbitMQ 有哪些集群模式？

**参考答案**：

| 模式 | 特点 | 高可用 |
|---|---|---|
| **普通集群** | 队列只在 Master 节点 | 低 |
| **镜像集群** | 队列镜像到所有节点 | 中 |
| **仲裁队列** | Raft 共识算法 | 高 |

## 四、实战应用篇

### 问题九：RabbitMQ 和 Kafka 有什么区别？

**参考答案**：

| 维度 | RabbitMQ | Kafka |
|---|---|---|
| **设计理念** | 消息路由 | 日志优先 |
| **吞吐量** | 1-5 万 QPS | 100 万+ QPS |
| **延迟** | 微秒级 | 毫秒级 |
| **路由** | Exchange + Binding | 分区 |
| **消息回溯** | 不支持 | 支持 |
| **事务** | 不支持 | 有限支持 |

---

### 问题十：如何选择 RabbitMQ 和 Kafka？

**参考答案**：

| 场景 | 选择 | 原因 |
|---|---|---|
| **日志收集** | Kafka | 高吞吐 |
| **复杂路由** | RabbitMQ | Exchange 灵活 |
| **任务队列** | RabbitMQ | 功能丰富 |
| **流处理** | Kafka | 生态完善 |
