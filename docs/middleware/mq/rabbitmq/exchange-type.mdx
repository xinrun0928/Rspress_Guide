# RabbitMQ 交换机类型：Direct、Fanout、Topic、Headers

## 场景切入：路由规则决定消息命运

消息发送到 Exchange 后，是如何路由到 Queue 的？

不同的 Exchange 类型有不同的路由规则。Direct 精确匹配，Fanout 广播所有，Topic 通配符匹配，Headers 按消息头匹配。选对 Exchange 类型，才能实现正确的消息分发。

## 一、四种交换机类型对比

| 类型 | 路由规则 | 复杂度 | 适用场景 |
|---|---|---|---|
| **Direct** | 完全匹配路由键 | 简单 | 点对点消息 |
| **Fanout** | 广播到所有队列 | 简单 | 发布订阅 |
| **Topic** | 通配符匹配路由键 | 中等 | 复杂路由 |
| **Headers** | 消息头匹配 | 复杂 | 按属性路由 |

---

## 二、Direct Exchange

### 2.1 原理

Direct Exchange 根据**完全匹配**的路由键进行路由：

```
Direct Exchange 路由：
Exchange: order-exchange (Direct)
Binding:
  order-queue ──"order.created"──> order-exchange
  payment-queue ──"payment.created"──> order-exchange

消息路由：
消息 1 (routingKey="order.created") ──> order-queue ✓
消息 2 (routingKey="payment.created") ──> payment-queue ✓
消息 3 (routingKey="order.updated") ──> 无匹配，丢弃 ✗
```

### 2.2 代码示例

```java
// Direct Exchange 示例
// 1. 声明 Direct 交换机
channel.exchangeDeclare("order-exchange", "direct", true);

// 2. 声明队列
channel.queueDeclare("order-queue", true, false, false, null);
channel.queueDeclare("payment-queue", true, false, false, null);

// 3. 绑定队列到交换机
channel.queueBind("order-queue", "order-exchange", "order.created");
channel.queueBind("payment-queue", "order-exchange", "payment.created");

// 4. 发送消息
channel.basicPublish("order-exchange", "order.created", null, "订单消息".getBytes());
channel.basicPublish("order-exchange", "payment.created", null, "支付消息".getBytes());
```

### 2.3 使用场景

| 场景 | 说明 |
|---|---|
| **订单消息** | 订单创建、支付、退款等不同类型的消息 |
| **通知消息** | 短信、邮件、推送不同渠道的通知 |
| **日志消息** | ERROR、WARN、INFO 不同级别的日志 |

## 三、Fanout Exchange

### 3.1 原理

Fanout Exchange 将消息**广播**到所有绑定的队列，忽略路由键：

```
Fanout Exchange 路由：
Exchange: notification-exchange (Fanout)
Binding:
  email-queue ───────────────────> notification-exchange
  sms-queue ────────────────────> notification-exchange
  push-queue ──────────────────> notification-exchange

消息路由：
消息 ──> notification-exchange ──广播──> email-queue
                                       ──广播──> sms-queue
                                       ──广播──> push-queue

所有绑定的队列都能收到消息
```

### 3.2 代码示例

```java
// Fanout Exchange 示例
// 1. 声明 Fanout 交换机
channel.exchangeDeclare("notification-exchange", "fanout", true);

// 2. 声明多个队列
channel.queueDeclare("email-queue", true, false, false, null);
channel.queueDeclare("sms-queue", true, false, false, null);
channel.queueDeclare("push-queue", true, false, false, null);

// 3. 绑定所有队列到交换机
channel.queueBind("email-queue", "notification-exchange", "");
channel.queueBind("sms-queue", "notification-exchange", "");
channel.queueBind("push-queue", "notification-exchange", "");

// 4. 发送消息（所有队列都能收到）
channel.basicPublish("notification-exchange", "", null, "通知消息".getBytes());
```

### 3.3 使用场景

| 场景 | 说明 |
|---|---|
| **系统通知** | 广播系统公告给所有服务 |
| **日志收集** | 日志同时发送到 ELK、监控、归档 |
| **事件总线** | 同一事件触发多个下游处理 |

## 四、Topic Exchange

### 4.1 原理

Topic Exchange 根据**通配符匹配**的路由键进行路由：

| 通配符 | 含义 | 示例 |
|---|---|---|
| `*` | 匹配一个词 | `order.*` 匹配 `order.created`、`order.updated` |
| `#` | 匹配零个或多个词 | `order.#` 匹配 `order`、`order.created`、`order.created.paid` |

```
Topic Exchange 路由：
Exchange: topic-exchange (Topic)
Binding:
  queue-a ──"*.created"──> topic-exchange    ── 匹配一个词
  queue-b ──"order.#"────> topic-exchange    ── 匹配零个或多个词
  queue-c ──"order.*"────> topic-exchange    ── 匹配一个词

消息路由：
消息 1 (routingKey="order.created")
  ✓ queue-a (*.created 匹配)
  ✓ queue-b (order.# 匹配)
  ✓ queue-c (order.* 匹配)

消息 2 (routingKey="payment.created")
  ✓ queue-a (*.created 匹配)
  ✗ queue-b (不匹配)
  ✗ queue-c (不匹配)

消息 3 (routingKey="order.created.paid")
  ✗ queue-a (不匹配)
  ✓ queue-b (order.# 匹配)
  ✗ queue-c (不匹配)
```

### 4.2 代码示例

```java
// Topic Exchange 示例
// 1. 声明 Topic 交换机
channel.exchangeDeclare("topic-exchange", "topic", true);

// 2. 声明多个队列
channel.queueDeclare("order-created-queue", true, false, false, null);
channel.queueDeclare("all-order-queue", true, false, false, null);
channel.queueDeclare("payment-queue", true, false, false, null);

// 3. 绑定队列到交换机（使用通配符）
channel.queueBind("order-created-queue", "topic-exchange", "order.created");
channel.queueBind("all-order-queue", "topic-exchange", "order.#");
channel.queueBind("payment-queue", "topic-exchange", "payment.*");

// 4. 发送消息
channel.basicPublish("topic-exchange", "order.created", null, "订单创建".getBytes());
channel.basicPublish("topic-exchange", "order.updated", null, "订单更新".getBytes());
channel.basicPublish("topic-exchange", "payment.created", null, "支付创建".getBytes());
```

### 4.3 使用场景

| 场景 | 说明 |
|---|---|
| **日志路由** | `log.#`、`error.#`、`warn.#` |
| **事件路由** | `user.created`、`user.updated`、`order.*` |
| **多租户路由** | `tenant-{id}.event.#` |

## 五、Headers Exchange

### 5.1 原理

Headers Exchange 根据**消息头的属性**进行路由：

| 参数 | 含义 |
|---|---|
| `x-match=all` | 所有属性必须匹配 |
| `x-match=any` | 任意属性匹配 |

```
Headers Exchange 路由：
Exchange: headers-exchange (Headers)
Binding (x-match=all):
  queue-a ──{format:json, type:order}──> headers-exchange
  queue-b ──{format:json, type:payment}──> headers-exchange
Binding (x-match=any):
  queue-c ──{format:json, type:order}──> headers-exchange

消息路由：
消息 1 (headers={format:json, type:order})
  ✓ queue-a (all 匹配)
  ✗ queue-b (all 不匹配)
  ✓ queue-c (any 匹配)

消息 2 (headers={format:json})
  ✗ queue-a (all 不匹配)
  ✗ queue-b (all 不匹配)
  ✗ queue-c (any 不匹配)
```

### 5.2 代码示例

```java
// Headers Exchange 示例
// 1. 声明 Headers 交换机
channel.exchangeDeclare("headers-exchange", "headers", true);

// 2. 声明队列
channel.queueDeclare("json-order-queue", true, false, false, null);
channel.queueDeclare("json-payment-queue", true, false, false, null);

// 3. 绑定队列（x-match=all）
Map<String, Object> orderArgs = new HashMap<>();
orderArgs.put("x-match", "all");
orderArgs.put("format", "json");
orderArgs.put("type", "order");
channel.queueBind("json-order-queue", "headers-exchange", "", orderArgs);

// 4. 绑定队列（x-match=any）
Map<String, Object> paymentArgs = new HashMap<>();
paymentArgs.put("x-match", "any");
paymentArgs.put("format", "json");
paymentArgs.put("type", "payment");
channel.queueBind("json-payment-queue", "headers-exchange", "", paymentArgs);

// 5. 发送消息（指定消息头）
Map<String, Object> headers = new HashMap<>();
headers.put("format", "json");
headers.put("type", "order");
AMQP.BasicProperties properties = new AMQP.BasicProperties.Builder()
    .headers(headers)
    .build();
channel.basicPublish("headers-exchange", "", properties, "消息".getBytes());
```

### 5.3 使用场景

| 场景 | 说明 |
|---|---|
| **多属性路由** | 根据消息的多个属性决定路由 |
| **内容路由** | 根据消息内容类型路由 |
| **版本路由** | 根据 API 版本路由 |

## 六、交换机类型对比

### 6.1 路由能力对比

| 类型 | 路由键匹配 | 多队列 | 灵活性 |
|---|---|---|---|
| **Direct** | 精确匹配 | 单队列 | 低 |
| **Fanout** | 无需匹配 | 所有队列 | 低 |
| **Topic** | 通配符匹配 | 多个队列 | 中 |
| **Headers** | 消息头匹配 | 多个队列 | 高 |

### 6.2 性能对比

| 类型 | 性能 | 说明 |
|---|---|---|
| **Direct** | 最高 | 简单哈希匹配 |
| **Fanout** | 高 | 无需匹配，直接广播 |
| **Topic** | 中 | 需要通配符解析 |
| **Headers** | 较低 | 需要匹配多个头 |

## 七、面试追问预判

### 追问一：如何实现消息的发布订阅？

使用 Fanout 类型的 Exchange：

```java
// 发布订阅
channel.exchangeDeclare("pub-sub-exchange", "fanout", true);
channel.queueBind("queue-a", "pub-sub-exchange", "");
channel.queueBind("queue-b", "pub-sub-exchange", "");
channel.basicPublish("pub-sub-exchange", "", null, message);
```

### 追问二：Topic 和 Headers Exchange 有什么区别？

| 类型 | 匹配方式 | 灵活性 |
|---|---|---|
| **Topic** | 路由键通配符匹配 | 中 |
| **Headers** | 消息头属性匹配 | 高 |

Topic 适合按路由键模式匹配，Headers 适合按多个属性匹配。

### 追问三：交换机可以重复声明吗？

可以，但参数必须一致。如果参数不一致，会抛出异常。

```java
// 幂等声明
channel.exchangeDeclare("exchange-name", "direct", true);
// true 表示如果已存在则不创建
```
