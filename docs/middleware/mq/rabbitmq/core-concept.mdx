# RabbitMQ 核心概念：Exchange、Queue、Binding、RouteKey

## 场景切入：RabbitMQ 的路由魔法

RabbitMQ 和 Kafka 的最大区别是什么？

Kafka 通过分区实现消息路由，Consumer Group 决定谁能消费。而 RabbitMQ 通过**交换机（Exchange）** 和**绑定（Binding）** 实现消息路由，这给了 RabbitMQ 更灵活的路由能力。

理解这四个核心概念，就理解了 RabbitMQ 的一半。

## 一、四大核心概念

### 1.1 概念总览

| 概念 | 说明 | 类比 |
|---|---|---|
| **Producer** | 消息生产者 | 快递员 |
| **Exchange** | 交换机，决定消息路由到哪个队列 | 快递分拣中心 |
| **Queue** | 队列，存储消息 | 快递柜 |
| **Consumer** | 消息消费者 | 收件人 |

```
数据流：
Producer ──发送消息──> Exchange ──路由规则──> Queue ──推送──> Consumer
                                    │
                                    ├── Binding (路由规则)
                                    └── RouteKey (路由键)
```

### 1.2 Exchange 的角色

Exchange 是 RabbitMQ 的核心组件，负责：

| 职责 | 说明 |
|---|---|
| **接收消息** | 接收 Producer 发送的消息 |
| **路由分发** | 根据 Binding 规则将消息分发到队列 |
| **类型决定路由** | 不同类型的 Exchange 有不同的路由算法 |

### 1.3 Queue 的角色

Queue 是消息存储的容器：

| 特性 | 说明 |
|---|---|
| **FIFO** | 队列中的消息按先进先出顺序消费 |
| **持久化** | 可以配置队列是否持久化 |
| **排他性** | 可以配置队列是否只属于一个 Connection |
| **自动删除** | 可以配置队列是否在最后一个 Consumer 离开后自动删除 |

### 1.4 Binding 的角色

Binding 是 Exchange 和 Queue 之间的绑定关系：

| 属性 | 说明 |
|---|---|
| **Exchange** | 绑定的交换机 |
| **Queue** | 绑定的队列 |
| **Routing Key** | 路由键（可选） |
| **Arguments** | 额外参数 |

## 二、Exchange 类型

### 2.1 四种 Exchange 类型

| 类型 | 路由规则 | 使用场景 |
|---|---|---|
| **Direct** | 完全匹配路由键 | 点对点消息 |
| **Fanout** | 广播到所有队列 | 发布订阅 |
| **Topic** | 按通配符匹配路由键 | 复杂路由 |
| **Headers** | 按消息头匹配 | 按属性路由 |

```
四种 Exchange 类型：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Direct ──────> Routing Key 完全匹配 ──────> Queue         │
│                                                             │
│  Fanout ──────> 广播到所有队列 ──────────> Queue A, B, C  │
│                                                             │
│  Topic ───────> 通配符匹配 ──────────────────> Queue        │
│               (* 匹配一个词, # 匹配多个词)                  │
│                                                             │
│  Headers ─────> 消息头匹配 ───────────────> Queue         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 数据流详解

```
完整数据流：
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   Producer                                                      │
│       │                                                         │
│       │ basicPublish(routingKey, message)                       │
│       │                                                         │
│       ▼                                                         │
│   Exchange                                                      │
│       │                                                         │
│       │ 根据类型和 Binding 规则路由                               │
│       │                                                         │
│       ├──> Queue A ──> Consumer 1                              │
│       │                                                         │
│       ├──> Queue B ──> Consumer 2                              │
│       │                                                         │
│       └──> Queue C ──> Consumer 3                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三、消息流生命周期

### 3.1 消息发送流程

```java
// 消息发送流程
Connection connection = connectionFactory.newConnection();
Channel channel = connection.createChannel();

// 1. 声明交换机
channel.exchangeDeclare("order-exchange", "direct", true);

// 2. 声明队列
channel.queueDeclare("order-queue", true, false, false, null);

// 3. 绑定交换机和队列
channel.queueBind("order-queue", "order-exchange", "order.created");

// 4. 发送消息
channel.basicPublish(
    "order-exchange",     // 交换机名
    "order.created",      // 路由键
    MessageProperties.PERSISTENT_TEXT_PLAIN,  // 消息属性
    "order data".getBytes()  // 消息体
);
```

### 3.2 消息消费流程

```java
// 消息消费流程
// 1. 创建 Connection 和 Channel
Connection connection = connectionFactory.newConnection();
Channel channel = connection.createChannel();

// 2. 设置 QoS
channel.basicQos(10);  // 每次只给消费者 10 条消息

// 3. 消费消息
channel.basicConsume(
    "order-queue",      // 队列名
    false,              // 不自动确认
    "consumer-tag",     // 消费者标签
    new DefaultConsumer(channel) {
        @Override
        public void handleDelivery(String consumerTag,
                                 Envelope envelope,
                                 AMQP.BasicProperties properties,
                                 byte[] body) {
            String message = new String(body);
            System.out.println("收到消息：" + message);
            // 4. 处理完成后确认
            channel.basicAck(envelope.getDeliveryTag(), false);
        }
    }
);
```

## 四、Connection 与 Channel

### 4.1 Connection

Connection 是 AMQP 连接：

| 特性 | 说明 |
|---|---|
| **TCP 连接** | 基于 TCP 的长连接 |
| **重量级** | 创建和销毁开销大 |
| **多路复用** | 一个 Connection 可以有多个 Channel |
| **自动重连** | 连接断开后可以自动重连 |

### 4.2 Channel

Channel 是 AMQP 虚拟连接：

| 特性 | 说明 |
|---|---|
| **轻量级** | 创建和销毁开销小 |
| **线程隔离** | 每个线程使用独立的 Channel |
| **共享连接** | 多个 Channel 共享一个 Connection |
| **顺序保证** | 同一 Channel 内的操作有序 |

```
Connection 与 Channel 的关系：
┌─────────────────────────────────────────────┐
│              Connection (TCP)                 │
│                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐    │
│  │Channel 1│  │Channel 2│  │Channel 3│    │
│  └─────────┘  └─────────┘  └─────────┘    │
│                                             │
└─────────────────────────────────────────────┘
```

## 五、Virtual Host

### 5.1 概念

Virtual Host 是 RabbitMQ 的逻辑隔离单元：

| 特性 | 说明 |
|---|---|
| **隔离性** | 不同 Virtual Host 互相隔离 |
| **独立配置** | 每个 Virtual Host 有独立的 Exchange、Queue、User |
| **默认 Virtual Host** | 默认的 Virtual Host 名为 `/` |
| **用途** | 多租户隔离、测试环境隔离 |

### 5.2 配置使用

```java
// 创建连接时指定 Virtual Host
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("localhost");
factory.setVirtualHost("my-vhost");  // 指定 Virtual Host
factory.setUsername("user");
factory.setPassword("password");
Connection connection = factory.newConnection();
```

## 六、面试追问预判

### 追问一：Exchange 和 Queue 的区别是什么？

| 组件 | 作用 |
|---|---|
| **Exchange** | 接收消息，根据规则路由到队列 |
| **Queue** | 存储消息，等待消费者拉取 |

Exchange 负责路由，Queue 负责存储。

### 追问二：多个 Consumer 订阅同一个 Queue 会怎样？

多个 Consumer 订阅同一个 Queue 时，消息会被**轮询分发**：

```
多个 Consumer 订阅同一个 Queue：
Queue: [msg1, msg2, msg3, msg4, msg5]
Consumer A: msg1, msg3, msg5
Consumer B: msg2, msg4
消息被轮询分发，每个消息只会被一个 Consumer 消费
```

### 追问三：如何实现消息的发布订阅？

使用 Fanout 类型的 Exchange：

```java
// 发布订阅模式
channel.exchangeDeclare("notification-exchange", "fanout", true);

// 多个队列绑定到同一个交换机
channel.queueBind("email-queue", "notification-exchange", "");
channel.queueBind("sms-queue", "notification-exchange", "");
channel.queueBind("push-queue", "notification-exchange", "");

// 发送消息，所有绑定的队列都能收到
channel.basicPublish("notification-exchange", "", null, message.getBytes());
```
