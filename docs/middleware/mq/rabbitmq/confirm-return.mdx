# RabbitMQ 消息确认机制：Confirm 与 Return

## 场景切入：消息发出去了，真的送到了吗？

你发送了一条消息到 Exchange，但你不确定它是否路由到了 Queue。

RabbitMQ 提供了两种确认机制：**Confirm** 和 **Return**。Confirm 确认消息到达了 Broker，Return 处理消息无法路由的情况。

## 一、Confirm 确认模式

### 1.1 什么是 Confirm

Confirm 机制用于确认消息是否成功到达 Broker：

| 状态 | 说明 |
|---|---|
| **ACK** | 消息成功到达 Broker |
| **NACK** | 消息到达 Broker 失败 |

```java
// 开启 Confirm 模式
channel.confirmSelect();

// 发送消息
channel.basicPublish(
    "order-exchange",
    "order.created",
    MessageProperties.PERSISTENT_TEXT_PLAIN,
    "order data".getBytes()
);

// 等待确认
if (channel.waitForConfirms()) {
    System.out.println("消息发送成功");
} else {
    System.out.println("消息发送失败");
}
```

### 1.2 异步 Confirm

```java
// 异步 Confirm
channel.confirmSelect();

channel.basicPublish(
    "order-exchange",
    "order.created",
    MessageProperties.PERSISTENT_TEXT_PLAIN,
    "order data".getBytes()
);

// 添加确认监听器
channel.addConfirmListener(new ConfirmListener() {
    @Override
    public void handleAck(long deliveryTag, boolean multiple) {
        // 消息被确认
        System.out.println("消息确认：tag=" + deliveryTag);
    }

    @Override
    public void handleNack(long deliveryTag, boolean multiple) {
        // 消息未被确认
        System.out.println("消息拒绝：tag=" + deliveryTag);
    }
});
```

### 1.3 批量 Confirm

```java
// 批量 Confirm
channel.confirmSelect();

for (int i = 0; i < 1000; i++) {
    channel.basicPublish(
        "order-exchange",
        "order.created",
        MessageProperties.PERSISTENT_TEXT_PLAIN,
        ("order " + i).getBytes()
    );
}

// 等待所有确认
channel.waitForConfirmsOrDie(5000);
```

## 二、Return 退信模式

### 2.1 什么是 Return

Return 机制用于处理消息无法路由到队列的情况：

| 场景 | 说明 |
|---|---|
| **消息无法路由** | 没有匹配的队列 |
| **队列不存在** | 交换机绑定的队列不存在 |

```
Return 场景：

Producer ──发送消息──> Exchange ──路由──> Queue
                                         │
                                         │ 无匹配队列
                                         ▼
                                    Return (无法路由)

可能的原因：
1. 路由键不匹配
2. 队列不存在
3. 队列已满
```

### 2.2 Return 配置

```java
// 配置 Return 监听器
channel.addReturnListener(new ReturnListener() {
    @Override
    public void handleReturn(int replyCode,
                           String replyText,
                           String exchange,
                           String routingKey,
                           AMQP.BasicProperties properties,
                           byte[] body) {
        System.out.println("消息无法路由：");
        System.out.println("replyCode=" + replyCode);
        System.out.println("replyText=" + replyText);
        System.out.println("exchange=" + exchange);
        System.out.println("routingKey=" + routingKey);

        // 处理无法路由的消息
        // 1. 记录日志
        // 2. 发送到死信队列
        // 3. 重试发送
    }
});

// 发送消息（必须设置 mandatory=true）
channel.basicPublish(
    "order-exchange",
    "order.unknown",      // 不存在的路由键
    true,                 // mandatory=true，消息无法路由时返回
    MessageProperties.PERSISTENT_TEXT_PLAIN,
    "order data".getBytes()
);
```

## 三、Confirm 与 Return 对比

### 3.1 对比表

| 机制 | 作用 | 触发时机 |
|---|---|---|
| **Confirm** | 确认消息到达 Broker | 消息写入 Broker |
| **Return** | 处理无法路由的消息 | 消息无法路由到队列 |

### 3.2 组合使用

```java
// 组合使用 Confirm 和 Return
channel.confirmSelect();

channel.addConfirmListener(new ConfirmListener() {
    @Override
    public void handleAck(long deliveryTag, boolean multiple) {
        System.out.println("消息确认：tag=" + deliveryTag);
    }

    @Override
    public void handleNack(long deliveryTag, boolean multiple) {
        System.out.println("消息拒绝：tag=" + deliveryTag);
        // 处理发送失败
    }
});

channel.addReturnListener(new ReturnListener() {
    @Override
    public void handleReturn(int replyCode,
                           String replyText,
                           String exchange,
                           String routingKey,
                           AMQP.BasicProperties properties,
                           byte[] body) {
        System.out.println("消息无法路由：routingKey=" + routingKey);
        // 处理无法路由的消息
    }
});

channel.basicPublish(
    "order-exchange",
    "order.created",
    true,  // mandatory=true
    MessageProperties.PERSISTENT_TEXT_PLAIN,
    "order data".getBytes()
);
```

## 四、面试追问预判

### 追问一：Confirm 和 Return 有什么区别？

| 机制 | 作用 | 触发时机 |
|---|---|---|
| **Confirm** | 确认消息到达 Broker | 消息写入 Broker |
| **Return** | 处理无法路由的消息 | 消息无法路由到队列 |

### 追问二：mandatory 参数是什么意思？

`mandatory=true` 表示消息无法路由时，Broker 会返回给 Producer。如果设置为 `false`，消息会直接丢弃。

### 追问三：如何保证消息一定到达 Broker？

1. 开启 Confirm 模式
2. 发送后等待确认
3. 确认失败时重试发送
