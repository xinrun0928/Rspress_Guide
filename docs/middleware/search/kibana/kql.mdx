# KQL 查询语言

当你第一次在 Kibana 的搜索框里输入内容时，可能会有一种似曾相识的感觉——这不就是 Google 搜索吗？

没错，KQL（Kibana Query Language）的设计目标就是让查询像 Google 搜索一样简单。但它的能力远不止于此。

## KQL 基础

### 字段查询

最基本的查询是指定字段和值：

```kql
status:200
```

这会返回所有 `status` 字段值为 `200` 的文档。如果字段值是字符串，记得加引号：

```kql
service:"order-service"
```

### 逻辑运算

使用 `AND`、`OR`、`NOT` 组合多个条件：

```kql
status:500 AND service:"order-service"
```

```kql
method:GET OR method:POST
```

```kql
status:200 AND NOT service:"health-check"
```

:::tip AND 和 OR 的优先级
KQL 中 AND 的优先级高于 OR。如果不加括号，会先计算 AND。习惯 SQL 写法的同学要注意这个区别。
:::

### 范围查询

对于数值或日期字段，可以使用范围查询：

```kql
price >= 100 AND price <= 500
```

```kql
@timestamp >= "2024-01-01" AND @timestamp <= "2024-01-31"
```

简写形式：

```kql
price:[100 TO 500]
@timestamp:[2024-01-01 TO 2024-01-31]
```

### 通配符查询

使用 `*` 匹配任意字符：

```kql
path:/api/user/*
```

使用 `?` 匹配单个字符：

```kql
path:/api/user/?
```

## Lucene 语法

除了 KQL，Kibana 还支持 Lucene 查询语法。在搜索框右侧切换查询语言即可。

### 基础语法

```lucene
status:500
service:"order-service"
```

### 通配符

```lucene
path:/api/user/*
```

### 模糊搜索

使用 `~` 进行模糊匹配：

```lucene
order~   # 匹配 order、orders、ordering 等
```

### 正则表达式

使用斜杠包裹正则表达式：

```lucene
service:/order[-_]service[0-9]*/
```

## KQL vs Lucene

| 特性 | KQL | Lucene |
|---|---|---|
| 语法风格 | 自然语言友好 | 更接近编程语言 |
| 自动补全 | 支持 | 不支持 |
| 范围查询 | `>=`、`TO` 都支持 | 仅 `TO` 语法 |
| 布尔运算 | `AND`、`OR`、`NOT` | `AND`、`OR`、`NOT`、`&&`、`||`、`!` |
| 存在性检查 | `exists:field` | `exists:field` |

:::tip 选择建议
对于简单查询，KQL 更直观。对于复杂查询（如正则、模糊搜索），Lucene 语法更强大。
:::

## 常用查询场景

### 查找错误日志

```kql
level:ERROR AND @timestamp:[now-1h TO now]
```

### 查找慢请求

```kql
response_time > 5000 AND @timestamp:[now-24h TO now]
```

### 查找特定用户的操作

```kql
user_id:"user-12345" AND @timestamp:[now-7d TO now]
```

### 排除健康检查

```kql
NOT path:"/health" AND NOT path:"/ping"
```

### 组合复杂查询

```kql
(
  status:500 OR status:502 OR status:503 OR status:504
) AND
(
  service:"order-service" OR service:"payment-service"
) AND
@timestamp:[now-1h TO now]
```

## 保存和分享

### 保存查询

点击搜索框右侧的保存按钮，输入查询名称即可保存。保存后的查询可以在 Discover 的「Saved Queries」中快速加载。

### 告警规则中使用

在创建告警规则时，可以使用 KQL 定义触发条件：

```kql
status:500 AND response_time > 10000
```

当这个条件匹配超过 10 条文档时，触发告警。

## 性能注意事项

### 避免通配符开头

```kql
# 慢：通配符在开头，无法使用索引
path:*user*

# 快：通配符在结尾，可以使用索引
path:/api/user*
```

### 精确值优于模糊匹配

```kql
# 较快：精确匹配
service:"order-service"

# 较慢：模糊匹配，性能随数据量下降
service:order*
```

### 范围查询优化

尽量缩小范围：

```kql
# 好：时间范围明确
@timestamp:[now-1h TO now]

# 差：时间范围过大
@timestamp:[now-30d TO now]
```

## KQL 函数

KQL 还支持一些内置函数：

### 日期函数

```kql
@timestamp > now-1d    # now 函数返回当前时间
```

### 字符串函数

```kql
message:join(*, *)     # 包含 join 和任意字符
message:split          # 包含 split
```

### 存在性检查

```kql
exists:user_id         # user_id 字段存在
NOT exists:user_id     # user_id 字段不存在
```

掌握 KQL，就掌握了在 Kibana 中高效探索数据的钥匙。多练习，多查询，你会发现数据中的规律。
