# Timelion 时序分析

在 Kibana 的众多功能中，Timelion 是一个独特的存在。它不像其他可视化工具那样需要配置聚合、选择图表类型——它更像是用一门小型查询语言来描述你的数据。

如果你熟悉 JavaScript，Timelion 会让你有宾至如归的感觉。如果不熟悉也没关系，它的语法非常简洁易学。

## Timelion 是什么？

Timelion 是 Kibana 内置的时序数据分析和可视化工具。它诞生于一个黑客马拉松项目，后来被官方收购并集成到 Kibana 中。

与普通可视化不同，Timelion 的核心是**表达式**。你通过编写表达式来告诉 Timelion 你想看什么数据、如何处理、如何展示。

```txt
.es(index=metrics-*), qos=*.http_server_requests_seconds_mean gauge
```

这行表达式展示了所有 HTTP 服务器请求的平均响应时间。

## 基础语法

### 数据源表达式

`.es()` 是 Elasticsearch 数据源，也是最常用的表达式：

```txt
.es(index=nginx-logs-*)
```

`.qds()` 是 Direct Elasticsearch 数据源，支持原始 Lucene 查询：

```txt
.qds(q=host:web*, metric=count)
```

### 链式操作

Timelion 的强大之处在于链式操作。每个表达式后面可以链上多个函数：

```txt
.es(index=nginx-logs-*).趋势().移动平均(window=10)
```

## 常用函数

### `.label()`

为数据系列添加标签：

```txt
.es(index=nginx-logs-*, metric=avg:response_time).label("平均响应时间")
```

### `.color()`

设置线条颜色：

```txt
.es(index=nginx-logs-*, metric=avg:response_time)
  .label("平均响应时间")
  .color(蓝色)
```

颜色支持多种格式：

- 颜色名称：`red`、`blue`、`green`
- 十六进制：`#FF5733`
- RGB：`rgb(255, 87, 51)`

### `.lines()`

自定义线条样式：

```txt
.es(index=nginx-logs-*).lines(fill=3, width=2)
```

参数说明：

| 参数 | 说明 | 范围 |
|---|---|---|
| fill | 填充透明度 | 0-10 |
| width | 线条宽度 | 0-10 |
| show | 是否显示线条 | true/false |
| shape | 线条形状 | linear/smooth/step |

### `.points()`

添加数据点标记：

```txt
.es(index=nginx-logs-*).points(radius=3)
```

## 数据处理函数

### `.divide()`

除法计算，常用于计算比率：

```txt
.es(index=nginx-logs-*, metric=count).divide(
  .es(index=nginx-logs-*, metric=count)
)
```

### `.subtract()`

减法，常用于计算差值或变化量：

```txt
.es(index=system-*, metric=avg:mem.used.pct).subtract(
  .es(index=system-*, metric=avg:mem.free.pct)
)
```

### `.multiply()`

乘法，常用于单位换算：

```txt
.es(index=nginx-logs-*, metric=avg:response_time).multiply(1000)
```

将秒转换为毫秒。

### `.abs()`

取绝对值：

```txt
.es().abs()
```

## 时间窗口函数

### `.movingaverage()`

移动平均，平滑数据波动：

```txt
.es(index=nginx-logs-*, metric=avg:response_time)
  .movingaverage(10)
```

参数 `10` 表示窗口大小为 10 个数据点。也可以使用时间单位：

```txt
.es().movingaverage(1h)
```

### `.holt()`

Holt-Winters 预测算法，可以预测未来趋势：

```txt
.es(index=nginx-logs-*, metric=count)
  .holt(alpha=0.3, beta=0.1)
```

参数：

- `alpha`：平滑系数，值越小越平滑
- `beta`：趋势系数

### `.trend()`

添加线性趋势线：

```txt
.es(index=nginx-logs-*).trend()
```

## 可视化增强

### `.title()`

设置图表标题：

```txt
.es(index=nginx-logs-*).title("请求量趋势")
```

### `.legend()`

设置图例位置：

```txt
.es().legend(position=nw)  # 西北角（左上）
```

### `.yaxis()`

自定义 Y 轴：

```txt
.es().yaxis(min=0, max=100, position=left)
```

## 实战案例

### 案例一：分析网站流量趋势

```txt
.es(index=nginx-logs-*, metric=count)
  .label("请求量")
  .title("网站流量趋势")
  .lines(fill=1)
  .color(蓝色)
```

### 案例二：比较多个服务的响应时间

```txt
.es(index=app-logs-*, service=order, metric=avg:response_time)
  .label("订单服务")
  .color(蓝色),
  
.es(index=app-logs-*, service=payment, metric=avg:response_time)
  .label("支付服务")
  .color(绿色),
  
.es(index=app-logs-*, service=user, metric=avg:response_time)
  .label("用户服务")
  .color(橙色)
```

### 案例三：计算错误率

```txt
.es(index=nginx-logs-*, status=500, metric=count)
  .divide(.es(index=nginx-logs-*, metric=count))
  .multiply(100)
  .label("错误率 (%)")
  .yaxis(min=0, max=10)
```

### 案例四：检测异常点

使用标准差检测异常：

```txt
.es(index=nginx-logs-*, metric=avg:response_time)
  .divide(.mvavg(30))
```

当比值远离 1 时，表示数据异常。

### 案例五：周同比分析

```txt
.es(index=nginx-logs-*, metric=count)
  .label("本周"),
  
.es(index=nginx-logs-*, metric=count, offset=-1w)
  .label("上周")
```

`offset=-1w` 表示向前偏移一周，非常适合做周环比。

## 性能注意事项

### 限制时间范围

Timelion 的计算量随时间范围增大而增加。避免查询过大的时间范围：

```txt
# 好：指定明确的时间范围
.es(index=nginx-logs-*, timefilter=("now-1d", "now"))

# 差：使用全量数据
.es(index=nginx-logs-*)
```

### 减少链式操作

链式操作每一步都会消耗资源。合并可以合并的操作：

```txt
# 优化前
.es().label("A").color(red)
.es().label("B").color(blue)

# 优化后：在同一个表达式中定义多个数据源
.es().label("A"), .es().label("B")
```

## 与普通可视化的对比

| 特性 | Timelion | 普通可视化 |
|---|---|---|
| 配置方式 | 表达式语言 | 图形界面 |
| 学习曲线 | 较陡 | 平缓 |
| 灵活性 | 高 | 中 |
| 多数据源对比 | 简单 | 较复杂 |
| 自定义计算 | 强大 | 有限 |

:::tip Timelion 使用建议
Timelion 适合高级用户和复杂分析场景。对于简单的可视化需求，使用 Kibana 的普通图表更高效。
:::

## 保存和使用

在 Timelion 编辑器中编写完表达式后，点击保存按钮保存。你可以将 Timelion 可视化添加到 Dashboard 中，与其他图表组合使用。

Timelion 是一门值得深入学习的技能。掌握了它，你就能用简洁的表达式揭示数据中的深层规律。
