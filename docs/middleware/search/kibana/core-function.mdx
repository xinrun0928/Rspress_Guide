# Kibana 核心功能

Kibana 有三个核心入口：Discover、Visualize、Dashboard。它们分别对应数据分析的三个阶段：探索、可视化、展示。理解这三个功能，就掌握了 Kibana 的精髓。

## Discover 数据探索

Discover 是你与数据对话的起点。当你需要排查问题、分析日志时，第一步通常是来这里。

### 时间范围选择

右上角的时间选择器决定了你要分析的数据范围。有几种常用模式：

**快速选择**：Last 15 minutes、Last 1 hour、Last 24 hours……适合快速查看最近状态。

**绝对时间**：指定具体的时间范围，如 2024-01-15 14:00:00 到 2024-01-15 15:00:00。适合排查已知时间的故障。

**相对时间**：Now-1h/h、Now-7d/d……以当前时间为基准向前推算。适合定期检查，比如每周一早上看上周数据。

:::tip 时间范围选择技巧
时间范围越宽，查询越慢。如果你能确定问题发生的大致时间，尽量缩小范围。
:::

### KQL 搜索框

搜索框使用 KQL（Kibana Query Language）语法，比 SQL 更直观：

```kql
status:500 AND service:"order-service" AND @timestamp:[now-1h TO now]
```

这段查询的意思是：查找最近 1 小时内，状态码为 500 且来自订单服务的日志。

### 字段过滤

左侧的字段列表显示了当前索引中的所有字段。点击字段旁边的加号或减号图标，可以快速添加过滤条件：

- **加号** `+`：只显示包含该值的文档
- **减号** `-`：排除包含该值的文档

### 文档详情

点击某一条日志，可以展开查看详细信息。JSON 格式的原始数据清晰可见，你还可以点击字段旁边的复制按钮快速复制值。

## Visualize 可视化创建

Visualize 是创建可视化图表的地方。每一种图表类型都有它的适用场景。

### 创建流程

1. 选择可视化类型（折线图、柱状图、饼图……）
2. 选择数据源（索引模式）
3. 配置 Metrics（数值聚合）和 Buckets（分组维度）
4. 调整样式和选项
5. 保存并添加到 Dashboard

### 常用可视化类型

| 类型 | 适用场景 | 示例 |
|---|---|---|
| 折线图 | 展示趋势变化 | CPU 使用率随时间变化 |
| 柱状图 | 对比不同类别 | 各服务的请求量对比 |
| 饼图 | 展示占比分布 | HTTP 状态码分布 |
| 数据表 | 展示详细数值 | Top 10 错误日志 |
| 标签云 | 突出高频词 | 热门搜索关键词 |
| 指标卡 | 展示单个数值 | 当前在线人数 |

### 聚合配置

Metrics 定义了「怎么算」：

- **Count**：文档数量
- **Average**：平均值
- **Sum**：总和
- **Max/Min**：最大/最小值
- **Percentiles**：百分位数，如 p50、p99

Buckets 定义了「怎么分」：

- **Date Histogram**：按时间分桶
- **Terms**：按字段值分桶
- **Range**：按数值范围分桶
- **Histogram**：按固定区间分桶

:::warning 聚合分桶过多的问题
Terms 聚合默认只返回前 10 个桶。如果你的数据中某字段有上百个值，排名靠后的会被忽略。需要用「Other」选项查看被忽略的 bucket。
:::

## Dashboard 仪表盘

Dashboard 是 Kibana 的终极武器——将多个可视化组合在一起，形成一个完整的数据视图。

### 添加面板

在 Dashboard 中点击「Add panel」按钮，可以添加已保存的可视化，或者直接创建新的可视化。

面板可以调整大小和位置，拖拽操作非常流畅。你可以像搭积木一样自由布局。

### 交互功能

Dashboard 中的每个面板都可以独立交互：

- 点击柱状图的某根柱子，Dashboard 会自动添加过滤条件
- 使用时间选择器，所有面板会同步更新
- 支持全屏模式，方便大屏展示

### 敏感词过滤

在 Dashboard 设置中，你可以启用敏感词过滤功能，自动隐藏日志中的敏感信息（如密码、手机号、身份证号）。

## 最佳实践

### 按角色创建 Dashboard

不要试图创建一个「万能 Dashboard」。建议按角色创建：

- **运维 Dashboard**：关注系统状态、错误率、响应时间
- **业务 Dashboard**：关注用户行为、转化率、业务指标
- **安全 Dashboard**：关注异常访问、未授权操作

### Dashboard 命名规范

使用统一的命名规范，方便搜索和分类：

```text
[环境] - [服务名] - [用途]
例如：Production - Order Service - 运维监控
```

### 控制面板数量

一个 Dashboard 的面板数量建议控制在 10-15 个以内。过多的面板会让界面拥挤，反而降低可读性。如果数据太多，考虑拆分成多个 Dashboard。
