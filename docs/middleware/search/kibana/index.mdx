# Kibana

当你第一次打开 Kibana 的界面时，你可能会觉得这只是一个花哨的数据展示工具。但如果你深入使用，就会发现它远不止于此——它是你与 Elasticsearch 数据对话的窗口，是洞察数据背后故事的利器。

凌晨 2 点，线上系统出现异常波动。你需要快速定位问题：哪些指标异常？什么时间开始的？关联日志有哪些？Kibana 就是这个场景下的瑞士军刀。

## Kibana 是什么？

Kibana 是 Elasticsearch 的官方可视化平台。它让你可以通过图形化界面探索、可视化、分析存储在 Elasticsearch 中的数据。

如果说 Elasticsearch 是一个强大的搜索引擎，那么 Kibana 就是它的「眼睛」——让你能够「看见」数据中的规律和异常。

## 核心价值

**快速数据探索**：不需要写代码，你可以通过点选、拖拽的方式探索数据。Discover 功能让你像使用高级搜索一样过滤和分析日志。

**直观可视化**：将枯燥的数据变成折线图、柱状图、热力图、地图……不同的可视化类型适合不同的分析场景。

**实时监控告警**：设置阈值告警，当指标超过预期时自动通知相关人员。

**团队协作**：通过 Spaces 功能隔离不同项目的数据，通过角色权限控制谁能看什么数据。

## 为什么需要 Kibana？

有人说：Elasticsearch 不是有 API 吗？我用 curl 也能查询数据。

这话没错，但想象一下这些场景：

你要分析过去 7 天的日志趋势，用 API 需要写聚合查询、处理响应、绘图；
在 Kibana 里，几秒钟就能创建一个可视化仪表盘。

你要排查一个跨多个服务的 bug，需要在海量日志中搜索关键词；
在 Kibana 里，KQL 查询语言让你可以像 Google 搜索一样简单。

你要向领导汇报系统运行状况；
在 Kibana 里，一个精心设计的 Dashboard 胜过千言万语。

## 本章内容结构

接下来的章节中，我们会全面掌握 Kibana：

- **核心功能**：Dashboard、Discover、Visualize 的使用技巧
- **KQL 查询**：比 SQL 更强大的查询语法
- **可视化类型**：选择正确的图表展示正确的数据
- **Timelion**：时序数据的趋势分析与预测
- **告警功能**：主动发现问题而不是等待用户反馈
- **安全功能**：多租户隔离与权限控制
- **分享功能**：将洞察传递给更多人

无论你是运维工程师、开发人员，还是数据分析师，Kibana 都能帮助你更好地理解你的数据。
