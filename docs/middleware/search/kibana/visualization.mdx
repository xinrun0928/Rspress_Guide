# 可视化类型详解

数据可视化是一门艺术，也是一门科学。同样一组数据，用不同的图表展示，效果可能天差地别。

折线图适合看趋势，饼图适合看占比，热力图适合看分布……每种图表都有它的「性格」，选择对了，数据会说话；选择错了，只会让读者困惑。

## 折线图（Line）

折线图是最常用的时序数据展示方式。它能够清晰地展示数据随时间变化的趋势。

### 适用场景

- 系统指标的时间趋势（CPU、内存、响应时间）
- 业务指标的变化（DAU、GMV、转化率）
- 性能对比（优化前后的性能曲线）

### 配置要点

```yaml
类型: Line
X 轴: @timestamp (Date Histogram, 1分钟/5分钟/1小时)
Y 轴: 平均响应时间 (Average)
分组: 按服务名分桶 (Terms)
```

### 实用技巧

- **平滑曲线 vs 锐利线条**：如果数据波动较大，关闭「Smooth curve」选项，让尖峰更明显
- **多Y轴**：当两条线的数值范围差异很大时，使用双Y轴避免大数值淹没小数值
- **区域填充**：开启「Fill」选项，可以在折线下方填充颜色，增强视觉效果

:::tip 折线图的最佳实践
不要在同一个图表中展示超过 5 条折线，否则会变成一团乱麻。如果数据源太多，考虑拆分成多个图表或使用堆叠面积图。
:::

## 柱状图（Bar/Histogram）

柱状图适合展示分类数据的对比，或者离散时间点的数值比较。

### 适用场景

- 不同服务的请求量对比
- 各 HTTP 状态码的分布
- Top N 用户/订单/商品

### 垂直柱 vs 水平柱

- **垂直柱**：时间序列、连续数值。X 轴通常是时间或数值。
- **水平柱**：分类数据。适合展示排名（如 Top 10 错误类型）。

### 分组模式

- **普通柱状图**：每个分类一个柱子
- **堆叠柱状图**：将多个指标堆叠在一起，适合展示部分与整体的关系
- **分组柱状图**：将多个指标并排展示，适合对比

```yaml
类型: Bar (Horizontal)
指标: 错误数量 (Count)
分桶: 错误类型 (Terms, 10 terms)
颜色: 按错误级别区分
```

## 饼图（Pie）

饼图展示的是部分与整体的关系。它告诉我们：各个部分占总量的百分比是多少。

### 适用场景

- HTTP 状态码分布
- 错误类型占比
- 服务调用量分布

### 注意事项

- **类别不宜过多**：超过 7 个类别时，饼图会变得难以阅读。剩余部分用「Other」汇总
- **从小到大排列**：将最大的扇区从 12 点钟位置开始，顺时针排列
- **避免 3D 效果**：3D 饼图会扭曲比例，让比较变得困难

```yaml
类型: Pie
指标: 请求数量 (Count)
分桶: 状态码 (Terms)
设置: 启用 "Include orthers" 选项
```

## 数据表（Data Table）

数据表是展示详细数值的最佳选择。当你需要看到具体数值，而不是图形趋势时，数据表是你的首选。

### 适用场景

- Top N 列表（Top 10 错误、最热门的商品）
- 详细指标对比
- 需要精确数值的场景

### 高级配置

```yaml
类型: Data Table
列配置:
  - 请求量 (Count, 降序)
  - 平均响应时间 (Average, 降序)
  - 错误率 (Scripted Metric)
分桶: 按服务名分组
```

### 计算列

如果内置聚合不够用，可以使用 Painless 脚本创建计算列：

```painless
doc['response_time'].value / doc['count'].value
```

:::warning 性能考虑
数据表会显示所有匹配的行，数据量大时可能导致 Kibana 卡顿。建议添加适当的过滤条件或限制返回行数。
:::

## 指标卡（Metric）

指标卡只显示一个数字。大道至简，有时候一个数字比一千张图表更有说服力。

### 适用场景

- 当前在线人数
- 今日收入
- 当前错误率
- 系统健康状态

### 配置技巧

- **大型数字格式化**：使用 `1,234,567` 而不是 `1234567`
- **颜色阈值**：当数值超过阈值时自动变色，如错误率 `` `>` `` 5% 时显示红色
- **趋势指示**：显示与上一时段的对比，如 `↑ 12%` 或 `↓ 3%`

```yaml
类型: Metric
指标: 今日活跃用户 (Unique Count, user_id)
格式化: #,##0
颜色: 
  - > 10000: 绿色
  - > 5000: 黄色
  - else: 红色
```

## 热力图（Heatmap）

热力图用颜色的深浅表示数值的大小，非常适合展示高密度数据。

### 适用场景

- 一年中每天每小时的请求量分布
- 不同维度的交叉分析
- 找出数据中的「热点」

### 配置要点

```yaml
类型: Heatmap
X 轴: 小时 (0-23)
Y 轴: 星期几 (Mon-Sun)
值: 请求数量 (Count)
颜色: 从浅绿到深红
```

### 颜色选择

热力图的颜色选择很重要：

- **顺序色**：适合单色渐变，从浅到深，如浅绿到深绿
- **发散色**：适合有中心值的情况，如低于预期（蓝）到高于预期（红）
- **避免彩虹色**：彩虹渐变不符合人类视觉感知，容易误导

## 地图（Maps）

当你的数据包含地理位置信息时，地图是最直观的选择。

### 适用场景

- 用户地域分布
- 服务节点分布
- 攻击来源地理分布

### 数据要求

地图需要 GeoIP 数据或显式的经纬度坐标：

```json
{
  "location": {
    "lat": 39.9042,
    "lon": 116.4074
  }
}
```

### 地图类型

- **坐标图（Coordinate Map）**：每个点是一个文档，点的大小表示数值
- **地区图（Choropleth Map）**：按国家/省份着色，颜色深浅表示数值
- **轨道图（Tile Map）**：适合展示轨迹数据

## 时序图表进阶

### 堆叠面积图

当你想同时展示总量和构成时，堆叠面积图是好选择：

```yaml
类型: Area (Stacked)
X 轴: 时间
Y 轴: 请求量
分组: 按状态码分桶
颜色: 
  - 2xx: 绿色
  - 3xx: 黄色
  - 4xx: 橙色
  - 5xx: 红色
```

### 比率折线图

如果想看某个指标占总体的比率：

```yaml
类型: Line
指标: 
  - 200 响应数 (Count, filter: status:200)
  - 总请求数 (Count)
公式: count_200 / count_total * 100
```

## 选择正确的图表

面对同样的数据，为什么不同的人会画出完全不同的图表？答案往往不是技术问题，而是对数据理解的问题。

**先问自己三个问题：**

1. 我想展示什么？（趋势、对比、分布、关系）
2. 我的数据是什么类型？（时间序列、分类、连续数值）
3. 我的受众是谁？（技术人员需要细节，管理层需要概览）

| 你想展示 | 推荐图表 |
|---|---|
| 随时间变化 | 折线图、面积图 |
| 部分占整体 | 饼图、堆叠柱状图 |
| 分类对比 | 柱状图 |
| 精确数值 | 数据表、指标卡 |
| 地理分布 | 地图 |
| 大量数据分布 | 热力图 |
| 相关性关系 | 散点图 |

记住：最好的图表是不需要解释就能让人理解的图表。
