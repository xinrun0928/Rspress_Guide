# 搜索引擎

凌晨 3 点，你的电商网站搜索功能突然变得极其缓慢，用户怨声载道。运营团队急得像热锅上的蚂蚁，你的老板在电话那头问你：「到底是什么问题？」

如果你不知道答案，那么 Elasticsearch 可能是你需要掌握的武器。

## 搜索引挚在现代系统中的定位

在海量数据的时代，如何让用户快速找到他们想要的信息？这是一个看似简单、实则极其复杂的问题。

想象一下：一个拥有 1000 万件商品的电商平台，用户输入「黑色运动鞋」，系统需要在毫秒级返回相关商品列表。如果用传统的关系型数据库做 `LIKE` 查询，这个操作可能需要几十秒——用户早就跑了。

搜索引挚正是为解决这类问题而生的。与关系型数据库的事务性存储不同，搜索引挚专门优化了**全文检索**、**模糊匹配**、**相关性排序**等场景。

### 搜索引挚 vs 关系型数据库

| 维度 | 关系型数据库 | 搜索引挚 |
|---|---|---|
| 数据模型 | 行、列、表 | 文档、字段、索引 |
| 查询能力 | 精确匹配为主 | 全文检索、模糊匹配 |
| 性能特点 | 数据量越大越慢 | 倒排索引保证高速检索 |
| 事务支持 | ACID 完整支持 | 无事务，弱一致性 |
| 适用场景 | 核心业务数据 | 日志分析、全文搜索 |

## ELK Stack 生态概览

说到搜索引挚，就不得不提 ELK Stack。ELK 是 Elasticsearch、Logstash、Kibana 三个开源软件的组合，它们共同构成了一个强大的**日志收集、存储、分析**平台。

### 组件职责

**Elasticsearch** 是整个栈的核心，负责数据的存储和检索。它基于 Apache Lucene 构建，使用倒排索引实现高速全文搜索。

**Logstash** 是数据收集和处理的管道。它从各种数据源（文件、数据库、消息队列）采集数据，进行过滤、转换，然后发送到 Elasticsearch。

**Kibana** 是可视化界面，让用户通过图形化方式探索数据、构建仪表盘、监控告警。

**Beats** 是一系列轻量级数据采集器，包括 Filebeat（日志文件）、Metricbeat（系统指标）、Packetbeat（网络流量）等。
