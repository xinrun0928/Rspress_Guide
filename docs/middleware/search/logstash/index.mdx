# Logstash

Logstash 是 ELK Stack 中的数据处理引擎，负责从各种数据源采集数据、转换数据格式、然后将处理后的数据发送到目的地。

想象一下：你的系统有日志文件、有数据库数据、有 Kafka 消息、有 HTTP API 推送的数据——这些数据格式各异，存储位置不同。Logstash 就是把这些杂乱的数据「清洗」「标准化」，最终变成 Elasticsearch 可以处理的统一格式。

## 为什么需要 Logstash？

有人说：Beats 也可以采集数据，为什么还要 Logstash？

Beats 是轻量级的采集器，适合简单场景。Logstash 的优势在于：

**强大的数据转换能力**：Grok 正则解析、JSON 解析、日期转换、字段操作……Logstash 有丰富的 Filter 插件，可以把混乱的日志变成结构化的数据。

**复杂的数据处理逻辑**：分支处理、条件路由、多数据源合并——Logstash 的 Pipeline 模型支持复杂的数据处理流程。

**缓冲与背压处理**：当 Elasticsearch 或下游服务不可用时，Logstash 可以缓冲数据，防止数据丢失。

## 核心概念

Logstash 有几个核心概念：

**Pipeline**：数据处理的完整流水线，由 Input、Filter、Output 三部分组成。

**Event**：Logstash 处理的数据单元，类似于 JSON 对象，包含字段和值。

**Plugin**：Input、Filter、Output 的扩展点。Logstash 提供了数百种插件，满足各种场景需求。
