# Logstash Input 插件

Input 是 Logstash Pipeline 的数据入口，负责从各种数据源读取数据。

Logstash 支持数十种 Input 插件，覆盖文件、数据库、消息队列、网络协议等各种数据源。这篇文章讲解常用 Input 插件的配置和使用。

## file 输入

### 基本配置

```ruby
input {
  file {
    path => "/var/log/nginx/access.log"
    start_position => "beginning"
    sincedb_path => "/var/lib/logstash/sincedb/access"
    codec => "json"
  }
}
```

| 参数 | 说明 |
|---|---|
| `path` | 文件路径，支持通配符 |
| `start_position` | 从头开始还是从尾部开始 |
| `sincedb_path` | 记录读取位置的文件 |
| `codec` | 数据编解码器 |

### 路径通配

```ruby
file {
  path => "/var/log/**/*.log"        # 递归匹配
  path => "/var/log/{access,error}.log"  # 花括号匹配
  exclude => "*.gz"                 # 排除特定文件
}
```

### 读取位置管理

`sincedb_path` 记录了每个文件当前的读取位置：

```bash
# sincedb 文件内容
inode  major  minor  inode  position  file
65482  8     1      131073 12345     /var/log/access.log
```

如果文件被删除后重建，inode 会变化，Logstash 会当作新文件从头读取。

### 常用参数

```ruby
file {
  path => "/var/log/**/*.log"
  type => "syslog"
  discover_interval => 10      # 扫描新文件间隔（秒）
  stat_interval => 1           # 检查文件变化间隔（秒）
  close_older => 3600          # 关闭多久没变化的文件（秒）
  max_open_files => 1024       # 最大打开文件数
}
```

## beats 输入

接收来自 Beats 系列产品的数据：

```ruby
input {
  beats {
    port => 5044
    host => "0.0.0.0"
    ssl => true
    ssl_certificate => "/etc/logstash/ssl/server.crt"
    ssl_key => "/etc/logstash/ssl/server.key"
  }
}
```

Filebeat、Metricbeat、Heartbeat 等 Beats 产品通常通过这个 Input 接收数据。

## jdbc 输入

从数据库读取数据：

```ruby
input {
  jdbc {
    jdbc_driver_library => "/opt/mysql-connector-java.jar"
    jdbc_driver_class => "com.mysql.jdbc.Driver"
    jdbc_connection_string => "jdbc:mysql://localhost:3306/mydb"
    jdbc_user => "logstash"
    jdbc_password => "password"

    statement => "SELECT * FROM orders WHERE updated_at > :sql_last_value"
    use_column_value => true
    tracking_column => "updated_at"
    tracking_column_type => "timestamp"

    schedule => "*/5 * * * *"  # 每5分钟执行一次
    last_run_metadata_path => "/var/lib/logstash/jdbc_last_run"
  }
}
```

### 增量同步策略

```ruby
jdbc {
  # 使用时间戳字段增量同步
  statement => "SELECT * FROM products WHERE updated_at > :sql_last_value"
  use_column_value => true
  tracking_column => "updated_at"

  # 使用自增ID增量同步
  statement => "SELECT * FROM products WHERE id > :sql_last_value"
  use_column_value => true
  tracking_column => "id"
  tracking_column_type => "numeric"
}
```

### 完整示例

```ruby
input {
  jdbc {
    jdbc_driver_library => "/opt/mysql-connector-java.jar"
    jdbc_driver_class => "com.mysql.jdbc.Driver"
    jdbc_connection_string => "jdbc:mysql://mysql:3306/shop"
    jdbc_user => "logstash"
    jdbc_password => "${JDBC_PASSWORD}"

    schedule => "*/10 * * * *"
    statement_filepath => "/etc/logstash/sql/orders.sql"

    jdbc_default_timezone => "Asia/Shanghai"
    jdbc_paging_enabled => true
    jdbc_page_size => 100000
  }
}
```

## kafka 输入

```ruby
input {
  kafka {
    bootstrap_servers => "kafka1:9092,kafka2:9092"
    topics_pattern => "app-logs-.*"
    group_id => "logstash-consumer"
    consumer_threads => 4

    codec => json
    decorate_events => true

    auto_offset_reset => "earliest"
    enable_auto_commit => true
  }
}
```

### 参数说明

| 参数 | 说明 |
|---|---|
| `bootstrap_servers` | Kafka 服务器地址 |
| `topics_pattern` | 主题匹配模式 |
| `group_id` | 消费者组 ID |
| `consumer_threads` | 消费线程数 |

### 高级配置

```ruby
input {
  kafka {
    bootstrap_servers => "kafka:9092"
    topics => ["app-logs"]
    group_id => "logstash"
    consumer_threads => 8

    # 安全配置
    security_protocol => "SASL_SSL"
    sasl_mechanism => "PLAIN"
    ssl_ca_locations => "/etc/logstash/ssl/ca.pem"

    # 偏移量管理
    auto_offset_reset => "earliest"
    offset_commit_interval => 5000

    # 过滤
    filter {
      # 排除特定消息
    }
  }
}
```

## http 输入

接收 HTTP POST 请求：

```ruby
input {
  http {
    port => 8080
    type => "api"
    codec => json
  }
}
```

### 常用配置

```ruby
http {
  port => 8080
  host => "0.0.0.0"

  # 认证
  user => "logstash"
  password => "${HTTP_PASSWORD}"

  # SSL
  ssl => true
  ssl_certificate => "/etc/logstash/ssl/server.crt"
  ssl_key => "/etc/logstash/ssl/server.key"

  # 请求方法
  request_methods => ["POST", "PUT"]
}
```

### 发送测试

```bash
curl -X POST -H "Content-Type: application/json" \
  -d '{"message": "test", "level": "info"}' \
  http://localhost:8080
```

## tcp 输入

接收 TCP 流数据：

```ruby
input {
  tcp {
    port => 5000
    codec => json_lines
  }
}
```

`json_lines` codec 适合处理每行一个 JSON 的流数据。

## syslog 输入

接收 syslog 协议数据：

input {

## 常用 Input 插件对比

| 插件 | 适用场景 | 特点 |
|---|---|---|
| `file` | 日志文件采集 | 支持通配符、断点续传 |
| `beats` | Beats 产品数据 | 轻量、安全 |
| `jdbc` | 数据库同步 | 支持增量查询 |
| `kafka` | 消息队列 | 高吞吐、持久化 |
| `http` | HTTP API 数据 | 简单易用 |
| `tcp` | 自定义协议 | 灵活扩展 |

## 总结

Input 插件是 Logstash 的数据入口。选择合适的 Input 插件，是搭建数据管道的第一步。

文件输入适合日志收集，JDBC 输入适合数据库同步，Kafka 输入适合高吞吐场景，HTTP 输入适合快速集成。

记住：Input 只负责采集数据，Filter 负责转换，Output 负责发送。合理分工，才能构建清晰的数据处理流程。
