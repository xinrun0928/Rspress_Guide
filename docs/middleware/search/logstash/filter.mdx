# Logstash Filter 插件

Filter 是 Logstash Pipeline 中最强大的部分，负责数据转换和处理。Input 读取原始数据，Filter 将其转换为结构化的格式，Output 发送到目标系统。

没有 Filter，你的 Pipeline 只是数据的搬运工。掌握 Filter，才能真正发挥 Logstash 的威力。

## Grok 插件

Grok 是 Logstash 最常用的 Filter，通过正则模式从非结构化文本中提取结构化字段。

### 基本语法

```ruby
filter {
  grok {
    match => { "message" => "%{IP:client_ip} %{WORD:method} %{URIPATHPARAM:request} %{NUMBER:status:int}" }
  }
}
```

Grok 模式语法：`%{PATTERN:FIELD_NAME}`

输入：
```
192.168.1.1 GET /api/users/123 200
```

输出：
```json
{
  "client_ip": "192.168.1.1",
  "method": "GET",
  "request": "/api/users/123",
  "status": 200
}
```

### 常用模式

| 模式 | 说明 | 示例 |
|---|---|---|
| `USERNAME` | 用户名 | `john` |
| `IP` | IP 地址 | `192.168.1.1` |
| `NUMBER` | 数字 | `200` |
| `WORD` | 单词 | `GET` |
| `PATH` | 路径 | `/var/log/app.log` |
| `URIPATHPARAM` | URI 路径和参数 | `/api/users?id=123` |
| `TIMESTAMP_ISO8601` | ISO8601 时间 | `2024-01-15T10:30:00Z` |
| `LOGLEVEL` | 日志级别 | `ERROR` |
| `GREEDYDATA` | 贪婪匹配剩余所有 | `剩余内容` |

### 自定义正则

```ruby
filter {
  grok {
    match => {
      "message" => "(?<timestamp>\d{4}-\d{2}-\d{2}) (?<level>\w+) (?<msg>.*)"
    }
  }
}
```

`(?<name>pattern)` 定义命名捕获组。

### 多个模式

```ruby
filter {
  grok {
    match => {
      "message" => [
        "%{IP:client} - %{WORD:user} \[%{HTTPDATE:timestamp}\] \"%{WORD:method} %{NOTSPACE:request}\"",
        "%{SYSLOGBASE}"
      ]
    }
    break_on_match => false
  }
}
```

按顺序尝试多个模式，匹配成功即停止（除非 `break_on_match => false`）。

### 常见日志格式

**Nginx 访问日志**：
```ruby
grok {
  match => {
    "message" => '%{IPORHOST:client_ip} - %{DATA:user} \[%{HTTPDATE:timestamp}\] "%{WORD:method} %{URIPATHPARAM:request} HTTP/%{NUMBER:http_version}" %{NUMBER:status:int} %{NUMBER:bytes:int} "%{DATA:referrer}" "%{DATA:user_agent}"'
  }
}
```

**Apache 访问日志**：
```ruby
grok {
  match => {
    "message" => '%{IPORHOST:client_ip} %{DATA:ident} %{DATA:auth} \[%{HTTPDATE:timestamp}\] "%{WORD:method} %{URIPATHPARAM:request} HTTP/%{NUMBER:http_version}" %{NUMBER:status:int} (?:%{NUMBER:bytes:int}|-)'
  }
}
```

## JSON 插件

解析 JSON 格式数据：

```ruby
filter {
  json {
    source => "message"
    target => "parsed"
  }
}
```

输入：`{"user": "john", "action": "login"}`

输出：
```json
{
  "message": "{\"user\": \"john\", \"action\": \"login\"}",
  "parsed": {
    "user": "john",
    "action": "login"
  }
}
```

### 展开到顶层

```ruby
filter {
  json {
    source => "message"
    target => ""
  }
}
```

设置 `target` 为空，字段会直接添加到 Event 顶层。

## Date 插件

统一时间戳格式：

```ruby
filter {
  date {
    match => ["timestamp", "ISO8601", "MMM dd yyyy HH:mm:ss", "yyyy/MM/dd HH:mm:ss"]
    target => "@timestamp"
  }
}
```

`match` 指定要匹配的时间格式，`target` 指定目标字段（默认覆盖 `@timestamp`）。

### 自定义格式

```ruby
date {
  match => ["timestamp", "dd/MMM/yyyy:HH:mm:ss Z"]
  target => "@timestamp"
  locale => "en"
}
```

## Mutate 插件

数据转换和修修改：

```ruby
filter {
  mutate {
    # 重命名字段
    rename => {
      "host" => "server_name"
      "msg" => "message"
    }

    # 删除字段
    remove_field => ["original_message", "unused_field"]

    # 替换字段值
    replace => { "message" => "New value for message" }

    # 类型转换
    convert => {
      "status" => "integer"
      "price" => "float"
      "is_active" => "boolean"
    }

    # 字符串操作
    gsub => [
      "path", "/var/log/", "",
      "message", "\s+", " "
    ]

    # 添加字段
    add_field => {
      "env" => "production"
      "app_name" => "myapp"
    }

    # 添加标签
    add_tag => ["processed"]
  }
}
```

### gsub 示例

```ruby
mutate {
  # 移除特殊字符
  gsub => [
    "message", "[\\x00-\\x1F]", ""
  ]

  # 替换协议名
  gsub => [
    "protocol", "HTTP/1.1", "HTTP/2"
  ]
}
```

## GeoIP 插件

根据 IP 地址添加地理位置信息：

```ruby
filter {
  geoip {
    source => "client_ip"
    target => "geoip"
    database => "/etc/logstash/GeoLite2-City.mmdb"
  }
}
```

输出：
```json
{
  "geoip": {
    "city_name": "Beijing",
    "country_code2": "CN",
    "country_name": "China",
    "location": {
      "lon": 116.4074,
      "lat": 39.9042
    }
  }
}
```

## UserAgent 插件

解析 User-Agent 字符串：

```ruby
filter {
  useragent {
    source => "user_agent"
    target => "ua"
  }
}
```

输出：
```json
{
  "ua": {
    "name": "Chrome",
    "os": "Windows 10",
    "os_name": "Windows",
    "device": "Other",
    "major": "120",
    "minor": "0"
  }
}
```

## Fingerprint 插件

生成指纹，用于去重或标识：

```ruby
filter {
  fingerprint {
    source => ["client_ip", "message"]
    target => "fingerprint"
    method => "SHA256"
    key => "secret_key"
  }
}
```

## 插件组合示例

### 完整的 Web 日志处理

```ruby
input {
  file {
    path => "/var/log/nginx/access.log"
    codec => json
  }
}

filter {
  # 解析日志格式
  grok {
    match => {
      "message" => '%{IPORHOST:client_ip} - %{DATA:user} \[%{HTTPDATE:timestamp}\] "%{WORD:method} %{URIPATHPARAM:request} HTTP/%{NUMBER:http_version}" %{NUMBER:status:int} %{NUMBER:bytes:int} "%{DATA:referrer}" "%{DATA:user_agent}"'
    }
  }

  # 转换时间
  date {
    match => ["timestamp", "dd/MMM/yyyy:HH:mm:ss Z"]
    target => "@timestamp"
  }

  # 添加地理位置
  geoip {
    source => "client_ip"
    target => "geo"
    database => "/etc/logstash/GeoLite2-City.mmdb"
  }

  # 解析 User-Agent
  useragent {
    source => "user_agent"
    target => "ua"
  }

  # 转换类型
  mutate {
    convert => {
      "bytes" => "integer"
      "status" => "integer"
    }
  }

  # 提取 URL 参数
  kv {
    field_split => "&?"
    target => "params"
    include_keys => ["id", "action", "page"]
  }
}

output {
  elasticsearch {
    hosts => ["http://localhost:9200"]
    index => "nginx-access-%{+YYYY.MM.dd}"
  }
}
```

## 总结

Filter 插件是 Logstash 的核心能力。

Grok 用于解析非结构化文本，是日志处理的瑞士军刀。JSON 插件处理结构化数据。Date 插件统一时间格式。Mutate 插件做各种数据转换。GeoIP 和 UserAgent 丰富数据维度。

实际使用中，通常是多个 Filter 插件组合使用。合理的 Filter 链，可以让混乱的原始数据变成干净的结构化数据。
