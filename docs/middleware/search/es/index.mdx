# Elasticsearch

Elasticsearch 是一个基于 Lucene 构建的分布式、RESTful 搜索和分析引擎。它能够解决海量数据下的全文检索、结构化查询、聚合分析等问题。

提到 Elasticsearch，就不得不提它的诞生背景。2010 年，Shay Banon 在构建一个房产搜索引擎时，觉得 Lucene 的 API 太底层了，于是开发了 Compass。后来他将 Compass 重写，诞生了 Elasticsearch。2015 年，Elasticsearch 与 Logstash、Kibana 合并，形成了今天我们熟知的 ELK Stack。

从诞生之初，Elasticsearch 就被设计为**水平扩展**的。你可以从单节点开始，随着数据量增长，逐步扩展到数十甚至数百个节点。它提供了近实时的搜索能力，数据写入后通常在 1 秒内就能被检索到。

## 核心能力

**全文检索**是 Elasticsearch 的看家本领。它支持模糊匹配、同义词查询、相关性评分，能够根据用户输入智能返回最相关的结果。

**分布式架构**让它可以处理 PB 级别的数据。通过分片机制，数据被分散存储在多个节点上，查询时可以并行执行，充分利用集群的计算能力。

**聚合分析**能力同样强大。它不仅能告诉你「有哪些结果」，还能告诉你「结果的分布情况」「平均值、最大值」「时间趋势」等信息。

**近实时性**是它的另一个特点。相比传统数据库需要等事务完成才能查询，Elasticsearch 默认每秒刷新一次索引，数据写入后几乎立即可被搜索到。

## 应用场景

日志分析与监控系统是 Elasticsearch 最经典的应用场景。配合 Logstash 收集日志、Kibana 可视化，你可以搭建一个强大的日志分析平台。

站内搜索是另一个常见场景。电商平台的商品搜索、文档管理系统的内容检索，都可以用 Elasticsearch 实现。

安全分析领域也在广泛使用 Elasticsearch。收集网络流量日志、用户行为日志，通过聚合分析发现异常行为。

价格监控、竞品追踪、用户行为分析……只要涉及到大量数据的检索和分析，Elasticsearch 都能派上用场。
