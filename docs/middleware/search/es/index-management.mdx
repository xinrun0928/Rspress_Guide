# ES 索引管理

索引是 Elasticsearch 中数据组织的基本单位。合理地管理索引——包括 Mapping 定义、Settings 配置、Alias 使用——是保证系统稳定性和性能的关键。

这篇文章讲解索引管理的核心内容。

## Mapping

### 什么是 Mapping

Mapping 定义了索引中文档的结构：有哪些字段、每个字段是什么类型、如何被索引和存储。

```json
PUT /my-index
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "analyzer": "standard"
      },
      "author": {
        "type": "keyword"
      },
      "published_at": {
        "type": "date"
      },
      "views": {
        "type": "integer"
      },
      "content": {
        "type": "text",
        "analyzer": "ik_max_word"
      }
    }
  }
}
```

### 动态映射

默认情况下，Elasticsearch 会根据写入的数据自动推断字段类型，这就是**动态映射**：

```json
POST /my-index/_doc
{
  "title": "Hello",
  "views": 100,
  "published_at": "2024-01-15",
  "price": 99.9
}
```

Elasticsearch 会自动推断：`title` → `text`，`views` → `long`，`published_at` → `date`，`price` → `float`。

动态映射虽然方便，但可能导致类型推断不准确。建议在生产环境中使用**显式映射**。

### 字段类型

| 类型 | 说明 | 示例 |
|---|---|---|
| `text` | 全文检索，会分词 | 文章内容 |
| `keyword` | 精确值，不分词 | 标签、分类 |
| `integer/long` | 整数类型 | 点赞数 |
| `float/double` | 浮点类型 | 价格 |
| `boolean` | 布尔类型 | 是否发布 |
| `date` | 日期类型 | 发布时间 |
| `keyword` + `text` | 联合类型 | 既可搜索又可精确匹配 |

```json
{
  "properties": {
    "email": {
      "type": "text",
      "fields": {
        "keyword": {
          "type": "keyword"
        }
      }
    }
  }
}
```

`email` 字段可以用 `match` 进行全文搜索，也可以用 `term` 按精确值过滤。

### 映射参数

常用映射参数：

- `analyzer`：指定分词器
- `index`：是否被索引（`false` 时只能存储，不能搜索）
- `doc_values`：是否启用列式存储（禁用可节省空间，但无法排序/聚合）
- `norms`：是否存储评分因子（禁用可节省内存，但无法用于相关性排序）
- `format`：日期格式

```json
{
  "properties": {
    "user_id": {
      "type": "keyword",
      "doc_values": false,
      "index": true
    },
    "description": {
      "type": "text",
      "norms": false
    }
  }
}
```

## Settings

### 索引级别配置

```json
PUT /my-index
{
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 1,
    "refresh_interval": "1s"
  }
}
```

### 分片与副本

```json
{
  "settings": {
    "number_of_shards": 5,
    "number_of_replicas": 2
  }
}
```

副本数可以动态调整：

```json
PUT /my-index/_settings
{
  "number_of_replicas": 1
}
```

### 刷新间隔

```json
{
  "settings": {
    "refresh_interval": "5s"
  }
}
```

默认刷新间隔是 1 秒。如果写入量大，可以适当增大；如果对实时性要求极高，可以设为 `-1` 禁用自动刷新，改为手动控制。

```json
POST /my-index/_refresh
```

###  analyzers

可以在 Settings 中定义自定义分词器：

```json
{
  "settings": {
    "analysis": {
      "analyzer": {
        "my_analyzer": {
          "type": "custom",
          "tokenizer": "standard",
          "filter": ["lowercase", "asciifolding"]
        }
      },
      "filter": {
        "my_synonym_filter": {
          "type": "synonym",
          "synonyms": [
            "computer,pc,laptop",
            "mobile,cellphone,phone"
          ]
        }
      }
    }
  }
}
```

## Alias

### 什么是 Alias

Alias（别名）是索引的虚拟名称，可以指向一个或多个索引。

```json
POST /_aliases
{
  "actions": [
    {
      "add": {
        "index": "my-index-2024.01",
        "alias": "my-index-latest"
      }
    }
  ]
}
```

### 别名用途

**读写分离**：

```json
POST /_aliases
{
  "actions": [
    {
      "add": {
        "index": "my-index-2024.01",
        "alias": "my-index-write",
        "is_write_index": true
      }
    },
    {
      "add": {
        "index": "my-index-2024.01",
        "alias": "my-index-read"
      }
    }
  ]
}
```

写入操作发送到 `my-index-write`，读取操作发送到 `my-index-read`。

**蓝绿部署**：

```json
// 蓝：旧版本索引
POST /_aliases
{
  "actions": [
    {
      "remove": {
        "index": "my-index-blue",
        "alias": "my-index"
      }
    },
    {
      "add": {
        "index": "my-index-green",
        "alias": "my-index"
      }
    }
  ]
}
```

切换时无需修改代码，改动别名即可。

**按时间滚动**：

```json
POST /_aliases
{
  "actions": [
    {
      "remove": {
        "index": "logs-2024.01",
        "alias": "logs-current"
      }
    },
    {
      "add": {
        "index": "logs-2024.02",
        "alias": "logs-current"
      }
    }
  ]
}
```

### 查看别名

```bash
GET /_cat/aliases?v

alias          index          filter  routing.index  routing.search
my-index       my-index-1     -       -               -
logs-current   logs-2024.02   -       -               -
```

## Index Template

### 什么是 Index Template

Index Template 定义了创建新索引时的默认配置和 Mapping。当索引名称匹配模板模式时，模板会自动生效。

```json
PUT /_index_template/logs-template
{
  "index_patterns": ["logs-*"],
  "template": {
    "settings": {
      "number_of_shards": 3,
      "number_of_replicas": 1,
      "refresh_interval": "5s"
    },
    "mappings": {
      "properties": {
        "timestamp": { "type": "date" },
        "level": { "type": "keyword" },
        "message": { "type": "text" },
        "host": { "type": "ip" }
      }
    }
  }
}
```

### 模板优先级

多个模板可能匹配同一索引名称。`priority` 决定应用顺序：

```json
{
  "index_patterns": ["logs-*"],
  "priority": 100,
  "template": { ... }
}
```

优先级高的模板会覆盖优先级低的。

### Component Template

7.11+ 版本引入了 Component Template，支持更灵活的模板组合：

```json
PUT /_component_template/logs-mappings
{
  "template": {
    "mappings": {
      "properties": {
        "timestamp": { "type": "date" },
        "level": { "type": "keyword" }
      }
    }
  }
}

PUT /_index_template/logs-template
{
  "index_patterns": ["logs-*"],
  "composed_of": ["logs-mappings"],
  "template": {
    "settings": {
      "number_of_shards": 3
    }
  }
}
```

## 索引生命周期管理（ILM）

### 什么是 ILM

ILM（Index Lifecycle Management）自动管理索引在不同阶段的行为：

- **Hot**：数据写入频繁期
- **Warm**：数据不再更新，但仍可能被查询
- **Cold**：数据很少被访问
- **Delete**：数据过期，删除释放空间

### 创建 ILM 策略

```json
PUT /_ilm/policy/logs-policy
{
  "policy": {
    "phases": {
      "hot": {
        "actions": {
          "rollover": {
            "max_age": "7d",
            "max_primary_shard_size": "50gb"
          },
          "set_priority": {
            "priority": 100
          }
        }
      },
      "warm": {
        "min_age": "30d",
        "actions": {
          "shrink": {
            "number_of_shards": 1
          },
          "forcemerge": {
            "max_num_segments": 1
          },
          "set_priority": {
            "priority": 50
          }
        }
      },
      "cold": {
        "min_age": "90d",
        "actions": {
          "set_priority": {
            "priority": 0
          }
        }
      },
      "delete": {
        "min_age": "365d",
        "actions": {
          "delete": {}
        }
      }
    }
  }
}
```

### 应用 ILM 到索引

```json
PUT /logs/_settings
{
  "index.lifecycle.name": "logs-policy"
}
```

## 总结

索引管理涵盖了 Mapping、Settings、Alias、Template、ILM 多个方面。

Mapping 定义了数据结构，要在准确性和灵活性之间找到平衡；Settings 控制索引行为，要根据实际负载调整；Alias 提供了索引抽象，是蓝绿部署和滚动更新的基础；ILM 让索引管理自动化，省去大量人工操作。

好的索引管理策略，能让系统运维变得轻松很多。
