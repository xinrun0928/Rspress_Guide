# ES 聚合分析

Elasticsearch 不仅能做全文检索，还能对数据进行统计分析。聚合分析让你能够从数据中挖掘 insights：总和、平均值、分布情况、时间趋势……

这篇文章系统讲解 Elasticsearch 的聚合功能，包括 Bucket 聚合、Metric 聚合和 Pipeline 聚合。

## 聚合的分类

Elasticsearch 的聚合分为三类：

**Bucket Aggregation（桶聚合）**：将文档划分到不同的桶中，每个桶代表一个条件或分组。类似于 SQL 的 `GROUP BY`。

**Metric Aggregation（指标聚合）**：计算单个指标值，如 `sum`、`avg`、`max`、`min`。可以作用于桶内或全局。

**Pipeline Aggregation（管道聚合）**：将其他聚合的输出作为输入，进行二次计算。类似于 SQL 的嵌套 `SELECT`。

## Bucket 聚合

### terms 聚合

按字段值分桶，比如按商品分类统计：

```json
{
  "aggs": {
    "by_category": {
      "terms": {
        "field": "category.keyword",
        "size": 10
      }
    }
  }
}
```

`size` 参数指定返回的桶数量，默认为 10。

### range 聚合

自定义范围分桶，适合数值或日期字段：

```json
{
  "aggs": {
    "price_ranges": {
      "range": {
        "field": "price",
        "ranges": [
          { "to": 50 },
          { "from": 50, "to": 100 },
          { "from": 100, "to": 500 },
          { "from": 500 }
        ]
      }
    }
  }
}
```

### date_histogram 聚合

按时间周期分桶，常用于时序数据分析：

```json
{
  "aggs": {
    "sales_over_time": {
      "date_histogram": {
        "field": "order_date",
        "calendar_interval": "month",
        "format": "yyyy-MM"
      }
    }
  }
}
```

支持的 `calendar_interval`：`minute`、`hour`、`day`、`week`、`month`、`quarter`、`year`

### filters 聚合

按多个过滤条件分桶：

```json
{
  "aggs": {
    "status_counts": {
      "filters": {
        "filters": {
          "errors": { "range": { "status_code": { "gte": 400, "lt": 500 } } },
          "success": { "range": { "status_code": { "lt": 400 } } },
          "server_errors": { "range": { "status_code": { "gte": 500 } } }
        }
      }
    }
  }
}
```

### 嵌套聚合

桶聚合可以嵌套，实现多级分组：

```json
{
  "aggs": {
    "by_category": {
      "terms": {
        "field": "category.keyword",
        "size": 10
      },
      "aggs": {
        "by_brand": {
          "terms": {
            "field": "brand.keyword",
            "size": 5
          },
          "aggs": {
            "avg_price": {
              "avg": {
                "field": "price"
              }
            }
          }
        }
      }
    }
  }
}
```

## Metric 聚合

### 基础统计

`stats` 聚合一次返回所有基础统计：

```json
{
  "aggs": {
    "price_stats": {
      "stats": {
        "field": "price"
      }
    }
  }
}
```

返回结果：

```json
{
  "price_stats": {
    "count": 100,
    "min": 10.0,
    "max": 999.99,
    "avg": 156.78,
    "sum": 15678.0
  }
}
```

### 单值聚合

```json
{
  "aggs": {
    "total_sales": { "sum": { "field": "sales" } },
    "avg_price": { "avg": { "field": "price" } },
    "max_price": { "max": { "field": "price" } },
    "min_price": { "min": { "field": "price" } },
    "unique_categories": { "cardinality": { "field": "category.keyword" } }
  }
}
```

`cardinality` 用于计算唯一值数量，类似于 SQL 的 `COUNT(DISTINCT)`。

### 多值聚合

`extended_stats` 提供更多统计指标：

```json
{
  "aggs": {
    "price_extended": {
      "extended_stats": {
        "field": "price"
      }
    }
  }
}
```

返回 `std_deviation`（标准差）、`sum_of_squares`（平方和）等。

## Pipeline 聚合

管道聚合将其他聚合的输出作为输入，实现复杂的分析逻辑。

### bucket_sort

对桶进行排序和截取：

```json
{
  "aggs": {
    "sales_by_month": {
      "date_histogram": {
        "field": "order_date",
        "calendar_interval": "month"
      },
      "aggs": {
        "total_sales": {
          "sum": { "field": "amount" }
        },
        "sales_sort": {
          "bucket_sort": {
            "sort": [
              { "total_sales": { "order": "desc" } }
            ],
            "size": 5
          }
        }
      }
    }
  }
}
```

### cumulative_sum

计算累计和：

```json
{
  "aggs": {
    "sales_by_month": {
      "date_histogram": {
        "field": "order_date",
        "calendar_interval": "month"
      },
      "aggs": {
        "monthly_sales": {
          "sum": { "field": "amount" }
        },
        "cumulative_sales": {
          "cumulative_sum": {
            "buckets_path": "monthly_sales"
          }
        }
      }
    }
  }
}
```

### moving_avg

移动平均值，用于平滑趋势：

```json
{
  "aggs": {
    "sales_by_month": {
      "date_histogram": {
        "field": "order_date",
        "calendar_interval": "month"
      },
      "aggs": {
        "monthly_sales": {
          "sum": { "field": "amount" }
        },
        "moving_avg_sales": {
          "moving_avg": {
            "buckets_path": "monthly_sales",
            "window": 3,
            "shift": 1
          }
        }
      }
    }
  }
}
```

`window` 指定窗口大小，`shift` 指定向后偏移量。

## 聚合与分析实战

### 销售数据分析

```json
{
  "size": 0,
  "aggs": {
    "by_category": {
      "terms": {
        "field": "category.keyword",
        "size": 20
      },
      "aggs": {
        "total_revenue": { "sum": { "field": "revenue" } },
        "avg_order_value": { "avg": { "field": "revenue" } },
        "order_count": { "value_count": { "field": "order_id" } },
        "by_region": {
          "terms": {
            "field": "region.keyword",
            "size": 5
          },
          "aggs": {
            "revenue": { "sum": { "field": "revenue" } }
          }
        }
      }
    }
  }
}
```

### 用户行为分析

```json
{
  "size": 0,
  "query": {
    "range": {
      "timestamp": {
        "gte": "now-7d/d",
        "lt": "now/d"
      }
    }
  },
  "aggs": {
    "hourly_distribution": {
      "date_histogram": {
        "field": "timestamp",
        "calendar_interval": "hour"
      }
    },
    "top_pages": {
      "terms": {
        "field": "page_url.keyword",
        "size": 10
      }
    },
    "user_stats": {
      "cardinality": {
        "field": "user_id.keyword"
      }
    }
  }
}
```

## 总结

聚合分析是 Elasticsearch 的强大能力之一。

Bucket 聚合帮你分组，Metric 聚合帮你计算，Pipeline 聚合帮你做更复杂的分析。三者组合使用，可以实现大部分数据分析需求。

实际使用中要注意：
- 聚合会遍历大量数据，大聚合可能很慢
- 使用 `size: 0` 只返回聚合结果，不返回搜索命中
- `shard_size` 参数可以控制分片级别的桶数量，提高精度但增加开销
