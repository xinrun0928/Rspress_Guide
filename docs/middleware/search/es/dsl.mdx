# ES 查询 DSL

Elasticsearch 的查询能力之所以强大，很大程度上归功于灵活而强大的 Query DSL（Domain Specific Language）。

Query DSL 让你可以精确表达各种查询意图：从简单的关键词匹配，到复杂的多条件布尔组合，再到嵌套文档的深层查询。

这篇文章系统讲解各种查询语法，通过实例帮你掌握 Elasticsearch 的查询技巧。

## 查询上下文

在深入具体查询之前，需要理解两种查询上下文：

**Query Context（查询上下文）**：计算相关性得分。`must`、`should`、`must_not` 等子句默认使用查询上下文。

**Filter Context（过滤上下文）**：不计算得分，只做二元判断（匹配/不匹配）。`filter` 子句使用过滤上下文，性能更好。

```json
{
  "query": {
    "bool": {
      "must": [
        { "match": { "title": "elasticsearch" } }
      ],
      "filter": [
        { "term": { "status": "published" } },
        { "range": { "published_at": { "gte": "2024-01-01" } } }
      ]
    }
  }
}
```

上面查询中：
- `must` 中的 `match` 会计算相关性得分
- `filter` 中的 `term` 和 `range` 不会计算得分，仅过滤

## 全文查询

### match 查询

最常用的全文查询语法：

```json
GET /my-index/_search
{
  "query": {
    "match": {
      "content": "elasticsearch guide"
    }
  }
}
```

默认使用 `operator: or`，只要文档包含「elasticsearch」或「guide」即匹配。如果需要同时包含两个词：

```json
{
  "match": {
    "content": {
      "query": "elasticsearch guide",
      "operator": "and"
    }
  }
}
```

### match_phrase 查询

短语查询，要求词条按顺序连续出现：

```json
{
  "match_phrase": {
    "content": "elasticsearch guide"
  }
}
```

文档「This is an elasticsearch guide」会匹配，但「elasticsearch is a guide」不会匹配。

可以使用 `slop` 参数允许词之间有一定间隔：

```json
{
  "match_phrase": {
    "content": {
      "query": "elasticsearch guide",
      "slop": 1
    }
  }
}
```

现在「elasticsearch is a guide」也能匹配，因为「elasticsearch」和「guide」之间隔了 1 个词。

### multi_match 查询

多个字段查询：

```json
{
  "multi_match": {
    "query": "elasticsearch",
    "fields": ["title^2", "content", "tags"]
  }
}
```

- `fields`：要查询的字段列表
- `^2`：字段权重，`title` 字段的得分翻倍
- 默认使用 `best_fields` 策略，取最高分字段的得分作为最终得分

其他策略包括：

- `most_fields`：各字段得分求和
- `cross_fields`：将所有字段当作一个字段处理
- `phrase`/`phrase_prefix`：基于短语匹配

## 精确查询

### term 查询

精确值查询，不分析（分词），用于 `keyword`、数值、日期、布尔类型字段：

```json
{
  "term": {
    "status": "published"
  }
}
```

等价于 SQL：`WHERE status = 'published'`

### terms 查询

匹配多个精确值：

```json
{
  "terms": {
    "category": ["tech", "news", "sports"]
  }
}
```

### range 查询

范围查询：

```json
{
  "range": {
    "published_at": {
      "gte": "2024-01-01",
      "lte": "2024-12-31"
    }
  }
}
```

支持的参数：`gt`（大于）、`gte`（大于等于）、`lt`（小于）、`lte`（小于等于）。

数值范围查询：

```json
{
  "range": {
    "price": {
      "gte": 100,
      "lte": 500
    }
  }
}
```

## 复合查询

### bool 查询

布尔查询是最强大的组合工具：

```json
{
  "bool": {
    "must": [
      { "match": { "title": "elasticsearch" } }
    ],
    "should": [
      { "match": { "content": "tutorial" } },
      { "match": { "content": "beginner" } }
    ],
    "must_not": [
      { "term": { "status": "draft" } }
    ],
    "filter": [
      { "term": { "category": "tech" } },
      { "range": { "published_at": { "gte": "2024-01-01" } } }
    ],
    "minimum_should_match": 1
  }
}
```

- `must`：必须匹配，计算得分
- `should`：应该匹配，满足则加分，不满足不加减分（除非设置 `minimum_should_match`）
- `must_not`：必须不匹配
- `filter`：过滤，不计算得分

### boosting 查询

需要保留某些匹配但降低其权重时，`boosting` 比 `bool` 更合适：

```json
{
  "boosting": {
    "positive": {
      "match": { "title": "elasticsearch" }
    },
    "negative": {
      "match": { "title": "obsolete" }
    },
    "negative_boost": 0.5
  }
}
```

包含「obsolete」的文档不会被排除，但得分会乘以 0.5，降低排名。

## 嵌套查询

### nested 查询

Elasticsearch 的文档可以包含嵌套对象（`nested` 类型）。嵌套对象需要使用专门的 `nested` 查询：

```json
{
  "nested": {
    "path": "comments",
    "query": {
      "bool": {
        "must": [
          { "match": { "comments.author": "John" } },
          { "match": { "comments.content": "great" } }
        ]
      }
    }
  }
}
```

`path` 指定嵌套对象的路径，`query` 在嵌套对象内部执行。

### has_child / has_parent

父子文档查询：

```json
{
  "has_child": {
    "type": "comment",
    "query": {
      "match": { "author": "John" }
    }
  }
}
```

查找所有有 John 评论的文章。

## 地理查询

### geo_distance 查询

按地理位置距离筛选：

```json
{
  "geo_distance": {
    "distance": "10km",
    "location": {
      "lat": 40.7128,
      "lon": -74.0060
    }
  }
}
```

### geo_bounding_box 查询

矩形区域查询：

```json
{
  "geo_bounding_box": {
    "location": {
      "top_left": { "lat": 41, "lon": -75 },
      "bottom_right": { "lat": 40, "lon": -73 }
    }
  }
}
```

## 特殊查询

### exists 查询

查询指定字段有值的文档：

```json
{
  "exists": {
    "field": "published_at"
  }
}
```

### missing 查询（已废弃）

查找指定字段没有值的文档，使用 `must_not` + `exists` 代替：

```json
{
  "bool": {
    "must_not": [
      { "exists": { "field": "published_at" } }
    ]
  }
}
```

### ids 查询

根据文档 ID 查询：

```json
{
  "ids": {
    "values": ["doc1", "doc2", "doc3"]
  }
}
```

## 常用组合示例

### 博客文章搜索

```json
{
  "query": {
    "bool": {
      "must": [
        {
          "multi_match": {
            "query": "elasticsearch performance",
            "fields": ["title^2", "content"],
            "type": "best_fields"
          }
        }
      ],
      "filter": [
        { "term": { "status": "published" } },
        { "range": { "published_at": { "gte": "now-1y" } } }
      ],
      "should": [
        { "term": { "featured": true } }
      ]
    }
  },
  "highlight": {
    "fields": {
      "title": {},
      "content": { "fragment_size": 150, "number_of_fragments": 3 }
    },
    "pre_tags": ["<em>"],
    "post_tags": ["</em>"]
  },
  "sort": [
    "_score",
    { "published_at": "desc" }
  ]
}
```

这个查询：
- 匹配「elasticsearch performance」
- `title` 字段命中权重更高
- 只查询已发布的文章
- 发布时间在过去一年内
- 优质文章加分
- 返回匹配片段高亮
- 按相关性得分和发布时间排序

### 电商商品搜索

```json
{
  "query": {
    "bool": {
      "must": [
        {
          "match": {
            "name": {
              "query": "wireless bluetooth headphones",
              "operator": "and"
            }
          }
        }
      ],
      "filter": [
        { "term": { "category_id": 123 } },
        { "range": { "price": { "gte": 50, "lte": 200 } } },
        { "term": { "in_stock": true } }
      ]
    }
  },
  "aggs": {
    "price_ranges": {
      "range": {
        "field": "price",
        "ranges": [
          { "to": 50 },
          { "from": 50, "to": 100 },
          { "from": 100, "to": 200 },
          { "from": 200 }
        ]
      }
    },
    "brands": {
      "terms": { "field": "brand.keyword", "size": 10 }
    }
  }
}
```

## 总结

Query DSL 是 Elasticsearch 最核心的能力之一。

理解查询上下文与过滤上下文的区别，才能写出高效的查询。全文查询、精确查询、复合查询、嵌套查询……每种查询都有其适用场景。

记住：能用 `filter` 的地方就不要用 `must`，能提前过滤的数据就不要留给排序阶段。查询优化的核心是**减少不必要的数据处理**。
