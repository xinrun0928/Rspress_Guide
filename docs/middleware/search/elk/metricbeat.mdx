# Metricbeat 系统指标采集

日志告诉你「发生了什么」，而指标告诉你「正在发生什么」。

如果说 Filebeat 是日志的眼睛，那么 Metricbeat 就是系统的体检仪。它定期采集服务器和服务的各项指标，让你能实时监控系统健康状态。

## Metricbeat 是什么？

Metricbeat 是 Elastic Stack 中的指标采集器。它定期从系统和服务收集指标数据，发送到 Elasticsearch 或 Logstash。

```
┌──────────────────────────────────────────────────────────┐
│                      Metricbeat                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐    │
│  │   system    │  │     mysql    │  │    nginx    │    │
│  │   模块      │  │     模块     │  │     模块    │    │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘    │
│         │                │                │             │
│         └────────────────┼────────────────┘             │
│                          │                              │
│                   ┌──────▼──────┐                       │
│                   │  指标数据    │                       │
│                   └──────┬──────┘                       │
└──────────────────────────┼──────────────────────────────┘
                           │
                    ┌──────▼──────┐
                    │ Elasticsearch│
                    └─────────────┘
```

Metricbeat 的特点：

- **定期采集**：按固定间隔采集指标，不是事件驱动
- **轻量级**：资源消耗低，可以部署在每台服务器
- **模块化**：预置了大量常用服务模块

## 安装与基本配置

### 安装

```bash
# Linux
curl -L -O https://artifacts.elastic.co/downloads/beats/metricbeat/metricbeat-8.11.0-linux-x86_64.tar.gz
tar xzf metricbeat-8.11.0-linux-x86_64.tar.gz

# macOS
brew install metricbeat

# Docker
docker run -d \
  --name=metricbeat \
  --network=host \
  -v /proc:/hostfs/proc:ro \
  -v /sys/fs/cgroup:/hostfs/sys/fs/cgroup:ro \
  -v /:/hostfs:ro \
  elastic/metricbeat:8.11.0 \
  metricbeat -e -strict.perms=false
```

### 基本配置

```yaml title="metricbeat.yml"
metricbeat.inputs:
  - type: system
    enabled: true
    period: 10s
    processes: ['.*']
    cpu.metrics: ['percentages']
    memory.metrics: ['percentages']

output.elasticsearch:
  hosts: ["es-node1:9200", "es-node2:9200"]
  index: "metrics-system-%{+yyyy.MM.dd}"
```

## 系统指标模块

### CPU 指标

```yaml
- type: cpu
  enabled: true
  period: 10s
```

采集的指标：

| 指标 | 说明 |
|---|---|
| system.cpu.user | 用户态 CPU 使用率 |
| system.cpu.system | 系统态 CPU 使用率 |
| system.cpu.idle | 空闲 CPU 比例 |
| system.cpu.iowait | I/O 等待比例 |
| system.cpu.stolen | 被虚拟机偷走的 CPU |
| system.cpu.cores | CPU 核心数 |

### 内存指标

```yaml
- type: memory
  enabled: true
  period: 10s
```

采集的指标：

| 指标 | 说明 |
|---|---|

| system.memory.actual.used | 实际使用内存 |
| system.memory.used | 总使用内存 |
| system.memory.swap.used | Swap 使用量 |

## 服务模块

### MySQL 模块

```yaml
- module: mysql
  enabled: true
  metricsets:
    - status
    - galera_status
  period: 10s
  hosts: ["root:password@tcp(127.0.0.1:3306)/"]
```

采集的指标：

| 指标 | 说明 |
|---|---|
| mysql.status.threads.connected | 当前连接数 |
| mysql.status.threads.running | 运行中的线程 |
| mysql.status.queries | 查询总数 |
| mysql.status.questions | 服务器收到的问题数 |
| mysql.status.innodb.row_lock_time | 行锁等待时间 |

## Kubernetes 模块

```yaml
- module: kubernetes
  enabled: true
  metricsets:
    - node
    - system
    - pod
    - container
    - volume
    - apiserver
  period: 15s
  hosts: ["kube-state-metrics:8080"]
  add_metadata: true
  bearer_token_file: /var/run/secrets/kubernetes.io/serviceaccount/token
```

:::tip Kubernetes 部署建议
在 Kubernetes 中，建议以 DaemonSet 方式部署 Metricbeat，采集每个节点的系统指标。同时可以部署单独的 Deployment 采集集群级别的指标。
:::

## 数据处理

### 使用 processors 丰富数据

```yaml
processors:
  - add_host_metadata:
      when.not.contains.tags: forwarded
  - add_cloud_metadata: ~
  - add_docker_metadata: ~
  - add_kubernetes_metadata: ~
  - add_fields:
      target: ''
      fields:
        env: production
        datacenter: dc1
```

### 指标计算

使用 Metricbeat 的 metricset 聚合功能：

```yaml
- type: system
  period: 30s
  metrics:
    - cpu:
        metrics: ['percentages']
    - process:
        include_cpu: true
        include_memory: true
        cmdline敛:
          - '/usr/sbin/sshd'
```

## 可视化与分析

### Kibana 预置仪表盘

Metricbeat 自带预配置的 Kibana 仪表盘：

```bash
# 加载预置仪表盘
metricbeat setup --dashboards
```

### 常用查询

#### CPU 使用率超过 80%

```kql
system.cpu.user.pct > 0.8 OR system.cpu.system.pct > 0.8
```

#### 内存使用率超过 90%

```kql
system.memory.used.pct > 0.9
```

#### 磁盘使用率

```kql
system.disk.used.pct > 0.8
```

### 告警规则示例

```yaml
# CPU 告警
rule: CPU 使用率超过 80%
condition: avg(system.cpu.user.pct) > 0.8 over 5m
frequency: 5m
actions:
  - webhook: slack
```

## 性能调优

### 采集间隔调整

```yaml
# 对精度要求不高的指标，使用更长的采集间隔
- type: memory
  period: 60s    # 1 分钟采集一次

- type: cpu
  period: 10s    # CPU 需要更频繁采集
```

### 禁用不需要的指标

## 常见问题

### Q: Metricbeat 和 Beats protocol 是什么关系？

Metricbeat 使用 Beats protocol 与 Elasticsearch/Logstash 通信。这是专门为 Beats 系列设计的传输协议，比普通 HTTP 更高效。

### Q: 如何添加自定义指标？

Metricbeat 支持通过外部脚本收集指标：

```yaml
- type: http
  enabled: true
  period: 10s
  hosts: ["http://localhost:8080/metrics"]
  format: json
```

### Q: Metricbeat 可以采集 Prometheus 指标吗？

可以使用 Metricbeat 的 HTTP 采集器，或使用专门的 Prometheus 采集器（通过 module）。
