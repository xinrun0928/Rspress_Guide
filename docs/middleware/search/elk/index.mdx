# ELK Stack 生态

当我们谈论日志收集与分析系统时，ELK Stack 是一个绕不开的话题。它由 Elasticsearch、Logstash、Kibana 三个核心组件组成，加上 Beats 数据采集器，形成了完整的日志处理链路。

但 ELK 不仅仅是三个工具的简单组合——它是一个生态系统，一个解决海量数据问题的完整方案。

## 什么是 ELK Stack？

ELK 是四个开源软件的缩写：

- **E**lasticsearch：分布式搜索和分析引擎
- **L**ogstash：数据处理管道
- **K**ibana：可视化平台
- **B**eats：轻量级数据采集器

这个组合的诞生，要从日志分析说起。

## 为什么需要 ELK？

在微服务架构下，一个请求可能经过十几个服务的处理。排查问题时，你需要在十几个日志文件中追踪一条请求的完整链路。传统方式下，这几乎是不可能完成的任务。

ELK 解决了这个问题：

```
杂乱日志 → 统一收集 → 集中存储 → 可视化分析
```

所有服务的日志汇聚到一个平台，你可以在一个界面中搜索、过滤、分析所有日志。这就是 ELK 的核心价值。

## ELK 演进历程

```
2010: Elasticsearch 诞生
2012: Kibana 加入
2013: Logstash 整合
2015: Beats 加入，ELK Stack 正式命名
2018: Elastic 公司商业化
至今: 云原生、Observability 方向演进
```

## 组件定位

| 组件 | 职责 | 特点 |
|---|---|---|
| Beats | 数据采集 | 轻量、低资源占用 |
| Logstash | 数据处理 | 功能强大、可扩展 |
| Elasticsearch | 数据存储与搜索 | 分布式、高性能 |
| Kibana | 数据可视化 | 图形化、交互丰富 |
