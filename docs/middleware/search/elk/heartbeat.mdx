# Heartbeat 站点健康检测

你的网站还在线吗？你的 API 还能访问吗？当用户打电话投诉时，你应该是最先知道答案的人。

Heartbeat 是 Elastic Stack 中的健康检测组件。它定期检查你的网站、服务、端口是否可达，在问题发生的第一时间通知你。

## Heartbeat 是什么？

Heartbeat 是 Elastic 的「健康检查员」。它通过发送 HTTP/TCP/ICMP 请求来探测目标服务是否可用，并将检测结果发送到 Elasticsearch。

```
┌─────────────┐
│   Heartbeat  │
│              │
│  ┌─────────┐ │
│  │ HTTP    │───▶ 检查 https://example.com
│  └─────────┘ │
│  ┌─────────┐ │
│  │ TCP     │───▶ 检查 127.0.0.1:5432
│  └─────────┘ │
│  ┌─────────┐ │
│  │ ICMP    │───▶ ping 检查
│  └─────────┘ │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│Elasticsearch│
└─────────────┘
       │
       ▼
┌─────────────┐
│   Kibana    │
│  (Uptime)   │
└─────────────┘
```

Heartbeat 的特点：

- **主动探测**：不是等故障报告，而是主动检查
- **URL 级检测**：检测到具体的 URL，不只是主机
- **TLS 检测**：检查证书过期时间
- **完整性检测**：不仅是可达，还检测响应内容

## 安装与基本配置

### 安装

```bash
# Linux
curl -L -O https://artifacts.elastic.co/downloads/beats/heartbeat/heartbeat-8.11.0-linux-x86_64.tar.gz
tar xzf heartbeat-8.11.0-linux-x86_64.tar.gz

# macOS
brew install heartbeat

# Docker
docker run -d \
  --name=heartbeat \
  -v /path/to/heartbeat.yml:/usr/share/heartbeat/heartbeat.yml \
  elastic/heartbeat:8.11.0
```

### 基本配置

```yaml title="heartbeat.yml"
heartbeat.monitors:
  - type: http
    enabled: true
    urls:
      - "https://example.com"
    schedule: '@every 10s'
    name: example-site
    fields:
      env: production

output.elasticsearch:
  hosts: ["es1:9200"]
  index: "heartbeat-%{+yyyy.MM.dd}"
```

## 监控类型

### HTTP 监控

最基本的监控方式，检测 URL 是否可达：

```yaml
- type: http
  enabled: true
  name: API 监控
  schedule: '@every 30s'
  urls:
    - https://api.example.com/health
    - https://api.example.com/v1/status
  timeout: 5s
  check.response:
    status: 200
  ssl:
    certificate_authorities: ['/etc/ca.crt']
```

### HTTP 响应内容检测

不仅检查状态码，还检查响应内容：

```yaml
- type: http
  enabled: true
  name: 首页检测
  schedule: '@every 30s'
  urls:
    - https://example.com
  check.response:
    status: 200
    body:
      positive:
        - "Welcome"
        - "Login"
      negative:
        - "Error"
        - "Maintenance"
    headers:
      Content-Type: "text/html"
```

### TCP 监控

检测端口是否开放：

```yaml
- type: tcp
  enabled: true
  name: 数据库连接检测
  schedule: '@every 30s'
  hosts:
    - "db.example.com:5432"
  timeout: 3s
  check:
    send: '\x08'      # 发送空字节
    expect: 'OK'     # 期望响应
```

### ICMP 监控

基础的 ping 检测：

```yaml
- type: icmp
  enabled: true
  name: 服务器 PING 检测
  schedule: '@every 60s'
  hosts:
    - "server1.example.com"
    - "server2.example.com"
  wait: 1s
```

:::warning ICMP 权限
ICMP 监控需要 root 权限或特定的 cap_net_raw 能力。在容器环境中可能需要特殊配置。
:::

## TLS 证书检测

Heartbeat 可以检测 SSL/TLS 证书的过期时间：

```yaml
- type: http
  enabled: true
  name: TLS 证书检测
  schedule: '@every 24h'     # 证书检查不需要太频繁
  urls:
    - "https://example.com"
  ssl:
    certificate: true       # 验证证书
    certificate_authorities:
      - "/etc/ssl/certs/ca-cert.crt"
  check:
    sent: []
```

### 告警规则：证书即将过期

```yaml
# Kibana Alert 配置
rule_name: "证书 30 天内过期"
condition: >-
  ctx.payload.certificates.any(c, 
    c.tls.cert.not_after >= now and 
    c.tls.cert.not_after <= now.plus(30d)
  )
```

## 配置详解

### 调度器

Heartbeat 支持灵活的调度：

```yaml
schedule: '@every 5s'      # 每 5 秒
schedule: '*/30 * * * * *' # 每 30 秒（cron 语法）
schedule: '0 0 * * * *'    # 每小时整点
schedule: '0 0,12 * * *'   # 每天 0 点和 12 点
```

### 超时设置

```yaml
timeout: 2s                # 单次检查超时
watch.poll_file.interval: 1s  # 监控文件更新间隔
```

### 认证配置

```yaml
- type: http
  enabled: true
  name: 认证 API 检测
  urls:
    - https://api.example.com/secure
  check.request:
    method: GET
    headers:
      Authorization: "Basic dXNlcjpwYXNz"
```

### 代理支持

```yaml
- type: http
  enabled: true
  urls:
    - https://example.com
  proxy_url: http://proxy.example.com:8080
  proxy_use_local_resolver: false
```

## Kubernetes 部署

```yaml title="heartbeat-kubernetes.yml"
apiVersion: v1
kind: ConfigMap
metadata:
  name: heartbeat-config
data:
  heartbeat.yml: |
    heartbeat.monitors:
      - type: http
        schedule: '@every 10s'
        urls:
          - "https://kubernetes.default.svc/healthz"
        ssl:
          verify: false
        check.response:
          status: 200
    
    output.elasticsearch:
      hosts: ['${ELASTICSEARCH_HOST:elasticsearch}:9200']
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: heartbeat
spec:
  replicas: 1
  template:
    spec:
      serviceAccountName: heartbeat
      containers:
        - name: heartbeat
          image: elastic/heartbeat:8.11.0
          args:
            - "-e"
            - "-E"
            - "output.elasticsearch.hosts=${ELASTICSEARCH_HOSTS}"
          env:
            - name: ELASTICSEARCH_HOSTS
              value: "elasticsearch:9200"
          volumeMounts:
            - name: config
              mountPath: /usr/share/heartbeat/heartbeat.yml
              subPath: heartbeat.yml
      volumes:
        - name: config
          configMap:
            name: heartbeat-config
```

## Kibana Uptime 应用

Heartbeat 采集的数据可以通过 Kibana 的 Uptime 应用展示：

### 功能特性

- **全球视图**：在世界地图上显示所有监控点
- **状态概览**：绿色（正常）、黄色（缓慢）、红色（失败）
- **历史记录**：查看过去的监控历史
- **告警集成**：与 Kibana Alerting 集成

### 预置仪表盘

```bash
# 加载 Uptime 仪表盘
heartbeat setup --dashboards
```

### 常用查询

#### 查看所有失败监控

```kql
monitor.status: down
```

#### 查看响应时间趋势

```kql
monitor.name: "example-site" AND observe.ssl.rtt.total: *
```

## 告警配置

### 创建告警规则

```yaml
# Stack Management → Rules → Create rule

规则名称: 服务不可用告警
类型: Heartbeat / Uptime alert
条件:
  - 监控器状态变为 down
  - 持续时间 > 1 分钟
通知:
  - Slack: #alerts-prod
  - PagerDuty: 关键告警
```

### 自定义告警脚本


## 最佳实践

### 监控策略

| 类型 | 检测间隔 | 超时设置 | 告警阈值 |
|---|---|---|---|
| 核心业务 API | 10-30s | 2-5s | 1 分钟不可用 |
| 普通服务 | 30-60s | 5-10s | 3 分钟不可用 |
| 非关键服务 | 1-5 分钟 | 10-30s | 10 分钟不可用 |
| TLS 证书 | 24 小时 | 10s | 30 天内过期 |

### 分层监控

```yaml
# 第一层：DNS/负载均衡
- type: tcp
  name: 入口检测
  hosts:
    - "api.example.com:443"

# 第二层：API 网关
- type: http
  name: 网关检测
  urls:
    - "https://api.example.com/gateway/health"

# 第三层：具体服务
- type: http
  name: 服务检测
  urls:
    - "https://api.example.com/service-a/health"
    - "https://api.example.com/service-b/health"
```

### 监控指标选择

不要只监控 HTTP 200：

```yaml
# 好：检查具体业务状态
check.response:
  status: 200
  body:
    positive:
      - '"status": "healthy"'
    negative:
      - '"status": "degraded"'

# 差：只检查 HTTP 状态
check.response:
  status: [200, 301, 302]
```

## 常见问题

### Q: Heartbeat 和外部监控服务（如 Pingdom）有什么优势？

Heartbeat 的优势在于：
- 与 ELK 生态深度集成
- 可以访问内网服务
- 统一的数据存储和分析
- 自托管，无额外费用

### Q: 如何监控多个区域的同一个服务？

可以使用多个 Heartbeat 实例部署在不同区域：

```yaml
# Asia instance
heartbeat.monitors:
  - type: http
    urls: ["https://example.com"]
    fields:
      region: asia
```

### Q: 监控的 URL 很多，配置会很复杂？

可以使用配置文件分离：

```yaml
heartbeat.config.monitors:
  path: ${path.config}/monitors.d/*.yml
  reload.enabled: true
  reload.period: 30s
```

然后在 `monitors.d/` 目录下创建多个配置文件。

Heartbeat 是你系统的「健康监测仪」。它默默运行，在问题发生的第一时间告诉你。
