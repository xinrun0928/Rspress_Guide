# Packetbeat 网络流量分析

日志告诉你「发生了什么」，指标告诉你「系统状态如何」，但如果想看到「谁在和谁通信」「传输了什么数据」，你需要 Packetbeat。

Packetbeat 是 Elastic Stack 中唯一的实时网络包分析器。它截获网络流量，解码协议，提取有用的信息。

## Packetbeat 是什么？

Packetbeat 通过 libpcap 在网络层面截获数据包，解码应用层协议（如 HTTP、DNS、MySQL），提取结构化的信息。

```
┌──────────────────────────────────────────────────────────────┐
│                        网络流量                               │
│                                                               │
│  Client ──────▶ [Packetbeat] ──────▶ Server                   │
│                    │                                          │
│                    │                                          │
│            捕获数据包                                         │
│            解码协议                                           │
│            提取信息                                           │
│                    │                                          │
└────────────────────┼──────────────────────────────────────────┘
                     │
                     ▼
              ┌─────────────┐
              │Elasticsearch│
              └─────────────┘
```

为什么选择 Packetbeat？

- **无代理**：不需要在服务器安装 agent
- **协议解析**：深入理解应用层协议
- **性能分析**：测量每个请求的响应时间
- **安全分析**：发现异常流量模式

:::warning 法律注意事项
Packetbeat 需要将网卡设置为混杂模式，这可能涉及法律风险。确保你有权监控相关网络流量，通常只用于自己的服务器网络。
:::

## 工作原理

### 数据包捕获

Packetbeat 使用 AF_PACKET（Linux）或 BPF（Berkeley Packet Filter）捕获数据包：

```yaml
interfaces:
  device: eth0
  type: af_packet
  buffer_size_mb: 50
  fanout_group: packetbeat
```

### 协议解码

支持的应用层协议：

| 协议 | 默认端口 | 说明 |
|---|---|---|
| HTTP | 80, 8080 | 请求方法、URL、状态码、响应时间 |
| DNS | 53 | 查询类型、响应码、延迟 |
| MySQL | 3306 | 查询语句、影响行数、错误 |
| PostgreSQL | 5432 | 查询语句、执行时间 |
| Redis | 6379 | 命令、键、响应时间 |
| MongoDB | 27017 | 操作、集合、响应时间 |
| TLS | 443 | 加密流量元数据 |
| Kafka | 9092 | 主题、分区、消息大小 |

### 数据流处理

```
数据包 ──▶ 分流 ──▶ 协议解析 ──▶ 重组 ──▶ 输出
            │                              │
            ▼                              ▼
        五元组                         Elasticsearch
        (连接跟踪)
```

## 安装与配置

### 安装

```bash
# Linux
curl -L -O https://artifacts.elastic.co/downloads/beats/packetbeat/packetbeat-8.11.0-linux-x86_64.tar.gz
tar xzf packetbeat-8.11.0-linux-x86_64.tar.gz

# macOS
brew install packetbeat

# Docker (需要特权模式)
docker run --cap-add=NET_RAW --cap-add=NET_ADMIN \
  --network=host \
  -v /path/to/packetbeat.yml:/usr/share/packetbeat/packetbeat.yml \
  elastic/packetbeat:8.11.0
```

:::warning Docker 限制
在 Docker 容器中运行 Packetbeat 需要 `--network=host` 和额外的 capabilities，这在某些环境中可能不可用。
:::

### 基本配置

```yaml title="packetbeat.yml"
packetbeat.interfaces:
  - type: af_packet
    device: eth0
    buffer_size_mb: 100

packetbeat.protocols:
  - type: http
    ports: [80, 8080, 8000]
  - type: dns
    ports: [53]
  - type: mysql
    ports: [3306]

packetbeat.flows:
  timeout: 30s
  period: 10s

output.elasticsearch:
  hosts: ["es1:9200"]
  index: "packetbeat-%{+yyyy.MM.dd}"
```

## 协议配置详解

### HTTP 监控

```yaml
- type: http
  ports: [80, 8080, 8000, 5000, 8001]
  
  # 请求解析配置
  parse_request: true
  parse_response: true
  
  # 隐藏敏感信息
  hide_keywords: ['pass', 'secret', 'token', 'authorization']
  replace_numbers: true
  
  # 超时设置
  transaction_timeout: 30s
  
  # 方法过滤
  send_headers:
    - "User-Agent"
    - "Content-Type"
    - "Host"
```

### DNS 监控

```yaml
- type: dns
  ports: [53]
  
  # 解析配置
  include_authorities: true
  include_additionals: true
  
  # 超时
  transaction_timeout: 5s
```

### 数据库监控

```yaml
# MySQL
- type: mysql
  ports: [3306]
  send_query: true
  send_response: false
  query_max_size: 10000

# PostgreSQL
- type: postgresql
  ports: [5432]
  send_query: true
  send_response: false
```

### Redis 监控

```yaml
- type: redis
  ports: [6379]
  send_request: true
  send_response: true
```

## 流量分析配置

### Flows 模式

Flows 模式记录网络连接统计：

```yaml
packetbeat.flows:
  enabled: true
  timeout: 30s        # 连接超时后输出统计
  period: 10s         # 定期输出统计
```

输出的字段：

| 字段 | 说明 |
|---|---|
| flows.final | 是否为最终统计 |
| flows.last время | 最后活跃时间 |
| source.stats.net_bytes_total | 源端总字节数 |
| dest.stats.net_bytes_total | 目标端总字节数 |
| source.packsts.count | 源端包数 |
| dest.packets.count | 目标端包数 |

### 事务记录

```yaml
packetbeat.transactions:
  enabled: true
  timeout: 5s          # 事务超时
```

## 性能调优

### 资源控制

```yaml
packetbeat.interfaces:
  - type: af_packet
    device: eth0
    buffer_size_mb: 100
    with_vlans: true
    copy_mode: true

# 丢弃不关心的流量
packetbeat.autodiscover:
  providers:
    - type: docker
      hints.enabled: true
```

### 过滤规则

使用 BPF 语法过滤流量：

```yaml
packetbeat.interfaces:
  - type: af_packet
    device: eth0
    bpf_filter: "tcp and (port 80 or port 8080)"
```

常见 BPF 过滤：

```bash
# 只监控 HTTP
bpf_filter: "tcp port 80 or tcp port 8080"

# 排除特定 IP
bpf_filter: "tcp and not (host 192.168.1.100)"

# 只监控 HTTPS
bpf_filter: "tcp port 443"
```

### 并发配置

```yaml
packetbeat.procs:
  enabled: true
  monitored:
    - process: nginx
    - process: httpd
    - process: mysqld
```

  # 检测爬虫

```yaml
# 检测大规模请求
rule: "每秒请求数超过 1000"
condition: >
  COUNT() > 1000 over 1s by client.ip
action: webhook
```

### 常用可视化

- **流量概览**：总流量、请求数、带宽
- **协议分布**：HTTP、DNS、MySQL 占比
- **Top 客户端**：请求最多的 IP
- **响应时间**：P50、P95、P99
- **错误分布**：4xx、5xx 分布

### 常用查询

#### 慢请求

```kql
type: http AND http.response_time_ms > 1000
```

#### 错误请求

```kql
type: http AND http.status >= 400
```

#### DNS 解析失败

```kql
type: dns AND dns.response_code: NXDOMAIN
```

## Kubernetes 部署

```yaml title="packetbeat-daemonset.yml"
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: packetbeat
spec:
  selector:
    matchLabels:
      app: packetbeat
  template:
    metadata:
      labels:
        app: packetbeat
    spec:
      hostNetwork: true
      containers:
        - name: packetbeat
          image: elastic/packetbeat:8.11.0
          securityContext:
            capabilities:
              add:
                - NET_ADMIN
                - NET_RAW
          volumeMounts:
            - name: config
              mountPath: /usr/share/packetbeat/packetbeat.yml
              subPath: packetbeat.yml
          env:
            - name: ELASTICSEARCH_HOSTS
              value: "elasticsearch:9200"
      volumes:
        - name: config
          configMap:
            name: packetbeat-config
```

:::warning Kubernetes 网络
在 Kubernetes 中使用 Packetbeat 需要考虑 CNI 的网络拓扑，可能需要特殊配置才能正确捕获流量。
:::

## 常见问题

### Q: Packetbeat 和 Metricbeat 有什么区别？

| 维度 | Packetbeat | Metricbeat |
|---|---|---|
| 数据来源 | 网络流量 | 系统/服务 API |
| 数据类型 | 请求/响应详情 | 聚合指标 |
| 粒度 | 单个事务 | 统计值 |
| 资源消耗 | 较高 | 较低 |
| 部署方式 | 旁路/镜像 | 代理 |

### Q: 如何处理高流量场景？

在高流量场景下，可以：

1. 使用 BPF 过滤不关心的流量
2. 限制采样率
3. 考虑只监控关键端口
4. 使用聚合减少数据量

### Q: Packetbeat 能监控加密流量吗？

Packetbeat 只能看到加密流量的元数据（源 IP、目标 IP、端口、TLS 版本、SNI），无法解密内容。如需解密，需要在负载均衡器或 Nginx 配置 SSL 终止。

Packetbeat 让你「看见」网络中的每一次对话。但请谨慎使用，确保合规。
