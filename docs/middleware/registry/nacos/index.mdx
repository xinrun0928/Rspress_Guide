# Nacos

Nacos（Naming and Configuration Service）是阿里巴巴开源的**一站式服务发现和配置管理平台**。它同时具备**注册中心**和**配置中心**的能力，是 Spring Cloud 生态中 Eureka + Spring Cloud Config 的有力替代者。

## 为什么选择 Nacos

在 Nacos 出现之前，Spring Cloud 项目通常需要组合多个组件：
- **注册中心**：Eureka、Consul 或 ZooKeeper
- **配置中心**：Spring Cloud Config、Apollo

这种组合虽然可行，但带来了额外的问题：
- 两个组件意味着两套部署、两套运维
- 组件之间缺乏联动，配置变更无法触发服务重注册
- 不同的数据模型导致开发体验不统一

Nacos 的目标是一站式解决这两个问题，让微服务只需要依赖一个组件。

## 核心特性

### 双模式支持：AP 与 CP

Nacos 支持两种一致性模型，可以根据业务场景灵活选择：

| 模式 | 协议 | 说明 |
|---|---|---|
| **AP 模式** | Distro 协议 | 最终一致性，适用于注册中心场景 |
| **CP 模式** | Raft 协议 | 强一致性，适用于配置中心场景 |

```
配置中心：选择 CP 模式，保证配置变更的强一致性
注册中心：选择 AP 模式，保证服务可用性和最终一致性
```

这个设计体现了 Nacos 的核心理念：**注册中心优先可用性，配置中心优先一致性**。

### 完整的功能矩阵

| 功能 | 说明 |
|---|---|
| **服务注册与发现** | 支持临时实例和永久实例，主动推送和订阅变更 |
| **配置管理** | namespace + group + dataId 三层隔离，热更新 |
| **健康检查** | TCP、HTTP、MySQL、主动上报多种方式 |
| **流量管理** | 权重、路由、熔断 |
| **服务元数据** | 标签、metadata 扩展 |
| **多环境支持** | namespace 隔离 dev/test/prod |

### 友好的管理界面

Nacos 提供完整的 Web 管理界面，可以：
- 可视化查看服务列表和实例信息
- 编辑配置并实时发布
- 监控服务健康状态
- 管理命名空间和分组

### Spring Cloud 集成

Nacos 与 Spring Cloud 深度集成，开箱即用：

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>
```

```yaml
# application.yml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: nacos-server:8848
        namespace: dev
      config:
        server-addr: nacos-server:8848
        file-extension: yaml
```

## Nacos 与其他注册中心的对比

| 维度 | Nacos | Eureka | ZooKeeper | Consul |
|---|---|---|---|---|
| **一致性模型** | AP + CP 可切换 | AP | CP | CP |
| **配置中心** | 原生支持 | 不支持 | 需二次开发 | 支持 KV 存储 |
| **健康检查** | TCP/HTTP/MySQL/Agent | 心跳续约 | 临时节点 | Script/HTTP/TCP/TTL |
| **多语言支持** | 好（HTTP/SDK） | 一般 | 好 | 非常好 |
| **运维复杂度** | 低 | 低 | 高 | 中 |
| **社区活跃度** | 非常活跃 | 低（停止维护） | 稳定 | 活跃 |

## Nacos 的演进路线

```
Nacos 1.x（2018）→ Nacos 2.x（2021）→ Nacos 3.x（规划中）

1.x：HTTP 通信，长连接需第三方网关
2.x：引入 gRPC 通信协议，支持长连接，主动推送
3.x：云原生优化，多协议支持
```

Nacos 2.0 最大的变化是引入了**gRPC 长连接**，大幅提升了服务变更推送的效率，弥补了 1.x 版本轮询拉取的延迟问题。

## 适用场景

| 场景 | 推荐理由 |
|---|---|
| **Spring Cloud 微服务** | 与 Spring Cloud 深度集成，开箱即用 |
| **需要配置中心 + 注册中心** | 一站式解决，无需多个组件 |
| **国内项目** | 阿里背景，中文社区活跃，文档完善 |
| **多环境配置隔离** | namespace 原生支持 dev/test/prod |
| **Dubbo 用户** | Nacos 是 Dubbo 官方推荐的注册中心 |
