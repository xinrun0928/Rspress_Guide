# Nacos 注册中心

Nacos 的注册中心功能解决了一个核心问题：**服务提供者如何告知消费者「我在这里」**。与传统的心跳续约机制不同，Nacos 支持多种健康检查方式，并能在服务变更时主动推送给订阅者。

## 服务注册流程

### 基本流程

当一个服务实例启动时，它会经历以下步骤完成注册：

```
1. 应用启动，读取 Nacos 配置
2. 向 Nacos Server 发送注册请求（HTTP/gRPC）
3. Server 存储实例信息到内存
4. Server 异步同步到其他节点
5. 订阅该服务的消费者收到推送
6. 消费者更新本地实例列表
```

```java
// 服务注册的核心逻辑（伪代码）
public class NacosServiceRegistry {

    public void register(Instance instance) throws Exception {
        // 构造注册请求
        NamingServiceRequest request = NamingRequest
            .builder()
            .setServiceName(instance.getServiceName())
            .setInstanceId(instance.getInstanceId())
            .setIp(instance.getIp())
            .setPort(instance.getPort())
            .setHealthy(instance.isHealthy())
            .setWeight(instance.getWeight())
            .setMetadata(instance.getMetadata())
            .build();

        // 发送到 Nacos Server
        namingProxy.registerService(
            getServiceName(),
            getGroupName(),
            instance
        );
    }
}
```

### 注册请求内容

服务注册时需要上报的信息：

| 字段 | 说明 | 示例 |
|---|---|---|
| `serviceName` | 服务名 | `order-service` |
| `groupName` | 分组名 | `DEFAULT_GROUP` |
| `instanceId` | 实例 ID | `192.168.1.10:8080` |
| `ip` | IP 地址 | `192.168.1.10` |
| `port` | 端口 | `8080` |
| `healthy` | 健康状态 | `true` |
| `weight` | 权重 | `1.0` |
| `metadata` | 元数据 | `{"version": "v1"}` |

### 临时实例与永久实例

Nacos 支持两种实例类型：

| 类型 | 说明 | 存储位置 | 健康检查 |
|---|---|---|---|
| **临时实例** | 客户端主动上报心跳 | 内存 | 依赖客户端心跳，超时删除 |
| **永久实例** | 注册后持久化存储 | 磁盘 | Nacos 主动探测，不删除 |

```
临时实例场景：
- 服务实例由客户端主动注册
- 客户端通过心跳保活
- 客户端宕机后，心跳停止，实例被自动删除

永久实例场景：
- 服务实例由运维手动注册或通过 Agent 注册
- Nacos 主动探测实例健康状态
- 即使实例宕机，实例信息也保留（标记为不健康）
```

```yaml
# 临时实例（默认）
spring:
  cloud:
    nacos:
      discovery:
        ephemeral: true  # 临时实例

# 永久实例
spring:
  cloud:
    nacos:
      discovery:
        ephemeral: false  # 永久实例
```

## 心跳保活机制

### 临时实例心跳

临时实例通过客户端主动发送心跳来保活。默认心跳间隔 5 秒，超时时间 15 秒：

```java
// 心跳任务调度
public class BeatTask implements Runnable {
    public void run() {
        // 构建心跳请求
        BeatRequest request = BeatRequest.builder()
            .setServiceName(serviceName)
            .setInstanceId(instanceId)
            .setIp(ip)
            .setPort(port)
            .setClusterName(clusterName)
            .setMark(true)  // 心跳标记
            .build();

        // 发送心跳
        namingProxy.sendBeat(request);
    }
}
```

| 参数 | 默认值 | 说明 |
|---|---|---|
| `beatInterval` | 5000ms | 心跳间隔 |
| `clientBeatTimeOut` | 15000ms | 心跳超时时间 |
| `instanceHeartTimeOut` | 15000ms | 实例过期时间 |

```
心跳机制：
- 每 5 秒发送一次心跳
- 连续 3 次心跳失败（15 秒）判定为不健康
- 连续 3 次心跳失败（15 秒）后删除实例
```

### 永久实例健康检查

永久实例由 Nacos Server 主动探测，不依赖客户端心跳：

| 检查类型 | 说明 | 配置 |
|---|---|---|
| **TCP 检查** | Nacos 向实例的任意端口发起 TCP 连接 | 默认 |
| **HTTP 检查** | Nacos 向指定路径发送 HTTP 请求 | `health-check: http` |
| **MySQL 检查** | Nacos 执行 SQL 验证数据库连接 | `health-check: mysql` |
| **自定义检查** | 开发者实现健康检查逻辑 | `health-check: custom` |

```yaml
# TCP 健康检查（默认）
spring:
  cloud:
    nacos:
      discovery:
        health-check:
          type: tcp
          interval: 10000  # 检查间隔 10 秒

# HTTP 健康检查
spring:
  cloud:
    nacos:
      discovery:
        cluster:
          health-check:
            type: http
            path: /actuator/health
            interval: 10000
```

## 服务发现

### 主动查询

服务消费者启动时，会从 Nacos Server 获取服务实例列表：

```java
// 服务发现
public List<Instance> selectInstances(String serviceName) {
    // 从本地缓存获取
    List<Instance> instances = serviceInfoHolder
        .getServiceInfo(serviceName)
        .getHosts();

    // 过滤不健康的实例
    return instances.stream()
        .filter(Instance::isHealthy)
        .collect(Collectors.toList());
}
```

### 订阅变更推送

Nacos 2.0 引入 gRPC 长连接，支持**服务变更主动推送**。当服务列表发生变化时，Nacos Server 会立即通知所有订阅者：

```
订阅流程：
1. 客户端订阅服务：建立 gRPC 长连接到 Server
2. Server 记录订阅关系：保存哪些客户端订阅了哪些服务
3. 服务变更时：Server 主动推送新的实例列表
4. 客户端更新本地缓存：刷新服务实例列表
```

```java
// 订阅服务变更
public void subscribe(String serviceName, EventListener listener) {
    // 注册订阅回调
    eventDispatcher.addListener(
        serviceName,
        groupName,
        clusterName,
        listener
    );

    // 建立订阅关系
    namingProxy.subscribe(serviceName, groupName);
}
```

### 服务变更通知

```java
// 订阅回调实现
NamingListener listener = new NamingListener() {
    @Override
    public void receiveNotify(ServiceInfo serviceInfo) {
        // 收到服务变更通知
        System.out.println("服务列表变更：" + serviceInfo.getHosts());
    }
};

// 订阅服务
namingService.subscribe("order-service", listener);
```

## 负载均衡

Nacos 支持多种负载均衡策略，通过权重和元数据实现灵活的流量分配。

### 权重负载均衡

实例权重决定了流量分配的相对比例：

| 实例 | 权重 | 流量占比 |
|---|---|---|
| 实例 A | 1 | 10% |
| 实例 B | 1 | 10% |
| 实例 C | 8 | 80% |

```java
// Nacos 内置的权重负载均衡器
public class NacosLoadBalancer {

    public Instance select(LoadBalancerRule rule) {
        // 根据权重选择实例
        return rule.choose(serviceInstanceList);
    }
}
```

```yaml
# 设置实例权重
spring:
  cloud:
    nacos:
      discovery:
        weight: 0.8  # 该实例承担 80% 流量
```

### 集群权重

```yaml
# nacos-cluster.yml
clusters:
  - cluster-name: HANGZHOU
    health-check-type: http
    health-check-url: /actuator/health
    weight: 2  # 杭州集群权重更高
  - cluster-name: SHANGHAI
    health-check-type: tcp
    weight: 1
```

### 熔断保护

Nacos 支持在实例不健康时自动降低权重或摘除流量：

```yaml
spring:
  cloud:
    nacos:
      discovery:
        # 开启实例熔断
        instance-healthy-check:
          enabled: true
          # 连续失败次数阈值
          failCount: 3
          # 恢复后重新加入的间隔
          interval: 10s
```

## 元数据与标签路由

### 服务元数据

每个实例可以附加键值对形式的元数据：

```java
// 设置实例元数据
@Bean
public NacosServiceMetadata serviceMetadata() {
    return new NacosServiceMetadata.Builder()
        .addMetadata("version", "v1")
        .addMetadata("region", "hangzhou")
        .addMetadata("env", "test")
        .build();
}
```

### 标签路由

通过元数据标签实现服务路由：

```yaml
# 只调用带有 v2 标签的实例
spring:
  cloud:
    nacos:
      discovery:
        metadata:
          version: v2
```

```java
// 自定义路由规则：优先选择同区域的实例
public Instance selectByRegion(List<Instance> instances, String currentRegion) {
    return instances.stream()
        // 优先选择同区域的实例
        .filter(i -> currentRegion.equals(i.getMetadata().get("region")))
        .findFirst()
        // 如果没有同区域实例，降级到任意实例
        .orElse(instances.get(0));
}
```

## 服务列表缓存

### 本地缓存

Nacos 客户端会在本地缓存服务实例列表，即使与 Server 断开连接，也能使用缓存继续工作：

```
缓存策略：
1. 客户端启动时，拉取完整服务列表
2. 订阅变更推送，实时更新本地缓存
3. 与 Server 断开时，使用本地缓存继续负载均衡
4. 重新连接后，同步最新服务列表
```

### 缓存容灾

```yaml
spring:
  cloud:
    nacos:
      discovery:
        cache:
          # 本地缓存目录
          cache-dir: /tmp/nacos-discovery
          # 缓存过期时间（单位：秒）
          cache-ttl: 300
```

## 总结

| 概念 | 说明 |
|---|---|
| **服务注册** | 实例启动时上报信息到 Nacos Server |
| **临时实例** | 客户端心跳保活，超时自动删除 |
| **永久实例** | Server 主动探测，不删除实例 |
| **心跳机制** | 临时实例 5 秒心跳，15 秒超时 |
| **服务发现** | 主动查询 + 订阅推送（gRPC 长连接） |
| **健康检查** | TCP、HTTP、MySQL、自定义多种方式 |
| **负载均衡** | 权重负载均衡 + 元数据路由 |
| **本地缓存** | 断连时使用缓存，保证可用性 |
