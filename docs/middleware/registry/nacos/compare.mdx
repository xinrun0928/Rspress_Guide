# Nacos vs ZooKeeper vs Consul 对比

注册中心的技术选型是微服务架构中重要的决策之一。本节从多个维度对比 Nacos、ZooKeeper 和 Consul，帮助你根据业务场景做出合理选择。

## 功能特性对比

| 特性 | Nacos | ZooKeeper | Consul |
|---|---|---|---|
| **注册中心** | 原生支持 | 可实现 | 原生支持 |
| **配置中心** | 原生支持 | 需二次开发 | 支持 KV 存储 |
| **一致性模型** | AP + CP 可切换 | CP | CP |
| **健康检查** | TCP/HTTP/MySQL/Agent | 临时节点心跳 | Script/HTTP/TCP/TTL |
| **多语言支持** | 好（HTTP/SDK） | 好（多语言客户端） | 非常好（DNS 接口） |
| **Web 管理界面** | 完整 UI | 无（需第三方） | 完整 UI |
| **配置热更新** | 支持 | 支持（Watch） | 支持 |
| **服务变更推送** | 主动推送（gRPC） | Watch 触发拉取 | HTTP Long Polling |
| **多数据中心** | 支持 | 不原生支持 | 原生支持 |
| **权限控制** | RBAC + ACL | ACL | ACL |
| **服务元数据** | 支持（metadata） | 有限支持 | 支持（metadata、tags） |

## CAP 维度对比

CAP 理论指出：分布式系统无法同时满足一致性（Consistency）、可用性（Availability）和分区容错（Partition tolerance）。注册中心需要在 AP 和 CP 之间做出权衡。

### CP vs AP

| 维度 | CP 注册中心 | AP 注册中心 |
|---|---|---|
| **代表** | ZooKeeper、Consul | Eureka、Nacos（AP 模式） |
| **一致性保证** | 强一致性，写入需要多数节点确认 | 最终一致性，允许短暂不一致 |
| **可用性** | Leader 宕机时可能不可写 | 任何节点都可写入 |
| **适用场景** | 配置中心、强一致需求 | 注册中心、高可用需求 |
| **脑裂风险** | 通过过半机制避免 | 允许短暂脑裂，自动恢复 |

### 三者的 CAP 选择

```
ZooKeeper：
- 定位：分布式协调服务（不仅是注册中心）
- 选择：CP，强一致性
- 理由：ZAB 协议保证数据不丢失，适合配置管理、分布式锁

Consul：
- 定位：服务网格基础设施
- 选择：CP，强一致性
- 理由：与 Vault 集成，需要强一致性保证

Nacos：
- 定位：一站式服务发现 + 配置管理
- 选择：AP + CP 可切换
- 理由：注册中心优先可用（AP），配置中心优先一致（CP）
```

## 架构与协议对比

### 一致性协议

| 注册中心 | 协议 | 说明 |
|---|---|---|
| **ZooKeeper** | ZAB 协议 | 主备复制，专为 ZooKeeper 设计 |
| **Consul** | Raft 共识协议 | 通用共识算法 |
| **Nacos（CP）** | JRaft | Raft 的 Java 实现 |
| **Nacos（AP）** | Distro 协议 | 最终一致性，适合注册中心 |

### 服务发现机制

| 维度 | ZooKeeper | Nacos | Consul |
|---|---|---|---|
| **注册方式** | 临时节点 | 临时/永久节点 | 临时节点 |
| **发现方式** | 客户端 Watch | 主动查询 + 订阅推送 | DNS/HTTP 查询 |
| **推送效率** | Watch 触发后拉取 | gRPC 长连接推送 | Long Polling |
| **客户端复杂度** | 高（需要处理 Watch 重连） | 低（SDK 封装完善） | 低（DNS 天然支持） |

## 运维成本对比

### 部署复杂度

| 维度 | ZooKeeper | Nacos | Consul |
|---|---|---|---|
| **依赖组件** | ZooKeeper 集群 | Nacos 集群 | Consul 集群 + Agent |
| **配置复杂度** | 中（需要 ZooKeeper ensemble） | 低（开箱即用） | 中（Server + Agent） |
| **监控集成** | JMX / Prometheus | 原生 Prometheus | 原生 Prometheus/DogStatsd |
| **升级维护** | 滚动升级需谨慎 | 支持滚动升级 | 支持滚动升级 |
| **资源消耗** | 低 | 中 | 中（Agent 占用资源） |

### 生态集成

| 维度 | ZooKeeper | Nacos | Consul |
|---|---|---|---|
| **Spring Cloud** | 需要额外集成 | 原生支持 | 需要额外集成 |
| **Dubbo** | 可用 | 官方推荐 | 可用 |
| **Kubernetes** | 无官方集成 | 无官方集成 | 官方支持（Consul Connect） |
| **Service Mesh** | 无 | 无 | 原生支持（Sidecar） |

## 选型建议

### 按业务场景选型

| 场景 | 推荐选择 | 理由 |
|---|---|---|
| **Spring Cloud 微服务** | Nacos | 与 Spring Cloud 深度集成，开箱即用 |
| **Dubbo 用户** | Nacos | Dubbo 官方推荐的注册中心 |
| **需要配置中心** | Nacos | 一站式解决，无需多个组件 |
| **分布式锁、Master 选举** | ZooKeeper | 成熟稳定的实现，生态丰富 |
| **多数据中心、跨云** | Consul | 原生支持多数据中心 |
| **服务网格** | Consul | Consul Connect 原生支持 Sidecar |
| **Kafka、HBase 用户** | ZooKeeper | 这些框架本身就依赖 ZooKeeper |
| **云原生、容器化** | Consul | 与 Kubernetes 深度集成 |

### 按团队能力选型

| 团队情况 | 推荐选择 |
|---|---|
| **没有专业运维** | Nacos（部署简单，UI 友好） |
| **已有 ZooKeeper 经验** | ZooKeeper |
| **HashiCorp 生态用户** | Consul |
| **快速启动** | Nacos |

### 按项目阶段选型

| 项目阶段 | 推荐选择 |
|---|---|
| **快速验证 POC** | Nacos（快速上手） |
| **中期发展** | Nacos 或 Consul |
| **大规模生产** | Consul（多数据中心）或 ZooKeeper（高一致性） |
| **长期演进** | Consul（云原生友好） |

## 迁移方案

### 从 ZooKeeper 迁移到 Nacos

```
迁移步骤：
1. 部署 Nacos 集群（与 ZooKeeper 并行运行）
2. 修改服务注册代码，切换到 Nacos
3. 配置 Nacos 与 ZooKeeper 的数据同步（可选）
4. 观察一段时间，确认稳定后下线 ZooKeeper
```

```yaml
# 双注册配置
spring:
  cloud:
    nacos:
      discovery:
        enabled: true
        server-addr: nacos:8848
    zookeeper:
      enabled: true
      connect-string: zookeeper:2181
```

### 从 Eureka 迁移到 Nacos

```
迁移步骤：
1. 部署 Nacos 集群
2. 引入 Nacos Discovery Starter（保留 Eureka 依赖）
3. 配置服务同时注册到 Nacos 和 Eureka
4. 逐步切换消费者到 Nacos
5. 确认无误后下线 Eureka
```

```xml
<!-- Eureka + Nacos 双注册 -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>
```

## 总结

| 选择 | 核心优势 | 最佳场景 |
|---|---|---|
| **Nacos** | 一站式、AP + CP 双模式、Spring Cloud 原生支持 | 国内微服务项目 |
| **ZooKeeper** | 强一致性、成熟稳定、分布式锁首选 | 需要强一致性的场景 |
| **Consul** | 多数据中心、DNS 接口、Service Mesh | 云原生、跨云部署 |
