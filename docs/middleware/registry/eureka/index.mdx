# Eureka

Eureka 是 Netflix 开源的服务注册与发现组件，曾是 Spring Cloud 生态中最广泛使用的注册中心。它以**简单、高可用**著称，与 Spring Boot、Spring Cloud 深度集成，开发者只需几行配置即可完成服务注册。

## Eureka 的历史

```
Netflix 的注册中心演进：
Eureka 1.x（2012）→ 内部使用 → 开源 → Spring Cloud 集成
Eureka 2.x（2018）→ 架构重构 → 停止维护（2020）

2020 年 4 月，Netflix 宣布 Eureka 2.x 停止维护，不再更新。
```

Eureka 的诞生源于 Netflix 内部微服务治理的需求。在大规模分布式系统中，服务数量众多、实例频繁变更，Netflix 需要一个能够**自我修复、高可用**的服务发现方案。Eureka 应运而生，并以其简单可靠的特性，成为 Spring Cloud 项目的标配。

## 核心设计理念

### 去中心化的 AP 模型

与 ZooKeeper 的强一致性不同，Eureka 从设计上选择了 **AP（可用性优先）**。这意味着：

- **不追求强一致性**：各节点之间不保证数据完全同步
- **优先保证可用性**：即使部分节点不可用，服务发现仍然工作
- **最终一致性**：各节点之间的数据会在后台异步同步

这个选择非常适合注册中心的场景——服务发现对可用性的要求远高于一致性。当部分实例不可用时，消费者可以暂时路由到其他可用实例。

### 自我保护机制

Eureka 有一个独特的**自我保护（Self-Preservation）机制**：当大量服务实例心跳超时被标记为不可用时，Eureka 会**停止剔除这些实例**，防止误剔除健康实例。

这个机制源于 Netflix 的实际经验：在云环境中，网络抖动或 JVM GC 暂停可能导致心跳暂时失败，但实例本身是健康的。自我保护机制防止了因瞬时网络问题导致的实例误剔除。

## Eureka 的组件

### Eureka Server

Eureka Server 是注册中心的服务端，负责：
- 接收服务实例的注册请求
- 存储服务实例信息
- 接收客户端的心跳续约
- 剔除过期服务实例
- 提供服务查询接口

```yaml
# application.yml
server:
  port: 8761

eureka:
  instance:
    hostname: localhost
  client:
    # Eureka Server 不需要注册到其他 Server
    register-with-eureka: false
    fetch-registry: false
    service-url:
      defaultZone: http://${eureka.instance.hostname}:${server.port}/eureka/
```

### Eureka Client

Eureka Client 是服务提供者/消费者，包含两个核心组件：

| 组件 | 说明 |
|---|---|
| **Application Client** | 服务消费者，从 Eureka Server 获取服务列表 |
| **Application Service** | 服务提供者，向 Eureka Server 注册 |

```yaml
# 服务提供者
spring:
  application:
    name: order-service

eureka:
  client:
    service-url:
      defaultZone: http://eureka-server:8761/eureka/
  instance:
    prefer-ip-address: true
    lease-renewal-interval-in-seconds: 30
```

## Eureka 与 Spring Cloud

Eureka 与 Spring Cloud 的集成非常简单：

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
</dependency>
```

```java
// 自动注册：添加 @EnableDiscoveryClient 后自动注册到 Eureka
@SpringBootApplication
@EnableDiscoveryClient
public class OrderApplication {
    public static void main(String[] args) {
        SpringApplication.run(OrderApplication.class, args);
    }
}
```

服务调用：

```java
// 使用服务名调用，不需要硬编码 IP 和端口
@Service
public class OrderService {

    @Autowired
    private RestTemplate restTemplate;

    public User getUser(Long userId) {
        // 直接使用服务名，Eureka 会自动解析为可用实例
        return restTemplate.getForObject(
            "http://user-service/user/" + userId,
            User.class
        );
    }
}
```

## Eureka 的局限性

| 局限性 | 说明 |
|---|---|
| **Eureka 2.x 停止维护** | 官方不再更新，存在安全风险 |
| **不支持配置中心** | 只能做服务发现，无法管理配置 |
| **多语言支持差** | 主要面向 Java 生态 |
| **自我保护机制复杂** | 生产环境需要合理配置 |
| **跨数据中心支持弱** | 不原生支持多数据中心部署 |
