# Guide：一份面向 Java 工程师的面试知识图谱

凌晨两点，你刚刷完一组 HashMap 相关的算法题，准备睡了。

但躺下后又忍不住想：HashMap 在 JDK 7 和 JDK 8 的实现有什么区别？头插法和尾插法在扩容时会有什么不同的后果？为什么面试官总是追着问这个问题？

你打开搜索引擎，发现答案要么太浅，要么太散，要么互相矛盾。你想找一个地方，把 Java 面试需要的知识点串成体系，而不是东学一点西学一点。

这就是 Guide 诞生的原因。

## 这个项目是什么

Guide 是一份面向 Java 后端工程师的**系统性面试知识图谱**。

它不是面经的堆砌，不是面试题的罗列，也不是某个框架的官方文档。它要做的事情只有一个：**把 Java 面试涉及的核心知识梳理成体系，让你不仅「知道答案」，更「知道为什么」。**

具体来说，Guide 覆盖以下知识领域：

| 模块 | 内容 | 目标 |
|---|---|---|
| Java 基础 | JVM 内存模型、GC 机制、并发编程、集合源码 | 理解 Java 运行原理，不只停留在 API 调用 |
| 计算机基础 | 操作系统、计网协议、数据结构与算法 | 打牢地基，面试中能举一反三 |
| 数据库 | MySQL 索引、事务与锁，Redis 缓存与持久化 | 高频考点逐一拆解，直击面试核心 |
| 常用框架 | Spring IoC/AOP、Spring Boot 自动装配、MyBatis 执行流程 | 知其然更知其所以然，不被框架细节问住 |
| 分布式 | CAP 理论、分布式事务、注册中心、消息队列 | 微服务架构核心概念系统梳理 |
| 高性能 | 缓存策略、异步消峰、读写分离、分库分表 | 性能优化思路与方案全掌握 |
| 面试准备 | 简历优化、算法训练、系统设计、HR 面与谈薪 | 从简历到 offer 的完整通关攻略 |

这份知识图谱的每一篇文档，都遵循同一套编写原则：**从实际场景出发，讲清楚原理，给出落地方案**。

## 为什么需要这样一份知识图谱

市面上 Java 面试相关的资料已经很多了——博客、公众号、视频课、开源仓库，质量参差不齐。那 Guide 和它们有什么区别？

**区别在于深度和体系化。**

大多数博客是「知识点驱动」的——遇到一个问题写一篇博客，点到为止。你看完十篇博客，可能还是不知道这些知识点之间的联系是什么。

Guide 是「体系驱动」的——先建立框架，再填充细节。每篇文档都知道自己属于哪个模块，和其他知识点有什么关系，最终帮助读者构建起完整的知识网络。

另一个常见的问题是：资料太多，时间太少。

你可能同时关注了十几个技术公众号，加了五个 Java 面试交流群，收藏夹里躺着三百篇文章。但真正需要用的时候，发现脑子里一片空白。

这不是你不努力，是知识没有形成结构。碎片化的信息在没有结构的情况下，大脑只能短期记忆，用完即忘。

Guide 的价值，就是帮你把零散的知识点组织成树状结构——有根有干有枝叶。当你能从根到叶把知识脉络理清楚，面试中被问到任何一个问题，都能快速定位到对应节点，然后顺着脉络展开。

## 怎么用 Guide

### 方式一：按模块系统学习

如果你刚开始准备面试，建议按模块顺序学习。

从 Java 基础开始，打牢地基；然后补充计算机基础和数据库知识，这是面试的常考领域；接着深入常用框架，理解 Spring 和 MyBatis 的核心原理；最后学习分布式和高性能内容，这是拉开差距的关键。

每个模块的入口文档会先给出该模块的知识框架图，帮助你建立全局视野。

### 方式二：按专题查漏补缺

如果你已经有一定基础，只是想针对某个薄弱环节强化，可以直接进入对应模块。

比如你觉得 JVM 调优是弱项，就进入 Java 基础模块，找到 JVM 相关的文章，从内存模型到 GC 原理到调优实战，逐一攻克。

### 方式三：按场景模拟面试

面试前，可以用「面试专项」模块做模拟训练。

这个模块按高频面试场景组织——秒杀系统设计、短链系统、Feed 流、故障排查案例……每个场景都给出完整的分析框架和常见追问方向，帮助你在短时间内建立面试的「肌肉记忆」。

## 内容的组织原则

在写每一篇文档时，Guide 遵循这几个原则：

**从场景出发，不从定义出发。**

不直接告诉你「什么是 HashMap」，而是先问：「为什么 JDK 7 的 HashMap 在并发场景下可能出现死循环？」场景引出问题，问题驱动学习。

**讲清楚原理，不只讲怎么用。**

知道怎么用只能应对简单提问，面试深度追问靠的是原理支撑。文档会从源码层面解释为什么这样设计，以及这样设计的代价是什么。

**给出可落地的方案。**

技术原理懂了还不够，面试官会问：「那你在实际项目中是怎么做的？」每篇文档都有实践相关的章节，给出具体方案和注意事项。

**留有思考空间。**

好的面试不是背答案，而是展示思考过程。文档在关键节点会抛出追问方向，帮助你理解面试官为什么这样问，以及还可以从哪些角度延伸。

## 给不同阶段读者的建议

### 在校学生 / 刚入职场的初级工程师

重点放在 Java 基础、计算机基础和数据库上。这三个模块是面试的门槛，门槛不过，后面的分布式和系统设计再强也没用。

算法每天都要练，不用多，但要有节奏。从简单题开始，建立信心，再逐步挑战中等难度和困难题。

### 2-3 年经验的中级工程师

这个阶段的人往往已经有一定的项目经验，但知识深度不够。面试官会开始追问「为什么」，而不是「做了什么」。

重点补齐 Java 并发编程、JVM 调优、Redis 高级特性、MySQL 索引原理这些深层内容。同时开始训练系统设计题的思路。

### 3 年以上的高级工程师 / 架构师

这个阶段的核心已经不是知识点，而是设计能力和架构视野。重点放在分布式系统设计、高并发场景实战、故障排查案例上。

面试中更多是「如果你来设计，你会怎么做」这类开放性问题。需要通过大量的案例积累，形成自己的设计框架。

## 写在最后

面试不是一场考试，而是一场对话。

面试官想看到的不是你背了多少答案，而是你解决问题的思路、你思考问题的方式、你面对追问时的反应。知识点是基础，但只有知识点是不够的。

Guide 希望能做你的面试伴侣——不是替你上场，而是帮你把装备准备齐全，把基本功练扎实，让你在面试场上能自信地展示自己。

准备好了，就从现在开始。
