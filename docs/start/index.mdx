# 快速入门

欢迎来到 Java 面试指南。

如果你刚刚开始准备 Java 后端面试，或者想系统性地梳理自己的知识体系，这部分内容就是为你设计的。我不会一上来就甩给你一堆八股文，而是先帮你搞清楚：**你应该学什么、从哪开始、按照什么顺序学**。

磨刀不误砍柴工，先把方向定清楚，后面的路才会顺。

## 目录概述

本模块是整个指南的入口，核心目标只有一个：**让你知道自己在哪里，要去哪里，以及怎么去**。

这里不堆知识点，只解决一个根本问题——很多候选人学了很久，发现脑子里是一团浆糊，越学越焦虑。本质上不是不够努力，而是缺乏一张清晰的地图。

## 内容范围

本模块目前包含以下内容：

- **指南介绍**：[一份面向 Java 工程师的系统性知识图谱](/start/introduction) — 介绍整个指南的设计理念、知识模块划分、以及如何高效使用这份资料

后续内容将陆续补充，包括学习路径规划、知识点优先级排序、以及不同目标（阿里P5/P6、字节1-2/1-3、美团L7/L8）的差异化准备策略。

## 学习路径指引

:::tip 💡
如果你时间充裕，建议按以下顺序学习：
1. 先读[指南介绍](/start/introduction)，搞清楚知识体系全貌
2. 根据自己的目标职级，明确优先级
3. 按模块逐个突破，不要跳着学
:::

如果你时间紧张，可以直接跳到自己最薄弱的核心模块——比如[JVM](/java/jvm/heap)、[并发编程](/java/collection/arraylist)、[MySQL](/database/mysql/)，哪里不会补哪里。但切记不要只背答案，要理解原理和实战场景。
