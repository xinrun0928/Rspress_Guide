# 快速上手

想象一下：你刚克隆完项目仓库，打开 `package.json`，看到一堆陌生的依赖和 scripts。`dev`、`build`、`preview`……这些命令分别做什么？配置改了一行，整个站点就不一样了，这背后的逻辑是什么？

别急，这套文档就是为你准备的。我们从零开始，一步步把「不知道从哪入手」变成「胸有成竹」。

## 模块内容

| [Introduction](./introduction) | 先搞懂我们在做什么，再动手 |

## 核心公式

```bash
# 开发调试
pnpm dev

# 构建生产版本
pnpm build

# 本地预览构建结果
pnpm preview
```

> 记住这三个命令，就足够应对日常 90% 的场景了。
