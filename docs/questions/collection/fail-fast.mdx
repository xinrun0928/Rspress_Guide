# fail-fast 与 fail-safe 机制

**目标级别**：P5 / P6

---

## 快速自测

面试官问：「fail-fast 和 fail-safe 有什么区别？ConcurrentHashMap 迭代时删除元素会抛异常吗？」

---

## 一、核心问题

### 🔴 什么是 fail-fast？

**快速失败**：检测到并发修改，抛出 `ConcurrentModificationException`

```java
// fail-fast 示例
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");

for (String s : list) {  // for-each 底层是 Iterator
    list.remove(s);  // 抛出 ConcurrentModificationException
}
```

### 🔴 什么是 fail-safe？

**安全失败**：对集合的复制进行遍历，修改不影响遍历

```java
// fail-safe 示例
CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
list.add("A");
list.add("B");

for (String s : list) {
    list.remove(s);  // 不会抛异常
}
```

---

## 二、fail-fast 原理

### 💡 modCount 和 expectedModCount

```java
// ArrayList 中的字段
protected transient int modCount = 0;  // 结构性修改次数

// Iterator 中的字段
int expectedModCount = modCount;  // 创建时记录

// 检查
final void checkForComodification() {
    if (modCount != expectedModCount)
        throw new ConcurrentModificationException();
}
```

## 三、fail-safe 原理

### 💡 CopyOnWriteArrayList 的快照

```java
// CopyOnWriteArrayList 迭代器
static final class COWIterator<E> implements ListIterator<E> {
    private final Object[] snapshot;  // 数组快照
    
    private COWIterator(Object[] elements, int cursor) {
        this.snapshot = elements;  // 直接引用当前数组
        this.cursor = cursor;
    }
    
    // 无 checkForComodification，因为是快照
    public E next() {
        return (E) snapshot[cursor++];
    }
}
```

## 四、对比表格

| 维度 | fail-fast | fail-safe |
|------|----------|-----------|
| 实现方式 | 检查 modCount | 复制集合快照 |
| 修改影响 | 抛异常 | 不影响遍历 |
| 内存开销 | 无 | 有（复制集合） |
| 遍历结果 | 可能过时 | 总是反映创建时的状态 |
| 代表实现 | ArrayList、HashMap | CopyOnWriteArrayList |
| 并发修改检测 | ✅ | ❌（不检测） |

---

## 五、哪些集合是 fail-fast？

### 🔴 fail-fast 集合

```java
// ArrayList - ✅ fail-fast
List<String> list = new ArrayList<>();

// LinkedList - ✅ fail-fast
List<String> linked = new LinkedList<>();

// HashMap - ✅ fail-fast
Map<String, Integer> map = new HashMap<>();

// TreeMap - ✅ fail-fast
Map<String, Integer> tree = new TreeMap<>();

// HashSet、TreeSet - ✅ fail-fast
Set<String> set = new HashSet<>();
```

### 🔴 fail-safe 集合

```java
// CopyOnWriteArrayList - ✅ fail-safe
List<String> cowList = new CopyOnWriteArrayList<>();

// CopyOnWriteArraySet - ✅ fail-safe
Set<String> cowSet = new CopyOnWriteArraySet<>();

// ConcurrentHashMap - ✅ 部分 fail-safe
Map<String, Integer> chm = new ConcurrentHashMap<>();
```

---

## 六、ConcurrentHashMap 的特殊性

### 💡 ConcurrentHashMap 是弱一致性

```java
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();
map.put("A", 1);
map.put("B", 2);

// ConcurrentHashMap 的迭代器不是快照
// 但不会抛 ConcurrentModificationException
for (String key : map.keySet()) {
    map.remove(key);  // 不会抛异常
    // 但遍历结果可能不完整
}
```

**原因**：`ConcurrentHashMap` 的迭代器不是快照，但迭代器的遍历不会因为其他线程的修改而抛异常。

## 七、面试题精讲

### 🔴 第一层：fail-fast 和 fail-safe 有什么区别？

> **参考答案**：
>
> 主要区别：
> 1. **fail-fast**：检测到并发修改，抛出 ConcurrentModificationException
> 2. **fail-safe**：对集合的快照进行遍历，修改不影响遍历
>
> 实现原理：
> - fail-fast 用 modCount 检测结构性修改
> - fail-safe 复制集合或使用并发容器

### 🟡 第二层：ConcurrentHashMap 迭代时删除元素会抛异常吗？

> **参考答案**：
>
> 不会抛异常：
> 1. ConcurrentHashMap 的迭代器不是 fail-fast
> 2. 它不会检测并发修改
> 3. 但遍历结果可能不完整（因为其他线程可能删除了元素）

### ⚠️ 面试官挖坑点

| 陷阱 | 错误回答 | 正确回答 |
|------|---------|----------|
| 「ConcurrentHashMap 是 fail-safe」 | 部分正确但不准确 | 是弱一致性的，不抛异常 |
| 「fail-safe 集合不需要同步」 | 忽略了快照问题 | 仍需同步，否则可能读到过期数据 |
| 「HashMap 的迭代是线程安全的」 | 搞混了 | HashMap 是 fail-fast，并发修改会抛异常 |

---

## 八、总结

**fail-fast vs fail-safe 核心要点**：

| 特性 | fail-fast | fail-safe |
|------|----------|-----------|
| 检测并发修改 | ✅ | ❌ |
| 抛异常 | ConcurrentModificationException | 不抛异常 |
| 遍历结果 | 可能不完整 | 快照 |
| 内存开销 | 无 | 有 |
| 代表 | ArrayList、HashMap | CopyOnWriteArrayList |
