# Vector 与 ArrayList 区别

**目标级别**：P5 / P6

---

## 快速自测

面试官问：「Vector 和 ArrayList 有什么区别？为什么现在很少用 Vector？」

---

## 一、核心问题

### 🔴 Vector 和 ArrayList 有什么区别？

| 对比维度 | Vector | ArrayList |
|---------|--------|-----------|
| 线程安全 | ✅（synchronized） | ❌ |
| 同步方式 | 每个方法加锁 | 不加锁 |
| 扩容倍数 | 2 倍 | 1.5 倍 |
| 初始容量 | 10 | 0（懒加载） |
| 迭代器 | fail-fast | fail-fast |
| 性能 | 慢（同步开销） | 快 |
| 推荐程度 | ❌ | ✅ |

---

## 二、源码对比

### ArrayList 扩容

```java
// ArrayList: 1.5 倍
private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1);  // 1.5 倍
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    elementData = Arrays.copyOf(elementData, newCapacity);
}
```

### Vector 扩容

```java
// Vector: 2 倍（默认）
private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (capacityIncrement > 0 ?
                capacityIncrement : oldCapacity);  // 2 倍或自定义增量
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    elementData = Arrays.copyOf(elementData, newCapacity);
}
```

### 💡 capacityIncrement

```java
// Vector 可以指定扩容增量
Vector<Integer> v = new Vector<>(10, 5);
// 每次扩容增加 5 个元素，而不是翻倍
```

## 三、线程安全对比

### Vector 的 synchronized

```java
// Vector 几乎所有方法都加锁
public synchronized boolean add(E e) {
    modCount++;
    ensureCapacityHelper(elementCount + 1);
    elementData[elementCount++] = e;
    return true;
}

public synchronized E get(int index) {
    if (index >= elementCount)
        throw new ArrayIndexOutOfBoundsException(index);
    return elementData(index);
}
```

### ArrayList 无锁

```java
// ArrayList 没有任何锁
public boolean add(E e) {
    ensureCapacityInternal(size + 1);
    elementData[size++] = e;
    return true;
}

public E get(int index) {
    rangeCheck(index);
    return elementData(index);
}
```

## 四、性能对比

### ⚠️ 为什么 Vector 性能差？

```java
// 单线程场景：Vector 的锁是多余的
Vector<String> v = new Vector<>();  // 不需要线程安全
v.add("A");
v.add("B");

// 多线程场景：应该用 ConcurrentHashMap/CopyOnWriteArrayList
ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
```

| 场景 | Vector | ArrayList |
|------|--------|-----------|
| 单线程 | 多余的同步开销 | 性能好 |
| 多线程 | 线程安全但性能差 | 不安全 |

---

## 五、替代方案

### 💡 多线程场景用什么？

| 需求 | 推荐方案 |
|------|---------|
| 高并发 Map | `ConcurrentHashMap` |
| 高并发 List | `CopyOnWriteArrayList` |
| 高并发 Queue | `ConcurrentLinkedQueue` |
| 需要 List + 同步 | `Collections.synchronizedList(new ArrayList<>())` |

## 六、面试题精讲

### 🔴 第一层：Vector 和 ArrayList 有什么区别？

> **参考答案**：
>
> 主要区别：
> 1. **线程安全**：Vector 是线程安全的（所有方法 synchronized），ArrayList 不是
> 2. **扩容倍数**：Vector 默认 2 倍，ArrayList 是 1.5 倍
> 3. **初始容量**：Vector 直接分配 10，ArrayList 懒加载
> 4. **性能**：Vector 因为同步开销大，性能差
> 5. **推荐**：现在很少用 Vector，推荐用 ArrayList

### ⚠️ 面试官挖坑点

| 陷阱 | 错误回答 | 正确回答 |
|------|---------|----------|
| 「Vector 更好因为线程安全」 | 不了解同步开销 | 线程安全但性能差 |
| 「ArrayList 不安全不能用」 | 不了解替代方案 | 应该根据场景选择合适的并发容器 |
| 「Vector 可以完全替代 ArrayList」 | 忽略了性能问题 | 在单线程场景下 Vector 性能更差 |

## 七、总结

**Vector vs ArrayList 核心要点**：

| 维度 | Vector | ArrayList |
|------|--------|-----------|
| 线程安全 | ✅ | ❌ |
| 扩容倍数 | 2 | 1.5 |
| 性能 | 慢 | 快 |
| 推荐场景 | 无 | 单线程首选 |
| 并发场景 | ❌ | ❌（用并发容器） |
