# 集合遍历时删除元素

**目标级别**：P5 / P6

---

## 快速自测

面试官问：「在遍历 ArrayList 时删除元素会出什么问题？怎么解决？」

---

## 一、核心问题

### 🔴 遍历时删除有什么问题？

```java
// 错误写法：ConcurrentModificationException
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

for (String s : list) {
    if ("B".equals(s)) {
        list.remove(s);  // ConcurrentModificationException!
    }
}
```

### ⚠️ 为什么会抛异常？

```java
// for-each 底层用的是 iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if ("B".equals(s)) {
        list.remove(s);  // 直接用 list.remove()，绕过了 iterator
        // modCount++，但 expectedModCount 没变
        // 迭代器检测到不一致，抛出异常
    }
}
```

---

## 二、正确删除方式

### 🔴 使用 Iterator.remove()

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if ("B".equals(s)) {
        it.remove();  // 正确：使用迭代器的 remove
    }
}
```

**为什么这样可以？**

```java
// Iterator.remove 会同步 expectedModCount
public void remove() {
    if (lastRet < 0)
        throw new IllegalStateException();
    checkForComodification();
    try {
        ArrayList.this.remove(lastRet);
        cursor = lastRet;
        lastRet = -1;
        expectedModCount = modCount;  // 同步
    }
}
```

### 💡 ListIterator.remove() 或 add()

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

ListIterator<String> it = list.listIterator();
while (it.hasNext()) {
    if ("B".equals(it.next())) {
        it.remove();  // 删除当前元素
    }
}

// 还可以用 add、set 等操作
```

## 三、倒序删除

### 💡 倒序遍历删除

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

// 倒序遍历，从后往前删
for (int i = list.size() - 1; i >= 0; i--) {
    if ("B".equals(list.get(i))) {
        list.remove(i);
    }
}
```

**为什么可以？**
- 倒序删除时，删除后面的元素不会影响前面元素的索引
- 不需要迭代器

---

## 四、removeIf（JDK 9+）

### 💡 现代写法

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

// JDK 9+ 推荐
list.removeIf(s -> "B".equals(s));
```

## 五、Map 的遍历删除

### 🔴 Map 遍历删除

```java
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);
map.put("B", 2);
map.put("C", 3);

// 错误写法
for (String key : map.keySet()) {
    if ("B".equals(key)) {
        map.remove(key);  // 可能 ConcurrentModificationException
    }
}

// 正确写法：使用 Iterator
Iterator<String> it = map.keySet().iterator();
while (it.hasNext()) {
    String key = it.next();
    if ("B".equals(key)) {
        it.remove();  // 使用 Iterator.remove
    }
}
```

## 六、面试题精讲

### 🔴 第一层：遍历时删除元素会出什么问题？

> **参考答案**：
>
> 会抛出 `ConcurrentModificationException`：
> 1. for-each 遍历底层用的是 Iterator
> 2. Iterator 维护了 expectedModCount
> 3. 直接调用 list.remove() 会增加 modCount
> 4. 迭代器检测到 modCount != expectedModCount，抛出异常
>
> 解决方案是使用 Iterator.remove()。

### 🟡 第二层：为什么 Iterator.remove() 可以？

> **参考答案**：
>
> Iterator.remove() 会同步 expectedModCount：
> 1. 在删除元素后，会执行 `expectedModCount = modCount`
> 2. 这样下次检查时两者相等，不会抛异常
> 3. 这是 fail-fast 机制的正确使用方式

### ⚠️ 面试官挖坑点

| 陷阱 | 错误回答 | 正确回答 |
|------|---------|----------|
| 「遍历时不能删除」 | 不了解正确方式 | 可以删除，但要使用 Iterator.remove() |
| 「removeIf 是最好的方法」 | 不了解兼容性 | JDK 9+ 才能用 |
| 「倒序遍历删除不需要迭代器」 | 没错但有风险 | 倒序删除确实可以，但要注意索引变化 |

## 七、总结

**集合遍历删除核心要点**：

1. **不能用 for-each + list.remove()**
2. **用 Iterator.remove()** 正确删除
3. **倒序删除** 可以避免索引问题
4. **removeIf** 是 JDK 9+ 的现代写法
