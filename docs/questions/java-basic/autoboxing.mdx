## 场景切入

面试官问：「`List<int>` 能编译通过吗？」你说「能」——然后面试官说「不能」。你愣住了。

Java 的泛型不支持基本类型，必须使用包装类。而自动装箱就是基本类型和包装类之间的自动转换。

## 一、基本概念

### 1.1 装箱 vs 拆箱

| 操作 | 说明 | 示例 |
|------|------|------|
| 装箱 | 基本类型 → 包装类 | int → Integer |
| 拆箱 | 包装类 → 基本类型 | Integer → int |

### 1.2 语法示例

```java
// 装箱（Boxing）
int primitive = 10;
Integer boxed = primitive;  // 自动装箱

// 拆箱（Unboxing）
Integer boxed2 = 20;
int primitive2 = boxed2;  // 自动拆箱
```

## 二、自动装箱的触发场景

### 2.1 赋值操作

```java
// 基本类型赋值给包装类
Integer a = 10;  // [!code highlight] 自动装箱

// 包装类赋值给基本类型
int b = new Integer(20);  // [!code highlight] 自动拆箱
```

### 2.2 方法参数

```java
public void process(Integer value) { }

int num = 100;
process(num);  // [!code highlight] 自动装箱

public int calculate(int value) { }

Integer boxed = 200;
int result = calculate(boxed);  // [!code highlight] 自动拆箱
```

### 2.3 集合操作

```java
List<Integer> list = new ArrayList<>();

// [!code highlight] 自动装箱
list.add(1);      // int -> Integer
list.add(2);      // int -> Integer
list.add(3);      // int -> Integer

// [!code highlight] 自动拆箱
int value = list.get(0);  // Integer -> int
```

### 2.4 运算操作

```java
Integer a = 10;
Integer b = 20;

// [!code highlight] 自动拆箱 -> 运算 -> 自动装箱
a + b;        // a.intValue() + b.intValue() = 30
a - b;        // a.intValue() - b.intValue() = -10
a * b;        // a.intValue() * b.intValue() = 200
a / b;        // a.intValue() / b.intValue() = 0
a++;          // a.intValue() + 1
```

## 三、自动装箱的原理

### 3.1 编译器处理

```java
// 源码
Integer a = 10;

// 编译后（伪代码）
Integer a = Integer.valueOf(10);  // [!code highlight]
```

### 3.2 自动拆箱的原理

```java
// 源码
int b = a;

// 编译后（伪代码）
int b = a.intValue();  // [!code highlight]
```

### 3.3 Integer.valueOf()

```java
// JDK 源码
public static Integer valueOf(int i) {
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);
}
```

:::tip 缓存优化
-128 到 127 之间的整数值会被缓存，所以这个范围内的自动装箱**不会创建新对象**。
:::

## 四、性能问题

### 4.1 性能开销

```java
// 反面案例：循环中的自动装箱
long start = System.currentTimeMillis();
Long sum = 0L;  // [!code warning] 使用 Long 而非 long

for (int i = 0; i < 10000000; i++) {
    sum += i;  // [!code warning] 每次都要装箱拆箱
}

System.out.println(sum);
System.out.println(System.currentTimeMillis() - start);  // 耗时明显

// 优化：使用基本类型
long start2 = System.currentTimeMillis();
long sum2 = 0L;  // [!code highlight]

for (int i = 0; i < 10000000; i++) {
    sum2 += i;  // [!code highlight] 无装箱拆箱
}

System.out.println(sum2);
System.out.println(System.currentTimeMillis() - start2);  // 快速
```

### 4.2 性能对比

| 操作 | 耗时（相对） | 说明 |
|------|--------------|------|
| long sum = 0L; sum += i | 1x | 基本类型，最快 |
| Long sum = 0L; sum += i | 10-50x | 每次装箱拆箱 |

:::warning 性能陷阱
在循环中使用包装类进行算术运算，会产生大量装箱拆箱操作，严重影响性能。
**高频循环中应使用基本类型。**
:::

## 五、常见错误与陷阱

### 5.1 空指针异常

```java
Integer a = null;
int b = a;  // [!code error] NullPointerException！

// 拆箱时自动调用 intValue()，null 调用方法会抛 NPE
```

:::warning 拆箱的 NPE
拆箱操作可能导致 NullPointerException，因为包装类是对象，可能为 null。
:::

### 5.2 自动装箱的 == 比较

```java
Integer a = 128;
Integer b = 128;
System.out.println(a == b);  // [!code warning] false！超出缓存范围

Integer c = 127;
Integer d = 127;
System.out.println(c == d);  // [!code highlight] true！缓存范围内
```

### 5.3 混合类型比较

```java
Integer a = 100;
int b = 100;
System.out.println(a == b);  // [!code highlight] true！a 自动拆箱

Integer c = 100;
Long d = 100L;
System.out.println(c == d);  // [!code warning] 编译错误！类型不同
```

## 七、最佳实践

### 7.1 循环中的性能优化

```java
// 错误：在循环中使用包装类
List<Integer> numbers = getNumbers();
long sum = 0L;
for (Integer num : numbers) {  // [!code warning]
    sum += num;  // [!code warning] 拆箱
}

// 正确：使用基本类型或增强 for 循环
long sum = 0L;
for (int num : numbers) {  // [!code highlight] 避免自动装箱
    sum += num;
}
```

### 7.2 方法参数优化

```java
// 错误
public static long sum(Integer... numbers) {  // [!code warning] 可变参数
    long sum = 0;
    for (Integer num : numbers) {
        sum += num;  // [!code warning]
    }
    return sum;
}

// 正确
public static long sum(int... numbers) {  // [!code highlight]
    long sum = 0;
    for (int num : numbers) {
        sum += num;
    }
    return sum;
}
```

## 八、加分回答

💡 **超出预期的深度**：

### 1. Integer 自动装箱字节码

```java
// 源码
Integer a = 10;

// 字节码（javap -c）
// iconst_10      // 将 10 压入栈
// invokestatic #2 // 调用 Integer.valueOf
// astore_1       // 存储到局部变量表
```

### 2. 基本类型 vs 包装类选择

| 场景 | 推荐类型 | 原因 |
|------|----------|------|
| 局部变量 | 基本类型 | 性能最优 |
| 方法参数 | 基本类型 | 性能最优 |
| 泛型类型参数 | 包装类 | 基本类型不支持泛型 |
| 表示「无值」 | 包装类 | 基本类型有默认值 |
| 高频循环 | 基本类型 | 避免装箱拆箱 |

### 3. 缓存池的源码

```java
// Boolean 缓存
public static Boolean valueOf(boolean b) {
    return b ? Boolean.TRUE : Boolean.FALSE;  // [!code highlight] 只缓存两个值
}

// Byte、Short、Long 缓存（类似）
// Character 缓存：0-127（ASCII）
```
