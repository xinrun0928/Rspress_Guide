## 场景切入

面试官问：「为什么要用 try-with-resources？」你说「因为会自动关闭资源」——然后面试官追问「close() 是在什么时候调用的？如果 close() 抛出异常会发生什么？」你愣了一下。

这是 Java 7 引入的重要特性，背后涉及编译器的自动资源管理和异常抑制机制。

## 一、基本语法

### 1.1 标准写法

```java
// 旧写法：手动关闭
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("test.txt"));
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (br != null) {
        try {
            br.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

// [!code highlight] 新写法：try-with-resources
try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();
}
```

### 1.2 多个资源

```java
// 多个资源同时管理
try (
    FileInputStream fis = new FileInputStream("input.txt");
    FileOutputStream fos = new FileOutputStream("output.txt")
) {
    byte[] buffer = new byte[1024];
    int len;
    while ((len = fis.read(buffer)) != -1) {
        fos.write(buffer, 0, len);
    }
}
```

## 二、AutoCloseable 接口

### 2.1 接口定义

```java
// JDK 源码：AutoCloseable.java
public interface AutoCloseable {
    /**
     * 关闭此资源，放弃任何底层资源。
     */
    void close() throws Exception;
}

// JDK 源码：Closeable.java
public interface Closeable extends AutoCloseable {
    void close() throws IOException;  // [!code highlight] 声明为 IOException
}
```

### 2.2 实现了 AutoCloseable 的类

| 类型 | 典型类 |
|------|--------|
| IO 流 | InputStream, OutputStream, Reader, Writer |
| NIO | Channel, Buffer |
| 数据库 | Connection, Statement, ResultSet |
| 其他 | Lock, ExecutorService |

### 2.3 自定义资源

```java
public class DatabaseConnection implements AutoCloseable {
    private Connection conn;

    public DatabaseConnection() throws SQLException {
        this.conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/test");
    }

    @Override
    public void close() {
        if (conn != null) {
            try {
                conn.close();
            } catch (SQLException e) {
                // 记录日志
            }
        }
    }
}

// 使用
try (DatabaseConnection db = new DatabaseConnection()) {
    // 使用数据库连接
} catch (SQLException e) {
    e.printStackTrace();
}  // [!code highlight] 自动调用 close()
```

## 三、编译器转换原理

### 3.1 转换过程

```mermaid
graph LR
    A[源码：try-with-resources] --> B[编译器转换]
    B --> C[生成 finally 块]
    B --> D[生成 close() 调用]
    B --> E[异常抑制处理]
```

### 3.2 编译器生成代码

```java
// 源码
try (Resource r = createResource()) {
    r.doSomething();
}

// 编译器转换后的伪代码
Resource r = createResource();
Throwable primaryExc = null;
try {
    r.doSomething();
} catch (Throwable t) {
    primaryExc = t;
    throw t;
} finally {
    if (r != null) {
        if (primaryExc != null) {
            try {
                r.close();
            } catch (Throwable suppressedExc) {
                primaryExc.addSuppressed(suppressedExc);  // [!code highlight] 异常抑制
            }
        } else {
            r.close();
        }
    }
}
```

## 四、异常抑制机制

### 4.1 基本原理

```java
try (MyResource r = new MyResource()) {
    r.doSomething();  // [!code warning] 抛出异常 A
    r.close();       // [!code warning] 也抛出异常 B
} catch (Exception e) {
    // [!code warning] 只抛出 B，A 被抑制
    e.getSuppressed();  // 获取被抑制的异常
}
```

### 4.2 suppressed 异常演示

```java
public class SuppressedDemo {
    static class MyResource implements AutoCloseable {
        @Override
        public void close() throws Exception {
            throw new RuntimeException("close exception");
        }
    }

    public static void main(String[] args) {
        try (MyResource r = new MyResource()) {
            throw new RuntimeException("use exception");
        } catch (Exception e) {
            System.out.println("Primary: " + e.getMessage());
            System.out.println("Suppressed: " + e.getSuppressed()[0].getMessage());
        }
    }
}

// 输出：
// Primary: use exception
// Suppressed: close exception
```

## 六、常见错误与陷阱

### ⚠️ 陷阱 1：资源声明在 try 外部

```java
Connection conn = null;
try (conn = DriverManager.getConnection(url)) {  // [!code warning] 语法错误
    // ...
}

// 正确写法
try (Connection conn = DriverManager.getConnection(url)) {  // [!code highlight]
    // ...
}
```

### ⚠️ 陷阱 2：忘记 AutoCloseable

```java
// 错误：自定义资源没有实现 AutoCloseable
public class MyResource {
    public void close() { }  // [!code warning] 不是 AutoCloseable，不会自动关闭
}

// 正确
public class MyResource implements AutoCloseable {  // [!code highlight]
    @Override
    public void close() { }
}
```

### ⚠️ 陷阱 3：变量遮蔽

```java
try (BufferedReader br = new BufferedReader(new FileReader("f1"))) {
    BufferedReader br = new BufferedReader(new FileReader("f2"));  // [!code error] 变量名冲突
}
```

## 七、最佳实践

### 7.1 资源关闭顺序

```java
// 如果有依赖关系，正确顺序是先打开后关闭
try (
    Connection conn = getConnection();  // [!code highlight]
    Statement stmt = conn.createStatement();  // [!code highlight] 依赖 conn
) {
    // 使用资源
}  // [!code highlight] 按相反顺序关闭：stmt 先关，conn 后关
```

### 7.2 带 catch 的 try-with-resources

```java
// 可以单独使用 try-with-resources
try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    br.readLine();
}  // [!code highlight] 自动关闭，不需要 catch
```

## 八、加分回答

💡 **超出预期的深度**：

### 1. Java 9 的改进： effectively-final 变量

```java
// Java 9+：可以在 try 外部声明
BufferedReader br = new BufferedReader(new FileReader("test.txt"));
try (br) {  // [!code highlight] 只需要变量是 effectively-final
    br.readLine();
}
```

### 2. 异常抑制的替代方案

```java
// 在 close() 中记录异常而不抛出
public void close() {
    try {
        connection.close();
    } catch (Exception e) {
        log.error("关闭连接失败", e);  // [!code highlight] 记录日志，不抛出
    }
}
```

### 3. 资源管理接口：Closeable vs Flushable

```java
// Flushable：刷新缓冲区
public interface Flushable {
    void flush() throws IOException;
}

// OutputStream 同时实现了 Closeable 和 Flushable
public abstract class OutputStream implements Closeable, Flushable {
    @Override
    public void close() throws IOException {
        flush();  // [!code highlight] close 时自动 flush
    }
}
```

## 九、扩展思考

面试结束前的延伸问题：

1. **try-with-resources 能完全替代 finally 吗？** —— 不能，用于资源管理，finally 仍用于通用清理
2. **close() 和 flush() 的区别？** —— flush 是刷新缓冲区，close 是关闭资源
3. **Java 9 的 try-with-resources 有什么改进？** —— 支持 outside-of-try 变量声明
