## 场景切入

面试官问：「Java 8 的 default 方法有什么用？」你说「可以在接口里写实现了」——然后面试官追问「如果两个接口都有同名的 default 方法，一个类同时实现这两个接口会发生什么？」你愣住了。

default 方法是 Java 8 最重大的变化之一，它让接口有了「部分实现」的能力，但也带来了新的复杂性。

## 一、为什么需要 default 方法？

### 1.1 问题背景

```java
// Java 8 之前：向接口添加新方法会导致所有实现类报错
interface List<E> {
    void add(E e);  // 原有方法
    // void sort(Comparator<E> c);  // [!code warning] 添加这个方法，所有实现类都要修改
}
```

### 1.2 解决方案

```java
// Java 8+：添加 default 方法，提供默认实现
interface List<E> {
    void add(E e);

    // [!code highlight] default 方法：现有实现类无需修改
    default void sort(Comparator<E> c) {
        // 默认实现：使用 Collections.sort
        Collections.sort((List<E>) this, c);
    }
}
```

:::tip default 方法的价值
在**不破坏现有实现**的情况下，向接口添加新方法。
:::

## 二、基本语法

### 2.1 定义 default 方法

```java
interface MyInterface {
    // 普通抽象方法
    void doSomething();

    // [!code highlight] default 方法：带实现
    default void doSomethingElse() {
        System.out.println("默认实现");
    }

    // default 方法可以调用其他方法
    default void log(String message) {
        // [!code highlight] private 方法（Java 9+）抽取公共逻辑
        writeLog(prefix() + message);
    }

    // [!code highlight] static 方法：属于接口本身
    static MyInterface create() {
        return new MyInterface() {
            @Override
            public void doSomething() {
                System.out.println("实现");
            }
        };
    }
}
```

### 2.2 实现类使用 default 方法

```java
class MyImpl implements MyInterface {
    @Override
    public void doSomething() {
        System.out.println("具体实现");
    }
    // [!code highlight] doSomethingElse() 自动继承，无需实现
}

MyImpl obj = new MyImpl();
obj.doSomething();       // 具体实现
obj.doSomethingElse();   // 默认实现
```

## 三、多接口冲突问题

### 3.1 问题场景

```java
interface A {
    default void hello() {
        System.out.println("Hello from A");
    }
}

interface B {
    default void hello() {
        System.out.println("Hello from B");
    }
}

// [!code error] 编译错误：两个接口都有 hello 方法
class MyClass implements A, B {
    // 必须解决冲突
}
```

### 3.2 解决方案

```java
// 方案1：重写方法
class MyClass implements A, B {
    @Override
    public void hello() {
        System.out.println("Hello from MyClass");
    }
}

// 方案2：调用指定接口的默认实现
class MyClass implements A, B {
    @Override
    public void hello() {
        A.super.hello();  // [!code highlight] 调用 A 的 hello
    }
}

// 方案3：同时调用多个
class MyClass implements A, B {
    @Override
    public void hello() {
        System.out.println("Before");
        A.super.hello();  // [!code highlight]
        B.super.hello();  // [!code highlight]
        System.out.println("After");
    }
}
```

### 3.3 冲突解决规则

```mermaid
graph TD
    A[类实现多个接口] --> B{有同名 default 方法？}
    B -->|是| C[类必须重写]
    B -->|否| D[直接继承唯一 default]
    C --> E[可以调用指定接口的 default<br/>InterfaceName.super.methodName()]
```

:::warning 冲突解决优先级
1. 类自己重写的方法优先
2. 如果类不重写，选择「最具体」的 default
3. 如果两个接口都提供了 default，必须显式选择
:::

## 四、与抽象类的区别

### 4.1 对比表

| 特性 | default 方法 | 抽象类方法 |
|------|--------------|------------|
| 构造器 | ❌ 不能有 | ✅ 可以有 |
| 状态 | ❌ 不能有实例字段 | ✅ 可以有 |
| 访问修饰符 | 隐式 public | 可自定义 |
| 多继承 | ✅ 接口可以多继承 | ❌ 类只能单继承 |

### 4.2 为什么 default 不能有状态？

```java
// [!code error] 编译错误：接口不能有实例字段
interface MyInterface {
    // String name;  // [!code error]
    // private int count = 0;  // [!code error]

    // 但可以有常量
    int CONSTANT = 100;  // [!code highlight] public static final
}
```

:::tip 设计原因
接口保持「纯粹」：只定义行为契约，不持有状态。这符合接口的原始设计意图。
:::

## 六、常见错误与陷阱

### ⚠️ 陷阱 1：忘记 default 方法的可见性

```java
interface MyInterface {
    // [!code warning] default 方法隐式是 public，不能用其他修饰符
    // private default void method() { }  // [!code error]
    // protected default void method() { }  // [!code error]
}
```

### ⚠️ 陷阱 2：diamond 问题

```java
interface A {
    default void method() { System.out.println("A"); }
}

interface B extends A {  // [!code highlight] B 继承 A 的 default
    // 默认继承 A.method()
}

interface C extends A {
    // 默认继承 A.method()
}

class MyClass implements B, C {
    // [!code highlight] 不冲突！只有一个 A.method()
    // 不需要重写
}

class MyClass2 implements B, C {
    @Override
    public void method() {
        // [!code highlight] 如果需要，可以显式调用 A 的方法
        A.super.method();
    }
}
```

### ⚠️ 陷阱 3：default 方法与抽象方法冲突

```java
interface A {
    default void method() { }
}

interface B extends A {
    // [!code warning] B 可以将 A 的 default 变为抽象
    void method();  // [!code highlight] 强制实现类重写
}

class MyClass implements B {
    @Override
    public void method() {
        // 必须实现
    }
}
```

## 七、实战应用

### 7.1 增强现有接口

```java
// Java 8 为 Collection 接口添加了大量 default 方法
interface Collection<E> {
    // 新增的 default 方法
    default Stream<E> stream() {  // [!code highlight]
        return StreamSupport.stream(spliterator(), false);
    }

    default Stream<E> parallelStream() {  // [!code highlight]
        return StreamSupport.stream(spliterator(), true);
    }

    default void forEach(Consumer<? super E> action) {  // [!code highlight]
        Objects.requireNonNull(action);
        for (E e : this) {
            action.accept(e);
        }
    }

    // 原有抽象方法
    Iterator<E> iterator();
    int size();
}
```

### 7.2 接口的多实现场景

```java
// 场景：需要在类中组合多个行为
interface Walkable { default void walk() { System.out.println("Walk"); } }
interface Runnable { default void run() { System.out.println("Run"); } }
interface Flyable { default void fly() { System.out.println("Fly"); } }

class Robot implements Walkable, Runnable {
    // [!code highlight] 组合多个接口的行为
}

Robot robot = new Robot();
robot.walk();  // Walk
robot.run();   // Run
// robot.fly();  // [!code error] Robot 没有实现 Flyable
```

### 1. Collection.sort 的实现

```java
// Java 8 之前
public interface List<E> {
    // 实现类必须提供 sort 方法
    void sort(Comparator<E> c);
}

// Java 8 之后
public interface List<E> {
    default void sort(Comparator<E> c) {
        // [!code highlight] 提供默认实现
        Collections.sort(this, c);
    }
}

// ArrayList 自动获得 sort 功能，无需修改
```
