## 一、注解的基本概念

### 1.1 什么是注解

> 注解（Annotation）是 JDK 5 引入的元数据机制，用于为代码添加额外信息。

```java
// 没有注解
public class UserService {
    public void save(User user) { }
}

// 有注解
public class UserService {
    @Override  // [!code highlight] 重写标识
    public void save(User user) {
        // ...
    }
}
```

### 1.2 注解与注释的区别

| 特性 | 注解 @ | 注释 // |
|------|--------|---------|
| 作用对象 | 代码、类、方法、字段 | 仅人类可读 |
| 编译器处理 | 可被编译器或注解处理器处理 | 完全被忽略 |
| 运行时 | 可保留到运行时读取 | 不存在 |
| 功能 | 提供元数据、生成代码、编译检查 | 代码说明 |

## 二、Java 内置注解

### 2.1 Java 预定义注解

| 注解 | 作用 | 位置 |
|------|------|------|
| @Override | 标记重写方法 | 方法 |
| @Deprecated | 标记过时 | 类、方法、字段 |
| @SuppressWarnings | 抑制编译器警告 | 类、方法、字段 |
| @SafeVarargs | 抑制 varargs 警告 | 方法、构造器 |
| @FunctionalInterface | 标记函数式接口 | 接口 |
| @Native | 标记 native 方法的字段 | 字段 |

### 2.2 使用示例

```java
@Deprecated  // [!code warning] 该类已过时
class OldService {
    @Deprecated  // [!code warning] 该方法已过时
    public void oldMethod() { }

    @SuppressWarnings(value = {"unchecked", "rawtypes"})
    public void rawTypes() {
        List list = new ArrayList();  // [!code highlight] raw type
        list.add("test");
    }
}
```

## 三、元注解（Meta-Annotation）

### 3.1 元注解定义

> 元注解是用于标注注解的注解。

### 3.2 四个核心元注解

```mermaid
graph TD
    A[元注解] --> B[@Target<br/>作用位置]
    A --> C[@Retention<br/>保留策略]
    A --> D[@Documented<br/>文档化]
    A --> E[@Inherited<br/>继承性]
```

### 3.3 @Target：标注作用位置

```java
// @Target 源码
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)  // [!code highlight] 可以标注在 TYPE 上
public @interface Component {
    String value() default "";
}

// ElementType 枚举值
ElementType.TYPE              // 类、接口、枚举
ElementType.FIELD             // 字段（包括枚举常量）
ElementType.METHOD            // 方法
ElementType.PARAMETER         // 方法参数
ElementType.CONSTRUCTOR       // 构造器
ElementType.LOCAL_VARIABLE    // 局部变量
ElementType.ANNOTATION_TYPE   // 注解类型
ElementType.PACKAGE           // 包
ElementType.TYPE_PARAMETER    // 类型参数（Java 8+）
ElementType.TYPE_USE          // 任何类型使用位置（Java 8+）
```

### 3.4 @Retention：标注保留策略

```java
// @Retention 源码
@Documented
@Retention(RetentionPolicy.RUNTIME)  // [!code highlight] 保留到运行时
@Target(ElementType.ANNOTATION_TYPE)
public @interface Retention {
    RetentionPolicy value();
}

// RetentionPolicy 枚举值
RetentionPolicy.SOURCE      // 仅在源码保留，编译时被丢弃
RetentionPolicy.CLASS      // 编译时保留，运行时丢弃（默认）
RetentionPolicy.RUNTIME    // 运行时保留，可通过反射读取  // [!code highlight]
```

:::tip 保留策略选择
- `@Override` 使用 SOURCE，因为编译时检查即可
- Spring 的 `@Autowired` 使用 RUNTIME，因为需要反射读取
- Dagger 的注解使用 RUNTIME 或 CLASS，用于代码生成
:::

### 3.5 @Documented：文档化

```java
@Documented  // [!code highlight] 使用此注解的类在生成 Javadoc 时会包含该注解
public @interface MyAnnotation {
    String value();
}
```

### 3.6 @Inherited：继承性

```java
@Inherited  // [!code highlight] 子类会继承父类的此注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyInheritedAnnotation { }

@MyInheritedAnnotation  // [!code highlight] 父类标注
class Parent { }

// [!code highlight] 子类会自动继承该注解
class Child extends Parent { }
```

## 四、自定义注解

### 4.1 注解语法

```java
// 简单注解
public @interface Author {
    String name();  // [!code highlight] 注解属性，必须有 default 或无默认值
}

// 带默认值的注解
public @interface Log {
    String value() default "INFO";
    int level() default 1;
}

// 复杂注解
public @interface TestCase {
    String description() default "";  // [!code highlight] String 默认值必须是 ""
    String[] tags() default {};
    boolean enabled() default true;
}
```

### 4.2 使用示例

```java
@Author(name = "张三")  // [!code highlight] 完整写法
@Log(value = "DEBUG", level = 2)  // [!code highlight] 如果只有一个 value，可省略
class MyService { }

@Log  // [!code highlight] 如果只有 value，且使用默认值
class AnotherService { }
```

### 4.3 注解处理器示例

```java
// 获取类上的注解
public static void main(String[] args) throws Exception {
    Class<?> clazz = MyService.class;

    // 获取所有注解
    Annotation[] annotations = clazz.getAnnotations();
    for (Annotation a : annotations) {
        System.out.println(a.annotationType().getSimpleName());
    }

    // 获取指定类型的注解
    Author author = clazz.getAnnotation(Author.class);
    if (author != null) {
        System.out.println("Author: " + author.name());
    }
}
```

## 六、常见错误与陷阱

### ⚠️ 陷阱 1：注解属性类型错误

```java
// [!code error] 编译错误：注解属性类型只能是基本类型、String、枚举、Class、注解、数组
public @interface BadAnnotation {
    Date value();  // [!code error] Date 不是合法类型
    Object value();  // [!code error]
    List<String> value();  // [!code error]
}

// 正确
public @interface GoodAnnotation {
    String value();  // [!code highlight]
    Class<?> value();  // [!code highlight]
    MyEnum value();  // [!code highlight]
    int[] value();  // [!code highlight]
}
```

### ⚠️ 陷阱 2：忘记设置默认值

```java
public @interface MissingDefault {
    String value();  // [!code warning] 没有默认值
}

// [!code error] 使用时必须显式赋值
@MissingDefault("test")  // 必须写
class MyClass { }
```

### ⚠️ 陷阱 3：注解继承问题

```java
@Inherited  // [!code highlight] 只有类上的注解才会被继承
@Target(ElementType.METHOD)
public @interface MyAnnotation { }

class Parent {
    @MyAnnotation  // [!code warning] 方法上的注解不会被继承！
    public void method() { }
}

// [!code warning] Child 不会继承该注解！
class Child extends Parent { }
```

```java

    String name();
}

@Retention(RetentionPolicy.RUNTIME)
public @interface Authors {
    Author[] value();  // [!code highlight] 容器注解
}

// 使用

```
