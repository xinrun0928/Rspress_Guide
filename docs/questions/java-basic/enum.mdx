## 一、枚举的本质

### 1.1 简单枚举

```java
public enum Season {
    SPRING,  // 编译器生成：public static final Season SPRING = new Season();
    SUMMER,
    AUTUMN,
    WINTER
}
```

### 1.2 编译后的字节码

```java
// 编译后的等价代码（伪代码）
public final class Season extends Enum<Season> {
    // [!code highlight] 继承自 Enum
    public static final Season SPRING = new Season();
    public static final Season SUMMER = new Season();
    public static final Season AUTUMN = new Season();
    public static final Season WINTER = new Season();

    // 私有构造器
    private Season() {}

    // values() 方法
    public static Season[] values() {
        return new Season[]{SPRING, SUMMER, AUTUMN, WINTER};
    }

    // valueOf(String) 方法
    public static Season valueOf(String name) {
        return Enum.valueOf(Season.class, name);
    }
}
```

:::tip 枚举的本质
枚举编译后是一个继承自 `java.lang.Enum` 的类。枚举值是这个类的静态常量实例。
:::

## 二、Enum 类的方法

### 2.1 常用方法

| 方法 | 说明 | 示例 |
|------|------|------|
| name() | 返回枚举常量名称 | Season.SPRING.name() → "SPRING" |
| ordinal() | 返回枚举位置（从 0 开始） | Season.SPRING.ordinal() → 0 |
| valueOf() | 根据名称获取枚举值 | Season.valueOf("SPRING") → SPRING |
| values() | 返回所有枚举值数组 | Season.values() → Season[4] |
| compareTo() | 比较枚举位置 | Season.SPRING.compareTo(Season.SUMMER) → -1 |
| getDeclaringClass() | 返回枚举类型的 Class | Season.SPRING.getDeclaringClass() → Season.class |

### 2.2 示例

```java
public enum Status {
    CREATED,
    PROCESSING,
    COMPLETED,
    FAILED;

    public boolean isSuccess() {
        return this == COMPLETED;
    }

    public boolean isPending() {
        return this == CREATED || this == PROCESSING;
    }
}
```

## 三、枚举的高级特性

### 3.1 枚举实现接口

```java
interface Describable {
    String getDescription();
}

enum Operation implements Describable {
    ADD {
        @Override
        public String getDescription() {
            return "加法运算";
        }
    },
    SUBTRACT {
        @Override
        public String getDescription() {
            return "减法运算";
        }
    };

    // 通用方法
    public int apply(int a, int b) {
        switch (this) {
            case ADD: return a + b;
            case SUBTRACT: return a - b;
            default: throw new IllegalStateException();
        }
    }
}

// 使用
System.out.println(Operation.ADD.getDescription());  // 加法运算
```

### 3.2 枚举抽象方法

```java
enum Operation {
    ADD {
        @Override
        public int apply(int a, int b) {
            return a + b;
        }
    },
    SUBTRACT {
        @Override
        public int apply(int a, int b) {
            return a - b;
        }
    },
    MULTIPLY {
        @Override
        public int apply(int a, int b) {
            return a * b;
        }
    };

    // [!code highlight] 抽象方法，每个枚举值必须实现
    public abstract int apply(int a, int b);
}
```

### 3.3 带参数的枚举

```java
enum Color {
    RED("#FF0000"),
    GREEN("#00FF00"),
    BLUE("#0000FF");

    private final String hexCode;  // [!code highlight] 私有字段

    // [!code highlight] 私有构造器
    Color(String hexCode) {
        this.hexCode = hexCode;
    }

    public String getHexCode() {
        return hexCode;
    }
}

// 使用
System.out.println(Color.RED.getHexCode());  // #FF0000
```

## 四、枚举的实际应用

### 4.1 替代常量类

```java
// 旧写法：常量类
class OrderStatus {
    public static final int CREATED = 1;
    public static final int PAID = 2;
    public static final int SHIPPED = 3;
    public static final int COMPLETED = 4;
}

// [!code highlight] 新写法：枚举
enum OrderStatus {
    CREATED,
    PAID,
    SHIPPED,
    COMPLETED
}

// 优势：
// 1. 类型安全（不能使用其他整数）
// 2. 可以在 switch 中使用
// 3. 有命名空间
// 4. 可以添加方法
```

### 4.2 单例模式

```java
// [!code highlight] 最简洁的线程安全单例
public enum Singleton {
    INSTANCE;

    public void doSomething() {
        System.out.println("Singleton doing something");
    }
}

// 使用
Singleton.INSTANCE.doSomething();
```

:::tip 枚举单例的优势
1. 线程安全（JVM 保证）
2. 反射无法破坏（JVM 阻止）
3. 序列化安全（JVM 保证）
4. 代码简洁
:::

### 4.3 策略模式

```java
enum CalculateStrategy {
    PLUS {
        @Override
        public int calculate(int a, int b) { return a + b; }
    },
    MINUS {
        @Override
        public int calculate(int a, int b) { return a - b; }
    },
    MULTIPLY {
        @Override
        public int calculate(int a, int b) { return a * b; }
    };

    public abstract int calculate(int a, int b);
}

// 使用
int result = CalculateStrategy.PLUS.calculate(1, 2);  // 3
```

## 六、常见错误与陷阱

### ⚠️ 陷阱 1：使用 == 比较不同枚举类型

```java
enum A { X, Y }
enum B { X, Y }

A.X == B.X  // [!code error] 编译错误！不同类型不能比较
```

### ⚠️ 陷阱 2：switch 中的 case 顺序

```java
enum Status {
    CREATED, PROCESSING, COMPLETED
}

switch (status) {
    case COMPLETED:  // [!code warning] switch 用的 name()
        // ...
        break;
    // case Status.COMPLETED: // [!code error] 编译错误
}
```

### ⚠️ 陷阱 3：序列化问题

```java
// 枚举序列化是安全的
enum Status {
    CREATED
}

// 反序列化会返回同一个对象（通过 valueOf）
Status status = Status.valueOf("CREATED");
```

### 2. 枚举与数据库映射

```java
// 使用 @Enumerated
public enum OrderStatus {
    CREATED, PAID, SHIPPED, COMPLETED
}

@Entity
public class Order {
    @Enumerated(EnumType.STRING)  // [!code highlight] 存为字符串
    private OrderStatus status;
}
```

### 3. EnumSet 和 EnumMap

```java
// EnumSet：高效集合
EnumSet<Status> statuses = EnumSet.allOf(Status.class);
statuses.add(Status.CREATED);
statuses.remove(Status.PAID);

// EnumMap：高效映射
EnumMap<Status, String> map = new EnumMap<>(Status.class);
map.put(Status.CREATED, "已创建");
map.put(Status.PAID, "已支付");
```
