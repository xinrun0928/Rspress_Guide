## 一、为什么不能用 double

### 1.1 double 的精度问题

```java
double d1 = 0.1;
double d2 = 0.2;
System.out.println(d1 + d2);  // [!code warning] 0.30000000000000004

double amount = 1.0;
for (int i = 0; i < 10; i++) {
    amount -= 0.1;
}
System.out.println(amount);  // [!code warning] 0.10000000000000006
```

### 1.2 原因：二进制浮点数

```mermaid
graph LR
    A[十进制 0.1] --> B[二进制 0.000110011...]
    B --> C[无限循环]
    C --> D[截断存储]
    D --> E[精度丢失]
```

:::danger 金额计算禁区
任何涉及金额的计算，都**绝对不能**使用 double 或 float。精度丢失会导致严重的财务问题。
:::

## 二、BigDecimal 基本用法

### 2.1 创建方式对比

```java
// 方式1：String 构造器（推荐）
BigDecimal bd1 = new BigDecimal("0.1");  // [!code highlight] 精确

// 方式2：double 构造器（不推荐）
BigDecimal bd2 = new BigDecimal(0.1);  // [!code warning] 仍然是 0.1000000000000000055511151231257827021181583404541015625

// 方式3：valueOf（推荐）
BigDecimal bd3 = BigDecimal.valueOf(0.1);  // [!code highlight] 内部用 String 转换
```

:::warning double 构造器的坑
`new BigDecimal(double)` 会精确转换 double 的二进制表示，而不是你期望的十进制值。
**永远不要用 double 初始化 BigDecimal！**
:::

### 2.2 基本运算

```java
BigDecimal a = new BigDecimal("0.1");
BigDecimal b = new BigDecimal("0.2");

// 加法
BigDecimal sum = a.add(b);  // 0.3

// 减法
BigDecimal diff = a.subtract(b);  // -0.1

// 乘法
BigDecimal product = a.multiply(b);  // 0.02

// 除法（需要指定精度和舍入模式）
BigDecimal quotient = a.divide(b, 2, RoundingMode.HALF_UP);  // 0.50

// 比较
boolean isEqual = a.compareTo(b) == 0;  // [!code highlight] 不要用 equals
```

## 三、BigDecimal 的坑

### 3.1 坑一：equals 和 compareTo

```java
BigDecimal bd1 = new BigDecimal("0.0");
BigDecimal bd2 = new BigDecimal("0.00");
BigDecimal bd3 = new BigDecimal("0");

System.out.println(bd1.equals(bd2));  // [!code warning] false！精度不同
System.out.println(bd1.equals(bd3));  // [!code warning] false！

System.out.println(bd1.compareTo(bd2) == 0);  // [!code highlight] true
System.out.println(bd1.compareTo(bd3) == 0);  // [!code highlight] true
```

:::warning equals 比较精度
BigDecimal 的 equals 方法同时比较**值**和**精度**。`0.0` 和 `0.00` 不相等。
**比较数值大小必须使用 compareTo！**
:::

### 3.2 坑二：除法精度

```java
BigDecimal dividend = new BigDecimal("10");
BigDecimal divisor = new BigDecimal("3");

// [!code error] 抛出 ArithmeticException：无精确结果
BigDecimal quotient = dividend.divide(divisor);

// 必须指定精度和舍入模式
BigDecimal quotient = dividend.divide(divisor, 2, RoundingMode.HALF_UP);  // 3.33
```

### 3.3 坑三：除不尽的情况

```java
// 使用 MathContext 限制精度
BigDecimal quotient = dividend.divide(divisor, new MathContext(4, RoundingMode.HALF_UP));
```

### 3.4 坑四：不可变性

```java
BigDecimal amount = new BigDecimal("100");
// amount.add(new BigDecimal("50"));  // [!code warning] 不改变原对象！
// amount 仍然是 100

amount = amount.add(new BigDecimal("50"));  // [!code highlight] 重新赋值
System.out.println(amount);  // 150
```

## 四、正确的金额计算

### 4.1 工具类封装

```java
public class MoneyUtil {

    // 加法
    public static BigDecimal add(BigDecimal a, BigDecimal b) {
        return a.add(b);
    }

    // 减法
    public static BigDecimal subtract(BigDecimal a, BigDecimal b) {
        return a.subtract(b);
    }

    // 乘法（价格 * 数量）
    public static BigDecimal multiply(BigDecimal a, BigDecimal b) {
        return a.multiply(b);
    }

    // 除法（金额 / 数量 = 单价）
    public static BigDecimal divide(BigDecimal dividend, BigDecimal divisor, int scale) {
        return dividend.divide(divisor, scale, RoundingMode.HALF_UP);
    }

    // 格式化
    public static String format(BigDecimal amount) {
        return amount.setScale(2, RoundingMode.HALF_UP).toPlainString();
    }
}
```

### 4.2 使用示例

```java
public class Order {
    private BigDecimal price;
    private int quantity;

    public BigDecimal getTotalPrice() {
        // [!code highlight] 使用 multiply 计算总价
        return price.multiply(BigDecimal.valueOf(quantity));
    }

    public BigDecimal getDiscount(BigDecimal discountRate) {
        // [!code highlight] 使用 multiply 计算折扣
        return getTotalPrice().multiply(discountRate);
    }

    public BigDecimal getFinalPrice(BigDecimal discountRate) {
        return getTotalPrice().subtract(getDiscount(discountRate));
    }
}
```

## 六、常见错误与陷阱

### ⚠️ 陷阱 1：使用 double 初始化

```java
// [!code error] 错误
BigDecimal amount = new BigDecimal(0.1);

// [!code highlight] 正确
BigDecimal amount = new BigDecimal("0.1");
BigDecimal amount = BigDecimal.valueOf(0.1);
```

### ⚠️ 陷阱 2：使用 equals 比较大小

```java
BigDecimal a = new BigDecimal("1.0");
BigDecimal b = new BigDecimal("1.00");

if (a.equals(b)) {  // [!code warning] false！
    System.out.println("相等");
}

if (a.compareTo(b) == 0) {  // [!code highlight] true
    System.out.println("相等");
}
```

### ⚠️ 陷阱 3：忘记处理除不尽

```java
BigDecimal a = new BigDecimal("10");
BigDecimal b = new BigDecimal("3");

// [!code error] ArithmeticException
a.divide(b);

// 正确
a.divide(b, 2, RoundingMode.HALF_UP);  // 3.33
```

## 七、加分回答

💡 **超出预期的深度**：

### 1. BigDecimal 内部表示

```java
// BigDecimal 内部结构
public class BigDecimal {
    private final BigInteger intVal;    // 整数部分
    private final int scale;           // 小数位数
    private transient String stringCache;  // 缓存字符串表示
}

// 0.1 表示为：intVal=1, scale=1
// 0.01 表示为：intVal=1, scale=2
// [!code highlight] equals 比较 intVal 和 scale，而不只是值
```

### 2. 舍入模式

| 模式 | 说明 | 示例（2位小数） |
|------|------|-----------------|
| UP | 远离零 | 1.234 -> 1.24 |
| DOWN | 接近零 | 1.235 -> 1.23 |
| HALF_UP | 四舍五入 | 1.235 -> 1.24 |
| HALF_DOWN | 五舍六入 | 1.235 -> 1.23 |
| CEILING | 向正无穷 | 1.234 -> 1.24 |
| FLOOR | 向负无穷 | 1.235 -> 1.23 |

### 3. 性能优化

```java
// 避免频繁创建 BigDecimal
private static final BigDecimal ZERO = BigDecimal.ZERO;
private static final BigDecimal ONE = BigDecimal.ONE;
private static final BigDecimal HUNDRED = new BigDecimal("100");

// 常用金额单位转换
public static BigDecimal toYuan(BigDecimal fen) {
    return fen.divide(HUNDRED, 2, RoundingMode.HALF_UP);
}
```
