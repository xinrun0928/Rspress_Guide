## 一、调试方式

### 1.1 常用调试方式

| 方式 | 说明 |
|------|------|
| System.out | 最简单，但侵入性强 |
| Logging | 日志框架，可配置级别 |
| IDE Debugger | 可视化断点调试 |
| JDB | 命令行调试工具 |

---

## 二、JDB 基础

### 2.1 启动 JDB

```bash
# 启动 JDB 调试
jdb -classpath your-app.jar MyClass

# 或附加到运行中的进程
jdb -attach 12345
```

### 2.2 常用命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `stop at <class>:<line>` | 设置断点 | `stop at MyClass:25` |
| `stop in <class>.<method>` | 方法入口断点 | `stop in MyClass.main` |
| `run` | 开始运行 | `run` |
| `cont` | 继续执行 | `cont` |
| `step` | 单步执行 | `step` |
| `next` | 单步执行（跳过方法） | `next` |
| `print <var>` | 打印变量 | `print x` |
| `locals` | 查看局部变量 | `locals` |
| `watch <class>.<field>` | 监视字段 | `watch MyClass.count` |
| `list` | 查看源代码 | `list` |
| `quit` | 退出 | `quit` |

## 三、调试示例

### 3.1 基本调试流程

```bash
# 1. 启动
jdb -classpath . MyClass

# 2. 设置断点
> stop at MyClass:25

# 3. 运行
> run

# 4. 停在断点处
> print x
x = 10

# 5. 单步执行
> step

# 6. 继续
> cont
```

### 3.2 查看调用栈

```bash
> where
  [1] MyClass.method1 (MyClass.java:20)
  [2] MyClass.main (MyClass.java:10)
```

## 五、常见错误与陷阱

### ⚠️ 陷阱 1：没有编译调试信息

```bash
# [!code warning] 需要使用 -g 编译
javac -g MyClass.java
```

### ⚠️ 陷阱 2：性能问题

```java
// [!code warning] 生产环境不要留调试代码
logger.debug("x = {}", x);  // 预判检查更好
if (logger.isDebugEnabled()) {
    logger.debug("x = {}", x);
}
```

## 六、扩展思考
