## 一、字符类

### 1.1 基本字符类

| 元字符 | 含义 | 示例 |
|--------|------|------|
| `[abc]` | 匹配 a、b 或 c | `[aeiou]` |
| `[^abc]` | 匹配除 a、b、c 外的字符 | `[^0-9]` |
| `[a-z]` | 匹配 a 到 z 的字符 | `[A-Za-z]` |
| `.` | 匹配任意字符（除换行） | `a.c` |

### 1.2 预定义字符类

| 元字符 | 含义 | 等价于 |
|--------|------|--------|
| `\d` | 数字 | `[0-9]` |
| `\D` | 非数字 | `[^0-9]` |
| `\w` | 单词字符 | `[a-zA-Z0-9_]` |
| `\W` | 非单词字符 | `[^\w]` |
| `\s` | 空白字符 | `[ \t\n\x0B\f\r]` |
| `\S` | 非空白字符 | `[^\s]` |

## 二、量词

### 2.1 基本量词

| 量词 | 含义 | 示例 |
|------|------|------|
| `*` | 零次或多次 | `a*` |
| `+` | 一次或多次 | `a+` |
| `?` | 零次或一次 | `a?` |
| `{n}` | 恰好 n 次 | `a{3}` |
| `{n,}` | 至少 n 次 | `a{3,}` |
| `{n,m}` | n 到 m 次 | `a{3,5}` |

### 2.2 贪婪 vs 非贪婪

```java
String text = "aaaa";

// 贪婪匹配：匹配尽可能多
text.matchAll("a+");  // "aaaa"

// [!code warning] 非贪婪：匹配尽可能少
text.matchAll("a+?"); // "a", "a", "a", "a"
```

:::tip 贪婪 vs 非贪婪
- 默认贪婪：匹配最多字符
- 加 `?` 非贪婪：匹配最少字符
:::

## 三、边界匹配

### 3.1 边界元字符

| 元字符 | 含义 | 示例 |
|--------|------|------|
| `^` | 行的开头 | `^abc` |
| `$` | 行的结尾 | `abc$` |
| `\b` | 单词边界 | `\bword\b` |
| `\B` | 非单词边界 | |

### 3.2 常用模式

```java
// 匹配开头
"hello world".matches("^hello");  // true

// 匹配结尾
"hello world".matches("world$");  // true

// 单词边界
"hello world hello".matches("\\bhello\\b");  // true
```

## 四、分组与捕获

### 4.1 捕获组

```java
String text = "2024-01-15";

// 捕获组
Pattern p = Pattern.compile("(\\d{4})-(\\d{2})-(\\d{2})");
Matcher m = p.matcher(text);

if (m.find()) {
    m.group(0);  // "2024-01-15" 整个匹配
    m.group(1);  // "2024" 第一个捕获组
    m.group(2);  // "01" 第二个捕获组
    m.group(3);  // "15" 第三个捕获组
}
```

### 4.2 非捕获组

```java
// (?:...) 不捕获分组
Pattern p = Pattern.compile("(?:\\d{4})-(?:\\d{2})-(\\d{2})");
// 只捕获日：m.group(1) = "15"
```

## 五、Java 中的正则

### 5.1 基本使用

```java
// 验证
boolean matches = "hello".matches("\\w+");  // true

// 分割
String[] parts = "a,b,c".split(",");  // ["a", "b", "c"]
String[] parts = "a,b,,c".split(",+");  // [!code highlight] 多个分隔符

// 替换
String result = "hello world".replaceAll("\\s+", "-");  // "hello-world"
String result = "hello".replaceFirst("l", "L");  // "heLlo"
```

### 5.2 Pattern 和 Matcher

```java
Pattern p = Pattern.compile("\\d+");
Matcher m = p.matcher("abc123def456");

// find：查找下一个匹配
while (m.find()) {
    System.out.println(m.group());  // "123", "456"
}

// replaceAll：替换所有匹配
String result = m.replaceAll("XXX");  // "abcXXXdefXXX"
```

## 六、常见正则模式

### 6.1 验证模式

```java
// 手机号（中国大陆）
"^1[3-9]\\d{9}$"

// 邮箱
"^[\\w.-]+@[\\w.-]+\\.\\w+$"

// 身份证（18位）
"^\\d{17}[\\dXx]$"

// URL
"^https?://[\\w.-]+(:\\d+)?(/.*)?$"

// IP 地址
"^((25[0-5]|2[0-4]\\d|[01]?\\d\\d?)\\.){3}(25[0-5]|2[0-4]\\d|[01]?\\d\\d?)$"
```

## 八、常见错误与陷阱

### ⚠️ 陷阱 1：忘记转义

```java
// [!code error] 错误：在字符串中需要双反斜杠
String regex = "\d+";  // [!code error] \d 不是有效转义

// [!code highlight] 正确：Java 字符串中需要 \\
String regex = "\\d+";
```

### ⚠️ 陷阱 2：. 需要转义

```java
// 匹配 . 号
String regex = "\\.";  // [!code highlight]

// 匹配 \ 号
String regex = "\\\\";
```

### ⚠️ 陷阱 3：边界不匹配

```java
// 错误：没有边界
"13812345678".matches("1[3-9]\\d{9}");  // true
"213812345678".matches("1[3-9]\\d{9}");  // [!code warning] true！

// [!code highlight] 正确：添加 ^ 和 $
"13812345678".matches("^1[3-9]\\d{9}$");  // true
"213812345678".matches("^1[3-9]\\d{9}$");  // false
```

## 九、加分回答

💡 **超出预期的深度**：

### 1. 预编译正则

```java
// [!code highlight] 预编译提高性能
private static final Pattern PHONE_PATTERN = Pattern.compile("^1[3-9]\\d{9}$");

public boolean isValidPhone(String phone) {
    return PHONE_PATTERN.matcher(phone).matches();
}
```

### 2. 零宽断言

```java
// 正向前断言：(?=...)
"hello world".replaceAll("(?=world)", "beautiful ");  // "hello beautiful world"

// 负向前断言：(?!...)
"hello world".replaceAll("\\w+(?!world)", "XXX");  // "XXX world"

// 正向后断言：(?<=...)
"hello world".replaceAll("(?<=hello\\s)", "beautiful ");  // "hello beautiful world"
```
