## 场景切入

面试官问：「你在代码里用过 Optional 吗？」你说「用过」——然后面试官追问「那你有没有用过 Optional.ofNullable？Optional.map 和 Optional.flatMap 有什么区别？」你愣了一下。

Optional 是 Java 8 引入的重要特性，用来优雅地处理 null，避免 NullPointerException。

## 一、为什么需要 Optional

### 1.1 问题背景

```java
// 传统写法：大量 null 检查
User user = getUser(id);
if (user != null) {
    Address address = user.getAddress();
    if (address != null) {
        String city = address.getCity();
        if (city != null) {
            System.out.println(city);
        }
    }
}
```

### 1.2 Optional 的优势

```java
// [!code highlight] Optional 写法：链式调用
String city = getUser(id)
    .flatMap(User::getAddress)
    .flatMap(Address::getCity)
    .orElse("未知城市");
```

## 二、Optional 的创建

### 2.1 创建方式

| 方法 | 说明 | 场景 |
|------|------|------|
| `Optional.empty()` | 创建空 Optional | 明确表示无值 |
| `Optional.of(value)` | 创建非空 Optional | value 不为 null |
| `Optional.ofNullable(value)` | 创建可能为空的 Optional | value 可能为 null |

```java
// 创建 Optional
Optional<String> empty = Optional.empty();  // 空
Optional<String> name = Optional.of("张三");  // 非空，不能为 null
Optional<String> nullable = Optional.ofNullable(getName());  // 可能为空
```

:::warning of vs ofNullable
- `Optional.of(null)` 会抛出 NullPointerException
- `Optional.ofNullable(null)` 返回 Optional.empty()
:::

## 三、常用方法

### 3.1 判断与获取

```java
Optional<String> opt = Optional.of("hello");

// 判断是否有值
if (opt.isPresent()) {  // true
    String value = opt.get();  // 获取值
}

// [!code highlight] 推荐：ifPresent
opt.ifPresent(value -> System.out.println(value));

// [!code highlight] 有值时执行，无值时执行另一个
opt.ifPresentOrElse(
    value -> System.out.println(value),
    () -> System.out.println("无值")
);
```

### 3.2 默认值方法

```java
Optional<String> opt = Optional.empty();

// orElse：总有默认值
String result1 = opt.orElse("默认值");  // "默认值"

// [!code highlight] orElseGet：延迟计算默认值
String result2 = opt.orElseGet(() -> computeDefault());  // 闭包延迟调用

// orElseThrow：有值返回值，无值抛异常
String result3 = opt.orElseThrow(() -> new RuntimeException("无值"));
```

:::tip orElse vs orElseGet
- `orElse`：总是计算默认值参数
- `orElseGet`：只在 Optional 为空时才计算默认值

**性能敏感场景用 orElseGet**。
:::

### 3.3 转换方法

```java
Optional<User> user = Optional.of(new User("张三", 25));

// [!code highlight] map：转换值
Optional<String> name = user.map(User::getName);  // Optional<String>

// [!code highlight] flatMap：扁平化转换
Optional<String> city = user.flatMap(User::getAddress)
                            .flatMap(Address::getCity);

// filter：过滤
Optional<User> adult = user.filter(u -> u.getAge() >= 18);
```

### 3.4 方法对比表

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| map | `Optional<T>` | `Optional<R>` | 转换值 |
| flatMap | `Optional<T>` | `Optional<R>` | 扁平化转换 |
| filter | `Optional<T>` | `Optional<T>` | 过滤 |

## 四、最佳实践

### 4.1 链式调用

```java
// 传统写法
public String getCityName(User user) {
    if (user == null) {
        return "未知";
    }
    Address address = user.getAddress();
    if (address == null) {
        return "未知";
    }
    City city = address.getCity();
    if (city == null) {
        return "未知";
    }
    return city.getName();
}

// [!code highlight] Optional 写法
public String getCityName(User user) {
    return Optional.ofNullable(user)
        .flatMap(User::getAddress)
        .flatMap(Address::getCity)
        .map(City::getName)
        .orElse("未知");
}
```

### 4.2 集合操作

```java
// 查找符合条件的元素
Optional<User> user = users.stream()
    .filter(u -> "admin".equals(u.getRole()))
    .findFirst();

// 或默认值
User admin = users.stream()
    .filter(u -> "admin".equals(u.getRole()))
    .findFirst()
    .orElse(createDefaultAdmin());
```

## 六、常见错误与陷阱

### ⚠️ 陷阱 1：Optional 用于方法参数

```java
// [!code warning] 不推荐：Optional 用于参数
void process(Optional<String> name) {
    // 调用方需要包装参数
}

// [!code highlight] 推荐：重载方法或使用 null
void process(String name) {
    // 调用方可以直接传值
}

void process(String name, String defaultValue) {
    // 提供默认值参数
}
```

### ⚠️ 陷阱 2：Optional 用于成员变量

```java
// [!code warning] Optional 不应该用于成员变量
class User {
    private Optional<String> phone;  // [!code warning]
}

// [!code highlight] 推荐：使用 null 或空字符串
class User {
    private String phone;  // phone == null 表示无值
}
```

### ⚠️ 陷阱 3：过度使用

```java
// [!code warning] 过度使用 Optional
Optional<String> opt = Optional.of("hello");
System.out.println(opt.isPresent() ? opt.get() : null);

// [!code highlight] 简洁写法
String result = opt.orElse(null);
```

### ⚠️ 陷阱 4：get() 不做检查

```java
Optional<String> opt = Optional.empty();
String value = opt.get();  // [!code error] NoSuchElementException
```

## 七、加分回答

💡 **超出预期的深度**：

### 1. Optional 的序列化

```java
// [!code warning] Optional 不支持序列化！
class User implements Serializable {
    private String name;
    private Optional<String> phone;  // [!code warning] 不推荐
}

// 替代方案：使用字段 + getter 组合
class User implements Serializable {
    private String name;
    private String phone;  // null 表示无值

    public Optional<String> getPhone() {
        return Optional.ofNullable(phone);
    }
}
```

### 2. Stream 与 Optional 组合

```java
// 从 List 中查找元素
Optional<User> admin = users.stream()
    .filter(u -> "admin".equals(u.getRole()))
    .findFirst();

// 聚合操作
int maxAge = users.stream()
    .mapToInt(User::getAge)
    .max()
    .orElse(0);
```

### 3. Optional 的实现原理

```java
// Optional.of(T value) 伪代码
public final class Optional<T> {
    private final T value;

    private Optional(T value) {
        this.value = Objects.requireNonNull(value);
    }

    public static <T> Optional<T> of(T value) {
        return new Optional<>(value);
    }

    public T orElse(T other) {
        return value != null ? value : other;
    }
}
```
