## 一、Lambda 的基本语法

### 1.1 语法形式

```java
// 完整形式
(int x, int y) -> { return x + y; }

// 省略形式：参数类型
(x, y) -> x + y

// 单参数省略括号
x -> x * 2

// 无参数
() -> System.out.println("Hello")
```

### 1.2 Lambda 与函数式接口

```java
// 函数式接口：只有一个抽象方法的接口
@FunctionalInterface  // [!code highlight] 注解帮助编译器检查
interface Calculator {
    int calculate(int a, int b);
}

// 使用 Lambda
Calculator add = (a, b) -> a + b;
System.out.println(add.calculate(1, 2));  // 3
```

## 二、Lambda vs 匿名内部类

### 2.1 语法对比

```java
// 匿名内部类
Runnable r1 = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello");
    }
};

// [!code highlight] Lambda 表达式
Runnable r2 = () -> System.out.println("Hello");
```

### 2.2 核心区别

| 对比 | Lambda | 匿名内部类 |
|------|--------|------------|
| this 引用 | 指向外围类的 this | 指向匿名内部类实例 |
| 编译产物 | invokeDynamic 指令 | 新增类文件 |
| 语义 | 行为参数化 | 创建对象 |
| 作用域 | 继承外围作用域 | 创建新作用域 |

### 2.3 this 的区别

```java
public class MyClass {
    Runnable r1 = new Runnable() {
        @Override
        public void run() {
            System.out.println(this.getClass().getName());  // MyClass$1
        }
    };

    Runnable r2 = () -> {
        System.out.println(this.getClass().getName());  // MyClass
    };
}
```

:::tip this 的区别
- Lambda 的 this 指向**外围类的实例**
- 匿名内部类的 this 指向**匿名内部类实例**
:::

## 三、JVM 实现原理

### 3.1 字节码对比

```java
// 源码
Runnable r = () -> System.out.println("Hello");
```

```bash
# 匿名内部类编译产物
MyClass$1.class  # 新增类文件

# Lambda 编译产物
# [!code highlight] 没有新增类文件，使用 invokedynamic
```

### 3.2 invokedynamic 指令

```java
// 编译后的字节码（伪代码）
INVOKEDYNAMIC makeConcatWithConstants  // [!code highlight]
    " Runnable.run ()V"           // Lambda 类型
    [ bootstrap method ]           // Lambda 引导方法
```

### 3.3 LambdaMetafactory

```java
// JVM 内部使用 LambdaMetafactory
// 在第一次调用时生成 Lambda 表达式对应的类

CallSite.makeConcatWithConstants(
    MethodHandles.Lookup(),      // 查找上下文
    "run",                        // 方法名
    MethodType,                   // 方法类型
    LambdaMetafactory.metafactory(...)  // [!code highlight] 引导方法
);
```

## 四、Lambda 的捕获

### 4.1 变量捕获

```java
int x = 10;  // effectively final
int y = 20;  // effectively final

// [!code highlight] Lambda 捕获外部变量
Calculator calc = (a, b) -> a + b + x + y;
```

### 4.2 捕获规则

```java
// [!code warning] 不能修改捕获的变量
int counter = 0;
Runnable r = () -> {
    counter++;  // [!code error] 编译错误
};
```

:::warning 捕获限制
Lambda 可以读取外部变量，但**不能修改**。这是因为 Lambda 可能在多线程中使用，修改会导致线程安全问题。
:::

### 4.3 实例字段捕获

```java
public class MyClass {
    private int count = 0;

    public void increment() {
        Runnable r = () -> {
            this.count++;  // [!code highlight] 可以修改实例字段
        };
        r.run();
    }
}
```

:::tip 可以修改实例字段
Lambda 不能修改**局部变量**，但可以修改**实例字段**和**静态字段**。
:::

## 六、常见错误与陷阱

### ⚠️ 陷阱 1：捕获可变变量

```java
// [!code warning] 并发场景下的陷阱
for (int i = 0; i < 10; i++) {
    new Thread(() -> System.out.println(i)).start();
}

// [!code warning] 输出不确定！因为 i 被共享
```

### ⚠️ 陷阱 2：return 语句

```java
// 匿名内部类中的 return
list.forEach(new Consumer<Integer>() {
    @Override
    public void accept(Integer i) {
        return;  // [!code warning] 返回的是 accept 方法
    }
});

// Lambda 中的 return
list.forEach(i -> {
    return;  // [!code warning] 返回的是 accept 方法，不是 forEach 方法
});
```

### ⚠️ 陷阱 3：异常处理

```java
// [!code warning] Lambda 不能抛出检查异常
list.forEach(name -> {
    throw new IOException();  // [!code error] 编译错误
});

// 解决方案：使用包装
list.forEach(name -> {
    try {
        // ...
    } catch (IOException e) {
        throw new RuntimeException(e);
    }
});
```

## 七、加分回答

💡 **超出预期的深度**：

### 1. Lambda 的创建时机

```java
// 引导方法在第一次调用时执行
// 之后调用直接使用缓存

// 第一次调用
Runnable r = () -> System.out.println("Hello");
r.run();  // 触发引导方法，生成 Lambda 类

// 后续调用
r.run();  // 直接调用已生成的 Lambda 类
```

### 2. Lambda 的序列化

```java
// [!code warning] Lambda 可以序列化，但不推荐
@FunctionalInterface
interface Serializer extends Serializable {
    void execute();
}

// [!code highlight] 更好的方式：使用方法引用
class MyClass implements Serializer {
    @Override
    public void execute() { }
}
```

### 3. 常见的函数式接口

| 接口 | 方法 | 用途 |
|------|------|------|
| `Function<T,R>` | `R apply(T t)` | 转换 |
| `Predicate<T>` | `boolean test(T t)` | 判断 |
| `Consumer<T>` | `void accept(T t)` | 消费 |
| `Supplier<T>` | `T get()` | 生产 |
| `Runnable` | `void run()` | 无参数无返回值 |
| `Callable<V>` | `V call()` | 有返回值 |
