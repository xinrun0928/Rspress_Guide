# Redis 25 题

> **目标级别**：P5/P6
> **面试频率**：Redis 是后端面试中最核心的缓存技术，无论是面试 Java、Golang 还是 Python 后端，Redis 都是高频考点。
> **面试特点**：Redis 面试不仅考察"会不会用"，更考察"为什么这样用"的底层原理。

面试官问：「Redis 为什么这么快？」你说「因为是内存数据库」——然后面试官紧接着追问「那 Memcached 也是内存数据库，为什么 Redis 比它快？Redis 单线程又是怎么回事？」你沉默了。

这就是 Redis 面试的真实面貌：表面上问的是使用，实际上考的是底层原理。80% 的候选人在第一层回答得头头是道，但被追问到第二层、第三层时就露馅了。

本专题精选 Redis 中最具面试价值的 25 道高频题，从「是什么」到「为什么」再到「怎么答」，覆盖数据结构、线程模型、持久化、缓存三问、集群五大核心模块。

## 学习路径建议

### P5 级别（1-3 年经验）

重点掌握前 18 题，确保每道题能回答到第二层追问。重点关注：
- 五种数据结构的底层实现
- 单线程模型与 IO 多路复用
- RDB/AOF 持久化机制
- 缓存三问的解决方案
- 主从复制与哨兵机制

### P6 级别（3-5 年经验）

全部 25 题都需要掌握，确保能回答到第三层追问。重点关注：
- SDS 动态字符串的内存优化
- 跳表 vs 红黑树的时间复杂度
- 多线程模型的适用场景
- RedLock 算法的争议与实现
- Cluster 集群的分片策略

## 常见面试场景

面试官通常不会直接问「Redis 了解吗」，而是把 Redis 知识融入到场景题或源码分析中：

- 「看你项目里用了 Redis，那缓存过期了怎么处理的？」
- 「你在项目中遇到过缓存穿透吗？怎么解决的？」
- 「Redis 的主从复制是怎么实现的？」
- 「Redis Cluster 集群数据是怎么分片的？」

这些追问才是真正考验 Redis 深度的时候。所以不要只背答案，要理解背后的原理。
