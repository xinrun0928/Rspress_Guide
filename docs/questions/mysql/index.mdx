# MySQL 30 题

> **目标级别**：P5/P6
> **面试频率**：MySQL 是后端面试中最核心的数据库知识，无论你是面试 Java、Golang 还是 Python 后端，MySQL 都是必考内容。

面试官问：「MySQL 的索引是什么？」你说「索引就像书的目录，能加快查询速度」——然后面试官紧接着追问「那索引底层是什么结构？为什么不用二叉树？为什么 B+Tree 比 B-Tree 更适合磁盘存储？」你沉默了。

这就是 MySQL 面试的真实面貌：表面上问的是概念，实际上考的是原理深度。80% 的候选人在第一层回答得头头是道，但被追问到第二层、第三层时就露馅了。

本专题精选 MySQL 中最具面试价值的 30 道高频题，从「是什么」到「为什么」再到「怎么答」，覆盖索引、事务、锁、主从复制、查询优化五大核心模块。

## 学习路径建议

### P5 级别（1-3 年经验）

重点掌握前 20 题，确保每道题能回答到第二层追问。重点关注：
- B+Tree 索引原理与数据结构
- 事务隔离级别与 MVCC
- 常见索引失效场景
- Explain 执行计划分析

### P6 级别（3-5 年经验）

全部 30 题都需要掌握，确保能回答到第三层追问。重点关注：
- MVCC 与 ReadView 的底层实现
- 间隙锁与临键锁的机制
- 主从复制延迟问题与解决方案
- 分库分表的设计与路由策略

## 常见面试场景

面试官通常不会直接问「MySQL 了解吗」，而是把 MySQL 知识融入到场景题或源码分析中：

- 「看你项目里用了 MySQL，那索引失效的场景有哪些？」
- 「你在项目中遇到过慢查询吗？怎么优化的？」
- 「MySQL 的事务隔离级别是怎么实现的？」
- 「数据库主从复制延迟怎么解决？」

这些追问才是真正考验 MySQL 深度的时候。所以不要只背答案，要理解背后的原理。
