# CompletableFuture 异步编程

> **目标级别**：P6
> **面试频率**：🔴 高频

面试官问：「CompletableFuture 是什么？」你说「异步编程」——然后面试官紧接着追问「那 CompletableFuture 和 Future 有什么区别？如何实现并行调用？」你沉默了。

CompletableFuture 是 Java 8 引入的异步编程利器，理解其API才能正确使用。

## 面试官最关心的 3 个问题

1. ⚠️ CompletableFuture 和 Future 的区别是什么？
2. ⚠️ 如何使用 CompletableFuture 实现异步编程？
3. ⚠️ CompletableFuture 的常用 API 有哪些？

## 核心原理

### CompletableFuture 的改进

| 特性 | Future | CompletableFuture |
|------|--------|-------------------|
| **手动完成** | ❌ | ✅ |
| **链式调用** | ❌ | ✅ |
| **组合多个** | ❌ | ✅ |
| **回调支持** | ❌ | ✅ |
| **异常处理** | 困难 | exceptionally/handle |
| **超时控制** | get(timeout) | orTimeout |

### 创建 CompletableFuture

```java
// 1. supplyAsync - 有返回值
CompletableFuture<String> future1 = CompletableFuture.supplyAsync(() -> {
    return "Hello";
});

// 2. runAsync - 无返回值
CompletableFuture<Void> future2 = CompletableFuture.runAsync(() -> {
    System.out.println("Running");
});

// 3. 手动完成
CompletableFuture<String> future3 = new CompletableFuture<>();
future3.complete("Manual result");
```

## 异步任务链

### thenApply - 转换

```java
// 同步版本
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "Hello")
    .thenApply(s -> s + " World");
System.out.println(future.get()); // Hello World
```

### thenCompose - 扁平化

```java
// thenCompose 用于返回 CompletableFuture 的场景
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "Hello")
    .thenCompose(s -> CompletableFuture.supplyAsync(() -> s + " World"));
```

### thenCombine - 组合

```java
// 等待两个 Future 完成并组合结果
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "Hello")
    .thenCombine(
        CompletableFuture.supplyAsync(() -> " World"),
        (s1, s2) -> s1 + s2
    );
```

## 异常处理

### exceptionally

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> {
        if (true) throw new RuntimeException("Error");
        return "result";
    })
    .exceptionally(ex -> {
        return "default value"; // 处理异常
    });
```

### handle

```java
// handle 无论成功还是失败都会被调用
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "result")
    .handle((result, ex) -> {
        if (ex != null) {
            return "error handling: " + ex.getMessage();
        }
        return result + " handled";
    });
```

## 并行执行

### allOf - 等待所有

```java
CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "one");
CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "two");
CompletableFuture<String> f3 = CompletableFuture.supplyAsync(() -> "three");

CompletableFuture<Void> all = CompletableFuture.allOf(f1, f2, f3);
all.join(); // 等待所有完成

String result1 = f1.join();
String result2 = f2.join();
String result3 = f3.join();
```

### anyOf - 等待任一

```java
CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> {
    Thread.sleep(1000);
    return "one";
});

CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "two");

CompletableFuture<Object> any = CompletableFuture.anyOf(f1, f2);
Object result = any.join(); // 返回最快完成的结果
```

## 实际应用场景

### 并行调用多个 API

```java
public CompletableFuture<UserVO> getUserInfo(Long userId) {
    CompletableFuture<User> userFuture = CompletableFuture
        .supplyAsync(() -> userService.getUser(userId));

    CompletableFuture<List<Order>> ordersFuture = CompletableFuture
        .supplyAsync(() -> orderService.getUserOrders(userId));

    CompletableFuture<List<Favorite>> favoritesFuture = CompletableFuture
        .supplyAsync(() -> favoriteService.getUserFavorites(userId));

    return userFuture.thenCombine(ordersFuture, (user, orders) ->
        new UserVO(user, orders, null)
    ).thenCombine(favoritesFuture, (userVO, favorites) -> {
        userVO.setFavorites(favorites);
        return userVO;
    });
}
```

### 超时控制

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> {
        Thread.sleep(5000);
        return "result";
    })
    .orTimeout(2, TimeUnit.SECONDS) // 2 秒超时
    .exceptionally(ex -> {
        return "timeout"; // 超时处理
    });
```

## 高频面试题

### 🔴 题目 1：CompletableFuture 和 Future 的区别？

**参考回答**：

| 区别 | Future | CompletableFuture |
|------|--------|-------------------|
| **手动完成** | ❌ | ✅ |
| **链式调用** | ❌ | ✅ |
| **组合多个** | ❌ | ✅ |
| **回调** | ❌ | ✅ |
| **异常处理** | 困难 | exceptionally |

### 🔴 题目 2：thenApply 和 thenCompose 的区别？

**参考回答**：

| 方法 | 输入 | 输出 | 适用场景 |
|------|------|------|---------|
| **thenApply** | `T -> U` | `CompletableFuture<U>` | 同步转换 |
| **thenCompose** | `T -> CompletableFuture<U>` | `CompletableFuture<U>` | 扁平化 |

```java
// thenApply：返回普通值
.thenApply(s -> s.toUpperCase());

// thenCompose：返回 CompletableFuture
.thenCompose(id -> getUser(id)); // 需要 flatten
```

### 🔴 题目 3：如何实现并行调用？

**参考回答**：

```java
CompletableFuture<User> userFuture = CompletableFuture
    .supplyAsync(() -> userService.getUser(id));

CompletableFuture<List<Order>> ordersFuture = CompletableFuture
    .supplyAsync(() -> orderService.getOrders(id));

// 并行执行，然后组合结果
userFuture.thenCombine(ordersFuture, (user, orders) ->
    new UserVO(user, orders)
).join();
```

## 常见错误与陷阱

### ⚠️ 陷阱 1：忘记 join/get

```java
// ❌ 异步任务不会自动执行
CompletableFuture
    .supplyAsync(() -> doSomething());
// 没有获取结果，任务可能不执行

// ✅ 使用 join 或 get
CompletableFuture
    .supplyAsync(() -> doSomething())
    .join();
```

### ⚠️ 陷阱 2：线程池配置不当

```java
// ❌ 使用默认的 ForkJoinPool.commonPool()
// 所有 CompletableFuture 共享一个线程池
CompletableFuture.supplyAsync(() -> doSomething());

// ✅ 使用自定义线程池
ExecutorService executor = Executors.newFixedThreadPool(10);
CompletableFuture
    .supplyAsync(() -> doSomething(), executor)
    .thenApply(...);
```

### ⚠️ 陷阱 3：异常丢失

```java
// ❌ 异常被吞掉
CompletableFuture.supplyAsync(() -> {
    throw new RuntimeException("error");
}).thenApply(s -> s.toUpperCase());

// ✅ 使用 exceptionally 或 handle 处理
CompletableFuture.supplyAsync(() -> {
    throw new RuntimeException("error");
}).exceptionally(ex -> {
    return "error: " + ex.getMessage();
});
```

## 加分回答

### 💡 thenApply vs thenApplyAsync

| 方法 | 说明 |
|------|------|
| **thenApply** | 在上一个任务执行的线程中执行 |
| **thenApplyAsync** | 在新线程中执行（使用线程池） |

```java
// thenApply：顺序执行
CompletableFuture
    .supplyAsync(() -> "step1")
    .thenApply(s -> {
        System.out.println("Step1 在: " + Thread.currentThread().getName());
        return s + " -> step2";
    });

// thenApplyAsync：在新线程中执行
CompletableFuture
    .supplyAsync(() -> "step1")
    .thenApplyAsync(s -> {
        System.out.println("Step2 在: " + Thread.currentThread().getName());
        return s + " -> step2";
    });
```

## 总结对比表

| API | 说明 |
|-----|------|
| `supplyAsync` | 创建异步任务（有返回值） |
| `runAsync` | 创建异步任务（无返回值） |
| `thenApply` | 转换 |
| `thenCompose` | 扁平化 |
| `thenCombine` | 组合两个 |
| `exceptionally` | 异常处理 |
| `handle` | 统一处理结果和异常 |
| `allOf` | 等待所有完成 |
| `anyOf` | 等待任一完成 |
| `orTimeout` | 超时控制 |
