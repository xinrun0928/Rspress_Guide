# Future 与 Callable

> **目标级别**：P5/P6
> **面试频率**：🔴 高频

面试官问：「Future 是什么？」你说「异步计算的结果」——然后面试官紧接着追问「那 Future 和 CompletableFuture 有什么区别？」你沉默了。

Future 是 Java 并发编程中异步任务的基础，理解其限制才能理解 CompletableFuture 的价值。

## 面试官最关心的 3 个问题

1. ⚠️ Future 和 Callable 是什么关系？
2. ⚠️ Future 有哪些局限性？
3. ⚠️ 如何正确使用 Future？

## 核心原理

### Callable 接口

```java
public interface Callable<V> {
    V call() throws Exception;
}
```

Callable 与 Runnable 的区别：

| 接口 | 返回值 | 异常 |
|------|--------|------|
| **Runnable** | void | 不能抛出受检异常 |
| **Callable** | V | 可以抛出受检异常 |

### Future 接口

```java
public interface Future<V> {
    boolean cancel(boolean mayInterruptIfRunning);
    boolean isCancelled();
    boolean isDone();
    V get() throws InterruptedException, ExecutionException;
    V get(long timeout, TimeUnit unit)
        throws InterruptedException, ExecutionException, TimeoutException;
}
```

### FutureTask

FutureTask 是 Future 和 Callable 的实现：

```java
public class FutureTask<V> implements RunnableFuture<V> {
    private final Callable<V> callable;
    private Object outcome; // 存储结果或异常
    private volatile int state; // NEW, COMPLETING, NORMAL, EXCEPTIONAL, CANCELLED, INTERRUPTING, INTERRUPTED
}
```

## 基本使用

### 提交 Callable 获取 Future

```java
ExecutorService executor = Executors.newFixedThreadPool(4);

// 提交 Callable
Future<String> future = executor.submit(() -> {
    Thread.sleep(1000);
    return "Result";
});

// 获取结果（阻塞）
String result = future.get(); // 等待完成
System.out.println(result);

// 非阻塞检查
if (future.isDone()) {
    String result = future.getNow(); // 假设有 getNow 方法
}
```

### FutureTask 的使用

```java
FutureTask<String> futureTask = new FutureTask<>(() -> {
    return "Result";
});

new Thread(futureTask).start();
String result = futureTask.get();
```

## Future 的局限性

### 局限性一：不能手动完成

```java
// ❌ 无法手动设置结果
Future<String> future = executor.submit(() -> {
    // 无法提前完成
});
```

### 局限性二：不能链式调用

```java
// ❌ 无法链式调用
Future<Integer> f1 = executor.submit(() -> 1);
Future<Integer> f2 = executor.submit(() -> f1.get() + 1); // 必须等 f1 完成
Future<Integer> f3 = executor.submit(() -> f2.get() + 1); // 必须等 f2 完成
```

### 局限性三：不能组合多个 Future

```java
// ❌ 无法并行执行并等待所有完成
Future<List<String>> results = executor.submit(() -> {
    List<Future<String>> futures = submitAll();
    return futures.stream().map(Future::get).collect(toList());
});
```

### 局限性四：不支持回调

```java
// ❌ 必须阻塞等待
Future<String> future = executor.submit(() -> "result");
String result = future.get(); // 无法注册完成回调
```

## CompletableFuture 的改进

### CompletableFuture vs Future

| 特性 | Future | CompletableFuture |
|------|--------|-------------------|
| 手动完成 | ❌ | ✅ |
| 链式调用 | ❌ | ✅ |
| 组合多个 | ❌ | ✅ |
| 回调支持 | ❌ | ✅ |
| 异步编排 | ❌ | ✅ |

## 高频面试题

### 🔴 题目 1：Future 和 Callable 有什么区别？

**参考回答**：

| 区别 | Callable | Runnable |
|------|----------|----------|
| **返回值** | V（泛型） | void |
| **异常** | 可抛出受检异常 | 不能抛�� |
| **提交方式** | submit(Callable) | submit(Runnable) |
| **Future** | 返回 Future | 返回 Future（结果为 null） |

### 🔴 题目 2：Future 有什么局限性？

**参考回答**：

Future 的局限性：

1. **不能手动完成**：无法提前设置结果
2. **不能链式调用**：不支持 then 系列方法
3. **不能组合多个**：无法等待多个 Future 完成
4. **不支持回调**：只能阻塞等待结果

### 🔴 题目 3：Future.get() 和 Future.get(timeout) 有什么区别？

**参考回答**：

| 方法 | 特点 | 适用场景 |
|------|------|---------|
| `get()` | 无限等待 | 不关心超时时间 |
| `get(timeout)` | 超时等待，返回剩余时间 | 有时效要求的任务 |

## 常见错误与陷阱

### ⚠️ 陷阱 1：在主线程中调用 get()

```java
// ❌ 如果线程池满，可能死锁
executor.submit(() -> {
    Future<String> future = executor.submit(() -> "result");
    future.get(); // 如果没有空闲线程，会死锁
});
```

### ⚠️ 陷阱 2：忽视 InterruptedException

```java
// ❌ 没有处理中断
String result = future.get(); // 抛出 InterruptedException

// ✅ 正确处理
try {
    String result = future.get();
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
    throw new RuntimeException(e);
}
```

### ⚠️ 陷阱 3：忘记处理 ExecutionException

```java
// ❌ 没有处理异常
String result = future.get(); // ExecutionException 包装了实际异常

// ✅ 正确处理
try {
    String result = future.get();
} catch (ExecutionException e) {
    throw e.getCause(); // 提取原始异常
}
```

## 加分回答

### 💡 使用 CompletableFuture 模拟 Future

```java
public <T> CompletableFuture<T> toCompletableFuture(ExecutorService executor) {
    CompletableFuture<T> result = new CompletableFuture<>();
    executor.submit(() -> {
        try {
            result.complete(callable.call());
        } catch (Exception e) {
            result.completeExceptionally(e);
        }
    });
    return result;
}
```

## 延伸思考

### 面试官可能会继续追问

1. 「CompletableFuture 和 Java 8 Stream 有什么关系？」
2. 「如何取消一个 Future？」
3. 「FutureTask 的 state 状态转换是什么？」

### 回答方向

关于 FutureTask 状态：
- NEW → COMPLETING：任务完成，准备存储结果
- COMPLETING → NORMAL：存储正常结果
- COMPLETING → EXCEPTIONAL：存储异常
- NEW → CANCELLED：任务被取消
- CANCELLED → INTERRUPTED：需要中断执行线程
