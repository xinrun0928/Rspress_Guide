# 并发编程 35 题

> **目标级别**：P5/P6/P6+
> **面试频率**：并发编程是 Java 后端面试的核心难点，P6 级别及以上几乎必问。本专题涵盖 35 道高频面试题，从基础概念到源码原理，帮助你建立完整的并发知识体系。

面试官问：「synchronized 是怎么保证线程安全的？」你说「通过加锁」——然后面试官紧接着追问「锁升级过程是怎样的？偏向锁什么时候会撤销？轻量级锁是怎么实现的？」你沉默了。

这就是并发编程面试的真实面貌：表面上问的是概念，实际上考的是原理深度。80% 的候选人在第一层回答得头头是道，但被追问到第二层、第三层时就露馅了。

本专题精选并发编程中最具面试价值的 35 道高频题，从「是什么」到「为什么」再到「怎么答」，帮助你建立完整的并发编程知识体系。

## 学习路径建议

### P5 级别（1-3 年经验）

重点掌握前 20 题，确保每道题能回答到第二层追问。重点关注：

- 线程基础概念与生命周期
- volatile 可见性与禁止重排序
- synchronized 基本原理
- 线程池核心参数与执行流程
- CountDownLatch、CyclicBarrier、Semaphore 使用

### P6 级别（3-5 年经验）

全部 35 题都需要掌握，确保能回答到第三层追问。重点关注：

- JMM 与 happens-before 原则的底层实现
- synchronized 锁升级的源码分析
- AQS 核心原理与模板方法模式
- LongAdder 高并发优化原理
- ThreadLocal 内存泄漏场景与解决方案

### P6+ 级别（5 年以上经验）

在 P6 基础上加深理解，重点关注：

- Mark Word 的位存储结构
- CAS 的底层实现（cmpxchg 指令）
- AQS 的实现细节（CLH 队列、state 状态管理）
- 生产环境线程池调优
- 并发编程中的性能优化

## 常见面试场景

面试官通常不会直接问「并发编程怎么样」，而是把并发题融入到场景题或源码分析中：

- 「看你用过线程池，那线程池的执行流程是怎样的？」
- 「你在项目里用过 volatile 吗？为什么用它？」
- 「synchronized 和 ReentrantLock 有什么区别？各自适用场景？」
- 「ThreadLocal 会出现内存泄漏，你知道是什么原因吗？」

这些追问才是真正考验并发编程功底的时候。所以不要只背答案，要理解背后的原理。
