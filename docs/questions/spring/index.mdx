# Spring 25 题：面试核心知识点全攻略

**目标级别**：P5/P6

## 开场：为什么 Spring 面试必须深入

面试官问：「Spring 有哪些核心模块？」你列举了 Spring Core、Spring AOP、Spring MVC——然后面试官追问：「Spring 是怎么实现依赖注入的？BeanFactory 和 ApplicationContext 有什么区别？」你沉默了。

这不是一道简单的概念题。Spring 面试考察的是**你对框架内部机制的理解深度**，以及**能否解释为什么这样设计**。很多人会用 Spring，却说不清 IoC 容器如何管理 Bean 的生命周期；能配置 `@Transactional`，却不知道它为什么会失效。

本专题精选 **25 道高频 Spring 面试题**，覆盖 IoC/DI 原理、Bean 生命周期、AOP 代理、事务管理、Spring Boot 自动配置五大核心模块。每道题都配有追问链、源码解析、对比表格和踩坑警示，帮你建立完整的 Spring 技术体系。

> 本专题适合准备 P5~P6 级别 Java 后端面试，涵盖 Spring Framework 和 Spring Boot 核心知识点。

## 面试官最关心的 3 个问题（快速自测）

在深入学习之前，先自测一下这三个���题：

1. **🔴 IoC 容器是如何完成依赖注入的？BeanFactory 和 ApplicationContext 在启动阶段有什么区别？**
2. **🔴 Spring 是如何解决循环依赖的？为什么构造器注入无法解决循环依赖？**
3. **🔴 @Transactional 是如何生效的？为什么在同一个类中调用 this 方法会失效？**

如果你对这三个问题的答案还不太确定，请认真阅读本专题的每一篇文章。

## 一、IoC 与 DI 原理（4 题）

| 题目 | 频率 | 难度 | 预计回答时间 |
|------|------|------|-------------|
| IoC 与 DI 原理 | 🔴 高频 | P6 | 3-5 分钟 |
| BeanFactory vs ApplicationContext | 🟡 中频 | P5 | 2-3 分钟 |
| 三级缓存解决循环依赖 | 🔴 高频 | P6 | 5-8 分钟 |
| 构造器注入无法解决循环依赖 | 🟡 中频 | P6 | 3-4 分钟 |

### 面试官视角

IoC（控制反转）和 DI（依赖注入）是 Spring 最核心的概念，也是面试中必问的基础题。面试官通常从以下几个角度追问：

- **第一层**：IoC 和 DI 是什么意思？有什么区别？
- **第二层**：Spring 是怎么实现依赖注入的？反射 + 暴力破解字段
- **第三层**：BeanFactory 和 ApplicationContext 在启动阶段做了什么不同的事？
- **隐藏层**：为什么 Spring 要设计三级缓存？只有两级缓存行不行？

### 常见错误

1. **❌ 把 IoC 等同于 DI**——IoC 是设计思想，DI 是实现方式，两者不是一回事
2. **❌ 说 ApplicationContext 是 BeanFactory 的子类**——ApplicationContext 组合了 BeanFactory，功能更丰富
3. **❌ 不知道循环依赖的触发场景**——构造器注入 + prototype bean + @Async 都会导致循环依赖检测失败

## 二、Bean 生命周期（4 题）

| 题目 | 频率 | 难度 | 预计回答时间 |
|------|------|------|-------------|
| Bean 生命周期 | 🔴 高频 | P5 | 2-3 分钟 |
| BeanPostProcessor 原理 | 🟡 中频 | P6 | 4-5 分钟 |
| 三级缓存与循环依赖 | 🔴 高频 | P6 | 5-8 分钟 |
| 构造器注入循环依赖 | 🟡 中频 | P6 | 3-4 分钟 |

### 面试官视角

Bean 生命周期是 Spring 源码中最核心的知识点之一。面试官经常从这个场景切入：

> 「我们在项目中用 @PostConstruct 初始化数据，和用 InitializingBean.afterPropertiesSet() 有什么区别？BeanPostProcessor.postProcessBeforeInitialization() 和 postProcessAfterInitialization() 分别在什么时候调用？」

如果你的回答只停留在「先后顺序」，而不知道 **BeanPostProcessor 是 Spring 扩展点的核心机制**，面试官会认为你只用过框架，没有深入理解框架的设计思想。

### 常见错误

1. **❌ 以为 @PostConstruct 优先级最高**——实际上 postProcessBeforeInitialization 最后一步才是 @PostConstruct
2. **❌ 把 init-method 和 afterPropertiesSet 混为一谈**——前者是自定义初始化方法，后者是 Spring 提供的接口
3. **❌ 不知道 BeanPostProcessor 的应用场景**——@Autowired、@Transactional 都依赖它

## 三、AOP 代理（4 题）

| 题目 | 频率 | 难度 | 预计回答时间 |
|------|------|------|-------------|
| AOP 原理与代理模式 | 🔴 高频 | P6 | 3-5 分钟 |
| JDK 动态代理 vs CGLIB | 🔴 高频 | P5 | 2-3 分钟 |
| 通知执行顺序 | 🟡 中频 | P6 | 3-4 分钟 |
| @Transactional 失效场景 | 🔴 高频 | P6 | 4-6 分钟 |

### 面试官视角

AOP 是 Spring 中最容易被问「为什么这样设计」的模块。面试官的追问路径通常是：

1. AOP 的核心概念：切点、切面、通知、织入
2. Spring AOP 和 AspectJ 的区别
3. 通知类型：@Before、@After、@AfterReturning、@AfterThrowing、@Around
4. 为什么 Spring 选 CGLIB 而不选 JDK 动态代理？

如果面试官想考验你的深度，可能会问：

> 「我们用 @Async 异步执行一个方法，发现异步没有生效，排查了一圈发现是 Spring 事务导致的问题——为什么？」

这道题的答案涉及 **Spring 事务与 AOP 代理的执行顺序**，很多人答不上来。

### 常见错误

1. **❌ 说 Spring AOP 支持方法调用拦截**——Spring AOP 只支持方法执行拦截，不支持属性访问拦截
2. **❌ 以为 @Async 异步方法在事务中是隔离的**——如果调用方有事务，异步方法会沿用调用方的事务上下文
3. **❌ 把 AOP 和设计模式中的代理模式画等号**——代理模式是实现 AOP 的一种手段，但不是全部

## 四、Spring 事务（5 题）

| 题目 | 频率 | 难度 | 预计回答时间 |
|------|------|------|-------------|
| Spring 事务管理原理 | 🔴 高频 | P6 | 4-6 分钟 |
| 事务传播行为 | 🔴 高频 | P6 | 3-5 分钟 |
| @Transactional 失效场景 | 🔴 高频 | P6 | 5-8 分钟 |
| 声明式事务 vs 编程式事务 | 🟡 中频 | P5 | 2-3 分钟 |

### 面试官视角

Spring 事务是面试中「表面简单，实际上坑很多」的模块。面试官最喜欢从这个场景切入：

> 「我们在 service 方法上加了 @Transactional，测试环境没问题，生产环境数据库真的回滚了吗？」

这道题涉及 **Spring 事务的生效条件**：非 public 方法不会生效、同一类内调用不会生效、异常被 catch 吞掉不会生效。很多人只知道「加 @Transactional 就万事大吉」，不知道 Spring 在底层做了多少工作。

### 常见错误

1. **❌ 以为 @Transactional 可以加在任何方法上**——只有 public 方法事务才会生效
2. **❌ 不知道 rollbackFor 的默认值**——默认只对 RuntimeException 回滚，checked exception 不会回滚
3. **❌ 误以为 Isolation 级别设得越高越好**——READ_UNCOMMITTED 性能最高，要根据业务场景选择

## 五、Spring Boot 自动配置（8 题）

| 题目 | 频率 | 难度 | 预计回答时间 |

| @SpringBootApplication 注解解析 | 🔴 高频 | P6 | 3-4 分钟 |
| Spring Boot 自动配置原理 | 🔴 高频 | P6 | 5-8 分钟 |
| Spring Boot Starter 原理 | 🟡 中频 | P6 | 4-5 分钟 |
| 自定义 Starter 实现 | 🟡 中频 | P6 | 5-6 分钟 |
| Spring Boot 启动流程 | 🟡 中频 | P5 | 3-4 分钟 |

### 面试官视角

Spring Boot 是现代 Java 开发的基础，但很多候选人只会用它，**不会解释为什么这样配置**。面试官经常从以下问题切入：

> 「@EnableAutoConfiguration 注解中有个 exclude 属性是干什么用的？为什么可以排除某个自动配置类？」

这道题考察的是你对 Spring Boot 自动配置机制的理解深度。如果你能回答「Spring Boot 通过 spring.factories 文件加载配置类，exclude 属性会将配置类从候选列表中移除」，面试官会认为你对框架有深入研究。

### 常见错误

1. **❌ 把 @SpringBootApplication 当作普通注解**——它是 @Configuration、@EnableAutoConfiguration、@ComponentScan 三个注解的组合
2. **❌ 以为自定义 Starter 可以随意命名**——约定大于配置，Starter 的命名有规范
3. **❌ 不了解自动配置的 conditional 注解**——@ConditionalOnClass、@ConditionalOnMissingBean 等注解控制配置是否生效

## 学习路径建议

### 零基础快速入门（1-2 天）

1. 先阅读 [IoC 与 DI 原理](/questions/spring/ioc-di) 和 [Bean 生命周期](/questions/spring/bean-lifecycle)，理解 Spring 的核心机制
2. 阅读 [AOP 原理与代理模式](/questions/spring/aop) 和 [@Transactional 失效场景](/questions/spring/transaction-failure)，掌握 Spring 代理机制
3. 阅读 [Spring Boot 自动配置原理](/questions/spring/auto-configuration)，理解 Spring Boot 的核心设计

### 中级进阶（3-5 天）

1. 深入阅读 [循环依赖与三级缓存](/questions/spring/circular-dependency) 和 [为什么需要三级缓存](/questions/spring/three-level-cache)，理解 Spring 解决循环依赖的设计思路
2. 阅读 [BeanPostProcessor 原理](/questions/spring/bean-post-processor)，掌握 Spring 的扩展点机制
3. 阅读 [事务传播行为](/questions/spring/transaction-propagation)，理解事务的嵌套行为

### 高级深化（5-7 天）

1. 阅读 [通知执行顺序](/questions/spring/advice-order)，理解 AspectJ 优先级机制
2. 阅读 [自定义 Starter 实现](/questions/spring/custom-starter)，动手实现一个自己的 Starter
3. 阅读 [Spring Boot 启动流程](/questions/spring/startup-process)，从源码层面理解 Spring Boot 的启动过程
