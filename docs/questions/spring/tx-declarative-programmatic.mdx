# 声明式事务与编程式事务

**目标级别**：P5/P6

## 开场：两种事务管理方式

面试官问：「声明式事务和编程式事务有什么区别？各有什么优缺点？」你说：「声明式事务用注解，编程式事务用代码。」面试官追问：「那什么时候应该用编程式事务？」

声明式事务和编程式事务是 Spring 提供的两种事务管理方式。理解它们的区别和使用场景，才能根据业务需求选择合适的方案。

## 面试官最关心的 3 个问题（快速自测）

1. **🟡 声明式事务和编程式事务的核心区别是什么？**
2. **🟡 什么情况下需要使用编程式事务？**
3. **🟡 TransactionTemplate 和 TransactionCallback 怎么使用？**

## 一、两种事务管理方式对比

### 1.1 基本概念

| 方式 | 说明 | 特点 |
|------|------|------|
| 声明式事务 | 使用 @Transactional 注解 | 非侵入式，配置简单 |
| 编程式事务 | 使用 TransactionTemplate 或 TransactionManager | 侵入式，控制精确 |

### 1.2 代码对比

**声明式事务**：

```java
@Service
public class UserService {
    
    @Transactional
    public void saveUser(User user) {
        userDao.save(user);
        orderService.createDefaultOrder(user);
    }
}
```

**编程式事务**：

```java
@Service
public class UserService {
    
    @Autowired
    private TransactionTemplate transactionTemplate;
    
    public void saveUser(User user) {
        transactionTemplate.executeWithoutResult(status -> {
            userDao.save(user);
            orderService.createDefaultOrder(user);
        });
    }
}
```

## 二、声明式事务

### 2.1 基本用法

```java
@Service
public class UserService {
    
    @Transactional(
        propagation = Propagation.REQUIRED,
        isolation = Isolation.DEFAULT,
        rollbackFor = Exception.class
    )
    public void saveUser(User user) {
        userDao.save(user);
    }
}
```

### 2.2 优缺点

| 优点 | 缺点 |
|------|------|
| 非侵入式 | 细粒度控制困难 |
| 配置简单 | 异常处理不灵活 |
| 代码简洁 | 自调用问题 |

## 三、编程式事务

### 3.1 TransactionTemplate

```java
@Service
public class UserService {
    
    @Autowired
    private TransactionTemplate transactionTemplate;
    
    public void saveUser(User user) {
        // 无返回值
        transactionTemplate.executeWithoutResult(status -> {
            userDao.save(user);
            orderService.createDefaultOrder(user);
        });
    }
    
    public Long saveUserAndGetId(User user) {
        // 有返回值
        return transactionTemplate.execute(status -> {
            userDao.save(user);
            return user.getId();
        });
    }
}
```

### 3.2 TransactionCallback

```java
@Service
public class UserService {
    
    @Autowired
    private TransactionTemplate transactionTemplate;
    
    public void complexTransaction(User user, Order order) {
        transactionTemplate.execute(new TransactionCallbackWithoutResult() {
            @Override
            protected void doInTransactionWithoutResult(TransactionStatus status) {
                try {
                    userDao.save(user);
                    orderDao.save(order);
                } catch (Exception e) {
                    // 手动回滚
                    status.setRollbackOnly();
                }
            }
        });
    }
}
```

### 3.3 手动控制事务

```java
@Service
public class UserService {
    
    @Autowired
    private PlatformTransactionManager transactionManager;
    
    public void saveUser(User user) {
        TransactionStatus status = transactionManager.getTransaction(
            new DefaultTransactionDefinition());
        
        try {
            userDao.save(user);
            orderService.createDefaultOrder(user);
            transactionManager.commit(status);
        } catch (Exception e) {
            transactionManager.rollback(status);
            throw e;
        }
    }
}
```

### 3.4 优缺点

| 优点 | 缺点 |
|------|------|
| 控制精确 | 代码侵入 |
| 异常处理灵活 | 代码复杂 |
| 适合复杂场景 | 容易出错 |

## 四、使用场景对比

### 4.1 声明式事务适用场景

```java
// 简单 CRUD 操作
@Service
public class UserService {
    
    @Transactional
    public void save(User user) {
        userDao.save(user);
    }
    
    @Transactional
    public void update(User user) {
        userDao.update(user);
    }
    
    @Transactional
    public void delete(Long id) {
        userDao.delete(id);
    }
}
```

### 4.2 编程式事务适用场景

```java
// 复杂业务逻辑
@Service
public class OrderService {
    
    public void createOrder(Order order) {
        transactionTemplate.execute(status -> {
            try {
                // 保存订单
                orderDao.save(order);
                
                // 扣减库存
                for (OrderItem item : order.getItems()) {
                    inventoryService.deduct(item.getProductId(), item.getQuantity());
                }
                
                // 扣减余额
                accountService.deduct(order.getUserId(), order.getAmount());
                
                // 发送消息（失败不影响主事务）
                try {
                    messageService.sendOrderCreated(order);
                } catch (Exception e) {
                    log.error("发送订单消息失败", e);
                }
                
            } catch (Exception e) {
                // 手动回滚
                status.setRollbackOnly();
                throw e;
            }
            
            return null;
        });
    }
}
```

## 五、源码解析

### 5.1 TransactionTemplate 执行流程

```java title="TransactionTemplate.java"
public class TransactionTemplate extends DefaultTransactionDefinition {
    
    public <T> T execute(TransactionCallback<T> action) {
        TransactionStatus status = transactionManager.getTransaction(this);
        
        try {
            T result = action.doInTransaction(status);
            transactionManager.commit(status);
            return result;
        } catch (RuntimeException ex) {
            rollbackOnException(status, ex);
            throw ex;
        } catch (Error err) {
            rollbackOnException(status, err);
            throw err;
        } catch (Exception ex) {
            // 其他异常，根据 rollbackOn 决定是否回滚
            if (transactionManager.shouldRollbackOn(ex)) {
                transactionManager.rollback(status);
            } else {
                transactionManager.commit(status);
            }
            throw ex;
        }
    }
}
```

### 5.2 声明式事务拦截器

```java title="TransactionInterceptor.java"
public class TransactionInterceptor extends TransactionAspectSupport {
    
    @Override
    public Object invoke(MethodInvocation invocation) throws Throwable {
        TransactionAttribute attr = getTransactionAttribute(
            invocation.getMethod(), invocation.getThis().getClass());
        
        TransactionInfo info = createTransactionIfNecessary(attr, invocation.getMethod());
        
        try {
            return invocation.proceed();
        } catch (Throwable ex) {
            completeTransactionAfterThrowing(info, ex);
            throw ex;
        } finally {
            cleanupTransactionInfo(info);
        }
    }
}
```

## 六、面试高频追问

### 追问链 1：性能对比

> **第一层**：声明式事务和编程式事务性能有差异吗？
> 
> 几乎没有差异，都是基于 AOP 代理。

> **第二层**：那为什么还要用编程式事务？
> 
> 为了更灵活地控制事务边界和异常处理。

> **第三层**：AOP 代理有什么性能开销？
> 
> 代理对象的创建和方法调用有少量开销，但现代 JVM 优化后影响很小。

### 追问链 2：异常处理

> **第一层**：编程式事务如何手动回滚？
> 
> 使用 `status.setRollbackOnly()` 标记回滚。

> **第二层**：setRollbackOnly 和 transactionManager.rollback 有什么区别？
> 
> setRollbackOnly 只是标记状态，commit 时会回滚；rollback 会立即回滚。

> **第三层**：什么时候用 setRollbackOnly？
> 
> 在 catch 块中处理异常后，标记事务回滚但不想抛异常。

## 七、常见错误与陷阱

### 错误 1：异常被吞掉

```java
@Service
public class BadService {
    
    @Autowired
    private TransactionTemplate template;
    
    public void save() {
        template.execute(status -> {
            try {
                userDao.save(user);
            } catch (Exception e) {
                // ⚠️ 异常被吞掉，事务可能不会回滚
                return null;
            }
            return null;
        });
    }
}

// 正确做法
public void save() {
    template.execute(status -> {
        userDao.save(user);  // 不要捕获异常，让框架处理
        return null;
    });
}
```

### 错误 2：忘记配置事务管理器

```java
@Configuration
public class Config {
    
    @Bean
    public TransactionTemplate transactionTemplate(
            PlatformTransactionManager transactionManager) {
        TransactionTemplate template = new TransactionTemplate(transactionManager);
        template.setIsolationLevel(TransactionDefinition.ISOLATION_DEFAULT);
        template.setPropagationBehavior(TransactionDefinition.PROPAGATION_REQUIRED);
        return template;
    }
}
```

### 错误 3：事务范围过大

```java
@Service
public class BadService {
    
    @Transactional  // ⚠️ 整个方法在一个事务中
    public void batchImport(List<ExcelRow> rows) {
        for (ExcelRow row : rows) {
            processRow(row);  // 大量数据，长事务
        }
    }
}

// 正确做法
@Service
public class GoodService {
    
    public void batchImport(List<ExcelRow> rows) {
        for (ExcelRow row : rows) {
            transactionTemplate.execute(status -> {
                processRow(row);
                return null;
            });
        }
    }
}
```

## 八、对比总结

### 特性对比

| 特性 | 声明式事务 | 编程式事务 |
|------|-----------|-----------|
| 代码侵入 | 无 | 高 |
| 配置复杂度 | 低 | 高 |
| 控制粒度 | 粗 | 细 |
| 异常处理 | 自动 | 手动 |
| 适合场景 | 简单 CRUD | 复杂业务 |
| 可读性 | 好 | 差 |
| 可维护性 | 高 | 低 |

### 选择建议

| 场景 | 推荐方式 |
|------|---------|
| 简单 CRUD | 声明式 |
| 复杂业务逻辑 | 编程式 |
| 需要细粒度控制 | 编程式 |
| 需要条件回滚 | 编程式 |
| 大多数场景 | 声明式 |

## 九、实战应用

### 9.1 混合使用

```java
@Service
public class OrderService {
    
    @Autowired
    private TransactionTemplate template;
    
    @Transactional
    public void createOrder(Order order) {
        // 主业务使用声明式事务
        orderDao.save(order);
        inventoryService.deduct(order.getItems());
        
        // 发送消息使用编程式事务
        template.executeWithoutResult(status -> {
            try {
                messageService.sendOrderCreated(order);
            } catch (Exception e) {
                // 消息发送失败不影响主事务
                log.error("发送订单消息失败", e);
            }
        });
    }
}
```

### 9.2 条件回滚

```java
@Service
public class UserService {
    
    @Autowired
    private TransactionTemplate template;
    
    public void registerUser(User user) {
        template.execute(status -> {
            userDao.save(user);
            
            // 根据业务条件决定是否回滚
            if (user.isVip() && !paymentService.charge(user)) {
                status.setRollbackOnly();
                throw new BusinessException("VIP 用户扣费失败");
            }
            
            return null;
        });
    }
}
```

> **💡 加分回答**：Spring Boot 2.2+ 提供了 `TransactionManager` 的自动配置，使用编程式事务更加简单。
