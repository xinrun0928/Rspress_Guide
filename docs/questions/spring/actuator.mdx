# Spring Boot Actuator 监控

**目标级别**：P5

## 开场：生产环境监控的重要性

面试官问：「Spring Boot Actuator 是什么？它能做什么？」你说：「是 Spring Boot 的监控功能。」面试官追问：「它有哪些常用的端点？如何保护这些端点？」

Spring Boot Actuator 是 Spring Boot 提供的生产级监控功能，在微服务架构中尤为重要。

## 面试官最关心的 3 个问题（快速自测）

1. **🟡 Spring Boot Actuator 的核心端点有哪些？**
2. **🟡 如何暴露 Actuator 端点？**
3. **🟡 如何保护 Actuator 端点？**

## 一、核心端点

### 1.1 内置端点

| 端点 | 说明 | 默认开启 |
|------|------|---------|
| /actuator/health | 应用健康检查 | ✅ |
| /actuator/info | 应用信息 | ✅ |
| /actuator/beans | 容器中所有 Bean | ❌ |
| /actuator/conditions | 自动配置条件 | ❌ |
| /actuator/configprops | 配置属性 | ❌ |
| /actuator/env | 环境变量 | ❌ |
| /actuator/mappings | URL 映射 | ❌ |
| /actuator/metrics | 指标信息 | ❌ |
| /actuator/threaddump | 线程信息 | ❌ |
| /actuator/heapdump | 堆转储 | ❌ |

### 1.2 端点配置

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics
        exclude: env,beans
  endpoint:
    health:
      show-details: when-authorized
  health:
    db:
      enabled: true
    redis:
      enabled: true
```

## 三、自定义端点

### 3.1 定义端点

```java
@Endpoint(id = "custom")
public class CustomEndpoint {
    
    @ReadOperation
    public Map<String, Object> info() {
        Map<String, Object> info = new HashMap<>();
        info.put("timestamp", System.currentTimeMillis());
        info.put("version", "1.0.0");
        return info;
    }
    
    @WriteOperation
    public String reload(String config) {
        return "Configuration reloaded: " + config;
    }
}
```

### 3.2 注册端点

```java
@Configuration
public class ActuatorConfig {
    
    @Bean
    @ConditionalOnAvailableEndpoint
    public CustomEndpoint customEndpoint() {
        return new CustomEndpoint();
    }
}
```

## 四、面试高频追问

### 追问链 1：端点安全

> **第一层**：如何保护 Actuator 端点？
> 
> 使用 Spring Security 配置端点访问权限。

> **第二层**：如何限制 health 端点显示详情？
> 
> 设置 `show-details: when-authorized`。

## 五、常见错误与陷阱

### 错误 1：暴露敏感端点

```yaml
# ⚠️ 错误：生产环境不应暴露所有端点
management:
  endpoints:
    web:
      exposure:
        include: "*"
```

## 六、对比总结

### 端点暴露方式对比

| 方式 | 说明 |
|------|------|
| web | 通过 HTTP 暴露 |
| jmx | 通过 JMX 暴露 |

> **💡 加分回答**：Spring Boot 3.0 改进了 Actuator 的安全配置，使用 `server.error.include-message` 等属性控制敏感信息暴露。
