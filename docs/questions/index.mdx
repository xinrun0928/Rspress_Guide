# 面试题库与实战复盘

## 目录概述

这道面试题库是按照 Java 工程师面试的高频模块逐一拆解的实战手册。

每个主题不只给你"标准答案"，而是站在面试官视角，告诉你：

- 这道题面试官真正想问的是什么
- 为什么你答对了却没拿到 offer
- 追问的边界在哪里，什么深度才叫"过关"
- 生产环境中这个问题背后有哪些坑

面试题不是背的，是理解之后能闭环讲的。如果你发现某个问题你能背出答案但讲不清楚原理，那这道题在真正的面试中仍然是你的软肋。

【面试官手记】
我面试了超过一千个候选人，发现一个规律：能通过面试的人，不是因为他知道得多，而是因为他说得清。能清晰解释一个知识点的人，说明他真的做过、踩过坑、复盘过。而那些只会背概念的人，往往在第一轮追问后就崩了。这道题库的每一篇文章，都是在帮你把"知道"变成"讲清楚"。

### Redis 🔴

Redis 的面试重点在于"数据结构 → 持久化 → 缓存策略 → 集群"这条链路的完整闭环，任何一个环节断了都会被追问到崩溃。

### JVM 虚拟机 🟡

JVM 部分主要覆盖内存模型、垃圾回收、类加载机制，是理解 Java 程序运行本质的关键。

### 分布式与系统设计 🟡

分布式相关面试题涉及 CAP/BASE 理论、一致性算法、分布式事务、消息队列等，是 P7 候选人的核心考察区。

### 场景设计题 🟡

系统设计类题目没有标准答案，考的是你在约束条件下的权衡能力和架构思维。

## 学习路径指引

### 新手通关路径（3-6 个月准备周期）

**第一阶段：Java 基础与集合（第 1-2 个月）**

Java 基础和集合是面试的第一关，也是最容易轻视的模块。建议先从 [equals 与 hashCode](/questions/java-basic/equals) 和 [HashMap put 流程](/questions/collection/hashmap-put) 开始，这两个是集合部分的核心枢纽。

**第二阶段：Spring 与框架（第 2-3 个月）**

Spring 的核心是 IOC/AOP 和 Bean 生命周期。[循环依赖与三级缓存](/questions/spring/circular-dependency) 是高频深水区，值得反复研究。

**第三阶段：并发编程（第 3-4 个月）**

并发是 P6 和 P7 的分水岭。[AQS 原理](/questions/concurrent/aqs) 和 [锁升级流程](/questions/concurrent/lock-upgrade) 是两个必须攻克的硬骨头。

**第四阶段：MySQL 与 Redis（第 4-5 个月）**

数据库部分建议先建索引体系认知，再深入事务机制。[MVCC 原理](/questions/mysql/mvcc) 是 InnoDB 的核心，也是面试高频深水区。

**第五阶段：JVM 与分布式（第 5-6 个月）**

JVM 偏原理，分布式偏架构。这两块是 P7 候选人的必考区，也是区分"用过"和"理解过"的关键。

### 针对性强化路径

| 场景 | 推荐阅读顺序 |
|------|-------------|
| 校招 P5 突击 | Java 基础 → 集合 → Spring IOC → MySQL 索引 → Redis 基本类型 |
| 社招 P6 突击 | 集合源码 → 并发原理 → Spring 源码 → MySQL 事务/锁 → Redis 持久化/集群 |
| P7 架构升级 | AQS → Spring 生态 → 分布式事务 → 系统设计 → 场景模拟 |

:::tip 💡
面试题库不是背的，是理解之后能用自己的话讲出来才算过关。每一篇文章看完后，试着向自己复述三遍：如果我是面试官，我会追问什么？这个问题的边界条件是什么？生产环境这个问题会导致什么后果？
:::

## 面试题分级速查

| 级别 | 核心考察点 | 过关标准 |
|------|-----------|----------|
| P5 | 基本概念、API 使用、常见区别 | 能背出流程，不怵简单追问 |
| P6 | 源码阅读、原理机制、生产问题 | 能回答追问，不怵连环追问 |
| P7 | 方案选型、架构权衡、生产复盘 | 有实战案例，能做 trade-off |
