# 系统设计 20 题

**目标级别**：P6/P7

---

面试官问：「设计一个秒杀系统」——这不是在考你知道多少高并发技术，而是在考你面对资源约束时，如何做出正确的技术权衡。

系统设计题是 P6+ 面试的标志性题型。它不像基础题那样有标准答案，而是要你在不确定性和时间压力下，展现自己的工程判断力。面试官真正想看的是：你能不能把一个模糊的需求拆解成具体的约束？你能不能在多个方案里说出 trade-off？你能不能在讨论中调整自己的思路？

本章节精选 20 道高频系统设计题，覆盖秒杀、短链、即时通讯、Feed 流、分布式 ID 等核心场景。每个题目都配有追问链、对比表格和避坑指南，帮助你在面试中展现出超出 P6 的技术深度。

## 学习路径建议

### 第一阶段：打牢基础（建议 3 天）

1. 先读「系统设计面试框架」，理解面试官想考什么
2. 掌握「容量估算方法」和「延迟数据速算表」，这是所有系统设计的起点
3. 完成基础篇的练习题

### 第二阶段：吃透核心场景（建议 5 天）

1. **秒杀系统**：理解高并发读写的核心矛盾
2. **短链系统**：理解 ID 生成与存储的权衡
3. **IM 系统**：理解长连接与消息可靠性的设计
4. **Feed 流**：理解数据分发与存储的取舍

### 第三阶段：拓展进阶场景（建议 3 天）

1. **分布式 ID**：雪花算法的原理与变体
2. **排行榜系统**：有序集合的设计
3. **配置中心/任务调度**：理解元数据管理的挑战

## 常见问题

### Q：系统设计题有没有标准答案？

没有。系统设计题考察的是你的思考过程，而不是某个「正确答案」。面试官会根据你的讨论表现打分：能不能正确理解需求、能不能识别关键约束、能不能分析 trade-off、能不能在讨论中接受反馈。

### Q：需要准备多少道题才能应付面试？

核心 5 道（秒杀、短链、IM、Feed、分布式 ID）必须精通，其他题目理解思路即可。P6 面试一般问 1-2 道系统设计题。

### Q：面试中卡住了怎么办？

可以向面试官求助：「这个系统的规模大概是多少？」或者「我需要考虑哪些约束？」好的面试是协作讨论，不是独角戏。展现出你愿意沟通、愿意在不确定中做假设的特质，比硬撑着答更重要。

---

> 💡 **面试官视角**：系统设计题考察的核心不是你知道多少高并发技术，而是你面对「不确定」时如何做决策。面试官会故意不给你明确的需求，让你自己去问、自己去做假设。你的提问质量和假设合理性，直接决定面试结果。
