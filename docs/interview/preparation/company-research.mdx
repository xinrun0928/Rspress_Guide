# 目标公司 / 岗位调研方法

> 「你们公司是做什么的？」——如果你在面试中被这个问题问住，这个 offer 基本就悬了。

这不是刁难，这是**诚意测试**。面试官想看的是：你是在认真考虑这个机会，还是在海投简历。

很多人海投简历，投出去就忘了自己投了哪家；等到 HR 打来电话，才手忙脚乱地回忆公司是做什么的。这种状态去面试，怎么可能成功？

**精准投递的前提是精准调研。** 花 20 分钟深入了解一家公司，远比花 20 分钟投 10 家公司更有效。

## 一、调研的三个层次

### 层次一：基础信息（必须掌握）

在面试前，你必须知道：

- 这家公司**做什么业务**？核心产品是什么？
- 公司的**发展阶段**是什么？（融资阶段、上市情况、市场地位）
- 你要投的**岗位**在哪个部门？做什么业务？

这些信息通过公司官网、招聘 JD、天眼查、脉脉等平台就能获取。

### 层次二：深度信息（拉开差距）

掌握基础信息后，你需要更进一步：

- 公司的**技术栈**是什么？（GitHub、招聘 JD、技术博客里能找到）
- 公司的**技术挑战**是什么？（ CTO 访谈、技术分享、招聘 JD 里的要求）
- 公司最近有什么**技术动态**或**业务动态**？（新闻、公司公众号）

这些信息能让你在面试中展示「我真的了解这家公司」，而不是泛泛而谈。

### 层次三：文化信息（决定匹配度）

最后，你需要了解软性信息：

- 公司的工作**文化**是什么？（加班情况、团队氛围、技术分享传统）
- 这个岗位的**成长空间**如何？
- 直属领导的**风格**如何？

通过脉脉、牛客、在职员工打听等方式获取。

## 二、调研渠道清单

### 官方渠道

| 渠道 | 获取什么信息 |
|---|---|
| 公司官网 | 业务介绍、发展历程、团队介绍 |
| 公司公众号 / 技术博客 | 技术动态、团队博客、技术方案 |
| 招聘 JD | 岗位要求、技术栈、工作内容 |
| IPO 招股书（上市公司） | 财务数据、业务结构、发展战略 |

### 社区渠道

| 渠道 | 获取什么信息 |
|---|---|
| 脉脉 | 公司文化、员工评价、薪资范围 |
| 牛客网 | 面试经验分享、offer 待遇 |
| Boss 直聘 / 拉勾 | 直接和 HR / 员工沟通的机会 |
| 天眼查 / 企查查 | 公司工商信息、融资历史 |
| GitHub / 技术博客 | 技术实力、开源项目 |

### 社交渠道

| 渠道 | 获取什么信息 |
|---|---|
| LinkedIn | 团队成员背景、团队规模 |
| 朋友 / 同学打听 | 第一手的内部信息 |
| 技术社群 | 在职员工的口碑和评价 |

## 三、按岗位类型定制调研

### 投大厂（BAT/TMD 等）

大厂的信息通常很透明，重点关注：

- **部门选择**：同一个公司，不同部门的差距巨大。选择核心业务部门还是边缘部门，决定了你未来的成长空间。
- **岗位匹配度**：JD 里写的技术栈和你简历里的匹配程度如何？
- **面试风格**：大厂通常有标准化的面试流程，可以在牛客上找同岗位的面试经验。

> **调研重点**：部门 > 团队 > 岗位

### 投中型公司（B轮-C轮）

中型公司信息不如大厂透明，重点关注：

- **业务前景**：融到哪一轮了？投资方是谁？业务增长情况如何？
- **技术栈**：有没有技术债务？技术团队规模如何？
- **稳定性**：公司现金流怎么样？有没有裁员历史？

> **调研重点**：业务前景 > 技术栈 > 稳定性

### 投创业公司（天使-A轮）

创业公司风险最高，重点关注：

- **赛道**：这个方向是不是长期有价值的？
- **团队背景**：创始人是什么背景？核心团队靠不靠谱？
- **商业模式**：公司怎么赚钱？有没有清晰的盈利路径？

> **调研重点**：赛道 > 团队 > 商业模式

## 四、JD 分析：读懂岗位要求

JD（Job Description）不只是岗位描述，它是一个**考试大纲**。分析 JD 是面试准备的第一步。

### 关键词拆解

把 JD 里出现的技术栈、工具、关键词列出来，和自己的简历对照：

- JD 里反复出现的词 = 高频考点
- JD 里的「加分项」= 你可以提前准备的亮点
- JD 里的「必须」= 你必须掌握的硬技能

### 反推面试问题

JD 分析透了，你就能反推出面试官会问什么：

> JD 写：「有高并发系统设计经验」 → 面试官可能问：你做的系统 QPS 是多少？怎么保证高可用的？
>
> JD 写：「熟悉 Redis 缓存」 → 面试官可能问：缓存穿透怎么解决？缓存和数据库一致性怎么处理？

### 匹配度自检

面试前问自己三个问题：

1. JD 里的「必须项」，我有没有对应的经历？
2. JD 里的「加分项」，我有没有能说的亮点？
3. JD 里没写但我会的技术，能不能找到合适的机会提？

## 五、调研结果的使用场景

### 自我介绍中展示

> 「我了解到贵司的交易系统目前面临 XX 挑战，这和我之前在 XX 公司做的 XX 项目高度相关……」

这比「我对贵司很有兴趣」有力一万倍。

### 反向提问中展示

面试结尾「你有什么想问我的」环节，可以问出有深度的问题：

> 「我看到贵司最近在做 XX 技术的升级，这个过程中最大的挑战是什么？」
>
> 「这个岗位的团队目前最大的技术债务是什么？有计划逐步解决吗？」

问出这种问题的前提是你做了调研。

### 谈薪时使用

了解公司的盈利情况和行业薪资水平，是你谈薪的底气：

> 「根据我的调研，这个岗位的市场价在 XX 万左右，结合我的经验和这个岗位的匹配度，我的期望是……」

## 六、调研 Checklist

面试前，确保以下信息你都已掌握：

- [ ] 公司做什么业务？（能用一句话说清楚）
- [ ] 公司的技术栈是什么？
- [ ] 你要去的部门是核心还是边缘？
- [ ] 公司目前处于什么发展阶段？
- [ ] 这个岗位的核心挑战是什么？
- [ ] 公司的文化是狼性还是弹性？
- [ ] 有没有在脉脉 / 牛客上找到相关面试经验？
- [ ] 有没有联系到在职员工打听内部情况？

---

## 最后

海投一份简历的成功率趋近于零。**精准投递的前提，是精准调研。**

每次面试前，花 20-30 分钟把调研做透。这 20 分钟，抵得上你多刷 50 道算法题的价值。

因为当你说出「我对贵司的 XX 业务很感兴趣」的时候，面试官知道你是认真的。而认真的候选人，永远比敷衍的候选人更受欢迎。
