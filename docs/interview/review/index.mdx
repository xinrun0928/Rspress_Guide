# 面试经验与复盘

面试凉了就凉了？

不。面试是「投入最小、反馈最快」的训练场——每一场面试，无论结果如何，都能让你更清楚地认识自己。

区别在于：你愿不愿意复盘。

有些人面了 20 场，每一场都在犯同样的错误。有些人面了 5 场，却能从每场中发现自己的问题并逐一改进。后者拿到 offer 的速度，往往比前者快得多。

复盘不是为了否定自己，而是**把每一次经历都变成下一次更好的垫脚石**。

## 复盘的正确姿势

很多人复盘是写流水账：「今天面了 XX 公司，算法题没做出来，结果凉了。」

这不是复盘，这是日记。

真正的复盘要回答三个问题：

```
1. 这次面试中，我表现好的是什么？（找到自己的优势）
2. 这次面试中，我暴露了什么问题？（找准自己的短板）
3. 针对这些问题，下一步具体做什么？（制定可执行的改进计划）
```

复盘不求全面，但求具体。与其写「算法还需要加强」，不如写「二分查找的边界条件处理不熟，需要把「寻找左侧边界」和「寻找右侧边界」两种模板各练 5 遍」。


## 面试复盘 Checklist

每次面试结束后，趁记忆还新鲜，24 小时内完成复盘：

```
□ 这场面试，面试官问了哪些类型的问题？
□ 哪道题/哪个问题我回答得最好？为什么？
□ 哪道题/哪个问题我没答好？卡在哪里？
□ 面试过程中，我的表达是否清晰？有没有词不达意？
□ 如果重来一次，有哪些地方可以做得更好？
□ 针对这些问题，下周我要做什么？
□ 有没有和面试官交流中，得到什么有价值的反馈？
□ 我对这家公司的业务/技术栈是否有了更深的了解？
```

不要等面试凉了才复盘。即使面试感觉良好，也要复盘——优势需要强化，良好需要变成优秀。


## 本模块核心公式

```
面试成长 = 每次面试 + 及时复盘 + 具体改进 + 刻意练习
```

一次深刻的复盘，胜过多场无思考的面试。

把每一次面试都当作一次「发现自己」的机会，不是考核，是成长。
