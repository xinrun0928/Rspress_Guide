# 离职原因与职业规划的表述技巧

> 「你为什么离职？」「你未来 3-5 年的规划是什么？」——这两道题回答不好，HR 直接在心里把你归类为「没想清楚的人」。

很多人把这两道题当成独立的 HR 问题来准备，但它们有一个共同本质：**都在考察你的稳定性。** HR 的底层逻辑是：你会不会在我们这也干不久？

## 一、离职原因：得体的表达方式

### 为什么「说真话」不一定是对的

我知道你可能会想：「我就实话实说呗，前公司确实烂。」

但问题是——HR 听到的不是「你前公司有多烂」，而是：

- 「这个人怨气很重」
- 「这个人会不会也这样评价我们公司」
- 「这个人是不是一个喜欢抱怨的人」

公司 HR 之间的圈子比你想象的小。你今天抱怨前公司，明天可能就会抱怨现公司。这个印象一旦形成，offer 就悬了。

**核心原则：正向表达，不抱怨，不撒谎。**

### 六类离职原因的分类应对

**类型一：钱没给够**

**❌ 直接说：**
> 「工资太低了，几年没涨薪。」

**✅ 正确姿势：**

不直接说「钱少」，而是把「薪资」和「价值」挂钩。

> 「在原公司我承担了更多的职责，薪资却没有相应的调整。这也让我开始重新思考自己的市场价值，希望找到一个能认可我贡献的平台。」

---

**类型二：领导不行**

**❌ 直接说：**
> 「领导能力差，不懂技术瞎指挥。」

**✅ 正确姿势：**

把矛盾包装成「发展诉求不一致」。

> 「我和直属领导在技术理念上有些分歧，我更希望能在技术深度上有更多探索。考虑到长期发展，我觉得换一个环境可能更适合我。」

---

**类型三：加班严重 / 996**

**❌ 直接说：**
> 「公司 996 受不了了，身体扛不住。」

**✅ 正确姿势：**

把「被迫离开」变成「主动选择」，强调你需要可持续的工作方式。

> 「上家公司节奏比较激进，短期内出活儿没问题，但长期来看，我担心这种模式不可持续。我希望找到一个能让我保持高效、同时兼顾成长的环境。高效和可持续，对我来说同样重要。」

---

**类型四：公司裁员 / 业务调整**

这是最不需要「编」的理由，实话实说就行。

> 「公司去年做了一轮战略调整，裁撤了整个产品线，我所在的团队也在其中。这段时间我重新梳理了自己的方向，明确了下一份工作我最看重的是什么。」

---

**类型五：晋升遇到瓶颈**

> 「在上一家公司，我负责的方向已经非常成熟了，继续待下去成长空间有限。我希望能找到一个有挑战的舞台，让自己的技术能力和管理能力都有发挥空间。」

---

**类型六：职业转型**

> 「过去几年我在 XX 领域积累了不少经验，但我越来越看好 XX 方向的发展。我花了一年半的时间自学和做项目，现在准备好了切换赛道。贵司的 XX 岗位正好是我准备已久的。」

**关键点**：转行的话，要展示你**已经做了充分的准备**，而不是「还没想清楚就跳了」。

### 绝对不能说出口的话

| 绝对不要说 | 原因 |
|---|---|
| 「领导是傻 X」 | HR 会想：你以后是不是也会这样说我 |
| 「同事内卷，勾心斗角」 | 暗示你处理不好人际关系 |
| 「公司拖欠工资 / 社保」 | 暗示你当初选择公司的时候判断力有问题 |
| 「就是为了钱」 | 太功利，显得你只认钱 |
| 「工作太累了」 | HR 会觉得你不够抗压 |

## 二、职业规划：技术路线 vs 管理路线

### 为什么 HR 要问职业规划

表面上是问「你想干什么」，实际上是看三件事：

1. **你对自己有没有清醒的认知**——你知道自己的长处吗？知道自己不擅长什么吗？
2. **你的野心和岗位匹不匹配**——你想升职加薪，但这个岗位能给你吗？你会不会把我们当跳板？
3. **你的规划和我们有没有交集**——你的规划在这个公司能实现吗？

### 技术路线 vs 管理路线，怎么选

**技术路线**适合对技术有热情、享受「解决问题」本身的人。回答示例：

> 「短期（1-2年）内，我希望能在一个技术方向上继续深耕，比如分布式系统或者中间件方向，争取成为团队在这个领域的核心力量。中期（3-5年），我希望能够承担架构设计的工作，从『能完成功能』升级到『能定义系统』。」

**管理路线**适合喜欢影响他人、享受「通过别人拿结果」的人。回答示例：

> 「短期（1-2年）内，我会继续打磨技术深度，同时开始尝试带人的工作，从 code review 和 technical design 开始，积累技术管理的感觉。中期（3-5年），我希望能够独立负责一个小团队。」

### 如果你不确定

> 「说实话，我目前更倾向于在技术深度上继续发展，我觉得在这个阶段我还是想把技术底子打扎实。但我也愿意保持开放心态，如果团队有需要，我也愿意尝试技术管理的工作。」

### 不同年限的程序员，职业规划怎么说

**1-3 年**：重点是**「在哪个方向扎根」**，而不是「要成为什么」。

> 「未来两年，我希望能在一个技术方向上（比如高并发系统或数据存储）扎下根来。具体来说，我希望从能独立完成开发任务，到能独立负责一个中型模块的设计和实现。」

**3-5 年**：重点是**「成为某个领域的专家」**。

> 「未来三年，我希望能在我专注的方向（分布式系统）上形成自己的深度。具体来说，我想从『能完成功能』升级到『能主导方案』，能够独立输出一个系统的技术方案并推动落地。」

**5-8 年**：重点是**「技术影响力」**。

> 「未来五年，我希望能从技术执行者转变为技术影响者。具体来说，我想做到：能定义系统的技术方向，不只是执行；能从架构层面思考业务问题。」

**8 年以上**：重点是**「战略思维」**。

> 「未来几年，我希望能在技术战略层面有所贡献。具体来说，我想做到：能参与产品技术路线图的制定，而不只是执行；能把业务目标和技术决策联系起来。」

## 三、一个完整的离职原因话术模板

如果你有多个原因，可以这样组合：

> 「我在上家公司工作了 X 年，学到了 XX 技能和 XX 经验，完成了 XX 项目。但随着时间推移，我在 XX 方向遇到了瓶颈——不是公司不好，而是我的职业规划需要更多的 XX 机会。了解到贵司的 XX 业务和团队文化，我觉得是一个很好的契合点，所以决定尝试一下。」

这个话术的逻辑：

1. **先肯定前公司**（不得罪，体现你是正常离开的）
2. **说明收获**（展示你不是白混的，有积累）
3. **点出瓶颈**（暗示你是主动选择，不是被迫离开）
4. **表达对新公司的兴趣**（展示动机，有具体信息，不是海投）

## 四、避坑清单

| 踩坑 | 正确 |
|---|---|
| 「我想当 CTO」 | 「我希望在技术深度上有成长空间，最终成为一个有影响力的技术 leader」 |
| 「我还没想好」 | 「我倾向于 XX 路线，但保持开放」 |
| 只说「学习」「成长」 | 给出具体的里程碑和方向 |
| 路线和岗位完全不匹配 | 呼应岗位的成长路径 |
| 过于谦虚，没有野心 | 适度展示你的目标和追求 |
| 把职位当终点 | 把能力成长当终点，职位只是结果 |

## 五、面试官心理分析

HR 问这两道题，其实是在找三个问题的答案：

1. **你是不是一个会主动思考未来的人？** ——没有规划的人，通常也不会主动规划自己的成长。
2. **你的野心和这个岗位匹配吗？** ——如果你想当 CTO，但这个岗位 5 年内都不可能给你这个位置，那你是不会稳定的。
3. **你的规划是不是在吹牛？** ——能说出具体路径的人，是真正想过的；只说大词的人，是没想清楚的。

---

## 六、Checklist

面试前准备离职原因和职业规划：

- [ ] 准备了 2-3 个「真实但不伤人的离职原因」备用
- [ ] 离职原因：正向表达，不抱怨前公司
- [ ] 职业规划有具体路径，不只是大词
- [ ] 规划分阶段：短期（1-2年）+ 中期（3-5年）
- [ ] 路线和岗位成长路径吻合
- [ ] 语气平和，不带怨气
- [ ] 组合话术能流畅表达（不超过 1 分钟）

---

## 最后

把你离职的原因，变成对新公司的期待，而不是对旧公司的控诉。把职业规划，变成对自己成长的真实思考，而不是写给 HR 看的作文。

同样一件事，「前公司太烂了」和「我需要更大的舞台」，给 HR 的感受完全不同。前者让人担心，后者让人期待。**先对自己诚实，面试才能自信。**
