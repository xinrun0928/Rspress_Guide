# 设计消息推送系统

你有没有想过：

- 微信消息是怎么做到秒到的？
- 为什么 App 关闭了还能收到推送？
- 为什么有些消息会延迟好几秒？

这些功能的背后，都是**消息推送系统**在默默工作。

今天，我们来深入探讨消息推送系统的设计与实现。

## 一、推送系统的挑战

### 1.1 推送系统 vs 普通消息系统

```
┌─────────────────────────────────────────────────────────┐
│                   推送系统的特殊性                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  普通消息系统：                                         │
│  - 消息存储在服务器，消费者主动拉取                      │
│  - 消费者在线才能处理                                    │
│                                                         │
│  推送系统：                                             │
│  - 消息主动推送到用户设备                               │
│  - 用户可能不在线，需要离线存储                          │
│  - 需要管理大量长连接                                   │
│  - 需要对接多个厂商通道                                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 技术挑战

```
┌─────────────────────────────────────────────────────────┐
│                 推送系统的技术挑战                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 实时性                                             │
│     - 消息要在毫秒级到达                                │
│     - 长连接保活                                        │
│                                                         │
│  2. 可靠性                                             │
│     - 消息不能丢失                                      │
│     - 不能重复推送                                      │
│                                                         │
│  3. 高并发                                             │
│     - 同时管理数百万长连接                              │
│     - 推送高峰（如春晚红包）                            │
│                                                         │
│  4. 离线消息                                           │
│     - 用户不在线时，消息需要存储                        │
│     - 用户上线后，需要同步离线消息                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```java title="src/main/java/PushGateway.java"
/**
 * 消息推送网关
 */
public class PushGateway {
    
    private MessageRouter router;
    private MessageValidator validator;
    
    /**
     * 推送消息
     */
    public PushResult push(PushRequest request) {
        // 1. 参数校验
        validator.validate(request);
        
        // 2. 生成消息 ID
        String messageId = idGenerator.nextId();
        
        // 3. 路由选择
        List<DeviceInfo> devices = resolveDevices(request);
        
        // 4. 发送结果
        List<PushResult.DeviceResult> results = new ArrayList<>();
        
        for (DeviceInfo device : devices) {
            PushResult.DeviceResult result = router.route(device, messageId, request);
            results.add(result);
        }
        
        // 5. 返回结果
        return new PushResult(messageId, results);
    }
    
    /**
     * 解析目标设备
     */
    private List<DeviceInfo> resolveDevices(PushRequest request) {
        switch (request.getTargetType()) {
            case USER:
                // 获取用户所有设备
                return deviceService.getUserDevices(request.getTargetId());
            case DEVICE:
                // 单个设备
                return Collections.singletonList(
                    deviceService.getDevice(request.getTargetId()));
            case ALIAS:
                // 别名（如手机号、邮箱）
                return deviceService.getDevicesByAlias(
                    request.getTargetId());
            case TAG:
                // 标签（如"活跃用户"）
                return deviceService.getDevicesByTag(request.getTargetId());
            default:
                throw new IllegalArgumentException("Unknown target type");
        }
    }
}
```

```java title="src/main/java/ConnectionManager.java"
/**
 * 长连接管理器
 */
public class ConnectionManager {
    
    /**
     * 使用 Netty 管理长连接
     */
    private EventLoopGroup bossGroup;
    private EventLoopGroup workerGroup;
    private ChannelGroup channelGroup;
    
    /**
     * 设备连接信息缓存
     */
    private LoadingCache<String, DeviceConnection> connectionCache;
    
    /**
     * 用户 ID 到通道的映射
     */
    private LoadingCache<Long, Set<Channel>> userChannels;
    
    /**
     * 建立连接
     */
    public void onConnect(ChannelHandlerContext ctx, ConnectRequest request) {
        // 1. 解析设备信息
        String deviceId = request.getDeviceId();
        long userId = request.getUserId();
        
        // 2. 创建连接信息
        DeviceConnection conn = new DeviceConnection();
        conn.setChannel(ctx.channel());
        conn.setDeviceId(deviceId);
        conn.setUserId(userId);
        conn.setConnectedAt(System.currentTimeMillis());
        
        // 3. 保存连接
        connectionCache.put(deviceId, conn);
        userChannels.get(userId).add(ctx.channel());
        
        // 4. 注册心跳
        registerHeartbeat(ctx, deviceId);
    }
    
    /**
     * 断开连接
     */
    public void onDisconnect(ChannelHandlerContext ctx) {
        DeviceConnection conn = ctx.channel().attr(DEVICE_CONN_KEY).get();
        if (conn != null) {
            // 移除缓存
            connectionCache.invalidate(conn.getDeviceId());
            userChannels.get(conn.getUserId()).remove(ctx.channel());
            
            // 清理离线消息
            handleOffline(conn.getUserId());
        }
    }
    
    /**
     * 发送消息
     */
    public void sendMessage(long userId, PushMessage message) {
        Set<Channel> channels = userChannels.get(userId);
        
        if (channels == null || channels.isEmpty()) {
            // 用户不在线，存储离线消息
            storeOfflineMessage(userId, message);
            return;
        }
        
        // 发送给用户的所有设备
        for (Channel channel : channels) {
            if (channel.isActive()) {
                channel.writeAndFlush(message);
            }
        }
    }
    
    /**
     * 心跳处理
     */
    private void registerHeartbeat(ChannelHandlerContext ctx, String deviceId) {
        // 使用 Netty IdleStateHandler 检测心跳
        // 客户端需要每 30 秒发送一次心跳
    }
}
```

```java title="src/main/java/MessageRouter.java"
/**
 * 消息路由
 */
public class MessageRouter {
    
    private ConnectionManager connectionManager;
    private OfflineStorage offlineStorage;
    private VendorPushService vendorPushService;
    
    /**
     * 路由消息
     */
    public PushResult.DeviceResult route(DeviceInfo device, String messageId, 
                                        PushRequest request) {
        // 1. 检查设备是否在线
        boolean online = connectionManager.isDeviceOnline(device.getDeviceId());
        
        if (online) {
            // 在线：通过长连接推送
            return pushViaLongConnection(device, messageId, request);
        } else {
            // 不在线：通过厂商通道或离线存储
            return pushViaVendorOrOffline(device, messageId, request);
        }
    }
    
    /**
     * 通过长连接推送
     */
    private PushResult.DeviceResult pushViaLongConnection(
            DeviceInfo device, String messageId, PushRequest request) {
        
        try {
            // 发送消息
            connectionManager.sendMessage(device.getUserId(), 
                buildMessage(messageId, request));
            
            return PushResult.DeviceResult.success(device.getDeviceId());
        } catch (Exception e) {
            // 发送失败，降级到厂商通道
            return pushViaVendor(device, messageId, request);
        }
    }
    
    /**
     * 通过厂商通道推送
     */
    private PushResult.DeviceResult pushViaVendor(
            DeviceInfo device, String messageId, PushRequest request) {
        
        String vendorType = device.getVendorType();
        
        switch (vendorType) {
            case "HMS": // 华为
                return huaweiPushService.push(device, messageId, request);
            case "MI": // 小米

            case "OPPO":
                return oppoPushService.push(device, messageId, request);
            case "VIVO":
                return vivoPushService.push(device, messageId, request);
            default:
                // 未知厂商，存储离线消息
                storeOfflineMessage(device.getUserId(), buildMessage(messageId, request));
                return PushResult.DeviceResult.offline(device.getDeviceId());
        }
    }
```

```java title="src/main/java/MessageAck.java"
/**
 * 消息确认机制

public class MessageAck {
    
    /**
     * 客户端确认收到消息
     */
    public void ack(String messageId, long deviceId) {
        // 1. 标记消息已送达
        messageLogService.markDelivered(messageId, deviceId);
        
        // 2. 发送应用层 ACK（可选）
        // 业务方可能需要知道消息是否真正被用户看到
    }
    
    /**
     * 客户端确认消息已读
     */
    public void read(String messageId, long deviceId) {
        // 1. 标记消息已读
        messageLogService.markRead(messageId, deviceId);
        
        // 2. 更新未读数
        unreadCountService.decrement(deviceId);
        
        // 3. 回调业务方（可选）
        notifyMessageRead(messageId);
    }
    
    /**
     * 消息处理超时
     */
    @Scheduled(fixedRate = 5000)
    public void handleTimeout() {
        // 1. 查询超时消息（发送超过 30 秒未确认）
        List<MessageRecord> timeoutMessages = 
            messageLogService.findTimeoutMessages(30000);
        
        for (MessageRecord record : timeoutMessages) {
            // 2. 重试推送
            retryPush(record.getMessageId(), record.getDeviceId());
        }
    }
    
    /**
     * 重试推送
     */
    private void retryPush(String messageId, long deviceId) {
        // 最多重试 3 次
        MessageRecord record = messageLogService.getRecord(messageId, deviceId);
        if (record.getRetryCount() >= 3) {
            // 重试次数用完，标记失败
            messageLogService.markFailed(messageId, deviceId, "Max retries exceeded");
            return;
        }
        
        // 重试
        router.route(deviceId, messageId);
        messageLogService.incrementRetryCount(messageId, deviceId);
    }
}
```

```java title="src/main/java/MessageDeduplication.java"
/**
 * 消息去重
 */
public class MessageDeduplication {
    
    private RedisTemplate<String, String> redis;
    
    /**
     * 检查消息是否已处理
     */
    public boolean isProcessed(String messageId, long deviceId) {
        String key = "msg:processed:" + deviceId + ":" + messageId;
        return Boolean.TRUE.equals(redis.hasKey(key));
    }
    
    /**
     * 标记消息已处理
     */
    public void markProcessed(String messageId, long deviceId) {
        String key = "msg:processed:" + deviceId + ":" + messageId;
        // 设置过期时间，避免内存溢出
        redis.opsForValue().set(key, "1", Duration.ofHours(24));
    }
    
    /**
     * 处理消息（带去重）
     */
    public void handleMessage(PushMessage message, long deviceId) {
        // 1. 去重检查
        if (isProcessed(message.getMessageId(), deviceId)) {
            return;
        }
        
        // 2. 处理消息
        processMessage(message);
        
        // 3. 标记已处理
        markProcessed(message.getMessageId(), deviceId);
    }
}
```

### 5.1 离线存储

```java title="src/main/java/OfflineStorage.java"
/**
 * 离线消息存储
 */
public class OfflineStorage {
    
    private RedisTemplate<String, String> redis;
    private MessageDao messageDao;
    
    /**
     * 存储离线消息
     */
    public void storeOfflineMessage(long userId, PushMessage message) {
        // 1. 存储到 Redis List（用户离线消息队列）
        String key = "offline:msg:" + userId;
        redis.opsForList().rightPush(key, JsonUtil.toJson(message));
        
        // 2. 限制离线消息数量（最多 100 条）
        redis.opsForList().trim(key, -100, -1);
        
        // 3. 持久化到数据库（可选，用于数据分析）
        messageDao.save(message);
    }
    
    /**
     * 获取离线消息
     */
    public List<PushMessage> getOfflineMessages(long userId) {
        String key = "offline:msg:" + userId;
        List<String> messages = redis.opsForList().range(key, 0, -1);
        
        return messages.stream()
            .map(JsonUtil::fromJson)
            .collect(Collectors.toList());
    }
    
    /**
     * 用户上线时同步离线消息
     */
    public List<PushMessage> syncOfflineMessages(long userId, long sinceTime) {
        // 1. 获取离线消息
        List<PushMessage> offlineMessages = getOfflineMessages(userId);
        
        // 2. 过滤指定时间之后的
        List<PushMessage> newMessages = offlineMessages.stream()
            .filter(m -> m.getCreatedAt() > sinceTime)
            .collect(Collectors.toList());
        
        // 3. 清除已同步的消息
        String key = "offline:msg:" + userId;
        redis.delete(key);
        
        // 4. 重新写入剩余消息
        offlineMessages.removeAll(newMessages);
        for (PushMessage msg : offlineMessages) {
            redis.opsForList().rightPush(key, JsonUtil.toJson(msg));
        }
        
        return newMessages;
    }
}
```

```java title="src/main/java/UnreadCountManager.java"
/**
 * 未读数管理
 */
public class UnreadCountManager {
    
    private RedisTemplate<String, String> redis;
    
    /**
     * 获取未读数
     */
    public int getUnreadCount(long userId) {
        String key = "unread:count:" + userId;
        String count = redis.opsForValue().get(key);
        return count != null ? Integer.parseInt(count) : 0;
    }
    
    /**
     * 增加未读数
     */
    public void increment(long userId, int delta) {
        String key = "unread:count:" + userId;
        redis.opsForValue().increment(key, delta);
    }
    
    /**
     * 清零未读数
     */
    public void clear(long userId) {
        String key = "unread:count:" + userId;
        redis.opsForValue().set(key, "0");
    }
    
    /**
     * 减少未读数
     */
    public void decrement(long userId, int delta) {
        String key = "unread:count:" + userId;
        Long result = redis.opsForValue().decrement(key, delta);
        if (result != null && result < 0) {
            redis.opsForValue().set(key, "0");
        }
    }
}
```

## 七、面试追问方向

### 问题一：「长连接如何保活？」

**回答思路**：

```
长连接保活方案：
1. 心跳机制
   - 客户端每 30 秒发送一次心跳
   - 服务端检测 60 秒无心跳则断开

2. TCP KeepAlive
   - 操作系统层面的连接检测
   - 默认 2 小时检测一次，太长

3. 重连机制
   - 客户端检测到断线后自动重连
   - 使用指数退避策略
```

### 问题二：「推送系统的性能如何优化？」

**回答思路**：

```
性能优化策略：
1. 批量发送
   - 一次 TCP 写入多条消息

2. 连接复用
   - 使用 HTTP/2 推送多个设备

3. 消息压缩
   - protobuf 替代 JSON

4. 异步处理
   - 消息写入 MQ，异步发送
```

### 问题三：「如何保证消息不丢失？」

**回答思路**：

```
消息可靠性保障：
1. 消息持久化
   - 离线消息存储到数据库

2. ACK 机制
   - 客户端确认收到消息
   - 未确认的消息重试

3. 消息去重
   - 使用 Redis SETNX 去重
```
