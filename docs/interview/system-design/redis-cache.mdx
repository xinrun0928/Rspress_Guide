# Redis 缓存：高性能系统的「加速器」

当你访问一个网站，首页加载只需要 0.1 秒。但数据库查询一次需要 50 毫秒。差 500 倍——是什么抹平了这个差距？**缓存。**

## 一、Redis 基础

### 1.1 Redis 数据结构

:::details Redis 数据结构概览

**String（字符串）**

- 常用命令：SET/GET
- 应用：缓存、计数器、分布式锁

**Hash（哈希）**

- 常用命令：HSET/HGET/HGETALL
- 应用：对象存储、购物车

**List（列表）**

- 常用命令：LPUSH/RPOP
- 应用：消息队列、关注列表

**Set（集合）**

- 常用命令：SADD/SMEMBERS/SISMEMBER
- 应用：标签、好友、抽奖

**ZSet（有序集合）**

- 常用命令：ZADD/ZRANGE/ZREVRANK
- 应用：排行榜、延时队列

:::

### 1.2 Redis 应用场景

:::details Redis 典型应用场景

**缓存层**

- 热点数据缓存
- Session 共享
- 页面缓存

**计数器**

- UV/PV 统计
- 点赞数、浏览量
- 限流计数器

**分布式锁**

- SETNX + EXPIRE
- Redisson 实现
- 分布式环境下互斥

**消息队列**

- List 实现轻量队列
- Stream 实现完整队列
- 延迟消息

**排行榜**

- ZSet 实现
- 游戏排名、销量榜
- 热搜榜

:::

```java
import org.springframework.data.redis.core.StringRedisTemplate;

public class RedisStringOps {

    private StringRedisTemplate redisTemplate;

    // 基本操作
    public void basicOps() {
        // 设置值
        redisTemplate.opsForValue().set("key", "value");

        // 设置值 + 过期时间
        redisTemplate.opsForValue().set("key", "value", 30, TimeUnit.MINUTES);

        // 获取值
        String value = redisTemplate.opsForValue().get("key");

        // SETNX：不存在才设置（分布式锁常用）
        Boolean success = redisTemplate.opsForValue()
            .setIfAbsent("lock:key", "1", 10, TimeUnit.SECONDS);

        // 批量操作
        redisTemplate.opsForValue().multiGet(Arrays.asList("key1", "key2"));
    }

    // 计数器
    public void counterOps() {
        // INCR：原子递增
        Long count = redisTemplate.opsForValue().increment("counter");

        // INCRBY：增加指定值
        Long count2 = redisTemplate.opsForValue().increment("counter", 10);

        // DECR：原子递减
        Long count3 = redisTemplate.opsForValue().decrement("counter");

        // 统计 UV
        redisTemplate.opsForValue().increment("uv:" + date);
    }
}
```

```java
public class RedisHashOps {

    private StringRedisTemplate redisTemplate;

    // Hash 操作：存储对象
    public void hashOps() {
        String key = "user:100";

        // 设置字段
        redisTemplate.opsForHash().put(key, "name", "张三");
        redisTemplate.opsForHash().put(key, "age", "25");
        redisTemplate.opsForHash().put(key, "city", "北京");

        // 获取单个字段
        Object name = redisTemplate.opsForHash().get(key, "name");

        // 获取所有字段
        Map<Object, Object> user = redisTemplate.opsForHash().entries(key);

        // 删除字段
        redisTemplate.opsForHash().delete(key, "city");

        // 检查字段是否存在
        Boolean exists = redisTemplate.opsForHash().hasKey(key, "name");
    }

    // 应用：存储用户信息
    public void cacheUser(User user) {
        String key = "user:" + user.getId();
        Map<String, String> map = new HashMap<>();
        map.put("name", user.getName());
        map.put("age", String.valueOf(user.getAge()));
        map.put("city", user.getCity());

        redisTemplate.opsForHash().putAll(key, map);
        redisTemplate.expire(key, 1, TimeUnit.HOURS);
    }

    public User getUser(Long userId) {
        String key = "user:" + userId;
        Map<Object, Object> map = redisTemplate.opsForHash().entries(key);

        if (map.isEmpty()) {
            return null;
        }

        User user = new User();
        user.setId(userId);
        user.setName((String) map.get("name"));
        user.setAge(Integer.parseInt((String) map.get("age")));
        user.setCity((String) map.get("city"));
        return user;
    }
}
```

### 2.3 List 操作

```java
public class RedisListOps {

    private StringRedisTemplate redisTemplate;

    // List 操作：消息队列
    public void listOps() {
        String queueKey = "queue:orders";

        // LPUSH：左插入
        redisTemplate.opsForList().leftPush(queueKey, "order:1");
        redisTemplate.opsForList().leftPush(queueKey, "order:2");

        // RPOP：右弹出
        String order = redisTemplate.opsForList().rightPop(queueKey);

        // BRPOP：阻塞右弹出（等待有元素）
        String blockingOrder = redisTemplate.opsForList()
            .rightPop(queueKey, 10, TimeUnit.SECONDS);
    }

    // 应用：最新消息列表
    public void latestMessages(Long userId, Message message) {
        String key = "user:" + userId + ":messages";

        // 左插入（最新的在最前面）
        redisTemplate.opsForList().leftPush(key, JSON.toJSONString(message));

        // 保持最新 100 条
        redisTemplate.opsForList().trim(key, 0, 99);
    }

    public List<Message> getLatestMessages(Long userId, int limit) {
        String key = "user:" + userId + ":messages";
        List<String> messages = redisTemplate.opsForList().range(key, 0, limit - 1);

        return messages.stream()
            .map(m -> JSON.parseObject(m, Message.class))
            .collect(Collectors.toList());
    }
}
```

```java
public class RedisZSetOps {

    private StringRedisTemplate redisTemplate;

    // ZSet 操作：排行榜
    public void zsetOps() {
        String leaderboardKey = "leaderboard:score";

        // 添加/更新分数
        redisTemplate.opsForZSet().add(leaderboardKey, "user:1", 1000);
        redisTemplate.opsForZSet().add(leaderboardKey, "user:2", 2000);
        redisTemplate.opsForZSet().add(leaderboardKey, "user:3", 1500);

        // 增加分数
        redisTemplate.opsForZSet().incrementScore(leaderboardKey, "user:1", 100);

        // 获取排名（从大到小，0 开始）
        Long rank = redisTemplate.opsForZSet().reverseRank(leaderboardKey, "user:1");

        // 获取 Top N
        Set<String> topN = redisTemplate.opsForZSet()
            .reverseRange(leaderboardKey, 0, 9);

        // 获取指定分数范围
        Set<String> range = redisTemplate.opsForZSet()
            .rangeByScore(leaderboardKey, 1000, 2000);
    }

    // 应用：热搜榜
    public void updateHotSearch(String keyword, double score) {
        String key = "hotsearch:daily:" + LocalDate.now();
        redisTemplate.opsForZSet().incrementScore(key, keyword, score);
    }

    public List<String> getHotSearch(int topN) {
        String key = "hotsearch:daily:" + LocalDate.now();
        Set<String> hot = redisTemplate.opsForZSet()
            .reverseRange(key, 0, topN - 1);
        return new ArrayList<>(hot);
    }
```

## 三、Redis 高级特性

### 3.1 Pipeline 批量操作

```java
public class RedisPipeline {

    private StringRedisTemplate redisTemplate;

    // Pipeline：减少网络往返
    public void batchOps(List<String> keys) {
        // 普通方式：每个 key 都会有一次网络往返
        // for (String key : keys) {
        //     redisTemplate.opsForValue().get(key);
        // }

        // Pipeline：批量操作，减少网络往返
        List<Object> results = redisTemplate.executePipelined(
            (RedisCallback<Object>) connection -> {
                for (String key : keys) {
                    connection.stringCommands().get(key.getBytes());
                }
                return null;
            }
        );
    }

    // 批量写入
    public void batchSet(Map<String, String> keyValues) {
        redisTemplate.executePipelined((RedisCallback<Object>) connection -> {
            for (Map.Entry<String, String> entry : keyValues.entrySet()) {
                connection.stringCommands()
                    .set(entry.getKey().getBytes(), entry.getValue().getBytes());
            }
            return null;
        });
    }
}
```

```java
public class RedisTransaction {

    private StringRedisTemplate redisTemplate;

    // 事务：保证原子性
    public void transactionOps() {
        // 开启事务
        redisTemplate.execute(new SessionCallback<Object>() {
            @Override
            public Object execute(RedisOperations operations) throws DataAccessException {
                operations.multi(); // 开启事务

                operations.opsForValue().set("key1", "value1");
                operations.opsForValue().set("key2", "value2");
                operations.opsForValue().increment("counter");

                return operations.exec(); // 执行事务
            }
        });
    }
}
```

## 四、Redis 在系统中的应用

### 4.1 缓存用户信息

```java
public class CacheUserService {

    private StringRedisTemplate redisTemplate;
    private UserDao userDao;

    private static final String USER_CACHE_KEY = "user:cache:";
    private static final Duration CACHE_EXPIRE = Duration.ofHours(1);

    // 查询用户（缓存优先）
    public User getUser(Long userId) {
        String cacheKey = USER_CACHE_KEY + userId;

        // 1. 先查缓存
        Map<Object, Object> cached = redisTemplate.opsForHash().entries(cacheKey);

        if (!cached.isEmpty()) {
            return convertToUser(userId, cached);
        }

        // 2. 缓存未命中，查数据库
        User user = userDao.findById(userId);

        if (user != null) {
            // 3. 回填缓存
            Map<String, String> cacheValue = new HashMap<>();
            cacheValue.put("name", user.getName());
            cacheValue.put("age", String.valueOf(user.getAge()));
            redisTemplate.opsForHash().putAll(cacheKey, cacheValue);
            redisTemplate.expire(cacheKey, CACHE_EXPIRE);
        }

        return user;
    }

    // 更新用户
    public void updateUser(User user) {
        // 1. 更新数据库
        userDao.update(user);

        // 2. 删除缓存（而非更新）
        String cacheKey = USER_CACHE_KEY + user.getId();
        redisTemplate.delete(cacheKey);
    }

    // 删除用户
    public void deleteUser(Long userId) {
        // 1. 删除数据库
        userDao.deleteById(userId);

        // 2. 删除缓存
        String cacheKey = USER_CACHE_KEY + userId;
        redisTemplate.delete(cacheKey);
    }

    private User convertToUser(Long userId, Map<Object, Object> cached) {
        User user = new User();
        user.setId(userId);
        user.setName((String) cached.get("name"));
        user.setAge(Integer.parseInt((String) cached.get("age")));
        return user;
    }
}
```

### 4.2 分布式锁

```java
public class RedisDistributedLock {

    private StringRedisTemplate redisTemplate;

    // 加锁
    public boolean tryLock(String resource, String requestId, long expireTime) {
        String key = "lock:" + resource;
        Boolean success = redisTemplate.opsForValue()
            .setIfAbsent(key, requestId, expireTime, TimeUnit.SECONDS);
        return Boolean.TRUE.equals(success);
    }

    // 解锁（Lua 脚本）
    public boolean unlock(String resource, String requestId) {
        String key = "lock:" + resource;

        String luaScript =
            "if redis.call('get', KEYS[1]) == ARGV[1] then " +
            "    return redis.call('del', KEYS[1]) " +
            "else " +
            "    return 0 " +
            "end";

        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(luaScript, Long.class),
            Collections.singletonList(key),
            requestId
        );

        return result != null && result > 0;
    }

    // 获取锁（带重试）
    public String getLockWithRetry(String resource, long waitTime,
                                   long retryInterval) {
        String requestId = UUID.randomUUID().toString();
        long deadline = System.currentTimeMillis() + waitTime;

        while (System.currentTimeMillis() < deadline) {
            if (tryLock(resource, requestId, 30)) {
                return requestId;
            }
            try {
                Thread.sleep(retryInterval);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }

        return null;
    }
}
```

### 4.3 延迟队列

```java
public class RedisDelayQueue {

    private StringRedisTemplate redisTemplate;

    // 添加延迟任务
    public void addTask(String taskId, long delayMillis) {
        String key = "delay:queue";
        long executeTime = System.currentTimeMillis() + delayMillis;

        // 使用 ZSet 存储，score 为执行时间
        redisTemplate.opsForZSet().add(key, taskId, executeTime);
    }

    // 消费延迟任务
    public List<String> pollTasks() {
        String key = "delay:queue";
        long now = System.currentTimeMillis();

        // 获取所有到期的任务
        Set<String> tasks = redisTemplate.opsForZSet()
            .rangeByScore(key, 0, now);

        if (tasks == null || tasks.isEmpty()) {
            return Collections.emptyList();
        }

        List<String> executedTasks = new ArrayList<>();

        for (String taskId : tasks) {
            // 原子性删除（防止重复执行）
            Long removed = redisTemplate.opsForZSet()
                .remove(key, taskId);

            if (removed != null && removed > 0) {
                executedTasks.add(taskId);
            }
        }

        return executedTasks;
    }
}
```

## 五、Redis 注意事项

### 5.1 缓存问题

:::details 缓存常见问题

**缓存穿透**

- 大量请求不存在的数据
- 解决：布隆过滤器、空值缓存

**缓存击穿**

- 热点 key 过期，瞬间大量请求
- 解决：互斥锁、热点永不过期

**缓存雪崩**

- 大量缓存同时过期
- 解决：过期时间随机、多级缓存

:::

### 5.2 内存管理

```
Redis 内存注意事项：

1. 内存淘汰策略
   - noeviction：不淘汰
   - allkeys-lru：所有 key LRU 淘汰
   - volatile-lru：设置过期 key 的 LRU 淘汰
   - allkeys-random：所有 key 随机淘汰

2. 内存优化
   - 合理设计 key
   - 避免 bigkey
   - 定期清理过期数据

3. 持久化
   - RDB：定期快照
   - AOF：日志追加
```
