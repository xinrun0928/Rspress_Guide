# 设计关注与粉丝系统

你有没有想过：

- 微博的关注列表和粉丝列表是怎么存的？
- 为什么关注后，立即就能看到对方的微博？

这些功能的背后，都是**关注与粉丝系统**在默默工作。它的实现，涉及到社交平台最核心的数据关系。

### 2.2 Redis 缓存设计

对于热点用户（粉丝数 > 10 万），关注关系存储在 Redis：

```java
/**
 * 关注关系缓存
 */
public class FollowRelationCache {

    private RedisTemplate<String, String> redis;

    /**
     * 缓存 Key 格式
     */
    private static final String FOLLOW_KEY = "follow:%d";        // 用户关注的 Set
    private static final String FAN_KEY = "fan:%d";              // 用户的粉丝 Set
    private static final String FOLLOW_COUNT_KEY = "count:follow:%d";  // 关注数
    private static final String FAN_COUNT_KEY = "count:fan:%d";        // 粉丝数

    /**
     * 关注（写缓存）
     */
    public void follow(Long userId, Long followId) {
        // 1. 添加到关注列表
        String followKey = String.format(FOLLOW_KEY, userId);
        redis.opsForSet().add(followKey, followId.toString());

        // 2. 添加到对方粉丝列表
        String fanKey = String.format(FAN_KEY, followId);
        redis.opsForSet().add(fanKey, userId.toString());

        // 3. 更新计数
        redis.opsForValue().increment(String.format(FOLLOW_COUNT_KEY, userId));
        redis.opsForValue().increment(String.format(FAN_COUNT_KEY, followId));
    }

    /**
     * 获取关注列表
     */
    public Set<Long> getFollowing(Long userId, int offset, int limit) {
        String followKey = String.format(FOLLOW_KEY, userId);
        Set<String> follows = redis.opsForSet().members(followKey);

        return follows.stream()
            .map(Long::parseLong)
            .skip(offset)
            .limit(limit)
            .collect(Collectors.toSet());
    }

    /**
     * 获取粉丝列表
     */
    public Set<Long> getFans(Long userId, int offset, int limit) {
        String fanKey = String.format(FAN_KEY, userId);
        Set<String> fans = redis.opsForSet().members(fanKey);

        return fans.stream()
            .map(Long::parseLong)
            .skip(offset)
            .limit(limit)
            .collect(Collectors.toSet());
    }

    /**
     * 是否关注
     */
    public boolean isFollowing(Long userId, Long followId) {
        String followKey = String.format(FOLLOW_KEY, userId);
        return Boolean.TRUE.equals(redis.opsForSet().isMember(followKey, followId.toString()));
    }
}
```

## 三、核心实现

### 3.1 关注操作

```java
/**
 * 关注服务
 */
public class FollowService {

    private FollowRelationRepository relationRepository;
    private UserCounterRepository counterRepository;
    private MQService mqService;
    private FollowRelationCache cache;

    /**
     * 关注用户
     */
    @Transactional
    public void follow(Long userId, Long followId) {
        // 1. 检查是否已关注
        if (relationRepository.exists(userId, followId)) {
            throw new BusinessException("已经关注过了");
        }

        // 2. 检查是否关注自己
        if (userId.equals(followId)) {
            throw new BusinessException("不能关注自己");
        }

        // 3. 保存关注关系
        relationRepository.save(userId, followId);

        // 4. 更新计数
        counterRepository.incFollowCount(userId);
        counterRepository.incFanCount(followId);

        // 5. 检查是否互相关注（对方也关注了我）
        if (relationRepository.exists(followId, userId)) {
            // 互粉
            relationRepository.saveMutual(userId, followId);
            counterRepository.incMutualCount(userId);
            counterRepository.incMutualCount(followId);
        }

        // 6. 发送消息（更新 Feed）
        mqService.send("follow:update", new FollowEvent(userId, followId));

        // 7. 更新缓存（如果用户是热点）
        if (isHotUser(followId)) {
            cache.follow(userId, followId);
        }
    }

    /**
     * 取消关注
     */
    @Transactional
    public void unfollow(Long userId, Long followId) {
        // 1. 删除关注关系
        relationRepository.delete(userId, followId);

        // 2. 更新计数
        counterRepository.decFollowCount(userId);
        counterRepository.decFanCount(followId);

        // 3. 检查是否之前是互粉
        if (relationRepository.isMutual(userId, followId)) {
            relationRepository.deleteMutual(userId, followId);
            counterRepository.decMutualCount(userId);
            counterRepository.decMutualCount(followId);
        }

        // 4. 发送消息（更新 Feed）
        mqService.send("follow:update", new UnfollowEvent(userId, followId));

        // 5. 更新缓存
        if (isHotUser(followId)) {
            cache.unfollow(userId, followId);
        }
    }

    /**
     * 是否是热点用户
     */
    private boolean isHotUser(Long userId) {
        return counterRepository.getFanCount(userId) > 100000;
    }
}
```

### 3.2 列表查询

```java
/**
 * 关注列表服务
 */
public class FollowListService {

    private FollowRelationRepository relationRepository;
    private UserRepository userRepository;
    private FollowRelationCache cache;

    /**
     * 获取关注列表（分页）
     */
    public Page<User> getFollowing(Long userId, int page, int size) {
        // 1. 先查数据库
        List<Long> followIds = relationRepository.getFollowing(userId, page, size);

        // 2. 如果是热点用户，查缓存
        if (isHotUser(userId)) {
            followIds = cache.getFollowing(userId, page * size, size)
                .stream().collect(Collectors.toList());
        }

        // 3. 批量查询用户信息
        List<User> users = userRepository.findByIds(followIds);

        return new Page<>(users, page, size);
    }

    /**
     * 获取粉丝列表（游标分页）
     */
    public List<User> getFans(Long userId, long cursor, int size) {
        // 1. 先查数据库
        List<Long> fanIds = relationRepository.getFansWithCursor(userId, cursor, size);

        // 2. 如果是热点用户，查缓存
        if (isHotUser(userId)) {
            fanIds = cache.getFans(userId, 0, size)
                .stream().collect(Collectors.toList());
        }

        // 3. 批量查询用户信息
        return userRepository.findByIds(fanIds);
    }

    /**
     * 获取互粉列表
     */
    public List<User> getMutualFriends(Long userId, int page, int size) {
        return relationRepository.getMutualFriends(userId, page, size)
            .stream()
            .map(userRepository::findById)
            .collect(Collectors.toList());
    }
}
```

### 3.3 计数服务

```java
/**
 * 计数服务
 */
public class CounterService {

    private UserCounterRepository counterRepository;
    private RedisTemplate<String, String> redis;

    private static final String FOLLOW_COUNT_KEY = "user:followCount:%d";
    private static final String FAN_COUNT_KEY = "user:fanCount:%d";
    private static final String MUTUAL_COUNT_KEY = "user:mutualCount:%d";

    /**
     * 获取关注数
     */
    public int getFollowCount(Long userId) {
        // 1. 先查缓存
        String cached = redis.opsForValue().get(String.format(FOLLOW_COUNT_KEY, userId));
        if (cached != null) {
            return Integer.parseInt(cached);
        }

        // 2. 查数据库
        int count = counterRepository.getFollowCount(userId);

        // 3. 写入缓存
        redis.opsForValue().set(String.format(FOLLOW_COUNT_KEY, userId), String.valueOf(count));

        return count;
    }

    /**
     * 获取粉丝数
     */
    public int getFanCount(Long userId) {
        String cached = redis.opsForValue().get(String.format(FAN_COUNT_KEY, userId));
        if (cached != null) {
            return Integer.parseInt(cached);
        }

        int count = counterRepository.getFanCount(userId);
        redis.opsForValue().set(String.format(FAN_COUNT_KEY, userId), String.valueOf(count));

        return count;
    }

    /**
     * 更新计数（异步）
     */
    public void updateCount(Long userId, String countType, int delta) {
        // 直接更新数据库
        counterRepository.updateCount(userId, countType, delta);

        // 删除缓存，下次查询会重新加载
        String key = String.format(countType, userId);
        redis.delete(key);
    }
}
```

## 四、热点用户处理

### 4.1 热点判定

```java
/**
 * 热点用户检测
 */
public class HotUserDetector {

    private UserCounterRepository counterRepository;
    private RedisTemplate<String, String> redis;

    private static final String HOT_USER_KEY = "hot:users";
    private static final int HOT_THRESHOLD = 100000; // 10 万粉丝

    /**
     * 检查是否为热点用户
     */
    public boolean isHotUser(Long userId) {
        // 1. 先查缓存
        Boolean cached = redis.opsForSet().isMember(HOT_USER_KEY, userId.toString());
        if (cached != null) {
            return cached;
        }

        // 2. 查数据库
        int fanCount = counterRepository.getFanCount(userId);
        boolean isHot = fanCount >= HOT_THRESHOLD;

        // 3. 写入缓存
        if (isHot) {
            redis.opsForSet().add(HOT_USER_KEY, userId.toString());
        }

        return isHot;
    }

    /**
     * 更新热点用户列表（定时任务）
     */
    public void refreshHotUsers() {
        // 查出粉丝数超过阈值的用户
        List<Long> hotUsers = counterRepository.getUsersByFanCount(HOT_THRESHOLD);

        // 清空并重建
        redis.delete(HOT_USER_KEY);
        for (Long userId : hotUsers) {
            redis.opsForSet().add(HOT_USER_KEY, userId.toString());
        }
    }
}
```

对于热点用户，使用 Redis Set 存储粉丝列表：

```java
/**
 * 热点用户存储
 */
public class HotUserStorage {

    private RedisTemplate<String, String> redis;

    /**
     * 批量添加粉丝
     */
    public void addFans(Long userId, List<Long> fanIds) {
        String fanKey = "hot:fans:" + userId;
        for (Long fanId : fanIds) {
            redis.opsForSet().add(fanKey, fanId.toString());
        }
    }

    /**
     * 批量删除粉丝
     */
    public void removeFans(Long userId, List<Long> fanIds) {
        String fanKey = "hot:fans:" + userId;
        for (Long fanId : fanIds) {
            redis.opsForSet().remove(fanKey, fanId.toString());
        }
    }

    /**
     * 获取粉丝数
     */
    public long getFanCount(Long userId) {
        String fanKey = "hot:fans:" + userId;
        return redis.opsForSet().size(fanKey);
    }

    /**
     * 获取粉丝列表（分页）
     */
    public List<Long> getFans(Long userId, int page, int size) {
        String fanKey = "hot:fans:" + userId;
        int start = page * size;
        int end = start + size - 1;

        Set<String> fans = redis.opsForSet().range(fanKey, start, end);
        return fans.stream()
            .map(Long::parseLong)
            .collect(Collectors.toList());
    }
}
```

## 五、反垃圾处理

### 5.1 关注频率限制

```java
/**
 * 关注反垃圾
 */
public class FollowAntiSpam {

    private RedisTemplate<String, String> redis;

    /**
     * 检查是否可以关注
     */
    public boolean canFollow(Long userId) {
        String dailyKey = "follow:daily:" + userId;
        String hourlyKey = "follow:hourly:" + userId;

        // 1. 检查每小时限制（每小时最多关注 100 人）
        String hourlyCount = redis.opsForValue().get(hourlyKey);
        if (hourlyCount != null && Integer.parseInt(hourlyCount) >= 100) {
            return false;
        }

        // 2. 检查每天限制（每天最多关注 500 人）
        String dailyCount = redis.opsForValue().get(dailyKey);
        if (dailyCount != null && Integer.parseInt(dailyCount) >= 500) {
            return false;
        }

        return true;
    }

    /**
     * 记录关注行为
     */
    public void recordFollow(Long userId) {
        String dailyKey = "follow:daily:" + userId;
        String hourlyKey = "follow:hourly:" + userId;

        // 每小时计数
        redis.opsForValue().increment(hourlyKey);
        redis.expire(hourlyKey, Duration.ofHours(1));

        // 每天计数
        redis.opsForValue().increment(dailyKey);
        redis.expire(dailyKey, Duration.ofDays(1));
    }

    /**
     * 检查冷却时间
     */
    public boolean checkCooldown(Long userId, Long targetUserId) {
        String cooldownKey = "follow:cooldown:" + userId + ":" + targetUserId;
        return redis.hasKey(cooldownKey);
    }

    /**
     * 设置冷却时间（取消关注后）
     */
    public void setCooldown(Long userId, Long targetUserId) {
        String cooldownKey = "follow:cooldown:" + userId + ":" + targetUserId;
        redis.opsForValue().set(cooldownKey, "1", Duration.ofMinutes(30));
    }
}
```

## 六、面试追问方向

### 问题一：「如何设计一个支持千万级粉丝的存储方案？」

**挑战**：单用户粉丝数达到千万级，常规存储无法满足。

**方案**：

1. 分片存储：按粉丝 ID 哈希分片到多个表
2. 热点缓存：热点用户的粉丝列表缓存在 Redis
3. 分层存储：热点用户用 Redis，普通用户用数据库
4. 只存储 ID：列表只存用户 ID，用户详情单独查询

### 问题二：「如何保证关注计数的准确性？」

**计数器一致性问题**：

1. 数据库计数是准确的，但查询慢
2. Redis 计数快，但可能不一致

**解决方案**：

1. 使用 Redis + 数据库双写
2. 异步任务定期同步
3. 计数器使用 Delta 更新
4. 大促前预热热点数据

### 问题三：「如何处理热点用户发布微博时的推送？」

**问题**：大 V 发微博，推送粉丝的压力巨大。

**解决方案**：

1. 不推送到粉丝收件箱
2. 粉丝读取时，单独拉取大 V 微博
3. 大 V 微博写入热门池
4. 主动推送改为拉取模式

## 七、总结

关注系统的核心是维护用户之间的关系网络。

:::tip 关注系统设计要点

**存储设计**

- 关注关系表：MySQL
- 热点用户缓存：Redis Set
- 计数器：MySQL + Redis 双写

**核心功能**

- 关注/取关：事务保证
- 列表查询：分页 + 游标
- 计数服务：实时 + 缓存

**热点处理**

- 热点判定：粉丝数 > 10 万
- 分片存储：哈希分片
- 推送优化：大 V 不推，拉取

:::

> 你有没有想过：微博的关注列表是如何支持双向查询的？既要知道「我关注了谁」，又要知道「谁关注了我」——这两种查询的数据结构有什么不同？这个问题涉及到了图数据库的核心设计思想。
