# 算法训练

刷了 200 道题，面试时遇到新题还是没思路？

这不是你的问题，是方法的问题。

很多人刷题的方式是「按序号刷」或「随机刷」，每道题做完就忘，下次遇到变体还是不会。刷题的核心不是数量，而是**建立题型直觉**——看到一道题，能在 30 秒内判断它属于哪个类型、该用什么思路。

本模块按题型分专题，每个专题覆盖高频考点、解题模板和变体思路。配合「三遍法则」，把每道题做透，而不是囫囵吞枣。

## 推荐刷题顺序

不要按序号刷，按类型集中突破：

```
第一轮（P0）：数组 + 哈希表 + 链表
→ 最基础，面试出现率最高，必须熟练

第二轮（P0）：二分查找 + 滑动窗口
→ 边界条件是难点，容易踩坑

第三轮（P1）：二叉树 + 栈与队列
→ 递归思维是核心，单调栈高频

第四轮（P1）：回溯 + DFS/BFS
→ 组合、排列、岛屿问题

第五轮（P2）：动态规划 + 图论
→ 最难的两个专题，放到最后
```


## 三遍刷题法则

每道题至少做三遍，才能真正变成自己的：

```
第一遍（理解）：读题，15 分钟没思路就看答案，理解后自己写一遍
第二遍（巩固）：隔一天重做，10 分钟内没思路说明还没掌握
第三遍（提速）：一周后再做，优化复杂度，练习多种解法
```

一道题做三遍，比三道题各做一遍有效得多。


## 本模块核心公式

```
高效刷题 = 分类专题突破 + 模板提炼 + 三遍法则 + 定期复盘
```

不要比谁刷的题多，要比谁遇到新题能更快找到思路。把每道题做透，比囫囵吞枣刷 500 道题有用得多。
