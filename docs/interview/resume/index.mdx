# 简历优化

你可能不知道，HR 看一份简历的平均时间是 **6~7 秒**。

在这 7 秒里，HR 不是在「阅读」，而是在「扫描」——扫关键词、扫数据、扫经历。如果扫不到重点，简历就进垃圾桶了。

简历不是「写你做过什么」，而是**「让面试官一眼看到你的价值」**。

本模块覆盖简历从零到一的完整流程：先搞清楚什么是好简历，再学会用 STAR 法则包装项目经历，然后针对常见问题逐个击破，最后打磨出一份能让你脱颖而出的版本。

## 学习路径

```
第一步（1 天）：读「优秀简历标准」和「各模块写法」
→ 知道好简历长什么样，心里有杆秤

第二步（2 天）：用 STAR 法则重写项目经历
→ 对照「项目包装」文档，把每个项目都过一遍

第三步（1 天）：过一遍「常见问题」
→ 把自己的简历代入检查，发现问题立刻改

第四步（持续）：用「模板」打磨最终版本
→ 格式统一、用词精准、细节无误

第五步（找有经验的人 review）：
→ 找比你资深的人帮你看一遍，你会发现自己看不到的问题
```


## 简历的三个「死亡陷阱」

**陷阱一：写得太多，什么都想展示**

简历篇幅有限，面试官注意力更有限。与其面面俱到，不如**精准击中目标岗位的核心需求**。针对目标公司 JD 定制简历，比海投一份「万能简历」有效得多。

**陷阱二：项目描述太模糊**

「负责后端开发」「参与系统优化」——这类描述等于没说。面试官想知道的是：**你做了什么、怎么做的、结果如何**。数据是最有力的语言。

**陷阱三：格式混乱**

字号不统一、日期格式乱跳、技术栈大小写随意——这些细节看起来是小事，但暴露的是**态度问题**。格式都管不好的人，代码质量能好到哪去？


## 本模块核心公式

```
好简历 = 精准定位 + STAR 法则 + 数据量化 + 无死角格式
```

打磨简历是一个反复迭代的过程。每投递一次都是一次反馈机会。不要写完就完事，每一次投递后根据结果调整，才能打磨出真正有效的版本。
