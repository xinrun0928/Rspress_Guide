# 组件选型指南

**目标读者**：P7 面试准备  
**面试级别**：P7 中频

## 快速自测

> **🔴 面试官最关心的 3 个问题**
>
> 1. 如何选择消息队列？
> 2. 如何选择缓存方案？
> 3. 如何选择数据库？

---

## 一、消息队列选型

| 特性 | Kafka | RabbitMQ | RocketMQ |
|------|-------|----------|----------|
| 吞吐量 | 百万级 | 万级 | 十万级 |
| 延迟 | 低 | 中 | 低 |
| 事务消息 | ❌ | ❌ | ✅ |
| 顺序消息 | 分区顺序 | 单队列顺序 | 分区顺序 |
| 优先级队列 | ❌ | ✅ | ❌ |
| 延迟队列 | ❌ | ✅ | ✅ |
| 生态 | 大数据友好 | 功能丰富 | 阿里系 |

### 选型建议

| 场景 | 推荐 |
|------|------|
| 日志采集、大数据 | Kafka |
| 电商交易、金融 | RocketMQ |
| 复杂路由、企业集成 | RabbitMQ |

---

## 二、缓存选型

| 特性 | Redis | Memcached |
|------|-------|----------|
| 数据结构 | 丰富 | KV |
| 持久化 | 支持 | 不支持 |
| 集群 | Cluster | 客户端分片 |
| 事务 | 支持 | 不支持 |
| 适用场景 | 缓存 + Session | 页面缓存 |

---

## 三、数据库选型

| 特性 | MySQL | PostgreSQL | MongoDB |
|------|-------|------------|---------|
| 类型 | 关系型 | 关系型 | 文档型 |
| 事务 | ACID | ACID | 支持 |
| JSON | 支持 | 支持 | 原生 |
| 全文搜索 | 插件 | 原生 | 插件 |
| 适用场景 | 业务数据 | 复杂查询 | 非结构化 |

---

>
> **第二层**：Kafka 和 RocketMQ 有什么区别？
>
> **第三层**：Redis 和 Memcached 有什么区别？
