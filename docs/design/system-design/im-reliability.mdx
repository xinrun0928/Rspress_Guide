## 二、消息投递机制

### 消息确认流程

```java
@Service
public class MessageDeliveryService {
    @Autowired
    private MessageProducer producer;

    public DeliveryResult deliver(Message message) {
        // 1. 消息入库
        messageMapper.insert(message);

        // 2. 消息投递
        DeliveryResult result = producer.deliver(message);

        // 3. 更新状态
        if (result.isSuccess()) {
            message.setStatus(MessageStatus.SENT);
        } else {
            // 投递失败，进入重试队列
            scheduleRetry(message);
        }

        return result;
    }
}
```

### 重试机制

```java
@Service
public class RetryService {
    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    // 延迟重试
    public void scheduleRetry(Message message) {
        int retryCount = message.getRetryCount();

        // 指数退避
        long delay = (long) Math.pow(2, retryCount) * 1000; // 1s, 2s, 4s, 8s...

        // 最多重试 5 次
        if (retryCount < 5) {
            message.setRetryCount(retryCount + 1);
            redisTemplate.opsForZSet().add(
                "message:retry",
                JSON.toJSONString(message),
                System.currentTimeMillis() + delay
            );
        } else {
            // 最终失败，更新状态
            messageMapper.updateStatus(message.getMsgId(), MessageStatus.FAILED);
        }
    }

    // 定时处理重试队列
    @Scheduled(fixedDelay = 1000)
    public void processRetryQueue() {
        Set<String> messages = redisTemplate.opsForZSet()
            .rangeByScore("message:retry", 0, System.currentTimeMillis());

        for (String msgJson : messages) {
            Message message = JSON.parseObject(msgJson, Message.class);
            try {
                deliverMessage(message);
                redisTemplate.opsForZSet().remove("message:retry", msgJson);
            } catch (Exception e) {
                // 继续留在队列，下次重试
            }
        }
    }
}
```

## 三、消息去重

### 幂等设计

```java
@Service
public class IdempotentService {
    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    private static final Duration DEDUP_WINDOW = Duration.ofMinutes(5);

    public boolean isDuplicate(String msgId) {
        String key = "msg:dedup:" + msgId;
        Boolean success = redisTemplate.setIfAbsent(key, "1", DEDUP_WINDOW);
        return !Boolean.TRUE.equals(success);
    }
}
```

## 四、消息顺序保证

### 分区保证顺序

```java
@Service
public class OrderedMessageService {
    // 按会话 ID 分区，保证同一会话消息有序
    public void sendMessage(Message message) {
        String topic = "im:message";
        String partitionKey = message.getConversationId();

        producer.send(topic, partitionKey, message);
    }
}
```

## 五、消息未读数同步

```java
@Service
public class UnreadSyncService {
    @KafkaListener(topics = "im.message", groupId = "unread-sync")
    public void syncUnread(Message message) {
        // 1. 增加发送者已发送数
        incrementSentCount(message.getFromUid());

        // 2. 增加接收者未读数
        if (!isUserOnline(message.getToUid())) {
            incrementUnreadCount(message.getToUid(), message.getConversationId());
        }

        // 3. 同步会话最后消息
        updateConversationLastMessage(message);
    }
}
```

## 六、消息可靠传输总结

| 保障措施 | 实现方式 |
|----------|----------|
| 消息持久化 | MySQL 存储 |
| 消息投递确认 | Kafka 消费确认 |
| 重试机制 | Redis ZSet 延迟队列 |
| 消息去重 | Redis 去重窗口 |
| 消息顺序 | 按会话分区 |
| 未读同步 | 实时更新 Redis |
