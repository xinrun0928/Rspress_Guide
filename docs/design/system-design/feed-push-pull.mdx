# Feed 流推拉模式对比

**目标读者**：P7 面试准备  
**面试级别**：P7 高频

---

## 一、三种模式对比

| 模式 | 原理 | 写复杂度 | 读复杂度 | 存储 | 适用场景 |
|------|------|----------|----------|------|----------|
| 推模式 | 写入时推送 | 高 | 低 | 大 | 小V、粉丝少 |
| 拉模式 | 读取时拉取 | 低 | 高 | 小 | 大V、粉丝多 |
| 混合模式 | 大V推、小V拉 | 中 | 中 | 中 | 大型平台 |

---

## 二、推模式详解

### 写入流程

```java
@Service
public class PushFeedService {
    @Autowired
    private FanService fanService;

    public void publish(Content content) {
        Long authorId = content.getUserId();
        List<Long> fans = fanService.getFans(authorId);

        // 推送给所有粉丝的收件箱
        for (Long fanId : fans) {
            String inboxKey = "timeline:inbox:" + fanId;
            redisTemplate.opsForList().leftPush(inboxKey, content.getId().toString());
            redisTemplate.opsForList().trim(inboxKey, 0, 999);
        }
    }
}
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 读取快（直接读收件箱）| 写放大（1 条内容推 N 份）|
| 用户体验好 | 存储成本高 |
| 离线用户也能看到 | 大V推送成本高 |

## 三、拉模式详解

### 读取流程

```java
@Service
public class PullFeedService {
    public List<Content> pullTimeline(Long userId, int limit) {
        // 1. 获取关注列表
        List<Long> followings = getFollowings(userId);

        // 2. 拉取关注者的最新内容
        List<Content> all = new ArrayList<>();
        for (Long followingId : followings) {
            List<Content> outbox = getOutbox(followingId, limit);
            all.addAll(outbox);
        }

        // 3. 排序、去重、分页
        return sortAndPaginate(all, limit);
    }
}
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 存储成本低 | 读取慢（多次查询）|
| 大V友好 | 无法离线查看 |
| 实现简单 | 难以保证内容新鲜度 |

## 四、混合模式实现

```java
@Service
public class HybridFeedService {
    private static final int PUSH_THRESHOLD = 10000;

    @Autowired
    private FanService fanService;

    public void handlePublish(Content content) {
        Long authorId = content.getUserId();
        int fanCount = fanService.getFanCount(authorId);

        if (fanCount < PUSH_THRESHOLD) {
            // 小博主：推模式
            pushToAllFans(content, fanService.getFans(authorId));
        } else {
            // 大V：混合模式
            // 前 1 万粉丝：推模式
            pushToFans(content, fanService.getFans(authorId, 0, PUSH_THRESHOLD));
            // 剩余粉丝：拉模式（只更新发件箱）
            updateOutbox(content, authorId);
        }
    }
}
```

## 五、Timeline 存储策略

### Redis 存储

```java
@Service
public class TimelineStorage {
    private static final int MAX_INBOX_SIZE = 1000;

    // 写入收件箱
    public void pushToInbox(Long userId, Long contentId) {
        String key = "timeline:inbox:" + userId;
        redisTemplate.opsForList().leftPush(key, contentId.toString());
        redisTemplate.opsForList().trim(key, 0, MAX_INBOX_SIZE - 1);
    }

    // 读取收件箱
    public List<Long> getInbox(Long userId, int offset, int limit) {
        String key = "timeline:inbox:" + userId;
        List<String> ids = redisTemplate.opsForList().range(key, offset, offset + limit - 1);
        return ids.stream().map(Long::parseLong).collect(Collectors.toList());
    }
}
```

## 六、选择建议

| 场景 | 推荐模式 |
|------|----------|
| 粉丝 < 1 万 | 推模式 |
| 粉丝 > 10 万 | 混合模式 |
| 实时性要求高 | 推模式 |
| 存储成本敏感 | 拉模式 |
