## 一、容量估算基础

### 核心公式

```
DAU（日活） → MAU（月活） → TPS/QPS → 存储
```

### 常用时间换算

| 时间单位 | 秒数 |
|----------|------|
| 1 分钟 | 60 |
| 1 小时 | 3600 |
| 1 天 | 86400 |
| 1 周 | 604800 |
| 1 月 | 2592000 |

---

## 二、QPS 估算

### 估算公式

```
QPS = DAU × 人均请求数 / 一天的秒数 × 峰值因子

TPS = QPS × 写操作占比
```

### 估算示例

```
场景：设计一个社交平台

已知：
- DAU = 1 亿（100,000,000）
- 人均每天操作 = 20 次
- 峰值因子 = 5

计算：
- 日总请求 = 1亿 × 20 = 20亿次
- 平均 QPS = 20亿 / 86400 ≈ 23000
- 峰值 QPS = 23000 × 5 ≈ 115000

结论：
- 平均 QPS ≈ 2.3 万
- 峰值 QPS ≈ 11.5 万
```

---

## 三、存储估算

### 存储计算

```
存储需求 = 数据量 × 单条大小 × 副本数
```

### 示例：评论系统

```
场景：设计一个评论系统

已知：
- DAU = 1000 万
- 人均每天评论 = 5 条
- 每条评论 ≈ 200 字节
- 数据保留 1 年

计算：
- 每天新增 = 1000万 × 5 = 5000万条
- 每天存储 = 5000万 × 200 = 10 GB
- 每年存储 = 10 GB × 365 = 3.65 TB

考虑副本（3副本）：
- 总存储 = 3.65 TB × 3 = 10.95 TB
```

---

## 四、带宽估算

### 带宽计算

```
带宽 = QPS × 平均请求大小 × 8 / 压缩比
```

### 示例

```
场景：API 网关带宽估算

已知：
- QPS = 10 万
- 平均请求 = 2 KB
- 响应 = 10 KB

计算：
- 入口带宽 = 10万 × 2KB = 2000 MB/s ≈ 16 Gbps
- 出口带宽 = 10万 × 10KB = 10 GB/s ≈ 80 Gbps
```

---

## 五、服务器估算

### 估算方法

```
服务器数量 = 峰值 QPS / 单机 QPS × 冗余因子
```

### 单机容量参考

| 服务类型 | 单机 QPS | 说明 |
|----------|----------|------|
| API 服务 | 1000-5000 | 取决于业务复杂度 |
| 数据库写入 | 1000-3000 | 取决于硬件配置 |
| Redis 操作 | 10万-50万 | 纯内存操作 |
| Kafka | 10万+ | 取决于配置 |

### 示例

```
场景：计算 API 服务集群规模

已知：
- 峰值 QPS = 10 万
- 单机 API QPS = 2000
- 冗余因子 = 2（高峰+故障）

计算：
- 基础服务器数 = 10万 / 2000 = 50 台
- 考虑冗余 = 50 × 2 = 100 台

结论：需要 100 台 API 服务器
```

## 六、缓存估算

### 热点数据估算

```
热点数据 = 总数据量 × 访问频率前 20% × 命中率
```

### 示例

```
场景：用户信息缓存

已知：
- 总用户 = 1 亿
- 活跃用户 = 1000 万（10%）
- 人均缓存 = 1 KB

计算：
- 热点数据 = 1000万 × 1KB = 10 GB
- 考虑冗余 = 10 GB × 1.5 = 15 GB

结论：Redis 集群 32GB 足够
```

---

## 七、综合估算模板

### 短链系统

```
1. 日均生成短链 = 1000 万
2. 日均访问短链 = 1 亿
3. 峰值 QPS = 1 亿 / 86400 × 5 ≈ 6000

存储：
- 每天新增 = 1000万 × 40字节 ≈ 400 MB
- 1年 = 400 MB × 365 ≈ 146 GB

缓存：
- 热点短链 ≈ 10% = 1000万
- 缓存大小 ≈ 1000万 × 40字节 ≈ 400 MB

服务器：
- API 服务：峰值 QPS 6000 / 2000 × 2 ≈ 6 台
```
