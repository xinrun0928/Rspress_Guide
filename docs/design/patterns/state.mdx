## 一、为什么需要状态模式

### if-else 的问题

```java
public class Order {
    private String status;

    public void pay() {
        if ("CREATED".equals(status)) {
            this.status = "PAID";
            sendNotification();
        } else if ("PAID".equals(status)) {
            // 不能重复支付
        }
    }

    public void deliver() {
        if ("PAID".equals(status)) {
            this.status = "DELIVERED";
            startDelivery();
        } else if ("DELIVERED".equals(status)) {
            // 已经发货
        }
    }
    // 问题：状态和逻辑耦合，难以维护
}
```

## 二、状态模式实现

```java
// 状态接口
public interface OrderState {
    void pay(OrderContext context);
    void deliver(OrderContext context);
    void receive(OrderContext context);
}

// 具体状态
public class CreatedState implements OrderState {
    @Override
    public void pay(OrderContext context) {
        context.setState(new PaidState());
        context.sendNotification("支付成功");
    }

    @Override
    public void deliver(OrderContext context) {
        throw new IllegalStateException("订单未支付，不能发货");
    }

    @Override
    public void receive(OrderContext context) {
        throw new IllegalStateException("订单未发货，不能收货");
    }
}

public class PaidState implements OrderState {
    @Override
    public void pay(OrderContext context) {
        throw new IllegalStateException("订单已支付");
    }

    @Override
    public void deliver(OrderContext context) {
        context.setState(new DeliveredState());
        context.startDelivery();
    }

    @Override
    public void receive(OrderContext context) {
        throw new IllegalStateException("订单未发货");
    }
}

// 上下文
public class OrderContext {
    private OrderState state = new CreatedState();

    public void setState(OrderState state) {
        this.state = state;
    }

    public void pay() { state.pay(this); }
    public void deliver() { state.deliver(this); }
    public void receive() { state.receive(this); }
}
```

## 三、状态模式 vs 策略模式

| 对比 | 状态模式 | 策略模式 |
|------|----------|----------|
| 状态切换 | 上下文切换状态 | 客户端指定策略 |
| 状态关系 | 状态之间有转换关系 | 策略之间无关系 |
| 客户端控制 | 部分由客户端触发 | 完全由客户端选择 |
