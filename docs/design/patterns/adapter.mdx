## 一、为什么需要适配器模式

### 接口不兼容

```java
// 第三方 SDK
public class ThirdPartyPay {
    public void makePayment(String transactionId, int cents) {
        // 第三方接口
    }
}

// 我们的接口
public interface Payment {
    void pay(String orderId, double amount);  // 元为单位的金额
}
```

---

## 二、适配器模式实现

### 对象适配器

```java
// 目标接口
public interface Payment {
    void pay(String orderId, double amount);
}

// 适配器
public class PaymentAdapter implements Payment {
    private ThirdPartyPay thirdPartyPay;

    @Override
    public void pay(String orderId, double amount) {
        // 适配：元 -> 分
        int cents = (int) (amount * 100);
        thirdPartyPay.makePayment(orderId, cents);
    }
}
```

### 类适配器

```java
// 继承方式
public class ClassAdapter extends ThirdPartyPay implements Payment {
    @Override
    public void pay(String orderId, double amount) {
        int cents = (int) (amount * 100);
        makePayment(orderId, cents);
    }
}
```

## 三、Spring 中的适配器

```java
// HandlerAdapter
public interface HandlerAdapter {
    boolean supports(Object handler);
    ModelAndView handle(HttpServletRequest request, HttpServletResponse response, Object handler);
}

// Spring MVC 的适配器
public class RequestMappingHandlerAdapter implements HandlerAdapter {
    @Override
    public boolean supports(Object handler) {
        return handler instanceof HandlerMethod;
    }

    @Override
    public ModelAndView handle(HttpServletRequest request,
                                HttpServletResponse response, Object handler) {
        HandlerMethod method = (HandlerMethod) handler;
        return method.getMethod().invoke(method.getBean(), request, response);
    }
}
```

## 四、对比总结

| 类型 | 实现方式 | 特点 |
|------|----------|------|
| 对象适配器 | 组合 | 灵活，可适配多个类 |
| 类适配器 | 继承 | 直接调用方法，但受限 |
