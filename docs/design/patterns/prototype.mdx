## 一、为什么需要原型模式

### 直接 clone 的问题

```java
// 创建复杂对象成本高
public class User {
    private String name;
    private List<Order> orders = new ArrayList<>();  // 复杂嵌套对象

    // 每个字段都要手动复制
    public User copy() {
        User copy = new User();
        copy.name = this.name;
        copy.orders = new ArrayList<>(this.orders);  // 手动复制
        return copy;
    }
}
```

## 二、原型模式实现

### Cloneable 接口

```java
public class User implements Cloneable {
    private String name;
    private int age;

    @Override
    protected User clone() {
        try {
            return (User) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

### 深拷贝实现

```java
public class User implements Cloneable {
    private String name;
    private Order order;  // 引用类型

    @Override
    public User clone() {
        try {
            User copy = (User) super.clone();
            if (this.order != null) {
                copy.order = this.order.clone();  // 深拷贝
            }
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

## 三、深拷贝 vs 浅拷贝

| 对比 | 浅拷贝 | 深拷贝 |
|------|--------|--------|
| 引用类型 | 复制引用 | 复制对象 |
| 原型修改 | 影响副本 | 不影响 |
| 实现难度 | 简单 | 复杂 |
