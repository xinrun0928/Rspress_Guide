## 一、为什么需要外观模式

### 子系统复杂

```java
// 复杂的子系统
public class CPU { public void start() { System.out.println("CPU启动"); } }
public class Memory { public void load() { System.out.println("加载内存"); } }
public class Disk { public void read() { System.out.println("读取磁盘"); } }
public class Display { public void init() { System.out.println("初始化显示器"); } }

// 客户端需要知道所有细节
public class Client {
    public void startComputer() {
        cpu.start();
        memory.load();
        disk.read();
        display.init();
    }
}
```

## 二、外观模式实现

```java
// 外观类
public class ComputerFacade {
    private CPU cpu = new CPU();
    private Memory memory = new Memory();
    private Disk disk = new Disk();
    private Display display = new Display();

    public void start() {
        cpu.start();
        memory.load();
        disk.read();
        display.init();
    }

    public void shutdown() {
        display.close();
        disk.write();
        memory.save();
        cpu.stop();
    }
}

// 客户端简化
public class Client {
    public void startComputer() {
        ComputerFacade computer = new ComputerFacade();
        computer.start();  // 一键启动
    }
}
```

## 三、外观模式 vs 适配器模式

| 对比 | 外观模式 | 适配器模式 |
|------|----------|------------|
| 目的 | 简化接口 | 转换接口 |
| 方向 | 多个接口 → 简单接口 | 接口 A → 接口 B |
| 关系 | 调用子系统 | 包装已有接口 |
