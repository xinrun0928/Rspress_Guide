## 一、为什么需要访问者模式

### 类型爆炸问题

```java
public class Circle { }
public class Square { }
public class Triangle { }

// 每次新增图形，都要修改所有操作类
public class AreaCalculator {
    public double calculate(Circle c) { return Math.PI * c.getR() * c.getR(); }
    public double calculate(Square s) { return s.getSide() * s.getSide(); }
    public double calculate(Triangle t) { return 0.5 * t.getBase() * t.getHeight(); }
}

public class Drawer {
    public void draw(Circle c) { }
    public void draw(Square s) { }
    public void draw(Triangle t) { }
}
```

## 二、访问者模式实现

```java
// 元素接口
public interface Shape {
    void accept(Visitor visitor);
}

// 具体元素
public class Circle implements Shape {
    private double radius;

    @Override
    public void accept(Visitor visitor) {
        visitor.visit(this);  // 双分派：传入 Visitor
    }
}

// 访问者接口
public interface Visitor {
    void visit(Circle circle);
    void visit(Square square);
    void visit(Triangle triangle);
}

// 具体访问者
public class AreaVisitor implements Visitor {
    @Override
    public void visit(Circle circle) {
        System.out.println("圆形面积: " + Math.PI * circle.getRadius() * circle.getRadius());
    }

    @Override
    public void visit(Square square) {
        System.out.println("正方形面积: " + square.getSide() * square.getSide());
    }

    @Override
    public void visit(Triangle triangle) {
        System.out.println("三角形面积");
    }
}
```

## 三、双分派机制

```java
// 第一层分派：运行时确定 Shape 类型
shape.accept(visitor);  // Circle.accept → visitor.visit(this)

// 第二层分派：运行时确定 Visitor 类型
visitor.visit(circle);  // AreaVisitor → AreaVisitor.visit(Circle)

// 结果：在编译时就确定了调用方法
```

## 四、优缺点

| 优点 | 缺点 |
|------|------|
| 符合开闭原则 | 元素必须稳定 |
| 可添加新操作 | 元素变更困难 |
| 封装操作逻辑 | 违反依赖倒置 |
