## 一、Spring 中的设计模式

### 1. 单例模式

```java
// Spring 默认 Bean 作用域
@Scope(scopeName = "singleton")
public class UserService {
    // 每次获取的都是同一个实例
}
```

### 2. 工厂模式

```java
// FactoryBean
public interface FactoryBean<T> {
    T getObject();  // 创建对象
    Class<?> getObjectType();  // 返回类型
    boolean isSingleton();  // 是否单例
}

// MyBatis 的 SqlSessionFactory
public class SqlSessionFactoryBean implements FactoryBean<SqlSessionFactory> {
    @Override
    public SqlSessionFactory getObject() {
        return buildSqlSessionFactory();
    }
}
```

### 3. 代理模式

```java
// AOP 自动代理
@Configuration
@EnableAspectJAutoProxy
public class AppConfig {
    // 所有 @Aspect 都会自动生成代理
}
```

### 4. 策略模式

```java
// PropertySourceLocator
public interface PropertySourceLocator {
    PropertySource<?> locate(Environment environment);
}

// Spring Cloud 配置中心
public class CompositePropertySourceLocator implements PropertySourceLocator {
    private final List<PropertySourceLocator> delegates;

    @Override
    public PropertySource<?> locate(Environment environment) {
        // 遍历所有策略选择配置
    }
}
```

### 5. 模板方法模式

```java
// JdbcTemplate
public class JdbcTemplate {
    public <T> T execute(StatementCallback<T> action) {
        Connection conn = null;
        try {
            conn = getConnection();
            Statement stmt = conn.createStatement();
            return action.doInStatement(stmt);
        } finally {
            closeStatement(stmt);
            releaseConnection(conn);
        }
    }
}
```

### 6. 观察者模式

```java
// ApplicationEventPublisher
public interface ApplicationEventPublisher {
    void publishEvent(ApplicationEvent event);
}

// Spring 事件
public class OrderCreatedEvent extends ApplicationEvent {
    public OrderCreatedEvent(Object source, Long orderId) {
        super(source);
    }
}

// 监听
@EventListener
public void handleOrderCreated(OrderCreatedEvent event) {
    // 处理事件
}
```

## 二、MyBatis 中的设计模式

### 1. 建造者模式

```java
// SqlSessionFactoryBuilder
public class SqlSessionFactoryBuilder {
    public SqlSessionFactory build(InputStream inputStream) {
        XMLConfigBuilder parser = new XMLConfigBuilder(inputStream);
        return build(parser.parse());
    }

    public SqlSessionFactory build(Configuration config) {
        return new DefaultSqlSessionFactory(config);
    }
}
```

### 2. 装饰器模式

```java
// Cache 接口是装饰器模式的抽象
public interface Cache {
    Object getObject(Object key);
    void putObject(Object key, Object value);
}

// 二级缓存装饰器
public class LoggingCache implements Cache {
    private final Cache delegate;

    @Override
    public Object getObject(Object key) {
        Object result = delegate.getObject(key);
        logCacheHit(key, result);
        return result;
    }
}
```

### 3. 代理模式

```java
// MapperProxy
public class MapperProxy<T> implements InvocationHandler {
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        SqlSession sqlSession = sqlSessionFactory.openSession();
        try {
            return sqlSession.selectOne(
                statement,
                args
            );
        } finally {
            sqlSession.close();
        }
    }
}
```

## 三、Spring Boot 中的设计模式

### 1. 工厂模式 + 策略模式

```java
// 自动配置
@Configuration
public class DispatcherServletConfiguration {
    @Bean
    public HandlerAdapter handlerAdapter() {
        // 根据策略选择适配器
        if (annotationDriven) {
            return new RequestMappingHandlerAdapter();
        }
        return new SimpleControllerHandlerAdapter();
    }
}
```

### 2. 模板方法模式

```java
// SpringApplicationRunListener
public abstract class SpringApplicationRunListener {
    public void starting() {}  // 钩子方法
    public void environmentPrepared(ConfigurableEnvironment environment) {}
    public void contextPrepared(ConfigurableApplicationContext context) {}
    public void contextLoaded(ConfigurableApplicationContext context) {}
    public void started(ConfigurableApplicationContext context) {}
    public void running(ConfigurableApplicationContext context) {}
}
```

## 四、对比总结

| 框架 | 核心设计模式 |
|------|--------------|
| Spring | 单例、工厂、代理、策略、模板方法、观察者 |
| MyBatis | 建造者、装饰器、代理 |
| Spring Boot | 工厂 + 策略、模板方法 |
| Spring MVC | 责任链（HandlerInterceptor） |
| Dubbo | 装饰器、代理、策略 |
