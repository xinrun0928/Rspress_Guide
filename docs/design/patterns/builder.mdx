## 一、为什么需要建造者模式

### 重叠构造函数的困境

```java
// 必需参数
public class User {
    private final String name;    // 必需
    private final String email;    // 必需
    private final int age;         // 可选
    private final String phone;    // 可选
    private final String address;   // 可选
    private final String hobby;     // 可选

    // 构造函数组合爆炸
    public User(String name, String email) { ... }
    public User(String name, String email, int age) { ... }
    public User(String name, String email, int age, String phone) { ... }
    // ... 更多组合
}
```

### setter 的问题

```java
// 使用 setter
User user = new User();
user.setName("张三");
user.setEmail("zhang@example.com");
// 问题：对象状态不一致（age、phone 等可能是 null）
```

## 二、传统建造者模式

### 实现

```java
public class User {
    private final String name;    // final，必须在构造函数中设置
    private final String email;
    private final int age;
    private final String phone;
    private final String address;
    private final String hobby;

    // 私有构造函数
    private User(Builder builder) {
        this.name = builder.name;
        this.email = builder.email;
        this.age = builder.age;
        this.phone = builder.phone;
        this.address = builder.address;
        this.hobby = builder.hobby;
    }

    // 静态内部类 Builder
    public static class Builder {
        // 必需参数
        private final String name;
        private final String email;

        // 可选参数，使用默认值
        private int age = 0;
        private String phone = null;
        private String address = null;
        private String hobby = null;

        public Builder(String name, String email) {
            this.name = name;
            this.email = email;
        }

        public Builder age(int age) {
            this.age = age;
            return this;  // 链式调用
        }

        public Builder phone(String phone) {
            this.phone = phone;
            return this;
        }

        public Builder address(String address) {
            this.address = address;
            return this;
        }

        public Builder hobby(String hobby) {
            this.hobby = hobby;
            return this;
        }

        public User build() {
            return new User(this);
        }
    }

    // getters...
}
```

### 使用

```java
public class Client {
    public static void main(String[] args) {
        // 链式调用，代码清晰
        User user = new User.Builder("张三", "zhang@example.com")
            .age(25)
            .phone("13800138000")
            .address("北京市")
            .hobby("编程")
            .build();

        // 只需要必需参数
        User user2 = new User.Builder("李四", "li@example.com")
            .build();
    }
}
```

## 三、建造者 vs 工厂模式

| 对比 | 建造者模式 | 工厂模式 |
|------|------------|----------|
| 目的 | 构建复杂对象 | 创建对象 |
| 参数 | 可选参数多 | 参数固定 |
| 过程 | 分步骤构建 | 一步创建 |
| 返回值 | this（链式调用） | 创建好的对象 |
| 适用场景 | 配置类、DTO、复杂实体 | 同一类型的不同变体 |

## 四、Lombok @Builder

### 简化写法

```java
import lombok.Builder;
import lombok.Getter;

@Builder
@Getter
public class User {
    private final String name;
    private final String email;
    private final int age;
    private final String phone;
    private final String address;
}
```

### 编译后生成

```java
// Lombok 自动生成的代码（等价于上面的手写版本）
public class User {
    private final String name;
    private final String email;
    private final int age;
    private final String phone;
    private final String address;

    private User(Builder builder) {
        this.name = builder.name;
        this.email = builder.email;
        this.age = builder.age;
        this.phone = builder.phone;
        this.address = builder.address;
    }

    public static Builder builder() {
        return new Builder();
    }

    public static class Builder {
        private String name;
        private String email;
        private int age;
        private String phone;
        private String address;

        public Builder name(String name) {
            this.name = name;
            return this;
        }
        // ... 其他 setter

        public User build() {
            return new User(this);
        }
    }
}
```

### @Builder.Default

```java
@Builder
public class Config {
    private String host = "localhost";  // 普通字段
    @Builder.Default
    private int port = 8080;            // 带默认值的字段
}
```

## 五、StringBuilder

### StringBuilder 的建造者模式

```java
// StringBuilder 本质上就是建造者模式
StringBuilder sb = new StringBuilder();
sb.append("Hello")
  .append(" ")
  .append("World")
  .append("!");

String result = sb.toString();
```

### 为什么不用 String +

```java
// 低效的写法
String result = "Hello" + " " + "World" + "!";  // 每次拼接都创建新对象

// 高效的写法（建造者模式）
StringBuilder sb = new StringBuilder();  // 只创建一次
sb.append("Hello");  // 复用同一对象
sb.append(" ");
sb.append("World");
```

## 六、实战应用

### 1. HTTP 请求构建

```java
public class HttpRequest {
    private final String url;
    private final String method;
    private final Map<String, String> headers;
    private final String body;

    private HttpRequest(Builder builder) {
        this.url = builder.url;
        this.method = builder.method;
        this.headers = Collections.unmodifiableMap(builder.headers);
        this.body = builder.body;
    }

    public static class Builder {
        private final String url;
        private String method = "GET";
        private Map<String, String> headers = new HashMap<>();
        private String body;

        public Builder(String url) {
            this.url = url;
        }

        public Builder method(String method) {
            this.method = method;
            return this;
        }

        public Builder header(String key, String value) {
            this.headers.put(key, value);
            return this;
        }

        public Builder body(String body) {
            this.body = body;
            return this;
        }

        public HttpRequest build() {
            return new HttpRequest(this);
        }
    }
}

// 使用
HttpRequest request = new HttpRequest.Builder("https://api.example.com/users")
    .method("POST")
    .header("Content-Type", "application/json")
    .header("Authorization", "Bearer token")
    .body("{\"name\":\"张三\"}")
    .build();
```

### 2. SQL 构建

```java
public class SqlBuilder {
    private final StringBuilder sql = new StringBuilder();
    private final List<Object> params = new ArrayList<>();

    public SqlBuilder select(String... columns) {
        sql.append("SELECT ");
        sql.append(String.join(", ", columns));
        return this;
    }

    public SqlBuilder from(String table) {
        sql.append(" FROM ").append(table);
        return this;
    }

    public SqlBuilder where(String column, Object value) {
        sql.append(" WHERE ").append(column).append(" = ?");
        params.add(value);
        return this;
    }

    public SqlBuilder and(String column, Object value) {
        sql.append(" AND ").append(column).append(" = ?");
        params.add(value);
        return this;
    }

    public String build() {
        return sql.toString();
    }

    public List<Object> getParams() {
        return params;
    }
}

// 使用
String sql = new SqlBuilder()
    .select("id", "name", "email")
    .from("users")
    .where("status", "active")
    .and("age", 18)
    .build();
```
