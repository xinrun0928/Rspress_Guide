## 一、核心思想

整洁架构（Clean Architecture）由 Robert C. Martin 提出，强调软件架构应该遵循以下原则：

1. **框架无关**：业务逻辑不依赖任何框架
2. **可测试**：业务逻辑可以在没有 UI、数据库的情况下测试
3. **UI 无关**：UI 变化不会影响业务逻辑
4. **数据库无关**：可以切换数据库
5. **外部代理无关**：业务逻辑不直接依赖外部系统

## 三、四层详解

| 层级 | 说明 | 依赖方向 |
|------|------|----------|
| Entities | 核心业务实体 | 无依赖 |
| Use Cases | 应用业务规则 | 依赖 Entities |
| Interface Adapters | 适配器（Controller、Gateway）| 依赖 Use Cases、Entities |
| Frameworks | 外部工具（DB、Web）| 依赖 Interface Adapters |

## 四、代码实现

### 1. Entities（实体层）

```java
// 实体：核心业务规则
public class Order {
    private Long id;
    private Customer customer;
    private List<OrderLineItem> lineItems;
    private OrderStatus status;
    private Money total;

    // 业务规则：订单金额不能为负
    public Money calculateTotal() {
        return lineItems.stream()
            .map(item -> item.getProduct().getPrice().multiply(item.getQuantity()))
            .reduce(Money.ZERO, Money::add);
    }

    // 业务规则：只有 DRAFT 状态才能添加商品
    public void addLineItem(Product product, int quantity) {
        if (this.status != OrderStatus.DRAFT) {
            throw new OrderException("订单已锁定，无法修改");
        }
        this.lineItems.add(new OrderLineItem(product, quantity));
    }

    // 业务规则：只有 PAID 状态才能发货
    public void ship() {
        if (this.status != OrderStatus.PAID) {
            throw new OrderException("订单未支付，不能发货");
        }
        this.status = OrderStatus.SHIPPED;
    }
}

// 值对象
public class Money {
    private final BigDecimal amount;
    private final String currency;

    public Money multiply(int quantity) {
        return new Money(this.amount.multiply(BigDecimal.valueOf(quantity)), this.currency);
    }

    public Money add(Money other) {
        return new Money(this.amount.add(other.amount), this.currency);
    }
}
```

### 2. Use Cases（用例层）

```java
// 用例接口
public interface CreateOrderInput {
    Long getCustomerId();
    List<OrderItemInput> getItems();
}

public interface CreateOrderOutput {
    void setOrder(OrderDTO order);
    void setError(String message);
}

// 用例实现
public class CreateOrderUseCase implements InputPort<CreateOrderInput> {
    private final OutputPort outputPort;
    private final OrderRepository orderRepository;
    private final CustomerRepository customerRepository;

    public CreateOrderUseCase(OutputPort outputPort,
                              OrderRepository orderRepository,
                              CustomerRepository customerRepository) {
        this.outputPort = outputPort;
        this.orderRepository = orderRepository;
        this.customerRepository = customerRepository;
    }

    @Override
    public void execute(CreateOrderInput input) {
        try {
            // 1. 验证客户
            Customer customer = customerRepository.findById(input.getCustomerId())
                .orElseThrow(() -> new CustomerNotFoundException(input.getCustomerId()));

            // 2. 创建订单
            Order order = new Order(customer);

            for (OrderItemInput itemInput : input.getItems()) {
                Product product = productRepository.findById(itemInput.getProductId())
                    .orElseThrow(() -> new ProductNotFoundException(itemInput.getProductId()));
                order.addLineItem(product, itemInput.getQuantity());
            }

            // 3. 保存
            Order savedOrder = orderRepository.save(order);

            // 4. 返回结果
            outputPort.present(toDTO(savedOrder));

        } catch (Exception e) {
            outputPort.presentError(e.getMessage());
        }
    }
}
```

### 3. Interface Adapters（接口适配器层）

```java
// Controller
@RestController
public class OrderController {
    private final InputPort<CreateOrderInput> createOrderUseCase;

    @PostMapping("/orders")
    public ResponseEntity<OrderDTO> createOrder(@RequestBody CreateOrderRequest request) {
        CreateOrderInput input = new CreateOrderInput() {
            @Override
            public Long getCustomerId() { return request.getCustomerId(); }
            @Override
            public List<OrderItemInput> getItems() { return request.getItems(); }
        };

        createOrderUseCase.execute(input);
        return ResponseEntity.ok(response.getOrder());
    }
}

// Gateway
public interface OrderRepository {

public interface CustomerRepository {
    Optional<Customer> findById(Long id);
}

public interface ProductRepository {
    Optional<Product> findById(Long id);
}
```

### 4. Frameworks（框架层）

```java
// Spring Data JPA 实现
@Repository
public class JpaOrderRepository implements OrderRepository {
    @Autowired
    private JpaOrderEntityRepository jpaRepository;

    @Override
    public Order save(Order order) {
        OrderEntity entity = OrderMapper.toEntity(order);
        entity = jpaRepository.save(entity);
        return OrderMapper.toDomain(entity);
    }

    @Override
    public Optional<Order> findById(Long id) {
        return jpaRepository.findById(id).map(OrderMapper::toDomain);
    }
}
```

## 五、整洁架构 vs 六边形架构 vs 洋葱架构

| 维度 | 整洁架构 | 六边形架构 | 洋葱架构 |
|------|----------|------------|----------|
| 提出者 | Robert C. Martin | Alistair Cockburn | Jeffrey Palermo |
| 核心概念 | 同心圆分层 | 端口与适配器 | 依赖向内 |
| 分层数量 | 4 层 | 2+ 层 | 4 层 |
| 依赖规则 | 只能向外依赖 | 端口隔离 | 依赖向内 |
| 适用场景 | 企业应用 | 集成系统 | DDD 项目 |

## 六、依赖规则详解

```java
/*
 * 整洁架构的依赖规则：
 *
 * 1. 实体层不依赖任何东西
 * 2. 用例层只依赖实体
 * 3. 适配器层依赖用例和实体
 * 4. 框架层依赖适配器
 */

// 错误示例：实体依赖了用例
public class Order {
    private CreateOrderUseCase useCase;  // ❌ 违反规则
}

// 正确示例：实体只包含业务规则
public class Order {
    // 纯粹的领域逻辑，无外部依赖
    public void addItem(Product product, int quantity) {
        // ...
    }
}
```

```java
    .resideInAnyPackage("..domain.entity..", "java.lang");
```
