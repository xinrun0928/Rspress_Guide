# 面试记录与复盘模板

上周有个学员给我看他的面试笔记，密密麻麻写了三页，但复盘的时候发现一个问题：记的都是流水账。

"HashMap问了三十分钟"、"JVM调优问了GC"——这种笔记有个鬼用，回头看根本想不起来当时怎么答的、哪里卡壳了、面试官追问的是什么。

真正有用的面试记录，是一份**作战地图**。你要记的不是"问了什么"，而是"我怎么答的"、"哪里被追问"、"暴露了什么弱点"、"下次怎么改"。

今天这套模板，是我带学员反复打磨出来的。照着写，坚持复盘，面试通过率能往上抬一个档次。

---

## 一、面试基本信息（5分钟内填完）

面试结束当天，趁记忆还热乎，5分钟内把基本信息填完。别拖，拖一晚上就忘了一半。

```markdown
## 面试记录

**公司**：[公司名]
**岗位**：[岗位名]
**面试官**：[面试官姓名/职位，如果记得的话]
**面试类型**：□一面 □二面 □三面 □HR面 □交叉面 □终面
**面试时长**：__分钟
**面试日期**：____年__月__日
**结果**：□通过 □待定 □淘汰 □未知
**薪资范围**：__K ~ __K（如果透露了的话）
```

这一Part看似简单，但80%的人压根不记。等你拿到offer要对比的时候，连当时面试官说了什么都想不起来。

【面试官手记】
我带过的学员里，有个习惯特别好的，每次面试完立刻在手机备忘录里写三行字：今天问了啥、我答得咋样、下次咋改。三个月下来，他的面试通过率从30%涨到了70%。差距就在这儿——有些人把每次面试当考试，有些人把每次面试当练习。

---

## 二、面试过程全记录

这是核心部分，要详细记。但记住，**不是记问答题答案，是记你的回答和面试官的反应**。

### 2.1 必记要素清单

| 要素 | 记什么 | 怎么记 |
| --- | --- | --- |
| **问题原文** | 面试官怎么问的 | 原话抄下来，特别是关键词 |
| **你的回答** | 你当时说了什么 | 用「我回答了...」开头，别记太详细 |
| **追问细节** | 面试官追问了什么 | 重点记！这是判断他满意度的关键 |
| **卡壳点** | 哪里卡了、停了、绕了 | 准确标记，这是你要改的地方 |
| **亮点** | 哪回答得自己都觉得漂亮 | 这要记下来，下次复用 |
| **氛围** | 面试官态度怎么样 | 友好/压迫/敷衍/感兴趣 |

### 2.2 记录模板

```markdown
### 问题1：[问题主题]

**面试官问**：________________[原话]________________

**我的回答**：
我回答了________________[简要描述你的回答]________________

**面试官追问**：
- 追问1：________________
- 追问2：________________
- 追问3：________________

**我的表现**：□满意 □一般 □不满意
**卡壳点**：________________[哪里卡了]________________
**下次改进**：________________[怎么答更好]________________
```

### 2.3 记录示例

```markdown
### 问题1：HashMap的put流程

**面试官问**："HashMap的put流程说一下。"

**我的回答**：
我说先计算hash，然后找到桶位置，插入元素，判断是否扩容。

**面试官追问**：
- 追问1："JDK8为什么引入红黑树？"
- 追问2："链表达到多少长度会树化？"
- 追问3："什么时候会退链表？"

**我的表现**：□满意 □一般 □不满意
**卡壳点**：被问到"为什么是8不是16"，我愣了两秒，开始瞎猜。
**下次改进**：要背熟树化阈值8和退链表阈值6的原文描述，理解泊松分布在其中的作用。
```

为什么要记追问？因为面试官不会无缘无故追问。追问的方向就是他想深入的地方，也是你暴露弱点的地方。

【面试官手记】
我观察过无数场面试，发现一个规律：面试官问完基础问题，如果候选人答得还行，下一个追问一定是顺着候选人答的某个点往下挖。这个"点"就是他的兴趣点，也可能是他的专业领域。你把这个点答好了，后面的问题都会围绕这个展开。你把这个点答崩了，整场面试就很难翻身了。

## 三、复盘分析（面试后24小时内）

基本信息是给自己看的，复盘分析是给自己提升用的。这一步要在面试后24小时内完成，否则新鲜感过了就懒得搞了。

### 3.1 通过/淘汰判定

```markdown
## 复盘判定

**本次面试表现**：□超常发挥 □正常水平 □失常发挥

**通过概率**：__%（主观判断）

**关键胜负手**：
________________[你觉得这场面试的关键在哪儿]________________
```

### 3.2 淘汰原因分析

如果你觉得这场要挂，或者已经挂了，认真分析原因：

```markdown
## 淘汰原因分析

**最大弱点**：________________[这场面试暴露的最严重问题]________________

**根本原因**：
- □ 知识盲区：________________[哪个知识点完全不会]________________
- □ 理解不深：________________[哪个点答出来了但被追问崩了]________________
- □ 表达混乱：________________[知道但说不清]________________
- □ 经验不足：________________[没有相关实战经验]________________
- □ 态度问题：________________[太紧张/太随意/太狂/太谦]________________
- □ 运气不好：________________[岗位不匹配/面试官眼缘]________________

**下次行动计划**：
1. ________________
2. ________________
3. ________________
```

### 3.3 通过原因分析

如果你觉得这场稳了，也要复盘：

```markdown
## 通过原因分析

**做对的点**：
- 1. ________________
- 2. ________________
- 3. ________________

**可以更好的点**：
- 1. ________________

**这些经验可以复用吗？怎么复用？**
________________[认真思考]________________
```

【面试官手记】
我带学员的时候，最怕两种人。一种是面试挂了不分析，觉得是运气不好，然后继续面、继续挂、继续怪运气。另一种是面试过了不分析，觉得是实力使然，然后下次用同样的方式挂另一家。真正能提升的人，不管是过还是挂，每次都会认真复盘，把偶然变成必然，把必然变成武器。

## 四、知识点追踪表

光记单场不够，还要追踪自己的知识点掌握情况。

```markdown
## 知识点追踪

| 知识点 | 掌握程度 | 挂过哪场 | 最近更新时间 |
| --- | --- | --- | --- |
| HashMap原理 | □精通 □熟悉 □了解 □陌生 | 字节x2、阿里x1 | 2024-01-15 |
| JVM调优 | □精通 □熟悉 □了解 □陌生 | 美团x1 | 2024-01-18 |
| Redis持久化 | □精通 □熟悉 □了解 □陌生 | 无 | 2024-01-20 |
```

这个表的价值在于：你能一眼看出自己的薄弱点在哪里，不用每次复盘都从头想"我哪个知识点不行"。

## 五、高频追问档案

把你被追问过的所有问题单独建档，这些都是**面试官最爱问的深水区**。

```markdown
## 高频追问档案

### HashMap相关
- **[追问问题]**：JDK8为什么引入红黑树？
- **[首次被问]**：____公司____岗位，20XX年XX月
- **[我当时答]**：________________
- **[标准答案]**：________________[自己总结或查资料补充]________________
- **[被追问次数]**：3次

### JVM相关
- **[追问问题]**：CMS和G1的区别是什么？
- **[首次被问]**：____公司____岗位，20XX年XX月
- **[我当时答]**：________________
- **[标准答案]**：________________
- **[被追问次数]**：2次
```

每被追问一次，就往里补充一次。积累多了，你会发现某些问题出现的频率特别高，这些就是你要重点准备的"面试高频深水区"。

【面试官手记】
我见过太多学员准备面试的方式：把网上能搜到的"Java面试题"背一遍，然后上场。问题是，网上那些题答案太标准化了，面试官稍微追问两句就穿帮。你真正要准备的不是"标准答案"，而是"被追问过的点"。这些点才是真正的高频考点，因为面试官真的在用这些问题筛人。

## 六、面试前检查清单

每次面试前，把这份清单过一遍：

```markdown
## 面试前检查清单

### 必查项
- [ ] 复习了最近挂过的3场面试记录
- [ ] 确认了公司业务和岗位要求
- [ ] 准备了3个可以讲的项目亮点
- [ ] 准备了好多问面试官的问题

### 心理建设
- [ ] 今晚早睡，明天精神好
- [ ] 准备好失败（降低预期，反而更放松）
- [ ] 穿什么衣服定好了

### 物品准备
- [ ] 简历打印了x份
- [ ] 笔记本和笔（有些公司要填表）
- [ ] 身份证/工牌
- [ ] 手机充满电
```

## 七、模板使用指南

这套模板用起来有三个关键：

**第一，当天记录，别拖。** 记忆这东西，第二天就忘一半，第三天全忘。面试结束地铁上、回家第一件事，把基本信息填了。详细记录可以晚上回家写，但基本框架要在当天拉起来。

**第二，24小时复盘，别懒。** 这个是真正的分水岭。记录只是输入，复盘才是内化。你要是不复盘，记录再多也是白搭。复盘不是写检讨，是分析"为什么"、"哪里可以改"、"下次怎么用"。

**第三，每周末回顾，别断。** 一周下来，可能面了三四场。这时候要抽出半小时，把这周的所有记录过一遍：你发现什么共性弱点？哪些点被反复追问？哪些地方有进步？

坚持一个月，你的面试状态会完全不一样。

【面试官手记】
有个学员跟我说，老师你这模板太麻烦了，我每次面试完都懒得写。我说你不写也行，那就继续挂。他后来咬着牙坚持了一个月，给我看他的记录本，说了一句话我印象特别深：原来我每次挂的都TM是同一个点。他后来专门花了两周把那个点怼透了，再面试，状态完全不一样。

## 附：快速记录模板（精简版）

如果你觉得完整版太麻烦，可以用这个精简版，地铁上都能写：

```markdown
## [公司]+[岗位]+[日期]

**表现**：□上 □中 □下

**亮点**：
- 1.________________

**卡壳**：
- 1.________________

**下次改进**：
- 1.________________

**知识点**：
- [掌握]________________
- [生疏]________________
```

先用精简版培养习惯，等习惯了再加详细的内容。不要一上来就追求完美，那只会让你坚持不下去。
