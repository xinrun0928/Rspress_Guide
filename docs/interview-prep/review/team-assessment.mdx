# 团队/业务/技术栈评估方法

「这家公司很好」——但你知道是哪家公司、哪个团队吗？

同一个大公司，不同团队之间的差距可能比不同公司之间的差距还大。本章帮你系统性地评估团队和业务。

## 面试官最关心的 3 个问题

> **问题 1**：如何评估一个团队是否值得加入？
> **问题 2**：如何了解业务的真实情况？
> **问题 3**：技术栈的选择重要吗？

## 团队评估

### 评估维度

| 维度 | 评估内容 | 方法 |
|------|----------|------|
| **领导** | 领导风格、能力、愿意培养人吗 | 面试观察、问 HR |
| **团队规模** | 多少人、是否有 HC | 问 HR |
| **协作方式** | 敏捷、扁平还是层级 | 面试观察 |
| **离职率** | 团队是否稳定 | 脉脉、牛客 |
| **技术氛围** | Code Review、技术分享 | 问面试官 |

### 评估方法

#### 方法 1：面试中观察

```
面试中注意：
- 面试官的态度和水平
- 团队的技术氛围
- 问的问题是否有深度
- 是否愿意回答你的问题
```

#### 方法 2：问面试官问题

```
✅ 可以问：
- 团队的技术栈是什么？
- 团队的技术分享机制是怎样的？
- 直属领导的管理风格如何？
- 试用期会有人带我熟悉业务吗？

❌ 不要问：
- 团队有没有加班
- 领导好不好相处（太主观）
```

#### 方法 3：通过外部渠道了解

```
渠道：
- 脉脉：公司点评、员工评价
- 牛客：员工分享、面试经验
- LinkedIn：前员工评价
- 朋友/同学：内部信息
```

### 评估清单

- [ ] 团队规模是否合适？（太大太散，太小太累）
- [ ] 领导是否愿意培养人？
- [ ] 团队技术氛围如何？
- [ ] 团队离职率如何？
- [ ] 工作节奏是否符合预期？

## 业务评估

### 评估维度

| 维度 | 评估内容 | 方法 |
|------|----------|------|
| **业务重要性** | 在公司的战略地位 | 问面试官、了解公司战略 |
| **业务增长** | 是否快速增长 | 财报、新闻 |
| **商业模式** | 能否盈利 | 财报、行业分析 |
| **竞争格局** | 竞争对手是谁 | 行业分析 |
| **政策风险** | 政策对业务的影响 | 新闻、政策 |

### 评估方法

#### 方法 1：了解公司战略

```
✅ 可以问：
- 这个业务在公司的战略地位如何？
- 未来一年的发展方向是什么？
- 业务的核心竞争力是什么？
```

#### 方法 2：行业分析

```
分析维度：
- 市场规模有多大？
- 行业增长率如何？
- 主要竞争对手是谁？
- 公司的差异化优势是什么？
```

#### 方法 3：财务数据

```
关注：
- 公司是否盈利？
- 业务增长率如何？
- 用户数据（DAU、GMV）增长情况？
```

### 业务评估清单

- [ ] 业务是否处于增长期？
- [ ] 业务的核心竞争力是什么？
- [ ] 竞争对手的威胁大吗？
- [ ] 有没有政策风险？
- [ ] 业务的盈利能力如何？

## 技术栈评估

### 评估维度

| 维度 | 评估内容 | 说明 |
|------|----------|------|
| **技术成熟度** | 技术是否主流 | 影响学习曲线 |
| **技术前沿性** | 是否跟进新技术 | 影响技术成长 |
| **技术匹配度** | 与你的技术背景是否匹配 | 影响面试难度 |
| **技术扩展性** | 能否扩展到其他领域 | 影响未来跳槽 |

### 技术栈类型

| 类型 | 示例 | 特点 |
|------|------|------|
| **主流技术** | Java、Spring、MySQL、Redis | 需求量大，好跳槽 |
| **前沿技术** | Rust、Go、云原生 | 薪酬高，门槛高 |
| **业务相关** | 金融、电商、AI | 与业务强相关 |
| **自研技术** | 内部框架、中间件 | 可迁移性差 |

### 技术栈评估清单

- [ ] 技术栈是否主流？
- [ ] 能否提升你的技术价值？
- [ ] 能否迁移到其他公司？
- [ ] 是否符合你的职业规划？

## ⚠️ 常见陷阱

### 陷阱 1：只看公司不看团队

```
❌ 错误：
「大公司就行，团队无所谓」

✅ 正确：
「先评估团队，再评估公司」
```

### 陷阱 2：只看业务不看技术

```
❌ 错误：
「业务好就行，技术不重要」

✅ 正确：
「好的技术栈能提升你的市场价值」
```

### 陷阱 3：被「前景好」忽悠

```
❌ 错误：
「面试官说业务前景很好」

✅ 正确：
「自己做一些调研，验证业务前景」
```

### 陷阱 4：忽视领导因素

```
❌ 错误：
「公司好就行，领导可以慢慢适应」

✅ 正确：
「领导是最影响你日常工作的人」
```

## 💡 评估优先级

### 评估优先级排序

```
1. 领导（最重要）
   - 好的领导能让你快速成长
   - 差的领导会让你痛苦不堪

2. 业务（第二重要）
   - 业务增长决定你的机会
   - 夕阳业务机会少

3. 技术栈（第三重要）
   - 好的技术栈提升市场价值
   - 影响未来跳槽

4. 薪资（在满足前三者后考虑）
```

## 常见问题

**Q：如何了解团队的真实情况？**

A：可以通过脉脉、牛客了解，也可以在面试中问面试官，或者找内部员工打听。

**Q：业务「前景好」是真的吗？**

A：需要自己判断。可以看财报、行业分析、政策导向等，不要只听面试官说。

**Q：技术栈和业务哪个更重要？**

A：都重要，但如果必须二选一，建议优先选择业务好的公司，技术栈可以慢慢学。
