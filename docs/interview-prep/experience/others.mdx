# 百度/京东/滴滴/拼多多

除了 BAT 三巨头，百度、京东、滴滴、拼多多也是很多程序员的热门选择。本章汇总了这些公司的面经。

## 百度面经

### 面试流程

| 特点 | 说明 |
|------|------|
| 流程 | 4-5 轮 |
| 风格 | 技术深度要求高 |
| 特色 | 自动驾驶、AI 相关岗位难度极高 |

### 高频考点

| 分类 | 高频考点 |
|------|----------|
| **算法** | 树、动态规划、回溯 |
| **搜索引擎** | 倒排索引、相关性计算 |
| **大数据** | Hadoop、Spark、Flink |
| **AI** | 机器学习、深度学习（AI 岗） |

### P6 面经

```
【一面】（60分钟）
- 自我介绍
- 项目介绍
- 分布式 ID 生成方案
- MySQL 索引优化
- Redis 持久化
- 算法题：二叉树层序遍历

【二面】（60分钟）
- 项目深挖
- 系统设计：设计一个搜索系统
- 缓存穿透、击穿、雪崩
- 算法题：编辑距离

【三面】（45分钟）
- 项目架构
- 技术判断力
- 职业规划
- 算法题：合并 K 个有序链表
```

## 京东面经

### 面试流程

| 特点 | 说明 |
|------|------|
| 流程 | 4 轮 |
| 风格 | 业务与技术并重 |
| 特色 | 重视电商业务理解 |

### 高频考点

| 分类 | 高频考点 |
|------|----------|
| **电商业务** | 订单系统、库存系统、支付系统 |
| **高并发** | 秒杀、抢购、削峰 |
| **分布式** | 分布式事务、分布式锁 |
| **中间件** | Redis、Kafka、Dubbo |

### P6 面经

```
【一面】（60分钟）
- 自我介绍
- 项目介绍
- 秒杀系统设计
- Redis 分布式锁
- MySQL 事务隔离级别
- 算法题：滑动窗口最大值

【二面】（60分钟）
- 项目深挖
- 库存超卖问题如何解决？
- 如何保证消息不丢失？
- 算法题：LRU 缓存

【三面】（45分钟）
- 电商业务理解
- 职业规划
- 算法题：接雨水
```

| 分类 | 高频考点 |

```

- 自我介绍
- 项目介绍
- 订单匹配算法
- Redis 实时统计
- 算法题：二叉树最近公共祖先

【二面】（60分钟）
- 项目深挖
- 调度算法设计
- 高并发订单处理
- 算法题：无重复字符最长子串

【三面】（45分钟）
- 系统设计：设计一个打车系统
- 职业规划
```

## 拼多多面经

### 面试流程

| 特点 | 说明 |
|------|------|
| 流程 | 3-4 轮 |
| 风格 | 算法为主、节奏快 |
| 特色 | 高强度工作、薪资高 |

### 高频考点

| 分类 | 高频考点 |
|------|----------|
| **算法** | 动态规划、滑动窗口、高频算法题 |
| **基础** | 网络、操作系统、数据库 |
| **系统设计** | 高并发设计 |

### P6 面经

```
【一面】（60分钟）
- 自我介绍
- 算法：编辑距离
- 算法：合并区间
- MySQL 索引结构
- Redis 数据结构

【二面】（60分钟）
- 项目介绍
- 算法：接雨水
- 算法：最小路径和
- 系统设计：设计一个秒杀系统

【三面】（45分钟）
- 项目深挖
- 算法：LRU 缓存
- 职业规划
```

## 公司对比

| 公司 | 难度 | 特色 | 薪资 |
|------|------|------|------|
| 百度 | 高 | AI、技术深度 | 高 |
| 京东 | 中高 | 电商业务 | 中高 |
| 滴滴 | 中高 | 实时系统 | 中高 |
| 拼多多 | 高 | 算法、节奏快 | 很高 |

## 常见问题

**Q：这些公司面试难度如何？**

A：总体难度从高到低：拼多多 > 百度 > 滴滴 > 京东。

**Q：哪个公司薪资最高？**

A：拼多多的薪资通常最高，但也对应更高的工作强度。
