# 岗位级别对标

「P6 对标阿里的什么级别？」「字节的 2-2 是 P7 吗？」——级别对标是面试准备中非常重要的一环。

## 面试官最关心的 3 个问题

> **问题 1**：国内各公司的级别如何对标？
> **问题 2**：不同级别对应的能力要求是什么？
> **问题 3**：如何判断自己应该投什么级别？

## 各级别能力要求

### P5 / T9 / L3（初级工程师）

| 维度 | 要求 |
|------|------|
| **技术** | 熟练使用常用技术，能独立完成分配的任务 |
| **项目** | 参与项目，能完成具体功能 |
| **协作** | 在指导下工作，能与团队协作 |
| **学习** | 学习能力强，能快速上手新技能 |

### P6 / T9-T10 / L4-L5（高级工程师）

| 维度 | 要求 |
|------|------|
| **技术** | 深入理解技术原理，能解决复杂问题 |
| **项目** | 独立负责模块，能主导项目 |
| **协作** | 指导 P5，能跨团队协作 |
| **架构** | 具备基本的架构设计能力 |

### P7 / T11 / L6（专家/Leader）

| 维度 | 要求 |
|------|------|
| **技术** | 深度掌握某一领域，能做技术选型 |
| **项目** | 主导跨团队项目 |
| **协作** | 培养团队，具备 Leadership |
| **架构** | 能设计中等规模系统 |
| **业务** | 理解业务，能平衡技术和业务 |

### P8 / T12 / L7（高级专家/架构师）

| 维度 | 要求 |
|------|------|
| **技术** | 多个领域精通，能定义技术方向 |
| **项目** | 主导公司级项目 |
| **协作** | 带领团队，影响多个团队 |
| **架构** | 能设计大规模系统 |
| **业务** | 深度理解业务，能驱动业务增长 |

## ⚠️ 级别选择的常见问题

### 问题 1：高估自己

```
表现：工作 3 年，投 P7/P8

后果：面试难度超出能力范围，容易挂

正确做法：评估自己的实际能力和经验，选择匹配的级别
```

### 问题 2：低估自己

```
表现：工作 6 年，投 P5/P6

后果：薪资涨幅受限，可能不被录用

正确做法：参考同级别同事的水平，合理评估
```

### 问题 3：跨级投递

```
表现：当前 P6，直接投 P8

后果：难度跳跃太大

正确做法：逐级晋升，或选择合适的级别
```

## 如何确定自己的级别

### 方法 1：对标同事

```
和同级别同事对比，评估自己的能力和经验
```

### 方法 2：看岗位要求

```
仔细阅读岗位 JD，对照自己的能力
```

### 方法 3：找内部人咨询

```
通过内推人了解真实的级别要求
```

## 常见问题

**Q：级别差一级面试难度差多少？**

A：级别越高，难度显著增加。P6 到 P7 的差距比 P5 到 P6 的差距更大。

**Q：如果级别投错了会怎么样？**

A：可能被降级录用，也可能被拒。建议仔细评估后再投递。

**Q：可以同时投递不同级别吗？**

A：通常不建议，同一公司只能投递一个岗位。
