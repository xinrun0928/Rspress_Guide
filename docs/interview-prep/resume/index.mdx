# 简历优化指南

【面试官手记】
我每年要看上千份简历，平均每份简历的阅读时间不超过30秒。在阿里的时候，我们组每周收到几十份简历，能进入面试环节的不超过10%。很多人不是能力不行，是简历写得不行。这篇指南，帮你把简历从"筛掉"变成"必看"。

## 目录概述

简历是求职的敲门砖，也是面试官对候选人的第一印象。一份好的简历，应该在30秒内让面试官看到：
- 你做过什么（项目经历）
- 你做到什么程度（成果量化）
- 你适合这个岗位（技能匹配）

本目录提供从简历框架到细节优化的全套指南，帮你把15年的技术经验浓缩进一张纸里。

## 学习路径指引

**第一步：搞清楚简历的筛选逻辑**

面试官看简历，只有两个核心问题：
1. 你能不能做这个活？（技能匹配）
2. 你做得怎么样？（成果说话）

30秒内回答不了这两个问题，简历就被扔进"待定"或"不合适"的堆里。

**第二步：掌握STAR法则**

项目描述用STAR法则结构化：
- **S**ituation：项目背景（团队、规模、挑战）
- **T**ask：你的职责（负责什么、承担什么角色）
- **A**ction：具体行动（怎么做的、用了什么技术）
- **R**esult：量化成果（带来什么价值、数据说话）

**第三步：关键词匹配**

JD（职位描述）里提到的技术栈和关键词，一定要在简历中体现。但不要堆砌，要自然融入项目描述。

## 避坑指南

**坑一：简历写成了JD（职位描述）**

很多人把简历写成"我负责XXX，负责YYY，负责ZZZ"。这是职位描述，不是你的简历。

正确写法：用STAR法则，重点写你**做了什么**，**怎么做的**，**做到了什么**。

**坑二：项目经历堆砌，没有重点**

最多写3-4个核心项目，每个项目2-4句话。面试官没时间看流水账。

**坑三：技术栈堆砌，不写深度**

不要只写"熟悉Spring、MySQL、Redis"，要写你在项目中怎么用的，用到了什么深度。

**坑四：成果没有量化**

"提升了系统性能"不是成果，"QPS从1000提升到5000，延迟从50ms降到5ms"才是成果。

**坑五：简历格式不规范**

- PDF优先，Word可能乱码
- 控制在一页或两页以内
- 字体统一，行距统一
- 联系方式要明显

**坑六：投不同的岗位用同一份简历**

根据JD调整简历侧重点，技术栈和项目经历要和目标岗位匹配。

:::tip 💡
写简历的核心心法：站在面试官的角度想问题。面试官想要一个能解决问题的人，所以你要展示的不是"你会什么"，而是"你用过什么解决了什么问题"。
:::

:::warning ⚠️
简历上的每一个字都要能经受追问。面试官问"你这个项目用了什么方案解决了什么问题"，你必须能讲清楚。如果写在简历上的东西讲不清楚，面试官会认为你是编的。
:::
