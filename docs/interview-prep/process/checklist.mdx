# 面试前中后 Checklist

面试是一场系统工程，每个环节都可能影响最终结果。很多候选人技术能力不错，却因为忽视了某些「小细节」而功亏一篑。本章将提供一份完整的面试前、中、后的 Checklist，帮助你把每个环节都做到位。

## 面试官最关心的 3 个问题

> **问题 1**：面试前最容易被忽视的准备工作是什么？
> **问题 2**：面试中如何展示自己的最佳状态？
> **问题 3**：面试后还有哪些事情需要做？

## 面试前 Checklist（面试前 1-2 天）

### 技术准备

- [ ] **核心知识点复习**：回顾目标公司的面试考察范围，确认自己的薄弱点
- [ ] **高频算法题**：完成 5-10 道高频算法题，保持手感
- [ ] **项目经历梳理**：用 STAR 法则整理每个项目的关键细节
- [ ] **系统设计准备**：如果是 P6+，准备 2-3 个系统设计案例
- [ ] **公司技术栈研究**：了解目标公司的技术栈、使用的框架和工具

### 信息准备

- [ ] **了解公司业务**：阅读公司的产品介绍、年度报告、新闻动态
- [ ] **了解团队情况**：如果是内推，了解团队的规模、技术方向、正在做的项目
- [ ] **确认面试时间**：提前和 HR 确认面试时间，避免记错
- [ ] **确认面试形式**：线上还是线下？视频面试需要提前测试设备
- [ ] **确认面试官信息**：如果有面试官的 LinkedIn 或技术博客，可以提前了解

### 物资准备

| 物品 | 作用 | 备注 |
|------|------|------|
| 简历 | 现场面试参考 | 多带几份备用 |
| 电脑/平板 | 白板写代码备用 | 确保电量充足 |
| 纸笔 | 记录思路或写提纲 | 有备无患 |
| 水杯 | 保持喉咙湿润 | 面试可能会很口渴 |
| 身份证/工牌 | 部分公司需要 | 入场凭证 |

### 环境准备

- [ ] **线上面试**：提前测试网络、摄像头、麦克风
- [ ] **线下面试**：提前查好路线，预留 15 分钟 buffer
- [ ] **安静空间**：确保面试环境安静，不被打扰
- [ ] **背景整洁**：如果是视频面试，确保背景干净整洁

### 心理准备

- [ ] **调整心态**：把面试当作「和同行交流技术」，而不是「被审判」
- [ ] **准备反问**：准备 3-5 个好的反问问题，展示你对公司的兴趣
- [ ] **预设问题**：预演可能被问到的问题，准备好回答
- [ ] **充足睡眠**：面试前一天保证 7-8 小时睡眠

### ⚠️ 面试前一天千万不要做

- ❌ 刷夜复习（影响第二天状态）
- ❌ 剧烈运动（避免受伤）
- ❌ 大量饮酒（影响思维清晰度）
- ❌ 临时修改简历（可能造成版本混乱）
- ❌ 临时换发型/穿新衣服（容易分散注意力）

## 面试中 Checklist

### 开场部分（前 5 分钟）

- [ ] **准时进入**：提前 5-10 分钟进入面试状态
- [ ] **礼貌问候**：简单的寒暄，展示职业素养
- [ ] **状态调整**：深呼吸，调整到最佳状态

### 自我介绍（2-3 分钟）

```
建议结构：
1. 基本信息：姓名、当前职位、工作年限
2. 核心技术栈：2-3 个你最擅长的技术方向
3. 代表性项目：1-2 个最能体现你能力的项目
4. 求职动机：为什么对这个岗位感兴趣
```

**⚠️ 陷阱**：自我介绍不要超过 3 分钟，信息量太大面试官记不住。

### 项目经历介绍

- [ ] **STAR 法则**：Situation（背景）、Task（任务）、Action（行动）、Result（结果）
- [ ] **数据说话**：用具体数字展示成果（如「QPS 提升 50%」）
- [ ] **技术深度**：展示你在项目中承担的技术决策和思考
- [ ] **团队贡献**：展示你的协作能力和 Leadership

**⚠️ 陷阱**：不要把项目说得太「个人英雄主义」，要展示团队协作。

### 算法/编码环节

- [ ] **确认题意**：先确认题目理解是否正确，不要急着写
- [ ] **思路先行**：先说思路，再写代码
- [ ] **边写边说**：不要闷头写，要展示思考过程
- [ ] **边界处理**：注意边界条件和异常情况
- [ ] **代码规范**：命名规范、格式整洁

**⚠️ 陷阱**：不要一看到题目就写，先想清楚再动手。

### 反问环节（最后 5 分钟）

这是你了解公司、展示自己的重要机会：

**好的反问问题**：

```
技术层面：
- 团队目前面临的技术挑战是什么？
- 团队的技术栈和架构是怎样的？
- 入职后会负责什么类型的项目？

团队层面：
- 团队的规模和组织结构是怎样的？
- 团队的工作节奏和协作方式是怎样的？
- 试用期会有人带我熟悉业务吗？

成长层面：
- 对这个岗位的成长路径有什么期待？
- 团队的技术分享和培训机制是怎样的？
- 之前在这个岗位的人，发展得怎么样？
```

**不好的反问问题**（避免问）：

```
❌ 「你们加班多吗？」
❌ 「薪资待遇怎么样？」
❌ 「这个岗位有晋升空间吗？」
❌ 「你们公司是做什么的？」（说明你之前没做功课）
```

### 面试中突发情况处理

| 情况 | 应对策略 |
|------|----------|
| 题目太难 | 「这个题目我之前没遇到过，让我分析一下……」展示思考过程 |
| 面试官态度不好 | 保持专业，专注于展示自己的能力 |
| 突然忘词 | 「让我想一下」给自己时间缓冲，不要硬撑 |
| 面试超时 | 配合面试官，但如果时间紧张可以礼貌提醒 |
| 网络卡顿（视频面试） | 提前告知，准备备用联系方式 |

## 面试后 Checklist

### 面试结束后（24 小时内）

- [ ] **发送感谢邮件**：简短、专业，感谢面试官的时间
- [ ] **记录面试内容**：趁记忆清晰，记录被问到的问题和自己的回答
- [ ] **评估表现**：给自己打分，找出需要改进的地方
- [ ] **跟进进度**：如果 3-5 天没收到回复，主动联系 HR 询问进度

### 感谢邮件模板

```
标题：感谢面试 - [姓名] - [岗位名称]

尊敬的 [面试官姓名]：

感谢您今天抽出宝贵时间面试我。

面试过程中，[面试官姓名] 对 [团队/业务方向] 的介绍让我印象深刻，尤其是 [提及面试中聊到的具体内容]，这让我更加确信贵司是我希望加入的平台。

如果面试中有任何我没有表达清楚的地方，欢迎随时联系我。

期待能有机会成为团队的一员。

祝工作顺利！

[姓名]
[联系方式]
[日期]
```

### 面试复盘（每场面试后）

| 维度 | 问题 |
|------|------|
| **整体表现** | 这次面试的整体感觉如何？ |
| **亮点** | 哪些问题回答得最好？为什么？ |
| **不足** | 哪些问题回答得不好？差在哪里？ |
| **改进** | 下次遇到类似问题，应该怎么回答？ |
| **知识盲点** | 暴露了哪些知识点需要补充？ |

### ⚠️ 面试后常见错误

**错误 1**：面试完就再也不管了

正确做法：面试后 24 小时内发送感谢邮件，3-5 天没回复要主动跟进。

**错误 2**：面试后立刻问结果

正确做法：不要当天问结果，等 HR 通知。如果流程上有问题，HR 会主动联系你。

**错误 3**：面试后停止准备

正确做法：继续准备下一场面试，每场面试都是一次练习。

**错误 4**：面试后到处抱怨

正确做法：保持职业素养，即使面试失败也不要公开批评面试官或公司。

### 一面 Checklist

- [ ] 基础知识扎实展示
- [ ] 项目经历真实性
- [ ] 算法题完成质量

### 二面 Checklist

- [ ] 技术深度展示

- [ ] 薪资期望合理表达

- [ ] 对公司的兴趣表达

## 💡 加分细节

### 面试中的加分小动作

| 动作 | 效果 |
|------|------|
| 面试前喝一杯温水 | 保持喉咙湿润，声音更清晰 |
| 带着微笑进会议室 | 第一印象很重要 |
| 面试前关掉手机 | 避免尴尬 |
| 眼神交流 | 展示自信和专注 |
| 记笔记 | 展示认真态度 |

### 面试后的加分行为

| 行为 | 效果 |
|------|------|
| 发送感谢邮件 | 展示职业素养 |
| 面试后给 HR 发消息 | 保持沟通 |
| 面试后给内推人反馈 | 维护关系 |
| 面试后在脉脉上关注面试官 | 扩大社交圈 |

## 常见问题

**Q：面试中可以录音吗？**

A：通常不建议，这会让面试官感到不舒服。但如果是视频面试且面试官同意，可以录音以便事后复盘。

**Q：面试中被问到不会的问题怎么办？**

A：诚实说「这个我不太确定」，但展示思考过程：「不过我分析可能是……」。不要瞎猜，更不要不懂装懂。

**Q：面试中感觉面试官对我不感兴趣怎么办？**

A：不要受面试官情绪影响，保持专业态度。有时候面试官只是性格内向，不代表你表现不好。

**Q：面试结束后可以询问反馈吗？**

A：可以，但通过 HR 渠道询问更合适。在感谢邮件中可以简单表达「希望了解自己的不足以便改进」。
