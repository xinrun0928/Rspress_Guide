# MyBatis Plus 快速上手：最少代码完成 CRUD

继承 `BaseMapper<T>`，居然就有了一堆 CRUD 方法？

MyBatis Plus 的入门非常简单——只需要引入依赖、配置连接、创建实体类，就能直接使用强大的 CRUD 功能。

这篇文章，带你用 10 分钟完成 MyBatis Plus 的快速入门。

## 引入依赖

### Maven

```xml
<dependencies>
    <!-- Spring Boot 集成方式（推荐） -->
    <dependency>
        <groupId>com.baomidou</groupId>
        <artifactId>mybatis-plus-boot-starter</artifactId>
        <version>3.5.5</version>
    </dependency>

    <!-- Spring MVC 集成方式 -->
    <dependency>
        <groupId>com.baomidou</groupId>
        <artifactId>mybatis-plus</artifactId>
        <version>3.5.5</version>
    </dependency>
</dependencies>
```

### Spring Boot 配置

```yaml
# application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mybatis_plus?useUnicode=true&characterEncoding=utf-8
    username: root
    password: password
    driver-class-name: com.mysql.cj.jdbc.Driver

# MyBatis Plus 配置
mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  configuration:
    map-underscore-to-camel-case: true
  global-config:
    db-config:
      id-type: auto  # 主键自增
```

```java

    }

```

## 核心注解

### @TableName：实体类与表名映射

```java
@TableName("sys_user")  // 指定表名
public class User {
    private Long id;
    private String name;
    private Integer age;
    private String email;
}
```

**自动映射规则**：
- 默认将类名转成下划线形式：`User` → `user`
- 如果实体类前缀是 `sys_`，可以使用 `autoResultMap = true`

```java
@TableName(value = "sys_user", autoResultMap = true)
public class User {
    // ...
}
```

### @TableId：主键策略

```java
@TableName("user")
public class User {

    @TableId(type = IdType.AUTO)  // 数据库自增
    private Long id;

    @TableId(type = IdType.INPUT)  // 用户输入
    private Long id;

    @TableId(type = IdType.ASSIGN_ID)  // 雪花算法（默认）
    private Long id;

    @TableId(type = IdType.ASSIGN_UUID)  // UUID
    private String id;

    private String name;
}
```

| 主键类型 | 说明 |
|---------|------|
| `AUTO` | 数据库自增 |
| `INPUT` | 用户输入 |
| `ASSIGN_ID` | 雪花算法生成（默认，推荐） |
| `ASSIGN_UUID` | UUID |
| `NONE` | 无主键 |

**雪花算法（ASSIGN_ID）**：

```java
// 生成的 ID 示例
1876543210123456789L
```

优点：分布式环境下保证 ID 唯一性
缺点：ID 是长整型，不够友好

### @TableField：字段映射

```java
@TableName("user")
public class User {

    @TableId(type = IdType.AUTO)
    private Long id;

    @TableField("user_name")  // 数据库字段名
    private String name;

    @TableField(exist = false)  // 非数据库字段
    private String extra;

    @TableField("age")
    private Integer age;

    @TableField(select = false)  // 查询时不返回
    private String password;
}
```

| 属性 | 说明 |
|------|------|
| `value` | 数据库字段名 |
| `exist` | 是否为数据库字段 |
| `select` | 查询时是否返回 |
| `fill` | 自动填充策略 |

### @TableLogic：逻辑删除

```java
@TableName("user")
public class User {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String name;

    @TableLogic  // 逻辑删除字段
    private Integer deleted;
}
```

**效果**：
- 删除操作：`UPDATE user SET deleted = 1 WHERE id = ?`
- 查询操作：`SELECT * FROM user WHERE deleted = 0`

## `BaseMapper<T>` 内置方法

继承 `BaseMapper<User>` 后，自动获得以下方法：

```java
public interface UserMapper extends BaseMapper<User> {
    // 自定义方法可以添加在这里
}
```

### 查询方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `selectById(id)` | 根据 ID 查询 | `userMapper.selectById(1L)` |
| `selectBatchIds(ids)` | 根据 ID 批量查询 | `userMapper.selectBatchIds(List.of(1, 2, 3))` |
| `selectByMap(map)` | 根据 Map 条件查询 | `userMapper.selectByMap(Map.of("name", "张三"))` |
| `selectList(wrapper)` | 条件查询 | `userMapper.selectList(wrapper)` |
| `selectMaps(wrapper)` | 查询 Map 列表 | `userMapper.selectMaps(wrapper)` |
| `selectObjs(wrapper)` | 查询单个字段 | `userMapper.selectObjs(wrapper)` |
| `selectCount(wrapper)` | 统计数量 | `userMapper.selectCount(wrapper)` |
| `selectOne(wrapper)` | 查询一条 | `userMapper.selectOne(wrapper)` |
| `selectPage(page, wrapper)` | 分页查询 | `userMapper.selectPage(page, wrapper)` |

### 插入方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `insert(entity)` | 插入一条记录 | `userMapper.insert(user)` |

```java
User user = new User();
user.setName("张三");
user.setAge(20);
user.setEmail("zhangsan@example.com");

// 插入后，主键会自动回填
int rows = userMapper.insert(user);
System.out.println(user.getId());  // 自动填充了 ID
```

### 更新方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `updateById(entity)` | 根据 ID 更新 | `userMapper.updateById(user)` |
| `update(entity, wrapper)` | 条件更新 | `userMapper.update(user, wrapper)` |

```java
// 更新指定字段
User user = new User();
user.setId(1L);
user.setName("李四");
userMapper.updateById(user);
// UPDATE user SET name = '李四' WHERE id = 1
```

### 删除方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `deleteById(id)` | 根据 ID 删除 | `userMapper.deleteById(1L)` |
| `deleteBatchIds(ids)` | 批量删除 | `userMapper.deleteBatchIds(List.of(1, 2))` |
| `deleteByMap(map)` | Map 条件删除 | `userMapper.deleteByMap(Map.of("name", "张三"))` |
| `delete(wrapper)` | 条件删除 | `userMapper.delete(wrapper)` |

## 完整示例

### 实体类

```java
package com.example.entity;

import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;

@Data
@TableName("user")
public class User {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String name;

    private Integer age;

    private String email;

    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;

    @TableLogic
    private Integer deleted;
}
```

### Mapper 接口

```java
package com.example.mapper;

import com.baomidou.mybatisplus.core.mapper.BaseMapper;
import com.example.entity.User;
import org.apache.ibatis.annotations.Mapper;

@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 自定义方法
    User selectByEmail(String email);
}
```

### Mapper XML

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.mapper.UserMapper">

    <select id="selectByEmail" resultType="User">
        SELECT * FROM user WHERE email = #{email}
    </select>

</mapper>
```

### Service 层

MyBatis Plus 还提供了 `IService` 接口：

```java
// Service 接口
public interface UserService extends IService<User> {
    // 自定义方法
}

// Service 实现
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {
    // 继承了大量 CRUD 方法
}
```

### Controller 层

```java
@RestController
@RequestMapping("/user")
public class UserController {

    @Autowired
    private UserService userService;

    @GetMapping("/{id}")
    public User getById(@PathVariable Long id) {
        return userService.getById(id);
    }

    @PostMapping
    public boolean save(@RequestBody User user) {
        return userService.save(user);
    }

    @PutMapping
    public boolean update(@RequestBody User user) {
        return userService.updateById(user);
    }

    @DeleteMapping("/{id}")
    public boolean delete(@PathVariable Long id) {
        return userService.removeById(id);
    }
}
```

## 全局配置

### application.yml

```yaml
mybatis-plus:
  # mapper XML 路径
  mapper-locations: classpath*:/mapper/**/*.xml

  # 全局配置
  global-config:
    db-config:
      # 主键策略
      id-type: assign_id
      # 逻辑删除字段
      logic-delete-field: deleted
      # 逻辑删除值
      logic-delete-value: 1
      # 未删除值
      logic-not-delete-value: 0
      # 表前缀
      table-prefix: sys_

  # 配置扫描 type-aliases
  type-aliases-package: com.example.entity

  # 配置扫描枚举类
  type-enums-package: com.example.enums
```

### Java 配置类

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        // 分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor());

        // 乐观锁插件
        interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());

        // 多租户插件
        // interceptor.addInnerInterceptor(new TenantLineInnerInterceptor(new TenantLineHandlerImpl()));

        return interceptor;
    }
}
```

## 常见问题

### Q1：主键返回为 0？

检查是否配置了 `useGeneratedKeys`：

```yaml
mybatis-plus:
  global-config:
    db-config:
      id-type: auto
```

### Q2：字段映射失败？

检查是否开启了驼峰转换：

```yaml
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
```

### Q3：逻辑删除不生效？

1. 确保字段加了 `@TableLogic` 注解
2. 确保全局配置了逻辑删除字段
