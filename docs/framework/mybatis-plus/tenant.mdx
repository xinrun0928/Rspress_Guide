# MyBatis Plus 多租户插件：TenantLineInnerInterceptor

多租户系统是什么？为什么要隔离租户数据？

在 SaaS 系统中，多个租户共享同一个数据库实例，但每个租户只能访问自己的数据。多租户插件就是实现这一隔离的利器。

## 多租户概念

### 什么是多租户

```
┌─────────────────────────────────────────────────────────────┐
│                        数据库                                 │
│  ┌─────────────────┐  ┌─────────────────┐                    │
│  │    租户 A 数据    │  │    租户 B 数据    │                    │
│  │  tenant_id = 1  │  │  tenant_id = 2  │                    │
│  └─────────────────┘  └─────────────────┘                    │
└─────────────────────────────────────────────────────────────┘
```

三种多租户架构：

| 架构 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 独立数据库 | 每个租户一个数据库 | 隔离性最好 | 成本高 |
| 共享数据库，独立 Schema | 每个租户一个 Schema | 成本较低 | 隔离性一般 |
| 共享数据库，共享 Schema | 所有租户共用一个数据库和表 | 成本最低 | 需要应用层隔离 |

MyBatis Plus 的多租户插件，适用于**共享数据库，共享 Schema** 的场景。

## 多租户插件配置

### 第一步：实现 TenantLineHandler

```java
@Component
public class TenantLineHandlerImpl implements TenantLineHandler {

    @Override
    public Expression getTenantId() {
        // 从 ThreadLocal 或 SecurityContext 获取当前租户 ID
        return new LongValue(TenantContext.getCurrentTenantId());
    }

    @Override
    public String getTenantIdColumn() {
        // 租户 ID 的列名
        return "tenant_id";
    }

    @Override
    public boolean ignoreTable(String tableName) {
        // 忽略租户隔离的表
        return "sys_user".equals(tableName) || "sys_role".equals(tableName);
    }
}
```

### 第二步：配置插件

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        // 多租户插件（必须在最前面）
        interceptor.addInnerInterceptor(new TenantLineInnerInterceptor(new TenantLineHandlerImpl()));

        // 分页插件（多租户之后添加）
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor());

        return interceptor;
    }
}
```

### 第三步：设置租户 ID

```java
// 在过滤器或拦截器中设置租户 ID
public class TenantFilter extends HandlerInterceptorAdapter {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) {
        String tenantId = request.getHeader("X-Tenant-ID");
        if (StringUtils.hasText(tenantId)) {
            TenantContext.setTenantId(Long.parseLong(tenantId));
        }
        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                 Object handler, Exception ex) {
        TenantContext.clear();
    }
}
```

## TenantLineHandler 接口详解

### 接口方法

```java
public interface TenantLineHandler {

    /**
     * 获取租户 ID 表达式
     */
    Expression getTenantId();

    /**
     * 获取租户 ID 列名
     */
    default String getTenantIdColumn() {
        return "tenant_id";
    }

    /**
     * 忽略的表（不走租户隔离）
     */
    default boolean ignoreTable(String tableName) {
        return false;
    }

    /**
     * 忽略的 SQL 类型
     */
    default boolean ignoreSql(SqlCommandType sqlCommandType) {
        return false;
    }
}
```

### 常用实现

```java
@Component
public class TenantLineHandlerImpl implements TenantLineHandler {

    /**
     * 从 ThreadLocal 获取租户 ID
     */
    @Override
    public Expression getTenantId() {
        Long tenantId = TenantContext.getCurrentTenantId();
        return new LongValue(tenantId);
    }

    /**
     * 租户 ID 列名
     */
    @Override
    public String getTenantIdColumn() {
        return "tenant_id";
    }

    /**
     * 忽略系统表
     */
    @Override
    public boolean ignoreTable(String tableName) {
        return "sys_config".equals(tableName)
            || "sys_user".equals(tableName)
            || "sys_role".equals(tableName);
    }
}
```

## 自动注入原理

### SQL 自动修改

```sql
-- 原始 SQL
SELECT * FROM user WHERE id = 1

-- 插件自动修改后
SELECT * FROM user WHERE tenant_id = 1 AND id = 1
```

```sql
-- 原始 SQL
INSERT INTO user (name, email) VALUES ('张三', 'zhangsan@example.com')

-- 插件自动修改后
INSERT INTO user (tenant_id, name, email) VALUES (1, '张三', 'zhangsan@example.com')
```

```sql
-- 原始 SQL
UPDATE user SET name = '李四' WHERE id = 1

-- 插件自动修改后
UPDATE user SET name = '李四' WHERE tenant_id = 1 AND id = 1
```

### 支持的 SQL 类型

| SQL 类型 | 自动修改 |
|---------|---------|
| SELECT | 添加 WHERE 条件 |
| INSERT | 添加租户 ID 列和值 |
| UPDATE | 添加 WHERE 条件 |
| DELETE | 添加 WHERE 条件 |

## 忽略特定查询

### 注解方式

```java
// Mapper 方法上使用
public interface UserMapper extends BaseMapper<User> {

    // 忽略租户隔离（查询所有租户的数据）
    @InterceptorIgnore(tenantLine = "true")
    List<User> selectAllTenants();
}
```

### 全局忽略

```java
@Override
public boolean ignoreTable(String tableName) {
    // 配置忽略的表
    return Arrays.asList("sys_user", "sys_role", "sys_config").contains(tableName);
}
```

### SQL 类型忽略

```java
@Override
public boolean ignoreSql(SqlCommandType sqlCommandType) {
    // 忽略 DELETE 操作的租户隔离
    return SqlCommandType.DELETE.equals(sqlCommandType);
}
```

## 多租户隔离实现思路

### 方案一：ThreadLocal 存储

```java
public class TenantContext {

    private static final ThreadLocal<Long> TENANT_ID = new ThreadLocal<>();

    public static void setTenantId(Long id) {
        TENANT_ID.set(id);
    }

    public static Long getTenantId() {
        return TENANT_ID.get();
    }

    public static void clear() {
        TENANT_ID.remove();
    }
}
```

### 方案二：拦截器自动设置

```java
@Component
public class TenantInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) {
        // 从请求头、Cookie 或 Token 中获取租户 ID
        String tenantId = getTenantId(request);

        if (tenantId != null) {
            TenantContext.setTenantId(Long.parseLong(tenantId));
        }

        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) {
        TenantContext.clear();
    }

    private String getTenantId(HttpServletRequest request) {
        // 优先级：Header > Cookie > Token
        String tenantId = request.getHeader("X-Tenant-ID");
        if (tenantId == null) {
            tenantId = request.getHeader("Tenant-ID");
        }
        return tenantId;
    }
}
```

## 完整示例

### 数据库表结构

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id BIGINT NOT NULL,
    name VARCHAR(50),
    email VARCHAR(100),
    INDEX idx_tenant_id (tenant_id)
);
```

### 实体类

```java
@TableName("user")
public class User {

    @TableId(type = IdType.AUTO)
    private Long id;

    // 不需要显式声明 tenant_id，插件会自动处理
    private String name;

    private String email;
}
```

### Mapper

```java
@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 所有方法自动支持多租户隔离
}
```

### 使用

```java
@RestController
public class UserController {

    @Autowired
    private UserMapper userMapper;

    @GetMapping("/users")
    public List<User> getUsers() {
        // 自动添加 tenant_id 条件
        // TenantContext 中设置的租户 ID 会被使用
        return userMapper.selectList(null);
    }

    @PostMapping("/users")
    public boolean addUser(@RequestBody User user) {
        // 自动注入 tenant_id
        return userMapper.insert(user) > 0;
    }

    @GetMapping("/users/all")
    public List<User> getAllUsers(UserMapper userMapper) {
        // 忽略租户隔离，查询所有租户的数据
        return userMapper.selectAllTenants();
    }
}
```

## 常见问题

### Q1：为什么多租户插件要放在最前面？

插件按顺序执行，放在前面的插件先拦截 SQL。多租户插件需要先修改 SQL，添加 WHERE 条件。

### Q2：如何查询跨租户的数据？

```java
// 使用 @InterceptorIgnore 注解忽略租户隔离
@InterceptorIgnore(tenantLine = "true")
List<User> selectAllTenants();
```

### Q3：系统表如何排除？

```java
@Override
public boolean ignoreTable(String tableName) {
    // 忽略系统表
    return "sys_config".equals(tableName)
        || "sys_user".equals(tableName);
}
```

### Q4：多租户和分页一起使用有问题吗？

确保插件顺序正确：

```java
interceptor.addInnerInterceptor(new TenantLineInnerInterceptor(...));  // 先
interceptor.addInnerInterceptor(new PaginationInnerInterceptor());     // 后
```

## 注意事项

### 索引问题

```sql
-- 必须为 tenant_id 添加索引
CREATE INDEX idx_tenant_id ON user(tenant_id);
```

### 唯一索引问题

如果表有唯一索引，需要包含 tenant_id：

```sql
-- 错误：跨租户的唯一性无法保证
CREATE UNIQUE INDEX uk_name ON user(name);

-- 正确：包含 tenant_id
CREATE UNIQUE INDEX uk_name_tenant ON user(name, tenant_id);
```

### 性能考虑

```sql
-- 好的查询（tenant_id 在前面）
WHERE tenant_id = 1 AND id = 1

-- 坏的查询（tenant_id 不在前面）
WHERE id = 1 AND tenant_id = 1
```

确保 SQL 中 tenant_id 条件在最前面，利用索引。

## 总结

### 配置步骤

1. **实现 TenantLineHandler**
2. **配置 TenantLineInnerInterceptor 插件**
3. **在请求入口设置租户 ID**

### 自动处理

| SQL 类型 | 处理方式 |
|---------|---------|
| SELECT | 添加 `WHERE tenant_id = ?` |
| INSERT | 添加 `tenant_id` 列和值 |
| UPDATE | 添加 `WHERE tenant_id = ?` |
| DELETE | 添加 `WHERE tenant_id = ?` |

### 忽略策略

| 方式 | 说明 |
|------|------|
| `ignoreTable()` | 全局忽略某些表 |
| `ignoreSql()` | 忽略某些 SQL 类型 |
| `@InterceptorIgnore` | 方法级别忽略 |

多租户插件是实现 SaaS 系统的关键技术，合理使用可以大大简化多租户数据隔离的实现。
