# MyBatis Plus 分页：PaginationInnerInterceptor

<Badge text="MyBatis Plus" type="tip" /> <Badge text="核心功能" type="info" />

为什么你用的 MyBatis Plus 分页一直不对？

很多人在使用 MyBatis Plus 分页时，发现分页没有生效——查询返回了全部数据，或者分页参数被忽略了。问题的根源往往是：**没有正确配置分页插件**。

这篇文章，让你彻底搞懂 MyBatis Plus 分页的实现原理和使用方法。

## 分页插件配置

### 第一步：添加依赖

```xml
<dependencies>
    <dependency>
        <groupId>com.baomidou</groupId>
        <artifactId>mybatis-plus-boot-starter</artifactId>
        <version>3.5.5</version>
    </dependency>
</dependencies>
```

### 第二步：配置分页插件

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        // 添加分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor());

        return interceptor;
    }
}
```

### 第三步：使用分页

```java
// 创建分页对象
Page<User> page = new Page<>(1, 10);  // 第 1 页，每页 10 条

// 执行分页查询
Page<User> result = userMapper.selectPage(page, null);

// 获取结果
List<User> users = result.getRecords();  // 当前页数据
long total = result.getTotal();          // 总记录数
long pages = result.getPages();          // 总页数
long current = result.getCurrent();       // 当前页码
long size = result.getSize();            // 每页条数
```

## Page 对象详解

### 创建方式

```java
// 方式一：构造方法
Page<User> page = new Page<>(1, 10);      // 第 1 页，每页 10 条
Page<User> page = new Page<>(1, 10, 100); // 第 1 页，每页 10 条，总数 100（不查数据库）

// 方式二：静态方法
Page<User> page = Page.of(1, 10);

// 方式三：只查数据，不查总数
Page<User> page = new Page<>(1, 10, false);  // optimizeCountQuery = false
```

### 分页参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `current` | 当前页码 | 1 |
| `size` | 每页条数 | 10 |
| `total` | 总记录数 | 0（自动查询） |
| `records` | 当前页数据 | - |
| `pages` | 总页数 | - |

### 分页属性

```java
Page<User> page = new Page<>(1, 10);

// 设置排序
page.addOrder(OrderItem.asc("name"));
page.addOrder(OrderItem.desc("create_time"));

// 设置搜索条件
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.like(User::getName, "张");

userMapper.selectPage(page, wrapper);
```

## 分页 SQL 原理

### 拦截机制

MyBatis Plus 通过 `PaginationInnerInterceptor` 拦截 SQL，自动在查询前后添加 LIMIT 和 COUNT。

```java
// 原始 SQL
SELECT * FROM user WHERE status = 1

// MyBatis Plus 转换后
-- 总记录数查询
SELECT COUNT(*) FROM user WHERE status = 1

-- 分页数据查询
SELECT * FROM user WHERE status = 1 LIMIT 0, 10
```

### 自动优化

当只查询数据不查询总数时：

```java
Page<User> page = new Page<>(1, 10, false);  // optimizeCountQuery = false
userMapper.selectPage(page, null);
// 不执行 COUNT 查询，直接返回 total = 0
```

## 分页与排序

### 基本用法

```java
Page<User> page = new Page<>(1, 10);
page.addOrder(OrderItem.asc("name"));
page.addOrder(OrderItem.desc("create_time"));

LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(User::getStatus, 1);

userMapper.selectPage(page, wrapper);
```

### 排序字段映射

```java
// 使用 Lambda 排序
page.addOrder(OrderItem.asc(User::getName));
page.addOrder(OrderItem.desc(User::getCreateTime));
```

### Wrapper 中的排序

```java
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(User::getStatus, 1)
       .orderByAsc(User::getName)
       .orderByDesc(User::getCreateTime);

userMapper.selectPage(page, wrapper);
```

## 分页参数不传递的问题

### 常见错误

```java
// 错误：Page 参数放错位置
Page<User> page = new Page<>(1, 10);
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(User::getStatus, 1);

// 错误：没有传入 wrapper
userMapper.selectPage(page, null);

// 正确：传入 wrapper
userMapper.selectPage(page, wrapper);
```

### 不传分页参数

```java
// 不传分页参数，默认查第一页，每页 10 条
Page<User> page = new Page<>();
```

### 全局默认配置

```yaml
mybatis-plus:
  global-config:
    db-config:
      # 默认每页条数
      page-size: 20
```

## 多表联查分页

### 自定义 SQL

```java
// Mapper 接口
Page<UserVO> selectUserDeptPage(Page<UserVO> page,
                                @Param("status") Integer status);
```

```xml
<select id="selectUserDeptPage" resultType="UserVO">
    SELECT u.id, u.name, d.name AS dept_name
    FROM user u
    LEFT JOIN department d ON u.dept_id = d.id
    WHERE u.status = #{status}
    ORDER BY u.create_time DESC
</select>
```

### COUNT 查询优化

多表联查时，COUNT 查询可能很慢。可以通过 `page.setSearchCount(false)` 禁用自动 COUNT：

```java
Page<UserVO> page = new Page<>(1, 10);
page.setSearchCount(false);  // 不查总数
```

## 分页插件高级配置

### 设置最大单页限制

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        PaginationInnerInterceptor paginationInterceptor = new PaginationInnerInterceptor();
        paginationInterceptor.setMaxLimit(100L);  // 最大单页限制
        paginationInterceptor.setOverflow(false);  // 超出时返回第一页

        interceptor.addInnerInterceptor(paginationInterceptor);

        return interceptor;
    }
}
```

| 配置项 | 说明 |
|--------|------|
| `setMaxLimit` | 最大单页限制（防止查询过多数据） |
| `setOverflow` | 页码溢出时是否回到第一页 |
| `setDbType` | 指定数据库类型 |
| `set dialect` | 指定分页方言 |

### 溢出处理

```java
// 请求第 10000 页，但只有 100 页
PaginationInnerInterceptor paginationInterceptor = new PaginationInnerInterceptor();
paginationInterceptor.setOverflow(true);  // 返回第一页数据
```

## 分页与性能

### 分页性能问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 深分页慢 | OFFSET 大时，数据库需要扫描大量数据 | 使用游标分页 |
| COUNT 慢 | 大表 COUNT 耗时 | 使用缓存、优化索引 |
| 数据量大 | 每页数据太多 | 限制最大单页条数 |

### 游标分页（Keyset Pagination）

当 OFFSET 很大时（如第 10000 页），MySQL 需要扫描前 10000 条数据：

```sql
-- 深分页慢的原因
SELECT * FROM user ORDER BY id LIMIT 100000, 10;
-- 需要扫描 100010 条数据
```

**游标分页**：使用上一页最后一条数据的 ID 作为起点：

```sql
-- 游标分页：基于 ID 的分页
SELECT * FROM user WHERE id > #{lastId} ORDER BY id LIMIT 10;
-- 只扫描 10 条数据
```

### BaseMapper 分页方法

```java
// 普通分页
Page<User> page = new Page<>(current, size);
userMapper.selectPage(page, wrapper);

// 只查记录数
Long count = userMapper.selectCount(wrapper);

// 只查分页数据
Page<User> page = new Page<>(current, size, false);  // 不查总数
```

## 多租户场景下的分页

### 配置多租户插件

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        // 先添加多租户插件
        interceptor.addInnerInterceptor(new TenantLineInnerInterceptor(new TenantLineHandlerImpl()));

        // 再添加分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor());

        return interceptor;
    }
}
```

**注意**：插件顺序很重要！多租户插件必须在分页插件之前添加。

## 常见问题

### Q1：分页不生效？

1. 检查是否配置了分页插件
2. 检查插件是否被 Spring 扫描到
3. 检查 Mapper 方法的参数顺序

### Q2：总记录数不对？

1. 检查是否有逻辑删除字段
2. 检查是否有多租户隔离
3. 检查 Wrapper 条件是否正确

### Q3：深分页很慢？

1. 使用游标分页代替 OFFSET 分页
2. 优化数据库索引
3. 限制最大单页条数

## 总结

### 分页使用步骤

1. **配置分页插件**：`PaginationInnerInterceptor`
2. **创建 Page 对象**：`new Page<>(current, size)`
3. **执行查询**：`mapper.selectPage(page, wrapper)`
4. **获取结果**：`page.getRecords()`、`page.getTotal()`

### 分页原理

```
用户请求
    ↓
MyBatis Plus 拦截 SQL
    ↓
判断是否分页查询
    ↓
添加 COUNT 查询
    ↓
添加 LIMIT 语句
    ↓
执行查询并返回
```

### 性能优化建议

1. 避免深分页（使用游标分页）
2. 合理设置每页条数（不要太大）
3. 使用 `optimizeCountQuery` 优化 COUNT 查询
4. 为分页字段添加索引

掌握 MyBatis Plus 分页，你就能轻松实现高效的分页查询功能。
