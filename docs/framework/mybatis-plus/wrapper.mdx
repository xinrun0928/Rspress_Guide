# MyBatis Plus Wrapper：链式调用的艺术

<Badge text="MyBatis Plus" type="tip" /> <Badge text="条件构造" type="info" />

QueryWrapper 和 LambdaQueryWrapper 是 MyBatis Plus 的核心功能之一，它们提供了强大的条件构造能力，让你可以用链式调用的方式构建复杂的查询条件。

## Wrapper 家族

MyBatis Plus 提供了多种 Wrapper：

| Wrapper | 说明 |
|---------|------|
| `QueryWrapper` | 普通 Wrapper，条件用字符串字段名 |
| `LambdaQueryWrapper` | Lambda Wrapper，条件用 Lambda 表达式 |
| `UpdateWrapper` | 更新 Wrapper，用于 UPDATE 操作 |
| `LambdaUpdateWrapper` | Lambda 更新 Wrapper |
| `LambdaQueryWrapper` | 查询 Wrapper |

## QueryWrapper 基础用法

### 创建方式

```java
// 方式一：new 方式
QueryWrapper<User> wrapper = new QueryWrapper<>();

// 方式二：lambda 方式
QueryWrapper<User> wrapper = new QueryWrapper<User>()
    .lambda()
    .eq(User::getName, "张三");

// 方式三：通过实体对象
QueryWrapper<User> wrapper = new QueryWrapper<>(user);
```

### eq：等于查询

```java
QueryWrapper<User> wrapper = new QueryWrapper<>();
wrapper.eq("name", "张三").eq("age", 20);
userMapper.selectList(wrapper);

// 生成的 SQL：SELECT * FROM user WHERE name = '张三' AND age = 20
```

### ne：不等于查询

```java
wrapper.ne("name", "张三");
```

### gt / ge / lt / le：比较查询

```java
wrapper.gt("age", 18);   // age > 18
wrapper.ge("age", 18);  // age >= 18
wrapper.lt("age", 60);  // age < 60
wrapper.le("age", 60);  // age <= 60
```

### between / notBetween：范围查询

```java
wrapper.between("age", 18, 30);
// age BETWEEN 18 AND 30

wrapper.notBetween("age", 18, 30);
// age NOT BETWEEN 18 AND 30
```

### like / notLike：模糊查询

```java
wrapper.like("name", "张");
// name LIKE '%张%'

wrapper.likeLeft("name", "三");
// name LIKE '%三'

wrapper.likeRight("name", "张");
// name LIKE '张%'

wrapper.notLike("name", "张");
// name NOT LIKE '%张%'
```

### isNull / isNotNull：NULL 查询

```java
wrapper.isNull("email");
// email IS NULL

wrapper.isNotNull("email");
// email IS NOT NULL
```

### in / notIn：IN 查询

```java
wrapper.in("id", 1, 2, 3);
// id IN (1, 2, 3)

wrapper.in("id", Arrays.asList(1, 2, 3));

wrapper.notIn("id", 1, 2, 3);
// id NOT IN (1, 2, 3)
```

### or / and：逻辑运算

```java
wrapper.eq("name", "张三").or().eq("name", "李四");
// name = '张三' OR name = '李四'

wrapper.and(w -> w.eq("age", 18).or().eq("age", 20));
// (age = 18 OR age = 20)
```

### nested：嵌套

```java
wrapper.nested(w -> w.eq("name", "张三").or().eq("name", "李四"));
// (name = '张三' OR name = '李四')
```

### 排序

```java
wrapper.orderByAsc("age", "name");
// ORDER BY age ASC, name ASC

wrapper.orderByDesc("age", "name");
// ORDER BY age DESC, name DESC

wrapper.orderBy(true, true, "age");
// ORDER BY age ASC

wrapper.orderBy(true, false, "age");
// ORDER BY age DESC
```

### 分组和 Having

```java
wrapper.select("name, SUM(age) as total_age")
      .groupBy("name")
      .having("SUM(age) > 100");
// SELECT name, SUM(age) as total_age FROM user GROUP BY name HAVING SUM(age) > 100
```

## LambdaQueryWrapper：类型安全的 Wrapper

### 为什么需要 Lambda？

`QueryWrapper` 的问题是：**字段名是字符串，容易写错**。

```java
// QueryWrapper：字段名可能写错
wrapper.eq("user_name", "张三");  // 运行时才发现错误

// LambdaQueryWrapper：编译时就报错
wrapper.eq(User::getName, "张三");  // 编译时检查
```

### 基本用法

```java
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();

wrapper.eq(User::getName, "张三")
      .ge(User::getAge, 18)
      .like(User::getEmail, "@example.com")
      .orderByDesc(User::getCreateTime);
```

### 等值查询

```java
// eq：等于
wrapper.eq(User::getName, "张三");

// ne：不等于
wrapper.ne(User::getName, "张三");
```

### 范围查询

```java
// gt：大于
wrapper.gt(User::getAge, 18);

// ge：大于等于
wrapper.ge(User::getAge, 18);

// lt：小于
wrapper.lt(User::getAge, 60);

// le：小于等于
wrapper.le(User::getAge, 60);

// between：范围
wrapper.between(User::getAge, 18, 30);

// notBetween：不在范围内
wrapper.notBetween(User::getAge, 18, 30);
```

### 模糊查询

```java
// like：两边模糊
wrapper.like(User::getName, "张");
// name LIKE '%张%'

// likeLeft：左边模糊
wrapper.likeLeft(User::getName, "三");
// name LIKE '%三'

// likeRight：右边模糊
wrapper.likeRight(User::getName, "张");
// name LIKE '张%'
```

### IN 查询

```java
wrapper.in(User::getId, 1, 2, 3);
// id IN (1, 2, 3)

wrapper.inSql(User::getId, "SELECT id FROM user WHERE age > 18");
// id IN (SELECT id FROM user WHERE age > 18)
```

### 聚合函数

```java
wrapper.select(User::getName, "SUM(age) as total_age")
      .groupBy(User::getName)
      .having("SUM(age) > 100");
```

## 常用查询示例

### 示例一：多条件查询

```java
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper
    .eq(StringUtils.hasText(name), User::getName, name)  // 条件成立时才添加
    .eq(age != null, User::getAge, age)
    .like(StringUtils.hasText(email), User::getEmail, email)
    .orderByDesc(User::getCreateTime);

List<User> users = userMapper.selectList(wrapper);
```

**关键技巧**：使用条件参数，只有条件成立时才添加 WHERE 子句。

### 示例二：OR 查询

```java
wrapper.and(w -> w
    .eq(User::getName, "张三")
    .or()
    .eq(User::getName, "李四")
);
// (name = '张三' OR name = '李四')
```

### 示例三：嵌套查询

```java
wrapper.nested(w -> w
    .eq(User::getAge, 18)
    .or()
    .eq(User::getAge, 20)
);
// (age = 18 OR age = 20)
```

### 示例四：子查询

```java
wrapper.inSql(User::getDeptId, "SELECT id FROM dept WHERE name = '研发部'");
// dept_id IN (SELECT id FROM dept WHERE name = '研发部')
```

### 示例五：指定查询字段

```java
wrapper.select(User::getId, User::getName, User::getEmail);
// SELECT id, name, email FROM user
```

### 示例六：排除某些字段

```java
wrapper.select(User.class, User::getId, User::getName);
// 排除 id 和 name 字段
```

## UpdateWrapper：更新条件构造

### 基本用法

```java
UpdateWrapper<User> wrapper = new UpdateWrapper<>();
wrapper.eq("name", "张三").set("age", 25);
userMapper.update(null, wrapper);

// UPDATE user SET age = 25 WHERE name = '张三'
```

### Lambda 更新

```java
LambdaUpdateWrapper<User> wrapper = new LambdaUpdateWrapper<>();
wrapper.eq(User::getName, "张三")
       .set(User::getAge, 25)
       .set(User::getEmail, "zhangsan@example.com");
userMapper.update(null, wrapper);
```

### set 和 setSql

```java
// set：设置字段值
wrapper.set("name", "王五");

// setSql：设置 SQL 片段
wrapper.setSql("age = age + 1");
// UPDATE user SET age = age + 1 WHERE ...
```

### LambdaUpdateWrapper 更新 null 值

```java
User user = new User();
user.setId(1L);
user.setName("张三");
user.setEmail(null);  // 想把这个字段设置为 NULL

LambdaUpdateWrapper<User> wrapper = new LambdaUpdateWrapper<>();
wrapper.eq(User::getId, 1L);

// 只更新 name，不更新 email
wrapper.set(User::getName, user.getName());
wrapper.set(user.getEmail() != null, User::getEmail, user.getEmail());

// 或者使用 setSql
wrapper.setSql("email = NULL");
```

## Wrapper 条件链式组合

### 链式调用规则

```java
// 多个条件默认用 AND 连接
wrapper.eq(User::getName, "张三")
      .eq(User::getAge, 20);
// WHERE name = '张三' AND age = 20

// 使用 or() 切换为 OR
wrapper.eq(User::getName, "张三")
      .or()
      .eq(User::getName, "李四");
// WHERE name = '张三' OR name = '李四'
```

### 条件分支

```java
// 方式一：if 条件
if (age != null) {
    wrapper.eq(User::getAge, age);
}

// 方式二：链式条件
wrapper.eq(age != null, User::getAge, age);
// 第一个参数为 true 时才添加条件
```

### 常用链式方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `eq(condition, column, value)` | 条件成立时添加等于 | `eq(age != null, User::getAge, age)` |
| `ge(condition, column, value)` | 条件成立时添加大于等于 | - |
| `gt(condition, column, value)` | 条件成立时添加大于 | - |
| `le(condition, column, value)` | 条件成立时添加小于等于 | - |
| `lt(condition, column, value)` | 条件成立时添加小于 | - |
| `ne(condition, column, value)` | 条件成立时添加不等于 | - |

## 完整示例

### 查询构建器

```java
public class UserQuery {
    private String name;
    private Integer minAge;
    private Integer maxAge;
    private String email;
    private List<Long> deptIds;
    private String sortField;
    private Boolean asc;
}

public List<User> search(UserQuery query) {
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();

    // 姓名模糊查询
    wrapper.like(StringUtils.hasText(query.getName()), User::getName, query.getName());

    // 年龄范围
    wrapper.ge(query.getMinAge() != null, User::getAge, query.getMinAge());
    wrapper.le(query.getMaxAge() != null, User::getAge, query.getMaxAge());

    // 邮箱精确查询
    wrapper.eq(StringUtils.hasText(query.getEmail()), User::getEmail, query.getEmail());

    // 部门 IN 查询
    if (query.getDeptIds() != null && !query.getDeptIds().isEmpty()) {
        wrapper.in(User::getDeptId, query.getDeptIds());
    }

    // 排序
    if ("name".equals(query.getSortField())) {
        wrapper.orderBy(true, query.getAsc(), User::getName);
    } else if ("age".equals(query.getSortField())) {
        wrapper.orderBy(true, query.getAsc(), User::getAge);
    } else if ("createTime".equals(query.getSortField())) {
        wrapper.orderBy(true, query.getAsc(), User::getCreateTime);
    } else {
        wrapper.orderByDesc(User::getCreateTime);
    }

    return userMapper.selectList(wrapper);
}
```

### 批量更新

```java
public int batchUpdateStatus(List<Long> ids, Integer status) {
    LambdaUpdateWrapper<User> wrapper = new LambdaUpdateWrapper<>();
    wrapper.in(User::getId, ids).set(User::getStatus, status);
    return userMapper.update(null, wrapper);
}
```

## 总结

### QueryWrapper vs LambdaQueryWrapper

| 特性 | QueryWrapper | LambdaQueryWrapper |
|------|--------------|-------------------|
| 字段名写法 | 字符串 | Lambda 表达式 |
| 类型安全 | 否 | 是 |
| IDE 自动补全 | 弱 | 强 |
| 重构安全 | 否 | 是（字段改名时编译报错） |

### 推荐使用

**推荐使用 `LambdaQueryWrapper`**，因为：
1. 编译时检查，减少运行时错误
2. IDE 自动补全，提高开发效率
3. 重构安全，字段改名时自动更新

### 条件链式技巧

1. 使用 `eq(condition, ...)` 实现条件分支
2. 使用 `and()` 和 `or()` 控制逻辑关系
3. 使用 `nested()` 实现条件嵌套
4. 使用 `inSql()` 实现子查询

掌握 Wrapper 的使用，你就能用最少的代码构建复杂的查询条件，大幅提升开发效率。
