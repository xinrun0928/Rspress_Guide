# MyBatis Plus SQL 分析：SqlExplainInterceptor

<Badge text="MyBatis Plus" type="tip" /> <Badge text="核心功能" type="info" />

防止误删全表数据，从 SQL 分析开始。

在生产环境中，最危险的操作不是查询慢，而是执行了一条没有 WHERE 条件的 UPDATE 或 DELETE——全表数据被清空。

MyBatis Plus 的 SQL 分析插件，可以帮你拦截这些危险操作。

## SqlExplainInterceptor

### 作用

`SqlExplainInterceptor` 会分析执行前的 SQL：
- 检查是否有潜在的全表更新/删除风险
- 如果有，使用 `EXPLAIN` 分析执行计划
- 如果是全表操作，抛出异常阻止执行

### 配置

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        // SQL 分析插件
        interceptor.addInnerInterceptor(new SqlExplainInterceptor());

        return interceptor;
    }
}
```

### 拦截场景

| SQL 类型 | 拦截条件 | 示例 |
|---------|---------|------|
| UPDATE | 没有 WHERE 条件 | `UPDATE user SET status = 1` |
| DELETE | 没有 WHERE 条件 | `DELETE FROM user` |
| UPDATE | WHERE 条件永真 | `UPDATE user SET status = 1 WHERE 1 = 1` |
| DELETE | WHERE 条件永真 | `DELETE FROM user WHERE 1 = 1` |

## 性能分析插件

### PerformanceInterceptor

除了安全分析，MyBatis Plus 还提供了性能分析插件：

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public PerformanceInterceptor performanceInterceptor() {
        PerformanceInterceptor performanceInterceptor = new PerformanceInterceptor();

        // SQL 执行超过 1 秒则记录日志
        performanceInterceptor.setMaxTime(1000L);

        // 开启 SQL 日志（格式化）
        performanceInterceptor.setFormat(true);

        return performanceInterceptor;
    }
}
```

### 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `maxTime` | SQL 最大执行时间（毫秒） | 0（不限制） |
| `format` | 是否格式化 SQL | false |
| `logRuntime` | 是否写入日志 | true |

### 日志输出

```log
 Time：25 ms - ID：com.example.mapper.UserMapper.selectById
 SQL：SELECT id, name, email FROM user WHERE id = 1
```

## SQL 执行效率分析

### 慢 SQL 的危害

| 危害 | 说明 |
|------|------|
| 数据库连接占用 | 长时间占用连接，影响其他请求 |
| CPU 消耗 | 数据库执行慢 SQL，消耗 CPU |
| 锁等待 | 长时间持有锁，导致其他事务等待 |
| 应用响应慢 | 用户等待时间长 |

### 性能优化建议

1. **添加索引**：分析 EXPLAIN 结果，确保走了索引
2. **优化 SQL**：避免 SELECT *，减少返回数据量
3. **分批处理**：大数据量操作分批执行
4. **异步处理**：非关键操作异步执行

## SQL 注入攻击防护

### 注入场景

```java
// 危险：通过参数拼接 SQL
@Select("SELECT * FROM user WHERE name = '${name}'")
List<User> search(String name);

// 如果 name = "' OR '1'='1"，会查出所有数据
```

### 防护措施

1. **使用参数绑定**：永远不要拼接 SQL
2. **参数校验**：校验输入参数格式
3. **SQL 日志**：记录所有执行的 SQL
4. **最小权限**：数据库用户只授予必要权限

### MyBatis Plus 的防护

MyBatis Plus 默认使用 `#{}` 参数绑定，自动防护 SQL 注入：

```java
// 安全：使用 #{}
userMapper.selectList(
    new LambdaQueryWrapper<User>()
        .like(User::getName, keyword)  // 自动转义
);

// 危险：使用 ${}（需要特殊场景才使用）
@Select("SELECT * FROM ${tableName}")
List<User> selectByTable(@Param("tableName") String tableName);
```

## 全表操作拦截

### 自定义拦截器

```java
@Component
@InterceptorIgnore(tenantLine = "true")
public class SqlBlockerInterceptor implements InnerInterceptor {

    @Override
    public void beforeQuery(Executor executor, MappedStatement ms, Object parameter,
                           RowBounds rowBounds, ResultHandler resultHandler,
                           BoundSql boundSql) {
        String sql = boundSql.getSql().toUpperCase();

        // 拦截全表查询
        if (sql.contains("SELECT *") && !sql.contains("WHERE")) {
            throw new SqlBlockException("禁止全表查询，请添加 WHERE 条件");
        }
    }
}
```

### 白名单机制

```java
@Component
public class SqlWhiteListInterceptor implements InnerInterceptor {

    private static final Set<String> WHITE_LIST = Set.of(
        "truncate_table",
        "sync_data"
    );

    @Override
    public void beforeUpdate(Executor executor, MappedStatement ms, Object parameter,
                            BoundSql boundSql) {
        String sqlId = ms.getId();

        // 检查是否在白名单中
        if (!WHITE_LIST.contains(sqlId)) {
            String sql = boundSql.getSql();

            // 检查是否有 WHERE 条件
            if (!sql.contains("WHERE") && !sql.contains("where")) {
                throw new SqlBlockException("禁止没有 WHERE 条件的更新操作");
            }
        }
    }
}
```

## 日志与监控

### 启用 SQL 日志

```yaml
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl  # 控制台输出
    # 或
    log-impl: org.apache.ibatis.logging.slf4j.Slf4jImpl     # SLF4J
```

### 自定义日志

```java
@Component
public class SqlLogInterceptor implements Interceptor {

    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        long start = System.currentTimeMillis();

        // 执行 SQL
        Object result = invocation.proceed();

        long end = System.currentTimeMillis();

        // 获取 SQL 信息
        Statement statement = (Statement) invocation.getArgs()[0];
        String sql = statement.toString();

        // 记录日志
        log.info("SQL: {} | 耗时: {}ms", sql, end - start);

        return result;
    }
}
```

## 常见问题

### Q1：SqlExplainInterceptor 会影响性能吗？

会，因为每次执行 UPDATE/DELETE 前都会调用 `EXPLAIN`。但在生产环境中，这是值得的——可以防止灾难性的全表操作。

### Q2：如何绕过 SqlExplainInterceptor？

```java
// 方式一：使用原生 JDBC（不推荐）
// 方式二：关闭插件（不推荐）
// 方式三：使用白名单（推荐）
```

### Q3：性能插件打印的 SQL 没有参数值？

需要开启日志中的参数打印：

```yaml
logging:
  level:
    com.example.mapper: DEBUG  # 需要 DEBUG 级别
```

## 总结

### SQL 分析插件

| 插件 | 作用 | 建议 |
|------|------|------|
| `SqlExplainInterceptor` | 防止全表更新/删除 | 生产环境开启 |
| `PerformanceInterceptor` | 分析 SQL 执行时间 | 开发环境开启 |
| `SqlLogInterceptor` | 记录所有 SQL | 按需开启 |

### 安全建议

1. **生产环境开启 SqlExplainInterceptor**
2. **使用参数绑定，禁止字符串拼接**
3. **添加数据库操作审计日志**
4. **限制数据库用户权限**
5. **定期审查慢 SQL**

### 配置示例

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        // SQL 分析插件（生产环境开启）
        interceptor.addInnerInterceptor(new SqlExplainInterceptor());

        // 分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor());

        return interceptor;
    }

    @Bean
    public PerformanceInterceptor performanceInterceptor() {
        PerformanceInterceptor performanceInterceptor = new PerformanceInterceptor();
        performanceInterceptor.setMaxTime(1000L);
        performanceInterceptor.setFormat(true);
        return performanceInterceptor;
    }
}
```

SQL 安全是生产环境最重要的事情之一。合理使用 MyBatis Plus 提供的分析插件，可以大大降低误操作的风险。
