# JDBC 执行流程：手写一遍，你就懂了

<Badge text="ORM" type="tip" /> <Badge text="JDBC" type="info" />

ORM 框架封装的到底是什么？JDBC 手写一遍就清楚了。

虽然现在很少有人直接使用 JDBC，但理解 JDBC 的执行流程，是理解 ORM 框架的基础。这一讲，我们手写一个完整的 JDBC 工具类，彻底搞懂数据库访问的本质。

## JDBC 七步走

JDBC 操作数据库的标准流程只有七步：

```
1. 加载驱动
     ↓
2. 获取连接
     ↓
3. 创建语句
     ↓
4. 执行 SQL
     ↓
5. 处理结果
     ↓
6. 关闭资源
     ↓
7. 异常处理
```

## 第一步：加载驱动

### 注册驱动

```java
// 方式一：Class.forName（传统方式）
Class.forName("com.mysql.cj.jdbc.Driver");

// 方式二：自动加载（JDBC 4.0+，MySQL 8.0+）
// 只需要在 classpath 中添加驱动 jar 包，会自动加载
```

### 驱动原理

```java
// DriverManager 会自动调用驱动类的静态代码块
public class Driver implements java.sql.Driver {

    // 静态代码块：自动注册到 DriverManager
    static {
        DriverManager.registerDriver(new Driver());
    }
}
```

## 第二步：获取连接

### Connection

```java
// 数据库连接配置
String url = "jdbc:mysql://localhost:3306/mybatis_plus?useUnicode=true&characterEncoding=utf-8";
String username = "root";
String password = "password";

// 获取连接
Connection connection = DriverManager.getConnection(url, username, password);

// 带参数的连接
Properties info = new Properties();
info.setProperty("user", "root");
info.setProperty("password", "password");
info.setProperty("useSSL", "false");
Connection connection = DriverManager.getConnection(url, info);
```

### URL 格式

```
jdbc:mysql://主机名:端口/数据库名?参数1=值1&参数2=值2
```

常用参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| `useUnicode` | 使用 Unicode | `true` |
| `characterEncoding` | 字符编码 | `utf-8` |
| `useSSL` | 是否使用 SSL | `false` |
| `serverTimezone` | 时区 | `Asia/Shanghai` |
| `allowPublicKeyRetrieval` | 允许公钥检索 | `true` |

## 第三步：创建语句

### Statement

```java
// 普通 SQL 语句
Statement statement = connection.createStatement();

// 预编译 SQL（推荐，防止 SQL 注入）
PreparedStatement preparedStatement = connection.prepareStatement(sql);

// 存储过程
CallableStatement callableStatement = connection.prepareCall(sql);
```

### Statement vs PreparedStatement

| 特性 | Statement | PreparedStatement |
|------|-----------|------------------|
| SQL 拼接 | 字符串拼接 | 参数绑定 |
| SQL 注入 | 不安全 | 安全 |
| 性能 | 每次编译 | 预编译，可复用 |
| 参数设置 | 无 | 支持 |

## 第四步：执行 SQL

### 查询

```java
String sql = "SELECT * FROM user WHERE id = ?";
PreparedStatement ps = connection.prepareStatement(sql);
ps.setLong(1, 1L);

ResultSet resultSet = ps.executeQuery();  // 查询用 executeQuery
```

### 增删改

```java
String sql = "INSERT INTO user (name, email) VALUES (?, ?)";
PreparedStatement ps = connection.prepareStatement(sql);
ps.setString(1, "张三");
ps.setString(2, "zhangsan@example.com");

int rows = ps.executeUpdate();  // 增删改用 executeUpdate
```

## 第五步：处理结果

### ResultSet

```java
ResultSet resultSet = ps.executeQuery();

while (resultSet.next()) {
    // 获取数据（方式一：按列名）
    Long id = resultSet.getLong("id");
    String name = resultSet.getString("name");
    String email = resultSet.getString("email");

    // 获取数据（方式二：按列索引，从 1 开始）
    Long id = resultSet.getLong(1);
    String name = resultSet.getString(2);
    String email = resultSet.getString(3);

    // 创建对象
    User user = new User(id, name, email);
}
```

### ResultSet 类型

| 类型 | 说明 |
|------|------|
| `TYPE_FORWARD_ONLY` | 只向前移动（默认） |
| `TYPE_SCROLL_INSENSITIVE` | 可滚动，不感知变化 |
| `TYPE_SCROLL_SENSITIVE` | 可滚动，感知变化 |

### 并发类型

| 类型 | 说明 |
|------|------|
| `CONCUR_READ_ONLY` | 只读（默认） |
| `CONCUR_UPDATABLE` | 可更新 |

## 第六步：关闭资源

### 资源泄漏问题

如果忘记关闭资源，会导致连接泄漏：

```java
// 错误：没有关闭资源
Connection conn = DriverManager.getConnection(url, username, password);
// 使用连接
// 忘记关闭...
```

### 正确关闭

```java
Connection connection = null;
PreparedStatement ps = null;
ResultSet rs = null;

try {
    connection = DriverManager.getConnection(url, username, password);
    ps = connection.prepareStatement(sql);
    rs = ps.executeQuery();

    while (rs.next()) {
        // 处理结果
    }
} catch (SQLException e) {
    e.printStackTrace();
} finally {
    // 关闭资源（倒序关闭）
    if (rs != null) {
        try {
            rs.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
    if (ps != null) {
        try {
            ps.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
    if (connection != null) {
        try {
            connection.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
```

### try-with-resources（推荐）

```java
// JDK 7+ 自动关闭资源
try (Connection connection = DriverManager.getConnection(url, username, password);
     PreparedStatement ps = connection.prepareStatement(sql);
     ResultSet rs = ps.executeQuery()) {

    while (rs.next()) {
        // 处理结果
    }
} catch (SQLException e) {
    e.printStackTrace();
}
```

## 第七步：异常处理

### SQLException

```java
try {
    // 数据库操作
} catch (SQLException e) {
    // 获取错误码
    int errorCode = e.getErrorCode();

    // 获取 SQL 状态
    String sqlState = e.getSQLState();

    // 获取错误信息
    String message = e.getMessage();

    // 获取根本原因
    Throwable cause = e.getCause();
}
```

### 常见错误码

| 错误码 | 说明 |
|--------|------|
| 1045 | 用户名或密码错误 |
| 1064 | SQL 语法错误 |
| 1146 | 表不存在 |
| 2003 | 无法连接到数据库 |

## 完整示例

### JDBC 工具类

```java
public class JdbcTemplate {

    private static final String DRIVER = "com.mysql.cj.jdbc.Driver";

    static {
        try {
            Class.forName(DRIVER);
        } catch (ClassNotFoundException e) {
            throw new RuntimeException("驱动加载失败", e);
        }
    }

    /**
     * 查询单条记录
     */
    public static <T> T queryOne(String sql, RowMapper<T> mapper, Object... params) {
        List<T> list = queryList(sql, mapper, params);
        return list.isEmpty() ? null : list.get(0);
    }

    /**
     * 查询多条记录
     */
    public static <T> List<T> queryList(String sql, RowMapper<T> mapper, Object... params) {
        List<T> result = new ArrayList<>();

        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(sql)) {

            // 设置参数
            setParameters(ps, params);

            // 执行查询
            try (ResultSet rs = ps.executeQuery()) {
                int index = 0;
                while (rs.next()) {
                    result.add(mapper.mapRow(rs, index++));
                }
            }
        } catch (SQLException e) {
            throw new RuntimeException("查询失败", e);
        }

        return result;
    }

    /**
     * 执行更新
     */
    public static int update(String sql, Object... params) {
        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(sql)) {

            // 设置参数
            setParameters(ps, params);

            // 执行更新
            return ps.executeUpdate();
        } catch (SQLException e) {
            throw new RuntimeException("更新失败", e);
        }
    }

    private static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(
            "jdbc:mysql://localhost:3306/mybatis_plus?useSSL=false",
            "root",
            "password"
        );
    }

    private static void setParameters(PreparedStatement ps, Object... params) throws SQLException {
        for (int i = 0; i < params.length; i++) {
            ps.setObject(i + 1, params[i]);
        }
    }
}

/**
 * 行映射器
 */
@FunctionalInterface
public interface RowMapper<T> {
    T mapRow(ResultSet rs, int rowNum) throws SQLException;
}
```

### 使用示例

```java
// 查询单条
String sql = "SELECT id, name, email FROM user WHERE id = ?";
User user = JdbcTemplate.queryOne(sql, (rs, rowNum) -> {
    User u = new User();
    u.setId(rs.getLong("id"));
    u.setName(rs.getString("name"));
    u.setEmail(rs.getString("email"));
    return u;
}, 1L);

// 查询列表
String sql = "SELECT id, name, email FROM user WHERE age > ?";
List<User> users = JdbcTemplate.queryList(sql, (rs, rowNum) -> {
    User u = new User();
    u.setId(rs.getLong("id"));
    u.setName(rs.getString("name"));
    u.setEmail(rs.getString("email"));
    return u;
}, 18);

// 更新
String sql = "UPDATE user SET name = ? WHERE id = ?";
int rows = JdbcTemplate.update(sql, "张三", 1L);
```

## SQL 执行效率分析

### 预编译的优势

```java
// 普通 SQL：每次都需要解析和编译
Statement stmt = connection.createStatement();
stmt.executeQuery("SELECT * FROM user WHERE id = 1");
stmt.executeQuery("SELECT * FROM user WHERE id = 2");
stmt.executeQuery("SELECT * FROM user WHERE id = 3");

// 预编译 SQL：编译一次，执行三次
PreparedStatement ps = connection.prepareStatement("SELECT * FROM user WHERE id = ?");
ps.setLong(1, 1);
ps.executeQuery();
ps.setLong(1, 2);
ps.executeQuery();
ps.setLong(1, 3);
ps.executeQuery();
```

### 批量操作

```java
// 非批量：执行 N 次
for (User user : users) {
    String sql = "INSERT INTO user (name, email) VALUES (?, ?)";
    PreparedStatement ps = connection.prepareStatement(sql);
    ps.setString(1, user.getName());
    ps.setString(2, user.getEmail());
    ps.executeUpdate();
}

// 批量：执行 1 次
String sql = "INSERT INTO user (name, email) VALUES (?, ?)";
PreparedStatement ps = connection.prepareStatement(sql);
for (User user : users) {
    ps.setString(1, user.getName());
    ps.setString(2, user.getEmail());
    ps.addBatch();  // 添加到批处理
}
ps.executeBatch();  // 执行批量
```

## 总结

### JDBC 七步流程

```
1. Class.forName()       → 加载驱动
2. getConnection()       → 获取连接
3. prepareStatement()    → 创建语句
4. executeQuery/Update() → 执行 SQL
5. ResultSet            → 处理结果
6. close()              → 关闭资源
7. try-catch            → 异常处理
```

### 核心要点

| 要点 | 说明 |
|------|------|
| 使用 PreparedStatement | 防止 SQL 注入 |
| 使用 try-with-resources | 自动关闭资源 |
| 异常处理 | 捕获 SQLException |
| 连接池 | 生产环境必须使用 |

### 进阶方向

1. **连接池**：Druid、HikariCP
2. **ORM 框架**：MyBatis、Hibernate
3. **事务管理**：ACID、隔离级别
4. **数据库连接池**：连接复用、性能优化

理解 JDBC 的执行流程，你就理解了所有 ORM 框架的底层原理。无论使用什么框架，底层都是 JDBC。
