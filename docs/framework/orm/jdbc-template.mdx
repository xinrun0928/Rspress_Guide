# Spring JDBC Template：比 JDBC 更好用，比 ORM 更轻量

在 Spring 生态中，JDBC Template 是一个常被忽视的利器。

它比原生 JDBC 好用，比 Hibernate/JPA 轻量。在某些场景下，JDBC Template 可能是最好的选择。

这篇文章，让你重新认识 Spring JDBC Template。

## JDBC Template 是什么？

### 定位

```
┌─────────────────────────────────────────────────────────────┐
│                       数据访问技术栈                          │
│                                                             │
│  ┌───────────────────┐                                       │
│  │      JPA/Hibernate│  功能强大，但重                       │
│  └─────────┬─────────┘                                       │
│            ↓                                                │
│  ┌───────────────────┐                                       │
│  │   MyBatis Plus   │  SQL 灵活，但需要 XML                  │
│  └─────────┬─────────┘                                       │
│            ↓                                                │
│  ┌───────────────────┐                                       │
│  │  Spring JDBC      │  轻量、灵活、Spring 生态               │
│  └─────────┬─────────┘                                       │
│            ↓                                                │
│  ┌───────────────────┐                                       │
│  │      JDBC         │  底层、繁琐                            │
│  └───────────────────┘                                       │
└─────────────────────────────────────────────────────────────┘
```

### 核心功能

```java
// Spring JDBC Template 封装了：
// 1. 资源管理（获取/关闭连接）
// 2. 异常处理（SQLException → DataAccessException）
// 3. 模板化操作（回调模式）
// 4. 参数绑定
// 5. 结果映射
```

## 基本使用

### 引入依赖

```xml
<dependencies>
    <!-- Spring Boot 自动引入 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-jdbc</artifactId>
    </dependencies>
</dependencies>
```

### 配置数据源

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mybatis_plus?useSSL=false
    username: root
    password: password
    driver-class-name: com.mysql.cj.jdbc.Driver
```

### 自动装配

Spring Boot 会自动创建 `JdbcTemplate`：

```java
@Service
public class UserService {

    @Autowired
    private JdbcTemplate jdbcTemplate;
}
```

## JdbcTemplate 方法

### queryForObject：查询单条

```java
// 查询单值
String name = jdbcTemplate.queryForObject(
    "SELECT name FROM user WHERE id = ?",
    String.class,
    1L
);

// 查询单条记录
User user = jdbcTemplate.queryForObject(
    "SELECT id, name, email FROM user WHERE id = ?",
    new Object[]{1L},
    (rs, rowNum) -> {
        User u = new User();
        u.setId(rs.getLong("id"));
        u.setName(rs.getString("name"));
        u.setEmail(rs.getString("email"));
        return u;
    }
);
```

### query：查询多条

```java
// 查询列表
List<User> users = jdbcTemplate.query(
    "SELECT id, name, email FROM user WHERE age > ?",
    new Object[]{18},
    (rs, rowNum) -> {
        User u = new User();
        u.setId(rs.getLong("id"));
        u.setName(rs.getString("name"));
        u.setEmail(rs.getString("email"));
        return u;
    }
);
```

### update：增删改

```java
// 插入
int rows = jdbcTemplate.update(
    "INSERT INTO user (name, email) VALUES (?, ?)",
    "张三", "zhangsan@example.com"
);

// 更新
int rows = jdbcTemplate.update(
    "UPDATE user SET name = ? WHERE id = ?",
    "李四", 1L
);

// 删除
int rows = jdbcTemplate.update(
    "DELETE FROM user WHERE id = ?",
    1L
);
```

### batchUpdate：批量操作

```java
String sql = "INSERT INTO user (name, email) VALUES (?, ?)";

List<Object[]> batchArgs = Arrays.asList(
    new Object[]{"张三", "zhangsan@example.com"},
    new Object[]{"李四", "lisi@example.com"},
    new Object[]{"王五", "wangwu@example.com"}
);

jdbcTemplate.batchUpdate(sql, batchArgs);
```

## RowMapper：结果映射

### 手写 RowMapper

```java
// 方式一：Lambda 表达式
User user = jdbcTemplate.queryForObject(
    "SELECT * FROM user WHERE id = ?",
    (rs, rowNum) -> {
        User u = new User();
        u.setId(rs.getLong("id"));
        u.setName(rs.getString("name"));
        u.setEmail(rs.getString("email"));
        return u;
    },
    1L
);
```

### BeanPropertyRowMapper

```java
// 方式二：使用 BeanPropertyRowMapper
// 自动映射字段（驼峰转换）
User user = jdbcTemplate.queryForObject(
    "SELECT id, name, email, create_time FROM user WHERE id = ?",
    new BeanPropertyRowMapper<>(User.class),
    1L
);

// 查询列表
List<User> users = jdbcTemplate.query(
    "SELECT id, name, email FROM user",
    new BeanPropertyRowMapper<>(User.class)
);
```

### BeanPropertyRowMapper 映射规则

| 数据库字段 | Java 属性 |
|-----------|-----------|
| `id` | `id` |
| `user_name` | `userName` |
| `create_time` | `createTime` |

**注意**：需要开启驼峰转换（Spring Boot 默认开启）。

## NamedParameterJdbcTemplate：命名参数

### 普通参数的问题

```java
// 普通参数：按位置
jdbcTemplate.update(
    "INSERT INTO user (name, email) VALUES (?, ?)",
    "张三", "zhangsan@example.com"
);

// 问题：如果 SQL 很长，很难对应参数位置
```

### 命名参数

```java
@Autowired
private NamedParameterJdbcTemplate namedJdbcTemplate;

// 使用命名参数
Map<String, Object> params = new HashMap<>();
params.put("name", "张三");
params.put("email", "zhangsan@example.com");

namedJdbcTemplate.update(
    "INSERT INTO user (name, email) VALUES (:name, :email)",
    params
);

// 查询
User user = namedJdbcTemplate.queryForObject(
    "SELECT * FROM user WHERE name = :name",
    params,
    new BeanPropertyRowMapper<>(User.class)
);
```

### SqlParameterSource

```java
// 方式一：MapSqlParameterSource
SqlParameterSource params = new MapSqlParameterSource()
    .addValue("name", "张三")
    .addValue("email", "zhangsan@example.com");

// 方式二：BeanPropertySqlParameterSource
SqlParameterSource params = new BeanPropertySqlParameterSource(user);
```

## @Repository 的 @Autowired 配合

### 正确写法

```java
@Repository
public class UserDao {

    @Autowired
    private JdbcTemplate jdbcTemplate;
}
```

### 为什么用 @Repository？

| 注解 | 适用场景 |
|------|---------|
| `@Repository` | DAO 层，数据库访问 |
| `@Mapper` | MyBatis Mapper 接口 |
| `@Component` | 通用组件 |

### JdbcTemplate 的事务支持

```java
@Transactional
public void updateUser(User user) {
    jdbcTemplate.update(
        "UPDATE user SET name = ? WHERE id = ?",
        user.getName(), user.getId()
    );
}
```

## JdbcTemplate vs JPA vs MyBatis

### 功能对比

| 特性 | JdbcTemplate | JPA | MyBatis |
|------|-------------|-----|---------|
| SQL 控制 | 完全控制 | 部分控制 | 完全控制 |
| 对象映射 | 手动/RownMapper | 自动 | 自动 |
| 动态 SQL | 拼接字符串 | Criteria API | XML/注解 |
| 缓存 | 无 | 一级/二级 | 一级/二级 |
| 性能 | 高 | 中 | 高 |
| 学习曲线 | 低 | 高 | 中 |

### 适用场景对比

| 场景 | 推荐 |
|------|------|
| 简单 CRUD | JdbcTemplate / MyBatis Plus |
| 复杂查询 | MyBatis / JdbcTemplate |
| 对象关系复杂 | JPA |
| 快速开发 | JPA / MyBatis Plus |
| SQL 精细控制 | JdbcTemplate / MyBatis |

## 完整示例

### UserDao

```java
@Repository
public class UserDao {

    @Autowired
    private NamedParameterJdbcTemplate namedJdbcTemplate;

    @Autowired
    private JdbcTemplate jdbcTemplate;

    public User findById(Long id) {
        String sql = "SELECT id, name, email, age, create_time FROM user WHERE id = :id";
        return namedJdbcTemplate.queryForObject(
            sql,
            Collections.singletonMap("id", id),
            new BeanPropertyRowMapper<>(User.class)
        );
    }

    public List<User> findByName(String name) {
        String sql = "SELECT * FROM user WHERE name LIKE :name";
        return namedJdbcTemplate.query(
            sql,
            Collections.singletonMap("name", "%" + name + "%"),
            new BeanPropertyRowMapper<>(User.class)
        );
    }

    public int insert(User user) {
        String sql = "INSERT INTO user (name, email, age) VALUES (:name, :email, :age)";
        return namedJdbcTemplate.update(sql, new BeanPropertySqlParameterSource(user));
    }

    public int update(User user) {
        String sql = "UPDATE user SET name = :name, email = :email WHERE id = :id";
        return namedJdbcTemplate.update(sql, new BeanPropertySqlParameterSource(user));
    }

    public int delete(Long id) {
        String sql = "DELETE FROM user WHERE id = ?";
        return jdbcTemplate.update(sql, id);
    }

    public Page<User> findByPage(int page, int size) {
        int offset = page * size;
        String sql = "SELECT * FROM user LIMIT ? OFFSET ?";
        List<User> users = jdbcTemplate.query(
            sql,
            new BeanPropertyRowMapper<>(User.class),
            size, offset
        );

        String countSql = "SELECT COUNT(*) FROM user";
        int total = jdbcTemplate.queryForObject(countSql, Integer.class);

        return new Page<>(users, total, page, size);
    }
}
```

### UserService

```java
@Service
@Transactional
public class UserService {

    @Autowired
    private UserDao userDao;

    public User getUser(Long id) {
        return userDao.findById(id);
    }

    public void saveUser(User user) {
        if (user.getId() == null) {
            userDao.insert(user);
        } else {
            userDao.update(user);
        }
    }

    public void deleteUser(Long id) {
        userDao.delete(id);
    }

    public Page<User> getUserPage(int page, int size) {
        return userDao.findByPage(page, size);
    }
}
```

## 常见问题

### Q1：如何获取自增主键？

```java
// 使用 KeyHolder
KeyHolder keyHolder = new GeneratedKeyHolder();

jdbcTemplate.update(connection -> {
    PreparedStatement ps = connection.prepareStatement(
        "INSERT INTO user (name, email) VALUES (?, ?)",
        Statement.RETURN_GENERATED_KEYS
    );
    ps.setString(1, user.getName());
    ps.setString(2, user.getEmail());
    return ps;
}, keyHolder);

// 获取自增主键
Number key = keyHolder.getKey();
user.setId(key.longValue());
```

### Q2：如何处理 NULL 值？

```java
// 使用 ResultSetExtractor
User user = jdbcTemplate.query(
    "SELECT * FROM user WHERE id = ?",
    (rs) -> {
        if (rs.next()) {
            User u = new User();
            u.setId(rs.getLong("id"));
            u.setName(rs.getString("name"));
            u.setEmail(rs.getString("email"));
            u.setAge(rs.getObject("age", Integer.class));  // 处理 NULL
            return u;
        }
        return null;
    },
    1L
);
```

### Q3：如何处理大数据？

```java
// 处理 CLOB
String content = jdbcTemplate.queryForObject(
    "SELECT content FROM article WHERE id = ?",
    (rs) -> {
        if (rs.next()) {
            Clob clob = rs.getClob("content");
            return clob != null ? clob.getSubString(1, (int) clob.length()) : null;
        }
        return null;
    },
    1L
);

// 处理 BLOB
byte[] data = jdbcTemplate.queryForObject(
    "SELECT data FROM file WHERE id = ?",
    (rs, rowNum) -> {
        if (rs.next()) {
            Blob blob = rs.getBlob("data");
            return blob != null ? blob.getBytes(1, (int) blob.length()) : null;
        }
        return null;
    },
    1L
);
```

## 总结

### JdbcTemplate 优势

| 优势 | 说明 |
|------|------|
| 轻量 | 只封装了 JDBC，依赖少 |
| 灵活 | SQL 完全可控 |
| Spring 集成 | 与 Spring 生态完美融合 |
| 简单 | 学习曲线低 |
| 高性能 | 直接操作 JDBC |

### 使用建议

| 场景 | 推荐 |
|------|------|
| 简单 CRUD | JdbcTemplate |
| 少量查询 | JdbcTemplate |
| 复杂映射 | BeanPropertyRowMapper |
| 命名参数 | NamedParameterJdbcTemplate |
| 批量操作 | batchUpdate |

### 选择建议

```
┌─────────────────────────────────────────────────────────────┐
│                       数据访问选择指南                         │
│                                                             │
│  如果：只需要简单的增删改查                                    │
│  → JdbcTemplate（轻量、简单）                                │
│                                                             │
│  如果：需要强大的对象映射                                      │
│  → JPA / MyBatis（功能强大）                                │
│                                                             │
│  如果：SQL 需要精细控制                                        │
│  → MyBatis / JdbcTemplate（SQL 灵活）                       │
│                                                             │
│  如果：想快速开发                                            │
│  → MyBatis Plus / JPA                                       │
└─────────────────────────────────────────────────────────────┘
```

Spring JDBC Template 是一个常被忽视的利器。在合适的场景下使用它，可以让你的代码既简洁又高效。
