## FastConnectionList：无锁链表

### 普通链表的并发问题

```java
// 普通 LinkedList：非线程安全
public class SlowConnectionList<T> {
    private Node<T> head;
    private Node<T> tail;

    public void add(T connection) {
        synchronized (this) {
            Node<T> newNode = new Node<>(connection);
            tail.next = newNode;  // 需要加锁
            tail = newNode;
        }
    }

    public T remove() {
        synchronized (this) {
            if (head == null) return null;
            T conn = head.value;
            head = head.next;  // 需要加锁
            return conn;
        }
    }
}
```

**问题**：每次操作都需要 synchronized，在高并发下成为瓶颈。

### FastConnectionList 的优化

```java
// FastConnectionList：使用 CAS 无锁操作
public class FastConnectionList<T> {

    // 使用 AtomicReference 实现无锁链表
    private final AtomicReference<Node<T>> head;
    private final AtomicReference<Node<T>> tail;

    public T remove() {
        Node<T> current;
        Node<T> next;

        // CAS 循环：无锁操作
        do {
            current = head.get();
            next = current.next.get();

            // 如果队列为空
            if (next == null) {
                return null;
            }
        } while (!head.compareAndSet(current, next));

        // 成功移除，返回值
        return current.value;
    }
}
```

**关键点**：使用 `AtomicReference` 和 `compareAndSet`，实现无锁操作。

## ConcurrentBag：高并发容器

### 传统池化方案的问题

```java
// 传统连接池：使用 synchronized
public class TraditionalPool {
    private List<Connection> pool = new ArrayList<>();

    public synchronized Connection acquire() {
        if (pool.isEmpty()) {
            return createConnection();
        }
        return pool.remove(pool.size() - 1);
    }

    public synchronized void release(Connection conn) {
        pool.add(conn);
    }
}
```

### ConcurrentBag 的设计

```java
public class ConcurrentBag<T> {

    // ThreadLocal：每个线程缓存自己的连接
    private final ThreadLocal<List<T>> threadList;

    // 共享队列：存放可复用的连接
    private final SynchronousQueue<T> sharedQueue;

    // 等待队列：获取连接的线程排队
    private final ConcurrentLinkedQueue<T> waitQueue;

    /**
     * 从池中获取连接
     */
    public T borrow(long timeout, TimeUnit timeunit) {
        // 1. 先从 ThreadLocal 获取（最快）
        List<T> list = threadList.get();
        if (!list.isEmpty()) {
            return list.remove(list.size() - 1);
        }

        // 2. 从共享队列获取
        T connection = sharedQueue.poll();
        if (connection != null) {
            return connection;
        }

        // 3. 创建新连接
        return createConnection();
    }

    /**
     * 归还连接到池中
     */
    public void add(T connection) {
        // 尝试给等待的线程
        if (!waitQueue.isEmpty()) {
            // 直接传递给等待者
            waitQueue.poll().put(connection);
            return;
        }

        // 放入共享队列
        sharedQueue.offer(connection);
    }
}
```

### steal() 和 add() 机制

## 线程局部存储：ThreadLocal

### ThreadLocal 的作用

```java
// HikariCP 使用 ThreadLocal 缓存连接
public class HikariPool {

    // 每个线程都有自己的连接缓存
    private final ThreadLocal<Connection> threadConnection;

    public Connection getConnection() {
        // 1. 先从 ThreadLocal 获取（最快）
        Connection conn = threadConnection.get();
        if (conn != null && conn.isValid()) {
            return conn;
        }

        // 2. 从连接池获取
        conn = pool.borrow();

        // 3. 存入 ThreadLocal
        threadConnection.set(conn);

        return conn;
    }
}
```

## 最小化连接池

### HikariCP 的理念

HikariCP 主张**最小化连接池**：

```yaml
hikari:
  # 连接池大小
  maximum-pool-size: 10  # 保持较小
  minimum-idle: 5
```

### 为什么连接池要小？

| 连接数 | 优点 | 缺点 |
|--------|------|------|
| 太多 | 并发能力强 | 资源浪费、数据库压力大 |
| 太少 | 资源节省 | 并发能力弱 |
| 合适 | 平衡 | - |

### HikariCP 的建议

```
┌─────────────────────────────────────────────────────────────┐

│                                                             │

│  • 4 核 CPU + 1 SSD = 9 连接                                │
│  • 8 核 CPU + 1 SSD = 17 连接                               │
│                                                             │
│  注意：这是上界，不是越大越好！                               │
```

## 总结

### HikariCP 高性能原因

| 原因 | 说明 |
|------|------|
| FastConnectionList | 无锁链表，减少锁竞争 |
| ConcurrentBag | ThreadLocal + 共享队列，减少竞争 |
| ThreadLocal | 每个线程缓存自己的连接，最快路径 |
| 最小化连接池 | 减少资源占用 |
| 简洁代码 | 代码量少，bug 少 |
