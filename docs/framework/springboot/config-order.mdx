## 一、配置文件加载顺序

Spring Boot 按照以下顺序加载配置文件（后面的会覆盖前面的）：

| 顺序 | 配置文件位置 | 说明 |
| --- | --- | --- |
| 1 | 命令行参数 | `--spring.config.location` |
| 2 | `SPRING_CONFIG_ADDITIONAL_LOCATION` | 环境变量 |
| 3 | 打包 jar 外的 application.yml |  |
| 4 | 打包 jar 内的 application.yml |  |
| 5 | `@PropertySource` 注解 |  |

## 二、多环境配置

### 配置文件命名

```
application.yml              # 默认配置
application-dev.yml          # 开发环境
application-test.yml         # 测试环境
application-prod.yml         # 生产环境
```

### 激活环境

```yaml title="application.yml"
spring:
  profiles:
    active: dev
```

```bash
# 命令行激活
java -jar app.jar --spring.profiles.active=prod
```

## 三、配置属性绑定

### @ConfigurationProperties

```java
@ConfigurationProperties(prefix = "app")
public class AppProperties {

    private String name;
    private int port;

    // getter/setter
}
```

```yaml
app:
  name: My Application
  port: 8080
```

## 四、高频面试题

### 🔴 第一层：配置文件的加载顺序是什么？

**答案要点**：
1. 命令行参数 > 环境变量 > jar 外配置 > jar 内配置
2. 后加载的配置会覆盖先加载的
3. 优先级高的配置文件可以覆盖低优先级的属性

### 🟡 第二层：如何覆盖 Spring Boot 的默认配置？

**答案要点**：
1. 通过 application.yml/properties 覆盖
2. 通过命令行参数覆盖
3. 通过自定义 @Configuration 类覆盖

---

## 五、常见陷阱

> ⚠️ **陷阱一**：多环境配置未激活

如果未正确激活环境，可能使用错误的配置。

> ⚠️ **陷阱二**：配置属性名不匹配

YAML 中的属性名需要与 Java 类中的属性名对应，支持松散绑定。
