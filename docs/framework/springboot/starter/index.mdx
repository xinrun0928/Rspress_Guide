# Spring Boot Starter

Spring Boot 的 Starter 机制是其「约定优于配置」理念的核心体现。一个 Starter 包含两部分：起步依赖（starter）和自动配置（auto-configure）。当你添加一个 Starter 依赖时，Spring Boot 会自动配置对应的功能，你甚至不需要写一行配置代码。

## Starter 的价值

### 开箱即用

以 `spring-boot-starter-web` 为例，添加依赖后，自动获得：

- **Web 框架**：Spring MVC
- **内嵌服务器**：Tomcat/Jetty/Undertow
- **JSON 处理**：Jackson
- **日志**：SLF4J + Logback
- **自动配置**：视图解析器、静态资源处理等

### 版本管理

Starter 通过 `spring-boot-dependencies` 统一管理版本：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <!-- 版本由 parent 统一管理 -->
</dependency>
```

你不需要关心依赖间的版本兼容性问题，Spring Boot 已经为你处理好了。

## 延伸思考

当你使用第三方 Starter 时，是否想过它是如何工作的？一个 Starter 实际上就是一个「配置包」——它打包了一组常用的依赖和自动配置类，让你可以一键启用某个功能。

理解 Starter 的工作原理，不仅能帮助你更好地使用第三方 Starter，还能让你开发出自己的 Starter，将团队内部的通用配置封装起来复用。
