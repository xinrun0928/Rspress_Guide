# 自定义 Starter 的开发流程

你是否想过，为什么添加 `spring-boot-starter-redis` 依赖后，Redis 就自动配置好了？是否想过把团队内部的通用组件封装成一个可复用的 Starter？

这一节，我们来完整学习如何开发一个自定义的 Spring Boot Starter。

## 一、Starter 的标准结构

一个标准的 Spring Boot Starter 由两个模块组成：

```
xxx-starter/
├── xxx-starter/              # 起步依赖模块
│   └── pom.xml
└── xxx-autoconfigure/        # 自动配置模块
    ├── pom.xml
    └── src/main/java/
        └── com/example/autoconfigure/
            └── XxxAutoConfiguration.java
```

### 模块职责划分

| 模块 | 职责 | 依赖内容 |
|---|---|---|
| `xxx-starter` | 仅包含依赖声明 | 依赖 `xxx-autoconfigure` 模块 |
| `xxx-autoconfigure` | 包含自动配置类 | Spring Boot 自动配置依赖 |

这种拆分的好处是：用户只需要引入 `xxx-starter`，自动配置的代码会自动被加载。

## 二、创建自动配置模块

### POM 配置

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>xxx-autoconfigure</artifactId>
    <version>1.0.0</version>
    <packaging>jar</packaging>

    <properties>
        <java.version>17</java.version>
        <spring-boot.version>3.2.0</spring-boot.version>
    </properties>

    <dependencies>
        <!-- Spring Boot 自动配置 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-autoconfigure</artifactId>
        </dependency>

        <!-- 业务依赖 -->
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>xxx-core</artifactId>
            <version>1.0.0</version>
        </dependency>

        <!-- 元数据生成（可选但推荐） -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-autoconfigure-processor</artifactId>
            <optional>true</optional>
        </dependency>
    </dependencies>

    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>${spring-boot.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>
</project>
```

### 定义配置属性类

```java
package com.example.autoconfigure;

import org.springframework.boot.context.properties.ConfigurationProperties;

@ConfigurationProperties(prefix = "example.service")
public class ExampleProperties {
    private String name = "default";
    private int timeout = 3000;
    private boolean enabled = true;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getTimeout() {
        return timeout;
    }

    public void setTimeout(int timeout) {
        this.timeout = timeout;
    }

    public boolean isEnabled() {
        return enabled;
    }

    public void setEnabled(boolean enabled) {
        this.enabled = enabled;
    }
}
```

### 创建自动配置类

```java
package com.example.autoconfigure;

import com.example.core.ExampleService;
import org.springframework.boot.autoconfigure.AutoConfiguration;
import org.springframework.boot.autoconfigure.condition.ConditionalOnClass;
import org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean;
import org.springframework.boot.autoconfigure.condition.ConditionalOnProperty;
import org.springframework.boot.context.properties.EnableConfigurationProperties;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@AutoConfiguration
@ConditionalOnClass(ExampleService.class)
@EnableConfigurationProperties(ExampleProperties.class)
@ConditionalOnProperty(prefix = "example.service", name = "enabled", havingValue = "true", matchIfMissing = true)
public class ExampleAutoConfiguration {

    private final ExampleProperties properties;

    public ExampleAutoConfiguration(ExampleProperties properties) {
        this.properties = properties;
    }

    @Bean
    @ConditionalOnMissingBean
    public ExampleService exampleService() {
        return new ExampleService(properties.getName(), properties.getTimeout());
    }
}
```

**关键注解解析：**

| 注解 | 作用 |
|---|---|
| `@AutoConfiguration` | 标记为自动配置类（Spring Boot 2.7+） |
| `@ConditionalOnClass` | 只有当 `ExampleService` 在 classpath 中时才生效 |
| `@EnableConfigurationProperties` | 启用 `ExampleProperties` 的属性绑定 |
| `@ConditionalOnProperty` | 只有当 `example.service.enabled=true` 时才生效 |
| `@ConditionalOnMissingBean` | 只有当容器中没有 `ExampleService` Bean 时才创建 |

### 注册自动配置类

Spring Boot 3.x 使用 `AutoConfiguration.imports` 文件：

```
src/main/resources/
└── META-INF/
    └── spring/
        └── org.springframework.boot.autoconfigure.AutoConfiguration.imports
```

```text
com.example.autoconfigure.ExampleAutoConfiguration
```

如果需要兼容 Spring Boot 2.7 之前版本，同时保留 `spring.factories`：

```
src/main/resources/
├── META-INF/
│   └── spring/
│       └── org.springframework.boot.autoconfigure.AutoConfiguration.imports
└── META-INF/spring.factories
```

```properties
org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
com.example.autoconfigure.ExampleAutoConfiguration
```

## 四、使用自定义 Starter

### 添加依赖

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>xxx-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 配置属性

```yaml
example:
  service:
    enabled: true
    name: my-service
    timeout: 5000
```

### 使用

```java
@Service
public class MyService {
    private final ExampleService exampleService;

    public MyService(ExampleService exampleService) {
        this.exampleService = exampleService;
    }

    public void doSomething() {
        exampleService.execute();
    }
}
```

## 五、自动配置的条件顺序

如果 Starter 依赖其他自动配置，需要指定加载顺序：

```java
@AutoConfiguration
@AutoConfigureAfter(DataSourceAutoConfiguration.class)
@ConditionalOnClass(ExampleService.class)
public class ExampleAutoConfiguration {
    // ...
}
```

### 常用注解

| 注解 | 作用 |
|---|---|
| `@AutoConfigureAfter` | 在指定配置类之后加载 |
| `@AutoConfigureBefore` | 在指定配置类之前加载 |
| `@AutoConfigureOrder` | 指定加载顺序（数值越小越先加载） |

## 六、Debug 自定义 Starter

### 开启自动配置报告

```bash
java -jar myapp.jar --debug
```

### 查看条件评估

启动时会输出 `CONDITIONS EVALUATION REPORT`，显示每个自动配置类的评估结果。

### 排除不需要的配置

```java
@SpringBootApplication(exclude = {ExampleAutoConfiguration.class})
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

或者在 YAML 中配置：

```yaml
spring:
  autoconfigure:
    exclude:
      - com.example.autoconfigure.ExampleAutoConfiguration
```

## 七、常见问题

### Q1：自动配置类没有被加载？

排查步骤：

1. 检查是否正确创建了 `AutoConfiguration.imports` 或 `spring.factories` 文件
2. 检查文件路径是否正确（`META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`）
3. 检查 `@ConditionalOnClass` 条件是否满足
4. 使用 `--debug` 模式查看自动配置报告

### Q2：自定义 Bean 被自动配置覆盖？

自动配置类使用 `@ConditionalOnMissingBean` 确保用户自定义 Bean 优先生效。但如果自动配置先于用户配置加载，可能导致问题。

Spring Boot 通过加载顺序确保用户配置优先生效。

### Q3：多个 Starter 冲突？

如果多个 Starter 都定义了相同的 Bean，会导致冲突。可以使用 `@AutoConfigureOrder` 或 `@AutoConfigureAfter` 控制加载顺序。

```
│     ├── 定义配置属性类（@ConfigurationProperties）               │

│     └── 注册自动配置（AutoConfiguration.imports）              │
│                                                                  │
│  2. 创建 xxx-starter 模块                                       │
│     └── 依赖 xxx-autoconfigure                                 │
│                                                                  │

│     ├── 添加依赖                                                │
│     └── 配置属性                                                │
└─────────────────────────────────────────────────────────────────┘
```

理解了这个流程，你就可以将团队内部的通用组件封装成 Starter，实现配置的复用和分发。
