# Starter 自动配置原理与条件装配

当你理解了 Spring Boot 的自动配置原理，你会发现自定义 Starter 的核心就是「把配置类和条件注解打包在一起」。这一节，我们深入分析 Starter 自动配置的完整流程。

```

│                    自动配置架构                                   │
├─────────────────────────────────────────────────────────────────┤
│  用户项目                                                        │
│  ├── 添加 Starter 依赖                                           │
│  └── @SpringBootApplication 启动                                 │
│                                                                  │
│  Spring Boot 自动配置机制                                        │
│  ├── @EnableAutoConfiguration                                  │

│  ├── SpringFactoriesLoader                                     │
│  └── 加载 AutoConfiguration.imports / spring.factories          │
│                                                                  │
│  自动配置类                                                      │

└─────────────────────────────────────────────────────────────────┘
```

## 二、自动配置的加载过程

### 1. @EnableAutoConfiguration

`@EnableAutoConfiguration` 通过 `@Import` 导入 `AutoConfigurationImportSelector`：

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Import(AutoConfigurationImportSelector.class)
public @interface EnableAutoConfiguration {
    // ...
}
```

### 2. AutoConfigurationImportSelector

`AutoConfigurationImportSelector` 实现了 `DeferredImportSelector` 接口：

```java
public class AutoConfigurationImportSelector
        implements DeferredImportSelector, BeanClassLoaderAware, ResourceLoaderAware {

    @Override
    public String[] selectImports(AnnotationMetadata annotationMetadata) {
        if (!isEnabled(annotationMetadata)) {
            return EMPTY_CONFIG;
        }

        AutoConfigurationEntry entry = getAutoConfigurationEntry(annotationMetadata);
        return entry.getConfigurations().toArray(new String[0]);
    }
}
```

### 3. 加载候选配置类

```java
protected AutoConfigurationEntry getAutoConfigurationEntry(
        AnnotationMetadata annotationMetadata) {
    // 获取启用的自动配置类列表
    List<String> configurations = getCandidateConfigurations(
        annotationMetadata, metadata);

    // 去重
    configurations = removeDuplicates(configurations);

    // 处理排除项
    Set<String> exclusions = getExclusions(annotationMetadata, attributes);
    configurations.removeAll(exclusions);

    // 条件过滤
    configurations = getConfigurationClassFilter().filter(configurations);

    // 发布事件
    fireAutoConfigurationImportEvents(configurations, exclusions);

    return new AutoConfigurationEntry(configurations, exclusions);
}
```

### 4. 条件过滤

加载到配置类列表后，Spring Boot 会根据条件注解进行过滤：

```java
private List<String> getConfigurationClassFilter().filter(List<String> configurations) {
    // 遍历所有 AutoConfigurationImportFilter
    for (AutoConfigurationImportFilter filter : this.filters) {
        boolean[] matches = filter.match(configurations, autoConfigurationMetadata);
        for (int i = 0; i < matches.length; i++) {
            if (!matches[i]) {
                configurations.remove(i);
            }
        }
    }
    return configurations;
}
```

## 三、条件装配的完整流程

### 条件注解的解析

Spring Boot 在解析 `@Configuration` 类时，会检查 `@Conditional` 注解：

```java
public class ConfigurationClassPostProcessor implements BeanFactoryPostProcessor {

    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        // 处理配置类
        processConfigBeanDefinitions(beanFactory);
    }

    private void processConfigBeanDefinitions(ConfigurableListableBeanFactory beanFactory) {
        // 解析配置类，检测 @Conditional 注解
        ConfigurationClassParser parser = new ConfigurationClassParser(...);
        parser.parse(candidates);

        // 检测 @Conditional 注解
        ConfigurationClassBeanDefinitionReader reader =
            new ConfigurationClassBeanDefinitionReader(...);
        reader.loadBeanDefinitions(configClasses);
    }
}
```

### @ConditionalOnClass 的实现

```java
public class ConditionalOnClassCondition extends SpringBootCondition {

    @Override
    protected ConditionAndSpec getConditionAndSpec(...) {
        // 解析 @ConditionalOnClass 注解
        MultiValueMap<String, Object> classes = metadata.getAllAnnotationAttributes(
            ConditionalOnClass.class.getName());

        List<String> classNames = new ArrayList<>();
        for (Object value : classes.get("value")) {
            classNames.add((String) value);
        }

        // 构建条件规格
        return new ConditionAndSpec(...);
    }

    @Override
    public ConditionOutcome getMatchOutcome(ConditionContext context,
            MultiValueMap<String, Object> attributes) {
        List<String> classNames = attributes.get("value", String.class);
        List<String> missing = new ArrayList<>();

        for (String className : classNames) {
            // 检查类是否存在
            if (!ClassUtils.isPresent(className, getClass().getClassLoader())) {
                missing.add(className);
            }
        }

        if (!missing.isEmpty()) {
            return ConditionOutcome.noMatch(
                "required class(es) not present: " + missing);
        }

        return ConditionOutcome.match();
    }
}
```

## 四、自动配置类的生命周期

### 1. 注册阶段

自动配置类被加载后，会被注册为 Bean 定义：

```java
// ConfigurationClassBeanDefinitionReader.loadBeanDefinitions()
for (String className : configClasses) {
    // 注册 Bean 定义
    reader.loadBeanDefinition(className);
}
```

### 2. 实例化阶段

在容器刷新阶段，非懒加载的单例 Bean 会被实例化：

```java
// AbstractApplicationContext.refresh()
finishBeanFactoryInitialization(beanFactory);

// DefaultListableBeanFactory.preInstantiateSingletons()
for (String beanName : beanNames) {
    // 实例化非懒加载的单例 Bean
    getBean(beanName);
}
```

### 3. 后置处理阶段

Bean 实例化后，会执行后置处理器：

```java
// AbstractAutowireCapableBeanFactory.initializeBean()
for (BeanPostProcessor bp : getBeanPostProcessors()) {
    result = bp.postProcessAfterInitialization(result, beanName);
}
```

## 五、@Configuration 与 @AutoConfiguration

Spring Boot 2.7 引入了 `@AutoConfiguration` 作为 `@Configuration` 的替代。

### 对比

| 特性 | @Configuration | @AutoConfiguration |
|---|---|---|
| 首次出现 | Spring Framework 3.0 | Spring Boot 2.7 |
| proxyBeanMethods | 默认 true | 默认 false |
| 适用场景 | 用户配置类 | 自动配置类 |
| 条件支持 | 需要额外添加 | 内置支持 |

### @AutoConfiguration 的优势

```java
@AutoConfiguration
@ConditionalOnClass(ExampleService.class)
@EnableConfigurationProperties(ExampleProperties.class)
public class ExampleAutoConfiguration {
    // proxyBeanMethods = false，更适合自动配置场景
    // 避免不必要的代理开销
}
```

**proxyBeanMethods = false 的优势：**

1. 避免 CGLIB 代理开销
2. 配置类更轻量
3. 适合一次性配置的场景

## 六、自动配置的顺序控制

### @AutoConfigureAfter


### @AutoConfigureBefore

```java

    // 在 WebMvcAutoConfiguration 之前加载
```

### @AutoConfigureOrder

```java
    // 最高优先级
```

## 七、自动配置事件

### AutoConfigurationImportEvent

Spring Boot 在自动配置过程中会发布事件：

```java
public class AutoConfigurationImportEvent extends ApplicationEvent {
    private final AnnotationMetadata metadata;
    private final Set<String> candidates;
    private final Set<String> exclusions;

    // 可以监听此事件修改候选配置类
}
```

### 自定义监听器

```java
@Component
public class MyAutoConfigurationImportListener
        implements ApplicationListener<AutoConfigurationImportEvent> {

    @Override
    public void onApplicationEvent(AutoConfigurationImportEvent event) {
        Set<String> candidates = event.getCandidateConfigurations();
        System.out.println("自动配置候选: " + candidates);
    }
}
```

## 八、面试高频追问

### Q1：@ConditionalOnMissingBean 和用户 @Bean 的执行顺序？

Spring Boot 确保：

1. `@ComponentScan` 先执行，注册用户配置类
2. `@EnableAutoConfiguration` 后执行，注册自动配置类

因此用户 `@Bean` 优先生效。

### Q2：如何让自动配置类在用户配置类之前执行？

可以通过 `BeanDefinitionRegistryPostProcessor` 手动注册 Bean，但这通常不推荐。

### Q3：spring.factories 和 AutoConfiguration.imports 可以同时存在吗？

可以。Spring Boot 会根据版本选择读取哪个文件：

- Spring Boot 2.7.x：优先读取 `AutoConfiguration.imports`
- Spring Boot 3.x：只读取 `AutoConfiguration.imports`
