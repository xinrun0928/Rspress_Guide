# Spring Boot Actuator 监控端点

在生产环境中，如何了解应用的运行状态？如何监控系统指标？如何在不重启应用的情况下修改日志级别？

Spring Boot Actuator 为你提供了开箱即用的生产监控能力。通过 HTTP 端点或 JMX，你可以查看应用的健康状态、配置信息、运行时指标等。

## 一、快速入门

### 添加依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

### 暴露端点

默认情况下，大部分端点是禁用的。需要在配置中暴露：

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics
  endpoint:
    health:
      show-details: when_authorized
```

### 访问端点

```bash
# 健康检查
curl http://localhost:8080/actuator/health

# 响应
{
    "status": "UP"
}
```

**健康检查聚合器**

## 三、端点配置

### 启用/禁用端点

```yaml
management:
  endpoint:
    health:
      enabled: true
    info:
      enabled: true
    metrics:
      enabled: true
```

### 路径配置

```yaml
management:
  endpoints:
    web:
      base-path: /manage
  endpoint:
    health:
      path: /health-check
```

现在健康检查端点变为 `/manage/health-check`。

### 健康检查详情

```yaml
management:
  endpoint:
    health:
      show-details: always  # 总是显示
      # show-details: when_authorized  # 授权用户显示
      # show-details: never  # 从不显示
```

## 四、自定义健康检查器

### 实现 HealthIndicator

```java
@Component
public class CustomHealthIndicator implements HealthIndicator {

    @Override
    public Health health() {
        try {
            // 执行健康检查逻辑
            boolean isHealthy = checkSomething();
            if (isHealthy) {
                return Health.up()
                    .withDetail("message", "Everything is OK")
                    .build();
            } else {
                return Health.down()
                    .withDetail("message", "Something is wrong")
                    .build();
            }
        } catch (Exception e) {
            return Health.down()
                .withException(e)
                .build();
        }
    }

    private boolean checkSomething() {
        // 检查逻辑
        return true;
    }
}
```

### 自定义健康状态聚合

```java
@Component
public class CustomHealthAggregator implements HealthAggregator {

    @Override
    public Status aggregate(Collection<HealthStatus> statuses) {
        // 自定义聚合逻辑
        // 例如：只要有一个 UP 就返回 UP
        for (HealthStatus status : statuses) {
            if (status.equals(Status.UP)) {
                return Status.UP;
            }
        }
        return Status.DOWN;
    }
}
```

## 五、自定义指标

### 使用 Micrometer

Spring Boot 使用 Micrometer 作为指标门面：

```java
@Service
public class OrderService {
    private final MeterRegistry meterRegistry;

    public OrderService(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }

    public void createOrder(Order order) {
        // 计数
        meterRegistry.counter("orders.created", "status", "success").increment();

        // 计时
        Timer timer = meterRegistry.timer("orders.process.time");
        timer.record(() -> {
            // 处理订单
            processOrder(order);
        });
    }
}
```

### 使用 @Timed 注解

```java
@Service
public class OrderService {
    @Timed(value = "orders.create.time", description = "订单创建耗时")
    public void createOrder(Order order) {
        // 处理订单
    }
}
```

## 七、安全配置

### 保护 Actuator 端点

```yaml
spring:
  security:
    user:
      name: admin
      password: secret

management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  endpoint:
    health:
      roles: ADMIN
```

```java
@Configuration
public class SecurityConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/actuator/health").permitAll()
                .requestMatchers("/actuator/**").hasRole("ADMIN")
            )
            .httpBasic();
        return http.build();
    }
}
```

public class MostSevereHealthAggregator implements HealthAggregator {

### Q3：Actuator 支持哪些监控指标？

Micrometer 支持多种指标：

| 类型 | 说明 | 示例 |
|---|---|---|
| Counter | 计数器 | 请求数 |
| Gauge | 仪表 | CPU 使用率 |
| Timer | 计时器 | 请求耗时 |
| Summary | 摘要 | 响应大小分布 |

## 九、总结

Spring Boot Actuator 的端点体系：

```
┌─────────────────────────────────────────────────────────────────┐
│                    Actuator 端点体系                              │
├─────────────────────────────────────────────────────────────────┤
│  健康检查                                                        │
│  ├── /actuator/health → 应用健康状态                            │
│  └── 自定义 HealthIndicator                                     │
│                                                                  │
│  信息端点                                                        │
│  ├── /actuator/info → 自定义信息                                │
│  ├── /actuator/env → 环境变量                                   │
│  └── /actuator/beans → Bean 列表                                │
│                                                                  │
│  指标端点                                                        │
│  ├── /actuator/metrics → 系统指标                               │
│  ├── /actuator/prometheus → Prometheus 格式                    │
│  └── Micrometer 自定义指标                                       │
│                                                                  │
│  操作端点                                                        │
│  ├── /actuator/loggers → 日志级别                               │
│  ├── /actuator/refresh → 刷新配置                              │
│  └── /actuator/shutdown → 关闭应用                              │
└─────────────────────────────────────────────────────────────────┘
```

Actuator 是 Spring Boot 生产监控的基础，掌握其使用对于运维和开发都至关重要。
