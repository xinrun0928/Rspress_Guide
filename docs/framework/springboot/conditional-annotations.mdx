## 一、条件注解分类

### 类级别条件

| 注解 | 条件 |
| --- | --- |
| @ConditionalOnClass | classpath 中存在指定类 |
| @ConditionalOnMissingClass | classpath 中不存在指定类 |

### Bean 级别条件

| 注解 | 条件 |
| --- | --- |
| @ConditionalOnBean | 容器中存在指定 Bean |
| @ConditionalOnMissingBean | 容器中不存在指定 Bean |

### 属性条件

| 注解 | 条件 |
| --- | --- |
| @ConditionalOnProperty | 配置属性满足条件 |
| @ConditionalOnResource | 指定资源存在 |

### Web 应用条件

| 注解 | 条件 |
| --- | --- |
| @ConditionalOnWebApplication | 是 Web 应用 |
| @ConditionalOnNotWebApplication | 不是 Web 应用 |

## 二、@ConditionalOnProperty 示例

```java
@Configuration
@ConditionalOnProperty(
    prefix = "spring.redis",
    name = {"enabled"},
    havingValue = "true",
    matchIfMissing = false
)
public class RedisAutoConfiguration {
    // ...
}
```

```yaml
spring:
  redis:
    enabled: true
```

---

## 三、@ConditionalOnBean 示例

```java
@Configuration
@ConditionalOnBean(DataSource.class)
@ConditionalOnMissingBean(JdbcTemplate.class)
public class JdbcTemplateAutoConfiguration {
    @Bean
    public JdbcTemplate jdbcTemplate(DataSource dataSource) {
        return new JdbcTemplate(dataSource);
    }
}
```

## 四、高频面试题

### 🔴 第一层：Spring Boot 有哪些条件注解？

**答案要点**：
1. @ConditionalOnClass / @ConditionalOnMissingClass
2. @ConditionalOnBean / @ConditionalOnMissingBean
3. @ConditionalOnProperty
4. @ConditionalOnWebApplication

### 🟡 第二层：@ConditionalOnProperty 的属性有哪些？

**答案要点**：
1. prefix：配置前缀
2. name：配置名
3. havingValue：期望值
4. matchIfMissing：缺失时的默认值

---

## 五、常见陷阱

> ⚠️ **陷阱一**：@ConditionalOnBean 在 Bean 注册后检查

如果 Bean 已经被注册，`@ConditionalOnBean` 可能不会生效。

> ⚠️ **陷阱二**：matchIfMissing 为 true 导致意外生效

如果配置缺失但 `matchIfMissing = true`，条件会意外生效。
