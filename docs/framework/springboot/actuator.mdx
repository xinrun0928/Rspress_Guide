1. Spring Boot Actuator 的主要端点有哪些？

---

## 一、常用端点

| 端点 | 说明 | 路径 |
| --- | --- | --- |
| health | 健康检查 | /actuator/health |
| info | 应用信息 | /actuator/info |
| beans | 容器中的 Bean | /actuator/beans |
| env | 环境变量 | /actuator/env |
| configprops | 配置属性 | /actuator/configprops |
| metrics | 指标信息 | /actuator/metrics |
| mappings | URL 映射 | /actuator/mappings |
| loggers | 日志配置 | /actuator/loggers |

---

## 二、配置端点

```yaml title="application.yml"
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics  # 暴露的端点
      base-path: /actuator            # 端点基础路径
  endpoint:
    health:
      show-details: when-authorized   # 健康详情显示策略
```

---

## 三、自定义健康指示器

```java
@Component
public class RedisHealthIndicator implements HealthIndicator {

    @Override
    public Health health() {
        try {
            // 检查 Redis 连接
            redisTemplate.opsForValue().get("health-check");
            return Health.up().build();
        } catch (Exception e) {
            return Health.down().withDetail("error", e.getMessage()).build();
        }
    }
}
```

## 四、高频面试题

### 🔴 第一层：Actuator 有哪些常用端点？

**答案要点**：
1. health：健康检查
2. metrics：指标信息
3. beans：容器中的 Bean
4. env：环境变量

### 🟡 第二层：如何保护 Actuator 端点？

**答案要点**：
1. 通过 Spring Security 配置端点权限
2. 限制暴露的端点
3. 配置端点访问策略

---

## 五、常见陷阱

> ⚠️ **陷阱一**：生产环境暴露敏感端点

生产环境应该只暴露必要的端点，如 health 和 info。

> ⚠️ **陷阱二**：端点信息泄露

Actuator 端点可能泄露敏感信息，需要做好安全防护。
