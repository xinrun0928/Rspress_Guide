## 二、创建步骤

### 1. 定义配置属性类

```java
@ConfigurationProperties(prefix = "my.service")
public class MyServiceProperties {

    private String host = "localhost";
    private int port = 8080;
    private boolean enabled = true;

    // getter/setter
}
```

### 2. 定义业务类

```java
public class MyService {

    private final MyServiceProperties properties;

    public MyService(MyServiceProperties properties) {
        this.properties = properties;
    }

    public void doSomething() {
        System.out.println("Connecting to " + properties.getHost() + ":" + properties.getPort());
    }
}
```

### 3. 定义自动配置类

```java
@Configuration
@ConditionalOnClass(MyService.class)           // MyService 在 classpath 中
@ConditionalOnProperty(                        // 配置开启
    prefix = "my.service",
    name = "enabled",
    havingValue = "true",
    matchIfMissing = true
)
@EnableConfigurationProperties(MyServiceProperties.class)
public class MyAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public MyService myService(MyServiceProperties properties) {
        return new MyService(properties);
    }
}
```

### 4. 注册自动配置

```text title="src/main/resources/META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports"
com.example.starter.MyAutoConfiguration
```

### 5. 创建 Starter 入口

```xml title="pom.xml"
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-autoconfigure</artifactId>
</dependency>
```

## 三、使用 Starter

### Maven 依赖

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>my-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 配置文件

```yaml title="application.yml"
my:
  service:
    enabled: true
    host: 127.0.0.1
    port: 9090
```

### Java 代码

```java
@Service
public class MyBusinessService {

    @Autowired
    private MyService myService;

    public void process() {
        myService.doSomething();
    }
}
```

## 四、完整示例

### pom.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>my-spring-boot-starter</artifactId>
    <version>1.0.0</version>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-autoconfigure</artifactId>
        </dependency>
    </dependencies>
</project>
```

### MyAutoConfiguration.java

```java
package com.example.starter;

import org.springframework.boot.autoconfigure.AutoConfiguration;
import org.springframework.boot.autoconfigure.condition.ConditionalOnClass;
import org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean;
import org.springframework.boot.autoconfigure.condition.ConditionalOnProperty;
import org.springframework.boot.context.properties.EnableConfigurationProperties;

@Configuration
@AutoConfiguration
@ConditionalOnClass(MyService.class)
@ConditionalOnProperty(prefix = "my.service", name = "enabled", havingValue = "true", matchIfMissing = true)
@EnableConfigurationProperties(MyServiceProperties.class)
public class MyAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public MyService myService(MyServiceProperties properties) {
        return new MyService(properties);
    }
}
```

## 五、高频面试题

### 🔴 第一层：如何创建一个 Spring Boot Starter？

**答案要点**：
1. 创建配置属性类（@ConfigurationProperties）
2. 创建业务类
3. 创建自动配置类（@Configuration + @Conditional）
4. 注册自动配置（AutoConfiguration.imports）

### 🟡 第二层：Starter 和 AutoConfiguration 有什么区别？

**答案要点**：
1. Starter 是对用户的接口，包含依赖管理
2. AutoConfiguration 是实现自动配置的核心类
3. 推荐分离为两个模块：xxx-starter 和 xxx-autoconfigure

## 六、常见陷阱

> ⚠️ **陷阱一**：starter 依赖了不必要的包

Starter 应该只包含必要的依赖，不应该引入具体的实现类。

> ⚠️ **陷阱二**：自动配置类没有条件判断

如果缺少 `@Conditional`，可能导致在某些环境下启动失败。

> ⚠️ **陷阱三**：配置属性没有使用 @EnableConfigurationProperties

如果不启用配置属性类，`@ConfigurationProperties` 注解的 Bean 不会被注册。
