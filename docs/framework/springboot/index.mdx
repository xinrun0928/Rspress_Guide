# Spring Boot

Spring Boot 是 Pivotal 团队在 2014 年发布的开源框架，它的核心设计理念是「约定优于配置」（Convention Over Configuration）。通过自动化配置、内嵌服务器、健康检查等特性，Spring Boot 让开发者能够快速搭建生产级别的 Spring 应用。

## 与 Spring Framework 的关系

很多初学者会混淆 Spring Framework 和 Spring Boot 的关系。简单来说：

- **Spring Framework**：提供了 IoC 容器、AOP、事务管理、Web MVC 等核心功能，是一切的基础
- **Spring Boot**：在 Spring Framework 之上，提供了自动化配置、内嵌服务器、健康检查等便利功能

Spring Boot 不是替代 Spring Framework，而是对 Spring Framework 的扩展和简化。

> [!TIP]
> 如果你对 Spring Framework 的核心概念还不够熟悉，建议先阅读 Spring Framework 相关文档，再来学习 Spring Boot。

## 延伸思考

Spring Boot 的「约定优于配置」理念，深刻影响了后续的 Java 生态。许多框架（如 Spring Cloud、Spring Security）都提供了对应的 Starter，进一步简化了集成成本。

但「约定」也意味着约束。当你的业务场景与约定不符时，如何优雅地覆盖默认配置？这正是下一节要讨论的问题。
