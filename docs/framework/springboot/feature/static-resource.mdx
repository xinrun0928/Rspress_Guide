# Spring Boot 静态资源配置

你有没有遇到过这样的情况：把 CSS/JS 文件放到 `src/main/resources/static` 目录下，但浏览器访问时 404 了。或者配置了自定义静态资源路径，但 Spring Boot 的默认配置覆盖了你的设置。

这一节，我们来深入学习 Spring Boot 的静态资源配置。

## 一、默认静态资源位置

Spring Boot 的静态资源默认位置：

```
classpath:/static/
classpath:/public/
classpath:/resources/
classpath:/META-INF/resources/
```

访问方式（假设文件路径为 `classpath:/static/css/style.css`）：

```
http://localhost:8080/css/style.css
```

**注意**：不需要加 `/static` 前缀。

## 二、自定义静态资源位置

### application.yml 配置

```yaml
spring:
  web:
    resources:
      static-locations:
        - classpath:/static/
        - classpath:/public/
        - classpath:/assets/
```

### Java 配置

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/static/**")
            .addResourceLocations("classpath:/static/");
    }
}
```

## 三、静态资源映射

### 多路径映射

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        // 映射到多个位置
        registry.addResourceHandler("/assets/**")
            .addResourceLocations(
                "classpath:/static/assets/",
                "file:/var/www/assets/"
            );
    }
}
```

### 不同路径不同映射

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        // CSS 文件
        registry.addResourceHandler("/css/**")
            .addResourceLocations("classpath:/static/css/");

        // JS 文件
        registry.addResourceHandler("/js/**")
            .addResourceLocations("classpath:/static/js/");

        // 图片
        registry.addResourceHandler("/images/**")
            .addResourceLocations("classpath:/static/images/", "file:/data/images/");
    }
}
```

## 四、缓存控制

### application.yml 配置

```yaml
spring:
  web:
    resources:
      cache:
        period: 3600  # 缓存 1 小时
```

### Java 配置

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/static/**")
            .addResourceLocations("classpath:/static/")
            .setCacheControl(CacheControl.maxAge(Duration.ofDays(30)));
    }
}
```

### CacheControl 选项

| 选项 | 说明 |
|---|---|
| `maxAge(Duration)` | 缓存最大时间 |
| `cachePublic()` | 公共缓存 |
| `cachePrivate()` | 私有缓存 |
| `noCache()` | 不缓存 |
| `noStore()` | 不存储 |

```java
.setCacheControl(CacheControl.maxAge(Duration.ofDays(30), CacheControl.Fixity.CLIENT_MAX_AGE))
```

## 五、WebJars

### 引入 WebJars

```xml
<dependency>
    <groupId>org.webjars</groupId>
    <artifactId>jquery</artifactId>
    <version>3.6.4</version>
</dependency>
```

### 访问方式

```
http://localhost:8080/webjars/jquery/3.6.4/jquery.min.js
```

### 配置 WebJars

Spring Boot 自动配置已经处理了 WebJars，通常不需要额外配置。

如果需要自定义：

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/webjars/**")
            .addResourceLocations("/webjars/")
            .setCacheControl(CacheControl.maxAge(Duration.ofDays(365)));
    }
}
```

## 六、欢迎页

### 默认配置

Spring Boot 会查找以下文件作为欢迎页：

```
classpath:/static/index.html
classpath:/public/index.html
classpath:/resources/index.html
classpath:/META-INF/resources/index.html
```

访问 `http://localhost:8080/` 时，会自动返回欢迎页。

### 自定义欢迎页

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/")
            .setViewName("index");
    }
}
```

## 七、Favicon

### 默认配置

Spring Boot 自动处理 `favicon.ico`：

```
classpath:/static/favicon.ico
classpath:/public/favicon.ico
classpath:/resources/favicon.ico
classpath:/META-INF/resources/favicon.ico
```

### 禁用 Favicon

```yaml
spring:
  web:
    resources:
      favicon:
        enabled: false
```

### 自定义 Favicon

将 `favicon.ico` 放在静态资源目录下即可。

## 八、静态资源版本控制

### 启用版本控制

```yaml
spring:
  web:
    resources:
      chain:
        strategy:
          content:
            enabled: true
            paths: /**
```

### 自定义版本策略

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/static/**")
            .addResourceLocations("classpath:/static/")
            .resourceChain(true)
            .addResolver(new VersionResourceResolver()
                .addContentVersionStrategy("/**"));
    }
}
```

访问时会自动添加 MD5 哈希：

```
/static/css/style.a1b2c3d4.css
```

## 九、跨域静态资源

### CORS 配置

如果静态资源需要跨域访问：

```yaml
spring:
  web:
    resources:
      cors:
        allowed-origins: "https://example.com"
```

### Java 配置

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addResourceHandler("/static/**")
            .addCorsMappings("/api/**")
            .allowedOrigins("https://example.com")
            .allowedMethods("GET", "POST")
            .maxAge(3600);
    }
}
```

## 十、常见问题

### Q1：静态资源 404？

排查步骤：

1. 确认文件放在正确位置（`classpath:/static/`）
2. 确认文件路径正确（`/static/css/style.css` 访问 `/css/style.css`）
3. 检查是否有 `addResourceHandlers` 覆盖了默认配置
4. 检查文件是否被打包到 JAR 中

### Q2：修改静态文件后浏览器缓存？

1. 使用 Ctrl+F5 强制刷新
2. 配置缓存控制 `Cache-Control: no-cache`
3. 使用版本化文件名

### Q3：War 包中静态资源位置？

在 War 包中，静态资源应该放在 `webapp` 目录：

```
src/
└── main/
    └── webapp/
        └── static/
            └── css/
                └── style.css
```

访问路径：

```
http://localhost:8080/app/static/css/style.css
```
