# Spring Boot 拦截器与过滤器

很多初学者分不清拦截器（Interceptor）和过滤器（Filter）的区别，甚至把它们混为一谈。面试官也经常问：「拦截器和过滤器有什么区别？Spring Boot 中如何注册？」

这一节，我们深入理解这两种组件的区别和使用场景。

## 一、过滤器和拦截器对比

### 基本概念

| 组件 | 来源 | 作用范围 | 接口 |
|---|---|---|---|
| Filter | Servlet 规范 | 请求到达 Servlet 之前 | `jakarta.servlet.Filter` |
| Interceptor | Spring MVC | Controller 处理之前 | `org.springframework.web.servlet.HandlerInterceptor` |

### 执行流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    请求处理流程                                   │
├─────────────────────────────────────────────────────────────────┤
│  1. Filter（多个，按 @Order 顺序）                              │
│     └── doFilter()                                              │
│                                                                  │
│  2. DispatcherServlet                                          │
│                                                                  │
│  3. HandlerInterceptor                                          │
│     └── preHandle() ← 控制器方法执行前                          │
│                                                                  │
│  4. Controller                                                  │
│                                                                  │
│  5. HandlerInterceptor                                          │
│     └── postHandle() ← 控制器方法执行后，视图渲染前             │
│                                                                  │
│  6. View 渲染                                                   │
│                                                                  │
│  7. HandlerInterceptor                                          │
│     └── afterCompletion() ← 视图渲染完成后                      │
│                                                                  │
│  8. Filter（多个，按 @Order 逆序）                              │
│     └── doFilter()                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 核心区别

| 对比项 | Filter | Interceptor |
|---|---|---|
| 规范 | Servlet 规范 | Spring MVC |
| 作用范围 | 整个请求（包含静态资源） | 仅 Controller（不包含静态资源） |
| 依赖 | Servlet API | Spring 容器 |
| 使用场景 | 字符编码、CORS、日志、请求包装 | 登录验证、权限校验、性能监控 |
| 配置方式 | @WebFilter、FilterRegistrationBean | WebMvcConfigurer |

## 二、过滤器（Filter）

### 实现 Filter 接口

```java
import jakarta.servlet.*;
import jakarta.servlet.http.HttpServletRequest;
import java.io.IOException;

public class LoggingFilter implements Filter {

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        // 初始化
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException {
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        long startTime = System.currentTimeMillis();

        try {
            chain.doFilter(request, response);
        } finally {
            long duration = System.currentTimeMillis() - startTime;
            System.out.println(httpRequest.getRequestURI() + " 耗时: " + duration + "ms");
        }
    }

    @Override
    public void destroy() {
        // 销毁
    }
}
```

### @WebFilter 注解

```java
@WebFilter(urlPatterns = "/api/*", filterName = "loggingFilter")
public class LoggingFilter implements Filter {
    // ...
}
```

需要添加组件扫描：

```java
@SpringBootApplication
@ServletComponentScan
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

### FilterRegistrationBean 配置

```java
@Configuration
public class FilterConfig {

    @Bean
    public FilterRegistrationBean<LoggingFilter> loggingFilter() {
        FilterRegistrationBean<LoggingFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new LoggingFilter());
        registration.setUrlPatterns(Arrays.asList("/api/*"));
        registration.setOrder(1);
        return registration;
    }
}
```

### 执行顺序

```java
@Bean
public FilterRegistrationBean<FilterA> filterA() {
    FilterRegistrationBean<FilterA> registration = new FilterRegistrationBean<>();
    registration.setFilter(new FilterA());
    registration.setOrder(1);  // 数值越小越先执行
    return registration;
}

@Bean
public FilterRegistrationBean<FilterB> filterB() {
    FilterRegistrationBean<FilterB> registration = new FilterRegistrationBean<>();
    registration.setFilter(new FilterB());
    registration.setOrder(2);  // 数值越大越后执行
    return registration;
}
```

## 三、拦截器（HandlerInterceptor）

### 实现 HandlerInterceptor

```java
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import org.springframework.web.servlet.HandlerInterceptor;

public class AuthInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
            Object handler) throws Exception {
        // 返回 true 继续执行，返回 false 阻止请求
        String token = request.getHeader("Authorization");
        if (token == null || !isValid(token)) {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.getWriter().write("{\"error\": \"Unauthorized\"}");
            return false;
        }
        return true;
    }

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response,
            Object handler, ModelAndView modelAndView) throws Exception {
        // Controller 方法执行后，视图渲染前
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
            Object handler, Exception ex) throws Exception {
        // 视图渲染完成后
        if (ex != null) {
            // 记录异常日志
        }
    }
}
```

### 注册拦截器

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new AuthInterceptor())
            .addPathPatterns("/api/**")           // 拦截的路径
            .excludePathPatterns("/api/login",   // 排除的路径
                                 "/api/public/**");
    }
}
```

### 拦截路径规则

```java
registry.addInterceptor(new AuthInterceptor())
    .addPathPatterns("/api/**")         // 拦截 /api/xxx
    .addPathPatterns("/admin/*")       // 拦截 /admin/xxx
    .addPathPatterns("/admin/**")      // 拦截 /admin/xxx/yyy
    .excludePathPatterns("/admin/login");  // 排除
```

### 拦截器中的依赖注入

拦截器默认不归 Spring 管理，无法直接注入 Bean。需要通过构造函数注入：

```java
@Component
public class AuthInterceptor implements HandlerInterceptor {

    private final UserService userService;

    public AuthInterceptor(UserService userService) {
        this.userService = userService;
    }

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
            Object handler) throws Exception {
        String token = request.getHeader("Authorization");
        User user = userService.validateToken(token);
        request.setAttribute("currentUser", user);
        return true;
    }
}
```

## 四、过滤器 vs 拦截器选择

### 使用过滤器的场景

| 场景 | 原因 |
|---|---|
| 字符编码 | 需要在请求到达 Spring 之前处理 |
| CORS | 跨域预检请求不经过 Controller |
| XSS 过滤 | 需要处理所有请求 |
| 请求日志 | 需要记录所有请求 |
| 静态资源处理 | 拦截器无法处理 |

### 使用拦截器的场景

| 场景 | 原因 |
|---|---|
| 登录验证 | 需要访问 Spring Bean |
| 权限校验 | 需要获取用户角色 |
| 性能监控 | 需要获取 Controller 执行时间 |
| 数据绑定 | 需要访问 ModelAndView |

## 五、组合使用

### 完整示例

```java
// 字符编码过滤器
@WebFilter(urlPatterns = "/*", filterName = "encodingFilter")
public class EncodingFilter implements Filter {
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        req.setCharacterEncoding("UTF-8");
        res.setCharacterEncoding("UTF-8");
        chain.doFilter(req, res);
    }
}

// 登录拦截器
@Component
public class LoginInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
            Object handler) throws Exception {
        // 验证登录状态
        return true;
    }
}

// 注册配置
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {
    @Autowired
    private LoginInterceptor loginInterceptor;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(loginInterceptor)
            .addPathPatterns("/**")
            .excludePathPatterns("/login", "/register", "/static/**");
    }
}
```

## 六、常见问题

### Q1：Filter 和 Interceptor 哪个先执行？

Filter 先执行。执行顺序：

```
Filter1 → Filter2 → ... → DispatcherServlet → Interceptor → Controller
```

### Q2：如何让 Filter 获取 Spring Bean？

```java
@Configuration
public class FilterConfig {

    @Bean
    public FilterRegistrationBean<MyFilter> myFilter(UserService userService) {
        FilterRegistrationBean<MyFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new MyFilter(userService));  // 注入 Spring Bean
        return registration;
    }
}
```

### Q3：Filter 中如何获取 Controller 方法信息？

```java
@Override
public void doFilter(ServletRequest request, ServletResponse response,
        FilterChain chain) throws IOException, ServletException {
    HttpServletRequest httpRequest = (HttpServletRequest) request;
    HandlerExecutionChain handlerChain =
        (HandlerExecutionChain) httpRequest.getAttribute(
            "jakarta.servlet.dispatch.handler");

    if (handlerChain != null) {
        HandlerMethod handler = (HandlerMethod) handlerChain.getHandler();
        Method method = handler.getMethod();
        System.out.println("Controller: " + method.getName());
    }
}
```

### Q4：拦截器中的异常如何处理？

```java
@Override
public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
        Object handler, Exception ex) throws Exception {
    if (ex != null) {
        // 记录异常日志
        // 发送异常告警
    }
}
```

## 七、面试高频追问

### Q1：拦截器如何获取请求参数？

```java
@Override
public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
        Object handler) throws Exception {
    // GET 参数
    String id = request.getParameter("id");

    // POST 参数（需要读取 body）
    BufferedReader reader = request.getReader();
    String body = reader.lines().collect(Collectors.joining());

    return true;
}
```

### Q2：拦截器如何修改响应？

拦截器的 `postHandle` 方法可以修改 ModelAndView，但无法修改已经写入的响应内容。如果需要修改响应，可以使用 ResponseBodyAdvice：

```java
@Component
@RestControllerAdvice
public class GlobalResponseAdvice implements ResponseBodyAdvice<Object> {

    @Override
    public Object beforeBodyWrite(Object body, MethodParameter returnType,
            MediaType selectedContentType, Class<? extends HttpMessageConverter<?>> selectedConverterType,
            ServerHttpRequest request, ServerHttpResponse response) {
        // 统一包装响应
        return new Result<>(body);
    }
}
```

### Q3：Filter 和 AOP 的区别？

| 对比项 | Filter | AOP |
|---|---|---|
| 作用范围 | Web 请求 | Bean 方法 |
| 粒度 | URL 级别 | 方法级别 |
| 织入时机 | 请求层面 | 方法层面 |
| 使用场景 | 通用功能 | 业务功能 |

## 八、总结

过滤器和拦截器的对比：

```
┌─────────────────────────────────────────────────────────────────┐
│                    过滤器和拦截器对比                              │
├─────────────────────────────────────────────────────────────────┤
│  Filter（Servlet 规范）                                          │
│  ├── 作用范围：所有请求（包括静态资源）                          │
│  ├── 执行时机：DispatcherServlet 之前                            │
│  ├── 使用场景：字符编码、CORS、请求日志                         │
│  └── 配置方式：@WebFilter / FilterRegistrationBean              │
│                                                                  │
│  Interceptor（Spring MVC）                                       │
│  ├── 作用范围：仅 Controller                                    │
│  ├── 执行时机：Controller 前后                                  │
│  ├── 使用场景：登录验证、权限校验                               │
│  └── 配置方式：WebMvcConfigurer                                 │
└─────────────────────────────────────────────────────────────────┘
```
