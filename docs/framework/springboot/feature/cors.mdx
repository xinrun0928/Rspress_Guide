# CORS 跨域配置

浏览器的同源策略（Same-Origin Policy）导致前端页面无法直接调用不同域的后端 API。这就是我们常说的「跨域问题」。

当你看到 `Access-Control-Allow-Origin` 这样的响应头时，说明 CORS 配置正在起作用。这一节，我们来全面学习 Spring Boot 中的 CORS 配置。

## 一、CORS 基础

### 什么是 CORS？

CORS（Cross-Origin Resource Sharing，跨域资源共享）是一种 W3C 标准，允许服务器通过响应头声明哪些来源可以访问其资源。

### 简单请求 vs 预检请求

| 类型 | 说明 | 示例 |
|---|---|---|
| 简单请求 | GET、HEAD、POST + 简单 Header | `Content-Type: text/plain` |
| 预检请求 | PUT、DELETE + 复杂 Header | `Content-Type: application/json` |

预检请求（OPTIONS）会先发送，询问服务器是否允许跨域。

### 关键响应头

| 响应头 | 说明 |
|---|---|
| `Access-Control-Allow-Origin` | 允许的来源 |
| `Access-Control-Allow-Methods` | 允许的 HTTP 方法 |
| `Access-Control-Allow-Headers` | 允许的请求头 |
| `Access-Control-Max-Age` | 预检请求缓存时间 |

## 二、@CrossOrigin 注解

### 基本用法

```java
@RestController
@RequestMapping("/api")
public class UserController {

    @CrossOrigin
    @GetMapping("/users")
    public List<User> getUsers() {
        return userService.findAll();
    }
}
```

### 指定来源

```java
@CrossOrigin(origins = "http://example.com")
@GetMapping("/users")
public List<User> getUsers() {
    return userService.findAll();
}
```

### 多个来源

```java
@CrossOrigin(origins = {"http://example.com", "http://example.org"})
@GetMapping("/users")
public List<User> getUsers() {
    return userService.findAll();
}
```

### 所有来源（生产环境慎用）

```java
@CrossOrigin(origins = "*")
@GetMapping("/users")
public List<User> getUsers() {
    return userService.findAll();
}
```

### 指定方法

```java
@CrossOrigin(methods = {RequestMethod.GET, RequestMethod.POST})
@GetMapping("/users")
public List<User> getUsers() {
    return userService.findAll();
}
```

### 指定请求头

```java
@CrossOrigin(allowedHeaders = {"X-Auth-Token", "Content-Type"})
@PostMapping("/users")
public User createUser(@RequestBody User user) {
    return userService.save(user);
}
```

### 暴露响应头

```java
@CrossOrigin(exposedHeaders = {"X-Total-Count"})
@GetMapping("/users")
public List<User> getUsers(HttpServletResponse response) {
    response.setHeader("X-Total-Count", "100");
    return userService.findAll();
}
```

## 三、类级别 @CrossOrigin

```java
@CrossOrigin(origins = "http://example.com")
@RestController
@RequestMapping("/api")
public class UserController {

    @GetMapping("/users")
    public List<User> getUsers() {
        return userService.findAll();
    }

    @CrossOrigin(origins = "http://example.org")  // 覆盖类级别
    @PostMapping("/users")
    public User createUser(@RequestBody User user) {
        return userService.save(user);
    }
}
```

## 四、全局 CORS 配置

### Java 配置

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins("http://example.com", "http://example.org")
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("*")
            .exposedHeaders("X-Total-Count")
            .allowCredentials(true)
            .maxAge(3600);
    }
}
```

### CorsRegistry 选项

| 方法 | 说明 | 默认值 |
|---|---|---|
| `addMapping(pattern)` | 路径模式 | |
| `allowedOrigins(origins)` | 允许的来源 | `*` |
| `allowedMethods(methods)` | 允许的 HTTP 方法 | `GET, HEAD, POST` |
| `allowedHeaders(headers)` | 允许的请求头 | `*` |
| `exposedHeaders(headers)` | 暴露的响应头 | |
| `allowCredentials(allow)` | 是否允许携带凭证 | `false` |
| `maxAge(seconds)` | 预检请求缓存时间 | 1800 秒 |

### 处理预检请求

Spring MVC 会自动处理 OPTIONS 预检请求，不需要手动配置。

## 五、Spring Security 中的 CORS

如果使用 Spring Security，需要额外配置：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            .csrf(csrf -> csrf.disable())
            // ... 其他配置
        return http.build();
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOrigins(Arrays.asList("http://example.com"));
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        configuration.setAllowedHeaders(Arrays.asList("*"));
        configuration.setAllowCredentials(true);
        configuration.setMaxAge(3600L);

        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/api/**", configuration);
        return source;
    }
}
```

## 六、Filter 方式配置

### 自定义 CorsFilter

```java
@Component
@Order(Ordered.HIGHEST_PRECEDENCE)
public class CorsFilter implements Filter {

    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletResponse response = (HttpServletResponse) res;
        HttpServletRequest request = (HttpServletRequest) req;

        response.setHeader("Access-Control-Allow-Origin", "http://example.com");
        response.setHeader("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS");
        response.setHeader("Access-Control-Allow-Headers", "Authorization, Content-Type");
        response.setHeader("Access-Control-Max-Age", "3600");

        if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
            response.setStatus(HttpServletResponse.SC_OK);
        } else {
            chain.doFilter(req, res);
        }
    }
}
```

## 七、常见问题

### Q1：CORS 预检请求失败？

预检请求使用 OPTIONS 方法，需要确保：

1. 过滤器链中没有阻止 OPTIONS 请求
2. Spring Security 中显式配置允许 OPTIONS
3. 负载均衡器允许 OPTIONS 请求通过

```java
// Spring Security 配置
http
    .requestMatchers(optionsRequestMatcher())
    .permitAll();

// Options 请求匹配器
private RequestMatcher optionsRequestMatcher() {
    return new AntPathRequestMatcher("/api/**", HttpMethod.OPTIONS.name());
}
```

### Q2：credentials 和 allowedOrigins 不能同时使用？

当 `allowCredentials = true` 时，不能使用 `allowedOrigins("*")`，必须指定具体域名：

```java
// 错误
configuration.setAllowedOrigins(Arrays.asList("*"));
configuration.setAllowCredentials(true);

// 正确
configuration.setAllowedOrigins(Arrays.asList("http://example.com"));
configuration.setAllowCredentials(true);
```

### Q3：Vary 响应头

如果服务器根据请求头返回不同的内容，需要设置 Vary 头：

```java
@GetMapping("/api/data")
public ResponseEntity<String> getData(HttpServletRequest request, HttpServletResponse response) {
    response.setHeader("Vary", "Origin");
    return ResponseEntity.ok("data");
}
```

## 八、最佳实践

### 生产环境建议

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Value("${cors.allowed-origins}")
    private List<String> allowedOrigins;

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins(allowedOrigins.toArray(new String[0]))
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("Authorization", "Content-Type", "X-Requested-With")
            .exposedHeaders("X-Total-Count", "X-Page-Number")
            .allowCredentials(true)
            .maxAge(3600);
    }
}
```

### 环境配置

```yaml
# application-dev.yml
cors:
  allowed-origins:
    - http://localhost:3000
    - http://localhost:8080

# application-prod.yml
cors:
  allowed-origins:
    - https://example.com
```

## 九、面试高频追问

### Q1：@CrossOrigin 和 WebMvcConfigurer.addCorsMappings() 的优先级？

类和方法级别的 `@CrossOrigin` 会覆盖全局配置。全局配置提供默认行为，注解提供细粒度控制。

### Q2：CORS 和 JSONP 的区别？

| 对比项 | CORS | JSONP |
|---|---|---|
| 支持方法 | 所有 HTTP 方法 | 仅 GET |
| 错误处理 | 支持 | 不支持 |
| 浏览器支持 | 现代浏览器 | 老旧浏览器 |
| 服务器配置 | 通过响应头 | 通过 `<script>` 标签 |

### Q3：Spring Security 中 CORS 的执行顺序？

CORS 过滤器早于 CSRF 过滤器执行，所以即使 CSRF 禁用，CORS 也能正常工作。
