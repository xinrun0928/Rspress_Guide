# Spring Boot 异常处理：ErrorController 与全局异常

当你的 API 抛出异常时，用户看到的是 Spring Boot 默认的错误页面，还是友好的 JSON 响应？对于 RESTful API，这个问题至关重要。

这一节，我们来深入学习 Spring Boot 的异常处理机制。

## 一、Spring Boot 默认异常处理

Spring Boot 提供默认的异常处理机制：

1. **浏览器请求**：返回 HTML 错误页面（Whitelabel Error Page）
2. **非浏览器请求**（如 API 调用）：返回 JSON 错误信息

```json
{
    "timestamp": "2024-01-15T10:30:00.000+0000",
    "status": 500,
    "error": "Internal Server Error",
    "message": "Something went wrong",
    "path": "/api/users"
}
```

## 二、ErrorController

### 自定义错误控制器

```java
@Controller
@RequestMapping("${server.error.path:${error.path:/error}}")
public class CustomErrorController implements ErrorController {

    @GetMapping
    public ResponseEntity<Map<String, Object>> error(HttpServletRequest request) {
        Map<String, Object> body = new HashMap<>();
        body.put("timestamp", new Date());
        body.put("status", request.getAttribute("jakarta.servlet.error.status_code"));
        body.put("error", request.getAttribute("jakarta.servlet.error.message"));
        body.put("path", request.getAttribute("jakarta.servlet.error.request_uri"));

        return ResponseEntity
            .status((Integer) body.get("status"))
            .body(body);
    }
}
```

### 错误属性

| 属性 | 说明 |
|---|---|
| `jakarta.servlet.error.status_code` | HTTP 状态码 |
| `jakarta.servlet.error.message` | 错误消息 |
| `jakarta.servlet.error.request_uri` | 请求路径 |
| `jakarta.servlet.error.exception` | 异常对象 |

## 三、@ControllerAdvice 全局异常处理

### 基本用法

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(NullPointerException.class)
    public ResponseEntity<ErrorResult> handleNullPointer(NullPointerException e) {
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(new ErrorResult("NULL_POINTER", "空指针异常"));
    }

    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ErrorResult> handleBusiness(BusinessException e) {
        return ResponseEntity
            .status(HttpStatus.BAD_REQUEST)
            .body(new ErrorResult(e.getCode(), e.getMessage()));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResult> handleGeneral(Exception e) {
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(new ErrorResult("INTERNAL_ERROR", "系统异常"));
    }
}
```

### ErrorResult 类

```java
public class ErrorResult {
    private String code;
    private String message;
    private LocalDateTime timestamp;
    private Map<String, Object> details;

    // 构造函数、Getter/Setter
}
```

## 四、@ExceptionHandler 详解

### 方法参数

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    // 1. 异常对象
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ErrorResult> handle(BusinessException e) {
        // e 是捕获的异常
    }

    // 2. HttpServletRequest
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResult> handle(HttpServletRequest request, Exception e) {
        String path = request.getRequestURI();
    }

    // 3. HttpServletResponse
    @ExceptionHandler(Exception.class)
    public void handle(HttpServletResponse response, Exception e) throws IOException {
        response.setStatus(HttpStatus.INTERNAL_SERVER_ERROR.value());
    }

    // 4. @RequestParam 参数
    @GetMapping("/test")
    public void test(@RequestParam(required = false) String name) {
        if (name == null) {
            throw new IllegalArgumentException("name 不能为空");
        }
    }
}
```

### 返回值

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    // 返回 ResponseEntity
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResult> handle1() { }

    // 返回 Map（自动包装为 JSON）
    @ExceptionHandler(Exception.class)
    public Map<String, Object> handle2() {
        return Map.of("code", 500, "message", "error");
    }

    // 返回 String（视图名）
    @ExceptionHandler(Exception.class)
    public String handle3() {
        return "error";
    }

    // 返回 void（直接写响应）
    @ExceptionHandler(Exception.class)
    public void handle4(HttpServletResponse response) throws IOException {
        response.getWriter().write("error");
    }
}
```

## 五、@ResponseStatus 注解

### 标记异常类

```java
@ResponseStatus(HttpStatus.NOT_FOUND)
public class ResourceNotFoundException extends RuntimeException {
    public ResourceNotFoundException(String message) {
        super(message);
    }
}
```

使用：

```java
@GetMapping("/users/{id}")
public User getUser(@PathVariable Long id) {
    User user = userService.findById(id);
    if (user == null) {
        throw new ResourceNotFoundException("用户不存在: " + id);
    }
    return user;
}
```

### @ResponseStatus + @ExceptionHandler

```java
@ExceptionHandler(ResourceNotFoundException.class)
@ResponseStatus(HttpStatus.NOT_FOUND)
public ErrorResult handleNotFound(ResourceNotFoundException e) {
    return new ErrorResult("NOT_FOUND", e.getMessage());
}
```

## 六、自定义业务异常

### 定义异常类

```java
public class BusinessException extends RuntimeException {
    private final String code;

    public BusinessException(String code, String message) {
        super(message);
        this.code = code;
    }

    public String getCode() {
        return code;
    }
}
```

### 预定义业务异常

```java
public enum BusinessError {
    USER_NOT_FOUND("USER_001", "用户不存在"),
    INVALID_PASSWORD("USER_002", "密码错误"),
    DUPLICATE_USERNAME("USER_003", "用户名已存在");

    private final String code;
    private final String message;

    BusinessError(String code, String message) {
        this.code = code;
        this.message = message;
    }

    public BusinessException get() {
        return new BusinessException(this.code, this.message);
    }

    public BusinessException get(Object... args) {
        String message = MessageFormat.format(this.message, args);
        return new BusinessException(this.code, message);
    }
}
```

使用：

```java
@GetMapping("/users/{id}")
public User getUser(@PathVariable Long id) {
    return userService.findById(id)
        .orElseThrow(() -> BusinessError.USER_NOT_FOUND.get(id));
}
```

## 七、异常处理顺序

`@ControllerAdvice` 支持多个，处理顺序：

1. **精确匹配**：先查找完全匹配的异常类型
2. **继承层次**：父类异常在后处理
3. **@Order 注解**：相同优先级按 Order 顺序

```java
@RestControllerAdvice
@Order(1)
public class SpecificExceptionHandler {

    @ExceptionHandler(NullPointerException.class)
    public ResponseEntity<ErrorResult> handle() { }
}

@RestControllerAdvice
@Order(2)
public class GeneralExceptionHandler {

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResult> handle() { }
}
```

## 八、Validation 异常处理

### 处理 MethodArgumentNotValidException

```java
@RestControllerAdvice
public class ValidationExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResult> handleValidation(MethodArgumentNotValidException e) {
        Map<String, String> errors = new HashMap<>();
        e.getBindingResult().getFieldErrors().forEach(error ->
            errors.put(error.getField(), error.getDefaultMessage())
        );

        return ResponseEntity
            .status(HttpStatus.BAD_REQUEST)
            .body(new ErrorResult("VALIDATION_ERROR", "参数校验失败", errors));
    }
}
```

### 处理 ConstraintViolationException

```java
@ExceptionHandler(ConstraintViolationException.class)
public ResponseEntity<ErrorResult> handleConstraint(ConstraintViolationException e) {
    Map<String, String> errors = new HashMap<>();
    e.getConstraintViolations().forEach(violation ->
        errors.put(violation.getPropertyPath().toString(), violation.getMessage())
    );

    return ResponseEntity
        .status(HttpStatus.BAD_REQUEST)
        .body(new ErrorResult("VALIDATION_ERROR", "参数校验失败", errors));
}
```

## 九、404 异常处理

### 方式一：ErrorController

```java
@Controller
public class NotFoundController implements ErrorController {

    @GetMapping("/error")
    public ResponseEntity<ErrorResult> error(HttpServletRequest request) {
        Integer status = (Integer) request.getAttribute("jakarta.servlet.error.status_code");
        if (status == 404) {
            return ResponseEntity
                .status(HttpStatus.NOT_FOUND)
                .body(new ErrorResult("NOT_FOUND", "资源不存在"));
        }
        return ResponseEntity
                .status(status)
                .body(new ErrorResult("ERROR", "未知错误"));
    }
}
```

### 方式二：HandlerFunction

```java
@Component
public class CustomErrorWebExceptionHandler implements ErrorWebExceptionHandler {

    @Override
    public Mono<Void> handle(ServerWebExchange exchange, Throwable ex) {
        if (exchange.getResponse().isCommitted()) {
            return Mono.error(ex);
        }

        ServerHttpResponse response = exchange.getResponse();
        response.setStatusCode(HttpStatus.NOT_FOUND);

        return response.setComplete();
    }
}
```

## 十、面试高频追问

### Q1：@ControllerAdvice 和 @RestControllerAdvice 的区别？

| 对比项 | @ControllerAdvice | @RestControllerAdvice |
|---|---|---|
| 返回值处理 | 不自动加 @ResponseBody | 自动加 @ResponseBody |
| 适用场景 | 需要返回视图 | RESTful API |

实际上 `@RestControllerAdvice` 是 `@ControllerAdvice` + `@ResponseBody` 的组合。

### Q2：异常处理器的优先级？

1. 精确匹配的异常类型优先
2. 子类异常优先于父类异常
3. 相同异常类型按 `@Order` 排序

### Q3：如何统一处理所有异常（包括 Filter 中的异常）？

```java
@Component
public class GlobalExceptionFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest request,
            HttpServletResponse response, FilterChain filterChain)
            throws ServletException, IOException {
        try {
            filterChain.doFilter(request, response);
        } catch (Exception e) {
            handleException(response, e);
        }
    }
}
```
