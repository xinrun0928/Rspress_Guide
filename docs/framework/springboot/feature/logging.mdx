# Spring Boot 日志配置：Logback、SLF4J

你有没有遇到过这种情况：本地日志输出正常，但部署到服务器后发现日志级别不对，或者日志格式混乱，甚至日志文件根本就没生成？

这些问题都与 Spring Boot 的日志配置有关。这一节，我们来深入学习 Spring Boot 的日志体系。

## 一、日志框架体系

Spring Boot 使用 SLF4J 作为日志门面，Logback 作为日志实现：

```
┌─────────────────────────────────────────────────────────────────┐
│                    日志框架体系                                    │
├─────────────────────────────────────────────────────────────────┤
│  日志门面（API）                                                  │
│  └── SLF4J（Simple Logging Facade for Java）                    │
│                                                                  │
│  日志实现                                                        │
│  ├── Logback（默认，Spring Boot Starter 中包含）                 │
│  ├── Log4j2                                                    │
│  └── JUL（java.util.logging）                                    │
│                                                                  │
│  桥接器                                                          │
│  └── jcl-over-slf4j → 桥接 Commons Logging                      │
│  └── jul-to-slf4j → 桥接 JUL                                   │
│  └── log4j-over-slf4j → 桥接 Log4j                             │
└─────────────────────────────────────────────────────────────────┘
```

## 二、默认日志配置

Spring Boot 默认配置已经足够日常使用：

```yaml
# application.yml
logging:
  level:
    root: INFO
    com.example: DEBUG
  pattern:
    console: "%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"
```

### 日志级别

| 级别 | 说明 | 使用场景 |
|---|---|---|
| `TRACE` | 最详细 | 追踪详细信息 |
| `DEBUG` | 调试级别 | 开发环境 |
| `INFO` | 信息级别 | 正常运行信息 |
| `WARN` | 警告级别 | 潜在问题 |
| `ERROR` | 错误级别 | 错误信息 |

## 三、Logback 配置文件

Spring Boot 自动扫描 `logback-spring.xml`、`logback.xml` 等配置文件。

### 基本结构

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <!-- 定义变量 -->
    <property name="LOG_HOME" value="/var/log/myapp"/>
    <property name="APP_NAME" value="my-application"/>

    <!-- Console Appender -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- File Appender -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_HOME}/${APP_NAME}.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_HOME}/${APP_NAME}.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <timeBasedFileNamingAndTriggeringPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedFNATP">
                <maxFileSize>100MB</maxFileSize>
            </timeBasedFileNamingAndTriggeringPolicy>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- Root Logger -->
    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="FILE"/>
    </root>

    <!-- Logger -->
    <logger name="com.example" level="DEBUG" additivity="false">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="FILE"/>
    </logger>
</configuration>
```

## 四、多环境日志配置

Spring Boot 支持根据 Profile 选择不同的日志配置。

### 使用 springProfile

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <include resource="org/springframework/boot/logging/logback/defaults.xml"/>

    <!-- Console Appender -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 开发环境：DEBUG 级别 -->
    <springProfile name="dev">
        <root level="DEBUG">
            <appender-ref ref="CONSOLE"/>
        </root>
        <logger name="com.example" level="DEBUG"/>
    </springProfile>

    <!-- 生产环境：INFO 级别 -->
    <springProfile name="prod">
        <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
            <file>/var/log/myapp/app.log</file>
            <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
                <fileNamePattern>/var/log/myapp/app.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
                <maxHistory>30</maxHistory>
            </rollingPolicy>
            <encoder>
                <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n</pattern>
            </encoder>
        </appender>

        <root level="INFO">
            <appender-ref ref="CONSOLE"/>
            <appender-ref ref="FILE"/>
        </root>
        <logger name="com.example" level="INFO"/>
    </springProfile>
</configuration>
```

### 使用 Spring Property

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <springProperty scope="context" name="APP_NAME" source="spring.application.name"/>

    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/${APP_NAME}.log</file>
        <!-- ... -->
    </appender>
</configuration>
```

## 五、日志格式详解

### 常用占位符

| 占位符 | 说明 | 示例 |
|---|---|---|
| `%d` | 日期时间 | `%d{yyyy-MM-dd HH:mm:ss}` |
| `%thread` | 线程名 | `[main]` |
| `%level` | 日志级别 | `INFO` |
| `%logger` | Logger 名称 | `c.e.s.UserService` |
| `%logger{36}` | 截取前36字符 | `c.e.s.UserService` |
| `%msg` | 日志消息 | `User logged in` |
| `%n` | 换行 | |
| `%exception{50}` | 异常信息，最多50行 | |

### 自定义格式

```xml
<encoder>
    <pattern>
        %d{yyyy-MM-dd HH:mm:ss.SSS} %highlight(%-5level) [%blue(%thread)] %cyan(%logger{50}) - %msg%n
    </pattern>
</encoder>
```

## 六、滚动策略

### TimeBasedRollingPolicy

按时间滚动：

```xml
<rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
    <fileNamePattern>logs/app.%d{yyyy-MM-dd}.log</fileNamePattern>
    <maxHistory>30</maxHistory>
</rollingPolicy>
```

### SizeAndTimeBasedRollingPolicy

按时间和大小滚动：

```xml
<rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
    <fileNamePattern>logs/app.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
    <maxFileSize>100MB</maxFileSize>
    <maxHistory>30</maxHistory>
</rollingPolicy>
```

### 保留策略

| 参数 | 说明 |
|---|---|
| `maxHistory` | 保留天数 |
| `totalSizeCap` | 保留总大小 |

```xml
<rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
    <fileNamePattern>logs/app.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
    <maxFileSize>100MB</maxFileSize>
    <maxHistory>30</maxHistory>
    <totalSizeCap>10GB</totalSizeCap>
</rollingPolicy>
```

## 七、动态日志级别

### 使用 Actuator

```yaml
management:
  endpoints:
    web:
      exposure:
        include: loggers
  endpoint:
    loggers:
      enabled: true
```

### 查看日志级别

```bash
curl http://localhost:8080/actuator/loggers/com.example
```

```json
{
    "effectiveLevel": "DEBUG"
}
```

### 修改日志级别

```bash
curl -X POST http://localhost:8080/actuator/loggers/com.example \
  -H "Content-Type: application/json" \
  -d '{"configuredLevel": "DEBUG"}'
```

## 八、异步日志

为了提高性能，可以将日志输出改为异步：

```xml
<appender name="ASYNC_FILE" class="ch.qos.logback.classic.AsyncAppender">
    <queueSize>512</queueSize>
    <discardingThreshold>0</discardingThreshold>
    <includeCallerData>false</includeCallerData>
    <appender-ref ref="FILE"/>
</appender>

<root level="INFO">
    <appender-ref ref="ASYNC_FILE"/>
</root>
```

## 九、面试高频追问

### Q1：Logback 和 Log4j2 的区别？

| 对比项 | Logback | Log4j2 |
|---|---|---|
| 性能 | 高 | 极高（支持异步） |
| 自动重载 | 支持 | 支持 |
| 配置格式 | XML/Groovy | XML/JSON/ YAML |
| 依赖 | 默认集成 | 需要单独引入 |

### Q2：Spring Boot 如何切换日志框架？

排除 Logback，引入 Log4j2：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-logging</artifactId>
        </exclusion>
    </exclusions>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-log4j2</artifactId>
</dependency>
```

### Q3：日志打印性能问题？

1. 使用 `{}` 占位符而非字符串拼接
2. 避免在日志方法内进行复杂计算
3. 生产环境使用异步日志
