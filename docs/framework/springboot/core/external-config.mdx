# Spring Boot 外部化配置：application.yml 与 @ConfigurationProperties

你有没有遇到过这样的场景：在本地开发时数据库连接正常，但部署到测试环境后，应用却启动失败。错误日志显示 `Could not resolve placeholder 'spring.datasource.url' in value "${spring.datasource.url}"`。

这很可能是因为你只配置了 `application.yml`，但没有正确处理不同环境的配置加载顺序。

**Spring Boot 的外部化配置是其「约定优于配置」理念的重要体现。** 理解配置加载顺序，是避免这类问题的关键。

## 一、外部化配置概述

外部化配置（Externalized Configuration）是 Spring Boot 的一大特性，允许将配置从代码中分离出来，通过文件、环境变量、命令行参数等方式提供。

### 配置加载层次

Spring Boot 的配置可以从多个来源加载，优先级从高到低：

```
┌─────────────────────────────────────────────────────────────────┐
│                    配置优先级（从高到低）                         │
├─────────────────────────────────────────────────────────────────┤
│  1. 命令行参数（--spring.config.location=/path/to/config）        │
│  2. 命令行属性（--server.port=8080）                             │
│  3. JVM 系统属性（-Dspring.config.location=/path）                │
│  4. 操作系统环境变量                                             │
│  5. application-{profile}.yml 或 application-{profile}.properties │
│  6. application.yml 或 application.properties                   │
│  7. @SpringApplicationTest 随机属性                             │
└─────────────────────────────────────────────────────────────────┘
```

**核心原则**：后面的配置会覆盖前面的同名配置。

## 二、application.yml 基础

### 基本语法

```yaml
# application.yml
spring:
  application:
    name: my-application
  datasource:
    url: jdbc:mysql://localhost:3306/test
    username: root
    password: password
    driver-class-name: com.mysql.cj.jdbc.Driver

server:
  port: 8080
  servlet:
    context-path: /api

logging:
  level:
    root: INFO
    com.example: DEBUG
```

### YAML 语法要点

1. **缩进**：使用空格缩进，不支持 Tab
2. **大小写敏感**：`name` 和 `Name` 是不同的属性
3. **列表**：`-` 表示列表项
4. **引用**：`&` 定义锚点，`*` 引用锚点

```yaml
# 列表示例
servers:
  - ip: 127.0.0.1
    port: 8080
  - ip: 127.0.0.2
    port: 9090

# 引用示例
defaults:
  host: &host "localhost"
  url: http://*host
```

## 三、application.yml 加载顺序

### Spring Boot 2.4 之前

Spring Boot 2.4 之前的加载顺序（从低到高）：

1. `classpath:/application.yml`
2. `classpath:/application.properties`
3. `classpath:/config/application.yml`
4. `classpath:/config/application.properties`
5. `file:./application.yml`
6. `file:./application.properties`
7. `file:./config/application.yml`
8. `file:./config/application.properties`
9. `spring.config.location` 指定的位置

### Spring Boot 2.4 之后

Spring Boot 2.4 重新设计了配置加载机制：

1. `classpath:/application.yml`
2. `classpath:/application.properties`
3. `classpath:/config/application.yml`
4. `classpath:/config/application.properties`
5. `file:./application.yml`
6. `file:./application.properties`
7. `file:./config/application.yml` （新增）
8. `file:./config/application.properties` （新增）
9. `spring.config.location` 指定的位置（作为额外位置）
10. `spring.config.import` 导入的配置文件

**重大变化**：`spring.config.location` 从覆盖行为改为追加行为，`spring.config.import` 成为新增的扩展机制。

## 四、Profile 配置

### 基本用法

```yaml
# application-dev.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/test
    username: root
    password: password

# application-prod.yml
spring:
  datasource:
    url: jdbc:mysql://prod:3306/prod
    username: prod_user
    password: ${DB_PASSWORD}
```

### 激活 Profile

```bash
# 命令行参数
java -jar app.jar --spring.profiles.active=prod

# JVM 参数
java -jar app.jar -Dspring.profiles.active=prod

# 环境变量
export SPRING_PROFILES_ACTIVE=prod
```

### 多 Profile 激活

```yaml
spring:
  profiles:
    active: dev,common
```

## 五、@ConfigurationProperties 详解

### 基本用法

```java
// application.yml
app:
  title: My Application
  cache:
    timeout: 300
    max-size: 1000
```

```java
// 配置属性类
@ConfigurationProperties(prefix = "app")
@Component
public class AppProperties {
    private String title;
    private CacheProperties cache = new CacheProperties();

    // Getter/Setter
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public CacheProperties getCache() { return cache; }
    public void setCache(CacheProperties cache) { this.cache = cache; }

    // 内部类
    public static class CacheProperties {
        private int timeout = 60;
        private int maxSize = 100;

        // Getter/Setter
        public int getTimeout() { return timeout; }
        public void setTimeout(int timeout) { this.timeout = timeout; }
        public int getMaxSize() { return maxSize; }
        public void setMaxSize(int maxSize) { this.maxSize = maxSize; }
    }
}
```

### 启用配置属性

Spring Boot 3.x 推荐使用 `@EnableConfigurationProperties` 或 `@ConfigurationPropertiesScan`：

```java
@SpringBootApplication
@ConfigurationPropertiesScan  // Spring Boot 2.2+
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

或者在配置类中启用：

```java
@Configuration
@EnableConfigurationProperties(AppProperties.class)
public class AppConfig {
    // ...
}
```

## 六、松散绑定（Relaxed Binding）

Spring Boot 支持「松散绑定」，即配置属性名与 POJO 字段名不必完全匹配。

### 绑定规则

| 配置属性 | Java 字段 | 说明 |
|---|---|---|
| `app.title` | `appTitle` | 烤串式 → 驼峰式 |
| `app.title` | `appTitle` | 烤串式 → 帕斯卡式 |
| `app.title` | `appTitle` | 全大写下划线 → 驼峰式 |
| `app.title` | `appTitle` | 全大写下划线 → 帕斯卡式 |

```yaml
# 都可以绑定到 AppProperties.title
app.title=My Application
app.title=My Application
app.TITLE=My Application
app.title=My Application
```

### 常用缩写

```yaml
# 这些都可以绑定
spring.jpa.show-sql=true
spring.jpa.showSql=true
spring.JPA.show-sql=true
```

## 七、配置属性校验

### JSR-303 校验

```java
@ConfigurationProperties(prefix = "app")
@Validated
public class AppProperties {
    @NotBlank
    private String title;

    @Min(1)
    @Max(100)
    private int cacheTimeout;

    @Email
    private String adminEmail;
}
```

```yaml
app:
  title: ""  # 会触发校验失败
  cache-timeout: 0  # 会触发校验失败
```

### 自定义校验

```java
@Constraint(validatedBy = UniqueEnvironmentValidator.class)
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
public @interface UniqueEnvironment {
    String message() default "Environment must be unique";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}
```

## 八、配置属性类型转换

Spring Boot 会自动转换配置值的类型。

### 基本类型转换

```yaml
# 字符串 → int
server:
  port: 8080

# 字符串 → boolean
spring:
  main:
    allow-bean-definition-overriding: false

# 字符串 → Duration
spring:
  session:
    timeout: 30m  # 30分钟
```

### Duration 类型

```yaml
# 支持的单位：ms, s, m, h, d
spring:
  session:
    timeout: 30m
  cache:
    ttl: 2h30m
```

### DataSize 类型

```yaml
# 支持的单位：B, KB, MB, GB, TB
spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 100MB
```

## 九、配置属性监听

### PropertySourcesPlaceholderConfigurer

```java
@Component
public class ConfigPropertiesReloader {
    @Autowired
    private AppProperties appProperties;

    @Scheduled(fixedDelay = 60000)
    public void reloadIfChanged() {
        // 监控配置变化并重新加载
    }
}
```

### EnvironmentChangeEvent

当配置变化时，Spring Boot 会发布 `EnvironmentChangeEvent`：

```java
@Component
public class MyEnvironmentChangeListener
        implements ApplicationListener<EnvironmentChangeEvent> {

    @Override
    public void onApplicationEvent(EnvironmentChangeEvent event) {
        for (String key : event.getKeys()) {
            System.out.println("Changed: " + key);
        }
    }
}
```

## 十、面试高频追问

### Q1：@ConfigurationProperties 和 @Value 的区别？

| 对比项 | @ConfigurationProperties | @Value |
|---|---|---|
| 绑定方式 | 批量绑定一组配置属性 | 逐个注入单个属性 |
| 功能 | 支持宽松绑定 | 不支持 |
| 元数据 | 支持 JSR-303 校验 | 不支持 |
| SPEL | 不支持 | 支持 `#{}` 表达式 |
| 默认值 | 无默认值 | 可以指定默认值 |
| 适用场景 | 配置类 POJO | 简单属性注入 |

### Q2：为什么 @ConfigurationProperties 不需要 @PropertySource？

`@ConfigurationProperties` 会自动从 `Environment` 中读取属性，而 `Environment` 已经包含了所有配置源（application.yml、环境变量等）。不需要显式指定 `@PropertySource`。

### Q3：如何让配置属性热更新？

Spring Boot 2.0 之后支持通过 `/actuator/refresh` 端点刷新配置：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

```yaml
management:
  endpoints:
    web:
      exposure:
        include: refresh
```

## 十一、总结

Spring Boot 的外部化配置机制：

```
┌─────────────────────────────────────────────────────────────────┐
│                    外部化配置体系                                  │
├─────────────────────────────────────────────────────────────────┤
│  配置源                                                          │
│  ├── application.yml / application.properties                   │
│  ├── Profile 配置（application-{profile}.yml）                  │
│  ├── 命令行参数                                                 │
│  ├── 环境变量                                                   │
│  └── spring.config.import                                       │
│                                                                  │
│  配置绑定                                                        │
│  ├── @ConfigurationProperties → POJO                            │
│  ├── @Value → 单个属性                                          │
│  └── Environment → 直接访问                                    │
│                                                                  │
│  配置特性                                                        │
│  ├── 松散绑定（Relaxed Binding）                                 │
│  ├── 类型转换（Duration、DataSize）                            │
│  ├── JSR-303 校验                                               │
│  └── 配置监听（EnvironmentChangeEvent）                         │
└─────────────────────────────────────────────────────────────────┘
```

理解外部化配置，你就能优雅地管理不同环境的配置，避免硬编码和配置冲突。
