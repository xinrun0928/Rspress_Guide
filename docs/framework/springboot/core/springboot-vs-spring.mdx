# Spring Boot vs Spring：自动配置与起步依赖

你一定见过这样的场景：面试官问「Spring Boot 和 Spring Framework 有什么区别？」，很多候选人会脱口而出「Spring Boot 是 Spring 的升级版」「Spring Boot 比 Spring 更好用」。这些回答都没错，但都没有触及问题的本质。

**Spring Boot 不是 Spring 的升级版，而是一套快速启动 Spring 应用的脚手架。** 它的核心价值在于：通过自动配置，解决了 Spring Framework 配置繁琐的问题。

## 一、为什么需要 Spring Boot？

### 传统 Spring 开发的痛点

在 Spring Boot 出现之前，使用 Spring Framework 开发一个 Web 应用，你需要手动配置大量的 XML 文件或 Java 配置类：

```xml
<!-- 传统 Spring MVC 配置 -->
<beans>
    <!-- 数据源配置 -->
    <bean id="dataSource" class="org.apache.commons.dbcp.BasicDataSource">
        <property name="driverClassName" value="com.mysql.jdbc.Driver"/>
        <property name="url" value="jdbc:mysql://localhost:3306/test"/>
        <property name="username" value="root"/>
        <property name="password" value="password"/>
    </bean>

    <!-- Session 工厂 -->
    <bean id="sessionFactory" class="org.springframework.orm.hibernate5.LocalSessionFactoryBean">
        <property name="dataSource" ref="dataSource"/>
        <!-- 还有一堆属性配置... -->
    </bean>

    <!-- 事务管理器 -->
    <bean id="transactionManager" class="org.springframework.orm.hibernate5.HibernateTransactionManager">
        <property name="sessionFactory" ref="sessionFactory"/>
    </bean>

    <!-- 组件扫描 -->
    <context:component-scan base-package="com.example"/>

    <!-- 事务注解驱动 -->
    <tx:annotation-driven transaction-manager="transactionManager"/>

    <!-- 视图解析器 -->
    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
        <property name="prefix" value="/WEB-INF/views/"/>
        <property name="suffix" value=".jsp"/>
    </bean>
</beans>
```

这还只是一个简化版的配置。实际项目中，一个完整的 Spring 应用可能需要几十甚至上百行配置。而且，这些配置大部分是模板化的——每个项目的数据源配置几乎一样，事务管理器的配置也几乎一样。

### Spring Boot 的解决思路

Spring Boot 的核心理念是**约定优于配置**（Convention Over Configuration）。它默认提供了大多数场景下的最佳配置，开发者只需要在少数地方进行定制。

```yaml
# Spring Boot 配置（同样功能的 Web 应用）
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/test
    username: root
    password: password
```

就这样，一个完整的 Web 应用配置就完成了。不需要 XML、不需要手动配置数据源、不需要配置事务管理器、不需要配置视图解析器——Spring Boot 都帮你做好了。

## 二、核心差异对比

### 1. 配置方式

| 对比项 | Spring Framework | Spring Boot |
|---|---|---|
| 配置方式 | XML 配置或 Java 配置类 | application.yml/properties |
| 配置量 | 大量模板化配置 | 极简配置或零配置 |
| 组件扫描 | 手动指定或默认扫描 | 默认扫描主类所在包 |
| 依赖管理 | 手动管理版本 | 起步依赖自动管理 |

### 2. 依赖管理

**Spring Framework** 需要手动管理所有依赖的版本：

```xml
<!-- Spring Framework -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-webmvc</artifactId>
    <version>5.3.25</version>
</dependency>
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-jdbc</artifactId>
    <version>5.3.25</version>
</dependency>
<!-- 还需要手动处理依赖冲突 -->
```

**Spring Boot** 通过起步依赖简化：

```xml
<!-- Spring Boot -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <!-- 版本由 parent 或 spring-boot-dependencies 统一管理 -->
</dependency>
```

`spring-boot-starter-web` 会自动引入所有必要的依赖：Spring MVC、Spring Core、Jackson（JSON 处理）、Tomcat（内嵌服务器）等。你不需要知道这些 transitive dependencies 具体是哪些，Spring Boot 都已经处理好了。

### 3. 应用部署

**Spring Framework** 需要将应用打包成 WAR 并部署到外部服务器：

```xml
<packaging>war</packaging>
```

**Spring Boot** 支持 JAR 包部署，内嵌服务器：

```xml
<packaging>jar</packaging>
```

```java
public static void main(String[] args) {
    SpringApplication.run(Application.class, args);
    // 内嵌 Tomcat/Jetty/Undertow，无需部署到外部服务器
}
```

### 4. 自动配置

这是 Spring Boot 和 Spring Framework 最大的区别。

Spring Boot 的自动配置会尝试根据添加的依赖自动配置 Spring 应用。例如：

- 如果 `spring-boot-starter-web` 在 classpath 中，Spring Boot 会自动配置 `DispatcherServlet`、视图解析器等
- 如果 `spring-boot-starter-data-jpa` 在 classpath 中，Spring Boot 会自动配置 `EntityManagerFactory`、事务管理器等
- 如果 `HikariCP` 在 classpath 中，Spring Boot 会自动配置 `HikariDataSource` 作为数据源

这些自动配置是通过 `@EnableAutoConfiguration` 机制实现的。

## 三、自动配置解决了什么问题？

### 问题 1：版本兼容

Spring Framework 有众多模块，不同版本之间可能存在兼容性问题。Spring Boot 通过 `spring-boot-dependencies` 统一管理所有依赖的版本：

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>3.2.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

### 问题 2：配置繁琐

模板化的配置（数据源、事务管理器、视图解析器等）在每个项目中几乎一样，Spring Boot 通过自动配置让这些配置「零成本」。

### 问题 3：依赖组合

一个完整的 Web 应用需要 dozens of JARs。起步依赖将常用的依赖组合打包：

| 起步依赖 | 包含内容 |
|---|---|
| spring-boot-starter-web | spring-webmvc, spring-web, jackson, tomcat |
| spring-boot-starter-data-jpa | spring-data-jpa, hibernate-core, spring-orm |
| spring-boot-starter-security | spring-security-web, spring-security-config |

### 问题 4：生产就绪

Spring Boot 提供了一键启用的生产特性：

- **Actuator**：健康检查、指标监控
- **外部化配置**：支持多环境配置
- **日志**：开箱即用的日志配置
- **Banner**：可定制的启动 Banner

## 四、面试追问

### Q1：Spring Boot 是否可以不使用自动配置？

可以。可以通过 `@SpringBootApplication(exclude = {...})` 排除自动配置类，或者设置 `spring.boot.enableautoconfiguration=false`。

### Q2：自动配置和 `@Bean` 配置同时存在会怎样？

Spring Boot 的设计原则是「用户自定义优先」。自动配置类通常使用 `@ConditionalOnMissingBean`，只有当容器中不存在对应的 Bean 时才会生效。

### Q3：起步依赖会不会导致 JAR 包过大？

Spring Boot 提供 `spring-boot-maven-plugin` 的 `exclude` 功能，可以排除不需要的依赖。同时，Spring Boot 支持分层 JAR（layered JAR），可以进一步优化镜像大小。

## 总结

Spring Boot vs Spring 的核心差异在于：

- **Spring Framework**：灵活但配置繁琐，需要手动管理依赖版本
- **Spring Boot**：约定优于配置，通过自动配置简化开发，但需要理解其原理才能正确定制

Spring Boot 不是要替代 Spring Framework，而是站在 Spring Framework 的肩膀上，提供一种更便捷的使用方式。
