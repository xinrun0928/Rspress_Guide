# 自动配置原理：@EnableAutoConfiguration 与 META-INF/spring.factories

很多人在面试时会背「Spring Boot 会自动配置 DataSource、DispatcherServlet」，但如果你追问「Spring Boot 是怎么知道要配置这些的？」，十个人里有九个答不上来。

这一节，我们深入自动配置的内部原理，从 `@EnableAutoConfiguration` 到 `spring.factories`，一步步揭开 Spring Boot 的自动化魔法。

## 一、自动配置的整体流程

在深入源码之前，先理解自动配置的整体流程：

```
┌────────────────────────────────────────────────────────────────┐
│                   自动配置整体流程                               │
├────────────────────────────────────────────────────────────────┤
│  1. @EnableAutoConfiguration                                   │
│     └── @Import(AutoConfigurationImportSelector)               │
│                                                                  │
│  2. AutoConfigurationImportSelector.selectImports()            │
│     └── 读取 META-INF/spring.factories                        │
│     └── 返回自动配置类名列表                                    │
│                                                                  │
│  3. Spring 容器处理配置类                                       │
│     └── 解析 @Configuration                                    │
│     └── 执行 @ConditionalOnClass 等条件判断                     │
│     └── 注册符合条件的 Bean                                     │
└────────────────────────────────────────────────────────────────┘
```

## 二、AutoConfigurationImportSelector 源码解析

`AutoConfigurationImportSelector` 实现了 `DeferredImportSelector` 接口，在容器刷新的早期阶段被处理。

### 核心方法：selectImports()

```java
public class AutoConfigurationImportSelector
        implements DeferredImportSelector, BeanClassLoaderAware, ResourceLoaderAware {

    @Override
    public String[] selectImports(AnnotationMetadata annotationMetadata) {
        if (!isEnabled(annotationMetadata)) {
            return EMPTY_CONFIG;
        }

        // 1. 获取自动配置入口点
        AutoConfigurationEntry autoConfigurationEntry =
            getAutoConfigurationEntry(annotationMetadata);

        // 2. 过滤排除的配置类
        // 3. 返回配置类数组
        return autoConfigurationEntry.getConfigurations().toArray(new String[0]);
    }
}
```

### 获取自动配置条目：getAutoConfigurationEntry()

```java
protected AutoConfigurationEntry getAutoConfigurationEntry(
        AnnotationMetadata annotationMetadata) {
    if (!isEnabled(annotationMetadata)) {
        return EMPTY_ENTRY;
    }

    // 1. 获取 @EnableAutoConfiguration 的属性
    AnnotationAttributes attributes =
        getAttributes(annotationMetadata);

    // 2. 加载候选配置类列表
    List<String> configurations = getCandidateConfigurations(
        attributes, metadata);

    // 3. 去重
    configurations = removeDuplicates(configurations);

    // 4. 处理排除项
    Set<String> exclusions = getExclusions(annotationMetadata, attributes);
    configurations.removeAll(exclusions);

    // 5. 根据条件过滤
    configurations = getConfigurationClassFilter().filter(configurations);

    // 6. 发布事件
    AutoConfigurationImportEvent event = new AutoConfigurationImportEvent(
        this, annotationMetadata, configurations);
    if (event.getEntries().size() > 0) {
        invokeAwareInterfaces(event);
    }

    return new AutoConfigurationEntry(configurations, exclusions);
}
```

**关键步骤解读：**

1. **getCandidateConfigurations()**：加载所有候选配置类（从 `spring.factories` 读取）
2. **removeDuplicates()**：去除重复的配置类名
3. **getExclusions()**：获取需要排除的配置类（来自 `exclude` 属性）
4. **filter()**：根据条件注解（`@ConditionalOnClass` 等）过滤

### 加载候选配置类：getCandidateConfigurations()

```java
protected List<String> getCandidateConfigurations(
        AnnotationMetadata metadata, AnnotationAttributes attributes) {
    // 关键方法：SpringFactoriesLoader.loadFactoryNames()
    List<String> configurations = SpringFactoriesLoader.loadFactoryNames(
        getSpringFactoriesLoaderFactoryClass(),  // = EnableAutoConfiguration.class
        getBeanClassLoader()
    );
    return configurations;
}
```

这里调用了 `SpringFactoriesLoader.loadFactoryNames()`，这是 Spring Framework 提供的一种 SPI 机制。

## 三、SpringFactoriesLoader 机制

`SpringFactoriesLoader` 是 Spring Framework 的核心工具类，用于加载 `META-INF/spring.factories` 文件。

### 文件格式

`spring.factories` 使用 Properties 格式，key 是接口/抽象类的全限定名，value 是实现类的全限定名列表（逗号分隔）：

```properties
# META-INF/spring.factories
org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
org.springframework.boot.autoconfigure.data.jpa.JpaRepositoriesAutoConfiguration,\
org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration,\
org.springframework.boot.autoconfigure.orm.jpa.HibernateJpaAutoConfiguration,\
org.springframework.boot.autoconfigure.web.servlet.WebMvcAutoConfiguration
```

### 源码实现

```java
public final class SpringFactoriesLoader {
    // spring.factories 文件位置
    public static final String FACTORIES_RESOURCE_LOCATION = "META-INF/spring.factories";

    public static List<String> loadFactoryNames(Class<?> factoryType, ClassLoader classLoader) {
        ClassLoader classLoaderToUse = classLoader;
        if (classLoader == null) {
            classLoader = SpringFactoriesLoader.class.getClassLoader();
        }

        // 加载 META-INF/spring.factories
        Enumeration<URL> urls = classLoader.getResources(FACTORIES_RESOURCE_LOCATION);
        List<String> result = new ArrayList<>();

        while (urls.hasMoreElements()) {
            URL url = urls.nextElement();
            // 解析 Properties 文件
            Properties properties = PropertiesLoaderUtils.loadProperties(new UrlResource(url));

            String factoryTypeName = factoryType.getName();
            String[] factoryImplementationNames =
                StringUtils.commaDelimitedListToStringArray(
                    properties.getProperty(factoryTypeName));

            for (String factoryImplementationName : factoryImplementationNames) {
                result.add(factoryImplementationName.trim());
            }
        }

        return result;
    }
}
```

**执行流程：**

1. 扫描 classpath 中所有的 `META-INF/spring.factories` 文件
2. 解析每个文件，查找 key 为 `EnableAutoConfiguration` 的条目
3. 收集所有匹配的实现类名
4. 返回配置类名列表

### 自动配置类的来源

Spring Boot 的自动配置类来自两个地方：

1. **Spring Boot 官方**：位于 `spring-boot-autoconfigure.jar` 中的 `META-INF/spring.factories`
2. **第三方 Starter**：每个 Starter 的 `spring-boot-autoconfigure` 模块中都有自己的 `META-INF/spring.factories`

## 四、条件过滤机制

加载到配置类名列表后，`AutoConfigurationImportSelector` 会根据条件注解进行过滤。

### 过滤流程

```java
// 在 getAutoConfigurationEntry() 方法中
configurations = getConfigurationClassFilter().filter(configurations);
```

`ConfigurationClassFilter` 会遍历所有 `Condition` 的实现类，逐一检查：

```java
private static class ConfigurationClassFilter {
    private final AutoConfigurationImportFilter[] filters;

    List<String> filter(List<String> configurations) {
        long startTime = System.nanoTime();
        String[] allFilters = getMetadataValueFactory().getClassNames(
            AutoConfigurationMetadata.class, "AutoConfiguration.autoConfigurationModule");

        boolean enabled = getMetadataValueFactory().getBoolean(
            AutoConfigurationMetadata.class, "AutoConfiguration.autoConfigurationModule");

        if (enabled && allFilters.length > 0) {
            for (AutoConfigurationImportFilter filter : this.filters) {
                // 执行过滤
                boolean[] election = filter.match(allFilters, autoConfigurationMetadata);
                for (int i = 0; i < election.length; i++) {
                    if (!election[i]) {
                        configurations.remove(allFilters[i]);
                    }
                }
            }
        }
        return configurations;
    }
}
```

### 核心条件注解

Spring Boot 提供了丰富的条件注解，用于控制配置类是否生效：

| 注解 | 条件 |
|---|---|
| `@ConditionalOnClass` | classpath 中存在指定类 |
| `@ConditionalOnMissingClass` | classpath 中不存在指定类 |
| `@ConditionalOnBean` | 容器中存在指定 Bean |
| `@ConditionalOnMissingBean` | 容器中不存在指定 Bean |
| `@ConditionalOnProperty` | 配置属性满足条件 |
| `@ConditionalOnWebApplication` | 是 Web 应用 |

### 示例：DataSourceAutoConfiguration

```java
@Configuration(proxyBeanMethods = false)
@ConditionalOnClass({ DataSource.class, EmbeddedDatabaseType.class })
@ConditionalOnMissingBean(type = "io.r2dbc.spi.ConnectionFactory")
@EnableConfigurationProperties(DataSourceProperties.class)
@Import({ DataSourcePoolMetadataProvidersConfiguration.class })
public class DataSourceAutoConfiguration {
    // ...
}
```

**解读：**

1. `@ConditionalOnClass({ DataSource.class, EmbeddedDatabaseType.class })`：只有当 classpath 中存在 `DataSource` 和 `EmbeddedDatabaseType` 时才生效
2. `@ConditionalOnMissingBean`：只有当容器中不存在 `ConnectionFactory` 类型的 Bean 时才生效
3. `@EnableConfigurationProperties`：启用 `DataSourceProperties` 的配置属性绑定

## 五、自动配置的执行顺序

自动配置类的执行顺序由 `@AutoConfigureAfter`、`@AutoConfigureBefore`、`@AutoConfigureOrder` 控制：

```java
@AutoConfigureAfter({ DataSourceAutoConfiguration.class })
public class HibernateJpaAutoConfiguration {
    // ...
}
```

这个配置确保 `HibernateJpaAutoConfiguration` 在 `DataSourceAutoConfiguration` 之后执行。

```yaml
      - org.springframework.boot.autoconfigure.orm.jpa.HibernateJpaAutoConfiguration
```

## 七、面试高频追问

### Q1：自动配置类的加载时机？

自动配置类在 `refresh()` 的 `invokeBeanFactoryPostProcessors()` 阶段被处理。具体来说，是由 `ConfigurationClassPostProcessor` 触发 `AutoConfigurationImportSelector` 的 `selectImports()` 方法。

### Q2：@ConditionalOnMissingBean 的坑？

`@ConditionalOnMissingBean` 可能在某些场景下导致意外结果：

1. 如果用户的 `@Bean` 定义先于自动配置类处理，`@ConditionalOnMissingBean` 会错误地认为 Bean 不存在
2. Spring Boot 通过 `initializer` 机制确保自动配置在用户配置之后处理

### Q3：spring.factories 文件太多，怎么避免冲突？

Spring Boot 使用 `@AutoConfigureAfter` 控制加载顺序。如果多个配置类需要按特定顺序执行，可以添加依赖声明。

## 八、总结

自动配置的核心机制可以概括为：

```
┌─────────────────────────────────────────────────────────────────┐
│                      自动配置核心机制                            │
├─────────────────────────────────────────────────────────────────┤
│  1. SpringFactoriesLoader 加载 spring.factories                 │
│     └── 读取 META-INF/spring.factories                          │
│     └── 收集 EnableAutoConfiguration 的所有实现类               │
│                                                                  │
│  2. AutoConfigurationImportSelector 处理                        │
│     └── 去重、排除、过滤                                         │
│     └── 根据 @ConditionalOnClass 等条件判断                     │
│                                                                  │
│  3. ConfigurationClassPostProcessor 解析                        │
│     └── 注册符合条件的自动配置类                                  │
│     └── 实例化 @Bean 方法                                        │
└─────────────────────────────────────────────────────────────────┘
```

理解了这个机制，你就掌握了 Spring Boot 自动配置的精髓。下次遇到自动配置不生效的问题，你就能快速定位原因。
