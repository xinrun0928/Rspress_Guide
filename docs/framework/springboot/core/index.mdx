# Spring Boot 核心

Spring Boot 的核心在于**自动配置**和**约定优于配置**。这一章将深入剖析 Spring Boot 的底层实现，从启动流程到自动配置原理，从条件装配到外部化配置，带你真正理解 Spring Boot 是如何工作的。

## 面试考察分析

Spring Boot 核心原理是面试的高频考点，主要集中在以下几个方向：

| 考察维度 | 常见问题 | 掌握程度 |
|---|---|---|
| 注解组合 | @SpringBootApplication 包含哪几个注解？ | 必须掌握 |
| 自动配置触发 | @EnableAutoConfiguration 是如何工作的？ | 必须掌握 |
| 条件装配 | @ConditionalOnClass 的底层实现？ | 必须掌握 |
| 配置加载顺序 | application.yml 的加载顺序是怎样的？ | 理解 |
| 源码流程 | SpringApplication.run() 的执行流程？ | 理解 |

## 学习路线

建议按照以下顺序学习：

1. **入门阶段**：先了解 Spring Boot vs Spring 的核心差异，理解「自动配置解决了什么问题」
2. **基础阶段**：掌握 @SpringBootApplication 注解的组合原理，了解外部化配置
3. **进阶阶段**：深入自动配置源码，理解条件装配的实现机制
4. **实战阶段**：能够自定义 Starter，理解 Spring Boot 的设计思想

## 延伸思考

Spring Boot 的自动配置机制看似复杂，但其核心思想非常简单：**通过条件判断，决定哪些配置类需要生效**。
