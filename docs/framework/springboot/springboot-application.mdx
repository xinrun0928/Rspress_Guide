# @SpringBootApplication 注解详解

> 目标级别：P6
>
> 面试命中率：80%

## 快速自测

1. @SpringBootApplication 由哪些注解组成？
2. @ComponentScan 默认扫描哪些包？
3. 如何排除特定的自动配置类？

---

## 一、注解组成

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Configuration
@EnableAutoConfiguration
@ComponentScan
public @interface SpringBootApplication {
    // ...
}
```

---

## 二、三个核心注解详解

### @Configuration

标记为 Spring 配置类，等价于 XML 配置中的 `<beans>` 标签：

```java
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

### @EnableAutoConfiguration

启用 Spring Boot 的自动配置机制：

```java
// 可以排除特定的自动配置类
@EnableAutoConfiguration(exclude = {DataSourceAutoConfiguration.class})
public class Application { }
```

### @ComponentScan

扫描 `@Component`、`@Service`、`@Repository`、`@Controller` 等注解：

```java
@SpringBootApplication
// 默认扫描主类所在的包及其子包
public class Application { }

// 自定义扫描路径
@ComponentScan(basePackages = {"com.example.service", "com.example.dao"})
public class Application { }
```

---

## 三、属性详解

```java
@SpringBootApplication(
    scanBasePackages = "com.example",      // 自定义扫描包
    scanBasePackageClasses = {Service.class}, // 指定扫描的类
    excludeName = {"RedisAutoConfiguration"}, // 排除的自动配置类
    exclude = {DataSourceAutoConfiguration.class} // 排除的自动配置类
)
public class Application { }
```

---

## 四、常见用法

### 排除自动配置类

```java
// 方式一：exclude 属性
@SpringBootApplication(exclude = {DataSourceAutoConfiguration.class})
public class Application { }

// 方式二：配置文件
@SpringBootApplication
public class Application { }
// application.yml
spring:
  autoconfigure:
    exclude:
      - org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
```

## 五、高频面试题

### 🔴 第一层：@SpringBootApplication 由哪些注解组成？

**答案要点**：
1. `@Configuration`：配置类
2. `@EnableAutoConfiguration`：自动配置
3. `@ComponentScan`：组件扫描

### 🟡 第二层：@ComponentScan ���认扫描哪些包？

**答案要点**：
1. 默认扫描主类所在的包及其子包
2. 如果需要自定义，可以通过 `scanBasePackages` 属性配置

---

## 六、常见陷阱

> ⚠️ **陷阱一**：ComponentScan 覆盖默认行为

如果自定义 `@ComponentScan`，会覆盖默认行为，只扫描指定的包。

> ⚠️ **陷阱二**：排除的类不存在

如果 `exclude` 中指定的类不存在，会抛出异常。
