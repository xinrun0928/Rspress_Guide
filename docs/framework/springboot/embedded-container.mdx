# 嵌入式 Web 容器

> 目标级别：P6
>
> 面试命中率：60%

## 快速自测

1. Spring Boot 支持哪些嵌入式 Web 容器？
2. 如何切换嵌入式容器？
3. 嵌入式容器的启动流程是怎样的？

---

## 一、支持的嵌入式容器

| 容器 | Starter | 默认 |
| --- | --- | --- |
| Tomcat | spring-boot-starter-tomcat | ✅ |
| Jetty | spring-boot-starter-jetty | ❌ |
| Undertow | spring-boot-starter-undertow | ❌ |

---

## 二、切换嵌入式容器

### 排除默认容器，引入新容器

```xml title="pom.xml"
<dependencies>
    <!-- 排除 Tomcat -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
        <exclusions>
            <exclusion>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-starter-tomcat</artifactId>
            </exclusion>
        </exclusions>
    </dependency>

    <!-- 引入 Undertow -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-undertow</artifactId>
    </dependency>
</dependencies>
```

---

## 三、配置嵌入式容器

```yaml title="application.yml"
server:
  port: 8080
  tomcat:
    threads:
      max: 200
      min-spare: 10
    max-connections: 10000
```

## 四、高频面试题

### 🔴 第一层：Spring Boot 支持哪些嵌入式容器？

**答案要点**：
1. Tomcat（默认）
2. Jetty
3. Undertow

### 🟡 第二层：如何切换嵌入式容器？

**答案要点**：
1. 排除默认的 Tomcat Starter
2. 引入新的容器 Starter
3. Spring Boot 会自动检测并使用新容器

---

## 五、对比总结

| 容器 | 特点 | 并发性能 | 适用场景 |
| --- | --- | --- | --- |
| Tomcat | 功能全面，稳定性好 | 中 | 通用场景 |
| Jetty | 轻量级，可定制 | 高 | 长连接场景 |
| Undertow | 高性能，低内存 | 最高 | 高并发场景 |
