# Shiro 实战：从认证授权到无状态改造

理论讲了一堆，是时候动手实战了。

本章将带你完成三个实战场景：**无状态 JWT 认证**、**权限数据模型设计**、**CSRF 防护**。这些是 Shiro 在真实项目中绕不开的话题。

## 为什么需要无状态认证？

传统的 Shiro Session 模式：

```
用户登录 → Shiro 创建 Session → 存入 Redis → 返回 Session ID

后续请求 → 携带 Session ID → 验证 Session → 处理请求
```

问题：

1. **Session 存储需要空间**：大量用户意味着大量 Session
2. **分布式 Session 同步**：多实例间需要 Session 复制或共享存储
3. **跨域限制**：Cookie 不能跨域传递
4. **移动端不友好**：移动端没有 Cookie

JWT 无状态认证：

```
用户登录 → 生成 JWT Token → 返回给客户端

后续请求 → 携带 JWT → 验证 JWT 签名 → 处理请求（无需查询 Session）
```

优势：

1. **无状态**：服务端不存储任何用户信息
2. **天然跨域**：Token 放在 Header 中即可
3. **适合微服务**：每个服务都可以验证 Token
4. **性能更好**：无需查询 Session 存储
