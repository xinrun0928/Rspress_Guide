# Shiro JSP 标签库：前端页面的权限控制

后端有了 Shiro 的认证授权，前端页面也需要根据权限显示或隐藏内容——管理员能看到删除按钮，普通用户看不到。

Shiro 提供了 JSP 标签库和 Thymeleaf 整合，让前端页面可以基于权限动态渲染。

## JSP 标签库

### 引入依赖

```xml
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-web</artifactId>
    <version>1.12.0</version>
</dependency>
```

### 在 JSP 中引入标签

```txt
<%@ taglib prefix="shiro" uri="http://shiro.apache.org/tags" %>
```

### 常用标签

#### guest 标签：匿名用户

```txt
<shiro:guest>
    <a href="/login">登录</a> |
    <a href="/register">注册</a>
</shiro:guest>
```

#### user 标签：已认证或 RememberMe

```txt
<shiro:user>
    欢迎, <shiro:principal /> |
    <a href="/logout">退出</a>
</shiro:user>
```

#### authenticated 标签：已认证（不含 RememberMe）

```txt
<shiro:authenticated>
    <a href="/settings">账户设置</a>
    <a href="/password/change">修改密码</a>
</shiro:authenticated>
```

#### principal 标签：显示主体信息

```txt
<!-- 显示用户名 -->
欢迎, <shiro:principal />

<!-- 显示特定类型的 principal -->
欢迎, <shiro:principal type="java.lang.String" />
```

#### hasRole 标签：角色判断

```txt
<!-- 单个角色 -->
<shiro:hasRole name="admin">
    <a href="/admin">管理后台</a>
</shiro:hasRole>

<!-- 多个角色（需要全部拥有） -->
<shiro:hasAllRoles name="admin,manager">
    <a href="/sensitive">敏感操作</a>
</shiro:hasAllRoles>

<!-- 多个角色之一 -->
<shiro:hasAnyRoles name="admin,user,vip">
    <a href="/vip">VIP 专区</a>
</shiro:hasAnyRoles>
```

#### hasPermission 标签：权限判断

```txt
<!-- 单个权限 -->
<shiro:hasPermission name="user:create">
    <a href="/user/create">创建用户</a>
</shiro:hasPermission>

<!-- 多个权限（需要全部拥有） -->
<shiro:hasAllPermissions name="user:read,user:update">
    <a href="/user/edit">编辑用户</a>
</shiro:hasAllPermissions>

<!-- 多个权限之一 -->
<shiro:hasAnyPermissions name="user:delete,user:admin">
    <a href="/user/delete">删除用户</a>
</shiro:hasAnyPermissions>
```

## 完整示例

### 后台管理页面

```txt
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ taglib prefix="shiro" uri="http://shiro.apache.org/tags" %>
<!DOCTYPE html>
<html>
<head>
    <title>后台管理</title>
</head>
<body>
    <shiro:authenticated>
        <h2>欢迎, <shiro:principal /></h2>
        <a href="/logout">退出</a>
        <hr>
    </shiro:authenticated>

    <div class="sidebar">
        <!-- 所有登录用户可见 -->
        <shiro:user>
            <a href="/dashboard">仪表盘</a>
            <a href="/profile">个人资料</a>
        </shiro:user>

        <!-- 管理员可见 -->
        <shiro:hasRole name="admin">
            <div class="admin-panel">
                <h3>管理功能</h3>
                <a href="/admin/users">用户管理</a>
                <a href="/admin/roles">角色管理</a>
                <a href="/admin/permissions">权限管理</a>
            </div>
        </shiro:hasRole>

        <!-- 有创建用户权限可见 -->
        <shiro:hasPermission name="user:create">
            <a href="/user/create">创建用户</a>
        </shiro:hasPermission>

        <!-- 有删除用户权限可见 -->
        <shiro:hasPermission name="user:delete">
            <a href="/user/delete" onclick="return confirm('确认删除?')">删除用户</a>
        </shiro:hasPermission>
    </div>

    <div class="content">
        <!-- 根据不同角色显示不同内容 -->
        <shiro:hasRole name="admin">
            <p>管理员可以看到所有数据</p>
        </shiro:hasRole>
        <shiro:hasRole name="manager">
            <p>经理可以看到部门数据</p>
        </shiro:hasRole>
        <shiro:hasRole name="user">
            <p>普通用户只能看到自己的数据</p>
        </shiro:hasRole>
    </div>

    <shiro:guest>
        <p>请 <a href="/login">登录</a> 后访问更多功能</p>
    </shiro:guest>
</body>
</html>
```

## Thymeleaf 整合

### 引入依赖

```xml
<dependency>
    <groupId>com.github.theborakompanioni</groupId>
    <artifactId>shiro-thymeleaf</artifactId>
    <version>1.1.0</version>
</dependency>
```

### Spring Boot 配置

```java
@Configuration
public class ShiroThymeleafConfig {

    @Bean
    public ShiroDialect shiroDialect() {
        return new ShiroDialect();
    }
}
```

### Thymeleaf 模板

```html title="src/main/resources/templates/user/list.html"
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org"
      xmlns:shiro="http://www.pollix.at/thymeleaf/shiro">
<head>
    <title>用户列表</title>
</head>
<body>
    <!-- 欢迎信息 -->
    <div shiro:authenticated>
        <p>欢迎, <span shiro:principal /></p>
    </div>

    <!-- 匿名用户 -->
    <div shiro:guest="">
        <a href="/login">登录</a>
    </div>

    <!-- 管理员功能 -->
    <div shiro:hasRole="admin">
        <a href="/admin">管理后台</a>
        <button shiro:hasPermission="user:create" onclick="showCreateForm()">
            创建用户
        </button>
        <button shiro:hasPermission="user:delete" onclick="deleteSelected()">
            删除选中
        </button>
    </div>

    <!-- 用户列表 -->
    <table>
        <tr th:each="user : ${users}">
            <td th:text="${user.username}">用户名</td>
            <td>
                <a shiro:hasPermission="user:read"
                   th:href="@{/user/view/{id}(id=${user.id})}">
                    查看
                </a>
                <a shiro:hasPermission="user:update"
                   th:href="@{/user/edit/{id}(id=${user.id})}">
                    编辑
                </a>
                <a shiro:hasPermission="user:delete"
                   th:href="@{/user/delete/{id}(id=${user.id})}"
                   onclick="return confirm('确认删除?')">
                    删除
                </a>
            </td>
        </tr>
    </table>

    <!-- 操作日志（管理员可见） -->
    <div shiro:hasRole="admin">
        <h3>操作日志</h3>
        <ul>
            <li th:each="log : ${logs}" th:text="${log}">日志</li>
        </ul>
    </div>

    <!-- 退出按钮 -->
    <form shiro:authenticated action="/logout" method="post">
        <button type="submit">退出登录</button>
    </form>
</body>
</html>
```

## Vue/React 前端集成

如果使用前后端分离架构，前端通过 API 获取权限：

### 后端接口

```java
@RestController
@RequestMapping("/api/auth")
public class AuthController {

    @GetMapping("/info")
    public Result<AuthInfo> getAuthInfo() {
        Subject subject = SecurityUtils.getSubject();
        if (subject.isAuthenticated()) {
            return Result.success(new AuthInfo(
                subject.getPrincipal().toString(),
                subject.hasRole("admin"),
                subject.hasPermission("user:create")
            ));
        }
        return Result.success(new AuthInfo(null, false, false));
    }
}

@Data
public class AuthInfo {
    private String username;
    private boolean isAdmin;
    private boolean canCreateUser;
}
```

### 前端（Vue）

```javascript
// api/auth.js
export function getAuthInfo() {
    return request.get('/api/auth/info');
}

// components/NavBar.vue
<template>
    <nav>
        <span v-if="authInfo.username">欢迎, {{ authInfo.username }}</span>
        <a v-if="authInfo.isAdmin" href="/admin">管理后台</a>
        <button v-if="authInfo.canCreateUser" @click="showCreate">
            创建用户
        </button>
    </nav>
</template>

<script>
export default {
    data() {
        return {
            authInfo: {
                username: null,
                isAdmin: false,
                canCreateUser: false
            }
        };
    },
    async mounted() {
        const res = await getAuthInfo();
        if (res.code === 200) {
            this.authInfo = res.data;
        }
    }
};
</script>
```

## 安全建议

### 不要完全依赖前端控制

```html
<!-- 不安全：仅前端控制，可被绕过 -->
<button shiro:hasPermission="user:delete" onclick="deleteUser(1)">
    删除用户
</button>
```

**正确做法**：前端控制体验，后端控制安全。

```java
// 后端必须再次检查权限
@RequiresPermissions("user:delete")
@PostMapping("/user/delete/{id}")
public Result deleteUser(@PathVariable Long id) {
    userService.delete(id);
    return Result.success();
}
```

### 敏感操作必须后端验证

```txt
<!-- 前端：可以根据权限显示/隐藏 -->
<shiro:hasPermission name="user:export">
    <button onclick="exportData()">导出数据</button>
</shiro:hasPermission>

<!-- 后端：必须检查权限 -->
@RequiresPermissions("user:export")
@PostMapping("/user/export")
public Result exportData() {
    // 导出逻辑
}
```

## 总结

Shiro 前端整合让权限控制延伸到视图层：

```
前端标签库使用场景：
├── 菜单项：根据角色显示/隐藏
├── 按钮：根据权限显示/隐藏
├── 区块：根据角色显示不同内容
├── 提示：根据认证状态显示欢迎信息
└── 安全建议：前端控制体验，后端控制安全
```
