# Shiro 缓存：CacheManager 与权限缓存

每次请求都要查数据库查权限？缓存来解决。

Shiro 内置了两层缓存：**认证缓存**和**授权缓存**。认证通过后，用户的角色和权限信息会被缓存起来，下次权限检查直接从缓存读取，避免频繁查询数据库。

## 为什么需要缓存？

### 没有缓存的流程

```
用户请求 /api/user/list
    │
    ▼
ShiroFilter 拦截
    │
    ▼
isPermitted("user:read")
    │
    ▼
Realm.doGetAuthorizationInfo()
    │
    ▼
查询数据库: SELECT * FROM sys_user WHERE username = 'zhangsan'
    │
    ▼
查询数据库: SELECT role_name FROM sys_user_role WHERE user_id = 1
    │
    ▼
查询数据库: SELECT permission FROM sys_role_permission WHERE role_id IN (1, 2)
    │
    ▼
返回权限列表
```

每个请求都要查 3 次数据库！高并发下数据库压力巨大。

### 有缓存的流程

```
第一次请求
    │
    ▼
Realm.doGetAuthorizationInfo()
    │
    ▼
查询数据库（3次）
    │
    ▼
存入 AuthorizationCache
    │
    ▼
返回权限列表

第二次及后续请求
    │
    ▼
先查 AuthorizationCache ← 命中！直接返回
    │
    ▼
不再查询数据库
```

## CacheManager 接口

Shiro 通过 `CacheManager` 管理缓存：

```java
public interface CacheManager {
    // 获取缓存
    <K, V> Cache<K, V> getCache(String name) throws CacheException;
}
```

## Shiro 内置缓存实现

| 实现 | 存储 | 适用场景 |
|---|---|---|
| `MemoryConstrainedCacheManager` | 内存 | 开发环境 |
| `EhCacheManager` | EhCache | 单节点应用 |
| `RedisCacheManager` | Redis | 分布式应用 |
| `HazelcastCacheManager` | Hazelcast | 集群应用 |
| `MemcachedCacheManager` | Memcached | 大型分布式 |

## MemoryConstrainedCacheManager

最简单的实现，使用内存缓存：

```java
@Bean
public MemoryConstrainedCacheManager cacheManager() {
    return new MemoryConstrainedCacheManager();
}

@Bean
public UserRealm userRealm(CacheManager cacheManager,
                           HashedCredentialsMatcher matcher) {
    UserRealm realm = new UserRealm();
    realm.setCredentialsMatcher(matcher);
    realm.setCacheManager(cacheManager);
    return realm;
}
```

:::warning 开发环境使用
MemoryConstrainedCacheManager 的缓存在应用重启后会丢失，而且无法跨 JVM 共享。只适合开发调试。
:::

## EhCacheManager

EhCache 是 Shiro 早期推荐使用的缓存：

### 引入依赖

```xml
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-ehcache</artifactId>
    <version>1.12.0</version>
</dependency>
```

### ehcache.xml 配置

```xml title="src/main/resources/ehcache.xml"
<?xml version="1.0" encoding="UTF-8"?>
<ehcache xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="http://ehcache.org/ehcache.xsd">

    <!-- 认证缓存 -->
    <cache name="authenticationCache"
           maxEntriesLocalHeap="10000"
           eternal="false"
           timeToIdleSeconds="1800"
           timeToLiveSeconds="0"
           overflowToDisk="false"
           statistics="false"/>

    <!-- 授权缓存 -->
    <cache name="authorizationCache"
           maxEntriesLocalHeap="10000"
           eternal="false"
           timeToIdleSeconds="1800"
           timeToLiveSeconds="0"
           overflowToDisk="false"
           statistics="false"/>

    <!-- Session 缓存 -->
    <cache name="activeSessionsCache"
           maxEntriesLocalHeap="10000"
           eternal="false"
           timeToIdleSeconds="1800"
           timeToLiveSeconds="0"
           overflowToDisk="false"
           statistics="false"/>

    <defaultCache
            maxEntriesLocalHeap="10000"
            eternal="false"
            timeToIdleSeconds="1800"
            timeToLiveSeconds="0"
            overflowToDisk="false"
            statistics="false"/>

</ehcache>
```

### 配置

```java
@Bean
public EhCacheManager cacheManager() {
    EhCacheManager cacheManager = new EhCacheManager();
    cacheManager.setCacheManagerConfigFileLoader("classpath:ehcache.xml");
    return cacheManager;
}
```

### 配置参数说明

| 参数 | 说明 | 推荐值 |
|---|---|---|
| `maxEntriesLocalHeap` | 最大缓存数量 | 10000 |
| `eternal` | 是否永不过期 | false |
| `timeToIdleSeconds` | 空闲过期时间 | 1800（30 分钟） |
| `timeToLiveSeconds` | 存活过期时间 | 0（不过期） |
| `overflowToDisk` | 超出内存时写入磁盘 | false |

## RedisCacheManager

生产环境推荐使用 Redis：

### 引入依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-redis</artifactId>
    <version>4.5.3</version>
</dependency>
```

### 配置

```java
@Bean
public RedisManager redisManager() {
    RedisManager manager = new RedisManager();
    manager.setHost("127.0.0.1:6379");
    manager.setPassword("123456");
    manager.setDatabase(0);
    manager.setTimeout(1800000);
    return manager;
}

@Bean
public CacheManager cacheManager(RedisManager redisManager) {
    RedisCacheManager cacheManager = new RedisCacheManager();
    cacheManager.setRedisManager(redisManager);
    // 缓存前缀
    cacheManager.setKeyPrefix("shiro:cache:");
    // 默认过期时间（秒）
    cacheManager.setExpire(1800);
    return cacheManager;
}
```

### 自定义 RedisCacheManager

如果 shiro-redis 不满足需求：

```java
@Component
public class ShiroRedisCacheManager implements CacheManager {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    private static final String KEY_PREFIX = "shiro:cache:";
    private static final long DEFAULT_EXPIRE = 1800;

    @Override
    public <K, V> Cache<K, V> getCache(String name) throws CacheException {
        return new ShiroRedisCache<>(redisTemplate, KEY_PREFIX + name, DEFAULT_EXPIRE);
    }
}

public class ShiroRedisCache<K, V> implements Cache<K, V> {

    private String name;
    private long expire;
    private RedisTemplate<String, Object> redisTemplate;

    public ShiroRedisCache(RedisTemplate<String, Object> redisTemplate,
                          String name, long expire) {
        this.redisTemplate = redisTemplate;
        this.name = name;
        this.expire = expire;
    }

    @Override
    public V get(K key) throws CacheException {
        String redisKey = name + ":" + key;
        return (V) redisTemplate.opsForValue().get(redisKey);
    }

    @Override
    public V put(K key, V value) throws CacheException {
        String redisKey = name + ":" + key;
        redisTemplate.opsForValue().set(redisKey, value, expire, TimeUnit.SECONDS);
        return value;
    }

    @Override
    public V remove(K key) throws CacheException {
        String redisKey = name + ":" + key;
        redisTemplate.delete(redisKey);
        return null;
    }

    @Override
    public void clear() throws CacheException {
        Set<String> keys = redisTemplate.keys(name + ":*");
        if (keys != null && !keys.isEmpty()) {
            redisTemplate.delete(keys);
        }
    }

    @Override
    public int size() {
        Set<String> keys = redisTemplate.keys(name + ":*");
        return keys != null ? keys.size() : 0;
    }

    @Override
    public Set<K> keys() {
        Set<String> keys = redisTemplate.keys(name + ":*");
        return (Set<K>) keys.stream()
            .map(k -> (K) k.substring(k.lastIndexOf(":") + 1))
            .collect(Collectors.toSet());
    }

    @Override
    public Collection<V> values() {
        Set<K> keys = keys();
        return keys.stream()
            .map(this::get)
            .collect(Collectors.toList());
    }
}
```

## Realm 启用缓存

### 完整配置

```java
@Bean
public UserRealm userRealm(CacheManager cacheManager,
                           HashedCredentialsMatcher matcher) {
    UserRealm realm = new UserRealm();
    realm.setCredentialsMatcher(matcher);

    // 启用缓存
    realm.setCachingEnabled(true);

    // 启用认证缓存
    realm.setAuthenticationCachingEnabled(true);
    realm.setAuthenticationCacheName("authenticationCache");

    // 启用授权缓存
    realm.setAuthorizationCachingEnabled(true);
    realm.setAuthorizationCacheName("authorizationCache");

    // 设置缓存管理器
    realm.setCacheManager(cacheManager);

    return realm;
}
```

### 缓存命名

| 缓存名 | 用途 | key |
|---|---|---|
| `authenticationCache` | 认证信息缓存 | principal |
| `authorizationCache` | 授权信息缓存 | principal |

## 缓存失效时机

### 登录时清除

```java
// 登录时，认证信息会更新，Shiro 自动清除旧缓存
subject.login(token);
```

### 退出时清除

```java
// 退出时，Shiro 自动清除认证和授权缓存
subject.logout();
```

### 手动清除

```java
@Service
public class RoleServiceImpl implements RoleService {

    @Autowired
    private Realm realm;

    @Override
    @Transactional
    public void updateUserRoles(String username, Set<String> newRoles) {
        // 1. 更新数据库
        userRoleMapper.updateRoles(username, newRoles);

        // 2. 清除缓存
        Cache<Object, AuthorizationInfo> cache =
            realm.getAuthorizationCache();
        if (cache != null) {
            // 清除该用户的授权缓存
            cache.remove(username);
        }
    }
}
```

### 批量清除

```java
// 清除所有授权缓存
Cache<Object, AuthorizationInfo> cache = realm.getAuthorizationCache();
cache.clear();

// 清除所有认证缓存
Cache<Object, AuthenticationInfo> authCache = realm.getAuthenticationCache();
if (authCache != null) {
    authCache.clear();
}
```

## 缓存配置最佳实践

### 缓存时间设置

```yaml
# application.yml
shiro:
  cache:
    # 授权缓存过期时间（秒）
    expire: 1800  # 30 分钟
```

### 缓存容量设置

| 场景 | 用户量 | 缓存大小 |
|---|---|---|
| 小型系统 | `<` 1000 | 10000 |
| 中型系统 | 1000-10000 | 50000 |
| 大型系统 | `>` 10000 | 使用分布式缓存 |

### 缓存预热

```java
@Component
public class CacheWarmUp implements ApplicationRunner {

    @Autowired
    private Realm realm;

    @Override
    public void run(ApplicationArguments args) {
        log.info("开始预热 Shiro 缓存...");

        // 预热活跃用户的权限缓存
        List<String> activeUsers = userMapper.selectActiveUsers();
        for (String username : activeUsers) {
            Subject subject = new Subject.Builder()
                .principals(new SimplePrincipalCollection(username, "userRealm"))
                .buildSubject();

            // 触发授权信息加载
            subject.hasRole("any");
        }

        log.info("Shiro 缓存预热完成，共 {} 个用户", activeUsers.size());
    }
}
```

## 面试高频问题

**问：Shiro 的缓存放在哪？**

可以放在内存（MemoryConstrainedCacheManager）、EhCache、Redis、Hazelcast 等。生产环境推荐 Redis。

**问：用户权限变更后如何清除缓存？**

调用 `realm.getAuthorizationCache().remove(username)` 清除该用户的授权缓存。或者在修改权限时主动清除。

**问：缓存会导致权限更新不及时吗？**

是的，这是缓存的通病。解决方案：

1. 设置合理的缓存过期时间
2. 权限变更时主动清除缓存
3. 敏感操作不依赖缓存

**问：Redis 缓存的序列化方式？**

默认使用 Java 序列化。可以配置使用 JSON 序列化：

```java
RedisTemplate<String, Object> redisTemplate = new RedisTemplate<>();
redisTemplate.setKeySerializer(new StringRedisSerializer());
redisTemplate.setValueSerializer(new GenericJackson2JsonRedisSerializer());
```

## 总结

Shiro 缓存是性能优化的关键：

```
缓存配置要点：
├── 认证缓存：存储 AuthenticationInfo
├── 授权缓存：存储 AuthorizationInfo
├── 缓存时间：建议 30 分钟
├── 缓存清除：权限变更时主动清除
└── 分布式：使用 Redis 实现多节点共享
```

合理配置缓存，可以将数据库查询次数降低 90% 以上。
