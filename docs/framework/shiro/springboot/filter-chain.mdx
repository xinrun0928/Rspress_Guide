# ShiroFilterFactoryBean：Spring Boot 集成 Shiro 的核心

如果说 SecurityManager 是 Shiro 的大脑，那 ShiroFilterFactoryBean 就是 Shiro 的四肢——它负责把 Shiro 连接到 Web 请求的处理链中。

Spring Boot 2.x 之后，Shiro 不再被自动集成，你需要手动配置 FilterChain。理解 ShiroFilterFactoryBean 的工作原理，是掌握 Shiro Web 集成的关键。

## ShiroFilterFactoryBean 的作用

`ShiroFilterFactoryBean` 做了三件事：

1. **创建 ShiroFilter**：一个 Servlet Filter，负责拦截所有请求
2. **配置过滤器链**：定义哪些 URL 需要什么过滤器
3. **注入 SecurityManager**：让 ShiroFilter 能够访问安全管理器

```java
@Bean
public ShiroFilterFactoryBean shiroFilterFactoryBean(SecurityManager securityManager) {
    ShiroFilterFactoryBean factory = new ShiroFilterFactoryBean();
    factory.setSecurityManager(securityManager);
    factory.setFilterChainDefinitionMap(filterChainMap);
    return factory;
}
```

## 过滤器链配置

### Map 方式配置

```java
@Bean
public ShiroFilterFactoryBean shiroFilterFactoryBean(SecurityManager securityManager) {
    ShiroFilterFactoryBean factory = new ShiroFilterFactoryBean();
    factory.setSecurityManager(securityManager);

    // 过滤器链定义
    Map<String, String> filterChain = new LinkedHashMap<>();

    // 匿名访问
    filterChain.put("/login", "anon");
    filterChain.put("/register", "anon");
    filterChain.put("/static/**", "anon");
    filterChain.put("/api/public/**", "anon");

    // 登出
    filterChain.put("/logout", "logout");

    // 需要角色
    filterChain.put("/admin/**", "roles[admin]");

    // 需要权限
    filterChain.put("/user/**", "perms[user:read]");

    // 其他需要认证
    filterChain.put("/**", "authc");

    factory.setFilterChainDefinitionMap(filterChain);
    return factory;
}
```

### ShiroFilterChainDefinition 方式

```java
@Bean
public ShiroFilterChainDefinition shiroFilterChainDefinition() {
    DefaultShiroFilterChainDefinition chainDefinition =
        new DefaultShiroFilterChainDefinition();

    chainDefinition.addPathDefinition("/login", "anon");
    chainDefinition.addPathDefinition("/logout", "logout");
    chainDefinition.addPathDefinition("/static/**", "anon");
    chainDefinition.addPathDefinition("/admin/**", "roles[admin]");
    chainDefinition.addPathDefinition("/**", "authc");

    return chainDefinition;
}

@Bean
public ShiroFilterFactoryBean shiroFilterFactoryBean(SecurityManager securityManager,
                                                     ShiroFilterChainDefinition chainDefinition) {
    ShiroFilterFactoryBean factory = new ShiroFilterFactoryBean();
    factory.setSecurityManager(securityManager);
    factory.setFilterChainDefinitionMap(chainDefinition.getFilterChainMap());
    return factory;
}
```

## ShiroFilter 执行流程

### 过滤器链执行

当一个请求进入时，ShiroFilter 会按顺序检查过滤器链：

```
请求: GET /admin/user/list
         │
         ▼
┌─────────────────────────────────────────────────┐
│ ShiroFilter 拦截请求                             │
├─────────────────────────────────────────────────┤
│ 按顺序检查过滤器链                               │
│                                                 │
│ /logout     → LogoutFilter      → 不匹配        │
│ /login      → AnonymousFilter    → 不匹配        │
│ /static/**  → AnonymousFilter    → 不匹配        │
│ /admin/**   → RolesAuthorizationFilter → 匹配!  │
│                                                 │
│ 执行 RolesAuthorizationFilter                    │
│ 检查用户是否有 admin 角色                        │
│                                                 │
│ ├── 有角色 → 继续下一个过滤器 → 最终到达 Controller│
│ └── 无角色 → 重定向到 unauthorizedUrl            │
└─────────────────────────────────────────────────┘
```

### OncePerRequestFilter 基类

所有 Shiro 过滤器都继承自 `OncePerRequestFilter`，保证每个请求只执行一次：

```java
// org.apache.shiro.web.filter.OncePerRequestFilter
public abstract class OncePerRequestFilter implements Filter {

    public final void doFilter(ServletRequest request,
                               ServletResponse response,
                               FilterChain chain) {
        String name = getName();
        String alreadyFilteredAttribute = name + ".FILTERED";
        request.setAttribute(alreadyFilteredAttribute, Boolean.TRUE);

        try {
            doFilterInternal(request, response, chain);
        } finally {
            request.removeAttribute(alreadyFilteredAttribute);
        }
    }

    // 子类实现这个方法
    protected abstract void doFilterInternal(ServletRequest request,
                                           ServletResponse response,
                                           FilterChain chain);
}
```

## 内置过滤器详解

### 认证相关过滤器

#### AnonymousFilter（anon）

不需要认证，匿名访问：

```java
filterChain.addPathDefinition("/login", "anon");
filterChain.addPathDefinition("/static/**", "anon");
```

#### FormAuthenticationFilter（authc）

表单认证过滤器。拦截未认证请求，重定向到登录页：

```java
// 默认配置
filterChain.addPathDefinition("/**", "authc");

// 自定义登录 URL
factory.setLoginUrl("/login");

// 自定义成功跳转 URL
factory.setSuccessUrl("/index");

// 自定义未授权跳转 URL
factory.setUnauthorizedUrl("/unauthorized");
```

#### BasicHttpAuthenticationFilter（authcBasic）

HTTP Basic 认证：

```java
filterChain.addPathDefinition("/api/**", "authcBasic");
```

浏览器会弹出用户名密码对话框：

```
Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=
```

### 授权相关过滤器

#### RolesAuthorizationFilter（roles）

需要特定角色：

```java
// 单个角色
filterChain.addPathDefinition("/admin/**", "roles[admin]");

// 多个角色（默认 AND，所有角色都需要）
filterChain.addPathDefinition("/manager/**", "roles[admin,manager]");

// 多个角色（OR，满足一个即可）
filterChain.addPathDefinition("/user/**", "authc, roles[admin,user]");
```

#### PermissionsAuthorizationFilter（perms）

需要特定权限：

```java
// 单个权限
filterChain.addPathDefinition("/user/create", "perms[user:create]");

// 多个权限（默认 AND）
filterChain.addPathDefinition("/user/**", "perms[user:read,user:write]");

// 多个权限（OR）
filterChain.addPathDefinition("/order/**", "authc, perms[order:read,order:admin]");
```

#### HttpMethodPermissionFilter（rest）

REST 风格权限控制：

```java
filterChain.addPathDefinition("/user/**", "rest[user]");
```

自动根据 HTTP 方法映射权限：

| HTTP 方法 | 映射权限 |
|---|---|
| GET | `user:read` |
| POST | `user:create` |
| PUT | `user:update` |
| DELETE | `user:delete` |

### 会话相关过滤器

#### UserFilter（user）

已认证或 RememberMe 用户：

```java
filterChain.addPathDefinition("/profile/**", "user");
```

和 `authc` 的区别：

| 过滤器 | 已认证 | RememberMe | 匿名 |
|---|---|---|---|
| `authc` | ✓ | ✗ | ✗ |
| `user` | ✓ | ✓ | ✗ |

#### NoSessionCreationFilter（noSessionCreation）

禁止创建新 Session：

```java
filterChain.addPathDefinition("/api/**", "noSessionCreation, authc");
```

### 其他过滤器

#### LogoutFilter（logout）

登出并清除 Session：

```java
filterChain.addPathDefinition("/logout", "logout");
```

#### SslFilter（ssl）

强制 HTTPS：

```java
filterChain.addPathDefinition("/secure/**", "ssl");
```

## 自定义过滤器

### 场景一：限制并发登录

```java
public class ConcurrentLoginFilter extends AccessControlFilter {

    private int maxSession = 1;  // 最多登录次数
    private boolean excludeCurrentAccount = false;  // 是否排除当前账户

    @Override
    protected boolean isAccessAllowed(ServletRequest request,
                                     ServletResponse response,
                                     Object mappedValue) {
        Subject subject = getSubject(request, response);
        Session session = subject.getSession();
        SessionDAO sessionDAO = sessionDAO();

        // 统计当前用户的所有 Session
        Collection<Session> activeSessions = sessionDAO.getActiveSessions();
        List<Session> userSessions = activeSessions.stream()
            .filter(s -> {
                PrincipalCollection principals =
                    (PrincipalCollection) s.getAttribute(DefaultSubjectContext.PRINCIPALS_SESSION_KEY);
                return principals != null && principals.getPrimaryPrincipal()
                    .equals(subject.getPrincipal());
            })
            .collect(Collectors.toList());

        if (userSessions.size() >= maxSession) {
            // 踢掉之前的 Session
            for (Session oldSession : userSessions.subList(0, userSessions.size() - 1)) {
                sessionDAO.delete(oldSession);
            }
        }

        return true;
    }

    @Override
    protected boolean onAccessDenied(ServletRequest request,
                                    ServletResponse response) {
        response.setContentType("application/json;charset=UTF-8");
        response.getWriter().write("{\"code\": 401, \"message\": \"已在其他设备登录\"}");
        return false;
    }
}
```

### 场景二：IP 黑名单

```java
public class IpBlacklistFilter extends AccessControlFilter {

    private Set<String> blacklist = new HashSet<>();

    public void setBlacklist(Set<String> blacklist) {
        this.blacklist = blacklist;
    }

    @Override
    protected boolean isAccessAllowed(ServletRequest request,
                                     ServletResponse response,
                                     Object mappedValue) {
        String ip = getClientIP(request);
        return !blacklist.contains(ip);
    }

    @Override
    protected boolean onAccessDenied(ServletRequest request,
                                    ServletResponse response) {
        response.setContentType("application/json;charset=UTF-8");
        response.getWriter().write("{\"code\": 403, \"message\": \"IP 被禁止访问\"}");
        return false;
    }

    private String getClientIP(ServletRequest request) {
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        String ip = httpRequest.getHeader("X-Forwarded-For");
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = httpRequest.getHeader("Proxy-Client-IP");
        }
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = httpRequest.getHeader("WL-Proxy-Client-IP");
        }
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = httpRequest.getRemoteAddr();
        }
        return ip;
    }
}
```

### 注册自定义过滤器

```java
@Bean
public ShiroFilterFactoryBean shiroFilterFactoryBean(SecurityManager securityManager,
                                                     ShiroFilterChainDefinition chainDefinition) {
    ShiroFilterFactoryBean factory = new ShiroFilterFactoryBean();
    factory.setSecurityManager(securityManager);
    factory.setFilterChainDefinitionMap(chainDefinition.getFilterChainMap());

    // 注册自定义过滤器
    Map<String, Filter> filters = new HashMap<>();
    filters.put("concurrentLogin", new ConcurrentLoginFilter());
    filters.put("ipBlacklist", new IpBlacklistFilter());
    factory.setFilters(filters);

    return factory;
}
```

## Filter 执行顺序

Shiro 过滤器的执行顺序很重要。错误的顺序可能导致过滤器失效：

```
正确顺序：
1. logout    → 先处理登出
2. anon       → 处理公开路径
3. ssl        → 处理 HTTPS
4. user       → 处理 RememberMe
5. authc      → 处理认证
6. authcBasic → 处理 Basic 认证
7. perms      → 处理权限
8. roles      → 处理角色
9. noSessionCreation → 禁止创建 Session

错误示例：
把 authc 放在 anon 前面，会导致 /login 也需要认证
```

## 面试高频问题

**问：Shiro 过滤器链和 Spring Security 过滤器链有什么区别？**

Spring Security 使用 `SecurityFilterChain` 配置，每个过滤器职责单一。Shiro 使用 `ShiroFilterFactoryBean` 配置，过滤器链通过 Map 定义，更直观但顺序依赖配置。

**问：如何自定义过滤器？**

继承 `AccessControlFilter` 或 `OncePerRequestFilter`，重写 `isAccessAllowed` 和 `onAccessDenied` 方法，然后注册到 `ShiroFilterFactoryBean`。

**问：Shiro 过滤器链的匹配规则是什么？**

第一个匹配的过滤器生效。使用前缀匹配（`*`），如 `/admin/**` 匹配 `/admin/user/list`。

## 总结

ShiroFilterFactoryBean 是 Shiro Web 集成的核心：

```
ShiroFilterFactoryBean 配置要点：
├── SecurityManager：注入安全管理器
├── FilterChain：配置 URL → 过滤器映射
├── LoginUrl：登录页面
├── SuccessUrl：登录成功页面
├── UnauthorizedUrl：未授权页面
└── 自定义过滤器：注册自定义过滤器
```

过滤器链是 Shiro 安全控制的第一道防线，合理配置可以做到**配置即安全**。
