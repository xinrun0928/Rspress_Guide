# Shiro 注解：@RequiresAuthentication 的方法级安全

过滤器链做的是 URL 级别的控制，但有些权限需要精确到方法级别——比如"只有创建者才能修改自己的订单"。

Shiro 提供了方法级的安全注解，配合 AOP 切面，可以在方法执行前进行权限校验。这种方式比过滤器链更灵活，也更符合业务逻辑。

## 核心注解一览

| 注解 | 说明 |
|---|---|
| `@RequiresAuthentication` | 需要已认证 |
| `@RequiresUser` | 需要已认证或 RememberMe |
| `@RequiresGuest` | 需要匿名访问 |
| `@RequiresRoles(roles)` | 需要特定角色 |
| `@RequiresPermissions(permissions)` | 需要特定权限 |
| `@RequiresHttpMethod(method)` | 需要特定 HTTP 方法 |

## 基本用法

### 认证注解

```java
@Controller
public class UserController {

    // 需要登录才能访问
    @RequiresAuthentication
    @GetMapping("/profile")
    public String profile(Model model) {
        Subject subject = SecurityUtils.getSubject();
        model.addAttribute("username", subject.getPrincipal());
        return "profile";
    }
}
```

### 角色注解

```java
@Service
public class UserService {

    // 需要 admin 角色
    @RequiresRoles("admin")
    public void deleteUser(Long userId) {
        userMapper.deleteById(userId);
    }

    // 需要 manager 或 admin 角色（OR）
    @RequiresRoles(value = {"manager", "admin"}, logical = Logical.OR)
    public void exportReport() {
        // 导出报表
    }
}
```

### 权限注解

```java
@Service
public class OrderService {

    // 需要 order:create 权限
    @RequiresPermissions("order:create")
    public Order createOrder(OrderDTO dto) {
        // 创建订单
    }

    // 需要 order:read 和 order:update 权限（AND）
    @RequiresPermissions({"order:read", "order:update"})
    public Order updateOrder(Long orderId, OrderDTO dto) {
        // 更新订单
    }

    // 需要 order:delete 或 order:admin 权限（OR）
    @RequiresPermissions(value = {"order:delete", "order:admin"}, logical = Logical.OR)
    public void deleteOrder(Long orderId) {
        // 删除订单
    }
}
```

## AOP 拦截机制

Shiro 的注解通过 AOP 切面生效。核心组件是 `AuthorizationAttributeSourceAdvisor`：

```java
// Shiro 通过 Spring AOP 拦截带注解的方法
@Aspect
@Component
public class AuthorizationAttributeSourceAdvisor extends StaticMethodMatcherPointcutAdvisor {

    private static final AttributeCheck check = new AuthorizingAnnotationMethodInterceptor();

    @Override
    public boolean matches(Method method, Class<?> targetClass) {
        return check.supports(targetClass) && hasAnnotation(method, targetClass);
    }

    @Override
    public Advice getAdvice() {
        return new AopAllianceAnnotationsAuthorizingMethodInterceptor();
    }
}
```

### 拦截链

```
方法调用
    │
    ▼
AOP 代理拦截
    │
    ▼
AuthorizationAttributeSourceAdvisor
    │
    ▼
AuthorizingAnnotationMethodInterceptor
    │
    ▼
MethodSecurityInterceptor
    │
    ├── @RequiresAuthentication → subject.isAuthenticated()
    ├── @RequiresUser → subject.isAuthenticated() || subject.isRemembered()
    ├── @RequiresRoles → subject.hasRole()
    └── @RequiresPermissions → subject.isPermitted()
    │
    ├── 通过 → 执行目标方法
    └── 拒绝 → 抛出 UnauthorizedException
```

## 配置启用注解

### Spring Boot 配置

```java
@Configuration
public class ShiroConfig {

    /**
     * 启用 Shiro 注解
     */
    @Bean
    public AuthorizationAttributeSourceAdvisor authorizationAttributeSourceAdvisor(
            SecurityManager securityManager) {
        AuthorizationAttributeSourceAdvisor advisor =
            new AuthorizationAttributeSourceAdvisor();
        advisor.setSecurityManager(securityManager);
        return advisor;
    }

    /**
     * Spring AOP 自动代理
     */
    @Bean
    @Role(BeanDefinition.ROLE_INFRASTRUCTURE)
    public DefaultAdvisorAutoProxyCreator defaultAdvisorAutoProxyCreator() {
        DefaultAdvisorAutoProxyCreator creator = new DefaultAdvisorAutoProxyCreator();
        creator.setProxyTargetClass(true);  // 使用 CGLIB 代理
        return creator;
    }
}
```

### Spring Boot 3.x 配置

Spring Boot 3.x 需要额外配置：

```xml
<dependency>
    <groupId>org.aspectj</groupId>
    <artifactId>aspectjweaver</artifactId>
</dependency>
```

## 注解失效的常见原因

### 原因一：未配置 AOP 代理

```java
// 错误：没有开启 AOP 自动代理
@Configuration
public class ShiroConfig {
    // 只有 Bean，缺少自动代理配置
}

// 正确：添加自动代理
@Bean
public DefaultAdvisorAutoProxyCreator defaultAdvisorAutoProxyCreator() {
    DefaultAdvisorAutoProxyCreator creator = new DefaultAdvisorAutoProxyCreator();
    creator.setProxyTargetClass(true);
    return creator;
}
```

### 原因二：同类内部调用

同类方法间的调用不走代理：

```java
@Service
public class UserService {

    @RequiresRoles("admin")
    public void deleteUser(Long userId) {
        // deleteUser 调用 deleteUserInternal，不走代理
        deleteUserInternal(userId);
    }

    // 这个方法有 @RequiresRoles，但同类内部调用不走代理
    private void deleteUserInternal(Long userId) {
        userMapper.deleteById(userId);
    }
}
```

解决方案：

1. 将注解方法抽取到另一个类
2. 使用 `AopContext.currentProxy()` 获取代理对象

```java
@Service
public class UserService {

    @Autowired
    private UserService self;  // 注入自身代理

    public void deleteUser(Long userId) {
        // 调用代理方法
        self.deleteUserInternal(userId);
    }

    @RequiresRoles("admin")
    public void deleteUserInternal(Long userId) {
        userMapper.deleteById(userId);
    }
}
```

### 原因三：方法在另一个 Service 中

如果注解方法在另一个 Service 中被调用：

```java
@Service
public class OrderService {

    @Autowired
    private UserService userService;

    public void processOrder() {
        // 调用 UserService 的方法，会走代理
        userService.deleteUser(1L);
    }
}

@Service
public class UserService {

    @RequiresRoles("admin")
    public void deleteUser(Long userId) {
        // 会被正确拦截
    }
}
```

## 异常处理

### 全局异常处理

```java
@RestControllerAdvice
public class ShiroExceptionHandler {

    @ExceptionHandler(UnauthorizedException.class)
    public Result handleUnauthorized(UnauthorizedException e) {
        return Result.fail(403, "没有权限访问该资源");
    }

    @ExceptionHandler(UnauthenticatedException.class)
    public Result handleUnauthenticated(UnauthenticatedException e) {
        return Result.fail(401, "请先登录");
    }

    @ExceptionHandler(AuthenticationException.class)
    public Result handleAuthentication(AuthenticationException e) {
        return Result.fail(401, "认证失败：" + e.getMessage());
    }
}
```

### ShiroFilter 处理

如果注解不生效，请求会到达 Controller 之前就被 Shiro 过滤器拦截：

```java
@Configuration
public class ShiroConfig {

    @Bean
    public ShiroFilterFactoryBean shiroFilterFactoryBean(...) {
        ShiroFilterFactoryBean factory = new ShiroFilterFactoryBean();
        // ...
        // 设置登录 URL
        factory.setLoginUrl("/login");
        // 设置未授权 URL
        factory.setUnauthorizedUrl("/unauthorized");
        return factory;
    }
}
```

## 与过滤器链的选择

| 场景 | 推荐方式 | 说明 |
|---|---|---|
| URL 级别控制 | 过滤器链 | 配置简单，性能好 |
| 业务逻辑相关 | 注解 | 更灵活，符合业务 |
| 动态权限 | 注解 + 数据库 | 权限可配置 |
| 简单角色判断 | 过滤器链 | 一行配置搞定 |
| 复杂权限逻辑 | 注解 | 可以在注解方法内写逻辑 |

## 面试高频问题

**问：Shiro 注解为什么不生效？**

常见原因：

1. 未配置 `AuthorizationAttributeSourceAdvisor`
2. 未配置 `DefaultAdvisorAutoProxyCreator`
3. 同类内部调用不走代理
4. Spring Boot 3.x 缺少 `aspectjweaver` 依赖

**问：@RequiresAuthentication 和 @RequiresUser 的区别？**

`@RequiresAuthentication` 要求 `subject.isAuthenticated() = true`，`@RequiresUser` 允许 `isAuthenticated()` 或 `isRemembered()` 都通过。

**问：如何实现动态权限（权限从数据库加载）？**

自定义 Realm 的 `doGetAuthorizationInfo` 方法从数据库加载权限，配合 Shiro 缓存，可以实现动态权限控制。

## 总结

Shiro 注解是方法级安全控制的利器：

```
注解使用要点：
├── @RequiresRoles：角色检查
├── @RequiresPermissions：权限检查
├── @RequiresAuthentication：认证检查
├── @RequiresUser：认证或 RememberMe
├── logical = Logical.OR：权限或逻辑
└── 注意 AOP 代理问题
```

配合过滤器链，可以实现**配置即安全 + 注解更灵活**的完整安全体系。
