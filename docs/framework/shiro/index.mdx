# Shiro 安全框架

想象一下：你接手了一个新项目，需要给后台管理系统加上登录和权限功能。

技术选型时，团队里有人推荐 Spring Security，说功能强大；有人推荐 Shiro，说简单易用。

你该怎么选？

Apache Shiro 是 Apache 基金会旗下的顶级项目，以「简单、安全、灵活」著称。相比 Spring Security，Shiro 的学习曲线更平缓，配置更简洁，特别适合中小型项目和传统 SSM（Spring + Spring MVC + MyBatis）项目。

## Shiro 是什么？

Apache Shiro（前身是 JSecurity）是 Java 世界的安全框架，用于：

- **身份认证（Authentication）**：用户登录验证
- **访问授权（Authorization）**：用户权限管理
- **会话管理（Session Management）**：独立于容器的 Session
- **加密服务（Cryptography）**：密码加密、数据加密
- **缓存管理（Caching）**：认证授权结果缓存

Shiro 的设计哲学是：**简单、强大、灵活**。

## 为什么要学习 Shiro？

### 优势一：学习曲线平缓

五分钟理解核心概念，十分钟跑通登录流程。不像 Spring Security 那样有一堆配置类和注解等着你。

```java
// Shiro 的最小化配置
IniRealm realm = new IniRealm("classpath:shiro.ini");
DefaultSecurityManager sm = new DefaultSecurityManager(realm);
SecurityUtils.setSecurityManager(sm);
Subject currentUser = SecurityUtils.getSubject();
currentUser.login(new UsernamePasswordToken("zhangsan", "password"));
```

### 优势二：无需 Spring

Shiro 独立于任何框架，可以用在 Servlet、Spring、Spring Boot、Guice 等任何 Java 环境中。

```

│       └────────────┴────────────┴────────────┘           │

```

### 优势三：功能完整

虽然简洁，但 Shiro 的功能覆盖了 Web 安全的大部分场景：

- URL 级别的过滤器链
- 方法级别的注解权限
- Session 管理（支持 Redis 分布式存储）
- RememberMe 功能
- 密码加密（支持多种算法和盐值）
- 缓存支持（EhCache、Redis）

## 与 Spring Security 的关系

很多人会问：既然 Spring Security 功能更强大，为什么还要学 Shiro？

**答案**：两者的定位不同。

| 维度 | Shiro | Spring Security |
|---|---|---|
| 定位 | 通用安全框架 | Spring 生态专属 |
| 学习曲线 | 低 | 高 |
| 适用场景 | 中小型项目、非 Spring 项目 | Spring Boot 项目 |
| 功能丰富度 | 核心功能 | 企业级全覆盖 |

Shiro 适合快速交付、团队经验不足、或非 Spring 环境的项目。Spring Security 适合 Spring Boot 项目、需要 OAuth2 等企业级功能的场景。

**两者并不冲突**。理解了 Shiro 的核心概念，再学习 Spring Security 会更轻松。

## 总结

Apache Shiro 是一个轻量级、功能完整、易于上手的安全框架：

- **核心组件**：Subject（门面）、SecurityManager（心脏）、Realm（数据源）
- **认证授权**：完整的认证流程和授权机制
- **Spring Boot 集成**：简洁的配置，快速上手
- **分布式支持**：Session 和缓存都可以存储到 Redis
- **面试重点**：认证流程、过滤器链、Realm 定制
