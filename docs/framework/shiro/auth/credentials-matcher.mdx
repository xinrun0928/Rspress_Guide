# HashedCredentialsMatcher：Shiro 的密码加密与盐值

密码怎么存才安全？

这是每一个安全系统都必须回答的问题。直接存明文是找死——数据库泄露等于密码泄露。存 MD5 也不够——彩虹表可以在秒级破解 99% 的常见密码。

Shiro 提供了 `HashedCredentialsMatcher`，通过**哈希算法 + 盐值 + 迭代次数**三重保护，让密码泄露的代价降到最低。

## 为什么不能只存 MD5

MD5 破解轻而易举：

```
彩虹表攻击：
MD5("123456") = 5f4dcc3b5aa765d61d8327deb882cf99
彩虹表存储：123456 → 5f4dcc3b5aa765d61d8327deb882cf99

攻击者只需要：
1. 拿到数据库
2. 查彩虹表
3. 几秒钟后，所有"123456"都暴露了
```

:::warning 血的教训
2013 年，Adobe 泄露了 1.5 亿用户密码，明文存储或简单加密，结果：用户密码、密码提示、甚至生日全部曝光。
:::

## 密码安全的正确姿势

### 三要素

| 要素 | 作用 | 例子 |
|---|---|---|
| 哈希算法 | 不可逆加密，防明文泄露 | SHA-256 |
| 盐值（Salt） | 防止彩虹表攻击 | `randomSalt_123` |
| 迭代次数 | 增加计算成本，防暴力破解 | 1024 次 |

### 加密过程

```
明文密码: "123456"
    │
    │ 添加盐值
    ▼
"123456" + "a3f8c9d2e1b4" = "123456a3f8c9d2e1b4"
    │
    │ SHA-256 哈希
    ▼
结果1: "8d969eef6ecad3c29a3a629280e686cf0c3f5d5a86aff3ca12020c923adc6c92..."
    │
    │ 再哈希（迭代 1023 次）
    ▼
最终存储: "f7c3bc1d808e47332a1471c..."
```

## HashedCredentialsMatcher 配置

### 基础配置

```java
HashedCredentialsMatcher matcher = new HashedCredentialsMatcher();
matcher.setHashAlgorithmName("SHA-256");     // 哈希算法
matcher.setHashIterations(1024);             // 迭代次数
matcher.setStoredCredentialsHexEncoded(false); // 存储格式：false=Base64, true=十六进制
realm.setCredentialsMatcher(matcher);
```

### 算法选择

| 算法 | 安全性 | 性能 | 推荐场景 |
|---|---|---|---|
| MD5 | 低 | 快 | 不推荐 |
| SHA-1 | 中低 | 快 | 不推荐 |
| SHA-256 | 中 | 中 | 推荐 |
| SHA-512 | 高 | 慢 | 高安全场景 |
| bcrypt | 高 | 慢 | 最高安全场景 |

:::tip 推荐
一般项目用 SHA-256 即可。如果对安全性要求极高，可以使用 bcrypt（需要引入 jbcrypt 库）。
:::

## 盐值的生成和使用

### 生成随机盐值

```java
public class SaltUtil {
    private static final SecureRandom RANDOM = new SecureRandom();

    /**
     * 生成 16 字节的随机盐值
     */
    public static String generateSalt() {
        byte[] salt = new byte[16];
        RANDOM.nextBytes(salt);
        return Base64.getEncoder().encodeToString(salt);
    }
}
```

### 用户注册时加密密码

```java
@Service
public class UserServiceImpl implements UserService {

    @Autowired
    private PasswordEncoder passwordEncoder;

    @Override
    @Transactional
    public void register(User user) {
        // 1. 生成盐值
        String salt = SaltUtil.generateSalt();

        // 2. 用盐值加密密码
        String hashedPassword = passwordEncoder.encode(user.getPassword(), salt);

        // 3. 保存到数据库
        user.setSalt(salt);
        user.setPassword(hashedPassword);
        userMapper.insert(user);
    }
}
```

### 手动实现 SHA-256 加密

```java
public class SHA256PasswordEncoder {

    private static final String ALGORITHM = "SHA-256";
    private static final int ITERATIONS = 1024;

    public String encode(CharSequence rawPassword, String salt) {
        try {
            // 1. 将盐值和密码组合
            byte[] saltedValue = merge(salt, rawPassword);

            // 2. 迭代哈希
            MessageDigest digest = MessageDigest.getInstance(ALGORITHM);
            byte[] hashed = digest.digest(saltedValue);

            for (int i = 1; i < ITERATIONS; i++) {
                digest.reset();
                hashed = digest.digest(hashed);
            }

            // 3. 返回 Base64 编码
            return Base64.getEncoder().encodeToString(hashed);

        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException("不支持的哈希算法", e);
        }
    }

    private byte[] merge(String salt, CharSequence password) {
        byte[] saltBytes = salt.getBytes(StandardCharsets.UTF_8);
        byte[] passwordBytes = password.toString().getBytes(StandardCharsets.UTF_8);
        byte[] merged = new byte[saltBytes.length + passwordBytes.length];
        System.arraycopy(saltBytes, 0, merged, 0, saltBytes.length);
        System.arraycopy(passwordBytes, 0, merged, saltBytes.length, passwordBytes.length);
        return merged;
    }
}
```

## Realm 中的盐值处理

### 自定义 Realm 配置盐值

```java
@Override
protected AuthenticationInfo doGetAuthenticationInfo(
        AuthenticationToken token) throws AuthenticationException {

    UsernamePasswordToken upToken = (UsernamePasswordToken) token;
    String username = upToken.getUsername();

    // 1. 查询用户
    User user = userMapper.findByUsername(username);
    if (user == null) {
        throw new UnknownAccountException("用户不存在");
    }

    // 2. 构建认证信息（包含盐值）
    return new SimpleAuthenticationInfo(
        user.getUsername(),
        user.getPassword(),
        ByteSource.Util.bytes(user.getSalt()),  // 关键：传入盐值
        getName()
    );
}
```

### HashedCredentialsMatcher 自动验证

Shiro 会自动处理密码验证：

```java
// Shiro 内部执行逻辑
public class HashedCredentialsMatcher {

    public boolean doCredentialsMatch(AuthenticationToken token,
                                       AuthenticationInfo info) {
        // 1. 获取用户输入的密码
        Object tokenCredentials = token.getCredentials();

        // 2. 获取数据库存储的密码
        Object accountCredentials = info.getCredentials();

        // 3. 获取盐值
        ByteSource salt = info.getCredentialsSalt();

        // 4. 对用户输入的密码用相同的盐值进行哈希
        Object hashedTokenCredentials = hashProvidedCredentials(
            tokenCredentials, salt);

        // 5. 比较哈希结果
        return equals(hashedTokenCredentials, accountCredentials);
    }

    protected Object hashProvidedCredentials(Object credentials, ByteSource salt) {
        String algorithmName = getHashAlgorithmName();
        int iterations = getHashIterations();

        // SHA-256(password + salt) 迭代 1024 次
        return new SimpleHash(algorithmName, credentials, salt, iterations);
    }
}
```

## 完整配置示例

### 配置类

```java
@Configuration
public class ShiroConfig {

    @Bean
    public HashedCredentialsMatcher hashedCredentialsMatcher() {
        HashedCredentialsMatcher matcher = new HashedCredentialsMatcher();
        matcher.setHashAlgorithmName("SHA-256");
        matcher.setHashIterations(1024);
        matcher.setStoredCredentialsHexEncoded(false);  // Base64
        return matcher;
    }

    @Bean
    public UserRealm userRealm(HashedCredentialsMatcher matcher) {
        UserRealm realm = new UserRealm();
        realm.setCredentialsMatcher(matcher);
        return realm;
    }
}
```

### Realm 实现

```java
@Component
public class UserRealm extends AuthorizingRealm {

    @Override
    protected AuthenticationInfo doGetAuthenticationInfo(
            AuthenticationToken token) throws AuthenticationException {

        UsernamePasswordToken upToken = (UsernamePasswordToken) token;
        String username = upToken.getUsername();

        User user = userService.findByUsername(username);
        if (user == null) {
            throw new UnknownAccountException("用户 [" + username + "] 不存在");
        }

        if (Boolean.TRUE.equals(user.getLocked())) {
            throw new LockedAccountException("账户已被锁定");
        }

        // 返回认证信息，Shiro 会自动用盐值验证密码
        return new SimpleAuthenticationInfo(
            user.getUsername(),
            user.getPassword(),
            ByteSource.Util.bytes(user.getSalt()),  // 盐值
            getName()
        );
    }

    @Override
    protected AuthorizationInfo doGetAuthorizationInfo(
            PrincipalCollection principals) {
        // ... 授权逻辑
    }
}
```

## BCrypt：更安全的替代方案

如果项目对密码安全要求极高，可以考虑使用 Spring Security 的 BCrypt（Shiro 本身不支持 BCrypt，但可以集成）。

### 引入依赖

```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-crypto</artifactId>
    <version>5.7.6</version>
</dependency>
```

### BCrypt 使用

```java
@Configuration
public class BCryptConfig {

    @Bean
    public BCryptPasswordEncoder passwordEncoder() {
        // strength 越高，计算越慢（4-31），默认 10
        return new BCryptPasswordEncoder(10);
    }
}

@Service
public class UserServiceImpl implements UserService {

    @Autowired
    private BCryptPasswordEncoder passwordEncoder;

    @Override
    public void register(User user) {
        // 注册时：BCrypt 自动生成盐值
        user.setPassword(passwordEncoder.encode(user.getPassword()));
        userMapper.insert(user);
    }

    @Override
    public boolean verifyPassword(String rawPassword, String encodedPassword) {
        // 验证时：BCrypt 自动从加密密码中提取盐值
        return passwordEncoder.matches(rawPassword, encodedPassword);
    }
}
```

### BCrypt vs HashedCredentialsMatcher

| 对比项 | BCrypt | HashedCredentialsMatcher |
|---|---|---|
| 盐值生成 | 自动 | 手动 |
| 算法强度 | 可调 | 固定 |
| 迭代次数 | 自动优化 | 手动设置 |
| 跨框架兼容 | Spring Security | Shiro 专用 |
| 性能 | 较慢（更安全） | 可调 |

```
┌─────────────────────────────────────────────────────────┐
│                    密码安全 checklist                     │
├─────────────────────────────────────────────────────────┤
│ ✓ 使用 SHA-256 或更高强度的算法                            │

└─────────────────────────────────────────────────────────┘
```

## 面试高频问题

**问：为什么密码要加盐？**

防止彩虹表攻击。如果没有盐值，攻击者可以用预先计算好的彩虹表快速破解所有常见密码。加盐后，即使两个用户使用相同密码，存储的结果也不同。

**问：迭代次数是不是越多越好？**

不是。迭代次数越多，计算成本越高，验证速度越慢。需要平衡安全性和性能。一般推荐 1024-4096 次。

**问：Shiro 的密码加密和 Spring Security 的 BCrypt 哪个更好？**

BCrypt 更安全，它的盐值生成、迭代次数都是自适应调整的，而且有专门的密码哈希算法设计（基于 Blowfish）。但 HashedCredentialsMatcher 更轻量，配置简单。根据项目需求选择。

**问：用户修改密码后，缓存怎么处理？**

修改密码时，调用 `realm.clearCache(principals)` 清除用户的认证缓存。下次登录时，会重新从数据库加载新的密码。

## 总结

密码安全是系统安全的第一道防线：

1. **不要存明文**：任何时候都不要
2. **不要只存 MD5/SHA-1**：彩虹表攻击太容易
3. **加盐哈希**：每个用户独立的随机盐值
4. **迭代加密**：增加计算成本
5. **Shiro 配置**：`HashedCredentialsMatcher + 盐值 + 多次迭代`
