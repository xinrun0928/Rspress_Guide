# Shiro 认证机制：Realm 的艺术

"Realm"这个词，直译是"领域"或"王国"。在 Shiro 中，它就是你的安全数据的"领域"——用户信息从哪里来、密码怎么验证、权限如何获取，都由 Realm 决定。

从最简单配置文件，到复杂的数据库查询，再到生产级的自定义实现，Realm 经历了三个阶段的演进。理解这三种 Realm，你才算真正掌握了 Shiro 的认证精髓。

## 为什么需要 Realm？

Shiro 的设计哲学是**框架与业务分离**。SecurityManager 负责协调认证授权流程，但用户数据从哪里来，它不关心。Realm 就是这个桥梁。

```
SecurityManager
    │
    │ 统一的认证/授权接口
    ▼
   Realm ←────── 数据源（INI、数据库、Redis、LDAP...）
```

这种设计带来几个好处：

1. **解耦**：SecurityManager 不关心数据存储方式
2. **可替换**：换一个 Realm，数据源就换了
3. **可组合**：多个 Realm 可以同时生效
4. **可测试**：可以注入 mock Realm 做单元测试

## 三种 Realm，从简单到生产

| Realm | 数据源 | 适用场景 | 复杂度 |
|---|---|---|---|
| `IniRealm` | INI 配置文件 | 演示、测试 | 最低 |
| `JdbcRealm` | JDBC 数据库 | 简单项目 | 中等 |
| `AuthorizingRealm` | 自定义实现 | 生产环境 | 最高 |

接下来，我们逐一拆解这三种 Realm 的用法。

## IniRealm：配置即认证

IniRealm 是 Shiro 提供的最轻量级 Realm。只需要一个 `shiro.ini` 文件，就能完成用户认证。

```ini title="src/main/resources/shiro.ini"
[users]
zhangsan = 123456, admin
lisi = 654321, user

[roles]
admin = user:*, order:*
user = user:read, user:update
```

几行配置，就能跑通认证。但它的缺点也很明显：**数据无法动态更新**，不适合生产环境。

## JdbcRealm：从数据库加载

JdbcRealm 迈出了关键一步——用户数据从数据库加载。

```java
JdbcRealm realm = new JdbcRealm();
realm.setDataSource(dataSource);
```

它使用固定的 SQL 查询数据库：

- `authenticationQuery`：查询用户密码
- `userRolesQuery`：查询用户角色
- `permissionsQuery`：查询角色权限

但它的 SQL 是固定的，定制能力有限。

## 自定义 Realm：生产级方案

生产环境必须使用自定义 Realm。通过继承 `AuthorizingRealm`，你可以完全掌控认证和授权的每一个细节：

```java
public class UserRealm extends AuthorizingRealm {

    @Override
    protected AuthenticationInfo doGetAuthenticationInfo(
            AuthenticationToken token) {
        // 自定义认证逻辑
    }

    @Override
    protected AuthorizationInfo doGetAuthorizationInfo(
            PrincipalCollection principals) {
        // 自定义授权逻辑
    }
}
```

配合 `HashedCredentialsMatcher`，可以实现**盐值加密密码**、**多次哈希迭代**，让密码泄露的代价降到最低。

## 本章小结

| 章节 | 核心内容 |
|---|---|
| `ini-realm` | 用 INI 文件配置用户，适合演示 |
| `jdbc-realm` | 从数据库加载用户，SQL 可配置 |
| `custom-realm` | 自定义认证授权逻辑，生产级方案 |
| `credentials-matcher` | 密码加密：哈希 + 盐值 + 迭代 |
| `session` | Shiro Session：脱离容器的会话管理 |
| `remember-me` | 自动登录：加密 Cookie + 安全配置 |
