# JdbcRealm：连接数据库实现认证授权

从 INI 配置文件到数据库，JdbcRealm 迈出了关键的一步。

如果你的项目已经有现成的用户表，不想额外维护一套配置，JdbcRealm 可以直接对接数据库。但它的 SQL 是固定的，定制能力有限。理解它的内部实现，才能为后面的自定义 Realm 打基础。

## JdbcRealm 简介

JdbcRealm 是 Shiro 内置的基于 JDBC 的 Realm 实现。它通过预定义的 SQL 查询，从数据库中获取用户信息、角色和权限。

```java
// 最简单的配置
JdbcRealm realm = new JdbcRealm();
DataSource dataSource = DruidDataSource.create(...);  // Druid 连接池
realm.setDataSource(dataSource);
```

Shiro 会自动创建三张表（如果没有的话）：

```sql
-- users 表
CREATE TABLE users (
    username VARCHAR(50) PRIMARY KEY,
    password VARCHAR(100) NOT NULL
);

-- user_roles 表（用户-角色关联）
CREATE TABLE user_roles (
    username VARCHAR(50),
    role_name VARCHAR(50),
    PRIMARY KEY (username, role_name)
);

-- roles_permissions 表（角色-权限关联）
CREATE TABLE roles_permissions (
    role_name VARCHAR(50),
    permission VARCHAR(100),
    PRIMARY KEY (role_name, permission)
);
```

## JdbcRealm 的 SQL 查询

JdbcRealm 使用固定的 SQL 查询数据库。如果这些 SQL 不符合你的表结构，就需要自定义 Realm。

### 认证查询

```java
// 获取用户密码
realm.authenticationQuery = "SELECT password FROM users WHERE username = ?"
```

### 角色查询

```java
// 获取用户的所有角色
realm.userRolesQuery = "SELECT role_name FROM user_roles WHERE username = ?"
```

### 权限查询

```java
// 获取角色的所有权限
realm.permissionsQuery = "SELECT permission FROM roles_permissions WHERE role_name = ?"
```

## 完整配置示例

### 数据库表结构

```sql
-- 用户表
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(100) NOT NULL,
    locked TINYINT(1) DEFAULT 0,
    create_time DATETIME
);

-- 角色表
CREATE TABLE sys_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_code VARCHAR(50) UNIQUE NOT NULL,
    role_name VARCHAR(100)
);

-- 权限表
CREATE TABLE sys_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    permission_code VARCHAR(100) UNIQUE NOT NULL,
    permission_name VARCHAR(100)
);

-- 用户-角色关联表
CREATE TABLE sys_user_role (
    user_id BIGINT,
    role_id BIGINT,
    PRIMARY KEY (user_id, role_id)
);

-- 角色-权限关联表
CREATE TABLE sys_role_permission (
    role_id BIGINT,
    permission_id BIGINT,
    PRIMARY KEY (role_id, permission_id)
);
```

### Java 配置

```java
public class JdbcRealmConfig {

    @Bean
    public DataSource dataSource() {
        DruidDataSource dataSource = new DruidDataSource();
        dataSource.setUrl("jdbc:mysql://localhost:3306/shiro_demo");
        dataSource.setUsername("root");
        dataSource.setPassword("123456");
        dataSource.setDriverClassName("com.mysql.cj.jdbc.Driver");
        return dataSource;
    }

    @Bean
    public JdbcRealm jdbcRealm(DataSource dataSource) {
        JdbcRealm realm = new JdbcRealm();
        realm.setDataSource(dataSource);
        realm.setPermissionsQuery("""
            SELECT p.permission_code
            FROM sys_permission p
            INNER JOIN sys_role_permission rp ON p.id = rp.permission_id
            INNER JOIN sys_role r ON rp.role_id = r.id
            WHERE r.role_code = ?
            """);
        realm.setUserRolesQuery("""
            SELECT r.role_code
            FROM sys_role r
            INNER JOIN sys_user_role ur ON r.id = ur.role_id
            INNER JOIN sys_user u ON ur.user_id = u.id
            WHERE u.username = ?
            """);
        // 启用权限查询（默认关闭）
        realm.setCachingEnabled(true);
        realm.setAuthorizationCachingEnabled(true);
        return realm;
    }
}
```

### 密码哈希配置

JdbcRealm 默认使用明文密码比较。如果数据库存储的是哈希密码，需要配置 `CredentialsMatcher`：

```java
@Bean
public JdbcRealm jdbcRealm(DataSource dataSource) {
    JdbcRealm realm = new JdbcRealm();

    // 配置哈希匹配器
    HashedCredentialsMatcher matcher = new HashedCredentialsMatcher();
    matcher.setHashAlgorithmName("SHA-256");
    matcher.setHashIterations(1024);
    matcher.setStoredCredentialsHexEncoded(false);  // 数据库存 Base64
    realm.setCredentialsMatcher(matcher);

    realm.setDataSource(dataSource);
    // ... 其他配置
    return realm;
}
```

## JdbcRealm 源码解析

### getAuthenticationInfo()

```java
// org.apache.shiro.realm.jdbc.JdbcRealm
public class JdbcRealm extends AuthorizingRealm {

    protected AuthenticationInfo doGetAuthenticationInfo(
            AuthenticationToken token) throws AuthenticationException {

        UsernamePasswordToken upToken = (UsernamePasswordToken) token;

        // 1. 执行认证查询
        String password = getPasswordForUser(upToken.getUsername());

        if (password == null) {
            throw new UnknownAccountException("用户不存在");
        }

        // 2. 返回 SimpleAuthenticationInfo
        // 密码验证交给 CredentialsMatcher
        return new SimpleAuthenticationInfo(
            upToken.getUsername(),
            password,
            getName()
        );
    }

    private String getPasswordForUser(String username) {
        Connection conn = null;
        PreparedStatement ps = null;
        ResultSet rs = null;
        try {
            conn = dataSource.getConnection();
            ps = conn.prepareStatement(authenticationQuery);
            ps.setString(1, username);
            rs = ps.executeQuery();

            if (rs.next()) {
                return rs.getString(1);
            }
            return null;
        } finally {
            closeQuietly(rs, ps, conn);
        }
    }
}
```

### doGetAuthorizationInfo()

```java
protected AuthorizationInfo doGetAuthorizationInfo(
        PrincipalCollection principals) {

    String username = (String) principals.getPrimaryPrincipal();

    // 1. 查询角色
    Set<String> roles = getRolesForUser(username);

    // 2. 查询权限
    Set<String> permissions = getPermissions(username, roles);

    // 3. 返回 AuthorizationInfo
    SimpleAuthorizationInfo info = new SimpleAuthorizationInfo();
    info.setRoles(roles);
    info.setStringPermissions(permissions);
    return info;
}

private Set<String> getRolesForUser(String username) {
    Connection conn = null;
    PreparedStatement ps = null;
    ResultSet rs = null;
    Set<String> roles = new HashSet<>();

    try {
        conn = dataSource.getConnection();
        ps = conn.prepareStatement(userRolesQuery);
        ps.setString(1, username);
        rs = ps.executeQuery();

        while (rs.next()) {
            roles.add(rs.getString(1));
        }
    } finally {
        closeQuietly(rs, ps, conn);
    }

    return roles;
}
```

## JdbcRealm 的局限性

虽然 JdbcRealm 比 IniRealm 强大，但它有几个明显的问题：

### 问题一：SQL 固定，难以适配复杂表结构

JdbcRealm 只有三个 SQL 配置项。如果你的表结构稍微复杂一点，比如：

- 用户表有 `salt` 字段
- 用户和权限是多对多关系（不通过角色）
- 需要同时查询菜单、按钮、数据权限

JdbcRealm 就力不从心了。

### 问题二：无缓存支持

JdbcRealm 默认没有启用缓存。每次权限检查都会查询数据库：

```
用户请求 → Shiro 过滤器 → isPermitted() → Realm.getAuthorizationInfo()
→ 每次都执行 SQL → 数据库压力
```

### 问题三：不支持盐值认证

JdbcRealm 的 `authenticationQuery` 只返回一个字段（密码），不支持同时查询盐值。如果要使用盐值加密，必须自己扩展。

## 自定义 JdbcRealm 解决 SQL 适配问题

针对复杂表结构，可以继承 JdbcRealm 并重写 SQL：

```java
public class CustomJdbcRealm extends JdbcRealm {

    // 自定义认证查询（包含盐值）
    @Override
    protected AuthenticationInfo doGetAuthenticationInfo(
            AuthenticationToken token) throws AuthenticationException {

        UsernamePasswordToken upToken = (UsernamePasswordToken) token;
        String username = upToken.getUsername();

        Connection conn = null;
        PreparedStatement ps = null;
        ResultSet rs = null;

        try {
            conn = dataSource.getConnection();
            // 自定义 SQL：同时查询密码和盐值
            String sql = """
                SELECT password, salt
                FROM sys_user
                WHERE username = ? AND locked = 0
                """;
            ps = conn.prepareStatement(sql);
            ps.setString(1, username);
            rs = ps.executeQuery();

            if (rs.next()) {
                String password = rs.getString("password");
                String salt = rs.getString("salt");
                return new SimpleAuthenticationInfo(
                    username,
                    password,
                    ByteSource.Util.bytes(salt),  // 传入盐值
                    getName()
                );
            }

            throw new UnknownAccountException("用户不存在或已被禁用");
        } catch (SQLException e) {
            throw new AuthenticationException(e);
        } finally {
            closeQuietly(rs, ps, conn);
        }
    }
}
```

## 生产环境优化：添加缓存

```java
@Bean
public JdbcRealm jdbcRealm(DataSource dataSource) {
    JdbcRealm realm = new JdbcRealm();
    realm.setDataSource(dataSource);
    // ... SQL 配置

    // 启用内存缓存
    realm.setCachingEnabled(true);
    realm.setAuthenticationCachingEnabled(true);      // 认证缓存
    realm.setAuthorizationCachingEnabled(true);       // 授权缓存
    realm.setCacheName("jdbcRealmCache");

    // 设置缓存管理器
    CacheManager cacheManager = new MemoryConstrainedCacheManager();
    realm.setCacheManager(cacheManager);

    return realm;
}
```

:::tip 缓存建议
生产环境推荐使用 Redis 缓存替代内存缓存。认证和授权缓存设置 TTL（如 30 分钟），并在用户权限变更时主动清除缓存。
:::

## 总结：什么时候用 JdbcRealm？

JdbcRealm 适合以下场景：

| 场景 | 建议 |
|---|---|
| 快速原型开发 | JdbcRealm，可以快速跑通流程 |
| 表结构简单 | users / roles / permissions 三张表 |
| 不需要盐值加密 | 明文或 MD5/SHA |
| 用户量小 | `<` 1000 并发用户 |

对于复杂业务场景，建议直接实现自定义 Realm。

> **面试追问**：JdbcRealm 和自定义 Realm 的区别是什么？
>
> JdbcRealm 的 SQL 是固定的，适合简单场景。自定义 Realm 可以完全控制认证和授权的逻辑，适合复杂业务。生产环境推荐自定义 Realm。
