# Shiro Session：独立于容器的会话管理

Servlet 的 Session 依赖 Web 容器（Tomcat/Jetty）——Session 存在哪里由容器决定，序列化由容器处理，超时时间也在容器配置里。听起来没问题，但当你需要**分布式 Session** 或者**跨服务共享 Session** 时，问题就来了。

Shiro 的 Session 机制解决了这个问题。它自己管理 Session，不依赖任何 Web 容器——你可以把 Session 存在内存、Redis、数据库，甚至 MongoDB 里。

## 为什么需要 Shiro Session？

### 传统 Session 的问题

| 问题 | 描述 | 影响 |
|---|---|---|
| 依赖容器 | Session 由 Tomcat 等容器管理 | 无法跨容器共享 |
| 序列化不可控 | 容器决定序列化方式 | 可能存在兼容性问题 |
| 集群困难 | 多节点需要 Session 同步 | 增加架构复杂度 |
| 超时配置分散 | 容器配置和应用配置分离 | 维护困难 |

### Shiro Session 的优势

```
传统方式：请求 → Tomcat Session（内存）
Shiro 方式：请求 → Shiro SessionManager → 任意存储（内存/Redis/DB）
```

- **存储可控**：Session 存在哪，你说了算
- **序列化可控**：可以用 JSON、Kryo、FST 等高效序列化
- **跨容器**：不依赖 Tomcat，随时迁移
- **统一配置**：超时时间在代码中配置

## Session 接口

Shiro 的 Session 接口定义：

```java
public interface Session {

    // 获取会话 ID
    Serializable getId();

    // 获取会话开始时间
    Date getStartTimestamp();

    // 获取上次访问时间
    Date getLastAccessTime();

    // 获取超时时间（毫秒）
    long getTimeout();

    // 设置超时时间
    void setTimeout(long maxIdleTimeInMs);

    // 停止会话
    void stop();

    // 获取属性
    Object getAttribute(Object key) throws InvalidSessionException;

    // 设置属性
    void setAttribute(Object key, Object value) throws InvalidSessionException;

    // 删除属性
    Object removeAttribute(Object key) throws InvalidSessionException;
}
```

## Session 的获取方式

### 方式一：通过 Subject 获取

```java
Subject subject = SecurityUtils.getSubject();

// 获取已有 Session（如果不存在，返回 null）
Session session = subject.getSession(false);

// 获取 Session（不存在则创建）
Session session = subject.getSession();
```

### 方式二：通过 SecurityUtils 获取

```java
Session session = SecurityUtils.getSubject().getSession();
session.setAttribute("user", currentUser);
```

:::warning 注意
`getSession(false)` vs `getSession(true)`：前者不创建新 Session，后者不存在则创建。
:::

## SessionManager 实现

Shiro 提供了多种 SessionManager 实现：

| 实现 | 说明 | 使用场景 |
|---|---|---|
| `DefaultSecurityManager` | 默认实现，不支持 Web | 普通 Java 应用 |
| `DefaultWebSecurityManager` | 支持 Web 环境 | Web 应用 |
| `DefaultWebSessionManager` | Web 环境专用 | 标准 Web 应用 |
| `ServletContainerSessionManager` | 委托给 Tomcat | 不推荐 |
| 自定义 SessionManager | 自定义存储 | 分布式环境 |

### 默认配置

```java
// 普通 Java 应用
DefaultSecurityManager securityManager = new DefaultSecurityManager();

// Web 应用
DefaultWebSecurityManager securityManager = new DefaultWebSecurityManager(realm);
```

## SimpleSession：会话的内部实现

`SimpleSession` 是 Session 接口的默认实现：

```java
// org.apache.shiro.session.simple.SimpleSession
public class SimpleSession implements Session {

    private Serializable id;
    private Date startTimestamp;
    private Date lastAccessTime;
    private long timeout = 1800000;  // 默认 30 分钟
    private boolean expired;
    private boolean stopped;
    private Map<Object, Object> attributes = new HashMap<>();

    // ... 省略方法实现
}
```

### 生命周期

```
会话创建
    │
    │ setTimeout() 设置超时
    ▼
会话活跃（每次访问更新 lastAccessTime）
    │
    │ lastAccessTime + timeout `<` 当前时间
    ▼
会话过期（expired = true）
    │
    │ 调用 stop()
    ▼
会话停止（stopped = true）
```

## Session 监听器

```java
public class MySessionListener implements SessionListener {

    @Override
    public void onStart(Session session) {
        log.info("会话创建: {}", session.getId());
    }

    @Override
    public void onStop(Session session) {
        log.info("会话停止: {}", session.getId());

    @Override
    public void onExpiration(Session session) {
```

## SessionDAO：持久化接口

`SessionDAO` 是 Session 持久化的核心接口：

```java
public interface SessionDAO {

    // 创建会话
    Serializable create(Session session);

    // 读取会话
    Session readSession(Serializable id) throws UnknownSessionException;

    // 更新会话
    void update(Session session) throws UnknownSessionException;

    // 删除会话
    void delete(Session session);

    // 获取所有活跃会话
    Collection<Session> getActiveSessions();
}
```

### AbstractSessionDAO：便捷基类

```java
public abstract class AbstractSessionDAO implements SessionDAO {

    private Serializable sessionIdGenerator = new JavaUuidSessionIdGenerator();

    @Override
    public Serializable create(Session session) {
        Serializable id = generateSessionId();
        assignSessionId(session, id);
        // 保存到存储
        doCreate(session);
        return id;
    }

    // 子类实现具体的持久化逻辑
    protected abstract void doCreate(Session session);
}
```

## 自定义 SessionDAO：内存实现

最简单的实现是内存存储：

```java
public class MemorySessionDAO extends AbstractSessionDAO {

    private Map<Serializable, Session> sessions = new ConcurrentHashMap<>();

    @Override
    protected void doCreate(Session session) {
        sessions.put(session.getId(), session);
    }

    @Override
    public Session readSession(Serializable id) {
        return sessions.get(id);
    }

    @Override
    public void update(Session session) throws UnknownSessionException {
        sessions.put(session.getId(), session);
    }

    @Override
    public void delete(Session session) {
        sessions.remove(session.getId());
    }

    @Override
    public Collection<Session> getActiveSessions() {
        return sessions.values();
    }
}
```

### RedisSessionDAO 配置

```java
@Configuration
public class ShiroConfig {

    @Bean
    public RedisManager redisManager() {
        RedisManager manager = new RedisManager();
        manager.setHost("127.0.0.1:6379");
        manager.setPassword("123456");
        manager.setDatabase(0);
        manager.setTimeout(1800000);
        return manager;
    }

    @Bean
    public RedisSessionDAO redisSessionDAO(RedisManager redisManager) {
        RedisSessionDAO sessionDAO = new RedisSessionDAO();
        sessionDAO.setRedisManager(redisManager);
        sessionDAO.setSessionIdGenerator(new JavaUuidSessionIdGenerator());
        return sessionDAO;
    }

    @Bean
    public DefaultWebSessionManager sessionManager(RedisSessionDAO redisSessionDAO) {
        DefaultWebSessionManager sessionManager = new DefaultWebSessionManager();
        sessionManager.setSessionDAO(redisSessionDAO);
        sessionManager.setGlobalSessionTimeout(1800000);  // 30 分钟
        return sessionManager;
    }
}
```

### 自定义 RedisSessionDAO

如果 shiro-redis 不满足需求，可以自己实现：

```java
@Component
public class CustomRedisSessionDAO extends AbstractSessionDAO {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String SESSION_KEY_PREFIX = "shiro:session:";
    private static final long SESSION_EXPIRE = 1800;  // 秒

    @Override
    protected void doCreate(Session session) {
        Serializable id = generateSessionId(session);
        assignSessionId(session, id);
        saveSession(session);
    }

    @Override
    public Session readSession(Serializable id) throws UnknownSessionException {
        if (id == null) {
            return null;
        }
        Session session = getSession(id);
        if (session == null) {
            throw new UnknownSessionException("Session 不存在: " + id);
        }
        return session;
    }

    @Override
    public void update(Session session) throws UnknownSessionException {
        saveSession(session);
    }

    @Override
    public void delete(Session session) {
        String key = SESSION_KEY_PREFIX + session.getId();
        redisTemplate.delete(key);
    }

    @Override
    public Collection<Session> getActiveSessions() {
        Set<String> keys = redisTemplate.keys(SESSION_KEY_PREFIX + "*");
        List<Session> sessions = new ArrayList<>();
        for (String key : keys) {
            Session session = getSession(key.substring(SESSION_KEY_PREFIX.length()));
            if (session != null) {
                sessions.add(session);
            }
        }
        return sessions;
    }

    private void saveSession(Session session) {
        if (session == null || session.getId() == null) {
            return;
        }
        String key = SESSION_KEY_PREFIX + session.getId();
        redisTemplate.opsForValue().set(key, session, SESSION_EXPIRE, TimeUnit.SECONDS);
    }

    private Session getSession(Serializable id) {
        String key = SESSION_KEY_PREFIX + id;
        return (Session) redisTemplate.opsForValue().get(key);
    }
}
```

## Session 序列化问题

RedisSessionDAO 存储 Session 时需要进行序列化。Shiro 默认使用 Java 原生序列化，效率较低。推荐使用 Kryo 或 JSON。

### Kryo 序列化

```xml
<dependency>
    <groupId>com.esotericsoftware</groupId>
    <artifactId>kryo</artifactId>
    <version>5.4.0</version>
</dependency>
```

```java
public class KryoSerializer {

    private static final ThreadLocal<Kryo> kryoLocal = ThreadLocal.withInitial(() -> {
        Kryo kryo = new Kryo();
        kryo.setRegistrationRequired(false);
        kryo.setReferences(true);
        kryo.register(SimpleSession.class);
        return kryo;
    });

    public byte[] serialize(Object obj) {
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        Output output = new Output(baos);
        kryoLocal.get().writeObject(output, obj);
        output.close();
        return baos.toByteArray();
    }

    public <T> T deserialize(byte[] bytes, Class<T> clazz) {
        ByteArrayInputStream bais = new ByteArrayInputStream(bytes);
        Input input = new Input(bais);
        return kryoLocal.get().readObject(input, clazz);
    }
}
```

## 与 Servlet Session 的关系

### DefaultWebSessionManager

Web 环境下，Shiro 默认使用 `DefaultWebSessionManager`。它会自动与 Servlet 容器集成：

```java
public class DefaultWebSessionManager extends DefaultSessionManager {

    // 是否验证 Servlet Session
    private boolean sessionIdCookieEnabled = true;

    // 获取 Session 时，如果 Shiro Session 不存在，会创建新的
    @Override
    protected Session retrieveSession(SessionContext context) {
        // 1. 尝试从请求中获取 Session ID
        // 2. 如果 Shiro Session 已存在，返回
        // 3. 如果不存在，从 SessionDAO 中读取
    }
}
```

### 禁用 Shiro Session

如果只需要 Subject 的身份标识，不需要 Session 功能，可以禁用：

```java
DefaultWebSecurityManager securityManager = new DefaultWebSecurityManager(realm);
// 禁用 Session
securityManager.setSubjectDAO(new DefaultSubjectDAO());
securityManager.getSubjectDAO().setSessionStorageEvaluator(
    new DefaultSessionStorageEvaluator() {
        @Override
        public boolean isSessionStorageEnabled(Subject subject) {
            return false;
        }
    }
);
```

## 面试高频问题

**问：Shiro 的 Session 和 Servlet 的 Session 有什么关系？**

默认情况下，Shiro 使用自己的 Session 机制，不依赖 Servlet 容器。但 `DefaultWebSessionManager` 会自动从请求中获取 Session ID，实现与 Web 环境的兼容。

**问：如何实现分布式 Session？**

使用 `SessionDAO` 将 Session 存储到 Redis。每次请求时，`RedisSessionDAO` 从 Redis 读取 Session 数据。

**问：Session 过期后如何处理？**

可以实现 `SessionListener` 监听过期事件，或者配置 Shiro 的过滤器链，在过滤器中检查 Session 有效性。

**问：Shiro Session 和 JWT Token 有什么区别？**

Session 需要服务端存储状态，Token 不需要（无状态）。微服务架构推荐 JWT，大型单体应用推荐 Shiro Session。

## 总结

Shiro Session 的核心优势是**可控性**：

1. **存储可控**：内存、Redis、数据库、Memcached 都可以
2. **序列化可控**：Java 原生、Kryo、JSON 都可以
3. **超时可控**：全局配置 + 单个 Session 配置
4. **事件可控**：通过 SessionListener 监听生命周期

```
分布式 Session 架构：

└──────┬──────┘

       ▼
┌─────────────┐     ┌─────────────┐
│ SessionDAO  │────▶│    Redis    │
│ (Redis)     │     │  (共享存储)  │
└─────────────┘     └─────────────┘
```
