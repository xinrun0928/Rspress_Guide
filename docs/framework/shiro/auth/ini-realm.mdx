# IniRealm：用 ini 文件配置用户和权限

不需要数据库，不需要复杂的配置——一个 `.ini` 文件，就能跑通 Shiro 的认证授权。

IniRealm 是 Shiro 提供的最轻量级 Realm，特别适合**演示项目、单元测试、或者快速验证 Shiro 流程**。它的缺点也很明显：不适合生产环境，数据无法动态更新。

## 最小可用的 Shiro 配置

假设你的项目结构如下：

```
src/
└── main/
    └── resources/
        └── shiro.ini
```

### shiro.ini 配置

```ini title="src/main/resources/shiro.ini"
[users]
# 用户名 = 密码, 角色1, 角色2, 权限1, 权限2
zhangsan = 123456, admin, user:create, user:delete
lisi = 654321, user, user:read, user:update
wangwu = abcdef, guest, user:read

[roles]
# 角色 = 权限1, 权限2, 权限3
admin = user:*, order:*, system:*
user = user:read, user:update
guest = user:read
```

配置解释：

- `[users]` 段：定义用户账号，格式是 `用户名 = 密码, 角色1, 角色2, ...`
- `[roles]` 段：定义角色和权限的映射，格式是 `角色名 = 权限1, 权限2, ...`
- 角色和权限都是可选的，可以为空
- 密码可以是明文，也可以是哈希值

### Java 代码加载配置

```java
public class IniRealmTest {
    public static void main(String[] args) {
        // 1. 创建 IniSecurityManagerFactory
        Factory<SecurityManager> factory =
            new IniSecurityManagerFactory("classpath:shiro.ini");

        // 2. 获取 SecurityManager
        SecurityManager securityManager = factory.getInstance();

        // 3. 设置到 SecurityUtils
        SecurityUtils.setSecurityManager(securityManager);

        // 4. 获取 Subject
        Subject subject = SecurityUtils.getSubject();

        // 5. 登录
        UsernamePasswordToken token =
            new UsernamePasswordToken("zhangsan", "123456");
        subject.login(token);

        // 6. 验证结果
        System.out.println("认证状态: " + subject.isAuthenticated());
        System.out.println("角色: " + subject.hasRole("admin"));
        System.out.println("权限: " + subject.isPermitted("user:create"));
    }
}
```

运行结果：

```
认证状态: true
角色: true
权限: true
```

## IniRealm 的加载过程

`IniSecurityManagerFactory` 负责解析 INI 文件，创建 SecurityManager。让我们看看它的内部逻辑：

### 第一步：解析 INI 文件

```java
// org.apache.shiro.config.IniSecurityManagerFactory
public class IniSecurityManagerFactory extends IniFactorySupport<SecurityManager> {

    public IniSecurityManagerFactory(String iniResourcePath) {
        // 加载 INI 资源
        Ini ini = Ini.fromResourcePath(iniResourcePath);
        // 调用父类初始化
        this.setIni(ini);
        // 创建 SecurityManager
        this.resolveSecurityManager();
    }
}
```

### 第二步：创建 Realm

```java
@Override
protected SecurityManager createSecurityManager(Ini ini) {
    // 从 [main] 段查找 realm 配置
    Map<String, String> mainSection = ini.getSection("main");
    IniRealm realm = (IniRealm) createInstance(mainSection.get("myRealm"));
    return realm;
}
```

### 第三步：IniRealm 解析用户数据

```java
// org.apache.shiro.realm.text.IniRealm
public class IniRealm extends AuthorizingRealm {

    private Ini ini;

    // 初始化时解析 INI
    public void onInit() {
        super.onInit();
        // 解析 [users] 段
        processIniUsers();
        // 解析 [roles] 段
        processIniRoles();
    }

    // 解析用户
    protected void processIniUsers() {
        Ini.Section users = ini.getSection("users");
        for (Map.Entry<String, String> entry : users.entrySet()) {
            String username = entry.getKey();
            String value = entry.getValue();

            // 解析: 密码, 角色1, 角色2, 权限1, 权限2
            String[] parts = value.split(",");
            String password = parts[0].trim();
            List<String> roles = new ArrayList<>();
            List<String> permissions = new ArrayList<>();

            // 解析角色（逗号分隔）
            for (int i = 1; i < parts.length; i++) {
                String item = parts[i].trim();
                // 根据 [roles] 段判断是角色还是权限
                if (ini.getSection("roles").containsKey(item)) {
                    roles.add(item);
                } else {
                    permissions.add(item);
                }
            }

            // 创建 SimpleAccount 并存入内存
            SimpleAccount account = new SimpleAccount(
                username, password, getName());
            account.setRoles(roles);
            account.setStringPermissions(permissions);
            add(account);
        }
    }
}
```

## 密码哈希：更安全的存储

明文密码在 `[users]` 中存储非常不安全。更好的做法是存储哈希值：

### MD5 哈希

```ini title="src/main/resources/shiro.ini"
[users]
# 密码是 MD5("123456")
# zhangsan 的 MD5 = 5f4dcc3b5aa765d61d8327deb882cf99
zhangsan = 5f4dcc3b5aa765d61d8327deb882cf99, admin
```

### 盐值哈希

单独的哈希仍然可能被彩虹表攻击。加入盐值（Salt）后更安全：

```ini
[users]
# 格式: 密码$盐值
zhangsan = 5f4dcc3b5aa765d61d8327deb882cf99$randomSalt, admin
```

但 IniRealm 原生不支持盐值加密，需要配置 `CredentialsMatcher`：

```java
// 创建哈希匹配器
HashedCredentialsMatcher matcher = new HashedCredentialsMatcher();
matcher.setHashAlgorithmName("MD5");
matcher.setHashIterations(1);
matcher.setStoredCredentialsHexEncoded(true);

// 应用到 IniRealm
IniRealm realm = new IniRealm("classpath:shiro.ini");
realm.setCredentialsMatcher(matcher);
```

## IniRealm 的局限性

虽然 IniRealm 用起来简单，但它有几个致命缺点：

| 局限性 | 说明 | 影响 |
|---|---|---|
| 数据无法动态更新 | INI 文件在启动时加载 | 用户变更后需要重启应用 |
| 不支持数据库 | 所有数据都在内存 | 无法存储大量用户 |
| SQL 固定 | JdbcRealm 有同样的问题 | 不适合复杂业务场景 |
| 无缓存机制 | 每次都重新解析 | 性能差 |

:::warning 生产环境注意
IniRealm 只适合开发和测试环境。生产环境必须使用自定义 Realm 对接数据库。
:::

## 动态更新 IniRealm 数据

虽然不推荐，但你可以在运行时动态添加/删除用户：

```java
// 获取 IniRealm
IniRealm realm = (IniRealm) securityManager.getRealms().iterator().next();

// 动态添加用户
SimpleAccount newUser = new SimpleAccount(
    "testuser",      // 用户名
    "testpass",      // 密码
    "testRole",      // 角色
    "user:read"     // 权限
);
realm.add(newUser);

// 动态删除用户
realm.remove("testuser");

// 动态更新密码
SimpleAccount user = realm.getAccount("zhangsan");
user.setCredentials("newpassword");
```

## 测试用例：完整示例

```java
class IniRealmTest {

    private SecurityManager securityManager;

    @BeforeEach
    void setUp() {
        // 加载 INI 配置
        Factory<SecurityManager> factory =
            new IniSecurityManagerFactory("classpath:shiro.ini");
        securityManager = factory.getInstance();
        SecurityUtils.setSecurityManager(securityManager);
    }

    @Test
    void testLoginSuccess() {
        Subject subject = SecurityUtils.getSubject();
        UsernamePasswordToken token =
            new UsernamePasswordToken("zhangsan", "123456");

        subject.login(token);

        assertTrue(subject.isAuthenticated());
        assertEquals("zhangsan", subject.getPrincipal());
        assertTrue(subject.hasRole("admin"));
    }

    @Test
    void testLoginWithWrongPassword() {
        Subject subject = SecurityUtils.getSubject();
        UsernamePasswordToken token =
            new UsernamePasswordToken("zhangsan", "wrongpassword");

        assertThrows(IncorrectCredentialsException.class,
            () -> subject.login(token));
    }

    @Test
    void testAuthorization() {
        Subject subject = SecurityUtils.getSubject();
        subject.login(new UsernamePasswordToken("lisi", "654321"));

        assertTrue(subject.isPermitted("user:read"));
        assertTrue(subject.isPermitted("user:update"));
        assertFalse(subject.isPermitted("user:delete"));
    }

    @Test
    void testLogout() {
        Subject subject = SecurityUtils.getSubject();
        subject.login(new UsernamePasswordToken("zhangsan", "123456"));

        assertTrue(subject.isAuthenticated());

        subject.logout();

        assertFalse(subject.isAuthenticated());
    }
}
```

## 总结：IniRealm 适用场景

IniRealm 是一个**快速入门**的好工具，但不是一个**生产级**的解决方案。

```
IniRealm 适用场景：
├── 快速验证 Shiro 流程
├── 编写 Shiro 单元测试
├── 演示项目
└── 小型内部工具（用户数 `<` 50，变更不频繁）

生产环境建议：
├── 自定义 Realm + 数据库
└── 配合 Redis 缓存
```
