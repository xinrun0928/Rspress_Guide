# Shiro RememberMe：自动登录的实现

"记住我"——这个看似简单的功能，背后藏着不少安全门道。

用户登录一次后，关闭浏览器，下次打开网站还是登录状态。这不是魔法，背后是浏览器 Cookie 存储的用户身份凭证。Shiro 提供了完整的 RememberMe 实现，但用不好，也会带来安全隐患。

## RememberMe 的工作原理

### 三步走

```
1. 用户登录时勾选"记住我"
         │
         ▼
2. Shiro 生成加密 Cookie，写入浏览器
   Cookie 内容 = AES(用户名 + 过期时间)
         │
         ▼
3. 下次访问时，Shiro 解析 Cookie
   自动创建 Subject（已认证但未登录）
```

### 与认证的区别

| 状态 | 说明 | 能做什么 |
|---|---|---|
| `isAuthenticated()` | 已认证（登录过） | 所有功能 |
| `isRemembered()` | RememberMe | 浏览、个性化设置 |
| 匿名用户 | 什么都没 | 只能访问公开资源 |

:::warning 安全边界
`isRemembered()` 状态下，用户不应执行敏感操作（如修改密码、转账）。这些操作必须强制要求重新输入密码。
:::

## SimpleRememberMeManager 核心原理

Shiro 通过 `RememberMeManager` 管理自动登录功能：

```java
public interface RememberMeManager {

    // 获取 RememberMe Cookie
    Serializable getRememberedIdentity(SubjectContext subjectContext);

    // 写入 RememberMe Cookie
    void rememberIdentity(SubjectContext subjectContext);

    // 清除 RememberMe Cookie
    void forgetIdentity(SubjectContext subjectContext);
}
```

`SimpleRememberMeManager` 是默认实现：

### Cookie 生成流程

```java
// org.apache.shiro.web.mgt.SimpleRememberMeManager
public class SimpleRememberMeManager implements RememberMeManager {

    // 加密密钥（建议运行时动态生成）
    private byte[] encryptionCipherKey;
    private byte[] signingCipherKey;

    @Override
    public void rememberIdentity(SubjectContext subjectContext) {
        // 1. 获取 principal（用户身份）
        PrincipalCollection principals = subjectContext.getPrincipals();
        String username = principals.getPrimaryPrincipal().toString();

        // 2. 序列化和加密
        byte[] bytes = serialize(principals);
        bytes = encrypt(bytes);

        // 3. 签名防篡改
        byte[] signed = sign(bytes);

        // 4. Base64 编码，写入 Cookie
        String encoded = Base64.getEncoder().encodeToString(signed);
        getCookie().saveToRequest(new SimpleCookie(encoded, subjectContext));
    }
}
```

### Cookie 读取流程

```java
@Override
public PrincipalCollection getRememberedIdentity(SubjectContext subjectContext) {
    // 1. 从 Cookie 读取值
    String encoded = getCookie().readValue();

    // 2. Base64 解码
    byte[] decoded = Base64.getDecoder().decode(encoded);

    // 3. 验签（防止篡改）
    byte[] bytes = verify(decoded);

    // 4. 解密
    bytes = decrypt(bytes);

    // 5. 反序列化
    PrincipalCollection principals = deserialize(bytes);

    return principals;
}
```

## 加密机制详解

### AES 加密 + 签名

Shiro 使用 AES + HMAC 的双重保护：

```
明文: "zhangsan"
         │
         │ 序列化成字节
         ▼
序列化: [B@12345678]
         │
         │ AES-CBC 加密
         ▼
密文: [0x1f, 0x2a, ...]
         │
         │ HMAC-SHA256 签名
         ▼
签名: [0x3b, 0x4c, ...|密文]
         │
         │ Base64 编码
         ▼
Cookie: "MzrDsaKfj2lsD...=="
```

### 密钥配置

```java
// 方式一：自动生成（每次启动随机生成，不推荐多实例）
SimpleRememberMeManager manager = new SimpleRememberMeManager();

// 方式二：手动配置（推荐，所有实例使用相同密钥）
SimpleRememberMeManager manager = new SimpleRememberMeManager();
manager.setCipherKey(Base64.decode("kPH+bIxk5D2deZiIxpm6TrYpDgF0mBJB8YdVHN2fN0Y="));
manager.setSigningCipherKey(Base64.decode("kPH+bIxk5D2deZiIxpm6TrYpDgF0mBJB8YdVHN2fN0Y="));
```

:::danger 密钥安全
硬编码的密钥如果泄露，攻击者可以伪造任意用户的 RememberMe Cookie。生产环境建议使用配置中心或环境变量管理密钥。
:::

## 配置 RememberMe

### shiro.ini 配置

```ini
[main]
# 配置 Cookie
rememberMeCookie = org.apache.shiro.web.servlet.SimpleCookie
rememberMeCookie.name = rememberMe
rememberMeCookie.maxAge = 604800  # 7 天
rememberMeCookie.httpOnly = true
rememberMeCookie.secure = false   # 生产环境设为 true

# 配置 RememberMeManager
rememberMeManager = org.apache.shiro.web.mgt.SimpleRememberMeManager
rememberMeManager.cookie = $rememberMeCookie
rememberMeManager.cipherKey = kPH+bIxk5D2deZiIxpm6TrYpDgF0mBJB8YdVHN2fN0Y=

# SecurityManager 配置
securityManager.subjectRememberMe = $rememberMeManager
```

### Spring Boot 配置

```java
@Configuration
public class ShiroConfig {

    @Bean
    public SimpleCookie rememberMeCookie() {
        SimpleCookie cookie = new SimpleCookie("rememberMe");
        cookie.setMaxAge(604800);  // 7 天
        cookie.setHttpOnly(true);
        cookie.setSecure(false);   // 生产环境设为 true
        return cookie;
    }

    @Bean
    public SimpleRememberMeManager rememberMeManager(SimpleCookie cookie) {
        SimpleRememberMeManager manager = new SimpleRememberMeManager();
        manager.setCookie(cookie);
        // Base64 编码的 AES 密钥
        manager.setCipherKey(Base64.decode(
            "kPH+bIxk5D2deZiIxpm6TrYpDgF0mBJB8YdVHN2fN0Y="));
        return manager;
    }

    @Bean
    public DefaultWebSecurityManager securityManager(Realm realm,
                                                       RememberMeManager rememberMeManager) {
        DefaultWebSecurityManager manager = new DefaultWebSecurityManager(realm);
        manager.setRememberMeManager(rememberMeManager);
        return manager;
    }
}
```

## 安全注意事项

### 敏感操作必须强制重新认证

```java
@Controller
public class AccountController {

    @RequestMapping("/settings")
    public String settings(HttpSession session) {
        Subject subject = SecurityUtils.getSubject();

        // RememberMe 用户不能访问敏感页面
        if (!subject.isAuthenticated()) {
            return "redirect:/login?force=1";
        }
        return "settings";
    }

    @RequestMapping("/password/change")
    public String changePassword(@RequestParam String newPassword) {
        Subject subject = SecurityUtils.getSubject();

        // 强制重新认证
        if (!subject.isAuthenticated()) {
            throw new UnauthorizedException("请先登录");
        }

        // 修改密码
        userService.changePassword(subject.getPrincipal().toString(), newPassword);
        return "success";
    }
}
```

### HTTPS 必须启用

```java
SimpleCookie cookie = new SimpleCookie("rememberMe");
cookie.setSecure(true);  // 只在 HTTPS 下传输
cookie.setHttpOnly(true); // 禁止 JavaScript 访问
```

### Cookie 配置最佳实践

```java
SimpleCookie cookie = new SimpleCookie("rememberMe");
cookie.setMaxAge(604800);     // 7 天，不要设置过长
cookie.setPath("/");
cookie.setDomain("example.com");
cookie.setSecure(true);       // HTTPS Only
cookie.setHttpOnly(true);     // JavaScript 无法读取
cookie.setSameSite("Strict"); // CSRF 保护
```

## 自定义 RememberMe 实现

如果需要更复杂的 RememberMe 功能，可以自定义实现：

### 场景：使用 Redis 存储 Token

```java
public class RedisRememberMeManager implements RememberMeManager {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String TOKEN_PREFIX = "remember:";
    private static final long TOKEN_EXPIRE = 604800;  // 7 天

    @Override
    public void rememberIdentity(SubjectContext subjectContext) {
        PrincipalCollection principals = subjectContext.getPrincipals();
        String username = principals.getPrimaryPrincipal().toString();

        // 生成随机 Token
        String token = UUID.randomUUID().toString();

        // 存入 Redis
        String key = TOKEN_PREFIX + token;
        redisTemplate.opsForValue().set(key, username, TOKEN_EXPIRE, TimeUnit.SECONDS);

        // 写入 Cookie
        Cookie cookie = new SimpleCookie("rememberMe");
        cookie.setValue(token);
        cookie.saveToRequest(subjectContext);
    }

    @Override
    public PrincipalCollection getRememberedIdentity(SubjectContext subjectContext) {
        // 从 Cookie 读取 Token
        String token = readToken(subjectContext);
        if (token == null) {
            return null;
        }

        // 从 Redis 获取用户名
        String key = TOKEN_PREFIX + token;
        String username = redisTemplate.opsForValue().get(key);
        if (username == null) {
            return null;
        }

        // 续期
        redisTemplate.expire(key, TOKEN_EXPIRE, TimeUnit.SECONDS);

        return new SimplePrincipalCollection(username, "redisRealm");
    }
}
```

## 面试高频问题

**问：RememberMe 的 Cookie 被窃取了怎么办？**

RememberMe Cookie 使用 AES 加密，即使被窃取，攻击者也无法解密。但如果密钥泄露，攻击者可以伪造任意用户的 Cookie。缓解措施：

1. Cookie 设置 `Secure=true`，只在 HTTPS 传输
2. Cookie 设置 `HttpOnly=true`，防止 XSS 读取
3. 定期更换加密密钥
4. 敏感操作强制重新认证

**问：RememberMe 和 Session 有什么区别？**

| 对比项 | RememberMe | Session |
|---|---|---|
| 存储位置 | 浏览器 Cookie | 服务器内存/Redis |
| 有效期 | 较长（7-30 天） | 较短（30 分钟） |
| 安全性 | 较低（Cookie 可被窃取） | 较高（存服务器） |
| 用途 | 自动登录 | 状态保持 |

**问：如何防止 RememberMe 被用于权限提升？**

RememberMe 用户只能执行公开操作。敏感操作前必须检查 `isAuthenticated()`，而不是 `isRemembered()`。

## 总结

RememberMe 是用户体验和安全的平衡：

```
RememberMe 使用原则：
├── Cookie 安全：HttpOnly + Secure + SameSite
├── 加密密钥：定期更换，不要硬编码
├── 过期时间：不超过 7 天
├── 敏感操作：必须检查 isAuthenticated()
└── Token 方案：可用 Redis 替代 Cookie
```
