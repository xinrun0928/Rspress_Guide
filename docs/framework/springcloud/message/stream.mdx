# Spring Cloud Stream：屏蔽 MQ 底层的消息框架

import { Badge } from '@rspress/core/theme';

Kafka 和 RabbitMQ 的 API 完全不同，怎么写出可以切换 MQ 的代码？

这是一个很现实的问题。你的系统现在用的是 Kafka，但业务方说想试试 RabbitMQ。更糟糕的是，Kafka 的 API 和 RabbitMQ 的 API 完全不同——如果直接写业务代码，后期切换 MQ 几乎等于重写。

**Spring Cloud Stream 的出现，就是为了解决这个问题。**

## 一、Binder 抽象：统一 MQ 接口

### 1.1 为什么需要 Binder

传统使用 MQ 的方式，代码直接依赖特定的 MQ 客户端：

```java
// Kafka 版本
@Autowired
private KafkaTemplate kafkaTemplate;
kafkaTemplate.send("topic", message);

// RabbitMQ 版本
@Autowired
private RabbitTemplate rabbitTemplate;
rabbitTemplate.convertAndSend("exchange", "routingKey", message);
```

如果后期要切换 MQ，代码要改很多——而且业务代码和 MQ 耦合在一起了。

### 1.2 Binder 的设计

Spring Cloud Stream 提出了 Binder 抽象层：

```
┌─────────────────────────────────────────────────────┐
│                    应用代码                          │
│  （使用统一的 Spring Cloud Stream API）              │
└─────────────────────────┬───────────────────────────┘
                          │
                          ↓
┌─────────────────────────────────────────────────────┐
│                    Binder 抽象层                     │
│  ┌─────────────┬─────────────┬─────────────┐       │
│  │    Kafka    │   RabbitMQ  │  RocketMQ   │       │
│  │   Binder    │    Binder   │   Binder    │       │
│  └─────────────┴─────────────┴─────────────┘       │
└─────────────────────────┬───────────────────────────┘
                          │
                          ↓
┌─────────────────────────────────────────────────────┐
│                   MQ 集群                           │
│  ┌─────────────┬─────────────┬─────────────┐       │
│  │    Kafka    │  RabbitMQ   │  RocketMQ   │       │
│  │   Cluster   │   Cluster   │   Cluster   │       │
│  └─────────────┴─────────────┴─────────────┘       │
└─────────────────────────────────────────────────────┘
```

Binder 是消息队列的抽象层，通过配置决定使用哪种 MQ，代码无需改动。

### 1.3 支持的 Binder

| Binder | 依赖 | 说明 |
| --- | --- | --- |
| Kafka | `spring-cloud-stream-binder-kafka` | Apache Kafka |
| RabbitMQ | `spring-cloud-stream-binder-rabbit` | RabbitMQ |
| RocketMQ | `spring-cloud-starter-stream-rocketmq` | Alibaba RocketMQ |
| Amazon Kinesis | `spring-cloud-stream-binder-kinesis` | AWS Kinesis |
| Google Pub/Sub | `spring-cloud-gcp-starter-binder-pubsub` | GCP Pub/Sub |

## 二、@Input / @Output 注解

### 2.1 定义输入输出通道

Spring Cloud Stream 通过注解定义消息通道：

```java
// 定义通道接口
public interface MyChannels {
    
    // 输入通道（消费消息）
    @Input("input-channel")
    SubscribableChannel input();
    
    // 输出通道（发送消息）
    @Output("output-channel")
    MessageChannel output();
}
```

### 2.2 简单接口

Stream 提供了两个简单接口：

```java
// Source：只能发送消息
public interface Source {
    String OUTPUT = "output";
    @Output(Source.OUTPUT)
    MessageChannel output();
}

// Sink：只能消费消息
public interface Sink {
    String INPUT = "input";
    @Input(Sink.INPUT)
    SubscribableChannel input();
}

// Processor：既能发送也能消费
public interface Processor extends Source, Sink {
}
```

### 2.3 使用示例

```java
@EnableBinding(MyChannels.class)
@Service
public class MyProcessor {
    
    @Autowired
    private MyChannels channels;
    
    public void process(String message) {
        // 发送消息到 output-channel
        channels.output().send(MessageBuilder.withPayload(message).build());
    }
    
    // 使用 @StreamListener 监听输入
    @StreamListener("input-channel")
    public void handleMessage(String message) {
        System.out.println("Received: " + message);
    }
}
```

## 三、@EnableBinding 和 @StreamListener

### 3.1 @EnableBinding

`@EnableBinding` 激活 Channel 绑定：

```java
// 绑定 Source 和 Sink
@EnableBinding(Source.class, Sink.class)
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

// 或者绑定自定义接口
@EnableBinding(MyChannels.class)
public class Application {
    // ...
}
```

### 3.2 @StreamListener

`@StreamListener` 监听消息，简化消费代码：

```java
@StreamListener(Sink.INPUT)
public void handleMessage(String message) {
    System.out.println("Received: " + message);
}

// 使用 @Payload 指定消息体
@StreamListener(Sink.INPUT)
public void handleOrder(@Payload Order order) {
    orderService.process(order);
}

// 使用 @Header 获取消息头
@StreamListener(Sink.INPUT)
public void handleMessage(@Payload Order order, 
                          @Header("Content-Type") String contentType) {
    System.out.println("Content-Type: " + contentType);
}
```

### 3.3 消息过滤

使用 `@StreamListener` 还能实现条件过滤：

```java
// 只处理 type=order 的消息
@StreamListener(value = Sink.INPUT, condition = "headers['type']=='order'")
public void handleOrder(Order order) {
    orderService.process(order);
}

// 处理 type=payment 的消息
@StreamListener(value = Sink.INPUT, condition = "headers['type']=='payment'")
public void handlePayment(Payment payment) {
    paymentService.process(payment);
}
```

## 四、Function 接口：现代化的编程模型

### 4.1 Supplier / Function / Consumer

从 Spring Cloud Stream 3.0 开始，推荐使用 Java 8 的 Function 接口，代码更简洁：

```java
@SpringBootApplication
@EnableBinding(Processor.class)
public class Application {
    
    // 消费者
    @Bean
    public Consumer<String> input() {
        return message -> {
            System.out.println("Received: " + message);
        };
    }
    
    // 处理器
    @Bean
    public Function<String, String> process() {
        return message -> {
            return message.toUpperCase();
        };
    }
}
```

### 4.2 Function 组合

可以组合多个 Function 实现流式处理：

```java
@SpringBootApplication
@EnableBinding(Processor.class)
public class Application {
    
    @Bean
    public Function<String, String> uppercase() {
        return String::toUpperCase;
    }
    
    @Bean
    public Function<String, String> addPrefix() {
        return message -> "PREFIX: " + message;
    }
}
```

### 4.3 Supplier：消息生产者

```java
@SpringBootApplication
@EnableBinding(Source.class)
public class Application {
    
    @Bean
    public Supplier<String> generate() {
        return () -> "Hello from Supplier at " + System.currentTimeMillis();
    }
}
```

## 五、分组和分区

### 5.1 消费者组

消费者组是 Stream 的重要概念：

| 场景 | 行为 |
| --- | --- |
| 同一消费者组 | 消息只被组内一个消费者消费（负载均衡） |
| 不同消费者组 | 消息被每个组消费（广播） |

```yaml title="application.yml"
spring:
  cloud:
    stream:
      bindings:
        input:
          destination: my-topic
          group: group-a  # 消费者组
```

### 5.2 分区

分区确保消息有序或按特定规则分配：

```java
// 发送时指定分区键
MessageBuilder.withPayload(order)
    .setHeader("partitionKey", order.getUserId())
    .build();
```

```yaml title="application.yml"
spring:
  cloud:
    stream:
      bindings:
        output:
          producer:
            partition-key-expression: headers['partitionKey']
            partition-count: 3
        input:
          consumer:
            partitioned: true
            instance-index: 0
```

## 六、总结

Spring Cloud Stream 通过 Binder 抽象屏蔽了 MQ 底层差异，使得应用可以在不改变代码的情况下切换消息队列。
