# Bus + Nacos：配置中心与消息总线的联合

import { Badge } from '@rspress/core/theme';

Nacos 作为配置中心，Spring Cloud Bus 作为消息总线，两者结合可以实现配置的自动刷新。

这是生产环境中常用的组合——Nacos 负责配置的存储和管理，Bus 负责配置的广播刷新。

## 一、Nacos 配置中心 + Spring Cloud Bus

### 1.1 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                     Nacos Server                             │
│  ┌─────────────┬─────────────┬─────────────┐               │
│  │ 配置存储     │ 配置推送     │ 变更通知    │               │
│  └─────────────┴─────────────┴─────────────┘               │
└──────────────────────────┬────────────────────────────────┘
                           │
              ┌────────────┴────────────┐
              │                         │
              ▼                         ▼
┌──────────────────────┐   ┌──────────────────────┐
│    RabbitMQ           │   │    RabbitMQ           │
│    Cluster            │   │    Cluster            │
└──────────────────────┬─┘   └──────────────────────┬─┘
                       │                             │
         ┌─────────────┴─────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│                  应用集群                                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ 实例 1   │  │ 实例 2   │  │ 实例 3   │  │ 实例 N   │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 工作流程

1. 用户在 Nacos 控制台修改配置
2. Nacos Server 推送配置变更到任意一个应用实例
3. 该实例收到变更后，向 RabbitMQ 发布刷新事件
4. RabbitMQ 广播刷新事件到所有应用实例
5. 所有实例执行 `@RefreshScope` 刷新

## 二、配置变更事件的传播链

### 2.1 完整的传播链路

```
用户修改 Nacos 配置
        ↓
Nacos Server 推送给 Config Client
        ↓
Config Client 收到推送
        ↓
ContextRefresher.refreshEnvironment()
        ↓
Spring Cloud Bus 发布 RefreshRemoteApplicationEvent
        ↓
RabbitMQ 广播到所有实例
        ↓
每个实例的 ApplicationListener 收到事件
        ↓
@RefreshScope 清空缓存
        ↓
配置生效
```

### 2.2 关键组件

| 组件 | 作用 |
| --- | --- |
| `NacosConfigAutoConfiguration` | Nacos 配置自动装配 |
| `NacosContextRefresher` | 监听 Nacos 配置变更 |
| `BusAutoConfiguration` | Bus 自动装配 |
| `RefreshBusListener` | 监听远程刷新事件 |
| `@RefreshScope` | 标记需要刷新的 Bean |

## 三、@KafkaListener / @RabbitListener 与 Stream 的对比

### 3.1 Spring AMQP

直接使用 RabbitMQ 的注解：

```java
@Configuration
public class RabbitMQConfig {
    
    @Bean
    public Queue orderQueue() {
        return new Queue("order-queue", true);
    }
    
    @Bean
    public Queue paymentQueue() {
        return new Queue("payment-queue", true);
    }
}

@Component
public class MessageListener {
    
    @RabbitListener(queues = "order-queue")
    public void handleOrder(String message) {
        System.out.println("处理订单消息: " + message);
    }
    
    @RabbitListener(queues = "payment-queue")
    public void handlePayment(String message) {
        System.out.println("处理支付消息: " + message);
    }
}
```

### 3.2 Spring Cloud Stream

使用 Stream 的统一抽象：

```java
@EnableBinding(Sink.class)
@Component
public class StreamListener {
    
    @StreamListener(Sink.INPUT)
    public void handle(String message) {
        System.out.println("处理消息: " + message);
    }
}
```

### 3.3 对比

| 维度 | Spring AMQP | Spring Cloud Stream |
| --- | --- | --- |
| MQ 依赖 | 直接依赖 RabbitMQ/Kafka | 通过 Binder 抽象 |
| API 风格 | Spring AMQP 原生 API | 统一的消息编程模型 |
| 多 MQ 支持 | 不支持 | 支持多种 MQ |
| 适用场景 | 单一 MQ | 需要切换 MQ 或多 MQ 共存 |

## 四、生产环境最佳实践

### 4.1 高可用配置

```yaml title="application.yml"
spring:
  cloud:
    bus:
      enabled: true
      trace:
        enabled: true
    nacos:
      config:
        server-addr: nacos-cluster:8848
        # 配置命名空间
        namespace: ${NACOS_NAMESPACE:prod}
        # 配置分组
        group: ${NACOS_GROUP:DEFAULT_GROUP}
      discovery:
        server-addr: nacos-cluster:8848
  rabbitmq:
    host: rabbitmq-cluster
    port: 5672
    # 高可用配置
    connection-timeout: 3000
    heartbeats: 60
    template:
      retry:
        enabled: true
        initial-interval: 1000
        max-attempts: 3
```

### 4.2 监控配置

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,bus-refresh,bus-env
  endpoint:
    health:
      show-details: always
```

## 五、总结

Bus + Nacos 的组合是生产环境中配置管理的最佳实践：

- **Nacos**：提供配置的存储、版本管理、变更推送
- **Spring Cloud Bus**：通过消息队列广播刷新事件
- **@RefreshScope**：自动刷新配置 Bean

两者结合，实现了配置的集中管理和集群内的实时同步。
