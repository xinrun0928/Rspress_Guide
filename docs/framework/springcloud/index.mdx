# Spring Cloud 全家桶

微服务架构的落地，从来不是「把一个巨石应用拆成多个小服务」这么简单。服务拆分之后，你面临的是：服务之间如何互相发现？配置变了怎么办？网关怎么统一入口？某个服务挂了怎么不让整个系统崩溃？链路怎么追踪？

Spring Cloud 给出的答案是一套完整的微服务治理生态——从服务注册发现、配置管理、负载均衡、熔断降级，到网关路由、消息总线、分布式事务，覆盖了微服务落地的每一个环节。

## 与 Spring Boot 的关系

很多人搞不清楚 Spring Cloud 和 Spring Boot 的关系，这里澄清一下：

**Spring Boot** 是快速构建单个 Spring 应用的脚手架，专注于「怎么更快地写一个应用」。

**Spring Cloud** 是构建分布式系统的全家桶，专注于「多个应用怎么协同工作」。

换个角度说：**Spring Cloud 是规范，Spring Boot 是实现**。Spring Cloud 定义了一套分布式系统的标准接口（如服务注册、负载均衡、熔断），而 Spring Boot 应用通过集成具体组件（Eureka、Nacos、Sentinel）来落地这些标准。

这也是为什么 Spring Cloud 的版本命名和 Spring Boot 是两套独立的体系，但它们之间有严格的版本对应关系。

## 面试考察分析

Spring Cloud 在面试中几乎是 Java 后端必考的内容，不同工作年限问的深度不同：

| 考察维度 | 常见问题 | 考察重点 |
|---|---|---|
| **基础概念** | Spring Cloud 和 Dubbo 的区别？ | 技术选型能力 |
| **服务注册** | Nacos 的 Namespace/Group/Data ID 三层隔离怎么理解？ | 配置管理能力 |
| **配置刷新** | @RefreshScope 的原理是什么？ | 原理深度 |
| **服务调用** | OpenFeign 和 Ribbon 是什么关系？ | 组件理解 |
| **网关** | Gateway 的 Predicates 和 Filters 有什么区别？ | 路由机制 |
| **容错** | Sentinel 和 Hystrix 的区别？滑动窗口怎么实现的？ | 原理深度 |
| **分布式事务** | Seata AT 模式的两阶段提交是怎么工作的？ | 分布式事务理解 |

这里有一个规律——**工作 1-3 年问配置和使用，3-5 年问原理，5 年以上问设计**。所以不要只停留在「会用」，要理解每个组件「为什么这样设计」。

## 学习路径建议

建议按照以下顺序学习 Spring Cloud 全家桶：

```
1. 基础篇：理解版本关系和组件职责（basic/）
   ↓
2. 注册配置：Nacos 的服务发现和配置中心（registry/）
   ↓
3. 服务调用：OpenFeign 声明式调用 + Gateway 网关（gateway/）
   ↓
4. 容错治理：Sentinel 流量控制和熔断降级（resilience/）
   ↓
5. 消息异步：Stream 屏蔽 MQ 差异 + Bus 配置广播（message/）
   ↓
6. 实战进阶：分布式事务 Seata + 统一认证 OAuth2（practice/）
```

每个模块都有对应的面试高频问题汇总，学完之后可以用 `practice/interview-summary` 来检验自己的掌握程度。

准备好了吗？让我们开始探索 Spring Cloud 的世界。
