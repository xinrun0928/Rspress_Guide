# OpenFeign：像调用本地方法一样调用远程服务

Ribbon + RestTemplate 的写法太繁琐，OpenFeign 改变了什么？

想象一下这个场景：你要调用订单服务的接口获取订单详情。用传统方式，你需要这样写：

```java
@Autowired
private RestTemplate restTemplate;

public Order getOrder(Long id) {
    return restTemplate.getForObject(
        "http://order-service/api/orders/" + id,
        Order.class
    );
}
```

每次调用都要写一长串 URL，参数拼接容易出错，接口定义不直观——你能一眼看出这个方法调的是哪个服务吗？

**OpenFeign 改变了这一切。**

---

## 一、@FeignClient 的基本使用

### 1.1 最简单的例子

使用 OpenFeign，你只需要定义一个接口：

```java
@FeignClient(name = "order-service")
public interface OrderClient {
    
    @GetMapping("/api/orders/{id}")
    Order getOrder(@PathVariable("id") Long id);
}
```

然后像使用本地 Bean 一样注入并调用：

```java
@Service
public class OrderServiceImpl {
    
    @Autowired
    private OrderClient orderClient;
    
    public Order getOrderDetail(Long id) {
        // 像调用本地方法一样调用远程服务
        return orderClient.getOrder(id);
    }
}
```

对比一下：

| 维度 | RestTemplate | OpenFeign |
| --- | --- | --- |
| 代码量 | 需要写 URL、拼接参数、处理响应 | 只需要定义接口 |
| 类型安全 | 参数拼接容易出错 | 编译期检查 |
| 接口清晰度 | 需要追踪代码才知道调了哪个服务 | 接口名直接标注服务名 |
| 可读性 | 一般 | 高 |

### 1.2 @FeignClient 的核心属性

`@FeignClient` 注解有多个属性，可以灵活配置：

```java
@FeignClient(
    name = "order-service",              // 服务名（必填）
    url = "https://api.example.com",    // 指定 URL（可选，用于测试环境直连）
    fallback = OrderClientFallback.class, // 降级处理类
    fallbackFactory = OrderFallbackFactory.class, // 降级工厂（可获取异常）
    configuration = FeignConfig.class,   // 自定义配置类
    path = "/orders"                     // 路径前缀
)
public interface OrderClient {
    // ...
}
```

| 属性 | 说明 | 注意事项 |
| --- | --- | --- |
| `name` | 服务名，用于从注册中心获取实例 | 必填，与 `value` 互为别名 |
| `url` | 固定 URL，通常用于测试环境直连 | 设置后不会从注册中心获取 |
| `fallback` | 熔断降级类 | 需要开启 Hystrix |
| `fallbackFactory` | 可获取异常原因的降级工厂 | 优先于 `fallback` |
| `configuration` | Feign 的自定义配置类 | 可以覆盖默认配置 |
| `path` | 接口方法的路径前缀 | 拼接在每个方法路径前 |

## 二、SpringMvcContract 注解解析

### 2.1 为什么用 Spring MVC 注解

OpenFeign 支持两种契约（Contract）：

1. **默认契约（Feign Contract）**：使用 `@RequestLine`、`@Headers`、`@Param` 等原生注解
2. **Spring MVC 契约**：使用 `@GetMapping`、`@PostMapping`、`@RequestParam` 等 Spring MVC 注解

Spring Cloud OpenFeign 默认使用 Spring MVC 契约，这意味着你可以用 Spring MVC 的注解来定义 Feign 接口：

```java
@FeignClient(name = "order-service")
public interface OrderClient {
    
    // Spring MVC 注解：简洁直观
    @GetMapping("/api/orders/{id}")
    Order getOrder(@PathVariable("id") Long id);
    
    @PostMapping("/api/orders")
    Order createOrder(@RequestBody CreateOrderRequest request);
    
    @GetMapping("/api/orders")
    Page<Order> listOrders(
        @RequestParam("page") int page,
        @RequestParam("size") int size,
        @RequestHeader("Authorization") String token
    );
}
```

### 2.2 注解解析原理

`SpringMvcContract` 负责解析 Spring MVC 注解，将其转换为 Feign 的请求模板：

```java
// SpringMvcContract 的简化解析逻辑
public class SpringMvcContract extends Contract {
    
    @Override
    public MethodMetadata parseAndValidateMetadata(Method method) {
        // 1. 解析 @RequestMapping 系列注解
        RequestMapping mapping = method.getAnnotation(RequestMapping.class);
        RequestMethod httpMethod = mapping.method()[0];
        String path = mapping.path()[0];
        
        // 2. 构建请求模板
        MethodMetadata metadata = new MethodMetadata();
        metadata.template().method(HttpMethod.valueOf(httpMethod.name()));
        metadata.template().uri(path);
        
        // 3. 解析方法参数
        Annotation[][] paramAnnotations = method.getParameterAnnotations();
        for (int i = 0; i < paramAnnotations.length; i++) {
            Annotation[] annotations = paramAnnotations[i];
            for (Annotation annotation : annotations) {
                if (annotation instanceof PathVariable) {
                    // 解析 @PathVariable
                    PathVariable pathVariable = (PathVariable) annotation;
                    metadata.template().uri(
                        replaceUriTemplate(
                            metadata.template().uri().toString(),
                            pathVariable.value(),
                            method.getParameters()[i].getName()
                        )
                    );
                } else if (annotation instanceof RequestParam) {
                    // 解析 @RequestParam
                    RequestParam requestParam = (RequestParam) annotation;
                    metadata.template().query(requestParam.value(), 
                        "{" + method.getParameters()[i].getName() + "}");
                }
                // ... 其他注解解析
            }
        }
        
        return metadata;
    }
}
```

### 2.3 常用注解对照

| Spring MVC 注解 | Feign 原生注解 | 说明 |
| --- | --- | --- |
| `@GetMapping` | `@RequestLine("GET ...")` | GET 请求 |
| `@PostMapping` | `@RequestLine("POST ...")` | POST 请求 |
| `@PutMapping` | `@RequestLine("PUT ...")` | PUT 请求 |
| `@DeleteMapping` | `@RequestLine("DELETE ...")` | DELETE 请求 |
| `@PathVariable` | `@Param` | URL 路径参数 |
| `@RequestParam` | `@QueryMap` | URL 查询参数 |
| `@RequestHeader` | `@Headers` | 请求头 |
| `@RequestBody` | `@Body` | 请求体 |

## 三、动态代理的生成

### 3.1 代理的创建过程

当你注入一个 Feign Client 时，Spring 返回给你的实际上是一个代理对象。这个代理对象的创建过程如下：

```java
// 简化后的 Feign 代理创建过程
public class FeignClientFactory {
    
    public <T> T create(Class<T> clazz, String name) {
        // 1. 获取 Feign 实例
        Feign feign = feignBuilder(name).build();
        
        // 2. 使用 ReflectiveFeign 创建代理
        ReflectiveFeign.FeignInvocationHandler handler = 
            new ReflectiveFeign.FeignInvocationHandler(feign);
        
        // 3. 创建 JDK 动态代理
        return (T) Proxy.newProxyInstance(
            clazz.getClassLoader(),
            new Class[]{clazz},
            handler
        );
    }
}
```

### 3.2 FeignInvocationHandler 的 invoke 方法

当调用 Feign 接口的方法时，实际执行的是 `FeignInvocationHandler.invoke()`：

```java
public class FeignInvocationHandler implements InvocationHandler {
    
    private final Map<Method, MethodHandler> dispatch;
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) {
        // 1. 过滤 Object 方法（toString、equals 等）
        if ("equals".equals(method.getName())) {
            return args.length == 1 && Objects.equals(args[0], proxy);
        }
        if ("hashCode".equals(method.getName())) {
            return System.identityHashCode(proxy);
        }
        if ("toString".equals(method.getName())) {
            return "Feign-proxy[" + target + "]";
        }
        
        // 2. 获取对应的 MethodHandler
        MethodHandler handler = dispatch.get(method);
        
        // 3. 执行请求
        return handler.invoke(args);
    }
}
```

### 3.3 SynchronousMethodHandler 的请求执行

`SynchronousMethodHandler` 负责实际发起 HTTP 请求：

```java
public class SynchronousMethodHandler implements MethodHandler {
    
    private final Client client;
    private final Target<?> target;
    private final MethodMetadata metadata;
    private final Logger logger;
    private final Level logLevel;
    
    @Override
    public Object invoke(Object[] args) throws Throwable {
        // 1. 构建请求模板
        RequestTemplate template = buildTemplate(args);
        
        // 2. 重试处理
        Retryer retryer = this.retryer.clone();
        while (true) {
            try {
                // 3. 发起请求
                Response response = client.execute(
                    Request.create(Request.HttpMethod.GET, 
                        target.url(template), 
                        template.headers())
                );
                
                // 4. 解析响应
                return decode(response);
            } catch (RetryableException e) {
                // 5. 如果可重试，则重试
                retryer.continueOrPropagate(e);
            }
        }
    }
}
```

## 四、Ribbon 的集成

### 4.1 自动集成

Spring Cloud OpenFeign 默认集成了 Ribbon，无需额外配置：

```yaml title="application.yml"
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            # 全局配置
            connectTimeout: 5000
            readTimeout: 5000
          order-service:
            # 针对 order-service 的配置
            connectTimeout: 3000
            readTimeout: 10000
```

### 4.2 负载均衡的原理

Feign 底层使用 Ribbon 进行负载均衡。当发起请求时：

1. **服务名解析**：从 `@FeignClient(name = "order-service")` 中提取服务名
2. **获取实例列表**：通过 `LoadBalancerFeignClient` 向注册中心查询服务实例
3. **负载均衡选择**：Ribbon 根据配置的策略选择一个实例
4. **发起请求**：将服务名替换为选中的实例地址

```java
// LoadBalancerFeignClient 的核心逻辑
public class LoadBalancerFeignClient implements Client {
    
    @Override
    public Response execute(Request request, Request.Options options) throws IOException {
        // 1. 提取 URL 中的服务名
        URL url = URL.parse(request.url());
        String clientName = url.getHost();
        
        // 2. 使用 Ribbon 获取目标服务器
        Server lbServer = loadBalancer.chooseServer(clientName);
        
        // 3. 构建新的请求 URL
        String newUrl = UriComponentsBuilder
            .fromHttpUrl(request.url())
            .host(lbServer.getHost())
            .port(lbServer.getPort())
            .build()
            .toUri()
            .toString();
        
        // 4. 发起真正的请求
        return delegate.execute(Request.create(request.method, newUrl, ...), options);
    }
}
```

### 4.3 负载均衡策略配置

```yaml title="application.yml"
# 全局配置
ribbon:
  # 负载均衡策略
  NFLoadBalancerRuleClassName: com.netflix.loadbalancer.RoundRobinRule
  # 连接超时
  ConnectTimeout: 3000
  # 读取超时
  ReadTimeout: 5000
  # 重试次数
  MaxAutoRetries: 1
  # 下一个服务器的重试次数
  MaxAutoRetriesNextServer: 2

# 针对特定服务配置
order-service:
  ribbon:
    NFLoadBalancerRuleClassName: com.netflix.loadbalancer.WeightedResponseTimeRule
```

## 五、超时配置

### 5.1 两个超时时间

Feign 有两个超时配置：

| 配置 | 说明 | 默认值 |
| --- | --- | --- |
| `connectTimeout` | 连接超时，等待建立 TCP 连接的时间 | 10 秒 |
| `readTimeout` | 读取超时，等待响应的最大时间 | 60 秒 |

```yaml title="application.yml"
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connectTimeout: 3000      # 连接超时 3 秒
            readTimeout: 10000        # 读取超时 10 秒
```

### 5.2 超时导致的问题

超时配置不合理会导致各种问题：

- **连接超时过短**：服务启动时，注册中心还没返回实例列表就超时了
- **读取超时过短**：复杂查询还没执行完就超时
- **连接超时过长**：服务不可用时，用户等待时间过长
- **读取超时过长**：单个慢请求占用线程太久

> [!WARNING]
> 超时配置不是越小越好，要根据实际业务场景调整。可以用 `logger.level.feign` 开启日志，逐步调优。

```java
@Configuration
public class FeignLogConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        // 可选值：NONE、BASIC、HEADERS、BODY
        return Logger.Level.FULL;
    }
}
```

| 日志级别 | 说明 | 输出内容 |
| --- | --- | --- |
| `NONE` | 不记录 | 无 |
| `BASIC` | 基本信息 | 请求方法、URL、响应状态码 |
| `HEADERS` | 请求头和响应头 | 请求头、响应头 |
| `BODY` | 完整内容 | 请求体、响应体 |

## 七、面试追问方向

- **OpenFeign 的工作原理是什么？** 通过动态代理生成接口实现，解析注解构建请求模板，使用 Ribbon 进行负载均衡。
- **@FeignClient 和 @EnableFeignClients 的区别是什么？** @FeignClient 标注接口，@EnableFeignClients 开启 Feign 扫描并注册这些接口。
- **Feign 和 Ribbon 是什么关系？** Feign 封装了 Ribbon，Feign 的请求最终由 Ribbon 进行负载均衡。
- **OpenFeign 如何实现请求重试？** 通过 Retryer 实现，默认会对连接失败和 5xx 错误重试。

## 八、总结

OpenFeign 通过**声明式接口**和**动态代理**，让远程服务调用变得像本地方法调用一样简单：

- **@FeignClient**：标注接口，声明要调用的服务
- **SpringMvcContract**：解析 Spring MVC 注解，构建请求模板
- **ReflectiveFeign**：创建 JDK 动态代理
- **SynchronousMethodHandler**：执行 HTTP 请求
- **Ribbon 集成**：自动完成服务发现和负载均衡

理解这套机制，才能在遇到问题时快速定位原因，也才能更好地使用 OpenFeign 的高级特性。
