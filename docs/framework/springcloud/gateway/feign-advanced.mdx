# OpenFeign 高级特性：传参与继承的艺术

import { Badge } from '@rspress/core/theme';

如果说 `@FeignClient` 是 OpenFeign 的门面，那参数传递就是它的内功。

很多开发者以为 OpenFeign 只能传递简单参数，但实际上，它支持完整的参数传递机制——路径参数、查询参数、请求头、请求体，甚至还有继承这种高级玩法。掌握了这些，你才能真正用好 OpenFeign。

## 一、@PathVariable、@RequestParam、@RequestHeader

### 1.1 @PathVariable：路径参数

URL 路径中的参数，使用 `@PathVariable`：

```java
@FeignClient(name = "order-service")
public interface OrderClient {
    
    @GetMapping("/api/orders/{orderId}")
    Order getOrder(@PathVariable("orderId") Long orderId);
    
    @GetMapping("/api/orders/{orderId}/items/{itemId}")
    OrderItem getOrderItem(
        @PathVariable("orderId") Long orderId,
        @PathVariable("itemId") Long itemId
    );
}
```

> [!TIP]
> 注意：`@PathVariable` 的 `value` 必须指定，不能省略。这是 Feign 的要求，与 Spring MVC 不同。

### 1.2 @RequestParam：查询参数

URL 查询字符串中的参数，使用 `@RequestParam`：

```java
@FeignClient(name = "order-service")
public interface OrderClient {
    
    @GetMapping("/api/orders")
    Page<Order> listOrders(
        @RequestParam("page") int page,
        @RequestParam("size") int size
    );
    
    @GetMapping("/api/orders/search")
    List<Order> searchOrders(
        @RequestParam("keyword") String keyword,
        @RequestParam(value = "status", required = false) String status,
        @RequestParam(value = "sort", defaultValue = "createTime") String sort
    );
}
```

生成的请求 URL：
```
GET /api/orders?page=1&size=10
GET /api/orders/search?keyword=手机&sort=createTime
```

### 1.3 @RequestHeader：请求头

传递 HTTP 请求头的参数，使用 `@RequestHeader`：

```java
@FeignClient(name = "order-service")
public interface OrderClient {
    
    @GetMapping("/api/orders/{orderId}")
    Order getOrder(
        @PathVariable("orderId") Long orderId,
        @RequestHeader("Authorization") String token,
        @RequestHeader("X-Request-Id") String requestId
    );
}
```

生成的请求：
```
GET /api/orders/123
Authorization: Bearer xxx
X-Request-Id: yyy
```

### 1.4 参数映射对照表

| Feign 注解 | 对应 HTTP 位置 | 示例 |
| --- | --- | --- |
| `@PathVariable` | URL 路径 | `GET /orders/{id}` |
| `@RequestParam` | URL 查询参数 | `GET /orders?id=123` |
| `@RequestHeader` | 请求头 | `Authorization: Bearer xxx` |
| `@RequestBody` | 请求体 | `POST /orders` + JSON Body |
| `@CookieValue` | Cookie | `Cookie: JSESSIONID=xxx` |

## 二、@RequestBody 的 JSON 序列化

### 2.1 基本用法

POST、PUT 请求通常需要传递 JSON 请求体：

```java
@FeignClient(name = "order-service")
public interface OrderClient {
    
    @PostMapping("/api/orders")
    Order createOrder(@RequestBody CreateOrderRequest request);
    
    @PutMapping("/api/orders/{orderId}")
    Order updateOrder(
        @PathVariable("orderId") Long orderId,
        @RequestBody UpdateOrderRequest request
    );
}
```

```java
// 创建订单请求
public class CreateOrderRequest {
    private Long userId;
    private List<Long> itemIds;
    private String remark;
    
    // getters and setters
}
```

生成的请求：
```
POST /api/orders
Content-Type: application/json

{"userId": 123, "itemIds": [1, 2, 3], "remark": "尽快发货"}
```

### 2.2 复杂的嵌套对象

OpenFeign 使用 Jackson 进行 JSON 序列化，支持复杂的嵌套对象：

```java
public class CreateOrderRequest {
    private Long userId;
    private List<OrderItem> items;
    private Address address;
    
    // 嵌套对象
    public static class OrderItem {
        private Long productId;
        private Integer quantity;
        // getters and setters
    }
    
    public static class Address {
        private String province;
        private String city;
        private String detail;
        // getters and setters
    }
}
```

序列化为：
```json
{
    "userId": 123,
    "items": [
        {"productId": 1, "quantity": 2},
        {"productId": 2, "quantity": 1}
    ],
    "address": {
        "province": "浙江",
        "city": "杭州",
        "detail": "文三路 123 号"
    }
}
```

### 2.3 日期格式配置

日期序列化需要统一格式，否则会出现格式不一致的问题：

```yaml title="application.yml"
spring:
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: GMT+8
```

## 三、Feign 的继承特性

### 3.1 为什么需要继承

当多个微服务都需要调用同一个服务的 API 时，传统的做法是每个调用方都写一遍接口定义：

```java
// 订单服务定义接口
@FeignClient(name = "order-service")
public interface OrderClient {
    @GetMapping("/api/orders/{id}")
    Order getOrder(@PathVariable("id") Long id);
}

// 用户服务也要调用订单服务
@FeignClient(name = "order-service")
public interface OrderFromUserService {
    @GetMapping("/api/orders/{id}")
    Order getOrder(@PathVariable("id") Long id);
}
```

问题：重复定义，维护困难。如果接口变了，两边都要改。

### 3.2 继承的实现方式

可以将接口定义抽取到一个公共模块：

```java title="order-api 模块"
public interface OrderApi {
    
    @GetMapping("/api/orders/{id}")
    Order getOrder(@PathVariable("id") Long id);
    
    @PostMapping("/api/orders")
    Order createOrder(@RequestBody CreateOrderRequest request);
    
    @GetMapping("/api/orders")
    Page<Order> listOrders(
        @RequestParam("page") int page,
        @RequestParam("size") int size
    );
}
```

```java title="用户服务"
@FeignClient(name = "order-service")
public interface OrderClient extends OrderApi {
    // 自动继承 OrderApi 中的所有方法
}
```

### 3.3 继承的注意事项

继承虽好，但有几个问题需要注意：

| 问题 | 说明 | 解决方案 |
| --- | --- | --- |
| 方法冲突 | 子接口和父接口有同名方法 | 保持简单，不要过度继承 |
| 注解冲突 | 参数注解解析可能出问题 | 确保参数注解完整 |
| 依赖耦合 | 调用方依赖 Provider 的 API 包 | 使用公共 API 模块解耦 |

> [!WARNING]
> 继承特性使用不当会导致接口膨胀和耦合加深。建议只在多服务需要调用同一个 Provider 时使用。

## 四、原生 Feign 注解

### 4.1 @RequestLine 和 @Headers

如果不想用 Spring MVC 注解，可以使用原生 Feign 注解：

```java
@FeignClient(name = "order-service")
public interface OrderClient {
    
    @RequestLine("GET /api/orders/{orderId}")
    Order getOrder(@Param("orderId") Long orderId);
    
    @RequestLine("GET /api/orders")
    List<Order> listOrders(@QueryMap Map<String, Object> params);
    
    @RequestLine("POST /api/orders")
    Order createOrder(@Body CreateOrderRequest request);
    
    @Headers("Authorization: {token}")
    @RequestLine("GET /api/orders/{orderId}")
    Order getOrderWithAuth(
        @Param("orderId") Long orderId,
        @Param("token") String token
    );
}
```

| 注解 | 说明 |
| --- | --- |
| `@RequestLine` | 定义请求方法和路径 |
| `@Param` | 路径参数或模板变量 |
| `@QueryMap` | 查询参数（Map 形式） |
| `@Body` | 请求体 |
| `@Headers` | 请求头，支持模板变量 |

### 4.2 何时使用原生注解

| 场景 | 推荐注解 |
| --- | --- |
| Spring Cloud 项目 | Spring MVC 注解 |
| 非 Spring 项目 | 原生 Feign 注解 |
| 需要高定制化 | 原生 Feign 注解 |

## 五、Encoder 和 Decoder 的自定义

### 5.1 默认的编解码器

OpenFeign 默认使用 `SpringEncoder` 和 `SpringDecoder`（如果引入了 Spring MVC），支持 JSON 序列化。如果需要支持其他格式（如 XML、Protobuf），可以自定义编解码器。

### 5.2 自定义 Encoder

```java
public class XmlEncoder implements Encoder {
    
    private final XStream xStream = new XStream();
    
    @Override
    public void encode(Object object, Type bodyType, RequestTemplate template) {
        // 将对象序列化为 XML
        String xml = xStream.toXML(object);
        template.body(xml, StandardCharsets.UTF_8);
    }
}
```

### 5.3 自定义 Decoder

```java
public class XmlDecoder implements Decoder {
    
    private final XStream xStream = new XStream();
    
    @Override
    public Object decode(Response response, Type type) throws IOException {
        String xml = response.body().asString(StandardCharsets.UTF_8);
        return xStream.fromXML(xml);
    }
}
```

### 5.4 配置自定义编解码器

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Encoder xmlEncoder() {
        return new XmlEncoder();
    }
    
    @Bean
    public Decoder xmlDecoder() {
        return new XmlDecoder();
    }
}
```

## 六、错误解码器（ErrorDecoder）

### 6.1 默认行为

默认情况下，Feign 对 4xx、5xx 响应会抛出 `FeignException`：

```java
// 400 Bad Request
throw new FeignException.BadRequest(...)

// 404 Not Found
throw new FeignException.NotFound(...)

// 500 Internal Server Error
throw new FeignException.InternalServerError(...)
```

### 6.2 自定义 ErrorDecoder

可以自定义错误处理逻辑：

```java
public class CustomErrorDecoder implements ErrorDecoder {
    
    private final ErrorDecoder defaultDecoder = new Default();
    
    @Override
    public Exception decode(String methodKey, Response response) {
        // 读取响应体
        String body = Util.toString(response.body().asReader(Util.UTF_8));
        
        if (response.status() == 400) {
            // 自定义处理 400 错误
            return new BusinessException("BAD_REQUEST", body);
        }
        
        if (response.status() == 401) {
            // 自定义处理 401 错误
            return new UnauthorizedException("未授权");
        }
        
        if (response.status() == 500) {
            // 自定义处理 500 错误
            return new ServerException("服务器内部错误");
        }
        
        return defaultDecoder.decode(methodKey, response);
    }
}
```

### 6.3 配置 ErrorDecoder

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public ErrorDecoder errorDecoder() {
        return new CustomErrorDecoder();
    }
}
```

## 七、重试机制（Retryer）

### 7.1 默认重试策略

Feign 默认使用 `Retryer.Default`，重试策略如下：

- 对 `RetryableException`（连接超时、读取超时、5xx 错误）进行重试
- 默认最多重试 5 次（初始请求 + 4 次重试）
- 重试间隔：100ms，后续重试间隔翻倍

### 7.2 自定义 Retryer

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Retryer retryer() {
        // 重试 3 次，间隔 200ms
        return new Retryer.Default(200, 2000, 3);
    }
}
```

### 7.3 关闭重试

如果业务上不需要重试，可以禁用：

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Retryer retryer() {
        // 不重试
        return Retryer.NEVER_RETRY;
    }
}
```

## 八、面试追问方向

- **Feign 如何传递复杂的请求参数？** 支持 `@RequestBody` 传递 JSON，Jackson 自动序列化；支持 `@RequestParam` 传递查询参数。
- **Feign 的继承特性有什么用？** 抽取公共接口，让多个 Feign Client 复用，减少重复代码。
- **Feign 如何自定义编解码器？** 实现 `Encoder` 和 `Decoder` 接口，注入到 `@Configuration` 类中。
- **Feign 的重试机制是怎样的？** 默认对连接异常和 5xx 错误重试，可通过 `Retryer` 自定义。

## 九、总结

OpenFeign 的高级特性让它不仅仅是"简化远程调用"，而是成为了一套完整的声明式 HTTP 客户端框架：

- **参数传递**：完整的 HTTP 参数支持，包括路径、查询、请求头、请求体
- **JSON 序列化**：自动使用 Jackson 序列化复杂对象
- **继承特性**：通过公共 API 模块减少重复定义
- **自定义编解码**：支持 XML、Protobuf 等多种格式
- **错误处理**：通过 `ErrorDecoder` 自定义错误处理逻辑
- **重试机制**：通过 `Retryer` 控制重试策略

掌握这些高级特性，才能用好 OpenFeign，应对各种复杂的业务场景。
