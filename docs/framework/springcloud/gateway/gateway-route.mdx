# Spring Cloud Gateway 路由：请求的第一步去哪里

import { Badge } from '@rspress/core/theme';

所有请求都经过网关，网关怎么知道该把请求路由到哪个服务？

这是网关最核心的问题。当一个请求进入 Gateway，它首先要做的事情就是：**匹配路由规则**，决定这个请求应该被路由到哪里去。

理解 Gateway 的路由匹配机制，是掌握 Spring Cloud Gateway 的第一步。

## 一、RoutePredicateHandlerMapping 的路由匹配

### 1.1 请求处理的入口

当一个 HTTP 请求进入 Gateway 时，首先由 `RoutePredicateHandlerMapping` 处理：

```java
// RoutePredicateHandlerMapping 的核心逻辑
public class RoutePredicateHandlerMapping extends AbstractHandlerMapping {
    
    @Override
    protected Mono<Object> getHandlerInternal(ServerWebExchange exchange) {
        // 1. 获取请求路径
        String path = exchange.getRequest().getPath().pathWithinApplication().value();
        
        // 2. 匹配路由
        return lookupRoute(path, exchange)
            .flatMap(route -> {
                // 3. 找到路由后，设置属性并返回
                exchange.getAttributes().put(GATEWAY_HANDLER_ATTR, route);
                return Mono.just(getHandler(route));
            });
    }
}
```

### 1.2 路由匹配的过程

`lookupRoute` 方法遍历所有路由，找出第一个匹配的：

```java
private Mono<Route> lookupRoute(String path, ServerWebExchange exchange) {
    return Flux.fromIterable(routeLocator.getRoutes())
        // 过滤出匹配的路由
        .filter(route -> {
            Predicate<ServerWebExchange> predicate = route.getPredicate();
            return predicate.test(exchange);
        })
        // 按优先级排序
        .sort(Comparator.comparing(Route::getOrder))
        // 返回第一个匹配的路由
        .next();
}
```

> [!TIP]
> **路由匹配的顺序很重要**。如果两个路由都匹配，会选择优先级更高（`order` 更小）的那个。建议将更具体的路由放在前面，更通用的路由放在后面。

## 二、Predicates 的 11 种实现

Predicate（谓词）是 Gateway 路由匹配的核心。每个 Predicate 都实现 `ServerWebExchange` → `boolean` 的判断逻辑。

### 2.1 路由谓词工厂一览

| 谓词 | 配置名 | 说明 | 示例 |
| --- | --- | --- | --- |
| After | `After` | 在指定时间之后 | `After=2024-01-01T00:00:00Z` |
| Before | `Before` | 在指定时间之前 | `Before=2024-12-31T23:59:59Z` |
| Between | `Between` | 在两个时间之间 | `Between=2024-01-01,2024-12-31` |
| Cookie | `Cookie` | Cookie 匹配 | `Cookie=name,张三` |
| Header | `Header` | 请求头匹配 | `Header=X-Request-Id,\d+` |
| Host | `Host` | Host 匹配 | `Host=**.example.com` |
| Method | `Method` | HTTP 方法匹配 | `Method=GET,POST` |
| Path | `Path` | 请求路径匹配 | `Path=/api/**` |
| Query | `Query` | 查询参数匹配 | `Query=status,active` |
| RemoteAddr | `RemoteAddr` | 客户端 IP 匹配 | `RemoteAddr=192.168.1.1/24` |
| Weight | `Weight` | 按权重分配 | `Weight=group1,2` / `Weight=group1,8` |

### 2.2 Path：路径匹配

最常用的谓词，匹配请求路径：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: order-route
          uri: lb://order-service
          predicates:
            - Path=/api/orders/**
          filters:
            - StripPrefix=1
```

```java title="Java DSL 方式"
@Bean
public RouteLocator customRouteLocator(RouteLocatorBuilder builder) {
    return builder.routes()
        .route("order-route", r -> r
            .path("/api/orders/**")
            .filters(f -> f.stripPrefix(1))
            .uri("lb://order-service"))
        .build();
}
```

### 2.3 Query：查询参数匹配

根据查询参数进行路由：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: search-route
          uri: https://search.example.com
          predicates:
            # 当有 status 参数且值为 vip 时，路由到 VIP 搜索
            - Query=status,vip
            # 当有 keyword 参数时，路由到搜索服务
            - Query=keyword
```

### 2.4 Method：HTTP 方法匹配

匹配特定的 HTTP 方法：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: api-route
          uri: lb://api-service
          predicates:
            - Method=GET,POST
        - id: admin-route
          uri: lb://admin-service
          predicates:
            - Method=DELETE
```

### 2.5 Header：请求头匹配

根据请求头进行路由：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: mobile-route
          uri: lb://mobile-service
          predicates:
            # X-Client-Type 为 mobile 时路由到移动端服务
            - Header=X-Client-Type,mobile
        - id: web-route
          uri: lb://web-service
          predicates:
            # X-Client-Type 为 web 时路由到 Web 服务
            - Header=X-Client-Type,web
```

支持正则表达式：
```yaml
- Header=X-Request-Id,\d+  # 请求 ID 必须是数字
```

### 2.6 Cookie：Cookie 匹配

根据 Cookie 进行路由：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: vip-route
          uri: lb://vip-service
          predicates:
            # 需要有 name 为 vip 的 Cookie
            - Cookie=name,vip
        - id: normal-route
          uri: lb://normal-service
          predicates:
            # 没有 vip Cookie 的走这里
            - Path=/api/**
```

### 2.7 Host：域名匹配

根据 Host 进行路由：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: api-host-route
          uri: lb://api-service
          predicates:
            # 匹配 api.example.com 和 *.api.example.com
            - Host=api.example.com,**.api.example.com
        - id: admin-host-route
          uri: lb://admin-service
          predicates:
            - Host=admin.example.com
```

### 2.8 Before / After / Between：时间匹配

在特定时间段内生效路由：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: maintenance-route
          uri: forward:/maintenance
          predicates:
            # 2024 年 12 月 31 日 23:00 到 2025 年 1 月 1 日 06:00 之间生效
            - Between=2024-12-31T23:00:00+08:00[Asia/Shanghai],2025-01-01T06:00:00+08:00[Asia/Shanghai]
```

### 2.9 RemoteAddr：IP 地址匹配

根据客户端 IP 进行路由：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: internal-route
          uri: lb://internal-service
          predicates:
            # 仅允许内网 IP 访问
            - RemoteAddr=192.168.0.0/16
        - id: public-route
          uri: lb://public-service
          predicates:
            - Path=/public/**
```

### 2.10 Weight：权重路由

按权重分配流量，用于灰度发布：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        # 流量权重分组：group1 组内，v1 占 20%，v2 占 80%
        - id: weight-v1
          uri: lb://order-service-v1
          predicates:
            - Weight=group1,v1
        - id: weight-v2
          uri: lb://order-service-v2
          predicates:
            - Weight=group1,v2
```

## 三、Route 的组成

### 3.1 Route 的三要素

Route（路由）由三个核心要素组成：

```
Route = ID + URI + Predicates + Filters
```

| 要素 | 说明 | 必填 |
| --- | --- | --- |
| ID | 路由的唯一标识 | 是 |
| URI | 目标地址（支持 `lb://` 负载均衡） | 是 |
| Predicates | 匹配条件数组 | 是 |
| Filters | 过滤器数组 | 否 |

### 3.2 Route 的完整配置

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: order-route
          uri: lb://order-service
          predicates:
            - Path=/api/orders/**
            - Method=GET,POST
            - Header=Authorization,.+
          filters:
            - StripPrefix=1
            - AddRequestHeader=X-Gateway,SpringCloudGateway
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 100
                redis-rate-limiter.burstCapacity: 200
```

## 四、路由配置方式对比

### 4.1 YAML 配置

最常用的方式，配置清晰易懂：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: order-route
          uri: lb://order-service
          predicates:
            - Path=/api/orders/**
          filters:
            - StripPrefix=1
```

### 4.2 Java DSL 配置

编程式配置，更灵活：

```java
@Bean
public RouteLocator customRouteLocator(RouteLocatorBuilder builder) {
    return builder.routes()
        .route("order-route", r -> r
            .path("/api/orders/**")
            .filters(f -> f
                .stripPrefix(1)
                .addRequestHeader("X-Gateway", "Gateway"))
            .uri("lb://order-service"))
        .route("user-route", r -> r
            .path("/api/users/**")
            .filters(f -> f.stripPrefix(1))
            .uri("lb://user-service"))
        .build();
}
```

### 4.3 配置中心动态路由

将路由配置放在 Nacos 等配置中心，支持动态修改：

```yaml title="Nacos 配置"

  cloud:
    gateway:
      routes:
        - id: order-route
          uri: lb://order-service
          predicates:
            - Path=/api/orders/**
```

## 五、动态路由

### 5.1 从注册中心获取服务

Gateway 支持从服务注册中心动态获取服务实例：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      discovery:
        locator:
          enabled: true  # 开启从注册中心获取服务
          lower-case-service-id: true  # 服务名转小写
          url-expression: "'lb://' + serviceId"  # 路由表达式
```

开启后，可以直接访问：
```
GET /order-service/api/orders  → 自动路由到 order-service
GET /user-service/api/users     → 自动路由到 user-service
```

### 5.2 自定义 RouteLocator

通过实现 `RouteLocator` 接口，可以自定义路由的获取方式：

```java
@Component
public class CustomRouteLocator implements RouteLocator {
    
    @Autowired
    private DiscoveryClient discoveryClient;
    
    @Override
    public Flux<Route> getRoutes() {
        // 从注册中心获取所有服务
        return Flux.fromIterable(discoveryClient.getServices())
            .flatMap(serviceId -> {
                // 为每个服务创建路由
                Route route = Route.async()
                    .id(serviceId + "-route")
                    .uri("lb://" + serviceId.toUpperCase())
                    .predicate(exchange -> {
                        String path = exchange.getRequest().getPath().value();
                        return path.startsWith("/" + serviceId);
                    })
                    .build();
                return Mono.just(route);
            });
    }
}
```

## 六、多个谓词的组合

### 6.1 and() 组合

多个谓词同时满足：

```yaml title="application.yml"
spring:
  cloud:
    gateway:
      routes:
        - id: multi-predicate-route
          uri: lb://order-service
          predicates:
            - Path=/api/orders/**
            - Method=GET
            - Header=Authorization,.+
```

这个路由只有在**同时满足**以下条件时才匹配：
- 路径以 `/api/orders/**` 开头
- HTTP 方法是 GET
- 有 Authorization 请求头

### 6.2 or() 组合

多个谓词满足其一即可（需要使用 Java DSL）：

```java
.route("payment-route", r -> r
    .path("/api/pay/**")
    .or()
    .path("/api/alipay/**")
    .or()
    .path("/api/wechat/**")
    .uri("lb://payment-service"))
```

### 6.3 复杂的谓词组合

```java
.route("complex-route", r -> r
    // 路径匹配
    .path("/api/vip/**")
    // 并且
    .and()
    // 满足以下任一条件：
    .or(
        // 条件1：Cookie 有 vip 标记
        r1 -> r1.cookie("vip", ".*"),
        // 条件2：Header 有 VIP-Token
        r2 -> r2.header("VIP-Token", ".*")
    )
    .uri("lb://vip-service"))
```

## 七、面试追问方向

- **Gateway 的路由匹配流程是什么？** 请求进入 → `RoutePredicateHandlerMapping` 获取路由 → 遍历所有 Route → 使用 Predicate 匹配 → 返回匹配的 Route。
- **多个路由都匹配时会选择哪个？** 选择 `order` 最小（优先级最高）的路由。如果 `order` 相同，则按配置顺序选择第一个。
- **Predicate 和 Filter 的区别是什么？** Predicate 决定"哪些请求要走这个路由"，Filter 决定"对匹配的请求做什么处理"。
- **如何实现动态路由？** 关闭默认路由，使用 `RouteLocator` 从配置中心或注册中心动态获取路由配置。

---

## 八、总结

Gateway 的路由机制是整个请求处理流程的起点：

- **RoutePredicateHandlerMapping**：负责找到匹配的路由
- **Predicates**：11 种匹配条件，满足条件的请求才会被路由
- **动态路由**：从注册中心或配置中心获取路由规则
- **谓词组合**：通过 `and()` 和 `or()` 实现复杂的匹配逻辑

理解路由匹配的原理，才能正确配置 Gateway，让请求准确到达目标服务。
