## 一、核心设计模式

### 1. 单例模式

Spring 默认将 Bean 作为单例管理：

```java
// DefaultSingletonBeanRegistry.java
private final Map<String, Object> singletonObjects = new ConcurrentHashMap<>(256);

public Object getSingleton(String beanName) {
    Object singletonObject = singletonObjects.get(beanName);
    if (singletonObject == null) {
        synchronized (singletonObjects) {
            singletonObject = singletonObjects.get(beanName);
            if (singletonObject == null) {
                singletonObject = createBean(beanName);
                singletonObjects.put(beanName, singletonObject);
            }
        }
    }
    return singletonObject;
}
```

### 2. 工厂模式

BeanFactory 是典型的工厂模式：

```java
public interface BeanFactory {
    Object getBean(String name);
    <T> T getBean(Class<T> requiredType);
}
```

### 3. 代理模式

Spring AOP 使用代理模式：

```java
public interface AopProxy {
    Object getProxy();
    Object getProxy(ClassLoader classLoader);
}
```

## 三、模板方法模式

JdbcTemplate 体现了模板方法模式：

```java
public abstract class JdbcAccessor {

    protected DataSource dataSource;

    public Object execute(ConnectionCallback action) {
        Connection con = null;
        try {
            con = dataSource.getConnection();
            return action.doInConnection(con);  // 回调
        } finally {
            closeConnection(con);
        }
    }
}

public class JdbcTemplate extends JdbcAccessor {

    public <T> T execute(ConnectionCallback<T> action) {
        Connection con = null;
        try {
            con = DataSourceUtils.getConnection(dataSource);
            return action.doInConnection(con);
        } finally {
            DataSourceUtils.releaseConnection(con);
        }
    }
}
```

## 五、高频面试题

### 🔴 第一层：Spring 用到了哪些设计模式？

**答案要点**：
1. 单例模式：Bean 默认单例
2. 工厂模式：BeanFactory
3. 代理模式：AOP
4. 模板方法模式：JdbcTemplate
5. 观察者模式：事件监听
6. 适配器模式：Controller 适配

### 🟡 第二层：Spring 如何保证单例 Bean 的线程安全？

**答案要点**：
1. 单例 Bean 是共享的，实例变量可能存在线程安全问题
2. 无状态的 Bean 可以使用单例模式
3. 有状态的 Bean 应该使用 Prototype 作用域或使用 ThreadLocal

---

## 六、常见陷阱

> ⚠️ **陷阱一**：单例 Bean 中使用可变成员变量

单例 Bean 在多线程环境下共享，如果成员变量可变，可能导致线程安全问题。

> ⚠️ **陷阱二**：BeanFactory 不是工厂模式

BeanFactory 是工厂模式的具体实现类，但不是简单的工厂模式，而是更复杂的容器。
