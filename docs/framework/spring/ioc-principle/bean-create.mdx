# Bean 创建流程：实例化、属性填充、初始化

Spring 是如何把一个 `@Component` 注解变成可用的 Bean 的？这个问题看似简单，背后却藏着 Spring 框架最核心的逻辑。

当你在代码里写 `context.getBean(UserService.class)` 时，Spring 到底做了什么？它是如何找到构造器、如何注入依赖、如何执行初始化方法的？

这一节，我们来深入 `doCreateBean()` 方法，搞清楚 Bean 创建的完整流程。

## 一、Bean 创建总览

### 1.1 核心入口：getBean()

```java
// AbstractBeanFactory.getBean()
public Object getBean(String name) throws BeansException {
    return doGetBean(name, null, null, false);
}

protected <T> T doGetBean(
        String name,
        Class<T> requiredType,
        Object[] args,
        boolean typeCheckOnly) throws BeansException {

    // 1. 转换 Bean 名称（去掉 & 前缀）
    final String beanName = transformedBeanName(name);

    // 2. 尝试从缓存获取（处理循环依赖）
    Object bean = getSingleton(beanName);

    if (bean != null) {
        // 单例 Bean 已存在，直接返回
        return bean;
    }

    // 3. 单例 Bean 不存在，创建新的
    // 检查是否正在创建（处理循环依赖）
    if (isPrototypeCurrentlyInCreation(beanName)) {
        throw new BeanCurrentlyInCreationException(beanName);
    }

    // 4. 获取 BeanDefinition
    BeanDefinition bd = getMergedLocalBeanDefinition(beanName);

    // 5. 创建 Bean
    if (bd.isSingleton()) {
        // 单例 Bean
        return createBean(beanName, bd, args);
    } else if (bd.isPrototype()) {
        // 原型 Bean
        Object prototypeInstance = createBean(beanName, bd, args);
        return prototypeInstance;
    } else {
        // 其他作用域...
    }
}
```

### 1.2 doCreateBean() 三大阶段

```java
// AbstractAutowireCapableBeanFactory.doCreateBean()
protected Object doCreateBean(String beanName, RootBeanDefinition mbd, Object[] args) {
    // 阶段 1：实例化（Instantiation）
    BeanWrapper instanceWrapper = createBeanInstance(beanName, mbd, args);
    Object bean = instanceWrapper.getWrappedInstance();

    // 阶段 2：属性填充（Population）
    populateBean(beanName, mbd, instanceWrapper);

    // 阶段 3：初始化（Initialization）
    bean = initializeBean(beanName, bean, mbd);

    return bean;
}
```

这就是 Bean 创建的核心流程：
- **实例化**：调用构造器创建对象
- **属性填充**：注入依赖
- **初始化**：执行Aware回调、初始化方法

## 二、阶段一：实例化（createBeanInstance）

### 2.1 选择实例化方式

```java
protected BeanWrapper createBeanInstance(String beanName, RootBeanDefinition mbd, Object[] args) {
    // 1. 如果有工厂方法，使用工厂方法创建
    if (mbd.getFactoryMethodName() != null) {
        return instantiateUsingFactoryMethod(beanName, mbd, args);
    }

    // 2. 使用构造器创建（重点）
    Constructor<?>[] ctors = determineConstructorsFromBeanPostProcessors(beanClass, beanName);
    if (ctors != null) {
        // 构造器自动注入
        return autowireConstructor(beanName, mbd, ctors, args);
    }

    // 3. 使用默认构造器
    return instantiateBean(beanName, mbd);
}
```

### 2.2 三种实例化方式

**方式一：工厂方法创建**

```java
// XML 配置
<bean id="userService" factory-method="createInstance"/>

// @Bean
@Configuration
public class AppConfig {
    @Bean
    public UserService userService() {
        return new UserService();
    }
}
```

**方式二：构造器自动注入**

```java
// 使用参数最多的构造器创建
public class UserService {
    private UserDao userDao;
    private RoleService roleService;

    // Spring 会选择这个构造器
    public UserService(UserDao userDao, RoleService roleService) {
        this.userDao = userDao;
        this.roleService = roleService;
    }
}
```

**方式三：默认构造器**

```java
// 直接使用默认构造器（无参构造器）
public class UserService {
    public UserService() { }
}
```

### 2.3 核心方法：instantiateBean()

```java
protected BeanWrapper instantiateBean(String beanName, RootBeanDefinition mbd) {
    try {
        Object beanInstance;

        // 1. 获取默认构造器
        Constructor<?> constructorToUse;
        if (System.getSecurityManager() != null) {
            constructorToUse = AccessController.doPrivileged(
                new PrivilegedAction<Constructor<?>>() {
                    @Override
                    public Constructor<?> run() {
                        return beanClass.getDeclaredConstructor();
                    }
                });
        } else {
            constructorToUse = beanClass.getDeclaredConstructor();
        }

        // 2. 使用反射创建实例
        beanInstance = constructorToUse.newInstance();

        // 3. 包装成 BeanWrapper 返回
        BeanWrapper bw = new BeanWrapperImpl(beanInstance);
        initBeanWrapper(bw);
        return bw;
    } catch (Exception ex) {
        // 异常处理...
    }
}
```

### 2.4 实例化阶段与循环依赖

实例化完成后，Spring 会立即将「实例化工厂」放入三级缓存，这是解决循环依赖的关键：

```java
// AbstractAutowireCapableBeanFactory.doCreateBean()
protected Object doCreateBean(String beanName, RootBeanDefinition mbd, Object[] args) {
    // 1. 实例化
    BeanWrapper instanceWrapper = createBeanInstance(beanName, mbd, args);
    Object bean = instanceWrapper.getWrappedInstance();

    // 2. 放入三级缓存（解决循环依赖的关键！）
    addSingletonFactory(beanName, () -> getEarlyBeanReference(beanName, mbd, bean));

    // 3. 属性填充
    populateBean(beanName, mbd, instanceWrapper);

    // ...
}
```

## 三、阶段二：属性填充（populateBean）

### 3.1 属性填充的核心逻辑

```java
protected void populateBean(String beanName, RootBeanDefinition mbd, BeanWrapper bw) {
    // 1. 获取属性值
    PropertyValues pvs = mbd.getPropertyValues();

    // 2. 处理 autowire（byName / byType）
    if (mbd.getAutowireMode() == AUTOWIRE_BY_NAME ||
        mbd.getAutowireMode() == AUTOWIRE_BY_TYPE) {
        MutablePropertyValues mpvs = new MutablePropertyValues(pvs);
        // 自动注入 byName / byType
        if (mbd.getAutowireMode() == AUTOWIRE_BY_NAME) {
            autowireByName(beanName, mbd, bw, mpvs);
        }
        if (mbd.getAutowireMode() == AUTOWIRE_BY_TYPE) {
            autowireByType(beanName, mbd, bw, mpvs);
        }
        pvs = mpvs;
    }

    // 3. 处理注解（@Autowired、@Resource 等）
    // 调用 InstantiationAwareBeanPostProcessor
    boolean hasInstAwareBpps = hasInstantiationAwareBeanPostProcessors();
    if (hasInstAwareBpps) {
        for (BeanPostProcessor bp : getBeanPostProcessors()) {
            if (bp instanceof InstantiationAwareBeanPostProcessor) {
                InstantiationAwareBeanPostProcessor ibp =
                    (InstantiationAwareBeanPostProcessor) bp;
                PropertyValues pvsToUse = ibp.postProcessProperties(pvs, bw.getWrappedInstance(), beanName);
                if (pvsToUse == null) {
                    return; // 跳过属性填充
                }
                pvs = pvsToUse;
            }
        }
    }

    // 4. 真正执行属性注入
    applyPropertyValues(beanName, mbd, bw, pvs);
}
```

### 3.2 InstantiationAwareBeanPostProcessor

这个接口允许在属性填充前做一些额外处理：

```java
public interface InstantiationAwareBeanPostProcessor extends BeanPostProcessor {
    // 实例化前（返回代理对象可替代默认实例化）
    Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName);

    // 实例化后（返回 false 可跳过属性填充）
    boolean postProcessAfterInstantiation(Object bean, String beanName);

    // 处理属性值
    PropertyValues postProcessProperties(PropertyValues pvs,
        Object bean, String beanName);
}
```

Spring 内置的 `InstantiationAwareBeanPostProcessor`：

| 处理器 | 作用 |
|---|---|
| `AutowiredAnnotationBeanPostProcessor` | 处理 `@Autowired`、`@Value` |
| `CommonAnnotationBeanPostProcessor` | 处理 `@Resource` |
| `RequiredAnnotationBeanPostProcessor` | 处理 `@Required`（已废弃） |

### 3.3 applyPropertyValues()：真正执行注入

```java
protected void applyPropertyValues(String beanName, BeanDefinition bd,
        BeanWrapper bw, PropertyValues pvs) {
    if (pvs.isEmpty()) {
        return;
    }

    // 1. 解析属性值（如果是 bean 引用，解析为真正的 bean）
    MutablePropertyValues mpvs = new MutablePropertyValues();
    for (PropertyValue pv : pvs.getPropertyValues()) {
        if (pv.isConverted()) {
            mpvs.addPropertyValue(pv);
        } else {
            String propertyName = pv.getName();
            Object originalValue = pv.getValue();

            // 解析依赖（如 ref="userDao" 解析为真正的 UserDao Bean）
            Object resolvedValue = resolveValue(
                new RuntimeBeanReference(propertyName), beanName);
            mpvs.addPropertyValue(propertyName, resolvedValue);
        }
    }

    // 2. 使用反射设置属性
    try {
        bw.setPropertyValues(mpvs);
    } catch (Exception ex) {
        throw new BeanCreationException(beanName, "set property values", ex);
    }
}
```

## 三、阶段三：初始化（initializeBean）

### 3.1 初始化流程总览

```java
protected Object initializeBean(String beanName, Object bean, RootBeanDefinition mbd) {
    // 1. 执行 Aware 回调
    invokeAwareMethods(beanName, bean);

    // 2. 执行前置处理器
    Object wrappedBean = applyBeanPostProcessorsBeforeInitialization(bean, beanName);

    // 3. 执行初始化方法
    invokeInitMethods(beanName, wrappedBean, mbd);

    // 4. 执行后置处理器
    wrappedBean = applyBeanPostProcessorsAfterInitialization(wrappedBean, beanName);

    return wrappedBean;
}
```

### 3.2 Aware 回调

```java
private void invokeAwareMethods(String beanName, Object bean) {
    if (bean instanceof Aware) {
        if (bean instanceof BeanNameAware) {
            ((BeanNameAware) bean).setBeanName(beanName);
        }
        if (bean instanceof BeanFactoryAware) {
            ((BeanFactoryAware) bean).setBeanFactory(this);
        }
        if (bean instanceof ApplicationContextAware) {
            ((ApplicationContextAware) bean).setApplicationContext(this);
        }
    }
}
```

### 3.3 初始化方法

```java
protected void invokeInitMethods(String beanName, Object bean, RootBeanDefinition mbd)
        throws Throwable {

    // 1. 调用 InitializingBean.afterPropertiesSet()
    if (bean instanceof InitializingBean) {
        ((InitializingBean) bean).afterPropertiesSet();
    }

    // 2. 调用自定义 init-method
    String initMethodName = mbd.getInitMethodName();
    if (StringUtils.hasLength(initMethodName) &&
        !(mbd.isEnforceInitMethod() &&
          initMethodName.equals(InitializingBean.INIT_METHOD_NAME))) {
        Method initMethod = determineCommonMethod(initMethodName);
        if (initMethod != null) {
            ReflectionUtils.makeAccessible(initMethod);
            ReflectionUtils.invokeMethod(initMethod, bean);
        }
    }
}
```

**执行顺序**：`@PostConstruct` 注解的方法会在 `InitializingBean.afterPropertiesSet()` 之前执行（因为它是通过 `CommonAnnotationBeanPostProcessor` 处理的）。

### 3.4 BeanPostProcessor 执行顺序

```java
public Object applyBeanPostProcessorsBeforeInitialization(Object existingBean, String beanName)
        throws BeansException {
    Object result = existingBean;
    for (BeanPostProcessor processor : getBeanPostProcessors()) {
        Object current = processor.postProcessBeforeInitialization(result, beanName);
        if (current == null) {
            return result;
        }
        result = current;
    }
    return result;
}

public Object applyBeanPostProcessorsAfterInitialization(Object existingBean, String beanName)
        throws BeansException {
    Object result = existingBean;
    for (BeanPostProcessor processor : getBeanPostProcessors()) {
        Object current = processor.postProcessAfterInitialization(result, beanName);
        if (current == null) {
            return result;
        }
        result = current;
    }
    return result;
}
```

## 五、面试追问方向

**「Bean 是先实例化还是先属性填充？」**

先实例化，后填充。实例化后放入三级缓存，然后进行属性填充。

**「Aware 回调在属性填充之后吗？」**

是的。在 `initializeBean()` 方法中，`invokeAwareMethods()` 在 `populateBean()` 之后执行。

**「BeanPostProcessor.postProcessBeforeInitialization() 和 postProcessAfterInitialization() 有什么区别？」**

前者在 `@PostConstruct`、`InitializingBean`、`init-method` 之前执行；后者在这些初始化方法之后执行。AOP 代理的生成通常在 `postProcessAfterInitialization()` 中完成。

**「BeanFactoryPostProcessor 和 BeanPostProcessor 的区别是什么？」**

前者操作 `BeanDefinition`，在 Bean 创建之前执行；后者操作 Bean 实例，在 Bean 创建过程中执行。

---

## 总结

Bean 创建的完整流程：

1. **实例化**：通过构造器或工厂方法创建实例
2. **属性填充**：注入 `@Autowired`、`@Resource` 等依赖
3. **初始化**：执行 Aware 回调、初始化方法

每个阶段都有对应的扩展点：
- 实例化前：`InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation()`
- 实例化后：`InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation()`
- 初始化前：`BeanPostProcessor.postProcessBeforeInitialization()`
- 初始化后：`BeanPostProcessor.postProcessAfterInitialization()`

理解这个流程，你就能在合适的时机插入自己的逻辑，实现更强大的功能。
