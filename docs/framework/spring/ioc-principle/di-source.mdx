# 依赖注入源码：setAutowiredField() 过程

你知道 `@Autowired` 注解是怎么工作的吗？当 Spring 遇到 `@Autowired` 注解时，它是如何找到对应的 Bean 并注入的？

这一节，我们来深入 `AutowiredAnnotationBeanPostProcessor` 的源码，分析依赖注入的完整过程。

## 一、@Autowired 注入的整体流程

### 1.1 入口：AutowiredAnnotationBeanPostProcessor

`@Autowired` 注解是由 `AutowiredAnnotationBeanPostProcessor` 处理的：

```java
public class AutowiredAnnotationBeanPostProcessor
        extends InstantiationAwareBeanPostProcessorAdapter
        implements MergedBeanDefinitionPostProcessor {

    // 构造函数中指定要处理的注解
    public AutowiredAnnotationBeanPostProcessor() {
        this.autowiredAnnotationTypes.add(Autowired.class);
        this.autowiredAnnotationTypes.add(Value.class);
        // JSR-330 的 @Inject
        try {
            this.autowiredAnnotationTypes.add((Class<? extends Annotation>)
                ClassUtils.forName("javax.inject.Inject", getClass().getClassLoader()));
        } catch (ClassNotFoundException ex) {
            // JSR-330 不存在，忽略
        }
    }
}
```

### 1.2 处理流程图

```
AutowiredAnnotationBeanPostProcessor
       ↓
postProcessProperties()
       ↓
1. findAutowiringMetadata()  → 找到所有 @Autowired 字段/方法
       ↓
2. InjectionMetadata.inject() → 遍历所有注入点
       ↓
3. AutowiredFieldElement.inject() → 注入字段
   或
   AutowiredMethodElement.inject() → 注入方法
```

## 二、第一步：findAutowiringMetadata() 找到注入点

### 2.1 源码解析

```java
private InjectionMetadata findAutowiringMetadata(Class<?> clazz) {
    // 1. 先检查缓存
    String cacheKey = clazz.getName();
    InjectionMetadata metadata = this.injectionMetadataCache.get(cacheKey);
    if (metadata != null) {
        return metadata;
    }

    // 2. 没有缓存，构建元数据
    metadata = buildAutowiringMetadata(clazz);
    this.injectionMetadataCache.put(cacheKey, metadata);

    return metadata;
}

private InjectionMetadata buildAutowiringMetadata(final Class<?> clazz) {
    // 存储所有需要注入的元素
    List<InjectionMetadata.InjectedElement> elements = new ArrayList<>();

    // 处理当前类
    Class<?> targetClass = clazz;

    do {
        final List<InjectionMetadata.InjectedElement> currElements = new ArrayList<>();

        // 遍历所有字段
        ReflectionUtils.doWithFields(targetClass, field -> {
            // 查找 @Autowired 注解
            Autowired ann = field.getAnnotation(Autowired.class);
            if (ann != null) {
                // 静态字段跳过
                if (Modifier.isStatic(field.getModifiers())) {
                    return;
                }
                // 构建注入元素
                currElements.add(new AutowiredFieldElement(field, ann.required()));
            }
        });

        // 遍历所有方法
        ReflectionUtils.doWithMethods(targetClass, method -> {
            // 查找 @Autowired 注解
            Autowired ann = method.getAnnotation(Autowired.class);
            if (ann != null) {
                // 跳过静态方法
                if (Modifier.isStatic(method.getModifiers())) {
                    return;
                }
                // 处理方法重写的情况
                PropertyDescriptor pd = BeanUtils.findPropertyForMethod(method);
                // 构建注入元素
                currElements.add(new AutowiredMethodElement(method, ann.required(), pd));
            }
        });

        elements.addAll(0, currElements);
        targetClass = targetClass.getSuperclass();
    }
    while (targetClass != null && targetClass != Object.class);

    return new InjectionMetadata(clazz, elements);
}
```

### 2.2 关键点

1. **缓存机制**：构建一次后缓存，避免重复解析
2. **递归处理**：处理当前类及其父类的所有注入点
3. **静态字段/方法跳过**：Spring 不处理静态成员的注入

## 三、第二步：InjectionMetadata.inject() 执行注入

### 3.1 源码解析

```java
public class InjectionMetadata {
    private final Class<?> targetClass;
    private final Collection<InjectedElement> injectedElements;

    public void inject(Object target, String beanName, PropertyValues pvs) throws Throwable {
        if (this.injectedElements != null) {
            for (InjectedElement element : this.injectedElements) {
                // 调用字段或方法的 inject()
                element.inject(target, beanName, pvs);
            }
        }
    }

    // 内部类：字段注入
    private class AutowiredFieldElement extends InjectedElement {
        private final boolean required;
        private volatile boolean cached;

        protected void inject(Object bean, String beanName, PropertyValues pvs) throws Throwable {
            Field field = (Field) this.member;
            Object value;

            if (this.cached) {
                // 从缓存获取
                value = resolveCachedArgument(beanName, this.cachedFieldValue);
            } else {
                // 创建依赖描述符
                DependencyDescriptor desc =
                    new DependencyDescriptor(new ResolvableType(field), field);

                // 设置当前 Bean 名称
                desc.setContainingClass(bean.getClass());
                desc.setName(field.getName());
                desc.setRequired(this.required);

                // 解析依赖
                value = beanFactory.resolveDependency(desc, beanName, autowiredBeanNames, typeConverter);
            }

            if (value != null) {
                // 反射设置字段值
                ReflectionUtils.makeAccessible(field);
                field.set(bean, value);
            }
        }
    }

    // 内部类：方法注入
    private class AutowiredMethodElement extends InjectedElement {
        private final boolean required;
        private volatile Object[] cachedMethodArguments;

        protected void inject(Object bean, String beanName, PropertyValues pvs) throws Throwable {
            Method method = (Method) this.member;
            Object[] arguments;

            if (this.cached) {
                // 从缓存获取
                arguments = resolveCachedArguments(beanName, this.cachedMethodArguments);
            } else {
                // 获取方法参数
                MethodParameter methodParam = new MethodParameter(method, -1);

                // 为每个参数创建依赖描述符
                DependencyDescriptor[] descriptors = new DependencyDescriptor[methodParam.getParameterCount()];
                for (int i = 0; i < descriptors.length; i++) {
                    MethodParameter param = new MethodParameter(method, i);
                    descriptors[i] = new DependencyDescriptor(param);
                    descriptors[i].setContainingClass(bean.getClass());
                }

                // 解析依赖
                arguments = beanFactory.resolveMultipleArguments(
                    descriptors, beanName, autowiredBeanNames, typeConverter);
            }

            if (arguments != null) {
                // 反射调用方法
                ReflectionUtils.makeAccessible(method);
                method.invoke(bean, arguments);
            }
        }
    }
}
```

## 四、第三步：resolveDependency() 解析依赖

### 4.1 源码解析

```java
// DefaultListableBeanFactory.resolveDependency()
public Object resolveDependency(DependencyDescriptor descriptor,
        String requestingBeanName, Set<String> autowiredBeanNames,
        TypeConverter typeConverter) throws BeansException {

    // 1. 处理 Optional 类型
    if (Optional.class == descriptor.getDependencyType()) {
        return createOptionalDependency(descriptor, requestingBeanName);
    }

    // 2. 处理 ObjectFactory / ObjectProvider
    if (ObjectFactory.class == descriptor.getDependencyType() ||
        ObjectProvider.class == descriptor.getDependencyType()) {
        return new DependencyObjectProvider(descriptor, requestingBeanName);
    }

    // 3. 处理 Stream/Array/Collection
    if (Stream.class == descriptor.getDependencyType() ||
        Iterator.class.isAssignableFrom(descriptor.getDependencyType())) {
        return StreamFactory.createStreamDependency(descriptor, requestingBeanName);
    }

    // 4. 普通依赖
    Object result = doResolveDependency(descriptor, requestingBeanName,
            autowiredBeanNames, typeConverter);

    return result;
}
```

### 4.2 doResolveDependency() 核心逻辑

```java
protected Object doResolveDependency(DependencyDescriptor descriptor,
        String requestingBeanName, Set<String> autowiredBeanNames,
        TypeConverter typeConverter) throws BeansException {

    // 1. 按类型查找所有候选 Bean
    Map<String, Object> matchingBeans = findAutowireCandidates(
        requestingBeanName, descriptor.getDependencyType(), descriptor);

    // 2. 如果只有一个候选，直接返回
    if (matchingBeans.size() == 1) {
        String beanName = matchingBeans.keySet().iterator().next();
        return matchingBeans.get(beanName);
    }

    // 3. 如果没有候选 Bean
    if (matchingBeans.isEmpty()) {
        if (descriptor.isRequired()) {
            // required=true 时抛出异常
            raiseNoMatchingBeanFound(descriptor, matchingBeans.keySet());
        }
        return null;
    }

    // 4. 有多个候选 Bean，需要确定使用哪一个
    if (matchingBeans.size() > 1) {
        // 确定候选者
        String primaryCandidate = determineAutowireCandidate(
            matchingBeans, descriptor);

        if (primaryCandidate != null) {
            return matchingBeans.get(primaryCandidate);
        }

        // 没有唯一候选者，抛出异常
        throw new NoUniqueBeanDefinitionException(
            descriptor.getDependencyType(), matchingBeans.keySet());
    }

    return null;
}
```

### 4.3 查找候选 Bean：findAutowireCandidates()

```java
protected Map<String, Object> findAutowireCandidates(
        String beanName, Class<?> requiredType, DependencyDescriptor descriptor) {

    // 1. 按类型查找所有 Bean
    String[] candidateNames = BeanFactoryUtils.beanNamesForTypeIncludingAncestors(
        this, requiredType);

    Map<String, Object> result = new LinkedHashMap<>(candidateNames.length);

    // 2. 遍历候选者
    for (String candidate : candidateNames) {
        // 跳过正在创建的 Bean（避免循环依赖时的死循环）
        if (!isSelfReference(beanName, candidate) && isAutowireCandidate(candidate, descriptor)) {
            result.put(candidate, getBean(candidate));
        }
    }

    return result;
}
```

### 4.4 确定唯一候选者：determineAutowireCandidate()

```java
protected String determineAutowireCandidate(Map<String, Object> candidates,
        DependencyDescriptor descriptor) {

    Class<?> requiredType = descriptor.getDependencyType();

    // 1. 优先使用 @Primary 标记的 Bean
    String primary = null;
    for (Map.Entry<String, Object> entry : candidates.entrySet()) {
        if (entry.getValue() instanceof AutowireCandidateQualifier) {
            Qualifier annot = entry.getValue().getAnnotation(Qualifier.class);
            if (annot != null && "primary".equals(annot.value())) {
                primary = entry.getKey();
                break;
            }
        }
    }
    if (primary != null) {
        return primary;
    }

    // 2. 使用 @Priority 优先级最高的 Bean
    Integer highestPriority = null;
    String candidateWithHighestPriority = null;
    for (String candidateName : candidates.keySet()) {
        Integer priority = getPriority(candidates.get(candidateName));
        if (priority != null) {
            if (highestPriority == null || priority < highestPriority) {
                highestPriority = priority;
                candidateWithHighestPriority = candidateName;
            }
        }
    }
    if (candidateWithHighestPriority != null) {
        return candidateWithHighestPriority;
    }

    // 3. 回退策略：按名称匹配
    for (String candidateName : candidates.keySet()) {
        if (requiredType.isInstance(candidates.get(candidateName))) {
            return candidateName;
        }
    }

    return null;
}
```

## 五、@Qualifier 与 @Primary

### 5.1 @Qualifier 指定 Bean

```java
@Service
@Qualifier("mysqlUserDao")
public class MysqlUserDao implements UserDao { }

@Service
@Qualifier("oracleUserDao")
public class OracleUserDao implements UserDao { }

// 使用
@Service
public class UserService {
    @Autowired
    @Qualifier("mysqlUserDao")
    private UserDao userDao;
}
```

### 5.2 @Primary 指定默认 Bean

```java
@Service
@Primary
public class MysqlUserDao implements UserDao { }

@Service
public class OracleUserDao implements UserDao { }

// 使用（不指定时使用 @Primary 的）
@Service
public class UserService {
    @Autowired
    private UserDao userDao; // 自动注入 MysqlUserDao
}
```

public class HandlerRegistry {

## 七、面试追问方向

**「@Autowired 注解是如何被解析的？」**

通过 `AutowiredAnnotationBeanPostProcessor.postProcessProperties()` 方法，在 Bean 属性填充阶段被调用。核心流程是：找到所有 `@Autowired` 注解 → 创建 `InjectionMetadata` → 执行注入 → 调用 `resolveDependency()` 解析依赖。

**「当有多个同类型 Bean 时，Spring 如何选择？」**

优先级：1. `@Qualifier` 指定 > 2. `@Primary` 标记 > 3. `@Priority` 优先级 > 4. 字段名匹配类型名。

**「@Autowired 和 @Resource 的区别？」**

`@Autowired` 先按类型匹配，再按名称匹配；`@Resource` 先按名称匹配，再按类型匹配。

**「为什么字段注入不推荐使用？」**

因为字段注入绕过了构造器，依赖关系不可见，难以进行单元测试。

---

## 总结

`@Autowired` 注解的注入过程：

1. **构建元数据**：`findAutowiringMetadata()` 找到所有 `@Autowired` 字段/方法，缓存到 `InjectionMetadata`
2. **执行注入**：`InjectionMetadata.inject()` 遍历所有注入点，调用 `inject()` 方法
3. **解析依赖**：`resolveDependency()` 按类型查找候选 Bean，按规则确定最终使用的 Bean
4. **反射赋值**：`field.set()` 或 `method.invoke()` 将依赖注入到目标对象

理解这个过程，能够帮助你更好地使用 `@Autowired` 注解，并排查相关问题。
