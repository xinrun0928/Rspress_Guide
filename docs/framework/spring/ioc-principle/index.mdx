## 源码学习方法

### 为什么要读源码？

很多开发者觉得「会用就行了」，读源码是浪费时间。但实际上，读懂源码能带来三大好处：

**1. 遇到问题时能快速定位根因**

当你遇到 `BeanCurrentlyInCreationException`、`NoSuchBeanDefinitionException` 这些错误时，知道源码逻辑，就能快速定位问题。

**2. 能更好地设计自己的扩展**

Spring 提供了很多扩展点，但如果你不知道 Bean 的生命周期，就不知道在哪个时机插入自己的逻辑。

**3. 面试加分项**

面试官问「Spring 是怎么创建 Bean 的」，能答出源码路径和关键方法的人，绝对是加分项。

### 源码阅读路线图

```
Spring 启动流程
├── AbstractApplicationContext.refresh()
│   ├── prepareBeanFactory() - 准备 BeanFactory
│   ├── invokeBeanFactoryPostProcessors() - 执行 BFPP
│   ├── registerBeanPostProcessors() - 注册 BPP
│   ├── initMessageSource() - 初始化消息源
│   ├── initApplicationEventMulticaster() - 初始化事件广播器
│   ├── onRefresh() - 子类扩展
│   ├── registerListeners() - 注册监听器
│   ├── finishBeanFactoryInitialization() - 初始化所有单例
│   └── finishRefresh() - 完成刷新
│
└── Bean 创建流程
    ├── doCreateBean()
    │   ├── createBeanInstance() - 实例化
    │   ├── populateBean() - 属性填充
    │   └── initializeBean() - 初始化
    │       ├── invokeAwareMethods() - Aware 回调
    │       ├── applyBeanPostProcessorsBeforeInitialization() - 前置处理
    │       ├── invokeInitMethods() - 初始化方法
    │       └── applyBeanPostProcessorsAfterInitialization() - 后置处理
    │
    └── 依赖注入
        └── AutowiredAnnotationBeanPostProcessor
            └── inject() - 字段/构造器注入
```

## 面试考察分析

**「Spring 启动流程是怎样的？」**

这是最经典的源码题。标准答案是说出 `refresh()` 方法的 12 个步骤，但更重要的是理解**为什么要按这个顺序**：

- 先注册 BeanPostProcessor，再初始化 Bean（因为 BPP 需要处理所有 Bean）
- 先初始化 MessageSource，再初始化其他 Bean（因为其他 Bean 可能依赖国际化）
- 先注册监听器，再初始化 Bean（因为初始化过程中可能发布事件）

**「Bean 是如何被创建的？」**

要回答这个问题，需要把 `doCreateBean()` 的三个阶段说清楚：实例化 → 属性填充 → 初始化。

**「BeanPostProcessor 和 BeanFactoryPostProcessor 的区别？」**

前者作用于 Bean 实例，后者作用于 BeanDefinition。

## 学习建议

1. **先跑通流程**：用 IDE 打断点，跟踪 Spring 启动过程
2. **抓住核心方法**：`refresh()`、`doCreateBean()`、`populateBean()`、`getBean()`
3. **理解扩展点**：Aware 接口、`BeanPostProcessor`、`BeanFactoryPostProcessor`
4. **动手验证**：写一个自定义的 `BeanPostProcessor`，观察它何时被调用

> 源码阅读是一个循序渐进的过程。刚开始可能会觉得复杂，但只要坚持读完几个核心方法，就会豁然开朗。记住：读源码不是为了背诵，而是为了理解设计思想。
