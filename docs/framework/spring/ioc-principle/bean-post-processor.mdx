# BeanPostProcessor 与前置/后置处理器

`BeanPostProcessor` 是 Spring 框架最核心的扩展点之一。你可能每天都在用它，但你真的了解它吗？

`@Autowired` 为什么能生效？`@Transactional` 为什么能自动代理？`@Async` 为什么能异步执行？这些功能的背后，都有 `BeanPostProcessor` 的身影。

这一节，我们来彻底搞清楚 `BeanPostProcessor` 的原理和应用。

---

## 一、BeanPostProcessor 是什么？

### 1.1 接口定义

```java
public interface BeanPostProcessor {
    // 初始化前回调
    Object postProcessBeforeInitialization(Object bean, String beanName)
            throws BeansException;

    // 初始化后回调
    Object postProcessAfterInitialization(Object bean, String beanName)
            throws BeansException;
}
```

### 1.2 执行时机

`BeanPostProcessor` 在 Bean 的初始化阶段被调用：

```
Bean 创建流程：
实例化 → 属性填充 → 初始化
                    ↓
        ┌───────────────────────────┐
        │  1. invokeAwareMethods()  │  ← Aware 回调
        │  2. 前置处理              │  ← BeanPostProcessor.postProcessBeforeInitialization()
        │  3. @PostConstruct        │
        │  4. InitializingBean      │
        │  5. init-method          │
        │  6. 后置处理              │  ← BeanPostProcessor.postProcessAfterInitialization()
        └───────────────────────────┘
```

### 1.3 核心特点

1. **作用于所有 Bean**：`BeanPostProcessor` 会对容器中**每一个** Bean 都执行
2. **可以返回代理对象**：通过返回不同的对象，可以替换原始 Bean
3. **执行顺序可控**：可以通过实现 `Ordered` 接口控制执行顺序

## 二、Spring 内置的 BeanPostProcessor

Spring 内置了许多 `BeanPostProcessor`，它们共同完成了依赖注入、AOP 代理等功能。

### 2.1 内置处理器一览

| 处理器 | 父类/接口 | 作用 |
|---|---|---|
| `ApplicationContextAwareProcessor` | — | 处理 Aware 接口回调 |
| `CommonAnnotationBeanPostProcessor` | `InstantiationAwareBeanPostProcessor` | 处理 `@PostConstruct`、`@PreDestroy`、`@Resource` |
| `AutowiredAnnotationBeanPostProcessor` | `InstantiationAwareBeanPostProcessor` | 处理 `@Autowired`、`@Value`、`@Inject` |
| `RequiredAnnotationBeanPostProcessor` | `InstantiationAwareBeanPostProcessor` | 处理 `@Required`（已废弃） |
| `ConfigurationClassPostProcessor` | `BeanDefinitionRegistryPostProcessor` | 处理 `@Configuration` 类 |
| `AopAutoProxyConfigurer` | — | 处理 AOP 代理 |

### 2.2 ApplicationContextAwareProcessor

负责处理 Aware 接口回调：

```java
// 源码简化
class ApplicationContextAwareProcessor implements BeanPostProcessor {
    private final ApplicationContext applicationContext;

    public ApplicationContextAwareProcessor(ApplicationContext applicationContext) {
        this.applicationContext = applicationContext;
    }

    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        if (bean instanceof EnvironmentAware) {
            ((EnvironmentAware) bean).setEnvironment(applicationContext.getEnvironment());
        }
        if (bean instanceof EmbeddedValueResolverAware) {
            ((EmbeddedValueResolverAware) bean).setEmbeddedValueResolver(
                new EmbeddedValueResolver(applicationContext.getEnvironment()));
        }
        if (bean instanceof ResourceLoaderAware) {
            ((ResourceLoaderAware) bean).setResourceLoader(applicationContext);
        }
        if (bean instanceof ApplicationEventPublisherAware) {
            ((ApplicationEventPublisherAware) bean).setApplicationEventPublisher(
                applicationContext);
        }
        if (bean instanceof MessageSourceAware) {
            ((MessageSourceAware) bean).setMessageSource(applicationContext);
        }
        if (bean instanceof ApplicationContextAware) {
            ((ApplicationContextAware) bean).setApplicationContext(applicationContext);
        }
        return bean;
    }
}
```

### 2.3 AutowiredAnnotationBeanPostProcessor

这是 Spring 最核心的处理器之一，负责处理 `@Autowired` 和 `@Value` 注解：

```java
// 源码简化
public class AutowiredAnnotationBeanPostProcessor
        extends InstantiationAwareBeanPostProcessorAdapter {

    // 在 postProcessProperties 中处理 @Autowired 注解
    @Override
    public PropertyValues postProcessProperties(
            PropertyValues pvs, Object bean, String beanName) {

        // 找到所有标注了 @Autowired 的字段
        InjectionMetadata metadata = findAutowiringMetadata(bean.getClass());

        try {
            // 执行注入
            metadata.inject(bean, beanName, pvs);
        } catch (BeanCreationException ex) {
            throw ex;
        } catch (Throwable ex) {
            throw new BeanCreationException(beanName, "Injection of autowired dependencies", ex);
        }
        return pvs;
    }
}
```

### 2.4 CommonAnnotationBeanPostProcessor

处理 `@Resource`、`@PostConstruct`、`@PreDestroy` 等注解：

```java
// 源码简化
public class CommonAnnotationBeanPostProcessor
        extends InstantiationAwareBeanPostProcessorAdapter {

    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        // 处理 @PostConstruct
        LifecycleMetadata metadata = findLifecycleMetadata(bean.getClass());
        metadata.invokeInitMethods(bean, beanName);
        return bean;
    }

    @Override
    public void postProcessProperties(PropertyValues pvs, Object bean, String beanName) {
        // 处理 @Resource
        InjectionMetadata metadata = findResourceMetadata(bean.getClass());
        metadata.inject(bean, beanName, pvs);
    }

    @Override
    public Object postProcessBeforeDestruction(Object bean, String beanName) {
        // 处理 @PreDestroy
        LifecycleMetadata metadata = findLifecycleMetadata(bean.getClass());
        metadata.invokeDestroyMethods(bean, beanName);
        return bean;
    }
}
```

## 三、自定义 BeanPostProcessor

### 3.1 基本示例

```java
@Component
public class MyBeanPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        System.out.println("BeforeInitialization: " + beanName);
        return bean;
    }

    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        System.out.println("AfterInitialization: " + beanName);
        return bean;
    }
}
```

### 3.2 只处理特定 Bean

```java
@Component
public class MyBeanPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        // 只处理名称以 Service 结尾的 Bean
        if (beanName.endsWith("Service")) {
            System.out.println("Processing: " + beanName);
        }
        return bean;
    }

    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        return bean;
    }
}
```

### 3.3 返回代理对象

这是最强大的能力——可以通过 BPP 返回代理对象，替换原始 Bean：

```java
@Component
public class PerformanceMonitorPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        // 如果是 UserService，返回代理对象
        if (bean instanceof UserService) {
            return Proxy.newProxyInstance(
                bean.getClass().getClassLoader(),
                bean.getClass().getInterfaces(),
                (proxy, method, args) -> {
                    long start = System.currentTimeMillis();
                    Object result = method.invoke(bean, args);
                    long end = System.currentTimeMillis();
                    System.out.println(beanName + "." + method.getName() +
                        " took " + (end - start) + "ms");
                    return result;
                }
            );
        }
        return bean;
    }
}
```

## 四、InstantiationAwareBeanPostProcessor

### 4.1 接口定义

```java
public interface InstantiationAwareBeanPostProcessor extends BeanPostProcessor {
    // 实例化前（返回非空对象可替代默认实例化）
    Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName)
            throws BeansException;

    // 实例化后（返回 false 可跳过属性填充和后续初始化）
    boolean postProcessAfterInstantiation(Object bean, String beanName)
            throws BeansException;

    // 处理属性值
    PropertyValues postProcessProperties(PropertyValues pvs,
            Object bean, String beanName) throws BeansException;
}
```

### 4.2 与 BeanPostProcessor 的区别

| 特性 | BeanPostProcessor | InstantiationAwareBeanPostProcessor |
|---|---|---|
| 执行时机 | 初始化前后 | 实例化前后 + 属性填充前后 |
| 可跳过实例化 | ❌ | ✅（返回非空对象） |
| 可跳过属性填充 | ❌ | ✅（返回 false） |
| 典型应用 | AOP 代理生成 | @Autowired 注入 |

### 4.3 使用场景

**场景一：自定义实例化逻辑**

```java
@Component
public class CustomInstantiationProcessor implements InstantiationAwareBeanPostProcessor {
    @Override
    public Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName) {
        // 如果是特定类型，返回自定义对象
        if (beanClass == UserService.class) {
            // 返回代理对象，替代默认实例化
            return createCustomUserServiceProxy();
        }
        return null; // 返回 null，使用默认实例化
    }
}
```

**场景二：动态注入属性**

```java
@Component
public class CustomPropertyInjectionProcessor implements InstantiationAwareBeanPostProcessor {
    @Override
    public PropertyValues postProcessProperties(PropertyValues pvs,
            Object bean, String beanName) {
        // 动态添加属性
        MutablePropertyValues mpvs = new MutablePropertyValues(pvs);
        mpvs.add("dynamicProperty", "dynamicValue");
        return mpvs;
    }
}
```

## 五、SmartInstantiationAwareBeanPostProcessor

### 5.1 接口定义

这是 `InstantiationAwareBeanPostProcessor` 的子接口，增加了更多回调：

```java
public interface SmartInstantiationAwareBeanPostProcessor
        extends InstantiationAwareBeanPostProcessor {

    // 预测 Bean 的类型
    Class<?> predictBeanType(Class<?> beanClass, String beanName) throws BeansException;

    // 选择候选构造器
    Constructor<?>[] determineCandidateConstructors(Class<?> beanClass, String beanName)
            throws BeansException;

    // 获取提前暴露的引用（用于循环依赖）
    Object getEarlyBeanReference(Object bean, String beanName) throws BeansException;
}
```

### 5.2 典型应用

**决定使用哪个构造器**：

```java
@Component
public class CustomConstructorSelector implements SmartInstantiationAwareBeanPostProcessor {
    @Override
    public Constructor<?>[] determineCandidateConstructors(Class<?> beanClass, String beanName) {
        if (beanClass == UserService.class) {
            // 返回指定的构造器数组
            return new Constructor<?>[] {
                beanClass.getDeclaredConstructor(String.class),
                beanClass.getDeclaredConstructor(String.class, Integer.class)
            };
        }
        return null; // 返回 null，使用默认逻辑
    }
}
```

## 六、执行顺序与优先级

### 6.1 PriorityOrdered 接口

可以通过实现 `PriorityOrdered` 接口控制 BPP 的执行顺序：

```java
@Component
public class HighPriorityProcessor implements BeanPostProcessor, PriorityOrdered {
    @Override
    public int getOrder() {
        return Ordered.HIGHEST_PRECEDENCE; // 最高优先级
    }
}

@Component
public class LowPriorityProcessor implements BeanPostProcessor, PriorityOrdered {
    @Override
    public int getOrder() {
        return Ordered.LOWEST_PRECEDENCE; // 最低优先级
    }
}
```

### 6.2 Spring 内置 BPP 的执行顺序

在 `AbstractBeanFactory.registerBeanPostProcessors()` 中，BPP 按以下顺序注册：

```
1. PriorityOrdered 类型的 BPP
   - ApplicationContextAwareProcessor（最高优先级）
   - BeanValidationPostProcessor
   - ...

2. Ordered 类型的 BPP
   - AutowiredAnnotationBeanPostProcessor
   - CommonAnnotationBeanPostProcessor
   - ...

3. 普通 BPP

4. 内部的 MergedBeanDefinitionPostProcessor
   - AutowiredAnnotationBeanPostProcessor
   - CommonAnnotationBeanPostProcessor
```

## 七、与 AOP 的关系

### 7.1 AOP 代理在哪个阶段生成？

AOP 代理的生成是在 `postProcessAfterInitialization()` 阶段完成的：

```java
// AbstractAutowireCapableBeanFactory.initializeBean()
protected Object initializeBean(String beanName, Object bean, RootBeanDefinition mbd) {
    // ...

    Object wrappedBean = bean;

    // 前置处理
    wrappedBean = applyBeanPostProcessorsBeforeInitialization(wrappedBean, beanName);

    // 初始化方法
    wrappedBean = invokeInitMethods(beanName, wrappedBean, mbd);

    // 后置处理（AOP 代理在这里生成！）
    wrappedBean = applyBeanPostProcessorsAfterInitialization(wrappedBean, beanName);

    return wrappedBean;
}
```

### 7.2 AopAutoProxyConfigurer

这是 Spring AOP 的核心处理器，负责创建代理对象：

```java
public class AopAutoProxyConfigurer {
    public static void configureAutoProxyCreator(
            AspectJAwareAdvisorAutoProxyCreator creator) {
        // 配置代理创建器
    }
}
```

## 八、常见问题

### 8.1 为什么自定义 BPP 不生效？

**可能原因**：

1. BPP 没有被 Spring 扫描到
2. BPP 在 Bean 创建之后才注册
3. BPP 的泛型类型过滤了目标 Bean

### 8.2 BPP 中的异常会影响容器启动吗？

会的。`BeanPostProcessor` 的异常会中断 Bean 的创建过程。如果需要容错，可以使用 `PersistenceExceptionTranslationPostProcessor` 等方式处理。

### 8.3 如何只让 BPP 处理特定包下的 Bean？

```java
@Component
public class PackageFilterBeanPostProcessor implements BeanPostProcessor {
    private final String targetPackage;

    public PackageFilterBeanPostProcessor(String targetPackage) {
        this.targetPackage = targetPackage;
    }

    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        if (bean.getClass().getPackage().getName().startsWith(targetPackage)) {
            // 处理特定包的 Bean
        }
        return bean;
    }
}
```

## 九、面试追问方向

**「BeanPostProcessor 和 BeanFactoryPostProcessor 的区别？」**

前者作用于 Bean 实例（在初始化前后回调），后者作用于 BeanDefinition（在 Bean 创建之前执行）。

**「BeanPostProcessor 可以改变 Bean 的类型吗？」**

`BeanPostProcessor.postProcessBeforeInitialization()` 和 `postProcessAfterInitialization()` 可以返回不同的对象，但实例化是由其他机制完成的。`InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation()` 可以返回非空对象，替代默认的实例化过程。

**「AOP 代理是在哪个阶段生成的？」**

在 `postProcessAfterInitialization()` 阶段生成。

**「@Autowired 为什么能生效？」**

因为 `AutowiredAnnotationBeanPostProcessor` 实现了 `InstantiationAwareBeanPostProcessor`，在 `postProcessProperties()` 方法中处理 `@Autowired` 注解的字段。

---

## 总结

`BeanPostProcessor` 是 Spring 框架最核心的扩展点之一：

1. **执行时机**：在 Bean 初始化前后被调用
2. **核心能力**：可以返回代理对象，替换原始 Bean
3. **内置应用**：
   - `AutowiredAnnotationBeanPostProcessor` → 处理依赖注入
   - `CommonAnnotationBeanPostProcessor` → 处理生命周期注解
   - `ApplicationContextAwareProcessor` → 处理 Aware 回调
   - `AopAutoProxyCreator` → 生成 AOP 代理

理解 `BeanPostProcessor`，是理解 Spring 框架内核的关键。
