# BeanFactoryPostProcessor 与属性占位符替换

import { Badge } from '@rspress/core/theme';

<Badge text="核心原理" type="danger" /> <Badge text="配置扩展" type="tip" /> <Badge text="面试高频" type="warning" />

你有没有想过，Spring 配置中的 `${jdbc.url}` 是怎么变成真实值的？`@Value` 注解是怎么从 properties 文件中读取值的？

这些功能的背后，都有 `BeanFactoryPostProcessor` 的身影。它是 Spring 框架另一个重要的扩展点，允许我们在 Bean 创建之前修改配置。

这一节，我们来深入理解 `BeanFactoryPostProcessor` 和 `PropertyPlaceholderConfigurer` 的原理。

---

## 一、BeanFactoryPostProcessor 是什么？

### 1.1 接口定义

```java
@FunctionalInterface
public interface BeanFactoryPostProcessor {
    // 在 Bean 定义加载完成后、Bean 实例化之前调用
    void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory)
            throws BeansException;
}
```

### 1.2 执行时机

`BeanFactoryPostProcessor` 在容器启动的 `refresh()` 方法中被调用，具体在：

```
refresh()
    ↓
obtainFreshBeanFactory()        ← 创建 BeanFactory
    ↓
prepareBeanFactory()            ← 配置 BeanFactory
    ↓
postProcessBeanFactory()        ← 子类扩展
    ↓
invokeBeanFactoryPostProcessors() ← 执行所有 BFFP
    ↓
registerBeanPostProcessors()    ← 注册 BPP
    ↓
finishBeanFactoryInitialization() ← 创建 Bean
```

**关键点**：`BeanFactoryPostProcessor` 执行时，Bean 实例还未创建，所有 Bean 的 `BeanDefinition` 已经加载完成。

### 1.3 与 BeanPostProcessor 的区别

| 对比项 | BeanFactoryPostProcessor | BeanPostProcessor |
|---|---|---|
| 操作对象 | `BeanDefinition` | Bean 实例 |
| 执行时机 | 所有 Bean 创建之前 | 每个 Bean 初始化前后 |
| 典型应用 | 修改 Bean 定义、属性占位符替换 | 依赖注入、AOP 代理 |

## 二、BeanDefinitionRegistryPostProcessor

### 2.1 接口定义

```java
public interface BeanDefinitionRegistryPostProcessor extends BeanFactoryPostProcessor {
    // 在标准 BeanFactoryPostProcessor 调用之前执行
    // 可以注册新的 BeanDefinition
    void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry)
            throws BeansException;
}
```

### 2.2 核心应用：ConfigurationClassPostProcessor

这是 Spring 最重要的 `BeanDefinitionRegistryPostProcessor`，负责解析 `@Configuration`、`@ComponentScan`、`@Bean` 等注解：

```java
public class ConfigurationClassPostProcessor implements BeanDefinitionRegistryPostProcessor,
        PriorityOrdered, ResourceLoaderAware, EnvironmentAware {

    @Override
    public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) {
        // 解析配置类，生成 BeanDefinition
        processConfigurationClass(new ConfigurationClass());
    }

    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        // 增强 BeanDefinition
        enhanceConfigurationClasses(beanFactory);
    }
}
```

**执行流程**：

```
Spring 启动
    ↓
invokeBeanFactoryPostProcessors()
    ↓
找到所有 BeanDefinitionRegistryPostProcessor
    ↓
先执行 postProcessBeanDefinitionRegistry()
    ├─ ConfigurationClassPostProcessor
    │   ├─ 解析 @ComponentScan
    │   ├─ 解析 @Bean 方法
    │   └─ 解析 @Import
    ↓
再执行 postProcessBeanFactory()
    └─ 增强 @Configuration 类的 CGLIB 代理
```

## 三、PropertyPlaceholderConfigurer：属性占位符替换

### 3.1 基本用法

```xml
<!-- jdbc.properties -->
jdbc.driver=com.mysql.cj.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/test
jdbc.username=root
jdbc.password=123456
```

```xml
<!-- applicationContext.xml -->
<bean class="org.springframework.beans.factory.config.PropertyPlaceholderConfigurer">
    <property name="location" value="classpath:jdbc.properties"/>
</bean>

<bean id="dataSource" class="com.mysql.cj.jdbc.DataSource">
    <property name="url" value="${jdbc.url}"/>
    <property name="username" value="${jdbc.username}"/>
    <property name="password" value="${jdbc.password}"/>
</bean>
```

### 3.2 Spring Boot 方式

```properties
# application.properties
jdbc.driver=com.mysql.cj.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/test
jdbc.username=root
jdbc.password=123456
```

```java
@Value("${jdbc.url}")
private String jdbcUrl;
```

### 3.3 原理分析

`PropertyPlaceholderConfigurer` 的核心原理：

```java
public class PropertyPlaceholderConfigurer
        extends PropertyResourceConfigurer
        implements BeanFactoryPostProcessor {

    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        try {
            // 1. 合并 properties 文件
            Properties mergedProps = mergeProperties();

            // 2. 解析占位符（如 ${jdbc.url}）
            resolvePlaceholders(mergedProps);

            // 3. 遍历所有 BeanDefinition
            String[] beanNames = beanFactory.getBeanDefinitionNames();
            for (String beanName : beanNames) {
                BeanDefinition bd = beanFactory.getBeanDefinition(beanName);

                // 4. 替换 BeanDefinition 中的属性值
                MutablePropertyValues pvs = bd.getPropertyValues();
                for (PropertyValue pv : pvs.getPropertyValues()) {
                    Object value = pv.getValue();
                    if (value instanceof String) {
                        String strVal = (String) value;
                        // 递归解析占位符
                        strVal = resolvePlaceholder(strVal, mergedProps);
                        pvs.addPropertyValue(pv.getName(), strVal);
                    }
                }
            }
        } catch (IOException ex) {
            throw new BeanInitializationException("Could not load properties", ex);
        }
    }
}
```

### 3.4 占位符解析过程

```
原始配置：
    <property name="url" value="${jdbc.url}"/>

第一步：找到 jdbc.properties
    jdbc.url=jdbc:mysql://localhost:3306/test

第二步：替换
    value = "${jdbc.url}" → "jdbc:mysql://localhost:3306/test"
```

## 四、PropertySourcesPropertyResolver

### 4.1 Spring 4 的新方式

Spring 4 之后，属性解析更加灵活，支持多个属性源：

```java
@Configuration
public class AppConfig {
    @Autowired
    private Environment env;

    public DataSource dataSource() {
        DataSource ds = new DataSource();
        // 从 Environment 中获取属性
        ds.setUrl(env.getProperty("jdbc.url"));
        ds.setUsername(env.getProperty("jdbc.username"));
        ds.setPassword(env.getProperty("jdbc.password"));
        return ds;
    }
}
```

### 4.2 @Value 注解的解析

`@Value` 注解是通过 `AutowiredAnnotationBeanPostProcessor` 处理的：

```java
public class AutowiredAnnotationBeanPostProcessor {
    private StringValueResolver valueResolver;

    // 在构建注入元数据时解析 @Value
    protected void buildAutowiringMetadata(Class<?> clazz) {
        ReflectionUtils.doWithLocalFields(clazz, field -> {
            AnnotationAttributes ann = findAutowiredAnnotation(field);
            if (ann != null) {
                if (Modifier.isStatic(field.getModifiers())) {
                    return;
                }

                Object extractedValue = null;

                // 处理 @Value 注解
                if (ann.presentAnnotation(Value.class)) {
                    extractedValue = field.getAnnotation(Value.class).value();
                    // 解析占位符
                    extractedValue = resolveStringValue((String) extractedValue);
                }

                // 构建元数据
                metadata.add(new AutowiredFieldElement(field, extractedValue));
            }
        });
    }
}
```

### 4.3 resolveStringValue() 源码

```java
protected Object resolveStringValue(String value) {
    // 如果是占位符，解析它
    if (getBeanFactory() instanceof ConfigurableBeanFactory) {
        return ((ConfigurableBeanFactory) getBeanFactory())
            .resolveEmbeddedValue(value);
    }
    return value;
}
```

## 五、自定义 BeanFactoryPostProcessor

### 5.1 基本示例

```java
@Component
public class CustomBeanFactoryPostProcessor implements BeanFactoryPostProcessor {
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        System.out.println("BeanFactory 后置处理器执行");
        System.out.println("当前 Bean 定义数量: " + beanFactory.getBeanDefinitionCount());

        // 遍历所有 Bean 定义
        String[] beanNames = beanFactory.getBeanDefinitionNames();
        for (String beanName : beanNames) {
            BeanDefinition bd = beanFactory.getBeanDefinition(beanName);
            System.out.println("  - " + beanName + ": " + bd.getBeanClassName());
        }
    }
}
```

### 5.2 修改 BeanDefinition

```java
@Component
public class ProfileBeanFactoryPostProcessor implements BeanFactoryPostProcessor {
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        // 遍历所有 Bean 定义
        String[] beanNames = beanFactory.getBeanDefinitionNames();
        for (String beanName : beanNames) {
            BeanDefinition bd = beanFactory.getBeanDefinition(beanName);

            // 将所有 Bean 标记为懒加载
            bd.setLazyInit(true);

            // 或者修改 Bean 的属性值
            if (bd.getPropertyValues().contains("enabled")) {
                bd.getPropertyValues().add("enabled", true);
            }
        }
    }
}
```

### 5.3 注册新的 Bean

```java
@Component
public class DynamicBeanRegistrar implements BeanDefinitionRegistryPostProcessor {
    @Override
    public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) {
        // 动态注册 Bean
        AbstractBeanDefinition beanDefinition = new RootBeanDefinition(DynamicService.class);
        beanDefinition.setScope(BeanDefinition.SCOPE_SINGLETON);
        beanDefinition.setLazyInit(false);

        // 注册
        registry.registerBeanDefinition("dynamicService", beanDefinition);
    }

    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        // 可以进一步修改刚注册的 Bean
    }
}
```

## 六、常见问题

### 6.1 属性占位符解析失败

**错误**：`Could not resolve placeholder 'xxx' in string value "${xxx}"`

**原因**：
1. 属性文件未正确加载
2. 占位符名称拼写错误
3. 属性文件中没有对应的 key

**解决方案**：
```java
// 设置默认值
@Value("${jdbc.url:default_url}")
private String jdbcUrl;

// 或者设置忽略解析失败
PropertyPlaceholderConfigurer configurer = new PropertyPlaceholderConfigurer();
configurer.setIgnoreUnresolvablePlaceholders(true);
```

### 6.2 BeanFactoryPostProcessor 执行顺序

可以通过实现 `Ordered` 或 `PriorityOrdered` 接口控制顺序：

```java
@Component
public class OrderedBeanFactoryPostProcessor implements BeanFactoryPostProcessor, Ordered {
    @Override
    public int getOrder() {
        return Ordered.LOWEST_PRECEDENCE;
    }

    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        // 执行逻辑
    }
}
```

## 七、Spring Boot 的自动配置

### 7.1 自动配置原理

Spring Boot 的自动配置是通过 `AutoConfigurationImportSelector` 实现的，它实现了 `BeanDefinitionRegistryPostProcessor`：

```java
public class AutoConfigurationImportSelector
        implements BeanDefinitionRegistryPostProcessor, DeferredImportSelector {

    @Override
    public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) {
        // 注册自动配置类
        AutoConfigurationEntry entry = getAutoConfigurationEntry();
        for (String configurationClassName : entry.getConfigurations()) {
            // 注册为 BeanDefinition
            registerBeanDefinition(configurationClassName, registry);
        }
    }
}
```

### 7.2 执行流程

```
Spring Boot 启动
    ↓
@SpringBootApplication
    ↓
@EnableAutoConfiguration
    ↓
AutoConfigurationImportSelector
    ↓
从 META-INF/spring.factories 加载配置
    ↓
根据 @Conditional 条件过滤
    ↓
注册候选配置类
```

## 八、面试追问方向

**「BeanFactoryPostProcessor 和 BeanPostProcessor 的执行顺序？」**

`BeanFactoryPostProcessor` 先执行（在 Bean 创建之前），`BeanPostProcessor` 后执行（在每个 Bean 初始化前后）。

**「PropertyPlaceholderConfigurer 是在什么时候替换占位符的？」**

在 `postProcessBeanFactory()` 方法中，遍历所有 `BeanDefinition`，替换其中的属性值字符串。

**「如何动态注册 Bean？」**

实现 `BeanDefinitionRegistryPostProcessor`，在 `postProcessBeanDefinitionRegistry()` 方法中使用 `registry.registerBeanDefinition()` 注册。

**「@Configuration 为什么默认是 CGLIB 代理？」**

因为 `@Configuration` 类的方法调用需要返回同一个 Bean 实例。如果不使用代理，每次调用 `@Bean` 方法都会创建新实例，这与 Spring 的单例语义冲突。

---

## 总结

`BeanFactoryPostProcessor` 是 Spring 框架的重要扩展点：

1. **操作对象**：`BeanDefinition`，在 Bean 创建之前执行
2. **核心应用**：
   - `PropertyPlaceholderConfigurer` → 属性占位符替换
   - `ConfigurationClassPostProcessor` → 解析配置类
   - `AutoConfigurationImportSelector` → 自动配置
3. **执行顺序**：先 `BeanDefinitionRegistryPostProcessor`，再 `BeanFactoryPostProcessor`

理解 `BeanFactoryPostProcessor`，能够帮助你实现更强大的配置扩展功能。
