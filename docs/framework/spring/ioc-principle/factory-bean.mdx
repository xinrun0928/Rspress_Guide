# FactoryBean 与 @Bean 注解解析

你知道 `FactoryBean` 和 `BeanFactory` 的区别吗？很多人容易搞混这两个概念。

更重要的是，`@Bean` 注解的方法是如何被 Spring 解析的？为什么 `@Bean` 方法可以直接返回实例，而不需要额外的配置？

这一节，我们来深入理解 `FactoryBean` 的原理和 `@Bean` 注解的解析过程。

## 一、FactoryBean vs BeanFactory

### 1.1 概念对比

| 概念 | 含义 | 层级 |
|---|---|---|
| `BeanFactory` | Spring IoC 容器的顶层接口 | 接口 |
| `FactoryBean` | 创建 Bean 的工厂对象 | Bean 实例 |

**BeanFactory** 是容器的接口，定义了 `getBean()` 等基本方法：

```java
public interface BeanFactory {
    Object getBean(String name);
    <T> T getBean(Class<T> requiredType);
    <T> T getBean(String name, Class<T> requiredType);
    // ...
}
```

**FactoryBean** 是工厂模式的实现，用于创建复杂的 Bean：

```java
public interface FactoryBean<T> {
    // 返回创建的 Bean 实例
    T getObject() throws Exception;

    // 返回 Bean 的类型
    Class<?> getObjectType();

    // 是否单例
    default boolean isSingleton() {
        return true;
    }
}
```

### 1.2 为什么要用 FactoryBean？

**场景一：创建复杂的 Bean**

有些 Bean 的创建过程很复杂，不适合用构造器或静态工厂方法：

```java
@Component
public class MyFactoryBean implements FactoryBean<MyObject> {
    @Override
    public MyObject getObject() {
        // 复杂的创建逻辑
        MyObject obj = new MyObject();
        obj.setConfig(loadConfig());
        obj.initialize();
        return obj;
    }

    @Override
    public Class<?> getObjectType() {
        return MyObject.class;
    }
}
```

**场景二：创建代理对象**

```java
@Component
public class ProxyFactoryBean implements FactoryBean<Object> {
    @Override
    public Object getObject() {
        return Proxy.newProxyInstance(
            getClass().getClassLoader(),
            new Class[]{TargetInterface.class},
            (proxy, method, args) -> {
                // 代理逻辑
                System.out.println("Before method: " + method.getName());
                Object result = method.invoke(target, args);
                System.out.println("After method: " + method.getName());
                return result;
            }
        );
    }

    @Override
    public Class<?> getObjectType() {
        return TargetInterface.class;
    }
}
```

**场景三：动态决定 Bean 类型**

```java
@Component
public class DynamicFactoryBean implements FactoryBean<Object> {
    @Override
    public Object getObject() {
        if (useCache) {
            return new CachedDataSource();
        } else {
            return new SimpleDataSource();
        }
    }

    @Override
    public Class<?> getObjectType() {
        return useCache ? CachedDataSource.class : SimpleDataSource.class;
    }
}
```

## 二、FactoryBean 的使用细节

### 2.1 获取 FactoryBean 本身

当 Bean 名称以 `&` 开头时，Spring 返回 FactoryBean 本身，而不是它创建的 Bean：

```java
// 获取 FactoryBean 创建的 Bean
Object bean = context.getBean("myFactoryBean"); // 返回 getObject() 的结果

// 获取 FactoryBean 本身
FactoryBean<?> factory = context.getBean("&myFactoryBean"); // 返回 FactoryBean 实例
```

### 2.2 源码解析

```java
// AbstractBeanFactory.doGetBean()
protected <T> T doGetBean(String name, Class<T> requiredType, Object[] args, boolean typeCheckOnly) {
    // 1. 转换 Bean 名称
    final String beanName = transformedBeanName(name);

    // 2. 检查 Bean 是否正在创建中
    Object bean = getSingleton(beanName);
    if (bean != null) {
        return bean;
    }

    // 3. 获取 Bean
    // ...

    // 4. 创建 Bean（如果不存在）
    if (mbd.isSingleton()) {
        return createBean(beanName, mbd, args);
    }

    return null;
}

// 关键：transformedBeanName() 处理 & 前缀
protected String transformedBeanName(String name) {
    return BeanFactoryUtils.transformedBeanName(name);
}

public static String transformedBeanName(String name) {
    String beanName = name;
    // 去掉 & 前缀
    while (beanName.startsWith("&")) {
        beanName = beanName.substring(1);
    }
    return beanName;
}
```

### 2.3 Spring 内置的 FactoryBean

**JndiObjectFactoryBean**：从 JNDI 获取对象

```java
@Bean
public DataSource jndiDataSource() {
    JndiObjectFactoryBean bean = new JndiObjectFactoryBean();
    bean.setJndiName("java:comp/env/jdbc/DataSource");
    bean.setExpectedType(DataSource.class);
    return (DataSource) bean.getObject();
}
```

**SqlMapClientFactoryBean**：iBatis/MyBatis 的 SqlMapClient

```xml
<bean class="org.springframework.orm.ibatis.SqlMapClientFactoryBean">
    <property name="configLocation" value="classpath:sql-map-config.xml"/>
    <property name="dataSource" ref="dataSource"/>
</bean>
```

## 三、@Bean 注解解析

### 3.1 @Bean 的基本用法

```java
@Configuration
@ComponentScan("com.example")
public class AppConfig {
    // 方法名就是 Bean 名称
    @Bean
    public UserService userService() {
        return new UserService();
    }

    // 自定义 Bean 名称
    @Bean(name = {"userService", "us"})
    public UserService userServiceImpl() {
        return new UserService();
    }

    // Bean 之间的依赖
    @Bean
    public OrderService orderService(UserService userService) {
        return new OrderService(userService);
    }
}
```

### 3.2 解析过程：ConfigurationClassPostProcessor

`@Bean` 注解是通过 `ConfigurationClassPostProcessor` 解析的：

```java
public class ConfigurationClassPostProcessor
        implements BeanDefinitionRegistryPostProcessor, PriorityOrdered {

    // 第一步：postProcessBeanDefinitionRegistry
    @Override
    public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) {
        processConfigurationClass(new ConfigurationClass());
    }

    // 第二步：postProcessBeanFactory
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        enhanceConfigurationClasses(beanFactory);
    }
}
```

### 3.3 @Bean 方法的扫描

```java
public class ConfigurationClass {
    // 解析配置类中的 @Bean 方法
    private void loadBeanMethodDefinitions(Set<ConfigurationClass> configurationModel) {
        // 1. 获取所有标注了 @Bean 的方法
        for (Method method : getDeclaredMethods()) {
            if (method.isAnnotationPresent(Bean.class)) {
                // 2. 创建 BeanDefinition
                BeanMethod beanMethod = new BeanMethod();
                beanMethod.setConfigurationClass(this);
                beanMethod.setMethod(method);
                beanMethod.setName(method.getName());

                // 3. 添加到配置类
                this.beanMethods.add(beanMethod);
            }
        }
    }
}
```

### 3.4 @Bean 方法的执行

`@Bean` 方法是如何被调用的？Spring 使用了两种方式：

**方式一：普通 @Bean 方法**

```java
@Configuration
public class AppConfig {
    @Bean
    public UserService userService() {
        return new UserService();
    }
}
```

解析后，Spring 会创建一个 `ConfigurationBeanMethod` 的 `BeanDefinition`，当需要 `userService` Bean 时，Spring 会调用 `AppConfig.userService()` 方法。

**方式二：@Configuration + @Bean = CGLIB 代理**

```java
@Configuration
public class AppConfig {
    @Bean
    public UserService userService() {
        return new UserService();
    }

    @Bean
    public OrderService orderService(UserService userService) {
        // 调用 userService() 返回的是容器中的单例
        return new OrderService(userService);
    }
}
```

**问题**：如果不用代理，`orderService()` 调用 `userService()` 会创建新实例，违背单例语义。

**解决方案**：Spring 对 `@Configuration` 类使用 CGLIB 代理，方法调用会被拦截：

```java
// CGLIB 代理后的 AppConfig
public class AppConfig$$EnhancerBySpringCGLIB$$ extends AppConfig {
    private ApplicationContext context;

    @Override
    public UserService userService() {
        // 实际调用容器的 getBean()
        return context.getBean(UserService.class);
    }
}
```

## 四、@Bean 与 FactoryBean 的结合

### 4.1 在 @Bean 中返回 FactoryBean

```java
@Configuration
public class AppConfig {
    @Bean
    public FactoryBean<UserService> userServiceFactory() {
        return new UserServiceFactoryBean();
    }
}
```

### 4.2 效果

```java
// 获取 FactoryBean 创建的 Bean（UserService 实例）
UserService userService = context.getBean(UserService.class);

// 获取 FactoryBean 本身
UserServiceFactoryBean factory = context.getBean("&userServiceFactory", UserServiceFactoryBean.class);
```

## 五、自定义 FactoryBean

### 5.1 实现 FactoryBean 接口

```java
@Component
public class JsonFactoryBean implements FactoryBean<ObjectMapper> {
    @Override
    public ObjectMapper getObject() {
        ObjectMapper mapper = new ObjectMapper();
        mapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false);
        return mapper;
    }

    @Override
    public Class<?> getObjectType() {
        return ObjectMapper.class;
    }

    @Override
    public boolean isSingleton() {
        return true;
    }
}
```

### 5.2 使用 @Bean 方式

```java
@Configuration
public class JacksonConfig {
    @Bean
    public ObjectMapper objectMapper() {
        ObjectMapper mapper = new ObjectMapper();
        mapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false);
        return mapper;
    }
}
```

### 5.3 两种方式的对比

| 对比 | FactoryBean | @Bean |
|---|---|---|
| 适用场景 | 需要实现特定接口的复杂创建逻辑 | 简单的创建逻辑 |
| 灵活性 | 可以实现 `FactoryBean` 接口的回调 | 直接返回对象 |
| 依赖注入 | 需要手动处理 | 可以直接注入其他 Bean |
| 名称控制 | 实现 `getObjectType()` | 方法名即 Bean 名称 |

## 六、面试追问方向

**「FactoryBean 和 BeanFactory 的区别？」**

`BeanFactory` 是 Spring IoC 容器的顶层接口，定义了获取 Bean 的基本方法。`FactoryBean` 是一个工厂 Bean，用于创建复杂的 Bean 实例。

**「@Configuration 为什么默认是 CGLIB 代理？」**

为了保证 `@Bean` 方法调用返回的是容器中的单例 Bean，而不是每次调用都创建新实例。

**「如何获取 FactoryBean 本身？」**

在 Bean 名称前加 `&` 前缀：`context.getBean("&factoryBeanName")`。

**「@Bean 方法可以返回 null 吗？」**

不能。Spring 要求 `@Bean` 方法必须返回非空对象，否则会抛出异常。

---

## 总结

`FactoryBean` 和 `@Bean` 是 Spring 中两种创建 Bean 的方式：

1. **FactoryBean**：实现 `FactoryBean` 接口，适合复杂的创建逻辑，可以返回任意对象
2. **@Bean**：在 `@Configuration` 类中定义方法，适合简单的创建逻辑
3. **关键区别**：`BeanFactory` 是容器，`FactoryBean` 是工厂
4. **@Configuration 代理**：Spring 对 `@Configuration` 类使用 CGLIB 代理，保证单例语义

理解这些概念，能够帮助你更好地设计 Bean 的创建方式。
