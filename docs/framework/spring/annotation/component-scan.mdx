# @ComponentScan 与组件扫描规则

`@ComponentScan` 是 Spring 组件扫描的核心。指定扫描路径后，Spring 会自动发现并注册带有 `@Component`、`@Service`、`@Repository`、`@Controller` 等注解的类。

这一节，我们来学习组件扫描的规则和过滤机制。

## 一、基本用法

### 1.1 指定扫描路径

```java
@Configuration
@ComponentScan("com.example.service")
public class AppConfig { }
```

### 1.2 多个扫描路径

```java
@Configuration
@ComponentScan({"com.example.service", "com.example.dao"})
public class AppConfig { }
```

### 1.3 扫描当前包及子包

```java
@Configuration
@ComponentScan  // 默认扫描当前配置类所在的包
public class AppConfig { }
```

## 二、过滤规则

### 2.1 includeFilters

只扫描符合规则的类：

```java
@Configuration
@ComponentScan(
    basePackages = "com.example",
    includeFilters = @Filter(type = FilterType.ANNOTATION, classes = Service.class)
)
public class AppConfig { }
```

### 2.2 excludeFilters

排除符合规则的类：

```java
@Configuration
@ComponentScan(
    basePackages = "com.example",
    excludeFilters = @Filter(type = FilterType.REGEX, pattern = ".*Test")
)
public class AppConfig { }
```

### 2.3 FilterType

| FilterType | 说明 | 示例 |
|---|---|---|
| ANNOTATION | 注解类型 | `@Filter(ANNOTATION, classes = Controller.class)` |
| ASSIGNABLE_TYPE | Assignable 类型 | `@Filter(ASSIGNABLE_TYPE, classes = BaseService.class)` |
| REGEX | 正则表达式 | `@Filter(REGEX, pattern = ".*Dao")` |
| ASPECTJ | AspectJ 表达式 | `@Filter(ASPECTJ, pattern = "com.example..*Dao+")` |
| CUSTOM | 自定义过滤器 | 实现 `TypeFilter` 接口 |

### 2.4 自定义过滤器

```java
@Configuration
@ComponentScan(
    basePackages = "com.example",
    includeFilters = @Filter(type = FilterType.CUSTOM, classes = MyFilter.class)
)
public class AppConfig { }

public class MyFilter implements TypeFilter {
    @Override
    public boolean match(MetadataReader metadataReader,
            MetadataReaderFactory metadataReaderFactory) throws IOException {
        ClassMetadata classMetadata = metadataReader.getClassMetadata();
        // 只扫描名字以 Dao 结尾的类
        return classMetadata.getClassName().endsWith("Dao");
    }
}
```

## 三、懒扫描

```java
@Configuration
@ComponentScan(lazyInit = true)  // 所有组件懒加载
public class AppConfig { }
```

## 四、组件命名规则

| 类名 | 默认 Bean 名称 |
|---|---|
| `UserService` | `userService` |
| `OrderDAO` | `orderDAO` |
| `URLHelper` | `URLHelper` |

自定义名称：

```java
@Service("customName")
public class UserService { }
```

## 五、常见问题

### 5.1 扫描不到 Bean

检查点：
1. 类是否在扫描路径内
2. 类是否标注了 stereotype 注解
3. 是否有 exclude 过滤

### 5.2 重复扫描

```java
@Configuration
@ComponentScan(basePackages = "com.example", useDefaultFilters = false)
```

`useDefaultFilters = false` 禁用默认过滤，只使用自定义过滤。

## 六、面试追问方向

**「@ComponentScan 的过滤规则有哪些？」**

includeFilters、excludeFilters，支持 ANNOTATION、ASSIGNABLE_TYPE、REGEX、ASPECTJ、CUSTOM 五种类型。

**「Spring 是如何发现组件的？」**

通过 `ComponentScanAnnotationParser` 解析 `@ComponentScan` 注解，然后使用 `ClassPathBeanDefinitionScanner` 扫描指定路径下的类。
