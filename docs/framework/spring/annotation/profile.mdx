# @Profile 环境隔离与多环境配置

在软件开发中，我们通常有 dev、test、prod 等多个环境。每个环境的配置可能不同：数据库连接、开发工具、数据初始化等。

`@Profile` 注解帮助我们实现环境隔离。

---

## 一、基本用法

### 1.1 @Profile 注解

```java
@Configuration
@Profile("dev")
public class DevDataSourceConfig {
    @Bean
    public DataSource dataSource() {
        return new EmbeddedDatabaseBuilder().build();
    }
}

@Configuration
@Profile("prod")
public class ProdDataSourceConfig {
    @Bean
    public DataSource dataSource() {
        return new HikariDataSource();
    }
}
```

### 1.2 激活 Profile

```properties
spring.profiles.active=dev
```

或 Java 代码：

```java
System.setProperty("spring.profiles.active", "dev");
```

### 1.3 @ActiveProfiles（测试）

```java
@SpringBootTest
@ActiveProfiles("test")
public class UserServiceTest { }
```

---

## 二、多 Profile 组合

### 2.1 同时激活多个

```properties
spring.profiles.active=dev,local
```

### 2.2 Profile 继承

```java
@Configuration
@Profile({"dev", "local"})
public class DevLocalConfig { }
```

---

## 三、Profile 特例

### 3.1 @Profile("default")

```java
@Configuration
@Profile("default")  // 没有激活任何 Profile 时生效
public class DefaultConfig { }
```

### 3.2 ! 取反

```java
@Configuration
@Profile("!prod")  // 非 prod 环境生效
public class NonProdConfig { }
```

---

## 四、Profile 配置文件

### 4.1 文件命名

```
application.properties
application-dev.properties
application-test.properties
application-prod.properties
```

### 4.2 @PropertySource 与 Profile

```java
@Configuration
@PropertySource("file:./config-${spring.profiles.active}.properties")
public class AppConfig { }
```

---

## 五、最佳实践

```java
// 主配置
@Configuration
@Profile("!test")  // 测试环境外都需要
public class AppConfig { }

// 开发环境特定
@Configuration
@Profile("dev")
public class DevConfig { }

// 生产环境特定
@Configuration
@Profile("prod")
public class ProdConfig { }
```

## 六、面试追问方向

**「@Profile 的原理是什么？」**

通过 `ProfileCondition` 检查当前激活的 Profile，只有匹配时才注册 Bean。

**「如何切换环境？」**

通过 `spring.profiles.active` 配置属性指定。
