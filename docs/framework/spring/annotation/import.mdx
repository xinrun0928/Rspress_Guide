# @Import 与 ImportSelector 接口

`@Import` 是 Spring 3.0 引入的注解，用于导入外部配置类或组件。它是 Spring Boot 自动配置的基础。

这一节，我们来学习 `@Import` 的使用方法和扩展接口。

---

## 一、基本用法

### 1.1 导入配置类

```java
@Configuration
@Import(AppConfig.class)
public class MainConfig {
    // ...
}
```

### 1.2 导入普通类

```java
@Configuration
@Import(UserService.class)
public class MainConfig {
    // UserService 会被注册为 Bean
}
```

### 1.3 导入多个

```java
@Configuration
@Import({AppConfig.class, DataSourceConfig.class, SecurityConfig.class})
public class MainConfig { }
```

## 二、ImportSelector 接口

### 2.1 接口定义

```java
public interface ImportSelector {
    String[] selectImports(AnnotationMetadata importingClassMetadata);
}
```

### 2.2 使用示例

```java
public class MyImportSelector implements ImportSelector {
    @Override
    public String[] selectImports(AnnotationMetadata importingClassMetadata) {
        // 根据注解元数据动态选择要导入的配置类
        Map<String, Object> attributes = importingClassMetadata
            .getAnnotationAttributes(EnableMyFeature.class.getName());

        if (attributes != null) {
            return new String[]{
                "com.example.config.FeatureAConfig",
                "com.example.config.FeatureBConfig"
            };
        }
        return new String[0];
    }
}

@Retention(RetentionPolicy.RUNTIME)
@Import(MyImportSelector.class)
public @interface EnableMyFeature { }
```

### 2.3 动态导入

```java
public class ConditionalImportSelector implements ImportSelector {
    @Override
    public String[] selectImports(AnnotationMetadata importingClassMetadata) {
        if (ClassUtils.isPresent("com.fasterxml.jackson.databind.ObjectMapper", null)) {
            return new String[]{"com.example.config.JacksonConfig"};
        } else {
            return new String[]{"com.example.config.GsonConfig"};
        }
    }
}
```

## 三、ImportBeanDefinitionRegistrar 接口

### 3.1 接口定义

```java
public interface ImportBeanDefinitionRegistrar {
    void registerBeanDefinitions(AnnotationMetadata importingClassMetadata,
            BeanDefinitionRegistry registry);
}
```

### 3.2 使用示例

```java
public class MyBeanDefinitionRegistrar implements ImportBeanDefinitionRegistrar {
    @Override
    public void registerBeanDefinitions(AnnotationMetadata importingClassMetadata,
            BeanDefinitionRegistry registry) {
        // 动态注册 Bean
        RootBeanDefinition beanDefinition = new RootBeanDefinition(MyBean.class);
        beanDefinition.getPropertyValues().add("property", "value");
        registry.registerBeanDefinition("myBean", beanDefinition);
    }
}

@Retention(RetentionPolicy.RUNTIME)
@Import(MyBeanDefinitionRegistrar.class)
public @interface EnableMyFeature { }
```

## 四、DeferredImportSelector 接口

### 4.1 与 ImportSelector 的区别

`DeferredImportSelector` 在所有配置类处理完成后执行，用于收集要导入的配置。

### 4.2 Spring Boot 自动配置

Spring Boot 的自动配置大量使用 `DeferredImportSelector`：

```java
public class AutoConfigurationImportSelector
        implements DeferredImportSelector {

    @Override
    public String[] selectImports(AnnotationMetadata importingClassMetadata) {
        // 从 META-INF/spring.factories 加载自动配置类
        return getCandidateConfigurations(importingClassMetadata).stream()
            .filter(isConfiguration)
            .filter(isAutoConfiguration)
            .toArray(String[]::new);
    }
}
```

## 五、组合使用

```java
@Configuration
@Import({AppConfig.class, MyImportSelector.class, MyBeanDefinitionRegistrar.class})
public class MainConfig { }
```

## 六、面试追问方向

**「@Import 的作用是什么？」**

导入外部配置类或组件，支持 ImportSelector、ImportBeanDefinitionRegistrar 等扩展。

**「ImportSelector 和 ImportBeanDefinitionRegistrar 的区别？」**

ImportSelector 通过返回类名数组导入配置；ImportBeanDefinitionRegistrar 直接操作 BeanDefinitionRegistry 注册 Bean。

**「Spring Boot 自动配置是如何实现的？」**

通过 `@Import(AutoConfigurationImportSelector)` 导入自动配置类，再通过 `@Conditional` 注解条件过滤。
