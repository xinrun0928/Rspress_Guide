# Spring 注解与配置

Spring 的配置方式经历了从 XML 到注解的演进。现在的 Spring 项目，几乎看不到 XML 配置文件了，全部用注解搞定。

这一章，我们来学习 Spring 注解与配置的核心机制，搞清楚 `@Configuration`、`@ComponentScan`、`@Conditional` 这些注解是怎么工作的。

## 核心思想

Spring 的注解配置，本质上是**约定大于配置**的体现。通过注解，Spring 告诉开发者「这样做就行」，减少了大量模板配置。

理解这些注解的底层原理，能帮助你在遇到问题时快速定位原因。
