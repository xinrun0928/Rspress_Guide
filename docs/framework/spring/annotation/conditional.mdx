# @Conditional 条件装配与底层实现

`@Conditional` 是 Spring 4.0 引入的注解，用于根据条件决定是否注册 Bean。它是 Spring Boot 自动配置的核心机制。

这一节，我们来学习 `@Conditional` 的使用方法和底层实现。

---

## 一、基本用法

### 1.1 @Conditional 注解

```java
@Configuration
public class AppConfig {
    @Bean
    @Conditional(OnClassCondition.class)  // 当条件满足时注册
    public DataSource dataSource() {
        return new DruidDataSource();
    }
}
```

### 1.2 Condition 接口

```java
public interface Condition {
    boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata);
}
```

### 1.3 实现示例

```java
public class OnClassCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        // 检查类路径是否存在某个类
        return ClassUtils.isPresent("com.alibaba.druid.pool.DruidDataSource",
            getClass().getClassLoader());
    }
}
```

## 二、Spring 内置的 @Conditional

### 2.1 @ConditionalOnClass

当类路径存在指定类时才注册 Bean：

```java
@Configuration
@ConditionalOnClass(name = "com.fasterxml.jackson.databind.ObjectMapper")
public class JacksonConfig {
    @Bean
    public ObjectMapper objectMapper() {
        return new ObjectMapper();
    }
}
```

### 2.2 @ConditionalOnMissingClass

当类路径不存在指定类时才注册：

```java
@Configuration
@ConditionalOnMissingClass("com.fasterxml.jackson.databind.ObjectMapper")
public class JacksonFallbackConfig {
    @Bean
    public ObjectMapper objectMapper() {
        return new Gson();
    }
}
```

### 2.3 @ConditionalOnProperty

当配置属性满足条件时才注册：

```java
@Configuration
@ConditionalOnProperty(name = "feature.enabled", havingValue = "true")
public class FeatureConfig {
    @Bean
    public FeatureService featureService() {
        return new FeatureService();
    }
}
```

### 2.4 @ConditionalOnBean

当容器中存在指定 Bean 时才注册：

```java
@Configuration
@ConditionalOnBean(DataSource.class)
public class DatabaseConfig {
    @Bean
    public JdbcTemplate jdbcTemplate(DataSource dataSource) {
        return new JdbcTemplate(dataSource);
    }
}
```

## 三、@Conditional 组合注解

Spring Boot 提供了大量条件注解：

| 注解 | 说明 |
|---|---|
| `@ConditionalOnClass` | 类路径有指定类 |
| `@ConditionalOnMissingClass` | 类路径无指定类 |
| `@ConditionalOnProperty` | 属性满足条件 |
| `@ConditionalOnBean` | 存在指定 Bean |
| `@ConditionalOnMissingBean` | 不存在指定 Bean |
| `@ConditionalOnExpression` | SpEL 表达式为 true |
| `@ConditionalOnWebApplication` | 是 Web 应用 |

## 四、自定义条件注解

### 4.1 定义组合注解

```java
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Conditional(OperatingSystemCondition.class)
public @interface ConditionalOnOS {
    String value();
}

public class OperatingSystemCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        String os = context.getEnvironment().getProperty("os.name");
        String requiredOS = (String) metadata.getAnnotationAttributes(
            ConditionalOnOS.class.getName()).get("value");
        return os.contains(requiredOS);
    }
}
```

### 4.2 使用

```java
@Configuration
@ConditionalOnOS(value = "Linux")
public class LinuxConfig {
    @Bean
    public FileSystem linuxFileSystem() {
        return FileSystems.getDefault();
    }
}
```

## 五、源码解析

### 5.1 ConfigurationClassPostProcessor 处理

```java
public class ConfigurationClassPostProcessor {
    public void processConfigBeanDefinitions(ConfigurationClass configClass) {
        // 处理每个配置类
        for (BeanMethod beanMethod : configClass.getBeanMethods()) {
            // 检查 @Conditional 条件
            if (beanMethod.getMetadata()
                    .isAnnotated(Conditional.class.getName())) {
                // 执行条件检查
                ConditionContextImpl context = new ConditionContextImpl(...);
                ConditionOutcome outcome = ConditionEvaluator
                    .evaluate(beanMethod.getMetadata(), context);
                if (!outcome.isMatch()) {
                    // 条件不满足，跳过该 Bean
                    continue;
                }
            }
        }
    }
}
```

## 六、面试追问方向

**「@Conditional 的原理是什么？」**

通过 `ConditionEvaluator` 在 Bean 注册前检查条件，只有条件满足时才注册。

**「Spring Boot 自动配置是如何实现的？」**

通过 `@Conditional` 系列注解，根据类路径、配置属性、Bean 存在性等条件动态注册 Bean。
