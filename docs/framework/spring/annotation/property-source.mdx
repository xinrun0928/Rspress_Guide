# Spring 属性注入：@Value 与 @PropertySource

你有没有遇到过这种场景：项目上线前，测试环境的数据库密码突然要改，你不得不在代码里搜索「root」这个字符串，祈祷自己能找到所有硬编码的地方。

更糟糕的是，你改了一处，却发现还有三处没改——因为它们分散在不同的文件里。

这就是为什么 Spring 要设计一套完整的属性注入机制。

这一节，我们来彻底掌握 Spring 的属性注入：`@Value`、`@PropertySource`、`@PropertySources`，以及如何用 `Environment` 接口灵活获取配置。

## 一、@Value 注解：基本用法

`@Value` 是最常用的属性注入注解，可以从配置文件、环境变量、甚至 SpEL 表达式中读取值。

### 1.1 读取配置文件属性

```java
@Component
public class DatabaseConfig {

    @Value("${db.url}")
    private String dbUrl;

    @Value("${db.username}")
    private String username;

    @Value("${db.password}")
    private String password;
}
```

这里的 `${...}` 是**占位符语法**，Spring 会从 `Environment` 中查找对应的属性值。

### 1.2 字面量注入

`@Value` 也可以直接注入字面量值：

```java
@Value("Hello World")
private String literal;

@Value("100")
private int number;

@Value("true")
private boolean enabled;
```

但这种用法比较少见，通常还是从配置文件中读取更有意义。

### 1.3 带默认值的属性

如果配置文件里没有某个属性，可以指定默认值：

```java
@Value("${app.port:8080}")
private int port;

@Value("${app.name:MyApp}")
private String appName;

@Value("${app.debug:false}")
private boolean debug;
```

冒号 `:` 后面就是默认值。如果不指定默认值而属性又不存在，Spring 启动时会报错。

:::warning
默认值中使用逗号需要转义

如果默认值中包含逗号，需要用反斜杠转义：

```java
@Value("${app.hosts:192.168.1.1\\,192.168.1.2}")
private List<String> hosts;
```

:::

### 1.4 类型自动转换

`@Value` 支持自动类型转换，不需要手动转换：

```java
@Value("${app.port}")
private int port;  // String -> int

@Value("${app.enabled}")
private boolean enabled;  // String -> boolean

@Value("${app.rate}")
private double rate;  // String -> double

@Value("${app.list}")
private List<String> list;  // String -> List

@Value("${app.map}")
private Map<String, String> map;  // String -> Map
```

配置文件中的写法：

```properties
app.list=a,b,c
app.map.key1=value1
app.map.key2=value2
```

## 二、SpEL 表达式：动态计算

`@Value` 支持 SpEL（Spring Expression Language），可以编写动态表达式。

### 2.1 基本 SpEL 表达式

```java
// 读取系统属性
@Value("#{systemProperties['user.region']}")
private String region;

// 算术运算
@Value("#{10 * 2}")
private int multiplier;  // 结果：20

// 调用方法
@Value("#{T(java.lang.Math).random()}")
private double random;
```

SpEL 表达式使用 `#{}` 包裹，与占位符 `${}` 区分开来。

### 2.2 复合表达式

可以结合占位符和 SpEL：

```java
@Value("#{'${app.name}'.toUpperCase()}")
private String upperName;
```

先取 `app.name` 的值，再调用 `toUpperCase()` 方法。

### 2.3 三元运算符

```java
@Value("#{'\${app.env}' == 'prod' ? '生产环境' : '非生产环境'}")
private String envDesc;
```

## 三、@PropertySource：加载外部配置

`@PropertySource` 用于加载外部配置文件，是 Spring 3.1 引入的注解。

### 3.1 基本用法

```java
@Configuration
@PropertySource("classpath:app.properties")
public class AppConfig {
    // 现在可以 @Value("${app.name") 读取了
}
```

`@PropertySource` 通常配合 `@Configuration` 使用，这样加载的属性就能被整个应用访问。

### 3.2 多配置文件

使用 `@PropertySources` 加载多个配置文件：

```java
@Configuration
@PropertySources({
    @PropertySource("classpath:app.properties"),
    @PropertySource("classpath:db.properties")
})
public class AppConfig {
}
```

### 3.3 加载 YAML 文件

Spring 5 之后，`@PropertySource` 支持加载 YAML 文件：

```java
@Configuration
@PropertySource(value = "classpath:config.yaml", factory = YamlPropertySourceFactory.class)
public class AppConfig {
}
```

但更推荐使用 Spring Boot 的自动配置——`application.yml` 或 `application.properties` 会自动加载。

### 3.4 编码和容错

```java
@Configuration
@PropertySource(
    value = "classpath:app.properties",
    encoding = "UTF-8",
    ignoreResourceNotFound = true  // 文件不存在时不报错
)
public class AppConfig {
}
```

### 3.5 配置文件示例

```properties
# application.properties
app.name=Guide
app.version=1.0.0
app.description=${app.name} 是指南项目

# 数据库配置
db.url=jdbc:mysql://localhost:3306/guide
db.username=root
db.password=secret

# Redis 配置
redis.host=localhost
redis.port=6379
```

```yaml
# application.yml（Spring Boot 推荐格式）
app:
  name: Guide
  version: 1.0.0
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/guide
    username: root
    password: secret
  redis:
    host: localhost
    port: 6379
```

## 四、Environment 接口：灵活的属性访问

`Environment` 是 Spring 的配置环境抽象，通过它可以编程式地访问配置属性。

### 4.1 获取属性

```java
@Service
public class ConfigService implements EnvironmentAware {

    private Environment environment;

    @Override
    public void setEnvironment(Environment environment) {
        this.environment = environment;
    }

    public void printConfigs() {
        String appName = environment.getProperty("app.name");
        int port = environment.getProperty("app.port", Integer.class);
        String dbUrl = environment.getProperty("db.url", "default-url");
    }
}
```

### 4.2 profiles 支持

`Environment` 支持 Spring Profile，可以根据环境切换配置：

```java
@Configuration
@Profile("dev")
@PropertySource("classpath:application-dev.yml")
public class DevConfig {
}

@Configuration
@Profile("prod")
@PropertySource("classpath:application-prod.yml")
public class ProdConfig {
}
```

### 4.3 判断属性是否存在

```java
boolean hasPort = environment.containsProperty("app.port");

// 或者获取默认值，只有属性不存在时才使用默认值
String value = environment.getRequiredProperty("app.name");  // 不存在则抛异常
```

## 五、@Value vs @ConfigurationProperties

这是面试中经常被问到的问题。

### 5.1 对比表

| 特性 | @Value | @ConfigurationProperties |
|---|---|---|
| 注入方式 | 逐个注入 | 批量注入 |
| 松散绑定 | 不支持 | 支持（user-name = userName） |
| SpEL 表达式 | 支持 | 不支持（默认） |
| 属性校验 | 不支持 | 支持（@Validated） |
| IDE 提示 | 需要手动配置 | 自动生成元信息 |

### 5.2 @ConfigurationProperties 示例

```java
@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {

    private String name;
    private int port;
    private boolean debug;
    private Map<String, String> extra;

    // getters and setters
}
```

配合 `@EnableConfigurationProperties` 使用：

```java
@Configuration
@EnableConfigurationProperties(AppProperties.class)
public class AppConfig {
}
```

### 5.3 何时用哪个

| 场景 | 推荐 |
|---|---|
| 单个属性注入 | @Value |
| 组件配置类（数据库、Redis 等） | @ConfigurationProperties |
| 需要 SpEL 表达式 | @Value |
| 需要属性校验 | @ConfigurationProperties + @Validated |

## 六、面试追问方向

**「@Value 属性值从哪里来？」**

Spring 的属性来源有多个层次，按优先级从高到低：
1. 命令行参数（`--server.port=8080`）
2. ServletConfig/ServletContext 参数
3. JNDI 属性
4. 环境变量
5. 配置文件（application.yml/properties）

后面的配置会覆盖前面的同名属性。

**「@PropertySource 和 @Import 的区别？」**

`@PropertySource` 只加载属性文件，不会注册 Bean。
`@Import` 加载配置类，会注册其中的 Bean 定义。

**「如何实现配置动态刷新？」**

Spring Boot 可以配合 `@RefreshScope` 实现配置刷新：

```java
@Component
@RefreshScope
public class DynamicConfig {

    @Value("${app.name}")
    private String name;
}
```

配合 Spring Cloud Config 使用，可以不重启应用刷新配置。

---

## 总结

Spring 的属性注入机制是一套完整的体系：

- **@Value**：适合单个属性注入，支持字面量、占位符、SpEL 表达式
- **@PropertySource**：加载外部配置文件，支持多文件、编码设置
- **Environment**：编程式访问配置，支持 profiles 和属性判断
- **@ConfigurationProperties**：适合批量注入配置类，支持松散绑定和校验

掌握这套体系，你就能优雅地管理应用配置，再也不用到处搜索硬编码的字符串了。
