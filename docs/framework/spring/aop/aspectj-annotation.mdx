# @AspectJ 注解驱动的 AOP 配置

现在我们终于可以动手写 AOP 代码了。Spring 提供了 `@AspectJ` 注解，让切面的编写变得非常简单。

这一节，我们来全面学习如何使用 `@AspectJ` 注解配置 AOP，包括切点表达式、通知类型、以及常见场景的最佳实践。

## 一、启用 @AspectJ 支持

### 1.1 Java 配置方式

```java
@Configuration
@EnableAspectJAutoProxy  // 启用 @AspectJ 支持
@EnableLoadTimeWeaving  // 如果需要加载时织入
public class AppConfig {
    // ...
}
```

### 1.2 XML 配置方式

```xml
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:aop="http://www.springframework.org/schema/aop">

    <!-- 启用 @AspectJ 支持 -->
    <aop:aspectj-autoproxy/>

    <!-- 或者强制使用 CGLIB -->
    <aop:aspectj-autoproxy proxy-target-class="true"/>

    <!-- 扫描切面类 -->
    <context:component-scan base-package="com.example.aspect"/>
</beans>
```

### 1.3 Spring Boot 方式

Spring Boot 默认支持 `@AspectJ`，只需引入 starter：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-aop</artifactId>
</dependency>
```

## 二、定义切面

### 2.1 基本结构

```java
@Aspect  // 标记为切面
@Component  // 注册为 Spring Bean
public class LoggingAspect {

    // 定义切点
    @Pointcut("execution(* com.example.service..*.*(..))")
    public void servicePointcut() {}

    // 定义通知
    @Before("servicePointcut()")
    public void beforeAdvice(JoinPoint jp) {
        System.out.println("方法执行前");
    }
}
```

### 2.2 切面的生命周期

```java
@Aspect
@Component
public class PerformanceAspect {
    // Spring 默认所有切面都是单例
    // 如果需要每次创建新实例，使用 @Scope("prototype")
}
```

## 三、切点表达式详解

### 3.1 常用切点指示符

| 指示符 | 说明 | 示例 |
|---|---|---|
| `execution` | 方法执行 | `execution(* com.example.service.*.*(..))` |
| `within` | 类型匹配 | `within(com.example.service.*)` |
| `this` | 代理对象类型 | `this(com.example.service.UserService)` |
| `target` | 目标对象类型 | `target(com.example.service.UserService)` |
| `args` | 参数匹配 | `args(Long, ..)` |
| `@within` | 带注解的类型 | `@within(@Transactional)` |
| `@target` | 目标类带注解 | `@target(org.springframework.transaction.annotation.Transactional)` |
| `@annotation` | 方法带注解 | `@annotation(@Transactional)` |
| `@args` | 参数带注解 | `@args(@Valid)` |
| `bean` | Bean 名称 | `bean(userService)` |

### 3.2 execution 详解

```
execution(modifiers? return-type declaring-type? method-name(param) throws?)
```

```java
// 匹配返回类型
execution(void)           // void 返回
execution(String)          // String 返回
execution(* )              // 任意返回

// 匹配方法名
execution(* find*(..))     // find 开头的方法
execution(* *ById(..))     // ById 结尾的方法

// 匹配参数
execution(* *(String))             // 单个 String 参数
execution(* *(String, ..))         // 第一个参数是 String
execution(* *(Integer, String))    // 两个特定参数
execution(* *(..))                  // 任意参数

// 匹配包
execution(* com.example.service.UserService.*(..))     // 精确类
execution(* com.example.service..*.*(..))              // service 包及子包
```

### 3.3 组合切点

```java
@Aspect
public class CombinedAspect {

    // 切点定义
    @Pointcut("execution(* com.example.service..*.*(..))")
    public void serviceLayer() {}

    @Pointcut("@annotation(Transactional)")
    public void transactional() {}

    @Pointcut("execution(* *.save(..))")
    public void saveOperation() {}

    // 组合使用
    @Before("serviceLayer() && transactional()")
    public void beforeTransactional() { }

    @Before("serviceLayer() && saveOperation()")
    public void beforeSave() { }

    // 或者使用 || 和 !
    @Pointcut("!within(com.example.controller..*)")
    public void notController() {}
}
```

## 四、五种通知类型

### 4.1 前置通知 @Before

```java
@Aspect
@Component
public class LoggingAspect {

    @Before("execution(* com.example.service..*.*(..))")
    public void before(JoinPoint jp) {
        // 获取方法签名
        String methodName = jp.getSignature().getName();
        // 获取参数
        Object[] args = jp.getArgs();
        System.out.println("执行方法: " + methodName);
    }

    // 带参数
    @Before("execution(* com.example.service..*.*(..)) && args(id, ..)")
    public void beforeWithId(Long id) {
        System.out.println("方法参数 ID: " + id);
    }
}
```

### 4.2 后置通知 @AfterReturning

```java
@Aspect
@Component
public class LoggingAspect {

    // 方法正常返回后执行
    @AfterReturning(pointcut = "execution(* com.example.service..*.*(..))",
                   returning = "result")
    public void afterReturning(JoinPoint jp, Object result) {
        System.out.println("方法返回: " + result);
    }

    // 限制返回类型
    @AfterReturning(
        pointcut = "execution(* com.example.service..*.find*(..))",
        returning = "result")
    public void afterFindReturning(JoinPoint jp, User result) {
        System.out.println("找到用户: " + result);
    }
}
```

### 4.3 异常通知 @AfterThrowing

```java
@Aspect
@Component
public class ExceptionAspect {

    // 方法抛出异常后执行
    @AfterThrowing(pointcut = "execution(* com.example.service..*.*(..))",
                   throwing = "ex")
    public void afterThrowing(JoinPoint jp, Exception ex) {
        System.out.println("方法异常: " + ex.getMessage());
    }

    // 只捕获特定异常
    @AfterThrowing(
        pointcut = "execution(* com.example.service..*.save(..))",
        throwing = "ex")
    public void afterSaveThrowing(JoinPoint jp, DataAccessException ex) {
        System.out.println("数据库异常: " + ex.getMessage());
    }
}
```

### 4.4 最终通知 @After

```java
@Aspect
@Component
public class ResourceAspect {

    // 方法执行后（无论成功失败）都执行
    @After("execution(* com.example.service..*.connect(..))")
    public void afterFinally() {
        System.out.println("连接已关闭");
    }
}
```

### 4.5 环绕通知 @Around

```java
@Aspect
@Component
public class PerformanceAspect {

    @Around("execution(* com.example.service..*.*(..))")
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        // 前置逻辑
        long start = System.currentTimeMillis();
        String method = pjp.getSignature().toShortString();

        // 执行目标方法
        Object result = pjp.proceed();

        // 后置逻辑
        long duration = System.currentTimeMillis() - start;
        System.out.println(method + " 执行时间: " + duration + "ms");

        return result;
    }

    // 修改参数和返回值
    @Around("@annotation(Cacheable)")
    public Object aroundCacheable(ProceedingJoinPoint pjp, Cacheable cacheable)
            throws Throwable {
        String key = cacheable.key();

        // 尝试从缓存获取
        Object cached = cache.get(key);
        if (cached != null) {
            return cached;
        }

        // 执行方法
        Object result = pjp.proceed();

        // 存入缓存
        cache.put(key, result);

        return result;
    }
}
```

## 五、JoinPoint 和 ProceedingJoinPoint

### 5.1 JoinPoint 方法

```java
@Before("execution(* com.example.service..*.*(..))")
public void advice(JoinPoint jp) {
    // 获取方法签名
    jp.getSignature();           // MethodSignature
    jp.getSignature().getName(); // 方法名
    jp.getSignature().toShortString(); // 简短签名

    // 获取目标对象
    jp.getTarget();              // 目标对象

    // 获取代理对象
    jp.getThis();                // 代理对象

    // 获取参数
    jp.getArgs();                // 参数数组

    // 获取类信息
    jp.getSourceLocation();       // 源码位置
    jp.getStaticPart();          // 静态部分
}
```

### 5.2 ProceedingJoinPoint 方法

```java
@Around("execution(* com.example.service..*.*(..))")
public Object advice(ProceedingJoinPoint pjp) throws Throwable {
    // 执行目标方法
    Object result = pjp.proceed();

    // 带参数执行
    Object[] args = pjp.getArgs();
    args[0] = modifiedArg;
    result = pjp.proceed(args);

    // 多次执行
    for (int i = 0; i < 3; i++) {
        pjp.proceed();
    }

    return result;
}
```

## 六、获取注解信息

### 6.1 方法级注解

```java
@Around("@annotation(Retryable)")
public Object aroundRetryable(ProceedingJoinPoint pjp, Retryable retryable)
        throws Throwable {
    int maxRetries = retryable.maxRetries();
    for (int i = 0; i < maxRetries; i++) {
        try {
            return pjp.proceed();
        } catch (Exception e) {
            if (i == maxRetries - 1) throw e;
        }
    }
    return null;
}
```

### 6.2 类级注解

```java
@Before("@within(Controller)")
public void beforeController(JoinPoint jp) {
    // 获取目标类上的注解
    Controller controller = jp.getTarget().getClass().getAnnotation(Controller.class);
}
```

### 6.3 参数级注解

```java
@Before("@args(Valid)")
public void beforeArgs(JoinPoint jp) {
    // 检查参数上的注解
    for (Object arg : jp.getArgs()) {
        if (arg != null) {
            Valid valid = arg.getClass().getAnnotation(Valid.class);
            // ...
        }
    }
}
```

## 七、常见应用场景

### 7.1 日志记录

```java
@Aspect
@Component
@Slf4j
public class LoggingAspect {

    @Pointcut("within(@org.springframework.stereotype.Service *)")
    public void serviceLayer() {}

    @Before("serviceLayer()")
    public void beforeService(JoinPoint jp) {
        log.info("调用: {}.{}",
            jp.getSignature().getDeclaringTypeName(),
            jp.getSignature().getName());
    }

    @AfterReturning(pointcut = "serviceLayer()", returning = "result")
    public void afterService(JoinPoint jp, Object result) {
        log.info("完成: {}.{} -> {}",
            jp.getSignature().getDeclaringTypeName(),
            jp.getSignature().getName(),
            result);
    }
}
```

### 7.2 性能监控

```java
@Aspect
@Component
public class PerformanceAspect {

    @Around("@annotation(Monitored)")
    public Object aroundMonitored(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.nanoTime();
        try {
            return pjp.proceed();
        } finally {
            long duration = System.nanoTime() - start;
            System.out.println(pjp.getSignature() + " took " +
                (duration / 1_000_000) + "ms");
        }
    }
}
```

### 7.3 参数校验

```java
@Aspect
@Component
public class ValidationAspect {

    @Before("@annotation(ValidParams) && args(..)")
    public void validateParams(JoinPoint jp) {
        for (Object arg : jp.getArgs()) {
            if (arg == null) {
                throw new IllegalArgumentException("参数不能为 null");
            }
        }
    }
}
```

## 八、常见问题

### 8.1 切面不生效

**原因一：切面类没被扫描到**

```java
@ComponentScan("com.example.aspect") // 确保切面类被扫描
```

**原因二：代理目标问题**

```java
@EnableAspectJAutoProxy(proxyTargetClass = true) // 强制使用 CGLIB
```

### 8.2 同类方法调用不生效

```java
@Service
public class UserService {
    public void methodA() {
        // this.methodB() 不会走代理
        this.methodB();
    }

    @Transactional
    public void methodB() {
        // 事务不会生效
    }
}

// 解决方案：注入自身
@Service
public class UserService {
    @Autowired
    private UserService self;

    public void methodA() {
        self.methodB(); // 通过代理调用
    }

    @Transactional
    public void methodB() {
        // 事务生效
    }
}
```

## 九、总结

`@AspectJ` 注解让 AOP 编程变得非常简单：

1. **启用**：`@EnableAspectJAutoProxy`
2. **定义切面**：`@Aspect` + `@Component`
3. **定义切点**：`@Pointcut`
4. **定义通知**：`@Before`、`@After`、`@Around` 等

掌握切点表达式的编写和五种通知类型的使用，就能够应对大多数 AOP 场景。
