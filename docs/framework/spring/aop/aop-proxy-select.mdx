# Spring AOP 代理选择规则

你有没有遇到过这种情况：明明加了 `@Transactional`，但事务就是不生效？明明用了 `@Async`，但方法还是同步执行？

很多时候，这是因为 Spring 选择了「错误」的代理方式——生成了 JDK 代理而不是 CGLIB 代理，或者反过来。

这一节，我们来搞清楚 Spring AOP 是如何选择代理方式的，以及如何正确配置。

---

## 一、ProxyFactory：代理创建的入口

### 1.1 核心类

Spring AOP 使用 `ProxyFactory` 来创建代理对象：

```java
public class ProxyFactory extends ProxyCreatorSupport {
    // 创建代理对象
    public Object getProxy() {
        return createAopProxy().getProxy();
    }

    // 创建 AopProxy
    protected final synchronized AopProxy createAopProxy() {
        // 根据配置决定使用哪种代理
        if (config.isProxyTargetClass() || hasNoUserSuppliedProxyInterfaces()) {
            // 需要代理类 → CGLIB
            return new ObjenesisCglibAopProxy(config);
        } else {
            // 可以代理接口 → JDK 动态代理
            return new JdkDynamicAopProxy(config);
        }
    }
}
```

### 1.2 关键配置

`ProxyFactory` 的关键配置：

```java
// 是否强制使用 CGLIB
proxyFactory.setProxyTargetClass(true);

// 目标对象
proxyFactory.setTarget(target);

// 切面
proxyFactory.addAdvisor(advisor);
```

## 二、自动选择规则

### 2.1 选择逻辑图

```
ProxyFactory.createAopProxy()
       ↓
┌──────────────────────────────────────────────┐
│  条件：config.isProxyTargetClass()            │
│        或 hasNoUserSuppliedProxyInterfaces()  │
└──────────────────────────────────────────────┘
       ↓ True                    ↓ False
┌─────────────────┐     ┌─────────────────────────┐
│  CGLIB 代理      │     │  有接口？               │
│  ObjenesisCglib │     │       ↓                 │
│  AopProxy        │     │   Yes → JDK 代理       │
└─────────────────┘     │   No  → CGLIB 代理     │
                        └─────────────────────────┘
```

### 2.2 条件详解

**条件一：`isProxyTargetClass()`**

配置项 `@EnableAspectJAutoProxy(proxyTargetClass = true/false)`。

**条件二：`hasNoUserSuppliedProxyInterfaces()`**

是否没有用户提供的接口。

```java
private boolean hasNoUserSuppliedProxyInterfaces() {
    // 如果目标类实现了接口（由用户定义）
    if (this.targetClass.getInterfaces().length > 0) {
        // 检查是否有特殊的代理接口（如 Spring 自己的 ScopedObject）
        for (Class<?> ifc : this.targetClass.getInterfaces()) {
            if (!ScopedObject.class.isAssignableFrom(ifc)) {
                return false; // 有接口，不使用 CGLIB
            }
        }
    }
    return true; // 没有接口，使用 CGLIB
}
```

## 三、@EnableAspectJAutoProxy 注解

### 3.1 基本用法

```java
@Configuration
@EnableAspectJAutoProxy  // 默认 proxyTargetClass = false
public class AppConfig { }

@Configuration
@EnableAspectJAutoProxy(proxyTargetClass = true)  // 强制使用 CGLIB
public class AppConfig { }
```

### 3.2 proxyTargetClass 属性

| 配置 | 效果 |
|---|---|
| `proxyTargetClass = false`（默认） | 优先使用 JDK 代理 |
| `proxyTargetClass = true` | 强制使用 CGLIB 代理 |

### 3.3 exposeProxy 属性

```java
@EnableAspectJAutoProxy(exposeProxy = true)
```

当 `exposeProxy = true` 时，可以通过 `AopContext.currentProxy()` 获取当前代理对象：

```java
@Service
public class UserService {
    public void methodA() {
        // 获取当前代理对象
        UserService proxy = (UserService) AopContext.currentProxy();
        // 调用同一类的另一个方法
        proxy.methodB();
    }

    @Transactional
    public void methodB() {
        // 事务会生效，因为通过代理调用
    }
}
```

## 四、Spring Boot 的默认行为

### 4.1 Spring Boot 2.x 的变化

Spring Boot 2.0 之前，`@EnableAspectJAutoProxy` 默认 `proxyTargetClass = false`。

Spring Boot 2.0 之后，**自动设置 `proxyTargetClass = true`**，即默认使用 CGLIB 代理。

### 4.2 为什么 Spring Boot 要改默认值？

**原因一：避免坑**

```java
@Service
public class UserServiceImpl implements UserService {
    @Override
    public void create() {
        // 调用同类中的另一个方法
        doCreate();
    }

    @Transactional
    public void doCreate() {
        // 事务不会生效！
        // 因为 this 调用不走代理
    }
}
```

如果使用 JDK 代理，代理对象只实现了 `UserService` 接口，不包含 `UserServiceImpl` 的内部方法。`this.doCreate()` 实际上调用的是原始对象的方法，不是代理对象的方法。

**原因二：代理 Object 方法**

JDK 代理基于接口，代理类只实现接口定义的方法，无法代理 `equals()`、`hashCode()`、`toString()` 等 Object 方法。

CGLIB 基于继承，可以代理所有非 `final` 方法，包括 Object 方法。

### 4.3 还原为 JDK 代理

如果确实需要使用 JDK 代理：

```java
@Configuration
@EnableAspectJAutoProxy(proxyTargetClass = false)
public class AppConfig { }
```

## 五、实战问题：同类方法调用不生效

### 5.1 问题场景

```java
@Service
public class OrderService {
    public void createOrder() {
        // 调用同类中的另一个方法
        saveOrder(); // 不会走代理！
    }

    @Transactional
    public void saveOrder() {
        // 事务不会生效
        // 因为 this.saveOrder() 调用的是原始对象，不是代理对象
    }
}
```

### 5.2 原因分析

```java
// JDK 代理生成的代理类结构
public class $Proxy12 implements UserService {
    private InvocationHandler h;

    public void createOrder() {
        // createOrder 方法没有被增强
        // 所以 this.saveOrder() 调用的是原始对象
    }
}

// CGLIB 代理生成的代理类结构
public class OrderService$$EnhancerBySpringCGLIB$$ extends OrderService {
    private MethodInterceptor CGLIB$CALLBACK_0;

    public void createOrder() {
        // createOrder 方法也没有被增强（因为没加 @Transactional）
        // this.saveOrder() 仍然调用原始对象
    }

    public void saveOrder() {
        // saveOrder 被增强，事务会生效
    }
}
```

### 5.3 解决方案

**方案一：注入自身**

```java
@Service
public class OrderService {
    @Autowired
    private OrderService self; // 注入自身

    public void createOrder() {
        self.saveOrder(); // 通过代理调用
    }

    @Transactional
    public void saveOrder() {
        // 事务生效
    }
}
```

**方案二：使用 AopContext**

```java
@EnableAspectJAutoProxy(exposeProxy = true)
public class OrderService {
    public void createOrder() {
        ((OrderService) AopContext.currentProxy()).saveOrder();
    }

    @Transactional
    public void saveOrder() {
        // 事务生效
    }
}
```

**方案三：AspectJ（推荐）**

AspectJ 在编译/加载时织入，不存在代理问题。

## 六、代理创建时机

### 6.1 延迟创建

默认情况下，Spring 在容器刷新时就会创建所有单例 Bean 的代理。但如果 Bean 被配置为懒加载，代理也会延迟创建。

```java
@Service
@Lazy  // 懒加载
public class LazyService {
    @Transactional
    public void doSomething() { }
}
```

### 6.2 获取代理对象

有时候需要手动获取代理对象：

```java
@Service
public class UserService {
    public void methodA() {
        // 获取代理对象
        UserService proxy = (UserService) AopContext.currentProxy();
        proxy.methodB();
    }

    @Transactional
    public void methodB() { }
}
```

## 七、面试追问方向

**「Spring 默认使用哪种代理？」**

Spring 3.x 之前默认使用 JDK 代理；Spring 4.x 之后，如果目标类有接口，默认使用 JDK 代理；Spring Boot 2.x 默认强制使用 CGLIB 代理。

**「为什么同类方法调用不生效？」**

因为 `this.method()` 调用的是原始对象，不是代理对象。Spring AOP 是基于代理的，只有通过代理对象调用方法才会被增强。

**「如何获取当前代理对象？」**

使用 `AopContext.currentProxy()`，需要配置 `@EnableAspectJAutoProxy(exposeProxy = true)`。

**「CGLIB 代理有什么限制？」**

无法代理 `final` 类和 `final` 方法。

---

## 总结

Spring AOP 的代理选择规则：

1. **默认行为**：
   - Spring Framework 3.x：`proxyTargetClass = false`，优先 JDK 代理
   - Spring Boot 2.x：`proxyTargetClass = true`，强制 CGLIB 代理

2. **选择条件**：
   - `proxyTargetClass = true` → CGLIB
   - `proxyTargetClass = false` 且有接口 → JDK
   - `proxyTargetClass = false` 且无接口 → CGLIB

3. **最佳实践**：
   - Spring Boot 项目：直接使用默认配置
   - 同类方法调用：注入自身或使用 `AopContext.currentProxy()`
   - 需要代理 Object 方法：必须使用 CGLIB

理解代理选择规则，能够帮助你避免常见的 AOP 陷阱。
