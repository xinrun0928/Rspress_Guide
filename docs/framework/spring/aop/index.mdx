# Spring AOP

你有没有想过，`@Transactional` 为什么能让方法自动开启事务？`@Async` 为什么能让方法异步执行？`@Cacheable` 为什么能自动缓存结果？

这些功能的背后，都是 AOP（面向切面编程）在起作用。

这一章，我们来彻底搞懂 Spring AOP——从核心概念到源码实现，从 JDK 动态代理到 CGLIB，从 AspectJ 到 Spring AOP，帮你建立完整的 AOP 知识体系。

## 面试考察分析

**「请解释一下 AOP 的核心概念」**

这是 AOP 最基础的题目，需要能把五个核心概念用自己的话说清楚：
- **连接点**：程序执行的某个位置，Spring AOP 中只有方法执行是连接点
- **切点**：用于匹配连接点的表达式，决定哪些连接点需要被增强
- **通知**：切面执行的具体逻辑，如「记录日志」
- **切面**：切点和通知的结合，定义了在哪里（切点）做什么（通知）
- **织入**：将切面应用到目标对象的过程

**「JDK 动态代理和 CGLIB 代理有什么区别？」**

需要从原理、使用场景、优缺点三个维度回答：
- JDK 代理：基于接口，代理对象实现目标接口
- CGLIB 代理：基于继承，代理对象继承目标类

**「@Transactional 是怎么实现事务的？」**

需要说出核心流程：注解解析 → 创建代理 → 事务拦截 → 提交/回滚。

## 学习建议

1. **先理解概念**：把 AOP 五个核心概念弄清楚
2. **理解代理原理**：搞懂 JDK 和 CGLIB 两种代理的区别
3. **动手实践**：写一个自定义切面，理解通知的执行顺序
4. **理解事务机制**：@Transactional 是最重要的 AOP 应用

> AOP 是 Spring 框架的核心能力之一。理解了 AOP，你才能真正理解 Spring 的事务管理、安全控制、缓存等高级特性。
