# AOP 核心概念：切点、切面、通知、连接点

AOP（面向切面编程）的概念听起来很抽象，很多教程一上来就讲术语，结果把人越讲越糊涂。

这一节，我们用生活化的例子，把 AOP 的五个核心概念彻底讲清楚。

---

## 一、从生活中的 AOP 说起

想象一下，你要给一座办公楼安装监控系统。你会怎么做？

**方案一：每个房间单独安装**——每个房间都要买设备、布线、维护，成本极高。

**方案二：统一安装**——在楼的入口处统一安装，所有房间共享，成本低，维护简单。

AOP 的思想就是「方案二」。它不是把通用逻辑分散到每个业务方法中，而是把这些通用逻辑「横向」抽取出来，集中处理。

## 二、五大核心概念

### 2.1 连接点（Join Point）

**定义**：程序执行的某个位置。

在 Spring AOP 中，连接点特指**方法执行**。类里的每个方法，都是一个连接点。

```java
public class OrderService {
    public void createOrder() { }     // 连接点 1
    public void cancelOrder() { }     // 连接点 2
    public Order findById() { }       // 连接点 3
}
```

### 2.2 切点（Pointcut）

**定义**：用于匹配连接点的表达式，告诉 AOP「我要增强哪些方法」。

连接点是「所有方法」，切点是从中筛选出「需要被增强的方法」。

```java
// 切点表达式：匹配 com.example.service 包下所有类的所有方法
execution(* com.example.service.*.*(..))
```

### 2.3 通知（Advice）

**定义**：切面要执行的「增强逻辑」。

| 通知类型 | 说明 | 执行时机 |
|---|---|---|
| `@Before` | 前置通知 | 方法执行**前** |
| `@After` | 最终通知 | 方法执行**后**（无论成功失败） |
| `@AfterReturning` | 后置通知 | 方法执行**成功后** |
| `@AfterThrowing` | 异常通知 | 方法执行**抛出异常后** |
| `@Around` | 环绕通知 | 方法执行**前后**都可以干预 |

### 2.4 切面（Aspect）

**定义**：切点和通知的组合。「在哪些位置」+「做什么」。

```java
@Aspect
@Component
public class LoggingAspect {
    // 切点：匹配所有 service 包下的方法
    @Pointcut("execution(* com.example.service.*.*(..))")
    public void servicePointcut() {}

    // 通知：在方法执行前记录日志
    @Before("servicePointcut()")
    public void before(JoinPoint jp) {
        System.out.println("方法执行: " + jp.getSignature());
    }
}
```

### 2.5 织入（Weaving）

**定义**：将切面应用到目标对象，创建代理对象的过程。

```
织入前：
┌─────────────┐
│  目标对象   │
│  OrderService│
│  createOrder │  → 只执行原有逻辑
└─────────────┘

织入后：
┌─────────────┐
│  代理对象   │  → 前置通知 → 原逻辑 → 后置通知
│  OrderService│
└─────────────┘
```

Spring AOP 采用**运行时织入**，即程序运行时通过动态代理创建代理对象。

## 三、Spring AOP 的局限

Spring AOP 只能增强**方法执行**这一种连接点。其他 AOP 实现（如 AspectJ）支持更丰富的连接点：

| 连接点类型 | Spring AOP | AspectJ |
|---|---|---|
| 方法调用 | ❌ | ✅ |
| 方法执行 | ✅ | ✅ |
| 构造器调用 | ❌ | ✅ |
| 构造器执行 | ❌ | ✅ |
| 字段读写 | ❌ | ✅ |
| 异常处理 | ❌ | ✅ |

## 四、切点表达式详解

### 4.1 基本语法

```
execution([修饰符] 返回类型 [类名.]方法名(参数) [异常])
```

### 4.2 常用表达式

```java
// 匹配所有 public 方法
execution(public * *(..))

// 匹配返回类型为 String 的方法
execution(String *(..))

// 匹配 com.example.service.UserService 的所有方法
execution(* com.example.service.UserService.*(..))

// 匹配 com.example.service 包下所有类的方法
execution(* com.example.service.*.*(..))

// 匹配 com.example.service.. 下所有子包的所有方法
execution(* com.example.service..*(..))

// 匹配第一个参数为 String 的方法
execution(* *(String, ..))

// 匹配以 find 开头的方法
execution(* find*(..))
```

### 4.3 切点组合

```java
@Pointcut("execution(* com.example.service.*.*(..))")
public void serviceLayer() {}

@Pointcut("execution(* com.example.dao.*.*(..))")
public void daoLayer() {}

@Pointcut("serviceLayer() && !toString()")
public void serviceNoToString() {}

@Pointcut("serviceLayer() || daoLayer()")
public void dataLayer() {}
```

## 五、通知执行顺序

### 5.1 单个切面的执行顺序

```java
@Aspect
@Component
public class LoggingAspect {

    @Around("execution(* com.example.service.*.*(..))")
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        System.out.println("环绕通知 - 前");
        Object result = pjp.proceed(); // 执行目标方法
        System.out.println("环绕通知 - 后");
        return result;
    }

    @Before("execution(* com.example.service.*.*(..))")
    public void before() {
        System.out.println("前置通知");
    }

    @AfterReturning("execution(* com.example.service.*.*(..))")
    public void afterReturning() {
        System.out.println("后置通知");
    }

    @AfterThrowing("execution(* com.example.service.*.*(..))")
    public void afterThrowing() {
        System.out.println("异常通知");
    }

    @After("execution(* com.example.service.*.*(..))")
    public void after() {
        System.out.println("最终通知");
    }
}
```

**正常执行顺序**：
```
环绕通知 - 前
前置通知
[目标方法执行]
后置通知
最终通知
环绕通知 - 后
```

**异常执行顺序**：
```
环绕通知 - 前
前置通知
[目标方法执行 - 抛出异常]
异常通知
最终通知
环绕通知 - 后（异常会抛出）
```

### 5.2 多个切面的执行顺序

当有多个切面时，Spring 按以下规则排序：

1. **按 `@Order` 注解的值**（值越小优先级越高）
2. **如果都用 `@Order`，按字典顺序**

```java
@Aspect
@Component
@Order(1) // 优先级最高
public class Aspect1 { }

@Aspect
@Component
@Order(2) // 优先级次之
public class Aspect2 { }
```

## 六、JoinPoint 对象

在通知方法中，可以注入 `JoinPoint` 对象获取方法信息：

```java
@Aspect
@Component
public class LoggingAspect {

    @Before("execution(* com.example.service.*.*(..))")
    public void before(JoinPoint jp) {
        // 获取方法签名
        String methodName = jp.getSignature().getName();
        String className = jp.getTarget().getClass().getSimpleName();

        // 获取参数
        Object[] args = jp.getArgs();

        // 获取目标对象
        Object target = jp.getTarget();

        System.out.println(className + "." + methodName + " 被调用，参数: " + Arrays.toString(args));
    }
}
```

### 6.1 ProceedingJoinPoint（环绕通知专用）

```java
@Around("execution(* com.example.service.*.*(..))")
public Object around(ProceedingJoinPoint pjp) throws Throwable {
    // 获取方法签名
    System.out.println("方法: " + pjp.getSignature());

    // 获取参数
    Object[] args = pjp.getArgs();

    // 修改参数
    args[0] = "modified";

    // 执行目标方法
    long start = System.currentTimeMillis();
    Object result = pjp.proceed(args); // 可以传递修改后的参数
    long end = System.currentTimeMillis();

    System.out.println("执行时间: " + (end - start) + "ms");
    return result;
}
```

## 七、面试追问方向

**「AOP 和 OOP 的区别是什么？」**

OOP（面向对象编程）主要关注垂直拆分，以类为单位；AOP（面向切面编程）主要关注水平拆分，以切面为单位。两者是互补关系，不是替代关系。

**「Spring AOP 和 AspectJ 的区别？」**

Spring AOP 是运行时织入，只支持方法级别的连接点；AspectJ 是编译时/加载时织入，支持字段、构造器等更丰富的连接点。

**「为什么 Spring AOP 默认只支持方法执行作为连接点？」**

因为 Spring AOP 使用动态代理实现，而动态代理只能代理方法调用。如果需要更丰富的 AOP 能力，需要使用 AspectJ。

## 总结

AOP 的五大核心概念：

| 概念 | 定义 | 类比 |
|---|---|---|
| **连接点** | 程序执行的位置 | 办公室的每个房间 |
| **切点** | 匹配连接点的表达式 | 监控覆盖的区域 |
| **通知** | 增强的逻辑 | 录像功能 |
| **切面** | 切点 + 通知 | 监控系统 |
| **织入** | 应用切面的过程 | 安装监控系统 |
