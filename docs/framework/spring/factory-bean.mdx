## 一、概念对比

| 名称 | 类型 | 作用 |
| --- | --- | --- |
| **BeanFactory** | 接口 | Spring 容器的核心接口，负责创建和管理 Bean |
| **FactoryBean** | 接口 | 工厂 Bean，用于创建复杂或特殊配置的 Bean |

---

## 二、FactoryBean 使用示例

### 定义 FactoryBean

```java
@Component
public class RedisFactoryBean implements FactoryBean<RedisClient> {

    @Value("${redis.host}")
    private String host;

    @Value("${redis.port}")
    private int port;

    @Override
    public RedisClient getObject() {
        // 创建并配置 Redis 客户端
        RedisClient client = new RedisClient(host, port);
        client.connect();
        return client;
    }

    @Override
    public Class<?> getObjectType() {
        return RedisClient.class;
    }

    @Override
    public boolean isSingleton() {
        return true;
    }
}
```

### 获取 Bean

```java
@Autowired
private ApplicationContext context;

// 通过 & 获取 FactoryBean 本身
RedisFactoryBean factory = context.getBean("&redisFactoryBean", RedisFactoryBean.class);

// 获取 FactoryBean 创建的对象
RedisClient client = context.getBean("redisFactoryBean", RedisClient.class);
```

## 三、源码解析

```java title="FactoryBean.java"
public interface FactoryBean<T> {
    // 返回创建的 Bean 对象
    T getObject();

    // 返回 Bean 的类型
    Class<?> getObjectType();

    // 是否单例
    default boolean isSingleton() {
        return true;
    }
}
```

## 四、MyBatis 整合 Spring 的原理

MyBatis 通过 `SqlSessionFactoryBean` 创建 `SqlSessionFactory`：

```java title="SqlSessionFactoryBean.java"
public class SqlSessionFactoryBean implements FactoryBean<SqlSessionFactory> {

    @Override
    public SqlSessionFactory getObject() {
        return buildSqlSessionFactory();
    }

    @Override
    public Class<?> getObjectType() {
        return SqlSessionFactory.class;
    }
}
```

```xml
<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="dataSource" ref="dataSource"/>
    <property name="mapperLocations" value="classpath:mapper/*.xml"/>
</bean>
```

## 五、高频面试题

### 🔴 第一层：FactoryBean 和 BeanFactory 有什么区别？

**答案要点**：
1. BeanFactory 是 Spring 容器的核心接口
2. FactoryBean 是用于创建特殊 Bean 的工厂接口
3. 通过 `&` 前缀可以获取 FactoryBean 本身

### 🟡 第二层：什么场景下需要使用 FactoryBean？

**答案要点**：
1. Bean 的创建过程比较复杂，需要自定义逻辑
2. 需要根据配置动态创建不同类型的 Bean
3. 第三方框架整合到 Spring 时

## 六、常见陷阱

> ⚠️ **陷阱一**：isSingleton 返回值错误

如果 `isSingleton` 返回 `false`，每次获取 Bean 都会调用 `getObject()` 创建新实例。

> ⚠️ **陷阱二**：忘记实现 getObjectType

`getObjectType` 用于类型推断，如果不实现可能导致注入时类型不匹配。
