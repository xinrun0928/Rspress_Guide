# Spring Bean 生命周期全流程

Spring Bean 的生命周期，是面试中最常被问到的问题之一。但大多数面试者只能说出「实例化 → 属性填充 → 初始化」这九个字， 深入追问就答不上来了。

这一节，我们来彻底搞清楚 Spring Bean 生命周期的每个阶段，以及每个阶段提供了哪些扩展点。

---

## 二、详细阶段解析

### 2.1 阶段一：BeanDefinition 注册

在 Spring 容器启动时，首先会扫描并解析配置，生成 `BeanDefinition`，然后注册到 `beanDefinitionMap` 中。

```java
// 简化版的注册逻辑
public class DefaultListableBeanFactory extends AbstractAutowireCapableBeanFactory {
    private final Map<String, BeanDefinition> beanDefinitionMap = new ConcurrentHashMap<>(256);

    public void registerBeanDefinition(String beanName, BeanDefinition beanDefinition) {
        this.beanDefinitionMap.put(beanName, beanDefinition);
    }
}
```

**BeanDefinition 包含的信息**：

| 属性 | 说明 |
|---|---|
| `beanClassName` | Bean 的全限定类名 |
| `scope` | 作用域（singleton/prototype） |
| `lazyInit` | 是否懒加载 |
| `constructorArgumentValues` | 构造器参数值 |
| `propertyValues` | 属性值 |
| `initMethodName` | 初始化方法名 |
| `destroyMethodName` | 销毁方法名 |
| `role` | Bean 的角色（用户/基础设施/内部） |

### 2.2 阶段二：实例化（Instantiation）

容器根据 `BeanDefinition` 创建 Bean 实例。最常见的是通过**构造器**创建。

```java
// AbstractAutowireCapableBeanFactory.createBeanInstance()
// 简化版逻辑
protected BeanWrapper createBeanInstance(String beanName, BeanDefinition mbd, Object[] args) {
    // 1. 使用工厂方法创建
    if (mbd.getFactoryMethodName() != null) {
        return instantiateUsingFactoryMethod(beanName, mbd, args);
    }

    // 2. 使用构造器创建（SmartInstantiationAwareBeanPostProcessor 决定用哪个构造器）
    Constructor<?>[] ctors = determineConstructorsFromBeanPostProcessors(beanClass, beanName);
    if (ctors != null) {
        return autowireConstructor(beanName, mbd, ctors, args);
    }

    // 3. 使用默认构造器
    return instantiateBean(beanName, mbd);
}
```

**扩展点：SmartInstantiationAwareBeanPostProcessor.determineCandidateConstructors()**

如果想自定义选择哪个构造器，可以实现这个方法：

```java
@Component
public class MyBeanPostProcessor implements SmartInstantiationAwareBeanPostProcessor {
    @Override
    public Constructor<?>[] determineCandidateConstructors(
            Class<?> beanClass, String beanName) throws BeansException {
        if (beanClass == UserService.class) {
            // 返回指定的构造器数组
            return new Constructor<?>[] {
                UserService.class.getDeclaredConstructor(String.class),
                UserService.class.getDeclaredConstructor(String.class, Integer.class)
            };
        }
        return null; // 返回 null 使用默认逻辑
    }
}
```

### 2.3 阶段三：属性填充（Population）

实例创建后，Spring 会自动注入标注了 `@Autowired`、`@Resource`、`@Inject` 等注解的字段。

```java
// AbstractAutowireCapableBeanFactory.populateBean()
protected void populateBean(String beanName, BeanDefinition mbd, BeanWrapper bw) {
    // 1. 处理 autowire byName / byType
    if (mbd.getAutowireMode() == AUTOWIRE_BY_NAME ||
        mbd.getAutowireMode() == AUTOWIRE_BY_TYPE) {
        MutablePropertyValues pvs = new MutablePropertyValues(pvs);
        if (mbd.getAutowireMode() == AUTOWIRE_BY_NAME) {
            autowireByName(beanName, mbd, bw, pvs);
        }
        if (mbd.getAutowireMode() == AUTOWIRE_BY_TYPE) {
            autowireByType(beanName, mbd, bw, pvs);
        }
    }

    // 2. 处理 @Autowired、@Value 等注解
    applyPropertyValues(beanName, mbd, bw, pvs);
}
```

### 2.4 阶段四：初始化（Initialization）

初始化是 Bean 生命周期中最复杂的阶段，包含多个扩展点。

#### 2.4.1 Aware 接口回调

Spring 提供了一系列 `Aware` 接口，Bean 可以通过实现这些接口获取容器内部资源：

| Aware 接口 | 获取资源 |
|---|---|
| `BeanNameAware` | 当前 Bean 的名称 |
| `BeanFactoryAware` | BeanFactory 实例 |
| `ApplicationContextAware` | ApplicationContext 实例 |
| `ResourceLoaderAware` | 资源加载器 |
| `MessageSourceAware` | 消息源 |
| `ApplicationEventPublisherAware` | 事件发布器 |

```java
@Service
public class UserService implements BeanNameAware, ApplicationContextAware {
    private String beanName;
    private ApplicationContext applicationContext;

    @Override
    public void setBeanName(String name) {
        this.beanName = name;
        System.out.println("Bean 名称: " + name);
    }

    @Override
    public void setApplicationContext(ApplicationContext context) {
        this.applicationContext = context;
        // 可以获取容器中的其他 Bean
        OtherService other = context.getBean(OtherService.class);
    }
}
```

#### 2.4.2 BeanPostProcessor 前置处理

`BeanPostProcessor.postProcessBeforeInitialization()` 在所有初始化方法之前调用：

```java
@Component
public class MyBeanPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        System.out.println("BeforeInitialization: " + beanName);
        return bean; // 可以返回代理对象
    }
}
```

#### 2.4.3 初始化方法

三种初始化方法的执行顺序：

```
@PostConstruct
    ↓
InitializingBean.afterPropertiesSet()
    ↓
自定义 init-method
```

```java
@Service
public class UserService implements InitializingBean {
    @PostConstruct
    public void init() {
        System.out.println("@PostConstruct 执行");
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("InitializingBean.afterPropertiesSet() 执行");
    }

    public void customInit() {
        System.out.println("自定义 init-method 执行");
    }
}
```

**执行结果**：
```
BeforeInitialization: userService
@PostConstruct 执行
InitializingBean.afterPropertiesSet() 执行
自定义 init-method 执行
AfterInitialization: userService
```

#### 2.4.4 BeanPostProcessor 后置处理

`BeanPostProcessor.postProcessAfterInitialization()` 在所有初始化方法之后调用，通常用于生成 AOP 代理。

```java
@Component
public class MyBeanPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        System.out.println("AfterInitialization: " + beanName);
        return bean;
    }
}
```

## 三、Bean 销毁阶段

当容器关闭时，会调用 Bean 的销毁方法：

```
@PreDestroy
    ↓
DisposableBean.destroy()
    ↓
自定义 destroy-method
```

```java
@Service
public class UserService implements DisposableBean {
    @PreDestroy
    public void preDestroy() {
        System.out.println("@PreDestroy 执行");
    }

    @Override
    public void destroy() throws Exception {
        System.out.println("DisposableBean.destroy() 执行");
    }

    public void customDestroy() {
        System.out.println("自定义 destroy-method 执行");
    }
}
```

**注意**：只有**单例 Bean** 才会自动执行销毁方法。多例 Bean 需要手动调用销毁逻辑。

## 四、完整生命周期代码演示

```java
@Component
public class LifeCycleBean implements InitializingBean, DisposableBean,
        BeanNameAware, BeanFactoryAware, ApplicationContextAware {

    // ========== 注入阶段 ==========

    @Autowired
    private DependencyService dependencyService;

    // ========== 生命周期回调 ==========

    @PostConstruct
    public void postConstruct() {
        System.out.println("4. @PostConstruct");
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("5. InitializingBean.afterPropertiesSet()");
    }

    // 自定义 init-method
    public void customInit() {
        System.out.println("6. customInit-method");
    }

    // ========== 销毁阶段 ==========

    @PreDestroy
    public void preDestroy() {
        System.out.println("@PreDestroy");
    }

    @Override
    public void destroy() throws Exception {
        System.out.println("DisposableBean.destroy()");
    }

    // 自定义 destroy-method
    public void customDestroy() {
        System.out.println("customDestroy-method");
    }

    // ========== Aware 回调 ==========

    @Override
    public void setBeanName(String name) {
        System.out.println("3. BeanNameAware.setBeanName(): " + name);
    }

    @Override
    public void setBeanFactory(BeanFactory beanFactory) throws BeansException {
        System.out.println("3. BeanFactoryAware.setBeanFactory()");
    }

    @Override
    public void setApplicationContext(ApplicationContext applicationContext) throws BeansException {
        System.out.println("3. ApplicationContextAware.setApplicationContext()");
    }
}
```

**执行顺序**：

1. Bean 构造器执行 → 实例化
2. 属性填充 → 注入 dependencyService
3. Aware 接口回调（BeanNameAware → BeanFactoryAware → ApplicationContextAware）
4. BeanPostProcessor.postProcessBeforeInitialization()
5. @PostConstruct
6. InitializingBean.afterPropertiesSet()
7. 自定义 init-method
8. BeanPostProcessor.postProcessAfterInitialization()
9. Bean 正常使用
10. 容器关闭
11. @PreDestroy
12. DisposableBean.destroy()
13. 自定义 destroy-method

## 五、BeanPostProcessor 详解

`BeanPostProcessor` 是 Spring 框架最重要的扩展点之一，贯穿整个 Bean 生命周期。

### 5.1 内置的 BeanPostProcessor

Spring 内置了许多 `BeanPostProcessor`：

| BeanPostProcessor | 作用 |
|---|---|
| `CommonAnnotationBeanPostProcessor` | 处理 `@PostConstruct`、`@PreDestroy`、`@Resource` |
| `AutowiredAnnotationBeanPostProcessor` | 处理 `@Autowired`、`@Value`、`@Inject` |
| `RequiredAnnotationBeanPostProcessor` | 处理 `@Required`（已废弃） |
| `ApplicationContextAwareProcessor` | 处理 Aware 接口回调 |
| `ConfigurationClassPostProcessor` | 处理 `@Configuration` 类 |
| `AopAutoProxyConfigurer` | 处理 AOP 代理 |

### 5.2 自定义 BeanPostProcessor

```java
@Component
public class PerformanceMonitorPostProcessor implements BeanPostProcessor {

    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        System.out.println("初始化前: " + beanName);
        return bean;
    }

    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        System.out.println("初始化后: " + beanName);
        // 可以在这里返回代理对象，实现切面逻辑
        return bean;
    }
}
```

### 5.3 BeanPostProcessor 与 AOP

AOP 代理的生成就是在 `postProcessAfterInitialization` 阶段完成的：

```java
// AbstractAutoWireCapableBeanFactory
protected Object initializeBean(String beanName, Object bean, RootBeanDefinition mbd) {
    // 1. 执行 Aware 回调
    invokeAwareMethods(beanName, bean);

    // 2. 前置处理
    Object wrappedBean = applyBeanPostProcessorsBeforeInitialization(bean, beanName);

    // 3. 初始化
    invokeInitMethods(beanName, wrappedBean, mbd);

    // 4. 后置处理（AOP 代理在这里生成）
    wrappedBean = applyBeanPostProcessorsAfterInitialization(wrappedBean, beanName);

    return wrappedBean;
}
```

## 六、面试追问方向

**「BeanPostProcessor 和 BeanFactoryPostProcessor 的区别是什么？」**

| 对比 | BeanPostProcessor | BeanFactoryPostProcessor |
|---|---|---|
| 作用对象 | Bean 实例 | BeanDefinition |
| 执行时机 | Bean 创建时 | 所有 BeanDefinition 加载完成后 |
| 典型应用 | AOP 代理生成、属性注入 | 修改 BeanDefinition、属性占位符替换 |

**「Spring 如何保证 Aware 回调在属性注入之后执行？」**

在 `AbstractAutowireCapableBeanFactory.initializeBean()` 方法中，先调用 `invokeAwareMethods()` 再调用 `applyBeanPostProcessorsBeforeInitialization()`，而 `@PostConstruct` 等注解处理是在 `init-method` 中执行的。

**「Bean 构造器中能注入依赖吗？」**

可以。通过构造器注入：

```java
@Service
public class UserService {
    private final DependencyService dependencyService;

    public UserService(DependencyService dependencyService) {
        // 在构造器中就可以使用 dependencyService
        this.dependencyService = dependencyService;
        this.dependencyService.doSomething(); // OK
    }
}
```

## 总结

Spring Bean 的生命周期包含完整的创建、初始化、销毁阶段，每个阶段都提供了丰富的扩展点：

- **实例化**：构造器创建对象
- **属性填充**：注入依赖
- **初始化**：Aware 回调 → 前置处理 → 初始化方法 → 后置处理
- **销毁**：销毁回调

理解这些扩展点，能够帮助你在合适的时机插入自定义逻辑，实现更灵活的应用。
