# @Autowired vs @Resource vs @Inject

import { Badge } from '@rspress/core/theme';

<Badge text="注入注解" type="tip" /> <Badge text="装配规则" type="info" />

你用过 `@Autowired`，也用过 `@Resource`，还可能见过 `@Inject`。这三个注解都能注入 Bean，它们有什么区别？什么时候用哪个？

这一节，我们来全面对比这三个注解，搞清楚它们的来源、装配规则和使用场景。

---

## 一、三个注解概览

| 注解 | 来源 | 包名 | 装配规则 |
|---|---|---|---|
| `@Autowired` | Spring | `org.springframework.beans.factory.annotation` | byType → byName |
| `@Resource` | JSR-250（Java 标准） | `javax.annotation` | byName → byType |
| `@Inject` | JSR-330（Java 标准） | `javax.inject` | byType |

---

## 二、@Autowired：Spring 原生的注入注解

### 2.1 基本用法

```java
@Service
public class UserService {
    @Autowired
    private UserDao userDao;

    @Autowired
    private RoleService roleService;
}
```

### 2.2 装配规则

`@Autowired` 的装配规则是：**先按类型匹配，再按名称匹配**。

```
@Autowired
private UserService userService;

执行过程：
1. 按类型查找：找到所有 UserService 类型的 Bean
2. 如果只有一个 → 直接注入
3. 如果有多个 → 按字段名（userService）匹配
4. 如果名称也匹配不上 → 报错 NoUniqueBeanDefinitionException
```

### 2.3 @Autowired 的特性

**1. required 属性**

```java
@Autowired(required = false) // 可选依赖，不存在也不报错
private SmsService smsService;
```

**2. @Qualifier 精确指定**

```java
@Autowired
@Qualifier("userDaoImpl") // 指定 Bean 名称
private UserDao userDao;
```

**3. 可以在构造器、Setter、字段上使用**

```java
@Service
public class UserService {
    private UserDao userDao;
    private RoleService roleService;

    // 构造器
    @Autowired
    public UserService(UserDao userDao) {
        this.userDao = userDao;
    }

    // Setter
    @Autowired
    public void setRoleService(RoleService roleService) {
        this.roleService = roleService;
    }

    // 字段
    @Autowired
    private ConfigService configService;
}
```

**4. 集合类型注入**

```java
@Autowired
private List<Handler> handlers; // 注入所有 Handler 类型的 Bean
```

## 三、@Resource：JSR-250 标准注解

### 3.1 基本用法

```java
@Service
public class UserService {
    @Resource
    private UserDao userDao;

    @Resource(name = "mysqlUserDao") // 指定 Bean 名称
    private UserDao userDao;
}
```

### 3.2 装配规则

`@Resource` 的装配规则是：**先按名称匹配，再按类型匹配**。

```
@Resource
private UserService userService;

执行过程：
1. 先按名称查找：找名为 "userService" 的 Bean
2. 找到了 → 注入
3. 没找到 → 按类型查找 UserService
4. 找到多个 → 报错
```

### 3.3 @Resource 的特性

**1. name 属性指定 Bean 名称**

```java
@Resource(name = "primaryUserDao")
private UserDao userDao;
```

**2. type 属性指定类型**

```java
@Resource(name = "userDao", type = UserDao.class)
private UserDao userDao;
```

**3. 只能用于字段和 Setter**

`@Resource` 不支持构造器注入。

```java
// ❌ 不支持
public class UserService {
    @Resource
    private UserDao userDao;

    // 只能用于 setter
    @Resource
    public void setUserDao(UserDao userDao) {
        this.userDao = userDao;
    }
}
```

## 四、@Inject：JSR-330 标准注解

### 4.1 基本用法

```java
@Service
public class UserService {
    @Inject
    private UserDao userDao;

    @Inject
    private RoleService roleService;
}
```

### 4.2 装配规则

`@Inject` 的装配规则是：**按类型匹配**（与 `@Autowired` 类似）。

```
@Inject
private UserService userService;

执行过程：
1. 按类型查找：找到所有 UserService 类型的 Bean
2. 如果只有一个 → 直接注入
3. 如果有多个 → 按字段名匹配（需要 @Named 配合）
```

### 4.3 @Inject 的特性

**1. 必须配合 @Named 使用才能按名称匹配**

```java
@Inject
@Named("userDaoImpl")
private UserDao userDao;
```

**2. required 属性需要额外处理**

`@Inject` 本身没有 `required` 属性，需要配合 `Provider` 使用：

```java
@Inject
private Provider<SmsService> smsServiceProvider; // 可选依赖

public void send() {
    SmsService sms = smsServiceProvider.get(); // 可能返回 null
    if (sms != null) {
        sms.send("message");
    }
}
```

**3. 需要额外依赖**

使用 `@Inject` 需要引入 `javax.inject` 包：

```xml
<dependency>
    <groupId>javax.inject</groupId>
    <artifactId>javax.inject</artifactId>
    <version>1</version>
</dependency>
```

## 五、三者对比

### 5.1 装配规则对比

| 注解 | 默认行为 | 精确指定名称 |
|---|---|---|
| `@Autowired` | byType → byName | `@Qualifier("name")` |
| `@Resource` | byName → byType | `name = "name"` |
| `@Inject` | byType | `@Named("name")` |

### 5.2 功能对比

| 特性 | @Autowired | @Resource | @Inject |
|---|---|---|---|
| 来源 | Spring | JSR-250 | JSR-330 |
| 构造器注入 | ✅ | ❌ | ✅ |
| 字段注入 | ✅ | ✅ | ✅ |
| Setter 注入 | ✅ | ✅ | ✅ |
| required 属性 | ✅ | ❌（需要用 @Nullable） | ❌（用 Provider） |
| 支持集合注入 | ✅ | ❌ | ✅ |
| 默认装配规则 | byType → byName | byName → byType | byType |

### 5.3 使用场景对比

**@Autowired 使用场景**

```java
// 1. 构造器注入（推荐）
@Service
@RequiredArgsConstructor
public class OrderService {
    private final UserService userService;
    private final OrderDao orderDao;
}

// 2. 可选依赖
@Bean
@Lazy
public SomeService someService() { ... }

@Autowired(required = false)
private SomeService someService;

// 3. 集合注入
@Autowired
private List<Handler> handlers;
```

**@Resource 使用场景**

```java
// 1. 当你想按名称明确指定 Bean 时
@Service
public class UserService {
    @Resource(name = "jdbcUserDao")
    private UserDao userDao;

    @Resource(name = "redisCacheService")
    private CacheService cacheService;
}

// 2. 在非 Spring 环境中也能工作（JSR-250 是 Java 标准）
```

**@Inject 使用场景**

```java
// 1. 想使用 Java 标准注解（跨框架兼容）
// 2. 想在 Spring 和 Guice/Dagger 之间切换

@Inject
private UserDao userDao;

@Inject
private Provider<SmsService> smsServiceProvider;
```

## 六、实战最佳实践

### 6.1 日常开发：用 @Autowired + 构造器注入

```java
@Service
@RequiredArgsConstructor
public class UserService {
    private final UserDao userDao;
    private final RoleService roleService;

    // 简洁、自动生成的构造器
}
```

### 6.2 需要按名称精确指定时：用 @Resource

```java
@Service
public class CacheService {
    // 明确指定使用哪个实现
    @Resource(name = "redisCacheService")
    private CacheService cacheService;
}
```

### 6.3 追求标准兼容时：用 @Inject

```java
@Inject
private UserDao userDao;

@Inject
private RoleService roleService;
```

### 6.4 避免混用

在一个项目中，建议统一使用一种注解。推荐组合：

- **Spring 项目**：使用 `@Autowired` + 构造器注入
- **追求标准**：使用 `@Inject` + `@Named`

## 七、装配顺序详解

### 7.1 @Autowired 的装配顺序

```
1. 按类型在 BeanFactory 中查找所有匹配类型的 Bean
2. 如果找到一个 → 注入
3. 如果找到多个
   ├─ 按字段名匹配（userService 匹配名为 userService 的 Bean）
   ├─ 匹配成功 → 注入
   └─ 匹配失败 → 报错 NoUniqueBeanDefinitionException
4. 如果找不到
   ├─ required = true → 报错 NoSuchBeanDefinitionException
   └─ required = false → 跳过
```

### 7.2 @Resource 的装配顺序

```
1. 如果指定了 name 属性
   ├─ 按 name 查找 → 找到 → 注入
   └─ 找不到 → 报错
2. 如果没指定 name 属性
   ├─ 按字段名查找（fieldName）→ 找到 → 注入
   └─ 找不到 → 按类型查找
       ├─ 按类型找到一个 → 注入
       └─ 按类型找到多个 → 报错
```

### 7.3 @Inject 的装配顺序

```
1. 按类型在 BeanFactory 中查找所有匹配类型的 Bean
2. 如果找到一个 → 注入
3. 如果找到多个
   ├─ 配合 @Named("name") 按名称匹配
   ├─ 匹配成功 → 注入
   └─ 匹配失败 → 报错
```

## 八、常见问题

### 8.1 找不到 Bean 报错

```java
// 错误：NoSuchBeanDefinitionException
@Autowired
private UserDao userDao;

// 解决方案：检查 Bean 是否存在、类型是否匹配、是否被扫描
```

### 8.2 找到多个 Bean 报错

```java
// 错误：NoUniqueBeanDefinitionException
@Autowired
private UserDao userDao; // 存在两个 UserDao 实现

// 解决方案：使用 @Qualifier 精确指定
@Autowired
@Qualifier("mysqlUserDao")
private UserDao userDao;

// 或者标记其中一个为主 Bean
@Primary
@Repository
public class MysqlUserDao implements UserDao { }
```

### 8.3 @Resource 找不到 Bean 报错

```java
// 如果 name 指定了但找不到，会直接报错
@Resource(name = "nonExistentBean")
private SomeService someService; // 报错 NameNotFoundException
```

## 九、面试追问方向

**「@Autowired 和 @Resource 的区别？」**

主要区别在于装配规则：`@Autowired` 先按类型再按名称，`@Resource` 先按名称再按类型。

**「为什么 @Autowired 可以用于构造器，@Resource 不行？」**

`@Resource` 是 JSR-250 标准注解，标准制定时只考虑了字段和 Setter 注入。Spring 在实现时做了扩展，支持构造器注入，但这是 Spring 的私有扩展，不是标准行为。

**「@Inject 和 @Autowired 有什么区别？」**

`@Inject` 是 JSR-330 标准注解，更通用，可以在不同 DI 框架间移植。`@Autowired` 是 Spring 私有注解，功能更丰富（required、@Qualifier 等）。

**「当存在多个同类型 Bean 时怎么办？」**

三种方式：
1. `@Qualifier("beanName")` 精确指定
2. `@Primary` 标记主 Bean
3. `@Autowired` + 字段名自动匹配

---

## 总结

三个注入注解各有特点：

| 注解 | 推荐场景 |
|---|---|
| `@Autowired` | Spring 项目首选，配合构造器注入最佳 |
| `@Resource` | 需要按名称明确指定 Bean 时 |
| `@Inject` | 追求 Java 标准注解，追求框架无关性 |

在实际项目中，**推荐统一使用 `@Autowired` + 构造器注入**，代码清晰、安全、可测试。
