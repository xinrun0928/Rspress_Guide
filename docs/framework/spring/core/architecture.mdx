## 二、核心模块详解

### 2.1 spring-core：根基中的根基

`spring-core` 是整个 Spring 的基础，其他所有模块都依赖它。这个模块包含：

**核心工具类**
- `ObjectUtils`：`Objects` 工具类，包含 null 检查、数组操作等
- `StringUtils`：字符串工具类，空串判断、字符串裁剪等
- `ClassUtils`：类操作工具，获取类名、判断是否内部类等
- `CollectionUtils`：集合工具，判断空集合、集合交集等

**资源抽象**
- `Resource` 接口：统一资源访问（文件、classpath、URL 等）
- `ResourceLoader` 接口：资源加载器
- 相关内容详见 [Spring 资源加载](../advanced/resource)

**类型转换系统**
- `ConversionService`：类型转换服务
- `Converter`、`ConverterFactory`：转换器
- 相关内容详见 [Spring 类型转换](../advanced/type-converter)

**反射与内省**
- `ReflectionUtils`：反射工具类
- `MethodInvoker`：方法调用封装

```java
// spring-core 常用工具示例
public class CoreDemo {
    public static void main(String[] args) {
        // 1. ObjectUtils - null 安全
        String str = null;
        System.out.println(ObjectUtils.isEmpty(str)); // true
        System.out.println(ObjectUtils.identityToString(str)); // null

        // 2. CollectionUtils - 集合判断
        List<String> list = Collections.emptyList();
        System.out.println(CollectionUtils.isEmpty(list)); // true

        // 3. StringUtils - 字符串判断
        System.out.println(StringUtils.hasText("  ")); // false
        System.out.println(StringUtils.hasText("hello")); // true
    }
}
```

### 2.2 spring-beans：Bean 的定义与创建

`spring-beans` 模块是 IoC 容器的核心，它定义了 Bean 的创建和管理机制。

**核心接口**
- `BeanFactory`：IoC 容器的顶层接口，提供 getBean() 方法
- `BeanDefinition`：Bean 的定义信息，包含类名、作用域、初始化方法等
- `BeanDefinitionReader`：读取 Bean 定义

**Bean 定义信息**

在 Spring 中，每个 Bean 都有一个对应的 `BeanDefinition`，它存储了创建 Bean 所需的所有信息：

| 属性 | 说明 | 对应注解/配置 |
|---|---|---|
| `beanClassName` | Bean 的全限定类名 | `@Component("xxx")` |
| `scope` | Bean 的作用域 | `@Scope("singleton")` |
| `lazyInit` | 是否懒加载 | `@Lazy` |
| `initMethodName` | 初始化方法名 | `@PostConstruct` |
| `destroyMethodName` | 销毁方法名 | `@PreDestroy` |
| `propertyValues` | 属性值（用于依赖注入） | `@Autowired` 字段 |

```java
// BeanDefinition 的核心属性
public interface BeanDefinition extends AttributeAccess {
    String getBeanClassName();
    void setBeanClassName(String beanClassName);

    String getScope();
    void setScope(String scope);

    boolean isLazyInit();
    void setLazyInit(boolean lazyInit);

    String getInitMethodName();
    void setInitMethodName(String initMethodName);

    String getDestroyMethodName();
    void setDestroyMethodName(String destroyMethodName);

    MutablePropertyValues getPropertyValues();
}
```

### 2.3 spring-context：IoC 容器实现

`spring-context` 模块在 `spring-beans` 的基础上，构建了完整的 IoC 容器。

**ApplicationContext**

`ApplicationContext` 是我们日常开发中最常用的接口，它继承自 `BeanFactory`，并扩展了：

| 扩展功能 | 说明 |
|---|---|
| 资源加载 | 从 classpath、文件系统、URL 加载资源 |
| 事件发布 | 支持应用内事件机制 |
| 国际化 | 支持多语言消息 |
| 消息源 | 内置 `MessageSource` 实现 |

**常用实现类**

| 实现类 | 适用场景 |
|---|---|
| `ClassPathXmlApplicationContext` | 从 classpath 加载 XML 配置 |
| `FileSystemXmlApplicationContext` | 从文件系统加载 XML 配置 |
| `AnnotationConfigApplicationContext` | 加载注解配置的类 |
| `AnnotationConfigWebApplicationContext` | Web 应用中使用注解配置 |
| `GenericApplicationContext` | 通用容器，支持手动注册 Bean |
| `GenericWebApplicationContext` | Web 环境下的通用容器 |

```java
// 不同 ApplicationContext 的使用方式

// 1. XML 配置（传统方式）
ApplicationContext xmlContext = new ClassPathXmlApplicationContext("application.xml");
UserService userService = xmlContext.getBean(UserService.class);

// 2. 注解配置（现代方式，推荐）
ApplicationContext annoContext = new AnnotationConfigApplicationContext(AppConfig.class);
UserService userService2 = annoContext.getBean(UserService.class);

// 3. 混合配置
ApplicationContext mixedContext = new AnnotationConfigApplicationContext();
mixedContext.getBeanFactory().registerBeanDefinition("dataSource", xmlBeanDefinition);
mixedContext.scan("com.example");
mixedContext.refresh();
```

## 三、Web 相关的模块

### 3.1 spring-web：Web 基础支持

`spring-web` 模块提供了 Web 开发的基础支持，是 Spring MVC 和 Spring WebFlux 的基石。

**核心功能**
-  multipart 文件上传支持
-  请求上下文管理（`RequestContextHolder`）
-  过滤器链配置（`FilterRegistrationBean`）
-  Servlet API 集成

**RequestContextHolder 原理**

Spring MVC 通过 `RequestContextHolder` 将在请求线程中绑定的请求信息存储在线程局部变量中，方便在任何地方获取当前请求：

```java
// 伪代码：RequestContextHolder 的实现
public abstract class RequestContextHolder {
    private static final ThreadLocal<RequestAttributes> requestAttributesHolder =
        new NamedThreadLocal<>("Request attributes");

    public static void setRequestAttributes(RequestAttributes attributes) {
        requestAttributesHolder.set(attributes);
    }

    public static RequestAttributes currentRequestAttributes() {
        RequestAttributes attributes = requestAttributesHolder.get();
        if (attributes == null) {
            throw new IllegalStateException("No thread-bound request found");
        }
        return attributes;
    }
}

// 使用示例：在任何地方获取当前请求
public class UserService {
    public void doSomething() {
        HttpServletRequest request = ((ServletRequestAttributes)
            RequestContextHolder.currentRequestAttributes()).getRequest();
        String token = request.getHeader("Authorization");
        // ...
    }
}
```

### 3.2 spring-webmvc：Spring MVC 框架

`spring-webmvc` 是我们熟悉的 Spring MVC 框架，提供了完整的 Web 请求处理能力。

**核心组件**
- `DispatcherServlet`：前端控制器
- `HandlerMapping`：请求到处理器的映射
- `HandlerAdapter`：处理器适配器
- `ViewResolver`：视图解析器
- `HandlerExceptionResolver`：异常处理器

详细内容见 [Spring MVC](../mvc/) 章节。

## 四、数据访问与事务

### 4.1 spring-jdbc：JDBC 封装

`spring-jdbc` 模块对 JDBC 进行了轻量级封装，消除样板式代码：

```java
// 传统 JDBC：样板代码多，容易出错
Connection conn = null;
PreparedStatement ps = null;
ResultSet rs = null;
try {
    conn = dataSource.getConnection();
    ps = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
    ps.setLong(1, id);
    rs = ps.executeQuery();
    if (rs.next()) {
        user = new User();
        user.setId(rs.getLong("id"));
        user.setName(rs.getString("name"));
    }
} catch (SQLException e) {
    e.printStackTrace();
} finally {
    // 关闭资源...
}

// Spring JDBC：简洁安全
public User findById(Long id) {
    return jdbcTemplate.queryForObject(
        "SELECT * FROM users WHERE id = ?",
        (rs, rowNum) -> {
            User user = new User();
            user.setId(rs.getLong("id"));
            user.setName(rs.getString("name"));
            return user;
        },
        id
    );
}
```

### 4.2 spring-tx：事务管理

`spring-tx` 模块提供了声明式事务和编程式事务两种方式。

**声明式事务**（推荐）

```java
@Service
public class UserService {
    @Transactional(rollbackFor = Exception.class)
    public void transfer(Long fromId, Long toId, BigDecimal amount) {
        // 转出
        accountDao.decrease(fromId, amount);
        // 转入
        accountDao.increase(toId, amount);
    }
}
```

详细内容见 [Spring 事务管理](../aop/transaction)。

## 五、模块依赖关系

理解 Spring 模块的依赖关系，对于做依赖分析和问题排查非常重要：

```
spring-core (所有模块的基石)
    ↓
spring-beans (依赖 spring-core)
    ↓
spring-context (依赖 spring-beans、spring-aop)
    ↓
spring-aop (依赖 spring-context)
    ↓
spring-webmvc (依赖 spring-context、spring-web)
```

**关键点**：
- `spring-core` 不依赖任何其他 Spring 模块
- `spring-beans` 只依赖 `spring-core`
- `spring-context` 依赖 `spring-beans` 和 `spring-aop`
- `spring-webmvc` 依赖 `spring-context` 和 `spring-web`

这意味着，如果你只需要用 Spring 的类型转换功能，只需要引入 `spring-core` 和 `spring-beans` 即可。

## 六、Maven 依赖选择

在实际项目中，通常有两种方式引入 Spring：

**方式一：引入完整的 spring-context**

```xml
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-context</artifactId>
    <version>6.1.0</version>
</dependency>
<!-- spring-context 会自动引入 spring-core、spring-beans、spring-aop -->
```

**方式二：按需引入**

```xml
<!-- 只用 IoC 容器 -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-context</artifactId>
</dependency>

<!-- 只用 JDBC -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-jdbc</artifactId>
</dependency>

<!-- 只用 Web MVC -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-webmvc</artifactId>
</dependency>
```

**Spring Boot 的 Starter**

Spring Boot 提供了各种 Starter，实际开发中推荐使用：

```xml
<!-- Spring Boot 自动引入所有需要的依赖 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

<!-- 等价于引入 -->
<!-- spring-webmvc + spring-context + jackson + tomcat + ... -->
```

## 七、面试追问方向

1. **「spring-context 依赖 spring-aop，这意味着什么？」**
   - 回答：IoC 容器本身依赖于 AOP 能力，用于支持 `@Required`、`@Autowired` 等注解的注入
   - 追问：那 BeanPostProcessor 是在哪个阶段被调用的？

2. **「为什么 Spring 要分层？把 core、beans、context 合成一个 jar 不行吗？」**
   - 回答：分层的好处是按需引入，减小依赖体积；同时各模块职责更清晰

3. **「Spring Framework 和 Spring Boot 有什么区别？」**
   - Spring Framework 是基础框架，提供了 IoC、AOP、MVC 等核心功能
   - Spring Boot 是基于 Spring Framework 的快速启动框架，提供了自动配置、嵌入式服务器等便利

## 总结

这一节我们从全局视角了解了 Spring Framework 的模块划分：

- **spring-core** 是根基，提供核心工具类和类型转换
- **spring-beans** 定义了 Bean 的创建和管理机制
- **spring-context** 构建了完整的 IoC 容器
- **spring-aop** 提供了面向切面编程能力
- **spring-web / spring-webmvc** 提供了 Web 开发支持
- **spring-jdbc / spring-tx** 提供了数据访问和事务管理
