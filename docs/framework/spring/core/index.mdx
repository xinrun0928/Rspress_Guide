如果说 Spring 是一个王国，那么「核心概念」就是这座王国的地基。你可能每天都在用 `@Autowired`、`@Service`、`@Component` 这些注解， 但你有没有想过：**为什么加上这些注解，Spring 就能帮我们管理对象？为什么字段会自动注入？为什么有些 Bean 是单例的，有些是多例的？**

这一章，我们将从最基础的概念出发，搞清楚 Spring 的核心思想和工作方式。无论你是刚入门的新手，还是想深化理解的老手， 这一章都是你必须掌握的基础。

## 面试考察分析

Spring 核心概念是面试中的高频考点，尤其是以下几个问题：

**「请解释一下 Spring 的 IoC 和 DI」**

这不是一道送分题，面试官想看到的不仅仅是背诵概念，而是你能用自己的话说清楚：
- IoC 是一种设计思想，把对象的创建和依赖管理交给容器
- DI 是 IoC 的一种实现方式，容器在创建对象时自动注入依赖
- 你能举出一个生活中的例子，比如乐高积木——你自己「控制」拼装顺序（主动），还是按照说明书让工厂「控制」（被动）

**「Bean 的生命周期是怎样的？」**

这道题考的是你对 Spring 扩展机制的理解。标准答案是：
1. 实例化 → 2. 属性填充 → 3. 初始化 → 4. 正常使用 → 5. 销毁

但更重要的是，你知道**在每个阶段，Spring 提供了哪些扩展点**？比如 `BeanPostProcessor` 可以在初始化前后做自定义处理，`InitializingBean` 接口可以在属性填充完成后做些什么。

**「Spring 如何解决循环依赖？」**

这是一个经典的八股文问题，但能答好的人不多。答案是「三级缓存」，但你要能说清楚：
- 为什么需要三级？两级不够吗？
- 什么是 early singleton？
- 哪些情况下循环依赖无法解决？（构造器注入、多例 Bean）

## 核心概念关系图

```
IoC 容器
├── BeanFactory（顶层接口）
│   └── ApplicationContext（扩展接口）
│       ├── 资源加载
│       ├── 事件发布
│       └── 国际化支持
│
├── Bean 定义
│   ├── @Component / @Service / @Repository / @Controller
│   ├── @Configuration + @Bean
│   └── XML <bean> 定义
│
├── Bean 作用域
│   ├── singleton（单例，默认）
│   ├── prototype（多例）
│   ├── request / session / application（Web 场景）
│   └── 自定义 scope
│
└── 依赖注入方式
    ├── 构造器注入（推荐）
    ├── Setter 注入
    └── 字段注入（不推荐）
```

## 学习路径建议

建议按照以下顺序学习：

1. **先搞懂架构**——知道 Spring 有哪些模块，各模块负责什么
2. **理解 IoC 思想**——这是 Spring 的核心，理解了它，后面的内容就顺理成章
3. **掌握 Bean 生命周期**——知道 Spring 是如何创建和管理 Bean 的
4. **学会注入方式**——知道如何让 Spring 帮你注入依赖，理解各种注入方式的优缺点
5. **理解循环依赖**——这是进阶内容，理解了它，你对 Spring 的理解就上了一个台阶

> 记住：学习 Spring 不是背诵概念，而是理解它为什么这样设计。每当你觉得「这样设计很奇怪」的时候，先别急着否定，想想它的设计初衷是什么。
