# HandlerMapping 与 HandlerAdapter

请求是如何找到对应的处理方法的？`@RequestMapping` 注解是如何被解析的？

这一节，我们来深入理解 HandlerMapping 和 HandlerAdapter 的工作原理。

## 一、HandlerMapping

### 1.1 HandlerMapping 体系

```
HandlerMapping
    ├── AbstractHandlerMapping
    │   ├── RequestMappingHandlerMapping（最常用）
    │   ├── BeanNameUrlHandlerMapping
    │   └── SimpleUrlHandlerMapping
    └── HandlerMapping（接口）
```

### 1.2 RequestMappingHandlerMapping

处理 `@RequestMapping` 注解：

```java
@RequestMapping("/users")
public class UserController {
    @RequestMapping(value = "/{id}", method = RequestMethod.GET)
    public User getUser(@PathVariable Long id) {
        return userService.findById(id);
    }
}
```

### 1.3 RequestMappingInfo

`@RequestMapping` 注解被解析为 `RequestMappingInfo`：

```java
public class RequestMappingInfo {
    private PatternsRequestCondition urlPatterns;
    private RequestMethodsRequestCondition methods;
    private ParamsRequestCondition params;
    private HeadersRequestCondition headers;
    private ConsumesRequestCondition consumes;
    private ProducesRequestCondition produces;
}
```

## 二、HandlerAdapter

### 2.1 HandlerAdapter 体系

```
HandlerAdapter
    ├── RequestMappingHandlerAdapter（最常用）
    ├── HttpRequestHandlerAdapter
    ├── SimpleControllerHandlerAdapter
    └── SimpleServletHandlerAdapter
```

### 2.2 RequestMappingHandlerAdapter

```java
public class RequestMappingHandlerAdapter extends AbstractHandlerMethodAdapter
        implements InitializingBean {

    @Override
    protected ModelAndView handleInternal(HttpServletRequest request,
            HttpServletResponse response, HandlerMethod handlerMethod) {
        // 1. 检查请求方法
        checkRequest(request);

        // 2. 执行预处理
        if (this.sessionAttributeHandler != null) {
            // ...
        }

        // 3. 执行处理方法
        Object returnValue = invokeHandlerMethod(request, response, handlerMethod);

        // 4. 处理返回值
        return getModelAndView(mavContainer, modelFactory, getMessageConverters());
    }
}
```

## 三、HandlerMethod

### 3.1 组成

```java
public class HandlerMethod {
    private final Object bean;
    private final BeanFactory beanFactory;
    private final Class<?> beanType;
    private final Method method;
    private final MethodParameter[] parameters;
}
```

### 3.2 创建过程

```java
// RequestMappingHandlerMapping.getHandlerInternal()
@Override
protected HandlerMethod getHandlerInternal(HttpServletRequest request) {
    String lookupPath = getUrlPathHelper().getLookupPathForRequest(request);
    // 查找匹配的 HandlerMethod
    return lookupHandlerMethod(lookupPath, request);
}
```

## 四、自定义 HandlerMapping

```java
@Component
public class CustomHandlerMapping implements HandlerMapping {
    @Override
    public HandlerExecutionChain getHandler(HttpServletRequest request) {
        String uri = request.getRequestURI();
        if (uri.startsWith("/api/")) {
            return new HandlerExecutionChain(new ApiHandler());
        }
        return null;
    }
}
```

## 五、面试追问方向

**「HandlerMapping 的作用是什么？」**

将请求 URL 映射到具体的处理方法（Handler）。

**「HandlerAdapter 的作用是什么？」**

执行 Handler，返回 ModelAndView。它屏蔽了不同 Handler 类型的差异。

**「@GetMapping 和 @RequestMapping 的关系？」**

`@GetMapping` 是 `@RequestMapping` 的组合注解，指定 `method = RequestMethod.GET`。
