当 Controller 抛出异常时，Spring MVC 是如何统一处理的？

这一节，我们来学习 Spring MVC 的异常处理机制。

---

## 一、异常处理体系

```
HandlerExceptionResolver
    ├── ExceptionHandlerExceptionResolver（@ExceptionHandler）
    ├── ResponseStatusExceptionResolver（@ResponseStatus）
    ├── DefaultHandlerExceptionResolver（Spring 默认异常）
    └── HandlerExceptionResolverComposite
```

---

## 二、@ExceptionHandler

### 2.1 基本用法

```java
@ControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(Exception.class)
    public Result handleException(Exception e) {
        return Result.error(e.getMessage());
    }

    @ExceptionHandler(UserNotFoundException.class)
    public Result handleUserNotFound(UserNotFoundException e) {
        return Result.error(404, e.getMessage());
    }
}
```

### 2.2 获取异常信息

```java
@ExceptionHandler(Exception.class)
public Result handleException(Exception e, HttpServletRequest request) {
    log.error("请求 {} 发生异常", request.getRequestURI(), e);
    return Result.error(e.getMessage());
}
```

---

## 三、@ControllerAdvice

### 3.1 基本用法

```java
@ControllerAdvice
public class GlobalExceptionHandler {
    // 处理所有异常
}
```

### 3.2 限定范围

```java
@ControllerAdvice(assignableTypes = {UserController.class})
public class UserExceptionHandler {
    // 只处理 UserController 的异常
}
```

## 四、ResponseStatusExceptionResolver

```java
@ResponseStatus(HttpStatus.NOT_FOUND)
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message) {
        super(message);
    }
}
```

## 五、全局异常处理最佳实践

```java
@ControllerAdvice
@Slf4j
public class GlobalExceptionHandler {
    @ExceptionHandler(BusinessException.class)
    public Result handleBusiness(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(ValidationException.class)
    public Result handleValidation(ValidationException e) {
        return Result.error(400, e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public Result handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error(500, "系统错误");
    }
}
```

## 六、面试追问方向

**「@ControllerAdvice 的作用是什么？」**

提供全局的异常处理、@ModelAttribute 绑定、@InitBinder 初始化。

**「@ExceptionHandler 的执行顺序？」**

按类中方法声明顺序匹配，先匹配具体的异常类型，再匹配宽泛的异常类型。
