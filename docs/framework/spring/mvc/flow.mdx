# Spring MVC 请求处理流程

当你在浏览器输入一个 URL，敲下回车，Spring MVC 是如何一步步处理这个请求的？

这一节，我们来完整分析 Spring MVC 的请求处理流程。

---

## 一、整体流程图

```
Spring MVC 请求处理流程
═══════════════════════════════════════════════════════════════════════

1. 用户请求
   ↓
2. DispatcherServlet 接收请求
   ↓
3. HandlerMapping 查找 Handler
   ↓
4. HandlerAdapter 执行 Handler
   ↓
5. Handler 返回 ModelAndView
   ↓
6. DispatcherServlet 视图解析
   ↓
7. View 渲染 Model
   ↓
8. 返回响应
   ↓
9. 请求结束
```

---

## 二、九步详解

### 2.1 请求进入 DispatcherServlet

所有请求都进入 `DispatcherServlet`：

```java
// DispatcherServlet.doGet() / doPost()
protected final void doGet(HttpServletRequest request, HttpServletResponse response)
        throws ServletException, IOException {
    processRequest(request, response);
}

protected final void doPost(HttpServletRequest request, HttpServletResponse response)
        throws ServletException, IOException {
    processRequest(request, response);
}

protected final void processRequest(HttpServletRequest request, HttpServletResponse response) {
    doDispatch(request, response);
}
```

### 2.2 HandlerMapping 查找 Handler

```java
// DispatcherServlet.doDispatch()
protected void doDispatch(HttpServletRequest request, HttpServletResponse response) {
    // 1. 获取 Handler
    HandlerExecutionChain mappedHandler = getHandler(request);
}

protected HandlerExecutionChain getHandler(HttpServletRequest request) {
    for (HandlerMapping mapping : handlerMappings) {
        HandlerExecutionChain handler = mapping.getHandler(request);
        if (handler != null) {
            return handler;
        }
    }
    return null;
}
```

### 2.3 HandlerAdapter 执行 Handler

```java
// 2. 获取 HandlerAdapter
HandlerAdapter ha = getHandlerAdapter(mappedHandler.getHandler());

// 3. 执行预处理拦截器
if (!mappedHandler.applyPreHandle(request, response)) {
    return;
}

// 4. 执行 Handler，返回 ModelAndView
ModelAndView mv = ha.handle(request, response, mappedHandler.getHandler());

// 5. 执行后处理拦截器
mappedHandler.applyPostHandle(request, response, mv);
```

### 2.4 视图解析

```java
// 6. 视图解析
processDispatchResult(request, response, mappedHandler, mv, dispatchException);

private void processDispatchResult(...) {
    // 解析视图
    render(mv, request, response);
}
```

## 三、完整流程代码

```java
protected void doDispatch(HttpServletRequest request, HttpServletResponse response) {
    Exception dispatchException = null;
    ModelAndView mv = null;

    try {
        // 1. 获取 Handler
        HandlerExecutionChain mappedHandler = getHandler(request);
        if (mappedHandler == null) {
            noHandlerFound(request, response);
            return;
        }

        // 2. 获取 HandlerAdapter
        HandlerAdapter ha = getHandlerAdapter(mappedHandler.getHandler());

        // 3. 执行预处理拦截器
        if (!mappedHandler.applyPreHandle(request, response)) {
            return;
        }

        // 4. 执行 Handler，返回 ModelAndView
        mv = ha.handle(request, response, mappedHandler.getHandler());

        // 5. 执行后处理拦截器
        mappedHandler.applyPostHandle(request, response, mv);

    } catch (Exception ex) {
        dispatchException = ex;
    } catch (Throwable err) {
        dispatchException = new NestedServletException("Handler dispatch failed", err);
    }

    // 6. 处理结果（渲染视图或处理异常）
    processDispatchResult(request, response, mappedHandler, mv, dispatchException);
}

private void processDispatchResult(...) {
    // 渲染视图
    if (mv != null && !mv.wasCleared()) {
        render(mv, request, response);
    }

    // 处理异常
    if (exception != null) {
        if (exception instanceof NoHandlerFoundException) {
            // 404
        } else {
            // 500
        }
    }
}
```

## 四、各组件职责

| 组件 | 职责 |
|---|---|
| DispatcherServlet | 前端控制器，协调各组件 |
| HandlerMapping | 映射请求到具体的 Handler |
| HandlerAdapter | 执行 Handler，返回 ModelAndView |
| HandlerInterceptor | 预处理/后处理请求 |
| ViewResolver | 解析视图名称到具体视图 |
| ExceptionHandler | 统一处理异常 |

---

## 五、面试追问方向

**「Spring MVC 的核心组件有哪些？」**

DispatcherServlet、HandlerMapping、HandlerAdapter、HandlerInterceptor、ViewResolver、ExceptionHandler。

**「请求处理流程是什么？」**

九步流程：请求 → DispatcherServlet → HandlerMapping → HandlerAdapter → Handler → ModelAndView → ViewResolver → View → 响应。

**「HandlerInterceptor 和 Filter 的区别？」**

Filter 依赖 Servlet API，作用范围更大；HandlerInterceptor 依赖 Spring MVC，粒度更细。
