# HandlerInterceptor 与 Filter 对比

import { Badge } from '@rspress/core/theme';

<Badge text="拦截器" type="tip" /> <Badge text="过滤器" type="info" /> <Badge text="对比分析" type="warning" />

Spring MVC 提供了 `HandlerInterceptor`，Servlet 规范提供了 `Filter`。两者都能拦截请求，有什么区别？

这一节，我们来全面对比 HandlerInterceptor 和 Filter。

---

## 一、基本概念

### 1.1 Filter（过滤器）

Filter 是 Servlet 规范的组件，位于 Web 应用层：

```
请求 → Filter1 → Filter2 → Servlet → Filter2 → Filter1 → 响应
```

### 1.2 HandlerInterceptor（拦截器）

HandlerInterceptor 是 Spring MVC 的组件，位于 Spring MVC 层：

```
请求 → DispatcherServlet → HandlerInterceptor1 → HandlerInterceptor2 → Controller
                                                                              ↓
响应 ← DispatcherServlet ← HandlerInterceptor2 ← HandlerInterceptor1 ← Controller
```

---

## 二、HandlerInterceptor

### 2.1 接口定义

```java
public interface HandlerInterceptor {
    // 预处理，返回 true 继续执行，false 终止
    default boolean preHandle(HttpServletRequest request,
            HttpServletResponse response, Object handler) throws Exception {
        return true;
    }

    // 后处理
    default void postHandle(HttpServletRequest request,
            HttpServletResponse response, Object handler,
            ModelAndView modelAndView) throws Exception {
    }

    // 整个请求处理完成后调用
    default void afterCompletion(HttpServletRequest request,
            HttpServletResponse response, Object handler,
            Exception ex) throws Exception {
    }
}
```

### 2.2 使用示例

```java
@Component
public class AuthInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request,
            HttpServletResponse response, Object handler) throws Exception {
        String token = request.getHeader("Authorization");
        if (token == null) {
            response.sendError(401, "Unauthorized");
            return false;
        }
        return true;
    }
}
```

### 2.3 注册拦截器

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new AuthInterceptor())
            .addPathPatterns("/api/**")
            .excludePathPatterns("/api/login");
    }
}
```

## 三、Filter

### 3.1 接口定义

```java
public interface Filter {
    void init(FilterConfig filterConfig);
    void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain);
    void destroy();
}
```

### 3.2 使用示例

```java
@Component
public class LoggingFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException {
        HttpServletRequest req = (HttpServletRequest) request;
        System.out.println("请求: " + req.getRequestURI());
        chain.doFilter(request, response);
    }
}
```

### 3.3 注册过滤器

```java
@Configuration
public class FilterConfig {
    @Bean
    public FilterRegistrationBean<LoggingFilter> loggingFilter() {
        FilterRegistrationBean<LoggingFilter> bean = new FilterRegistrationBean<>();
        bean.setFilter(new LoggingFilter());
        bean.addUrlPatterns("/api/*");
        return bean;
    }
}
```

## 四、两者对比

| 对比维度 | HandlerInterceptor | Filter |
|---|---|---|
| **来源** | Spring MVC | Servlet 规范 |
| **作用范围** | Spring MVC 范围内 | Web 应用范围内 |
| **执行顺序** | 按注册顺序 | 按过滤链顺序 |
| **参数** | HttpServletRequest/Response | ServletRequest/Response |
| **获取 Bean** | 可以直接注入 | 需要通过 WebApplicationContext |
| **典型应用** | 权限校验、日志记录 | 字符编码、跨域处理 |

---

## 五、执行顺序

### 5.1 Filter 执行顺序

```
请求 → CharacterEncodingFilter → CORSFilter → LoggingFilter → DispatcherServlet
                                                              ↓
响应 ← CharacterEncodingFilter ← CORSFilter ← LoggingFilter ←
```

### 5.2 HandlerInterceptor 执行顺序

```
DispatcherServlet.doDispatch()
    ↓
HandlerInterceptor1.preHandle() → true
    ↓
HandlerInterceptor2.preHandle() → true
    ↓
Controller 执行
    ↓
HandlerInterceptor2.postHandle()
    ↓
HandlerInterceptor1.postHandle()
    ↓
视图渲染
    ↓
HandlerInterceptor2.afterCompletion()
    ↓
HandlerInterceptor1.afterCompletion()
```

## 六、选择使用

| 场景 | 选择 |
|---|---|
| 字符编码 | Filter |
| 跨域处理 | Filter |
| Spring Bean 依赖 | HandlerInterceptor |
| 权限校验 | HandlerInterceptor |
| 请求日志 | 两者皆可 |

## 七、面试追问方向

**「Filter 和 HandlerInterceptor 的区别？」**

Filter 是 Servlet 规范组件，作用范围更大；HandlerInterceptor 是 Spring MVC 组件，粒度更细，可以获取 Spring 容器中的 Bean。

**「Filter 的执行顺序由什么决定？」**

由 `FilterRegistrationBean` 的添加顺序或 `@Order` 注解决定。

**「HandlerInterceptor.postHandle() 和 afterCompletion() 的区别？」**

`postHandle()` 在 Controller 执行后、视图渲染前调用；`afterCompletion()` 在视图渲染完成后调用。
