# Spring MVC

Spring MVC 是 Spring 框架的 Web 层模块，负责处理 HTTP 请求和响应。从请求进入的那一刻起，到响应返回给客户端，Spring MVC 经历了怎样的处理流程？

这一章，我们来全面学习 Spring MVC 的工作原理、核心组件和实战技巧。

## 知识图谱

```
Spring MVC 架构
─────────────────────────────────────────────────────────────────────

┌─────────────────────────────────────────────────────────────────┐
│                        DispatcherServlet                          │
│  1. HandlerMapping 找到 Handler                                  │
│  2. HandlerAdapter 执行 Handler                                 │
│  3. ViewResolver 解析视图                                       │
│  4. 异常统一处理                                                │
└─────────────────────────────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────────────────────────────┐
│                         核心组件                                  │
│  HandlerMapping     → 映射请求到 Handler                         │
│  HandlerAdapter     → 执行 Handler，返回 ModelAndView            │
│  HandlerInterceptor → 拦截请求，做预处理                         │
│  ViewResolver       → 解析视图名称到具体视图                     │
│  ExceptionHandler   → 统一处理异常                              │
└─────────────────────────────────────────────────────────────────┘
```

## 面试考察分析

**「Spring MVC 的请求处理流程是什么？」**

标准答案：9 步流程图。从请求进入 `DispatcherServlet` 到最后返回视图。

**「DispatcherServlet 的核心方法是什么？」**

`doDispatch()`，它协调各个组件完成请求处理。

**「@RequestParam 和 @RequestBody 的区别？」**

`@RequestParam` 用于 GET/POST 的 query string 和 form data，`@RequestBody` 用于请求体（通常用于 JSON）。

理解 Spring MVC 的工作原理，是成为 Web 开发高手的基础。
