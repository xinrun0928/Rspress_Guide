这一节，我们来学习 RESTful API 的设计规范和 Spring MVC 实现。

---

## 一、RESTful 核心原则

### 1.1 资源定位

```
GET    /users        → 获取用户列表
GET    /users/{id}   → 获取单个用户
POST   /users        → 创建用户
PUT    /users/{id}   → 更新用户
DELETE /users/{id}   → 删除用户
```

### 1.2 HTTP 方法语义

| 方法 | 语义 | 幂等 | 安全 |
|---|---|---|---|
| GET | 查询 | ✅ | ✅ |
| POST | 创建 | ❌ | ❌ |
| PUT | 完整更新 | ✅ | ❌ |
| PATCH | 部分更新 | ❌ | ❌ |
| DELETE | 删除 | ✅ | ❌ |

## 二、Spring MVC RESTful 实现

### 2.1 @RestController

```java
@RestController
@RequestMapping("/api/users")
public class UserApiController {
    // 自动返回 JSON
}
```

### 2.2 @PathVariable

```java
@GetMapping("/{id}")
public User getUser(@PathVariable Long id) {
    return userService.findById(id);
}
```

### 2.3 @RequestBody

```java
@PostMapping
public User createUser(@RequestBody @Valid User user) {
    return userService.save(user);
}
```

## 三、统一响应结构

```java
@Data
public class Result<T> {
    private int code;
    private String message;
    private T data;

    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setData(data);
        return result;
    }

    public static <T> Result<T> error(String message) {
        Result<T> result = new Result<>();
        result.setCode(500);
        result.setMessage(message);
        return result;
    }
}
```

## 四、分页查询

```java
@GetMapping
public Result<Page<User>> listUsers(
        @RequestParam(defaultValue = "1") int page,
        @RequestParam(defaultValue = "10") int size) {
    return Result.success(userService.findAll(page, size));
}
```

## 五、HTTP 状态码

| 状态码 | 含义 |
|---|---|
| 200 | 成功 |
| 201 | 创建成功 |
| 204 | 删除成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

## 六、面试追问方向

**「RESTful 的核心原则是什么？」**

资源定位（URL 表示资源）、HTTP 方法表示操作、状态码表示结果、无状态。

**「PUT 和 PATCH 的区别？」**

PUT 是完整更新，PATCH 是部分更新。
