# Spring 类型转换：PropertyEditor 与 Converter

import { Badge } from '@rspress/core/theme';

<Badge text="类型转换" type="tip" /> <Badge text="Converter" type="warning" /> <Badge text="PropertyEditor" type="info" />

`String` → `Integer`、`String` → `Date`、`String` → `Boolean`，Spring 是怎么自动转换的？

当你使用 `@Value("${app.port}")` 注入一个整数时，Spring 自动把字符串 "8080" 转成了 int。这个转换是谁做的？又是怎么做的？

这一节，我们深入解析 Spring 的类型转换系统。

---

## 类型转换的两种方式

### 1. PropertyEditor（传统方式）

JDK 标准的属性编辑器，基于 `PropertyEditor` 接口：

```java
public interface PropertyEditor {
    // 设置字符串值
    void setAsText(String text) throws java.lang.IllegalArgumentException;

    // 获取字符串值
    String getAsText();

    // 获取/设置值对象
    void setValue(Object value);
    Object getValue();

    // 其他方法...
}
```

### 2. Converter（Spring 3+）

Spring 3 引入的通用转换器，基于 `Converter` 接口：

```java
@FunctionalInterface
public interface Converter<S, T> {
    // 将 S 类型转换为 T 类型
    T convert(S source);
}
```

**Spring 推荐使用 Converter**，更通用、更类型安全。

---

## Converter 接口

### 基本用法

```java
// String -> Integer 转换器
public class StringToIntegerConverter implements Converter<String, Integer> {
    @Override
    public Integer convert(String source) {
        return Integer.parseInt(source);
    }
}
```

### 注册转换器

```java
@Configuration
public class ConversionConfig {

    @Bean
    public ConversionService conversionService() {
        DefaultConversionService conversionService = new DefaultConversionService();

        // 注册转换器
        conversionService.addConverter(new StringToIntegerConverter());

        return conversionService;
    }
}
```

### 使用转换服务

```java
@Service
public class ConfigService {

    @Autowired
    private ConversionService conversionService;

    public void useConversion() {
        // 使用转换服务
        Integer port = conversionService.convert("8080", Integer.class);

        // 也可以直接注入转换器
    }
}
```

## ConverterFactory

### 接口定义

```java
public interface ConverterFactory<S, R> {
    // 获取从 S 到 T 的转换器
    <T extends R> Converter<S, T> getConverter(Class<T> targetType);
}
```

### 使用示例

```java
// String -> Number 转换器工厂
public class StringToNumberConverterFactory implements ConverterFactory<String, Number> {

    @Override
    public <T extends Number> Converter<String, T> getConverter(Class<T> targetType) {
        if (targetType == Integer.class) {
            return new StringToIntegerConverter();
        } else if (targetType == Long.class) {
            return new StringToLongConverter();
        } else if (targetType == Double.class) {
            return new StringToDoubleConverter();
        }
        // ...
        throw new IllegalArgumentException("Unsupported type: " + targetType);
    }
}
```

## ConverterRegistry

### 接口定义

```java
public interface ConverterRegistry {
    // 添加转换器
    void addConverter(Converter<?, ?> converter);

    // 添加带泛型的转换器
    void addConverter(TypeDescriptor sourceType, TypeDescriptor targetType, Converter<?, ?> converter);

    // 添加转换器工厂
    void addConverterFactory(ConverterFactory<?, ?> factory);

    // 添加泛型转换器
    <S, T> void addConverter(Class<S> sourceType, Class<T> targetType,
                            Converter<? super S, ? extends T> converter);

    // 移除转换器
    void removeConvertible(Class<?> sourceType, Class<?> targetType);
}
```

### 使用示例

```java
@Configuration
public class ConversionConfig {

    @Bean
    public ConversionServiceFactoryBean conversionService() {
        ConversionServiceFactoryBean factory = new ConversionServiceFactoryBean();

        factory.setConverters(Set.of(
            new StringToIntegerConverter(),
            new StringToBooleanConverter(),
            new StringToDateConverter()
        ));

        return factory;
    }
}
```

## ConversionService

### 接口定义

```java
public interface ConversionService {
    // 判断是否可以转换
    boolean canConvert(Class<?> sourceType, Class<?> targetType);

    // 判断是否可以转换（带类型描述）
    boolean canConvert(TypeDescriptor sourceType, TypeDescriptor targetType);

    // 转换
    <T> T convert(Object source, Class<T> targetType);

    // 转换（带类型描述）
    Object convert(Object source, TypeDescriptor sourceType, TypeDescriptor targetType);
}
```

### Spring 内置的转换器

| 转换器 | 说明 |
|---|---|
| `StringToBooleanConverter` | "true"/"false" → boolean |

```java
@Configuration
public class FormattingConfig {

    @Bean
    public FormattingConversionService formattingConversionService() {
        FormattingConversionServiceFactoryBean factory =
            new FormattingConversionServiceFactoryBean();

        // 设置是否使用 JSR-354 Money & Currency
        factory.setRegisterDefaultFormatters(false);

        // 添加格式化注解
```

## 自定义 Converter

### String → Date 转换器

```java
public class StringToDateConverter implements Converter<String, Date> {
    private final DateTimeFormatter formatter;

    public StringToDateConverter(String pattern) {
        this.formatter = DateTimeFormatter.ofPattern(pattern);
    }

    @Override
    public Date convert(String source) {
        if (StringUtils.isBlank(source)) {
            return null;
        }

        LocalDate date = LocalDate.parse(source, formatter);
        return Date.from(date.atStartOfDay(ZoneId.systemDefault()).toInstant());
    }
}
```

### String → LocalDate 转换器

```java
public class StringToLocalDateConverter implements Converter<String, LocalDate> {

    private final DateTimeFormatter[] formatters = {
        DateTimeFormatter.ISO_DATE,
        DateTimeFormatter.ofPattern("yyyy-MM-dd"),
        DateTimeFormatter.ofPattern("yyyy/MM/dd")
    };

    @Override
    public LocalDate convert(String source) {
        if (StringUtils.isBlank(source)) {
            return null;
        }

        for (DateTimeFormatter formatter : formatters) {
            try {
                return LocalDate.parse(source, formatter);
            } catch (DateTimeParseException ignored) {
            }
        }

        throw new IllegalArgumentException("Cannot parse date: " + source);
    }
}
```

### 注册自定义转换器

```java
@Configuration
public class ConversionConfig {

    @Bean
    public ConversionServiceFactoryBean conversionService() {
        ConversionServiceFactoryBean factory = new ConversionServiceFactoryBean();

        Set<Converter<?, ?>> converters = new HashSet<>();
        converters.add(new StringToDateConverter("yyyy-MM-dd"));
        converters.add(new StringToLocalDateConverter());

        factory.setConverters(converters);

        return factory;
    }
}
```

## PropertyEditor 与 Converter 的区别

### 对比

| 特性 | PropertyEditor | Converter |
|---|---|---|
| 来源 | JDK | Spring 3+ |
| 泛型支持 | 不支持 | 支持 |
| 线程安全 | 否 | 是（通常） |
| 使用场景 | 表单绑定、BeanWrapper | 通用类型转换 |

### PropertyEditor 的局限

```java
// PropertyEditor 是非泛型的
public class CustomEditor extends PropertyEditorSupport {
    @Override
    public void setAsText(String text) throws IllegalArgumentException {
        // 只能处理 String -> Object
        setValue(parse(text));
    }

    @Override
    public String getAsText() {
        // Object -> String
        return format((MyType) getValue());
    }
}
```

### Converter 的优势

```java
// Converter 是泛型的
public class CustomConverter implements Converter<SourceType, TargetType> {
    @Override
    public TargetType convert(SourceType source) {
        // 编译时类型安全
    }
}
```

## 面试追问方向

**「PropertyEditor 和 Converter 有什么区别？」**

- `PropertyEditor`：JDK 标准，非泛型，用于表单绑定
- `Converter`：Spring 3+ 引入，泛型支持，用于通用类型转换

**「Spring 是如何自动注册转换器的？」**

Spring Boot 通过 `DefaultFormattingConversionService` 自动注册了常用转换器。

**「自定义转换器怎么注册？」**

通过 `ConversionServiceFactoryBean` 或 `FormattingConversionServiceFactoryBean` 注册。

---

## 总结

Spring 的类型转换机制：

| 组件 | 说明 |
|---|---|
| `Converter<S, T>` | 通用转换器接口 |
| `ConverterFactory<S, R>` | 转换器工厂 |
| `ConverterRegistry` | 转换器注册表 |
| `ConversionService` | 转换服务 |
| `PropertyEditor` | JDK 标准属性编辑器（已过时） |

推荐使用 `Converter`，更类型安全、更通用。
