# Spring 国际化：MessageSource 的实现

一个国际化（i18n）的场景：你开发了一个系统，需要同时支持中文、英文、日文。用户登录失败时，中文用户看到"用户名或密码错误"，英文用户看到"Invalid username or password"。

在传统写法中，你可能会写一堆 `if-else` 来判断语言。这种方式不仅代码臃肿，而且难以维护。Spring 提供了 **`MessageSource`** 来优雅地解决这个问题。

## 一、MessageSource 接口

### 接口定义

```java
public interface MessageSource {
    
    // 获取消息，没有参数时使用
    String getMessage(String code, Object[] args, String defaultMessage, 
                      Locale locale);
    
    // 获取消息，找不到时抛出异常
    String getMessage(String code, Object[] args, Locale locale)
        throws NoSuchMessageException;
    
    // 获取消息，支持占位符
    String getMessage(MessageSourceResolvable resolvable, Locale locale)
        throws NoSuchMessageException;
}
```

### 核心方法解析

- `code`：消息的 key，比如 `login.error`
- `args`：消息中的参数，用于填充 `{0}`、`{1}` 等占位符
- `defaultMessage`：找不到消息时使用的默认消息
- `locale`：地区标识，如 `Locale.CHINA`、`Locale.US`

## 二、基本配置

### Spring Boot 自动配置

Spring Boot 已经自动配置了 `MessageSource`：

```yaml
# application.yml
spring:
  messages:
    basename: messages            # 资源文件名，默认 messages
    encoding: UTF-8               # 文件编码，默认 UTF-8
    fallback-to-system-locale: true  # 找不到时是否使用系统默认地区
```

`basename` 指定消息资源文件的基本名称，Spring 会自动加载：

- `messages.properties`（默认语言）
- `messages_zh_CN.properties`（简体中文）
- `messages_zh_TW.properties`（繁体中文）
- `messages_en_US.properties`（美国英语）
- `messages_ja_JP.properties`（日语）

### 手动配置

如果需要更灵活的配置：

```java
@Bean
public MessageSource messageSource() {
    ResourceBundleMessageSource messageSource = 
        new ResourceBundleMessageSource();
    messageSource.setBasename("i18n/messages");
    messageSource.setDefaultEncoding("UTF-8");
    messageSource.setUseCodeAsDefaultMessage(false);
    return messageSource;
}
```

## 三、消息文件编写

### 基础消息

```
# messages.properties
app.name=My Application
login.title=Login
login.username=Username
login.password=Password
login.submit=Submit
login.error=Invalid username or password
```

### 中文消息

```
# messages_zh_CN.properties
app.name=我的应用
login.title=登录
login.username=用户名
login.password=密码
login.submit=提交
login.error=用户名或密码错误
```

### 支持占位符

```
# messages.properties
user.welcome={0}, welcome to {1}!
order.total=Total: ${0}
```

占位符可以用数字 `{0}`、`{1}` 或者 SpEL 语法 `#{0}`。

## 四、在代码中使用 MessageSource

### 注入 MessageSource

```java
@Service
public class LoginService {
    
    @Autowired
    private MessageSource messageSource;
    
    public void loginFailed(HttpServletRequest request) {
        Locale locale = request.getLocale();
        String message = messageSource.getMessage(
            "login.error", null, locale
        );
        
        // 返回国际化消息
        return message;
    }
}
```

### 带参数的消息

```java
public String getWelcomeMessage(String username, String appName) {
    return messageSource.getMessage(
        "user.welcome",
        new Object[]{username, appName},
        Locale.getDefault()
    );
    // 输出: "John, welcome to My Application!"
}
```

### 使用 MessageSourceResolvable

```java
public String getMessage(String code, Locale locale) {
    DefaultMessageSourceResolvable resolvable = 
        new DefaultMessageSourceResolvable(
            new String[]{code, "default.code"},  // 多个 code 依次尝试
            null,
            "Default Message"  // 最终默认消息
        );
    return messageSource.getMessage(resolvable, locale);
}
```

## 五、Thymeleaf 中使用国际化

### 配置 Thymeleaf

```java
@Bean
public MessageSource messageSource() {
    ResourceBundleMessageSource source = new ResourceBundleMessageSource();
    source.setBasename("messages");
    source.setDefaultEncoding("UTF-8");
    return source;
}

@Bean
public TemplateEngine templateEngine(MessageSource messageSource) {
    SpringTemplateEngine engine = new SpringTemplateEngine();
    engine.setTemplateResolver(templateResolver());
    
    // 注册国际化处理器
    engine.addDialect(new SpringSecurityDialect());
    return engine;
}
```

### Thymeleaf 模板中使用

```html
<h1 th:text="#{login.title}">登录</h1>

<input type="text" th:placeholder="#{login.username}">

<p th:text="#{user.welcome(${user.name}, ${app.name})}">
    Welcome message
</p>
```

## 六、SessionLocaleResolver 与 LocaleResolver

### LocaleResolver

Spring 提供了多种 `LocaleResolver` 实现：

| 实现类 | 说明 |
| --- | --- |
| `AcceptHeaderLocaleResolver` | 从 HTTP 请求头 Accept-Language 获取（默认） |
| `SessionLocaleResolver` | 从 Session 中获取 |
| `CookieLocaleResolver` | 从 Cookie 中获取 |
| `FixedLocaleResolver` | 固定使用某个 Locale |

### 配置 LocaleResolver

```java
@Bean
public LocaleResolver localeResolver() {
    SessionLocaleResolver resolver = new SessionLocaleResolver();
    resolver.setDefaultLocale(Locale.CHINA);
    return resolver;
}
```

### 切换语言

```java
@Controller
public class LanguageController {
    
    @Autowired
    private LocaleResolver localeResolver;
    
    @GetMapping("/change-language")
    public String changeLanguage(
            @RequestParam("lang") String lang,
            HttpServletRequest request,
            HttpServletResponse response) {
        
        Locale newLocale;
        if ("en".equals(lang)) {
            newLocale = Locale.US;
        } else if ("zh".equals(lang)) {
            newLocale = Locale.CHINA;
        } else {
            newLocale = Locale.getDefault();
        }
        
        localeResolver.setLocale(request, response, newLocale);
        
        return "redirect:/";
    }
}
```

## 七、ReloadingMessageSource

### 动态刷新消息

Spring 还提供了 `ReloadableResourceBundleMessageSource`，支持热更新消息文件：

```java
@Bean
public MessageSource messageSource() {
    ReloadableResourceBundleMessageSource source = 
        new ReloadableResourceBundleMessageSource();
    source.setBasename("classpath:i18n/messages");
    source.setDefaultEncoding("UTF-8");
    
    // 缓存时间：-1 表示永不过期，0 表示每次都检查，>0 为毫秒数
    source.setCacheSeconds(60);
    
    return source;
}
```

### 配合 @RefreshScope

在 Spring Cloud 场景下，配合 `@RefreshScope` 实现配置刷新：

```java
@Component
@RefreshScope
public class RefreshableMessageSource {
    
    @Autowired
    private MessageSource messageSource;
    
    public String getMessage(String code, Locale locale) {
        return messageSource.getMessage(code, null, locale);
    }
}
```

## 八、消息文件命名规范

### 标准命名

| 文件名 | 说明 |
| --- | --- |
| `messages.properties` | 默认语言（通常是英文） |
| `messages_zh_CN.properties` | 简体中文 |
| `messages_zh_TW.properties` | 繁体中文 |
| `messages_en_GB.properties` | 英式英语 |
| `messages_ja_JP.properties` | 日语 |

### Locale 的构成

`Locale` 由语言代码、国家代码和变体组成：

```
语言_国家_变体
  |     |      |
 zh   CN      Simplified
 en   US
 ja   JP
```

### 搜索顺序

当查找一个消息时，Spring 按以下顺序搜索：

```
messages_zh_CN_Simplified.properties
messages_zh_CN.properties
messages_zh.properties
messages.properties
```

如果都找不到，使用默认消息或抛出异常。

## 九、常见问题

### 问题一：消息文件不生效

检查：

1. 文件是否放在 `resources` 或 `src/main/resources` 下
2. `basename` 是否正确（不含语言后缀）
3. 文件编码是否为 UTF-8

### 问题二：中文乱码

确保：

1. `application.yml` 中设置 `spring.messages.encoding: UTF-8`
2. 手动配置时调用 `setDefaultEncoding("UTF-8")`
3. IDE 设置文件编码为 UTF-8

### 问题三：切换语言后消息不更新

确保 `LocaleResolver` 配置正确，并且 `Locale` 被正确保存在 Session 或 Cookie 中。

## 总结

Spring 国际化的核心要点：

1. **`MessageSource`** 是 Spring 国际化的核心接口
2. **消息文件命名规范**：`basename_语言_国家.properties`
3. **占位符**：`{0}`、`{1}` 用于参数替换
4. **`LocaleResolver`** 控制语言解析策略：Accept-Language / Session / Cookie
5. **`ReloadableResourceBundleMessageSource`** 支持消息热更新
6. **Spring Boot 自动配置**，只需创建消息文件即可

---

下次当你需要实现国际化功能时，不要再写 `if (locale == zh) return "中文";` 了——用 `MessageSource`，代码更优雅，维护更方便。
