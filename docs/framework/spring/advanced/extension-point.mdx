## 一、容器启动前

### BeanDefinitionRegistryPostProcessor

**接口定义**

```java
public interface BeanDefinitionRegistryPostProcessor extends BeanFactoryPostProcessor {
    void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) throws BeansException;
}
```

**调用时机**：所有 BeanDefinition 加载之前

**作用**：动态注册 BeanDefinition

**典型应用**：
- MyBatis 的 `@MapperScan`
- Spring Boot 的自动配置

**示例**

```java
@Component
public class CustomRegistrar implements BeanDefinitionRegistryPostProcessor {
    @Override
    public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) {
        // 动态注册 Bean
        BeanDefinitionBuilder builder = BeanDefinitionBuilder
            .genericBeanDefinition(MyService.class);
        registry.registerBeanDefinition("myService", builder.getBeanDefinition());
    }
}
```

## 二、Bean 定义加载阶段

### BeanFactoryPostProcessor

**接口定义**

```java
public interface BeanFactoryPostProcessor {
    void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException;
}
```

**调用时机**：所有 BeanDefinition 加载之后、Bean 实例化之前

**作用**：修改 BeanDefinition

**典型应用**：
- `PropertyPlaceholderConfigurer`：解析占位符
- `PropertyOverrideConfigurer`：属性覆盖
- `CustomEditorConfigurer`：注册属性编辑器

**示例**

```java
@Component
public class CustomProcessor implements BeanFactoryPostProcessor {
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        // 修改所有 BeanDefinition
        for (String beanName : beanFactory.getBeanDefinitionNames()) {
            BeanDefinition bd = beanFactory.getBeanDefinition(beanName);
            bd.setLazyInit(true);
        }
    }
}
```

## 三、Bean 实例化前

### InstantiationAwareBeanPostProcessor

**接口定义**

```java
public interface InstantiationAwareBeanPostProcessor extends BeanPostProcessor {
    // 实例化前
    Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName);

    // 实例化后、属性填充前
    boolean postProcessAfterInstantiation(Object bean, String beanName);

    // 属性填充
    PropertyValues postProcessProperties(PropertyValues pvs, Object bean, String beanName);
}
```

**调用时机**：Bean 实例化前后

**作用**：
- `postProcessBeforeInstantiation`：返回自定义 Bean 对象，替代默认实例化
- `postProcessAfterInstantiation`：控制是否填充属性
- `postProcessProperties`：处理属性值

**典型应用**：
- Spring AOP 的 `AbstractAutoProxyCreator`
- `@Required` 注解处理

**示例**

```java
@Component
public class CustomInstantiationProcessor implements InstantiationAwareBeanPostProcessor {
    @Override
    public Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName) {
        // 返回非 null 会跳过实例化
        return null;
    }

    @Override
    public boolean postProcessAfterInstantiation(Object bean, String beanName) {
        // 返回 false 会跳过属性填充
        return true;
    }

    @Override
    public PropertyValues postProcessProperties(PropertyValues pvs, Object bean, String beanName) {
        // 可以修改属性值
        return pvs;
    }
}
```

public interface MergedBeanDefinitionPostProcessor extends BeanPostProcessor {

## 五、Bean 初始化

### Aware 接口

| 接口 | 回调方法 | 注入内容 |
|---|---|---|
| `BeanNameAware` | `setBeanName(String name)` | Bean 名称 |
| `BeanFactoryAware` | `setBeanFactory(BeanFactory factory)` | BeanFactory |
| `ApplicationContextAware` | `setApplicationContext(ApplicationContext context)` | ApplicationContext |
| `EmbeddedValueResolverAware` | `setEmbeddedValueResolver(StringValueResolver resolver)` | 占位符解析器 |
| `BeanClassLoaderAware` | `setBeanClassLoader(ClassLoader classLoader)` | 类加载器 |
| `BootstrapContextAware` | `setBootstrapContext(BootstrapContext context)` | BootstrapContext |

**示例**

```java
@Component
public class CustomBean implements BeanNameAware, ApplicationContextAware {
    private String beanName;
    private ApplicationContext context;

    @Override
    public void setBeanName(String name) {
        this.beanName = name;
    }

    @Override
    public void setApplicationContext(ApplicationContext context) {
        this.context = context;
    }
}
```

### BeanPostProcessor（初始化阶段）

**接口定义**

```java
public interface BeanPostProcessor {
    // 初始化前
    default Object postProcessBeforeInitialization(Object bean, String beanName) {
        return bean;
    }

    // 初始化后
    default Object postProcessAfterInitialization(Object bean, String beanName) {
        return bean;
    }
}
```

**典型应用**：
- `@Autowired`、`@Value` 注入（`AutowiredAnnotationBeanPostProcessor`）
- `@Async` 异步处理（`AsyncAnnotationBeanPostProcessor`）
- `@Transactional` 事务处理（`InfrastructureAdvisorAutoProxyCreator`）

### InitializationBean

```java
public interface InitializingBean {
    void afterPropertiesSet() throws Exception;
}
```

**调用时机**：属性填充完成后

```java
@Service
public class UserService implements InitializingBean {
    @Override
    public void afterPropertiesSet() {
        // 初始化逻辑
    }
}
```

### @PostConstruct

**调用时机**：InitializingBean.afterPropertiesSet() 之后

```java
@Service
public class UserService {
    @PostConstruct
    public void init() {
        // 初始化逻辑
    }
}
```

**执行顺序**

```
@PostConstruct
    ↓
InitializingBean.afterPropertiesSet()
    ↓
BeanPostProcessor.postProcessAfterInitialization()
```

## 七、按调用顺序分类

### 完整调用顺序

```
1. BeanDefinitionRegistryPostProcessor.postProcessBeanDefinitionRegistry()
   ↓
2. BeanFactoryPostProcessor.postProcessBeanFactory()
   ↓
3. BeanPostProcessor.postProcessBeforeInstantiation() [InstantiationAware]
   ↓
4. Bean 构造方法执行
   ↓
5. MergedBeanDefinitionPostProcessor.postProcessMergedBeanDefinition()
   ↓
6. InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation()
   ↓
7. InstantiationAwareBeanPostProcessor.postProcessProperties()
   ↓
8. BeanNameAware.setBeanName()
   ↓
9. BeanFactoryAware.setBeanFactory()
   ↓
10. ApplicationContextAware.setApplicationContext()
   ↓
11. BeanPostProcessor.postProcessBeforeInitialization()
   ↓
12. @PostConstruct
   ↓
13. InitializingBean.afterPropertiesSet()
   ↓
14. @Bean(initMethod)
   ↓
15. BeanPostProcessor.postProcessAfterInitialization()
   ↓
16. SmartInitializingSingleton.afterSingletonsInstantiated()
   ↓
[Bean 就绪]
   ↓
17. @PreDestroy
   ↓
18. DisposableBean.destroy()
   ↓
19. @Bean(destroyMethod)
   ↓
20. DestructionAwareBeanPostProcessor.postProcessBeforeDestruction()
```

## 十、面试追问方向

**「BeanFactoryPostProcessor 和 BeanPostProcessor 的区别？」**

- `BeanFactoryPostProcessor`：操作 BeanDefinition，在 Bean 创建前
- `BeanPostProcessor`：操作 Bean 实例，在 Bean 创建后

**「Spring AOP 是在哪个扩展点织入的？」**

`AbstractAutoProxyCreator` 继承自 `SmartInstantiationAwareBeanPostProcessor`，在 `postProcessBeforeInstantiation` 中返回代理对象。

**「循环依赖是怎么处理的？」**

通过 `getEarlyBeanReference` 方法提前暴露 Bean 引用，配合三级缓存实现。

## 总结

Spring 的扩展点是框架设计的精华：

| 阶段 | 扩展点 |
|---|---|
| 容器启动前 | `BeanDefinitionRegistryPostProcessor` |
| 定义加载 | `BeanFactoryPostProcessor` |
| 实例化前 | `InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation` |
| 属性填充 | `InstantiationAwareBeanPostProcessor.postProcessProperties` |
| 初始化前 | `BeanPostProcessor.postProcessBeforeInitialization` |
| 初始化后 | `BeanPostProcessor.postProcessAfterInitialization` |
| 销毁前 | `DestructionAwareBeanPostProcessor.postProcessBeforeDestruction` |

理解这些扩展点，就理解了 Spring 的核心。
