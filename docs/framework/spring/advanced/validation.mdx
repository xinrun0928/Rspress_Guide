# Spring 验证：JSR-303 与 Validator

import { Badge } from '@rspress/core/theme';

<Badge text="验证" type="tip" /> <Badge text="JSR-303" type="warning" /> <Badge text="@Validated" type="info" />

当用户提交表单时，你怎么校验输入？

是写一堆 if-else 判断？还是让 Spring 帮你校验？

Spring 提供了完整的验证框架，基于 JSR-303（Bean Validation）标准，让校验变得声明式、标准化。

这一节，我们深入解析 Spring 的验证机制。

---

## JSR-303 简介

### 规范定义

JSR-303 是 Java EE 6 引入的 Bean Validation 规范，定义了一套注解用于声明式校验：

```java
@NotNull
private String name;

@Size(min = 6, max = 20)
private String password;

@Email
private String email;
```

### 实现

| 实现 | 说明 |
|---|---|
| Hibernate Validator | 常用实现 |
| Apache BVal | 另一个实现 |
| Spring Validator | Spring 自带的验证器 |

## 常用校验注解

### 空值校验

| 注解 | 说明 |
|---|---|
| `@NotNull` | 不能为 null |
| `@NotBlank` | 不能为 null、空字符串、纯空格 |
| `@NotEmpty` | 不能为 null、空集合、空字符串 |

### 长度校验

| 注解 | 说明 |
|---|---|
| `@Size(min, max)` | 字符串/集合长度范围 |
| `@Length(min, max)` | Hibernate Validator 扩展 |

### 数值校验

| 注解 | 说明 |
|---|---|
| `@Min(value)` | 最小值 |
| `@Max(value)` | 最大值 |
| `@DecimalMin(value)` | 最小值（支持小数） |
| `@DecimalMax(value)` | 最大值（支持小数） |
| `@Digits(integer, fraction)` | 整数和小数位数 |
| `@Positive` | 正数 |
| `@Negative` | 负数 |
| `@PositiveOrZero` | 非负数 |
| `@NegativeOrZero` | 非正数 |

### 格式校验

| 注解 | 说明 |
|---|---|
| `@Email` | 邮箱格式 |
| `@URL` | URL 格式 |
| `@Pattern(regexp)` | 正则表达式 |
| `@NotBlank` | 非空字符串 |

### 日期校验

| 注解 | 说明 |
|---|---|
| `@Past` | 过去的时间 |
| `@Future` | 未来的时间 |
| `@PastOrPresent` | 过去或现在 |
| `@FutureOrPresent` | 未来或现在 |

## 基本用法

### 定义校验规则

```java
public class User {
    @NotNull(message = "用户名不能为空")
    @Size(min = 3, max = 20, message = "用户名长度为 3-20 个字符")
    private String username;

    @NotBlank(message = "密码不能为空")
    @Size(min = 6, max = 20, message = "密码长度为 6-20 个字符")
    private String password;

    @Email(message = "邮箱格式不正确")
    private String email;

    @Min(value = 0, message = "年龄不能小于 0")
    @Max(value = 150, message = "年龄不能大于 150")
    private Integer age;

    @Pattern(regexp = "^1[3-9]\\d{9}$", message = "手机号格式不正确")
    private String phone;
}
```

### 在 Controller 中校验

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @PostMapping
    public Result<User> createUser(@Valid @RequestBody User user) {
        // @Valid 触发校验
        return Result.ok(userService.create(user));
    }
}
```

### 处理校验错误

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<?> handleValidation(MethodArgumentNotValidException e) {
        List<String> errors = e.getBindingResult()
            .getFieldErrors()
            .stream()
            .map(FieldError::getDefaultMessage)
            .collect(Collectors.toList());

        return Result.error(400, "校验失败: " + errors);
    }
}
```

## @Validated 注解

### @Validated vs @Valid

| 特性 | @Valid | @Validated |
|---|---|---|
| 来源 | javax.validation（JSR-303） | org.springframework.validation.annotation |
| 嵌套校验 | 支持 | 支持 |
| 分组校验 | 不支持 | 支持 |
| 位置 | 参数、方法、字段 | 类和方法参数 |

### 使用

```java
@RestController
@RequestMapping("/api/users")
@Validated
public class UserController {

    @PostMapping
    public Result<User> createUser(@Valid @RequestBody User user) {
        return Result.ok(userService.create(user));
    }

    @GetMapping("/{id}")
    public Result<User> getUser(@PathVariable @Min(1) Long id) {
        // @Validated 支持方法参数校验
        return Result.ok(userService.findById(id));
    }
}
```

## 分组校验

### 定义分组

```java
// 分组接口
public interface Create {
}

public interface Update {
}
```

### 在实体中使用分组

```java
public class User {
    @Null(groups = Update.class, message = "创建时不能指定 ID")
    @NotNull(groups = Update.class, message = "更新时必须指定 ID")
    private Long id;

    @NotBlank(groups = {Create.class, Update.class})
    @Size(min = 3, max = 20, groups = {Create.class, Update.class})
    private String username;

    @NotBlank(groups = Create.class)
    @Size(min = 6, max = 20, groups = {Create.class, Update.class})
    private String password;
}
```

### 在 Controller 中指定分组

```java
@RestController
public class UserController {

    @PostMapping
    public Result<User> create(@Validated(Create.class) @RequestBody User user) {
        return Result.ok(userService.create(user));
    }

    @PutMapping
    public Result<User> update(@Validated(Update.class) @RequestBody User user) {
        return Result.ok(userService.update(user));
    }
}
```

## 自定义校验

### 自定义校验注解

```java
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PhoneValidator.class)
@Documented
public @interface Phone {
    String message() default "手机号格式不正确";

    Class<?>[] groups() default {};

    Class<? extends Payload>[] payload() default {};
}
```

### 实现校验器

```java
public class PhoneValidator implements ConstraintValidator<Phone, String> {

    private static final Pattern PHONE_PATTERN =
        Pattern.compile("^1[3-9]\\d{9}$");

    @Override
    public void initialize(Phone constraintAnnotation) {
        // 初始化
    }

    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null || value.isEmpty()) {
            return true; // 使用 @NotNull 校验
        }
        return PHONE_PATTERN.matcher(value).matches();
    }
}
```

### 使用

```java
public class User {
    @Phone
    private String phone;
}
```

## BindingResult

### 获取校验结果

```java
@RestController
public class UserController {

    @PostMapping
    public Result<?> createUser(@Valid @RequestBody User user,
                               BindingResult bindingResult) {
        // 如果有校验错误
        if (bindingResult.hasErrors()) {
            Map<String, String> errors = new HashMap<>();
            bindingResult.getFieldErrors().forEach(error ->
                errors.put(error.getField(), error.getDefaultMessage())
            );
            return Result.error(400, errors);
        }

        return Result.ok(userService.create(user));
    }
}
```

### 常用方法

| 方法 | 说明 |
|---|---|
| `hasErrors()` | 是否有错误 |
| `hasFieldErrors()` | 是否有字段错误 |
| `hasGlobalErrors()` | 是否有全局错误 |
| `getFieldErrors()` | 获取所有字段错误 |
| `getGlobalErrors()` | 获取所有全局错误 |
| `getFieldError(field)` | 获取指定字段的错误 |
| `getErrorCount()` | 获取错误数量 |

## Spring Validator

### Spring 自带的验证器

除了 JSR-303，Spring 还提供了自己的验证器接口：

```java
public interface Validator {
    // 是否支持这个类
    boolean supports(Class<?> clazz);

    // 校验逻辑
    void validate(Object target, Errors errors);
}
```

### 实现

```java
@Component
public class UserValidator implements Validator {

    @Override
    public boolean supports(Class<?> clazz) {
        return User.class.isAssignableFrom(clazz);
    }

    @Override
    public void validate(Object target, Errors errors) {
        ValidationUtils.rejectIfEmpty(errors, "username", "username.empty");
        ValidationUtils.rejectIfEmpty(errors, "password", "password.empty");

        User user = (User) target;
        if (user.getPassword().length() < 6) {
            errors.rejectValue("password", "password.tooShort");
        }
    }
}
```

### 使用

```java
@Controller
public class UserController {

    @Autowired
    private UserValidator userValidator;

    @PostMapping
    public Result<?> createUser(@ModelAttribute User user, Errors errors) {
        userValidator.validate(user, errors);

        if (errors.hasErrors()) {
            // 处理错误
        }

        return Result.ok(userService.create(user));
    }
}
```

## 面试追问方向

**「@Valid 和 @Validated 的区别？」**

- `@Valid`：JSR-303 标准，支持嵌套校验
- `@Validated`：Spring 提供，支持分组校验

**「如何自定义校验注解？」**

定义注解 + 实现 `ConstraintValidator`。

**「级联校验怎么实现？」**

在嵌套对象上使用 `@Valid`。

## 总结

Spring 验证机制基于 JSR-303：

| 注解 | 说明 |
|---|---|
| `@NotNull` | 不能为 null |
| `@NotBlank` | 不能为空字符串 |
| `@NotEmpty` | 不能为空集合 |
| `@Size` | 长度范围 |
| `@Min/@Max` | 数值范围 |
| `@Email` | 邮箱格式 |
| `@Pattern` | 正则表达式 |
| `@Valid` | 嵌套校验 |
| `@Validated` | 分组校验 |
