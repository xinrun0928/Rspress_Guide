---

## 一、核心概念

`BeanPostProcessor` 是 Spring 框架的核心扩展点，允许在 Bean 初始化前后进行自定义处理。

```java title="BeanPostProcessor 接口"
public interface BeanPostProcessor {
    // 初始化前回调
    Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException;

    // 初始化后回调
    Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException;
}
```

---

## 二、常见实现类

| 实现类 | 作用 | 执行时机 |
| --- | --- | --- |
| AutowiredAnnotationBeanPostProcessor | 处理 @Autowired、@Value | 属性填充阶段 |
| CommonAnnotationBeanPostProcessor | 处理 @Resource、@PostConstruct | 属性填充阶段 |
| RequiredAnnotationBeanPostProcessor | 处理 @Required | 属性填充阶段 |
| AbstractAutoProxyCreator | 创建 AOP 代理 | 初始化后 |
| ApplicationContextAwareProcessor | 注入 Aware 接口 | 初始化前 |

---

## 三、自定义 BeanPostProcessor

```java
@Component
public class MyBeanPostProcessor implements BeanPostProcessor {

    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        System.out.println("初始化前: " + beanName);
        return bean;
    }

    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        System.out.println("初始化后: " + beanName);
        // 可以返回代理对象替换原对象
        return bean;
    }
}
```

## 四、执行时机

```mermaid

    participant Container as Spring 容器
    participant Create as 实例化
    participant Populate as 属性填充
    participant BPP_Before as BeanPostProcessor<br/>.postProcessBeforeInitialization
    participant Init as 初始化方法
    participant BPP_After as BeanPostProcessor<br/>.postProcessAfterInitialization
    participant Done as Bean 就绪

    Container->>Create: 实例化 Bean
    Create->>Populate: 属性填充
    Populate->>BPP_Before: 执行所有 BPP 前置处理
    BPP_Before->>Init: 调用初始化方法
    Init->>BPP_After: 执行所有 BPP 后置处理
    BPP_After->>Done: Bean 创建完成

    Note over BPP_Before: AOP 代理在这里创建！
```

## 五、高频面试题

### 🔴 第一层：BeanPostProcessor 有什么作用？

**答案要点**：
1. Spring 框架的核心扩展点
2. 在 Bean 初始化前后进行拦截
3. @Autowired、@Transactional、AOP 都依赖它实现

### 🟡 第二层：@Autowired 是通过 BeanPostProcessor 实现的吗？

**答案要点**：
1. 是的，通过 `AutowiredAnnotationBeanPostProcessor`
2. 它在属性填充阶段将依赖注入到 Bean 中

---

## 六、常见陷阱

> ⚠️ **陷阱一**：BeanPostProcessor 会对所有 Bean 执行

如果没有过滤条件，自定义的 BeanPostProcessor 会对容器中所有 Bean 执行，可能影响性能。

> ⚠️ **陷阱二**：postProcessAfterInitialization 中返回代理对象

如果返回的代理对象类型与原 Bean 类型不兼容，可能导致类型转换异常。
