# Spring Security 高级特性：框架深处的安全能力

认证、授权、JWT……这些只是 Spring Security 能力的冰山一角。

Spring Security 还有许多高级特性，用于防护各种安全威胁：CSRF 攻击、CORS 跨域、XSS 注入……

这一章，就是来探索这些「框架深处的安全能力」。

## 安全威胁与防护

### 常见的 Web 安全威胁

| 威胁 | 说明 | 防护 |
|---|---|---|
| CSRF | 跨站请求伪造 | CSRF Token |
| CORS | 跨域资源共享问题 | 正确的 CORS 配置 |
| XSS | 跨站脚本攻击 | 输入转义、Content-Security-Policy |
| 点击劫持 | 恶意页面嵌套 | X-Frame-Options |
| 会话固定 | 攻击者固定用户 Session | Session 换 ID |

## 面试核心问题

### Q1：Spring Security 的安全防护有哪些？

Spring Security 提供了多层次的安全防护：
- 认证：表单登录、OAuth2、JWT 等
- 授权：URL 权限、方法权限、数据权限
- CSRF 防护：Synchronizer Token Pattern
- CORS：跨域资源共享控制
- XSS：输入转义、Content-Security-Policy
- 点击劫持：X-Frame-Options
- 会话安全：Session 固定防护、并发控制

### Q2：CSRF 的原理是什么？

攻击者诱使已登录用户访问恶意页面，恶意页面自动发送请求到目标网站，由于浏览器会自动带上用户的 Cookie，服务器无法区分是用户主动发起的请求还是恶意页面伪造的请求。

### Q3：如何禁用 Spring Security 的某个安全功能？

```java
http.csrf(csrf -> csrf.disable())  // 禁用 CSRF
http.cors(cors -> cors.disable())  // 禁用 CORS
```

## 延伸思考

Spring Security 的高级特性，远不止这些。

当你深入了解 Spring Security 的源码，你会发现：

- 每个安全机制都有其存在的理由
- 每个配置选项都有其适用的场景
- 框架的设计处处体现安全最佳实践

理解这些高级特性，你不仅能更好地使用 Spring Security，还能写出更安全的代码。
