# 短信验证码登录：自定义 AuthenticationProvider

表单登录靠密码，短信登录靠验证码，Spring Security 怎么统一处理？

密码登录是「你知道什么」，短信登录是「你拥有什么」——两种不同的认证方式，但 Spring Security 的认证框架可以优雅地统一处理它们。

## 短信登录的整体流程

```
┌─────────────────────────────────────────────────────────────┐
│                  短信验证码登录流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【发送验证码阶段】                                           │
│  用户输入手机号                                              │
│       ↓                                                      │
│  后端生成 6 位验证码                                          │
│       ↓                                                      │
│  验证码存入 Redis（有效期 5 分钟）                            │
│       ↓                                                      │
│  调用短信网关发送验证码                                        │
│       ↓                                                      │
│  【登录阶段】                                                 │
│  用户提交手机号 + 验证码                                      │
│       ↓                                                      │
│  SmsCodeAuthenticationFilter 拦截                             │
│       ↓                                                      │
│  生成 SmsCodeAuthenticationToken（未认证）                      │
│       ↓                                                      │
│  SmsCodeAuthenticationProvider 认证                           │
│       ↓                                                      │
│  校验 Redis 中的验证码                                         │
│       ↓                                                      │
│  校验通过 → 加载用户 → 认证成功                               │
│       ↓                                                      │
│  设置 SecurityContext                                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 核心组件

要实现短信登录，需要三样东西：

1. **AuthenticationToken**：封装手机号和验证码
2. **AuthenticationProvider**：校验验证码和加载用户
3. **AuthenticationFilter**：拦截请求，生成 Token

## 自定义 SmsCodeAuthenticationToken

### 继承体系

```
Authentication
    ↑
AbstractAuthenticationToken
    ↑
UsernamePasswordAuthenticationToken
    ↑
SmsCodeAuthenticationToken（自定义）
```

### 实现

```java
public class SmsCodeAuthenticationToken extends AbstractAuthenticationToken {

    private final Object principal;  // 手机号
    private String smsCode;          // 验证码（认证前）

    // 认证前构造器：只有手机号
    public SmsCodeAuthenticationToken(String mobile) {
        super(null);
        this.principal = mobile;
        this.smsCode = null;
        setAuthenticated(false);
    }

    // 认证后构造器：有 UserDetails
    public SmsCodeAuthenticationToken(Object principal,
            Collection<? extends GrantedAuthority> authorities) {
        super(authorities);
        this.principal = principal;
        this.smsCode = null;
        setAuthenticated(true);
    }

    @Override
    public Object getCredentials() {
        return smsCode;  // 认证后返回 null
    }

    @Override
    public Object getPrincipal() {
        return principal;  // 始终返回手机号
    }

    public void setSmsCode(String smsCode) {
        this.smsCode = smsCode;
    }

    public String getSmsCode() {
        return this.smsCode;
    }
}
```

## 自定义 SmsCodeAuthenticationFilter

### 继承 `AbstractAuthenticationProcessingFilter`

```java
public class SmsCodeAuthenticationFilter extends
        AbstractAuthenticationProcessingFilter {

    public static final String SPRING_SECURITY_FORM_MOBILE_KEY = "mobile";
    public static final String SPRING_SECURITY_FORM_SMS_CODE_KEY = "smsCode";

    private String mobileParameter = SPRING_SECURITY_FORM_MOBILE_KEY;
    private String smsCodeParameter = SPRING_SECURITY_FORM_SMS_CODE_KEY;

    public SmsCodeAuthenticationFilter() {
        // 拦截短信登录接口
        super(new AntPathRequestMatcher("/login/sms", "POST"));
    }

    @Override
    public Authentication attemptAuthentication(
            HttpServletRequest request,
            HttpServletResponse response) throws AuthenticationException {

        String mobile = obtainMobile(request);
        String smsCode = obtainSmsCode(request);

        if (mobile == null || mobile.isEmpty()) {
            mobile = "";
        }
        if (smsCode == null || smsCode.isEmpty()) {
            smsCode = "";
        }

        mobile = mobile.trim();
        smsCode = smsCode.trim();

        SmsCodeAuthenticationToken authRequest =
            new SmsCodeAuthenticationToken(mobile);

        // 设置详细信息
        setDetails(request, authRequest);

        // 交给 AuthenticationManager 认证
        return this.getAuthenticationManager().authenticate(authRequest);
    }

    private String obtainMobile(HttpServletRequest request) {
        return request.getParameter(mobileParameter);
    }

    private String obtainSmsCode(HttpServletRequest request) {
        return request.getParameter(smsCodeParameter);
    }

    private void setDetails(HttpServletRequest request,
            SmsCodeAuthenticationToken authRequest) {
        authRequest.setDetails(
            authenticationDetailsSource.buildDetails(request));
    }
}
```

## 自定义 SmsCodeAuthenticationProvider

### 实现 `AuthenticationProvider`

```java
@Component
public class SmsCodeAuthenticationProvider implements AuthenticationProvider {

    private UserDetailsService userDetailsService;

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    private static final String SMS_CODE_KEY_PREFIX = "sms:code:";
    private static final int SMS_CODE_EXPIRE_SECONDS = 300;  // 5 分钟

    @Override
    public Authentication authenticate(Authentication authentication)
            throws AuthenticationException {

        SmsCodeAuthenticationToken token =
            (SmsCodeAuthenticationToken) authentication;

        String mobile = (String) token.getPrincipal();
        String smsCode = token.getSmsCode();

        // 1. 从 Redis 获取验证码
        String cachedCode = redisTemplate.opsForValue()
            .get(SMS_CODE_KEY_PREFIX + mobile);

        // 2. 校验验证码
        if (cachedCode == null) {
            throw new AuthenticationServiceException("验证码已过期");
        }

        if (!cachedCode.equals(smsCode)) {
            throw new BadCredentialsException("验证码错误");
        }

        // 3. 验证通过后删除验证码（一次性使用）
        redisTemplate.delete(SMS_CODE_KEY_PREFIX + mobile);

        // 4. 加载用户
        UserDetails userDetails = userDetailsService.loadUserByUsername(mobile);

        // 5. 返回已认证的 Token
        return new SmsCodeAuthenticationToken(
            userDetails,
            userDetails.getAuthorities()
        );
    }

    @Override
    public boolean supports(Class<?> authentication) {
        // 只处理 SmsCodeAuthenticationToken
        return SmsCodeAuthenticationToken.class.isAssignableFrom(authentication);
    }

    public void setUserDetailsService(UserDetailsService userDetailsService) {
        this.userDetailsService = userDetailsService;
    }
}
```

## 验证码发送服务

### 发送验证码接口

```java
@RestController
public class SmsController {

    @Autowired
    private SmsService smsService;

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    private static final String SMS_CODE_KEY_PREFIX = "sms:code:";
    private static final int SMS_CODE_EXPIRE_SECONDS = 300;

    @GetMapping("/sms/send")
    public Result<?> sendSms(@RequestParam String mobile) {
        // 1. 校验手机号格式
        if (!isValidMobile(mobile)) {
            return Result.error("手机号格式错误");
        }

        // 2. 检查发送频率（60 秒内不能重复发送）
        String rateKey = "sms:rate:" + mobile;
        Boolean hasRate = redisTemplate.hasKey(rateKey);
        if (Boolean.TRUE.equals(hasRate)) {
            return Result.error("发送太频繁，请稍后再试");
        }

        // 3. 生成 6 位验证码
        String smsCode = String.format("%06d", new Random().nextInt(1000000));

        // 4. 存入 Redis
        redisTemplate.opsForValue().set(
            SMS_CODE_KEY_PREFIX + mobile,
            smsCode,
            SMS_CODE_EXPIRE_SECONDS,
            TimeUnit.SECONDS
        );

        // 5. 设置发送频率限制
        redisTemplate.opsForValue().set(rateKey, "1", 60, TimeUnit.SECONDS);

        // 6. 调用短信网关（生产环境）
        // smsService.send(mobile, "您的验证码是：" + smsCode);

        // 测试环境直接返回验证码
        return Result.success(Map.of("code", smsCode, "message", "验证码已发送"));
    }

    private boolean isValidMobile(String mobile) {
        return mobile != null && mobile.matches("^1[3-9]\\d{9}$");
    }
}
```

## Security 配置

### 注入 Provider 并添加 Filter

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Autowired
    private SmsCodeAuthenticationProvider smsCodeAuthenticationProvider;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/sms/send", "/login/sms").permitAll()
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginProcessingUrl("/login")
                .permitAll()
            )
            .csrf(csrf -> csrf
                .ignoringRequestMatchers("/sms/send", "/login/sms")
            )
            // 添加短信登录过滤器
            .addFilterBefore(smsCodeAuthenticationFilter(),
                UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public SmsCodeAuthenticationFilter smsCodeAuthenticationFilter() {
        SmsCodeAuthenticationFilter filter = new SmsCodeAuthenticationFilter();
        // 设置 AuthenticationManager
        filter.setAuthenticationManager(
            authenticationManager(authenticationConfiguration));
        return filter;
    }

    @Bean
    public AuthenticationManager authenticationManager(
            AuthenticationConfiguration authenticationConfiguration)
            throws Exception {
        return authenticationConfiguration.getAuthenticationManager();
    }
}
```

### 在 ProviderManager 中注册 Provider

需要确保 `SmsCodeAuthenticationProvider` 能被 `AuthenticationManager` 使用：

```java
@Bean
public AuthenticationManager authenticationManager(
        AuthenticationConfiguration config) throws Exception {
    AuthenticationManager manager = config.getAuthenticationManager();
    // 将 SmsCodeAuthenticationProvider 注册到 ProviderManager
    ProviderManager providerManager = (ProviderManager) manager;
    providerManager.getProviders().add(smsCodeAuthenticationProvider);
    return manager;
}
```

或者直接在配置中构建：

```java
@Bean
public AuthenticationManager authenticationManager() {
    ProviderManager manager = new ProviderManager(
        Arrays.asList(
            daoAuthenticationProvider(),
            smsCodeAuthenticationProvider()
        )
    );
    return manager;
}
```

## 多认证方式共存

### 表单登录 + 短信登录

```
请求 /login（POST，用户名+密码）
    ↓
UsernamePasswordAuthenticationFilter 拦截
    ↓
交给 DaoAuthenticationProvider 认证
    ↓
成功/失败

请求 /login/sms（POST，手机号+验证码）
    ↓
SmsCodeAuthenticationFilter 拦截
    ↓
交给 SmsCodeAuthenticationProvider 认证
    ↓
成功/失败
```

### Provider 的匹配逻辑

`ProviderManager` 会遍历所有 `AuthenticationProvider`，找到 `supports()` 返回 `true` 的那个来处理认证：

```java
for (AuthenticationProvider provider : providers) {
    if (provider.supports(authentication.getClass())) {
        return provider.authenticate(authentication);
    }
}
```

所以 `SmsCodeAuthenticationProvider.supports()` 必须返回 `true` 当且仅当 `SmsCodeAuthenticationToken` 被传入。

## 常见问题与解决方案

### 问题一：验证码总是验证失败

排查步骤：

1. 确认 Redis 连接正常
2. 确认 Redis 中的 key 格式正确（`sms:code:13800138000`）
3. 确认验证码没有过期（5 分钟）
4. 确认验证码没有一次性使用后被删除

### 问题二：短信登录和密码登录冲突

如果用户同时支持密码登录和短信登录，`UserDetailsService` 需要同时支持用户名和手机号查询：

```java
@Service
public class CustomUserDetailsService implements UserDetailsService {

    @Autowired
    private UserMapper userMapper;

    @Override
    public UserDetails loadUserByUsername(String usernameOrMobile)
            throws UsernameNotFoundException {
        // 尝试按用户名查询
        UserEntity user = userMapper.findByUsername(usernameOrMobile);
        if (user == null) {
            // 尝试按手机号查询
            user = userMapper.findByMobile(usernameOrMobile);
        }
        if (user == null) {
            throw new UsernameNotFoundException("用户不存在");
        }
        // 构建权限...
    }
}
```

### 问题三：过滤器顺序问题

`SmsCodeAuthenticationFilter` 必须放在 `UsernamePasswordAuthenticationFilter` **之前**：

```java
// 错误
.addFilterAfter(smsCodeAuthenticationFilter(),
    UsernamePasswordAuthenticationFilter.class);

// 正确
.addFilterBefore(smsCodeAuthenticationFilter(),
    UsernamePasswordAuthenticationFilter.class);
```

原因：如果 `UsernamePasswordAuthenticationFilter` 先处理，它不认识手机号参数，会尝试按用户名密码处理。

## 面试追问预判

### Q1：自定义 AuthenticationProvider 需要注意什么？

1. `supports()` 方法必须精确匹配要处理的 `AuthenticationToken` 类型
2. 认证成功后必须返回一个新的 `Authentication` 对象，`authenticated = true`
3. 认证失败必须抛出 `AuthenticationException` 子类
4. 不要在 Provider 中做太多事情，职责单一

### Q2：短信登录的安全考虑有哪些？

1. **验证码生成**：使用 `SecureRandom`，不要用可预测的算法
2. **验证码有效期**：不要太长，5 分钟足够
3. **发送频率限制**：防止短信轰炸
4. **验证码只能使用一次**：防止重放攻击
5. **日志脱敏**：验证码不能写入日志
6. **IP 限制**：同一 IP 每天发送次数限制

### Q3：Spring Security 还有哪些内置的认证方式？

- `UsernamePasswordAuthenticationToken`：表单登录
- `RememberMeAuthenticationToken`：Remember-Me 自动登录
- `OpenIDAuthenticationToken`：OpenID 认证
- `JaasAuthenticationToken`：JAAS 认证
- `PreAuthenticatedAuthenticationToken`：预认证（如 Nginx 认证）

## 总结

短信登录的实现，展示了 Spring Security 的高度可扩展性：

1. **AuthenticationToken**：封装认证凭证（手机号 + 验证码）
2. **AuthenticationProvider**：校验凭证 + 加载用户
3. **AuthenticationFilter**：拦截请求，生成 Token
4. **配置注入**：将 Provider 注册到 AuthenticationManager

理解了这些，你就能实现任何自定义的认证方式——邮箱登录、企业微信登录、指纹登录……
