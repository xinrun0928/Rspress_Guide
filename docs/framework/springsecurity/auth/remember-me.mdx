# Remember-Me：两周免登录是怎么实现的

用户登录时勾选「记住我」，两周内再次访问时无需重新登录——这是怎么做到的？

表单登录靠 Session，Session 依赖 Cookie 传递 Session ID。但 Session 有超时时间——默认 30 分钟不活动就失效。用户关闭浏览器，隔天再访问，Session 早就没了。

Remember-Me 就是来解决这个问题的。

---

## Remember-Me 的工作原理

### 核心思想

用户登录成功后，服务器生成一个 Token，写入浏览器的 Cookie。下次访问时，浏览器自动带上这个 Cookie，服务器验证 Token 的有效性，如果有效就自动恢复登录状态。

```
┌─────────────────────────────────────────────────────────────┐
│                 Remember-Me 完整流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【登录时】                                                  │
│  用户登录，勾选「记住我」                                      │
│       ↓                                                      │
│  Spring Security 生成 Token                                   │
│       ↓                                                      │
│  Token 写入浏览器 Cookie                                       │
│       ↓                                                      │
│  【后续访问】                                                 │
│  用户访问网站（不携带登录凭证）                                │
│       ↓                                                      │
│  RememberMeFilter 拦截请求                                    │
│       ↓                                                      │
│  从 Cookie 读取 Token                                         │
│       ↓                                                      │
│  验证 Token 签名（服务端有密钥）                               │
│       ↓                                                      │
│  Token 有效 → 自动登录 → 设置 SecurityContext                  │
│       ↓                                                      │
│  【登出时】                                                   │
│  用户点击「退出登录」                                         │
│       ↓                                                      │
│  删除 Cookie 中的 Token                                        │
│       ↓                                                      │
│  清除服务端 Token 记录（如果用数据库存储）                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 和 Session 的区别

| 维度 | Session | Remember-Me |
|---|---|---|
| 存储位置 | 服务端 Session | Cookie（Token 本身） |
| 有效期 | 通常 30 分钟超时 | 通常 2 周 |
| 服务端资源 | 需要存储 Session | 可选（内存或数据库） |
| 适用场景 | 短期登录 | 长期登录 |
| 依赖 | 需要 Cookie 传递 Session ID | 需要 Cookie 存储 Token |

## Token 的生成与存储

### 基于签名的 Token

Spring Security 默认使用 `TokenRepository` 存储和管理 Token。

```java
public interface TokenRepository {
    void createNewToken(PersistentRememberMeToken token);
    void updateToken(String series, String tokenValue, Date lastUsed);
    PersistentRememberMeToken getTokenForSeries(String seriesId);
    void removeUserTokens(String username);
}
```

### PersistentRememberMeToken

```java
public class PersistentRememberMeToken implements Serializable {
    private final String username;
    private final String series;     // 系列号，用于标识这个设备
    private final String tokenValue; // Token 值
    private final Date date;        // 最后使用时间
    // ...
}
```

**为什么需要 series（系列号）？**

series 用于标识「哪个设备」。同一个用户可以在多个设备上勾选「记住我」，每个设备对应一个 series。登出时，可以选择删除单个设备或所有设备。

### Token 的生成过程

```java
public class DefaultTokenRepositoryImpl implements TokenRepository {
    // 生成 Token
    public void createNewToken(PersistentRememberMeToken token) {
        // 生成安全的随机 Token
        String tokenValue = generateToken();
        // 保存到内存或数据库
        saveToken(token);
    }

    private String generateToken() {
        byte[] randomBytes = new byte[16];
        new SecureRandom().nextBytes(randomBytes);
        return Base64.getEncoder().encodeToString(randomBytes);
    }
}
```

### 签名验证

```
用户提交 Token
    ↓
从 Token 中解析出 username 和 series
    ↓
从数据库/内存中查询对应的 Token
    ↓
比对 Token 值是否一致
    ↓
检查 Token 是否过期
    ↓
验证通过 → 加载用户信息 → 自动登录
```

## Remember-Me 配置详解

### 基本配置

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Autowired
    private DataSource dataSource;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginProcessingUrl("/login")
                .permitAll()
            )
            // Remember-Me 配置
            .rememberMe(rem -> rem
                .tokenRepository(tokenRepository())  // Token 存储
                .tokenValiditySeconds(1209600)      // 有效期：14 天（秒）
                .rememberMeParameter("remember-me") // 表单参数名
                .rememberMeCookieName("remember-me")// Cookie 名称
            );

        return http.build();
    }

    @Bean
    public PersistentTokenRepository tokenRepository() {
        // 方式一：内存存储（重启后失效）
        return new InMemoryTokenRepositoryImpl();

        // 方式二：数据库存储（推荐）
        return new JdbcTokenRepositoryImpl(dataSource);
    }
}
```

### 数据库存储

如果使用 `JdbcTokenRepositoryImpl`，需要创建表：

```sql
CREATE TABLE persistent_logins (
    username VARCHAR(64) NOT NULL,
    series VARCHAR(64) PRIMARY KEY,
    token VARCHAR(64) NOT NULL,
    last_used TIMESTAMP NOT NULL
);
```

### Token 的有效期

```java
.tokenValiditySeconds(1209600)  // 14 天
.tokenValiditySeconds(86400)    // 1 天
```

这个时间从「用户勾选 Remember-Me 时」开始计算，还是从「最后一次使用时」开始计算？

答案是：**从「用户勾选 Remember-Me 时」开始计算**，即使你每天都使用 Token，14 天后 Token 仍然会过期。

> 如果你想实现「滑动窗口」（每次使用都刷新有效期），需要自定义 `TokenRepository` 和 `RememberMeServices`。

## Remember-Me 的安全机制

### 为什么是安全的？

1. **签名防伪造**：Token 由服务端密钥签名，无法伪造
2. **Token 随机性**：使用 `SecureRandom` 生成，无法猜测
3. **可撤销**：服务端可以删除 Token，登出时清除
4. **Cookie 安全属性**：`httpOnly`、`secure`、`sameSite`

### Cookie 安全配置

```java
.rememberMe(rem -> rem
    .tokenRepository(tokenRepository())
    .tokenValiditySeconds(1209600)
)
.cookie()  // Cookie 额外配置
    .name("remember-me")
    .httpOnly(true)      // 禁止 JavaScript 读取
    .secure(true)        // 仅 HTTPS
    .sameSite("Strict")  // 严格 SameSite
;
```

### 和 CSRF 的关系

Remember-Me 通常用于表单登录，表单登录需要 CSRF Token。所以 Remember-Me 的自动登录流程：

```
恶意页面诱导用户访问（携带 Cookie）
    ↓
RememberMeFilter 识别用户
    ↓
自动登录成功
    ↓
但恶意页面的请求没有 CSRF Token → 被 CSRF Filter 拦截 → 失败
```

**结论**：Remember-Me 本身是安全的，但需要配合 CSRF 防护使用。

## 自定义 Remember-Me 实现

### 实现 TokenRepository

```java
@Component
public class RedisTokenRepository implements TokenRepository {
    private final StringRedisTemplate redisTemplate;
    private static final String KEY_PREFIX = "remember-me:";

    @Override
    public void createNewToken(PersistentRememberMeToken token) {
        String key = KEY_PREFIX + token.getSeries();
        redisTemplate.opsForHash().put(key, "token", token.getTokenValue());
        redisTemplate.opsForHash().put(key, "username", token.getUsername());
        redisTemplate.opsForHash().put(key, "lastUsed",
            String.valueOf(token.getDate().getTime()));
        // 设置过期时间
        redisTemplate.expire(key, Duration.ofSeconds(1209600));
    }

    @Override
    public void updateToken(String series, String tokenValue, Date lastUsed) {
        String key = KEY_PREFIX + series;
        redisTemplate.opsForHash().put(key, "token", tokenValue);
        redisTemplate.opsForHash().put(key, "lastUsed",
            String.valueOf(lastUsed.getTime()));
        // 刷新过期时间
        redisTemplate.expire(key, Duration.ofSeconds(1209600));
    }

    @Override
    public PersistentRememberMeToken getTokenForSeries(String seriesId) {
        String key = KEY_PREFIX + seriesId;
        Map<Object, Object> entries = redisTemplate.opsForHash().entries(key);
        if (entries.isEmpty()) {
            return null;
        }
        return new PersistentRememberMeToken(
            (String) entries.get("username"),
            seriesId,
            (String) entries.get("token"),
            new Date(Long.parseLong((String) entries.get("lastUsed")))
        );
    }

    @Override
    public void removeUserTokens(String username) {
        // 遍历删除（生产环境用 SCAN）
        Set<String> keys = redisTemplate.keys(KEY_PREFIX + "*");
        for (String key : keys) {
            if (username.equals(redisTemplate.opsForHash().get(key, "username"))) {
                redisTemplate.delete(key);
            }
        }
    }
}
```

## 常见问题与解决方案

### 问题一：Remember-Me 不生效

排查步骤：

1. 确认表单中有 `remember-me` 参数
2. 确认 `rememberMe()` 配置已添加
3. 确认 `TokenRepository` 已配置
4. 检查浏览器是否阻止了 Cookie

### 问题二：Token 泄漏导致的安全问题

如果 Token 被盗用，攻击者可以在 Token 有效期内冒充用户。

**缓解措施**：

1. 限制 Token 有效期（不要设置太长）
2. 使用 `secure` 和 `httpOnly` Cookie
3. 登录时记录 IP 和 User-Agent，异常时报警
4. 实现 Token 刷新机制

### 问题三：多设备登录的 Token 管理

```java
// 踢掉其他设备的登录
@Autowired
private PersistentTokenRepository tokenRepository;

public void kickOutOtherDevices(String username) {
    tokenRepository.removeUserTokens(username);
}

// 只踢掉指定设备
public void kickOutDevice(String series) {

}
```

## 面试追问预判

### Q1：Remember-Me 和 JWT Token 的区别？

| 维度 | Remember-Me | JWT Token |
|---|---|---|
| 存储 | Token 存在服务端（内存/数据库） | Token 存在客户端 |
| 状态 | 有状态 | 无状态 |
| 撤销 | 可实时撤销 | 只能等过期或维护黑名单 |
| 适用场景 | 传统 Web 应用 | RESTful API、微服务 |

### Q2：Remember-Me 怎么实现「每次使用刷新有效期」？

默认不支持，需要自定义 `TokenRepository`：

1. 覆写 `updateToken()` 方法
2. 每次自动登录成功时，调用 `updateToken()`
3. 同时刷新 Cookie 的过期时间

### Q3：用户改密码后 Remember-Me Token 需要清除吗？

**是的，需要。** 密码泄露后，Remember-Me Token 也应该失效：

```java
@PostConstruct
public void init() {
    eventPublisher.publishEvent(new PasswordChangedEvent(currentUser));
}

// 然后在监听器中清除 Token
@Component
public class RememberMeTokenClearListener {
    @Autowired
    private PersistentTokenRepository tokenRepository;

    @EventListener
    public void onPasswordChanged(PasswordChangedEvent event) {
        tokenRepository.removeUserTokens(event.getUsername());
    }
}
```

## 总结

Remember-Me 是 Session 的补充机制：

1. **原理**：服务器生成签名 Token → 写入 Cookie → 下次访问时验证
2. **存储**：`TokenRepository` 支持内存和数据库两种方式
3. **安全**：签名防伪造 + Cookie 安全属性 + CSRF 配合
4. **自定义**：可以基于 Redis 等实现分布式 Token 存储
