# OAuth2 登录：QQ、微信、GitHub 第三方登录

点一下「微信登录」，Spring Security 怎么知道你是哪个微信用户？

OAuth2 登录的原理比表单登录复杂得多——涉及重定向、授权码、Token 交换、用户信息获取等一系列步骤。

但 Spring Security 把这些复杂性封装好了，我们只需要配置几个参数就能用。

---

## OAuth2 登录的核心概念

### 什么是 OAuth2？

OAuth 2.0（Open Authorization）是一个授权协议，允许用户授权第三方应用访问自己在某个平台上的资源，而不需要提供用户名密码。

### OAuth2 授权码模式

OAuth2 有多种授权模式，第三方登录使用的是「授权码模式」（Authorization Code Grant）：

```
┌─────────────────────────────────────────────────────────────┐
│              OAuth2 授权码模式完整流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【第一步：用户发起登录】                                       │
│  用户点击「微信登录」                                          │
│       ↓                                                      │
│  跳转微信授权页面（携带 client_id、redirect_uri、state）         │
│       ↓                                                      │
│  【第二步：用户授权】                                          │
│  用户在微信页面确认授权                                        │
│       ↓                                                      │
│  微信跳转回 redirect_uri，携带 authorization_code 和 state     │
│       ↓                                                      │
│  【第三步：获取 Token】                                        │
│  后端用 authorization_code 换取 access_token（不经过浏览器）    │
│       ↓                                                      │
│  【第四步：获取用户信息】                                       │
│  用 access_token 调用微信 API 获取用户信息                       │
│       ↓                                                      │
│  【第五步：登录/注册】                                        │
│  根据用户信息决定是登录已有账号还是注册新账号                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 为什么用授权码模式而不是隐式模式？

隐式模式（Implicit Grant）直接在 URL 中返回 Token，不需要 code 换取 Token 的步骤，看起来更简单。但隐式模式有安全风险——Token 直接暴露在浏览器中。

授权码模式的优势在于 Token 交换发生在后端，不会暴露给浏览器。

## Spring Security OAuth2 登录的配置

### 引入依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>
```

`spring-boot-starter-oauth2-client` 已经包含了 Spring Security 和 OAuth2 Client 相关的依赖。

### 配置 OAuth2 客户端

Spring Boot 2.x 提供了简化的配置方式：

```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          github:
            client-id: your-github-client-id
            client-secret: your-github-client-secret
            scope: read:user, user:email
          wechat:
            client-id: your-wechat-client-id
            client-secret: your-wechat-client-secret
            authorization-grant-type: authorization_code
            redirect-uri: "{baseUrl}/login/oauth2/code/{registrationId}"
            scope: snsapi_login
        provider:
          github:
            authorization-uri: https://github.com/login/oauth/authorize
            token-uri: https://github.com/login/oauth/access_token
            user-info-uri: https://api.github.com/user
            user-name-attribute: login
```

或者用 Java 配置：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public ClientRegistrationRepository clientRegistrationRepository() {
        return new InMemoryClientRegistrationRepository(
            githubClientRegistration(),
            wechatClientRegistration()
        );
    }

    private ClientRegistration githubClientRegistration() {
        return ClientRegistration.withRegistrationId("github")
            .clientId("your-github-client-id")
            .clientSecret("your-github-client-secret")
            .authorizationGrantType(AuthorizationGrantType.AUTHORIZATION_CODE)
            .redirectUri("{baseUrl}/login/oauth2/code/{registrationId}")
            .scope("read:user", "user:email")
            .authorizationUri("https://github.com/login/oauth/authorize")
            .tokenUri("https://github.com/login/oauth/access_token")
            .userInfoUri("https://api.github.com/user")
            .userNameAttributeName("login")
            .clientName("GitHub")
            .build();
    }
}
```

### Security 配置

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/login/**", "/error").permitAll()
                .anyRequest().authenticated()
            )
            .oauth2Login(oauth2 -> oauth2
                .loginPage("/login")
                .authorizationEndpoint()
                    .baseUri("/oauth2/authorization")
            );

        return http.build();
    }
}
```

这样配置后，访问 `/oauth2/authorization/github` 就会跳转到 GitHub 授权页面。

## OAuth2 登录过滤器链

### OAuth2LoginAuthenticationFilter

```
请求 /oauth2/authorization/{provider}
    ↓
OAuth2LoginAuthenticationFilter 拦截
    ↓
重定向到 GitHub 授权页面
    ↓
```

## 用户信息获取与处理

### OAuth2UserService

Spring Security 使用 `OAuth2UserService` 来获取用户信息：

```java
public interface OAuth2UserService<R extends OAuth2UserRequest, U extends OAuthUser> {
    OAuth2User loadUser(OAuth2UserRequest userRequest)
        throws OAuth2AuthenticationException;
}
```

默认实现是 `DefaultOAuth2UserService`，它会：

1. 用 access_token 调用用户信息接口
2. 将返回的 JSON 反序列化为 `OAuth2User`

### 自定义用户信息处理

你可以覆写 `loadUser()` 方法来自定义处理逻辑：

```java
@Service
public class CustomOAuth2UserService implements OAuth2UserService
        <OAuth2UserRequest, OAuth2User> {

    @Autowired
    private OAuth2UserService<OAuth2UserRequest, OAuth2User> delegate;

    @Autowired
    private UserService userService;

    @Override
    public OAuth2User loadUser(OAuth2UserRequest userRequest)
            throws OAuth2AuthenticationException {

        OAuth2User oauth2User = delegate.loadUser(userRequest);

        // 获取第三方平台的用户信息
        Map<String, Object> attributes = oauth2User.getAttributes();
        String provider = userRequest.getClientRegistration().getRegistrationId();
        String providerId = (String) attributes.get("id");  // 或其他标识

        // 业务逻辑：根据第三方信息查找或创建用户
        User user = userService.processOAuthLogin(provider, providerId, attributes);

        // 返回自定义的 UserDetails
        return new CustomOAuth2User(user, attributes, oauth2User.getName());
    }
}
```

### 用户注册/登录逻辑

```java
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    @Autowired
    private RoleRepository roleRepository;

    public User processOAuthLogin(String provider, String providerId,
            Map<String, Object> attributes) {

        // 查找是否已存在该第三方账号绑定的用户
        Optional<User> existingUser = userRepository
            .findByProviderAndProviderId(provider, providerId);

        if (existingUser.isPresent()) {
            return existingUser.get();
        }

        // 查找是否已存在相同邮箱的用户
        String email = (String) attributes.get("email");
        Optional<User> userByEmail = userRepository.findByEmail(email);

        if (userByEmail.isPresent()) {
            // 绑定第三方账号
            User user = userByEmail.get();
            user.setProvider(provider);
            user.setProviderId(providerId);
            return userRepository.save(user);
        }

        // 创建新用户
        User newUser = new User();
        newUser.setProvider(provider);
        newUser.setProviderId(providerId);
        newUser.setEmail(email);
        newUser.setNickname((String) attributes.get("name"));
        newUser.setAvatar((String) attributes.get("avatar_url"));
        newUser.setRole(roleRepository.findByName("USER"));

        return userRepository.save(newUser);
    }
}
```

## 多平台登录的用户合并

### 场景

同一个用户，可能既用邮箱注册过，又用 GitHub 登录过，还用微信登录过——这三个「身份」应该是同一个用户。

### 合并策略

1. **以邮箱为基准**：如果邮箱相同，认为是同一个用户
2. **用户主动绑定**：用户可以在个人中心手动绑定多个第三方账号
3. **强制合并**：如果系统检测到多个身份可能属于同一人，提示用户合并

### 实现

```java
public class OAuth2UserMergeService {

    @Autowired
    private UserRepository userRepository;

    public void mergeIfNeeded(String email, String provider, String providerId) {
        // 如果该第三方账号已绑定，跳过
        if (userRepository.findByProviderAndProviderId(provider, providerId).isPresent()) {
            return;
        }

        // 查找相同邮箱的已有账号
        Optional<User> existingUser = userRepository.findByEmail(email);

        if (existingUser.isPresent()) {
            // 绑定第三方账号
            User user = existingUser.get();
            user.setProvider(provider);
            user.setProviderId(providerId);
            userRepository.save(user);
        }
    }
}
```

## 登录成功后的处理

### OAuth2LoginAuthenticationSuccessHandler

```java
public class CustomOAuth2LoginSuccessHandler
        extends SimpleUrlAuthenticationSuccessHandler {

    @Autowired
    private JwtTokenProvider tokenProvider;

    @Autowired
    private UserService userService;

    @Override
    public void onAuthenticationSuccess(HttpServletRequest request,
            HttpServletResponse response, Authentication authentication)
            throws IOException {

        OAuth2User oauth2User = (OAuth2User) authentication.getPrincipal();

        // 生成 JWT Token（如果需要）
        String token = tokenProvider.generateToken(oauth2User);

        // 返回 JSON（如果是前后端分离）
        response.setContentType("application/json");
        response.setCharacterEncoding("UTF-8");
        response.getWriter().write(
            "{\"message\": \"登录成功\", \"token\": \"" + token + "\"}"
        );
    }
}
```

### 配置使用自定义 Handler

```java
.oauth2Login(oauth2 -> oauth2
    .userInfoEndpoint()
        .userService(customOAuth2UserService())
    .and()
    .successHandler(oAuth2LoginSuccessHandler())
)
```

## 常见问题与解决方案

### 问题一：回调地址不匹配

微信等平台要求回调地址必须是在微信公众平台后台配置的地址。确保 `redirect-uri` 与配置一致。

### 问题二：获取用户信息失败

某些平台（特别是微信）的用户信息接口可能需要额外权限或特殊处理。

### 问题三：Token 过期

OAuth2 的 access_token 有有效期。如果过期，需要使用 refresh_token 刷新。

## 面试追问预判

### Q1：OAuth2 和 SSO 的区别？

OAuth2 是授权协议，SSO（Single Sign-On）是单点登录。OAuth2 可以用于实现 SSO，但 SSO 不一定用 OAuth2。

### Q2：OAuth2 登录的实现流程？

用户点击第三方登录后，前端或后端跳转到授权页面，用户确认授权后获取授权码，用授权码在服务端换取访问令牌，最后用令牌获取用户信息。

### Q3：OAuth2 的安全性如何保证？

授权码模式确保令牌不暴露给浏览器，state 参数防止 CSRF 攻击，HTTPS 加密传输，token 设置合理的有效期。

## 总结

Spring Security OAuth2 登录的关键在于：

1. **ClientRegistration**：配置授权信息（client_id、client_secret、scope、endpoint）
2. **OAuth2UserService**：处理用户信息获取（可自定义）
3. **用户合并**：处理同一用户多平台登录的合并逻辑
4. **SuccessHandler**：自定义登录成功后的处理

理解了这些，你就能实现 GitHub、微信、QQ 等任何 OAuth2 平台的登录。
