# RBAC 权限模型：用户-角色-权限的三层设计

权限系统的设计，RBAC 是工业标准。

几乎所有的企业级权限系统，都或多或少地借鉴了 RBAC 的设计思想。理解 RBAC，就理解了权限系统的半壁江山。

## RBAC 的核心思想

### 什么是 RBAC？

RBAC（Role-Based Access Control，基于角色的访问控制）是一种以角色为中心的权限管理模型。

```
┌─────────────────────────────────────────────────────────────┐
│                    RBAC 核心思想                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  用户 ────── 拥有 ────── 角色 ────── 拥有 ────── 权限         │
│                                                              │
│  用户张三 → 拥有角色：管理员 → 拥有权限：用户增删改查          │
│  用户李四 → 拥有角色：普通员工 → 拥有权限：用户查看             │
│                                                              │
│  好处：张三和李四不直接关联权限，而是通过角色间接关联          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 为什么不直接给用户分配权限？

假设有 1000 个用户，50 种权限。如果直接分配：

- 用户-权限关系：1000 × 50 = 50000 条记录
- 每次修改权限，需要更新大量用户记录
- 权限变更难以追踪

使用 RBAC 后：

- 用户-角色关系：1000 × 10 = 10000 条记录
- 角色-权限关系：10 × 50 = 500 条记录
- 修改权限只需要修改角色

## RBAC 的四个级别

### RBAC0：最基础模型

用户-角色-权限三层，是 RBAC 的基础。

```
用户 ────── 用户角色关联 ────── 角色 ────── 角色权限关联 ────── 权限
```

### RBAC1：角色继承

角色可以有继承关系，子角色自动拥有父角色的所有权限。

```
          角色：超级管理员
                 ↑
         ┌───────┴───────┐
         ↑               ↑
      角色：管理员    角色：运维
         ↑               ↑
         └───────┬───────┘
                 ↑
              角色：普通员工
```

配置：

```java
RoleHierarchyImpl hierarchy = new RoleHierarchyImpl();
hierarchy.setHierarchy(
    "ROLE_SUPER_ADMIN > ROLE_ADMIN > ROLE_USER"
);
```

### RBAC2：约束模型

在 RBAC0 基础上增加约束条件：

| 约束 | 说明 | 示例 |
|---|---|---|
| 角色互斥 | 用户不能同时拥有互斥角色 | 一个人不能同时是「会计」和「出纳」 |
| 基数约束 | 限制角色拥有的权限数量 | 「部门经理」最多 3 个 |
| 先决条件 | 获得某角色前必须先拥有另一个角色 | 晋升为「经理」前必须先是「员工」 |

### RBAC3：完整模型

RBAC1 + RBAC2，是最完整的 RBAC 模型。

## 数据库表设计

### 基础五表

```sql
-- 用户表
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    email VARCHAR(100),
    status TINYINT DEFAULT 1 COMMENT '1:启用 0:禁用',
    create_time DATETIME,
    update_time DATETIME
);

-- 角色表
CREATE TABLE sys_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_name VARCHAR(50) NOT NULL UNIQUE COMMENT '角色标识',
    role_desc VARCHAR(200) COMMENT '角色描述',
    status TINYINT DEFAULT 1,
    create_time DATETIME
);

-- 权限表
CREATE TABLE sys_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    permission_name VARCHAR(100) NOT NULL COMMENT '权限标识',
    permission_desc VARCHAR(200) COMMENT '权限描述',
    type VARCHAR(20) COMMENT 'menu:菜单 button:按钮 api:接口',
    parent_id BIGINT DEFAULT 0 COMMENT '父权限ID',
    url VARCHAR(200) COMMENT '菜单URL',
    perms VARCHAR(100) COMMENT '权限标识符',
    create_time DATETIME
);

-- 用户角色关联表
CREATE TABLE sys_user_role (
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    PRIMARY KEY (user_id, role_id)
);

-- 角色权限关联表
CREATE TABLE sys_role_permission (
    role_id BIGINT NOT NULL,
    permission_id BIGINT NOT NULL,
    PRIMARY KEY (role_id, permission_id)
);
```

### 角色层级表（RBAC1）

```sql
-- 角色层级关系表
CREATE TABLE sys_role_hierarchy (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    parent_role_id BIGINT NOT NULL COMMENT '父角色ID',
    child_role_id BIGINT NOT NULL COMMENT '子角色ID',
    FOREIGN KEY (parent_role_id) REFERENCES sys_role(id),
    FOREIGN KEY (child_role_id) REFERENCES sys_role(id)
);
```

### 角色互斥表（RBAC2）

```sql
-- 角色互斥表
CREATE TABLE sys_role_mutex (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_id_1 BIGINT NOT NULL,
    role_id_2 BIGINT NOT NULL,
    FOREIGN KEY (role_id_1) REFERENCES sys_role(id),
    FOREIGN KEY (role_id_2) REFERENCES sys_role(id)
);
```

## Spring Security 中的 RBAC 实现

### UserDetailsService 实现

```java
@Service
public class CustomUserDetailsService implements UserDetailsService {

    @Autowired
    private UserMapper userMapper;

    @Autowired
    private RoleMapper roleMapper;

    @Autowired
    private PermissionMapper permissionMapper;

    @Override
    public UserDetails loadUserByUsername(String username)
            throws UsernameNotFoundException {

        // 1. 查询用户
        User user = userMapper.findByUsername(username);
        if (user == null) {
            throw new UsernameNotFoundException("用户不存在: " + username);
        }

        // 2. 查询用户角色
        List<Role> roles = roleMapper.findByUserId(user.getId());

        // 3. 查询角色权限
        Set<String> permissions = new HashSet<>();
        for (Role role : roles) {
            List<Permission> perms = permissionMapper.findByRoleId(role.getId());
            perms.forEach(p -> permissions.add(p.getPerms()));
        }

        // 4. 构建权限列表
        List<GrantedAuthority> authorities = new ArrayList<>();

        // 添加角色（带 ROLE_ 前缀）
        roles.forEach(role ->
            authorities.add(new SimpleGrantedAuthority("ROLE_" + role.getRoleName()))
        );

        // 添加权限
        permissions.forEach(perm ->
            authorities.add(new SimpleGrantedAuthority(perm))
        );

        // 5. 返回 UserDetails
        return new CustomUserDetails(
            user.getId(),
            user.getUsername(),
            user.getPassword(),
            user.getEmail(),
            user.getStatus() == 1,
            authorities
        );
    }
}
```

### 角色层级配置

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public RoleHierarchyImpl roleHierarchy() {
        RoleHierarchyImpl hierarchy = new RoleHierarchyImpl();
        // 角色层级配置
        hierarchy.setHierarchy(
            "ROLE_SUPER_ADMIN > ROLE_ADMIN " +
            "and ROLE_ADMIN > ROLE_MANAGER " +
            "and ROLE_MANAGER > ROLE_USER"
        );
        return hierarchy;
    }

    @Bean
    public AccessDecisionManager accessDecisionManager() {
        List<AccessDecisionVoter<?>> voters = Arrays.asList(
            new WebExpressionVoter(),
            new RoleHierarchyVoter(roleHierarchy())  // 使用角色层级
        );
        return new AffirmativeBased(voters);
    }
}
```

### 动态权限配置

```java
@Configuration
@EnableMethodSecurity
public class MethodSecurityConfig {

    @Bean
    public MethodSecurityExpressionHandler methodSecurityExpressionHandler() {
        DefaultMethodSecurityExpressionHandler handler =
            new DefaultMethodSecurityExpressionHandler();
        handler.setPermissionEvaluator(new CustomPermissionEvaluator());
        return handler;
    }
}

@Component
public class CustomPermissionEvaluator implements PermissionEvaluator {

    @Autowired
    private PermissionMapper permissionMapper;

    @Override
    public boolean hasPermission(Authentication authentication,
                                 Object targetDomainObject,
                                 Object permission) {
        if (authentication == null || targetDomainObject == null) {
            return false;
        }

        String perm = (String) permission;
        Collection<? extends GrantedAuthority> authorities =
            authentication.getAuthorities();

        // 检查是否有指定权限
        return authorities.stream()
            .anyMatch(a -> a.getAuthority().equals(perm));
    }

    @Override
    public boolean hasPermission(Authentication authentication,
                                 Serializable targetId,
                                 String targetType,
                                 Object permission) {
        if (authentication == null || targetType == null || permission == null) {
            return false;
        }

        String perm = (String) permission;
        Collection<? extends GrantedAuthority> authorities =
            authentication.getAuthorities();

        return authorities.stream()
            .anyMatch(a -> a.getAuthority().equals(perm));
    }
}
```

## 权限变更的原子性

### 批量修改用户角色

```java
@Service
public class RoleService {

    @Autowired
    private UserRoleMapper userRoleMapper;

    @Transactional
    public void assignRoles(Long userId, List<Long> roleIds) {
        // 1. 删除用户现有角色
        userRoleMapper.deleteByUserId(userId);

        // 2. 添加新角色
        for (Long roleId : roleIds) {
            UserRole ur = new UserRole();
            ur.setUserId(userId);
            ur.setRoleId(roleId);
            userRoleMapper.insert(ur);
        }
    }
}
```

### 检查角色互斥

```java
@Service
public class RoleMutexService {

    @Autowired
    private UserRoleMapper userRoleMapper;

    @Autowired
    private RoleMutexMapper roleMutexMapper;

    public boolean canAssignRoles(Long userId, List<Long> newRoleIds) {
        // 1. 查询用户现有角色
        List<Long> existingRoleIds = userRoleMapper.findRoleIdsByUserId(userId);

        // 2. 查询新角色之间的互斥关系
        for (Long newRoleId : newRoleIds) {
            List<Long> mutexWithNewRole =
                roleMutexMapper.findMutexRoles(newRoleId);

            // 3. 检查是否与现有角色互斥
            for (Long existingRoleId : existingRoleIds) {
                if (mutexWithNewRole.contains(existingRoleId)) {
                    return false;  // 互斥
                }
            }
        }

        return true;
    }
}
```

## 常见问题与解决方案

### 问题一：用户太多，权限加载慢

**解决**：引入缓存

```java
@Cacheable(value = "userPermissions", key = "#username")
@Override
public UserDetails loadUserByUsername(String username) {
    // 加载权限
}
```

### 问题二：权限实时性要求高

如果权限修改后需要立即生效：

```java
@CacheEvict(value = "userPermissions", key = "#username")
public void updateUserRoles(String username, List<Long> roleIds) {
    // 修改权限
}
```

### 问题三：超级管理员权限太大

**解决**：在代码层面限制超级管理员的操作

```java
@PreAuthorize("hasRole('SUPER_ADMIN')")
public void superAdminOnly() {
    // 只有超级管理员能执行
}
```

## 面试追问预判

### Q1：RBAC 和 ACL 的区别？

ACL（Access Control List，访问控制列表）是直接给用户分配权限。RBAC 是通过角色间接分配权限。ACL 更灵活但管理成本高，RBAC 更易管理但灵活性稍差。

### Q2：如何实现数据权限？

数据权限指「用户只能看到自己的数据」。这通常在 SQL 层面实现：

```java
@Select("SELECT * FROM orders WHERE user_id = #{userId}")
List<Order> findMyOrders(Long userId);
```

或者使用 MyBatis-Plus 的数据权限插件。

### Q3：超级管理员如何绕过权限检查？

超级管理员的权限通常在系统层面实现，而不是依赖业务代码：

```java
@Aspect
@Component
public class SuperAdminAspect {

    @Around("@annotation(adminOnly)")
    public Object checkAdmin(ProceedingJoinPoint point, AdminOnly adminOnly) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth.getAuthorities().stream()
                .anyMatch(a -> a.getAuthority().equals("ROLE_SUPER_ADMIN"))) {
            return point.proceed();  // 放行
        }
        // 走正常权限检查
    }
}
```

## 总结

RBAC 是企业级权限系统的基础模型：

1. **核心三层**：用户 → 角色 → 权限
2. **四个级别**：RBAC0（基础）、RBAC1（继承）、RBAC2（约束）、RBAC3（完整）
3. **数据库设计**：五张基础表 + 层级表 + 互斥表
4. **Spring Security 实现**：通过 `UserDetailsService` 和 `RoleHierarchyVoter`
