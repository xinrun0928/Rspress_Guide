# @EnableMethodSecurity：方法级安全的总开关

`@PreAuthorize` 为什么有时候不生效？

这个问题的答案，往往是忘记了一个注解——`@EnableMethodSecurity`。

## @EnableMethodSecurity 的作用

`@EnableMethodSecurity` 是 Spring Security 方法级安全的大门。只有添加了这个注解，下面的注解才会生效：

- `@PreAuthorize`
- `@PostAuthorize`
- `@Secured`
- `@RolesAllowed`

### 基本用法

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity  // 开启方法级安全
public class SecurityConfig {
    // ...
}
```

---

## 配置选项详解

### prePostEnabled

启用 `@PreAuthorize` 和 `@PostAuthorize`：

```java
@EnableMethodSecurity(prePostEnabled = true)  // 默认 true
```

```java
@PreAuthorize("hasRole('ADMIN')")
public void deleteUser(Long id) { }

@PostAuthorize("returnObject.owner == authentication.name")
public User getUser(Long id) { }
```

### securedEnabled

启用 `@Secured`：

```java
@EnableMethodSecurity(securedEnabled = true)
```

```java
@Secured({"ROLE_ADMIN", "ROLE_USER"})
public void doSomething() { }
```

### jsr250Enabled

启用 JSR-250 标准的 `@RolesAllowed`、`@DenyAll`、`@PermitAll`：

```java
@EnableMethodSecurity(jsr250Enabled = true)
```

```java
@RolesAllowed({"ROLE_ADMIN"})
public void adminOnly() { }

@PermitAll
public void publicAccess() { }

@DenyAll
public void nobodyAccess() { }
```

### proxyTargetClass

控制代理方式：

```java
@EnableMethodSecurity(proxyTargetClass = true)  // 使用 CGLIB 代理
@EnableMethodSecurity(proxyTargetClass = false) // 使用 JDK 动态代理
```

通常不需要显式设置，Spring Security 会自动选择。

## SpEL 评估过程

### 表达式解析

```
@PreAuthorize("hasRole('ADMIN')")
    ↓
解析为 SpEL 表达式
    ↓
创建 EvaluationContext
    ↓
注入变量（authentication, principal, method, args, etc）
    ↓
评估表达式
    ↓
返回 true/false
```

### 可用的变量

| 变量 | 说明 |
|---|---|
| `authentication` | 当前认证对象 |
| `principal` | 当前用户主体 |
| `hasRole()` | 检查角色 |
| `hasAuthority()` | 检查权限 |
| `hasAnyRole()` | 检查任一角色 |
| `hasAnyAuthority()` | 检查任一权限 |
| `permitAll()` | 始终返回 true |
| `denyAll()` | 始终返回 false |
| `isAnonymous()` | 是否匿名用户 |
| `isAuthenticated()` | 是否已认证 |
| `isRememberMe()` | 是否 Remember-Me |
| `hasPermission()` | 检查对象权限 |

### 自定义变量注入

```java
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public MethodSecurityExpressionHandler methodSecurityExpressionHandler() {
        DefaultMethodSecurityExpressionHandler handler =
            new DefaultMethodSecurityExpressionHandler();
        handler.setPermissionEvaluator(new CustomPermissionEvaluator());
        return handler;
    }
}
```

## @PostAuthorize：返回后检查

### 使用场景

有时候需要在方法执行**后**才能判断是否有权访问：

```java
// 方法执行后，检查返回值
@PostAuthorize("returnObject != null && " +
    "returnObject.owner == authentication.name")
public Order getOrder(Long orderId) {
    // 返回订单，但只能返回属于当前用户的订单
}
```

### returnObject 变量

`returnObject` 是 `@PostAuthorize` 特有的变量，代表方法的返回值：

```java
@PostAuthorize("returnObject.status != 'DELETED'")
public Document updateDocument(Long id, Document doc) {
    return documentRepository.findById(id)
        .map(d -> documentRepository.save(d))
        .orElse(null);
}
```

### 访问返回对象的属性

```java
@PostAuthorize("returnObject.id == #userId")
public User getUser(Long userId) { }

@PostAuthorize("returnObject.username == authentication.name")
public UserProfile getProfile() { }
```

## 方法参数注入

### @P 注解

使用 `@P` 注解直接引用方法参数：

```java
@PreAuthorize("@userService.canAccess(#p0, #p1)")
public void doSomething(Long userId, String resource) { }
```

等价于：

```java
@PreAuthorize("@userService.canAccess(#userId, #resource)")
public void doSomething(@P("userId") Long userId,
                        @P("resource") String resource) { }
```

### @AuthenticationPrincipal

注入当前认证的用户：

```java
@GetMapping("/profile")
public User getProfile(
        @AuthenticationPrincipal UserDetails user) {
    return user;
}
```

### 注入元数据

```java
@GetMapping("/example")
public String example(
        @AuthenticationPrincipal UserDetails user,
        MethodSecurityExpressionRoot root) {
    return "example";
}
```

## 性能影响

### 拦截开销

每次方法调用都要经过 AOP 拦截和 SpEL 评估，有一定性能开销：

```
方法调用
    ↓
AOP 拦截
    ↓
解析 @PreAuthorize 注解
    ↓
构建 SpEL 上下文
    ↓
评估表达式
    ↓
返回结果
```

### 优化建议

1. **表达式缓存**：Spring Security 会缓存已解析的表达式
2. **简化表达式**：避免复杂的嵌套调用
3. **使用 @Secured 替代**：如果只需要简单角色检查

## 与 @PreFilter / @PostFilter 配合

### @PreFilter

方法执行前过滤集合参数：

```java
@PreFilter("filterObject.owner == authentication.name")
public void processOrders(List<Order> orders) {
    // 只处理当前用户的订单
}
```

### @PostFilter

方法执行后过滤返回集合：

```java
@PostFilter("filterObject.owner == authentication.name")
public List<Order> getOrders() {
    return orderRepository.findAll();
    // 返回的列表只包含当前用户的订单
}
```

:::warning
`@PostFilter` 在返回集合较大时性能较差，因为它会遍历整个集合进行过滤。如果数据量大，建议在数据库查询时直接用 WHERE 条件过滤。
:::

## 常见问题与解决方案

### 问题一：@PreAuthorize 不生效

```java
// 错误：在接口上加注解
public interface UserService {
    @PreAuthorize("hasRole('ADMIN')")
    void deleteUser(Long id);
}

// 正确：在实现类上加注解
@Service
public class UserServiceImpl implements UserService {
    @Override
    @PreAuthorize("hasRole('ADMIN')")
    public void deleteUser(Long id) { }
}
```

Spring AOP 默认代理接口上的注解不生效。

### 问题二：内部方法调用不拦截

```java
@Service
public class UserService {
    public void outer() {
        inner();  // 不走代理，不拦截
    }

    @PreAuthorize("hasRole('ADMIN')")
    public void inner() { }  // 不会被拦截
}
```

**解决**：注入自身：

```java
@Autowired
private UserService self;

public void outer() {
    self.inner();  // 走代理，被拦截
}
```

### 问题三：权限不足但异常被吞

```java
try {
    userService.deleteUser(id);
} catch (AccessDeniedException e) {
    // 异常被吞掉了
}
```

**解决**：不要 catch `AccessDeniedException`，让它传播到全局异常处理器。

## 完整配置示例

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity(
    prePostEnabled = true,
    securedEnabled = true,
    jsr250Enabled = true
)
public class SecurityConfig {

    @Bean
    public MethodSecurityExpressionHandler methodSecurityExpressionHandler() {
        DefaultMethodSecurityExpressionHandler handler =
            new DefaultMethodSecurityExpressionHandler();
        handler.setPermissionEvaluator(customPermissionEvaluator());
        handler.setRoleHierarchy(roleHierarchy());
        return handler;
    }

    @Bean
    public RoleHierarchyImpl roleHierarchy() {
        RoleHierarchyImpl hierarchy = new RoleHierarchyImpl();
        hierarchy.setHierarchy("ROLE_ADMIN > ROLE_USER");
        return hierarchy;
    }

    @Bean
    public CustomPermissionEvaluator customPermissionEvaluator() {
        return new CustomPermissionEvaluator();
    }
}
```

## 面试追问预判

### Q1：@PreAuthorize 和 @PostAuthorize 哪个先执行？

`@PreAuthorize` 在方法执行**前**执行，如果权限不足，直接拒绝，不执行方法。`@PostAuthorize` 在方法执行**后**执行，可以根据返回值判断是否有权访问。

### Q2：Spring Security 的 AOP 和事务的 AOP 哪个先执行？

Spring Security 的方法安全 AOP 在事务 AOP 之前执行。如果方法权限不足，事务不会开始。

### Q3：如何自定义权限判断逻辑？

1. 实现 `PermissionEvaluator` 接口
2. 注册到 `MethodSecurityExpressionHandler`
3. 在 SpEL 表达式中使用 `hasPermission()`

## 总结

`@EnableMethodSecurity` 是方法级安全的总开关：

1. **启用注解**：`@PreAuthorize`、`@PostAuthorize`、`@Secured`、`@RolesAllowed`
2. **配置选项**：`prePostEnabled`、`securedEnabled`、`jsr250Enabled`
3. **SpEL 表达式**：支持条件组合、参数引用、方法调用
4. **性能考虑**：复杂表达式有开销，大集合过滤建议用数据库 WHERE
