# AccessDecisionManager：投票器模式的权限决策

三个投票器，两个投通过，一个投拒绝，最终放行还是拒绝？

这是 Spring Security 投票器模式的核心问题。`AccessDecisionManager` 负责收集投票结果，根据策略决定最终的访问权限。

## 投票器模式的设计

### 为什么用投票器？

Spring Security 的权限决策不是简单的「是/否」，而是采用投票器模式。原因：

1. **灵活性**：不同的投票器可以从不同维度判断
2. **可扩展**：可以添加自定义投票器
3. **解耦**：每个投票器职责单一

## AccessDecisionVoter 接口

### 投票结果

```java
public interface AccessDecisionVoter<S> {
    // 投票结果
    int ACCESS_GRANTED = 1;   // 赞成
    int ACCESS_DENIED = -1;   // 反对
    int ACCESS_ABSTAIN = 0;   // 弃权
}
```

### Voter 的判断逻辑

```java
public interface AccessDecisionVoter<S> {

    // 投票方法
    int vote(Authentication authentication,
             S object,
             Collection<ConfigAttribute> attributes);

    // 是否支持这个 ConfigAttribute
    boolean supports(ConfigAttribute attribute);

    // 是否支持这个类型的对象
    boolean supports(Class<?> clazz);
}
```

## 内置投票器

### WebExpressionVoter

URL 级别权限投票器，支持 SpEL 表达式：

```java
public class WebExpressionVoter implements AccessDecisionVoter<FilterInvocation> {

    private SecurityExpressionHandler<FilterInvocation> expressionHandler =
        new DefaultWebSecurityExpressionHandler();

    @Override
    public int vote(Authentication authentication,
                    FilterInvocation object,
                    Collection<ConfigAttribute> attributes) {

        WebSecurityExpressionRoot expressionRoot =
            (WebSecurityExpressionRoot) expressionHandler
                .createSecurityExpressionRoot(authentication, object);

        // 解析配置中的 SpEL 表达式
        ConfigAttribute attribute = attributes.iterator().next();
        String expr = attribute.getAttribute();
        return ExpressionUtils.evaluateAsBoolean(expr, expressionRoot)
            ? ACCESS_GRANTED
            : ACCESS_DENIED;
    }
}
```

### RoleHierarchyVoter

角色层级投票器，支持角色继承：

```java
public class RoleHierarchyVoter implements AccessDecisionVoter<Object> {

    private RoleHierarchy roleHierarchy;

    @Override
    public int vote(Authentication authentication,
                    Object object,
                    Collection<ConfigAttribute> attributes) {
        Collection<? extends GrantedAuthority> authorities =
            authentication.getAuthorities();

        for (ConfigAttribute attribute : attributes) {
            if (!this.supports(attribute)) {
                continue;
            }

            // 获取配置需要的角色
            String requiredRole =
                attribute.getAttribute().substring("ROLE_".length());

            // 检查是否有该角色或其上级角色
            for (GrantedAuthority authority : authorities) {
                if (roleHierarchy.getReachableGrantedAuthorities(
                        Collections.singletonList(authority))
                        .stream()
                        .anyMatch(a -> a.getAuthority()
                            .equals("ROLE_" + requiredRole))) {
                    return ACCESS_GRANTED;
                }
            }
        }

        return ACCESS_DENIED;
    }
}
```

## AccessDecisionManager 三种策略

### AffirmativeBased（默认）

**任一投票器赞成即通过**：

```java
// 伪代码
public void decide(Authentication authentication,
                   Object object,
                   Collection<ConfigAttribute> attributes) {
    int grant = 0;
    for (AccessDecisionVoter voter : voters) {
        int result = voter.vote(authentication, object, attributes);
        if (result == ACCESS_GRANTED) {
            return;  // 直接通过
        }
        if (result == ACCESS_DENIED) {
            grant--;
        }
    }
    if (grant < 0) {
        throw new AccessDeniedException("拒绝访问");
    }
}
```

### ConsensusBased

**多数票决定**：

```java
public void decide(...) {
    int grant = 0;
    int deny = 0;

    for (AccessDecisionVoter voter : voters) {
        int result = voter.vote(authentication, object, attributes);
        if (result == ACCESS_GRANTED) {
            grant++;
        } else if (result == ACCESS_DENIED) {
            deny++;
        }
    }

    if (grant > deny) {
        return;  // 通过
    } else if (deny > grant) {
        throw new AccessDeniedException("拒绝访问");
    }

    // 平票时，检查 allowIfEqualGrantedDeniedDecisions
    if (allowIfEqualGrantedDeniedDecisions) {
        return;
    }
    throw new AccessDeniedException("拒绝访问");
}
```

### UnanimousBased

**全票赞成才通过**：

```java
public void decide(...) {
    int grant = 0;

    for (AccessDecisionVoter voter : voters) {
        int result = voter.vote(authentication, object, attributes);
        if (result == ACCESS_DENIED) {
            throw new AccessDeniedException("拒绝访问");
        }
        if (result == ACCESS_GRANTED) {
            grant++;
        }
    }

    // 如果有投票器投了赞成票，且没有反对票，通过
    // 如果所有投票器都弃权，也通过
}
```

## 决策过程源码分析

### FilterSecurityInterceptor 中的决策

```java
public class FilterSecurityInterceptor extends AbstractSecurityInterceptor
        implements Filter {

    private AccessDecisionManager accessDecisionManager;

    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException {

        FilterInvocation fi = new FilterInvocation(request, response);

        InterceptorStatusToken token = super.beforeInvocation(fi);

        try {
            fi.getChain().doFilter(fi.getRequest(), fi.getResponse());
        } finally {
            super.afterInvocation(token, null);
        }
    }

    @Override
    protected InterceptorStatusToken beforeInvocation(Object object) {
        // 1. 获取请求需要的 ConfigAttribute
        Collection<ConfigAttribute> attributes =
            obtainSecurityMetadataSource().getAttributes(object);

        // 2. 如果没有配置权限要求，直接放行
        if (attributes == null || attributes.isEmpty()) {
            return null;
        }

        // 3. 检查是否已认证
        Authentication authenticated =
            authenticateIfRequired();

        // 4. 核心：投票决策
        try {
            this.accessDecisionManager.decide(
                authenticated, object, attributes);
        } catch (AccessDeniedException ex) {
            // 抛出异常，由 ExceptionTranslationFilter 处理
            throw ex;
        }
    }
}
```

### decide() 方法

```java
public void decide(Authentication authentication,
                  Object object,
                  Collection<ConfigAttribute> configAttributes)
          throws AccessDeniedException {

    // 遍历所有投票器
    for (AccessDecisionVoter voter : getVoters()) {
        int result = voter.vote(authentication, object, configAttributes);

        if (result == ACCESS_ABSTAIN) {
            continue;  // 弃权，继续下一个
        }

        if (result == ACCESS_GRANTED) {
            // 根据策略处理
            // AffirmativeBased: 直接返回
            // ConsensusBased: 计数
            // UnanimousBased: 继续
        }

        if (result == ACCESS_DENIED) {
            // 根据策略处理
        }
    }
}
```

## 自定义 Voter

### 实现场景

假设你需要根据 IP 地址限制访问：

```java
public class IpAddressVoter implements AccessDecisionVoter<FilterInvocation> {

    private static final String IP_PREFIX = "IP_";

    private List<String> allowedIps = Arrays.asList(
        "192.168.1.*",
        "10.0.0.*"
    );

    @Override
    public int vote(Authentication authentication,
                    FilterInvocation object,
                    Collection<ConfigAttribute> attributes) {

        for (ConfigAttribute attribute : attributes) {
            if (!supports(attribute)) {
                continue;
            }

            String requiredIpPrefix = attribute.getAttribute();
            if (!requiredIpPrefix.startsWith(IP_PREFIX)) {
                continue;
            }

            String requiredIp = requiredIpPrefix.substring(IP_PREFIX.length());
            String clientIp = object.getHttpRequest().getRemoteAddr();

            if (matchesIp(clientIp, requiredIp)) {
                return ACCESS_GRANTED;
            } else {
                return ACCESS_DENIED;
            }
        }

        return ACCESS_ABSTAIN;  // 不涉及 IP 检查，弃权
    }

    private boolean matchesIp(String clientIp, String pattern) {
        if (pattern.contains("*")) {
            String regex = pattern.replace(".", "\\.").replace("*", ".*");
            return clientIp.matches(regex);
        }
        return clientIp.equals(pattern);
    }

    @Override
    public boolean supports(ConfigAttribute attribute) {
        return attribute.getAttribute() != null
            && attribute.getAttribute().startsWith(IP_PREFIX);
    }

    @Override
    public boolean supports(Class<?> clazz) {
        return FilterInvocation.class.isAssignableFrom(clazz);
    }
}
```

### 注册自定义 Voter

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public AccessDecisionManager accessDecisionManager() {
        List<AccessDecisionVoter<? extends Object>> voters = Arrays.asList(
            new WebExpressionVoter(),
            new RoleHierarchyVoter(roleHierarchy()),
            new IpAddressVoter()
        );

        return new AffirmativeBased(voters);
    }

    @Bean
    public RoleHierarchyImpl roleHierarchy() {
        RoleHierarchyImpl hierarchy = new RoleHierarchyImpl();
        hierarchy.setHierarchy("ROLE_ADMIN > ROLE_USER");
        return hierarchy;
    }
}
```

## 常见问题与解决方案

### 问题一：自定义 Voter 不生效

排查步骤：

1. 确认 Voter 注册到了 `AccessDecisionManager`
2. 确认 Voter 的 `supports()` 方法正确
3. 确认投票逻辑正确

### 问题二：投票器弃权太多

如果所有投票器都弃权，`AffirmativeBased` 会通过，`UnanimousBased` 也会通过。

**解决**：添加一个总是投赞成票或反对票的投票器。

### 问题三：角色继承不生效

确认配置了 `RoleHierarchyVoter`：

```java
@Bean
public AccessDecisionManager accessDecisionManager() {
    return new AffirmativeBased(Arrays.asList(
        new WebExpressionVoter(),
        new RoleHierarchyVoter(roleHierarchy())  // 必须配置
    ));
}
```

## 面试追问预判

### Q1：三种决策策略分别适用什么场景？

- `AffirmativeBased`：大多数场景，默认策略
- `ConsensusBased`：需要「少数服从多数」的场景
- `UnanimousBased`：需要「全票通过」的高安全场景

### Q2：投票器返回 ACCESS_ABSTAIN 意味着什么？

弃权。投票器表示「我不关心这个权限决策」，由其他投票器决定。

### Q3：如何在决策时获取更多信息？

可以在 `vote()` 方法中访问 `FilterInvocation`，从中获取请求 URL、参数等信息：

```java

                FilterInvocation object,
                Collection<ConfigAttribute> attributes) {

    HttpServletRequest request = object.getHttpRequest();
    String ip = request.getRemoteAddr();
    String url = request.getRequestURI();

    // ...
}
```

## 总结

`AccessDecisionManager` 是 Spring Security 权限决策的核心：

1. **投票器模式**：每个投票器独立判断，职责单一
2. **三种策略**：`AffirmativeBased`（默认）、`ConsensusBased`、`UnanimousBased`
3. **内置投票器**：`WebExpressionVoter`、`RoleHierarchyVoter`、`AuthenticatedVoter`
4. **自定义投票器**：实现 `AccessDecisionVoter` 接口，注册到 `AccessDecisionManager`
