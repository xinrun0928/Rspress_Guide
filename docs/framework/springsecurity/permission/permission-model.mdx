# 接口权限数据模型：菜单树、按钮权限与数据权限

菜单、按钮、数据——三级权限该怎么设计？

这是权限系统设计的三个层次。菜单权限决定用户能看到哪些页面，按钮权限决定用户能做哪些操作，数据权限决定用户能看到哪些数据。

---

## 权限控制的三个层次

```
├─────────────────────────────────────────────────────────────┤
│                                                              │

│  ├── 系统管理（可见/不可见）                                  │
│  │   ├── 用户管理（可见/不可见）                              │

│  └── 报表中心（可见/不可见）                                  │

│  第二层：按钮权限（能做哪些操作）                              │
│  ├── 用户管理页面                                            │
│  │   ├── [新增用户] 按钮（有/无权限）                         │
│  │   ├── [编辑] 按钮（有/无权限）                            │
│  │   └── [删除] 按钮（有/无权限）                            │
```

#### MyBatis-Plus 递归查询

```java
@Service
public class MenuService {

    @Autowired
    private MenuMapper menuMapper;

    public List<MenuTree> getMenuTree() {
        // 查询所有菜单
        List<Menu> allMenus = menuMapper.selectList(
            new LambdaQueryWrapper<Menu>()
                .eq(Menu::getStatus, 1)
                .orderByAsc(Menu::getSort)
        );

        // 构建树
        return buildTree(allMenus, 0L);
    }

    private List<MenuTree> buildTree(List<Menu> menus, Long parentId) {
        return menus.stream()
            .filter(m -> m.getParentId().equals(parentId))
            .map(m -> {
                MenuTree tree = new MenuTree();
                tree.setId(m.getId());
                tree.setName(m.getName());
                tree.setUrl(m.getUrl());
                tree.setIcon(m.getIcon());
                tree.setPerms(m.getPerms());
                tree.setChildren(buildTree(menus, m.getId()));
                return tree;
            })
            .collect(Collectors.toList());
    }
}
```

### 按钮权限的加载

```java
@Service
public class PermissionService {

    @Autowired
    private MenuMapper menuMapper;

    @Autowired
    private RolePermissionMapper rolePermissionMapper;

    public Set<String> getUserPermissions(Long userId) {
        Set<String> permissions = new HashSet<>();

        // 查询用户的菜单权限（按钮权限）
        List<String> perms = menuMapper.selectUserPermissions(userId);
        permissions.addAll(perms);

        return permissions;
    }
}
```

```xml
<select id="selectUserPermissions" resultType="String">

    WHERE ur.user_id = #{userId}

      AND p.status = 1
```

### 前端权限控制

#### Vue 权限指令

```javascript
// permission.js
export function hasPermission(permission) {
  const permissions = store.getters && store.getters.permissions
  return permissions.includes(permission)
}

export default {
  install(Vue) {
    Vue.directive('permission', {
      inserted(el, binding) {
        const permission = binding.value
        if (permission && !hasPermission(permission)) {
          el.parentNode.removeChild(el)
        }
      }
    })
  }
}
```

#### 使用

```html
<template>
  <div>
    <!-- 只有有 system:user:add 权限才显示 -->
    <el-button v-permission="'system:user:add'" type="primary">
      新增用户
    </el-button>

    <!-- 有编辑或删除权限才显示 -->
    <el-button v-permission="['system:user:edit', 'system:user:delete']" type="danger">
      操作
    </el-button>
  </div>
</template>
```

#### React 权限组件

```jsx

  const { permissions } = useContext(AuthContext);

    return null;
  }

  return children;
```

## URL 权限设计

### URL 权限的正则匹配

Spring Security 支持 Ant 风格的路径匹配：

```java
http.authorizeHttpRequests(auth -> auth
    // 精确匹配
    .requestMatchers("/user/123").hasRole("USER")
    // Ant 风格匹配
    .requestMatchers("/user/*").hasRole("USER")
    .requestMatchers("/user/**").hasRole("USER")
    // Ant 风格 + HTTP 方法
    .requestMatchers(HttpMethod.GET, "/api/users").hasAuthority("user:list")
    .requestMatchers(HttpMethod.POST, "/api/users").hasAuthority("user:add")
    // 正则匹配
    .requestMatchers(RegexRequestMatcher.regexMatchers("/api/user/\\d+"))
        .hasAuthority("user:view")
)
```

### AntPathMatcher

Spring 的 `AntPathMatcher` 用于路径匹配：

```java
AntPathMatcher matcher = new AntPathMatcher();

matcher.match("/user/*", "/user/123");     // true
matcher.match("/user/**", "/user/123/edit"); // true
matcher.match("/user/?", "/user/1");      // true
```

### URL 权限的动态加载

有些场景需要从数据库动态加载 URL 权限：

```java
@Component
public class DynamicUrlFilter extends OncePerRequestFilter {

    @Autowired
    private UrlPermissionMapper urlPermissionMapper;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain)
            throws ServletException, IOException {

        String requestUri = request.getRequestURI();
        String method = request.getMethod();

        // 查询需要的权限
        UrlPermission permission =
            urlPermissionMapper.findByUrlAndMethod(requestUri, method);

        if (permission == null) {
            // 没有配置权限要求的 URL，直接放行
            filterChain.doFilter(request, response);
            return;
        }

        // 检查权限
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        boolean hasPermission = auth.getAuthorities().stream()
            .anyMatch(a -> a.getAuthority().equals(permission.getRequiredPerm()));

        if (hasPermission) {
            filterChain.doFilter(request, response);
        } else {
            response.setStatus(403);
            response.getWriter().write("{\"message\": \"没有权限\"}");
        }
    }
}
```

```sql
CREATE TABLE sys_url_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    url VARCHAR(200) NOT NULL COMMENT 'URL路径',
    method VARCHAR(10) COMMENT 'HTTP方法',
    required_perms VARCHAR(100) COMMENT '需要的权限',
    status TINYINT DEFAULT 1,
    UNIQUE KEY uk_url_method (url, method)
);
```

## 数据权限设计

### 数据权限的分类

| 类型 | 说明 | 示例 |
|---|---|---|
| 行级权限 | 只能看某些行的数据 | 只能看自己的订单 |
| 列级权限 | 只能看某些列的数据 | 只能看订单的基本信息 |
| 单元格权限 | 只能看某些单元格的数据 | 高管才能看薪资 |

### 行级权限实现

#### SQL 层面

```java
@Select("SELECT * FROM orders WHERE user_id = #{userId}")
List<Order> findMyOrders(Long userId);
```

#### AOP 拦截

```java
@Aspect
@Component
public class DataPermissionAspect {

    @Autowired
    private DataScopeMapper dataScopeMapper;

    @Before("execution(* com.example.mapper.*.*(..))")
    public void before(JoinPoint point) {
        // 获取当前用户
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        String username = auth.getName();

        // 查询用户的数据权限范围
        DataScope scope = dataScopeMapper.findByUsername(username);

        // 将数据权限注入到 ThreadLocal
        DataPermissionContext.setDataScope(scope);
    }

    @After("execution(* com.example.mapper.*.*(..))")
    public void after() {
        DataPermissionContext.clear();
    }
}
```

#### MyBatis-Plus 数据权限插件

```java
@Configuration
public class DataPermissionConfig {

    @Bean
    public DataPermissionHandler dataPermissionHandler() {
        return new DataPermissionHandler() {
            @Override
            public Expression getExpression(
                    MapperMetadata mapperMetadata,
                    Class<?> userClass) {

                Authentication auth = SecurityContextHolder.getContext().getAuthentication();
                String username = auth.getName();

                // 根据用户类型添加不同的 WHERE 条件
                if ("admin".equals(username)) {
                    return null;  // 管理员无限制
                }

                // 普通员工只能看自己的数据
                return new StringExpression("create_by = '" + username + "'");
            }
        };
    }
}
```

### 数据权限字段设计

```sql
-- 用户表增加数据权限字段
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY,
    username VARCHAR(50),
    -- 数据权限相关
    data_scope VARCHAR(20) DEFAULT 'SELF' COMMENT '数据范围: ALL/DEPT/DEPT_AND_CHILD/SELF',
    dept_id BIGINT COMMENT '所属部门',
    -- ...
);

-- 数据权限枚举
-- ALL: 所有数据
-- DEPT: 本部门数据
-- DEPT_AND_CHILD: 本部门及下级部门数据
-- SELF: 仅自己的数据
```

### 数据权限 SQL 拼接

```java
@Component
public class DataScopeBuilder {

    public String build(String tableAlias, String userIdField) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        String username = auth.getName();

        // 查询用户信息
        User user = userMapper.findByUsername(username);

        switch (user.getDataScope()) {
            case "ALL":
                return "";  // 无限制
            case "SELF":
                return String.format("%s.%s = '%s'", tableAlias, userIdField, username);
            case "DEPT":
                return String.format("%s.dept_id = %d", tableAlias, user.getDeptId());
            case "DEPT_AND_CHILD":
                List<Long> deptIds = deptMapper.findChildDeptIds(user.getDeptId());
                return String.format("%s.dept_id IN (%s)",
                    tableAlias, String.join(",", deptIds.stream()
                        .map(String::valueOf)
                        .collect(Collectors.joining(","))));
            default:
                return "";
        }
    }
}
```

## 权限的动态加载

### 启动时注册所有接口

```java
@Component
public class ApiPermissionScanner {

    @Autowired
    private RequestMappingHandlerMapping handlerMapping;

    @Autowired
    private UrlPermissionMapper urlPermissionMapper;

    @PostConstruct
    public void scanAndRegister() {
        // 获取所有接口
        Map<RequestMappingInfo, HandlerMethod> handlerMethods =
            handlerMapping.getHandlerMethods();

        for (Map.Entry<RequestMappingInfo, HandlerMethod> entry : handlerMethods.entrySet()) {
            RequestMappingInfo mappingInfo = entry.getKey();
            HandlerMethod handlerMethod = entry.getValue();

            // 获取 URL 和方法
            PatternsRequestCondition patternsCondition =
                mappingInfo.getPatternsCondition();

            if (patternsCondition != null) {
                for (String pattern : patternsCondition.getPatterns()) {
                    for (RequestMethod method : mappingInfo.getMethodsCondition().getMethods()) {
                        // 检查是否有 @PreAuthorize 注解
                        PreAuthorize preAuthorize =
                            handlerMethod.getMethodAnnotation(PreAuthorize.class);

                        if (preAuthorize != null) {
                            // 注册到数据库
                            registerPermission(pattern, method.name(),
                                extractPermission(preAuthorize.value()[0]));
                        }
                    }
                }
            }
        }
    }

    private String extractPermission(String expression) {
        // 从 SpEL 表达式中提取权限
        // hasRole('ADMIN') -> ADMIN
        // hasAuthority('user:list') -> user:list
        if (expression.contains("hasRole('")) {
            return expression.substring(
                expression.indexOf("hasRole('") + 9,
                expression.indexOf("')")
            );
        }
        if (expression.contains("hasAuthority('")) {
            return expression.substring(
                expression.indexOf("hasAuthority('") + 15,
                expression.indexOf("')")
            );
        }
        return null;
    }
}
```

## 常见问题与解决方案

### 问题一：权限变更后需要重新登录

**解决**：使用缓存，权限变更时清除缓存

```java
@CacheEvict(value = "userPermissions", key = "#username")
public void updateUserPermissions(String username) {
    // 更新权限
}
```

### 问题二：超级管理员绕过权限检查

**解决**：在关键操作上加强校验

```java
@PreAuthorize("hasRole('ADMIN') and #id != 1")
public void deleteUser(Long id) {
    // 防止删除超级管理员
}
```

### 问题三：权限粒度过细导致配置复杂

**解决**：使用权限组

```sql
INSERT INTO sys_permission_group (name, permissions)
VALUES ('用户管理', 'user:list,user:add,user:edit,user:delete');
```

## 总结

接口权限数据模型设计三个层次：

1. **菜单权限**：控制用户能看到哪些页面
2. **按钮权限**：控制用户能做哪些操作
3. **数据权限**：控制用户能看到哪些数据

Spring Security 的 `AccessDecisionManager` 负责 URL 级别的权限决策，配合 `PermissionEvaluator` 实现更细粒度的数据权限控制。
