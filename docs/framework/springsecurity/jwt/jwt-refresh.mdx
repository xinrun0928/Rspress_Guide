# JWT 刷新机制：双 Token 方案与黑名单设计

Access Token 过期了，用户被迫登出？Refresh Token 解决了这个问题。

单 Token 方案有一个致命缺陷：Token 过期 = 用户被迫登出。用户正在填写表单，Token 突然过期，体验很差。

双 Token 方案解决了这个问题。

## 为什么需要刷新机制？

### 单 Token 的问题

```
Access Token 有效期：15 分钟
用户上午 10:00 登录
    ↓
10:15 访问 → Token 有效，正常访问
    ↓
10:30 访问 → Token 过期，被迫登出
```

如果 Token 设置得短，频繁过期，用户体验差。
如果 Token 设置得长，安全风险大。

### 双 Token 的解决方案

```
Access Token：短期（15 分钟）
    ↓
用户登录
    ↓
颁发 Access Token + Refresh Token
    ↓
Access Token 过期 → 用 Refresh Token 换取新的 Access Token
    ↓
用户无感知，继续使用
```

## 双 Token 方案的整体架构

```
┌─────────────────────────────────────────────────────────────┐
│              双 Token 认证流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【登录】                                                    │
│  用户登录 → 颁发 Access Token + Refresh Token                 │
│       ↓                                                     │
│  【正常访问】                                                │
│  请求带上 Access Token → 验证通过 → 正常访问                  │
│       ↓                                                     │
│  【Access Token 过期】                                       │
│  请求带上过期的 Access Token → 验证失败                       │
│       ↓                                                     │
│  【刷新 Token】                                             │
│  请求带上 Refresh Token → 验证通过 → 颁发新的 Access Token    │
│       ↓                                                     │
│  【Refresh Token 过期】                                       │
│  需要重新登录                                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Redis 操作

```java
@Service
public class RefreshTokenService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String KEY_PREFIX = "refresh_token:";
    private static final Duration REFRESH_TOKEN_TTL = Duration.ofDays(7);

    /**
     * 存储 Refresh Token
     */
    public void storeRefreshToken(Long userId, RefreshToken refreshToken) {
        String key = KEY_PREFIX + userId;
        redisTemplate.opsForValue().set(key, JsonUtil.toJson(refreshToken), REFRESH_TOKEN_TTL);
    }

    /**
     * 获取 Refresh Token
     */
    public RefreshToken getRefreshToken(Long userId) {
        String key = KEY_PREFIX + userId;
        String json = redisTemplate.opsForValue().get(key);
        if (json == null) {
            return null;
        }
        return JsonUtil.fromJson(json, RefreshToken.class);
    }

    /**
     * 删除 Refresh Token（登出）
     */
    public void removeRefreshToken(Long userId) {
        String key = KEY_PREFIX + userId;
        redisTemplate.delete(key);
    }

    /**
     * 验证 Refresh Token
     */
    public boolean validateRefreshToken(Long userId, String tokenId) {
        RefreshToken storedToken = getRefreshToken(userId);
        if (storedToken == null) {
            return false;
        }
        return storedToken.getTokenId().equals(tokenId);
    }
}
```

## Token 生成和刷新服务

### TokenProvider

```java
@Service
public class TokenProvider {

    @Autowired
    private JwtTokenProvider jwtTokenProvider;

    @Autowired
    private RefreshTokenService refreshTokenService;

    private final long accessTokenExpireMs = 15 * 60 * 1000;  // 15 分钟
    private final long refreshTokenExpireMs = 7 * 24 * 60 * 60 * 1000;  // 7 天

    /**
     * 生成 Token 对
     */
    public TokenPair generateTokenPair(Long userId, String username, List<String> roles) {
        // 1. 生成 Access Token
        String accessToken = jwtTokenProvider.generateAccessToken(
            userId, username, roles, accessTokenExpireMs);

        // 2. 生成 Refresh Token
        String refreshTokenId = UUID.randomUUID().toString();
        RefreshToken refreshToken = new RefreshToken();
        refreshToken.setTokenId(refreshTokenId);
        refreshToken.setUserId(userId);
        refreshToken.setUsername(username);
        refreshToken.setCreateTime(new Date());
        refreshToken.setExpireTime(new Date(
            System.currentTimeMillis() + refreshTokenExpireMs));

        // 3. 存储 Refresh Token
        refreshTokenService.storeRefreshToken(userId, refreshToken);

        return new TokenPair(accessToken, refreshTokenId, accessTokenExpireMs / 1000);
    }

    /**
     * 刷新 Access Token
     */
    public TokenPair refreshToken(String refreshTokenId) {
        // 1. 解析 Refresh Token（不验证过期）
        Claims claims = jwtTokenProvider.parseClaims(refreshTokenId);

        Long userId = Long.parseLong(claims.getSubject());
        String username = claims.get("username", String.class);
        List<String> roles = claims.get("roles", List.class);

        // 2. 验证 Refresh Token
        RefreshToken storedToken = refreshTokenService.getRefreshToken(userId);
        if (storedToken == null || !storedToken.getTokenId().equals(refreshTokenId)) {
            throw new TokenRefreshException("Refresh Token 无效");
        }

        // 3. 验证过期
        if (storedToken.getExpireTime().before(new Date())) {
            refreshTokenService.removeRefreshToken(userId);
            throw new TokenRefreshException("Refresh Token 已过期");
        }

        // 4. 生成新的 Token 对
        return generateTokenPair(userId, username, roles);
    }
}
```

## 刷新接口实现

### Controller

@RestController
@RequestMapping("/auth")
public class AuthController {

    @Autowired
    private TokenProvider tokenProvider;

    @PostMapping("/refresh")
    public ResponseEntity<?> refreshToken(@RequestBody RefreshTokenRequest request) {
        try {
            TokenPair tokenPair = tokenProvider.refreshToken(request.getRefreshToken());
            return ResponseEntity.ok(tokenPair);
        } catch (TokenRefreshException e) {
            return ResponseEntity.status(401)
                .body(Map.of("message", e.getMessage()));
        }
    }
}

## Token 黑名单设计

### 退出登录时的处理

当用户退出登录时，需要让 Refresh Token 失效：

```java
@Service
public class AuthService {

    @Autowired
    private RefreshTokenService refreshTokenService;

    @Autowired
    private JwtTokenProvider jwtTokenProvider;

    public void logout(String accessToken) {
        // 1. 解析 Access Token 获取用户 ID
        Claims claims = jwtTokenProvider.parseClaims(accessToken);
        Long userId = Long.parseLong(claims.getSubject());

        // 2. 删除 Refresh Token
        refreshTokenService.removeRefreshToken(userId);

        // 3. 将 Access Token 加入黑名单
        long ttl = claims.getExpiration().getTime() - System.currentTimeMillis();
        if (ttl > 0) {
            tokenBlacklistService.addToBlacklist(accessToken, ttl);
        }
    }
}
```

### Token 黑名单服务

```java
@Service
public class TokenBlacklistService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String BLACKLIST_PREFIX = "token_blacklist:";

    /**
     * 将 Token 加入黑名单
     */
    public void addToBlacklist(String token, long ttlMillis) {
        String key = BLACKLIST_PREFIX + token;
        redisTemplate.opsForValue().set(key, "1", Duration.ofMillis(ttlMillis));
    }

    /**
     * 检查 Token 是否在黑名单
     */
    public boolean isBlacklisted(String token) {
        String key = BLACKLIST_PREFIX + token;
        return Boolean.TRUE.equals(redisTemplate.hasKey(key));
    }
}
```

## 滑动窗口刷新

### 什么是滑动窗口？

每次使用 Refresh Token 时，都刷新其有效期。这样可以减少 Refresh Token 泄漏的窗口。

```
场景：Refresh Token 有效期 7 天

非滑动窗口：
用户 1月1日 登录
    ↓
1月7日 使用 Refresh Token（过期时间不变）
    ↓
1月8日 Refresh Token 过期

滑动窗口：
用户 1月1日 登录（过期时间：1月8日）
    ↓
1月2日 使用 Refresh Token（过期时间刷新：1月9日）
    ↓
1月5日 使用 Refresh Token（过期时间刷新：1月12日）
    ↓
只要用户持续使用，就持续有效
```

### 实现

```java
public TokenPair refreshToken(String refreshTokenId) {
    // ... 验证逻辑 ...

    // 删除旧的 Refresh Token
    refreshTokenService.removeRefreshToken(userId);

    // 生成新的 Token 对（会自动创建新的 Refresh Token）
    return generateTokenPair(userId, username, roles);
}
```

## 并发刷新控制

### 问题

用户在前端同时发起多个请求，每个请求都触发 Token 刷新，导致多个 Refresh Token 被创建。

### 解决方案

使用 Redis 分布式锁：

```java
public TokenPair refreshToken(String refreshTokenId) {
    // 1. 解析和验证
    Claims claims = jwtTokenProvider.parseClaims(refreshTokenId);
    Long userId = Long.parseLong(claims.getSubject());

    // 2. 获取锁
    String lockKey = "refresh_lock:" + userId;
    String lockValue = UUID.randomUUID().toString();

    Boolean acquired = redisTemplate.opsForValue()
        .setIfAbsent(lockKey, lockValue, Duration.ofSeconds(10));

    if (!Boolean.TRUE.equals(acquired)) {
        // 等待重试
        try {
            Thread.sleep(100);
            return refreshToken(refreshTokenId);  // 递归重试
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new TokenRefreshException("刷新失败，请重试");
        }
    }

    try {
        // 3. 执行刷新逻辑
        return doRefreshToken(userId, claims);
    } finally {
        // 4. 释放锁
        if (lockValue.equals(redisTemplate.opsForValue().get(lockKey))) {
            redisTemplate.delete(lockKey);
        }
    }
}
```

## 常见问题与解决方案

### 问题一：Refresh Token 被盗用

**解决**：
1. 使用 HTTPS
2. 设置较短的 Refresh Token 有效期
3. 设备绑定
4. 异常登录检测（IP、设备变化时报警）

### 问题二：多个设备登录的 Token 管理

**解决**：Redis 按 userId + deviceId 存储

```java
// 存储 key
String key = "refresh_token:" + userId + ":" + deviceId;
```

### 问题三：用户改密码后 Token 失效

**解决**：改密码时清除所有 Refresh Token

```java
public void changePassword(Long userId, String newPassword) {
    // 1. 更新密码
    userRepository.updatePassword(userId, newPassword);

    // 2. 清除所有 Refresh Token
    Set<String> keys = redisTemplate.keys("refresh_token:" + userId + ":*");
    if (keys != null && !keys.isEmpty()) {
        redisTemplate.delete(keys);
    }
}
```

## 面试追问预判

### Q1：双 Token 方案和单 Token 相比有什么优势？

单 Token 方案中，Token 过期就意味着用户被迫登出。双 Token 方案用短期 Access Token 保证安全性，用长期 Refresh Token 保证用户体验。

### Q2：Refresh Token 存储在哪里？

通常存储在服务端（Redis）。也可以存储在客户端（如 HTTPOnly Cookie），但安全性较低。

### Q3：如何防止 Refresh Token 重放攻击？

1. 使用 HTTPS
2. 每次刷新都生成新的 Refresh Token
3. 并发控制（分布式锁）
4. 检查 Refresh Token 的使用频率

## 总结

JWT 刷新机制是保障用户体验和安全的关键：

1. **双 Token 方案**：Access Token 短期有效，Refresh Token 长期有效
2. **Redis 存储**：存储 Refresh Token，支持主动撤销
3. **滑动窗口**：每次使用都刷新 Refresh Token 有效期
4. **并发控制**：防止并发刷新导致的问题
