# JWT 防盗用：设备指纹、IP 绑定与 Token 撤销

Token 被盗了怎么办？

JWT 的优势是无状态，但劣势也是无状态——一旦签发，服务端无法主动撤销。Token 被盗后，在过期前攻击者都可以使用。

这一节，我们来探讨 JWT 的防盗用策略。

## JWT 安全风险分析

### Token 泄漏的场景

```
1. 网络传输中被截获（未使用 HTTPS）
2. XSS 攻击盗取 localStorage
3. CSRF 攻击通过 Cookie 盗用
4. 日志中泄露 Token
5. 代理服务器/网关记录 Token
```

### Token 被盗后的危害

```
Token 被盗
    ↓
攻击者可以冒充受害者
    ↓
在 Token 过期前（可能 15 分钟到 7 天）
    ↓
攻击者可以任意操作
```

## 设备指纹

### 什么是设备指纹？

设备指纹是一组用于唯一标识设备的特征信息。通过对比 Token 中的设备指纹和请求中的设备指纹，可以检测 Token 是否被盗用。

### 常用特征

| 特征 | 说明 | 稳定性 |
|---|---|---|
| User-Agent | 浏览器信息 | 中 |
| Accept-Language | 语言偏好 | 高 |
| Screen 分辨率 | 屏幕尺寸 | 中 |
| Timezone | 时区 | 高 |
| Canvas 指纹 | Canvas 渲染特征 | 高 |
| WebGL 渲染器 | GPU 信息 | 高 |
| 已安装字体 | 字体列表 | 高 |

### 设备指纹生成

```java
@Data
public class DeviceFingerprint {
    private String userAgent;       // User-Agent
    private String language;       // Accept-Language
    private String screen;         // 屏幕分辨率
    private String timezone;       // 时区
    private String canvasHash;     // Canvas 指纹
    private String webglRenderer;  // WebGL 渲染器
    private String fontsHash;      // 字体指纹
}

@Service
public class FingerprintService {

    /**
     * 生成设备指纹哈希
     */
    public String generateFingerprint(DeviceFingerprint fingerprint) {
        String raw = fingerprint.getUserAgent()
            + "|" + fingerprint.getLanguage()
            + "|" + fingerprint.getScreen()
            + "|" + fingerprint.getTimezone()
            + "|" + fingerprint.getCanvasHash()
            + "|" + fingerprint.getFontsHash();

        // 使用 MD5 生成指纹
        return DigestUtils.md5Hex(raw);
    }

    /**
     * 验证设备指纹
     */
    public boolean validateFingerprint(String storedFingerprint,
                                     String requestFingerprint) {
        if (storedFingerprint == null || requestFingerprint == null) {
            return false;
        }
        return storedFingerprint.equals(requestFingerprint);
    }
}
```

        .claim("exp", expiryDate.getTime())
        .signWith(secretKey)
        .compact();
}

## IP 绑定

### 什么是 IP 绑定？

Token 签发时记录用户的 IP，后续请求必须来自同一 IP。

### Token 中存储 IP

```java
public String generateToken(HttpServletRequest request, Long userId,
                            String username, List<String> roles) {
    String clientIp = getClientIp(request);

    return Jwts.builder()
        .subject(String.valueOf(userId))
        .claim("username", username)
        .claim("roles", roles)
        .claim("ip", clientIp)  // 存储 IP
        .signWith(secretKey)
        .compact();
}
```

### IP 验证

```java
public class IpBindingFilter extends OncePerRequestFilter {

    @Autowired
    private JwtTokenProvider jwtTokenProvider;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                   HttpServletResponse response,
                                   FilterChain filterChain)
            throws ServletException, IOException {

        String jwt = getJwtFromRequest(request);

        if (jwt != null && jwtTokenProvider.validateToken(jwt)) {
            String storedIp = jwtTokenProvider.getIpFromToken(jwt);
            String currentIp = getClientIp(request);

            // IP 不匹配，可能是 Token 被盗用
            if (storedIp != null && !storedIp.equals(currentIp)) {
                logger.warn("IP 不匹配！存储的 IP: {}, 当前 IP: {}",
                    storedIp, currentIp);
                // 可以选择拒绝请求或报警
                // throw new SecurityException("IP 不匹配");
            }
        }

        filterChain.doFilter(request, response);
    }
}
```

### IP 获取方法

```java
private String getClientIp(HttpServletRequest request) {
    String ip = request.getHeader("X-Forwarded-For");
    if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
        ip = request.getHeader("Proxy-Client-IP");
    }
    if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
        ip = request.getHeader("WL-Proxy-Client-IP");
    }
    if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
        ip = request.getHeader("HTTP_CLIENT_IP");
    }
    if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
        ip = request.getHeader("HTTP_X_FORWARDED_FOR");
    }
    if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
        ip = request.getRemoteAddr();
    }
    // 多级代理时，取第一个 IP
    if (ip != null && ip.contains(",")) {
        ip = ip.split(",")[0].trim();
    }
    return ip;
}
```

## Token 黑名单机制

### 为什么需要黑名单？

JWT 无法主动撤销，但可以通过黑名单机制实现类似效果。

### 黑名单设计

```java
@Service
public class TokenBlacklistService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String BLACKLIST_PREFIX = "token_blacklist:";

    /**
     * 将 Token 加入黑名单
     */
    public void addToBlacklist(String token, long expirationTime) {
        long ttl = expirationTime - System.currentTimeMillis();
        if (ttl > 0) {
            String key = BLACKLIST_PREFIX + token;
            redisTemplate.opsForValue().set(key, "1", Duration.ofMillis(ttl));
        }
    }

    /**
     * 检查 Token 是否在黑名单
     */
    public boolean isBlacklisted(String token) {
        String key = BLACKLIST_PREFIX + token;
        return Boolean.TRUE.equals(redisTemplate.hasKey(key));
    }
}
```

### 黑名单检查

```java
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired
    private TokenBlacklistService tokenBlacklistService;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                   HttpServletResponse response,
                                   FilterChain filterChain) {
        String jwt = getJwtFromRequest(request);

        if (jwt != null && jwtTokenProvider.validateToken(jwt)) {
            // 检查是否在黑名单
            if (tokenBlacklistService.isBlacklisted(jwt)) {
                SecurityContextHolder.clearContext();
                response.setStatus(401);
                return;
            }

            // ... 设置 Authentication ...
        }

        filterChain.doFilter(request, response);
    }
}
```

### 登出时加入黑名单

```java
@Service
public class AuthService {

    @Autowired
    private TokenBlacklistService tokenBlacklistService;

    @Autowired
    private JwtTokenProvider jwtTokenProvider;

    public void logout(String token) {
        // 解析 Token 获取过期时间
        Claims claims = jwtTokenProvider.parseClaims(token);
        long expirationTime = claims.getExpiration().getTime();

        // 加入黑名单
        tokenBlacklistService.addToBlacklist(token, expirationTime);
    }
}
```

## 综合安全策略

### 安全检查流程

```
请求到达
    ↓
检查 Token 是否在黑名单
    ↓
验证 Token 签名
    ↓
检查 Token 是否过期
    ↓
验证设备指纹（可选）
    ↓
验证 IP 地址（可选）
    ↓
检查异常行为
    ↓
通过 → 继续处理
失败 → 拒绝并报警
```

### 异常行为检测

```java
@Service
public class AnomalyDetectionService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    /**
     * 记录请求
     */
    public void recordRequest(String userId, String ip) {
        String key = "request_count:" + userId + ":" + ip;
        redisTemplate.opsForValue().increment(key);
        redisTemplate.expire(key, Duration.ofMinutes(1));
    }

    /**
     * 检测异常
     */
    public boolean detectAnomaly(String userId, String ip,
                                String deviceFingerprint) {
        // 1. 检查请求频率
        String requestKey = "request_count:" + userId + ":" + ip;
        String count = redisTemplate.opsForValue().get(requestKey);
        if (count != null && Integer.parseInt(count) > 1000) {
            return true;  // 请求过于频繁
        }

        // 2. 检查 IP 变化
        String lastIpKey = "last_ip:" + userId;
        String lastIp = redisTemplate.opsForValue().get(lastIpKey);
        if (lastIp != null && !lastIp.equals(ip)) {
            // IP 发生变化，可能存在安全问题
            return true;
        }

        // 3. 检查设备变化
        String lastFpKey = "last_fp:" + userId;
        String lastFp = redisTemplate.opsForValue().get(lastFpKey);
        if (lastFp != null && !lastFp.equals(deviceFingerprint)) {
            // 设备发生变化
            return true;
        }

        return false;
    }

    /**
     * 更新 IP 和设备信息
     */
    public void updateLastSeen(String userId, String ip, String deviceFingerprint) {
        redisTemplate.opsForValue().set("last_ip:" + userId, ip,
            Duration.ofHours(24));
        redisTemplate.opsForValue().set("last_fp:" + userId, deviceFingerprint,
            Duration.ofHours(24));
    }
}
```

## 完整的 Token 防盗用策略

### 配置

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .anyRequest().authenticated())
            .addFilterBefore(
                new CompositeSecurityFilter(
                    jwtAuthenticationFilter(),
                    ipBindingFilter(),
                    deviceFingerprintFilter(),
                    anomalyDetectionFilter()
                ),
                UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }
}
```

### 综合过滤器

```java
public class CompositeSecurityFilter extends OncePerRequestFilter {

    private final JwtAuthenticationFilter jwtFilter;
    private final IpBindingFilter ipFilter;
    private final DeviceFingerprintFilter fpFilter;
    private final AnomalyDetectionFilter anomalyFilter;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                   HttpServletResponse response,
                                   FilterChain filterChain)
            throws ServletException, IOException {

        // 按顺序执行检查
        if (!jwtFilter.isAuthenticated(request)) {
            // JWT 验证失败
            return;
        }

        if (!ipFilter.validate(request)) {
            // IP 不匹配
            return;
        }

        if (!fpFilter.validate(request)) {
            // 设备指纹不匹配
            return;
        }

        if (anomalyFilter.detect(request)) {
            // 检测到异常行为
            return;
        }

        filterChain.doFilter(request, response);
    }
}
```

## 常见问题与解决方案

### 问题一：用户 VPN 切换 IP 导致误报

**解决**：
1. 使用更宽松的 IP 验证（如 IP 段）
2. 使用设备指纹作为主要验证
3. 对 IP 变化做二次验证

### 问题二：Token 过多导致黑名单过大

**解决**：
1. 只黑名单 Refresh Token
2. Access Token 有效期足够短，无需黑名单
3. 使用 Token 版本号代替黑名单

### 问题三：前端存储 Token 的安全问题

**解决**：
1. 优先使用 HTTPOnly Cookie
2. 如果使用 localStorage，确保无 XSS 漏洞
3. 敏感操作需要额外验证

## 面试追问预判

### Q1：JWT 的安全性如何保证？

1. 使用 HTTPS 防止传输被截获
2. Access Token 短期有效，减少被盗用的窗口
3. 使用 Refresh Token + 黑名单实现主动撤销
4. 加入设备指纹和 IP 验证
5. 异常行为检测

### Q2：如何让 JWT Token 失效？

由于 JWT 无状态，无法直接让 Token 失效。可以通过：
1. 短期 Token
2. Token 黑名单
3. Token 版本号

### Q3：设备指纹能被伪造吗？

可以，但有一定成本。设备指纹主要用于检测异常，而不是防止伪造。配合 IP 验证和行为分析，可以有效检测 Token 盗用。

## 总结

JWT 防盗用是一个系统工程：

1. **设备指纹**：标识设备，检测异常设备
2. **IP 绑定**：验证请求来源 IP
3. **Token 黑名单**：实现主动撤销
4. **异常行为检测**：发现 Token 被盗用的迹象
5. **HTTPS**：防止传输被截获

这些措施组合使用，可以大大提升 JWT 的安全性。理解了这四节 JWT 相关的内容，我们来看 [Spring Security 高级特性](/framework/springsecurity/advanced/index)——框架深处的安全能力。
