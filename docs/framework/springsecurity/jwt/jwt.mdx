# JWT：Token 化的身份令牌

JWT 为什么能在各方之间可靠地传递身份信息？

理解 JWT 的结构，是理解无状态认证的基础。

## JWT 的三部分结构

一个 JWT 由三部分组成，用点号（`.`）分隔：

```
Header.Payload.Signature
```

### 实际例子

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.
eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```

### 为什么这样设计？

- **Header**：告诉接收方这是什么类型的 Token，用什么算法签名
- **Payload**：存放用户信息（Claims）
- **Signature**：防篡改的签名

---

## Header（头部）

### 内容

```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

- `alg`：签名算法（HMAC-SHA256 / RSA / ECDSA）
- `typ`：Token 类型

### Base64 编码

```java
// Java 实现
String header = "{\"alg\":\"HS256\",\"typ\":\"JWT\"}";
String encodedHeader = Base64.getUrlEncoder()
    .withoutPadding()
    .encodeToString(header.getBytes(StandardCharsets.UTF_8));
```

输出：`eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9`

## Payload（载荷）

### Registered Claims（预定义声明）

| Claim | 说明 |
|---|---|
| `iss` | 签发者（Issuer） |
| `sub` | 主题（Subject），通常是用户 ID |
| `aud` | 受众（Audience），Token 发给谁 |
| `exp` | 过期时间（Expiration Time） |
| `nbf` | 生效时间（Not Before） |
| `iat` | 签发时间（Issued At） |
| `jti` | JWT 唯一 ID（JWT ID） |

### Public Claims（公共声明）

可以随意添加，如：

```json
{
  "name": "John Doe",
  "role": "admin",
  "email": "john@example.com"
}
```

### Private Claims（私有声明）

双方约定的自定义声明：

```json
{
  "userId": 12345,
  "username": "zhangsan"
}
```

### 完整示例

```json
{
  "sub": "1234567890",
  "name": "John Doe",
  "role": "admin",
  "permissions": ["user:read", "user:write"],
  "iat": 1516239022,
  "exp": 1516242622
}
```

### Base64 编码

```java
String payload = "{\"sub\":\"1234567890\",\"name\":\"John Doe\",\"iat\":1516239022,\"exp\":1516242622}";
String encodedPayload = Base64.getUrlEncoder()
    .withoutPadding()
    .encodeToString(payload.getBytes(StandardCharsets.UTF_8));
```

## Signature（签名）

### 生成过程

```
Signature = HMACSHA256(
    base64UrlEncode(header) + "." + base64UrlEncode(payload),
    secret
)
```

### Java 实现

```java
public class JwtTokenProvider {

    private final String secret = "your-256-bit-secret";

    public String createToken(String header, String payload) {
        try {
            // 1. 拼接 header.payload
            String data = base64UrlEncode(header) + "." + base64UrlEncode(payload);

            // 2. 使用 HMAC-SHA256 签名
            Mac mac = Mac.getInstance("HmacSHA256");
            SecretKeySpec secretKeySpec = new SecretKeySpec(
                secret.getBytes(StandardCharsets.UTF_8), "HmacSHA256");
            mac.init(secretKeySpec);

            // 3. 生成签名
            byte[] hmacBytes = mac.doFinal(data.getBytes(StandardCharsets.UTF_8));
            String signature = Base64.getUrlEncoder()
                .withoutPadding()
                .encodeToString(hmacBytes);

            // 4. 拼接 token
            return data.substring(0, data.indexOf(".")) + "." +
                   payload.split("\\.")[1] + "." + signature;

        } catch (NoSuchAlgorithmException | InvalidKeyException e) {
            throw new RuntimeException(e);
        }
    }
}
```

### 验证签名

```java
public boolean verifyToken(String token) {
    try {
        // 1. 解析 token
        String[] parts = token.split("\\.");
        String header = parts[0];
        String payload = parts[1];
        String signature = parts[2];

        // 2. 重新计算签名
        String data = header + "." + payload;
        String expectedSignature = calculateSignature(data);

        // 3. 比对签名
        return signature.equals(expectedSignature);
    } catch (Exception e) {
        return false;
    }
}
```

## JJWT 库的使用

JJWT（Java JWT）是最流行的 Java JWT 库。

### 引入依赖

```xml

    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-jackson</artifactId>
    <version>0.12.3</version>
    <scope>runtime</scope>
</dependency>
```

### 生成 Token

```java
@Service
public class JwtTokenProvider {

    private final SecretKey secretKey;

    public JwtTokenProvider() {
        // 生成签名密钥（生产环境需要安全存储）
        this.secretKey = Keys.secretKeyFor(SignatureAlgorithm.HS256);
    }

    public String generateToken(Long userId, String username, List<String> roles) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + 86400000);  // 24小时

        return Jwts.builder()
            .subject(String.valueOf(userId))           // 用户ID
            .claim("username", username)               // 用户名
            .claim("roles", roles)                    // 角色列表
            .issuedAt(now)                             // 签发时间
            .expiration(expiryDate)                   // 过期时间
            .signWith(secretKey)                      // 签名
            .compact();
    }
}
```

### 解析和验证 Token

```java
public Claims parseToken(String token) {
    try {
        return Jwts.parser()
            .verifyWith(secretKey)               // 使用密钥验证签名
            .build()
            .parseSignedClaims(token)             // 解析 Token
            .getPayload();                        // 获取 Payload
    } catch (ExpiredJwtException e) {
        throw new TokenExpiredException("Token 已过期");
    } catch (JwtException e) {
        throw new InvalidTokenException("无效的 Token");
    }
}
```

### 提取 Claims

```java
public Long getUserId(String token) {
    Claims claims = parseToken(token);
    return Long.parseLong(claims.getSubject());
}

public String getUsername(String token) {
    Claims claims = parseToken(token);
    return claims.get("username", String.class);
}

public List<String> getRoles(String token) {
    Claims claims = parseToken(token);
    return claims.get("roles", List.class);
}
```

### 验证过期时间

```java
public boolean isTokenExpired(String token) {
    try {
        Claims claims = parseToken(token);
        return claims.getExpiration().before(new Date());
    } catch (ExpiredJwtException e) {
        return true;
    }
}
```

## JWT 的安全问题

### 不要存储敏感信息

Payload 只是 Base64 编码，可以被解码：

```java
// 恶意用户可以解码 JWT
String payload = new String(Base64.getUrlDecoder().decode(payloadPart));
```

**解决方案**：敏感信息不要放在 JWT 中，或加密存储。

### Token 有效期要合理

| Token 类型 | 推荐有效期 |
|---|---|
| Access Token | 15 分钟 ~ 1 小时 |
| Refresh Token | 7 天 ~ 30 天 |

### 使用 HTTPS

JWT 在请求中传输，必须使用 HTTPS 防止中间人攻击。

### 密钥要安全存储

```java
// 错误：密钥硬编码
private final String secret = "my-secret-key";

// 正确：从配置文件或密钥服务获取
@Value("${jwt.secret}")
private String secret;
```

### 考虑使用非对称加密

如果微服务架构中多个服务需要验证 Token，可以考虑使用 RSA 非对称加密：

- 签发服务使用私钥签名
- 其他服务使用公钥验证
- 不需要每个服务都存储密钥

```java
// 签发时使用私钥
KeyPair keyPair = Keys.keyPairFor(SignatureAlgorithm.RS256);
String token = Jwts.builder()
    .subject(userId)
    .signWith(keyPair.getPrivate())
    .compact();

// 验证时使用公钥
Jwts.parser()
    .verifyWith(keyPair.getPublic())
    .build()
    .parseSignedClaims(token);
```

## 常见问题与解决方案

### 问题一：JWT 被盗用

**场景**：Token 被攻击者截获

**解决方案**：
- 使用 HTTPS
- 设置短期 Access Token
- 使用 Refresh Token + 黑名单
- 加入设备指纹验证

### 问题二：Token 过期后用户被迫登出

**解决方案**：使用双 Token 机制

- Access Token：短期（如 15 分钟）
- Refresh Token：长期（如 7 天）
- Access Token 过期后，用 Refresh Token 换取新的 Access Token

### 问题三：多个微服务如何验证 Token

**解决方案**：
- 方案一：共享密钥（简单但不安全）
- 方案二：使用非对称加密（推荐）
- 方案三：统一认证中心验签

## 面试追问预判

### Q1：JWT 为什么能防篡改？

因为 Signature 部分是用密钥对 Header + Payload 计算的签名。如果 Payload 被篡改，重新签名需要密钥，服务端验证签名时会发现不一致。

### Q2：JWT 和 Session 的区别？

Session 的认证信息存在服务端，客户端只持有 Session ID。JWT 的认证信息编码在 Token 中，服务端只需要验证签名。Session 可以主动失效，JWT 只能等过期。

### Q3：如何让 JWT Token 失效？

JWT 无状态，无法直接让 Token 失效。可以使用：
1. 短期 Token（过期前有效期短）
2. Token 黑名单（Redis 存储已撤销的 Token）
3. Token 版本号（撤销时版本号 +1）

## 总结

JWT 是无状态认证的基础：

1. **结构**：Header.Payload.Signature
2. **签名**：使用 HMAC-SHA256 等算法防篡改
3. **Claims**：存放用户信息的标准格式
4. **JJWT**：Java 生态中处理 JWT 的标准库
5. **安全**：不存敏感信息、使用 HTTPS、短期有效
