# WebSecurityConfigurerAdapter 配置方式演进

很多老项目里，你可能见过这样的配置：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .anyRequest().authenticated()
            .and()
            .formLogin();
    }
}
```

这段代码曾经是 Spring Security 的「标准写法」。但在 Spring Security 6.x 中，`WebSecurityConfigurerAdapter` 已经被**完全移除**。

如果你正在维护旧项目，需要理解这个抽象类的历史；如果你从零开始，需要知道新的配置方式是什么。这一节，完整讲述这段演进历程。

---

## WebSecurityConfigurerAdapter 的三生三世

### 第一世：JDK 5 时代的配置方式

在遥远的 JDK 5 时代（2006 年左右），Spring Security 2.x 使用 `WebSecurityConfigurerAdapter` 作为配置基类：

```java
// Spring Security 2.x
public abstract class WebSecurityConfigurerAdapter {
    // 认证配置
    protected void configure(AuthenticationManagerBuilder auth) throws Exception;

    // HTTP 安全配置
    protected void configure(HttpSecurity http) throws Exception;

    // Web 安全配置
    public void configure(WebSecurity web) throws Exception;
}
```

这种继承式的配置方式，对当时的 Java 开发者来说非常直观：
- 重写 `configure(AuthenticationManagerBuilder)` 配置用户
- 重写 `configure(HttpSecurity)` 配置授权规则
- 重写 `configure(WebSecurity)` 配置全局安全规则

### 第二世：JDK 8 + Spring 5 时代的 Lambda DSL

Java 8 带来了 Lambda 表达式，Spring Security 5.x 顺势推出了 Lambda DSL：

```java
// Spring Security 5.x
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests(authorize -> authorize
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login")
                .permitAll()
            )
            .logout(logout -> logout
                .logoutUrl("/logout")
                .logoutSuccessUrl("/login?logout")
            );
    }
}
```

相比之前的 `and()` 链式调用，Lambda DSL 更简洁、更类型安全：

| 写法 | 特点 |
|---|---|
| `.and().formLogin().xxx()` | 返回 `FormLoginConfigurer`，只能配登录 |
| `.formLogin(form -> form.xxx())` | Lambda 参数就是 `FormLoginConfigurer`，更直观 |

### 第三世：Spring Security 6.x 的完全重构

Spring Security 6.0 在 2022 年发布，最大的变化是：**移除了 `WebSecurityConfigurerAdapter`**。

原因：
1. **组件化优于继承**：继承关系让定制变得困难
2. **Lambda DSL 已经成熟**：不需要抽象类提供类型安全
3. **Spring Security 团队的方向**：走向更灵活的组件配置

新的配置方式完全基于 `SecurityFilterChain` Bean：

```java
// Spring Security 6.x
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    @Order(1)
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/public/**").permitAll()
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login")
                .permitAll()
            );
        return http.build();
    }
}
```

## 新旧配置对比

### 用户认证配置

**旧方式**：`WebSecurityConfigurerAdapter` + `AuthenticationManagerBuilder`

```java
@Override
protected void configure(AuthenticationManagerBuilder auth) throws Exception {
    auth.jdbcAuthentication()
        .dataSource(dataSource)
        .withDefaultSchema()
        .withUser("user")
            .password("password")
            .roles("USER")
        .and()
        .withUser("admin")
            .password("admin")
            .roles("USER", "ADMIN");
}
```

**新方式**：直接定义 `AuthenticationManager` Bean

```java
@Bean
public UserDetailsService userDetailsService() {
    return new InMemoryUserDetailsManager(
        User.withUsername("user")
            .password("password")
            .roles("USER")
            .build(),
        User.withUsername("admin")
            .password("admin")
            .roles("USER", "ADMIN")
            .build()
    );
}

@Bean
public PasswordEncoder passwordEncoder() {
    return PasswordEncoderFactories.createDelegatingPasswordEncoder();
}
```

### HTTP 安全配置

**旧方式**：继承 `WebSecurityConfigurerAdapter`

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/public/**").permitAll()
                .antMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            .and()
            .formLogin()
                .loginPage("/login")
                .permitAll()
            .and()
            .csrf()
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
            .and()
            .httpBasic();
    }
}
```

**新方式**：`SecurityFilterChain` Bean

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/public/**").permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login")
                .permitAll()
            )
            .csrf(csrf -> csrf
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
            )
            .httpBasic(basic -> {});

        return http.build();
    }
}
```

**主要变化**：

| 旧 API | 新 API | 说明 |
|---|---|---|
| `.antMatchers()` | `.requestMatchers()` | 方法名变化 |
| `.antMatchers(HttpMethod, String)` | `.requestMatchers(HttpMethod, String)` | 同上 |
| `.csrf().xxx()` | `.csrf(csrf -> csrf.xxx())` | Lambda 封装 |
| `formLogin().xxx()` | `formLogin(form -> form.xxx())` | Lambda 封装 |

### Web 安全配置

**旧方式**：重写 `configure(WebSecurity)`

```java
@Override
public void configure(WebSecurity web) throws Exception {
    web
        .ignoring()
        .antMatchers("/static/**", "/public/**")
        .and()
        .debug(true);
}
```

**新方式**：注入 `WebSecurityCustomizer`

```java
@Bean
public WebSecurityCustomizer<WebSecurity> webSecurityCustomizer() {
    return web -> web
        .ignoring()
            .requestMatchers("/static/**", "/public/**");
}
```

## 常见迁移问题

### 问题一：@EnableGlobalMethodSecurity 的替代

**旧方式**：

```java
@Configuration
@EnableWebSecurity
@EnableGlobalMethodSecurity(prePostEnabled = true, securedEnabled = true)
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    // ...
}
```

**新方式**：

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity(prePostEnabled = true, securedEnabled = true)
public class SecurityConfig {
    // WebSecurityConfigurerAdapter 已移除，不再继承
}
```

### 问题二：AuthenticationManager 的获取

**旧方式**：直接注入或调用 `authenticationManagerBean()`

```java
@Autowired
private AuthenticationManager authenticationManager;

@Override
protected void configure(AuthenticationManagerBuilder auth) throws Exception {
    auth.authenticationManager(
        new MyAuthenticationManager()
    );
}
```

**新方式**：显式定义 `AuthenticationManager` Bean

```java
@Bean
public AuthenticationManager authenticationManager(
        AuthenticationConfiguration config) throws Exception {
    return config.getAuthenticationManager();
}

@Bean
public AuthenticationProvider myAuthProvider() {
    return new MyAuthenticationProvider();
}
```

### 问题三：多 SecurityFilterChain 配置

**旧方式**：多个 `WebSecurityConfigurerAdapter` + `@Order`

```java
@Configuration
@Order(1)
public static class ApiSecurityConfig
        extends WebSecurityConfigurerAdapter {
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .antMatcher("/api/**")
            .authorizeRequests()
            .anyRequest().hasRole("API_USER");
    }
}

@Configuration
@Order(2)
public static class FormSecurityConfig
        extends WebSecurityConfigurerAdapter {
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
            .anyRequest().authenticated()
            .and()
            .formLogin();
    }
}
```

**新方式**：多个 `SecurityFilterChain` + `@Order`

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    @Order(1)
    public SecurityFilterChain apiFilterChain(HttpSecurity http)
            throws Exception {
        http
            .securityMatcher("/api/**")
            .authorizeHttpRequests(auth -> auth
                .anyRequest().hasRole("API_USER")
            );
        return http.build();
    }

    @Bean
    @Order(2)
    public SecurityFilterChain formFilterChain(HttpSecurity http)
            throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .anyRequest().authenticated()
            )
            .formLogin(form -> form.permitAll());
        return http.build();
    }
}
```

## 面试追问预判

### Q1：为什么 Spring Security 要移除 WebSecurityConfigurerAdapter？

考察你对框架设计的理解。回答要点：

1. **继承 vs 组合**：继承导致子类与父类强耦合，定制困难
2. **Lambda DSL 的成熟**：Java 8 之后，Lambda DSL 已经能提供类型安全
3. **组件化的趋势**：Spring 框架整体走向组件化配置（@Bean > 继承）
4. **消除魔法**：移除隐藏的自动配置，让行为更可预测

### Q2：新的配置方式有什么优势？

- **灵活性更高**：可以自由组合不同的 SecurityFilterChain
- **测试更简单**：不需要创建测试子类
- **Bean 生命周期更清晰**：所有组件都是 Spring 管理的 Bean
- **Lambda DSL 更简洁**：避免 `and()` 的链式调用

### Q3：如果项目还在用 WebSecurityConfigurerAdapter，能升级到 6.x 吗？

可以，但需要迁移。迁移步骤：

1. 创建 `SecurityFilterChain` Bean
2. 将 `configure(HttpSecurity)` 的内容迁移到 lambda DSL
3. 将 `configure(AuthenticationManagerBuilder)` 的内容迁移到 `UserDetailsService` Bean
4. 删除 `extends WebSecurityConfigurerAdapter`
5. 删除 `@EnableWebSecurity`（6.x 默认启用 Web Security）

## 总结

`WebSecurityConfigurerAdapter` 的演进，是 Spring Security 从「继承式配置」到「组件化配置」转变的缩影：

```
JDK 5 时代：继承 + 方法重写
    ↓
JDK 8 时代：继承 + Lambda DSL
    ↓
Spring Security 6.x：组件化 + Lambda DSL
```

无论配置方式怎么变，核心概念不变：
- `SecurityFilterChain` 定义过滤器链
- `AuthenticationManager` 管理认证
- `UserDetailsService` 加载用户信息
- `GrantedAuthority` 定义权限
