## 知识图谱

```
SqlSessionFactoryBuilder.build()
        ↓
XmlConfigBuilder.parseConfiguration()
        ↓
Configuration 对象（核心容器）
        ↓
DefaultSqlSessionFactory
        ↓
openSession() → DefaultSqlSession
        ↓
getMapper() → MapperProxy (JDK 动态代理)
        ↓
代理对象调用方法 → Executor → StatementHandler → ParameterHandler → JDBC
        ↓
ResultSetHandler → 结果映射
```

## 为什么要深入理解 MyBatis 核心原理？

大多数开发者会用 MyBatisCRUD，但遇到以下问题时会束手无策：

- **SQL 找不到**：Mapper 接口明明定义了方法，为什么提示找不到 SQL？
- **参数绑定失败**：#{} 和 ${} 到底有什么区别？什么时候用哪个？
- **缓存不生效**：一级缓存和二级缓存的区别是什么？为什么增删改会清缓存？
- **N+1 问题**：嵌套查询为什么会产生 N+1 问题？如何解决？
- **性能瓶颈**：为什么同一个查询第二次执行会快很多？

理解核心原理后，这些问题都将迎刃而解。

## 面试考察分析

MyBatis 核心原理是 Java 后端面试的高频考点，尤其是：

1. **四大组件的协作关系**：Executor、StatementHandler、ParameterHandler、ResultSetHandler
2. **Mapper 代理机制**：JDK 动态代理的 invoke() 方法如何路由到正确的 SQL
3. **缓存机制**：一级缓存（SqlSession 级别）和二级缓存（Mapper 级别）的实现差异

这些问题不仅考察你是否"会用"，更考察你是否理解框架底层的设计思想。
