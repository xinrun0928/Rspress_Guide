# SqlSessionFactory 创建流程与生命周期

一个应用只需要一个 SqlSessionFactory，还是每个线程需要一个？SqlSessionFactory 什么时候创建？什么时候销毁？

这些问题看似简单，但错误的生命周期管理可能导致数据库连接泄漏、事务混乱，甚至生产环境的诡异 bug。

## SqlSessionFactory 的创建

### 创建入口

```java
String resource = "mybatis-config.xml";
InputStream inputStream = Resources.getResourceAsStream(resource);
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
```

`SqlSessionFactoryBuilder` 会根据配置创建一个 `DefaultSqlSessionFactory`：

```java title="SqlSessionFactoryBuilder.java"
public SqlSessionFactory build(InputStream inputStream) {
    return build(new XMLConfigBuilder(inputStream).parse());
}

public SqlSessionFactory build(Configuration config) {
    return new DefaultSqlSessionFactory(config);
}
```

### DefaultSqlSessionFactory 的结构

```java title="DefaultSqlSessionFactory.java"
public class DefaultSqlSessionFactory implements SqlSessionFactory {

    private final Configuration configuration;

    public DefaultSqlSessionFactory(Configuration configuration) {
        this.configuration = configuration;
    }

    @Override
    public SqlSession openSession() {
        return openSessionFromDataSource(
            configuration.getDefaultExecutorType(),
            TransactionIsolationLevel.NONE,
            false
        );
    }

    @Override
    public SqlSession openSession(boolean autoCommit) {
        return openSessionFromDataSource(
            configuration.getDefaultExecutorType(),
            TransactionIsolationLevel.NONE,
            autoCommit
        );
    }

    @Override
    public SqlSession openSession(Connection connection) {
        return new DefaultSqlSession(
            configuration,
            connection,
            false
        );
    }

    // 从数据源创建 SqlSession
    private SqlSession openSessionFromDataSource(ExecutorType execType,
                                                  TransactionIsolationLevel level,
                                                  boolean autoCommit) {
        Transaction tx = null;
        try {
            // 从 Configuration 获取 Environment
            final Environment environment = configuration.getEnvironment();
            // 获取数据源
            final DataSource dataSource = getDataSource(environment);
            // 获取事务工厂
            final TransactionFactory transactionFactory = getTransactionFactory(environment);

            // 创建事务
            tx = transactionFactory.newTransaction(dataSource, level, autoCommit);

            // 创建执行器
            final Executor executor = configuration.newExecutor(tx, execType);

            // 创建 SqlSession
            return new DefaultSqlSession(configuration, executor, autoCommit);
        } catch (Exception e) {
            closeTransaction(tx);
            throw ExceptionFactory.wrapException("Error opening session", e);
        }
    }
}
```

## 生命周期管理

### 正确的生命周期

| 对象 | 作用域 | 创建时机 | 销毁时机 |
|---|---|---|---|
| SqlSessionFactoryBuilder | 方法内 | 使用时 | 方法结束自动回收 |
| SqlSessionFactory | 应用级 | 应用启动时（一次） | 应用关闭时 |
| SqlSession | 方法内 | 每次数据库操作 | 方法结束时关闭 |
| Transaction | 方法内 | 每个 SqlSession | 提交或回滚后 |
| Executor | SqlSession 内 | 每次 openSession() | SqlSession 关闭时 |

### 应用级单例：SqlSessionFactory

**SqlSessionFactory 应该是单例的**，在应用生命周期内只创建一个实例。

```java title="MyBatisUtil.java"
public class MyBatisUtil {

    private static SqlSessionFactory sqlSessionFactory;

    static {
        try {
            String resource = "mybatis-config.xml";
            InputStream inputStream = Resources.getResourceAsStream(resource);
            sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
        } catch (IOException e) {
            throw new ExceptionInInitializerError(e);
        }
    }

    public static SqlSessionFactory getSqlSessionFactory() {
        return sqlSessionFactory;
    }

    public static SqlSession getSqlSession() {
        return sqlSessionFactory.openSession();
    }
}
```

### 方法级：SqlSession

**SqlSession 不能是单例的**，每个线程应该有自己的 SqlSession：

```java
// 错误示例：SqlSession 作为静态变量
private static SqlSession sqlSession; // ❌ 线程不安全！

// 正确示例：每次获取新的 SqlSession
public void doSomething() {
    try (SqlSession session = sqlSessionFactory.openSession()) {
        UserMapper mapper = session.getMapper(UserMapper.class);
        User user = mapper.selectById(1);
        session.commit();
    } // 自动关闭
}
```

### SqlSession 关闭的重要性

SqlSession 必须关闭，否则数据库连接会泄漏：

```java
// 方式一：try-with-resources（推荐）
try (SqlSession session = sqlSessionFactory.openSession()) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    mapper.insert(user);
    session.commit();
}

// 方式二：finally 块
SqlSession session = null;
try {
    session = sqlSessionFactory.openSession();
    // ... 操作
    session.commit();
} finally {
    if (session != null) {
        session.close();
    }
}
```

## 与 Spring 整合时的生命周期

### Spring 管理 SqlSessionFactory

在 Spring 环境中，SqlSessionFactory 由 Spring 管理：

```xml
<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="dataSource" ref="dataSource"/>
    <property name="mapperLocations" value="classpath:mapper/*.xml"/>
</bean>
```

```java
@Autowired
private SqlSessionFactory sqlSessionFactory;
```

### SqlSessionTemplate：线程安全的 SqlSession

Spring 中使用 `SqlSessionTemplate` 替代原生的 `SqlSession`，它是线程安全的：

```java
@Component
public class UserService {

    @Autowired
    private SqlSessionTemplate sqlSessionTemplate;

    public User selectById(Long id) {
        // SqlSessionTemplate 内部管理 SqlSession
        return sqlSessionTemplate.selectOne("com.example.UserMapper.selectById", id);
    }
}
```

**SqlSessionTemplate 的原理**：

```java title="SqlSessionTemplate.java"
public class SqlSessionTemplate implements SqlSession {

    private final SqlSessionFactory sqlSessionFactory;
    private final ExecutorType executorType;

    // 每个线程使用不同的 SqlSession
    private final SqlSessionFactory sqlSessionFactoryProxy;

    @Override
    public <T> T selectOne(String statement) {
        // 动态代理，每次获取真实的 SqlSession
        SqlSession sqlSession = getSqlSession(
            sqlSessionFactory,
            executorType,
            exceptionTranslator
        );
        return sqlSession.selectOne(statement);
    }
}
```

### MapperScannerConfigurer：自动扫描注册

Spring Boot 和 SSM 框架中，使用 MapperScannerConfigurer 自动扫描 Mapper 接口：

```xml
<bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
    <property name="basePackage" value="com.example.mapper"/>
</bean>
```

```java
// 等价于 Spring Boot 的 @MapperScan
@MapperScan("com.example.mapper")
public class Application { }
```

自动扫描后，Mapper 接口会被注册为 Spring Bean，可以直接 `@Autowired` 注入。

## 事务管理

### Spring 声明式事务

```java
@Service
public class UserService {

    @Autowired
    private UserMapper userMapper;

    @Transactional
    public void transfer(Long fromId, Long toId, BigDecimal amount) {
        userMapper.decreaseBalance(fromId, amount);
        // 如果这里抛出异常，事务会回滚
        userMapper.increaseBalance(toId, amount);
    }
}
```

### 手动事务控制

```java
public void doSomething() {
    SqlSession session = sqlSessionFactory.openSession();
    try {
        UserMapper mapper = session.getMapper(UserMapper.class);

        // 手动开启事务（默认 autoCommit = false）
        mapper.insert(user);

        // 手动提交
        session.commit();
    } catch (Exception e) {
        // 回滚
        session.rollback();
        throw e;
    } finally {
        session.close();
    }
}
```

## 面试重点

| 问题 | 回答要点 |
|---|---|
| SqlSessionFactory 为什么是单例？ | 配置只解析一次，创建成本高 |
| SqlSession 为什么不能是单例？ | 不是线程安全的，包含数据库连接 |
| SqlSession.close() 做了什么？ | 关闭 Executor，释放数据库连接回连接池 |
| Spring 中 Mapper 是如何被注入的？ | MapperScannerConfigurer 扫描注册为 Spring Bean |

---

## 延伸思考

如果在 `session.commit()` 之前就调用了 `session.close()`，事务会提交吗？

答案是**不会**。`close()` 方法会先判断是否有活跃事务，如果有则先回滚。这是为了防止忘记提交时数据不会丢失，但也可能导致一些意外的行为——比如你以为事务没提交，实际上已经回滚了。
