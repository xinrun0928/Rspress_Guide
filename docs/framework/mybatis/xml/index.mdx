# MyBatis XML 映射：SQL 的另一种表达方式

MyBatis 支持两种定义 SQL 的方式：XML 映射和注解映射。

这一章节，我们聚焦 XML 映射。XML 文件是 MyBatis 最核心的配置文件，在这里你可以定义 SQL 语句、结果映射、动态 SQL，几乎所有的 MyBatis 功能都离不开 XML 配置。

## 为什么需要 XML 映射？

在开始之前，先回答一个问题：既然可以用注解写 SQL，为什么还需要 XML？

| 特性 | XML 映射 | 注解映射 |
|------|---------|---------|
| SQL 可读性 | 高（SQL 可以换行、格式化） | 低（SQL 在一行） |
| 动态 SQL | 强大（9 个标签） | 弱（需要拼接字符串） |
| 复杂映射 | 支持 resultMap | 支持但不直观 |
| 团队协作 | 易于版本管理 | 易于冲突 |
| SQL 调试 | 可以在 IDE 中格式化 | 需要运行才能看到 |

对于复杂的业务系统，XML 映射仍然是主流选择。

## XML mapper 文件结构

一个完整的 MyBatis XML mapper 文件如下：

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.UserMapper">

    <!-- 缓存配置 -->
    <cache eviction="LRU" flushInterval="60000" size="256" readOnly="true"/>

    <!-- 可复用的 SQL 片段 -->
    <sql id="userColumns">
        id, username, email, created_at, updated_at
    </sql>

    <!-- parameterMap（已不推荐使用） -->
    <parameterMap id="userParamMap" type="com.example.User">
        <parameter property="id" jdbcType="BIGINT"/>
        <parameter property="name" jdbcType="VARCHAR"/>
    </parameterMap>

    <!-- 结果映射 -->
    <resultMap id="userResultMap" type="com.example.User">
        <id property="id" column="id"/>
        <result property="username" column="username"/>
        <result property="email" column="email"/>
    </resultMap>

    <!-- SELECT 语句 -->
    <select id="selectById" resultMap="userResultMap">
        SELECT <include refid="userColumns"/>
        FROM user
        WHERE id = #{id}
    </select>

    <!-- INSERT 语句 -->
    <insert id="insert" parameterType="com.example.User">
        INSERT INTO user (username, email)
        VALUES (#{username}, #{email})
    </insert>

    <!-- UPDATE 语句 -->
    <update id="update">
        UPDATE user
        SET username = #{username}, email = #{email}
        WHERE id = #{id}
    </update>

    <!-- DELETE 语句 -->
    <delete id="delete">
        DELETE FROM user WHERE id = #{id}
    </delete>

</mapper>
```

## `<mapper>` 根元素

`<mapper>` 是 XML 文件的根元素，它有一个关键属性：`namespace`。

### namespace 的作用

**第一：与接口绑定**

```xml
<mapper namespace="com.example.UserMapper">
```

```java
public interface UserMapper {
    User selectById(Long id);
}
```

MyBatis 会将 XML 中的 SQL 与接口方法一一对应：

- `namespace` + `id` = 唯一标识
- 方法名与 `id` 必须一致

**第二：分类管理 SQL**

不同 Mapper 的 SQL 不会冲突：

```xml
<!-- UserMapper.xml -->
<mapper namespace="com.example.UserMapper">
    <select id="selectById">...</select>
</mapper>

<!-- OrderMapper.xml -->
<mapper namespace="com.example.OrderMapper">
    <select id="selectById">...</select>
</mapper>
```

**第三：支持增删改的自动刷新**

增删改操作会自动清空对应 namespace 的二级缓存。

## 四种 SQL 标签

| 标签 | 用途 | 返回值 |
|------|------|--------|
| `<select>` | 查询语句 | `resultType` 或 `resultMap` |
| `<insert>` | 插入语句 | `int`（影响行数）或 `KeyHolder`（自增主键） |
| `<update>` | 更新语句 | `int`（影响行数） |
| `<delete>` | 删除语句 | `int`（影响行数） |

### `<select>` 标签属性

```xml
<select
    id="selectById"                           <!-- 唯一标识（必须） -->
    parameterType="Long"                      <!-- 参数类型 -->
    resultType="com.example.User"            <!-- 返回类型（二选一） -->
    resultMap="userResultMap"                 <!-- 结果映射（二选一） -->
    flushCache="false"                        <!-- 是否刷新缓存 -->
    useCache="true"                           <!-- 是否使用二级缓存 -->
    timeout="10000"                           <!-- 超时时间（毫秒） -->
    fetchSize="100"                           <!-- 抓取大小 -->
    statementType="PREPARED"                  <!-- Statement 类型 -->
    resultSetType="FORWARD_ONLY">            <!-- ResultSet 类型 -->
    SELECT * FROM user WHERE id = #{id}
</select>
```

### `<insert>` 标签属性

```xml
<insert
    id="insert"
    parameterType="com.example.User"
    flushCache="true"
    statementType="PREPARED"
    useGeneratedKeys="true"                  <!-- 使用自增主键 -->
    keyProperty="id"                          <!-- 主键属性 -->
    keyColumn="id">                          <!-- 主键列 -->
    INSERT INTO user (name, email)
    VALUES (#{name}, #{email})
</insert>
```

### 自增主键获取

```xml
<!-- MySQL 自增主键 -->
<insert id="insert" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO user (name, email)
    VALUES (#{name}, #{email})
</insert>
```

```java
User user = new User();
user.setName("张三");
user.setEmail("zhangsan@example.com");
mapper.insert(user);
System.out.println(user.getId());  // 自动填充了自增 ID
```

### 非自增主键获取

```xml
<!-- Oracle 序列 -->
<insert id="insert">
    <selectKey keyProperty="id" resultType="long" order="BEFORE">
        SELECT user_seq.NEXTVAL FROM DUAL
    </selectKey>
    INSERT INTO user (id, name, email)
    VALUES (#{id}, #{name}, #{email})
</insert>
```

## `<sql>` 可复用片段

### 定义和引用

```xml
<sql id="userColumns">
    id, username, email, created_at, updated_at
</sql>

<select id="selectAll" resultType="com.example.User">
    SELECT <include refid="userColumns"/>
    FROM user
</select>

<select id="selectById" resultType="com.example.User">
    SELECT <include refid="userColumns"/>
    FROM user
    WHERE id = #{id}
</select>
```

### 支持变量替换

```xml
<sql id="userColumns">
    ${prefix}_id, ${prefix}_name, ${prefix}_email
</sql>

<select id="selectAll" resultType="com.example.User">
    SELECT <include refid="userColumns">
        <property name="prefix" value="u"/>
    </include>
    FROM user u
</select>
```

## `<cache>` 缓存配置

```xml
<!-- 基础配置 -->
<cache/>

<!-- 自定义配置 -->
<cache
    eviction="LRU"           <!-- 淘汰策略：LRU/FIFO/SOFT/WEAK -->
    flushInterval="60000"    <!-- 刷新间隔（毫秒） -->
    size="256"               <!-- 缓存容量 -->
    readOnly="true"/>        <!-- 是否只读 -->
```

## `<parameterMap>` 参数映射

**注意**：`<parameterMap>` 已被官方标记为过时，推荐直接使用 `parameterType`。

```xml
<!-- 过时写法 -->
<parameterMap id="userParamMap" type="com.example.User">
    <parameter property="id" jdbcType="BIGINT" mode="IN"/>
    <parameter property="name" jdbcType="VARCHAR" mode="IN"/>
</parameterMap>

<!-- 推荐写法 -->
<insert id="insert" parameterType="com.example.User">
    INSERT INTO user (id, name)
    VALUES (#{id}, #{name})
</insert>
```

## `<resultMap>` 结果映射

resultMap 是 MyBatis 最强大的功能之一：

```xml
<resultMap id="userResultMap" type="com.example.User">
    <!-- 主键映射（用于缓存 key 计算） -->
    <id property="id" column="user_id"/>

    <!-- 普通字段映射 -->
    <result property="username" column="user_name"/>
    <result property="email" column="user_email"/>

    <!-- 一对一关联 -->
    <association property="department" javaType="com.example.Department">
        <id property="id" column="dept_id"/>
        <result property="name" column="dept_name"/>
    </association>

    <!-- 一对多关联 -->
    <collection property="orders" ofType="com.example.Order">
        <id property="id" column="order_id"/>
        <result property="orderNo" column="order_no"/>
    </collection>
</resultMap>
```

## XML 校验：DTD 与 XSD

MyBatis XML 使用 DTD（Document Type Definition）进行校验：

```xml
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
```

DTD 文件定义了 XML 中可以使用哪些元素、哪些属性、哪些枚举值。通过 DTD 校验，IDE 可以提供智能提示和错误检查。

## 文件组织建议

### 目录结构

```
src/
└── main/
    └── resources/
        └── mapper/
            ├── user/
            │   ├── UserMapper.xml
            │   └── UserExtendMapper.xml
            └── order/
                └── OrderMapper.xml
```

### 配置文件关联

```xml
<!-- mybatis-config.xml -->
<mappers>
    <!-- 方式一：单个文件注册 -->
    <mapper resource="mapper/user/UserMapper.xml"/>

    <!-- 方式二：包扫描（推荐） -->
    <package name="mapper"/>
</mappers>
```

## 延伸思考

### XML 和注解如何选择？

根据团队经验和项目特点来选择：

**推荐使用 XML**：
- SQL 复杂度高，需要动态 SQL
- 团队人多，容易冲突
- 需要频繁调试 SQL
- resultMap 复杂

**推荐使用注解**：
- SQL 简单（单表 CRUD）
- 项目小，SQL 少
- 动态 SQL 需求少

**混合使用**：
- 简单 SQL 用注解
- 复杂 SQL 用 XML

### 你可能想问的问题

**Q1：namespace 可以不配置吗？**

可以，但会失去接口绑定功能，只能通过 `SqlSession` 的 `selectOne("statementId", param)` 方式调用。

**Q2：多个 XML 文件可以有相同的 namespace 吗？**

可以，但 `id` 必须唯一。MyBatis 会合并同一 namespace 下的所有 SQL。

**Q3：XML 文件修改后需要重启吗？**

如果使用 `SqlSessionFactory` 的默认方式构建，需要重启。但如果使用 Spring Boot 的热部署或者配置了 mapper locations 的热加载，则不需要。
