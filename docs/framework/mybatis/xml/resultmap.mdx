# resultMap：MyBatis 结果映射的终极武器

<Badge text="MyBatis" type="tip" /> <Badge text="核心原理" type="info" />

resultType 自动映射简单，resultMap 才能搞定复杂场景。

大多数简单的 SELECT 查询，用 `resultType` 就够了——MyBatis 会自动把字段映射到同名的 Java 属性上。但当遇到以下场景时，`resultType` 就力不从心了：

- 数据库字段名和 Java 属性名不一致
- 一对一、一对多关联查询
- 需要特殊处理的数据转换
- 多态映射（同一个查询返回不同类型的对象）

这时候，`resultMap` 就是你的终极武器。

## 基础结构

```xml
<resultMap id="userResultMap" type="com.example.entity.User">
    <!-- 主键映射（必须，用于缓存 key 计算） -->
    <id property="id" column="user_id"/>

    <!-- 普通字段映射 -->
    <result property="username" column="user_name"/>
    <result property="email" column="user_email"/>
    <result property="createdAt" column="created_at"/>
</resultMap>
```

**核心属性**：

| 属性 | 说明 |
|------|------|
| `id` | resultMap 的唯一标识 |
| `type` | 映射的目标类型（全限定类名或别名） |
| `autoMapping` | 是否开启自动映射，默认 `true` |

## `<id>` vs `<result>`

```xml
<resultMap id="userResultMap" type="User">
    <!-- id：主键映射 -->
    <id property="id" column="id"/>

    <!-- result：普通字段映射 -->
    <result property="username" column="username"/>
</resultMap>
```

**两者的区别**：

| 特性 | `<id>` | `<result>` |
|------|--------|------------|
| 用途 | 主键映射 | 普通字段映射 |
| 缓存 key | 参与计算 | 不参与 |
| 嵌套映射去重 | 会去重 | 不去重 |
| 性能 | 更快 | 稍慢 |

**为什么 id 更快？**

主键是唯一的，MyBatis 可以用它做快速查找和去重。当处理嵌套结果集时，id 相同的记录会被认为是同一个对象。

## `<association>` 一对一映射

### 嵌套 select 方式

```xml
<!-- 先查用户，再根据 dept_id 查部门 -->
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>
    <result property="username" column="username"/>

    <!-- column: 传给子查询的参数 -->
    <association property="department"
                 column="dept_id"
                 select="com.example.mapper.DepartmentMapper.selectById"/>
</resultMap>

<select id="selectAll" resultMap="userResultMap">
    SELECT * FROM user
</select>
```

**生成的 SQL**：

```sql
-- 第一次查询
SELECT * FROM user

-- 第二次查询（N 次，每次查一个部门）
SELECT * FROM department WHERE id = ?
SELECT * FROM department WHERE id = ?
...
```

**问题**：会产生 N+1 查询！

### 嵌套 resultMap 方式（推荐）

```xml
<!-- 一次 JOIN 查完 -->
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>
    <result property="username" column="username"/>

    <association property="department" javaType="Department">
        <id property="id" column="dept_id"/>
        <result property="name" column="dept_name"/>
        <result property="code" column="dept_code"/>
    </association>
</resultMap>

<select id="selectAll" resultMap="userResultMap">
    SELECT
        u.id, u.username, u.dept_id,
        d.id AS dept_id, d.name AS dept_name, d.code AS dept_code
    FROM user u
    LEFT JOIN department d ON u.dept_id = d.id
</select>
```

**生成的 SQL**：

```sql
-- 一次查询，JOIN 搞定
SELECT
    u.id, u.username, u.dept_id,
    d.id AS dept_id, d.name AS dept_name, d.code AS dept_code
FROM user u
LEFT JOIN department d ON u.dept_id = d.id
```

**优化点**：只产生 1 次数据库查询！

### fetchType 控制加载方式

```xml
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>
    <result property="username" column="username"/>

    <!-- 强制立即加载（覆盖全局延迟加载配置） -->
    <association property="department"
                 javaType="Department"
                 fetchType="eager">
        <id property="id" column="dept_id"/>
        <result property="name" column="dept_name"/>
    </association>
</resultMap>
```

| fetchType | 效果 |
|-----------|------|
| `eager` | 立即加载，不等待访问关联对象 |
| `lazy` | 延迟加载，访问关联对象时才查询 |

## `<collection>` 一对多映射

### 嵌套 select 方式

```xml
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>
    <result property="username" column="username"/>

    <collection property="orders"
                ofType="Order"
                column="id"
                select="com.example.mapper.OrderMapper.selectByUserId"/>
</resultMap>
```

**问题**：会产生 1 + N 次查询！

### 嵌套 resultMap 方式（推荐）

```xml
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>
    <result property="username" column="username"/>

    <collection property="orders" ofType="Order">
        <id property="id" column="order_id"/>
        <result property="orderNo" column="order_no"/>
        <result property="totalAmount" column="total_amount"/>
    </collection>
</resultMap>

<select id="selectAll" resultMap="userResultMap">
    SELECT
        u.id, u.username,
        o.id AS order_id, o.order_no, o.total_amount
    FROM user u
    LEFT JOIN `order` o ON u.id = o.user_id
</select>
```

**关键点**：
- `ofType`：集合中元素的类型（不是 Java 类型）
- JOIN 查询后，MyBatis 会自动按主键去重，构建一对多关系

### 嵌套集合

```xml
<!-- 用户 → 订单 → 订单明细 -->
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>

    <collection property="orders" ofType="Order">
        <id property="id" column="order_id"/>

        <!-- 嵌套的 collection -->
        <collection property="items" ofType="OrderItem">
            <id property="id" column="item_id"/>
            <result property="productName" column="product_name"/>
        </collection>
    </collection>
</resultMap>
```

## `<constructor>` 构造器注入

默认情况下，MyBatis 会调用无参构造器创建对象，然后通过 setter 方法设置属性。但如果实体类没有无参构造器，或者想通过构造器注入，就需要使用 `<constructor>`。

### 基本用法

```java
public class User {
    private final Long id;
    private final String username;

    // 只能通过构造器创建（没有无参构造器）
    public User(Long id, String username) {
        this.id = id;
        this.username = username;
    }
}
```

```xml
<resultMap id="userResultMap" type="User">
    <constructor>
        <idArg column="id" javaType="long"/>
        <arg column="username" javaType="String"/>
    </constructor>
</resultMap>
```

### 指定参数名

如果构造器参数名和列名不一致，可以使用 `name` 属性：

```xml
<resultMap id="userResultMap" type="User">
    <constructor>
        <idArg column="user_id" javaType="long" name="id"/>
        <arg column="user_name" javaType="String" name="username"/>
    </constructor>
</resultMap>
```

### 混合使用

```xml
<resultMap id="userResultMap" type="User">
    <!-- 通过构造器注入 -->
    <constructor>
        <idArg column="id" javaType="long"/>
        <arg column="username" javaType="String"/>
    </constructor>

    <!-- 通过 setter 注入 -->
    <result property="email" column="email"/>
    <result property="createdAt" column="created_at"/>
</resultMap>
```

## `<discriminator>` 鉴别器

鉴别器可以根据某个字段的值，选择不同的映射规则。这在处理多态数据时非常有用。

### 场景：订单类型不同，返回不同的对象

```java
// 订单类型
public class Order { ... }

// VIP 订单
public class VipOrder extends Order { ... }

// 普通订单
public class NormalOrder extends Order { ... }
```

### 配置鉴别器

```xml
<resultMap id="orderResultMap" type="Order">
    <id property="id" column="id"/>
    <result property="orderNo" column="order_no"/>
    <result property="totalAmount" column="total_amount"/>

    <!-- 鉴别器：根据 order_type 字段选择映射规则 -->
    <discriminator javaType="string" column="order_type">
        <!-- 值为 VIP 时，使用 vipOrderResultMap -->
        <case value="VIP" resultMap="vipOrderResultMap"/>

        <!-- 值为 NORMAL 时，使用 normalOrderResultMap -->
        <case value="NORMAL" resultMap="normalOrderResultMap"/>

        <!-- 默认情况，使用 baseResultMap -->
        <case value="DEFAULT" resultType="Order"/>
    </discriminator>
</resultMap>

<resultMap id="vipOrderResultMap" type="VipOrder" extends="orderResultMap">
    <result property="vipLevel" column="vip_level"/>
    <result property="discount" column="discount"/>
</resultMap>

<resultMap id="normalOrderResultMap" type="NormalOrder" extends="orderResultMap">
    <result property="shippingFee" column="shipping_fee"/>
</resultMap>
```

### 继承机制

`resultMap` 支持继承（`extends`），可以复用公共映射：

```xml
<!-- 父 resultMap -->
<resultMap id="baseResultMap" type="User">
    <id property="id" column="id"/>
    <result property="username" column="username"/>
    <result property="email" column="email"/>
</resultMap>

<!-- 子 resultMap 继承父 -->
<resultMap id="vipUserResultMap" type="VipUser" extends="baseResultMap">
    <result property="vipLevel" column="vip_level"/>
    <result property="expireDate" column="expire_date"/>
</resultMap>
```

## 延迟加载配置

### 全局配置

```xml
<settings>
    <!-- 开启延迟加载（默认 true） -->
    <setting name="lazyLoadingEnabled" value="true"/>

    <!-- aggressiveLazyLoading：false = 按需加载，true = 访问任一属性就加载全部 -->
    <setting name="aggressiveLazyLoading" value="false"/>
</settings>
```

### 局部配置

```xml
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>

    <!-- 强制延迟加载（即使全局配置 eager） -->
    <association property="department"
                 javaType="Department"
                 fetchType="lazy">
        <id property="id" column="dept_id"/>
        <result property="name" column="dept_name"/>
    </association>

    <!-- 强制立即加载 -->
    <collection property="orders"
                ofType="Order"
                fetchType="eager">
        <id property="id" column="order_id"/>
    </collection>
</resultMap>
```

## autoMappingBehavior 配置

```xml
<settings>
    <!--
        NONE：不自动映射
        PARTIAL：自动映射非嵌套的 resultMap（默认）
        FULL：自动映射所有
    -->
    <setting name="autoMappingBehavior" value="PARTIAL"/>
</settings>
```

### PARTIAL（默认）

```xml
<resultMap id="userResultMap" type="User">
    <!-- 显式映射 -->
    <id property="id" column="user_id"/>
    <result property="username" column="user_name"/>

    <!-- 自动映射：email 会自动映射（因为没有显式配置） -->
    <!-- 自动映射：created_at → createdAt（驼峰转换） -->
</resultMap>
```

### FULL

```xml
<resultMap id="userResultMap" type="User">
    <!-- 即使没有配置，也会自动映射所有字段 -->
    <id property="id" column="user_id"/>
</resultMap>
<!-- email、created_at 也会被自动映射 -->
```

**注意**：FULL 模式下，如果有多表 JOIN，需要小心字段名冲突。

## 常见问题

### Q1：字段名和属性名不一致怎么办？

**方案一**：使用 column + property 映射

```xml
<result property="userName" column="user_name"/>
```

**方案二**：开启驼峰映射

```xml
<settings>
    <setting name="mapUnderscoreToCamelCase" value="true"/>
</settings>
```

```xml
<!-- 数据库：user_name，Java：userName -->
<result property="userName" column="user_name"/>
```

### Q2：association 和 collection 怎么选？

| 关系 | 使用标签 |
|------|---------|
| 一对一（一个用户有一个部门） | `<association>` |
| 一对多（一个用户有多个订单） | `<collection>` |

### Q3：N+1 问题怎么解决？

**最佳方案**：使用嵌套 resultMap + JOIN

```xml
<collection property="orders" ofType="Order">
    <id property="id" column="order_id"/>
    <result property="orderNo" column="order_no"/>
</collection>
```

避免使用嵌套 select：

```xml
<!-- 不要这样写 -->
<collection property="orders" column="id" select="selectOrdersByUserId"/>
```

## 总结

### resultMap 元素一览

| 元素 | 作用 | 关键属性 |
|------|------|---------|
| `<id>` | 主键映射 | property, column |
| `<result>` | 普通字段映射 | property, column |
| `<association>` | 一对一关联 | property, javaType, column, select |
| `<collection>` | 一对多关联 | property, ofType, column, select |
| `<constructor>` | 构造器注入 | - |
| `<discriminator>` | 鉴别器 | javaType, column |

### 选择建议

| 场景 | 推荐方式 |
|------|---------|
| 单表查询，字段名一致 | `resultType` |
| 单表查询，字段名不一致 | `resultType` + 驼峰映射 |
| 一对一关联 | `association` + 嵌套 resultMap |
| 一对多关联 | `collection` + 嵌套 resultMap |
| 多态数据 | `discriminator` |

理解 resultMap 的用法，你就能应对任何复杂的结果映射场景。配合动态 SQL，MyBatis 的映射能力可以说是无限强大。
