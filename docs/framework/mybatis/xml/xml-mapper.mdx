# MyBatis XML mapper：结构决定一切

MyBatis XML mapper 文件是整个框架的核心阵地。

所有的 SQL 语句、结果映射、缓存配置都在这里定义。一个结构良好的 mapper 文件，不仅让代码更易维护，还能充分发挥 MyBatis 的能力。

这一讲，我们深入解析 XML mapper 的每一个元素。

## 文件结构总览

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<!-- 根元素：namespace 是关键 -->
<mapper namespace="com.example.mapper.UserMapper">

    <!-- 1. 缓存配置 -->
    <cache
        eviction="LRU"
        flushInterval="60000"
        size="512"
        readOnly="true"/>

    <!-- 2. SQL 片段（可复用） -->
    <sql id="baseColumns">
        id, username, email, status, created_at, updated_at
    </sql>

    <!-- 3. 参数映射（已过时，不推荐） -->
    <parameterMap id="userParamMap" type="com.example.entity.User">
        <parameter property="id" jdbcType="BIGINT"/>
    </parameterMap>

    <!-- 4. 结果映射 -->
    <resultMap id="userResultMap" type="com.example.entity.User">
        <id property="id" column="id"/>
        <result property="username" column="username"/>
        <result property="email" column="email"/>
    </resultMap>

    <!-- 5. 查询语句 -->
    <select id="selectById" resultMap="userResultMap">
        SELECT <include refid="baseColumns"/>
        FROM user
        WHERE id = #{id}
    </select>

    <!-- 6. 插入语句 -->
    <insert id="insert" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO user (username, email)
        VALUES (#{username}, #{email})
    </insert>

    <!-- 7. 更新语句 -->
    <update id="update">
        UPDATE user
        SET username = #{username}, email = #{email}
        WHERE id = #{id}
    </update>

    <!-- 8. 删除语句 -->
    <delete id="delete">
        DELETE FROM user WHERE id = #{id}
    </delete>

</mapper>
```

## `<mapper>` 根元素

### namespace 属性

`namespace` 是 mapper 文件的身份证，具有以下作用：

**第一：与 Java 接口绑定**

```xml
<mapper namespace="com.example.mapper.UserMapper">
```

```java
package com.example.mapper;

public interface UserMapper {
    User selectById(Long id);
    int insert(User user);
}
```

MyBatis 会自动将接口方法与 XML 中的 SQL 关联起来：

| 接口方法 | XML SQL |
|---------|---------|
| `User selectById(Long id)` | `<select id="selectById" ...>` |
| `int insert(User user)` | `<insert id="insert" ...>` |

**第二：防止 SQL id 冲突**

```xml
<!-- UserMapper.xml -->
<select id="selectById">...</select>

<!-- OrderMapper.xml -->
<select id="selectById">...</select>
```

两个 `selectById` 通过 namespace 区分，完整 ID 是：
- `com.example.mapper.UserMapper.selectById`
- `com.example.mapper.OrderMapper.selectById`

**第三：支持增删改自动刷新缓存**

执行增删改时，MyBatis 会自动清空对应 namespace 的二级缓存。

### 没有 namespace 的情况

```xml
<mapper>
    <select id="selectById">
        SELECT * FROM user WHERE id = #{id}
    </select>
</mapper>
```

这种情况下，SQL id 只有 `selectById`，调用时直接写：

```java
sqlSession.selectOne("selectById", 1L);
```

**不推荐**：失去了接口绑定的类型安全优势。

## 四种 SQL 标签详解

### `<select>`

```xml
<select
    id="selectById"                           <!-- 唯一标识（必须） -->
    parameterType="Long"                      <!-- 参数类型（可选） -->
    resultType="com.example.User"            <!-- 返回类型（resultType/resultMap 二选一） -->
    resultMap="userResultMap"                 <!-- 结果映射 -->
    flushCache="false"                        <!-- 是否刷新缓存，默认 false -->
    useCache="true"                           <!-- 是否使用二级缓存，默认 true -->
    timeout="30000"                           <!-- 超时时间（毫秒） -->
    fetchSize="100"                           <!-- JDBC fetchSize -->
    statementType="PREPARED"                  <!-- STATEMENT/PREPARED/CALLABLE -->
    resultSetType="DEFAULT"                   <!-- DEFAULT/FORWARD_ONLY/SCROLL_SENSITIVE... -->
    databaseId="mysql">                       <!-- 数据库类型标识 -->
    SELECT * FROM user WHERE id = #{id}
</select>
```

**示例：复杂查询**

```xml
<select id="searchUsers" resultMap="userResultMap">
    SELECT
        u.id,
        u.username,
        u.email,
        d.name AS dept_name,
        COUNT(o.id) AS order_count
    FROM user u
    LEFT JOIN department d ON u.dept_id = d.id
    LEFT JOIN `order` o ON u.id = o.user_id
    WHERE u.status = #{status}
    GROUP BY u.id, u.username, u.email, d.name
    HAVING COUNT(o.id) >= #{minOrderCount}
    ORDER BY u.created_at DESC
    LIMIT #{offset}, #{pageSize}
</select>
```

### `<insert>`

```xml
<insert
    id="insert"
    parameterType="com.example.User"
    flushCache="true"
    statementType="PREPARED"
    useGeneratedKeys="false"                 <!-- 是否使用自增主键 -->
    keyProperty=""                           <!-- 主键对应的属性名 -->
    keyColumn=""                             <!-- 主键对应的列名 -->
    databaseId="mysql">
    INSERT INTO user (username, email)
    VALUES (#{username}, #{email})
</insert>
```

**useGeneratedKeys 示例**：

```xml
<!-- MySQL 自增主键 -->
<insert id="insert" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO user (username, email)
    VALUES (#{username}, #{email})
</insert>
```

```java
User user = new User();
user.setUsername("张三");
user.setEmail("zhangsan@example.com");
mapper.insert(user);
System.out.println(user.getId());  // 自动填充了自增主键！
```

**selectKey 示例**：

```xml
<!-- Oracle 使用序列 -->
<insert id="insert">
    <selectKey keyProperty="id" resultType="Long" order="BEFORE">
        SELECT user_seq.NEXTVAL FROM DUAL
    </selectKey>
    INSERT INTO user (id, username, email)
    VALUES (#{id}, #{username}, #{email})
</insert>
```

| order 属性 | 说明 |
|-----------|------|
| `BEFORE` | 先执行 selectKey，再执行 INSERT |
| `AFTER` | 先执行 INSERT，再执行 selectKey（用于获取自增 ID） |

### `<update>`

```xml
<update
    id="update"
    parameterType="com.example.User"
    flushCache="true"
    statementType="PREPARED"
    timeout="30000">
    UPDATE user
    SET username = #{username}, email = #{email}
    WHERE id = #{id}
</update>
```

**返回值的含义**：

```java
int rows = mapper.update(user);
if (rows == 0) {
    // 没有更新任何记录（可能 ID 不存在）
} else if (rows == 1) {
    // 成功更新 1 条记录
} else {
    // 更新了多条记录（通常不应该发生，除非 WHERE 条件有问题）
}
```

### `<delete>`

```xml
<delete
    id="delete"
    parameterType="Long"
    flushCache="true"
    statementType="PREPARED"
    timeout="30000">
    DELETE FROM user WHERE id = #{id}
</delete>
```

## `<sql>` 可复用片段

### 基本用法

```xml
<sql id="userColumns">
    id, username, email, status, created_at, updated_at
</sql>

<select id="selectAll" resultType="com.example.User">
    SELECT <include refid="userColumns"/>
    FROM user
</select>
```

### 变量替换

```xml
<sql id="userColumns">
    ${alias}.id, ${alias}.username, ${alias}.email
</sql>

<select id="selectAll" resultType="com.example.User">
    SELECT <include refid="userColumns">
        <property name="alias" value="u"/>
    </include>
    FROM user u
</select>
```

**生成的 SQL**：

```sql
SELECT u.id, u.username, u.email FROM user u
```

### 引用其他 sql 片段

```xml
<sql id="selectColumns">
    id, username, email
</sql>

<sql id="fullColumns">
    <include refid="selectColumns"/>, status, created_at
</sql>
```

## `<cache>` 缓存配置

### 完整配置

```xml
<cache
    eviction="LRU"           <!-- 淘汰策略 -->
    flushInterval="60000"    <!-- 刷新间隔（毫秒） -->
    size="512"               <!-- 缓存容量 -->
    readOnly="true"          <!-- 是否只读 -->
    blocking="false"/>       <!-- 是否阻塞 -->
```

| 属性 | 说明 | 可选值 |
|------|------|--------|
| `eviction` | 淘汰策略 | `LRU`（默认）、`FIFO`、`SOFT`、`WEAK` |
| `flushInterval` | 刷新间隔 | 毫秒，默认不刷新 |
| `size` | 缓存容量 | 默认 1024 |
| `readOnly` | 是否只读 | `true`（只读，性能好）、`false`（可读写） |
| `blocking` | 是否阻塞 | `true`、`false`（默认） |

### 缓存实现类

```xml
<!-- 使用 Redis 缓存 -->
<cache type="org.mybatis.caches.redis.RedisCache"/>

<!-- 使用 Ehcache -->
<cache type="org.mybatis.caches.ehcache.EhcacheCache"/>
```

## `<parameterMap>` 参数映射

**警告**：`parameterMap` 是过时的 API，不推荐使用。

### 过时写法

```xml
<parameterMap id="userParamMap" type="com.example.User">
    <parameter property="id" jdbcType="BIGINT"/>
    <parameter property="username" jdbcType="VARCHAR"/>
    <parameter property="email" jdbcType="VARCHAR"/>
</parameterMap>

<insert id="insert" parameterMap="userParamMap">
    INSERT INTO user (id, username, email)
    VALUES (?, ?, ?)
</insert>
```

### 推荐写法

```xml
<insert id="insert" parameterType="com.example.User">
    INSERT INTO user (id, username, email)
    VALUES (#{id}, #{username}, #{email})
</insert>
```

## `<resultMap>` 结果映射

resultMap 是 MyBatis 最强大的特性：

```xml
<resultMap id="userResultMap" type="com.example.User">
    <!-- 主键映射 -->
    <id property="id" column="id"/>

    <!-- 普通字段映射 -->
    <result property="username" column="username"/>
    <result property="email" column="email"/>

    <!-- 一对一关联 -->
    <association property="department" javaType="com.example.Department">
        <id property="id" column="dept_id"/>
        <result property="name" column="dept_name"/>
    </association>

    <!-- 一对多关联 -->
    <collection property="orders" ofType="com.example.Order">
        <id property="id" column="order_id"/>
        <result property="orderNo" column="order_no"/>
    </collection>
</resultMap>
```

resultMap 的详细内容，将在下一章深入讲解。

## XML 校验：DTD

### DTD 的作用

```xml
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
```

DTD（Document Type Definition）定义了：

1. **允许的元素**：mapper、select、insert、update、delete 等
2. **元素的属性**：每个元素可以有什么属性
3. **嵌套规则**：哪些元素可以嵌套在哪些元素内
4. **枚举值**：属性可以取哪些值

### IDE 中的作用

现代 IDE（IntelliJ IDEA、Eclipse）会根据 DTD 提供：

- **智能提示**：输入 `<` 时显示可用的标签
- **语法检查**：错误的标签或属性会标红
- **文档查看**：查看标签的说明

## 文件组织最佳实践

### 目录结构

```
resources/
└── mapper/
    ├── user/
    │   ├── UserMapper.xml
    │   └── UserAccountMapper.xml
    ├── order/
    │   └── OrderMapper.xml
    └── product/
        └── ProductMapper.xml
```

### 配置关联

```xml
<!-- mybatis-config.xml -->
<mappers>
    <!-- 方式一：单个文件 -->
    <mapper resource="mapper/user/UserMapper.xml"/>

    <!-- 方式二：包扫描（推荐） -->
    <package name="mapper"/>
</mappers>
```

### 接口和 XML 放在一起

如果使用 Maven，可以将接口和 XML 放在同一目录：

```xml
<!-- pom.xml -->
<build>
    <resources>
        <resource>
            <directory>src/main/java</directory>
            <includes>
                <include>**/*.xml</include>
            </includes>
        </resource>
    </resources>
</build>
```

```
src/main/java/
└── com/example/mapper/
    ├── UserMapper.java
    └── UserMapper.xml  <!-- 放在一起 -->
```

这样就不需要在 mybatis-config.xml 中配置 mapper 路径了。

## 总结

### mapper 文件核心要素

| 元素 | 作用 | 必须 |
|------|------|------|
| `<mapper namespace="">` | 命名空间，与接口绑定 | 是 |
| `<select>` | 查询语句 | - |
| `<insert>` | 插入语句 | - |
| `<update>` | 更新语句 | - |
| `<delete>` | 删除语句 | - |
| `<resultMap>` | 结果映射 | 否 |
| `<sql>` | SQL 片段 | 否 |
| `<cache>` | 缓存配置 | 否 |

### 延伸思考

**Q1：XML 文件热加载需要配置什么？**

```xml
<!-- mybatis-config.xml -->
<settings>
    <setting name="mapperCacheFiles" value="false"/>
</settings>
```

Spring Boot 环境下，默认开启热加载。

**Q2：可以省略 resultType 吗？**

可以省略的情况：
- 使用 `resultMap`
- 返回 `void` 或基本类型

**Q3：多个 XML 可以有相同 namespace 吗？**

可以，但 id 必须唯一。MyBatis 会合并它们。
