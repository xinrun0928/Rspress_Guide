# MyBatis 动态 SQL：9 个标签的组合艺术

WHERE 条件不确定该怎么写 SQL？

业务开发中，最常见的问题就是：WHERE 条件不确定。有时候需要按姓名搜索，有时候需要按状态筛选，有时候需要按时间范围过滤……

MyBatis 的动态 SQL 就是为解决这类问题而生的。它提供了 9 个强大的标签，让你可以根据条件动态拼接 SQL。

## 动态 SQL 标签一览

| 标签 | 作用 | 关键词 |
|------|------|--------|
| `<if>` | 条件判断 | test 属性 |
| `<where>` | 智能 WHERE | 自动处理 AND/OR |
| `<set>` | 智能 SET | 自动处理逗号 |
| `<trim>` | 万能 trim | prefix、suffix、trim |
| `<choose>` | 分支选择 | when、otherwise |
| `<foreach>` | 循环遍历 | collection、item、open、close |
| `<bind>` | 变量绑定 | OGNL 表达式 |
| `<bind>` | SQL 片段 | include、refid |

## `<if>`：基础条件判断

### 基本用法

```xml
<select id="search" resultType="User">
    SELECT * FROM user
    WHERE 1=1
    <if test="name != null">
        AND name = #{name}
    </if>
    <if test="age != null">
        AND age = #{age}
    </if>
</select>
```

**注意**：`test` 属性使用 OGNL 表达式，必须符合 Java 语法。

### 常见判断条件

```xml
<!-- 不为空字符串 -->
<if test="name != null and name != ''">
    AND name = #{name}
</if>

<!-- 数值大于 0 -->
<if test="age != null and age > 0">
    AND age = #{age}
</if>

<!-- 集合不为空 -->
<if test="ids != null and ids.size > 0">
    AND id IN
    <foreach collection="ids" item="id" open="(" separator="," close=")">
        #{id}
    </foreach>
</if>
```

### 判断字符串长度

```xml
<!-- 判断字符串长度 -->
<if test="name != null and name.length > 0">
    AND name LIKE CONCAT('%', #{name}, '%')
</if>
```

## `<where>`：智能 WHERE

### 问题：第一个条件前的 AND

```xml
<!-- 问题写法：WHERE 1=1 -->
<select id="search" resultType="User">
    SELECT * FROM user
    WHERE 1=1
    <if test="name != null">
        AND name = #{name}
    </if>
    <if test="status != null">
        AND status = #{status}
    </if>
</select>
```

### 使用 `<where>` 解决

```xml
<select id="search" resultType="User">
    SELECT * FROM user
    <where>
        <if test="name != null">
            name = #{name}
        </if>
        <if test="status != null">
            AND status = #{status}
        </if>
        <if test="createdAfter != null">
            AND created_at >= #{createdAfter}
        </if>
    </where>
</select>
```

**`<where>` 的智能之处**：
- 如果 `<where>` 块内所有条件都不满足，生成的 SQL 不包含 WHERE
- 如果第一个条件成立，`WHERE` 关键字会自动添加
- 如果后续条件以 `AND`/`OR` 开头，会自动处理

### `<where>` 的内部处理逻辑

```java
// 伪代码展示 where 标签的内部处理
public class WhereSqlSource implements SqlSource {

    @Override
    public BoundSql getBoundSql(Object parameterObject) {
        // 1. 解析所有 <if> 标签
        // 2. 过滤掉不满足条件的部分
        // 3. 处理第一个 AND/OR
        // 4. 生成最终 SQL
    }
}
```

## `<set>`：智能 SET

### 问题：UPDATE 语句的逗号

```xml
<!-- 问题写法：可能产生多余的逗号 -->
<update id="update">
    UPDATE user
    SET
    <if test="name != null">name = #{name},</if>
    <if test="email != null">email = #{email},</if>
    <if test="status != null">status = #{status},</if>
</update>
```

如果只有 `name` 不为空，生成的 SQL 是：

```sql
UPDATE user SET name = '张三', WHERE id = 1  -- 语法错误！
```

### 使用 `<set>` 解决

```xml
<update id="update">
    UPDATE user
    <set>
        <if test="name != null">name = #{name},</if>
        <if test="email != null">email = #{email},</if>
        <if test="status != null">status = #{status},</if>
    </set>
    WHERE id = #{id}
</update>
```

**`<set>` 的智能之处**：
- 自动去掉最后一个多余的逗号
- 如果所有条件都不满足，`<set>` 不会生成任何内容

## `<trim>`：万能 trim

### 自定义前缀后缀处理

```xml
<!-- 等价于 <where> -->
<trim prefix="WHERE" prefixOverrides="AND |OR ">
    <if test="name != null">name = #{name}</if>
    <if test="status != null">AND status = #{status}</if>
</trim>

<!-- 等价于 <set> -->
<trim prefix="SET" suffixOverrides=",">
    <if test="name != null">name = #{name},</if>
    <if test="email != null">email = #{email},</if>
</trim>
```

| 属性 | 说明 |
|------|------|
| `prefix` | 添加到 SQL 片段开头的字符串 |
| `suffix` | 添加到 SQL 片段结尾的字符串 |
| `prefixOverrides` | 去掉 SQL 片段开头的指定字符串 |
| `suffixOverrides` | 去掉 SQL 片段结尾的指定字符串 |

### 复杂场景

```xml
<!-- 自定义搜索条件 -->
<select id="search" resultType="User">
    SELECT * FROM user
    <trim prefix="WHERE" prefixOverrides="AND |OR " suffixOverrides=" AND|OR">
        <if test="name != null">name = #{name}</if>
        <if test="status != null">AND status = #{status}</if>
        <if test="email != null">OR email = #{email}</if>
    </trim>
</select>
```

## `<choose>`：分支选择

### 类似 Java 的 switch-case

```xml
<select id="search" resultType="User">
    SELECT * FROM user
    <where>
        <choose>
            <when test="id != null">
                id = #{id}
            </when>
            <when test="name != null">
                name = #{name}
            </when>
            <when test="email != null">
                email LIKE CONCAT('%', #{email}, '%')
            </when>
            <otherwise>
                status = 1
            </otherwise>
        </choose>
    </where>
</select>
```

**执行逻辑**：
1. 从上到下判断 `when` 的 `test` 条件
2. 第一个满足条件的分支会被执行
3. 如果都不满足，执行 `otherwise`

## `<foreach>`：循环遍历

### 批量 IN 查询

```xml
<select id="selectByIds" resultType="User">
    SELECT * FROM user
    WHERE id IN
    <foreach collection="ids" item="id" open="(" separator="," close=")">
        #{id}
    </foreach>
</select>
```

**生成的 SQL**：

```sql
SELECT * FROM user WHERE id IN (1, 2, 3, 4, 5)
```

### 批量插入

```xml
<insert id="batchInsert">
    INSERT INTO user (name, email) VALUES
    <foreach collection="users" item="user" separator=",">
        (#{user.name}, #{user.email})
    </foreach>
</insert>
```

**生成的 SQL**：

```sql
INSERT INTO user (name, email) VALUES ('张三', 'zhangsan@example.com'),
                                       ('李四', 'lisi@example.com'),
                                       ('王五', 'wangwu@example.com')
```

### 批量更新

```xml
<update id="batchUpdate">
    <foreach collection="users" item="user" separator=";">
        UPDATE user
        SET name = #{user.name}
        WHERE id = #{user.id}
    </foreach>
</update>
```

**生成的 SQL**：

```sql
UPDATE user SET name = '张三' WHERE id = 1;
UPDATE user SET name = '李四' WHERE id = 2;
UPDATE user SET name = '王五' WHERE id = 3;
```

### 动态 UPDATE SET

```xml
<update id="dynamicUpdate">
    UPDATE user
    <trim prefix="SET" suffixOverrides=",">
        <foreach collection="params" item="param">
            <if test="param.value != null">
                ${param.key} = #{param.value},
            </if>
        </foreach>
    </trim>
    WHERE id = #{id}
</update>
```

## `<bind>`：变量绑定

### OGNL 表达式

```xml
<select id="search" resultType="User">
    <bind name="pattern" value="'%' + name + '%'"/>
    SELECT * FROM user
    WHERE name LIKE #{pattern}
</select>
```

**注意**：`${name}` 是字符串拼接，而 `#{name}` 是参数绑定。

### 多字段模糊搜索

```xml
<select id="search" resultType="User">
    <bind name="namePattern" value="'%' + name + '%'"/>
    <bind name="emailPattern" value="'%' + email + '%'"/>
    SELECT * FROM user
    <where>
        <if test="name != null">
            name LIKE #{namePattern}
        </if>
        <if test="email != null">
            OR email LIKE #{emailPattern}
        </if>
    </where>
</select>
```

### 使用正则表达式

```xml
<select id="search" resultType="User">
    <bind name="emailPattern" value="'[a-z]+@[a-z]+\\.com'"/>
    SELECT * FROM user
    WHERE REGEXP(email, #{emailPattern})
</select>
```

## `<sql>` 和 `<include>`：SQL 片段复用

### 定义 SQL 片段

```xml
<sql id="userColumns">
    id, username, email, status, created_at
</sql>

<sql id="whereClause">
    <where>
        <if test="status != null">
            status = #{status}
        </if>
        <if test="keyword != null">
            AND (username LIKE #{keyword} OR email LIKE #{keyword})
        </if>
    </where>
</sql>
```

### 引用 SQL 片段

```xml
<select id="selectAll" resultType="User">
    SELECT <include refid="userColumns"/>
    FROM user
    <include refid="whereClause"/>
</select>

<select id="count" resultType="int">
    SELECT COUNT(*) FROM user
    <include refid="whereClause"/>
</select>
```

### 变量替换

```xml
<sql id="selectColumns">
    ${alias}.id, ${alias}.username, ${alias}.email
</sql>

<select id="selectAll" resultType="User">
    SELECT <include refid="selectColumns">
        <property name="alias" value="u"/>
    </include>
    FROM user u
</select>
```

## 动态 SQL 源码解析

### SqlSource 接口

```java
public interface SqlSource {
    BoundSql getBoundSql(Object parameterObject);
}
```

### DynamicSqlSource

处理动态 SQL（包含 `<if>`、`<where>` 等标签）：

```java
public class DynamicSqlSource implements SqlSource {

    private final SqlNode rootSqlNode;

    @Override
    public BoundSql getBoundSql(Object parameterObject) {
        // 1. 创建 DynamicContext，存储参数
        DynamicContext context = new DynamicContext(parameterObject);

        // 2. 处理 SQL 树，生成 SQL 片段
        rootSqlNode.apply(context);

        // 3. 创建 SqlSourceBuilder 解析 ? 占位符
        SqlSource sqlSource = new SqlSourceBuilder(context.getSql())
            .parse(parameterObject.getClass());

        // 4. 返回最终的 BoundSql
        return sqlSource.getBoundSql(parameterObject);
    }
}
```

### MixedSqlSource

混合静态和动态 SQL：

```java
public class MixedSqlSource implements SqlSource {

    @Override
    public BoundSql getBoundSql(Object parameterObject) {
        // 根据是否包含动态标签，选择不同的处理方式
    }
}
```

## 完整示例：动态搜索

```xml
<select id="dynamicSearch" resultType="User">
    SELECT * FROM user
    <where>
        <!-- 精确匹配 -->
        <if test="id != null">
            AND id = #{id}
        </if>

        <!-- 模糊匹配 -->
        <if test="name != null and name != ''">
            AND name LIKE CONCAT('%', #{name}, '%')
        </if>

        <!-- 范围查询 -->
        <if test="minAge != null">
            AND age >= #{minAge}
        </if>
        <if test="maxAge != null">
            AND age &lt;= #{maxAge}
        </if>

        <!-- 枚举匹配 -->
        <if test="status != null">
            AND status = #{status}
        </if>

        <!-- 日期范围 -->
        <if test="startDate != null">
            AND created_at >= #{startDate}
        </if>
        <if test="endDate != null">
            AND created_at &lt;= #{endDate}
        </if>

        <!-- 集合查询 -->
        <if test="deptIds != null and deptIds.size > 0">
            AND dept_id IN
            <foreach collection="deptIds" item="deptId" open="(" separator="," close=")">
                #{deptId}
            </foreach>
        </if>
    </where>
    ORDER BY
    <choose>
        <when test="sortField == 'name'">
            name
        </when>
        <when test="sortField == 'createdAt'">
            created_at
        </when>
        <otherwise>
            id
        </otherwise>
    </choose>
    <choose>
        <when test="sortOrder == 'desc'">
            DESC
        </when>
        <otherwise>
            ASC
        </otherwise>
    </choose>
</select>
```

## #{} 和 ${} 在动态 SQL 中的处理

### #{} 预编译

```xml
<!-- #{} 会被处理成 ?，参数单独绑定 -->
<if test="name != null">
    AND name = #{name}
</if>

<!-- 生成的 SQL -->
AND name = ?  -- 参数：'张三'
```

### ${} 替换

```xml
<!-- ${} 直接替换，存在 SQL 注入风险 -->
<if test="column != null">
    ORDER BY ${column} ${sortOrder}
</if>

<!-- 生成的 SQL -->
ORDER BY name ASC  -- 直接拼接到 SQL
```

### 安全的排序字段

```xml
<!-- 方案一：白名单 -->
<choose>
    <when test="sortField == 'name'">name</when>
    <when test="sortField == 'createdAt'">created_at</when>
    <otherwise>id</otherwise>
</choose>

<!-- 方案二：使用 bind + OGNL -->
<bind name="safeColumn" value="'name'"/>
ORDER BY ${safeColumn}
```

## 总结

### 标签对比

| 标签 | 适用场景 | 注意事项 |
|------|---------|---------|
| `<if>` | 单条件判断 | test 属性使用 OGNL |
| `<where>` | WHERE 子句 | 自动处理 AND/OR |
| `<set>` | UPDATE SET | 自动去掉多余逗号 |
| `<trim>` | 自定义前后缀 | 功能最灵活 |
| `<choose>` | 分支选择 | 类似 switch-case |
| `<foreach>` | 循环操作 | 批量 IN、批量插入 |
| `<bind>` | OGNL 表达式 | 创建临时变量 |

### 使用建议

1. **优先使用 `<where>` 和 `<set>`**，它们会自动处理边界情况
2. **`${}` 只在必要时使用**，如动态表名、排序字段
3. **`#{}` 用于参数绑定**，可以防 SQL 注入
4. **使用 `<sql>` 复用公共片段**，保持代码整洁

掌握动态 SQL 的组合使用，你就能应对任何复杂的查询场景。MyBatis 的动态 SQL 虽不如 Hibernate 的 Criteria API 那样面向对象，但在 SQL 控制力和可读性上更胜一筹。
