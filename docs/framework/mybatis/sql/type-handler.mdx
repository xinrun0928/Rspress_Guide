# TypeHandler：MyBatis 的类型转换器

<Badge text="MyBatis" type="tip" /> <Badge text="核心原理" type="info" />

Java 类型和 JDBC 类型之间的转换，全靠 TypeHandler。

当你在 Mapper 中写 `#{user.birthDate}` 时，有没有想过这个 `java.util.Date` 是怎么变成数据库的 `DATE` 类型的？反过来，数据库返回的 `DATE` 又怎么变回 `java.util.Date`？

这一切都是 TypeHandler 的功劳。

## 为什么需要 TypeHandler？

JDBC 和 Java 有各自独立的类型系统：

| JDBC 类型 | Java 类型 |
|-----------|-----------|
| VARCHAR | String |
| INTEGER | int / Integer |
| DATE | java.sql.Date |
| TIMESTAMP | java.sql.Timestamp |
| BOOLEAN | boolean / Boolean |
| BLOB | byte[] / Blob |
| CLOB | String / Clob |

虽然名字相似，但**它们是不同的类型**。TypeHandler 就是负责这两种类型之间互相转换的桥梁。

```java
// JDBC 的世界
java.sql.Date

// Java 的世界
java.util.Date
```

## TypeHandler 接口

```java
public interface TypeHandler<T> {

    // 设置参数（Java → JDBC）
    void setParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) throws SQLException;

    // 获取结果（JDBC → Java）
    T getResult(ResultSet rs, String columnName) throws SQLException;

    // 获取结果（JDBC → Java，通过列索引）
    T getResult(ResultSet rs, int columnIndex) throws SQLException;

    // 获取 CallableStatement 的输出参数
    T getResult(CallableStatement cs, int columnIndex) throws SQLException;
}
```

这是一个**泛型接口**，`T` 表示 Java 类型。MyBatis 为每种常见的类型都提供了默认实现。

## EnumTypeHandler：枚举的序列化

MyBatis 提供了两种枚举处理器，适用于不同场景。

### EnumTypeHandler（默认）

使用枚举的**名称**进行转换：

```java
// 枚举定义
public enum Status {
    PENDING,    // 0
    ACTIVE,     // 1
    INACTIVE,   // 2
    DELETED     // 3
}

// 实体类
public class User {
    private Status status;
}
```

```xml
<!-- 存储时：ACTIVE → 'ACTIVE'（字符串） -->
<insert id="insert">
    INSERT INTO user (name, status) VALUES (#{name}, #{status})
</insert>

<!-- 查询时：'ACTIVE' → Status.ACTIVE -->
<select id="selectById" resultType="User">
    SELECT * FROM user WHERE id = #{id}
</select>
```

### EnumOrdinalTypeHandler

使用枚举的**序号**进行转换：

```xml
<!-- 存储时：ACTIVE → 1（数字） -->
<insert id="insert">
    INSERT INTO user (name, status) VALUES (#{name}, #{status, typeHandler=org.apache.ibatis.type.EnumOrdinalTypeHandler})
</insert>

<!-- 查询时：1 → Status.ACTIVE -->
```

### 自定义枚举处理器

```java
public class MyEnumTypeHandler<E extends Enum<E>> implements TypeHandler<E> {

    private Class<E> type;
    private E[] enums;

    @Override
    public void setParameter(PreparedStatement ps, int i, E parameter, JdbcType jdbcType) {
        // 存储枚举的 code 值（假设枚举有 getCode() 方法）
        ps.setInt(i, parameter.getCode());
    }

    @Override
    public E getResult(ResultSet rs, String columnName) throws SQLException {
        int code = rs.getInt(columnName);
        return CodeEnum.findByCode(type, code);
    }
}
```

## 自定义 TypeHandler：三步完成

### 第一步：实现 TypeHandler 接口

```java
@MappedTypes(String.class)        // 声明处理的 Java 类型
@MappedJdbcTypes(JdbcType.VARCHAR) // 声明处理的 JDBC 类型
public class MyStringTypeHandler implements TypeHandler<String> {

    @Override
    public void setParameter(PreparedStatement ps, int i, String parameter, JdbcType jdbcType)
            throws SQLException {
        ps.setString(i, parameter);
    }

    @Override
    public String getResult(ResultSet rs, String columnName) throws SQLException {
        return rs.getString(columnName);
    }

    @Override
    public String getResult(ResultSet rs, int columnIndex) throws SQLException {
        return rs.getString(columnIndex);
    }

    @Override
    public String getResult(CallableStatement cs, int columnIndex) throws SQLException {
        return cs.getString(columnIndex);
    }
}
```

### 第二步：注册 TypeHandler

```xml
<!-- 方式一：全包名注册 -->
<typeHandlers>
    <typeHandler handler="com.example.MyStringTypeHandler"/>
</typeHandlers>

<!-- 方式二：包扫描（推荐） -->
<typeHandlers>
    <package name="com.example.handler"/>
</typeHandlers>
```

### 第三步：在 XML 中使用

```xml
<insert id="insert">
    INSERT INTO user (name, status)
    VALUES (#{name, typeHandler=com.example.MyStringTypeHandler}, #{status})
</insert>

<!-- 或者在 resultMap 中指定 -->
<result column="name" property="name" typeHandler="com.example.MyStringTypeHandler"/>
```

## 实战：JSON 字段的 TypeHandler

在 MySQL 中存储 JSON 数据时，需要自定义 TypeHandler：

### 依赖

```xml
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.15.2</version>
</dependency>
```

### 实现 JacksonTypeHandler

```java
@MappedTypes(Object.class)
@MappedJdbcTypes(JdbcType.VARCHAR)
public class JacksonTypeHandler<T> implements TypeHandler<T> {

    private static final ObjectMapper objectMapper = new ObjectMapper();

    private Class<T> clazz;

    @Override
    public void setParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) {
        try {
            // Java 对象 → JSON 字符串
            String json = objectMapper.writeValueAsString(parameter);
            ps.setString(i, json);
        } catch (Exception e) {
            throw new RuntimeException("序列化失败", e);
        }
    }

    @Override
    public T getResult(ResultSet rs, String columnName) throws SQLException {
        try {
            String json = rs.getString(columnName);
            if (json == null) {
                return null;
            }
            // JSON 字符串 → Java 对象
            return objectMapper.readValue(json, clazz);
        } catch (Exception e) {
            throw new RuntimeException("反序列化失败", e);
        }
    }

    @Override
    public T getResult(ResultSet rs, int columnIndex) throws SQLException {
        try {
            String json = rs.getString(columnIndex);
            if (json == null) {
                return null;
            }
            return objectMapper.readValue(json, clazz);
        } catch (Exception e) {
            throw new RuntimeException("反序列化失败", e);
        }
    }

    @Override
    public T getResult(CallableStatement cs, int columnIndex) throws SQLException {
        try {
            String json = cs.getString(columnIndex);
            if (json == null) {
                return null;
            }
            return objectMapper.readValue(json, clazz);
        } catch (Exception e) {
            throw new RuntimeException("反序列化失败", e);
        }
    }
}
```

### 使用

```java
public class User {
    private Long id;
    private String name;

    // 扩展信息，存储为 JSON
    private Map<String, Object> extraInfo;
}
```

```xml
<resultMap id="userResultMap" type="User">
    <id property="id" column="id"/>
    <result property="name" column="name"/>
    <result property="extraInfo" column="extra_info"
            typeHandler="com.example.handler.JacksonTypeHandler"/>
</resultMap>
```

## TypeHandler 的注册机制

### TypeRegistry

MyBatis 使用 `TypeHandlerRegistry` 管理所有的 TypeHandler：

```java
public class TypeHandlerRegistry {

    // Java 类型 → TypeHandler
    private final Map<JdbcType, TypeHandler<?>> jdbcTypeHandlerMap = new EnumMap<>(JdbcType.class);

    // Java 类型 → TypeHandler（泛型）
    private final Map<Type, TypeHandler<?>> typeHandlerMap = new ConcurrentHashMap<>();

    // TypeReference：用于获取泛型类型
    private final Map<Type, TypeHandler<?>> allTypeHandlersMap = new HashMap<>();

    public TypeHandlerRegistry(Configuration configuration) {
        // 注册内置的 TypeHandler
        register(Boolean.class, new BooleanTypeHandler());
        register(String.class, new StringTypeHandler());
        // ...
    }

    // 根据 Java 类型获取 TypeHandler
    public <T> TypeHandler<T> getTypeHandler(Class<T> type) {
        return typeHandlerMap.get(type);
    }

    // 根据 JDBC 类型获取 TypeHandler
    public TypeHandler<?> getTypeHandler(JdbcType jdbcType) {
        return jdbcTypeHandlerMap.get(jdbcType);
    }
}
```

### 注册优先级

```java
// 1. 显式注册的 TypeHandler 优先级最高
register(String.class, new MyStringTypeHandler());

// 2. 泛型 TypeHandler 次之
register(Long.class, new LongTypeHandler());

// 3. 内置 TypeHandler 最低
```

## TypeHandler 的查找流程

```java
public class DefaultParameterHandler {

    public void setParameters(PreparedStatement ps) {
        for (ParameterMapping parameterMapping : parameterMappings) {
            TypeHandler typeHandler = parameterMapping.getTypeHandler();

            if (typeHandler == null) {
                // 1. 尝试根据 Java 类型查找
                typeHandler = typeHandlerRegistry.getTypeHandler(parameterMapping.getJavaType());

                // 2. 尝试根据 JDBC 类型查找
                if (typeHandler == null) {
                    typeHandler = typeHandlerRegistry.getTypeHandler(parameterMapping.getJdbcType());
                }
            }

            // 设置参数
            typeHandler.setParameter(ps, position, value, parameterMapping.getJdbcType());
        }
    }
}
```

## 总结：TypeHandler 工作流程

```
参数设置（Java → JDBC）
┌──────────────────────────────────────────────────────────────┐
│ User.userType = UserType.ACTIVE                              │
│                              ↓                                │
│ EnumTypeHandler.setParameter()                               │
│                              ↓                                │
│ active.name() = 'ACTIVE'                                     │
│                              ↓                                │
│ ps.setString(1, 'ACTIVE')                                    │
│                              ↓                                │
│ 数据库存储：'ACTIVE'                                          │
└──────────────────────────────────────────────────────────────┘

结果映射（JDBC → Java）
┌──────────────────────────────────────────────────────────────┐
│ 数据库返回：'ACTIVE'                                          │
│                              ↓                                │
│ rs.getString("user_type") = 'ACTIVE'                         │
│                              ↓                                │
│ EnumTypeHandler.getResult()                                  │
│                              ↓                                │
│ Status.valueOf("ACTIVE") = Status.ACTIVE                     │
│                              ↓                                │
│ User.userType = UserType.ACTIVE                              │
└──────────────────────────────────────────────────────────────┘
```

## 延伸思考

### 为什么需要 TypeHandlerRegistry？

如果不统一管理，每个查询都需要知道使用哪个 TypeHandler。统一管理后：

1. **配置集中**：所有类型转换规则在一处配置
2. **可扩展**：可以注册自定义 TypeHandler
3. **优先级明确**：显式注册 > 内置类型

### 你可能想问的问题

**Q1：TypeHandler 在什么时候被调用？**

- **参数设置阶段**：`PreparedStatement.setXxx()` 之前
- **结果映射阶段**：`ResultSet.getXxx()` 之后

**Q2：如何让所有枚举使用同一个 TypeHandler？**

可以创建一个通用的枚举处理器，根据枚举值查找：

```java
public class GenericEnumTypeHandler<E extends Enum<E>> extends BaseTypeHandler<E> {

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, E parameter, JdbcType jdbcType) {
        // 使用枚举的 code 或 name
        ps.setString(i, parameter.name());
    }

    @Override
    public E getNullableResult(ResultSet rs, String columnName) {
        String value = rs.getString(columnName);
        return Enum.valueOf((Class<E>) type, value);
    }
}
```

**Q3：JSON 字段在不同数据库中如何处理？**

- **MySQL**：使用 `VARCHAR` 或 `JSON` 类型 + 自定义 TypeHandler
- **PostgreSQL**：使用 `JSONB` 类型 + 自定义 TypeHandler
- **Oracle**：使用 `CLOB` 类型 + 自定义 TypeHandler

理解 TypeHandler 的设计，你就能在处理特殊数据类型时游刃有余。JSON、枚举、自定义对象……只要有 TypeHandler，就没有处理不了的类型转换。
