## 问题一：#{} 和 ${} 的区别

这是 MyBatis 最常考的题目，没有之一。

| 特性 | `#{}` | `${}` |
|---|---|---|
| 预编译 | 是，参数绑定 | 否，字符串替换 |
| SQL 注入 | 安全 | 不安全 |
| 类型处理 | TypeHandler 转换 | 无 |
| NULL 值 | 自动处理 | 需要手动处理 |
| 使用场景 | 大多数场景 | 动态表名/列名 |

**最佳回答**：

> `#{}` 使用 PreparedStatement 的参数绑定机制，参数会被 `setString()` 等方法设置，SQL 语句先预编译，参数后绑定，因此可以防止 SQL 注入攻击。
>
> `${}` 是直接的字符串替换，参数直接拼接到 SQL 中，不经过预编译，因此有 SQL 注入风险。只适合动态表名、动态列名等无法使用参数绑定的场景。

**追问**：什么场景必须用 `${}`？

```xml
<!-- 动态表名 -->
SELECT * FROM ${tableName}

<!-- 动态列名 -->
SELECT ${columnName} FROM user

<!-- ORDER BY 动态列 -->
ORDER BY ${sortColumn} ${sortOrder}
```

## 问题二：MyBatis 执行 SQL 的流程

考察对 MyBatis 四大核心组件的理解。

**完整流程**：

```
SqlSession.selectList()
        ↓
Executor.query()
        ↓
CachingExecutor → 检查二级缓存
        ↓
SimpleExecutor/ReuseExecutor
        ↓
StatementHandler.prepare() → SQL 预编译
        ↓
ParameterHandler.setParameters() → 参数绑定
        ↓
StatementHandler.query() → 执行 SQL
        ↓
ResultSetHandler.handleResultSets() → 结果映射
        ↓
返回 Java 对象
```

**四大组件职责**：

| 组件 | 职责 |
|---|---|
| Executor | SQL 执行调度、缓存管理 |
| StatementHandler | 创建 Statement、SQL 预编译 |
| ParameterHandler | 将 Java 参数绑定到 PreparedStatement |
| ResultSetHandler | 将 ResultSet 转换为 Java 对象 |

## 问题三：一级缓存和二级缓存的区别

| 特性 | 一级缓存 | 二级缓存 |
|---|---|---|
| 作用域 | SqlSession | Mapper（跨 SqlSession） |
| 存储位置 | 内存（HashMap） | 可配置（内存、Redis 等） |
| 默认开启 | 是 | 否（需配置） |
| 失效时机 | SqlSession 关闭或 clearCache | 增删改操作、flushCache、flushInterval |
| 缓存共享 | 不共享 | 同一个 Mapper 共享 |

**追问**：二级缓存什么时候会失效？

1. 增删改操作会清空该 Mapper 的缓存
2. `flushCache="true"` 显式刷新
3. `flushInterval` 过期
4. 手动调用 `clearCache()`
5. 跨 Mapper 修改数据时不会自动清空（需要使用 `cache-ref`）

## 问题四：MyBatis 是如何找到 Mapper 接口对应的 SQL

考察对 JDK 动态代理的理解。

**完整流程**：

1. `getMapper()` 创建代理对象
2. 调用接口方法触发 `MapperProxy.invoke()`
3. 拼接 statementId = `namespace + "." + methodName`
4. 从 Configuration 的 `mappedStatements` 查找 MappedStatement
5. 调用 SqlSession 执行 SQL

```java
// namespace = "com.example.mapper.UserMapper"
// methodName = "selectById"
// statementId = "com.example.mapper.UserMapper.selectById"
```

**为什么需要 namespace？**

用于区分不同 Mapper 的同名方法。例如 UserMapper 和 OrderMapper 都可能有 `selectById` 方法，通过 namespace 区分。

## 问题五：嵌套查询的 N+1 问题

**什么是 N+1 问题？**

查询 N 个主对象时，额外执行 N 次关联查询。

**示例**：

```xml
<resultMap id="userResultMap" type="User">
    <collection property="orders" column="id"
               select="com.example.OrderMapper.selectByUserId"/>
</resultMap>
```

```java
List<User> users = userMapper.selectAll();
// 触发 1 + N 次 SQL（N = users.size()）
```

**解决方案**：

1. 使用嵌套结果，一次 JOIN 查询
2. 配置 `fetchType="eager"` 禁用懒加载
3. 使用 `cache-ref` 共享缓存
4. 手动分步查询，内存组装

## 问题七：动态 SQL 的原理

MyBatis 使用 **SqlNode** 体系处理动态 SQL：

```
SqlNode (接口)
├── StaticTextSqlNode      → 静态文本
├── TextSqlNode            → ${} 文本
├── IfSqlNode              → <if test="...">
├── WhereSqlNode           → <where>
├── SetSqlNode            → <set>
├── TrimSqlNode           → <trim>
├── ForEachSqlNode        → <foreach>
├── ChooseSqlNode         → <choose>
└── MixedSqlNode          → 组合多个 SqlNode
```

**if 标签的执行原理**：

```java
public class IfSqlNode implements SqlNode {
    @Override
    public boolean apply(DynamicContext context) {
        // 评估 test 条件
        if (evaluator.evaluateBoolean(test, context.getBindings())) {
            // 条件满足，执行子节点
            return contents.apply(context);
        }
        return true;
    }
}
```

## 问题九：MyBatis 延迟加载的原理

延迟加载（Lazy Loading）使得关联对象在真正访问时才加载。

**配置**：

```xml
<settings>
    <setting name="lazyLoadingEnabled" value="true"/>
</settings>
```

**原理**：

1. 嵌套查询使用 `select` 属性指定查询语句
2. 访问关联属性时触发延迟加载
3. MyBatis 生成代理对象，在访问属性时执行查询

**问题**：延迟加载可能产生 N+1 问题。

## 问题十：MyBatis 插件机制

MyBatis 允许插件拦截四大组件的方法：

```java
@Intercepts({
    @Signature(type = Executor.class, method = "query",
               args = {MappedStatement.class, Object.class, RowBounds.class, ResultHandler.class})
})
public class MyPlugin implements Interceptor {
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        // 拦截逻辑
        return invocation.proceed();
    }
}
```

**可拦截的方法**：

| 组件 | 可拦截方法 |
|---|---|
| Executor | query、update、commit、rollback |
| StatementHandler | prepare、parameterize、query、update |
| ParameterHandler | getParameterObject、setParameters |
| ResultSetHandler | handleResultSets、handleOutputParameters |

## 问题十三：openSession() 和 getMapper() 的区别

| 方式 | 说明 | 适用场景 |
|---|---|---|
| `openSession()` | 获取 SqlSession，手动调用 API | 临时查询、批量操作 |
| `getMapper()` | 获取 Mapper 接口的代理对象 | 常规 CRUD |

```java
// 方式一：SqlSession
SqlSession session = sqlSessionFactory.openSession();
User user = session.selectOne("com.example.UserMapper.selectById", 1L);
session.close();

// 方式二：Mapper 代理（推荐）
UserMapper mapper = session.getMapper(UserMapper.class);
User user = mapper.selectById(1L);
```

## 问题十四：MyBatis 如何防止 SQL 注入

1. **使用 #{} 参数绑定**：PreparedStatement 的参数绑定机制
2. **不使用 ${}**：除非必要，避免字符串拼接
3. **使用 TypeHandler**：确保参数类型正确转换
4. **配置安全设置**：如 Oracle 的 `dbcp.jmxEnabled=false`

## 面试追问方向

### 追问一：MyBatis 的缓存会不会导致脏数据？

会。二级缓存是 Mapper 级别的，当多个 SqlSession 操作同一个 Mapper 时，一个 SqlSession 的修改不会通知其他 SqlSession，导致脏读。

**解决方案**：
- 使用 `flushCache="true"` 配置刷新
- 使用 `cache-ref` 引用其他 Mapper 的缓存
- 关闭缓存（`useCache="false"`）

### 追问二：MyBatis 的批量插入怎么做？

```java
try (SqlSession session = sqlSessionFactory.openSession(ExecutorType.BATCH)) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    for (User user : users) {
        mapper.insert(user);
    }
    session.commit();
}
```

### 追问三：MyBatis 的分页怎么做？

MyBatis 本身没有分页功能，需要：
1. 手写分页 SQL（使用 LIMIT/OFFSET）
2. 使用 RowBounds（内存分页，不推荐）
3. 使用 PageHelper 等分页插件
