# 字节码入门

Java 源代码需要经过编译才能被 JVM 执行，但 JVM 执行的并不是机器码，而是字节码（Bytecode）。字节码是一种中间表示，介于源代码和机器码之间，兼顾了可移植性和执行效率。理解字节码是深入理解 Java 运行机制的关键。

## 什么是字节码

<Badge>What is Bytecode</Badge>

字节码是由 Java 编译器（javac）生成的 .class 文件中包含的指令集。每条字节码指令是一个字节的操作码（opcode），后面可能跟着零到多个操作数（operand）。

```bash
# 查看类的字节码
javap -c MyClass.class

# 查看详细字节码
javap -v MyClass.class
```

```java
// 源代码
public class BytecodeDemo {
    public int add(int a, int b) {
        return a + b;
    }
}
```

编译后生成的字节码：

```
public int add(int, int);
    Code:
      Stack=2, Locals=3, Args_size=3
      0: iload_1        // 将局部变量 1 (参数 a) 压入操作数栈
      1: iload_2        // 将局部变量 2 (参数 b) 压入操作数栈
      2: iadd           // 弹出栈顶两个 int，相加，结果压回栈顶
      3: ireturn        // 返回栈顶的 int 值
```

## 字节码的结构

<Badge>Bytecode Structure</Badge>

一个 .class 文件包含以下部分：

**魔数（Magic Number）**：0xCAFEBABE，标识 Java class 文件。

**版本号**：minor version + major version，标识 class 文件格式版本。

**常量池（Constant Pool）**：存储字面量和符号引用。

**访问标志（Access Flags）**：public、final 等修饰符。

**类/接口信息**：类名、父类名、接口列表。

**字段表（Field Table）**：类的字段信息。

**方法表（Method Table）**：类的方法信息。

**属性表（Attribute Table）**：额外的元数据信息。

## 常用字节码指令

<Badge>Common Instructions</Badge>

### 局部变量指令

```java
iload_1   // 将局部变量 1（int 类型）压入栈
iload_2   // 将局部变量 2（int 类型）压入栈
aload_0   // 将局部变量 0（引用类型，通常是 this）压入栈
aload_1   // 将局部变量 1（引用类型）压入栈
```

### 操作数栈指令

```java
bipush 10    // 将 byte 常量 10 压入栈
sipush 100   // 将 short 常量 100 压入栈
iconst_0     // 将 int 常量 0 压入栈
aconst_null  // 将 null 压入栈
```

### 对象操作指令

```java
new ArrayList     // 在堆上分配新对象
dup               // 复制栈顶值
astore_1          // 将栈顶引用存入局部变量 1
getfield          // 获取对象字段
putfield          // 设置对象字段
```

### 方法调用指令

```java
invokevirtual   // 调用实例方法（虚方法调用）
invokespecial   // 调用构造方法、私有方法、父类方法
invokestatic    // 调用静态方法
invokeinterface // 调用接口方法
```

### 控制流指令

```java
ifeq 10          // 如果栈顶为 0，跳转到第 10 字节
if_icmplt 20    // 如果 int1 < int2，跳转到第 20 字节
goto 30         // 无条件跳转到第 30 字节
tableswitch     // switch 语句（稀疏 case）
lookupswitch    // switch 语句（密集 case）
```

### 算术指令

```java
iadd      // int 相加
isub      // int 相减
imul      // int 相乘
idiv      // int 相除
irem      // int 取模
```

## 字节码分析示例

<Badge>Bytecode Analysis Example</Badge>

```java
public class MethodCall {
    public void sayHello() {
        System.out.println("Hello");
    }
}
```

对应字节码：

```
public void sayHello();
    Code:
      0: getstatic     #2                  // 获取 System.out
      3: ldc           #3                  // 加载字符串常量 "Hello"
      5: invokevirtual #4                  // 调用 println
      8: return                             // 返回
```

## synchronized 的字节码

<Badge>Synchronized Bytecode</Badge>

```java
public class SynchronizedDemo {
    public synchronized void method() {
        // 同步方法
    }
}
```

对应字节码：

```
public synchronized void method();
    Code:
      0: return
      LocalVariableTable:
        Start  Slot  Slot Name
          0     1    this
```

`synchronized` 关键字在字节码层面的实现：

```java
public void method() {
    synchronized (this) {
        // 临界区
    }
}
```

```
monitorenter    // 进入同步块，获取 monitor
// 临界区代码
monitorexit     // 退出同步块，释放 monitor
```

## for 循环的字节码

<Badge>For Loop Bytecode</Badge>

```java
public class ForLoopDemo {
    public int sum(int[] arr) {
        int sum = 0;
        for (int i = 0; i < arr.length; i++) {
            sum += arr[i];
        }
        return sum;
    }
}
```

对应字节码：

```
public int sum(int[]);
    Code:
      0: iconst_0          // 常量 0
      1: istore_2          // sum = 0
      2: iconst_0          // 常量 0
      3: istore_3          // i = 0
      4: goto 15           // 跳转到循环条件判断
      7: aload_1           // 加载数组
      8: iload_3           // 加载 i
      9: iaload            // arr[i]
     10: iload_2           // 加载 sum
     11: iadd              // sum += arr[i]
     12: istore_2          // 保存 sum
     13: iinc 3, 1         // i++
     16: iload_3           // 加载 i
     17: aload_1           // 加载数组
     18: arraylength       // arr.length
     19: if_icmplt 7       // i < arr.length ? 跳转 : 继续
     22: iload_2           // 加载 sum
     23: ireturn           // 返回 sum
```

## 面试高频考点

**问题一：什么是字节码？** 字节码是 Java 源代码编译后生成的中间代码，由 JVM 执行，不可直接被 CPU 执行。

**问题二：字节码指令有哪些类别？** 局部变量操作指令、操作数栈指令、对象操作指令、方法调用指令、控制流指令等。

**问题三：什么是 class 文件的结构？** 包含魔数、版本号、常量池、访问标志、类/接口信息、字段表、方法表、属性表等部分。

**问题四：invokevirtual 和 invokestatic 的区别是什么？** invokevirtual 调用实例方法，需要动态分派；invokestatic 调用静态方法，不需要对象实例。

**问题五：synchronized 在字节码层面是如何实现的？** 实例方法通过 ACC_SYNCHRONIZED 标志实现；同步块通过 monitorenter 和 monitorexit 指令实现。

字节码是 Java 运行时的"机器语言"，理解它让我们能够"读懂"JVM 的想法。那么，你最想分析哪个 Java 特性的字节码实现？
