# JVM

你知道 OOM（OutOfMemoryError）有多少种吗？

Heap Space OOM、Metaspace OOM、Direct Buffer OOM、StackOverflowError、Unable to create new native thread——每一种都对应着不同的内存区域和不同的根因。理解它们，你才算真正理解了 JVM。

JVM（Java Virtual Machine）是 Java 能实现「一次编译，到处运行」的核心，也是深入理解 Java 语言的必经之路。从 .class 文件的加载，到堆内存的分配，到垃圾的回收，到字节码的执行——每一步都藏着大量知识点。

## 学习路线建议

```
第一阶段：JVM 架构 → 类加载 → 双亲委派（全局认知）
         ↓
第二阶段：运行时数据区 → 方法区 vs Metaspace ★（理解内存布局）
         ↓
第三阶段：GC 算法 → 各收集器 → CMS vs G1 vs ZGC ★（GC 全貌）
         ↓
第四阶段：OOM 排查 → JVM 工具 → 性能调优（工程实践）
         ↓
第五阶段：字节码 → JIT → 逃逸分析（深度理解执行过程）
```

## 面试核心公式

```
类加载顺序 = 加载 → 验证 → 准备 → 解析 → 初始化（双亲委派贯穿全程）
         ↓
JDK 7 vs JDK 8 = PermGen（固定大小）vs Metaspace（动态扩展）
         ↓
GC 算法 = 标记-清除（碎片）/ 复制（减半空间）/ 标记-整理（移动对象）
         ↓
GC 收集器 = Serial → ParNew/CMS → G1 → ZGC（演进即优化）
```
