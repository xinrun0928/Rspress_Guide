# MAT 内存分析工具实战

MAT（Memory Analyzer Tool）是 Eclipse 基金会开发的 Java 堆内存分析工具，是排查内存泄漏和内存占用问题的利器。它能够快速定位堆转储文件中内存占用最大的对象，分析对象引用链，找出内存泄漏的根源。

## MAT 的安装与使用

<Badge>MAT Installation</Badge>

### 安装 MAT

```bash
# 下载地址：https://eclipse.dev/mat/downloads.php
# 也可以通过 Eclipse IDE 安装 MAT 插件

# 如果使用命令行分析
java -jar mat-parser.jar heapdump.hprof
```

### 基本使用流程

```bash
# 1. 生成堆转储文件
jcmd <pid> GC.heap_dump /path/to/heapdump.hprof

# 2. 使用 MAT 打开堆转储文件
mat.sh heapdump.hprof

# 3. 选择分析报告类型
# - Leak Suspects Report：自动分析泄漏嫌疑
# - Component Report：按组件分析
# - Heap Dump Overview：堆概览
```

## MAT 的核心视图

<Badge>MAT Views</Badge>

### 1. Leak Suspects Report（泄漏嫌疑人报告）

这是最常用的视图，MAT 会自动分析最可能的内存泄漏点。

```
Leak Suspects Report 会显示：
1. 问题概述：总内存占用、泄漏嫌疑的内存量
2. 泄漏嫌疑人：可能导致内存泄漏的对象
3. 简要分析：问题的基本描述
```

### 2. Histogram（直方图）

直方图按类统计对象数量和内存占用。

```bash
# 在 MAT 中可以：
# - 按对象数量排序：找出数量最多的类
# - 按浅堆排序：找出占用内存最大的类
# - 按深堆排序：找出包含引用链最深的类

# 常用 OQL 查询
select * from java.util.HashMap where size > 1000
select * from java.lang.String where value.length > 100
select * from char[] where @length > 1000
```

### 3. Dominator Tree（支配树）

支配树显示对象之间的支配关系，可以快速找出持有大量内存的对象。

```
支配树的特性：
- 如果 A 支配 B，则 A 被回收时 B 也会被回收
- 支配树中直接父节点是直接持有该对象的 GC Root
- 通过支配树可以快速定位内存占用的根源
```

## 实战案例：缓存未清理导致的内存泄漏

<Badge>Case Study</Badge>

### 问题描述

某 Web 应用启动后内存持续增长，最终 OOM。

### 分析过程

```bash
# 1. 生成堆转储
jcmd <pid> GC.heap_dump /tmp/heapdump.hprof

# 2. 使用 MAT 打开
# 选择 Leak Suspects Report
```

### 泄漏分析

```
在 MAT 中发现的泄漏点：

1. java.util.HashMap
   - 对象数量：1
   - 浅堆大小：500MB
   - 深堆大小：800MB
   
2. 该 HashMap 被以下对象引用：
   - com.example.service.CacheService.cache (静态字段)
   
3. HashMap 包含 1000000 个 Entry
   - 每个 Entry 包含 key 和 value
   - key 是 String 类型，value 是 User 对象
```

### 定位问题代码

```java
// 问题代码
public class CacheService {
    private static Map<String, User> cache = new HashMap<>();
    
    public void addUser(String key, User user) {
        cache.put(key, user);  // 永远不清理
    }
}
```

### 解决方案

```java
// 方案一：使用 WeakHashMap
private static Map<String, User> cache = new WeakHashMap<>();

// 方案二：使用 LRU 缓存
private static Map<String, User> cache = 
    Collections.synchronizedMap(
        new LinkedHashMap<>(1000, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry eldest) {
                return size() > 10000;
            }
        }
    );

// 方案三：使用缓存框架
private static Cache<String, User> cache = 
    Caffeine.newBuilder()
        .maximumSize(10000)
        .expireAfterAccess(Duration.ofMinutes(30))
        .build();
```

## 常用 OQL 查询

<Badge>OQL Query</Badge>

MAT 支持 OQL（Object Query Language）查询，可以精确定位对象。

```sql
-- 查询大字符串
SELECT * FROM java.lang.String WHERE value.length > 1000

-- 查询大集合
SELECT * FROM java.util.HashMap WHERE table.length > 100

-- 查询特定包下的对象
SELECT * FROM "com.example.*" WHERE objectsize > 1048576

-- 查询包含特定字段的对象
SELECT * FROM java.util.ArrayList WHERE elementData.length > 100

-- 查询空集合
SELECT * FROM java.util.HashMap WHERE size = 0
```

## MAT 进阶技巧

<Badge>Advanced Tips</Badge>

### 计算对象保留集

通过支配树可以计算某个对象的保留集（Retained Set），即回收该对象时会被同时回收的所有对象。

### 比较两个堆转储

MAT 支持比较两个不同时间点的堆转储，找出内存增长的对象。

```bash
# 在 MAT 中选择 File -> Compare to Another Heap Dump
```

### 分析 Thread 信息

通过线程视图可以查看线程相关的对象占用，包括线程栈和线程本地变量。

## 面试高频考点

**问题一：MAT 的主要功能是什么？** 分析堆转储文件，定位内存泄漏点，计算对象引用链，显示支配树。

**问题二：Leak Suspects Report 是什么？** MAT 自动分析的最可能的内存泄漏嫌疑人，列出内存占用最大的对象和引用链。

**问题三：Dominator Tree 的作用是什么？** 显示对象之间的支配关系，帮助快速定位持有大量内存的对象的根源。

**问题四：Histogram 和 Dominator Tree 有什么区别？** Histogram 按类统计，Dominator Tree 按对象实例统计。Dominator Tree 更适合分析具体的内存占用根源。

**问题五：浅堆和深堆有什么区别？** 浅堆是对象本身占用的内存，深堆是对象及其引用链所有对象的总内存。

MAT 是排查内存问题的瑞士军刀，但工具只是手段，找到问题的根源才是目的。那么，你有什么排查内存问题的好方法吗？
