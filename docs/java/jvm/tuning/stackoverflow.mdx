# 虚拟机栈溢出排查

虚拟机栈溢出（StackOverflowError）是 Java 程序中另一个常见的错误。与 OOM 不同，StackOverflowError 通常是由程序本身的逻辑错误导致的，比如无限递归。理解栈溢出的成因和排查方法，对于编写健壮的代码至关重要。

## 栈溢出的成因

<Badge>StackOverflow Cause</Badge>

栈溢出发生的条件很简单：线程请求的栈深度超过了 JVM 允许的最大深度。在 Java 中，最典型的场景就是无限递归。

```java
public class StackOverflowDemo {
    public static void main(String[] args) {
        recursive();
    }
    
    private static void recursive() {
        // 递归调用没有终止条件
        recursive();  // 无限递归
    }
}
```

运行这个程序，很快就会看到：

```
Exception in thread "main" java.lang.StackOverflowError
    at StackOverflowDemo.recursive(StackOverflowDemo.java:10)
```

## 递归调用与栈帧

<Badge>Stack Frame</Badge>

理解栈溢出需要理解虚拟机栈的工作原理。每一次方法调用都会创建一个栈帧（Stack Frame），包含局部变量表、操作数栈、动态链接等信息。递归调用会在栈上累积大量栈帧。

```java
public class StackFrameUnderstanding {
    public static void main(String[] args) {
        // 每次调用 methodA，栈上增加一个栈帧
        methodA(1);
    }
    
    static void methodA(int depth) {
        System.out.println("进入方法 A，深度: " + depth);
        methodB(depth + 1);  // 调用 B
    }
    
    static void methodB(int depth) {
        System.out.println("进入方法 B，深度: " + depth);
        methodA(depth + 1);  // 又调用 A，递归！
    }
}
```

栈帧的大小取决于方法参数数量、局部变量数量等因素。可以通过 `-Xss` 参数调整栈大小：

```bash
# 默认栈大小（通常是 1MB）
java -Xss1m StackOverflowDemo

# 减小栈大小，更容易触发 StackOverflowError
java -Xss256k StackOverflowDemo

# 增大栈大小，可以支持更深的递归
java -Xss2m StackOverflowDemo
```

## 栈溢出的常见场景

<Badge>Common Scenarios</Badge>

**场景一：递归调用没有终止条件**

```java
public class RecursionWithoutTermination {
    // 错误：递归方法没有终止条件
    public static int sum(int n) {
        return n + sum(n - 1);  // 永远不会停止
    }
    
    // 正确：递归方法必须有终止条件
    public static int sumCorrect(int n) {
        if (n <= 1) {
            return 1;
        }
        return n + sumCorrect(n - 1);
    }
    
    // 或者使用循环，避免栈溢出
    public static int sumIterative(int n) {
        int result = 0;
        for (int i = 1; i <= n; i++) {
            result += i;
        }
        return result;
    }
}
```

**场景二：循环依赖导致间接递归**

```java
public class IndirectRecursion {
    public void methodA() {
        methodB();
    }
    
    public void methodB() {
        methodC();
    }
    
    public void methodC() {
        methodA();  // 间接递归
    }
}
```

**场景三：过深的调用链**

```java
public class DeepCallChain {
    public static void main(String[] args) {
        methodLevel1(1);
    }
    
    static void methodLevel1(int level) { methodLevel2(level + 1); }
    static void methodLevel2(int level) { methodLevel3(level + 1); }
    static void methodLevel3(int level) { methodLevel4(level + 1); }
    static void methodLevel4(int level) { methodLevel5(level + 1); }
    static void methodLevel5(int level) { methodLevel1(level + 1); } // 循环
    
    // 即使方法很简单，过深的调用链也会导致栈溢出
}
```

**场景四：大量局部变量**

```java
public class LargeLocalVariables {
    public static void main(String[] args) {
        largeMethod(1);
    }
    
    static void largeMethod(int level) {
        // 每个栈帧占用大量空间
        byte[] buffer1 = new byte[1024];
        byte[] buffer2 = new byte[1024];
        byte[] buffer3 = new byte[1024];
        // 大量局部变量会消耗更多栈空间
        
        largeMethod(level + 1);
    }
}
```

## 栈溢出的排查方法

<Badge>Troubleshooting</Badge>

### 方法一：分析堆栈信息

```bash
Exception in thread "main" java.lang.StackOverflowError
    at RecursiveDemo.sum(RecursiveDemo.java:5)
    at RecursiveDemo.sum(RecursiveDemo.java:5)
    at RecursiveDemo.sum(RecursiveDemo.java:5)
    ...

# 从堆栈信息可以看到：
# 1. 错误类型是 StackOverflowError
# 2. 发生在 RecursiveDemo.sum 方法
# 3. 方法自己调用自己（递归）
```

### 方法二：增加栈大小

```bash
# 如果业务确实需要深递归，增加栈大小
java -Xss2m -Xss4m MyApplication

# 但这只能延缓问题，不是根本解决方案
```

### 方法三：改用循环或尾递归

```java
public class TailRecursionOptimization {
    // 尾递归实现：递归调用在最后一行
    public static int factorial(int n, int acc) {
        if (n <= 1) {
            return acc;
        }
        return factorial(n - 1, n * acc);  // 尾递归
    }
    
    // 或者使用循环代替递归
    public static int factorialIterative(int n) {
        int result = 1;
        for (int i = 2; i <= n; i++) {
            result *= i;
        }
        return result;
    }
}
```

## 面试高频考点

**问题一：StackOverflowError 和 OutOfMemoryError 的区别是什么？** StackOverflowError 是栈空间耗尽，通常由无限递归导致；OutOfMemoryError 是堆内存耗尽，通常由内存泄漏或配置不足导致。

**问题二：如何排查 StackOverflowError？** 查看堆栈信息定位错误发生的方法；分析代码检查是否有无限递归；使用递归的方法改为循环实现；必要时增大栈大小。

**问题三：如何预防 StackOverflowError？** 避免无限递归、使用循环代替深度递归、合理配置栈大小、注意间接递归。
