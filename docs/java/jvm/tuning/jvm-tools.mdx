# JVM 诊断工具使用

JVM 提供了一系列命令行工具用于诊断和分析运行时问题。这些工具位于 JDK 的 bin 目录下，无需额外安装即可使用。熟练掌握这些工具，是排查生产环境问题的必备技能。

## jstat：GC 统计信息

<Badge>jstat</Badge>

jstat 是最常用的 GC 监控工具，可以实时查看堆内存使用、GC 频率、类加载统计等信息。

```bash
# 基本语法
jstat -<option> <pid> [interval] [count]

# 查看 GC 统计信息
jstat -gcutil <pid> 1000

# 输出解释：
# S0C S1C S0U S1U EC EU OC OU MC MU CCSC CCSU YGC YGCT FGC FGCT GCT
# S0C/S1C: S0/S1 当前容量（KB）
# S0U/S1U: S0/S1 已使用（KB）
# EC/EU: Eden 区容量/已使用（KB）
# OC/OU: Old 区容量/已使用（KB）
# MC/MU: Metaspace 容量/已使用（KB）
# YGC/YGCT: 年轻代 GC 次数/总时间（秒）
# FGC/FGCT: Full GC 次数/总时间（秒）
# GCT: GC 总时间（秒）
```

```bash
# 查看类加载统计
jstat -class <pid>

# 输出：
# Loaded Bytes  Used Bytes  Bytes  Long  Class Flag  Classes
#    12345      11234     1024     0      123    1      456
```

```bash
# 查看编译统计
jstat -compiler <pid>

# 查看 GC 历史（需要 -XX:+UseG1GC）
jstat -gcold <pid>
```

## jmap：内存映射与堆转储

<Badge>jmap</Badge>

jmap 用于生成堆转储文件、分析内存分配、查看 finalizer 队列等。

```bash
# 生成堆转储文件
jmap -dump:format=b,file=heapdump.hprof <pid>

# 查看堆使用摘要
jmap -heap <pid>

# 输出包括：
# - Heap Configuration（堆配置）
# - Heap Usage（堆使用情况）
# - Survivor Space（Survivor 区）
# - Old Generation（老年代）
# - Metaspace（元空间）
```

```bash
# 查看对象的统计信息
jmap -histo <pid> | head -30

# 输出：
#  num     #instances         #bytes  class name
# -----------------------------------------------
#    1:         12345       12345678  [B
#    2:          5678        2345678  java.lang.String
#    3:          3456        1234567  java.util.HashMap$Node
```

```bash
# 查看 finalizer 队列（可能导致 F-Queue 阻塞）
jmap -finalizerinfo <pid>
```

## jstack：线程堆栈分析

<Badge>jstack</Badge>

jstack 用于生成线程的堆栈跟踪信息，是分析 CPU 高占用、死锁等问题的首选工具。

```bash
# 生成线程堆栈
jstack <pid>

# 输出包括：
# - 每条线程的状态和堆栈信息
# - 锁信息（waiting on monitor, blocked by）
# - 死锁检测（Found one Java-level deadlock）
```

```bash
# 查看死锁
jstack -l <pid> | grep -A 20 "Found one"

# 查看热点线程
jstack <pid> | grep "tid=" | awk '{print $2}' | sort | uniq -c | sort -rn
```

```bash
# 实时监控线程（每秒输出一次）
watch -n 1 'jstack <pid>'
```

## jcmd：多功能诊断工具

<Badge>jcmd</Badge>

jcmd 是 JDK 8 引入的统一诊断工具，可以执行多种诊断命令。

```bash
# 列出可用的诊断命令
jcmd <pid> help

# 生成堆转储
jcmd <pid> GC.heap_dump /path/to/heapdump.hprof

# 查看 VM 标志
jcmd <pid> VM.flags

# 查看系统属性
jcmd <pid> VM.system_properties

# 查看 JMX 信息
jcmd <pid> VM.native_memory summary

# 查看 classloader 统计
jcmd <pid> GC.class_stats
```

## jinfo：运行时配置查看

<Badge>jinfo</Badge>

jinfo 用于查看和修改 JVM 的运行时配置。

```bash
# 查看 JVM 配置
jinfo <pid>

# 查看特定标志的值
jinfo -flag MaxHeapSize <pid>
jinfo -flag PrintGCDetails <pid>

# 动态修改标志（部分标志支持）
jinfo -flag +PrintGCDetails <pid>
jinfo -flag -UseG1GC <pid>
```

## jhsdb：调试工具

<Badge>jhsdb</Badge>

jhsdb（Java HotSpot Debugger）提供低级调试功能。

```bash
# 启动 clhsdb（命令行调试器）
jhsdb clhsdb

# 常用命令：
# help - 显示帮助
# universe - 显示堆配置
# runFinalizer - 运行 finalizer
# inspect - 检查对象
```

## 工具对比与选择

<Badge>Tool Comparison</Badge>

| 工具 | 主要用途 | 适用场景 |
|------|----------|----------|
| jstat | GC 监控 | 实时监控 GC 状态 |
| jmap | 堆转储 | 生成堆转储文件分析 |
| jstack | 线程分析 | 排查死锁、CPU 高占用 |
| jcmd | 综合诊断 | 统一入口，多种功能 |
| jinfo | 配置查看 | 查看/修改 JVM 参数 |

## 面试高频考点

**问题一：如何使用 jstat 查看 GC 情况？** 使用 `jstat -gcutil <pid>` 查看 GC 统计信息，包括各区使用率、GC 次数和总时间。

**问题二：如何使用 jmap 生成堆转储？** 使用 `jmap -dump:format=b,file=heapdump.hprof <pid>` 生成堆转储文件。

**问题三：如何使用 jstack 分析线程问题？** 使用 `jstack <pid>` 生成线程堆栈，用于定位死锁、CPU 高占用等问题。

**问题四：jcmd 有什么优势？** jcmd 是统一的诊断入口，可以执行多种诊断命令，包括堆转储、GC、类统计等，比单独的 jmap/jstack 更方便。

**问题五：这些工具在生产环境使用有什么注意事项？** jmap、jstack 可能导致长时间暂停，影响服务；建议在低峰期使用，或使用 -XX:+HeapDumpOnOutOfMemoryError 自动转储。
