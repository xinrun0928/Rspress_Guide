# JVM 参数配置与调优经验

JVM 参数配置是 Java 性能调优的核心内容之一。合理的参数配置可以让应用运行得更加稳定高效，而错误的配置可能导致性能下降甚至系统崩溃。本篇文章将系统性地介绍 JVM 常用参数及其调优经验。

## JVM 参数分类

<Badge>JVM Parameter Types</Badge>

JVM 参数可以分为三类：标准参数、非标准参数和实验性参数。

**标准参数**：以 `-` 开头，如 `-server`、`-client`，所有 JVM 实现都必须支持。

**非标准参数**：以 `-X` 开头，如 `-Xms512m`、`-Xmx2g`，JVM 实现可能支持但不做保证。

**实验性参数**：以 `-XX` 开头，如 `-XX:+UseG1GC`，主要用于 JVM 内部调试，可能在未来版本中变化。

## 堆内存参数

<Badge>Heap Parameters</Badge>

```bash
# 设置堆初始大小
-Xms512m

# 设置堆最大大小
-Xmx4g

# 推荐：将初始和最大设为相同，避免运行时调整开销
-Xms4g -Xmx4g

# 设置年轻代大小
-Xmn2g

# 设置年轻代/老年代比例
-XX:NewRatio=2
# NewRatio=2 表示 老年代:年轻代 = 2:1

# 设置 Eden/Survivor 比例
-XX:SurvivorRatio=8
# SurvivorRatio=8 表示 Eden:S0:S1 = 8:1:1
```

## GC 收集器参数

<Badge>GC Collector Parameters</Badge>

```bash
# 选择 Serial 收集器（单线程，适合小内存）
-XX:+UseSerialGC

# 选择 ParNew + CMS 组合（低停顿）
-XX:+UseParNewGC
-XX:+UseConcMarkSweepGC

# 选择 Parallel Scavenge + Parallel Old（高吞吐）
-XX:+UseParallelGC
-XX:+UseParallelOldGC

# 选择 G1（平衡型，JDK 9+ 默认）
-XX:+UseG1GC

# 选择 ZGC（极低停顿）
-XX:+UseZGC
-XX:MaxGCPauseMillis=10
```

## G1 调优参数

<Badge>G1 Tuning Parameters</Badge>

```bash
# 设置停顿时间目标（软约束）
-XX:MaxGCPauseMillis=200

# 设置 Region 大小（1MB-32MB）
-XX:G1HeapRegionSize=4m

# 设置触发 Mixed GC 的堆占用阈值
-XX:InitiatingHeapOccupancyPercent=45

# 设置 Mixed GC 目标
-XX:G1MixedGCLiveThresholdPercent=85
-XX:G1OldCSetRegionThresholdPercent=10
```

## GC 日志参数

<Badge>GC Log Parameters</Badge>

```bash
# JDK 8 及之前
-XX:+PrintGCDetails
-XX:+PrintGCDateStamps
-Xloggc:/var/log/gc.log

# JDK 9+ 统一日志
-Xlog:gc*=info:file=/var/log/gc.log:time,uptime,level,tags

# OOM 时生成堆转储
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/var/log/
```

## JVM 调优经验总结

<Badge>Tuning Experience</Badge>

### 经验一：避免动态调整堆大小

```bash
# 推荐：将初始和最大设为相同
-Xms4g -Xmx4g
```

动态调整堆大小会消耗 CPU 资源，且可能导致内存抖动。

### 经验二：根据应用特点选择 GC 策略

```bash
# 追求高吞吐（批处理）
-XX:+UseParallelGC
-XX:MaxGCPauseMillis=500
-XX:GCTimeRatio=19

# 追求低延迟（Web 服务）
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200

# 追求极致低延迟
-XX:+UseZGC
-XX:MaxGCPauseMillis=10
```

### 经验三：监控是关键

```bash
# 使用 jstat 监控
jstat -gcutil <pid> 1000

# 使用 arthas dashboard
arthas> dashboard
```

### 经验四：避免过度调优

JVM 的默认参数在大多数场景下已经足够好。调优应该基于实际监控数据，而不是猜测。

### 经验五：容器环境特殊考虑

```bash
# 容器环境建议
-Xms2g -Xmx2g
-XX:+UseContainerSupport
-XX:MaxRAMPercentage=75.0
```

## 调优实战配置示例

<Badge>Real World Examples</Badge>

### Web 应用配置

```bash
java -server \
     -Xms2g -Xmx2g \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -XX:+HeapDumpOnOutOfMemoryError \
     -XX:HeapDumpPath=/var/log/ \
     -Xlog:gc*:file=/var/log/gc.log:time,uptime,level \
     -jar myapp.jar
```

### 批处理应用配置

```bash
java -server \
     -Xms4g -Xmx4g \
     -Xmn2g \
     -XX:+UseParallelGC \
     -XX:ParallelGCThreads=8 \
     -XX:+UseAdaptiveSizePolicy \
     -Xlog:gc*:file=/var/log/gc.log \
     -jar myapp.jar
```

### 低延迟应用配置

```bash
java -server \
     -Xms8g -Xmx8g \
     -XX:+UseZGC \
     -XX:MaxGCPauseMillis=5 \
     -XX:+HeapDumpOnOutOfMemoryError \
     -Xlog:gc*:file=/var/log/gc.log:time,uptime \
     -jar myapp.jar
```

## 面试高频考点

**问题一：如何配置堆大小？** 使用 -Xms 和 -Xmx 参数，建议设为相同值避免动态调整开销。

**问题二：如何选择 GC 收集器？** 追求高吞吐用 Parallel，追求低延迟用 G1，追求极低延迟用 ZGC。

**问题三：G1 的核心参数有哪些？** MaxGCPauseMillis（停顿目标）、G1HeapRegionSize（Region 大小）、InitiatingHeapOccupancyPercent（触发阈值）。

**问题四：如何避免 GC 抖动？** 合理设置堆大小、避免创建过多短期对象、选择合适的 GC 收集器。

**问题五：如何监控 JVM 状态？** 使用 jstat、jvisualvm、arthas 等工具监控 GC、内存、线程等指标。

**问题六：容器环境中 JVM 参数有什么特殊考虑？** 需要考虑容器内存限制，使用 MaxRAMPercentage 或设置与容器限制相匹配的堆大小。
