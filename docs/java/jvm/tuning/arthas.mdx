# Arthas 诊断工具使用

Arthas（阿尔萨斯）是阿里巴巴开源的 Java 诊断工具，可以实时诊断 Java 应用的各种问题，包括类加载、方法执行、内存分析、线程分析等。相比 JDK 自带的命令行工具，Arthas 提供了更友好、更强大的交互界面，是生产环境问题排查的利器。

## Arthas 的安装与启动

<Badge>Arthas Installation</Badge>

```bash
# 方式一：下载 jar 包
curl -O https://arthas.aliyun.com/arthas-boot.jar
java -jar arthas-boot.jar

# 方式二：使用 as.sh（Linux/Mac）
curl -L https://arthas.aliyun.com/install.sh | sh
./as.sh

# 方式三：IDEA 插件
# 在 IDEA 插件市场搜索 Arthas
```

启动后会列出当前 Java 进程，选择要诊断的进程即可。

## Arthas 的核心命令

<Badge>Arthas Commands</Badge>

### 1. dashboard：实时仪表盘

```bash
# 查看应用实时状态
dashboard

# 显示内容：
# - 线程信息（ID、名称、状态、CPU 占用）
# - 内存信息（各区使用情况）
# - GC 信息（GC 次数、时间）
# - 运行环境信息
```

### 2. thread：线程分析

```bash
# 查看所有线程
thread

# 查看 CPU 占用最高的线程
thread -n 5

# 查看指定线程的堆栈
thread <tid>

# 查看阻塞状态的线程
thread -b

# 查看死锁
thread --state BLOCKED
```

### 3. jad：反编译类

```bash
# 反编译指定类
jad com.example.MyService

# 反编译并保存
jad com.example.MyService -d /tmp/output

# 反编译指定方法
jad com.example.MyService getUser
```

### 4. watch：方法调用监控

```bash
# 监控方法入参和返回值
watch com.example.MyService getUser '{params, returnObj}'

# 监控方法执行时间
watch com.example.MyService getUser '{params, returnObj, throwExp}' -x 3

# 监控指定异常
watch com.example.MyService * '{params, throwExp}' -e
```

### 5. trace：方法调用链路

```bash
# 追踪方法调用耗时
trace com.example.MyService getUser

# 追踪调用链路
trace com.example.MyService *

# 只显示耗时超过 100ms 的调用
trace com.example.MyService * '#cost > 100'
```

### 6. monitor：方法调用统计

```bash
# 统计方法调用次数和平均耗时
monitor -c 5 com.example.MyService getUser

# 输出：
# timestamp         class           method     total  success  fail  avg-rt  fail-rate
# 2024-01-01       MyService       getUser   1000   990     10    12ms    1.00%
```

### 7. stack：方法调用堆栈

```bash
# 查看方法调用堆栈
stack com.example.MyService getUser

# 查看深度堆栈
stack com.example.MyService getUser -n 5
```

### 8. sc：类搜索

```bash
# 搜索已加载的类
sc com.example.*

# 搜索类的方法
sc -d com.example.MyService

# 搜索所有包含特定字符串的类
sc *Controller
```

### 9. sm：方法搜索

```bash
# 查看类的方法
sm com.example.MyService

# 查看方法的详细信息
sm -d com.example.MyService getUser
```

### 10. heapdump：堆转储

```bash
# 生成堆转储
heapdump

# 生成指定路径的堆转储
heapdump /tmp/heapdump.hprof

# 只 dump 存活对象
heapdump --live /tmp/heapdump_live.hprof
```

## Arthas 实战案例

<Badge>Arthas Case Study</Badge>

### 案例一：排查接口响应慢

```bash
# 1. 找到慢接口
trace com.example.* * '#cost > 1000'

# 2. 发现 getUser 方法耗时 2s
trace com.example.MyService getUser

# 3. 查看方法入参
watch com.example.MyService getUser '{params, returnObj}'

# 4. 发现是数据库查询慢
# 解决方案：添加索引或优化 SQL
```

### 案例二：排查内存泄漏

```bash
# 1. 查看内存使用
dashboard

# 2. 查看对象数量变化
heapdump /tmp/heap.hprof

# 3. 在 MAT 中分析发现 HashMap 持续增长

# 4. 查看 HashMap 的操作
watch com.example.CacheService * '{params, returnObj}'
```

### 案例三：排查 CPU 高占用

```bash
# 1. 查看 CPU 占用最高的线程
thread -n 3

# 2. 查看线程堆栈
thread <tid>

# 3. 发现是某个方法的循环导致
# 4. 使用 trace 追踪具体方法
trace com.example.*.* '#cost > 500'
```

## Arthas 的高级用法

<Badge>Advanced Usage</Badge>

### 使用 Profiler 生成火焰图

```bash
# 启动 profiler
profiler start

# 运行一段时间后获取结果
profiler stop --format html

# 查看火焰图
# 输出 HTML 文件，可以用浏览器打开
```

### 热更新代码

```bash
# 重新定义类（热更新）
redefine /tmp/MyService.class

# 注意：热更新有限制，不能修改方法签名
```

### 多进程诊断

```bash
# 连接到指定进程
arthas -c "session 'target-pid'"

# 同时监控多个进程
arthas -c "multi-instance"
```

## 面试高频考点

**问题一：Arthas 有什么特点？** 交互式诊断工具，无需重启应用即可诊断，提供 dashboard、trace、watch、jad 等强大功能。

**问题二：watch 和 trace 命令有什么区别？** watch 监控方法的入参、返回值和异常；trace 追踪方法的调用链路和耗时。

**问题三：如何在生产环境使用 Arthas？** 可以通过 jvm attach 方式连接正在运行的应用，对业务零影响。建议在低峰期使用密集监控命令。

**问题四：Arthas 的 profiler 命令有什么作用？** 生成火焰图，直观显示 CPU 时间消耗在各方法中的分布。

**问题五：Arthas 支持热更新吗？** 支持，使用 redefine 命令，但有限制，不能修改方法签名。

Arthas 让生产环境的诊断变得简单，但工具终究只是手段，更重要的是理解问题的本质。那么，你遇到过哪些 Arthas 解决不了的问题？
