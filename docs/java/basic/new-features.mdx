# Java 新特性演进：从 JDK 8 到 JDK 21

Java 自 1995 年诞生以来，已经走过了近三十年的历程。在这段时间里，Java 从一个简单的面向对象语言，发展成为一个功能丰富、生态庞大的企业级开发平台。每一次 JDK 版本的更新，都为 Java 注入了新的活力。理解这些新特性，不仅能帮助你写出更简洁、更高效的代码，也能让你跟上 Java 生态的发展趋势。

让我们从 JDK 8 开始，回顾 Java 最重要的新特性。JDK 8 可能是 Java 历史上最重要的一次更新，它引入的 Lambda 表达式和 Stream API 彻底改变了 Java 的编程风格。

## JDK 8：Lambda 表达式与 Stream API

### Lambda 表达式：函数式编程的入口

Lambda 表达式是 JDK 8 最引人注目的新特性之一。它让 Java 具备了函数式编程的能力，让代码更加简洁：

```java
// 传统方式：使用匿名内部类
Runnable runnable = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello");
    }
};

// Lambda 方式
Runnable runnable = () -> System.out.println("Hello");

// 传统方式：排序
Collections.sort(users, new Comparator<User>() {
    @Override
    public int compare(User u1, User u2) {
        return u1.getAge().compareTo(u2.getAge());
    }
});

// Lambda 方式
users.sort((u1, u2) -> u1.getAge().compareTo(u2.getAge()));

// 更简洁：方法引用
users.sort(Comparator.comparing(User::getAge));
```

Lambda 表达式的语法简洁明了。`(a, b) -> a + b` 表示一个接受两个参数并返回它们和的函数。箭头左边是参数，右边是函数体。

### Stream API：集合操作的新范式

Stream API 为集合操作提供了一种声明式的编程方式。它让数据处理代码变得更加简洁和易读：

```java
List<String> names = users.stream()
    .filter(u -> u.getAge() > 18)           // 过滤成年人
    .map(User::getName)                     // 提取名字
    .sorted()                               // 排序
    .collect(Collectors.toList());          // 收集结果

// 统计计算
int totalAge = users.stream()
    .mapToInt(User::getAge)
    .sum();

double averageAge = users.stream()
    .mapToInt(User::getAge)
    .average()
    .orElse(0);

// 分组统计
Map<String, List<User>> usersByCity = users.stream()
    .collect(Collectors.groupingBy(User::getCity));

Map<String, Long> cityCount = users.stream()
    .collect(Collectors.groupingBy(User::getCity, Collectors.counting()));
```

Stream 不是数据结构，它不存储元素，而是按需计算（惰性求值）。这种特性让流可以处理无限序列，同时节省内存。

### Optional：消除空指针的优雅方式

Optional 是 JDK 8 引入的容器类，用于表示值存在或不存在的情况。它提供了一种更优雅的方式来处理可能为 null 的值：

```java
// 传统方式：多层嵌套的 null 检查
public String getCityName(User user) {
    if (user != null) {
        Address address = user.getAddress();
        if (address != null) {
            City city = address.getCity();
            if (city != null) {
                return city.getName();
            }
        }
    }
    return "Unknown";
}

// Optional 方式
public String getCityName(User user) {
    return Optional.ofNullable(user)
        .map(User::getAddress)
        .map(Address::getCity)
        .map(City::getName)
        .orElse("Unknown");
}
```

Optional 提供了多种方法来进行链式调用：map、flatMap、filter 用于转换和过滤，orElse、orElseGet、orElseThrow 用于提供默认值或抛出异常。

### 接口的默认方法

JDK 8 允许在接口中定义默认方法（带方法体的方法）：

```java
public interface List<E> extends Collection<E> {
    // 新增的默认方法
    default void sort(Comparator<? super E> c) {
        Collections.sort(this, c);
    }
    
    default Stream<E> stream() {
        return StreamSupport.stream(spliterator(), false);
    }
    
    default boolean isEmpty() {
        return size() == 0;
    }
}
```

默认方法让接口的演进成为可能——在不破坏现有实现的情况下向接口添加新方法。这是 Java 应对向后兼容性挑战的巧妙方案。

## JDK 9：模块系统与更多语法糖

### 模块系统（Project Jigsaw）

JDK 9 引入了 Java 平台模块系统（JPMS），允许将应用和 JDK 本身组织成模块：

```java
// module-info.java
module com.myapp {
    requires com.example.library;
    exports com.myapp.api;
}
```

模块系统提供了更好的封装和依赖管理，让大型应用的组织更加清晰。

### 接口的私有方法

JDK 9 允许在接口中定义私有方法，这对于共享默认方法之间的代码非常有用：

```java
public interface MyInterface {
    default void methodA() {
        // 共享的逻辑
        helper();
    }
    
    default void methodB() {
        // 使用相同的辅助方法
        helper();
    }
    
    private void helper() {
        // 私有实现细节
    }
}
```

### var 类型推断（局部变量类型推断）

虽然 JDK 10 才正式引入 var，但 JDK 9 做了很多准备工作：

```java
// JDK 10+
var list = new ArrayList<String>();
var stream = list.stream();
```

var 不是关键字，它只是一个类型名称的占位符。编译器会根据右侧的表达式推断变量的实际类型。

## JDK 10~14：持续的演进

### var 类型推断（JDK 10）

JDK 10 正式引入了 var，让局部变量声明更加简洁：

```java
var message = "Hello, World!";
var numbers = List.of(1, 2, 3, 4, 5);
var map = new HashMap<String, Integer>();
```

### switch 表达式（JDK 14 预览，JDK 21 正式）

传统的 switch 语句在 JDK 14 得到了增强，可以作为表达式使用：

```java
// JDK 14+：switch 表达式
int result = switch (day) {
    case MONDAY, FRIDAY, SUNDAY -> 6;
    case TUESDAY -> 7;
    case THURSDAY, SATURDAY -> 8;
    case WEDNESDAY -> 9;
    default -> 0;
};
```

### Records（JDK 16 正式）

Records 是一种特殊的类，用于表示不可变的数据载体：

```java
// JDK 16+：定义一个 Record
public record User(String name, int age, String email) {
    // 编译器自动生成：
    // - private final 字段
    // - 全参构造函数
    // - getter 方法 (name(), age(), email())
    // - equals(), hashCode(), toString()
}

// 使用
var user = new User("张三", 25, "zhang@example.com");
System.out.println(user.name());  // 而不是 getName()
```

## JDK 15~17：安全性与现代化

### Sealed Classes（JDK 17 正式）

密封类限制了哪些类可以继承某个类或接口：

```java
public abstract sealed class Shape permits Circle, Rectangle, Square {}

// Circle、Rectangle、Square 必须：
// - 声明为 final（最终类），或
// - 声明为 sealed，或
// - 声明为 non-sealed
public final class Circle extends Shape {}
public sealed class Rectangle extends Shape permits TransparentRect, FilledRect {}
public non-sealed class Square extends Shape {}
```

密封类与模式匹配结合，可以让编译器穷尽检查，成为类型安全的代数数据类型（ADT）。

### 模式匹配 for instanceof（JDK 16 预览，JDK 21 正式）

```java
// JDK 16+
if (obj instanceof String s && s.length() > 5) {
    System.out.println(s.toUpperCase());
}

// JDK 21+
public String describe(Object obj) {
    return switch (obj) {
        case Integer i -> "整数: " + i;
        case String s && s.length() > 10 -> "长字符串: " + s;
        case String s -> "字符串: " + s;
        case null -> "空值";
        default -> "未知类型";
    };
}
```

## JDK 19~21：虚拟线程与未来

### 虚拟线程（Project Loom）

虚拟线程是 JDK 21 最重要的新特性，它大幅降低了 Java 并发编程的复杂度：

```java
// JDK 21：创建虚拟线程
Thread virtualThread = Thread.ofVirtual()
    .name("my-virtual-thread")
    .start(() -> {
        System.out.println("运行在虚拟线程中");
    });

// 使用 ExecutorService 创建虚拟线程
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 10000).forEach(i -> {
        executor.submit(() -> {
            Thread.sleep(Duration.ofSeconds(1));
            return i;
        });
    });
}
```

虚拟线程使得"每任务一个线程"的编程模型成为可能，而不需要担心数百万线程的内存开销。阻塞一个虚拟线程不会阻塞底层平台线程，这让异步代码可以用同步的风格编写。

### Switch 模式匹配的完整版（JDK 21）

JDK 21 完善了 switch 的模式匹配，让它可以处理所有类型：

```java
record Point(int x, int y) {}

String describe(Object obj) {
    return switch (obj) {
        case null -> "空值";
        case Point(int x, int y) when x < 0 && y < 0 -> 
            String.format("负象限点 (%d, %d)", x, y);
        case Point p -> String.format("点 (%d, %d)", p.x(), p.y());
        case String s -> "字符串: " + s;
        case int[] arr -> "int 数组，长度 " + arr.length;
        default -> "未知类型: " + obj.getClass().getName();
    };
}
```

### String Templates（JDK 21 预览）

JDK 21 预览了字符串模板，让字符串插值更加灵活：

```java
// JDK 21 预览
String name = "张三";
int age = 25;
String message = STR."Hello, \{name}! You are \{age} years old.";

// 可以创建自定义的模板处理器
LocalDate today = LocalDate.now();
String html = HTML."""
    <html>
        <body>
            <p>今天是 \{today}。</p>
        </body>
    </html>
    """;
```

## 其他值得关注的特性

### Stream API 的增强

JDK 9 引入了 `takeWhile`、`dropWhile`、`ofNullable` 等方法；JDK 16 引入了 `mapMulti`；JDK 21 引入了 `toList()` 的便捷方法。

### 集合的便捷工厂方法

JDK 9 引入了不可变集合的工厂方法：

```java
List<String> list = List.of("a", "b", "c");
Set<Integer> set = Set.of(1, 2, 3);
Map<String, Integer> map = Map.of("a", 1, "b", 2);
```

### HTTP Client API（JDK 11 正式）

JDK 11 引入了新的 HTTP Client API，支持 HTTP/2 和 WebSocket：

```java
HttpClient client = HttpClient.newHttpClient();

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/data"))
    .header("Accept", "application/json")
    .GET()
    .build();

HttpResponse<String> response = client.send(request, 
    HttpResponse.BodyHandlers.ofString());
```

## Java 演进的启示

Java 的演进体现了几个重要的原则：保持向后兼容性、提供渐进式的语法改进、在保留核心价值观的同时接受新思想。Java 不追求第一个采纳新特性，而是追求采纳经过充分验证的特性。

Lambda 表达式让 Java 进入了函数式编程时代，Stream API 改变了集合操作的风格，虚拟线程将彻底改变并发编程的模式。每一次演进都在解决真实的问题，而不是为了变而变。

## 总结

从 JDK 8 到 JDK 21，Java 经历了多次重大变革。Lambda 表达式和 Stream API 带来了编程范式的转变，模块系统让大型应用更加可控，Records 和 Sealed Classes 让数据建模更加简洁，虚拟线程将彻底改变 Java 的并发模型。

掌握这些新特性，不仅能让你写出更简洁、更高效的代码，也能让你更好地理解 Java 生态中的框架和工具。每一次 JDK 升级，都值得我们去学习和探索。

你是否已经在使用 JDK 21 的新特性？虚拟线程会如何改变你的项目架构？Records 和 Sealed Classes 会如何简化你的代码？这些问题值得每个 Java 开发者思考。
