# String：Java 中最常用的类型

如果问你 Java 中使用最多的类型是什么，恐怕没有任何争议——`String` 绝对是当之无愧的冠军。从日志输出到文件路径，从用户输入到网络请求，`String` 无处不在。据统计，在典型的 Java 应用中，`String` 对象占到了堆内存的很大比例，有些项目甚至能达到 30% 以上。这意味着，理解 `String` 的内部机制，对于写出高效、正确的 Java 代码至关重要。

然而，`String` 看似简单，实则暗藏玄机。为什么 String 是不可变的？字符串常量池是什么？为什么不建议使用 `new String()`？StringBuilder 和 StringBuffer 有什么区别？这些问题看似基础，却难倒了不少有经验的 Java 开发者。在这一章节中，我们将深入探索 String 的方方面面，从不可变性到字符串常量池，从 JDK 8 到 JDK 9+ 的实现变化，带你彻底理解这个每天都在使用的类型。

## JDK 版本的秘密：String 实现的演变

你知道吗？String 在 JDK 8 和 JDK 9+ 的内部实现有重大变化。JDK 9 引入了紧凑字符串（Compact Strings），让 String 不再默认使用 char 数组，而是根据内容选择 byte 数组或 char 数组。这个变化对性能有什么影响？「JDK 8 vs JDK 9+ String 实现差异」将为你深入分析。
