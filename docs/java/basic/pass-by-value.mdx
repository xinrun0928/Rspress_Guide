腾讯P6面试间里，面试官看了看候选人小周的简历，上面写着"扎实的Java基础"。

"问个简单的，"面试官说，"Java 是值传递还是引用传递？"

小周说："Java 是引用传递。"

面试官笑了笑，写下代码：

```java
public void change(String s, StringBuilder sb) {
    s = "changed";           // 改变了参数
    sb.append(" added");     // 改变了对象内容
    sb = new StringBuilder("new");  // 改变了引用
}

public static void main(String[] args) {
    String str = "original";
    StringBuilder sb = new StringBuilder("hello");
    change(str, sb);
    System.out.println(str);   // 输出什么？
    System.out.println(sb);    // 输出什么？
}
```

"这两个输出什么？"

小周说："str 应该是 changed，sb 应该是 new。"

面试官摇摇头："错了。"

小周愣住了。

【面试官心理】
这道题我用来测试候选人对 Java 内存模型的深层理解。90% 的候选人会说"Java 是引用传递"——这是错的。Java 永远是值传递，只是传递的值是引用（对象地址）。能准确说出上面代码输出"original"和"hello added"的候选人，说明他真正理解了值传递和引用传递的本质区别。

## 一、核心结论：Java 永远是值传递 🔴

### 1.1 问题拆解

**第一层：怎么用？**
面试官问："Java 是值传递还是引用传递？"

**第二层：底层实现**
追问："如果是值传递，那传递引用类型时传的是什么？"

**第三层：边界缺陷**
追问："交换两个对象能不能通过参数交换实现？为什么？"

**第四层：选型 trade-off**
追问："你遇到过什么因为值传递/引用传递导致的 bug？"

### 1.2 标准回答

**Java 永远是值传递，没有引用传递**。

这句话听起来矛盾，因为很多人觉得"传递对象时就是引用传递"。关键在于理解：**传递的是引用的副本，不是引用本身**。

```java
// 基本类型：传递的是值的副本
int a = 10;
changePrimitive(a);
System.out.println(a);  // 10，没变化

// 引用类型：传递的是引用的副本（地址的副本）
StringBuilder sb = new StringBuilder("hello");
changeReference(sb);
System.out.println(sb);  // "hello world"，内容变了
```

### 1.3 什么是值传递？什么是引用传递？

| 类型 | 定义 | 效果 |
| --- | --- | --- |
| **值传递** | 把参数的**值**（可能是基本值，也可能是地址值）复制一份传给方法 | 方法内部改变的是副本，不影响原变量 |
| **引用传递** | 把变量的**引用本身**传给方法，方法内可以改变原变量 | 方法内部可以改变外部变量 |

## 二、基本类型 vs 引用类型 🔴

### 2.1 基本类型：值传递值

```java
public void changePrimitive(int x) {
    x = 999;  // 改变的是局部副本
}

public static void main(String[] args) {
    int a = 10;
    changePrimitive(a);
    System.out.println(a);  // 10，没变
}
```

```
栈内存:
main 方法: a = 10
changePrimitive 方法: x = 999 (副本)
```

### 2.2 引用类型：值传递引用（地址）

```java
public void changeReference(StringBuilder sb) {
    sb.append(" world");     // 通过引用副本修改了堆中的对象
}

public static void main(String[] args) {
    StringBuilder sb = new StringBuilder("hello");
    changeReference(sb);
    System.out.println(sb);  // "hello world"，变了！
}
```

```
栈内存:
main: sb = 0x100 (指向堆中的 StringBuilder)
changeReference: 副本 sb = 0x100 (同样的地址)

堆内存:
StringBuilder@0x100 = "hello world" (被修改了)
```

:::tip 💡
**关键区别**：引用类型传递的是地址副本。方法内通过这个副本访问的是同一个对象，所以可以修改对象内容。但方法内如果改变这个副本本身（让它指向新对象），外部的原引用不受影响。
:::

## 三、详解开篇代码 🔴

### 3.1 问题拆解

面试官追问："回到刚才那道题，为什么 str 是 original，sb 是 hello added？"

### 3.2 逐行分析

```java
public void change(String s, StringBuilder sb) {
    s = "changed";           // ①
    sb.append(" added");     // ②
    sb = new StringBuilder("new");  // ③
}

public static void main(String[] args) {
    String str = "original";          // str -> 0x100 ("original")
    StringBuilder sb = new StringBuilder("hello");  // sb -> 0x200 ("hello")
    change(str, sb);
    System.out.println(str);   // ① str 是什么？
    System.out.println(sb);    // ② sb 是什么？
}
```

执行流程：

```
main 栈:
  str -> 0x100 ("original")
  sb  -> 0x200 ("hello")

change 栈（参数接收副本）:
  s  -> 0x100 ("original")  ← str 的副本，指向同一个 String
  sb -> 0x200 ("hello")     ← sb 的副本，指向同一个 StringBuilder

change 执行:
  ① s = "changed"
     s 现在 -> 0x300 ("changed")
     但 main 中的 str 仍然是 0x100，不受影响！

  ② sb.append(" added")
     通过 sb 副本 (0x200) 访问堆中的对象
     StringBuilder@0x200 现在是 "hello added"

  ③ sb = new StringBuilder("new")
     sb 现在 -> 0x400 ("new")
     但 main 中的 sb 仍然是 0x200，不受影响！

main 打印:
  str = "original"  ← s 的改变不影响 str
  sb  = "hello added"  ← sb.append() 改变了对象内容
```

**最终输出**：
- `str`：**"original"**（s 的重新赋值不影响 str）
- `sb`：**"hello added"**（append 改变了对象内容）

:::warning ⚠️
**记住一个核心规则**：方法可以修改引用类型对象的**内容**（通过引用副本访问同一个对象），但不能修改调用者的**引用**（重新赋值只影响副本）。
:::

## 四、面试高频追问 🔴

### 4.1 追问：能否通过参数交换两个变量？

```java
public void swap(String a, String b) {
    String temp = a;
    a = b;
    b = temp;
}

public static void main(String[] args) {
    String x = "a";
    String y = "b";
    swap(x, y);
    System.out.println(x + " " + y);  // "a b" —— 交换失败了！
}
```

**不能**。因为 `a` 和 `b` 是 `x` 和 `y` 的副本，交换 `a` 和 `b` 不影响 `x` 和 `y`。

### 4.2 追问：如何通过参数"返回"多个值？

```java
// 方法1：返回数组/集合
public String[] getNames() {
    return new String[]{"Tom", "Jerry"};
}

// 方法2：用包装类
public void getNames(String[] output) {
    output[0] = "Tom";
    output[1] = "Jerry";
}

// 方法3：用可变参数
public void getNames(List<String> output) {
    output.add("Tom");
    output.add("Jerry");
}
```

:::tip 💡
Java 不支持引用传递，但可以通过包装类（如数组、List、自定义包装类）来"模拟"引用传递的效果。
:::

### 4.3 追问：String 为什么表现得像值传递？

```java
public void changeString(String s) {
    s = s + " world";  // 看起来像在修改 String
}

public static void main(String[] args) {
    String str = "hello";
    changeString(str);
    System.out.println(str);  // "hello" —— 没变！
}
```

因为 String 是**不可变**的。当执行 `s = s + " world"` 时，实际上是创建了一个新的 String 对象，让 `s` 指向它。而原来的 `str` 仍然指向原来的 `"hello"`。

## 五、对象引用的真相 🟡

### 5.1 Java 没有指针，只有引用

```java
// 表面上看：传递的是对象
StringBuilder sb = new StringBuilder("hello");
method(sb);

// 实际上：传递的是引用的副本
method(sb);  // 编译后等价于
method(sb);  // method 接收的是 sb 的副本（地址值 0x100）
```

### 5.2 为什么说"Java 没有引用传递"？

因为引用传递的本质是**方法可以直接操作外部变量本身**。在 C++ 的引用传递中：

```cpp
// C++ 引用传递
void swap(string& a, string& b) {
    string temp = a;
    a = b;
    b = temp;
}

int main() {
    string x = "a", y = "b";
    swap(x, y);
    // x = "b", y = "a" —— 真正交换了！
}
```

C++ 的引用传递中，`swap` 函数拿到的是 `x` 和 `y` 的**引用本身**，可以直接交换它们。而 Java 传递的是**地址副本**，无法直接交换外部变量。

## 六、生产避坑 🟡

### 6.1 坑：误以为对象参数可以被重新赋值

```java
// ❌ 错误假设：方法可以改变外部引用
public void reset(List<String> list) {
    list = new ArrayList<>();  // 这行代码无效！
    list.add("new item");
}

public static void main(String[] args) {
    List<String> list = new ArrayList<>();
    list.add("original");
    reset(list);
    System.out.println(list);  // 仍然只有一个 "original"！
}
```

### 6.2 坑：可变对象作为参数的副作用

```java
public void processOrder(List<Order> orders) {
    for (Order order : orders) {
        order.setStatus("PROCESSED");  // 修改了原始对象！
    }
    orders.clear();  // 清空了原始列表！
}

public static void main(String[] args) {
    List<Order> orders = new ArrayList<>();
    orders.add(new Order("O001"));
    processOrder(orders);
    System.out.println(orders);  // [] —— 被清空了！
}
```

:::warning ⚠️
传递可变对象（List、Map、自定义对象）到方法中时，方法内部可以修改这个对象的内容。如果不想被修改，应该在调用前做防御性拷贝：`method(new ArrayList<>(originalList))`。
:::

最后出一道实战题，检验你是否真正理解了这道题：

```java
public void foo(String s, StringBuilder sb) {
    s = "changed";
    sb.append(" world");
}

public static void main(String[] args) {
    String s = "original";
    StringBuilder sb = new StringBuilder("hello");
    foo(s, sb);
    System.out.println(s + " " + sb);
}
```

输出是：`original hello world`。记住：**String 参数的重新赋值不影响原变量**（因为 String 不可变），**但 StringBuilder 的 append 影响原对象**（因为通过副本访问的是同一个堆对象）。
