# 内部类与 Lambda 表达式

在现实世界中，组合无处不在。一辆汽车由发动机、轮胎、方向盘等部件组成；一家公司由不同部门组成，每个部门又有各自的员工。这种"整体与部分"的关系，在 Java 中可以通过内部类来表达。而 Lambda 表达式，则像是给你的代码装上了加速器——它让曾经冗长的匿名内部类变成了一行简洁的表达式。

## 内部类：藏在内部的伙伴

### 为什么需要内部类？

乍一看，在类内部再定义一个类是件奇怪的事情。但当你深入理解后，会发现内部类解决了很多设计问题：

**逻辑分组**：如果一个类只被另一个类使用，而且两者在概念上紧密相关，把它们放在一起更符合直觉。就像一个团队，成员之间天然是内部关系。

**增强封装性**：内部类可以访问外部类的所有成员，包括私有成员。这让你可以更精细地控制哪些代码能看到哪些内容。

**代码可读性和维护性**：把使用范围很小的类隐藏起来，减少了命名空间的污染。

### 成员内部类：作为外层类的成员

成员内部类是最常见的内部类形式。它就像外层类的一个成员，可以访问外层类的所有成员：

```java
public class Outer {
    private String outerField = "外部类字段";

    public class Inner {
        private String innerField = "内部类字段";

        public void display() {
            // 内部类可以直接访问外部类的所有成员
            System.out.println("外部类: " + outerField);
            System.out.println("内部类: " + innerField);

            // 还可以通过 Outer.this 明确引用外部类
            System.out.println("外部类(this): " + Outer.this.outerField);
        }
    }

    public void createInner() {
        Inner inner = new Inner();
        inner.display();
    }
}

// 使用方式
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();  // 注意语法：通过外部类实例.new 创建内部类实例
inner.display();
```

<Badge text="重要提醒" type="warning" /> 成员内部类隐式持有外部类的引用，所以当外部类对象存在时，内部类对象才有可能存在。这个引用可能会导致内存泄漏，在设计时要考虑生命周期。

### 静态内部类：不需要外部实例

有时候你需要一个内部类，但它不需要访问外部类的实例成员。这时可以使用静态内部类：

```java
public class Outer {
    private static String staticField = "静态字段";
    private String instanceField = "实例字段";

    public static class StaticInner {
        public void display() {
            // 只能访问外部类的静态成员
            System.out.println("静态字段: " + staticField);
            // System.out.println(instanceField);  // 编译错误！

            Outer outer = new Outer();  // 如果需要访问实例字段，需要显式创建实例
            System.out.println("实例字段: " + outer.instanceField);
        }
    }
}

// 静态内部类可以直接创建，不需要外部类实例
Outer.StaticInner inner = new Outer.StaticInner();
```

静态内部类的典型用途是：作为外部类的辅助类，与外部类的实例无关。比如链表的节点定义：

```java
public class LinkedList<E> {
    private Node head;

    // 节点是链表结构的一部分，但不需要访问 LinkedList 的实例成员
    private static class Node<E> {
        E data;
        Node<E> next;
    }
}
```

### 局部内部类：方法里的秘密

局部内部类定义在方法、构造函数或代码块内部。它的作用域仅限于定义它的块内部：

```java
public class Outer {
    public void process() {
        // 局部内部类
        class LocalClass {
            private int value;

            public LocalClass(int value) {
                this.value = value;
            }

            public int getValue() {
                return value;
            }
        }

        LocalClass local = new LocalClass(100);
        System.out.println(local.getValue());
    }

    public void anotherMethod() {
        // LocalClass local = new LocalClass();  // 编译错误，LocalClass 在此作用域不存在
    }
}
```

局部内部类可以访问外部类的成员，也可以访问局部变量（但必须是 final 或 effectively final）：

```java
public class Outer {
    private int outerField = 10;

    public Executable createExecutable(final int param) {
        int localVar = 20;  // effectively final

        return new Executable() {
            @Override
            public void execute() {
                // 可以访问外部类实例字段
                System.out.println(outerField);
                // 可以访问局部变量（必须是 final 或 effectively final）
                System.out.println(param);
                System.out.println(localVar);
            }
        };
    }
}

interface Executable {
    void execute();
}
```

<Badge text="重要提醒" type="warning" /> 在 JDK 8 之前，局部内部类访问的局部变量必须显式声明为 `final`。JDK 8 引入了 "effectively final" 概念：如果局部变量在赋值后不再改变，编译器会把它当作 final 处理。

### 匿名内部类：没有名字的类

匿名内部类是局部内部类的一种特殊形式，它没有显式的类名，在定义的同时就创建实例：

```java
public interface HelloService {
    void sayHello(String name);
}

// 传统方式：定义一个类实现接口
public class HelloServiceImpl implements HelloService {
    @Override
    public void sayHello(String name) {
        System.out.println("你好，" + name);
    }
}

// 使用：需要先创建实现类
HelloService service = new HelloServiceImpl();
service.sayHello("张三");

// 使用匿名内部类：不需要显式创建新的类文件
HelloService anonymousService = new HelloService() {
    @Override
    public void sayHello(String name) {
        System.out.println("你好，" + name);
    }
};
anonymousService.sayHello("李四");
```

匿名内部类常用于创建一次性实现，尤其在 GUI 编程和事件处理中非常常见：

```java
button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("按钮被点击了！");
    }
});
```

## Lambda 表达式：简洁的力量

<Badge text="JDK 8+" type="tip" /> JDK 8 引入了 Lambda 表达式，这是 Java 历史上最重要的新特性之一。它让函数式编程成为可能，让代码更加简洁、表达力更强。

### Lambda 的基本语法

Lambda 表达式本质上是函数的简洁表示。其基本语法是：

```java
// 完整语法
(参数列表) -> { 方法体 }

// 简写规则
// 1. 参数类型可以省略（编译器可以从上下文推断）
// 2. 如果只有一个参数，圆括号可以省略
// 3. 如果方法体只有一条语句，花括号可以省略
// 4. 如果那条语句是 return 语句，return 关键字也要省略
```

下面是一些等价的写法：

```java
// 完整的匿名内部类
Runnable r1 = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello");
    }
};

// Lambda 表达式 - 完整语法
Runnable r2 = () -> {
    System.out.println("Hello");
};

// Lambda 表达式 - 简写
Runnable r3 = () -> System.out.println("Hello");

// 带参数的 Lambda
Comparator<String> c1 = (String a, String b) -> {
    return a.compareTo(b);
};

// 带参数的 Lambda - 简写
Comparator<String> c2 = (a, b) -> a.compareTo(b);
```

### 函数式接口

Lambda 表达式只能用于函数式接口。函数式接口是只有一个抽象方法的接口（可以有默认方法、静态方法、私有方法）。

Java 8 引入了 `@FunctionalInterface` 注解，用于标记函数式接口：

```java
@FunctionalInterface
public interface MathOperation {
    int operate(int a, int b);

    // 可以有默认方法
    default int absOperate(int a, int b) {
        return Math.abs(operate(a, b));
    }
}

MathOperation add = (a, b) -> a + b;
MathOperation subtract = (a, b) -> a - b;
MathOperation multiply = (a, b) -> a * b;

System.out.println(add.operate(10, 5));      // 15
System.out.println(subtract.operate(10, 5)); // 5
System.out.println(multiply.operate(10, 5)); // 50
```

### 方法引用：指向方法的指针

<Badge text="JDK 8+" type="tip" /> 方法引用是 Lambda 表达式的简化形式，当 Lambda 表达式只是调用一个现有方法时，可以使用方法引用。

```java
List<String> names = Arrays.asList("Charlie", "Alice", "Bob");

// Lambda 表达式
names.forEach(name -> System.out.println(name));

// 方法引用 - 更简洁
names.forEach(System.out::println);

// 静态方法引用
List<Integer> lengths = names.stream()
    .map(s -> s.length())           // Lambda
    .collect(Collectors.toList());

List<Integer> lengths2 = names.stream()
    .map(String::length)            // 方法引用
    .collect(Collectors.toList());

// 实例方法引用
String prefix = "Hello ";
List<String> greetings = names.stream()
    .map(name -> prefix.concat(name))    // Lambda
    .collect(Collectors.toList());

List<String> greetings2 = names.stream()
    .map(prefix::concat)                   // 实例方法引用
    .collect(Collectors.toList());

// 构造函数引用
Supplier<ArrayList<String>> listSupplier = () -> new ArrayList<>();
Supplier<ArrayList<String>> listSupplier2 = ArrayList::new;

Function<Integer, ArrayList<String>> listWithCapacity =
    size -> new ArrayList<>(size);
Function<Integer, ArrayList<String>> listWithCapacity2 =
    ArrayList::new;
```

### Lambda 与匿名内部类的对比

```java
// 使用匿名内部类
new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("线程执行中");
    }
}).start();

// 使用 Lambda 表达式
new Thread(() -> System.out.println("线程执行中")).start();

// 匿名内部类中可以定义字段和方法
Runnable r = new Runnable() {
    private int count = 0;

    @Override
    public void run() {
        count++;
        System.out.println("执行了 " + count + " 次");
    }

    public void reset() {
        count = 0;
    }
};

// Lambda 中不能定义额外的方法和字段
Runnable r2 = () -> {
    // int count = 0;  // 不能定义状态！
    System.out.println("线程执行中");
};
```

<Badge text="重要提醒" type="warning" /> 匿名内部类是一个完整的类，可以定义字段和方法；Lambda 表达式只是一个代码块，不能定义额外的状态（除非使用外部变量，但那也必须是 final 或 effectively final 的）。

### Lambda 的作用域

Lambda 表达式的作用域规则与匿名内部类有所不同：

```java
int effectivelyFinal = 10;

// Lambda 表达式不引入新的作用域
// this 引用指向包围它的类的实例
Runnable r = () -> {
    // 可以访问外部变量（必须是 final 或 effectively final）
    System.out.println(effectivelyFinal);

    // this 指代外部类的实例
    System.out.println(this.getClass().getName());
};

// 匿名内部类引入新的作用域
Runnable r2 = new Runnable() {
    @Override
    public void run() {
        // this 指代当前匿名内部类实例
        System.out.println(this.getClass().getName());

        // 如果需要访问外部类实例
        System.out.println(OuterClass.this.getClass().getName());
    }
};
```

## 内部类的实际应用

### 回调模式

内部类常用于实现回调模式：

```java
public class DataLoader {
    private OnDataLoadedListener listener;

    public void setOnDataLoadedListener(OnDataLoadedListener listener) {
        this.listener = listener;
    }

    public void loadData() {
        // 模拟加载数据
        String data = "加载的数据";

        // 加载完成后通知监听器
        if (listener != null) {
            listener.onSuccess(data);
        }
    }

    public interface OnDataLoadedListener {
        void onSuccess(String data);
        void onError(Exception e);
    }
}

// 使用
DataLoader loader = new DataLoader();
loader.setOnDataLoadedListener(new DataLoader.OnDataLoadedListener() {
    @Override
    public void onSuccess(String data) {
        System.out.println("成功: " + data);
    }

    @Override
    public void onError(Exception e) {
        System.out.println("失败: " + e.getMessage());
    }
});
loader.loadData();
```

### 迭代器模式

内部类非常适合实现迭代器：

```java
public class BookCollection {
    private Book[] books = {
        new Book("《Java核心技术》"),
        new Book("《Effective Java》"),
        new Book("《深入理解Java虚拟机》")
    };
    private int size = books.length;

    public Iterator iterator() {
        return new BookIterator();
    }

    private class BookIterator implements Iterator<Book> {
        private int position = 0;

        @Override
        public boolean hasNext() {
            return position < size;
        }

        @Override
        public Book next() {
            return books[position++];
        }
    }
}

// 使用
BookCollection collection = new BookCollection();
Iterator<Book> iterator = collection.iterator();
while (iterator.hasNext()) {
    System.out.println(iterator.next().getTitle());
}
```

<Badge text="JDK 5+" type="tip" /> JDK 5 引入了 for-each 循环，配合 `Iterable` 接口可以让迭代更简洁：

```java
public class BookCollection implements Iterable<Book> {
    // ... 同上

    @Override
    public Iterator<Book> iterator() {
        return new BookIterator();
    }
}

// 使用 for-each
for (Book book : collection) {
    System.out.println(book.getTitle());
}
```

## 方法引用与 Lambda 的选择

方法引用和 Lambda 都能让代码更简洁，但应该优先使用哪个？

<Badge text="最佳实践" type="tip" /> 当 Lambda 表达式只是调用一个现有方法时，优先使用方法引用。方法引用更简洁，表达意图更清晰。

```java
// 优先使用
list.forEach(System.out::println);
map.forEach((k, v) -> System.out.println(k + "=" + v));  // 必须用 Lambda

// 静态方法引用
strings.stream().map(Integer::parseInt);

// 实例方法引用（特定对象）
String prefix = "Hello ";
strings.stream().map(prefix::concat);

// 实例方法引用（任意对象）
strings.stream().map(String::toUpperCase);
```

## 常见陷阱与最佳实践

**Lambda 中的闭包陷阱**：Lambda 可以访问外部变量，但这些变量必须是 final 或 effectively final。

```java
// 编译错误
int counter = 0;
runnable = () -> counter++;  // 错误！counter 不是 final

// 正确做法：使用数组或包装类
int[] counter = {0};
runnable = () -> counter[0]++;  // 正确
```

**避免过长的 Lambda**：如果 Lambda 表达式超过几行，考虑提取为方法引用或普通方法。

```java
// 太长，难以阅读
list.stream()
    .filter(x -> x > 0)
    .map(x -> {
        if (x < 10) return "small";
        else if (x < 100) return "medium";
        else return "large";
    });

// 改进：提取为方法
list.stream()
    .filter(x -> x > 0)
    .map(this::categorize);
```

**类型推断的边界**：Lambda 的类型推断在某些情况下会失败，需要显式指定类型。

```java
// 编译器无法推断
BinaryOperator<Long> add = (x, y) -> x + y;  // 错误！

// 需要显式指定
BinaryOperator<Long> add = (Long x, Long y) -> x + y;  // 正确
```

内部类和 Lambda 表达式是 Java 中两个重要的语言特性，它们让代码更加组织化、简洁。内部类解决了类的封装和逻辑分组问题，而 Lambda 则带来了函数式编程的能力。下一章我们将深入探讨 String 的不可变性，这是理解 Java 字符串处理的基础。
