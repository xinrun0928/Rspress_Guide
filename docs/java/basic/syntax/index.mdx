# Java 基础语法

当你第一次打开 Eclipse 或 IntelliJ IDEA，准备写下人生中第一行 Java 代码时，你可能会想：为什么 Java 代码要这样写？为什么变量需要声明类型？为什么有些地方用 `int`，有些地方用 `Integer`？这些看似基础的问题，实际上蕴含着 Java 语言设计者数十年的智慧结晶。

Java 的基础语法不仅仅是语法的规则，更是面向对象编程思想的入门钥匙。在这一章节中，我们将从数据类型与类型转换开始，逐步深入到面向对象的核心概念：封装、继承与多态。然后我们会探讨抽象类与接口的区别，以及它们各自的使用场景。最后，我们还会深入了解内部类与 Lambda 表达式，这是现代 Java 编程中不可或缺的利器。

## 数据类型：Java 的地基工程

想象一下，如果你要建造一栋高楼，你会选择什么样的地基？数据类型就是 Java 程序的地基。Java 是一种强类型语言，这意味着每一个变量、每一个表达式都有明确的类型。类型不仅决定了变量能存储什么数据，还决定了这些数据能进行什么样的操作。

在 Java 中，数据类型分为两大阵营：基本数据类型和引用数据类型。基本数据类型就像是建筑中的砖块，简单、直接、高效；而引用数据类型则像是预制好的建筑模块，可以组合成更复杂的结构。理解这两种类型的区别和联系，是深入学习 Java 的第一步。

## 面向对象：封装、继承与多态

如果说数据结构是 Java 的骨架，那么面向对象就是 Java 的灵魂。Java 不仅仅是一门编程语言，更是一种编程思想。封装让我们把数据和操作数据的方法打包在一起，隐藏内部实现细节，只暴露必要的接口；继承让我们可以在现有类的基础上创建新类，复用代码的同时还能进行扩展；多态则让我们可以用统一的接口处理不同类型的对象，编写出更加灵活和可扩展的代码。

这三个概念看似简单，但要真正理解和运用它们，需要大量的实践和思考。在这一章节中，我们将通过大量的实例，让你不仅知道这些概念是什么，更要理解为什么是这样，以及如何在实际开发中运用它们。

## 抽象类与接口：代码的契约设计

在软件设计中，我们经常需要定义一些"规范"或"契约"，让不同的类遵循相同的规则。抽象类和接口就是 Java 中实现这种契约设计的两种主要方式。它们看起来很相似，都能定义方法签名让子类去实现，但在设计理念和使用场景上有着微妙的区别。

抽象类是一种"是...的特例"的关系，比如"狗是动物的一种特例"；而接口则是一种"能做什么"的关系，比如"会游泳"、"能飞"。理解这种区别的微妙之处，能够帮助你在设计类层次结构时做出更明智的选择。

## 内部类与 Lambda：让代码更简洁

随着 Java 8 引入 Lambda 表达式和函数式接口，Java 编程进入了一个新的时代。Lambda 表达式不仅让代码更加简洁，还为函数式编程铺平了道路。而内部类作为 Lambda 的前身，同样在 Java 中扮演着重要的角色。

内部类让我们可以在一个类的内部定义另一个类，这种看似奇怪的语法实际上解决了许多设计问题。从成员内部类到局部内部类，从匿名内部类到静态内部类，每一种都有其独特的用途和适用场景。而 Lambda 表达式，则是对这些场景的进一步简化。
