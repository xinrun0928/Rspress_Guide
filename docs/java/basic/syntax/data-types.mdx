# 数据类型与类型转换

你有没有想过，当你在 Java 中写下 `int age = 25;` 这样的代码时，背后究竟发生了什么？计算机的内存中是如何存储这个数字的？为什么 Java 要把数据类型分成这么多种？这些问题看似基础，却隐藏着计算机底层的核心原理。理解数据类型，不仅仅是记住关键字那么简单，而是理解数据在计算机中如何表示、存储和运算的关键。

## 基本数据类型：Java 的原子

Java 提供了八种基本数据类型，它们是 Java 语言的基础构件。就像物质由原子组成一样，Java 程序由这些基本数据类型构成。这八种类型分为四类：整数类型、浮点类型、字符类型和布尔类型。

### 整数类型：精确的数学

整数类型包括 `byte`、`short`、`int` 和 `long`，它们的取值范围各不相同。选择合适的数据类型，就像选择合适的容器来装水——用太小的容器会溢出，用太大的容器又浪费空间。

`byte` 是最小的整数类型，占用 1 个字节（8 位），取值范围是 -128 到 127。这个范围看起来很小，但在处理网络数据或二进制文件时非常有用。比如，读取一个字节的数据，用 `byte` 数组再合适不过。

```java
byte fileHeader = 0x4A;  // 'J' 的 ASCII 码
byte[] magic = {0xCA, 0xFE, 0xBA, 0xBE};  // Java class 文件的魔数
```

`short` 占用 2 个字节，取值范围是 -32768 到 32767。在现代 Java 开发中，这个类型已经很少使用了，主要出现在与旧系统交互的场景中。

`int` 是最常用的整数类型，占用 4 个字节，取值范围大约是 -21 亿到 21 亿。日常开发中的大多数整数场景，使用 `int` 就足够了。

```java
int pageSize = 20;                    // 分页大小
int maxRetries = 3;                   // 最大重试次数
int userCount = 1_000_000;            // 用户数量，下划线增强可读性
```

`long` 占用 8 个字节，用于存储更大的整数。当你需要表示超过 21 亿的数字时，就该 `long` 出场了。金融系统中处理金额、科学计算中处理大数值，都是 `long` 的用武之地。

```java
long population = 7_900_000_000L;     // 全球人口，需要 L 后缀
long timestamp = System.currentTimeMillis();  // 时间戳
```

<Badge text="JDK 7+" type="tip" /> 从 JDK 7 开始，你可以在数值字面量中使用下划线来增强可读性，比如 `1_000_000`，编译器会自动忽略这些下划线。

### 浮点类型：近似之美

浮点类型包括 `float` 和 `double`，用于表示小数。它们的设计理念是：用有限的位表示一个近似的数值范围，代价是牺牲精确度。这就像用一把尺子测量长度——你永远只能得到一个近似值。

`float` 占用 4 个字节，单精度浮点数，有效数字约为 6-7 位。`double` 占用 8 个字节，双精度浮点数，有效数字约为 15-16 位。在现代 Java 开发中，`double` 是更常见的选择，因为它的精度更高，而且大多数数学函数都返回 `double`。

```java
float pi = 3.14159f;          // 需要 f 或 F 后缀
double e = 2.718281828459045; // 自然常数
double price = 99.99;         // 商品价格
```

<Badge text="重要提醒" type="warning" /> 浮点数不适合进行精确比较。比如 `0.1 + 0.2 != 0.3`，因为 0.1 在二进制中是一个无限循环小数，只能截断存储。如果你需要精确计算（如金融计算），应该使用 `BigDecimal`。

### 字符类型：文字的数字密码

`char` 用于表示单个字符，占用 2 个字节，采用 Unicode 编码。这意味着它可以表示世界上绝大多数的文字符号，包括中文、日文、阿拉伯文等。

```java
char chinese = '中';
char emoji = '\uD83D\uDE00';  // Unicode 表示的笑脸 emoji
char letter = 'A';
```

有趣的是，`char` 本质上是一个无符号整数，存储的是字符的 Unicode 码点。所以你可以对它进行数值运算：

```java
char c = 'A';
System.out.println(c + 1);    // 输出 66，即 'B' 的码点
System.out.println((char)(c + 1));  // 输出 'B'
```

### 布尔类型：真与假

`boolean` 是 Java 中最简单的类型，只有两个可能的值：`true` 和 `false`。它的大小没有明确规定，取决于 JVM 的实现，可能是 1 位、1 字节，甚至是 4 字节。

```java
boolean isActive = true;
boolean hasPermission = user.checkPermission("READ");
boolean isEmpty = list.isEmpty();
```

<Badge text="重要提醒" type="warning" /> 在 Java 中，布尔类型不能转换为其他类型，其他类型也不能转换为布尔类型。这种设计避免了其他语言中常见的"真值假值"陷阱，让代码更加清晰。

## 引用数据类型：对象的坐标

与基本数据类型不同，引用数据类型存储的是对象的"地址"——一个指向对象在堆内存中位置的引用。想象一下，基本数据类型像是直接持有物品，而引用数据类型像是持有物品所在位置的纸条。

```java
String name = new String("Alice");  // name 存储的是 String 对象的引用
int[] scores = {90, 85, 78};        // scores 存储的是数组的引用
Date today = new Date();            // today 存储的是 Date 对象的引用
```

当你比较两个引用类型变量时，`==` 比较的是引用是否指向同一个对象，而不是对象的值是否相等。这就是为什么字符串比较通常要用 `equals()` 方法：

```java
String a = new String("hello");
String b = new String("hello");
System.out.println(a == b);        // false，不同的对象
System.out.println(a.equals(b));  // true，内容相同
```

## 类型转换：数据的变形术

类型转换允许我们将一种类型的值转换为另一种类型。在 Java 中，类型转换分为两种：自动类型转换和强制类型转换。

### 自动类型转换：自然的升级

当把一个范围小的类型赋值给一个范围大的类型时，Java 会自动进行类型转换。这就像是把一杯水倒进一个更大的杯子，不需要任何操作。

转换顺序（从小到大）是：`byte` → `short` → `int` → `long` → `float` → `double`

```java
int i = 100;
long l = i;        // 自动转换，int 到 long
double d = i;      // 自动转换，int 到 double

float f = 3.14f;
double dd = f;     // 自动转换，float 到 double
```

值得注意的是，`int` 到 `float` 或 `long` 到 `double` 的转换可能会丢失精度，但 Java 仍然允许自动转换，因为范围确实更大了。

### 强制类型转换：人工的干预

当需要把一个范围大的类型赋值给范围小的类型时，必须进行强制类型转换。这就像把一大桶水倒进一个小杯子，你必须明确告诉编译器："我知道这可能会丢失数据，但我仍然要这么做。"

```java
double pi = 3.14159;
int truncated = (int) pi;    // 结果是 3，小数部分被截断

long bigNum = 100L;
byte small = (byte) bigNum;   // 结果是 100，只要在范围内就不会丢失数据

long huge = 130L;
byte overflow = (byte) huge;  // 结果是 -126，发生了溢出！
```

<Badge text="重要提醒" type="warning" /> 强制类型转换时，如果值超出了目标类型的范围，会发生溢出。比如 `byte` 的最大值是 127，把 130 强制转换为 `byte` 会得到 -126（因为 130 = 127 + 3，溢出后从 -128 开始计数）。这是初学者最容易踩的坑之一。

### 表达式中的类型提升

当一个表达式中有多种类型的操作数时，Java 会自动进行类型提升：

```java
int a = 5;
int b = 2;
System.out.println(a / b);        // 输出 2，不是 2.5

double result = a / b;            // 仍然是 2，因为 a / b 先计算
double correct = (double) a / b;  // 正确的结果 2.5
```

## 包装类：基本类型的对象包装

每一种基本数据类型都有对应的包装类：`Integer`、`Long`、`Double`、`Boolean` 等。包装类把基本类型包装成对象，这样就可以在需要对象的场景（如集合类）中使用基本类型。

```java
Integer count = Integer.valueOf(100);  // 基本类型到包装类
int primitive = count.intValue();       // 包装类到基本类型

// JDK 5+ 的自动装箱/拆箱
Integer boxed = 100;   // 自动装箱，等价于 Integer.valueOf(100)
int unboxed = boxed;   // 自动拆箱，等价于 boxed.intValue()
```

<Badge text="JDK 5+" type="tip" /> 自动装箱和拆箱是 JDK 5 引入的特性，大大简化了基本类型和包装类之间的转换。但要注意，自动装箱可能创建不必要的对象，影响性能和内存使用。

## 数字字面量的特殊表示

Java 支持多种进制的数字字面量表示：

```java
int decimal = 100;           // 十进制
int octal = 0144;            // 八进制，以 0 开头
int hex = 0x64;              // 十六进制，以 0x 开头
int binary = 0b1100100;     // 二进制，JDK 7+ 支持

System.out.println(decimal == octal);    // true，都是 100
System.out.println(decimal == hex);     // true
System.out.println(decimal == binary);  // true
```

<Badge text="JDK 7+" type="tip" /> 二进制字面量（`0b` 或 `0B` 前缀）是 JDK 7 引入的特性，在此之前只能通过位运算来表示二进制。

## 实践建议

理解了数据类型和类型转换后，以下是一些实战建议：

**选择合适的数据类型**：不要为了"保险"而总是使用 `long` 或 `double`。选择能够容纳数据的最小的类型，不仅节省内存，有时候还能提高性能。

**警惕精度问题**：涉及金额、小数比较等场景，优先使用 `BigDecimal`。浮点数的精度陷阱是 Java 初学者最常见的 bug 来源之一。

**注意自动转换的陷阱**：`int` 除以 `int` 结果仍然是 `int`，如果你需要小数结果，确保至少有一个操作数是浮点类型。

**理解引用类型和基本类型的区别**：`==` 对于引用类型比较的是引用，不是值。这个区别在比较字符串和包装类时特别容易出错。

数据类型是 Java 编程的起点，理解它们的细微差别，能让你在编写代码时更加得心应手。下一章我们将深入探讨面向对象编程，看看这些数据类型是如何被封装成对象的。
