阿里P6面试间里，面试官看着候选人小马，问了一道看似基础的题：

"Java 有哪些内部类？分别有什么区别？"

小马说："有四种：成员内部类、静态内部类、局部内部类和匿名内部类。"

面试官点点头，写下了这段代码：

```java
public class Outer {
    private int value = 10;

    public class Inner {
        private int value = 20;

        public void print() {
            System.out.println(value);         // 打印哪个？
            System.out.println(this.value);    // 打印哪个？
            System.out.println(Outer.this.value); // 打印哪个？
        }
    }
}
```

"这三个分别打印什么？"

小马说："第一个打印 20...第二个也打印 20...第三个...Outer.this.value 应该是 10？"

面试官继续追问："那静态内部类和成员内部类的区别是什么？"

小马说："静态内部类...不需要创建外部类对象就能访问？"

面试官又问："匿名内部类和局部内部类有什么区别？匿名内部类能访问局部变量吗？有什么限制？"

小张彻底卡住了。

【面试官心理】
这道题我用来测试候选人对 Java 闭包机制的理解。内部类看似简单，但涉及到作用域、变量捕获、`this` 引用等深层机制。能准确回答 Inner 类中三个打印值、能说出局部变量必须是 final 或 effectively final 的候选人，说明真正理解了 Java 的变量作用域。

## 一、四种内部类概览 🔴

### 1.1 问题拆解

**第一层：怎么用？**
面试官问："Java 有哪些内部类？分别用在什么场景？"

**第二层：底层实现**
追问："内部类在编译后生成什么？内部类怎么持有外部类的引用？"

**第三层：边界缺陷**
追问："匿名内部类和局部内部类在访问外部变量时有什么限制？"

**第四层：选型 trade-off**
追问："什么时候用内部类而不是单独的文件？静态内部类和成员内部类的选择依据是什么？"

### 1.2 四种内部类一览

| 类型 | 定义位置 | 修饰符 | 是否持有外部类引用 | 能否访问外部类成员 |
| --- | --- | --- | --- | --- |
| 成员内部类 | 类级别，作为外部类的成员 | 无/其他 | ✅ 是（隐式） | ✅ 全部（包括 private） |
| 静态内部类 | 类级别，作为外部类的成员 | static | ❌ 否 | ❌ 只能访问外部类的静态成员 |
| 局部内部类 | 方法/代码块内部 | 无 | ✅ 是 | ✅ 全部（包括 private） |
| 匿名内部类 | 表达式内部，没有名字 | 无 | ✅ 是 | ✅ 全部（包括 private） |

## 二、成员内部类 🔴

### 2.1 基本用法

```java
public class Outer {
    private String outerField = "outer";

    // 成员内部类：作为外部类的一个成员
    public class Inner {
        private String innerField = "inner";

        public void accessOuter() {
            // 可以直接访问外部类的所有成员，包括 private
            System.out.println(outerField);
            System.out.println(Outer.this.outerField);  // 显式指定外部类引用
        }
    }
}

// 创建方式：必须通过外部类实例
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
```

### 2.2 编译后的结构

```java
// 编译后会生成两个 .class 文件：
// Outer.class
// Outer$Inner.class

// 编译器为内部类添加的隐式引用
public class Outer$Inner {
    // 编译器添加：指向外部类实例的引用
    final Outer this$0;

    public Outer$Inner(Outer outer) {
        this.this$0 = outer;
    }

    public void accessOuter() {
        // 编译后的代码：
        System.out.println(this$0.outerField);
    }
}
```

### 2.3 this 引用详解

```java
public class Outer {
    private int value = 10;

    public class Inner {
        private int value = 20;

        public void print() {
            int value = 30;

            System.out.println(value);              // 局部变量：30
            System.out.println(this.value);         // Inner 的 this：20
            System.out.println(Outer.this.value);  // 外部类的引用：10
        }
    }
}
```

**规则**：在内部类中，`this` 默认指向当前内部类实例。如果需要访问外部类的 `this`，用 `Outer.this`。

## 三、静态内部类 🔴

### 3.1 基本用法

```java
public class Outer {
    private static String staticField = "static outer";
    private String instanceField = "instance outer";

    // 静态内部类：不持有外部类实例引用
    public static class StaticInner {
        public void access() {
            // 只能访问外部类的静态成员
            System.out.println(staticField);
            // System.out.println(instanceField);  // ❌ 编译错误！

            // 创建静态内部类不需要外部类实例
            StaticInner si = new StaticInner();
        }
    }
}

// 创建方式：不需要外部类实例
Outer.StaticInner si = new Outer.StaticInner();
```

### 3.2 静态内部类的特点

| 维度 | 成员内部类 | 静态内部类 |
| --- | --- | --- |
| 是否持有外部类引用 | ✅ 持有（隐式 final 字段） | ❌ 不持有 |
| 是否需要外部类实例 | ✅ 需要 `outer.new Inner()` | ❌ 不需要 |
| 能访问外部类成员 | 全部（包括 private） | 只能访问静态成员 |
| 能否定义 static 成员 | ❌ 不能 | ✅ 可以 |
| 内存泄漏风险 | 有（持有外部类引用） | 无（不持有外部类引用） |

### 3.3 典型应用：Map.Entry

```java
// JDK 中的静态内部类典型应用
public class HashMap<K, V> {
    // Node 是 HashMap 的静态内部类
    static class Node<K, V> implements Map.Entry<K, V> {
        final int hash;
        final K key;
        V value;
        Node<K, V> next;

        // 不需要持有外部 HashMap 的引用
        // 只依赖自身的字段
    }
}
```

## 四、局部内部类 🔴

### 4.1 基本用法

```java
public class Outer {
    private String outerField = "outer";

    public void method() {
        final int localVar = 100;

        // 局部内部类：定义在方法内部
        class LocalInner {
            private String innerField = "local inner";

            public void access() {
                // 可以访问外部类的所有成员
                System.out.println(outerField);
                // 可以访问局部变量（必须 effectively final）
                System.out.println(localVar);
            }
        }

        LocalInner li = new LocalInner();
        li.access();
    }
}
```

### 4.2 变量捕获机制

这是面试的高频考点。局部内部类可以访问**局部变量**，但有一个关键限制：**局部变量必须是 final 或 effectively final**。

```java
public class Outer {
    public Runnable createRunnable() {
        int count = 0;  // effectively final（没有被修改）

        class LocalRunnable implements Runnable {
            @Override
            public void run() {
                System.out.println(count);  // OK：count 是 effectively final
            }
        }

        return new LocalRunnable();
    }
}
```

但如果局部变量被修改：

```java
public Runnable createRunnable() {
    int count = 0;
    count++;  // count 不再是 effectively final

    class LocalRunnable implements Runnable {
        @Override
        public void run() {
            System.out.println(count);  // ❌ 编译错误！
        }
    }

    return new LocalRunnable();
}
```

:::warning ⚠️
局部内部类捕获的是**局部变量的副本**，而不是变量本身。如果允许在内部类使用过程中修改变量，就会产生歧义——内部类看到的是哪个版本？所以 Java 要求局部变量必须是 final 或 effectively final。
:::

### 4.3 编译后的局部内部类

编译后生成的类名：`Outer$1LocalInner.class`（带序号）

```java
// 编译器为局部内部类生成的代码
class Outer$1LocalInner {
    final int val$count;          // 局部变量的副本
    final Outer this$0;           // 外部类引用

    Outer$1LocalInner(Outer outer, int count) {
        this.this$0 = outer;
        this.val$count = count;
    }
}
```

## 五、匿名内部类 🔴

### 5.1 基本用法

```java
// 匿名内部类：没有名字的局部内部类
public class Outer {
    public Runnable createRunnable() {
        return new Runnable() {
            @Override
            public void run() {
                System.out.println("匿名内部类运行中");
            }
        };
    }
}

// 匿名内部类常见用法
public class Demo {
    public static void main(String[] args) {
        // 1. 作为参数传递
        new Thread(new Runnable() {
            @Override
            public void run() {
                System.out.println("线程运行");
            }
        }).start();

        // 2. 继承式
        Animal dog = new Animal() {
            @Override
            public void sound() {
                System.out.println("汪汪");
            }
        };
    }
}

abstract class Animal {
    abstract void sound();
}
```

### 5.2 匿名内部类的限制

```java
// 限制1：只能继承一个类或实现一个接口
// ✅ OK
Runnable r = new Runnable() { ... };

// ❌ 错误：不能同时继承和实现
Runnable r = new Animal() implements Runnable { ... };

// 限制2：不能有构造函数
// 匿名内部类没有名字，所以不能定义构造函数

// 限制3：只能访问 effectively final 的局部变量
public Runnable create(final String name) {
    return new Runnable() {
        @Override
        public void run() {
            System.out.println(name);  // OK：name 是 final 或 effectively final
            // name = "new name";  // ❌ 编译错误！
        }
    };
}
```

### 5.3 匿名内部类 vs Lambda（Java 8+）

```java
// 匿名内部类
Runnable r1 = new Runnable() {
    @Override
    public void run() {
        System.out.println("匿名内部类");
    }
};

// Lambda 表达式（函数式接口的简化写法）
Runnable r2 = () -> System.out.println("Lambda");

// 两者在字节码层面的区别
// 匿名内部类：生成 Outer$1.class
// Lambda：生成 lambda 表达式的调用点（invokedynamic），不生成新类
```

| 维度 | 匿名内部类 | Lambda |
| --- | --- | --- |
| 本质 | 类 | 方法（函数式接口的实例） |
| 编译产物 | 新的 .class 文件 | invokedynamic 指令，不生成类 |
| this 引用 | 指向匿名内部类实例 | 指向包围它的类 |
| 变量捕获 | 捕获外部变量（必须是 effectively final） | 捕获外部变量（必须是 effectively final） |
| 用途 | 实现抽象类或接口 | 实现函数式接口（只有一个抽象方法） |

## 六、生产避坑 🔴

### 6.1 坑：内存泄漏

```java
// 危险：成员内部类持有外部类引用
public class Outer {
    private byte[] largeData = new byte[10 * 1024 * 1024];  // 10MB

    public Runnable createTask() {
        // 这个内部类持有外部类引用
        return new Runnable() {
            @Override
            public void run() {
                System.out.println("task running");
            }
        };
    }
}

public static void main(String[] args) {
    Outer outer = new Outer();
    Runnable task = outer.createTask();
    // 如果只是不用 outer 但仍然持有 task
    // outer 不会被 GC 回收（因为内部类持有引用）
    outer = null;  // outer 仍然是可达的！
}
```

:::warning ⚠️
成员内部类会隐式持有外部类的引用。如果内部类的生命周期比外部类长（比如作为线程任务、回调、监听器），就会导致外部类无法被垃圾回收，造成内存泄漏。**解决方案**：使用静态内部类（不持有外部类引用）或在不使用时主动清理引用。
:::

### 6.2 坑：匿名内部类的 this 误解

```java
public class Outer {
    private String name = "outer";

    public Runnable createRunnable() {
        return new Runnable() {
            private String name = "anonymous";

            @Override
            public void run() {
                System.out.println(this.name);  // 打印 "anonymous"！
                // 这里的 this 是匿名内部类 Runnable 的实例，不是 Outer
            }
        };
    }
}
```

### 6.3 坑：匿名内部类在循环中的变量捕获

```java
// ❌ 危险：循环中创建匿名内部类
for (String s : list) {
    new Thread(new Runnable() {
        @Override
        public void run() {
            System.out.println(s);  // 所有线程看到的是同一个变量
            // 如果 s 在循环中被修改，所有线程看到的都是最终值
        }
    }).start();
}

// ✅ 正确：使用临时变量隔离
for (String s : list) {
    final String captured = s;  // 明确捕获
    new Thread(new Runnable() {
        @Override
        public void run() {
            System.out.println(captured);
        }
    }).start();
}

// ✅ Java 8+ 更好：用 Lambda
for (String s : list) {
    new Thread(() -> System.out.println(s)).start();
}
```

## 七、面试高频追问 🔴

### 7.1 追问：为什么局部变量要被捕获？

这是 Java 闭包（Closure）的实现机制：

```java
public class Outer {
    public Runnable createTask(final int initial) {
        int count = initial;  // 局部变量在栈上，方法返回后栈帧销毁

        class LocalRunnable implements Runnable {
            public void run() {
                // 如果内部类直接用 count，方法返回后 count 不存在了
                // 所以编译器复制了一份 count 到堆中
                System.out.println(count);
            }
        }

        return new LocalRunnable();
    }
}
```

局部变量在**栈**上，方法返回后栈帧销毁。但内部类可能被**堆**中持有（作为返回值被外部引用），所以编译器把局部变量**复制**了一份到堆中。这就是变量捕获的原理。

### 7.2 追问：四种内部类的选择

| 场景 | 推荐 | 原因 |
| --- | --- | --- |
| 工具类，逻辑上属于外部类但不需要外部引用 | 静态内部类 | 不持有外部引用，避免内存泄漏 |
| 需要访问外部类实例状态 | 成员内部类 | 持有外部引用 |
| 逻辑复杂，需要命名 | 局部内部类 | 作用域明确，不会污染类命名空间 |
| 一次性使用，不需要类名 | 匿名内部类 | 代码更简洁 |

最后出一道综合题：

```java
public class Outer {
    private static int staticVal = 1;
    private int instanceVal = 2;

    public Runnable createTask(final int param) {
        int localVal = 3;

        return new Runnable() {
            private int innerVal = 4;

            @Override
            public void run() {
                // 以下哪些可以访问？分别打印什么？
                System.out.println(staticVal);      // ①
                System.out.println(instanceVal);    // ②
                System.out.println(param);         // ③
                System.out.println(localVal);      // ④
                System.out.println(innerVal);      // ⑤
            }
        };
    }
}
```

答案：①~⑤ 全部可以访问。① static 成员，直接访问；② 通过 `Outer$1Runnable.this$0.instanceVal` 访问；③④⑤ 是局部变量的副本和内部类自身的字段，直接访问。
