# Java 基础核心

面试官翻到简历上"扎实的 Java 基础"这一行，嘴角微微一抽——因为他知道，接下来这个问题，80%的候选人会在第三层追问上崩掉。

Java 基础不是"会写代码"那么简单。它是你理解所有高级特性的地基：泛型为什么有类型擦除？equals 和 `==` 到底比什么？String 为什么是不可变的？这些问题答不清楚，P6 就是天花板。

这一模块，带你把 Java 基础从"能用"升级到"能用且能讲清楚为什么"。

## 学习路径指引

### P5 候选人（校招/初级社招）

这一层的核心任务是**把基础概念讲清楚**，不需要深入源码，但必须理解底层逻辑。

**必过关卡**：

1. String 的不可变性解决了什么问题？`StringBuilder` vs `StringBuffer` 的区别是什么？
2. equals 方法的通用实现模板是什么？为什么重写 equals 必须重写 hashCode？
3. 什么是值传递？Java 里没有引用传递，那对象作为参数传递时底层发生了什么？
4. 接口和抽象类的区别是什么？各自适用什么场景？

这四道题能流畅答出来，基础关就算过了。

### P6 候选人（中级社招）

这一层需要在**概念理解的基础上，能回答追问**，尤其是"为什么这样设计"。

**拉开差距的问题**：

- `String.intern()` 在 JDK6 和 JDK7/8 的行为有什么区别？为什么要改变？
- 泛型的类型擦除会引发哪些问题？什么是桥接方法？
- 方法重载的匹配规则是什么？PECS 原则怎么应用？
- 异常捕获的顺序有什么讲究？finally 里 return 会发生什么？

这些问题是 P5 和 P6 的分水岭。能答出来的，才算真正理解了这门语言。

### P7 候选人（高级社招/架构方向）

P7 的 Java 基础不是在考你"会不会"，而是在考你**能不能从设计者的视角看问题**。

**典型的 P7 级追问**：

- String 为什么设计成不可变？这种设计带来了哪些优缺点？
- 如果让你设计一套新的泛型系统，你会怎么改进 Java 的类型擦除？
- 注解处理器在编译期做了什么？它和反射的区别是什么？

这些问题的核心是：你有没有能力去评判一门语言的设计，而不是只会使用它。

## 生产避坑指南

Java 基础看似简单，但生产事故往往就藏在这些"基础常识"里：

- **String 拼接**：循环内拼接字符串导致 OOM，`StringBuilder` 容量没设好导致频繁扩容
- **equals 坑**：自定义对象放进 HashSet/HashMap 但没重写 hashCode，导致"找不到"
- **泛型擦除**：Gson 反序列化时类型丢失，泛型数组不能 new，`T[]` 必须用反射
- **异常处理**：catch 了 Exception 然后吞掉、finally 里修改返回值、抛出 checked 异常导致 API 污染

:::warning ⚠️
生产上最常见的"基础翻车"：在 for 循环里用 `+=` 拼接字符串，数据量一上来直接 OOM。StringBuilder 容量默认是 16，拼接后超过 16 会扩容 2 倍 + 2，小心这个暗坑。
:::
