# 线程安全与死锁概述

线程安全是 Java 并发编程中最核心的概念之一。当多个线程同时访问共享资源时，如果没有适当的同步机制，就会出现各种并发问题：数据竞争、死锁、活锁、饥饿……这些问题，轻则导致程序行为异常，重则导致系统崩溃。

## 什么是线程安全

线程安全是指：在多线程环境下，无论线程如何调度交替执行，代码总能产生正确的结果。

这个定义有几个关键点：

1. **正确性**：程序行为符合预期
2. **线程调度**：不考虑线程的执行顺序
3. **共享资源**：涉及对共享状态的访问

一个线程不安全的程序，可能在单核处理器上运行正常，在多核处理器上却出现 bug。这是因为单核处理器同一时刻只有一个线程执行，不存在真正的"并发"。

## 线程安全问题的分类

线程安全问题可以分为几类：

**竞态条件（Race Condition）**：多个线程对共享资源的访问顺序影响了最终结果。典型的如"检查-执行"（check-then-act）模式。

**可见性问题（Visibility Problem）**：一个线程对共享变量的修改，对其他线程不可见。

**原子性问题（Atomicity Problem）**：复合操作不是原子的，被其他线程打断。

**死锁（Deadlock）**：多个线程相互等待对方持有的资源。

**活锁（Livelock）**：线程虽然没有被阻塞，但不断重复相同的操作。

**饥饿（Starvation）**：某个线程长时间无法获得所需资源。

## 线程安全的三大特性

线程安全涉及三个核心特性：

**原子性**：操作要么完全执行，要么完全不执行，不会被中断。

**可见性**：一个线程对共享变量的修改，对其他线程可见。

**有序性**：程序指令按照代码顺序执行。

这三个特性相互关联，共同决定了并发程序的安全性。
