# 哲学家就餐问题

哲学家就餐问题是并发编程中最著名的同步问题之一，由 Dijkstra 提出。它形象地描述了死锁、活锁、资源饥饿等问题，是理解多线程同步机制的经典案例。

## 问题规则

1. 哲学家要么**思考**，要么**吃饭**
2. 哲学家只能拿起**左右两边**的筷子
3. 筷子不能被抢走，必须**自愿放下**
4. 吃完后，必须**放下筷子**，其他哲学家才能使用

## 问题分析

### 为什么容易死锁？

如果每个哲学家都先拿起左边的筷子，然后等待右边的筷子，就会形成死锁：

```java
public class PhilosopherDeadlock {
    
    // 每位哲学家都先拿左边的筷子
    // 然后等待右边的筷子
    // 形成循环等待 -> 死锁
    
    public void philosopherMethod(int id) {
       筷子 left = 筷子[id];
        筷子 right = 筷子[(id + 1) % 5];
        
        while (true) {
            synchronized (left) {
                synchronized (right) {
                    eat();  // 吃饭
                }
            }
            think();  // 思考
        }
    }
}
```

## 解决方案

### 方案一：资源顺序法（推荐）

让奇数编号的哲学家先拿左边的筷子，偶数编号的先拿右边的筷子：

```java
public class PhilosopherOrderSolution {
    
    private final Object[] chopsticks;
    private final int philosopherCount;
    
    public PhilosopherOrderSolution(int count) {
        this.philosopherCount = count;
        this.chopsticks = new Object[count];
        for (int i = 0; i < count; i++) {
            chopsticks[i] = new Object();
        }
    }
    
    public void philosopherMethod(int id) {
        Object left = chopsticks[id];
        Object right = chopsticks[(id + 1) % philosopherCount];
        
        // 确保总是先拿编号小的筷子
        Object first = id < (id + 1) % philosopherCount ? left : right;
        Object second = id < (id + 1) % philosopherCount ? right : left;
        
        while (true) {
            synchronized (first) {
                synchronized (second) {
                    eat(id);
                }
            }
            think(id);
        }
    }
    
    private void eat(int id) {
        System.out.println("哲学家 " + id + " 正在吃饭");
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    private void think(int id) {
        System.out.println("哲学家 " + id + " 正在思考");
    }
}
```

### 方案二：最多允许 N-1 位哲学家同时就餐

```java
public class PhilosopherSemaphoreSolution {
    
    private final Semaphore table;
    private final Object[] chopsticks;
    
    public PhilosopherSemaphoreSolution(int count) {
        this.table = new Semaphore(count - 1);  // 最多 N-1 位哲学家
        this.chopsticks = new Object[count];
        for (int i = 0; i < count; i++) {
            chopsticks[i] = new Object();
        }
    }
    
    public void philosopherMethod(int id) {
        Object left = chopsticks[id];
        Object right = chopsticks[(id + 1) % chopsticks.length];
        
        while (true) {
            try {
                // 获取就餐桌位许可
                table.acquire();
                
                synchronized (left) {
                    synchronized (right) {
                        eat(id);
                    }
                }
                
                table.release();
                think(id);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }
    
    private void eat(int id) {
        System.out.println("哲学家 " + id + " 正在吃饭");
    }
    
    private void think(int id) {
        System.out.println("哲学家 " + id + " 正在思考");
    }
}
```

### 方案三：使用 ReentrantLock 尝试获取

```java
public class PhilosopherTryLockSolution {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition[] eating = new Condition[5];
    private final boolean[] forks = new boolean[5];
    
    public PhilosopherTryLockSolution() {
        for (int i = 0; i < 5; i++) {
            eating[i] = lock.newCondition();
        }
    }
    
    public void philosopherMethod(int id) {
        while (true) {
            try {
                takeForks(id);
                eat(id);
                putForks(id);
                think(id);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }
    
    private void takeForks(int id) throws InterruptedException {
        lock.lock();
        try {
            while (forks[id] || forks[(id + 1) % 5]) {
                eating[id].await();
            }
            forks[id] = true;
            forks[(id + 1) % 5] = true;
        } finally {
            lock.unlock();
        }
    }
    
    private void putForks(int id) {
        lock.lock();
        try {
            forks[id] = false;
            forks[(id + 1) % 5] = false;
            eating[(id + 4) % 5].signal();
            eating[(id + 1) % 5].signal();
        } finally {
            lock.unlock();
        }
    }
    
    private void eat(int id) {
        System.out.println("哲学家 " + id + " 正在吃饭");
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    private void think(int id) {
        System.out.println("哲学家 " + id + " 正在思考");
    }
}
```

### 方案四：服务员安排（Arbiter）

```java
public class PhilosopherArbiterSolution {
    
    private final Object[] chopsticks;
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition[] conditions;
    private final boolean[] forks;
    private final Condition permissionGranted;
    
    public PhilosopherArbiterSolution(int count) {
        this.chopsticks = new Object[count];
        for (int i = 0; i < count; i++) {
            chopsticks[i] = new Object();
        }
        this.conditions = new Condition[count];
        this.forks = new boolean[count];
        for (int i = 0; i < count; i++) {
            conditions[i] = lock.newCondition();
        }
        this.permissionGranted = lock.newCondition();
    }
    
    public void requestToEat(int id) throws InterruptedException {
        lock.lock();
        try {
            while (forks[id] || forks[(id + 1) % 5]) {
                conditions[id].await();
            }
            forks[id] = true;
            forks[(id + 1) % 5] = true;
        } finally {
            lock.unlock();
        }
    }
    
    public void finishEating(int id) {
        lock.lock();
        try {
            forks[id] = false;
            forks[(id + 1) % 5] = false;
            conditions[(id + 4) % 5].signal();
            conditions[(id + 1) % 5].signal();
        } finally {
            lock.unlock();
        }
    }
}
```

## 完整示例：带超时和监控

```java
public class PhilosopherDemo {
    
    private final int philosopherCount;
    private final ReentrantLock[] forks;
    private volatile boolean running = true;
    
    public PhilosopherDemo(int count) {
        this.philosopherCount = count;
        this.forks = new ReentrantLock[count];
        for (int i = 0; i < count; i++) {
            forks[i] = new ReentrantLock();
        }
    }
    
    public void start() {
        for (int i = 0; i < philosopherCount; i++) {
            final int philosopherId = i;
            new Thread(() -> philosopher(philosopherId)).start();
        }
    }
    
    public void stop() {
        running = false;
    }
    
    private void philosopher(int id) {
        while (running) {
            think(id);
            try {
                eat(id);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }
    
    private void think(int id) throws InterruptedException {
        System.out.println("哲学家 " + id + " 正在思考");
        Thread.sleep(500);
    }
    
    private void eat(int id) throws InterruptedException {
        // 获取叉子的顺序
        int left = id;
        int right = (id + 1) % philosopherCount;
        
        // 总是先获取编号小的叉子，避免死锁
        if (left > right) {
            int temp = left;
            left = right;
            right = temp;
        }
        
        // 尝试获取两把叉子
        if (forks[left].tryLock(1, TimeUnit.SECONDS)) {
            try {
                if (forks[right].tryLock(1, TimeUnit.SECONDS)) {
                    try {
                        System.out.println("哲学家 " + id + " 正在吃饭");
                        Thread.sleep(300);
                    } finally {
                        forks[right].unlock();
                    }
                }
            } finally {
                forks[left].unlock();
            }
        }
    }
    
    public static void main(String[] args) {
        PhilosopherDemo demo = new PhilosopherDemo(5);
        demo.start();
        
        // 运行 10 秒后停止
        try {
            Thread.sleep(10000);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        demo.stop();
    }
}
```

## 问题变体

### 变体1：饥饿问题

如果一位哲学家长时间占用餐位，其他哲学家可能饥饿：

```java
public class AvoidStarvation {
    
    // 解决方案：使用公平锁
    private final ReentrantLock[] fairForks;
    
    public AvoidStarvation(int count) {
        this.fairForks = new ReentrantLock[count];
        for (int i = 0; i < count; i++) {
            fairForks[i] = new ReentrantLock(true);  // 公平锁
        }
    }
}
```

### 变体2：活锁问题

如果哲学家每次都同时拿起和放下叉子，可能导致活锁：

```java
public class AvoidLivelock {
    
    // 解决方案：引入随机等待
    private void eat(int id) throws InterruptedException {
        Random random = new Random();
        Thread.sleep(random.nextInt(100));  // 随机等待时间
    }
}
```

## 总结

哲学家就餐问题展示了并发编程中的核心挑战：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 死锁 | 循环等待 | 资源顺序法 |
| 饥饿 | 某些哲学家得不到资源 | 公平锁 |
| 活锁 | 同时让步 | 随机等待 |

这个问题的解决思路可以应用到实际的系统设计中。
