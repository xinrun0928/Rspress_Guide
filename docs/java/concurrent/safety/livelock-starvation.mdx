# 活锁与饥饿

在并发编程中，死锁是最著名的问题，但并不是唯一的问题。**活锁**和**饥饿**同样是多线程环境下的常见陷阱。活锁的线程没有阻塞，但不断地重复同样的操作；饥饿的线程虽然能运行，但永远得不到所需的资源。理解这些问题，是编写高效并发程序的关键。

## 活锁（Starvation）

### 什么是活锁？

活锁是指线程虽然没有阻塞，但始终无法完成工作。因为某些原因，线程总是得不到执行的机会。

```java
public class LivelockConcept {
    
    // 活锁示例：两个线程互相让步，导致谁都执行不了
    public void livelockDemo() {
        final Object resource = new Object();
        
        Thread thread1 = new Thread(() -> {
            while (true) {
                try {
                    Thread.sleep(100);  // 让步
                    synchronized (resource) {
                        System.out.println("Thread 1 got resource");
                    }
                    break;
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        
        Thread thread2 = new Thread(() -> {
            while (true) {
                try {
                    Thread.sleep(100);  // 让步
                    synchronized (resource) {
                        System.out.println("Thread 2 got resource");
                    }
                    break;
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        
        thread1.start();
        thread2.start();
    }
}
```

### 活锁的常见场景

#### 场景1：数据库连接池饥饿

```java
public class ConnectionPoolStarvation {
    
    private final Semaphore permits = new Semaphore(2);
    
    // 长时间占用连接的线程导致其他线程饥饿
    public void badConnectionUsage() {
        try {
            permits.acquire();
            // 获取连接后，进行耗时操作
            Thread.sleep(10000);  // 占用连接 10 秒
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            permits.release();
        }
    }
}
```

#### 场景2：优先级反转

```java
public class PriorityInversion {
    
    // 高优先级线程等待低优先级线程持有的锁
    // 如果低优先级线程被其他中优先级线程抢占，可能导致高优先级线程饥饿
    private final Object lock = new Object();
    
    public void lowPriorityThread() {
        synchronized (lock) {
            // 低优先级任务
        }
    }
    
    public void highPriorityThread() {
        synchronized (lock) {
            // 高优先级任务，但可能永远等不到锁
        }
    }
}
```

## 饥饿（Starvation）

### 什么是饥饿？

饥饿是指线程需要获取某个资源，但这个资源永远被其他线程占用，导致该线程无法执行。

```java
public class ResourceStarvation {
    
    // 饥饿示例：总是有新线程插队
    public void starveDemo() {
        while (true) {
            synchronized (this) {
                // 每次都执行很短的操作
            }
            
            // 但不断有新线程加入
            if (Math.random() > 0.9) {
                new Thread(() -> {
                    synchronized (this) {
                        // 长时间占用
                        try {
                            Thread.sleep(10000);
                        } catch (InterruptedException e) {
                            Thread.currentThread().interrupt();
                        }
                    }
                }).start();
            }
        }
    }
}
```

### 饥饿的典型场景

#### 场景1：非公平锁饥饿

```java
public class UnfairLockStarvation {
    
    // 非公平锁可能导致某些线程永远得不到锁
    private final ReentrantLock unfairLock = new ReentrantLock(false);
    
    public void unfairLockUsage() {
        while (true) {
            unfairLock.lock();
            try {
                // 执行工作
            } finally {
                unfairLock.unlock();
            }
        }
    }
    
    // 解决方案：使用公平锁
    private final ReentrantLock fairLock = new ReentrantLock(true);
}
```

#### 场景2：独占资源饥饿

```java
public class ExclusiveResourceStarvation {
```

## 解决活锁的方法

### 方法一：引入随机性

```java
public class RandomBackoff {
    
    private final Object lock = new Object();
    
    public void withRandomBackoff() {
        Random random = new Random();
        
        while (true) {
            if (lock.tryLock()) {
                try {
                    // 执行操作
                    break;
                } finally {
                    lock.unlock();
                }
            } else {
                // 随机等待，避免所有线程同时重试
                int sleepTime = random.nextInt(100);
                try {
                    Thread.sleep(sleepTime);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                    break;
                }
            }
        }
    }
}
```

### 方法二：限制重试次数

```java
public class LimitedRetry {
    
    public void withRetryLimit() {
        int maxRetries = 5;
        int retries = 0;
        
        while (retries < maxRetries) {
            if (tryAcquire()) {
                try {
                    // 执行操作
                    return;
                } finally {
                    release();
                }
            }
            retries++;
            // 每次重试等待时间递增
            try {
                Thread.sleep(retries * 100);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
        
        // 超过重试次数，记录日志或降级处理
        System.out.println("获取锁失败");
    }
}
```

### 方法三：使用公平锁

```java
public class FairLockSolution {
    
    // 公平锁：按 FIFO 顺序获取锁，避免饥饿
    private final ReentrantLock fairLock = 
        new ReentrantLock(true);
    
    public void fairLockUsage() {
        fairLock.lock();
        try {
            // 执行操作
        } finally {
            fairLock.unlock();
        }
    }
}
```

### 方法二：资源分配优先级

```java
public class PriorityAllocation {
    
    // 使用优先级队列，确保资源合理分配
    private final BlockingQueue<Runnable> workQueue;
    
    public PriorityAllocation() {
        // 优先级队列，短任务优先级高
        this.workQueue = new PriorityBlockingQueue<>(11, 
            Comparator.comparingInt((Runnable r) -> {
                if (r instanceof ShortTask) return 1;
                if (r instanceof MediumTask) return 2;
                return 3;
            }));
    }
    
    static abstract class Task implements Runnable {}
    static class ShortTask extends Task {}
    static class MediumTask extends Task {}
    static class LongTask extends Task {}
}
```

### 方法三：限时等待

```java
public class TimeoutForStarvation {
    
    private final ReentrantLock lock = new ReentrantLock();
    
    public boolean tryExecute(long timeoutMs) {
        long deadline = System.currentTimeMillis() + timeoutMs;
        
        while (true) {
            if (lock.tryLock()) {
                try {
                    // 执行操作
                    return true;
                } finally {
                    lock.unlock();
                }
            }
            
            long remaining = deadline - System.currentTimeMillis();
            if (remaining <= 0) {
                return false;  // 超时
            }
            
            try {
                Thread.sleep(Math.min(100, remaining));
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                return false;
            }
        }
    }
}
```

## 三者的对比

```java
public class Comparison {
    
    // 死锁 vs 活锁 vs 饥饿
    /*
     * 死锁：线程都在等待对方释放资源，程序完全僵住
     * 活锁：线程不断让步，但谁都无法前进
     * 饥饿：某些线程永远得不到资源
     */
    
    // 死锁：需要获取多个锁，循环等待
    public void deadlock() {
        // 线程 A: lockA -> lockB
        // 线程 B: lockB -> lockA
        // 互相等待，死锁
    }
    
    // 活锁：都在主动让步，但没有人前进
    public void livelock() {
        // 线程 A: 获取锁 -> 失败 -> 让步 -> 重试
        // 线程 B: 获取锁 -> 失败 -> 让步 -> 重试
        // 始终无法前进
    }
    
    // 饥饿：某些线程得不到执行机会
    public void starvation() {
        // 线程 A: 持续占用资源
        // 线程 B: 永远等不到资源
    }
}
```

| 问题 | 特征 | 解决方案 |
|------|------|---------|
| **死锁** | 线程互相等待，完全阻塞 | 固定锁顺序、超时获取 |
| **活锁** | 线程不断让步，无法前进 | 随机退避、限制重试 |
| **饥饿** | 线程得不到资源 | 公平锁、优先级调度 |

## 总结

活锁和饥饿是比死锁更隐蔽的问题：

- **活锁**：线程不断让步，谁都无法前进
- **饥饿**：线程得不到执行机会

解决方法：
1. **引入随机性**：打破同步的步调
2. **限制重试次数**：避免无限重试
3. **使用公平锁**：按序分配资源
4. **优先级调度**：合理分配资源

理解这三种问题及其区别，才能在并发编程中游刃有余。
