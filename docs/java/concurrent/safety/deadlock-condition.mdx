# 死锁产生的四个必要条件

死锁（Deadlock）是多线程编程中最危险的问题之一。一旦发生死锁，整个程序就会僵住不动，既不能前进也不能后退。理解死锁产生的**四个必要条件**，是预防和解决死锁问题的前提。

## 什么是死锁？

死锁是指两个或多个线程相互持有对方需要的资源，导致每个线程都在等待对方释放资源，从而形成僵持的局面。

```java
public class DeadlockDefinition {
    
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    
    // 线程 A
    public void methodA() {
        synchronized (lockA) {
            System.out.println("线程 A 获取了锁 A");
            synchronized (lockB) {
                System.out.println("线程 A 获取了锁 B");
            }
        }
    }
    
    // 线程 B
    public void methodB() {
        synchronized (lockB) {
            System.out.println("线程 B 获取了锁 B");
            synchronized (lockA) {
                System.out.println("线程 B 获取了锁 A");
            }
        }
    }
}
```

## 四个必要条件

### 条件一：互斥条件（Mutual Exclusion）

资源一次只能被一个线程占用。

```java
public class MutualExclusion {
    
    // 每个锁同一时刻只能被一个线程持有
    // 其他想要获取锁的线程必须等待
    private final Object lock = new Object();
    
    public void useLock() {
        synchronized (lock) {
            // 同一时刻只有一个线程能进入这个代码块
            // 其他线程在此等待
        }
    }
    
    // 互斥条件的例子：
    // 1. synchronized 锁
    // 2. ReentrantLock
    // 3. 数据库行锁
    // 4. 文件锁
}
```

### 条件二：占有并等待（Hold and Wait）

线程在持有资源的同时，等待获取其他线程持有的资源。

```java
public class HoldAndWait {
    
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    
    // 占有并等待的例子
    // 线程先获取 lockA（占有），然后等待获取 lockB（等待其他资源）
    public void holdAndWait() {
        synchronized (lockA) {      // 占有 lockA
            // 在持有 lockA 的同时，等待 lockB
            synchronized (lockB) {  // 等待 lockB
                // 执行操作
            }
        }
    }
    
    // 改进：使用 tryLock 避免占有并等待
    public void noHoldAndWait() {
        if (lockA.tryLock()) {
            try {
                if (lockB.tryLock()) {
                    try {
                        // 执行操作
                    } finally {
                        lockB.unlock();
                    }
                }
            } finally {
                lockA.unlock();
            }
        }
    }
}
```

### 条件三：不可抢占（No Preemption）

资源不能被强制从持有它的线程中抢占。

```java
public class NoPreemption {
    
    private final ReentrantLock lock = new ReentrantLock();
    
    // synchronized 和 ReentrantLock 都是不可抢占的
    // 线程获取锁后，只能自己主动释放
    // 其他线程无法强制抢夺
    
    public void cannotPreempt() {
        lock.lock();
        try {
            // 锁只能在这里被持有
            // 其他线程无法抢占
        } finally {
            lock.unlock();  // 只能自己释放
        }
    }
    
    // 对比：LockSupport.park() 可以被 unpark() 唤醒
    // 这是一种可以"唤醒"的操作，但仍然不能强制抢占锁
}
```

### 条件四：循环等待（Circular Wait）

形成一个线程资源的循环等待链。

```java
public class CircularWait {
    
    // 循环等待：
    // 线程 A 等待线程 B 的资源
    // 线程 B 等待线程 A 的资源
    // 形成 A → B → A 的循环
    
    // 示例：
    // 线程 A: 持有 R1，等待 R2
    // 线程 B: 持有 R2，等待 R1
    // 形成 R1 ← A → R2 ← B → R1 的循环
    
    // 打破循环等待的方法：固定获取顺序
    public void fixedOrder() {
        // 总是先获取 lockA，再获取 lockB
        synchronized (lockA) {
            synchronized (lockB) {
                // 不会死锁
            }
        }
    }
}
```

## 实际案例分析

### 经典案例：转账死锁

```java
public class TransferDeadlock {
    
    // 问题代码
    public void badTransfer(Account from, Account to, int amount) {
        synchronized (from) {
            // 持有 from 的锁
            synchronized (to) {
                // 等待 to 的锁
                // 如果另一个线程以相反顺序转账，就死锁
                from.balance -= amount;
                to.balance += amount;
            }
        }
    }
    
    // 解决方案1：固定获取顺序（按 ID 排序）
    public void safeTransfer1(Account from, Account to, int amount) {
        // 获取锁的顺序
        Object first = from.id < to.id ? from : to;
        Object second = from.id < to.id ? to : from;
        
        synchronized (first) {
            synchronized (second) {
                from.balance -= amount;
                to.balance += amount;
            }
        }
    }
    
    // 解决方案2：使用 tryLock
    public boolean safeTransfer2(Account from, Account to, int amount) {
        while (true) {
            if (from.lock.tryLock(1, TimeUnit.SECONDS)) {
                try {
                    if (to.lock.tryLock(1, TimeUnit.SECONDS)) {
                        try {
                            from.balance -= amount;
                            to.balance += amount;
                            return true;
                        } finally {
                            to.lock.unlock();
                        }
                    }
                } finally {
                    from.lock.unlock();
                }
            }
            // 获取失败，随机等待后重试
            Thread.sleep(10);
        }
    }
    
    static class Account {
        int id;
        ReentrantLock lock = new ReentrantLock();
        int balance;
    }
}
```

### 案例：哲学家就餐问题

```java
public class PhilosopherDining {
    
    // 五位哲学家，五根筷子
    // 每个哲学家需要左右两根筷子才能吃饭
    // 如果同时拿起左边的筷子，可能死锁
    
    public void philosopherProblem() {
        Object[] chopsticks = new Object[5];
        for (int i = 0; i < 5; i++) {
            chopsticks[i] = new Object();
        }
        
        // 问题：每个哲学家都先拿左边的筷子
        // 然后等待右边的筷子
        // 可能导致所有人都拿着左边的，等待右边的
        // 形成循环等待 → 死锁
        
        // 解决方案：让最后一位哲学家先拿右边
        for (int i = 0; i < 5; i++) {
            final int philosopher = i;
            if (i == 4) {
                // 最后一个哲学家先拿右边
                new Thread(() -> {
                    synchronized (chopsticks[(philosopher + 1) % 5]) {
                        synchronized (chopsticks[philosopher]) {
                            eat();
                        }
                    }
                }).start();
            } else {
                new Thread(() -> {
                    synchronized (chopsticks[philosopher]) {
                        synchronized (chopsticks[(philosopher + 1) % 5]) {
                            eat();
                        }
                    }
                }).start();
            }
        }
    }
    
    private void eat() {
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 总结

死锁产生的四个必要条件：

1. **互斥条件**：资源一次只能被一个线程占用
2. **占有并等待**：线程持有资源的同时等待其他资源
3. **不可抢占**：资源不能被强制从持有者手中抢夺
4. **循环等待**：形成线程-资源的循环等待链

破坏任一条件即可避免死锁：
- 破坏互斥：使用资源分配图等
- 破坏占有并等待：一次性获取所有资源
- 破坏不可抢占：允许强制抢占
- 破坏循环等待：固定资源获取顺序

理解这四个条件，是解决死锁问题的第一步。
