# 生产者-消费者问题

生产者-消费者问题是并发编程中最经典的同步问题之一。问题的核心是：一组生产者线程生成数据，一组消费者线程处理数据，它们之间通过一个**有界缓冲区**通信。这个问题涉及线程间的同步、互斥和通信，是理解 Java 并发工具的绝佳案例。

## 问题描述

```
┌────────────────────────────────────────────────────────────────┐
│                      生产者-消费者问题                              │
│                                                                 │
│   生产者线程 ──┐                                                │
│                │  produce()                                     │
│                ▼                                                │
│   ┌────────────────────────┐                                   │
│   │      有界缓冲区          │                                   │
│   │   (生产者-消费者队列)    │                                   │
│   └────────────────────────┘                                   │
│                │  consume()                                     │
│                ▼                                                │
│   消费者线程 ──┘                                                │
│                                                                 │
│   约束：                                                        │
│   - 缓冲区满时，生产者必须等待                                    │
│   - 缓冲区空时，消费者必须等待                                    │
│   - 多个生产者/消费者可以并发访问缓冲区                            │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## 解决方案一：synchronized + wait/notify

```java
public class ProducerConsumerSync {
    
    private final int capacity = 10;
    private final Queue<Object> queue = new LinkedList<>();
    
    // 生产者
    public synchronized void produce(Object item) throws InterruptedException {
        while (queue.size() >= capacity) {
            wait();  // 缓冲区满，等待消费者
        }
        
        queue.add(item);
        System.out.println("生产: " + item);
        notifyAll();  // 通知消费者
    }
    
    // 消费者
    public synchronized Object consume() throws InterruptedException {
        while (queue.isEmpty()) {
            wait();  // 缓冲区空，等待生产者
        }
        
        Object item = queue.remove();
        System.out.println("消费: " + item);
        notifyAll();  // 通知生产者
        return item;
    }
}
```

## 解决方案二：ReentrantLock + Condition

```java
public class ProducerConsumerCondition {
    
    private final int capacity = 10;
    private final Queue<Object> queue = new LinkedList<>();
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();
    
    // 生产者
    public void produce(Object item) throws InterruptedException {
        lock.lock();
        try {
            while (queue.size() >= capacity) {
                notFull.await();  // 等待不满
            }
            
            queue.add(item);
            System.out.println("生产: " + item);
            notEmpty.signal();  // 通知非空
        } finally {
            lock.unlock();
        }
    }
    
    // 消费者
    public Object consume() throws InterruptedException {
        lock.lock();
        try {
            while (queue.isEmpty()) {
                notEmpty.await();  // 等待非空
            }
            
            Object item = queue.remove();
            System.out.println("消费: " + item);
            notFull.signal();  // 通知不满
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

## 解决方案三：BlockingQueue（推荐）

```java
public class ProducerConsumerBlockingQueue {
    
    private final BlockingQueue<Object> queue;
    
    public ProducerConsumerBlockingQueue(int capacity) {
        this.queue = new ArrayBlockingQueue<>(capacity);
    }
    
    // 生产者
    public void produce(Object item) throws InterruptedException {
        queue.put(item);  // 阻塞直到队列不满
        System.out.println("生产: " + item);
    }
    
    // 消费者
    public Object consume() throws InterruptedException {
        Object item = queue.take();  // 阻塞直到队列非空
        System.out.println("消费: " + item);
        return item;
    }
    
    // 带超时的生产
    public boolean produceWithTimeout(Object item, long timeout, TimeUnit unit) 
            throws InterruptedException {
        return queue.offer(item, timeout, unit);
    }
    
    // 带超时的消费
    public Object consumeWithTimeout(long timeout, TimeUnit unit) 
            throws InterruptedException {
        return queue.poll(timeout, unit);
    }
}
```

## 完整示例：多生产者多消费者

```java
public class MultiProducerConsumer {
    
    private final BlockingQueue<Integer> queue;
    private volatile boolean running = true;
    
    public MultiProducerConsumer(int capacity) {
        this.queue = new ArrayBlockingQueue<>(capacity);
    }
    
    // 生产者线程
    public class Producer implements Runnable {
        private final int id;
        
        public Producer(int id) {
            this.id = id;
        }
        
        @Override
        public void run() {
            int count = 0;
            while (running) {
                try {
                    int item = id * 1000 + count++;
                    queue.put(item);
                    System.out.println("生产者 " + id + " 生产: " + item);
                    Thread.sleep(100);  // 模拟生产时间
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                    break;
                }
            }
        }
    }
    
    // 消费者线程
    public class Consumer implements Runnable {
        private final int id;
        
        public Consumer(int id) {
            this.id = id;
        }
        
        @Override
        public void run() {
            while (running) {
                try {
                    Integer item = queue.poll(1, TimeUnit.SECONDS);
                    if (item != null) {
                        System.out.println("消费者 " + id + " 消费: " + item);
                        Thread.sleep(150);  // 模拟消费时间
                    }
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                    break;
                }
            }
        }
    }
    
    public void start(int producerCount, int consumerCount) {
        ExecutorService executor = Executors.newFixedThreadPool(
            producerCount + consumerCount
        );
        
        for (int i = 0; i < producerCount; i++) {
            executor.submit(new Producer(i));
        }
        
        for (int i = 0; i < consumerCount; i++) {
            executor.submit(new Consumer(i));
        }
    }
    
    public void stop() {
        running = false;
    }
}
```

```java
public class TaskDispatcher {
    
    private final BlockingQueue<Task> taskQueue = new ArrayBlockingQueue<>(100);
    
    // 提交任务
    public void submit(Task task) throws InterruptedException {
        taskQueue.put(task);

    }
```

### 场景3：消息队列

```java
public class SimpleMessageQueue<T> {
    
    private final BlockingQueue<Message<T>> queue;
    private volatile boolean running = true;
    
    public SimpleMessageQueue(int capacity) {
        this.queue = new LinkedBlockingQueue<>(capacity);
    }
    
    // 发送消息
    public void send(T data) throws InterruptedException {
        if (!running) {
            throw new IllegalStateException("Queue is closed");
        }
        queue.put(new Message<>(data));
    }
    
    // 接收消息
    public Message<T> receive() throws InterruptedException {
        return queue.take();
    }
    
    // 关闭队列
    public void close() {
        running = false;
    }
    
    static class Message<T> {
        final T data;
        final long timestamp;
        
        Message(T data) {
            this.data = data;
            this.timestamp = System.currentTimeMillis();
        }
    }
}
```

## 注意事项

### 1. 使用 offer 而不是 put（带拒绝策略）

```java
public class WithRejectionPolicy {
    
    private final BlockingQueue<Task> queue;
    
    public WithRejectionPolicy() {
        this.queue = new ArrayBlockingQueue<>(100, true, 
            (r, executor) -> {
                // 拒绝策略：记录日志
                System.err.println("任务被拒绝: " + r);
            }
        );
    }
}
```

### 2. 正确关闭队列

```java
public class GracefulShutdown {
    
    private final ExecutorService executor;
    private final BlockingQueue<Task> queue;
    
    public GracefulShutdown() {
        this.queue = new LinkedBlockingQueue<>(100);
        this.executor = Executors.newFixedThreadPool(10);
    }
    
    public void shutdown() throws InterruptedException {
        // 1. 停止接受新任务
        executor.shutdown();
        
        // 2. 等待现有任务完成
        if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
            // 3. 超时后强制停止
            executor.shutdownNow();
        }
        
        // 4. 处理队列中剩余的任务
        while (!queue.isEmpty()) {
            Task task = queue.poll();
            process(task);
        }
    }
}
```

## 总结

生产者-消费者问题的关键点：

1. **同步**：生产者等待队列不满，消费者等待队列不空
2. **互斥**：多线程访问共享缓冲区
3. **有界队列**：容量限制，需要控制生产速度

Java 解决方案对比：

| 方案 | 优点 | 缺点 |
|------|------|------|
| synchronized + wait/notify | 简单 | 不够灵活 |
| Lock + Condition | 灵活，支持多个条件 | 需要手动释放锁 |
| BlockingQueue | **推荐**，简单安全 | 功能有限 |

实际应用中，推荐使用 `BlockingQueue`，它已经帮我们处理好了所有同步问题。
