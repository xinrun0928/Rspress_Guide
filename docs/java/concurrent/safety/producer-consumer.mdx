# 生产者-消费者问题

你去奶茶店点单，看到柜台后面的店员正在悠闲地擦杯子，柜台前的队伍已经排到门口了。

这就是典型的**供需不平衡**——生产者（点单的顾客）太快，消费者（店员）太慢。

反过来，如果店员手忙脚乱地做奶茶，顾客却稀稀拉拉没人来，店员也会空闲等待。

**生产者-消费者问题**描述的就是这种场景：**一个或多个线程生产数据，放到缓冲区；一个或多个线程从缓冲区取走数据消费**。核心矛盾是——如何让生产者和消费者协调工作，既不浪费资源（缓冲区空时消费者等待），也不丢失数据（缓冲区满时生产者等待）？

## 问题模型

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   生产者 ──> [  缓冲区（队列）  ] ──> 消费者             │
│                 ↑                    ↑                   │
│                 │                    │                   │
│           放数据（满则等）        取数据（空则等）        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**关键约束**：
1. 缓冲区满时，生产者必须等待
2. 缓冲区空时，消费者必须等待
3. 缓冲区有数据时，消费者可以取
4. 缓冲区有空位时，生产者可以放

## 使用 synchronized 实现

### 最基础的版本

```java
public class BasicProducerConsumer {
    private static final int BUFFER_SIZE = 10;
    private final Object[] buffer = new Object[BUFFER_SIZE];
    private int count = 0;      // 缓冲区中的数据数量
    private int in = 0;          // 下一个放入的位置
    private int out = 0;         // 下一个取出的位置

    // 生产者调用
    public synchronized void produce(Object item) throws InterruptedException {
        // 缓冲区满，等待
        while (count == BUFFER_SIZE) {
            wait();
        }

        // 放入数据
        buffer[in] = item;
        in = (in + 1) % BUFFER_SIZE;
        count++;

        // 通知消费者
        notify();
    }

    // 消费者调用
    public synchronized Object consume() throws InterruptedException {
        // 缓冲区空，等待
        while (count == 0) {
            wait();
        }

        // 取出数据
        Object item = buffer[out];
        buffer[out] = null;  // 帮助 GC
        out = (out + 1) % BUFFER_SIZE;
        count--;

        // 通知生产者
        notify();

        return item;
    }
}
```

**为什么用 while 而不是 if？**

因为 `notify()` 唤醒后，线程从 `wait()` 处返回时，条件可能已经再次不满足了（被其他线程抢先）。必须用 while 重新检查条件。

### 完整示例

```java
public class ProducerConsumerDemo {
    public static void main(String[] args) {
        BasicProducerConsumer pc = new BasicProducerConsumer();

        // 生产者线程
        for (int i = 0; i < 3; i++) {
            final int id = i;
            new Thread(() -> {
                for (int j = 0; j < 10; j++) {
                    try {
                        String item = "Item-" + id + "-" + j;
                        pc.produce(item);
                        System.out.println("生产者" + id + " 生产: " + item);
                        Thread.sleep((long) (Math.random() * 100));
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }, "Producer-" + i).start();
        }

        // 消费者线程
        for (int i = 0; i < 2; i++) {
            final int id = i;
            new Thread(() -> {
                for (int j = 0; j < 15; j++) {
                    try {
                        Object item = pc.consume();
                        System.out.println("消费者" + id + " 消费: " + item);
                        Thread.sleep((long) (Math.random() * 200));
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }, "Consumer-" + i).start();
        }
    }
}
```

## 使用 ReentrantLock + Condition 实现

`synchronized` 的 `wait/notify` 过于底层，JDK 5 引入的 `Condition` 提供了更灵活的控制：

```java
import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.ReentrantLock;

public class ConditionProducerConsumer {
    private static final int BUFFER_SIZE = 10;
    private final Object[] buffer = new Object[BUFFER_SIZE];
    private int count = 0;
    private int in = 0;
    private int out = 0;

    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();   // 缓冲区不满
    private final Condition notEmpty = lock.newCondition();  // 缓冲区不空

    public void produce(Object item) throws InterruptedException {
        lock.lock();
        try {
            while (count == BUFFER_SIZE) {
                notFull.await();  // 等待缓冲区不满
            }

            buffer[in] = item;
            in = (in + 1) % BUFFER_SIZE;
            count++;

            notEmpty.signal();  // 通知消费者
        } finally {
            lock.unlock();
        }
    }

    public Object consume() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();  // 等待缓冲区不空
            }

            Object item = buffer[out];
            buffer[out] = null;
            out = (out + 1) % BUFFER_SIZE;
            count--;

            notFull.signal();  // 通知生产者
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

**为什么需要两个 Condition？**

| 方法 | synchronized | Condition |
| --- | --- | --- |
| 等待 | `wait()` | `await()` |
| 唤醒单个 | `notify()` | `signal()` |
| 唤醒全部 | `notifyAll()` | `signalAll()` |
| 条件分组 | 不支持 | 支持（多个 Condition） |

使用 `notifyAll()` 会有**惊群效应**——唤醒所有等待的线程，但只有一个能获取锁，其他线程又得等待。用两个 Condition 可以精确唤醒。

## 使用 BlockingQueue（推荐方式）

实际开发中，推荐直接使用 JDK 提供的 `BlockingQueue`，它已经封装好了所有逻辑：

```java
import java.util.concurrent.ArrayBlockingQueue;
import java.util.concurrent.BlockingQueue;

public class BlockingQueueDemo {
    public static void main(String[] args) {
        // 有界阻塞队列，容量 10
        BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);

        // 生产者
        for (int i = 0; i < 3; i++) {
            final int id = i;
            new Thread(() -> {
                for (int j = 0; j < 10; j++) {
                    try {
                        String item = "Item-" + id + "-" + j;
                        queue.put(item);  // 队列满时阻塞
                        System.out.println("生产者" + id + " 生产: " + item);
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }, "Producer-" + i).start();
        }

        // 消费者
        for (int i = 0; i < 2; i++) {
            final int id = i;
            new Thread(() -> {
                for (int j = 0; j < 15; j++) {
                    try {
                        String item = queue.take();  // 队列空时阻塞
                        System.out.println("消费者" + id + " 消费: " + item);
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }, "Consumer-" + i).start();
        }
    }
}
```

### BlockingQueue 的方法对照

| 方法 | 队列满时的行为 | 队列空时的行为 |
| --- | --- | --- |
| `put(e)` | 阻塞等待 | 不适用 |
| `offer(e, timeout)` | 阻塞等待，最多 timeout | 不适用 |
| `offer(e)` | 立即返回 false | 不适用 |
| `take()` | 不适用 | 阻塞等待 |
| `poll(timeout)` | 不适用 | 阻塞等待，最多 timeout |
| `poll()` | 不适用 | 立即返回 null |

### 无界阻塞队列

```java
// 无界队列，不会满，但可能导致 OOM
BlockingQueue<String> unbounded = new LinkedBlockingQueue<>();

// 指定容量的有界队列（推荐）
BlockingQueue<String> bounded = new LinkedBlockingQueue<>(100);
```

## 使用信号量实现

虽然不常见，但可以用 `Semaphore` 模拟生产者-消费者：

```java
import java.util.concurrent.Semaphore;

public class SemaphoreDemo {
    private static final int BUFFER_SIZE = 10;
    private final Object[] buffer = new Object[BUFFER_SIZE];
    private int count = 0;
    private int in = 0;
    private int out = 0;

    // empty：空槽数量，初始为 BUFFER_SIZE
    // full：产品数量，初始为 0
    private final Semaphore empty = new Semaphore(BUFFER_SIZE);
    private final Semaphore full = new Semaphore(0);
    private final Semaphore mutex = new Semaphore(1);  // 互斥访问缓冲区

    public void produce(Object item) throws InterruptedException {
        empty.acquire();  // 占用一个空槽
        mutex.acquire();  // 进入临界区

        buffer[in] = item;
        in = (in + 1) % BUFFER_SIZE;
        count++;

        mutex.release();  // 离开临界区
        full.release();   // 增加一个产品
    }

    public Object consume() throws InterruptedException {
        full.acquire();   // 消耗一个产品
        mutex.acquire();  // 进入临界区

        Object item = buffer[out];
        buffer[out] = null;
        out = (out + 1) % BUFFER_SIZE;
        count--;

        mutex.release();  // 离开临界区
        empty.release();   // 释放一个空槽

        return item;
    }
}
```

## 变体：多个生产者-多个消费者

### 场景：订单处理系统

```java
public class OrderProcessingSystem {
    private final BlockingQueue<Order> orderQueue = new LinkedBlockingQueue<>(1000);
    private final ExecutorService executor = Executors.newFixedThreadPool(10);

    // 生产者：接收订单
    public void submitOrder(Order order) {
        try {
            orderQueue.put(order);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }

    // 消费者：处理订单（多个线程并发处理）
    public void startProcessing(int consumerCount) {
        for (int i = 0; i < consumerCount; i++) {
            executor.submit(() -> {
                while (true) {
                    try {
                        Order order = orderQueue.take();
                        processOrder(order);
                    } catch (InterruptedException e) {
                        break;
                    }
                }
            });
        }
    }

    private void processOrder(Order order) {
        // 模拟处理
        try {
            Thread.sleep(100);
            System.out.println("处理订单: " + order.getId());
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 常见问题与最佳实践

### 问题一：线程安全的单例生产者

```java
// 错误的实现：多个实例
public class BadSingleton {
    private static BadSingleton instance;
    public static BadSingleton getInstance() {
        if (instance == null) {
            instance = new BadSingleton();
        }
        return instance;
    }
}

// 正确的实现：双重检查锁定
public class GoodSingleton {
    private static volatile GoodSingleton instance;
    public static GoodSingleton getInstance() {
        if (instance == null) {
            synchronized (GoodSingleton.class) {
                if (instance == null) {
                    instance = new GoodSingleton();
                }
            }
        }
        return instance;
    }
}
```

### 问题二：消费者比生产者快怎么办？

```java
// 使用 poll 带超时 + 优雅退出
public class GracefulConsumer {
    private final BlockingQueue<Task> queue;
    private volatile boolean running = true;

    public void run() {
        while (running || !queue.isEmpty()) {
            try {
                Task task = queue.poll(1, TimeUnit.SECONDS);
                if (task != null) {
                    process(task);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }

    public void stop() {
        running = false;
    }
}
```

### 最佳实践总结

```
□ 优先使用 BlockingQueue，而不是自己实现
□ 有界队列防止 OOM，无界队列慎用
□ 生产者和消费者数量可以不对称
□ 使用 take() 而非 poll() 避免空轮询
□ 考虑使用拒绝策略处理溢出
□ 优雅关闭：不再接收新任务，但处理完队列中的任务
```

## 总结

生产者-消费者是经典的线程协作问题：

- **synchronized + wait/notify**：最基础，但容易出错
- **ReentrantLock + Condition**：更灵活，支持精确唤醒
- **BlockingQueue**：推荐方式，JDK 已经封装好所有逻辑
- **Semaphore**：可以模拟，但不推荐

实际开发中，**直接使用 `BlockingQueue`**，通常配合 `ExecutorService` 使用，是最简单、最安全的做法。
