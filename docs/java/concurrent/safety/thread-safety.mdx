# 线程安全三大特性：原子性、可见性、有序性

线程安全是 Java 并发编程中最核心的概念之一。判断一段代码是否线程安全，需要从三个维度来分析：**原子性**、**可见性**和**有序性**。这三个特性相互关联，共同决定了多线程环境下程序的行为是否正确。理解它们，是编写正确并发代码的基础。

## 为什么需要线程安全？

在单线程环境下，代码的执行顺序是确定的。但多线程环境下，情况变得复杂：

```java
public class WhyThreadSafety {
    
    private int counter = 0;
    
    public void increment() {
        counter++;  // 在多线程下可能出问题
    }
    
    // 单线程：counter 每次 +1
    // 多线程：counter 可能只增加了几次
}
```

多线程导致的并发问题：
1. **竞态条件**：多个线程同时访问共享资源
2. **内存可见性问题**：线程间的数据不一致
3. **指令重排序**：执行顺序与预期不符

## 原子性：不可分割的操作

### 什么是原子性？

原子性是指操作要么全部执行，要么全部不执行，中间不会被其他线程干扰。

```java
public class AtomicityConcept {
    
    // 原子操作：不可分割
    int a = 1;           // 赋值操作
    boolean flag = true;   // 布尔赋值
    Object obj = null;    // 引用赋值
    
    // 非原子操作：可分割
    counter++;             // 读-改-写
    a = b + c;           // 读-加-写
    a += b;              // 读-改-写
}
```

### Java 中的原子操作

| 操作 | 原子性 | 原因 |
|------|--------|------|
| 基本类型赋值 | ✓ | 单条指令 |
| 引用赋值 | ✓ | 单条指令 |
| volatile 读写 | ✓ | JMM 保证 |
| CAS 操作 | ✓ | CPU 原子指令 |
| `i++` | ✗ | 读-改-写 |
| `i = i + 1` | ✗ | 读-改-写 |
| `a += b` | ✗ | 读-改-写 |

### 保证原子性的方式

```java
public class GuaranteeAtomicity {
    
    // 方式1：synchronized
    private int counter1 = 0;
    
    public synchronized void incrementSync() {
        counter1++;  // 原子操作
    }
    
    // 方式2：AtomicInteger
    private AtomicInteger counter2 = new AtomicInteger(0);
    
    public void incrementAtomic() {
        counter2.incrementAndGet();  // 原子操作
    }
    
    // 方式3：ReentrantLock
    private final ReentrantLock lock = new ReentrantLock();
    private int counter3 = 0;
    
    public void incrementLock() {
        lock.lock();
        try {
            counter3++;
        } finally {
            lock.unlock();
        }
    }
}
```

## 可见性：一个线程的修改对其他线程可见

### 什么是可见性？

可见性是指一个线程对共享变量的修改，能被其他线程及时看到。

```java
public class VisibilityProblem {
    
    private boolean ready = false;
    private int data = 0;
    
    // 线程 A
    public void writer() {
        data = 42;       // 写 data
        ready = true;   // 写 ready
    }
    
    // 线程 B
    public void reader() {
        if (ready) {           // 可能读到 true
            System.out.println(data);  // 可能读到 0！
        }
    }
}
```

### 可见性问题产生的原因

```
┌────────────────────────────────────────────────────────────────┐
│                      CPU 缓存导致的可见性问题                       │
│                                                                 │
│   线程 A                   主内存              线程 B               │
│   ┌──────────┐           ┌────────┐         ┌──────────┐       │
│   │ CPU 核心1 │           │        │         │ CPU 核心2 │       │
│   │ ┌──────┐ │           │ data=42│         │ ┌──────┐ │       │
│   │ │Cache  │ │           │ready=1 │         │ │Cache  │ │       │
│   │ │data=42 │ │ ◄───    │        │         │ │data=0  │ │       │
│   │ │ready=1 │ │  同步     │        │  同步 ──►│ │ready=0 │ │       │
│   │ └──────┘ │           │        │         │ └──────┘ │       │
│   └──────────┘           └────────┘         └──────────┘       │
│        │                                              │          │
│        │ 对缓存的修改，可能还没有同步到主内存              │          │
│        └──────────────────────────────────────────────►          │
│                      可能读到旧值                               │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 保证可见性的方式

```java
public class GuaranteeVisibility {
    
    // 方式1：volatile
    private volatile boolean ready = false;
    private int data = 0;
    
    public void writer() {
        data = 42;       // Happens-Before ready
        ready = true;   // volatile 写
    }
    
    public void reader() {
        if (ready) {    // volatile 读
            // 能看到 data = 42
        }
    }
    
    // 方式2：synchronized
    private final Object lock = new Object();
    private boolean ready2 = false;
    private int data2 = 0;
    
    public synchronized void writerSync() {
        data2 = 42;
        ready2 = true;
    }
    
    public synchronized void readerSync() {
        if (ready2) {
            // 能看到 data2 = 42
        }
    }
    
    // 方式3：final
    private final int[] array = new int[10];  // 构造函数中初始化
    
    // final 字段在构造函数完成后对其他线程可见
}
```

## 有序性：程序执行的顺序

### 什么是有序性？

有序性是指程序执行的顺序与代码的书写顺序一致。但由于编译器和 CPU 的优化，指令可能被重排序。

```java
public class OrderingProblem {
    
    private int a = 0;
    private int b = 0;
    
    public void method() {
        a = 1;    // 可能与下面一行重排序
        b = 2;
    }
}
```

### 重排序的类型

| 类型 | 发生层级 | 说明 |
|------|---------|------|
| **编译器重排序** | 编译器 | 编译器调整指令顺序 |
| **指令级重排序** | CPU | CPU 调整指令顺序 |
| **内存系统重排序** | CPU/Cache | CPU 缓存导致的可见性问题 |

### 有序性问题示例

```java
public class OrderingIssue {
    
    private int x = 0;
    private int y = 0;
    private boolean ready = false;
    
    // 线程 A
    public void writer() {
        x = 1;         // 指令 1
        y = 2;         // 指令 2
        ready = true;  // 指令 3
    }
    
    // 线程 B
    public void reader() {
        if (ready) {  // 读取 ready
            // x 和 y 可能都是默认值
            System.out.println("x = " + x + ", y = " + y);
        }
    }
}
```

### 保证有序性的方式

```java
public class GuaranteeOrdering {
    
    // 方式1：synchronized
    private final Object lock = new Object();
    private int x = 0;
    private int y = 0;
    
    public synchronized void writer() {
        x = 1;
        y = 2;
    }
    
    public synchronized void reader() {
        if (x == 1) {
            // x 和 y 的写入一定已完成
        }
    }
    
    // 方式2：volatile
    private volatile boolean ready = false;
    
    public void writerVolatile() {
        x = 1;         // volatile 前的写
        y = 2;         // volatile 前的写
        ready = true;  // volatile 写，禁止与前面的写重排序
    }
}
```

## Happens-Before 规则

Java 内存模型（JMM）通过 Happens-Before 规则来定义可见性：

```java
public class HappensBeforeRules {
    
    // 1. 程序顺序规则
    // 同一线程中，前面的操作 Happens-Before 后面的操作
    int a = 1;  // Happens-Before
    int b = a + 1;
    
    // 2. 监视器锁规则
    // 解锁 Happens-Before 后续对这个锁的加锁
    synchronized (lock) {
        a = 2;  // Happens-Before
    }          // 解锁
    
    synchronized (lock) {
        // 能看到 a = 2
    }
    
    // 3. volatile 规则
    // volatile 写 Happens-Before volatile 读
    volatile boolean ready = false;
    
    // 4. 线程启动规则
    // Thread.start() Happens-Before 被启动线程中的任何操作
    thread.start();  // Happens-Before
    // 线程内能看到主线程的所有操作
    
    // 5. 传递性
    // A Happens-Before B，B Happens-Before C，则 A Happens-Before C
}
```

## 实战分析

### 分析一个方法是否线程安全

```java
public class ThreadSafetyAnalysis {
    
    // 示例1：不安全
    private int counter = 0;
    
    public void increment() {
        counter++;  // 非原子操作
        // 问题：非原子性
    }
    
    // 示例2：安全（局部变量）
    public void safeLocal() {
        int local = 0;  // 局部变量，线程私有
        local++;
        // 安全：每个线程有自己的 local 副本
    }
    
    // 示例3：安全（不可变对象）
    private final String name = "immutable";  // 不可变对象
    // 安全：不可变对象天然线程安全
    
    // 示例4：安全（volatile）
    private volatile boolean active = true;
    
    public void setActive(boolean value) {
        active = value;  // volatile 保证可见性
    }
    
    public boolean isActive() {
        return active;  // volatile 保证可见性
    }
}
```

## 总结

线程安全的三大特性：

1. **原子性**：操作不可分割
   - 保证方式：`synchronized`、`Lock`、`Atomic*`
   
2. **可见性**：一个线程的修改对其他线程可见
   - 保证方式：`volatile`、`synchronized`、`final`

3. **有序性**：操作按照预期的顺序执行
   - 保证方式：`volatile`、`synchronized`

理解这三个特性及其相互关系，是分析线程安全问题和选择正确同步手段的基础。
