# 死锁定位与排查方法

死锁一旦发生，程序就会僵住不动。定位和排查死锁是每个 Java 开发者必须掌握的技能。JDK 提供了一系列工具帮助我们检测和定位死锁，从简单的命令行工具到专业的分析软件，让我们一步步揭开死锁的神秘面纱。

## 死锁的典型症状

```java
public class DeadlockSymptoms {
    
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    
    public void method1() {
        synchronized (lockA) {
            try {
                Thread.sleep(1000);  // 模拟耗时操作
                synchronized (lockB) {
                    System.out.println("Method1 done");
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
    
    public void method2() {
        synchronized (lockB) {
            try {
                Thread.sleep(1000);
                synchronized (lockA) {
                    System.out.println("Method2 done");
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
    
    public static void main(String[] args) {
        DeadlockSymptoms demo = new DeadlockSymptoms();
        
        // 线程 A
        new Thread(demo::method1, "Thread-A").start();
        
        // 线程 B
        new Thread(demo::method2, "Thread-B").start();
        
        // 程序运行后，会发现两个线程都没有输出
        // 程序僵住，无法退出
    }
}
```

## 方法一：使用 jstack 排查

`jstack` 是 JDK 自带的线程堆栈分析工具，可以生成 Java 进程的线程堆栈快照。

### 基本用法

```bash
# 1. 找到 Java 进程的 PID
jps -l

# 2. 使用 jstack 生成线程堆栈
jstack <pid>

# 3. 查看死锁信息
jstack -l <pid>
```

## 方法五：程序化检测

### 使用 ThreadMXBean

```java
public class ProgrammaticDeadlockDetection {
    
    public static void main(String[] args) throws InterruptedException {
        // 获取 ThreadMXBean
        ThreadMXBean threadMXBean = ManagementFactory.getThreadMXBean();
        
        // 创建死锁检测线程
        Thread detectionThread = new Thread(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                // 检测死锁
                long[] deadlockedThreads = threadMXBean.findDeadlockedThreads();
                
                if (deadlockedThreads != null) {
                    System.out.println("检测到死锁！线程 IDs: " + 
                        Arrays.toString(deadlockedThreads));
                    
                    // 获取线程信息
                    ThreadInfo[] infos = threadMXBean.getThreadInfo(deadlockedThreads);
                    for (ThreadInfo info : infos) {
                        System.out.println("死锁线程: " + info.getThreadName());
                        System.out.println("等待锁: " + info.getLockName());
                        System.out.println("持有锁: " + info.getLockOwnerName());
                    }
                }
                
                try {
                    Thread.sleep(5000);  // 每5秒检测一次
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                    break;
                }
            }
        });
        
        detectionThread.setDaemon(true);
        detectionThread.start();
        
        // 运行测试代码...
        runDeadlockTest();
    }
    
    private static void runDeadlockTest() throws InterruptedException {
        Object lockA = new Object();
        Object lockB = new Object();
        
        Thread t1 = new Thread(() -> {
            synchronized (lockA) {
                try {
                    Thread.sleep(100);
                    synchronized (lockB) {
                        System.out.println("Thread 1 done");
                    }
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        
        Thread t2 = new Thread(() -> {
            synchronized (lockB) {
                try {
                    Thread.sleep(100);
                    synchronized (lockA) {
                        System.out.println("Thread 2 done");
                    }
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        
        t1.start();
        t2.start();
        t1.join();
        t2.join();
    }
}
```

### 使用 ReentrantLock 的 tryLock 超时

```java
public class TimeoutDetection {
    
    private final ReentrantLock lockA = new ReentrantLock();
    private final ReentrantLock lockB = new ReentrantLock();
    
    public void method1() {
        try {
            if (lockA.tryLock(1, TimeUnit.SECONDS)) {
                try {
                    if (lockB.tryLock(1, TimeUnit.SECONDS)) {
                        try {
                            System.out.println("Method1 done");
                        } finally {
                            lockB.unlock();
                        }
                    } else {
                        System.out.println("获取 lockB 超时");
                    }
                } finally {
                    lockA.unlock();
                }
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    public void method2() {
        try {
            if (lockB.tryLock(1, TimeUnit.SECONDS)) {
                try {
                    if (lockA.tryLock(1, TimeUnit.SECONDS)) {
                        try {
                            System.out.println("Method2 done");
                        } finally {
                            lockA.unlock();
                        }
                    } else {
                        System.out.println("获取 lockA 超时");
                    }
                } finally {
                    lockB.unlock();
                }
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

### 日志解读要点

```
解读要点：
1. "waiting to lock" - 正在等待的锁
2. "locked" - 当前持有的锁
3. 行号定位 - 可以定位到具体代码行
4. 对象地址 - 0x00000007af800e90 标识锁对象
```

## 常见死锁模式

### 模式1：嵌套锁

```java
public class NestedLock {
    public synchronized void methodA() {
        methodB();  // 在持有锁的情况下调用
    }
    
    public synchronized void methodB() {
        // 可重入，所以不会死锁
    }
}

public class NestedLockFixed {
    public void methodA() {
        synchronized (this) {
            methodB();  // 安全
        }
    }
    
    public void methodB() {
        synchronized (this) {
            // 安全
        }
    }
}
```

### 模式2：多对象锁

```java
public class MultiObjectLock {
    Object lock1 = new Object();
    Object lock2 = new Object();
    
    public void method1() {
        synchronized (lock1) {
            synchronized (lock2) {
                // 操作
            }
        }
    }
    
    public void method2() {
        synchronized (lock2) {  // 不同顺序
            synchronized (lock1) {
                // 可能死锁
            }
        }
    }
}
```
