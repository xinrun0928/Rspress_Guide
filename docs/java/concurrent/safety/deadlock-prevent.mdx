# 如何避免死锁：破坏四大条件

理解了死锁产生的四个必要条件（互斥、占有并等待、不可抢占、循环等待），我们就可以针对性地破坏其中任何一个条件，从而有效地避免死锁。本文将详细介绍各种避免死锁的方法和最佳实践。

## 预防死锁的策略

### 策略一：破坏互斥条件

互斥条件是资源的固有属性，很难被破坏。但有些场景下可以绕过：

```java
public class BreakMutualExclusion {
    
    // 方案1：使用无锁数据结构
    // 不是避免互斥，而是使用无锁算法替代互斥
    private AtomicInteger counter = new AtomicInteger(0);
    
    public void increment() {
        counter.incrementAndGet();  // 无锁
    }
    
    // 方案2：使用读写锁分离
    // 读操作可以并发，只有写操作需要互斥
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
    
    public void read() {
        rwLock.readLock().lock();
        try {
            // 读操作
        } finally {
            rwLock.readLock().unlock();
        }
    }
    
    public void write() {
        rwLock.writeLock().lock();
        try {
            // 写操作
        } finally {
            rwLock.writeLock().unlock();
        }
    }
}
```

### 策略二：破坏占有并等待

#### 方案1：一次性获取所有资源

```java
public class AcquireAllAtOnce {
    
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    
    // 方案1.1：使用 ReentrantLock 的 tryLock
    private final ReentrantLock lock1 = new ReentrantLock();
    private final ReentrantLock lock2 = new ReentrantLock();
    
    public void transfer() {
        while (true) {
            // 一次性尝试获取所有锁
            if (lock1.tryLock()) {
                try {
                    if (lock2.tryLock()) {
                        try {
                            // 执行转账操作
                            doTransfer();
                            return;  // 成功，退出
                        } finally {
                            lock2.unlock();
                        }
                    }
                } finally {
                    lock1.unlock();
                }
            }
            // 获取失败，随机等待后重试
            LockSupport.parkNanos(1000);
        }
    }
    
    private void doTransfer() {
        // 转账操作
    }
}
```

#### 方案2：释放持有的资源再等待

```java
public class ReleaseBeforeWait {
    
    // 方案2：先释放已持有的资源，等待后再重新获取
    // 这种方式可能导致活锁
    
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    
    public void methodWithRetry() {
        boolean acquiredA = false;
        boolean acquiredB = false;
        
        while (!acquiredA || !acquiredB) {
            if (!acquiredA) {
                if (tryAcquire(lockA)) {
                    acquiredA = true;
                }
            }
            
            if (!acquiredB) {
                if (tryAcquire(lockB)) {
                    acquiredB = true;
                }
            }
            
            if (!acquiredA || !acquiredB) {
                // 释放已获取的锁
                if (acquiredA) {
                    lockA.release();
                    acquiredA = false;
                }
                if (acquiredB) {
                    lockB.release();
                    acquiredB = false;
                }
                // 等待后重试
                Thread.sleep(10);
            }
        }
        
        try {
            // 执行操作
        } finally {
            lockA.release();
            lockB.release();
        }
    }
}
```

### 策略三：破坏不可抢占

使用能够响应中断的锁获取方式：

```java
public class BreakNoPreemption {
    
    private final ReentrantLock lockA = new ReentrantLock();
    private final ReentrantLock lockB = new ReentrantLock();
    
    // 使用 lockInterruptibly 可以响应中断
    public void methodWithInterrupt() throws InterruptedException {
        try {
            lockA.lockInterruptibly();
            try {
                lockB.lockInterruptibly();
                try {
                    // 执行操作
                } finally {
                    lockB.unlock();
                }
            } finally {
                lockA.unlock();
            }
        } catch (InterruptedException e) {
            // 可以处理中断
            Thread.currentThread().interrupt();
            throw e;
        }
    }
    
    // 使用 tryLock(timeout)
    public boolean methodWithTimeout() {
        try {
            if (lockA.tryLock(5, TimeUnit.SECONDS)) {
                try {
                    if (lockB.tryLock(5, TimeUnit.SECONDS)) {
                        try {
                            // 执行操作
                            return true;
                        } finally {
                            lockB.unlock();
                        }
                    }
                } finally {
                    lockA.unlock();
                }
            }
            return false;
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            return false;
        }
    }
}
```

### 策略四：破坏循环等待（推荐）

#### 固定锁获取顺序

这是最常用也最有效的方案：

```java
public class FixedLockOrder {
    
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    
    // 始终按照固定顺序获取锁
    public void method1() {
        // 始终先获取 lockA，再获取 lockB
        synchronized (lockA) {
            synchronized (lockB) {
                // 操作
            }
        }
    }
    
    public void method2() {
        // 同样先获取 lockA，再获取 lockB
        synchronized (lockA) {
            synchronized (lockB) {
                // 操作
            }
        }
    }
}
```

#### 使用锁排序

```java
public class LockOrdering {
    
    // 基于对象的 hashCode 或 ID 确定顺序
    public void transfer(Account from, Account to, int amount) {
        // 确保始终按 ID 大小顺序获取锁
        Object first = from.id < to.id ? from : to;
        Object second = from.id < to.id ? to : from;
        
        synchronized (first) {
            synchronized (second) {
                from.balance -= amount;
                to.balance += amount;
            }
        }
    }
    
    static class Account {
        int id;
        int balance;
    }
}
```

## 最佳实践

### 实践一：缩小锁的范围

```java
public class NarrowLockScope {
    
    private final Object dataLock = new Object();
    private volatile int data;
    
    // 糟糕：锁范围太大
    public void badPattern() {
        synchronized (dataLock) {
            // 读取数据
            int temp = data;
            // 计算
            temp = temp * 2;
            // 其他耗时操作
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            // 保存结果
            data = temp;
        }
    }
    
    // 改进：只锁必要的操作
    public void goodPattern() {
        int temp;
        synchronized (dataLock) {
            temp = data;  // 只在必要时加锁
        }
        
        // 耗时计算不需要锁
        temp = temp * 2;
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        
        synchronized (dataLock) {
            data = temp;  // 只在写回时加锁
        }
    }
}
```

### 实践二：避免嵌套锁

```java
public class AvoidNestedLocks {
    
    private final Object outerLock = new Object();
    private final Object innerLock = new Object();
    
    // 糟糕：嵌套锁
    public void nestedLock() {
        synchronized (outerLock) {
            synchronized (innerLock) {
                // 操作
            }
        }
    }
    
    // 改进：提取为单独方法，使用可重入特性
    public void improvedLock() {
        synchronized (outerLock) {
            doInnerWork();  // innerLock 在这个方法里获取
        }
    }
    
    private synchronized void doInnerWork() {
        // 持有 this 锁
        // 但在 improvedLock 已持有 outerLock 的情况下
        // 这里应该获取 innerLock 而不是 this
    }
}
```

### 实践三：使用高级并发工具

```java
public class UseHighLevelTools {
    
    // 使用 ConcurrentHashMap 代替 synchronized HashMap
    private final Map<String, Object> map = new ConcurrentHashMap<>();
    
    public void put(String key, Object value) {
        map.put(key, value);  // 线程安全
    }
    
    // 使用 AtomicReference 代替锁
    private final AtomicReference<Counter> counterRef = 
        new AtomicReference<>(new Counter(0));
    
    public void increment() {
        while (true) {
            Counter current = counterRef.get();
            Counter next = new Counter(current.value + 1);
            if (counterRef.compareAndSet(current, next)) {
                return;
            }
        }
    }
    
    static class Counter {
        final int value;
        Counter(int value) {
            this.value = value;
        }
    }
}
```

### 实践四：设置锁超时

```java
public class LockTimeout {
    
    private final ReentrantLock lock = new ReentrantLock();
    
    public void methodWithTimeout() {
        try {
            // 尝试获取锁，最多等待 5 秒
            if (lock.tryLock(5, TimeUnit.SECONDS)) {
                try {
                    // 执行操作
                } finally {
                    lock.unlock();
                }
            } else {
                // 获取锁超时，可以记录日志或采取其他措施
                System.out.println("获取锁超时");
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 总结

避免死锁的策略：

| 条件 | 破坏方式 | 适用场景 |
|------|---------|---------|
| 互斥 | 无锁数据结构、读写锁 | 读多写少 |
| 占有并等待 | 一次性获取所有锁、释放再等待 | 资源数量有限 |
| 不可抢占 | tryLock 超时、lockInterruptibly | 需要响应中断 |
| 循环等待 | **固定锁顺序（推荐）** | 任何场景 |

最佳实践：
1. **固定锁获取顺序**（最简单有效）
2. 缩小锁的范围
3. 避免嵌套锁
4. 使用高级并发工具
5. 设置锁超时
