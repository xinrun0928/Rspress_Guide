# AtomicStampedReference：解决 ABA 问题

前面我们讲了 CAS 的 ABA 问题——一个变量的值从 A 变成 B，又变回 A，CAS 认为是「没变化」，但实际上中间发生了修改。

现在的问题是：**怎么让 CAS 知道「变化了多少次」？**

答案就是 `AtomicStampedReference`。

## ABA 问题的严重性

先看一个实际场景：

```java
// 栈数据结构（简化版）
class Node {
    int value;
    Node next;
}

// 无锁栈
class LockFreeStack {
    private AtomicReference<Node> top = new AtomicReference<>();

    public void push(Node node) {
        Node oldTop;
        do {
            oldTop = top.get();
            node.next = oldTop;
        } while (!top.compareAndSet(oldTop, node));  // ABA 问题！
    }

    public Node pop() {
        Node oldTop, newTop;
        do {
            oldTop = top.get();
            if (oldTop == null) return null;
            newTop = oldTop.next;
        } while (!top.compareAndSet(oldTop, newTop));  // ABA 问题！
        return oldTop;
    }
}
```

**ABA 问题的危险**：

```
初始状态：A -> B -> C

线程1：准备 pop A
      读取 top = A
      读取 A.next = B

线程2：pop A，pop B，push D（变成 D -> C）
      pop C，push A（变成 A -> C）
      // 此时栈：A -> C

线程1：CAS(top, A, B)，成功！
      // 但 B.next 已经是 null 了！
      // 导致 C 节点丢失
```

## AtomicStampedReference 的设计

### 核心思想：版本号

`AtomicStampedReference` 为每次修改关联一个**版本号（stamp）**：

```java
public class AtomicStampedReference<V> {
    // 内部使用 Pair 存储值和版本号
    private static class Pair<T> {
        final T reference;
        final int stamp;
        private Pair(T reference, int stamp) {
            this.reference = reference;
            this.stamp = stamp;
        }
    }

    private volatile Pair<V> pair;
}
```

CAS 操作不仅比较值，还比较版本号：

```java
public boolean compareAndSet(V expectedReference,
                             V newReference,
                             int expectedStamp,
                             int newStamp) {
    Pair<V> current = pair;
    return
        expectedReference == current.reference &&
        expectedStamp == current.stamp &&           // 版本号也要匹配！
        ((newReference == current.reference &&
          newStamp == current.stamp) ||
         casPair(current, Pair.of(newReference, newStamp)));
}
```

### 基本用法

```java
public class StampedReferenceDemo {
    public static void main(String[] args) {
        // 初始值：reference = "A"，版本号 = 1
        AtomicStampedReference<String> ref =
            new AtomicStampedReference<>("A", 1);

        // 获取当前版本号
        int[] stamp = new int[1];
        String value = ref.get(stamp);
        System.out.println("初始值: " + value + ", 版本号: " + stamp[0]);

        // 正确更新：值变为 B，版本号变为 2
        ref.compareAndSet("A", "B", stamp[0], stamp[0] + 1);
        value = ref.get(stamp);
        System.out.println("更新后: " + value + ", 版本号: " + stamp[0]);

        // 错误的更新：版本号不匹配，失败
        boolean success = ref.compareAndSet("A", "C", 1, 2);
        System.out.println("使用旧版本号更新: " + success);  // false

        // 正确的更新
        success = ref.compareAndSet("B", "C", stamp[0], stamp[0] + 1);
        System.out.println("使用正确版本号更新: " + success);  // true
    }
}
```

## ABA 问题的解决

### 修复无锁栈

```java
class SafeNode {
    int value;
    SafeNode next;
}

class SafeLockFreeStack {
    // Stamp = 更新次数
    private AtomicStampedReference<SafeNode> top =
        new AtomicStampedReference<>(null, 0);

    public void push(SafeNode node) {
        SafeNode oldTop;
        int[] stamp = new int[1];

        do {
            oldTop = top.get(stamp);
            node.next = oldTop;
        } while (!top.compareAndSet(oldTop, node, stamp[0], stamp[0] + 1));
    }

    public SafeNode pop() {
        SafeNode oldTop, newTop;
        int[] stamp = new int[1];

        do {
            oldTop = top.get(stamp);
            if (oldTop == null) return null;

            newTop = oldTop.next;
            // 版本号不匹配会失败，防止 ABA
        } while (!top.compareAndSet(oldTop, newTop, stamp[0], stamp[0] + 1));

        return oldTop;
    }
}
```

### 模拟 ABA 场景

```java
public class ABASolutionDemo {
    public static void main(String[] args) {
        AtomicStampedReference<String> ref =
            new AtomicStampedReference<>("A", 1);

        // 线程1：准备修改
        new Thread(() -> {
            int[] stamp = new int[1];
            String value = ref.get(stamp);
            System.out.println("线程1读取: " + value + ", 版本号: " + stamp[0]);

            try { Thread.sleep(100); } catch (InterruptedException ignored) {}

            // 尝试 CAS（使用旧版本号）
            boolean success = ref.compareAndSet(value, "B", stamp[0], stamp[0] + 1);
            System.out.println("线程1 CAS结果: " + success + " (版本号过期)");
        }).start();

        // 线程2：A -> B -> A
        new Thread(() -> {
            int[] stamp = new int[1];
            ref.get(stamp);

            ref.compareAndSet("A", "B", stamp[0], stamp[0] + 1);
            System.out.println("线程2: A -> B");

            ref.get(stamp);
            ref.compareAndSet("B", "A", stamp[0], stamp[0] + 1);
            System.out.println("线程2: B -> A");
        }).start();
    }
}
```

输出：

```
线程1读取: A, 版本号: 1
线程2: A -> B
线程2: B -> A
线程1 CAS结果: false (版本号过期)  // 版本号变成 2 了，CAS 失败！
```

## AtomicMarkableReference：更轻量的版本号

有时候我们不需要完整的版本号，只需要一个**布尔标记**（比如「已删除」）。

`AtomicMarkableReference` 就是为此设计的：

```java
public class AtomicMarkableReference<T> {
    private static class Pair<T> {
        final T reference;
        final boolean mark;
        // ...
    }

    // compareAndSet 只比较 mark，不比较完整版本号
    public boolean compareAndSet(V expectedReference,
                                  V newReference,
                                  boolean expectedMark,
                                  boolean newMark) {
        // ...
    }
}
```

### 典型应用：逻辑删除

```java
class Node {
    int value;
    AtomicMarkableReference<Node> next;
}

class LogicalDeleteStack {
    private AtomicMarkableReference<Node> top =
        new AtomicMarkableReference<>(null, false);

    public void markAsDeleted(Node node) {
        // 只改变 mark，不改变 reference
        // ...
    }
}
```

### AtomicStampedReference vs AtomicMarkableReference

| 特性 | AtomicStampedReference | AtomicMarkableReference |
| --- | --- | --- |
| 版本信息 | int 版本号 | boolean 标记 |
| 内存开销 | 稍大 | 更小 |
| 适用场景 | 需要知道修改次数 | 只需要二元状态（已删除/未删除） |
| 示例 | 栈、队列 | 逻辑删除、标记移除 |

## 手写简易版本

理解原理后，我们可以实现一个简化版：

```java
public class SimpleStampedReference<T> {
    private volatile Pair<T> pair;

    private static class Pair<T> {
        final T reference;
        final int stamp;

        Pair(T reference, int stamp) {
            this.reference = reference;
            this.stamp = stamp;
        }

        static <T> Pair<T> of(T reference, int stamp) {
            return new Pair<>(reference, stamp);
        }
    }

    public SimpleStampedReference(T initialRef, int initialStamp) {
        pair = Pair.of(initialRef, initialStamp);
    }

    public T getReference() {
        return pair.reference;
    }

    public int getStamp() {
        return pair.stamp;
    }

    public T get(int[] stampHolder) {
        stampHolder[0] = pair.stamp;
        return pair.reference;
    }

    public boolean compareAndSet(T expectedReference,
                                  T newReference,
                                  int expectedStamp,
                                  int newStamp) {
        Pair<T> current = pair;
        return
            expectedReference == current.reference &&
            expectedStamp == current.stamp &&
            ((newReference == current.reference &&
              newStamp == current.stamp) ||
             casPair(current, Pair.of(newReference, newStamp)));
    }

    private boolean casPair(Pair<T> expect, Pair<T> update) {
        if (pair == expect) {
            pair = update;
            return true;
        }
        return false;
    }
}
```

## 总结

`AtomicStampedReference` 和 `AtomicMarkableReference` 是解决 ABA 问题的利器：

- **核心机制**：为每次修改关联一个版本号或标记
- **CAS 语义**：不仅比较值，还要比较版本号/标记
- **应用场景**：无锁数据结构（栈、队列）、需要版本追踪的场景
- **选择建议**：只需要二元状态用 `AtomicMarkableReference`，需要计数用 `AtomicStampedReference`

理解这两个类的设计思想，对于实现无锁算法和理解 Java 并发工具箱都有重要意义。
