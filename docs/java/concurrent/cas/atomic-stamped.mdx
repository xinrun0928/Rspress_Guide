# AtomicStampedReference：解决 ABA 问题

import { Badge } from '@rspress/core/theme';

<Badge type="info" value="JDK 1.5+" />

ABA 问题是 CAS 操作的一个经典陷阱。当你使用 AtomicReference 时，可能遇到这样的场景：值从 A 变成 B 又变回 A，但你的 CAS 仍然成功了。

**AtomicStampedReference 就是来解决这个问题的**——它不仅比较值，还比较版本号。

## 为什么需要版本号

### ABA 问题的场景

```
没有版本号：
初始：A → 线程A读取A
      B → 线程B修改A→B
      A → 线程B修改B→A（变回来了）
      C → 线程A的CAS(A→C)成功！← 不知道中间经历了什么

有版本号：
初始：A, v1 → 线程A读取(A, v1)
      B, v2 → 线程B修改(A→B, v1→v2)
      A, v3 → 线程B修改(B→A, v2→v3)  ← 版本号不同了！
      A, v1 → 线程A的CAS(A→C, v1)失败！
```

## AtomicStampedReference 详解

### 基本结构

```java
public class AtomicStampedReference<V> {
    // Pair 类存储值和版本号
    private static class Pair<T> {
        final T reference;   // 值
        final int stamp;     // 版本号（时间戳）
        // ...
    }

    private volatile Pair<V> pair;
}
```

### 核心方法

```java
// 获取当前值和版本号
public V get(int[] stampHolder) {
    Pair<V> pair = this.pair;
    stampHolder[0] = pair.stamp;
    return pair.reference;
}

// 比较并设置
public boolean compareAndSet(
    V expectedReference,  // 期望值
    V newReference,       // 新值
    int expectedStamp,    // 期望版本号
    int newStamp         // 新版本号
) {
    Pair<V> current = pair;
    // 只有值和版本号都匹配时才更新
    return expectedReference == current.reference &&
           expectedStamp == current.stamp &&
           ((newReference == current.reference &&
             newStamp == current.stamp) ||
            casPair(current, Pair.of(newReference, newStamp)));
}
```

## 使用示例

### 基本用法

```java
import java.util.concurrent.atomic.AtomicStampedReference;

public class StampedDemo {
    private AtomicStampedReference<Integer> ref = 
        new AtomicStampedReference<>(100, 0);

    public void demo() {
        int[] stamp = new int[1];

        // 获取当前值和版本号
        Integer value = ref.get(stamp);
        System.out.println("值: " + value + ", 版本号: " + stamp[0]);

        // CAS 更新
        boolean success = ref.compareAndSet(100, 200, stamp[0], stamp[0] + 1);
        System.out.println("更新成功: " + success);

        // 再次获取
        value = ref.get(stamp);
        System.out.println("新值: " + value + ", 新版本号: " + stamp[0]);
    }
}
```

### 模拟 ABA 问题场景

```java
import java.util.concurrent.atomic.AtomicStampedReference;

public class ABAWithStamp {
    private AtomicStampedReference<String> ref = 
        new AtomicStampedReference<>("A", 0);

    public void abaScenario() throws InterruptedException {
        int[] stamp = new int[1];

        // 线程A：读取初始值
        String initial = ref.get(stamp);
        int initialStamp = stamp[0];
        System.out.println("线程A读取: " + initial + ", 版本号: " + initialStamp);

        // 线程B：修改 A → B → A
        Thread threadB = new Thread(() -> {
            int[] bStamp = new int[1];
            ref.get(bStamp);

            // A → B
            ref.compareAndSet("A", "B", bStamp[0], bStamp[0] + 1);
            System.out.println("线程B: A → B");

            // B → A
            ref.get(bStamp);
            ref.compareAndSet("B", "A", bStamp[0], bStamp[0] + 1);
            System.out.println("线程B: B → A, 版本号: " + (bStamp[0] + 1));
        });

        threadB.start();
        threadB.join();

        // 线程A：尝试 CAS，版本号不匹配会失败
        boolean success = ref.compareAndSet("A", "C", initialStamp, initialStamp + 1);
        System.out.println("线程A CAS结果: " + success);
        // 即使值相同（都是A），但版本号已变，CAS失败！
    }
}
```

## 实战：实现线程安全的计数器

### 带历史记录的计数器

```java
import java.util.concurrent.atomic.AtomicStampedReference;

public class HistoryCounter {
    private final AtomicStampedReference<Long> value = 
        new AtomicStampedReference<>(0L, 0);
    private final AtomicStampedReference<Long> minValue = 
        new AtomicStampedReference<>(Long.MAX_VALUE, 0);
    private final AtomicStampedReference<Long> maxValue = 
        new AtomicStampedReference<>(Long.MIN_VALUE, 0);

    public void increment() {
        while (true) {
            long[] holder = new long[2];
            Long current = value.get(holder);
            int stamp = (int) holder[1];

            long[] minHolder = new long[2];
            Long currentMin = minValue.get(minHolder);
            int minStamp = (int) minHolder[1];

            long[] maxHolder = new long[2];
            Long currentMax = maxValue.get(maxHolder);
            int maxStamp = (int) maxHolder[1];

            if (value.compareAndSet(current, current + 1, stamp, stamp + 1) &&
                (current + 1 < currentMin || 
                 minValue.compareAndSet(currentMin, current + 1, minStamp, minStamp + 1)) &&
                (current + 1 > currentMax || 
                 maxValue.compareAndSet(currentMax, current + 1, maxStamp, maxStamp + 1))) {
                return;
            }
        }
    }

    public long get() {
        return value.getReference();
    }

    public long getMin() {
        return minValue.getReference();
    }

    public long getMax() {
        return maxValue.getReference();
    }
}
```

### 乐观锁的版本控制

```java
import java.util.concurrent.atomic.AtomicStampedReference;

public class OptimisticEntity {
    private final AtomicStampedReference<VersionedData> dataRef = 
        new AtomicStampedReference<>(new VersionedData("initial", 1), 0);

    public boolean update(String newValue) {
        while (true) {
            VersionedData[] holder = new VersionedData[1];
            int stamp = dataRef.get(holder);
            VersionedData current = holder[0];

            VersionedData newData = new VersionedData(newValue, current.version + 1);
            
            if (dataRef.compareAndSet(current, newData, stamp, stamp + 1)) {
                return true;
            }
            // 更新失败，自旋重试
        }
    }

    public VersionedData get() {
        return dataRef.getReference();
    }

    public static class VersionedData {
        public final String value;
        public final int version;

        public VersionedData(String value, int version) {
            this.value = value;
            this.version = version;
        }
    }
}
```

## 与 AtomicMarkableReference 的区别

| 特性 | AtomicStampedReference | AtomicMarkableReference |
|------|----------------------|----------------------|
| 版本类型 | int（整数版本号） | boolean（标记位） |
| 版本含义 | 递增的版本号 | 二进制标记 |
| 适用场景 | 需要追踪修改次数 | 只需知道是否修改过 |
| 示例 | 乐观锁、计数器 | 逻辑删除、生命周期 |

```java
// AtomicMarkableReference：适用于"软删除"场景
private AtomicMarkableReference<Node> head = 
    new AtomicMarkableReference<>(null, false);

// 标记为已删除
head.compareAndSet(node, node, false, true);
```

## 性能考虑

### 版本号溢出的问题

```java
// 版本号是 int，会溢出
// 最大值：Integer.MAX_VALUE = 2147483647
// 溢出后变成负数

// 解决方案：使用 Long 版本号，或定期重置
public class OverflowSafeStamp {
    private final AtomicStampedReference<String> ref = 
        new AtomicStampedReference<>("A", 0);

    public void safeUpdate() {
        while (true) {
            String[] holder = new String[1];
            int stamp = ref.get(holder);

            // 检查是否接近溢出
            if (stamp == Integer.MAX_VALUE) {
                // 重置版本号
                ref.compareAndSet(holder[0], holder[0], stamp, 0);
            }

            if (ref.compareAndSet(holder[0], "B", stamp, stamp + 1)) {
                return;
            }
        }
    }
}
```

## 面试核心问题

**问题 1**：AtomicStampedReference 是如何解决 ABA 问题的？

**答案**：AtomicStampedReference 不仅存储值，还存储一个版本号（stamp）。compareAndSet() 方法需要同时满足期望值、期望版本号、新值、新版本号四个条件才能成功。即使值从 A 变到 B 又变回 A，版本号会持续递增，所以 CAS 会失败。

**问题 2**：为什么版本号能解决 ABA 问题？

**答案**：因为版本号是递增的。即使值"变回来"了，版本号也不会相同。例如：A(v1) → B(v2) → A(v3)，版本号从 v1 变成 v3，线程尝试用 v1 的版本号做 CAS 会失败，因为它期望的是旧版本。

**问题 3**：AtomicStampedReference 和 AtomicMarkableReference 的区别是什么？

**答案**：StampedReference 使用 int 版本号，可以追踪完整的修改历史，适用于乐观锁等需要知道"修改了多少次"的场景。MarkableReference 使用 boolean 标记，只能表示"修改过"或"未修改过"，适用于软删除等只需要知道"是否修改"的场景。

**问题 4**：版本号溢出怎么处理？

**答案**：版本号是 int 类型，会溢出变成负数。处理方法：1）检测版本号接近 Integer.MAX_VALUE 时进行重置；2）使用其他机制（如时间戳 + 版本号的组合）；3）在极端情况下接受溢出（因为溢出会导致版本号变得很大，CAS 仍会失败）。
