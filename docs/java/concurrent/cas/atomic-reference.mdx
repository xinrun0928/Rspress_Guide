# AtomicReference 与 ABA 问题

`AtomicReference` 是 Java 并发包中用于原子性更新**对象引用**的类。与 `AtomicInteger` 只能操作基本类型不同，`AtomicReference` 可以操作任意对象。但这也带来了一个经典问题——**ABA 问题**。理解 `AtomicReference` 的使用和 ABA 问题，是深入理解 CAS 无锁编程的重要一环。

## AtomicReference 概览

### 基本用法

```java
public class AtomicReferenceDemo {
    
    private final AtomicReference<String> reference = 
        new AtomicReference<>("A");
    
    public void basicUsage() {
        // 设置值
        reference.set("B");
        
        // 获取值
        String value = reference.get();
        
        // CAS 设置
        reference.compareAndSet("B", "C");
        
        // 获取并设置
        String old = reference.getAndSet("D");
    }
}
```

### 完整方法列表

```java
public class AtomicReferenceMethods {
    
    AtomicReference<String> ref = new AtomicReference<>("A");
    
    public void allMethods() {
        // 构造函数
        AtomicReference<String> r1 = new AtomicReference<>();           // null
        AtomicReference<String> r2 = new AtomicReference<>("B");        // "B"
        
        // 获取和设置
        String v1 = ref.get();                    // 获取值
        ref.set("C");                             // 设置值
        String v2 = ref.getAndSet("D");          // 获取并设置
        
        // CAS 操作
        ref.compareAndSet("D", "E");             // 成功返回 true
        ref.weakCompareAndSet("E", "F");         // 弱 CAS
        
        // 懒设置
        ref.lazySet("G");                        // 不保证立即可见
        
        // 更新
        ref.updateAndGet(s -> s + "H");          // JDK 8+
        ref.accumulateAndGet("I", (a, b) -> a + b);  // JDK 8+
        
        // toString
        ref.toString();                          // "G"
    }
}
```

## 实战应用场景

### 场景1：乐观锁

```java
public class OptimisticLock<T> {
    
    private final AtomicReference<VersionedValue<T>> ref = 
        new AtomicReference<>();
    
    public OptimisticLock(T initialValue) {
        ref.set(new VersionedValue<>(initialValue, 0));
    }
    
    public boolean update(T newValue) {
        VersionedValue<T> current;
        VersionedValue<T> newVersion;
        do {
            current = ref.get();
            newVersion = new VersionedValue<>(newValue, current.version + 1);
        } while (!ref.compareAndSet(current, newVersion));
        
        return true;
    }
    
    public T get() {
        return ref.get().value;
    }
    
    private static class VersionedValue<T> {
        final T value;
        final long version;
        
        VersionedValue(T value, long version) {
            this.value = value;
            this.version = version;
        }
    }
}
```

### 场景2：无锁栈

```java
public class LockFreeStack<E> {
    
    private final AtomicReference<Node<E>> top = 
        new AtomicReference<>();
    
    public void push(E item) {
        Node<E> newNode = new Node<>(item);
        Node<E> oldTop;
        do {
            oldTop = top.get();
            newNode.next = oldTop;
        } while (!top.compareAndSet(oldTop, newNode));
    }
    
    public E pop() {
        Node<E> oldTop;
        Node<E> newTop;
        do {
            oldTop = top.get();
            if (oldTop == null) {
                return null;
            }
            newTop = oldTop.next;
        } while (!top.compareAndSet(oldTop, newTop));
        
        return oldTop.item;
    }
    
    private static class Node<E> {
        final E item;
        volatile Node<E> next;
        
        Node(E item) {
            this.item = item;
        }
    }
}
```

### 场景3：线程安全的配置更新

```java
public class ThreadSafeConfig {
    
    private final AtomicReference<Map<String, String>> config = 
        new AtomicReference<>(Collections.emptyMap());
    
    // 获取配置（无锁读取）
    public String get(String key) {
        return config.get().get(key);
    }
    
    // 原子更新配置
    public void update(String key, String value) {
        while (true) {
            Map<String, String> current = config.get();
            Map<String, String> updated = new HashMap<>(current);
            updated.put(key, value);
            
            if (config.compareAndSet(current, updated)) {
                return;
            }
            // 更新失败，重试
        }
    }
    
    // 原子替换整个配置
    public void replace(Map<String, String> newConfig) {
        config.set(newConfig);
    }
}
```

## ABA 问题的详解

### 什么是 ABA 问题？

CAS 只检查"当前值是否等于期望值"，但不能检测值是否被修改后再改回来：

```java
public class ABAProblemDemo {
    
    public static void main(String[] args) {
        AtomicReference<String> ref = new AtomicReference<>("A");
        
        // 线程 A 读取值 = "A"
        System.out.println("Thread A reads: " + ref.get());
        
        // 线程 B 修改：A → B → A
        new Thread(() -> {
            ref.compareAndSet("A", "B");  // A → B
            ref.compareAndSet("B", "A");  // B → A（变回来了！）
        }).start();
        
        // 线程 A 再次尝试修改：A → C
        // CAS 检查：期望 "A"，当前 "A"，相等！修改成功！
        boolean success = ref.compareAndSet("A", "C");
        System.out.println("Thread A CAS result: " + success);
        System.out.println("Final value: " + ref.get());
    }
}
```

### ABA 问题的危害

#### 场景1：栈的 ABA 问题

```java
public class StackABAProblem {
    
    public void demonstrate() {
        LockFreeStack<Integer> stack = new LockFreeStack<>();
        
        // 初始：top -> A
        stack.push(1);
        stack.push(2);
        stack.push(3);
        // 栈: 3 -> 2 -> 1
        
        // 线程 A：读取 top = 3，准备 pop
        Integer topA = stack.pop();
        
        // 线程 B：执行 push/pop 操作
        // pop: 3 -> 2 -> 1
        stack.pop(); stack.pop();  // 移除 3, 2
        
        // push: 3 -> 2 -> 1 (又变回来了)
        stack.push(2);
        stack.push(3);
        
        // 线程 A：CAS 成功，以为栈没变化
        // 但实际上栈的内容已经改变了！
    }
}
```

#### 场景2：链表操作的 ABA 问题

```java
public class LinkedListABAProblem {
    
    private final AtomicReference<Node> head = new AtomicReference<>();
    
    public void demonstrate() {
        // 初始：head -> A -> B -> C
        head.set(new Node("C"));
        head.get().next = new Node("B");
        head.get().next.next = new Node("A");
        
        // 线程 A：读取 head = A
        Node expectedHead = head.get();
        
        // 线程 B：删除并重新插入节点 A
        // 删除: head -> B -> C
        // 插入: head -> A' -> B -> C
        // 现在 A 和 A' 是不同的对象，但值相同
        
        // 线程 A：CAS 成功
        // 但实际上链表结构已经改变了
    }
}
```

## ABA 问题的解决方案

### 方案1：AtomicStampedReference（带版本号）

```java
public class StampedReferenceSolution {
    
    public void solveABA() {
        // 每个引用附带一个戳（版本号）
        AtomicStampedReference<String> ref = 
            new AtomicStampedReference<>("A", 1);
        
        // 获取当前值和版本号
        int[] stamp = new int[1];
        String value = ref.get(stamp);
        int currentStamp = stamp[0];
        
        // CAS 操作：值和版本号都匹配才能成功
        boolean success = ref.compareAndSet(
            "A",          // 期望值
            "B",          // 新值
            currentStamp, // 期望版本号
            currentStamp + 1  // 新版本号
        );
        
        System.out.println("CAS success: " + success);
    }
}
```

### 方案2：AtomicMarkableReference（带标记）

```java
public class MarkableReferenceSolution {
    
    // AtomicMarkableReference 用 boolean 标记
    // 适用于"删除标记"等场景
    public void solve() {
        AtomicMarkableReference<String> ref = 
            new AtomicMarkableReference<>("A", false);
        
        // 获取值和标记
        boolean[] mark = new boolean[1];
        String value = ref.get(mark);
        
        // CAS 操作：值和标记都匹配才能成功
        ref.compareAndSet("A", "B", mark[0], true);
    }
}
```

### 实战：带版本号的栈

```java
public class VersionedStack<E> {
    
    private final AtomicStampedReference<Node<E>> top = 
        new AtomicStampedReference<>(null, 0);
    
    public void push(E item) {
        Node<E> newNode = new Node<>(item);
        int[] stamp = new int[1];
        Node<E> currentTop;
        
        do {
            currentTop = top.get(stamp);
            newNode.next = currentTop;
        } while (!top.compareAndSet(
            currentTop,          // 期望值
            newNode,             // 新值
            stamp[0],           // 期望版本号
            stamp[0] + 1        // 新版本号
        ));
    }
    
    public E pop() {
        int[] stamp = new int[1];
        Node<E> currentTop;
        Node<E> newTop;
        
        do {
            currentTop = top.get(stamp);
            if (currentTop == null) {
                return null;
            }
            newTop = currentTop.next;
        } while (!top.compareAndSet(
            currentTop,
            newTop,
            stamp[0],
            stamp[0] + 1
        ));
        
        return currentTop.item;
    }
    
    private static class Node<E> {
        final E item;
        volatile Node<E> next;
        
        Node(E item) {
            this.item = item;
        }
    }
}
```

## 注意事项

### 1. 泛型擦除的问题

```java
public class ErasureProblem {
    
    // AtomicReference<String> 和 AtomicReference<Integer>
    // 在运行时都是 AtomicReference<Object>
    // 需要在业务层面保证类型安全
}

public class TypeSafety {
    
    private final AtomicReference<Object> ref = 
        new AtomicReference<>();
    
    // 使用泛型方法包装
    public <T> void setIfAbsent(Class<T> clazz, T value) {
        ref.compareAndSet(null, value);
    }
    
    public <T> T get(Class<T> clazz) {
        return clazz.cast(ref.get());
    }
}
```

### 2. 对象引用的原子性

```java
public class ObjectReferenceAtomicity {
    
    private final AtomicReference<User> userRef = 
        new AtomicReference<>();
    
    public void badPractice() {
        User user = userRef.get();
        user.setName("Bob");  // 这个操作不是原子的！
        // 应该使用新的对象
    }
    
    public void goodPractice() {
        userRef.updateAndGet(current -> {
            if (current == null) {
                return new User("Bob");
            } else {
                return new User(current.getName() + "_modified");
            }
        });
    }
}
```

## 总结

`AtomicReference` 的核心：

- **CAS 原子性**：保证引用更新的原子性
- **ABA 问题**：值变回原值时 CAS 仍会成功
- **解决方案**：使用 `AtomicStampedReference` 或 `AtomicMarkableReference`

使用场景：
- 乐观锁实现
- 无锁数据结构
- 线程安全的配置管理
