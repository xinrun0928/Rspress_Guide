# AtomicInteger/AtomicLong 源码解析

`AtomicInteger` 和 `AtomicLong` 是 Java 并发包中最基础也最常用的原子类。它们内部使用 CAS（Compare-And-Swap）实现线程安全的递增、递减等操作，是理解 Java 无锁编程的绝佳窗口。今天我们就来深入解析它们的实现原理。

## AtomicInteger 概览

```java
public class AtomicInteger extends Number implements java.io.Serializable {
    
    // 使用 Unsafe 进行 CAS 操作
    private static final Unsafe unsafe = Unsafe.getUnsafe();
    
    // value 字段在内存中的偏移量
    private static final long valueOffset;
    
    // 实际存储的值，使用 volatile 保证可见性
    private volatile int value;
    
    // 静态初始化块：获取 value 字段的偏移量
    static {
        try {
            valueOffset = unsafe.objectFieldOffset(
                AtomicInteger.class.getDeclaredField("value")
            );
        } catch (Exception e) {
            throw new Error(e);
        }
    }
    
    // 构造函数
    public AtomicInteger(int initialValue) {
        value = initialValue;
    }
    
    public AtomicInteger() {
    }
}
```

## 核心 CAS 方法

### getAndAddInt 的实现

`AtomicInteger.incrementAndGet()` 调用了 `Unsafe.getAndAddInt()`：

```java
// Unsafe.java 中的实现（JDK 源码）
public final int getAndAddInt(Object o, long offset, int delta) {
    int expected;
    do {
        // 获取当前值
        expected = getIntVolatile(o, offset);
    } while (!compareAndSwapInt(o, offset, expected, expected + delta));
    
    // 返回旧值
    return expected;
}
```

### 增量操作的完整流程

```java
public class IncrementFlow {
    
    AtomicInteger counter = new AtomicInteger(0);
    
    public void incrementAndGetFlow() {
        // 1. 获取当前值
        int current = counter.get();
        
        // 2. CAS 尝试设置新值
        boolean success = counter.compareAndSet(current, current + 1);
        
        // 3. 如果失败，重试
        while (!success) {
            current = counter.get();
            success = counter.compareAndSet(current, current + 1);
        }
        
        // 4. 返回新值
        // current + 1
    }
}
```

### 6. compareAndSet()：CAS 操作

```java
public final boolean compareAndSet(int expectedValue, int newValue) {
    return unsafe.compareAndSwapInt(
        this, valueOffset, expectedValue, newValue
    );
}

// 使用示例
AtomicInteger counter = new AtomicInteger(10);

// 成功：期望值匹配
boolean success1 = counter.compareAndSet(10, 20);  // true, counter = 20

// 失败：期望值不匹配
boolean success2 = counter.compareAndSet(10, 30);  // false, counter = 20
```

### 7. weakCompareAndSet()：弱 CAS

```java
public final boolean weakCompareAndSet(int expectedValue, int newValue) {
    return unsafe.compareAndSwapInt(
        this, valueOffset, expectedValue, newValue
    );
}

// weakCompareAndSet 与 compareAndSet 的区别：
// 1. weakCompareAndSet 可能在不通知其他线程的情况下失败
// 2. 在某些平台上性能更好
// 3. JMM 保证比 compareAndSet 更弱
```

## 完整方法列表

```java
public class AllMethods {
    
    AtomicInteger counter = new AtomicInteger(0);
    
    public void demonstrateAllMethods() {
        // 构造函数
        AtomicInteger a1 = new AtomicInteger();        // 初始值 0
        AtomicInteger a2 = new AtomicInteger(100);     // 初始值 100
        
        // 获取值
        int v1 = counter.get();
        
        // 设置值
        counter.set(100);                  // 无条件设置
        counter.lazySet(100);              // 最终设置（性能更好，不保证立即可见）
        counter.getAndSet(100);           // 设置并返回旧值
        
        // 递增
        counter.incrementAndGet();         // ++i
        counter.getAndIncrement();         // i++
        
        // 递减
        counter.decrementAndGet();         // --i
        counter.getAndDecrement();         // i--
        
        // 加法
        counter.addAndGet(10);            // i += 10
        counter.getAndAdd(10);            // i += 10，返回旧值
        
        // CAS
        counter.compareAndSet(10, 20);    // 期望 10，设置 20
        counter.weakCompareAndSet(10, 20); // 弱 CAS
        
        // 更新（JDK 8+）
        counter.updateAndGet(x -> x + 1);  // 函数式更新
        counter.accumulateAndGet(10, Integer::sum);  // 累加
        
        // toString
        counter.toString();                // 返回字符串
    }
}
```

## JDK 8 新增方法

### updateAndGet

```java
// 使用函数式接口更新
public final int updateAndGet(IntUnaryOperator updateFunction) {
    int prev, next;
    do {
        prev = get();
        next = updateFunction.applyAsInt(prev);
    } while (!compareAndSet(prev, next));
    return next;
}

// 使用示例
AtomicInteger counter = new AtomicInteger(10);
int result = counter.updateAndGet(x -> x * 2);  // result = 20
```

### accumulateAndGet

```java
// 使用累加函数
public final int accumulateAndGet(int x, IntBinaryOperator accumulatorFunction) {
    int prev, next;
    do {
        prev = get();
        next = accumulatorFunction.applyAsInt(prev, x);
    } while (!compareAndSet(prev, next));
    return next;
}

// 使用示例
AtomicInteger counter = new AtomicInteger(10);
counter.accumulateAndGet(5, Integer::sum);  // counter = 15
counter.accumulateAndGet(3, (a, b) -> a * b);  // counter = 45
```

## AtomicLong 的实现

`AtomicLong` 与 `AtomicInteger` 类似，但处理 64 位数值：

```java
public class AtomicLong extends Number implements java.io.Serializable {
    
    private static final Unsafe unsafe = Unsafe.getUnsafe();
    private static final long valueOffset;
    
    private volatile long value;
    
    static {
        try {
            valueOffset = unsafe.objectFieldOffset(
                AtomicLong.class.getDeclaredField("value")
            );
        } catch (Exception e) {
            throw new Error(e);
        }
    }
    
    public final long getAndAddLong(Object o, long offset, long delta) {
        long expected;
        do {
            expected = getLongVolatile(o, offset);
        } while (!compareAndSwapLong(o, offset, expected, expected + delta));
        return expected;
    }
    
    // 在 32 位 JVM 中，long 的写入需要 volatile
    // 64 位 JVM 中，volatile 保证了对 long 操作的原子性
}
```

## 实战：计数器实现

### 简单的线程安全计数器

```java
public class SafeCounter {
    
    private final AtomicInteger count = new AtomicInteger(0);
    
    public void increment() {
        count.incrementAndGet();
    }
    
    public int get() {
        return count.get();
    }
    
    public void reset() {
        count.set(0);
    }
}
```

### 分段计数器（高并发优化）

```java
public class SegmentedCounter {
    
    private final AtomicInteger[] counters;
    private final int segmentCount;
    
    public SegmentedCounter(int segmentCount) {
        this.segmentCount = segmentCount;
        this.counters = new AtomicInteger[segmentCount];
        for (int i = 0; i < segmentCount; i++) {
            counters[i] = new AtomicInteger(0);
        }
    }
    
    public void increment() {
        int index = ThreadLocalRandom.current().nextInt(segmentCount);
        counters[index].incrementAndGet();
    }
    
    public int getTotal() {
        int sum = 0;
        for (AtomicInteger counter : counters) {
            sum += counter.get();
        }
        return sum;
    }
}
```

### 带边界检查的计数器

```java
public class BoundedCounter {
    
    private final AtomicInteger value;
    private final int min;
    private final int max;
    
    public BoundedCounter(int initial, int min, int max) {
        if (initial < min || initial > max) {
            throw new IllegalArgumentException();
        }
        this.value = new AtomicInteger(initial);
        this.min = min;
        this.max = max;
    }
    
    public boolean increment() {
        while (true) {
            int current = value.get();
            if (current >= max) {
                return false;  // 已达上限
            }
            if (value.compareAndSet(current, current + 1)) {
                return true;
            }
            // CAS 失败，重试
        }
    }
    
    public boolean decrement() {
        while (true) {
            int current = value.get();
            if (current <= min) {
                return false;  // 已达下限
            }
            if (value.compareAndSet(current, current - 1)) {
                return true;
            }
        }
    }
}
```

## 注意事项

### 1. 使用 getAndAdd 而不是 get + set

```java
public class CorrectUsage {
    
    AtomicInteger counter = new AtomicInteger(0);
    
    public void wrong() {
        // 错误：不是原子操作
        counter.set(counter.get() + 1);
    }
    
    public void correct() {
        // 正确：使用原子操作
        counter.incrementAndGet();
        // 或
        counter.getAndAdd(1);
    }
}
```

### 2. 避免过度使用

```java
public class OveruseProblem {
    
    AtomicInteger a = new AtomicInteger(0);
    AtomicInteger b = new AtomicInteger(0);
    AtomicInteger c = new AtomicInteger(0);
    
    // 问题：复合操作不是原子的
    public int getSum() {
        // 不是原子操作
        return a.get() + b.get() + c.get();
    }
    
    // 正确：使用同步或 LongAdder
    public int getSumWithLock() {
        synchronized (this) {
            return a.get() + b.get() + c.get();
        }
    }
}
```

## 总结

`AtomicInteger` 和 `AtomicLong` 的核心原理：

1. **Unsafe**：使用底层 CAS 指令
2. **volatile**：保证可见性
3. **死循环 CAS**：实现原子操作
4. **JDK 8 新增**：函数式 API（updateAndGet、accumulateAndGet）

理解源码能帮助我们：
- 正确使用原子类
- 避免常见错误
- 在高并发场景下选择合适的方案
