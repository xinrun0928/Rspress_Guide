# Java 并发编程

synchronized 关键字，你真的了解它吗？

大多数人会说「synchronized 可以实现同步」。但追问下去：synchronized 锁的是什么？偏向锁、轻量级锁、重量级锁分别是什么时候升级的？为什么 JDK 6 之后 synchronized 的性能突飞猛进？

并发编程是 Java 体系中最高阶也是最复杂的部分。当多个线程同时访问共享资源，当 CPU 的每一个核心都被压榨到极限，当死锁、活锁、饥饿轮番出现——你需要这一模块的知识。

## 学习路线建议

```
第一阶段：线程基础 → synchronized → volatile → JMM（Happens-Before）
         ↓
第二阶段：CAS → 原子类 → ReentrantLock → AQS 原理（深度理解底层）
         ↓
第三阶段：线程池 → JUC 工具 → CompletableFuture（工程实践）
         ↓
第四阶段：死锁 → 生产者消费者 → 面试汇总（查漏补缺）
```

## 面试核心公式

```
synchronized 锁升级 = 无锁 → 偏向锁 → 轻量级锁 → 重量级锁（不可逆）
         ↓
volatile 保证 = 可见性（StoreLoad 屏障）+ 有序性（禁止重排序）
         ↓
线程池拒绝策略 = Abort（抛异常）/ CallerRuns（调用者执行）/ Discard（丢弃）/ DiscardOldest（丢弃最老）
         ↓
死锁四大条件 = 互斥 + 占有并等待 + 不可抢占 + 循环等待（破坏任一即可）
```
