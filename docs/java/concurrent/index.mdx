# 线程与进程

# 线程是 CPU 调度的最小单位，进程是资源分配的最小单位。线程共享进程的资源，但拥有独立的栈内存和程序计数器。

想象一下，你正在经营一家餐厅。进程就像是这家餐厅的整个建筑本身——包括厨房、用餐区、仓库等所有空间和设备。而线程呢？线程就像是餐厅里的不同员工——厨师、服务员、收银员等，他们共享餐厅的所有资源（厨房设备、食材、桌椅），但各自做着不同的工作，彼此协作让餐厅正常运转。

在 Java 的世界里，**进程**是程序的一次执行过程，是系统进行资源分配和调度的基本单位。每个 Java 应用运行时会创建一个 JVM 进程，主方法就是其中的一个线程。**线程**则是进程中的一个执行路径，是 CPU 调度的最小单位，同一个进程中的多个线程共享堆内存和方法区，但每个线程有自己的程序计数器、虚拟机栈和本地方法栈。

Java 天生就是多线程的。你是否注意到，每次运行一个简单的 `main` 方法，实际上 JVM 会创建多个线程？这个看似简单的程序背后，隐藏着比想象中复杂得多的并发世界。

本章节将带你深入探索 Java 并发编程的方方面面，从基础的线程概念，到 synchronized 的锁机制，再到 JUC 并发包的核心组件，以及线程池、JMM 内存模型、CAS 无锁算法等高级主题。每一个知识点都是面试中的高频考点，也是实际开发中必须掌握的硬技能。
