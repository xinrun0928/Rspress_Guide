# JMM 八大 Happens-Before 规则

Happens-Before 是 Java 内存模型（JMM）的核心概念，它定义了多线程环境下操作的**可见性边界**。理解这八大规则，是编写正确并发代码的关键。当面试官问你"什么是 Happens-Before？"时，你能否完整地说出这八条规则？今天我们就来深入解析每一条规则。

## 什么是 Happens-Before？

简单来说，如果操作 A Happens-Before 操作 B，那么：

1. **A 的结果对 B 可见**
2. **A 的执行顺序在 B 之前**

这并不意味着 A 在时间上一定先于 B 执行（编译器和 CPU 可能重排序），而是保证了**可见性**和**有序性**。

```
┌────────────────────────────────────────────────────────────────┐
│                      Happens-Before 示意                          │
│                                                                 │
│   操作 A ────────────► 操作 B                                     │
│       │                   │                                      │
│       │                   │                                      │
│       ▼                   ▼                                      │
│   ┌────────┐          ┌────────┐                               │
│   │ 写入 X │          │ 读取 X │                                │
│   │  = 1   │  ───────▶│   = ? │                               │
│   └────────┘          └────────┘                               │
│       │                                                       │
│       │ A Happens-Before B                                      │
│       └─────────────────────────────────▶ X 的值一定是 1        │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## 八大 Happens-Before 规则

### 规则一：程序顺序规则（Program Order Rule）

> **同一个线程中，前面的操作 Happens-Before 后面的操作**

在同一个线程内，代码的执行顺序与书写顺序一致。

```java
public class ProgramOrderDemo {
    
    public void singleThread() {
        int a = 1;           // 操作 1
        int b = 2;           // 操作 2
        int c = a + b;      // 操作 3
        
        // 操作 1 Happens-Before 操作 2
        // 操作 2 Happens-Before 操作 3
        // 因此：a=1 对 c=a+b 可见
    }
}
```

这条规则看似简单，但它允许编译器和 CPU 在**不改变单线程执行结果**的前提下进行**指令重排序**。

```java
public class ReorderingExample {
    
    int a = 0;
    int b = 0;
    
    public void method() {
        a = 1;      // 可能与下面一行重排序
        b = 2;      // 如果 b 没有被其他线程读取
        
        // 单线程下：无论哪个先执行，a 和 b 最终都是 1 和 2
        // 多线程下：可能出现问题
    }
}
```

### 规则二：监视器锁规则（Monitor Lock Rule）

> **同一个锁的 unlock 操作 Happens-Before 后续对这个锁的 lock 操作**

```java
public class MonitorLockDemo {
    
    private final Object lock = new Object();
    private int value = 0;
    
    // 线程 A
    public void writer() {
        synchronized (lock) {
            value = 100;  // unlock 释放锁
        }  // Happens-Before
    }
    
    // 线程 B
    public void reader() {
        synchronized (lock) {
            // Happens-Before
            int v = value;  // 一定能读到 100
        }
    }
}
```

这条规则保证：
- 线程 A 对 value 的修改 Happens-Before 线程 B 获取锁
- 线程 B 获取锁 Happens-Before 线程 B 读取 value
- 传递性：线程 A 的修改 Happens-Before 线程 B 的读取

### 规则三：volatile 变量规则（Volatile Variable Rule）

> **volatile 变量的写操作 Happens-Before 后续对这个变量的读操作**

```java
public class VolatileRuleDemo {
    
    volatile boolean ready = false;
    int data = 0;
    
    // 线程 A
    public void writer() {
        data = 42;       // 操作 1
        ready = true;    // 操作 2（volatile）
    }
    
    // 线程 B
    public void reader() {
        if (ready) {    // 操作 3（volatile）
            int v = data;  // 操作 4
            // 一定能读到 data = 42
        }
    }
}
```

等价于：
```
操作 1 Happens-Before 操作 2 (程序顺序)
操作 2 Happens-Before 操作 3 (volatile 规则)
操作 3 Happens-Before 操作 4 (程序顺序)
传递性：操作 1 Happens-Before 操作 4
```

### 规则四：线程启动规则（Thread Start Rule）

> **Thread.start() 操作 Happens-Before 被启动线程中的任何操作**

```java
public class ThreadStartDemo {
    
    int x = 0;
    
    public void startRule() {
        Thread thread = new Thread(() -> {
            // 一定能读到 x = 100
            System.out.println(x);
        });
        
        x = 100;        // 操作 1
        thread.start(); // 操作 2
    }
}
```

因为：
- 操作 1 Happens-Before 操作 2（程序顺序）
- 操作 2 Happens-Before lambda 表达式中的代码（线程启动规则）

### 规则五：线程终止规则（Thread Termination Rule）

> **线程中的所有操作 Happens-Before 其他线程检测到该线程终止**

```java
public class ThreadTerminationDemo {
    
    int x = 0;
    
    public void terminationRule() throws InterruptedException {
        Thread thread = new Thread(() -> {
            x = 100;  // 线程内的操作
        });
        
        thread.start();
        thread.join();  // 等待线程结束
        
        // 一定能读到 x = 100
        System.out.println(x);
    }
}
```

因为：
- 线程内的代码 Happens-Before `thread.join()` 返回（线程终止规则）
- `thread.join()` 返回 Happens-Before `println(x)`（程序顺序）

### 规则六：中断规则（Thread Interruption Rule）

> **对线程 interrupt() 的调用 Happens-Before 被中断线程检测到中断**

```java
public class InterruptRuleDemo {
    
    volatile boolean interrupted = false;
    
    public void interruptRule() throws InterruptedException {
        Thread thread = new Thread(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                // 业务逻辑
            }
            // 一定能检测到中断
            interrupted = true;
        });
        
        thread.start();
        Thread.sleep(100);
        thread.interrupt();  // 中断线程
        
        thread.join();
        // interrupted 一定为 true
    }
}
```

### 规则七：终结器规则（Finalizer Rule）

> **对象的构造函数中的操作 Happens-Before 该对象的 finalize() 方法的开始**

```java
public class FinalizerRuleDemo {
    
    // 构造函数
    public SafeClass() {
        data = 42;  // 一定 Happens-Before finalize()
    }
    
    @Override
    protected void finalize() throws Throwable {
        // 一定能读到 data = 42
        System.out.println(data);
    }
}
```

### 规则八：传递性规则（Transitivity）

> **如果 A Happens-Before B，且 B Happens-Before C，则 A Happens-Before C**

这是最重要也最常用的规则，用于组合其他规则。

```java
public class TransitivityDemo {
    
    volatile int a = 0;
    int b = 0;
    
    public void transitivity() {
        // 线程 A
        b = 100;      // 操作 1
        a = 1;        // 操作 2（volatile）
        
        // 线程 B
        if (a == 1) { // 操作 3（volatile）
            int v = b; // 操作 4
            // v 一定等于 100
        }
    }
}
```

证明：
- 操作 1 Happens-Before 操作 2（程序顺序）
- 操作 2 Happens-Before 操作 3（volatile 规则）
- 操作 3 Happens-Before 操作 4（程序顺序）
- 传递性：操作 1 Happens-Before 操作 4

## 综合示例：单例模式

```java
public class Singleton {
    
    private static volatile Singleton instance;
    
    // 双重检查锁定
    public static Singleton getInstance() {
        if (instance == null) {  // 第一次检查
            synchronized (Singleton.class) {
                if (instance == null) {  // 第二次检查
                    instance = new Singleton();  // volatile 保证
                }
            }
        }
        return instance;
    }
    
    // 问题：为什么 instance 要声明为 volatile？
    // 因为 "instance = new Singleton()" 分为三步：
    // 1. 分配内存
    // 2. 调用构造函数
    // 3. 将引用赋值给 instance
    //
    // 没有 volatile 时，步骤 2 和 3 可能重排序：
    // 1. 分配内存
    // 3. 将引用赋值给 instance（instance != null，但未初始化）
    // 2. 调用构造函数
    //
    // 其他线程可能读到未初始化的 instance
    //
    // 有 volatile 后，volatile 的 StoreLoad 屏障禁止这种重排序
}
```

## 实际应用

### 正确使用 volatile

```java
public class CorrectVolatileUsage {
    
    // 场景：状态标志
    volatile boolean running = true;
    
    public void start() {
        running = true;  // Happens-Before
    }
    
    public void stop() {
        running = false;  // Happens-Before
    }
    
    public void run() {
        while (running) {  // 一定能读取到最新值
            // 业务逻辑
        }
    }
}
```

### 正确使用 synchronized

```java
public class CorrectSynchronizedUsage {
    
    private final Object lock = new Object();
    private int count = 0;
    
    public void increment() {
        synchronized (lock) {
            count++;  // Happens-Before
        }
    }
    
    public int getCount() {
        synchronized (lock) {
            return count;  // Happens-Before
        }
    }
}
```

## 总结：八大规则一览

| 序号 | 规则名称 | 描述 |
|------|---------|------|
| 1 | 程序顺序规则 | 同一线程内，前面的操作 Happens-Before 后面的操作 |
| 2 | 监视器锁规则 | unlock Happens-Before 后续的 lock |
| 3 | volatile 变量规则 | volatile 写 Happens-Before volatile 读 |
| 4 | 线程启动规则 | Thread.start() Happens-Before 被启动线程 |
| 5 | 线程终止规则 | 线程内操作 Happens-Before 其他线程 join() 返回 |
| 6 | 中断规则 | interrupt() 调用 Happens-Before 被中断线程检测到 |
| 7 | 终结器规则 | 构造函数 Happens-Before finalize() |
| 8 | 传递性规则 | A Happens-Before B，B Happens-Before C，则 A Happens-Before C |
