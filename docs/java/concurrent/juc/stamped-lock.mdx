# StampedLock：乐观读 vs 悲观读

`StampedLock` 是 JDK 8 引入的一种新型锁，它是 `ReadWriteLock` 的增强版。传统读写锁虽然允许读读并发，但在读取数据时仍然需要加锁（悲观读）。`StampedLock` 引入了**乐观读**的概念：如果在读取期间没有写操作发生，就不需要真正的加锁，从而进一步提升并发性能。

## StampedLock 的设计思想

想象你去图书馆借书。传统读写锁像是：你要借书时，先在前台登记（加锁），才能去书架拿书（读取）。即使书架上书很多（读操作），你也得在前台等着。

而 `StampedLock` 的乐观读像是：你先去书架拿书，同时记录下当前时间戳（戳记）。等你拿完书，再回去前台核对时间戳——如果在你拿书期间有人动过书架（写操作），时间戳会变化，你需要重新去拿。

```
┌────────────────────────────────────────────────────────────────┐
│                    乐观读 vs 悲观读                               │
│                                                                 │
│  悲观读（ReadWriteLock）:                                        │
│                                                                 │
│  线程A ───────▶ [读锁] ───────▶ 读取 ───────▶ [解锁]            │
│  线程B ───────▶ [读锁] ───────▶ 读取 ───────▶ [解锁]            │
│  线程C ───▶ [写锁等待] ───▶ [写锁] ───▶ 写入 ───▶ [解锁]        │
│                                                                 │
│  乐观读（StampedLock）:                                          │
│                                                                 │
│  线程A ──▶ 获取戳 ──▶ 读取(不锁) ──▶ 验证戳 ──▶ 通过             │
│  线程B ──▶ 获取戳 ──▶ 读取(不锁) ──▶ 验证戳 ──▶ 失败，需重试      │
│  线程C ──▶ 获取写戳 ──▶ 写入 ──▶ 释放戳                          │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## StampedLock 的三种锁模式

```java
public class StampedLockDemo {
    
    private final StampedLock stampedLock = new StampedLock();
    private double x, y;
    
    // 1. 写锁：独占锁，与读写互斥
    public void write(double newX, double newY) {
        long stamp = stampedLock.writeLock();
        try {
            x = newX;
            y = newY;
        } finally {
            stampedLock.unlockWrite(stamp);
        }
    }
    
    // 2. 悲观读锁：与写锁互斥，与读锁共享
    public double[] readPessimistic() {
        long stamp = stampedLock.readLock();
        try {
            return new double[]{x, y};
        } finally {
            stampedLock.unlockRead(stamp);
        }
    }
    
    // 3. 乐观读：不加锁，通过戳验证
    public double[] readOptimistic() {
        // 获取乐观戳
        long stamp = stampedLock.tryOptimisticRead();
        
        // 读取数据（不阻塞）
        double currentX = x;
        double currentY = y;
        
        // 验证戳是否有效
        if (!stampedLock.validate(stamp)) {
            // 戳无效（有写锁介入），升级为悲观读
            stamp = stampedLock.readLock();
            try {
                currentX = x;
                currentY = y;
            } finally {
                stampedLock.unlockRead(stamp);
            }
        }
        
        return new double[]{currentX, currentY};
    }
}
```

## 戳记（Stamp）的概念

戳记是 `StampedLock` 的核心概念，它是一个 `long` 类型的值，不同的锁模式返回不同的戳记：

| 模式 | 方法 | 返回值 | 说明 |
|------|------|--------|------|
| 写锁 | `writeLock()` | 非零戳记 | 独占访问 |
| 悲观读 | `readLock()` | 非零戳记 | 共享访问 |
| 乐观读 | `tryOptimisticRead()` | 戳记 | 零表示失败 |
| 获取锁 | `tryConvertToWriteLock()` | 新戳记 | 锁升级 |

```java
public class StampConcept {
    
    private final StampedLock lock = new StampedLock();
    
    public void stampDemo() {
        // 乐观读戳
        long optimisticStamp = lock.tryOptimisticRead();
        System.out.println("乐观读戳: " + optimisticStamp);
        
        // 悲观读戳
        long readStamp = lock.readLock();
        System.out.println("悲观读戳: " + readStamp);
        lock.unlockRead(readStamp);
        
        // 写锁戳
        long writeStamp = lock.writeLock();
        System.out.println("写锁戳: " + writeStamp);
        lock.unlockWrite(writeStamp);
    }
}
```

## 乐观读的完整示例

```java
public class Point {
    
    private final StampedLock lock = new StampedLock();
    private double x, y;
    
    public Point(double x, double y) {
        this.x = x;
        this.y = y;
    }
    
    // 移动到新位置
    public void move(double deltaX, double deltaY) {
        long stamp = lock.writeLock();
        try {
            x += deltaX;
            y += deltaY;
        } finally {
            lock.unlockWrite(stamp);
        }
    }
    
    // 距离原点的距离（乐观读）
    public double distanceFromOrigin() {
        // 1. 获取乐观戳
        long stamp = lock.tryOptimisticRead();
        
        // 2. 读取数据（此时可能正在被写）
        double currentX = x;
        double currentY = y;
        
        // 3. 验证戳
        if (!lock.validate(stamp)) {
            // 戳无效：写锁正在操作
            
            // 4. 升级为悲观读
            stamp = lock.readLock();
            try {
                currentX = x;
                currentY = y;
            } finally {
                lock.unlockRead(stamp);
            }
        }
        
        // 5. 使用数据
        return Math.sqrt(currentX * currentX + currentY * currentY);
    }
    
    // 距离原点的距离（悲观读）
    public double distanceFromOriginPessimistic() {
        long stamp = lock.readLock();
        try {
            return Math.sqrt(x * x + y * y);
        } finally {
            lock.unlockRead(stamp);
        }
    }
}
```

## 锁升级：tryConvertToLock

`StampedLock` 支持锁的转换和升级：

```java
public class LockUpgradeDemo {
    
    private final StampedLock lock = new StampedLock();
    private double value = 0;
    
    // 场景：从乐观读升级为写锁
    public void upgradeFromOptimisticToWrite() {
        // 1. 乐观读
        long stamp = lock.tryOptimisticRead();
        
        // 2. 读取
        double currentValue = value;
        
        // 3. 发现需要修改，尝试升级为写锁
        long writeStamp = lock.tryConvertToWriteLock(stamp);
        
        if (writeStamp != 0) {
            // 升级成功
            stamp = writeStamp;
            value = currentValue + 1;
            lock.unlockWrite(stamp);
        } else {
            // 升级失败（可能有其他读锁），释放并重新获取
            lock.unlockRead(stamp);
            stamp = lock.writeLock();
            try {
                value = value + 1;
            } finally {
                lock.unlockWrite(stamp);
            }
        }
    }
    
    // 场景：读写组合
    public double readAndCheck() {
        long stamp = lock.readLock();
        try {
            if (value > 0) {
                // 满足条件，需要修改
                // 释放读锁，获取写锁
                long writeStamp = lock.tryConvertToWriteLock(stamp);
                if (writeStamp != 0) {
                    stamp = writeStamp;
                    value = -value;
                }
            }
            return value;
        } finally {
            lock.unlock(stamp);
        }
    }
}
```

## StampedLock vs ReadWriteLock

| 特性 | StampedLock | ReadWriteLock |
|------|-------------|---------------|
| 乐观读 | 支持 | 不支持 |
| 锁升级 | 支持（乐观→悲观→写） | 不支持 |
| 性能 | 更优 | 良好 |
| 可重入 | 否 | 是 |
| 公平性 | 非公平 | 公平/非公平可选 |
| Condition | 不支持直接支持 | 支持 |

```java
public class ComparisonDemo {
    
    private final StampedLock stamped = new StampedLock();
    private final ReentrantReadWriteLock.ReadLock rwRead = 
        new ReentrantReadWriteLock().readLock();
    
    // StampedLock 乐观读：无需加锁
    public double stampedOptimistic() {
        long stamp = stamped.tryOptimisticRead();
        return compute();
    }
    
    // ReadWriteLock 悲观读：必须加锁
    public double rwPessimistic() {
        rwRead.lock();
        try {
            return compute();
        } finally {
            rwRead.unlock();
        }
    }
    
    private double compute() {
        return Math.random();
    }
}
```

## 实战：高性能缓存

```java
public class StampedLockCache<K, V> {
    
    private final Map<K, V> cache = new HashMap<>();
    private final StampedLock lock = new StampedLock();
    
    // 获取值：先乐观读，失败则悲观读
    public V get(K key) {
        long stamp = lock.tryOptimisticRead();
        V value = cache.get(key);
        
        if (value != null && lock.validate(stamp)) {
            return value;
        }
        
        // 乐观读失败，升级为悲观读
        stamp = lock.readLock();
        try {
            return cache.get(key);
        } finally {
            lock.unlockRead(stamp);
        }
    }
    
    // 更新值：写锁
    public V put(K key, V value) {
        long stamp = lock.writeLock();
        try {
            return cache.put(key, value);
        } finally {
            lock.unlockWrite(stamp);
        }
    }
    
    // 计算并缓存：乐观读 + 写锁组合
    public V computeIfAbsent(K key, Function<K, V> loader) {
        // 1. 先尝试乐观读
        long stamp = lock.tryOptimisticRead();
        V value = cache.get(key);
        
        if (value != null && lock.validate(stamp)) {
            return value;
        }
        
        // 2. 乐观读失败，获取写锁重试
        stamp = lock.writeLock();
        try {
            // 双重检查
            value = cache.get(key);
            if (value == null) {
                value = loader.apply(key);
                cache.put(key, value);
            }
            return value;
        } finally {
            lock.unlockWrite(stamp);
        }
    }
}
```

## 注意事项

### 1. StampedLock 不支持重入

```java
public class NoReentrant {
    
    private final StampedLock lock = new StampedLock();
    
    public void wrongMethod() {
        long stamp = lock.writeLock();
        try {
            // 在持有写锁时再次获取写锁
            long another = lock.writeLock();  // ❌ 会阻塞，导致死锁
            lock.unlockWrite(another);
        } finally {
            lock.unlockWrite(stamp);
        }
    }
}
```

### 2. 乐观读后必须验证

```java
public class MustValidate {
    
    private final StampedLock lock = new StampedLock();
    
    public void mustValidate() {
        long stamp = lock.tryOptimisticRead();
        
        // ❌ 错误：没有验证就使用数据
        doSomething();
        
        // ✓ 正确：使用前验证
        if (lock.validate(stamp)) {
            doSomething();
        }
    }
}
```

### 3. 不要在 finally 中 unlock 无效戳

```java
public class CorrectUnlock {
    
    private final StampedLock lock = new StampedLock();
    
    public void correctUnlock() {
        long stamp = lock.tryOptimisticRead();
        try {
            // 业务逻辑
        } finally {
            // 必须根据戳的类型调用正确的 unlock 方法
            if (StampedLock.isReadLockStamp(stamp)) {
                lock.unlockRead(stamp);
            } else if (StampedLock.isWriteLockStamp(stamp)) {
                lock.unlockWrite(stamp);
            }
        }
    }
    
    // 更好的方式：统一使用 unlock
    public void unifiedUnlock() {
        long stamp = lock.readLock();
        try {
            // 业务逻辑
        } finally {
            lock.unlock(stamp);  // JDK 8+ 支持
        }
    }
}
```

## 总结

`StampedLock` 是 JDK 8 的重要改进：

- **乐观读**：`tryOptimisticRead()` 不加锁，通过戳验证，适合读多写少
- **悲观读**：`readLock()` 传统读锁，与写锁互斥
- **写锁**：`writeLock()` 独占访问
- **锁升级**：`tryConvertToWriteLock()` 支持锁转换

使用建议：
1. 读多写少场景首选 `StampedLock`
2. 注意乐观读后必须验证戳
3. `StampedLock` 不支持重入，需要小心使用
