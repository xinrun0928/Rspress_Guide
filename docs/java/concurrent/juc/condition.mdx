# Condition：精准等待与唤醒

在 Java 并发编程中，我们经常需要在线程间协调工作进度。`Object.wait()/notify()/notifyAll()` 是最基础的等待/通知机制，但它们有一个明显的局限：**所有等待的线程都在同一个队列中**。如果我们需要区分不同的等待条件，比如"等待数据准备好"和"等待队列不满"，就无能为力了。`Condition` 接口正是为解决这个问题而生的——它提供了比 `Object.wait/notify` 更灵活、更强大的等待/通知机制。

## Condition 的基本概念

`Condition` 是在 `ReentrantLock` 的基础上实现的，每个 `ReentrantLock` 可以创建多个 `Condition`：

```java
public class ConditionConcept {
    
    private final ReentrantLock lock = new ReentrantLock();
    
    // 创建两个条件变量
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();
    
    private final Object[] items = new Object[100];
    private int count = 0;
    
    // 生产者：等待队列不满
    public void put(Object item) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length) {
                notFull.await();  // 等待"不满"条件
            }
            items[count++] = item;
            notEmpty.signal();   // 通知"非空"条件
        } finally {
            lock.unlock();
        }
    }
    
    // 消费者：等待队列非空
    public Object take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();  // 等待"非空"条件
            }
            Object item = items[--count];
            notFull.signal();       // 通知"不满"条件
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

### Condition vs Object.wait

| 特性 | Condition | Object.wait |
|------|-----------|-------------|
| 条件数量 | 多个独立条件 | 单一条件 |
| 获取方式 | `lock.newCondition()` | `synchronized` 块中调用 |
| 等待队列 | 多个独立队列 | 一个共享队列 |
| 超时等待 | `await(long, TimeUnit)` | `wait(long)` |
| 精确唤醒 | `signal()` 指定队列 | `notify()` 随机唤醒 |
| 公平性 | 支持 | 不支持 |

## Condition 的核心方法

```java
public class ConditionMethods {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition condition = lock.newCondition();
    
    public void allMethods() throws InterruptedException {
        
        // 1. await() - 等待，直到被唤醒或中断
        condition.await();
        
        // 2. awaitUninterruptibly() - 不可中断的等待
        condition.awaitUninterruptibly();
        
        // 3. awaitNanos(nanos) - 等待指定纳秒
        long remainingNanos = condition.awaitNanos(1000000);
        // 返回值 < 0 表示超时
        
        // 4. await(time, unit) - 等待指定时间
        boolean timedOut = !condition.await(5, TimeUnit.SECONDS);
        
        // 5. awaitUntil(deadline) - 等待直到绝对时间点
        boolean beforeDeadline = condition.awaitUntil(
            new Date(System.currentTimeMillis() + 5000)
        );
        
        // 6. signal() - 唤醒一个等待线程
        condition.signal();
        
        // 7. signalAll() - 唤醒所有等待线程
        condition.signalAll();
    }
}
```

## Condition 的实现原理

`Condition` 的实现依赖于 AQS 的内部结构：

```java
public class ConditionObject implements Condition, Serializable {
    
    // 等待队列：存储等待该条件的线程
    private transient Node firstWaiter;
    private transient Node lastWaiter;
    
    // 添加等待节点到队列尾部
    private Node addConditionWaiter() {
        Node node = new Node(Thread.currentThread(), Node.CONDITION);
        
        if (lastWaiter == null) {
            firstWaiter = lastWaiter = node;
        } else {
            lastWaiter.nextWaiter = node;
            lastWaiter = node;
        }
        return node;
    }
    
    // await() 的实现
    public final void await() throws InterruptedException {
        if (Thread.interrupted()) {
            throw new InterruptedException();
        }
        
        // 1. 将当前线程加入等待队列
        Node node = addConditionWaiter();
        
        // 2. 释放已获取的锁
        int savedState = fullyRelease(node);
        
        // 3. 阻塞当前线程
        boolean interrupted = false;
        while (!isOnSyncQueue(node)) {
            LockSupport.park(this);
            if (Thread.interrupted()) {
                interrupted = true;
            }
        }
        
        // 4. 重新获取锁
        acquireQueued(node, savedState);
        
        if (interrupted) {
            selfInterrupt();
        }
    }
    
    // signal() 的实现
    public final void signal() {
        if (!isHeldExclusively()) {
            throw new IllegalMonitorStateException();
        }
        
        Node first = firstWaiter;
        if (first != null) {
            doSignal(first);  // 移动到同步队列
        }
    }
    
    // 移动节点到同步队列
    private void doSignal(Node first) {
        do {
            firstWaiter = first.nextWaiter;
            if (firstWaiter == null) {
                lastWaiter = null;
            }
            first.nextWaiter = null;
            
            // 转移到 AQS 同步队列
            if (transferForSignal(first)) {
                LockSupport.unpark(first.thread);
            }
        } while (firstWaiter != null);
    }
}
```

## 实战应用场景

### 场景1：阻塞队列

```java
public class MyBlockingQueue<E> {
    
    private final Object[] items;
    private int count, takeIndex, putIndex;
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();
    
    public MyBlockingQueue(int capacity) {
        items = new Object[capacity];
    }
    
    public void put(E e) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length) {
                notFull.await();  // 队列满，等待
            }
            items[putIndex] = e;
            if (++putIndex == items.length) {
                putIndex = 0;
            }
            count++;
            notEmpty.signal();  // 通知消费者
        } finally {
            lock.unlock();
        }
    }
    
    public E take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();  // 队列空，等待
            }
            @SuppressWarnings("unchecked")
            E e = (E) items[takeIndex];
            items[takeIndex] = null;
            if (++takeIndex == items.length) {
                takeIndex = 0;
            }
            count--;
            notFull.signal();  // 通知生产者
            return e;
        } finally {
            lock.unlock();
        }
    }
    
    // 带超时的操作
    public boolean offer(E e, long timeout, TimeUnit unit) 
            throws InterruptedException {
        
        long nanos = unit.toNanos(timeout);
        lock.lockInterruptibly();
        try {
            while (count == items.length) {
                if (nanos <= 0) {
                    return false;
                }
                nanos = notFull.awaitNanos(nanos);
            }
            items[putIndex] = e;
            if (++putIndex == items.length) {
                putIndex = 0;
            }
            count++;
            notEmpty.signal();
            return true;
        } finally {
            lock.unlock();
        }
    }
}
```

```java
public class Turnstile {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition canPass = lock.newCondition();
    private volatile boolean isGreen = false;
    
    public void open() {
        lock.lock();
        try {
            isGreen = true;
            canPass.signalAll();  // 唤醒所有等待的线程
        } finally {
            lock.unlock();
        }
    }
    
    public void close() {
        lock.lock();
        try {
            isGreen = false;
        } finally {
            lock.unlock();
        }
    }
    
    public void waitForGreen() throws InterruptedException {
        lock.lock();
        try {
            while (!isGreen) {
                canPass.await();  // 等待绿灯
            }
        } finally {
            lock.unlock();
        }
    }
}
```

### 场景3：多条件资源池

```java
public class Pool<E> {
    
    private final Set<E> available = new HashSet<>();
    private final Set<E> inUse = new HashSet<>();
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition hasAvailable = lock.newCondition();
    private final Condition hasInUse = lock.newCondition();
    
    public Pool(Collection<E> initialItems) {
        available.addAll(initialItems);
    }
    
    public E acquire() throws InterruptedException {
        lock.lock();
        try {
            while (available.isEmpty()) {
                hasAvailable.await();  // 等待有可用资源
            }
            E item = available.iterator().next();
            available.remove(item);
            inUse.add(item);
            return item;
        } finally {
            lock.unlock();
        }
    }
    
    public void release(E item) {
        lock.lock();
        try {
            inUse.remove(item);
            available.add(item);
            hasAvailable.signal();  // 通知等待获取的线程
        } finally {
            lock.unlock();
        }
    }
    
    public int getAvailableCount() {
        lock.lock();
        try {
            return available.size();
        } finally {
            lock.unlock();
        }
    }
}
```

### 场景4：分布式任务协调

```java
public class TaskCoordinator {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition taskReady = lock.newCondition();
    private final Condition resultReady = lock.newCondition();
    
    private volatile Task currentTask;
    private volatile Result currentResult;
    private volatile boolean taskProcessed = false;
    
    public void submitTask(Task task) throws InterruptedException {
        lock.lock();
        try {
            while (currentTask != null) {
                taskReady.await();  // 等待任务槽空闲
            }
            currentTask = task;
            taskProcessed = false;
            // 通知处理器
        } finally {
            lock.unlock();
        }
    }
    
    public Result waitForResult() throws InterruptedException {
        lock.lock();
        try {
            while (!taskProcessed) {
                resultReady.await();  // 等待结果
            }
            return currentResult;
        } finally {
            lock.unlock();
        }
    }
    
    public void processTask() throws InterruptedException {
        lock.lock();
        try {
            while (currentTask == null) {
                taskReady.await();  // 等待任务
            }
            // 处理任务
            currentResult = currentTask.execute();
            taskProcessed = true;
            currentTask = null;
            resultReady.signal();  // 通知结果已就绪
        } finally {
            lock.unlock();
        }
    }
}
```

## signal vs signalAll

```java
public class SignalVsSignalAll {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition condition = lock.newCondition();
    
    // signal()：只唤醒一个等待线程
    public void singleSignal() {
        lock.lock();
        try {
            condition.signal();  // 唤醒一个线程
        } finally {
            lock.unlock();
        }
    }
    
    // signalAll()：唤醒所有等待线程
    public void allSignal() {
        lock.lock();
        try {
            condition.signalAll();  // 唤醒所有线程
        } finally {
            lock.unlock();
        }
    }
}
```

### 选择建议

- **signal()**：效率更高，适合只需要一个线程处理的情况
- **signalAll()**：更安全，确保所有等待的线程都能继续

```java
public class SignalSelection {
    
    // 使用 signal() 的场景
    public void singleConsumer() {
        // 只有一个消费者，等待数据
        // signal() 唤醒一个就足够了
        lock.lock();
        try {
            while (data == null) {
                condition.await();
            }
            process(data);
        } finally {
            lock.unlock();
        }
    }
    
    // 使用 signalAll() 的场景
    public void allNeedToKnow() {
        // 多个线程等待同一个条件
        // signalAll() 确保所有线程都能感知变化
        lock.lock();
        try {
            state = newState;
            condition.signalAll();  // 所有线程都需要知道状态变化
        } finally {
            lock.unlock();
        }
    }
}
```

## 常见问题

### 问题1：虚假唤醒

```java
public class SpuriousWakeup {
    
    private volatile boolean ready = false;
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition condition = lock.newCondition();
    
    // 错误：使用 if
    public void wrongWait() {
        lock.lock();
        try {
            if (!ready) {  // ❌ 可能虚假唤醒
                condition.await();
            }
            // 处理
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            lock.unlock();
        }
    }
    
    // 正确：使用 while
    public void correctWait() {
        lock.lock();
        try {
            while (!ready) {  // ✓ 循环检查
                condition.await();
            }
            // 处理
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            lock.unlock();
        }
    }
}
```

### 问题2：忘记释放锁

```java
public class LockRelease {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition condition = lock.newCondition();
    
    // 错误：异常时忘记释放锁
    public void wrongMethod() throws InterruptedException {
        lock.lock();
        try {
            condition.await();
            throw new RuntimeException("error");  // 锁未释放
        } finally {
            lock.unlock();
        }
    }
    
    // 正确：使用 finally 确保释放
    public void correctMethod() throws InterruptedException {
        lock.lock();
        try {
            condition.await();
        } finally {
            lock.unlock();  // ✓ 一定释放
        }
    }
}
```

## 总结

`Condition` 是 `ReentrantLock` 的强大伴侣：

- **多条件队列**：每个 Condition 有独立的等待队列
- **精准唤醒**：可以唤醒特定条件的等待者
- **超时等待**：`awaitNanos()` 支持精确的超时控制
- **可中断等待**：`await()` 响应线程中断

与 `Object.wait/notify` 相比：
- `Condition` 支持多个独立条件
- `Condition` 提供更精确的等待/唤醒控制
- `Condition` 必须配合 `ReentrantLock` 使用
