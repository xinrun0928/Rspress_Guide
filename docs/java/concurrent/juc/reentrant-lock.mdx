# ReentrantLock：可重入锁与公平/非公平策略

在 Java 并发编程中，`synchronized` 是最基础的锁机制，但它的功能相对有限。`ReentrantLock`（可重入锁）是 JDK 5 引入的高级锁机制，提供了更强的灵活性和更丰富的功能。与 synchronized 相比，ReentrantLock 支持**公平/非公平**切换、**可中断**获取、**超时获取**等特性，而这些强大功能的背后，都依赖于一个核心组件——**AQS（AbstractQueuedSynchronizer）**。

## 为什么需要 ReentrantLock？

`synchronized` 的局限性：

```java
// synchronized 的局限
public class SynchronizedLimitations {
    
    private final Object lock = new Object();
    
    public void business() {
        synchronized (lock) {
            // 无法尝试获取锁
            // 无法设置超时
            // 无法中断等待
            // 无法实现公平锁
        }
    }
}
```

`ReentrantLock` 的优势：

```java
public class ReentrantLockAdvantages {
    
    private final ReentrantLock lock = new ReentrantLock();
    
    public void business() {
        // 1. tryLock()：尝试获取，可中断
        try {
            if (lock.tryLock(1, TimeUnit.SECONDS)) {
                try {
                    // 业务逻辑
                } finally {
                    lock.unlock();  // 必须在 finally 中释放
                }
            } else {
                // 获取失败
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        
        // 2. lockInterruptibly()：可中断等待
        try {
            lock.lockInterruptibly();
            // 业务逻辑
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}
```

## ReentrantLock 的可重入性

与 synchronized 一样，ReentrantLock 也是可重入的。同一线程可以多次获取同一把锁，而不会死锁。

```java
public class ReentrantDemo {
    
    private final ReentrantLock lock = new ReentrantLock();
    
    public void outer() {
        lock.lock();
        try {
            System.out.println("outer 获取锁，重入计数: " + 
                lock.getHoldCount());
            inner();  // 重入
        } finally {
            lock.unlock();
        }
    }
    
    public void inner() {
        lock.lock();
        try {
            System.out.println("inner 获取锁，重入计数: " + 
                lock.getHoldCount());
        } finally {
            lock.unlock();
        }
    }
    
    public static void main(String[] args) {
        ReentrantDemo demo = new ReentrantDemo();
        demo.outer();
        // 输出：
        // outer 获取锁，重入计数: 1
        // inner 获取锁，重入计数: 2
    }
}
```

可重入的实现原理：在 AQS 的 `state` 中记录重入次数。

## 公平锁 vs 非公平锁

ReentrantLock 支持两种模式：

```java
public class FairVsUnfair {
    
    // 公平锁：严格按照等待顺序获取锁
    private final ReentrantLock fairLock = 
        new ReentrantLock(true);
    
    // 非公平锁：允许插队，可能饥饿
    private final ReentrantLock unfairLock = 
        new ReentrantLock(false);  // 默认
    
    // 公平锁的实现
    static final class FairSync extends Sync {
        protected final boolean tryAcquire(int acquires) {
            Thread current = Thread.currentThread();
            int c = getState();
            if (c == 0) {
                // 关键区别：检查等待队列
                if (!hasQueuedPredecessors() &&  // FIFO 检查
                    compareAndSetState(0, acquires)) {
                    setExclusiveOwnerThread(current);
                    return true;
                }
            } else if (current == getExclusiveOwnerThread()) {
                int nextc = c + acquires;
                setState(nextc);
                return true;
            }
            return false;
        }
    }
    
    // 非公平锁的实现
    static final class NonfairSync extends Sync {
        protected final boolean tryAcquire(int acquires) {
            Thread current = Thread.currentThread();
            int c = getState();
            if (c == 0) {
                // 关键区别：直接 CAS，不检查等待队列
                if (compareAndSetState(0, acquires)) {
                    setExclusiveOwnerThread(current);
                    return true;
                }
            } else if (current == getExclusiveOwnerThread()) {
                int nextc = c + acquires;
                setState(nextc);
                return true;
            }
            return false;
        }
    }
}
```

### 关键区别：hasQueuedPredecessors()

```java
// FairSync 独有的检查
public final boolean hasQueuedPredecessors() {
    Node t = tail;
    Node h = head;
    Node s;
    
    // 如果 head == tail，说明队列为空或只有 head
    // 如果 h.next == null 或 s.thread != current
    // 说明有线程在当前线程之前等待
    return h != t && 
           ((s = h.next) == null || 
            s.thread != Thread.currentThread());
}
```

### 公平锁 vs 非公平锁的选择

| 特性 | 公平锁 | 非公平锁 |
|------|--------|----------|
| **吞吐量** | 较低 | 较高 |
| **等待时间** | 确定性 | 不确定 |
| **饥饿风险** | 无 | 有 |
| **适用场景** | 需要公平性的场景 | 大多数场景 |

```java
// 选择建议
public class LockSelection {
    
    // 场景1：短临界区，高并发 → 非公平锁
    private final ReentrantLock highThroughputLock = 
        new ReentrantLock();  // 默认非公平
    
    // 场景2：长临界区，资源稀缺 → 公平锁
    private final ReentrantLock fairLock = 
        new ReentrantLock(true);
    
    // 场景3：数据库连接池 → 公平锁（避免饥饿）
    private final ReentrantLock dbPoolLock = 
        new ReentrantLock(true);
}
```

## ReentrantLock 的核心方法

```java
public class ReentrantLockMethods {
    
    private final ReentrantLock lock = new ReentrantLock();
    
    public void allMethods() {
        // 1. lock() - 获取锁，不响应中断
        lock.lock();
        try {
            // 业务逻辑
        } finally {
            lock.unlock();
        }
        
        // 2. lockInterruptibly() - 可中断获取
        try {
            lock.lockInterruptibly();
            try {
                // 业务逻辑
            } finally {
                lock.unlock();
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        
        // 3. tryLock() - 尝试获取，立即返回
        if (lock.tryLock()) {
            try {
                // 获取成功
            } finally {
                lock.unlock();
            }
        } else {
            // 获取失败
        }
        
        // 4. tryLock(timeout) - 超时获取
        try {
            if (lock.tryLock(5, TimeUnit.SECONDS)) {
                try {
                    // 业务逻辑
                } finally {
                    lock.unlock();
                }
            } else {
                // 超时
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        
        // 5. unlock() - 释放锁
        lock.unlock();
        
        // 6. isHeldByCurrentThread() - 是否由当前线程持有
        boolean held = lock.isHeldByCurrentThread();
        
        // 7. getHoldCount() - 重入次数
        int count = lock.getHoldCount();
        
        // 8. isLocked() - 是否被任意线程持有
        boolean locked = lock.isLocked();
    }
}
```

## ReentrantLock 与 Condition

`Condition` 是 ReentrantLock 的高级特性，提供了比 `Object.wait/notify` 更灵活的等待/通知机制。

```java
public class ConditionExample {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();
    private final Object[] items = new Object[100];
    private int count = 0;
    
    public void put(Object item) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length) {
                notFull.await();  // 等待不满
            }
            items[count++] = item;
            notEmpty.signal();  // 通知非空
        } finally {
            lock.unlock();
        }
    }
    
    public Object take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();  // 等待非空
            }
            Object item = items[--count];
            notFull.signal();  // 通知不满
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

一个 ReentrantLock 可以创建多个 Condition：

```java
// 与 Object.wait/notify 的对比
public class ConditionVsObjectWait {
    
    // Object.wait/notify 的局限
    private final Object primitiveLock = new Object();
    
    public void primitiveApproach() throws InterruptedException {
        synchronized (primitiveLock) {
            // 只能有一个等待条件
            while (conditionA) {
                primitiveLock.wait();
            }
            // 无法区分是哪个条件被满足
        }
    }
    
    // Condition 的优势
    private final ReentrantLock advancedLock = new ReentrantLock();
    private final Condition conditionA = advancedLock.newCondition();
    private final Condition conditionB = advancedLock.newCondition();
    
    public void advancedApproach() throws InterruptedException {
        advancedLock.lock();
        try {
            // 可以有多个独立的等待条件
            while (!conditionACheck()) {
                conditionA.await();
            }
            
            while (!conditionBCheck()) {
                conditionB.await();
            }
            
            // 精确通知
            if (conditionACheck()) {
                conditionA.signal();  // 只唤醒 conditionA 的等待者
            }
        } finally {
            advancedLock.unlock();
        }
    }
}
```

## ReentrantLock 的实现原理

### 核心数据结构：AQS

```java
// AQS 的核心状态
public abstract class AbstractQueuedSynchronizer 
        extends AbstractOwnableSynchronizer {
    
    // 同步状态
    private volatile int state;
    
    // 等待队列
    private transient volatile Node head;
    private transient volatile Node tail;
    
    // Node 定义
    static final class Node {
        volatile int waitStatus;
        volatile Node prev;
        volatile Node next;
        volatile Thread thread;
        
        Node nextWaiter;  // 共享模式或独占模式
        
        static final int EXCLUSIVE = null;  // 独占模式
        static final int SHARED = "SHARED"; // 共享模式
    }
}
```

### 获取锁的流程

```java
// ReentrantLock.Lock() 的完整流程
public void lock() {
    sync.lock();
}

// NonfairSync.lock()
final void lock() {
    // 1. 第一次尝试：直接 CAS 获取
    if (compareAndSetState(0, 1)) {
        setExclusiveOwnerThread(Thread.currentThread());
        return;
    }
    
    // 2. 获取失败，调用 acquire（模板方法）
    acquire(1);
}

// AQS.acquire() - 模板方法
public final void acquire(int arg) {
    if (!tryAcquire(arg) &&           // 尝试获取
        acquireQueued(addWaiter(Node.EXCLUSIVE), arg)) {  // 入队等待
        selfInterrupt();
    }
}

// NonfairSync.tryAcquire()
protected final boolean tryAcquire(int acquires) {
    return nonfairTryAcquire(acquires);
}

// Sync.nonfairTryAcquire()
final boolean nonfairTryAcquire(int acquires) {
    Thread current = Thread.currentThread();
    int c = getState();
    
    if (c == 0) {
        // 状态为0，尝试 CAS
        if (compareAndSetState(0, acquires)) {
            setExclusiveOwnerThread(current);
            return true;
        }
    } else if (current == getExclusiveOwnerThread()) {
        // 可重入：增加计数
        int nextc = c + acquires;
        setState(nextc);
        return true;
    }
    return false;
}
```

### 释放锁的流程

```java
// ReentrantLock.unlock() 的完整流程
public void unlock() {
    sync.release(1);
}

// AQS.release() - 模板方法
public final boolean release(int arg) {
    if (tryRelease(arg)) {
        Node h = head;
        if (h != null && h.waitStatus != 0) {
            unparkSuccessor(h);  // 唤醒后继节点
        }
        return true;
    }
    return false;
}

// Sync.tryRelease()
protected final boolean tryRelease(int releases) {
    int c = getState() - releases;
    
    if (Thread.currentThread() != getExclusiveOwnerThread()) {
        throw new IllegalMonitorStateException();
    }
    
    boolean free = false;
    if (c == 0) {
        // 状态为0，完全释放
        free = true;
        setExclusiveOwnerThread(null);
    }
    setState(c);
    return free;
}
```

## 实战：ReentrantLock 应用场景

### 场景1：实现缓存

```java
public class CacheDemo<K, V> {
    
    private final Map<K, V> cache = new ConcurrentHashMap<>();
    private final ReentrantLock lock = new ReentrantLock();
    
    public V get(K key, Supplier<V> loader) {
        // 先尝试从缓存获取
        V value = cache.get(key);
        if (value != null) {
            return value;
        }
        
        // 缓存未命中，加载数据
        lock.lock();
        try {
            // 双重检查
            value = cache.get(key);
            if (value != null) {
                return value;
            }
            
            // 加载并缓存
            value = loader.get();
            cache.put(key, value);
            return value;
        } finally {
            lock.unlock();
        }
    }
}
```

### 场景2：实现信号量

```java
public class SemaphoreDemo {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();
    private final int[] buffer;
    private int count = 0;
    private int in = 0;
    private int out = 0;
    
    public SemaphoreDemo(int size) {
        this.buffer = new int[size];
    }
    
    public void produce(int item) throws InterruptedException {
        lock.lock();
        try {
            while (count == buffer.length) {
                notFull.await();
            }
            buffer[in] = item;
            in = (in + 1) % buffer.length;
            count++;
            notEmpty.signal();
        } finally {
            lock.unlock();
        }
    }
    
    public int consume() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();
            }
            int item = buffer[out];
            out = (out + 1) % buffer.length;
            count--;
            notFull.signal();
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

## 总结

`ReentrantLock` 相比 `synchronized` 提供了更丰富的功能：

- **可重入**：同一线程可多次获取
- **公平/非公平**：可选择是否保证 FIFO
- **可中断**：`lockInterruptibly()` 允许等待时被中断
- **超时获取**：`tryLock(timeout)` 避免无限等待
- **多条件变量**：`newCondition()` 支持多个独立等待条件

这些功能的实现都依赖于 AQS 的精巧设计。你是否曾好奇，为什么 ReentrantLock 叫"可重入锁"？答案就藏在其内部维护的 state 计数器中。
