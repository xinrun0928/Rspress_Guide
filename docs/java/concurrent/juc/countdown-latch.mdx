# CountDownLatch：线程倒计时协调

在日常生活和工作中，我们经常需要等待某些前置任务完成才能开始后续工作。比如在组织一场会议时，主持人需要等待所有参会人员都到齐才能开始；在并行计算中，主任务需要等待所有子任务完成才能汇总结果。`CountDownLatch`（倒计时门闩）正是为这种场景设计的——它让一个或多个线程等待其他线程完成一组操作后再继续执行。

## CountDownLatch 的基本概念

```java
public class CountDownLatchConcept {
    
    // 创建倒计时器，初始计数为3
    private final CountDownLatch latch = new CountDownLatch(3);
    
    // 工作线程：完成工作后计数减1
    public void worker() {
        try {
            // 执行任务
            doWork();
        } finally {
            latch.countDown();  // 计数减1
        }
    }
    
    // 主线程：等待计数归零
    public void awaitAll() throws InterruptedException {
        latch.await();  // 阻塞，直到计数为0
        System.out.println("所有工作已完成");
    }
}
```

### 核心方法

```java
public class CountDownLatchMethods {
    
    private final CountDownLatch latch = new CountDownLatch(3);
    
    public void allMethods() throws InterruptedException {
        
        // 1. await() - 等待计数归零
        latch.await();  // 一直等待
        
        // 2. await(timeout) - 超时等待
        boolean completed = latch.await(5, TimeUnit.SECONDS);
        if (!completed) {
            System.out.println("等待超时");
        }
        
        // 3. countDown() - 计数减1
        latch.countDown();
        
        // 4. getCount() - 获取当前计数
        long count = latch.getCount();
    }
}
```

## CountDownLatch 的实现原理

`CountDownLatch` 内部依赖 AQS 的共享模式实现：

```java
public class CountDownLatch extends AbstractQueuedSynchronizer {
    
    // 内部类：Sync
    private static final class Sync extends AbstractQueuedSynchronizer {
        private final int count;
        
        Sync(int count) {
            this.count = count;
            setState(count);
        }
        
        // 获取共享锁
        protected int tryAcquireShared(int ignores) {
            return getState() == 0 ? 1 : -1;
        }
        
        // 释放共享锁
        protected boolean tryReleaseShared(int releases) {
            for (;;) {
                int c = getState();
                if (c == 0) {
                    return false;  // 已经归零
                }
                int nextc = c - 1;
                if (compareAndSetState(c, nextc)) {
                    return nextc == 0;  // 返回是否归零
                }
            }
        }
    }
    
    private final Sync sync;
    
    public void await() throws InterruptedException {
        sync.acquireSharedInterruptibly(1);
    }
    
    public void countDown() {
        sync.releaseShared(1);
    }
}
```

## 实战应用场景

### 场景1：并行计算

```java
public class ParallelComputation {
    
    private final double[] data;
    private final int threadCount;
    private final CountDownLatch latch;
    
    public ParallelComputation(double[] data, int threadCount) {
        this.data = data;
        this.threadCount = threadCount;
        this.latch = new CountDownLatch(threadCount);
    }
    
    public double computeSum() throws InterruptedException {
        double[] partialSums = new double[threadCount];
        int chunkSize = data.length / threadCount;
        
        // 启动多个线程并行计算
        for (int i = 0; i < threadCount; i++) {
            final int threadIndex = i;
            final int start = i * chunkSize;
            final int end = (i == threadCount - 1) ? data.length : (i + 1) * chunkSize;
            
            new Thread(() -> {
                double sum = 0;
                for (int j = start; j < end; j++) {
                    sum += data[j];
                }
                partialSums[threadIndex] = sum;
                latch.countDown();
            }).start();
        }
        
        // 等待所有线程完成
        latch.await();
        
        // 汇总结果
        double total = 0;
        for (double partial : partialSums) {
            total += partial;
        }
        return total;
    }
}
```

### 场景3：多数据源查询

```java
public class MultiDataSourceQuery {
    
    public QueryResult query(List<DataSource> sources, String query) 
            throws InterruptedException {
        
        Map<String, Object> results = new ConcurrentHashMap<>();
        CountDownLatch latch = new CountDownLatch(sources.size());
        AtomicReference<Exception> error = new AtomicReference<>();
        
        for (DataSource source : sources) {
            new Thread(() -> {
                try {
                    Object result = source.execute(query);
                    results.put(source.getName(), result);
                } catch (Exception e) {
                    error.set(e);
                } finally {
                    latch.countDown();
                }
            }).start();
        }
        
        // 等待所有数据源返回
        latch.await(10, TimeUnit.SECONDS);
        
        if (error.get() != null) {
            throw new RuntimeException("查询失败", error.get());
        }
        
        return new QueryResult(results);
    }
}
```

### 场景4：压力测试

```java
public class LoadTester {
    
    public void runLoadTest(int totalRequests, int concurrency) 
            throws InterruptedException {
        
        CountDownLatch startLatch = new CountDownLatch(1);  // 控制所有线程同时开始
        CountDownLatch endLatch = new CountDownLatch(concurrency);  // 等待所有线程结束
        
        AtomicInteger successCount = new AtomicInteger();
        AtomicInteger failCount = new AtomicInteger();
        
        for (int i = 0; i < concurrency; i++) {
            new Thread(() -> {
                try {
                    // 所有线程等待主线程发出开始信号
                    startLatch.await();
                    
                    // 执行请求
                    int requestsPerThread = totalRequests / concurrency;
                    for (int j = 0; j < requestsPerThread; j++) {
                        if (executeRequest()) {
                            successCount.incrementAndGet();
                        } else {
                            failCount.incrementAndGet();
                        }
                    }
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                } finally {
                    endLatch.countDown();
                }
            }).start();
        }
        
        // 主线程：发出开始信号
        System.out.println("开始压测");
        startLatch.countDown();  // 所有线程同时开始
        
        // 等待所有线程结束
        endLatch.await();
        
        System.out.println("压测完成");
        System.out.println("成功: " + successCount.get());
        System.out.println("失败: " + failCount.get());
    }
    
    private boolean executeRequest() {
        // 执行请求
        return true;
    }
}
```

## CountDownLatch vs CyclicBarrier

| 特性 | CountDownLatch | CyclicBarrier |
|------|----------------|---------------|
| 计数方向 | 递减到0 | 递增到 parties |
| 可重用 | 否（一次性） | 是（可循环） |
| 等待线程 | 任意数量 | 固定数量（parties） |
| 完成后的动作 | 唤醒等待线程 | 执行 barrierAction |
| 典型用途 | 等待多任务完成 | 多线程汇合点 |

```java
public class ComparisonDemo {
    
    // CountDownLatch：一次性，等待多个事件
    public void countDownLatchUsage() throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(2);
        
        new Thread(() -> {
            doTaskA();
            latch.countDown();
        }).start();
        
        new Thread(() -> {
            doTaskB();
            latch.countDown();
        }).start();
        
        latch.await();  // 等待 taskA 和 taskB 都完成
        doTaskC();      // 然后执行 taskC
    }
    
    // CyclicBarrier：可循环，多线程汇合
    public void cyclicBarrierUsage() throws InterruptedException {
        CyclicBarrier barrier = new CyclicBarrier(2, () -> {
            System.out.println("所有线程都到达了");
        });
        
        new Thread(() -> {
            doTask();
            barrier.await();  // 等待另一个线程
        }).start();
        
        new Thread(() -> {
            doTask();
            barrier.await();  // 等待另一个线程
        }).start();
    }
    
    private void doTaskA() {}
    private void doTaskB() {}
    private void doTaskC() {}
    private void doTask() {}
}
```

## 常见错误

### 错误1：计数设置错误

```java
public class WrongUsage {
    
    public void wrongCount() {
        // 错误：计数为0，门闩已打开
        CountDownLatch latch = new CountDownLatch(0);
        // await() 会立即返回，可能导致逻辑错误
    }
    
    public void correctCount(int taskCount) {
        // 正确：计数等于任务数量
        CountDownLatch latch = new CountDownLatch(taskCount);
    }
}
```

### 错误2：忘记 countDown

```java
public class ForgetCountDown {
    
    private final CountDownLatch latch = new CountDownLatch(2);
    
    public void wrongMethod() {
        try {
            doTask1();
            // 忘记 latch.countDown();
            latch.await();  // 会永远等待
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    public void correctMethod() {
        try {
            doTask1();
            latch.countDown();  // 确保计数
        } finally {
            // 不需要在 finally 中调用 await
        }
    }
}
```

## 总结

`CountDownLatch` 是协调多线程执行顺序的利器：

- **核心思想**：用计数器控制等待，一个或多个线程等待计数器归零
- **一次性**：计数归零后不可重置
- **不可中断的等待**：`await()` 可响应中断
- **典型场景**：等待多任务完成、服务启动、压力测试

与 `CyclicBarrier` 的区别：
- `CountDownLatch`：计数器递减，不可重用
- `CyclicBarrier`：计数器递增，可重用，适合多线程反复汇合
