# Semaphore：信号量控制并发数

信号量（Semaphore）是操作系统中的经典同步原语，JDK 将其引入 Java，为我们提供了一种灵活控制**同时访问资源的线程数量**的能力。想象一个停车场的入口：它有 N 个车位，每当一辆车进入，剩余车位减一；每当一辆车离开，剩余车位加一。如果车位已满，后来的车辆必须等待。`Semaphore` 就是这样一位尽职的"停车场管理员"。

## Semaphore 的基本概念

```java
public class SemaphoreConcept {
    
    // 创建信号量：同时允许3个线程访问
    private final Semaphore semaphore = new Semaphore(3);
    
    public void access() {
        try {
            // 获取许可证（acquire）
            semaphore.acquire();
            try {
                // 访问共享资源
                System.out.println("线程 " + Thread.currentThread().getName() + " 正在访问");
                Thread.sleep(1000);
            } finally {
                // 释放许可证（release）
                semaphore.release();
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

### 核心方法

```java
public class SemaphoreMethods {
    
    private final Semaphore semaphore = new Semaphore(3);
    
    public void allMethods() throws InterruptedException {
        
        // 1. acquire() - 获取一个许可证，阻塞直到可用
        semaphore.acquire();
        try {
            // 业务逻辑
        } finally {
            semaphore.release();
        }
        
        // 2. acquire(n) - 获取多个许可证
        semaphore.acquire(2);
        try {
            // 需要2个许可证的操作
        } finally {
            semaphore.release(2);
        }
        
        // 3. tryAcquire() - 尝试获取，立即返回
        if (semaphore.tryAcquire()) {
            try {
                // 获取成功
            } finally {
                semaphore.release();
            }
        } else {
            // 获取失败
        }
        
        // 4. tryAcquire(timeout) - 超时获取
        if (semaphore.tryAcquire(5, TimeUnit.SECONDS)) {
            try {
                // 获取成功
            } finally {
                semaphore.release();
            }
        } else {
            // 超时
        }
        
        // 5. release() - 释放许可证
        semaphore.release();
        
        // 6. availablePermits() - 查看可用许可证数
        int available = semaphore.availablePermits();
        
        // 7. drainPermits() - 获取并清空所有可用许可证
        int drained = semaphore.drainPermits();
        
        // 8. reducePermits(n) - 减少许可证数
        semaphore.reducePermits(2);
    }
}
```

## Semaphore 的实现原理

`Semaphore` 内部使用 **AQS 的共享模式**实现：

```java
public class Semaphore extends AbstractQueuedSynchronizer {
    
    private final Sync sync;
    
    // 非公平同步器
    static final class NonfairSync extends Sync {
        protected int tryAcquireShared(int permits) {
            return nonfairTryAcquireShared(permits);
        }
    }
    
    // 公平同步器
    static final class FairSync extends Sync {
        protected int tryAcquireShared(int permits) {
            for (;;) {
                // 公平模式：检查等待队列
                if (hasQueuedPredecessors()) {
                    return -1;
                }
                int available = getState();
                int next = available - permits;
                if (next < 0 || compareAndSetState(available, next)) {
                    return next;
                }
            }
        }
    }
    
    // 核心获取逻辑
    final int nonfairTryAcquireShared(int permits) {
        for (;;) {
            int available = getState();
            int remaining = available - permits;
            
            if (remaining < 0 ||
                compareAndSetState(available, remaining)) {
                return remaining;
            }
        }
    }
}
```

### 许可证的存储

Semaphore 将许可证数量存储在 AQS 的 `state` 字段中：

```
┌────────────────────────────────────────────────────────────────┐
│                      AQS state                                  │
├────────────────────────────────────────────────────────────────┤
│                                                                  │
│   state = 3  表示有3个许可证可用                                   │
│                                                                  │
│   每 acquire(1) 一次，state 减 1                                 │
│   每 release(1) 一次，state 加 1                                 │
│                                                                  │
│   当 state = 0 时，其他线程的 acquire 会阻塞                       │
│                                                                  │
└────────────────────────────────────────────────────────────────┘
```

## 实战应用场景

### 场景1：限流器

```java
public class RateLimiter {
    
    private final Semaphore permits;
    
    public RateLimiter(int maxConcurrent) {
        this.permits = new Semaphore(maxConcurrent);
    }
    
    // 限流执行
    public <T> T execute(Supplier<T> task) throws InterruptedException {
        permits.acquire();
        try {
            return task.get();
        } finally {
            permits.release();
        }
    }
    
    // 限流执行（带超时）
    public <T> Optional<T> executeWithTimeout(
            Supplier<T> task, 
            long timeout, 
            TimeUnit unit) throws InterruptedException {
        
        if (permits.tryAcquire(timeout, unit)) {
            try {
                return Optional.ofNullable(task.get());
            } finally {
                permits.release();
            }
        }
        return Optional.empty();
    }
}

// 使用示例
public class RateLimiterDemo {

                // 实际处理逻辑
                return handleRequest(request);
            });
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

### 场景3：任务限流

```java
public class TaskThrottler {
    
    private final Semaphore semaphore;
    private final ExecutorService executor;
    
    public TaskThrottler(int maxConcurrent, ExecutorService executor) {
        this.semaphore = new Semaphore(maxConcurrent);
        this.executor = executor;
    }
    
    public Future<?> submit(Runnable task) {
        return executor.submit(() -> {
            try {
                semaphore.acquire();
                try {
                    task.run();
                } finally {
                    semaphore.release();
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
    }
    
    public <T> Future<T> submit(Callable<T> task) {
        return executor.submit(() -> {
            try {
                semaphore.acquire();
                try {
                    return task.call();
                } finally {
                    semaphore.release();
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                throw new RuntimeException(e);
            }
        });
    }
}
```

### 场景4：资源池

```java
public class ResourcePool<T> {
    
    private final Set<T> resources = new HashSet<>();
    private final Semaphore semaphore;
    private final boolean fair;
    
    public ResourcePool(Collection<T> initialResources, boolean fair) {
        this.fair = fair;
        this.resources.addAll(initialResources);
        this.semaphore = new Semaphore(initialResources.size(), fair);
    }
    
    public T acquire() throws InterruptedException {
        semaphore.acquire();
        return takeResource();
    }
    
    public void release(T resource) {
        if (returnResource(resource)) {
            semaphore.release();
        }
    }
    
    private synchronized T takeResource() {
        Iterator<T> iter = resources.iterator();
        if (iter.hasNext()) {
            return iter.next();
        }
        throw new IllegalStateException("No resources available");
    }
    
    private synchronized boolean returnResource(T resource) {
        return resources.add(resource);
    }
}
```


## 与其他并发工具的对比

| 工具 | 用途 | 特点 |
|------|------|------|
| `Semaphore` | 控制并发数 | 固定数量许可证 |
| `CountDownLatch` | 一次性同步 | 计数递减到0 |
| `CyclicBarrier` | 多线程汇合 | 全部到达后一起继续 |
| `ReentrantLock` | 互斥 | 独占访问 |

## 总结

`Semaphore` 是控制并发访问的强大工具：

- **核心思想**：用许可证控制同时访问资源的线程数
- **基本操作**：`acquire()` 获取许可证，`release()` 释放许可证
- **支持超时**：`tryAcquire()` 支持非阻塞和超时获取
- **公平性**：可选公平和非公平模式

使用场景：
1. 限流器：控制同时请求数
2. 连接池：限制数据库连接数
3. 资源池：管理有限的物理资源
