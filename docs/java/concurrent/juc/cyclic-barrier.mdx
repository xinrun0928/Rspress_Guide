# CyclicBarrier：循环栅栏

想象一个登山队在山顶集合的场景：队员们从不同的路线出发，但必须全员到齐才能一起冲顶。`CyclicBarrier`（循环栅栏）正是为这种场景设计的——它让一组线程相互等待，直到所有线程都到达某个"汇合点"后，再一起继续执行。这与 `CountDownLatch` 有本质区别：`CountDownLatch` 是"倒计时"，一旦归零就结束；而 `CyclicBarrier` 可以"循环"使用，线程到达后等待，直到所有人都到齐，然后一起继续，之后又可以再次使用。

## CyclicBarrier 的基本概念

```java
public class CyclicBarrierConcept {
    
    // 创建栅栏， parties = 3 表示需要3个线程同时到达
    private final CyclicBarrier barrier = new CyclicBarrier(3);
    
    // 线程执行到这个点会等待
    public void task() {
        doPartialWork();
        
        try {
            barrier.await();  // 等待其他线程
            // 所有线程都到达这里后，一起继续执行
            doFinalWork();
        } catch (InterruptedException | BrokenBarrierException e) {
            // 处理异常
        }
    }
}
```

## CyclicBarrier 的实现原理

`CyclicBarrier` 内部使用 `ReentrantLock` 和 `Condition` 实现：

```java
public class CyclicBarrier {
    
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition trip = lock.newCondition();
    
    private final int parties;  // 需要到达的线程数
    private final Runnable barrierCommand;  // 到达后执行的命令
    
    private int count;  // 还需要等待的线程数
    private Generation generation;  // 代数，用于区分循环
    
    // Generation：用于处理 BrokenBarrierException
    private static class Generation {
        boolean broken = false;
    }
    
    public CyclicBarrier(int parties, Runnable barrierAction) {
        this.parties = parties;
        this.count = parties;
        this.barrierCommand = barrierAction;
        this.generation = new Generation();
    }
    
    public int await() throws InterruptedException, BrokenBarrierException {
        lock.lock();
        try {
            int index = --count;
            
            if (index == 0) {  // 最后一个线程到达
                Runnable command = barrierCommand;
                if (command != null) {
                    command.run();  // 执行 barrierAction
                }
                nextGeneration();  // 开启新一代
                return 0;
            }
            
            // 不是最后一个线程，等待
            for (;;) {
                try {
                    trip.await();  // 等待
                    break;
                } catch (InterruptedException ie) {
                    if (g != generation || g.broken) {
                        throw ie;
                    }
                }
            }
            
            if (g.broken) {
                throw new BrokenBarrierException();
            }
            
            return index;
        } finally {
            lock.unlock();
        }
    }
    
    // 开启新一代
    private void nextGeneration() {
        trip.signalAll();  // 唤醒所有等待的线程
        count = parties;    // 重置计数
        generation = new Generation();  // 新一代
    }
}
```

```
│   初始：parties = 3, count = 3                                 │

```

## 实战应用场景

### 场景1：并行计算汇总

```java
public class ParallelSum {
    
    private final double[] data;
    private final int threadCount;
    private final CyclicBarrier barrier;
    
    public ParallelSum(double[] data, int threadCount) {
        this.data = data;
        this.threadCount = threadCount;
        this.barrier = new CyclicBarrier(threadCount, () -> {
            System.out.println("所有分片计算完成，汇总结果");
        });
    }
    
    public double compute() throws InterruptedException {
        double[] results = new double[threadCount];
        int chunkSize = data.length / threadCount;
        CountDownLatch doneSignal = new CountDownLatch(threadCount);
        
        for (int i = 0; i < threadCount; i++) {
            final int index = i;
            final int start = i * chunkSize;
            final int end = (i == threadCount - 1) ? data.length : (i + 1) * chunkSize;
            
            new Thread(() -> {
                try {
                    double sum = 0;
                    for (int j = start; j < end; j++) {
                        sum += data[j];
                    }
                    results[index] = sum;
                    
                    barrier.await();  // 等待其他线程
                    // 所有线程都到达后，继续执行
                    
                    // 可以在这里做汇总操作
                } catch (Exception e) {
                    Thread.currentThread().interrupt();
                } finally {
                    doneSignal.countDown();
                }
            }).start();
        }
        
        doneSignal.await();
        return Arrays.stream(results).sum();
    }
}
```

```java
public class MultiFileProcessor {
    
    private final CyclicBarrier barrier;
    private final List<String> filePaths;
    private final Map<String, Result> results = new ConcurrentHashMap<>();
    
    public MultiFileProcessor(List<String> filePaths) {
        this.filePaths = filePaths;
        this.barrier = new CyclicBarrier(filePaths.size(), () -> {
            // 所有文件处理完成后，生成报告
            generateReport();
        });
    }
    
    public void processAll() throws InterruptedException {
        List<Thread> threads = new ArrayList<>();
        
        for (String path : filePaths) {
            threads.add(new Thread(() -> {
                try {
                    Result result = processFile(path);
                    results.put(path, result);
                    
                    // 等待其他文件处理完成
                    barrier.await();
                    
                } catch (Exception e) {
                    // 处理异常
                }
            }));
        }
        
        threads.forEach(Thread::start);
    }
    
    private Result processFile(String path) {
        // 处理文件的逻辑
        return new Result();
    }
    
    private void generateReport() {
        System.out.println("所有文件处理完成，共处理 " + results.size() + " 个文件");
    }
}
```

### 场景4：分阶段计算

```java
public class StagedComputation {
    
    private final CyclicBarrier barrier1 = new CyclicBarrier(3);
    private final CyclicBarrier barrier2 = new CyclicBarrier(3);
    private final CyclicBarrier barrier3 = new CyclicBarrier(3);
    
    public void execute() throws InterruptedException {
        for (int i = 0; i < 3; i++) {
            final int threadId = i;
            
            new Thread(() -> {
                try {
                    // 阶段1：初始化
                    init(threadId);
                    barrier1.await();
                    
                    // 阶段2：计算
                    compute(threadId);
                    barrier2.await();
                    
                    // 阶段3：汇总
                    aggregate(threadId);
                    barrier3.await();
                    
                } catch (Exception e) {
                    // 处理异常
                }
            }).start();
        }
    }
    
    private void init(int id) { System.out.println("初始化 " + id); }
    private void compute(int id) { System.out.println("计算 " + id); }
    private void aggregate(int id) { System.out.println("汇总 " + id); }
}
```

## BrokenBarrierException

当栅栏被"破坏"时，会抛出 `BrokenBarrierException`：

```java
public class BrokenBarrierDemo {
    
    private final CyclicBarrier barrier = new CyclicBarrier(3);
    
    public void brokenBarrierDemo() throws InterruptedException {
        Thread t1 = new Thread(() -> {
            try {
                barrier.await();
            } catch (BrokenBarrierException e) {
                System.out.println("栅栏已被破坏");
            }
        });
        
        t1.start();
        Thread.sleep(100);
        
        // 重置栅栏（模拟异常情况）
        barrier.reset();
        
        // 之前等待的线程会收到 BrokenBarrierException
    }
}
```

### 栅栏被破坏的情况

1. 等待中的线程被中断
2. 调用 `reset()` 方法
3. 调用 `await()` 超时

## CyclicBarrier vs CountDownLatch

| 特性 | CyclicBarrier | CountDownLatch |
|------|---------------|----------------|
| 计数方向 | 递增到 parties | 递减到 0 |
| 可重用 | 是 | 否 |
| 等待线程 | 固定数量 | 任意数量 |
| 完成后动作 | barrierAction | 唤醒等待线程 |
| 典型场景 | 多线程汇合 | 等待多任务完成 |

```java
public class CompareDemo {
    
    // CountDownLatch：一次性倒计时
    // 适合：主线程等待多个子任务完成
    public void countDownLatchExample() throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(3);
        
        for (int i = 0; i < 3; i++) {
            new Thread(() -> {
                doWork();
                latch.countDown();  // 计数递减
            }).start();
        }
        
        latch.await();  // 等待计数归零
        doFinalWork();
    }
    
    // CyclicBarrier：可循环的汇合点
    // 适合：多个线程互相等待，全部到达后再继续
    public void cyclicBarrierExample() throws InterruptedException {
        CyclicBarrier barrier = new CyclicBarrier(3);
        
        for (int i = 0; i < 3; i++) {
            new Thread(() -> {
                doPartialWork();
                barrier.await();  // 等待所有人到达
                doFinalWork();    // 所有人一起继续
            }).start();
        }
    }
    
    private void doWork() {}
    private void doPartialWork() {}
    private void doFinalWork() {}
}
```

## 总结

`CyclicBarrier` 是多线程汇合同步的好帮手：

- **核心思想**：一组线程相互等待，全部到达后一起继续执行
- **可循环**：完成后自动重置，可重复使用
- **可选动作**：支持 barrierAction，在所有线程到达后执行
- **BrokenBarrier**：支持异常检测，线程中断或重置会破坏栅栏

使用场景：
1. 多线程分阶段计算，需要同步点
2. 多数据源并行查询后汇总
3. 游戏多玩家加载同步
