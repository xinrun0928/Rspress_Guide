# ThreadLocal 详解

**ThreadLocal** 是 Java 提供的线程本地存储机制。它为每个线程提供了独立的变量副本，实现了线程间的数据隔离。

## 为什么需要 ThreadLocal

### 线程共享的问题

```java
public class SharedData {
    private String data = "";  // 所有线程共享

    public void set(String data) {
        this.data = data;  // 可能被其他线程修改
    }
}
```

### ThreadLocal 的解决方案

```java
public class ThreadLocalData {
    private static final ThreadLocal<String> data = new ThreadLocal<>();

    public static void set(String value) {
        data.set(value);  // 每个线程都有自己的副本
    }

    public static String get() {
        return data.get();  // 获取当前线程的副本
    }
}
```

## 基本用法

### 创建和使用

```java
public class ThreadLocalDemo {
    private static final ThreadLocal<Integer> threadId = ThreadLocal.withInitial(() -> {
        return Thread.currentThread().hashCode();
    });

    public static void main(String[] args) {
        System.out.println("主线程: " + threadId.get());

        new Thread(() -> {
            System.out.println("子线程1: " + threadId.get());
        }).start();

        new Thread(() -> {
            System.out.println("子线程2: " + threadId.get());
        }).start();
    }
}
```

### 常用方法

```java
threadLocal.set(value);     // 设置值
threadLocal.get();          // 获取值
threadLocal.remove();       // 移除值
ThreadLocal.withInitial();  // 创建带初始值的 ThreadLocal
```

## 实际应用

### 1. 存储用户上下文

```java
public class UserContext {
    private static final ThreadLocal<User> currentUser = new ThreadLocal<>();

    public static void setUser(User user) {
        currentUser.set(user);
    }

    public static User getUser() {
        return currentUser.get();
    }

    public static void clear() {
        currentUser.remove();
    }
}

// 使用
public class UserInterceptor {
    public void intercept(HttpRequest request) {
        User user = authenticate(request);
        UserContext.setUser(user);
        try {
            // 业务逻辑
        } finally {
            UserContext.clear();  // 清理
        }
    }
}
```

### 2. 日期格式化

```java
public class DateFormatter {
    private static final ThreadLocal<DateFormat> df = ThreadLocal.withInitial(() -> {
        return new SimpleDateFormat("yyyy-MM-dd");
    });

    public static String format(Date date) {
        return df.get().format(date);
    }
}
```

### 3. 事务管理

```java
public class TransactionManager {
    private static final ThreadLocal<Transaction> currentTx = new ThreadLocal<>();

    public static void begin() {
        Transaction tx = new Transaction();
        currentTx.set(tx);
    }

    public static Transaction getCurrent() {
        return currentTx.get();
    }

    public static void commit() {
        Transaction tx = currentTx.get();
        if (tx != null) {
            tx.commit();
            currentTx.remove();
        }
    }
}
```

## ThreadLocal 内存泄漏

### 原因

```
┌────────────────────────────────────────────────────────────┐
│                    内存泄漏原因                               │
│                                                              │
│   ThreadLocalMap 的 Entry 引用 ThreadLocal                  │
│   ┌──────────────────────────────────────────────────┐  │
│   │  ThreadLocalMap                                    │  │
│   │  ┌──────────────────────────────────────────┐   │  │
│   │  │  Entry extends WeakReference<ThreadLocal> │   │  │
│   │  │  key (弱引用)    │  value (强引用)       │   │  │
│   │  └──────────────────────────────────────────┘   │  │
│   └──────────────────────────────────────────────────┘  │
│                                                              │
│   当 ThreadLocal 对象没有强引用时，会被 GC 回收              │
│   但 value 仍然被 Entry 持有，导致内存泄漏                   │
│                                                              │
└────────────────────────────────────────────────────────────┘
```

### 解决方案

```java
public class SafeThreadLocal<T> {
    private final ThreadLocal<T> threadLocal = new ThreadLocal<>();

    public void set(T value) {
        try {
            threadLocal.set(value);
        } finally {
            // 使用后移除
            // threadLocal.remove();
        }
    }

    public T get() {
        try {
            return threadLocal.get();
        } finally {
            threadLocal.remove();
        }
    }
}
```

### 最佳实践

```java
public class BestPractice {
    private static final ThreadLocal<String> local = new ThreadLocal<>();

    public void process() {
        try {
            local.set("value");
            // 业务逻辑
        } finally {
            local.remove();  // 始终清理
        }
    }
}
```

## 面试核心问题

**问题1**：ThreadLocal 是什么？解决了什么问题？

**答案**：ThreadLocal 为每个线程提供独立的变量副本，实现线程间的数据隔离。常用于存储用户上下文、事务管理、日志追踪等。

**问题2**：ThreadLocal 为什么会内存泄漏？

**答案**：ThreadLocalMap 的 Entry 继承 WeakReference，key（ThreadLocal）可能被 GC 回收，但 value 仍然被强引用，导致内存泄漏。

**问题3**：如何避免 ThreadLocal 内存泄漏？

**答案**：使用完 ThreadLocal 后调用 remove() 方法清理；在 finally 块中清理；使用 inheritThreadLocals 实现父子线程共享。

**问题4**：ThreadLocal 和 synchronized 的区别？

**答案**：synchronized 通过锁实现线程同步；ThreadLocal 通过副本机制实现线程隔离。前者共享变量，后者隔离变量。

ThreadLocal 是 Java 并发编程的重要工具，但需要正确使用以避免内存泄漏。
