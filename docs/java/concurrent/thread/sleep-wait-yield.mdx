# sleep、wait、yield：三剑客的区别

在多线程编程中，`Thread` 类提供了三个看似相似但实际上完全不同的工作方法：`sleep()`、`wait()` 和 `yield()`。它们都能影响线程的执行，但原理和适用场景截然不同。理解它们的区别，是编写正确并发代码的基础。

## sleep()：线程睡眠

`Thread.sleep()` 静态方法会使当前线程进入计时等待（TIMED_WAITING）状态，暂停执行一段时间。睡眠期间，线程不会释放任何锁（如果有的话），也不会参与 CPU 调度。

### 基本用法

```java
public class SleepDemo {
    public static void main(String[] args) throws InterruptedException {
        System.out.println("开始时间: " + System.currentTimeMillis());

        Thread.sleep(3000); // 睡眠3秒

        System.out.println("结束时间: " + System.currentTimeMillis());
    }
}
```

### sleep() 的特点

**不释放锁**：如果线程持有某个对象的锁，调用 `sleep()` 不会释放这个锁。

```java
public class SleepWithLock {
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        Thread thread1 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("线程1 获取锁，准备睡眠");
                try {
                    Thread.sleep(5000); // 持有锁5秒，期间不释放
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "Thread-1");

        Thread thread2 = new Thread(() -> {
            synchronized (lock) { // 必须等待 Thread-1 释放锁
                System.out.println("线程2 获取到锁");
            }
        }, "Thread-2");

        thread1.start();
        Thread.sleep(100); // 确保 thread1 先获取锁
        thread2.start();
    }
}
```

在这个例子中，Thread-1 持有锁后睡眠 5 秒，Thread-2 必须等待 Thread-1 睡眠结束并释放锁后才能进入同步块。这就是为什么说 `sleep()` 不释放锁。

**响应中断**：`sleep()` 可能会抛出 `InterruptedException`。当其他线程中断当前睡眠线程时，睡眠会提前结束。

```java
public class SleepInterrupt {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                System.out.println("线程开始睡眠");
                Thread.sleep(10000); // 睡眠10秒
                System.out.println("线程正常醒来");
            } catch (InterruptedException e) {
                System.out.println("线程被中断醒来");
            }
        });

        thread.start();
        Thread.sleep(2000); // 2秒后中断线程
        thread.interrupt();
    }
}
```

**线程状态变化**：NEW → RUNNABLE → TIMED_WAITING → RUNNABLE → TERMINATED

## wait()：等待与通知

`wait()` 是 `Object` 类的方法，不是 `Thread` 类的方法。这是有意为之的设计——因为 `wait()` 总是与某个对象的监视器锁配合使用。

### 基本用法

`wait()` 有三个重载版本：

```java
public class WaitDemo {
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            synchronized (lock) {
                System.out.println("线程获取锁，等待中...");
                try {
                    lock.wait(); // 释放锁并等待
                    System.out.println("线程被通知，继续执行");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        thread.start();
        Thread.sleep(1000);

        // 主线程获取锁并通知
        synchronized (lock) {
            lock.notify(); // 通知等待的线程
        }
    }
}
```

### wait() 的核心特点

**释放锁**：这是 `wait()` 与 `sleep()` 最本质的区别。调用 `wait()` 时，当前线程会释放它持有的监视器锁。这是实现线程间通信的关键机制。

```java
public class WaitReleaseLock {
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        Thread thread1 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("线程1 获取锁，准备等待");
                try {
                    lock.wait(); // 释放锁
                    System.out.println("线程1 重新获取锁，继续执行");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "Thread-1");

        Thread thread2 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("线程2 获取锁");
            }
        }, "Thread-2");

        thread1.start();
        Thread.sleep(100); // 确保 thread1 先获取锁并进入等待
        thread2.start(); // thread2 可以获取锁，因为 thread1 已释放
    }
}
```

在这个例子中，Thread-1 调用 `wait()` 后立即释放了锁，所以 Thread-2 能够获取锁继续执行。而 `sleep()` 绝对不会释放锁。

**必须在同步块中调用**：`wait()` 必须在 `synchronized` 代码块或方法中调用，否则会抛出 `IllegalMonitorStateException`。

```java
public class WaitWithoutSync {
    public static void main(String[] args) {
        Object obj = new Object();

        try {
            obj.wait(); // 抛出 IllegalMonitorStateException
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}
```

**等待队列**：`wait()` 调用后，线程会进入对象的等待队列（Wait Set）。当其他线程调用 `notify()` 或 `notifyAll()` 时，JVM 会从等待队列中选择一个或全部线程唤醒，将它们移到同步队列（Entry List）中竞争锁。

### wait() vs wait(long timeout)

```java
// 无限期等待
obj.wait();

// 限时等待，超时自动唤醒
obj.wait(3000); // 等待3秒
obj.wait(3000, 500); // 等待3秒500纳秒
```

带超时的 `wait()` 在超时后会自己醒来继续执行，而不需要其他线程通知。

## yield()：谦让 CPU

`yield()` 是一个静态方法，表示当前线程愿意让出 CPU 使用权，让出后线程可能立即被调度回来继续执行。这是给调度器的一个提示，不保证任何事。

### 基本用法

```java
public class YieldDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("线程执行: " + i);
                Thread.yield(); // 提示让出 CPU
            }
        });

        thread.start();
    }
}
```

### yield() 的特点

**不释放锁**：与 `sleep()` 类似，`yield()` 也不会释放任何锁。如果线程在同步块中调用 `yield()`，它仍然持有锁。

**不阻塞**：`yield()` 不会让线程进入等待或阻塞状态，它只是让线程从 RUNNING 状态回到 RUNNABLE 状态，重新参与 CPU 竞争。

**不确定**：`yield()` 只是一个提示，线程调度器可以完全忽略它。调用 `yield()` 后，线程可能立即又被调度回来继续执行。

### yield() 的使用场景

`yield()` 在现代 Java 应用中很少使用，因为它太不可靠。但在某些特定场景下可能有用：

```java
public class YieldUsage {
    // 忙等待的另一种实现
    private volatile boolean ready = false;

    public void producer() {
        ready = true;
    }

    public void consumer() {
        while (!ready) {
            Thread.yield(); // 让出 CPU，避免完全占满 CPU
        }
        System.out.println("消费数据");
    }
}
```

不过更好的做法是使用 `wait()` 或 `LockSupport.park()`。

## 详细对比

| 特性 | sleep() | wait() | yield() |
|------|---------|--------|---------|
| 方法归属 | Thread 类 | Object 类 | Thread 类 |
| 释放锁 | 否 | 是 | 否 |
| 使用场景 | 同步块外/内都可以 | 必须在同步块内 | 任意位置 |
| 被中断 | 抛异常 | 抛异常 | 无异常 |
| 状态变化 | RUNNING → TIMED_WAITING | RUNNING → WAITING | RUNNING → RUNNABLE |
| 唤醒方式 | 超时/中断 | notify/notifyAll/中断/超时 | 自动被调度 |

## 实际应用场景

### 场景一：等待超时模式

```java
public class WaitTimeout {
    private static final Object lock = new Object();
    private static boolean condition = false;

    public static void main(String[] args) throws InterruptedException {
        Thread waiter = new Thread(() -> {
            synchronized (lock) {
                while (!condition) {
                    try {
                        long start = System.currentTimeMillis();
                        long remaining = 5000; // 超时时间
                        lock.wait(remaining);
                        long elapsed = System.currentTimeMillis() - start;

                        if (condition) {
                            System.out.println("条件满足，继续执行");
                        } else {
                            System.out.println("等待超时，等待了 " + elapsed + "ms");
                        }
                    } catch (InterruptedException e) {
                        throw new RuntimeException(e);
                    }
                }
            }
        });

        waiter.start();
        Thread.sleep(1000);
        // 不设置 condition=true，模拟超时
    }
}
```

### 场景二：简单延时

```java
public class SimpleDelay {
    public static void main(String[] args) throws InterruptedException {
        System.out.println("开始下载...");
        Thread.sleep(3000); // 简单延时3秒
        System.out.println("下载完成");
    }
}
```

### 场景三：模拟礼让

```java
public class YieldSimulation {
    private static volatile int count = 0;

    public static void main(String[] args) {
        Thread thread1 = new Thread(() -> {
            while (count < 100) {
                int current = count;
                count = current + 1;
                System.out.println("线程1: " + count);
                Thread.yield(); // 礼让，让其他线程有机会执行
            }
        });

        Thread thread2 = new Thread(() -> {
            while (count < 100) {
                int current = count;
                count = current + 1;
                System.out.println("线程2: " + count);
                Thread.yield();
            }
        });

        thread1.start();
        thread2.start();
    }
}
```

## 面试中的常见问题

**问题 1**：sleep() 和 wait() 有什么区别？

**答案**：
1. 归属不同：`sleep()` 是 `Thread` 类的方法，`wait()` 是 `Object` 类的方法
2. 释放锁：`sleep()` 不释放锁，`wait()` 会释放锁
3. 使用位置：`sleep()` 可以在任意位置调用，`wait()` 必须在同步块/方法中调用
4. 唤醒方式：`sleep()` 通过超时或中断唤醒，`wait()` 通过 notify/notifyAll/超时/中断唤醒

**问题 2**：yield() 方法有什么用？

**答案**：`yield()` 是一种让步机制，表示当前线程愿意让出 CPU，但可能会立即被重新调度。它主要用于调试或不太常见的高性能场景。由于其不确定性和低可靠性，实际开发中使用很少。

**问题 3**：为什么 wait() 是 Object 的方法而不是 Thread 的方法？

**答案**：因为 `wait()` 必须与对象的监视器锁配合使用，调用 `wait()` 的线程必须持有该对象的锁。设计者认为锁属于对象而不是线程，所以 `wait()` 应该是对象的方法。这也迫使开发者必须在同步块中使用 `wait()`，避免误用。
