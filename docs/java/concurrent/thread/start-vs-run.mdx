# start() vs run()：线程启动的秘密

很多初学者会困惑：`start()` 和 `run()` 都能执行线程中的代码，那它们有什么区别？为什么必须调用 `start()` 而不是 `run()`？这个看似简单的问题背后，隐藏着线程创建的核心原理。

## 一图胜千言

```java
public class StartVsRun {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("线程执行中: " + Thread.currentThread().getName());
        });

        // 这两种方式有什么区别？
        thread.run();   // 方式一
        thread.start(); // 方式二
    }
}
```

表面上看，两种方式都会执行 Lambda 表达式中的代码。但实际上，它们有着本质的区别。

## run()：普通方法调用

`run()` 就是一个普通的方法调用。当你在主线程中调用 `thread.run()` 时，它会在主线程中顺序执行 `run()` 方法的内容。**不会有新的线程被创建。**

```java
public class RunDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            for (int i = 0; i < 3; i++) {
                System.out.println("run() 执行: " + i + ", 线程: " 
                    + Thread.currentThread().getName());
            }
        });

        thread.run(); // 在主线程中执行

        System.out.println("main 方法结束");
    }
}
```

输出可能是：

```
run() 执行: 0, 线程: main
run() 执行: 1, 线程: main
run() 执行: 2, 线程: main
main 方法结束
```

所有代码都在 main 线程中执行，没有任何并发效果。

## start()：真正创建新线程

`start()` 方法才会真正创建一个新的操作系统线程，然后在这个新线程中执行 `run()` 方法的内容。这是一个重量级操作，涉及系统调用和资源分配。

```java
public class StartDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            for (int i = 0; i < 3; i++) {
                System.out.println("start() 执行: " + i + ", 线程: " 
                    + Thread.currentThread().getName());
            }
        });

        thread.start(); // 创建新线程并执行

        System.out.println("main 方法结束");
    }
}
```

输出可能是（顺序不确定）：

```
main 方法结束
start() 执行: 0, 线程: Thread-0
start() 执行: 1, 线程: Thread-0
start() 执行: 2, 线程: Thread-0
```

或者：

```
start() 执行: 0, 线程: Thread-0
main 方法结束
start() 执行: 1, 线程: Thread-0
start() 执行: 2, 线程: Thread-0
```

`main` 方法和 `Thread-0` 是并行执行的，谁先谁后完全取决于 CPU 调度。

## start() 的底层原理

让我们深入 `Thread` 类的源码，看看 `start()` 到底做了什么：

```java
public class Thread implements Runnable {
    private Runnable target; // 要执行的任务

    @Override
    public void run() {
        if (target != null) {
            target.run();
        }
    }

    public synchronized void start() {
        // A native method that will eventually call run()
        start0();
    }

    private native void start0();
}
```

`start()` 是一个 `synchronized` 方法，里面调用了本地方法 `start0()`。这个本地方法会：

1. 向操作系统请求创建一个新的线程
2. 为新线程分配栈内存
3. 将新线程添加到线程组
4. 准备好线程的执行上下文
5. 调用操作系统的调度器，将新线程标记为可运行状态

关键是：**新线程一旦获得 CPU 时间片，就会立即调用 `run()` 方法**。但这个 `run()` 调用是在新线程中执行的，不是在调用 `start()` 的线程中。

## 为什么不能多次调用 start()？

这是一个常见的问题。如果多次调用 `start()`，会发生什么？

```java
public class DoubleStart {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("线程执行中");
        });

        thread.start();
        thread.start(); // 第二次调用会怎样？
    }
}
```

运行后会抛出 `IllegalThreadStateException`：

```
Exception in thread "main" java.lang.IllegalThreadStateException
    at java.lang.Thread.start0(Native Method)
    at java.lang.Thread.start(Thread.java:1482)
    at com.example.DoubleStart.main(DoubleStart.java:12)
```

原因很简单：线程从 NEW 状态转换到 RUNNABLE 状态是单向的，不可逆。一旦线程被启动（调用过 `start()`），它的状态就不再是 NEW 了，再次调用 `start()` 就是非法的。

这就像你已经点了一把火，不能再点第二次。每次火焰都是新的火苗，而不是原来的火焰重新燃烧。

## start() 和 run() 的执行时序

为了更清楚地展示两者的区别，我们用时序图来分析：

**调用 run() 的时序：**

```
┌─────────────────────────────────────────┐
│              main 线程                   │
├─────────────────────────────────────────┤
│                                         │
│  thread.run() ──────► 执行完成           │
│       │                                  │
│       │  (在 main 线程中执行)            │
│       │                                  │
│  继续执行主线程后续代码                    │
│                                         │
└─────────────────────────────────────────┘
```

**调用 start() 的时序：**

```
┌─────────────────────────────────────────┐
│              main 线程                   │
├─────────────────────────────────────────┤
│                                         │
│  thread.start() ──► 返回（不等待）       │
│       │                                 │
│       │  (立即返回，不阻塞)              │
│       │                                 │
│  继续执行主线程后续代码                    │
│       │                                 │
│       │         ┌──────────────────────┤
│       │         │    Thread-0 线程      │
│       │         ├──────────────────────┤
│       │         │                      │
│       │         │  执行 thread.run()   │
│       │         │                      │
│       │         └──────────────────────┘
│       │
└───────┘
```

## 实际开发中的坑

### 坑一：误用 run() 导致串行执行

```java
public class Pitfall1 {
    public static void main(String[] args) {
        List<Thread> threads = new ArrayList<>();

        for (int i = 0; i < 10; i++) {
            final int taskId = i;
            threads.add(new Thread(() -> {
                // 模拟耗时任务
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("任务 " + taskId + " 完成");
            }));
        }

        long start = System.currentTimeMillis();

        // 错误写法：调用 run() 而不是 start()
        for (Thread thread : threads) {
            thread.run(); // 串行执行，总耗时约 10 秒
        }

        System.out.println("总耗时: " + (System.currentTimeMillis() - start));
    }
}
```

如果误用了 `run()`，这 10 个任务会串行执行，总耗时约 10 秒。但正确使用 `start()` 后，由于是并行执行，总耗时约 1 秒。

### 坑二：FutureTask 配合 Thread 的误解

```java
public class Pitfall2 {
    public static void main(String[] args) throws Exception {
        Callable<Integer> callable = () -> {
            Thread.sleep(2000);
            return 42;
        };

        FutureTask<Integer> future = new FutureTask<>(callable);
        Thread thread = new Thread(future);

        thread.start(); // 启动线程
        thread.start(); // 再次启动！抛出 IllegalThreadStateException
    }
}
```

### 坑三：start() 非阻塞导致的竞态条件

```java
public class Pitfall3 {
    private static boolean done = false;

    public static void main(String[] args) {
        Thread worker = new Thread(() -> {
            // 模拟初始化
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            done = true;
        });

        worker.start();

        // 错误假设 worker 已经完成
        if (done) { // 此时 done 可能是 false
            System.out.println("完成");
        }
    }
}
```

`start()` 立即返回，不等待线程执行完成。如果需要等待，必须使用 `join()` 方法。

## 正确的使用方式

```java
public class CorrectUsage {
    public static void main(String[] args) throws InterruptedException {
        // 1. 创建线程
        Thread thread = new Thread(() -> {
            System.out.println("新线程开始执行");
        }, "MyThread");

        // 2. 启动线程（不要直接调用 run()）
        thread.start();

        // 3. 如果需要等待线程执行完成
        thread.join();

        // 4. 检查线程是否还在运行
        if (!thread.isAlive()) {
            System.out.println("线程已执行完毕");
        }
    }
}
```

## 面试中的考察点

面试官经常通过这个问题考察候选人对线程创建过程的理解：

**问题**：调用 `start()` 方法会执行 `run()` 吗？为什么不直接调用 `run()`？

**参考答案**：
1. `start()` 会执行 `run()`，但不是立即执行，而是在新创建的线程中执行
2. `run()` 是普通方法调用，在调用它的线程中顺序执行
3. 调用 `start()` 会真正创建操作系统线程，涉及系统资源分配；调用 `run()` 只是一个简单的方法调用
4. `start()` 只能调用一次；`run()` 可以调用多次
5. 多次调用 `start()` 会抛出 `IllegalThreadStateException`

理解 `start()` 和 `run()` 的区别，是理解 Java 线程模型的关键一步。下一篇文章我们将探讨 `sleep()`、`wait()` 和 `yield()` 这三个控制线程行为的重要方法。
