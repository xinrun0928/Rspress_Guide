# start() vs run()：为什么必须调用 start()

面试官问了一个看似简单的问题：「启动线程是调用 `start()` 还是 `run()`？」

你回答：「当然是 `start()`。」

面试官追问：「为什么不能直接调用 `run()`？」

你：「因为...」

这个问题看似基础，但背后的原理却涉及 JVM 的线程创建机制。

## 一行代码的陷阱

```java
Thread thread = new Thread(() -> {
    System.out.println("Hello from thread");
});

// 错误方式：直接调用 run()
thread.run();  // 这是在主线程中执行，不是新线程！

// 正确方式：调用 start()
thread.start();  // 真正创建新线程
```

表面上看，`run()` 和 `start()` 都能执行代码。但实际上，它们有本质区别。

## 执行结果对比

```java
public class StartVsRunDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("Thread: " + Thread.currentThread().getName());
        });

        System.out.println("Main: " + Thread.currentThread().getName());

        thread.run();  // 方式一
        thread.start(); // 方式二
    }
}
```

输出可能是：

```text
Main: main
Thread: main          // run() 在主线程执行
Thread: Thread-0      // start() 真正创建了新线程
```

**关键区别**：`run()` 只是在当前线程中执行任务，而 `start()` 才是真正创建新线程。

## start() 的底层原理

`start()` 方法会触发 JVM 创建新线程，整个过程如下：

```java
public class Thread {
    private Runnable target;

    public synchronized void start() {
        // 一个线程只能启动一次
        if (threadStatus != 0)
            throw new IllegalThreadStateException();

        // 将线程添加到线程组
        group.add(this);

        boolean started = false;
        try {
            // 调用 native 方法创建线程
            start0();
            started = true;
        } finally {
            if (!started) {
                try {
                    group.threadStartFailed(this);
                } catch (Throwable ignore) {
                }
            }
        }
    }

    // native 方法，真正创建线程
    private native void start0();
}
```

### start() 的执行流程

```
┌─────────────────────────────────────────────────────────┐
│                    start() 执行流程                       │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. 状态检查：threadStatus == 0                          │
│            ↓                                             │
│  2. 添加到线程组：group.add(this)                        │
│            ↓                                             │
│  3. 调用 native start0()                                │
│     ┌──────────────────────────────────────┐            │
│     │    OS 层面创建新线程                   │            │
│     │    - 分配线程栈                        │            │
│     │    - 注册到 OS 线程调度器              │            │
│     │    - 分配 threadId                      │            │
│     └──────────────────────────────────────┘            │
│            ↓                                             │
│  4. 新线程执行 run() 方法                               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### run() 只是普通方法

```java
public void run() {
    if (target != null) {
        target.run();
    }
}
```

`run()` 只是一个普通的实例方法，没有启动新线程的能力。

## 为什么不能多次调用 start()？

```java
Thread thread = new Thread(() -> {});
thread.start();  // 第一次：创建新线程
thread.start();  // 第二次：抛出 IllegalThreadStateException
```

因为 `start()` 会修改线程的 `threadStatus` 状态：

```java
if (threadStatus != 0)
    throw new IllegalThreadStateException();
```

线程一旦启动（`threadStatus` 变为非 0），就不能再次启动。

## 正确的使用方式

### 方式一：继承 Thread

```java
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Running in: " + getName());
    }
}

MyThread thread = new MyThread();
thread.start();
```

### 方式二：实现 Runnable

```java
Runnable task = () -> {
    System.out.println("Running in: " + Thread.currentThread().getName());
};

Thread thread = new Thread(task);
thread.start();
```

### 方式三：实现 Callable + FutureTask

```java
Callable<Integer> callable = () -> {
    System.out.println("Computing...");
    return 42;
};

FutureTask<Integer> future = new FutureTask<>(callable);
Thread thread = new Thread(future);
thread.start();

Integer result = future.get();  // 获取返回值
```

## 常见误区

### 误区一：在循环中反复调用 run()

```java
// 错误代码
while (true) {
    Thread thread = new Thread(task);
    thread.run();  // 错误：在主线程执行，没有创建新线程
}
```

### 误区二：任务完成后重新 start()

```java
// 错误代码
Thread thread = new Thread(task);
thread.start();
thread.join();  // 等待完成

thread.start();  // 再次 start：抛出 IllegalThreadStateException
```

正确做法是创建新线程：

```java
Thread thread1 = new Thread(task);
Thread thread2 = new Thread(task);
thread1.start();
thread2.start();
```

### 误区三：start() 会立即执行

```java
Thread thread = new Thread(() -> {
    System.out.println("Task started");
});

thread.start();
// 主线程和子线程是并发的
// 这行代码可能在子线程执行前、后、或执行中输出
System.out.println("Main continued");
```

`start()` 只是告诉 JVM「创建一个新线程」，不保证新线程立即执行。

## 面试追问

### Q1：start() 调用 run() 吗？

技术上，`start()` 不直接调用 `run()`。`start()` 通过 native 方法 `start0()` 创建 OS 层面的线程，新线程创建后会执行 `run()` 方法。

```java
// JVM 层面的伪代码
private void start0() {
    // 创建 OS 线程
    // 新线程执行：
    //   1. 调用 init() 初始化
    //   2. 调用 start0() 再次确认
    //   3. 执行 run() 方法
}
```

### Q2：start() 和 run() 的性能差异？

`start()` 需要：
- 分配 OS 线程资源
- 创建线程上下文
- 注册到线程调度器

`run()` 只是普通方法调用。

在高并发场景下，频繁创建线程有性能问题，应该使用线程池。

### Q3：为什么 Java 要设计成 start() + run() 分离？

这是设计上的权衡：

1. **灵活性**：同一份任务代码可以：
   - 用 `start()` 创建新线程执行
   - 直接用 `run()` 在当前线程执行
2. **接口分离**：`Runnable` 接口只定义 `run()`，不关心线程管理
3. **延迟执行**：`Thread` 对象创建后，可以延迟调用 `start()`

## 总结

`start()` 和 `run()` 的区别：

| 特性 | `start()` | `run()` |
|---|---|---|
| 线程创建 | 真正创建新线程 | 不创建线程 |
| 执行线程 | 新线程执行 | 当前线程执行 |
| 调用次数 | 仅限一次 | 可多次调用 |
| 性能开销 | 有（创建线程） | 无（普通方法调用） |
| 并发 | 支持 | 不支持 |

记住一个原则：**启动线程用 `start()`，执行任务用 `run()`**。
