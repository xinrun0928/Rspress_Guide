# 创建线程的 N 种方式

想象一下，你要在一场足球比赛中安排球员上场。在 Java 中，线程就像是球员，而创建线程就是选择让球员上场的方式。你可以用直接上场（继承 Thread）、让教练指挥（实现 Runnable）、还是请替补球员（实现 Callable）？每种方式都有自己的特点和适用场景。

## 方式一：继承 Thread 类

这是最直接的方式，就像让一个人直接上场踢球。你只需要继承 `java.lang.Thread` 类，重写 `run()` 方法，然后调用 `start()` 启动线程。

```java
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("线程执行中: " + Thread.currentThread().getName());
    }

    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.setName("我的线程");
        thread.start(); // 不要调用 run()，要调用 start()
    }
}
```

这里有一个重要的坑：**必须调用 `start()` 而不是 `run()`**。`start()` 方法会创建一个新的线程，然后在新线程中执行 `run()` 方法。如果直接调用 `run()`，那它就是一个普通的方法调用，只会在主线程中顺序执行，不会有新线程被创建。

继承 Thread 类的优点是简单直观，缺点是 Java 单继承的限制——如果已经继承了其他类，就无法再用这种方式了。

## 方式二：实现 Runnable 接口

这是更灵活的方式。`Runnable` 接口只有一个 `run()` 方法，实现它不会占用你的继承名额。

```java
public class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("通过 Runnable 创建线程: " + Thread.currentThread().getName());
    }

    public static void main(String[] args) {
        Thread thread = new Thread(new MyRunnable(), "Runnable线程");
        thread.start();
    }
}
```

这种方式实现了任务与线程的分离。`Runnable` 代表要执行的任务，`Thread` 代表执行任务的线程。你可以复用同一个 Runnable 实例创建多个线程，它们会执行相同的任务代码。

## 方式三：使用 Lambda 表达式

在 Java 8 以后，由于 `Runnable` 接口只有一个抽象方法（函数式接口），我们可以使用 Lambda 表达式来简化代码：

```java
public class LambdaThread {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("Lambda 线程执行中: " + Thread.currentThread().getName());
        }, "LambdaThread");
        thread.start();
    }
}
```

这种方式代码简洁，是现代 Java 编程的主流写法。

## 方式四：实现 Callable 接口 + FutureTask

`Runnable` 的 `run()` 方法没有返回值，也无法抛出异常。但在实际开发中，我们经常需要获取线程的执行结果，这时就需要 `Callable`。

```java
import java.util.concurrent.Callable;
import java.util.concurrent.FutureTask;

public class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        // 模拟计算任务
        Thread.sleep(1000);
        return 42;
    }

    public static void main(String[] args) throws Exception {
        MyCallable callable = new MyCallable();
        FutureTask<Integer> futureTask = new FutureTask<>(callable);
        Thread thread = new Thread(futureTask, "Callable线程");
        thread.start();

        // 获取线程执行结果，会阻塞直到结果返回
        Integer result = futureTask.get();
        System.out.println("线程返回结果: " + result);
    }
}
```

`FutureTask` 实现了 `RunnableFuture` 接口，而 `RunnableFuture` 又同时继承了 `Runnable` 和 `Future` 接口。它既可以作为 `Runnable` 被线程执行，又可以像 `Future` 一样获取执行结果。

`Future` 接口提供了几个重要方法：

- `get()`：获取执行结果，会阻塞等待
- `get(timeout, unit)`：带超时获取结果
- `isDone()`：判断任务是否完成
- `cancel()`：取消任务执行

## 方式五：使用线程池

在实际开发中，我们几乎不会手动创建线程，而是使用线程池。线程池就像是一个球员储备库，你可以从中获取线程执行任务，用完后归还而不是销毁。

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadPoolDemo {
    public static void main(String[] args) {
        // 创建固定大小的线程池
        ExecutorService executor = Executors.newFixedThreadPool(3);

        for (int i = 0; i < 5; i++) {
            final int taskId = i;
            executor.execute(() -> {
                System.out.println("任务 " + taskId + " 由线程 "
                    + Thread.currentThread().getName() + " 执行");
            });
        }

        executor.shutdown();
    }
}
```

线程池的优势在于：复用线程减少创建开销、控制并发数量、易于管理。关于线程池的详细内容，我们会在后面的章节深入探讨。

## 方式六：使用 CompletableFuture

Java 8 引入的 `CompletableFuture` 是更现代的异步编程方式，提供了强大的链式调用、组合操作和异常处理能力：

```java
import java.util.concurrent.CompletableFuture;

public class CompletableFutureDemo {
    public static void main(String[] args) throws Exception {
        CompletableFuture<String> future = CompletableFuture
            .supplyAsync(() -> {
                // 模拟耗时操作
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                return "Hello";
            })
            .thenApply(result -> result + " World")
            .thenApply(String::toUpperCase);

        System.out.println(future.get()); // 输出: HELLO WORLD
    }
}
```

## 方式七：使用 Timer 和 TimerTask

这是较老的方式，适用于定时任务场景：

```java
import java.util.Timer;
import java.util.TimerTask;

public class TimerDemo {
    public static void main(String[] args) {
        Timer timer = new Timer();
        timer.schedule(new TimerTask() {
            @Override
            public void run() {
                System.out.println("定时任务执行");
            }
        }, 0, 1000); // 延迟0ms后开始，每秒执行一次
    }
}
```

不过 `Timer` 的缺点是只创建一个线程，如果一个任务执行时间过长，会影响其他任务的准时执行。推荐使用 `ScheduledThreadPoolExecutor`。

## 方式八：使用 Fork/Join 框架

Java 7 引入了 Fork/Join 框架，适用于分治算法的并行计算：

```java
import java.util.concurrent.ForkJoinPool;
import java.util.concurrent.RecursiveTask;

public class SumTask extends RecursiveTask<Long> {
    private static final int THRESHOLD = 1000;
    private long[] array;
    private int start;
    private int end;

    public SumTask(long[] array, int start, int end) {
        this.array = array;
        this.start = start;
        this.end = end;
    }

    @Override
    protected Long compute() {
        if (end - start <= THRESHOLD) {
            long sum = 0;
            for (int i = start; i < end; i++) {
                sum += array[i];
            }
            return sum;
        }

        int mid = (start + end) / 2;
        SumTask leftTask = new SumTask(array, start, mid);
        SumTask rightTask = new SumTask(array, mid, end);
        leftTask.fork();
        rightTask.fork();
        return leftTask.join() + rightTask.join();
    }

    public static void main(String[] args) throws Exception {
        ForkJoinPool pool = new ForkJoinPool();
        long[] array = new long[10000];
        // 初始化数组...
        SumTask task = new SumTask(array, 0, array.length);
        Long result = pool.invoke(task);
        System.out.println("Sum = " + result);
    }
}
```

## 总结与对比

| 方式 | 返回值 | 多继承 | 适用场景 |
|------|--------|--------|----------|
| 继承 Thread | 无 | 否 | 简单场景 |
| 实现 Runnable | 无 | 是 | 通用场景 |
| Lambda | 无 | 是 | Java 8+ 简洁写法 |
| Callable + FutureTask | 有 | 是 | 需要获取结果 |
| 线程池 | 无 | 是 | 生产环境 |
| CompletableFuture | 有 | 是 | 复杂异步流程 |
| Fork/Join | 有 | 是 | 分治并行计算 |

在实际开发中，**推荐使用线程池或 CompletableFuture**，因为它们能更好地管理线程资源，避免创建过多线程导致的系统资源耗尽。
