# 守护线程：幕后的默默守护者

在一个餐厅的运营中，除了在前台服务的服务员（用户线程），还有后台默默工作的厨师、清洁工等（守护线程）。他们负责洗碗、准备食材、打扫卫生，当所有服务员下班后，厨师和清洁工也没必要继续工作了——即使碗还没洗完，餐厅也可以打烊。

Java 中的守护线程（Daemon Thread）就是这样一种特殊线程。当 JVM 中只剩下守护线程时，JVM 会直接退出，不管守护线程是否还有任务没执行完。

## 守护线程的概念

守护线程是一种为其他线程提供服务的线程。当所有用户线程（non-daemon threads）都结束时，JVM 会自动终止所有守护线程。最典型的守护线程就是垃圾回收器（GC）线程。

```java
public class DaemonDemo {
    public static void main(String[] args) {
        Thread daemonThread = new Thread(() -> {
            while (true) {
                System.out.println("守护线程运行中...");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "Daemon-Thread");

        daemonThread.setDaemon(true); // 设置为守护线程
        daemonThread.start();

        System.out.println("主线程执行完毕，即将退出");
    }
}
```

运行这个程序，你会发现 JVM 几乎立即退出，而不会等待守护线程执行完毕。如果把 `setDaemon(true)` 去掉，程序会一直运行。

## 设置守护线程

使用 `setDaemon(true)` 将线程设置为守护线程。**必须在启动之前设置**，否则会抛出 `IllegalThreadStateException`。

```java
public class SetDaemon {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("线程执行中");
        });

        thread.setDaemon(true); // 启动前设置
        thread.start();

        // thread.setDaemon(true); // 如果在这里设置，会抛出 IllegalThreadStateException
    }
}
```

## 守护线程的特点

### 特点一：JVM 退出会立即终止

```java
public class DaemonTermination {
    public static void main(String[] args) throws InterruptedException {
        Thread daemon = new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                System.out.println("守护线程打印: " + i);
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            System.out.println("守护线程执行完毕");
        });

        daemon.setDaemon(true);
        daemon.start();

        Thread.sleep(2000); // 主线程只睡2秒
        System.out.println("主线程结束");
    }
}
```

输出可能是：

```
守护线程打印: 0
守护线程打印: 1
守护线程打印: 2
守护线程打印: 3
主线程结束
```

守护线程可能只打印到 3 或 4，而不是完整的 0-9。当主线程结束后，JVM 会立即终止守护线程。

### 特点二：不finally块保护

守护线程被终止时，finally 块中的代码**不保证执行**：

```java
public class DaemonFinally {
    public static void main(String[] args) throws InterruptedException {
        Thread daemon = new Thread(() -> {
            try {
                System.out.println("守护线程开始");
                Thread.sleep(1000);
                System.out.println("守护线程 sleep 结束"); // 不保证打印
            } finally {
                // finally 块不保证执行！
                System.out.println("守护线程 finally"); // 不保证打印
            }
        });

        daemon.setDaemon(true);
        daemon.start();

        Thread.sleep(200); // 主线程睡200毫秒后结束
        System.out.println("主线程结束");
    }
}
```

这是一个重要的陷阱！如果你的守护线程中有资源清理逻辑（如关闭文件、释放锁），不能依赖 finally 块来保证执行。

### 特点三：自动继承

如果一个线程是守护线程，那么它创建的子线程默认也是守护线程：

```java
public class DaemonInherit {
    public static void main(String[] args) throws InterruptedException {
        Thread parent = new Thread(() -> {
            // 创建子线程
            Thread child = new Thread(() -> {
                System.out.println("子线程是守护线程: " + Thread.currentThread().isDaemon());
            });
            child.start();

            System.out.println("父线程是守护线程: " + Thread.currentThread().isDaemon());
        });

        parent.setDaemon(true);
        parent.start();

        Thread.sleep(500);
    }
}
```

输出显示父线程和子线程都是守护线程。

## 守护线程的应用场景

### 场景一：后台任务调度

```java
import java.util.concurrent.Executors;
import java.util.concurrent.ScheduledExecutorService;
import java.util.concurrent.TimeUnit;

public class ScheduledDaemon {
    public static void main(String[] args) {
        ScheduledExecutorService scheduler = 
            Executors.newScheduledThreadPool(1, r -> {
                Thread t = new Thread(r);
                t.setDaemon(true); // 设置为守护线程
                return t;
            });

        // 每5秒执行一次清理任务
        scheduler.scheduleAtFixedRate(() -> {
            System.out.println("执行定时清理...");
            cleanup();
        }, 0, 5, TimeUnit.SECONDS);

        // 主线程可以结束，JVM 会保持守护线程运行
        System.out.println("主线程退出");
    }

    private static void cleanup() {
        // 模拟清理工作
    }
}
```

### 场景二：监控服务

```java
public class MonitoringDaemon {
    public static void main(String[] args) throws InterruptedException {
        Thread monitor = new Thread(() -> {
            while (true) {
                try {
                    checkSystemHealth();
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    break;
                }
            }
        }, "Monitor");

        monitor.setDaemon(true);
        monitor.start();

        // 模拟主业务
        Thread.sleep(10000);
        System.out.println("主业务完成，程序退出");
    }

    private static void checkSystemHealth() {
        Runtime runtime = Runtime.getRuntime();
        long memory = runtime.maxMemory() / 1024 / 1024;
        System.out.println("系统内存上限: " + memory + "MB");
    }
}
```

### 场景三：热更新检查

```java
public class HotDeployChecker {
    public static void main(String[] args) {
        Thread checker = new Thread(() -> {
            while (true) {
                try {
                    checkForUpdates();
                    Thread.sleep(60000); // 每分钟检查一次
                } catch (Exception e) {
                    // 记录错误但不中断
                }
            }
        }, "HotDeploy-Checker");

        checker.setDaemon(true);
        checker.start();

        System.out.println("应用启动完成，等待热更新...");
    }

    private static void checkForUpdates() {
        // 检查是否有新的版本
    }
}
```

## 守护线程 vs 用户线程

| 特性 | 守护线程 | 用户线程 |
|------|----------|----------|
| 用途 | 为其他线程提供服务 | 执行实际业务逻辑 |
| JVM 退出 | 随着 JVM 退出 | 必须主动结束或等待结束 |
| finally 执行 | 不保证 | 保证执行 |
| 创建子线程 | 默认是守护线程 | 默认是用户线程 |
| 示例 | GC 线程、 JIT 编译线程 | main 线程、业务处理线程 |

## 常见误区

### 误区一：数据库连接池线程一定是守护线程

实际上，数据库连接池（如 HikariCP）的线程通常是**用户线程**，因为它们负责重要的资源管理，JVM 退出时需要优雅关闭。如果设置不当，可能导致连接泄漏。

### 误区二：守护线程的异常会导致 JVM 退出

守护线程和用户线程抛出未捕获异常的默认行为是一样的——都会打印异常堆栈。但守护线程的异常不会阻止 JVM 退出，而用户线程的异常可能影响主线程的执行。

```java
public class DaemonException {
    public static void main(String[] args) {
        Thread daemon = new Thread(() -> {
            throw new RuntimeException("守护线程异常");
        });
        daemon.setDaemon(true);
        daemon.start();

        Thread.sleep(100);
        System.out.println("主线程继续执行");
    }
}
```

输出会显示异常信息，但 JVM 不会崩溃，主线程继续执行。

### 误区三：所有后台服务都应该用守护线程

不是的！如果后台服务有重要的清理或收尾工作（如写入持久化状态、发送关闭信号），应该使用用户线程，并正确处理优雅关闭。

```java
public class NonDaemonService {
    private volatile boolean running = true;

    public void start() {
        Thread worker = new Thread(() -> {
            while (running) {
                process();
            }
            // 重要：这里是用户线程，finally 保证执行
            try {
                cleanup();
            } catch (Exception e) {
                log(e);
            }
        });

        worker.setDaemon(false); // 明确设置用户线程
        worker.start();
    }

    public void stop() {
        running = false;
    }

    private void process() {}
    private void cleanup() {}
}
```

## 线程池与守护线程

使用 `Executors` 创建的线程池，默认是用户线程。如果需要创建守护线程的线程池，需要自定义线程工厂：

```java
public class DaemonThreadPool {
    public static void main(String[] args) {
        // 创建守护线程的线程池
        ExecutorService executor = Executors.newFixedThreadPool(2, r -> {
            Thread t = new Thread(r);
            t.setDaemon(true);
            return t;
        });

        executor.submit(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("守护线程任务: " + i);
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        executor.shutdown();
        // 立即返回，因为线程池中的线程是守护线程
    }
}
```

## 面试问题

**问题 1**：守护线程和用户线程有什么区别？

**答案**：当 JVM 中没有用户线程而只有守护线程时，JVM 会直接退出，不等待守护线程执行完毕。守护线程中的 `finally` 块也不保证执行。典型的守护线程有 GC 线程和 JIT 编译线程。

**问题 2**：为什么守护线程的 finally 块不保证执行？

**答案**：因为 JVM 退出时是直接终止线程，不会给线程执行清理代码的机会。这是 JVM 的设计决策，目的是确保 JVM 能够快速退出。如果需要保证清理代码执行，应该使用用户线程并正确处理优雅关闭。

**问题 3**：如何创建守护线程池？

**答案**：可以通过自定义 `ThreadFactory` 来创建守护线程的线程池：

```java
ThreadFactory daemonFactory = r -> {
    Thread t = new Thread(r);
    t.setDaemon(true);
    return t;
};
ExecutorService executor = Executors.newFixedThreadPool(2, daemonFactory);
```

**问题 4**：守护线程在生产环境中应该注意什么？

**答案**：主要注意以下几点：

1. 不要在守护线程中执行不能中断的业务逻辑
2. 不要依赖 `finally` 块进行资源清理
3. 需要优雅关闭的服务应该使用用户线程
4. 如果守护线程崩溃，不应该影响整个应用
