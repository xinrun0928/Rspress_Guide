# 进程与线程

## 进程：程序的一次执行

进程（Process）是计算机中的程序关于某数据集合上的一次运行活动，是系统进行资源分配和调度的基本单位。你可以理解进程就是一个正在运行的程序，它拥有独立的内存空间、文件句柄、处理器资源等。

想象一下，当你打开一个浏览器查看网页时，操作系统会为这个浏览器程序创建一个进程。这个进程拥有自己的内存地址空间，存放着浏览器的代码、数据和运行时信息。即使你同时打开两个浏览器窗口（实际上是两个标签页进程），它们之间也是相互独立的，一个崩溃不会影响另一个。

在 Java 中，运行一个 `main` 方法就会启动一个 JVM 进程。JVM 是用 C++ 编写的，它本身就是一个进程，Java 代码运行在这个进程之上。每个 Java 应用通常对应一个 JVM 进程。

## 线程：进程内的执行单元

线程（Thread）是进程中的一个执行流，是 CPU 调度的最小单位。一个进程可以包含多个线程，这些线程共享进程的资源，但拥有独立的执行栈和程序计数器。

继续用餐厅的比喻：如果进程是整个餐厅，那么线程就是餐厅里的各个员工。厨师、服务员、收银员共享餐厅的所有资源——厨房设备、食材库存、桌椅板凳。但每个员工都有自己的记事本和工作状态——线程的栈内存和程序计数器。

**为什么需要线程？** 因为线程比进程轻量得多。创建线程比创建进程快，线程之间的切换开销也远小于进程。而且，线程之间可以通过共享的内存空间直接通信，不需要像进程间通信那样复杂的机制。

## 进程与线程的区别

**资源拥有者不同**。进程是资源分配的基本单位，拥有独立的地址空间；线程是调度的基本单位，共享进程的资源。

**独立性不同**。进程之间相互独立，一个进程崩溃不会影响其他进程；而线程共享地址空间，一个线程崩溃可能导致整个进程崩溃。这就像餐厅里如果一个员工出了问题，可能影响整个餐厅的运转；但如果两家餐厅中的一家出了问题，另一家完全不受影响。

**通信方式不同**。进程间通信需要借助管道、消息队列、共享内存、套接字等机制；而线程间可以直接通过共享的内存进行通信（当然需要同步机制保证线程安全）。

**开销不同**。创建和销毁进程需要分配和回收独立的地址空间，开销较大；线程创建和销毁只需分配/回收栈寄存器和栈内存，开销小得多。线程上下文切换也比进程快得多。

## Java 中的线程

Java 天生支持多线程。从你写下第一个 `main` 方法的那一刻起，JVM 就已经创建了多个线程。看看这个简单的程序：

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
```

你以为这只是单线程运行？错了！JVM 启动时会创建这些线程：

- **主线程**：执行 `main` 方法的线程
- **GC 线程**：垃圾回收的后台线程
- **编译器线程**：JIT 编译相关
- **信号分发线程**：处理操作系统信号
- **其他 JVM 内部线程**

所以即使是最简单的 Java 程序，也是多线程运行的。这就是为什么 Java 能高效地同时处理多个任务——无论是用户交互还是后台处理，都能各司其职、互不干扰。

## 线程的内存模型

每个线程都有自己的程序计数器（Program Counter, PC）和虚拟机栈（VM Stack）。程序计数器记录当前线程执行的字节码行号；虚拟机栈存储局部变量、方法参数、方法调用栈帧等信息。

而堆（Heap）和方法区（Method Area）则是所有线程共享的。堆中存放对象实例，方法区存放类信息、常量、静态变量等。这就是为什么多个线程访问同一个对象时需要同步——因为它们共享堆内存。

```java
public class MemoryModel {
    // 静态变量 - 线程共享，存放在方法区
    private static int sharedValue = 0;

    // 实例变量 - 线程共享，存放在堆中
    private int instanceValue;

    public void method() {
        // 局部变量 - 线程私有，存放在栈中
        int localValue = 0;
        // ...
    }
}
```

理解进程与线程的区别，是理解 Java 并发编程的第一步。进程提供资源隔离，线程提供执行并行。两者相辅相成，构成了现代操作系统和 Java 程序的基石。
