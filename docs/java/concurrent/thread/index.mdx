# 线程基础：从进程到线程的进化

当你启动一个 Java 程序时，操作系统会创建一个进程（Process），每个进程都有自己独立的内存空间和系统资源。而在进程内部，可以创建多个线程（Thread），它们共享进程的内存，但各自拥有独立的执行栈和程序计数器。这就是 Java 并发编程的基础——理解线程，就是理解 Java 并发世界的第一步。

## 进程与线程：执行单位的演进

### 进程：独立的内存空间

进程是操作系统分配资源的基本单位。每个进程都有自己独立的地址空间、文件描述符、内存区域。进程之间的通信需要通过操作系统提供的 IPC（进程间通信）机制，如管道、消息队列、共享内存、套接字等。

进程的主要特点是隔离性：一个进程的崩溃不会影响其他进程。这种隔离带来稳定性，但也有代价——进程之间的切换需要保存和恢复完整的上下文，开销较大。

### 线程：轻量级的执行单位

线程是进程内的执行单位，共享进程的堆内存，但各自拥有独立的栈空间、寄存器和程序计数器。线程的创建、切换、销毁开销远小于进程，因为不需要分配独立的内存空间。

```
进程 A
├── 线程 1 (栈 A1 | PC A1 | 寄存器 A1)
├── 线程 2 (栈 A2 | PC A2 | 寄存器 A2)
└── 共享资源 (堆 A | 文件描述符 A | ...)
    |
    +--------------------------------------------+
    |                                            |
    v                                            v
进程 B                                        进程 C
└── ...                                       └── ...
```

Java 虚拟机本身就是一个进程。在 JVM 内部，主线程负责执行 `main()` 方法，同时可以创建其他线程执行并发任务。

## Java 线程的创建：三种方式

### 方式一：继承 Thread 类

最直接的方式是继承 `Thread` 类并重写 `run()` 方法：

```java
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("线程执行中: " + Thread.currentThread().getName());
    }
}

// 启动线程
MyThread thread = new MyThread();
thread.start();  // 注意是 start()，不是 run()
```

注意区分 `start()` 和 `run()`：`start()` 会创建新线程并执行 `run()` 方法；如果直接调用 `run()`，则是在当前线程中同步执行，不会创建新线程。

### 方式二：实现 Runnable 接口

由于 Java 不支持多重继承，更好的方式是实现 `Runnable` 接口：

```java
public class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("线程执行中: " + Thread.currentThread().getName());
    }
}

// 启动线程
Thread thread = new Thread(new MyRunnable());
thread.start();

// 也可以使用 Lambda（Java 8+）
new Thread(() -> {
    System.out.println("Lambda 线程");
}).start();
```

`Runnable` 接口的实现更加灵活，类可以继承其他类的同时实现 `Runnable`。

### 方式三：实现 Callable 接口

如果线程需要返回结果或抛出异常，需要使用 `Callable` 和 `Future`：

```java
public class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        return 42;
    }
}

// 使用 FutureTask 包装
FutureTask<Integer> future = new FutureTask<>(new MyCallable());
new Thread(future).start();

// 获取结果（会阻塞直到完成）
Integer result = future.get();
```

## 线程的状态与生命周期

Java 线程有六种状态，通过 `Thread.State` 枚举定义：

```java
public enum State {
    NEW,        // 新建：线程创建但未启动
    RUNNABLE,   // 可运行：正在 JVM 中执行或等待 CPU 时间片
    BLOCKED,    // 阻塞：等待获取监视器锁
    WAITING,    // 等待：无限期等待另一个线程
    TIMED_WAITING, // 超时等待：有限期等待
    TERMINATED  // 终止：run() 方法执行完毕
}
```

线程状态转换图：

```
NEW ──── start() ────> RUNNABLE
                           │
            ┌──────────────┼──────────────┐
            │              │              │
            v              v              v
        BLOCKED      WAITING         TIMED_WAITING
    (synchronized)  (wait/join)    (sleep/join/await)
            │              │              │
            │              │              │
            └──────────────┴──────────────┘
                           │
                           v
                    TERMINATED
```

### 阻塞 vs 等待：两种暂停的区别

**BLOCKED** 状态发生在线程尝试获取对象锁但锁不可用时。比如，两个线程竞争同一个 `synchronized` 方法或代码块，未获得锁的线程会进入 BLOCKED 状态。

**WAITING** 状态发生在线程调用 `Object.wait()`、`Thread.join()`（无参数）或 `LockSupport.park()` 时。处于 WAITING 状态的线程需要等待其他线程显式唤醒。

**TIMED_WAITING** 是 WAITING 的有时间限制版本。`Thread.sleep(millis)`、`Object.wait(millis)`、`Thread.join(millis)`、`LockSupport.parkNanos()` 等操作都会让线程进入 TIMED_WAITING 状态。

## 线程的常用操作

### sleep：休眠不释放锁

```java
Thread.sleep(1000);  // 休眠 1 秒，不释放已持有的锁
```

`sleep()` 静态方法让当前线程暂停执行指定时间。常用于轮询场景，或模拟网络延迟。但注意，`sleep()` 不会释放已持有的锁，如果线程持有某个对象的锁并在 `synchronized` 块中 `sleep()`，锁不会被释放。

### join：等待线程结束

```java
Thread t = new Thread(() -> {
    // 耗时操作
    try {
        Thread.sleep(5000);
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
});
t.start();

// 主线程等待 t 执行完毕
t.join();  // 阻塞直到 t 结束
t.join(3000);  // 最多等待 3 秒

System.out.println("t 执行完毕或等待超时，继续执行");
```

`join()` 用于等待一个线程执行完毕，常用于主线程收集子线程的执行结果。

### yield：谦让但不保证

```java
Thread.yield();
```

`yield()` 提示调度器当前线程愿意让出 CPU 时间片，但调度器可能忽略这个提示。这不是一个可靠的编程工具，通常仅用于调试或测试。

## 守护线程：默默奉献的后台工作者

Java 线程分为用户线程（User Thread）和守护线程（Daemon Thread）。主线程和所有你创建的线程默认都是用户线程。当所有用户线程结束时，JVM 会立即终止所有守护线程。

```java
Thread daemonThread = new Thread(() -> {
    while (true) {
        // 守护线程的使命
        System.out.println("守护线程运行中");
    }
});
daemonThread.setDaemon(true);  // 必须在 start() 前设置
daemonThread.start();
```

常见的守护线程包括：垃圾回收线程（GC）、JIT 编译器线程、应用服务器的连接清理线程等。守护线程的典型特征是：为用户线程提供服务，但不应该做为主要任务的执行者。

## 线程优先级：调度的参考而非保证

```java
thread.setPriority(Thread.MAX_PRIORITY);  // 10
thread.setPriority(Thread.NORM_PRIORITY); // 5
thread.setPriority(Thread.MIN_PRIORITY);  // 1
```

线程优先级只是一个提示，调度器可能完全忽略它。在不同操作系统上，优先级的效果也不同。依赖优先级来保证正确性的程序是脆弱的——应该使用更可靠的同步机制。
