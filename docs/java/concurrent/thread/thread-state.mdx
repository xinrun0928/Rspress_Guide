# Thread 生命周期与状态转换

面试官问：「Thread 有哪些状态？它们之间是怎么转换的？」

你回答：「New、Runnable、Blocked、Waiting、TimedWaiting、Terminated……」

面试官点点头：「那从 Waiting 怎么到 Blocked？」

你犹豫了一下。

这个问题看似简单，但要准确描述六种状态之间的转换关系，还是需要仔细梳理。

## 六种状态的完整定义

Java 中，线程有六种明确的状态：

```java
public enum State {
    NEW,              // 新建
    RUNNABLE,          // 可运行
    BLOCKED,          // 阻塞
    WAITING,          // 等待
    TIMED_WAITING,    // 限时等待
    TERMINATED         // 终止
}
```

## 状态详解

### NEW：出生的瞬间

当用 `new Thread()` 创建线程对象时，线程处于 NEW 状态。

```java
Thread thread = new Thread(() -> {
    System.out.println("I'm running");
});
// 此时 thread 的状态是 NEW
System.out.println(thread.getState());  // NEW
```

NEW 状态的线程还没有开始执行，只是一个普通的 Java 对象。

### RUNNABLE：整装待发

调用 `start()` 后，线程进入 RUNNABLE 状态。

```java
thread.start();
System.out.println(thread.getState());  // RUNNABLE
```

RUNNABLE 是一个「复合状态」，包含两个子状态：

- **Running**：线程正在 CPU 上执行
- **Ready**：线程在等待 CPU 时间片

```
┌─────────────────────────────────────────┐
│              RUNNABLE                    │
│  ┌─────────────┐    ┌─────────────┐    │
│  │   Running   │◄──►│    Ready    │    │
│  │ (正在执行)   │    │ (等待CPU)   │    │
│  └─────────────┘    └─────────────┘    │
└─────────────────────────────────────────┘
```

JVM 的调度器决定哪个 RUNNABLE 线程获得 CPU 时间片。

### BLOCKED：等待锁的无奈

当线程试图获取一个被其他线程持有的锁时，会进入 BLOCKED 状态。

```java
// 线程 A 持有锁
synchronized (lock) {
    // 模拟耗时操作
    Thread.sleep(10000);
}

// 线程 B 尝试获取同一个锁
synchronized (lock) {  // 被阻塞，进入 BLOCKED
    // ...
}
```

BLOCKED 状态的线程在等待获得锁，只有以下情况会退出 BLOCKED：

- 持锁线程释放了锁，BLOCKED 线程获得了锁
- BLOCKED 线程被中断，抛出 `InterruptedException`

### WAITING：无尽的等待

当线程调用以下方法时会进入 WAITING 状态：

- `Object.wait()`
- `Thread.join()`（不带超时）
- `LockSupport.park()`

```java
// 线程 A
synchronized (lock) {
    lock.wait();  // 进入 WAITING，释放锁
}

// 线程 B
synchronized (lock) {
    lock.notify();  // 唤醒线程 A
}
```

WAITING 状态的线程必须被其他线程显式唤醒：

- `Object.notify()` 或 `Object.notifyAll()`
- 被 join 的线程执行完毕
- `LockSupport.unpark()`

### TIMED_WAITING：限时的等待

带超时的等待，线程会在指定时间后自动苏醒：

```java
// 这些方法都会进入 TIMED_WAITING
Thread.sleep(1000);                    // sleep
object.wait(1000);                    // wait with timeout
thread.join(1000);                    // join with timeout
LockSupport.parkNanos(1000000);       // park with timeout
LockSupport.parkUntil(System.currentTimeMillis() + 1000);
```

| 方法 | TIMED_WAITING 超时条件 |
|---|---|
| `Thread.sleep(ms)` | 经过指定毫秒数 |
| `Object.wait(ms)` | 经过指定毫秒数或被唤醒 |
| `Thread.join(ms)` | 经过指定毫秒数或被 join 的线程结束 |
| `LockSupport.parkNanos(ns)` | 经过指定纳秒数 |
| `LockSupport.parkUntil(deadline)` | 到达指定时间点 |

### TERMINATED：使命完成

线程的 `run()` 方法执行完毕，或者 `run()` 中抛出未捕获的异常，线程进入 TERMINATED 状态。

```java
Thread thread = new Thread(() -> {
    System.out.println("Task completed");
});

thread.start();
thread.join();  // 等待线程结束
System.out.println(thread.getState());  // TERMINATED
```

TERMINATED 是线程的最终状态，不可逆转。

## 状态转换实战

### 场景一：synchronized 锁竞争

```java
public class LockContentionDemo {
    public static void main(String[] args) throws InterruptedException {
        Object lock = new Object();

        Thread thread1 = new Thread(() -> {
            synchronized (lock) {
                try {
                    Thread.sleep(2000);  // 持有锁 2 秒
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        Thread thread2 = new Thread(() -> {
            synchronized (lock) {  // 等待获得锁
                System.out.println("Thread 2 got the lock");
            }
        });

        // 初始状态
        System.out.println("Before start: " + thread1.getState());  // NEW
        System.out.println("Before start: " + thread2.getState());  // NEW

        thread1.start();
        Thread.sleep(100);  // 等待 thread1 获取锁

        System.out.println("Thread1 running: " + thread1.getState());  // RUNNABLE
        System.out.println("Thread2 blocked: " + thread2.getState());  // BLOCKED

        thread2.start();
        thread2.join();

        thread1.join();
        System.out.println("After complete: " + thread1.getState());  // TERMINATED
    }
}
```

### 场景二：wait/notify 模式

```java
public class WaitNotifyDemo {
    public static void main(String[] args) throws InterruptedException {
        Object lock = new Object();

        Thread waiter = new Thread(() -> {
            synchronized (lock) {
                try {
                    System.out.println("Waiting for signal...");
                    lock.wait();  // 进入 WAITING
                    System.out.println("Received signal!");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        Thread notifier = new Thread(() -> {
            synchronized (lock) {
                System.out.println("Sending signal...");
                lock.notify();  // 唤醒等待的线程
            }
        });

        waiter.start();
        Thread.sleep(100);
        System.out.println("Waiter state: " + waiter.getState());  // WAITING

        notifier.start();
        waiter.join();

        System.out.println("Final waiter state: " + waiter.getState());  // TERMINATED
    }
}
```

### 场景三：join 的等待

```java
public class JoinDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread longTask = new Thread(() -> {
            try {
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });

        longTask.start();
        System.out.println("Before join: " + longTask.getState());  // TIMED_WAITING

        longTask.join(1000);  // 等待 1 秒
        System.out.println("After 1 second: " + longTask.getState());  // TIMED_WAITING

        longTask.join();  // 等待完成
        System.out.println("After complete: " + longTask.getState());  // TERMINATED
    }
}
```

## 状态转换表

| 当前状态 | 操作/条件 | 目标状态 |
|---|---|---|
| NEW | `start()` | RUNNABLE |
| RUNNABLE | 获得 CPU 时间片 | RUNNABLE（Running） |
| RUNNABLE | 失去 CPU 时间片 | RUNNABLE（Ready） |
| RUNNABLE | `synchronized` 获取不到锁 | BLOCKED |
| RUNNABLE | `wait()` | WAITING |
| RUNNABLE | `join()` | WAITING |
| RUNNABLE | `sleep()` | TIMED_WAITING |
| BLOCKED | 获得锁 | RUNNABLE |
| WAITING | `notify()` / `notifyAll()` | BLOCKED（竞争锁） |
| WAITING | 被中断 | BLOCKED（竞争锁） |
| TIMED_WAITING | 超时结束 | RUNNABLE |
| TIMED_WAITING | 被唤醒 | BLOCKED（竞争锁） |
| TIMED_WAITING | 被中断 | BLOCKED（竞争锁） |
| 任意状态 | `run()` 结束或抛异常 | TERMINATED |

## 面试追问

### Q1：为什么 RUNNABLE 还要细分？

这是 Java 对线程状态的抽象。实际上 JVM 层面（Native 线程）的状态和 Java层面的状态不完全对应。RUNNABLE 包含了「在 JVM 中可执行」和「正在 CPU 上执行」两种情况。

### Q2：WAITING 和 BLOCKED 的区别？

WAITING 是线程主动调用 `wait()`、`join()`、`park()` 等方法进入的。BLOCKED 是线程被动等待锁。WAITING 的线程需要被其他线程显式唤醒。

### Q3：`Object.wait()` 和 `Thread.sleep()` 的区别？

- `wait()` 会释放锁，`sleep()` 不会
- `wait()` 只能在 synchronized 中调用，`sleep()` 可以在任何地方调用
- `wait()` 被唤醒后需要重新竞争锁

## 总结

线程的六种状态构成了 Java 并发编程的基础：

- **NEW → RUNNABLE**：调用 `start()`
- **RUNNABLE ↔ BLOCKED**：获取 synchronized 锁
- **RUNNABLE → WAITING**：调用 `wait()`、`join()`、`park()`
- **WAITING → BLOCKED**：被唤醒后竞争锁
- **任意 → TERMINATED**：任务执行完毕

理解状态转换，才能写出正确的并发代码。
