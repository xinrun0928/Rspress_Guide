# 线程状态与生命周期

想象一个员工从入职到离职的全过程：首先是新员工报到（NEW），然后开始熟悉工作（RUNNABLE），可能在午休时暂停工作（TIMED_WAITING），可能在等待同事的文件（WAITING），可能在专心处理紧急任务（BLOCKED），最后完成任务离开公司（TERMINATED）。这个过程和 Java 线程的生命周期惊人地相似。

Java 线程有六种状态，定义在 `java.lang.Thread.State` 枚举中。理解这些状态及其转换，是掌握多线程编程的基础。

## 六种线程状态

### NEW（新建状态）

这是线程被创建但尚未启动的状态。就好比你定义了一个 Runnable 实现类，但还没有创建 Thread 对象，或者创建了 Thread 对象但还没有调用 `start()` 方法。

```java
Thread thread = new Thread(() -> {
    // 线程体
});
// 此时 thread.getState() == NEW
System.out.println(thread.getState()); // NEW
```

在这个状态下，线程还没有分配任何系统资源，仅仅是 Java 层面的对象。

### RUNNABLE（可运行状态）

一旦调用了 `start()` 方法，线程就进入了 RUNNABLE 状态。这个状态实际上是"可运行"状态，而不是"正在运行"状态。在单核 CPU 上，同一时刻只能有一个线程真正在执行，其他 RUNNABLE 状态的线程在等待 CPU 时间片。

```java
thread.start();
// 此时 thread.getState() == RUNNABLE
System.out.println(thread.getState()); // RUNNABLE
```

RUNNABLE 状态涵盖了操作系统的两种状态：

- **Ready（就绪）**：线程已准备好，只等 CPU 调度
- **Running（运行）**：线程正在 CPU 上执行

Java 将这两种状态合并为 RUNNABLE，因为在 JVM 看来，线程一旦获得了 CPU 时间片就是可运行的。

### BLOCKED（阻塞状态）

线程在等待获取监视器锁（Monitor Lock）时，会进入 BLOCKED 状态。当一个线程试图进入一个被其他线程持有的 synchronized 块/方法时，就会被阻塞在这个锁的入口处。

```java
// 线程A持有锁
synchronized (lock) {
    Thread.sleep(10000); // 长时间持有锁
}

// 线程B尝试进入这个同步块
// 此时线程B的状态是 BLOCKED
```

BLOCKED 状态是线程间竞争锁失败时的状态。关于 synchronized 的锁机制，我们后面会深入讲解。

### WAITING（无限期等待状态）

线程进入 WAITING 状态表示它正在等待另一个线程执行特定的操作。有以下几种方式会使线程进入 WAITING 状态：

- 调用 `Object.wait()`
- 调用 `Thread.join()`（不带超时参数）
- 调用 `LockSupport.park()`

```java
public class WaitingDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            // 主线程调用 thread.join()，主线程会进入 WAITING
        });
        thread.start();
        thread.join(); // 主线程在这里等待 thread 执行完成
        System.out.println("thread 执行完了，主线程继续");
    }
}
```

处于 WAITING 状态的线程需要等待其他线程显式地唤醒：
- 被 `wait()` 等待的线程需要其他线程调用 `notify()` 或 `notifyAll()`
- 被 `join()` 等待的线程需要目标线程执行完成

### TIMED_WAITING（限时等待状态）

与 WAITING 类似，但有超时限制。以下几种方式会使线程进入 TIMED_WAITING 状态：

- 调用 `Thread.sleep(milliseconds)`
- 调用 `Object.wait(timeout)`
- 调用 `Thread.join(timeout)`
- 调用 `LockSupport.parkNanos()` 或 `LockSupport.parkUntil()`

```java
public class TimedWaitingDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            try {
                Thread.sleep(5000); // 睡眠5秒
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        thread.start();

        // 线程刚启动时是 RUNNABLE，sleep 后变成 TIMED_WAITING
        System.out.println("线程状态: " + thread.getState()); // TIMED_WAITING
    }
}
```

### TERMINATED（终止状态）

线程的 `run()` 方法正常执行完毕，或者抛出未捕获的异常，线程就会进入 TERMINATED 状态。一旦线程终止，就不能再重新启动了。

```java
Thread thread = new Thread(() -> {
    System.out.println("线程执行中");
});
thread.start();
thread.join(); // 等待线程结束
System.out.println("线程状态: " + thread.getState()); // TERMINATED
```

## 常见状态转换示例

### 场景一：锁竞争导致的 BLOCKED

```java
public class BlockedDemo {
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        // 线程1：获取锁后长时间持有
        Thread thread1 = new Thread(() -> {
            synchronized (lock) {
                try {
                    Thread.sleep(10000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "Thread-1");

        // 线程2：无法获取锁，被阻塞
        Thread thread2 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("Thread-2 获取到锁");
            }
        }, "Thread-2");

        thread1.start();
        Thread.sleep(100); // 确保 thread1 先获取锁
        thread2.start();
        Thread.sleep(100);

        System.out.println("Thread-1 状态: " + thread1.getState()); // TIMED_WAITING
        System.out.println("Thread-2 状态: " + thread2.getState()); // BLOCKED
    }
}
```

### 场景二：join() 导致的 WAITING

```java
public class JoinDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("子线程执行完成");
        }, "Child-Thread");

        thread.start();
        thread.join(); // main 线程等待，直到子线程执行完毕

        System.out.println("main 线程继续执行");
        // 此时 thread 的状态是 TERMINATED
    }
}
```

### 场景三：sleep vs wait

```java
public class SleepVsWait {
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            synchronized (lock) {
                try {
                    // sleep 不释放锁，线程状态是 TIMED_WAITING
                    Thread.sleep(3000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "Sleep-Thread");

        thread.start();
        Thread.sleep(100);
        System.out.println("Sleep-Thread 状态: " + thread.getState()); // TIMED_WAITING

        Thread thread2 = new Thread(() -> {
            synchronized (lock) {
                // wait 释放锁，线程状态是 WAITING
                try {
                    lock.wait();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "Wait-Thread");

        // 注意：由于 sleep 线程持有锁，这个线程会被 BLOCKED
        // 等 sleep 释放锁后，它才会执行 wait()，变成 WAITING
        thread2.start();
    }
}
```

## 状态监控的实际应用

理解线程状态对于排查多线程问题至关重要。在生产环境中，如果发现大量线程处于 BLOCKED 状态，往往意味着严重的锁竞争问题；如果大量线程处于 WAITING 状态，可能是死锁或者线程间协作出现了问题。

```java
import java.util.Map;
import java.util.concurrent.*;

public class StateMonitor {
    public static void main(String[] args) throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(5);
        CountDownLatch latch = new CountDownLatch(5);

        for (int i = 0; i < 5; i++) {
            executor.submit(() -> {
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    latch.countDown();
                }
            });
        }

        // 监控线程状态
        Thread monitor = new Thread(() -> {
            while (latch.getCount() > 0) {
                Map<Thread, StackTraceElement[]> allThreads = Thread.getAllStackTraces();
                System.out.println("========== 线程状态快照 ==========");
                allThreads.forEach((thread, stack) -> {
                    if (thread.getState() != Thread.State.RUNNABLE) {
                        System.out.printf("%s: %s%n", thread.getName(), thread.getState());
                    }
                });
                System.out.println("==================================");
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    break;
                }
            }
        });
        monitor.start();

        latch.await();
        executor.shutdown();
    }
}
```
