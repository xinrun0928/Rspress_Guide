# Fork/Join 框架与工作窃取算法

现代多核处理器时代，如何充分利用 CPU 的每一个核心成为性能优化的关键。`Fork/Join` 框架是 JDK 7 引入的并行计算框架，它专门设计用于**分治策略**的并行化执行。其核心是**工作窃取算法**——让空闲的线程"偷走"其他线程的任务来执行，从而实现负载均衡和高效利用 CPU。

## Fork/Join 的设计思想

### 什么是 Fork/Join？

Fork/Join 的思想源于"分而治之"：

1. **Fork（分叉）**：把大任务分解成多个小任务
2. **Join（合并）**：等待子任务完成，合并结果

```
┌────────────────────────────────────────────────────────────────┐
│                      Fork/Join 工作流程                          │
│                                                                 │
│                        [大任务]                                   │
│                           │                                     │
│                    ┌───────┴───────┐                           │
│                    ▼               ▼                            │
│               [任务1]          [任务2]                           │
│               /      \         /      \                        │
│              ▼        ▼       ▼        ▼                       │
│          [子任务] [子任务] [子任务] [子任务]                      │
│              │        │       │        │                        │
│              └────────┴───────┴────────┘                       │
│                           │                                     │
│                     [合并结果]                                   │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 为什么需要 Fork/Join？

传统的线程池存在负载不均的问题：

```java
public class TraditionalProblem {
    
    private final ExecutorService executor = 
        Executors.newFixedThreadPool(4);
    
    public long sum(long start, long end) {
        if (end - start <= THRESHOLD) {
            return compute(start, end);
        }
        
        long mid = (start + end) / 2;
        Future<Long> left = executor.submit(() -> sum(start, mid));
        Future<Long> right = executor.submit(() -> sum(mid, end));
        
        return left.get() + right.get();
    }
    
    // 问题：
    // 1. 每个子任务都要 submit，产生额外开销
    // 2. 线程数固定，可能导致某些线程过忙而其他空闲
    // 3. get() 可能阻塞
}
```

Fork/Join 框架解决了这些问题。

### 工作窃取算法

```
┌────────────────────────────────────────────────────────────────┐
│                    工作窃取算法                                   │
│                                                                 │
│   Thread-1 队列      Thread-2 队列      Thread-3 队列           │
│   ┌─────────┐       ┌─────────┐       ┌─────────┐              │
│   │ Task-A  │       │ Task-D  │       │ Task-G  │              │
│   │ Task-B  │       │ Task-E  │       │ Task-H  │              │
│   │ Task-C  │       │ Task-F  │       │         │              │
│   │ (尾部)  │       │ (尾部)  │       │         │              │
│   └────┬────┘       └─────────┘       └─────────┘              │
│        │                                                         │
│        │ 执行                            空闲时偷 Task-A         │
│        ▼                              ◄─────────────            │
│   [执行 Task-C] ──────────────────────────────────►            │
│        │                                                         │
│   执行 Task-B ◄─── 队列空了，从 Thread-1 偷                      │
│        │                                                         │
│   执行 Task-A ◄─── 继续偷                                        │
│                                                                 │
│   特点：                                                         │
│   1. 每个线程从自己的队列尾部取任务（生产者）                      │
│   2. 从其他线程的队列头部偷任务（消费者）                          │
│   3. 减少竞争，提高缓存命中率                                      │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## ForkJoinTask：任务抽象

### RecursiveTask：有返回值的任务

```java
public class SumTask extends RecursiveTask<Long> {
    
    private static final long THRESHOLD = 10000;
    private final long[] array;
    private final int start;
    private final int end;
    
    public SumTask(long[] array, int start, int end) {
        this.array = array;
        this.start = start;
        this.end = end;
    }
    
    @Override
    protected Long compute() {
        int length = end - start;
        
        // 小任务直接计算
        if (length <= THRESHOLD) {
            return computeDirectly();
        }
        
        // 大任务分解
        int mid = start + length / 2;
        SumTask leftTask = new SumTask(array, start, mid);
        SumTask rightTask = new SumTask(array, mid, end);
        
        // Fork：异步执行子任务
        leftTask.fork();
        rightTask.fork();
        
        // Join：合并结果
        long leftResult = leftTask.join();
        long rightResult = rightTask.join();
        
        return leftResult + rightResult;
    }
    
    private long computeDirectly() {
        long sum = 0;
        for (int i = start; i < end; i++) {
            sum += array[i];
        }
        return sum;
    }
}

// 使用
public class SumExample {
    
    public long parallelSum(long[] array) {
        ForkJoinPool pool = ForkJoinPool.commonPool();
        SumTask task = new SumTask(array, 0, array.length);
        return pool.invoke(task);
    }
}
```

## ForkJoinPool 的工作原理

### 双端队列

```java
// 每个工作线程维护一个双端队列
class WorkQueue {
    
    // 数组存储任务
    ForkJoinTask<?>[] array;
    
    // 头部索引（从头部取）
    volatile int head;
    
    // 尾部索引（从尾部放）
    int top;
    
    // 线程从自己的队列尾部取任务（LIFO）
    final ForkJoinTask<?> pollAt(int index) {
        int mask = array.length - 1;
        return array[(mask & head)];
    }
    
    // 从其他线程的队列头部偷任务（FIFO）
    final ForkJoinTask<?> poll() {
        int mask = array.length - 1;
        ForkJoinTask<?> t = array[mask & top];

    }
}
```

### 工作窃取的执行流程

```java
public class ForkJoinWorkerThread {
    
    final ForkJoinPool pool;
    final ForkJoinWorkerThread workQueue;
    
    void run() {
        while (!java.util.concurrent.locks.LockSupport.interrupted()) {
            ForkJoinTask<?> task;
            
            if ((task = scan()) != null) {
                // 窃取成功，执行任务
                task.doExec();
            } else {
                // 没有任务，等待
                awaitWork();
            }
        }
    }
    
    // 扫描并窃取任务
    private ForkJoinTask<?> scan() {
        // 1. 先尝试从自己的队列获取
        if (workQueue != null) {
            ForkJoinTask<?> task = workQueue.poll();
            if (task != null)
                return task;
        }
        
        // 2. 随机扫描其他线程的队列，窃取任务
        for (int i = 0; i < poolSize; i++) {
            WorkQueue q = pool.queues[random.nextInt(poolSize)];
            if (q != null && q != workQueue) {
                ForkJoinTask<?> task = q.poll();
                if (task != null)
                    return task;
            }
        }
        
        return null;
    }
}
```

## 实际应用场景

### 场景1：并行数组处理

```java
public class ParallelArrayProcessing {
    
    public <T> void parallelForEach(
            List<T> list, 
            Consumer<T> action,
            ForkJoinPool pool) {
        
        int threshold = 1000;
        int size = list.size();
        
        RecursiveAction task = new RecursiveAction() {
            @Override
            protected void compute() {
                if (size <= threshold) {
                    list.forEach(action);
                } else {
                    int mid = size / 2;
                    invokeAll(
                        new SubTask(list, 0, mid, action),
                        new SubTask(list, mid, size, action)
                    );
                }
            }
        };
        
        pool.invoke(task);
    }
    
    private static class SubTask extends RecursiveAction {
        private final List<?> list;
        private final int start;
        private final int end;
        private final Consumer<?> action;
        
        SubTask(List<?> list, int start, int end, Consumer<?> action) {
            this.list = list;
            this.start = start;
            this.end = end;
            this.action = action;
        }
        
        @Override
        protected void compute() {
            list.subList(start, end).forEach(action);
        }
    }
}
```

### 场景2：并行树遍历

```java
public class ParallelTreeTraversal {
    
    public class TreeNode<T> {
        T value;
        TreeNode<T> left;
        TreeNode<T> right;
    }
    
    public <T> List<T> parallelInOrder(TreeNode<T> root) {
        ForkJoinPool pool = ForkJoinPool.commonPool();
        List<T> result = Collections.synchronizedList(new ArrayList<>());
        
        pool.invoke(new InOrderTask<>(root, result));
        return result;
    }
    
    private class InOrderTask<T> extends RecursiveAction {
        private final TreeNode<T> node;
        private final List<T> result;
        
        InOrderTask(TreeNode<T> node, List<T> result) {
            this.node = node;
            this.result = result;
        }
        
        @Override
        protected void compute() {
            if (node == null) return;
            
            InOrderTask<T> leftTask = new InOrderTask<>(node.left, result);
            InOrderTask<T> rightTask = new InOrderTask<>(node.right, result);
            
            // 并行执行
            invokeAll(leftTask, rightTask);
            
            // 收集结果
            result.add(node.value);
        }
    }
}
```

## 注意事项

### 1. 任务不能有依赖

```java
public class WrongDependency {
    
    // 错误示例：任务之间有依赖
    public void wrongExample() {
        ForkJoinPool pool = ForkJoinPool.commonPool();
        
        // task2 依赖 task1 的结果
        RecursiveAction task1 = new RecursiveAction() {
            @Override
            protected void compute() { /* ... */ }
        };
        
        RecursiveAction task2 = new RecursiveAction() {
            @Override
            protected void compute() { 
                task1.join();  // 错误：在 compute 中等待
            }
        };
        
        // ForkJoinPool 假设任务可以并行执行
        // 在 compute 中等待会导致死锁
    }
}
```

### 3. 避免阻塞

```java
public class AvoidBlocking {
    
    public void avoidBlocking() {
        // 错误：在 compute 中调用阻塞方法
        public class BadTask extends RecursiveAction {
            @Override
            protected void compute() {
                // 阻塞会导致工作线程闲置
                Thread.sleep(1000);  // 错误！
            }
        }
        
        // 正确：使用 ForkJoinPool 的ManagedBlocker
        public class GoodTask extends RecursiveAction {
            @Override
            protected void compute() {
                ForkJoinPool.managedBlock(new ForkJoinPool.ManagedBlocker() {
                    @Override
                    public boolean block() throws InterruptedException {
                        Thread.sleep(1000);  // 正确
                        return true;
                    }
                    
                    @Override
                    public boolean isReleasable() {
                        return false;
                    }
                });
            }
        }
    }
}
```

## 总结

`Fork/Join` 框架是 JDK 7 的重要特性：

- **工作窃取算法**：空闲线程从其他线程队列偷任务
- **双端队列**：减少竞争，提高缓存命中率
- **RecursiveTask/RecursiveAction**：支持分治策略的任务
- **公共线程池**：`ForkJoinPool.commonPool()` 可直接使用

适用场景：
- 分治策略的并行化
- 大数据量的并行处理
- 递归任务的并行执行
