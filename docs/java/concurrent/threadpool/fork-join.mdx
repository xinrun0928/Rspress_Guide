# Fork/Join 框架与工作窃取算法

想象一个场景：需要计算 1 到 100 万整数的和。如果是单线程，累加 100 万次即可。但如果用多线程，每个线程计算一部分，最后汇总结果，理论上可以快 4 倍。

但有个问题：每个子任务的计算量可能不均衡——第一个线程计算 1-50 万，第二个计算 50-100 万，万一第一个线程的数据更复杂呢？

**Fork/Join 框架** 解决了这个问题。它采用**工作窃取（Work Stealing）**算法，让空闲线程可以从忙碌线程的队列尾部「偷」任务，确保所有线程都有事可做。

## 为什么需要 Fork/Join？

传统的线程池存在两个问题：

1. **任务分配不均**：分割后的子任务计算量可能差异很大
2. **结果汇总复杂**：需要额外的同步机制合并结果

Fork/Join 框架的设计哲学是：

```
分而治之 + 工作窃取 = 高效并行
```

```java
// 使用 Fork/Join 计算整数和
public class SumTask extends RecursiveTask<Long> {
    private final long[] array;
    private final int start, end;

    public SumTask(long[] array, int start, int end) {
        this.array = array;
        this.start = start;
        this.end = end;
    }

    @Override
    protected Long compute() {
        int length = end - start;
        // 小任务直接计算
        if (length <= 10000) {
            return sumSequentially();
        }

        // 大任务拆分为两个子任务
        int middle = start + length / 2;
        SumTask left = new SumTask(array, start, middle);
        SumTask right = new SumTask(array, middle, end);

        // 并行执行：fork 异步执行，join 等待结果
        left.fork();
        Long rightResult = right.compute();
        Long leftResult = left.join();

        return leftResult + rightResult;
    }

    private long sumSequentially() {
        long sum = 0;
        for (int i = start; i < end; i++) {
            sum += array[i];
        }
        return sum;
    }
}
```

执行方式：

```java
ForkJoinPool pool = ForkJoinPool.commonPool();
long[] array = new long[1_000_000];
// 初始化数组...

SumTask task = new SumTask(array, 0, array.length);
Long result = pool.invoke(task);
System.out.println("Sum: " + result);
```

## Fork/Join 的核心原理

### 任务分解（Fork）

Fork/Join 的核心是 `RecursiveTask`（有返回值）或 `RecursiveAction`（无返回值）：

```java
public abstract class RecursiveTask<V> extends ForkJoinTask<V> {
    public final V fork() { ... }      // 异步提交到队列
    public final V join() { ... }      // 阻塞等待结果
    protected abstract V compute();   // 子类实现计算逻辑
}
```

**递归分解的艺术**：

```
任务 [0, 1000000)
         │
    fork / join
         │
    ┌────┴────┐
[0,500000)  [500000, 1000000)
    │
   fork / join
    │
  ┌──┴──┐
[0,250k) [250k,500k)
   ...继续分解直到阈值
```

### 工作窃取（Work Stealing）

这是 Fork/Join 最精妙的设计。每个工作线程有自己的双端队列：

```
线程 A 的队列：  [任务3] [任务2] [任务1]  <- 头部（处理）
                尾部（窃取）

线程 B 的队列：  [任务6] [任务5] [任务4]
```

- **正常情况**：线程从自己的队列头部取任务（后进先出，LIFO）
- **空闲时**：从其他线程的队列尾部「偷」任务（FIFO）

```java
// ForkJoinPool 的工作窃取实现
final void workStealing() {
    ForkJoinWorkerThread wt = thread;
    WorkQueue[] ws = workQueues;
    int n = ws.length;

    while (queueIsNotEmpty(wt)) {
        ForkJoinTask<?> t;
        WorkQueue w = findNonEmptyQueue(wt);
        if (w != null && (t = w.poll()) != null) {
            executeTask(t, wt);
        }
    }
}
```

:::tip 为什么从尾部窃取？
从尾部窃取可以让被窃取的线程继续处理队列头部的任务，避免与窃取者竞争同一条数据。
多个线程访问同一队列的头部会产生竞争，从尾部窃取可以减少这种竞争。
:::

### join 的阻塞机制

`join()` 不是简单的等待，它会**触发工作窃取**：

```java
public final V join() {
    int s;
    if ((s = status) < 0)
        return getRawResult();

    // 如果任务还没完成，当前线程尝试帮忙执行
    if (s != 0) {
        Thread t = Thread.currentThread();
        if (t instanceof ForkJoinWorkerThread)
            drainHelp((ForkJoinWorkerThread) t);
        else
            helpInterruptibleWorker();
    }
    return getRawResult();
}
```

这意味着：当你调用 `left.join()` 时，如果 `left` 还在执行，当前线程会帮助执行 `left` 的任务，而不是空等！

## ForkJoinPool 的特点

### 两种创建方式

```java
// 方式1：使用公共池（推荐）
ForkJoinPool pool = ForkJoinPool.commonPool();

// 方式2：创建独立池
ForkJoinPool pool2 = new ForkJoinPool(
    Runtime.getRuntime().availableProcessors()  // 并行度
);
```

### 公共池的特殊性

`ForkJoinPool.commonPool()` 是 JVM 级别的单例：

```java
private static final ForkJoinPool common;
static {
    common = commonPool();
}

// commonPool() 的默认并行度
public static ForkJoinPool commonPool() {
    return common;
}
```

:::warning 公共池的陷阱
公共池是所有使用它的任务共享的。如果某个任务占满了公共池，其他使用公共池的任务就会受影响。建议创建独立池来隔离不同业务。
:::

### asyncMode

ForkJoinPool 支持一种特殊的模式——`asyncMode`：

```java
ForkJoinPool pool = new ForkJoinPool(
    parallelism,
    ForkJoinPool.defaultForkJoinWorkerThreadFactory,
    null,
    true  // asyncMode: 任务不保证 FIFO 顺序
);
```

当设置为 `asyncMode = true` 时，任务以 FIFO 顺序处理，适合**事件驱动型任务**，而不适合需要合并结果的递归任务。

## 实际应用场景

### 场景1：并行排序

```java
public class ParallelMergeSort extends RecursiveAction {
    private final long[] array;
    private final int low, high;

    public ParallelMergeSort(long[] array, int low, int high) {
        this.array = array;
        this.low = low;
        this.high = high;
    }

    @Override
    protected void compute() {
        if (low >= high) return;

        int middle = (low + high) >>> 1;

        // 并行排序左右两半
        ParallelMergeSort left = new ParallelMergeSort(array, low, middle);
        ParallelMergeSort right = new ParallelMergeSort(array, middle + 1, high);

        invokeAll(left, right);  // 并行执行两个子任务

        merge(low, middle, high);
    }

    private void merge(int low, int middle, int high) {
        // 标准的归并排序合并逻辑
        long[] temp = new long[high - low + 1];
        int i = low, j = middle + 1, k = 0;
        while (i <= middle && j <= high) {
            temp[k++] = array[i] <= array[j] ? array[i++] : array[j++];
        }
        while (i <= middle) temp[k++] = array[i++];
        while (j <= high) temp[k++] = array[j++];
        System.arraycopy(temp, 0, array, low, temp.length);
    }
}
```

### 场景2：并行遍历树

```java
public class TreeNode {
    long value;
    TreeNode left, right;
}

public class TreeSumTask extends RecursiveTask<Long> {
    private final TreeNode node;

    public TreeSumTask(TreeNode node) {
        this.node = node;
    }

    @Override
    protected Long compute() {
        if (node == null) return 0L;

        // 并行处理左右子树
        TreeSumTask leftTask = new TreeSumTask(node.left);
        TreeSumTask rightTask = new TreeSumTask(node.right);

        leftTask.fork();
        long rightSum = rightTask.compute();
        long leftSum = leftTask.join();

        return node.value + leftSum + rightSum;
    }
}
```

## 性能调优建议

| 参数 | 默认值 | 调优建议 |
| --- | --- | --- |
| `parallelism` | CPU 核心数 - 1 | I/O 密集型可适当增加 |
| `forkJoinPool.common.parallelism` | JVM 属性 | 可通过 `-D` 参数调整 |
| 任务阈值 | 1000 | 根据实际计算量调整，太小增加调度开销 |

:::tip 阈值的选择
任务阈值太小会导致任务过多，调度开销大于计算收益；太大则并行度不足。可以通过实验找到最优值。
:::


## 总结

Fork/Join 框架是 Java 7 引入的专为**递归任务**设计的并行框架：

- **分而治之**：通过 `fork()` 分解任务，`join()` 合并结果
- **工作窃取**：空闲线程从其他线程队列尾部偷任务，保证负载均衡
- **协作式执行**：调用 `join()` 时，当前线程会帮忙执行待处理任务
- **公共池 vs 独立池**：公共池共享但便捷，独立池隔离但需要管理

理解工作窃取算法的设计思想，对于编写高效的并行程序至关重要。
