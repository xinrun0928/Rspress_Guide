# 线程池参数配置与调优

线程池调优是一项艺术，需要平衡多个因素：吞吐量、响应时间、资源消耗和系统稳定性。没有"一刀切"的最佳配置，只有最适合具体场景的配置。

## 调优前的准备工作

在开始调优之前，我们需要了解几个关键信息：

**任务特性分析**：

- CPU 密集型：任务主要是计算，如加密、压缩、算法计算
- I/O 密集型：任务涉及大量等待，如网络请求、文件读写、数据库查询
- 混合型：计算和 I/O 混合

```java
// 判断任务类型
public class TaskAnalyzer {
    public static void main(String[] args) {
        int cpuCores = Runtime.getRuntime().availableProcessors();
        System.out.println("CPU 核心数: " + cpuCores);
    }
}
```

**系统资源评估**：

- 可用 CPU 核心数
- 内存大小
- 最大允许线程数（受限于进程地址空间）

## CPU 密集型任务配置

CPU 密集型任务的特点是持续利用 CPU，很少阻塞。对于这类任务，线程数过多会增加上下文切换开销，线程数过少则无法充分利用 CPU。

**推荐配置公式**：

```
最佳线程数 = CPU 核心数 + 1
```

为什么 +1？因为计算线程偶尔也会发生 Page Fault，需要等待，加一个额外的线程可以利用这段时间。

```java
// CPU 密集型线程池
int cpuCores = Runtime.getRuntime().availableProcessors();
ThreadPoolExecutor cpuBoundPool = new ThreadPoolExecutor(
    cpuCores + 1,
    cpuCores + 1,
    0L, TimeUnit.MILLISECONDS,
    new LinkedBlockingQueue<>(100),
    new ThreadFactoryBuilder().setNameFormat("cpu-pool-%d").build(),
    new ThreadPoolExecutor.AbortPolicy()
);
```

## I/O 密集型任务配置

I/O 密集型任务的特点是大量时间在等待 I/O 完成，CPU 处于空闲状态。对于这类任务，可以配置更多的线程来提高 CPU 利用率。

**推荐配置公式**：

```
最佳线程数 = CPU 核心数 * (1 + I/O 等待时间 / CPU 计算时间)
```

更简单的经验公式：

```
最佳线程数 = CPU 核心数 * 2（或更多）
```

```java
// I/O 密集型线程池
int cpuCores = Runtime.getRuntime().availableProcessors();
ThreadPoolExecutor ioBoundPool = new ThreadPoolExecutor(
    cpuCores * 2,
    cpuCores * 4,
    60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(200),
    new ThreadFactoryBuilder().setNameFormat("io-pool-%d").build(),
    new ThreadPoolExecutor.AbortPolicy()
);
```

## 队列容量配置

队列容量决定了任务堆积的能力。配置过小容易触发拒绝策略，配置过大可能耗尽内存。

**配置原则**：

- 队列大小应该与最大线程数配合
- 考虑任务的最长等待时间和最大堆积数量

```java
// 配置公式
队列容量 = (最大并发时间 / 单任务平均时间) * 最大容忍延迟

// 示例
// 单任务 100ms，期望最大堆积 1000 个任务
// 队列容量 = 1000
LinkedBlockingQueue<Runnable> queue = new LinkedBlockingQueue<>(1000);
```

## 监控与调整

线程池调优是一个持续的过程，需要监控和调整。

```java
// 监控线程池状态
public class ThreadPoolMonitor {
    public static void printStatus(ThreadPoolExecutor executor) {
        System.out.println("线程池状态:");
        System.out.println("  核心线程数: " + executor.getCorePoolSize());
        System.out.println("  最大线程数: " + executor.getMaximumPoolSize());
        System.out.println("  当前线程数: " + executor.getPoolSize());
        System.out.println("  活跃线程数: " + executor.getActiveCount());
        System.out.println("  队列任务数: " + executor.getQueue().size());
        System.out.println("  完成任务数: " + executor.getCompletedTaskCount());
    }
}
```

## 总结

线程池调优需要根据任务特性选择合适的线程数和队列容量。CPU 密集型任务线程数设置为 CPU 核心数 + 1；I/O 密集型任务线程数设置为 CPU 核心数的 2-4 倍。队列容量需要根据任务堆积能力需求来配置。调优是持续的过程，需要监控线程池状态并不断调整。
