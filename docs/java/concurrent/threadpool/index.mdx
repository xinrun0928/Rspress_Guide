# Java 线程池概述

在 Java 并发编程中，线程池可以说是一把"瑞士军刀"——几乎所有需要多线程的场景都能看到它的身影。线程池解决了两个核心问题：一是避免了频繁创建和销毁线程的开销；二是提供了对线程数量的统一管理和控制。没有线程池，每个任务都创建新线程，很快就会耗尽系统资源。

## 为什么需要线程池

直接创建线程（`new Thread()`）的问题显而易见：线程是宝贵的系统资源，创建和销毁线程有显著的开销。更重要的是，无节制地创建线程会导致系统资源耗尽，引发 OOM（OutOfMemoryError）。想象一下，如果每个 HTTP 请求都创建一个新线程来处理，一秒钟内几千个请求就会创建几千个线程，系统很快就崩溃了。

线程池的核心思想是"复用"。预先创建一组工作线程，不断从任务队列中取任务执行，执行完成后不销毁，而是继续取下一个任务。这样就避免了线程创建销毁的开销，也能有效控制并发数量。

## Java 线程池的发展历程

Java 线程池的发展经历了几个重要阶段：

JDK 1.5 引入了 `Executor` 框架，包括 `ExecutorService`、`ThreadPoolExecutor`、`ScheduledExecutorService` 等核心接口和类。这是现代 Java 并发编程的基石。

JDK 7 引入了 `ForkJoinPool`，专门用于分治算法的并行执行，如归并排序、快速排序等。它采用了"工作窃取"算法，让空闲线程帮助其他线程完成任务。

JDK 8 引入了并行流（Parallel Stream），底层使用 `ForkJoinPool.commonPool()`，让集合操作更容易并行化。

## 线程池的核心组件

Java 线程池的核心接口和类包括：

`Executor`：最顶层的接口，只有一个 `execute(Runnable)` 方法。

`ExecutorService`：扩展了 Executor，添加了生命周期管理和返回 Future 的能力。

`ThreadPoolExecutor`：最常用的线程池实现，提供了丰富的配置参数。

`ScheduledExecutorService`：支持定时任务和周期性任务的线程池。

`ForkJoinPool`：专门用于分治算法的线程池，采用工作窃取算法。

## 线程池的工作流程

线程池的核心工作流程是这样的：

1. 提交任务到线程池
2. 如果线程池的工作线程数小于核心线程数，创建新线程处理任务
3. 如果工作线程数已达到核心线程数，任务进入等待队列
4. 如果等待队列已满且工作线程数小于最大线程数，创建新线程处理任务
5. 如果工作线程数已达到最大线程数，执行拒绝策略

理解这个流程对于合理配置线程池参数至关重要。
