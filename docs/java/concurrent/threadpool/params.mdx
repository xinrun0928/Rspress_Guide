# ThreadPoolExecutor 七大参数详解

`ThreadPoolExecutor` 是 Java 并发包中最核心的线程池实现，理解它的七个参数是掌握线程池的关键。这七个参数看似简单，但它们之间的配合却决定了线程池的行为模式和性能特征。深入理解这些参数，能帮助我们根据业务场景配置出最优的线程池。

## 七大参数概览

```java
public class ThreadPoolExecutorParameters {
    
    public void createThreadPool() {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            int corePoolSize,              // 核心线程数
            int maximumPoolSize,           // 最大线程数
            long keepAliveTime,            // 空闲线程存活时间
            TimeUnit unit,                 // keepAliveTime 的时间单位
            BlockingQueue<Runnable> workQueue,  // 任务队列
            ThreadFactory threadFactory,   // 线程工厂
            RejectedExecutionHandler handler  // 拒绝策略
        );
    }
}
```

## 参数一：corePoolSize（核心线程数）

核心线程是线程池中的"常驻人口"，即使它们处于空闲状态也不会被回收。

### 核心线程的特点

```java
public class CorePoolSizeDemo {
    
    private final ExecutorService executor = 
        new ThreadPoolExecutor(
            4,                      // 核心线程数
            10,                     // 最大线程数
            60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(),
            Executors.defaultThreadFactory(),
            new ThreadPoolExecutor.AbortPolicy()
        );
    
    public void demonstrate() {
        // 1. 初始时，没有核心线程在运行
        System.out.println(executor.getPoolSize());  // 0
        
        // 2. 提交任务后，创建核心线程
        executor.execute(() -> {
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        System.out.println(executor.getPoolSize());  // 1
        
        // 3. 核心线程会一直存在
        // 即使没有任务，也不会被回收（默认情况）
    }
}
```

### 核心线程的预热

默认情况下，核心线程不会预先创建，只有在提交任务时才创建。如果需要预创建核心线程：

```java
public class CorePoolPreheating {
    
    private final ThreadPoolExecutor executor = 
        new ThreadPoolExecutor(4, 10, 60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>());
    
    public void preheat() {
        // 预热：让核心线程都创建出来
        executor.prestartAllCoreThreads();
        
        // 或者预热单个核心线程
        executor.prestartCoreThread();
    }
}
```

## 参数二：maximumPoolSize（最大线程数）

最大线程数是线程池的上限，包括核心线程和临时线程。

### 核心线程 vs 临时线程

```java
public class MaxPoolSizeDemo {
    
    private final ThreadPoolExecutor executor = 
        new ThreadPoolExecutor(
            2,                      // 核心线程数：2
            5,                      // 最大线程数：5（包含核心线程）
            60L, TimeUnit.SECONDS,
            new SynchronousQueue<>(),  // 直接提交队列
            Executors.defaultThreadFactory(),
            new ThreadPoolExecutor.AbortPolicy()
        );
    
    public void demonstrate() throws InterruptedException {
        // 提交多个任务，观察线程创建
        
        for (int i = 0; i < 10; i++) {
            final int taskId = i;
            executor.execute(() -> {
                System.out.println("任务" + taskId + 
                    " 由 " + Thread.currentThread().getName() + " 执行");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            });
            
            Thread.sleep(50);  // 让任务有间隔地提交
        }
        
        // 观察输出：
        // 前2个任务：创建2个核心线程
        // 后面的任务：创建临时线程（最多3个，因为最大是5，核心是2）
    }
}
```

## 参数三、四：keepAliveTime 和 unit（空闲线程存活时间）

keepAliveTime 决定了非核心线程可以空闲存活的时间。

### 空闲线程的回收

```java
public class KeepAliveTimeDemo {
    
    private final ThreadPoolExecutor executor = 
        new ThreadPoolExecutor(
            2,                      // 核心线程数
            5,                      // 最大线程数
            5L,                      // keepAliveTime
            TimeUnit.SECONDS,        // 单位：秒
            new SynchronousQueue<>()
        );
    
    public void demonstrate() throws InterruptedException {
        // 提交任务，耗尽所有线程
        for (int i = 0; i < 5; i++) {
            executor.execute(() -> {
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            });
        }
        
        Thread.sleep(200);
        System.out.println("活跃线程数: " + executor.getPoolSize());  // 5
        
        // 等待空闲线程被回收（超过5秒）
        Thread.sleep(6000);
        System.out.println("回收后线程数: " + executor.getPoolSize());  // 2（核心线程）
    }
}
```

### allowCoreThreadTimeOut

默认情况下，核心线程不会被回收。如果希望核心线程也能被回收：

```java
public class CoreThreadTimeout {
    
    private final ThreadPoolExecutor executor = 
        new ThreadPoolExecutor(
            2,                      // 核心线程数
            5,                      // 最大线程数
            5L, TimeUnit.SECONDS,   // 空闲存活时间
            new LinkedBlockingQueue<>()
        );
    
    public void enableCoreTimeout() {
        // 允许核心线程超时回收
        executor.allowCoreThreadTimeOut(true);
    }
}
```

## 参数五：workQueue（任务队列）

任务队列用于存储等待执行的任务。队列的选择直接影响线程池的行为。

### 队列类型对比

```java
public class QueueTypesDemo {
    
    // 1. 无界队列：LinkedBlockingQueue（默认容量 Integer.MAX_VALUE）
    // 风险：任务堆积可能导致内存溢出
    public ExecutorService unboundedQueue() {
        return new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>()  // 无界
        );
    }
    
    // 2. 有界队列：ArrayBlockingQueue 或带容量的 LinkedBlockingQueue
    // 需要配合拒绝策略使用
    public ExecutorService boundedQueue() {
        return new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(100)  // 有界，容量100
        );
    }
    
    // 3. 同步队列：SynchronousQueue
    // 不存储任务，直接提交给线程
    // 必须配合足够的最大线程数
    public ExecutorService synchronousQueue() {
        return new ThreadPoolExecutor(
            2, Integer.MAX_VALUE, 60L, TimeUnit.SECONDS,
            new SynchronousQueue<>()  // 同步队列
        );
    }
    
    // 4. 延迟队列：DelayedWorkQueue
    // 用于 ScheduledThreadPoolExecutor
    public ScheduledExecutorService delayedQueue() {
        return Executors.newScheduledThreadPool(4);
    }
}
```

### 队列选择策略

```java
public class QueueSelection {
    
    // 策略1：CPU密集型 + 有界队列
    // 核心线程满后，任务入队；队满后创建临时线程
    public ExecutorService cpuBoundWithBound() {
        int coreCount = Runtime.getRuntime().availableProcessors();
        return new ThreadPoolExecutor(
            coreCount,
            coreCount + 1,
            60L, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(100)
        );
    }
    
    // 策略2：IO密集型 + 无界队列
    // 适合任务量可控，不会突然激增的场景
    public ExecutorService ioBoundUnbounded() {
        int coreCount = Runtime.getRuntime().availableProcessors();
        return new ThreadPoolExecutor(
            coreCount * 2,
            coreCount * 2,
            60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>()
        );
    }
    
    // 策略3：SynchronousQueue
    // 适合任务量大但执行快的场景
    public ExecutorService synchronousQueueStrategy() {
        return new ThreadPoolExecutor(
            0, Integer.MAX_VALUE,
            60L, TimeUnit.SECONDS,
            new SynchronousQueue<>()
        );
    }
}
```

## 参数六：threadFactory（线程工厂）

线程工厂负责创建新线程，可以自定义线程的命名、优先级、守护状态等。

### 自定义线程工厂

```java
public class CustomThreadFactory {
    
    public static class DaemonThreadFactory implements ThreadFactory {
        
        private final AtomicInteger counter = new AtomicInteger(1);
        
        @Override
        public Thread newThread(Runnable r) {
            Thread thread = new Thread(r);
            thread.setName("Daemon-Worker-" + counter.getAndIncrement());
            thread.setDaemon(true);  // 守护线程
            return thread;
        }
    }
    
    public static class NamedThreadFactory implements ThreadFactory {
        
        private final String prefix;
        private final AtomicInteger counter = new AtomicInteger(1);
        private final ThreadGroup group;
        
        public NamedThreadFactory(String prefix) {
            this.prefix = prefix;
            this.group = new ThreadGroup(prefix + "-group");
        }
        
        @Override
        public Thread newThread(Runnable r) {
            Thread thread = new Thread(group, r, 
                prefix + "-" + counter.getAndIncrement());
            thread.setPriority(Thread.NORM_PRIORITY);
            return thread;
        }
    }
    
    public void useCustomFactory() {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(),
            new DaemonThreadFactory()
        );
    }
}
```

## 参数七：handler（拒绝策略）

当线程池无法接受新任务时，会触发拒绝策略。

### 四种内置拒绝策略

```java
public class RejectionPolicies {
    
    // 1. AbortPolicy（默认）：抛出 RejectedExecutionException
    public void abortPolicy() {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(10),
            new ThreadPoolExecutor.AbortPolicy()
        );
        
        try {
            executor.execute(() -> System.out.println("任务"));
        } catch (RejectedExecutionException e) {
            // 任务被拒绝，可以记录日志、重试等
        }
    }
    
    // 2. CallerRunsPolicy：由调用线程执行被拒绝的任务
    public void callerRunsPolicy() {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(10),
            new ThreadPoolExecutor.CallerRunsPolicy()
        );
        // 当线程池满时，任务由调用线程执行
    }
    
    // 3. DiscardPolicy：直接丢弃被拒绝的任务
    public void discardPolicy() {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(10),
            new ThreadPoolExecutor.DiscardPolicy()
        );
        // 任务被静默丢弃，不抛出异常
    }
    
    // 4. DiscardOldestPolicy：丢弃最老的任务
    public void discardOldestPolicy() {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(10),
            new ThreadPoolExecutor.DiscardOldestPolicy()
        );
        // 丢弃队列中最老的任务，然后重新提交当前任务
    }
    
    // 5. 自定义拒绝策略
    public void customPolicy() {
        RejectedExecutionHandler customHandler = (r, executor) -> {
            // 记录日志
            System.err.println("任务被拒绝: " + r);
            
            // 重试逻辑
            if (!executor.isShutdown()) {
                executor.execute(r);
            }
        };
        
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(10),
            customHandler
        );
    }
}
```

### 常用配置公式

```java
public class CommonConfigurations {
    
    // CPU密集型任务
    // 线程数 = CPU核心数 + 1
    public ExecutorService forCpuBound() {
        int coreCount = Runtime.getRuntime().availableProcessors();
        return new ThreadPoolExecutor(
            coreCount + 1,
            coreCount + 1,
            0L, TimeUnit.MILLISECONDS,
            new LinkedBlockingQueue<>(),
            new ThreadFactoryBuilder().setNameFormat("cpu-pool-%d").build()
        );
    }
    
    // IO密集型任务
    // 线程数 = CPU核心数 * 2（或更多，取决于IO阻塞比例）
    public ExecutorService forIoBound() {
        int coreCount = Runtime.getRuntime().availableProcessors();
        return new ThreadPoolExecutor(
            coreCount * 2,
            coreCount * 2,
            60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(100),
            new ThreadFactoryBuilder().setNameFormat("io-pool-%d").build()
        );
    }
    
    // 混合型任务
    // 线程数 = CPU核心数 * (1 + 阻塞时间/执行时间)
    public ExecutorService forMixed(
            int coreCount, 
            double blockingRatio) {
        
        int threads = (int) (coreCount * (1 + blockingRatio));
        return new ThreadPoolExecutor(
            threads,
            threads * 2,
            60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(100),
            new ThreadFactoryBuilder().setNameFormat("mixed-pool-%d").build()
        );
    }
}
```

## 总结

`ThreadPoolExecutor` 的七大参数：

1. **corePoolSize**：核心线程数，常驻线程
2. **maximumPoolSize**：最大线程数，上限
3. **keepAliveTime**：临时线程的空闲存活时间
4. **unit**：时间单位
5. **workQueue**：任务队列，选择很重要
6. **threadFactory**：线程工厂，自定义线程行为
7. **handler**：拒绝策略，处理无法执行的任务

理解这些参数的配合关系，才能配置出适合业务的线程池。
