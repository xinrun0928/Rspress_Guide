# 为什么要用线程池？

在 Java 并发编程中，线程是一种昂贵的资源。每创建一个线程，JVM 需要分配栈内存、操作系统需要调度CPU时间片、频繁的上下文切换更会带来巨大的性能开销。如果每次执行任务都创建新线程，程序很快就会崩溃。线程池正是解决这个问题的利器——它复用线程，减少创建销毁的开销，同时提供可控的并发度。

## 不用线程池的问题

### 1. 线程创建销毁开销大

```java
public class WithoutThreadPool {
    
    public void processTasks(List<Task> tasks) {
        for (Task task : tasks) {
            // 每个任务创建一个线程
            new Thread(() -> task.execute()).start();
        }
    }
}
```

线程的创建和销毁涉及：
- 调用 native 方法创建系统线程
- 分配线程栈内存（默认 1MB）
- JVM 注册线程到线程管理结构
- OS 调度CPU时间片

对于大量短任务，这开销是灾难性的。

### 2. 线程数量不可控

```java
public class UncontrolledThreads {
    
    // 如果有一百万个任务
    public void disaster(List<Task> tasks) {
        for (Task task : tasks) {
            // 一百万个线程同时运行
            new Thread(task::execute).start();
        }
    }
}
```

无限制创建线程会导致：
- 内存耗尽（每个线程1MB栈空间）
- 系统资源耗尽
- 上下文切换开销巨大
- 程序响应变慢甚至崩溃

### 3. 无法复用线程

```java
public class NoReuse {
    
    public void badPattern() {
        // 每次执行都创建新线程
        for (int i = 0; i < 1000; i++) {
            int id = i;
            new Thread(() -> {
                // 每个线程只执行一次
                process(id);
            }).start();
        }
    }
}
```

线程执行完任务后就销毁，无法复用。

## 线程池的优势

### 1. 复用线程，减少开销

```java
public class WithThreadPool {
    
    private final ExecutorService executor = 
        Executors.newFixedThreadPool(4);
    
    public void processTasks(List<Task> tasks) {
        for (Task task : tasks) {
            // 提交任务，线程池复用已有线程
            executor.submit(task::execute);
        }
    }
}
```

线程池中的工作线程会被重复使用，执行多个任务。

### 2. 控制并发度

```java
public class ControlledConcurrency {
    
    // 固定4个线程，同时最多4个任务并发
    private final ExecutorService executor = 
        Executors.newFixedThreadPool(4);
    
    // 任何时候最多只有4个任务在执行
    public void submitTasks(List<Task> tasks) {
        for (Task task : tasks) {
            executor.submit(task::execute);
        }
    }
}
```

线程池可以限制同时执行的线程数量。

### 3. 提供线程管理能力

```java
public class ThreadPoolManagement {
    
    private final ThreadPoolExecutor executor = 
        new ThreadPoolExecutor(
            4,                      // corePoolSize
            8,                      // maximumPoolSize
            60L, TimeUnit.SECONDS,  // 空闲线程存活时间
            new LinkedBlockingQueue<>(100)  // 任务队列
        );
    
    // 查看线程池状态
    public void monitor() {
        System.out.println("活跃线程数: " + executor.getActiveCount());
        System.out.println("队列任务数: " + executor.getQueue().size());
        System.out.println("已完成任务数: " + executor.getCompletedTaskCount());
    }
    
    // 优雅关闭
    public void shutdown() {
        executor.shutdown();  // 不再接受新任务，等待已提交任务完成
        // executor.shutdownNow();  // 尝试停止所有正在执行的任务
    }
}
```

### 4. 支持多种任务执行策略

```java
public class TaskStrategies {
    
    // 固定大小线程池
    public ExecutorService fixedThreadPool() {
        return Executors.newFixedThreadPool(4);
    }
    
    // 缓存线程池（按需创建）
    public ExecutorService cachedThreadPool() {
        return Executors.newCachedThreadPool();
    }
    
    // 单线程池（保证顺序执行）
    public ExecutorService singleThreadPool() {
        return Executors.newSingleThreadExecutor();
    }
    
    // 定时任务线程池
    public ScheduledExecutorService scheduledPool() {
        return Executors.newScheduledThreadPool(4);
    }
}
```

## 线程池的工作原理

```
┌────────────────────────────────────────────────────────────────┐
│                      线程池工作原理                               │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐    │
│   │                      线程池                             │    │
│   │  ┌─────────┐  ┌─────────┐  ┌─────────┐               │    │
│   │  │Thread-1 │  │Thread-2 │  │Thread-3 │  ...         │    │
│   │  └────┬────┘  └────┬────┘  └────┬────┘               │    │
│   │       │            │            │                      │    │
│   └───────┼────────────┼────────────┼───────────────────────┘    │
│           │            │            │                            │
│           ▼            ▼            ▼                            │
│   ┌─────────────────────────────────────────────────────────┐    │
│   │                    任务队列                              │    │
│   │   [task5] [task6] [task7] [task8] [task9] ...          │    │
│   └─────────────────────────────────────────────────────────┘    │
│                          ▲                                      │
│                          │                                      │
│                   submit(task)                                  │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

线程池的工作流程：

1. **提交任务**：调用 `execute()` 或 `submit()` 提交任务
2. **判断核心线程**：如果核心线程未满，创建新核心线程执行
3. **加入队列**：核心线程满，将任务加入队列等待
4. **创建临时线程**：队列满且未达最大线程数，创建临时线程
5. **执行拒绝策略**：超过最大线程数且队列满，执行拒绝策略

## 线程池的核心参数

```java
public class CoreParameters {
    
    public ThreadPoolExecutor create() {
        return new ThreadPoolExecutor(
            2,                      // corePoolSize：核心线程数
            4,                      // maximumPoolSize：最大线程数
            60L,                    // keepAliveTime：空闲线程存活时间
            TimeUnit.SECONDS,       // 时间单位
            new LinkedBlockingQueue<>(100),  // 任务队列
            Executors.defaultThreadFactory(),  // 线程工厂
            new ThreadPoolExecutor.AbortPolicy()  // 拒绝策略
        );
    }
}
```

- **corePoolSize**：核心线程数，即使空闲也不会被回收
- **maximumPoolSize**：最大线程数，包括核心线程和临时线程
- **keepAliveTime**：临时线程的空闲存活时间
- **workQueue**：任务队列，存储等待执行的任务
- **threadFactory**：线程工厂，用于创建新线程
- **handler**：拒绝策略，当任务无法执行时的处理方式

## 线程池的适用场景

### 场景1：Web服务器处理请求

```java
public class WebServer {
    
    private final ExecutorService executor = 
        new ThreadPoolExecutor(
            10,     // 10个核心线程
            200,    // 最多200个线程
            60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(1000),
            new ThreadFactoryBuilder()
                .setNameFormat("http-handler-%d")
                .build()
        );
    
    public void handleRequest(HttpRequest request) {
        executor.submit(() -> {
            // 处理HTTP请求
            return process(request);
        });
    }
}
```

### 场景2：异步任务处理

```java
public class AsyncProcessor {
    
    private final ExecutorService executor = 
        Executors.newFixedThreadPool(Runtime.getRuntime().availableProcessors());
    
    public CompletableFuture<Result> processAsync(Data data) {
        return CompletableFuture.supplyAsync(() -> {
            return heavyComputation(data);
        }, executor);
    }
}
```

### 场景3：定时任务

```java
public class ScheduledTasks {
    
    private final ScheduledExecutorService scheduler = 
        Executors.newScheduledThreadPool(4);
    
    public void scheduleTasks() {
        // 延迟执行
        scheduler.schedule(() -> {
            System.out.println("延迟1秒后执行");
        }, 1, TimeUnit.SECONDS);
        
        // 固定频率执行
        scheduler.scheduleAtFixedRate(() -> {
            System.out.println("每3秒执行一次");
        }, 1, 3, TimeUnit.SECONDS);
        
        // 固定延迟执行
        scheduler.scheduleWithFixedDelay(() -> {
            System.out.println("每次执行后延迟2秒");
        }, 1, 2, TimeUnit.SECONDS);
    }
}
```

## 线程池的注意事项

### 1. 合理设置线程数

```java
public class ThreadCountGuidelines {
    
    // CPU密集型：线程数 = CPU核心数 + 1
    public ExecutorService cpuIntensive() {
        int coreCount = Runtime.getRuntime().availableProcessors();
        return Executors.newFixedThreadPool(coreCount + 1);
    }
    
    // IO密集型：线程数 = CPU核心数 * 2 或更多
    public ExecutorService ioIntensive() {
        int coreCount = Runtime.getRuntime().availableProcessors();
        return Executors.newFixedThreadPool(coreCount * 2);
    }
    
    // 混合型：根据实际情况调整
    public ExecutorService mixed() {
        int coreCount = Runtime.getRuntime().availableProcessors();
        int threads = (int) (coreCount * (1 + 0.5));  // 预估比例
        return Executors.newFixedThreadPool(threads);
    }
}
```

### 2. 正确关闭线程池

```java
public class ProperShutdown {
    
    private final ExecutorService executor = 
        Executors.newFixedThreadPool(4);
    
    public void gracefulShutdown() throws InterruptedException {
        // 禁止提交新任务
        executor.shutdown();
        
        // 等待已提交任务完成（最多等待1分钟）
        if (!executor.awaitTermination(1, TimeUnit.MINUTES)) {
            // 超时后强制停止
            executor.shutdownNow();
        }
    }
    
    public void immediateShutdown() {
        // 立即停止所有任务
        executor.shutdownNow();
    }
}
```

### 3. 避免任务丢失

```java
public class TaskSafety {
    
    private final ThreadPoolExecutor executor = 
        new ThreadPoolExecutor(4, 8, 60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(100),
            new ThreadPoolExecutor.CallerRunsPolicy());  // 用调用线程执行
    
    public void submitSafely(Runnable task) {
        try {
            executor.execute(task);
        } catch (RejectedExecutionException e) {
            // 拒绝策略触发时的处理
            // CallerRunsPolicy 会用调用线程执行
            task.run();
        }
    }
}
```

## 总结

使用线程池的原因：

1. **复用线程**：减少创建销毁开销
2. **控制并发**：限制同时执行的线程数
3. **管理能力**：提供监控、关闭、拒绝策略等
4. **异步执行**：提高程序响应性

核心参数：
- `corePoolSize`：核心线程数
- `maximumPoolSize`：最大线程数
- `workQueue`：任务队列
- `handler`：拒绝策略
