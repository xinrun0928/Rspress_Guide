# 4 种常用线程池：Fixed、Cached、Single、Scheduled

你写了一个 HTTP 服务器，每收到一个请求就 `new Thread(() -> { handle(); }).start()`。

系统上线第一天，100 个并发请求，服务器稳如老狗。

第三天，10000 个并发请求，服务器原地升天。

问题出在哪？**线程不是越多越好，无限制创建线程是要付出代价的。**

今天，我们来聊聊线程池，特别是 JDK 提供的 4 种常用线程池。

## 为什么需要线程池？

直接创建线程的问题：

```
┌─────────────────────────────────────────────────────────┐
│           频繁创建线程的代价                              │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  每次 new Thread()：                                     │
│  1. 分配线程栈内存（通常 1MB）                          │
│  2. 向 OS 注册线程                                       │
│  3. 创建线程内核对象                                      │
│  4. 分配 PID                                              │
│                                                          │
│  线程创建耗时：约 1-2ms                                  │
│  线程销毁耗时：约 0.5-1ms                                │
│                                                          │
│  如果每秒创建 1000 个线程：                              │
│  - 30% 的时间花在创建/销毁线程上                        │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

线程池的核心思想：**预先创建一组线程，用完了放回池里，需要时再取出来**。

## Executor 框架

JDK 的线程池体系以 `Executor` 为起点：

```java
public interface Executor {
    void execute(Runnable command);
}
```

层层递进：

```
Executor
  └── ExecutorService
        └── AbstractExecutorService
              └── ThreadPoolExecutor  ← 核心实现

ScheduledExecutorService  ← 定时执行
  └── ScheduledThreadPoolExecutor
```

## 4 种常用线程池

### 1. FixedThreadPool：固定大小线程池

```java
// 固定 10 个线程的线程池
ExecutorService executor = Executors.newFixedThreadPool(10);

for (int i = 0; i < 100; i++) {
    final int taskId = i;
    executor.submit(() -> {
        System.out.println("Task " + taskId + " running in " + Thread.currentThread().getName());
    });
}

executor.shutdown();
```

**特点**：

- 核心线程数 = 最大线程数 = n
- 线程池创建后，线程数固定不变
- 所有线程都在运行，不会被回收
- 任务队列无界，可能导致 OOM

**适用场景**：

- CPU 密集型任务
- 需要限制并发数的场景

### 2. CachedThreadPool：缓存线程池

```java
// 可缓存的线程池
ExecutorService executor = Executors.newCachedThreadPool();

for (int i = 0; i < 100; i++) {
    final int taskId = i;
    executor.submit(() -> {
        System.out.println("Task " + taskId + " running in " + Thread.currentThread().getName());
    });
}

executor.shutdown();
```

**特点**：

- 核心线程数 = 0
- 最大线程数 = `Integer.MAX_VALUE`（几乎无限）
- 线程空闲 60 秒后回收
- 任务提交时，有空余线程就用，没有就创建新的

**适用场景**：

- 短命任务
- 突发性任务
- **警告**：慎用！最大线程数无限制，可能导致 OOM

### 3. SingleThreadExecutor：单线程池

```java
// 单线程的线程池
ExecutorService executor = Executors.newSingleThreadExecutor();

for (int i = 0; i < 100; i++) {
    final int taskId = i;
    executor.submit(() -> {
        System.out.println("Task " + taskId + " running in " + Thread.currentThread().getName());
    });
}

executor.shutdown();
```

**特点**：

- 核心线程数 = 最大线程数 = 1
- 只有一个线程在执行任务
- 保证任务串行执行
- 线程异常退出后，会创建新线程补充

**适用场景**：

- 需要保证顺序执行的场景
- 日志写入
- 消息队列消费

### 4. ScheduledThreadPool：定时线程池

```java
// 定时线程池
ScheduledExecutorService executor = Executors.newScheduledThreadPool(5);

// 延迟 3 秒后执行一次
executor.schedule(() -> {
    System.out.println("Delayed task");
}, 3, TimeUnit.SECONDS);

// 延迟 1 秒后开始执行，之后每 2 秒执行一次
executor.scheduleAtFixedRate(() -> {
    System.out.println("Fixed rate task");
}, 1, 2, TimeUnit.SECONDS);

// 上次执行完成后，延迟 1 秒再执行
executor.scheduleWithFixedDelay(() -> {
    System.out.println("Fixed delay task");
}, 1, 2, TimeUnit.SECONDS);

executor.shutdown();
```

**特点**：

- 核心线程数 = 指定数量
- 最大线程数 = `Integer.MAX_VALUE`
- 支持定时和周期性任务

**适用场景**：

- 定时任务调度
- 心跳检测
- 延迟处理

## 线程池参数

`ThreadPoolExecutor` 的核心参数：

```java
public ThreadPoolExecutor(
    int corePoolSize,      // 核心线程数
    int maximumPoolSize,   // 最大线程数
    long keepAliveTime,    // 空闲线程存活时间
    TimeUnit unit,         // 时间单位
    BlockingQueue<Runnable> workQueue,  // 任务队列
    ThreadFactory threadFactory,         // 线程工厂
    RejectedExecutionHandler handler     // 拒绝策略
)
```

### 线程池的执行流程

```
┌─────────────────────────────────────────────────────────┐
│           线程池执行流程                                  │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  提交任务 ───────────────────────────────────────────┐  │
│     │                                                │  │
│     ▼                                                │  │
│  线程数 < 核心线程数？                                │  │
│     ├── 是 → 创建核心线程执行                        │  │
│     │    │                                           │  │
│     │    ◄──────────────────────────┘               │  │
│     │                                                │  │
│     ▼ 否                                              │  │
│  队列未满？                                           │  │
│     ├── 是 → 加入任务队列                            │  │
│     │    │                                           │  │
│     │    ◄──────────────────────────┘               │  │
│     │                                                │  │
│     ▼ 否                                              │  │
│  线程数 < 最大线程数？                               │  │
│     ├── 是 → 创建临时线程执行                        │  │
│     │    │                                           │  │
│     │    ◄──────────────────────────┘               │  │
│     │                                                │  │
│     ▼ 否                                              │  │
│  执行拒绝策略                                         │  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 四种拒绝策略

```java
// 1. AbortPolicy（默认）：抛异常
new ThreadPoolExecutor.AbortPolicy();

// 2. CallerRunsPolicy：由调用线程执行
new ThreadPoolExecutor.CallerRunsPolicy();

// 3. DiscardPolicy：静默丢弃
new ThreadPoolExecutor.DiscardPolicy();

// 4. DiscardOldestPolicy：丢弃队列中最老的任务
new ThreadPoolExecutor.DiscardOldestPolicy();
```

## 手写一个自定义线程池

```java
public class CustomThreadPoolDemo {
    public static void main(String[] args) {
        // 核心 5 个线程，最大 10 个，空闲 60 秒后回收
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            5,                          // corePoolSize
            10,                         // maximumPoolSize
            60L,                        // keepAliveTime
            TimeUnit.SECONDS,           // unit
            new LinkedBlockingQueue<>(100),  // workQueue
            new ThreadFactory() {       // threadFactory
                private AtomicInteger counter = new AtomicInteger(1);
                @Override
                public Thread newThread(Runnable r) {
                    Thread t = new Thread(r);
                    t.setName("MyPool-" + counter.getAndIncrement());
                    t.setDaemon(false);
                    return t;
                }
            },
            new ThreadPoolExecutor.CallerRunsPolicy()  // handler
        );

        // 提交任务
        for (int i = 0; i < 150; i++) {
            final int taskId = i;
            executor.submit(() -> {
                System.out.println("Task " + taskId + " executed by " +
                    Thread.currentThread().getName());
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            });
        }

        executor.shutdown();
    }
}
```

## 线程池的陷阱

### 陷阱一：Executors 创建的线程池可能有 OOM 风险

```java
// 危险！
ExecutorService executor = Executors.newCachedThreadPool();
// 最大线程数 Integer.MAX_VALUE，可能导致 OOM

ExecutorService executor = Executors.newFixedThreadPool(100);
// 无界队列，可能导致 OOM
```

**正确做法**：使用 `new ThreadPoolExecutor(...)` 手动指定参数。

### 陷阱二：线程池中的线程不一定会被回收

`allowCoreThreadTimeOut(true)` 允许回收核心线程：

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(5, 10, 60L,
    TimeUnit.SECONDS, new LinkedBlockingQueue<>());

// 允许回收核心线程
executor.allowCoreThreadTimeOut(true);
```

### 陷阱三：shutdown() 和 shutdownNow() 的区别

```java
// shutdown()：拒绝新任务，等待已有任务执行完毕
executor.shutdown();
boolean finished = executor.awaitTermination(10, TimeUnit.SECONDS);

// shutdownNow()：尝试停止所有正在执行的任务
List<Runnable> tasks = executor.shutdownNow();
```

## 最佳实践

### 配置线程数

```java
// CPU 密集型：线程数 = CPU 核心数 + 1
int cpuCores = Runtime.getRuntime().availableProcessors();
ExecutorService cpuPool = new ThreadPoolExecutor(
    cpuCores + 1, cpuCores + 1, 0L, TimeUnit.MILLISECONDS,
    new LinkedBlockingQueue<>()
);

// IO 密集型：线程数 = CPU 核心数 * 2（或根据 IO 阻塞比例调整）
int ioThreads = cpuCores * 2;
ExecutorService ioPool = new ThreadPoolExecutor(
    ioThreads, ioThreads, 0L, TimeUnit.MILLISECONDS,
    new LinkedBlockingQueue<>()
);
```

### 监控线程池

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(5, 10, 60L,
    TimeUnit.SECONDS, new LinkedBlockingQueue<>());

// 监控指标
public void monitor(ThreadPoolExecutor executor) {
    System.out.println("Active threads: " + executor.getActiveCount());
    System.out.println("Pool size: " + executor.getPoolSize());
    System.out.println("Queue size: " + executor.getQueue().size());
    System.out.println("Completed tasks: " + executor.getCompletedTaskCount());
    System.out.println("Total tasks: " + executor.getTaskCount());
}
```

## 总结

| 线程池 | 核心线程 | 最大线程 | 队列 | 适用场景 |
|---|---|---|---|---|
| FixedThreadPool | n | n | 无界 | CPU 密集、固定并发 |
| CachedThreadPool | 0 | 无穷 | 同步队列 | 短命任务、突发任务 |
| SingleThreadExecutor | 1 | 1 | 无界 | 顺序执行、保证因果 |
| ScheduledThreadPool | n | 无穷 | 无界 | 定时任务、周期任务 |

**关键原则**：

1. 不要用 `Executors` 创建线程池，用 `new ThreadPoolExecutor(...)`
2. 根据任务类型（CPU 密集/IO 密集）配置线程数
3. 监控线程池状态，防止 OOM
4. 记得 shutdown，避免资源泄漏
