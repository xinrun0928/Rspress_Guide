# 四种常用线程池：Fixed、Cached、Single、Scheduled

JDK 提供了几种常用的线程池工厂方法，通过 `Executors` 类可以方便地创建。这些预定义的线程池适用于不同的场景，理解它们的特点能帮助我们快速选择合适的线程池。

## FixedThreadPool（固定大小线程池）

`FixedThreadPool` 创建一个固定线程数量的线程池。

```java
ExecutorService executor = Executors.newFixedThreadPool(4);
```

**特点**：

- 核心线程数和最大线程数相同，都是指定的线程数
- 没有超时机制，线程不会被回收
- 使用无界队列 LinkedBlockingQueue
- 适合线程数量固定、需要控制并发数的场景

**源码分析**：

```java
public static ExecutorService newFixedThreadPool(int nThreads) {
    return new ThreadPoolExecutor(
        nThreads,          // corePoolSize
        nThreads,          // maximumPoolSize
        0L,                // keepAliveTime（不生效）
        TimeUnit.MILLISECONDS,
        new LinkedBlockingQueue<Runnable>()  // 无界队列
    );
}
```

**适用场景**：适合执行长期任务、需要控制并发数的场景。比如处理固定数量的并发请求、数据库连接池等。

**潜在问题**：由于使用无界队列，如果任务提交速度持续大于处理速度，可能导致队列无限增长，耗尽内存。

## CachedThreadPool（缓存线程池）

`CachedThreadPool` 创建一个可缓存的线程池，线程数是弹性的。

```java
ExecutorService executor = Executors.newCachedThreadPool();
```

**特点**：

- 核心线程数为 0
- 最大线程数为 Integer.MAX_VALUE（实际可创建数量受限于系统资源）
- 线程有 60 秒超时机制
- 使用 SynchronousQueue（不存储元素）

**源码分析**：

```java
public static ExecutorService newCachedThreadPool() {
    return new ThreadPoolExecutor(
        0,                        // corePoolSize = 0
        Integer.MAX_VALUE,        // maximumPoolSize
        60L, TimeUnit.SECONDS,    // 60秒超时
        new SynchronousQueue<Runnable>()  // 直接交接
    );
}
```

**适用场景**：适合执行大量短期异步任务、任务提交频率不固定的场景。线程池会根据任务数量自动调整。

**潜在问题**：

- 由于最大线程数无限制，可能创建过多线程，耗尽系统资源
- SynchronousQueue 不存储任务，如果线程池空闲超过 60 秒，线程会被回收
- 不适合长期运行的任务

## SingleThreadExecutor（单线程池）

`SingleThreadExecutor` 创建一个单线程的线程池。

```java
ExecutorService executor = Executors.newSingleThreadExecutor();
```

**特点**：

- 核心线程数和最大线程数都是 1
- 使用无界队列 LinkedBlockingQueue
- 任务按顺序执行

**源码分析**：

```java
public static ExecutorService newSingleThreadExecutor() {
    return new FinalizableDelegatedExecutorService(
        new ThreadPoolExecutor(
            1, 1, 0L, TimeUnit.MILLISECONDS,
            new LinkedBlockingQueue<Runnable>()
        )
    );
}
```

**适用场景**：适合需要保证任务顺序执行的场景。比如日志写入、消息队列消费等。

**潜在问题**：单点故障——如果这个唯一的线程异常终止，线程池会创建一个新的线程继续执行。

## ScheduledThreadPool（定时线程池）

`ScheduledThreadPool` 创建一个定时执行的线程池。

```java
ScheduledExecutorService executor = Executors.newScheduledThreadPool(4);
```

**特点**：

- 核心线程数由指定值决定
- 最大线程数为 Integer.MAX_VALUE
- 使用 DelayedWorkQueue（延迟队列）

**源码分析**：

```java
public static ScheduledExecutorService newScheduledThreadPool(int corePoolSize) {
    return new ScheduledThreadPoolExecutor(corePoolSize);
}

public ScheduledThreadPoolExecutor(int corePoolSize) {
    super(corePoolSize, Integer.MAX_VALUE, 0, NANOSECONDS,
          new DelayedWorkQueue());
}
```

**使用示例**：

```java
ScheduledExecutorService executor = Executors.newScheduledThreadPool(4);

// 延迟执行（3秒后执行一次）
executor.schedule(() -> System.out.println("Delayed"), 3, TimeUnit.SECONDS);

// 固定频率执行（上一个任务开始后2秒执行下一个）
executor.scheduleAtFixedRate(() -> System.out.println("Fixed Rate"),
    1, 2, TimeUnit.SECONDS);

// 固定延迟执行（上一个任务结束后2秒执行下一个）
executor.scheduleWithFixedDelay(() -> System.out.println("Fixed Delay"),
    1, 2, TimeUnit.SECONDS);
```

**适用场景**：适合定时任务、周期性任务的场景。比如定时数据同步、定时报表生成等。

## SingleThreadScheduledExecutor（单定时线程池）

```java
ScheduledExecutorService executor = Executors.newSingleThreadScheduledExecutor();
```

类似 SingleThreadExecutor，但支持定时执行。

## 线程池的常见问题

虽然 Executors 提供了便捷的创建方法，但阿里巴巴开发规范明确指出：不推荐使用 Executors 创建线程池。原因在于：

**FixedThreadPool 和 SingleThreadExecutor**：使用无界队列，可能导致 OOM。

**CachedThreadPool 和 ScheduledThreadPool**：最大线程数为 Integer.MAX_VALUE，可能创建过多线程。

**推荐做法**：使用 `new ThreadPoolExecutor` 明确指定参数：

```java
// 推荐写法
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    10,                                     // corePoolSize
    20,                                     // maximumPoolSize
    60L, TimeUnit.SECONDS,                  // keepAliveTime
    new LinkedBlockingQueue<>(100),         // 有界队列
    new ThreadFactoryBuilder().build(),     // 自定义线程工厂
    new ThreadPoolExecutor.AbortPolicy()    // 拒绝策略
);
```

## 总结

四种常用线程池各有特点：FixedThreadPool 适合固定并发数；CachedThreadPool 适合短期异步任务；SingleThreadExecutor 适合顺序执行；ScheduledThreadPool 适合定时任务。

虽然 Executors 工厂方法使用方便，但存在潜在风险。在生产环境中，推荐使用 `new ThreadPoolExecutor` 明确配置参数，避免 OOM 和线程数失控的问题。
