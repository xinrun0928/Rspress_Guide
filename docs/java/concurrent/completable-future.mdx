# CompletableFuture：异步编程的瑞士军刀

候选人小李在面试中遇到了这样一个问题：

面试官问："你知道CompletableFuture吗？和Future有什么区别？"

小李说："CompletableFuture是JDK 8引入的异步编程工具，比Future更强大。"

面试官追问："那你说说CompletableFuture是怎么实现异步的？thenApply和thenCompose有什么区别？"

小张停顿了半天，说不出区别。

【面试官心理】
CompletableFuture是JDK 8引入的最重要的并发工具之一。80%的候选人知道它"可以异步执行"，但只有30%能说清楚它和Future的区别，只有10%能区分thenApply、thenCompose、thenCombine的用法。CompletableFuture的链式API是函数式编程在Java并发中的最佳实践，能正确使用它的候选人，在异步编程上已经有了实战经验。

## 一、为什么需要CompletableFuture：Future的局限 🔴

### 1.1 Future的局限性

```java
// Future的典型用法
Future<String> future = executor.submit(() -> {
    return callRemoteService();
});

String result = future.get(); // 阻塞等待！
// 问题一：get()是阻塞的
// 问题二：没有回调通知机制
// 问题三：不能链式组合多个Future
// 问题四：不能手动完成
```

**Future的三大局限**：
1. **无法链式调用**：多个异步任务串行执行时，必须嵌套
2. **无法组合多个Future**：无法等待多个Future全部完成
3. **无法手动完成**：无法在任意时刻触发完成

### 1.2 CompletableFuture的解决方案

```java
// CompletableFuture：非阻塞 + 链式调用 + 组合 + 手动完成
CompletableFuture.supplyAsync(() -> callRemoteService())
    .thenApply(String::toUpperCase)       // 链式转换
    .thenAccept(result -> System.out.println(result)) // 消费
    .exceptionally(ex -> {                // 异常处理
        log.error("发生错误", ex);
        return null;
    });
```

## 二、CompletableFuture的执行模型 🔴

### 2.1 异步执行的两大模式

```java
// 模式一：supplyAsync / runAsync（使用默认线程池 ForkJoinPool.commonPool）
CompletableFuture<String> future1 =
    CompletableFuture.supplyAsync(() -> "Hello"); // 使用公共线程池

CompletableFuture<Void> future2 =
    CompletableFuture.runAsync(() -> System.out.println("Hi")); // 公共线程池

// 模式二：指定线程池
ExecutorService executor = Executors.newFixedThreadPool(10);
CompletableFuture<String> future3 =
    CompletableFuture.supplyAsync(() -> callService(), executor);
```

### 2.2 默认线程池：ForkJoinPool.commonPool

JDK 8引入了ForkJoinPool.commonPool作为默认的异步执行池：

```java
// ForkJoinPool.commonPool的配置（可调整）
// 默认线程数 = CPU核心数 - 1
// 对于IO密集型任务，这是次优配置！

// 建议：IO密集型任务使用自定义线程池
ExecutorService ioExecutor = new ThreadPoolExecutor(
    corePoolSize, maxPoolSize,
    60L, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(1000)
);

CompletableFuture.supplyAsync(() -> {
    return fetchFromDB(); // IO密集型
}, ioExecutor); // 使用IO线程池！
```

### 2.3 线程切换：thenApply vs thenApplyAsync

```java
CompletableFuture<String> future =
    CompletableFuture.supplyAsync(() -> "hello", executor);

// thenApplyAsync：切换线程池
future.thenApplyAsync(s -> {
    // 默认使用ForkJoinPool.commonPool
    return s.toUpperCase();
});

// thenApply：沿用上一个线程
future.thenApply(s -> {
    // 继续使用executor
    return s.toUpperCase();
});
```

## 三、核心API详解 🔴

### 3.1 转换：thenApply vs thenApplyAsync

```java
// thenApply：同步转换
CompletableFuture<Integer> future =
    CompletableFuture.supplyAsync(() -> "123")
        .thenApply(Integer::parseInt); // 在同一线程执行

// thenApplyAsync：异步转换
CompletableFuture<Integer> future =
    CompletableFuture.supplyAsync(() -> "123")
        .thenApplyAsync(Integer::parseInt); // 可能切换线程
```

### 3.2 组合：thenCompose vs thenCombine

这是最容易被混淆的两个API：

```java
// thenCompose：扁平化（用于处理返回CompletableFuture的操作）
// 场景：先查用户ID，再查用户详情（两个异步操作串行）

// ❌ 错误：thenApply会产生嵌套Future
CompletableFuture<CompletableFuture<UserDetail>> nestedFuture =
    CompletableFuture.supplyAsync(() -> getUserId())
        .thenApply(userId -> getUserDetailAsync(userId)); // 返回CompletableFuture！

// ✅ 正确：thenCompose扁平化
CompletableFuture<UserDetail> future =
    CompletableFuture.supplyAsync(() -> getUserId())
        .thenCompose(userId -> getUserDetailAsync(userId)); // 自动扁平化
```

```java
// thenCombine：合并两个独立的CompletableFuture
// 场景：同时查询用户信息和商品信息，然后组合

CompletableFuture<User> userFuture =
    CompletableFuture.supplyAsync(() -> getUser());
CompletableFuture<Product> productFuture =
    CompletableFuture.supplyAsync(() -> getProduct());

CompletableFuture<Order> orderFuture =
    userFuture.thenCombine(productFuture, (user, product) ->
        new Order(user, product)  // 两个都完成后执行
    );
```

### 3.3 thenApply vs thenCompose 对比

| API | 作用 | 返回类型 | 使用场景 |
|-----|------|---------|---------|
| thenApply | 转换 | `CompletableFuture<R>` | 同步转换 |
| thenCompose | 扁平化组合 | `CompletableFuture<R>` | 异步操作串行 |
| thenCombine | 并行合并 | `CompletableFuture<R>` | 异步操作并行 |

### 3.4 消费：thenAccept

```java
// thenAccept：最终消费，无返回值
CompletableFuture.supplyAsync(() -> "Hello")
    .thenAccept(result -> System.out.println("Result: " + result));
// Result: Hello

// thenRun：不关心结果，只关心完成
CompletableFuture.supplyAsync(() -> "Hello")
    .thenRun(() -> System.out.println("完成")); // 不传递结果
```

### 3.5 异常处理：exceptionally vs handle

```java
CompletableFuture.supplyAsync(() -> {
    if (true) throw new RuntimeException("Oops!");
    return "success";
})
// exceptionally：捕获异常，返回替代值
.thenApply(result -> result.toUpperCase())
.exceptionally(ex -> {
    log.error("发生异常", ex);
    return "DEFAULT"; // 返回默认值
});

// handle：无论正常还是异常都会执行
.thenApply(result -> {
    return result.toUpperCase();
})
.handle((result, ex) -> {
    if (ex != null) {
        log.error("异常", ex);
        return "DEFAULT";
    }
    return result;
});
```

## 四、实战场景：全链路异步化 🟡

### 4.1 并行查询多个服务

```java
// 场景：用户下单前，需要同时查询用户信息、库存信息、价格信息

long start = System.nanoTime();

CompletableFuture<User> userFuture =
    CompletableFuture.supplyAsync(() -> queryUser(userId));
CompletableFuture<Inventory> inventoryFuture =
    CompletableFuture.supplyAsync(() -> queryInventory(productId));
CompletableFuture<Price> priceFuture =
    CompletableFuture.supplyAsync(() -> queryPrice(productId));

// allOf：等待所有Future完成
CompletableFuture.allOf(userFuture, inventoryFuture, priceFuture)
    .thenAccept(v -> {
        long duration = (System.nanoTime() - start) / 1_000_000;
        System.out.println("并行查询耗时: " + duration + "ms");
    });

// 如果用串行查询：
// userFuture.get() + inventoryFuture.get() + priceFuture.get()
// 耗时 = t1 + t2 + t3（串行）

// 如果用并行查询：
// 耗时 ≈ max(t1, t2, t3)（并行）
```

### 4.2 anyOf：最先完成的决定结果

```java
// 场景：同时查询多个供应商的报价，返回最快响应的那一个
List<CompletableFuture<Quote>> futures = suppliers.stream()
    .map(s -> CompletableFuture.supplyAsync(() -> s.getQuote(productId)))
    .collect(Collectors.toList());

CompletableFuture<Object> firstCompleted =
    CompletableFuture.anyOf(futures.toArray(new CompletableFuture[0]));

Quote bestQuote = (Quote) firstCompleted.join();
```

### 4.3 完整的异步编排示例

```java
public CompletableFuture<OrderResult> createOrder(Order order) {
    return CompletableFuture
        // 第一步：验证用户（异步）
        .supplyAsync(() -> validateUser(order.getUserId()), userExecutor)

        // 第二步：查询库存（依赖用户验证结果）
        .thenCompose(user -> CompletableFuture
            .supplyAsync(() -> checkInventory(order.getProductId(), order.getQuantity()),
                        inventoryExecutor))

        // 第三步：计算价格和创建订单（依赖库存）
        .thenCombine(
            CompletableFuture.supplyAsync(() -> calculatePrice(order), priceExecutor),
            (inventory, price) -> {
                order.setPrice(price);
                order.setInventory(inventory);
                return order;
            })

        // 第四步：持久化订单
        .thenApplyAsync(order -> {
            saveOrder(order);
            return order;
        }, dbExecutor)

        // 第五步：发送通知（不阻塞主流程）
        .thenAccept(order -> {
            CompletableFuture.runAsync(() ->
                sendNotification(order.getUserId()), notificationExecutor);
        })

        // 异常处理
        .exceptionally(ex -> {
            log.error("订单创建失败: orderId={}", order.getId(), ex);
            throw new RuntimeException("订单创建失败", ex);
        });
}
```

## 五、生产避坑：CompletableFuture的陷阱 ⚠️

### 5.1 线程池配置不当

```java
// ❌ 危险：默认使用ForkJoinPool.commonPool（线程数=CPU-1）
CompletableFuture.supplyAsync(() -> fetchFromDB()); // IO密集型任务用错了线程池！

// ✅ 正确：为IO密集型任务配置专用线程池
ExecutorService ioExecutor = new ThreadPoolExecutor(
    100, 200, 60L, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(1000),
    new ThreadFactoryBuilder().setNameFormat("io-pool-%d").build()
);
CompletableFuture.supplyAsync(() -> fetchFromDB(), ioExecutor);
```

### 5.2 异步调用被同步阻塞

```java
// ❌ 危险：CompletableFuture中有同步阻塞调用
CompletableFuture.supplyAsync(() -> {
    CompletableFuture<User> userFuture = getUserAsync(); // 异步
    userFuture.join(); // 同步阻塞！破坏了异步的意义
});

// ✅ 正确：完全异步化
CompletableFuture.supplyAsync(() -> getUserAsync())
    .thenCompose(userFuture -> userFuture); // 扁平化
```

### 5.3 不处理异常

```java
// ❌ 危险：没有异常处理
CompletableFuture.supplyAsync(() -> riskyCall())
    .thenApply(result -> process(result)); // 异常被吞掉！

// ✅ 正确：必须处理异常
CompletableFuture.supplyAsync(() -> riskyCall())
    .exceptionally(ex -> {
        log.error("调用失败", ex);
        throw new RuntimeException(ex);
    })
    .thenApply(result -> process(result));
```

## 六、❌ 错误示范

**候选人原话**："thenApply和thenCompose是一样的，都能链式调用。"

**问题诊断**：
- thenApply用于同步转换，返回值
- thenCompose用于返回CompletableFuture的场景，自动扁平化
- 混淆两者会导致嵌套Future问题

**候选人原话**："CompletableFuture是异步的，所以一定比同步快。"

**问题诊断**：
- 异步的好处是释放线程，不是"快"
- 如果是CPU密集型任务，用异步反而增加线程切换开销
- 异步适用于IO密集型任务

【面试官心理】
CompletableFuture的thenApply/thenCompose/thenCombine区别是Java并发编程中最经典的API混淆题。能正确区分三者的候选人，说明他对函数式编程和异步编程都有实战经验。CompletableFuture不是银弹——它只是让异步代码看起来像同步代码，性能仍然取决于任务特性和线程池配置。

## 七、面试应对脚本

**面试官**："thenApply和thenCompose的区别是什么？"
**标准回答**："thenApply用于对CompletableFuture的结果进行同步转换，返回一个新的CompletableFuture。比如thenApply(Integer::parseInt)将String类型转为Integer类型。thenCompose用于需要返回CompletableFuture的场景，它会自动扁平化，避免嵌套Future。比如thenCompose(userId -> getUserDetailAsync(userId))，如果用thenApply会产生CompletableFuture<CompletableFuture<UserDetail>>，用thenCompose则产生CompletableFuture<UserDetail>。thenCombine用于合并两个独立的CompletableFuture，当两个都完成后执行合并函数。"

【面试官心理】
能回答thenApply/thenCompose/thenCombine区别的候选人，说明他对Java函数式编程和异步编程都有深入理解。这类候选人在处理复杂的异步业务流程时，能设计出清晰、高效的异步编排方案。
