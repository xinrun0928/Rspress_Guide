# CompletableFuture：现代 Java 异步编程利器

如果说 `Future` 是异步编程的 1.0 时代，那么 `CompletableFuture` 就是 2.0 时代。Java 8 引入的 `CompletableFuture` 解决了 `Future` 的所有痛点——它支持回调而非阻塞、支持链式组合、支持手动完成、支持异常处理。掌握 `CompletableFuture`，就掌握了现代 Java 异步编程的核心。

## CompletableFuture 的诞生背景

`Future` 虽然解决了异步返回值的问题，但它的局限性在实际应用中暴露无遗：

```java
// Future 的局限性：需要轮询或阻塞
Future<Integer> f = executor.submit(() -> calculation());

// 只能阻塞等待
Integer result = f.get();

// 或者轮询（浪费 CPU）
while (!f.isDone()) {
    doOtherWork();
}
result = f.get();
```

而且，`Future` 无法：
- 将多个 `Future` 串联执行
- 在 `Future` 完成后自动执行后续操作
- 手动设置 `Future` 的结果
- 处理异常或超时

`CompletableFuture` 就是为了解决这些问题而生的。

## 创建 CompletableFuture

### 手动创建已完成的任务

```java
// 创建一个已完成的任务
CompletableFuture<String> completed = CompletableFuture.completedFuture("Hello");
CompletableFuture<String> failed = CompletableFuture.failedFuture(
    new RuntimeException("Oops")
);
```

### 基于 Runnable 或 Supplier

```java
ExecutorService executor = Executors.newFixedThreadPool(4);

// 基于 Supplier（可返回结果）
CompletableFuture<String> future1 = CompletableFuture.supplyAsync(
    () -> "Hello", executor
);

// 基于 Runnable（无返回值）
CompletableFuture<Void> future2 = CompletableFuture.runAsync(
    () -> doSomething(), executor
);

// 使用默认线程池（ForkJoinPool.commonPool）
CompletableFuture<String> future3 = CompletableFuture.supplyAsync(
    () -> fetchFromRemote()
);
```

### 手动完成

```java
CompletableFuture<String> future = new CompletableFuture<>();

// 在某个线程中手动完成
new Thread(() -> {
    try {
        String result = fetchData();
        future.complete(result);  // 正常完成
    } catch (Exception e) {
        future.completeExceptionally(e);  // 异常完成
    }
}).start();

// 消费结果
future.thenAccept(System.out::println);
```

## 转换与链式调用

`CompletableFuture` 提供了丰富的链式 API：

### thenApply：转换结果

```java
// 同步转换：thenApply
CompletableFuture<String> f1 = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenApply(String::toUpperCase);  // "HELLO"

// 异步转换：thenApplyAsync（在新线程中执行）
CompletableFuture<String> f2 = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenApplyAsync(s -> {
        sleep(100);
        return s.toUpperCase();
    });
```

### thenCompose：扁平化链式调用

当转换函数返回另一个 `Future` 时，需要使用 `thenCompose` 避免嵌套：

```java
// 使用 thenApply（会嵌套）
CompletableFuture<CompletableFuture<String>> nested = 
    CompletableFuture.supplyAsync(() -> "hello")
        .thenApply(s -> CompletableFuture.supplyAsync(() -> s + " world"));

// 使用 thenCompose（扁平化）
CompletableFuture<String> flat = 
    CompletableFuture.supplyAsync(() -> "hello")
        .thenCompose(s -> CompletableFuture.supplyAsync(() -> s + " world"));
```

这类似于函数式编程中的 `flatMap`（扁平化 map）。

### thenCombine：组合两个 Future

```java
CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "Hello");
CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "World");

// 等待两个 Future 都完成后，合并结果
CompletableFuture<String> combined = f1.thenCombine(f2, (s1, s2) -> s1 + " " + s2);
```

### thenAccept：消费结果

```java
// 同步消费
CompletableFuture.supplyAsync(() -> "Hello")
    .thenAccept(System.out::println);

// 异步消费
CompletableFuture.supplyAsync(() -> "Hello")
    .thenAcceptAsync(System.out::println);
```

### thenRun：不关心结果

```java
// 不需要前一个结果，也不产生新结果
CompletableFuture.supplyAsync(() -> calculation())
    .thenRun(() -> System.out.println("计算完成"));
```

## 异常处理

### exceptionally：处理异常

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> {
        if (true) throw new RuntimeException("Oops");
        return "success";
    })
    .exceptionally(ex -> {
        System.out.println("发生异常: " + ex.getMessage());
        return "fallback";  // 返回默认值
    });
```

### handle：无论如何都处理

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> doSomething())
    .handle((result, ex) -> {
        if (ex != null) {
            System.out.println("发生异常: " + ex.getMessage());
            return "default";
        }
        return result;
    });
```

### whenComplete：完成时回调

```java
CompletableFuture.supplyAsync(() -> doSomething())
    .whenComplete((result, ex) -> {
        if (ex != null) {
            System.out.println("失败: " + ex.getMessage());
        } else {
            System.out.println("成功: " + result);
        }
    });
```

## 超时处理

### orTimeout（JDK 9+）

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> doSomething())
    .orTimeout(3, TimeUnit.SECONDS)  // 3秒超时
    .exceptionally(ex -> "超时了");
```

### 手动实现超时（JDK 8）

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> doSomething())
    .applyToEither(
        CompletableFuture.delayedFuture(3, TimeUnit.SECONDS),
        s -> s
    );
```

## 实际应用案例

### 并行执行多个任务并等待

```java
CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> fetchUser(1));
CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> fetchUser(2));
CompletableFuture<String> f3 = CompletableFuture.supplyAsync(() -> fetchUser(3));

// 等待所有任务完成
CompletableFuture.allOf(f1, f2, f3).join();

// 获取结果
List<String> users = Stream.of(f1, f2, f3)
    .map(CompletableFuture::join)
    .collect(Collectors.toList());
```

### 竞速：取最快的结果

```java
CompletableFuture<String> fast = CompletableFuture
    .anyOf(
        CompletableFuture.supplyAsync(() -> queryDatabase()),
        CompletableFuture.supplyAsync(() -> queryCache()),
        CompletableFuture.supplyAsync(() -> queryRemote())
    )
    .thenApply(Object::toString);
```

### 流水线处理

```java
CompletableFuture.supplyAsync(() -> parseRequest(request))
    .thenApply(this::validateRequest)
    .thenApply(this::authorizeRequest)
    .thenApply(this::executeRequest)
    .thenApply(this::formatResponse)
    .whenComplete((response, ex) -> {
        if (ex != null) {
            log.error("请求处理失败", ex);
        }
    });
```

### 组合多个异步操作

```java
CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(() -> getUser());
CompletableFuture<List<Order>> ordersFuture = CompletableFuture.supplyAsync(() -> getOrders());

userFuture.thenCombine(ordersFuture, (user, orders) -> 
    new UserProfile(user, orders)
).thenApply(profile ->
    enrichWithRecommendations(profile)
).thenAccept(profile ->
    displayProfile(profile)
);
```

## 性能考虑

### 避免不必要的异步

```java
// 不要为简单操作创建异步
CompletableFuture.supplyAsync(() -> "hello")  // 不必要
CompletableFuture.completedFuture("hello")    // 直接完成

// 正确的场景：真正的耗时操作
CompletableFuture.supplyAsync(() -> fetchFromNetwork())
```

### 选择合适的线程池

```java
// CPU 密集型：使用专用线程池
ExecutorService cpuPool = Executors.newFixedThreadPool(
    Runtime.getRuntime().availableProcessors()
);

// IO 密集型：可以使用更多线程
ExecutorService ioPool = Executors.newFixedThreadPool(100);

// 每个任务使用独立线程池（避免相互影响）
CompletableFuture.supplyAsync(() -> doWork(), dedicatedPool);
```
