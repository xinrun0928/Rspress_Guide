# CompletableFuture 优势：异步 + 链式编排

`CompletableFuture` 是 JDK 8 引入的异步编程利器，它解决了 `Future` 的三大局限性：阻塞式获取、无链式调用、无法组合。与 `Future` 相比，`CompletableFuture` 提供了丰富的 API，支持异步任务链式编排、异常处理、多任务组合。

## Future 的局限回顾

```java
public class FutureLimitations {
    
    public void futureProblems() throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(3);
        
        // Future 的问题1：必须阻塞等待结果
        Future<String> f1 = executor.submit(() -> "result");
        String result = f1.get();  // 阻塞！
        
        // Future 的问题2：无法链式调用
        Future<String> f2 = executor.submit(() -> {
            // 依赖 f1 的结果？
            // 必须嵌套 get()
            return f1.get() + " world";
        });
        
        // Future 的问题3：无法组合多个 Future
        // 必须手动等待所有完成
    }
}
```

## 基本用法对比

```java
public class Comparison {
    
    // Future 方式：阻塞
    public void futureWay() throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        
        Future<String> future = executor.submit(() -> {
            Thread.sleep(1000);
            return "Hello";
        });
        
        String result = future.get();  // 阻塞 1 秒
        System.out.println(result);
    }
    
    // CompletableFuture 方式：非阻塞
    public void completableFutureWay() {
        CompletableFuture.supplyAsync(() -> {
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            return "Hello";
        }).thenAccept(result -> {
            System.out.println(result);  // 回调，非阻塞
        });
    }
}
```

## 异步链式调用

```java
public class AsyncChain {
    
    public void chainDemo() {
        CompletableFuture.supplyAsync(() -> {
                // 第一步：查询用户
                return queryUser();
            })
            .thenApply(user -> {
                // 第二步：查询用户订单
                return queryOrders(user);
            })
            .thenApply(orders -> {
                // 第三步：处理订单
                return processOrders(orders);
            })
            .thenAccept(result -> {
                // 最终结果处理
                System.out.println("完成: " + result);
            })
            .exceptionally(ex -> {
                // 异常处理
                System.err.println("出错: " + ex.getMessage());
                return null;
            });
    }
    
    private String queryUser() { return "user"; }
    private String queryOrders(String user) { return "orders"; }
    private String processOrders(String orders) { return "processed"; }
}
```

## 创建 CompletableFuture

```java
public class Creation {
    
    public void creationDemo() {
        // 方式1：supplyAsync - 有返回值
        CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "result");
        
        // 方式2：runAsync - 无返回值
        CompletableFuture<Void> f2 = CompletableFuture.runAsync(() -> {
            System.out.println("异步执行");
        });
        
        // 方式3：completedFuture - 已完成
        CompletableFuture<String> f3 = CompletableFuture.completedFuture("done");
        
        // 方式4：failedFuture - 失败
        CompletableFuture<String> f4 = CompletableFuture.failedFuture(
            new RuntimeException("error")
        );
        
        // 方式5：allOf - 等待所有
        CompletableFuture.allOf(f1, f2, f3);
        
        // 方式6：anyOf - 等待任一
        CompletableFuture.anyOf(f1, f2, f3);
    }
}
```

## 与线程池配合

```java
public class WithExecutor {
    
    private final ExecutorService executor = Executors.newFixedThreadPool(4);
    
    public void withExecutorDemo() {
        // 默认使用 ForkJoinPool.commonPool()
        CompletableFuture.supplyAsync(() -> "default")
            .thenApply(s -> s + " pool");

        CompletableFuture.supplyAsync(() -> "parallel")
            .thenApplyAsync(s -> s + " async");
```

## 组合操作

```java
public class Combination {
    
    public void combineDemo() {
        CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "Hello");
        CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "World");
        
        // thenCombine: 组合两个 Future
        CompletableFuture<String> combined = f1.thenCombine(f2, (a, b) -> a + " " + b);
        
        // thenCompose: 扁平化链式调用
        CompletableFuture<String> f3 = f1.thenCompose(s -> 
            CompletableFuture.supplyAsync(() -> s + " World")
        );
    }
    
    public void allOfDemo() {
        CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "a");
        CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "b");
        CompletableFuture<String> f3 = CompletableFuture.supplyAsync(() -> "c");
        
        // 等待所有完成
        CompletableFuture.allOf(f1, f2, f3).join();
        
        // 获取所有结果
        String result = Stream.of(f1, f2, f3)
            .map(CompletableFuture::join)
            .collect(Collectors.joining(","));
    }
}
```

## 异常处理

```java
public class ExceptionHandling {
    
    public void exceptionDemo() {
        CompletableFuture.supplyAsync(() -> {
            if (Math.random() > 0.5) {
                throw new RuntimeException("随机错误");
            }
            return "success";
        })
        .exceptionally(ex -> {
            // 处理异常
            System.err.println("出错: " + ex.getMessage());
            return "fallback";
        })
        .thenAccept(result -> {
            System.out.println(result);
        });
    }
}
```

## 总结

CompletableFuture 的优势：

1. **非阻塞**：通过回调处理结果
2. **链式调用**：thenApply, thenCompose, thenCombine
3. **组合操作**：allOf, anyOf, join
4. **异常处理**：exceptionally, handle, whenComplete
5. **多种执行方式**：sync/async, 默认/自定义线程池

相比 Future，CompletableFuture 是现代异步编程的首选。
