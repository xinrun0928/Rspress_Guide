# anyOf 实现超时降级与快速失败

系统响应时间超过 3 秒，用户就会开始不耐烦；超过 5 秒，超过一半的用户会离开页面。在分布式系统中，一次请求可能涉及多个服务调用，任何一个服务的延迟都会拖慢整个系统。超时降级和快速失败，是保障系统可用性的关键策略。`anyOf` 正是实现这些策略的重要工具。

## anyOf 的基本概念

`CompletableFuture.anyOf()` 与 `allOf` 相对。`allOf` 等待**所有** `CompletableFuture` 完成，`anyOf` 等待**任一** `CompletableFuture` 完成。当第一个 `CompletableFuture` 完成时，`anyOf` 就立即完成。

```java
public class AnyOfBasicDemo {

    public CompletableFuture<String> queryFromMultipleSources(String id) {
        CompletableFuture<String> primarySource = CompletableFuture.supplyAsync(() ->
            queryPrimary(id)
        );

        CompletableFuture<String> backupSource = CompletableFuture.supplyAsync(() ->
            queryBackup(id)
        );

        // 任一完成即可
        return CompletableFuture.anyOf(primarySource, backupSource)
            .thenApply(result -> (String) result);  // anyOf 返回 Object，需要转型
    }

    private String queryPrimary(String id) {
        // 主数据源，可能较慢
        sleep(2000);
        return "primary result";
    }

    private String queryBackup(String id) {
        // 备用数据源，通常较快
        sleep(500);
        return "backup result";
    }

    private void sleep(int ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) {}
    }
}
```

## 超时降级的经典实现

超时降级是最常见的 `anyOf` 使用场景：

```java
public class TimeoutDegradation {

    public CompletableFuture<QueryResult> queryWithTimeout(String query) {
        // 实际查询任务
        CompletableFuture<QueryResult> actualQuery = CompletableFuture.supplyAsync(() ->
            performExpensiveQuery(query)
        );

        // 超时任务
        CompletableFuture<QueryResult> timeoutFuture = CompletableFuture.supplyAsync(() -> {
            sleep(3000);  // 3 秒超时
            return QueryResult.timeout();
        });

        // 返回最快的结果
        return CompletableFuture.anyOf(actualQuery, timeoutFuture)
            .thenApply(result -> (QueryResult) result)
            .exceptionally(ex -> {
                // 任意一方完成都会触发，这里需要处理
                return QueryResult.error();
            });
    }

    private QueryResult performExpensiveQuery(String query) {
        // 模拟慢查询
        sleep(10000);
        return new QueryResult("expensive result");
    }

    private void sleep(int ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) {}
    }
}
```

但上面的实现有一个问题：如果实际查询在超时后完成，它仍在后台运行，造成资源浪费。更完善的实现需要取消失败的任务：

```java
public class TimeoutDegradationV2 {

    public CompletableFuture<QueryResult> queryWithTimeout(String query) {
        CompletableFuture<QueryResult> actualQuery = CompletableFuture.supplyAsync(() ->
            performExpensiveQuery(query)
        );

        CompletableFuture<QueryResult> timeoutFuture = CompletableFuture.supplyAsync(() -> {
            Thread.sleep(3000);
            return QueryResult.timeout();
        });

        return CompletableFuture.anyOf(actualQuery, timeoutFuture)
            .thenApply(result -> (QueryResult) result)
            .handle((result, ex) -> {
                // 无论成功还是超时，都需要取消另一个任务
                if (result != null && result.isTimeout()) {
                    actualQuery.cancel(true);  // 取消实际查询
                } else if (result == null || ex != null) {
                    // 查询失败，取消超时任务（实际上它可能已经完成了）
                    timeoutFuture.cancel(true);
                }
                return result != null ? result : QueryResult.error();
            });
    }

    private QueryResult performExpensiveQuery(String query) {
        sleep(10000);
        return new QueryResult("expensive result");
    }

    private void sleep(int ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) {}
    }
}
```

## 更优雅的超时方案：orTimeout

JDK 9 引入了 `orTimeout` 方法，让超时处理更加优雅：

```java
public CompletableFuture<User> getUser(String userId) {
    return CompletableFuture
        .supplyAsync(() -> queryUserFromDatabase(userId))
        .orTimeout(3, TimeUnit.SECONDS)  // 设置 3 秒超时
        .exceptionally(ex -> {
            if (ex instanceof TimeoutException) {
                log.warn("用户查询超时: {}", userId);
                return getDefaultUser(userId);
            }
            return User.ANONYMOUS;
        });
}
```

结合 `completeOnTimeout`（JDK 9+），可以实现更简洁的降级逻辑：

```java
public CompletableFuture<User> getUser(String userId) {
    return CompletableFuture
        .supplyAsync(() -> queryUserFromDatabase(userId))
        .orTimeout(3, TimeUnit.SECONDS)
        .completeOnTimeout(User.ANONYMOUS, 3, TimeUnit.SECONDS)  // 超时后使用默认值
        .exceptionally(ex -> {
            // 这里只处理真正的异常，超时已经被 completeOnTimeout 处理了
            log.error("用户查询异常", ex);
            return User.ERROR;
        });
}
```

## 快速失败策略

"快速失败"意味着当某个条件不满足时，立即返回失败，而不是继续等待：

```java
public class FastFailService {

    public CompletableFuture<ServiceResult> processOrder(Order order) {
        // 并行检查多个前置条件
        CompletableFuture<Boolean> inventoryCheck = CompletableFuture.supplyAsync(() ->
            checkInventory(order)
        );

        CompletableFuture<Boolean> creditCheck = CompletableFuture.supplyAsync(() ->
            checkCredit(order)
        );

        CompletableFuture<Boolean> riskCheck = CompletableFuture.supplyAsync(() ->
            checkRisk(order)
        );

        // 等待所有检查完成
        return CompletableFuture.allOf(inventoryCheck, creditCheck, riskCheck)
            .thenApply(v -> {
                // 汇总结果
                if (!inventoryCheck.join()) {
                    return ServiceResult.failed("库存不足");
                }
                if (!creditCheck.join()) {
                    return ServiceResult.failed("信用不足");
                }
                if (!riskCheck.join()) {
                    return ServiceResult.failed("风控拦截");
                }
                return ServiceResult.success();
            });
    }
}
```

但有时我们需要"任一失败就失败"的逻辑：

```java
public CompletableFuture<ServiceResult> processOrderWithFastFail(Order order) {
    CompletableFuture<ServiceResult>[] checks = new CompletableFuture[]{
        CompletableFuture.supplyAsync(() -> checkInventory(order))
            .thenApply(ok -> ok ? null : ServiceResult.failed("库存不足")),

        CompletableFuture.supplyAsync(() -> checkCredit(order))
            .thenApply(ok -> ok ? null : ServiceResult.failed("信用不足")),

        CompletableFuture.supplyAsync(() -> checkRisk(order))
            .thenApply(ok -> ok ? null : ServiceResult.failed("风控拦截"))
    };

    return CompletableFuture.anyOf(checks)
        .thenApply(result -> {
            ServiceResult serviceResult = (ServiceResult) result;
            if (serviceResult != null) {
                return serviceResult;
            }
            // 所有检查都通过
            return ServiceResult.success();
        });
}
```

## 实战：多级缓存查询

```java
public class MultiLevelCacheService {

    private final Map<String, String> l1Cache = new ConcurrentHashMap<>();
    private final Map<String, String> l2Cache = new ConcurrentHashMap<>();

    public CompletableFuture<String> get(String key) {
        // L1 缓存命中
        String l1Result = l1Cache.get(key);
        if (l1Result != null) {
            return CompletableFuture.completedFuture(l1Result);
        }

        // L1 未命中，并行查询 L2 和数据库
        CompletableFuture<String> l2Future = CompletableFuture.supplyAsync(() ->
            l2Cache.get(key)
        );

        CompletableFuture<String> dbFuture = CompletableFuture.supplyAsync(() ->
            queryFromDatabase(key)
        );

        // 返回任一结果
        return CompletableFuture.anyOf(l2Future, dbFuture)
            .thenApply(result -> (String) result)
            .thenApply(cached -> {
                // 回填 L1 缓存
                if (cached != null) {
                    l1Cache.put(key, cached);
                }
                return cached;
            })
            .exceptionally(ex -> {
                log.error("查询失败: {}", key, ex);
                return null;
            });
    }

    private String queryFromDatabase(String key) {
        return "db:value";
    }
}
```

## anyOf 的注意事项

### 类型转换问题

`anyOf` 返回的是 `CompletableFuture<Object>`，因为它不知道哪个会先完成，所以返回的是通用的 `Object` 类型：

```java
// 需要手动转型
CompletableFuture.anyOf(future1, future2, future3)
    .thenApply(result -> (String) result);  // 假设所有 Future 都返回 String
```

更好的方式是使用泛型方法封装：

```java
public <T> CompletableFuture<T> anyOf(CompletableFuture<T>... futures) {
    return CompletableFuture.anyOf(futures)
        .thenApply(result -> (T) result);
}
```

### 不支持 cancel

与 `allOf` 不同，`anyOf` 返回的 `CompletableFuture` 无法直接取消未完成的任务。如果需要取消，需要在后续处理中显式调用 `cancel`。

### 与超时的结合

`anyOf` 最适合与超时任务结合使用。在实际任务和超时任务之间"赛跑"，谁先完成就用谁的结果：

```java
public <T> CompletableFuture<T> withTimeout(
        CompletableFuture<T> task,
        long timeout,
        TimeUnit unit,
        T fallbackValue) {

    CompletableFuture<T> timeoutFuture = CompletableFuture.supplyAsync(() -> {
        unit.sleep(timeout);
        return fallbackValue;
    });

    return CompletableFuture.anyOf(task, timeoutFuture)
        .thenApply(result -> (T) result)
        .thenCompose(finalResult -> {
            // 如果是超时任务的结果，取消原任务
            if (finalResult == fallbackValue) {
                task.cancel(true);
            }
            return CompletableFuture.completedFuture(finalResult);
        });
}
```

## 面试高频考点

**问题一：anyOf 和 allOf 的区别是什么？**

`allOf` 等待**所有** `CompletableFuture` 完成后才完成；`anyOf` 等待**任一** `CompletableFuture` 完成后就立即完成。

**问题二：anyOf 返回的类型是什么？**

`anyOf` 返回 `CompletableFuture<Object>`，因为不知道哪个 `CompletableFuture` 会先完成，所以使用通用类型。

**问题三：如何使用 anyOf 实现超时？**

创建一个超时任务的 `CompletableFuture`，然后用 `anyOf` 让实际任务和超时任务"赛跑"，谁先完成就用谁的结果。

**问题四：anyOf 能取消失败的任务吗？**

`anyOf` 本身不支持取消。但可以在获取结果后，手动调用 `cancel()` 取消不需要的任务。

**问题五：orTimeout 和 anyOf 实现的超时有什么区别？**

`orTimeout` 是 JDK 9 引入的标准 API，语义更清晰，实现更简洁。`anyOf` 更灵活，可以实现"超时降级"（返回默认值）或者"快速失败"（任务失败就立即返回）等复杂场景。

超时降级和快速失败，是保障系统高可用的重要策略。`anyOf` 让我们能够优雅地实现这些策略，在响应性和健壮性之间找到平衡。
