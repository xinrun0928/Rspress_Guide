# allOf 聚合多路查询结果并处理异常

想象这样一个场景：你在开发一个聚合搜索功能，用户搜索一个关键词，需要同时查询多个数据源——数据库、Elasticsearch、Redis 缓存、第三方 API。每个数据源的响应时间不同，有的快有的慢；有的可能成功，有的可能失败。你需要等待所有数据源返回后，合并结果返回给用户。`allOf` 就是解决这类问题的利器。

## allOf 的基本用法

`CompletableFuture.allOf()` 接收一个可变参数的 `CompletableFuture` 数组，返回一个新的 `CompletableFuture`。当所有传入的 `CompletableFuture` 都正常完成（不是异常完成）时，这个新的 `CompletableFuture` 才会完成。

```java
public class AllOfBasicDemo {

    public CompletableFuture<List<String>> searchFromMultipleSources(String query) {
        // 创建多个独立的异步任务
        CompletableFuture<List<String>> dbSearch = CompletableFuture.supplyAsync(() ->
            searchFromDatabase(query)
        );

        CompletableFuture<List<String>> esSearch = CompletableFuture.supplyAsync(() ->
            searchFromElasticsearch(query)
        );

        CompletableFuture<List<String>> cacheSearch = CompletableFuture.supplyAsync(() ->
            searchFromCache(query)
        );

        // 使用 allOf 等待所有任务完成
        return CompletableFuture.allOf(dbSearch, esSearch, cacheSearch)
            .thenApply(v -> {
                // 所有任务都完成后，合并结果
                List<String> results = new ArrayList<>();
                results.addAll(dbSearch.join());
                results.addAll(esSearch.join());
                results.addAll(cacheSearch.join());
                return results;
            });
    }

    private List<String> searchFromDatabase(String query) {
        // 模拟数据库查询
        return List.of("db:result1", "db:result2");
    }

    private List<String> searchFromElasticsearch(String query) {
        // 模拟 ES 查询
        return List.of("es:result1");
    }

    private List<String> searchFromCache(String query) {
        // 模拟缓存查询
        return List.of("cache:result1", "cache:result2", "cache:result3");
    }
}
```

## allOf 的陷阱：异常处理

`allOf` 有一个重要的特性需要特别注意：**即使部分 `CompletableFuture` 异常完成，只要有一个 `CompletableFuture` 正常完成，返回的 `CompletableFuture` 就会完成**。

这意味着默认情况下，你无法通过 `allOf` 的返回值感知到任何异常：

```java
CompletableFuture.allOf(task1, task2, task3)
    .thenApply(v -> {
        // 即使 task2 失败了，这里仍然会执行
        // 因为 task1 和 task3 都正常完成了
        return mergeResults(task1.join(), task2.join(), task3.join());
    })
    .exceptionally(ex -> {
        // 这个回调永远不会执行！
        // 因为 allOf 返回的 Future 不会因为部分失败而异常完成
        return Collections.emptyList();
    });
```

### 正确的异常处理方式

如果需要在 `allOf` 中处理部分失败，需要手动检查每个 `CompletableFuture` 的状态：

```java
public CompletableFuture<AggregatedResult> searchWithErrorHandling(String query) {
    CompletableFuture<List<String>> dbFuture = CompletableFuture.supplyAsync(() ->
        searchFromDatabase(query)
    );

    CompletableFuture<List<String>> esFuture = CompletableFuture.supplyAsync(() ->
        searchFromElasticsearch(query)
    );

    CompletableFuture<List<String>> cacheFuture = CompletableFuture.supplyAsync(() ->
        searchFromCache(query)
    );

    return CompletableFuture.allOf(dbFuture, esFuture, cacheFuture)
        .thenApply(v -> {
            List<String> results = new ArrayList<>();
            List<String> errors = new ArrayList<>();

            // 手动检查每个 Future 的状态
            collectResults(dbFuture, results, errors, "数据库");
            collectResults(esFuture, results, errors, "Elasticsearch");
            collectResults(cacheFuture, results, errors, "缓存");

            return new AggregatedResult(results, errors);
        });
}

private void collectResults(
        CompletableFuture<List<String>> future,
        List<String> results,
        List<String> errors,
        String source) {
    if (future.isCompletedExceptionally()) {
        errors.add(source + " 查询失败: " + future.exceptionNow().getMessage());
    } else {
        results.addAll(future.join());
    }
}
```

## 实战场景：电商商品聚合

一个典型的电商商品详情页需要聚合多个数据源：

```java
public class ProductAggregationService {

    public CompletableFuture<ProductPageData> getProductPage(String productId) {
        // 并行发起多个查询
        CompletableFuture<ProductBasicInfo> basicFuture = CompletableFuture.supplyAsync(() ->
            productRepository.getBasicInfo(productId)
        );

        CompletableFuture<List<ProductImage>> imageFuture = CompletableFuture.supplyAsync(() ->
            productRepository.getImages(productId)
        );

        CompletableFuture<List<Review>> reviewFuture = CompletableFuture.supplyAsync(() ->
            reviewService.getReviews(productId)
        );

        CompletableFuture<ProductStock> stockFuture = CompletableFuture.supplyAsync(() ->
            inventoryService.getStock(productId)
        );

        CompletableFuture<List<Coupon>> couponFuture = CompletableFuture.supplyAsync(() ->
            couponService.getAvailableCoupons(productId)
        );

        // 使用 allOf 聚合结果
        return CompletableFuture.allOf(
                basicFuture, imageFuture, reviewFuture,
                stockFuture, couponFuture
            )
            .thenApply(v -> {
                // 处理异常情况，提供默认值
                ProductBasicInfo basic = getOrDefault(basicFuture, ProductBasicInfo.EMPTY);
                List<ProductImage> images = getOrDefault(imageFuture, Collections.emptyList());
                List<Review> reviews = getOrDefault(reviewFuture, Collections.emptyList());
                ProductStock stock = getOrDefault(stockFuture, ProductStock.UNAVAILABLE);
                List<Coupon> coupons = getOrDefault(couponFuture, Collections.emptyList());

                return new ProductPageData(basic, images, reviews, stock, coupons);
            })
            .exceptionally(ex -> {
                log.error("获取商品页数据失败: {}", productId, ex);
                return ProductPageData.error();
            });
    }

    private <T> T getOrDefault(CompletableFuture<T> future, T defaultValue) {
        if (future.isCompletedExceptionally()) {
            return defaultValue;
        }
        try {
            return future.join();
        } catch (Exception e) {
            return defaultValue;
        }
    }
}
```

## 实战场景：多数据源配置加载

应用启动时需要从多个配置源加载配置：

```java
public class ConfigLoader {

    public CompletableFuture<AppConfig> loadConfig() {
        CompletableFuture<Map<String, String>> appPropsFuture = CompletableFuture.supplyAsync(() ->
            loadFromFile("application.properties")
        );

        CompletableFuture<Map<String, String>> dbPropsFuture = CompletableFuture.supplyAsync(() ->
            loadFromFile("database.properties")
        );

        CompletableFuture<Map<String, String>> envPropsFuture = CompletableFuture.supplyAsync(() ->
            loadFromEnvironment()
        );

        CompletableFuture<Map<String, String>> remotePropsFuture = CompletableFuture.supplyAsync(() ->
            loadFromConfigServer()
        );

        return CompletableFuture.allOf(appPropsFuture, dbPropsFuture, envPropsFuture, remotePropsFuture)
            .thenApply(v -> {
                Map<String, String> merged = new HashMap<>();

                // 按优先级合并：远程 > 环境变量 > 数据库配置 > 应用配置
                merge(merged, appPropsFuture.join());
                merge(merged, dbPropsFuture.join());
                merge(merged, envPropsFuture.join());
                merge(merged, remotePropsFuture.join());

                return new AppConfig(merged);
            })
            .exceptionally(ex -> {
                log.warn("部分配置加载失败，使用默认配置", ex);
                return AppConfig.defaultConfig();
            });
    }

    private void merge(Map<String, String> target, Map<String, String> source) {
        if (source != null) {
            target.putAll(source);
        }
    }
}
```

## 处理动态数量的 Future

有时候，Future 的数量在运行时才能确定（比如根据用户 ID 列表查询每个用户的信息）：

```java
public CompletableFuture<List<UserInfo>> getUserInfos(List<String> userIds) {
    // 创建所有查询 Future
    List<CompletableFuture<UserInfo>> futures = userIds.stream()
        .map(id -> CompletableFuture.supplyAsync(() -> queryUser(id)))
        .collect(Collectors.toList());

    // 转换为数组
    CompletableFuture<UserInfo>[] array = futures.toArray(new CompletableFuture[0]);

    return CompletableFuture.allOf(array)
        .thenApply(v ->
            futures.stream()
                .map(CompletableFuture::join)
                .collect(Collectors.toList())
        );
}
```

## 使用 CompletableFuture.join() vs get() 

在 `allOf` 的回调中，推荐使用 `join()` 而不是 `get()`：

```java
.allOf(task1, task2, task3)
    .thenApply(v -> {
        // join() 和 get() 的区别：
        // get() 抛出受检异常 InterruptedException, ExecutionException
        // join() 抛出非受检异常 CompletionException

        // 由于 allOf 已经确保所有任务完成，这里用 join() 更简洁
        return merge(task1.join(), task2.join(), task3.join());
    })
```

## 性能优化：设置超时

对于多数据源查询，设置合理的超时时间很重要：

```java
public CompletableFuture<List<String>> searchWithTimeout(String query) {
    CompletableFuture<List<String>> dbFuture = CompletableFuture.supplyAsync(() ->
        searchFromDatabase(query)
    ).orTimeout(3, TimeUnit.SECONDS);  // 数据库超时 3 秒

    CompletableFuture<List<String>> esFuture = CompletableFuture.supplyAsync(() ->
        searchFromElasticsearch(query)
    ).orTimeout(2, TimeUnit.SECONDS);  // ES 超时 2 秒

    CompletableFuture<List<String>> cacheFuture = CompletableFuture.supplyAsync(() ->
        searchFromCache(query)
    ).orTimeout(500, TimeUnit.MILLISECONDS);  // 缓存超时 500ms

    return CompletableFuture.allOf(dbFuture, esFuture, cacheFuture)
        .thenApply(v -> {
            List<String> results = new ArrayList<>();
            // 即使部分超时，也要收集成功的结果
            if (!dbFuture.isCompletedExceptionally()) results.addAll(dbFuture.join());
            if (!esFuture.isCompletedExceptionally()) results.addAll(esFuture.join());
            if (!cacheFuture.isCompletedExceptionally()) results.addAll(cacheFuture.join());
            return results;
        });
}
```

## 面试高频考点

**问题一：allOf 返回的 CompletableFuture 什么时候完成？**

当所有传入的 `CompletableFuture` 都正常完成（非异常完成）时，`allOf` 返回的 `CompletableFuture` 才会完成。如果任一 `CompletableFuture` 异常完成，`allOf` 的结果不受影响。

**问题二：如何知道 allOf 中的哪个 Future 失败了？**

`allOf` 本身不提供这个信息。需要手动检查每个 `CompletableFuture` 的状态：`isCompletedExceptionally()` 或 `exceptionNow()`。

**问题三：allOf 有异常时，exceptionally 会被触发吗？**

不会。`allOf` 返回的 `CompletableFuture` 只会在所有传入的 `CompletableFuture` 都正常完成时完成。任一异常完成不会导致 `allOf` 的 `exceptionally` 被触发。

**问题四：allOf 和 whenComplete 配合使用有什么技巧？**

可以在 `allOf` 之后使用 `whenComplete` 来做一些清理工作或记录日志，但不建议在 `whenComplete` 中修改结果，因为 `allOf` 已经确保了所有任务完成。

**问题五：如何处理 allOf 中的超时？**

使用 `CompletableFuture.orTimeout()` 为每个 `Future` 设置超时时间，或者使用 `CompletableFuture.anyOf()` 实现"任一完成"模式。

`allOf` 是处理多路并行查询的利器，但它的异常处理机制需要特别注意。理解了这个特点，就能写出既高效又健壮的多路查询代码。
