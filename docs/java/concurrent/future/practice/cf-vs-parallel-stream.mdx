# CompletableFuture vs ParallelStream vs 响应式编程对比

现代 Java 提供了多种异步编程方式，了解它们的适用场景很重要。

## CompletableFuture

```java
public class CompletableFutureVsStream {
    
    // CompletableFuture: 灵活的任务编排
    public void completableFuture() {
        CompletableFuture.supplyAsync(() -> queryFromDB())
            .thenApply(this::transform)
            .thenAccept(this::process);
    }
    
    // 优点：灵活控制、异常处理丰富、支持依赖链
    // 缺点：语法稍复杂
}
```

## ParallelStream

```java
public class ParallelStreamVs {
    
    // ParallelStream: 数据并行处理
    public void parallelStream() {
        List<String> data = getData();
        
        List<Result> results = data.parallelStream()
            .map(this::process)
            .collect(Collectors.toList());
    }
    
    // 优点：简洁、数据并行
    // 缺点：共享状态麻烦、不支持任务依赖
}
```

## 对比表

| 特性 | CompletableFuture | ParallelStream | 响应式 |
|------|-----------------|---------------|--------|
| 任务依赖 | ✓ | ✗ | ✓ |
| 数据并行 | ✗ | ✓ | ✓ |
| 异常处理 | ✓ | ✗ | ✓ |
| 灵活性 | 高 | 中 | 高 |
| 学习曲线 | 中 | 低 | 高 |
