# CompletableFuture 性能调优：自定义线程池

`CompletableFuture` 默认使用 `ForkJoinPool.commonPool()`，但在某些场景下需要自定义线程池。

## 默认线程池的问题

```java
public class DefaultPoolProblem {
    
    public void problem() {
        // commonPool 是共享的
        // 长时间运行的任务会阻塞其他任务
        
        CompletableFuture.supplyAsync(() -> {
            Thread.sleep(10000);  // 阻塞 commonPool
            return "done";
        });
        
        // 其他任务可能受影响
    }
}
```

## 自定义线程池

```java
public class CustomExecutor {
    
    private final ExecutorService executor = Executors.newFixedThreadPool(10);
    
    public void withCustomPool() {
        CompletableFuture.supplyAsync(() -> {
            return queryDB();
        }, executor)
        .thenApplyAsync(this::process, executor)
        .thenAcceptAsync(this::save, executor);
    }
    
    private String queryDB() { return "data"; }
    private String process(String data) { return data; }
    private void save(String data) { System.out.println(data); }
}
```

## 最佳实践

```java
public class BestPractice {
    
    // 建议：为每类任务创建专用线程池
    private final ExecutorService ioExecutor = 
        Executors.newFixedThreadPool(20);  // IO密集型
    
    private final ExecutorService cpuExecutor = 
        Executors.newFixedThreadPool(
            Runtime.getRuntime().availableProcessors()
        );  // CPU密集型
    
    public void separatePools() {
        // IO任务使用大线程池
        CompletableFuture.supplyAsync(() -> queryFromDB(), ioExecutor);
        
        // CPU任务使用小线程池
        CompletableFuture.supplyAsync(() -> compute(), cpuExecutor);
    }
}
```

## 总结

1. **长时间任务**：使用自定义线程池
2. **IO密集型**：大线程池
3. **CPU密集型**：小线程池
