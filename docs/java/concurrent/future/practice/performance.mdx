# CompletableFuture 性能调优：自定义线程池

你有没有遇到过这种情况：线上服务 CPU 使用率不高，但接口响应时间很慢？或者线程池配置看起来没问题，但并发请求一多就开始超时？问题很可能出在 `CompletableFuture` 使用的线程池上。`CompletableFuture` 默认使用 `ForkJoinPool.commonPool()`，这个共享线程池很容易成为性能瓶颈。

## 默认线程池的问题

### 问题一：共享线程池导致资源竞争

```java
public class SharedPoolProblem {

    public void problematicCode() {
        // 默认使用 commonPool
        CompletableFuture.supplyAsync(() -> {
            // 假设这是一个慢查询
            return slowQuery();
        });

        // 另一个模块的代码也使用 commonPool
        CompletableFuture.supplyAsync(() -> {
            // 这个任务可能被阻塞
            return anotherSlowOperation();
        });

        // 所有任务都在争抢同一个线程池！
    }
}
```

`ForkJoinPool.commonPool()` 是 JVM 中所有并行流和 `CompletableFuture` 默认使用的线程池。如果你的应用有多个模块都在使用它，它们会相互竞争资源。

### 问题二：commonPool 的并行度受限

`ForkJoinPool.commonPool()` 的并行度默认为 `CPU 核心数 - 1`。对于 8 核机器，只有 7 个线程。这意味着如果你的任务是 I/O 密集型的，很快就会耗尽线程。

```java
public class PoolSizeDemo {

    public void showPoolSize() {
        // commonPool 的并行度
        int commonPool parallelism = ForkJoinPool.commonPool().getParallelism();
        System.out.println("commonPool 并行度: " + commonPool parallelism);

        // 实际 CPU 核心数
        int availableProcessors = Runtime.getRuntime().availableProcessors();
        System.out.println("可用 CPU 核心数: " + availableProcessors);
    }
}
```

### 问题三：阻塞任务影响其他任务

`commonPool` 使用的是工作窃取算法，适合 CPU 密集型任务。如果在 commonPool 中执行 I/O 密集型或阻塞任务，会导致线程阻塞，影响整个池的性能：

```java
// 危险：在 commonPool 中执行阻塞操作
CompletableFuture.supplyAsync(() -> {
    Thread.sleep(5000);  // 阻塞！会耗尽 commonPool 的线程
    return result;
});
```

## 自定义线程池的方案

### 方案一：为 IO 密集型任务创建专用线程池

```java
public class IoIntensiveExecutor {

    private final ExecutorService ioExecutor = Executors.newFixedThreadPool(
        100,  // I/O 密集型任务可以使用更多线程
        new ThreadFactory() {
            private int count = 1;

            @Override
            public Thread newThread(Runnable r) {
                Thread t = new Thread(r, "io-pool-" + count++);
                t.setDaemon(true);  // 设置为守护线程，JVM 可以正常退出
                return t;
            }
        }
    );

    public CompletableFuture<Result> fetchData(String id) {
        return CompletableFuture
            .supplyAsync(() -> fetchFromRemote(id), ioExecutor)
            .thenApplyAsync(this::processData, ioExecutor)
            .thenAcceptAsync(this::saveToCache, ioExecutor);
    }

    // 记得在应用关闭时关闭线程池
    public void shutdown() {
        ioExecutor.shutdown();
        try {
            if (!ioExecutor.awaitTermination(60, TimeUnit.SECONDS)) {
                ioExecutor.shutdownNow();
            }
        } catch (InterruptedException e) {
            ioExecutor.shutdownNow();
        }
    }
}
```

### 方案二：为 CPU 密集型任务创建专用线程池

```java
public class CpuIntensiveExecutor {

    private final ExecutorService cpuExecutor = Executors.newFixedThreadPool(
        Runtime.getRuntime().availableProcessors(),  // CPU 密集型任务，线程数 = 核心数
        new ThreadFactory() {
            private int count = 1;

            @Override
            public Thread newThread(Runnable r) {
                Thread t = new Thread(r, "cpu-pool-" + count++);
                t.setPriority(Thread.MAX_PRIORITY);  // 设置最高优先级
                return t;
            }
        }
    );

    public CompletableFuture<CalculationResult> calculate(String taskId) {
        return CompletableFuture
            .supplyAsync(() -> performCalculation(taskId), cpuExecutor)
            .thenApply(this::aggregateResults);
    }
}
```

### 方案三：为不同类型任务隔离线程池

```java
public class IsolatedExecutorService {

    private final ExecutorService dbExecutor = createPool("db-pool", 20);
    private final ExecutorService httpExecutor = createPool("http-pool", 50);
    private final ExecutorService computeExecutor = createPool(
        "compute-pool",
        Runtime.getRuntime().availableProcessors()
    );

    private ExecutorService createPool(String name, int size) {
        return new ThreadPoolExecutor(
            size,  // corePoolSize
            size,  // maximumPoolSize
            0L, TimeUnit.MILLISECONDS,
            new LinkedBlockingQueue<>(1000),  // 有界队列，防止内存溢出
            new ThreadFactory() {
                private int count = 1;

                @Override
                public Thread newThread(Runnable r) {
                    Thread t = new Thread(r, name + "-" + count++);
                    t.setUncaughtExceptionHandler((thread, ex) ->
                        log.error("线程 {} 异常", thread.getName(), ex)
                    );
                    return t;
                }
            },
            new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略：调用者执行
        );
    }

    // 数据库操作
    public CompletableFuture<User> getUser(String userId) {
        return CompletableFuture
            .supplyAsync(() -> jdbcTemplate.queryForObject(
                "SELECT * FROM users WHERE id = ?",
                userRowMapper,
                userId
            ), dbExecutor)
            .exceptionally(ex -> {
                log.error("查询用户失败: {}", userId, ex);
                return User.ANONYMOUS;
            });
    }

    // HTTP 调用
    public CompletableFuture<ExternalData> fetchExternalData(String url) {
        return CompletableFuture
            .supplyAsync(() -> restTemplate.getForObject(url, ExternalData.class), httpExecutor)
            .orTimeout(5, TimeUnit.SECONDS)
            .exceptionally(ex -> {
                log.warn("获取外部数据失败: {}", url);
                return ExternalData.EMPTY;
            });
    }

    // 计算任务
    public CompletableFuture<Report> generateReport(String reportId) {
        return CompletableFuture
            .supplyAsync(() -> performHeavyCalculation(reportId), computeExecutor)
            .thenApply(this::formatReport);
    }
}
```

## 线程池参数调优指南

### 核心参数说明

```java
new ThreadPoolExecutor(
    corePoolSize,        // 核心线程数
    maximumPoolSize,     // 最大线程数
    keepAliveTime,       // 空闲线程存活时间
    unit,                // 时间单位
    workQueue,           // 任务队列
    threadFactory,       // 线程工厂
    handler              // 拒绝策略
);
```

### 不同场景的参数配置

```java
public class ThreadPoolConfig {

    /**
     * CPU 密集型任务配置
     * 线程数 = CPU 核心数 + 1
     * 原因：CPU 密集型任务几乎 100% 占用 CPU，额外线程可以在 CPU 等待时起作用
     */
    public ExecutorService cpuIntensivePool() {
        int threads = Runtime.getRuntime().availableProcessors() + 1;
        return new ThreadPoolExecutor(
            threads, threads,
            0L, TimeUnit.MILLISECONDS,
            new LinkedBlockingQueue<>(),
            new DefaultThreadFactory("cpu")
        );
    }

    /**
     * I/O 密集型任务配置
     * 线程数 = CPU 核心数 * 2 或更多
     * 原因：I/O 密集型任务大部分时间在等待，使用更多线程可以提高并发度
     */
    public ExecutorService ioIntensivePool() {
        int cores = Runtime.getRuntime().availableProcessors();
        int threads = cores * 2;
        return new ThreadPoolExecutor(
            cores, threads,
            60L, TimeUnit.SECONDS,  // 较长的空闲存活时间
            new LinkedBlockingQueue<>(1000),
            new DefaultThreadFactory("io")
        );
    }

    /**
     * 混合型任务配置
     * 使用更大的队列和更高的最大线程数
     */
    public ExecutorService mixedPool() {
        int cores = Runtime.getRuntime().availableProcessors();
        return new ThreadPoolExecutor(
            cores,
            cores * 4,
            60L, TimeUnit.SECONDS,
            new SynchronousQueue<>(),  // 直接提交模式
            new DefaultThreadFactory("mixed"),
            new ThreadPoolExecutor.CallerRunsPolicy()
        );
    }
}
```

## 避免的常见错误

### 错误一：使用 Executors 创建线程池

```java
// 错误：OOM 的潜在风险
ExecutorService cached = Executors.newCachedThreadPool();  // 无限线程
ExecutorService single = Executors.newSingleThreadExecutor();  // 队列无限大

// 正确：使用有界队列，设置合理的线程数
ExecutorService safe = new ThreadPoolExecutor(
    10, 50,
    60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(1000)  // 有界队列，防止 OOM
);
```

### 错误二：不关闭线程池

```java
// 错误：线程池泄漏
public class BadService {
    private final ExecutorService executor = Executors.newFixedThreadPool(10);

    public void process() {
        executor.submit(() -> doWork());  // 如果不调用 shutdown，线程池永远不会关闭
    }
}

// 正确：实现优雅关闭
public class GoodService implements AutoCloseable {
    private final ExecutorService executor = Executors.newFixedThreadPool(10);

    @Override
    public void close() {
        executor.shutdown();
        try {
            if (!executor.awaitTermination(30, TimeUnit.SECONDS)) {
                executor.shutdownNow();
            }
        } catch (InterruptedException e) {
            executor.shutdownNow();
            Thread.currentThread().interrupt();
        }
    }
}
```

### 错误三：忘记异常处理

```java
// 错误：异常被吞掉
CompletableFuture.supplyAsync(() -> riskyOperation());

// 正确：添加异常处理
CompletableFuture.supplyAsync(() -> riskyOperation())
    .exceptionally(ex -> {
        log.error("操作失败", ex);
        return defaultValue;
    });
```

## 监控与诊断

### 添加监控指标

```java
public class MonitoredExecutor implements ExecutorService {

    private final ExecutorService delegate;
    private final MeterRegistry meterRegistry;

    public MonitoredExecutor(ExecutorService delegate, MeterRegistry meterRegistry) {
        this.delegate = delegate;
        this.meterRegistry = meterRegistry;
    }

    @Override
    public <T> Future<T> submit(Runnable task, T result) {
        return delegate.submit(wrap(task), result);
    }

    // ... 其他方法类似

    private Runnable wrap(Runnable task) {
        return () -> {
            long start = System.nanoTime();
            try {
                task.run();
            } finally {
                long duration = System.nanoTime() - start;
                meterRegistry.timer("task.duration").record(duration, TimeUnit.NANOSECONDS);
            }
        };
    }
}
```

### 线程池状态监控

```java
public class PoolMonitor {

    public void dumpPoolStatus(ExecutorService executor, String name) {
        if (executor instanceof ThreadPoolExecutor) {
            ThreadPoolExecutor tpe = (ThreadPoolExecutor) executor;
            System.out.printf("[%s] Pool Status:%n", name);
            System.out.printf("  Active: %d%n", tpe.getActiveCount());
            System.out.printf("  Completed: %d%n", tpe.getCompletedTaskCount());
            System.out.printf("  Queue size: %d%n", tpe.getQueue().size());
            System.out.printf("  Core pool: %d%n", tpe.getCorePoolSize());
            System.out.printf("  Max pool: %d%n", tpe.getMaximumPoolSize());
        }
    }
}
```

## 面试高频考点

**问题一：CompletableFuture 默认使用什么线程池？**

默认使用 `ForkJoinPool.commonPool()`，并行度为 `CPU 核心数 - 1`。

**问题二：为什么 I/O 密集型任务需要更大的线程池？**

I/O 密集型任务大部分时间在等待（网络请求、磁盘读写），而不是使用 CPU。更多的线程可以提高并发度，让 CPU 在等待期间处理其他任务。

**问题三：线程池过大或过小有什么问题？**

过大会导致线程竞争激烈，消耗过多内存，上下文切换开销增大。过小会导致任务排队，响应时间增加，CPU 利用率不足。

**问题四：如何选择合适的线程池类型？**

CPU 密集型使用 `FixedThreadPool`，线程数等于 CPU 核心数 + 1。I/O 密集型使用 `CachedThreadPool` 或自定义的 `ThreadPoolExecutor`，线程数 = CPU 核心数 * 2 或更多。混合型使用 `ScheduledThreadPoolExecutor`。

**问题五：CompletableFuture 如何使用自定义线程池？**

使用 `supplyAsync(Supplier, Executor)` 或 `thenApplyAsync(Function, Executor)` 传入自定义的 `ExecutorService`。

理解线程池的原理和调优方法，是写出高性能 `CompletableFuture` 代码的基础。线程池不是配置一次就完事的，需要根据实际负载持续调整。
