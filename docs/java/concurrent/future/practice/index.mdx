# CompletableFuture 实践：真实场景中的异步编程

理论终究要服务于实践。本章将通过几个真实的业务场景，展示如何使用 `CompletableFuture` 解决实际问题，包括并行查询、异步流水线、服务降级等常见模式。

## 场景一：并行查询优化

### 问题背景

用户详情页需要展示：用户基本信息、用户订单、用户积分、推荐商品。这些数据来自不同的服务，在同步串行调用时耗时可能是 100ms + 200ms + 50ms + 150ms = 500ms。

### 使用 CompletableFuture 并行化

```java
public CompletableFuture<UserProfile> getUserProfile(String userId) {
    // 并行启动所有查询
    CompletableFuture<User> userFuture = userService.getUser(userId);
    CompletableFuture<List<Order>> ordersFuture = orderService.getOrders(userId);
    CompletableFuture<Integer> pointsFuture = pointsService.getPoints(userId);
    CompletableFuture<List<Product>> recommendFuture = productService.getRecommendations(userId);
    
    // 等待所有查询完成
    return userFuture.thenCombine(ordersFuture, UserProfile::new)
        .thenCombine(pointsFuture, UserProfile::withPoints)
        .thenCombine(recommendFuture, UserProfile::withRecommendations);
}

// 使用
getUserProfile("U123")
    .thenAccept(profile -> renderProfile(profile))
    .exceptionally(ex -> {
        logger.error("获取用户资料失败", ex);
        return null;
    });
```

### 超时控制

```java
public CompletableFuture<UserProfile> getUserProfileWithTimeout(String userId) {
    CompletableFuture<User> userFuture = userService.getUser(userId)
        .orTimeout(3, TimeUnit.SECONDS);
    CompletableFuture<List<Order>> ordersFuture = orderService.getOrders(userId)
        .orTimeout(5, TimeUnit.SECONDS);
    CompletableFuture<Integer> pointsFuture = pointsService.getPoints(userId)
        .orTimeout(2, TimeUnit.SECONDS);
    CompletableFuture<List<Product>> recommendFuture = productService.getRecommendations(userId)
        .orTimeout(3, TimeUnit.SECONDS);
    
    return CompletableFuture.allOf(userFuture, ordersFuture, pointsFuture, recommendFuture)
        .thenApply(ignored -> {
            try {
                return new UserProfile(
                    userFuture.join(),
                    ordersFuture.join(),
                    pointsFuture.join(),
                    recommendFuture.join()
                );
            } catch (Exception e) {
                throw new CompletionException(e);
            }
        })
        .exceptionally(ex -> {
            logger.error("获取用户资料失败", ex);
            return UserProfile.empty(userId);
        });
}
```

## 场景二：异步流水线处理

### 问题背景

订单处理需要经过：数据校验 -> 库存扣减 -> 支付扣款 -> 发货通知。每个步骤可能失败，需要支持回滚。

### 流水线实现

```java
public CompletableFuture<OrderResult> processOrder(Order order) {
    return CompletableFuture
        .supplyAsync(() -> validateOrder(order))        // 步骤1：校验
        .thenCompose(this::deductInventory)             // 步骤2：扣减库存
        .thenCompose(this::processPayment)             // 步骤3：支付扣款
        .thenCompose(this::sendNotification)           // 步骤4：发送通知
        .exceptionallyCompose(ex -> {
            // 任何步骤失败，执行补偿/回滚
            return compensate(order, ex);
        });
}

private CompletableFuture<Order> validateOrder(Order order) {
    if (order.getItems().isEmpty()) {
        return CompletableFuture.failedFuture(
            new BusinessException("订单不能为空")
        );
    }
    return CompletableFuture.completedFuture(order);
}

private CompletableFuture<Order> deductInventory(Order order) {
    return CompletableFuture.supplyAsync(() -> {
        for (OrderItem item : order.getItems()) {
            if (!inventoryService.deduct(item.getProductId(), item.getQuantity())) {
                throw new BusinessException("库存不足: " + item.getProductId());
            }
        }
        return order;
    });
}

private CompletableFuture<Order> processPayment(Order order) {
    return paymentService.charge(order.getUserId(), order.getTotalAmount())
        .thenApply(paymentId -> {
            order.setPaymentId(paymentId);
            return order;
        });
}

private CompletableFuture<Order> sendNotification(Order order) {
    return notificationService.sendOrderConfirmation(order)
        .thenApply(notificationId -> order);
}

private CompletableFuture<OrderResult> compensate(Order order, Throwable ex) {
    logger.warn("订单处理失败，开始补偿: {}", ex.getMessage());
    
    CompletableFuture<Void> inventoryRollback = CompletableFuture.completedFuture(null);
    CompletableFuture<Void> paymentRollback = CompletableFuture.completedFuture(null);
    
    // 补偿库存（如果已扣减）
    if (order.getItems() != null) {
        for (OrderItem item : order.getItems()) {
            inventoryRollback = inventoryRollback
                .thenCompose(v -> inventoryService.rollback(item.getProductId(), item.getQuantity()));
        }
    }
    
    // 补偿支付（如果已扣款）
    if (order.getPaymentId() != null) {
        paymentRollback = paymentService.refund(order.getPaymentId());
    }
    
    return CompletableFuture.allOf(inventoryRollback, paymentRollback)
        .thenApply(v -> OrderResult.failed(ex.getMessage()));
}
```

## 场景三：多级降级策略

### 问题背景

调用远程服务时，可能遇到服务不可用、网络超时等问题。需要实现多级降级：本地缓存 -> 降级服务 -> 默认值。

### 降级实现

```java
public CompletableFuture<ProductInfo> getProductInfo(String productId) {
    // 尝试从本地缓存获取
    return CompletableFuture
        .supplyAsync(() -> localCache.get(productId))
        .thenCompose(cached -> {
            if (cached != null) {
                return CompletableFuture.completedFuture(cached);
            }
            return fetchFromRemote(productId);
        })
        .exceptionallyCompose(ex -> {
            logger.warn("获取商品信息失败，降级到本地数据源: {}", ex.getMessage());
            return fetchFromDatabase(productId);
        })
        .exceptionallyCompose(ex -> {
            logger.warn("数据库查询失败，使用默认数据: {}", ex.getMessage());
            return CompletableFuture.completedFuture(ProductInfo.defaultInfo(productId));
        })
        .thenApply(info -> {
            // 异步更新缓存
            localCache.put(productId, info);
            return info;
        });
}

private CompletableFuture<ProductInfo> fetchFromRemote(String productId) {
    return productService.getRemoteProduct(productId)
        .orTimeout(2, TimeUnit.SECONDS)
        .thenApply(this::convertToProductInfo)
        .exceptionally(ex -> {
            throw new CompletionException(ex);
        });
}

private CompletableFuture<ProductInfo> fetchFromDatabase(String productId) {
    return CompletableFuture.supplyAsync(() -> 
        database.query("SELECT * FROM products WHERE id = ?", productId)
    ).thenApply(this::convertToProductInfo);
}
```

## 场景四：批量处理与进度跟踪

### 问题背景

批量导入订单时，需要处理大量数据，同时提供进度反馈。

### 批量处理实现

```java
public CompletableFuture<BatchResult> importOrders(List<Order> orders, 
                                                   Consumer<BatchProgress> progressCallback) {
    AtomicInteger processed = new AtomicInteger(0);
    AtomicInteger success = new AtomicInteger(0);
    AtomicInteger failed = new AtomicInteger(0);
    List<String> errors = Collections.synchronizedList(new ArrayList<>());
    
    ExecutorService executor = Executors.newFixedThreadPool(10);
    
    List<CompletableFuture<OrderImportResult>> futures = orders.stream()
        .map(order -> CompletableFuture
            .supplyAsync(() -> processOrderImport(order), executor)
            .whenComplete((result, ex) -> {
                int index = processed.incrementAndGet();
                if (ex != null) {
                    failed.incrementAndGet();
                    errors.add(order.getId() + ": " + ex.getMessage());
                } else {
                    success.incrementAndGet();
                }
                // 回调进度
                progressCallback.accept(new BatchProgress(
                    orders.size(), index, success.get(), failed.get()
                ));
            })
        )
        .collect(Collectors.toList());
    
    return CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
        .thenApply(ignored -> {
            executor.shutdown();
            return new BatchResult(
                orders.size(),
                success.get(),
                failed.get(),
                errors
            );
        })
        .exceptionally(ex -> {
            executor.shutdown();
            return new BatchResult(orders.size(), 0, orders.size(), 
                Collections.singletonList(ex.getMessage()));
        });
}

// 使用
importOrders(orders, progress -> {
    System.out.printf("进度: %d/%d (成功: %d, 失败: %d)%n",
        progress.processed, progress.total, 
        progress.success, progress.failed);
}).thenAccept(result -> {
    System.out.println("导入完成: " + result);
});
```

## 场景五：竞态查询（请求合并）

### 问题背景

高并发下，同一商品的详情请求可能同时发起多次，造成资源浪费。需要合并相同请求。

### 请求合并实现

```java
public class RequestDeduplicator<K, V> {
    private final ConcurrentHashMap<K, CompletableFuture<V>> inFlight = new ConcurrentHashMap<>();
    private final Function<K, V> computation;
    
    public RequestDeduplicator(Function<K, V> computation) {
        this.computation = computation;
    }
    
    public V apply(K key) {
        // 尝试获取已有的 Future
        CompletableFuture<V> existing = inFlight.get(key);
        if (existing != null) {
            return existing.join();  // 等待相同请求完成
        }
        
        // 创建新的 Future
        CompletableFuture<V> future = CompletableFuture.supplyAsync(() -> {
            try {
                return computation.apply(key);
            } finally {
                inFlight.remove(key);  // 完成后移除
            }
        });
        
        // 如果成功插入，返回新的 Future
        // 如果已存在，返回现有的 Future
        CompletableFuture<V> result = inFlight.merge(key, future, (a, b) -> a);
        
        return result.join();
    }
}

// 使用
private RequestDeduplicator<String, ProductDetail> productDeduplicator = 
    new RequestDeduplicator<>(this::fetchProductDetail);

public ProductDetail getProduct(String productId) {
    return productDeduplicator.apply(productId);
}
```

## 场景六：异步限流

### 问题背景

异步调用也需要限流，避免压垮下游服务。

### 令牌桶实现

```java
public <T> CompletableFuture<T> withRateLimit(
        Supplier<CompletableFuture<T>> action,
        int permitsPerSecond) {
    
    // 使用 Semaphore 作为简单的限流器
    Semaphore semaphore = new Semaphore(permitsPerSecond);
    
    return CompletableFuture.supplyAsync(() -> {
        try {
            if (!semaphore.tryAcquire(1, 1, TimeUnit.SECONDS)) {
                throw new RateLimitException("请求过于频繁");
            }
            return action.get().get(10, TimeUnit.SECONDS);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RateLimitException("请求被中断");
        } catch (TimeoutException e) {
            throw new RateLimitException("请求超时");
        } finally {
            semaphore.release();
        }
    });
}

// 使用
CompletableFuture<Order> future = withRateLimit(
    () -> orderService.createOrder(order),
    100  // 每秒最多 100 个请求
);
```

## 本章小结

`CompletableFuture` 的实际应用远不止这些。关键是要理解它的核心能力：

1. **并行执行**：使用 `allOf` 或分别启动多个 Future
2. **链式处理**：使用 `thenCompose` 构建异步流水线
3. **异常处理**：使用 `exceptionally` 和 `handle` 实现降级
4. **超时控制**：使用 `orTimeout` 防止无限等待
5. **请求合并**：使用 `ConcurrentHashMap` 减少重复请求
6. **限流保护**：结合 `Semaphore` 实现异步限流

实际项目中，往往需要组合使用这些技术。异步编程虽然增加了复杂度，但带来的性能提升是显著的。你在项目中是否有过类似的实践？
