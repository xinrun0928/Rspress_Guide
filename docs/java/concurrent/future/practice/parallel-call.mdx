# 多服务并行调用：thenCompose 链式编排实战

在电商系统中，用户打开商品详情页可能需要加载：商品基本信息、用户评价、库存状态、推荐商品、优惠券信息……如果这些接口串行调用，一个接口 100ms，5 个接口就是 500ms 的等待时间。用户体验？那叫一个糟糕。但如果并行调用这些接口，等待时间就降到了 100ms 左右。`thenCompose` 就是实现这种并行编排的关键工具。

## 串行 vs 并行：性能差距有多大

先看一个简单的对比：

```java
public class SerialVsParallel {
    public void serialCalls() {
        long start = System.currentTimeMillis();

        // 串行：每个服务等待前一个完成才开始
        String user = getUserInfo();        // 100ms
        String orders = getUserOrders(user); // 200ms
        String recommendations = getRecommendations(orders); // 150ms

        System.out.println("串行耗时: " + (System.currentTimeMillis() - start));
        // 总耗时约 450ms
    }

    public void parallelCalls() {
        long start = System.currentTimeMillis();

        // 并行：所有服务同时开始
        CompletableFuture<String> userFuture = CompletableFuture.supplyAsync(this::getUserInfo);
        CompletableFuture<String> ordersFuture = CompletableFuture.supplyAsync(this::getUserOrders);
        CompletableFuture<String> recommendationsFuture = CompletableFuture.supplyAsync(this::getRecommendations);

        // 等待所有完成
        String user = userFuture.join();
        String orders = ordersFuture.join();
        String recommendations = recommendationsFuture.join();

        System.out.println("并行耗时: " + (System.currentTimeMillis() - start));
        // 总耗时约 200ms（最长那个）
    }

    // 模拟服务调用
    private String getUserInfo() { sleep(100); return "user"; }
    private String getUserOrders(String user) { sleep(200); return "orders"; }
    private String getRecommendations(String orders) { sleep(150); return "recommendations"; }

    private void sleep(int ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) {}
    }
}
```

串行需要 450ms，并行只需要 200ms，性能提升超过 50%。这就是并行调用的威力。

## thenCompose：处理有依赖的链式调用

但是，实际业务中并不是所有接口都可以并行。有些接口必须等待前一个接口返回后才能调用——比如获取用户信息后才能获取用户订单，因为订单需要用户 ID。这时就需要 `thenCompose`。

### thenCompose vs thenApply：区别是什么

理解 `thenCompose` 首先要理解它和 `thenApply` 的区别：

```java
// thenApply：变换结果，返回值直接是结果
CompletableFuture<Integer> future1 = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenApply(s -> s.length());  // String -> Integer

// thenCompose：变换结果，返回值是另一个 Future
CompletableFuture<Integer> future2 = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenCompose(s -> CompletableFuture.supplyAsync(() -> s.length()));  // 返回 CF<Integer>
```

关键区别在于：`thenApply` 用于同步变换，`thenCompose` 用于异步链式调用。当你需要在当前异步操作完成后，启动另一个异步操作时，用 `thenCompose`。

### thenCompose 实战：用户中心编排

```java
public class UserCenterService {

    public CompletableFuture<UserDashboard> getUserDashboard(String userId) {
        return CompletableFuture
            // 第一步：获取用户基本信息
            .supplyAsync(() -> queryUserInfo(userId), executor)
            .thenCompose(user -> {
                // userInfo 返回后才执行第二步
                // 这里返回一个新的 CompletableFuture，代表后续的异步操作
                return CompletableFuture
                    // 第二步：并行获取用户订单和收藏
                    .allOf(
                        CompletableFuture.supplyAsync(() -> queryUserOrders(user.getId()), executor),
                        CompletableFuture.supplyAsync(() -> queryUserFavorites(user.getId()), executor),
                        CompletableFuture.supplyAsync(() -> queryUserCoupons(user.getId()), executor)
                    )
                    .thenCompose(v -> {
                        // 所有并行任务完成后，组装数据
                        List<Order> orders = orderFuture.join();
                        List<Product> favorites = favoriteFuture.join();
                        List<Coupon> coupons = couponFuture.join();

                        return CompletableFuture.completedFuture(
                            new UserDashboard(user, orders, favorites, coupons)
                        );
                    });
            })
            .exceptionally(ex -> {
                log.error("获取用户仪表盘失败: {}", ex.getMessage());
                return UserDashboard.empty();
            });
    }

    private UserInfo queryUserInfo(String userId) { /* ... */ return new UserInfo(); }
    private List<Order> queryUserOrders(String userId) { /* ... */ return new ArrayList<>(); }
    private List<Product> queryUserFavorites(String userId) { /* ... */ return new ArrayList<>(); }
    private List<Coupon> queryUserCoupons(String userId) { /* ... */ return new ArrayList<>(); }
}
```

## 实战场景：订单服务编排

一个典型的订单创建流程需要多个步骤，每个步骤可能依赖前一步的结果：

```java
public class OrderService {

    public CompletableFuture<OrderResult> createOrder(OrderRequest request) {
        return CompletableFuture
            .supplyAsync(() -> validateRequest(request), executor)
            .thenCompose(validation -> {
                if (!validation.isValid()) {
                    // 验证失败，返回失败结果
                    return CompletableFuture.completedFuture(
                        OrderResult.failed(validation.getErrorMessage())
                    );
                }
                // 验证通过，继续下一步
                return checkInventory(request);
            })
            .thenCompose(inventoryResult -> {
                if (!inventoryResult.hasStock()) {
                    // 库存不足
                    return CompletableFuture.completedFuture(
                        OrderResult.failed("库存不足")
                    );
                }
                // 库存充足，锁定库存并计算价格
                return lockInventoryAndCalculatePrice(request, inventoryResult);
            })
            .thenCompose(priceResult -> {
                // 价格计算完成，创建订单
                return createOrderRecord(request, priceResult);
            })
            .thenCompose(orderRecord -> {
                // 订单创建完成，发送消息通知
                return sendOrderNotification(orderRecord)
                    .thenApply(notificationSent -> orderRecord);
            })
            .exceptionally(ex -> {
                log.error("订单创建失败", ex);
                return OrderResult.failed("订单创建失败: " + ex.getMessage());
            });
    }

    private ValidationResult validateRequest(OrderRequest request) { /* ... */ return new ValidationResult(); }
    private InventoryResult checkInventory(OrderRequest request) { /* ... */ return new InventoryResult(); }
    private PriceResult lockInventoryAndCalculatePrice(OrderRequest request, InventoryResult inventory) { /* ... */ return new PriceResult(); }
    private OrderRecord createOrderRecord(OrderRequest request, PriceResult price) { /* ... */ return new OrderRecord(); }
    private CompletableFuture<Boolean> sendOrderNotification(OrderRecord order) { /* ... */ return CompletableFuture.completedFuture(true); }
}
```

这个例子展示了 `thenCompose` 的强大能力：它让异步代码的阅读体验接近同步代码，同时保持了非阻塞的特性。

## thenCompose 与并行分支的组合

更复杂的场景中，`thenCompose` 可以与 `allOf` 结合，实现"先串后并再串"的复杂编排：

```java
public CompletableFuture<ProductDetail> getProductDetail(String productId) {
    return CompletableFuture
        .supplyAsync(() -> getProductBasicInfo(productId), executor)
        .thenCompose(basicInfo ->
            // 获取基础信息后，并行获取详情、评价、库存
            CompletableFuture.allOf(
                CompletableFuture.supplyAsync(() -> getProductDescription(productId), executor),
                CompletableFuture.supplyAsync(() -> getProductReviews(productId), executor),
                CompletableFuture.supplyAsync(() -> getProductStock(productId), executor)
            ).thenApply(v ->
                new ProductDetail(
                    basicInfo,
                    descriptionFuture.join(),
                    reviewsFuture.join(),
                    stockFuture.join()
                )
            )
        );
}
```

## 避免 thenCompose 误用

### 误用一：thenApply 代替 thenCompose

```java
// 错误：thenApply 返回的不是 CompletableFuture
CompletableFuture.supplyAsync(() -> queryData())
    .thenApply(data -> CompletableFuture.supplyAsync(() -> process(data)))
    // 返回的是 CompletableFuture<CompletableFuture<T>>，需要额外处理

// 正确：使用 thenCompose
CompletableFuture.supplyAsync(() -> queryData())
    .thenCompose(data -> CompletableFuture.supplyAsync(() -> process(data)))
    // 返回的是 CompletableFuture<T>
```

### 误用二：忘记处理异常

`thenCompose` 链中任何一个环节抛出异常，都会跳过后续的 `thenCompose`，直接进入 `exceptionally`。这通常是期望的行为，但如果你需要在某个环节做特殊处理：

```java
.thenCompose(result -> {
    if (result.requiresSpecialHandling()) {
        // 需要特殊处理，启动另一个异步流程
        return handleSpecially(result);
    }
    // 正常流程继续
    return CompletableFuture.completedFuture(normalProcess(result));
})
```

## 面试高频考点

**问题一：thenCompose 和 thenApply 的区别是什么？**

`thenApply` 用于同步转换，输入是 T，输出是 U，返回 `CompletableFuture<U>`。`thenCompose` 用于异步链式调用，输入是 T，输出是 `CompletableFuture<U>`，返回 `CompletableFuture<U>`。简单说，`thenCompose` 用于"异步任务完成后启动另一个异步任务"的场景。

**问题二：thenCompose 和 thenCombine 的区别是什么？**

`thenCompose` 用于处理依赖关系（A 完成后执行 B）。`thenCombine` 用于合并两个独立的异步任务（两个任务同时执行，完成后合并结果）。

**问题三：thenCompose 链中如何处理部分失败？**

默认情况下，`thenCompose` 链中任何环节抛出异常都会跳过后续环节。如果需要在某个环节做容错处理，可以在那个环节内部使用 `exceptionally` 返回一个默认值或使用 `handle` 做兜底。

**问题四：thenCompose 的性能考虑有哪些？**

`thenCompose` 本身几乎没有性能开销，但需要注意：链不要太长（增加调试难度和异常传播层次），考虑使用 `allOf` 并行化独立的异步任务，避免在 `thenCompose` 中做耗时的同步操作。

`thenCompose` 是 `CompletableFuture` 中最强大的组合方法之一。掌握它，就能写出既简洁又高效的异步代码。
