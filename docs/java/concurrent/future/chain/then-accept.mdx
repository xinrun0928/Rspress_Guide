# thenAccept / thenRun：结果消费与动作执行

`thenAccept` 和 `thenRun` 是 `CompletableFuture` 中用于消费结果的方法。两者都会等待前一个阶段完成，但处理方式不同。

## thenAccept：消费结果

```java
public class ThenAcceptDemo {
    
    public void thenAccept() {
        CompletableFuture.supplyAsync(() -> {
            return "Hello";
        })
        .thenAccept(result -> {
            // 消费前一个阶段的结果
            System.out.println("收到结果: " + result);
        })
        .join();
    }
    
    // 链式消费
    public void chainedAccept() {
        CompletableFuture.supplyAsync(() -> 100)
            .thenAccept(x -> System.out.println("步骤1: " + x))
            .thenAccept(x -> System.out.println("步骤2: " + (x * 2)))
            .join();
    }
}
```

## thenRun：不关心结果

```java
public class ThenRunDemo {
    
    public void thenRun() {
        CompletableFuture.supplyAsync(() -> {
            return "result";
        })
        .thenRun(() -> {
            // 不关心前一个阶段的结果
            System.out.println("前一个阶段完成");
        })
        .join();
    }
    
    // thenRun 访问不到前一个阶段的结果
    public void thenRunVsAccept() {
        CompletableFuture.supplyAsync(() -> "data")
            .thenAccept(data -> {
                // data 可用
                System.out.println("收到: " + data);
            })
            .thenRun(() -> {
                // 前一个阶段的结果不可用
                System.out.println("只关心完成状态");
            })
            .join();
    }
}
```

## 区别对比

```java
public class Comparison {
    
    public void demo() {
        // thenAccept: 接收前一个阶段的结果
        CompletableFuture.supplyAsync(() -> "result")
            .thenAccept(data -> System.out.println(data));  // data = "result"
        
        // thenRun: 不接收结果
        CompletableFuture.supplyAsync(() -> "result")
            .thenRun(() -> System.out.println("完成"));  // 无法访问 "result"
    }
}
```

## 异常处理

```java
public class ExceptionHandling {
    
    public void demo() {
        CompletableFuture.supplyAsync(() -> {
            throw new RuntimeException("错误");
        })
        .thenAccept(data -> {
            // 前一个阶段异常，这里不会执行
        })
        .exceptionally(ex -> {
            // 需要在这里处理异常
            System.err.println("处理异常: " + ex.getMessage());
            return null;
        });
    }
}
```

## 总结

| 方法 | 参数 | 使用场景 |
|------|------|---------|
| `thenAccept` | Consumer | 需要使用前一个阶段的结果 |
| `thenRun` | Runnable | 不需要结果，只关心完成 |
