# Future 链式调用：异步流水线的构建艺术

`CompletableFuture` 相比传统 `Future` 最强大的特性，就是支持链式调用。通过一系列的 `then*` 方法，我们可以将多个异步操作串联成一条流水线，让代码既保持同步的清晰易读，又拥有异步的高性能。本章将深入探讨各种链式调用的用法和使用场景。

## thenApply：结果转换

`thenApply` 是最基础的转换方法，它接收一个函数，将前一个 `CompletableFuture` 的结果转换为另一个值：

```java
// 语法
<U> CompletableFuture<U> thenApply(Function<? super T, ? extends U> fn);
<U> CompletableFuture<U> thenApplyAsync(Function<? super T, ? extends U> fn);

// 示例
CompletableFuture<String> f1 = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenApply(s -> s + " world");  // "hello world"
```

### thenApply vs thenApplyAsync

```java
// thenApply：在同一个线程中执行
CompletableFuture<String> f1 = CompletableFuture
    .supplyAsync(() -> {
        System.out.println("step1: " + Thread.currentThread().getName());
        return "hello";
    })
    .thenApply(s -> {
        System.out.println("step2: " + Thread.currentThread().getName());
        return s + " world";
    });

// 输出：
// step1: ForkJoinPool.commonPool-worker-1
// step2: ForkJoinPool.commonPool-worker-1  <- 同一线程

// thenApplyAsync：在新线程中执行
CompletableFuture<String> f2 = CompletableFuture
    .supplyAsync(() -> {
        System.out.println("step1: " + Thread.currentThread().getName());
        return "hello";
    })
    .thenApplyAsync(s -> {
        System.out.println("step2: " + Thread.currentThread().getName());
        return s + " world";
    });

// 输出：
// step1: ForkJoinPool.commonPool-worker-1
// step2: ForkJoinPool.commonPool-worker-2  <- 不同线程
```

选择 `thenApply` 还是 `thenApplyAsync` 取决于你的需求：
- `thenApply`：同步执行，如果转换操作很快，可以减少线程切换开销
- `thenApplyAsync`：异步执行，如果转换操作耗时，可以让调用线程继续做其他事

## thenCompose：扁平化嵌套 Future

这是最容易混淆的方法。`thenCompose` 用于处理返回 `CompletableFuture` 的函数，避免产生嵌套结构：

```java
// 假设有这两个方法
CompletableFuture<User> getUser(String id);
CompletableFuture<Order> getOrders(String userId);  // 返回 CompletableFuture

// 使用 thenApply（错误）：产生嵌套
CompletableFuture<CompletableFuture<Order>> nested = 
    getUser("123")
        .thenApply(user -> getOrders(user.getId()));

// 使用 thenCompose（正确）：扁平化
CompletableFuture<Order> flat = 
    getUser("123")
        .thenCompose(user -> getOrders(user.getId()));
```

### thenCompose 的语义

`thenCompose` 的函数签名为：
```java
<U> CompletableFuture<U> thenCompose(Function<? super T, ? extends CompletableFuture<? extends U>> fn);
```

它类似于函数式编程中的 `flatMap`：
```
flatMap: (T -> Future<U>) -> Future<U>
```

### 实际应用：异步调用链

```java
CompletableFuture<User> future = userService.findById(userId)
    .thenCompose(user -> {
        if (user == null) {
            return CompletableFuture.failedFuture(new UserNotFoundException());
        }
        return orderService.findByUserId(user.getId());
    })
    .thenCompose(orders -> {
        if (orders.isEmpty()) {
            return CompletableFuture.failedFuture(new OrderNotFoundException());
        }
        return paymentService.calculateTotal(orders);
    })
    .thenCompose(total -> discountService.applyDiscount(total, "SUMMER2024"));
```

## thenCombine：合并两个 Future

当需要组合两个独立执行的 `CompletableFuture` 时，使用 `thenCombine`：

```java
// 语法
<U, V> CompletableFuture<V> thenCombine(
    CompletableFuture<? extends U> other,
    BiFunction<? super T, ? super U, ? extends V> fn
);

// 示例
CompletableFuture<String> name = CompletableFuture.supplyAsync(() -> "Alice");
CompletableFuture<Integer> age = CompletableFuture.supplyAsync(() -> 30);

CompletableFuture<String> profile = name.thenCombine(age, (n, a) -> n + " is " + a + " years old");
// 结果: "Alice is 30 years old"
```

### thenCombine vs thenCompose

```
thenCombine:  Future<T> + Future<U> -> Future<V>
              合并两个 Future 的结果

thenCompose:  Future<T> + (T -> Future<U>) -> Future<U>
              使用第一个 Future 的结果生成新的 Future
```

### 实际应用：聚合多个数据源

```java
CompletableFuture<User> userFuture = userService.findById(userId);
CompletableFuture<List<Order>> ordersFuture = orderService.findByUserId(userId);
CompletableFuture<BigDecimal> balanceFuture = accountService.getBalance(userId);

CompletableFuture<UserProfile> profileFuture = userFuture
    .thenCombine(ordersFuture, UserProfile::new)
    .thenCombine(balanceFuture, UserProfile::withBalance);
```

## allOf 与 anyOf：批量组合

### allOf：等待所有任务完成

```java
List<CompletableFuture<String>> futures = Arrays.asList(
    CompletableFuture.supplyAsync(() -> fetchFromService1()),
    CompletableFuture.supplyAsync(() -> fetchFromService2()),
    CompletableFuture.supplyAsync(() -> fetchFromService3())
);

// 等待所有完成
CompletableFuture<Void> allDone = CompletableFuture.allOf(
    futures.toArray(new CompletableFuture[0])
);

allDone.thenRun(() -> {
    // 所有任务都完成了
    List<String> results = futures.stream()
        .map(CompletableFuture::join)
        .collect(Collectors.toList());
});
```

### anyOf：取最快的结果

```java
CompletableFuture<String> fastest = CompletableFuture.anyOf(
    CompletableFuture.supplyAsync(() -> queryDatabase()),
    CompletableFuture.supplyAsync(() -> queryCache()),
    CompletableFuture.supplyAsync(() -> queryRemote())
).thenApply(result -> (String) result);
```

## thenAccept 与 thenRun：终端操作

### thenAccept：消费结果但不返回

```java
CompletableFuture.supplyAsync(() -> "Hello")
    .thenAccept(System.out::println);  // 打印 "Hello"

// 接收两个参数
CompletableFuture<String> name = CompletableFuture.supplyAsync(() -> "Alice");
CompletableFuture<Integer> age = CompletableFuture.supplyAsync(() -> 30);

name.thenAcceptBoth(age, (n, a) -> 
    System.out.println(n + " is " + a + " years old")
);
```

### thenRun：不关心结果

```java
CompletableFuture.supplyAsync(() -> doCalculation())
    .thenRun(() -> System.out.println("计算完成"));  // 只关心是否完成
```

## 方法选择指南

```
需要做什么？
├── 转换值 -> thenApply / thenApplyAsync
├── 扁平化嵌套 -> thenCompose
├── 合并两个结果 -> thenCombine
├── 等待多个完成 -> allOf
├── 取最快结果 -> anyOf
├── 消费结果 -> thenAccept / thenAcceptBoth
└── 执行后续操作 -> thenRun
```

## 性能优化：避免不必要的异步

### 短操作用同步

```java
// 不必要：为简单操作创建异步开销
CompletableFuture.supplyAsync(() -> "hello")
    .thenApply(s -> s.toUpperCase())
    .thenApply(s -> s + "!");

// 更好：使用同步方法链
CompletableFuture.completedFuture("hello")
    .thenApply(s -> s.toUpperCase())
    .thenApply(s -> s + "!");
```

### 正确拆分同步和异步

```java
// 假设 validation 是同步的，fetchData 是异步的
CompletableFuture
    .completedFuture(request)
    .thenApply(this::validateRequest)  // 同步
    .thenApplyAsync(this::fetchData)   // 异步
    .thenApply(this::processResponse); // 同步
```

## 链式调用的取消

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> doLongTimeWork())
    .thenApply(this::processResult);

// 取消
future.cancel(true);  // mayInterruptIfRunning = true
```

取消后，后续的 `thenApply` 不会执行，`get()` 会抛出 `CancellationException`。

## 本章小结

链式调用是 `CompletableFuture` 最强大的特性。通过合理组合 `thenApply`、`thenCompose`、`thenCombine` 等方法，我们可以构建出清晰、高效的异步流水线。记住几个关键点：

1. `thenApply` 转换值，`thenCompose` 扁平化
2. `thenCombine` 合并两个独立 Future 的结果
3. 选择同步还是异步版本取决于具体场景
4. 避免不必要的异步开销
