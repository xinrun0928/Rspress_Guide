# unchecked 异常与 CancellationException 处理

`CompletableFuture` 对异常的处理有特殊规则，理解这些规则对于正确处理异步异常至关重要。

## 异常类型

```java
public class ExceptionTypes {
    
    public void exceptionTypes() {
        // 1. unchecked 异常（RuntimeException, Error）
        // 会被包装在 CompletionException 中
        CompletableFuture.supplyAsync(() -> {
            throw new RuntimeException("unchecked");
        })
        .exceptionally(ex -> {
            // ex 是 CompletionException
            System.err.println(ex.getCause().getMessage());  // unchecked
            return null;
        });
        
        // 2. checked 异常
        // 会被直接抛出
        CompletableFuture.supplyAsync(() -> {
            throw new IOException("checked");
        })
        .exceptionally(ex -> {
            // ex 可能是 IOException
            return null;
        });
        
        // 3. CancellationException
        // 通过 cancel() 方法触发
        CompletableFuture<String> future = new CompletableFuture<>();
        future.cancel(true);
    }
}
```

## CancellationException

```java
public class CancellationDemo {
    
    public void demo() {
        CompletableFuture<String> future = new CompletableFuture<>();
        
        future.cancel(true);
        
        future.exceptionally(ex -> {
            if (ex instanceof CancellationException) {
                System.out.println("任务被取消");
            }
            return null;
        });
    }
}
```

## 总结

1. **unchecked 异常**：包装在 `CompletionException` 中
2. **CancellationException**：通过 `cancel()` 触发
