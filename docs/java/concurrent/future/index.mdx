# Future 与异步编程：跨越时间的桥梁

在单线程顺序执行的世界里，一切都那么简单直接——你调用一个方法，等待它返回结果，然后继续下一步。但在多线程时代，事情变得复杂了：我们需要启动一个异步任务，不阻塞当前线程，同时又能获取任务的结果。`Future` 就是解决这个问题的方案，它像是跨越时间的桥梁——你在"现在"提交任务，在"未来"获取结果。

## Future 的诞生：从 Runnable 到 Callable

在 `Future` 出现之前，`Runnable` 是创建线程的唯一选择，但 `Runnable` 不能返回结果，也不能抛出受检异常：

```java
// Runnable 无法获取返回值
new Thread(() -> {
    Object result = doLongTimeWork();  // 结果只能内部处理
}).start();
```

`Callable` 接口解决了这个问题：

```java
public interface Callable<V> {
    V call() throws Exception;  // 可以返回结果，可以抛出异常
}
```

但 `Callable` 如何与 `Thread` 配合？`Thread` 只接受 `Runnable`。这就引出了 `FutureTask`——它是连接 `Callable` 和 `Thread` 的桥梁：

```java
public class FutureTask<V> implements RunnableFuture<V> {
    
    private Callable<V> callable;
    private Object outcome;  // 存储执行结果
    
    public FutureTask(Callable<V> callable) {
        this.callable = callable;
    }
    
    public void run() {
        if (state != NEW ||
            !UNSAFE.compareAndSwapObject(this, runOffset, null, Thread.currentThread()))
            return;
        try {
            Callable<V> c = callable;
            if (c != null && state == NEW) {
                V result;
                boolean ran;
                try {
                    result = c.call();  // 执行任务
                    ran = true;
                } catch (Throwable ex) {
                    result = null;
                    ran = false;
                    setException(ex);  // 设置异常
                }
                if (ran)
                    set(result);  // 设置结果
            }
        } finally {
            callable = null;
        }
    }
}
```

## Future 接口：定义异步操作的结果契约

`Future` 接口定义了获取异步操作结果的规范：

```java
public interface Future<V> {
    // 获取结果，会阻塞直到结果可用
    V get() throws InterruptedException, ExecutionException;
    
    // 获取结果，最多等待指定时间
    V get(long timeout, TimeUnit unit) 
        throws InterruptedException, ExecutionException, TimeoutException;
    
    // 判断任务是否完成
    boolean isDone();
    
    // 取消任务
    boolean cancel(boolean mayInterruptIfRunning);
    
    // 判断任务是否被取消
    boolean isCancelled();
}
```

### Future 的使用示例

```java
ExecutorService executor = Executors.newFixedThreadPool(4);

// 提交 Callable 任务
Future<Integer> future = executor.submit(() -> {
    Thread.sleep(2000);
    return 42;
});

// 执行其他操作
doSomethingElse();

// 获取结果（会阻塞等待）
try {
    Integer result = future.get();  // 阻塞直到完成
    System.out.println("Result: " + result);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
} catch (ExecutionException e) {
    Throwable cause = e.getCause();  // 获取任务抛出的异常
    cause.printStackTrace();
}
```

### Future 的局限性

`Future` 虽然解决了异步返回值的问题，但有明显局限：

1. **阻塞式获取**：调用 `get()` 会阻塞当前线程
2. **无法组合**：不能将多个 Future 串联起来执行
3. **无法手动完成**：无法提前设置结果
4. **错误处理繁琐**：需要捕获 `ExecutionException`

## FutureTask：Future + Runnable

`FutureTask` 实现了 `RunnableFuture`，同时是 `Future` 和 `Runnable`：

```java
public interface RunnableFuture<V> extends Runnable, Future<V> {
    void run();
}
```

这让它可以直接被 `Thread` 或 `Executor` 执行：

```java
// 方式一：作为 Runnable 提交给 Executor
Future<Integer> f = executor.submit(new FutureTask<>(() -> 42));

// 方式二：直接用 Thread
FutureTask<Integer> task = new FutureTask<>(() -> 42);
new Thread(task).start();
Integer result = task.get();
```

## 实际应用场景

### 并行计算：MapReduce 模式

```java
ExecutorService executor = Executors.newFixedThreadPool(10);
List<Future<Integer>> futures = new ArrayList<>();

// 提交多个并行任务
for (Data data : dataset) {
    futures.add(executor.submit(() -> process(data)));
}

// 收集结果
int total = 0;
for (Future<Integer> f : futures) {
    total += f.get();  // 阻塞获取
}

executor.shutdown();
```

### 异步超时控制

```java
Future<String> future = executor.submit(() -> {
    return callRemoteService();
});

try {
    String result = future.get(3, TimeUnit.SECONDS);  // 最多等3秒
} catch (TimeoutException e) {
    future.cancel(true);  // 超时后取消任务
    return "超时了";
}
```

### 取消长时间运行的任务

```java
Future<?> future = executor.submit(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 执行任务
    }
});

// 在另一个线程中取消
future.cancel(true);  // true 表示如果任务正在运行，中断它
```

## Future 与 CompletableFuture

JDK 8 引入了 `CompletableFuture`，它是 `Future` 的扩展，提供了更强大的异步编程能力：

```java
// 创建已完成的任务
CompletableFuture<String> completed = CompletableFuture.completedFuture("done");

// 创建异步任务
CompletableFuture<Integer> async = CompletableFuture.supplyAsync(() -> 42);

// 链式操作
CompletableFuture<Integer> result = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenApply(String::toUpperCase)
    .thenApply(s -> s.length());
```

`CompletableFuture` 解决了 `Future` 的局限性，是现代 Java 异步编程的首选。
