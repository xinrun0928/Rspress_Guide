# Future 局限性：get() 阻塞与无法链式调用

`Future` 是 Java 并发编程的重要组件，但它有明显的局限性。在 JDK 8 引入 `CompletableFuture` 之前，开发者必须忍受这些限制。理解 `Future` 的局限性，才能更好地理解为什么需要 `CompletableFuture`。

## Future 的三大局限

### 局限一：阻塞式 get()

`Future.get()` 是阻塞的，要么一直等待，要么等待超时。在等待期间，调用线程什么都做不了。

```java
public class BlockingGetProblem {
    
    public void blockingGetDemo() throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        
        Future<Integer> future = executor.submit(() -> {
            Thread.sleep(3000);
            return 42;
        });
        
        System.out.println("任务已提交...");
        
        // 这里会阻塞，直到任务完成
        Integer result = future.get();  // 阻塞 3 秒
        
        System.out.println("得到结果: " + result);
    }
    
    // 问题：调用线程在等待期间无法做任何事
    // 理想情况：让调用线程在等待期间执行其他任务
}
```

### 局限二：无法链式调用

`Future` 没有提供链式调用的 API。多个 Future 之间的依赖关系很难表达。

```java
public class NoChainProblem {
    
    public void noChainDemo() throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(3);
        
        // 场景：查询用户 → 查询订单 → 查询商品详情
        
        Future<User> userFuture = executor.submit(() -> queryUser());
        
        // 必须等待 userFuture 完成才能查询订单
        // 无法优雅地表达依赖关系
        User user = userFuture.get();
        
        Future<List<Order>> orderFuture = executor.submit(() -> queryOrders(user));
        
        // 又要等待
        List<Order> orders = orderFuture.get();
        
        Future<Map<Long, Product>> productFuture = executor.submit(
            () -> queryProducts(orders)
        );
        
        Map<Long, Product> products = productFuture.get();
        
        // 一堆嵌套的 get()，代码丑陋
    }
    
    // 问题：无法表达 "A完成后执行B，B完成后执行C" 的链式关系
}
```

### 局限三：无法组合多个 Future

无法等待多个 Future 都完成，也无法在任意一个 Future 完成时得到通知。

```java
public class NoCombineProblem {
    
    public void noCombineDemo() throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(3);
        
        Future<Integer> f1 = executor.submit(() -> queryFromDB());
        Future<Integer> f2 = executor.submit(() -> queryFromCache());
        Future<Integer> f3 = executor.submit(() -> queryFromAPI());
        
        // 问题1：如何等待所有 Future 都完成？
        // 必须手动写一个 while 循环
        while (!f1.isDone() || !f2.isDone() || !f3.isDone()) {
            Thread.sleep(100);
        }
        
        // 问题2：如何获取第一个完成的结果？
        // 必须自己实现
        // ...
        
        // 问题3：如何在一个完成后取消其他？
        // 必须自己实现
    }
}
```

## Future 的这些问题

```java
public class FutureProblems {
    
    // 问题总结：
    /*
     * 1. 阻塞式 get()
     *    - 主线程调用 get() 会阻塞
     *    - 没有回调机制
     *    - 必须轮询检查 isDone()
     * 
     * 2. 链式调用困难
     *    - 无法表达任务依赖
     *    - 必须嵌套 get()
     *    - 代码不直观
     * 
     * 3. 无法组合
     *    - 无法等待多个 Future
     *    - 无法获取第一个完成的结果
     *    - 无法取消一组任务
     */
}
```

## CompletableFuture 如何解决

```java
public class CompletableFutureSolution {
    
    public void chainSolution() {
        CompletableFuture.supplyAsync(() -> queryUser())
            .thenCompose(user -> queryOrders(user))      // 链式
            .thenCompose(orders -> queryProducts(orders))
            .thenAccept(products -> {
                // 处理结果
                System.out.println("完成: " + products);
            });
    }
    
    public void combineSolution() {
        CompletableFuture<Integer> f1 = CompletableFuture.supplyAsync(() -> queryFromDB());
        CompletableFuture<Integer> f2 = CompletableFuture.supplyAsync(() -> queryFromCache());
        CompletableFuture<Integer> f3 = CompletableFuture.supplyAsync(() -> queryFromAPI());
        
        // 等待所有完成
        CompletableFuture.allOf(f1, f2, f3).join();
        
        // 获取第一个完成
        CompletableFuture.anyOf(f1, f2, f3).join();
    }
    
    private String queryUser() { return "user"; }
    private List<Order> queryOrders(String user) { return null; }
    private Map<Long, Product> queryProducts(List<Order> orders) { return null; }
    private Integer queryFromDB() { return 1; }
    private Integer queryFromCache() { return 2; }
    private Integer queryFromAPI() { return 3; }
    
    static class Order {}
    static class Product {}
}
```

## 总结

Future 的局限：

1. **阻塞式 get()**：没有回调
2. **无法链式调用**：任务依赖难表达
3. **无法组合**：多 Future 协作困难

这些局限性催生了 `CompletableFuture`，它提供了：
- 非阻塞的回调机制
- 强大的链式调用
- 丰富的组合操作
