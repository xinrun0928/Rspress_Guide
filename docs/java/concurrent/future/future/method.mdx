# JDK Future 接口方法详解

`Future.get()` 到底会不会阻塞？为什么有时候设置超时也没用？

这是很多开发者对 `Future` 的困惑。`Future` 是 JDK 5 引入的接口，定义了异步计算的结果获取机制。看起来简单，但细节很多，用不好就会出问题。

这篇文章会详细讲解 `Future` 的三个核心方法：`get()`、`cancel()`、`isDone()`，以及它们的重载版本和注意事项。

## Future 接口回顾

### 基本定义

```java
public interface Future<V> {
    // 获取结果，阻塞直到完成
    V get() throws InterruptedException, ExecutionException;

    // 获取结果，阻塞直到完成或超时
    V get(long timeout, TimeUnit unit)
        throws InterruptedException, ExecutionException, TimeoutException;

    // 判断是否完成
    boolean isDone();

    // 判断是否取消
    boolean isCancelled();

    // 取消任务
    boolean cancel(boolean mayInterruptIfRunning);
}
```

### 简单使用

```java
ExecutorService executor = Executors.newFixedThreadPool(2);

Future<Integer> future = executor.submit(() -> {
    Thread.sleep(2000);
    return 42;
});

try {
    Integer result = future.get();  // 阻塞直到完成
    System.out.println("结果: " + result);
} catch (ExecutionException e) {
    System.out.println("执行失败: " + e.getCause());
} catch (InterruptedException e) {
    System.out.println("线程被中断");
}
```

## get() 方法详解

### 阻塞特性

`get()` 是**阻塞式**的：

```java
Future<String> future = executor.submit(() -> {
    Thread.sleep(10000);  // 模拟耗时操作
    return "结果";
});

System.out.println("提交任务后");
String result = future.get();  // 在这里阻塞
System.out.println("获取到结果: " + result);

// 输出顺序：
// 1. 提交任务后
// 2. (等待 10 秒)
// 3. 获取到结果: 结果
```

### get() 阻塞的原因

`get()` 内部使用 `LockSupport.park()` 阻塞当前线程：

```java
// 简化实现
public V get() throws InterruptedException, ExecutionException {
    if (isDone()) {
        return report();
    }

    // 阻塞直到被唤醒
    while (!isDone()) {
        LockSupport.park(this);
        if (isCancelled()) {
            throw new CancellationException();
        }
    }
    return report();
}
```

### 抛出哪些异常

```java
try {
    future.get();
} catch (InterruptedException e) {
    // 当前线程被中断
    // 应该恢复中断状态
    Thread.currentThread().interrupt();
} catch (ExecutionException e) {
    // 任务执行过程中抛出异常
    Throwable cause = e.getCause();
    System.out.println("任务失败: " + cause.getMessage());
} catch (CancellationException e) {
    // 任务被取消（Java 8+，通常不显式声明）
}
```

### get() vs get(timeout)

```java
// 无限制等待
try {
    future.get();  // 可能永远阻塞
} catch (ExecutionException | InterruptedException e) {
    e.printStackTrace();
}

// 有时间限制
try {
    // 最多等待 5 秒
    future.get(5, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    System.out.println("超时！");
} catch (ExecutionException | InterruptedException e) {
    e.printStackTrace();
}
```

## 超时重载详解

### TimeoutException

当超过指定时间任务仍未完成时，抛出 `TimeoutException`：

```java
Future<String> future = executor.submit(() -> {
    Thread.sleep(10000);
    return "结果";
});

try {
    String result = future.get(3, TimeUnit.SECONDS);
    System.out.println("结果: " + result);
} catch (TimeoutException e) {
    // 3 秒后任务还没完成
    System.out.println("超时，任务可能被取消");
} catch (ExecutionException | InterruptedException e) {
    e.printStackTrace();
}
```

### 超时后任务会怎样

**重要**：超时后任务**不会自动取消**！

```java
Future<String> future = executor.submit(() -> {
    // 任务继续运行，不受影响
    Thread.sleep(10000);
    return "结果";
});

try {
    future.get(3, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    // 超时，但任务仍在后台运行
    // 下次调用 get() 仍可能获取到结果
    System.out.println(future.isDone());  // false
}
```

### 超时后的正确处理

```java
try {
    String result = future.get(3, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    // 超时，可以选择：
    // 1. 取消任务
    future.cancel(true);

    // 2. 返回默认值
    return "默认值";

    // 3. 继续等待
    try {
        return future.get();  // 无限等待
    } catch (ExecutionException ex) {
        return "默认值";
    }
}
```

## cancel() 方法详解

### 基本用法

```java
Future<String> future = executor.submit(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 模拟长时间任务
        doWork();
    }
    return "完成";
});

// 取消任务
boolean cancelled = future.cancel(true);
System.out.println("取消结果: " + cancelled);
```

### cancel(true) vs cancel(false)

```java
public boolean cancel(boolean mayInterruptIfRunning) {
    if (isDone()) {
        return false;  // 已经完成，无法取消
    }

    if (isCancelled()) {
        return false;  // 已经取消
    }

    if (mayInterruptIfRunning) {
        // 中断正在运行的线程
        thread.interrupt();
    }

    // 标记为取消状态
    state = CANCELLED;
    return true;
}
```

**参数含义**：

| 参数 | 行为 |
|---|---|
| `cancel(true)` | 如果任务正在运行，尝试中断线程 |
| `cancel(false)` | 不中断线程，只标记取消状态，任务可以继续运行直到完成 |

### cancel(true) 的陷阱

**陷阱一：线程可能不响应中断**

```java
executor.submit(() -> {
    while (true) {
        // 不检查中断标志，死循环！
        // cancel(true) 的 interrupt() 对这个线程无效
        if (Thread.interrupted()) {
            break;  // 应该检查中断状态
        }
        doWork();
    }
});
```

**陷阱二：中断可能丢失**

```java
executor.submit(() -> {
    try {
        while (running) {
            // 阻塞在 I/O 上，interrupt() 可以唤醒
            socket.read(buffer);

            // 或者阻塞在 sleep 上
            Thread.sleep(100);
        }
    } catch (InterruptedException e) {
        // 正确处理中断异常
        return;
    }
});
```

### 正确实现可取消任务

```java
Future<String> future = executor.submit(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 关键：定期检查中断状态
        if (Thread.currentThread().isInterrupted()) {
            return "被中断";
        }

        // 模拟工作单元
        processOneUnit();

        // 模拟阻塞操作
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            // 收到中断请求，清理资源后退出
            Thread.currentThread().interrupt();  // 恢复中断状态
            return "被中断";
        }
    }
    return "正常完成";
});
```

## isDone() 方法

### 基本用法

```java
Future<String> future = executor.submit(() -> {
    Thread.sleep(5000);
    return "结果";
});

while (!future.isDone()) {
    System.out.println("任务进行中...");
    Thread.sleep(500);
}

System.out.println("完成: " + future.get());
```

### isDone() vs isCancelled()

```java
Future<String> future = executor.submit(() -> {
    Thread.sleep(3000);
    return "结果";
});

// isDone()：任务是否完成（正常完成、异常、取消都算）
// isCancelled()：任务是否被取消

System.out.println(future.isDone());      // false
System.out.println(future.isCancelled()); // false

future.cancel(true);

System.out.println(future.isDone());      // true
System.out.println(future.isCancelled()); // true
```

### 状态机

```
Future 状态机：

        submit()
            |
            v
    +---------------+
    |    PENDING    | ← isDone() = false, isCancelled() = false
    +---------------+
            |
    +-------+-------+
    |               |
 正常完成         异常完成
    |               |
    v               v
+-----------+   +---------------+
| COMPLETED |   |   EXCEPTION   |
+-----------+   +---------------+
    |               |
    +-------+-------+
            |
            v
    +---------------+
    |  CANCELLED    | ← isDone() = true, isCancelled() = true
    +---------------+

注意：
- CANCELLED 后 isDone() = true
- EXCEPTION 后 isDone() = true，但 isCancelled() = false
```

## 组合使用示例

### 示例一：超时 + 降级

```java
public String fetchWithTimeout(String key) {
    CompletableFuture<String> future = CompletableFuture
        .supplyAsync(() -> fetchFromDatabase(key));

    try {
        // 3 秒超时
        return future.get(3, TimeUnit.SECONDS);
    } catch (TimeoutException e) {
        log.warn("从数据库获取 {} 超时，使用缓存", key);
        return getFromCache(key);
    } catch (ExecutionException | InterruptedException e) {
        log.error("获取失败", e);
        return "默认值";
    }
}
```

### 示例二：超时 + 取消

```java
public String process(String input) {
    Future<String> future = executor.submit(() -> {
        return slowProcess(input);
    });

    try {
        return future.get(10, TimeUnit.SECONDS);
    } catch (TimeoutException e) {
        // 超时，取消任务
        future.cancel(true);
        return "处理超时，请重试";
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        return "处理被中断";
    } catch (ExecutionException e) {
        return "处理失败: " + e.getCause().getMessage();
    }
}
```

### 示例三：轮询模式

```java
public void pollForResult(Future<String> future) {
    int retries = 0;
    while (!future.isDone() && retries < MAX_RETRIES) {
        System.out.println("等待完成... (重试 " + retries + ")");

        try {
            // 轮询间隔
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            return;
        }
        retries++;
    }

    if (future.isDone()) {
        try {
            System.out.println("结果: " + future.get());
        } catch (ExecutionException e) {
            System.out.println("失败: " + e.getCause());
        }
    } else if (retries >= MAX_RETRIES) {
        System.out.println("达到最大重试次数");
        future.cancel(true);
    }
}
```

## 常见问题

### Q1：get() 阻塞了主线程怎么办？

```java
// 不推荐：在主线程阻塞
future.get();  // 不要这样做

// 推荐：用 CompletableFuture 或回调
CompletableFuture.supplyAsync(() -> task())
    .thenAccept(result -> System.out.println(result));

// 或者用线程池
executor.submit(() -> {
    try {
        String result = future.get();
        System.out.println(result);
    } catch (ExecutionException e) {
        e.printStackTrace();
    }
});
```

### Q2：超时后任务还在运行？

`cancel(true)` 尝试中断，但任务可能不响应中断。需要任务本身检查中断状态。

### Q3：多次调用 get() 会有问题吗？

```java
Future<String> future = executor.submit(() -> "结果");

// 第一次 get()
String r1 = future.get();

// 第二次 get()
String r2 = future.get();  // 不会再次执行，直接返回结果

// 多次 get() 是安全的，但第一次 get() 会阻塞直到完成
```

### Q4：任务完成后还能取消吗？

```java
Future<String> future = executor.submit(() -> "结果");
future.get();  // 等待完成

// 尝试取消
future.cancel(true);  // 返回 false，已经完成无法取消
```

### Q5：isDone() 和 isCancelled() 的区别？

```java
future.cancel(true);  // 取消
future.isDone();      // true
future.isCancelled(); // true

future.get();         // 正常完成
future.isDone();      // true
future.isCancelled(); // false

// 任务抛异常
future.get();         // ExecutionException
future.isDone();      // true
future.isCancelled(); // false
```

## 面试问题

**Q1：Future.get() 和 Future.get(timeout) 的区别？**

`get()` 无限等待直到任务完成；`get(timeout)` 最多等待指定时间，超时抛出 `TimeoutException`。

**Q2：cancel(true) 和 cancel(false) 的区别？**

`cancel(true)` 尝试中断正在运行的线程；`cancel(false)` 不中断任务，只标记取消状态。

**Q3：cancel(true) 能保证任务停止吗？**

不一定。如果任务不检查中断状态，cancel(true) 只能设置中断标志，任务可能继续运行。

**Q4：isDone() 和 isCancelled() 的区别？**

`isDone()` 返回任务是否完成（包括正常完成、异常、取消）；`isCancelled()` 只在任务被取消时返回 true。

**Q5：为什么 Future 没有 `thenApply()` 等回调方法？**

因为 `Future` 是 JDK 5 引入的，设计时没有考虑回调。回调方法是 `CompletableFuture`（JDK 8）新增的。

## 总结

| 方法 | 行为 | 异常 |
|---|---|---|
| `get()` | 阻塞等待完成 | InterruptedException, ExecutionException |
| `get(timeout, unit)` | 阻塞等待，最多超时时间 | + TimeoutException |
| `cancel(true/false)` | 取消任务 | 无 |
| `isDone()` | 判断是否完成 | 无 |
| `isCancelled()` | 判断是否被取消 | 无 |

**最佳实践**：

1. **优先使用带超时的 get()**：避免无限等待
2. **超时时考虑取消任务**：避免资源浪费
3. **实现可取消的任务**：检查中断状态
4. **CompletableFuture 是更好的选择**：支持回调和链式调用
