# Future 基础：理解异步操作的结果封装

在 Java 并发编程中，`Future` 是一个看似简单却内涵丰富的重要概念。它是 Java 5 引入的接口，用于表示一个异步计算的结果。当你提交一个任务给线程池执行时，`Future` 就是你获取结果的"凭证"——现在提交，未来兑现。

## 为什么需要 Future？

传统的同步调用简单直接：

```java
// 同步调用：阻塞等待结果
Result result = expensiveOperation();
// 继续使用 result
```

但在现代应用中，这种阻塞等待往往不可接受——如果 `expensiveOperation()` 需要 5 秒，那整个程序就要卡住 5 秒。更合理的做法是：

1. 提交任务，不等待执行
2. 继续做其他事情
3. 稍后获取任务结果

`Future` 正是这种模式的抽象。

## Future 接口详解

`Future` 接口定义了异步结果的所有操作：

```java
public interface Future<V> {
    
    /**
     * 获取计算结果，阻塞直到结果可用
     * @throws CancellationException 如果任务被取消
     * @throws ExecutionException 如果计算过程抛出异常
     * @throws InterruptedException 如果当前线程被中断
     */
    V get() throws InterruptedException, ExecutionException;
    
    /**
     * 获取计算结果，最多等待指定时间
     */
    V get(long timeout, TimeUnit unit) 
        throws InterruptedException, ExecutionException, TimeoutException;
    
    /** 判断任务是否已完成（正常完成/异常/取消） */
    boolean isDone();
    
    /** 尝试取消任务
     * @param mayInterruptIfRunning 如果为 true，任务正在运行时会被中断
     */
    boolean cancel(boolean mayInterruptIfRunning);
    
    /** 判断任务是否被取消 */
    boolean isCancelled();
}
```

## Future 的状态转换

`Future` 任务有五种状态：

```
        NEW（创建）
            |
            v
    COMPLETING（即将完成）
            |
      +-----+-----+
      |           |
      v           v
   NORMAL      EXCEPTIONAL
 (正常完成)   (异常完成)
      |
      v
  CANCELLED
 (被取消)
```

```java
// FutureTask 的状态定义
private static final int NEW          = 0;
private static final int COMPLETING   = 1;
private static final int NORMAL       = 2;
private static final int EXCEPTIONAL  = 3;
private static final int CANCELLED    = 4;
```

一旦任务进入终态（NORMAL、EXCEPTIONAL、CANCELLED），就不会再改变。

## FutureTask：Future 的标准实现

`FutureTask` 是 `Future` 接口最常用的实现类，它同时实现了 `Runnable` 接口，可以直接交给线程或线程池执行：

```java
public class FutureTask<V> implements RunnableFuture<V> {
    
    // 任务状态
    private volatile int state;
    private Callable<V> callable;
    private Object outcome;  // 存储结果或异常
    
    // 执行线程
    private volatile Thread runner;
    
    // 等待结果的等待者链表（WaitNode）
    private volatile WaitNode waiters;
    
    protected void done() {
        // 模板方法，子类可以重写
    }
    
    protected void set(V v) {
        // 设置成功结果
        if (STATE.compareAndSet(this, NEW, COMPLETING)) {
            outcome = v;
            STATE.setRelease(this, NORMAL);
            finishCompletion();  // 唤醒所有等待者
        }
    }
    
    protected void setException(Throwable t) {
        // 设置异常
        if (STATE.compareAndSet(this, NEW, COMPLETING)) {
            outcome = t;
            STATE.setRelease(this, EXCEPTIONAL);
            finishCompletion();
        }
    }
}
```

### FutureTask 的 run 方法

```java
public void run() {
    // 检查状态，防止重复执行
    if (state != NEW ||
        !RUNNER.compareAndSet(this, null, Thread.currentThread()))
        return;
    try {
        Callable<V> c = callable;
        if (c != null && state == NEW) {
            V result;
            boolean ran;
            try {
                result = c.call();  // 执行任务
                ran = true;
            } catch (Throwable ex) {
                result = null;
                ran = false;
                setException(ex);   // 捕获异常
            }
            if (ran)
                set(result);        // 设置结果
        }
    } finally {
        runner = null;
        int s = state;
        if (s >= INTERRUPTING)
            handlePossibleCancellationInterrupt(s);
    }
}
```

### FutureTask 的 get 方法

```java
public V get() throws InterruptedException, ExecutionException {
    int s = state;
    if (s <= COMPLETING)
        s = awaitDone(false, 0L);  // 阻塞等待
    return report(s);  // 报告结果
}

public V get(long timeout, TimeUnit unit) 
        throws InterruptedException, ExecutionException, TimeoutException {
    long nanos = unit.toNanos(timeout);
    int s = state;
    if (s <= COMPLETING)
        s = awaitDone(true, nanos);  // 超时等待
    return report(s);
}

private int awaitDone(boolean timed, long nanos) {
    long startTime = 0;
    int s = state;
    for (;;) {
        if (Thread.interrupted()) {
            removeWaiter(waiters);
            throw new InterruptedException();
        }
        
        if (s > COMPLETING) {
            if (timed)
                startTime = System.nanoTime();
            removeWaiter(waiters);
            return s;
        }
        
        if (s == COMPLETING && timed) {
            Thread.yield();  // 让出 CPU
        } else if (timed) {
            long now = System.nanoTime();
            nanos -= now - startTime;
            if (nanos <= 0)
                return awaitDone(false, 0L);
            LockSupport.parkNanos(this, nanos);
        } else {
            LockSupport.park(this);  // 阻塞
        }
    }
}
```

## 使用场景：与线程池配合

`Future` 通常与 `ExecutorService` 配合使用：

```java
ExecutorService executor = Executors.newFixedThreadPool(4);

// 提交 Callable 任务
Future<Integer> future = executor.submit(() -> {
    // 模拟耗时计算
    Thread.sleep(2000);
    return 42;
});

// 提交 Runnable 任务（Future 只关心完成状态）
Future<?> runnableFuture = executor.submit(() -> {
    // 执行不需要返回值的任务
    doSomething();
}, null);

// 做其他事情
System.out.println("任务已提交，我继续执行...");

// 获取结果
try {
    Integer result = future.get();  // 阻塞等待
    System.out.println("结果: " + result);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
    future.cancel(true);
} catch (ExecutionException e) {
    // 任务执行过程中抛出的异常
    Throwable cause = e.getCause();
    cause.printStackTrace();
}

// 关闭线程池
executor.shutdown();
```

## Future 的局限性

`Future` 虽然解决了异步返回值的问题，但功能有限：

| 特性 | Future 支持 | Future 不支持 |
|------|-----------|-------------|
| 获取结果 | `get()` 阻塞 | 无法注册回调 |
| 超时 | `get(timeout)` 支持 | 只能被动等待 |
| 组合 | 无法组合多个 Future | 需要自己轮询 |
| 完成通知 | 无法在完成时自动通知 | 需要 `get()` 轮询 |

这些问题促成了 `CompletableFuture` 的诞生，它在 Java 8 中引入，提供了更强大的异步编程能力。
