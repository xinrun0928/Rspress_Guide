# AQS 核心思想：state + FIFO 队列

`AbstractQueuedSynchronizer`（AQS）是 Java 并发包的基石，几乎所有的高级并发工具都依赖于它。`ReentrantLock`、`Semaphore`、`CountDownLatch`、`CyclicBarrier`、`ReentrantReadWriteLock`……这些我们常用的同步工具，背后都是 AQS 在支撑。理解 AQS 的核心思想，就等于理解了 Java 并发框架的半壁江山。

## AQS 的设计思想

AQS 采用了**模板方法模式**，它定义了一套获取和释放同步状态的框架，而具体的实现由子类提供：

```java
public abstract class AbstractQueuedSynchronizer 
        extends AbstractOwnableSynchronizer 
        implements java.io.Serializable {
    
    // 同步状态
    private volatile int state;
    
    // 等待队列的头尾节点
    private transient volatile Node head;
    private transient volatile Node tail;
    
    // 模板方法 - 子类需要实现
    protected abstract boolean tryAcquire(int arg);
    protected abstract boolean tryRelease(int arg);
    protected abstract int tryAcquireShared(int arg);
    protected abstract boolean tryReleaseShared(int arg);
    protected abstract boolean isHeldExclusively();
}
```

### AQS 的两大模式

AQS 支持两种同步模式：

| 模式 | 说明 | 应用 |
|------|------|------|
| **独占模式** | 同一时刻只有一个线程能获取同步状态 | `ReentrantLock` |
| **共享模式** | 多个线程可以同时获取同步状态 | `Semaphore`、`CountDownLatch` |

## 核心数据结构：CLH 队列

AQS 内部维护了一个 **CLH 队列**（Craig, Landin, and Hagersten 队列）的变体，这是一个 FIFO 双向队列：

```java
static final class Node {
    // 等待状态
    volatile int waitStatus;
    
    // 节点状态常量
    static final int CANCELLED =  1;    // 已取消
    static final int SIGNAL    = -1;    // 后继节点需要唤醒
    static final int CONDITION = -2;    // 节点在条件队列中等待
    static final int PROPAGATE = -3;    // 共享模式下传播
    
    // 节点链接
    volatile Node prev;
    volatile Node next;
    
    // 等待的线程
    volatile Thread thread;
    
    // 指向下一个在 Condition 上等待的节点
    Node nextWaiter;
}
```

## 同步状态（state）

`state` 是 AQS 的核心，它是一个 `volatile int` 类型的变量：

```java
// state 的含义由子类定义
private volatile int state;

// ReentrantLock 中：
// state = 0：锁未被持有
// state > 0：锁被持有，值表示重入次数

// Semaphore 中：
// state = 可用许可证数量

// CountDownLatch 中：
// state = 倒计时初始值
```

## 独占模式：获取同步状态

```java
public final void acquire(int arg) {
    // 1. 尝试获取（模板方法）
    if (!tryAcquire(arg)) {
        // 2. 获取失败，加入等待队列
        Node node = addWaiter(Node.EXCLUSIVE);
        
        // 3. 在队列中等待获取
        if (acquireQueued(node, arg)) {
            // 4. 获取过程中被中断
            selfInterrupt();
        }
    }
}

// 添加等待节点
private Node addWaiter(Node mode) {
    Node node = new Node(Thread.currentThread(), mode);
    
    // 快速入队
    Node pred = tail;
    if (pred != null) {
        node.prev = pred;
        if (compareAndSetTail(pred, node)) {
            pred.next = node;
            return node;
        }
    }
    
    // 队列为空或 CAS 失败，通过 enq 自旋入队
    enq(node);
    return node;
}

// 自旋入队
private Node enq(Node node) {
    for (;;) {
        Node t = tail;
        if (t == null) {
            // 初始化队列，创建空的头节点
            if (compareAndSetHead(new Node())) {
                tail = head;
            }
        } else {
            // CAS 入队
            node.prev = t;
            if (compareAndSetTail(t, node)) {
                t.next = node;
                return node;
            }
        }
    }
}
```

## 独占模式：释放同步状态

```java
public final boolean release(int arg) {
    // 1. 尝试释放
    if (tryRelease(arg)) {
        Node h = head;
        if (h != null && h.waitStatus != 0) {
            // 2. 唤醒后继节点
            unparkSuccessor(h);
        }
        return true;
    }
    return false;
}

// 唤醒后继节点
private void unparkSuccessor(Node node) {
    int ws = node.waitStatus;
    if (ws < 0) {
        compareAndSetWaitStatus(node, ws, 0);
    }
    
    Node s = node.next;
    if (s == null || s.waitStatus > 0) {
        // 找到最前面的有效节点
        s = null;
        for (Node t = tail; t != null && t != node; t = t.prev) {
            if (t.waitStatus <= 0) {
                s = t;
            }
        }
    }
    
    if (s != null) {
        LockSupport.unpark(s.thread);
    }
}
```

## 共享模式：获取同步状态

```java
public final void acquireShared(int arg) {
    // 尝试获取共享锁
    if (tryAcquireShared(arg) < 0) {
        // 获取失败，加入队列等待
        doAcquireShared(arg);
    }
}

private void doAcquireShared(int arg) {
    // 添加共享节点到队列
    Node node = addWaiter(Node.SHARED);
    
    boolean failed = true;
    try {
        boolean interrupted = false;
        for (;;) {
            Node p = node.predecessor();
            if (p == head) {
                // 再次尝试获取
                int r = tryAcquireShared(arg);
                if (r >= 0) {
                    // 获取成功，设置头节点并传播
                    setHeadAndPropagate(node, r);
                    p.next = null;
                    if (interrupted) {
                        selfInterrupt();
                    }
                    failed = false;
                    return;
                }
            }
            
            // 检查是否需要挂起
            if (shouldParkAfterFailedAcquire(p, node) &&
                parkAndCheckInterrupt()) {
                interrupted = true;
            }
        }
    } finally {
        if (failed) {
            cancelAcquire(node);
        }
    }
}
```

## 共享模式：释放同步状态

```java
public final boolean releaseShared(int arg) {
    // 尝试释放
    if (tryReleaseShared(arg)) {
        // 唤醒后继节点
        doReleaseShared();
        return true;
    }
    return false;
}

private void doReleaseShared() {
    for (;;) {
        Node h = head;
        if (h != null && h != tail) {
            int ws = h.waitStatus;
            if (ws == Node.SIGNAL) {
                // 唤醒后继节点
                if (compareAndSetWaitStatus(h, SIGNAL, 0)) {
                    unparkSuccessor(h);
                }
            } else if (ws == 0 &&
                       !compareAndSetWaitStatus(h, 0, Node.PROPAGATE)) {
                // 传播状态
            }
        }
        
        if (h == head) {
            break;
        }
    }
}
```

                       │
                设置 prev 链接

步骤2：CAS 设置 tail

## 总结

AQS 的核心思想：

1. **同步状态（state）**：由子类定义含义
2. **CLH 队列**：FIFO 双向队列，存储等待线程
3. **模板方法**：定义获取和释放的框架
4. **独占/共享模式**：支持两种同步方式

AQS 是 Java 并发包的基石，理解它就能理解大部分并发工具的实现原理。
