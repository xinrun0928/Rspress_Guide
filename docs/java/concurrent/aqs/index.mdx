# AQS 队列同步器：Java 并发框架的基石

如果你仔细研究 `ReentrantLock`、`Semaphore`、`CountDownLatch`、`CyclicBarrier` 等并发工具的源码，会发现它们都有一个共同的基类——`AbstractQueuedSynchronizer`，简称 **AQS**。AQS 是 Java 并发包的基石，它用极简的设计实现了强大的同步功能。理解 AQS 的工作原理，就等于理解了 Java 并发框架的半壁江山。

## AQS 的设计思想：模板方法模式

AQS 采用模板方法模式设计：它定义了同步状态的获取和释放的框架逻辑，而具体的实现由子类提供。AQS 本身不实现任何同步语义，它只是提供了一组模板方法供子类调用。

```java
public abstract class AbstractQueuedSynchronizer 
        extends AbstractOwnableSynchronizer 
        implements java.io.Serializable {
    
    // 同步状态
    private volatile int state;
    
    // 等待队列的头尾节点
    private transient volatile Node head;
    private transient volatile Node tail;
    
    // 模板方法 - 子类需要实现
    protected int tryAcquire(int arg);
    protected boolean tryRelease(int arg);
    protected int tryAcquireShared(int args);
    protected boolean tryReleaseShared(int args);
    protected boolean isHeldExclusively();
}
```

子类需要实现的方法分为两类：

1. **独占模式**（如 `ReentrantLock`）：同一时刻只有一个线程能获取同步状态
   - `tryAcquire(int arg)`：尝试获取同步状态
   - `tryRelease(int arg)`：尝试释放同步状态

2. **共享模式**（如 `Semaphore`、`CountDownLatch`）：多个线程可以同时获取同步状态
   - `tryAcquireShared(int arg)`：尝试获取同步状态，返回值表示剩余资源
   - `tryReleaseShared(int arg)`：尝试释放同步状态

## 等待队列：FIFO 双向链表

AQS 内部维护了一个 FIFO 双向队列，用于存储等待获取同步状态的线程：

```java
static final class Node {
    // 等待状态
    static final int CANCELLED =  1;
    static final int SIGNAL    = -1;
    static final int CONDITION = -2;
    static final int PROPAGATE = -3;
    
    volatile int waitStatus;
    
    volatile Node prev;
    volatile Node next;
    
    volatile Thread thread;  // 等待的线程
    
    Node nextWaiter;  // 指向下一个在 Condition 上等待的节点
}
```

队列结构如下：

```
         head                        tail
          |                           |
          v                           v
    +--------+     +--------+     +--------+
    | dummy  | <-> | Node 1 | <-> | Node 2 |
    +--------+     +--------+     +--------+
      (null)        Thread A        Thread B
```

注意：队列的第一个节点是"空"的，它不代表任何等待线程，只是作为一个基准点。

## 独占模式：ReentrantLock 的实现

`ReentrantLock` 是 AQS 最典型的应用。`Sync` 类是抽象基类，`NonfairSync` 和 `FairSync` 分别实现非公平锁和公平锁：

```java
// 非公平锁的实现
static final class NonfairSync extends Sync {
    protected final boolean tryAcquire(int acquires) {
        return nonfairTryAcquire(acquires);
    }
}

abstract static class Sync extends AbstractQueuedSynchronizer {
    // 非公平的获取逻辑
    final boolean nonfairTryAcquire(int acquires) {
        final Thread current = Thread.currentThread();
        int c = getState();
        if (c == 0) {
            // 尝试 CAS 获取锁（不检查队列）
            if (compareAndSetState(0, acquires)) {
                setExclusiveOwnerThread(current);
                return true;
            }
        } else if (current == getExclusiveOwnerThread()) {
            // 可重入：同一线程再次获取
            int nextc = c + acquires;
            setState(nextc);
            return true;
        }
        return false;
    }
}

// 公平锁的实现
static final class FairSync extends Sync {
    protected final boolean tryAcquire(int acquires) {
        final Thread current = Thread.currentThread();
        int c = getState();
        if (c == 0) {
            // 公平锁必须检查队列
            if (!hasQueuedPredecessors() &&  // 检查是否有更早等待的线程
                compareAndSetState(0, acquires)) {
                setExclusiveOwnerThread(current);
                return true;
            }
        } else if (current == getExclusiveOwnerThread()) {
            int nextc = c + acquires;
            setState(nextc);
            return true;
        }
        return false;
    }
}
```

非公平锁直接尝试 CAS 获取锁，不检查队列，所以可能"插队"；公平锁多了 `hasQueuedPredecessors()` 检查，确保只有队列最前面的线程才能获取锁。

## acquire 方法：独占获取的流程

`acquire(int arg)` 是模板方法，定义了获取同步状态的完整流程：

```java
public final void acquire(int arg) {
    if (!tryAcquire(arg) &&        // 1. 尝试获取
        acquireQueued(addWaiter(Node.EXCLUSIVE), arg)) {  // 2. 获取失败，加入等待队列
        selfInterrupt();
    }
}

private Node addWaiter(Node mode) {
    Node node = new Node(Thread.currentThread(), mode);
    Node pred = tail;
    if (pred != null) {
        node.prev = pred;
        if (compareAndSetTail(pred, node)) {
            pred.next = node;
            return node;
        }
    }
    enq(node);  // 队列为空，初始化
    return node;
}

final boolean acquireQueued(final Node node, int arg) {
    boolean failed = true;
    try {
        boolean interrupted = false;
        for (;;) {
            final Node p = node.predecessor();
            if (p == head && tryAcquire(arg)) {
                // 如果是队列第一个，且获取成功
                setHead(node);
                p.next = null;  // 帮助 GC
                failed = false;
                return interrupted;
            }
            // 获取失败，检查是否需要挂起
            if (shouldParkAfterFailedAcquire(p, node) &&
                parkAndCheckInterrupt()) {
                interrupted = true;
            }
        }
    } finally {
        if (failed)
            cancelAcquire(node);
    }
}
```

流程总结：

1. 调用 `tryAcquire()` 尝试获取同步状态
2. 获取失败，调用 `addWaiter()` 创建节点加入等待队列
3. 调用 `acquireQueued()` 在队列中自旋等待
4. 如果是队列第一个节点，尝试获取；否则挂起等待

## release 方法：独占释放的流程

```java
public final boolean release(int arg) {
    if (tryRelease(arg)) {  // 1. 尝试释放
        Node h = head;
        if (h != null && h.waitStatus != 0) {
            unparkSuccessor(h);  // 2. 唤醒后继节点
        }
        return true;
    }
    return false;
}

private void unparkSuccessor(Node node) {
    int ws = node.waitStatus;
    if (ws < 0)
        compareAndSetWaitStatus(node, ws, 0);
    
    Node s = node.next;
    if (s == null || s.waitStatus > 0) {
        // 找到最前面的有效节点
        s = null;
        for (Node t = tail; t != null && t != node; t = t.prev) {
            if (t.waitStatus <= 0)
                s = t;
        }
    }
    if (s != null)
        LockSupport.unpark(s.thread);  // 唤醒线程
}
```

释放时，如果 `tryRelease()` 返回 true，唤醒等待队列中最前面的有效节点。

## 共享模式：Semaphore 的实现

`Semaphore`（信号量）控制同时访问资源的线程数量。使用 AQS 的共享模式实现：

```java
static final class NonfairSync extends Sync {
    protected int tryAcquireShared(int reduces) {
        for (;;) {
            int available = getState();
            int remaining = available - reduces;
            if (remaining < 0 ||
                compareAndSetState(available, remaining))
                return remaining;
        }
    }
}

public void acquire() throws InterruptedException {
    sync.acquireSharedInterruptibly(1);
}

public void release() {
    sync.releaseShared(1);
}
```

`tryAcquireShared()` 返回负数表示获取失败，返回非负数表示还有剩余资源。

## 条件变量：await/signal

AQS 还支持条件变量，实现 `Condition` 接口：

```java
public class ConditionObject implements Condition, Serializable {
    private transient Node firstWaiter;
    private transient Node lastWaiter;
    
    public final void await() throws InterruptedException {
        // 将线程加入条件队列
        // ...
    }
    
    public final void signal() {
        // 移动条件队列的节点到同步队列
        // ...
    }
}
```

`Condition` 允许线程在等待某个条件满足时挂起，比 `Object.wait/notify` 更灵活。

## AQS 的价值

AQS 的设计体现了很多优秀的编程思想：

1. **模板方法模式**：定义骨架，子类提供实现
2. **策略模式**：独占和共享两种获取策略
3. **FIFO 队列**：公平性的保证
4. **CAS + 自旋**：无锁编程的经典实现

JDK 中大量工具类都基于 AQS 构建：

- `ReentrantLock`、`ReentrantReadWriteLock`
- `Semaphore`、`CountDownLatch`、`CyclicBarrier`
- `Exchanger`（线程间交换数据）
- `ArrayBlockingQueue`、`LinkedBlockingQueue`

理解 AQS 的原理，就理解了 Java 并发框架的精髓，为深入学习其他并发工具打下坚实基础。你是否曾好奇过这些"高级"并发工具是如何实现的？答案就在 AQS 之中。
