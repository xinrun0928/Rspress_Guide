# AQS 两种模式：独占锁与共享锁

AQS 支持两种同步模式：**独占模式**和**共享模式**。理解这两种模式的区别和使用场景，是深入掌握 Java 并发框架的关键。

## 两种模式的区别

```
┌────────────────────────────────────────────────────────────────┐
│                    独占模式 vs 共享模式                              │
│                                                                 │
│   独占模式（Exclusive）                                          │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   同一时刻只有一个线程能获取同步状态                      │  │
│   │                                                         │  │
│   │   Thread A ───────▶ [持有锁]                            │  │
│   │   Thread B ───▶ [等待]                                  │  │
│   │   Thread C ───▶ [等待]                                  │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   共享模式（Shared）                                             │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   多个线程可以同时获取同步状态                            │  │
│   │                                                         │  │
│   │   Thread A ──▶ [持有] ──┐                              │  │
│   │   Thread B ──▶ [持有] ──┼──▶ 状态值递减               │  │
│   │   Thread C ──▶ [持有] ──┘                              │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## 模式对比表

| 特性 | 独占模式 | 共享模式 |
|------|---------|---------|
| **获取方法** | `acquire()` | `acquireShared()` |
| **释放方法** | `release()` | `releaseShared()` |
| **返回值** | `tryAcquire()` 返回 boolean | `tryAcquireShared()` 返回 int |
| **成功返回值** | `true` | `>= 0`（剩余资源数） |
| **失败返回值** | `false` | `< 0` |
| **应用** | `ReentrantLock` | `Semaphore`、`CountDownLatch` |
| **队列处理** | 独占节点 | 共享节点 |

## 独占模式

### 核心方法

```java
public abstract class AbstractQueuedSynchronizer {
    
    // 独占模式获取
    public final void acquire(int arg) {
        if (!tryAcquire(arg)) {
            // 获取失败，加入队列
            Node node = addWaiter(Node.EXCLUSIVE);
            acquireQueued(node, arg);
        }
    }
    
    // 独占模式释放
    public final boolean release(int arg) {
        if (tryRelease(arg)) {
            Node h = head;
            if (h != null && h.waitStatus != 0) {
                unparkSuccessor(h);
            }
            return true;
        }
        return false;
    }
    
    // 子类需要实现的方法
    protected abstract boolean tryAcquire(int arg);
    protected abstract boolean tryRelease(int arg);
    protected abstract boolean isHeldExclusively();
}
```

### 独占模式的特点

1. **互斥**：同一时刻只有一个线程能获取
2. **可重入**：同一线程可以多次获取
3. **FIFO**：等待线程按顺序获取
4. **中断支持**：支持可中断等待

## 共享模式

### 核心方法

```java
public abstract class AbstractQueuedSynchronizer {
    
    // 共享模式获取
    public final void acquireShared(int arg) {
        if (tryAcquireShared(arg) < 0) {
            // 获取失败，加入队列
            doAcquireShared(arg);
        }
    }
    
    // 共享模式释放
    public final boolean releaseShared(int arg) {
        if (tryReleaseShared(arg)) {
            doReleaseShared();
            return true;
        }
        return false;
    }
    
    // 子类需要实现的方法
    protected abstract int tryAcquireShared(int arg);
    protected abstract boolean tryReleaseShared(int arg);
}
```

### 共享模式的特点

1. **并行**：多个线程可以同时获取
2. **计数**：state 表示资源数量
3. **传播**：释放时唤醒后继节点
4. **支持中断**：支持可中断等待

## ReentrantLock：独占模式

```java
// ReentrantLock.NonfairSync
static final class NonfairSync extends Sync {
    protected final boolean tryAcquire(int acquires) {
        return nonfairTryAcquire(acquires);
    }
}

// Sync.nonfairTryAcquire
final boolean nonfairTryAcquire(int acquires) {
    Thread current = Thread.currentThread();
    int c = getState();
    
    if (c == 0) {
        // 锁未被持有，尝试 CAS 获取
        if (compareAndSetState(0, acquires)) {
            setExclusiveOwnerThread(current);
            return true;
        }
    } else if (current == getExclusiveOwnerThread()) {
        // 可重入：当前线程已持有锁
        int nextc = c + acquires;
        setState(nextc);
        return true;
    }
    return false;
}

// ReentrantLock 总是独占模式
// 同一时刻只有一个线程能持有锁
```

## Semaphore：共享模式

```java
// Semaphore.NonfairSync
static final class NonfairSync extends Sync {
    protected final int tryAcquireShared(int acquires) {
        for (;;) {
            int available = getState();
            int remaining = available - acquires;
            
            if (remaining < 0 ||
                compareAndSetState(available, remaining)) {
                return remaining;
            }
        }
    }
}

// Semaphore 可以有多个线程同时获取许可证
// state 表示可用许可证数量
// 每次获取递减 state
```

## CountDownLatch：共享模式

```java
// CountDownLatch.Sync
protected int tryAcquireShared(int ignores) {
    return getState() == 0 ? 1 : -1;
}

protected boolean tryReleaseShared(int releases) {
    for (;;) {
        int c = getState();
        if (c == 0) {
            return false;  // 已经归零，不能再释放
        }
        int nextc = c - 1;
        if (compareAndSetState(c, nextc)) {
            return nextc == 0;  // 返回是否完全倒计时
        }
    }
}

// CountDownLatch 也是共享模式
// 所有等待的线程同时被唤醒
```

## 队列节点的区别

```java
public class NodeModes {

    // 独占节点
    static class ExclusiveNode extends Node {

        }
    }
}
```

## 模式转换

### 独占转共享

```java
// ReentrantReadWriteLock
// 读锁是共享模式，写锁是独占模式
// 获取写锁前必须释放所有读锁

public class ReadWriteLockExample {
    
    private final ReentrantReadWriteLock rwLock = 
        new ReentrantReadWriteLock();
    
    public void read() {
        rwLock.readLock().lock();
        try {
            // 读操作
        } finally {
            rwLock.readLock().unlock();
        }
    }
    
    public void write() {
        rwLock.writeLock().lock();
        try {
            // 写操作
        } finally {
            rwLock.writeLock().unlock();
        }
    }
}
```

## 实战：自定义同步器

### 自定义互斥锁

```java
public class Mutex implements Lock, java.io.Serializable {
    
    private final Sync sync = new Sync();
    
    // 使用 AQS 独占模式
    private static class Sync extends AbstractQueuedSynchronizer {
        protected boolean tryAcquire(int arg) {
            // CAS 尝试获取锁
            if (compareAndSetState(0, 1)) {
                setExclusiveOwnerThread(Thread.currentThread());
                return true;
            }
            return false;
        }
        
        protected boolean tryRelease(int arg) {
            if (getState() == 0) {
                throw new IllegalMonitorStateException();
            }
            setExclusiveOwnerThread(null);
            setState(0);
            return true;
        }
        
        protected boolean isHeldExclusively() {
            return getExclusiveOwnerThread() == Thread.currentThread();
        }
        
        Condition newCondition() {
            return newConditionObject();
        }
    }
}
```

### 自定义信号量

```java
public class SimpleSemaphore implements java.io.Serializable {
    
    private final Sync sync;
    
    public SimpleSemaphore(int permits) {
        sync = new Sync(permits);
    }
    
    // 使用 AQS 共享模式
    private static class Sync extends AbstractQueuedSynchronizer {
        Sync(int permits) {
            setState(permits);
        }
        
        protected int tryAcquireShared(int reduces) {
            for (;;) {
                int available = getState();
                int remaining = available - reduces;
                
                if (remaining < 0 ||
                    compareAndSetState(available, remaining)) {
                    return remaining;
                }
            }
        }
        
        protected boolean tryReleaseShared(int restores) {
            for (;;) {
                int current = getState();
                int next = current + restores;
                if (compareAndSetState(current, next)) {
                    return true;
                }
            }
        }
    }
}
```

## 总结

AQS 的两种模式：

| 模式 | 获取方法 | 释放方法 | 应用 |
|------|---------|---------|------|
| **独占** | `acquire()` | `release()` | `ReentrantLock` |
| **共享** | `acquireShared()` | `releaseShared()` | `Semaphore`、`CountDownLatch` |

选择建议：
- 需要互斥访问：`ReentrantLock`（独占）
- 控制并发数量：`Semaphore`（共享）
- 等待多个事件完成：`CountDownLatch`（共享）

理解两种模式，才能灵活使用 Java 并发工具。
