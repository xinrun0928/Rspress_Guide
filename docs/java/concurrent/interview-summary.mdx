# Java 并发面试高频问题汇总

Java 并发是面试中的重中之重，本章汇总高频面试问题及答案。

## synchronized 相关

### Q1: synchronized 和 Lock 的区别？

| 特性 | synchronized | Lock |
|------|-------------|------|
| 获取方式 | 自动释放 | 手动释放 |
| 公平性 | 非公平 | 可选公平/非公平 |
| 中断 | 不可中断 | 可中断 |
| 条件 | Object.wait | Condition |
| 性能 | JVM 优化 | JDK 优化 |

### Q2: synchronized 锁的是什么？

锁的是对象的 Mark Word，包括：
- 偏向锁：锁对象头中的线程 ID
- 轻量级锁：指向栈中 Lock Record 的指针
- 重量级锁：指向 Monitor 的指针

## 线程池相关

### Q3: 线程池参数？

```java
new ThreadPoolExecutor(
    corePoolSize,      // 核心线程数
    maximumPoolSize,    // 最大线程数
    keepAliveTime,     // 空闲存活时间
    unit,               // 时间单位
    workQueue,         // 任务队列
    threadFactory,      // 线程工厂
    handler            // 拒绝策略
);
```

### Q4: 线程池执行流程？

1. corePoolSize 未满 → 创建核心线程
2. 队列未满 → 加入队列
3. maximumPoolSize 未满 → 创建临时线程
4. 执行拒绝策略

## JMM 相关

### Q5: volatile 保证了什么？

- **可见性**：一个线程的修改对其他线程可见
- **有序性**：禁止指令重排序
- **不保证原子性**：复合操作仍需同步

### Q6: happens-before 规则？

1. 程序顺序规则
2. 监视器锁规则
3. volatile 规则
4. 线程启动规则
5. 线程终止规则
6. 传递性

## 并发工具类

### Q7: CountDownLatch vs CyclicBarrier？

| 特性 | CountDownLatch | CyclicBarrier |
|------|----------------|---------------|
| 计数方向 | 递减到0 | 递增到 parties |
| 可重用 | 否 | 是 |
| 等待线程 | 任意数量 | 固定数量 |

### Q8: ReentrantLock 可重入原理？

通过 AQS 的 state 计数器实现：
- 获取锁：state + 1
- 释放锁：state - 1
- state = 0 时完全释放

## CAS 相关

### Q9: CAS 原理？

Compare-And-Swap，CPU 原子指令：
1. 比较内存值与期望值
2. 相等则更新
3. 返回是否成功

### Q10: CAS 的 ABA 问题？

值被改成 A→B→A，CAS 检查通过但中间有变化。

解决方案：使用 `AtomicStampedReference` 带版本号。

## 实战问题

### Q11: 如何实现线程安全的单例？

```java
// 方案1：饿汉式
class HungrySingleton {
    private static final HungrySingleton INSTANCE = new HungrySingleton();
}

// 方案2：双重检查 + volatile
class DCLSingleton {
    private static volatile DCLSingleton instance;
    
    public static DCLSingleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

// 方案3：静态内部类
class StaticInnerSingleton {
    private static class Holder {
        private static final StaticInnerSingleton INSTANCE = new StaticInnerSingleton();
    }
    
    public static StaticInnerSingleton getInstance() {
        return Holder.INSTANCE;
    }
}
```

### Q12: 如何避免死锁？

1. 固定锁获取顺序
2. 使用 tryLock 超时
3. 减少锁的范围
4. 避免嵌套锁
