# Java I/O 与网络

select/poll/epoll，你知道它们的区别吗？

大多数后端面试都会问这个问题，但 90% 的人只会背「epoll 是事件驱动，比 select 性能好」。追问下去：epoll 的 LT 和 ET 模式有什么区别？为什么 Netty 选择了 NIO 而不是 AIO？

I/O（输入/输出）和网络编程，是 Java 连接外部世界的桥梁。从最底层的 Socket 通信，到经典的 BIO/NIO/AIO 模型，到高性能网络框架 Netty，再到 Tomcat 的线程模型——每一步都值得深入理解。

## 面试核心公式

```
BIO = 阻塞 I/O = 一个线程处理一个连接 = 无法应对高并发
         ↓
NIO = 非阻塞 I/O + 多路复用 = 一个线程处理多个连接 = Reactor 模式
         ↓
epoll 优势 = 红黑树 O(1) 查找 + LT/ET 模式 + 无最大连接数限制
         ↓
Netty 线程模型 = BossGroup（accept） + WorkerGroup（read/write） + EventLoop 单线程执行
```
