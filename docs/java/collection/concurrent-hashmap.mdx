**目标级别**：P6

## 一、核心问题

### 🔴 ConcurrentHashMap 为什么是线程安全的？

HashMap 不是线程安全的，并发 put 可能导致数据覆盖、死循环等问题。ConcurrentHashMap 通过分段锁（JDK7）或 CAS + synchronized（JDK8）实现线程安全。

### JDK7 vs JDK8 对比

| 维度 | JDK7 | JDK8 |
|------|------|------|
| **数据结构** | Segment[] + HashEntry[]（分段锁） | Node[] + CAS + synchronized |
| **锁粒度** | 16 个 Segment，每个是一把锁 | 每个桶一把锁（头节点） |
| **并发度** | 最多 16（可调整） | 理论上无限 |
| **实现复杂度** | 复杂（双重数据结构） | 简洁 |
| **put 性能** | 较好（分段锁） | 更好（乐观锁 + 悲观锁） |

## 二、JDK7 分段锁实现

### 核心数据结构

```java title="ConcurrentHashMap.java (JDK7)"
public class ConcurrentHashMap<K, V> extends AbstractMap<K, V>
        implements ConcurrentMap<K, V>, Serializable {
    
    // 分段锁数组（不可变 final）
    final Segment<K,V>[] segments;
    
    // 默认分段锁数量
    static final int DEFAULT_CONCURRENCY_LEVEL = 16;
    
    // Segment 继承 ReentrantLock
    static final class Segment<K,V> extends ReentrantLock {
        transient volatile HashEntry<K,V>[] table;
        transient int count;
        transient int modCount;
        transient int threshold;
    }
    
    // HashEntry 是链表节点
    static final class HashEntry<K,V> {
        final int hash;
        final K key;
        volatile V value;
        volatile HashEntry<K,V> next;
    }
}
```

### put 流程（JDK7）

```java
public V put(K key, V value) {
    Segment<K,V> s;
    if (value == null) throw new NullPointerException();
    
    // 1. 计算 key 的 hash
    int hash = hash(key);
    
    // 2. 计算 Segment 索引
    int j = (hash >>> segmentShift) & (segmentMask);
    
    // 3. 获取或创建 Segment
    if ((s = (Segment<K,V>)UNSAFE.getObjectVolatile(
            segments, u)) == null) {
        s = ensureSegment(j);
    }
    
    // 4. 在 Segment 内 put（需要加锁）
    return s.put(key, hash, value, false);
}

// Segment.put
final V put(K key, int hash, V value, boolean onlyIfAbsent) {
    // 加锁（可重入）
    lock();
    try {
        HashEntry<K,V>[] tab = table;
        int index = (tab.length - 1) & hash;
        HashEntry<K,V> first = tab[index];
        
        // 遍历链表...
        for (HashEntry<K,V> e = first;;) {
            if (e != null) {
                K k;
                if (e.hash == hash &&
                    ((k = e.key) == key || key.equals(k))) {
                    V oldValue = e.value;
                    if (!onlyIfAbsent) {
                        e.value = value;
                    }
                    return oldValue;
                }
                e = e.next;
            } else {
                // 头插法
                if (casNext(e, new HashEntry<K,V>(hash, key, value, first))) {
                    break;
                }
            }
        }
    } finally {
        unlock();
    }
}
```

## 三、JDK8 CAS + Synchronized 实现

### 核心数据结构

```java title="ConcurrentHashMap.java (JDK8)"
public class ConcurrentHashMap<K, V> extends AbstractMap<K, V>
        implements ConcurrentMap<K, V>, Serializable {
    
    // 数组 + 链表 + 红黑树（JDK8 统一）
    transient volatile Node<K,V>[] table;
    
    // 链表节点
    static class Node<K,V> implements Map.Entry<K,V> {
        final int hash;
        final K key;
        volatile V value;
        volatile Node<K,V> next;
    }
    
    // 红黑树节点
    static final class TreeNode<K,V> extends Node<K,V> {
        TreeNode<K,V> parent;
        TreeNode<K,V> left;
        TreeOpenObject<K,V> right;
        boolean red;
    }
}
```

### 核心属性

```java
// 数组
private transient volatile Node<K,V>[] table;

// 表的初始化和大小调整控制
private transient volatile int sizeCtl;

// 各种状态值
private static final int MOVED     = -1; // hash FORWARDING
private static final int TREEBIN   = -2; // hash TBIN
private static final int RESERVED  = -3; // hash RESERVED
```

### put 流程（JDK8）

```java title="ConcurrentHashMap.java"
public V put(K key, V value) {
    return putVal(key, value, false);
}

final V putVal(K key, V value, boolean onlyIfAbsent) {
    if (key == null || value == null) throw new NullPointerException();
    
    int hash = spread(key.hashCode());
    int binCount = 0;
    
    // 自旋 + CAS
    for (Node<K,V>[] tab = table;;) {
        Node<K,V> f; int n, i, fh;
        
        // 1. table 为空，初始化
        if (tab == null || (n = tab.length) == 0)
            tab = initTable();
        
        // 2. 该位置为空，CAS 插入
        else if ((f = tabAt(tab, i = (n - 1) & hash)) == null) {
            if (casTabAt(tab, i, null,
                         new Node<K,V>(hash, key, value, null)))
                break;
        }
        
        // 3. 正在扩容，帮助扩容
        else if ((fh = f.hash) == MOVED)
            tab = helpTransfer(tab, f);
        
        // 4. 该位置有节点（哈希冲突）
        else {
            V oldVal = null;
            
            // 加锁（锁定头节点）
            synchronized (f) {
                // 再次确认头节点未变
                if (tabAt(tab, i) == f) {
                    // fh >= 0 是普通链表
                    if (fh >= 0) {
                        binCount = 1;
                        for (Node<K,V> e = f;; ++binCount) {
                            K ek;
                            if (e.hash == hash &&
                                ((ek = e.key) == key ||
                                 key.equals(ek))) {
                                oldVal = e.value;
                                if (!onlyIfAbsent)
                                    e.value = value;
                                break;
                            }
                            Node<K,V> pred = e;
                            if ((e = e.next) == null) {
                                pred.next = new Node<K,V>(hash, key,
                                                          value, null);
                                break;
                            }
                        }
                    }
                    // 红黑树节点
                    else if (f instanceof TreeBin) {
                        Node<K,V> p;
                        binCount = 2;
                        p = ((TreeBin<K,V>)f).putTreeVal(hash, key,
                                                       value);
                        if (p != null) {
                            oldVal = p.value;
                            if (!onlyIfAbsent)
                                p.value = value;
                        }
                    }
                }
            }
            
            if (binCount != 0) {
                // 链表长度 >= 8，转红黑树
                if (binCount >= TREEIFY_THRESHOLD)
                    treeifyBin(tab, i);
                if (oldVal != null)
                    return oldVal;
                break;
            }
        }
    }
    
    // 5. 添加计数
    addCount(1L, binCount);
    return null;
}
```

## 四、初始化与扩容

### initTable（JDK8）

```java
private final Node<K,V>[] initTable() {
    Node<K,V>[] tab; int sc;
    while ((tab = table) == null || tab.length == 0) {
        // sizeCtl < 0 表示正在初始化
        if ((sc = sizeCtl) < 0)
            Thread.yield();
        
        // CAS 设置 sizeCtl = -1
        else if (U.compareAndSwapInt(this, SIZECTL, sc, -1)) {
            try {
                if ((tab = table) == null || tab.length == 0) {
                    int n = (sc > 0) ? sc : DEFAULT_CAPACITY;
                    @SuppressWarnings("unchecked")
                    Node<K,V>[] nt = (Node<K,V>[])new Node<?,?>[n];
                    table = tab = nt;
                    sc = n - (n >>> 2);
                }
            } finally {
                sizeCtl = sc;
            }
            break;
        }
    }
    return tab;
}
```

### helpTransfer 扩容（JDK8）

```java
final Node<K,V>[] helpTransfer(Node<K,V>[] tab, Node<K,V> f) {
    Node<K,V>[] nextTab; int sc;
    
    // f 是 ForwardingNode
    if (tab != null && (f instanceof ForwardingNode) &&
        (nextTab = ((ForwardingNode<K,V>)f).nextTable) != null) {
        
        int rs = resizeStamp(tab.length);
        while (nextTab == nextTable && table == tab &&
               (sc = sizeCtl) < 0) {
            if ((sc >>> RESIZE_STAMP_SHIFT) != rs || sc == rs + 1 ||
                sc == rs + MAX_RESIZERS || transferIndex <= 0)
                break;
            
            // CAS 增加 sizeCtl，参与扩容的线程数 +1
            if (U.compareAndSwapInt(this, SIZECTL, sc, sc + 1)) {
                transfer(tab, nextTab);
                break;
            }
        }
    }
    return nextTab;
}
```

### JDK7 vs JDK8 扩容对比

| 特性 | JDK7 | JDK8 |
|------|------|------|
| **锁粒度** | Segment 锁（16 把锁） | Node 锁（桶锁） |
| **并发度** | 最多 16 | 理论上无限 |
| **实现方式** | 先加锁再扩容 | 多线程并发扩容 |
| **转移方式** | 单线程顺序转移 | 多线程分段转移 |

## 五、get 方法

### JDK7 get（无锁）

```java
public V get(Object key) {
    Segment<K,V> s;
    HashEntry<K,V>[] tab;
    int h = hash(key);
    
    // 定位 Segment
    long u = (((h >>> segmentShift) & segmentMask) << SSHIFT) + SBASE;
    if ((s = (Segment<K,V>)UNSAFE.getObjectVolatile(segments, u)) != null &&
        (tab = s.table) != null) {
        // 遍历链表
        for (HashEntry<K,V> e = (HashEntry<K,V>) 
                 UNSAFE.getObjectVolatile(tab, ((long)(((tab.length - 1) & h)) << TSHIFT) + TBASE);
             e != null; e = e.next) {
            K k;
            if (e.hash == h && ((k = e.key) == key || key.equals(k)))
                return e.value;
        }
    }
    return null;
}
```

### JDK8 get（无锁）

```java
public V get(Object key) {
    Node<K,V>[] tab; Node<K,V> e, p; int n, eh; K ek;
    
    int h = spread(key.hashCode());
    if ((tab = table) != null && (n = tab.length) > 0 &&
        (e = tabAt(tab, (n - 1) & h)) != null) {
        // 首节点命中
        if ((eh = e.hash) == h) {
            if ((ek = e.key) == key || (ek != null && key.equals(ek)))
                return e.value;
        }
        // 树节点
        else if (eh < 0)
            p = ((TreeBin<K,V>)e).find(h, key);
        // 链表
        else {
            do {
                if (e.hash == h &&
                    ((ek = e.key) == key || (ek != null && key.equals(ek))))
                    return e.value;
            } while ((e = e.next) != null);
        }
    }
    return null;
}
```

### get 方法为什么不需要加锁？

**volatile 保证可见性**：

```java
// Node.value 是 volatile
static class Node<K,V> implements Map.Entry<K,V> {
    volatile V value;  // volatile 保证可见性
    volatile Node<K,V> next;
}
```

## 六、size 方法对比

### JDK7 size

```java
public int size() {
    Segment<K,V>[] segments = this.segments;
    int size;
    long sum = 0;
    long overflow = 0;
    int sumMod = 0;
    
    // 先不加锁统计两次
    for (int i = 0; i < RETRIES_BEFORE_LOCK; ++i) {
        sum = 0;
        sumMod = 0;
        overflow = 0;
        
        for (int j = 0; j < segments.length; ++j) {
            Segment<K,V> s = segments[j];
            int c = s.count;
            sumMod += c;
            if (c < 0)
                overflow = Long.MIN_VALUE;
        }
        
        // 两次结果相同，说明没有并发修改
        if (sumMod == sum)
            break;
        
        // 否则加锁重试
        sum = 0;
        for (int j = 0; j < segments.length; ++j)
            sum += segments[j].size();
    }
    
    return (int)(sum + overflow);
}
```

### JDK8 size（CounterCell）

```java
// 使用 CounterCell 分散计数
private transient volatile long baseCount;
private transient volatile CounterCell[] counterCells;

public int size() {
    long n = sumCount();
    return ((n < 0L) ? 0 : (n > (long)Integer.MAX_VALUE) ? Integer.MAX_VALUE : (int)n);
}

final long sumCount() {
    CounterCell[] as = counterCells;
    long sum = baseCount;
    if (as != null) {
        for (CounterCell a : as) {
            sum += a.value;
        }
    }
    return sum;
}
```

## 七、面试题精讲

### 🔴 第一层：ConcurrentHashMap 和 HashMap 的区别

> **参考答案**：
>
> 1. **线程安全性**：HashMap 线程不安全，ConcurrentHashMap 线程安全
> 2. **实现方式**：HashMap 无锁，ConcurrentHashMap 用 CAS + synchronized
> 3. **数据结构**：JDK8 后两者都是数组 + 链表 + 红黑树
> 4. **null 支持**：HashMap 允许 null key/value，ConcurrentHashMap 不允许
> 5. **性能**：HashMap 在单线程下性能好，ConcurrentHashMap 在多线程下性能好

### 🟡 第二层：JDK7 和 JDK8 的区别

> **参考答案**：
>
> | 维度 | JDK7 | JDK8 |
> |------|------|------|
> | **数据结构** | Segment分段锁 | CAS + synchronized |
> | **锁粒度** | 每个Segment一把锁（16把） | 每个桶一把锁 |
> | **并发度** | 固定16 | 动态扩展 |
> | **复杂度** | 复杂 | 简洁 |
> | **优化** | 无 | 链表转红黑树、并发扩容 |

### 🟡 第三层：为什么 JDK8 要放弃分段锁？

> **参考答案**：
>
> 1. **分段锁开销大**：16个Segment占用额外内存，且需要继承ReentrantLock
> 2. **并发度固定**：最多16个线程并发，热门桶仍是瓶颈
> 3. **实现复杂度**：双重数组结构，代码复杂
> 4. **JDK8优化**：CAS + synchronized 可以让每个桶独立加锁，理论上无限并发

### 💡 第四层：JDK8 的锁细化

> **参考答案**：
>
> JDK8 的 synchronized 锁细化优化：
>
> 1. **轻量级锁获取**：如果头节点无锁，用 CAS 尝试获取
> 2. **偏向锁**：无竞争时，同一线程重复获取无需加锁
> 3. **锁膨胀**：竞争激烈时膨胀为重量级锁
> 4. **桶锁**：只锁定当前桶，不影响其他桶的操作

## 八、常见错误与陷阱

### ⚠️ 陷阱 1：ConcurrentHashMap 是完全线程安全的

ConcurrentHashMap 只保证单个操作的原子性（如 put、get），**复合操作不是线程安全的**：

```java
// 非原子操作，不是线程安全的
if (map.get(key) == null) {
    map.put(key, value);  // 可能被其他线程覆盖
}

// 正确做法
map.putIfAbsent(key, value);  // 原子操作
```

### ⚠️ 陷阱 2：JDK7 的 key 不能为 null

JDK7 和 JDK8 的 ConcurrentHashMap **都不允许 null key/value**，因为多线程环境下无法区分「未找到」和「值为 null」。

```java
// 会抛 NullPointerException
map.put(null, "value");
map.put("key", null);
```

### ⚠️ 陷阱 3：遍历时修改集合

```java
// 迭代器是弱一致性的，遍历时修改可能看到部分更新
for (Map.Entry<String, String> entry : map.entrySet()) {
    if (entry.getValue().equals("delete")) {
        map.remove(entry.getKey());  // 可能 ConcurrentModificationException
    }
}

// 正确做法
map.entrySet().removeIf(entry -> entry.getValue().equals("delete"));
```

## 九、对比总结表

| 维度 | HashMap | Hashtable | ConcurrentHashMap |
|------|---------|-----------|-------------------|
| **线程安全** | 否 | 是（全局锁） | 是（桶锁/CAS） |
| **并发性能** | 低 | 低 | 高 |
| **null key/value** | 允许 | 不允许 | 不允许 |
| **数据结构** | 数组+链表 | 数组+链表 | 数组+链表+红黑树 |
| **JDK8 优化** | 无 | 无 | CAS+synchronized |

| 维度 | JDK7 分段锁 | JDK8 CAS+synchronized |
|------|-------------|----------------------|
| **锁粒度** | Segment（16把锁） | 桶（每桶一把锁） |
| **并发度上限** | 16 | 无限（理论） |
| **内存占用** | 高（双重数组） | 低 |
| **实现复杂度** | 复杂 | 简洁 |
| **扩容** | 单线程 | 多线程并发 |

## 十、扩展思考

> **追问**：为什么 ConcurrentHashMap 不直接用 synchronized 锁整个数组？

因为锁粒度太粗，所有 put 操作都会串行，严重影响并发性能。JDK8 用 synchronized 只锁桶节点，实现更细粒度的并发控制。

> **追问**：ConcurrentHashMap 的迭代器是线程安全的吗？

不是。迭代器是弱一致性的（ weakly consistent），它可能看不到并发修改的全部更新，但在遍历过程中不会抛出 ConcurrentModificationException。
