# BlockingQueue 体系与生产者-消费者模式

在多线程编程中，生产者-消费者模式是最经典的并发设计模式之一。一个或多个线程生成数据（生产者），一个或多个线程处理数据（消费者），它们之间通过一个队列来传递数据。BlockingQueue（阻塞队列）是实现这一模式的核心组件，它提供了线程安全的入队和出队操作，以及可选的阻塞能力。

## BlockingQueue 的核心接口

BlockingQueue 扩展自 Queue 接口，添加了阻塞操作：

```java
public interface BlockingQueue<E> extends Queue<E> {
    // 抛出异常的入队方法
    void add(E e);
    boolean offer(E e);
    
    // 阻塞的入队方法
    void put(E e) throws InterruptedException;
    boolean offer(E e, long timeout, TimeUnit unit) throws InterruptedException;
    
    // 抛出异常的出队方法
    E remove();
    E poll();
    
    // 阻塞的出队方法
    E take() throws InterruptedException;
    E poll(long timeout, TimeUnit unit) throws InterruptedException;
}
```

核心区别在于：

- `put()` / `take()`：队列满/空时会一直阻塞，直到操作成功
- `offer()` / `poll()`：队列满/空时立即返回 false/null
- `offer(e, timeout)` / `poll(timeout)`：队列满/空时等待指定时间

## BlockingQueue 的实现体系

Java 提供了多种 BlockingQueue 实现：

**ArrayBlockingQueue**：基于数组的有界队列，需要指定容量。

```java
BlockingQueue<Task> queue = new ArrayBlockingQueue<>(100);
```

**LinkedBlockingQueue**：基于链表的可选有界队列，默认无界（Integer.MAX_VALUE）。

```java
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100); // 有界
BlockingQueue<Task> queue = new LinkedBlockingQueue<>();    // 无界
```

**PriorityBlockingQueue**：基于数组的无界优先级队列，元素按优先级排序。

```java
BlockingQueue<Task> queue = new PriorityBlockingQueue<>();
```

**DelayQueue**：延迟队列，元素必须实现 Delayed 接口，只有延迟到期才能出队。

```java
BlockingQueue<DelayedTask> queue = new DelayQueue<>();
```

**SynchronousQueue**：不存储元素的队列，每个 put 必须等待一个 take，反之亦然。

```java
BlockingQueue<Task> queue = new SynchronousQueue<>();
```

## 生产者-消费者模式实战

```java
public class ProducerConsumerDemo {
    
    private static final int QUEUE_CAPACITY = 10;
    private static final int NUM_PRODUCERS = 2;
    private static final int NUM_CONSUMERS = 4;
    
    public static void main(String[] args) {
        BlockingQueue<Task> queue = new LinkedBlockingQueue<>(QUEUE_CAPACITY);
        
        // 启动生产者
        for (int i = 0; i < NUM_PRODUCERS; i++) {
            final int producerId = i;
            new Thread(() -> {
                int taskId = 0;
                while (true) {
                    Task task = new Task(producerId, taskId++);
                    try {
                        queue.put(task); // 队列满时阻塞
                        System.out.println("Producer " + producerId + " produced: " + task);
                        Thread.sleep(100); // 模拟生产时间
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }).start();
        }
        
        // 启动消费者
        for (int i = 0; i < NUM_CONSUMERS; i++) {
            final int consumerId = i;
            new Thread(() -> {
                while (true) {
                    try {
                        Task task = queue.take(); // 队列空时阻塞
                        System.out.println("Consumer " + consumerId + " consumed: " + task);
                        process(task);
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }).start();
        }
    }
    
    private static void process(Task task) throws InterruptedException {
        Thread.sleep(500); // 模拟处理时间
    }
}
```

这个示例展示了如何使用 BlockingQueue 解耦生产者和消费者：

- 生产者不断产生任务，放入队列，满时自动阻塞
- 消费者不断从队列取任务处理，空时自动阻塞
- 队列满了，生产者停止生产；队列空了，消费者停止消费

## ArrayBlockingQueue vs LinkedBlockingQueue

| 特性 | ArrayBlockingQueue | LinkedBlockingQueue |
|------|-------------------|-------------------|
| 底层结构 | 有界数组 | 链表节点 |
| 容量 | 必须指定 | 可选，默认 Integer.MAX_VALUE |
| 吞吐量 | 较低 | 较高（入队出队使用独立锁） |
| 内存 | 预分配，固定 | 动态分配，按需增长 |
| 内存占用 | 稳定 | 与元素数量成正比 |

LinkedBlockingQueue 的入队和出队使用两把独立的锁（putLock 和 takeLock），可以实现真正的并发入队出队，吞吐量更高。

## DelayQueue：定时任务调度

DelayQueue 适合实现定时任务调度：

```java
public class DelayedTask implements Delayed {
    private final String name;
    private final long startTime;
    private final long delayMs;
    
    public DelayedTask(String name, long delayMs) {
        this.name = name;
        this.delayMs = delayMs;
        this.startTime = System.currentTimeMillis() + delayMs;
    }
    
    @Override
    public long getDelay(TimeUnit unit) {
        return unit.convert(startTime - System.currentTimeMillis(), TimeUnit.MILLISECONDS);
    }
    
    @Override
    public int compareTo(Delayed other) {
        return Long.compare(this.startTime, ((DelayedTask) other).startTime);
    }
}

// 使用
DelayQueue<DelayedTask> queue = new DelayQueue<>();

queue.put(new DelayedTask("Task1", 5000)); // 5秒后到期
queue.put(new DelayedTask("Task2", 2000)); // 2秒后到期

while (true) {
    DelayedTask task = queue.take(); // 阻塞直到有任务到期
    System.out.println("Executing: " + task.name);
}
```

## SynchronousQueue：直接传递

SynchronousQueue 是一种特殊的队列，它不存储任何元素。每个 put 必须等待一个 take，反之亦然：

```java
SynchronousQueue<Integer> queue = new SynchronousQueue<>();

// 线程 A
new Thread(() -> {
    try {
        Integer value = queue.take();
        System.out.println("Got: " + value);
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
}).start();

// 线程 B
new Thread(() -> {
    try {
        Thread.sleep(1000);
        queue.put(42);
        System.out.println("Put: 42");
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
}).start();
```

SynchronousQueue 适合用于两个线程直接交换数据的场景，比如线程间的信号传递。

## 选择合适的 BlockingQueue

选择正确的 BlockingQueue 需要考虑多个因素：

**需要容量限制**：使用 ArrayBlockingQueue（有界数组）或指定容量的 LinkedBlockingQueue。

**追求高吞吐量**：LinkedBlockingQueue，入队出队可并行。

**需要优先级排序**：PriorityBlockingQueue。

**需要延迟执行**：DelayQueue。

**直接传递数据**：SynchronousQueue。

理解 BlockingQueue 体系，你就能在生产者-消费者场景中做出正确的选择。
