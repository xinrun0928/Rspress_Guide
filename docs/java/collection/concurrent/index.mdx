# 并发容器体系全景

在多线程环境下，普通的集合类（如 HashMap、ArrayList）是不安全的。多个线程同时修改同一个集合，可能导致数据不一致、ConcurrentModificationException，甚至死循环。Java 并发包（java.util.concurrent）提供了一系列线程安全且高性能的并发容器，解决了这一难题。

## 并发容器的必要性

为什么普通集合在多线程环境下不安全？考虑一个简单的场景：

```java
List<String> list = new ArrayList<>();
// 线程 A 执行 list.add("A");
// 线程 B 执行 list.add("B");
```

如果线程 A 和 B 同时执行，ArrayList 的内部数组可能同时被修改，导致数组越界、数据丢失或其他不可预测的问题。

Java 早期使用 `Collections.synchronizedList()` 等包装方法来创建线程安全集合，但这些方法只是在每个方法上加了 synchronized 锁，性能很差，无法满足高并发场景的需求。

## 并发容器的分类

Java 并发包中的容器可以分为几大类：

**并发 Map**：ConcurrentHashMap、ConcurrentSkipListMap 等。提供高并发的键值对存储。

**并发 List**：CopyOnWriteArrayList。适合读多写少的场景。

**并发 Set**：CopyOnWriteArraySet、ConcurrentHashMapKeySet 等。

**并发 Queue**：ConcurrentLinkedQueue、LinkedBlockingQueue、ArrayBlockingQueue、PriorityBlockingQueue、DelayQueue 等。

**并发 Deque**：ConcurrentLinkedDeque、LinkedBlockingDeque。

## 核心设计思想

并发容器的设计围绕两个核心问题：

**如何减少锁竞争**：传统 synchronized 锁的粒度过粗，导致所有线程排队等待。并发容器通过分段锁、无锁算法等手段，让多个线程可以同时操作集合的不同部分。

**如何保证线程安全**：并发修改可能导致数据不一致。并发容器使用 CAS（Compare-And-Swap）、volatile 变量、不可变对象等技术来保证线程安全。

## ConcurrentHashMap 的演进

ConcurrentHashMap 是 Java 并发容器中最重要的一员，它经历了两个重要版本：

**JDK 7：Segment 分段锁**

使用 Segment 数组，每个 Segment 持有一个独立的 HashMap。不同 Segment 之间可以并发操作，显著减少锁竞争。

**JDK 8+：CAS + synchronized**

放弃了 Segment 结构，使用数组加链表/红黑树。每个桶单独加锁（synchronized），同时使用 CAS 进行无锁读取。结构与 HashMap 趋于一致。

## 并发 Queue 的重要性

并发队列（Queue）是实现生产者-消费者模式的核心组件：

```java
BlockingQueue<Task> queue = new LinkedBlockingQueue<>();

// 生产者
new Thread(() -> {
    while (hasMoreTasks()) {
        queue.put(produceTask());
    }
}).start();

// 消费者
for (int i = 0; i < NUM_CONSUMERS; i++) {
    new Thread(() -> {
        while (true) {
            Task task = queue.take();
            process(task);
        }
    }).start();
}
```

BlockingQueue 提供了阻塞的 put 和 take 操作，当队列满时 put 会阻塞，当队列空时 take 会阻塞，非常适合解耦生产者和消费者。

## 选择并发容器的考量

选择合适的并发容器需要考虑多个因素：

**并发度**：期望的并发读写量。ConcurrentHashMap 适合高并发读写，CopyOnWriteArrayList 只适合读多写少。

**一致性**：是否可以容忍弱一致性。ConcurrentHashMap 的迭代器是弱一致的，允许在迭代期间修改。

**阻塞行为**：是否需要阻塞操作。BlockingQueue 提供阻塞的入队出队，适合生产者-消费者场景。

**内存占用**：无锁算法通常需要更多内存（如 ConcurrentHashMap 的 CounterCell）。

理解并发容器的设计原理和使用场景，你才能在多线程编程中做出正确的选择。
