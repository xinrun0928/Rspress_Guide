# TreeSet 与红黑树

如果说 HashSet 是速度的追求者，那 TreeSet 就是有序的守护者。TreeSet 能够让元素自动排序，支持范围查询、子集获取等有序操作。这一切的背后，是一颗精心设计的红黑树在默默支撑。

## TreeSet 的有序世界

TreeSet 与 HashSet 的关系，与 HashMap 和 LinkedHashMap 的关系类似。HashSet 内部是 HashMap，TreeSet 内部是 TreeMap：

```java
public class TreeSet<E> extends AbstractSet<E> 
    implements NavigableSet<E>, Cloneable, Serializable {
    
    private transient NavigableMap<E,Object> m;
    private static final Object PRESENT = new Object();
    
    TreeSet(NavigableMap<E,Object> m) {
        this.m = m;
    }
    
    public TreeSet() {
        this(new TreeMap<E,Object>());
    }
    
    public TreeSet(Comparator<? super E> comparator) {
        this(new TreeMap<>(comparator));
    }
    
    public boolean add(E e) {
        return m.put(e, PRESENT) == null;
    }
}
```

TreeSet 使用 `PRESENT` 作为所有元素的占位值（与 HashSet 相同的技巧），真正的排序逻辑由内部的 `TreeMap` 实现。

## TreeMap：会排序的 Map

TreeMap 是 Java 中唯一直接实现有序 Map 的类。它基于红黑树（Red-Black Tree）实现，能够按照键的自然顺序或自定义比较器顺序来存储键值对。

红黑树是一种自平衡的二叉搜索树，它通过一系列旋转和着色操作来保持树的平衡，确保查找、插入、删除操作的时间复杂度都是 O(log n)。

## 红黑树：平衡的艺术

红黑树的名字来源于每个节点上存储的"颜色"属性，可以是红色或黑色。正是这个简单的颜色标记，使得红黑树能够在插入和删除时通过局部调整来保持整体平衡。

红黑树必须满足以下五个性质：

1. 每个节点或者是红色的，或者是黑色的
2. 根节点是黑色的
3. 每个叶子节点（NIL 节点，空节点）是黑色的
4. 如果一个节点是红色的，则它的两个子节点都是黑色的（即不能有两个连续的红色节点）
5. 对每个节点，从该节点到其所有后代叶子节点的简单路径上，均包含相同数目的黑色节点

这些性质确保了红黑树的高度大致为 O(log n)。由于红黑树的查找、插入、删除操作都需要从根节点走到叶子节点，树的高度直接影响操作的时间复杂度。

## 红黑树的旋转：保持平衡的手段

当插入或删除节点可能破坏红黑树的性质时，需要通过旋转操作来恢复平衡。旋转分为左旋和右旋：

**左旋**：将某个节点旋转为其右子树的根节点，被旋转节点变为其右子节点的左子节点。

```
    p               p
    |               |
    x       -->     y
   / \             / \
  lx  y           x   ry
     / \         / \
    ly ry       lx ly
```

**右旋**：将某个节点旋转为其左子树的根节点，被旋转节点变为其左子节点的右子节点。

```
    p               p
    |               |
    y       -->     x
   / \             / \
  x   ry          lx  y
 / \                 / \
lx ly              ly ry
```

旋转操作本身是 O(1) 的，但插入或删除时可能需要多次旋转来恢复红黑性质。

## 插入操作：变色与旋转

TreeMap 插入新节点时，首先按照二叉搜索树的方式找到插入位置，然后将节点着色为红色（为什么是红色？为了尽量少破坏黑高平衡）。之后通过变色和旋转来恢复红黑树的性质。

插入修复分为三种情况（假设新节点是 z，父节点是 p，叔节点是 u，祖父节点是 g）：

**情况 1**：z 的叔叔节点 u 是红色。此时将 p 和 u 都着色为黑色，g 着色为红色，然后继续向上检查 g。

**情况 2**：z 的叔叔节点 u 是黑色，且 z 是 p 的右子节点。此时对 p 进行左旋，转换为情况 3。

**情况 3**：z 的叔叔节点 u 是黑色，且 z 是 p 的左子节点。此时对 g 进行右旋，交换 p 和 g 的颜色。

这三种情况覆盖了所有可能的插入场景，通过最多两次旋转即可恢复红黑树的平衡。

## Comparable vs Comparator：两种排序方式

TreeMap（以及 TreeSet）支持两种排序方式：自然顺序和比较器顺序。

**Comparable** 是对象自身的排序能力。如果一个类实现了 Comparable 接口，就可以使用自然顺序存储到 TreeMap/TreeSet 中：

```java
public interface Comparable<T> {
    public int compareTo(T o);
}
```

例如，Integer、String 等都实现了 Comparable 接口，可以直接添加到 TreeSet 中并自动排序。

```java
TreeSet<Integer> set = new TreeSet<>();
set.add(5);
set.add(2);
set.add(8);
set.add(1);
System.out.println(set); // [1, 2, 5, 8]
```

**Comparator** 是外部定义的排序能力。如果你不希望修改类的定义，或者需要多种排序方式，可以使用 Comparator：

```java
TreeSet<Person> set = new TreeSet<>(new Comparator<Person>() {
    @Override
    public int compare(Person p1, Person p2) {
        return p1.getAge() - p2.getAge();
    }
});

set.add(new Person("Alice", 25));
set.add(new Person("Bob", 30));
set.add(new Person("Charlie", 20));
// 按年龄排序：[Charlie, Alice, Bob]
```

Comparator 还可以使用 lambda 表达式简化：

```java
TreeSet<Person> set = new TreeSet<>((p1, p2) -> p1.getAge() - p2.getAge());
```

## TreeSet 的独特操作

由于 TreeSet 内部是红黑树，它提供了一系列普通 Set 没有的有序操作：

```java
TreeSet<Integer> set = new TreeSet<>();
set.addAll(Arrays.asList(1, 5, 3, 7, 2, 8, 4, 6));

set.first();                         // 1
set.last();                          // 8
set.lower(5);                        // 4（严格小于 5 的最大元素）
set.higher(5);                       // 6（严格大于 5 的最小元素）
set.floor(5);                        // 5（小于等于 5 的最大元素）
set.ceiling(5);                      // 5（大于等于 5 的最小元素）

set.subSet(2, 6);                    // [2, 3, 4, 5]（前闭后开）
set.headSet(5);                      // [1, 2, 3, 4]
set.tailSet(5);                     // [5, 6, 7, 8]
```

这些方法的时间复杂度都是 O(log n)，因为红黑树支持高效的范围查询。

## 为什么选择红黑树

为什么 Java 的 TreeMap 选择红黑树而不是其他平衡二叉搜索树（如 AVL 树）？

AVL 树比红黑树更加平衡，查找性能更好（最大高度约为 1.44 log n，而红黑树约为 2 log n）。但 AVL 树的插入和删除操作需要更多的旋转来维持平衡。

红黑树的插入最多只需要两次旋转，删除最多只需要三次旋转。虽然平衡性稍差，但换来了更快的修改操作。在大多数应用场景中，修改和查询的频率相近，红黑树是更好的选择。

此外，红黑树与 B 树有深刻的联系——红黑树可以视为 3 阶 B 树的等价表示。这种联系使得红黑树在大规模数据存储和数据库索引中有广泛应用。

理解了红黑树的原理，你不仅掌握了 TreeSet/TreeMap 的核心，也理解了 Java 集合框架中最重要的数据结构之一。
