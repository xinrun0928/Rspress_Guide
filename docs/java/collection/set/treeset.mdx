# TreeSet 与红黑树

你写了一段代码，用 HashSet 存储用户ID：

```java
Set<Integer> ids = new HashSet<>();
ids.add(5);
ids.add(2);
ids.add(8);
ids.add(1);
// 遍历
for (Integer id : ids) {
    System.out.println(id);
}
```

输出是：`5, 2, 8, 1`——乱序的。

你皱了皱眉：「明明按顺序插入的，为什么输出不是 `1, 2, 5, 8`？」

这就是 HashSet 和 TreeSet 的本质区别：**一个用哈希表，一个用红黑树**。

## 为什么需要有序集合？

HashSet 的查找、插入、删除都是 O(1)，性能极好。但它有一个致命缺陷：**无序**。

有些场景下，我们需要：

- 按自然顺序遍历元素
- 快速找到最大/最小元素
- 按范围查询元素
- 保持元素的有序状态

这时，TreeSet 就派上用场了。

## TreeSet 底层结构

TreeSet 内部依赖 **TreeMap**（NavigableMap），而 TreeMap 的底层是 **红黑树**（Red-Black Tree）。

```java
public class TreeSet<E> extends AbstractSet<E>
        implements NavigableSet<E>, Cloneable, Serializable {

    private transient NavigableMap<E, Object> m;

    // 所有元素都映射到一个固定的Value
    private static final Object PRESENT = new Object();

    public TreeSet() {
        m = new TreeMap<>();
    }

    public TreeSet(Comparator<? super E> comparator) {
        m = new TreeMap<>(comparator);
    }

    public boolean add(E e) {
        return m.put(e, PRESENT) == null;
    }
}
```

TreeSet 只是一个「外壳」，真正存储数据的是 TreeMap。TreeMap 使用红黑树来维护 key 的顺序。

## 红黑树：自平衡二叉搜索树

### 二叉搜索树（BST）的问题

普通的二叉搜索树在最坏情况下会退化成链表：

```
      1
       \
        2
         \
          3
           \
            4
```

插入 1, 2, 3, 4, 5 后，树退化为链表，查找变成 O(n)。

### 红黑树的平衡规则

红黑树是一种「近似平衡」的二叉搜索树，通过以下规则保持平衡：

1. 每个节点要么是红色，要么是黑色
2. 根节点是黑色
3. 所有叶子节点（NIL）是黑色
4. 红节点的子节点必须是黑色（不能有连续的红节点）
5. 从任一节点到其每个叶子的路径上，黑节点数量相同

```
        ┌───(B)──┐
       (B)      (R)
      /  \      /  \
    (R)  (B)  (B)  (B)
```

### 平衡效果

红黑树保证：**从根到叶子的最长路径不超过最短路径的 2 倍**。

这意味着红黑树的高度大致在 `O(log n)`，所以查找、插入、删除都是 **O(log n)**。

## 红黑树的操作

### 插入：O(log n)

```java
public class RedBlackTree {
    private static final boolean RED = true;
    private static final boolean BLACK = false;

    private class Node {
        int key;
        boolean color;
        Node left, right, parent;
    }

    // 左旋
    private void rotateLeft(Node x) {
        Node y = x.right;
        x.right = y.left;
        if (y.left != null) y.left.parent = x;
        y.parent = x.parent;
        if (x.parent == null) root = y;
        else if (x == x.parent.left) x.parent.left = y;
        else x.parent.right = y;
        y.left = x;
        x.parent = y;
    }

    // 右旋
    private void rotateRight(Node x) {
        Node y = x.left;
        x.left = y.right;
        if (y.right != null) y.right.parent = x;
        y.parent = x.parent;
        if (x.parent == null) root = y;
        else if (x == x.parent.right) x.parent.right = y;
        else x.parent.left = y;
        y.right = x;
        x.parent = y;
    }
}
```

插入新节点后，需要通过「旋转」和「重新着色」来恢复红黑树的性质。

### 删除：O(log n)

删除操作更复杂，需要考虑三种情况：

1. 删除叶子节点：直接删除
2. 删除有一个孩子的节点：用孩子替换
3. 删除有两个孩子的节点：用后继节点替换，然后删除后继

## TreeSet 的核心方法

### 基本操作

```java
TreeSet<Integer> set = new TreeSet<>();

// 添加
set.add(5);
set.add(2);
set.add(8);
set.add(1);
set.add(9);

// 遍历（自动有序）
for (Integer n : set) {
    System.out.println(n);  // 输出: 1, 2, 5, 8, 9
}

// 大小
set.size();  // 5

// 是否包含
set.contains(3);  // false
set.contains(5);  // true
```

### 导航方法（TreeSet 独有）

```java
TreeSet<Integer> set = new TreeSet<>();
set.addAll(Arrays.asList(1, 3, 5, 7, 9, 11, 13));

// 找最近的元素
set.lower(7);   // 小于 7 的最大元素: 5
set.floor(7);   // 小于等于 7 的最大元素: 7
set.higher(7);  // 大于 7 的最小元素: 9
set.ceiling(7); // 大于等于 7 的最小元素: 7

// 第一个和最后一个
set.first();  // 1
set.last();   // 13

// 子集
set.subSet(3, 9);   // [3, 9): 3, 5, 7
set.subSet(3, true, 9, true);  // [3, 9]: 3, 5, 7, 9
set.headSet(7);     // 小于 7: 1, 3, 5
set.tailSet(7);      // 大于等于 7: 7, 9, 11, 13
```

### 删除操作

```java
TreeSet<String> set = new TreeSet<>();
set.addAll(Arrays.asList("apple", "banana", "cherry", "date"));

// 删除
set.remove("banana");  // true

// 删除第一个/最后一个
set.pollFirst();  // 返回并删除 "apple"
set.pollLast();   // 返回并删除 "date"

// 清空
set.clear();
```

## 自定义排序：Comparator

### 使用自然顺序

```java
TreeSet<Integer> set = new TreeSet<>();
// 默认使用自然顺序（Integer.compareTo）
```

### 使用自定义比较器

```java
// 按字符串长度排序
TreeSet<String> set = new TreeSet<>((a, b) -> {
    int cmp = a.length() - b.length();
    if (cmp != 0) return cmp;
    return a.compareTo(b);  // 长度相同按字典序
});

set.add("apple");
set.add("香蕉");
set.add("hi");
set.add("长颈鹿");

for (String s : set) {
    System.out.println(s);  // hi, apple, 香蕉, 长颈鹿
}
```

### 使用 Collections.reverseOrder

```java
// 降序排列
TreeSet<Integer> descSet = new TreeSet<>(Collections.reverseOrder());
descSet.addAll(Arrays.asList(3, 1, 4, 1, 5, 9, 2, 6));
for (Integer n : descSet) {
    System.out.println(n);  // 9, 6, 5, 4, 3, 2, 1
}
```

## HashSet vs TreeSet

| 特性 | HashSet | TreeSet |
|---|---|---|
| 底层结构 | 哈希表 | 红黑树 |
| 元素顺序 | 无序 | 有序 |
| `add()` 时间复杂度 | O(1) | O(log n) |
| `contains()` 时间复杂度 | O(1) | O(log n) |
| 额外操作 | 无 | `lower()`、`floor()`、`subSet()` 等 |
| 需要元素实现 | `hashCode()`、`equals()` | `Comparable` 或 `Comparator` |
| 空元素支持 | 支持（一个 null） | 不支持（除非用 `Comparator`） |

## 实战场景

### 场景一：排行榜

```java
TreeSet<Player> leaderboard = new TreeSet<>(Comparator
    .comparingInt(Player::getScore).reversed()
    .thenComparing(Player::getName));

leaderboard.add(new Player("Alice", 1000));
leaderboard.add(new Player("Bob", 1500));
leaderboard.add(new Player("Charlie", 1000));

// 获取前 3 名
int count = 0;
for (Player p : leaderboard) {
    if (count++ >= 3) break;
    System.out.println(p.getName() + ": " + p.getScore());
}
```

### 场景二：航班时刻表

```java
TreeSet<LocalTime> departures = new TreeSet<>();
departures.add(LocalTime.of(8, 30));
departures.add(LocalTime.of(10, 15));
departures.add(LocalTime.of(14, 0));
departures.add(LocalTime.of(18, 45));

// 找下一班航班
LocalTime now = LocalTime.of(11, 0);
LocalTime next = departures.higher(now);  // 14:00
```

## 总结

TreeSet 是 Java 集合框架中「有序」的代名词：

- **底层是红黑树**：O(log n) 的查找、插入、删除
- **自动排序**：元素按自然顺序或自定义顺序排列
- **导航方法**：`lower()`、`floor()`、`subSet()` 等是 HashSet 没有的能力
- **选择原则**：需要有序时选 TreeSet，追求极致 O(1) 性能时选 HashSet

理解红黑树的平衡机制，才能真正理解 TreeSet 为什么能做到「既有序又高效」。
