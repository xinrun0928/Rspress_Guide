# ArrayList 扩容机制与位运算

如果问你 Java 中最常用的类是什么，`ArrayList` 一定榜上有名。它简单、高效、几乎出现在每一个 Java 项目的代码里。但在这份简单的背后，有一个精巧而强大的机制支撑着它的灵活性——扩容机制。理解了这个机制，你就理解了 `ArrayList` 乃至整个集合框架的设计精髓。

## 从一个谜题开始

为什么 `ArrayList` 的初始容量是 10？为什么扩容因子是 1.5？为什么使用 `oldCapacity >> 1` 而不是 `oldCapacity * 1.5`？为什么最大容量是 `Integer.MAX_VALUE - 8`？

这些问题看似琐碎，但每一道后面都藏着 Java 设计者的深思熟虑。让我们一层层揭开 `ArrayList` 扩容机制的神秘面纱。

## ArrayList 的内部结构

`ArrayList` 的核心只有一个数组：

```java
public class ArrayList<E> {
    transient Object[] elementData;  // 存储元素的数组
    private int size;                 // 当前元素数量
}
```

就这么简单。一个数组 + 一个计数器，构成了 `ArrayList` 的全部。数组的长度就是容量，`size` 是当前元素数量。两者不一定是相等的——容量可能大于元素数量。

## 容量与大小的区别

这是一个经常被误解的概念。**容量（Capacity）**是数组的长度，表示底层数组能容纳多少元素。**大小（Size）**是实际存储的元素数量。

```java
ArrayList<String> list = new ArrayList<>(20);  // 容量是20
list.add("A");                                  // 大小是1
list.add("B");                                  // 大小是2
System.out.println(list.size());                // 输出2，不是20
```

当你创建一个 `ArrayList` 时，容量并不立即分配 20 个对象的空间——它只是创建了一个能容纳 20 个引用的数组。只有当你真正添加元素时，才会创建具体的对象。

## 扩容的触发时机

`ArrayList` 的扩容发生在添加元素时，而不是创建对象时：

```java
public boolean add(E e) {
    ensureCapacityInternal(size + 1);  // 确保容量足够
    elementData[size++] = e;
    return true;
}

public void add(int index, E e) {
    rangeCheck(index);
    ensureCapacityInternal(size + 1);
    System.arraycopy(elementData, index, elementData, index + 1, size - index);
    elementData[index] = e;
    size++;
}
```

无论是 `add(E e)` 尾部添加，还是 `add(int index, E e)` 中间插入，都需要先确保容量足够。扩容的触发条件是：`size + 1 > elementData.length`。

## 扩容的完整流程

让我们追踪一次完整的扩容过程：

```java
private void ensureCapacityInternal(int minCapacity) {
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        // 第一次添加元素时
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
    }
    ensureExplicitCapacity(minCapacity);
}

private void ensureExplicitCapacity(int minCapacity) {
    modCount++;  // 修改计数器，用于fail-fast机制
    if (minCapacity - elementData.length > 0) {
        grow(minCapacity);
    }
}

private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1);  // 位运算：oldCapacity * 1.5
    
    if (newCapacity - minCapacity < 0) {
        newCapacity = minCapacity;  // 如果1.5倍还不够，直接用所需的最小容量
    }
    
    if (newCapacity - MAX_ARRAY_SIZE > 0) {
        newCapacity = hugeCapacity(minCapacity);
    }
    
    elementData = Arrays.copyOf(elementData, newCapacity);
}
```

### 位运算：oldCapacity >> 1

这里有一个精妙的位运算：`oldCapacity >> 1`。

`>>` 是有符号右移运算符，`>> 1` 表示除以 2 并向下取整。所以 `oldCapacity + (oldCapacity >> 1)` 等价于 `oldCapacity + oldCapacity/2 = oldCapacity * 1.5`。

为什么使用位运算而不是直接乘以 1.5？

**第一，执行效率更高**。在底层，位运算只需要一个 CPU 指令，而乘法和浮点运算需要更多 cycles。虽然现代 CPU 上这个差异已经很小，但在 Java 早期版本中，这是重要的优化。

**第二，结果必然是整数**。数组容量必须是整数，`oldCapacity * 1.5` 在 oldCapacity 为奇数时会产生浮点数，而位运算 `>> 1` 总是产生整数。

**第三，历史习惯**。C/C++ 风格的代码中，位运算常用于性能敏感的场景。Java 早期很多代码都能看到这种风格的影响。

## 扩容示例

让我们通过具体例子理解扩容过程：

```java
ArrayList<Integer> list = new ArrayList<>();

// 第一次 add
list.add(1);  // size=1, capacity=10（延迟初始化）

// 第十次 add
list.add(10); // size=10, capacity=10

// 第十一次 add，触发扩容
list.add(11); // size=11, capacity=15 (10 * 1.5 = 15)

// 容量增长：10 -> 15 -> 22 -> 33 -> 49 -> 73 -> 109 -> 163 -> 244 ...
```

每次扩容大约增加 50% 的容量。这种增长策略在时间（扩容次数）和空间（预分配内存）之间取得了平衡。

## 为什么是 1.5 倍？

1.5 这个数字是经过精心选择的。如果扩容太少（比如 1.1 倍），会频繁扩容；如果扩容太多（比如 2 倍），会浪费空间。

1.5 倍的优势在于：

1. **空间效率较高**：平均预留 50% 的空闲空间，既不会太浪费，又能提供足够的增长空间。
2. **扩容次数适中**：相比 1.2 倍，扩容次数明显减少；相比 2 倍，空间利用率更高。
3. **数学特性好**：1.5 = 3/2，可以用移位和加法实现，避免浮点运算。

理论上，我们可以根据具体场景自定义扩容策略。但实际上，`ArrayList` 的默认策略对大多数应用来说已经足够好。

## 默认容量 10 的由来

当你使用无参构造函数创建 `ArrayList` 时：

```java
public ArrayList() {
    this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
}
```

它创建的是一个空数组！真正的默认容量 10 是在第一次 `add()` 时赋予的：

```java
private void ensureCapacityInternal(int minCapacity) {
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
        // DEFAULT_CAPACITY = 10
    }
}
```

为什么采用延迟初始化？因为并非所有 `ArrayList` 都会被使用，很多可能只是声明后被传递或返回。延迟初始化节省了不必要的内存分配。

那为什么默认值是 10？这是一个经验值。10 是一个"足够大又不太大"的数字——对于大多数小型集合来说够用，对于大型集合来说，初始的几步扩容也很快追上。

## MAX_ARRAY_SIZE 的秘密

`ArrayList` 有一个最大容量限制，不是 `Integer.MAX_VALUE`，而是 `Integer.MAX_VALUE - 8`：

```java
private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;

private static int hugeCapacity(int minCapacity) {
    if (minCapacity < 0) throw new OutOfMemoryError();
    return (minCapacity > MAX_ARRAY_SIZE) ?
        Integer.MAX_VALUE : MAX_ARRAY_SIZE;
}
```

为什么要减 8？这是为了在某些 JVM 实现中避免内存对齐带来的问题。一些 JVM 会在数组对象头中存储数组长度，如果数组过长可能导致整数溢出。减 8 留出了安全余量。

## 扩容的开销

扩容操作的核心是数组拷贝：`Arrays.copyOf(elementData, newCapacity)`。这是一个 O(n) 的操作，涉及到：

1. 分配新的数组内存
2. 将原数组的所有元素复制到新数组
3. GC 回收旧数组（如果没有其他引用）

对于大型 `ArrayList`，这可能是一个昂贵的操作。这就是为什么在已知元素数量时，预先指定初始容量是一个好习惯：

```java
// 预估容量，避免扩容
List<MyObject> list = new ArrayList<>(expectedSize);
```

## trimToSize：收缩艺术

`ArrayList` 还提供了 `trimToSize()` 方法，可以将容量收缩到当前大小：

```java
public void trimToSize() {
    if (size < elementData.length) {
        elementData = (size == 0)
            ? EMPTY_ELEMENTDATA
            : Arrays.copyOf(elementData, size);
    }
}
```

这个方法在确定不再添加元素后很有用，可以释放多余的内存：

```java
ArrayList<String> list = new ArrayList<>();
// ... 添加元素 ...
list.trimToSize();  // 容量 = size，节省内存
```

## 扩容与迭代：ConcurrentModificationException

`ArrayList` 的扩容机制还与一个常见的异常有关——`ConcurrentModificationException`。这个异常通常在迭代过程中修改集合时抛出：

```java
ArrayList<String> list = new ArrayList<>(Arrays.asList("a", "b", "c"));

for (String s : list) {  // 迭代器遍历
    if (s.equals("b")) {
        list.remove(s);   // 直接修改集合，抛出异常！
    }
}
```

`ArrayList` 维护了一个 `modCount` 变量，每次修改（添加、删除、清空）都会递增。迭代器在创建时会记录初始的 `modCount`，每次迭代时会检查是否变化。如果检测到变化，就会抛出 `ConcurrentModificationException`。

正确的做法是使用迭代器的 `remove()` 方法：

```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().equals("b")) {
        it.remove();  // 使用迭代器删除，不触发异常
    }
}
```

## 扩容机制的演进

`ArrayList` 的扩容机制在 JDK 8 有一个小变化：引入了 `DEFAULTCAPACITY_EMPTY_ELEMENTDATA` 和 `EMPTY_ELEMENTDATA` 两个空数组实例的区别。

```java
private static final Object[] DEFAULTCAPACITY_EMPTY_ELEMENTDATA = {};
private static final Object[] EMPTY_ELEMENTDATA = {};
```

这个优化的目的是：**避免无参构造的 ArrayList 和指定容量为 0 的 ArrayList 在第一次添加时产生不同的扩容行为**。

- 无参构造：`minCapacity` 会与 `DEFAULT_CAPACITY(10)` 比较，取较大值
- 指定容量 0：`minCapacity` 直接使用

## 写在最后

`ArrayList` 的扩容机制是 Java 集合框架设计精髓的缩影。它用简单的位运算实现了高效的容量增长，用延迟初始化节省了不必要的开销，用 `MAX_ARRAY_SIZE` 考虑了 JVM 的限制，用 `modCount` 维护了迭代的安全性。

理解这些细节，不仅能帮助你在面试中回答"ArrayList 扩容机制"这类问题，更能让你在实际开发中写出更高效的代码。记住：**预估容量，避免扩容**——这八个字是使用 `ArrayList` 的第一原则。
