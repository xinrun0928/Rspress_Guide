# Collection 接口结构与分类

面试官问你：「Java 集合框架都有哪些接口？」

你脱口而出：「List、Set、Queue、Map 四大接口。」

面试官点点头，又追问：「那 `Collection` 和 `Map` 是什么关系？」

你愣了一下——这个问题看似基础，却难住了不少人。

## 集合框架的两大家族

Java 集合框架（JCF）实际上分为两大体系：**Collection** 和 **Map**。

```
Iterable
  └── Collection
        ├── List（有序、可重复）
        │     ├── ArrayList
        │     ├── LinkedList
        │     └── Vector
        ├── Set（无序、去重）
        │     ├── HashSet
        │     ├── LinkedHashSet
        │     └── TreeSet
        └── Queue（队列、FIFO）
              ├── LinkedList
              ├── PriorityQueue
              └── BlockingQueue
  └── Map（键值对）
        ├── HashMap
        ├── LinkedHashMap
        ├── TreeMap
        └── Hashtable
```

这个继承关系有一个关键点：**`Map` 不是 `Collection` 的子接口**。它们是两个独立的根接口，这是 Java 集合设计中最容易被误解的地方。

## Iterable：集合的迭代之源

`Iterable` 是所有集合的顶级接口，它定义了集合的迭代能力：

```java
public interface Iterable<T> {
    Iterator<T> iterator();
    default void forEach(Consumer<? super T> action);
    default Spliterator<T> spliterator();
}
```

`Iterable` 接口的引入，使得所有实现了它的集合都支持 `for-each` 循环：

```java
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
for (String name : names) {
    System.out.println(name);
}
```

## Collection 接口：三大子接口的共性

`Collection` 是 List、Set、Queue 的共同父接口，定义了集合的通用操作：

```java
public interface Collection<E> extends Iterable<E> {
    int size();                    // 元素数量
    boolean isEmpty();             // 是否为空
    boolean contains(Object o);   // 是否包含元素
    Iterator<E> iterator();       // 获取迭代器

    // 操作方法
    boolean add(E e);             // 添加元素
    boolean remove(Object o);     // 移除元素
    boolean addAll(Collection<? extends E> c);  // 批量添加

    // 批量操作
    void clear();                  // 清空
    boolean containsAll(Collection<?> c);  // 是否包含所有
    boolean removeAll(Collection<?> c);     // 批量移除
    boolean retainAll(Collection<?> c);     // 取交集

    // 转换
    Object[] toArray();            // 转为数组
    <T> T[] toArray(T[] a);     // 指定类型数组
}
```

### List：有序且可重复

`List` 是有序集合，允许重复元素，每个元素都有索引：

```java
public interface List<E> extends Collection<E> {
    // 索引操作
    E get(int index);
    E set(int index, E element);
    void add(int index, E element);
    E remove(int index);

    // 搜索
    int indexOf(Object o);
    int lastIndexOf(Object o);

    // 视图
    ListIterator<E> listIterator();
    ListIterator<E> listIterator(int index);
    List<E> subList(int fromIndex, int toIndex);
}
```

List 额外提供了基于索引的操作，以及 `ListIterator`（双向迭代器）。

### Set：不包含重复元素

`Set` 是无重复元素的集合：

```java
public interface Set<E> extends Collection<E> {
    // Set 没有新增方法
    // 但语义不同：add() 如果元素已存在，返回 false
}
```

Set 继承了 Collection 的所有方法，但语义不同：

- `add()`：如果元素已存在，返回 `false`
- `addAll()`：取并集，自动去重

### Queue：队列的抽象

`Queue` 是在 Collection 基础上增加了队列语义：

```java
public interface Queue<E> extends Collection<E> {
    // 插入
    boolean offer(E e);    // 队列满时返回 false（不抛异常）
    boolean add(E e);     // 队列满时抛异常

    // 移除
    E poll();              // 队列空时返回 null（不抛异常）
    E remove();            // 队列空时抛异常

    // 查看
    E peek();              // 队列空时返回 null
    E element();           // 队列空时抛异常
}
```

这里体现了 Java 设计的一个原则：**操作失败时，用两种方式处理**：

- `xxx()` 方法：抛异常
- `yyy()` 方法：返回特殊值（`false` / `null`）

### Deque：双端队列

`Deque` 是 Queue 的扩展，支持两端操作：

```java
public interface Deque<E> extends Queue<E> {
    // 队首操作
    void addFirst(E e);
    void addLast(E e);
    E removeFirst();
    E removeLast();
    E getFirst();
    E getLast();

    // 对应的安全版本
    boolean offerFirst(E e);
    boolean offerLast(E e);
    E pollFirst();
    E pollLast();
    E peekFirst();
    E peekLast();
}
```

`LinkedList` 同时实现了 `List` 和 `Deque`，因此既可以当列表用，也可以当队列用：

```java
LinkedList<String> deque = new LinkedList<>();

// 当 Queue 使用（FIFO）
deque.offer("A");
deque.offer("B");
deque.poll();  // "A"

// 当 Stack 使用（LIFO）
deque.push("C");
deque.pop();   // "C"

// 当 Deque 使用
deque.offerFirst("D");
deque.offerLast("E");
```

## Map：键值对的映射

`Map` 是独立于 Collection 的接口家族：

```java
public interface Map<K, V> {
    // 基本操作
    int size();
    boolean isEmpty();
    boolean containsKey(Object key);
    boolean containsValue(Object value);
    V get(Object key);
    V put(K key, V value);
    V remove(Object key);

    // 批量操作
    void putAll(Map<? extends K, ? extends V> m);
    void clear();

    // 视图
    Set<K> keySet();
    Collection<V> values();
    Set<Map.Entry<K, V>> entrySet();

    // Entry 接口
    interface Entry<K, V> {
        K getKey();
        V getValue();
        V setValue(V value);
    }
}
```

### Map 与 Collection 的关键区别

| 特性 | Map | Collection |
|---|---|---|
| 元素结构 | 键值对 `Entry<K, V>` | 单元素 |
| 父接口 | `Object` | `Iterable` |
| 唯一性 | 键唯一，值可重复 | Set 唯一、List 可重复 |
| 空值支持 | 键最多一个 null，值可有多个 null | 取决于具体实现 |

## 三大集合的适用场景

| 集合类型 | 特点 | 适用场景 |
|---|---|---|
| `ArrayList` | 随机访问 O(1)，插入删除 O(n) | 频繁随机访问，少量增删 |
| `LinkedList` | 插入删除 O(1)，随机访问 O(n) | 频繁增删，少量随机访问 |
| `HashSet` | 无序，去重，查找 O(1) | 需要去重，不需要顺序 |
| `TreeSet` | 有序，去重，查找 O(log n) | 需要有序遍历 |
| `HashMap` | 键值对，查找 O(1) | 键值映射存储 |
| `TreeMap` | 有序键值对 | 需要按键排序 |

## 不可修改集合

Java 9 引入了 `List.of()`、`Set.of()`、`Map.of()` 快速创建不可修改集合：

```java
List<String> immutable = List.of("A", "B", "C");
// immutable.add("D");  // UnsupportedOperationException
```

这些集合适用于：

- 防御性编程
- 作为常量集合
- 避免并发修改问题

## 总结

Java 集合框架的设计体现了几个核心原则：

1. **接口与实现分离**：接口定义契约，实现类负责细节
2. **两大家族并行**：`Collection` 和 `Map` 是独立的体系
3. **语义明确**：`List` 有序、`Set` 去重、`Queue` 队列、`Map` 映射
4. **设计一致**：`xxx()` vs `yyy()` 的异常/返回值约定

理解了这个继承结构，你就掌握了 Java 集合框架的「地图」，遇到任何集合相关的问题，都能快速定位答案。
