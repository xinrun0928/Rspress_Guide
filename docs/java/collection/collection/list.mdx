# List：ArrayList、LinkedList、Vector

凌晨两点，你正在排查一个奇怪的性能问题——系统负载不高，但某个接口响应时间忽快忽慢。

你翻开代码，发现同事写了一个「万能列表」：

```java
List<Object> list = new ArrayList<>();
// ... 各种 add、remove 操作
```

问题就在这里：**ArrayList 的随机删除是 O(n)，而 LinkedList 才是 O(1)**。

但如果你问他为什么选 ArrayList，他可能会说：「ArrayList 不是比 LinkedList 快吗？」

这正是今天要聊的话题——**三种 List 的真实性能与适用场景**。

## ArrayList：动态数组的默认选择

`ArrayList` 是 Java 中最常用的 List 实现，本质上是一个**动态数组**。

### 底层结构

```java
public class ArrayList<E> extends AbstractList<E>
        implements List<E>, RandomAccess, Cloneable, Serializable {
    transient Object[] elementData;  // 存储元素的数组
    private static final int DEFAULT_CAPACITY = 10;  // 默认容量

    // 扩容逻辑
    private void grow(int minCapacity) {
        int oldCapacity = elementData.length;
        int newCapacity = oldCapacity + (oldCapacity >> 1);  // 1.5 倍扩容
        elementData = Arrays.copyOf(elementData, newCapacity);
    }
}
```

### 核心操作复杂度

| 操作 | 时间复杂度 | 说明 |
|---|---|---|
| `get(int index)` | O(1) | 数组直接索引 |
| `set(int index, E e)` | O(1) | 数组直接索引 |
| `add(E e)` | O(1) 均摊 | 偶尔扩容时 O(n) |
| `add(int index, E e)` | O(n) | 移动 index 之后的所有元素 |
| `remove(int index)` | O(n) | 移动被删除后的所有元素 |
| `contains(E e)` | O(n) | 线性搜索 |

### ArrayList 的优势

1. **随机访问极快**：数组的索引操作是 CPU 直接计算的，O(1) 复杂度
2. **内存连续**：CPU 缓存亲和度高，遍历速度快
3. **空间效率高**：只有 elementData 一个数组，没有额外指针开销

```java
ArrayList<Integer> list = new ArrayList<>(1000);  // 预分配容量
for (int i = 0; i < 1000; i++) {
    list.add(i);  // 避免频繁扩容
}
```

### ArrayList 的劣势

1. **中间插入/删除慢**：需要移动大量元素
2. **扩容有成本**：扩容时需要复制整个数组
3. **线程不安全**：多线程并发修改需要额外同步

## LinkedList：双向链表的优雅

`LinkedList` 是双向链表的实现，同时实现了 `List` 和 `Deque` 接口。

### 底层结构

```java
public class LinkedList<E> extends AbstractSequentialList<E>
        implements List<E>, Deque<E>, Cloneable, Serializable {
    transient Node<E> first;  // 头节点
    transient Node<E> last;    // 尾节点

    private static class Node<E> {
        E item;
        Node<E> prev;
        Node<E> next;
    }
}
```

每个节点包含数据域和两个指针（指向前后节点），形成双向链表。

### 核心操作复杂度

| 操作 | 时间复杂度 | 说明 |
|---|---|---|
| `get(int index)` | O(n) | 需要从头/尾遍历 |
| `set(int index, E e)` | O(n) | 需要先找到位置 |
| `add(E e)` | O(1) | 尾部追加 |
| `add(int index, E e)` | O(1) 找到位置后 | 关键是找到位置 |
| `addFirst(E e)` | O(1) | 头部插入 |
| `remove(int index)` | O(1) 找到位置后 | 链表节点删除 |
| `removeFirst()` | O(1) | 头部删除 |

### LinkedList 的优势

1. **首尾操作快**：只需要操作头尾指针
2. **中间插入/删除快**：找到位置后，调整指针即可
3. **无容量限制**：不需要预分配，不会扩容

```java
LinkedList<Integer> list = new LinkedList<>();
list.addFirst(1);    // 头部插入：O(1)
list.addLast(3);     // 尾部插入：O(1)
list.add(1, 2);      // 中间插入：找到位置后 O(1)
list.removeFirst();  // 头部删除：O(1)
list.removeLast();   // 尾部删除：O(1)
```

### LinkedList 的劣势

1. **随机访问慢**：需要从头/尾遍历
2. **内存开销大**：每个节点需要额外存储前后指针
3. **CPU 缓存不友好**：节点可能分散在堆的各个位置
4. **遍历慢**：`forEach` 时每次都需要 hasNext() 检查

## Vector：线程安全的古老实现

`Vector` 是最早期的 List 实现，与 ArrayList 的主要区别是**线程安全**。

### 同步机制

```java
public class Vector<E> extends AbstractList<E>
        implements List<E>, RandomAccess, Cloneable, Serializable {
    protected Object[] elementData;

    // 所有公开方法都加了 synchronized
    public synchronized boolean add(E e) {
        modCount++;
        ensureCapacityHelper(elementCount + 1);
        elementData[elementCount++] = e;
        return true;
    }

    public synchronized E get(int index) {
        if (index >= elementCount)
            throw new ArrayIndexOutOfBoundsException(index);
        return elementData(index);
    }
}
```

### Vector vs ArrayList

| 特性 | Vector | ArrayList |
|---|---|---|
| 线程安全 | synchronized 方法 | 非同步 |
| 性能 | 同步开销大 | 无同步开销，更快 |
| 扩容策略 | 2 倍扩容 | 1.5 倍扩容 |
| 迭代器 | 迭代时修改抛异常 | 迭代时修改抛异常 |

### 何时使用 Vector

现代 Java 中，几乎不需要使用 Vector。但如果你的代码需要：

1. **向后兼容**：维护几十年前的老代码
2. **简单线程安全**：不需要复杂的并发控制

更推荐使用 `Collections.synchronizedList()` 包装，或者直接使用 `CopyOnWriteArrayList`。

## 如何选择

选择 List 实现时，遵循以下原则：

| 场景 | 推荐 | 原因 |
|---|---|---|
| 随机访问为主 | `ArrayList` | O(1) 随机访问 |
| 只在尾部增删 | `ArrayList` | 扩容成本可控 |
| 队列/栈 | `LinkedList` | 首尾操作 O(1) |
| 频繁中间增删 | `LinkedList` | O(1) 增删（找到位置后） |
| 需要线程安全 | `CopyOnWriteArrayList` | 读多写少场景 |
| 不知道选什么 | `ArrayList` | 默认选择 |

## 最佳实践

### 预分配容量

```java
// 预估容量，避免频繁扩容
ArrayList<String> list = new ArrayList<>(expectedSize);
```

### 选择合适的迭代方式

```java
List<String> list = new ArrayList<>();

// ArrayList 推荐：索引循环
for (int i = 0; i < list.size(); i++) {
    // 使用 i 进行计算或访问
}

// LinkedList 推荐：迭代器或增强 for
for (String s : list) {
    // 不要用索引循环
}

// LinkedList 如果必须用索引
ListIterator<String> it = list.listIterator();
while (it.hasNext()) {
    // it.nextIndex() 获取当前位置
}
```

## 总结

三种 List 各有特点：

- **ArrayList**：动态数组，随机访问之王，尾部追加效率高
- **LinkedList**：双向链表，首尾操作快，中间增删有优势
- **Vector**：线程安全，但性能已落后于 modern 并发集合

记住一个原则：**没有最好的 List，只有最合适的 List**。根据你的操作模式选择实现，才能写出高性能的代码。
