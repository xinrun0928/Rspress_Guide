# List：ArrayList、LinkedList、Vector

List 是 Java 集合框架中最常用的接口之一，它代表一个有序、可重复的元素序列。三国鼎立，群雄逐鹿——在 List 的世界里，`ArrayList`、`LinkedList` 和 `Vector` 三分天下，各有各的地盘和粉丝。

## ArrayList：数组的华丽转身

如果说数据结构世界也有"草根逆袭"的故事，那一定是 `ArrayList` 的故事。它用最朴素的数据结构——数组——创造了一个高效、灵活、几乎无处不在的列表实现。

`ArrayList` 的核心其实就是一个 `Object[]` 数组：

```java
public class ArrayList<E> extends AbstractList<E>
        implements List<E>, RandomAccess, Cloneable, java.io.Serializable {
    
    transient Object[] elementData;
    private int size;
    // ...
}
```

这个设计看似简单，却暗藏玄机。数组本身是固定长度的，但 `ArrayList` 通过**动态扩容**机制，在保持数组随机访问 O(1) 性能优势的同时，解决了容量固定的问题。

### ArrayList 的时间复杂度

`ArrayList` 的各操作时间复杂度如下：

- **随机访问**：`get(index)` — O(1)。这是数组的天然优势，通过索引直接计算内存地址。
- **尾部操作**：`add(element)` — O(1) amortized。均摊复杂度是常数时间，因为扩容不是每次都发生。
- **头部/中间插入删除**：`add(index, element)`、`remove(index)` — O(n)。需要移动后续所有元素。
- **查找元素**：`contains()`、`indexOf()` — O(n)。需要遍历数组进行equals比较。

这种复杂度特征决定了 `ArrayList` 的最佳使用场景：**写少读多，或者主要在尾部进行操作**。

### ArrayList 的构造函数

`ArrayList` 提供了三个构造函数：

```java
// 默认构造，初始容量为10（第一次add时才真正创建数组）
public ArrayList() {
    this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
}

// 指定初始容量
public ArrayList(int initialCapacity) {
    if (initialCapacity > 0) {
        this.elementData = new Object[initialCapacity];
    } else if (initialCapacity == 0) {
        this.elementData = EMPTY_ELEMENTDATA;
    } else {
        throw new IllegalArgumentException("Illegal Capacity: " + initialCapacity);
    }
}

// 从已有集合创建
public ArrayList(Collection<? extends E> c) {
    elementData = c.toArray();
    if ((size = elementData.length) != 0) {
        // ...
    } else {
        this.elementData = EMPTY_ELEMENTDATA;
    }
}
```

值得注意的是，无参构造函数实际上创建的是一个空数组！真正的默认容量 10 是在第一次 `add()` 时才赋予的。这种"延迟初始化"的策略既节省了内存，又保证了性能。

### ArrayList 的扩容机制

这是面试中的高频考点。`ArrayList` 的扩容发生在 `add()` 方法中：

```java
public boolean add(E e) {
    ensureCapacityInternal(size + 1);  // Increments modCount!!
    elementData[size++] = e;
    return true;
}

private void ensureCapacityInternal(int minCapacity) {
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
    }
    ensureExplicitCapacity(minCapacity);
}

private void ensureExplicitCapacity(int minCapacity) {
    modCount++;
    if (minCapacity - elementData.length > 0)
        grow(minCapacity);
}

private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1);  // 扩容为原来的1.5倍
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    elementData = Arrays.copyOf(elementData, newCapacity);  // 数组拷贝
}
```

扩容过程可以总结为：

1. 容量不足时，调用 `grow()` 方法
2. 新容量 = 旧容量 × 1.5（通过 `oldCapacity + (oldCapacity >> 1)` 实现）
3. 如果还不够，直接使用所需容量
4. 最后通过 `Arrays.copyOf()` 创建新数组并拷贝数据

为什么选择 1.5 倍而不是 2 倍？这是一个时间和空间权衡的结果。1.5 倍增量相对较小，可以更节省内存；而 2 倍增量虽然扩容次数更少，但可能造成更多的空间浪费。

### ArrayList 的线程安全性

`ArrayList` 是**非线程安全**的。在多线程环境下，可能出现以下问题：

```java
// 线程A和线程B同时add
// 线程A: elementData[size] = e;
// 线程B: elementData[size] = f;
// 结果：后写入的元素覆盖了先写入的，造成数据丢失
```

如果你需要在多线程环境中使用 List，可以选择：
- `Collections.synchronizedList(new ArrayList<>())` — 包装成同步版本
- `CopyOnWriteArrayList` — 读写分离的并发 List
- `Vector` — 古老的同步实现（不推荐）

## LinkedList：链表的力量

与 `ArrayList` 的数组底层不同，`LinkedList` 采用**双向链表**作为底层数据结构：

```java
public class LinkedList<E>
        extends AbstractSequentialList<E>
        implements List<E>, Deque<E>, Cloneable, java.io.Serializable {
    
    transient Node<E> first;
    transient Node<E> last;
    private int size = 0;
    
    private static class Node<E> {
        E item;
        Node<E> next;
        Node<E> prev;
    }
}
```

每个元素被封装成一个 `Node`，包含数据本身、指向前一个节点的引用和指向后一个节点的引用。这种结构带来了一些独特的特性。

### LinkedList 的时间复杂度

- **随机访问**：`get(index)` — O(n)。需要从头或尾遍历到指定位置。
- **头部操作**：`addFirst()`、`removeFirst()` — O(1)。
- **尾部操作**：`addLast()`、`removeLast()` — O(1)。
- **查找元素**：`contains()`、`indexOf()` — O(n)。

可以看出，`LinkedList` 的优势在于头部和尾部的 O(1) 操作，以及插入和删除的高效性（如果你已经有目标节点的引用）。

### LinkedList 既是 List 也是 Deque

`LinkedList` 实现了 `List` 接口和 `Deque` 接口，这意味着它既可以当作列表使用，也可以当作双端队列使用：

```java
// 作为 List
List<String> list = new LinkedList<>();
list.add("apple");
list.get(0);

// 作为 Deque
Deque<String> deque = new LinkedList<>();
deque.addFirst("first");
deque.addLast("last");
deque.pop();  // 相当于 removeFirst()
```

### ArrayList vs LinkedList：性能对比

| 操作 | ArrayList | LinkedList |
|------|-----------|------------|
| 随机访问 | O(1) | O(n) |
| 头部插入/删除 | O(n) | O(1) |
| 尾部插入/删除 | O(1) amortized | O(1) |
| 中间插入/删除 | O(n) | O(1) 但需先定位 |
| 内存占用 | 紧凑，连续 | 每个节点有额外指针开销 |

实际开发中，`ArrayList` 几乎是默认选择。只有当你明确知道需要频繁在头部操作，或者需要队列/栈结构时，才考虑 `LinkedList`（实际上，`ArrayDeque` 通常比 `LinkedList` 更适合做栈和队列）。

## Vector：时代的眼泪

`Vector` 是 JDK 1.0 就存在的古老类，比 `ArrayList` 早了整整两年。它的设计与 `ArrayList` 非常相似——同样基于数组，同样支持动态扩容——但有一个关键区别：**所有方法都是 synchronized 的**。

```java
public class Vector<E> extends AbstractList<E>
        implements List<E>, RandomAccess, Cloneable, java.io.Serializable {
    
    protected Object[] elementData;
    protected int elementCount;
    protected int capacityIncrement;  // 扩容增量
    
    public synchronized boolean add(E e) {
        modCount++;
        ensureCapacityHelper(elementCount + 1);
        elementData[elementCount++] = e;
        return true;
    }
    
    public synchronized E get(int index) {
        if (index >= elementCount)
            throw new IndexOutOfBoundsException();
        return elementData(index);
    }
    // ...
}
```

因为 `Vector` 的每个操作都需要获取锁，所以在单线程环境下性能不如 `ArrayList`。而在多线程环境下，我们通常也不会选择 `Vector`，而是使用专门的并发集合如 `CopyOnWriteArrayList`。

`Vector` 还有一个 `capacityIncrement` 参数，允许自定义扩容增量：

```java
// 创建一个Vector，初始容量10，每次扩容增加5
Vector<String> v = new Vector<>(10, 5);
```

但这个特性并没有让 `Vector` 翻身——在实际应用中，很少有人会根据具体场景调优这个参数。

### Vector 的遗留价值

虽然 `Vector` 已经不再是推荐的选择，但它在历史上的贡献不可磨灭：

1. **Stack 类**：JDK 提供了一个 `Stack` 类，继承自 `Vector`。虽然 `Stack` 本身也不推荐使用（因为 `Deque` 实现更好），但你可能在遗留代码中遇到它。
2. **并发基础**：`Vector` 的同步机制虽然粗糙，但它证明了在 Java 中实现线程安全集合的可能性，为后来的并发集合框架积累了经验。

## 最佳实践

### 选择合适的初始容量

如果你能估计元素数量，在创建 `ArrayList` 时指定初始容量可以避免不必要的扩容：

```java
// 反例：频繁扩容
List<String> list1 = new ArrayList<>();
for (String s : largeCollection) {
    list1.add(s);
}

// 正例：预估容量
List<String> list2 = new ArrayList<>(largeCollection.size());
for (String s : largeCollection) {
    list2.add(s);
}
```

### 善用 JDK 9+ 的工厂方法

```java
// JDK 9+
List<String> immutable = List.of("a", "b", "c");
List<String> mutable = new ArrayList<>(List.of("a", "b", "c"));
```

### 避免在遍历时修改集合

```java
// 反例：ConcurrentModificationException
for (String s : list) {
    if (s.startsWith("test")) {
        list.remove(s);
    }
}

// 正例：使用 Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().startsWith("test")) {
        it.remove();
    }
}
```

## 写在最后

`ArrayList`、`LinkedList`、`Vector`——三个名字，三种哲学。`ArrayList` 用最朴素的数组创造了最高的随机访问效率；`LinkedList` 用链表的灵活性换取了插入删除的便利；`Vector` 则用同步机制守护着数据的安全。

在实际工作中，99% 的情况下你应该选择 `ArrayList`。它简单、高效、足够用。只有在真正需要队列或栈的场景下，才考虑 `LinkedList` 或 `ArrayDeque`。而 `Vector`，除非你必须维护古老的遗留代码，否则应该让它安息在历史的角落里。
