## 一、ArrayList 的底层数据结构 🔴

### 1.1 问题拆解

先问自己：ArrayList 和数组有什么区别？为什么 ArrayList 能动态增长？

### 1.2 ❌ 错误示范

**候选人原话**：
> "ArrayList 是用数组实现的，它可以自动扩容。"

**问题诊断**：
- 只知道"数组"，不知道 ArrayList 内部用 `Object[] elementData` 存储
- 不理解 `transient` 的含义
- 不清楚 `size` 和 `capacity` 的区别

### 1.3 标准回答

**ArrayList 核心属性**

```java
// [!code highlight] ArrayList 底层核心属性
public class ArrayList<E> extends AbstractList<E>
        implements List<E>, RandomAccess, Cloneable, java.io.Serializable {

    // 存储元素的数组
    transient Object[] elementData;

    // 当前元素个数（不等于 capacity）
    private int size;
}
```

> [!warning]
> 注意 `elementData` 是 `transient` 的，这意味着它会被序列化忽略。ArrayList 自定义了 `writeObject` 和 `readObject` 来处理序列化，为什么要这么做？后面会讲。

**容量 vs 大小**

| 概念 | 含义 | 示例 |
|------|------|------|
| `capacity` | 数组的长度，即能存储多少元素 | `new ArrayList()` 后 capacity = 0 |
| `size` | 实际存储的元素个数 | add 3 个元素后 size = 3 |

```java
ArrayList list = new ArrayList();  // elementData = [], size = 0, capacity = 0
list.add("A");                       // elementData = [A], size = 1, capacity = 10
```

【面试官心理】

面试官问数据结构，他真正想知道的是：
1. 你有没有看过 ArrayList 的源码
2. 你理不理解 `capacity` 和 `size` 的区别
3. 你知不知道 `transient` 序列化的问题

## 二、add 流程详解 🔴

### 2.1 问题拆解

当我们调用 `list.add("hello")` 时，底层发生了什么？

### 2.2 标准回答

**无参 add**

```java
public boolean add(E e) {
    // [!code highlight] 确保容量够用
    ensureCapacityInternal(size + 1);

    // [!code highlight] 赋值后 size++
    elementData[size++] = e;
    return true;
}

private void ensureCapacityInternal(int minCapacity) {
    // 第一次添加时，elementData 是空数组
    if (elementData == EMPTY_ELEMENTDATA) {
        // minCapacity = 1，DEFAULT_CAPACITY = 10
        // 所以第一次 add 后 capacity = 10
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
    }
    ensureExplicitCapacity(minCapacity);
}

private void ensureExplicitCapacity(int minCapacity) {
    if (minCapacity - elementData.length > 0)
        grow(minCapacity);  // 需要扩容
}
```

**带索引的 add**

```java
public void add(int index, E element) {
    rangeCheck(index);  // 检查索引是否越界

    ensureCapacityInternal(size + 1);

    // [!code warning] 核心操作：数组拷贝
    System.arraycopy(elementData, index,
                     elementData, index + 1,
                     size - index);

    elementData[index] = element;
    size++;
}
```

> [!warning]
> `System.arraycopy` 是浅拷贝！如果是对象，会直接复制引用。这也是为什么 ArrayList 不是线程安全的——两个线程同时 add，可能覆盖彼此的值。

### 2.3 追问升级

**面试官追问**：
> "ArrayList 的 add(int index, E element) 和 add(E e) 有什么区别？"

**参考答案**：
- `add(E e)` 直接加到末尾，时间复杂度 O(1)（均摊）
- `add(int index, E element)` 需要把 index 及之后的元素往后移一位，时间复杂度 O(n)

## 三、扩容机制（grow）🔴

### 3.1 问题拆解

这是 ArrayList 最核心的部分，也是面试官最爱的追问点。

### 3.2 标准回答

**扩容核心代码**

```java
private void grow(int minCapacity) {
    int oldCapacity = elementData.length;

    // [!code highlight] 新容量 = 旧容量 + 旧容量/2 = 1.5 倍
    int newCapacity = oldCapacity + (oldCapacity >> 1);

    // 溢出处理：如果 oldCapacity = Integer.MAX_VALUE
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;

    // [!code warning] 容量上限是 Integer.MAX_VALUE - 8
    // 为什么要减 8？JVM 数组对象头需要 8 或 12 字节
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);

    // [!code highlight] 核心操作：数组拷贝
    elementData = Arrays.copyOf(elementData, newCapacity);
}
```

### 3.3 ❌ 错误示范

**候选人原话**：
> "ArrayList 扩容是 1.5 倍，因为这样空间利用率最高。"

**问题诊断**：
- 只背了结论，不知道 1.5 倍是怎么算出来的
- 不理解 `oldCapacity >> 1` 这个位运算
- 不知道为什么要用 `hugeCapacity` 做上限处理

### 3.4 为什么是 1.5 倍？

```java
int newCapacity = oldCapacity + (oldCapacity >> 1);
// oldCapacity = 10
// oldCapacity >> 1 = 5
// newCapacity = 10 + 5 = 15
```

**空间与时间的权衡**：

| 扩容倍数 | 空间利用率 | 扩容频率 | 拷贝代价 |
|----------|------------|----------|----------|
| 1.5 倍 | ~67% | 中等 | 中等 |
| 2 倍 | 50% | 低 | 低 |
| 1.1 倍 | ~91% | 高 | 非常高 |

> [!tip]
> 1.5 倍不是拍脑袋想出来的，而是经验值：既能保证空间利用率（>50%），又能控制扩容频率。JDK 选择了一个折中方案。

【面试官心理】

追问"为什么是 1.5 倍"，面试官想看的是：
1. 你有没有思考过工程上的权衡
2. 你能不能说出"空间换时间"这个基本概念
3. 你有没有意识到扩容是有代价的

## 四、容量与阈值 🟡

### 4.1 默认容量

```java
// 空数组，标记为"懒加载"
private static final Object[] EMPTY_ELEMENTDATA = {};

// 第一次 add 时使用这个容量
private static final int DEFAULT_CAPACITY = 10;
```

**懒加载机制**：

```java
// 第一次 add 时的处理
if (elementData == EMPTY_ELEMENTDATA) {
    minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
}
```

这意味着 `new ArrayList()` 并不会立即分配 10 个元素的数组，只有第一次 add 时才会。

### 4.2 容量上限

```java
private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;

private static int hugeCapacity(int minCapacity) {
    if (minCapacity < 0)  // overflow
        throw new OutOfMemoryError();
    return (minCapacity > MAX_ARRAY_SIZE)
        ? Integer.MAX_VALUE
        : MAX_ARRAY_SIZE;
}
```

> [!warning]
> 为什么要减 8？这是给 JVM 的对象头留空间。数组对象在 JVM 中有额外的元数据（长度等），如果不留余量，可能在创建数组时溢出。

## 五、序列化问题 🟡

### 5.1 问题拆解

ArrayList 的 `elementData` 声明为 `transient`，那它是怎么序列化的？

### 5.2 标准回答

**自定义序列化**

```java
private void writeObject(java.io.ObjectOutputStream s)
        throws java.io.IOException {
    int expectedModCount = modCount;
    s.defaultWriteObject();  // 写入 size

    // [!code highlight] 只写入 size 个元素，不写入整个 capacity
    s.writeInt(size);

    for (int i = 0; i < size; i++) {
        s.writeObject(elementData[i]);
    }

    if (expectedModCount != modCount)
        throw new ConcurrentModificationException();
}
```

**为什么要这么设计？**

```java
ArrayList list = new ArrayList(10000);  // capacity = 10000
list.add("hello");                       // size = 1
```

如果序列化整个 `elementData`，会写入 10000 个数组位置，但实际上只有 1 个元素是有意义的。

> [!tip]
> 这也是"空间换时间"的另一个例子：浪费一点序列化时间，换取更小的序列化文件大小。

【面试官心理】

问到序列化，面试官可能是在试探你对 Java 基础知识的掌握深度。能回答出这个问题的候选人，说明他真的看过源码，并且思考过为什么要这么设计。

## 七、生产避坑 🟡

### 7.1 扩容导致的性能问题

> [!warning]
> 曾经在一次大促中，某服务在零点大量请求超时，排查发现是 ArrayList 扩容导致的 GC 压力。

**问题分析**：

```java
// 错误的写法：循环内不断 add
List<User> users = new ArrayList<>();
for (User user : hugeDataset) {
    users.add(user);  // 每次扩容都可能触发数组拷贝
}
```

**正确写法**：

```java
// 预分配容量
List<User> users = new ArrayList<>(hugeDataset.size());
for (User user : hugeDataset) {
    users.add(user);
}
```

> [!tip]
> 如果预知数据量，务必在初始化时指定容量，避免频繁扩容。

### 7.2 遍历时的扩容问题

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");

for (String s : list) {
    // [!code warning] 迭代过程中不要修改 list
    if ("A".equals(s)) {
        list.remove(s);  // 抛出 ConcurrentModificationException
    }
}
```

## 【面试官手记】

面过一个 P6 候选人，简历上写着"精通 Java 集合"。

我问："ArrayList 的 grow 方法里，新容量是怎么算出来的？"

他说："是 1.5 倍。"

真正的"精通"，不是知道答案，而是能解释"为什么是这个答案"。
