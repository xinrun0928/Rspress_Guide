# Queue 体系全景

Queue（队列）是 Java 集合框架中用于表示"排队等待"这一概念的数据结构。想象餐厅门口排队的顾客、银行柜台前等待办理业务的客户——它们都有一个共同特点：先进先出（FIFO，First In First Out）。Queue 接口正是这种"先来先服务"语义的抽象。

## Queue 接口的设计

Queue 继承自 Collection 接口，它的核心操作分为三类：

```java
public interface Queue<E> extends Collection<E> {
    // 添加元素
    boolean add(E e);           // 队列满时抛出异常
    boolean offer(E e);         // 队列满时返回 false
    
    // 移除元素
    E remove();                  // 队列空时抛出异常
    E poll();                   // 队列空时返回 null
    
    // 检查元素（不移除）
    E element();                // 队列空时抛出异常
    E peek();                  // 队列空时返回 null
}
```

这种设计遵循了两个原则：

1. **操作分为三类**：add/offer、remove/poll、element/peek
2. **成对设计**：每个操作有两个版本，一个抛出异常，一个返回特殊值

这种"优雅的设计"让 API 既能满足需要异常语义的场景，也能满足需要返回值语义的场景。

## Queue 的分类体系

Java 的 Queue 实现可以分为两大类：

**阻塞队列（Blocking Queue）**：在队列为空或满时提供阻塞能力，是实现生产者-消费者模式的核心。

**非阻塞队列（Non-blocking Queue）**：操作立即返回，不阻塞线程。

```
Queue
├── Deque
│   ├── ArrayDeque
│   ├── LinkedList
│   └── LinkedBlockingDeque
├── AbstractQueue
│   ├── PriorityQueue
│   └── ConcurrentLinkedQueue
└── BlockingQueue
    ├── ArrayBlockingQueue
    ├── LinkedBlockingQueue
    ├── PriorityBlockingQueue
    └── DelayQueue
```

## Deque：双端队列

Deque（Double Ended Queue，双端队列）扩展了 Queue 接口，允许从两端插入和删除元素：

```java
public interface Deque<E> extends Queue<E> {
    // 队列头操作
    void addFirst(E e);
    void addLast(E e);
    boolean offerFirst(E e);
    boolean offerLast(E e);
    
    // 队列尾操作
    E removeFirst();
    E removeLast();
    E pollFirst();
    E pollLast();
    
    // 查看操作
    E getFirst();
    E getLast();
    E peekFirst();
    E peekLast();
}
```

Deque 兼容两种数据结构：

- **既可以作为 Queue 使用**（先进先出）
- **也可以作为 Stack 使用**（先进后出）

```java
Deque<String> deque = new ArrayDeque<>();

// 作为 Queue 使用
deque.add("A");      // 入队尾
deque.add("B");
String first = deque.poll();  // A

// 作为 Stack 使用
deque.push("X");    // 入栈顶
deque.push("Y");
String top = deque.pop();     // Y
```

## 常用实现对比

| 实现 | 底层结构 | 线程安全 | 有界性 | 特点 |
|------|----------|----------|--------|------|
| ArrayDeque | 循环数组 | 否 | 可选有界 | 高性能，比 LinkedList 快 |
| LinkedList | 双向链表 | 否 | 无界 | 支持 null，内存开销大 |
| PriorityQueue | 堆 | 否 | 无界 | 按优先级排序 |
| ConcurrentLinkedQueue | 链表 | 是 | 无界 | 无锁高性能 |
| ArrayBlockingQueue | 数组 | 是 | 有界 | 阻塞操作 |
| LinkedBlockingQueue | 链表 | 是 | 可选有界 | 阻塞操作 |

## 典型应用场景

**任务队列**：线程池内部使用 BlockingQueue 存储待执行任务。

**广度优先搜索**：使用 Queue 存储待访问的节点。

**消息队列**：生产者将消息放入 Queue，消费者取出处理。

**滑动窗口**：使用 Deque 实现滑动窗口最大值等问题。

## Java 8+ 的 Stream 集成

Queue 与 Java 8 的 Stream API 完美集成：

```java
Queue<Integer> queue = new PriorityQueue<>();
queue.addAll(Arrays.asList(3, 1, 4, 1, 5, 9, 2, 6));

queue.stream()
     .sorted()
     .forEach(System.out::println);
```

## 选择合适的 Queue

选择 Queue 实现需要考虑：

1. **是否需要阻塞**：生产者-消费者场景用 BlockingQueue
2. **是否需要排序**：按优先级处理用 PriorityQueue
3. **是否需要高性能**：单线程用 ArrayDeque
4. **是否需要线程安全**：多线程用 ConcurrentLinkedQueue

理解 Queue 体系的架构和选择原则，你就能在合适的场景选择最合适的数据结构。
