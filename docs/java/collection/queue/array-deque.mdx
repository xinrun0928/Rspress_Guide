# ArrayDeque：比 LinkedList 更快的双端队列

ArrayDeque 是 Java 7 引入的 Deque 实现，它使用可动态扩展的循环数组来实现双端队列。与 LinkedList 相比，ArrayDeque 在大多数场景下具有更好的性能。这让它成为实现栈、队列或双端队列的首选数据结构。

## 为什么需要 ArrayDeque

在 ArrayDeque 出现之前，如果需要一个高性能的队列，通常选择 LinkedList。但 LinkedList 有几个固有的问题：

1. **每个节点需要额外的内存开销**：每个节点需要存储 prev 和 next 两个指针
2. **CPU 缓存不友好**：链表节点的内存分散，遍历时缓存命中率低
3. **频繁的内存分配**：每次插入都可能创建新节点

ArrayDeque 使用数组存储元素，避免了这些问题。

## ArrayDeque 的核心结构

```java
public class ArrayDeque<E> extends AbstractCollection<E> 
    implements Deque<E>, Cloneable, Serializable {
    
    transient Object[] elements;
    transient int head;
    transient int tail;
    
    private static final int MIN_INITIAL_CAPACITY = 8;
}
```

ArrayDeque 使用循环数组（circular buffer）实现：

- `elements`：存储元素的数组
- `head`：指向队头元素
- `tail`：指向下一个可以插入的位置

循环数组的关键特性是：当 head 或 tail 到达数组末尾时，会"绕回"到数组开头。这使得入队和出队操作都是 O(1) 时间复杂度。

## 循环数组的工作原理

```
初始状态:
[ null, null, null, null, null, null, null, null ]
  head                              tail (0)

添加 A, B, C:
[ A, B, C, null, null, null, null, null ]
            head               tail (3)

取出 A:
[ null, B, C, null, null, null, null, null ]
              head           tail (3)
    head 向前移动一位

添加 D, E, F, G:
[ null, B, C, D, E, F, G, null ]
              head       tail (7)

添加 H（触发扩容）:
[ G, null, null, null, null, null, H, A, B, C, D, E, F ]
                                                    tail (6)
```

注意添加 H 后，原数组扩容并重新排列，元素顺序被反转。这种扩容行为是 ArrayDeque 的一个特点。

## 入队操作

```java
public void addLast(E e) {
    if (e == null)
        throw new NullPointerException();
    elements[tail] = e;
    if ((tail = (tail + 1) & (elements.length - 1)) == head)
        doubleCapacity();
}

private void doubleCapacity() {
    int p = head;
    int n = elements.length;
    int r = n - p;
    int newCapacity = n << 1;
    if (newCapacity < 0)
        throw new IllegalStateException("Deque too big");
    Object[] a = new Object[newCapacity];
    System.arraycopy(elements, p, a, 0, r);
    System.arraycopy(elements, 0, a, r, p);
    elements = a;
    head = 0;
    tail = n;
}
```

`addLast` 将元素放到 tail 位置，然后将 tail 向后移动一位。使用位运算 `(tail + 1) & (elements.length - 1)` 而不是取模 `%`，因为数组长度始终是 2 的幂次方。

当 tail 与 head 相遇时（数组满了），调用 `doubleCapacity()` 扩容为原来的两倍。

## 出队操作

```java
public E pollFirst() {
    h = head;
    result = (E) elements[h];
    if (result == null)
        return null;
    elements[h] = null;
    head = (h + 1) & (elements.length - 1);
    return result;
}

public E pollLast() {
    t = (tail - 1) & (elements.length - 1);
    result = (E) elements[t];
    if (result == null)
        return null;
    elements[t] = null;
    tail = t;
    return result;
}
```

出队操作同样使用位运算计算新位置。由于循环数组的特性，head 和 tail 可以"绕回"数组开头。

## ArrayDeque vs LinkedList

| 特性 | ArrayDeque | LinkedList |
|------|-----------|-----------|
| 底层结构 | 数组 | 双向链表 |
| 内存开销 | 固定开销 | 每节点 2 个引用 |
| 缓存友好性 | 好 | 差 |
| 插入时间复杂度 | O(1) 平均 | O(1) |
| 随机访问 | O(1) | O(n) |
| 内存分配 | 偶尔扩容 | 每次插入 |
| null 支持 | 不支持 | 支持 |
| 迭代器 | 弱一致 | 弱一致 |

**ArrayDeque 的优势**：

- **更快的速度**：数组的缓存命中率远高于链表节点
- **更低的内存开销**：不需要存储前后指针
- **更少的内存分配**：扩容时批量分配数组

**LinkedList 的优势**：

- **支持 null**：ArrayDeque 不允许 null 元素
- **迭代时删除安全**：迭代器删除操作更安全
- **容量无限制**：ArrayDeque 需要扩容

## 何时使用 ArrayDeque

**适合使用 ArrayDeque**：

```java
// 作为高性能队列
ArrayDeque<Event> eventQueue = new ArrayDeque<>();
eventQueue.offer(event);
Event e = eventQueue.poll();

// 作为栈
ArrayDeque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.push(2);
int top = stack.pop(); // 2

// 作为双端队列
ArrayDeque<String> deque = new ArrayDeque<>();
deque.addFirst("start");
deque.addLast("end");
String first = deque.removeFirst();
String last = deque.removeLast();
```

**不适合使用 ArrayDeque**：

```java
// 需要存储 null 元素
// LinkedList 是唯一选择

// 需要在迭代过程中安全删除
for (String s : linkedList) {
    if (s.equals("remove")) {
        linkedList.remove(s); // 安全
    }
}

// 需要随机访问（虽然可以，但不推荐）
// 使用 ArrayList 更合适
```

## 扩容与内存

ArrayDeque 的初始容量总是 2 的幂次方，最小是 8。每次扩容容量翻倍。

```java
public ArrayDeque() {
    elements = new Object[16]; // 默认容量 16
}

private void grow(int neededSpace) {
    int newCapacity = elements.length << 1;
    // ...
}
```

当元素数量接近容量时，ArrayDeque 会自动扩容。由于是 O(1) 的摊销成本，整体性能仍然很好。

## 线程安全性

ArrayDeque 不是线程安全的。如果需要在多线程环境下使用，可以：

1. 使用 `Collections.asLifoQueue()` 包装（但仍是线程不安全的底层）
2. 使用 `ConcurrentLinkedDeque`（无锁实现）
3. 使用外部同步

对于单线程场景或不需要线程安全的场景，ArrayDeque 是最佳选择。

## 最佳实践

1. **作为栈使用**：ArrayDeque 比 Stack 更快（Stack 继承自 Vector，是同步的）
2. **作为队列使用**：ArrayDeque 比 LinkedList 更快、更省内存
3. **预估容量**：如果能预估元素数量，指定初始容量可以避免扩容

```java
// 预估容量，避免扩容
ArrayDeque<Event> events = new ArrayDeque<>(expectedEventCount);
```

理解 ArrayDeque 的循环数组实现和与 LinkedList 的对比，你就能在合适场景选择最高效的数据结构。
