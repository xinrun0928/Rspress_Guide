# Queue 接口与实现类

Queue（队列）是 Java 集合框架中最具"队列语义"的数据结构。它模拟了现实世界中的排队场景——先来的人先接受服务，后来的人排在队尾。理解 Queue 接口及其实现类是掌握 Java 集合框架的重要一环。

## Queue 接口的契约

Queue 接口定义了队列的核心操作：

```java
public interface Queue<E> extends Collection<E> {
    // 添加元素
    boolean add(E e);        // 成功返回 true，队列满则抛出 IllegalStateException
    boolean offer(E e);      // 成功返回 true，队列满则返回 false
    
    // 移除元素
    E remove();              // 返回并移除队头，队列空则抛出 NoSuchElementException
    E poll();                // 返回并移除队头，队列空则返回 null
    
    // 检查元素
    E element();             // 返回队头但不移除，队列空则抛出异常
    E peek();                // 返回队头但不移除，队列空则返回 null
}
```

这六种方法形成了两两对应的三对：

| 操作 | 抛出异常 | 返回特殊值 |
|------|----------|-----------|
| 入队 | add() | offer() |
| 出队 | remove() | poll() |
| 检查 | element() | peek() |

这种设计允许调用者根据场景选择合适的语义——是需要明确的异常反馈，还是需要优雅的降级处理。

## Queue 的语义约定

Queue 对元素的顺序（出队顺序）有一个核心约定：**FIFO（先进先出）**。元素按照插入的先后顺序出队，最早插入的元素最先被移除。

但这个约定不是强制的。PriorityQueue 就不遵循 FIFO 顺序，而是按照元素的优先级出队。因此，Queue 接口的规范是"不保证排序"，调用者在选择实现时需要注意。

## ArrayDeque：高性能双端队列

ArrayDeque 是 Java 7 引入的Deque 实现，它使用循环数组实现，性能通常优于 LinkedList：

```java
Queue<String> queue = new ArrayDeque<>();
Deque<String> deque = new ArrayDeque<>();

queue.offer("A");
queue.offer("B");
queue.offer("C");
System.out.println(queue.poll()); // A
```

ArrayDeque 的优势：

- **无容量限制**：自动扩容
- **性能优秀**：使用循环数组，插入和删除都是 O(1)
- **比 LinkedList 快**：数组的缓存友好性优于链表节点
- **不支持 null**：与 LinkedList 不同，ArrayDeque 不允许 null 元素

ArrayDeque 不支持 null 是因为它使用 null 来标记空槽，避免与有效元素混淆。

## LinkedList：万金油实现

LinkedList 同时实现了 List 和 Deque 接口：

```java
List<String> list = new LinkedList<>();
Queue<String> queue = new LinkedList<>();
Deque<String> deque = new LinkedList<>();
```

LinkedList 作为 Queue 使用时的特点：

- **支持 null 元素**：与 ArrayDeque 不同
- **内存开销大**：每个节点都需要额外的指针存储
- **插入删除 O(1)**：但随机访问是 O(n)
- **线程不安全**：需要外部同步

如果你需要 List 的随机访问能力，用 ArrayList；如果你只需要 Queue 的 FIFO 语义，用 ArrayDeque。

## PriorityQueue：优先级队列

PriorityQueue 实现了按优先级排序的队列：

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.add(30);
pq.add(10);
pq.add(50);
pq.add(20);

System.out.println(pq.poll()); // 10
System.out.println(pq.poll()); // 20
System.out.println(pq.poll()); // 30
```

PriorityQueue 默认使用自然顺序（Comparable），也可以传入自定义 Comparator：

```java
PriorityQueue<String> pq = new PriorityQueue<>(Comparator.reverseOrder());
```

PriorityQueue 的底层结构是二叉堆（Binary Heap），插入和删除操作的时间复杂度是 O(log n)。

## 线程安全的 Queue

Queue 的线程安全实现分为两类：

**非阻塞队列**：操作不阻塞，线程安全通过无锁算法实现。

```java
Queue<Integer> concurrentQueue = new ConcurrentLinkedQueue<>();
```

**阻塞队列**：在队列空或满时阻塞操作，是实现生产者-消费者模式的基础。

```java
BlockingQueue<Integer> blockingQueue = new LinkedBlockingQueue<>(100);
blockingQueue.put(42); // 队列满时会阻塞
Integer val = blockingQueue.take(); // 队列空时会阻塞
```

## Queue 的使用模式

**模式一：任务队列**

```java
Queue<Runnable> tasks = new ArrayDeque<>();
tasks.offer(() -> System.out.println("Task 1"));
tasks.offer(() -> System.out.println("Task 2"));

while (!tasks.isEmpty()) {
    Runnable task = tasks.poll();
    task.run();
}
```

**模式二：生产者-消费者**

```java
BlockingQueue<Task> queue = new LinkedBlockingQueue<>();

// 生产者
new Thread(() -> {
    while (hasMoreTasks()) {
        queue.put(produceTask());
    }
}).start();

// 消费者
new Thread(() -> {
    while (true) {
        Task task = queue.take();
        process(task);
    }
}).start();
```

**模式三：滑动窗口最大值**

使用 Deque 实现滑动窗口相关算法：

```java
public int[] maxSlidingWindow(int[] nums, int k) {
    Deque<Integer> deque = new ArrayDeque<>();
    int[] result = new int[nums.length - k + 1];
    
    for (int i = 0; i < nums.length; i++) {
        while (!deque.isEmpty() && deque.peekFirst() <= i - k)
            deque.pollFirst();
        while (!deque.isEmpty() && nums[deque.peekLast()] < nums[i])
            deque.pollLast();
        deque.offerLast(i);
        if (i >= k - 1)
            result[i - k + 1] = nums[deque.peekFirst()];
    }
    return result;
}
```
