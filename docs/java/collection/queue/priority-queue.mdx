# PriorityQueue 堆结构与排序

PriorityQueue 是 Java 中实现优先级队列的类。与普通队列的 FIFO（先进先出）不同，PriorityQueue 按照元素的优先级顺序出队——优先级最高的元素最先被移除。这使得 PriorityQueue 成为实现调度器、任务队列、贪心算法等场景的理想选择。

## 什么是堆

PriorityQueue 的底层数据结构是堆（Heap）。堆是一种特殊的完全二叉树，它满足堆属性：

- **最大堆（Max Heap）**：每个节点的值都大于或等于其子节点的值。根节点是最大值。
- **最小堆（Min Heap）**：每个节点的值都小于或等于其子节点的值。根节点是最小值。

PriorityQueue 默认实现是最小堆。

堆可以用数组高效存储，因为完全二叉树的节点可以通过索引计算父子关系。对于索引为 i 的节点：

- 左子节点索引：2 * i + 1
- 右子节点索引：2 * i + 2
- 父节点索引：(i - 1) / 2

```
数组索引:   0   1   2   3   4   5   6   7
            ├───┬───┬───┬───┬───┬───┬───┬───┤
最小堆:     │ 1 │ 3 │ 5 │ 7 │ 9 │ 4 │ 6 │10 │
            │   │   │   │   │   │   │   │   │
            └───┴───┴───┴───┴───┴───┴───┴───┘
                      1
                    /   \
                   3     5
                  / \   / \
                 7   9 4   6
                /
               10
```

## PriorityQueue 的核心结构

```java
public class PriorityQueue<E> extends AbstractQueue<E> 
    implements Serializable {
    
    private static final int DEFAULT_INITIAL_CAPACITY = 11;
    
    private final Comparator<? super E> comparator;
    transient Object[] queue;
    private int size;
    private final Comparator<? super E> comparator;
    
    private void siftUp(int k, E x) {
        if (comparator != null)
            siftUpUsingComparator(k, x);
        else
            siftUpComparable(k, x);
    }
    
    private void siftDown(int k, E x) {
        if (comparator != null)
            siftDownUsingComparator(k, x);
        else
            siftDownComparable(k, x);
    }
}
```

核心字段说明：

- `queue`：存储元素的数组，索引 0 是堆的根节点
- `size`：元素数量
- `comparator`：比较器，为 null 则使用自然顺序

## 上浮操作：siftUp

当向堆中插入新元素时，需要执行上浮操作来恢复堆属性：

```java
private void siftUpUsingComparator(int k, E x) {
    while (k > 0) {
        int parent = (k - 1) >>> 1;
        Object e = queue[parent];
        if (comparator.compare(x, (E) e) >= 0)
            break;
        queue[k] = e;
        k = parent;
    }
    queue[k] = x;
}
```

以最小堆为例，siftUp 的过程：

1. 将新元素 x 放到数组末尾（索引 k）
2. 比较 x 与其父节点：如果 x 更小，交换位置
3. 继续向上比较，直到 x 不小于父节点或到达根节点

时间复杂度：O(log n)，最多比较到根节点的路径长度。

```
插入 2:
     1                    1
    / \                 / \
   3   5      -->      2   5
  / \                / \
 7   9              3   9
                    /
                   7
```

## 下沉操作：siftDown

当从堆中移除根节点或修改堆中某个元素时，需要执行下沉操作：

```java
private void siftDownUsingComparator(int k, E x) {
    int half = size >>> 1;
    while (k < half) {
        int child = (k * 2) + 1;
        Object c = queue[child];
        int right = child + 1;
        if (right < size && comparator.compare((E) c, (E) queue[right]) > 0)
            child = right;
        if (comparator.compare(x, (E) c) <= 0)
            break;
        queue[k] = c;
        k = child;
    }
    queue[k] = x;
}
```

siftDown 的过程：

1. 从当前位置 k 开始，检查子节点
2. 选择较小的子节点（最小堆）或较大的子节点（最大堆）
3. 如果子节点更小/更大，交换位置
4. 继续向下，直到 x 应该在的位置或到达叶子节点

时间复杂度：O(log n)。

```
移除根节点 1:
     1                    3
    / \                 / \
   3   5      -->      7   5
  / \                /
 7   9              9
```

## 添加元素：add/offer

```java
public boolean offer(E e) {
    if (e == null)
        throw new NullPointerException();
    int i = size;
    if (i >= queue.length)
        grow(i + 1);
    size = i + 1;
    if (i == 0)
        queue[0] = e;
    else
        siftUp(i, e);
    return true;
}
```

offer 方法的核心逻辑：

1. 如果数组已满，扩容（通常是 2 倍）
2. 将新元素放到数组末尾（索引 i）
3. 如果不是第一个元素，调用 siftUp 将元素上浮到正确位置

## 移除元素：poll

```java
public E poll() {
    if (size == 0)
        return null;
    int s = --size;
    E result = (E) queue[0];
    E x = (E) queue[s];
    queue[s] = null;
    if (s != 0)
        siftDown(0, x);
    return result;
}
```

poll 方法的核心逻辑：

1. 保存根节点（最小/最大值）作为返回值
2. 将最后一个元素移到根位置
3. 调用 siftDown 将该元素下沉到正确位置
4. 返回保存的根节点

## Heapify：批量建堆

PriorityQueue 提供了一个构造函数，可以接受现有集合并批量建堆：

```java
public PriorityQueue(Collection<? extends E> c) {
    if (c instanceof SortedSet) {
        SortedSet<? extends E> ss = (SortedSet<? extends E>) c;
        comparator = ss.comparator();
        heapify();
    } else if (c instanceof PriorityQueue) {
        PriorityQueue<? extends E> pq = (PriorityQueue<? extends E>) c;
        comparator = pq.comparator();
        heapify();
    } else {
        comparator = null;
        heapify();
    }
}

private void heapify() {
    Object[] es = queue;
    int n = size;
    int i = (n >>> 1) - 1;
    for (; i >= 0; i--)
        siftDown(i, (E) es[i]);
}
```

heapify 的时间复杂度是 O(n)，而不是 O(n log n)。这是因为从最后一个非叶子节点开始下沉，利用了叶子节点不需要下沉的特点。

## 典型应用场景

**Top-K 问题**：找出数组中的最大/最小的 K 个元素

```java
public int[] findKSmallest(int[] arr, int k) {
    PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
    
    for (int num : arr) {
        if (maxHeap.size() < k) {
            maxHeap.offer(num);
        } else if (num < maxHeap.peek()) {
            maxHeap.poll();
            maxHeap.offer(num);
        }
    }
    
    return maxHeap.stream().mapToInt(Integer::intValue).toArray();
}
```

**合并有序文件**：将 K 个有序数组合并成一个有序数组

```java
public int[] mergeKSortedArrays(List<int[]> arrays) {
    PriorityQueue<int[]> minHeap = new PriorityQueue<>(
        Comparator.comparingInt(a -> a[0])
    );
    
    int totalSize = 0;
    for (int i = 0; i < arrays.size(); i++) {
        int[] arr = arrays.get(i);
        if (arr.length > 0) {
            minHeap.offer(new int[]{arr[0], i, 0});
            totalSize += arr.length;
        }
    }
    
    int[] result = new int[totalSize];
    int index = 0;
    while (!minHeap.isEmpty()) {
        int[] top = minHeap.poll();
        result[index++] = top[0];
        int arrIndex = top[1];
        int elemIndex = top[2];
        if (elemIndex + 1 < arrays.get(arrIndex).length) {
            int next = arrays.get(arrIndex)[elemIndex + 1];
            minHeap.offer(new int[]{next, arrIndex, elemIndex + 1});
        }
    }
    return result;
}
```

**定时任务调度**：使用 PriorityQueue 管理定时任务

```java
public class TaskScheduler {
    private final PriorityQueue<Task> taskQueue;
    
    public TaskScheduler() {
        taskQueue = new PriorityQueue<>();
    }
    
    public void schedule(Task task) {
        taskQueue.offer(task);
    }
    
    public Task nextTask() {
        return taskQueue.poll();
    }
}
```

## 与 TreeSet 的对比

PriorityQueue 和 TreeSet 都能实现排序集合，但有重要区别：

| 特性 | PriorityQueue | TreeSet |
|------|--------------|---------|
| 底层结构 | 堆 | 红黑树 |
| 是否允许重复 | 允许（相同值都保留） | 不允许 |
| 删除任意元素 | O(n)（需要遍历） | O(log n) |
| 查看最值 | O(1) | O(log n) |
| 迭代器 | 弱顺序保证 | 有序迭代 |

如果需要有序遍历和删除任意元素，用 TreeSet；如果只需要获取最大/最小值，用 PriorityQueue。

理解 PriorityQueue 的堆结构和 siftUp/siftDown 操作，你就能理解这个重要数据结构的核心原理。
