# Java 集合框架：从入门到精通

想象一下，你正在经营一家图书馆。没有书架、书柜、分类标签的图书馆会是什么样子？书籍堆叠在一起，想要找一本特定的書，可能需要翻遍整个房间。Java 的集合框架就是程序世界中的"图书馆管理系统"——它提供了一套精心设计的容器，让数据的存储、检索和管理变得井然有序。

集合框架是 Java 最核心的 API 之一，也是面试中最常考察的知识点。无论是初级的 `ArrayList` 扩容机制，还是高级的 `ConcurrentHashMap` 分段锁原理，都可能成为面试官检验你技术深度的试金石。本系列文章将带你深入探索 Java 集合框架的每一个角落，从底层原理到实战应用，从单线程到并发场景，系统性地构建你的集合框架知识体系。

## 为什么需要集合框架？

在早期编程中，如果需要存储多个数据，我们可能会这样做：

```java
// 原始方式：使用数组
String[] names = new String[100];
names[0] = "Alice";
names[1] = "Bob";
// 问题：容量固定，扩容麻烦
```

数组的问题显而易见：容量固定、插入删除不便、缺乏现成的方法支持。Java 1.2 引入了集合框架，优雅地解决了这些问题。集合框架不仅仅是一堆类的集合，它是一套设计精良的架构，体现在以下几个核心原则之上：

**接口与实现分离**：集合框架定义了清晰的接口层次，如 `List`、`Set`、`Map`、`Queue`，每个接口可以有多种实现。你可以根据具体场景选择最合适的实现——是需要快速查找？有序遍历？还是线程安全？不同的需求指向不同的选择。

**统一的操作语义**：无论你使用的是 `ArrayList` 还是 `LinkedList`，`add()`、`remove()`、`get()` 这些操作的行为是一致的。这种一致性大大降低了学习成本，也使得代码更容易理解和维护。

**算法的可复用性**：`Collections` 和 `Arrays` 类提供了丰富的静态方法，包括排序、二分查找、洗牌等。这些算法实现经过了精心优化，可以直接应用于任何集合。

## 集合框架全景图

Java 集合框架可以划分为两大分支：`Collection` 和 `Map`。这两者的区别看似简单——`Collection` 存储单个元素，`Map` 存储键值对——但背后的设计却蕴含着深刻的考量。

```
Iterable
├── Collection
│   ├── List（有序、可重复）
│   │   ├── ArrayList
│   │   ├── LinkedList
│   │   └── Vector
│   ├── Set（无序、不可重复）
│   │   ├── HashSet
│   │   ├── LinkedHashSet
│   │   └── TreeSet
│   └── Queue（队列）
│       ├── LinkedList
│       ├── PriorityQueue
│       └── Deque
│           └── ArrayDeque
└── Map
    ├── HashMap
    ├── LinkedHashMap
    ├── TreeMap
    ├── Hashtable
    └── ConcurrentHashMap
```

`List` 接口的代表实现是 `ArrayList`。这个名字本身就透露了它的本质——基于数组的实现。但"数组"前面加上"动态"二字，才是 `ArrayList` 的真正威力所在。它在保证数组随机访问 O(1) 性能的同时，通过巧妙的扩容机制解决了容量固定的问题。`LinkedList` 则采用了链表结构，在插入和删除操作上性能更优，但随机访问需要 O(n) 的时间复杂度。`Vector` 作为早期实现，现已式微，但在理解线程安全演变历史时仍有参考价值。

`Set` 接口的实现都围绕一个核心问题：如何保证元素唯一性？`HashSet` 通过哈希表实现，依赖 `hashCode()` 和 `equals()` 方法来判断元素是否相等。`LinkedHashSet` 在 `HashSet` 基础上增加了双向链表，维护插入顺序或访问顺序。`TreeSet` 则使用红黑树结构，实现了元素的自然排序或自定义排序。

`Map` 接口存储键值对，是日常开发中使用频率最高的集合类型之一。`HashMap` 是当之无愧的主角，其底层结构经历了从 JDK 7 的数组加链表到 JDK 8+ 的数组加链表加红黑树的演进。`LinkedHashMap` 可以实现 LRU 缓存，`TreeMap` 提供了基于键的排序能力，`Hashtable` 虽然古老但已不再推荐使用。

`Queue` 和 `Deque` 接口则将我们带入数据结构的经典领域。队列遵循 FIFO 原则，双端队列支持从两端操作。`PriorityQueue` 基于堆结构，实现了优先级队列；`ArrayDeque` 以数组实现双端队列，性能通常优于 `LinkedList`。

## 并发集合的特殊地位

在多线程环境下，普通的集合类可能引发 `ConcurrentModificationException`，更严重的是数据竞争导致的脏读和数据丢失。为此，Java 提供了专门的并发集合框架：

`ConcurrentHashMap` 是其中的明星产品。JDK 7 采用分段锁设计，将数据分成多个 Segment，每个 Segment 独立加锁。JDK 8+ 则采用 CAS + synchronized 的方式，进一步提升了并发性能。`CopyOnWriteArrayList` 采用了"写时复制"的策略，适合读多写少的场景。`BlockingQueue` 接口及其实现构成了生产者-消费者模式的基石，`LinkedBlockingQueue`、`ArrayBlockingQueue`、`SynchronousQueue` 各有适用场景。
