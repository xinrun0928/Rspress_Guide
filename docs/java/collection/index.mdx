# 集合框架

你知道 ArrayList 的 `add()` 方法，在 JDK 源码里藏着一个 1.5 倍扩容的秘密吗？

当你往 ArrayList 里 add 100 万个元素，它会经历多少次扩容？每次扩容要复制多少数据？为什么 HashMap 的负载因子是 0.75 而不是 0.8 或 0.5？

集合框架是 Java 开发中使用最频繁的 API，也是面试的必考战场。从最简单的 `ArrayList` 到高性能的 `ConcurrentHashMap`，从无序的 `HashSet` 到有序的 `TreeSet`——每一个集合背后都有精心设计的数据结构与算法。

```
第一阶段：Collection 体系 → List → ArrayList 扩容（建立基础）
         ↓
第二阶段：HashMap 底层结构 → HashMap 专题（JDK 7 vs JDK 8）★ 重点
         ↓
第三阶段：Set 与散列表 → Map 其他实现 → 并发容器 → Queue
```

## 面试核心公式

```
HashMap 查找 = hash(key) → index = (n-1) & hash → 遍历链表/红黑树
         ↓
扩容触发 = size > threshold (capacity × loadFactor)
         ↓
JDK 7 死循环 = 头插法 + 多线程 + 扩容 → 环形链表 → get() 死循环
         ↓
JDK 8 优化 = 红黑树（>8）+ 尾插法（避免死循环）+ 不重新计算 hash
```
