# JDK 8：数组 + 链表/红黑树 + 尾插法

JDK 8 是 HashMap 发展史上最重要的里程碑。这一版本对 HashMap 进行了脱胎换骨的改造，引入了红黑树、尾插法，并优化了扰动函数。这些改变使得 HashMap 在保证高性能的同时，彻底解决了 JDK 7 的并发死循环问题。

## JDK 8 核心数据结构

JDK 8 的 HashMap 使用数组加链表或红黑树的组合结构：

```java
public class HashMap<K,V> extends AbstractMap<K,V> 
    implements Map<K,V>, Cloneable, Serializable {
    
    transient Node<K,V>[] table;
    transient int size;
    transient int modCount;
    int threshold;
    final float loadFactor;
    
    static class Node<K,V> implements Map.Entry<K,V> {
        final int hash;
        final K key;
        V value;
        Node<K,V> next;
        
        Node(int hash, K key, V value, Node<K,V> next) {
            this.hash = hash;
            this.key = key;
            this.value = value;
            this.next = next;
        }
    }
    
    static final class TreeNode<K,V> extends LinkedHashMap.Entry<K,V> {
        TreeNode<K,V> parent;
        TreeNode<K,V> left;
        TreeNode<K,V> right;
        TreeNode<K,V> prev;
        boolean red;
        
        TreeNode(int hash, K key, V val, Node<K,V> next, TreeNode<K,V> parent) {
            super(hash, key, val, next);
            this.parent = parent;
        }
    }
}
```

注意两个关键变化：

1. **Node 类替代了 Entry 类**：类名从 Entry 改为 Node，但核心结构变化不大
2. **引入了 TreeNode 类**：这是一个继承自 `LinkedHashMap.Entry` 的红黑树节点类

TreeNode 的设计很有意思，它继承自 `LinkedHashMap.Entry`，而 `LinkedHashMap.Entry` 又继承自 `HashMap.Node`。这意味着 TreeNode 既有红黑树所需的 `parent`、`left`、`right`、`prev` 和 `red` 字段，也有链表所需的 `next` 字段。这种双重继承的设计使得树节点仍然可以组成链表（在扩容时用于遍历）。

## 尾插法：安全的插入方式

JDK 8 最大的改变之一是将头插法改为尾插法：

```java
Node<K,V> newNode(int hash, K key, V value, Node<K,V> next) {
    return new Node<>(hash, key, value, next);
}

TreeNode<K,V> newTreeNode(int hash, K key, V value, Node<K,V> next) {
    return new TreeNode<>(hash, key, value, next, null);
}

void afterNodeAccess(Node<K,V> p) { }
void afterNodeInsertion(boolean evict) { }

final V putVal(int hash, K key, V value, boolean onlyIfAbsent, boolean evict) {
    Node<K,V>[] tab; Node<K,V> p; int n, i;
    if ((tab = table) == null || (n = tab.length) == 0)
        n = (tab = resize()).length;
    if ((p = tab[i = (n - 1) & hash]) == null)
        tab[i] = newNode(hash, key, value, null);
    else {
        Node<K,V> e; K k;
        if (p.hash == hash &&
            ((k = p.key) == key || (key != null && key.equals(k))))
            e = p;
        else if (p instanceof TreeNode)
            e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
        else {
            for (int binCount = 0; ; ++binCount) {
                if ((e = p.next) == null) {
                    p.next = newNode(hash, key, value, null);
                    if (binCount >= TREEIFY_THRESHOLD - 1)
                        treeifyBin(tab, hash);
                    break;
                }
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    break;
                p = e;
            }
        }
        if (e != null) {
            V oldValue = e.value;
            if (!onlyIfAbsent || oldValue == null)
                e.value = value;
            afterNodeAccess(e);
            return oldValue;
        }
    }
    ++modCount;
    if (++size > threshold)
        resize();
    afterNodeInsertion(evict);
    return null;
}
```

关键变化在于 `p.next = newNode(hash, key, value, null)` 这行代码。新节点被追加到链表的末尾，而非插入到头部。

## 红黑树转换：树化与拆分

当链表长度超过阈值（默认为 8）时，链表会转换成红黑树：

```java
static final int TREEIFY_THRESHOLD = 8;

final void treeifyBin(Node<K,V>[] tab, int hash) {
    int n, index; Node<K,V> e;
    if (tab == null || (n = tab.length) < MIN_TREEIFY_CAPACITY)
        resize();
    else if ((e = tab[index = (n - 1) & hash]) != null) {
        TreeNode<K,V> hd = null, tl = null;
        do {
            TreeNode<K,V> p = replacementTreeNode(e, null);
            if (tl == null)
                hd = p;
            else {
                p.prev = tl;
                tl.next = p;
            }
            tl = p;
        } while ((e = e.next) != null);
        if ((tab[index] = hd) != null)
            hd.treeify(tab);
    }
}
```

`treeifyBin` 方法做了两件事：

1. **检查容量**：如果数组容量小于 64（`MIN_TREEIFY_CAPACITY`），优先扩容而非树化
2. **转换链表**：将 Node 链表转换成 TreeNode 链表，然后调用 `treeify()` 构建红黑树

反过来，当红黑树中的元素被删除导致节点数过少时（默认阈值 6），红黑树会"拆分"回链表：

```java
static final int UNTREEIFY_THRESHOLD = 6;
```

## 扩容时的树处理

JDK 8 扩容时对红黑树有特殊处理：

```java
final Node<K,V>[] resize() {
    Node<K,V>[] oldTab = table;
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    int oldThr = threshold;
    int newCap, newThr = 0;
    
    // ... 计算新的容量和阈值 ...
    
    @SuppressWarnings({"rawtypes","unchecked"})
    Node<K,V>[] newTab = (Node<K,V>[]) new Node[newCap];
    table = newTab;
    
    if (oldTab != null) {
        for (int j = 0; j < oldCap; ++j) {
            Node<K,V> e;
            if ((e = oldTab[j]) != null) {
                oldTab[j] = null;
                if (e.next == null)
                    newTab[e.hash & (newCap - 1)] = e;
                else if (e instanceof TreeNode)
                    ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                else {
                    // 链表保持顺序的低位部分
                    Node<K,V> loHead = null, loTail = null;
                    // 链表保持顺序的高位部分
                    Node<K,V> hiHead = null, hiTail = null;
                    Node<K,V> next;
                    do {
                        next = e.next;
                        if ((e.hash & oldCap) == 0) {
                            if (loTail == null)
                                loHead = e;
                            else
                                loTail.next = e;
                            loTail = e;
                        } else {
                            if (hiTail == null)
                                hiHead = e;
                            else
                                hiTail.next = e;
                            hiTail = e;
                        }
                    } while ((e = next) != null);
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }
                }
            }
        }
    }
    return newTab;
}
```

关键改进是链表扩容时的处理。JDK 8 不再反转链表顺序，而是分成两部分：

- **低位链表（lo）**：元素的新位置与旧位置相同（j）
- **高位链表（hi）**：元素的新位置是旧位置加上旧容量（j + oldCap）

判断依据是 `(e.hash & oldCap) == 0`。由于 oldCap 是 2 的幂次方，`e.hash & oldCap` 只检查 hash 值新增的那一位（从 0 变成 1）。这一位为 0 的元素留在原位置，为 1 的元素移动到新位置。

对于红黑树，也有类似的 `split()` 方法，会将树分成两部分，如果某一部分节点太少，就转换回链表。

## 为什么尾插法能避免死循环

JDK 8 的尾插法为什么能避免扩容死循环？因为尾插法保证了扩容后链表的元素顺序与扩容前**一致**。

假设原链表顺序是 A -> B -> C，扩容后使用尾插法重新插入：

1. 处理 A：`newTab[i] = A`（newTab[i] 为空）
2. 处理 B：`A.next = B`，`newTab[i] = B`
3. 处理 C：`B.next = C`，`newTab[i] = C`

最终链表仍然是 A -> B -> C，顺序保持不变。

更重要的是，在并发环境下，即使两个线程同时扩容，由于顺序保持不变，不会形成环形引用。每个线程最终都能正确完成扩容。

## JDK 8 的其他优化

除了红黑树和尾插法，JDK 8 还做了以下优化：

1. **扰动函数简化**：`hash = (h = key.hashCode()) ^ (h >>> 16)` 简化为一次异或
2. **容量计算优化**：使用 `tableSizeFor()` 方法确保初始容量是 2 的幂次方
3. **空指针检查优化**：对 key 的 null 检查更合理
4. **注释更详细**：源码注释详细解释了设计决策

理解 JDK 8 的改进，不仅仅是学习一个数据结构，更是理解软件工程中"发现问题、解决问题"的迭代过程。
