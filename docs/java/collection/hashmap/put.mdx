# HashMap.put() 全流程源码解析

`put()` 方法是 HashMap 中最核心的操作之一。每次向 HashMap 添加一个键值对，都会经历哈希计算、寻址、冲突处理、扩容判断等一系列复杂步骤。深入理解 put 的全流程，是掌握 HashMap 的关键。

## put 方法的入口

`put()` 方法是对用户暴露的 API，它会调用内部的 `putVal()` 方法完成实际工作：

```java
public V put(K key, V value) {
    return putVal(hash(key), key, value, false, true);
}

static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

final V putVal(int hash, K key, V value, boolean onlyIfAbsent, boolean evict) {
    Node<K,V>[] tab; Node<K,V> p; int n, i;
    
    // 第一步：检查数组是否为空，如果是则扩容
    if ((tab = table) == null || (n = tab.length) == 0)
        n = (tab = resize()).length;
    
    // 第二步：计算桶位置，检查是否为空
    if ((p = tab[i = (n - 1) & hash]) == null)
        tab[i] = newNode(hash, key, value, null);
    else {
        Node<K,V> e; K k;
        
        // 第三步：桶不为空，处理冲突
        // 3.1 第一个节点的 key 与要插入的 key 相同
        if (p.hash == hash &&
            ((k = p.key) == key || (key != null && key.equals(k))))
            e = p;
        // 3.2 如果是树节点，调用树节点的插入方法
        else if (p instanceof TreeNode)
            e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
        // 3.3 否则遍历链表
        else {
            for (int binCount = 0; ; ++binCount) {
                // 到达链表尾部，没有找到相同的 key
                if ((e = p.next) == null) {
                    p.next = newNode(hash, key, value, null);
                    // 如果链表长度达到阈值，进行树化
                    if (binCount >= TREEIFY_THRESHOLD - 1)
                        treeifyBin(tab, hash);
                    break;
                }
                // 找到相同的 key
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    break;
                p = e;
            }
        }
        
        // 第四步：key 已存在，更新 value
        if (e != null) {
            V oldValue = e.value;
            if (!onlyIfAbsent || oldValue == null)
                e.value = value;
            afterNodeAccess(e);
            return oldValue;
        }
    }
    
    // 第五步：插入新节点，修改次数加一
    ++modCount;
    // 第六步：检查是否需要扩容
    if (++size > threshold)
        resize();
    afterNodeInsertion(evict);
    
    return null;
}
```

这个方法虽然不长，但逻辑非常丰富。让我们逐步分析每个步骤。

## 第一步：空表检查与初始化

```java
if ((tab = table) == null || (n = tab.length) == 0)
    n = (tab = resize()).length;
```

如果 table 数组为空（首次插入元素时），调用 `resize()` 方法进行初始化。`resize()` 在这里有两个作用：

1. 初始化数组（当 table 为 null 时）
2. 扩容（当 table 不为空时）

初始化时，`resize()` 会创建一个默认容量为 16 的数组，并设置阈值为 `16 * 0.75 = 12`。

## 第二步：计算桶位置

```java
if ((p = tab[i = (n - 1) & hash]) == null)
    tab[i] = newNode(hash, key, value, null);
```

`(n - 1) & hash` 是 HashMap 的寻址公式，它的效率远高于 `hash % n`。当桶位置为空时，直接将新节点放入即可，这是最理想的情况——O(1) 时间复杂度。

## 第三步：处理哈希冲突

当桶不为空时，需要处理哈希冲突。有三种情况：

### 情况一：第一个节点就是目标

```java
if (p.hash == hash &&
    ((k = p.key) == key || (key != null && key.equals(k))))
    e = p;
```

hash 值相同且 `equals()` 返回 true，说明找到了相同的 key。此时直接用新 value 覆盖旧 value 即可。

注意这里的短路逻辑：先比较 hash，再比较引用，最后才调用 `equals()`。`hashCode()` 是 O(1) 的，而 `equals()` 可能涉及复杂的对象比较。通过 hash 过滤，大多数不相等的对象在 `equals()` 之前就被排除了。

### 情况二：红黑树节点

```java
else if (p instanceof TreeNode)
    e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
```

如果第一个节点是 TreeNode（说明已经树化），调用红黑树的插入方法 `putTreeVal()`。红黑树的插入会保持树的有序性，并可能在必要时进行旋转平衡。

### 情况三：链表节点

```java
else {
    for (int binCount = 0; ; ++binCount) {
        if ((e = p.next) == null) {
            p.next = newNode(hash, key, value, null);
            if (binCount >= TREEIFY_THRESHOLD - 1)
                treeifyBin(tab, hash);
            break;
        }
        if (e.hash == hash &&
            ((k = e.key) == key || (key != null && key.equals(k))))
            break;
        p = e;
    }
}
```

遍历链表寻找目标 key。找到则更新 value；没找到则在链表末尾追加新节点（尾插法）。如果链表长度达到 8，调用 `treeifyBin()` 尝试将链表转换成红黑树。

## 第四步：更新已存在的 key

```java
if (e != null) {
    V oldValue = e.value;
    if (!onlyIfAbsent || oldValue == null)
        e.value = value;
    afterNodeAccess(e);
    return oldValue;
}
```

如果找到了已存在的 key，保存旧 value，然后根据 `onlyIfAbsent` 参数决定是否更新。`afterNodeAccess(e)` 是一个空方法，供 LinkedHashMap 等子类重写，用于维护访问顺序。

## 第五步和第六步：维护计数与扩容判断

```java
++modCount;
if (++size > threshold)
    resize();
afterNodeInsertion(evict);
```

每次插入成功后，`size` 加一。如果 `size` 超过阈值，调用 `resize()` 进行扩容。扩容会使容量翻倍，并重新分布所有元素。

## put 方法的复杂度分析

put 方法的时间复杂度取决于多种因素：

- **最佳情况**（桶为空）：O(1)
- **哈希冲突少**（链表长度 1-7）：O(1) ~ O(log n)
- **长链表未树化**：O(n)
- **红黑树**（已树化）：O(log n)

理论上，最坏情况是 O(n)，但由于负载因子 0.75 和树化阈值 8 的设计，正常情况下 HashMap 表现非常稳定。

## get 方法的对应流程

理解了 put，get 方法就很简单了：

```java
public V get(Object key) {
    Node<K,V> e;
    return (e = getNode(hash(key), key)) == null ? null : e.value;
}

final Node<K,V> getNode(int hash, Object key) {
    Node<K,V>[] tab; Node<K,V> first, e; int n; K k;
    if ((tab = table) != null && (n = tab.length) > 0 &&
        (first = tab[(n - 1) & hash]) != null) {
        if (first.hash == hash &&
            ((k = first.key) == key || (key != null && key.equals(k))))
            return first;
        if ((e = first.next) != null) {
            if (first instanceof TreeNode)
                return ((TreeNode<K,V>)first).find(hash, key, null);
            do {
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    return e;
            } while ((e = e.next) != null);
        }
    }
    return null;
}
```

get 方法遵循类似的逻辑：先定位桶位置，再检查第一个节点，然后遍历链表或搜索红黑树。
