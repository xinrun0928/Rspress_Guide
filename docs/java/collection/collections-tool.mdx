# Collections 工具类常用方法

Collections 是 Java 提供的一个工具类，它包含大量用于操作集合的静态方法。虽然 Java 8 之后集合类本身增加了大量默认方法，但 Collections 中的一些方法仍然非常实用，熟练掌握可以大大简化代码。

## 不可变集合

创建不可变（只读）集合是 Collections 的核心功能之一：

```java
// 空集合
List<String> emptyList = Collections.emptyList();
Set<Integer> emptySet = Collections.emptySet();
Map<String, Integer> emptyMap = Collections.emptyMap();

// 单元素集合
List<String> singletonList = Collections.singletonList("only");
Set<String> singleton = Collections.singleton("only");
Map.Entry<String, Integer> entry = Collections.singletonMap("key", 1);

// 包装为不可变集合
List<String> immutable = Collections.unmodifiableList(new ArrayList<>());
Set<Integer> readOnly = Collections.unmodifiableSet(new HashSet<>());
Map<String, Integer> unmodifiableMap = Collections.unmodifiableMap(new HashMap<>());
```

Java 9+ 提供了更简洁的语法：

```java
List<String> list = List.of("a", "b", "c");
Set<Integer> set = Set.of(1, 2, 3);
Map<String, Integer> map = Map.of("a", 1, "b", 2);
```

但 Collections 的方法在需要包装现有集合时仍然有用。

## 线程安全包装

Collections 提供了将普通集合包装为线程安全集合的方法：

```java
List<String> syncList = Collections.synchronizedList(new ArrayList<>());
Set<Integer> syncSet = Collections.synchronizedSet(new HashSet<>());
Map<String, Integer> syncMap = Collections.synchronizedMap(new HashMap<>());
Deque<String> syncDeque = Collections.synchronizedDeque(new ArrayDeque<>());
```

这些包装方法为集合的每个操作都加了 synchronized 锁，性能较差。仅在确实无法使用并发容器时才考虑使用。

Java 并发包提供了更好的替代：`ConcurrentHashMap`、`ConcurrentLinkedQueue`、`CopyOnWriteArrayList` 等。

## 排序相关方法

### sort：集合排序

```java
// 自然排序
List<Integer> numbers = new ArrayList<>(Arrays.asList(3, 1, 4, 1, 5));
Collections.sort(numbers);
System.out.println(numbers); // [1, 1, 3, 4, 5]

// 使用 Comparator
List<String> strings = new ArrayList<>(Arrays.asList("banana", "apple", "cherry"));
Collections.sort(strings, Comparator.comparingInt(String::length));
System.out.println(strings); // [apple, banana, cherry]

// 按多个条件排序
List<User> users = new ArrayList<>();
users.sort(Comparator.comparing(User::getAge).thenComparing(User::getName));
```

### reverse：反转顺序

```java
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));
Collections.reverse(list);
System.out.println(list); // [5, 4, 3, 2, 1]
```

### shuffle：打乱顺序

```java
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));
Collections.shuffle(list); // 使用默认随机源
System.out.println(list); // [3, 1, 5, 2, 4]（随机）

// 使用特定随机源
Random random = new Random(42);
Collections.shuffle(list, random);
System.out.println(list); // [2, 4, 1, 5, 3]（固定顺序，可复现）
```

shuffle 在实现洗牌算法、生成随机测试数据时很有用。

### rotate：循环移位

```java
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));
Collections.rotate(list, 2); // 向右旋转 2 位
System.out.println(list); // [4, 5, 1, 2, 3]

Collections.rotate(list, -2); // 向左旋转 2 位
System.out.println(list); // [1, 2, 3, 4, 5]
```

## 查找相关方法

### binarySearch：二分查找

```java
List<Integer> sorted = new ArrayList<>(Arrays.asList(1, 3, 5, 7, 9));

int index = Collections.binarySearch(sorted, 5);
System.out.println(index); // 2

int notFound = Collections.binarySearch(sorted, 4);
System.out.println(notFound); // -3（插入点是 index 2，即 -2-1）
```

**重要**：binarySearch 要求集合**已排序**，否则结果是未定义的。

### min/max：最小最大值

```java
List<Integer> numbers = new ArrayList<>(Arrays.asList(3, 1, 4, 1, 5, 9, 2, 6));

int min = Collections.min(numbers);
int max = Collections.max(numbers);
System.out.println(min + ", " + max); // 1, 9

// 使用 Comparator
String shortest = Collections.min(strings, Comparator.comparingInt(String::length));
```

## 填充与替换

### fill：填充集合

```java
List<String> list = new ArrayList<>(Arrays.asList("a", "b", "c"));
Collections.fill(list, "x");
System.out.println(list); // [x, x, x]
```

### replaceAll：替换所有匹配元素

```java
List<String> list = new ArrayList<>(Arrays.asList("a", "b", "a", "c", "a"));
Collections.replaceAll(list, "a", "A");
System.out.println(list); // [A, b, A, c, A]
```

## 添加与复制

### addAll：批量添加

```java
List<String> list = new ArrayList<>();
Collections.addAll(list, "a", "b", "c");
System.out.println(list); // [a, b, c]

// 从数组添加
String[] arr = {"x", "y", "z"};
Collections.addAll(list, arr);
System.out.println(list); // [a, b, c, x, y, z]
```

### copy：复制集合

```java
List<String> source = Arrays.asList("x", "y", "z");
List<String> dest = new ArrayList<>(Arrays.asList("a", "b", "c", "d", "e"));

Collections.copy(dest, source);
System.out.println(dest); // [x, y, z, d, e]

// dest 必须有足够的容量容纳 source
```

## 频率与全等检查

### frequency：计算频率

```java
List<String> list = Arrays.asList("a", "b", "a", "c", "a");
int countA = Collections.frequency(list, "a");
System.out.println(countA); // 3
```

### disjoint：检查是否无交集

```java
List<Integer> list1 = Arrays.asList(1, 2, 3);
List<Integer> list2 = Arrays.asList(4, 5, 6);
List<Integer> list3 = Arrays.asList(3, 4, 5);

System.out.println(Collections.disjoint(list1, list2)); // true（无交集）
System.out.println(Collections.disjoint(list1, list3)); // false（有交集 3）
```

## 交换与旋转

### swap：交换元素

```java
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));
Collections.swap(list, 0, 4);
System.out.println(list); // [5, 2, 3, 4, 1]
```

### rotate：循环移位

```java
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));
Collections.rotate(list, 2); // 向右移动 2 位
System.out.println(list); // [4, 5, 1, 2, 3]
```

## nCopies：创建重复元素的不可变集合

```java
List<String> repeated = Collections.nCopies(5, "item");
System.out.println(repeated); // [item, item, item, item, item]

// 这个集合的内存开销极小，因为底层只是一个对象加计数
```

## 使用 Collections 的注意事项

1. **优先使用 Java 9+ 的工厂方法**：`List.of()`、`Set.of()`、`Map.of()` 更简洁
2. **避免在循环中调用 Collections 方法**：考虑直接操作集合
3. **binarySearch 前确保已排序**：未排序的结果不可预测
4. **线程安全包装性能差**：能用并发容器就别用 synchronized 包装
5. **不可变集合适合作为常量**：创建后不再修改的集合

```java
// 好的实践
public static final List<String> CONSTANTS = List.of("A", "B", "C");

// 避免的做法
public static final List<String> CONSTANTS;
static {
    CONSTANTS = new ArrayList<>();
    Collections.addAll(CONSTANTS, "A", "B", "C");
}
```

Collections 工具类虽然历史悠久，但其中的许多方法在日常编程中仍然非常实用。熟练掌握这些方法，可以让你的代码更加简洁高效。
