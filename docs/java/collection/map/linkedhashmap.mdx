# LinkedHashMap 与 LRU 缓存实现

HashMap 是 Java 中最常用的键值对存储结构，但它的元素顺序是不可预测的——遍历 HashMap 的结果与插入顺序无关。如果你需要在保持高性能的同时维护元素的插入顺序或访问顺序，LinkedHashMap 就是答案。

## LinkedHashMap 的继承关系

LinkedHashMap 直接继承自 HashMap，它在 HashMap 的基础上增加了一条双向链表来记录元素的顺序：

```java
public class LinkedHashMap<K,V> extends HashMap<K,V> implements Map<K,V> {
    
    transient LinkedHashMap.Entry<K,V> head;
    transient LinkedHashMap.Entry<K,V> tail;
    
    final boolean accessOrder;
    
    static class Entry<K,V> extends HashMap.Node<K,V> {
        Entry<K,V> before, after;
        Entry(int hash, K key, V value, Node<K,V> next) {
            super(hash, key, value, next);
        }
    }
}
```

LinkedHashMap.Entry 继承自 HashMap.Node，增加了 `before` 和 `after` 两个指针，分别指向前一个和后一个节点。这条双向链表从头到尾串联起所有元素，遍历顺序由这条链表决定，与哈希桶的位置无关。

## 双向链表的维护机制

LinkedHashMap 通过重写 HashMap 的几个关键方法来维护双向链表：

```java
void afterNodeAccess(Node<K,V> p) { }
void afterNodeInsertion(boolean evict) { }
void afterNodeRemoval(Node<K,V> p) { }
```

HashMap 中的这些方法是空实现，LinkedHashMap 重写了它们：

```java
void afterNodeAccess(Node<K,V> e) {
    LinkedHashMap.Entry<K,V> last;
    if (accessOrder && (last = tail) != e) {
        LinkedHashMap.Entry<K,V> p =
            (LinkedHashMap.Entry<K,V>)e, b = p.before, a = p.after;
        p.after = null;
        if (b == null)
            head = a;
        else
            b.after = a;
        if (a != null)
            a.before = b;
        else
            last = b;
        if (last == null)
            head = p;
        else {
            p.before = last;
            last.after = p;
        }
        last = p;
        tail = last;
    }
}

void afterNodeInsertion(boolean evict) {
    LinkedHashMap.Entry<K,V> first;
    if (accessOrder && (first = head) != null && evict) {
        LinkedHashMap.Entry<K,V> eldest = first;
        if (removeEldestEntry(eldest)) {
            K key = eldest.key;
            removeNode(hash(key), key, null, false, true);
        }
    }
}

void afterNodeRemoval(Node<K,V> e) {
    LinkedHashMap.Entry<K,V> p =
        (LinkedHashMap.Entry<K,V>)e, b = p.before, a = p.after;
    if (b == null)
        head = a;
    else
        b.after = a;
    if (a != null)
        a.before = b;
    else
        tail = b;
}
```

这三个方法分别在**访问后**、**插入后**、**删除后**被调用，用于维护双向链表的完整性。

## 插入顺序模式（默认）

默认情况下，LinkedHashMap 的 `accessOrder = false`，处于插入顺序模式。在这种模式下：

- 元素按照插入的先后顺序排列
- 每次插入新元素，都将其添加到链表末尾
- 访问元素（get）不会改变顺序
- 删除元素会从链表中移除

```java
LinkedHashMap<String, Integer> map = new LinkedHashMap<>();
map.put("A", 1);
map.put("B", 2);
map.put("C", 3);
map.put("D", 4);

System.out.println(map.keySet()); // [A, B, C, D]

map.get("B"); // 访问 B，顺序不变

System.out.println(map.keySet()); // [A, B, C, D]，顺序保持不变
```

## 访问顺序模式：LRU 的秘密

当构造方法传入 `accessOrder = true` 时，LinkedHashMap 进入访问顺序模式。这种模式是实现 LRU 缓存的基础：

```java
LinkedHashMap<String, Integer> map = 
    new LinkedHashMap<>(16, 0.75f, true); // accessOrder = true
map.put("A", 1);
map.put("B", 2);
map.put("C", 3);
map.put("D", 4);

map.get("A"); // 访问 A

System.out.println(map.keySet()); // [B, C, D, A]
// A 被移到了链表末尾！
```

在访问顺序模式下，每次调用 `get()` 或 `put()`（更新已存在的 key）都会触发 `afterNodeAccess()` 方法，将被访问的节点移动到链表末尾。这使得链表头部永远是"最久未被访问"的元素。

## removeEldestEntry：自动淘汰的钩子

`afterNodeInsertion()` 方法中调用了 `removeEldestEntry()` 方法：

```java
protected boolean removeEldestEntry(Map.Entry<K,V> eldest) {
    return false;
}
```

这是一个 protected 方法，默认返回 false，不进行任何淘汰。如果你想实现自动淘汰最老元素的功能，可以重写这个方法：

```java
LinkedHashMap<Integer, String> cache = 
    new LinkedHashMap<>(16, 0.75f, true) {
        private static final int MAX_SIZE = 3;
        
        @Override
        protected boolean removeEldestEntry(Map.Entry<Integer, String> eldest) {
            return size() > MAX_SIZE;
        }
    };
```

当 `removeEldestEntry()` 返回 true 时，`afterNodeInsertion()` 会调用 `removeNode()` 删除链表头部（最老的元素）。

## 完整的 LRU 缓存实现

结合以上机制，我们可以实现一个功能完整的 LRU 缓存：

```java
public class LRUCache<K, V> {
    
    private final LinkedHashMap<K, V> cache;
    private final int capacity;
    
    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.cache = new LinkedHashMap<>(16, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
                return size() > LRUCache.this.capacity;
            }
        };
    }
    
    public V get(K key) {
        return cache.get(key);
    }
    
    public void put(K key, V value) {
        cache.put(key, value);
    }
    
    public int size() {
        return cache.size();
    }
    
    public void clear() {
        cache.clear();
    }
    
    public static void main(String[] args) {
        LRUCache<String, Integer> cache = new LRUCache<>(3);
        cache.put("A", 1);
        cache.put("B", 2);
        cache.put("C", 3);
        System.out.println(cache.size()); // 3
        System.out.println(cache.get("A")); // 1
        
        // 添加 D 会淘汰 B（最久未被访问）
        cache.put("D", 4);
        System.out.println(cache.size()); // 3
        System.out.println(cache.get("B")); // null，已被淘汰
        System.out.println(cache.get("A")); // 1
        System.out.println(cache.get("D")); // 4
    }
}
```

这个实现简洁而优雅，仅用 20 行代码就实现了线程不安全的 LRU 缓存。

## LinkedHashMap 与 HashMap 的性能对比

LinkedHashMap 在 HashMap 的基础上增加了双向链表的维护，这会带来一定的开销：

**空间开销**：每个 Entry 需要额外的 `before` 和 `after` 指针，在 64 位 JVM 上，每个指针 8 字节，每个 Entry 额外消耗 16 字节。

**时间开销**：每次插入、访问、删除时需要更新双向链表，这是 O(1) 的操作。但如果频繁访问元素，会有额外的链表调整开销。

**查询性能**：与 HashMap 相同，O(1) 平均时间复杂度。

**遍历性能**：与元素总数成正比 O(n)，而非与桶数量相关。对于只需要按顺序遍历的场景，LinkedHashMap 的遍历是连续的，缓存友好的。

## LinkedHashMap 的典型应用场景

除了 LRU 缓存，LinkedHashMap 还有其他实用场景：

**访问历史的记录**：浏览器的历史记录、文件的访问记录等。

**保持插入顺序的去重**：遍历时按照插入顺序，可以快速找到第一个重复的元素。

**配置项的顺序存储**：配置文件中的键值对通常需要保持定义的顺序。

**快速遍历优化**：当你需要频繁遍历 Map 并按插入顺序处理时，LinkedHashMap 的遍历性能优于 HashMap。

## 线程安全性

LinkedHashMap 本身不是线程安全的。如果在多线程环境下使用，需要外部同步或使用 `Collections.synchronizedMap()` 包装：

```java
Map<String, Integer> syncMap = 
    Collections.synchronizedMap(new LinkedHashMap<>());
```

或者使用并发容器 `ConcurrentHashMap`，但它不保证任何顺序。
