# TreeMap：Comparable vs Comparator

TreeMap 是 Java 中唯一直接实现有序 Map 的类。与 HashMap 的无序不同，TreeMap 能够按照键的自然顺序或自定义规则对键进行排序。这使得 TreeMap 成为需要有序遍历或范围查询场景的首选。但 TreeMap 的排序能力来自两种不同的机制：Comparable 和 Comparator，理解它们的区别和适用场景至关重要。

## TreeMap 的排序依赖

TreeMap 底层基于红黑树实现，树的排序依赖于键的比较。有两种方式提供比较规则：

1. **自然顺序**：key 类实现 Comparable 接口
2. **外部比较器**：创建 TreeMap 时传入 Comparator

```java
// 使用自然顺序
TreeMap<String, Integer> byNatural = new TreeMap<>();
byNatural.put("banana", 1);
byNatural.put("apple", 2);
byNatural.put("cherry", 3);
System.out.println(byNatural.keySet()); // [apple, banana, cherry]

// 使用自定义比较器（降序）
TreeMap<String, Integer> byCustom = new TreeMap<>(Comparator.reverseOrder());
byCustom.put("banana", 1);
byCustom.put("apple", 2);
byCustom.put("cherry", 3);
System.out.println(byCustom.keySet()); // [cherry, banana, apple]
```

## Comparable：对象自身的排序能力

Comparable 是 Java 提供的一个接口，定义了对象的"自然顺序"：

```java
public interface Comparable<T> {
    int compareTo(T o);
}
```

如果一个类实现了 Comparable，就说明这个类知道如何与同类型的其他实例比较。Java 的很多核心类都实现了 Comparable：

- 数值类型：Integer、Long、Double 等
- 字符串：String
- 日期时间：Date、LocalDate 等
- 集合类：Collections、Arrays（用于排序）

```java
public class User implements Comparable<User> {
    private String name;
    private int age;
    
    @Override
    public int compareTo(User other) {
        // 按年龄升序，年龄相同按姓名升序
        if (this.age != other.age) {
            return this.age - other.age;
        }
        return this.name.compareTo(other.name);
    }
}
```

使用自然顺序的 TreeMap：

```java
TreeMap<User, String> users = new TreeMap<>();
users.put(new User("Alice", 25), "A");
users.put(new User("Bob", 30), "B");
users.put(new User("Charlie", 20), "C");
// 按 User 的 compareTo 规则排序
```

## Comparator：外部定义的排序规则

Comparator 是独立于对象的比较器接口：

```java
@FunctionalInterface
public interface Comparator<T> {
    int compare(T o1, T o2);
    
    // Java 8+ 的默认方法
    default Comparator<T> reversed() { ... }
    default <U> Comparator<T> thenComparing(...) { ... }
    // ...
}
```

Comparator 适合以下场景：

1. **类没有实现 Comparable**：无法修改第三方类或遗留代码
2. **需要多种排序规则**：同一个类可能需要不同的排序方式
3. **不想污染原有类**：保持类的单一职责

```java
// 按姓名长度排序
TreeMap<String, Integer> byLength = new TreeMap<>(
    Comparator.comparingInt(String::length)
);
byLength.put("apple", 1);
byLength.put("hi", 2);
byLength.put("banana", 3);
// [hi, apple, banana] — 按长度排序

// 先按长度，再按字母序
TreeMap<String, Integer> byLengthThenAlpha = new TreeMap<>(
    Comparator.comparingInt(String::length).thenComparing(Comparator.naturalOrder())
);
```

## 两种方式的优先级

当 TreeMap 同时拥有自然顺序和比较器时，比较器优先级更高：

```java
TreeMap<String, Integer> map = new TreeMap<>(Comparator.reverseOrder());
map.put("apple", 1);
map.put("banana", 2);
map.put("cherry", 3);
// 按 reverseOrder() 排序：[cherry, banana, apple]
// 忽略 String 的自然顺序
```

如果使用无参构造，TreeMap 会使用 key 的自然顺序（要求 key 实现 Comparable）。

如果 key 没有实现 Comparable 且没有提供比较器，TreeMap 会在首次插入时抛出 `ClassCastException`。

## compareTo 和 compare 的契约

Comparable 的 compareTo 方法和 Comparator 的 compare 方法都需要遵守相同的契约：

```java
// sgn(compare(x, y)) == -sgn(compare(y, x))
// 即 compare(x, y) 和 compare(y, x) 符号相反

// 如果 compare(x, y) > 0 且 compare(y, z) > 0，则 compare(x, z) > 0
// 即传递性

// 如果 compare(x, y) == 0，则 compare(x, z) 和 compare(y, z) 符号相同
// 即与 equals 一致性

// （可选）强烈建议 compare(x, y) == 0 时，equals(x, y) 也为 true
// 这样才能保证与 SortedSet/SortedMap 行为一致
```

违反这些契约会导致 TreeMap 的行为不可预测，特别是当比较器不一致时。

## 实际应用场景

**使用 Comparable** 的场景：类本身有明确的、不会改变的排序语义，如日期、数值、字母序字符串。

**使用 Comparator** 的场景：

1. 遗留代码或第三方类

```java
// 第三方类，无法修改
public class LegacyItem {
    private int id;
    private String name;
    // getters, setters...
}

// 按 id 降序排列
TreeMap<LegacyItem, String> items = new TreeMap<>(
    Comparator.comparingInt(LegacyItem::getId).reversed()
);
```

2. 需要运行时决定的排序

```java
// 根据用户选择排序
String sortField = userInput; // "name" 或 "date" 或 "price"
Comparator<Item> comparator = switch(sortField) {
    case "name" -> Comparator.comparing(Item::getName);
    case "date" -> Comparator.comparing(Item::getDate);
    case "price" -> Comparator.comparingDouble(Item::getPrice);
    default -> Comparator.naturalOrder();
};
TreeMap<Item, Integer> sortedItems = new TreeMap<>(comparator);
```

3. 多条件排序

```java
TreeMap<Item, Integer> map = new TreeMap<>(
    Comparator.comparing(Item::getCategory)
             .thenComparing(Item::getPrice)
             .thenComparing(Item::getName)
);
```

## 红黑树的插入与比较

TreeMap 每次插入新元素时，都需要通过比较来确定插入位置：

```java
public V put(K key, V value) {
    Entry<K,V> t = root;
    if (t == null) {
        compare(key, key); // type check
        root = new Entry<>(key, value, null);
        size = 1;
        modCount++;
        return null;
    }
    int cmp;
    Entry<K,V> parent;
    Comparator<? super K> cpr = comparator;
    if (cpr != null) {
        do {
            parent = t;
            cmp = cpr.compare(key, t.key);
            if (cmp < 0)
                t = t.left;
            else if (cmp > 0)
                t = t.right;
            else
                return t.setValue(value);
        } while (t != null);
    } else {
        if (key == null)
            throw new NullPointerException();
        @SuppressWarnings("unchecked")
        Comparable<? super K> k = (Comparable<? super K>) key;
        do {
            parent = t;
            cmp = k.compareTo(t.key);
            if (cmp < 0)
                t = t.left;
            else if (cmp > 0)
                t = t.right;
            else
                return t.setValue(value);
        } while (t != null);
    }
    // 创建新节点并插入
    Entry<K,V> e = new Entry<>(key, value, parent);
    if (cmp < 0)
        parent.left = e;
    else
        parent.right = e;
    fixAfterInsertion(e);
    size++;
    modCount++;
    return null;
}
```

代码中的 `compare()` 方法会优先使用 Comparator（如果存在），否则使用 Comparable。

理解 Comparable 和 Comparator 的区别和使用场景，你就能在不同的排序需求中做出正确的选择。
