# WeakHashMap 与弱引用

Java 的内存管理通常被认为是自动的——垃圾回收器会自动回收不再使用的对象。但"不再使用"的定义是什么？对于普通的强引用，即使对象已经没有任何实际用途，只要还有引用指向它，垃圾回收器就不会回收它。WeakHashMap 引入了一种特殊的能力：允许垃圾回收器回收只有弱引用指向的对象。

## Java 的四种引用类型

理解 WeakHashMap 之前，需要先了解 Java 的四种引用类型：

**强引用（Strong Reference）**：最常见的引用类型，如 `String s = new String("hello")`。只要强引用存在，对象就不会被垃圾回收。

**软引用（Soft Reference）**：在内存不足时会被回收。`SoftReference<T>` 用于实现内存敏感的缓存。

**弱引用（Weak Reference）**：只要垃圾回收器运行，就会被回收。`WeakReference<T>` 用于实现规范化映射（canonicalizing mappings）。

**虚引用（Phantom Reference）**：无法通过虚引用获取对象，仅用于跟踪对象被垃圾回收的时间。`PhantomReference<T>` 用于安排清理操作。

```java
// 强引用
Object strong = new Object();

// 软引用
SoftReference<Object> soft = new SoftReference<>(new Object());

// 弱引用
WeakReference<Object> weak = new WeakReference<>(new Object());

// 虚引用（必须配合 ReferenceQueue 使用）
ReferenceQueue<Object> queue = new ReferenceQueue<>();
PhantomReference<Object> phantom = new PhantomReference<>(new Object(), queue);
```

## WeakHashMap 的核心机制

WeakHashMap 的特殊之处在于它的键使用弱引用存储：

```java
public class WeakHashMap<K,V> extends AbstractMap<K,V> implements Map<K,V> {
    
    private final ReferenceQueue<Object> queue = new ReferenceQueue<>();
    
    private static class Entry<K,V> extends WeakReference<Object> implements Map.Entry<K,V> {
        V value;
        int hash;
        Entry<K,V> next;
        
        Entry(Object key, V value, ReferenceQueue<Object> queue) {
            super(key, queue);
            this.value = value;
            this.hash = System.identityHashCode(key);
            this.next = null;
        }
    }
}
```

注意 `Entry` 继承自 `WeakReference<Object>`，并且将 key 作为弱引用的对象。当 key 只有 WeakHashMap 在引用时，垃圾回收器会回收这个 key。

## ReferenceQueue：回收通知机制

WeakReference 和 ReferenceQueue 配合使用，能够在对象被回收时获得通知：

```java
WeakHashMap<String, Integer> map = new WeakHashMap<>();
map.put("key", 1);

// 触发垃圾回收
System.gc();

// 处理被回收的键
Reference<? extends String> ref = map.pollReferencedEntries();
while (ref != null) {
    // 清理操作
    ref = map.pollReferencedEntries();
}
```

WeakHashMap 在每次操作（get、put、size 等）时，都会自动清理已经被回收的 Entry。这个过程通过 `expungeStaleEntries()` 方法实现：

```java
private void expungeStaleEntries() {
    for (Object x; (x = queue.poll()) != null; ) {
        synchronized (queue) {
            @SuppressWarnings("unchecked")
            Entry<K,V> e = (Entry<K,V>) x;
            int i = indexFor(e.hash, table.length);
            Entry<K,V> prev = table[i];
            Entry<K,V> p = prev;
            while (p != null) {
                Entry<K,V> next = p.next;
                if (p == e) {
                    if (prev == e)
                        table[i] = next;
                    else
                        prev.next = next;
                    e.value = null;
                    size--;
                    break;
                }
                prev = p;
                p = next;
            }
        }
    }
}
```

## WeakHashMap 的典型应用场景

**缓存**：当内存不足时自动清理缓存项

```java
WeakHashMap<ImageKey, Image> cache = new WeakHashMap<>();

public Image loadImage(ImageKey key) {
    Image image = cache.get(key);
    if (image == null) {
        image = loadFromDisk(key);
        cache.put(key, image);
    }
    return image;
}
```

这段代码实现了一个简单的缓存，当 ImageKey 对象不再被其他地方引用时，对应的缓存条目会被自动清理。

**规范映射**：当 key 对象生命周期结束时，自动清理映射

WeakHashMap 常用于实现"规范映射"——对象的某些属性与其他对象关联，但这个关联关系不应该阻止对象被回收。

**会话映射**：Web 应用中的用户会话数据

```java
WeakHashMap<HttpSession, SessionData> sessions = new WeakHashMap<>();
```

当用户的 HttpSession 被销毁时，对应的 SessionData 会自动从 Map 中移除。

## WeakHashMap 的注意事项

**键必须可哈希且 equals 稳定**

WeakHashMap 对键有特殊要求：键必须正确实现 equals() 和 hashCode()，并且在作为弱引用期间，这些方法的行为必须一致。如果键的哈希值在垃圾回收前后发生变化，会导致严重的内存泄漏。

**不适合作为普通缓存**

WeakHashMap 的清理时机不确定，不适合需要精确缓存控制的场景。如果需要内存敏感的强缓存，考虑使用 `WeakHashMap<SoftReference<K>, V>` 或专门的缓存库如 Guava Cache。

**非线程安全**

WeakHashMap 不是线程安全的，需要外部同步或使用 `Collections.synchronizedMap()` 包装。

**迭代期间的清理**

如果在迭代过程中有大量 Entry 被清理，可能会影响迭代器的行为。

## 弱引用与软引用的选择

选择弱引用还是软引用，取决于清理时机的需求：

**弱引用**：希望对象在下次垃圾回收时立即被清理，不管内存是否充足。适用于键值对映射，当键不再使用时希望立即清理。

**软引用**：希望对象在内存不足时才被清理，保留对象直到确实需要内存。适用于内存敏感但不紧急的缓存。

```java
// 软引用缓存
Map<K, SoftReference<V>> softCache = new HashMap<>();

// 弱引用映射
WeakHashMap<K, V> weakMap = new WeakHashMap<>();
```

理解 WeakHashMap 和弱引用的机制，你就能在合适的场景使用它们，实现更精细的内存管理和自动清理。
