# Map 接口体系全景

Map（映射）是 Java 集合框架中最重要的接口之一。与 Collection 体系（List、Set、Queue）不同，Map 用于存储键值对（Key-Value Pairs），每个键最多只能对应一个值。从电话簿（姓名 -> 电话号码）到字典（单词 -> 解释），Map 在现实世界中有无数对应的例子。

## Map 接口的设计哲学

Map 接口的设计围绕"键值对映射"这一核心概念展开。它的基本操作包括：

```java
public interface Map<K, V> {
    V put(K key, V value);
    V get(Object key);
    V remove(Object key);
    boolean containsKey(Object key);
    boolean containsValue(Object value);
    int size();
    boolean isEmpty();
    void putAll(Map<? extends K, ? extends V> m);
    void clear();
}
```

这些方法构成了 Map 的基础 API，Java 8 之后又引入了大量默认方法和函数式编程支持。

## Map 体系的三大分支

Java 的 Map 体系呈现出清晰的三层架构，每一层都有其独特的设计目标：

**第一层：通用 Map**

以 HashMap 为核心，还包括 LinkedHashMap（维护插入顺序）、TreeMap（按键排序）、WeakHashMap（弱引用键）等。这一层是日常开发中使用最频繁的。

**第二层：并发 Map**

以 ConcurrentHashMap 为代表，专门为高并发场景设计。这一层在 JDK 5 引入的 ConcurrentHashMap 起步，JDK 8 进行了重大改进。还有 ConcurrentSkipListMap 等变体。

**第三层：遗留 Map**

Hashtable 和 HashMap 的区别常常让人困惑。Hashtable 是 JDK 1.0 就存在的古老类，所有方法都是 synchronized 的，性能较差，已不推荐使用。

## HashMap vs LinkedHashMap vs TreeMap

这三个 Map 是最常用的选择，它们的特性对比：

| 特性 | HashMap | LinkedHashMap | TreeMap |
|------|---------|---------------|---------|
| 底层结构 | 数组 + 链表/红黑树 | HashMap + 双向链表 | 红黑树 |
| 时间复杂度 | O(1) 平均 | O(1) 平均 | O(log n) |
| 元素顺序 | 无序 | 插入顺序/访问顺序 | 排序顺序 |
| 键要求 | 无 | 无 | 可比较或提供比较器 |
| 额外开销 | 无 | 双向链表指针 | 树节点额外字段 |

选择的核心依据是：**你是否需要元素有序**？

- 不需要顺序：HashMap
- 需要插入顺序：LinkedHashMap
- 需要按键排序：TreeMap
- 需要 LRU 缓存：LinkedHashMap（accessOrder=true）

## Map 的视图（Views）

Map 提供了三个"视图"方法，可以将 Map 的内容以不同方式展示：

```java
Set<K> keySet();           // 所有键的集合
Collection<V> values();    // 所有值的集合
Set<Map.Entry<K, V>> entrySet(); // 所有键值对的集合
```

这些视图与底层 Map 是关联的——通过视图修改元素，等同于直接修改 Map：

```java
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);
map.put("B", 2);

Set<String> keys = map.keySet();
keys.remove("A"); // 从 keySet 中移除
System.out.println(map); // {B=2}，Map 也被修改了
```

这个特性既是便利也是陷阱。在遍历 Map 时，如果直接遍历 keySet 并删除元素（ConcurrentModificationException），应该使用 iterator.remove() 或 Map 的 remove 方法。

## Map 的默认方法（Java 8+）

Java 8 为 Map 接口添加了大量默认方法，使代码更加简洁：

```java
// getOrDefault：安全的获取，键不存在时返回默认值
int count = map.getOrDefault("key", 0);

// putIfAbsent：仅当键不存在时才插入
map.putIfAbsent("key", value);

// compute 系列：计算新值
map.compute(key, (k, v) -> v == null ? 1 : v + 1);

// merge：合并操作
map.merge(key, 1, Integer::sum);

// forEach：遍历
map.forEach((k, v) -> System.out.println(k + ": " + v));
```

这些方法大大简化了常见的 Map 操作，同时保持线程安全（单线程环境下）。

## equals() 与 hashCode()：Map 的契约

对于将自定义对象作为 Map 的 key，必须正确实现 equals() 和 hashCode() 方法。这是 HashMap 等基于哈希的 Map 正确工作的前提。

如果两个对象 equals()，它们的 hashCode() 必须相同。如果 hashCode() 相同，它们不一定 equals()（哈希碰撞）。

一个常见的错误是：在创建对象后修改了影响 hashCode() 或 equals() 的字段。这会导致在 HashMap 中"丢失"这个键。

## Map 与对象的关系

在 Java 中，Map 不是继承自 Collection 接口。但它与 Collection 体系有着密切的关系：

- Map 的 values() 返回 Collection
- Map 的 keySet() 和 entrySet() 返回 Set
- HashSet 内部使用 HashMap，TreeSet 内部使用 TreeMap

这种设计体现了 Java 集合框架的模块化思想——不同的接口关注不同的抽象层次。

理解 Map 体系的架构和选择原则，你就能在合适的场景选择最合适的数据结构，写出更高效、更优雅的代码。
