# Java 17+ 新特性

Java 从 1995 年诞生至今，已经走过了近三十年的历程。从最早的 Oak 语言，到如今的现代化编程平台，Java 经历了无数次重大变革。每一个 LTS（长期支持）版本，都像是一座里程碑，记录着 Java 生态的进化史。

<Badge text="JDK 9" type="warning" /> <Badge text="JDK 11" type="tip" /> <Badge text="JDK 17" type="tip" /> <Badge text="JDK 21" type="tip" />

你可能听说过"Java 5 的泛型革命"、"Java 8 的函数式编程时代"，但你知道吗？**Java 17 到 21 这五年，是 Java 诞生以来变化最密集的五年**。虚拟线程、Record 类型、模式匹配、密封类、开关表达式增强……这些新特性不是小打小闹的语法糖，而是从底层改变了我们编写 Java 代码的方式。

想象一下：过去启动一万个线程来处理并发任务，你需要担心堆栈大小、上下文切换开销、JVM 参数调优；但有了虚拟线程，一百万个并发任务也可以轻松搞定。过去定义一个"不可变数据传输对象"，你需要写一个类、添加字段、生成 getter/setter/equals/hashCode/toString，一不小心还可能写错；但有了 Record，一行代码就能搞定。

本章节将带你系统梳理 **JDK 9 到 JDK 21** 的核心新特性。我们会从"这些东西怎么来的、为什么要这样设计"讲起，让你不仅知道怎么用，更理解为什么这样设计，以及背后的权衡取舍。

准备好了吗？让我们开始这场 Java 新特性的探索之旅。
