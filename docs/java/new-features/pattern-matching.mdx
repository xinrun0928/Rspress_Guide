## 一、instanceof 的模式匹配 🔴

### 1.1 问题拆解

模式匹配是Java 14~21逐步引入的系列特性，目的是让代码更简洁、更类型安全。这道题考察候选人对Java类型系统演进的掌握程度。

**第一层：基本语法**
- 面试官问："`instanceof`的模式匹配怎么用？"
- 考察点：类型模式、局部变量模式

**第二层：语法演进**
- 面试官追问："模式匹配经历了哪些Java版本的演进？"
- 考察点：JEP 394/405/441等版本路线

**第三层：switch模式匹配**
- 面试官继续问："switch的模式匹配比传统写法好在哪里？"
- 考察点：穷举性检查、guard条件

**第四层：record模式**
- 面试官问："Record怎么配合模式匹配使用？"
- 考察点：嵌套模式、解构

### 1.2 ❌ 错误示范

**错误回答1："instanceof模式匹配就是语法糖，写法更简洁"**

不完全是。模式匹配不仅仅是语法简化，它还带来**编译器穷举性检查**——当你用sealed class + switch模式匹配时，漏掉任何case编译器都会报错。

**错误回答2："switch的模式匹配就是`case String s`这样用"**

这只是第一层。Java 21的switch模式匹配支持**guard条件**（`when`子句）、**record嵌套解构**、**null处理**等多种高级用法。

**错误回答3："模式匹配会影响性能"**

模式匹配是**编译时特性**，在字节码层面和手写的`instanceof + cast`等价，不会引入额外运行时开销。

【面试官心理】
模式匹配这道题我通常用来试探候选人是否跟进Java新版本的特性。知道instanceof模式匹配的占50%，知道switch模式匹配演进历史的占20%，能说出guard条件和record嵌套模式的占10%。Java 21是目前的LTS，模式匹配已经相当成熟。

### 1.3 标准回答

#### 第一步：instanceof模式匹配——Java 16正式版（JEP 394）

```java
// Java 16之前
if (obj instanceof String) {
    String s = (String) obj; // 重复类型声明
    System.out.println(s.length());
}

// Java 16+
if (obj instanceof String s) { // 模式变量s立即生效
    System.out.println(s.length()); // 直接用s，不用转型
}
```

**关键语义**：

```java
Object obj = getData();

// 1. 模式变量s的作用域
if (obj instanceof String s && s.length() > 3) { // ✅ s在&&右侧可见
    System.out.println(s); // ✅ 在if体内可见
}

// 2. null检查隐含在模式中
if (obj instanceof String s) {
    // obj是null时，条件为false，不会进入
    System.out.println(s);
}

// 3. 多重条件
if (obj instanceof String s && s.startsWith("prefix")) {
    System.out.println(s);
}
```

**作用域规则**：模式变量的作用域被**智能地限制**在条件成立时。如果`instanceof`在`&&`或`||`的左侧，模式变量在右侧可见。

#### 第二步：switc模式匹配——Java 21正式版（JEP 441）

```java
// Java 21之前
String describe(Object obj) {
    if (obj instanceof Integer) {
        return "整数: " + (Integer) obj;
    } else if (obj instanceof String) {
        return "字符串: " + ((String) obj).length();
    } else if (obj instanceof List) {
        return "列表: " + ((List<?>) obj).size();
    } else {
        return "未知";
    }
}

// Java 21+
String describe(Object obj) {
    return switch (obj) {
        case Integer i -> "整数: " + i;
        case String s -> "字符串: " + s.length();
        case List<?> l -> "列表: " + l.size();
        case null -> "是null";
        case default -> "未知: " + obj;
    };
}
```

#### 第三步：Guard条件（`when`子句）

```java
// 只有特定条件才匹配
String classify(Object obj) {
    return switch (obj) {
        case Integer i when i > 0 -> "正整数: " + i;
        case Integer i when i < 0 -> "负整数: " + i;
        case Integer i -> "零";
        case String s when s.length() > 10 -> "长字符串: " + s;
        case String s -> "短字符串: " + s;
        default -> "其他";
    };
}
```

**Guard条件规则**：
- `when`子句在case标签后，`->`前
- Guard条件为false时，继续尝试下一个case
- 相当于在匹配后加了一个额外的条件判断

#### 第四步：Record模式——嵌套解构

```java
record Point(int x, int y) {}
record Circle(Point center, int radius) {}

String describeShape(Object shape) {
    return switch (shape) {
        case Circle(Point(int x, int y), int r) ->
            "圆心在(" + x + "," + y + ")，半径" + r;
        // 嵌套record模式：直接解构内层record
        case null -> "是null";
        case default -> "其他形状";
    };
}
```

```java
// 更复杂的嵌套
record Box<T>(T content) {}
record Pair<A, B>(A first, B second) {}

String process(Object obj) {
    return switch (obj) {
        case Box<Pair<String, Integer>>(Pair(String s, Integer i)) ->
            "盒子装着一对: " + s + "和" + i;
        // 解构类型和值同时进行
        case Box<Integer>(Integer i) -> "盒子装着整数: " + i;
        case null -> "是null";
        case default -> "其他";
    };
}
```

:::tip 💡
**Record模式的好处**：不需要在case中手动转型和解构。比如`case Circle(Point p, int r)`，如果你用传统写法，需要先`instanceof Circle`，再转型，再从Circle对象中调用`p.center().x()`。Record模式直接一步到位。
:::

### 2.1 各版本JEP一览

| Java版本 | JEP | 特性 |
| --- | --- | --- |
| Java 14 | JEP 305 | instanceof模式匹配（预览） |
| Java 16 | JEP 394 | instanceof模式匹配（正式） |
| Java 17 | JEP 406 | switch模式匹配（预览） |
| Java 21 | JEP 441 | switch模式匹配（正式） |
| Java 21 | JEP 439 | 模式匹配for循环（预览） |
| Java 22 | JEP 454 | 模式匹配中future/external配置（预览） |

### 2.2 完整演进示例

```java
// Java 8：传统写法
if (obj instanceof Integer) {
    Integer i = (Integer) obj;
    System.out.println(i * 2);
}

// Java 16：instanceof带模式
if (obj instanceof Integer i) {
    System.out.println(i * 2);
}

// Java 21：switch带模式匹配
return switch (obj) {
    case Integer i -> "整数的两倍: " + (i * 2);
    case String s when s.length() > 5 -> "长字符串: " + s;
    case String s -> "短字符串: " + s;
    case null -> "null值";
    default -> "其他类型: " + obj.getClass().getName();
};
```

## 三、穷举性检查 🟡

### 3.1 sealed class + switch模式匹配 = 编译器保证

```java
sealed interface Expr permits Literal, Add, Variable {}
record Literal(int value) implements Expr {}
record Add(Expr left, Expr right) implements Expr {}
record Variable(String name) implements Expr {}

// 编译器穷举检查：所有Expr实现都必须处理
int eval(Expr e) {
    return switch (e) {
        case Literal(int v) -> v;
        case Add(Expr l, Expr r) -> eval(l) + eval(r);
        case Variable(String name) -> lookup(name);
    };
}
// 如果忘记处理某个case，编译直接报错
```

### 3.2 穷举性的边界

```java
// 当default存在时，穷举性检查被绕过
return switch (obj) {
    case Integer i -> "整数";
    case String s -> "字符串";
    default -> "其他"; // 有default，不强制穷举
};
```

:::warning ⚠️
**default和穷举性的关系**：当你使用sealed接口但不想强制穷举时，可以加`default`分支。但这会失去编译器检查的好处。最佳实践是：**优先穷举，不加default**——新增实现类时，编译器会强制你补全所有switch。
:::

## 四、实战应用 🟢

### 4.1 工厂模式的类型安全版本

```java
sealed interface Shape permits Circle, Rectangle, Triangle {}
record Circle(double radius) implements Shape {}
record Rectangle(double w, double h) implements Shape {}
record Triangle(double base, double h) implements Shape {}

double area(Shape shape) {
    return switch (shape) {
        case Circle(double r) -> Math.PI * r * r;
        case Rectangle(double w, double h) -> w * h;
        case Triangle(double base, double h) -> base * h / 2;
    };
}
```

### 4.2 JSON解析中的模式匹配

```java
sealed interface JsonValue permits JsonObject, JsonArray, JsonString, JsonNumber, JsonNull {}
record JsonObject(Map<String, JsonValue> fields) implements JsonValue {}
record JsonArray(List<JsonValue> elements) implements JsonValue {}
record JsonString(String value) implements JsonValue {}
record JsonNumber(double value) implements JsonValue {}
record JsonNull() implements JsonValue {}

String toPretty(JsonValue v, int indent) {
    return switch (v) {
        case JsonObject(Map<String, JsonValue> fields) -> formatObject(fields, indent);
        case JsonArray(List<JsonValue> elements) -> formatArray(elements, indent);
        case JsonString(String s) -> "\"" + s + "\"";
        case JsonNumber(Double d) -> d.toString();
        case JsonNull() -> "null";
    };
}
```
