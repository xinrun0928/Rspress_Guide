## 一、文本块到底是什么？🔴

### 1.1 问题拆解

这道题是**P6了解级**题目，Java 15引入的文本块在实际项目中应用场景有限，但面试官会追问细节来考察候选人对Java语言特性的钻研程度。

**第一层：基本使用**
- 面试官问："文本块怎么用？"
- 考察点："""三引号语法、缩进处理

**第二层：转义序列**
- 面试官追问："`\n`、`\s`、`\t`在文本块里有什么区别？"
- 考察点：行终止符自动处理、结尾空格处理

**第三层：格式化集成**
- 面试官问："文本块和String.formatted()怎么结合？"
- 考察点：stripIndent()、格式化控制

### 1.2 ❌ 错误示范

**错误回答1："文本块就是Java 15的语法糖，用三引号表示多行字符串"**

文本块不仅仅是语法糖，它有严格的语义规则——**缩进处理、行终止符规范化、转义序列**。

**错误回答2："文本块会自动去掉所有空格"**

错误！文本块只会根据closing delimiter的位置做**最小缩进移除**，**行末空格会被保留**（除非用`\`明确去除）。

**错误回答3："文本块可以直接用`""`表示空字符串"**

文本块的closing delimiter是`"""`，前面有空格会导致编译错误：`""" content """`是合法的，但空文本块必须是`"""`。

### 1.3 标准回答

#### 第一步：文本块解决的问题

```java
// Java 14之前的写法——丑陋、易错
String json = "{\n" +
              "    \"name\": \"张三\",\n" +
              "    \"age\": 30\n" +
              "}";

// 文本块写法——清晰、可维护
String json = """
        {
            "name": "张三",
            "age": 30
        }
        """;
```

三大改进：
1. **多行字符串无需手动拼接**
2. **无需转义双引号**（在文本块内）
3. **提高可读性和可维护性**

#### 第二步：缩进规则——最容易被忽略的细节

文本块的缩进规则是**以closing delimiter为基准做最小缩进移除**：

```java
// 原始写法
String s = """
        line 1
        line 2
        """;

// 编译后s的内容（不包含左边的缩进空格）：
// "line 1\nline 2\n"
// 移除的是"最小公共缩进"——即"        "这8个空格

// closing delimiter的位置决定了移除多少
String s = """
        line 1
    """; // closing delimiter只有4个空格，所以移除4个空格
// 结果："line 1\n    "  行末保留了2个空格
```

**closing delimiter的位置决定缩进基准**：

```java
String s1 = """
        line 1
        """;    // 基准缩进=8空格

String s2 = """
        line 1
    """;   // 基准缩进=4空格，s1和s2内容相同
```

**行末空格**：

```java
String s = """  // closing delimiter前没有空格，行末空格会被保留
        line 1
        line 2
        """;  // 2后面没有空格

// 如果想在行末保留空格，用\s
String s = """\
        line 1\
        """;  // "line 1 " 末尾有空格
```

#### 第三步：转义序列

| 转义 | 含义 | 说明 |
| --- | --- | --- |
| `\"` | 双引号 | 文本块内不需要，但在行首时需要 |
| `\\` | 反斜杠 | 避免被解释为续行符 |
| `\n` | 换行 | 显式换行 |
| `\r\n` | 回车+换行 | Windows风格行终止符 |
| `\s` | 空格（space） | 显式保留行末空格 |
| `\t` | Tab | 制表符 |
| `\<line-terminator>` | 行续符 | 禁止自动换行 |

**`\<line-terminator>`——禁止自动换行**：

```java
// 普通写法：会自动换行
String s1 = """
        SELECT * FROM users
        WHERE id = 1
        """;

// 行续符：两行变成一行
String s2 = """
        SELECT * FROM users \
        WHERE id = 1
        """;

System.out.println(s1); // 包含换行符
System.out.println(s2); // 没有换行符："SELECT * FROM users WHERE id = 1"
```

## 二、文本块的格式化控制 🟡

### 2.1 formatted方法——与String.formatted集成

```java
String sql = """
        SELECT * FROM %s
        WHERE id = %d
          AND status = '%s'
        """.formatted("users", 1, "active");
// "SELECT * FROM users\nWHERE id = 1\n  AND status = 'active'\n"
```

### 2.2 indent方法——Java 15+的动态缩进调整

```java
String s = """
        line 1
        line 2
        """;

// indent(+4)：给每行加4个空格
s.indent(4);

// indent(-2)：给每行减2个空格
s.indent(-2);

// indent的工作原理：先split("\n")，每行加/减缩进，再join("\n")
```

### 2.3 stripIndent——去除缩进

```java
// 用于从字符串中移除"假装的"缩进空白
String raw = "\n    line 1\n    line 2\n";
String cleaned = raw.stripIndent();
// "line 1\nline 2\n"
```

:::tip 💡
**实际应用场景**：文本块最适合用在**JSON/SQL/HTML/XML**等多行字符串场景。避免了大量的`"` + `"` + `"`拼接，代码可读性大幅提升。在代码生成器、SQL Builder、模板渲染等场景特别有价值。
:::

## 三、文本块与模板引擎的取舍 🟢

### 3.1 什么时候用文本块？

| 场景 | 推荐 | 原因 |
| --- | --- | --- |
| 内嵌JSON/SQL/HTML | ✅ 文本块 | 静态内容，不需要变量替换 |
| 多行错误信息 | ✅ 文本块 | 静态，可读性优先 |
| 动态内容 | ⚠️ 文本块+formatted | 简单格式化 |
| 复杂模板（条件、循环） | ❌ 文本块 | 需要模板引擎 |
| 需要运行时拼装 | ❌ 文本块 | 文本块是编译期常量 |

### 3.2 JSON处理——文本块 vs 序列化

```java
// 场景：测试时需要硬编码JSON
// 文本块方案
String requestJson = """
        {
            "name": "张三",
            "age": 30
        }
        """;

// 序列化方案（更推荐）
User user = new User("张三", 30);
String requestJson = objectMapper.writeValueAsString(user);
```

:::warning ⚠️
**文本块的一个坑**：文本块不能用于注解的字符串常量，除非是Java 15+的预览特性。编译时常量文本块在某些场景下可能有性能优化（String字面量会被intern），但动态文本块不会。
:::
