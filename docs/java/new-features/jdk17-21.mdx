# JDK 17~21 LTS 新特性

2018 年，Java 突然宣布：从 Java 11 开始，每六个版本发布一个 LTS（长期支持）版本。节奏从原来的"缓慢演进"变成了"持续交付"——每六个月，Java 社区就会迎来一次新特性预览或正式发布。这种" train model"（火车模型）发布节奏，让 Java 的现代化进程以前所未有的速度推进。

而 JDK 17，作为 2021 年 9 月发布的 LTS 版本，标志着 Java 现代化的一个重要节点——从这一版本开始，大量预览特性正式转正，新特性开始密集出现。本节，我们就来全面梳理 JDK 17 到 JDK 21 期间那些你不得不知的重要变革。

## JDK 17：现代 Java 的起点

JDK 17 是一个里程碑式的 LTS 版本。它不仅将 JDK 12~16 的多个预览特性正式转正，还包含了许多面向未来的新能力。

**密封类正式转正**（JDK 15 预览）。密封类通过 `sealed` 修饰符和 `permits` 子句，让父类精确控制继承层次，从根本上防止未授权的扩展。这个特性在类型系统和架构设计上意义重大——它让"穷尽性检查"成为可能，编译器可以确保每一种可能的类型都被处理。

**Switch 表达式正式转正**（JDK 12 预览，JDK 14 完善）。`case L ->` 箭头语法消除了 fall-through 的陷阱，switch 作为表达式返回值让函数式风格更加自然。结合 instanceof 模式匹配，switch 可以匹配类型、null、具体值，语法表达能力大幅增强。

**Records 正式转正**（JDK 14 预览）。`public record Point(int x, int y) {}` 一行替代了几十行模板代码。Record 天生不可变、基于内容相等性、自带 toString，是定义"值对象"的完美工具。结合密封接口，Record 可以构建完整的代数数据类型体系。

**随机数生成器增强**。JDK 17 引入了更灵活、更安全的随机数生成 API。新的 `RandomGenerator` 接口和 `SplittableRandomGenerator` 等实现类，让随机数测试、游戏开发等场景有了更可靠的基础设施。

**移除 Applet API**。 Applet 这个从 JDK 1.0 就存在的古老技术，在 JDK 17 中被正式移除。如果你还在维护使用 Applet 的遗留系统，现在是时候彻底告别了。

## JDK 18~20：承前启后的过渡期

JDK 18、19、20 继续以每六个月一个版本的速度发布，继续完善预览特性和引入新的实验性 API。这三个版本虽然没有新的 LTS，但预览了 JDK 21 中一些最激动人心的特性。

**JEP 422：Linux/RISC-V 移植**（JDK 20 正式）。Java 对 RISC-V 架构的官方支持，让 Java 在物联网和嵌入式领域有了更广阔的天地。

**字符集 UTF-8 为默认**（JDK 18）。此前 Java 在不同操作系统上使用不同默认字符集，导致跨平台行为不一致。JDK 18 统一使用 UTF-8 作为默认字符集，减少了"在 Windows 上开发、在 Linux 上部署"的字符编码坑。

**外部函数和内存 API**（Foreign Function & Memory API）。这个特性在 JDK 18 孵化、JDK 19 预览、JDK 21 正式发布。它让你可以在纯 Java 中安全地调用 native 代码（C/C++）和操作堆外内存，无需 JNI 的复杂模板：

```java
// JDK 21 正式 API：通过纯 Java 调用 native C 函数
MemorySegment segment = MemorySegment.ofArray(new byte[1024]);
// 告别 System.loadLibrary() 和 JNI 的样板代码
```

这对高性能计算、科学计算、以及与系统底层交互的场景意义重大——既保持了 Java 的安全性，又获得了接近 native 的性能。

**Vector API**（JDK 16 孵化，JDK 21 第三轮预览）。在 CPU SIMD 指令集上运行向量计算，让数值计算性能提升数倍：

```java
// JDK 21 Vector API：对数组进行向量化计算
var vector = FloatVector.fromArray(SPECIES_256, values, 0);
var result = vector.mul(vector).add(vector);
```

## JDK 21：集大成之作

JDK 21 是继 JDK 17 之后的又一个 LTS 版本，被 Oracle 称为"史上最大的 Java 发行版"。它汇集了从 JDK 18 到 JDK 20 的所有预览特性，并正式推出了几项革命性的新功能。

**虚拟线程正式转正**（JDK 21 GA）。这是 Java 21 最具影响力的特性，我们将单独用一整节来深入讲解。简而言之：虚拟线程让 Java 的并发编程彻底改变——以前需要精心维护的线程池，现在可以"按需创建、随手丢弃"，百万并发的门槛大幅降低。

**模式匹配 for switch 正式转正**（JDK 21 GA）。从 JDK 17 开始预览的模式匹配 for switch，终于在 JDK 21 正式可用：

```java
String describe = switch (obj) {
    case Integer i when i > 0 -> "正整数 " + i;
    case Integer i -> "非正整数 " + i;
    case String s -> "字符串 '" + s + "'";
    case null, default -> "未知类型";
};
```

注意 `when` 子句——它允许在类型匹配的基础上进一步加条件判断。这种"类型 + 条件"的模式组合，让代码的表达能力达到了前所未有的高度。

**Record 模式**（JDK 21 GA）。模式和 Record 的结合，让解构嵌套数据结构变得异常优雅：

```java
// 解构嵌套的 Record 和 List
void printSum(Object obj) {
    if (obj instanceof Point(int x, int y)) {
        System.out.println(x + y);
    }
}

// switch 中使用 Record 模式
return switch (e) {
    case Add(Number n1, Number n2) -> new Number(n1.value() + n2.value());
    case Multiply(Number n, Expr e2), case Multiply(Expr e1, Number n)
        -> simplifyProduct(n, e1, e2); // 逗号分隔多个模式
    default -> e;
};
```

**有序集合（Sequenced Collections）**。JDK 21 引入了一组新的接口：`SequencedCollection`、`SequencedSet`、`SequencedMap`，为所有有序集合提供了统一的 API。这些接口定义了 `getFirst()`、`getLast()`、`reversed()` 等方法，结束了 Java 多年以来 List 有 `get(0)`、Queue 有 `peek()`、Deque 有 `getFirst()` 而没有统一 API 的混乱局面。

**String Templates**（JDK 21 预览）。字符串模板终于来了！`"Hello, #{name}!"` 语法的引入，让字符串插值不再需要 `String.format()` 或 `formatted()`：

```java
String message = "Hello, #{user.name}! You have #{user.messageCount} messages.";
```

但要注意：JDK 21 正式版中 String Templates 实际上是第四轮预览（第一轮在 JDK 21 的早期版本中），最终在 JDK 21 中以预览形式存在，JDK 22 才正式发布。

**unnamed Patterns and Variables**（JDK 21 GA）。用下划线 `_` 表示"我不关心"的模式变量，让代码意图更明确：

```java
// 只关心第一个和第三个元素，第二个无所谓
switch (point) {
    case Point(int x, _, int z) -> System.out.println(x + z);
}
```

## 版本选择的建议

面对如此多的新版本，如何选择？

生产环境优先选择 **LTS 版本**：JDK 17 和 JDK 21 是目前最稳妥的选择。JDK 17 成熟稳定，生态完善；JDK 21 面向未来，功能更强大。JDK 11 作为上一个 LTS 也依然有大量使用，但新项目建议至少以 JDK 17 为起点。

如果你的项目使用了 Spring Framework 6 或 Jakarta EE，现代框架对 JDK 版本的要求也在逐步提高——JDK 17 是当前的"最低水位"，JDK 21 则是面向未来的选择。

---

JDK 17 到 JDK 21 这段时期，是 Java 诞生以来新特性最密集、现代化程度最高的阶段。虚拟线程重新定义了 Java 的并发模型，模式匹配让类型系统更强大，Record 和密封类让数据建模更优雅。
