# IaC 工具对比：选择适合你的方案

「Ansible、Terraform、Pulumi——都在说基础设施即代码，到底该用哪个？」

这三个工具虽然都归属于「基础设施即代码」范畴，但它们的设计哲学、核心能力、适用场景都有显著差异。选择正确的工具，事半功倍；选错了，可能寸步难行。

## 三大工具定位

### Ansible：配置管理专家

Ansible 诞生于 2012 年，核心定位是**配置管理和应用部署**。它的强项是「在已存在的基础设施上配置服务器」。

```
┌─────────────────────────────────────────────────────────────┐
│                    Ansible 工作模型                           │
│                                                              │
│  ┌─────────────┐                                            │
│  │ Ansible    │  ← Control Node                           │
│  │ Playbook   │     (安装 Ansible 的机器)                    │
│  │            │                                            │
│  │ inventory  │                                            │
│  │ modules    │                                            │
│  └──────┬──────┘                                            │
│         │ SSH                                               │
│  ┌──────┴──────┐    ┌──────┴──────┐    ┌──────┴──────┐     │
│  │  Server 1  │    │  Server 2  │    │  Server 3  │     │
│  │  (目标)    │    │  (目标)    │    │  (目标)    │     │
│  └─────────────┘    └─────────────┘    └─────────────┘     │
│                                                              │
│  特点：无 Agent（通过 SSH）、幂等性、丰富的模块                │
└─────────────────────────────────────────────────────────────┘
```

### Terraform：基础设施创建专家

Terraform 专注**创建和管理云基础设施资源**。它的强项是「从零开始创建一套完整的环境」。

```
┌─────────────────────────────────────────────────────────────┐
│                    Terraform 工作模型                        │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  Terraform Core                      │   │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐              │   │
│  │  │ State   │  │  Plan   │  │  Apply  │              │   │
│  │  │ Manager │  │ Engine  │  │ Engine  │              │   │
│  │  └─────────┘  └─────────┘  └─────────┘              │   │
│  └─────────────────────────────────────────────────────┘   │
│                              │                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ AWS         │  │ Azure       │  │ GCP         │         │
│  │ Provider    │  │ Provider    │  │ Provider    │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                              │
│  特点：声明式、状态管理、Provider 生态                         │
└─────────────────────────────────────────────────────────────┘
```

### Pulumi：编程式基础设施

Pulumi 用**真正的编程语言**（Python/TypeScript/Go）写基础设施代码，兼具配置管理和编程能力。

```
┌─────────────────────────────────────────────────────────────┐
│                    Pulumi 工作模型                           │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   Pulumi Runtime                     │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │  你的代码 (Python/TS/Go)                     │   │   │
│  │  │  const vpc = new aws.ec2.Vpc("main", {...})│   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │  语言运行时 + 资源 Provider                   │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
│                              │                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ AWS         │  │ Kubernetes  │  │ 更多...     │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                              │
│  特点：编程能力、类型安全、测试友好                           │
└─────────────────────────────────────────────────────────────┘
```

## 核心对比

### 功能定位

| 维度 | Ansible | Terraform | Pulumi |
|---|---|---|---|
| **核心定位** | 配置管理 | 基础设施创建 | 通用 IaC |
| **主要用途** | 服务器配置 | 云资源创建 | 云资源创建 + 配置 |
| **资源类型** | 服务器、网络 | 云资源 | 云资源 + 应用 |
| **执行方式** | SSH 连接 | Provider API | Provider API |
| **是否无状态** | 无状态 | 有状态（State）| 有状态（State）|

### 语言能力

| 维度 | Ansible | Terraform | Pulumi |
|---|---|---|---|
| **配置语言** | YAML (Playbook) | HCL (DSL) | Python/TS/Go |
| **循环** | Jinja2 loop | count/meta | for 循环 |
| **条件** | when | count/ternary | if/else |
| **函数** | Jinja2 filter | HCL 函数 | 语言原生函数 |
| **抽象** | Roles/Includes | Modules | 类/函数 |
| **测试** | 有限 | Terratest | 单元/集成测试 |

### 云支持

| 维度 | Ansible | Terraform | Pulumi |
|---|---|---|---|
| **AWS** | ✅ 丰富 | ✅ 丰富 | ✅ 丰富 |
| **Azure** | ✅ 丰富 | ✅ 丰富 | ✅ 丰富 |
| **GCP** | ✅ 丰富 | ✅ 丰富 | ✅ 丰富 |
| **Kubernetes** | ✅ 模块 | ✅ Provider | ✅ 原生支持 |
| **VMware** | ✅ | ⚠️ 需要 Provider | ⚠️ |
| **裸金属** | ✅ 优秀 | ❌ 不支持 | ❌ 不支持 |

### 状态管理

| 维度 | Ansible | Terraform | Pulumi |
|---|---|---|---|
| **状态管理** | 无状态 | State 文件 | State 文件 |
| **状态位置** | 无 | 本地/远程 | 本地/云端 |
| **状态锁定** | 无 | 支持（S3+DynamoDB）| 支持（Pulumi Cloud）|
| **状态导入** | 支持 | 支持 | 支持 |

## 详细对比

### 创建资源 vs 配置资源

```yaml
# Ansible: 配置已存在的服务器
- name: 安装 Nginx
  hosts: webservers
  become: yes
  tasks:
    - name: 安装 Nginx
      yum:
        name: nginx
        state: present

    - name: 配置 Nginx
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/nginx.conf
      notify: 重启 Nginx
```

```hcl
# Terraform: 创建云资源
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  tags = {
    Name = "web-server"
  }
}
```

```typescript
// Pulumi: 用代码创建云资源
const instance = new aws.ec2.Instance("web", {
    ami: "ami-0c55b159cbfafe1f0",
    instanceType: "t3.micro",
    tags: { Name: "web-server" },
});
```

### 幂等性对比

| 工具 | 幂等性保证 | 说明 |
|---|---|---|
| **Ansible** | 原生支持 | 重复执行结果相同 |
| **Terraform** | 声明式 | State 驱动，天然幂等 |
| **Pulumi** | 声明式 | 同 Terraform |

### 场景推荐

| 场景 | 推荐方案 | 原因 |
|---|---|---|
| 服务器配置管理 | Ansible | 原生支持，模块丰富 |
| 云基础设施创建 | Terraform | State 管理，Provider 生态 |
| 混合云管理 | Ansible + Terraform | 各取所长 |
| Kubernetes 配置 | Pulumi/Ansible | Pulumi 原生支持 |
| 复杂逻辑基础设施 | Pulumi | 编程能力 |

| 无 Agent 配置 | Ansible | SSH 无 Agent |
| 应用部署 | Ansible | 成熟生态 |

## 组合使用

### Ansible + Terraform

```
┌─────────────────────────────────────────────────────────────┐
│                    最佳组合模式                               │
│                                                              │
│  Terraform                                    Ansible        │
│  ┌─────────────────┐                        ┌────────────┐ │
│  │ 1. 创建基础设施  │                        │ 配置服务器  │ │
│  │   - VPC         │                        │ - 安装软件  │ │
│  │   - 子网         │                        │ - 配置应用  │ │
│  │   - 安全组       │                        │ - 部署代码  │ │
│  │   - 服务器       │ ────────────────────→ │ - 健康检查  │ │
│  └─────────────────┘     (SSH 目标)          └────────────┘ │
│                                                              │
│  Terraform: 创建                              Ansible: 配置  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 实战示例

```hcl
# Terraform: 创建基础设施
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"
  key_name      = aws_key_pair.mykey.key_name

  tags = {
    Name = "web-server"
    Role = "web"
  }
}

# Terraform 输出 Ansible 主机清单
output "ansible_inventory" {
  value = <<-EOT
[webservers]
${aws_instance.web.public_ip} ansible_user=ubuntu ansible_python_interpreter=/usr/bin/python3
EOT
}
```

```yaml
# Ansible: 配置服务器
- name: 配置 Web 服务器
  hosts: webservers
  become: yes
  tasks:
    - name: 安装 Nginx
      apt:
        name: nginx
        state: present

    - name: 部署应用代码
      git:
        repo: https://github.com/myorg/myapp.git
        dest: /var/www/myapp

    - name: 重启 Nginx
      service:
        name: nginx
        state: restarted
```

## 总结

三个工具不是互斥的，而是互补的：

- **Terraform**：云基础设施创建的标杆，State 管理完善
- **Ansible**：服务器配置的老将，无 Agent、模块丰富
- **Pulumi**：IaC 的未来，编程能力带来无限可能

<Badge text="最佳实践" type="tip" />

**推荐组合：**

```yaml
# 小团队，简单需求
Terraform  # 够用

# 中型团队，云 + 配置
Terraform + Ansible  # 创建 + 配置

# 大型团队，复杂逻辑
Terraform + Ansible + Pulumi  # 各取所长

# 云原生团队
Pulumi  # 一站式
```

**记住：**
- Terraform 不擅长配置管理，Ansible 不擅长创建资源
- Pulumi 试图成为「全能选手」，但生态不如 Terraform 成熟
- 最好的工具是能解决你问题的工具，不是功能最多的工具
