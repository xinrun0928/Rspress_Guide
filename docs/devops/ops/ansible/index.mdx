# Ansible 自动化运维

想象一个场景：凌晨 3 点，你需要给 100 台服务器更新 SSL 证书。手动 SSH 登录每一台？重复操作 100 次？万一中途密码改了怎么办？

这就是 Ansible 诞生的背景——**让运维工程师从重复劳动中解放出来**。

## Ansible 是什么

Ansible 是 Red Hat 出品的自动化运维工具，用最简单的方式实现：

- **配置管理**：服务器初始化、软件安装、配置文件同步
- **应用部署**：一键部署应用到多台机器
- **任务编排**：按顺序执行一系列任务
- **批量操作**：批量执行 Shell 命令

Ansible 的核心哲学是：**简单、高效、无代理**。

## 核心特性

| 特性 | 说明 |
|---|---|
| **Agentless** | 无需在被管理节点安装任何软件 |
| **幂等性** | 同一任务执行多次，结果一致 |
| **YAML 语法** | playbook 使用人类可读的 YAML 格式 |
| **模块化** | 丰富的内置模块，涵盖各种场景 |
| **SSH 通信** | 通过 SSH 协议与被管理节点通信 |

## 架构一览

```
┌─────────────────┐
│  Control Node   │  ← 运行 Ansible 的机器（你的笔记本/服务器）
│  (Ansible 安装)  │
└────────┬────────┘
         │ SSH
         ▼
┌─────────────────┐
│ Managed Nodes   │  ← 被管理的目标机器
│  (无 Agent)     │     只需要 SSH 即可
└─────────────────┘
```

Control Node（控制节点）上安装 Ansible，通过 SSH 连接到 Managed Nodes（被管理节点），推送并执行任务。

## 快速入门

### 安装 Ansible

```bash
# macOS
brew install ansible

# Ubuntu/Debian
sudo apt update
sudo apt install ansible

# 验证安装
ansible --version
```

### 编写 Inventory 文件

Inventory 文件定义了 Ansible 要管理哪些主机：

```ini title="inventory.ini"
[webservers]
web01.example.com
web02.example.com
web03.example.com

[dbservers]
db01.example.com
db02.example.com

[all:vars]
ansible_user=admin
ansible_port=22
```

### 基础命令

```bash
# 测试连通性
ansible all -i inventory.ini -m ping

# 查看主机信息
ansible all -i inventory.ini -m setup

# 执行临时命令
ansible all -i inventory.ini -m command -a "uptime"

# 执行 Shell 命令
ansible all -i inventory.ini -b -m shell -a "systemctl status nginx"
```

`-b` 参数表示使用 sudo 提权执行。

## Ansible 能做什么

### 场景一：批量安装软件

```bash
# 一次性在所有 web 服务器上安装 Nginx
ansible webservers -i inventory.ini -b -m apt -a "name=nginx state=present"
```

### 场景二：配置文件管理

```bash
# 同步配置文件到所有服务器
ansible all -i inventory.ini -b -m copy -a "src=./nginx.conf dest=/etc/nginx/nginx.conf"
```

### 场景三： playbook 编排

当任务复杂时，用 Playbook 剧本化：

```bash
# 执行 playbook
ansible-playbook -i inventory.ini deploy.yml
```

## 模块生态系统

Ansible 拥有超过 3000 个模块，覆盖各种场景：

| 类别 | 模块示例 |
|---|---|
| 系统管理 | `user`、`group`、`systemd`、`cron` |
| 软件包 | `apt`、`yum`、`dnf`、`pip` |
| 文件操作 | `copy`、`template`、`lineinfile`、`synchronize` |
| 云服务 | `aws_*`、`azure_*`、`gcp_*`、`k8s_*` |
| 容器 | `docker_container`、`docker_image`、`podman` |
| 数据库 | `mysql_db`、`postgresql_db`、`mongodb` |
| 网络 | `uri`、`get_url`、`slurp` |

:::tip
想知道某个模块怎么用？使用 `ansible-doc <module-name>` 查看详细文档。

```bash
ansible-doc copy
ansible-doc template
ansible-doc docker_container
```
:::
