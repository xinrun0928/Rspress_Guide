# Ansible 架构原理

你有没有想过——SSH 登录 100 台服务器，手动执行命令需要多久？如果要改一个配置，要一台一台改吗？

Ansible 就是来解决这个问题的。Ansible 通过 SSH 连接到被管理节点，执行自动化任务，无需在被管理节点安装任何代理。

## Ansible 核心概念

### 四大核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                      Ansible 架构                             │
│                                                              │
│  ┌─────────────────┐                                       │
│  │  Inventory      │ ← 定义主机清单                         │
│  │  (主机列表)      │    192.168.1.10, 192.168.1.20...   │
│  └────────┬────────┘                                       │
│           │                                                  │
│  ┌────────┴────────┐                                       │
│  │    Playbook      │ ← 定义任务清单                        │
│  │    (YAML)        │    - hosts: webservers               │
│  └────────┬────────┘      tasks:                           │
│           │                 - name: Install nginx            │
│  ┌────────┴────────┐                                       │
│  │     Module       │ ← 执行具体操作                        │
│  │    (插件)        │    yum, copy, template, service...   │
│  └────────┬────────┘                                       │
│           │                                                  │
│  ┌────────┴────────┐                                       │
│  │    Ansible Core  │                                       │
│  │                  │                                       │
│  │  ┌─────────────┐ │                                       │
│  │  │  pyYAML    │ │                                       │
│  │  │  Paramiko   │ │  ← SSH 连接                         │
│  │  │  Jinja2     │ │  ← 模板引擎                         │
│  │  └─────────────┘ │                                       │
│  └────────┬────────┘                                       │
│           │ SSH                                              │
│  ┌────────┴────────┐                                       │
│  │  Managed Nodes   │                                       │
│  │  (无 Agent)      │                                       │
│  │  只需要 SSH      │                                       │
│  └─────────────────┘                                       │
└─────────────────────────────────────────────────────────────┘
```

### Inventory：主机清单

Inventory 定义了 Ansible 管理的主机列表，可以是静态文件或动态脚本：

```ini title="inventory.ini"
# 基础主机
[webservers]
web01.example.com
web02.example.com
web03.example.com

[dbservers]
db01.example.com
db02.example.com

[all:vars]
ansible_user=admin
ansible_port=22
ansible_ssh_private_key_file=~/.ssh/id_rsa
```

```yaml title="inventory.yaml"
all:
  children:
    webservers:
      hosts:
        web01.example.com:
          http_port: 80
          max_connections: 1000
        web02.example.com:
          http_port: 80
          max_connections: 1000
    dbservers:
      hosts:
        db01.example.com:
          db_port: 3306
        db02.example.com:
          db_port: 3306
  vars:
    ansible_user: admin
    ansible_ssh_private_key_file: ~/.ssh/id_rsa
```

## 模块系统

### 常用模块

Ansible 拥有超过 3000 个内置模块，按功能分类：

| 类别 | 模块 | 说明 |
|---|---|---|
| **包管理** | `yum`、`apt`、`dnf` | 安装/卸载软件包 |
| **文件操作** | `copy`、`template`、`file` | 复制、模板、文件属性 |
| **服务管理** | `service`、`systemd` | 启动/停止/重启服务 |
| **命令执行** | `command`、`shell`、`script` | 执行命令或脚本 |
| **Git** | `git` | 代码仓库操作 |
| **Docker** | `docker_container`、`docker_image` | Docker 操作 |
| **K8s** | `k8s`、`helm` | Kubernetes 操作 |
| **云服务** | `aws_*`、`azure_*`、`gcp_*` | 云资源管理 |
| **数据库** | `mysql_db`、`postgresql_db` | 数据库操作 |
| **通知** | `slack`、`mail`、`webhook` | 发送通知 |

### 模块使用示例

```yaml
# 文件复制
- name: Copy nginx config
  copy:
    src: ./nginx.conf
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: '0644'
    backup: yes  # 备份原文件

# 模板渲染
- name: Render application config
  template:
    src: app.conf.j2
    dest: /etc/myapp/config.conf
    owner: app
    group: app
    mode: '0640'
    validate: '/usr/sbin/nginx -t -c %s'

# 服务管理
- name: Start nginx
  service:
    name: nginx
    state: started
    enabled: yes
    use: systemd
```

## Playbook：任务清单

### Playbook 结构

```yaml title="deploy.yml"
---
# 定义主机组
- hosts: webservers
  # 远程执行用户
  remote_user: root
  # sudo 提权
  become: yes
  become_user: app
  become_method: sudo

  # 收集主机信息
  gather_facts: yes

  # 变量
  vars:
    app_version: "1.2.3"
    app_path: /opt/myapp

  # 任务列表
  tasks:
    - name: Ensure app directory exists
      file:
        path: "{{ app_path }}"
        state: directory
        owner: app
        group: app
        mode: '0755'

    - name: Deploy application
      synchronize:
        src: ./dist/
        dest: "{{ app_path }}/current"
        delete: yes
        rsync_opts:
          - "--exclude=.git"

    - name: Restart application
      systemd:
        name: myapp
        state: restarted
      notify:
        - Wait for application to start

  # 处理程序
  handlers:
    - name: Wait for application to start
      pause:
        seconds: 5
```

### 条件执行

```yaml
- name: Install nginx on Debian
  apt:
    name: nginx
    state: present
  when: ansible_os_family == "Debian"

- name: Install nginx on RedHat
  yum:
    name: nginx
    state: present
  when: ansible_os_family == "RedHat"
```

### 循环

```yaml
# 循环创建用户
- name: Create users
  user:
    name: "{{ item }}"
    state: present
    groups: wheel
  loop:
    - alice
    - bob
    - charlie

# 循环创建多个目录
- name: Create directories
  file:
    path: "/opt/{{ item }}"
    state: directory
    mode: '0755'
  loop:
    - app
    - logs
    - backups
```

## Roles：角色组织

### Role 目录结构

```
roles/
└── myapp/
    ├── defaults/          # 默认变量（最低优先级）
    │   └── main.yml
    ├── vars/              # 角色变量（高优先级）
    │   └── main.yml
    ├── handlers/          # 处理程序
    │   └── main.yml
    ├── tasks/            # 任务
    │   └── main.yml
    ├── templates/         # 模板文件
    │   └── config.j2
    ├── files/            # 静态文件
    │   └── init.sh
    ├── meta/              # 依赖关系
    │   └── main.yml
    └── README.md
```

### Role 示例：安装 Nginx

```yaml title="roles/nginx/tasks/main.yml"
---
- name: Install nginx
  apt:
    name: nginx
    state: present
    update_cache: yes

- name: Copy nginx config
  template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: '0644'
  notify: Restart nginx

- name: Enable nginx
  systemd:
    name: nginx
    enabled: yes
    state: started
```

```yaml title="roles/nginx/handlers/main.yml"
---
- name: Restart nginx
  systemd:
    name: nginx
    state: restarted
```

```yaml title="roles/nginx/templates/nginx.conf.j2"
server {
    listen {{ nginx_port | default(80) }};
    server_name {{ nginx_server_name | default('_') }};

    location / {
        root {{ nginx_root | default('/var/www/html') }};
        index index.html;
    }
}
```

### Playbook 使用 Role

```yaml title="site.yml"
---
- hosts: webservers
  become: yes
  roles:
    - role: nginx
      nginx_port: 8080
      nginx_server_name: myapp.example.com
    - role: myapp
      myapp_version: "1.2.3"
```

## 动态 Inventory

### AWS EC2 动态清单

```bash
# 安装 AWS inventory 插件
ansible-galaxy collection install amazon.aws

# 配置 ec2.ini
[ec2]
regions = us-east-1,us-west-2
instance_filters = tag:Environment=production

[credentials]
aws_access_key_id = AKIA...
aws_secret_access_key = ...
```

```bash
# 使用动态清单
ansible-inventory -i ec2.py --list
ansible all -i ec2.py -m ping
```

### 自定义动态清单脚本

```python title="my_inventory.py"
#!/usr/bin/env python3
import json
import sys

def get_inventory():
    return {
        "webservers": {
            "hosts": ["web01.example.com", "web02.example.com"]
        },
        "dbservers": {
            "hosts": ["db01.example.com", "db02.example.com"]
        },
        "_meta": {
            "hostvars": {
                "web01.example.com": {"ansible_host": "192.168.1.10"},
                "web02.example.com": {"ansible_host": "192.168.1.11"},
            }
        }
    }

if __name__ == "__main__":
    if "--list" in sys.argv:
        print(json.dumps(get_inventory()))
    elif "--host" in sys.argv:
        print(json.dumps({}))
```

### ansible.cfg 配置

```ini title="ansible.cfg"
[defaults]
inventory = inventory/production/hosts
roles_path = roles:/usr/share/ansible/roles
host_key_checking = False
retry_files_enabled = False
gathering = smart
fact_caching = jsonfile
fact_caching_connection = /tmp/ansible_facts
callback_whitelist = timer, profile_tasks

[privilege_escalation]
become = True
become_method = sudo
become_user = root
become_ask_pass = False

[ssh_connection]
pipelining = True
ssh_args = -o ControlMaster=auto -o ControlPersist=60s
```

## 总结

Ansible 的核心要点：

1. **无 Agent**：通过 SSH 连接，零部署成本
2. **幂等性**：同一任务执行多次，结果一致
3. **YAML Playbook**：人类可读的自动化脚本
4. **模块化**：3000+ 内置模块，覆盖各种场景
5. **Roles**：可复用的任务组织方式
6. **动态 Inventory**：灵活的主机管理

Ansible 是自动化运维的利器，掌握它可以让你的运维工作事半功倍。
