# Ansible Roles 与 Galaxy

你有没有想过——每次写 Playbook 都要重复配置 Nginx、安装 Java 环境？有没有办法把这些通用的任务封装成可复用的模块？

Ansible Roles 就是来解决这个问题的。Roles 让你把任务、变量、模板、处理器组织成可分享、可复用的结构，配合 Ansible Galaxy，可以直接使用社区贡献的成熟 Roles。

## Role 是什么

Role 是 Ansible 中组织任务的标准方式，将相关的任务、变量、模板、文件打包成一个可复用的单元。

### Role vs 普通 Playbook

```yaml
# 普通 Playbook：所有内容堆在一起
- hosts: webservers
  tasks:
    - name: Install nginx
      apt:
        name: nginx
    - name: Copy config
      copy:
        src: nginx.conf
        dest: /etc/nginx/nginx.conf
    - name: Start nginx
      service:
        name: nginx
        state: started
```

```yaml
# 使用 Role：任务封装在 roles/nginx 中
- hosts: webservers
  roles:
    - nginx
```

## Role 目录结构

### 完整结构

```
roles/
└── myapp/
    ├── defaults/              # 默认变量（最低优先级）
    │   └── main.yml
    ├── vars/                  # 角色变量（高优先级）
    │   └── main.yml
    ├── tasks/                # 主任务列表
    │   ├── main.yml
    │   └── install.yml
    ├── handlers/              # 处理程序
    │   └── main.yml
    ├── templates/              # Jinja2 模板
    │   ├── config.j2
    │   └── nginx.conf.j2
    ├── files/                  # 静态文件
    │   ├── init.sh
    │   └── backup.sh
    ├── meta/                   # 依赖关系
    │   └── main.yml
    ├── library/                # 自定义模块
    │   └── my_filter.py
    ├── module_utils/           # 模块工具
    │   └── my_util.py
    └── README.md               # 文档
```

### 最小 Role 结构

```
roles/
└── basic/
    ├── tasks/
    │   └── main.yml
    ├── handlers/
    │   └── main.yml
    └── defaults/
        └── main.yml
```

## Role 编写指南

### tasks/main.yml：核心任务

```yaml title="roles/myapp/tasks/main.yml"
---
# tasks/main.yml
- name: Include install tasks
  include_tasks: install.yml
  when: myapp_action in ['install', 'deploy']

- name: Include configure tasks
  include_tasks: configure.yml
  when: myapp_action in ['configure', 'deploy']

- name: Include service tasks
  import_tasks: service.yml
```

```yaml title="roles/myapp/tasks/install.yml"
---
- name: Create application user
  user:
    name: "{{ myapp_user }}"
    system: yes
    shell: /bin/bash
    create_home: no

- name: Create application directories
  file:
    path: "{{ item }}"
    state: directory
    owner: "{{ myapp_user }}"
    group: "{{ myapp_group }}"
    mode: '0755'
  loop:
    - "{{ myapp_home }}"
    - "{{ myapp_log_dir }}"
    - "{{ myapp_data_dir }}"

- name: Download application package
  get_url:
    url: "{{ myapp_download_url }}"
    dest: /tmp/{{ myapp_package_name }}
    mode: '0644'
  when: myapp_download_url is defined
```

```yaml title="roles/myapp/tasks/service.yml"
---
- name: Ensure service is running
  systemd:
    name: "{{ myapp_service_name }}"
    state: "{{ myapp_service_state }}"
    enabled: "{{ myapp_service_enabled }}"
    daemon_reload: yes

- name: Check service health
  uri:
    url: "http://{{ myapp_listen_host }}:{{ myapp_listen_port }}/health"
    status_code: 200
  register: health_check
  until: health_check.status == 200
  retries: 10
  delay: 3
  when: myapp_service_state == 'started'
```

### defaults/main.yml：默认变量

```yaml title="roles/myapp/defaults/main.yml"
---
# 基本配置
myapp_user: app
myapp_group: app
myapp_home: /opt/myapp
myapp_version: "1.0.0"

# 变量
myapp_action: deploy
myapp_service_name: myapp
myapp_service_state: started
myapp_service_enabled: yes

# 端口和主机
myapp_listen_host: "0.0.0.0"
myapp_listen_port: 8080

# 目录
myapp_log_dir: "{{ myapp_home }}/logs"
myapp_data_dir: "{{ myapp_home }}/data"

# 下载（可选）
myapp_download_url: ""
myapp_package_name: "myapp-{{ myapp_version }}.tar.gz"

# 配置模板
myapp_config_template: "config.yml.j2"
```

### handlers/main.yml：处理程序

```yaml title="roles/myapp/handlers/main.yml"
---
- name: Restart myapp
  systemd:
    name: "{{ myapp_service_name }}"
    state: restarted

- name: Reload myapp
  systemd:
    name: "{{ myapp_service_name }}"
    state: reloaded

- name: Restart nginx
  systemd:
    name: nginx
    state: restarted
```

### templates：Jinja2 模板

```yaml title="roles/myapp/templates/config.yml.j2"
# Application Configuration
# Generated by Ansible

app:
  name: {{ myapp_service_name }}
  version: {{ myapp_version }}
  home: {{ myapp_home }}

server:
  host: {{ myapp_listen_host }}
  port: {{ myapp_listen_port }}

logging:
  level: {{ myapp_log_level | default('INFO') }}
  path: {{ myapp_log_dir }}/app.log

database:
{% if myapp_db_host is defined %}
  host: {{ myapp_db_host }}
  port: {{ myapp_db_port | default(3306) }}
  name: {{ myapp_db_name }}
{% endif %}
```

### meta/main.yml：依赖关系

```yaml title="roles/myapp/meta/main.yml"
---
galaxy_info:
  author: Your Name
  description: My application role
  company: Your Company
  license: MIT
  min_ansible_version: "2.9"
  platforms:
    - name: EL
      versions:
        - "7"
        - "8"
    - name: Ubuntu
      versions:
        - focal
        - jammy
  galaxy_tags:
    - app
    - web
    - deployment

dependencies:
  - role: geerlingguy.java
    java_version: "17"
  - role: common
    common_timezone: "{{ myapp_timezone }}"
```

## Role 使用方式

### 在 Playbook 中使用

```yaml title="playbooks/deploy.yml"
---
- name: Deploy myapp
  hosts: webservers
  become: yes

  roles:
    # 直接指定 role（使用默认变量）
    - role: myapp

    # 带参数指定
    - role: myapp
      myapp_version: "1.2.3"
      myapp_action: deploy

    # 条件执行
    - role: nginx
      when: enable_nginx | default(true)

    # 标签（用于 --tags 选择执行）
    - role: myapp
      tags: [app, deploy]
```

### Pre-tasks 和 Post-tasks

```yaml
- name: Deploy application
  hosts: webservers
  become: yes

  pre_tasks:
    - name: Gather facts
      setup:
        filter: ansible_distribution*
      when: ansible_facts == {}

    - name: Notify Slack before deployment
      slack:
        token: "{{ slack_token }}"
        msg: "Starting deployment to {{ inventory_hostname }}"
      changed_when: false

  roles:
    - myapp

  tasks:
    - name: Additional task after role
      debug:
        msg: "Role execution completed"

  post_tasks:
    - name: Notify deployment complete
      slack:
        token: "{{ slack_token }}"
        msg: "Deployment to {{ inventory_hostname }} completed"
      changed_when: false
```

## Ansible Galaxy

### Galaxy 是什么

Ansible Galaxy（galaxy.ansible.com）是 Ansible 的社区角色市场，托管了数千个由社区贡献的预构建 Roles 和 Collections。

### 常用 Galaxy Roles

| Role | 作者 | 说明 |
|---|---|---|
| `geerlingguy.nginx` | Jeff Geerling | 安装和配置 Nginx |
| `geerlingguy.redis` | Jeff Geerling | 安装和配置 Redis |
| `geerlingguy.docker` | Jeff Geerling | 安装 Docker |
| `geerlingguy.java` | Jeff Geerling | 安装 Java |
| `ANXS.postgresql` | ANXS | 安装 PostgreSQL |
| `elastic.elasticsearch` | Elastic | 安装 Elasticsearch |

### 安装和使用

```bash
# 搜索角色
ansible-galaxy search nginx
ansible-galaxy search --author geerlingguy

# 查看角色详情
ansible-galaxy role info geerlingguy.nginx

# 安装角色
ansible-galaxy role install geerlingguy.nginx
ansible-galaxy role install geerlingguy.redis geerlingguy.docker

# 查看已安装的角色
ansible-galaxy role list

# 删除角色
ansible-galaxy role remove geerlingguy.nginx
```

### requirements.yml 管理依赖

```yaml title="requirements.yml"
# roles/requirements.yml
roles:
  # 从 Galaxy 安装
  - name: geerlingguy.nginx
  - name: geerlingguy.redis
    version: "3.1.0"  # 指定版本
  - name: geerlingguy.docker
    src: geerlingguy.docker  # 可以省略 Galaxy 用户名
    version: "4.2.0"

  # 从 Git 安装
  - name: my-custom-role
    src: https://github.com/myorg/ansible-role-myapp
    scm: git
    version: main

  # 从 Tar 包安装
  - name: internal-role
    src: /path/to/internal-role.tar.gz
```

```bash
# 安装 requirements.yml 中的所有依赖
ansible-galaxy install -r roles/requirements.yml

# 安装到指定目录
ansible-galaxy install -r requirements.yml -p ./roles
```

### Collections

Collections 是 Ansible 2.9+ 引入的新一代打包格式，包含 Roles、Modules、Plugins：

```yaml title="requirements.yml"
collections:
  - name: kubernetes.core
    version: "2.4.0"
  - name: amazon.aws
    version: "6.0.0"
  - name: community.general
```

```bash
# 安装 Collection
ansible-galaxy collection install kubernetes.core

# 从 requirements.yml 安装
ansible-galaxy collection install -r requirements.yml
```

## 完整示例：Web 应用部署

### 目录结构

```
project/
├── inventory/
│   └── production/
│       ├── hosts
│       └── group_vars/
│           └── webservers.yml
├── playbooks/
│   └── web.yml
├── roles/
│   ├── common/
│   │   └── tasks/
│   │       └── main.yml
│   ├── nginx/
│   │   └── tasks/
│   │       └── main.yml
│   └── myapp/
│       ├── defaults/
│       │   └── main.yml
│       ├── tasks/
│       │   ├── main.yml
│       │   └── install.yml
│       ├── handlers/
│       │   └── main.yml
│       └── templates/
│           └── config.yml.j2
└── ansible.cfg
```

### Playbook

```yaml title="playbooks/web.yml"
---
- name: Deploy web application
  hosts: webservers
  become: yes
  gather_facts: yes

  roles:
    - common
    - role: nginx
      nginx_server_port: 8080
    - role: myapp
      myapp_version: "1.2.3"
      myapp_action: deploy

  post_tasks:
    - name: Verify deployment
      uri:
        url: "http://{{ inventory_hostname }}:8080/health"
        status_code: 200
      register: health
      until: health.status == 200
      retries: 5
      delay: 5
```

### 执行

```bash
# 执行整个 Playbook
ansible-playbook -i inventory/production/hosts playbooks/web.yml

# 只执行特定 Role
ansible-playbook -i inventory/production/hosts playbooks/web.yml --tags myapp

# 跳过特定 Role
ansible-playbook -i inventory/production/hosts playbooks/web.yml --skip-tags nginx

# 检查模式（不实际执行）
ansible-playbook -i inventory/production/hosts playbooks/web.yml --check

# 增量执行（diff）
ansible-playbook -i inventory/production/hosts playbooks/web.yml --diff
```

## 最佳实践

### 1. 变量优先级

```
Role 默认变量（defaults/main.yml）  ← 最低优先级
  ↓
Inventory 变量（hosts/group_vars）
  ↓
Playbook 变量
  ↓
命令行变量（-e var=value）  ← 最高优先级
```

### 2. 敏感变量处理

```yaml
# 错误：敏感信息写在 defaults 中
myapp_db_password: "secret123"

# 正确：使用 vault 加密
# ansible-vault encrypt_string --stdin-name 'myapp_db_password'
myapp_db_password: !vault |
  $ANSIBLE_VAULT;1.1;AES256
  613862613861386132386138...

# 正确：使用环境变量
myapp_db_password: "{{ lookup('env', 'DB_PASSWORD') }}"
```

### 3. 错误处理

```yaml
- name: Backup database
  block:
    - name: Create backup directory
      file:
        path: /backup
        state: directory

    - name: Run backup
      command: /usr/local/bin/backup.sh
      register: backup_result

    - name: Verify backup
      assert:
        that:
          - backup_result.rc == 0
          - "'backup_complete' in backup_result.stdout"
  rescue:
    - name: Notify backup failure
      slack:
        token: "{{ slack_token }}"
        msg: "Backup failed on {{ inventory_hostname }}"
      fail:
        msg: "Backup failed"

  always:
    - name: Cleanup temp files
      file:
        path: /tmp/backup_*
        state: absent
```

### 4. 测试 Role

```yaml title="molecule/default/converge.yml"
---
- name: Converge
  hosts: all
  become: true
  roles:
    - role: myapp
      myapp_version: "1.0.0"
```

```bash
# 使用 Molecule 测试 Role
molecule create
molecule converge
molecule verify
molecule destroy
```

## 总结

Ansible Roles 的核心要点：

1. **结构化组织**：tasks、handlers、templates、files、vars 各司其职
2. **变量层级**：defaults 最低，命令行最高
3. **依赖管理**：meta/main.yml 声明 Role 依赖
4. **Galaxy 市场**：社区贡献的成熟 Roles，开箱即用
5. **requirements.yml**：统一管理 Role 和 Collection 依赖
6. **最佳实践**：敏感变量用 vault、错误处理用 block/ rescue/always

善用 Roles 和 Galaxy，可以让你的自动化脚本从「一次性脚本」变成「可复用的资产」。
