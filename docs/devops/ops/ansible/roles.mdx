# Ansible Roles 角色化组织

当你写了一个 500 行的 Playbook，每一个任务都堆在一起。问题是：**这个 Playbook 能给其他项目用吗？**

答案是：很难。因为所有东西都混在一起——变量、任务、处理器、模板、文件。

Roles 就是来解决这个问题的：**把 Playbook 打散成可复用的组件，每个 Role 专注做一件事**。

## Role 是什么

Role 是一种组织 Playbook 的方式，它将：

- **变量** 放在 `defaults/` 和 `vars/`
- **任务** 放在 `tasks/`
- **处理器** 放在 `handlers/`
- **模板** 放在 `templates/`
- **文件** 放在 `files/`
- **元数据** 放在 `meta/`

每个 Role 独立存在，可以被多个 Playbook 引用。

## Role 目录结构

```
roles/
└── nginx/
    ├── defaults/              # 默认变量（优先级最低）
    │   └── main.yml
    ├── vars/                  # 角色变量（优先级更高）
    │   └── main.yml
    ├── tasks/                 # 任务列表
    │   ├── main.yml
    │   └── install.yml
    ├── handlers/              # 处理器
    │   └── main.yml
    ├── templates/             # Jinja2 模板文件
    │   ├── nginx.conf.j2
    │   └── default.conf.j2
    ├── files/                # 静态文件
    │   └── mime.types
    ├── meta/                  # 角色元数据（依赖）
    │   └── main.yml
    └── README.md              # 角色文档
```

### 各目录说明

| 目录 | 用途 | 说明 |
|---|---|---|
| `defaults/main.yml` | 默认变量 | 可被 Playbook 中的变量覆盖 |
| `vars/main.yml` | 角色变量 | 通常不应该被覆盖 |
| `tasks/main.yml` | 主任务文件 | Role 执行的入口 |
| `handlers/main.yml` | 处理器 | 被 tasks 中的 notify 调用 |
| `templates/` | 模板文件 | Jinja2 模板，支持变量 |
| `files/` | 静态文件 | 不需要模板处理的纯文件 |
| `meta/main.yml` | 依赖关系 | 声明 Role 依赖 |

## 创建 Role

### 使用 ansible-galaxy 初始化

```bash
# 创建 Role 目录结构
ansible-galaxy role init nginx

# 指定路径
ansible-galaxy role init nginx --init-path ./roles/
```

生成的文件结构：

```
roles/nginx/
├── defaults/
│   └── main.yml
├── handlers/
│   └── main.yml
├── meta/
│   └── main.yml
├── README.md
├── tasks/
│   └── main.yml
├── tests/
│   ├── inventory
│   └── test.yml
└── vars/
    └── main.yml
```

### 手动创建

如果不想用 `ansible-galaxy`，可以手动创建：

```bash
mkdir -p roles/nginx/{defaults,handlers,meta,tasks,templates,files,vars}
```

## Role 示例：Nginx

### 定义默认变量

```yaml title="roles/nginx/defaults/main.yml"
---
# 默认变量（可被覆盖）
nginx_version: "1.24.0"
nginx_port: 80
nginx_worker_processes: auto
nginx_worker_connections: 1024
nginx_server_name: localhost
nginx_docroot: /var/www/html
nginx_enable_status: false
nginx_remove_default_site: true
```

### 定义角色变量

```yaml title="roles/nginx/vars/main.yml"
---
# 角色内部使用的变量（通常不应该被覆盖）
nginx_packages:
  - nginx
nginx_service_name: nginx
nginx_config_owner: root
nginx_config_group: root
nginx_config_mode: "0644"
```

### 编写任务

```yaml title="roles/nginx/tasks/main.yml"
---
# tasks/main.yml - Role 入口
- name: Include OS-specific variables
  ansible.builtin.include_vars: "{{ item }}"
  loop:
    - "{{ ansible_os_family }}.yml"
  when:
    - ansible_os_family in ['Debian', 'RedHat']

- name: Include install.yml
  ansible.builtin.import_tasks: install.yml
  when: nginx_install_mode == "package"

- name: Include configure.yml
  ansible.builtin.import_tasks: configure.yml

- name: Include manage.yml
  ansible.builtin.import_tasks: manage.yml
```

```yaml title="roles/nginx/tasks/install.yml"
---
# 安装 Nginx
- name: Install Nginx
  ansible.builtin.apt:
    name: "{{ nginx_packages }}"
    state: present
    update_cache: yes
  notify: Start Nginx
```

```yaml title="roles/nginx/tasks/configure.yml"
---
# 配置 Nginx
- name: Copy Nginx configuration
  ansible.builtin.template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
    owner: "{{ nginx_config_owner }}"
    group: "{{ nginx_config_group }}"
    mode: "{{ nginx_config_mode }}"
    validate: nginx -t -c %s
  notify: Reload Nginx

- name: Create site configuration
  ansible.builtin.template:
    src: site.conf.j2
    dest: "/etc/nginx/sites-available/{{ nginx_site_name }}"
    mode: "0644"
  notify: Reload Nginx

- name: Enable site
  ansible.builtin.file:
    src: "/etc/nginx/sites-available/{{ nginx_site_name }}"
    dest: "/etc/nginx/sites-enabled/{{ nginx_site_name }}"
    state: link
  notify: Reload Nginx
```

```yaml title="roles/nginx/tasks/manage.yml"
---
# 管理 Nginx 服务
- name: Ensure Nginx is running
  ansible.builtin.service:
    name: "{{ nginx_service_name }}"
    state: started
    enabled: yes
```

### 编写处理器

```yaml title="roles/nginx/handlers/main.yml"
---
- name: Start Nginx
  ansible.builtin.service:
    name: "{{ nginx_service_name }}"
    state: started

- name: Reload Nginx
  ansible.builtin.service:
    name: "{{ nginx_service_name }}"
    state: reloaded

- name: Restart Nginx
  ansible.builtin.service:
    name: "{{ nginx_service_name }}"
    state: restarted
```

### 编写模板

```txt title="roles/nginx/templates/nginx.conf.j2"
user www-data;
worker_processes {{ nginx_worker_processes }};
pid /run/nginx.pid;
error_log /var/log/nginx/error.log;

events {
    worker_connections {{ nginx_worker_connections }};
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    access_log /var/log/nginx/access.log;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    include /etc/nginx/conf.d/*.conf;
    include /etc/nginx/sites-enabled/*;
}
```

```txt title="roles/nginx/templates/site.conf.j2"
server {
    listen {{ nginx_port }};
    server_name {{ nginx_server_name }};

    root {{ nginx_docroot }};
    index index.html index.htm;

    location / {
        try_files $uri $uri/ =404;
    }

    {% if nginx_enable_status %}
    location /nginx_status {
        stub_status on;
        access_log off;
        allow 127.0.0.1;
        deny all;
    }
    {% endif %}
}
```

### 定义依赖

```yaml title="roles/nginx/meta/main.yml"
---
dependencies:
  - role: common
    tags: ["base"]
  - role: firewall
    when: nginx_firewall_enabled | default(false)
```

## 使用 Role

### 在 Playbook 中引用

```yaml title="site.yml"
---
- name: Deploy web servers
  hosts: webservers
  become: yes

  roles:
    - role: nginx
      vars:
        nginx_version: "1.24.0"
        nginx_port: 8080
    - role: app
      vars:
        app_port: 8080
        app_env: production
```

### Role 变量覆盖优先级

```
命令行 -e                    (最高)
Play vars/vars_files
Role vars/
set_fact / registered vars
Playbook inventory vars
Host facts
Role defaults                 (最低)
```

因此 `defaults/main.yml` 中的变量可以被任何上层变量覆盖。

### import_tasks vs include_tasks

```yaml
# import_tasks：在解析 Playbook 时静态加载，适合不会变化的公共任务
- name: Include common tasks
  ansible.builtin.import_tasks: tasks/common.yml

# include_tasks：在执行到该任务时动态加载，适合条件加载的场景
- name: Include OS-specific tasks
  ansible.builtin.include_tasks: "{{ ansible_os_family | lower }}.yml"
  when: ansible_os_family == "RedHat"
```

:::tip 何时使用哪个
| 场景 | 推荐 |
|---|---|
| 条件任务 | `include_tasks` |
| 循环加载 | `include_tasks` |
| 公共任务 | `import_tasks` |
| Handler | `import_tasks` |
| 静态检查 | `import_tasks` |
:::

## Ansible Galaxy

### 搜索社区 Role

```bash
# 搜索 Role
ansible-galaxy search nginx

# 查看 Role 详情
ansible-galaxy role info username.rolename

# 列出已安装的 Role
ansible-galaxy role list
```

### 安装社区 Role

```bash
# 安装单个 Role
ansible-galaxy role install geerlingguy.nginx

# 从文件安装多个 Role
ansible-galaxy role install -r requirements.yml

# requirements.yml 示例
roles:
  - name: geerlingguy.nginx
    version: "4.5.0"
  - name: geerlingguy.redis
    version: "3.3.0"
  - name: geerlingguy.mysql
    version: "3.8.0"
```

## Role 依赖管理

### meta/main.yml 语法

```yaml title="roles/nodejs/meta/main.yml"
---
galaxy_info:
  author: yourname
  description: Node.js installation role
  company: yourcompany
  license: MIT
  min_ansible_version: "2.9"
  platforms:
    - name: EL
      versions:
        - 7
        - 8
    - name: Ubuntu
      versions:
        - bionic
        - focal
  galaxy_tags:
    - nodejs
    - javascript
    - runtime

dependencies:
  - role: common
  - role: users
    vars:
      users_to_create:
        - nodejs
```

### 依赖执行顺序

当 Playbook 引用一个有依赖的 Role 时：

```yaml
roles:
  - role: myapp
```

执行顺序：

1. 依赖 Role 1
2. 依赖 Role 2
3. ...其他依赖
4. myapp Role

### 主 Playbook 示例

```yaml title="playbooks/site.yml"
---
- name: Setup base infrastructure
  hosts: all
  become: yes
  roles:
    - common

- name: Setup webservers
  hosts: webservers
  become: yes
  roles:
    - nginx
    - firewall
    - myapp

```

## 总结

Role 是 Ansible 代码组织的最佳实践：

1. **标准化结构**：每个 Role 都有固定的目录结构
2. **可复用性**：Role 可以被多个 Playbook 引用
3. **可组合性**：通过依赖关系组合多个 Role
4. **社区共享**：通过 Ansible Galaxy 分享和使用社区 Role

掌握 Role 后，你的 Playbook 将从"一次性脚本"变成"可维护、可复用、可分享"的工程化代码。
