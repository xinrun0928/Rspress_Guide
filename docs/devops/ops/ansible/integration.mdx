# Ansible 集成实战

学会了 Playbook 和 Role，你已经能完成大部分自动化任务。但现实环境中，**容器化和 Kubernetes 才是主角**。

这一节，我们看看 Ansible 如何与 Docker 和 Kubernetes 无缝集成。

## Ansible 与 Docker

### Docker 模块概述

Ansible 提供了完整的 Docker 模块生态：

| 模块 | 用途 |
|---|---|
| `docker_container` | 管理容器（创建、启动、停止、删除） |
| `docker_image` | 管理镜像（构建、拉取、推送） |
| `docker_image_info` | 获取镜像信息 |
| `docker_network` | 管理网络 |
| `docker_volume` | 管理存储卷 |
| `docker_stack` | 管理 Docker Stack（Swarm） |
| `docker_prune` | 清理未使用的资源 |

### 安装 Docker SDK

部分模块需要 Python Docker SDK：

```bash
pip install docker
```

### 镜像管理

```yaml title="docker-image.yml"
---
- name: Build and push Docker image
  hosts: build-server
  become: yes
  tasks:
    - name: Build Docker image
      ansible.builtin.shell: |
        docker build -t myapp:{{ app_version }} ./app
      args:
        chdir: /opt/build

    - name: Tag image for registry
      community.docker.docker_image:
        name: myapp
        tag: "{{ app_version }}"
        repository: registry.example.com/myapp
        tag_latest: yes
        push: yes
        force_tag: yes
      register: image_build

    - name: Display image info
      ansible.builtin.debug:
        msg: "Image built: {{ image_build.image.tag }}"
```

### 容器管理

```yaml title="docker-container.yml"
---
- name: Deploy application containers
  hosts: docker-hosts
  become: yes
  tasks:
    - name: Pull latest image
      community.docker.docker_image:
        name: registry.example.com/myapp
        tag: "{{ app_version }}"
        source: pull
        force_source: yes

    - name: Create and start container
      community.docker.docker_container:
        name: myapp-{{ item.instance_id }}
        image: registry.example.com/myapp:{{ app_version }}
        state: started
        restart_policy: always
        ports:
          - "{{ item.port }}:8080"
        env:
          APP_ENV: "{{ app_env }}"
          DB_HOST: "{{ db_host }}"
          REDIS_HOST: "{{ redis_host }}"
        volumes:
          - myapp-data:/data
          - /etc/localtime:/etc/localtime:ro
        networks:
          - name: frontend
          - name: backend
        cleanup: yes
        containers_to_keep: 2
      loop: "{{ app_instances }}"
      vars:
        app_instances:
          - { instance_id: 1, port: 8080 }
          - { instance_id: 2, port: 8081 }
          - { instance_id: 3, port: 8082 }

    - name: Create network
      community.docker.docker_network:
        name: frontend
        driver: bridge
        ipam_config:
          - subnet: 172.20.0.0/16

    - name: Create volume
      community.docker.docker_volume:
        name: myapp-data
        driver: local
```

### Docker Compose（完整应用栈）

```yaml title="docker-compose-app.yml"
---
- name: Deploy Docker Compose application
  hosts: docker-hosts
  become: yes
  vars:
    compose_version: "3.8"
    app_version: "2.1.0"

  tasks:
    - name: Create app directory
      ansible.builtin.file:
        path: /opt/myapp
        state: directory
        owner: deploy
        group: deploy
        mode: "0755"

    - name: Deploy Docker Compose file
      community.docker.docker_compose_v2:
        project_src: /opt/myapp
        definition:
          version: "{{ compose_version }}"
          services:
            app:
              image: registry.example.com/myapp:{{ app_version }}
              ports:
                - "8080:8080"
              environment:
                - DB_HOST=postgres
                - REDIS_HOST=redis
              depends_on:
                - postgres
                - redis
              restart: always
              networks:
                - backend

            postgres:
              image: postgres:15-alpine
              environment:
                POSTGRES_DB: myapp
                POSTGRES_USER: myapp
                POSTGRES_PASSWORD: "{{ postgres_password }}"
              volumes:
                - postgres-data:/var/lib/postgresql/data
              restart: always
              networks:
                - backend

            redis:
              image: redis:7-alpine
              command: redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru
              restart: always
              networks:
                - backend

            nginx:
              image: nginx:alpine
              ports:
                - "80:80"
                - "443:443"
              volumes:
                - ./nginx.conf:/etc/nginx/nginx.conf:ro
              depends_on:
                - app
              restart: always
              networks:
                - frontend
                - backend

          networks:
            frontend:
              driver: bridge
            backend:
              driver: bridge

          volumes:
            postgres-data:
      register: compose_result

    - name: Wait for services to be ready
      ansible.builtin.wait_for:
        port: 80
        delay: 10
        timeout: 60
      register: port_check
      ignore_errors: yes

    - name: Verify deployment
      ansible.builtin.debug:
        msg: "Docker Compose deployment {{ 'succeeded' if compose_result is succeeded else 'needs attention' }}"
```

## Ansible 与 Kubernetes

### Kubernetes 模块概述

Ansible 提供了 `k8s` 模块系列：

| 模块 | 用途 |
|---|---|
| `k8s` | 创建/更新/删除 K8s 资源 |
| `k8s_info` | 获取 K8s 资源信息 |
| `k8s_auth` | K8s 认证配置 |
| `k8s_cluster_info` | 获取集群信息 |
| `k8s_logs` | 获取 Pod 日志 |
| `k8s_exec` | 在 Pod 中执行命令 |
| `k8s_facts` | 获取资源详情（已废弃，推荐 k8s_info） |

### 安装依赖

```bash
pip install kubernetes
```

### 基本用法

```yaml title="k8s-basic.yml"
---
- name: Deploy to Kubernetes
  hosts: localhost
  connection: local
  gather_facts: no
  tasks:
    - name: Get Kubernetes cluster info
      k8s_cluster_info:
        kubeconfig: /path/to/kubeconfig
      register: cluster_info

    - name: Create namespace
      k8s:
        kubeconfig: /path/to/kubeconfig
        definition:
          apiVersion: v1
          kind: Namespace
          metadata:
            name: production

    - name: Create Deployment
      k8s:
        kubeconfig: /path/to/kubeconfig
        namespace: production
        definition:
          apiVersion: apps/v1
          kind: Deployment
          metadata:
            name: myapp
            labels:
              app: myapp
              version: "{{ app_version }}"
          spec:
            replicas: 3
            selector:
              matchLabels:
                app: myapp
            template:
              metadata:
                labels:
                  app: myapp
                  version: "{{ app_version }}"
              spec:
                containers:
                  - name: myapp
                    image: registry.example.com/myapp:{{ app_version }}
                    ports:
                      - containerPort: 8080
                    resources:
                      requests:
                        memory: "128Mi"
                        cpu: "100m"
                      limits:
                        memory: "256Mi"
                        cpu: "500m"
                    livenessProbe:
                      httpGet:
                        path: /health
                        port: 8080
                      initialDelaySeconds: 10
                      periodSeconds: 5
                    readinessProbe:
                      httpGet:
                        path: /ready
                        port: 8080
                      initialDelaySeconds: 5
                      periodSeconds: 3

    - name: Create Service
      k8s:
        kubeconfig: /path/to/kubeconfig
        namespace: production
        definition:
          apiVersion: v1
          kind: Service
          metadata:
            name: myapp
          spec:
            type: ClusterIP
            ports:
              - port: 80
                targetPort: 8080
            selector:
              app: myapp

    - name: Create Ingress
      k8s:
        kubeconfig: /path/to/kubeconfig
        namespace: production
        definition:
          apiVersion: networking.k8s.io/v1
          kind: Ingress
          metadata:
            name: myapp
            annotations:
              nginx.ingress.kubernetes.io/rewrite-target: /
          spec:
            ingressClassName: nginx
            rules:
              - host: myapp.example.com
                http:
                  paths:
                    - path: /
                      pathType: Prefix
                      backend:
                        service:
                          name: myapp
                          port:
                            number: 80
```

### 使用模板

```yaml title="templates/deployment.yaml.j2"
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ app_name }}
  namespace: {{ app_namespace }}
  labels:
    app: {{ app_name }}
    version: {{ app_version }}
spec:
  replicas: {{ app_replicas | default(3) }}
  selector:
    matchLabels:
      app: {{ app_name }}
  template:
    metadata:
      labels:
        app: {{ app_name }}
        version: {{ app_version }}
    spec:
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
            - weight: 100
              podAffinityTerm:
                labelSelector:
                  matchExpressions:
                    - key: app
                      operator: In
                      values:
                        - {{ app_name }}
                topologyKey: kubernetes.io/hostname
      containers:
        - name: {{ app_name }}
          image: {{ app_registry }}/{{ app_name }}:{{ app_version }}
          ports:
            - containerPort: {{ app_port }}
          resources:
            requests:
              memory: "{{ app_memory_request }}"
              cpu: "{{ app_cpu_request }}"
            limits:
              memory: "{{ app_memory_limit }}"
              cpu: "{{ app_cpu_limit }}"
          env:
            {% for env_var in app_env_vars %}
            - name: {{ env_var.name }}
              value: {{ env_var.value }}
            {% endfor %}
```

Playbook 中使用模板：

```yaml title="k8s-template.yml"
---
- name: Deploy application to Kubernetes
  hosts: localhost
  connection: local
  vars:
    app_name: myapp
    app_namespace: production
    app_version: "2.1.0"
    app_replicas: 3
    app_port: 8080
    app_registry: registry.example.com
    app_memory_request: "128Mi"
    app_cpu_request: "100m"
    app_memory_limit: "256Mi"
    app_cpu_limit: "500m"
    app_env_vars:
      - name: DATABASE_URL
        value: "postgresql://db:5432/myapp"
      - name: REDIS_URL
        value: "redis://redis:6379"
      - name: LOG_LEVEL
        value: "info"

  tasks:
    - name: Create deployment from template
      k8s:
        kubeconfig: /path/to/kubeconfig
        definition: "{{ lookup('template', 'deployment.yaml.j2') | from_yaml }}"
        apply: yes
```

### 获取 K8s 资源信息

```yaml
- name: Get deployment info
  k8s_info:
    kubeconfig: /path/to/kubeconfig
    kind: Deployment
    namespace: production
    name: myapp
  register: myapp_deployment

- name: Display deployment info
  ansible.builtin.debug:
    msg: |
      Name: {{ myapp_deployment.resources[0].metadata.name }}
      Replicas: {{ myapp_deployment.resources[0].spec.replicas }}
      Available: {{ myapp_deployment.resources[0].status.availableReplicas }}

- name: Get all pods
  k8s_info:
    kubeconfig: /path/to/kubeconfig
    kind: Pod
    namespace: production
    label_selectors:
      - app=myapp
  register: myapp_pods

- name: Check pod status
  ansible.builtin.debug:
    msg: "Pod {{ item.metadata.name }} is {{ item.status.phase }}"
  loop: "{{ myapp_pods.resources }}"
```

### 在 Pod 中执行命令

```yaml
- name: Execute command in pod

```

## Kubernetes 动态 Inventory

Ansible 可以使用 Kubernetes 作为动态 Inventory，自动发现集群中的所有 Pod、Service 等资源。

### 安装 k8s Inventory 插件

```bash
pip install kubernetes
```

### 配置文件

```ini title="k8s.ini"
# Kubernetes 动态 Inventory 配置
plugin=community.kubernetes.k8s

# 认证方式
kubeconfig=/path/to/kubeconfig

# 或者使用 in-cluster 认证
host=https://kubernetes.default.svc
api_key={{ lookup('env', 'KUBERNETES_SERVICE_HOST') }}
api_key_var=KUBERNETES_SERVICE_HOST
```

```yaml title="ansible.cfg"
[inventory]
enable_plugins = yaml, ini, k8s
```

### Inventory 使用

```bash
# 列出所有 pods
ansible-inventory -i k8s.yml --list

# 按 namespace 过滤
ansible-inventory -i k8s.yml --graph --vars

# 在 playbook 中使用
ansible-playbook -i k8s.yml -m ping all
```

### 自定义 Inventory 行为

```yaml title="k8s-inventory.yml"
plugin: community.kubernetes.k8s

kubeconfig: /path/to/kubeconfig

# 包含哪些资源
include_namespaces:
  - production
  - staging

# 资源类型
kinds:
  - Pod
  - Service
  - Deployment

# 标签过滤
keyed_groups:
  - key: metadata.labels.app
    prefix: app
  - key: metadata.namespace
    prefix: namespace

# 变量
groups:
  production: metadata.namespace == 'production'
```

## 实战案例：应用全流程部署

```yaml title="deploy-app.yml"
---
- name: Build and deploy application
  hosts: localhost
  connection: local
  vars:
    app_name: myapp
    app_version: "{{ lookup('env', 'APP_VERSION') }}"
    registry: registry.example.com
    k8s_namespace: production

  tasks:
    - name: Verify variables
      ansible.builtin.assert:
        that:
          - app_version is defined
          - app_version != ""
        fail_msg: "APP_VERSION must be set"

    - name: Build Docker image
      when: build_image | default(true)
      block:
        - name: Checkout code
          ansible.builtin.git:
            repo: "https://github.com/example/{{ app_name }}.git"
            dest: "/tmp/{{ app_name }}"
            version: "{{ app_version }}"

        - name: Build image
          community.docker.docker_image:
            name: "{{ app_name }}"
            tag: "{{ app_version }}"
            build:
              path: "/tmp/{{ app_name }}"
              pull: yes
            force_source: yes

        - name: Tag for registry
          community.docker.docker_image:
            name: "{{ app_name }}"
            tag: "{{ app_version }}"
            repository: "{{ registry }}/{{ app_name }}"
            push: yes
            force_tag: yes

    - name: Deploy to Kubernetes
      block:
        - name: Create namespace
          k8s:
            kubeconfig: "{{ kubeconfig_path }}"
            definition:
              apiVersion: v1
              kind: Namespace
              metadata:
                name: "{{ k8s_namespace }}"

        - name: Apply Kubernetes manifests
          k8s:
            kubeconfig: "{{ kubeconfig_path }}"
            namespace: "{{ k8s_namespace }}"
            definition: "{{ lookup('template', 'k8s/manifests.yml.j2') }}"
            apply: yes
            wait: yes
            wait_timeout: 300

        - name: Wait for rollout
          ansible.builtin.shell: |
            kubectl rollout status deployment/{{ app_name }} -n {{ k8s_namespace }}
          register: rollout
          until: rollout.rc == 0
          retries: 10
          delay: 30

        - name: Verify deployment
          k8s_info:
            kubeconfig: "{{ kubeconfig_path }}"
            kind: Deployment
            namespace: "{{ k8s_namespace }}"
            name: "{{ app_name }}"
          register: deployment

        - name: Report success
          ansible.builtin.debug:
            msg: |
              Successfully deployed {{ app_name }}:{{ app_version }}
              Namespace: {{ k8s_namespace }}
              Replicas: {{ deployment.resources[0].spec.replicas }}
              Image: {{ registry }}/{{ app_name }}:{{ app_version }}

      rescue:
        - name: Get pod logs on failure
          k8s_logs:
            kubeconfig: "{{ kubeconfig_path }}"
            namespace: "{{ k8s_namespace }}"
            name: "{{ app_name }}"
            container: "{{ app_name }}"
            tail_lines: 50
          register: logs
          ignore_errors: yes

        - name: Report failure
          ansible.builtin.fail:
            msg: |
              Deployment failed!
              Pod logs:
              {{ logs.log_lines | join('\n') }}
```

## 总结

Ansible 与容器生态的集成非常完善：

1. **Docker 集成**：`docker_container`、`docker_image`、`docker_compose` 模块让你管理容器生命周期
2. **Kubernetes 集成**：`k8s` 模块系列让你通过声明式方式管理 K8s 资源
3. **动态 Inventory**：K8s 可以作为 Ansible 的 Inventory 源
4. **Helm 支持**：通过 `helm` 模块管理 Helm Release

掌握这些集成技能，你就能用 Ansible 实现**从代码构建到容器部署的全流程自动化**。
