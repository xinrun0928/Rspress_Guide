# 日志收集器选型与实践

日志收集器是日志架构的「搬运工」。它负责从各个角落收集日志，送到中央存储系统。在 Kubernetes 环境中，常见的收集器有 Fluentd、Fluent Bit、Filebeat、Vector 等。

如何选择？各有优劣。

## 收集器对比

| 收集器 | 资源占用 | 插件生态 | Kubernetes 支持 | 适用场景 |
|---|---|---|---|---|
| **Fluentd** | 中等 | 丰富 | 好 | 需要复杂处理的场景 |
| **Fluent Bit** | 极低 | 一般 | 优秀 | Kubernetes 边缘收集 |
| **Filebeat** | 低 | 一般 | 一般 | 简单文件收集 |
| **Vector** | 低 | 丰富 | 好 | 高性能需求 |
| **Logstash** | 高 | 最丰富 | 一般 | 复杂 ETL 场景 |

## Filebeat：轻量收集

### 核心特点

- 资源占用低，适合边缘部署
- 配置简单，上手快
- 与 ELK 栈无缝集成
- 自动发现日志文件

### Kubernetes 部署

```yaml title="filebeat-daemonset.yaml"
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: filebeat
  namespace: kube-logging
  labels:
    app: filebeat
spec:
  selector:
    matchLabels:
      app: filebeat
  template:
    metadata:
      labels:
        app: filebeat
    spec:
      serviceAccount: filebeat
      containers:
      - name: filebeat
        image: docker.elastic.co/beats/filebeat:8.11.0
        args: ["-c", "/etc/filebeat.yml", "-e"]
        securityContext:
          runAsUser: 0
        volumeMounts:
        - name: config
          mountPath: /etc/filebeat.yml
          readOnly: true
          subPath: filebeat.yml
        - name: data
          mountPath: /usr/share/filebeat/data
        - name: varlibdockercontainers
          mountPath: /var/lib/docker/containers
          readOnly: true
        - name: varlog
          mountPath: /var/log
          readOnly: true
      volumes:
      - name: config
        configMap:
          name: filebeat-config
      - name: data
        hostPath:
          path: /var/lib/filebeat
      - name: varlibdockercontainers
        hostPath:
          path: /var/lib/docker/containers
      - name: varlog
        hostPath:
          path: /var/log
```

### 配置文件

```yaml title="filebeat-configmap.yaml"
apiVersion: v1
kind: ConfigMap
metadata:
  name: filebeat-config
  namespace: kube-logging
data:
  filebeat.yml: |
    filebeat.inputs:
    - type: container
      enabled: true
      paths:
        - /var/log/containers/*.log
      processors:
        - add_kubernetes_metadata:
            host: ${NODE_NAME}
            matchers:
              - logs_path:
                  logs_path: "/var/log/containers/"
        - add_host_metadata:
            when.not.contains.tags: forwarded
    
    output.elasticsearch:
      hosts: ["elasticsearch:9200"]
      index: "filebeat-%{[agent.version]}-%{+yyyy.MM.dd}"
    
    setup.template:
      enabled: true
      name: "filebeat"
      pattern: "filebeat-*"
    
    processors:
      - decode_json_fields:
          fields: ["message"]
          target: ""
      - drop_fields:
          fields: ["agent", "ecs", "input", "log"]
```

## Fluent Bit：云原生首选

### 核心特点

- 资源占用极低（5-10MB 内存）
- C 语言实现，性能优秀
- Kubernetes 原生支持
- 与 Fluentd 生态兼容

### 解析插件

```txt title="parsers.conf"
[PARSER]
    Name   docker
    Format json
    Time_Key time
    Time_Format %Y-%m-%dT%H:%M:%S.%L

[PARSER]
    Name   json
    Format json
    Time_Key timestamp
    Time_Format %Y-%m-%dT%H:%M:%S.%NZ

[PARSER]
    Name   syslog
    Format regex
    Regex  ^(?<time>[^ ]* {1,2}[^ ]* [^ ]*) (?<host>[^ ]*) (?<ident>[^ ]*): (?<message>.*)$
    Time_Key time
    Time_Format %b %d %H:%M:%S
```

### 多目标输出

```txt title="fluent-bit-multi-output.conf"
[SERVICE]
    Flush         5
    Daemon        Off
    Log_Level     info
    Parsers_File  parsers.conf

[INPUT]
    Name              tail
    Path              /var/log/containers/*.log
    Parser            docker
    Tag               kube.*
    Refresh_Interval  5
    DB                /run/fluent-bit/flb_kube.db

[FILTER]
    Name                kubernetes
    Match               kube.*
    Kube_URL            https://kubernetes.default.svc:443
    Kube_Tag_Prefix     kube.var.log.containers.

[OUTPUT]
    Name            es
    Match           kube.*
    Host            elasticsearch
    Port            9200
    Logstash_Format On
    Retry_Limit     False

[OUTPUT]
    Name            prometheus_exporter
    Match           kube.*
    Host            0.0.0.0
    Port            2021
    Metrics         bytes_processed,bytes_sent
```

## Vector：高性能后起之秀

### 核心特点

- Rust 语言实现，性能卓越
- 低资源占用，高吞吐量
- 配置简单（TOML/YAML）
- 可替换 Fluentd/Filebeat

### Kubernetes 部署

```yaml title="vector-daemonset.yaml"
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: vector
  namespace: monitoring
spec:
  selector:
    matchLabels:
      app: vector
  template:
    spec:
      containers:
      - name: vector
        image: timberio/vector:0.34.0-rc1
        args: ["--config-dir", "/etc/vector"]
        env:
        - name: VECTOR_SELF_NODE_NAME
          valueFrom:
            fieldRef:
              fieldPath: spec.nodeName
        ports:
        - name: api
          containerPort: 9001
        volumeMounts:
        - name: config
          mountPath: /etc/vector
        - name: data
          mountPath: /vector-data
        - name: logpodlogs
          mountPath: /var/log/pods
          readOnly: true
        - name: varlibdockercontainers
          mountPath: /var/lib/docker/containers
          readOnly: true
      volumes:
      - name: config
        configMap:
          name: vector-config
      - name: data
        emptyDir: {}
      - name: logpodlogs
        hostPath:
          path: /var/log/pods
      - name: varlibdockercontainers
        hostPath:
          path: /var/lib/docker/containers
```

### Vector 配置

```toml title="vector-config.yaml"
[sources.kubernetes]
type = "kubernetes_logs"
exclude_nodes_glob = "master.*"

[transforms.parse_json]
type = "json_parser"
inputs = ["kubernetes"]
fields = ["message"]

[transforms.add_metadata]
type = "remap"
inputs = ["parse_json"]
source = """
.parsed_timestamp = del(.timestamp)
.timestamp = now()
"""

[sinks.elasticsearch]
type = "elasticsearch"
inputs = ["add_metadata"]
endpoint = "http://elasticsearch:9200"
bulk.index = "vector-logs-%Y.%m.%d"

[sinks.prometheus]
type = "prometheus_exporter"
inputs = ["kubernetes"]
address = "0.0.0.0:9090"
```

## Fluentd：功能全面

### Kubernetes 部署

```yaml title="fluentd-configmap.yaml"
apiVersion: v1
kind: ConfigMap
metadata:
  name: fluentd-config
  namespace: kube-logging
data:
  fluent.conf: |
    # 接收 Fluent Bit 转发的日志
    <source>
      @type forward
      @id input_forward
      port 24224
      bind 0.0.0.0
    </source>

    # 收集系统日志
    <source>
      @type syslog
      @id input_syslog
      port 5140
      bind 0.0.0.0
      protocol_type tcp
    </source>

    # Kubernetes 元数据过滤
    <filter kubernetes.**>
      @type kubernetes_metadata
      @id filter_kubernetes_metadata
    </filter>

    # JSON 解析
    <filter docker.**>
      @type parser
      @id filter_parser_docker
      key_name log
      reserve_data true
      <parse>
        @type json
        time_format %Y-%m-%dT%H:%M:%S.%NZ
      </parse>
    </filter>

    # 添加标签
    <filter kubernetes.**>
      @type record_transformer
      @id filter_add_tag
      <record>
        tag ${tag_suffix_parts[0]}
        hostname ${hostname}
      </record>
    </filter>

    # 输出到 Elasticsearch
    <match kubernetes.**>
      @type elasticsearch
      @id output_elasticsearch
      host elasticsearch.logging.svc.cluster.local
      port 9200
      logstash_format true
      logstash_prefix kubernetes
      <buffer>
        @type file
        path /var/log/fluentd-buffers/kubernetes.buffer
        flush_mode interval
        flush_interval 5s
        retry_type exponential_backoff
      </buffer>
    </match>
```

## 收集器组合使用

### Fluent Bit + Fluentd

最佳组合：Fluent Bit 负责边缘收集，Fluentd 负责聚合处理：

```
Pod → Fluent Bit → Fluentd → Elasticsearch
  (收集)           (处理)
```

```title="fluent-bit-to-fluentd.conf"
[OUTPUT]
    Name            forward
    Match           kube.*
    Host            fluentd.logging.svc.cluster.local
    Port            24224
    Retry_Limit     False
```

### 日志处理能力对比

| 场景 | 推荐方案 |
|---|---|
| 简单文件收集 | Filebeat |
| Kubernetes 日志 | Fluent Bit |
| 需要复杂处理 | Fluent Bit + Fluentd |
| 高性能需求 | Vector |
| ELK 栈集成 | Filebeat + Elasticsearch |

## 最佳实践

### 1. 标签设计

```yaml
# 好的标签设计
labels:
  app: order-service          # 应用名
  version: v1.2.3              # 版本
  environment: production      # 环境
  region: us-east-1           # 区域

# 避免的标签
labels:
  hostname: pod-xxx           # 避免使用主机名
  random_id: abc123           # 避免随机值
```

### 2. 缓冲配置

```yaml
# Fluent Bit 内存缓冲（推荐用于 Kubernetes）
[OUTPUT]
    Name            es
    Match           kube.*
    Host            elasticsearch
    Port            9200
    buffer_type     memory
    buffer_path     /var/log/fluent-bit/buffers
    buffer_queue_limit 100
    buffer_chunk_size 128k

# Fluentd 文件缓冲（推荐用于生产）
<buffer>
  @type file
  path /var/log/fluentd-buffers/output.buffer
  flush_mode interval
  flush_interval 10s
  chunk_limit_size 256M
  queue_limit_length 512
</buffer>
```

### 3. 资源限制

```yaml
resources:
  requests:
    cpu: 100m
    memory: 128Mi
  limits:
    cpu: 500m
    memory: 256Mi
```

### 4. 监控指标

```yaml
# Fluent Bit 暴露 Prometheus 指标
[SERVICE]
    HTTP_Server    On
    HTTP_Listen    0.0.0.0
    HTTP_Port      2020
    Health_Check    On
```

## 总结

日志收集器选型建议：

| 场景 | 推荐 |
|---|---|
| 小规模 / 测试环境 | Filebeat |
| Kubernetes 生产环境 | Fluent Bit |
| 需要复杂处理 | Fluent Bit + Fluentd |
| 高性能 / 低延迟 | Vector |
| ELK 完整生态 | Filebeat + Logstash |

关键配置原则：

1. **边缘轻量化**：收集端资源占用要低
2. **缓冲可靠**：防止数据丢失
3. **标签准确**：方便后续查询
4. **监控完善**：实时了解收集状态
