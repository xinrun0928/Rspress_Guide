# Prometheus 服务发现

想象一下：你的 Kubernetes 集群里有 100 个 Pod，每次扩容或缩容，你都需要手动更新 Prometheus 配置吗？

当然不是。**服务发现（Service Discovery）** 让 Prometheus 自动感知集群的变化。

## 为什么需要服务发现

### 传统方式的痛点

```yaml
# 传统方式：静态配置
scrape_configs:
  - job_name: 'node'
    static_configs:
      - targets: 
        - '192.168.1.10:9100'
        - '192.168.1.11:9100'
        - '192.168.1.12:9100'
```

问题：
- 服务器 IP 变了？手动改
- 新增服务器？手动加
- 服务器下线？手动删
- 20 个集群 × 100 台服务器？你疯了

### 服务发现的优势

```
┌─────────────────────────────────────────────────────────────┐
│                     Service Discovery                        │
│                                                              │
│   Prometheus ← 自动发现 → Kubernetes / Consul / DNS / ...     │
│                                                              │
│   • 新增目标：自动加入监控                                      │
│   • 目标下线：自动移除监控                                      │
│   • 配置变更：自动感知                                         │
└─────────────────────────────────────────────────────────────┘
```

## Kubernetes 服务发现

### 配置方式

```yaml title="prometheus.yml"
scrape_configs:
  - job_name: 'kubernetes-pods'
    kubernetes_sd_configs:
      - role: pod          # 发现 Pod
        namespaces:
          names:
            - production
            - staging
        selectors:
          - role: pod
            label: "app"
```

### 支持的角色

| 角色 | 说明 |
|---|---|
| `pod` | 发现集群中所有 Pod |
| `service` | 发现集群中所有 Service |
| `endpoints` | 发现 Service 关联的 Pod 端点 |
| `node` | 发现集群中所有节点 |
| `ingress` | 发现集群中所有 Ingress |

### Pod 发现示例

```yaml
scrape_configs:
  - job_name: 'kubernetes-pods'
    kubernetes_sd_configs:
      - role: pod
    relabel_configs:
      # 使用 Pod 的命名空间作为 job 标签
      - source_labels: [__meta_kubernetes_pod_namespace]
        target_label: namespace
      
      # 使用 Pod 名称作为 instance 标签
      - source_labels: [__meta_kubernetes_pod_name]
        target_label: pod
      
      # 只监控带有 prometheus.io/scrape="true" 注解的 Pod
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
        action: keep
        regex: true
      
      # 从注解中获取路径
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_path]
        action: replace
        target_label: __metrics_path__
        regex: (.+)
      
      # 从注解中获取端口
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_port]
        action: replace
        regex: (.+)
        target_label: __address__
        replacement: $1
```

### 生成的元标签

Prometheus 从 Kubernetes API 获取数据后，会生成以下元标签：

| 元标签 | 说明 |
|---|---|
| `__meta_kubernetes_namespace` | Pod 所在命名空间 |
| `__meta_kubernetes_pod_name` | Pod 名称 |
| `__meta_kubernetes_pod_ip` | Pod IP 地址 |
| `__meta_kubernetes_pod_label_*` | Pod 上的标签 |
| `__meta_kubernetes_pod_annotation_*` | Pod 上的注解 |
| `__meta_kubernetes_container_name` | 容器名称 |
| `__meta_kubernetes_container_runtime` | 容器运行时 |

### Pod 配置示例

在 Pod 中添加注解：

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
  annotations:
    prometheus.io/scrape: "true"
    prometheus.io/path: "/metrics"
    prometheus.io/port: "8080"
spec:
  containers:
    - name: myapp
      image: myapp:latest
      ports:
        - containerPort: 8080
```

Prometheus 会自动发现这个 Pod 并以 `/metrics:8080` 为目标抓取指标。

### Endpoints 发现

```yaml
scrape_configs:
  - job_name: 'kubernetes-endpoints'
    kubernetes_sd_configs:
      - role: endpoints
    relabel_configs:
      # 只保留带有 app=prometheus 标签的 Endpoints
      - source_labels: [__meta_kubernetes_service_label_app]
        regex: prometheus
        action: keep
      
      # 从 Service 名称生成 job 标签
      - source_labels: [__meta_kubernetes_service_name]
        target_label: job
      
      # 从 Endpoint 地址生成 target
      - source_labels: [__meta_kubernetes_endpoint_address_target_name]
        target_label: pod_name
```

## Consul 服务发现

### 配置 Consul

```yaml title="prometheus.yml"
scrape_configs:
  - job_name: 'consul-services'
    consul_sd_configs:
      - server: 'consul-server:8500'
        datacenter: dc1
        tag_separator: ','
        services:
          - nginx
          - mysql
          - redis
```

### 生成的元标签

| 元标签 | 说明 |
|---|---|
| `__meta_consul_service` | 服务名称 |
| `__meta_consul_tags` | 服务标签（逗号分隔） |
| `__meta_consul_address` | 服务地址 |
| `__meta_consul_port` | 服务端口 |
| `__meta_consul_dc` | 数据中心 |

### Consul 服务注册

服务需要先注册到 Consul：

```json
{
  "Name": "myapp",
  "Address": "192.168.1.100",
  "Port": 8080,
  "Tags": ["production", "v2"],
  "Meta": {
    "version": "2.0.0"
  },
  "Check": {
    "HTTP": "http://192.168.1.100:8080/health",
    "Interval": "10s"
  }
}
```

## 文件发现

### 配置方式

当无法使用自动发现时，可以使用文件发现：

```yaml
scrape_configs:
  - job_name: 'file-sd'
    file_sd_configs:
      - files:
          - /etc/prometheus/targets/*.json
        refresh_interval: 30s
```

### JSON 格式

```json title="/etc/prometheus/targets/webservers.json"
[
  {
    "targets": ["192.168.1.10:9100", "192.168.1.11:9100"],
    "labels": {
      "job": "node",
      "env": "production"
    }
  }
]
```

### YAML 格式

```yaml title="/etc/prometheus/targets/dbservers.yml"
- targets:
    - "192.168.1.20:9100"
    - "192.168.1.21:9100"
  labels:
    job: mysql
    env: production
```

### 自动更新

当文件内容变化时，Prometheus 自动重新加载：

```bash
# 编辑配置文件
vim /etc/prometheus/targets/webservers.json

# Prometheus 自动感知变化（refresh_interval 内）
```

## Relabeling（重标签）

### 为什么需要 Relabeling

服务发现返回的标签往往不够友好，或者需要进一步加工。Relabeling 允许我们对标签进行：

- 选择（keep/drop）
- 修改（replace/labelmap）
- 创建（set）
- 删除（labeldrop）

### Relabeling 流程

```
┌─────────────────────────────────────────────────────────────┐
│                  Relabeling Pipeline                         │
│                                                              │
│  服务发现      Relabel Config 1     Relabel Config 2        │
│  元标签         → 转换处理 1      →  转换处理 2              │
│                                                              │
│  __address__   __address__          192.168.1.10:9100       │
│  __meta_*      __meta_*             (删除)                   │
│                                                      最终标签 │
└─────────────────────────────────────────────────────────────┘
```

### 常见 Relabeling 操作

**1. Keep：只保留匹配的目标**

```yaml
relabel_configs:
  # 只保留带 prod 标签的目标
  - source_labels: [env]
    regex: prod
    action: keep
```

**2. Drop：删除匹配的目标**

```yaml
relabel_configs:
  # 删除测试环境的目标
  - source_labels: [env]
    regex: test
    action: drop
```

**3. Replace：替换标签值**

```yaml
relabel_configs:
  # 从 __address__ 提取 IP 和端口
  - source_labels: [__address__]
    regex: '(.*):(.*)'
    target_label: __address__
    replacement: '${1}:9100'
```

**4. LabelMap：批量添加标签**

```yaml
relabel_configs:
  # 将 __meta_kubernetes_pod_label_app 映射为 app
  - source_labels: [__meta_kubernetes_pod_label_app]
    regex: (.+)
    action: labelmap
    replacement: ${1}
```

**5. HashMod：为目标分组**

```yaml
relabel_configs:
  # 按 instance 哈希分组，确保同一 instance 的指标发到同一 remote_write
  - source_labels: [instance]
    target_label: __tmp_hash
    modulus: 4
    action: hashmod
  - source_labels: [__tmp_hash]
    regex: '0'
    action: keep
```

**6. 设置固定标签**

```yaml
relabel_configs:
  # 添加固定标签
  - target_label: cluster
    replacement: prod-us-east-1
  
  - target_label: datacenter
    replacement: aws
```

### 实际案例：K8s 服务发现完整配置

```yaml
scrape_configs:
  # 监控 Node（节点）
  - job_name: 'kubernetes-nodes'
    kubernetes_sd_configs:
      - role: node
    relabel_configs:
      - action: labelmap
        regex: __meta_kubernetes_node_label_(.+)
      - target_label: __address__
        replacement: kubernetes.default.svc:443
      - source_labels: [__meta_kubernetes_node_name]
        regex: (.+)
        target_label: __metrics_path__
        replacement: /api/v1/nodes/${1}/proxy/metrics

  # 监控 Pod
  - job_name: 'kubernetes-pods'
    kubernetes_sd_configs:
      - role: pod
    relabel_configs:
      # 只监控标记为需要监控的 Pod
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
        action: keep
        regex: true
      
      # 从注解获取路径和端口
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_path]
        action: replace
        target_label: __metrics_path__
        regex: (.+)
      
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_port]
        action: replace
        regex: (.+)
        target_label: __address__
        replacement: $1
      
      # 使用 Pod IP 作为 instance
      - source_labels: [__meta_kubernetes_pod_ip]
        action: replace
        regex: (.+)
        target_label: instance
        replacement: $1
      
      # 保留原始标签
      - source_labels: [__meta_kubernetes_namespace]
        target_label: namespace
      - source_labels: [__meta_kubernetes_pod_name]
        target_label: pod
      
      # 添加集群标签
      - target_label: cluster
        replacement: prod-cluster

  # 监控 Service
  - job_name: 'kubernetes-services'
    kubernetes_sd_configs:
      - role: service
    relabel_configs:
      # 只监控标记为需要监控的 Service
      - source_labels: [__meta_kubernetes_service_annotation_prometheus_io_scrape]
        action: keep
        regex: true
      
      # 使用 Service 名称作为 job
      - source_labels: [__meta_kubernetes_service_name]
        target_label: job
      
      # 添加端口
      - source_labels: [__meta_kubernetes_service_annotation_prometheus_io_port]
        action: replace
        target_label: __param_target_port
        regex: (.+)
```

## HTTP 服务发现

### 配置方式

Prometheus 支持通过 HTTP 接口进行服务发现：

```yaml
scrape_configs:
  - job_name: 'http-sd'
    http_sd_configs:
      - url: http://service-discovery:8080/sd
        refresh_interval: 30s
```

### HTTP SD 响应格式

```json
[
  {
    "targets": ["192.168.1.10:9100", "192.168.1.11:9100"],
    "labels": {
      "job": "node",
      "env": "production"
    }
  }
]
```

### 自定义服务发现

```python
# Python 实现一个简单的 HTTP SD 服务
from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/sd')
def service_discovery():
    targets = []
    
    # 从配置中心、CMDB 等获取目标
    services = get_services_from_config_center()
    
    for service in services:
        targets.append({
            "targets": [f"{service['ip']}:{service['port']}"],
            "labels": {
                "job": service['name'],
                "env": service['env'],
                "region": service['region']
            }
        })
    
    return jsonify(targets)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)
```

## DNS 服务发现

### 配置方式

```yaml
scrape_configs:
  - job_name: 'dns-sd'
    dns_sd_configs:
      - names:
          - 'prometheus.service.consul'
        refresh_interval: 30s
        type: A
        port: 9100
```

### DNS 记录类型

| 类型 | 说明 |
|---|---|
| `A` | IPv4 地址 |
| `AAAA` | IPv6 地址 |
| `SRV` | 服务记录（包含端口） |

## 服务发现最佳实践

### 1. 合理的标签设计

```yaml
# 推荐：使用 relabel 添加有意义的标签
relabel_configs:
  - source_labels: [__meta_kubernetes_pod_label_app]
    target_label: app
  
  - target_label: cluster
    replacement: prod-us-east-1
```

### 2. 过滤不需要的目标

```yaml
# 只监控 production 环境
relabel_configs:
  - source_labels: [namespace]
    regex: production
    action: keep
```

### 3. 统一的服务发现配置

将通用的 relabel 配置抽成模板：

```yaml
# 使用 yaml 重构
scrape_configs:
  - $import: templates/kubernetes-pod.yml
```

### 4. 监控服务发现本身

```bash
# 检查服务发现状态
curl http://prometheus:9090/api/v1/targets

# 查看发现的目标数量
curl http://prometheus:9090/api/v1/label/__name__/values
```

## 总结

服务发现是 Prometheus 实现自动化监控的关键：

1. **Kubernetes SD**：自动发现 Pod、Service、Node 等资源
2. **Consul SD**：发现注册到 Consul 的服务
3. **File SD**：通过文件配置静态目标
4. **HTTP SD**：通过 HTTP 接口自定义发现逻辑
5. **DNS SD**：通过 DNS 记录发现服务
6. **Relabeling**：对发现的标签进行转换和过滤

合理使用服务发现，可以让你的监控系统**无需人工干预**，自动适应基础设施的变化。
