# PromQL 查询语言

PromQL（Prometheus Query Language）是 Prometheus 的核心。它让你能够灵活地查询、聚合、计算时序数据。

想象一下：你想知道过去 5 分钟内，HTTP 500 错误的比例是多少？怎么查？

```txt
sum(rate(http_requests_total{status="500"}[5m])) / sum(rate(http_requests_total[5m]))
```

这就是 PromQL 的威力。

## 基础语法

### 指标查询

```txt
# 查询所有 http_requests_total 指标
http_requests_total

# 带标签过滤
http_requests_total{method="GET", status="200"}

# 正则匹配标签
http_requests_total{method=~"GET|POST"}

# 排除某个标签
http_requests_total{status!="404"}
```

### 时间范围

Prometheus 支持两种向量类型：

| 类型 | 说明 | 语法 |
|---|---|---|
| **Instant Vector** | 当前时刻的数据 | `up` |
| **Range Vector** | 一段时间内的数据 | `up[5m]` |

```txt
# Instant Vector：当前时刻的 CPU 使用率
node_cpu_usage

# Range Vector：过去 5 分钟的 CPU 使用率
node_cpu_usage[5m]

# 支持的时间单位
[5m]    # 5 分钟
[1h]    # 1 小时
[2d]    # 2 天
[1w]    # 1 周
[30s]   # 30 秒
```

## 标签匹配

### 匹配运算符

| 运算符 | 说明 | 示例 |
|---|---|---|
| `=` | 精确匹配 | `method="GET"` |
| `!=` | 不等于 | `method!="GET"` |
| `=~` | 正则匹配 | `method=~"GET\|POST"` |
| `!~` | 正则不匹配 | `method!~"DELETE\|PUT"` |

```txt
# 匹配多个值
http_requests_total{method=~"GET|POST|DELETE"}

# 排除多个值
http_requests_total{method!~"OPTIONS|HEAD"}

# 组合条件
http_requests_total{method="GET", status=~"2.."}
```

## 聚合操作

### 内置聚合函数

| 函数 | 说明 |
|---|---|
| `sum()` | 求和 |
| `avg()` | 平均值 |
| `min()` | 最小值 |
| `max()` | 最大值 |
| `count()` | 计数 |
| `stddev()` | 标准差 |
| `stdvar()` | 方差 |
| `topk()` | 返回最大的 k 个值 |
| `bottomk()` | 返回最小的 k 个值 |

### 聚合语法

```txt
# 语法
<聚合函数>(by|without) (label1, label2, ...) (metric)

# 示例：按 job 聚合，求和
sum by (job) (http_requests_total)

# 示例：排除 instance 标签，求平均
avg without (instance) (node_cpu_usage)

# 示例：统计每个 namespace 的 Pod 数量
count by (namespace) (kube_pod_info)
```

### without vs by

```txt
# 有 100 个指标，按 job 求和，保留所有其他标签
sum by (job) (http_requests_total)
# 结果：{job="nginx"} 1000, {job="mysql"} 500

# 有 100 个指标，按 job 求和，移除 namespace
sum without (namespace) (http_requests_total)
# 结果：{job="nginx", instance="server-01"} 300, ...
```

### topk 和 bottomk

```txt
# 返回 CPU 使用率最高的 5 个实例
topk(5, node_cpu_usage{mode="idle"})

# 返回内存使用率最低的 3 个实例
bottomk(3, (node_memory_Used_bytes / node_memory_total_bytes * 100))
```

## 计算函数

### rate 和 increase

```txt
# rate：计算每秒平均增长率
rate(http_requests_total[5m])

# increase：计算时间范围内的总增量
increase(http_requests_total[5m])

# 使用场景
# rate：用于计算速率、告警
# increase：用于计算增量、展示
```

**为什么用 `rate` 而不是直接除？**

```txt
# 错误：直接除只考虑了两次采样之间的增量
http_requests_total / time

# 正确：rate 考虑了整个时间范围内的平均速率
rate(http_requests_total[5m])
```

### irate 和 increase

```txt
# irate：基于最后两个点计算瞬时速率
irate(http_requests_total[5m])

# rate：基于整个时间范围计算平均速率
rate(http_requests_total[5m])

# 选择建议：
# - 绘图用 rate（平滑）
# - 告警用 irate（敏感）
```

### histogram_quantile

计算 Histogram 的百分位数：

```txt
# 计算 P95 延迟
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))

# 计算 P99 延迟
histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m]))

# 计算 P50（中位数）
histogram_quantile(0.50, rate(http_request_duration_seconds_bucket[5m]))
```

:::warning 使用注意
`histogram_quantile` 需要配合 `rate()` 使用，否则计算结果没有意义：

```txt
# 错误：没有 rate
histogram_quantile(0.95, http_request_duration_seconds_bucket)

# 正确：有 rate
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))
```
:::

### 常用计算函数

| 函数 | 说明 | 示例 |
|---|---|---|
| `rate()` | 平均每秒增长率 | `rate(http_requests_total[5m])` |
| `increase()` | 时间范围内的增量 | `increase(http_requests_total[5m])` |
| `irate()` | 瞬时增长率 | `irate(http_requests_total[5m])` |
| `histogram_quantile()` | 计算百分位数 | `histogram_quantile(0.95, ...)` |
| `abs()` | 绝对值 | `abs(-5) = 5` |
| `ceil()` | 向上取整 | `ceil(4.2) = 5` |
| `floor()` | 向下取整 | `floor(4.8) = 4` |
| `round()` | 四舍五入 | `round(4.5) = 5` |
| `clamp_max()` | 限制最大值 | `clamp_max(value, 100)` |
| `clamp_min()` | 限制最小值 | `clamp_min(value, 0)` |

## 时间函数

### offset

```txt
# 当前值
http_requests_total

# 5 分钟前的值
http_requests_total offset 5m

# 与当前值比较
http_requests_total - http_requests_total offset 5m

# 计算增长率（5分钟前到现在）
increase(http_requests_total[5m]) / (5 * 60)
```

### 时间位移

```txt
# 过去 24 小时的同时间段
sum(increase(http_requests_total[1h] offset 24h))

# 与上周同期对比
sum by (job) (rate(http_requests_total[5m])) 
/ 
sum by (job) (rate(http_requests_total[5m] offset 1w))
```

## 运算符

### 算术运算符

```txt
# 加减乘除
node_memory_total_bytes - node_memory_available_bytes

(node_memory_used_bytes / node_memory_total_bytes) * 100

# 计算使用率
rate(http_requests_total{status="5xx"}[5m]) 
/ 
rate(http_requests_total[5m]) 
* 100
```

### 比较运算符

```txt
# 过滤：只保留 CPU 使用率大于 80% 的实例
node_cpu_usage > 80

# 与或非组合
(node_cpu_usage > 80) and (node_cpu_usage < 95)

(node_cpu_usage > 80) or (node_memory_usage > 90)

# bool 修饰符：返回 0 或 1
node_cpu_usage > bool 80
```

### 逻辑运算符

```txt
# and：交集
http_requests_total{status="500"} and http_requests_total{method="GET"}

# or：并集
http_requests_total{method="GET"} or http_requests_total{method="POST"}

# unless：差集
http_requests_total and on(instance) node_memory_usage{type="free"}
```

## 组合查询

### 多个指标组合

```txt
# 展示 CPU、内存、磁盘使用率
{instance=~".*:9100"}
  and on(instance) 
  (node_cpu_usage > 0)
```

### 多行查询

Grafana 支持多行 PromQL：

```txt
# Panel 1: 请求速率
sum by (job) (rate(http_requests_total[5m]))

# Panel 2: 错误率
sum by (job) (rate(http_requests_total{status=~"5.."}[5m])) 
/ 
sum by (job) (rate(http_requests_total[5m])) * 100
```

## 实际查询示例

### 1. HTTP 请求统计

```txt
# 总请求数
sum(increase(http_requests_total[1h]))

# 每秒请求数
sum(rate(http_requests_total[5m]))

# 按状态码分布
sum by (status) (rate(http_requests_total[5m]))

# 按路径统计 Top 10
topk(10, sum by (path) (rate(http_requests_total[5m])))
```

### 2. 性能监控

```txt
# CPU 使用率
100 - (avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# 内存使用率
(1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100

# 磁盘使用率
100 - (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"} * 100)

# 网络 I/O
rate(node_network_receive_bytes_total{device!="lo"}[5m])
rate(node_network_transmit_bytes_total{device!="lo"}[5m])
```

### 3. 业务指标

```txt
# DAU（日活跃用户）
sum by (app) (increase(user_active_total[24h]))

# 订单量
sum(increase(orders_total[1h]))

# 订单金额
sum(increase(order_amount_total[1h]))

# 支付成功率
sum(rate(pay_requests_total{status="success"}[5m])) 
/ 
sum(rate(pay_requests_total[5m])) * 100
```

### 4. Kubernetes 监控

```txt
# Pod CPU 使用率
sum by (namespace, pod) (rate(container_cpu_usage_seconds_total[5m])) 
/ 
sum by (namespace, pod) (container_spec_cpu_quota / container_spec_cpu_period) * 100

# Pod 内存使用率
sum by (namespace, pod) (container_memory_working_set_bytes)
/
sum by (namespace, pod) (container_spec_memory_limit_bytes) * 100

# Pod 数量
count by (namespace) (kube_pod_info)

# Deployment 副本数
kube_deployment_spec_replicas

# HPA 当前副本数
kube_horizontalpodautoscaler_status_desired_replicas
```

## Recording Rules

Recording Rules 预计算常用查询，避免实时计算的性能开销。

### 配置

```yaml title="prometheus.yml"
groups:
  - name: recording_rules
    interval: 30s
    rules:
      # 保存预计算结果
      - record: job:http_requests_per_second:rate5m
        expr: sum by (job) (rate(http_requests_total[5m]))
      
      - record: job:http_request_duration_p95:rate5m
        expr: histogram_quantile(0.95, sum by (job, le) (rate(http_request_duration_seconds_bucket[5m])))
      
      - record: instance:cpu_usage:percent
        expr: 100 - (avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)
```

### 使用

```txt
# 使用 Recording Rule
job:http_requests_per_second:rate5m{job="nginx"}

# 比直接计算快得多
sum by (job) (rate(http_requests_total[5m]))
```

## 常见错误排查

### 1. 查不到数据

```txt
# 检查指标名是否正确
{__name__=~"http.*"}

# 检查标签是否存在
{job="nginx", nonexistent_label="value"}  # 永远查不到
```

### 2. 数据缺失

```txt
# 使用默认值的技巧
sum(http_requests_total) or vector(0)
```

### 3. 计算结果异常

```txt
# 避免除零
value / (total + 0.001)

# 或者使用 clamp
clamp_min(value / (total or vector(1)), 0)
```

## 总结

PromQL 核心要点：

1. **向量类型**：Instant Vector（当前）和 Range Vector（范围）
2. **标签匹配**：精确匹配、正则匹配、组合条件
3. **聚合函数**：sum、avg、min、max、count、topk 等
4. **rate/increase**：计算速率和增量，配合 Range Vector 使用
5. **histogram_quantile**：计算百分位数，必须配合 rate
6. **Recording Rules**：预计算常用查询，提升性能

熟练掌握 PromQL，你就能从 Prometheus 中提取出任何想要的数据。
