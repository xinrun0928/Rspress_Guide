# Prometheus Exporter

Exporter 是 Prometheus 监控体系中的"翻译官"。它将各种系统和应用的指标，转换成 Prometheus 能理解的格式。

## Exporter 是什么

### 工作原理

```
┌─────────────┐                    ┌─────────────┐
│  MySQL      │                    │ Prometheus  │
│  Server     │                    │   Server    │
│             │                    │             │
│  Status     │  Exporter          │  Scrape     │
│  Variables  │ ────────────────►  │  Metrics    │
│  (内部)     │                    │  (标准化)   │
└─────────────┘                    └─────────────┘
```

Exporter 的核心职责：
1. 连接到目标系统（MySQL、Nginx、Kubernetes 等）
2. 收集原始指标
3. 转换成 Prometheus 格式（`metric_name{labels} value`）
4. 通过 HTTP 端点暴露给 Prometheus

### 为什么需要 Exporter

| 方案 | 优点 | 缺点 |
|---|---|---|
| 直接集成 | 无额外依赖 | 侵入代码 |
| Agent | 通用性好 | 需要部署 |
| Exporter | 轻量、无侵入 | 特定系统需要特定 Exporter |

## Node Exporter

Node Exporter 是最常用的 Exporter，用于采集主机硬件和操作系统指标。

### 安装

```bash
# 二进制安装
wget https://github.com/prometheus/node_exporter/releases/download/v1.7.0/node_exporter-1.7.0.linux-amd64.tar.gz
tar xvfz node_exporter-*.tar.gz
cd node_exporter-*
./node_exporter

# Docker 运行
docker run -d \
  --name node-exporter \
  --network host \
  prom/node-exporter:v1.7.0 \
  --collector.textfile.directory=/var/lib/node_exporter/textfile_collector
```

### 默认端口

```
http://localhost:9100/metrics
```

### 常用指标

| 指标 | 说明 |
|---|---|
| `node_cpu_seconds_total` | CPU 时间 |
| `node_memory_MemTotal_bytes` | 总内存 |
| `node_memory_MemAvailable_bytes` | 可用内存 |
| `node_filesystem_size_bytes` | 文件系统大小 |
| `node_filesystem_avail_bytes` | 可用空间 |
| `node_disk_read_bytes_total` | 磁盘读取字节 |
| `node_disk_written_bytes_total` | 磁盘写入字节 |
| `node_network_receive_bytes_total` | 网卡接收字节 |
| `node_network_transmit_bytes_total` | 网卡发送字节 |

### 常用查询

```promql
# CPU 使用率
100 - (avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# 内存使用率
(1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100

# 磁盘使用率
100 - ((node_filesystem_avail_bytes * 100) / node_filesystem_size_bytes)

# 网卡流量
rate(node_network_receive_bytes_total{device!="lo"}[5m])
rate(node_network_transmit_bytes_total{device!="lo"}[5m])
```

### 文本收集器

Node Exporter 支持从文本文件收集自定义指标：

```bash
# 创建文本文件目录
mkdir -p /var/lib/node_exporter/textfile_collector

# 写入指标
cat > /var/lib/node_exporter/textfile_collector/custom_metrics.prom << 'EOF'
# HELP node_custom_info Custom information
# TYPE node_custom_info gauge
node_custom_info{os_version="Ubuntu 22.04", kernel="5.15.0"} 1

# HELP node_last_update Last update timestamp
# TYPE node_last_update gauge
node_last_update 1699999999
EOF
```

## MySQL Exporter

### 安装

```bash
# 安装 mysqld_exporter
wget https://github/prometheus/mysqld_exporter/releases/download/v0.15.0/mysqld_exporter-0.15.0.linux-amd64.tar.gz
tar xvfz mysqld_exporter-*.tar.gz

# 创建数据库用户
mysql -e "CREATE USER 'exporter'@'localhost' IDENTIFIED BY 'password';"
mysql -e "GRANT PROCESS, REPLICATION CLIENT, SELECT ON *.* TO 'exporter'@'localhost';"
mysql -e "FLUSH PRIVILEGES;"

# 运行
./mysqld_exporter --config.my-cnf=./.my.cnf
```

### 配置文件

```ini title=".my.cnf"
[client]
user=exporter
password=password
host=localhost
port=3306
```

### 常用指标

| 指标 | 说明 |
|---|---|
| `mysql_up` | MySQL 是否在线 |
| `mysql_global_status_threads_connected` | 当前连接数 |
| `mysql_global_status_threads_running` | 正在运行的查询数 |
| `mysql_global_status_commands_total` | SQL 命令统计 |
| `mysql_global_variables_long_query_time` | 慢查询阈值 |
| `mysql_global_status_slow_queries` | 慢查询数量 |
| `mysql_global_status_innodb_row_lock_time` | 行锁等待时间 |
| `mysql_slave_status_seconds_behind_master` | 主从延迟 |

### 常用查询

```promql
# MySQL 在线状态
mysql_up{instance="mysql-prod:9104"}

# 连接数使用率
(mysql_global_status_threads_connected / mysql_global_variables_max_connections) * 100

# QPS
sum(rate(mysql_global_status_questions[5m]))

# 慢查询
rate(mysql_global_status_slow_queries[5m])

# 主从延迟
mysql_slave_status_seconds_behind_master{instance="mysql-replica:9104"}
```

### 常见配置问题

```bash
# 问题：Too many connections
# 解决：增加 max_connections
mysql -e "SET GLOBAL max_connections = 1000;"

# 问题：InnoDB buffer pool 命中率低
# 查看：show engine innodb status;
```

## Blackbox Exporter

Blackbox Exporter 用于探测 HTTP、TCP、ICMP 等协议。

### 安装

```bash
docker run -d \
  --name blackbox-exporter \
  -p 9115:9115 \
  prom/blackbox-exporter:v0.24.0 \
  --config.file=/etc/blackbox/blackbox.yml
```

### 配置文件

```yaml title="blackbox.yml"
modules:
  http_2xx:
    prober: http
    http:
      valid_http_versions: ["HTTP/1.1", "HTTP/2"]
      valid_status_codes: [200, 201, 204]
      method: GET
      prefer_server_protobuf: false
  
  http_post_2xx:
    prober: http
    http:
      method: POST
      headers:
        Content-Type: application/json
  
  tcp_connect:
    prober: tcp
  
  icmp:
    prober: icmp

  dns:
    prober: dns
    dns:
      preferred_ip_protocol: ip4
      query_record: A
```

### Prometheus 配置

```yaml
scrape_configs:
  - job_name: 'blackbox-http'
    metrics_path: /probe
    params:
      module: [http_2xx]
    static_configs:
      - targets:
          - https://example.com
          - https://api.example.com/health
    relabel_configs:
      - source_labels: [__address__]
        target_label: __param_target
      - source_labels: [__param_target]
        target_label: instance
      - target_label: __address__
        replacement: blackbox-exporter:9115
  
  - job_name: 'blackbox-tcp'
    metrics_path: /probe
    params:
      module: [tcp_connect]
    static_configs:
      - targets:
          - mysql:3306
          - redis:6379
          - kafka:9092
```

### 常用查询

```promql
# HTTP 可用性
probe_success{job="blackbox-http"}

# HTTP 响应时间
probe_duration_seconds{job="blackbox-http"}

# TLS 证书过期时间
probe_ssl_earliest_cert_expiry{job="blackbox-http"}

# DNS 解析时间
probe_dns_lookup_duration_seconds{job="blackbox-dns"}
```

## Redis Exporter

### 安装

```bash
docker run -d \
  --name redis-exporter \
  -p 9121:9121 \
  oliver006/redis_exporter:v1.57.0 \
  --redis.addr=redis://redis:6379 \
  --redis.password=secret
```

### 常用指标

| 指标 | 说明 |
|---|---|
| `redis_up` | Redis 是否在线 |
| `redis_connected_clients` | 连接数 |
| `redis_memory_used_bytes` | 内存使用 |
| `redis_mem_fragmentation_ratio` | 内存碎片率 |
| `redis_rdb_changes_since_last_save` | RDB 变化数 |
| `redis_commands_total` | 命令统计 |
| `redis_keyspace_hits_total` | Key 命中 |
| `redis_keyspace_misses_total` | Key 未命中 |

### 常用查询

```promql
# Redis 内存使用率
redis_memory_used_bytes / redis_memory_max_bytes * 100

# 命中率
redis_keyspace_hits_total / (redis_keyspace_hits_total + redis_keyspace_misses_total) * 100

# 连接数
redis_connected_clients{instance="redis-prod:9121"}
```

## 自定义 Exporter

### Go 示例

使用 client_golang 编写自定义 Exporter：

```go
package main

import (
    "net/http"
    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promhttp"
)

var (
    httpRequestsTotal = prometheus.NewCounterVec(
        prometheus.CounterOpts{
            Name: "http_requests_total",
            Help: "Total number of HTTP requests",
        },
        []string{"method", "status"},
    )
    
    httpRequestDuration = prometheus.NewHistogramVec(
        prometheus.HistogramOpts{
            Name:    "http_request_duration_seconds",
            Help:    "HTTP request duration in seconds",
            Buckets: prometheus.DefBuckets,
        },
        []string{"method", "path"},
    )
)

func init() {
    prometheus.MustRegister(httpRequestsTotal)
    prometheus.MustRegister(httpRequestDuration)
}

func handler(w http.ResponseWriter, r *http.Request) {
    // 处理请求
    httpRequestsTotal.WithLabelValues(r.Method, "200").Inc()
}

func main() {
    http.HandleFunc("/metrics", promhttp.Handler())
    http.HandleFunc("/", handler)
    http.ListenAndServe(":8080", nil)
}
```

### Python 示例

使用 prometheus_client 库：

```python
from prometheus_client import Counter, Histogram, generate_latest, start_http_server
import random
import time

# 定义指标
REQUEST_COUNT = Counter(
    'http_requests_total',
    'Total HTTP requests',
    ['method', 'status']
)

REQUEST_LATENCY = Histogram(
    'http_request_duration_seconds',
    'HTTP request latency',
    buckets=[0.01, 0.05, 0.1, 0.5, 1.0, 2.0]
)

@app.route('/metrics')
def metrics():
    return generate_latest()

@app.route('/api/users')
def users():
    with REQUEST_LATENCY.time():
        # 处理请求
        time.sleep(random.random())
    
    REQUEST_COUNT.labels(method='GET', status='200').inc()
    return {'users': []}
```

### Java 示例

使用 Micrometer：

```java
import io.micrometer.core.instrument.MeterRegistry;
import io.micrometer.core.instrument.Counter;
import io.micrometer.core.instrument.Timer;

@RestController
public class UserController {
    
    private final Counter requestCounter;
    private final Timer requestTimer;
    
    public UserController(MeterRegistry registry) {
        this.requestCounter = Counter.builder("http_requests_total")
            .tag("method", "GET")
            .register(registry);
        
        this.requestTimer = Timer.builder("http_request_duration_seconds")
            .register(registry);
    }
    
    @GetMapping("/users")
    public List<String> getUsers() {
        return requestTimer.record(() -> {
            requestCounter.increment();
            // 业务逻辑
            return Arrays.asList("user1", "user2");
        });
    }
}
```

## Exporter 最佳实践

### 1. 合理选择端口

| Exporter | 默认端口 |
|---|---|
| Node Exporter | 9100 |
| MySQL Exporter | 9104 |
| Redis Exporter | 9121 |
| Blackbox Exporter | 9115 |
| Postgres Exporter | 9187 |
| Kafka Exporter | 9308 |

### 2. 配置 TLS

```yaml
scrape_configs:
  - job_name: 'node-exporter-tls'
    static_configs:
      - targets: ['node-exporter:9100']
    scheme: https
    tls_config:
      ca_file: /etc/prometheus/ca.crt
      cert_file: /etc/prometheus/prometheus.crt
      key_file: /etc/prometheus/prometheus.key
```

### 3. 认证

```yaml
scrape_configs:
  - job_name: 'node-exporter-auth'
    static_configs:
      - targets: ['node-exporter:9100']
    basic_auth:
      username: prometheus
      password: secret
```

### 4. 避免高基数

```bash
# 错误：高基数字段作为标签
node_filesystem_size_bytes{mount_point="/data/users/123"}  # 每个用户都不同

# 正确：聚合后再分析
sum by (device) (node_filesystem_size_bytes)
```

## 总结

Exporter 是 Prometheus 监控体系的关键组件：

1. **Node Exporter**：主机硬件和系统指标
2. **MySQL Exporter**：数据库性能指标
3. **Blackbox Exporter**：HTTP/TCP/ICMP 探测
4. **Redis Exporter**：缓存服务指标
5. **自定义 Exporter**：业务指标

选择合适的 Exporter，可以让监控系统覆盖所有你需要关注的组件。
