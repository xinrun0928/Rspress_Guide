# K3s：轻量级 Kubernetes

你有没有想过——Kubernetes 功能强大，但部署和管理起来也相当复杂。对于边缘计算、IoT、开发测试环境这些资源受限的场景，有没有更轻量的选择？

K3s 就是来解决这个问题的。K3s 是 Rancher（现 SUSE）出品的轻量级 Kubernetes 发行版，一个不到 100MB 的二进制文件，包含了一切你需要的东西。

## K3s 是什么

K3s 是经过 CNCF 认证的 Kubernetes 发行版，专为以下场景设计：

- **边缘计算**：资源受限的边缘节点
- **IoT**：嵌入式设备和网关
- **开发测试**：本地 K8s 环境
- **CI/CD**：短暂的构建环境
- **ARM 架构**：树莓派、ARM 服务器

### K3s vs 标准 K8s

| 维度 | K3s | 标准 K8s |
|---|---|---|
| **二进制大小** | ~100MB | ~200MB+ |
| **内存占用** | ~512MB | ~2GB+ |
| **安装时间** | < 1 分钟 | 10-30 分钟 |
| **离线安装** | 原生支持 | 需要手动配置 |
| **数据库** | SQLite（默认）/ etcd | etcd |
| **容器运行时** | containerd（内置） | Docker / containerd |
| **Kubernetes 版本** | 同步上游 | 同步上游 |
| **生产可用性** | 测试/边缘/开发 | 大规模生产 |

## 安装 K3s

### 单节点安装

```bash
# 快速安装
curl -sfL https://get.k3s.io | sh -

# 安装完成，查看状态
sudo systemctl status k3s

# 获取 kubectl 配置
cat /etc/rancher/k3s/k3s.yaml
# 或直接使用 k3s kubectl
sudo k3s kubectl get nodes

# 卸载
/usr/local/bin/k3s-uninstall.sh
```

### 高可用安装

K3s 支持使用嵌入式 etcd 实现高可用：

```bash
# 第一个节点（server）
curl -sfL https://get.k3s.io | sh - server \
  --cluster-init \
  --token <CLUSTER_TOKEN> \
  --tls-san <LOAD_BALANCER_IP>

# 其他 server 节点
curl -sfL https://get.k3s.io | sh - server \
  --server https://<LOAD_BALANCER>:6443 \
  --token <CLUSTER_TOKEN> \
  --tls-san <LOAD_BALANCER_IP>

# agent 节点
curl -sfL https://get.k3s.io | K3S_URL=https://<LOAD_BALANCER>:6443 \
  K3S_TOKEN=<CLUSTER_TOKEN> sh -
```

### 离线安装

```bash
# 1. 下载安装包和镜像包
wget https://github.com/k3s-io/k3s/releases/download/v1.28.4+k3s1/k3s
wget https://github.com/k3s-io/k3s/releases/download/v1.28.4+k3s1/k3s-images.txt

# 2. 将镜像导入到私有仓库
# 3. 安装
curl -sfL https://get.k3s.io | INSTALL_K3S_SKIP_START=true sh -
```

### 内置组件

K3s 将以下组件打包到一个二进制文件中：

- **Kubernetes API Server**：`kube-apiserver`
- **Scheduler**：`kube-scheduler`
- **Controller Manager**：`kube-controller-manager`
- **Kubelet**：`kubelet`
- **Kube Proxy**：`kube-proxy`
- **Container Runtime**：内置 `containerd`
- **CNI**：内置 `bridge`、`host-local`、`loopback`、`portmap`、`bandwidth`
- **Metrics Server**：内置 metrics-server
- **CoreDNS**：内置 CoreDNS
- **Traefik**：内置 Traefik（默认 Ingress Controller）
- **Service Load Balancer**：内置 Klipper（简化版 LB）
- **Local Storage Provider**：内置 Local Path Provisioner

## 配置与优化

### 配置文件

```bash title="/etc/rancher/k3s/k3s.conf"
# K3s 配置
K3S_KUBECONFIG_MODE=0644
K3S_TOKEN=<CLUSTER_TOKEN>
K3S_CLUSTER_SECRET=<CLUSTER_SECRET>

# 容器运行时配置
K3S/containerd_template=/var/lib/rancher/k3s/agent/containerd/config.toml.tmpl

# 网络配置
K3S_CLUSTER_CIDR=10.42.0.0/16
K3S_SERVICE_CIDR=10.43.0.0/16
K3S_NODE_CIDR=10.0.0.0/16

# Kubelet 配置
K3S_KUBelet_ARGS="--eviction-hard=memory.available<500Mi"

# 禁用内置组件
K3S_DISABLE=traefik,servicelb,local-storage,metrics-server
```

### Helm 集成

K3s 内置了 Helm Controller，可以通过 CRD 管理 Helm Release：

```bash
# 安装 Traefik（K3s 内置）
# 默认已安装，可通过禁用
kubectl get crd helmcharts.helm.cattle.io

# 使用 HelmChart 部署应用
kubectl apply -f - <<EOF
apiVersion: helm.cattle.io/v1
kind: HelmChart
metadata:
  name: redis
  namespace: kube-system
spec:
  chart: redis
  repo: https://charts.bitnami.com/bitnami
  targetNamespace: default
  version: "18.0.0"
  valuesContent: |
    architecture: standalone
    auth:
      enabled: true
      password: "secret123"
EOF
```

### 资源限制

```bash
# 启动时限制资源
curl -sfL https://get.k3s.io | sh - server \
  --kubelet-arg=eviction-hard=memory.available<500Mi \
  --kubelet-arg=eviction-minimum-reclaim=memory.available=100Mi
```

## 数据持久化

### Local Path Provisioner

K3s 内置了 Local Path Provisioner，为 Pod 提供本地存储：

```bash
# 查看存储类
kubectl get storageclass
# NAME                   PROVISIONER
# local-path             rancher.io/local-path

# 使用本地存储
kubectl apply -f - <<EOF
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: local-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
  storageClassName: local-path
EOF
```

### 使用外部数据库

对于需要更强数据持久性的场景，可以使用外部数据库：

```bash
# 使用外部 PostgreSQL
curl -sfL https://get.k3s.io | sh - server \
  --datastore-endpoint="postgres://user:password@localhost:5432/k3s"
```

## 在边缘计算场景使用 K3s

### 边缘节点架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         云端数据中心                               │
│                                                                   │
│  ┌──────────────────┐                                          │
│  │   Rancher / Fleet │  ← 统一管理多集群                        │
│  └──────────────────┘                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
        ┌──────────┐   ┌──────────┐   ┌──────────┐
        │ 边缘节点1  │   │ 边缘节点2  │   │ 边缘节点3  │
        │ K3s Agent │   │ K3s Agent │   │ K3s Agent │
        │ (ARM/IoT) │   │ (ARM/IoT) │   │ (x86)    │
        └──────────┘   └──────────┘   └──────────┘
```

### Fleet：GitOps 管理 K3s 集群

Fleet 是 Rancher 的 GitOps 工具，可以管理上千个 K3s 集群：

```yaml title="fleet.yaml"
apiVersion: fleet.cattle.io/v1alpha1
kind: GitRepo
metadata:
  name: myapp
  namespace: fleet-default
spec:
  repo: https://github.com/example/myapp-k3s
  branch: main
  targets:
  - name: production
    clusterSelector:
      matchLabels:
        env: production
  - name: edge
    clusterSelector:
      matchLabels:
        env: edge
```

### 离线同步

边缘节点可能无法持续联网，Fleet 支持定时同步：

```yaml
spec:
  repo: https://github.com/example/myapp-k3s
  syncInterval: 1h
  retry:
    maxAttempts: 5
    interval: 1m
```

## 使用场景对比

| 场景 | 推荐方案 | 原因 |
|---|---|---|
| **本地开发** | K3s | 安装简单，资源占用低 |
| **边缘计算** | K3s + Fleet | 轻量，支持离线场景 |
| **CI/CD** | K3s 或 Kind | 短暂环境，快速启动 |
| **小型生产** | K3s HA | 资源受限的生产环境 |
| **大规模生产** | 标准 K8s | 高可用、大规模场景 |

## 常见问题

### Pod 无法调度

```bash
# 查看节点状态
kubectl get nodes
kubectl describe node <node-name>

# 常见原因：
# 1. 资源不足
# 2. 污点不匹配
# 3. 磁盘空间不足
df -h
```

### 镜像拉取失败

```bash
# 配置私有镜像仓库
vim /etc/rancher/k3s/registries.yaml
```

```yaml title="/etc/rancher/k3s/registries.yaml"
mirrors:
  "registry.example.com":
    endpoint:
      - "https://registry.example.com"
  docker.io:
    endpoint:
      - "https://registry.docker.io"
```

### 服务无法访问

```bash
# K3s 内置 ServiceLB，可能与其他 LB 冲突
# 禁用 ServiceLB，使用其他 LB
curl -sfL https://get.k3s.io | sh - server \
  --disable servicelb
```

## 总结

K3s 是轻量级 Kubernetes 的最佳选择：

1. **极简安装**：一条命令，不到 100MB
2. **资源友好**：512MB 内存即可运行
3. **功能完整**：包含 K8s 核心功能和常用插件
4. **离线支持**：专为边缘计算场景设计
5. **Fleet 集成**：GitOps 管理多集群

对于边缘计算、IoT、开发测试等场景，K3s 是标准 K8s 的完美替代。
