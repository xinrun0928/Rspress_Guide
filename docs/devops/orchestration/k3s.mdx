# K3s：轻量级 Kubernetes

「我想用 Kubernetes，但我只有 1GB 内存的树莓派。」

K3s 正是为这种场景而生——它把 Kubernetes 打包成一个不到 100MB 的二进制文件，可以在 512MB 内存的机器上运行。

**K3s 不是 Kubernetes 的简化版，而是 Kubernetes 的轻量化发行版。** 官方将它定位为：「 Anywhere, Anytime, Any device 」。

## K3s 是什么？

K3s 是 Rancher Labs（后被 SUSE 收购）开发的 Kubernetes 发行版，于 2019 年发布。它通过了 CNCF 的一致性认证，功能上与标准 K8s 高度兼容，但资源占用大幅降低。

### 与标准 K8s 的对比

| 维度 | 标准 Kubernetes | K3s |
|---|---|---|
| **二进制大小** | 多个组件，数百 MB | 单二进制，约 70MB |
| **内存占用** | 1-2GB（Master）| < 512MB |
| **磁盘占用** | 数 GB | < 1GB |
| **安装时间** | 10-30 分钟 | < 1 分钟 |
| **默认存储** | 无（需配置）| SQLite（可换 etcd）|
| **容器运行时** | Docker/containerd | containerd（内置）|
| **K8s 兼容性** | - | 100%（CNCF 认证）|
| **边缘计算** | 困难 | 原生支持 |

## K3s 架构

```
┌─────────────────────────────────────────────────────────────┐
│                       K3s Server                            │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │           Control Plane                       │   │   │
│  │  │  API Server │ Scheduler │ Controller │ etcd │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │           Container Runtime                   │   │   │
│  │  │  containerd + CNI + CoreDNS + Traefik       │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  特点：单进程、内置 SQLite、轻量化                           │
└─────────────────────────────────────────────────────────────┘
```

**K3s 内置组件：**

- **containerd**：容器运行时（替换 Docker）
- **CNI**：网络插件
- **CoreDNS**：服务发现
- **Traefik**：Ingress Controller（默认）
- **Klipper LB**：内置负载均衡器（替代 MetalLB）
- **SQLite/etcd**：数据存储（可选）

## 安装与部署

### 快速安装

```bash
# 一键安装（Server 节点）
curl -sfL https://get.k3s.io | sh -

# 查看状态
kubectl get nodes
kubectl get pods -A

# 获取 token（用于添加 Agent）
cat /var/lib/rancher/k3s/server/node-token
```

### 添加 Agent 节点

```bash
# 在 Agent 节点执行
curl -sfL https://get.k3s.io | K3S_URL=https://server-node:6443 \
    K3S_TOKEN=${NODE_TOKEN} sh -
```

### 离线安装

```bash
# 1. 下载安装包
wget https://github.com/k3s-io/k3s/releases/download/v1.28.4+k3s2/k3s
wget https://github.com/k3s-io/k3s/releases/download/v1.28.4+k3s2/k3s-airgap-images-amd64.tar.gz

# 2. 放到指定位置
sudo mkdir -p /var/lib/rancher/k3s/agent/images/
sudo cp k3s-airgap-images-amd64.tar.gz /var/lib/rancher/k3s/agent/images/

# 3. 安装
sudo chmod +x k3s
sudo cp k3s /usr/local/bin/
```

## 配置与调优

### 配置文件

```bash
# /etc/systemd/system/k3s.service.env
K3S_KUBECONFIG_MODE="644"
K3S_TOKEN="your-secure-token"
K3S_CLUSTER_SECRET="your-cluster-secret"
```

### 禁用内置组件

```bash
# 禁用 Traefik（使用自己的 Ingress）
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--disable traefik" sh -

# 禁用 ServicelB（使用云 LB）
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--disable servicelb" sh -

# 禁用 Traefik 和 Metrics Server
curl -sfL https://get.k3s.io | sh -s - server \
  --disable traefik \
  --disable metrics-server
```

### 高可用配置

```bash
# 初始化高可用（3 台 Server）
curl -sfL https://get.k3s.io | K3S_TOKEN="token" sh -
# 在其他 Server 节点
curl -sfL https://get.k3s.io | K3S_TOKEN="token" \
    K3S_URL="https://first-server:6443" \
    K3S_CLUSTER_INIT="true" sh -
```

## K3s 数据存储

### SQLite（默认）

```bash
# 默认使用 SQLite
# 数据文件位置
/var/lib/rancher/k3s/server/db/state.db
```

**适用场景：**
- 单 Server 节点
- 边缘计算节点
- 开发测试环境

### etcd（生产推荐）

```bash
# 启动时指定 etcd
curl -sfL https://get.k3s.io | sh -s - server \
    --cluster-init \
    --cluster-init \
    --cluster-init \
    --cluster-init \
    --cluster-init
```

:::warning SQLite vs etcd
SQLite 适合边缘场景，但不支持高可用。生产环境建议使用 etcd 外部数据库或 K3s 高可用模式。
:::

## K3d：在本地运行 K3s

K3d 是在 Docker 中运行 K3s 的工具，适合本地开发和测试。

### 安装 K3d

```bash
brew install k3d
```

### 创建集群

```bash
# 创建 3 节点的 K3s 集群
k3d cluster create mycluster \
  --servers 1 \
  --agents 3 \
  --port "80:80@loadbalancer" \
  --port "443:443@loadbalancer"

# 查看集群
k3d cluster list
kubectl config use-context k3d-mycluster
```

### 多集群管理

```bash
# 创建开发集群
k3d cluster create dev --agents 1

# 创建测试集群
k3d cluster create test --agents 2

# 创建生产集群（模拟）
k3d cluster create prod --agents 3

# 切换上下文
kubectl config use-context k3d-dev
```

## 边缘计算场景

### 树莓派部署

```bash
# ARM64 架构安装
curl -sfL https://get.k3s.io | K3S_NODE_NAME="pi-1" sh -
```

```yaml
# 在边缘节点部署应用
apiVersion: apps/v1
kind: Deployment
metadata:
  name: iot-gateway
spec:
  replicas: 1
  selector:
    matchLabels:
      app: iot-gateway
  template:
    spec:
      nodeSelector:
        kubernetes.io/arch: arm64
      tolerations:
      - key: "node-role.kubernetes.io/control-plane"
        operator: "Exists"
        effect: "NoSchedule"
      containers:
      - name: gateway
        image: myorg/iot-gateway:latest
        resources:
          limits:
            memory: "256Mi"
            cpu: "500m"
```

### 边缘场景特点

| 场景 | K3s 优势 |
|---|---|
| 低配置设备 | < 512MB 内存运行 |
| 离线部署 | 支持 air-gap 安装 |
| 多节点管理 | K3s GitOps 支持 |
| 网络不稳定 | 内置重连机制 |
| ARM 支持 | 原生支持 ARM64/ARMv7 |

## 与 K8s 的差异

### 移除的组件

K3s 移除了 Kubernetes 中不常用的组件：

- cloud-providers（云厂商集成）
- in-tree storage provisioners
- etcd（默认使用 SQLite）

### K3s 特有的 API

```yaml
# Helium（K3s 存储配置）
apiVersion: k3s.cattle.io/v1
kind: HelmChart
metadata:
  name: traefik
spec:
  chart: https://%{K3S_UPSTREAM_REPO%/charts/traefik-21.2.0.tgz
  set:
  - name: rbac.enabled
    value: "true"
```

### kubectl 兼容性

```bash
# K3s 自带 kubectl
k3s kubectl get nodes

# 或者配置 ~/.kube/config
export KUBECONFIG=/etc/rancher/k3s/k3s.yaml
kubectl get nodes
```

## 性能对比

### 资源占用测试

| 指标 | 标准 K8s | K3s | 节省 |
|---|---|---|---|
| 安装包大小 | ~500MB | ~70MB | 86% |
| 内存占用（Idle）| ~1.5GB | ~300MB | 80% |
| 启动时间 | ~5 分钟 | ~30 秒 | 83% |
| Pod 启动时间 | ~10 秒 | ~5 秒 | 50% |

### 压力测试

```bash
# 创建 100 个 Pod
kubectl create deployment nginx --image=nginx --replicas=100

# 监控启动时间
kubectl rollout status deployment/nginx --timeout=300s
```

## 使用场景总结

### 适合用 K3s 的场景

| 场景 | 说明 |
|---|---|
| 边缘计算 | ARM 设备、物联网网关 |
| 开发测试 | 本地 K8s 体验 |
| CI/CD | 轻量级测试集群 |
| 资源受限环境 | 内存 < 4GB 的机器 |
| 嵌入式系统 | IoT 设备 |
| 学习 K8s | 低成本学习环境 |

### 不适合用 K3s 的场景

| 场景 | 说明 |
|---|---|
| 大规模生产集群 | K3s < 50 节点 |
| 需要 etcd 外部存储 | 使用标准 K8s |
| 需要多云管理 | 使用 Rancher |
| 需要完整 K8s 特性 | 云厂商托管 K8s |

## K3s 生态

### K3s + K3d

```
本地开发：k3d create cluster → 快速迭代
    │
    │ (镜像推送)
    ▼
边缘部署：k3s agent → 轻量运行
```

### K3s + Rancher

```
Rancher 管理平面
    │
    ├── K3s 集群 1（边缘）
    ├── K3s 集群 2（分支办公室）
    └── RKE 集群（数据中心）
```

## 总结

K3s 是 Kubernetes 生态的重要补充：

- **轻量**：70MB 二进制，< 512MB 内存
- **兼容**：CNCF 认证，功能与 K8s 一致
- **简单**：单进程，安装即用
- **边缘**：原生支持 ARM 和边缘场景

<Badge text="选型建议" type="tip" />

- **本地开发**：用 k3d 快速创建集群
- **边缘计算**：用 K3s 部署到 ARM 设备
- **生产小规模**：K3s 高可用模式
- **大规模生产**：标准 Kubernetes 或 RKE2

<Badge text="趋势" type="info" /> 随着边缘计算和 IoT 的发展，K3s 的应用场景会越来越广。
