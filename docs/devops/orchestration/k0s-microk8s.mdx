# k0s 与 MicroK8s：更多轻量级选择

除了 K3s，业界还有两个值得关注的轻量级 Kubernetes 发行版：**k0s** 和 **MicroK8s**。它们各有特色，适合不同的场景。

## k0s vs MicroK8s vs K3s

| 维度 | k0s | MicroK8s | K3s |
|---|---|---|---|
| **开发公司** | Mirantis | Canonical (Ubuntu) | SUSE (Rancher) |
| **发布年份** | 2020 | 2019 | 2019 |
| **二进制大小** | ~150MB | ~400MB | ~70MB |
| **内存占用** | ~500MB | ~1GB | < 512MB |
| **默认存储** | kine (SQLite/etcd) | Dqlite/LXD | SQLite |
| **架构** | 单二进制 | snap/AppImage | 单二进制 |
| **高可用** | 原生支持 | 需配置 | 原生支持 |
| **CNCF 认证** | 是 | 是 | 是 |

## k0s：零依赖的 Kubernetes

### k0s 的理念

k0s 的名字来源于「Kubernetes零依赖」。它通过将 Kubernetes 控制平面打包为单一二进制，同时保持与上游 K8s 的完全兼容。

```
┌─────────────────────────────────────────────────────────────┐
│                         k0s 架构                             │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                      k0s 二进制                       │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │              Control Plane                    │   │   │
│  │  │  API Server │ Scheduler │ Controller │ kine │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │              Container Runtime               │   │   │
│  │  │  containerd │ CNI │ CoreDNS                 │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  特点：零外部依赖、灵活存储后端                               │
└─────────────────────────────────────────────────────────────┘
```

### 安装 k0s

```bash
# 快速安装
curl -sSLf https://get.k0s.sh | sudo sh

# 初始化集群
sudo k0s install controller
sudo k0s start

# 使用 kubectl
sudo k0s kubectl get nodes
```

### 高可用配置

```bash
# 初始化高可用
k0s init --enable-worker \
  --token-file controller-token \
  --controller-count 3
```

### k0sctl 集群管理

```yaml
# k0sctl.yaml
spec:
  hosts:
    - address: 192.168.1.10
      role: controller+worker
    - address: 192.168.1.11
      role: controller+worker
    - address: 192.168.1.12
      role: controller+worker
```

```bash
# 部署集群
k0sctl apply --config k0sctl.yaml
```

## MicroK8s：Ubuntu 原生体验

### MicroK8s 的理念

MicroK8s 由 Canonical 开发，专为 Ubuntu 和物联网设备优化。它提供 snaps 包安装，与 Ubuntu 系统深度集成。

### 安装 MicroK8s

```bash
# Ubuntu 安装
sudo snap install microk8s --classic

# 添加用户到 microk8s 组
sudo usermod -a -G microk8s $USER
newgrp microk8s

# 查看状态
microk8s status
```

### 启用附加组件

```bash
# 启用 DNS
microk8s enable dns

# 启用存储
microk8s enable storage

# 启用 Ingress
microk8s enable ingress

# 启用 GPU 支持
microk8s enable gpu

# 启用 Prometheus
microk8s enable prometheus

# 查看可用插件
microk8s status
```

### kubectl 集成

```bash
# 使用 microk8s kubectl
microk8s kubectl get nodes

# 或创建别名
alias kubectl='microk8s kubectl'
```

### 高可用模式

```bash
# 添加高可用节点
microk8s add-node

# 在新节点执行生成的命令
microk8s join ...
```

## 功能对比详解

### 存储后端

| 发行版 | 支持的存储后端 |
|---|---|
| **k0s** | SQLite (kine), etcd, MySQL, PostgreSQL |
| **MicroK8s** | Dqlite (内置), Ceph, Longhorn |
| **K3s** | SQLite (内置), etcd |

### 网络插件

| 发行版 | 内置 CNI | 可选 |
|---|---|---|
| **k0s** | kube-router (默认) | Calico, Cilium |
| **MicroK8s** | CNI (基础) | Calico, Cilium |
| **K3s** | Flannel (默认) | Calico, Cilium, Canal |

### 升级机制

| 发行版 | 升级方式 | 自动升级 |
|---|---|---|
| **k0s** | `k0s upgrade` | 支持 |
| **MicroK8s** | `snap refresh` | 支持 |
| **K3s** | 自动检查更新 | 可配置 |

## 选型指南

### 场景对比

| 场景 | 推荐方案 | 原因 |
|---|---|---|
| Ubuntu/Debian 环境 | MicroK8s | snap 原生支持 |
| 需要 MySQL/PostgreSQL 存储 | k0s | kine 支持多种后端 |
| 最小资源占用 | K3s | 70MB，< 512MB 内存 |
| 多云/混合云 | k0s | 灵活架构 |
| 树莓派/ARM 设备 | K3s | ARM 优化最佳 |
| 快速本地开发 | 任意 | 都比标准 K8s 快 |
| 企业生产（小规模）| k0s | 功能全面 |
| Ubuntu LTS 用户 | MicroK8s | 官方支持 |

### 系统要求对比

| 发行版 | 最低 CPU | 最低内存 | 最低磁盘 |
|---|---|---|---|
| **k0s** | 2 核 | 1GB | 2GB |
| **MicroK8s** | 2 核 | 2GB | 20GB |
| **K3s** | 1 核 | 512MB | 2GB |

## 常见问题

### Q: k0s 和 K3s 哪个更好？

**A:** 没有绝对的好坏，只有适合的场景：

- 如果你需要最小资源占用 → K3s
- 如果你需要灵活存储后端 → k0s
- 如果你用 Ubuntu → MicroK8s

### Q: MicroK8s 和标准 K8s 完全兼容吗？

**A:** 基本兼容，但有细微差别。MicroK8s 默认只启用核心插件，其他功能需要手动 `microk8s enable`。

### Q: 轻量级 K8s 能用于生产吗？

**A:** 可以，但有限制：

- 建议 < 50 节点的集群
- 需要评估厂商支持
- 高可用需要额外配置

## 总结

**三剑客的定位：**

| 发行版 | 核心优势 | 最佳场景 |
|---|---|---|
| **K3s** | 极简、最轻量 | 边缘计算、资源受限 |
| **k0s** | 灵活、无依赖 | 混合云、多存储后端 |
| **MicroK8s** | Ubuntu 原生 | Ubuntu 用户、快速开发 |

**选择建议：**

```bash
# 你用 Ubuntu 吗？
#   是 → MicroK8s

# 你的内存 < 1GB？
#   是 → K3s

# 你需要 MySQL 作为存储后端？
#   是 → k0s

# 都不满足？
#   是 → K3s（社区最大）
```

**记住：**
- 三者都与上游 Kubernetes 兼容
- 学习其中任何一个，掌握的知识可迁移
- 没有银弹，只有最适合的选择
