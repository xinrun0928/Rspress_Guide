# KubeSphere 流水线语法

"这个 Jenkinsfile 能跑，但每次改配置都要登录 Jenkins 服务器，改完还容易出错。有没有办法把流水线配置也版本化？"

这是一个很好的问题。答案是：**把 Jenkinsfile 放在代码仓库里，用 Git 管理**。

## 流水线配置方式

### 方式一：流水线即代码（推荐）

将 Jenkinsfile 放在项目代码仓库中：

```
my-project/
├── src/
│   └── main/java/
├── Dockerfile
├── Jenkinsfile          # ← 流水线配置
└── pom.xml
```

### 方式二：图形化编辑

在 KubeSphere 控制台通过图形界面编辑，配置自动保存。

### 方式三：混合模式

图形化创建基础配置，代码中定义复杂逻辑。

## Jenkinsfile 语法详解

### 基础结构

```groovy
// Jenkinsfile 必须以 pipeline 块开始
pipeline {
    // 1. Agent: 流水线在哪里执行
    agent {
        // any: 任意可用节点
        // none: 不分配节点，由 stage 指定
        // label: 指定节点标签
        // kubernetes: 在 Kubernetes Pod 中执行
        any
    }
    
    // 2. Environment: 全局环境变量
    environment {
        APP_NAME = 'user-service'
        DOCKER_REGISTRY = 'harbor.example.com'
    }
    
    // 3. Options: 流水线选项
    options {
        // 构建超时时间
        timeout(time: 1, unit: 'HOURS')
        // 构建失败时保留日志
        buildDiscarder(logRotator(numToKeepStr: '10'))
        // 禁用并发构建
        disableConcurrentBuilds()
    }
    
    // 4. Stages: 阶段定义
    stages {
        stage('阶段名称') {
            steps {
                // 步骤定义
            }
        }
    }
    
    // 5. Post: 构建后操作
    post {
        always {
            // 无论成功失败都执行
        }
        success {
            // 构建成功时执行
        }
        failure {
            // 构建失败时执行
        }
    }
}
```

## Agent 配置

### 在 Kubernetes 中执行

<Badge text="推荐配置" type="tip" /> KubeSphere 推荐使用 Kubernetes Agent，它可以为每个流水线创建独立的 Pod 环境。

```groovy
agent {
    kubernetes {
        // Pod 模板定义
        label 'maven-builder'
        defaultContainer 'maven'
        
        // 定义多个容器
        yaml '''
apiVersion: v1
kind: Pod
metadata:
  name: maven-builder
spec:
  containers:
  - name: maven
    image: maven:3.8-openjdk-17
    command: ['cat']
    tty: true
    resources:
      requests:
        memory: "1Gi"
        cpu: "500m"
      limits:
        memory: "2Gi"
        cpu: "1000m"
  - name: docker
    image: docker:20.10
    command: ['cat']
    tty: true
    securityContext:
      privileged: true
'''
    }
}
```

### 多容器协作

```groovy
agent {
```

## Steps 常用指令

### Shell 命令

```groovy
steps {
    // 基本命令
    sh 'echo "Hello World"'
    
    // 多行命令
    sh '''
        echo "Line 1"
        echo "Line 2"
        mvn clean package
    '''
    
    // 命令返回值
    script {
        def result = sh(script: 'mvn --version', returnStdout: true)
        echo result
    }
}
```

### 条件执行

```groovy
steps {
    // when 条件
    when {
        // 分支匹配
        branch 'main'
        
        // 环境变量
        environment name: 'DEPLOY_ENV', value: 'production'
        
        // 表达式
        expression {
            return params.ENABLE_DEPLOY == 'true'
        }
        
        // 组合条件
        allOf {
            branch 'main'
            expression {
                return env.BUILD_NUMBER.toInteger() > 10
            }
        }
    }
}
```

### 参数化构建

```groovy
pipeline {
    parameters {
        string(name: 'BRANCH', defaultValue: 'main', description: 'Git 分支')
        booleanParam(name: 'ENABLE_DEPLOY', defaultValue: false, description: '是否部署')
        choice(name: 'ENV', choices: ['dev', 'test', 'prod'], description: '环境')
        password(name: 'SECRET_TOKEN', description: '访问令牌')
    }
    
    stages {
        stage('构建') {
            steps {
                echo "分支: ${params.BRANCH}"
                echo "环境: ${params.ENV}"
            }
        }
    }
}
```

## 环境变量

### 全局环境变量

```groovy
environment {
    APP_NAME = 'user-service'
    APP_VERSION = "${env.BUILD_NUMBER}"
    DOCKER_REGISTRY = 'harbor.example.com'
}
```

### Stage 级环境变量

```groovy
stages {
    stage('初始化') {
        environment {
            BUILD_ENV = 'development'
        }
        steps {
            echo "环境: ${env.BUILD_ENV}"
        }
    }
}
```

### 动态环境变量

```groovy
environment {
    // 动态获取 Git 提交信息
    GIT_COMMIT_SHORT = sh(script: 'git rev-parse --short HEAD', returnStdout: true).trim()
    
    // 获取当前日期
    BUILD_DATE = sh(script: 'date +%Y%m%d', returnStdout: true).trim()
    
    // 组合变量
    IMAGE_TAG = "${env.GIT_COMMIT_SHORT}-${env.BUILD_DATE}"
}
```

### 凭证使用

```groovy
environment {
    // 使用凭证（安全）
    DOCKER_USERNAME = credentials('docker-hub-username')
    DOCKER_PASSWORD = credentials('docker-hub-password')
}

stages {
    stage('登录镜像仓库') {
        steps {
            withCredentials([usernamePassword(
                credentialsId: 'docker-hub-credential',
                usernameVariable: 'DOCKER_USER',
                passwordVariable: 'DOCKER_PASS'
            )]) {
                sh '''
                    echo $DOCKER_PASS | docker login -u $DOCKER_USER \
                        $DOCKER_REGISTRY --password-stdin
                '''
            }
        }
    }
}
```

## 凭证类型详解

### 用户名密码

```groovy
withCredentials([usernamePassword(
    credentialsId: 'db-credential',
    usernameVariable: 'DB_USER',
    passwordVariable: 'DB_PASS'
)]) {
    // 在这个闭包内可以使用变量
    sh "mysql -u$DB_USER -p$DB_PASS ..."
}
```

### SSH 密钥

```groovy
withCredentials([sshUserPrivateKey(
    credentialsId: 'git-ssh-key',
    keyFileVariable: 'SSH_KEY',
    usernameVariable: 'SSH_USER'
)]) {
    sh 'git clone git@github.com:org/repo.git'
}
```

### 秘密文本

```groovy
withCredentials([string(
    credentialsId: 'api-token',
    variable: 'API_TOKEN'
)]) {
    sh 'curl -H "Authorization: Bearer $API_TOKEN" ...'
}
```

### 文件凭证

```groovy

    credentialsId: 'kubeconfig',
    variable: 'KUBECONFIG'
```

## 高级语法

### 矩阵式构建

```groovy
stage('多环境构建') {
    steps {
        script {
            def configs = [
                [env: 'dev', version: '1.0'],
                [env: 'test', version: '1.0'],
                [env: 'prod', version: '1.0']
            ]
            
            def builds = configs.collectEntries { config ->
                ["${config.env}": {
                    stage("构建 ${config.env}") {
                        environment {
                            TARGET_ENV = config.env
                            TARGET_VERSION = config.version
                        }
                        steps {
                            sh 'make build'
                        }
                    }
                }]
            }
            
            parallel builds
        }
    }
}
```

### 失败重试

```groovy
stages {
    stage('拉取代码') {
        steps {
            retry(3) {
                sh 'git clone ...'
            }
        }
    }
    
    stage('推送镜像') {
        steps {
            timeout(time: 5, unit: 'MINUTES') {
                retry(2) {
                    sh 'docker push ...'
                }
            }
        }
    }
}
```

### 错误处理

```groovy
stages {
    stage('执行可能失败的操作') {
        steps {
            script {
                try {
                    sh 'mvn test'
                } catch (Exception e) {
                    echo "测试失败: ${e.message}"
                    // 可以在这里决定是继续还是失败
                    // throw e  // 重新抛出异常
                }
            }
        }
    }
}
```

## 完整示例

```groovy
@Library('shared-pipeline') _

pipeline {
    agent {
        kubernetes {
            label 'java-builder'
            defaultContainer 'maven'
            yaml '''
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: maven
    image: maven:3.8-openjdk-17
    command: [cat]
    tty: true
    resources:
      requests:
        memory: "2Gi"
        cpu: "1"
  - name: docker
    image: docker:20.10
    command: [cat]
    tty: true
    securityContext:
      privileged: true
'''
        }
    }
    
    options {
        timeout(time: 30, unit: 'MINUTES')
        buildDiscarder(logRotator(numToKeepStr: '30'))
        disableConcurrentBuilds()
    }
    
    parameters {
        choice(name: 'BRANCH', choices: ['main', 'develop'], description: '分支')
        booleanParam(name: 'SKIP_TEST', defaultValue: false, description: '跳过测试')
        choice(name: 'ENV', choices: ['dev', 'test', 'prod'], description: '部署环境')
    }
    
    environment {
        DOCKER_REGISTRY = 'harbor.example.com'
        PROJECT_NAME = 'my-app'
        DOCKER_CREDENTIAL = 'harbor-credential'
        KUBECONFIG_CREDENTIAL = 'kubeconfig'
    }
    
    stages {
        stage('代码检出') {
            steps {
                git branch: params.BRANCH,
                    url: 'git@github.com:example/my-app.git',
                    credentialsId: 'github-ssh'
            }
        }
        
        stage('单元测试') {
            when {
                expression { return !params.SKIP_TEST }
            }
            steps {
                container('maven') {
                    sh 'mvn test -Dmaven.test.failure.ignore=false'
                }
            }
            post {
                always {
                    junit 'target/surefire-reports/*.xml'
                }
            }
        }
        
        stage('代码扫描') {
            steps {
                container('maven') {
                    sh '''
                        mvn sonar:sonar \
                            -Dsonar.projectKey=${PROJECT_NAME} \
                            -Dsonar.host.url=http://sonarqube:9000 \
                            -Dsonar.login=$SONAR_TOKEN
                    '''
                }
            }
        }
        
        stage('构建镜像') {
            steps {
                container('docker') {
                    script {
                        def imageTag = "${env.DOCKER_REGISTRY}/${params.ENV}/${env.PROJECT_NAME}:${env.BUILD_NUMBER}"
                        env.FINAL_IMAGE = imageTag
                        
                        sh """
                            docker build \
                                --build-arg VERSION=${env.BUILD_NUMBER} \
                                -t ${imageTag} .
                        """
                    }
                }
            }
        }
        
        stage('推送镜像') {
            steps {
                container('docker') {
                    withCredentials([usernamePassword(
                        credentialsId: env.DOCKER_CREDENTIAL,
                        usernameVariable: 'DOCKER_USER',
                        passwordVariable: 'DOCKER_PASS'
                    )]) {
                        sh '''
                            echo $DOCKER_PASS | docker login \
                                $DOCKER_REGISTRY -u $DOCKER_USER --password-stdin
                            docker push $FINAL_IMAGE
                        '''
                    }
                }
            }
        }
        
        stage('部署到环境') {
            steps {
                withCredentials([file(
                    credentialsId: env.KUBECONFIG_CREDENTIAL,
                    variable: 'KUBECONFIG'
                )]) {
                    sh '''
                        # 更新镜像版本
                        sed -i "s|image:.*|image: $FINAL_IMAGE|" k8s/${ENV}/deployment.yaml
                        
                        # 部署
                        kubectl apply -f k8s/${ENV}/ --namespace=${ENV}
                        
                        # 等待就绪
                        kubectl rollout status deployment/${PROJECT_NAME} \
                            -n ${ENV} --timeout=5m
                    '''
                }
            }
        }
    }
    
    post {
        success {
            echo '流水线执行成功！'
            dingTalk(
                type: 'LINK',
                title: '构建成功',
                text: "${env.PROJECT_NAME} 构建并部署成功",
                messageUrl: env.BUILD_URL
            )
        }
        failure {
            echo '流水线执行失败！'
            dingTalk(
                type: 'TEXT',
                text: "构建失败！\n项目: ${env.PROJECT_NAME}\n构建号: ${env.BUILD_NUMBER}"
            )
        }
        always {
            // 清理工作目录
            cleanWs()
        }
    }
}
```

## 常见问题

### 问题一：中文乱码

```groovy
// 在 steps 开始前设置编码
stages {
    stage('构建') {
        steps {
            // 设置 JVM 编码
            environment {
                JAVA_TOOL_OPTIONS = '-Dfile.encoding=UTF-8'
            }
            sh 'mvn clean package'
        }
    }
}
```

### 问题二：凭证无法使用

```
排查步骤：
1. 确认凭证 ID 正确
2. 确认凭证类型匹配（usernamePassword vs secretText）
3. 确认凭证有权限被 Jenkins Job 使用
4. 检查 Jenkins 日志是否有凭证解密错误
```

### 问题三：Pod 资源不足

```groovy
// 增加资源请求
agent {
    kubernetes {
        yaml '''
spec:

  - name: maven

        memory: "4Gi"    # 增加内存
        cpu: "2"         # 增加 CPU
      limits:
        memory: "8Gi"
        cpu: "4"
'''
    }
}
```
