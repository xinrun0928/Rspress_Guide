# KubeSphere 制品仓库

你有没有想过——KubeSphere 的流水线构建完成后，产物（Docker 镜像、Jar 包、Helm Chart）应该存到哪里？

KubeSphere 内置了多种制品仓库，支持 Harbor、Docker Registry、GitLab 等多种类型。本篇介绍如何配置和管理 KubeSphere 的制品仓库。

## 制品仓库概述

KubeSphere 支持以下类型的制品仓库：

| 类型 | 说明 | 适用场景 |
|---|---|---|
| **Harbor** | 企业级镜像仓库 | 生产环境镜像存储 |
| **Docker Registry** | Docker 官方 Registry | 测试环境 |
| **AWS S3 / MinIO** | 对象存储 | 二进制制品、Helm Chart |
| **GitLab** | 代码仓库 | 依赖存储 |
| **JFrog Artifactory** | 通用制品仓库 | 多类型制品 |

## 配置 Harbor 仓库

### 前提条件

- 已安装 Harbor 并配置好 HTTPS
- 创建了项目（Project）和有权限的用户

### 在 KubeSphere 中添加 Harbor

1. **进入制品管理**：左侧菜单 → 制品管理 → 创建制品仓库
2. **选择类型**：Harbor
3. **填写配置**：

```yaml
# 制品仓库配置
名称：my-harbor
类型：Harbor
Harbor URL：https://harbor.example.com
项目名称：library  # Harbor 中的项目名
用户名：admin
密码：Harbor12345
```

### 使用仓库

```bash
# 在流水线中使用 Harbor
pipeline {
  agent {
    kubernetes {
      yaml '''
        apiVersion: v1
        kind: Pod
        spec:
          containers:
          - name: builder
            image: maven:3.9-eclipse-temurin-17
            command: sleep 3600
          - name: docker
            image: docker:latest
            command: sleep 3600
            env:
            - name: DOCKER_API_VERSION
              value: "1.39"
      '''
    }
  }
  stages {
    stage('构建镜像') {
      steps {
        container('docker') {
          sh '''
            # 登录 Harbor
            echo ${HARBOR_PASSWORD} | docker login ${HARBOR_URL} -u ${HARBOR_USER} --password-stdin

            # 构建并推送镜像
            docker build -t ${HARBOR_URL}/library/myapp:${BRANCH_NAME}-${BUILD_NUMBER} .
            docker push ${HARBOR_URL}/library/myapp:${BRANCH_NAME}-${BRANCH_NAME}-${BUILD_NUMBER}
          '''
        }
      }
    }
  }
}
```

## 配置 S3/MinIO 仓库

### MinIO 快速部署

```yaml title="minio.yaml"
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: minio-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 50Gi
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: minio
spec:
  replicas: 1
  selector:
    matchLabels:
      app: minio
  template:
    metadata:
      labels:
        app: minio
    spec:
      containers:
      - name: minio
        image: minio/minio:latest
        args:
          - server
          - /data
          - --console-address
          - ":9001"
        env:
        - name: MINIO_ROOT_USER
          value: admin
        - name: MINIO_ROOT_PASSWORD
          value: minio123456
        ports:
        - containerPort: 9000
        - containerPort: 9001
        volumeMounts:
        - name: data
          mountPath: /data
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: minio-pvc
---
apiVersion: v1
kind: Service
metadata:
  name: minio
spec:
  type: LoadBalancer
  ports:
  - port: 9000
    targetPort: 9000
    name: api
  - port: 9001
    targetPort: 9001
    name: console
  selector:
    app: minio
```

### 在 KubeSphere 中添加 MinIO

1. **进入制品管理**：左侧菜单 → 制品管理 → 创建制品仓库
2. **选择类型**：S3
3. **填写配置**：

```yaml
# 制品仓库配置
名称：my-minio
类型：S3
Bucket：my-artifacts
Endpoint：http://minio:9000
Access Key：admin
Secret Key：minio123456
Region：us-east-1  # MinIO 使用默认区域
```

## 在流水线中使用制品仓库

### 推送 Docker 镜像

```groovy
pipeline {
  agent {
    kubernetes {
      yaml '''
        apiVersion: v1
        kind: Pod
        spec:
          containers:
          - name: builder
            image: maven:3.9-eclipse-temurin-17
            command: sleep 3600
          - name: docker
            image: docker:latest
            command: sleep 3600
            env:
            - name: DOCKER_API_VERSION
              value: "1.39"
            volumeMounts:
            - name: docker-socket
              mountPath: /var/run/docker.sock
          volumes:
          - name: docker-socket
            hostPath:
              path: /var/run/docker.sock
      '''
    }
  }
  environment {
    HARBOR_URL = 'harbor.example.com'
    HARBOR_USER = 'admin'
    HARBOR_PASSWORD = credentials('harbor-password')
    IMAGE_NAME = 'library/myapp'
  }
  stages {
    stage('代码编译') {
      steps {
        container('builder') {
          sh 'mvn clean package -DskipTests'
        }
      }
    }
    stage('构建镜像') {
      steps {
        container('docker') {
          sh '''
            # 登录 Harbor
            docker login ${HARBOR_URL} -u ${HARBOR_USER} -p ${HARBOR_PASSWORD}

            # 构建镜像
            docker build -t ${HARBOR_URL}/${IMAGE_NAME}:${BRANCH_NAME}-${BUILD_NUMBER} .

            # 推送镜像
            docker push ${HARBOR_URL}/${IMAGE_NAME}:${BRANCH_NAME}-${BUILD_NUMBER}

            # 打标签为 latest
            docker tag ${HARBOR_URL}/${IMAGE_NAME}:${BRANCH_NAME}-${BUILD_NUMBER} ${HARBOR_URL}/${IMAGE_NAME}:latest
            docker push ${HARBOR_URL}/${IMAGE_NAME}:latest
          '''
        }
      }
    }
  }
}
```

### 推送 Helm Chart

```groovy
stage('推送 Helm Chart') {
  steps {
    container('helm') {
      sh '''
        # 配置 Helm 仓库
        helm repo add myharbor https://${HARBOR_URL}/chartrepo/${PROJECT_NAME}

        # 打包 Chart
        helm package ./charts/myapp

        # 推送 Chart
        curl -u ${HARBOR_USER}:${HARBOR_PASSWORD} \
          -T myapp-1.0.0.tgz \
          "https://${HARBOR_URL}/api/chartrepo/${PROJECT_NAME}/charts"
      '''
    }
  }
}
```

### 推送二进制制品

```groovy
stage('上传制品') {
  steps {
    script {
      def s3Artifact = [
        bucket: 'my-artifacts',
        region: 'us-east-1',
        accessKey: env.AWS_ACCESS_KEY_ID,
        secretKey: env.AWS_SECRET_ACCESS_KEY,
        uploadPath: "builds/${BRANCH_NAME}/${BUILD_NUMBER}/"
      ]

      // 上传 Jar 文件
      sh '''
        aws s3 cp target/myapp.jar \
          s3://${S3_BUCKET}/builds/${BRANCH_NAME}/${BUILD_NUMBER}/myapp.jar
      '''
    }
  }
}
```

## 管理制品

### 查看制品

```bash
# 在 KubeSphere 控制台
# 左侧菜单 → 制品管理 → 选择仓库 → 查看制品列表

# 或通过 kubectl
kubectl get artifacts -n ${NAMESPACE}
```

### 清理制品

```yaml
# 配置自动清理策略
apiVersion: core.kubesphere.io/v1alpha1
kind: Retention
metadata:
  name: cleanup-policy
spec:
  scope:
    selector:
      matchLabels:
        app: myapp
  rules:
  - maxAge: 30d  # 保留 30 天
    maxCount: 100   # 最多保留 100 个版本
```

### 制品签名

```bash
# 使用 Notary 对镜像签名
docker trust signer add --key cert.pem admin harbor.example.com

# 签名镜像
DOCKER_CONTENT_TRUST=1 docker push harbor.example.com/library/myapp:latest
```

## 最佳实践

### 1. 仓库隔离

```yaml
# 开发环境
harbor-dev:
  url: https://harbor-dev.example.com
  project: dev

# 测试环境
harbor-test:
  url: https://harbor-test.example.com
  project: test

# 生产环境
harbor-prod:
  url: https://harbor-prod.example.com
  project: prod
```

### 2. 镜像标签策略

```bash
# 推荐标签格式
# Git SHA 标签：git-${GIT_COMMIT_SHA:0:8}
# 分支+构建号：${BRANCH_NAME}-${BUILD_NUMBER}
# 环境标签：latest、stable、canary

# 示例
harbor.example.com/library/myapp:git-abc12345
harbor.example.com/library/myapp:feature-login-100
harbor.example.com/library/myapp:latest
```

### 3. 镜像安全扫描

在 Harbor 中启用 Trivy 扫描：

```bash
# 推送镜像后自动触发扫描
# Harbor 自动扫描 CVE 漏洞

# 查看扫描结果
curl -u admin:password \
  "https://harbor.example.com/api/v2.0/projects/library/repositories/myapp/artifacts/latest/additions/vulnerabilities"
```

## 总结

KubeSphere 制品仓库管理的核心要点：

1. **Harbor**：企业级镜像仓库，支持安全扫描和镜像签名
2. **S3/MinIO**：适合存储二进制制品和 Helm Chart
3. **流水线集成**：在 CI/CD 流水线中自动推送制品
4. **清理策略**：配置保留策略，避免存储成本过高
5. **安全扫描**：使用 Trivy 扫描镜像漏洞

合理的制品仓库管理，可以让 CI/CD 流水线更加高效和安全。
