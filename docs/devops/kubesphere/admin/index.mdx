# KubeSphere 运维指南

> 「KubeSphere 装好了，跑起来了，然后呢？」

作为 Kubernetes 的企业级管理平台，KubeSphere 提供了远超原生 K8s 的用户体验。但装好只是起点——真正的挑战在于如何高效地运维这个平台：添加节点、管理存储、配置网络、处理升级。

本章节聚焦 KubeSphere 的运维实践，从节点管理到存储配置，从 RBAC 权限控制到故障排查，手把手教你成为一个合格的 KubeSphere 管理员。

## 章节内容

### 节点管理

Kubernetes 集群的节点是所有工作负载的底座。添加新节点、配置污点、管理节点角色——这些操作直接影响集群的稳定性和资源利用率。

### 存储管理

有状态应用需要持久化存储。从 StorageClass 的创建到 PersistentVolume 的动态供给，从配额管理到存储后端选型，存储往往是新手的第一个坑。

### 网络配置

Pod 之间如何通信？外部流量如何进入集群？KubeSphere 支持多种网络方案，从内置的 Calico 到负载均衡器 PorterLB，配置不当会导致服务无法访问。

### 升级与维护

Kubernetes 版本迭代频繁，KubeSphere 本身也在快速演进。升级是风险最高的操作——备份、灰度、回滚，每个环节都不能出错。

### 权限控制

多租户是 KubeSphere 的核心能力之一。从集群管理员到项目观察者，角色决定了你能看到什么、操作什么。权限过大是安全风险，权限过小影响效率。

### 故障排查

集群不工作？组件报错？本章节提供系统化的排查思路和实用工具，让你在「战争」中快速定位问题。

:::tip 学习路径建议
1. 先完成节点管理，确保集群基础健康
2. 配置存储，这是大多数有状态应用的前提
3. 根据业务需求配置网络和 Ingress
4. 理解 RBAC，建立多租户权限体系
5. 制定升级计划，定期维护集群
:::
