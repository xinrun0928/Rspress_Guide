# KubeSphere 安装部署

安装是所有故事的起点。

很多团队在评估 KubeSphere 时，第一个问题就是："安装复不复杂？"答案是：取决于你选择哪种模式。KubeSphere 提供了从"尝鲜"到"生产"的全套安装方案，总有一款适合你。

这一节，我们详细讲解 KubeSphere 的各种安装方式，以及安装过程中需要注意的坑。

## 安装模式概览

KubeSphere 提供了多种安装模式，从简单到复杂：

| 模式 | 适用场景 | 复杂度 | 硬件要求 |
|---|---|---|---|
| All-in-One | 尝鲜、学习 | 极简 | 2核4G |
| 多节点 | 准生产、小规模生产 | 中等 | 4核8G * 3 |
| 高可用 | 大规模生产 | 复杂 | 8核16G * 3 + LB |

<Badge text="安装工具" type="tip" /> KubeSphere 推荐使用 KubeKey 进行安装，这是一个 Go 写的安装工具，支持 Kubernetes 集群的一键部署和 KubeSphere 的插件化安装。

### 硬件要求

| 数据盘 | 100 GB | 200 GB SSD（用于存储） |

### 软件要求

| 软件 | 版本要求 |
|---|---|
| 操作系统 | Ubuntu 16.04/18.04/20.04, CentOS 7.x/8.x, Debian |
| Kubernetes | 1.19.x ~ 1.24.x |
| Docker | 19.3.1+ |
| kubectl | 与 Kubernetes 版本匹配 |

### 网络要求

| 配置项 | 说明 |
|---|---|
| 外网访问 | 安装过程需要拉取镜像 |
| 节点互通 | 所有节点内网互通 |
| 端口开放 | 见下表 |

#### 必要端口

| 端口 | 协议 | 服务 | 说明 |
|---|---|---|---|

| 6443 | TCP | kube-apiserver | API Server |
| 10250-10255 | TCP | kubelet | 节点通信 |
| 30000-32767 | TCP | NodePort | 服务暴露 |

## 安装方式一：All-in-One

All-in-One 是最简单的方式，适合快速体验 KubeSphere。它会把 Kubernetes 和 KubeSphere 全部安装在一台机器上。

### 前置条件

确保机器上没有 Kubernetes 相关组件：

```bash
# 检查是否有 Kubernetes
kubectl get nodes 2>/dev/null
systemctl status kubelet 2>/dev/null

# 如果有残留，清理干净
kubeadm reset -f
rm -rf ~/.kube
```

### 安装步骤

#### 1. 下载 KubeKey

```bash
export KKZONE=cn  # 国内用户设置代理
curl -sfL https://get-kk.kubesphere.io | sh -

# 国内镜像加速
curl -sfL https://get-kk.kubesphere.io | sh -s 2.0.2 -f
```

#### 2. 创建配置文件

```bash
cat > config-sample.yaml << 'EOF'
apiVersion: kubekey.kubesphere.io/v1alpha2
kind: Cluster
metadata:
  name: sample
spec:
  hosts:
  - {name: node1, address: 192.168.1.10, internalAddress: 192.168.1.10, user: root, password: "your-password"}
  roleGroups:
    etcd:
    - node1
    master:
    - node1
    worker:
    - node1
  kubernetes:
    version: v1.23.10
    containerManager: docker
  network:
    plugin: calico
    kubePodsCIDR: 10.233.64.0/18
    kubeServiceCIDR: 10.233.0.0/18
  registry:
    registryMirrors: []
    insecureRegistries: []
  addons: []
EOF
```

#### 3. 执行安装

```bash
./kk create cluster -f config-sample.yaml
```

<Badge text="耐心等待" type="warning" /> 首次安装需要下载大量镜像，根据网络状况可能需要 10-30 分钟。

#### 4. 安装 KubeSphere

```bash
# 方式一：安装时直接指定
./kk create cluster -f config-sample.yaml --with-kubesphere v3.4.1

# 方式二：先安装 Kubernetes，再安装 KubeSphere
./kk create cluster -f config-sample.yaml
./kubectl apply -f https://raw.githubusercontent.com/kubesphere/ks-installer/master/deploy/kubesphere-installer.yaml
./kubectl apply -f https://raw.githubusercontent.com/kubesphere/ks-installer/master/deploy/cluster-configuration.yaml
```

#### 5. 验证安装

```bash
# 检查 KubeSphere 命名空间
kubectl get ns | grep kubesphere

# 查看所有 Pod 状态
kubectl get pods -A

# 查看 KubeSphere 版本
kubectl exec -n kubesphere-system ks-console-0 -- cat /app/node_modules/@kubesphere/console/package.json | grep version
```

#### 6. 访问控制台

```
# 获取 Admin 密码
kubectl get secret -n kubesphere-system admin-user -o jsonpath='{.iterations}'  # 获取迭代次数
kubectl describe secret -n kubesphere-system admin-user

# 默认账户
# 用户名: admin
# 密码: P@88w0rd

# 访问地址（根据实际情况调整）
http://192.168.1.10:30880
```

## 安装方式二：多节点集群

多节点模式适合准生产环境，可以体验完整的多租户功能。

### 集群规划

```
┌─────────────────────────────────────────────────────────────┐
│                    Load Balancer                           │
│                 (可选，如 Nginx)                            │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐        │
│  │   Master 1  │  │   Master 2  │  │   Master 3  │        │
│  │  (etcd+API) │  │  (etcd+API) │  │  (etcd+API) │        │
│  └─────────────┘  └─────────────┘  └─────────────┘        │
│  ┌─────────────┐  ┌─────────────┐                        │
│  │   Worker 1  │  │   Worker 2  │                        │
│  │  (计算节点) │  │  (计算节点) │                        │
│  └─────────────┘  └─────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

### 配置文件示例

```yaml
apiVersion: kubekey.kubesphere.io/v1alpha2
kind: Cluster
metadata:
  name: multi-node
spec:
  hosts:
  # Master 节点
  - {name: master1, address: 192.168.1.10, internalAddress: 192.168.1.10, user: root, password: "pass1", port: 22}
  - {name: master2, address: 192.168.1.11, internalAddress: 192.168.1.11, user: root, password: "pass2", port: 22}
  - {name: master3, address: 192.168.1.12, internalAddress: 192.168.1.12, user: root, password: "pass3", port: 22}
  # Worker 节点
  - {name: worker1, address: 192.168.1.20, internalAddress: 192.168.1.20, user: root, password: "pass4", port: 22}
  - {name: worker2, address: 192.168.1.21, internalAddress: 192.168.1.21, user: root, password: "pass5", port: 22}

  roleGroups:
    etcd:
    - master1
    - master2
    - master3
    master:
    - master1
    - master2
    - master3
    worker:
    - worker1
    - worker2

  kubernetes:
    version: v1.23.10
    containerManager: docker
    masqueradeade: true
    hostnameCheck: false
  network:
    plugin: calico
    kubePodsCIDR: 10.233.64.0/18
    kubeServiceCIDR: 10.233.0.0/18
  # KubeSphere 组件
  addons:
  - name: kubesphere
    namespace: kubesphere-system
    sources:
      chart:
        name: kubesphere
        repo: https://charts.kubesphere.io
        version: v3.4.1
        values:
          etcd:
            endpointIps: 192.168.1.10,192.168.1.11,192.168.1.12
            port: 2379
          common:
            redis:
              enabled: true
            openldap:
              enabled: true
            minio:
              enabled: true
            prometheus:
              enabled: true
            logging:
              enabled: true
            alerting:
              enabled: true
            devops:
              enabled: true
            servicemesh:
              enabled: true
```

### 安装步骤

```bash
# 执行安装
./kk create cluster -f config-multi-node.yaml

# 验证集群状态
kubectl get nodes
kubectl get pods -n kubesphere-system
```

## 组件选择性安装

KubeSphere 提供了模块化的安装选项，可以根据需要选择性安装组件。

### 安装清单

| 组件 | 标识 | 说明 |
|---|---|---|
| KubeSphere Core | 默认安装 | 最小化安装，只包含基础功能 |
| DevOps | --with-devops | Jenkins 流水线和制品库 |
| 日志系统 | --with-logging | Elasticsearch + FluentBit |
| 监控 | --with-monitoring | Prometheus + Grafana |
| 服务网格 | --with-servicemesh | Istio |
| 告警 | --with-alerting | AlertManager |
| 网络策略 | --with-networkpolicies | NetworkPolicy 支持 |
| 边缘节点 | --with-edge | KubeEdge 边缘计算支持 |

### 最小化安装

```bash
./kk create cluster -f config.yaml --with-kubesphere v3.4.1
```

### 完整安装

```bash
./kk create cluster -f config.yaml \
  --with-kubesphere v3.4.1 \
  --with-kubernetes v1.23.10 \
  --with-devops \
  --with-logging \
  --with-monitoring \
  --with-servicemesh \
  --with-alerting
```

### 插件化安装（安装后添加）

如果最初选择了最小化安装，之后可以单独开启某个功能：

```bash
# 编辑 cluster-configuration
kubectl edit cc -n kubesphere-system ks-installer
```

```yaml
# 开启 DevOps
spec:
  devops:
    enabled: true
    jenkinsVolumeSize: 20Gi
    jobsNamespace: kubesphere-devops-jobs

# 开启日志系统
spec:
  logging:
    enabled: true
    elasticsearchVolumeSize: 20Gi
    logsidecarVolumeSize: 2Gi
```

## 安装后配置

### 初始化管理员密码

首次登录后，建议立即修改默认密码：

```bash
# 通过控制台修改
# 系统设置 → 账户管理 → admin → 修改密码

# 命令行方式（需重置）
kubectl patch user admin -n kubesphere-system \
  -p '{"spec":{"password":"$2a$10$xxxx"}}' --type=merge
```

### 配置外网访问

```yaml
# 编辑 ks-console 服务，改为 NodePort
kubectl patch svc ks-console -n kubesphere-system \
  -p '{"spec":{"type":"NodePort"}}'

# 或创建 Ingress
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: ks-console-ingress
  namespace: kubesphere-system
spec:
  rules:
  - host: ks.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: ks-console
            port:
              number: 80
```

## 常见问题排查

### 问题一：安装超时

```
症状：安装过程卡在某一步，超过 30 分钟没有进展

解决方案：
1. 检查网络连接
2. 使用国内镜像源
3. 手动拉取镜像

# 列出需要拉取的镜像
./kk create config -f config.yaml

# 手动拉取镜像（示例）
docker pull registry.cn-beijing.aliyuncs.com/kubesphere/ks-console:v3.4.1
```

### 问题二：etcd 启动失败

```
症状：etcd Pod 处于 CrashLoopBackOff 状态

解决方案：
1. 检查时间同步
timedatectl status
timedatectl set-ntp true

2. 检查磁盘空间
df -h

3. 检查内存是否足够
free -h
```

### 问题三：组件启动失败

```bash
# 查看组件状态
kubectl get pods -n kubesphere-system

# 查看特定组件日志
kubectl logs -n kubesphere-system -l app=ks-console
kubectl logs -n kubesphere-system -l app=ks-apiserver

# 查看安装日志
kubectl logs -n kubesphere-system ks-installer
```

## 卸载 KubeSphere

如果需要卸载 KubeSphere 但保留 Kubernetes：

```bash
# 卸载 KubeSphere
./kk delete cluster -f config.yaml --remove-packages

# 完全卸载（包含 Kubernetes）
./kk delete cluster -f config.yaml
```
