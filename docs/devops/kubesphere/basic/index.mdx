# KubeSphere 基础

这一章节是 KubeSphere 的入门篇，适合刚接触 KubeSphere 或计划在团队中推广 KubeSphere 的同学。

## 学习路径

在开始之前，先给你一张学习地图：

```
KubeSphere 基础
│
├── 平台概览        ← 了解 KubeSphere 能做什么
├── 产品对比        ← 知道 KubeSphere 和竞品的差异
├── 系统架构        ← 理解 KubeSphere 的设计原理
├── 多租户体系      ← 掌握 KubeSphere 的权限管理核心
└── 安装部署        ← 亲手搭建一个 KubeSphere 环境
```

## 你将学到什么

通过这一章节的学习，你将能够：

1. **理解 KubeSphere 的定位**：清楚 KubeSphere 解决的是什么问题，适合什么场景
2. **掌握系统架构**：了解 KubeSphere 的组件构成和服务通信机制
3. **理解多租户体系**：掌握工作空间、项目、DevOps 项目的概念和关系
4. **完成安装部署**：能够在不同场景下安装 KubeSphere（All-in-One 和多节点）

:::tip 学习建议
建议先通读这一章节，对 KubeSphere 有整体认知后再深入各个模块。如果你想先看效果，可以直接跳到「安装部署」，用 All-in-One 模式快速体验。
:::

## 前置知识

虽然 KubeSphere 降低了 Kubernetes 的使用门槛，但你至少需要了解以下基础知识：

| 概念 | 了解程度 | 说明 |
|---|---|---|
| Docker 容器 | 必须 | 理解容器、镜像、仓库的概念 |
| Kubernetes | 推荐 | 了解 Pod、Deployment、Service 等核心概念 |
| Linux 基础 | 必须 | 会用命令行，会看日志 |
| 网络基础 | 推荐 | 了解 Ingress、Service、Ingress 的区别 |

如果你对 Kubernetes 还不太熟悉，建议先阅读本指南的 Kubernetes 相关章节。
