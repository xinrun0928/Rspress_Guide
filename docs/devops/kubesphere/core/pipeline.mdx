# KubeSphere CI/CD 流水线

凌晨 3 点，你被一条报警叫醒：生产环境出问题了。

你睡眼惺忪地打开电脑，登录服务器，开始排查。翻了半天日志，发现是一个小时前的手动部署出了问题——测试环境明明是好的，但你部署的时候改了一个参数，生产环境没改……

这是一个典型的 CI/CD 场景问题。

手动部署是高风险操作。KubeSphere 的 CI/CD 流水线就是为了解决这个问题——**让代码提交到生产部署的一切都是自动化的、可追溯的、可靠的**。

## 流水线概述

KubeSphere 的 CI/CD 功能基于 Jenkins 构建，但做了大量的封装和增强，让它变得更易用、更适合团队协作。

### 为什么选择 Jenkins

Jenkins 是 CI/CD 领域的"老大哥"，虽然近年来出现了 GitLab CI、GitHub Actions 等新选手，但它依然是企业级 CI/CD 的主流选择：

| 优势 | 说明 |
|---|---|
| 生态成熟 | 插件丰富，几乎可以做任何事 |
| 可扩展性强 | 可以自定义插件和步骤 |
| 社区活跃 | 遇到问题容易找到解决方案 |
| 企业级功能 | LDAP 集成、权限控制、分布式构建 |

### KubeSphere 对 Jenkins 的增强

<Badge text="差异化价值" type="tip" /> KubeSphere 不是简单地把 Jenkins 包装一下，它在 Jenkins 基础上构建了一套完整的多租户 CI/CD 体系。


## 流水线核心概念

### Pipeline（流水线）

Pipeline 是 KubeSphere CI/CD 的核心，它定义了一个软件从代码提交到部署的完整流程。

```
代码提交 → 编译构建 → 单元测试 → 镜像构建 → 镜像推送 → 部署测试 → 部署生产
   │          │          │          │          │          │          │
   └──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
                               Jenkins Pipeline
```

### Pipeline 的结构

```groovy
pipeline {
    // 1. 代理配置：流水线在哪里执行
    agent any
    
    // 2. 环境变量
    environment {
        DOCKER_REGISTRY = 'harbor.example.com'
        APP_NAME = 'my-app'
    }
    
    // 3. 阶段定义
    stages {
        stage('拉取代码') {
            steps {
                git 'https://github.com/example/my-app.git'
            }
        }
        
        stage('编译构建') {
            steps {
                sh 'mvn clean package'
            }
        }
        
        stage('单元测试') {
            steps {
                sh 'mvn test'
            }
        }
        
        stage('构建镜像') {
            steps {
                sh 'docker build -t ${APP_NAME}:${BUILD_NUMBER} .'
            }
        }
        
        stage('推送镜像') {
            steps {
                sh 'docker push ${DOCKER_REGISTRY}/${APP_NAME}:${BUILD_NUMBER}'
            }
        }
    }
    
    // 4. 通知配置
    post {
        success {
            echo '流水线执行成功！'
        }
        failure {
            echo '流水线执行失败！'
        }
    }
}
```

## 凭证管理

### 支持的凭证类型

| 类型 | 用途 |
|---|---|
| 用户名/密码 | Git、镜像仓库登录 |
| SSH 密钥 | Git SSH 拉取 |
| 访问令牌 | GitHub/GitLab Token |
| 秘密文本 | 敏感配置值 |

### 创建凭证

```
DevOps 项目 → 凭证管理 → 创建凭证
  ├── 类型：用户名/密码
  ├── 名称：docker-hub-credential
  ├── 描述：Docker Hub 登录凭证
  └── 内容：
        用户名: admin
        密码: ******
```

### 在流水线中使用凭证

```groovy
pipeline {
    agent any
    
    environment {
        // 从凭证中读取，不需要硬编码
        DOCKER_USERNAME = credentials('docker-credential')
    }
    
    stages {
        stage('登录镜像仓库') {
            steps {
                withCredentials([usernamePassword(
                    credentialsId: 'docker-credential',
                    usernameVariable: 'DOCKER_USER',
                    passwordVariable: 'DOCKER_PASS'
                )]) {
                    sh 'echo $DOCKER_USER | docker login -u $DOCKER_USER --password-stdin'
                }
            }
        }
    }
}
```

### 自定义模板

团队可以根据需要创建自己的模板：

```groovy
// my-team-template.groovy
def call(Map config) {
    pipeline {
        agent {
            kubernetes {
                label 'maven-builder'
                defaultContainer 'maven'
            }
        }
        
        stages {
            stage('初始化') {
                steps {
                    container('maven') {
                        sh 'echo "开始构建 ${config.appName}"'
                    }
                }
            }
            
            stage('代码检出') {
                steps {
                    git config.user.email "ci@example.com"
                    git config user.name "CI Bot"
                }
            }
            
            stage('编译') {
                steps {
                    container('maven') {
                        sh config.buildCommand ?: 'mvn clean package'
                    }
                }
            }
            
            // ... 更多阶段
        }
    }
}
```

### 构建状态通知

```groovy
post {
    always {
        // 记录构建状态
        echo "构建状态: ${currentBuild.result}"
    }
    
    success {
        // 发送成功通知
        dingTalk(
            robot: 'devops-robot',
            type: 'LINK',
            title: '构建成功',
            text: '流水线执行成功',
            messageUrl: "${env.BUILD_URL}"

    failure {
        // 发送失败通知

        )
    }
}
```

## 实践：完整的 Java 应用流水线

### 场景描述

- 代码仓库：GitHub
- 构建工具：Maven
- 镜像仓库：Harbor
- 部署目标：KubeSphere 项目

### 流水线配置

```groovy
pipeline {
    agent {
        kubernetes {
            label 'maven-jdk17'
            defaultContainer 'maven'
        }
    }
    
    environment {
        DOCKER_REGISTRY = 'harbor.example.com'
        DOCKER_CREDENTIAL_ID = 'harbor-credential'
        GIT_CREDENTIAL_ID = 'github-ssh'
        REGISTRY_PROJECT = 'java-apps'
        APP_NAME = 'user-service'
        KUBECONFIG_CREDENTIAL_ID = 'kubeconfig-dev'
    }
    
    stages {
        stage('拉取代码') {
            steps {
                git url: 'git@github.com:example/user-service.git',
                    branch: 'main',
                    credentialsId: env.GIT_CREDENTIAL_ID
            }
        }
        
        stage('单元测试') {
            steps {
                container('maven') {
                    sh 'mvn test -Dmaven.test.failure.ignore=false'
                }
            }
        }
        
        stage('代码质量检查') {
            steps {
                container('maven') {
                    sh '''
                        mvn sonar:sonar \
                        -Dsonar.projectKey=${APP_NAME} \
                        -Dsonar.host.url=http://sonarqube:9000
                    '''
                }
            }
        }
        
        stage('构建镜像') {
            steps {
                container('maven') {
                    sh '''
                        mvn package -DskipTests
                        docker build \
                            -t ${DOCKER_REGISTRY}/${REGISTRY_PROJECT}/${APP_NAME}:${BUILD_NUMBER} \
                            -f Dockerfile .
                    '''
                }
            }
        }
        
        stage('推送镜像') {
            steps {
                container('maven') {
                    withCredentials([usernamePassword(
                        credentialsId: env.DOCKER_CREDENTIAL_ID,
                        usernameVariable: 'DOCKER_USER',
                        passwordVariable: 'DOCKER_PASS'
                    )]) {
                        sh '''
                            echo $DOCKER_PASS | docker login -u $DOCKER_USER \
                                $DOCKER_REGISTRY --password-stdin
                            docker push ${DOCKER_REGISTRY}/${REGISTRY_PROJECT}/${APP_NAME}:${BUILD_NUMBER}
                        '''
                    }
                }
            }
        }
        
        stage('部署到测试环境') {
            steps {
                container('maven') {
                    withCredentials([file(
                        credentialsId: env.KUBECONFIG_CREDENTIAL_ID,
                        variable: 'KUBECONFIG'
                    )]) {
                        sh '''
                            sed -i "s|image:.*|image: ${DOCKER_REGISTRY}/${REGISTRY_PROJECT}/${APP_NAME}:${BUILD_NUMBER}|" \
                                k8s/deployment.yaml
                            kubectl apply -f k8s/
                        '''
                    }
                }
            }
        }
    }
    
    post {
        success {
            echo '流水线执行成功，镜像已部署到测试环境'
        }
        failure {
            echo '流水线执行失败，请检查日志'
        }
    }
}
```
