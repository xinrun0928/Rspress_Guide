# SonarQube 代码质量管理

代码写完了，测试也通过了，就能上线了吗？不，你还需要 SonarQube——它能发现测试覆盖不到的代码问题：潜在 Bug、代码异味、安全漏洞、技术债务。

## SonarQube 是什么

### 核心功能

| 功能 | 说明 |
|---|---|
| **代码质量检测** | Bug、漏洞、代码异味 |
| **代码覆盖** | 集成测试覆盖率 |
| **技术债务** | 估算修复时间 |
| **安全热点** | 发现安全漏洞 |
| **重复代码** | 发现复制粘贴的代码 |

### 架构

```
开发提交代码
    │
    ├── GitLab/GitHub Hook 触发
    │
    ▼
SonarScanner 扫描代码
    │
    ▼
分析结果上报
    │
    ▼
SonarQube Server 分析存储
    │
    ▼
SonarQube UI 查看报告
```

## GitLab CI 集成

### 基础配置

```yaml title=".gitlab-ci.yml"
sonarqube:
  stage: quality
  image: sonarsource/sonar-scanner-cli:latest
  variables:
    SONAR_USER_HOME: ${CI_PROJECT_DIR}/.sonar
  cache:
    key: ${CI_COMMIT_REF_SLUG}
    paths:
      - .sonar/cache
  script:
    - sonar-scanner
  allow_failure: true
  only:
    - merge_requests
    - main
    - develop
```

## SonarQube 配置

### 质量配置

```yaml title="sonar-project.properties"
sonar.projectKey=my-java-app
sonar.projectName=My Java Application
sonar.projectVersion=1.0

sonar.sources=src/main/java
sonar.tests=src/test/java
sonar.language=java

# 覆盖率
sonar.java.binaries=target/classes
sonar.coverage.jacoco.xmlReportPaths=target/site/jacoco/jacoco.xml

# 排除
sonar.exclusions=**/*.proto,**/generated/**,**/test/**
sonar.test.exclusions=**/*Test.java

# 质量门禁
sonar.qualitygate.wait=true
```

### 质量规则

SonarQube 内置了丰富的质量规则：

| 规则类型 | 说明 | 示例 |
|---|---|---|
| Bug | 明显的错误 | NullPointerException 可能 |
| Vulnerability | 安全漏洞 | SQL 注入、XSS |
| Code Smell | 代码异味 | 重复代码、方法过长 |
| Security Hotspot | 安全热点 | 硬编码密码、Crypto |

### 质量门禁

```yaml title="quality-gate.yml"
# 质量门禁规则
quality_gate:
  on_default_gate: true
  gitlab_comment: true
```

质量门禁条件示例：

```
- 新增代码覆盖率 >= 80%
- 新增 Bug 数 = 0
- 新增漏洞数 = 0
- 重复度 <= 3%
- 技术债务 <= 1 天
```

## Maven 集成详解

### 完整配置

```xml title="pom.xml"
<properties>
    <sonar.host.url>${SONAR_HOST_URL}</sonar.host.url>
    <sonar.login>${SONAR_TOKEN}</sonar.login>
    <jacoco.version>0.8.11</jacoco.version>

    <plugins>
        <!-- JaCoCo 覆盖率 -->
        <plugin>
            <groupId>org.jacoco</groupId>
            <artifactId>jacoco-maven-plugin</artifactId>
            <version>${jacoco.version}</version>
            <executions>
                <execution>
                    <goals>
                        <goal>prepare-agent</goal>
                    </goals>
                </execution>
                <execution>
                    <id>report</id>
                    <phase>test</phase>
                    <goals>
                        <goal>report</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>

        <!-- Sonar Scanner -->
        <plugin>
            <groupId>org.sonarsource.scanner.maven</groupId>
            <artifactId>sonar-maven-plugin</artifactId>
            <version>3.11.0.3922</version>
        </plugin>
    </plugins>
</build>
```

```bash
  -Dsonar.host.url=http://sonarqube:9000 \
  -Dsonar.token=${SONAR_TOKEN}

mvn verify sonar:sonar
```

## Gradle 集成详解

### 完整配置

```groovy title="build.gradle"
plugins {
    id 'org.sonarqube' version '4.4.0.3356'
}

sonarqube {
    properties {
        property "sonar.projectName", "${project.name}"
        property "sonar.projectKey", "${project.group}:${project.name}"
        property "sonar.projectVersion", project.version
        
        // Java 配置
        property "sonar.language", "java"
        property "sonar.java.source", rootProject.sourceCompatibility
        property "sonar.java.target", rootProject.targetCompatibility
        
        // 源代码
        property "sonar.sources", sourceSets.main.java.srcDirs.join(',')
        property "sonar.tests", sourceSets.test.java.srcDirs.join(',')
        property "sonar.java.binaries", sourceSets.main.java.output.classesDirs.asPath
        
        // 覆盖率
        property "sonar.coverage.jacoco.xmlReportPaths", 
            "${project.layout.buildDirectory.get()}/reports/jacoco/test/jacocoTestReport.xml"
        
        // 排除
        property "sonar.exclusions", "**/*Test.java,**/generated/**,**/*.proto"
    }
}

test {
    useJUnit()
    finalizedBy jacocoTestReport
}

jacocoTestReport {
    reports {
        xml.required = true
    }
}
```

## MR 评论集成

### GitLab MR 页面显示

```yaml title="gitlab-mr-comment.yml"
sonarqube:
  stage: quality
  script:
    - |
      SONAR_ANALYSIS=$(cat <<'EOF'
      {
        "pScannerUuid": "${CI_PIPELINE_ID}",
        "scannerUrl": "${CI_PROJECT_URL}/-/quality_projects",
        "dashboardUrl": "${CI_PROJECT_URL}/-/quality_projects",
        "status": "passed"
      }
      EOF
      )
      
      # 上报分析结果，GitLab 会自动显示在 MR 页面
      echo "$SONAR_ANALYSIS" > sonar_result.json
  artifacts:
    paths:
      - sonar_result.json
```

## 安全扫描

### 依赖漏洞扫描

```yaml title="dependency-check.yml"
dependency_scan:
  stage: security
  image: owasp/dependency-check:8.4
  script:
    - |
      dependency-check.sh \
        --project "My Application" \
        --scan "target/*.jar" \
        --format "ALL" \
        --out "."
    - mv dependency-check-report.* target/
  artifacts:
    paths:
      - target/dependency-check-report.*
    expire_in: 1 week
```

### SonarQube 安全规则

| 类别 | 规则示例 | 严重程度 |
|---|---|---|
| SQL 注入 | 使用字符串拼接构建 SQL | Critical |
| XSS | 未转义的用户输入输出 | Critical |
| 硬编码密码 | 代码中包含明文密码 | Major |
| 不安全的加密 | 使用弱加密算法 | Major |
| 反序列化 | 不安全的反序列化 | Critical |

## 最佳实践

### 1. 质量门禁

```json title="quality-gate.json"
{
  "conditions": [
    {
      "metric": "new_coverage",
      "operator": "LESS_THAN",
      "value": "80"
    },
    {
      "metric": "new_bugs",
      "operator": "GREATER_THAN",
      "value": "0"
    },
    {
      "metric": "new_vulnerabilities",
      "operator": "GREATER_THAN",
      "value": "0"
    },
    {
      "metric": "new_code_smells",
      "operator": "GREATER_THAN",
      "value": "10"
    }
  ]
}
```

### 2. PR 评论自动化

```yaml title="pr-comment.yml"
sonarqube:
  stage: quality
  script:
    - |
      # 获取 SonarQube 分析结果
      RESULT=$(curl -s -u ${SONAR_TOKEN}: \
        "${SONAR_HOST_URL}/api/qualitygates/project_status?projectKey=${CI_PROJECT_KEY}&pullRequest=${CI_MERGE_REQUEST_IID}")
      
      STATUS=$(echo $RESULT | jq -r '.projectStatus.status')
      
      # 在 MR 评论中显示结果
      if [ "$STATUS" == "ERROR" ]; then
        echo "❌ Quality Gate Failed" > mr_comment.md
        curl --request POST \
          --header "PRIVATE-TOKEN: ${GITLAB_TOKEN}" \
          --data-urlencode "body=$(cat mr_comment.md)" \
          "${CI_API_V4_URL}/projects/${CI_PROJECT_ID}/merge_requests/${CI_MERGE_REQUEST_IID}/notes"
      fi
```

### 3. 问题分配

```groovy title="auto-assign.gradle"
sonarqube {
    properties {
        property "sonar.issues.assignees.max", "1"
        property "sonar.newCode.referenceBranch", "main"
    }
}
```

## 总结

SonarQube 是代码质量保障的重要工具：

| 能力 | 说明 |
|---|---|
| 代码质量 | Bug、漏洞、代码异味 |
| 覆盖率 | 单元测试、集成测试覆盖 |
| 质量门禁 | CI/CD 中自动检查 |
| MR 集成 | 在 MR 页面显示问题 |

最佳实践：

1. **在 CI 中集成**：每次 MR 自动扫描
2. **设置质量门禁**：不达标不允许合并
3. **关注新增代码**：避免引入新问题
4. **定期清理**：对老代码制定清理计划

记住：SonarQube 是辅助工具，代码质量的核心还是靠人。
