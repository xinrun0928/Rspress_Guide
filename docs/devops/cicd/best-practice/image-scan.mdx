# 镜像安全扫描

你的容器镜像安全吗？你知道镜像里有多少漏洞吗？

大多数团队 Pull 一个基础镜像，安装几个依赖，打包上线，从未想过这个镜像里藏着多少 CVE 漏洞。直到某天，被安全扫描工具检测出 200+ 高危漏洞。

镜像安全扫描，是容器安全的第一道防线。

## 镜像安全威胁

### 常见威胁

| 威胁类型 | 说明 | 示例 |
|---|---|---|
| **系统漏洞** | 基础镜像的系统包漏洞 | glibc、openssl |
| **应用漏洞** | 运行时依赖的漏洞 | Log4j、Spring4Shell |
| **恶意软件** | 镜像中包含恶意代码 | 挖矿软件、后门 |
| **敏感信息** | 镜像中泄露密钥 | AWS 密钥、密码 |
| **不安全的配置** | 默认配置不安全 | root 用户运行 |

### 漏洞来源

```
基础镜像（OS 包漏洞）
    │
    ├── 语言运行时（Node.js、Python、JVM）
    │
    ├── 依赖库（Log4j、Jackson）
    │
    └── 应用代码（自研代码漏洞）
```

## Trivy 扫描

### 简介

Trivy 是 VMware 开源的容器镜像扫描工具，简单易用，扫描速度快。

### Kubernetes 部署

```yaml title="trivy-operator.yaml"
apiVersion: apps/v1
kind: Deployment
metadata:
  name: trivy-operator
  namespace: security
spec:
  replicas: 1
  selector:
    matchLabels:
      app: trivy-operator
  template:
    spec:
      containers:
      - name: trivy-operator
        image: aquasec/trivy-operator:0.16.0
        env:
        - name: TRIVY_SKIP_DB_UPDATE
          value: "true"
        - name: TRIVY_OFFLINE_SCAN
          value: "true"
```

### GitLab CI 集成

```yaml title=".gitlab-ci.yml"
trivy:
  stage: security
  image:
    name: aquasec/trivy:latest
    entrypoint: [""]
  before_script:
    - trivy version
  script:
    - trivy image --exit-code 1 --severity HIGH,CRITICAL $IMAGE_NAME
  variables:
    IMAGE_NAME: $CI_REGISTRY_IMAGE:$CI_COMMIT_SHORT_SHA
  allow_failure: true
  only:
    - main
    - merge_requests
```

### 扫描报告

```bash
# 基础扫描
trivy image nginx:latest

# 只显示漏洞
trivy image --severity HIGH,CRITICAL nginx:latest

# 生成报告
trivy image --format json --output report.json nginx:latest

# 扫描目录（用于 Dockerfile）
trivy fs --severity HIGH,CRITICAL /path/to/project
```

## Grype 扫描

### 简介

Anchore 推出的镜像扫描工具，数据库更全面，误报率更低。

### 安装与使用

```bash
# 安装
curl -sSfL https://raw.githubusercontent.com/anchore/grype/main/install.sh | sh

# 扫描镜像
grype nginx:latest

# 输出格式
grype nginx:latest -o json > report.json
grype nginx:latest -o cyclonedx > report.cdx
```

### GitLab CI 集成

```yaml title="grype-gitlab.yml"
grype:
  stage: security
  image: anchore/grype:latest
  before_script:
    - grype version
  script:
    - grype $CI_REGISTRY_IMAGE:$CI_COMMIT_SHORT_SHA \
        --severity HIGH,CRITICAL \
        --fail-on HIGH \
        --output json \
        --file /tmp/grype.json
  artifacts:
    reports:
      container_scanning: /tmp/grype.json
    expire_in: 1 week
```

## Snyk 扫描

### 简介

Snyk 是商业化的安全平台，支持容器、代码、依赖安全扫描。

### CLI 使用

```bash
# 安装
npm install -g snyk

# 认证
snyk auth

# 扫描镜像
snyk container test nginx:latest

# 扫描 Dockerfile
snyk container test --file=Dockerfile .
```

### GitLab CI 集成

```yaml title="snyk-gitlab.yml"
snyk:
  stage: security
  image: snyk/snyk:docker
  services:
    - docker:24-dind
  variables:
    SNYK_TOKEN: ${SNYK_TOKEN}
  script:
    - snyk container test $IMAGE_NAME \
        --severity-threshold=high \
        --json-file-output=snyk-report.json
  artifacts:
    paths:
      - snyk-report.json
    expire_in: 1 week
```

## Clair 扫描

### 简介

Quay 推出的开源容器漏洞分析工具，适合自建扫描服务。

### Kubernetes 部署

```yaml title="clair.yaml"
apiVersion: apps/v1
kind: Deployment
metadata:
  name: clair
  namespace: security
spec:
  replicas: 2
  template:
    spec:
      containers:
      - name: clair
        image: quay.io/coreos/clair:v4.8.0
        ports:
        - containerPort: 6060
        - containerPort: 6061
        env:
        - name: POSTGRES_CONNECTION_STRING
          value: "postgres://clair:password@postgres:5432/clair?sslmode=disable"
        - name: CONFIG_PATH
          value: "/etc/clair/config.yaml"
        volumeMounts:
        - name: config
          mountPath: /etc/clair
      volumes:
      - name: config
        configMap:
          name: clair-config
```

### Klar 客户端

```bash
# Klar 是简单的 Clair 客户端
docker run --rm \
  -e CLAIR_URL=http://clair:6060 \
  -e IMAGE_NAME=myapp:latest \
  wearedevops/klar
```

## Harbor 镜像扫描

### 内置扫描

Harbor 内置 Trivy 扫描器：

```yaml title="harbor-scanner.yaml"
apiVersion: goharbor.go/v1alpha2
kind: Scanner
metadata:
  name: trivy
spec:
  provider: trivy
  url: http://trivy-svc:8080
  disabled: false
```

### 自动扫描策略

```yaml title="harbor-policy.yaml"
apiVersion: goharbor.go/v1alpha2
kind: ReplicationPolicy
metadata:
  name: scan-on-push
spec:
  trigger:
    type: event_based
  filters:
  - type: name
    value: "harbor.example.com/**"
  actions:
  - type: scan
```

## Docker Scout

### 简介

Docker 官方推出的镜像安全工具，集成在 Docker Desktop 中。

### 使用

```bash
# 登录 Docker Hub
docker scout cves nginx:latest

# 输出
ocker Scout vulnerabilities for localhost/library/nginx:latest (debian 12)


    ✓ No vulnerable packages detected


    ✓ Image tested for vulnerabilities

    Package threat detection
    ✓ 0 critical, 0 high, 0 medium, 0 low

    Configuration checks
    ✓ 0 checks passed
```

### CI 集成

```yaml title="docker-scout.yml"
docker:scout:
  stage: security
  services:
    - docker:24-dind
  before_script:
    - |
      docker scout login -u ${DOCKER_USERNAME} -p ${DOCKER_PASSWORD}
  script:
    - docker build -t $IMAGE_NAME .
    - docker scout cves --exit-code --ignore-unchanged $IMAGE_NAME
```

## 扫描结果处理

### 失败策略

```yaml title="scan-fail-policy.yml"
# 阻断高危漏洞
trivy:
  stage: security
  script:
    - |
      trivy image --exit-code 1 \
        --severity CRITICAL \
        --ignore-unfixed \
        $IMAGE_NAME
      
      # 如果有 CRITICAL 漏洞则失败
      if [ $? -eq 1 ]; then
        echo "❌ Critical vulnerabilities detected!"
        exit 1
      fi
```

### 忽略漏洞

```yaml title="trivy-ignore.yml"
trivy:
  script:
    - |
      trivy image \
        --severity HIGH,CRITICAL \
        --ignorefile .trivyignore \
        $IMAGE_NAME
```

```text title=".trivyignore"
# 忽略特定漏洞
CVE-2023-28370
CVE-2023-44487
```

### SBOM 生成

```bash
# Syft 生成 SBOM
syft nginx:latest -o cyclonedx-json > sbom.json

# Trivy 使用 SBOM
trivy image --sbom sbom.json nginx:latest
```

## 最佳实践

### 1. 镜像选择

| 实践 | 说明 |
|---|---|
| 使用官方镜像 | 镜像来源可信 |
| 选择 Alpine | 更小的攻击面 |
| 定期更新基础镜像 | 及时修复漏洞 |
| 使用 Distroless | 只包含运行时必要文件 |

### 2. 多层扫描

```yaml title="multi-layer-scan.yml"
stages:
  - build
  - scan
  
scan:dependencies:
  stage: scan
  script:
    - |
      # 扫描依赖
      snyk test --file=package-lock.json
      
scan:dockerfile:
  stage: scan
  script:
    - |
      # 扫描 Dockerfile
      hadolint Dockerfile
      
scan:image:
  stage: scan
  script:
    - |
      # 扫描最终镜像
      trivy image --severity HIGH,CRITICAL $IMAGE_NAME
```

### 3. 质量门禁

```yaml title="quality-gate.yml"
security:gate:
  stage: security
  script:
    - |
      CRITICAL=$(trivy image --severity CRITICAL $IMAGE_NAME 2>/dev/null | grep -c "CRITICAL" || echo "0")
      
      if [ $CRITICAL -gt 0 ]; then
        echo "❌ Found $CRITICAL CRITICAL vulnerabilities"
        exit 1
      fi
      
      echo "✅ Security check passed"
```

## 总结

镜像安全扫描是容器安全的基础：

| 工具 | 特点 | 适用场景 |
|---|---|---|
| Trivy | 轻量、快速、开源 | 快速扫描 |
| Grype | 误报率低 | 精确扫描 |
| Snyk | 功能全面 | 企业级 |
| Clair | 自建服务 | 内网环境 |
| Docker Scout | Docker 官方 | Docker 用户 |

最佳实践：

1. **每次构建都扫描**：CI/CD 中集成
2. **设置质量门禁**：阻断高危漏洞
3. **定期全量扫描**：发现历史漏洞
4. **及时修复**：漏洞不过夜
5. **使用轻量镜像**：减少攻击面

记住：安全扫描不是终点，而是起点。
