# Jenkinsfile 流水线

import { Badge } from '@rspress/core/theme';

<Badge text=" PRINCIPLE 原理类" type="warning" />

2016 年，Jenkins 2.0 发布，Pipeline（流水线）成为核心功能。在这之前，Jenkins 的 Job 配置只能通过 Web UI 完成——步骤难以版本化、难以复用、难以审查。当你想修改构建流程时，必须登录 Jenkins，点开 Job 配置页面，一步步修改，然后保存。

Jenkinsfile 的出现改变了这一切。你可以在代码仓库中创建一个 `Jenkinsfile`，用代码描述整个 CI/CD 流程。提交代码时，顺便提交构建流程——这就是 Pipeline as Code。

## 两种语法：Declarative vs Scripted

Jenkinsfile 支持两种语法：Declarative（声明式）和 Scripted（脚本式）。

| 特性 | Declarative | Scripted |
|---|---|---|
| 语法风格 | YAML-like，结构化 | Groovy 脚本，灵活 |
| 学习曲线 | 平缓 | 较陡 |
| 灵活性 | 固定结构 | 几乎无限 |
| 错误检查 | IDE 支持较好 | 需要运行时检查 |
| 推荐程度 | **首选** | 复杂场景使用 |

### Declarative 语法示例

```groovy
pipeline {
    agent any
    
    environment {
        APP_NAME = 'my-app'
        REGISTRY = 'registry.example.com'
    }
    
    options {
        timeout(time: 1, unit: 'HOURS')
        disableConcurrentBuilds()
    }
    
    stages {
        stage('Checkout') {
            steps {
                echo 'Checking out source code...'
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                echo 'Building application...'
                sh 'mvn clean package -DskipTests=false'
            }
        }
        
        stage('Test') {
            steps {
                echo 'Running tests...'
                sh 'mvn test'
            }
            post {
                always {
                    junit 'target/surefire-reports/*.xml'
                }
            }
        }
        
        stage('Docker Build') {
            steps {
                echo 'Building Docker image...'
                sh '''
                    docker build -t ${REGISTRY}/${APP_NAME}:${BUILD_NUMBER} .
                    docker build -t ${REGISTRY}/${APP_NAME}:latest .
                '''
            }
        }
        
        stage('Deploy') {
            when {
                branch 'main'
            }
            steps {
                echo 'Deploying to production...'
                sh './deploy.sh production'
            }
        }
    }
    
    post {
        always {
            echo 'Pipeline execution completed'
        }
        success {
            echo 'Build and deployment successful!'
        }
        failure {
            echo 'Build or deployment failed!'
        }
    }
}
```

### Scripted 语法示例

```groovy
node('maven') {
    stage('Checkout') {
        checkout scm
    }
    
    stage('Build') {
        def mvnHome = tool 'Maven-3.9'
        sh "${mvnHome}/bin/mvn clean package"
    }
    
    stage('Test') {
        try {
            sh "${mvnHome}/bin/mvn test"
        } finally {
            junit 'target/surefire-reports/*.xml'
        }
    }
    
    if (env.BRANCH_NAME == 'main') {
        stage('Deploy') {
            sh './deploy.sh production'
        }
    }
}
```

## 核心概念

### agent

指定流水线在哪里执行。

```groovy
// 在任何可用的 Agent 上执行
agent any

// 在特定标签的 Agent 上执行
agent { label 'maven' }

// 在 Kubernetes Pod 中执行
agent {
    kubernetes {
        label 'jenkins-agent'
        defaultContainer 'jnlp'
        yaml '''
apiVersion: v1
kind: Pod
spec:
  containers:
    - name: maven
      image: maven:3.9-eclipse-temurin-17
      command: sleep
      args: infinity
'''
    }
}

// 不分配 Agent（用于 post 步骤等）
agent none
```

### stages 与 stage

组织流水线的逻辑单元。

```groovy
stages {
    stage('Build') {
        steps {
            // 具体步骤
        }
    }
    
    stage('Test') {
        steps {
            // 测试步骤
        }
    }
    
    stage('Deploy') {
        steps {
            // 部署步骤
        }
    }
}
```

### steps

每个 stage 中的具体操作。

```groovy
stage('Build') {
    steps {
        // Shell 命令
        sh 'mvn clean package'
        
        // Windows 命令
        bat 'mvn clean package'
        
        // 打印信息
        echo 'Building...'
        
        // 发送通知
        slackSend message: 'Build started'
        
        // 脚本
        script {
            def version = readVersion()
            env.IMAGE_TAG = version
        }
    }
}
```

### environment

定义环境变量。

```groovy
pipeline {
    environment {
        APP_NAME = 'my-app'
        REGISTRY = 'registry.example.com'
        // 凭证引用
        DOCKER_CREDS = credentials('docker-hub-credentials')
    }
    
    stages {
        stage('Build') {
            steps {
                echo "Building ${APP_NAME}"
                sh 'echo $REGISTRY'
            }
        }
    }
}
```

### options

流水线配置选项。

```groovy
pipeline {
    options {
        // 超时控制
        timeout(time: 1, unit: 'HOURS')
        
        // 禁止并发构建
        disableConcurrentBuilds()
        
        // 构建历史保留策略
        buildDiscarder(logRotator(numToKeepStr: '10'))
        
        // 跳过默认 checkout
        skipDefaultCheckout()
        
        // 添加时间戳
        timestamps()
    }
}
```

### when

条件执行。

```groovy
stage('Deploy') {
    when {
        // 只有 main 分支执行
        branch 'main'
        
        // 或者满足多个条件
        allOf {
            branch 'main'
            environment name: 'DEPLOY_ENABLED', value: 'true'
        }
        
        // 或者任意条件
        anyOf {
            branch 'main'
            branch 'release/*'
        }
        
        // 自定义条件
        expression {
            return params.DEPLOY == 'true'
        }
    }
    
    steps {
        sh './deploy.sh'
    }
}
```

## Post 块：构建后处理

Post 定义了流水线或 stage 执行完成后的处理逻辑。

```groovy
pipeline {
    stages {
        stage('Test') {
            steps {
                sh 'mvn test'
            }
            
            post {
                // 总是执行
                always {
                    echo 'Test stage completed'
                }
                
                // 仅成功时执行
                success {
                    echo 'Tests passed!'
                    junit 'target/surefire-reports/*.xml'
                }
                
                // 仅失败时执行
                failure {
                    echo 'Tests failed!'
                    slackSend color: 'danger', message: 'Build failed!'
                }
                
                // 仅不稳定状态时执行
                unstable {
                    echo 'Build is unstable'
                }
                
                // 仅被中断时执行
                aborted {
                    echo 'Build was aborted'
                }
            }
        }
    }
}
```

## 凭证管理

### 在 Jenkins 中配置凭证

1. **Manage Jenkins → Manage Credentials → Add Credentials**

2. **类型选择**：

| 类型 | 用途 |
|---|---|
| Username with password | Git 仓库、服务的用户名密码认证 |
| SSH Username with private key | SSH 密钥认证 |
| Secret text | API Token、密钥等 |
| Username | Git username（password 单独配置） |

### 在流水线中引用凭证

```groovy
pipeline {
    environment {
        // 使用 credentials() 自动解密
        DOCKER_TOKEN = credentials('docker-hub-token')
        SSH_KEY = credentials('ssh-private-key')
    }
    
    stages {
        stage('Docker Login') {
            steps {
                sh '''
                    echo $DOCKER_TOKEN_USR
                    echo $DOCKER_TOKEN_PSW
                    docker login -u $DOCKER_TOKEN_USR -p $DOCKER_TOKEN_PSW
                '''
            }
        }
        
        stage('Deploy with SSH') {
            steps {
                sshagent(credentials: ['ssh-deployment-key']) {
                    sh 'ssh -i /tmp/jenkinssshkey user@server ./deploy.sh'
                }
            }
        }
    }
}
```

## 高级特性

### 矩阵构建（Matrix）

```groovy
pipeline {
    stages {
        stage('Test on Multiple Platforms') {
            matrix {
                axes {
                    axis {
                        name 'PLATFORM'
                        values 'linux', 'windows', 'macos'
                    }
                    axis {
                        name 'JDK'
                        values '11', '17', '21'
                    }
                }
                
                agent { label PLATFORM }
                
                steps {
                    echo "Testing on ${PLATFORM} with JDK ${JDK}"
                    sh "mvn test"
                }
            }
        }
    }
}
```

### 参数化构建

```groovy
pipeline {
    parameters {
        string(name: 'DEPLOY_ENV', defaultValue: 'staging', description: 'Deployment environment')
        choice(name: 'VERSION', choices: ['stable', 'beta', 'alpha'], description: 'Version to deploy')
        booleanParam(name: 'RUN_INTEGRATION_TESTS', defaultValue: true, description: 'Run integration tests')
        password(name: 'API_SECRET', defaultValue: '', description: 'API Secret')
    }
    
    stages {
        stage('Deploy') {
            steps {
                echo "Deploying ${params.VERSION} to ${params.DEPLOY_ENV}"
                sh "./deploy.sh ${params.DEPLOY_ENV} ${params.VERSION}"
            }
        }
    }
}
```

### input 指令：人工审批

```groovy
pipeline {
    stages {
        stage('Deploy Approval') {
            steps {
                input {
                    message 'Deploy to production?'
                    ok 'Approve'
                    submitter 'admin,devops'
                    parameters {
                        choice(name: 'ENVIRONMENT', choices: ['staging', 'production'], description: 'Target environment')
                    }
                }
            }
        }
        
        stage('Deploy') {
            steps {
                echo "Deploying to ${ENVIRONMENT}"
                sh "./deploy.sh ${ENVIRONMENT}"
            }
        }
    }
}
```

### 重试与超时

```groovy
pipeline {
    stages {
        stage('Deploy') {
            steps {
                retry(3) {
                    sh './deploy.sh'
                }
            }
        }
        
        stage('Health Check') {
            steps {
                timeout(time: 5, unit: 'MINUTES') {
                    waitUntil {
                        script {
                            def response = httpRequest 'http://api.example.com/health'
                            return response.status == 200
                        }
                    }
                }
            }
        }
    }
}
```

## 完整示例

```groovy
pipeline {
    agent {
        kubernetes {
            label 'jenkins-agent'
            defaultContainer 'jnlp'
            yaml '''
apiVersion: v1
kind: Pod
spec:
  containers:
    - name: maven
      image: maven:3.9-eclipse-temurin-17
      command: ["sleep"]
      args: ["infinity"]
      resources:
        requests:
          cpu: "500m"
          memory: "1Gi"
        limits:
          cpu: "1000m"
          memory: "2Gi"
    - name: docker
      image: docker:24-dind
      securityContext:
        privileged: true
      volumeMounts:
        - name: docker-socket
          mountPath: /var/run/docker.sock
  volumes:
    - name: docker-socket
      hostPath:
        path: /var/run/docker.sock
'''
        }
    }
    
    environment {
        APP_NAME = 'my-spring-boot-app'
        REGISTRY = 'registry.example.com'
        DOCKER_CREDS = credentials('docker-hub')
    }
    
    options {
        timeout(time: 30, unit: 'MINUTES')
        buildDiscarder(logRotator(numToKeepStr: '10'))
        timestamps()
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                container('maven') {
                    sh 'mvn clean package -DskipTests=false'
                }
            }
        }
        
        stage('Test') {
            steps {
                container('maven') {
                    sh 'mvn test'
                }
            }
            post {
                always {
                    junit 'target/surefire-reports/**/*.xml'
                }
            }
        }
        
        stage('Docker Build') {
            steps {
                container('docker') {
                    sh """
                        docker build -t ${REGISTRY}/${APP_NAME}:${BUILD_NUMBER} .
                        docker build -t ${REGISTRY}/${APP_NAME}:latest .
                        docker login -u ${DOCKER_CREDS_USR} -p ${DOCKER_CREDS_PSW}
                        docker push ${REGISTRY}/${APP_NAME}:${BUILD_NUMBER}
                        docker push ${REGISTRY}/${APP_NAME}:latest
                    """
                }
            }
        }
        
        stage('Deploy to Staging') {
            when {
                anyOf { branch 'main'; branch 'release/*' }
            }
            steps {
                sh './deploy.sh staging'
            }
        }
        
        stage('Deploy to Production') {
            when {
                branch 'main'
            }
            steps {
                input {
                    message 'Deploy to production?'
                    ok 'Deploy'
                }
                sh './deploy.sh production'
            }
        }
    }
    
    post {
        success {
            slackSend channel: '#ci-cd', color: 'good', message: "Build ${BUILD_NUMBER} succeeded: ${env.BUILD_URL}"
        }
        failure {
            slackSend channel: '#ci-cd', color: 'danger', message: "Build ${BUILD_NUMBER} failed: ${env.BUILD_URL}"
        }
    }
}
```

> [!TIP]
> 建议将 Jenkinsfile 纳入版本控制，配合分支保护规则，确保流水线的任何修改都经过 Code Review。
