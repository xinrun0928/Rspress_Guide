# Jenkins 插件生态

Jenkins 的核心只是一个任务调度器——它知道什么时候该做什么，但不知道怎么做。「如何构建 Java 项目」「如何与 Git 集成」「如何部署到 Kubernetes」——这些能力都来自插件。

毫不夸张地说：**插件的质量决定了 Jenkins 的能力上限**。一个配置得当的 Jenkins 实例，配合精选的插件矩阵，可以满足任何复杂的 CI/CD 需求。

## 核心插件详解

### Git 插件

```groovy
// Git 插件在流水线中的使用
pipeline {
    agent any
    
    stages {
        stage('Checkout') {
            steps {
                git(
                    branch: 'main',
                    url: 'https://github.com/example/project.git',
                    credentialsId: 'git-credentials',
                    // 或者使用多仓库
                    userRemoteConfigs: [
                        [
                            url: 'https://github.com/example/frontend.git',
                            credentialsId: 'git-creds'
                        ],
                        [
                            url: 'https://github.com/example/backend.git',
                            credentialsId: 'git-creds'
                        ]
                    ]
                )
            }
        }
    }
}
```

### Docker Pipeline 插件

```groovy
// Docker Pipeline 语法
pipeline {
    agent {
        docker {
            image 'maven:3.9-eclipse-temurin-17'
            label 'docker-enabled'
            args '-v /root/.m2:/root/.m2'
        }
    }
    
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
    }
}

// 多容器场景
pipeline {
    agent {
        docker {
            image 'postgres:15'
            label 'docker-enabled'
        }
    }
    
    stages {
        stage('Integration Test') {
            steps {
                sh '''
                    docker ps
                    # 测试数据库连接
                    psql -h localhost -U postgres -c "SELECT 1"
                '''
            }
        }
    }
}
```

### Kubernetes 插件

```groovy
// Kubernetes Agent 配置
pipeline {
    agent {
        kubernetes {
            label 'jenkins-agent'
            defaultContainer 'jnlp'
            yaml '''
apiVersion: v1
kind: Pod
spec:
  containers:
    - name: maven
      image: maven:3.9-eclipse-temurin-17
      command: ["sleep"]
      args: ["infinity"]
      ttyEnabled: true
    - name: docker
      image: docker:24-dind
      securityContext:
        privileged: true
      env:
        - name: DOCKER_TLS_CERTDIR
          value: ""
'''
            yamlMergeStrategy: merge()
        }
    }
    
    stages {
        stage('Build with Maven') {
            steps {
                container('maven') {
                    sh 'mvn clean package'
                }
            }
        }
        
        stage('Build Docker Image') {
            steps {
                container('docker') {
                    sh 'docker build -t myapp:${BUILD_NUMBER} .'
                }
            }
        }
    }
}
```

## SonarQube 集成

### 安装 SonarQube Scanner 插件

### 配置 SonarQube Server

```bash
# Manage Jenkins → Configure System → SonarQube servers

# 添加 SonarQube Server
# Server URL: http://sonarqube:9000
# Server authentication token: (在 SonarQube 中生成的 token)
```

### 流水线集成

```groovy
pipeline {
    environment {
        SCANNER_HOME = tool 'SonarQubeScanner'
        DOCKER_TAG = "${BRANCH_NAME}-${BUILD_NUMBER}"
    }
    
    stages {
        stage('SonarQube Analysis') {
            steps {
                withSonarQubeEnv('SonarQube') {
                    sh '''
                        ${SCANNER_HOME}/bin/sonar-scanner \
                            -Dsonar.projectKey=my-project \
                            -Dsonar.sources=src \
                            -Dsonar.java.binaries=target/classes \
                            -Dsonar.branch.name=${BRANCH_NAME}
                    '''
                }
            }
        }
        
        stage('Quality Gate') {
            steps {
                timeout(time: 1, unit: 'HOURS') {
                    waitForQualityGate abortPipeline: true
                }
            }
        }
    }
}
```

### SonarQube Quality Gate 配置

```bash
# 在 SonarQube 中创建 Quality Gate
# 建议配置：
# - Coverage on New Code: < 80% → 失败
# - Duplicated Lines on New Code: > 3% → 失败
# - Blocker Issues on New Code: > 0 → 失败
# - Critical Issues on New Code: > 0 → 失败
```

## 权限管理插件

### Role-Based Authorization Strategy

```groovy
// 安装 Role Strategy 插件后
// Manage Jenkins → Configure Global Security → Authorization

// 选择 "Role-Based Strategy"

// 管理 Roles: Manage and Assign Roles
// Global Roles: 基础权限（admin, authenticated）
// Project Roles: 项目级别权限（build, configure, delete）
// Agent Roles: Agent 级别权限
```

```bash
# 使用 Jenkins CLI 配置角色
# 创建全局角色
jenkins-cli.jar assign-role globalRoles authenticated user1

# 创建项目角色
jenkins-cli.jar create-role -name "dev-role" -permission hudson.model.Item.Read
jenkins-cli.jar assign-role -name "dev-role" -roleType project -user user2
```

### 流水线中的权限控制

```groovy
pipeline {
    agent any
    
    options {
        // 只有特定用户可以触发
        buildDiscarder(logRotator(numToKeepStr: '5'))
    }
    
    parameters {
        choice(name: 'ENVIRONMENT', choices: ['dev', 'staging', 'production'], 
               description: 'Select environment')
    }
    
    stages {
        stage('Deploy to Production') {
            when {
                expression { params.ENVIRONMENT == 'production' }
            }
            
            steps {
                // 生产环境部署需要更高权限
                echo 'Deploying to production...'
            }
        }
    }
}
```

## 插件管理最佳实践

### 插件版本管理

```bash
# 定期更新插件（建议每月一次）
# Manage Jenkins → Manage Plugins → Updates

# 更新前建议：
# 1. 在测试环境验证
# 2. 备份 Jenkins 配置
# 3. 阅读插件更新日志
# 4. 检查兼容性
```

### 插件依赖检查

```bash
# 查看插件依赖
# Manage Jenkins → Manage Plugins → Installed

# 点击某个插件，查看 "Dependencies" 标签
# 确保所有依赖插件也保持更新
```

### 插件故障排查

```bash
# 1. 检查插件日志
# Manage Jenkins → System Log → All Log

# 2. 重启 Jenkins（安全重启）
# Manage Jenkins → Reload Configuration from Disk

# 3. 完整的重启
# systemctl restart jenkins
```

> [!TIP]
> 建议使用 Jenkins Configuration as Code (JCasC) 管理插件配置，确保插件版本和配置可以在不同环境间一致部署。
