# Git 常用命令与原理

Git 是每个开发者每天都要用到的工具，但你真的了解它的工作原理吗？`git pull` 和 `git rebase` 有什么区别？merge 和 rebase 各自适合什么场景？当 conflict 发生时，如何处理才能让提交历史保持清晰？

理解 Git 的底层原理，才能用好这些命令。

## Git 核心概念

### 四个区域

```
┌─────────────────────────────────────────────────────────────┐
│                      工作区 (Working Directory)               │
│  你正在编辑的文件                                             │
└─────────────────────────────┬───────────────────────────────┘
                              │ git add
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    暂存区 (Staging Area / Index)             │
│  准备提交的文件快照                                          │
└─────────────────────────────┬───────────────────────────────┘
                              │ git commit
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   本地仓库 (Local Repository)                 │
│  .git 目录，存放所有提交历史                                 │
└─────────────────────────────┬───────────────────────────────┘
                              │ git push
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   远程仓库 (Remote Repository)                 │
│  GitHub / GitLab / Gitee                                    │
└─────────────────────────────────────────────────────────────┘
```

### 三种对象

| 对象 | 说明 | 存储内容 |
|---|---|---|
| **blob** | 文件快照 | 文件内容 |
| **tree** | 目录结构 | 文件名 + blob/tree 引用 |
| **commit** | 提交记录 | tree 引用 + 父 commit 引用 + 作者信息 |

```
commit 对象：
┌─────────────────────────────────────┐
│ commit SHA: abc123...               │
│                                     │
│ tree: refs # tree 对象              │
│ parent: refs # 上一个 commit         │
│ author: xxx                         │
│ committer: xxx                     │
│ message: "feat: add login"        │
└─────────────────────────────────────┘
```

## 基础命令

### 工作区操作

```bash
# 查看工作区状态
git status
# 简略输出
git status -s
# 输出示例：
#  M README.md        # M = 已修改（已跟踪）
# A  new-file.md      # A = 已暂存（新增文件）
# ?? untracked.md     # ?? = 未跟踪的新文件
# D  deleted.md       # D = 已删除

# 查看具体修改
git diff          # 工作区 vs 暂存区
git diff --staged # 暂存区 vs 上次提交
git diff HEAD     # 工作区 vs HEAD

# 暂存文件
git add file.txt           # 暂存单个文件
git add .                 # 暂存所有变更
git add -p                # 交互式暂存（选择性暂存部分修改）

# 撤销操作
git checkout -- file.txt   # 丢弃工作区修改（危险！）
git restore file.txt        # 同上，git 2.23+ 推荐
git reset HEAD file.txt    # 取消暂存（保留工作区修改）
```

### 提交操作

```bash
# 基本提交
git commit -m "feat: add user login"

# 跳过 pre-commit 钩子（不推荐常规使用）
git commit --no-verify -m "quick fix"

# 修改最后一次提交（追加暂存区内容到上一个 commit）
git add forgotten-file.txt
git commit --amend --no-edit
# 等价于：
# 1. 撤销上次的 commit
# 2. 合并暂存区
# 3. 重新 commit

# 查看提交历史
git log
git log --oneline           # 简洁模式
git log --graph --oneline   # 图形化
git log -n 5                # 最近 5 条
git log --author="zhang"   # 按作者筛选
git log --since="2024-01-01"  # 按时间筛选
```

## 分支操作

### 分支基础

```bash
# 创建分支
git branch feature-login

# 切换分支
git checkout feature-login
# 新建并切换（git 2.23+ 推荐）
git switch -c feature-login

# 查看分支
git branch              # 本地分支
git branch -r           # 远程分支
git branch -a           # 所有分支

# 删除分支
git branch -d feature-login   # 安全删除（有未合并内容会失败）
git branch -D feature-login   # 强制删除

# 重命名分支
git branch -m old-name new-name
```

### 合并分支

```bash
# 合并分支
git checkout main
git merge feature-login

# 合并时创建 merge commit（即使 fast-forward）
git merge --no-ff feature-login

# 取消合并（当 merge 产生冲突时）
git merge --abort
```

**Fast-forward vs 三路合并**：

```
Fast-forward（快进）：
main:  A---B---C
                └── feature: D---E
                         │
                         ▼
main:  A---B---C---D---E  ← 直接移动指针

三路合并（no-ff）：
main:  A---B---C
                └── feature: D---E
                         │
                         ▼
main:  A---B---C---F
                \        /
                 D------E
                    ↑
               merge commit
```

### Rebase：变基

`rebase` 把当前分支的提交「重放」到目标分支上，让提交历史变成一条直线。

```bash
# 将 feature 分支变基到 main
git checkout feature-login
git rebase main

# 等价于：
# 1. 找到 feature 和 main 的共同祖先
# 2. 提取 feature 分支的提交
# 3. 在 main 分支上依次应用这些提交
```

**rebase 的工作原理**：

```
变基前：
main:  A---B---C
                └── feature: D---E

变基后：
main:  A---B---C
                └── feature: D'---E'
                            ↑
                      重新生成的提交（D' 基于 C）
```

:::warning 黄金法则
**不要对已经推送到远程的提交执行 rebase！** 如果其他人基于你的旧提交开发，rebase 会导致他们的分支失效。
:::

### Cherry-pick：挑选提交

cherry-pick 把指定的提交「摘取」到当前分支：

```bash
# 将某个提交应用到当前分支
git cherry-pick abc123

# 挑选多个提交
git cherry-pick abc123 def456

# 挑选并继续（如果有冲突需要解决）
git cherry-pick --continue

# 取消 cherry-pick
git cherry-pick --abort
```

## 远程操作

### 基本操作

```bash
# 克隆仓库
git clone https://github.com/example/repo.git
git clone --depth 1    # 浅克隆（只含最新提交）
git clone --branch develop  # 克隆指定分支

# 添加远程仓库
git remote add origin https://github.com/example/repo.git
git remote -v           # 查看远程仓库

# 拉取并合并
git fetch origin        # 只拉取，不合并
git pull                # fetch + merge
git pull --rebase       # fetch + rebase

# 推送
git push origin main
git push -u origin feature-login   # 首次推送，设置上游分支
git push --force-with-lease       # 安全的强制推送
```

### Fetch vs Pull

```
git fetch：
  远程仓库 ──► origin/main
                    ↑
              从远程拉取，但不影响本地 main

git pull (= fetch + merge)：
  远程 main ──► 本地 main
              └── merge origin/main 到本地 main

git pull --rebase (= fetch + rebase)：
  远程 main ──► 本地 main（变基）
```

## Stash：暂存修改

当你在一个分支上工作到一半，突然需要切换到另一个分支处理紧急问题，可以用 stash 暂存工作区修改：

```bash
# 暂存当前修改
git stash
git stash save "WIP: user login feature"

# 查看暂存列表
git stash list
# 输出：
# stash@{0}: WIP: user login feature
# stash@{1}: on main: hotfix: fix bug

# 应用暂存（保留暂存记录）
git stash apply stash@{0}

# 应用暂存（删除暂存记录）
git stash pop

# 查看暂存内容
git stash show
git stash show -p   # 详细 diff

# 删除暂存
git stash drop stash@{0}
git stash clear     # 清空所有 stash
```

## 重写历史

### Reset：重置提交

```bash
# 软重置（保留修改在暂存区）
git reset --soft HEAD~1

# 混合重置（保留修改在工作区）
git reset --mixed HEAD~1
git reset HEAD~1      # 默认是 --mixed

# 硬重置（丢弃所有修改）
git reset --hard HEAD~1
# ⚠️ 危险！会丢失未提交的修改
```

**三种 reset 模式**：

```
软重置 --soft：
  HEAD~1 ──► 指向 上一个提交
  暂存区 ──► 保留
  工作区 ──► 保留

混合重置 --mixed（默认）：
  HEAD~1 ──► 指向 上一个提交
  暂存区 ──► 丢弃
  工作区 ──► 保留

硬重置 --hard：
  HEAD~1 ──► 指向 上一个提交
  暂存区 ──► 丢弃
  工作区 ──► 丢弃
```

### Reflog：找回丢失的提交

```bash
# 查看所有操作的引用日志
git reflog
# 输出：
# abc123 (HEAD -> main) commit: add feature
# def456 HEAD~1 commit: fix bug
# ...

# 找回被 reset 的提交
git checkout abc123
# 或
git reset --hard abc123
```

## 高级操作

### 子模块（Submodule）

```bash
# 添加子模块
git submodule add https://github.com/example/lib.git libs/lib

# 克隆含子模块的仓库
git clone --recursive https://github.com/example/repo.git

# 更新子模块
git submodule update --remote libs/lib

# 在子模块中工作
cd libs/lib
git checkout develop
git push
cd ../..
git add libs/lib
git commit -m "update lib to latest"
```

### 子树（Subtree）

```bash
# 添加子树
git subtree add --prefix=libs/lib https://github.com/example/lib.git main

# 更新子树
git subtree pull --prefix=libs/lib https://github.com/example/lib.git main

# 推送子树修改到上游
git subtree push --prefix=libs/lib https://github.com/example/lib.git main
```

### Interactive Rebase：交互式变基

```bash
# 修改最近 3 个提交
git rebase -i HEAD~3

# 在编辑器中：
pick abc123 commit 1
pick def456 commit 2
pick ghi789 commit 3

# 可以修改为：
pick abc123 commit 1
squash def456 commit 2   # 合并到上一个
pick ghi789 commit 3
reword ghi789 commit 3    # 修改提交信息

# 命令说明：
# pick - 保留提交
# reword - 修改提交信息
# edit - 暂停，可以修改提交
# squash - 合并到上一个提交
# drop - 删除提交
```

## 解决冲突

### 冲突标记

```bash
# 合并或变基时产生冲突
<<<<<<< HEAD
const name = 'Alice';
=======
const name = 'Bob';
>>>>>>> feature-branch
```

### 解决冲突

```bash
# 1. 编辑文件，选择保留的内容
# 2. 删除冲突标记

# 3. 标记为已解决
git add file.txt

# 4. 完成 rebase 或 merge
git rebase --continue
git merge --continue

# 或者放弃
git rebase --abort
git merge --abort
```

### 常见策略

```bash
# 保留当前分支的修改
git checkout --ours file.txt

# 保留合并分支的修改
git checkout --theirs file.txt

# 接受双方修改（冲突标记全部保留）
git checkout --merge file.txt

# 使用 mergetool 可视化解决
git mergetool
```

## Git 别名与配置

### 常用别名

```bash
# 命令行配置
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.last 'log -1 HEAD'

# 图形化日志
git config --global alias.lg "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"
```

### 常用配置

```bash
# 用户信息
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# 记住凭证（15 分钟）
git config --global credential.helper cache

# 默认分支
git config --global init.defaultBranch main

# 拉取策略
git config --global pull.rebase false   # merge
git config --global pull.rebase true    # rebase

# 中文文件名支持
git config --global core.quotepath false
```

## 总结

Git 命令虽多，但核心就几个概念：

1. **四个区域**：工作区、暂存区、本地仓库、远程仓库
2. **三个对象**：blob（文件）、tree（目录）、commit（提交）
3. **merge vs rebase**：merge 保留历史，rebase 简化历史
4. **stash**：临时保存工作区修改
5. **reset**：调整 HEAD 位置，改变提交历史
6. **reflog**：找回丢失的提交

理解原理，比死记硬背命令更重要。
