# GitHub Actions

import { Badge } from '@rspress/core/theme';

<Badge text=" PRINCIPLE 原理类" type="warning" />

2019 年 GitHub Actions 正式发布，彼时很多人质疑：GitHub 不是已经有很多 CI/CD 集成方案了吗？Travis CI、CircleCI、Jenkins——为什么还需要 GitHub 自家的工具？

答案在于**生态系统**。Actions 市场让任何人都能创建和分享可复用的工作流组件。你可以找到一个 Action 来构建 Docker 镜像、部署到 AWS、甚至发一条 Slack 消息——而不需要自己写脚本。

更重要的是，对于开源项目来说，GitHub Actions 是**免费**的。每个月有大量的免费构建分钟数，对于大多数开源项目来说绰绰有余。

## 核心概念

### Workflow

Workflow（工作流）是 GitHub Actions 的顶层概念。一个仓库可以有多个 Workflow，每个 Workflow 由一个或多个 Job 组成。

**存放位置**：`.github/workflows/*.yml` 或 `.github/workflows/*.yaml`

### Job

Job 是一组在相同 Runner 上执行的 Steps。每个 Job 可以包含多个 Step。

### Step

Step 是 Job 中的最小执行单元，可以是 Shell 命令或 Action。

### Action

Action 是可复用的工作流单元。你可以使用市场中的 Action，也可以自己创建。

### Runner

Runner 是执行 Workflow 的服务器。GitHub 提供 Ubuntu、Windows、macOS 的托管 Runner，也可以使用自托管 Runner。

## Workflow 语法

### 基本结构

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

env:
  APP_NAME: my-app
  NODE_VERSION: '20'

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Build
        run: npm run build
      
      - name: Test
        run: npm test
      
      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: build-artifacts
          path: dist/
```

### trigger 触发条件

```yaml
on:
  # 1. Push 触发
  push:
    branches:
      - main
      - 'releases/**'
    tags:
      - 'v*'
    paths:
      - 'src/**'
      - '*.js'
    paths-ignore:
      - 'docs/**'
  
  # 2. Pull Request 触发
  pull_request:
    types:
      - opened
      - synchronize
      - closed
    branches:
      - main
  
  # 3. 定时触发
  schedule:
    - cron: '0 2 * * *'  # 每天凌晨 2 点
    - cron: '0 */4 * * *'  # 每 4 小时
  
  # 4. 手动触发
  workflow_dispatch:
    inputs:
      environment:
        description: 'Deployment environment'
        required: true
        default: 'staging'
        type: choice
        options:
          - staging
          - production
  
  # 5. 其他事件
  release:
    types: [published]
  issues:
    types: [opened, closed]
  issue_comment:
    types: [created, edited]
```

### jobs

```yaml
jobs:
  # 1. 单 job
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: echo "Building..."
  
  # 2. 多 job（并行执行）
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm run build
  
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm test
  
  # 3. job 依赖
  build:
    runs-on: ubuntu-latest
    outputs:
      version: ${{ steps.version.outputs.number }}
    steps:
      - id: version
        run: echo "number=1.0.$GITHUB_RUN_NUMBER" >> $GITHUB_OUTPUT
  
  test:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - run: echo "Testing version ${{ needs.build.outputs.version }}"
```

### 环境与 secrets

```yaml
jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: production
      url: https://example.com
    
    steps:
      - name: Deploy
        env:
          API_KEY: ${{ secrets.API_KEY }}
          DB_HOST: ${{ secrets.DB_HOST }}
        run: ./deploy.sh production
```

### 条件执行

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Build for PR
        if: github.event_name == 'pull_request'
        run: npm run build:pr
      
      - name: Build for Release
        if: github.event_name == 'release'
        run: npm run build:release
      
      - name: Conditional deploy
        if: github.ref == 'refs/heads/main' && github.event_name == 'push'
        run: ./deploy.sh
```

## 常用 Actions

### actions/checkout

```yaml
- uses: actions/checkout@v4
  with:
    fetch-depth: 0      # 获取完整历史
    lfs: true           # 启用 Git LFS
    token: ${{ secrets.GITHUB_TOKEN }}  # 自动授权
```

### actions/setup-node

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: '20'
    cache: 'npm'        # 自动缓存 node_modules
    # cache: 'yarn'
```

### actions/setup-java

```yaml
- uses: actions/setup-java@v4
  with:
    distribution: 'temurin'
    java-version: '17'
    cache: 'maven'      # 自动缓存 .m2/repository
```

### actions/upload-artifact

```yaml
- uses: actions/upload-artifact@v4
  with:
    name: build-artifacts
    path: dist/
    retention-days: 7
    
- uses: actions/upload-artifact@v4
  if: failure()
  with:
    name: test-results
    path: test-results/
```

### actions/download-artifact

```yaml
- uses: actions/download-artifact@v4
  with:
    name: build-artifacts
    path: dist/
```

### actions/cache

```yaml
- uses: actions/cache@v4
  with:
    path: |
      ~/.npm
      ~/.cache/pip
    key: ${{ runner.os }}-deps-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-deps-
```

## 完整 CI/CD 示例

### Node.js 应用

```yaml
name: Node.js CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:
  NODE_VERSION: '20'
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run ESLint
        run: npm run lint

  test:
    runs-on: ubuntu-latest
    needs: lint
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_DB: test
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432
    env:
      DATABASE_URL: postgres://test:test@localhost:5432/test
      
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test
      
      - name: Upload coverage
        uses: codecov/codecov-action@v4
        with:
          token: ${{ secrets.CODECOV_TOKEN }}

  build:
    runs-on: ubuntu-latest
    needs: test
    permissions:
      contents: read
      packages: write
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Docker Buildx
        uses: docker/setup-buildx-action@v3
      
      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy:
    runs-on: ubuntu-latest
    needs: build
    if: github.ref == 'refs/heads/main'
    environment:
      name: production
      url: https://example.com
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Deploy to production
        env:
          REGISTRY: ${{ env.REGISTRY }}
          IMAGE: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
        run: |
          kubectl set image deployment/app app=${{ env.IMAGE }}
          kubectl rollout status deployment/app
```

### Docker 多平台构建

```yaml
name: Multi-platform Docker Build

on:
  push:
    branches: [main]
    tags:
      - 'v*'

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        platform: [linux/amd64, linux/arm64]
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up QEMU
        uses: docker/setup-qemu-action@v3
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3
      
      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          platforms: ${{ matrix.platform }}
          push: true
          tags: ghcr.io/${{ github.repository }}:${{ github.sha }}
```

## 复用工作流

### 使用 reusable workflow

```yaml
# .github/workflows/reusable-deploy.yml
name: Reusable Deploy Workflow

on:
  workflow_call:
    inputs:
      environment:
        required: true
        type: string
    secrets:
      DEPLOY_TOKEN:
        required: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: ${{ inputs.environment }}
    steps:
      - uses: actions/checkout@v4
      
      - name: Deploy
        env:
          TOKEN: ${{ secrets.DEPLOY_TOKEN }}
        run: ./deploy.sh ${{ inputs.environment }}
```

```yaml
# .github/workflows/production.yml
name: Production Deployment

on:
  push:
    branches: [main]

jobs:
  deploy:
    uses: ./.github/workflows/reusable-deploy.yml
    with:
      environment: production
    secrets:
      DEPLOY_TOKEN: ${{ secrets.DEPLOY_TOKEN }}
```

### Composite Action

```yaml
# .github/actions/my-action/action.yml
name: 'My Composite Action'
description: 'A reusable action'
inputs:
  version:
    description: 'Version to use'
    required: true
    default: 'latest'
outputs:
  result:
    description: 'The result'
runs:
  using: composite
  steps:
    - shell: bash
      run: |
        echo "Version: ${{ inputs.version }}"
        echo "result=success" >> $GITHUB_OUTPUT
    - shell: bash
      run: ./script.sh
```

## 最佳实践

### 1. 使用缓存加速

```yaml
- uses: actions/cache@v4
  with:
    path: |
      ~/.npm
      node_modules
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
```

### 2. 避免暴露敏感信息

```yaml
# 正确：使用 secrets
env:
  API_KEY: ${{ secrets.API_KEY }}

# 错误：硬编码
env:
  API_KEY: 'sk-live-xxxxxxx'
```

### 3. 使用超时限制

```yaml
jobs:
  build:
    timeout-minutes: 30  # 防止无限运行的构建
    runs-on: ubuntu-latest
```

### 4. 充分利用并发控制

```yaml
jobs:
  deploy:
    runs-on: ubuntu-latest
    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
      cancel-in-progress: true  # 取消同一分支的重复运行
```

> [!TIP]
> GitHub Actions 的 Actions 市场是最大的财富。在写自己的脚本之前，先去市场看看有没有现成的 Action 可以使用。
