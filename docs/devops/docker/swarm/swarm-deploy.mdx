# Swarm 服务部署

Docker Swarm 的服务（Service）是定义和管理容器化应用的核心概念。本篇介绍如何创建、配置和管理 Swarm 服务。

## 服务基本操作

### 创建服务

```bash
# 创建一个简单的 nginx 服务
docker service create \
  --name web \
  --replicas 3 \
  --publish 80:80 \
  nginx:alpine
```

### 查看服务

```bash
# 列出所有服务
docker service ls

# 查看服务详情
docker service inspect web

# 查看服务任务分布
docker service ps web

# 查看服务日志
docker service logs web -f
```

### 更新服务

```bash
# 更新镜像
docker service update \
  --image nginx:1.25 \
  web

# 更新副本数
docker service scale web=5

# 更新环境变量
docker service update \
  --env-add NODE_ENV=production \
  --env-rm DEBUG \
  api

# 更新端口
docker service update \
  --publish-rm 8080:80 \
  --publish-add 8080:8080 \
  api
```

### 删除服务

```bash
docker service rm web
```

## 服务副本模式

### Replicated 模式

默认模式，运行指定数量的副本：

```yaml
services:
  web:
    image: nginx:alpine
    deploy:
      replicas: 5
```

```bash
# 扩缩容
docker service scale web=10
docker service scale web=2
```

### Global 模式

每个节点运行一个容器：

```yaml
services:
  monitoring-agent:
    image: myagent:latest
    deploy:
      mode: global
```

```bash
# 运行后，每个节点都会有一个 monitoring-agent 容器
```

## 滚动更新

### 配置滚动更新

```yaml title="docker-compose.yml"
services:
  api:
    image: myapi:v1.0.0
    deploy:
      replicas: 4
      update_config:
        parallelism: 1        # 同时更新 1 个
        delay: 10s            # 更新间隔
        failure_action: rollback  # 失败时回滚
        monitor: 5s           # 监控窗口
        max_failure_ratio: 0.1  # 允许 10% 失败
      rollback_config:
        parallelism: 1
        delay: 5s
```

### 执行滚动更新

```bash
# 更新镜像版本
docker service update \
  --image myapi:v2.0.0 \
  myapi

# 手动回滚
docker service rollback myapi

# 查看更新历史
docker service inspect --pretty myapi | grep -A 5 "Update status"
```

### 滚动更新流程

```
旧版本 (4 replicas)  ──▶  1. 启动 1 个新容器
                            │
                            ▼
                       2. 等待健康
                            │
                            ▼
                       3. 停止 1 个旧容器
                            │
                            ▼
                       4. 重复直到全部更新
```

## 健康检查集成

### 在 Dockerfile 中定义

```dockerfile
FROM node:18-alpine
HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1
```

### 在 Compose 中定义

```yaml
services:
  api:
    image: myapi:latest
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 3s
      retries: 3
      start_period: 10s
    deploy:
      replicas: 3
      update_config:
        monitor: 10s           # 等待健康检查
```

### 健康检查与更新

当健康检查与滚动更新配合时：

```yaml
update_config:
  monitor: 10s  # 在标记失败前等待健康检查通过的时间
```

## 资源限制

### CPU 和内存限制

```yaml
services:
  api:
    image: myapi:latest
    deploy:
      replicas: 3
      resources:
        limits:
          cpus: "1.0"
          memory: 512M
        reservations:
          cpus: "0.5"
          memory: 256M
```

```bash
# CLI 方式
docker service create \
  --name api \
  --limit-cpu 1 \
  --limit-memory 512M \
  --reserve-cpu 0.5 \
  --reserve-memory 256M \
  myapi:latest
```

## 网络配置

### overlay 网络

```yaml
services:
  web:
    image: nginx:alpine
    networks:
      - frontend
    ports:
      - "80:80"

  api:
    image: myapi:latest
    networks:
      - frontend
      - backend
    depends_on:
      - db

  db:
    image: postgres:15-alpine
    networks:
      - backend

networks:
  frontend:
    driver: overlay
  backend:
    driver: overlay
```

### 创建自定义 overlay 网络

```bash
# 创建 overlay 网络
docker network create \
  --driver overlay \
  --attachable \
  myoverlay

# 查看网络
docker network ls
```

:::tip attachable 标志
如果需要非 Swarm 服务（如普通容器）也能加入 overlay 网络，使用 `--attachable` 标志。
:::

## 持久化存储

### 使用 volumes

```yaml
services:
  db:
    image: postgres:15-alpine
    volumes:
      - db-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=secret
    deploy:
      placement:
        constraints:
          - "node.role==worker"

volumes:
  db-data:
```

### 约束到特定节点

```yaml
services:
  storage:
    image: minio:latest
    volumes:
      - minio-data:/data
    deploy:
      placement:
        constraints:
          - "node.hostname==storage-node-1"
```

## 完整示例：多层应用

```yaml title="docker-compose.yml"
version: "3.8"

services:
  frontend:
    image: nginx:alpine
    deploy:
      replicas: 3
      update_config:
        parallelism: 1
        delay: 10s
      restart_policy:
        condition: on-failure
    ports:
      - "80:80"
    networks:
      - webnet
    depends_on:
      - api

  api:
    image: myapi:latest
    deploy:
      replicas: 4
      update_config:
        parallelism: 2
        delay: 15s
        monitor: 10s
        failure_action: rollback
      resources:
        limits:
          cpus: "1.0"
          memory: 512M
    environment:
      - DB_HOST=db
      - REDIS_HOST=redis
    networks:
      - webnet
      - backend
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started

  db:
    image: postgres:15-alpine
    volumes:
      - pgdata:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=myapp
      - POSTGRES_USER=admin
      - POSTGRES_PASSWORD_FILE=/run/secrets/db_password
    secrets:
      - db_password
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U admin"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - backend
    deploy:
      placement:
        constraints:
          - "node.role==worker"

  redis:
    image: redis:7-alpine
    volumes:
      - redis-data:/data
    networks:
      - backend
    deploy:
      replicas: 1

networks:
  webnet:
    driver: overlay
  backend:
    driver: overlay

volumes:
  pgdata:
  redis-data:

secrets:
  db_password:
    file: ./secrets/db_password.txt
```

## 服务部署命令

```bash
# 部署服务栈
docker stack deploy -c docker-compose.yml myapp

# 查看栈状态
docker stack services myapp

# 查看栈任务
docker stack ps myapp

# 查看栈配置
docker stack config myapp

# 移除栈
docker stack rm myapp
```

## 调试技巧

```bash
# 查看服务日志
docker service logs myapi -f --tail 100

# 查看容器日志（跨节点）
docker ps
docker logs container-id

# 强制重新调度
docker service update --force myapi

# 节点排空（迁移容器）
docker node update --availability drain node-1
```

## 总结

Swarm 服务部署的关键点：

- **副本模式**：`replicated` vs `global`
- **滚动更新**：配置 `parallelism`、`delay`、`monitor`
- **健康检查**：与更新监控配合
- **资源限制**：CPU、内存、GPU
- **网络隔离**：overlay 网络分段
- **数据持久化**：volumes + 节点约束

正确配置这些选项，可以实现高可用、可扩展的容器化服务。
