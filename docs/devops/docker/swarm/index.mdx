# Docker Swarm

单机 Docker 能做的事有限。当需要管理多个宿主机上的数百个容器、进行滚动更新、故障自动恢复时，就需要容器编排工具。

Docker Swarm 是 Docker 内置的原生编排方案，学习曲线平缓，适合中小规模容器集群。

## Docker Swarm 简介

Docker Swarm 是 Docker Engine 内置的集群管理和容器编排功能。相比 Kubernetes，Swarm 更轻量、配置更简单。

```
┌─────────────────────────────────────────────────────┐
│                  Docker Swarm 集群                    │
├─────────────────────────────────────────────────────┤
│                                                      │
│  ┌──────────────────┐   ┌──────────────────┐        │
│  │   Manager Node    │   │   Manager Node  │        │
│  │   (Leader)        │   │                 │        │
│  │   管理集群         │   │   管理集群       │        │
│  │   调度任务         │   │                 │        │
│  └──────────────────┘   └──────────────────┘        │
│          ▲                       ▲                    │
│          │        Raft          │                    │
│          │       Consensus       │                    │
│          ▼                       ▼                    │
│  ┌──────────────────┐   ┌──────────────────┐        │
│  │   Worker Node     │   │   Worker Node     │        │
│  │   运行容器         │   │   运行容器        │        │
│  └──────────────────┘   └──────────────────┘        │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## Swarm vs Kubernetes

| 特性 | Docker Swarm | Kubernetes |
|---|---|---|
| 学习曲线 | 平缓 | 陡峭 |
| 复杂度 | 低 | 高 |
| 部署难度 | 简单（内置） | 复杂 |
| 功能完整性 | 基础 | 完整 |
| 生态 | 一般 | 丰富 |
| 适用规模 | 中小规模 | 大规模 |
| YAML 配置 | docker-compose.yml | k8s yaml |
| 滚动更新 | 支持 | 支持 |
| 自动扩缩容 | 支持（有限） | 支持 |

## Swarm 架构

### Manager 与 Worker

| 节点类型 | 功能 | 数量建议 |
|---|---|---|
| **Manager** | 集群管理、调度、状态存储 | 奇数（1/3/5） |
| **Worker** | 运行容器 | 按需 |

### Raft 一致性

Swarm Manager 使用 Raft 共识协议保证状态一致性：

- **奇数原则**：需要多数节点（n/2+1）同意才能达成共识
- **3 节点建议**：生产环境至少 3 个 Manager
- **脑裂防护**：网络分区时，只有多数派能继续工作

### 工作节点发现

```bash
# 查看加入命令
docker swarm join-token worker

# 输出
docker swarm join --token SWMTKN-1-xxx worker:2377

# 查看 Manager 加入命令
docker swarm join-token manager
```

## 初始化 Swarm

```bash
# 初始化 Swarm（当前机器成为 Manager）
docker swarm init

# 指定监听地址（多网卡机器）
docker swarm init --advertise-addr 192.168.1.100

# 查看 Swarm 状态
docker info | grep Swarm

# 查看节点列表
docker node ls
```

## 部署服务

### 基本服务部署

```bash
# 部署服务
docker service create \
  --name web \
  --replicas 3 \
  --publish 80:80 \
  nginx:alpine

# 查看服务
docker service ls
docker service ps web

# 扩缩容
docker service scale web=5
docker service scale web=2
```

### docker stack 部署

使用 docker-compose.yml 部署服务栈：

```yaml title="docker-compose.yml"
version: "3.8"

services:
  web:
    image: nginx:alpine
    deploy:
      replicas: 3
      placement:
        constraints:
          - "node.role==worker"
      resources:
        limits:
          cpus: "0.5"
          memory: 256M
      update_config:
        parallelism: 1
        delay: 10s
        failure_action: rollback
      restart_policy:
        condition: on-failure
    ports:
      - "80:80"

  api:
    image: myapi:latest
    deploy:
      replicas: 2
      endpoint_mode: dnsrr
    environment:
      - NODE_ENV=production
```

```bash
# 部署 stack
docker stack deploy -c docker-compose.yml myapp

# 查看 stack
docker stack ls
docker stack ps myapp

# 查看服务
docker stack services myapp

# 移除 stack
docker stack rm myapp
```

## 滚动更新

### 配置滚动更新

```yaml
services:
  api:
    image: myapi:v1.0.0
    deploy:
      replicas: 4
      update_config:
        parallelism: 1          # 同时更新 1 个容器
        delay: 10s              # 更新间隔
        failure_action: rollback # 失败时回滚
        monitor: 5s             # 监控时间
        max_failure_ratio: 0.1  # 允许 10% 失败
      rollback_config:
        parallelism: 1
        delay: 5s
```

### 执行滚动更新

```bash
# 更新镜像
docker service update \
  --image myapi:v2.0.0 \
  myapp_api

# 查看更新进度
docker service ps myapp_api

# 手动回滚
docker service rollback myapp_api
```

## 故障恢复

### 自动恢复机制

```yaml
services:
  api:
    image: myapi:latest
    deploy:
      replicas: 3
      restart_policy:
        condition: on-failure   # 失败时重启
        delay: 5s              # 重启延迟
        max_attempts: 3        # 最大尝试次数
        window: 120s           # 评估窗口
```

### 健康检查集成

```yaml
services:
  api:
    image: myapi:latest
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 3s
      retries: 3
    deploy:
      replicas: 3
      update_config:
        monitor: 10s           # 等待健康检查通过
```

## 网络配置

### Overlay 网络

```yaml
services:
  web:
    image: nginx:alpine
    networks:
      - frontend
    ports:
      - "80:80"

  api:
    image: myapi:latest
    networks:
      - frontend
      - backend

networks:
  frontend:
    driver: overlay
  backend:
    driver: overlay
    internal: true  # 外部无法访问
```

### Ingress 网络

Swarm 自动创建 `ingress` 网络用于负载均衡：

```
外部请求
    │
    ▼
Ingress Network (IPVS 负载均衡)
    │
    ├──▶ Container 1
    ├──▶ Container 2
    └──▶ Container 3
```

## 总结

Docker Swarm 适合：

- 小到中型规模集群（几个到几十个节点）
- 已有 Docker Compose 项目迁移
- 需要快速上手容器编排
- 不愿运维复杂的 Kubernetes 集群

主要功能：

- **集群管理**：节点加入/离开、自动故障恢复
- **服务编排**：声明式部署、滚动更新、回滚
- **负载均衡**：内置 Ingress 网络、DNS 轮询
- **安全**：TLS 加密通信、证书自动更新

虽然功能不如 Kubernetes 完整，但 Swarm 的简单易用使其在特定场景下仍是很好的选择。
