# Docker 容器化指南

凌晨 2 点，你正准备上线新功能，需要在服务器上部署一套 Java 环境。安装 JDK、配置环境变量、解决依赖冲突……折腾了半小时，终于让服务跑起来。但下一个问题来了：测试环境和生产环境的配置不一致。这种「在我机器上能跑」的困境，你一定不陌生。

Docker 正是为了解决这些问题而生的。它让你把应用连同整个运行环境一起打包成一个「镜像」，在任何装了 Docker 的机器上都能一键运行，分毫不差。

## Docker 能做什么

Docker 是一个开源的容器化平台，它将应用及其依赖打包成标准化的单元——**容器（Container）**。容器不同于虚拟机，它共享宿主机的内核，启动更快、资源占用更少、可移植性更强。

想象一下：以前部署一个微服务，需要在服务器上装 JDK、配置 Tomcat、部署 War 包，步骤繁琐且容易出错。有了 Docker，你只需要一个 `Dockerfile` 定义构建步骤，然后 `docker build` 生成镜像，`docker run` 启动容器，整个过程几分钟内完成，且在任何环境中表现一致。

## Docker 生态概览

Docker 生态系统包含多个核心组件：

| 组件 | 说明 |
|---|---|
| **Docker Engine** | 核心运行时，负责构建和运行容器 |
| **Dockerfile** | 镜像构建脚本，定义容器的构建步骤 |
| **Docker Compose** | 多容器应用的编排工具 |
| **Docker Hub** | 公共镜像仓库，托管官方镜像 |
| **Docker Swarm** | 原生容器编排和集群管理工具 |
| **Harbor** | 企业级私有镜像仓库，支持镜像扫描和签名 |

## 本章节学习路径

本章节按照学习曲线循序渐进，分为三大模块：

**Docker 基础** — 从零开始理解容器化

这一模块面向 Docker 初学者，覆盖容器与虚拟机的本质区别、镜像与容器的核心概念、Dockerfile 语法基础、数据持久化与网络通信、Docker Compose 使用，以及私有镜像仓库的搭建。学完这一模块，你能够独立完成日常开发中的容器化任务。

**Docker 进阶** — 深入容器运行机制

这一模块面向有一定 Docker 基础的开发者，深入探讨容器间通信原理、存储驱动对比与选型、资源限制与 cgroups 机制、容器安全加固、日志与健康检查、性能调优，以及构建缓存优化。学完这一模块，你能够排查生产环境中的容器问题，并进行针对性优化。

**Docker Swarm** — 容器编排与集群管理

这一模块介绍 Docker 原生的编排方案，包括 Swarm 的架构原理、服务部署与滚动更新、负载均衡与 DNS 服务发现。学完这一模块，你能够使用 Docker Swarm 部署和管理生产级容器集群。
