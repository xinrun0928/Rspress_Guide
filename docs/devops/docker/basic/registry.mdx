# Docker Registry

Docker 镜像是容器化应用的分发单元，而 **Registry（镜像仓库）**就是存储和分发镜像的服务。

## Docker Registry 架构

Docker Registry 是无状态的存储服务，核心功能是存储镜像的各层数据，并提供 HTTP API 供客户端 push 和 pull。

```
┌──────────────────────────────────────────────────────┐
│                 Docker Registry                       │
├──────────────────────────────────────────────────────┤
│                                                      │
│  ┌────────────────────────────────────────────────┐ │
│  │              Registry API (HTTP)                 │ │
│  │  /v2/                   # API 端点               │ │
│  │  /v2/library/nginx/blobs/  # 镜像层             │ │
│  │  /v2/library/nginx/manifests/ # 镜像清单        │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│  ┌────────────────────────────────────────────────┐ │
│  │              Storage Layer                       │ │
│  │  /docker/registry/v2/blobs/     # 镜像层数据     │ │
│  │  /docker/registry/v2/repositories/ # 仓库元数据  │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
└──────────────────────────────────────────────────────┘
         ▲ push                    ▼ pull
         │                         │
┌─────────────────┐        ┌─────────────────┐
│   Docker CLI    │        │   Docker CLI    │
│   docker push   │        │   docker pull   │
└─────────────────┘        └─────────────────┘
```

## 公共 Registry vs 私有 Registry

| 类型 | 示例 | 特点 |
|---|---|---|
| 公共 | Docker Hub、Quay.io、Google GCR | 开箱即用，托管在云端 |
| 私有 | Docker Registry、Harbor、阿里云 ACr | 数据自主可控，适合企业内网 |

## Docker Hub 使用

### 拉取镜像

```bash
# 拉取官方镜像（默认从 Docker Hub）
docker pull nginx:latest

# 拉取用户镜像
docker pull username/repository:tag

# 拉取特定版本
docker pull node:18.17.0-alpine

# 从其他 Registry 拉取
docker pull gcr.io/google-containers/pause:3.0
```

### 推送镜像

```bash
# 登录 Docker Hub
docker login

# 标记镜像
docker tag myapp:latest myusername/myapp:latest

# 推送镜像
docker push myusername/myapp:latest
```

## 部署私有 Registry

### 快速启动

```bash
# 运行 Registry 容器
docker run -d \
  --name registry \
  --restart=always \
  -p 5000:5000 \
  -v registry-data:/var/lib/registry \
  registry:2
```

### 推送和拉取

```bash
# 标记镜像
docker tag myapp:latest localhost:5000/myapp:latest

# 推送到本地 Registry
docker push localhost:5000/myapp:latest

# 从本地 Registry 拉取
docker pull localhost:5000/myapp:latest
```

### 验证 Registry

```bash
# 查看仓库列表
curl http://localhost:5000/v2/_catalog

# 查看镜像标签
curl http://localhost:5000/v2/myapp/tags/list
```

## 配置认证

Registry 支持 Basic Auth 认证：

### 创建认证文件

```bash
# 创建密码文件目录
mkdir -p /opt/registry/auth

# 创建用户（-c 创建新文件，首次创建必须）
docker run --rm \
  --entrypoint htpasswd \
  httpd:2 \
  -Bbn admin mypassword > /opt/registry/auth/htpasswd
```

### 启动带认证的 Registry

```bash
docker run -d \
  --name registry \
  --restart=always \
  -p 5000:5000 \
  -v /opt/registry/data:/var/lib/registry \
  -v /opt/registry/auth:/auth \
  -e "REGISTRY_AUTH=htpasswd" \
  -e "REGISTRY_AUTH_HTPASSWD_REALM=Registry Realm" \
  -e "REGISTRY_AUTH_HTPASSWD_PATH=/auth/htpasswd" \
  registry:2
```

### 使用认证

```bash
# 登录
docker login localhost:5000 -u admin

# 之后可以正常 push/pull
docker tag myapp:latest localhost:5000/myapp:latest
docker push localhost:5000/myapp:latest
```

## TLS 配置

生产环境中应配置 HTTPS：

### 使用 Let's Encrypt

```bash
# 使用 nginx 反向代理 + Let's Encrypt
docker run -d \
  --name registry-proxy \
  --restart=always \
  -p 443:443 \
  -v /opt/certs:/etc/nginx/certs:ro \
  -v /opt/registry/data:/var/lib/registry \
  nginx:alpine
```

### 简化版（开发/测试）

如果使用 `localhost`，可以通过配置 Docker 允许不安全 Registry：

```json title="/etc/docker/daemon.json"
{
  "insecure-registries": ["registry.example.com:5000"]
}
```

```bash
# 重启 Docker
sudo systemctl restart docker
```

:::warning 安全警告
`insecure-registries` 仅用于开发和测试环境。生产环境必须使用 HTTPS。
:::

## 镜像清理与维护

### 垃圾回收

Registry 会积累未引用的层，需要定期清理：

```bash
# 启用垃圾回收
docker run -d \
  --name registry \
  -p 5000:5000 \
  registry:2

# 执行垃圾回收（进入容器执行）
docker exec registry garbage-collect /etc/docker/registry/config.yml

# 删除已停用的仓库
docker exec registry registry garbage-collect \
  -m /etc/docker/registry/config.yml \
  /etc/docker/registry/config.yml
```

### 清理脚本

```bash title="cleanup-registry.sh"
#!/bin/bash
# 停止 Registry
docker stop registry

# 运行垃圾回收
docker run --rm \
  -v registry-data:/var/lib/registry \
  -v $(pwd)/config.yml:/etc/docker/registry/config.yml \
  registry:2 \
  garbage-collect /etc/docker/registry/config.yml

# 重新启动 Registry
docker start registry
```

## Registry API 常用端点

```bash
# API 版本信息
curl http://localhost:5000/v2/

# 列出所有仓库
curl http://localhost:5000/v2/_catalog

# 列出镜像的所有标签
curl http://localhost:5000/v2/myapp/tags/list

# 获取镜像清单
curl http://localhost:5000/v2/myapp/manifests/latest

# 获取镜像层信息
curl http://localhost:5000/v2/myapp/blobs/sha256:abc123...
```

## 镜像生命周期管理

### 镜像命名

```bash
# 官方镜像（library 命名空间）
nginx:latest                    # 实际是 library/nginx:latest

# 用户镜像
myusername/myapp:v1.0

# 组织镜像
myorganization/myapp:v1.0

# 完整路径（私有 Registry）
registry.example.com:5000/myorg/myapp:v1.0
```

### 标签管理

```bash
# 为镜像添加多个标签
docker tag myapp:latest registry.example.com/myapp:v1.0
docker tag myapp:latest registry.example.com/myapp:latest
docker tag myapp:latest registry.example.com/myapp:stable

# 一次性推送所有标签
docker push registry.example.com/myapp --all-tags
```

### 删除镜像

```bash
# 从本地删除
docker rmi myapp:v1.0

# 从 Registry 删除（需要启用删除功能）
# 1. 启用 delete 功能（在 config.yml 中）
docker exec registry rm /var/lib/registry/docker/registry/v2/repositories/myapp/_manifests/tags/v1.0
```

## Docker Registry vs Harbor

| 特性 | Registry | Harbor |
|---|---|---|
| 镜像存储 | 原生支持 | 原生支持 + S3/GCS/Azure |
| 认证 | Basic Auth | RBAC + LDAP/OIDC |
| 镜像扫描 | 无 | Trivy/Clair |
| 签名 | 无 | Notary |
| 复制 | 无 | 多主复制 |
| 镜像清理 | 手动 GC | Web UI |
| 适用场景 | 简单内部 Registry | 企业级镜像管理 |

## 总结

Docker Registry 是镜像存储和分发的基础设施：

- **基础 Registry**：适合小团队，简单易用
- **认证配置**：生产环境必须配置 HTTPS 和认证
- **清理维护**：定期执行垃圾回收
- **安全考虑**：避免在镜像中存储敏感信息

对于大多数生产场景，建议使用 Harbor 获得更完善的企业级功能。
