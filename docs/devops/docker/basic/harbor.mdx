# Harbor 私有镜像仓库

当你需要管理数百个镜像、进行安全扫描、签名验证、多仓库复制时，Docker Registry 就力不从心了。**Harbor** 是 CNCF 毕业的项目，专为企业级镜像仓库设计。

## Harbor 架构

Harbor 是一个完整的企业级 Registry 解决方案：

```

│  │              Harbor Portal (Web UI)                 │ │
│  └────────────────────────────────────────────────────┘ │
│  ┌────────────────────────────────────────────────────┐ │
│  │                   Nginx (反向代理)                    │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐   │
│  │   Core API   │  │    Jobs      │  │   Harbor    │   │

│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐   │

```

### 核心组件

| 组件 | 作用 |
|---|---|
| **Harbor Core** | 核心 API 和业务逻辑 |
| **Harbor Portal** | Web 管理界面 |
| **Registry** | 镜像存储（Docker Distribution） |
| **Job Service** | 复制、漏洞扫描等异步任务 |
| **Notary** | 镜像签名和验证 |
| **Trivy/Clair** | 镜像安全漏洞扫描 |
| **PostgreSQL** | 元数据存储 |
| **Redis** | 会话和缓存 |

## 安装 Harbor

### 使用 Docker Compose（推荐）

```bash
# 下载 Harbor 安装包
wget https://github.com/goharbor/harbor/releases/download/v2.10.0/harbor-offline-installer.tgz
tar xzf harbor-offline-installer.tgz
cd harbor

# 配置 Harbor
cp harbor.yml.tmpl harbor.yml
vim harbor.yml
```

### 配置文件

```yaml title="harbor.yml"
# 基本配置
hostname: harbor.example.com
http:
  port: 80
https:
  port: 443
  certificate: /path/to/cert.pem
  private_key: /path/to/key.pem

# 管理员密码
harbor_admin_password: Harbor12345

# 数据库配置
database:
  password: root123
  max_idle_conns: 50
  max_open_conns: 1000

# 存储配置
storage:
  filesystem:
    maxthreads: 8
  # 也支持 S3/GCS/Azure
  # s3:
  #   region: us-west-1
  #   bucket: my-harbor

# 日志配置
log:
  level: info
  location: /var/log/harbor

# Clair 漏洞扫描器（可选）
clair:
  enabled: true
  vsulnerabilities_update_interval: 24h

# Notary 签名服务（可选）
notary:
  enabled: true
```

### 执行安装

```bash
# 安装 Docker Compose 插件
# 如果需要启动 Harbor 的所有组件（包括 notary、trivy）
./install.sh --with-notary --with-trivy --with-chartmuseum

# 快速安装（仅核心组件）
./install.sh
```

### 启动和停止

```bash
# 启动
docker compose up -d

# 停止
docker compose down

# 完全停止并清理
docker compose down -v
```

## Harbor 核心功能

### 1. 多项目管理

Harbor 支持项目隔离：

| 项目类型 | 说明 |
|---|---|
| **公开项目** | 所有用户可拉取镜像 |
| **私有项目** | 需要成员权限 |
| **系统项目** | Harbor 内置（如 library） |

```bash
# 推送镜像到 Harbor
docker login harbor.example.com -u admin
docker tag myapp:latest harbor.example.com/myproject/myapp:v1.0
docker push harbor.example.com/myproject/myapp:v1.0
```

### 2. 镜像仓库复制

支持跨 Harbor 实例的镜像同步：

```yaml title="复制策略配置"
# 源 Harbor 配置
source:
  url: https://harbor-source.example.com
  credential:
    access_key: AKIAIOSFODNN7EXAMPLE
    access_secret: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

# 目标 Harbor 配置
destination:
  url: https://harbor-dest.example.com
  credential:
    access_key: AKIAIOSFODNN7EXAMPLE
    access_secret: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

# 过滤器
filters:
  - type: name
    value: "library/**"
  - type: tag
    value: "v*"

# 触发方式
trigger:
  type: manual # manual/scheduled/event-based
  schedule: "0 0 * * * *" # cron 表达式
```

### 3. 镜像安全扫描

Harbor 集成了 Trivy 和 Clair 进行漏洞扫描：

```bash
# 扫描结果示例
镜像: harbor.example.com/myproject/myapp:v1.0

严重漏洞 (Critical): 2
高危漏洞 (High): 5
中危漏洞 (Medium): 12
低危漏洞 (Low): 8

CVE-2023-12345 | openssl | Critical | 在 OpenSSL 1.1.1 中发现缓冲区溢出
CVE-2023-67890 | libcurl | High | 允许拒绝服务攻击
```

:::tip 安全策略
在项目设置中可以配置阻止未通过扫描的镜像部署：

- 禁止部署有 Critical 漏洞的镜像
- 禁止部署未扫描的镜像
- 强制要求所有镜像签名
:::

### 4. 镜像签名与验证

Notary 实现镜像签名：

```bash
# 启用内容信任（Docker Content Trust）
export DOCKER_CONTENT_TRUST=1
export DOCKER_CONTENT_TRUST_SERVER=https://notary.harbor.example.com

# 签名并推送镜像
docker push harbor.example.com/myproject/signed-image:v1.0
# 首次推送会提示创建根密钥，设置密码

# 拉取签名镜像（仅拉取已签名）
docker pull harbor.example.com/myproject/signed-image:v1.0
```

### 5. Robot 账户

为 CI/CD 创建机器人账户（不暴露用户密码）：

```
# 创建 robot 账户
名称: ci-pipeline
权限: 拉取镜像 (docker pull)
到期: 30 天

# 获取凭证
用户名: robot$ci-pipeline
密码: 1f0a2b3c4d5e6f7a8b9c0d1e2f3a4b5c
```

```bash
# 使用 robot 账户登录
docker login harbor.example.com -u robot$ci-pipeline -p 1f0a2b3c4d5e6f7a8b9c0d1e2f3a4b5c

# 推送镜像
docker push harbor.example.com/myproject/myapp:v1.0
```

## Helm Chart 仓库

Harbor 还支持 Helm Chart 存储：

```bash
# 添加 Harbor 作为 Helm repo
helm repo add myrepo https://harbor.example.com/chartrepo/myproject

# 上传 Chart
helm cm-push mychart-1.0.0.tgz myrepo

# 搜索 Chart
helm search repo myrepo/myapp

# 安装
helm install myapp myrepo/myapp --version 1.0.0
```

## API 使用

Harbor 提供 RESTful API：

```bash
# 获取项目列表
curl -u admin:Harbor12345 \
  https://harbor.example.com/api/v2.0/projects

# 创建项目
curl -u admin:Harbor12345 \
  -X POST \
  -H "Content-Type: application/json" \
  https://harbor.example.com/api/v2.0/projects \
  -d '{"project_name":"newproject","public":false}'

# 获取镜像标签
curl -u admin:Harbor12345 \
  https://harbor.example.com/api/v2.0/projects/myproject/repositories/myapp/artifacts?with_tag=true
```

## 高可用配置

Harbor 支持高可用部署：

```yaml title="docker-compose.yml"
version: '3'
services:
  harbor-core:
    extends:
      file: docker-compose.yml
      service: harbor-core
    deploy:
      replicas: 3

  harbor-portal:
    extends:
      file: docker-compose.yml
      service: harbor-portal
    deploy:
      replicas: 3

  nginx:
    image: goharbor/nginx-photon:v2.10.0
    ports:
      - "80:8080"
      - "443:8443"
    depends_on:
      - harbor-core
    deploy:
      replicas: 2
```

:::tip 高可用要点
- PostgreSQL 使用主从复制
- Redis 使用集群模式
- 共享存储（S3/NFS）确保镜像数据一致
- 外部负载均衡器分发请求
:::

## 总结

Harbor 是企业级镜像仓库的首选：

| 功能 | 说明 |
|---|---|
| **多项目** | 权限隔离，适合多团队 |
| **安全扫描** | Trivy/Clair 自动扫描漏洞 |
| **签名验证** | Notary 确保镜像可信 |
| **复制同步** | 多 Harbor 实例镜像同步 |
| **Robot 账户** | CI/CD 集成最佳实践 |
| **Helm 支持** | 同时管理 Helm Chart |

从 Docker Registry 到 Harbor，是从「能存镜像」到「安全管镜像」的升级。
