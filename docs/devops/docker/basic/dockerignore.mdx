# .dockerignore 文件

你有没有遇到过这种情况：`docker build` 突然变得很慢，或者镜像体积莫名其妙地大？

这很可能是因为你没有使用 `.dockerignore` 文件，把大量不需要的文件都发送给了 Docker 构建上下文。

## 构建上下文是什么

在理解 `.dockerignore` 之前，我们需要先理解 **构建上下文（Build Context）**。

当你执行 `docker build .` 时，当前目录会被打包成一个 tar 压缩包，发送给 Docker daemon 进行构建。这个目录就是构建上下文。

```
构建上下文发送过程：

你的项目目录                    Docker daemon
┌─────────────────┐           ┌─────────────────┐
│ ./my-app/       │   ──────> │  解压后构建      │
│   ├── src/      │  tar.gz   │  上下文          │
│   ├── test/     │           │  /my-app/       │
│   ├── docs/     │           │    ├── src/     │
│   ├── .git/     │           │    ├── test/    │
│   ├── node_modules/         │    └── ...      │
│   └── .env      │           └─────────────────┘
└─────────────────┘
```

**关键问题**：构建上下文会包含目录下**所有文件**，即使 Dockerfile 的 `COPY` 指令只复制了部分文件。

如果你有一个 1GB 的 `node_modules` 目录但 Dockerfile 不复制它，构建上下文仍然会发送 1GB 的数据，浪费大量时间和带宽。

## .dockerignore 的作用

`.dockerignore` 文件的语法与 `.gitignore` 相同，用于排除不需要发送到构建上下文的文件和目录。

```bash
# 示例：.dockerignore 的效果
# 构建前
./my-app/ (500MB)
├── src/ (10MB)         ← 需要
├── node_modules/ (450MB) ← 不需要，COPY package*.json 后会自动安装
├── .git/ (30MB)        ← 不需要
├── docs/ (10MB)        ← 不需要
└── .env (1KB)          ← 敏感信息，绝对不能发送

# 使用 .dockerignore 后
./my-app/ (20MB)
├── src/ (10MB)
├── package.json (1KB)
├── package-lock.json (5KB)
└── Dockerfile (5KB)
```

## 基本语法

`.dockerignore` 使用 glob 模式匹配文件：

```bash
# 注释
# 这是一个注释行

# 排除特定文件
Dockerfile
.dockerignore
.env

# 排除所有 node_modules 目录
node_modules/

# 排除所有 .log 文件
*.log

# 排除 docs 目录及其内容
docs/

# 排除所有 .md 文件（但不包含 README.md）
*.md
!README.md

# 排除 test 目录（但保留 unit 子目录）
test/**
!test/unit/**
```

## 常见错误

### 1. 排除 Dockerfile 本身

```bash
# 错误：Dockerfile 被排除，构建失败
Dockerfile

# 正确：保留 Dockerfile
# 不要在 .dockerignore 中写 Dockerfile
```

### 2. 过度排除

```bash
# 错误：排除了所有 .json 文件，包括 package.json
*.json

# 正确：只排除不需要的 .json 文件
tsconfig*.json
*.config.json
```

### 3. 忘记重新构建

修改 `.dockerignore` 后，之前缓存的构建上下文仍然有效。如果想确保生效，使用 `--no-cache` 或添加虚拟参数：

```bash
docker build --no-cache -t myapp .

# 或者添加构建参数触发重建
docker build --build-arg CACHE_BUST=$(date +%s) -t myapp .
```

## 调试 .dockerignore

如果构建上下文仍然很大，可以用以下方法调试：

```bash
# 查看构建上下文大小
docker build --progress=plain . 2>&1 | grep "sending build context"

# 或者创建一个 tar 文件查看
tar -czvf context.tar.gz .

# 查看 tar 内容大小分布
tar -tzf context.tar.gz | awk '{print $5, $6}' | sort -rh | head -20
```

## 总结

`.dockerignore` 是 Docker 构建的重要优化手段：

- **减小构建上下文**：减少发送到 Docker daemon 的数据量
- **加快构建速度**：上下文越小，传输和解析越快
- **保护敏感信息**：防止密钥等文件进入镜像层
- **减小镜像体积**：不必要的文件不会被复制

正确的 `.dockerignore` 配置应该是项目启动的第一步之一。
