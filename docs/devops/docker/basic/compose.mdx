# Docker Compose

应用从来不是孤立的。一个 Web 服务可能需要数据库、缓存、消息队列……这些组件组合在一起，才是一个完整的应用。

Docker Compose 正是为多容器应用而生的工具——它允许你使用一个 YAML 文件定义和管理多个容器。

## 为什么需要 Docker Compose

假设你有一个典型的微服务架构：

```
┌─────────────────────────────────────────────────────┐
│                 完整应用栈                             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐     │
│  │   Web    │───▶│   API    │───▶│   DB     │     │
│  │  Nginx   │    │  Node.js │    │ Postgres │     │
│  └──────────┘    └──────────┘    └──────────┘     │
│                       │                             │
│                       ▼                             │
│                 ┌──────────┐                       │
│                 │  Redis   │                       │
│                 │  缓存     │                       │
│                 └──────────┘                       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

没有 Docker Compose，你需要：

```bash
# 手动启动每个容器，并处理依赖关系
docker network create mynet
docker run -d --name db --network mynet postgres:15
docker run -d --name redis --network mynet redis:alpine
docker run -d --name api --network mynet -p 3000:3000 myapi:latest
docker run -d --name web --network mynet -p 80:80 nginx:alpine
```

用 Docker Compose，只需要一个文件：

```bash
docker compose up -d
```

## 核心概念

Docker Compose 使用三个核心概念：

- **Services（服务）**：一个容器及其配置
- **Networks（网络）**：容器间通信的网络
- **Volumes（卷）**：持久化数据存储

## docker-compose.yml 基础

### 最小示例

```yaml title="docker-compose.yml"
version: "3.8"

services:
  web:
    image: nginx:alpine
    ports:
      - "80:80"
```

### 完整结构

```yaml title="docker-compose.yml"
version: "3.8"

services:
  web:
    image: nginx:alpine
    container_name: my-web
    restart: unless-stopped
    
    ports:
      - "80:80"
      - "443:443"
    
    volumes:
      - ./html:/usr/share/nginx/html:ro
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - nginx-logs:/var/log/nginx
    
    environment:
      - NODE_ENV=production
      - APP_VERSION=1.0.0
    
    networks:
      - frontend
    depends_on:
      - api

  api:
    image: node:18-alpine
    working_dir: /app
    command: node server.js
    volumes:
      - ./src:/app
    environment:
      - DATABASE_URL=postgres://db:5432/mydb
      - REDIS_URL=redis://redis:6379
    networks:
      - frontend
      - backend
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started

  db:
    image: postgres:15-alpine
    volumes:
      - postgres-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=mydb
      - POSTGRES_USER=admin
      - POSTGRES_PASSWORD_FILE=/run/secrets/db_password
    secrets:
      - db_password
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U admin -d mydb"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - backend

  redis:
    image: redis:7-alpine
    volumes:
      - redis-data:/data
    networks:
      - backend

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge

volumes:
  nginx-logs:
  postgres-data:
  redis-data:

secrets:
  db_password:
    file: ./secrets/db_password.txt
```

## 服务定义详解

### image

指定容器使用的镜像：

```yaml
services:
  web:
    image: nginx:alpine
    image: registry.example.com/myapp:v1.2.3
```

### container_name

自定义容器名称：

```yaml
services:
  web:
    container_name: my-nginx
```

### restart

容器重启策略：

```yaml
services:
  web:
    restart: "no"           # 不重启（默认）
    restart: always         # 始终重启
    restart: on-failure      # 仅在失败时重启
    restart: unless-stopped # 除非手动停止，否则重启
```

### ports

端口映射（宿主机端口:容器端口）：

```yaml
services:
  web:
    ports:
      - "80:80"             # 简单映射
      - "127.0.0.1:80:80"   # 仅绑定本地回环
      - "443:443/tcp"       # 指定协议
      - "5000-5010:5000-5010" # 端口范围
```

### volumes

挂载数据卷：

```yaml
services:
  web:
    volumes:
      - ./html:/usr/share/nginx/html      # 绑定挂载
      - nginx-data:/var/log/nginx         # 命名卷
      - /tmp:/tmp:ro                      # 只读挂载
```

### environment

设置环境变量：

```yaml
services:
  api:
    environment:
      - NODE_ENV=production
      - DEBUG=false
      - DATABASE_URL=postgres://user:pass@db:5432/mydb
```

或者使用 `env_file` 批量设置：

```yaml
services:
  api:
    env_file:
      - .env.common
      - .env.prod
```

### networks

指定容器加入的网络：

```yaml
services:
  web:
    networks:
      - frontend
      - backend
```

### depends_on

声明服务依赖关系：

```yaml
services:
  web:
    depends_on:
      - api
      - db

  api:
    depends_on:
      db:
        condition: service_healthy  # 等待 db 健康检查通过
```

## 健康检查

`condition` 支持三种状态：

| 条件 | 说明 |
|---|---|
| `service_started` | 容器启动即可（默认） |
| `service_healthy` | 等待健康检查通过 |
| `service_completed_successfully` | 一次性任务执行成功 |

```yaml
services:
  db:
    image: postgres:15-alpine
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s

  web:
    depends_on:
      db:
        condition: service_healthy
```

## 多环境配置

### 使用 profiles

```yaml
services:
  web:
    image: nginx:alpine
  
  db:
    image: postgres:15-alpine
  
  adminer:
    image: adminer:4
    profiles:
      - admin

  redis:
    image: redis:alpine
    profiles:
      - cache
```

```bash
# 只启动 web 和 db
docker compose up -d

# 启动包括 adminer 的完整栈
docker compose --profile admin up -d

# 启动包括 redis 的缓存栈
docker compose --profile cache up -d
```

### 环境变量文件

创建 `.env` 文件：

```bash title=".env"
COMPOSE_PROJECT_NAME=myapp
TAG=latest
DATABASE_PASSWORD=secret123
```

在 `docker-compose.yml` 中引用：

```yaml
services:
  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_PASSWORD: ${DATABASE_PASSWORD}
```

## Compose 命令

### 启动和停止

```bash
# 启动所有服务（后台运行）
docker compose up -d

# 前台运行（查看日志）
docker compose up

# 带重建启动
docker compose up -d --build

# 停止所有服务
docker compose down

# 停止并删除数据卷
docker compose down -v
```

### 扩缩容

```bash
# 扩展服务实例数
docker compose up -d --scale api=3

# 或者在 compose 文件中定义 replicas
docker compose up -d --scale web=5
```

### 查看日志

```bash
# 查看所有服务日志
docker compose logs

# 实时跟踪日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f web

# 最近 100 行
docker compose logs --tail 100
```

### 调试命令

```bash
# 进入容器
docker compose exec web sh

# 在服务中执行命令
docker compose exec db psql -U postgres

# 查看服务状态
docker compose ps

# 查看资源使用
docker compose top
```

### 其他命令

```bash
# 构建镜像
docker compose build

# 拉取最新镜像
docker compose pull

# 重启服务
docker compose restart

# 暂停/恢复服务
docker compose pause
docker compose unpause

# 删除停止的服务
docker compose rm

# 列出配置（检查配置是否正确）
docker compose config

# 列出正在使用的镜像
docker compose images
```

## 多文件组合

Docker Compose 支持多个配置文件，用于不同环境：

```bash
# 基础配置
docker-compose.yml

# 开发环境覆盖
docker-compose.override.yml

# 生产环境覆盖
docker-compose.prod.yml
```

```bash
# 开发环境（自动加载 override）
docker compose up -d

# 生产环境
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

## 完整示例：LAMP 栈

```yaml title="docker-compose.yml"
version: "3.8"

services:
  web:
    image: php:8.2-apache
    container_name: lamp-web
    ports:
      - "80:80"
    volumes:
      - ./html:/var/www/html
      - ./php.ini:/usr/local/etc/php/conf.d/custom.ini
    depends_on:
      db:
        condition: service_healthy
    networks:
      - lamp-net

  mysql:
    image: mysql:8.0
    container_name: lamp-db
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
      MYSQL_DATABASE: ${DB_NAME}
      MYSQL_USER: ${DB_USER}
      MYSQL_PASSWORD: ${DB_PASSWORD}
    volumes:
      - mysql-data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - lamp-net

  phpmyadmin:
    image: phpmyadmin:latest
    container_name: lamp-pma
    ports:
      - "8080:80"
    environment:
      PMA_HOST: mysql
      PMA_USER: root
      PMA_PASSWORD: ${DB_ROOT_PASSWORD}
    depends_on:
      db:
        condition: service_healthy
    networks:
      - lamp-net
    profiles:
      - admin

networks:
  lamp-net:
    driver: bridge

volumes:
  mysql-data:
```

```bash title=".env"
DB_ROOT_PASSWORD=root_secret_pass
DB_NAME=myapp
DB_USER=myapp
DB_PASSWORD=app_secret_pass
```

## 总结

Docker Compose 是多容器应用管理的利器：

- **简化部署**：一条命令启动完整应用栈
- **环境一致**：开发、测试、生产使用相同配置
- **依赖管理**：`depends_on` 自动处理启动顺序
- **健康检查**：`condition: service_healthy` 确保依赖服务就绪
- **多环境支持**：profiles 和多文件组合

掌握 Docker Compose，是容器化开发者的必备技能。
