# 健康检查

如何知道一个容器是「健康」的？如果容器内的应用卡死了，Docker 如何发现并处理？

Docker 的健康检查机制提供了自动检测容器状态的能力。

## HEALTHCHECK 指令

在 Dockerfile 中使用 `HEALTHCHECK` 定义容器健康检查：

```dockerfile
FROM nginx:alpine

# 健康检查：每 30 秒检查一次，超时 3 秒，重试 3 次
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost/ || exit 1
```

### 参数说明

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--interval` | 检查间隔 | `30s` |
| `--timeout` | 单次检查超时 | `30s` |
| `--retries` | 连续失败次数视为不健康 | `3` |
| `--start-period` | 容器启动后多久开始检查 | `0s` |

## 健康检查命令

健康检查使用容器的退出码判断健康状态：

| 退出码 | 含义 |
|---|---|
| `0` | 容器健康 |
| `1` | 容器不健康 |
| `2` | 保留（不应使用） |

### 常用检查方式

```dockerfile
# HTTP 健康检查
HEALTHCHECK --interval=30s --timeout=3s CMD curl -f http://localhost/health || exit 1

# TCP 连接检查
HEALTHCHECK --interval=30s --timeout=3s CMD nc -z localhost 5432 || exit 1

# 执行脚本检查
HEALTHCHECK --interval=30s --timeout=3s CMD /opt/check-health.sh || exit 1

# ping 检查
HEALTHCHECK --interval=30s --timeout=3s CMD wget -qO- http://localhost/ping || exit 1
```

### 多层健康检查

```dockerfile
# 检查脚本示例
#!/bin/sh
# /opt/check-health.sh

# 检查进程是否运行
if ! pgrep -x "python" > /dev/null; then
    echo "Process not running"
    exit 1
fi

# 检查端口是否监听
if ! nc -z localhost 8000; then
    echo "Port not listening"
    exit 1
fi

# 检查健康端点
if ! curl -sf http://localhost:8000/health; then
    echo "Health check failed"
    exit 1
fi

exit 0
```

## 容器状态

容器有六种状态：

| 状态 | 说明 |
|---|---|
| `created` | 已创建但未启动 |
| `running` | 运行中 |
| `paused` | 暂停 |
| `restarting` | 重启中 |
| `exited` | 已退出 |
| `dead` | Docker 认为容器已死 |

健康检查增加了第四种运行状态：

| 健康状态 | 说明 |
|---|---|
| `(healthy)` | 健康检查通过 |
| `(unhealthy)` | 健康检查连续失败 |
| `(starting)` | 启动期间，未完成初始检查 |

## Docker Compose 中的健康检查

```yaml title="docker-compose.yml"
services:
  web:
    image: nginx:alpine
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost/"]
      interval: 30s
      timeout: 3s
      retries: 3
      start_period: 10s

  api:
    image: node:18-alpine
    healthcheck:
      test: ["CMD-SHELL", "node /app/check-health.js"]
      interval: 30s
      timeout: 5s
      retries: 3

  db:
    image: postgres:15-alpine
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
```

## 依赖健康检查

在 Docker Compose 中使用 `condition` 确保依赖服务健康：

```yaml title="docker-compose.yml"
services:
  web:
    image: nginx:alpine
    depends_on:
      api:
        condition: service_healthy

  api:
    image: node:18-alpine
    depends_on:
      db:
        condition: service_healthy
      cache:
        condition: service_healthy

  db:
    image: postgres:15-alpine
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  cache:
    image: redis:7-alpine
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 3
```

### condition 选项

| 条件 | 说明 |
|---|---|
| `service_started` | 容器启动（默认） |
| `service_healthy` | 等待健康检查通过 |
| `service_completed_successfully` | 一次性任务成功 |

## Swarm 模式中的健康检查

在 Docker Swarm 中，健康检查与滚动更新紧密集成：

```yaml title="docker-compose.yml"
services:
  api:
    image: myregistry/api:v1.0.0
    replicas: 3
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 3s
      retries: 3
    deploy:
      replicas: 3
      update_config:
        parallelism: 1
        delay: 10s
        monitor: 5s
        failure_action: rollback
      restart_policy:
        condition: on-failure
```

### Swarm 健康检查行为

- 不健康的容器会被 Swarm 标记
- 滚动更新会等待容器健康后才继续
- 可以配置失败时自动回滚

## 手动健康检查

使用 `docker healthcheck` 命令：

```bash
# 手动触发健康检查
docker healthcheck inspect web

# 手动标记容器为不健康（测试用）
docker update --health-flags "unhealthy" web
```

## 应用健康端点设计

应用应提供专门的健康检查端点：

### 简单检查

```javascript
// Express.js
app.get('/health', (req, res) => {
  res.status(200).json({ status: 'ok' });
});
```

### 完整检查（依赖外部服务）

```javascript
app.get('/health', async (req, res) => {
  const checks = {
    database: await checkDatabase(),
    redis: await checkRedis(),
    disk: await checkDisk(),
    memory: process.memoryUsage().heapUsed < 500 * 1024 * 1024
  };

  const healthy = Object.values(checks).every(Boolean);

  res.status(healthy ? 200 : 503).json({
    status: healthy ? 'healthy' : 'unhealthy',
    checks,
    timestamp: new Date().toISOString()
  });
});

async function checkDatabase() {
  try {
    await db.query('SELECT 1');
    return true;
  } catch {
    return false;
  }
}
```

### 就绪检查 vs 存活检查

| 类型 | 端点 | 说明 |
|---|---|---|
| 存活检查（liveness） | `/livez` | 应用是否存活（避免重启） |
| 就绪检查（readiness） | `/readyz` | 应用是否就绪（避免接收流量） |

```javascript
// Kubernetes 风格
app.get('/livez', (req, res) => {
  res.status(200).json({ status: 'alive' });
});

app.get('/readyz', async (req, res) => {
  const ready = await app.isReady();
  res.status(ready ? 200 : 503).json({
    status: ready ? 'ready' : 'not ready'
  });
});
```

## 常见问题

### 健康检查频繁失败

```bash
# 检查健康检查输出
docker inspect --format '{{json .State.Health.Log}}' container-name | jq

# 检查容器内命令是否可用
docker exec container-name curl --version
```

### 超时时间太短

如果健康检查总是超时，增加 `--timeout` 值：

```dockerfile
HEALTHCHECK --timeout=10s CMD curl -f http://localhost/health || exit 1
```

### 启动时间较长

使用 `--start-period` 给予足够的启动时间：

```dockerfile
HEALTHCHECK --start-period=60s CMD curl -f http://localhost/health || exit 1
```

## 总结

健康检查是容器自愈能力的关键：

- **HEALTHCHECK 指令**：在 Dockerfile 中定义健康检查
- **自动检测**：Docker 自动执行检查并更新容器状态
- **依赖管理**：Compose 中 `condition: service_healthy` 确保依赖就绪
- **Swarm 集成**：健康检查与滚动更新集成
- **端点设计**：应用应提供有意义的健康端点

正确的健康检查配置能提高应用的可用性和自愈能力。
