# Harbor 企业级镜像仓库

你有没有想过——Docker Hub 虽然方便，但把公司代码放到别人服务器上总是让人不放心。尤其是金融、医疗、政府项目，对数据主权有严格要求。

Harbor 就是来解决这个问题的。Harbor 是 VMware（现属 Broadcom）开源的企业级 Registry，不仅支持私有部署，还提供了图形化界面、镜像安全扫描、镜像签名、镜像复制等企业级功能。

## Harbor 是什么

Harbor 是一个开源的容器镜像仓库，专为企业级使用场景设计：

- **私有部署**：数据完全在自己服务器上
- **图形化界面**：不用记命令，直接 Web 管理
- **镜像安全扫描**：自动检查镜像中的 CVE 漏洞
- **镜像签名**：确保镜像来源可信
- **镜像复制**：多数据中心同步镜像
- **RBAC 权限控制**：细粒度管理谁可以 push/pull

## 安装 Harbor

### 使用 Docker Compose 安装（单节点）

```bash
# 1. 下载安装包
wget https://github.com/goharbor/harbor/releases/download/v2.10.0/harbor-offline-installer-v2.10.0.tgz
tar -xzf harbor-offline-installer-v2.10.0.tgz
cd harbor

# 2. 配置
cp harbor.yml.tmpl harbor.yml
vim harbor.yml

# 3. 关键配置项
# hostname: registry.example.com
# http.port: 80
# https.port: 443
# certificate: /path/to/cert.pem
# private_key: /path/to/key.pem
# harbor_admin_password: Harbor12345

# 4. 配置数据存储（可选，使用外部存储）
# vi harbor.yml
# storage_service:
#   ca_bundle: /path/to/ca-bundle.crt
#   s3:
#     region: us-west-1
#     bucket: harbor-registry
#     accesskey: aws-access-key
#     secretkey: aws-secret-key

# 5. 启动
./install.sh --with-notary --with-trivy --with-chartmuseum
```

### 关键配置说明

```yaml title="harbor.yml"
# 基础配置
hostname: registry.example.com
http:
  port: 80
https:
  port: 443
  certificate: /data/cert/server.crt
  private_key: /data/cert/server.key

# 数据库配置（默认使用内置 PostgreSQL，生产环境建议外部数据库）
database:
  password: postgres_password

# 数据持久化
persistence:
  persistentVolumeClaim:
    harbor:
      storage_class: "nfs-storage"
      access_mode: ReadWriteOnce
      size: 100Gi

# 管理员密码
harbor_admin_password: ChangeMe123!

# 邮件通知
mail:
  host: smtp.example.com
  port: 587
  username: harbor@example.com
  password: mail_password
  from_address: harbor@example.com
```

## Docker 客户端配置

### 登录 Harbor

```bash
# 登录
docker login registry.example.com
# Username: admin
# Password: Harbor12345

# 登录后配置文件在 ~/.docker/config.json
```

### push/pull 镜像

```bash
# 打标签
docker tag myapp:latest registry.example.com/library/myapp:v1.0.0

# 推送
docker push registry.example.com/library/myapp:v1.0.0

# 拉取
docker pull registry.example.com/library/myapp:v1.0.0
```

### Kubernetes 使用

```yaml
# 创建 Secret
kubectl create secret docker-registry harbor-secret \
  --docker-server=registry.example.com \
  --docker-username=admin \
  --docker-password=Harbor12345 \
  --docker-email=admin@example.com

# Pod 配置
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
  - name: myapp
    image: registry.example.com/library/myapp:v1.0.0
  imagePullSecrets:
  - name: harbor-secret
```

## 安全扫描

### 内置 Trivy 扫描

Harbor 内置了 Trivy 作为默认扫描器：

```bash
# Harbor 会自动对 push 的镜像进行 CVE 扫描
# 扫描结果分为 4 个等级：
# - Critical：立即修复
# - High：高风险，限期修复
# - Medium：中风险，建议修复
# - Low：低风险，可延后处理
```

### 手动触发扫描

```bash
# 通过 Harbor API 触发扫描
curl -X POST \
  -u "admin:Harbor12345" \
  "https://registry.example.com/api/v2.0/projects/library/repositories/myapp/artifacts/v1.0.0/scan"
```

### 查看扫描报告

```bash
# 获取扫描报告
curl -u "admin:Harbor12345" \
  "https://registry.example.com/api/v2.0/projects/library/repositories/myapp/artifacts/v1.0.0/additions/vulnerabilities"
```

## 镜像复制（Replication）

### 同构复制（Harbor to Harbor）

```yaml title="harbor-replication-rule"
# 创建复制规则
# Harbor Web UI → 系统管理 → 复制管理 → 新建规则
name: replicate-to-dr
# 源 Registry
source_registry:
  provider: harbor
  url: https://primary-harbor.example.com
  credential_ref:
    name: harbor-admin
# 过滤器
filters:
  - type: name
    value: "library/**"
  - type: tag
    value: "v*"
# 目标
target:
  endpoint: https://dr-harbor.example.com
  namespace: library
  credential_ref:
    name: harbor-admin
# 触发方式
trigger:
  type: event_based  # 基于事件（镜像 push 时触发）
# 覆盖模式
deletion: true  # 允许删除已复制的镜像
```

### 异构复制（Docker Hub / 阿里云）

```yaml
# 从阿里云 ACR 复制镜像到本地 Harbor
name: replicate-from-aliyun
source_registry:
  provider: google-gcr  # 阿里云用 aliyun
  url: https://registry.cn-shanghai.aliyuncs.com
  credential_ref:
    name: aliyun-creds
filters:
  - type: name
    value: "myproject/**"
trigger:
  type: scheduled
  trigger_settings:
    cron: "0 0 * * * *"  # 每小时同步一次
```

## RBAC 权限控制

### 内置角色

| 角色 | 说明 |
|---|---|
| **Guest** | 只读权限，可以 pull 镜像 |
| **Developer** | 可以 pull/push 镜像 |
| **Maintainer** | 可以管理项目成员和标签策略 |
| **Project Admin** | 完全控制，包括删除项目 |

### 添加项目成员

```bash
# 添加成员
curl -X POST \
  -u "admin:Harbor12345" \
  -H "Content-Type: application/json" \
  "https://registry.example.com/api/v2.0/projects/library/members" \
  -d '{
    "member_user": {
      "username": "developer1"
    },
    "role_id": 2  # Developer
  }'
```

## Helm Chart 仓库

Harbor 内置了 ChartMuseum，支持存储 Helm Chart：

```bash
# 添加 Harbor 作为 Helm 仓库
helm repo add myrepo https://registry.example.com/chartrepo/library \
  --username=admin \
  --password=Harbor12345

# 上传 Chart
helm cm-push mychart-1.0.0.tgz myrepo

# 搜索 Chart
helm search repo myrepo/

# 安装 Chart
helm install myapp myrepo/mychart
```

## 高可用部署

```
┌─────────────┐
│   Nginx     │ ← LB
│   (负载均衡)|
└──────┬──────┘
       │
   ┌───┴───┐
   ▼       ▼
┌─────┐ ┌─────┐
│Harbor│ │Harbor│
│  Node│ │ Node│
│  1   │ │  2  │
└──┬───┘ └──┬───┘
   │        │
   ▼        ▼
┌─────┐ ┌─────┐
│ PG  │ │ PG  │ ← PostgreSQL 主从
│  主 │ │ 从  │
└─────┘ └─────┘
   │
   ▼
┌─────────────┐
│   Redis     │ ← Session 共享
└─────────────┘
```

关键点：

- 多 Harbor 实例共享同一个 PostgreSQL 和 Redis
- 使用共享存储（NFS、Ceph）存放镜像数据
- Nginx/LB 做前端负载均衡
- 建议至少 3 个 Harbor 节点保证高可用

## 运维管理

### 查看日志

```bash
# 查看 Harbor 日志
docker logs harbor-core -f
docker logs harbor-jobservice -f

# 查看 PostgreSQL 日志
docker logs harbor-db -f
```

### 常见问题排查

```bash
# 镜像 push 失败
# 1. 检查网络连通性
curl -I https://registry.example.com

# 2. 检查认证
docker login registry.example.com

# 3. 检查磁盘空间
df -h /data

# 扫描服务不工作
# 1. 检查 Trivy 配置
docker logs harbor-trivy-adapter -f

# 2. 重启扫描服务
docker-compose restart trivy-adapter
```

### 升级 Harbor

```bash
# 1. 备份数据
pg_dump -U postgres -h localhost -p 5432 harbor > harbor_backup.sql
cp -r /data/database /data/harbor-db-backup

# 2. 下载新版本
wget https://github.com/goharbor/harbor/releases/download/v2.11.0/harbor-offline-installer-v2.11.0.tgz

# 3. 更新配置
cp harbor.yml harbor.yml.bak
vim harbor.yml  # 更新版本信息

# 4. 执行升级
./install.sh --with-notary --with-trivy --with-chartmuseum
```

## 总结

Harbor 的核心价值：

1. **私有化部署**：数据完全可控
2. **安全扫描**：自动发现 CVE 漏洞
3. **镜像复制**：多数据中心同步
4. **细粒度权限**：RBAC 保护镜像访问
5. **多租户**：一个平台支持多个团队
6. **Helm 支持**：原生 Chart 仓库

对于企业来说，Harbor 是 Docker Registry 的最佳选择之一。
