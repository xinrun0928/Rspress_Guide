# 日志处理

容器运行时产生的日志是排查问题的关键。但容器的日志在哪里？如何收集？如何处理大量日志？

理解 Docker 的日志机制，是生产环境运维的基础。

## 日志驱动

Docker 使用插件化的日志驱动架构，默认使用 `json-file` 驱动。

### 支持的日志驱动

| 驱动 | 说明 | 适用场景 |
|---|---|---|
| `json-file` | JSON 格式存储（默认） | 开发/测试，简单日志 |
| `syslog` | 写入 syslog | 系统日志收集 |
| `journald` | 写入 systemd journal | RHEL/CentOS |
| `fluentd` | 写入 Fluentd | 集中式日志收集 |
| `awslogs` | 写入 CloudWatch | AWS 环境 |
| `gelf` | 写入 Graylog | Graylog 收集 |
| `splunk` | 写入 Splunk | Splunk 收集 |
| `gcplogs` | 写入 Google Cloud Logging | GCP 环境 |
| `none` | 禁用日志 | 临时容器 |

### 配置默认日志驱动

```json title="/etc/docker/daemon.json"
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3",
    "compress": "true"
  }
}
```

### 为容器指定日志驱动

```bash
# 使用特定日志驱动
docker run -d \
  --log-driver=syslog \
  --log-opt syslog-address=tcp://logserver:514 \
  --name app \
  nginx:alpine
```

## json-file 日志

默认的 `json-file` 驱动将日志存储在 `/var/lib/docker/containers/<container-id>/<container-id>-json.log`。

### 日志选项

```bash
# 配置单个容器的日志
docker run -d \
  --log-driver=json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  --log-opt compress=true \
  --name app \
  nginx:alpine
```

| 选项 | 说明 | 默认值 |
|---|---|---|
| `max-size` | 单个日志文件最大大小 | 无限制 |
| `max-file` | 最大日志文件数 | 无限制 |
| `compress` | 轮转时压缩旧日志 | `true` |

:::warning 磁盘空间
如果不配置 `max-size` 和 `max-file`，日志会无限增长，可能耗尽磁盘空间。
:::

### 查看日志

```bash
# 查看日志（默认显示最后 100 行）
docker logs my_container

# 实时跟踪日志
docker logs -f my_container

# 查看最近 N 行
docker logs --tail 100 my_container

# 显示时间戳
docker logs -t my_container

# 过滤日志
docker logs --since "2024-01-15T10:00:00" my_container
docker logs --since 10m my_container

# 只显示 stderr
docker logs --stderr my_container
```

### 直接读取日志文件

```bash
# 查看原始日志文件
cat /var/lib/docker/containers/<container-id>/<container-id>-json.log

# 格式化查看
cat /var/lib/docker/containers/<container-id>/<container-id>-json.log | jq -r '.log'
```

## Fluentd 日志收集

对于多主机环境，推荐使用 Fluentd 集中收集日志。

### Fluentd 配置

```xml title="fluent.conf"
<source>
  @type forward
  port 24224
  bind 0.0.0.0
</source>

<filter docker.**>
  @type parser
  @type json
  key_name log
  time_key time
  time_format %Y-%m-%dT%H:%M:%S
</filter>

<match docker.**>
  @type elasticsearch
  host elasticsearch
  port 9200
  logstash_format true
  logstash_prefix docker
</match>
```

### 容器配置

```bash
# 启动 Fluentd 容器
docker run -d \
  --name fluentd \
  -p 24224:24224 \
  -v /path/to/fluent.conf:/fluentd/etc/fluent.conf \
  fluent/fluentd:latest

# 启动应用容器，使用 Fluentd 驱动
docker run -d \
  --log-driver=fluentd \
  --log-opt fluentd-address=localhost:24224 \
  --log-opt tag=docker.{{.Name}} \
  --name app \
  nginx:alpine
```

### Filebeat 收集 Docker 日志

```yaml title="filebeat.yml"
filebeat.inputs:
  - type: container
    paths:
      - /var/lib/docker/containers/*/*.log
    json.keys_under_root: true
    json.add_error_key: true
    processors:
      - add_docker_metadata: ~

output.elasticsearch:
  hosts: ["elasticsearch:9200"]
```

## 日志轮转

### logrotate 配置

对于 `json-file` 日志，需要在宿主机配置 logrotate：

```bash title="/etc/logrotate.d/docker"
/var/lib/docker/containers/*/*.log {
    rotate 7
    daily
    compress
    delaycompress
    missingok
    notifempty
    copytruncate
}
```

### Docker 自身日志轮转

Docker 的 `--log-opt max-size` 和 `--log-opt max-file` 会自动轮转 Docker 的日志文件：

```bash
# 配置全局日志轮转（/etc/docker/daemon.json）
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "50m",
    "max-file": "5"
  }
}
```

## 日志最佳实践

### 结构化日志

在应用中输出 JSON 格式日志，便于解析和分析：

```javascript
// Node.js 示例
const winston = require('winston');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [
    new winston.transports.Console()
  ]
});

logger.info('Request processed', {
  requestId: req.id,
  userId: req.userId,
  duration: Date.now() - startTime
});
```

### 日志级别

```bash
# 使用环境变量控制日志级别
docker run -d \
  -e LOG_LEVEL=debug \
  --name app \
  myapp:latest
```

### 日志内容建议

| 应该记录 | 避免记录 |
|---|---|
| 请求 ID、用户 ID | 密码、密钥 |
| 操作时间戳 | 完整请求/响应体 |
| HTTP 方法、路径、状态码 | 大文件内容 |
| 错误堆栈（脱敏后） | PII（个人身份信息） |
| 性能指标 | 数据库完整 SQL（可脱敏） |

### Docker Compose 日志配置

```yaml title="docker-compose.yml"
services:
  web:
    image: nginx:alpine
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
        tag: "web"
    labels:
      - "app=web"

  api:
    image: node:18-alpine
    logging:
      driver: "fluentd"
      options:
        fluentd-address: "localhost:24224"
        tag: "api"
    environment:
      - LOG_LEVEL=info
```

## 清理日志

```bash
# 查看容器日志大小
du -h /var/lib/docker/containers/*/*-json.log

# 清理单个容器日志
truncate -s 0 /var/lib/docker/containers/<container-id>/*-json.log

# 清理所有容器日志
truncate -s 0 /var/lib/docker/containers/*/*-json.log
```

## 总结

Docker 日志处理的要点：

- **选择合适的日志驱动**：测试用 json-file，生产用 Fluentd/ELK
- **配置日志轮转**：防止磁盘空间耗尽
- **结构化日志**：JSON 格式便于分析
- **集中收集**：多主机环境使用集中式日志方案
- **安全考虑**：避免日志中泄露敏感信息

合理的日志策略是生产环境运维的关键。
