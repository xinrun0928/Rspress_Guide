# Maven 常用命令

Maven 的命令行提供了丰富的功能。熟练掌握常用命令，能大幅提升开发和调试效率。

## 基础命令

### mvn clean package

最常用的构建命令：

```bash
# 清理并打包（跳过测试）
mvn clean package

# 清理并打包（运行测试）
mvn clean verify

# 打包成 JAR
mvn clean package

# 打包成 WAR
mvn clean package -Dpackaging=war
```

### mvn clean install

安装到本地仓库，供其他项目使用：

```bash
# 清理并安装到本地仓库
mvn clean install

# 安装时跳过测试
mvn clean install -DskipTests
```

### mvn clean deploy

部署到远程仓库（需要配置 distributionManagement）：

```bash
# 部署到私服
mvn clean deploy
```

## 依赖分析命令

### mvn dependency:tree

查看完整依赖树：

```bash
# 查看依赖树
mvn dependency:tree

# 过滤特定依赖
mvn dependency:tree -Dincludes=com.alibaba:fastjson

# 输出详细版本信息
mvn dependency:tree -Dverbose
```

### mvn dependency:analyze

分析依赖使用情况：

```bash
# 分析依赖
mvn dependency:analyze
```

输出会提示：

| 类型 | 说明 |
|---|---|
| `Used undeclared dependencies` | 代码使用了但未声明的依赖 |
| `Unused declared dependencies` | 声明了但未使用的依赖 |

### mvn dependency:resolve

解析所有依赖：

```bash
# 解析依赖
mvn dependency:resolve

# 解析插件依赖
mvn dependency:resolve-plugins
```

## 版本管理命令

### mvn versions:display-dependency-updates

检查依赖更新：

```bash
# 检查可更新的依赖
mvn versions:display-dependency-updates
```

输出示例：

```
[INFO] The following version updates are available:
[INFO]   com.alibaba:fastjson : 1.2.83 -> 1.2.84
[INFO]   org.projectlombok:lombok : 1.18.24 -> 1.18.26
```

### mvn versions:update-properties

批量更新版本属性：

```bash
# 更新 properties 中的版本
mvn versions:update-properties
```

## 多模块构建

### -pl：指定项目

```bash
# 只构建指定模块
mvn clean install -pl module-a

# 构建多个模块
mvn clean install -pl module-a,module-b
```

### -am：同时构建依赖

```bash
# 构建 module-a 及其依赖
mvn clean install -pl module-a -am
```

### -pl !：排除项目

```bash
# 排除某个模块
mvn clean install -pl !module-to-exclude
```

## 调试与诊断

### -X：调试模式

```bash
# 输出详细调试信息
mvn clean package -X
```

### -e：显示错误堆栈

```bash
# 显示错误详情
mvn clean package -e
```

### -o：离线模式

```bash
# 使用本地仓库，不连接远程仓库
mvn clean package -o
```

:::warning 离线模式限制
如果依赖在本地仓库不存在，离线模式会失败。这个模式适合本地已有所有依赖的情况下加速构建。
:::

## 跳过测试

| 命令 | 效果 |
|---|---|
| `-DskipTests` | 跳过测试执行，但编译测试代码 |
| `-Dmaven.test.skip=true` | 跳过测试编译和执行 |
| `-Dtest=MyTest` | 只运行指定测试类 |
| `-Dtest=MyTest#method` | 只运行指定测试方法 |

```bash
# 跳过所有测试
mvn clean package -DskipTests

# 只运行特定测试
mvn test -Dtest=UserServiceTest

# 运行多个测试
mvn test -Dtest=UserServiceTest,OrderServiceTest
```

## 常用参数汇总

```bash
# 完整示例：并行构建，跳过测试，指定 Java 版本
mvn clean verify \
    -T 4 \
    -DskipTests \
    -Dmaven.compiler.source=17 \
    -Dmaven.compiler.target=17
```

| 参数 | 说明 |
|---|---|
| `-T <线程数>` | 并行构建线程数 |
| `-DskipTests` | 跳过测试 |
| `-o` | 离线模式 |
| `-U` | 强制更新 SNAPSHOT |
| `-q` | 安静模式（减少输出） |
| `-B` | 批处理模式（非交互） |

## 在 CI/CD 中的使用

### Jenkinsfile 示例

```groovy title="Jenkinsfile"
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package -DskipTests -B'
            }
        }

        stage('Test') {
            steps {
                sh 'mvn test -B'
            }
            post {
                always {
                    junit '**/target/surefire-reports/*.xml'
                }
            }
        }

        stage('Deploy to Nexus') {
            when {
                branch 'main'
            }
            steps {
                sh 'mvn deploy -DskipTests -B'
            }
        }
    }
}
```

### GitLab CI 示例

```yaml title=".gitlab-ci.yml"
variables:
  MAVEN_OPTS: "-Dmaven.repo.local=.m2/repository"

cache:
  paths:
    - .m2/repository/
    - target/

build:
  stage: build
  script:
    - mvn clean package -DskipTests -B

test:
  stage: test
  script:
    - mvn test -B
  artifacts:
    reports:
      junit: target/surefire-reports/*.xml
```
