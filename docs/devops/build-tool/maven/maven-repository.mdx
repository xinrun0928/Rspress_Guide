# Maven 仓库管理

Maven 仓库是依赖的存储和分发中心。本地仓库、本地私服、中央仓库——理解它们的关系和配置，是搭建可靠 CI/CD 流程的基础。

## 仓库的三种类型

### 本地仓库

本地仓库位于 `~/.m2/repository`，是 Maven 在本地的缓存。

```bash
# 查看本地仓库位置
mvn -v
# 输出包含：Local repository: /Users/username/.m2/repository
```

首次构建时，Maven 从远程仓库下载依赖到本地仓库，后续构建直接从本地读取，速度更快。

### 远程仓库

远程仓库包括中央仓库和私服：

| 类型 | 地址 | 说明 |
|---|---|---|
| 中央仓库 | `https://repo.maven.apache.org/maven2` | Maven 官方仓库 |
| 阿里云仓库 | `https://maven.aliyun.com/repository/public` | 国内镜像，加速下载 |
| Google 仓库 | `https://maven.google.com` | Android 依赖 |

### 私服

私服是企业自建的 Maven 仓库，常用软件有 Sonatype Nexus 和 JFrog Artifactory。

```xml title="pom.xml"
<repositories>
    <repository>
        <id>nexus</id>
        <url>https://company-nexus.example.com/repository/maven-public/</url>
    </repository>
</repositories>
```

## 配置镜像加速

在 `settings.xml` 中配置镜像，拦截中央仓库请求：

```xml title="~/.m2/settings.xml"
<settings>
    <mirrors>
        <mirror>
            <id>aliyun</id>
            <mirrorOf>central</mirrorOf>
            <name>Aliyun Maven</name>
            <url>https://maven.aliyun.com/repository/public</url>
        </mirror>
    </mirrors>
</settings>
```

`mirrorOf` 参数指定被拦截的仓库 ID：`central` 拦截中央仓库；`*` 拦截所有仓库。

## 私服的三大功能

### 1. 代理（Proxy）

私服代理中央仓库，缓存下载过的依赖：

```
开发者 → 私服（无缓存）→ 中央仓库
开发者 → 私服（有缓存）→ 直接返回
```

配置代理仓库：

```xml title="Nexus/Artifactory 配置"
<remote-storage>
    <url>https://repo.maven.apache.org/maven2/</url>
</remote-storage>
```

### 2. 托管（Hosted）

托管公司内部的私有 JAR 包：

```bash
# 安装本地 JAR 到本地仓库
mvn install:install-file \
    -Dfile=/path/to/library.jar \
    -DgroupId=com.company \
    -DartifactId=internal-lib \
    -Dversion=1.0.0 \
    -Dpackaging=jar

# 或者部署到私服
mvn deploy:deploy-file \
    -Dfile=/path/to/library.jar \
    -DgroupId=com.company \
    -DartifactId=internal-lib \
    -Dversion=1.0.0 \
    -Dpackaging=jar \
    -Durl=https://company-nexus.example.com/repository/maven-releases/ \
    -DrepositoryId=nexus-releases
```

### 3. 聚合（Group）

聚合多个仓库，提供统一访问入口：

```
Group: maven-public
├── 代理：Maven Central
├── 代理：Google
└── 托管：company-releases
└── 托管：company-snapshots
```

## Release 与 Snapshot 仓库

Maven 区分 Release（正式版）和 Snapshot（快照版）仓库：

| 类型 | 说明 | 版本示例 |
|---|---|---|
| Release | 正式发布的稳定版本 | `1.0.0`、`2.3.5` |
| Snapshot | 开发期间的频繁更新版本 | `1.0.0-SNAPSHOT` |

配置多个仓库：

```xml title="pom.xml"
<distributionManagement>
    <repository>
        <id>nexus-releases</id>
        <url>https://company-nexus.example.com/repository/maven-releases/</url>
    </repository>
    <snapshotRepository>
        <id>nexus-snapshots</id>
        <url>https://company-nexus.example.com/repository/maven-snapshots/</url>
    </snapshotRepository>
</distributionManagement>
```

## 认证配置

访问私服需要认证：

```xml title="~/.m2/settings.xml"
<settings>
    <servers>
        <server>
            <id>nexus-releases</id>
            <username>admin</username>
            <password>admin123</password>
        </server>
        <server>
            <id>nexus-snapshots</id>
            <username>admin</username>
            <password>admin123</password>
        </server>
    </servers>
</settings>
```

:::warning 安全建议
不要在 `settings.xml` 中明文存储密码。使用 `mvn -encrypt-password` 加密密码，或使用 Maven Credential Helper。
:::

## 常用命令

```bash
# 查看本地仓库中的依赖
mvn dependency:list

# 复制依赖到指定目录
mvn dependency:copy-dependencies -DoutputDirectory=target/lib

# 解析依赖（显示实际版本）
mvn dependency:resolve

# 查看依赖树
mvn dependency:tree

# 分析依赖
mvn dependency:analyze
```
