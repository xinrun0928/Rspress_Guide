# Maven 指南

Apache Maven 是 Java 生态最成熟的构建工具。自 2004 年发布以来，Maven 凭借其强大的依赖管理、约定优于配置的理念，以及丰富的插件生态，成为企业级 Java 项目的首选构建工具。

即便 Gradle 在灵活性和构建速度上更具优势，Maven 依然是大多数 Java 项目的选择。这不仅因为 Maven 的稳定性，更因为其完善的企业级特性和社区支持。

## Maven 的核心优势

为什么在 Gradle 出现多年后，Maven 仍然占据主导地位？

**成熟的依赖管理体系**。Maven 的依赖管理是其最核心的优势。通过中央仓库，`pom.xml` 声明的依赖可以一键下载、自动管理传递依赖、解决版本冲突。这套机制经过十几年打磨，稳定性极高。

**约定优于配置**。Maven 规定了标准的项目结构、构建生命周期、目录布局。新成员加入项目时，不需要理解复杂的构建脚本，按照约定即可工作。这降低了团队协作的摩擦。

**丰富的插件生态**。Maven 有超过 3000 个插件，覆盖编译、测试、打包、部署的各个环节。主流框架（Spring、Hibernate、MyBatis）都提供官方的 Maven 插件。

**稳定性和可预测性**。Maven 的构建过程是确定的，同样的 POM 配置产生同样的产物。对于追求稳定性的企业项目，这一点至关重要。

**依赖管理**篇是 Maven 最核心的内容。传递依赖、依赖调解、依赖冲突排查——这些技能在实际项目中天天用到。

**作用域**篇详细讲解 Maven 的六种依赖作用域，理解它们才能正确配置依赖。

**生命周期**篇介绍 Maven 的三套生命周期及其阶段，理解生命周期是掌握 Maven 的关键。

**插件机制**篇讲解插件如何绑定到生命周期、自定义插件的开发方法。

**POM 高级特性**篇涵盖 `dependencyManagement`、`build` 配置、`profiles` 等实用技巧。

**多模块项目**篇讲解父子 POM 的设计模式，这是大型项目必备的技能。

**仓库管理**篇介绍如何搭建 Nexus/Artifactory 私服，优化依赖下载速度。

**BOM 模式**篇讲解如何用 BOM 统一管理依赖版本，这是 Spring Boot 的核心技术。

**命令与性能**篇汇总常用命令，讲解加速构建的技巧。

无论你是 Maven 新手还是希望进阶的老兵，这一章节都将帮助你更好地掌握 Maven。
