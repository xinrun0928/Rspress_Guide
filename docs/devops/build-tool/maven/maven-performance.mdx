# Maven 性能优化

大型项目的 Maven 构建可能需要几分钟甚至更长时间。优化构建性能，能显著提升开发效率和 CI/CD 速度。

## 并行构建

### -T 参数

Maven 支持并行构建：

```bash
# 使用 4 个线程
mvn clean install -T 4

# 根据 CPU 核心数自动确定线程数
mvn clean install -T 1C

# 使用 2 个线程并行构建
mvn clean install -T 2
```

### 线程数选择

| 项目规模 | 推荐线程数 |
|---|---|
| 小型项目（< 10 模块） | `1C` 或 `2` |
| 中型项目（10-50 模块） | `2C` 或 `4` |
| 大型项目（> 50 模块） | `CPU 核心数 - 1` |

:::warning 线程安全问题
并行构建要求模块间没有循环依赖。如果存在循环依赖，并行构建可能失败或产生不一致结果。先用 `mvn reactor:make -Dmake.order=reverse` 检查依赖关系。
:::

## 增量编译

### maven-compiler-plugin 增量编译

```xml title="pom.xml"
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <version>3.10.1</version>
    <configuration>
        <useIncrementalCompilation>true</useIncrementalCompilation>
        <fork>true</fork>
        <compilerArgs>
            <arg>-Xlint:none</arg>
        </compilerArgs>
    </configuration>
</plugin>
```

### 小技巧：跳过不必要的阶段

```bash
# 只需要 class 文件，不打包
mvn compiler:compile

# 只编译测试，不运行
mvn compiler:testCompile

# 跳过 javadoc 生成
mvn package -Dmaven.javadoc.skip=true
```

## 本地仓库优化

### 使用本地镜像

国内用户使用阿里云镜像：

```xml title="~/.m2/settings.xml"
<settings>
    <mirrors>
        <mirror>
            <id>aliyun</id>
            <mirrorOf>central</mirrorOf>
            <name>Aliyun Maven</name>
            <url>https://maven.aliyun.com/repository/public</url>
        </mirror>
        <mirror>
            <id>aliyun-spring</id>
            <mirrorOf>spring.milestones,spring.plugins.release</mirrorOf>
            <name>Aliyun Spring</name>
            <url>https://maven.aliyun.com/repository/spring</url>
        </mirror>
    </mirrors>
</settings>
```

### 离线模式

本地仓库完备后，使用离线模式：

```bash
mvn clean package -o
```

## 依赖下载优化

### 批量下载

```xml title="pom.xml"
<settings>
    <profiles>
        <profile>
            <id>download</id>
            <properties>
                <maven.resolver.transport>wagon</maven.resolver.transport>
            </properties>
        </profile>
    </profiles>
</settings>
```

### 预下载依赖

CI 服务器启动时预下载依赖：

```bash
# 先下载所有依赖
mvn dependency:go-offline

# 然后在后续构建中使用离线模式
mvn package -o
```

## 构建缓存服务器

对于超大型项目，可以使用构建缓存服务器：

| 产品 | 厂商 |
|---|---|
| Sonatype Nexus Repository | Sonatype |
| JFrog Mission Control | JFrog |
| Apache Maven Repository | Apache（仅代理） |

### 配置构建缓存

```xml title="pom.xml"
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <useIncrementalCompilation>true</useIncrementalCompilation>
        <fork>true</fork>
        <meminitial>512m</meminitial>
        <maxmem>2048m</maxmem>
    </configuration>
</plugin>
```

## JVM 参数调优

### 设置 Maven 进程内存

```bash title="~/.m2/.mvn/jvm.config"
-Xms2g
-Xmx4g
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
```

### MAVEN_OPTS 环境变量

```bash title="~/.bash_profile"
export MAVEN_OPTS="-Xmx2g -Xms512m"
```

## 常见优化场景

### 场景一：只修改了一个类

```bash
# 编译主代码（不运行测试）
mvn compiler:compile

# 或只编译测试代码
mvn compiler:testCompile
```

### 场景二：跳过测试快速打包

```bash
mvn package -DskipTests -B
```

### 场景三：只分析依赖不构建

```bash
mvn dependency:analyze
mvn dependency:tree -Dincludes=com.alibaba:fastjson
```

## 性能对比

优化前后的典型对比：

| 优化项 | 优化前 | 优化后 |
|---|---|---|
| 无优化 | 10 分钟 | - |
| 并行构建 `-T 4` | 10 分钟 | 3 分钟 |
| + 阿里云镜像 | 3 分钟 | 1.5 分钟 |
| + 离线模式 | 1.5 分钟 | 30 秒 |

:::tip 最佳实践
开发阶段使用 `-T 1C -o` 离线快速构建；CI 服务器首次构建使用完整命令，后续使用离线模式；定期在构建服务器上运行 `mvn dependency:purge-local-repository` 清理无用依赖。
:::
