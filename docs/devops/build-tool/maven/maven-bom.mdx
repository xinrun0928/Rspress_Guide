# Maven BOM 模式

BOM（Bill of Materials，物料清单）是 Maven 中用于统一管理依赖版本的模式。理解 BOM，是掌握 Spring Boot 依赖管理的关键。

## 什么是 BOM

BOM 本身是一个 POM 文件，它只声明依赖和版本，不包含实际代码：

```xml title="spring-boot-dependencies-2.7.0.pom"
<!-- BOM 示例：只声明版本 -->
<project>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-dependencies</artifactId>
    <version>2.7.0</version>
    <packaging>pom</packaging>

    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>com.fasterxml.jackson</groupId>
                <artifactId>jackson-bom</artifactId>
                <version>2.13.3</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
            <dependency>
                <groupId>org.springframework</groupId>
                <artifactId>spring-core</artifactId>
                <version>5.3.20</version>
            </dependency>
            <!-- ...上百个依赖版本 -->
        </dependencies>
    </dependencyManagement>
</project>
```

引入 BOM 后，子项目声明依赖时可以省略版本号：

```xml title="pom.xml"
<project>
    <!-- 引入 Spring Boot BOM -->
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>2.7.0</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <dependencies>
        <!-- 不需要指定版本，由 BOM 管理 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- Jackson 版本也由 Spring Boot BOM 统一管理 -->
        <dependency>
            <groupId>com.fasterxml.jackson.core</groupId>
            <artifactId>jackson-databind</artifactId>
        </dependency>
    </dependencies>
</project>
```

## BOM 的优势

### 版本统一

没有 BOM 时，每个项目可能使用不同版本的 Jackson：

```
项目 A → jackson-databind 2.12.0
项目 B → jackson-databind 2.13.3
项目 C → jackson-databind 2.11.0
```

引入 BOM 后，所有项目使用统一版本：

```
Spring Boot BOM → jackson-databind 2.13.3

项目 A → 使用 2.13.3
项目 B → 使用 2.13.3
项目 C → 使用 2.13.3
```

### 依赖传递版本覆盖

BOM 会覆盖传递依赖的版本：

```
Spring Boot Starter Web
 └── spring-core 5.3.20
 └── jackson 2.12.x

Spring Boot BOM
 └── jackson 2.13.3  ← 覆盖传递依赖的版本
```

## Spring Boot Starter

Spring Boot Starter 是 BOM 模式的典型应用。每一个 Starter 都是对某个场景的依赖封装：

```xml title="pom.xml"
<!-- Spring Boot 的 BOM 通过 spring-boot-starter-parent 引入 -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.7.0</version>
</parent>
```

`spring-boot-starter-parent` 继承自 `spring-boot-dependencies`，内含 BOM 功能。

### Starter 生态

| Starter | 用途 |
|---|---|
| `spring-boot-starter-web` | Web 开发 |
| `spring-boot-starter-data-jpa` | JPA 数据库访问 |
| `spring-boot-starter-security` | 安全认证 |
| `spring-boot-starter-test` | 单元测试 |
| `spring-boot-starter-actuator` | 应用监控 |

## 自定义 BOM

公司内部可以创建自己的 BOM，统一管理内部依赖版本：

```xml title="company-bom/pom.xml"
<project>
    <groupId>com.example</groupId>
    <artifactId>company-bom</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>

    <dependencyManagement>
        <dependencies>
            <!-- 内部框架版本 -->
            <dependency>
                <groupId>com.example</groupId>
                <artifactId>framework-core</artifactId>
                <version>3.2.1</version>
            </dependency>
            <dependency>
                <groupId>com.example</groupId>
                <artifactId>framework-utils</artifactId>
                <version>2.8.0</version>
            </dependency>

            <!-- 第三方依赖版本锁定 -->
            <dependency>
                <groupId>com.alibaba</groupId>
                <artifactId>fastjson</artifactId>
                <version>1.2.83</version>
            </dependency>
        </dependencies>
    </dependencyManagement>
</project>
```

子项目引入 BOM：

```xml title="子项目 pom.xml"
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>company-bom</artifactId>
            <version>1.0.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

## 多 BOM 组合

可以同时引入多个 BOM，Maven 按声明顺序处理：

```xml title="pom.xml"
<dependencyManagement>
    <dependencies>
        <!-- 先引入 Spring Boot BOM -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>2.7.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>

        <!-- 再引入公司内部 BOM -->
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>company-bom</artifactId>
            <version>1.0.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

## BOM vs 父 POM

BOM 和父 POM 的 `<dependencies>` 都能统一管理版本，但有细微差别：

| 特性 | BOM (`import`) | 父 POM (`<dependencies>`) |
|---|---|---|
| 版本覆盖 | 覆盖传递依赖版本 | 覆盖传递依赖版本 |
| 依赖传递 | 不传递 | 传递 |
| 子模块依赖 | 需显式声明 | 自动继承 |
| 适用场景 | 统一版本管理 | 强制依赖管理 |

对于多模块项目，推荐做法：

1. **父 POM**：定义 `<dependencyManagement>` 和 `<plugins>`，子模块继承
2. **BOM POM**（可选）：作为父 POM 的一部分，管理第三方依赖版本

## 面试追问

**面试官可能问**：什么是 Maven BOM？它解决了什么问题？

BOM 是一个特殊的 POM 文件，通过 `<dependencyManagement>` 声明依赖版本但不实际引入依赖。引入 BOM 后，子项目声明相同依赖时可以省略版本号，由 BOM 统一管理。这解决了两个问题：多项目依赖版本不一致、依赖传递导致版本混乱。Spring Boot 的 BOM 机制确保了所有 Starter 使用经过测试的依赖版本组合。

**面试官可能问**：`spring-boot-starter-parent` 和 `spring-boot-dependencies` 的关系是什么？

`spring-boot-starter-parent` 继承自 `spring-boot-dependencies`，在继承基础上增加了默认插件配置、资源过滤等。换句话说，`spring-boot-dependencies` 只负责版本管理，而 `spring-boot-starter-parent` 还负责构建配置。大多数据情况下，使用 `spring-boot-starter-parent` 即可；如果你的项目需要继承其他父 POM，则需要在 `<dependencyManagement>` 中手动引入 `spring-boot-dependencies`。
