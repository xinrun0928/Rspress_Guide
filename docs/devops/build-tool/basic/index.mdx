# 构建工具基础

本节介绍 Java 构建工具的基础知识和演进历史，帮助你理解为什么会有 Maven、Gradle 这样的工具，以及它们各自解决了什么问题。

如果你对「为什么需要构建工具」还存在疑问，建议先阅读 [构建工具指南](../index)，那里有更详细的背景介绍。

## 构建工具的选择

<Badge text="Maven" type="tip" /> 和 <Badge text="Gradle" type="tip" /> 是当前 Java 生态最主流的两大构建工具。根据 JetBrains 2024 年的调查，超过 70% 的 Java 项目使用这两种工具之一。

| 特性 | Maven | Gradle |
|---|---|---|
| 学习曲线 | 平缓 | 较陡 |
| 构建速度 | 较慢 | 快（增量构建） |
| 灵活性 | 一般 | 高 |
| 插件生态 | 成熟 | 丰富 |
| 企业采用率 | 高 | 越来越高 |

选择哪个工具，取决于你的项目需求和团队情况。大多数传统企业项目仍选择 Maven，而新项目和移动端项目更倾向 Gradle。
