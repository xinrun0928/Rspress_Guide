# Gradle 脚本基础

Gradle 构建脚本定义了 Project 的构建逻辑。每个脚本都是一个程序，你可以通过 DSL 配置 Project、定义 Task、设置依赖。

理解脚本基础，是掌握 Gradle 的第一步。

## Project 和 Task

### Project

在 Gradle 中，每个参与构建的模块都是一个 Project：

```groovy title="build.gradle"
// 访问当前 Project
println "Project: ${project.name}"
println "Project dir: ${project.projectDir}"

// 设置 Project 属性
project.description = "My awesome project"
group = "com.example"
version = "1.0.0"
```

### Task

Task 是 Gradle 的工作单元：

```groovy title="build.gradle"
// 注册 Task（新版 API）
tasks.register('hello') {
    doLast {
        println 'Hello, World!'
    }
}

// 或使用旧的 register 方法
task hello {
    doLast {
        println 'Hello, World!'
    }
}
```

## Task 定义详解

### Task 完整结构

```groovy title="build.gradle"
tasks.register('myTask') {
    // Task 描述（gradle tasks 时显示）
    description = '这是一个示例任务'

    // Task 分组（用于组织任务列表）
    group = 'build'

    // Task 依赖
    dependsOn 'otherTask'

    // Task 输入（用于增量构建）
    inputs.file 'src/input.txt'
    inputs.property 'version', version

    // Task 输出（用于增量构建）
    outputs.file 'build/output.txt'
    outputs.dir 'build/output'

    // Task 动作
    doFirst {
        println 'Task 即将执行'
    }

    doLast {
        println 'Task 执行完成'
    }
}
```

### Task 动作：doFirst 和 doLast

```groovy title="build.gradle"
tasks.register('message') {
    doFirst {
        // 在任务开始时执行
        println '开始执行'
    }

    doLast {
        // 在任务结束时执行
        println '执行结束'
    }
}

// 添加动作到已存在的 Task
tasks.register('build') {
    doLast {
        println '构建完成'
    }
}

build.doLast {
    println 'build 任务追加的动作'
}
```

## Task 依赖

### 声明依赖

```groovy title="build.gradle"
// 方式一：通过 Task 名
tasks.register('taskA') {
    doLast { println 'A' }
}

tasks.register('taskB') {
    dependsOn 'taskA'
    doLast { println 'B' }
}

// 方式二：通过 Task 对象
tasks.register('taskC') {
    dependsOn(tasks.named('taskA'))
    doLast { println 'C' }
}

// 方式三：动态依赖
tasks.register('allTasks') {
    dependsOn tasks.matching { it.name.startsWith('task') }
}
```

### Task 执行顺序

Task 的执行顺序由依赖关系决定，没有依赖的 Task 并行执行：

```groovy title="build.gradle"
tasks.register('A') { doLast { println 'A' } }
tasks.register('B') { doLast { println 'B' } }
tasks.register('C') {
    dependsOn 'A', 'B'
    doLast { println 'C (最后)' }
}
```

输出：`A`、`B` 的顺序不确定（并行），但 `C` 一定最后。

## Lifecycle Tasks

Gradle 提供内置的生命周期 Task：

| Task | 说明 |
|---|---|
| `build` | 执行完整构建 |
| `check` | 运行所有检查 |
| `clean` | 清理构建产物 |
| `assemble` | 组装所有产物 |
| `run` | 运行应用（需要 application 插件）|

### 自定义生命周期 Task

```groovy title="build.gradle"
tasks.register('deploy') {
    group = 'deployment'
    description = '部署应用到服务器'
    dependsOn 'build', 'upload'
}

tasks.register('upload') {
    doLast { println '上传中...' }
}
```

## 条件执行

### onlyIf

Task 只在条件满足时执行：

```groovy title="build.gradle"
tasks.register('deployToProd') {
    dependsOn 'build'
    doLast { println '部署到生产环境' }
}

// 只有当 version 是 Release 版本时才执行
deployToProd.onlyIf {
    version.toString().contains('RELEASE')
}
```

### enabled

Task 可以被禁用：

```groovy title="build.gradle"
tasks.register('report') {
    doLast { println '生成报告' }
}

// 禁用 report 任务
report.enabled = false

// 或动态决定
report.enabled = project.hasProperty('generateReport')
```

## 默认 Task

项目默认提供的 Task（由插件提供）：

```groovy title="build.gradle"
plugins {
    id 'java'
}
```

内置 Task：

| Task | 插件 | 说明 |
|---|---|---|
| `compileJava` | Java | 编译 Java 源码 |
| `processResources` | Java | 复制资源文件 |
| `classes` | Java | 组装 classes |
| `jar` | Java | 创建 JAR 文件 |
| `test` | Java | 运行测试 |
| `build` | Java | 执行完整构建 |
| `clean` | Base | 清理输出 |

## 面试追问

**面试官可能问**：Gradle 的 Task 和 Maven 的 Phase/Goal 有什么区别？

这是理解两个工具区别的关键。Maven 的 Phase 是生命周期中的一个步骤，本身不执行任何操作，真正工作的是绑定到 Phase 的 Goal（插件目标）。Gradle 的 Task 是一个完整的工作单元，有输入、输出和动作。Task 可以独立执行，也可以声明依赖关系。Gradle 的 Task 比 Maven 的 Goal 更灵活，因为你可以自定义 Task 的行为和依赖。

**面试官可能问**：如何让 Gradle Task 按特定顺序执行？

有两种方式：声明 `dependsOn` 依赖关系，或使用 `mustRunAfter`/`shouldRunAfter`/`finalizedBy` 指定执行顺序。`dependsOn` 是强制的，Task B 依赖 Task A，意味着 Task A 必须先完成。`mustRunAfter` 是建议性的，在并行执行时生效。`finalizedBy` 定义清理动作，Task 完成后自动执行。
