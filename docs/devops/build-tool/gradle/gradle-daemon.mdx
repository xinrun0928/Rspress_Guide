# Gradle Daemon 原理与优化

你有没有注意到，每次运行 `./gradlew build` 时，都有一个 Java 进程在后台默默工作？这就是 Gradle Daemon——一个让你「秒开」构建的秘密武器。

## 什么是 Gradle Daemon

### 工作原理

Daemon 是 Gradle 的常驻进程。它避免了每次构建都要重新启动 JVM 的开销：

```
无 Daemon：每次构建
├── 启动 JVM（~3-5秒）
├── 加载 Gradle 核心（~2-3秒）
├── 执行构建（~10秒）
└── JVM 退出（~0.5秒）
总计：约 15 秒

有 Daemon：后续构建
├── 连接现有 Daemon（~0.1秒）
├── 执行构建（~10秒）
└── 释放连接（~0.1秒）
总计：约 10 秒
```

看起来只省了几秒，但如果你每天构建 50 次，就是 4 分钟的差距。

### Daemon 日志位置

```bash
# 查看 Daemon 日志
~/.gradle/daemon/<version>/
├── daemon.out.log      # 标准输出
├── daemon.err.log      # 标准错误
└── daemon-<pid>.log    # 进程日志
```

## Daemon 配置

### 启用与禁用

```groovy title="gradle.properties"
# 启用 Daemon（默认开启）
org.gradle.daemon=true

# 禁用 Daemon
org.gradle.daemon=false
```

命令行覆盖：

```bash
# 临时禁用 Daemon
./gradlew build --no-daemon

# 强制启动新的 Daemon
./gradlew build --foreground
```

### 内存配置

```groovy title="gradle.properties"
# Daemon 堆内存
org.gradle.jvmargs=-Xmx2048m -Dfile.encoding=UTF-8

# Daemon 堆内存（Gradle 8+ 推荐）
org.gradle.heap.size.default=2g
```

:::tip
Gradle 8+ 使用 `org.gradle.heap.size.default` 代替 JVM args 中的 `-Xmx`，可以避免 Daemon 和子进程的混淆。
:::

### 并行构建

```groovy title="gradle.properties"
# 允许多个项目并行构建
org.gradle.parallel=true

# 并行构建时的线程数
org.gradle.workers.max=8
```

## Daemon 状态管理

### 查看 Daemon 状态

```bash
./gradlew --status

# 输出示例
   PID | Status   | Kind     | JDK | Daemon addr
 12345 | Busy     | Gradle 8 | 17  | 127.0.0.1:56321
 12346 | Idle     | Gradle 8 | 17  | 127.0.0.1:56322
```

### 停止 Daemon

```bash
# 停止所有 Daemon
./gradlew --stop

# 或
gradle --stop
```

## Daemon 内存调优

### 分析 Daemon 内存使用

```groovy title="build.gradle"
tasks.register('diagnoseMemory') {
    doLast {
        def runtime = Runtime.runtime
        println "Max Memory: ${runtime.maxMemory() / 1024 / 1024} MB"
        println "Total Memory: ${runtime.totalMemory() / 1024 / 1024} MB"
        println "Free Memory: ${runtime.freeMemory() / 1024 / 1024} MB"
        println "Available Processors: ${runtime.availableProcessors()}"
    }
}
```

### 内存泄漏排查

```groovy title="gradle.properties"
# 开启 GC 日志
org.gradle.jvmargs=-Xmx4g -XX:+HeapDumpOnOutOfMemoryError \
  -XX:HeapDumpPath=$HOME/gradle-oom.hprof \
  -XX:+PrintGCDetails -XX:+PrintGCDateStamps \
  -Xloggc:$HOME/gradle-gc.log
```

## Daemon 与 CI/CD

### CI 环境中禁用 Daemon

CI 环境通常每次构建都是新进程，Daemons 可能无法有效复用：

```groovy title="gradle.properties"
# CI 环境检测
org.gradle.daemon=${CI:-false}
```

```yaml title=".gitlab-ci.yml"
build:
  variables:
    CI: "true"
  script:
    - ./gradlew build
```

### 容器环境优化

Docker 容器中运行 Gradle：

```dockerfile title="Dockerfile"
FROM eclipse-temurin:17-jdk

# 设置 Gradle 堆内存
ENV GRADLE_OPTS="-Xmx2g -Dorg.gradle.daemon=false"

WORKDIR /app
COPY . .
RUN ./gradlew dependencies --no-daemon
```

:::warning
在容器环境中，内存限制更严格。如果不设置合理的堆内存，OOM 会频繁发生。
:::

## Daemon 启动优化

### 预热 Daemon

```groovy title="build.gradle"
tasks.register('warmUp') {
    doLast {
        // 预编译所有配置
        tasks.configureEach { task ->
            task.enabled = false
        }
        
        // 执行基础任务预热
        project(":").tasks.named("help").get()
    }
}
```

### 构建缓存

```groovy title="gradle.properties"
# 启用构建缓存
org.gradle.caching=true

# 缓存目录
org.gradle.caches.dir=/path/to/shared/cache
```

```bash
# 导出缓存到 CI artifact
./gradlew --build-cache --parallel assemble

# 导入缓存
./gradlew --build-cache
```

## Daemon 故障排查

### Daemon 启动失败

```bash
# 查看详细错误
./gradlew build --stacktrace

# 查看 Daemon 日志
cat ~/.gradle/daemon/8.5/daemon.out.log
```

### 端口冲突

```groovy title="gradle.properties"
# 指定 Daemon 端口
org.gradle.daemon.port=54321

# 或禁用端口绑定（仅支持单 Daemon）
org.gradle.daemon.permanent=false
```

### JDK 版本不匹配

```groovy title="gradle.properties"
# 强制使用特定 JDK
org.gradle.java.home=/path/to/jdk-17
```

## 性能对比

| 场景 | 无 Daemon | 有 Daemon | 节省 |
|---|---|---|---|
| 增量构建（无变化） | 8.2s | 1.1s | 87% |
| 增量构建（单文件变更） | 12.5s | 4.8s | 62% |
| Clean 构建（缓存冷） | 45s | 38s | 16% |
| Clean 构建（缓存热） | 45s | 28s | 38% |

## 总结

Daemon 是 Gradle 性能优化的基础：

- **开发时**：始终启用 Daemon，享受「秒开」体验
- **CI 环境**：根据实际情况选择是否启用
- **内存配置**：根据项目规模调整堆内存
- **缓存配合**：配合构建缓存，效果翻倍

下次觉得 Gradle 慢的时候，先检查一下 Daemon 状态。
