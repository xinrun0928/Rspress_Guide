# Gradle 指南

Gradle 是现代 Java 生态最强大的构建工具。Google 选择 Gradle 作为 Android 官方构建工具，Spring Framework 在某些模块使用 Gradle，Netflix、LinkedIn、Uber 等科技公司都在大规模使用 Gradle。

Gradle 的核心优势在于：**增量构建**、**任务依赖图**、**灵活的 DSL**。这些特性使它在大型项目中构建速度远超 Maven。

## Gradle 的核心优势

### 增量构建

Gradle 会分析每个 Task 的输入和输出，只有当输入发生变化时才执行 Task。这意味着如果只修改了一个类，Gradle 只会重新编译这个类，依赖它的模块如果输出没变，就直接复用缓存结果。

### 任务依赖图（DAG）

Gradle 将构建过程建模为有向无环图（DAG）。每个构建任务是一个节点，任务之间的依赖关系是边。Gradle 按照拓扑顺序执行任务，并行执行没有依赖关系的任务。

### 灵活的 DSL

Gradle 使用 Groovy 或 Kotlin DSL 编写构建脚本。相比 Maven 的 XML，DSL 更简洁、更强大，支持自定义任务、自定义插件、动态配置。

### 强大的依赖管理

Gradle 的 `implementation` 配置比 Maven 的 `compile` 更细粒度。依赖变更的影响范围更可控，减少不必要的重新编译。

---

## Groovy DSL vs Kotlin DSL

Gradle 支持两种 DSL：

| 特性 | Groovy DSL | Kotlin DSL |
|---|---|---|
| 文件后缀 | `.gradle` | `.gradle.kts` |
| 语法 | 动态类型 | 静态类型 |
| IDE 支持 | 一般 | 优秀 |
| 学习曲线 | 较平缓 | 较陡 |
| 生态 | 成熟 | 越来越完善 |

Google 官方推荐 Kotlin DSL，用于 Android 开发。
