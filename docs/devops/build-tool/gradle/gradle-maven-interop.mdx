# Gradle 与 Maven 互通

「我们团队要迁移到 Gradle，但依赖还在 Maven 仓库里，能用吗？」

当然能。Gradle 从诞生之日起就支持 Maven 仓库，而且它和 Maven 的互操作性远不止「能读取 Maven POM」这么简单。

## Maven 仓库：Gradle 的天然盟友

### 配置 Maven 仓库

```groovy title="build.gradle"
repositories {
    mavenCentral()
    maven { url 'https://repo.spring.io/milestone' }
    maven { url 'https://repo.example.com/private-repo' }
}
```

这段配置和 Maven 的 `<repositories>` 完全等价。Gradle 内置了对 Maven 仓库协议的完整支持，包括认证、代理、镜像。

### 配置认证

```groovy title="build.gradle"
repositories {
    maven {
        url 'https://repo.example.com/private'
        credentials {
            username = project.findProperty('repoUsername') ?: System.getenv('REPO_USERNAME')
            password = project.findProperty('repoPassword') ?: System.getenv('REPO_PASSWORD')
        }
    }
}
```

:::warning
敏感信息不要硬编码在 `build.gradle` 中。优先使用环境变量或 Gradle properties 文件。
:::

## Maven POM 依赖导入

### 直接导入

Gradle 可以直接读取 Maven POM 并转换为 Gradle 依赖：

```groovy title="build.gradle"
dependencies {
    implementation 'com.google.guava:guava:32.1.3-jre'
}
```

背后发生的事情：

1. Gradle 向 Maven Central 发起请求：`/com/google/guava/guava/32.1.3-jre/guava-32.1.3-jre.pom`
2. 下载 POM 文件，解析 `<dependency>` 和 `<dependencyManagement>`
3. 下载对应的 JAR 文件

### BOM 支持

Maven BOM（Bill of Materials）是一种特殊的 POM，用于统一管理依赖版本：

```groovy title="build.gradle"
dependencies {
    // 方式一：Spring Boot BOM
    implementation platform('org.springframework.boot:spring-boot-dependencies:3.2.0')
    
    // 方式二：Dependency Management 插件
    implementation platform('io.spring.platform:platform-bom:2023.0.0')
}

dependencyManagement {
    imports {
        mavenBom 'org.springframework.cloud:spring-cloud-dependencies:2023.0.0'
    }
}
```

## 发布到 Maven 仓库

### 配置 Maven Publish 插件

```groovy title="build.gradle"
plugins {
    id 'maven-publish'
}

publishing {
    publications {
        mavenJava(MavenPublication) {
            from components.java
            
            pom {
                name = 'My Library'
                description = 'A library published by Gradle'
                url = 'https://example.com/library'
                
                licenses {
                    license {
                        name = 'Apache-2.0'
                        url = 'https://www.apache.org/licenses/LICENSE-2.0'
                    }
                }
                
                developers {
                    developer {
                        id = 'developer'
                        name = 'Developer Name'
                        email = 'dev@example.com'
                    }
                }
                
                scm {
                    connection = 'scm:git:https://example.com/repo.git'
                    developerConnection = 'scm:git:https://example.com/repo.git'
                    url = 'https://example.com/repo'
                }
            }
        }
    }
    
    repositories {
        maven {
            name = 'MavenCentral'
            url = version.endsWith('SNAPSHOT') 
                ? 'https://s01.oss.sonatype.org/content/repositories/snapshots'
                : 'https://s01.oss.sonatype.org/service/local/staging/deploy/maven2/'
            
            credentials {
                username = project.findProperty('mavenCentralUsername')
                password = project.findProperty('mavenCentralPassword')
            }
        }
    }
}
```

### 签名配置

```groovy title="build.gradle"
plugins {
    id 'signing'
}

signing {
    sign publishing.publications.mavenJava
}

gradle.taskGraph.whenReady { taskGraph ->
    if (taskGraph.hasTask(':publish')) {
        println 'Publishing task detected - ensure signing is configured'
    }
}
```

## 混合使用：Maven 模块 + Gradle 构建

### 从 Maven 模块继承

假设你有一个 Maven 模块需要迁移到 Gradle：

```xml title="maven-library/pom.xml"
<?xml version="1.0" encoding="UTF-8"?>
<project>
    <groupId>com.example</groupId>
    <artifactId>maven-library</artifactId>
    <version>1.0.0</version>
    
    <dependencies>
        <dependency>
            <groupId>org.apache.commons</groupId>
            <artifactId>commons-lang3</artifactId>
            <version>3.14.0</version>
        </dependency>
    </dependencies>
</project>
```

迁移后的 Gradle 配置：

```groovy title="maven-library/build.gradle"
plugins {
    id 'java-library'
}

java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.apache.commons:commons-lang3:3.14.0'
}
```

### Gradle 生成 Gradle Module Metadata

Gradle 5.0+ 引入了 Gradle Module Metadata（`.module.json`），类似于 Maven POM，但支持更丰富的语义：

```groovy title="build.gradle"
plugins {
    id 'java-library'
    id 'maven-publish'
}

java {
    withSourcesJar()
    withJavadocJar()
}

publishing {
    publications {
        mavenJava(MavenPublication) {
            from components.java
            
            // Gradle Module Metadata 会自动生成
            // 包含 variants、constraints 等信息
        }
    }
}
```

## 依赖解析差异与桥接

### Scope 映射

Maven 的 scope 和 Gradle 的配置不完全对应：

| Maven Scope | Gradle 配置 | 说明 |
|---|---|---|
| compile | implementation | 编译和运行时可见 |
| provided | compileOnly | 编译时需要，运行时由容器提供 |
| runtime | runtimeOnly | 仅运行时需要 |
| test | testImplementation | 仅测试时需要 |
| system | 无等价物 | 需要用 `files()` 或 `flatDir` |

### 依赖冲突解决

```groovy title="build.gradle"
configurations.all {
    resolutionStrategy {
        // 强制使用特定版本
        force 'com.google.guava:guava:32.1.3-jre'
        
        // 替换传递依赖
        dependencySubstitution {
            substitute(module('com.example:legacy:1.0'))
                .using(module('com.example:new:2.0'))
        }
        
        // 缓存刷新
        cacheChangingModulesFor 0, 'seconds'
        cacheDynamicVersionsFor 0, 'seconds'
    }
}
```

## 实战：Maven 到 Gradle 迁移策略

### 第一步：创建 build.gradle

```groovy title="build.gradle"
plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
    id 'io.spring.dependency-management' version '1.1.4'
}

group = 'com.example'
version = '1.0.0'

repositories {
    mavenCentral()
}

dependencies {
    implementation platform('org.springframework.boot:spring-boot-dependencies:3.2.0')
    
    implementation 'org.springframework.boot:spring-boot-starter-web'
}
```

### 第二步：运行 Gradle 初始化

```bash
# 生成 wrapper
gradle wrapper

# 验证构建
./gradlew build
```

### 第三步：逐个转换依赖

Maven：

```xml
<dependency>
    <groupId>com.google.code.gson</groupId>
    <artifactId>gson</artifactId>
    <version>2.10.1</version>
</dependency>
```

Gradle：

```groovy
implementation 'com.google.code.gson:gson:2.10.1'
```

### 第四步：验证

```bash
./gradlew dependencies --configuration runtimeClasspath
```

## 总结

Gradle 与 Maven 的互通性是渐进式迁移的基础。你不需要「全量重写」，而是可以：

1. **先用 Gradle 构建 Maven 项目**：验证依赖解析是否正确
2. **逐步迁移依赖配置**：从 POM 到 `build.gradle`
3. **发布回 Maven 仓库**：让其他 Maven 项目继续使用你的库

这不是非此即彼的选择，而是渐进式的共存。
