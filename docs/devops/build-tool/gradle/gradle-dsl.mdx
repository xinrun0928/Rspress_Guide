# Gradle DSL 语法

Gradle 的构建脚本可以用两种语言写：Groovy DSL（`.gradle`）和 Kotlin DSL（`.gradle.kts`）。Kotlin DSL 提供了更好的 IDE 支持（类型安全、自动补全），而 Groovy DSL 更加简洁。

## Groovy DSL vs Kotlin DSL

```groovy
// Groovy DSL (.gradle)
plugins {
    id 'java'
}
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
}
```

```kotlin
// Kotlin DSL (.gradle.kts)
plugins {
    java
}
dependencies {
    implementation("org.springframework.boot:spring-boot-starter-web")
}
```

### 核心差异

| 维度 | Groovy DSL | Kotlin DSL |
|---|---|---|
| **文件后缀** | `.gradle` | `.gradle.kts` |
| **语法风格** | 动态语言，灵活 | 静态语言，类型安全 |
| **IDE 支持** | 一般 | 优秀（自动补全、跳转） |
| **编译时检查** | 无 | 有 |
| **学习曲线** | 平缓 | 较陡（需要 Kotlin 基础） |

### 选择建议

- 新项目：推荐 Kotlin DSL（更好的 IDE 支持和类型安全）
- 已有 Groovy 项目：可以逐步迁移，或保持现状
- 追求简洁：Groovy DSL

## 基础语法

### 项目配置

```groovy
// Groovy DSL
plugins {
    id 'java'
    id 'application'
}
group 'com.example'
version '1.0.0'

// 项目属性
base {
    group = 'com.example'
    version = '1.0.0'
    archivesBaseName = 'myapp'
}
```

```kotlin
// Kotlin DSL
plugins {
    java
    application
}
group = "com.example"
version = "1.0.0"
```

### 仓库配置

```groovy
// Groovy DSL
repositories {
    mavenCentral()
    google()
    maven { url 'https://jitpack.io' }
    maven {
        name = 'Aliyun'
        url = uri('https://maven.aliyun.com/repository/public')
    }
}
```

```kotlin
// Kotlin DSL
repositories {
    mavenCentral()
    google()
    maven { url = uri("https://jitpack.io") }
    maven {
        name.set("Aliyun")
        url.set(uri("https://maven.aliyun.com/repository/public"))
    }
}
```

### 依赖配置

```groovy
// Groovy DSL
dependencies {
    // 依赖配置
    implementation 'org.springframework.boot:spring-boot-starter-web'
    api 'com.google.guava:guava:32.1.2-jre'
    runtimeOnly 'mysql:mysql-connector-java:8.0.33'
    compileOnly 'org.projectlombok:lombok'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'

    // 配置类路径
    classpath 'com.github.jengelman.gradle.plugins:shadow:7.1.2'
}
```

```kotlin
// Kotlin DSL
dependencies {
    // 依赖配置
    implementation("org.springframework.boot:spring-boot-starter-web")
    api("com.google.guava:guava:32.1.2-jre")
    runtimeOnly("mysql:mysql-connector-java:8.0.33")
    compileOnly("org.projectlombok:lombok")
    testImplementation("org.springframework.boot:spring-boot-starter-test")
}
```

### 依赖配置隔离（Java Library 插件）

Java Library 插件引入了 `api` 配置，解决了依赖传递的透明性问题：

```groovy
// Groovy DSL
plugins {
    id 'java-library'
}
dependencies {
    api 'org.apache.commons:commons-lang3:3.12.0'  // 传递依赖
    implementation 'org.slf4j:slf4j-api:2.0.7'    // 不传递
}
```

```kotlin
// Kotlin DSL
plugins {
    java-library
}
dependencies {
    api("org.apache.commons:commons-lang3:3.12.0")  // 传递依赖
    implementation("org.slf4j:slf4j-api:2.0.7")    // 不传递
}
```

### 任务定义

```groovy
// Groovy DSL
task hello {
    doLast {
        println 'Hello, Gradle!'
    }
}

// 简写
task hello << {
    println 'Hello, Gradle!'  // << 已废弃
}
```

```kotlin
// Kotlin DSL
tasks.register("hello") {
    doLast {
        println("Hello, Gradle!")
    }
}

// 或使用 tasks.named
tasks.named("hello") {
    doLast {
        println("Hello, Gradle!")
    }
}
```

## 高级语法

### 条件配置

```groovy
// Groovy DSL
tasks.withType(JavaCompile) {
    options.encoding = 'UTF-8'
    if (project.hasProperty('release')) {
        options.compilerArgs.add('-release')
    }
}
```

```kotlin
// Kotlin DSL
tasks.withType<JavaCompile> {
    options.encoding = "UTF-8"
    if (project.hasProperty("release")) {
        options.compilerArgs.add("-release")
    }
}
```

### 扩展属性

```groovy
// Groovy DSL
ext {
    springVersion = '6.0.9'
    lombokVersion = '1.18.28'
}
dependencies {
    implementation "org.springframework:spring-core:$springVersion"
}
```

```kotlin
// Kotlin DSL
extra["springVersion"] = "6.0.9"
extra["lombokVersion"] = "1.18.28"
dependencies {
    implementation("org.springframework:spring-core:${project.extra["springVersion"]}")
}
```

### 自定义任务

```groovy
// Groovy DSL
class DeployTask extends DefaultTask {
    @Input
    String environment
    @Input
    String artifact

    @TaskAction
    void deploy() {
        println "Deploying $artifact to $environment"
    }
}

task deploy(type: DeployTask) {
    environment = 'production'
    artifact = 'myapp-1.0.0.jar'
}
```

```kotlin
// Kotlin DSL
abstract class DeployTask : DefaultTask() {
    @get:Input
    abstract var environment: String
    @get:Input
    abstract var artifact: String

    @TaskAction
    fun deploy() {
        println("Deploying $artifact to $environment")
    }
}

tasks.register<DeployTask>("deploy") {
    environment = "production"
    artifact = "myapp-1.0.0.jar"
}
```

### 约定优于配置

Gradle 遵循约定优于配置的原则，标准项目结构：

```
project/
├── build.gradle
├── build.gradle.kts
├── settings.gradle
├── settings.gradle.kts
├── src/
│   ├── main/
│   │   ├── java/
│   │   ├── resources/
│   │   └── kotlin/
│   └── test/
│       ├── java/
│       └── kotlin/
├── build/
└── gradle/
    └── wrapper/
```

## Gradle 属性与配置

### gradle.properties

```properties
# gradle.properties
org.gradle.jvmargs=-Xmx4g -XX:+HeapDumpOnOutOfMemoryError
org.gradle.parallel=true
org.gradle.caching=true
org.gradle.daemon=true
```

### settings.gradle

```groovy
// Groovy DSL
rootProject.name = 'myproject'
include 'module-a', 'module-b'
project(':module-a').name = 'core'
```

```kotlin
// Kotlin DSL
rootProject.name = "myproject"
include(":module-a", ":module-b")
```

## Gradle Kotlin DSL 优势

### 类型安全

```kotlin
// Kotlin DSL 提供类型安全
// 编译时就能发现错误
dependencies {
    // 错误的配置会在 IDE 中标红
    implementation("org.springframework.boot:spring-boot-starter-web")
}
```

### 强大的 IDE 支持

```kotlin
// IDE 可以提供：
// 1. 自动补全
// 2. 类型检查
// 3. 重构支持
// 4. 跳转到定义
plugins {
    java
    `application`
}
```

### Gradle 企业版功能

```properties
# 在 gradle.properties 中启用
org.gradle.configuration-cache=true
org.gradle.configuration-cache.problems=warn
```

## 最佳实践

### 依赖管理

1. **使用 `api` vs `implementation`**：
   - Library：对外暴露的用 `api`，内部的用 `implementation`
   - Application：`implementation` 就够了
2. **统一版本管理**：通过 `extra` 属性或 BOM 管理
3. **使用 gradle.lockfile**：锁定依赖版本

### 任务组织

1. **使用任务分组**：给任务添加分组和描述
2. **避免重复任务**：使用增量构建
3. **约定优于配置**：遵循 Gradle 标准

### 构建性能

1. **启用并行构建**：`org.gradle.parallel=true`
2. **启用构建缓存**：`org.gradle.caching=true`
3. **启用配置缓存**：`org.gradle.configuration-cache=true`

---

## 总结

Groovy DSL 和 Kotlin DSL 各有优劣：

1. **Groovy DSL**：简洁灵活，适合简单项目
2. **Kotlin DSL**：类型安全，IDE 支持好，适合大型项目

选择时需要考虑团队的技术背景、项目规模和长期维护成本。
