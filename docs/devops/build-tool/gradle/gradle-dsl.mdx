# Gradle DSL 详解

Gradle 支持两种 DSL：Groovy DSL（`.gradle`）和 Kotlin DSL（`.gradle.kts`）。理解两者的语法差异，能帮助你在实际项目中做出更好的选择。

## Groovy DSL vs Kotlin DSL

| 特性 | Groovy DSL | Kotlin DSL |
|---|---|---|
| 文件后缀 | `.gradle` | `.gradle.kts` |
| 语言基础 | Groovy | Kotlin |
| 类型系统 | 动态类型 | 静态类型 |
| IDE 支持 | 一般 | 优秀（自动补全、类型检查）|
| 学习曲线 | 较平缓 | 较陡 |
| Null 安全 | 无 | 原生支持 |
| 生态 | 成熟 | 越来越完善 |

从趋势看，Kotlin DSL 越来越受欢迎，Google 官方推荐 Android 项目使用 Kotlin DSL。

## Groovy DSL 基础

### build.gradle 语法

```groovy title="build.gradle"
plugins {
    id 'java'
    id 'org.springframework.boot' version '2.7.0'
}

group = 'com.example'
version = '1.0.0'

repositories {
    mavenCentral()
    maven { url 'https://repo.example.com' }
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web:2.7.0'
    testImplementation 'org.springframework.boot:spring-boot-starter-test:2.7.0'
}

java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}
```

### Groovy 语法特点

```groovy
// 方法调用可以省略括号
println 'Hello'

// 属性访问用点号
project.name

// 字符串插值用双引号
println "Project: ${project.name}"

// 列表和 Map
list = [1, 2, 3]
map = [a: 1, b: 2]

// 方法定义
def myMethod(param) {
    return param * 2
}
```

## Kotlin DSL 基础

### build.gradle.kts 语法

```kotlin title="build.gradle.kts"
plugins {
    java
    id("org.springframework.boot") version "2.7.0"
}

group = "com.example"
version = "1.0.0"

repositories {
    mavenCentral()
    maven { url = uri("https://repo.example.com") }
}

dependencies {
    implementation("org.springframework.boot:spring-boot-starter-web:2.7.0")
    testImplementation("org.springframework.boot:spring-boot-starter-test:2.7.0")
}

java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}
```

### Kotlin DSL 语法特点

```kotlin
// 方法调用必须括号
println("Hello")

// 命名参数
maven { url = uri("https://...") }

// 字符串模板
println("Project: ${project.name}")

// 列表和 Map
val list = listOf(1, 2, 3)
val map = mapOf("a" to 1, "b" to 2)

// Lambda 最后一个参数
dependencies {
    implementation("com.example:lib:1.0")
}
```

## 语法对比

### 定义任务

**Groovy DSL**：

```groovy title="build.gradle"
tasks.register('hello') {
    doLast {
        println 'Hello, World!'
    }
}
```

**Kotlin DSL**：

```kotlin title="build.gradle.kts"
tasks.register("hello") {
    doLast {
        println("Hello, World!")
    }
}
```

### 配置插件

**Groovy DSL**：

```groovy title="build.gradle"
plugins {
    id 'java'
    id 'org.springframework.boot' version '2.7.0'
    id 'io.spring.dependency-management' version '1.0.11.RELEASE'
}
```

**Kotlin DSL**：

```kotlin title="build.gradle.kts"
plugins {
    java
    id("org.springframework.boot") version "2.7.0"
    id("io.spring.dependency-management") version "1.0.11.RELEASE"
}
```

### 定义依赖

**Groovy DSL**：

```groovy title="build.gradle"
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web:2.7.0'
    testImplementation 'org.junit.jupiter:junit-jupiter:5.9.0'
}
```

**Kotlin DSL**：

```kotlin title="build.gradle.kts"
dependencies {
    implementation("org.springframework.boot:spring-boot-starter-web:2.7.0")
    testImplementation("org.junit.jupiter:junit-jupiter:5.9.0")
}
```

### 配置 Java 版本

**Groovy DSL**：

```groovy title="build.gradle"
java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}
```

**Kotlin DSL**：

```kotlin title="build.gradle.kts"
java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}
```

## 迁移策略

### 从 Groovy 迁移到 Kotlin

1. 重命名文件后缀

```bash
mv build.gradle build.gradle.kts
mv settings.gradle settings.gradle.kts
```

2. 更新语法

| Groovy | Kotlin DSL |
|---|---|
| `'string'` | `"string"` |
| `plugin: 'java'` | `java` |
| `dependencies { implementation 'x' }` | `dependencies { implementation("x") }` |
| `sourceCompatibility = JavaVersion.VERSION_17` | `sourceCompatibility = JavaVersion.VERSION_17` |

3. 运行迁移工具

```bash
./gradlew gradleKotlinDslMigrationReport
```

### 混合使用

可以在同一项目混用两种 DSL：

```
project/
├── build.gradle      # Groovy DSL
├── build.gradle.kts  # 不建议，会冲突
├── settings.gradle   # Groovy DSL
└── app/
    ├── build.gradle      # Groovy DSL
    └── build.gradle.kts  # 不建议，会冲突
```

## 选择建议

| 场景 | 推荐 DSL |
|---|---|
| Android 项目 | Kotlin DSL（Google 官方推荐）|
| 新项目 | Kotlin DSL |
| 团队熟悉 Kotlin | Kotlin DSL |
| 快速脚本 | Groovy DSL |
| 遗留项目 | Groovy DSL 或保持不变 |

:::tip 最佳实践
无论选择哪种 DSL，保持项目风格一致。不要在一个项目中混用两种 DSL。
:::

## 面试追问

**面试官可能问**：Groovy DSL 和 Kotlin DSL 各有什么优缺点？

这是理解 Gradle DSL 选择的关键问题。Groovy DSL 的优点是语法简洁，学习曲线平缓，适合快速编写脚本；缺点是动态类型导致 IDE 支持有限，运行时才能发现错误。Kotlin DSL 的优点是静态类型带来优秀的 IDE 支持（自动补全、类型检查、导航），代码更安全；缺点是语法稍繁琐，需要 Kotlin 基础知识。从长期维护角度，Kotlin DSL 更适合大型项目。

**面试官可能问**：Gradle 的 DSL 是如何工作的？

Gradle DSL 本质上是 Kotlin 或 Groovy 的语言特性。`build.gradle` 文件实际上是一个 Groovy 脚本，`build.gradle.kts` 是一个 Kotlin 脚本。DSL 的简洁语法（如 `implementation 'x'`）是通过方法重载、属性访问约定等语言特性实现的。Gradle 提供了一组约定和方法，使得 DSL 看起来像是专门的语言，但实际上只是宿主语言（Grovy/Kotlin）的巧妙运用。
