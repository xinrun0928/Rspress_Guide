# Gradle 插件开发

你有没有想过——当 Gradle 内置的任务和配置不能满足你的需求时，该怎么办？

Gradle 的插件系统就是来解决这个问题的。通过编写自定义插件，你可以将重复的构建逻辑封装起来，在多个项目间复用。

## 插件基础

### 插件类型

| 类型 | 说明 | 适用场景 |
|---|---|---|
| **Script Plugin** | `.gradle` 文件，直接引用 | 简单的构建逻辑 |
| **Binary Plugin** | 实现 `Plugin` 接口的类 | 复杂的、可复用的插件 |
| **Precompiled Script Plugin** | `.gradle.kts` 编译为插件 | Kotlin DSL 插件开发 |

### 插件的三个阶段

```
┌─────────────────────────────────────────────────┐
│                 Plugin Application               │
│                                                  │
│  1. 解析阶段 (Resolve)                          │
│     插件从仓库下载，类加载                        │
│                                                  │
│  2. 应用阶段 (Apply)                            │
│     plugin.apply(project) 被调用                 │
│     添加扩展、创建 Task、配置项目                │
│                                                  │
│  3. 配置阶段 (Configure)                         │
│     用户配置 extension，插件应用配置              │
└─────────────────────────────────────────────────┘
```

## 编写 Binary Plugin

### 创建插件项目结构

```
my-plugin/
├── build.gradle
├── settings.gradle
└── src/
    └── main/
        └── groovy/
            └── com/example/
                └── MyPlugin.groovy
```

### 实现 Plugin 接口

```groovy
// Groovy DSL
package com.example

import org.gradle.api.Plugin
import org.gradle.api.Project

class MyPlugin implements Plugin<Project> {

    @Override
    void apply(Project project) {
        // 1. 创建扩展
        def extension = project.extensions.create('myPlugin', MyPluginExtension)

        // 2. 创建配置任务
        project.task('myPluginConfig') {
            doLast {
                println "MyPlugin 配置: ${extension.message}"
            }
        }

        // 3. 创建主要任务
        project.tasks.register('myPluginTask') {
            dependsOn project.tasks.named('myPluginConfig')
            doLast {
                println "执行 MyPlugin 任务"
                println "消息: ${extension.message}"
                println "输出目录: ${extension.outputDir}"
            }
        }
    }
}

// 扩展类
class MyPluginExtension {
    String message = "默认消息"
    String outputDir = "build/myplugin"
    boolean verbose = false
}
```

```kotlin
// Kotlin DSL
package com.example

import org.gradle.api.Plugin
import org.gradle.api.Project
import org.gradle.api.provider.Property

abstract class MyPlugin : Plugin<Project> {
    override fun apply(project: Project) {
        // 1. 创建扩展
        val extension = project.extensions.create("myPlugin", MyPluginExtension::class.java)

        // 2. 创建任务
        project.tasks.register("myPluginTask") {
            doLast {
                println("执行 MyPlugin 任务")
                println("消息: ${extension.message.get()}")
                println("输出目录: ${extension.outputDir.get()}")
            }
        }
    }
}

// 扩展类
abstract class MyPluginExtension {
    val message: Property<String> = project.objects.property(String::class).convention("默认消息")
    val outputDir: Property<String> = project.objects.property(String::class).convention("build/myplugin")
    val verbose: Property<Boolean> = project.objects.property(Boolean::class).convention(false)
}
```

### 发布插件


## 使用 Plugin DSL 应用插件

### 传统方式 vs Plugin DSL

```groovy
// 传统方式（Build Script）
buildscript {
    repositories {
        mavenCentral()
    }
    dependencies {
        classpath 'com.example:my-plugin:1.0.0'
    }
}
apply plugin: 'com.example.my-plugin'
```

```groovy
// Plugin DSL（现代方式）
plugins {
    id 'com.example.my-plugin' version '1.0.0'
}
```

### 配置插件仓库

```groovy
// settings.gradle
pluginManagement {
    repositories {
        gradlePluginPortal()
        mavenCentral()
        maven {
            url = uri('https://maven.example.com/releases')
        }
    }
    resolutionStrategy {
        eachPlugin {
            if (requested.id.id == 'com.example.my-plugin') {
                useModule("com.example:my-plugin:${requested.version}")
            }
        }
    }
}
```

## 预编译脚本插件（BuildSrc）

### 什么是 BuildSrc

BuildSrc 是 Gradle 的一个特殊目录，其中的代码会在构建脚本之前编译，可作为插件或共享代码使用。

```
project/
├── buildSrc/
│   ├── build.gradle
│   └── src/
│       └── main/
│           └── groovy/
│               └── com/example/
│                   └── MyBuildPlugin.groovy
├── build.gradle
└── settings.gradle
```

### 常用场景

```groovy
// buildSrc/src/main/groovy/com/example/VersionCatalog.groovy
package com.example

class VersionCatalog {
    static def springBoot = '3.1.2'
    static def kotlin = '1.9.0'
    static def jUnit = '5.10.0'
}
```

```groovy
// 使用
plugins {
    id 'org.springframework.boot' version VersionCatalog.springBoot
}
dependencies {
    implementation "org.jetbrains.kotlin:kotlin-stdlib:${VersionCatalog.kotlin}"
}
```

## 插件与 Extension 配合

### 完整的插件示例

```groovy
// MyReportingPlugin.groovy
package com.example

import org.gradle.api.Plugin
import org.gradle.api.Project
import org.gradle.api.tasks.bundling.Zip

class MyReportingPlugin implements Plugin<Project> {

    @Override
    void apply(Project project) {
        // 创建扩展
        def extension = project.extensions.create('reporting', ReportingExtension)

        // 创建配置任务
        def configTask = project.tasks.register('configReport', ConfigReportTask) {
            outputFile.set(project.layout.buildDirectory.file("report-config.json"))
        }

        // 创建打包任务
        def packageTask = project.tasks.register('packageReport', Zip) {
            dependsOn configTask
            from extension.reportDir
            archiveFileName.set("report-${extension.reportType.get()}.zip")
            destinationDirectory.set(project.layout.buildDirectory.dir("dist"))
        }

        // 配置项目
        project.tasks.named('check') {
            dependsOn packageTask
        }
    }
}

// 扩展类
class ReportingExtension {
    final Property<String> reportType = project.objects.property(String).convention("summary")
    final Property<String> reportDir = project.objects.property(String).convention("build/reports")

    void setReportType(String type) {
        reportType.set(type)
    }
}
```

```kotlin
// MyReportingPlugin.kt
package com.example

import org.gradle.api.Plugin
import org.gradle.api.Project
import org.gradle.api.provider.Property
import org.gradle.api.tasks.bundling.Zip

class MyReportingPlugin : Plugin<Project> {
    override fun apply(project: Project) {
        // 创建扩展
        val extension = project.extensions.create("reporting", ReportingExtension::class.java, project)

        // 创建任务
        val packageTask = project.tasks.register<Zip>("packageReport") {
            from(extension.reportDir)
            archiveFileName.set("report-${extension.reportType.get()}.zip")
            destinationDirectory.set(project.layout.buildDirectory.dir("dist"))
        }

        // 配置项目
        project.tasks.named("check") {
            dependsOn(packageTask)
        }
    }
}

abstract class ReportingExtension(project: Project) {
    val reportType: Property<String> = project.objects.property(String::class.java).convention("summary")
    val reportDir: Property<String> = project.objects.property(String::class.java).convention("build/reports")
}
```

## 插件发布到 Gradle Plugin Portal

```groovy
// build.gradle
plugins {
    id 'groovy'
    id 'maven-publish'
    id 'com.gradle.plugin-publish' version '1.2.1'
}

group = 'com.example'
version = '1.0.0'

publishing {
    publications {
        mavenJava(MavenPublication) {
            from components.java
        }
    }
}

// Gradle Plugin Portal 配置
plugins {
    id 'com.gradle.plugin-publish'
}

gradlePlugin {
    plugins {
        myPlugin {
            id = 'com.example.my-plugin'
            displayName = 'My Custom Plugin'
            description = 'A custom Gradle plugin for reporting'
            tags = ['reporting', 'build', 'automation']
        }
    }
}
```

```bash
# 发布到 Gradle Plugin Portal
# 1. 在 https://plugins.gradle.com 注册获取 API Key
# 2. 配置 API Key
# ~/.gradle/gradle.properties
pluginPortal.token=YOUR_API_KEY
# 3. 发布
./gradlew publishPlugins
```

## 最佳实践

### 1. 使用 Extension 暴露配置

```groovy
// 好的设计：通过 Extension 配置
plugins {
    id 'com.example.my-plugin'
}
myPlugin {
    message = "Hello"
    outputDir = "build/output"
}

// 避免：直接修改 Task 属性
```

### 2. 使用 Provider API

```groovy
// 好的设计：使用 Provider
def version = project.objects.property(String)
version.set("1.0.0")

// 避免：直接赋值
def version = "1.0.0"
```

### 3. 遵循 Gradle 命名约定

- Task 名称：使用小写字母和连字符（如 `generateReport`）
- Extension 名称：使用 camelCase（如 `myReporting`）
- Task Group：使用有意义的描述（如 `reporting`、`build`）

## 总结

Gradle 插件开发的核心要点：

1. **插件类型**：Script Plugin 简单直接，Binary Plugin 可复用
2. **插件结构**：扩展 + 任务 + 配置
3. **Extension**：提供声明式配置 API
4. **BuildSrc**：共享构建逻辑的最佳实践
5. **发布**：通过 Plugin Portal 或私有仓库分发

将重复的构建逻辑封装为插件，可以大幅提升团队协作效率。
