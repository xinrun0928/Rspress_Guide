# Gradle 插件系统

你有没有想过，Gradle 是怎么做到「一行配置就支持 Spring Boot」的？为什么 `apply plugin: 'java'` 就能让项目支持 Java 编译？

答案在于 Gradle 的插件系统。这是 Gradle 区别于 Maven 最重要的特性之一——**插件是构建逻辑的复用单元**。

## 插件的本质

### 从 Task 视角看插件

当你执行 `./gradlew tasks` 时，会看到一堆预定义的 Task：compileJava、jar、javadoc……这些 Task 哪来的？答案是一个个插件。

```groovy
plugins {
    id 'java'  // 应用 Java 插件
}
```

应用 `java` 插件后，Gradle 会：

1. 创建 `compileJava` Task
2. 创建 `processResources` Task
3. 创建 `classes` Task（依赖于前两者）
4. 配置 `sourceSets`
5. 配置 `dependencies`

插件的本质就是：**一组预定义的 Task、配置和约定**。

### 插件的三种类型

| 类型 | 说明 | 示例 |
|---|---|---|
| 脚本插件 | 直接写在构建脚本中的代码 | `apply from: 'common.gradle'` |
| 对象插件（ binary ） | 编译好的插件类，打包为 jar | `id 'org.springframework.boot'` |
| 预编译脚本插件 | 用 Groovy/Kotlin 编写的插件源码 | `id 'com.example.myplugin'` |

## 插件的应用

### 基本语法

```groovy title="build.gradle"
plugins {
    id 'java'                           // 核心插件，直接用 ID
    id 'org.springframework.boot' version '3.2.0'  // 版本化插件
    id 'com.github.johnrengelman.shadow' version '8.1.1'  // 第三方插件
}
```

### 应用方式对比

```groovy
// 方式一：新语法（推荐）
plugins {
    id 'java'
}

// 方式二：旧语法
apply plugin: 'java'

// 方式三：带条件的应用
if (project.hasProperty('android')) {
    apply plugin: 'com.android.application'
}
```

### 延迟应用与强制应用

```groovy
// 延迟应用：在需要时才生效
pluginManager.withPlugin('java-library') {
    tasks.named('jar') {
        manifest.attributes['Implementation-Version'] = project.version
    }
}
```

## 常用核心插件

### Java 插件生态

```groovy title="build.gradle"
// Java 基础插件
plugins {
    id 'java'                              // Java 编译
    id 'java-library'                       // Java 库（支持 api/implementation）
    id 'application'                        // Java 应用程序
}

// 配置源代码版本
java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}

// 配置主类
application {
    mainClass = 'com.example.MainApplication'
}
```

plugins {

## 自定义插件

### 创建简单任务插件

```groovy title="buildSrc/src/main/groovy/com/example/hello-plugin.gradle"
task hello {
    doLast {
        println "Hello from custom plugin!"
    }
}
```

```groovy title="build.gradle"
plugins {
    id 'com.example.hello-plugin'
}
```

### 创建任务类型的插件

```groovy title="buildSrc/src/main/groovy/com/example/replace-task.gradle"
import org.gradle.api.DefaultTask
import org.gradle.api.tasks.TaskAction

class ReplaceTask extends DefaultTask {
    String source
    String target
    String replacement
    
    @TaskAction
    void replace() {
        def content = file(source).text
        def replaced = content.replaceAll(target, replacement)
        file(target).write(replaced)
        logger.lifecycle "Replaced '${target}' with content from '${source}'"
    }
}

tasks.register('replaceVersion', ReplaceTask) {
    source = 'build/version.tmpl'
    target = 'src/main/resources/version.txt'
}
```

### 创建真正的插件类

```groovy title="buildSrc/src/main/groovy/com/example/ReportingPlugin.groovy"
package com.example

import org.gradle.api.Plugin
import org.gradle.api.Project

class ReportingPlugin implements Plugin<Project> {
    @Override
    void apply(Project project) {
        project.extensions.create('reportConfig', ReportExtension)
        
        project.tasks.register('generateReport', GenerateReportTask) {
            it.outputDir = project.reportConfig.outputDir
            it.reportName = project.reportConfig.reportName
        }
    }
}

class ReportExtension {
    String outputDir = 'build/reports'
    String reportName = 'default-report'
}
```

## 插件的依赖管理

### 插件管理配置

```groovy title="settings.gradle"
pluginManagement {
    repositories {
        gradlePluginPortal()
        mavenCentral()
        google()
        maven { url 'https://repo.spring.io/milestone' }
    }
    
    resolutionStrategy {
        eachPlugin {
            if (requested.id.id == 'org.springframework.boot') {
                useModule("org.springframework.boot:spring-boot-gradle-plugin:${requested.version}")
            }
        }
    }
}
```

### 插件版本管理最佳实践

## 插件开发最佳实践

### 1. 约定优于配置

```groovy
// 为插件用户提供合理的默认值
class MyPlugin implements Plugin<Project> {
    @Override
    void apply(Project project) {
        def extension = project.extensions.create('myPlugin', MyPluginExtension)
        
        extension.with {
            outputDir = 'build/myplugin'
            fileName = 'report.txt'
            verbose = false
        }
        
        // 提供基于约定的默认 Task
        project.tasks.register('generateMyReport', GenerateTask) {
            it.group = 'reporting'
            it.description = 'Generate custom report'
            it.outputDir = extension.outputDir
        }
    }
}
```

### 2. 避免硬编码

```groovy
// 错误：硬编码路径
file('src/main/java')

// 正确：使用配置
sourceSets.main.java.srcDirs
```

### 3. 支持增量构建

```groovy
class MyTask extends DefaultTask {
    @InputFile
    File inputFile
    
    @OutputFile
    File outputFile
    
    @TaskAction
    void execute() {
        // Gradle 会自动处理增量更新
    }
}
```

## 总结

Gradle 插件系统是构建逻辑复用的核心。掌握插件的应用、自定义和开发，你就拥有了：

- **复用能力**：一个插件，多个项目使用
- **封装能力**：隐藏复杂配置，提供简单接口
- **扩展能力**：遇到特殊需求，自己写插件解决

下次当你觉得某个构建逻辑「要写很多次」的时候，想想：是不是该写个插件了？
