# Gradle 命令行详解

你可能每天都在用 `./gradlew`，但你知道 Gradle 命令行有多少能力吗？从任务执行到日志控制，从多项目构建到构建扫描，命令行是你掌控 Gradle 的直接窗口。

## 基础命令

### Gradle Wrapper

```bash
# Unix/macOS
./gradlew <task>

# Windows
gradlew.bat <task>

# 首次运行后会自动下载 Gradle
```

### 常用任务

```bash
# 构建项目
./gradlew build

# 清理构建产物
./gradlew clean

# 编译源代码
./gradlew compileJava

# 运行测试
./gradlew test

# 运行应用
./gradlew run

# 打包应用
./gradlew bootJar
```

### 任务选择

```bash
# 指定项目（多项目构建）
./gradlew :module-name:task-name

# 列出所有项目
./gradlew projects

# 列出所有任务
./gradlew tasks

# 列出任务详情
./gradlew tasks --all
```

## 日志与输出控制

### 日志级别

```bash
# 无日志（静默）
./gradlew build -q

# 错误日志
./gradlew build --quiet

# 基本信息
./gradlew build

# 详细信息
./gradlew build --info

# 调试级别
./gradlew build --debug
```

### 日志输出控制

```bash
# 彩色输出
./gradlew build --console=plain
./gradlew build --console=auto

# 显示任务执行时间
./gradlew build --console=verbose

# 实时渲染进度
./gradlew build --console=rich
```

### 任务执行跟踪

```bash
# 显示执行的任务链
./gradlew build --dry-run

# 显示任务执行详情
./gradlew build --info | grep "Task :"

# 显示完整的任务执行树
./gradlew build --rerun-tasks
```

## 任务过滤与排除

### 按名称过滤

```bash
# 只执行匹配的任务
./gradlew test --tests "*.IntegrationTest"

# 排除特定任务
./gradlew build -x test
./gradlew build -x checkstyleMain -x checkstyleTest
```

### 按项目过滤

```bash
# 只构建特定项目
./gradlew :web-backend:build

# 排除特定项目
./gradlew build -p web-backend --exclude-task test
```

### 条件执行

```bash
# 仅在文件变更时执行
./gradlew build --continue

# 如果有任务失败，继续执行后续任务
./gradlew build --continue
```

## 构建扫描

### 启用构建扫描

```bash
./gradlew build --scan

# 首次运行会提示接受协议
# Build scan license accepted.
```

### 上传到 Gradle Enterprise

```groovy title="gradle.properties"
# Gradle Enterprise URL
services.gradle.attributions.buildScan.url=https://scans.gradle.com

# 自定义服务器
services.gradle.attributions.buildScan.url=https://enterprise.example.com
```

构建扫描可以查看：

- 构建时间分解
- 任务执行依赖图
- 依赖解析详情
- 缓存命中率
- 并行执行效果

## 依赖管理命令

### 查看依赖树

```bash
# 查看运行时依赖
./gradlew dependencies

# 查看特定配置的依赖
./gradlew :web-backend:dependencies --configuration runtimeClasspath

# 仅显示特定依赖
./gradlew dependencies --configuration compileClasspath | grep spring

# 反向依赖（谁依赖了我）
./gradlew :common-utils:dependencies --configuration compileClasspath -i | grep ":common-utils"
```

### 依赖分析

```bash
# 检测重复依赖
./gradlew buildEnvironment

# 查看可更新的依赖
./gradlew buildEnvironment -q

# 生成依赖报告
./gradlew htmlDependencyReport
```

## 多项目构建命令

### 树形结构

```bash
# 显示项目依赖树
./gradlew projects

# 输出示例
Root project 'my-platform'
+--- Project ':common-utils'
+--- Project ':web-backend'
|    +--- Project ':web-backend:api-client'
|    \--- Project ':web-backend:service-core'
\--- Project ':web-frontend'
```

### 顺序控制

```bash
# 从根项目执行所有子项目
./gradlew build

# 从叶子项目开始构建（自动处理依赖顺序）
./gradlew build --parallel

# 强制指定构建顺序
./gradlew :web-backend:build :api-gateway:build
```

## 性能相关

### 并行与缓存

```bash
# 启用并行构建
./gradlew build --parallel

# 启用构建缓存
./gradlew build --build-cache

# 两者同时启用
./gradlew build --parallel --build-cache
```

### 增量更新

```bash
# 强制重新执行任务
./gradlew build --rerun-tasks

# 刷新依赖
./gradlew build --refresh-dependencies

# 仅编译变化的模块
./gradlew build --configure-on-demand
```

### Daemon 控制

```bash
# 停止所有 Daemon
./gradlew --stop

# 不使用 Daemon 运行
./gradlew build --no-daemon

# 前台运行（调试用）
./gradlew build --foreground
```

## 高级用法

### 传递系统属性

```bash
./gradlew build -DmyProperty=value

# 在 build.gradle 中读取
def myValue = project.findProperty('myProperty') ?: 'default'
```

### 传递 Project 属性

```bash
./gradlew build -Pversion=2.0.0

# 在 build.gradle 中读取
version = project.findProperty('version') ?: '1.0.0'
```

### 环境变量

```bash
# 通过环境变量传递
export ORG_GRADLE_PROJECT_version=2.0.0
./gradlew build

# 读取
project.findProperty('version')
```

### 初始化脚本

```bash
# 使用自定义初始化脚本
./gradlew build --init-script init.gradle

# 全局初始化脚本
~/.gradle/init.gradle
```

## 常用命令速查

| 命令 | 说明 |
|---|---|
| `./gradlew build` | 完整构建 |
| `./gradlew clean build` | 清理后构建 |
| `./gradlew build -x test` | 跳过测试 |
| `./gradlew test --tests "*.Test"` | 只运行单元测试 |
| `./gradlew dependencies` | 查看依赖树 |
| `./gradlew projects` | 查看项目结构 |
| `./gradlew tasks` | 列出所有任务 |
| `./gradlew --scan` | 启用构建扫描 |
| `./gradlew --stop` | 停止 Daemon |
| `./gradlew build --parallel` | 并行构建 |
| `./gradlew build --build-cache` | 使用构建缓存 |

## 总结

Gradle 命令行不只是执行构建，它是你与 Gradle 交互的主要方式。掌握这些命令，你就能：

- **快速调试**：通过日志级别和任务跟踪定位问题
- **性能优化**：通过并行和缓存加速构建
- **依赖管理**：通过依赖树和报告解决冲突
- **高级控制**：通过属性和初始化脚本定制构建

下次构建卡住的时候，别急着重启 IDE——先试试命令行。
