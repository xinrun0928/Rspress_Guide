# 多环境配置：不同环境不同策略

「开发环境能跑，测试环境报错，生产环境挂了」——这是环境差异导致的经典问题。

多环境配置的核心挑战是：**同一份代码，在不同环境下需要不同的配置**。数据库连接、API 地址、日志级别、资源配额——这些都不能用同一套值。

## Maven 多环境

### Profile 机制

Maven Profile 允许你定义多套配置，通过激活不同的 Profile 来选择不同的配置。

```xml
<!-- pom.xml -->
<profiles>
    <!-- 开发环境 -->
    <profile>
        <id>dev</id>
        <activation>
            <activeByDefault>true</activeByDefault>
        </activation>
        <properties>
            <env>dev</env>
            <db.url>jdbc:mysql://localhost:3306/myapp_dev</db.url>
            <db.username>dev</db.username>
            <db.password>dev123</db.password>
            <api.base-url>http://localhost:8080</api.base-url>
        </properties>
    </profile>

    <!-- 测试环境 -->
    <profile>
        <id>test</id>
        <properties>
            <env>test</env>
            <db.url>jdbc:mysql://test-mysql:3306/myapp_test</db.url>
            <db.username>test</db.username>
            <db.password>${TEST_DB_PASSWORD}</db.password>
            <api.base-url>http://test-api.internal</api.base-url>
        </properties>
    </profile>

    <!-- 生产环境 -->
    <profile>
        <id>prod</id>
        <properties>
            <env>prod</env>
            <db.url>jdbc:mysql://prod-mysql:3306/myapp</db.url>
            <db.username>${PROD_DB_USERNAME}</db.username>
            <db.password>${PROD_DB_PASSWORD}</db.password>
            <api.base-url>https://api.example.com</api.base-url>
        </properties>
    </profile>
</profiles>
```

### 资源过滤

```yaml

    name: my-app

  level:

```

### 激活 Profile

**方式一：命令行激活**

```bash
# 指定环境构建
mvn clean package -Pprod

# 同时指定多个
mvn clean package -Pprod,cloud
```

**方式二：环境变量激活**

```xml
<profile>
    <id>prod</id>
    <activation>
        <property>
            <name>env</name>
            <value>prod</value>
        </property>
    </activation>
    <!-- ... -->
</profile>
```

```bash
mvn clean package -Denv=prod
```

**方式三：settings.xml 激活**

```xml
<!-- ~/.m2/settings.xml -->
<activeProfiles>
    <activeProfile>dev</activeProfile>
</activeProfiles>
```

### 多环境目录结构

```
src/
├── main/
│   ├── resources/
│   │   ├── application.yml          # 公共配置
│   │   ├── application-dev.yml      # 开发专用
│   │   ├── application-test.yml     # 测试专用
│   │   ├── application-prod.yml     # 生产专用
│   │   └── config/
│   │       ├── db-dev.properties
│   │       ├── db-test.properties
│   │       └── db-prod.properties
│   └── java/
└── test/
```

**主配置文件引用：**

```yaml
# application.yml
spring:
  profiles:
    active: @env@
  config:
    import: optional:file:./config/db-${env}.properties
```

## Gradle 多环境

### ProductFlavors

```kotlin
// build.gradle.kts
plugins {
    id("org.springframework.boot")
}

android {
    // ... Android 项目
}

// Spring Boot 多环境
springBoot {
    matches = listOf("*-boot-*.jar")
}

tasks.withType<Jar> {
    archiveFileName.set("${project.name}-${project.version}-${findProperty("env")}.jar")
}
```

### 定义 Flavor

```kotlin
// build.gradle.kts
plugins {
    id("org.springframework.boot") version "2.7.18"
}

val dev by creating {
    val dbUrl = "jdbc:mysql://localhost:3306/myapp_dev"
    val dbUser = "dev"
    val dbPassword = "dev123"
    val apiBaseUrl = "http://localhost:8080"
}

val test by creating {
    val dbUrl = "jdbc:mysql://test-mysql:3306/myapp_test"
    val dbUser = "test"
    val dbPassword = System.getenv("TEST_DB_PASSWORD") ?: ""
    val apiBaseUrl = "http://test-api.internal"
}

val prod by creating {
    val dbUrl = "jdbc:mysql://prod-mysql:3306/myapp"
    val dbUser = System.getenv("PROD_DB_USERNAME") ?: ""
    val dbPassword = System.getenv("PROD_DB_PASSWORD") ?: ""
    val apiBaseUrl = "https://api.example.com"
}
```

### SourceSet 分离

```kotlin
// build.gradle.kts
sourceSets {
    val dev by creating {
        resources.srcDirs("src/main/resources-dev")
    }
    val test by creating {
        resources.srcDirs("src/main/resources-test")
    }
    val prod by creating {
        resources.srcDirs("src/main/resources-prod")
    }
}
```


## Spring Profile 机制

### @Profile 注解

```java
// 只在 dev 环境启用的配置
@Configuration
@Profile("dev")
public class DevDatabaseConfig {
    @Bean
    public DataSource dataSource() {
        return new EmbeddedDatabaseBuilder()
            .setType(EmbeddedDatabaseType.H2)
            .build();
    }
}

// 只在 prod 环境启用的配置
@Configuration
@Profile("prod")
public class ProdDatabaseConfig {
    @Bean
    public DataSource dataSource() {
        HikariConfig config = new HikariConfig();
        config.setJdbcUrl(env.getProperty("db.url"));
        config.setUsername(env.getProperty("db.username"));
        config.setPassword(env.getProperty("db.password"));
        return new HikariDataSource(config);
    }
}
```

### application-{profile}.yml

```yaml
# application-dev.yml
spring:
  h2:
    console:
      enabled: true  # 开发环境启用 H2 控制台

logging:
  level:
    com.myapp: DEBUG
    org.springframework.web: DEBUG

management:
  endpoints:
    web:
      exposure:
        include: "*"  # 开发环境暴露所有端点
```

```yaml
# application-prod.yml
logging:
  level:
    com.myapp: INFO
    org.springframework.web: WARN

management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics  # 生产环境只暴露必要端点
  endpoint:
    health:
      show-details: when_authorized
```

### 条件配置 @ConditionalOnProperty

```java
@Configuration
@ConditionalOnProperty(name = "feature.new-payment", havingValue = "true")
public class NewPaymentConfig {
    // 只有 feature.new-payment=true 时才加载
}
```

## Kubernetes 环境变量注入

### ConfigMap 挂载

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: myapp-config
data:
  DB_URL: "jdbc:mysql://mysql:3306/myapp"
  API_BASE_URL: "http://api-service:8080"
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  template:
    spec:
      containers:
      - name: myapp
        image: myapp:latest
        envFrom:
        - configMapRef:
            name: myapp-config
        env:
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: myapp-secrets
              key: db-password
```

### Helm values 文件分离

```yaml
# values-dev.yaml
replicaCount: 1
image:
  repository: myapp-dev
  tag: latest

resources:
  limits:
    cpu: 500m
    memory: 512Mi

ingress:
  enabled: false

env:
  PROFILE: dev
```

```yaml
# values-prod.yaml
replicaCount: 3
image:
  repository: myapp-prod
  tag: v1.2.3

resources:
  limits:
    cpu: 2000m
    memory: 2Gi

ingress:
  enabled: true
  className: nginx
  host: myapp.example.com

env:
  PROFILE: prod
```

```bash
# 使用不同 values 文件部署
helm upgrade --install myapp ./chart \
  -f values-prod.yaml \
  --namespace production
```

## 敏感信息管理

### 不在配置文件中明文存储

```yaml
# 错误：敏感信息写在文件里
api:
  password: "123456"

# 正确：使用环境变量或 Secret
api:
  password: ${API_PASSWORD}
```

### Maven 安全配置

```xml
<!-- settings.xml 中的服务器配置 -->
<servers>
    <server>
        <id>prod-maven-repo</id>
        <username>${env.MAVEN_REPO_USERNAME}</username>
        <password>${env.MAVEN_REPO_PASSWORD}</password>
    </server>
</servers>
```

### Gradle 安全配置

```kotlin
// build.gradle.kts
val prodDbPassword: String = providers.environmentVariable("PROD_DB_PASSWORD")
    .forUseAtConfigurationTime()
    .get()

dependencies {
    implementation("com.example:secure-lib:${prodDbPassword}")
}
```

## 环境一致性保证

### Docker 镜像分层

```dockerfile
# 多阶段构建
FROM maven:3.9-eclipse-temurin-17 AS builder
COPY pom.xml .
RUN mvn dependency:go-offline
COPY src ./src
RUN mvn package -Pprod -DskipTests

FROM eclipse-temurin:17-jre
COPY --from=builder /target/myapp.jar /app/myapp.jar
ENTRYPOINT ["java", "-jar", "/app/myapp.jar"]
```

### CI/CD 中的环境注入

```yaml
# GitHub Actions
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK
        uses: actions/setup-java@v4
        with:
          java-version: '17'

      - name: Build with Maven
        run: mvn clean package -P${{ github.ref == 'refs/heads/main' && 'prod' || 'dev' }}
        env:
          PROD_DB_PASSWORD: ${{ secrets.PROD_DB_PASSWORD }}

      - name: Build with Gradle
        run: ./gradlew bootJar -Penv=${{ env.ENV }}
        env:
          PROD_DB_PASSWORD: ${{ secrets.PROD_DB_PASSWORD }}
```

## 总结

多环境配置的核心原则：

| 原则 | 说明 |
|---|---|
| **环境差异外置** | 敏感信息和环境特定配置放在代码外 |
| **Profile 隔离** | 使用 Maven Profile 或 Gradle Flavor |
| **敏感信息加密** | 使用 Secret、Vault 或环境变量 |
| **CI/CD 注入** | 构建时根据目标环境注入配置 |
| **一致性保证** | 本地、CI、生产使用同一套配置源 |

**推荐目录结构：**

```
project/
├── src/main/resources/
│   ├── application.yml           # 公共配置
│   ├── application-dev.yml       # 开发环境
│   ├── application-test.yml      # 测试环境
│   └── application-prod.yml      # 生产环境
├── chart/
│   ├── values.yaml               # 默认值
│   ├── values-dev.yaml
│   ├── values-test.yaml
│   └── values-prod.yaml
└── src/main/config/
    └── config-prod.properties
```
