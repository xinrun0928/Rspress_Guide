# 构建工具最佳实践

你以为「依赖管好、版本统一」就是最佳实践了？错。真正的最佳实践，是让构建成为团队的速度倍增器，而不是噩梦的开始。

## 为什么需要最佳实践

大多数项目的构建问题是逐渐累积的：

- 依赖版本混乱：`Spring Boot 2.4` 和 `Spring Boot 3.0` 共存
- 依赖冲突频发：排了一个 another 又来一个
- 构建速度感人：每次 CI 跑 20 分钟起步
- 发布流程繁琐：手动打包、手动上传、手动验证

**最佳实践的目标**：让构建可预测、可重复、可维护。

## 依赖管理最佳实践

### 统一版本管理

```groovy title="build.gradle (Maven pom.xml 类似)"
subprojects {
    ext {
        springBootVersion = '3.2.0'
        lombokVersion = '1.18.30'
        guavaVersion = '32.1.3-jre'
    }
}
```

:::tip
使用 `dependencyManagement` 或 BOM 是更优雅的方案——只需在一处定义版本。
:::

### 依赖冲突排查

```bash
# Maven
mvn dependency:tree -Dverbose
mvn dependency:tree -Dincludes=com.example

# Gradle
./gradlew dependencies --configuration runtimeClasspath
./gradlew :app:dependencies --configuration compileClasspath | grep <artifact>
```

### 排除传递依赖

```groovy title="Gradle"
implementation('com.example:bad-lib:1.0') {
    exclude group: 'org.apache.commons', module: 'commons-collections'
}

// 或全局配置
configurations.all {
    resolutionStrategy {
        force 'com.google.guava:guava:32.1.3-jre'
    }
}
```

## 构建速度优化

### 增量构建

```groovy title="Gradle"
tasks.register('incrementalCompile', JavaCompile) {
    source = fileTree('src/main/java')
    destinationDirectory = file('build/classes')
    
    options.incremental = true
}
```

### 并行构建

```groovy title="gradle.properties"
org.gradle.parallel=true
org.gradle.caching=true
org.gradle.daemon=true
```

### 跳过不必要的任务

```bash
./gradlew build -x test -x checkstyleMain -x checkstyleTest
```

## 多环境配置

### Profile 切换

Maven Profile：

```xml title="pom.xml"
<profiles>
    <profile>
        <id>dev</id>
        <properties>
            <env>dev</env>
            <db.url>jdbc:mysql://localhost:3306/dev</db.url>
        </properties>
    </profile>
    <profile>
        <id>prod</id>
        <properties>
            <env>prod</env>
            <db.url>jdbc:mysql://prod-db:3306/app</db.url>
        </properties>
    </profile>
</profiles>
```

Gradle 方式：

```groovy title="build.gradle"
def profile = project.findProperty('profile') ?: 'dev'

sourceSets {
    "${profile}" {
        java {
            srcDirs = ["src/${profile}/java"]
        }
        resources {
            srcDirs = ["src/${profile}/resources"]
        }
    }
}
```

## 安全最佳实践

### 依赖安全扫描

```bash
# OWASP Dependency-Check
./gradlew dependencyCheckAnalyze

# Maven
mvn org.owasp:dependency-check-maven:check
```

### 签名与验证

```groovy title="Gradle"
plugins {
    id 'signing'
}

signing {
    sign publishing.publications.mavenJava
}
```

## 总结

构建最佳实践的核心原则：

| 原则 | 说明 |
|---|---|
| 统一管理 | 版本集中，BOM 优先 |
| 可重复 | 相同输入，相同输出 |
| 可追踪 | 依赖树清晰，冲突可查 |
| 速度快 | 增量优先，缓存复用 |
| 安全 | 扫描、签名、验证 |

记住：好的构建实践不是为了炫技，而是让团队把时间花在真正重要的事情上。
