# 构建安全：依赖漏洞扫描

「你的代码可能是安全的，但你的依赖呢？」

据统计，现代应用 80% 以上的代码来自第三方依赖。而这些依赖中，或多或少都存在安全漏洞。心血漏洞（Heartbleed）、Log4Shell、Spring4Shell——这些耳熟能详的安全事件，无一不是通过依赖漏洞入侵的。

**依赖安全不是可选项，而是必修课。**

## OWASP 依赖检查

### OWASP Top 10 简介

OWASP（Open Web Application Security Project）每年发布的 Top 10 是应用安全的权威指南，其中「A06:2021 - Vulnerable and Outdated Components」直接针对依赖安全问题。

### OWASP Dependency-Check

OWASP Dependency-Check 是开源的依赖漏洞扫描工具，支持 Maven、Gradle、CLI 等多种使用方式。

#### Maven 集成

```xml
<plugin>
    <groupId>org.owasp</groupId>
    <artifactId>dependency-check-maven</artifactId>
    <version>8.4.0</version>
    <configuration>
        <skip>false</skip>
        <assemblyAnalyzerEnabled>false</assemblyAnalyzerEnabled>
        <centralRepoId>NVD</centralRepoId>
    </configuration>
    <executions>
        <execution>
            <goals>
                <goal>check</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

```bash
# 运行检查
mvn dependency-check:check

# 生成报告
mvn dependency-check:check dependency-check-report:report

# 查看报告
open target/dependency-check-report.html
```

#### Gradle 集成

```kotlin
// build.gradle.kts
plugins {
    id("org.owasp.dependencycheck") version "8.4.0"
}

dependencyCheck {
    failBuildOnCVSS = 7.0f  // CVSS 分数 >= 7.0 时构建失败
    suppressionFile = "config/dependency-check-suppressions.xml"
}
```

```bash
./gradlew dependencyCheck

# 查看报告
open build/reports/dependency-check-report.html
```

**CVSS 评分说明：**

| 分数 | 等级 | 行动 |
|---|---|---|
| 0.0 - 3.9 | 低 | 记录，建议修复 |
| 4.0 - 6.9 | 中 | 计划修复 |
| 7.0 - 8.9 | 高 | 尽快修复 |
| 9.0 - 10.0 | 严重 | 立即修复 |

## 已知漏洞库

### NVD（National Vulnerability Database）

美国国家漏洞数据库，是最权威的 CVE 来源：

- **数据源**：https://nvd.nist.gov/
- **更新频率**：每小时
- **数据格式**：CPE、CVE、CVSS

### Maven 中央仓库元数据

```bash
# 查询特定依赖的已知漏洞
curl -s "https://search.maven.org/solrsearch/select?q=g:org.apache.logging.log4j+a:log4j-core&rows=5&wt=json" | jq
```

### GitHub Advisory Database

GitHub 托管的开源软件漏洞库：

```graphql
# 查询特定依赖的漏洞
query {

        ghsaId
        summary
        severity
        publishedAt
        updatedAt
        description
      }
      vulnerableVersionRange
      firstPatchedVersion {
        identifier
      }
    }
  }
}
```

## 许可证合规

### 许可证类型分类

| 类别 | 许可证 | 使用限制 |
|---|---|---|
| ** permissive** | MIT, BSD, Apache 2.0 | 几乎无限制 |
| **weak copyleft** | LGPL, MPL | 静态链接需开源 |
| **strong copyleft** | GPL, AGPL | 衍生作品必须开源 |
| **商业** | proprietary | 需购买许可 |

### License Check 插件

```xml
<plugin>
    <groupId>org.codehaus.mojo</groupId>
    <artifactId>license-maven-plugin</artifactId>
    <version>2.3.0</version>
    <configuration>
        <licenses>
            <license>
                <name>Apache License 2.0</name>
                <url>https://www.apache.org/licenses/LICENSE-2.0.txt</url>
                <distribution>repo</distribution>
            </license>
        </licenses>
        <exceptions>
            <exception>
                <name>Classpath Exception</name>
                <url>https://www.gnu.org/software/classpath/license.html</url>
            </exception>
        </exceptions>
        <excludeTransitiveDependencies>true</excludeTransitiveDependencies>
    </configuration>
</plugin>
```

```bash
# 检查许可证
mvn license:check

# 更新许可证头
mvn license:update-file-header

# 下载许可证文本
mvn license:download-licenses
```

### 禁止特定许可证

```xml
<plugin>
    <groupId>org.apache.rat</groupId>
    <artifactId>apache-rat-plugin</artifactId>
    <version>0.15</version>
    <configuration>
        <excludes>
            <exclude>**/licenses/**</exclude>
            <exclude>**/*.txt</exclude>
        </excludes>
    </configuration>
</plugin>
```

## CI/CD 集成

### GitHub Actions

```yaml
name: Security Scan

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Dependency Check
        run: |
          mvn org.owasp:dependency-check-maven:check
        continue-on-error: true

      - name: Upload Report
        uses: actions/upload-artifact@v4
        with:
          name: dependency-check-report
          path: target/dependency-check-report.html

      - name: Check for critical vulnerabilities
        run: |
          if grep -q "Critical" target/dependency-check-report.html; then
            echo "Critical vulnerabilities found!"
            exit 1
          fi
```

### GitLab CI

```yaml
dependency-check:
  stage: security
  image: maven:3.9-eclipse-temurin-17
  script:
    - mvn org.owasp:dependency-check-maven:check
    - mvn org.owasp:dependency-check-maven:report
  artifacts:
    reports:
      dependency_scanning: dependency-check-report.html
    when: always
```

### Jenkins Pipeline

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.9-eclipse-temurin-17'
        }
    }
    stages {
        stage('Dependency Check') {
            steps {
                sh '''
                    mvn org.owasp:dependency-check-maven:check \
                        dependency-check-report:report \
                        -DfailBuildOnCVSS=7
                '''
            }
            post {
                always {
                    publishHTML([
                        allowMissing: false,
                        alwaysLinkToLastBuild: true,
                        keepAll: true,
                        reportDir: 'target',
                        reportFiles: 'dependency-check-report.html',
                        reportName: 'Dependency Check Report'
                    ])
                }
            }
        }
    }
}
```

## 依赖审查流程

### 1. 引入前审查

引入新依赖时，回答以下问题：

- [ ] 这个库来自可信来源吗？（官方仓库/知名组织）
- [ ] 许可证是否允许在我们的产品中使用？
- [ ] 最近一次更新是什么时候？
- [ ] 是否有已知漏洞？（查询 NVD）
- [ ] 是否有活跃的社区维护？

### 2. 定期扫描

```bash
#!/bin/bash
# schedule-scan.sh

# 每周一早上执行扫描
0 8 * * 1 /opt/scripts/dependency-scan.sh

# 或使用 Renovate Bot 自动检测更新
```

### 3. 自动化告警

```yaml
# GitHub Actions 告警到 Slack
- name: Notify on vulnerabilities
  if: steps.check.outputs.vuln_count > 0
  uses: slackapi/slack-github-action@v1
  with:
    payload: |
      {
        "text": "🚨 依赖漏洞告警",
        "blocks": [{
          "type": "section",
          "text": {
            "type": "mrkdwn",
            "text": "*项目*: ${{ github.repository }}\n*漏洞数*: ${{ steps.check.outputs.vuln_count }}\n*链接*: ${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}"
          }
        }]
      }
```

## 依赖升级策略

### Semantic Versioning 遵循

| 变更类型 | 版本号变化 | 是否需要代码修改 |
|---|---|---|
| Bug 修复 | 1.0.**0** → 1.0.**1** | 否 |
| 新功能（向后兼容）| 1.**0**.1 → 1.**1**.0 | 可能 |
| 破坏性变更 | **1**.0.1 → **2**.0.0 | 是 |

### Dependabot 自动更新

```yaml
# .github/dependabot.yml
version: 2
updates:
  - package-ecosystem: "maven"
    directory: "/"
    schedule:
      interval: "weekly"
    open-pull-requests-limit: 10
    reviewers:
      - team:security
    labels:
      - "dependencies"
      - "security"
    ignore:
      - dependency-name: "org.springframework.boot"
        update-types: ["version-update:semver-major"]
```

### Renovate 高级配置

```json
{
  "packageRules": [
    {
      "matchPackagePatterns": ["*"],
      "groupName": "all dependencies"
    },
    {
      "matchPackagePatterns": ["log4j", "spring-core"],
      "labels": ["security"],
      "automerge": false
    },
    {
      "matchPackagePatterns": ["junit", "mockito"],
      "labels": ["testing"],
      "automerge": true
    }
  ]
}
```

## 总结

依赖安全是应用安全的第一道防线：

- **扫描是基础**：OWASP Dependency-Check 是标配
- **CI/CD 集成**：每次构建都应扫描
- **许可证合规**：了解你引入的每一条依赖的许可证
- **自动更新**：使用 Dependabot/Renovate 保持依赖新鲜
- **漏洞响应**：建立漏洞响应流程，快速修复

**建议的最低安全标准：**

1. 每次 PR 必须通过依赖扫描
2. 禁止引入 GPL/AGPL 许可证的依赖
3. 严重漏洞（CVSS >= 9.0）必须 24 小时内修复
4. 高危漏洞（CVSS >= 7.0）必须一周内修复
5. 每季度进行一次全量依赖审计
