# 构建安全：依赖漏洞扫描

你有没有想过——你的项目依赖了几百个第三方库，其中有多少是安全的？

2021 年的 Log4Shell 漏洞（CVE-2021-44228）让整个 Java 生态为之震动。这个漏洞藏在 Log4j 这个几乎所有 Java 项目都在用的日志库里，却存在了十几年才被发现。

OWASP Dependency-Check 就是为了解决这个问题——在构建阶段自动扫描依赖中的已知漏洞。

## 为什么需要依赖漏洞扫描

### 依赖安全现状

```
一个典型的 Spring Boot 项目依赖数量：
├── spring-boot-starter-web
│   ├── spring-web
│   ├── spring-webmvc
│   └── jackson-databind
│       └── jackson-core
│           └── jackson-annotations
│               └── ...
├── spring-boot-starter-data-jpa
│   └── hibernate-core
│       └── ...
└── ...

总依赖数量：约 150-200 个
直接依赖：约 20-30 个
间接依赖：约 130-170 个
```

你知道自己间接依赖了哪些库吗？你知道这些库有没有安全漏洞吗？

### 漏洞的严重性

| 漏洞等级 | CVSS 评分 | 说明 | 示例 |
|---|---|---|---|
| **Critical** | 9.0-10.0 | 远程代码执行 | Log4Shell |
| **High** | 7.0-8.9 | 可导致数据泄露 | Spring4Shell |
| **Medium** | 4.0-6.9 | 有限影响 | 信息泄露 |
| **Low** | 0.1-3.9 | 最小影响 | 拒绝服务 |

## OWASP Dependency-Check

### 安装与使用

```bash
# Maven 插件
# pom.xml
<plugin>
    <groupId>org.owasp</groupId>
    <artifactId>dependency-check-maven</artifactId>
    <version>8.4.0</version>
    <configuration>
        <skip>false</skip>
        <failBuildOnCVSS>7</failBuildOnCVSS>  <!-- 超过 7 分则构建失败 -->
        <outputDirectory>target/dependency-check-report</outputDirectory>
    </configuration>
    <executions>
        <execution>
            <goals>
                <goal>check</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

```bash
# Gradle 插件
# build.gradle.kts
plugins {
    id("org.owasp.dependencycheck") version "8.4.0"
}

dependencyCheck {
    failBuildOnCVSS = 7f
    outputDirectory = project.layout.buildDirectory.dir("dependency-check-report").get().asFile
}

tasks.withType<DependencyCheckAnalyze> {
    enabled = true
}
```

```bash
# 命令行执行
mvn dependency-check:check
./gradlew dependencyCheckAnalyze

# 生成报告
mvn dependency-check:check dependency-check-report
```

- **引用链接**：官方修复建议

## GitLab CI / GitHub Actions 集成

### GitLab CI

```yaml title=".gitlab-ci.yml"
dependency-check:
  stage: test
  image: maven:3.9-eclipse-temurin-17
  script:
    - mvn org.owasp:dependency-check-maven:8.4.0:check
    - mvn org.owasp:dependency-check-maven:8.4.0:dependency-check-report
  artifacts:
    reports:
      dependency_scanning: target/dependency-check-report.html
    when: always
    paths:
      - target/dependency-check-report.html
  allow_failure: true  # 发现漏洞时允许继续（但不合并）
```

### GitHub Actions

```yaml title=".github/workflows/security-scan.yml"
name: Security Scan

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  dependency-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
          cache: 'maven'

      - name: Run OWASP Dependency Check
        run: |
          mvn org.owasp:dependency-check-maven:8.4.0:check \
            org.owasp:dependency-check-maven:8.4.0:dependency-check-report

      - name: Upload Dependency Check Report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: dependency-check-report
          path: target/dependency-check-report.html

      - name: Add dependency-review github actions
        uses: actions/dependency-review-action@v4
        with:
          fail-on-severity: high
          deny-packages: |
            # 禁止使用这些包（示例）
```

## 常见漏洞库与修复

### Log4j

```xml

    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-core</artifactId>
    <version>2.14.1</version>

    <version>2.23.1</version>
```

## 最佳实践

### 1. 强制门禁

```xml
<!-- Maven：发现高危漏洞则构建失败 -->
<configuration>
    <failBuildOnCVSS>7</failBuildOnCVSS>
    <cvssEvaluator>CVSSv3</cvssEvaluator>
</configuration>
```

```kotlin
// Gradle
dependencyCheck {
    failBuildOnCVSS = 7f
    failBuildOnAnyKnownVulnerability = false
}
```

### 2. 增量更新 CVE 数据库

```xml
<configuration>
    <!-- 每天只更新一次 CVE 数据库 -->
    <autoupdateEnabled>true</autoupdateEnabled>
    <autoUpdateSkipNewVersion>true</autoUpdateSkipNewVersion>
</configuration>
```

### 3. 排除误报

```xml
<configuration>
    <suppressionFiles>
        <suppressionFile>dependency-check-suppression.xml</suppressionFile>
    </suppressionFiles>
</configuration>
```

```xml title="dependency-check-suppression.xml"
<?xml version="1.0" encoding="UTF-8"?>
<suppressions xmlns="https://jeremylong.github.io/DependencyCheck/dependency-suppression.anchor.xsd">
    <suppress>
        <notes>
            已知误报：此漏洞在此版本中已修复，但 NVD 尚未更新
        </notes>
        <cve>CVE-2023-12345</cve>
        <packageUrl>pkg:maven/com.example/library@1.2.3</packageUrl>
        <vulnerabilityName>Known False Positive</vulnerabilityName>
    </suppress>
</suppressions>
```

### 4. 与依赖管理结合

```xml title="pom.xml"
<!-- 使用 BOM 统一管理版本 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>3.2.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

## 总结

依赖漏洞扫描是构建安全的重要环节：

1. **OWASP Dependency-Check**：自动化扫描 Maven/Gradle 项目的依赖漏洞
2. **CI/CD 集成**：在构建流水线中发现漏洞，阻止带漏洞的代码上线
3. **强制门禁**：配置 CVSS 阈值，高危漏洞阻断构建
4. **持续监控**：每次构建都更新 CVE 数据库，及时发现新漏洞
5. **快速修复**：发现漏洞后，尽快升级到安全版本

将依赖漏洞扫描纳入构建流程，可以在漏洞爆发前提前发现和修复，保护系统的安全。
