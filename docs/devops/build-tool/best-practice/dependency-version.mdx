# 依赖版本管理：构建的「地基」

「为什么我的项目能跑，但同事电脑上就报错了？」

依赖版本不一致是 Java 项目最常见的问题之一。你用 Spring Boot 2.7，他用 2.6，API 差异导致行为不同；你引入 fastjson 2.14，他用的是 2.12，漏洞扫描报告天差地别。

**版本管理的核心目标是：让团队所有人、让本地和 CI 环境，构建出完全相同的依赖树。**

## Maven 版本管理

### 依赖传递与冲突

Maven 的依赖传递是双刃剑——它简化了依赖声明，但也带来了版本冲突的风险。

```xml
<!-- 项目依赖 Spring Boot -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <version>2.7.18</version>
</dependency>

<!-- Spring Boot 2.7.18 依赖 Jackson 2.13.x -->
<!-- 如果你同时引入其他依赖声明了 Jackson 2.11 -->
<!-- Maven 会根据「最短路径优先」原则选择 -->
```

**最短路径优先原则：**

```
项目 → A(v1.0) → X(1.0)
项目 → B(v1.0) → X(1.5)
    ↓
选择 X(1.5)（路径更短）
```

### 版本冲突解决

**方式一：显式声明（推荐）**

```xml
<!-- 显式声明版本，覆盖传递依赖 -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.15.2</version>
</dependency>
```

**方式二：依赖管理（dependencyManagement）**

```xml
<!-- 在父 POM 中统一管理版本 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.fasterxml.jackson</groupId>
            <artifactId>jackson-bom</artifactId>
            <version>2.15.2</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<!-- 子项目继承版本，无需指定 version -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <!-- 不需要指定 version，自动使用 BOM 中的版本 -->
</dependency>
```

## BOM 的艺术

### Spring Boot 的 BOM 策略

Spring Boot 采用了「starter + BOM」的策略：

```xml
<!-- spring-boot-dependencies 定义了所有依赖的版本 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>2.7.18</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

**为什么这样做？**

- 你只需要记住一个版本：Spring Boot 版本
- Spring Boot 团队测试过所有依赖的兼容性组合
- 升级 Spring Boot 版本时，所有依赖自动协调升级

### 自定义企业 BOM

```xml
<!-- corporate-bom/pom.xml -->
<project>
    <groupId>com.example</groupId>
    <artifactId>corporate-bom</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>

    <dependencyManagement>
        <dependencies>
            <!-- Spring Boot -->
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>${spring-boot.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>

            <!-- 我们的规范版本 -->
            <dependency>
                <groupId>com.example</groupId>
                <artifactId>core-lib</artifactId>
                <version>2.3.0</version>
            </dependency>

            <!-- 第三方库统一版本 -->
            <dependency>
                <groupId>org.apache.commons</groupId>
                <artifactId>commons-lang3</artifactId>
                <version>3.12.0</version>
            </dependency>
        </dependencies>
    </dependencyManagement>
</project>
```

## Gradle 版本管理

### Platform 和 BOM

Gradle 7+ 提供了原生 BOM 支持：

```kotlin
// settings.gradle.kts
dependencyResolutionManagement {
    repositories {
        mavenCentral()
    }
}

// 使用 Maven BOM
dependencies {
    implementation(platform("org.springframework.boot:spring-boot-dependencies:2.7.18"))
    // 不需要指定 version
    implementation("com.fasterxml.jackson.core:jackson-databind")
}
```

### 版本目录（Gradle 版本插件）

```kotlin
// settings.gradle.kts
plugins {
    id("io.spring.dependency-management") version "1.1.4"
}

// build.gradle.kts
dependencyManagement {
    imports {
        mavenBom("org.springframework.boot:spring-boot-dependencies:2.7.18")
    }
    dependencies {
        dependency("com.example:core-lib:2.3.0")
    }
}

dependencies {
    implementation("com.example:core-lib")  // 自动使用 2.3.0
}
```

### 版本约束（Version Catalog）

Gradle 7+ 推荐使用版本目录：

```toml
# gradle/libs.versions.toml
[versions]
spring-boot = "2.7.18"
jackson = "2.15.2"
kotlin = "1.9.10"

[libraries]
spring-boot-starter = { module = "org.springframework.boot:spring-boot-starter" }
spring-boot-web = { module = "org.springframework.boot:spring-boot-starter-web" }
jackson-databind = { module = "com.fasterxml.jackson.core:jackson-databind", version.ref = "jackson" }

[plugins]
spring-boot = { id = "org.springframework.boot", version.ref = "spring-boot" }
kotlin-jvm = { id = "org.jetbrains.kotlin.jvm", version.ref = "kotlin" }
```

```kotlin
// build.gradle.kts
plugins {
    alias(libs.plugins.spring.boot)
    alias(libs.plugins.kotlin.jvm)
}

dependencies {
    implementation(libs.spring.boot.web)
    implementation(libs.jackson.databind)
}
```

## SNAPSHOT 陷阱

### SNAPSHOT 的危害

SNAPSHOT（快照版本）是不稳定依赖的代表：

```xml
<!-- 使用 SNAPSHOT = 不稳定 -->
<dependency>
    <groupId>com.example</groupId>
    <artifactId>common-lib</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>
```

**SNAPSHOT 的问题：**

1. **不可重现**：每次构建可能拉取不同的代码
2. **缓存问题**：IDE/本地仓库可能缓存旧版本
3. **CI 不确定性**：不同时间构建结果不同
4. **难以回溯**：无法定位问题出在哪个「快照」

### 生产环境禁止 SNAPSHOT

| 环境 | 允许 SNAPSHOT | 推荐做法 |
|---|---|---|
| 开发环境 | ✅ 允许 | 开发分支使用 SNAPSHOT |
| 测试环境 | ❌ 禁止 | 使用发布版本 |
| 预发布 | ❌ 禁止 | 使用发布版本 |
| 生产环境 | ❌ 禁止 | 使用发布版本 + 校验签名 |

### 版本号规范

| 版本类型 | 格式 | 示例 | 场景 |
|---|---|---|---|
| 主版本 | X.0.0 | 2.0.0 | 不兼容的 API 变更 |
| 次版本 | X.Y.0 | 2.1.0 | 向下兼容的新功能 |
| 补丁版本 | X.Y.Z | 2.1.1 | 向下兼容的 bug 修复 |
| 里程碑 | X.Y.Z-M1 | 2.1.0-M1 | 发布前测试版本 |
| 快照 | X.Y.Z-SNAPSHOT | 2.1.1-SNAPSHOT | 开发中版本 |

## 版本锁定

### Maven Enforcer 插件

防止意外的依赖变更：

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-enforcer-plugin</artifactId>
    <version>3.4.1</version>
    <executions>
        <execution>
            <id>enforce-dependency-convergence</id>
            <goals>
                <goal>enforce</goal>
            </goals>
            <configuration>
                <rules>
                    <dependencyConvergence/>
                    <requireReleaseDeps>
                        <message>SNAPSHOT dependencies are not allowed!</message>
                        <excludes>
                            <exclude>*:SNAPSHOT</exclude>
                        </excludes>
                    </requireReleaseDeps>
                </rules>
            </configuration>
        </execution>
    </executions>
</plugin>
```

### Gradle 依赖锁

```kotlin
// build.gradle.kts
dependencyLocking {
    lockAllConfigurations()
}
```

```bash
# 生成锁文件
./gradlew dependencies --write-locks

# 查看锁文件
cat build/generated-resources/main/dependencies.lock
```

## 实战：搭建企业依赖治理体系

### 1. 创建公司级 BOM

```xml
<!-- corporate-parent/pom.xml -->
<project>
    <artifactId>corporate-parent</artifactId>
    <packaging>pom</packaging>

    <properties>
        <java.version>17</java.version>
        <spring-boot.version>2.7.18</spring-boot.version>
        <mybatis-plus.version>3.5.3.1</mybatis-plus.version>
        <hutool.version>5.8.21</hutool.version>
    </properties>

    <dependencyManagement>
        <dependencies>
            <!-- Spring Boot -->
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>${spring-boot.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>

            <!-- 统一内部库 -->
            <dependency>
                <groupId>com.example</groupId>
                <artifactId>corporate-dependencies</artifactId>
                <version>${project.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>

            <!-- 其他统一版本 -->
            <dependency>
                <groupId>cn.hutool</groupId>
                <artifactId>hutool-all</artifactId>
                <version>${hutool.version}</version>
            </dependency>
        </dependencies>
    </dependencyManagement>
</project>
```

### 2. 子项目继承

```xml
<!-- child-project/pom.xml -->
<project>
    <parent>
        <groupId>com.example</groupId>
        <artifactId>corporate-parent</artifactId>
        <version>1.0.0</version>
    </parent>

    <artifactId>child-project</artifactId>

    <dependencies>
        <!-- 不需要指定 version，继承父 POM -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <dependency>
            <groupId>cn.hutool</groupId>
            <artifactId>hutool-all</artifactId>
        </dependency>
    </dependencies>
</project>
```

## 总结

依赖版本管理是构建系统的地基：

- **统一 BOM**：让团队使用一致的版本集合
- **禁止 SNAPSHOT**：生产环境必须使用稳定版本
- **版本锁定**：确保每次构建结果可重现
- **强制检查**：用 Enforcer/Lock 插件防止意外
- **定期升级**：及时更新到安全版本

**最佳实践：**

```xml
<!-- 依赖管理检查 -->
<enforcer>
    <requireReleaseDeps/>
    <banDuplicatePomDependencyVersions/>
</enforcer>
```

版本管理做好了，构建问题少一半。
