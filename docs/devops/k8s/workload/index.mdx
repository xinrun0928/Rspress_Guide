# Kubernetes 工作负载

工作负载是 Kubernetes 中用于运行应用的对象。本模块涵盖 Deployment、StatefulSet、DaemonSet、Job、CronJob 等核心资源类型的使用方法和最佳实践。

## 工作负载的类型

Kubernetes 中的工作负载分为两大类：**无状态工作负载** 和 **有状态工作负载**。

**无状态工作负载**不关心实例的身份，任何实例都可以替代另一个实例。Deployment 是最典型的无状态工作负载，适合运行 Web 服务、API 服务等可以水平扩展的应用。

**有状态工作负载**需要维护实例的身份和状态。每个实例可能有不同的网络标识、不同的存储卷。StatefulSet 专门用于管理这类应用，如数据库、消息队列等。

:::tip 使用场景
- **Deployment**：无状态 Web 服务、API 服务
- **StatefulSet**：数据库、消息队列、需要稳定网络标识的应用
- **DaemonSet**：日志收集器、监控代理、网络插件
- **Job**：一次性批处理任务
- **CronJob**：定时执行的批处理任务
:::
