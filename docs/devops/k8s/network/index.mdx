# Kubernetes 网络

网络是 Kubernetes 中最复杂的子系统之一。本模块深入讲解 Kubernetes 的网络模型、DNS 服务发现、各种 Service 类型，以及 Ingress 和网络策略的配置。

## 为什么 Kubernetes 网络如此重要？

在传统单体应用中，服务之间的调用通常是 localhost 或者固定 IP。但在 Kubernetes 环境中：

- Pod 的 IP 是临时的，随时可能变化
- Pod 可能被调度到集群中的任何节点
- 应用需要动态发现其他服务的位置
- 外部流量需要正确路由到目标 Pod

Kubernetes 定义了一套清晰的网络模型，让不同的 CNI 插件可以提供一致的网络能力。理解这套模型，是编写生产级网络配置的前提。
