# NetworkPolicy 网络策略

在 Kubernetes 的默认配置下，集群内的 Pod 可以自由通信。这意味着如果某个 Pod 被攻破，攻击者可以横向移动到其他 Pod。

NetworkPolicy 提供了 Pod 级别的网络隔离能力，让你可以定义「谁可以访问谁」的精确规则。

## 核心概念

NetworkPolicy 是一种网络资源的访问控制列表（ACL）。它由以下几部分组成：

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: api-network-policy
  namespace: production
spec:
  podSelector:
    matchLabels:
      app: api-server
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: nginx
    ports:
    - protocol: TCP
      port: 8080
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 5432
```

### policyTypes

| 类型 | 说明 | 必需 |
|---|---|---|
| `Ingress` | 入站流量控制 | 当有 ingress 规则时自动添加 |
| `Egress` | 出站流量控制 | 当有 egress 规则时自动添加 |

:::warning CNI 支持要求
NetworkPolicy 需要 CNI 插件支持。Flannel 默认不支持，Calico、Cilium、Weave 等支持。选择 CNI 插件时需要确认这一点。
:::

## 流量选择器

### podSelector

选择具有特定标签的 Pod：

```yaml
podSelector:
  matchLabels:
    app: api-server
```

### namespaceSelector

选择特定命名空间的 Pod：

```yaml
ingress:
- from:
  - namespaceSelector:
      matchLabels:
        tenant: finance
    podSelector:
      matchLabels:
        app: dashboard
```

### ipBlock

基于 CIDR 范围选择：

```yaml
egress:
- to:
  - ipBlock:
      cidr: 10.0.0.0/8
      except:
      - 10.0.0.0/16
  ports:
  - protocol: TCP
    port: 443
```

## 访问控制规则

### Ingress 规则

控制谁可以访问被选中的 Pod：

```yaml
# 只允许带有 app=nginx 标签的 Pod 访问 80 端口
ingress:
- from:
  - podSelector:
      matchLabels:
        app: nginx
  ports:
  - protocol: TCP
    port: 80
```

### Egress 规则

控制被选中的 Pod 可以访问谁：

```yaml
# 只允许访问 DNS 和特定的数据库
egress:
- to:
  - podSelector:
      matchLabels:
        app: database
  ports:
  - protocol: TCP
    port: 5432
- to:
  - namespaceSelector: {}  # 匹配所有命名空间
    podSelector:
      matchLabels:
        k8s-app: kube-dns
  ports:
  - protocol: UDP
    port: 53
```

## 端口规范

```yaml
ports:
- protocol: TCP  # TCP/UDP/SCTP
  port: 8080     # 端口号
  endPort: 8090  # 端口范围（Kubernetes 1.16+）
```

## 默认策略

### 默认拒绝所有入站流量

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-ingress
spec:
  podSelector: {}  # 空选择器，选择所有 Pod
  policyTypes:
  - Ingress
```

### 默认允许所有入站流量（显式）

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-all-ingress
spec:
  podSelector: {}
  ingress:
  - {}
```

### 完整的命名空间隔离

```yaml
---
# 命名空间标签
metadata:
  labels:
    isolation: strict

---
# 允许同命名空间内访问
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-same-namespace
spec:
  podSelector: {}
  ingress:
  - from:
    - podSelector: {}

---
# 允许特定标签的命名空间访问
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-trusted-namespaces
spec:
  podSelector: {}
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          trusted: "true"
```

## 实际案例

### Web 应用三层架构

```yaml
# Nginx 层：只允许来自外部的 80/443 端口
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: nginx-policy
  namespace: production
spec:
  podSelector:
    matchLabels:
      tier: frontend
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - ipBlock:
        cidr: 0.0.0.0/0
    ports:
    - protocol: TCP
      port: 80
    - protocol: TCP
      port: 443
  egress:
  - to:
    - podSelector:
        matchLabels:
          tier: api
    ports:
    - protocol: TCP
      port: 8080
```

```yaml
# API 层：只允许来自 Nginx 的请求
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: api-policy
  namespace: production
spec:
  podSelector:
    matchLabels:
      tier: api
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          tier: frontend
    ports:
    - protocol: TCP
      port: 8080
  egress:
  - to:
    - podSelector:
        matchLabels:
          tier: database
    ports:
    - protocol: TCP
      port: 5432
```

```yaml
# Database 层：只允许来自 API 层的请求
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: database-policy
  namespace: production
spec:
  podSelector:
    matchLabels:
      tier: database
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          tier: api
    ports:
    - protocol: TCP
      port: 5432
  egress: []  # 数据库不主动发起连接
```

## CIDR 规则详解

### 内外部 CIDR

```yaml
# 允许外部 CIDR
ingress:
- from:
  - ipBlock:
      cidr: 192.168.1.0/24

# 排除特定子网
ingress:
- from:
  - ipBlock:
      cidr: 10.0.0.0/8
      except:
      - 10.0.0.0/16
      - 10.1.0.0/16
```

:::tip 排除集群 Pod CIDR
如果允许外部 IP 访问，需要排除集群的 Pod CIDR 和 Service CIDR，否则 ClusterIP 流量可能受影响。
:::

## DNS 访问

几乎所有 Pod 都需要访问 DNS，确保 NetworkPolicy 不会阻断 DNS 请求：

```yaml
# 允许访问 DNS
egress:
- to:
  - namespaceSelector:
      matchLabels:
        kubernetes.io/metadata.name: kube-system
    podSelector:
      matchLabels:
        k8s-app: kube-dns
  ports:
  - protocol: UDP
    port: 53
  - protocol: TCP
    port: 53
```

## 常见错误

### 只定义入站规则但忘记 DNS

```yaml
# 错误：Pod 无法解析 DNS
spec:
  policyTypes:
  - Ingress
  egress:
  - {}  # 禁止所有出站，包括 DNS
```

### 规则顺序问题

```yaml
# 错误：空规则表示允许所有，但后面的规则无法执行
ingress:
- from:
  - podSelector: {}  # 允许所有 Pod
- from:
  - podSelector:
      matchLabels:
        app: specific-app
```

## CNI 支持情况

| CNI 插件 | NetworkPolicy 支持 |
|---|---|
| Calico | 完整支持 |
| Cilium | 完整支持 |
| Weave | 部分支持 |
| Flannel | 不支持 |
| Amazon VPC CNI | 部分支持 |
| GKE Dataplane V2 | 完整支持 |

## 总结

NetworkPolicy 是 Kubernetes 网络安全的基础：

- **Pod 级别隔离**：精确控制 Pod 间的通信
- **多层防护**：Ingress + Egress 双向控制
- **灵活选择**：支持标签、命名空间、IP 段
- **默认拒绝**：通过空选择器实现白名单

正确的 NetworkPolicy 策略应该遵循最小权限原则：默认拒绝，按需开放。
