# Kubernetes 指南

Kubernetes 是目前最流行的容器编排平台，它将容器化应用的生命周期管理提升到了一个全新的高度。从单机的 Docker Compose 到跨集群的云原生架构，Kubernetes 已经成为现代云原生基础设施的事实标准。

想象这样一个场景：你的应用从单机部署开始，随着用户增长，需要水平扩展。最初你可能只是手动启动更多容器实例，但很快就会面临一系列问题——如何发现这些实例？如何实现负载均衡？如何管理配置变更而不影响服务？如何保证应用的高可用？

Kubernetes 正是为解决这些问题而生。它不仅仅是一个容器调度器，更是一个**声明式的自动化运维平台**。你需要做的，只是告诉 Kubernetes 你期望的状态——"我需要 3 个 Nginx 实例均匀分布"，Kubernetes 会自动完成剩余的工作。

## 核心能力概览

| 能力 | 说明 |
|---|---|
| **自动化部署与扩缩容** | 根据负载自动调整 Pod 数量 |
| **服务发现与负载均衡** | 无需修改应用即可实现服务寻址 |
| **自愈能力** | 自动重启失败容器，替换不健康的节点 |
| **配置管理与密钥管理** | 分离配置与镜像，实现配置复用 |
| **存储编排** | 挂载多种存储后端，包括本地、云存储、网络存储 |
| **批量执行** | 支持 Job 和 CronJob 执行定时任务 |

## 学习路径建议

对于刚接触 Kubernetes 的开发者，建议按照以下顺序学习：

1. **先理解架构**：阅读 [架构设计](/devops/k8s/basic/architecture) 和 [核心概念](/devops/k8s/basic/core-concept)，了解 Kubernetes 的设计哲学
2. **掌握基本操作**：熟悉 Deployment、Service、ConfigMap 的基本用法
3. **深入核心机制**：学习 Pod 调度、探针、资源管理，这些是编写生产级 YAML 的基础
4. **扩展网络存储**：理解网络模型和存储卷，为有状态应用做准备

> [!TIP]
> 学习 Kubernetes 的最佳方式不是死记硬背 YAML 模板，而是理解**声明式 API 的思维模式**——你描述期望状态，Kubernetes 负责达成这个状态。
