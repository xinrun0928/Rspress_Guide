# Helm 概览：Kubernetes 包管理器

Helm 是 K8s 的包管理器。想象一下：如果没有 Helm，部署一个 MySQL 集群需要手动编写多少 YAML？Deployment、Service、ConfigMap、Secret、PVC...每个环境还要复制一份，配置稍有不同就要改多个文件。这就是 Helm 存在的意义——**让你一键部署复杂应用，一处配置，处处生效**。

本章节将系统讲解 Helm 的使用方法，包括 Chart 开发、模板语法、Hook 机制和 Helmfile 管理。

## Helm 核心概念

### Chart

Chart 是 Helm 的包格式，包含：

- `Chart.yaml`：应用元信息
- `values.yaml`：默认配置
- `templates/`：K8s 资源模板
- `charts/`：依赖的子 Chart
- `crds/`：自定义 CRD

### Release

Release 是 Chart 的安装实例：

```bash
# 安装 Chart 创建 Release
helm install my-mysql bitnami/mysql

# 查看所有 Release
helm list

# 升级 Release
helm upgrade my-mysql bitnami/mysql --set replicaCount=3

# 回滚 Release
helm rollback my-mysql 1
```

### Repository

Repository 是 Chart 的存储仓库：

```bash
# 添加仓库
helm repo add bitnami https://charts.bitnami.com/bitnami

# 更新仓库索引
helm repo update

# 搜索 Chart
helm search repo mysql

# 移除仓库
helm repo remove bitnami
```

## Helm vs kubectl vs Kustomize

| 维度 | Helm | kubectl | Kustomize |
|---|---|---|---|
| 定位 | 包管理器 | CLI 工具 | 配置管理器 |
| 模板 | Go template | 无 | Overlay |
| 版本管理 | Release 版本化 | 无 | Git 版本化 |
| 依赖 | Chart 依赖 | 无 | 无 |
| 回滚 | 支持 | 不支持 | 不支持 |
| 学习曲线 | 较高 | 低 | 中 |

:::tip 选择建议
- **Helm**：复杂应用、需要版本管理、回滚、多环境部署
- **kubectl**：简单资源、快速验证
- **Kustomize**：GitOps、多环境配置覆盖

实际项目中，可以组合使用：Helm 管理应用，Kustomize 管理环境差异化。
:::
