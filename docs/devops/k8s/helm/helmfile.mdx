# Helmfile：管理多环境多 Chart 部署

当你的项目有多个环境（dev、staging、production），每个环境有多个 Chart 需要部署时，单纯用 Helm 命令管理会变得非常繁琐。每个环境都要执行一遍 `helm upgrade --install`，参数各不相同，容易出错。

Helmfile 就是解决这个问题的——**它是 Helm Release 的声明式配置，让你用 GitOps 的方式管理多环境多 Chart**。

## Helmfile 基础

### 核心概念

```yaml title="helmfile.yaml"
repositories:
- name: bitnami
  url: https://charts.bitnami.com/bitnami
- name: prometheus-community
  url: https://prometheus-community.github.io/helm-charts

releases:
- name: nginx
  chart: bitnami/nginx
  namespace: production
  values:
  - values/production/nginx.yaml
  - values/production/nginx-secrets.yaml.gotmpl
  set:
  - name: replicaCount
    value: 3
```

### 安装 Helmfile

```bash
# macOS
brew install helmfile

# Linux
curl -LO https://github.com/roboll/helmfile/releases/download/v0.156.0/helmfile_linux_amd64
chmod +x helmfile_linux_amd64
sudo mv helmfile_linux_amd64 /usr/local/bin/helmfile

# 验证
helmfile --version
```

## 基础配置

### repositories 声明

```yaml title="helmfile.yaml"
repositories:
- name: bitnami
  url: https://charts.bitnami.com/bitnami
  oci: true  # OCI 仓库

- name: elastic
  url: https://helm.elastic.co

- name: stable
  url: https://charts.helm.sh/stable
  # 认证
  username: {{ requiredEnv "HELM_REPO_USER" }}
  password: {{ requiredEnv "HELM_REPO_PASS" }}
```

### releases 定义

```yaml title="基础 Release"
releases:
# 简单配置
- name: redis
  chart: bitnami/redis

# 完整配置
- name: myapp
  chart: ./charts/myapp
  namespace: production
  version: 1.2.3
  createNamespace: true
  missingFileHandler: Error  # Error, Warn, Info, Debug
  # values 文件
  values:
  - values/{{ .Environment.Name }}/myapp.yaml
  - values/{{ .Environment.Name }}/myapp-secrets.yaml.gotmpl
  # 内联 values
  set:
  - name: replicaCount
    value: 3
  - name: image.tag
    value: "v2.0.0"
  # 条件渲染
  condition: myapp.enabled
  # 标签过滤
  labels:
    app: myapp
    tier: backend
```

## 环境管理

### 环境定义

```yaml title="helmfile.yaml"
environments:
  development:
    values:
    - environments/development.yaml
  staging:
    values:
    - environments/staging.yaml
  production:
    values:
    - environments/production.yaml
```

```yaml title="environments/production.yaml"
namespace: production
replicaCount: 5
resources:
  limits:
    cpu: 2000m
    memory: 4Gi
ingress:
  className: nginx
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
tls:
  enabled: true
```

### 默认环境

```bash
# 使用默认环境（default）
helmfile apply

# 指定环境
helmfile --environment production apply

# 环境变量
export HELMFILE_ENVIRONMENT=production
helmfile apply
```

### 环境覆盖

```yaml title="helmfile.yaml"
releases:
- name: myapp
  chart: ./charts/myapp
  values:
  - values/common.yaml
  - values/{{ .Environment.Name }}/myapp.yaml
  set:
  {{- if eq .Environment.Name "production" }}
  - name: replicaCount
    value: 10
  {{- else }}
  - name: replicaCount
    value: 2
  {{- end }}
```

## 模板语法

### 内置变量

```yaml
{{ .Environment.Name }}    # 环境名称
{{ .Environment.Values }}  # 环境配置值
{{ .Release.Name }}        # Release 名称
{{ .HelmBinary }}          # Helm 二进制路径
{{ .KubeContext }}         # KubeContext
{{ .Namespace }}           # Namespace
```

### 自定义模板函数

```yaml title="templates.gotmpl"
{{- define "release.namespace" -}}
{{- if eq .Environment.Name "production" -}}
prod-{{ .Release.Name }}
{{- else -}}
{{ .Release.Name }}
{{- end -}}
{{- end -}}

{{- define "image.tag" -}}
{{- if eq .Environment.Name "production" -}}
{{ .Values.image.prodTag }}
{{- else -}}
{{ .Values.image.devTag }}
{{- end -}}
{{- end -}}
```

```yaml title="helmfile.yaml"
templates:
  templateFiles:
  - templates.gotmpl

releases:
- name: myapp
  namespace: {{ include "release.namespace" . }}
  values:
  - tag: {{ include "image.tag" . }}
```

## 多环境架构

```

├── helmfile.yaml
├── templates.gotmpl
├── environments/
│   ├── development.yaml
│   ├── staging.yaml
│   └── production.yaml
├── values/
│   ├── common.yaml
│   ├── development/

│   ├── staging/
│   │   └── myapp.yaml
│   └── production/
│       └── myapp.yaml
└── charts/
    └── myapp/
        ├── Chart.yaml
        └── templates/
```

### 分层 helmfile

```yaml title="helmfile.yaml（根）"
helmfiles:
- path: apps/frontend/helmfile.yaml
- path: apps/backend/helmfile.yaml
- path: apps/database/helmfile.yaml
```

```yaml title="apps/frontend/helmfile.yaml"
{{- if eq .Environment.Name "production" }}
{{- $.Values.root.Values.frontend.prod | toYaml }}
{{- else }}
{{- $.Values.root.Values.frontend.dev | toYaml }}
{{- end }}
```

## Stacks

### Stack 概念

Stack 是多个 Release 的组合，便于分组管理：

```yaml title="helmfile.yaml"
stacks:
- name: monitoring
  labels:
    group: monitoring
  values:
  - stacks/monitoring/values.yaml
  targets:
  - name: prometheus
    chart: prometheus-community/prometheus
  - name: grafana
    chart: grafana/grafana
  - name: alertmanager
    chart: prometheus-community/alertmanager

- name: database
  labels:
    group: database
  values:
  - stacks/database/values.yaml
  targets:
  - name: postgresql
    chart: bitnami/postgresql
  - name: redis
    chart: bitnami/redis
```

## CI/CD 集成

### GitHub Actions

```yaml title=".github/workflows/helmfile.yml"
name: Helmfile Deploy

on:
  push:
    branches: [main]
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3

    - name: Setup Helmfile
      uses: azure/setup-helm@v3
      with:
        version: v3.12.0

    - name: Setup Helmfile
      uses: mikoim/helmfile-actions@v1.0.0

    - name: Lint Helmfile
      run: helmfile --environment ${{ github.ref == 'refs/heads/main' && 'production' || 'staging' }} lint

    - name: Diff
      run: |
        helmfile --environment ${{ github.ref == 'refs/heads/main' && 'production' || 'staging' }} diff

    - name: Apply
      if: github.ref == 'refs/heads/main'
      run: helmfile --environment production apply
      env:
        KUBECONFIG_FILE: ${{ secrets.KUBECONFIG }}
```

### GitOps 流程

```bash
# 1. 修改配置
vim values/production/myapp.yaml

# 2. 提交到 Git
git commit -m "Update myapp replica count"
git push

# 3. ArgoCD/Fleet 自动检测变更
# 4. 自动应用变更到集群
```

## 常用命令

### 基本操作

```bash
# 本地渲染
helmfile template

# 差异对比
helmfile diff

# 预览变更
helmfile diff --context 3

# 应用变更
helmfile apply

# 只应用带特定标签的 Release
helmfile apply --selector app=myapp

# 同步（删除不在配置中的 Release）
helmfile sync

# 清理（删除所有 Release）
helmfile destroy
```

### 环境相关

```bash
# 指定环境
helmfile --environment production apply

# 列出所有环境
helmfile environments

# 查看环境值
helmfile --environment production printenv
```

### 调试

```bash
# 详细输出
helmfile --debug apply

# 只执行验证
helmfile lint

# dry-run
helmfile apply --dry-run

# 跟踪执行
helmfile apply --concurrency 1
```

## 最佳实践

### 结构组织

```yaml title="推荐的项目结构"
# helmfile.yaml 保持简洁
bases:
- environments.yaml

helmfiles:
- path: applications/helmfile.yaml
- path: infrastructure/helmfile.yaml
```

```yaml title="applications/helmfile.yaml"
{{- if eq .Environment.Name "production" }}
{{- include "production.overrides" $ }}
{{- end }}

releases:
- name: frontend
  chart: ./charts/frontend
- name: backend
  chart: ./charts/backend
```

### 密钥管理

```yaml title="使用外部密钥"
releases:
- name: myapp
  chart: ./charts/myapp
  secrets:
  - secrets/{{ .Environment.Name }}/myapp-secrets.enc.yaml
```

```bash
# 使用 SOPS 加密
sops --encrypt secrets/production/myapp-secrets.yaml > secrets/production/myapp-secrets.enc.yaml

# helmfile-secrets 插件
helm plugin install https://github.com/jkroepke/helmfile-secrets
```

### 错误处理

```yaml
# missingFileHandler
releases:
- name: myapp
  chart: ./charts/myapp
  missingFileHandler: Warn  # 文件不存在时警告而非报错
  values:
  - values/{{ .Environment.Name }}/myapp.yaml  # 可能不存在
```

## Helmfile vs 纯 Helm

| 维度 | Helmfile | 纯 Helm |
|---|---|---|
| 多环境 | 原生支持 | 需要脚本 |
| 多 Chart | 统一管理 | 逐个管理 |
| GitOps | 友好 | 需要脚本 |
| 学习曲线 | 较高 | 低 |
| 灵活性 | 高 | 更高 |

:::tip 选择建议
- **小团队、少 Chart**：直接用 Helm + 脚本
- **大团队、多环境**：用 Helmfile
- **GitOps 优先**：Helmfile + ArgoCD
:::

## 面试追问方向

1. **Helmfile 和 Helm 的区别？**
   - Helmfile 是 Helm 的声明式配置层
   - Helmfile 管理多个 Release 和环境

2. **Helmfile 如何实现 GitOps？**
   - Git 存储配置
   - ArgoCD/Fleet 监听 Git 变化
   - 自动同步到集群

3. **Helmfile 的模板能力？**
   - Go Template 语法
   - 环境变量
   - 自定义函数

4. **Helmfile 如何处理敏感信息？**
   - helmfile-secrets 插件
   - SOPS 加密
   - External Secrets

## 总结

Helmfile 是多环境多 Chart 管理的利器：

- **声明式配置**：统一管理所有 Release
- **环境支持**：dev/staging/production 差异化
- **模板能力**：Go Template 灵活配置
- **GitOps 友好**：配合 ArgoCD 实现自动化

掌握 Helmfile，你就能高效管理复杂的 K8s 应用部署。
