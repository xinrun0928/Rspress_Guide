# Helm 模板语法：打造可配置的 Chart

Helm 模板是 Helm 的核心能力。它让你编写一次模板，根据不同的 values 生成不同的资源清单。没有模板，Helm 就只是一个配置文件分发工具。**有了模板，Helm 才能真正实现「一次编写，多处复用」**。

## Go Template 基础

### 模板语法

Helm 使用 Go 的 text/template 包：

```yaml title="templates/deployment.yaml"
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ .Release.Name }}-deployment
spec:
  replicas: {{ .Values.replicaCount }}
```

渲染结果：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-release-deployment
spec:
  replicas: 3
```

### 注释

```yaml
{{/* 这是一个注释 */}}
# 单行注释（YAML 风格，不推荐在模板中用）
```

## 内置对象

Helm 提供了多个内置对象：

| 对象 | 说明 |
|---|---|
| `.Release` | Release 信息 |
| `.Values` | values.yaml 中的值 |
| `.Chart` | Chart.yaml 中的值 |
| `.Capabilities` | K8s 版本信息 |
| `.Template` | 当前模板信息 |
| `.Files` | Chart 中的文件 |
| `.Capabilities.APIVersions` | 支持的 API 版本 |

### Release 对象

```yaml
.Release.Name      # Release 名称
.Release.Namespace # Release 所在 Namespace
.Release.IsInstall # 是否是安装操作
.Release.IsUpgrade # 是否是升级操作
.Release.Service   # 服务名称（通常是 "Helm"）
.Release.Revision  # Release 版本号
```

### Chart 对象

```yaml
.Chart.Name        # Chart 名称
.Chart.Version     # Chart 版本
.Chart.AppVersion  # 应用版本
.Chart.Description  # Chart 描述
.Chart.Keywords     # 关键词
.Chart.Maintainers  # 维护者
```

### Capabilities 对象

```yaml
# 检查 K8s 版本支持
{{- if .Capabilities.APIVersions.Has "networking.k8s.io/v1/Ingress" }}
apiVersion: networking.k8s.io/v1
{{- else }}
apiVersion: networking.k8s.io/v1beta1
{{- end }}

# 获取 K8s 版本
{{ .Capabilities.KubeVersion.Version }}
{{ .Capabilities.KubeVersion.GitVersion }}
```

## Values 访问

### 基本访问

```yaml
# values.yaml
replicaCount: 3
image:
  repository: nginx
  tag: "latest"
  pullPolicy: IfNotPresent
```

```yaml
# 访问方式
{{ .Values.replicaCount }}              # 3
{{ .Values.image.repository }}           # nginx
{{ .Values.image.tag }}                  # latest
{{ .Values.image.pullPolicy }}           # IfNotPresent
```

### 默认值

```yaml
# 使用 default
image: {{ .Values.image | default "nginx:latest" }}

# 使用 required
{{ required "image.repository is required" .Values.image.repository }}

# 使用 pluck 和 default
{{ index .Values "image" "repository" | default "nginx" }}
```

## 函数与管道

### 常用函数

```yaml
# quote - 添加引号
name: {{ quote .Values.name }}

# upper/lower - 大小写转换
name: {{ upper .Values.name }}

# default - 默认值
image: {{ .Values.image | default "nginx:latest" }}

# indent - 缩进
{{ .Values.config | indent 2 }}

# nindent - 换行后缩进
{{ .Values.config | nindent 2 }}
```

### 字符串函数

```yaml
# trim - 去除首尾空白
{{ trim .Values.name }}

# trunc - 截断
{{ trunc 63 .Values.name }}

# replace - 替换
{{ replace "-" "_" .Values.name }}

# lower/upper - 大小写
{{ lower .Values.NAME }}
{{ upper .Values.name }}
```

### 数学函数

```yaml
# add - 加法
replicas: {{ add .Values.replicaCount 1 }}

# sub - 减法
timeout: {{ sub .Values.timeout 5 }}

# mul - 乘法
cpu: {{ mul .Values.cpuPerReplica 2 }}

# div - 除法
memoryPerReplica: {{ div .Values.totalMemory .Values.replicaCount }}
```

### 逻辑函数

```yaml
# and - 逻辑与
{{- if and .Values.ingress.enabled .Values.ingress.tls.enabled }}

# or - 逻辑或
{{- if or .Values.postgresql.enabled .Values.externalDb.host }}

# not - 逻辑非
{{- if not .Values.ingress.enabled }}
```

## 流程控制

### if/else

```yaml
{{- if .Values.ingress.enabled }}
apiVersion: networking.k8s.io/v1
kind: Ingress
{{- else }}
# 不创建 Ingress
{{- end }}
```

```yaml
{{- if eq .Values.environment "production" }}
replicas: 10
{{- else if eq .Values.environment "staging" }}
replicas: 3
{{- else }}
replicas: 1
{{- end }}
```

### 运算符

```yaml
{{- if eq .Values.image.tag "latest" }}       # 等于
{{- if ne .Values.env "dev" }}               # 不等于
{{- if lt .Values.replicas 3 }}              # 小于
{{- if le .Values.replicas 3 }}              # 小于等于
{{- if gt .Values.replicas 10 }}             # 大于
{{- if ge .Values.replicas 10 }}             # 大于等于
{{- if and (eq .Values.env "prod") (gt .Values.replicas 5) }}  # 组合
```

### range - 循环

```yaml
# values.yaml
hosts:
- name: app1.example.com
  path: /api
- name: app2.example.com
  path: /v2
```

```yaml
# 遍历列表
{{- range .Values.hosts }}
- host: {{ .name }}
  http:
    paths:
    - path: {{ .path }}
      pathType: Prefix
{{- end }}
```

渲染结果：

```yaml
- host: app1.example.com
  http:
    paths:
    - path: /api
      pathType: Prefix
- host: app2.example.com
  http:
    paths:
    - path: /v2
      pathType: Prefix
```

### range - 遍历 map

```yaml
# values.yaml
env:
  DATABASE_URL: "postgres://localhost:5432"
  LOG_LEVEL: "info"
  NODE_ENV: "production"
```

```yaml
# 遍历字典
{{- range $key, $value := .Values.env }}
- name: {{ $key }}
  value: {{ $value | quote }}
{{- end }}
```

渲染结果：

```yaml
- name: DATABASE_URL
  value: "postgres://localhost:5432"
- name: LOG_LEVEL
  value: "info"
- name: NODE_ENV
  value: "production"
```

### with - 作用域

```yaml
# with 修改当前作用域
{{- with .Values.image }}
repository: {{ .repository }}
tag: {{ .tag | default "latest" }}
{{- end }}
```

### 索引访问

```yaml
# 访问列表元素
{{ index .Values.hosts 0 }}

# 访问字典元素
{{ index .Values.config "database.url" }}

# 嵌套访问
{{ (index .Values.services 0).port }}
```

## 命名模板

### 定义模板

```yaml title="templates/_helpers.tpl"
{{/* 全局名称 */}}
{{- define "myapp.name" -}}
{{ .Release.Name }}-{{ .Chart.Name }}
{{- end -}}

{{/* 标签 */}}
{{- define "myapp.labels" -}}
app.kubernetes.io/name: {{ include "myapp.name" . }}
app.kubernetes.io/instance: {{ .Release.Name }}
app.kubernetes.io/version: {{ .Chart.AppVersion }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
{{- end -}}
```

### 调用模板

```yaml
# include 调用命名模板
metadata:
  name: {{ include "myapp.name" . }}
  labels:
    {{- include "myapp.labels" . | nindent 4 }}

# nindent 自动处理缩进
```

## 条件渲染

### when 子句

```yaml
# when 等同于 if...else
replicas: {{ .Values.replicaCount | default 1 | int }}
{{- when .Values.ingress.enabled }}
---
apiVersion: networking.k8s.io/v1
kind: Ingress
{{- end }}
```

### coalesce

```yaml
# 多个值中取第一个非空
{{- $image := coalesce .Values.image.tag .Values.image.repository "nginx:latest" }}
```

## 特殊字符处理

### 特殊字符转义

```yaml
# 使用 \ 转义
{{ "value with \"quotes\"" }}

# 使用 toJson
{{ .Values.config | toJson }}
```

### 多行字符串

```yaml
# 使用 | 保留格式
data:
  config.yaml: |
    {{ .Values.config | nindent 4 }}
  entrypoint.sh: |
    #!/bin/bash
    {{ .Values.script | nindent 2 }}
```

## 高级用法

### lookup 函数

获取集群中现有资源：

```yaml
{{- $secret := lookup "v1" "Secret" .Release.Namespace "my-secret" }}
{{- if $secret }}
data:
  {{- range $key, $value := $secret.data }}
  {{ $key }}: {{ $value | b64dec }}
  {{- end }}
{{- end }}
```

### include 和 template

```yaml
# 两种调用方式等效
{{ include "myapp.labels" . }}
{{ template "myapp.labels" . }}

# 配合管道
{{ include "myapp.labels" . | nindent 4 }}
```

### 模板函数参考

| 函数 | 说明 |
|---|---|
| `default X Y` | X 为空时使用 Y |
| `required X Y` | X 为空时报错 |
| `coalesce X Y...` | 返回第一个非空值 |
| `lookup G V N R` | 查询集群资源 |
| `toYaml` | 转换为 YAML |
| `toJson` | 转换为 JSON |
| `fromYaml` | 从 YAML 解析 |
| `b64enc` / `b64dec` | Base64 编码/解码 |
| `sha256sum` | SHA256 哈希 |

## 常见问题

### 问题一：缩进不生效

```yaml
# 错误：indent 在模板内不起作用
metadata:
  labels:
    {{ include "myapp.labels" . }}

# 正确：使用 nindent
metadata:
  labels:
    {{- include "myapp.labels" . | nindent 4 }}
```

### 问题二：空值导致渲染失败

```yaml
# 安全访问方式
{{ .Values.config.database.url | default "" | quote }}

# 使用 required
{{ required "database.url is required" .Values.config.database.url }}
```

### 问题三：布尔值判断

```yaml
# Helm 中，空值、false、"false"、"0" 都视为假
{{- if .Values.enabled }}
# ...
{{- end }}

# 显式检查布尔值
{{- if eq (toString .Values.enabled) "true" }}
# ...
{{- end }}
```

## 面试追问方向

1. **Helm 模板函数的执行顺序？**
   - 模板在渲染时执行
   - values 覆盖顺序：Chart defaults < parent values < command-line --set

2. **lookup 函数的用途？**
   - 获取集群中现有资源
   - 用于配置迁移

3. **如何调试 Helm 模板？**
   - `helm template` 本地渲染
   - `helm install --dry-run --debug` 查看详细输出

4. **命名模板的最佳实践？**
   - 统一命名规范（如 `chartname.object`）
   - 使用 `_helpers.tpl` 集中定义

## 总结

Helm 模板是 Chart 灵活性的核心：

- **内置对象**：Release、Values、Chart、Capabilities
- **函数**：字符串、数字、逻辑处理
- **流程控制**：if/else、range、with
- **命名模板**：代码复用
- **高级特性**：lookup、条件渲染

掌握这些，你就能编写出灵活、可配置的 Helm Chart。
