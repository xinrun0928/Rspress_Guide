# Kubernetes 基础

本模块深入讲解 Kubernetes 的核心概念和底层机制。理解这些内容，是编写生产级 YAML 配置的第一步。

## 为什么需要深入学习基础？

很多开发者使用 Kubernetes 的姿势是"复制粘贴 YAML，改改名字就跑"。这种做法在学习和测试环境没有问题，但在生产环境中，你迟早会面临这些问题：

- Pod 一直被驱逐，原因是什么？
- 资源 Limit 设置了，但应用还是被 OOM Kill 了
- 探针配置不对，导致服务刚启动就被停止
- 调度结果不符合预期，Pod 全挤在同一台 Node 上

这些问题的根源，往往是对 Kubernetes 基础概念的误解或忽视。本模块的目标，就是让你**不仅会用，而且懂为什么**。
