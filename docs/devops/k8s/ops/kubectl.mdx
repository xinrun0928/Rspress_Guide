# kubectl 速查手册：Kubernetes 命令行工具

kubectl 是 K8s 的命令行瑞士军刀。熟练掌握 kubectl，可以让你在终端里完成一切操作——从查看 Pod 状态到调试网络问题，从部署应用到排查故障。本篇将系统讲解 kubectl 的常用命令、配置技巧和实战场景。

## kubectl 安装与配置

### 安装 kubectl

```bash
# macOS 安装
brew install kubectl

# Linux 安装
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl
sudo mv kubectl /usr/local/bin/

# 验证安装
kubectl version --client
```

### kubeconfig 配置

kubectl 通过 kubeconfig 文件管理集群访问：

```bash
# 默认位置：~/.kube/config
# 查看当前配置
kubectl config view

# 查看所有上下文
kubectl config get-contexts

# 切换上下文
kubectl config use-context production

# 查看当前上下文
kubectl config current-context
```

:::tip kubeconfig 合并
如果需要同时访问多个集群，可以直接编辑 kubeconfig 或使用 `KUBECONFIG` 环境变量指定多个文件：

```bash
KUBECONFIG=~/.kube/config:/path/to/extra/config kubectl get nodes
```
:::

## 核心命令

### 资源查看

```bash
# 查看所有 Pod（默认 namespace）
kubectl get pods

# 查看所有命名空间的 Pod
kubectl get pods -A

# 查看特定命名空间的 Pod
kubectl get pods -n production

# 精简输出（只显示名称）
kubectl get pods -o name

# 宽表输出（显示更多信息）
kubectl get pods -o wide

# 查看所有资源类型
kubectl api-resources
```

```bash
# 查看 Pod 详细信息
kubectl describe pod nginx-deployment-7fb96c846b-abcde

# 查看节点信息
kubectl describe node worker-node-1

# 查看 Service 信息
kubectl describe svc my-service

# 查看 Events（按时间排序）
kubectl get events --sort-by='.lastTimestamp'
```

### 资源创建与修改

```bash
# 从文件创建资源
kubectl apply -f deployment.yaml

# 从目录创建所有资源
kubectl apply -f ./configs/

# 查看资源变更预览（不实际执行）
kubectl diff -f deployment.yaml

# 删除资源
kubectl delete -f deployment.yaml

# 删除特定 Pod
kubectl delete pod nginx-abcde

# 删除所有 Pod（保留 Deployment）
kubectl delete pods --all -n production

# 标签筛选删除
kubectl delete pods -l app=nginx -n production
```

### 缩放与滚动更新

```bash
# 扩缩容
kubectl scale deployment/nginx --replicas=5

# 自动扩缩容（HPA）
kubectl autoscale deployment/nginx --min=2 --max=10 --cpu-percent=80

# 滚动更新
kubectl set image deployment/nginx nginx=nginx:1.25

# 查看滚动更新状态
kubectl rollout status deployment/nginx

# 回滚到上一个版本
kubectl rollout undo deployment/nginx

# 回滚到特定版本
kubectl rollout undo deployment/nginx --to-revision=2

# 查看滚动历史
kubectl rollout history deployment/nginx
```

### 日志与调试

```bash
# 查看 Pod 日志
kubectl logs nginx-deployment-7fb96c846b-abcde

# 实时查看日志
kubectl logs -f nginx-deployment-7fb96c846b-abcde

# 多容器 Pod 查看特定容器日志
kubectl logs nginx-pod -c sidecar

# 查看最近 100 行日志
kubectl logs --tail=100 nginx-deployment-7fb96c846b-abcde

# 查看历史日志（Pod 重启后）
kubectl logs --previous nginx-deployment-7fb96c846b-abcde
```

### 进入容器

```bash
# 进入容器（创建临时 Pod）
kubectl exec -it nginx-deployment-7fb96c846b-abcde -- /bin/sh

# 执行特定命令
kubectl exec nginx-deployment-7fb96c846b-abcde -- ls /app

# 多容器 Pod 进入特定容器
kubectl exec -it nginx-pod -c main-container -- /bin/bash
```

### 端口转发与代理

```bash
# 端口转发（本地访问集群内服务）
kubectl port-forward svc/nginx-service 8080:80

# 端口转发到 Pod
kubectl port-forward nginx-pod 8080:80

# 启动 kubectl proxy（Web UI 访问）
kubectl proxy --port=8001

# 启动 API Server 代理（直接访问）
kubectl proxy --address='0.0.0.0' --port=8001
```

### 服务暴露

```bash
# 创建 NodePort Service
kubectl expose deployment/nginx --type=NodePort --port=80

# 创建 LoadBalancer Service
kubectl expose deployment/nginx --type=LoadBalancer --port=80

# 创建 Ingress
kubectl create ingress nginx-ingress \
  --rule="nginx.example.com/*=nginx-service:80"
```

## 高效用法

### 输出格式化

```bash
# JSON 输出
kubectl get pods -o json

# YAML 输出
kubectl get pods -o yaml

# 自定义列输出
kubectl get pods -o custom-columns='NAME:.metadata.name,STATUS:.status.phase,IP:.status.podIP'

# 宽表输出
kubectl get pods -o wide

# JSONPath 表达式
kubectl get pods -o jsonpath='{.items[*].status.podIP}'
```

### 标签与选择器

```bash
# 按标签筛选
kubectl get pods -l app=nginx,env=production

# 按标签排除
kubectl get pods -l 'app notin (nginx,redis)'

# 查看标签
kubectl get pods --show-labels

# 添加标签
kubectl label pods nginx-pod env=staging

# 更新标签
kubectl label pods nginx-pod env=production --overwrite

# 删除标签
kubectl label pods nginx-pod env-
```

### Watch 模式

```bash
# 持续监控资源变化
kubectl get pods -w

# 监控特定资源
kubectl get nodes -w

# 监控多个资源
kubectl get pods,nodes -w
```

### 资源管理

```bash
# 排除节点进行维护
kubectl drain worker-node-1 --ignore-daemonsets --delete-emptydir-data

# 恢复节点
kubectl uncordon worker-node-1

# 标记节点不可调度
kubectl cordon worker-node-1

# 节点打标签
kubectl label node worker-node-1 disk=ssd

# 查看节点污点
kubectl describe node worker-node-1 | grep -A 10 Taints
```

## API 参考

### kubectl explain

```bash
# 查看资源字段定义
kubectl explain pod
kubectl explain pod.spec
kubectl explain pod.spec.containers

# 递归查看完整结构
kubectl explain pod --recursive
```

### API 版本

```bash
# 查看资源支持的版本
kubectl api-resources -o wide

# 查看特定资源的 API 版本
kubectl api-resources --api-group=apps

# 强制替换（使用指定版本）
kubectl replace --force -f updated-deployment.yaml
```

## Shell 自动补全

```bash
# bash
echo 'source <(kubectl completion bash)' >> ~/.bashrc

# zsh
echo 'source <(kubectl completion zsh)' >> ~/.zshrc

# 启用别名
echo 'alias k=kubectl' >> ~/.bashrc
echo 'complete -F __start_kubectl k' >> ~/.bashrc
```

### 常用别名

```bash
# ~/.bashrc 或 ~/.zshrc
alias k='kubectl'
alias kgp='kubectl get pods'

alias kga='kubectl get all'

alias kdf='kubectl delete -f'
```

## 常见问题排查

### Pod 无法启动

```bash
# 1. 查看 Pod 状态
kubectl get pod nginx -o wide

# 2. 查看详细事件
kubectl describe pod nginx

# 3. 查看日志
kubectl logs nginx --previous

# 4. 检查资源限制
kubectl top pod nginx
```

### Service 无法访问

```bash
# 1. 检查 Service 是否存在
kubectl get svc my-service

# 2. 检查 Endpoint
kubectl get endpoints my-service

# 3. 检查 Selector 是否匹配
kubectl describe svc my-service

# 4. 测试连通性（临时 Pod）
kubectl run test --rm -it --image=busybox -- wget -qO- http://my-service:80
```

### 调度问题

```bash
# 1. 检查 Pod 状态
kubectl get pod nginx -o wide

# 2. 查看调度原因
kubectl describe pod nginx | grep -A 5 Events

# 3. 检查节点资源
kubectl describe nodes | grep -A 5 "Allocated resources"

# 4. 检查污点
kubectl get nodes -o custom-columns='NODE:metadata.name,TAINTS:spec.taints[*]'
```

## 面试追问方向

1. **kubectl apply 和 kubectl create 的区别？**
   - apply 是声明式，支持增量更新
   - create 是命令式，适合快速创建

2. **kubectl diff 的作用？**
   - 预览变更但不实际执行
   - 三路合并：本地文件、当前状态、集群状态

3. **如何快速查看资源定义？**
   - `kubectl get <resource> -o yaml` 导出
   - `kubectl explain` 查看字段定义

4. **kubectl drain 和 cordon 的区别？**
   - drain 驱逐 Pod 并标记不可调度
   - cordon 只标记不可调度
