# Kubernetes Dashboard 与可视化工具

kubectl 是终端利器，但有时候图形界面更直观。Kubernetes Dashboard 和 GUI 工具提供了可视化的集群管理方式，适合快速查看资源状态、监控应用健康、排查基础问题。本篇将介绍这些工具的选择和配置。

## Kubernetes Dashboard

### 什么是 Dashboard

Kubernetes Dashboard 是 K8s 官方提供的 Web UI，提供了一个直观的界面来管理集群资源。虽然不适合作为主要运维工具，但在快速查看状态、临时调试场景下非常有用。

### 部署 Dashboard

```bash
# 推荐方式：Helm 部署
helm repo add kubernetes-dashboard https://kubernetes.github.io/dashboard
helm repo update
helm install kubernetes-dashboard kubernetes-dashboard/kubernetes-dashboard \
  --namespace kube-system \
  --set=protocolHttp=true
```

或者使用 kubectl 部署：

```bash
kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.7.0/aio/deploy/recommended.yaml
```

### 访问 Dashboard

Dashboard 默认不暴露外部访问，有几种访问方式：

#### 方式一：kubectl proxy

```bash
# 启动代理
kubectl proxy

# 访问
# http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/
```

#### 方式二：NodePort

```bash
# 修改 Service 为 NodePort
kubectl edit svc kubernetes-dashboard -n kubernetes-dashboard

# 修改 type: ClusterIP -> type: NodePort
# 添加 nodePort: 30000
```

#### 方式三：Ingress

```yaml title="Ingress 配置"
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: dashboard-ingress
  namespace: kubernetes-dashboard
  annotations:
    nginx.ingress.kubernetes.io/ssl-passthrough: "true"
    nginx.ingress.kubernetes.io/auth-type: basic
    nginx.ingress.kubernetes.io/auth-secret: dashboard-auth
spec:
  ingressClassName: nginx
  rules:
  - host: dashboard.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: kubernetes-dashboard
            port:
              number: 443
```

### 获取访问令牌

```bash
# 创建服务账号
kubectl create serviceaccount admin -n kube-system

# 创建集群角色绑定
kubectl create clusterrolebinding admin-binding \
  --clusterrole=cluster-admin \
  --serviceaccount=kube-system:admin

# 获取令牌
kubectl create token admin -n kube-system
```

:::warning 权限控制
上述配置授予了 cluster-admin 权限，仅适合开发/测试环境。生产环境应该创建更细粒度的 RBAC 配置。
:::

### Dashboard 功能概览

| 功能 | 说明 |
|---|---|
| 命名空间管理 | 查看、切换 Namespace |
| Workloads | Deployment、StatefulSet、DaemonSet、Job |
| Service | Service、Ingress、NetworkPolicy |
| Config | ConfigMap、Secret |
| 存储 | PV、PVC、StorageClass |
| 日志查看 | Pod 日志实时查看 |
| Shell | 进入容器终端 |
| 资源编辑 | YAML 在线编辑 |

:::tip Dashboard 局限性
Dashboard 不支持日志聚合、监控图表、多集群管理。对于生产环境，推荐使用专业的监控工具。
:::

## Lens（推荐）

### 为什么选择 Lens

Lens 是目前最受欢迎的 K8s 桌面端工具，相比 Dashboard 有以下优势：

| 特性 | Dashboard | Lens |
|---|---|---|
| 部署方式 | Web UI | 桌面应用 |
| 多集群 | 需要配置 | 原生支持 |
| 资源管理 | 有限 | 完整 CRUD |
| 性能 | 一般 | 流畅 |
| 本地调试 | 不支持 | 支持 |
| 价格 | 免费 | 免费（个人）/ 订阅（团队） |

### 安装 Lens

```bash
# macOS
brew install --cask lens

# Windows
# 从 https://k8slens.dev 下载安装包

# Linux
sudo snap install kontena-lens --classic
```

### 添加集群

1. 打开 Lens
2. 点击左上角 `+` 或 `Add Cluster`
3. 选择 kubeconfig 文件或直接导入

```bash
# 复制 kubeconfig 到 Lens 配置目录
mkdir -p ~/.kube
cp /path/to/your/kubeconfig ~/.kube/config-lens
```

### Lens 核心功能

**资源浏览器**
- 左菜单树形结构，清晰展示资源层级
- 右详情面板，支持 YAML 编辑
- 支持批量操作

**实时监控**
- 内置 Prometheus 集成（需要已安装）
- Pod 资源使用图表
- 历史趋势查看

**日志聚合**
- 多容器日志同屏查看
- 日志搜索和过滤
- 导出日志

**终端集成**
- 右键菜单直接进入 Pod Shell
- kubectl 命令直接执行
- 快捷命令面板

**Helm 集成**
- Helm Release 管理
- Chart 安装和升级
- Values 可视化编辑

## k9s

### k9s 简介

k9s 是终端里的可视化工具，提供类 Vim 的键盘操作体验，适合习惯终端操作的工程师。

```bash
# macOS
brew install k9s

# Linux
curl -sL https://github.com/derailed/k9s/releases/download/v0.27.0/k9s_0.27.0_Linux_amd64.tar.gz | tar xz
sudo mv k9s /usr/local/bin/

# 验证
k9s version
```

### 常用快捷键

| 快捷键 | 功能 |
|---|---|
| `:` | 打开命令面板 |
| `q` | 退出 |
| `1-9` | 切换视图 |
| `d` | 查看详情 |
| `e` | 编辑资源 |
| `x` | 删除资源 |
| `l` | 查看日志 |
| `s` | Shell 进入容器 |
| `u` | 卸载资源 |
| `p/s` | 上一屏/下一屏 |
| `/` | 搜索过滤 |

```bash
# 常用命令
:pod              # Pod 视图
:deployment       # Deployment 视图
:svc              # Service 视图
:ingress          # Ingress 视图
:namespace        # Namespace 视图
:ctx              # 切换上下文
:profit           # 利润最大化（彩蛋）
```

### k9s 配置

```yaml title="~/.k9s/config.yml"
k9s:
  clusters:
    production:
      contexts:
        production:
          cluster: production
          user: admin
          namespace: default
  ui:
    enableMouse: true
    headless: false
  liveView:
    refreshRate: 2s
  screen:
    dumpOnExit: false
```

## 其他工具

### Octant

VMware 开发的 Web 工具，适合作为 Dashboard 的替代：

```bash
brew install octant

# 运行
octant
# 访问 http://localhost:7777
```

### Stern

日志聚合工具，支持多 Pod、多容器日志同时查看：

```bash
# 安装
brew install stern

# 查看所有 nginx Pod 的日志
stern nginx

# 过滤日志内容
stern nginx --since=15m | grep "ERROR"

# 彩色输出
stern nginx --color=always
```

### kubectx / kubens

集群和命名空间快速切换：

```bash
brew install kubectx

# 切换上下文
kubectx production

# 切换命名空间
kubens kube-system

# 恢复上个上下文/命名空间
kubectx -
kubens -
```

## 工具选择建议

| 场景 | 推荐工具 |
|---|---|
| 快速查看状态 | Dashboard / Lens |
| 日常运维 | Lens / k9s |
| 多集群管理 | Lens |
| 终端爱好者 | k9s |
| 日志排查 | Stern |
| 快速切换上下文 | kubectx |

:::tip 最佳实践
Lens 作为主力工具，配合 kubectl 处理批量操作和脚本自动化，k9s 作为备用选项。这样可以覆盖大多数运维场景。
:::

## 面试追问方向

1. **Dashboard 和 Lens 的区别？**
   - Dashboard 需要部署到集群，Lens 是独立应用
   - Lens 支持多集群，Dashboard 单集群
   - Lens 功能更丰富，性能更好

2. **如何安全暴露 Dashboard？**
   - 使用 Ingress + HTTPS
   - 配置认证（Basic Auth 或 SSO）
   - 网络层限制访问来源

3. **k9s 和 Lens 哪个更适合你？**
   - 习惯终端 + Vim 操作 → k9s
   - 习惯桌面应用 → Lens

## 总结

可视化工具是 kubectl 的补充，而不是替代。掌握 1-2 个工具，配合终端命令，可以大幅提升运维效率。

- **Dashboard**：快速查看，无需本地安装
- **Lens**：功能全面的桌面应用
- **k9s**：终端用户的最佳选择

选择你顺手的工具，熟练使用，比安装多个工具更有价值。
