# Kubernetes 运维概览

Kubernetes 运维是一个系统工程，从日常的 kubectl 操作到高可用集群部署，从资源配额管理到 Prometheus 监控，每一项都需要深入理解。本章节将系统讲解 K8s 运维的各个方面。

## 日常运维工具

### kubectl：K8s 的命令行工具

kubectl 是 K8s 的核心 CLI 工具，几乎所有 K8s 操作都可以通过 kubectl 完成。本章节将详解：

- 常用命令用法：`get`、`describe`、`apply`、`delete`、`exec`、`logs`
- 多集群配置和切换
- 上下文和 Namespace 管理
- Shell 自动补全配置

### Dashboard 与 GUI 工具

命令行不是唯一的选择。可视化工具可以提升运维效率：

- **Kubernetes Dashboard**：Web 界面，适合快速查看资源状态
- **Lens/k9s**：强大的桌面端 CLI 增强工具
- **Octant**：VMware 开发的 Web 工具

:::tip 工具选择
小团队推荐 Lens，一站式管理多集群；大团队推荐 k9s，终端操作更高效。Dashboard 适合临时查看，不建议作为主要运维工具。
:::

## 高可用集群

生产环境必须部署高可用集群，避免单点故障。高可用涉及：

- **控制平面高可用**：多 master 节点部署
- **Etcd 高可用**：奇数节点组成 Raft 集群
- **负载均衡**：API Server 的外部 LB 配置
- **工作节点分布**：跨故障域分布

### Etcd 数据备份

Etcd 存储了整个集群的状态，备份是重中之重：

- 定时快照机制
- S3/Velero 备份集成
- 灾难恢复流程
- Etcd 压缩和碎片整理

### 集群升级

K8s 版本升级需要谨慎规划：

- 版本偏移策略（最多两个小版本）
- 升级顺序：Etcd → 控制平面 → 工作节点
- 滚动升级与回滚策略
- Canary 升级方法

## 资源管理

### 资源配额

通过 ResourceQuota 和 LimitRange 控制资源使用：

| 机制 | 作用域 | 作用 |
|---|---|---|
| ResourceQuota | Namespace | 限制 Namespace 总资源 |
| LimitRange | Namespace | 设置容器默认/最大资源 |

### 自动扩缩容

K8s 提供多种自动扩缩容机制：

| 机制 | 适用场景 | 指标类型 |
|---|---|---|
| HPA | Deployment/ReplicaSet | CPU、内存、自定义指标 |
| VPA | Pod 推荐 | 基于历史资源使用 |
| Cluster Autoscaler | Node | Pod 调度失败 |

## 监控与可观测性

### Prometheus 监控

Prometheus 是 K8s 监控的事实标准：

- Prometheus Server：指标采集和存储
- kube-state-metrics：K8s 对象状态指标
- node-exporter：节点级别指标
- Alertmanager：告警管理
- Grafana：可视化面板

### Operator 模式

Operator 简化了有状态应用的管理：

- CRD：扩展 K8s API
- Controller：自定义控制逻辑
- Kubebuilder：快速构建 Operator
