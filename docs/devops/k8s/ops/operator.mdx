# Kubernetes Operator：自定义控制器实战

有状态应用（如数据库、消息队列）为什么难管理？因为它们有「状态」——需要初始化、备份、故障恢复。这些操作不能靠简单的 YAML 配置完成，需要**领域知识**。

Operator 模式就是让 K8s「学会」这些领域知识。你告诉 Operator「我要一个 MySQL 集群」，Operator 会自动处理初始化、主从复制、故障切换、备份等一切。

## Operator 是什么

### 定义

Operator 是 K8s 的扩展模式，通过自定义资源和控制器自动化管理有状态应用的生命周期。

```
┌─────────────────────────────────────────────────────────────┐
│                         K8s API Server                       │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  内置资源：Pod、Deployment、Service                      │ │
│  │  自定义资源：MySQLCluster、RedisCluster、Elasticsearch   │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ 监听资源变化
                              │
┌─────────────────────────────────────────────────────────────┐
│                       Operator Controller                    │
│  ┌───────────────┐  ┌───────────────┐  ┌─────────────────┐  │
│  │  Reconcile    │  │   Watch       │  │   Control Loop   │  │
│  │   Loop        │  │               │  │                  │  │
│  └───────────────┘  └───────────────┘  └─────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ 实际管理
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    有状态应用                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │

│  │  Primary │  │ Replica1 │  │ Replica2 │  │   Job    │    │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### 核心概念

| 概念 | 说明 |
|---|---|
| CRD（Custom Resource Definition） | 定义新的资源类型 |
| Custom Resource | CRD 的实例 |
| Controller | 监听 CR 变化，实际执行操作 |
| Reconciliation Loop | 控制循环，驱动状态向期望状态收敛 |

## CRD 定义

### 基本 CRD

```yaml title="定义 MySQLCluster CRD"
apiVersion: apiextensions.k8s.io/v1
kind: CustomResourceDefinition
metadata:
  name: mysqlclusters.mysql.example.com
  labels:
    app: mysql-operator
spec:
  group: mysql.example.com
  names:
    kind: MySQLCluster
    plural: mysqlclusters
    shortNames:
    - mc
  scope: Namespaced
  versions:
  - name: v1
    served: true
    storage: true
    schema:
      openAPIV3Schema:
        type: object
        properties:
          spec:
            type: object
            required: ["version", "replicas"]
            properties:
              version:
                type: string
                pattern: "^[0-9]+\\.[0-9]+$"
              replicas:
                type: integer
                minimum: 1
                maximum: 9
              storage:
                type: object
                properties:
                  size:
                    type: string
                    pattern: "^[0-9]+Gi$"
                  storageClass:
                    type: string
              backup:
                type: object
                properties:
                  enabled:
                    type: boolean
                  schedule:
                    type: string
          status:
            type: object
            properties:
              phase:
                type: string
              primary:
                type: string
              replicas:
                type: integer
```

### 使用 CR


## Controller 架构

### Reconcile Loop

```go title="Controller 核心逻辑"
func (r *MySQLClusterReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
    // 1. 获取 MySQLCluster 资源
    cluster := &mysqlv1.MySQLCluster{}
    if err := r.Get(ctx, req.NamespacedName, cluster); err != nil {
        return ctrl.Result{}, client.IgnoreNotFound(err)
    }

    // 2. 计算期望状态
    desiredState := r.calculateDesiredState(cluster)

    // 3. 获取当前状态
    currentState := &mysqlv1.MySQLClusterStatus{}
    // ...

    // 4. 调和差异
    if err := r.reconcileResources(ctx, cluster, desiredState); err != nil {
        return ctrl.Result{}, err
    }

    // 5. 更新状态
    cluster.Status.Phase = "Running"
    if err := r.Status().Update(ctx, cluster); err != nil {
        return ctrl.Result{}, err
    }

    return ctrl.Result{RequeueAfter: 30 * time.Second}, nil
}
```

## Webhook 处理

### 验证 Webhook

```go title="ValidateCreate 实现"
func (r *MySQLCluster) ValidateCreate() error {
    // 验证副本数
    if r.Spec.Replicas < 1 || r.Spec.Replicas > 9 {
        return field.Invalid(
            field.NewPath("spec").Child("replicas"),
            r.Spec.Replicas,
            "replicas must be between 1 and 9",
        )
    }

    // 验证存储大小
    if r.Spec.Storage.Size == "" {
        return field.Required(
            field.NewPath("spec").Child("storage").Child("size"),
            "storage size is required",
        )
    }

    return nil
}
```

### 默认值 Webhook

```go title="Default 实现"
func (r *MySQLCluster) SetDefaults() {
    // 设置默认值
    if r.Spec.Replicas == 0 {
        r.Spec.Replicas = 1
    }

    if r.Spec.Storage.Size == "" {
        r.Spec.Storage.Size = "10Gi"
    }

    if r.Spec.Backup != nil && r.Spec.Backup.Schedule == "" {
        r.Spec.Backup.Schedule = "0 2 * * *"
    }
}
```

## 常见 Operator 示例

### cert-manager

cert-manager 是最流行的 K8s Operator：

```yaml title="Certificate CR"
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: example-com-tls
  namespace: production
spec:
  secretName: example-com-tls
  issuerRef:
    name: letsencrypt-prod
    kind: ClusterIssuer
  dnsNames:
  - example.com
  - "*.example.com"
  duration: 2160h
  renewBefore: 360h
```

cert-manager 自动：
- 向 Let's Encrypt 请求证书
- 验证域名所有权
- 续期即将过期的证书
- 存储证书到 Secret

## 最佳实践

### 错误处理

```go title="重试机制"
func (r *MySQLClusterReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
    // 带有重试的协调逻辑
    var err error
    var requeue bool

    for attempt := 0; attempt < 3; attempt++ {
        err = r.reconcileOneStep(ctx, req)
        if err == nil {
            return ctrl.Result{}, nil
        }

        if !isRetryableError(err) {
            return ctrl.Result{}, err
        }

        time.Sleep(time.Duration(attempt+1) * time.Second)
    }

    return ctrl.Result{
        Requeue:      true,
        RequeueAfter: 30 * time.Second,
    }, err
}
```

### 状态管理

```go title="状态更新"
type Phase string

const (
    PhaseCreating Phase = "Creating"
    PhaseRunning  Phase = "Running"
    PhaseUpgrading Phase = "Upgrading"
    PhaseError    Phase = "Error"
)

func (r *MySQLClusterReconciler) updateStatus(ctx context.Context, cluster *mysqlv1.MySQLCluster, phase Phase) error {
    cluster.Status.Phase = string(phase)
    cluster.Status.ObservedGeneration = cluster.Generation
    cluster.Status.LastUpdateTime = metav1.Now()
    return r.Status().Update(ctx, cluster)
}
```

### Owner Reference

```go title="设置 Owner Reference"
func setOwnerReference(owner, object metav1.Object) {
    ref := metav1.OwnerReference{
        APIVersion:         owner.GetObjectKind().GroupVersionKind().GroupVersion().String(),
        Kind:               owner.GetObjectKind().GroupVersionKind().Kind,
        Name:               owner.GetName(),
        UID:                owner.GetUID(),
        BlockOwnerDeletion: pointer.Bool(true),
        Controller:         pointer.Bool(true),
    }

    ownerRefs := object.GetOwnerReferences()
    for i, existing := range ownerRefs {
        if existing.UID == ref.UID {
            ownerRefs[i] = ref
            object.SetOwnerReferences(ownerRefs)
            return
        }
    }

    ownerRefs = append(ownerRefs, ref)
    object.SetOwnerReferences(ownerRefs)
}
```

## Operator SDK vs Kubebuilder

| 维度 | Kubebuilder | Operator SDK |
|---|---|---|
| 定位 | 脚手架工具 | 完整框架 |
| 难度 | 较低 | 较高 |
| 功能 | 基础 CRD + Controller | Helm + Ansible + Go |
| 适用 | 自定义 CR + Controller | 复杂 Operator |

```bash
# Operator SDK 初始化
operator-sdk init --domain example.com --repo github.com/example/operator

# 创建 API
operator-sdk create api --group cache --version v1 --kind Memcached --resource --controller
```

## 面试追问方向

1. **Operator 和普通 Controller 的区别？**
   - 本质相同，Operator 是特定模式的 Controller
   - Operator 专注于有状态应用管理

2. **Reconcile Loop 的核心逻辑？**
   - 获取期望状态（Spec）
   - 获取当前状态（Status）
   - 调和差异
   - 更新状态

3. **为什么需要 Owner Reference？**
   - 级联删除：删除 CR 时自动删除相关资源
   - 垃圾回收

4. **Webhook 的作用？**
   - Validation：验证 CR 合法性
   - Mutating：设置默认值

## 总结

Operator 是 K8s 扩展的核心模式：

- **CRD**：定义新的资源类型
- **Controller**：实现业务逻辑
- **Webhook**：验证和默认值
- **Owner Reference**：资源生命周期管理

掌握 Operator 模式，你就能将 K8s 打造成管理任意应用的平台。
