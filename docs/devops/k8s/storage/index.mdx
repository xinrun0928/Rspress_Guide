# Kubernetes 存储

存储是 Kubernetes 中管理有状态应用的关键。本模块涵盖 Volume、PV/PVC、StorageClass、ConfigMap 和 Secret 的使用方法和最佳实践。

## 容器存储的挑战

容器最大的特点之一是**无状态**——容器可以被随时销毁和重建，数据不会自动保留。但在实际应用中，我们需要：

- 数据库需要持久化存储
- 配置文件需要能够动态更新
- 敏感信息（密码、密钥）需要安全存储
- 多个 Pod 可能需要访问同一个存储卷

Kubernetes 通过一套存储抽象来应对这些挑战。这套抽象既支持临时存储（随 Pod 生命周期），也支持持久化存储（独立于 Pod 存在）。

:::tip 存储选择建议
- **临时数据**：使用 emptyDir
- **节点级别的持久化**：使用 hostPath（需注意单节点限制）
- **跨节点共享存储**：使用网络存储（NFS、Ceph、Cloud Storage）
- **无状态配置**：使用 ConfigMap
- **敏感数据**：使用 Secret（注意加密）
:::
