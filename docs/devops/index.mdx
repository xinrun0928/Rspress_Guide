# DevOps 与云原生技术指南

想象一下这个场景：你刚写完代码，点击「提交」，几分钟后，应用就自动部署到了生产环境——单元测试通过、代码质量扫描通过、容器镜像构建完成、安全漏洞检查通过、灰度发布验证无误。这整个过程，没有人手动干预。

这就是 DevOps 的终极愿景：**让软件交付变得像流水线一样高效、可靠、可重复。**

DevOps 不是某一项技术，而是一套文化、实践和工具的集合。它打破了「开发只管写代码，运维只管上线」的传统边界，让开发和运维成为同一个团队的共同责任。在这个过程中，容器化、编排、自动化、可观测性成为缺一不可的基础设施。

## 你能从这里学到什么

**Docker 部分**，你会深入理解容器化的核心原理——从镜像分层到 Copy-on-Write，从网络通信到存储驱动，从单机使用到集群编排。这些知识是理解 Kubernetes 的基础。

**Kubernetes 部分**，这是本模块的核心。K8s 已经成为容器编排的事实标准，你需要掌握其核心概念、调度机制、网络模型、存储抽象、安全体系，以及日常运维技能。

**构建工具部分**，Maven 和 Gradle 是 Java 项目构建的两大支柱。你需要理解它们的依赖管理机制、生命周期、插件系统，以及如何优化构建性能。

**CI/CD 部分**，从 Git 工作流到 Jenkins 流水线，从 GitLab CI 到 GitHub Actions，从 ArgoCD 到 Tekton——你会了解业界主流的持续交付工具链，以及如何设计高效的流水线。

**自动化运维部分**，Ansible 让配置管理变得可重复，Prometheus + Grafana 让你看清系统的每一个指标，ELK/EFK/Loki 让你追踪每一条日志。

**KubeSphere 部分**，作为国内领先的企业级容器平台，KubeSphere 提供了开箱即用的 DevOps、可观测性、服务网格能力，适合快速搭建企业级容器平台。

## 学习建议

| 场景 | 推荐学习顺序 |
|---|---|
| 零基础入门 | Docker 基础 → Docker Compose → K8s 基础 → K8s 核心资源 → CI/CD |
| 已有 Docker 经验 | K8s 基础 → K8s 网络 → K8s 存储 → K8s 安全 → 自动化运维 |
| 准备面试 | K8s 核心概念 → K8s 网络模型 → K8s 存储 → CI/CD → 监控体系 |
| 企业级平台搭建 | KubeSphere 基础 → DevOps 流水线 → 监控日志 → 运维管理 |

每一章都配有详细的原理解析和实战代码，原理篇的篇幅会更长、更深入，确保你不仅「会用」，更「懂为什么这样设计」。
