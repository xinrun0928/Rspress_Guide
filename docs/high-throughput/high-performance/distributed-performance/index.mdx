# 分布式性能优化

## 模块定位

当单点优化到达极限后，系统性能的天花板就取决于**分布式架构的设计**了。

单机能扛 10 万 QPS，分布式集群能扛 1000 万 QPS——但分布式带来了新的挑战：网络延迟、跨节点协调、数据一致性。分布式性能优化的核心任务，就是解决这些挑战。

本模块聚焦分布式场景下的性能优化，覆盖**批量任务优化**、**链路监控**、**序列化**和**连接池**四大方向。

## 分布式性能 vs 单机性能

| 维度 | 单机优化 | 分布式优化 |
|---|---|---|
| 瓶颈 | CPU、内存、磁盘 | 网络延迟、数据分片、分布式协调 |
| 扩展方式 | 垂直扩展（更大机器） | 水平扩展（更多机器） |
| 一致性 | 无 | 需要权衡（CAP） |
| 复杂度 | 低 | 高 |

## 学习路径

```
distributed-batch（批量任务）→ distributed-metrics（链路监控）
→ distributed-serialization（序列化优化）
→ distributed-connection-pool（连接池）→ distributed-file（文件存储）
```

## 面试考察

分布式性能优化的面试题通常围绕场景设计：

- 「如何设计一个每天定时同步大量数据的任务？」
- 「如何实现全链路压测？」
- 「分布式场景下序列化有什么需要注意的？」

分布式优化没有标准答案，关键是说清楚权衡和取舍。
