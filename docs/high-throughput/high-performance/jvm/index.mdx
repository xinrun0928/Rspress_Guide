# JVM 性能调优

## 模块定位

JVM 是 Java 程序的运行环境，JVM 调优的目标是让程序在各种压力下保持稳定高效的运行。

JVM 调优的核心是两件事：**内存管理**和**垃圾回收**。内存分配不合理会 OOM，GC 停顿过长会影响 RT。两者配合不好，再快的代码也跑不快。

本模块覆盖 JVM 调优的三大方向：**JVM 参数体系**、**GC 调优实战**和**内存优化与 OOM 排查**。

## 面试考察

JVM 调优是 Java 面试的最高频领域之一，考察深度从「会用」到「理解原理」不等：

- 「JVM 内存区域有哪些？」
- 「Minor GC 和 Full GC 的区别？」
- 「什么情况下会触发 Full GC？」
- 「G1 和 CMS 的区别是什么？」
- 「线上 OOM 怎么排查？」
- 「CPU 100% 怎么定位？」

理解原理是基础，能在生产环境中解决问题才是最终目标。
