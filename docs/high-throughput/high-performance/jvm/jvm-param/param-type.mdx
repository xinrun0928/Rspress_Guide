# JVM 参数类型与分类

面对几百个 JVM 参数，你是否感到无从下手？

别担心，这篇文章帮你梳理清楚 JVM 参数的分类体系。三种类型（标准参数、非标准参数、不稳定参数），加上参数命名规范，让你在配置 JVM 时不再迷茫。

## 一、JVM 参数三大类型

### 1.1 类型总览

```
JVM 参数分类体系
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

标准参数（Standard Options）
├─ 所有 JVM 实现必须支持
├─ 稳定性高，跨版本兼容
└─ 示例：-server, -client, -version

非标准参数（Non-Standard Options）
├─ JVM 实现可选支持
├─ 以 -X 开头
└─ 示例：-Xmx, -Xms, -Xss

不稳定参数（XX Options）
├─ 可能随时变更，不保证稳定
├─ 以 -XX 开头
└─ 示例：-XX:+UseG1GC, -XX:MaxGCPauseMillis=200

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 1.2 标准参数（- 开头）

标准参数是 JVM 规范的一部分，所有 JVM 实现都必须支持。

**常用标准参数**：

| 参数 | 说明 |
|---|---|
| `-server` | 使用 Server VM（JDK 8+ 默认） |
| `-client` | 使用 Client VM（JDK 8 默认 Server） |
| `-version` | 显示版本信息 |
| `-help` | 显示帮助信息 |
| `-jar <jarfile>` | 执行 JAR 文件 |
| `-classpath <path>` | 指定类搜索路径 |

```bash
# 查看版本
java -version

# 输出示例
openjdk version "1.8.0_362"
OpenJDK Runtime Environment (Temurin-8.8.0.362) (build 1.8.0_362-b09)
OpenJDK 64-Bit Server VM (build 25.362-b09, mixed mode)
```

### 1.3 非标准参数（-X 开头）

非标准参数是 JVM 的通用选项，以 `-X` 开头。

**常用非标准参数**：

| 参数 | 说明 | 默认值 |
|---|---|---|
| `-Xms<size>` | 初始堆大小 | 物理内存的 1/64 |
| `-Xmx<size>` | 最大堆大小 | 物理内存的 1/4 |
| `-Xss<size>` | 线程栈大小 | 1MB |
| `-Xmn<size>` | 年轻代大小 | - |
| `-Xloggc:<file>` | GC 日志文件 | - |
| `-Xprof` | 启用 CPU 分析 | - |
| `-Xshare:auto` | 共享类数据（默认） | auto |
| `-Xshare:off` | 不使用共享数据 | - |
| `-Xshare:on` | 强制使用共享数据 | - |

```bash
# 常用配置
java -Xms2g -Xmx2g -Xss1m -jar app.jar

# -Xms 和 -Xmx 建议设置为相同值
# 避免运行时调整堆大小带来的开销
```

### 1.4 不稳定参数（-XX 开头）

不稳定参数是 JVM 的高级配置，以 `-XX` 开头，用于调优 GC、内存、 JIT 编译等。

**命名规则**：

```
-XX:<+/-><name>        布尔选项
-XX:<name>=<value>     带值选项
```

```bash
# 布尔选项（+ 启用，- 禁用）
-XX:+UseG1GC           # 启用 G1 GC
-XX:-UseStringDeduplication  # 禁用字符串去重

# 带值选项
-XX:MaxGCPauseMillis=200    # 设置最大 GC 停顿时间
-XX:MetaspaceSize=256m      # 设置元空间初始大小
```

## 二、参数命名规范

### 2.1 布尔选项

```
命名模式：UseXXX 或 AlwaysXXX
```

| 参数 | 说明 |
|---|---|
| `-XX:+UseG1GC` | 使用 G1 收集器 |
| `-XX:+UseStringDeduplication` | 启用字符串去重 |
| `-XX:+PrintGCDetails` | 打印 GC 详情 |
| `-XX:+HeapDumpOnOutOfMemoryError` | OOM 时导出堆 |
| `-XX:+AlwaysPreTouch` | 启动时预触摸堆内存 |

### 2.2 数值选项

```
命名模式：XXXSize, XXXThreshold, XXXRatio 等
```

| 参数 | 说明 |
|---|---|
| `-XX:MetaspaceSize=256m` | 元空间大小 |
| `-XX:MaxGCPauseMillis=200` | 最大 GC 停顿时间 |
| `-XX:NewRatio=2` | 新老年代比例 |
| `-XX:SurvivorRatio=8` | Eden 与 Survivor 比例 |
| `-XX:G1HeapRegionSize=4m` | G1 Region 大小 |

### 2.3 百分比选项

```
命名模式：XXXPercent
```

| 参数 | 说明 |
|---|---|
| `-XX:MetaspaceSize=128m` | 元空间大小 |
| `-XX:MaxRAMPercentage=75.0` | 堆占容器内存百分比 |
| `-XX:G1HeapWastePercent=5` | G1 堆浪费百分比 |

## 三、常用参数速查表

### 3.1 堆内存参数

| 参数 | 说明 | 推荐值 |
|---|---|---|
| `-Xms` | 初始堆大小 | 与 -Xmx 相同 |
| `-Xmx` | 最大堆大小 | 容器内存的 50-75% |
| `-Xmn` | 年轻代大小 | 堆大小的 30-50% |
| `-XX:NewSize` | 年轻代初始大小 | - |
| `-XX:MaxNewSize` | 年轻代最大大小 | - |
| `-XX:NewRatio` | 新老年代比例 | 2（年轻代:老年代=1:2） |
| `-XX:SurvivorRatio` | Eden 与 Survivor 比例 | 8（Eden:S0:S1=8:1:1） |

### 3.2 元空间参数

| 参数 | 说明 | 推荐值 |
|---|---|---|
| `-XX:MetaspaceSize` | 元空间初始大小 | 128-256MB |
| `-XX:MaxMetaspaceSize` | 元空间最大大小 | 256-1024MB |

### 3.3 线程栈参数

| 参数 | 说明 | 默认值 |
|---|---|---|
| `-Xss` | 线程栈大小 | 1MB |
| `-XX:ThreadStackSize` | 与 -Xss 相同 | 1MB |

### 3.4 GC 相关参数

| 参数 | 说明 |
|---|---|
| `-XX:+UseSerialGC` | 使用 Serial GC |
| `-XX:+UseParallelGC` | 使用 Parallel GC |
| `-XX:+UseConcMarkSweepGC` | 使用 CMS GC |
| `-XX:+UseG1GC` | 使用 G1 GC |
| `-XX:+UseZGC` | 使用 ZGC |
| `-XX:+UseShenandoahGC` | 使用 Shenandoah GC |

### 3.5 GC 日志参数（JDK 8）

| 参数 | 说明 |
|---|---|
| `-XX:+PrintGCDetails` | 详细 GC 日志 |
| `-XX:+PrintGCDateStamps` | 日期时间戳 |
| `-XX:+PrintGCTimeStamps` | JVM 启动时间戳 |
| `-Xloggc:<file>` | GC 日志文件 |

### 3.6 GC 日志参数（JDK 9+）

| 参数 | 说明 |
|---|---|
| `-Xlog:gc*:<options>` | 所有 GC 日志 |
| `-Xlog:gc=info:<options>` | GC info 日志 |

## 四、查看 JVM 参数

### 4.1 PrintFlagsFinal

`PrintFlagsFinal` 是查看 JVM 参数的重要工具。

```bash
# 查看所有 JVM 参数
java -XX:+PrintFlagsFinal -version

# 查看特定参数
java -XX:+PrintFlagsFinal -version | grep -i MaxGCPause
```

**输出示例**：

```
uintx MaxGCPauseMillis                    = 200
bool  UseG1GC                             = false
size_t MetaspaceSize                      = 21807104
```

**列说明**：

| 列 | 说明 |
|---|---|
| `uintx` | 无符号整数类型 |
| `bool` | 布尔类型 |
| `size_t` | 内存大小类型 |
| `=` | 默认值 |
| `:=` | 修改后的值 |

### 4.2 jinfo 查看运行时参数

```bash
# 查看所有参数
jinfo -flags <pid>

# 查看指定参数
jinfo -flag MaxHeapSize <pid>
jinfo -flag UseG1GC <pid>

# 查看系统属性
jinfo -sysprops <pid>
```

### 4.3 jcmd 查看参数

```bash
# 查看所有 VM 参数
jcmd <pid> VM.flags

# 查看版本和参数
jcmd <pid> VM.version
jcmd <pid> VM.system_properties
```


## 六、参数设置方式

### 6.1 命令行设置

```bash
# 启动时设置
java -Xmx2g -Xms2g -XX:+UseG1GC -XX:MaxGCPauseMillis=200 -jar app.jar
```

### 6.2 环境变量设置（JDK 9+）

```bash
# 设置 JAVA_TOOL_OPTIONS
export JAVA_TOOL_OPTIONS="-Xmx1g -XX:+PrintGCDetails"

# 所有 JVM 进程都会读取这个变量
java -jar app1.jar
java -jar app2.jar
```

### 6.3 配置文件设置

```bash
# 创建配置文件
cat > /etc/java/java.conf << 'EOF'
-Xmx2g
-Xms2g
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
EOF

# 通过 @ 符号加载
java @/etc/java/java.conf -jar app.jar
```

### 6.4 运行时修改（部分参数）

```bash
# jinfo 可以修改部分运行时参数
jinfo -flag +PrintGCDetails <pid>    # 启用
jinfo -flag -PrintGCDetails <pid>    # 禁用
jinfo -flag MaxGCPauseMillis=300 <pid>  # 修改值

# 注意：并非所有参数都支持运行时修改
```

## 八、面试实战

### 面试题一：标准参数和非标准参数有什么区别？

**参考答案**：

标准参数是 JVM 规范的一部分，所有 JVM 实现都必须支持，稳定性和兼容性最好。

非标准参数是 JVM 的通用选项，以 `-X` 开头，部分 JVM 实现可能不支持。

不稳定参数以 `-XX` 开头，用于高级调优，可能在不同版本之间变更。

---

### 面试题二：如何查看 JVM 的所有参数？

**参考答案**：

1. **命令行查看**：`java -XX:+PrintFlagsFinal -version`
2. **运行时查看**：`jinfo -flags <pid>`
3. **指定参数查看**：`jinfo -flag MaxHeapSize <pid>`
4. **JDK 9+ 日志查看**：使用 `-Xlog` 框架

---

### 面试题三：-Xmx 和 -XX:MaxHeapSize 有什么区别？

**参考答案**：

本质上没有区别。`-Xmx` 是 `-XX:MaxHeapSize` 的简写形式。

- `-Xmx2g`：非标准参数，简洁写法
- `-XX:MaxHeapSize=2g`：不稳定参数，完整写法

两者效果相同，建议使用 `-Xmx`，因为更简洁且兼容性好。
