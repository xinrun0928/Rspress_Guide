# JVM 参数体系

## 模块定位

JVM 参数是 GC 调优和内存优化的基础。

不理解参数，就无法看懂 GC 日志；不理解参数，就无法配置一个合理的 JVM 环境。JVM 参数看起来很多很杂，但本质上分为几大类：内存参数、GC 参数、调试参数。

本模块帮你梳理 JVM 参数的完整体系，理解每个参数的作用和配置思路。

## JVM 参数速查

| 类别 | 核心参数 | 说明 |
|---|---|---|
| 堆内存 | -Xms / -Xmx | 初始/最大堆大小 |
| 年轻代 | -Xmn / -XX:NewRatio | 年轻代大小 / 比例 |
| 元空间 | -XX:MetaspaceSize | 初始元空间大小 |
| GC 算法 | -XX:+UseG1GC | 启用 G1 收集器 |
| 停顿时间 | -XX:MaxGCPauseMillis | G1 目标停顿时间 |
| GC 日志 | -Xlog:gc* | JDK 9+ 统一日志 |
| 诊断 | -XX:+HeapDumpOnOutOfMemoryError | OOM 时导出堆转储 |

## 学习路径

```
param-type（参数分类）→ heap-param（堆内存）
→ gc-param（GC 算法选择）→ metaspace-param（元空间）
→ g1-param（G1 调优）→ debug-param（调试与诊断）
```

## 面试考察

JVM 参数的面试题通常从两个角度切入：

**角度一：参数含义**
- 「-Xms 和 -Xmx 的区别？」
- 「-XX:+UseG1GC 是什么意思？」
- 「MaxMetaspaceSize 和 MaxPermSize 有什么区别？」

**角度二：参数配置**
- 「生产环境 JVM 参数怎么配置？」
- 「堆内存设置多少合适？」
- 「G1 的参数怎么调？」

参数本身不难记，理解背后的原理才能灵活运用。
