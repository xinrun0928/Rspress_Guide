# GC 调优实战

## 模块定位

GC（垃圾回收）是 JVM 调优中最需要「实战经验」的领域。

理论上，你知道 CMS 有五个阶段、G1 有 SATB 算法，但生产环境里 Full GC 频繁了怎么查？GC 停顿时间 P99 超过 500ms 怎么调？这些问题，只有真正调过 GC 的人才知道答案。

本模块从**分析 GC 日志**开始，到**定位 GC 问题**，再到**逐个 GC 收集器的调优**，覆盖 GC 调优的完整流程。

## 面试考察

GC 调优的面试题通常围绕三个维度：

**维度一：原理理解**
- 「Minor GC 和 Full GC 的区别？」
- 「G1 的工作原理是什么？」
- 「CMS 的五个阶段是什么？」

**维度二：问题排查**
- 「Full GC 频繁怎么排查？」
- 「GC 日志怎么分析？」
- 「P99 GC 停顿时间过长怎么调？」

**维度三：参数配置**
- 「G1 的参数怎么配置？」
- 「CMS 和 G1 怎么选择？」

理解原理是基础，能解决实际问题才是关键。
