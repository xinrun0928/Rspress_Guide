# NoSQL 与搜索引擎

## 模块定位

关系型数据库不是万能的。当数据量达到千万级、查询模式变得复杂、业务对实时性有更高要求时，NoSQL 和专业搜索引擎就成了必然选择。

本模块聚焦面试中出现频率最高的两个方案：**Elasticsearch**（搜索和分析）和 **ClickHouse**（OLAP 分析），MongoDB 作为补充了解即可。

## ES vs MySQL：什么时候用 ES？

| 场景 | 推荐 |
|---|---|
| 精确查询、事务、强一致性 | MySQL |
| 全文搜索、多条件组合查询 | ES |
| 大数据量聚合分析 | ClickHouse |
| 海量数据模糊搜索 | ES + MySQL |
