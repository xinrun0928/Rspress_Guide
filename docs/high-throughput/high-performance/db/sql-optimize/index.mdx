# SQL 优化：原理与实战

## 模块定位

SQL 优化是数据库性能优化的根基，也是 Java 面试中出现频率最高的领域之一。

大多数面试官不会直接问你「MySQL 的 MVCC 是怎么实现的」（当然也会问），但几乎每个面试都会问：「你的 SQL 优化经验有哪些？」「分页查询很慢怎么解决？」「索引失效的场景有哪些？」

本模块从**定位慢查询**开始，到**分析执行计划**，再到**逐个解决**各类 SQL 性能问题，覆盖 SQL 优化的完整链路。

## SQL 优化的三层境界

**第一层：会用**——能写 SQL，知道加索引能让查询变快。

**第二层：会分析**——能读懂 EXPLAIN，知道 type、key、Extra 字段的含义，能找到索引失效的原因。

**第三层：会设计**——能根据业务场景设计索引，知道什么场景不该加索引，理解索引的代价。

本模块覆盖第一层到第三层，帮你建立完整的 SQL 优化知识体系。

## 学习路径

```
sql-slow（发现慢 SQL）→ explain（分析执行计划）
→ filesort-temporary + index-optimize（索引层面优化）
→ join-optimize + pagination（多表和分页优化）
→ batch-optimize + count-optimize（批量和聚合优化）
```
