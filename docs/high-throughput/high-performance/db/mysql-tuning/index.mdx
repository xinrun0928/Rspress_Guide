# MySQL 调优：配置与生产实践

## 模块定位

索引和 SQL 是 MySQL 优化的「内功」，而 MySQL 配置和架构层面的调优，是连接「优化 SQL」和「生产稳定运行」的最后一环。

很多工程师能写出高效的 SQL，却对 MySQL 的配置参数一头雾水——连接池大小设多少合适？buffer_pool 为什么是 MySQL 最重要的配置？主从延迟怎么处理？

这些问题的答案，都在这个子模块里。

## 核心内容

本模块聚焦 MySQL 在生产环境中的配置调优和架构设计：

| 章节 | 核心问题 | 面试考察 |
|---|---|---|
| [MySQL 配置优化](./mysql-config) | innodb_buffer_pool_size、max_connections 怎么配置？ | 高频 |
| [并发与锁](./concurrency) | MVCC 是怎么实现的？MySQL 如何处理并发更新？ | 高频 |
| [读写分离](./read-write) | 读写分离的延迟问题如何解决？延迟从哪个环节来？ | 高频 |
| [热点行问题](./hot-row) | 大促期间同一商品被大量并发购买，如何处理？ | 大厂高频 |
| [连接池配置](./connection-pool) | 数据库连接池大小如何配置？ | 高频 |

## 学习路径

```
mysql-config（理解 MySQL 核心配置）→ concurrency（理解并发控制原理）
→ read-write（理解主从架构）→ hot-row（理解热点问题处理）
→ connection-pool（理解连接管理）
```

## 面试考察

MySQL 调优的面试题通常围绕两个维度：

**维度一：配置参数**
- 「innodb_buffer_pool_size 设置多少合适？」
- 「max_connections 设太大有什么风险？」

**维度二：生产场景**
- 「MySQL 主从延迟怎么解决？」
- 「大促期间热点行问题怎么处理？」
- 「MVCC 是怎么实现的？」

理解配置背后的原理，比记住参数值更重要。
