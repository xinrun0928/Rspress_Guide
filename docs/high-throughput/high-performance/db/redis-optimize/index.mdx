# Redis 性能优化

## 模块定位

Redis 是高并发系统的缓存核心，也是面试中被问到频率最高的 NoSQL 数据库。

大多数候选人能用 Redis 实现缓存功能，但被问到「Redis 为什么这么快」「大 key 怎么处理」「集群扩容时数据怎么迁移」等问题时，往往答不上来。

本模块帮你从「会用 Redis」提升到「理解 Redis」，覆盖 Redis 性能优化的核心知识点。

## Redis 为什么快？

三个原因缺一不可：

1. **纯内存操作**：数据存储在内存中，访问速度比磁盘快 10 万倍
2. **单线程模型**：避免了锁竞争和上下文切换的开销（Redis 6.0 引入多线程 I/O）
3. **IO 多路复用**：使用 epoll 高效管理海量连接
