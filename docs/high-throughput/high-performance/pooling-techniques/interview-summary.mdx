# 池化技术面试题汇总

池化技术是 Java 面试的高频考点，涵盖线程池、连接池、内存池等多种技术。这篇文章帮你系统性地整理所有核心面试题。

## 一、线程池面试题

### 1.1 线程池有哪些参数？工作流程是什么？

**普通回答**：有 corePoolSize、maximumPoolSize、keepAliveTime、workQueue 等参数。先判断线程数是否小于 corePoolSize，小于则创建线程，否则放入队列，队列满则创建临时线程，临时线程也满则拒绝。

**优秀回答**：

> ThreadPoolExecutor 有七个核心参数：corePoolSize（核心线程数）、maximumPoolSize（最大线程数）、keepAliveTime（空闲存活时间）、TimeUnit（时间单位）、workQueue（任务队列）、threadFactory（线程工厂）、handler（拒绝策略）。
>
> 工作流程分三步：
> 第一，判断当前线程数是否小于 corePoolSize。如果是，则创建新核心线程处理任务。
> 第二，如果核心线程满了，则尝试把任务放入队列。如果队列未满，任务进入队列等待。
> 第三，如果队列也满了，再判断当前线程数是否小于 maximumPoolSize。如果是，则创建临时线程处理任务。如果也达到上限，则执行拒绝策略。
>
> 关键点：线程池不会一开始就创建 corePoolSize 个线程，而是按需创建。

### 1.2 CPU 密集型和 IO 密集型线程池如何配置？

**深入回答**：

> **CPU 密集型任务**（大量计算）：核心线程数 = CPU 核心数 + 1。原因是 CPU 密集型任务几乎不阻塞，线程数等于 CPU 核心数即可充分利用 CPU。
>
> **IO 密集型任务**（数据库查询、网络请求）：核心线程数 = CPU 核心数 × 期望 CPU 利用率 × (1 + IO 耗时/CPU 耗时)。原因是 IO 等待时 CPU 是空闲的，所以需要更多线程来保持 CPU 利用率。
>
> 举例：4 核 CPU，每个任务 IO 等待 100ms，CPU 计算 20ms：
> 核心线程数 = 4 × 1 × (1 + 100/20) = 4 × 6 = 24

### 1.3 线程池的队列有哪些类型？

**深入回答**：

> **LinkedBlockingQueue**：可配置容量，默认无界（风险最大，可能 OOM）。
> **ArrayBlockingQueue**：必须指定容量，有界队列。
> **SynchronousQueue**：不存储任何任务，每个 put 必须等待一个 take。
>
> 生产环境中，LinkedBlockingQueue 必须指定容量。

### 1.4 线程池使用不当会引发什么问题？

**深入回答**：

> **队列无界导致 OOM**：使用 Executors.newFixedThreadPool 默认的 LinkedBlockingQueue 无界队列。
> **拒绝策略未处理**：使用默认的 AbortPolicy 但没有 catch 异常，导致任务丢失。
> **线程泄漏**：核心线程不会超时退出，长时间无任务时白白占用资源。

## 二、连接池面试题

### 2.1 数据库连接池有哪些核心参数？

**深入回答**：

> **容量参数**：minimumIdle（最小空闲）、maximumPoolSize（最大连接数）。
>
> **超时参数**：connectionTimeout（获取连接超时）、idleTimeout（空闲超时）、maxLifetime（最大生命周期）。
>
> **验证参数**：testOnBorrow（获取时验证）、testWhileIdle（空闲时验证）。

### 2.2 HikariCP 为什么这么快？

**深入回答**：

> **FastList 替代 ArrayList**：连接归还通常是最后归还，FastList 从后往前遍历更快。
>
> **字节码生成**：通过 Javassist 编译期生成方法，替代反射调用，减少调用开销。
>
> **无等待设计**：使用 CAS 替代锁，减少线程竞争。

## 三、池化通用面试题

### 3.1 池化技术的核心思想是什么？

**深入回答**：

> 池化的核心是**以空间换时间，预分配加复用**。
>
> 时间维度：复用已有资源，避免重复创建。
>
> 空间维度：池的大小有上限，防止资源耗尽。

### 3.2 对象池和内存池的区别？

**深入回答**：

> **对象池**：复用已创建的对象（如数据库连接、线程、HTTP 连接）。适用于创建成本高、生命周期长的资源。
>
> **内存池**：预分配内存块，对象从内存池分配，对象销毁时归还到内存池而不是释放给操作系统。适用于高频率分配/释放对象的场景。

## 四、面试追问预判

线程池追问方向：
- 线程池的线程如何回收？核心线程也会被回收吗？
- 如果一个任务执行时间很长，会影响其他任务吗？
- ForkJoinPool 和 ThreadPoolExecutor 的区别是什么？

连接池追问方向：
- 连接泄漏怎么排查？
- 如何判断连接池大小是否合适？

---

## 总结

池化技术是高性能系统的基础设施。理解其背后的「以空间换时间」思想，就能以不变应万变地应对所有池化相关的问题。

面试中，最重要的不是背参数名称，而是理解**为什么需要这些参数**，以及**如何根据业务场景配置参数**。
