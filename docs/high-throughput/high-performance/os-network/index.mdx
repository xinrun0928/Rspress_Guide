# 操作系统与网络优化

## 模块定位

操作系统是所有软件的底层运行环境。很多性能问题，表面上出在应用层，但根因却在 OS 层面。

TIME_WAIT 堆积导致端口耗尽、GC 停顿其实是 SWAP 的锅、网络带宽被占满导致接口超时——这些问题的发现和解决，都需要操作系统和网络的知识。

本模块覆盖操作系统优化和网络性能优化两个方向，是高性能优化的底层补充。

## 操作系统层面常见的性能问题

| 问题 | 表现 | 根因 |
|---|---|---|
| TIME_WAIT 堆积 | 无法建立新连接 | 短连接过多，2MSL 等待 |
| CLOSE_WAIT 堆积 | 连接未关闭 | 代码中未正确关闭连接 |
| SWAP 使用 | GC 停顿变长 | 堆内存分配过大，内存不足 |
| 上下文切换过多 | CPU 利用率高但吞吐低 | 线程数过多，锁竞争严重 |

## 网络层面常见的性能问题

| 问题 | 表现 | 根因 |
|---|---|---|
| 连接超时 | RT 忽高忽低 | TIME_WAIT / 端口耗尽 |
| 带宽占满 | 传输速度慢 | 未启用压缩、未用 CDN |
| RTT 过高 | 跨机房延迟大 | 未做同机房路由 |

## 学习路径

```
os-optimize（操作系统基础）→ network-optimize（网络优化）
```

## 面试考察

OS 和网络优化的面试题通常比较综合：

- 「TIME_WAIT 过多有什么危害？怎么解决？」
- 「如何排查网络延迟问题？」
- 「CPU 使用率不高但系统响应慢，可能是什么原因？」

理解 OS 层面的知识，可以帮助你从更底层定位问题。
