# 接口优化与最佳实践

## 模块定位

一个接口的性能，取决于两个关键环节：**数据传输**和**调用方式**。

- 数据传输慢：序列化体积大、响应未压缩、网络传输耗时
- 调用方式低效：重复调用、连接未复用、幂等性未保证

接口优化是高性能优化中最「贴近业务」的环节——不像 GC 调优需要理解 JVM 底层，但它直接影响用户体验和系统吞吐。

本模块不追求面面俱到，聚焦高频考点：**序列化选型**、**幂等性保证**、**连接池配置**、**响应压缩**，这些是面试和实战中出现频率最高的四个方向。

## 学习路径

```
serialization（序列化）→ api-idempotent（幂等性）
→ http-keepalive + connection-pool-api（连接复用）
→ compression + cdn（传输优化）
→ api-checklist（优化 checklist）
```

先理解「为什么慢」（序列化、重复请求），再解决「怎么快」（连接复用、压缩），最后用 checklist 查漏补缺。

## 面试考察

接口优化类面试题的考察点通常比较直接：

- 「JSON 和 Protobuf 的区别？各自适用场景？」
- 「接口幂等性怎么实现？」
- 「HTTP/1.1 和 HTTP/2 的区别？」
- 「如何保证接口不被重复调用？」

这些问题的答案都不复杂，关键是能说出「为什么这样选」——这才是面试官真正想听到的。
