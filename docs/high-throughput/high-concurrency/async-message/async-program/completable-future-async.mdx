# CompletableFuture 异步编排实战：从入门到精通

假设你正在开发一个商品详情页面，后端需要聚合多个数据源：用户信息、商品基本信息、商品评价、库存状态、推荐商品。这五个接口之间没有依赖关系，完全可以并行调用。

如果用同步方式串行调用，每个接口耗时 100ms，总耗时就是 500ms。但如果用 `CompletableFuture` 并行调用，总耗时就是 100ms（取最长）——性能提升 5 倍！

这就是 `CompletableFuture` 的威力。在高并发场景下，它是我们提升系统吞吐量、优化响应时间的利器。

## 一、Future 接口的局限性回顾

在说 `CompletableFuture` 之前，我们先回顾一下它的前身——`Future` 接口。

### 1.1 Future 接口的用法

<Badge text="JDK 5+" type="tip" /> JDK 5 引入了 `Future` 接口，作为异步计算的结果：

```java
public interface Future<V> {
    V get() throws InterruptedException, ExecutionException;
    V get(long timeout, TimeUnit unit) throws ...;
    boolean cancel(boolean mayInterruptIfRunning);
    boolean isDone();
    boolean isCancelled();
}
```

```java
ExecutorService executor = Executors.newFixedThreadPool(10);

Future<String> future = executor.submit(() -> {
    // 模拟耗时操作
    Thread.sleep(2000);
    return "Hello";
});

// 主线程可以做其他事情
doSomethingElse();

// 阻塞等待结果
String result = future.get();  // 可能阻塞很久
```

### 1.2 Future 的三大局限

| 局限 | 说明 |
|---|---|
| **无法组合** | 无法将两个 Future 结果组合成一个，无法链式调用 |
| **无法手动完成** | 无法提前设置结果，只能等待计算完成 |
| **缺少回调** | 没有回调机制，只能通过 get() 阻塞等待 |

```java
// Future 无法优雅地处理这种场景：
// 1. 查询用户信息
Future<User> userFuture = executor.submit(() -> getUser(1L));
// 2. 根据用户信息查询订单
Future<List<Order>> ordersFuture = executor.submit(() -> getOrders(userFuture.get()));  // 无法直接用 userFuture
// 必须这样：先 get() 阻塞，再提交新的任务
```

## 二、CompletableFuture 的创建

### 2.1 supplyAsync 与 runAsync

<Badge text="JDK 8+" type="tip" /> `CompletableFuture` 是 JDK 8 引入的异步编程利器，支持链式调用、组合操作和回调。

```java
// supplyAsync: 有返回值
CompletableFuture<String> future1 = CompletableFuture.supplyAsync(() -> {
    return "Hello";
});

// runAsync: 无返回值
CompletableFuture<Void> future2 = CompletableFuture.runAsync(() -> {
    System.out.println("Hello");
});

// 默认使用 ForkJoinPool.commonPool()
CompletableFuture<String> future3 = CompletableFuture.supplyAsync(() -> {
    return "Hello";
});

// 指定线程池
ExecutorService executor = Executors.newFixedThreadPool(10);
CompletableFuture<String> future4 = CompletableFuture.supplyAsync(() -> {
    return "Hello";
}, executor);
```

### 2.2 completedFuture 与 failedFuture

```java
// 直接返回一个已完成的 Future
CompletableFuture<String> successFuture = CompletableFuture.completedFuture("result");

// 返回一个已失败的 Future
CompletableFuture<String> failedFuture = CompletableFuture.failedFuture(new RuntimeException("error"));

// 实际使用场景
public CompletableFuture<User> getUserById(Long id) {
    if (id == null) {
        return CompletableFuture.failedFuture(new IllegalArgumentException("id 不能为空"));
    }
    User user = userRepository.findById(id);
    return CompletableFuture.completedFuture(user);
}
```

### 2.3 创建方式的对比

| 方法 | 返回值 | 线程池 | 典型场景 |
|---|---|---|---|
| `supplyAsync(Supplier)` | 有 | ForkJoinPool.commonPool() | 有返回值的异步任务 |
| `runAsync(Runnable)` | 无 | ForkJoinPool.commonPool() | 无返回值的异步任务 |
| `supplyAsync(Supplier, Executor)` | 有 | 自定义线程池 | 需要控制线程资源 |
| `runAsync(Runnable, Executor)` | 无 | 自定义线程池 | 需要控制线程资源 |
| `completedFuture(T)` | 有（同步） | 无 | 同步返回已知结果 |
| `failedFuture(Throwable)` | 有（同步） | 无 | 同步返回已知异常 |

## 三、转换方法：thenApply、thenAccept、thenRun

### 3.1 thenApply：转换结果

`thenApply` 用于对上一步的结果进行转换，返回一个新的 `CompletableFuture`：

```java
// 同步版本：thenApply
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "Hello")           // 返回 String
    .thenApply(result -> result + " World") // 转换，返回 String
    .thenApply(result -> result + "!");    // 继续转换

String result = future.get();  // "Hello World!"
```

```java
// 实际场景：用户 ID -> 用户信息 -> 用户订单 -> 订单数量
CompletableFuture<Integer> future = CompletableFuture
    .supplyAsync(() -> getUserId("zhangsan"))           // Long
    .thenApply(userId -> userRepository.findById(userId))  // User
    .thenApply(user -> orderRepository.findByUserId(user.getId())) // List<Order>
    .thenApply(orders -> orders.size());  // Integer

int orderCount = future.get();
```

### 3.2 thenAccept：消费结果

`thenAccept` 用于消费上一步的结果，不返回新值：

```java
CompletableFuture.supplyAsync(() -> "Hello")
    .thenAccept(result -> {
        System.out.println("收到结果: " + result);
    })
    .thenRun(() -> {
        System.out.println("上一步完成后执行，不需要上一步的结果");
    });
```

### 3.3 thenRun：不依赖结果

`thenRun` 完全不依赖前一步的结果，只在前一步完成后执行：

```java
CompletableFuture.supplyAsync(() -> {
    // 异步任务 A
    return "result";
})
.thenRun(() -> {
    // 不关心上一步的结果，只关心上一步是否完成
    System.out.println("任务 A 完成了，我可以开始任务 B");
});
```

### 3.4 Async 变体

每个转换方法都有对应的 Async 版本，会在线程池中异步执行：

```java
// thenApply: 同步执行（复用前一步的线程）
.thenApply(result -> transform(result));

// thenApplyAsync: 异步执行（使用线程池）
.thenApplyAsync(result -> transform(result));

// thenApplyAsync(executor): 异步执行（使用指定线程池）
.thenApplyAsync(result -> transform(result), customExecutor);
```

```java
// 场景：数据处理比较耗时，适合异步执行
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> fetchFromDatabase())
    .thenApplyAsync(data -> {           // 异步执行，不阻塞主线程
        return processData(data);
    }, dataProcessingExecutor)
    .thenApply(result -> formatResult(result));
```

## 四、组合方法：thenCompose、thenCombine

### 4.1 thenCompose：扁平化链式调用

`thenCompose` 用于处理两个异步任务之间存在依赖关系的情况：第二个任务依赖第一个任务的结果。

```java
// 错误示例：使用 thenApply 导致嵌套 Future
CompletableFuture<CompletableFuture<List<Order>>> future = CompletableFuture
    .supplyAsync(() -> getUserId("zhangsan"))
    .thenApply(userId -> getOrdersAsync(userId)); // 返回 CompletableFuture<List<Order>>

// 正确示例：使用 thenCompose 扁平化
CompletableFuture<List<Order>> future = CompletableFuture
    .supplyAsync(() -> getUserId("zhangsan"))
    .thenCompose(userId -> getOrdersAsync(userId)); // 直接返回 List<Order>
```

```java
// thenCompose 适用场景：级联调用
public CompletableFuture<OrderDetail> getOrderDetail(Long orderId) {
    return CompletableFuture
        .supplyAsync(() -> orderRepository.findById(orderId))  // 查订单
        .thenCompose(order -> CompletableFuture
            .supplyAsync(() -> userService.getUser(order.getUserId()))  // 查用户
            .thenCombine(
                CompletableFuture.supplyAsync(() -> productService.getProduct(order.getProductId())),
                (user, product) -> new OrderDetail(order, user, product)  // 组合
            )
        );
}
```

### 4.2 thenCombine：合并两个结果

`thenCombine` 用于处理两个独立的异步任务，它们都完成后合并结果：

```java
// 查询用户信息和商品信息，并行执行
CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(() -> getUser(1L));
CompletableFuture<Product> productFuture = CompletableFuture.supplyAsync(() -> getProduct(100L));

// 合并结果
CompletableFuture<Order> orderFuture = userFuture.thenCombine(
    productFuture,                          // 第二个 Future
    (user, product) -> new Order(user, product)  // 合并函数
);
```

### 4.3 场景对比

| 方法 | 依赖关系 | 返回类型 |
|---|---|---|
| `thenCompose` | 第二个任务依赖第一个任务的结果 | 扁平化（不嵌套） |
| `thenCombine` | 两个任务独立，完成后合并结果 | 合并后的结果 |

```java
// thenCompose: 级联调用
future1.thenCompose(result1 -> future2(result1));  // result1 是 future2 的输入

// thenCombine: 并行合并
future1.thenCombine(future2, (result1, result2) -> merge(result1, result2));
```

## 五、allOf 与 anyOf：并行等待

### 5.1 allOf：等待所有任务完成

`allOf` 等待所有传入的 `CompletableFuture` 全部完成：

```java
// 并行调用 5 个接口
CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(() -> getUser(id));
CompletableFuture<Product> productFuture = CompletableFuture.supplyAsync(() -> getProduct(id));
CompletableFuture<List<Review>> reviewFuture = CompletableFuture.supplyAsync(() -> getReviews(id));
CompletableFuture<Stock> stockFuture = CompletableFuture.supplyAsync(() -> getStock(id));
CompletableFuture<List<Product>> recommendFuture = CompletableFuture.supplyAsync(() -> getRecommend(id));

// 等待所有完成
CompletableFuture<Void> allFuture = CompletableFuture.allOf(
    userFuture, productFuture, reviewFuture, stockFuture, recommendFuture
);

allFuture.get();  // 阻塞直到所有完成

// 获取结果（allOf 返回 Void，需要手动获取每个结果）
User user = userFuture.get();
Product product = productFuture.get();
```

```java
// 实际封装
public CompletableFuture<PageData> getProductDetail(Long productId) {
    CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(() -> getCurrentUser());
    CompletableFuture<Product> productFuture = CompletableFuture.supplyAsync(() -> productService.getProduct(productId));
    CompletableFuture<List<Review>> reviewFuture = CompletableFuture.supplyAsync(() -> reviewService.getReviews(productId));
    CompletableFuture<Stock> stockFuture = CompletableFuture.supplyAsync(() -> stockService.getStock(productId));

    return CompletableFuture.allOf(userFuture, productFuture, reviewFuture, stockFuture)
        .thenApply(ignored -> {
            try {
                return new PageData(
                    userFuture.get(),
                    productFuture.get(),
                    reviewFuture.get(),
                    stockFuture.get()
                );
            } catch (Exception e) {
                throw new RuntimeException(e);
            }
        });
}
```

### 5.2 anyOf：竞速获取

`anyOf` 在任意一个 `CompletableFuture` 完成后就返回：

```java
// 场景：多机房部署，优先使用最快响应的机房
CompletableFuture<String> beijingFuture = CompletableFuture.supplyAsync(() ->
    remoteService.callBeijing("data"));
CompletableFuture<String> shanghaiFuture = CompletableFuture.supplyAsync(() ->
    remoteService.callShanghai("data"));
CompletableFuture<String> guangzhouFuture = CompletableFuture.supplyAsync(() ->
    remoteService.callGuangzhou("data"));

// 任意一个完成即可
CompletableFuture<Object> fastestFuture = CompletableFuture.anyOf(
    beijingFuture, shanghaiFuture, guangzhouFuture
);

String result = (String) fastestFuture.get();  // 最快返回的结果
```

### 5.3 对比

| 方法 | 等待条件 | 返回值 | 典型场景 |
|---|---|---|---|
| `allOf` | 所有完成 | Void | 并行调用多个接口，最后聚合结果 |
| `anyOf` | 任意一个完成 | Object（需要强转） | 多机房竞速、重试机制 |

## 六、异常处理

### 6.1 exceptionally：异常时返回默认值

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> {
        if (true) throw new RuntimeException("Oops!");
        return "success";
    })
    .exceptionally(ex -> {
        System.out.println("捕获异常: " + ex.getMessage());
        return "default value";  // 返回默认值
    });

String result = future.get();  // "default value"
```

### 6.2 handle：无论成功失败都处理

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> {
        if (true) throw new RuntimeException("Oops!");
        return "success";
    })
    .handle((result, ex) -> {
        if (ex != null) {
            System.out.println("处理异常: " + ex.getMessage());
            return "default value";
        }
        System.out.println("处理结果: " + result);
        return result;
    });
```

### 6.3 whenComplete：副作用处理

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> callRemoteService())
    .whenComplete((result, ex) -> {
        // 不影响主流程的清理操作
        if (ex != null) {
            log.error("调用失败", ex);
            metrics.recordFailure();
        } else {
            metrics.recordSuccess();
        }
    });
```

### 6.4 exceptionallyCompose：异常时执行另一个 Future

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> callPrimaryService())
    .exceptionallyCompose(ex -> {
        log.warn("主服务调用失败，尝试备用服务", ex);
        return callBackupService();  // 返回一个新的 CompletableFuture
    });
```

### 6.5 异常处理的对比

| 方法 | 返回值 | 场景 |
|---|---|---|
| `exceptionally` | 返回默认值或抛出新异常 | 简单的异常恢复 |
| `handle` | 根据结果或异常返回对应值 | 需要区分成功和失败的场景 |
| `whenComplete` | 不改变原结果（副作用） | 记录日志、监控等 |
| `exceptionallyCompose` | 返回新的 CompletableFuture | 需要进行复杂的异常恢复 |

## 七、实战演练

### 7.1 场景一：并行调用多个下游接口

```java
public CompletableFuture<ProductDetailVO> getProductDetail(Long productId) {
    // 1. 并行查询多个数据源
    CompletableFuture<Product> productFuture = CompletableFuture
        .supplyAsync(() -> productService.getProduct(productId), executor);
    CompletableFuture<List<Review>> reviewFuture = CompletableFuture
        .supplyAsync(() -> reviewService.getReviews(productId), executor);
    CompletableFuture<Stock> stockFuture = CompletableFuture
        .supplyAsync(() -> stockService.getStock(productId), executor);
    CompletableFuture<List<Product>> recommendFuture = CompletableFuture
        .supplyAsync(() -> recommendService.getRecommend(productId), executor);

    // 2. 等待所有结果
    return CompletableFuture.allOf(productFuture, reviewFuture, stockFuture, recommendFuture)
        .thenApply(ignored -> {
            try {
                return ProductDetailVO.builder()
                    .product(productFuture.get())
                    .reviews(reviewFuture.get())
                    .stock(stockFuture.get())
                    .recommends(recommendFuture.get())
                    .build();
            } catch (Exception e) {
                throw new RuntimeException("构建商品详情失败", e);
            }
        })
        .exceptionally(ex -> {
            log.error("获取商品详情失败, productId={}", productId, ex);
            throw new ServiceException("获取商品详情失败");
        });
}
```

### 7.2 场景二：级联调用优化

```java
// 同步版本：串行调用，总耗时 = sum(每个调用耗时)
public UserOrderVO getUserOrderSync(Long userId) {
    User user = userService.getUser(userId);      // 100ms
    List<Order> orders = orderService.getOrders(userId);  // 200ms
    List<Product> products = productService.getProducts(orders);  // 300ms
    return new UserOrderVO(user, orders, products);  // 总耗时 600ms
}

// 异步版本：并行调用，总耗时 = max(最长调用耗时)
public CompletableFuture<UserOrderVO> getUserOrderAsync(Long userId) {
    CompletableFuture<User> userFuture = CompletableFuture
        .supplyAsync(() -> userService.getUser(userId), executor);
    CompletableFuture<List<Order>> ordersFuture = userFuture
        .thenCompose(user -> CompletableFuture
            .supplyAsync(() -> orderService.getOrders(user.getId()), executor));
    CompletableFuture<List<Product>> productsFuture = ordersFuture
        .thenCompose(orders -> CompletableFuture
            .supplyAsync(() -> productService.getProducts(orders), executor));

    return userFuture
        .thenCombine(ordersFuture, (user, orders) ->
            new UserOrderVO(user, orders, null))
        .thenCombine(productsFuture, (vo, products) -> {
            vo.setProducts(products);
            return vo;
        });
    // 总耗时约 300ms（最长的 productService.getProducts）
}
```

### 7.3 场景三：超时控制

```java
// 设置超时：orTimeout (JDK 9+)
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> callRemoteService())
    .orTimeout(5, TimeUnit.SECONDS)  // 5 秒超时
    .exceptionally(ex -> {
        if (ex instanceof TimeoutException) {
            return "超时了，返回默认值";
        }
        throw new RuntimeException(ex);
    });

// JDK 8 实现超时：使用 CompletableFuture.anyOf
public <T> CompletableFuture<T> withTimeout(CompletableFuture<T> future, long timeout, TimeUnit unit) {
    CompletableFuture<T> timeoutFuture = CompletableFuture.supplyAsync(() -> {
        throw new TimeoutException("操作超时");
    });

    return (CompletableFuture<T>) CompletableFuture.anyOf(future, timeoutFuture)
        .thenCompose(result -> {
            if (result instanceof TimeoutException) {
                throw new RuntimeException("超时");
            }
            return CompletableFuture.completedFuture(result);
        });
}
```

## 八、线程池的选择

### 8.1 默认线程池：ForkJoinPool.commonPool()

`CompletableFuture` 默认使用 `ForkJoinPool.commonPool()`：

```java
// 源码
public static <U> CompletableFuture<U> supplyAsync(Supplier<U> supplier) {
    return asyncSupplyStage(screenExecutor(supplier), maker-supply(supplier));
}
```

```java
// ForkJoinPool.commonPool() 的特点：
// - 线程数 = Runtime.getRuntime().availableProcessors() - 1
// - 所有 CompletableFuture 默认共享
// - 适合 CPU 密集型任务
```

### 8.2 自定义线程池

```java
// IO 密集型任务：使用更大的线程池
ExecutorService ioExecutor = new ThreadPoolExecutor(
    20,                    // 核心线程数
    50,                    // 最大线程数
    60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(500),
    new ThreadFactoryBuilder().setNameFormat("io-async-%d").build(),
    new ThreadPoolExecutor.CallerRunsPolicy()
);

CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> callRemoteService(), ioExecutor)  // 指定线程池
    .thenApplyAsync(result -> process(result), ioExecutor);  // 也指定
```

### 8.3 线程池配置建议

| 任务类型 | CPU 核心数 | 线程池大小建议 |
|---|---|---|
| CPU 密集型 | N | N 或 N+1 |
| IO 密集型 | N | N * 2 或更多 |

```java
// 最佳实践：为 IO 密集型任务配置合理的线程池
int cores = Runtime.getRuntime().availableProcessors();
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    cores * 2,             // 核心线程数
    cores * 4,             // 最大线程数
    60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(1000),
    new ThreadFactoryBuilder()
        .setNameFormat("cf-async-%d")
        .setDaemon(true)   // 设置为守护线程
        .build(),
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略：调用方执行
);
```

## 九、常见坑与最佳实践

### 9.1 不指定线程池的问题

```java
// 错误：大量异步任务可能耗尽共享线程池
for (int i = 0; i < 1000; i++) {
    CompletableFuture.supplyAsync(() -> process(i));  // 使用 commonPool
}

// 正确：使用自定义线程池
for (int i = 0; i < 1000; i++) {
    CompletableFuture.supplyAsync(() -> process(i), customExecutor);
}
```

### 9.2 异常被忽略的问题

```java
// 错误：异常被吞掉，没有处理
CompletableFuture.supplyAsync(() -> callService())
    .thenApply(this::process);  // 如果 callService 失败，异常丢失

// 正确：添加异常处理
CompletableFuture.supplyAsync(() -> callService())
    .exceptionally(ex -> {
        log.error("服务调用失败", ex);
        return getDefaultValue();
    })
    .thenAccept(this::process);
```

### 9.3 get() 阻塞的问题

```java
// 错误：在异步流程中调用 get()，导致阻塞
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> callService1())
    .thenApply(result1 -> {
        // 错误：在 thenApply 中调用 get()，失去异步优势
        String result2 = future2.get();
        return result1 + result2;
    });

// 正确：使用 thenCompose 扁平化
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> callService1())
    .thenCompose(result1 ->
        CompletableFuture.supplyAsync(() -> callService2())
            .thenApply(result2 -> result1 + result2)
    );
```

### 9.4 最佳实践清单

| 实践 | 说明 |
|---|---|
| **指定线程池** | 避免使用默认线程池，为 IO 密集型任务配置专用线程池 |
| **处理异常** | 每个异步链都应添加异常处理，避免异常丢失 |
| **避免 get() 阻塞** | 使用 thenCompose 替代嵌套 get() |
| **设置超时** | 为远程调用设置合理的超时时间 |
| **复用 Future** | 相同的异步调用结果应复用，不要重复提交 |
| **线程安全** | 在异步回调中访问共享变量需注意线程安全 |

## 总结

`CompletableFuture` 是 JDK 8 提供的异步编程利器，它弥补了 `Future` 的三大局限：支持链式调用、支持组合操作、支持回调。通过 `supplyAsync` 和 `runAsync` 创建异步任务，通过 `thenApply`、`thenAccept`、`thenRun` 进行转换，通过 `thenCompose` 和 `thenCombine` 进行组合，通过 `allOf` 和 `anyOf` 实现并行等待，通过 `exceptionally`、`handle`、`whenComplete` 处理异常。

在实际使用中，需要注意：自定义线程池避免线程资源耗尽、为远程调用设置超时、正确处理异常避免异常丢失、使用 `thenCompose` 扁平化嵌套的 `Future`。

## 面试追问预判

**Q1：CompletableFuture 和 Future 有什么区别？**

答：Future 是 JDK 5 引入的异步计算结果接口，支持获取结果和取消任务，但有三个主要局限：无法组合多个 Future、无法手动完成、缺少回调机制。CompletableFuture 是 JDK 8 在 Future 基础上的扩展，支持链式调用、组合操作（thenCompose、thenCombine）、allOf/anyOf 并行等待、内置异常处理，可以完全替代 Future 且更强大。

**Q2：thenCompose 和 thenCombine 的区别是什么？**

答：thenCompose 用于处理有依赖关系的两个异步任务，第二个任务的输入依赖第一个任务的结果，返回类型是扁平的 CompletableFuture。thenCombine 用于处理两个独立的异步任务，两个任务都完成后合并结果，返回合并后的新结果。选择依据是：两个任务之间是否有数据依赖，有则用 thenCompose，无则用 thenCombine。

**Q3：CompletableFuture 的线程池如何选择？**

答：默认使用 ForkJoinPool.commonPool()，线程数 = CPU 核心数 - 1，适合 CPU 密集型任务。对于 IO 密集型任务（如远程调用），需要自定义线程池，线程数通常设置为 CPU 核心数的 2-4 倍。重要原则是：整个异步调用链最好使用同一个线程池，避免线程池切换开销。

**Q4：CompletableFuture 有哪些常见坑？**

答：常见坑包括：1）不指定线程池，使用默认 commonPool，可能被其他任务影响；2）异常被忽略，thenApply 链中的异常可能丢失，必须添加 exceptionally；3）在异步链中调用 get() 导致阻塞，失去异步优势，应使用 thenCompose；4）忘记设置超时，远程调用可能无限等待；5）忽略线程安全问题，在回调中访问共享变量需要同步。
