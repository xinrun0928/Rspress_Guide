# 短链系统设计：从 URL 缩短到跳转全流程

一条微博只能发 140 个字，一条短信链接占 40 个字符，一条微信分享最好控制在 25 个字符以内……

长 URL 的问题，你一定遇到过。

短链系统，就是来解决这个问题的。一个「https://t.cn/abc123」比「https://example.com/product/detail?id=123456789」短了 80%。

但短链系统真的只是「把 URL 变短」吗？面试官追问你几个问题，你就知道这里面的水有多深了：

- 短码怎么生成的？会冲突吗？
- 301 和 302 重定向有什么区别？
- 每天生成 1000 万条短链，数据库怎么扛？
- 短链如何防止被人恶意生成？

这篇文章，把短链系统讲透。

## 一、短链系统的核心问题

### 1.1 什么是短链系统

短链系统的本质是一个「映射服务」：

```
长 URL  ──映射──▶  短码  ──存储──▶  映射关系
                              │
短码  ◀──查询──  映射关系  ◀──存储──▶  长 URL
```

用户访问短链时，系统根据短码找到对应的长 URL，然后重定向到长 URL。

### 1.2 短链的业务价值

| 场景 | 价值 | 示例 |
|---|---|---|
| 短信营销 | 节省短信费用，提升点击率 | 「您有一张优惠券，点击 https://u.jd.com/abc123 领取」 |
| 社交分享 | 分享更简洁，兼容性更好 | 微博、微信限制字符数 |
| 数据统计 | 追踪每个链接的点击量 | UTM 参数、埋点分析 |
| 域名统一 | 统一品牌域名，避免长链 | 公众号、外链管理 |

### 1.3 核心技术挑战

| 挑战 | 说明 |
|---|---|
| **短码生成** | 如何生成唯一、不重复的短码 |
| **存储设计** | 如何存储海量映射关系 |
| **跳转性能** | 如何保证毫秒级跳转 |
| **容量规划** | 如何支撑亿级甚至百亿级存储 |
| **安全防护** | 如何防止恶意生成和滥用 |

## 二、短码生成策略

### 2.1 短码的基本要求

一个好的短码应该满足：

1. **足够短**：6-8 位是最常见的长度
2. **唯一不重复**：绝对不能出现两个长 URL 映射到同一个短码
3. **不可预测**：防止被人枚举所有短码
4. **URL 友好**：只包含 URL 安全字符（a-zA-Z0-9_-）

### 2.2 方案一：Base62 编码

**原理**：将数字 ID 转换为 Base62（0-9, a-z, A-Z）字符串。

```java
public class Base62Util {
    
    private static final String BASE62_CHARS = 
        "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz";
    private static final int BASE = 62;
    
    /**
     * 数字 ID 转 Base62 字符串
     * 例如: 100 -> "1B"
     */
    public static String encode(long num) {
        StringBuilder sb = new StringBuilder();
        while (num > 0) {
            int remainder = (int) (num % BASE);
            sb.append(BASE62_CHARS.charAt(remainder));
            num = num / BASE;
        }
        return sb.reverse().toString();
    }
    
    /**
     * Base62 字符串转数字 ID
     * 例如: "1B" -> 100
     */
    public static long decode(String str) {
        long result = 0;
        for (int i = 0; i < str.length(); i++) {
            char c = str.charAt(i);
            int index = BASE62_CHARS.indexOf(c);
            result = result * BASE + index;
        }
        return result;
    }
    
    /**
     * 生成固定长度的短码（不足左边补 0）
     */
    public static String encodeFixedLength(long num, int length) {
        String encoded = encode(num);
        return String.format("%" + length + "s", encoded).replace(' ', '0');
    }
}
```

**容量计算**：

| 短码长度 | 组合数 | 可存储 URL 数量 |
|---|---|---|
| 6 位 | 62^6 ≈ 568 亿 | 足够大部分场景 |
| 7 位 | 62^7 ≈ 3.5 万亿 | 超大规模 |
| 8 位 | 62^8 ≈ 218 万亿 | 天量级 |

### 2.3 方案二：雪花算法（Snowflake）

**原理**：使用分布式 ID 生成器生成唯一 ID，再 Base62 编码。

```java
public class SnowflakeIdGenerator {
    
    private final long epoch = 1609459200000L;  // 2021-01-01
    private final long workerIdBits = 5;
    private final long datacenterIdBits = 5;
    private final long sequenceBits = 12;
    
    private final long maxWorkerId = ~(-1L << workerIdBits);
    private final long maxDatacenterId = ~(-1L << datacenterIdBits);
    
    private final long workerIdShift = sequenceBits;
    private final long datacenterIdShift = sequenceBits + workerIdBits;
    private final long timestampLeftShift = sequenceBits + workerIdBits + datacenterIdBits;
    private final long sequenceMask = ~(-1L << sequenceBits);
    
    private final long workerId;
    private final long datacenterId;
    private long sequence = 0L;
    private long lastTimestamp = -1L;
    
    public SnowflakeIdGenerator(long workerId, long datacenterId) {
        if (workerId > maxWorkerId || workerId < 0) {
            throw new IllegalArgumentException("workerId 超出范围");
        }
        if (datacenterId > maxDatacenterId || datacenterId < 0) {
            throw new IllegalArgumentException("datacenterId 超出范围");
        }
        this.workerId = workerId;
        this.datacenterId = datacenterId;
    }
    
    public synchronized long nextId() {
        long timestamp = timeGen();
        
        if (timestamp < lastTimestamp) {
            throw new RuntimeException("时钟回拨");
        }
        
        if (timestamp == lastTimestamp) {
            sequence = (sequence + 1) & sequenceMask;
            if (sequence == 0) {
                timestamp = tilNextMillis(lastTimestamp);
            }
        } else {
            sequence = 0L;
        }
        
        lastTimestamp = timestamp;
        
        return ((timestamp - epoch) << timestampLeftShift)
            | (datacenterId << datacenterIdShift)
            | (workerId << workerIdShift)
            | sequence;
    }
    
    private long tilNextMillis(long lastTimestamp) {
        long timestamp = timeGen();
        while (timestamp <= lastTimestamp) {
            timestamp = timeGen();
        }
        return timestamp;
    }
    
    private long timeGen() {
        return System.currentTimeMillis();
    }
}
```

### 2.4 方案三：数据库自增 ID

**原理**：使用数据库自增主键作为短码的来源。

```sql

    long_url TEXT NOT NULL,

```

```java
@Service

    @Autowired
    private ShortUrlMapper shortUrlMapper;
    
    /**
     * 生成短码（基于数据库自增 ID）
     */
    public String generateShortCode() {

```

### 2.5 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| Base62 | 简单，实现容易 | ID 大时编码后较长 | ID 范围可控 |
| 雪花算法 | 分布式唯一，时间有序 | 实现复杂 | 多节点部署 |
| 数据库自增 | 绝对连续，无冲突 | 性能受限，单点 | 小规模系统 |

**推荐方案**：雪花算法 + Base62。分布式唯一，高性能。

## 四、跳转服务实现

### 4.1 301 vs 302 重定向

这是短链系统最常被问到的问题。

| 类型 | 含义 | 特点 | 适用场景 |
|---|---|---|---|
| **301** | 永久重定向 | 浏览器会缓存，后续请求直接访问目标 URL | 永久迁移的链接 |
| **302** | 临时重定向 | 每次都经过短链服务，便于统计 | 营销链接、追踪链接 |

```java
@RestController
public class ShortUrlController {
    
    @Autowired
    private ShortUrlService shortUrlService;
    
    /**
     * 301 永久重定向
     */
    @GetMapping("/{shortCode}")
    public ResponseEntity<Void> redirectPermanent(@PathVariable String shortCode) {
        String longUrl = shortUrlService.getLongUrl(shortCode);
        if (longUrl == null) {
            return ResponseEntity.notFound().build();
        }
        // 301: 永久重定向，浏览器会缓存
        return ResponseEntity.status(HttpStatus.MOVED_PERMANENTLY)
            .location(URI.create(longUrl))
            .build();
    }
    
    /**
     * 302 临时重定向
     */
    @GetMapping("/t/{shortCode}")
    public ResponseEntity<Void> redirectTemporary(@PathVariable String shortCode) {
        String longUrl = shortUrlService.getLongUrl(shortCode);
        if (longUrl == null) {
            return ResponseEntity.notFound().build();
        }
        // 302: 临时重定向，每次都经过短链服务
        return ResponseEntity.status(HttpStatus.FOUND)
            .location(URI.create(longUrl))
            .build();
    }
}
```

:::warning SEO 注意事项

- 301 重定向会被搜索引擎认为是「永久迁移」，权重会传递
- 302 不会被传递权重，可能被搜索引擎视为作弊
- 营销追踪链接通常使用 302，便于统计每次点击

### 4.2 跳转性能优化：缓存策略

```java
@Service
public class ShortUrlCacheService {
    
    private static final String SHORT_URL_CACHE_KEY = "short:url:";
    private static final String LONG_URL_CACHE_KEY = "short:long:";
    private static final long CACHE_EXPIRE_HOURS = 24;
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    /**
     * 获取长 URL（多级缓存）
     */
    public String getLongUrl(String shortCode) {
        // 1. 尝试从缓存获取
        String cacheKey = LONG_URL_CACHE_KEY + shortCode;
        String cached = redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) {
            // 异步记录点击
            recordClickAsync(shortCode);
            return cached;
        }
        
        // 2. 缓存未命中，从数据库获取
        ShortUrlEntity entity = shortUrlMapper.selectByShortCode(shortCode);
        if (entity == null || entity.getStatus() != 1) {
            return null;
        }
        
        // 3. 写入缓存
        redisTemplate.opsForValue().set(
            cacheKey, 
            entity.getLongUrl(), 
            CACHE_EXPIRE_HOURS, 
            TimeUnit.HOURS
        );
        
        // 4. 异步记录点击
        recordClickAsync(shortCode);
        
        return entity.getLongUrl();
    }
    
    /**
     * 生成短链并缓存
     */
    public void cacheShortUrl(String shortCode, String longUrl) {
        String shortCacheKey = SHORT_URL_CACHE_KEY + longUrl;
        String longCacheKey = LONG_URL_CACHE_KEY + shortCode;
        
        redisTemplate.opsForValue().set(shortCacheKey, shortCode, CACHE_EXPIRE_HOURS, TimeUnit.HOURS);
        redisTemplate.opsForValue().set(longCacheKey, longUrl, CACHE_EXPIRE_HOURS, TimeUnit.HOURS);
    }
    
    /**
     * 异步记录点击
     */
    @Async
    public void recordClickAsync(String shortCode) {
        // 更新点击数
        redisTemplate.opsForHyperLogLog().add("click:uv:" + LocalDate.now(), shortCode);
        redisTemplate.opsForValue().increment("click:count:" + shortCode);
    }
}
```

## 六、自定义短码与别名

### 6.1 业务需求

很多场景下，用户希望自定义短码：

- 品牌宣传：「https://xxx.com/summer-sale」
- 易于记忆：「https://xxx.com/jd-coupon」

### 6.2 实现方案

```java
@Service
public class CustomShortCodeService {
    
    @Autowired
    private ShortUrlMapper shortUrlMapper;
    
    /**
     * 检查短码是否可用
     */
    public boolean isAvailable(String shortCode) {
        // 长度检查
        if (shortCode.length() < 4 || shortCode.length() > 20) {
            return false;
        }
        
        // 必须是 URL 安全字符
        if (!shortCode.matches("^[a-zA-Z0-9_-]+$")) {
            return false;
        }
        
        // 保留词检查
        if (isReserved(shortCode)) {
            return false;
        }
        
        // 数据库检查
        return shortUrlMapper.selectByShortCode(shortCode) == null;
    }
    
    /**
     * 注册自定义短码
     */
    @Transactional
    public String registerCustomCode(String shortCode, String longUrl, Long userId) {
        // 1. 权限检查（只有 VIP 用户可以自定义）
        if (!userService.isVip(userId)) {
            throw new BusinessException("自定义短码仅对 VIP 用户开放");
        }
        
        // 2. 可用性检查
        if (!isAvailable(shortCode)) {
            throw new BusinessException("短码已被占用");
        }
        
        // 3. 插入数据库
        ShortUrlEntity entity = new ShortUrlEntity();
        entity.setShortCode(shortCode);
        entity.setLongUrl(longUrl);
        entity.setUserId(userId);
        entity.setCustom(true);
        shortUrlMapper.insert(entity);
        
        // 4. 写入缓存
        redisTemplate.opsForValue().set(
            "short:long:" + shortCode, 
            longUrl, 
            30, TimeUnit.DAYS
        );
        
        return shortCode;
    }
    
    /**
     * 检查是否是保留词
     */
    private boolean isReserved(String shortCode) {
        Set<String> reserved = Set.of(
            "admin", "login", "logout", "api", "www", 
            "static", "assets", "images", "css", "js",
            "about", "contact", "terms", "privacy"
        );
        return reserved.contains(shortCode.toLowerCase());
    }
}
```

```java
@Service
public class ClickAnalyticsService {
    
    @Autowired
    private ClickRecordMapper clickRecordMapper;
    
    /**
     * 记录点击信息
     */
    public void recordClick(ShortCodeRequest request, HttpServletRequest httpRequest) {
        ClickRecord record = new ClickRecord();
        record.setShortCode(request.getShortCode());
        record.setClickTime(LocalDateTime.now());
        record.setIp(getClientIp(httpRequest));
        record.setUserAgent(httpRequest.getHeader("User-Agent"));

        record.setBrowser(parseBrowser(httpRequest.getHeader("User-Agent")));
        record.setOs(parseOs(httpRequest.getHeader("User-Agent")));
        
        // 异步写入数据库
        CompletableFuture.runAsync(() -> clickRecordMapper.insert(record));
    }
    
    /**
     * 获取短链统计信息
     */
    public ShortUrlStats getStats(String shortCode) {
        ShortUrlStats stats = new ShortUrlStats();
        
        // 今日 UV
        String todayKey = "click:uv:" + LocalDate.now();
        Long todayUv = redisTemplate.opsForHyperLogLog().size(todayKey);
        stats.setTodayUv(todayUv != null ? todayUv : 0);
        
        // 今日 PV
        String todayPvKey = "click:count:" + shortCode;
        String todayPv = redisTemplate.opsForValue().get(todayPvKey);
        stats.setTodayPv(todayPv != null ? Long.parseLong(todayPv) : 0);
        
        // 从数据库获取总量
        ShortUrlEntity entity = shortUrlMapper.selectByShortCode(shortCode);
        if (entity != null) {
            stats.setTotalPv(entity.getClickCount());
        }
        
        return stats;
    }
}
```

```java
    public RealtimeStats getRealtimeStats() {
        RealtimeStats stats = new RealtimeStats();
        
        // 当前秒 QPS
        String qpsKey = "stats:qps:" + LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss"));
        String qps = redisTemplate.opsForValue().get(qpsKey);
        stats.setCurrentQps(qps != null ? Long.parseLong(qps) : 0);
        
        // 分钟级趋势
        List<MinuteTrend> trends = new ArrayList<>();
        for (int i = 0; i < 60; i++) {
            String minuteKey = "stats:minute:" + 
                LocalDateTime.now().minusMinutes(i).format(DateTimeFormatter.ofPattern("yyyyMMddHHmm"));
            String count = redisTemplate.opsForValue().get(minuteKey);
            trends.add(new MinuteTrend(i, count != null ? Long.parseLong(count) : 0));
        }
        stats.setMinuteTrends(trends);
        
        return stats;
    }
}
```

## 九、安全防护

### 9.1 恶意生成防护

```java
@Service
public class SecurityService {
    
    /**
     * 频率限制：单用户每小时最多生成 100 条
     */
    public boolean checkRateLimit(Long userId) {
        String key = "rate:create:" + userId;
        Long count = redisTemplate.opsForValue().increment(key);
        if (count != null && count == 1) {
            redisTemplate.expire(key, 1, TimeUnit.HOURS);
        }
        return count == null || count <= 100;
    }
    
    /**
     * 黑名单检查
     */
    public boolean isBlacklisted(String url) {
        String domain = extractDomain(url);
        return redisTemplate.opsForSet().isMember("blacklist:domains", domain);
    }
    
    /**
     * 恶意域名检测
     */
    public boolean isMaliciousUrl(String url) {
        // 1. URL 格式检查
        if (!url.startsWith("http://") && !url.startsWith("https://")) {
            return true;
        }
        
        // 2. 已知恶意域名列表
        if (isBlacklisted(url)) {
            return true;
        }
        
        // 3. 钓鱼检测（相似域名）
        String domain = extractDomain(url);
        if (isSimilarToBrand(domain)) {
            return true;
        }
        
        return false;
    }
}
```

## 十、面试高频问题

### 问题一：短链如何实现防重放攻击？

**参考答案**：

1. **时间戳签名**：在短链中加入时效性签名，如 `t.cn/abc123?token=md5(shortCode+timestamp+secret)`
2. **一次性链接**：设置最大点击次数，达到后自动失效
3. **IP 白名单**：只有白名单 IP 才能访问
4. **访问日志**：记录每次访问的 IP、UA 等信息，异常时告警

### 问题二：如何设计一个每天 10 亿请求的短链系统？

**参考答案**：

| 层级 | 方案 | 预期 QPS |
|---|---|---|
| CDN 层 | 静态化跳转页面，减少回源 | 1000 万+ |
| 缓存层 | Redis Cluster，多级缓存 | 100 万+ |
| 数据库层 | 写多库，读多从，分库分表 | 10 万+ |
| ID 生成 | 雪花算法，本地缓存号段 | 无瓶颈 |

### 问题三：短链系统的容灾设计？

**参考答案**：

1. **多级缓存**：本地缓存 + Redis + 数据库
2. **降级策略**：缓存失效时，直接查询数据库（限流）
3. **数据备份**：定期全量 + 增量备份
4. **跨机房部署**：主备机房自动切换

## 总结

短链系统的设计，核心在于三点：

1. **短码生成**：雪花算法 + Base62，分布式唯一高性能
2. **存储架构**：分库分表 + 多级缓存，支撑海量数据
3. **跳转优化**：301/302 根据场景选择，多级缓存兜底

理解了这三个核心，你就掌握了短链系统的精髓。
