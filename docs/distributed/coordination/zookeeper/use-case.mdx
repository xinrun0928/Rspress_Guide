# ZooKeeper 应用场景

凌晨两点，你的电商平台突然无法创建订单。监控系统显示：库存服务和订单服务「失联」了——它们互相不知道对方的存在，导致整个交易链路断裂。

这是一个典型的**服务发现**问题。在分布式系统中，服务实例可能随时增加或减少，如何让服务消费者动态感知服务提供者？ZooKeeper 就是解决这类问题的利器。

## ZooKeeper 的核心应用场景

ZooKeeper 解决了分布式系统中的四类典型问题：

| 问题类型 | 具体场景 | ZooKeeper 方案 |
|---|---|---|
| **服务发现** | 服务注册与订阅 | 临时节点 + Watch |
| **配置管理** | 动态配置更新 | 持久节点 + Watch |
| **分布式锁** | 互斥访问共享资源 | 临时顺序节点 |
| **Master 选举** | 选出一个主节点执行任务 | 临时节点 + Watch |
| **分布式队列** | 按顺序处理任务 | 临时顺序节点 |

## 服务注册与发现

### 原理

服务启动时向 ZooKeeper 注册自己的地址，服务消费者通过 Watch 监听服务列表变化。

```
┌─────────────────────────────────────────────────────────────┐
│                      服务注册与发现                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────┐    注册     ┌─────────────────┐                │
│  │ Order   │───────────▶│ /services/order │                │
│  │ Service │◀─ Watch ───│  ├── 192.168.1.1 │                │
│  └─────────┘             │  └── 192.168.1.2 │                │
│                          └─────────────────┘                │
│  ┌─────────┐    注册     ┌─────────────────┐                │
│  │Inventory│───────────▶│ /services/inventory               │
│  │ Service │◀─ Watch ───│  └── 192.168.1.3 │                │
│  └─────────┘             └─────────────────┘                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 实现代码

```java
@Service
public class ServiceRegistry {

    private static final String SERVICE_PATH = "/services";

    private ZooKeeper zk;

    /**
     * 注册服务实例
     */
    public void register(String serviceName, String host, int port) throws Exception {
        String instancePath = SERVICE_PATH + "/" + serviceName + "/" +
                             host + ":" + port;

        // 创建临时节点（实例下线自动删除）
        String path = zk.create(instancePath,
            (host + ":" + port).getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.EPHEMERAL);

        System.out.println("服务注册成功: " + path);
    }

    /**
     * 发现服务实例列表
     */
    public List<String> discover(String serviceName) throws Exception {
        String servicePath = SERVICE_PATH + "/" + serviceName;

        try {
            return zk.getChildren(servicePath, false);
        } catch (NoNodeException e) {
            return Collections.emptyList();
        }
    }

    /**
     * 监听服务变化
     */
    public void watchService(String serviceName, Runnable callback) throws Exception {
        String servicePath = SERVICE_PATH + "/" + serviceName;

        Watcher watcher = event -> {
            if (event.getType() == Event.EventType.NodeChildrenChanged) {
                System.out.println("服务列表变化，触发回调");
                callback.run();
            }
        };

        zk.getChildren(servicePath, watcher);
    }
}
```

### 服务消费端

```java
@Service
public class ServiceConsumer {

    @Autowired
    private ServiceRegistry registry;

    @Autowired
    private LoadBalancer loadBalancer;

    /**
     * 调用服务（带负载均衡）
     */
    public String callService(String serviceName, String path) {
        // 获取服务实例列表
        List<String> instances = registry.discover(serviceName);

        if (instances.isEmpty()) {
            throw new RuntimeException("没有可用的服务实例: " + serviceName);
        }

        // 负载均衡选择
        String instance = loadBalancer.select(instances);
        String[] hostPort = instance.split(":");

        // 调用服务
        return restTemplate.getForObject(
            "http://" + hostPort[0] + ":" + hostPort[1] + path,
            String.class
        );
    }

    /**
     * 启动时订阅服务变化
     */
    @PostConstruct
    public void subscribe() throws Exception {
        registry.watchService("order", () -> {
            System.out.println("Order 服务列表已更新，刷新本地缓存");
            // 刷新本地缓存
            refreshLocalCache("order");
        });
    }
}
```

## 动态配置管理

### 原理

配置存储在 ZooKeeper 的持久节点中，应用通过 Watch 监听配置变化，实现热更新。

```
┌─────────────────────────────────────────────────────────────┐
│                      动态配置管理                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ZooKeeper                                                  │
│  ┌─────────────────────────────────────────┐                │
│  │ /configs                                │                │
│  │   ├── database (NodeDataChanged)        │                │
│  │   │   ├── host = 192.168.1.100          │                │
│  │   │   └── port = 3306                   │                │
│  │   └── redis                             │                │
│  │       └── host = 192.168.1.101          │                │
│  └─────────────────────────────────────────┘                │
│                    │ Watch 通知                              │
│                    ▼                                        │
│  应用实例 A ──── 监听到变化 ──── 更新本地配置                  │
│  应用实例 B ──── 监听到变化 ──── 更新本地配置                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 实现代码

```java
@Service
public class ConfigCenter {

    private static final String CONFIG_PATH = "/configs";

    private volatile Map<String, String> configCache = new ConcurrentHashMap<>();
    private ZooKeeper zk;

    /**
     * 初始化并监听配置
     */
    public void init() throws Exception {
        watchConfig(CONFIG_PATH);

        // 加载初始配置
        loadConfig(CONFIG_PATH);
    }

    /**
     * 加载配置到本地缓存
     */
    private void loadConfig(String path) throws Exception {
        List<String> children = zk.getChildren(path, false);

        for (String child : children) {
            String childPath = path + "/" + child;
            Stat stat = new Stat();
            byte[] data = zk.getData(childPath, false, stat);

            if (stat.getNumChildren() == 0) {
                // 叶子节点，存储配置值
                configCache.put(childPath, new String(data));
            } else {
                // 目录，递归加载
                loadConfig(childPath);
            }
        }
    }

    /**
     * 监听配置变化
     */
    private void watchConfig(String path) throws Exception {
        Watcher watcher = event -> {
            if (event.getType() == Event.EventType.NodeDataChanged) {
                // 配置值变化
                try {
                    String newValue = new String(
                        zk.getData(event.getPath(), false, null));
                    configCache.put(event.getPath(), newValue);
                    System.out.println("配置更新: " + event.getPath() + " = " + newValue);
                    notifyConfigChange(event.getPath(), newValue);
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }

            // 重新注册 Watch
            try {
                zk.getChildren(event.getPath(), this::watchConfig);
            } catch (Exception e) {
                e.printStackTrace();
            }
        };

        List<String> children = zk.getChildren(path, watcher);
        for (String child : children) {
            String childPath = path + "/" + child;
            Stat stat = new Stat();
            byte[] data = zk.getData(childPath, watcher, stat);

            if (stat.getNumChildren() == 0) {
                configCache.put(childPath, new String(data));
            }
        }
    }

    /**
     * 获取配置
     */
    public String getConfig(String key) {
        return configCache.get(CONFIG_PATH + "/" + key);
    }

    /**
     * 更新配置
     */
    public void updateConfig(String key, String value) throws Exception {
        String path = CONFIG_PATH + "/" + key;
        zk.setData(path, value.getBytes(), -1);
    }

    /**
     * 通知配置变更
     */
    private void notifyConfigChange(String path, String value) {
        // 可以发送 MQ 消息、HTTP 回调等
        System.out.println("配置变更通知: " + path + " = " + value);
    }
}
```

## Master 选举

### 原理

多个实例竞争创建同一个临时节点，只有创建成功的成为 Master。其他实例监听该节点变化，Master 宕机时重新竞争。

```
┌─────────────────────────────────────────────────────────────┐
│                      Master 选举                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  初始状态：                                                   │
│  /master (不存在)                                            │
│                                                              │
│  实例 A ──── create /master ──── 成功 ──── 成为 Master       │
│  实例 B ──── create /master ──── 失败 ──── Watch /master     │
│  实例 C ──── create /master ──── 失败 ──── Watch /master     │
│                                                              │
│  Master (A) 宕机：                                           │
│  /master 节点删除 ──── 触发 B、C 的 Watch                      │
│  B ──── 重新 create /master ──── 成功 ──── 成为新 Master     │
│  C ──── 继续 Watch                                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 实现代码

```java
@Service
public class MasterElection {

    private static final String MASTER_PATH = "/master";

    private ZooKeeper zk;
    private volatile boolean isMaster = false;
    private String instanceId;

    /**
     * 参与 Master 选举
     */
    public void runForMaster() throws Exception {
        instanceId = InetAddress.getLocalHost().getHostAddress() + ":" +
                    ManagementFactory.getRuntimeMXBean().getPid();

        attemptBecomeMaster();
    }

    private void attemptBecomeMaster() throws Exception {
        try {
            // 尝试创建临时节点
            String path = zk.create(MASTER_PATH,
                instanceId.getBytes(),
                ZooDefs.Ids.OPEN_ACL_UNSAFE,
                CreateMode.EPHEMERAL);

            isMaster = true;
            System.out.println("成为 Master: " + instanceId);

            // 注册死亡监听（当前实例不再是 Master 时触发）
            ((CuratorFramework) zk).newWatch()
                .forPath(MASTER_PATH)
                .withWatcher(event -> {
                    if (event.getType() == Event.EventType.NodeDeleted) {
                        System.out.println("Master 宕机，重新竞争");
                        isMaster = false;
                        try {
                            attemptBecomeMaster();
                        } catch (Exception e) {
                            e.printStackTrace();
                        }
                    }
                });

        } catch (NodeExistsException e) {
            // 已经存在 Master，监听其变化
            isMaster = false;
            System.out.println("当前不是 Master，监听 Master 变化");

            Watcher watcher = event -> {
                if (event.getType() == Event.EventType.NodeDeleted) {
                    System.out.println("Master 宕机，重新竞争");
                    try {
                        attemptBecomeMaster();
                    } catch (Exception ex) {
                        ex.printStackTrace();
                    }
                }
            };

            zk.exists(MASTER_PATH, watcher);
        }
    }

    /**
     * 执行 Master 专属任务
     */
    public void executeIfMaster(Runnable task) {
        if (isMaster) {
            task.run();
        } else {
            System.out.println("不是 Master，跳过执行");
        }
    }

    /**
     * 检查是否是 Master
     */
    public boolean isMaster() {
        return isMaster;
    }
}
```

## 分布式队列

### 原理

使用临时顺序节点实现分布式队列，生产者创建节点入队，消费者按序号消费。

```java
@Service
public class DistributedQueue {

    private static final String QUEUE_PATH = "/queue";

    private ZooKeeper zk;

    /**
     * 入队
     */
    public void enqueue(String task) throws Exception {
        String taskPath = zk.create(QUEUE_PATH + "/task-",
            task.getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.PERSISTENT_SEQUENTIAL);

        System.out.println("任务入队: " + taskPath);
    }

    /**
     * 出队（阻塞）
     */
    public String dequeue() throws Exception {
        while (true) {
            List<String> tasks = zk.getChildren(QUEUE_PATH, false);

            if (tasks.isEmpty()) {
                // 队列为空，等待
                CountDownLatch latch = new CountDownLatch(1);
                zk.getChildren(QUEUE_PATH, event -> {
                    if (event.getType() == Event.EventType.NodeChildrenChanged) {
                        latch.countDown();
                    }
                });
                latch.await();
                continue;
            }

            // 按序号排序
            Collections.sort(tasks);
            String minTask = tasks.get(0);

            // 尝试删除（原子操作，防止并发消费）
            try {
                byte[] data = zk.getData(QUEUE_PATH + "/" + minTask, false, null);
                zk.delete(QUEUE_PATH + "/" + minTask, -1);
                System.out.println("任务出队: " + minTask);
                return new String(data);
            } catch (NoNodeException e) {
                // 任务已被其他消费者取走，重试
                continue;
            }
        }
    }
}
```

## 分布式唯一 ID 生成器

### 原理

利用 ZooKeeper 的顺序节点，生成带序号的唯一 ID。

```java
@Service
public class IdGenerator {

    private static final String ID_PATH = "/ids";

    private ZooKeeper zk;

    /**
     * 生成分布式唯一 ID
     */
    public String generateId(String category) throws Exception {
        String path = zk.create(ID_PATH + "/" + category + "-",
            new byte[0],
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.PERSISTENT_SEQUENTIAL);

        // 提取序号
        String sequence = path.substring(path.lastIndexOf("-") + 1);
        return category + "-" + System.currentTimeMillis() + "-" + sequence;
    }
}
```

## 总结

ZooKeeper 的四大核心应用场景：

1. **服务注册与发现**：临时节点 + Watch，实时感知服务上下线
2. **配置管理**：持久节点 + Watch，实现配置热更新
3. **Master 选举**：临时节点竞争，宕机自动重新选举
4. **分布式锁**：临时顺序节点，保证互斥访问

这些场景都充分利用了 ZooKeeper 的核心特性：**临时节点**保证可靠性，**Watch** 保证实时性，**顺序节点**保证公平性。
