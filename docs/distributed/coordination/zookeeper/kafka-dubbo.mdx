# ZooKeeper 在 Dubbo 和 Kafka 中的应用

|Dubbo| |Kafka| |ZooKeeper|

Dubbo 是阿里巴巴开源的高性能 RPC 框架，Kafka 是LinkedIn 开源的分布式消息队列。这两个系统在分布式架构中举足轻重，而它们都选择 ZooKeeper 作为核心的协调组件。

本节将深入探讨 ZooKeeper 在这两个系统中的具体应用。

## Dubbo 与 ZooKeeper

### Dubbo 的服务治理架构

Dubbo 是一个分布式服务框架，核心功能包括：

| 功能 | 说明 |
|---|---|
| 远程调用 | RPC 通信 |
| 服务注册与发现 | 多注册中心支持 |
| 负载均衡 | 多种策略 |
| 容错机制 | 失败自动切换 |
| 服务治理 | 限流、降级 |

### 服务注册流程

```java
// Dubbo 服务注册简化流程

// 1. 服务提供者启动时
public class ServiceProvider {

    private ZooKeeper zk;

    public void export(String serviceName, String host, int port) throws Exception {
        String interfaceName = serviceName;

        // 创建服务节点
        String providerPath = "/dubbo/" + interfaceName + "/providers/";

        // 注册 URL
        String url = buildProviderUrl(serviceName, host, port);
        String path = zk.create(providerPath + url,
            url.getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.EPHEMERAL);

        System.out.println("服务已注册: " + path);
    }

    private String buildProviderUrl(String serviceName, String host, int port) {
        return String.format(
            "dubbo://%s:%d/%s?category=configurators&timeout=3000",
            host, port, serviceName);
    }
}

// 2. 服务消费者启动时
public class ServiceConsumer {

    private ZooKeeper zk;
    private Map<String, List<String>> serviceCache = new ConcurrentHashMap<>();

    public void subscribe(String serviceName) throws Exception {
        String consumerPath = "/dubbo/" + serviceName + "/consumers/";

        // 注册消费者
        String url = buildConsumerUrl();
        zk.create(consumerPath + url,
            url.getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.EPHEMERAL);

        // 订阅 providers 变化
        subscribeProviders(serviceName);
    }

    public void subscribeProviders(String serviceName) throws Exception {
        String providersPath = "/dubbo/" + serviceName + "/providers";

        // Watch providers 列表变化
        List<String> providers = zk.getChildren(providersPath, event -> {
            if (event.getType() == Event.EventType.NodeChildrenChanged) {
                System.out.println("服务提供者列表变化，刷新缓存");
                refreshProviderCache(serviceName);
            }
        });

        // 更新缓存
        serviceCache.put(serviceName, providers);
    }

    private void refreshProviderCache(String serviceName) {
        try {
            String providersPath = "/dubbo/" + serviceName + "/providers";
            List<String> providers = zk.getChildren(providersPath, true);
            serviceCache.put(serviceName, providers);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### Dubbo 的配置中心

Dubbo 支持动态配置，功能开关存储在 ZooKeeper 中：

```java
// 配置覆盖机制
public class ConfigOverride {

    /**
     * 禁用某个服务
     */
    public void disableService(String serviceName) throws Exception {
        String overridePath = "/dubbo/" + serviceName +
            "/configurators/override://0.0.0.0/" + serviceName +
            "?enabled=false";

        zk.create(overridePath,
            overridePath.getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.PERSISTENT);

        System.out.println("服务已禁用: " + serviceName);
    }

    /**
     * 修改超时时间
     */
    public void updateTimeout(String serviceName, int timeout) throws Exception {
        String overridePath = "/dubbo/" + serviceName +
            "/configurators/override://0.0.0.0/" + serviceName +
            "?timeout=" + timeout;

        zk.create(overridePath,
            overridePath.getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.PERSISTENT);
    }
}
```

### Broker 注册

Kafka Broker 启动时向 ZooKeeper 注册：

```java
// Broker 注册路径
/brokers
├── /ids
│   ├── 0 -> {"jmx_port":-1,"timestamp":"1234567890",
│   │         "port":9092,"host":"kafka1","version":1}
│   ├── 1 -> {"jmx_port":-1,"timestamp":"1234567891",
│   │         "port":9092,"host":"kafka2","version":1}
│   └── 2 -> {"jmx_port":-1,"timestamp":"1234567892",
│             "port":9092,"host":"kafka3","version":1}
├── /seqid
│   └── 0 -> 100
└── /topics
    └── ...
```

```java
// Kafka Broker 注册简化实现

public class KafkaBroker {

    private ZooKeeper zk;
    private int brokerId;
    private String host;
    private int port;

    /**
     * Broker 注册到 ZooKeeper
     */
    public void register() throws Exception {
        String brokerIdPath = "/brokers/ids/" + brokerId;

        // Broker 信息
        Map<String, Object> brokerInfo = new HashMap<>();
        brokerInfo.put("jmx_port", -1);
        brokerInfo.put("timestamp", System.currentTimeMillis());
        brokerInfo.put("port", port);
        brokerInfo.put("host", host);
        brokerInfo.put("version", 1);

        String json = new ObjectMapper().writeValueAsString(brokerInfo);

        // 创建临时节点
        try {
            zk.create(brokerIdPath,
                json.getBytes(),
                ZooDefs.Ids.OPEN_ACL_UNSAFE,
                CreateMode.EPHEMERAL);
            System.out.println("Broker 已注册: " + brokerId);
        } catch (NodeExistsException e) {
            // Broker ID 已存在，可能需要重启或重新分配 ID
            throw new RuntimeException("Broker ID 冲突: " + brokerId);
        }

        // 监听 Broker 列表变化（Controller 关心其他 Broker 的上下线）
        zk.getChildren("/brokers/ids", event -> {
            if (event.getType() == Event.EventType.NodeChildrenChanged) {
                System.out.println("Broker 列表变化，触发 Controller 重选检查");
            }
        });
    }
}
```

### Controller 选举

Kafka 集群中只有一个 Controller，负责管理 partition 的 leader 选举。

```java
// Controller 选举

public class KafkaController {

    private ZooKeeper zk;
    private static final String CONTROLLER_PATH = "/controller";
    private static final String CONTROLLER_EPOCH_PATH = "/controller_epoch";

    /**
     * 尝试成为 Controller
     */
    public boolean tryBecomeController() throws Exception {
        try {
            // 尝试创建临时节点
            long timestamp = System.currentTimeMillis();
            String json = "{\"brokerid\":" + brokerId +
                         ",\"timestamp\":" + timestamp + "}";

            String path = zk.create(CONTROLLER_PATH,
                json.getBytes(),
                ZooDefs.Ids.OPEN_ACL_UNSAFE,
                CreateMode.EPHEMERAL);

            System.out.println("成为 Controller: " + brokerId);

            // 初始化 epoch
            zk.create(CONTROLLER_EPOCH_PATH,
                "1".getBytes(),
                ZooDefs.Ids.OPEN_ACL_UNSAFE,
                CreateMode.PERSISTENT);

            return true;

        } catch (NodeExistsException e) {
            // 已有 Controller
            return false;
        }
    }

    /**
     * 监听 Controller 变化
     */
    public void watchController() throws Exception {
        Watcher watcher = event -> {
            if (event.getType() == Event.EventType.NodeDeleted) {
                System.out.println("Controller 宕机，重新竞争");
                tryBecomeController();
            }
        };

        // 监听节点变化
        byte[] data = zk.getData(CONTROLLER_PATH, watcher, null);

        if (data != null) {
            System.out.println("当前 Controller: " + new String(data));
        }
    }
}
```

### Partition Leader 选举

当 partition 的 leader 副本所在 broker 宕机时，需要选举新的 leader：

```java
// Partition Leader 选举

public class PartitionLeaderElection {

    private ZooKeeper zk;

    /**
     * 副本状态变更时触发 leader 选举
     */
    public void handleStateChange(String topic, int partition,
                                   String newLeader) throws Exception {
        String path = "/brokers/topics/" + topic +
                     "/partitions/" + partition + "/state";

        // 更新 partition 的 leader 状态
        String state = "{\"leader\":" + newLeader +
                      ",\"leader_epoch\":1,\"version\":1}";

        zk.setData(path, state.getBytes(), -1);

        System.out.println("Partition leader 选举完成: " +
                          topic + "-" + partition + " -> " + newLeader);
    }
}
```

### Consumer Group 协调

Kafka 新版（0.9+）使用内置的 group coordinator 管理 Consumer，但在早期版本中依赖 ZooKeeper：

```java
// 旧版 Consumer Group 协调（Kafka 0.8）

public class ConsumerCoordination {

    private ZooKeeper zk;

    /**
     * Consumer 注册到 Group
     */
    public void registerConsumer(String groupId, String consumerId,
                                 String topic) throws Exception {
        // 注册 Consumer
        String consumerPath = "/consumers/" + groupId +
            "/ids/" + consumerId;
        String consumerInfo = "{\"topic\":\"" + topic +
            "\",\"instance_id\":\"" + consumerId + "\"}";

        zk.create(consumerPath, consumerInfo.getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.EPHEMERAL);

        // 注册 Consumer 的 offset
        String offsetPath = "/consumers/" + groupId +
            "/offsets/" + topic + "/" + partition;
        zk.create(offsetPath, "0".getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);

        // 注册到 topic 的 consumer 列表
        String ownersPath = "/consumers/" + groupId +
            "/owners/" + topic + "/" + partition;
        zk.create(ownersPath, consumerId.getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.EPHEMERAL);
    }
}
```

## ZooKeeper 的负担

随着 Kafka 集群规模增大，ZooKeeper 的负担也越来越重：

| ZooKeeper 节点 | 问题 |
|---|---|
| `/brokers` | Broker 注册，变更频繁 |
| `/controller` | Controller 选举，高并发 |
| `/consumers` | Consumer 注册（旧版） |
| `/admin` | 管理操作 |

:::warning
Kafka 正在逐步移除对 ZooKeeper 的依赖，引入 KRaft 模式（Kafka 自己的 Raft 实现）。Kafka 3.6+ 已支持 KRaft，生产可用预计在 Kafka 4.0。
:::

## 总结

ZooKeeper 在 Dubbo 和 Kafka 中的核心作用：

| 系统 | ZooKeeper 用途 |
|---|---|
| **Dubbo** | 服务注册中心、配置中心、动态路由 |
| **Kafka** | Broker 注册、Controller 选举、Partition Leader 选举（旧版） |

两者都充分利用了 ZooKeeper 的：
- **临时节点**：服务/ Broker 的生命周期管理
- **Watch 机制**：实时感知变化
- **顺序节点**：分布式协调

但随着系统规模增长，ZooKeeper 也面临压力。Kafka 的 KRaft 模式、etcd 的 Raft 实现，都是在寻找更轻量的替代方案。
