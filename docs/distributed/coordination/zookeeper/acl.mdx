# ZooKeeper ACL 权限控制

你的团队有 10 个人，其中 3 个人负责运维，5 个人负责业务开发，2 个人是测试。你不能让测试人员修改生产配置，也不能让普通开发人员删除关键节点。

ZooKeeper 的 ACL（Access Control List）机制，就是来解决这个问题的。

## ACL 的基本概念

### ACL 组成

ZooKeeper 的 ACL 由三部分组成：

| 组成 | 说明 | 示例 |
|---|---|---|
| **Scheme** | 授权方式 | `world`、`auth`、`digest`、`ip` |
| **ID** | 授权对象 | `anyone`、`user:password`、`192.168.1.0/24` |
| **Permission** | 权限类型 | `crwda`（增删读写管理） |

### ACL 语法

```java
// 设置 ACL
zk.setACL("/path",
    zooKeeper.get ZooDefs.Ids.CREATOR_ALL_ACL,  // 所有权限
    -1);

// 或使用自定义 ACL
List<ACL> aclList = new ArrayList<>();
ACL acl = new ACL(Perms.ALL, new Id("auth", "user:password"));
aclList.add(acl);
zk.setACL("/path", aclList, -1);
```

### 内置的 Scheme

| Scheme | 说明 | ID 示例 |
|---|---|---|
| `world` | 所有人 | `anyone` |
| `auth` | 已认证用户 | `user:password` |
| `digest` | 用户名:密码（Base64） | `user:password` |
| `ip` | IP 地址或 CIDR | `192.168.1.1` 或 `192.168.1.0/24` |
| `x509` | X.509 证书 | 证书 DN |

## Permission 权限类型

### 五种权限

| 权限 | 字符 | 说明 | 允许的操作 |
|---|---|---|---|
| CREATE | `c` | 创建子节点 | `create("/path/child", data)` |
| READ | `r` | 读取数据和子节点列表 | `getData()`、`getChildren()` |
| WRITE | `w` | 修改数据 | `setData("/path", data)` |
| DELETE | `d` | 删除子节点 | `delete("/path/child")` |
| ADMIN | `a` | 设置 ACL | `setACL()` |

:::tip
注意 `CREATE` 和 `DELETE` 是针对**子节点**的，不是当前节点。
:::

### 权限示例

```java
// 设置只读权限（只能读取）
zk.setACL("/configs/public",
    Collections.singletonList(new ACL(Perms.READ, new Id("world", "anyone"))),
    -1);

// 设置完全控制权限
zk.setACL("/configs/admin",
    Collections.singletonList(new ACL(Perms.ALL, new Id("auth", "admin:password"))),
    -1);
```

## 常用 ACL 策略

### 公开读

```java
// 所有人可读，只有认证用户可写
List<ACL> publicReadACL = new ArrayList<>();
publicReadACL.add(new ACL(Perms.READ, new Id("world", "anyone")));
publicReadACL.add(new ACL(Perms.ALL, new Id("auth", "admin:secret")));

zk.setACL("/public/data", publicReadACL, -1);
```

### 私有数据

```java
// 只有创建者可以访问
zk.setACL("/private/data",
    Collections.singletonList(new ACL(Perms.ALL, new Id("auth", "creator:password"))),
    -1);
```

### IP 白名单

```java
// 只有特定 IP 可访问
List<ACL> ipWhitelist = new ArrayList<>();
ipWhitelist.add(new ACL(Perms.ALL, new Id("ip", "192.168.1.100")));
ipWhitelist.add(new ACL(Perms.READ, new Id("ip", "192.168.1.0/24")));

zk.setACL("/internal/services", ipWhitelist, -1);
```

### 内置 ACL

ZooKeeper 提供了一些预定义的 ACL：

```java
// 公开可读，认证可写
zooKeeper.get ZooDefs.Ids.OPEN_ACL_UNSAFE

// 只有创建者可以访问
ZooDefs.Ids.CREATOR_ALL_ACL

// 匿名访问（仅限读）
ZooDefs.Ids.READ_ACL_UNSAFE
```

```java
// OPEN_ACL_UNSAFE = world:anyone:cdrwa
// 相当于：
List<ACL> openACL = new ArrayList<>();
openACL.add(new ACL(Perms.READ, new Id("world", "anyone")));
openACL.add(new ACL(Perms.CREATE | Perms.DELETE | Perms.READ | Perms.WRITE | Perms.ADMIN,
                    new Id("world", "anyone")));

// CREATOR_ALL_ACL = auth:user:cdrwa
// 相当于：
List<ACL> creatorACL = new ArrayList<>();
creatorACL.add(new ACL(Perms.ALL, new Id("auth", "")));
```

## Digest 认证

Digest 是最常用的认证方式，使用用户名和密码：

### 设置认证信息

```java
// 添加认证信息
zk.addAuthInfo("digest", "admin:secret".getBytes());

// 创建节点（使用认证权限）
zk.create("/protected",
    "data".getBytes(),
    ZooDefs.Ids.CREATOR_ALL_ACL,  // 只有认证用户可访问
    CreateMode.PERSISTENT);
```

### 验证 ACL

```java
// 不同的认证信息访问同一节点
ZooKeeper zk1 = new ZooKeeper("localhost:2181", 3000, watcher);
zk1.addAuthInfo("digest", "admin:secret".getBytes());
byte[] data1 = zk1.getData("/protected", false, null);  // ✓ 成功

ZooKeeper zk2 = new ZooKeeper("localhost:2181", 3000, watcher);
zk2.addAuthInfo("digest", "guest:guest".getBytes());
byte[] data2 = zk2.getData("/protected", false, null);  // ✗ NoAuthException
```

### 密码加密

```java
// ZooKeeper 使用 SHA-1 + Base64 加密密码
// user:password -> Base64(SHA1(user:password))

// 生成密码
public static String generatePassword(String userPassword) {
    try {
        MessageDigest md = MessageDigest.getInstance("SHA-1");
        byte[] digest = md.digest(userPassword.getBytes());
        return Base64.getEncoder().encodeToString(digest);
    } catch (NoSuchAlgorithmException e) {
        throw new RuntimeException(e);
    }
}

// 使用加密后的密码
String encrypted = generatePassword("admin:secret");
// 写入 ACL
new ACL(Perms.ALL, new Id("digest", "admin:" + encrypted));
```

## 多用户 ACL

### 场景：不同角色不同权限

```java
public class MultiUserACL {

    private ZooKeeper zk;

    /**
     * 创建带 ACL 的节点
     */
    public void createWithACL(String path, String data,
                               Map<String, List<ACL>> rolePermissions) throws Exception {
        List<ACL> aclList = new ArrayList<>();

        for (Map.Entry<String, List<ACL>> entry : rolePermissions.entrySet()) {
            String role = entry.getKey();
            List<ACL> permissions = entry.getValue();
            aclList.addAll(permissions);
        }

        zk.create(path, data.getBytes(), aclList, CreateMode.PERSISTENT);
    }

    /**
     * 管理员可读写，运维可读，应用只读
     */
    public void createServiceNode(String serviceName) throws Exception {
        List<ACL> aclList = new ArrayList<>();

        // 管理员：所有权限
        aclList.add(new ACL(Perms.ALL,
            new Id("digest", "admin:" + encrypt("admin:secret"))));

        // 运维：读 + 管理（设置 ACL）
        aclList.add(new ACL(Perms.READ | Perms.ADMIN,
            new Id("digest", "ops:" + encrypt("ops:password"))));

        // 应用：只读
        aclList.add(new ACL(Perms.READ,
            new Id("digest", "app:" + encrypt("app:password"))));

        zk.create("/services/" + serviceName,
            "metadata".getBytes(), aclList, CreateMode.PERSISTENT);
    }
}
```

## Curator 的 ACL 支持

Curator 提供了更便捷的 ACL 操作：

```java
public class CuratorACL {

    private CuratorFramework client;

    public void withACL() throws Exception {
        // 使用特定的 ACL 创建节点
        client.create()
              .withACL(Collections.singletonList(
                  new ACL(Perms.ALL, new Id("auth", "admin:secret"))))
              .forPath("/protected", "data".getBytes());
    }

    public void addACL() throws Exception {
        // 添加额外的 ACL
        client.create()
              .withACL(ZooDefs.Ids.CREATOR_ALL_ACL)
              .forPath("/admin-only", "data".getBytes());

        // 添加更多权限
        client.addACL().forPath("/admin-only",
            Collections.singletonList(
                new ACL(Perms.READ, new Id("ip", "192.168.1.0/24"))));
    }

    /**
     * 使用 ACL 链
     */
    public void aclChain() throws Exception {
        // 设置默认 ACL
        ACL defaultACL = new ACL(Perms.READ, new Id("world", "anyone"));
        ACL adminACL = new ACL(Perms.ALL, new Id("digest", "admin:" + encrypt("admin:secret")));

        client.create()
              .withACL(Arrays.asList(defaultACL, adminACL))
              .forPath("/config", "config".getBytes());
    }
}
```

## 继承与覆盖

### 子节点继承父节点 ACL

默认情况下，子节点会继承父节点的 ACL：

```
/configs (ACL: admin:rw, ops:r, app:r)
    ├── /configs/db (继承父节点 ACL)
    │       ├── /configs/db/host (继承 db 的 ACL)
    └── /configs/redis (继承父节点 ACL)
```

### 覆盖 ACL

```java
// 创建时覆盖 ACL
zk.create("/configs/sensitive",
    "secret".getBytes(),
    Collections.singletonList(
        new ACL(Perms.ALL, new Id("auth", "admin:secret"))),
    CreateMode.PERSISTENT);

// 修改 ACL
zk.setACL("/configs/sensitive",
    Collections.singletonList(
        new ACL(Perms.READ, new Id("world", "anyone"))),
    -1);
```

## 实际应用场景

### 服务注册的安全控制

```java
@Service
public class SecureServiceRegistry {

    private CuratorFramework client;

    /**
     * 注册服务（只有该服务可以修改）
     */
    public void registerService(String serviceName, String instanceId)
            throws Exception {
        // 添加认证
        String auth = serviceName + ":" + instanceId;

        CuratorACL acl = new CuratorACL(
            Collections.singletonList(
                new ACL(Perms.ALL, new Id("auth", auth))),
            auth
        );

        client.create()
              .withACL(acl.getAclList())
              .forPath("/services/" + serviceName + "/" + instanceId,
                       instanceId.getBytes());
    }

    /**
     * 读取服务（公开可读）
     */
    public List<String> getServices(String serviceName) throws Exception {
        return client.getChildren()
                     .forPath("/services/" + serviceName);
    }
}
```

### 配置中心权限

```java
@Service
public class ConfigCenterACL {

    /**
     * 创建配置（管理员可写，所有人可读）
     */
    public void createConfig(String key, String value) throws Exception {
        List<ACL> aclList = new ArrayList<>();

        // 管理员：完全控制
        aclList.add(new ACL(Perms.ALL,
            new Id("digest", "admin:" + encrypt("admin:secret"))));

        // 所有人：只读
        aclList.add(new ACL(Perms.READ, new Id("world", "anyone")));

        client.create()
              .withACL(aclList)
              .forPath("/configs/" + key, value.getBytes());
    }

    /**
     * 只能修改自己创建的配置
     */
    public void createPrivateConfig(String key, String value, String owner)
            throws Exception {
        List<ACL> aclList = new ArrayList<>();

        // 创建者：完全控制
        aclList.add(new ACL(Perms.ALL, new Id("auth", owner)));

        client.create()
              .withACL(aclList)
              .forPath("/private-configs/" + key, value.getBytes());
    }
}
```

## 常见问题

### Q：ACL 支持组吗？

A：ZooKeeper 本身不支持组。可以通过约定命名规范（如 `user:role:admin`）来实现简单的组功能。

### Q：ACL 变更会影响子节点吗？

A：默认不影响。子节点默认继承父节点 ACL，变更父节点 ACL 不会自动更新子节点。需要单独修改。

### Q：忘记管理员密码怎么办？

A：只能通过重新启动 ZooKeeper 并重置 ACL。这是一个运维风险，建议使用 ip 白名单作为备用认证。

## 总结

ZooKeeper 的 ACL 机制提供了精细化的权限控制：

1. **多种 Scheme**：world、auth、digest、ip 满足不同场景
2. **五种权限**：CREATE、READ、WRITE、DELETE、ADMIN
3. **继承机制**：子节点默认继承父节点 ACL
4. **Digest 认证**：最常用的安全认证方式

合理使用 ACL，可以有效保护 ZooKeeper 中的敏感数据和配置。
