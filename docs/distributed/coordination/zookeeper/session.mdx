# ZooKeeper 会话机制

你正在使用一个在线协作工具，突然网络断开了。当你重新连接后，你惊讶地发现——你的编辑内容还在，协作状态没有丢失。这就是会话（Session）的力量。

ZooKeeper 的会话机制与此类似：客户端与服务端之间建立持久的会话，即使连接短暂中断，只要在超时时间内重连成功，之前创建的所有数据都不会丢失。

## 什么是会话

会话（Session）是 ZooKeeper 客户端与服务端之间的逻辑连接。每个 ZooKeeper 客户端在连接服务端时都会创建一个会话，会话有一个超时时间（SessionTimeout）。

### 会话的生命周期

```
                    ┌─────────────────────────────────────┐
                    │                                     │
   CONNECTING ──────┼──▶ CONNECTED ───────────────────────┼────▶ CLOSED
                    │         │                           │
                    │         │  断开连接                  │
                    │         ▼                           │
                    │    DISCONNECTED                      │
                    │         │                           │
                    │         │  重连成功（在超时时间内）    │
                    │         ▼                           │
                    │    RECONNECTING                     │
                    │         │                           │
                    │         │  重连失败（超时）          │
                    │         ▼                           │
                    │      EXPIRED                        │
                    │                                     │
                    └─────────────────────────────────────┘
```

### 会话状态

| 状态 | 说明 |
|---|---|
| `CONNECTING` | 正在连接 |
| `CONNECTED` | 已连接 |
| `RECONNECTING` | 正在重连 |
| `DISCONNECTED` | 已断开 |
| `CLOSED` | 会话已关闭 |

## 会话超时

### 超时时间设置

```java
// 创建 ZooKeeper 客户端
ZooKeeper zk = new ZooKeeper(
    "localhost:2181",           // 连接地址
    30000,                       // Session 超时时间（毫秒）
    watcher                      // 连接 watcher
);
```

### 超时时间的意义

```
Session 超时 = 30 秒

客户端                            服务端
   │                                │
   │── CONNECT ──────────────────▶│
   │◀── CONNECTED ─────────────────│
   │                                │
   │   网络断开                     │
   │── PING ──▶ ✗                  │
   │                                │
   │   服务端开始计时                │
   │   30 秒内未收到心跳             │
   │                                │── 删除临时节点
   │◀── SESSION_EXPIRED ───────────│
   │                                │
```

### 超时时间的选择

| 因素 | 建议 |
|---|---|
| 网络质量好 | 可以设置较短的超时（如 10-30 秒） |
| 网络不稳定 | 设置较长的超时（如 60 秒以上） |
| 临时节点重要 | 超时不要太短，给重连留有时间 |
| 服务发现 | 超时短可以更快发现服务下线 |

:::tip
ZooKeeper 的建议超时时间范围是 2 * tickTime 到 20 * tickTime。tickTime 默认是 2000 毫秒，所以建议范围是 4 秒到 40 秒。
:::

## 心跳机制

### PING

ZooKeeper 使用 PING 来检测客户端是否存活。客户端定期向服务端发送 PING，服务端在超时时间内没收到 PING 就认为客户端已失效。

```java
// ZooKeeper 客户端源码简化
public class ZooKeeper {
    private final SendThread sendThread;

    // 定时发送 PING
    void startPinger() {
        while (state == States.CONNECTED) {
            try {
                Thread.sleep(minSessionTimeout / 3);
                sendPing();  // 发送 PING
            } catch (InterruptedException e) {
                // 处理
            }
        }
    }

    void sendPing() {
        // 发送 PING 请求
        queuePacket(new PingRequest());
    }
}
```

### 服务端的超时检测

```java
public class ZooKeeperServer {
    // Session 超时管理器
    private SessionTrackerImpl sessionTracker;

    // 处理心跳
    public void processPulse() {
        sessionTracker.dumpSessions();
    }
}

public class SessionTrackerImpl {
    // 会话超时检查
    public void run() {
        while (running) {
            long currentTime = System.currentTimeMillis();

            // 检查所有会话
            for (Session session : sessions.values()) {
                if (currentTime - session.getLastHeard() > session.getTimeout()) {
                    // 会话超时
                    expireSession(session.getSessionId());
                }
            }

            Thread.sleep(tickTime);
        }
    }
}
```

## 临时节点与会话

### 临时节点的特性

临时节点与会话绑定：当会话关闭（正常或超时），服务端会自动删除该会话创建的所有临时节点。

```java
public class EphemeralNodeDemo {

    private ZooKeeper zk;

    public void createEphemeralNode() throws Exception {
        // 创建临时节点
        String path = zk.create("/services/worker1",
            "alive".getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.EPHEMERAL);

        System.out.println("临时节点创建: " + path);

        // 当会话断开（关闭或超时），节点自动删除
    }

    public void closeSession() throws Exception {
        // 正常关闭会话
        zk.close();
        // 临时节点被删除
    }
}
```

### 应用：服务注册

```java
@Service
public class ServiceRegistry {

    private ZooKeeper zk;

    /**
     * 注册服务（临时节点）
     */
    public void register(String serviceName, String host, int port) throws Exception {
        // 创建临时节点，服务端实例标识
        String instancePath = zk.create(
            "/services/" + serviceName + "/" + host + ":" + port,
            "alive".getBytes(),
            ZooDefs.Ids.OPEN_ACL_UNSAFE,
            CreateMode.EPHEMERAL);

        System.out.println("服务注册成功: " + instancePath);

        // 当实例崩溃或网络中断，会话超时
        // ZooKeeper 自动删除临时节点
        // 其他服务感知到服务下线
    }

    /**
     * 监听服务变化
     */
    public void watchServices(String serviceName) throws Exception {
        String path = "/services/" + serviceName;

        // 监听子节点变化
        List<String> services = zk.getChildren(path, event -> {
            if (event.getType() == Event.EventType.NodeChildrenChanged) {
                System.out.println("服务列表变化，需要重新获取");
                try {
                    List<String> newServices = zk.getChildren(path, true);
                    System.out.println("当前服务: " + newServices);
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }
        });

        System.out.println("当前服务列表: " + services);
    }
}
```

## 会话重连

### 重连流程

```java
public class ReconnectionDemo {

    private ZooKeeper zk;
    private CountDownLatch reconnectLatch = new CountDownLatch(1);

    public void connect() throws Exception {
        Watcher watcher = event -> {
            switch (event.getState()) {
                case SyncConnected:
                    System.out.println("连接成功");
                    reconnectLatch.countDown();
                    break;
                case Disconnected:
                    System.out.println("连接断开");
                    break;
                case Expired:
                    System.out.println("会话过期，需要重新创建");
                    reconnect();
                    break;
            }
        };

        zk = new ZooKeeper("localhost:2181", 30000, watcher);

        // 等待连接成功
        reconnectLatch.await();
    }

    public void reconnect() throws Exception {
        // 关闭旧会话
        if (zk != null) {
            zk.close();
        }

        // 创建新会话
        zk = new ZooKeeper("localhost:2181", 30000, watcher);
        reconnectLatch = new CountDownLatch(1);
        reconnectLatch.await();
    }
}
```

### 重连时的状态保持

```
会话重连成功前：                        会话重连成功后：
                                        │
客户端：CLOSED 会话                     客户端：原会话恢复
客户端：重新创建新会话                   客户端：临时节点恢复
        │                                    │
        ▼                                    ▼
服务端：原会话标记为 DISCONNECTED      服务端：会话状态恢复
服务端：临时节点保留（未过期）           服务端：临时节点保持
                                        │
                                        ▼
                              之前创建的所有数据都在
                              （包括临时节点）
```

## Curator 的会话监听

Curator 提供了更易用的会话监听 API：

```java
public class CuratorSessionDemo {

    public void watchSession(CuratorFramework client) throws Exception {
        // 使用 ConnectionStateListener 监听会话状态
        client.getConnectionStateListenable().addListener((c, newState) -> {
            switch (newState) {
                case CONNECTED:
                    System.out.println("连接成功");
                    break;
                case SUSPENDED:
                    System.out.println("连接暂停（网络中断）");
                    // 暂停业务操作，等待重连
                    break;
                case RECONNECTED:
                    System.out.println("重连成功");
                    // 恢复业务操作
                    break;
                case LOST:
                    System.out.println("会话丢失");
                    // 需要重新初始化
                    break;
                case READ_ONLY:
                    System.out.println("只读模式");
                    break;
            }
        });
    }

    /**
     * 等待连接成功
     */
    public void blockUntilConnected(CuratorFramework client)
            throws InterruptedException {
        client.blockUntilConnected(30, TimeUnit.SECONDS);
    }
}
```

## 常见问题

### Q：会话超时后，临时节点会立即删除吗？

A：是的。服务端检测到会话超时会立即删除该会话创建的所有临时节点。这个过程是异步的，但很快（毫秒级）。

### Q：网络抖动导致短暂断开，会话会过期吗？

A：只要在 SessionTimeout 时间内重新连接成功，会话就不会过期。ZooKeeper 使用的是「宽松」的超时机制，给了网络抖动一定的容忍度。

### Q：多个 ZooKeeper 服务器如何保证会话一致？

A：会话信息存储在所有服务器上（通过 Zab 协议同步）。Leader 服务器负责会话超时检测和临时节点清理。

## 总结

ZooKeeper 的会话机制是分布式协调的基础：

1. **会话生命周期**：CONNECTING → CONNECTED → DISCONNECTED → RECONNECTING/EXPIRED
2. **超时机制**：心跳检测 + SessionTimeout 保证可靠性
3. **临时节点绑定**：会话关闭时自动清理
4. **重连恢复**：超时时间内重连，会话状态可恢复

理解会话机制后，你可以更好地设计高可用的 ZooKeeper 客户端应用。
