# Nacos 简介

你的微服务架构越来越复杂：几十个服务注册在不同的机器上，配置散落在各处，每次发布都要手动更新配置。能不能有一个平台，把服务注册、配置管理都统一管起来？

Nacos 就是来解决这个问题的。

## Nacos 是什么

Nacos（Naming and Configuration Service）是阿里巴巴开源的服务发现与配置管理平台。它提供了一整套解决方案：

| 核心功能 | 说明 |
|---|---|
| **服务注册与发现** | 注册中心，支持临时/持久实例 |
| **配置管理** | 配置中心，支持热更新和多环境 |
| **服务管理** | 实例管理、健康检查 |
| **元数据管理** | 命名空间、分组、标签 |

## 典型应用场景

### 场景一：微服务注册发现

```yaml
# application.yml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
        namespace: dev
        group: order-system
```

```java
// 服务提供者
@SpringBootApplication
@EnableDiscoveryClient
public class OrderServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(OrderServiceApplication.class, args);
    }
}

// 服务消费者
@RestController
public class OrderController {

    @Autowired
    private RestTemplate restTemplate;

    @Autowired
    private DiscoveryClient discoveryClient;

    public String callInventory() {
        // 获取服务实例
        List<ServiceInstance> instances =
            discoveryClient.getInstances("inventory-service");

        // 负载均衡选择
        ServiceInstance selected = instances.get(0);

        // 调用服务
        return restTemplate.getForObject(
            "http://" + selected.getHost() + ":" + selected.getPort() + "/inventory",
            String.class
        );
    }
}
```

### 场景二：配置中心

```yaml
# application.yml
spring:
  cloud:
    nacos:
      config:
        server-addr: localhost:8848
        file-extension: yaml
        namespace: dev
        group: order-system
        refresh-enabled: true
```

```java
// 配置监听
@RestController
@RefreshScope  // 支持热更新
public class ConfigController {

    @Value("${app.config:default}")
    private String config;

    @GetMapping("/config")
    public String getConfig() {
        return config;
    }
}
```

## 总结

Nacos 是国产优秀的服务发现与配置管理平台：

1. **开箱即用**：服务注册、配置管理一步到位
2. **中文文档**：国内团队维护，文档友好
3. **多环境支持**：命名空间、分组满足复杂场景
4. **双协议支持**：CP + AP，可灵活切换

如果你的项目在国内，Nacos 是服务治理的不错选择。
