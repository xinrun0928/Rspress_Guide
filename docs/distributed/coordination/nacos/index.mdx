# Nacos：Spring Cloud 的注册中心 + 配置中心

凌晨 2 点，你在发布新版本。

灰度发布策略：先让 10% 的流量走新版本。但等等——流量怎么分配？负载均衡器怎么知道哪个实例是哪个版本？

**答案是：服务注册与发现。**

Nacos 是阿里巴巴开源的分布式服务发现和配置管理平台，同时支持注册中心和配置中心。相比 Eureka（已停止维护）、Spring Cloud Config（功能单一），Nacos 是 Spring Cloud 生态下最完整的一站式解决方案。

## Nacos 双模式

```
┌─────────────────────────────────────────────────────────┐
│                   Nacos 双模式架构                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────────────┐    ┌──────────────────┐         │
│  │    注册中心        │    │    配置中心        │         │
│  ├──────────────────┤    ├──────────────────┤         │
│  │ 服务注册 / 注销   │    │ 配置发布 / 获取    │         │
│  │ 服务发现          │    │ 配置监听 / 推送    │         │
│  │ 健康检查          │    │ 版本管理           │         │
│  │ 权重 / 标签分组   │    │ 命名空间隔离       │         │
│  └──────────────────┘    └──────────────────┘         │
│                                                         │
│  统一入口，统一管理，减少运维复杂度                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## AP + CP 双模式

Nacos 同时支持 AP（可用性）和 CP（一致性）两种模式：

```
临时实例（ephemeral）：使用 AP 模式，适合普通微服务
  └─ 基于 Distro 协议，注册后心跳保活，不存在则自动剔除

持久实例（persistent）：使用 CP 模式，适合数据类服务
  └─ 基于 Raft 协议，数据强一致，不存在自动删除
```

## 适用场景

| 场景 | 推荐模式 | 说明 |
|------|---------|------|
| 微服务注册发现 | AP（临时实例） | 高可用，快速响应实例变化 |
| 配置中心 | CP（持久实例） | 配置一致性，避免配置漂移 |
| 灰度发布 | AP + 标签分组 | 按标签路由流量 |
| 异地多机房 | 持久实例 | 跨机房数据一致性 |

## 核心公式

```
Nacos = 注册中心（AP/CP）+ 配置中心 + 命名空间隔离

选型理由：
- Spring Cloud 全家桶 → Nacos（开箱即用）
- 需要配置中心 → Nacos（注册+配置二合一）
- 需要 CP 一致性 → 持久实例 + Raft
- 需要 AP 高可用 → 临时实例 + Distro
```
