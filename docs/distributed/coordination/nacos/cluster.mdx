# Nacos 集群模式

单机版的 Nacos 只能用于开发和测试环境。在生产环境中，我们需要部署 Nacos 集群来保证高可用。

本节将详细介绍如何部署和管理 Nacos 集群。

## 集群架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Nacos 集群架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│                      VIP / SLB                              │
│                         │                                   │
│       ┌─────────────────┼─────────────────┐                 │
│       ▼                 ▼                 ▼                 │
│  ┌─────────┐        ┌─────────┐        ┌─────────┐         │
│  │ Nacos 1 │◀──────▶│ Nacos 2 │◀──────▶│ Nacos 3 │         │
│  │(Leader) │        │(Follower)│       │(Follower)│        │
│  └────┬────┘        └────┬────┘        └────┬────┘         │
│       │                   │                   │              │
│       └───────────────────┴───────────────────┘              │
│                      Raft 协议                               │
│                                                              │
│  ┌───────────────────────────────────────────────────────┐  │
│  │                  MySQL 集群                            │  │
│  │         (主从复制，数据持久化)                          │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 集群部署

### 环境要求

| 组件 | 要求 |
|---|---|
| CPU | 4 核+ |
| 内存 | 8GB+ |
| 磁盘 | SSD 100GB+ |
| MySQL | 5.7+ / 8.0+ |
| JDK | 1.8+ |

### MySQL 配置

```sql
-- 创建数据库
CREATE DATABASE nacos_config CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 创建用户
CREATE USER 'nacos'@'%' IDENTIFIED BY 'nacos_password';
GRANT ALL PRIVILEGES ON nacos_config.* TO 'nacos'@'%';

-- 初始化数据库
mysql -u nacos -p nacos_password nacos_config < nacos-mysql.sql
```

### 集群配置文件

```properties
# cluster.conf
# 集群节点列表
192.168.1.10:8848
192.168.1.11:8848
192.168.1.12:8848
```

### application.properties

```properties
# 服务端口
server.port=8848

# MySQL 数据源
spring.datasource.platform=mysql
db.num=1
db.url.0=jdbc:mysql://mysql-master:3306/nacos_config?characterEncoding=utf8&connectTimeout=1000&socketTimeout=3000&autoReconnect=true&useUnicode=true&useSSL=false&serverTimezone=UTC
db.user=nacos
db.password=nacos_password

# 集群配置
nacos.inetutils.prefer-hostname-over-ip=false
nacos.inetutils.ip-address=192.168.1.10

# 认证配置（生产环境必须开启）
nacos.core.auth.enabled=true
nacos.core.auth.server.identity.key=nacos-identity-key
nacos.core.auth.server.identity.value=nacos-identity-value
nacos.core.auth.plugin.nacos.token.cache.enable=true
nacos.core.auth.token.expire.seconds=18000
nacos.core.auth.plugin.nacos.token.secret.key=NacosServerIdentityKey

# 监控配置
management.endpoints.web.exposure.include=*
management.metrics.export.elastic.enabled=false
management.metrics.export.influx.enabled=false
```

### 启动集群

```bash
# 在每个节点上执行
./startup.sh

# 查看集群状态
curl "http://localhost:8848/nacos/v1/ops/cluster/leader/health"
```

### Leader 切换

```
Nacos Leader 故障时：

1. Follower 检测到 Leader 超时
2. 发起 Leader 选举
3. 多数派投票
4. 选出新 Leader
5. 通知客户端（通过 gRPC）

整个过程通常在秒级完成
```

## 常见问题

### Q：集群节点数必须是奇数吗？

A：是的。Raft 协议需要多数派投票，偶数节点不会提高容错能力。

| 节点数 | 容错能力 | 说明 |
|---|---|---|
| 1 | 0 | 不推荐 |
| 3 | 1 | 最小集群 |
| 5 | 2 | 推荐生产使用 |
| 7 | 3 | 大规模集群 |

### Q：如何添加新节点？

A：

```bash
# 1. 在新节点上安装 Nacos
# 2. 配置 cluster.conf
192.168.1.10:8848
192.168.1.11:8848
192.168.1.12:8848
192.168.1.13:8848  # 新节点

# 3. 启动新节点
./startup.sh

# 4. 验证集群状态
curl "http://localhost:8848/nacos/v1/ops/cluster/nodes"
```

### Q：如何处理脑裂？

A：Nacos 使用 Raft 协议的多数派机制避免脑裂：

- 只有获得多数派投票的节点才能成为 Leader
- 网络分区后，只有多数派一侧可以工作
- 网络恢复后，少数派一侧会自动同步新 Leader 的数据

## 总结

Nacos 集群模式的核心要点：

1. **Raft 一致性**：通过 Raft 协议保证数据一致性
2. **MySQL 持久化**：配置数据持久化到 MySQL
3. **高可用**：节点故障自动切换
4. **水平扩展**：可以增加节点提升容量
5. **监控运维**：提供丰富的运维 API 和监控指标
