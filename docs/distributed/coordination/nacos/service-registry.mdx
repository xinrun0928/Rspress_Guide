# Nacos 服务注册与发现

你的微服务架构有几十个服务，每个服务可能有多个实例。当新增一个服务实例时，如何让其他服务感知到？当一个实例崩溃时，如何快速将流量切换到健康的实例？

这就是服务注册与发现要解决的问题。

## 服务注册原理

### 注册流程

```

│                    服务注册流程                               │

│  │  :8080       │                                            │

│         │                                                     │
│         │ POST /nacos/v1/ns/instance                        │
│         │                                                   │
│         │ {                                                 │
│         │   "serviceName": "order-service",                │
│         │   "ip": "192.168.1.10",                          │
```


### 服务提供者配置

```yaml
# application.yml
spring:
  application:
    name: order-service

  cloud:
    nacos:
      discovery:
        # Nacos 服务地址
        server-addr: nacos-server:8848
        # 命名空间（用于环境隔离）
        namespace: dev
        # 分组（用于业务隔离）
        group: order-system
        # 集群名称
        cluster-name: DEFAULT
        # 实例权重
        weight: 1.0
        # 是否开启实例元数据
        instance-enabled: true
        # 元数据
        metadata:
          version: v1
          environment: prod

# 健康检查配置
management:
  endpoints:
    web:
      exposure:
        include: health,info
  endpoint:
    health:
      show-details: always
```

### 服务消费者配置

```yaml
# application.yml
spring:
  application:
    name: user-consumer
  cloud:
    nacos:
```

```java
// 使用 DiscoveryClient 获取服务实例
@RestController
public class UserController {

    @Autowired
    private DiscoveryClient discoveryClient;

    @Autowired
    private RestTemplate restTemplate;

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    @GetMapping("/order/{id}")
    public String getOrder(@PathVariable Long id) {
        // 获取服务实例列表
        List<ServiceInstance> instances =
            discoveryClient.getInstances("order-service");

        if (instances.isEmpty()) {
            return "服务不可用";
        }

        // 简单负载均衡：选择第一个
        ServiceInstance instance = instances.get(0);

        // 调用服务
        String url = String.format("http://%s:%d/orders/%d",
            instance.getHost(),
            instance.getPort(),
            id);

        return restTemplate.getForObject(url, String.class);
    }
}
```

### 订阅服务变化

```java
// 使用 Nacos Watcher 订阅服务变化
@Autowired
private NamingService namingService;

@PostConstruct
public void subscribeService() throws NacosException {
    namingService.subscribe("order-service", "DEFAULT_GROUP", event -> {
        if (event instanceof NamingEvent) {
            NamingEvent namingEvent = (NamingEvent) event;
            List<Instance> instances = namingEvent.getInstances();

            System.out.println("服务实例列表变化:");
            for (Instance instance : instances) {
                if (instance.isHealthy()) {
                    System.out.println("- " + instance.getIp() + ":" + instance.getPort());
                }
            }
        }
    });
}
```

## 负载均衡

### 内置负载均衡策略

| 策略 | 说明 |
|---|---|
| `权重` | 根据权重比例分配流量 |
| `随机` | 随机选择实例 |
| `轮询` | 按顺序轮流选择 |
| `一致性哈希` | 相同请求哈希到同一实例 |
| `最少连接` | 选择连接数最少的实例 |

### 配置 Ribbon 负载均衡

```yaml
# 方式一：配置文件
spring:
  cloud:
    nacos:
      discovery:
        server-addr: nacos-server:8848

# order-service 的负载均衡配置
order-service:
  ribbon:
    NFLoadBalancerRuleClassName: com.alibaba.nacos.client.ribbon.NacosRule
    # 或使用其他策略
    # NFLoadBalancerRuleClassName: com.netflix.loadbalancer.RandomRule
    # NFLoadBalancerRuleClassName: com.netflix.loadbalancer.RoundRobinRule

# 方式二：代码配置
@Configuration
public class RibbonConfig {

    @Bean
    public IRule ribbonRule() {
        // 使用 Nacos 的权重负载均衡
        return new NacosRule();
        // 或使用随机策略
        // return new RandomRule();
        // 或使用重试策略
        // return new RetryRule(new RoundRobinRule(), 3);
    }
}
```

### 自定义负载均衡

```java
// 自定义负载均衡器
@Component
public class CustomLoadBalancer implements ServiceInstanceListSupplier {

    @Autowired
    private DiscoveryClient discoveryClient;

    @Override
    public String getServiceId() {
        return "order-service";
    }

    @Override
    public Flux<List<ServiceInstance>> get() {
        return Flux.fromIterable(() -> {
            List<ServiceInstance> instances = discoveryClient.getInstances(getServiceId());
            // 只返回健康实例
            return instances.stream()
                .filter(ServiceInstance::isHealthy)
                .collect(Collectors.toList());
        });
    }
}
```

## 健康检查

### 心跳机制

```
┌─────────────────────────────────────────────────────────────┐
│                    心跳机制                                 │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  服务实例                                                    │
│     │                                                        │
│     │ 每 5 秒发送心跳                                        │
│     │ ──────────────────────────────────────────────────▶  │
│     │                                                        │
│  ┌────────────────────────────────────────────────────────┐ │
│  │                   Nacos Server                          │ │
│  │                                                         │ │
│  │  15 秒内未收到心跳 → 标记为不健康                        │ │
│  │  30 秒内未收到心跳 → 移除实例                            │ │
│  │                                                         │ │
│  └────────────────────────────────────────────────────────┘ │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 心跳配置

```yaml
spring:
  cloud:
    nacos:
      discovery:
        # 心跳间隔（毫秒）
        heart-beat-interval: 5000
        # 心跳超时（毫秒）
        heart-beat-timeout: 15000
        # 实例删除时间（毫秒）
        ip-delete-timeout: 30000
```

### 健康检查类型

| 类型 | 说明 | 配置 |
|---|---|---|
| TCP | 检测端口是否可达 | 客户端上报 |
| HTTP | 检测 HTTP 端点 | 客户端上报 |
| MySQL | 检测 MySQL 连接 | 服务端探测 |
| 自定义 | 自定义健康检查逻辑 | 实现 HealthChecker |

## 常见问题

### Q：服务注册后发现实例不可用？

A：检查以下几点：
1. 服务是否正常启动
2. 网络是否通畅
3. 健康检查端点是否返回正确状态
4. Nacos 服务地址是否配置正确

### Q：如何实现灰度发布？

A：使用 Nacos 的元数据和权重功能：

```yaml
# 灰度实例（10% 流量）
spring:
  cloud:
    nacos:
      discovery:
        metadata:
          version: gray
        weight: 0.1

# 正式实例（90% 流量）
spring:
  cloud:
    nacos:
      discovery:
        metadata:
          version: release
        weight: 0.9
```

## 总结

Nacos 服务注册与发现的核心要点：

1. **简单易用**：Spring Cloud 注解自动注册
2. **多协议支持**：HTTP 和 gRPC 双重通信
3. **健康检查**：客户端心跳 + 服务端探测
4. **灵活隔离**：命名空间 + 分组实现多环境
5. **负载均衡**：多种策略可选
