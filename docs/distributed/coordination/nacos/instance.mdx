# Nacos 临时实例与持久实例

|Nacos| |临时实例| |持久实例|

你的系统中有些服务需要长期运行（如数据库、消息队列），有些服务可能频繁启停（如无状态的应用服务）。这两种场景需要不同的服务注册策略。

Nacos 通过**临时实例**和**持久实例**来满足这两种需求。

## 两种实例类型对比

```
┌─────────────────────────────────────────────────────────────┐
│                  临时实例 vs 持久实例                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  临时实例 (Ephemeral)                                        │
│  ─────────────────────────────────────────                   │
│  • 客户端心跳维持                                            │
│  • 停止发送心跳自动删除                                      │
│  • 无状态应用首选                                            │
│  • AP 模式                                                  │
│                                                              │
│  持久实例 (Persistent)                                      │
│  ─────────────────────────────────────────                   │
│  • 服务端持久存储                                            │
│  • 需要手动注销                                              │
│  • 有状态服务                                                │
│  • CP 模式                                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 核心区别

| 维度 | 临时实例 | 持久实例 |
|---|---|---|
| **存储方式** | 内存 | MySQL |
| **注册方式** | 心跳维持 | 持久化存储 |
| **删除方式** | 心跳停止自动删除 | 手动注销 |
| **一致性协议** | Distro (AP) | Raft (CP) |
| **适用场景** | 无状态微服务 | 中间件、数据库 |
| **数据可靠性** | 依赖心跳 | 持久化存储 |

## 临时实例

### 使用场景

```
适合临时实例的服务：
• Spring Cloud 微服务
• 无状态的 HTTP/gRPC 服务
• 短期运行的任务
• 定时任务节点
```

### 注册临时实例

```yaml
# 默认注册为临时实例
spring:
  cloud:
    nacos:
      discovery:
        # ephemeral = true 表示临时实例（默认）
        ephemeral: true
        # 心跳间隔（毫秒）
        heart-beat-interval: 5000
        # 心跳超时（毫秒）
        heart-beat-timeout: 15000
        # 实例删除时间（毫秒）
        ip-delete-timeout: 30000
```

```java
// 手动注册临时实例
@Service
public class InstanceRegistry {

    @Autowired
    private NacosServiceManager nacosServiceManager;

    public void registerInstance(String serviceName, String ip, int port) {
        try {
            NamingService namingService =
                nacosServiceManager.getNamingService();

            Instance instance = new Instance();
            instance.setIp(ip);
            instance.setPort(port);
            instance.setInstanceId(IdUtils.generateInstanceId(namingService.getServicesOfServer(1, 1000, serviceName).size()));
            instance.setWeight(1.0);
            instance.setEnabled(true);
            instance.setHealthy(true);
            instance.setEphemeral(true);  // 临时实例

            namingService.registerInstance(serviceName, instance);

        } catch (NacosException e) {
            e.printStackTrace();
        }
    }
}
```

### 心跳机制

```
心跳流程：

客户端 ───────────────────────────────▶ Nacos Server
                                              │
│  每 5 秒发送心跳                               │
│  content-type: 80                            │
│                                              │
│  ┌──────────────────────────────────────┐   │
│  │  15 秒内未收到心跳 → 标记为不健康     │   │
│  │  30 秒内未收到心跳 → 自动删除         │   │
│  └──────────────────────────────────────┘   │
│                                              │
◀─────────────── 响应 ─────────────────────────
│  code: 0
│  message: success
```

### 心跳配置

```yaml
spring:
  cloud:
    nacos:
      discovery:
        # 心跳间隔（默认 5000ms）
        heart-beat-interval: 5000

        # 心跳超时（默认 15000ms）
        # 超过这个时间未收到心跳，标记为不健康
        heart-beat-timeout: 15000

        # IP 删除超时（默认 30000ms）
        # 超过这个时间未收到心跳，删除实例
        ip-delete-timeout: 30000
```

## 持久实例

### 使用场景

```
适合持久实例的服务：
• MySQL、Redis 等数据库
• Kafka、RocketMQ 等消息队列
• ZooKeeper、etcd 等协调服务
• 定时任务调度器（如 XXL-JOB）
• 需要固定 IP 的服务
```

### 注册持久实例

```yaml
# 注册为持久实例
spring:
  cloud:
    nacos:
      discovery:
        # ephemeral = false 表示持久实例
        ephemeral: false
```

```java
// 手动注册持久实例
@Service
public class PersistentInstanceRegistry {

    @Autowired
    private NacosServiceManager nacosServiceManager;

    public void registerPersistentInstance(String serviceName, String ip, int port) {
        try {
            NamingService namingService =
                nacosServiceManager.getNamingService();

            Instance instance = new Instance();
            instance.setIp(ip);
            instance.setPort(port);
            instance.setWeight(1.0);
            instance.setEnabled(true);
            instance.setHealthy(true);
            instance.setEphemeral(false);  // 持久实例

            namingService.registerInstance(serviceName, instance);

        } catch (NacosException e) {
            e.printStackTrace();
        }
    }

    // 手动注销持久实例
    public void deregisterInstance(String serviceName, String ip, int port) {
        try {
            NamingService namingService =
                nacosServiceManager.getNamingService();

            namingService.deregisterInstance(serviceName, ip, port);

        } catch (NacosException e) {
            e.printStackTrace();
        }
    }
}
```

### 持久实例的元数据

```yaml
spring:
  cloud:
    nacos:
      discovery:
        ephemeral: false
        # 持久实例的元数据
        metadata:
          role: database
          version: v1
          cluster: master
```

## 混合使用

### 同一服务不同实例类型

```java
// 场景：数据库集群
// 主库 - 持久实例
@Service
public class MasterDbRegistry {

    @PostConstruct
    public void registerMaster() {
        Instance instance = new Instance();
        instance.setIp(masterHost);
        instance.setPort(masterPort);
        instance.setEphemeral(false);  // 持久实例
        instance.setMetadata(Map.of("role", "master"));

        namingService.registerInstance("mysql-cluster", instance);
    }
}

// 从库 - 临时实例
@Service
public class SlaveDbRegistry {

    @Autowired
    private NacosServiceManager nacosServiceManager;

    @Autowired
    private NacosDiscoveryProperties discoveryProperties;

    @PostConstruct
    public void registerSlaves() {
        InstanceProperties properties = discoveryProperties.getInstance();
        String instanceId = discoveryProperties.getInstanceId();

        // 从库作为临时实例注册
        namingService.registerInstance("mysql-cluster", instanceId,
            properties.getPort(), properties.getMetadata());
    }
}
```

### 按环境区分

```yaml
# 开发环境 - 临时实例（频繁启停）
spring:
  cloud:
    nacos:
      discovery:
        ephemeral: true

# 生产环境 - 持久实例（需要高可靠）
spring:
  cloud:
    nacos:
      discovery:
        ephemeral: false
```

## 一致性协议选择

### Distro 协议（临时实例）

```java
// Distro 协议特点
public class DistroProtocol {
    // 1. 数据分片存储
    // 每个节点负责一部分数据

    // 2. 异步同步
    // 数据变更异步同步到其他节点

    // 3. 最终一致
    // 不保证强一致，但保证高可用
}
```

### Raft 协议（持久实例）

```java

    // 1. Leader 处理写请求
    // 所有写操作经过 Leader

    // 2. 日志复制
    // 写入前需要多数派确认

    // 3. 强一致
    // 读取可以走 Leader 保证一致
}
```

### 持久实例健康检查

```java
// 持久实例使用服务端探测
public class HealthCheckForPersistent {

    // 服务端 TCP 健康检查
    public boolean tcpHealthCheck(String ip, int port) {
        try (Socket socket = new Socket()) {
            socket.connect(new InetSocketAddress(ip, port), 3000);
            return true;
        } catch (IOException e) {
            return false;
        }
    }

    // 服务端 HTTP 健康检查
    public boolean httpHealthCheck(String ip, int port, String healthPath) {
        try {
            URL url = new URL("http://" + ip + ":" + port + healthPath);
            HttpURLConnection conn = (HttpURLConnection) url.openConnection();
            conn.setRequestMethod("GET");
            conn.setConnectTimeout(3000);
            return conn.getResponseCode() == 200;
        } catch (Exception e) {
            return false;
        }
    }
}
```

## 常见问题

### Q：临时实例宕机后会立即被删除吗？

A：不会。心跳超时后，会先标记为不健康（15 秒），再删除（30 秒）。

```
时间线：
T0:     实例正常
T0+5s:  发送心跳
T0+10s: 发送心跳
...（每 5 秒）
T0+15s: 未收到心跳，标记为不健康
T0+20s: 未收到心跳，继续等待
...（每 5 秒）
T0+30s: 未收到心跳，删除实例
```

### Q：可以混合使用两种实例类型吗？

A：可以。Nacos 支持同一服务的不同实例使用不同类型：

```yaml
# 同一服务不同实例
# 实例 A - 持久实例
spring:
  cloud:
    nacos:
      discovery:
        ephemeral: false

# 实例 B - 临时实例
spring:
  cloud:
    nacos:
      discovery:
        ephemeral: true
```

### Q：如何选择实例类型？

A：

| 场景 | 推荐类型 | 原因 |
|---|---|---|
| Spring Cloud 微服务 | 临时实例 | 频繁启停，自动清理 |
| Kubernetes Pod | 临时实例 | Pod 随时可能被调度 |
| 数据库主库 | 持久实例 | 需要固定注册 |
| 消息队列 | 持久实例 | 高可用，不轻易下线 |
| 定时任务 | 临时实例 | 执行完自动下线 |

## 总结

临时实例和持久实例的选择建议：

1. **无状态服务**：使用临时实例（Spring Cloud、gRPC 等）
2. **有状态服务**：使用持久实例（数据库、消息队列）
3. **混合架构**：按需选择，Nacos 支持同一服务混合使用
4. **配置建议**：生产环境对可靠性要求高的服务使用持久实例

理解两种实例类型的区别，有助于设计更合理的服务注册策略。
