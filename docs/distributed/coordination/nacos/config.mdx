# Nacos 配置管理

你的微服务有几十个，每个微服务都有自己的配置文件。每次修改一个配置，都要登录到服务器上改文件、重启服务。有没有办法把配置集中管理，修改后自动推送到所有服务？

Nacos 配置中心就是来解决这个问题的。

## 配置管理原理

### 配置数据模型

```
┌─────────────────────────────────────────────────────────────┐
│                    Nacos 配置数据模型                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Namespace (命名空间)                                        │
│  ├── public (默认)                                          │
│  ├── dev                                                     │
│  ├── test                                                    │
│  └── prod                                                    │
│       │                                                      │
│       └── Group (分组)                                       │
│           ├── DEFAULT_GROUP (默认)                           │
│           └── order-system                                  │
│                │                                             │
│                └── DataId (配置ID)                          │
│                    ├── application.yaml                     │
│                    ├── datasource.yaml                      │
│                    └── redis.yaml                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 配置查找规则

```
配置查找优先级（从高到低）：
1. {namespace}/{group}/{dataId}.{fileExtension}
2. {group}/{dataId}.{fileExtension}
3. {dataId}.{fileExtension}

示例：
dataId = "order-service.yaml"
group = "order-system"

完整路径 = public/order-system/order-service.yaml
```

## Spring Cloud 集成

### Maven 依赖

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2022.0.0.0</version>
</dependency>
```

### 配置文件

```yaml
# bootstrap.yml（必须使用 bootstrap.yml）
spring:
  application:
    name: order-service
  cloud:
    nacos:
      config:
        # Nacos 地址
        server-addr: nacos-server:8848
        # 命名空间
        namespace: dev
        # 分组
        group: order-system
        # 文件扩展名
        file-extension: yaml
        # 共享配置
        shared-configs:
          - data-id: common.yaml
            group: DEFAULT_GROUP
            refresh: true
          - data-id: datasource.yaml
            group: order-system
            refresh: true
        # 开启配置自动刷新
        refresh-enabled: true
```

### 读取配置

```java
// 简单配置
@Value("${app.name}")
private String appName;

// 对象配置
@Configuration
@ConfigurationProperties(prefix = "datasource")
public class DataSourceConfig {
    private String url;
    private String username;
    private String password;
}

// 数组配置
@Value("${redis.nodes}")
private List<String> redisNodes;

// Map 配置
@ConfigurationProperties(prefix = "app")
public class AppConfig {
    private Map<String, String> settings;
}
```

### 配置热更新

```java
// 方式一：@RefreshScope
@RestController
@RefreshScope  // 自动刷新配置
public class ConfigController {

    @Value("${app.config}")
    private String config;

    @GetMapping("/config")
    public String getConfig() {
        return config;
    }
}

// 方式二：@ConfigurationProperties + @RefreshScope
@RestController
@RefreshScope
@ConfigurationProperties(prefix = "app")
public class AppConfigProperties {

    private String config;

    @GetMapping("/config")
    public String getConfig() {
        return config;
    }
}
```

### 配置监听

```java
// 监听配置变化
@Configuration
public class ConfigListener {

    @NacosConfigurationProperties(dataId = "order.yaml", autoRefreshed = true)
    @ConfigurationProperties(prefix = "order")
    public static class OrderConfig {
        private int timeout;
        private int maxRetries;
    }

    // 使用 @NacosValue 监听
    @NacosValue(value = "${app.config:default}", autoRefreshed = true)
    private String config;

    // 使用 NacosConfigListener
    @Bean
    public ApplicationRunner runner(NacosConfigProperties configProperties) {
        return args -> {
            Properties properties = new Properties();
            properties.put("serverAddr", "localhost:8848");

            ConfigService configService = NacosFactory.createConfigService(properties);

            // 添加监听器
            configService.addListener("order.yaml", "DEFAULT_GROUP", new Listener() {
                @Override
                public Executor getExecutor() {
                    return Executors.newSingleThreadExecutor();
                }

                @Override
                public void receiveConfigInfo(String configInfo) {
                    System.out.println("配置变化: " + configInfo);
                }
            });
        };
    }
}
```

## 多环境配置

### 环境隔离

```yaml
# 开发环境 namespace: dev
spring:
  cloud:
    nacos:
      config:
        namespace: dev

# 生产环境 namespace: prod
spring:
  cloud:
    nacos:
      config:
        namespace: prod
```

### 共享配置

```yaml
# bootstrap.yml
spring:
  cloud:
    nacos:
      config:
        # 共享配置列表
        shared-configs:
          # 通用配置
          - data-id: common.yaml
            group: SHARED_GROUP
            refresh: true
          # 数据源配置
          - data-id: datasource.yaml
            group: SHARED_GROUP
            refresh: true
          # Redis 配置
          - data-id: redis.yaml
            group: SHARED_GROUP
            refresh: false
```

### 扩展配置

```yaml
# bootstrap.yml
spring:
  cloud:
    nacos:
      config:
        # 扩展配置（优先级低于 shared-configs）
        extension-configs:
          - data-id: order-ext.yaml
            group: ORDER_GROUP
            refresh: true
```

## 配置的优先级

```
配置优先级（从高到低）：
1. 启动参数 -Dspring.cloud.nacos.configOverride=
2. 扩展配置 extension-configs[n]
3. 共享配置 shared-configs[n]
4. 主配置 {application}.{file-extension}
5. 主配置 {application}-{profile}.{file-extension}
```

## 配置回滚

### 历史版本

```bash
# 查询配置历史
curl "http://nacos-server:8848/nacos/v1/cs/history" \
  "?dataId=order.yaml" \
  "&group=DEFAULT_GROUP"

# 返回示例
{
  "data": [
    {
      "id": 100,
      "dataId": "order.yaml",
      "group": "DEFAULT_GROUP",
      "content": "...",
      "md5": "abc123",
      "gmtModified": 1700000000000,
      "opType": "UPDATE"
    }
  ]
}
```

### 回滚配置

```bash
# 回滚到指定版本
curl -X POST "http://nacos-server:8848/nacos/v1/cs/history/rollback" \
  -d "dataId=order.yaml" \
  -d "group=DEFAULT_GROUP" \
  -d "version=100"
```

## 配置加密

### 加密算法

Nacos 支持 AES 加密存储敏感配置：

```bash
# 在 Nacos 配置加密算法
# application.properties
nacos.config.encrypt.key=your-256-bit-key

# 使用加密配置
# 在 Nacos 控制台配置
dataId: sensitive-config
content: |
  db.password: AES(${cipher:base64-aes-encrypted-password})
```

### 敏感配置处理

```yaml
# bootstrap.yml
# 不将敏感配置推送到 Git
spring:
  cloud:
    nacos:
      config:
        # 敏感配置加密处理
        secret-key: ${NACOS_SECRET_KEY}
```

## 配置管理最佳实践

### 配置文件命名规范

```
{service-name}-{profile}.{file-extension}

示例：
order-service-dev.yaml      # 开发环境
order-service-test.yaml     # 测试环境
order-service-prod.yaml     # 生产环境
```

### 配置分组策略

```
Group 命名规则：
- DEFAULT_GROUP          # 默认分组
- {system}-{env}         # 系统-环境
- {team}-{project}       # 团队-项目

示例：
order-system-dev
order-system-prod
inventory-system-prod
```

## 常见问题

### Q：配置改了但没生效？

A：检查以下几点：

1. 配置的 DataId、Group、Namespace 是否正确
2. `@RefreshScope` 注解是否添加
3. 配置是否在正确的命名空间和分组
4. 是否有语法错误（YAML 格式检查）

### Q：如何实现配置版本管理？

A：使用 Nacos 的历史记录功能：

```bash
# 查询历史版本
curl "http://localhost:8848/nacos/v1/cs/history?dataId=xxx&group=xxx"

# 对比版本差异
curl "http://localhost:8848/nacos/v1/cs/history/diff" \
  -d "dataId=xxx" \
  -d "group=xxx" \
  -d "version1=10" \
  -d "version2=11"
```

### Q：配置中心挂了怎么办？

A：Nacos 支持本地缓存：

```yaml
spring:
  cloud:
    nacos:
      config:
        # 开启本地缓存
        cache-enabled: true
        # 缓存目录
        cache-path: /tmp/nacos/config
```

## 总结

Nacos 配置管理的核心要点：

1. **集中管理**：所有配置存储在 Nacos
2. **热更新**：配置变更自动推送到应用
3. **多环境**：命名空间 + 分组实现环境隔离
4. **版本控制**：配置变更历史可追溯
5. **高可用**：本地缓存防止配置中心故障
