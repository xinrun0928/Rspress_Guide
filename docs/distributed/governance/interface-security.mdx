# 接口安全

你的 API 被爬虫抓取了。你的接口被 CSRF 攻击了。你的用户数据被 SQL 注入了。你的系统被 DDoS 打瘫了。

接口安全不是锦上添花，而是生产系统的必需品。一旦出现安全漏洞，轻则数据泄露，重则系统瘫痪，更严重的可能面临法律风险。

## 接口安全面临的威胁

### 常见攻击类型

```yaml
# 接口安全威胁一览
1. SQL 注入
   攻击方式：在参数中注入 SQL 语句
   危害：数据泄露、数据篡改、数据库瘫痪
   示例：?id=1 OR 1=1

2. XSS 跨站脚本攻击
   攻击方式：在页面中注入恶意脚本
   危害：窃取 Cookie、Session 劫持、页面篡改
   示例：<script>alert('hack')</script>

3. CSRF 跨站请求伪造
   攻击方式：利用用户的登录状态发起恶意请求
   危害：用户不知情地执行操作
   示例：钓鱼邮件中的恶意链接

4. 重放攻击
   攻击方式：捕获并重放合法请求
   危害：重复执行操作、绕过一次性验证
   示例：重放登录请求获取 Token

5. DDoS 攻击
   攻击方式：大量请求涌向服务器
   危害：服务不可用
   应对：限流、CDN、清洗服务

6. 越权访问
   攻击方式：篡改参数访问他人数据
   危害：数据泄露
   示例：?userId=123 → ?userId=456

7. 敏感数据泄露
   攻击方式：接口返回敏感信息或错误信息过于详细
   危害：信息泄露
   示例：返回数据库错误、堆栈信息

8. 密码暴力破解
   攻击方式：使用字典或穷举尝试登录
   危害：账号被盗
   应对：限流、验证码、账户锁定
```

## 防护方案

### 1. 参数校验与过滤

永远不要相信用户的输入。

```java
// 参数校验示例
@Service
public class UserService {

    // 使用 @Valid 注解进行参数校验
    public Result createUser(@Valid @RequestBody CreateUserRequest request) {
        // 参数校验由 Spring Validation 自动完成
        return Result.success(userRepository.save(convertToEntity(request)));
    }

    // 手动校验（需要额外校验的场景）
    public Result updateUser(@RequestBody UpdateUserRequest request) {
        // 校验 ID
        if (request.getUserId() == null || request.getUserId() <= 0) {
            throw new IllegalArgumentException("无效的用户 ID");
        }

        // 校验手机号格式
        if (!isValidPhone(request.getPhone())) {
            throw new IllegalArgumentException("无效的手机号格式");
        }

        // 校验邮箱格式
        if (!isValidEmail(request.getEmail())) {
            throw new IllegalArgumentException("无效的邮箱格式");
        }

        // 校验密码强度
        if (!isStrongPassword(request.getPassword())) {
            throw new IllegalArgumentException("密码强度不足");
        }

        // 校验枚举值
        if (!EnumUtils.isValidEnum(UserType.class, request.getUserType())) {
            throw new IllegalArgumentException("无效的用户类型");
        }

        return Result.success(userRepository.save(convertToEntity(request)));
    }

    // 自定义校验器
    public static boolean isStrongPassword(String password) {
        if (password == null || password.length() < 8) {
            return false;
        }
        // 包含大写字母
        boolean hasUpper = password.chars().anyMatch(Character::isUpperCase);
        // 包含小写字母
        boolean hasLower = password.chars().anyMatch(Character::isLowerCase);
        // 包含数字
        boolean hasDigit = password.chars().anyMatch(Character::isDigit);
        // 包含特殊字符
        boolean hasSpecial = password.chars().anyMatch(c -> !Character.isLetterOrDigit(c));

        return hasUpper && hasLower && hasDigit && hasSpecial;
    }
}

// 请求对象
public class CreateUserRequest {

    @NotBlank(message = "用户名不能为空")
    @Size(min = 3, max = 20, message = "用户名长度必须在 3-20 之间")

    private String username;

    @Size(min = 8, max = 20, message = "密码长度必须在 8-20 之间")

    private String phone;

    @NotBlank(message = "邮箱不能为空")
    @Email(message = "邮箱格式不正确")
    private String email;

    // 枚举值校验
    @NotNull(message = "用户类型不能为空")
    private UserType userType;
}
```

### 3. XSS 防护

对输出进行转义。

```java
// 全局 XSS 过滤器
@Component
@WebFilter(urlPatterns = "/*")
public class XssFilter implements Filter {

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        chain.doFilter(new XssHttpServletRequestWrapper((HttpServletRequest) request), response);
    }
}

// XSS 过滤包装器
public class XssHttpServletRequestWrapper extends HttpServletRequestWrapper {

    private static final HtmlEscaper htmlEscaper = new HtmlEscaper();

    public XssHttpServletRequestWrapper(HttpServletRequest request) {
        super(request);
    }

    @Override
    public String getParameter(String name) {
        String value = super.getParameter(name);
        return htmlEscaper.escape(value);
    }

    @Override
    public String[] getParameterValues(String name) {
        String[] values = super.getParameterValues(name);
        if (values == null) {
            return null;
        }
        return Arrays.stream(values)
                .map(htmlEscaper::escape)
                .toArray(String[]::new);
    }

    @Override
    public String getHeader(String name) {
        String value = super.getHeader(name);
        return htmlEscaper.escape(value);
    }
}

// 使用 OWASP Java Encoder
@Service
public class HtmlEscaper {

    public String escape(String input) {
        if (input == null) {
            return null;
        }
        // 使用 OWASP Java Encoder
        return Encoder.forHtml(input);
    }
}
```

### 4. CSRF 防护

使用 Token 验证请求来源。

```java
// CSRF Token 配置
@Configuration
public class CsrfConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf
                // 启用 CSRF 保护
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
                // 对 API 请求进行特殊处理
                .ignoringAntMatchers("/api/public/**")
            )
            .addFilterAfter(new CsrfTokenInterceptor(), BasicAuthenticationFilter.class);
        return http.build();
    }
}

// 自定义 CSRF 拦截器
public class CsrfTokenInterceptor extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain)
            throws ServletException, IOException {

        // 获取 CSRF Token
        String token = request.getHeader("X-CSRF-Token");
        if (StringUtils.isBlank(token)) {
            token = request.getParameter("_csrf");
        }

        // 验证 Token
        CsrfToken csrfToken = (CsrfToken) request.getAttribute("_csrf");
        if (csrfToken != null && !csrfToken.getToken().equals(token)) {
            response.setStatus(HttpServletResponse.SC_FORBIDDEN);
            response.getWriter().write("CSRF Token 无效");
            return;
        }

        filterChain.doFilter(request, response);
    }
}
```

```javascript
// 前端处理 CSRF Token
class ApiClient {
    constructor() {
        this.csrfToken = '';
    }

    // 从 Cookie 或 DOM 中获取 CSRF Token
    getCsrfToken() {
        // 从 meta 标签获取
        if (!this.csrfToken) {
            const meta = document.querySelector('meta[name="csrf-token"]');
            this.csrfToken = meta ? meta.content : '';
        }
        return this.csrfToken;
    }

    async request(url, options = {}) {
        return fetch(url, {
            ...options,
            headers: {
                ...options.headers,
                'X-CSRF-Token': this.getCsrfToken(),
                'Content-Type': 'application/json',
            },
            credentials: 'include',  // 带上 Cookie
        });
    }
}
```

### 5. 请求签名与重放防护

防止请求被篡改和重放。

```java
// 请求签名服务
@Service
public class RequestSignatureService {

    private static final String SECRET_KEY = "your-secret-key";  // 应当从配置中心获取

    /**
     * 生成签名
     * @param timestamp 时间戳
     * @param nonce 随机数
     * @param body 请求体（可选）
     */
    public String generateSignature(String timestamp, String nonce, String body) {
        // 排序拼接
        String content = String.format("timestamp=%s&nonce=%s&body=%s",
                timestamp, nonce, body != null ? body : "");

        // 使用 HMAC-SHA256
        try {
            Mac mac = Mac.getInstance("HmacSHA256");
            SecretKey key = new SecretKeySpec(SECRET_KEY.getBytes(), "HmacSHA256");
            mac.init(key);
            byte[] hash = mac.doFinal(content.getBytes());
            return Hex.encodeHexString(hash);
        } catch (NoSuchAlgorithmException | InvalidKeyException e) {
            throw new RuntimeException("签名生成失败", e);
        }
    }

    /**
     * 验证签名
     */
    public boolean verifySignature(String timestamp, String nonce,
                                   String body, String signature) {
        // 验证时间戳（5 分钟内有效）
        long ts = Long.parseLong(timestamp);
        long now = System.currentTimeMillis();
        if (Math.abs(now - ts) > 5 * 60 * 1000) {
            return false;  // 请求已过期
        }

        // 验证签名
        String expected = generateSignature(timestamp, nonce, body);
        return expected.equals(signature);
    }
}

// 重放攻击防护
@Service
public class ReplayProtectionService {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    private static final String REPLAY_PREFIX = "request:nonce:";
    private static final long NONCE_EXPIRE_SECONDS = 300;  // 5 分钟

    /**
     * 检查 nonce 是否已使用
     */
    public boolean isNonceUsed(String nonce) {
        String key = REPLAY_PREFIX + nonce;
        return Boolean.TRUE.equals(redisTemplate.hasKey(key));
    }

    /**
     * 标记 nonce 已使用
     */
    public void markNonceUsed(String nonce) {
        String key = REPLAY_PREFIX + nonce;
        redisTemplate.opsForValue().set(key, "1", NONCE_EXPIRE_SECONDS, TimeUnit.SECONDS);
    }
}

// 签名验证过滤器
@Component
@WebFilter(urlPatterns = "/api/secure/*")
public class SignatureFilter extends OncePerRequestFilter {

    @Autowired
    private RequestSignatureService signatureService;

    @Autowired
    private ReplayProtectionService replayService;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain chain) throws ServletException, IOException {

        String timestamp = request.getHeader("X-Timestamp");
        String nonce = request.getHeader("X-Nonce");
        String signature = request.getHeader("X-Signature");

        // 检查必要参数
        if (timestamp == null || nonce == null || signature == null) {
            response.setStatus(HttpServletResponse.SC_BAD_REQUEST);
            response.getWriter().write("缺少签名参数");
            return;
        }

        // 检查 nonce 是否已使用（防止重放）
        if (replayService.isNonceUsed(nonce)) {
            response.setStatus(HttpServletResponse.SC_FORBIDDEN);
            response.getWriter().write("请求已被处理");
            return;
        }

        // 读取请求体
        String body = StreamUtils.copyToString(request.getInputStream(),
                StandardCharsets.UTF_8);

        // 验证签名
        if (!signatureService.verifySignature(timestamp, nonce, body, signature)) {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.getWriter().write("签名验证失败");
            return;
        }

        // 标记 nonce 已使用
        replayService.markNonceUsed(nonce);

        chain.doFilter(request, response);
    }
}
```

```javascript
// 客户端签名生成
class SignedRequestClient {
    constructor(secretKey) {
        this.secretKey = secretKey;
    }

    async sign(url, method = 'GET', body = null) {
        const timestamp = Date.now().toString();
        const nonce = this.generateNonce();
        const bodyString = body ? JSON.stringify(body) : '';

        // 生成签名
        const content = `timestamp=${timestamp}&nonce=${nonce}&body=${bodyString}`;
        const signature = await this.computeSignature(content);

        return {
            url,
            method,
            headers: {
                'X-Timestamp': timestamp,
                'X-Nonce': nonce,
                'X-Signature': signature,
                'Content-Type': 'application/json',
            },
            body: bodyString || undefined,
        };
    }

    generateNonce() {
        return Math.random().toString(36).substring(2, 15) +
               Math.random().toString(36).substring(2, 15);
    }

    async computeSignature(content) {
        const encoder = new TextEncoder();
        const key = await crypto.subtle.importKey(
            'raw',
            encoder.encode(this.secretKey),
            { name: 'HMAC', hash: 'SHA-256' },
            false,
            ['sign']
        );
        const signature = await crypto.subtle.sign('HMAC', key, encoder.encode(content));
        return Array.from(new Uint8Array(signature))
            .map(b => b.toString(16).padStart(2, '0'))
            .join('');
    }
}
```

```java
// 基于角色的权限控制
@Service
public class PermissionService {

    /**
     * 检查用户是否有权限访问
     */
    public boolean hasPermission(Long userId, String resource, String action) {
        List<Role> roles = roleRepository.findByUserId(userId);
        return roles.stream()
                .anyMatch(role -> role.hasPermission(resource, action));
    }

    /**
     * 数据权限控制：用户只能访问自己的数据
     */
    @Transactional(readOnly = true)
    public List<Order> getUserOrders(Long userId, Long queryUserId) {
        // 查询者只能查询自己的数据（除非是管理员）
        if (!userId.equals(queryUserId) && !isAdmin(userId)) {
            throw new UnauthorizedException("无权访问他人数据");
        }
        return orderRepository.findByUserId(queryUserId);
    }

    private boolean isAdmin(Long userId) {
        return userRepository.findById(userId)
                .map(User::isAdmin)
                .orElse(false);
    }
}

// 方法级权限控制
@Service
public class OrderService {

    // 只能操作自己的订单
    @PreAuthorize("#order.userId == authentication.userId or hasRole('ADMIN')")

    }

    // 使用自定义权限表达式
    @PreAuthorize("@permissionService.canAccessOrder(#orderId, authentication.userId)")

// 越权访问防护
@Component
public class DataIsolationAspect {

    @Autowired
    private SecurityContext securityContext;

    @Around("execution(* com.example.repository.*Repository.find*(..))")
    public Object checkDataAccess(ProceedingJoinPoint joinPoint) throws Throwable {
        // 获取当前用户
        User currentUser = securityContext.getCurrentUser();
        if (currentUser == null || currentUser.isAdmin()) {
            return joinPoint.proceed();  // 管理员跳过检查
        }

        // 检查方法参数是否包含 userId
        MethodSignature signature = (MethodSignature) joinPoint.getSignature();
        String methodName = signature.getName();
        Object[] args = joinPoint.getArgs();

        // 如果方法包含 userId 参数，必须匹配当前用户
        Long methodUserId = extractUserIdFromArgs(args, signature.getParameterNames());
        if (methodUserId != null && !methodUserId.equals(currentUser.getId())) {
            throw new ForbiddenException("无权访问此数据");
        }

        return joinPoint.proceed();
    }
}
```

```java
// 敏感数据脱敏
@Service
public class DataMaskingService {

    // 手机号脱敏：138****5678
    public String maskPhone(String phone) {
        if (phone == null || phone.length() < 7) {
            return phone;
        }
        return phone.substring(0, 3) + "****" + phone.substring(phone.length() - 4);
    }

    // 身份证脱敏：320***********1234
    public String maskIdCard(String idCard) {
        if (idCard == null || idCard.length() < 10) {
            return idCard;
        }
        return idCard.substring(0, 3) + "***********" + idCard.substring(idCard.length() - 4);
    }

    // 银行卡脱敏：622202*********1234
    public String maskBankCard(String bankCard) {
        if (bankCard == null || bankCard.length() < 6) {
            return bankCard;
        }
        return bankCard.substring(0, 4) + "***********" + bankCard.substring(bankCard.length() - 4);
    }

    // 邮箱脱敏：t***@example.com
    public String maskEmail(String email) {
        if (email == null || !email.contains("@")) {
            return email;
        }
        String[] parts = email.split("@");
        String name = parts[0];
        String domain = parts[1];
        if (name.length() <= 1) {
            return name + "@" + domain;
        }
        return name.charAt(0) + "***" + "@" + domain;
    }
}
```

## 总结

接口安全是系统安全的第一道防线。需要从多个层面进行防护：输入校验防止注入、Token 验证防止 CSRF、签名机制防止篡改和重放、权限控制防止越权。

**核心要点回顾：**

- 参数校验是一切安全的基础，永远不要相信用户输入
- SQL 注入防护：使用参数化查询
- XSS 防护：输出转义
- CSRF 防护：使用 Token 验证请求来源
- 重放防护：签名 + 时间戳 + Nonce
- 权限控制：数据级别隔离
- 敏感数据：加密存储、脱敏展示
- 限流防爬：保护系统免受 DDoS 和爬虫

这些安全措施不是可选的，而是生产系统的必需品。
