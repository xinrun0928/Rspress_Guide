# 服务治理：微服务稳定性保障

你有没有见过这种场景：

系统正常时响应 50ms，突然下游数据库慢了一下，所有请求都堆积在网关。

然后是服务 A 超时 → 服务 B 堆积 → 服务 C 崩溃 → 全链路雪崩。

**这就是服务治理要解决的问题。**

服务治理是微服务架构的"保护伞"——在系统正常时放行流量，在系统异常时及时止损。

## 保护系统的四板斧

| 策略 | 作用 | 触发条件 |
|------|------|---------|
| 熔断 | 保护调用方，快速失败 | 下游持续超时/错误 |
| 降级 | 保护系统，牺牲非核心 | 资源不足/系统过载 |
| 限流 | 保护基础设施 | 流量超过阈值 |
| 超时 | 防止资源耗尽 | 请求超过预期时间 |

## 核心公式

```
服务治理 = 服务发现 + 负载均衡 + 容错保护 + 安全幂等

系统保护四件套：
- 熔断：当下游持续失败，直接短路
- 降级：核心功能优先，非核心功能降级
- 限流：控制入口流量，不超过系统容量
- 超时：避免无限等待，及时释放资源

组合策略：限流兜底 → 超时传递 → 熔断止损 → 降级保命
```
