# 可靠消息最终一致性

<Badge text="分布式事务" type="tip" /> <Badge text="消息队列" type="warning" /> <Badge text="最终一致" type="info" />

你有没有想过：为什么不直接用消息队列来做分布式事务？

消息队列天生就是用来解耦和异步化的。如果能保证「本地事务执行成功」和「消息发送成功」的一致性，就能通过消息队列实现最终一致性。

**可靠消息最终一致性**就是基于这个思想实现的。

## 核心思想

```java
/**
 * 可靠消息的核心思想
 */

// 目标：保证「本地事务」和「消息发送」的一致性

// 问题：如何保证本地事务成功后，消息一定能发送成功？

// 解决方案：
// 1. 本地消息表：将消息和业务放在同一个数据库中
// 2. 事务消息：将消息发送纳入事务管理
```

## 本地消息表方案

### 原理

```java
/**
 * 本地消息表方案
 */

// 1. 创建消息表（与业务表在同一个数据库）
// 2. 本地事务中，同时操作业务和消息
// 3. 后台任务轮询消息表，发送消息
// 4. 消息发送成功后，标记消息为已发送

/**
 * 流程
 */

// 业务处理 + 消息入库（同一事务）：
// BEGIN;
//   INSERT INTO orders (...) VALUES (...);  -- 业务
//   INSERT INTO message_outbox (...) VALUES (...);  -- 消息
// COMMIT;

// 消息发送（异步）：
// SELECT * FROM message_outbox WHERE status = 'PENDING';
// 发送消息...
// UPDATE message_outbox SET status = 'SENT' WHERE id = ?;

// 补偿机制：
// - 定时扫描 PENDING 状态的消息
// - 重试发送
// - 死信处理
```

### 代码实现

```sql
-- 消息表结构
CREATE TABLE `message_outbox` (
    `id` bigint NOT NULL AUTO_INCREMENT,
    `business_key` varchar(64) DEFAULT NULL COMMENT '业务 Key',
    `message_type` varchar(32) DEFAULT NULL COMMENT '消息类型',
    `message_body` text COMMENT '消息体',
    `status` tinyint DEFAULT '0' COMMENT '状态：0-待发送，1-已发送，2-发送失败',
    `retry_count` int DEFAULT '0' COMMENT '重试次数',
    `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
    `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_status_create_time` (`status`,`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

```java
/**
 * 本地消息表实现
 */
@Service
public class LocalMessageService {
    
    @Autowired
    private MessageOutboxDao messageOutboxDao;
    
    @Autowired
    private TransactionTemplate transactionTemplate;
    
    /**
     * 发送可靠消息
     */
    public void sendMessage(String businessKey, String messageType, Object messageBody) {
        // 使用事务保证原子性
        transactionTemplate.executeWithoutResult(status -> {
            // 1. 执行业务逻辑（在调用方的事务中）
            
            // 2. 保存消息到本地消息表
            MessageOutbox message = new MessageOutbox();
            message.setBusinessKey(businessKey);
            message.setMessageType(messageType);
            message.setMessageBody(JSON.toJSONString(messageBody));
            message.setStatus(MessageStatus.PENDING);
            messageOutboxDao.insert(message);
        });
    }
}

/**
 * 消息发送任务
 */
@Service
public class MessageSendTask {
    
    @Autowired
    private MessageOutboxDao messageOutboxDao;
    
    @Autowired
    private MessageQueue messageQueue;
    
    private static final int MAX_RETRY = 3;
    
    /**
     * 定时发送消息
     */
    @Scheduled(fixedDelay = 1000)
    public void sendPendingMessages() {
        // 1. 查询待发送消息
        List<MessageOutbox> messages = messageOutboxDao.selectPending(
            MessageStatus.PENDING, 
            MAX_RETRY,
            100
        );
        
        // 2. 发送每条消息
        for (MessageOutbox message : messages) {
            try {
                // 3. 发送消息
                messageQueue.send(message.getMessageType(), message.getMessageBody());
                
                // 4. 更新状态
                messageOutboxDao.updateStatus(message.getId(), MessageStatus.SENT);
                
            } catch (Exception e) {
                // 5. 发送失败，增加重试次数
                messageOutboxDao.incrementRetryCount(message.getId());
                
                // 6. 如果超过最大重试次数，标记为失败
                if (message.getRetryCount() >= MAX_RETRY) {
                    messageOutboxDao.updateStatus(message.getId(), MessageStatus.FAILED);
                }
            }
        }
    }
}
```

### 消费者处理

```java
/**
 * 消息消费者
 */
@Service
public class OrderMessageConsumer {
    
    @Autowired
    private OrderService orderService;
    
    /**
     * 处理消息
     */
    @Consume(topic = "order.created", group = "order-service")
    public void handleOrderCreated(OrderCreatedEvent event) {
        try {
            // 1. 幂等检查
            if (isProcessed(event.getOrderId())) {
                return;
            }
            
            // 2. 处理业务
            orderService.processOrderNotification(event);
            
            // 3. 标记已处理
            markAsProcessed(event.getOrderId());
            
        } catch (Exception e) {
            // 4. 处理失败，抛出异常触发重试
            throw new RuntimeException("处理订单消息失败", e);
        }
    }
    
    /**
     * 幂等检查
     */
    private boolean isProcessed(Long orderId) {
        return orderProcessedDao.exists(orderId);
    }
    
    /**
     * 标记已处理
     */
    private void markAsProcessed(Long orderId) {
        OrderProcessed record = new OrderProcessed();
        record.setOrderId(orderId);
        record.setProcessedTime(new Date());
        orderProcessedDao.insert(record);
    }
}
```

## RocketMQ 事务消息

### 原理

```java
/**
 * RocketMQ 事务消息
 */

// RocketMQ 4.3+ 支持事务消息
// 原理：两阶段提交 + 事务回查

/**
 * 流程
 */

// 阶段一：发送半消息（Half Message）
// - 消息先发送到 RocketMQ
// - 消息标记为「不可投递」

// 阶段二：执行本地事务
// - 执行业务逻辑
// - 返回事务结果（COMMIT / ROLLBACK）

// 阶段三：提交或回滚
// - COMMIT：消息变为可投递
// - ROLLBACK：消息被删除

// 阶段四：事务回查
// - 如果阶段二超时或失败
// - RocketMQ 会回查消息状态
// - 生产者返回最终状态
```

### 代码实现

```java
/**
 * RocketMQ 事务消息发送
 */
@Service
public class TransactionMessageService {
    
    @Autowired
    private TransactionMQProducer producer;
    
    @Autowired
    private OrderDao orderDao;
    
    /**
     * 发送事务消息
     */
    public void sendOrderCreatedMessage(Order order) {
        // 1. 构建消息
        OrderCreatedMessage message = new OrderCreatedMessage();
        message.setOrderId(order.getId());
        message.setUserId(order.getUserId());
        message.setTotalAmount(order.getTotalAmount());
        
        // 2. 发送事务消息
        // TransactionListener 执行本地事务
        // 返回事务结果
        producer.sendMessageInTransaction(
            "order-topic:created",
            message,
            new TransactionListener() {
                @Override
                public LocalTransactionState executeLocalTransaction(
                    Message msg, Object arg) {
                    try {
                        // 执行本地事务
                        // 这里不需要额外操作，订单已经创建
                        return LocalTransactionState.COMMIT_MESSAGE;
                    } catch (Exception e) {
                        return LocalTransactionState.ROLLBACK_MESSAGE;
                    }
                }
                
                @Override
                public LocalTransactionState checkLocalTransaction(MessageExt msg) {
                    // 事务回查：检查本地事务状态
                    String orderId = extractOrderId(msg);
                    
                    // 检查订单是否存在
                    Order order = orderDao.selectById(orderId);
                    if (order != null) {
                        return LocalTransactionState.COMMIT_MESSAGE;
                    } else {
                        return LocalTransactionState.ROLLBACK_MESSAGE;
                    }
                }
            },
            order.getId()
        );
    }
}

/**
 * 事务监听器实现
 */
@Component
public class OrderTransactionListener implements TransactionListener {
    
    @Autowired
    private OrderDao orderDao;
    
    @Override
    public LocalTransactionState executeLocalTransaction(Message msg, Object arg) {
        Long orderId = (Long) arg;
        
        try {
            // 检查本地事务状态
            Order order = orderDao.selectById(orderId);
            if (order != null) {
                // 本地事务已提交
                return LocalTransactionState.COMMIT_MESSAGE;
            } else {
                // 本地事务未提交或已回滚
                return LocalTransactionState.ROLLBACK_MESSAGE;
            }
        } catch (Exception e) {
            return LocalTransactionState.UNKNOWN;
        }
    }
    
    @Override
    public LocalTransactionState checkLocalTransaction(MessageExt msg) {
        // 事务回查
        String orderId = extractOrderId(msg);
        
        Order order = orderDao.selectById(orderId);
        if (order != null) {
            return LocalTransactionState.COMMIT_MESSAGE;
        } else {
            return LocalTransactionState.ROLLBACK_MESSAGE;
        }
    }
}
```

### 消费者实现

```java
/**
 * 消费者处理事务消息
 */
@Service
public class OrderConsumer {
    
    @Autowired
    private InventoryService inventoryService;
    
    /**
     * 处理订单创建消息
     */
    @RocketMQMessageListener(
        topic = "order-topic",
        selectorExpression = "created",
        consumerGroup = "inventory-consumer-group"
    )
    public class OrderCreatedListener implements RocketMQListener<OrderCreatedMessage> {
        
        @Override
        public void onMessage(OrderCreatedMessage message) {
            try {
                // 1. 幂等检查
                if (isProcessed(message.getOrderId())) {
                    return;
                }
                
                // 2. 扣减库存
                inventoryService.deductStock(
                    message.getProductId(), 
                    message.getCount()
                );
                
                // 3. 标记已处理
                markAsProcessed(message.getOrderId());
                
            } catch (Exception e) {
                // 4. 失败时抛出异常，触发重试
                throw new RuntimeException("处理失败", e);
            }
        }
    }
}
```

## RabbitMQ 事务消息

```java
/**
 * RabbitMQ 实现
 */

// RabbitMQ 不原生支持事务消息
// 需要结合手动确认和消息表实现

/**
 * 实现思路
 */

// 1. 发送消息前，先把消息入库
// 2. 发送消息时使用 publisher confirms
// 3. 收到确认后，标记消息为已发送
// 4. 未确认的消息定时重发

@Service
public class RabbitMQTransactionService {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @Autowired
    private MessageOutboxDao messageOutboxDao;
    
    @Autowired
    private TransactionTemplate transactionTemplate;
    
    /**
     * 发送事务消息
     */
    public void sendTransactionalMessage(String exchange, String routingKey, 
                                        Object message, Long businessId) {
        // 1. 使用事务
        transactionTemplate.executeWithoutResult(status -> {
            // 2. 业务处理（在调用方的事务中）
            
            // 3. 保存消息到本地消息表
            MessageOutbox outbox = new MessageOutbox();
            outbox.setBusinessId(businessId);
            outbox.setExchange(exchange);
            outbox.setRoutingKey(routingKey);
            outbox.setMessageBody(JSON.toJSONString(message));
            outbox.setStatus(MessageStatus.PENDING);
            messageOutboxDao.insert(outbox);
        });
        
        // 4. 异步发送消息（不在事务中）
        asyncSendMessage(outbox.getId());
    }
    
    /**
     * 异步发送消息
     */
    private void asyncSendMessage(Long messageId) {
        CompletableFuture.runAsync(() -> {
            MessageOutbox message = messageOutboxDao.selectById(messageId);
            
            if (message.getStatus() != MessageStatus.PENDING) {
                return;
            }
            
            rabbitTemplate.execute(channel -> {
                // 设置 publisher confirms
                channel.confirmSelect();
                
                // 发送消息
                channel.basicPublish(
                    message.getExchange(),
                    message.getRoutingKey(),
                    MessageProperties.PERSISTENT,
                    message.getMessageBody().getBytes()
                );
                
                return null;
            });
        });
    }
}
```

## 方案对比

```java
/**
 * 本地消息表 vs 事务消息
 */

// 本地消息表：
// - 优点：不依赖特定 MQ，支持所有消息队列
// - 缺点：需要维护消息表，增加数据库压力
// - 适用：RabbitMQ、ActiveMQ 等不支持事务消息的 MQ

// 事务消息（RocketMQ）：
// - 优点：MQ 原生支持，无需额外表
// - 缺点：依赖特定 MQ
// - 适用：RocketMQ
```

## 总结

可靠消息最终一致性是实现分布式事务的重要方案：

1. **本地消息表**：通过消息表保证消息发送的可靠性
2. **事务消息**：通过 MQ 原生支持实现
3. **最终一致**：允许短暂不一致，通过重试保证最终一致
4. **幂等处理**：消费者需要实现幂等性

这种方案适合异步化场景，是分布式事务的轻量级解决方案。
