## Seata 架构概览

### 三大角色

```java
/**
 * Seata 的三大角色
 */

// 1. TC（Transaction Coordinator）- 事务协调者
// - Seata Server
// - 维护全局事务状态
// - 协调分支事务提交/回滚
// - 管理者全局锁

// 2. TM（Transaction Manager）- 事务管理器
// - Seata Client（业务应用）
// - 开启全局事务
// - 发起全局提交或回滚
// - 管理事务边界

// 3. RM（Resource Manager）- 资源管理器
// - Seata Client（业务应用）
// - 管理分支事务
// - 注册资源到 TC
// - 执行分支事务
```

## 核心组件详解

### TC（Transaction Coordinator）

```java
/**
 * TC 的核心职责
 */

// 1. 全局事务管理
// - 创建全局事务
// - 维护全局事务状态
// - 协调全局事务提交/回滚

// 2. 分支事务管理
// - 注册分支事务
// - 维护分支事务状态
// - 协调分支事务提交/回滚

// 3. 全局锁管理
// - 管理全局锁
// - 处理锁竞争
// - 防止脏写

/**
 * TC 的核心类
 */
public class DefaultCoordinator {
    
    // 会话管理器：管理全局会话
    private SessionManager sessionManager;
    
    // 锁管理器：管理全局锁
    private LockManager lockManager;
    
    // 事务处理器：处理事务请求
    private TransactionHandlerManager transactionHandlerManager;
    
    // 高可用组件：保证 TC 自身的高可用
    private HAStrategy haStrategy;
}

/**
 * TC 的核心流程
 */

// 1. 接收 TM 的请求
// 2. 创建全局事务会话
// 3. 注册分支事务
// 4. 处理分支事务结果
// 5. 决定全局事务提交或回滚
// 6. 通知 RM 执行分支提交/回滚
```

### Session Manager

```java
/**
 * Session Manager
 */

// 管理所有全局事务的会话
// 每个全局事务对应一个 Session

/**
 * Session 结构
 */
public class GlobalSession {
    
    private String xid;                    // 全局事务 ID
    private String applicationId;           // 应用 ID
    private String transactionServiceGroup; // 事务组
    private TransactionStatus status;       // 事务状态
    
    private List<BranchSession> branchSessions;  // 分支会话列表
    
    private Date createTime;               // 创建时间
    private Date activeTime;               // 最后活跃时间
}

/**
 * Session 状态
 */
public enum TransactionStatus {
    BEGIN,              // 开始
    COMMITTING,         // 提交中
    COMMITTED,          // 已提交
    COMMIT_FAILED,      // 提交失败
    ROLLBACKING,        // 回滚中
    ROLLBACKED,         // 已回滚
    ROLLBACK_FAILED,    // 回滚失败
    TIMEOUTROLLING,     // 超时回滚中
    TIMEOUT_ROLLBACK_FAILED,  // 超时回滚失败
    FINISHED           // 已结束
}
```

### TM（Transaction Manager）

```java
/**
 * TM 的核心职责
 */

// 1. 开启全局事务
// - 生成 XID
// - 向 TC 注册全局事务
// - 返回给应用

// 2. 管理全局事务边界
// - 在事务结束时发起提交或回滚
// - 处理异常时触发回滚

// 3. 传播 XID
// - 将 XID 传递到各个 RM

/**
 * TM 核心类
 */
public class GlobalTransactionProxy {
    
    /**
     * 获取全局事务代理
     */
    @GlobalTransactional(name = "create-order", rollbackFor = Exception.class)
    public void createOrder(OrderDTO orderDTO) {
        // 在代理中执行
        // 1. 开启全局事务
        // 2. 执行方法
        // 3. 提交或回滚
    }
}

/**
 * TM 注解
 */
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface GlobalTransactional {
    
    String name() default "";
    
    int timeoutMills() default 60000;
    
    Class<? extends Throwable>[] rollbackFor() default {};
    
    String[] rollbackForClassName() default {};
}
```

### RM（Resource Manager）

```java
/**
 * RM 的核心职责
 */

// 1. 管理数据源
// - 包装数据源
// - 拦截 SQL

// 2. 分支事务管理
// - 注册分支事务到 TC
// - 执行分支事务
// - 报告分支事务状态

// 3. Undo Log 管理
// - 生成 Undo Log
// - 回滚时使用 Undo Log

/**
 * RM 核心类
 */
public class DataSourceProxy implements DataSource {
    
    private ConnectionProxy connectionProxy;  // 代理连接
    private String resourceId;               // 资源 ID
    
    /**
     * 获取代理连接
     */
    @Override
    public Connection getConnection() throws SQLException {
        return new ConnectionProxy(this);
    }
}

/**
 * Connection Proxy
 */
public class ConnectionProxy implements Connection {
    
    private Connection originalConnection;    // 原始连接
    private DataSourceProxy dataSourceProxy;
    private String xid;                     // 全局事务 ID
    
    /**
     * 执行 SQL
     */
    @Override
    public Statement createStatement() throws SQLException {
        return new StatementProxy(this, 
            originalConnection.createStatement());
    }
    
    /**
     * 提交
     */
    @Override
    public void commit() throws SQLException {
        // 1. 执行分支提交
        // 2. 向 TC 注册分支事务
        // 3. 记录 Undo Log
        // 4. 提交本地事务
    }
}
```

## AT 模式的执行流程

```java
/**
 * AT 模式执行流程
 */

// 1. TM 开启全局事务
// TM Proxy.begin() -> TC.registerGlobalSession() -> 返回 XID

// 2. TM 执行分支事务
// 执行 SQL -> RM 拦截 -> 生成前后镜像 -> 执行 SQL -> 提交本地

// 3. TM 提交全局事务
// TM Proxy.commit() -> TC.commit() -> 删除 Undo Log

// 4. 或 TM 回滚全局事务
// TM Proxy.rollback() -> TC.rollback() -> RM 回滚 -> 使用 Undo Log 回滚
```

## 总结

Seata 的整体架构设计清晰、模块化：

1. **三大角色**：TC（协调者）、TM（事务管理器）、RM（资源管理器）
2. **核心组件**：Session Manager（会话管理）、Lock Manager（锁管理）
3. **AT 模式**：通过 Undo Log 实现自动回滚
4. **高性能**：本地事务先提交，全局锁持有时间短

理解 Seata 的架构，对于深入理解分布式事务至关重要。
