# 流量控制：限流、削峰与弹性降级

双十一零点，你的系统迎来了 10 倍于平时的流量。

数据库只有 1000 QPS 的处理能力，但瞬时请求量达到了 10000 QPS。

**结果是：数据库被打垮，所有请求都超时，用户什么都访问不了。**

这就是流量控制的起点：**当系统扛不住时，要优雅地拒绝，而不是全面崩溃。**

## 流量控制的核心策略

```
┌─────────────────────────────────────────────────────────┐
│                  流量控制四板斧                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 限流（Rate Limiting）                               │
│     └─ 控制进入系统的请求速率，不超过系统容量             │
│                                                         │
│  2. 削峰（Peak Shaving）                                │
│     └─ 把瞬时高峰摊平，用消息队列缓冲                    │
│                                                         │
│  3. 降级（Degradation）                                 │
│     └─ 牺牲非核心功能，保证核心功能可用                  │
│                                                         │
│  4. 熔断（Circuit Breaker）                             │
│     └─ 当下游持续失败，直接短路，不再调用下游            │
│                                                         │
│  组合使用：限流兜底 → 削峰缓冲 → 降级保核心 → 熔断止损  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 限流方案对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 计数器 | 固定时间窗口计数 | 简单 | 有临界突变 |
| 滑动窗口 | 滑动时间窗口计数 | 平滑 | 实现复杂 |
| 令牌桶 | 按固定速率放令牌 | 允许突发 | 实现复杂 |
| 漏桶 | 恒定速率消费 | 完全平滑 | 不允许突发 |
| Sentinel | 滑动窗口 + 冷启动 | 功能完善 | 引入依赖 |

## 削峰方案

```
瞬时高峰                    削峰后
                   
10000 QPS ─────┐     2000 QPS ──→ 数据库
 (瞬时)        │     (平稳)
               ▼
          ┌────────┐
          │  MQ    │ ← 消息队列作为缓冲
          └────────┘
```

削峰的核心思路：**用空间换时间。** 把瞬时洪峰存入消息队列，后端按自己的节奏慢慢消费。

## 核心公式

```
流量控制 = 限流（入口控制）+ 削峰（队列缓冲）+ 降级（功能取舍）+ 熔断（故障隔离）

Sentinel 三剑客：
-限流规则（flow）→ 控制入口流量
-降级规则（degrade）→ 异常比例/响应时间降级
-系统规则（system）→ 根据系统 Load/QPS/响应时间自适应限流

组合策略：Sentinel 限流 + MQ 削峰 + 降级兜底 = 系统韧性
```
