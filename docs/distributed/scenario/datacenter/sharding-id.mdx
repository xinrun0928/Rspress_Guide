# 全局递增 ID 在分库分表中的实现

数据库分库分表后，主键 ID 怎么生成？如果用数据库自增 ID，不同分片之间会冲突；如果用 UUID，太长且无序。

全局递增 ID，是分布式系统的基础组件。

## 为什么需要全局 ID？

### 分库分表后的问题

```

│                    分库分表后的 ID 问题                           │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
```

### 全局 ID 的要求

| 要求 | 说明 | 重要性 |
|---|---|---|
| 全局唯一 | 不同分片生成的 ID 不能冲突 | 必须 |
| 趋势递增 | 新生成的 ID 大于旧的，利于索引 | 高 |
| 高性能 | 每秒可生成数万甚至百万个 ID | 高 |
| 高可用 | 服务挂掉不影响业务 | 高 |
| 可反解 | 从 ID 可提取时间、分片信息 | 中 |
| 交互友好 | ID 是数字，不是超长字符串 | 中 |

## 方案一：数据库号段模式

### 原理

```

│                      号段模式原理                                 │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│   ID 生成器服务                                                   │

│   │  号段表        │                                             │

│          │                                                        │
│          │ 每次取 step=1000                                      │

│   ┌─────────────────────────────────────┐                        │

│   │  next() -> 1  │                                             │

│   │  ...          │                                             │
│   │  next() -> 1000 (触发重新获取)      │                        │
```

### 服务端实现

```java
@Service
public class DistributedIdGenerator {

    private final JdbcTemplate jdbcTemplate;
    private final Map<String, AtomicReference<SegmentId>> segmentCache = new ConcurrentHashMap<>();

    /**
     * 获取下一个 ID
     */
    public long nextId(String bizTag) {
        // 1. 获取或创建号段
        SegmentId segment = segmentCache.computeIfAbsent(bizTag, k -> {
            return new AtomicReference<>(fetchNextSegment(bizTag));
        });

        // 2. 尝试获取 ID
        while (true) {
            long current = segment.get().current;
            long max = segment.get().max;

            if (current < max) {
                // 当前号段还有 ID
                if (segment.compareAndSet(
                        segment.get(),
                        new Segment(segment.get().value, current + 1, max))) {
                    return current + 1;
                }
            } else {
                // 号段耗尽，获取新号段
                SegmentId newSegment = fetchNextSegment(bizTag);
                segment.set(newSegment);
            }
        }
    }

    /**
     * 从数据库获取新号段
     */
    private Segment fetchNextSegment(String bizTag) {
        // 方式一：使用 UPDATE + SELECT（简单但不安全）
        jdbcTemplate.update(
            "UPDATE sequence SET max_id = max_id + step WHERE biz_tag = ?",
            bizTag
        );

        Map<String, Object> result = jdbcTemplate.queryForMap(
            "SELECT max_id, step FROM sequence WHERE biz_tag = ?",
            bizTag
        );

        long maxId = ((Number) result.get("max_id")).longValue();
        int step = ((Number) result.get("step")).intValue();

        return new Segment(maxId - step + 1, maxId, maxId);

        // 方式二：使用 CAS 更新（推荐）
        // 见下面的实现
    }

    /**
     * 使用 CAS 获取新号段（线程安全）
     */
    private synchronized Segment fetchNextSegmentWithCAS(String bizTag) {
        // 双重检查
        SegmentId currentSegment = segmentCache.get(bizTag);
        if (currentSegment != null && currentSegment.get().current < currentSegment.get().max) {
            return currentSegment.get();
        }

        // 使用乐观锁更新
        int retryCount = 3;
        while (retryCount > 0) {
            // 1. 获取当前值
            Map<String, Object> current = jdbcTemplate.queryForMap(
                "SELECT max_id, step FROM sequence WHERE biz_tag = ?",
                bizTag
            );

            long oldMaxId = ((Number) current.get("max_id")).longValue();
            int step = ((Number) current.get("step")).intValue();
            long newMaxId = oldMaxId + step;

            // 2. CAS 更新
            int updated = jdbcTemplate.update(
                "UPDATE sequence SET max_id = ?, version = version + 1 " +
                "WHERE biz_tag = ? AND version = ?",
                newMaxId, bizTag, oldMaxId
            );

            if (updated == 1) {
                // 更新成功
                return new Segment(oldMaxId + 1, newMaxId, newMaxId);
            }

            // 更新失败，重试
            retryCount--;
            try {
                Thread.sleep(10);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }

        throw new RuntimeException("获取号段失败");
    }
}

@Data
class Segment {
    private final long value;    // 号段起始值
    private volatile long current;
    private final long max;

    public Segment(long current, long max, long value) {
        this.current = current;
        this.max = max;
        this.value = value;
    }
}
```

### 客户端实现（高性能）

```java
/**
 * 高性能 ID 生成器（客户端缓存号段）
 */
public class SegmentIdGenerator {

    private final String bizTag;
    private final IdGeneratorService idGeneratorService;

    // 本地缓存
    private volatile Segment currentSegment;
    private final Object lock = new Object();

    public SegmentIdGenerator(String bizTag, IdGeneratorService idGeneratorService) {
        this.bizTag = bizTag;
        this.idGeneratorService = idGeneratorService;
        this.currentSegment = new Segment(0, 0, 0);
    }

    /**
     * 获取下一个 ID
     */
    public long nextId() {
        while (true) {
            long current = currentSegment.current;
            long max = currentSegment.max;

            if (current < max) {
                return ++current;
            }

            // 号段耗尽，获取新号段
            synchronized (lock) {
                if (current >= currentSegment.max) {
                    // 再次检查，可能其他线程已经更新
                    currentSegment = idGeneratorService.fetchNextSegment(bizTag);
                }
            }
        }
    }

    /**
     * 批量获取 ID
     */
    public long[] nextIds(int count) {
        long[] ids = new long[count];
        for (int i = 0; i < count; i++) {
            ids[i] = nextId();
        }
        return ids;
    }
}
```

## 方案二：雪花算法（Snowflake）

### 原理

```
┌──────────────────────────────────────────────────────────────────┐
│                      雪花算法 ID 结构                             │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  符号位   │   时间戳（41位）   │  机器ID（10位） │ 序列号（12位）│ │
│  │  1 bit   │    41 bits        │    10 bits    │   12 bits   │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                                                                   │
│  时间戳：使用相对时间，从某个 epoch 开始                            │
│          41 位可以表示约 69 年的时间                                │
│                                                                   │
│  机器ID：数据中心（5位）+ 机器ID（5位）                            │
│          最多支持 32 个数据中心，每个数据中心 32 台机器             │
│                                                                   │
│  序列号：每毫秒内递增，每毫秒最多 4096 个                          │
│                                                                   │
│  优势：高性能，不依赖数据库                                       │
│  劣势：依赖时钟，如果时钟回拨会冲突                                │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### Java 实现

```java
public class SnowflakeIdGenerator {

    // 开始时间截（2020-01-01 00:00:00）
    private final long epoch = 1577836800000L;

    // 机器ID占的位数
    private final long workerIdBits = 5L;
    // 数据中心ID占的位数
    private final long dataCenterIdBits = 5L;
    // 序列号占的位数
    private final long sequenceBits = 12L;

    // 机器ID能拥有的最大值
    private final long maxWorkerId = ~(-1L << workerIdBits);
    // 数据中心ID能拥有的最大值
    private final long maxDataCenterId = ~(-1L << dataCenterCenterIdBits);

    // 机器ID左移位数
    private final long workerIdShift = sequenceBits;
    // 数据中心ID左移位数
    private final long dataCenterIdShift = sequenceBits + workerIdBits;
    // 时间截左移位数
    private final long timestampLeftShift = sequenceBits + workerIdBits + dataCenterIdBits;
    // 序列号掩码
    private final long sequenceMask = ~(-1L << sequenceBits);

    private final long workerId;
    private final long dataCenterId;
    private long sequence = 0L;
    private long lastTimestamp = -1L;

    public SnowflakeIdGenerator(long workerId, long dataCenterId) {
        if (workerId > maxWorkerId || workerId < 0) {
            throw new IllegalArgumentException("workerId 不能大于 " + maxWorkerId + " 或小于 0");
        }
        if (dataCenterId > maxDataCenterId || dataCenterId < 0) {
            throw new IllegalArgumentException("dataCenterId 不能大于 " + maxDataCenterId + " 或小于 0");
        }

        this.workerId = workerId;
        this.dataCenterId = dataCenterId;
    }

    /**
     * 生成下一个 ID
     */
    public synchronized long nextId() {
        long timestamp = timeGen();

        // 时钟回拨检测
        if (timestamp < lastTimestamp) {
            throw new RuntimeException(
                "时钟回拨了 " + (lastTimestamp - timestamp) + " 毫秒"
            );
        }

        // 同一毫秒内，重置序列号
        if (lastTimestamp == timestamp) {
            sequence = (sequence + 1) & sequenceMask;
            if (sequence == 0) {
                // 序列号溢出，等待下一毫秒
                timestamp = tilNextMillis(lastTimestamp);
            }
        } else {
            sequence = 0L;
        }

        lastTimestamp = timestamp;

        return ((timestamp - epoch) << timestampLeftShift)
            | (dataCenterId << dataCenterIdShift)
            | (workerId << workerIdShift)
            | sequence;
    }

    /**
     * 等待下一毫秒
     */
    protected long tilNextMillis(long lastTimestamp) {
        long timestamp = timeGen();
        while (timestamp <= lastTimestamp) {
            timestamp = timeGen();
        }
        return timestamp;
    }

    /**
     * 获取当前时间戳
     */
    protected long timeGen() {
        return System.currentTimeMillis();
    }

    /**
     * 从 ID 中反解各部分
     */
    public static IdInfo parse(long id) {
        long timestamp = (id >> timestampLeftShift) + epoch;
        long dataCenterId = (id >> dataCenterIdShift) & maxDataCenterId;
        long workerId = (id >> workerIdShift) & maxWorkerId;
        long sequence = id & sequenceMask;

        return new IdInfo(timestamp, dataCenterId, workerId, sequence);
    }
}

@Data
@AllArgsConstructor
class IdInfo {
    private long timestamp;
    private long dataCenterId;
    private long workerId;
    private long sequence;
}
```

### 时钟回拨处理

```java
/**
 * 支持时钟回拨处理的雪花算法
 */
public class RobustSnowflakeIdGenerator extends SnowflakeIdGenerator {

    // 使用 Redis 记录上次生成的时间戳
    private final RedisTemplate<String, Object> redisTemplate;
    private final String workerId;
    private final String timeKey;

    public RobustSnowflakeIdGenerator(
            long workerId,
            long dataCenterId,
            RedisTemplate<String, Object> redisTemplate) {
        super(workerId, dataCenterId);
        this.redisTemplate = redisTemplate;
        this.workerId = String.valueOf(workerId);
        this.timeKey = "snowflake:last_time:" + workerId;
    }

    @Override
    protected long tilNextMillis(long lastTimestamp) {
        long timestamp = timeGen();

        // 如果当前时间仍然小于上次时间，说明时钟回拨
        while (timestamp <= lastTimestamp) {
            // 1. 从 Redis 获取允许的最大时间
            Long maxAllowedTime = (Long) redisTemplate.opsForValue().get(timeKey);

            if (maxAllowedTime != null && timestamp < maxAllowedTime) {
                // 时钟回拨了，等待追上
                try {
                    Thread.sleep(maxAllowedTime - timestamp + 1);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                timestamp = timeGen();
            } else {
                // 直接使用 lastTimestamp + 1
                break;
            }
        }

        return timestamp;
    }

    @Override
    protected long timeGen() {
        return System.currentTimeMillis();
    }
}
```

## 方案三：UidGenerator（百度）

```java
/**
 * 百度 UidGenerator：改进版雪花算法
 * 使用 RingBuffer 缓存 ID，支持批量获取
 */
@Service
public class BaiduUidGenerator {

    private final UidGenerator uidGenerator;

    @PostConstruct
    public void init() {
        // 配置
        CachedUidGenerator generator = new CachedUidGenerator();
        generator.setTimeBits(29);      // 时间戳：2^29 / (365*24*3600) ≈ 17 年
        generator.setWorkerBits(21);   // 工作机器：2^21 = 209 万台
        generator.setSeqBits(13);      // 序列号：2^13 = 8192
        generator.setEpochStr("2020-01-01");  // 起始时间

        // 设置 RingBuffer 大小（必须为 2^n）
        generator.setBufferSize(6000);

        // 拒绝策略：当 Buffer 满时的处理
        generator.setRejectedPutHandler((uid, ringBuffer) -> {
            // 拒绝策略：等待
            try {
                Thread.sleep(50);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        generator.setRejectedTakeHandler(ringBuffer -> {
            // 获取不到时的策略：生成后备 ID
            return generateSimpleUid();
        });

        generator.afterPropertiesSet();

        this.uidGenerator = generator;
    }

    /**
     * 获取下一个 ID
     */
    public long nextId() {
        return uidGenerator.getUID();
    }

    /**
     * 解析 ID
     */
    public String parseUID(long uid) {
        return uidGenerator.parseUID(uid);
    }

    /**
     * 简单兜底方案
     */
    private long generateSimpleUid() {
        return SnowflakeIdGeneratorHolder.INSTANCE.nextId();
    }
}
```

## 方案对比与选型

| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 号段模式 | 不依赖时钟，高可用 | 需要数据库 | 数据量中等（< 10万/秒） |
| 雪花算法 | 性能高，不依赖数据库 | 依赖时钟 | 数据量大（> 10万/秒） |
| 百度 UidGenerator | 批量获取，低延迟 | 依赖组件 | 超高并发场景 |

```java
// 对 ID 进行加密，避免暴露真实数据量
public class SecureIdGenerator {

    private final IdGenerator idGenerator;
    private final AesCipher aesCipher;

    public SecureIdGenerator(IdGenerator idGenerator, String secretKey) {
        this.idGenerator = idGenerator;
        this.aesCipher = new AesCipher(secretKey);
    }

    public String nextSecureId() {
        long id = idGenerator.nextId();
        // 使用 AES 加密，但只取固定位数
        String encrypted = aesCipher.encrypt(String.valueOf(id));
        return encrypted.substring(0, 16);
    }
}
```

## 总结

全局 ID 生成是分布式系统的核心组件：

| 方案 | QPS | 延迟 | 依赖 |
|---|---|---|---|
| 号段模式 | 10万+ | < 1ms | MySQL |
| 雪花算法 | 100万+ | < 1ms | 无 |
| UidGenerator | 1000万+ | < 0.1ms | 无 |

选型建议：

- **初期业务**：号段模式，简单可靠
- **中等规模**：雪花算法，性能好
- **超大规模**：UidGenerator 或自研

记住：**ID 生成不是瓶颈，业务才是**。不要过度设计，选择能满足当前需求的方案即可。
