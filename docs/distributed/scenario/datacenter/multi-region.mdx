# 异地多活架构：单元化部署与数据同步方案

2015 年，携程遭遇了一次大规模服务中断，整个系统瘫痪了近 12 小时。这次事故的教训是：**单机房部署，无法应对区域级的灾难**。

异地多活，成了大型互联网系统的标配。

## 什么是异地多活？

### 架构演进

```
┌──────────────────────────────────────────────────────────────────┐
│                      架构演进历程                                 │
├──────────────────────────────────────────────────────────────────┤
```

### 多活的关键指标

| 指标 | 说明 | 目标值 |
|---|---|---|
| RTO（Recovery Time Objective） | 故障恢复时间 | < 30 分钟 |
| RPO（Recovery Point Objective） | 数据恢复点 | < 5 分钟 |
| 单元独立性 | 单个单元故障不影响其他单元 | 100% |
| 流量切换时间 | 故障时切换流量耗时 | < 5 分钟 |

## 单元化架构设计

### 单元的定义

单元（Unit）是能够独立运行、业务闭环的最小部署单元。每个单元包含：

- **完整的应用层**：所有业务服务都在单元内
- **完整的数据层**：独立的数据库实例
- **完整的中间件**：消息队列、缓存等
- **独立的入口**：独立的 VIP 或 DNS

### 单元划分规则

```java
/**
 * 用户单元归属计算
 * 核心原则：同一用户的读写操作必须在同一单元内
 */
public class UnitRouter {

    private static final int UNIT_COUNT = 3;  // 单元数量

    /**
     * 根据用户 ID 计算归属单元
     */
    public static int calculateUnit(String userId) {
        // 方法一：哈希取模
        int hash = Math.abs(userId.hashCode());
        return hash % UNIT_COUNT;

        // 方法二：城市代码分区
        // String cityCode = getCityCode(userId);
        // return CITY_TO_UNIT_MAPPING.get(cityCode);
    }

    /**
     * 判断请求是否在正确单元
     */
    public static boolean isInCorrectUnit(String userId, String currentUnit) {
        int correctUnit = calculateUnit(userId);
        return currentUnit.equals("unit-" + correctUnit);
    }

    /**
     * 获取单元名称
     */
    public static String getUnitName(int unitIndex) {
        return "unit-" + unitIndex;
    }
}
```

### 网关路由层

```java
@Component
public class UnitAwareRouteFilter extends ZuulFilter {

    private final UnitRouter unitRouter;

    @Override
    public String filterType() {
        return "route";
    }

    @Override
    public int filterOrder() {
        return 10;
    }

    @Override
    public boolean shouldFilter() {
        return true;
    }

    @Override
    public Object run() {
        RequestContext ctx = RequestContext.getCurrentContext();
        HttpServletRequest request = ctx.getRequest();

        // 1. 获取用户 ID（从 Token 或 Cookie）
        String userId = extractUserId(request);
        String requestUnit = extractUnit(request);

        // 2. 计算用户应该在哪一单元
        int correctUnitIndex = unitRouter.calculateUnit(userId);
        String correctUnit = unitRouter.getUnitName(correctUnitIndex);

        // 3. 如果请求不在正确单元，重定向
        if (!correctUnit.equals(requestUnit)) {
            String targetUrl = buildTargetUrl(correctUnit, request);
            ctx.setSendZuulResponse(false);
            ctx.setResponseStatusCode(302);
            ctx.addOriginResponseHeader("Location", targetUrl);
            return null;
        }

        // 4. 记录单元信息到请求上下文
        ctx.addZuulRequestHeader("X-Unit-Id", correctUnit);
        return null;
    }

    private String extractUserId(HttpServletRequest request) {
        // 从 JWT Token 中获取用户 ID
        String token = request.getHeader("Authorization");
        if (token != null && token.startsWith("Bearer ")) {
            String userId = JwtUtil.getUserId(token.substring(7));
            return userId;
        }

        // 或从 Cookie 中获取
        Cookie userIdCookie = findCookie(request, "user_id");
        return userIdCookie != null ? userIdCookie.getValue() : null;
    }

    private String extractUnit(HttpServletRequest request) {
        Cookie unitCookie = findCookie(request, "unit_id");
        if (unitCookie != null) {
            return unitCookie.getValue();
        }

        // 从域名解析
        String host = request.getHeader("Host");
        return parseUnitFromHost(host);
    }
}
```

### 基于 Canal 的数据同步

```java
/**
 * Canal 客户端：订阅 MySQL Binlog，实现数据同步
 */
@Service
public class CanalDataSyncService {

    private final CanalConnector connector;
    private final DataSyncService dataSyncService;
    private final ElasticsearchTemplate esTemplate;
    private final RedisTemplate<String, Object> redisTemplate;

    /**
     * 启动数据同步
     */
    @PostConstruct
    public void startSync() {
        // 1. 连接 Canal Server
        connector = CanalConnectors.newClusterConnector(
            "canal-server:11111",  // Canal 集群地址
            "destination",          // 目的地名称
            "",                    // 用户名
            ""                     // 密码
        );

        connector.connect();
        connector.subscribe(".*\\..*");  // 订阅所有表

        // 2. 启动同步线程
        new Thread(this::syncLoop).start();
    }

    /**
     * 同步循环
     */
    private void syncLoop() {
        while (true) {
            try {
                // 1. 获取批量消息
                Message message = connector.get(batchSize);

                if (message.getEntries().isEmpty()) {
                    Thread.sleep(1000);  // 没有数据，等待
                    continue;
                }

                // 2. 处理每个 Entry
                for (Entry entry : message.getEntries()) {
                    if (entry.getEntryType() == EntryType.ROWDATA) {
                        processRowData(entry);
                    }
                }

                // 3. 确认已消费
                connector.ack();

            } catch (Exception e) {
                log.error("同步异常", e);
                connector.rollback();

                try {
                    Thread.sleep(5000);  // 失败后等待重试
                } catch (InterruptedException ie) {
                    Thread.currentThread().interrupt();
                }
            }
        }
    }

    /**
     * 处理行数据变化
     */
    private void processRowData(Entry entry) {
        try {
            RowChange rowChange = RowChange.parseFrom(entry.getStoreValue());

            for (RowData rowData : rowChange.getRowDatasList()) {
                String tableName = entry.getHeader().getTableName();

                switch (rowChange.getEventType()) {
                    case INSERT:
                        handleInsert(tableName, rowData);
                        break;
                    case UPDATE:
                        handleUpdate(tableName, rowData);
                        break;
                    case DELETE:
                        handleDelete(tableName, rowData);
                        break;
                }
            }

        } catch (Exception e) {
            log.error("处理行数据失败", e);
        }
    }

    private void handleInsert(String tableName, RowData rowData) {
        Map<String, String> data = parseRow(rowData.getAfterColumnsList());

        switch (tableName) {
            case "user":
                // 同步到 ES
                User user = convertToUser(data);
                esTemplate.save(user);
                // 更新 Redis 缓存
                redisTemplate.opsForValue().set("user:" + user.getId(), user);
                break;
            case "product":
                // 同步商品数据
                break;
        }
    }

    private void handleUpdate(String tableName, RowData rowData) {
        Map<String, String> before = parseRow(rowData.getBeforeColumnsList());
        Map<String, String> after = parseRow(rowData.getAfterColumnsList());

        String id = after.getOrDefault("id", before.get("id"));

        switch (tableName) {
            case "user":
                User user = convertToUser(after);
                esTemplate.save(user);
                redisTemplate.opsForValue().set("user:" + id, user);
                break;
        }
    }

    private void handleDelete(String tableName, RowData rowData) {
        Map<String, String> data = parseRow(rowData.getBeforeColumnsList());
        String id = data.get("id");

        switch (tableName) {
            case "user":
                esTemplate.delete("user", id);
                redisTemplate.delete("user:" + id);
                break;
        }
    }
}
```

```java
@Service
public class UnitDataSyncService {

    private final KafkaTemplate<String, UnitDataMessage> kafkaTemplate;
    private final RedisTemplate<String, Object> redisTemplate;

    /**
     * 发送跨单元同步消息
     */
    public void syncToAllUnits(String dataType, Object data) {
        UnitDataMessage message = new UnitDataMessage()
            .setId(UUID.randomUUID().toString())
            .setDataType(dataType)
            .setData(data)
            .setTimestamp(System.currentTimeMillis())
            .setSourceUnit(getCurrentUnit());

        // 发送到所有单元（除本单元外）
        List<String> targetUnits = getAllUnitsExceptCurrent();
        for (String targetUnit : targetUnits) {
            kafkaTemplate.send(
                "unit-sync-" + targetUnit,
                message.getId(),
                message
            );
        }
    }

    /**
     * 处理跨单元同步消息
     */
    @KafkaListener(topics = "unit-sync-*", groupId = "unit-sync-consumer")
    public void handleSyncMessage(UnitDataMessage message) {
        // 幂等处理：检查是否已处理
        String key = "sync:processed:" + message.getId();
        Boolean processed = redisTemplate.hasKey(key);
        if (Boolean.TRUE.equals(processed)) {
            return;
        }

        // 处理数据
        switch (message.getDataType()) {
            case "USER_UPDATE":
                handleUserUpdate(message.getData());
                break;
            case "ACCOUNT_CHANGE":
                handleAccountChange(message.getData());
                break;
            case "CONFIG_CHANGE":
                handleConfigChange(message.getData());
                break;
        }

        // 标记已处理
        redisTemplate.opsForValue().set(key, "1", Duration.ofDays(7));
    }
}
```

### 流量切换流程

```java
/**
 * 单元故障切换服务
 */
@Service
public class UnitFailoverService {

    private final LoadBalancer loadBalancer;
    private final RedisTemplate<String, Object> redisTemplate;
    private final NamingService namingService;  // Nacos 等服务发现

    /**
     * 执行单元切换
     */
    public void switchUnit(String fromUnit, String toUnit) {
        // 1. 更新路由配置
        updateRouteConfig(fromUnit, toUnit);

        // 2. 更新 DNS 解析
        updateDnsRouting(fromUnit, toUnit);

        // 3. 通知所有服务
        notifyServicesUnitSwitch(fromUnit, toUnit);

        // 4. 记录切换事件
        recordFailoverEvent(fromUnit, toUnit);
    }

    /**
     * 更新路由配置
     */
    private void updateRouteConfig(String fromUnit, String toUnit) {
        // 1. 更新本单元的路由规则：故障单元的用户路由到目标单元
        String routingKey = "unit:routing:" + fromUnit;
        Map<String, Object> routing = new HashMap<>();
        routing.put("redirectTo", toUnit);
        routing.put("updatedAt", System.currentTimeMillis());
        redisTemplate.opsForHash().putAll("global:routing", routing);

        // 2. 发布路由变更事件
        publishRouteChangeEvent(fromUnit, toUnit);
    }

    /**
     * DNS 切换
     */
    private void updateDnsRouting(String fromUnit, String toUnit) {
        // 获取故障单元的域名
        String fromDomain = fromUnit + ".yourdomain.com";

        // 解析到目标单元的 IP
        String toIp = namingService.getHealthyInstance(
            "api-service",
            toUnit
        ).getIp();

        // 更新 DNS 记录（使用 DNSPod 或阿里云 DNS API）
        dnsClient.updateRecord(
            UpdateRecordRequest.builder()
                .DomainName("yourdomain.com")
                .RR(fromUnit)
                .Type("A")
                .Value(toIp)
                .TTL(60)  // 短 TTL，加快生效
                .build()

    }
```

```java
/**
 * 单元故障恢复后的回滚
 */
@Service
public class UnitRollbackService {

    /**
     * 回滚到原单元
     */
    public void rollbackToOriginal(String originalUnit, String currentUnit) {
        // 1. 确认原单元已恢复
        if (!isUnitHealthy(originalUnit)) {
            throw new IllegalStateException("原单元尚未恢复，不能回滚");
        }

        // 2. 逐步切回流量（灰度）
        int percentage = 10;
        while (percentage <= 100) {
            // 按百分比逐步将流量切回原单元
            gradualSwitch(originalUnit, percentage);
            percentage += 10;

            // 等待并监控
            monitorMetrics(originalUnit);
            sleep(30000);  // 每步等待 30 秒
        }

        // 3. 恢复 DNS
        restoreDns(originalUnit);
    }

    /**
     * 逐步切换流量
     */
    private void gradualSwitch(String targetUnit, int percentage) {
        String key = "unit:traffic:ratio:" + targetUnit;
        redisTemplate.opsForValue().set(key, percentage);

        log.info("流量切换到 {}: {}%", targetUnit, percentage);
    }
}
```

### 最终一致性策略

```java
/**
 * 单元化数据一致性处理
 */
@Service
public class UnitConsistencyService {

    /**
     * 处理分布式事务
     * 使用 TCC（Try-Confirm-Cancel）模式
     */
    @Transactional
    public void transferBetweenUnits(String fromUserId, String toUserId, BigDecimal amount) {
        // 本地单元：扣减余额
        String fromUnit = UnitRouter.getUnitName(UnitRouter.calculateUnit(fromUserId));
        deductBalance(fromUserId, amount);

        // 远程单元：增加余额
        String toUnit = UnitRouter.getUnitName(UnitRouter.calculateUnit(toUserId));

        if (!fromUnit.equals(toUnit)) {
            // 发送跨单元消息
            CompletableFuture<Boolean> future = sendAddBalanceMessage(toUserId, amount);

            // 如果发送失败，回滚本地
            if (!future.join()) {
                throw new BusinessException("转账失败");
            }
        } else {
            addBalance(toUserId, amount);
        }
    }

    /**
     * 补偿机制
     */
    @Scheduled(fixedRate = 60000)  // 每分钟检查
    public void checkAndCompensate() {
        // 1. 查询未完成的分布式事务
        List<PendingTransaction> pendingTxns = transactionRepository.findPending();

        for (PendingTransaction txn : pendingTxns) {
            // 2. 根据事务状态执行补偿
            switch (txn.getStatus()) {
                case PENDING:
                    // 超时未完成，查询实际状态
                    checkTransactionStatus(txn);
                    break;
                case CONFIRM_FAILED:
                    // 确认失败，重试确认
                    retryConfirm(txn);
                    break;
                case CANCEL_FAILED:
                    // 取消失败，重试取消
                    retryCancel(txn);
                    break;
            }
        }
    }
}
```

## 常见问题与解决

### 问题一：单元划分不均匀

```java
// 使用一致性哈希减少数据倾斜
public class ConsistentHashUnitRouter {

    private final TreeMap<Long, String> ring = new TreeMap<>();
    private final int virtualNodes;

    public ConsistentHashUnitRouter(List<String> units, int virtualNodes) {
        this.virtualNodes = virtualNodes;
        for (String unit : units) {
            addUnit(unit);
        }
    }

    public void addUnit(String unit) {
        for (int i = 0; i < virtualNodes; i++) {
            long hash = hash(unit + ":" + i);
            ring.put(hash, unit);
        }
    }

    public String route(String userId) {
        long hash = hash(userId);
        Map.Entry<Long, String> entry = ring.ceilingEntry(hash);
        return entry != null ? entry.getValue() : ring.firstEntry().getValue();
    }

    private long hash(String key) {
        return MD5.hash(key).getLong();
    }
}
```

| 单元化部署 | 业务闭环，独立运行 |
| 用户路由 | 确保用户访问正确单元 |
| 数据同步 | 保持数据最终一致 |
| 故障切换 | 快速恢复服务 |

实施异地多活的三个阶段：

1. **同城双活**：先验证架构可行性
2. **单元化改造**：业务代码按单元改造
3. **异地多活**：逐步扩展到多个城市

记住：**不是所有系统都需要异地多活**。它的复杂度很高，只有当业务规模达到一定程度、数据可靠性要求极高时，才值得投入。
