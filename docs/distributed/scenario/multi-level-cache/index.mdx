# 多级缓存：本地缓存 → Redis → 数据库

你有没有注意过一个现象？

双十一零点抢购，页面刷不出来，系统没崩，数据库也活着——但用户就是进不去页面。

**问题是：数据库承受不住那么多请求。**

如果每个请求都直接打数据库，再好的数据库也会跪。而多级缓存要解决的，就是这个问题。

## 为什么需要多级缓存

每往上一层缓存，数据访问就快一个数量级：

```
┌─────────────────────────────────────────────────────────┐
│                 缓存层级性能对比                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  L1 本地缓存（Caffeine / Guava）                        │
│  └─ 纳秒级，比 Redis 快 1000 倍                        │
│  └─ 适用：极高热点数据                                   │
│                                                         │
│  L2 分布式缓存（Redis）                                 │
│  └─ 微秒级，比数据库快 10-100 倍                       │
│  └─ 适用：准热点数据、跨进程共享                        │
│                                                         │
│  L3 数据库                                              │
│  └─ 毫秒级，最终数据源                                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 缓存三难题

| 难题 | 问题 | 解决方案 |
|------|------|---------|
| 缓存穿透 | 查询不存在的数据直达数据库 | 布隆过滤器 / 空值缓存 |
| 缓存击穿 | 热点 key 过期瞬间大量请求 | 互斥锁 / 永不过期 + 异步更新 |
| 缓存雪崩 | 大量 key 同时过期 | 过期时间随机化 + 热点数据永不过期 |

## 缓存一致性

```
┌─────────────────────────────────────────────────────────┐
│               Cache-Aside 读写策略                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  读流程：                                               │
│  L1 缓存 → 命中返回                                     │
│     ↓ 未命中                                            │
│  L2 缓存 → 命中返回，写入 L1                            │
│     ↓ 未命中                                            │
│  数据库 → 返回数据，写入 L2 + L1                         │
│                                                         │
│  写流程：                                               │
│  更新数据库 → 删除 L2 缓存 → 删除 L1 缓存               │
│  （不是更新缓存，是删除！）                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 核心公式

```
多级缓存 = L1本地缓存 + L2分布式缓存 + 缓存一致性

Cache-Aside 最佳实践：
- 读：Cache → Miss → DB → Cache
- 写：DB → Delete Cache（不是 Update Cache）

三难题应对：
- 穿透：布隆过滤器（不存在一定过滤）
- 击穿：互斥锁 / 永不过期 + 异步更新
- 雪崩：过期时间加随机值（±10%）

核心原则：让每一层缓存各司其职，不是简单堆叠
```
