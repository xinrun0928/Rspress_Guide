# 典型分布式场景方案

理论学了一大堆，最终还是要落地。

这一模块，将分布式理论与工程实践结合，讲解真实生产环境中的典型场景：多级缓存、统一认证、文件存储、多机房部署……

**每一个场景，都是前人踩过的坑。**

## 场景与方案对照

```
┌──────────────────┬──────────────────────────────────────┐
│ 场景             │ 典型方案                              │
├──────────────────┼──────────────────────────────────────┤
│ 高并发读          │ 多级缓存 + 热点探测                   │
│ 高并发写          │ 消息队列削峰 + 分库分表               │
│ 缓存一致性         │ Cache Aside + 延迟双删              │
│ 缓存异常          │ 穿透（布隆）、击穿（互斥）、雪崩（打散）│
│ 统一认证          │ JWT / Spring Session / CAS           │
│ 文件存储          │ MinIO / FastDFS + CDN               │
│ 多机房部署         │ 同城双活 / 异地多活 / 单元化         │
│ 链路追踪          │ TraceId + MDC / Skywalking           │
│ 服务限流          │ Sentinel / 令牌桶 / 滑动窗口           │
└──────────────────┴──────────────────────────────────────┘
```

## 核心公式

```
场景方案 = 问题理解 + 方案选型 + 落地实现 + 权衡取舍

学习方法：
- 多级缓存：理解每一层缓存的意义和取舍
- 统一认证：理解 Session 和 Token 的本质区别
- 文件存储：理解对象存储和文件系统的适用场景
- 多机房部署：理解 CAP 理论在工程中的体现
- 流量控制：理解限流的本质是「保护系统」
- 可观测性：理解 Metrics、Tracing、Logging 的关系

核心原则：场景方案的答案不是唯一的，重要的是理解为什么这样选。
```
