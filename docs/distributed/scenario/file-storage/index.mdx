# 分布式文件存储：从 MinIO 到 CDN 加速

你有没有遇到这种需求？

用户上传头像，1MB，但 100 万用户，存储空间不是问题。

用户上传视频，500MB，1000 个并发上传，存储空间 + 带宽 + 上传速度都是问题。

用户要下载安装包，2GB，全国各地的用户都要能快速下载，传统服务器磁盘 IO 直接打满。

**这就是分布式文件存储要解决的问题。**

## 文件存储的三个层次

```
┌─────────────────────────────────────────────────────────┐
│                  文件存储三层架构                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  第一层：本地存储                                        │
│  └─ 单机磁盘，容量有限，适合临时文件                     │
│                                                         │
│  第二层：分布式对象存储                                  │
│  └─ MinIO / FastDFS / OSS / S3，可横向扩展              │
│                                                         │
│  第三层：CDN 分发                                        │
│  └─ 把文件缓存在全国各地的边缘节点，用户就近下载          │
│                                                         │
│  选择依据：小文件本地存，大文件上对象存储，热文件上CDN   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 主流方案对比

| 方案 | 特点 | 适用场景 | 不适用场景 |
|------|------|---------|-----------|
| MinIO | S3 兼容、自部署、私有化 | 企业内网、混合云 | 需要全托管 |
| FastDFS | 轻量级、小文件优化 | 图片/短视频 | 大文件、冷存储 |
| 阿里云 OSS | 全托管、高可用 | 互联网应用 | 成本敏感 |
| 自建 NAS | 传统文件协议 | 遗留系统 | 容器环境 |

## 文件分片上传原理

大文件上传的核心是**分片**：把一个大文件拆成多个小块，并行上传，每个块独立重试，最后合并。

```
上传流程：

┌─────────────────────────────────────────────────────────┐
│  1. 前端请求上传接口，获取唯一 uploadId                  │
│                                                         │
│  2. 前端对文件分片（例如每块 5MB）                       │
│                                                         │
│  3. 前端并行上传各分片到 OSS（携带 uploadId + 分片序号） │
│                                                         │
│  4. 服务端标记分片上传完成                               │
│                                                         │
│  5. 前端请求合并接口，服务端合并分片                     │
│                                                         │
│  6. 清理临时分片，返回最终文件 URL                       │
└─────────────────────────────────────────────────────────┘

关键点：
- 分片上传可断点续传（从失败的分片继续）
- 并行上传提升速度（充分利用带宽）
- 秒传（服务端计算 MD5，已存在则直接返回 URL）
```

## 核心公式

```
文件存储 = 方案选型 + 分片上传 + CDN 加速 + 权限控制

方案选择：
- 小文件 + 高访问 → FastDFS
- 大文件 + S3兼容 → MinIO
- 不想运维 → 阿里云OSS / 腾讯云COS
- 热数据分发 → CDN（阿里云CDN / Cloudflare）
```
