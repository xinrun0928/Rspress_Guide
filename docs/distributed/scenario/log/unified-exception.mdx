# 微服务统一异常处理：全局异常码 + 标准化响应体

微服务架构下，每个服务都可能抛出异常。如果每个服务各自定义自己的错误码和响应格式，前端调用方将苦不堪言——A 服务返回 `{"code": -1}`, B 服务返回 `{"error": "系统错误"}`, C 服务返回 `"success": false`……

统一异常处理，是微服务走向成熟的第一步。

## 问题分析：微服务异常处理的乱象

```
前端 → 调用 A 服务
    返回：{"code": -1, "msg": "参数错误"}
    
前端 → 调用 B 服务
    返回：{"status": 500, "message": "Internal Server Error"}

前端 → 调用 C 服务
    返回：{"success": false, "reason": "数据不存在"}
    
前端崩溃：到底该解析哪个字段？
```

## 标准化响应体设计

### 统一响应结构

```java
public class ApiResponse<T> {
    
    /** 响应码：0=成功，非0=失败 */
    private int code;
    
    /** 响应消息 */
    private String message;
    
    /** 响应数据 */
    private T data;
    
    /** 请求追踪 ID */
    private String traceId;
    
    /** 响应时间 */
    private long timestamp;
    
    // 工厂方法
    public static <T> ApiResponse<T> success(T data) {
        ApiResponse<T> response = new ApiResponse<>();
        response.code = 0;
        response.message = "success";
        response.data = data;
        response.traceId = TraceContext.getTraceId();
        response.timestamp = System.currentTimeMillis();
        return response;
    }
    
    public static <T> ApiResponse<T> fail(int code, String message) {
        ApiResponse<T> response = new ApiResponse<>();
        response.code = code;
        response.message = message;
        response.traceId = TraceContext.getTraceId();
        response.timestamp = System.currentTimeMillis();
        return response;
    }
    
    // 业务异常
    public static <T> ApiResponse<T> fail(BizException e) {
        return fail(e.getCode(), e.getMessage());
    }
}
```

### 业务异常码体系

```java
// 全局异常码定义
public class GlobalErrorCode {
    
    // 系统级错误（1xxx）
    public static final int SYSTEM_ERROR = 1000;
    public static final int INVALID_PARAM = 1001;
    public static final int REQUEST_TIMEOUT = 1002;
    public static final int SERVICE_UNAVAILABLE = 1003;
    
    // 认证授权错误（2xxx）
    public static final int UNAUTHORIZED = 2000;
    public static final int TOKEN_EXPIRED = 2001;
    public static final int FORBIDDEN = 2003;
    
    // 业务错误（3xxx）
    public static final int USER_NOT_FOUND = 3001;
    public static final int ORDER_NOT_FOUND = 3002;
    public static final int STOCK_NOT_ENOUGH = 3003;
    public static final int DUPLICATE_ORDER = 3004;
}

// 业务异常类
public class BizException extends RuntimeException {
    
    private final int code;
    
    public BizException(int code, String message) {
        super(message);
        this.code = code;
    }
    
    public BizException(GlobalErrorCode errorCode) {
        super(errorCode.message());
        this.code = errorCode.code();
    }
}
```

## 全局异常处理器

Spring Boot 的 `@ControllerAdvice` 注解是全局异常处理的核心：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    // 业务异常
    @ExceptionHandler(BizException.class)
    public ApiResponse<Void> handleBizException(BizException e) {
        log.warn("业务异常: code={}, message={}", e.getCode(), e.getMessage());
        return ApiResponse.fail(e);
    }
    
    // 参数校验异常
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ApiResponse<Void> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
            .map(FieldError::getDefaultMessage)
            .collect(Collectors.joining("; "));
        
        log.warn("参数校验失败: {}", message);
        return ApiResponse.fail(GlobalErrorCode.INVALID_PARAM.code(), message);
    }
    
    // 限流异常
    @ExceptionHandler(FlowLimitException.class)
    public ApiResponse<Void> handleFlowLimitException(FlowLimitException e) {
        log.warn("触发限流: {}", e.getMessage());
        return ApiResponse.fail(GlobalErrorCode.SERVICE_UNAVAILABLE.code(), "系统繁忙，请稍后重试");
    }
    
    // 服务调用异常（Feign / OpenFeign）
    @ExceptionHandler(FeignException.class)
    public ApiResponse<Void> handleFeignException(FeignException e) {
        log.error("Feign 调用失败: status={}, message={}", 
            e.status(), e.getMessage());
        
        if (e.status() == HttpStatus.NOT_FOUND.value()) {
            return ApiResponse.fail(GlobalErrorCode.RESOURCE_NOT_FOUND);
        }
        
        return ApiResponse.fail(GlobalErrorCode.SERVICE_UNAVAILABLE.code(), "服务暂不可用");
    }
    
    // 兜底：未捕获的异常
    @ExceptionHandler(Exception.class)
    public ApiResponse<Void> handleException(Exception e) {
        log.error("未捕获的系统异常", e);
        return ApiResponse.fail(GlobalErrorCode.SYSTEM_ERROR.code(), "系统异常，请联系管理员");
    }
}
```

## 服务间调用的异常传递

微服务之间互相调用时，异常如何在服务链路上正确传递？

### Feign 降级与异常处理

```java
// 定义 Feign 客户端时指定降级类
@FeignClient(
    name = "user-service",
    fallbackFactory = UserServiceFallbackFactory.class,
    configuration = FeignConfiguration.class
)
public interface UserServiceClient {
    
    @GetMapping("/user/{id}")
    ApiResponse<User> getUser(@PathVariable("id") Long id);
}

// 降级工厂
@Component
public class UserServiceFallbackFactory implements FallbackFactory<UserServiceClient> {
    
    @Override
    public UserServiceClient create(Throwable cause) {
        return new UserServiceClient() {
            @Override
            public ApiResponse<User> getUser(Long id) {
                log.error("调用 UserService 失败，降级返回", cause);
                return ApiResponse.fail(
                    GlobalErrorCode.SERVICE_UNAVAILABLE.code(), 
                    "用户服务暂时不可用"
                );
            }
        };
    }
}
```

### 远程异常的正确传递

当 A 服务调用 B 服务时，B 服务的异常应该被 A 服务识别并重新抛出：

```java
// A 服务调用 B 服务
@Service
public class OrderService {
    
    @Autowired
    private UserServiceClient userService;
    
    public void createOrder(CreateOrderRequest request) {
        ApiResponse<User> userResponse = userService.getUser(request.getUserId());
        
        if (userResponse.getCode() != 0) {
            // 识别远程服务返回的业务异常
            throw new BizException(
                userResponse.getCode(), 
                userResponse.getMessage()
            );
        }
        
        // 正常逻辑
        User user = userResponse.getData();
    }
}
```

## 异常日志与追踪

```java
// 异常日志切面：自动记录异常上下文
@Aspect
@Component
public class ExceptionLogAspect {
    
    @Around("execution(* com.example..*Controller.*(..))")
    public Object logException(ProceedingJoinPoint point) throws Throwable {
        try {
            return point.proceed();
        } catch (Exception e) {
            // 记录完整的异常上下文
            log.error("API 异常: method={}, args={}, traceId={}, error={}",
                point.getSignature().getName(),
                JsonUtil.toJson(point.getArgs()),
                TraceContext.getTraceId(),
                ExceptionUtil.getRootCauseMessage(e)
            );
            throw e;
        }
    }
}
```

## 总结

微服务统一异常处理的核心是**标准化**：

1. **响应体标准化**：所有服务返回统一格式的 `ApiResponse`
2. **异常码标准化**：全局定义异常码，不同服务复用
3. **异常处理标准化**：全局异常处理器统一捕获
4. **服务间异常传递标准化**：远程调用时正确识别并传递异常

统一异常处理不是一次性的工作，而是需要在实践中不断完善异常码体系，让前端和调用方有据可依。
