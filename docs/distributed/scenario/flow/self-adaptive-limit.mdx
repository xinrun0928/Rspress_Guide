# 系统自适应限流：Sentinel 与 Prometheus 联动

你的系统平时运行平稳，但某天突然遭遇流量高峰。问题是：限流阈值应该设多少？设高了怕压垮系统，设低了又浪费资源。

自适应限流来了——让系统自己告诉你，应该限多少。

### 自适应限流的思路

自适应限流的核心是**根据系统当前状态动态调整限流阈值**。

## Sentinel 自适应限流

### Sentinel 核心概念

| 概念 | 说明 |
|---|---|
| Resource | 资源，即被保护的业务代码 |
| Rule | 规则，限流、降级、熔断的规则 |
| Node | 统计节点，收集资源调用数据 |
| Entry | 入口，每次调用资源的入口点 |
| Slot | 插槽，处理不同功能（统计、限流、降级） |

### Sentinel 规则配置

```java
@Configuration
public class SentinelConfig {

    /**
     * 初始化 Sentinel 规则
     */
    @PostConstruct
    public void initSentinelRules() {
        initFlowRules();      // 限流规则
        initDegradeRules();   // 降级规则
        initSystemRules();    // 系统规则
    }

    /**
     * 流控规则
     */
    private void initFlowRules() {
        List<FlowRule> rules = new ArrayList<>();

        // 规则 1：针对 /api/user 的流控
        FlowRule userRule = new FlowRule();
        userRule.setResource("getUser");
        userRule.setCount(100);                    // 阈值
        userRule.setGrade(RuleConstant.FLOW_GRADE_QPS);  // QPS 模式
        userRule.setLimitApp("default");
        userRule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);
        rules.add(userRule);

        // 规则 2：针对 /api/order 的流控
        FlowRule orderRule = new FlowRule();
        orderRule.setResource("createOrder");
        orderRule.setCount(50);                     // 阈值
        orderRule.setGrade(RuleConstant.FLOW_GRADE_QPS);
        orderRule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_WARM_UP);  // 预热模式
        orderRule.setWarmUpPeriodSec(10);           // 预热时长
        rules.add(orderRule);

        FlowRuleManager.loadRules(rules);
    }

    /**
     * 降级规则
     */
    private void initDegradeRules() {
        List<DegradeRule> rules = new ArrayList<>();

        DegradeRule rule = new DegradeRule();
        rule.setResource("remoteCall");
        rule.setCount(5);                           // 异常数
        rule.setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_COUNT);  // 按异常数
        rule.setTimeWindow(10);                     // 时间窗口
        rule.setMinRequestAmount(5);                // 最小请求数
        rule.setStatIntervalMs(60000);              // 统计时长 1 分钟
        rules.add(rule);

        DegradeRuleManager.loadRules(rules);
    }

    /**
     * 系统规则（自适应限流的核心）
     */
    private void initSystemRules() {
        List<SystemRule> rules = new ArrayList<>();

        SystemRule rule = new SystemRule();
        rule.setHighestSystemLoad(3.0);              // 系统负载
        rule.setQps(10000);                         // 全局 QPS
        rule.setAvgRT(100);                         // 平均响应时间
        rule.setMaxThread(200);                      // 最大线程数
        rules.add(rule);

        SystemRuleManager.loadRules(rules);
    }
}
```

### Sentinel 使用示例

```java
@Service
public class SentinelProtectedService {

    /**
     * 使用 Sentinel API 保护资源
     */
    public User getUser(Long userId) {
        // 1. 手动定义资源
        try (Entry entry = SphU.entry("getUser")) {
            // 业务逻辑
            return userRepository.findById(userId);

        } catch (BlockException e) {
            // 2. 被限流了
            log.warn("getUser 被限流, userId={}", userId);
            return getUserFallback(userId);
        }
    }

    /**
     * 使用注解方式保护资源
     */
    @SentinelResource(
        value = "createOrder",
        blockHandler = "createOrderBlockHandler",
        fallback = "createOrderFallback",
        blockHandlerClass = SentinelBlockHandler.class,
        fallbackClass = SentinelFallback.class
    )
    public Order createOrder(OrderRequest request) {
        return orderService.create(request);
    }

    /**
     * 限流处理
     */
    public Order createOrderBlockHandler(OrderRequest request, BlockException e) {
        log.warn("createOrder 被限流: {}", e.getClass().getSimpleName());
        return Order.degraded();
    }

    /**
     * 降级处理
     */
    public Order createOrderFallback(OrderRequest request, Throwable t) {
        log.error("createOrder 降级: {}", t.getMessage());
        return Order.fallback();
    }
}
```

```java
/**

@Configuration
public class SentinelMetricsExporter {

    /**
     * 注册 Sentinel 的 MetricFetcher
     */
    @Bean
    public MetricHandler metricHandler() {
        return new MetricHandler();
    }

    /**
     * 定时采集并暴露指标
     */
    @Scheduled(fixedRate = 1000)
    public void exportMetrics() {
        // 获取所有资源的指标
        Map<String, MetricEntity> metrics = MetricFetcher.getInstance().getMetrics();

        for (Map.Entry<String, MetricEntity> entry : metrics.entrySet()) {
            String resource = entry.getKey();
            MetricEntity metric = entry.getValue();

            // QPS
            recordGauge("sentinel_qps_pass",
                metric.getPassQps(),
                "resource", resource);
            recordGauge("sentinel_qps_block",
                metric.getBlockQps(),
                "resource", resource);

            // 响应时间
            recordGauge("sentinel_rt_avg",
                metric.getSuccessRt().doubleValue(),
                "resource", resource);

            // 线程数
            recordGauge("sentinel_thread",
                metric.getThread(),
                "resource", resource);
        }
    }
}
```

## 自适应限流实现

### 基于系统负载的自适应限流

```java
/**
 * 自适应限流器
 * 根据系统当前负载动态调整限流阈值
 */
@Service
public class AdaptiveRateLimiter {

    private final AtomicReference<Double> currentThreshold = new AtomicReference<>(10000.0);
    private final LoadIndicator loadIndicator;

    @Scheduled(fixedRate = 1000)  // 每秒调整
    public void adjustThreshold() {
        // 1. 获取当前系统指标
        SystemMetrics metrics = loadIndicator.getCurrentMetrics();

        double cpuLoad = metrics.getCpuLoad();
        double memoryUsage = metrics.getMemoryUsage();
        double currentQps = metrics.getCurrentQps();
        double avgResponseTime = metrics.getAvgResponseTime();

        // 2. 计算新的阈值
        double newThreshold = calculateThreshold(metrics);

        // 3. 平滑调整阈值
        double current = currentThreshold.get();
        double delta = newThreshold - current;
        double smoothed = current + delta * 0.1;  // 渐进调整

        currentThreshold.set(smoothed);

        // 4. 更新 Sentinel 规则
        updateFlowRule(smoothed);

        // 5. 记录日志
        log.info("自适应限流: CPU={}, 内存={}, 当前QPS={}, 新阈值={}",
            cpuLoad, memoryUsage, currentQps, smoothed);
    }

    /**
     * 计算限流阈值
     */
    private double calculateThreshold(SystemMetrics metrics) {
        double cpuLoad = metrics.getCpuLoad();
        double memoryUsage = metrics.getMemoryUsage();
        double baseline = 10000.0;  // 基础阈值

        // 1. CPU 负载评估
        double cpuFactor = 1.0;
        if (cpuLoad > 0.9) {
            cpuFactor = 0.3;  // CPU > 90%，降为 30%
        } else if (cpuLoad > 0.7) {
            cpuFactor = 0.6;  // CPU > 70%，降为 60%
        } else if (cpuLoad > 0.5) {
            cpuFactor = 0.8;  // CPU > 50%，降为 80%
        }

        // 2. 内存负载评估
        double memoryFactor = 1.0;
        if (memoryUsage > 0.9) {
            memoryFactor = 0.5;
        } else if (memoryUsage > 0.8) {
            memoryFactor = 0.7;
        }

        // 3. 响应时间评估
        double rtFactor = 1.0;
        double avgRt = metrics.getAvgResponseTime();
        if (avgRt > 1000) {
            rtFactor = 0.5;  // RT > 1s，限流 50%
        } else if (avgRt > 500) {
            rtFactor = 0.8;  // RT > 500ms，限流 80%
        }

        // 4. 综合计算
        double factor = Math.min(cpuFactor, Math.min(memoryFactor, rtFactor));
        return baseline * factor;
    }

    /**
     * 更新限流规则
     */
    private void updateFlowRule(double threshold) {
        List<FlowRule> rules = FlowRuleManager.getRules();

        for (FlowRule rule : rules) {
            if ("global".equals(rule.getLimitApp())) {
                rule.setCount(threshold);
            }
        }

        FlowRuleManager.loadRules(rules);
    }
}
```

### 基于排队队列的自适应限流

```java
/**
 * 基于排队的自适应限流
 */
@Service
public class QueueBasedAdaptiveLimiter {

    private final BlockingQueue<Request> queue;
    private final AtomicInteger currentProcessing = new AtomicInteger(0);
    private final int maxProcessing;

    public QueueBasedAdaptiveLimiter() {
        this.queue = new LinkedBlockingQueue<>(10000);
        this.maxProcessing = 200;  // 最大并发处理数
    }

    /**
     * 提交请求
     */
    public CompletableFuture<Response> submit(Request request) {
        return CompletableFuture.supplyAsync(() -> {
            // 1. 尝试加入队列
            if (!queue.offer(request)) {
                throw new RejectedExecutionException("队列已满");
            }

            // 2. 限流检查
            if (!tryAcquire()) {
                queue.remove(request);
                throw new RejectedExecutionException("系统繁忙");
            }

            try {
                // 3. 处理请求
                return process(request);
            } finally {
                release();
            }
        });
    }

    /**
     * 限流检查
     */
    private boolean tryAcquire() {
        // 当前处理数少于最大并发，可以处理
        return currentProcessing.get() < maxProcessing;
    }

    /**
     * 释放资源
     */
    private void release() {
        currentProcessing.decrementAndGet();
    }

    /**
     * 动态调整最大并发
     */
    @Scheduled(fixedRate = 5000)
    public void adjustMaxProcessing() {
        SystemMetrics metrics = loadIndicator.getCurrentMetrics();

        int newMax = calculateMaxProcessing(metrics);
        this.maxProcessing = newMax;

        log.info("调整最大并发数: {}", newMax);
    }
}
```

## 限流策略选择

| 策略 | 原理 | 适用场景 |
|---|---|---|
| QPS 限流 | 固定阈值 | 稳定的接口 |
| 并发限流 | 控制线程数 | 资源密集型操作 |
| 预热限流 | 逐渐增加 | 流量突增 |
| 排队等待 | 请求排队 | 允许延迟的场景 |
| 自适应限流 | 动态调整 | 生产环境 |

## 常见问题与解决

### 问题一：限流过于严格

```java
// 调整限流因子
private double calculateThreshold(SystemMetrics metrics) {
    double baseThreshold = 10000.0;
    double factor = calculateFactor(metrics);

    // 放宽阈值（乘以系数）
    return baseThreshold * factor * 1.2;  // 放宽 20%
}
```

### 问题二：限流抖动

```java
// 使用滑动窗口平均，避免抖动
private double calculateSmoothedThreshold(SystemMetrics metrics) {
    // 获取最近 10 次的指标
    List<Double> recentThresholds = history.getRecentThresholds(10);

    // 计算平均值
    double avg = recentThresholds.stream()
        .mapToDouble(Double::doubleValue)
        .average()
        .orElse(10000.0);

    return avg;
}
```

## 总结

自适应限流的核心是**感知 + 调整**：

| 组件 | 作用 |
|---|---|
| 指标采集 | 收集 CPU、内存、RT、QPS 等指标 |
| 阈值计算 | 根据指标计算合适的限流阈值 |
| 规则更新 | 动态更新 Sentinel 规则 |
| 效果验证 | 监控限流效果，验证策略 |

自适应限流的优势：

1. **无需人工调参**：系统自动感知负载
2. **更精确**：根据实际状态调整
3. **更稳定**：避免过载或浪费

记住：**自适应不是万能的**，在复杂场景下仍需要人工干预和调优。
