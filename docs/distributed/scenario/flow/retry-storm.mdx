# 超时重试风暴治理：重试策略与熔断降级组合

你的服务 A 调用服务 B，B 超时了，A 重试；B 还没恢复，A 继续重试；同时服务 C 也在重试。结果呢？B 直接被请求淹没，彻底崩溃。

这就是超时重试风暴。

## 重试的本质

### 重试的双刃剑

重试可以提高系统的容错能力，但也会带来风险：

```
┌──────────────────────────────────────────────────────────────────┐
│                    重试风暴形成过程                               │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│  时刻 T0：服务 B 响应变慢                                         │
│  ┌─────────┐              ┌─────────┐                          │
│  │  服务 A  │ ───────────> │  服务 B  │  超时                    │
│  └─────────┘              └─────────┘                          │
│                                                                   │
│  时刻 T1：A 重试，同时 C 也在调用 B                             │
│  ┌─────────┐              ┌─────────┐                          │
│  │  服务 A  │ ───────────> │  服务 B  │ × 3                    │
│  └─────────┘              └─────────┘                          │
│  ┌─────────┐              ┌─────────┐                          │
│  │  服务 C  │ ───────────> │  服务 B  │ × 2                    │
│  └─────────┘              └─────────┘                          │
│                                                                   │
│  结果：B 的请求量从 1x 暴增到 5x，彻底崩溃                        │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 重试的原则

| 原则 | 说明 |
|---|---|
| 可幂等 | 重试的操作必须幂等 |
| 有上限 | 重试次数必须有上限 |
| 要退避 | 重试间隔要递增，不要频繁重试 |
| 可熔断 | 被调方压力大时要熔断 |
| 要监控 | 重试量和成功率必须监控 |

## 重试策略

### 退避算法

```java
/**
 * 重试退避策略
 */
public class BackoffStrategy {

    /**
     * 固定间隔退避
     */
    public static long fixedDelay(int attempt) {
        return 1000;  // 每次间隔 1 秒
    }

    /**
     * 线性递增退避
     */
    public static long linearDelay(int attempt, long baseDelay) {
        return baseDelay * attempt;  // 1s, 2s, 3s...
    }

    /**
     * 指数退避（推荐）
     */
    public static long exponentialDelay(int attempt, long baseDelay) {
        // 1s, 2s, 4s, 8s, 16s...
        return (long) (baseDelay * Math.pow(2, attempt - 1));
    }

    /**
     * 指数退避 + 抖动（推荐用于生产环境）
     * 加入随机抖动，避免多客户端同时重试
     */
    public static long exponentialDelayWithJitter(int attempt, long baseDelay) {
        long exponentialDelay = (long) (baseDelay * Math.pow(2, attempt - 1));
        // 添加 0-50% 的随机抖动
        long jitter = (long) (exponentialDelay * (Math.random() * 0.5));
        return exponentialDelay + jitter;
    }
}
```

### Spring Retry 配置

```java
@Configuration
@EnableRetry
public class RetryConfig {

    /**
     * 全局重试配置
     */
    @Bean
    public RetryTemplate retryTemplate() {
        RetryTemplate template = new RetryTemplate();

        // 1. 设置重试策略
        // 最大重试 3 次
        SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy();
        retryPolicy.setMaxAttempts(3);

        // 2. 设置退避策略
        ExponentialBackOffPolicy backOffPolicy = new ExponentialBackOffPolicy();
        backOffPolicy.setInitialInterval(1000);      // 初始间隔 1 秒
        backOffPolicy.setMultiplier(2.0);            // 指数退避
        backOffPolicy.setMaxInterval(10000);          // 最大间隔 10 秒

        template.setRetryPolicy(retryPolicy);
        template.setBackOffPolicy(backOffPolicy);

        return template;
    }

    /**
     * 带熔断的重试模板
     */
    @Bean
    public RetryTemplate circuitBreakerRetryTemplate() {
        RetryTemplate template = new RetryTemplate();

        // 使用 CircuitBreakerRetryPolicy
        CircuitBreakerRetryPolicy retryPolicy = new CircuitBreakerRetryPolicy(
            new SimpleRetryPolicy()
        );

        // 熔断器配置
        retryPolicy.setOpenTimeout(5000);      // 熔断器打开时间
        retryPolicy.setResetTimeout(30000);    // 熔断器重置时间

        template.setRetryPolicy(retryPolicy);

        return template;
    }
}
```

### 使用示例

```java
@Service
public class RemoteCallService {

    @Autowired
    private RetryTemplate retryTemplate;

    private final RestTemplate restTemplate;

    /**
     * 使用 RetryTemplate 重试
     */
    public User getUserById(Long userId) {
        return retryTemplate.execute(context -> {
            // context.getRetryCount() 可以获取重试次数
            log.info("尝试获取用户，次数={}", context.getRetryCount() + 1);

            return restTemplate.getForObject(
                "http://user-service/api/users/" + userId,
                User.class
            );
        });
    }

    /**
     * 使用注解方式重试
     */
    @Retryable(
        value = { RemoteException.class, SocketTimeoutException.class },
        maxAttempts = 3,
        backoff = @Backoff(delay = 1000, multiplier = 2, maxDelay = 10000)
    )
    public Order getOrder(Long orderId) {
        return orderClient.getOrder(orderId);
    }

    /**
     * 重试失败后的处理
     */
    @Retryable(
        value = Exception.class,
        maxAttempts = 3,
        recover = "getOrderFallback"
    )
    public Order getOrderWithFallback(Long orderId) {
        return orderClient.getOrder(orderId);
    }

    /**
     * 重试失败后的兜底方法
     */
    public Order getOrderFallback(Long orderId, Exception e) {
        log.error("获取订单失败，orderId={}, 错误={}", orderId, e.getMessage());
        return orderRepository.findById(orderId).orElse(null);
    }
}
```

## 熔断器模式

### 熔断器原理

```
┌──────────────────────────────────────────────────────────────────┐
│                      熔断器状态转换                               │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│   ┌─────────┐                                                     │
│   │  CLOSED │ ←─── 失败率 `<=` 阈值 ───┐                         │
│   │  关闭    │                         │                         │
│   │         │                         │                         │
│   │ 正常请求  │                         │                         │
│   │ 通过     │                         │                         │
│   └────┬────┘                         │                         │
│        │                              │                         │
│        │ 失败率 > 阈值               │                         │
│        ▼                              │                         │
│   ┌─────────┐                         │                         │
│   │  OPEN   │ ←─── 超时时间 ──────────┘                         │
│   │  打开    │                                                 │
│   │         │                                                 │
│   │ 快速失败 │ ──── 半开 ────────── ┐                         │
│   │ 返回默认 │                    │                         │
│   └────┬────┘                    │                         │
│        │                         │                         │
│        │ 成功                    │ 失败                     │

│   ┌─────────┐                    │                         │
│   │ HALF-   │ ──────────────────  │                         │
│   │ OPEN    │                    │                         │
│   │ 半开    │                    │                         │
│   └─────────┘                    │                         │
│                                 │                         │
│                                 ▼                         │
│                           ┌─────────┐                      │
│                           │ OPEN    │                      │
│                           │ 打开     │                      │
│                           └─────────┘                      │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### Resilience4j 熔断器

```java
@Configuration
public class Resilience4jConfig {

    @Bean
    public CircuitBreakerRegistry circuitBreakerRegistry() {
        // 创建熔断器注册表
        CircuitBreakerRegistry registry = CircuitBreakerRegistry.of(
            CircuitBreakerConfig.custom()
                // 滑动窗口配置
                .slidingWindowType(SlidingWindowType.COUNT_BASED)  // 基于请求次数
                .slidingWindowSize(100)                            // 滑动窗口大小
                .minimumNumberOfCalls(20)                           // 最小调用次数

                // 失败率阈值
                .failureRateThreshold(50)  // 失败率 50% 时触发熔断

                // 慢调用阈值
                .slowCallRateThreshold(80)   // 80% 慢调用时触发熔断
                .slowCallDurationThreshold(Duration.ofSeconds(5))

                // 熔断持续时间
                .waitDurationInOpenState(Duration.ofSeconds(30))

                // 半开状态下允许的调用次数
                .permittedNumberOfCallsInHalfOpenState(10)

                // 自动从打开变为半开
                .automaticTransitionFromOpenToHalfOpenEnabled(true)

                .build()
        );

        return registry;
    }

    /**
     * 为不同服务创建不同的熔断器
     */
    @Bean
    public CircuitBreaker userServiceCircuitBreaker(CircuitBreakerRegistry registry) {
        return registry.circuitBreaker("userService");
    }

    @Bean
    public CircuitBreaker paymentServiceCircuitBreaker(CircuitBreakerRegistry registry) {
        CircuitBreakerConfig config = CircuitBreakerConfig.custom()
            // 支付服务更严格
            .failureRateThreshold(30)
            .slowCallDurationThreshold(Duration.ofSeconds(2))
            .build();

        return registry.circuitBreaker("paymentService", config);
    }
}
```

### 熔断器使用

```java
@Service
public class CircuitBreakerRemoteCallService {

    private final CircuitBreaker userServiceCircuitBreaker;
    private final CircuitBreaker paymentServiceCircuitBreaker;
    private final UserClient userClient;
    private final PaymentClient paymentClient;

    public CircuitBreakerRemoteCallService(
            @Qualifier("userServiceCircuitBreaker") CircuitBreaker userServiceCircuitBreaker,
            @Qualifier("paymentServiceCircuitBreaker") CircuitBreaker paymentServiceCircuitBreaker,
            UserClient userClient,
            PaymentClient paymentClient) {
        this.userServiceCircuitBreaker = userServiceCircuitBreaker;
        this.paymentServiceCircuitBreaker = paymentServiceCircuitBreaker;
        this.userClient = userClient;
        this.paymentClient = paymentClient;
    }

    /**
     * 用户服务调用（带熔断）
     */
    public User getUser(Long userId) {
        return userServiceCircuitBreaker.executeSupplier(() -> {
            return userClient.getUser(userId);
        });
    }

    /**
     * 支付服务调用（带熔断和降级）
     */
    public PaymentResult pay(PaymentRequest request) {
        return Decorators.ofSupplier(() -> paymentClient.pay(request))
            // 配置熔断器
            .withCircuitBreaker(paymentServiceCircuitBreaker)
            // 配置超时
            .withTimeout(Duration.ofSeconds(5))
            // 配置降级
            .withFallback(List.of(Exception.class),
                e -> {
                    log.warn("支付服务熔断，返回降级结果", e);
                    return PaymentResult.degraded();
                })
            .decorateSupplier()
            .get();
    }
}
```

## 组合策略：重试 + 熔断

### 完整配置

```java
/**
 * 重试 + 熔断的完整配置
 */
@Configuration
public class RetryAndCircuitBreakerConfig {

    @Bean
    public RetryTemplate resilientRetryTemplate(CircuitBreakerRegistry registry) {
        RetryTemplate template = new RetryTemplate();

        // 熔断重试策略
        CircuitBreakerRetryPolicy retryPolicy = new CircuitBreakerRetryPolicy(
            new SimpleRetryPolicy()
        );
        retryPolicy.setCircuitBreaker(
            registry.circuitBreaker("default")
        );

        // 退避策略
        ExponentialBackOffPolicy backOff = new ExponentialBackOffPolicy();
        backOff.setInitialInterval(1000);
        backOff.setMultiplier(2);
        backOff.setMaxInterval(10000);

        template.setRetryPolicy(retryPolicy);
        template.setBackOffPolicy(backOff);

        return template;
    }
}

/**
 * 使用示例
 */
@Service
public class ResilientRemoteService {

    @Autowired
    private RetryTemplate retryTemplate;

    @Autowired
    private CircuitBreakerRegistry circuitBreakerRegistry;

    /**
     * 带完整保护的远程调用
     */
    public <T> T callWithFullProtection(
            Supplier<T> supplier,
            Supplier<T> fallback,
            String serviceName) {

        // 1. 获取或创建熔断器
        CircuitBreaker circuitBreaker = circuitBreakerRegistry.circuitBreaker(serviceName);

        // 2. 使用熔断器包装
        Supplier<T> decoratedSupplier = Decorators.ofSupplier(supplier)
            .withCircuitBreaker(circuitBreaker)
            .withFallback(List.of(Exception.class), e -> fallback.get())
            .decorate();

        // 3. 使用重试模板执行
        return retryTemplate.execute(context -> {
            log.info("调用 {}, 第 {} 次尝试", serviceName, context.getRetryCount() + 1);
            return decoratedSupplier.get();
        }, context -> {
            // 重试耗尽后的处理
            log.error("调用 {} 重试耗尽，共尝试 {} 次",
                serviceName, context.getRetryCount());
            return fallback.get();
        });
    }
}
```

```java
public class DegradationStrategyService {

     * 返回缓存数据
     */

        // 从 Redis 获取缓存
        List<Product> cached = redisTemplate.opsForValue().get("products:cache");
        if (cached != null) {
            return cached;
        }

        return Collections.emptyList();
    }

    /**
     * 返回友好错误提示
     */
    public Order createOrderFallback(OrderRequest request, Exception e) {

        Order order = new Order();
        order.setId("DEGRADED-" + System.currentTimeMillis());
        order.setStatus("PENDING");
        order.setRemark("系统繁忙，订单稍后处理");

        // 放入延迟队列，稍后重试
        delayedQueue.put(request);

        return order;
    }

     * 限流降级
     */
    public Response rateLimitFallback(String resource, Exception e) {
        return Response.error("系统繁忙，请稍后重试", 429);
    }
```

## 监控与告警

### 熔断器监控

```java
/**
 * 熔断器监控
 */
@Service
public class CircuitBreakerMonitoringService {

    @Autowired
    private CircuitBreakerRegistry registry;

    @Scheduled(fixedRate = 30000)  // 每 30 秒
    public void monitorCircuitBreakers() {
        for (String name : registry.getAllCircuitBreakers().stream()
                .map(CircuitBreaker::getName)
                .collect(Collectors.toList())) {

            CircuitBreaker cb = registry.circuitBreaker(name);
            CircuitBreakerMetrics metrics = cb.getMetrics();

            // 获取关键指标
            long failureRate = metrics.getFailureRate();
            long slowCallRate = metrics.getSlowCallRate();
            int bufferedCalls = metrics.getNumberOfBufferedCalls();
            int failedCalls = metrics.getNumberOfFailedCalls();

            // 记录到监控系统
            recordMetrics(name, metrics);

            // 触发告警
            if (failureRate > 80) {
                alert("严重告警: " + name + " 熔断器打开，失败率=" + failureRate + "%");
            }
        }
    }

    /**
     * 获取熔断器状态
     */
    public Map<String, CircuitBreakerStatus> getAllCircuitBreakerStatus() {
        Map<String, CircuitBreakerStatus> statusMap = new HashMap<>();

        for (CircuitBreaker cb : registry.getAllCircuitBreakers()) {
            CircuitBreakerMetrics metrics = cb.getMetrics();

            statusMap.put(cb.getName(), new CircuitBreakerStatus()
                .setName(cb.getName())
                .setState(cb.getState().name())
                .setFailureRate(metrics.getFailureRate())
                .setSlowCallRate(metrics.getSlowCallRate())
                .setBufferedCalls(metrics.getNumberOfBufferedCalls())
                .setFailedCalls(metrics.getNumberOfFailedCalls())
                .setSuccessCalls(metrics.getNumberOfSuccessfulCalls())
            );
        }

        return statusMap;
    }
}
```

## 常见问题与解决

### 问题一：幂等性

```java
// 所有可能重试的操作必须幂等
@Service
public class IdempotentOperationService {

    /**
     * 幂等扣款
     */
    @Transactional
    public boolean deductBalance(String userId, BigDecimal amount, String transactionId) {
        // 1. 检查是否已处理
        String key = "transaction:processed:" + transactionId;
        if (Boolean.TRUE.equals(redisTemplate.hasKey(key))) {
            log.info("交易已处理: transactionId={}", transactionId);
            return true;
        }

        // 2. 执行扣款
        int affected = jdbcTemplate.update(
            "UPDATE account SET balance = balance - ? WHERE user_id = ? AND balance >= ?",
            amount, userId, amount
        );

        if (affected > 0) {
            // 3. 标记已处理
            redisTemplate.opsForValue().set(key, "1", Duration.ofDays(7));
            return true;
        }

        return false;
    }
}
```

## 总结

超时重试风暴治理的核心是**保护 + 恢复**：

| 策略 | 作用 |
|---|---|
| 退避算法 | 避免同时重试 |
| 熔断器 | 快速失败，保护下游 |
| 降级 | 优雅降级，不影响主流程 |
| 监控 | 及时发现问题 |

最佳实践：

1. **幂等优先**：任何可能重试的操作必须幂等
2. **有限重试**：设置最大重试次数
3. **指数退避**：重试间隔递增
4. **熔断保护**：防止雪崩
5. **降级兜底**：有备无患

记住：**重试是把双刃剑**，用好了提高可用性，用不好制造故障。
