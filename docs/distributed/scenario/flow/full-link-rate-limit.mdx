# 全链路限流

双十一零点，你的系统能抗住 10 万 QPS，但上游只给你 1 万 QPS 的流量。剩下的 9 万请求怎么办？直接拒绝，用户体验极差；全部接收，系统扛不住。

这就是**限流**要解决的问题。今天我们深入剖析全链路限流的设计与实现。

## 一、限流的核心问题

### 1.1 限流的三大场景

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        限流三大场景                                    │
│                                                                         │
│   1. 入口限流：                                                       │
│   用户请求 ──▶ 网关限流 ──▶ 允许 N QPS 通过                               │
│                                                                         │
│   2. 链路限流：                                                       │
│   服务A ──▶ 服务B ──▶ 服务C ──▶ 每层都限流                               │
│                                                                         │
│   3. 资源限流：                                                       │
│   CPU 80% ──▶ 限流 50% ──▶ CPU 60% ──▶ 逐步恢复                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 限流算法对比

| 算法 | 原理 | 优点 | 缺点 |
|---|---|---|---|
| 计数器 | 固定时间窗口内计数 | 简单 | 有临界问题 |
| 滑动窗口 | 滑动时间窗口 | 精确 | 实现复杂 |
| 令牌桶 | 以固定速率放入令牌 | 支持突发 | 需要存储 |
| 漏桶 | 以固定速率处理请求 | 平滑 | 不支持突发 |

### 2.2 限流配置

```java title="Sentinel 配置")
@Configuration
public class SentinelConfig {
    @PostConstruct
    public void init() {
        // 配置限流规则
        List<FlowRule> rules = new ArrayList<>();

        // 接口级别限流
        FlowRule rule1 = new FlowRule();
        rule1.setResource("order/create");
        rule1.setGrade(RuleConstant.FLOW_GRADE_QPS);
        rule1.setCount(1000); // 1000 QPS
        rule1.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);
        rules.add(rule1);

        // 服务级别限流
        FlowRule rule2 = new FlowRule();
        rule2.setResource("orderService");
        rule2.setGrade(RuleConstant.FLOW_GRADE_QPS);
        rule2.setCount(5000); // 5000 QPS
        rule2.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_WARM_UP);
        rule2.setWarmUpPeriodSec(10); // 预热时间
        rules.add(rule2);

        FlowRuleManager.loadRules(rules);
    }
}
```

### 2.3 限流注解

```java title="限流注解")
@Service
public class OrderService {
    /**
     * 订单创建限流
     */
    @SentinelResource(value = "createOrder",
            blockHandler = "createOrderBlockHandler",
            fallback = "createOrderFallback")
    public Order createOrder(OrderDTO dto) {
        // 业务逻辑
        return orderRepository.save(dto);
    }

    /**
     * 限流处理
     */
    public Order createOrderBlockHandler(OrderDTO dto, BlockException e) {
        log.warn("订单创建被限流: {}", e.getClass().getSimpleName());
        throw new ServiceUnavailableException("系统繁忙，请稍后重试");
    }

    /**
     * 降级处理
     */
    public Order createOrderFallback(OrderDTO dto, Throwable e) {
        log.error("订单创建失败", e);
        throw new ServiceException("服务异常，请稍后重试");
    }
}
```

## 三、全链路限流设计

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        全链路限流架构                                  │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                      配置中心                                   │  │
│   │              (规则下发、动态配置)                               │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                              │                                          │
│          ┌─────────────────┼─────────────────┐                      │
│          ▼                 ▼                 ▼                      │
│   ┌───────────┐      ┌───────────┐      ┌───────────┐            │
│   │  网关层   │      │  服务 A   │      │  服务 B   │            │
│   │  限流组件  │      │  限流组件  │      │  限流组件  │            │
│   └───────────┘      └───────────┘      └───────────┘            │
│                                                                         │
│   特点：                                                               │
│   - 每层都有限流阈值                                                    │
│   - 上层阈值 <= 下层阈值之和                                            │
│   - 动态配置，实时生效                                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 配置下发

```java title="动态规则配置")
@Service
@Slf4j
public class FlowRuleService {
    @Autowired
    private ConfigService configService;

    /**
     * 监听配置变化
     */
    @PostConstruct
    public void init() {
        configService.addListener("flow-rules", changeEvent -> {
            String newValue = changeEvent.getValue();
            List<FlowRule> rules = JSON.parseArray(newValue, FlowRule.class);
            FlowRuleManager.loadRules(rules);
            log.info("限流规则更新: {} 条", rules.size());
        });
    }

    /**
     * 更新限流规则
     */
    public void updateRules(List<FlowRule> rules) {
        String ruleJson = JSON.toJSONString(rules);
        configService.setConfig("flow-rules", ruleJson);
    }
}
```

### 3.3 层级限流策略

```java title="层级限流")
@Service
@Slf4j
public class TieredRateLimiter {
    /**
     * 网关层限流：10000 QPS
     */
    @Value("${rate.limit.gateway:10000}")
    private int gatewayLimit;

    /**
     * 服务层限流：5000 QPS
     */
    @Value("${rate.limit.service:5000}")
    private int serviceLimit;

    /**
     * 底层限流：2000 QPS
     */
    @Value("${rate.limit.database:2000}")
    private int databaseLimit;

    /**
     * 层级限流检查
     */
    public boolean checkLimit(String tier, int currentQps) {
        int limit = getLimitForTier(tier);
        return currentQps < limit;
    }

    private int getLimitForTier(String tier) {
        switch (tier) {
            case "gateway":
                return gatewayLimit;
            case "service":
                return serviceLimit;
            case "database":
                return databaseLimit;
            default:
                return 1000;
        }
    }
}
```

## 四、热点限流

### 4.1 热点参数限流

```java title="热点参数限流")
@Service
public class HotParamLimiter {
    /**
     * 热点商品限流
     */
    @SentinelResource(value = "getProduct",
            blockHandlerClass = HotParamBlockHandler.class,
            blockHandler = "handleBlock")
    public Product getProduct(Long productId) {
        return productRepository.findById(productId);
    }
}

public class HotParamBlockHandler {
    public static Product handleBlock(Long productId, BlockException e) {
        log.warn("热点商品被限流: productId={}", productId);
        throw new ServiceUnavailableException("商品访问太频繁，请稍后重试");
    }
}
```

### 4.2 配置热点规则

```java title="热点规则配置")
@Service
public class HotSpotRuleConfig {
    @PostConstruct
    public void init() {
        List<ParamFlowRule> rules = new ArrayList<>();

        // 热点商品限流：每个商品ID最多 100 QPS
        ParamFlowRule rule = new ParamFlowRule("getProduct")
                .setParamIdx(0) // 第一个参数（productId）
                .setCount(100)
                .setGrade(RuleConstant.FLOW_GRADE_QPS)
                .setDurationInSec(1)
                .setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);

        rules.add(rule);
        ParamFlowRuleManager.loadRules(rules);
    }
}
```

## 五、集群限流

### 5.1 Token Server 模式

```java title="集群限流配置")
@Configuration
public class ClusterFlowConfig {
    @Value("${token.server.host}")
    private String tokenServerHost;

    @Value("${token.server.port}")
    private int tokenServerPort;

    @PostConstruct
    public void init() {
        // 配置 Token Server
        ClusterFlowTokenClient clusterTokenClient = new ClusterFlowTokenClient();
        clusterTokenClient.start(tokenServerHost, tokenServerPort);

        // 设置为客户端模式
        FlowRuleManager.setProperty(new ClusterClientProperty() {
            @Override
            public List<FlowRule> load() {
                return ClusterFlowConfig.getRules();
            }

            @Override
            public void store(List<FlowRule> rules) {
                ClusterFlowConfig.updateRules(rules);
            }
        });
    }
}
```

### 5.2 集群规则配置

```java title="集群限流规则")
@Service
public class ClusterFlowRuleService {
    @Autowired
    private ConfigService configService;

    /**
     * 设置集群限流规则
     */
    public void setClusterRule(String resource, int threshold) {
        ClusterFlowRule rule = new ClusterFlowRule();
        rule.setResource(resource);
        rule.setCount(threshold);
        rule.setFlowId(1L);
        rule.setSampleCount(10);
        rule.setSlideWindowMs(1000);

        // 客户端自己统计
        rule.setStrategy(ClusterFlowRule.STRATEGY_NORMAL);

        ClusterFlowConfig.addClusterRule(rule);
    }
}
```

## 七、总结

| 限流层级 | 阈值设置 | 策略 |
|---|---|---|
| 网关层 | 入口流量 100% | 直接拒绝 |
| 服务层 | 入口流量 50% | 冷启动 |
| 资源层 | 入口流量 20% | 排队等待 |

:::tip 实战建议
1. **上层阈值 >= 下层阈值之和**：避免下层先扛不住
2. **冷启动是必须的**：大促前预热，让系统渐进到高水位
3. **热点限流很重要**：防止爆款商品打死系统
:::

## 结尾

全链路限流是保护系统的「多道防线」。记住三个要点：

1. **每层都要限流**：单层限流不够用
2. **阈值要合理**：上层 >= 下层之和
3. **策略要选对**：突发用冷启动，平滑用漏桶
