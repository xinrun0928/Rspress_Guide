# 分布式 Session：Spring Session + Redis 方案

用户刚刚登录了你的系统，下一秒请求被负载均衡到了另一台服务器。用户状态丢了，又要重新登录。这就是分布式 Session 的经典问题。

Session 不一致，是分布式系统中最容易出错的环节之一。

## 为什么需要分布式 Session？

### 单机 Session 的问题

```java
// 单机 Tomcat 的 Session 存储
HttpSession session = request.getSession();
session.setAttribute("user", currentUser);

// 问题：用户下次请求可能被路由到另一台服务器
// 这台服务器上没有这个用户的 Session
```

当你的系统部署了多个实例，用户请求被随机分配时，Session 状态丢失的问题就出现了：

1. **水平扩展困难**：无法简单增加服务器数量
2. **负载不均**：某些策略可能加剧 Session 丢失
3. **故障转移失败**：主服务器宕机时，用户状态全部丢失

### 分布式 Session 解决方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| Session 复制 | 实现简单 | 网络开销大，延迟高 | 集群规模小（< 5 台） |
| Session 绑定 | 可控性好 | 失去负载均衡优势 | 有状态服务 |
| Cookie 存储 | 无存储成本 | 安全风险，容量有限 | 简单场景 |
| Redis 集中存储 | 高性能，易扩展 | 增加复杂度 | **生产环境推荐** |

## 快速集成 Spring Session

### Maven 依赖

```xml
<dependencies>
    <!-- Spring Session Core -->
    <dependency>
        <groupId>org.springframework.session</groupId>
        <artifactId>spring-session-core</artifactId>
    </dependency>

    <!-- Spring Session + Redis -->
    <dependency>
        <groupId>org.springframework.session</groupId>
        <artifactId>spring-session-data-redis</artifactId>
    </dependency>

    <!-- Lettuce 客户端（推荐，比 Jedis 性能更好）-->
    <dependency>
        <groupId>io.lettuce</groupId>
        <artifactId>lettuce-core</artifactId>
    </dependency>

    <!-- Redis 连接池 -->
    <dependency>
        <groupId>org.apache.commons</groupId>
        <artifactId>commons-pool2</artifactId>
    </dependency>
</dependencies>
```

### 配置类

```java
// Spring Session 配置
@Configuration
@EnableRedisHttpSession(maxInactiveIntervalInSeconds = 1800)  // Session 30 分钟过期
public class SessionConfig {

    @Value("${spring.redis.host}")
    private String redisHost;

    @Value("${spring.redis.port}")
    private int redisPort;

    @Value("${spring.redis.password:}")
    private String redisPassword;

    @Value("${spring.redis.database:0}")
    private int redisDatabase;

    @Bean
    public LettuceConnectionFactory redisConnectionFactory() {
        RedisStandaloneConfiguration config = new RedisStandaloneConfiguration();
        config.setHostName(redisHost);
        config.setPort(redisPort);
        config.setPassword(RedisPassword.of(redisPassword));
        config.setDatabase(redisDatabase);

        // 连接池配置
        LettucePoolingClientConfiguration poolConfig = LettucePoolingClientConfiguration.builder()
            .commandTimeout(Duration.ofSeconds(2))
            .poolConfig(new GenericObjectPoolConfig() {{
                setMaxTotal(100);
                setMaxIdle(20);
                setMinIdle(5);
                setMaxWait(Duration.ofMillis(3000));
            }})
            .build();

        return new LettuceConnectionFactory(config, poolConfig);
    }
}
```

## Session 数据操作

### 基本操作

```java
@RestController
@RequestMapping("/api/session")
public class SessionController {

    @GetMapping("/test")
    public ResponseEntity<?> testSession(HttpSession session) {
        // 获取 Session ID
        String sessionId = session.getId();

        // 存储数据
        session.setAttribute("user", "张三");
        session.setAttribute("loginTime", LocalDateTime.now());

        // 读取数据
        String user = (String) session.getAttribute("user");
        LocalDateTime loginTime = (LocalDateTime) session.getAttribute("loginTime");

        Map<String, Object> result = new HashMap<>();
        result.put("sessionId", sessionId);
        result.put("user", user);
        result.put("loginTime", loginTime);

        return ResponseEntity.ok(result);
    }

    @DeleteMapping("/clear")
    public ResponseEntity<?> clearSession(HttpSession session) {
        // 使 Session 失效
        session.invalidate();
        return ResponseEntity.ok("Session 已清除");
    }
}
```

### 分布式锁保护 Session

在并发场景下，多个请求可能同时修改 Session：

```java
@Service
public class DistributedSessionService {

    private final StringRedisTemplate redisTemplate;
    private static final String LOCK_PREFIX = "session:lock:";

    public DistributedSessionService(StringRedisTemplate redisTemplate) {
        this.redisTemplate = redisTemplate;
    }

    /**
     * 分布式锁保护的 Session 操作
     */
    public <T> T executeWithLock(String sessionId, SessionCallback<T> callback) {
        String lockKey = LOCK_PREFIX + sessionId;
        String lockValue = UUID.randomUUID().toString();

        try {
            // 获取锁（最多等待 5 秒，锁自动 30 秒过期）
            boolean locked = redisTemplate.opsForValue()
                .setIfAbsent(lockKey, lockValue, Duration.ofSeconds(30));

            if (!locked) {
                throw new RuntimeException("获取 Session 锁失败");
            }

            // 执行操作
            return callback.execute();

        } finally {
            // 释放锁（只能释放自己的锁）
            String currentValue = redisTemplate.opsForValue().get(lockKey);
            if (lockValue.equals(currentValue)) {
                redisTemplate.delete(lockKey);
            }
        }
    }

    /**
     * 原子性增加积分
     */
    public void addPoints(String sessionId, int points) {
        executeWithLock(sessionId, () -> {
            String pointsKey = "session:points:" + sessionId;

            // 获取当前积分
            String currentPoints = redisTemplate.opsForValue().get(pointsKey);
            int newPoints = (currentPoints == null ? 0 : Integer.parseInt(currentPoints)) + points;

            // 更新积分
            redisTemplate.opsForValue().set(pointsKey, String.valueOf(newPoints));

            return newPoints;
        });
    }
}

@FunctionalInterface
interface SessionCallback<T> {
    T execute();
}
```

## 常见问题与解决

### 问题一：Session 序列化失败

Session 数据存储到 Redis 需要序列化。复杂对象必须实现序列化：

```java
// 方式一：实现 Serializable 接口
public class User implements Serializable {
    private static final long serialVersionUID = 1L;
    private Long id;
    private String username;
    private String email;
    private String password;  // 敏感信息不建议存在 Session
}

// 方式二：使用 JSON 序列化
@Configuration
public class JsonSessionConfig {

    @Bean
    public RedisSerializer<Object> springSessionDefaultRedisSerializer() {
        return new GenericJackson2JsonRedisSerializer(
            new ObjectMapper()
                .registerModule(new JavaTimeModule())  // 支持 Java 8 时间
                .activateDefaultTyping(
                    BasicPolymorphicTypeValidator.builder()
                        .allowIfBaseType(Object.class)
                        .build()
                )
        );
    }
}
```

### 问题二：Session 并发竞争

多个请求同时修改 Session，可能导致数据覆盖：

```java
// 使用乐观锁
@Bean
public RedisTemplate<String, Object> redisTemplate() {
    RedisTemplate<String, Object> template = new RedisTemplate<>();
    template.setKeySerializer(new StringRedisSerializer());
    template.setValueSerializer(new Jackson2JsonRedisSerializer<>(Object.class));

    // 开启事务支持
    template.setEnableTransactionSupport(true);
    return template;
}

// 服务层使用事务
public void updateSessionWithTransaction(HttpSession session, String key, Object value) {
    redisTemplate.execute(new SessionCallback<Object>() {
        @Override
        public Object execute(RedisOperations operations) throws DataAccessException {
            operations.watch(key);  // 监视 key

            // 在事务中执行操作
            operations.multi();
            operations.opsForValue().set(getSessionKey(session) + ":" + key, value);
            operations.exec();

            return null;
        }
    });
}
```

```java

HttpSession session = request.getSession();
```

## 总结

分布式 Session 的核心是**集中存储 + 共享访问**：

| 组件 | 作用 |
|---|---|
| Redis | Session 数据存储 |
| Spring Session | Session 生命周期管理 |
| Filter/拦截器 | 自动处理 Session 读写 |

Spring Session + Redis 的优势：

1. **无侵入**：几乎不需要修改业务代码
2. **高性能**：Redis 内存存储，毫秒级响应
3. **易扩展**：增加实例只需配置 Redis 连接
4. **高可用**：配合 Redis Cluster 或哨兵模式

记住三个最佳实践：

1. Session 中只存储必要的数据，不要存储大对象
2. 敏感信息（如密码）不要存入 Session
3. 设置合理的 Session 过期时间，平衡安全与用户体验
