# OAuth 2.0 在微服务中的使用：授权服务与资源服务分离

你用微信登录了一个第三方应用，它怎么知道你是谁？它能读取你的微信聊天记录吗？你的微信密码它知道吗？

这一切的答案，都在 OAuth 2.0 里。

## 为什么需要 OAuth 2.0？

### 传统方式的问题

假设我们有一个第三方应用「云音乐」，用户想用微信登录：

**方案一：用户把微信密码给云音乐**

```java
// 云音乐服务器
public void loginWithWechat(String username, String password) {
    // 传给微信验证
    boolean valid = wechatApi.verify(username, password);
    // 云音乐服务器拿到了用户的微信密码！
}
```

这太危险了。云音乐拿到了用户的微信密码，它可以：

- 读取用户微信聊天记录
- 以用户身份发消息
- 修改用户微信设置

**方案二：OAuth 2.0**

用户授权云音乐访问「获取用户信息」的能力，但不暴露密码。云音乐只能看到用户同意它看的。

## OAuth 2.0 四种授权模式

| 模式 | 适用场景 | 安全性 |
|---|---|---|
| 授权码模式（Authorization Code） | 有后端服务器的 Web 应用 | 最高 |
| 隐式授权模式（Implicit） | 纯前端 SPA 应用 | 中（已废弃） |
| 密码凭证模式（Password Credentials） | 受信任的第一方应用 | 低 |
| 客户端凭证模式（Client Credentials） | 服务间通信 | 中 |

### 授权码模式（最安全，推荐使用）

### 核心配置

```java
// OAuth2 服务器配置
@Configuration
@EnableAuthorizationServer
public class OAuth2AuthorizationConfig extends AuthorizationServerConfigurerAdapter {

    @Autowired
    private AuthenticationManager authenticationManager;

    @Autowired
    private TokenStore tokenStore;

    @Autowired
    private ClientDetailsService clientDetailsService;

    @Override
    public void configure(AuthorizationServerEndpointsConfigurer endpoints) throws Exception {
        endpoints
            .authenticationManager(authenticationManager)
            .tokenStore(tokenStore)
            .authorizationCodeServices(authorizationCodeServices())
            .tokenServices(tokenServices())
            .allowedTokenEndpointRequestMethods(HttpMethod.GET, HttpMethod.POST);
    }

    @Override
    public void configure(ClientDetailsServiceConfigurer clients) throws Exception {
        clients.withClientDetails(clientDetailsService);
    }

    @Bean
    public AuthorizationCodeServices authorizationCodeServices() {
        return new JdbcAuthorizationCodeServices(dataSource);
    }

    @Bean
    public AuthorizationServerTokenServices tokenServices() {
        DefaultAuthorizationServerTokenServices services = new DefaultAuthorizationServerTokenServices();
        services.setSupportRefreshToken(true);
        services.setTokenStore(tokenStore);
        services.setClientDetailsService(clientDetailsService);
        services.setAccessTokenValiditySeconds(3600);      // 1 小时
        services.setRefreshTokenValiditySeconds(86400 * 7); // 7 天
        return services;
    }
}
```

### 授权端点实现

```java
@RestController
@RequestMapping("/oauth")
public class OAuthController {

    @Autowired
    private AuthorizationServerTokenServices tokenServices;

    @Autowired
    private ClientDetailsService clientDetailsService;

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    /**
     * 授权码模式 - 授权请求
     * GET /oauth/authorize?response_type=code&client_id=app1&redirect_uri=...
     */
    @GetMapping("/authorize")
    public void authorize(Authentication authentication,
                          @RequestParam String response_type,
                          @RequestParam String client_id,
                          @RequestParam String redirect_uri,
                          @RequestParam(required = false) String scope,
                          @RequestParam(required = false) String state,
                          HttpServletResponse response) throws IOException {

        // 1. 验证客户端
        ClientDetails client = clientDetailsService.loadClientByClientId(client_id);
        if (client == null) {
            throw new OAuth2Exception("未知客户端");
        }

        // 2. 验证回调地址
        if (!client.getRegisteredRedirectUri().contains(redirect_uri)) {
            throw new OAuth2Exception("无效的回调地址");
        }

        // 3. 生成授权确认页面（简化版，实际应该有前端页面）
        String confirmPage = buildConfirmPage(client_id, scope, redirect_uri, state);
        response.setContentType("text/html");
        response.getWriter().write(confirmPage);
    }

    /**
     * 授权确认
     * POST /oauth/authorize (with form params: user_oauth_approval=true)
     */
    @PostMapping("/authorize")
    public ResponseEntity<?> authorizeConfirm(Authentication authentication,
                                             @RequestParam String client_id,
                                             @RequestParam String redirect_uri,
                                             @RequestParam(required = false) String state,
                                             @RequestParam Map<String, String> scopeMap) {

        // 1. 生成授权码
        String authCode = generateAuthCode(authentication.getName(), client_id, scopeMap);

        // 2. 存储授权码（有效期 10 分钟）
        String key = "oauth:authcode:" + authCode;
        AuthCode code = new AuthCode(authCode, authentication.getName(), client_id,
            redirect_uri, scopeMap, System.currentTimeMillis());
        redisTemplate.opsForValue().set(key, code, Duration.ofMinutes(10));

        // 3. 重定向回客户端
        String redirectUrl = redirect_uri + "?code=" + authCode;
        if (state != null) {
            redirectUrl += "&state=" + state;
        }

        return ResponseEntity.status(302)
            .header("Location", redirectUrl)
            .build();
    }

    /**
     * 换取 Token
     * POST /oauth/token
     */
    @PostMapping("/token")
    public ResponseEntity<OAuth2AccessToken> token(
            @RequestParam String grant_type,
            @RequestParam String code,
            @RequestParam String redirect_uri,
            @RequestParam String client_id,
            @RequestHeader("Authorization") String authHeader) {

        // 1. 验证客户端
        ClientDetails client = validateClient(client_id, authHeader);

        // 2. 验证授权码
        String key = "oauth:authcode:" + code;
        AuthCode authCode = (AuthCode) redisTemplate.opsForValue().get(key);
        if (authCode == null) {
            throw new OAuth2Exception("授权码无效或已过期");
        }

        // 3. 验证回调地址和客户端
        if (!authCode.getClientId().equals(client_id) || !authCode.getRedirectUri().equals(redirect_uri)) {
            throw new OAuth2Exception("参数不匹配");
        }

        // 4. 删除授权码（一次性使用）
        redisTemplate.delete(key);

        // 5. 构建 Token 请求
        TokenRequest tokenRequest = new TokenRequest(
            new HashMap<>(), client_id, client.getScope(), grant_type
        );

        // 6. 生成 Token
        OAuth2AccessToken token = tokenServices.createAccessToken(
            new OAuth2Authentication(client, null, parseScopes(authCode.getScopes()))
        );

        return ResponseEntity.ok(token);
    }

    /**
     * 刷新 Token
     * POST /oauth/token?grant_type=refresh_token&refresh_token=...
     */
    @PostMapping("/token")
    public ResponseEntity<OAuth2AccessToken> refreshToken(
            @RequestParam String grant_type,
            @RequestParam String refresh_token,
            @RequestParam String client_id,
            @RequestHeader("Authorization") String authHeader) {

        if (!"refresh_token".equals(grant_type)) {
            throw new OAuth2Exception("不支持的授权类型");
        }

        OAuth2RefreshToken refreshToken = new DefaultOAuth2RefreshToken(refresh_token);
        OAuth2AccessToken token = tokenServices.refreshAccessToken(refreshToken, null);

        return ResponseEntity.ok(token);
    }
}
```

### Token 存储策略

```java
// Redis Token 存储（推荐）
@Configuration
public class TokenStoreConfig {

    @Bean
    public TokenStore tokenStore(RedisConnectionFactory factory) {
        // 使用 Redis 存储，支持集群部署
        RedisTokenStore tokenStore = new RedisTokenStore(factory);
        tokenStore.setPrefix("oauth:token:");

        // 设置序列化方式
        tokenStore.setSerializationStrategy(new Jackson2SerializationRedisSerializer<>(OAuth2AccessToken.class));

        return tokenStore;
    }
}
```

## 资源服务实现

### 资源服务器配置

```java
@Configuration
@EnableResourceServer
public class ResourceServerConfig extends ResourceServerConfigurerAdapter {

    @Value("${auth.server.url}")
    private String authServerUrl;

    @Override
    public void configure(ResourceServerSecurityConfigurer resources) throws Exception {
        resources
            .tokenServices(tokenServices())
            .resourceId("user-api");
    }

    @Override
    public void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/public/**").permitAll()
                .antMatchers("/actuator/**").permitAll()
                .anyRequest().authenticated();
    }

    @Bean
    public ResourceServerTokenServices tokenServices() {
        // 使用远程 Token 验证（授权服务和资源服务分离）
        RemoteTokenServices services = new RemoteTokenServices();
        services.setCheckTokenEndpointUrl(authServerUrl + "/oauth/check_token");
        services.setClientId("resource-server");
        services.setClientSecret("resource-secret");
        services.setAccessTokenConverter(accessTokenConverter());
        return services;
    }

    @Bean
    public JwtAccessTokenConverter accessTokenConverter() {
        // 如果使用 JWT，这里配置签名密钥
        JwtAccessTokenConverter converter = new JwtAccessTokenConverter();
        converter.setSigningKey("your-signing-key");
        return converter;
    }
}
```

### 受保护的资源接口

```java
@RestController
@RequestMapping("/api/user")
public class UserController {

    @GetMapping("/info")
    public ResponseEntity<?> getUserInfo(Principal principal) {
        if (principal == null) {
            return ResponseEntity.status(401).body("Unauthorized");
        }

        // 从 Authentication 中获取用户信息
        Map<String, Object> userInfo = new HashMap<>();
        userInfo.put("username", principal.getName());
        userInfo.put("authorities", ((OAuth2AuthenticationDetails) principal)
            .getOAuth2Request().getScope());

        return ResponseEntity.ok(userInfo);
    }

    @GetMapping("/profile")
    @PreAuthorize("#oauth2.hasScope('profile')")
    public ResponseEntity<?> getUserProfile(Principal principal) {
        // 只有请求了 profile 权限才能访问
        return ResponseEntity.ok(getUserProfileData(principal.getName()));
    }

    @PostMapping("/update")
    @PreAuthorize("hasAuthority('SCOPE_update')")
    public ResponseEntity<?> updateProfile(Principal principal,
                                          @RequestBody UserUpdateRequest request) {
        // 有 update 权限才能修改
        return ResponseEntity.ok("更新成功");
    }
}
```

## 常见问题与解决

### 问题一：Token 被截获

```java
// 生产环境必须使用 HTTPS
// 同时验证请求来源
@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        .requiresChannel()
            .anyRequest().requiresSecure()  // 强制 HTTPS
        .and()
        .csrf()
            .requireCsrfProtectionMatcher(request ->
                ".*oauth.*".matches(request.getRequestURI()) ?
                    false : true  // Token 接口启用 CSRF
            );
}
```

### 问题二：Token 泄露后的保护

```java
// 使用 Token 的指纹验证
public class TokenSecurityService {

    public String createSecureToken(OAuth2AccessToken token, HttpServletRequest request) {
        // 添加客户端指纹
        String fingerprint = getFingerprint(request);
        String secureToken = token.getValue() + "." + fingerprint;

        // 存储指纹到 Redis
        String key = "token:fp:" + token.getValue();
        redisTemplate.opsForValue().set(key, fingerprint, Duration.ofHours(1));

        return secureToken;
    }

    public boolean validateFingerprint(String token, HttpServletRequest request) {
        String expectedFingerprint = getFingerprint(request);
        String key = "token:fp:" + token;

        String storedFingerprint = (String) redisTemplate.opsForValue().get(key);
        return expectedFingerprint.equals(storedFingerprint);
    }

    private String getFingerprint(HttpServletRequest request) {
        // 结合多个因素生成指纹
        String userAgent = request.getHeader("User-Agent");
        String accept = request.getHeader("Accept");
        return DigestUtils.md5Hex(userAgent + "|" + accept);
    }
}
```

### 问题三：Token 撤销

```java
// 注销时撤销 Token
@Service
public class TokenRevocationService {

    @Autowired
    private TokenStore tokenStore;

    public void revokeToken(String tokenValue) {
        OAuth2AccessToken token = tokenStore.readAccessToken(tokenValue);
        if (token != null) {
            // 删除 Access Token
            tokenStore.removeAccessToken(token);

            // 如果有 Refresh Token，也删除
            OAuth2RefreshToken refreshToken = token.getRefreshToken();
            if (refreshToken != null) {
                tokenStore.removeRefreshToken(refreshToken);
            }
        }
    }

    // 根据用户撤销所有 Token
    public void revokeAllUserTokens(String username) {
        Collection<OAuth2AccessToken> tokens = tokenStore.findTokensByUserName(username);
        tokens.forEach(tokenStore::removeAccessToken);
    }
}
```

## 总结

OAuth 2.0 的核心是**授权与资源分离**：

| 组件 | 职责 |
|---|---|
| 授权服务器 | 验证用户身份、签发 Token、管理授权 |
| 资源服务器 | 验证 Token、返回受保护的资源 |
| 客户端 | 代表用户访问资源，但不获取用户密码 |

四种授权模式的选择：

| 场景 | 推荐模式 |
|---|---|
| 有后端的 Web 应用 | 授权码模式 |
| 纯前端 SPA | 授权码模式 + PKCE |
| 公司内部系统 | 密码凭证模式 |
| 服务间调用 | 客户端凭证模式 |

记住一个原则：**永远不要把 Token 直接暴露在前端 URL 中**。Token 应该通过 Authorization 头传递。
