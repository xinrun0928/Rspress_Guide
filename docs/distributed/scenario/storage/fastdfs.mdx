# FastDFS 分布式文件系统

如果 MinIO 是为云原生时代设计的对象存储，那 FastDFS 就是为小文件场景而生的「轻量级选手」。

FastDFS 起源于中国，是一款纯 C 实现的轻量级分布式文件系统，特别适合图片、音视频等中小文件的存储场景。国内很多互联网公司早期都用过它。

今天我们来深入剖析 FastDFS 的架构、部署和实战应用。

## 一、FastDFS 简介

### 1.1 核心特点

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        FastDFS 特点                                     │
│                                                                         │
│   ✓ 轻量级：纯 C 实现，代码简洁                                         │
│   ✓ 高性能：同步 IO，支持大并发                                         │
│   ✓ 高可用：Group 分组，存储节点冗余                                    │
│   ✓ 无中心：Tracker 无状态，可水平扩展                                   │
│   ✓ 简单协议：自定义协议，避免 HTTP 复杂开销                            │
│   ✓ 小文件优化：针对中小文件特别优化                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 与 MinIO 对比

| 维度 | FastDFS | MinIO |
|---|---|---|
| 协议 | 自定义协议 | S3 HTTP API |
| 语言 | C | Go |
| 适用文件 | 中小文件 | 任意大小 |
| 特点 | 小文件性能高 | 功能丰富 |
| 生态 | 一般 | 丰富（S3 生态） |
| 社区 | 国内为主 | 国际 |

## 二、架构设计

### 2.1 核心组件

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        FastDFS 架构                                    │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                        客户端                                    │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                    │                                    │
│                                    ▼                                    │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                    Tracker Server（调度器）                      │  │
│   │                    - 心跳检测                                    │  │
│   │                    - 负载均衡                                    │  │
│   │                    - 存储调度                                    │  │
│   │                    - 无状态，可集群                               │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                    │                                    │
│              ┌─────────────────────┼─────────────────────┐             │
│              ▼                     ▼                     ▼             │
│   ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐      │
│   │   Group 1       │   │   Group 2       │   │   Group 3       │      │
│   │  ┌───────────┐  │   │  ┌───────────┐  │   │  ┌───────────┐  │      │
│   │  │Storage 1.1│  │   │  │Storage 2.1│  │   │  │Storage 3.1│  │      │
│   │  │Storage 1.2│  │   │  │Storage 2.2│  │   │  │Storage 3.2│  │      │
│   │  └───────────┘  │   │  └───────────┘  │   │  └───────────┘  │      │
│   └─────────────────┘   └─────────────────┘   └─────────────────┘      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 核心概念

| 概念 | 说明 |
|---|---|
| Tracker | 调度器，管理 Storage 节点，不存储文件 |
| Storage | 存储节点，实际存储文件，可配置多副本 |
| Group | 分组，不同组可存不同类型文件 |
| file_id | 文件唯一标识，格式：`group/目录/文件名/维度` |

### 2.3 文件路由流程

```
客户端上传文件：
1. 客户端请求 Tracker 获取可用 Storage
2. Tracker 返回一个 Storage 地址
3. 客户端直接连接 Storage 上传文件
4. Storage 生成 file_id，返回给客户端

客户端下载文件：
1. 客户端携带 file_id 请求 Tracker
2. Tracker 查询 file_id 所在 Storage
3. Tracker 返回 Storage 地址
4. 客户端直接连接 Storage 下载文件
```

## 三、部署配置

### 3.1 Tracker 配置

```ini title="tracker.conf"
# tracker 服务端口
port=22122

# 存储日志目录
base_path=/fastdfs/tracker

# 存储配置文件目录
# 会在此目录创建 storage_groups_servers.conf
store_lookup=2
store_group=group1

# 绑定 IP（多网卡时使用）
bind_addr=

# Tracker 集群配置
# 可配置多个 tracker server 地址
```

### 3.2 Storage 配置

```ini title="storage.conf"
# storage 服务端口
port=23000

# 所属 group
group_name=group1

# 存储日志目录
base_path=/fastdfs/storage

# 存储数据目录（可配置多个）
store_path0=/fastdfs/storage/data
# store_path1=/mnt/disk2/fastdfs

# tracker 服务器地址
tracker_server=192.168.1.101:22122
tracker_server=192.168.1.102:22122

# 绑定 IP
bind_addr=

# 存储文件同步配置
sync_log_buff_interval=10
sync_stat_file_interval=300
```

### 3.3 启动服务

```bash title="启动 FastDFS"
# 启动 Tracker
fdfs_trackerd /etc/fdfs/tracker.conf start

# 启动 Storage
fdfs_storaged /etc/fdfs/storage.conf start

# 检查服务状态
fdfs_monitor /etc/fdfs/client.conf
```

### 4.3 文件上传服务

```java title="FastDFS 文件服务")
@Service
@Slf4j
public class FastDFSFileService {
    @Autowired
    private FastFileStorageClient storageClient;

    /**
     * 上传文件
     */
    public String uploadFile(MultipartFile file) throws Exception {
        String originalFilename = file.getOriginalFilename();
        String extension = originalFilename.substring(originalFilename.lastIndexOf(".") + 1);

        StorePath storePath = storageClient.uploadFile(
                file.getInputStream(),
                file.getSize(),
                extension,
                null
        );

        return storePath.getFullPath();
    }

    /**
     * 上传图片（带缩略图）
     */
    public String uploadImage(MultipartFile file) throws Exception {
        String originalFilename = file.getOriginalFilename();
        String extension = originalFilename.substring(originalFilename.lastIndexOf(".") + 1);

        StorePath storePath = storageClient.uploadImageAndCreateThumbImage(
                file.getInputStream(),
                file.getSize(),
                extension,
                null
        );

        return storePath.getFullPath();
    }

    /**
     * 获取缩略图路径
     */
    public String getThumbImagePath(String filePath) {
        // FastDFS 缩略图路径格式：原图路径_thumb
        return filePath + "_150x150";
    }

    /**
     * 下载文件
     */
    public byte[] downloadFile(String filePath) throws Exception {
        return storageClient.downloadFile(
                filePath.substring(0, filePath.indexOf("/")),
                filePath.substring(filePath.indexOf("/") + 1),
                new DownloadByteArray()
        );
    }

    /**
     * 删除文件
     */
    public void deleteFile(String filePath) {
        storageClient.deleteFile(filePath);
    }

    /**
     * 获取文件访问 URL
     */
    public String getAccessUrl(String filePath, String nginxUrl) {
        return nginxUrl + "/" + filePath;
    }
}
```

### 4.4 文件上传控制器

```java title="上传控制器")
@RestController
@RequestMapping("/api/file")
public class FileController {
    @Autowired
    private FastDFSFileService fileService;

    @Value("${file.nginx-url}")
    private String nginxUrl;

    @PostMapping("/upload")
    public Result<String> upload(@RequestParam("file") MultipartFile file) {
        try {
            String filePath = fileService.uploadFile(file);
            String accessUrl = fileService.getAccessUrl(filePath, nginxUrl);
            return Result.success(accessUrl);
        } catch (Exception e) {
            log.error("文件上传失败", e);
            return Result.fail("上传失败：" + e.getMessage());
        }
    }

    @PostMapping("/upload/image")
    public Result<Map<String, String>> uploadImage(@RequestParam("file") MultipartFile file) {
        try {
            String filePath = fileService.uploadImage(file);
            String accessUrl = fileService.getAccessUrl(filePath, nginxUrl);
            String thumbUrl = fileService.getAccessUrl(
                    fileService.getThumbImagePath(filePath), nginxUrl);

            Map<String, String> result = new HashMap<>();
            result.put("url", accessUrl);
            result.put("thumbUrl", thumbUrl);
            return Result.success(result);
        } catch (Exception e) {
            log.error("图片上传失败", e);
            return Result.fail("上传失败：" + e.getMessage());
        }
    }

    @DeleteMapping("/delete")
    public Result<Void> delete(@RequestParam String filePath) {
        try {
            fileService.deleteFile(filePath);
            return Result.success();
        } catch (Exception e) {
            return Result.fail("删除失败：" + e.getMessage());
        }
    }
}
```

    server 192.168.1.103:23000;
    server 192.168.1.104:23000;

server {
    listen 80;
    server_name file.example.com;

    # FastDFS 模块配置
    location /group1/M00 {
        ngx_fastdfs_module;

### 8.2 Java 监控服务

```java title="监控服务")
@Service
public class FastDFSMonitorService {
    @Autowired
    private TrackerClient trackerClient;

    public Map<String, Object> getStorageInfo() throws Exception {
        TrackerServer trackerServer = trackerClient.getConnection();
        try {
            List<GroupInfo> groupInfos = trackerClient.listGroups(trackerServer);
            Map<String, Object> result = new HashMap<>();

            for (GroupInfo group : groupInfos) {
                List<StorageServer> storageServers =
                        trackerClient.getStorageServers(trackerServer, group.getGroupName());

                for (StorageServer storage : storageServers) {
                    StorageInfo storageInfo = new StorageInfo();
                    storageInfo.setGroupName(group.getGroupName());
                    storageInfo.setIp(storage.getIpAddr());
                    storageInfo.setPort(storage.getPort());
                    storageInfo.setTotalSpace(storage.getTotalSpace());
                    storageInfo.setFreeSpace(storage.getFreeSpace());

                    result.put(storage.getIpAddr(), storageInfo);
                }
            }
            return result;
        } finally {
            trackerServer.close();
        }
    }
}
```

## 九、常见问题

### 9.1 文件 ID 为空

```
原因：Storage 未正确注册到 Tracker
解决：检查网络连通性，确认 Storage 和 Tracker 配置一致
```

### 9.2 下载 404

```
原因：文件同步未完成
解决：等待同步完成，或检查同步配置
```

### 9.3 性能问题

```
原因：单 Storage 压力大
解决：增加同组 Storage 节点，或增加新 Group
```

## 十、总结对比

| 场景 | 推荐方案 |
|---|---|
| 小文件（`< 4MB`） | FastDFS |
| 中等文件（4MB - 500MB） | FastDFS / MinIO |
| 大文件（`>` 500MB） | MinIO |
| 需要 S3 兼容 | MinIO |
| 需要国际化生态 | MinIO |
| 国内互联网项目 | FastDFS |

:::tip 选型建议
- **新项目**：直接选择 MinIO，功能更丰富
- **小文件场景**：FastDFS 性能更好
- **已有 FastDFS**：可以继续使用，没必要迁移
:::

## 结尾

FastDFS 是一款经典的国产分布式文件系统，它的「简单」和「高效」让它在特定场景下依然是首选。

记住两个要点：

1. **Tracker 无状态**：可以水平扩展，不怕单点故障
2. **小文件场景**：FastDFS 依然是性价比最高的选择
