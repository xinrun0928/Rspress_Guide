# CDN + OSS 分发加速架构设计

你的用户遍布全国，有人访问东北节点，有人访问华南节点。如果所有请求都回源到杭州的服务器，东北用户打开一张图片可能要 3 秒。

CDN + OSS 就是来解决这个问题的：把文件放到离用户更近的地方。

## CDN + OSS 架构

### 为什么需要 CDN？

```
┌──────────────────────────────────────────────────────────────────┐
│                        无 CDN 架构                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│   用户 A (北京)  ─────────────────────>  OSS (杭州)  延迟 80ms    │
│   用户 B (广州)  ─────────────────────>  OSS (杭州)  延迟 60ms    │
│   用户 C (上海)  ─────────────────────>  OSS (杭州)  延迟 20ms    │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│                        CDN + OSS 架构                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│   用户 A (北京)  ────> CDN 节点 (北京) ── Cache Hit ──> OSS (杭州)│
│   用户 B (广州)  ────> CDN 节点 (广州) ── Cache Hit ──> OSS (杭州)│
│   用户 C (上海)  ────> CDN 节点 (上海) ── Cache Hit ──> OSS (杭州)│
│                                                                   │
│   首次访问：用户 ──> CDN 节点 ──> OSS（回源，延迟 20ms）          │
│   后续访问：用户 ──> CDN 节点 ── 直接返回（命中缓存，延迟 5ms）   │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 核心优势

| 优势 | 说明 |
|---|---|
| 降低延迟 | 用户访问最近的 CDN 节点，比直连 OSS 快 5-10 倍 |
| 减轻源站压力 | 热点内容由 CDN 节点直接返回，OSS 压力大幅降低 |
| 提升可用性 | CDN 节点分布广泛，单节点故障不影响整体服务 |
| 节省带宽成本 | OSS 外网流量计费，CDN 内网回源成本更低 |

### 存储分层

| 层级 | 存储类型 | 特点 | 适用场景 |
|---|---|---|---|
| L0 | CDN 边缘节点 | 就近访问，毫秒级响应 | 热点图片、视频 |
| L1 | CDN 区域中心 | 命中边缘节点缓存后自动降温 | 中等热度内容 |
| L2 | OSS 标准存储 | 成本低，容量大 | 常规文件存储 |
| L3 | OSS 归档存储 | 成本极低，取回需分钟 | 冷数据、备份文件 |

### CDN 缓存策略

```java
@Service
public class CdnCacheService {

    private final CDNClient cdnClient;

    /**
     * 配置 URL 级别的缓存策略
     */
    public void configureCacheRules() {
        // 1. 图片缓存 30 天
        CdnCacheRule imageRule = CdnCacheRule.builder()
            .setRuleType("suffix")
            .setRuleValue(Arrays.asList(".jpg", ".jpeg", ".png", ".gif", ".webp"))
            .setCacheTTL(30 * 24 * 3600)  // 30 天
            .setCacheUnit("day")
            .setWeight(1)
            .build();

        // 2. 视频缓存 7 天
        CdnCacheRule videoRule = CdnCacheRule.builder()
            .setRuleType("suffix")
            .setRuleValue(Arrays.asList(".mp4", ".webm", ".m3u8"))
            .setCacheTTL(7 * 24 * 3600)  // 7 天
            .setCacheUnit("day")
            .setWeight(1)
            .build();

        // 3. JS/CSS 缓存 1 天（带 hash 的文件名）
        CdnCacheRule staticRule = CdnCacheRule.builder()
            .setRuleType("regex")
            .setRuleValue("\\.[0-9a-f]{8}\\.(js|css)$")
            .setCacheTTL(24 * 3600)
            .setCacheUnit("second")
            .setWeight(2)  // 优先级更高
            .build();

        // 4. HTML 不缓存
        CdnCacheRule noCacheRule = CdnCacheRule.builder()
            .setRuleType("suffix")
            .setRuleValue(Collections.singletonList(".html"))
            .setCacheTTL(0)
            .setCacheUnit("second")
            .setWeight(10)  // 最高优先级
            .build();

        cdnClient.setCacheRules(Arrays.asList(
            noCacheRule, imageRule, videoRule, staticRule
        ));
    }

    /**
     * 刷新 CDN 缓存
     */
    public void refreshCache(String url) {
        RefreshObjectRequest request = RefreshObjectCachesRequest.builder()
            .setObjectType("File")
            .setObjectPath(url)
            .build();

        RefreshObjectCachesResponse response = cdnClient.refreshObjectCaches(request);
        System.out.println("刷新任务 ID: " + response.getRefreshTaskId());
    }

    /**
     * 批量刷新缓存
     */
    public void batchRefreshCache(List<String> urls) {
        RefreshObjectCachesRequest request = RefreshObjectCachesRequest.builder()
            .setObjectType("File")
            .setObjectPaths(urls)
            .build();

        cdnClient.refreshObjectCaches(request);
    }

    /**
     * 预热资源到 CDN
     * 将热点资源提前加载到 CDN 节点
     */
    public void preloadToCache(String url) {
        PushObjectCacheRequest request = PushObjectCacheRequest.builder()
            .setObjectPath(url)
            .build();

        cdnClient.pushObjectCache(request);
    }
}
```

### 防盗链配置

```java
@Service
public class CdnSecurityService {

    private final OSS ossClient;
    private final CDNClient cdnClient;

    /**
     * 配置 Referer 防盗链
     */
    public void configureRefererAntiLeech() {
        // 设置允许的 Referer
        List<String> refererList = Arrays.asList(
            "https://www.yourdomain.com",
            "https://*.yourdomain.com"
        );

        cdnClient.setRefererConfig(
            SetRefererConfigRequest.builder()
                .setRefererType("whitelist")
                .setReferers(refererList)
                .setAllowEmpty(true)  // 允许空 Referer
                .build()
        );
    }

    /**
     * 生成 URL 鉴权的签名
     */
    public String generateSignedUrl(String originalUrl, long expirationTime) {
        // 鉴权方式 A
        // 格式: http://DomainName/Filename?auth_key=timestamp-rand-uid-md5hash

        String timestamp = String.valueOf(expirationTime / 1000);
        String rand = UUID.randomUUID().toString().replace("-", "").substring(0, 16);
        String uid = "0";  // 用户 ID

        String stringToSign = originalUrl + timestamp + rand + uid + SECRET_KEY;
        String md5hash = DigestUtils.md5Hex(stringToSign);

        return originalUrl + "?auth_key=" + timestamp + "-" + rand + "-" + uid + "-" + md5hash;
    }

    /**
     * 生成带签名的播放器 Token
     * 用于视频防盗链
     */
    public VideoPlaybackToken generateVideoToken(String videoPath,
                                                  String clientIp,
                                                  long expiresIn) {
        long timestamp = System.currentTimeMillis() / 1000 + expiresIn;

        // 构建签名内容
        String signContent = String.format("%s-%s-%d-%s",
            videoPath, clientIp, timestamp, SECRET_KEY);
        String signature = DigestUtils.md5Hex(signContent);

        return new VideoPlaybackToken()
            .setToken(signature)
            .setTimestamp(timestamp)
            .setVideoPath(videoPath);
    }
}
```

## 文件上传与分发流程

### 完整的上传分发流程

```java
@Service
public class FileDistributionService {

    private final OSS ossClient;
    private final CDNClient cdnClient;
    private final FileUploadService uploadService;

    /**
     * 上传文件并自动配置 CDN
     */
    public FileUploadResult uploadFile(MultipartFile file) {
        try {
            // 1. 生成唯一的文件路径
            String objectName = generateObjectPath(file.getOriginalFilename());

            // 2. 上传到 OSS
            String bucketName = "your-bucket";
            ossClient.putObject(
                PutObjectRequest.builder()
                    .bucket(bucketName)
                    .object(objectName)
                    .inputStream(file.getInputStream())
                    .contentLength(file.getSize())
                    .contentType(file.getContentType())
                    .build()
            );

            // 3. 设置 HTTP 头（缓存控制、CORS 等）
            ossClient.setObjectMeta(
                SetObjectMetaRequest.builder()
                    .bucket(bucketName)
                    .object(objectName)
                    .contentType(file.getContentType())
                    .cacheControl("max-age=31536000")  // 缓存 1 年
                    .build()
            );

            // 4. 生成 CDN 访问 URL
            String cdnDomain = "https://cdn.yourdomain.com";
            String cdnUrl = cdnDomain + "/" + objectName;

            // 5. 返回结果
            return new FileUploadResult()
                .setObjectName(objectName)
                .setOssUrl(ossClient.getObjectUrl(bucketName, objectName))
                .setCdnUrl(cdnUrl)
                .setFileSize(file.getSize());

        } catch (Exception e) {
            throw new UploadException("上传文件失败", e);
        }
    }

    /**
     * 处理大文件（分片上传 + CDN）
     */
    public String uploadLargeFile(String uploadId, List<UploadedPart> parts) {
        // 1. 完成 OSS 分片上传
        String objectName = uploadService.completeUpload(uploadId, parts);

        // 2. 生成 CDN URL
        String cdnUrl = "https://cdn.yourdomain.com/" + objectName;

        // 3. 预热到 CDN
        cdnCacheService.preloadToCache(cdnUrl);

        return cdnUrl;
    }

    /**
     * 批量处理并分发
     */
    public List<FileUploadResult> batchUploadAndDistribute(List<MultipartFile> files) {
        return files.parallelStream()
            .map(this::uploadFile)
            .collect(Collectors.toList());
    }

    /**
     * 发布新版本静态资源
     * 1. 上传带 hash 的新文件
     * 2. 刷新旧文件缓存
     * 3. 返回新的 URL
     */
    public String publishNewVersion(String originalPath, MultipartFile newFile) {
        // 1. 生成新路径（带 hash）
        String newPath = generateHashedPath(originalPath, newFile);

        // 2. 上传到 OSS
        String objectName = "static/" + newPath;
        ossClient.putObject(
            PutObjectRequest.builder()
                .bucket("your-bucket")
                .object(objectName)
                .inputStream(newFile.getInputStream())
                .contentLength(newFile.getSize())
                .contentType(newFile.getContentType())
                .build()
        );

        // 3. 刷新旧版本缓存
        cdnCacheService.refreshCache("https://cdn.yourdomain.com/" + originalPath);

        return "https://cdn.yourdomain.com/" + objectName;
    }
}
```

## 智能缓存策略

### 基于访问模式的缓存优化

```java
@Service
public class IntelligentCacheService {

    private final CDNClient cdnClient;
    private final RedisTemplate<String, Object> redisTemplate;

    private static final String HOT_RESOURCE_KEY = "cdn:hot:resources";
    private static final String ACCESS_COUNT_KEY = "cdn:access:count:";

    /**
     * 分析访问热度，动态调整缓存策略
     */
    public void analyzeAndOptimizeCache() {
        // 1. 获取 CDN 访问日志
        List<CdnAccessLog> logs = fetchAccessLogs();

        // 2. 统计每个资源的访问次数
        Map<String, Long> resourceAccessCount = logs.stream()
            .collect(Collectors.groupingBy(
                CdnAccessLog::getObjectPath,
                Collectors.counting()
            ));

        // 3. 识别热点资源
        List<String> hotResources = resourceAccessCount.entrySet().stream()
            .filter(e -> e.getValue() > 10000)  // 访问量超过 1 万
            .sorted(Map.Entry.<String, Long>comparingByValue().reversed())
            .limit(100)
            .map(Map.Entry::getKey)
            .collect(Collectors.toList());

        // 4. 预热热点资源
        for (String resource : hotResources) {
            String cdnUrl = "https://cdn.yourdomain.com/" + resource;
            cdnClient.pushObjectCache(
                PushObjectCacheRequest.builder()
                    .setObjectPath(cdnUrl)
                    .build()
            );
        }

        // 5. 更新 Redis 热点列表
        redisTemplate.delete(HOT_RESOURCE_KEY);
        redisTemplate.opsForList().rightPushAll(HOT_RESOURCE_KEY, hotResources);
    }

    /**
     * 根据访问热度返回不同的 CDN 节点
     * 热点资源优先返回边缘节点
     */
    public String getOptimalCdnUrl(String resourcePath) {
        // 检查是否是热点资源
        Boolean isHot = redisTemplate.opsForSet().isMember(HOT_RESOURCE_KEY, resourcePath);

        if (Boolean.TRUE.equals(isHot)) {
            // 热点资源：优先返回边缘节点
            return "https://cdn.yourdomain.com/" + resourcePath;
        } else {
            // 非热点资源：返回区域中心节点
            return "https://regional-cdn.yourdomain.com/" + resourcePath;
        }
    }
}
```

### 缓存失效策略

```java
@Service
public class CacheInvalidationService {

    private final CDNClient cdnClient;

    /**
     * 版本化缓存失效
     * 所有静态资源使用版本号，修改时改版本号即可
     */
    public String getVersionedUrl(String resourcePath, String version) {
        // 原路径: /static/app.js
        // 版本路径: /static/v1.0.0/app.js
        return "https://cdn.yourdomain.com/static/" + version + "/" +
            resourcePath.replace("/static/", "");
    }

    /**
     * 目录级刷新
     */
    public void refreshDirectory(String directoryPath) {
        // 刷新整个目录
        String fullPath = "https://cdn.yourdomain.com/" + directoryPath + "/*";
        cdnClient.refreshObjectCaches(
            RefreshObjectCachesRequest.builder()
                .setObjectType("Directory")
                .setObjectPath(fullPath)
                .build()
        );
    }

    /**
     * 使用 Tag 刷新
     * 给资源打标签，按标签批量刷新
     */
    public void refreshByTag(String tag) {
        // 获取该标签下的所有资源
        List<String> resourcePaths = getResourcesByTag(tag);

        // 批量刷新
        for (String path : resourcePaths) {
            cdnClient.refreshObjectCaches(
                RefreshObjectCachesRequest.builder()
                    .setObjectType("File")
                    .setObjectPath("https://cdn.yourdomain.com/" + path)
                    .build()
            );
        }
    }
}
```

```java
            .setCondition("> 20")
            .setDuration(5)

            .setMessage("CDN 回源率过高: ${value}%")
```

## 总结

CDN + OSS 的核心是**就近访问 + 缓存分发**：

| 组件 | 作用 |
|---|---|
| CDN 边缘节点 | 离用户最近的访问入口 |
| CDN 区域中心 | 二级缓存，减轻回源压力 |
| OSS 源站 | 文件的持久化存储 |
| 回源 | 缓存未命中时，从 OSS 获取 |

CDN 优化的关键点：

1. **合理设置缓存时间**：静态资源长缓存，动态资源短或不缓存
2. **版本化文件名**：JS/CSS 使用内容 hash，修改后自动失效
3. **防盗链**：防止他人盗用你的 CDN 资源
4. **预热热点资源**：发布后主动推送到 CDN 节点
5. **监控命中率**：命中率是衡量 CDN 效果的核心指标

记住：**CDN 不是银弹**，对于首次访问、随机访问的冷门资源，CDN 效果有限。CDN 适合内容稳定、访问集中的场景。
