# CDN + OSS 分发加速

你的用户在地球的另一端访问你的网站，图片加载要 10 秒。用户骂骂咧咧地关掉了页面，你损失了一个潜在客户。

这是一个关于**距离**的问题：数据在美国，服务在中国，用户在欧洲。再好的服务器也扛不住光速的延迟。

今天我们来看如何用 CDN（内容分发网络）解决这个问题，让用户就近访问，速度飞起。

## 一、为什么需要 CDN

### 1.1 没有 CDN 的问题

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        无 CDN 的访问                                    │
│                                                                         │
│   用户（欧洲） ──▶ 中国服务器 ──▶ 延迟 200ms                             │
│                        │                                                │
│                   图片、视频、JS 等静态资源                              │
│                        │                                                │
│                   所有资源都跨洲传输                                      │
│                        │                                                │
│                   用户等待 10 秒加载                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 有 CDN 的变化

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        有 CDN 的访问                                    │
│                                                                         │
│   用户（欧洲） ──▶ 欧洲 CDN 节点 ──▶ 延迟 10ms                        │
│                              │                                          │
│                         缓存命中？                                       │
│                              │                                          │
│                         是 ──▶ 直接返回（毫秒级）                       │
│                         否 ──▶ 回源（中国服务器）                       │
│                              │                                          │
│                         返回并缓存到欧洲节点                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.3 CDN 的核心价值

| 价值 | 说明 |
|---|---|
| 就近访问 | 用户访问最近的 CDN 节点，减少延迟 |
| 减轻源站压力 | 静态资源由 CDN 承担，减少源站带宽 |
| 可用性保障 | CDN 节点冗余，单点故障不影响 |
| 安全防护 | DDoS 防护、CC 防护等 |
| 成本优化 | 按流量计费，比带宽便宜 |

## 二、CDN 核心概念

### 2.1 工作原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        CDN 工作原理                                     │
│                                                                         │
│   1. 用户请求 www.example.com/image.jpg                                │
│   2. DNS 解析到 CDN 的调度系统                                         │
│   3. 调度系统返回最近的 CDN 节点 IP                                     │
│   4. 用户访问 CDN 节点                                                  │
│   5. CDN 节点检查缓存                                                   │
│      ├─ 命中 ──▶ 直接返回                                              │
│      └─ 未命中 ──▶ 回源获取 ──▶ 缓存 ──▶ 返回                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 核心术语

| 术语 | 说明 |
|---|---|
| 加速域名 | 需要 CDN 加速的域名 |
| 源站 | 真实资源所在的服务器 |
| CDN 节点 | 分布在全球的缓存服务器 |
| 缓存策略 | 哪些资源缓存、缓存多久 |
| 回源 | CDN 缓存未命中时，向源站获取资源 |
| 预热 | 提前将资源推送到 CDN 节点 |

### 3.1 OSS 配置

```java title="OSS 配置类")
@Configuration
public class OssConfig {
    @Value("${aliyun.oss.endpoint}")
    private String endpoint;

    @Value("${aliyun.oss.accessKeyId}")
    private String accessKeyId;

    @Value("${aliyun.oss.accessKeySecret}")
    private String accessKeySecret;

    @Value("${aliyun.oss.bucketName}")
    private String bucketName;

    @Bean
    public OSS ossClient() {
        return OSSClientBuilder.oss()
                .endpoint(endpoint)
                .credentialsProvider(
                        new DefaultCredentialProvider(accessKeyId, accessKeySecret)
                )
                .build();
    }
}
```

### 3.2 上传到 OSS

```java title="OSS 上传服务")
@Service
@Slf4j
public class OssFileService {
    @Autowired
    private OSS ossClient;

    @Value("${aliyun.oss.bucketName}")
    private String bucketName;

    @Value("${aliyun.cdn.domain}")
    private String cdnDomain;

    /**
     * 上传文件到 OSS
     */
    public String uploadFile(String key, InputStream inputStream, long size, String contentType) {
        PutObjectRequest request = new PutObjectRequest(bucketName, key, inputStream);
        request.setMetadata(new ObjectMetadata());
        request.getMetadata().setContentType(contentType);
        request.getMetadata().setContentLength(size);

        ossClient.putObject(request);
        log.info("上传文件到 OSS: {}", key);

        return getCdnUrl(key);
    }

    /**
     * 上传本地文件
     */
    public String uploadFile(String key, File file) throws FileNotFoundException {
        FileInputStream fis = new FileInputStream(file);
        return uploadFile(key, fis, file.length(), getContentType(file.getName()));
    }

    /**
     * 获取 CDN 访问 URL
     */
    public String getCdnUrl(String key) {
        return "https://" + cdnDomain + "/" + key;
    }

    /**
     * 获取 OSS 私有 URL（签名 URL）
     */
    public String getPrivateUrl(String key, Date expire) {
        return ossClient.generatePresignedUrl(bucketName, key, expire).toString();
    }

    /**
     * 生成图片样式 URL
     */
    public String getImageStyleUrl(String key, String style) {
        return getCdnUrl(key) + "?" + style;
    }

    /**
     * 删除文件
     */
    public void deleteFile(String key) {
        ossClient.deleteObject(bucketName, key);
    }
}
```

### 3.3 CDN 配置

```java title="CDN 服务")
@Service
@Slf4j
public class CdnService {
    @Autowired
    private CdnClient cdnClient;

    /**
     * 刷新 CDN 缓存
     */
    public void refreshCache(String url) {
        RefreshObjectCachesRequest request = new RefreshObjectCachesRequest();
        request.setObjectType("File");
        request.setObjectPaths(Collections.singletonList(url));

        RefreshObjectCachesResponse response = cdnClient.refreshObjectCaches(request);
        log.info("刷新 CDN 缓存: {}, taskId: {}", url, response.getRefreshTaskId());
    }

    /**
     * 批量刷新缓存
     */
    public void refreshCacheBatch(List<String> urls) {
        RefreshObjectCachesRequest request = new RefreshObjectCachesRequest();
        request.setObjectType("File");
        request.setObjectPaths(urls);

        cdnClient.refreshObjectCaches(request);
        log.info("批量刷新 CDN 缓存: {}", urls.size());
    }

    /**
     * 预热缓存
     */
    public void preloadCache(String url) {
        PushObjectCacheRequest request = new PushObjectCacheRequest();
        request.setObjectPath(url);

        cdnClient.pushObjectCache(request);
        log.info("预热 CDN 缓存: {}", url);
    }

    /**
     * 查询缓存命中率
     */
    public CdnDomainStatisticResult getStatistics(String domain, Date startDate, Date endDate) {
        DescribeDomainQpsDataRequest request = new DescribeDomainQpsDataRequest();
        request.setDomain(domain);
        request.setStartTime(startDate);
        request.setEndTime(endDate);

        return cdnClient.describeDomainQpsData(request);
    }
}
```

## 四、缓存策略配置

### 4.1 CDN 缓存策略

```java title="缓存策略服务")
@Service
public class CacheStrategyService {
    @Autowired
    private CdnClient cdnClient;

    /**
     * 设置缓存策略
     */
    public void setCacheRules(String domainName) {
        SetCacheRuleConfigRequest request = new SetCacheRuleConfigRequest();
        request.setDomain(domainName);

        // 缓存规则
        List<CacheRule> rules = new ArrayList<>();

        // HTML：不缓存
        rules.add(new CacheRule("*.html", "0"));

        // 图片：缓存 30 天
        rules.add(new CacheRule("*.jpg", "2592000"));
        rules.add(new CacheRule("*.png", "2592000"));
        rules.add(new CacheRule("*.gif", "2592000"));
        rules.add(new CacheRule("*.webp", "2592000"));

        // CSS/JS：缓存 7 天
        rules.add(new CacheRule("*.css", "604800"));
        rules.add(new CacheRule("*.js", "604800"));

        // 音视频：缓存 30 天
        rules.add(new CacheRule("*.mp4", "2592000"));
        rules.add(new CacheRule("*.mp3", "2592000"));

        request.setCacheRules(rules);
        cdnClient.setCacheRuleConfig(request);
    }

    /**
     * 设置 Referer 黑名单/白名单
     */
    public void setRefererRules(String domainName) {
        SetRefererConfigRequest request = new SetRefererConfigRequest();
        request.setDomain(domainName);
        request.setRefererType("whitelist"); // 白名单模式
        request.setReferList("*.example.com,example.com");

        cdnClient.setRefererConfig(request);
    }

    /**
     * 设置 IP 黑名单
     */
    public void setIpBlackList(String domainName, List<String> ips) {
        SetIpBlackListConfigRequest request = new SetIpBlackListConfigRequest();
        request.setDomain(domainName);
        request.setIpList(ips);

        cdnClient.setIpBlackListConfig(request);
    }
}
```

### 4.2 资源版本管理

```java title="版本化资源 URL")
@Service
public class VersionedResourceService {
    @Autowired
    private OssFileService ossFileService;

    /**
     * 生成版本化 URL
     */
    public String getVersionedUrl(String resourcePath) {
        String version = getResourceVersion(resourcePath);
        return ossFileService.getCdnUrl(resourcePath) + "?v=" + version;
    }

    /**
     * 获取资源版本号
     */
    private String getResourceVersion(String resourcePath) {
        // 方案1：使用文件 MD5
        // return calculateFileMd5(resourcePath);

        // 方案2：使用部署时间戳
        // return String.valueOf(System.currentTimeMillis());

        // 方案3：使用 Git commit hash
        return System.getenv("GIT_COMMIT") != null ?
                System.getenv("GIT_COMMIT").substring(0, 7) :
                String.valueOf(System.currentTimeMillis());
    }
}
```

### 5.2 JavaScript 中的 CDN 访问

```javascript title="CDN 资源加载")
class CdnResourceLoader {
    constructor(cdnDomain) {
        this.cdnDomain = cdnDomain;
    }

    /**
     * 加载图片
     */
    loadImage(path, version) {
        return `${this.cdnDomain}/${path}?v=${version}`;
    }

    /**
     * 预加载资源
     */
    preload(urls) {
        urls.forEach(url => {
            const link = document.createElement('link');
            link.rel = 'preload';
            link.as = this.getResourceType(url);
            link.href = url;
            document.head.appendChild(link);
        });
    }

    /**
     * 获取资源类型
     */
    getResourceType(url) {
        if (url.endsWith('.css')) return 'style';
        if (url.endsWith('.js')) return 'script';
        if (url.match(/\.(jpg|png|gif|webp)$/)) return 'image';
        if (url.match(/\.(mp4|webm)$/)) return 'video';
        return 'fetch';
    }
}

// 使用示例
const loader = new CdnResourceLoader('https://cdn.example.com');

// 加载图片
const imageUrl = loader.loadImage('images/avatar.jpg', '1.0.0');
document.querySelector('img.avatar').src = imageUrl;

// 预加载关键资源
loader.preload([
    'https://cdn.example.com/images/banner.jpg',
    'https://cdn.example.com/css/critical.css'
]);
```

## 六、性能优化

### 6.1 图片优化

```java title="图片处理服务")
@Service
@Slf4j
public class ImageOptimizeService {
    @Autowired
    private OssFileService ossFileService;

    /**
     * 生成响应式图片 URL
     */
    public String getResponsiveImageUrl(String originalUrl, int width, int quality) {
        // 使用 CDN 图片处理参数
        return originalUrl + "?x-oss-process=image/resize,w_" + width + "/quality,q_" + quality;
    }

    /**
     * 生成 WebP 格式 URL
     */
    public String getWebpUrl(String originalUrl) {
        return originalUrl + "?x-oss-process=image/format,webp";
    }

    /**
     * 生成缩略图 URL
     */
    public String getThumbnailUrl(String originalUrl, int width, int height) {
        return originalUrl + "?x-oss-process=image/resize,m_fill,w_" + width + ",h_" + height;
    }

    /**
     * 生成渐进式 JPEG
     */
    public String getProgressiveJpegUrl(String originalUrl) {
        return originalUrl + "?x-oss-process=image/interlace,1";
    }
}
```

## 七、HTTPS 配置

### 7.1 配置 HTTPS 证书

```java title="HTTPS 配置")
@Service
public class CdnHttpsService {
    @Autowired
    private CdnClient cdnClient;

    /**
     * 设置 HTTPS
     */
    public void setHttps(String domainName, String certificate, String privateKey) {
        SetCdnDomainCertificateRequest request = new SetCdnDomainCertificateRequest();
        request.setDomainName(domainName);
        request.setCertType("upload");
        request.setSSLProtocol("on");
        request.setCertificate(certificate);
        request.setPrivateKey(privateKey);

        cdnClient.setCdnDomainCertificate(request);
    }

    /**
     * 设置强制 HTTPS
     */
    public void setForceHttps(String domainName) {
        SetHttpHeaderConfigRequest request = new SetHttpHeaderConfigRequest();
        request.setDomainName(domainName);
        request.setHeaderName("Content-Type");
        request.setHeaderValue("application/octet-stream");

        // 设置强制跳转
        cdnClient.setForceRedirectConfig(
                domainName,
                "on",
                "https"
        );
    }
}
```

## 八、监控与告警

### 8.1 性能监控

```java title="CDN 监控服务")
@Service
@Slf4j
public class CdnMonitorService {
    @Autowired
    private CdnClient cdnClient;

    @Autowired
    private AlertService alertService;

    /**
     * 监控 CDN 命中率
     */
    @Scheduled(fixedRate = 300000) // 5 分钟
    public void monitorHitRate() {
        try {
            DescribeDomainHitRateResponse response = cdnClient.describeDomainHitRate();
            double hitRate = response.getQpsHitRate();

            if (hitRate < 0.8) {
                alertService.alert("cdn.hitrate.low",
                        String.format("CDN 命中率低于 80%: %.2f%%", hitRate * 100));
            }

            log.info("CDN 命中率: {:.2f}%", hitRate * 100);
        } catch (Exception e) {
            log.error("CDN 监控失败", e);
        }
    }

    /**
     * 监控回源率
     */
    @Scheduled(fixedRate = 300000)
    public void monitorBackSourceRate() {
        try {
            DescribeDomainUpperPathRateResponse response = cdnClient.describeDomainUpperPathRate();
            double backSourceRate = response.getBackSourceQpsHitRate();

            if (backSourceRate > 0.3) {
                alertService.alert("cdn.backsource.high",
                        String.format("回源率过高: %.2f%%", backSourceRate * 100));
            }
        } catch (Exception e) {
            log.error("回源率监控失败", e);
        }
    }
}
```

## 九、总结对比

| 方案 | 适用场景 | 成本 |
|---|---|---|
| 自建 CDN | 有足够运维能力 | 硬件成本 |
| 阿里云 CDN + OSS | 国内业务 | 按流量计费 |
| CloudFlare | 海外业务 | 免费版够用 |
| 混合云 CDN | 国内外业务 | 组合计费 |

:::tip 选型建议
1. **国内业务**：阿里云 CDN + OSS 是最成熟方案
2. **海外业务**：CloudFlare 免费版就很强
3. **全球业务**：多 CDN 回源，或使用第三方 CDN 服务
:::

## 结尾

CDN 是静态资源加速的不二之选。记住三个要点：

1. **就近访问**：CDN 节点分布全球，用户访问最近节点
2. **缓存策略**：合理配置缓存时间，减少回源
3. **性能优化**：配合图片压缩、懒加载等前端优化
