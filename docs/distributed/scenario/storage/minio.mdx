# MinIO 分布式对象存储

你的用户开始上传图片、视频、文档了。最开始存在服务器本地，后来发现：文件打不开、单点故障、扩容困难……

这就是**文件存储**的经典困境。本地存储不是不能用，而是当业务做大后，它会成为系统的阿喀琉斯之踵。

今天我们来聊一聊 MinIO——一个高性能、云原生的分布式对象存储，如何用它优雅地解决文件存储问题。

## 二、MinIO 核心概念

## 三、MinIO 部署

### 3.1 Docker 单机部署

```bash title="Docker 启动 MinIO"
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e "MINIO_ROOT_USER=minioadmin" \
  -e "MINIO_ROOT_PASSWORD=minioadmin" \
  -v /data/minio/data:/data \
  -v /data/minio/config:/root/.minio \
  minio/minio server /data --console-address ":9001"
```

### 3.2 分布式集群部署

```bash title="分布式部署（4节点）"
# Node 1
minio server http://node1:9000/data http://node2:9000/data \
  http://node3:9000/data http://node4:9000/data \
  --console-address ":9001"

# Node 2, 3, 4 同样配置
```

### 4.2 MinIO 客户端配置

```java title="MinIO 配置类"
@Configuration
public class MinioConfig {
    @Value("${minio.endpoint}")
    private String endpoint;

    @Value("${minio.accessKey}")
    private String accessKey;

    @Value("${minio.secretKey}")
    private String secretKey;

    @Bean
    public MinioClient minioClient() {
        return MinioClient.builder()
                .endpoint(endpoint)
                .credentials(accessKey, secretKey)
                .build();
    }
}
```

### 4.3 文件上传服务

```java title="文件上传服务")
@Service
@Slf4j
public class MinioFileService {
    @Autowired
    private MinioClient minioClient;

    @Value("${minio.bucketName}")
    private String bucketName;

    @PostConstruct
    public void init() {
        try {
            // 初始化 bucket
            boolean exists = minioClient.bucketExists(BucketExistsArgs.builder()
                    .bucket(bucketName)
                    .build());
            if (!exists) {
                minioClient.makeBucket(MakeBucketArgs.builder()
                        .bucket(bucketName)
                        .build());
            }
        } catch (Exception e) {
            log.error("初始化 MinIO Bucket 失败", e);
        }
    }

    /**
     * 上传文件
     */
    public String uploadFile(MultipartFile file) throws Exception {
        // 1. 生成文件名
        String originalFilename = file.getOriginalFilename();
        String extension = originalFilename.substring(originalFilename.lastIndexOf("."));
        String fileName = UUID.randomUUID().toString().replace("-", "") + extension;

        // 2. 上传到 MinIO
        PutObjectArgs args = PutObjectArgs.builder()
                .bucket(bucketName)
                .object(fileName)
                .stream(file.getInputStream(), file.getSize(), -1)
                .contentType(file.getContentType())
                .build();
        minioClient.putObject(args);

        // 3. 返回访问 URL
        return getFileUrl(fileName);
    }

    /**
     * 上传文件（带路径）
     */
    public String uploadFile(String path, MultipartFile file) throws Exception {
        String originalFilename = file.getOriginalFilename();
        String extension = originalFilename.substring(originalFilename.lastIndexOf("."));
        String fileName = path + "/" + UUID.randomUUID().toString().replace("-", "") + extension;

        PutObjectArgs args = PutObjectArgs.builder()
                .bucket(bucketName)
                .object(fileName)
                .stream(file.getInputStream(), file.getSize(), -1)
                .contentType(file.getContentType())
                .build();
        minioClient.putObject(args);

        return getFileUrl(fileName);
    }

    /**
     * 获取文件 URL（签名 URL）
     */
    public String getPresignedUrl(String fileName) throws Exception {
        return minioClient.getPresignedObjectUrl(
                GetPresignedObjectUrlArgs.builder()
                        .method(Method.GET)
                        .bucket(bucketName)
                        .object(fileName)
                        .expiry(3600) // 1 小时
                        .build()
        );
    }

    /**
     * 获取公开文件 URL
     */
    public String getFileUrl(String fileName) {
        return String.format("%s/%s/%s", minioEndpoint, bucketName, fileName);
    }

    /**
     * 下载文件
     */
    public InputStream getFile(String fileName) throws Exception {
        return minioClient.getObject(GetObjectArgs.builder()
                .bucket(bucketName)
                .object(fileName)
                .build());
    }

    /**
     * 删除文件
     */
    public void deleteFile(String fileName) throws Exception {
        minioClient.removeObject(RemoveObjectArgs.builder()
                .bucket(bucketName)
                .object(fileName)
                .build());
    }
}
```

## 五、策略配置

### 5.1 生命周期策略

```java title="生命周期管理")
@Service
public class MinioLifecycleService {
    @Autowired
    private MinioClient minioClient;

    /**
     * 设置文件过期策略
     */
    public void setLifecyclePolicy(String bucketName) throws Exception {
        String lifecycle = """
            {
              "Rules": [
                {
                  "ID": "expire-old-files",
                  "Status": "Enabled",
                  "Filter": {
                    "Prefix": "logs/"
                  },
                  "Expiration": {
                    "Days": 30
                  }
                },
                {
                  "ID": "expire-temp-files",
                  "Status": "Enabled",
                  "Filter": {
                    "Prefix": "temp/"
                  },
                  "Expiration": {
                    "Days": 7
                  }
                }
              ]
            }
            """;

        minioClient.setBucketLifecycle(SetBucketLifecycleArgs.builder()
                .bucket(bucketName)
                .config(new ByteArrayInputStream(lifecycle.getBytes()))
                .build());
    }
}
```

### 5.2 访问策略

```java title="Bucket 访问策略")
@Service
public class MinioPolicyService {
    @Autowired
    private MinioClient minioClient;

    /**
     * 设置公开只读策略
     */
    public void setPublicReadPolicy(String bucketName) throws Exception {
        String policy = """
            {
              "Version": "2012-10-17",
              "Statement": [
                {
                  "Effect": "Allow",
                  "Principal": {"AWS": "*"},
                  "Action": ["s3:GetObject"],
                  "Resource": ["arn:aws:s3:::%s/*"]
                }
              ]
            }
            """.formatted(bucketName);

        minioClient.setBucketPolicy(SetBucketPolicyArgs.builder()
                .bucket(bucketName)
                .config(new ByteArrayInputStream(policy.getBytes()))
                .build());
    }
}
```

## 六、前端直传 MinIO

### 6.1 后端生成预签名上传 URL

```java title="预签名上传")
@RestController
@RequestMapping("/api/upload")
public class UploadController {
    @Autowired
    private MinioClient minioClient;

    @Value("${minio.bucketName}")
    private String bucketName;

    @GetMapping("/presigned-url")
    public Result<String> getPresignedUploadUrl(
            @RequestParam String fileName,
            @RequestParam String contentType) throws Exception {

        String objectName = "uploads/" + UUID.randomUUID().toString() + getExtension(fileName);

        String presignedUrl = minioClient.getPresignedObjectUrl(
                GetPresignedObjectUrlArgs.builder()
                        .method(Method.PUT)
                        .bucket(bucketName)
                        .object(objectName)
                        .expiry(300) // 5 分钟有效期
                        .contentType(contentType)
                        .build()
        );

        return Result.success(presignedUrl);
    }
}
```

### 6.2 前端直传

```javascript title="前端直传")
async function uploadFile(file) {
    // 1. 获取预签名 URL
    const response = await fetch(`/api/upload/presigned-url?fileName=${file.name}&contentType=${file.type}`);
    const { data: presignedUrl } = await response.json();

    // 2. 直接上传到 MinIO
    await fetch(presignedUrl, {
        method: 'PUT',
        body: file,
        headers: {
            'Content-Type': file.type
        }
    });

    // 3. 获取文件访问地址
    const fileUrl = presignedUrl.split('?')[0];
    console.log('上传成功:', fileUrl);
}
```

## 七、高可用配置

### 7.1 纠删码模式

MinIO 使用纠删码技术，即使丢失一半的磁盘，数据依然可以恢复。

```bash
# 分布式部署建议
# 最小 4 节点，每节点 4 盘
# 数据冗余：N/2 节点容错
minio server \
  http://node1:9000/data1 \
  http://node1:9000/data2 \
  http://node2:9000/data1 \
  http://node2:9000/data2 \
  http://node3:9000/data1 \
  http://node3:9000/data2 \
  http://node4:9000/data1 \
  http://node4:9000/data2
```

```java title="健康检查")
@Service
public class MinioHealthService {
    @Autowired
    private MinioClient minioClient;

    public boolean isHealthy() {
        try {
            minioClient.listBuckets();
            return true;
        } catch (Exception e) {
            return false;
        }
    }

    public Map<String, Object> getClusterInfo() throws Exception {
        Map<String, Object> info = new HashMap<>();
        info.put("buckets", minioClient.listBuckets().size());
        info.put("healthy", isHealthy());
        return info;
    }
}
```

## 九、总结对比

| 特性 | MinIO | Ceph | 阿里云 OSS |
|---|---|---|---|
| 部署复杂度 | 低 | 高 | 无需部署 |
| S3 兼容性 | 完全兼容 | 部分兼容 | 原生支持 |
| Kubernetes 集成 | 原生支持 | 一般 | 需要 CSI |
| 适用场景 |私有云/混合云|大规模存储|公有云|
| 成本 | 开源免费 | 硬件成本 | 按量付费 |

:::tip 选型建议
- **私有云部署**：MinIO 是最佳选择，部署简单，功能完整
- **海量存储**：Ceph 或商业存储方案
- **公有云**：直接使用云厂商的对象存储
:::

## 结尾

MinIO 是一款专为云原生时代设计的对象存储，它用简单的方式解决了复杂的文件存储问题。记住三个要点：

1. **分布式是核心**：水平扩展、多副本容错
2. **S3 兼容是优势**：生态丰富，迁移成本低
3. **纠删码是保障**：高可靠低冗余
