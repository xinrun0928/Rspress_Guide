# 文件分片上传与断点续传实现

用户上传一个 2GB 的视频文件，结果网络断了，前功尽弃——没有断点续传的话，用户体验极差。

分片上传就是解决方案：把大文件切成小块，每块独立上传，断点后只重传失败的块，而不是整个文件。

## 分片上传原理

### 核心流程

```
┌──────────────────────────────────────────────────────────────────┐
│                      文件分片上传流程                              │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│   客户端                      服务端                          OSS │
│      │                          │                               │
│      │  1. 初始化上传（获取 uploadId）  │                          │
│      │ ───────────────────────> │                               │
│      │                          │                               │
│      │  2. 切分文件为多个 chunk   │                               │
│      │                          │                               │
│      │  3. 上传 chunk 1         │ 4. 存储 chunk 1                │
│      │ ───────────────────────> │ ───────────────────────────> │ │
│      │      成功，返回 checksum   │                               │
│      │ <─────────────────────── │                               │
│      │                          │                               │
│      │  5. 上传 chunk 2         │ 6. 存储 chunk 2                │
│      │ ───────────────────────> │ ───────────────────────────> │ │
│      │      成功，返回 checksum   │                               │
│      │ <─────────────────────── │                               │
│      │                          │                               │
│      │  ... (继续上传剩余 chunks) │                               │
│      │                          │                               │
│      │  7. 合并 chunks           │ 8. 组装完整文件                 │
│      │ ───────────────────────> │ ───────────────────────────> │ │
│      │                          │                               │
│      │  9. 上传完成，返回 fileId │                               │
│      │ <─────────────────────── │                               │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 分片策略

| 分片大小 | 适用场景 | 优缺点 |
|---|---|---|
| 1MB | 普通文件（< 100MB） | 网络稳定时效率高 |
| 5MB | 大文件（100MB - 1GB） | 平衡重传成本与并发效率 |
| 10MB | 超大文件（> 1GB） | 减少请求数，但重传成本高 |
| 动态计算 | 通用场景 | 根据文件大小自动调整 |

## 服务端实现

### 文件上传服务

```java
@Service
public class FileUploadService {

    private final MinioClient minioClient;
    private final RedisTemplate<String, Object> redisTemplate;
    private static final String UPLOAD_KEY_PREFIX = "upload:";
    private static final int DEFAULT_PART_SIZE = 5 * 1024 * 1024;  // 5MB

    /**
     * 初始化分片上传
     * 返回 uploadId 和预签名 URL
     */
    public InitUploadResponse initUpload(InitUploadRequest request) {
        // 1. 生成唯一 uploadId
        String uploadId = UUID.randomUUID().toString();

        // 2. 计算分片数量
        int partCount = (int) Math.ceil((double) request.getFileSize() / DEFAULT_PART_SIZE);

        // 3. 创建 OSS 分片上传任务
        String objectName = generateObjectName(request.getFileName());
        String bucketName = "uploads";

        try {
            String ossUploadId = minioClient.initiateMultipartUpload(
                InitiateMultipartUploadArgs.builder()
                    .bucket(bucketName)
                    .object(objectName)
                    .build()
            );

            // 4. 存储上传上下文
            UploadContext context = new UploadContext()
                .setUploadId(uploadId)
                .setOssUploadId(ossUploadId)
                .setObjectName(objectName)
                .setBucketName(bucketName)
                .setFileName(request.getFileName())
                .setFileSize(request.getFileSize())
                .setPartCount(partCount)
                .setUploadedParts(new ConcurrentHashMap<>())
                .setCreatedAt(System.currentTimeMillis());

            String contextKey = UPLOAD_KEY_PREFIX + uploadId;
            redisTemplate.opsForValue().set(contextKey, context, Duration.ofHours(24));

            // 5. 生成分片预签名 URL（支持直接上传到 OSS）
            List<PresignedPart> presignedParts = new ArrayList<>();
            for (int i = 0; i < partCount; i++) {
                long start = (long) i * DEFAULT_PART_SIZE;
                long end = Math.min(start + DEFAULT_PART_SIZE, request.getFileSize()) - 1;

                String presignedUrl = minioClient.getPresignedObjectUrl(
                    GetPresignedObjectUrlArgs.builder()
                        .method(Method.PUT)
                        .bucket(bucketName)
                        .object(objectName)
                        .expiry(1, TimeUnit.HOURS)
                        .extraQueryParam("uploadId", ossUploadId)
                        .extraQueryParam("partNumber", String.valueOf(i + 1))
                        .build()
                );

                presignedParts.add(new PresignedPart()
                    .setPartNumber(i + 1)
                    .setUrl(presignedUrl)
                    .setStart(start)
                    .setEnd(end)
                    .setSize(end - start + 1)
                );
            }

            return new InitUploadResponse()
                .setUploadId(uploadId)
                .setObjectName(objectName)
                .setPartCount(partCount)
                .setPartSize(DEFAULT_PART_SIZE)
                .setPresignedParts(presignedParts);

        } catch (Exception e) {
            throw new UploadException("初始化上传失败", e);
        }
    }

    /**
     * 确认分片上传完成
     * 合并所有分片
     */
    public CompleteUploadResponse completeUpload(String uploadId, List<UploadedPart> uploadedParts) {
        String contextKey = UPLOAD_KEY_PREFIX + uploadId;
        UploadContext context = (UploadContext) redisTemplate.opsForValue().get(contextKey);

        if (context == null) {
            throw new UploadException("上传上下文不存在或已过期");
        }

        try {
            // 1. 对上传的分片按 ETag 排序
            uploadedParts.sort(Comparator.comparingInt(UploadedPart::getPartNumber));

            // 2. 完成 OSS 分片上传
            List<Part> parts = uploadedParts.stream()
                .map(p -> Part.from(p.getPartNumber(), p.getEtag()))
                .collect(Collectors.toList());

            ObjectWriteResponse response = minioClient.completeMultipartUpload(
                CompleteMultipartUploadArgs.builder()
                    .bucket(context.getBucketName())
                    .object(context.getObjectName())
                    .uploadId(context.getOssUploadId())
                    .parts(parts)
                    .build()
            );

            // 3. 清理上传上下文
            redisTemplate.delete(contextKey);

            // 4. 返回文件访问 URL
            String fileUrl = String.format("%s/%s/%s",
                minioClient.getObjectUrl(context.getBucketName(), context.getObjectName()));

            return new CompleteUploadResponse()
                .setFileId(uploadId)
                .setFileUrl(fileUrl)
                .setObjectName(context.getObjectName())
                .setFileSize(context.getFileSize());

        } catch (Exception e) {
            throw new UploadException("完成上传失败", e);
        }
    }

    /**
     * 查询已上传的分片（用于断点续传）
     */
    public List<UploadedPart> listUploadedParts(String uploadId) {
        String contextKey = UPLOAD_KEY_PREFIX + uploadId;
        UploadContext context = (UploadContext) redisTemplate.opsForValue().get(contextKey);

        if (context == null) {
            return Collections.emptyList();
        }

        // 返回已成功上传的分片
        return context.getUploadedParts().values().stream()
            .sorted(Comparator.comparingInt(UploadedPart::getPartNumber))
            .collect(Collectors.toList());
    }

    /**
     * 取消上传
     */
    public void abortUpload(String uploadId) {
        String contextKey = UPLOAD_KEY_PREFIX + uploadId;
        UploadContext context = (UploadContext) redisTemplate.opsForValue().get(contextKey);

        if (context != null) {
            try {
                // 取消 OSS 上的分片上传任务
                minioClient.abortMultipartUpload(
                    AbortMultipartUploadArgs.builder()
                        .bucket(context.getBucketName())
                        .object(context.getObjectName())
                        .uploadId(context.getOssUploadId())
                        .build()
                );
            } catch (Exception e) {
                log.warn("取消 OSS 分片上传失败", e);
            }

            // 清理本地上下文
            redisTemplate.delete(contextKey);
        }
    }

    private String generateObjectName(String originalFileName) {
        String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy/MM/dd"));
        String uuid = UUID.randomUUID().toString().replace("-", "");
        String ext = getFileExtension(originalFileName);
        return String.format("uploads/%s/%s%s", timestamp, uuid, ext);
    }

    private String getFileExtension(String fileName) {
        int lastDot = fileName.lastIndexOf('.');
        return lastDot > 0 ? fileName.substring(lastDot) : "";
    }
}
```

```java
public class UploadContext implements Serializable {

    private String bucketName;        // OSS bucket

    private long createdAt;           // 创建时间

    private String etag;              // OSS 返回的 ETag
    private long size;                // 分片大小
    private long uploadedAt;          // 上传时间
}
```

```java
@RestController
@RequestMapping("/api/upload")
public class FileUploadController {

    private final FileUploadService fileUploadService;

    public FileUploadController(FileUploadService fileUploadService) {
        this.fileUploadService = fileUploadService;
    }

    /**
     * 初始化上传
     */
    @PostMapping("/init")
    public ResponseEntity<?> initUpload(@RequestBody InitUploadRequest request) {
        InitUploadResponse response = fileUploadService.initUpload(request);
        return ResponseEntity.ok(response);
    }

    /**
     * 确认上传完成
     */
    @PostMapping("/complete")
    public ResponseEntity<?> completeUpload(
            @RequestParam String uploadId,
            @RequestBody List<UploadedPart> uploadedParts) {
        CompleteUploadResponse response = fileUploadService.completeUpload(uploadId, uploadedParts);
        return ResponseEntity.ok(response);
    }

    /**
     * 查询已上传分片（断点续传）
     */
    @GetMapping("/status/{uploadId}")
    public ResponseEntity<?> getUploadStatus(@PathVariable String uploadId) {
        List<UploadedPart> uploadedParts = fileUploadService.listUploadedParts(uploadId);
        return ResponseEntity.ok(Map.of(
            "uploadId", uploadId,
            "uploadedParts", uploadedParts
        ));
    }

    /**
     * 取消上传
     */
    @DeleteMapping("/{uploadId}")
    public ResponseEntity<?> abortUpload(@PathVariable String uploadId) {
        fileUploadService.abortUpload(uploadId);
        return ResponseEntity.ok("上传已取消");
    }
}
```

## 客户端实现

### JavaScript 分片上传

```javascript
class ChunkUploader {
    constructor(options = {}) {
        this.file = options.file;
        this.chunkSize = options.chunkSize || 5 * 1024 * 1024; // 5MB
        this.concurrency = options.concurrency || 3;  // 并发数
        this.initUrl = options.initUrl;
        this.completeUrl = options.completeUrl;
        this.statusUrl = options.statusUrl;

        this.uploadId = null;
        this.uploadedChunks = new Set();
        this.totalChunks = Math.ceil(this.file.size / this.chunkSize);
    }

    /**
     * 开始上传
     */
    async start() {
        // 1. 检查是否有未完成的上传（断点续传）
        const savedUploadId = localStorage.getItem(this.getUploadIdKey());
        if (savedUploadId) {
            this.uploadId = savedUploadId;
            const status = await this.checkUploadStatus();
            if (status) {
                console.log('断点续传，已上传', status.uploadedParts.length, '个分片');
                this.uploadedChunks = new Set(status.uploadedParts.map(p => p.partNumber));
            }
        } else {
            // 2. 初始化上传
            const initResponse = await fetch(this.initUrl, {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({
                    fileName: this.file.name,
                    fileSize: this.file.size,
                    fileHash: await this.getFileHash()
                })
            });
            const initData = await initResponse.json();
            this.uploadId = initData.uploadId;
            localStorage.setItem(this.getUploadIdKey(), this.uploadId);
        }

        // 3. 上传剩余的分片
        await this.uploadChunks();

        // 4. 完成上传
        await this.completeUpload();
    }

    /**
     * 检查上传状态（断点续传）
     */
    async checkUploadStatus() {
        try {
            const response = await fetch(`${this.statusUrl}/${this.uploadId}`);
            if (response.ok) {
                return await response.json();
            }
        } catch (e) {
            console.warn('检查上传状态失败', e);
        }
        return null;
    }

    /**
     * 上传所有未完成的分片
     */
    async uploadChunks() {
        const pendingChunks = [];
        for (let i = 1; i <= this.totalChunks; i++) {
            if (!this.uploadedChunks.has(i)) {
                pendingChunks.push(i);
            }
        }

        // 使用 Promise 并发控制
        const results = await this.uploadWithConcurrency(pendingChunks, this.concurrency);

        // 检查失败的分片并重试
        const failedChunks = results.filter(r => !r.success);
        if (failedChunks.length > 0) {
            console.warn('有', failedChunks.length, '个分片上传失败，开始重试');
            await this.retryFailedChunks(failedChunks.map(f => f.partNumber));
        }
    }

    /**
     * 并发上传
     */
    async uploadWithConcurrency(chunks, concurrency) {
        const results = [];
        let index = 0;

        const workers = Array(concurrency).fill(null).map(async () => {
            while (index < chunks.length) {
                const currentIndex = index++;
                const partNumber = chunks[currentIndex];
                try {
                    const result = await this.uploadChunk(partNumber);
                    results.push({ partNumber, success: true, result });
                } catch (e) {
                    results.push({ partNumber, success: false, error: e });
                }
            }
        });

        await Promise.all(workers);
        return results;
    }

    /**
     * 上传单个分片
     */
    async uploadChunk(partNumber) {
        const start = (partNumber - 1) * this.chunkSize;
        const end = Math.min(start + this.chunkSize, this.file.size);
        const chunk = this.file.slice(start, end);

        // 获取预签名 URL
        const presignedParts = await this.getPresignedUrl(partNumber);
        const presignedUrl = presignedParts.url;

        // 使用预签名 URL 直接上传到 OSS
        const response = await fetch(presignedUrl, {
            method: 'PUT',
            body: chunk,
            headers: {
                'Content-Type': 'application/octet-stream'
            }
        });

        if (!response.ok) {
            throw new Error(`分片 ${partNumber} 上传失败`);
        }

        // 获取 ETag
        const etag = response.headers.get('ETag');
        this.uploadedChunks.add(partNumber);

        // 记录进度
        this.onProgress({
            uploaded: this.uploadedChunks.size,
            total: this.totalChunks,
            percent: Math.round(this.uploadedChunks.size / this.totalChunks * 100)
        });

        return { partNumber, etag };
    }

    /**
     * 获取预签名 URL
     */
    async getPresignedUrl(partNumber) {
        const response = await fetch(`${this.initUrl}/${this.uploadId}/part/${partNumber}`);
        return response.json();
    }

    /**
     * 完成上传
     */
    async completeUpload() {
        const response = await fetch(this.completeUrl, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
                uploadId: this.uploadId,
                parts: Array.from(this.uploadedChunks).map(partNumber => ({
                    partNumber,
                    etag: this.getEtag(partNumber)
                }))
            })
        });

        const result = await response.json();

        // 清理本地存储
        localStorage.removeItem(this.getUploadIdKey());

        return result;
    }

    /**
     * 重试失败的分片
     */
    async retryFailedChunks(failedChunks, maxRetries = 3) {
        for (let retry = 0; retry < maxRetries; retry++) {
            const stillFailing = [];

            for (const partNumber of failedChunks) {
                try {
                    await this.uploadChunk(partNumber);
                } catch (e) {
                    stillFailing.push(partNumber);
                }
            }

            if (stillFailing.length === 0) {
                break;
            }

            // 指数退避
            await new Promise(resolve => setTimeout(resolve, Math.pow(2, retry) * 1000));
            failedChunks = stillFailing;
        }

        if (failedChunks.length > 0) {
            throw new Error(`有 ${failedChunks.length} 个分片上传失败`);
        }
    }

    /**
     * 获取文件的唯一标识（用于断点续传）
     */
    getUploadIdKey() {
        return `upload_${this.file.name}_${this.file.size}_${this.file.lastModified}`;
    }

    /**
     * 计算文件 Hash（秒传验证）
     */
    async getFileHash() {
        const chunkSize = 2 * 1024 * 1024;  // 2MB
        const chunks = Math.ceil(this.file.size / chunkSize);
        const spark = new SparkMD5();

        for (let i = 0; i < chunks; i++) {
            const chunk = this.file.slice(i * chunkSize, (i + 1) * chunkSize);
            const buffer = await chunk.arrayBuffer();
            spark.append(buffer);
        }

        return spark.end();
    }

    /**
     * 进度回调
     */
    onProgress(info) {
        if (this.options.onProgress) {
            this.options.onProgress(info);
        }
    }
}
```

## 秒传功能

### 秒传原理

用户上传前先计算文件 MD5，服务端检查是否已存在相同文件：

```java
/**
 * 秒传检查
 */
public FileCheckResponse checkFileExists(String fileHash, String fileName, long fileSize) {
    // 1. 查询文件 Hash 表
    FileRecord record = fileRepository.findByHash(fileHash);

    if (record != null && record.getFileSize() == fileSize) {
        // 2. 文件已存在，直接返回
        return new FileCheckResponse()
            .setExists(true)
            .setFileId(record.getFileId())
            .setFileUrl(record.getFileUrl());
    }

    // 3. 返回不存在，需要上传
    return new FileCheckResponse()
        .setExists(false)
        .setUploadId(null);
}
```

## 常见问题与解决

### 问题一：并发上传导致服务器压力

```java
// 使用信号量限制并发
@Service
public class ThrottledUploadService {

    private final Semaphore semaphore = new Semaphore(50);  // 最多 50 个并发

    public PresignedUrlResponse getUploadUrl(String uploadId, int partNumber) {
        try {
            semaphore.acquire();
            return generatePresignedUrl(uploadId, partNumber);
        } catch (InterruptedException e) {
            throw new UploadException("服务器繁忙，请稍后重试");
        } finally {
            semaphore.release();
        }
    }
}
```

```javascript
const axiosInstance = axios.create({
    timeout: 60000,  // 60 秒超时

    retryDelay: 1000

// 请求拦截器处理重试
axiosInstance.interceptors.response.use(
    response => response,
    error => {
        const config = error.config;
        if (!config || !config.retry) return Promise.reject(error);

        config.retry -= 1;
        if (config.retry <= 0) return Promise.reject(error);

        return new Promise(resolve =>
            setTimeout(() => resolve(axiosInstance(config)), config.retryDelay)
        );
    }
);
```

```java
// 完整性校验
public void validateUploadComplete(String uploadId, List<UploadedPart> uploadedParts) {
    UploadContext context = getUploadContext(uploadId);

    // 1. 检查分片数量
    if (uploadedParts.size() != context.getPartCount()) {
        throw new UploadException("分片数量不匹配");
    }

    // 2. 检查分片编号连续性
    Set<Integer> partNumbers = uploadedParts.stream()
        .map(UploadedPart::getPartNumber)
        .collect(Collectors.toSet());

    for (int i = 1; i <= context.getPartCount(); i++) {
        if (!partNumbers.contains(i)) {
            throw new UploadException("缺少分片: " + i);
        }
    }
}
```

## 总结

分片上传的核心是**化整为零 + 断点续传**：

| 组件 | 职责 |
|---|---|
| 初始化 | 生成 uploadId，计算分片数 |
| 分片上传 | 并发上传各分片，记录进度 |
| 续传检查 | 查询已上传分片，跳过已完成的 |
| 合并完成 | 合并所有分片，清理临时数据 |

实现要点：

1. **预签名 URL**：服务端生成签名，客户端直传 OSS，避免流量经过应用服务器
2. **进度持久化**：将已上传分片信息存储到 Redis，支持重启后续传
