# 分布式唯一任务执行：乐观锁 + 分布式锁双保险

两个定时任务同时触发，都要处理同一批数据。如果不加控制，它们会重复执行——你可能给用户发了两次短信，扣了两次钱。

分布式唯一任务执行，就是为了解决这个问题。

## 为什么需要唯一执行？

### 重复执行的问题

### 解决方案对比

| 方案 | 原理 | 优点 | 缺点 |
|---|---|---|---|
| 定时调度器单点 | 只有一个节点执行 | 简单 | 可用性低 |
| 分布式锁 | 抢锁执行 | 简单可靠 | 锁竞争 |
| 乐观锁 | 版本号控制 | 无锁竞争 | 可能失败重试 |
| 任务表 | 数据库记录 | 可靠 | 增加数据库负担 |

## 分布式锁实现

### Redis 分布式锁

```java
/**
 * Redis 分布式锁
 */
@Service
public class RedisDistributedLock {

    private final RedisTemplate<String, Object> redisTemplate;
    private static final String LOCK_PREFIX = "lock:";

    /**
     * 尝试获取锁
     */
    public boolean tryLock(String key, long expireTime, TimeUnit unit) {
        String lockKey = LOCK_PREFIX + key;
        String lockValue = generateLockValue();

        // SET key value NX PX milliseconds
        Boolean result = redisTemplate.opsForValue().setIfAbsent(
            lockKey,
            lockValue,
            Duration.ofMillis(unit.toMillis(expireTime))
        );

        return Boolean.TRUE.equals(result);
    }

    /**
     * 释放锁
     */
    public boolean unlock(String key, String lockValue) {
        String lockKey = LOCK_PREFIX + key;

        // Lua 脚本：只有锁持有者才能释放
        String script =
            "if redis.call('get', KEYS[1]) == ARGV[1] then " +
            "  return redis.call('del', KEYS[1]) " +
            "else " +
            "  return 0 " +
            "end";

        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(script, Long.class),
            Collections.singletonList(lockKey),
            lockValue
        );

        return result != null && result > 0;
    }

    /**
     * 可重入锁（同一线程可多次获取）
     */
    public boolean tryReentrantLock(String key, long expireTime, TimeUnit unit) {
        String lockKey = LOCK_PREFIX + key;
        String lockValue = generateLockValue();
        String threadId = getCurrentThreadId();

        // 检查是否是同一线程持有
        Object currentValue = redisTemplate.opsForValue().get(lockKey);
        if (currentValue != null) {
            LockInfo lockInfo = parseLockValue(currentValue.toString());
            if (lockInfo.getThreadId().equals(threadId)) {
                // 同一线程，增加重入计数
                lockInfo.setCount(lockInfo.getCount() + 1);
                redisTemplate.opsForValue().set(lockKey,
                    serializeLockInfo(lockInfo),
                    Duration.ofMillis(unit.toMillis(expireTime)));
                return true;
            }
        }

        // 尝试获取锁
        Boolean result = redisTemplate.opsForValue().setIfAbsent(
            lockKey,
            serializeLockInfo(new LockInfo(lockValue, threadId, 1)),
            Duration.ofMillis(unit.toMillis(expireTime))
        );

        return Boolean.TRUE.equals(result);
    }

    /**
     * 生成锁值（包含时间戳和随机数）
     */
    private String generateLockValue() {
        return System.currentTimeMillis() + ":" + UUID.randomUUID().toString();
    }

    private String getCurrentThreadId() {
        return Thread.currentThread().getName() + "-" + Thread.currentThread().getId();
    }
}
```

### 分布式锁注解

```java
/**
 * 分布式锁注解
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface DistributedLock {

    /**
     * 锁的 key
     */
    String key();

    /**
     * 锁过期时间
     */
    long expireTime() default 30;

    /**
     * 时间单位
     */
    TimeUnit timeUnit() default TimeUnit.SECONDS;

    /**
     * 获取锁等待时间
     */
    long waitTime() default 0;

    /**
     * 是否可重入
     */
    boolean reentrant() default false;
}

/**
 * 分布式锁切面
 */
@Aspect
@Component
public class DistributedLockAspect {

    private final RedisDistributedLock lockService;

    @Around("@annotation(distributedLock)")
    public Object around(ProceedingJoinPoint joinPoint,
                        DistributedLock distributedLock) throws Throwable {

        String key = resolveKey(distributedLock.key(), joinPoint);
        String lockValue = UUID.randomUUID().toString();

        boolean acquired = false;

        try {
            // 1. 尝试获取锁
            acquired = lockService.tryLock(
                key,
                distributedLock.expireTime(),
                distributedLock.timeUnit()
            );

            if (!acquired) {
                // 2. 获取失败，根据配置决定行为
                if (distributedLock.waitTime() > 0) {
                    // 等待一段时间
                    Thread.sleep(distributedLock.waitTime());
                    acquired = lockService.tryLock(key,
                        distributedLock.expireTime(),
                        distributedLock.timeUnit());
                }

                if (!acquired) {
                    throw new LockAcquisitionException("获取锁失败: " + key);
                }
            }

            // 3. 执行目标方法
            return joinPoint.proceed();

        } finally {
            // 4. 释放锁
            if (acquired) {
                lockService.unlock(key, lockValue);
            }
        }
    }

    private String resolveKey(String keyTemplate, ProceedingJoinPoint joinPoint) {
        // 支持 SpEL 表达式
        if (keyTemplate.contains("#")) {
            EvaluationContext context = new StandardEvaluationContext();
            String[] parameterNames = getParameterNames(joinPoint);
            Object[] args = joinPoint.getArgs();

            for (int i = 0; i < parameterNames.length; i++) {
                ((StandardEvaluationContext) context).setVariable(parameterNames[i], args[i]);
            }

            return new SpelExpressionParser().parseExpression(keyTemplate)
                .getValue(context, String.class);
        }
        return keyTemplate;
    }
}
```

### 乐观锁服务

```java
/**
 * 乐观锁任务服务
 */
@Service
public class OptimisticLockTaskService {

    @Autowired
    private JdbcTemplate jdbcTemplate;

    /**
     * 尝试获取任务执行权
     * 使用乐观锁，只有版本号匹配才能更新成功
     */
    public TaskClaimResult claimTask(String taskKey, long maxExecuteTime) {
        // 1. 获取当前时间
        long now = System.currentTimeMillis();

        // 2. 尝试获取任务（乐观锁）
        String sql =
            "UPDATE scheduled_task " +
            "SET status = 'RUNNING', " +
            "    last_execute_time = ?, " +
            "    max_execute_time = ?, " +
            "    version = version + 1 " +
            "WHERE task_key = ? " +
            "  AND status IN ('IDLE', 'COMPLETED', 'FAILED') " +
            "  AND (next_execute_time IS NULL OR next_execute_time <= ?) " +
            "  AND version = ?";

        // 先查询当前版本
        Task currentTask = findByTaskKey(taskKey);
        if (currentTask == null) {
            return TaskClaimResult.FAILED_TASK_NOT_FOUND;
        }

        // 尝试更新
        int updated = jdbcTemplate.update(sql,
            new Date(now),
            new Date(maxExecuteTime),
            taskKey,
            new Date(now),
            currentTask.getVersion()
        );

        if (updated == 1) {
            // 获取成功
            Task claimedTask = findByTaskKey(taskKey);
            return TaskClaimResult.success(claimedTask);
        } else {
            // 获取失败（被其他节点抢走）
            return TaskClaimResult.FAILED_LOCK_CONFLICT;
        }
    }

    /**
     * 完成任务
     */
    public void completeTask(String taskKey, String result) {
        String sql =
            "UPDATE scheduled_task " +
            "SET status = 'COMPLETED', " +
            "    result = ?, " +
            "    version = version + 1 " +
            "WHERE task_key = ? AND status = 'RUNNING'";

        jdbcTemplate.update(sql, result, taskKey);
    }

    /**
     * 标记任务失败
     */
    public void failTask(String taskKey, String errorMessage) {
        String sql =
            "UPDATE scheduled_task " +
            "SET status = 'FAILED', " +
            "    result = ?, " +
            "    version = version + 1 " +
            "WHERE task_key = ? AND status = 'RUNNING'";

        jdbcTemplate.update(sql, errorMessage, taskKey);
    }

    /**
     * 释放任务（用于超时处理）
     */
    public void releaseTask(String taskKey) {
        String sql =
            "UPDATE scheduled_task " +
            "SET status = 'FAILED', " +
            "    result = '执行超时', " +
            "    version = version + 1 " +
            "WHERE task_key = ? " +
            "  AND status = 'RUNNING' " +
            "  AND max_execute_time < ?";

        jdbcTemplate.update(sql, taskKey, new Date());
    }
}
```

## 双保险实现

### 乐观锁 + 分布式锁

```java
/**
 * 分布式定时任务执行器
 * 结合乐观锁和分布式锁，保证任务唯一执行
 */
@Service
public class DistributedTaskExecutor {

    private final OptimisticLockTaskService optimisticLockService;
    private final RedisDistributedLock distributedLock;
    private final TaskExecutor taskExecutor;

    private static final String TASK_LOCK_PREFIX = "task:lock:";

    /**
     * 执行定时任务（双保险）
     */
    public void executeTask(String taskKey) {
        String distributedLockKey = TASK_LOCK_PREFIX + taskKey;
        String lockValue = UUID.randomUUID().toString();

        boolean lockAcquired = false;

        try {
            // 第一层：分布式锁（快速失败）
            lockAcquired = distributedLock.tryLock(distributedLockKey, lockValue, 30, TimeUnit.SECONDS);

            if (!lockAcquired) {
                log.info("未能获取分布式锁，跳过执行: {}", taskKey);
                return;
            }

            // 第二层：乐观锁（保证唯一）
            TaskClaimResult claimResult = optimisticLockService.claimTask(
                taskKey,
                System.currentTimeMillis() + 5 * 60 * 1000  // 5 分钟后超时
            );

            if (!claimResult.isSuccess()) {
                log.info("乐观锁获取失败: taskKey={}, reason={}",
                    taskKey, claimResult.getFailReason());
                return;
            }

            // 执行任务
            executeTaskInternal(taskKey, claimResult.getTask());

        } catch (Exception e) {
            log.error("执行任务异常: taskKey={}", taskKey, e);
            optimisticLockService.failTask(taskKey, e.getMessage());

        } finally {
            // 释放分布式锁
            if (lockAcquired) {
                distributedLock.unlock(distributedLockKey, lockValue);
            }
        }
    }

    private void executeTaskInternal(String taskKey, Task task) {
        try {
            // 执行具体的业务逻辑
            Object result = taskExecutor.execute(task);

            // 任务完成
            optimisticLockService.completeTask(taskKey, toJson(result));

            log.info("任务执行成功: taskKey={}", taskKey);

        } catch (Exception e) {
            log.error("任务执行失败: taskKey={}", taskKey, e);
            optimisticLockService.failTask(taskKey, e.getMessage());
        }
    }
}
```

```java
/**
 * 分布式定时任务调度器
 */
@Configuration
public class DistributedScheduler {

    @Autowired
    private TaskRepository taskRepository;

    @Autowired
    private DistributedTaskExecutor taskExecutor;

    /**
     * 分布式定时任务（不再使用 @Scheduled）
     */
    @Scheduled(fixedDelay = 60000)  // 1 分钟调度一次
    public void scheduleTasks() {
        // 1. 查询待执行的任务
        List<Task> tasks = taskRepository.findRunnableTasks(
            new Date(),
            10  // 每次最多调度 10 个
        );

        for (Task task : tasks) {
            // 2. 异步执行任务
            CompletableFuture.runAsync(() ->
                taskExecutor.executeTask(task.getTaskKey())
            );
        }
    }

    /**
     * 处理超时任务
     */
    @Scheduled(fixedDelay = 60000)  // 1 分钟检查一次
    public void handleTimeoutTasks() {
        List<Task> timeoutTasks = taskRepository.findTimeoutTasks(new Date());

        for (Task task : timeoutTasks) {
            log.warn("发现超时任务: taskKey={}, maxExecuteTime={}",
                task.getTaskKey(), task.getMaxExecuteTime());

            // 释放任务（乐观锁）
            optimisticLockService.releaseTask(task.getTaskKey());
        }
    }
}
```

### 顺序执行任务

```java
/**
 * 顺序执行任务
 * 保证同一业务的任务按顺序执行
 */
@Service
public class SequentialTaskExecutor {

    private final RedisTemplate<String, Object> redisTemplate;
    private static final String SEQUENTIAL_LOCK_PREFIX = "sequential:lock:";

    /**
     * 按顺序执行任务
     */
    public <T> void executeSequentially(String businessKey, Supplier<T> task) {
        String lockKey = SEQUENTIAL_LOCK_PREFIX + businessKey;

        // 使用 Redis 队列保证顺序
        redisTemplate.opsForList().rightPush("sequential:queue:" + businessKey, businessKey);

        // 获取执行权
        while (true) {
            // 检查是否轮到当前任务
            Object front = redisTemplate.opsForList().get("sequential:queue:" + businessKey, 0);
            if (businessKey.equals(front)) {
                // 是第一个，执行任务
                try {
                    task.get();
                } finally {
                    // 完成后移除
                    redisTemplate.opsForList().leftPop("sequential:queue:" + businessKey);
                }
                break;
            }

            // 等待
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }
}
```

## 监控与告警

### 任务监控

```java
/**
 * 任务执行监控
 */
@Service
public class TaskExecutionMonitor {

    @Autowired
    private TaskRepository taskRepository;

    @Scheduled(fixedRate = 60000)
    public void monitorTaskExecution() {
        // 1. 检查正在运行的任务
        List<Task> runningTasks = taskRepository.findByStatus("RUNNING");

        for (Task task : runningTasks) {
            // 检查是否超时
            if (isTaskTimeout(task)) {
                handleTaskTimeout(task);
            }
        }

        // 2. 统计任务执行情况
        reportTaskStatistics();
    }

    private boolean isTaskTimeout(Task task) {
        if (task.getMaxExecuteTime() == null) {
            return false;
        }
        return System.currentTimeMillis() > task.getMaxExecuteTime().getTime();
    }

    private void handleTaskTimeout(Task task) {
        log.warn("任务执行超时: taskKey={}, maxExecuteTime={}",
            task.getTaskKey(), task.getMaxExecuteTime());

        // 释放任务
        optimisticLockService.releaseTask(task.getTaskKey());

        // 发送告警
        alertService.sendAlert("任务执行超时",
            AlertLevel.WARNING,
            Map.of("taskKey", task.getTaskKey()));
    }

    private void reportTaskStatistics() {
        // 上报任务执行指标到监控系统
        Map<String, Object> stats = taskRepository.getTaskStatistics();
        metricsService.record("task.running", stats.get("running"));
        metricsService.record("task.failed", stats.get("failed"));
        metricsService.record("task.completed", stats.get("completed"));
    }
}
```

## 常见问题与解决

### 问题一：锁释放失败

```java
// 使用看门狗自动续期
@Service
public class LockRenewalService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    private final ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);

    public String acquireLockWithWatchdog(String key, long expireTime) {
        String lockValue = UUID.randomUUID().toString();
        String lockKey = "lock:" + key;

        // 尝试获取锁
        Boolean acquired = redisTemplate.opsForValue().setIfAbsent(
            lockKey, lockValue, Duration.ofSeconds(expireTime));

        if (!Boolean.TRUE.equals(acquired)) {
            return null;
        }

        // 启动看门狗，自动续期
        scheduleRenewal(lockKey, expireTime);

        return lockValue;
    }

    private void scheduleRenewal(String lockKey, long expireTime) {
        scheduler.scheduleAtFixedRate(() -> {
            // 检查锁是否还存在
            if (Boolean.TRUE.equals(redisTemplate.hasKey(lockKey))) {
                // 续期
                redisTemplate.expire(lockKey, Duration.ofSeconds(expireTime));
            }
        }, expireTime / 3, expireTime / 3, TimeUnit.SECONDS);
    }
}
```

## 总结

分布式唯一任务执行的核心是**双重保障**：

| 机制 | 作用 | 特点 |
|---|---|---|
| 分布式锁 | 第一道防线，快速失败 | 性能好，可能死锁 |
| 乐观锁 | 第二道防线，唯一保证 | 可靠，可能重试 |

最佳实践：

1. **优先使用分布式锁**：性能好，大部分场景够用
2. **关键任务加乐观锁**：对数据一致性要求高的任务
3. **设置任务超时**：防止任务卡死导致锁无法释放
4. **做好监控告警**：及时发现和处理异常

记住：**没有银弹**。不同的场景需要不同的方案，要根据实际需求选择。
