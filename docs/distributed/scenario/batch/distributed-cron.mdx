# 定时任务的分布式化：广播模式与分片模式

凌晨 3 点，你的数据同步定时任务开始执行。5 分钟后，运营同事打电话来：数据怎么同步了两份？

单机定时任务直接部署多实例时，每个实例都会执行相同的任务。这就是分布式定时任务的问题：**如何让多个节点协调工作？**

## 分布式定时任务的问题

### 多实例执行问题

```
┌──────────────────────────────────────────────────────────────────┐
│                    多实例执行问题                                 │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│  配置: 每 5 分钟执行一次数据同步                                 │
│                                                                   │
│  单实例:                                                         │
│  ┌─────────────┐                                                │
│  │ 实例 A     │ ──> 每 5 分钟执行一次                           │
│  └─────────────┘                                                │
│                                                                   │
│  多实例（未做处理）:                                            │
│  ┌─────────────┐  ┌─────────────┐                               │
│  │ 实例 A     │  │ 实例 B     │                               │
│  │ 每 5 分钟  │  │ 每 5 分钟  │  ← 重复执行！                   │
│  └─────────────┘  └─────────────┘                               │
│                                                                   │
│  后果: 数据同步重复，可能导致数据错误                             │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 解决方案对比

| 模式 | 原理 | 适用场景 | 代表框架 |
|---|---|---|---|
| 抢占模式 | 多个节点抢锁，只有一个执行 | 任务不可拆分 | XXL-Job |
| 分片模式 | 任务拆分成多个分片，分配给不同节点 | 任务可并行 | XXL-Job、ElasticJob |
| 广播模式 | 所有节点都执行 | 需要所有节点同步 | Quartz 集群 |
| 工作流模式 | 按 DAG 顺序执行 | 任务有依赖关系 | DolphinScheduler |

### Redis Pub/Sub 实现广播

```java
/**
 * 基于 Redis Pub/Sub 的广播调度器
 */
@Service
public class RedisBroadcastScheduler {

    private final RedisTemplate<String, Object> redisTemplate;
    private final RedisMessageListenerContainer listenerContainer;

    /**
     * 发布广播消息
     */
    public void broadcast(String jobName, Object payload) {
        String channel = "broadcast:job:" + jobName;

        BroadcastMessage message = new BroadcastMessage()
            .setJobName(jobName)
            .setPayload(payload)
            .setTimestamp(System.currentTimeMillis())
            .setSenderId(getInstanceId());

        redisTemplate.convertAndSend(channel, message);
    }

    /**
     * 订阅广播消息
     */
    @PostConstruct
    public void subscribeBroadcast() {
        listenerContainer.addMessageListener(
            (message, pattern) -> {
                BroadcastMessage broadcast = (BroadcastMessage)
                    redisTemplate.getValueSerializer().deserialize(message.getBody());

                // 忽略自己发送的消息
                if (broadcast.getSenderId().equals(getInstanceId())) {
                    return;
                }

                // 执行任务
                executeJob(broadcast.getJobName(), broadcast.getPayload());

            },
            new PatternTopic("broadcast:job:*")
        );
    }

    private String getInstanceId() {
        return InetAddress.getLocalHost().getHostName() + "-" + getPid();
    }
}
```

### Zookeeper 实现广播

```java
/**
 * 基于 Zookeeper 的分布式锁（抢占模式）
 */
@Service
public class ZookeeperDistributedLock {

    private final CuratorFramework zkClient;
    private static final String LOCK_PATH = "/distributed/locks";

    /**
     * 尝试获取锁
     */
    public boolean tryLock(String lockName, long timeout, TimeUnit unit) {
        String lockPath = LOCK_PATH + "/" + lockName;

        try {
            // 创建一个临时顺序节点
            String nodePath = zkClient.create()
                .withMode(CreateMode.EPHEMERAL_SEQUENTIAL)
                .forPath(lockPath + "/");

            // 获取所有子节点
            List<String> children = zkClient.getChildren().forPath(LOCK_PATH + "/" + lockName);
            Collections.sort(children);

            // 检查是否是第一个
            String firstNode = lockPath + "/" + children.get(0);
            return firstNode.equals(nodePath);

        } catch (Exception e) {
            log.error("获取分布式锁失败", e);
            return false;
        }
    }

    /**
     * 监听节点变化
     */
    public void watchAndExecute(String lockName, Runnable task) {
        String lockPath = LOCK_PATH + "/" + lockName;

        try {
            // 获取锁
            if (tryLock(lockName, 30, TimeUnit.SECONDS)) {
                try {
                    task.run();
                } finally {
                    unlock(lockName);
                }
            }

        } catch (Exception e) {
            log.error("执行任务失败", e);
        }
    }
}
```

### 分片任务实现

```java
/**
 * 分片任务抽象
 */
public abstract class ShardingJob {

    /**
     * 获取分片总数
     */
    public abstract int getShardingTotalCount();

    /**
     * 执行分片任务
     */
    public abstract void execute(ShardingContext context);

    /**
     * 获取任务名称
     */
    public abstract String getJobName();
}

/**
 * 分片上下文
 */
@Data
public class ShardingContext {
    private int shardingItem;      // 当前分片编号
    private int shardingTotalCount; // 分片总数
    private String jobParameter;    // 任务参数
    private DateTime shardingParameter; // 分片时间参数
}

/**
 * 数据同步分片任务
 */
public class DataSyncShardingJob extends ShardingJob {

    @Override
    public int getShardingTotalCount() {
        return 10;  // 10 个分片
    }

    @Override
    public void execute(ShardingContext context) {
        int shardingItem = context.getShardingItem();
        int total = context.getShardingTotalCount();

        log.info("开始执行数据同步分片任务: sharding={}/{}", shardingItem, total);

        // 计算数据范围
        long startId = shardingItem * 10000L;
        long endId = (shardingItem + 1) * 10000L - 1;

        // 分批处理
        processBatch(startId, endId, 1000);

        log.info("数据同步分片任务完成: sharding={}", shardingItem);
    }

    private void processBatch(long startId, long endId, int batchSize) {
        long currentStart = startId;

        while (currentStart <= endId) {
            long currentEnd = Math.min(currentStart + batchSize - 1, endId);

            // 处理数据
            List<Entity> entities = queryData(currentStart, currentEnd);
            syncData(entities);

            currentStart = currentEnd + 1;
        }
    }
}
```

### XXL-Job 分片任务

```java
/**
 * XXL-Job 分片任务
 */
@Component
public class ShardingJobHandler {

    /**
     * 分片广播任务
     * 所有节点都会执行，但根据分片参数执行不同逻辑
     */
    @XxlJob("dataSyncShardingJob")
    public ReturnT<String> execute() {
        // 1. 获取分片参数
        ShardingEnv shardingEnv = XxlJobHelper.getShardingEnv();
        int shardingIndex = shardingEnv.getIndex();  // 当前分片索引
        int shardingTotal = shardingEnv.getTotal();  // 总分片数

        log.info("分片任务开始: index={}, total={}", shardingIndex, shardingTotal);

        try {
            // 2. 执行分片任务
            List<Long> dataIds = calculateShardingIds(shardingIndex, shardingTotal);

            log.info("分片任务数据量: index={}, count={}", shardingIndex, dataIds.size());

            // 3. 逐条处理
            int successCount = 0;
            int failCount = 0;

            for (Long id : dataIds) {
                try {
                    processData(id);
                    successCount++;
                } catch (Exception e) {
                    log.error("处理数据失败: id={}", id, e);
                    failCount++;
                }
            }

            // 4. 返回结果
            String result = String.format("成功: %d, 失败: %d", successCount, failCount);
            log.info("分片任务完成: {}", result);

            return failCount > 0
                ? new ReturnT<>(ReturnT.SUCCESS_CODE, result)
                : ReturnT.SUCCESS;

        } catch (Exception e) {
            log.error("分片任务执行失败", e);
            return new ReturnT<>(ReturnT.FAIL_CODE, e.getMessage());
        }
    }

    /**
     * 计算分片数据范围

        // 从数据库查询所有待处理的 ID

                result.add(allIds.get(i));
            }
        }

        return result;
    }
```

```java
/**
 * 单机调度策略（抢占到才执行）
 */
@Service
public class SingleNodeScheduler {

    private final RedisDistributedLock distributedLock;
    private static final String JOB_LOCK_PREFIX = "job:lock:";

    /**
     * 分布式定时任务
     */
    @Scheduled(cron = "0 0 2 * * ?")
    public void executeDistributedJob() {
        String lockKey = JOB_LOCK_PREFIX + "dataSyncJob";

        // 尝试获取分布式锁
        if (!distributedLock.tryLock(lockKey, 30, TimeUnit.SECONDS)) {
            log.info("未获取到执行锁，跳过本次执行");
            return;
        }

        try {
            // 执行任务
            doExecute();

        } finally {
            distributedLock.unlock(lockKey);
        }
    }
}
```

### 分片调度

```java
/**
 * 分片调度策略
 */
@Service
public class ShardingScheduler {

    @Autowired
    private RegistryCenter registryCenter;

    /**
     * 分片定时任务
     */
    @Scheduled(fixedDelay = 60000)
    public void executeShardingJob() {
        // 1. 获取所有注册的分片节点
        List<ServerNode> servers = registryCenter.getServers();

        // 2. 计算当前节点负责的分片
        String currentServer = getCurrentServerId();
        List<Integer> shardingItems = calculateShardingItems(
            currentServer,
            servers,
            getTotalShardingCount()
        );

        // 3. 执行分片任务
        for (Integer shardingItem : shardingItems) {
            executeShardingTask(shardingItem);
        }
    }

    private List<Integer> calculateShardingItems(
            String server,
            List<ServerNode> servers,
            int totalSharding) {

        int serverCount = servers.size();
        int shardingPerServer = totalSharding / serverCount;
        int extraSharding = totalSharding % serverCount;

        int serverIndex = servers.indexOf(server);

        // 计算起始和结束分片
        int start = serverIndex * shardingPerServer + Math.min(serverIndex, extraSharding);
        int end = start + shardingPerServer + (serverIndex < extraSharding ? 1 : 0);

        List<Integer> result = new ArrayList<>();
        for (int i = start; i < end; i++) {
            result.add(i);
        }

        return result;
    }
}
```

### 任务执行监控

```java
/**
 * 任务执行监控
 */
@Service
public class JobMonitoringService {

    @Autowired
    private JobExecutionRepository executionRepository;

    @Scheduled(fixedRate = 60000)
    public void monitorJobExecution() {
        // 1. 检查正在执行的任务
        List<JobExecution> runningExecutions = executionRepository.findRunningExecutions();

        for (JobExecution execution : runningExecutions) {
            // 检查是否超时
            if (isTimeout(execution)) {
                handleTimeoutExecution(execution);
            }

            // 检查执行进度
            checkExecutionProgress(execution);
        }

        // 2. 统计任务执行情况
        reportExecutionStatistics();
    }

    private boolean isTimeout(JobExecution execution) {
        long runningTime = System.currentTimeMillis() - execution.getStartTime().getTime();
        long timeout = getJobTimeout(execution.getJobName());
        return runningTime > timeout;
    }

    private void handleTimeoutExecution(JobExecution execution) {
        log.warn("任务执行超时: job={}, executionId={}, runningTime={}",
            execution.getJobName(),
            execution.getId(),
            runningTime);

        // 发送告警
        alertService.sendAlert(
            AlertType.JOB_TIMEOUT,
            AlertLevel.WARNING,
            execution.getJobName()
        );
    }

    private void reportExecutionStatistics() {
        // 计算任务成功率
        long total = executionRepository.countTodayExecutions();
        long success = executionRepository.countTodaySuccessExecutions();
        long failed = executionRepository.countTodayFailedExecutions();

        double successRate = total > 0 ? (double) success / total * 100 : 0;

        // 上报指标
        metricsService.record("job.execution.total", total);
        metricsService.record("job.execution.success", success);
        metricsService.record("job.execution.failed", failed);
        metricsService.record("job.execution.success_rate", successRate);
    }
}
```

```java
// 使用分布式锁 + 任务执行记录
public class IdempotentJobExecutor {

    private final RedisTemplate<String, Object> redisTemplate;

    public void executeWithIdempotency(String jobName, Runnable task) {
        String executionKey = "job:execution:" + jobName + ":" + getCurrentDate();

        // 1. 检查是否已执行
        if (Boolean.TRUE.equals(redisTemplate.hasKey(executionKey))) {
            log.info("任务今日已执行，跳过: {}", jobName);
            return;
        }

        // 2. 执行任务
        try {
            task.run();

            // 3. 标记已执行
            redisTemplate.opsForValue().set(executionKey, "1", Duration.ofDays(1));

        } catch (Exception e) {
            log.error("任务执行失败", e);
            throw e;
        }
    }
}
```

### 问题二：分片不均匀

```java
// 使用加权分片策略
public class WeightedShardingStrategy implements ShardingStrategy {

    @Override
    public Map<Integer, String> sharding(
            List<String> jobName,
            LinkedHashSet<String> shardingItems,
            JobConfiguration jobConfiguration) {

        // 获取各节点的处理能力
        Map<String, Integer> serverCapacity = getServerCapacity();

        // 计算权重
        Map<Integer, String> result = new LinkedHashMap<>();
        int totalWeight = serverCapacity.values().stream().mapToInt(Integer::intValue).sum();

        int index = 0;
        for (Map.Entry<String, Integer> entry : serverCapacity.entrySet()) {
            int shardingCount = (int) Math.ceil(
                (double) entry.getValue() / totalWeight * shardingItems.size()
            );

            for (int i = 0; i < shardingCount && index < shardingItems.size(); i++) {
                result.put(index, entry.getKey());
                index++;
            }
        }

        return result;
    }
}
```

```java
// 使用超时控制
public class TimeoutJobExecutor {

    public <T> T executeWithTimeout(Callable<T> task, long timeout, TimeUnit unit) {
        ExecutorService executor = Executors.newSingleThreadExecutor();
        Future<T> future = executor.submit(task);

        try {
            return future.get(timeout, unit);
        } catch (TimeoutException e) {
            future.cancel(true);
            throw new JobTimeoutException("任务执行超时");
        } catch (Exception e) {
            throw new RuntimeException(e);
        } finally {
            executor.shutdownNow();
        }
    }
}
```

## 总结

分布式定时任务的模式选择：

| 模式 | 特点 | 适用场景 |
|---|---|---|
| 抢占模式 | 只有一个节点执行 | 任务不可并行 |
| 分片模式 | 任务分配到不同节点 | 大数据量处理 |
| 广播模式 | 所有节点都执行 | 同步配置、清理缓存 |
| 混合模式 | 组合使用 | 复杂业务场景 |

框架选择：

| 框架 | 特点 | 适用场景 |
|---|---|---|
| XXL-Job | 轻量，易用 | 中小型项目 |
| ElasticJob | 功能丰富，支持分片 | 大型项目 |
| Quartz | 功能全面 | 需要事务支持 |
| DolphinScheduler | 工作流 | ETL 场景 |

记住：**没有最好的框架，只有最适合的方案**。要根据团队技术栈和业务需求选择。
