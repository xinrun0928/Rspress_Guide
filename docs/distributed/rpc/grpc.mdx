# gRPC：高性能跨语言 RPC

你有多个服务，分别用 Java、Go、Python 编写。它们之间需要相互调用。

你有几个选择：HTTP JSON、Thrift、或者——gRPC。

gRPC 是 Google 主导的 RPC 框架，凭借 Protobuf 和 HTTP/2 的加持，在性能和跨语言支持上几乎无人能敌。

## 什么是 gRPC

gRPC（Google Remote Procedure Call）是一个高性能、开源的通用 RPC 框架，支持多种语言，实现跨平台、跨语言的服务调用。

```yaml
# gRPC 核心特点
特性                  说明
─────────────────────────────────────────────────────────────────
跨语言                支持 C++、Java、Python、Go、C# 等 10+ 语言
高性能                基于 HTTP/2，支持多路复用
强类型                基于 Protocol Buffers，编译时类型检查
双向流                支持客户端和服务端双向流式调用
自动生成代码          .proto 文件自动生成客户端/服务端代码
```

## Protocol Buffers

Protocol Buffers（简称 Protobuf）是 gRPC 的接口定义语言和数据序列化格式。

### 定义服务

```protobuf
// user.proto
syntax = "proto3";

package user;

option java_multiple_files = true;
option java_package = "com.example.grpc.user";
option java_outer_classname = "UserServiceProto";

// 用户服务
service UserService {
    // 简单 RPC
    rpc GetUser (GetUserRequest) returns (User);

    // 服务端流式 RPC
    rpc ListUsers (ListUsersRequest) returns (stream User);

    // 客户端流式 RPC
    rpc BatchGetUsers (stream GetUserRequest) returns (BatchGetUsersResponse);

    // 双向流式 RPC
    rpc StreamChat (stream ChatMessage) returns (stream ChatMessage);
}

// 请求消息
message GetUserRequest {
    int64 id = 1;
}

message ListUsersRequest {
    int32 page = 1;
    int32 page_size = 2;
}

// 响应消息
message User {
    int64 id = 1;
    string name = 2;
    string email = 3;
    string phone = 4;
    int32 age = 5;
    repeated string roles = 6;
    map<string, string> metadata = 7;
}

message BatchGetUsersResponse {
    repeated User users = 1;
}

// 聊天消息
message ChatMessage {
    string content = 1;
    string sender = 2;
    int64 timestamp = 3;
}
```

### 数据类型映射

```yaml
# Protobuf 数据类型与各语言对应
Protobuf 类型       Java 类型        Python 类型      说明
─────────────────────────────────────────────────────────────────
double              double           float           双精度浮点
float               float            float           单精度浮点
int32               int              int            32 位整数
int64               long             int            64 位整数
bool                boolean          bool           布尔值
string              String           str            字符串
bytes               ByteString       bytes          字节数组
repeated T          List<T>          List[T]        数组
map<K, V>           Map<K, V>        Dict[K, V]     映射
```

## gRPC 服务端

### Java 服务端

```java
// 实现服务
public class UserServiceImpl extends UserServiceGrpc.UserServiceImplBase {

    private final UserRepository userRepository;

    public UserServiceImpl(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    // 简单 RPC
    @Override
    public void getUser(GetUserRequest request, StreamObserver<User> responseObserver) {
        Long userId = request.getId();

        Optional<UserEntity> userOpt = userRepository.findById(userId);

        if (userOpt.isPresent()) {
            UserEntity entity = userOpt.get();
            User user = User.newBuilder()
                    .setId(entity.getId())
                    .setName(entity.getName())
                    .setEmail(entity.getEmail())
                    .setAge(entity.getAge())
                    .addAllRoles(entity.getRoles())
                    .putAllMetadata(entity.getMetadata())
                    .build();

            responseObserver.onNext(user);
            responseObserver.onCompleted();
        } else {
            responseObserver.onError(
                    Status.NOT_FOUND
                            .withDescription("User not found: " + userId)
                            .asRuntimeException()
            );
        }
    }

    // 服务端流式 RPC
    @Override
    public void listUsers(ListUsersRequest request,
                         StreamObserver<User> responseObserver) {
        int page = request.getPage();
        int pageSize = request.getPageSize();

        List<UserEntity> users = userRepository.findAll(page, pageSize);

        for (UserEntity entity : users) {
            User user = convertToProto(entity);
            responseObserver.onNext(user);
        }

        responseObserver.onCompleted();
    }
}

// 启动服务
public class GrpcServer {

    public static void main(String[] args) throws IOException, InterruptedException {
        Server server = ServerBuilder.forPort(50051)
                .addService(new UserServiceImpl(new UserRepository()))
                .build()
                .start();

        System.out.println("gRPC Server started on port 50051");

        Runtime.getRuntime().addShutdownHook(new Thread(() -> {
            System.out.println("Shutting down gRPC server");
            server.shutdown();
        }));

        server.awaitTermination();
    }
}
```

### Go 服务端

```go
// server/main.go
package main

import (
    "context"
    "log"
    "net"

    "google.golang.org/grpc"
    pb "example.com/user"
)

type UserServer struct {
    pb.UnimplementedUserServiceServer
}

func (s *UserServer) GetUser(ctx context.Context, req *pb.GetUserRequest) (*pb.User, error) {
    user, err := repository.FindByID(req.Id)
    if err != nil {
        return nil, status.Errorf(codes.NotFound, "user not found: %v", err)
    }

    return &pb.User{
        Id:    user.ID,
        Name:  user.Name,
        Email: user.Email,
    }, nil
}

func (s *UserServer) ListUsers(req *pb.ListUsersRequest, stream pb.UserService_ListUsersServer) error {
    users, err := repository.FindAll(int(req.Page), int(req.PageSize))
    if err != nil {
        return err
    }

    for _, user := range users {
        stream.Send(&pb.User{
            Id:    user.ID,
            Name:  user.Name,
            Email: user.Email,
        })
    }

    return nil
}

func main() {
    lis, _ := net.Listen("tcp", ":50051")
    s := grpc.NewServer()
    pb.RegisterUserServiceServer(s, &UserServer{})
    log.Printf("Server started on :50051")
    s.Serve(lis)
}
```

## gRPC 客户端

### Java 客户端

```java
public class GrpcClient {

    public static void main(String[] args) {
        // 创建 Channel
        ManagedChannel channel = ManagedChannelBuilder
                .forAddress("localhost", 50051)
                .usePlaintext()  // 生产环境应使用 TLS
                .build();

        try {
            // 创建 Stub
            UserServiceGrpc.UserServiceBlockingStub blockingStub =
                    UserServiceGrpc.newBlockingStub(channel);

            // 简单 RPC 调用
            GetUserRequest request = GetUserRequest.newBuilder()
                    .setId(1L)
                    .build();

            User user = blockingStub.getUser(request);
            System.out.println("User: " + user.getName());

            // 服务端流式调用
            ListUsersRequest listRequest = ListUsersRequest.newBuilder()
                    .setPage(1)
                    .setPageSize(10)
                    .build();

            Iterator<User> users = blockingStub.listUsers(listRequest);
            while (users.hasNext()) {
                System.out.println("User: " + users.next().getName());
            }

        } finally {
            channel.shutdown();
        }
    }
}
```

## 四种调用模式

### 1. 简单 RPC（Unary RPC）

```protobuf
rpc GetUser (GetUserRequest) returns (User);
```

```java
// 客户端发送一个请求，服务器返回一个响应
GetUserRequest request = GetUserRequest.newBuilder()
        .setId(1L)
        .build();
User user = stub.getUser(request);
```

### 2. 服务端流式 RPC（Server Streaming RPC）

```protobuf
rpc ListUsers (ListUsersRequest) returns (stream User);
```

```java
// 客户端发送一个请求，服务器返回多个响应
ListUsersRequest request = ListUsersRequest.newBuilder()
        .setPage(1)
        .setPageSize(100)
        .build();

Iterator<User> users = stub.listUsers(request);
while (users.hasNext()) {
    User user = users.next();
    System.out.println(user.getName());
}
```

### 3. 客户端流式 RPC（Client Streaming RPC）

```protobuf
rpc BatchGetUsers (stream GetUserRequest) returns (BatchGetUsersResponse);
```

```java
// 客户端发送多个请求，服务器返回一个响应
StreamObserver<GetUserRequest> requestStream = stub.batchGetUsers(
        new StreamObserver<BatchGetUsersResponse>() {
            @Override
            public void onNext(BatchGetUsersResponse response) {
                response.getUsersList().forEach(user ->
                        System.out.println(user.getName()));
            }

            @Override
            public void onError(Throwable t) {
                t.printStackTrace();
            }

            @Override
            public void onCompleted() {
                System.out.println("Batch request completed");
            }
        });

// 发送多个请求
for (Long userId : userIds) {
    requestStream.onNext(GetUserRequest.newBuilder()
            .setId(userId)
            .build());
}
requestStream.onCompleted();
```

### 4. 双向流式 RPC（Bidirectional Streaming RPC）

```protobuf
rpc StreamChat (stream ChatMessage) returns (stream ChatMessage);
```

```java
// 客户端和服务器都可以发送多个消息
StreamObserver<ChatMessage> chatStream = stub.streamChat(
        new StreamObserver<ChatMessage>() {
            @Override
            public void onNext(ChatMessage message) {
                System.out.println("Received: " + message.getContent());
            }

            @Override
            public void onError(Throwable t) {
                t.printStackTrace();
            }

            @Override
            public void onCompleted() {
                System.out.println("Chat ended");
            }
        });

// 发送消息
chatStream.onNext(ChatMessage.newBuilder()
        .setContent("Hello!")
        .setSender("client")
        .setTimestamp(System.currentTimeMillis())
        .build());
```

## HTTP/2 特性

```yaml
# HTTP/2 相比 HTTP/1.1 的优势
特性                  HTTP/1.1              HTTP/2
─────────────────────────────────────────────────────────────────
多路复用             ❌                    ✅
header 压缩          ❌                    ✅ (HPACK)
流控制               依赖 TCP              ✅
```

## gRPC vs REST

```yaml
# gRPC vs REST 对比
维度                  gRPC                  REST
─────────────────────────────────────────────────────────────────
协议                  HTTP/2                HTTP/1.1 / HTTP/2
序列化                Protobuf              JSON / XML
接口定义              .proto                OpenAPI/Swagger
类型安全              编译时检查            运行时检查
代码生成              原生支持              需要工具
浏览器支持            需要 grpc-web         原生支持
流式支持              原生支持              需用 SSE/WebSocket
性能                  极高                  一般
学习曲线              较陡                  较平缓
生态                  较小                  庞大
```

## 最佳实践

### 实践一：使用 TLS

```yaml
# 生产环境必须使用 TLS
grpc:
  server:
    port: 50051
    security:
      enabled: true
      certificate: /path/to/cert.pem
      private-key: /path/to/key.pem
```

### 实践二：合理使用流

```java
// 大量数据传输使用流式调用
// 好：服务端流式返回
stub.listUsers(request)

// 不好：一次性返回所有数据
stub.getAllUsers(request)  // 可能导致 OOM
```

### 实践三：设置超时

```java
// 始终设置超时
 ManagedChannel channel = ManagedChannelBuilder
        .forAddress("localhost", 50051)
        .usePlaintext()
        .build();

try {
    UserServiceGrpc.UserServiceBlockingStub stub =
            UserServiceGrpc.newBlockingStub(channel)
                    .withDeadlineAfter(5, TimeUnit.SECONDS);

    User user = stub.getUser(request);
} catch (Status.DEADLINE_EXCEEDED e) {
    // 超时处理
}
```

## 总结

gRPC 是一个高性能、跨语言的 RPC 框架，特别适合：

- 微服务之间的高性能通信
- 多语言混合开发场景
- 需要流式处理的场景
- 对性能要求极高的系统

**核心要点回顾：**

- gRPC 基于 HTTP/2 和 Protocol Buffers，性能极高
- 支持四种调用模式：简单 RPC、服务端流、客户端流、双向流
- .proto 文件定义接口，自动生成多语言代码
- 生产环境应使用 TLS 加密
- 始终设置合理的超时时间
