# Feign 压缩与连接池优化

你的 Feign 调用延迟高，吞吐量上不去。除了优化业务代码，网络层的配置也很重要。

Feign 支持 GZIP 压缩和连接池，可以显著减少网络开销。

## 请求压缩

### 启用压缩

```yaml
# application.yml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          # 压缩的 MIME 类型
          mime-types: text/xml,application/xml,application/json
          # 最小压缩阈值
          min-request-size: 2048
        response:
          enabled: true
```

### Java 配置

```java
@Configuration
public class CompressionConfig {

    @Bean
    public RequestInterceptor compressionInterceptor() {
        return template -> {
            // 添加压缩请求头
            template.header("Accept-Encoding", "gzip, deflate");
            template.header("Content-Encoding", "gzip");
        };
    }
}
```

## 连接池优化

### 默认 HttpURLConnection

Feign 默认使用 JDK 的 `HttpURLConnection`，每次请求都会创建新连接，性能较差。

```yaml
# 默认配置的问题
问题：
- 每次请求创建新连接
- TCP 握手耗时
- 无法复用连接
```

### 使用 Apache HttpClient 连接池

```xml
<!-- 引入 HttpClient 依赖 -->
<dependency>
    <groupId>io.github.openfeign</groupId>
    <artifactId>feign-httpclient</artifactId>
</dependency>
```

```yaml
# 配置 HttpClient 连接池
spring:
  cloud:
    openfeign:
      httpclient:
        enabled: true  # 启用 HttpClient
        hc:
          # 连接池配置
          pool:
            max-connections: 200           # 最大连接数
            max-connections-per-route: 50 # 每路由最大连接数
            time-to-live: 900             # 连接空闲存活时间（秒）
            time-to-live-unit: SECONDS    # 时间单位
          # 连接配置
          connect-timeout: 3000           # 连接超时
          read-timeout: 5000             # 读取超时
          write-timeout: 5000             # 写入超时
          follow-redirects: true           # 跟随重定向
```

### 使用 OkHttp 连接池

```xml
<!-- 引入 OkHttp 依赖 -->
<dependency>
    <groupId>io.github.openfeign</groupId>
    <artifactId>feign-okhttp</artifactId>
</dependency>
```

```yaml
# 配置 OkHttp
spring:
  cloud:
    openfeign:
      okhttp:
        enabled: true
```

```java
// 自定义 OkHttp 配置
@Configuration
public class OkHttpConfig {

    @Bean
    public OkHttpClient okHttpClient() {
        return new OkHttpClient.Builder()
                // 连接池
                .connectionPool(new ConnectionPool(
                        200,            // 最大连接数
                        15,             // 空闲存活时间
                        TimeUnit.MINUTES
                ))
                // 超时配置
                .connectTimeout(3, TimeUnit.SECONDS)
                .readTimeout(5, TimeUnit.SECONDS)
                .writeTimeout(5, TimeUnit.SECONDS)
                // 连接升级
                .protocols(List.of(Protocol.HTTP_1_1, Protocol.HTTP_2))
                // 重试
                .retryOnConnectionFailure(true)
                // 拦截器
                .addInterceptor(chain -> {
                    Request request = chain.request().newBuilder()
                            .header("X-Client", "OkHttp")
                            .build();
                    return chain.proceed(request);
                })
                .build();
    }
}
```

## 连接池原理

### 连接池工作原理

```yaml
# 连接池工作原理
┌─────────────────────────────────────────────────────────────┐
│                    连接池工作原理                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  连接池                                                    │
│  ┌─────────────────────────────────────────────────────┐  │
│  │ Connection Pool                                      │  │
│  │                                                      │  │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐         │  │
│  │  │ Conn 1   │  │ Conn 2   │  │ Conn 3   │  ...    │  │
│  │  │ user-svc │  │ user-svc │  │ order-.. │         │  │
│  │  │ ACTIVE   │  │ IDLE     │  │ ACTIVE   │         │  │
│  │  └──────────┘  └──────────┘  └──────────┘         │  │
│  │                                                      │  │
│  │  Max Connections: 200                               │  │
│  │  Per-Route Max: 50                                  │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                             │
│  请求流程:                                                 │
│  1. 请求到来，检查连接池是否有可用连接                      │
│  2. 有空闲连接，复用                                       │
│  3. 无空闲连接，创建新连接                                 │
│  4. 连接用完，归还连接池                                    │
│  5. 空闲连接超时，关闭连接                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 连接池参数说明

```yaml
# 连接池参数说明
参数                        说明                  推荐值
─────────────────────────────────────────────────────────────────
max-connections            最大连接数             根据 QPS 和延迟调整
max-connections-per-route  每路由最大连接数       估算并发请求数
time-to-live             连接空闲存活时间        5-15 分钟
time-to-live-unit        时间单位               MINUTES
connect-timeout          建立连接超时           3-5 秒
read-timeout             读取超时               5-10 秒
```

## 性能优化实践

### 完整配置示例

```yaml
# application.yml 完整配置
spring:
  application:
    name: order-service

  cloud:
    openfeign:
      # 压缩配置
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 1024
        response:
          enabled: true

      # HttpClient 连接池
      httpclient:
        enabled: true
        hc:
          pool:
            max-connections: 300
            max-connections-per-route: 100
            time-to-live: 900
            time-to-live-unit: SECONDS
          connect-timeout: 3000
          read-timeout: 5000
          write-timeout: 5000
          follow-redirects: true

      # 日志配置
      logging:
        level:
          com.example.feign: DEBUG

    # LoadBalancer 配置
    loadbalancer:
      ribbon:
        enabled: false
```

### Java 配置类

```java
@Configuration
public class FeignPerformanceConfig {

    // HttpClient 连接池
    @Bean
    public ApacheHttpClient apacheHttpClient() {
        RequestConfig config = RequestConfig.custom()
                .setConnectTimeout(3000)
                .setSocketTimeout(5000)
                .setConnectionRequestTimeout(3000)
                .setRedirectsEnabled(true)
                .setRelativeRedirectsAllowed(true)
                .build();

        return new ApacheHttpClient(
                HttpClientBuilder.create()
                        .setDefaultRequestConfig(config)
                        .setMaxConnTotal(300)
                        .setMaxConnPerRoute(100)
                        .evictIdleConnections(15, TimeUnit.SECONDS)
                        .build()
        );
    }

    // 超时配置
    @Bean
    public Options feignOptions() {
        return new Request.Options(
                3000, TimeUnit.MILLISECONDS,    // ConnectTimeout
                5000, TimeUnit.MILLISECONDS,    // ReadTimeout
                true                            // FollowRedirects
        );
    }

    // 日志级别
    @Bean
    public Logger.Level feignLoggerLevel() {
        return Logger.Level.BASIC;
    }

    // 请求拦截器
    @Bean
    public RequestInterceptor requestInterceptor() {
        return template -> {
            template.header("X-Request-Id", UUID.randomUUID().toString());
        };
    }
}
```

## 重试配置

### 重试机制

```yaml
# Feign 重试配置
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            # 禁用重试
            retryer: never
```

```java
// 自定义重试策略
@Configuration
public class RetryConfig {

    @Bean
    public Retryer retryer() {
        return new Retryer.Default(
                100,                  // 初始间隔（毫秒）
                1000,                 // 最大间隔（毫秒）
                3                     // 最大重试次数
        );
    }
}

// 永久重试
@Bean
public Retryer foreverRetryer() {
    return new Retryer() {
        @Override
        public void continueOrPropagate(RetryableException e) {
            if (e.getAttemptCount() > 5) {
                throw e;
            }
            try {
                Thread.sleep((long) Math.min(1000L * Math.pow(2, e.getAttemptCount()), 5000));
            } catch (InterruptedException ex) {
                Thread.currentThread().interrupt();
            }
            throw e;
        }

        @Override
        public Retryer clone() {
            return this;
        }
    };
}
```

## 最佳实践

### 实践一：选择合适的连接池

```yaml
# 连接池选择建议
场景                  推荐              说明
─────────────────────────────────────────────────────────────────
简单场景              默认              无需额外配置
中等并发              Apache HttpClient  成熟稳定
高并发低延迟           OkHttp           性能更好，支持 HTTP/2
```

### 实践二：合理设置连接池大小

```yaml
# 连接池大小计算
公式：
max-connections = QPS × 平均延迟（秒） × 2

示例：
QPS = 1000
平均延迟 = 100ms = 0.1s
max-connections = 1000 × 0.1 × 2 = 200
```

### 实践三：监控关键指标

```java
// 关键指标监控
@Component
public class FeignHealthIndicator implements HealthIndicator {

    private final ApacheHttpClient client;

    @Override
    protected Health health() {
        PoolingHttpClientConnectionManager pool = (PoolingHttpClientConnectionManager) client
                .getHttpClient();

        Health health = Health.up()
                .withDetail("leased", pool.getStats().getLeased())
                .withDetail("available", pool.getStats().getAvailable())
                .withDetail("pending", pool.getStats().getPending())
                .withDetail("max", pool.getTotalStats().getMax());

        if (pool.getTotalStats().getLeased() >= pool.getTotalStats().getMax() * 0.8) {
            health.down().withDetail("reason", "Connection pool nearly exhausted");
        }

        return health.build();
    }
}
```

## 总结

Feign 的压缩和连接池优化可以显著提升 HTTP 调用的性能。

**核心要点回顾：**

- 启用 GZIP 压缩减少网络传输量
- 使用连接池避免重复建立 TCP 连接
- Apache HttpClient 和 OkHttp 都是优秀的连接池实现
- 合理配置连接池大小：`max-connections = QPS × 延迟 × 2`
- 监控连接池状态，及时发现瓶颈
