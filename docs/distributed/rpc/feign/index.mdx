# OpenFeign：声明式 HTTP 客户端

你有没有被这种代码折磨过？

```java
RestTemplate template = new RestTemplate();
HttpHeaders headers = new HttpHeaders();
headers.setContentType(MediaType.APPLICATION_JSON);

HttpEntity<Request> entity = new HttpEntity<>(request, headers);

ResponseEntity<Response> response = template.exchange(
    "http://order-service/api/orders",
    HttpMethod.POST,
    entity,
    Response.class
);
```

一个 HTTP 调用，写了 5 行代码，还容易出错。

**OpenFeign 的核心理念是：让 HTTP 调用像调用本地方法一样简单。**

## OpenFeign 解决了什么问题

```
┌─────────────────────────────────────────────────────────┐
│                   传统 HTTP 调用                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 手动拼接 URL（字符串拼接，容易出错）                │
│  2. 手动处理 HTTP 头（Content-Type、Authorization...）   │
│  3. 手动处理序列化/反序列化（Object → JSON → Object）   │
│  4. 手动处理超时、重试、错误码                           │
│                                                         │
│  结果：10行代码，5行是模板                               │
│                                                         │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                   OpenFeign 声明式调用                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  @FeignClient("order-service")                          │
│  public interface OrderClient {                          │
│                                                         │
│      @PostMapping("/api/orders")                        │
│      Response createOrder(@RequestBody Request request);│
│                                                         │
│  }                                                      │
│                                                         │
│  结果：1行接口定义，代码即文档                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## OpenFeign 的核心能力

| 能力 | 说明 | 配置方式 |
|------|------|----------|
| 声明式接口 | 用注解定义 HTTP 调用 | @FeignClient + @GetMapping |
| 负载均衡 | 集成 Ribbon，自动负载均衡 | ribbon.listOfServers |
| 超时控制 | 配置读写超时时间 | feign.client.config |
| 重试机制 | 自动重试失败请求 | Retryer |
| 拦截器 | 请求/响应拦截 | RequestInterceptor |
| 压缩 | GZIP 压缩减少传输量 | feign.compression |
| 日志 | 打印 HTTP 请求日志 | Logger.Level |

## OpenFeign vs RestTemplate

| 维度 | RestTemplate | OpenFeign |
|------|-------------|-----------|
| 代码风格 | 命令式（过程调用） | 声明式（接口定义） |
| URL 拼接 | 字符串拼接 | 注解自动拼接 |
| 参数映射 | 手动处理 | 注解自动映射 |
| 负载均衡 | 需要手动配置 | 内置 Ribbon |
| 可测试性 | 难以测试 | 容易 Mock |
| 代码复用 | 每个调用点重复代码 | 接口可复用 |

## 核心公式

```
OpenFeign = 声明式接口 + 自动序列化 + Ribbon负载均衡 + 拦截器链

优势：代码简洁、可测试、可复用
劣势：不支持响应式、性能不如 Dubbo
适用：Spring Cloud 生态、RESTful API 调用
```
