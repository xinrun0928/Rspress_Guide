# Dubbo：高性能 RPC 框架

Dubbo 是阿里巴巴开源的高性能 Java RPC 框架，专为分布式服务治理场景设计。在微服务架构中，服务间的通信是核心问题，Dubbo 通过其强大的 RPC 能力和完善的服务治理功能，成为国内最受欢迎的分布式服务框架之一。

Dubbo 诞生于 2008 年，最初用于解决阿里巴巴内部大规模服务拆分后的通信问题。2011 年开源后，迅速在国内普及。2017 年 Dubbo 捐献给 Apache 基金会，并于 2019 年正式成为 Apache 顶级项目。目前 Dubbo 3.x 版本主打云原生、服务发现和应用级服务治理，是 Dubbo 历史上最重要的架构升级。

## 为什么需要 Dubbo？

单体应用中，方法调用不过是 JVM 内部的事。拆成微服务后，服务 A 要调用服务 B，网络通信取代了方法调用——这带来了前所未有的复杂度：

- 服务在哪里？（服务发现）
- 调用哪台机器？（负载均衡）
- 调用失败了怎么办？（容错）
- 调用太慢了怎么办？（超时控制）
- 流量太大了怎么办？（限流降级）

Dubbo 的核心价值，就是把这些分布式通信问题封装好，让业务开发者只需要关注业务逻辑本身。

## Dubbo 的三层架构

Dubbo 的设计采用了清晰的三层架构：

```
┌────────────────────────────────────────────────────┐
│                   应用层（Application Layer）        │
│            Service 层：业务接口定义与实现            │
├────────────────────────────────────────────────────┤
│                   RPC 层（RPC Layer）               │
│   Config 层：配置管理    Proxy 层：透明代理          │
│   Registry 层：注册发现   Cluster 层：负载均衡与容错  │
├────────────────────────────────────────────────────┤
│                   Remoting 层（Remote Layer）        │
│      Exchange 层：请求响应   Network 层：网络传输    │
│      Serialize 层：数据序列化  Transport 层：NIO     │
└────────────────────────────────────────────────────┘
```

三层架构的好处是：**每层都可以独立替换和扩展**。序列化层可以从 Hessian 换成 Protobuf，网络层可以从 Netty 换成 gRPC，而不影响上层业务。

## Dubbo 3.x 的核心变化

Dubbo 3.x 相比 2.x 有三个核心变化：

### 一、应用级服务发现

2.x 版本使用**接口级服务发现**：每个服务（接口）对应一个注册中心节点，注册压力大。

3.x 改为**应用级服务发现**：以应用为粒度注册，一个应用只需注册一个节点。这在拥有数百个服务的超大规模集群中，可以将注册中心压力降低 90% 以上。

### 二、Triple 协议（兼容 gRPC）

Dubbo 3.x 引入了 Triple 协议，完全兼容 gRPC 协议。这意味着 Dubbo 服务可以被任何 gRPC 客户端调用，打破了 Dubbo 服务只能被 Dubbo 客户端调用的限制。

### 三、云原生支持

- 支持 Kubernetes 原生服务发现（DNS + Kubernetes Service）
- 支持 Proxyless Mesh 模式
- 支持环境变量注入配置
