# Dubbo 集群容错

服务调用失败了怎么办？重试？快速失败？还是直接返回降级结果？

Dubbo 提供了多种集群容错策略，让你可以根据业务场景灵活选择。

## 什么是集群容错

在分布式系统中，服务调用失败是常态而非例外。网络抖动、服务过载、临时不可用……这些情况随时可能发生。集群容错（Cluster Fault Tolerance）就是在调用失败时，提供一套兜底机制，保证系统的可用性。

## Dubbo 内置容错策略

Dubbo 内置了六种集群容错策略：

```yaml
# Dubbo 集群容错策略
策略                  类名                          特点
─────────────────────────────────────────────────────────────────
Failover              失败自动切换                  自动重试其他服务器
Failfast              快速失败                      只调用一次，失败直接报错
Failsafe              失败安全                      调用失败，记录日志，返回空结果
Failback              失败自动恢复                  失败后异步重试
Forking               并行调用                      同时调用多个服务器，一个成功即返回
Broadcast             广播调用                      逐个调用所有服务器，全部成功才算成功
```

### 1. Failover - 失败自动切换

当调用失败时，自动切换到其他服务器重试。**这是最常用的容错策略。**

```java
// Failover 实现
public class FailoverClusterInvoker<T> extends AbstractClusterInvoker<T> {

    @Override
    public Result doInvoke(Invocation invocation, List<Invoker<T>> invokers, LoadBalance loadbalance) {
        List<Invoker<T>> copy = new ArrayList<>(invokers);
        int len = getUrl().getMethodParameter(invocation.getMethodName(), RETRIES_KEY, DEFAULT_RETRIES) + 1;

        RpcException exception = null;
        Invoker<T> invoked = null;

        for (int i = 0; i < len; i++) {
            if (i > 0) {
                // 重试前检查是否被销毁
                checkWhetherDestroyed();
                // 重新获取 invokers（可能有变化）
                copy = new ArrayList<>(list(invocation));
            }

            Invoker<T> selected = loadbalance.select(copy, invocation);
            invoked = selected;

            try {
                // 执行调用
                Result result = selected.invoke(invocation);
                return result;
            } catch (RpcException e) {
                // 记录异常
                exception = e;
            } catch (Throwable e) {
                exception = new RpcException(e);
            }
        }

        // 所有重试都失败
        throw new RpcException(exception);
    }
}
```

**特点**：

- 适合读操作
- 默认重试 2 次（总共 3 次调用）
- 幂等操作使用

**配置**：

```java
// 注解配置
@Reference(
    cluster = "failover",
    retries = 3
)
private UserService userService;
```

### 2. Failfast - 快速失败

只调用一次，如果失败立即抛出异常。**适合非幂等操作。**

```java
// Failfast 实现
public class FailfastClusterInvoker<T> extends AbstractClusterInvoker<T> {

    @Override
    public Result doInvoke(Invocation invocation, List<Invoker<T>> invokers, LoadBalance loadbalance) {
        checkInvokers(invokers, invocation);

        Invoker<T> selected = loadbalance.select(invokers, invocation);

        try {
            return selected.invoke(invocation);
        } catch (RpcException e) {
            // 抛出异常，不重试
            throw e;
        }
    }
}
```

**特点**：

- 适合非幂等操作，如创建订单
- 失败后立即返回，不浪费时间
- 需要业务自行处理失败

**配置**：

```java
@Reference(cluster = "failfast")
private OrderService orderService;
```

### 3. Failsafe - 失败安全

调用失败时，记录日志并返回空结果，不抛出异常。**适合审计日志等非核心服务。**

```java
// Failsafe 实现
public class FailsafeClusterInvoker<T> extends AbstractClusterInvoker<T> {

    @Override
    public Result doInvoke(Invocation invocation, List<Invoker<T>> invokers, LoadBalance loadbalance) {
        try {
            checkInvokers(invokers, invocation);

            Invoker<T> selected = loadbalance.select(invokers, invocation);

            return selected.invoke(invocation);
        } catch (RpcException e) {
            // 记录日志，返回空结果
            logger.warn("Failsafe call failed", e);
            return AsyncRpcResult.newDefaultAsyncResult(null, null, invocation);
        }
    }
}
```

**特点**：

- 调用失败不抛异常
- 返回空结果
- 适合审计日志等场景

**配置**：

```java
@Reference(cluster = "failsafe")
private AuditLogService auditLogService;
```

### 4. Failback - 失败自动恢复

调用失败后，在后台定时重试。**适合消息通知等场景。**

```java
// Failback 实现
public class FailbackClusterInvoker<T> extends AbstractClusterInvoker<T> {

    private static final long RETRY_INTERVAL = 5000;  // 重试间隔
    private final ConcurrentMap<Long, FailedInvocation> failedInvocations = new ConcurrentHashMap<>();

    @Override
    public Result doInvoke(Invocation invocation, List<Invoker<T>> invokers, LoadBalance loadbalance) {
        try {
            checkInvokers(invokers, invocation);
            Invoker<T> selected = loadbalance.select(invokers, invocation);
            return selected.invoke(invocation);
        } catch (RpcException e) {
            // 记录失败的调用，稍后重试
            addFailedInvocation(invocation);
            // 返回空结果
            return AsyncRpcResult.newDefaultAsyncResult(null, null, invocation);
        }
    }

    private void addFailedInvocation(Invocation invocation) {
        FailedInvocation old = failedInvocations.putIfAbsent(
            invocation.getInvokerId(),
            new FailedInvocation(invocation));

        if (old != null) {
            // 已经在队列中
            return;
        }

        // 定时重试
        retryExecutor.schedule(() -> {
            try {
                FailedInvocation failed = failedInvocations.remove(invocation.getInvokerId());
                if (failed != null) {
                    failed.retry();
                }
            } catch (Exception e) {
                logger.error("Retry failed", e);
            }
        }, RETRY_INTERVAL, TimeUnit.MILLISECONDS);
    }
}
```

**特点**：

- 失败后不立即返回错误
- 在后台异步重试
- 适合不关心即时结果但最终要成功的场景

**配置**：

```java
@Reference(cluster = "failback")
private MessageNotificationService notificationService;
```

### 5. Forking - 并行调用

同时调用多个服务器，只要有一个成功就返回。

```java
// Forking 实现
public class ForkingClusterInvoker<T> extends AbstractClusterInvoker<T> {

    private final ExecutorService executor = Executors.newCachedThreadPool();

    @Override
    public Result doInvoke(Invocation invocation, List<Invoker<T>> invokers, LoadBalance loadbalance) {
        try {
            checkInvokers(invokers, invocation);

            int forks = getUrl().getMethodParameter(invocation.getMethodName(), FORKS_KEY, DEFAULT_FORKS);
            forks = Math.min(forks, invokers.size());

            ExecutorService executor = Executors.newThreadPoolExecutor(forks, forks, ...);

            List<Callable<Result>> calls = new ArrayList<>();
            for (Invoker<T> invoker : invokers.subList(0, forks)) {
                calls.add(() -> {
                    return invoker.invoke(invocation);
                });
            }

            // 并行执行
            List<Result> results = executor.invokeAll(calls);

            // 返回第一个成功的结果
            for (Result result : results) {
                if (result != null && result.hasException() == false) {
                    return result;
                }
            }

            // 所有都失败，抛出异常
            return results.get(0);
        } catch (Exception e) {
            throw new RpcException(e);
        }
    }
}
```

**特点**：

- 并行调用多个服务器
- 只要有一个成功就返回
- 延迟最低，但资源消耗大

**配置**：

```java
@Reference(
    cluster = "forking",
    forks = 4,
    timeout = 5000
)
private IdempotentService idempotentService;
```

### 6. Broadcast - 广播调用

逐个调用所有服务器，所有服务器都成功才算成功。

```java
// Broadcast 实现
public class BroadcastClusterInvoker<T> extends AbstractClusterInvoker<T> {

    @Override
    public Result doInvoke(Invocation invocation, List<Invoker<T>> invokers, LoadBalance loadbalance) {
        checkInvokers(invokers, invocation);

        RpcException exception = null;

        // 逐个调用
        for (Invoker<T> invoker : invokers) {
            try {
                invoker.invoke(invocation);
            } catch (RpcException e) {
                exception = e;
                // 继续调用其他服务器
            }
        }

        // 如果有任何失败，抛出异常
        if (exception != null) {
            throw exception;
        }

        return AsyncRpcResult.newDefaultAsyncResult(null, null, invocation);
    }
}
```

**特点**：

- 调用所有服务器
- 所有成功才算成功
- 适合通知类操作，如刷新缓存

**配置**：

```java
@Reference(cluster = "broadcast")
private CacheRefreshService cacheRefreshService;
```

## 集群容错配置

### 全局配置

```yaml
# application.yml
dubbo:
  consumer:
    cluster: failover          # 全局默认容错策略
    retries: 2                  # 全局默认重试次数
```

### 服务级别配置

```java
// 服务级别配置
@Reference(
    cluster = "failover",
    retries = 3,
    timeout = 5000
)
private UserService userService;

// 非幂等操作
@Reference(cluster = "failfast")
private OrderService orderService;
```

### 方法级别配置

```java
// 方法级别配置
@Reference(
    methods = {
        @Method(name = "getUser", cluster = "failover", retries = 2),
        @Method(name = "createOrder", cluster = "failfast")
    }
)
private UserService userService;
```

## 集群容错最佳实践

### 实践一：幂等操作用 Failover

```java
// 幂等操作使用 Failover
@Reference(cluster = "failover", retries = 2)
private UserService userService;

// 幂等操作示例：
// - GET 请求
// - UPDATE（根据 ID 更新）
// - DELETE（根据 ID 删除）
// - 查询类操作
```

### 实践二：非幂等操作用 Failfast

```java
// 非幂等操作使用 Failfast
@Reference(cluster = "failfast")
private OrderService orderService;

// 非幂等操作示例：
// - 创建订单（可能重复创建）
// - 扣款操作（可能重复扣款）
// - 发送短信（可能重复发送）
```

### 实践三：非核心服务用 Failsafe

```java
// 非核心服务使用 Failsafe
@Reference(cluster = "failsafe")
private AuditLogService auditLogService;

// 非核心服务示例：
// - 审计日志
// - 数据统计
// - 行为分析
```

### 实践四：核心服务配合降级

```java
// 核心服务配合降级
@Reference(
    cluster = "failover",
    retries = 2,
    fallback = "userServiceFallback"
)
private UserService userService;

public User userServiceFallback(long userId) {
    // 降级返回
    return getUserFromCache(userId);
}
```

## 总结

Dubbo 提供了六种集群容错策略，覆盖了不同的业务场景。

**核心要点回顾：**

- Failover：自动切换，适合幂等操作（最常用）
- Failfast：快速失败，适合非幂等操作
- Failsafe：失败安全，适合非核心操作
- Failback：失败恢复，适合异步通知
- Forking：并行调用，适合高性能场景
- Broadcast：广播调用，适合所有节点都要执行
- 实际应用中，常配合降级、熔断一起使用
