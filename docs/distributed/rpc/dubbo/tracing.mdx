# Dubbo 链路追踪

你的用户反馈页面加载很慢。问题是订单服务慢，还是商品服务慢，还是它们之间的网络慢？

没有链路追踪，这个问题根本无从定位。

链路追踪是分布式系统可观测性的重要组成部分，帮助你理解请求在系统中的完整流转路径。

## 什么是链路追踪

链路追踪（Distributed Tracing）是通过为每个请求生成一个唯一的追踪 ID（TraceId），并在请求经过的每个服务间传递，从而还原请求的完整调用链。

```yaml
# 链路追踪原理
┌─────────────────────────────────────────────────────────────┐
│                     链路追踪原理                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  TraceId: abc123                                           │
│                                                             │
│  ┌────────────┐                                            │
│  │  Gateway   │ ─── SpanId: 0.1 ──────────────────────►  │
│  │            │                                            │
│  └────────────┘                                            │
│        │                                                   │
│        ▼                                                   │
│  ┌────────────┐                                            │
│  │Order Service│ ◄── SpanId: 0.1.1 ── parent: 0.1 ──────► │
│  │            │                                            │
│  └────────────┘                                            │
│        │                                                   │
│        ├──► ┌────────────┐                                │
│        │    │User Service │ ◄── SpanId: 0.1.1.1 ────────► │
│        │    └────────────┘                                │
│        │                                                   │
│        └──► ┌────────────┐                                │
│             │Product Svc │ ◄── SpanId: 0.1.1.2 ────────► │
│             └────────────┘                                │
│                                                             │
│  Span: 一次 RPC 调用                                        │
│  Trace: 一次完整请求的所有 Span                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 核心概念

```yaml
# 链路追踪核心概念
概念              说明
─────────────────────────────────────────────────────────────────
Trace            一次分布式请求的完整路径
Span             一次 RPC 调用或内部操作
SpanId           Span 在 Trace 中的位置
ParentSpanId     父 Span 的 ID，形成调用树
Annotation       标记事件，如 CS、CR、SS、SR
```

```yaml
# Span 生命周期事件
CS (Client Send)     客户端发起请求
CR (Client Receive)   客户端收到响应
SS (Server Start)    服务端开始处理
SR (Server Receive)  服务端收到请求
```

## Dubbo 集成链路追踪

### 1. 使用 Apache SkyWalking

SkyWalking 是 Apache 顶级项目，对 Dubbo 有良好的支持。

```xml
<!-- 引入 SkyWalking Agent -->
<dependency>
    <groupId>org.apache.skywalking</groupId>
    <artifactId>apm-toolkit-opentracing</artifactId>
    <version>8.12.0</version>
</dependency>
```

```yaml
# SkyWalking 配置
# skywalking-agent/config/agent.config
agent.service_name=dubbo-provider
collector.backend_service=127.0.0.1:11800
```

```java
// 启动时添加 Agent
// java -javaagent:skywalking-agent.jar -jar dubbo-provider.jar
```

### 2. 使用 Zipkin

Zipkin 是 Twitter 开源的链路追踪系统。

```xml
<!-- 引入 Zipkin Brave -->
<dependency>
    <groupId>io.zipkin.brave</groupId>
    <artifactId>brave-dubbo</artifactId>
    <version>5.13.4</version>
</dependency>
```

```java
// 配置 Brave Filter
@Configuration
public class TracingConfig {

    @Bean
    public Tracing tracing() {
        return Tracing.newBuilder()
                .localServiceName("dubbo-provider")
                .spanReporter(Reporter.NO_OP)  // 使用 Reporter 异步上报
                .build();
    }

    @Bean
    public Filter tracingFilter(Tracing tracing) {
        return new TracingFilter(tracing);
    }
}
```

### 3. 使用 OpenTelemetry

OpenTelemetry 是 CNCF 的可观测性标准。

```xml
<!-- 引入 OpenTelemetry -->
<dependency>
    <groupId>io.opentelemetry</groupId>
    <artifactId>opentelemetry-api</artifactId>
    <version>1.20.0</version>
</dependency>
```

```java
// OpenTelemetry 配置
@Configuration
public class OtelConfig {

    @Bean
    public OpenTelemetry openTelemetry() {
        return OpenTelemetrySdk.builder()
                .setTracerProvider(
                    SdkTracerProvider.builder()
                        .addSpanProcessor(
                            BatchSpanProcessor.builder(
                                OtlpGrpcSpanExporter.getDefault()
                            ).build()
                        )
                        .build()
                )
                .build();
    }
}
```

## Dubbo 链路追踪实现

### 自定义链路追踪过滤器

```java
// Dubbo 链路追踪过滤器
@Activate(group = {Constants.CONSUMER, Constants.PROVIDER})
public class TracingFilter implements Filter {

    private Tracer tracer;

    public TracingFilter(Tracer tracer) {
        this.tracer = tracer;
    }

    @Override
    public Result invoke(Invoker<?> invoker, Invocation invocation) throws RpcException {
        RpcContext context = RpcContext.getContext();

        // 获取或创建 TraceId
        String traceId = context.getAttachment("X-TraceId");
        if (traceId == null) {
            traceId = generateTraceId();
        }

        // 获取或创建 SpanId
        String spanId = generateSpanId();

        // 获取父 SpanId
        String parentSpanId = context.getAttachment("X-ParentSpanId");

        // 构建 Span
        Span span = tracer.buildSpan(invocation.getMethodName())
                .withTag("service", invoker.getInterface().getName())
                .withTag("method", invocation.getMethodName())
                .withTag("trace.id", traceId)
                .withTag("span.id", spanId)
                .start();

        try (Scope scope = tracer.withSpan(span)) {
            // 将 TraceId 传递给下游
            context.setAttachment("X-TraceId", traceId);
            context.setAttachment("X-ParentSpanId", spanId);

            // 执行调用
            Result result = invoker.invoke(invocation);

            // 记录结果
            if (result.hasException()) {
                span.recordException(result.getException());
            }

            return result;
        } finally {
            span.end();
        }
    }

    private String generateTraceId() {
        return UUID.randomUUID().toString().replace("-", "");
    }

    private String generateSpanId() {
        return String.valueOf(System.nanoTime() % 1000000);
    }
}
```

### 配置过滤器

```yaml
# 启用追踪过滤器
dubbo:
  provider:
    filter: tracing
  consumer:
    filter: tracing
```

## 链路追踪最佳实践

### 实践一：关键参数透传

```java
// 确保 TraceId 在整个调用链中透传
public class TraceContextFilter implements Filter {

    @Override
    public Result invoke(Invoker<?> invoker, Invocation invocation) {
        RpcContext context = RpcContext.getContext();

        // 透传 TraceId
        String traceId = context.getAttachment("X-TraceId");
        if (traceId == null) {
            traceId = TraceContext.current().getTraceId();
            context.setAttachment("X-TraceId", traceId);
        }

        return invoker.invoke(invocation);
    }
}
```

### 实践二：合理设置采样率

```yaml
# 高 QPS 服务设置采样率
dubbo:
  tracing:
    sampling:
      rate: 0.1    # 10% 采样
      # 10000 QPS → 1000 条链路
```

### 实践三：添加业务标签

```java
// 添加业务标签
Span span = tracer.buildSpan("createOrder")
        .withTag("order.id", orderId)
        .withTag("user.id", userId)
        .withTag("amount", amount.toString())
        .start();
```

## 总结

链路追踪是分布式系统可观测性的三大支柱之一（还有 Metrics 和 Logging）。通过链路追踪，可以清晰地看到请求在系统中的完整流转路径，快速定位性能瓶颈。

**核心要点回顾：**

- TraceId 追踪一次完整请求，SpanId 追踪一次 RPC 调用
- 常见链路追踪工具：SkyWalking、Zipkin、Jaeger、OpenTelemetry
- Dubbo 通过 Filter 机制集成链路追踪
- 链路数据通过 Collector 收集，存储到 ES/Cassandra 等
- 合理设置采样率，高 QPS 服务不要全量采集
