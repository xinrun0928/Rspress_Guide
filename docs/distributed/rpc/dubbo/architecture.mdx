# Dubbo 整体架构

Dubbo 这个名字来自一个动画角色——它是一个负责「连接」的重要角色。而 Dubbo 框架的核心使命，也正是如此：**在分布式系统中，建立起高效、可靠的服务连接。**

Dubbo 是阿里巴巴开源的高性能 RPC 框架，目前已经是 Apache 顶级项目。它在国内企业中的使用非常广泛，尤其是在中大型互联网公司中，几乎是微服务架构的标配。

## Dubbo 的演进历史

在深入架构之前，有必要了解一下 Dubbo 的演进历史，这有助于理解它的设计哲学。

```yaml
# Dubbo 演进历程
版本          时间          特点
─────────────────────────────────────────────────────────────
Dubbo 2.x     2011-2017    阿里巴巴内部使用后开源，奠定 RPC 基础
Service Mesh   2017-2018    社区沉寂，Istio/Envoy 崛起
Dubbo 2.7.x   2018-2019    重启维护，新增注解、Zookeeper 优化
Dubbo 3.x     2021 至今    云原生优先，应用级服务发现，Proxyless Mesh
```

Dubbo 3.x 提出了「应用级服务发现」的概念，相比 2.x 的「接口级服务发现」，在服务治理规模和性能上都有质的飞跃。

### 核心层职责

```yaml
# 各层职责说明
层级            组件            职责
─────────────────────────────────────────────────────────────
业务层          Service         用户定义的业务接口实现
配置层          Config          解析 XML、注解、API 配置
代理层          Proxy           生成代理类，屏蔽 RPC 调用细节
注册层          Registry        服务注册与发现
集群层          Cluster         负载均衡、路由、容错
监控层          Monitor         统计服务调用次数、耗时等
协议层          Protocol        封装 RPC 调用，定义协议
信息交换层      Exchange        请求响应模型封装
传输层          Transport       NIO 框架：Netty、Mina
序列化层        Serialize       对象序列化：Hessian、JSON、Protobuf
```

## 核心组件详解

### 1. Proxy 层

Proxy 层是 Dubbo 实现「透明 RPC 调用」的关键。它通过 Java 动态代理或 Javassist，生成一个实现了两大核心接口 `Invoker` 和 `Exporter` 的代理类，让调用方以为自己是在调用本地方法。

```java
// 代理层核心接口
public interface Invoker<T> extends Node {
    // 获取服务接口
    Class<T> getInterface();
    // 执行调用
    Result invoke(Invocation invocation) throws RpcException;
}

// 消费者端代理创建过程
public class ReferenceConfig<T> {
    private T createProxy(URL url) {
        // 通过 ProxyFactory 创建代理
        // 默认使用 JavassistProxyFactory
        ProxyFactory proxyFactory = ExtensionLoader
                .getExtensionLoader(ProxyFactory.class)
                .getAdaptiveExtension();

        // 创建 Invoker
        Invoker<T> invoker = new DubboInvoker<T>(serviceInterface, ...);

        // 创建代理
        T proxy = proxyFactory.getProxy(invoker);
        return proxy;
    }
}
```

### 2. Cluster 层

Cluster 层负责处理多个服务提供者的情况，提供负载均衡、路由、容错等能力。

```java
// Cluster 层核心接口
public interface Cluster {
    // 将多个 Invoker 聚合成一个 Cluster Invoker
    <T> Invoker<T> join(Directory<T> directory) throws RpcException;
}

// Cluster Invoker 执行流程
public class AbstractClusterInvoker<T> {
    public Result invoke(Invocation invocation) throws RpcException {
        // 1. 获取所有可用 Invoker
        List<Invoker<T>> invokers = list(invocation);

        // 2. 执行负载均衡
        Invoker<T> selected = loadbalance.select(invokers);

        // 3. 执行调用
        return doInvoke(invocation, selected);
    }
}
```

### 3. Protocol 层

Protocol 层是 RPC 的核心，它定义了 RPC 调用的协议格式和网络通信方式。

```java
// Protocol 核心接口
@SPI("dubbo")
public interface Protocol {
    // 导出服务（服务端）
    <T> Exporter<T> export(Invoker<T> invoker) throws RpcException;

    // 引用服务（客户端）
    <T> Invoker<T> refer(Class<T> type, URL url) throws RpcException;
}

// Dubbo Protocol 执行流程
public class DubboProtocol {
    // 服务导出
    public <T> Exporter<T> export(Invoker<T> invoker) {
        // 1. 打开 Server
        openServer(url);

        // 2. 优化序列化
        optimizeSerialization(url);

        // 3. 创建 Exporter
        return new DubboExporter<T>(invoker, exportKey, exporterMap);
    }

    // 服务引用
    public <T> Invoker<T> refer(Class<T> type, URL url) {
        // 1. 创建 Invoker
        DubboInvoker<T> invoker = new DubboInvoker<T>(type, url, ...);

        // 2. 加载 Invoker
        invokers.add(invoker);

        return invoker;
    }
}
```

### 4. Registry 层

Registry 层负责服务的注册与发现，是 Dubbo 实现服务治理的基础。

```java
// 注册中心核心接口
public interface Registry {
    // 注册服务
    void register(URL url);

    // 订阅服务
    void subscribe(URL url, NotifyListener listener);

    // 取消注册
    void unregister(URL url);

    // 取消订阅
    void unsubscribe(URL url, NotifyListener listener);
}

// 服务注册流程
public class ZookeeperRegistry {
    public void register(URL url) {
        // 在 Zookeeper 中创建临时节点
        // path: /dubbo/com.example.UserService/providers/...
        String path = toCategoryPath(url);
        zkClient.create(path, url.toFullString(), ZookeeperRegistry.EPHEMERAL);
    }

    public void subscribe(URL url, NotifyListener listener) {
        // 订阅 providers、consumers、routers、configurators 节点
        zkClient.subscribe(toCategoryPath(url), listener);
    }
}
```

## 核心配置解析

Dubbo 支持多种配置方式：XML 配置、注解配置、API 配置、properties 配置。

```xml
<!-- XML 配置示例 -->
<beans>
    <!-- 服务提供者配置 -->
    <dubbo:application name="demo-provider"/>

    <dubbo:registry address="zookeeper://127.0.0.1:2181"/>

    <dubbo:protocol name="dubbo" port="20880"/>

    <dubbo:service
        interface="com.example.UserService"
        ref="userService"
        version="1.0.0"
        group="prod"
        timeout="3000"
        retries="2"/>

    <!-- 服务消费者配置 -->
    <dubbo:reference
        interface="com.example.UserService"
        version="1.0.0"
        group="prod"
        timeout="5000"
        check="false"/>
</beans>
```

```java
// 注解配置示例
@Configuration
public class DubboConfig {

    @Bean
    public ApplicationConfig applicationConfig() {
        ApplicationConfig config = new ApplicationConfig();
        config.setName("demo-provider");
        return config;
    }

    @Bean
    public RegistryConfig registryConfig() {
        RegistryConfig config = new RegistryConfig();
        config.setAddress("zookeeper://127.0.0.1:2181");
        return config;
    }

    @Bean
    public ProtocolConfig protocolConfig() {
        ProtocolConfig config = new ProtocolConfig();
        config.setName("dubbo");
        config.setPort(20880);
        return config;
    }
}

// 使用注解定义服务
@Service(version = "1.0.0", group = "prod")
@Component
public class UserServiceImpl implements UserService {

    @Override
    @DubboReference(version = "1.0.0", group = "prod")
    private OrderService orderService;

    public User getUser(Long userId) {
        // 使用 Dubbo 调用远程服务
        Order order = orderService.getOrder(userId);
        // ...
    }
}
```

## 总结

Dubbo 是一个功能完善、性能优秀的高性能 RPC 框架。它的分层架构设计，使得每一层都有清晰的职责，便于扩展和定制。

**核心要点回顾：**

- Dubbo 采用分层架构：Business、RPC、Remoting
- 核心组件包括：Proxy、Cluster、Protocol、Registry
- 服务调用流程：注册→订阅→代理→负载均衡→序列化→网络传输→反序列化→执行
- 支持多种配置方式：XML、注解、API、properties
- Dubbo 3.x 提出了应用级服务发现，更适合云原生场景
