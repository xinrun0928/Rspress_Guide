# 分布式理论

<Badge text="分布式系统" type="tip" /> <Badge text="一致性" type="warning" /> <Badge text="共识算法" type="info" />

凌晨 2 点，某电商平台的库存服务突然出现了一个诡异的问题：用户下单时显示有库存，支付成功后却提示库存不足。运维团队连夜排查，发现是库存数据在多个节点间的同步延迟导致的。这个场景，或许你在生产环境中已经经历过，或许正在经历。

分布式系统带来了横向扩展能力和高可用性，但也引入了一个核心问题：**如何在分布式环境下保证数据的一致性？**

## 为什么需要分布式理论？

单机系统的 ACID 特性在分布式环境下不再天然成立。节点之间的网络通信是不可靠的，消息可能丢失、延迟、乱序、甚至重复。网络分区（Network Partition）随时可能发生，而系统在面对网络分区时的行为，直接决定了它能提供什么样的服务承诺。

分布式理论正是为了回答这个问题而生。它从数学和逻辑的角度，证明了分布式系统在一致性、可用性和分区容错性之间的根本矛盾，指导我们如何在实际场景中做出合理的权衡。

## 理论体系总览

分布式理论体系可以分为几个层次：

**第一层：基础理论**
- CAP 定理告诉我们 C-A-P 三者不可兼得
- BASE 理论提供了在弱一致性场景下的实践指导
- FLP 不可能性原理揭示了异步系统中共识算法的根本限制

**第二层：一致性模型**
- 强一致性：线性一致性、顺序一致性
- 弱一致性：最终一致性、因果一致性
- 不同的一致性模型适用于不同的业务场景

**第三层：核心机制**
- 分布式互斥：确保资源在同一时刻只能被一个节点访问
- 领导者选举：在多个节点中选出一个协调者
- 共识算法：Paxos、Raft、ZAB 等协议解决分布式一致性问题

**第四层：协调协议**
- 两阶段提交（2PC）、三阶段提交（3PC）
- 分布式事务的挑战与应对策略
