# 2PC

<Badge text="分布式系统" type="tip" /> <Badge text="分布式事务" type="warning" /> <Badge text="两阶段提交" type="info" />

想象一个场景：你在电商平台下一笔订单，需要同时扣减库存、扣减账户余额、创建订单这三个操作，分别在三个不同的数据库中。

如何保证这三个操作要么全部成功，要么全部失败？

**两阶段提交（2PC）**就是来解决这个问题的——它通过两个阶段的协调，确保分布式事务的原子性。

## 为什么需要分布式事务？

```java
/**
 * 单机事务 vs 分布式事务
 */

// 单机事务：
// 所有操作在一个数据库中
// BEGIN TRANSACTION
// UPDATE inventory SET count = count - 1 WHERE id = 1;
// UPDATE account SET balance = balance - 100 WHERE id = 2;
// INSERT INTO orders (...) VALUES (...);
// COMMIT;
// 数据库自动保证原子性

// 分布式事务：
// 操作分布在多个数据库中
// DB1: UPDATE inventory SET count = count - 1 WHERE id = 1;
// DB2: UPDATE account SET balance = balance - 100 WHERE id = 2;
// DB3: INSERT INTO orders (...) VALUES (...);
// 
// 问题：如何保证三个操作要么全部成功，要么全部失败？
// 答案：两阶段提交
```

## 两阶段提交的角色

```java
/**
 * 2PC 的角色
 */

// 1. Coordinator（协调者）
// - 发起和管理整个事务
// - 决定事务提交还是回滚
// - 通常是应用服务器

// 2. Participant（参与者）
// - 执行实际的业务操作
// - 响应协调者的指令
// - 通常是数据库服务器

/**
 * 2PC 的两阶段
 */

// 第一阶段：投票阶段（Voting Phase）
// - 协调者询问所有参与者：能否提交？
// - 参与者执行prepare，但不提交
// - 参与者投票：Yes 或 No

// 第二阶段：提交阶段（Commit Phase）
// - 如果所有参与者都投票 Yes：提交
// - 如果有任何参与者投票 No：回滚
```

## 2PC 的详细流程

```java
/**
 * 2PC 详细流程
 */

// 阶段一：投票阶段
// 
// Coordinator                    Participant1  Participant2  Participant3
//     |                               |            |            |
//     |--- CanCommit? --------------> |            |            |
//     |<-- Yes ---------------------- |            |            |
//     |--- CanCommit? ----------------------------> |            |
//     |<-- Yes ---------------------------------- |            |
//     |--- CanCommit? ------------------------------------------------> |
//     |<-- Yes ------------------------------------------------------ |
//     |                               |            |            |
//     v                               |            |            |
//                                   执行 PREPARE，不提交          |
//                                   写 redo log                 |
//     |                               |            |            |
//     v                               |            |            |
// 
// 阶段二：提交阶段（所有参与者都投 Yes）
// 
// Coordinator                    Participant1  Participant2  Participant3
//     |                               |            |            |
//     |--- DoCommit -----------------> |            |            |
//     |                               |            |            |
//     |<-- Ack ---------------------- |            |            |
//     |--- DoCommit ----------------------------> |            |
//     |<-- Ack ---------------------------------- |            |
//     |--- DoCommit ------------------------------------------------> |
//     |<-- Ack ------------------------------------------------------ |
//     |                               |            |            |
//     |                               v            v            v
//     |                            提交事务     提交事务     提交事务
//     |                               |            |            |
//     v                               |            |            |
//                                   完成          完成          完成
```

## 2PC 的代码实现

### 协调者实现

```java
/**
 * 2PC 协调者
 */
public class TwoPhaseCommitCoordinator {
    
    private List<Participant> participants;
    
    /**
     * 提交分布式事务
     */
    public void commit(Transaction transaction) {
        // 阶段一：投票阶段
        boolean allCanCommit = doCommitPhase(transaction);
        
        if (allCanCommit) {
            // 阶段二：提交阶段
            doCommit(transaction);
        } else {
            // 阶段二：回滚阶段
            doRollback(transaction);
        }
    }
    
    /**
     * 第一阶段：收集投票
     */
    private boolean doCommitPhase(Transaction transaction) {
        List<Future<Boolean>> responses = new ArrayList<>();
        
        // 向所有参与者发送 CanCommit 请求
        for (Participant participant : participants) {
            Future<Boolean> response = executor.submit(() -> 
                participant.canCommit(transaction)
            );
            responses.add(response);
        }
        
        // 等待所有响应
        boolean allCanCommit = true;
        for (Future<Boolean> response : responses) {
            try {
                Boolean canCommit = response.get(TIMEOUT, TimeUnit.SECONDS);
                if (!canCommit) {
                    allCanCommit = false;
                    break;
                }
            } catch (TimeoutException e) {
                allCanCommit = false;
                break;
            } catch (ExecutionException e) {
                allCanCommit = false;
                break;
            }
        }
        
        return allCanCommit;
    }
    
    /**
     * 第二阶段：提交
     */
    private void doCommit(Transaction transaction) {
        for (Participant participant : participants) {
            try {
                participant.doCommit(transaction);
            } catch (Exception e) {
                // 记录日志，稍后补偿
                log.error("Commit failed for participant", e);
                scheduleRetry(participant, transaction);
            }
        }
    }
    
    /**
     * 第二阶段：回滚
     */
    private void doRollback(Transaction transaction) {
        for (Participant participant : participants) {
            try {
                participant.doRollback(transaction);
            } catch (Exception e) {
                // 记录日志，稍后补偿
                log.error("Rollback failed for participant", e);
                scheduleRetry(participant, transaction);
            }
        }
    }
}
```

### 参与者实现

```java
/**
 * 2PC 参与者
 */
public class TwoPhaseCommitParticipant {
    
    private Connection connection;
    private String participantId;
    private TransactionLog transactionLog;
    
    /**
     * 响应 CanCommit 请求
     */
    public boolean canCommit(Transaction transaction) {
        try {
            // 1. 检查业务逻辑是否允许提交
            if (!checkBusinessRules(transaction)) {
                return false;
            }
            
            // 2. 获取数据库锁
            acquireLocks(transaction);
            
            // 3. 记录事务状态为 PREPING
            transactionLog.logStatus(transaction.getId(), 
                TransactionStatus.PREPING, participantId);
            
            // 4. 执行 PREPARE（不提交）
            PreparedStatement ps = connection.prepareStatement(
                "PREPARE TRANSACTION ?"
            );
            ps.setString(1, transaction.getId());
            ps.execute();
            
            // 5. 记录事务状态为 PREPARED
            transactionLog.logStatus(transaction.getId(), 
                TransactionStatus.PREPARED, participantId);
            
            return true;
            
        } catch (Exception e) {
            log.error("Prepare failed", e);
            transactionLog.logStatus(transaction.getId(), 
                TransactionStatus.FAILED, participantId);
            return false;
        }
    }
    
    /**
     * 执行提交
     */
    public void doCommit(Transaction transaction) {
        try {
            // 1. 检查事务状态
            TransactionStatus status = 
                transactionLog.getStatus(transaction.getId());
            
            if (status == TransactionStatus.PREPARED) {
                // 2. 执行 COMMIT PREPARED
                PreparedStatement ps = connection.prepareStatement(
                    "COMMIT PREPARED ?"
                );
                ps.setString(1, transaction.getId());
                ps.execute();
                
                // 3. 释放锁
                releaseLocks(transaction);
                
                // 4. 记录事务状态为 COMMITTED
                transactionLog.logStatus(transaction.getId(), 
                    TransactionStatus.COMMITTED, participantId);
            }
            
        } catch (Exception e) {
            log.error("Commit failed", e);
            throw new TransactionException("Commit failed", e);
        }
    }
    
    /**
     * 执行回滚
     */
    public void doRollback(Transaction transaction) {
        try {
            // 1. 检查事务状态
            TransactionStatus status = 
                transactionLog.getStatus(transaction.getId());
            
            if (status == TransactionStatus.PREPARED) {
                // 2. 执行 ROLLBACK PREPARED
                PreparedStatement ps = connection.prepareStatement(
                    "ROLLBACK PREPARED ?"
                );
                ps.setString(1, transaction.getId());
                ps.execute();
            } else {
                // 3. 还没 prepare，直接回滚本地事务
                connection.rollback();
            }
            
            // 4. 释放锁
            releaseLocks(transaction);
            
            // 5. 记录事务状态为 ROLLED_BACK
            transactionLog.logStatus(transaction.getId(), 
                TransactionStatus.ROLLED_BACK, participantId);
            
        } catch (Exception e) {
            log.error("Rollback failed", e);
            throw new TransactionException("Rollback failed", e);
        }
    }
}
```

## 2PC 的问题

### 1. 协调者单点故障

```java
/**
 * 2PC 的单点故障问题
 */

// 场景：协调者在发送 DoCommit 前崩溃
// 
// Coordinator                    Participant
//     |                               |
//     |<-- Yes ---------------------- |
//     |                               |
//     |          [Coordinator 崩溃]    |
//     |                               |
//     X                               v
//                                   等待 DoCommit...
//                                   永远等待！
// 
// 参与者不知道该提交还是该回滚
// 事务处于不确定状态（in-doubt state）
```

### 2. 阻塞问题

```java
/**
 * 2PC 的阻塞问题
 */

// 参与者在投票后必须等待协调者的最终指令
// 在此期间，参与者持有的锁不能释放
// 
// 问题：
// 1. 协调者崩溃，参与者永久阻塞
// 2. 网络分区，参与者永久阻塞
// 3. 其他事务无法获取相同的锁

/**
 * 解决方案：超时机制
 */
public class ParticipantWithTimeout {
    
    private volatile TransactionStatus status;
    
    /**
     * 投票后的超时处理
     */
    public void onVoteTimeout() {
        switch (status) {
            case PREPARED:
                // 投票 Yes 后等待，可以安全提交
                // 因为其他参与者也投票了多数派
                // 决定：提交
                selfCommit();
                break;
                
            case VOTING:
                // 还没投票，可以安全回滚
                selfRollback();
                break;
        }
    }
    
    /**
     * 自主决定提交
     */
    private void selfCommit() {
        // 需要查看事务日志来决定
        TransactionLog log = transactionLog.get(transaction.getId());
        if (log != null && log.getDecision() == Decision.COMMIT) {
            doCommit(transaction);
        } else {
            doRollback(transaction);
        }
    }
}
```

### 3. 数据不一致风险

```java

 * 2PC 的数据不一致风险
 */

//     |--- DoCommit -----------------> |            |
//     |<-- Ack ---------------------- |            |
//     |                               v            |
//     |                            提交            |
//     X                               |            |
//                                      |            |
//                                      |   [没收到，阻塞等待]
//                                      v
//                                     永久等待！
// 
// 解决方案：
// 1. 协调者重试发送 DoCommit
// 2. 参与者超时后查询协调者状态
```

## 2PC 的优化

### 瞻前顾后策略

```java
/**
 * 瞻前顾后（Presumed Abort / Presumed Commit）
 */

// Presumed Abort：
// - 如果协调者在发送 DoAbort 前崩溃，参与者可以单方面回滚
// - 减少协调者的日志开销

// Presumed Commit：
// - 如果协调者在发送 DoCommit 前崩溃，参与者可以单方面提交
// - 减少协调者的日志开销

/**
 * Presumed Commit 实现
 */
public class PresumedCommitCoordinator {
    
    /**
     * 提交事务
     */
    public void commit(Transaction transaction) {
        // 阶段一：收集投票
        if (collectVotes(transaction)) {
            // 阶段二：发送提交
            for (Participant p : participants) {
                p.doCommit(transaction);
            }
            
            // 注意：提交成功后，不写日志
            // 如果协调者此时崩溃，参与者可以单方面提交
        } else {
            // 回滚
            rollback(transaction);
            // 同样，不写日志
        }
    }
}
```

## 总结

2PC 是分布式事务的基础协议：

1. **两阶段**：投票阶段 + 提交/回滚阶段
2. **原子性**：通过协调者统一决策保证
3. **阻塞**：参与者在投票后必须等待
4. **单点**：协调者是潜在的故障点

虽然 2PC 有其局限性，但它仍然是许多分布式事务方案的基础。
