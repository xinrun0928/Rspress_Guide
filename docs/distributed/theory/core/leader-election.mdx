# 分布式选举算法

<Badge text="分布式系统" type="tip" /> <Badge text="领导者选举" type="warning" /> <Badge text="核心算法" type="info" />

你有没有想过：当 Zookeeper 集群中主节点宕机后，是谁决定哪个 Follower 成为新的主节点？这个「选举」过程是怎么进行的？

在分布式系统中，领导者选举是一个核心问题。许多分布式协议（如 Raft、ZAB）都需要一个领导者来协调操作。**领导者选举算法决定了：当集群中没有领导者时，谁能成为新的领导者。**

## 为什么需要领导者选举？

在分布式系统中，领导者（Leader）扮演着「协调者」的角色：

```java
// 没有领导者时的混乱
// 场景：三个节点都需要写入数据
Node node1 = new Node();
Node node2 = new Node();
Node node3 = new Node();

// 问题：谁来决定写入顺序？
// 如果同时写入，可能导致数据不一致
node1.write("key", "value1");  // 同时
node2.write("key", "value2");  // 同时
node3.write("key", "value3");  // 同时

// 最终数据是什么？不确定！
```

```java
// 有领导者时的有序
// 节点 2 成为领导者
Leader leader = node2;
Follower follower1 = node1;
Follower follower3 = node3;

// 所有写入都经过领导者
leader.write("key", "value1");  // 领导者写入
follower1.replicateFrom(leader);  // 复制
follower3.replicateFrom(leader);  // 复制

// 数据一致性得到保证
```

## 领导者选举的要求

```java
/**
 * 领导者选举的基本要求
 */
public class LeaderElectionRequirements {
    
    /**
     * 1. 领导者唯一性（Leader Uniqueness）
     * 任何时刻，系统中最多只有一个领导者
     */
    boolean leaderUniqueness();
    
    /**
     * 2. 活跃性（Liveness）
     * 如果没有领导者，最终会选出一个领导者
     */
    boolean liveness();
    
    /**
     * 3. 合法性（Legitimacy）
     * 选出的领导者必须是集群中的有效成员
     */
    boolean legitimacy();
    
    /**
     * 4. 公平性（Fairness）
     * 每个节点都有机会成为领导者
     * 不会某个节点总是成为领导者
     */
    boolean fairness();
}
```

## Bully 算法

Bully 算法是最简单的领导者选举算法之一。它的核心思想是：**ID 最大的节点获胜。**

```java
public class BullyAlgorithm {
    
    private final int nodeId;
    private final List<Integer> allNodeIds;
    private volatile int coordinator = -1;
    
    public BullyAlgorithm(int nodeId, List<Integer> allNodeIds) {
        this.nodeId = nodeId;
        this.allNodeIds = allNodeIds;
    }
    
    /**
     * 发起选举
     */
    public void startElection() {
        System.out.println("节点 " + nodeId + " 发起选举");
        
        // 1. 向所有 ID 更大的节点发送选举消息
        List<Integer> higherNodes = allNodeIds.stream()
            .filter(id -> id > nodeId)
            .sorted()
            .collect(Collectors.toList());
        
        if (higherNodes.isEmpty()) {
            // 没有更高 ID 的节点，自己成为领导者
            becomeLeader();
        } else {
            // 等待更高 ID 节点的回复
            boolean receivedAnswer = false;
            for (Integer higherId : higherNodes) {
                try {
                    sendMessage(higherId, MessageType.ELECTION);
                    receivedAnswer = true;
                } catch (Exception e) {
                    // 节点无响应，继续尝试下一个
                }
            }
            
            if (!receivedAnswer) {
                // 所有更高 ID 的节点都无响应，成为领导者
                becomeLeader();
            }
        }
    }
    
    /**
     * 处理选举消息
     */
    public void onElectionMessage(int fromNodeId) {
        if (fromNodeId > nodeId) {
            // 有更高 ID 的节点存在，回复 OK
            sendMessage(fromNodeId, MessageType.OK);
        } else {
            // 自己是最高 ID，等待更高 ID 的节点失败后成为领导者
        }
    }
    
    /**
     * 成为领导者
     */
    private void becomeLeader() {
        coordinator = nodeId;
        System.out.println("节点 " + nodeId + " 成为领导者");
        
        // 向所有节点发送领导者声明
        for (Integer nodeId : allNodeIds) {
            if (nodeId != this.nodeId) {
                sendMessage(nodeId, MessageType.COORDINATOR);
            }
        }
    }
}
```

### Bully 算法的优缺点

```java
// 优点
// 1. 实现简单
// 2. 容易理解

// 缺点
// 1. 消息开销大：最坏情况需要 O(N²) 条消息
// 2. 不公平：ID 大的节点总是优先
// 3. 对故障敏感：最高 ID 节点频繁故障会导致频繁选举
```

## Raft 中的领导者选举

Raft 算法使用一种改进的领导者选举机制：

```java
public class RaftLeaderElection {
    
    // 节点状态
    public enum State { FOLLOWER, CANDIDATE, LEADER }
    
    // 持久化状态（需要在故障后恢复）
    private long currentTerm = 0;      // 当前任期
    private Integer votedFor = null;  // 当前任期投票给了谁
    private List<LogEntry> log = new ArrayList<>();  // 日志
    
    // -volatile 状态
    private State state = State.FOLLOWER;
    private Integer leaderId = null;
    
    // 配置参数
    private static final long ELECTION_TIMEOUT_MIN = 150;  // ms
    private static final long ELECTION_TIMEOUT_MAX = 300;  // ms
    
    private final Random random = new Random();
    private volatile long lastHeartbeat = 0;
    
    /**
     * 开始选举
     */
    public void startElection() {
        state = State.CANDIDATE;
        currentTerm++;  // 任期号增加
        
        // 给自己投票
        votedFor = nodeId;
        
        // 向所有其他节点发送投票请求
        int votesNeeded = (clusterSize / 2) + 1;
        AtomicInteger votesReceived = new AtomicInteger(1);  // 给自己投了一票
        
        for (Integer peerId : peers) {
            if (canVoteFor(peerId)) {
                sendRequestVote(peerId, currentTerm, 
                    getLastLogIndex(), getLastLogTerm());
            }
        }
        
        // 等待投票结果
        awaitVoteResults(votesNeeded, votesReceived);
    }
    
    /**
     * 处理投票请求
     */
    public VoteResponse onRequestVote(int candidateId, long term, 
                                      long lastLogIndex, long lastLogTerm) {
        // 1. 如果任期号更小，拒绝投票
        if (term < currentTerm) {
            return new VoteResponse(currentTerm, false);
        }
        
        // 2. 如果当前节点没有投票，或者投给了候选者
        // 并且候选者的日志至少和自己一样新
        if ((votedFor == null || votedFor == candidateId) 
            && isLogUpToDate(lastLogIndex, lastLogTerm)) {
            votedFor = candidateId;
            return new VoteResponse(currentTerm, true);
        }
        
        return new VoteResponse(currentTerm, false);
    }
    
    /**
     * 判断候选者的日志是否比自己的日志更新
     */
    private boolean isLogUpToDate(long lastLogIndex, long lastLogTerm) {
        long myLastLogTerm = log.isEmpty() ? 0 : log.get(log.size() - 1).term;
        long myLastLogIndex = log.size();
        
        // 如果最后一条日志的任期号不同，term 更大的更新
        // 如果任期号相同，index 更大的更新
        return lastLogTerm > myLastLogTerm 
            || (lastLogTerm == myLastLogTerm && lastLogIndex >= myLastLogIndex);
    }
}
```

### Raft 选举的关键机制

```java
/**
 * Raft 选举安全的三个保证
 */

// 1. 选举超时（Election Timeout）
// Follower 在超时时间内没有收到心跳，就会发起选举
// 随机化超时时间避免多个节点同时发起选举
long electionTimeout = ELECTION_TIMEOUT_MIN 
    + random.nextInt(ELECTION_TIMEOUT_MAX - ELECTION_TIMEOUT_MIN);

// 2. 多数派投票
// 只有获得多数派投票的候选者才能成为领导者
// 保证新领导者包含所有已提交的日志
int majority = (clusterSize / 2) + 1;

// 3. 任期号递增
// 每次选举都会递增任期号
// 如果发现更大的任期号，更新自己的任期号并转为 Follower
if (response.term > currentTerm) {
    currentTerm = response.term;
    state = State.FOLLOWER;
}
```

## Zookeeper 中的领导者选举

Zookeeper 使用 ZAB（Zookeeper Atomic Broadcast）协议进行领导者选举：

```java
public class ZABLeaderElection {
    
    // ZAB 选举状态
    public enum ElectionState { LOOKING, FOLLOWING, LEADING }
    
    // 投票结构
    static class Vote {
        long zxid;      // 事务 ID
        long epoch;     // 轮次（election epoch）
        int serverId;   // 服务器 ID
        
        public Vote(long zxid, long epoch, int serverId) {
            this.zxid = zxid;
            this.epoch = epoch;
            this.serverId = serverId;
        }
        
        // 比较规则：epoch > zxid > serverId
        public boolean isGreaterThan(Vote other) {
            if (this.epoch != other.epoch) {
                return this.epoch > other.epoch;
            }
            if (this.zxid != other.zxid) {
                return this.zxid > other.zxid;
            }
            return this.serverId > other.serverId;
        }
    }
    
    private volatile Vote currentVote;
    private volatile ElectionState electionState = ElectionState.LOOKING;
    
    /**
     * FastLeaderElection 核心逻辑
     */
    public void lookForLeader() {
        electionState = ElectionState.LOOKING;
        currentVote = new Vote(getServerId(), getEpoch(), getServerId());
        
        // 广播投票
        broadcastVote(currentVote);
        
        // 收集投票，直到选出领导者
        while (electionState == ElectionState.LOOKING) {
            // 等待其他节点的投票
            Vote receivedVote = waitForVote();
            
            // 比较投票
            if (receivedVote.isGreaterThan(currentVote)) {
                currentVote = receivedVote;
                broadcastVote(currentVote);  // 广播更优的投票
            }
            
            // 检查是否已经选出领导者
            if (isElectionFinished()) {
                ElectionOutcome outcome = getElectionOutcome();
                if (outcome.hasLeader()) {
                    electionState = outcome.isMeLeader() 
                        ? ElectionState.LEADING 
                        : ElectionState.FOLLOWING;
                }
            }
        }
    }
}
```

## 选举算法对比

| 算法 | 时间复杂度 | 消息复杂度 | 特点 |
|---|---|---|---|
| Bully | O(N) | O(N²) | 简单，但不公平 |
| Raft | O(N) | O(N) | 工程实现广泛 |
| ZAB | O(N) | O(N) | Zookeeper 使用 |
| Paxos | O(N) | O(N) | 理论优雅 |

## 总结

领导者选举是分布式系统中的核心问题之一。不同的选举算法有不同的特点：

- **Bully 算法**：最简单，但不适用于大规模系统
- **Raft 选举**：工程实现广泛，通过随机超时保证安全性
- **ZAB 选举**：Zookeeper 使用，通过事务 ID 和轮次保证选举的正确性

理解领导者选举的原理，对于理解 Raft、Zookeeper 等分布式系统至关重要。
