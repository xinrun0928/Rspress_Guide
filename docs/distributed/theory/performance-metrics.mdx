# 分布式性能指标

<Badge text="分布式系统" type="tip" /> <Badge text="性能指标" type="warning" /> <Badge text="度量体系" type="info" />

如何衡量一个分布式系统的性能？你可能会说「快不快」「稳不稳」。但这些模糊的描述无法指导我们优化系统。

在分布式系统中，性能指标是一套完整的度量体系。理解这些指标，才能真正把握系统的性能状况。

## 核心性能指标

### 延迟（Latency）

```java
/**
 * 延迟的定义
 */

// 延迟是请求从发起到收到响应的时间
// 通常用 P50、P90、P99、P999 等分位数表示

/**
 * 延迟的分位数
 */

// P50（中位数）：50% 的请求延迟低于此值
// P90：90% 的请求延迟低于此值
// P99：99% 的请求延迟低于此值
// P999：99.9% 的请求延迟低于此值

/**
 * 为什么用分位数？
 */

// 平均延迟容易受极端值影响
// 分位数更能反映用户体验

// 假设有 1000 个请求：
// - 999 个请求延迟 10ms
// - 1 个请求延迟 10s

// 平均延迟：约 20ms（看起来不错）
// P999：10s（实际有 0.1% 的用户很慢）

/**
 * 延迟的可视化
 */

// 直方图示例：
// 
// 请求数
// ^
// |                    *
// |                *   *
// |            *   *   *
// |        *   *   *   *
// |    *   *   *   *   *
// |*   *   *   *   *   *
// +-------------------------> 延迟 (ms)
// 1    5   10  50  100  500
//      ^P50  ^P90    ^P99
```

### 吞吐量（Throughput）

```java
/**
 * 吞吐量的定义
 */

// 吞吐量是单位时间内系统处理的请求数量
// 常用单位：QPS、TPS、RPS

/**
 * 相关指标
 */

// QPS（Queries Per Second）
// 每秒查询数

// TPS（Transactions Per Second）
// 每秒事务数

// RPS（Requests Per Second）
// 每秒请求数

/**
 * 吞吐量 vs 延迟
 */

// 高吞吐量 ≠ 低延迟
// 系统可能高吞吐但延迟高
// 例如：批量处理系统

// 低延迟 ≠ 高吞吐
// 系统可能低延迟但吞吐低
// 例如：资源受限的小系统

/**
 * 饱和度（Saturation）
 */

// 系统达到最大吞吐量时的状态
// 常用指标：
// - CPU 使用率
// - 内存使用率
// - 线程池使用率
// - 连接池使用率

/**
 * 吞吐量拐点
 */

// 吞吐量曲线：
// 
//        吞吐量
//           ^
//           |      ____
//           |    /
//           |   /
//           |  /
//           | /
//           |/
//           +----------------------> 负载
//                  ^拐点
// 
// 拐点前：吞吐量随负载线性增长
// 拐点后：吞吐量下降，延迟上升
```

### 可用性（Availability）

```java
/**
 * 可用性的定义
 */

// 可用性是系统正常运行的概率
// 常用百分比表示

/**
 * 可用性级别
 */

// 99%：每年宕机约 3.65 天
// 99.9%：每年宕机约 8.76 小时
// 99.99%：每年宕机约 52.6 分钟
// 99.999%：每年宕机约 5.26 分钟

/**
 * 可用性计算
 */

// Availability = MTBF / (MTBF + MTTR)

// MTBF（Mean Time Between Failures）
// 平均故障间隔时间

// MTTR（Mean Time To Recovery）
// 平均恢复时间

/**
 * 可用性 vs 可靠性
 */

// 可用性：系统随时可用的概率
// 可靠性：系统长时间无故障运行的能力

// 高可用 ≠ 高可靠
// 例如：一个经常故障但恢复快的系统可用性高
// 但可靠性不一定高
```

## 分布式系统特有指标

### CAP 指标

```java
/**
 * 一致性指标
 */

// 一致性级别
// - 强一致性：每次读取都是最新写入
// - 最终一致性：最终会达到一致
// - 读后写一致性：能读到自己的写入

/**
 * 一致性延迟
 */

// 数据同步到多数派节点的时间
// 例如：
// - Raft 写操作需要多数派确认
// - 确认时间就是一致性延迟

/**
 * 数据同步延迟
 */

// 副本间数据同步的延迟
// 常用指标：
// - 主从复制延迟
// - 跨机房同步延迟
```

/**
 * 事务性能指标

// 事务吞吐量

// 事务延迟
// 从开始到提交/回滚的时间

// 事务成功率
// 成功提交的事务占总事务的比例

// 分布式事务占比

/**

### 服务间通信指标

```java
/**
 * RPC 指标
 */

// 调用延迟
// - P50、P90、P99 延迟
// - 按服务维度的延迟分布

// 调用成功率
// - 总成功数 / 总调用数

// 调用重试率
// - 重试调用数 / 总调用数

/**
 * 负载均衡指标
 */

// 节点分布
// 各节点请求量是否均衡

// 最小连接数
// 当前连接数最少的节点

// 故障节点数
// 被摘除的故障节点数量
```

## 监控指标体系

### USE 方法

```java
/**
 * USE 方法（Utilization-Saturation-Errors）
 */

// 适用于资源指标监控

/**
 * Utilization（利用率）
 */

// 资源忙于工作的时间百分比
// 例如：CPU 利用率 80%

/**
 * Saturation（饱和度）
 */

// 资源排队等待工作的程度
// 例如：CPU 运行队列长度

/**
 * Errors（错误）
 */

// 错误发生的频率
// 例如：网络丢包率
```

### RED 方法

```java
/**
 * RED 方法（Rate-Errors-Duration）
 */

// 适用于服务指标监控

/**
 * Rate（速率）
 */

// 每秒请求数
// QPS、RPS

/**
 * Errors（错误）
 */

// 每秒错误数
// 按错误类型分类

/**
 * Duration（延迟）
 */

// 请求处理时间
// P50、P90、P99 延迟
```

### Google Four Golden Signals

```java
/**
 * 四个黄金信号
 */

// 1. Latency（延迟）
// 延迟分布，关注慢请求

// 2. Traffic（流量）
// 系统负载，QPS、并发数

// 3. Errors（错误）
// 错误率，关注异常

// 4. Saturation（饱和度）
// 资源使用率，关注瓶颈
```

## 实际监控实现

### 埋点方案

```java
/**
 * 端点埋点
 */

// 在请求入口和出口埋点
public class EndpointInterceptor {
    
    @Autowired
    private MetricsCollector collector;
    
    public Object preHandle(Request request) {
        // 记录开始时间
        long start = System.currentTimeMillis();
        request.setAttribute("start", start);
        
        // 记录请求信息
        collector.recordRequest(
            request.getService(),
            request.getMethod(),
            System.currentTimeMillis()
        );
    }
    
    public void postHandle(Request request, Response response) {
        long start = (Long) request.getAttribute("start");
        long duration = System.currentTimeMillis() - start;
        
        // 记录延迟
        collector.recordLatency(
            request.getService(),
            duration
        );
        
        // 记录结果
        collector.recordResult(
            request.getService(),
            response.isSuccess() ? "success" : "error"
        );
    }
}
```

### 指标采集

```java
/**
 * Prometheus 指标采集
 */

// 1. Counter（计数器）
// 只增不减的指标
public class RequestCounter {
    private Counter counter;
    
    public void increment() {
        counter.inc();
    }
}

// 2. Gauge（仪表盘）
// 可以任意变化的指标
public class ThreadPoolMonitor {
    private Gauge gauge;
    
    public void record(int activeCount) {
        gauge.set(activeCount);
    }
}

// 3. Histogram（直方图）
// 延迟分布
public class LatencyRecorder {
    private Histogram histogram;
    
    public void observe(long duration) {
        histogram.observe(duration);
    }
}

// 4. Summary（摘要）
// 计算百分位数
public class LatencySummary {
    private Summary summary;
    
    public void observe(long duration) {
        summary.observe(duration);
    }
}
```

## 总结

分布式系统的性能指标是一套完整的体系：

1. **基础指标**：延迟、吞吐量、可用性
2. **分布式特有**：CAP 指标、事务指标、通信指标
3. **监控方法**：USE、RED、Four Golden Signals
4. **优化方向**：分析瓶颈、针对性优化

建立完善的监控体系，持续关注这些指标，才能保证分布式系统的稳定运行。
