# Redisson 分布式锁

你花了一周时间，手写了一个 Redis 分布式锁。自测通过，上线了。三天后，凌晨三点，监控系统报警——两台机器同时扣减了同一笔订单的库存。你翻开代码，逐行审查，终于发现了那个隐藏的 bug：锁续期没做，主从切换时丢锁，释放时还可能误删他人的锁。

你开始怀疑人生：就一个分布式锁，怎么这么多坑？有没有现成的轮子可以用？

有。Redisson。

## Redisson 是什么

Redisson 是一个在 Redis 基础上实现的 Java 驻内存数据网格（In-Memory Data Grid）。它不仅提供了 Redis 客户端的基本功能，还封装了一系列分布式对象和服务，包括分布式锁、分布式集合、分布式队列等。

```xml
<dependency>
    <groupId>org.redisson</groupId>
    <artifactId>redisson-spring-boot-starter</artifactId>
    <version>3.25.0</version>
</dependency>
```

```yaml
# application.yml
spring:
  data:
    redis:
      host: localhost
      port: 6379
```

```java
// 配置
@Configuration
public class RedissonConfig {

    @Bean
    public RedissonClient redissonClient() {
        Config config = new Config();
        config.useSingleServer()
              .setAddress("redis://localhost:6379")
              .setConnectionPoolSize(64)
              .setConnectionMinimumIdleSize(24);
        return Redisson.create(config);
    }
}
```

## 基本使用

Redisson 分布式锁的使用非常简单，API 设计几乎与 Java 标准库的 `ReentrantLock` 一致：

```java
@Service
public class OrderService {

    @Autowired
    private RedissonClient redissonClient;

    public void createOrder(String productId, int quantity) {
        // 获取锁
        RLock lock = redissonClient.getLock("order:lock:" + productId);

        try {
            // 加锁（阻塞等待，默认 30 秒过期）
            lock.lock();

            // 业务逻辑
            checkInventory(productId, quantity);
            deductInventory(productId, quantity);
            saveOrder(productId, quantity);

        } finally {
            // 释放锁
            lock.unlock();
        }
    }

    // 非阻塞获取锁
    public boolean tryCreateOrder(String productId, int quantity) {
        RLock lock = redissonClient.getLock("order:lock:" + productId);

        // 尝试获取锁，等待 0 秒，最多锁定 10 秒
        boolean acquired = lock.tryLock(0, 10, TimeUnit.SECONDS);
        if (!acquired) {
            return false; // 获取锁失败
        }

        try {
            checkInventory(productId, quantity);
            deductInventory(productId, quantity);
            saveOrder(productId, quantity);
            return true;
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}
```

## 核心特性

### Watch Dog 自动续期

这是 Redisson 区别于原生 Redis 锁的核心特性之一。

当你调用 `lock()` 不带参数时，锁的默认过期时间是 30 秒。但 Redisson 内部有一个 **Watch Dog** 机制：如果持有锁的线程还没释放锁，Watch Dog 会每 10 秒自动将锁的过期时间延长到 30 秒。

```java
// 源码简化版
// lock() 内部调用
internalLockLeaseTime = 30_000L; // 默认 30 秒
scheduleExpirationRenewal(threadId); // 启动 Watch Dog

// Watch Dog 任务：每 10 秒执行一次
private void scheduleExpirationRenewal(long threadId) {
    renewalTask.schedule(new Runnable() {
        @Override
        public void run() {
            // 续期到 30 秒
            renewExpiration();
        }
    }, internalLockLeaseTime / 3, TimeUnit.MILLISECONDS); // 10 秒
}
```

:::tip
Watch Dog 只在**没有指定过期时间**或**使用 `lock()` 无参数方法**时生效。如果你手动指定了过期时间，Redisson 认为你知道自己在做什么，不会自动续期。
:::

### 多种锁类型

Redisson 提供了多种类型的分布式锁：

| 锁类型 | 说明 | 适用场景 |
|---|---|---|
| `RLock` | 可重入锁，最常用 | 大部分场景 |
| `RFairLock` | 公平锁，按请求顺序获取 | 需要保证顺序的场景 |
| `RReadWriteLock` | 读写锁 | 读多写少场景 |
| `RSemaphore` | 信号量 | 限流、资源池控制 |
| `RMultLock` | 联锁，多把锁同时持有 | 跨多个资源的事务 |

### 可重入机制

同一个线程可以多次获取同一把锁：

```java
RLock lock = redissonClient.getLock("resource");

lock.lock();
try {
    doSomething();        // 第一次获取

    lock.lock();
    try {
        doOtherThing();   // 第二次获取，可重入
    } finally {
        lock.unlock();
    }
} finally {
    lock.unlock();        // 必须调用两次 unlock
}
```

Redisson 通过 `ThreadLocal` 记录每个线程的持有计数：

```java
// 简化实现
private final ThreadLocal<Map<String, Integer>> threadCache = new ThreadLocal<>();

public void lock() {
    String key = getLockKey();
    Map<String, Integer> locks = threadCache.get();
    if (locks == null) {
        locks = new HashMap<>();
        threadCache.set(locks);
    }

    Integer count = locks.get(key);
    if (count == null) {
        // 第一次获取，真正执行加锁
        acquireLock(key);
        locks.put(key, 1);
    } else {
        // 重入，增加计数
        locks.put(key, count + 1);
    }
}
```

## 完整示例

一个完整的分布式锁使用示例，包含异常处理和最佳实践：

```java
@Service
public class InventoryService {

    private static final Logger log = LoggerFactory.getLogger(InventoryService.class);

    @Autowired
    private RedissonClient redissonClient;

    @Autowired
    private InventoryMapper inventoryMapper;

    /**
     * 扣减库存（标准用法）
     */
    public void deductStock(String productId, int quantity) {
        RLock lock = redissonClient.getLock("inventory:lock:" + productId);

        // 获取锁，设置最大等待时间为 0（立即返回），锁定时间 30 秒
        boolean locked = false;
        try {
            locked = lock.tryLock(0, 30, TimeUnit.SECONDS);
            if (!locked) {
                throw new BusinessException("系统繁忙，请稍后重试");
            }

            // 执行业务逻辑
            Inventory inventory = inventoryMapper.selectByProductId(productId);
            if (inventory.getStock() < quantity) {
                throw new BusinessException("库存不足");
            }

            inventory.setStock(inventory.getStock() - quantity);
            inventoryMapper.update(inventory);

            log.info("库存扣减成功: productId={}, quantity={}", productId, quantity);

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new BusinessException("系统异常，请稍后重试");
        } finally {
            // 必须在 finally 中释放锁
            if (locked && lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }

    /**
     * 批量扣减库存（使用红锁）
     */
    public void batchDeductStock(List<String> productIds, Map<String, Integer> quantities) {
        // 红锁需要多个 RedissonClient 实例
        String[] nodes = {"redis://localhost:6379", "redis://localhost:6378", "redis://localhost:6377"};
        RedissonRedLock redLock = new RedissonRedLock(
            Arrays.stream(nodes)
                  .map(redisUrl -> {
                      Config config = new Config();
                      config.useSingleServer().setAddress(redisUrl);
                      return Redisson.create(config);
                  })
                  .toArray(RedissonClient[]::new)
        );

        try {
            // 同时锁定所有产品
            redLock.lock(30, TimeUnit.SECONDS);

            for (String productId : productIds) {
                deductStock(productId, quantities.get(productId));
            }

        } finally {
            redLock.unlock();
        }
    }
}
```

## 常见问题

### Q：锁过期了业务还没执行完怎么办？

A：使用 `lock()` 无参数方法或指定较长的锁定时间。Redisson 的 Watch Dog 会自动续期。

```java
// 方式一：使用无参数 lock()，Watch Dog 自动续期
lock.lock();

// 方式二：指定锁定时间，不自动续期，需要业务预估时间
lock.lock(30, TimeUnit.SECONDS);
```

### Q：为什么 unlock 前要判断 isHeldByCurrentThread？

A：锁可能因为过期自动释放后再调用 unlock，导致报错或误删他人锁。

```java
// 推荐写法
if (lock.isHeldByCurrentThread()) {
    lock.unlock();
}
```

### Q：RedissonClient 是线程安全的吗？

A：是的，RedissonClient 是线程安全的，可以注入到单例 Bean 中供所有线程共享使用。

## 总结

Redisson 是目前 Java 生态中最完善的 Redis 分布式锁解决方案，它解决了原生实现的大部分痛点：

- **Watch Dog** 解决锁续期问题
- **可重入** 满足递归调用场景
- **多种锁类型** 覆盖不同业务需求
- **API 友好** 与 ReentrantLock 设计一致

对于生产环境，强烈建议使用 Redisson 而不是手写分布式锁——你永远不知道还有哪些边界情况没有考虑到。
