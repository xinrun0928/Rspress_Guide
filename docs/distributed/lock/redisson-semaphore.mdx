# Redisson 信号量与 CountDownLatch

单机 Semaphore 用 AQS，分布式环境下怎么实现？Redisson 给了答案——还是用 Redis。

Semaphore（信号量）和 CountDownLatch（倒计时门闩）是 Java 并发编程中的重要工具。它们在单机环境下很好用，但到了分布式环境呢？

Redisson 提供了分布式版本的 Semaphore 和 CountDownLatch，让你在分布式环境中也能使用这些并发原语。

## 一、Semaphore 信号量

### 1.1 什么是信号量

Semaphore（信号量）用于控制同时访问资源的线程数量：

```
Semaphore permits = 3;

线程 A: acquire() → permits = 2 ✓
线程 B: acquire() → permits = 1 ✓
线程 C: acquire() → permits = 0 ✓
线程 D: acquire() → 阻塞，等待
线程 E: acquire() → 阻塞，等待

线程 A: release() → permits = 1
线程 D: acquire() → permits = 0 ✓
```

### 1.2 为什么需要分布式信号量

单机 Semaphore 只能在单 JVM 内控制并发。分布式环境下：
- 不同服务器的线程无法协调
- 需要跨进程的并发控制
- 比如：限制对某个接口的调用频率

### 1.3 Redisson Semaphore 实现

```java
// 获取分布式信号量
RSemaphore semaphore = redissonClient.getSemaphore("my-semaphore");
// 初始化信号量（许可数量）
semaphore.trySetPermits(10);
```

### 1.4 核心方法

```java
public interface RSemaphore extends RObject {

    /**
     * 获取一个许可，如果没有则阻塞
     */
    void acquire();

    /**
     * 获取指定数量的许可
     */
    void acquire(int permits);

    /**
     * 尝试获取许可（非阻塞）
     */
    boolean tryAcquire();

    /**
     * 尝试获取许可（带超时）
     */
    boolean tryAcquire(long waitTime, TimeUnit unit);

    /**
     * 释放许可
     */
    void release();

    /**
     * 释放指定数量的许可
     */
    void release(int permits);

    /**
     * 获取当前可用许可数量
     */
    int availablePermits();
}
```

## 二、Redis 中的实现

### 2.1 存储结构

Redisson Semaphore 使用 Redis String 存储许可计数器：

```
Semaphore key: my-semaphore
    │
    └── String: "10"  (许可数量)
```

### 2.2 获取许可 Lua 脚本

```lua
-- 获取许可
-- KEYS[1] = 信号量 key
-- ARGV[1] = 需要的许可数量

local current = tonumber(redis.call('get', KEYS[1]));
if current >= tonumber(ARGV[1]) then
    -- 许可足够，扣减
    redis.call('decrby', KEYS[1], ARGV[1]);
    return 1;
end;
-- 许可不足
return 0;
```

### 2.3 释放许可 Lua 脚本

```lua
-- 释放许可
-- KEYS[1] = 信号量 key
-- ARGV[1] = 释放的许可数量

redis.call('incrby', KEYS[1], ARGV[1]);
return 1;
```

## 三、使用示例

### 3.1 连接池限流

```java
@Service
public class ConnectionPoolService {

    private final RSemaphore semaphore;

    public ConnectionPoolService(RedissonClient redissonClient) {
        // 初始化信号量，允许 100 个并发连接
        this.semaphore = redissonClient.getSemaphore("connection-pool");
        this.semaphore.trySetPermits(100);
    }

    public Connection getConnection() {
        try {
            // 获取许可（阻塞直到获取）
            semaphore.acquire();

            // 创建连接
            return createConnection();

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException("获取连接被中断");
        }
    }

    public void releaseConnection(Connection conn) {
        try {
            // 关闭连接
            conn.close();
        } finally {
            // 释放许可
            semaphore.release();
        }
    }
}
```

### 3.2 接口限流

```java
@Service
public class RateLimitService {

    private final RSemaphore semaphore;

    public RateLimitService(RedissonClient redissonClient) {
        // 每秒允许 1000 次请求
        this.semaphore = redissonClient.getSemaphore("rate-limit:api");
        this.semaphore.trySetPermits(1000);
    }

    public boolean tryAcquire() {
        // 尝试获取许可，最多等待 0 秒
        return semaphore.tryAcquire(0, TimeUnit.MILLISECONDS);
    }

    public void release() {
        semaphore.release();
    }
}

// 使用拦截器实现限流
@Component
public class RateLimitInterceptor {

    @Autowired
    private RateLimitService rateLimitService;

    @Autowired
    private RedissonClient redissonClient;

    @Autowired
    private RSemaphore semaphore;

    @PostConstruct
    public void init() {
        // 每秒重置许可数量
        ScheduledExecutorService scheduler = Executors.newSingleThreadScheduledExecutor();
        scheduler.scheduleAtFixedRate(() -> {
            semaphore.trySetPermits(1000);
        }, 0, 1, TimeUnit.SECONDS);
    }

    public boolean preHandle() {
        return rateLimitService.tryAcquire();
    }
}
```

### 3.3 批量处理限流

```java
@Service
public class BatchProcessService {

    private final RSemaphore semaphore;

    public BatchProcessService(RedissonClient redissonClient) {
        // 限制同时只有 5 个任务在处理
        this.semaphore = redissonClient.getSemaphore("batch-process");
        this.semaphore.trySetPermits(5);
    }

    public void processBatch(List<Task> tasks) {
        for (Task task : tasks) {
            try {
                // 获取许可
                semaphore.acquire();

                // 异步处理
                CompletableFuture.runAsync(() -> {
                    try {
                        processTask(task);
                    } finally {
                        semaphore.release();
                    }
                });

            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }
}
```

## 四、CountDownLatch 倒计时门闩

### 4.1 什么是 CountDownLatch

CountDownLatch（倒计时门闩）用于等待一组线程完成：

```java
// 初始化为 3
CountDownLatch latch = new CountDownLatch(3);

Thread t1 = new Thread(() -> { doWork(); latch.countDown(); });
Thread t2 = new Thread(() -> { doWork(); latch.countDown(); });
Thread t3 = new Thread(() -> { doWork(); latch.countDown(); });

t1.start(); t2.start(); t3.start();

// 主线程等待
latch.await();  // 阻塞，直到 count 变为 0
// 继续执行
```

### 4.2 为什么需要分布式 CountDownLatch

单机 CountDownLatch 只能在单 JVM 内工作。分布式环境下：
- 需要等待多个服务节点完成
- 需要跨进程的同步
- 比如：等待所有数据加载完成后启动应用

### 4.3 Redisson CountDownLatch 实现

```java
// 获取分布式 CountDownLatch
RCountDownLatch latch = redissonClient.getCountDownLatch("my-latch");
// 初始化计数
latch.trySetCount(3);
```

### 4.4 核心方法

```java
public interface RCountDownLatch {

    /**
     * 等待计数归零
     */
    void await() throws InterruptedException;

    /**
     * 等待计数归零（带超时）
     */
    boolean await(long timeout, TimeUnit unit) throws InterruptedException;

    /**
     * 计数减 1
     */
    void countDown();

    /**
     * 获取当前计数
     */
    long getCount();
}
```

## 五、Redis 中的实现

### 5.1 存储结构

Redisson CountDownLatch 使用 Redis Key + TTL 实现：

```
Latch key: my-latch
    │
    └── String: "3"  (计数)
    │
    └── TTL: 无（直到计数归零才删除）
```

### 5.2 实现原理

```java
// await 实现
public void await() throws InterruptedException {
    long currentCount = getCount();
    while (currentCount > 0) {
        // 等待一段时间后重试
        // 使用 Redis 的发布订阅机制优化
        waitForCountZero();
        currentCount = getCount();
    }
}

// countDown 实现
public void countDown() {
    Long newCount = redisTemplate.opsForValue().decrement("my-latch");
    if (newCount <= 0) {
        // 计数归零，删除 key 并通知等待者
        redisTemplate.delete("my-latch");
        publishCountZeroEvent();
    }
}
```

## 六、使用示例

### 6.1 等待多服务启动完成

```java
@Service
public class ApplicationStartupService {

    private final RCountDownLatch latch;

    public ApplicationStartupService(RedissonClient redissonClient) {
        // 需要等待 3 个服务就绪
        this.latch = redissonClient.getCountDownLatch("app:startup:latch");
    }

    public void onServiceReady(String serviceName) {
        log.info("服务 {} 就绪", serviceName);
        latch.countDown();
    }

    public void waitForAllReady() {
        try {
            log.info("等待所有服务就绪...");
            latch.await(30, TimeUnit.SECONDS);
            log.info("所有服务已就绪，应用启动完成");
        } catch (InterruptedException e) {
            log.error("等待被打断");
            Thread.currentThread().interrupt();
        }
    }
}
```

### 6.2 分布式任务协调

```java
@Service
public class DistributedTaskCoordination {

    private final RCountDownLatch latch;

    public DistributedTaskCoordination(RedissonClient redissonClient) {
        this.latch = redissonClient.getCountDownLatch("task:coordinator");
    }

    // Worker 节点调用
    public void reportTaskComplete(String taskId, String workerId) {
        log.info("任务 {} 由 {} 完成", taskId, workerId);
        latch.countDown();
    }

    // 主节点等待所有 Worker 完成
    public void waitForTaskCompletion(String taskId, int workerCount) {
        // 初始化计数
        latch.trySetCount(workerCount);

        try {
            log.info("等待 {} 个 Worker 完成...", workerCount);
            boolean completed = latch.await(10, TimeUnit.MINUTES);
            if (completed) {
                log.info("所有 Worker 完成，任务 {} 成功", taskId);
            } else {
                log.warn("等待超时，任务 {} 未完成", taskId);
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 七、与单机版本的对比

| 维度 | Java 并发原语 | Redisson 分布式版本 |
|---|---|---|
| **作用域** | 单 JVM | 分布式 |

## 八、面试高频问题

**Q：分布式信号量如何实现？**

A：使用 Redis String 存储许可计数器。`acquire()` 时用 `decrby` 扣减计数，`release()` 时用 `incrby` 增加计数。原子性通过 Lua 脚本保证。

**Q：CountDownLatch 在 Redis 中如何实现？**

A：使用 Redis String 存储计数。`countDown()` 时用 `decr` 递减，`await()` 时轮询检查计数是否为 0。计数归零后删除 key。

**Q：什么场景需要分布式信号量？**

A：跨进程的并发控制，如 API 接口限流、数据库连接池管理、批量任务并发控制等。

**Q：什么场景需要分布式 CountDownLatch？**

A：跨进程的任务协调，如等待多个服务启动完成、等待分布式任务的所有 Worker 完成等。

## 总结

Redisson 的 Semaphore 和 CountDownLatch 为分布式环境提供了与单机环境一致的并发原语：

1. **Semaphore**：控制并发数量，用 Redis String 存储计数
2. **CountDownLatch**：等待一组操作完成，用 Redis Key + TTL 实现
3. **基于 Redis**：天然支持分布式和持久化

这些工具在服务协调、限流、批量处理等场景中非常有用。
