# 分布式锁：跨机器的互斥

你有没有想过这个问题：

单机环境下，`synchronized` 和 `ReentrantLock` 足够。

但如果是集群部署呢？多台机器同时抢一把锁，怎么保证互斥？

**单机锁解决的是进程内的问题，分布式锁解决的是跨机器的问题。**

## 分布式锁五大要求

```
┌─────────────────────────────────────────────────────────┐
│                   分布式锁要求图                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 互斥性                                              │
│     └─ 在任意时刻，只有一个客户端持有锁                  │
│                                                         │
│  2. 不死锁                                              │
│     └─ 客户端崩溃时，锁能被 TTL 自动释放                │
│                                                         │
│  3. 可重入（可选）                                      │
│     └─ 同一客户端可多次获取同一把锁                     │
│                                                         │
│  4. 公平性（可选）                                      │
│     └─ 按请求顺序分配锁（FIFO）                        │
│                                                         │
│  5. 高性能                                              │
│     └─ 获取/释放锁的操作必须快                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 实现方案对比

| 方案 | 性能 | 可靠性 | 实现复杂度 | 推荐场景 |
|------|------|--------|-----------|---------|
| Redis SETNX | 最高 | 中 | 中 | 高并发首选 |
| Redisson | 高 | 高 | 低 | 生产环境推荐 |
| ZooKeeper | 中 | 高 | 高 | 高可靠性场景 |
| MySQL | 低 | 高 | 低 | 备选过渡 |

## Redis 分布式锁最佳实践

使用 Redisson，它是目前最成熟的 Redis 分布式锁实现：

```
┌─────────────────────────────────────────────────────────┐
│                  Redisson 锁机制                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  获取锁                                                 │
│  └─ SET key value NX PX timeout                        │
│                                                         │
│  看门狗（Watchdog）                                     │
│  └─ 每 10 秒自动续期，防止业务未完成锁过期              │
│                                                         │
│  释放锁                                                 │
│  └─ Lua 脚本保证原子性：只有持有者才能释放              │
│                                                         │
│  可重入                                                 │
│  └─ Hash 结构存储：field=客户端ID, value=重入计数       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 核心公式

```
分布式锁 = 互斥（SETNX）+ TTL（不死锁）+ 看门狗（续期）+ 原子释放

方案选择：
- 高性能首选 → Redis + Redisson
- 高可靠性首选 → ZooKeeper
- 简单备选 → MySQL 唯一索引

避坑指南：
- 锁的 TTL 要大于业务执行时间
- 释放锁要验证持有者身份
- 集群部署要考虑 RedLock 防主从切换丢锁
```
