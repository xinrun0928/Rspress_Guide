# 分布式锁

凌晨两点，你正盯着服务器监控屏幕，忽然发现一个诡异的现象：两台机器上的定时任务同时触发了，它们同时去查询库存、同时去扣减库存、同时生成了订单。你揉了揉眼睛，以为自己看错了——但数据不会骗人，同一个商品被超卖了三次。

这不是孤例。在分布式系统里，没有一把「锁」来协调多台机器的访问，类似的问题几乎每天都在发生。单机环境下的 `synchronized` 或 `ReentrantLock` 在多节点场景下完全失效，你需要的是**分布式锁**。

## 什么是分布式锁

分布式锁是一种跨 JVM 的同步机制，用于在分布式环境中协调多个进程对共享资源的访问。它的核心思想与单机锁一致：**同一时刻只有一个持有者能访问临界资源**。

在单机环境下，锁的实现可以依赖操作系统提供的底层原语（如 mutex、semaphore）。但到了分布式环境，不同机器上的进程无法直接共享这些底层原语，必须借助外部的「协调服务」来实现锁语义。

常见的分布式锁载体包括：

| 载体 | 特点 | 适用场景 |
|---|---|---|
| Redis | 性能极高，支持过期时间 | 追求性能的业务场景 |
| ZooKeeper | 强一致性，有临时节点和监听机制 | 对可靠性要求极高的场景 |
| etcd | Raft 协议保证一致性 | Kubernetes 等云原生场景 |
| MySQL | 兼容性好，依赖数据库 | 团队已有数据库基础设施 |

## 分布式锁应满足的条件

一个合格的分布式锁必须满足以下五个核心条件，少了任何一个都可能引发严重问题。

### 互斥性

互斥性是锁的最基本要求：在任意时刻，只能有一个客户端持有锁。

```java
// 理想情况：只有一个客户端能获取到锁
if (redis.set(key, value, "NX", "PX", 30000)) {
    // 获取锁成功，执行临界区代码
} else {
    // 获取锁失败
}
```

### 可重入性

同一线程获取锁之后，可以重复获取该锁而不会死锁。`Redisson` 通过 `threadLocal` 记录持有次数来实现可重入。

### 锁超时

锁必须有过期时间，防止持有锁的客户端崩溃后锁无法释放，导致死锁。超时时间需要根据业务执行时间合理设置。

:::warning
如果业务执行时间超过锁超时时间，可能导致两个问题：一是锁被自动释放后其他客户端进入临界区，二是原客户端执行完成后释放了别人的锁。
:::

### 非阻塞获取

支持尝试获取锁，立即返回结果，而不是一直阻塞等待。

### 公平性（可选）

公平锁按照客户端请求的先后顺序依次获取锁，非公平锁则允许插队。Redis 原生实现是非公平的，`Redisson` 提供了公平锁的实现。

## 分布式锁的常见问题

在实际使用中，分布式锁面临几个经典挑战：

### 锁续期问题

如果持有锁的客户端执行时间过长，可能在锁过期后被其他客户端抢走。`Redisson` 的 `Watch Dog` 机制会在后台自动续期锁。

### 主从切换问题

如果 Redis 采用主从架构，主节点宕机后从节点晋升，此时之前设置的锁可能丢失。`Redisson` 通过红锁（RedLock）来解决这个问题。

### 时钟漂移问题

多台机器的时钟可能存在误差，导致锁过期时间的计算出现偏差。

### 原子性问题

获取锁和设置过期时间必须是原子操作，否则可能出现获取锁成功但设置过期时间失败的情况。

```java
// 非原子操作，存在问题
if (redis.setnx(key, value)) {
    redis.expire(key, 30); // 如果这里失败，锁永远不会过期
}

// 原子操作，Redis SET NX PX 就是原子的
redis.set(key, value, "NX", "PX", 30000);
```

## 分布式锁 vs 单机锁

| 维度 | 单机锁 | 分布式锁 |
|---|---|---|
| 作用范围 | 单 JVM 进程 | 多台机器、多个进程 |
| 实现方式 | 操作系统原语 | 外部协调服务（Redis/ZK/etcd） |
| 性能 | 极高，无网络开销 | 有网络延迟 |
| 可靠性 | 依赖 JVM | 依赖协调服务的可用性 |
| 适用场景 | 单机应用 | 微服务、分布式系统 |

## 总结

分布式锁是分布式系统中不可或缺的基础设施。选择哪种实现，取决于你的业务场景：对性能要求高选 Redis，对可靠性要求极高选 ZooKeeper 或 etcd。

但无论选择哪种实现，都要记住：**分布式锁不是万能的**。它只能保证互斥性，无法解决所有并发问题。过度依赖分布式锁可能导致系统性能下降，甚至引入新的问题。
