# Rspress 特殊字符转义规范

> 本文件是 CLAUDE.md 中转义规则的详细说明文档。
> 目的：彻底解决 Rspress/MDX 渲染特殊字符时的异常问题。

## 为什么需要转义

Rspress 基于 MDX，MDX 中某些字符有特殊语义，直接书写会导致渲染异常或解析错误。常见问题：

- `<` 和 `>` 与 JSX 标签语法冲突
- `{` 和 `}` 与 MDX 表达式语法冲突
- 泛型尖括号 `<T>` `<K, V>` 在某些配置下被误解析为 JSX
- Markdown 元字符在表格、列表中可能破坏格式

## 转义规则

### 1. 运算符（正文/列表/表格）

所有运算符前后加反引号：

| 运算符 | 转义写法 |
|---|---|
| `<=` | `` `<=` `` |
| `>=` | `` `>=` `` |
| `==` | `` `==` `` |
| `!=` | `` `!=` `` |
| `&&` | `` `&&` `` |
| `\|\|` | `` `\|\|` `` |
| `=>` | `` `=>` `` |
| `->` | `` `->` `` |
| `<-` | `` `<-` `` |
| `<<` `>>` `>>>` | `` `<<` `` `` `>>` `` `` `>>>` `` |
| `&` `\|` `^` `~` | `` `&` `` `` `\|` `` `` `^` `` `` `~` `` |

### 2. 泛型尖括号（正文/列表/表格）

```markdown
// ✅ 正确 — 泛型用反引号包裹
`List<T>`
`Map<K, V>`
`HashMap<K, V>`
`ConcurrentHashMap<K, V>`
`CompletableFuture<T>`
`Class<?>`

// ❌ 错误 — 未转义会导致 MDX 解析问题
List<T> Map<K, V>
```

### 3. Markdown 元字符

| 字符 | 转义写法 |
|---|---|
| `$` | `` `$` `` |
| `*` | `` `*` `` |
| `_` | `` `_` `` |
| `` ` `` | `` ` `` （用 `` ` `` 包裹内敛号） |
| `#` | `` `#` `` （仅在非标题语境中） |

### 4. 特殊符号

| 符号 | 场景 | 转义写法 |
|---|---|---|
| `<` `>` | 正文/表格 | `` `<` `` `` `>` `` |
| `{}` | 正文/表格（非代码块） | `` `{` `` `` `}` `` |
| `[]` | 歧义场景 | `` `[` `` `` `]` `` |

### 5. 不需要转义的场景

- **代码块内**（` ``` ` 包裹的代码）：运算符和泛型不需要转义，MDX 不解析代码块内容
- **标题**（`#` `##` 后）：标题中的运算符和泛型不需要额外转义
- **链接文本**：`[text](/path)` 中的 text 部分无需转义
- **图片**：`![alt](url)` 无需转义

## 快速检查清单

生成内容后，逐项核对：

- [ ] `<=` `>=` `==` `!=` 都用了反引号
- [ ] `&&` `||` 都用了反引号
- [ ] `List<T>` `Map<K, V>` 等泛型都用了反引号
- [ ] `=>` `->` 等箭头符号都用了反引号
- [ ] 位运算符 `&` `|` `^` `~` 都用了反引号
- [ ] `` `$` `` 在非代码块场景用了反引号
- [ ] 代码块内的泛型和运算符没有额外转义
